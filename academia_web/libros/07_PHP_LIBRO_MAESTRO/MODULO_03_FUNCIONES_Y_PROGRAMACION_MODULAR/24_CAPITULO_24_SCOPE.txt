===============================================================================
1. QUÉ ES EL SCOPE
===============================================================================

El scope (ámbito) define en qué parte del programa una variable es
accesible. PHP tiene dos alcances principales para variables:

  • **Scope global**: variables declaradas fuera de cualquier función.
  • **Scope local**: variables declaradas dentro de una función.

```php
<?php

$nombre = "Global"; // Variable GLOBAL

function mostrar_nombre(): void
{
    $nombre = "Local"; // Variable LOCAL (diferente)
    echo $nombre; // "Local"
}

mostrar_nombre();   // "Local"
echo $nombre;       // "Global"
```

Principio fundamental: las variables definidas dentro de una función
NO son accesibles desde fuera, y las variables globales NO son
accesibles desde dentro de una función (sin ayuda explícita).

```php
<?php

$mensaje = "Hola desde fuera";

function probar(): void
{
    // Esto NO accede a $mensaje global
    // Crea una variable local $mensaje (o da error si está declarada)
    echo $mensaje; // Warning: Undefined variable $mensaje
}

probar();
```

===============================================================================
2. SCOPE LOCAL
===============================================================================

El scope local es el interior de una función. Las variables declaradas
allí solo existen mientras la función se ejecuta.

```php
<?php

function calcular(): void
{
    $x = 10; // Variable local
    echo $x; // 10
}

calcular();
echo $x; // Undefined variable $x (fuera del scope)
```

Cada invocación de función crea un NUEVO scope local. Las variables
de invocaciones anteriores no persisten (a menos que sean estáticas).

```php
<?php

function incrementar(): void
{
    $contador = 0;
    $contador++;
    echo $contador;
}

incrementar(); // 1
incrementar(); // 1 (nuevo scope, $contador se reinicia)
incrementar(); // 1
```

Los parámetros de función también pertenecen al scope local:

```php
<?php

function duplicar(int $numero): int
{
    // $numero es local a esta función
    $resultado = $numero * 2;
    return $resultado;
}

echo duplicar(5); // 10
echo $numero; // Undefined variable
```

===============================================================================
3. SCOPE GLOBAL
===============================================================================

El scope global es todo lo que está fuera de las funciones. Las
variables globales existen desde que se declaran hasta que el script
termina.

```php
<?php

$app_name = "PHP Master Book";  // Global
$version = "3.0.0";             // Global
$autor = "Lucas EGA Academy";   // Global

function mostrar_info(): void
{
    // No podemos acceder directamente a las globales
    // echo $app_name; // Undefined variable
}

// Pero podemos acceder a ellas fuera de funciones
echo $app_name; // PHP Master Book
echo $version;  // 3.0.0
```

Principio importante: el scope global de PHP es especial porque las
variables globales no están disponibles dentro de funciones sin
declaración explícita.

===============================================================================
4. LA PALABRA CLAVE `global`
===============================================================================

Para acceder a una variable global DENTRO de una función, usamos la
palabra `global`:

```php
<?php

$contador_global = 0;

function incrementar_contador(): void
{
    global $contador_global;
    $contador_global++;
}

incrementar_contador();
incrementar_contador();
echo $contador_global; // 2
```

Puedes declarar múltiples globales en una sola línea:

```php
<?php

$db_host = "localhost";
$db_user = "root";
$db_pass = "secret";

function conectar_bd(): PDO
{
    global $db_host, $db_user, $db_pass;

    $dsn = "mysql:host=$db_host";
    return new PDO($dsn, $db_user, $db_pass);
}
```

También puedes usar `global` con arrays para modificar elementos:

```php
<?php

$config = [
    'debug' => false,
    'db' => ['host' => 'localhost'],
];

function habilitar_debug(): void
{
    global $config;
    $config['debug'] = true;
}

habilitar_debug();
echo $config['debug'] ? "true" : "false"; // true
```

Advertencia: `global` crea una referencia a la variable global. Si la
modificas dentro de la función, la variable global se modifica.

===============================================================================
5. PROBLEMAS DE LAS VARIABLES GLOBALES
===============================================================================

Aunque `global` es útil, su uso excesivo trae problemas graves:

Problema 1: Acoplamiento oculto

```php
<?php

$impuesto = 0.16;

function calcular_total(float $subtotal): float
{
    global $impuesto;
    return $subtotal * (1 + $impuesto);
}

// ¿De dónde viene $impuesto? Quien lee la función no lo sabe
// sin buscar en el código. La firma no revela la dependencia.
```

Problema 2: Efectos secundarios inesperados

```php
<?php

$cache = [];

function obtener_datos(string $clave): ?array
{
    global $cache;

    if (isset($cache[$clave])) {
        return $cache[$clave];
    }

    // Simular consulta costosa
    $cache[$clave] = ['dato' => "Valor de $clave"];
    return $cache[$clave];
}

// La función modifica $cache como efecto secundario
// Si otra función también usa $cache, pueden haber conflictos
```

Problema 3: Dificultad para testing

```php
<?php

$db_conexion = null;

function obtener_usuarios(): array
{
    global $db_conexion;

    if ($db_conexion === null) {
        $db_conexion = new PDO("mysql:host=localhost", "root", "");
    }

    return $db_conexion->query("SELECT * FROM usuarios")->fetchAll();
}

// Para probar esta función necesitas configurar $db_conexion
// y una base de datos real. No puedes simplemente pasar un mock.
```

Alternativas a las globales:

  1. Pasar como parámetro (inyección de dependencias).
  2. Usar clases y propiedades estáticas.
  3. Usar contenedores de servicios (DI Container).
  4. Usar patrones como Singleton (con cuidado).

```php
<?php

// ✅ Mejor: inyección de dependencias
function obtener_usuarios(PDO $db): array
{
    return $db->query("SELECT * FROM usuarios")->fetchAll();
}

// Más fácil de probar: pasas un PDO mockeado
```

===============================================================================
6. EL ARRAY SUPER GLOBAL $GLOBALS
===============================================================================

`$GLOBALS` es un array superglobal que contiene TODAS las variables
globales del script. Las claves son los nombres de las variables.

```php
<?php

$nombre = "Ana";
$edad = 25;
$pais = "México";

function mostrar_globales(): void
{
    echo $GLOBALS['nombre']; // Ana
    echo $GLOBALS['edad'];   // 25
}

mostrar_globales();
```

Modificar variables a través de `$GLOBALS`:

```php
<?php

$contador = 0;

function aumentar(): void
{
    $GLOBALS['contador']++;
}

aumentar();
aumentar();
echo $contador; // 2
```

`$GLOBALS` vs `global`:

  • `global $var;` — crea una referencia local a la variable global.
  • `$GLOBALS['var']` — accede directamente al array de globales.

Ambos logran lo mismo, pero `$GLOBALS` hace más explícito que estamos
accediendo al ámbito global. Sin embargo, ambos deben usarse con
moderación.

```php
<?php

$x = 10;

function ejemplo(): void
{
    // Estas dos líneas son equivalentes:
    global $x;
    echo $x; // 10

    echo $GLOBALS['x']; // 10
}
```

Listado de superglobales en PHP:

  • `$GLOBALS` — todas las variables globales.
  • `$_SERVER` — información del servidor y entorno.
  • `$_GET` — parámetros de la URL.
  • `$_POST` — datos de formularios POST.
  • `$_FILES` — archivos subidos.
  • `$_COOKIE` — cookies HTTP.
  • `$_SESSION` — datos de sesión.
  • `$_REQUEST` — combinación de GET, POST y COOKIE.
  • `$_ENV` — variables de entorno.

Las superglobales están disponibles en TODOS los ámbitos, incluso
dentro de funciones, sin necesidad de `global`.

```php
<?php

function detectar_navegador(): string
{
    return $_SERVER['HTTP_USER_AGENT'] ?? 'Desconocido';
}

function obtener_parametro_get(string $clave): ?string
{
    return $_GET[$clave] ?? null;
}
```

===============================================================================
7. VARIABLES ESTÁTICAS
===============================================================================

Las variables estáticas se declaran dentro de una función con la
palabra `static`. A diferencia de las locales normales, las estáticas
**recuerdan su valor** entre invocaciones.

```php
<?php

function contador(): int
{
    static $contador = 0;
    $contador++;
    return $contador;
}

echo contador(); // 1
echo contador(); // 2
echo contador(); // 3
```

Características de las variables estáticas:

  • Se inicializan solo UNA VEZ (en la primera invocación).
  • PERSISTEN entre llamadas a la función.
  • Son locales a la función (no accesibles desde fuera).
  • Útiles para memorización (caching) y contadores.

```php
<?php

function obtener_numero_pedido(): string
{
    static $secuencia = 0;
    $secuencia++;
    return "PED-" . str_pad($secuencia, 6, "0", STR_PAD_LEFT);
}

echo obtener_numero_pedido(); // PED-000001
echo obtener_numero_pedido(); // PED-000002
echo obtener_numero_pedido(); // PED-000003
```

Memorización con static:

```php
<?php

function factorial(int $n): int
{
    static $cache = [];

    if ($n < 0) {
        throw new InvalidArgumentException("Número negativo");
    }

    if (isset($cache[$n])) {
        return $cache[$n];
    }

    if ($n <= 1) {
        $cache[$n] = 1;
        return 1;
    }

    $cache[$n] = $n * factorial($n - 1);
    return $cache[$n];
}

echo factorial(10); // 3628800 (calcula)
echo factorial(10); // 3628800 (desde cache, más rápido)
print_r($GLOBALS); // No incluye $cache porque es local estática
```

Limitaciones de static:

  • No se pueden compartir entre funciones (cada función tiene su
    propia variable static aunque se llame igual).
  • No son accesibles desde fuera de la función.
  • La inicialización solo acepta expresiones constantes simples.

```php
<?php

function ejemplo_static(): void
{
    // ✅ Correcto
    static $a = 0;
    static $b = null;
    static $c = 'texto';

    // ❌ Incorrecto (PHP < 8.1): expresión no constante
    // static $d = date('Y');
    // static $e = new DateTime();

    // PHP 8.1+ permite inicialización dinámica con static
    static $f = rand(1, 100); // Válido en PHP 8.1+
}
```

===============================================================================
8. COMPARACIÓN DE ÁMBITOS
===============================================================================

| Característica       | Local         | Global        | Estática      | Superglobal |
|---------------------|---------------|---------------|---------------|-------------|
| Visibilidad         | Solo función  | Todo el script| Solo función  | Todo        |
| Persistencia        | Una llamada   | Todo el script| Entre llamadas| Todo el script|
| Declaración         | `$var`        | Fuera de fn   | `static $var` | Automática  |
| Acceso desde fn     | Directo       | `global`/`$GLOBALS`| Directo  | Directo     |
| Riesgo de colisión  | Bajo          | Alto          | Medio         | Medio       |

```php
<?php

$global = "Soy global";

function demostracion(): void
{
    $local = "Soy local";
    static $estatica = 0;
    $estatica++;

    echo $local;           // "Soy local"
    echo $GLOBALS['global']; // "Soy global"
    echo $estatica;         // 1, 2, 3...
    echo $_SERVER['PHP_SELF']; // Superglobal
}

demostracion();
echo $global;    // "Soy global"
echo $local;     // Undefined
// echo $estatica; // Undefined
```

9. EJEMPLO COMPLETO: SISTEMA DE LOGGING
===============================================================================

```php
<?php

declare(strict_types=1);

// Configuración global (solo lectura, no mutable)
$config_log = [
    'archivo' => 'app.log',
    'nivel_minimo' => 'INFO',
    'formato' => '[{fecha}] [{nivel}] {mensaje}',
];

$niveles = [
    'DEBUG' => 0,
    'INFO' => 1,
    'WARNING' => 2,
    'ERROR' => 3,
];

/**
 * Registra un mensaje en el log.
 */
function loggear(string $mensaje, string $nivel = 'INFO'): void
{
    global $config_log, $niveles;

    if (!isset($niveles[$nivel])) {
        throw new InvalidArgumentException("Nivel '$nivel' no válido");
    }

    if ($niveles[$nivel] < $niveles[$config_log['nivel_minimo']]) {
        return; // No registrar si es menos prioritario
    }

    $formato = str_replace(
        ['{fecha}', '{nivel}', '{mensaje}'],
        [date('Y-m-d H:i:s'), $nivel, $mensaje],
        $config_log['formato']
    );

    file_put_contents($config_log['archivo'], $formato . PHP_EOL, FILE_APPEND);
}

/**
 * Lleva un contador de eventos de log por nivel.
 */
function contar_evento(string $nivel): array
{
    static $conteo = [
        'DEBUG' => 0,
        'INFO' => 0,
        'WARNING' => 0,
        'ERROR' => 0,
    ];

    if (isset($conteo[$nivel])) {
        $conteo[$nivel]++;
    }

    return $conteo;
}

/**
 * Obtiene el resumen de eventos (usa estática para persistencia).
 */
function obtener_resumen_eventos(): string
{
    $conteo = contar_evento(''); // Solo para obtener el array actual
    // Necesitamos acceder al static de contar_evento
    // Para este ejemplo, usamos una variable estática local
    static $resumen_anterior = '';

    return implode(', ', array_map(
        fn(string $n, int $c) => "$n: $c",
        $conteo,
        $conteo
    ));
}

/**
 * Genera un ID único basado en timestamp (usa static).
 */
function generar_id_unico(): string
{
    static $contador = 0;
    $contador++;
    return date('Ymd') . '-' . str_pad($contador, 4, '0', STR_PAD_LEFT);
}

// --- USO ---
loggear("Aplicación iniciada", "INFO");
loggear("Usuario 'admin' autenticado", "INFO");
loggear("Intento de acceso a ruta restringida", "WARNING");
loggear("Error de conexión a base de datos", "ERROR");

echo "IDs generados:\n";
echo generar_id_unico() . "\n";
echo generar_id_unico() . "\n";
echo generar_id_unico() . "\n";

$eventos = contar_evento('INFO');
$eventos = contar_evento('ERROR');
echo "Resumen de eventos: " . obtener_resumen_eventos() . "\n";
```

===============================================================================
10. BUENAS PRÁCTICAS DE SCOPE
===============================================================================

  • Prefiere variables locales sobre globales.
  • Si necesitas estado compartido, considera inyección de dependencias.
  • Usa `global` solo en scripts pequeños o prototipos.
  • En aplicaciones profesionales, evita completamente `$GLOBALS`.
  • Usa variables estáticas para memorización interna de funciones.
  • Documenta claramente cuando una función depende de estado global.
  • Las superglobales (`$_GET`, `$_POST`, etc.) son aceptables para
    entrada de datos, pero no las modifiques.
  • Considera usar clases y namespaces en lugar de variables globales.

===============================================================================
ERRORES COMUNES CON SCOPE
===============================================================================

Error 1: Asumir que una variable global es accesible dentro de una
función:

```php
<?php

$descuento = 0.10;

function calcular_precio(float $precio): float
{
    // ❌ $descuento no está disponible aquí
    return $precio * (1 - $descuento);
}
```

Error 2: Modificar globales sin darte cuenta:

```php
<?php

$config = ['debug' => false];

function probar(): void
{
    global $config;
    $config = "accidental"; // ❌ Sobrescribe el array global
}
```

Error 3: Confiar en que una variable estática se reinicia:

```php
<?php

function probar(): void
{
    static $contador = 0;
    $contador++;
    return $contador;
}

echo probar(); // 1
echo probar(); // 2 (no se reinicia)
```

Error 4: Usar `global` cuando el parámetro es más claro:

```php
<?php

// ❌ Oscuro — dependencia oculta
function conectar(): PDO
{
    global $host, $user, $pass;
    return new PDO("mysql:host=$host", $user, $pass);
}

// ✅ Claro — dependencia explícita
function conectar(string $host, string $user, string $pass): PDO
{
    return new PDO("mysql:host=$host", $user, $pass);
}
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- El scope define dónde es visible una variable en el programa.
- Las variables locales solo existen dentro de la función que las declara.
- Las variables globales existen fuera de las funciones y no son
  accesibles dentro de ellas sin `global` o `$GLOBALS`.
- `$GLOBALS` es un array superglobal que contiene todas las variables
  globales del script.
- Las variables estáticas (`static`) persisten entre llamadas a la
  función pero son locales a ella.
- El uso excesivo de globales dificulta el testing y el mantenimiento.
- Las superglobales (`$_SERVER`, `$_GET`, etc.) están disponibles en
  todos los ámbitos sin declaración explícita.
- La inyección de dependencias es la alternativa profesional al uso
  de variables globales.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo usar `global` y `static` juntos?**

No. `static` mantiene el valor entre llamadas dentro de la función
pero no hace que la variable sea global. Son conceptos separados.

❓ **¿Las variables de clase (propiedades) siguen las mismas reglas?**

No. Las propiedades de clase usan `public`, `protected` y `private`.
El scope en POO es diferente al de las funciones.

❓ **¿Es malo usar `$GLOBALS`?**

No es malo intrínsecamente, pero su uso excesivo indica un diseño
que puede mejorarse. En scripts pequeños es aceptable; en aplicaciones
grandes, evítalo.

❓ **¿Qué pasa si declaro `global $var` y no existe?**

PHP crea la variable global con valor `null`. Es buena práctica
verificar que existe antes de usarla.

❓ **¿Las funciones internas de PHP usan globales?**

Algunas funciones internas usan configuraciones globales (como
`date_default_timezone_set`), pero en general el núcleo de PHP
está bien diseñado para evitar contaminación del scope.

DESAFÍO DEL ARQUITECTO
===============================================================================

🏗️ **Sistema de Caché Multinivel con Scope Control**

Diseña un sistema de caché que opere en diferentes niveles de scope:

  • `cache_local(string $clave, callable $generador): mixed` — caché
    que usa variable estática. Los datos persisten solo durante la
    ejecución del script.

  • `cache_global(string $clave, callable $generador): mixed` — caché
    que usa una variable global compartida. Los datos persisten y son
    accesibles desde cualquier función.

  • `cache_con_ttl(string $clave, callable $generador, int $ttl_segundos): mixed`
    — caché global que invalida entradas después de TTL segundos
    (usa time() para comparar).

  • `limpiar_cache(?string $nivel = null): void` — limpia la caché
    local, global o ambas según el parámetro.

  • `estadisticas_cache(): array` — devuelve hits, misses, tamaño y
    tiempo promedio de generación.

  • `cache_operacion_costosa(int $n): int` — calcula el n-ésimo número
    de Fibonacci usando cache_local para optimizar llamadas recursivas.

Requisitos:
  - Usa strict_types y tipos de retorno.
  - Implementa una variable global `$_CACHE` para la caché global.
  - Implementa variables estáticas para la caché local.
  - La función `cache_con_ttl` debe limpiar automáticamente entradas
    expiradas al acceder.
  - Documenta con PHPDoc completo.
  - Incluye un ejemplo que calcule Fibonacci(40) con y sin caché,
    mostrando la diferencia de rendimiento.

Este desafío integra scope local, global y estático en un sistema
práctico y útil para aplicaciones reales.
