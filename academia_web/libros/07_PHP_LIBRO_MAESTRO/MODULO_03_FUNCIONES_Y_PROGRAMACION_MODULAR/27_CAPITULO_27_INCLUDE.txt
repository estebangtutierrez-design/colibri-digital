===============================================================================
1. QUÉ ES INCLUDE
===============================================================================

`include` es una directiva de PHP que inserta el contenido de un
archivo en el punto donde se invoca. Es como si copiaras y pegaras
el contenido del archivo incluido en esa posición.

```php
<?php

// archivo: config.php
$db_host = 'localhost';
$db_user = 'root';
$db_pass = '';

// archivo: index.php
echo "Antes del include\n";
include 'config.php';
echo "Después del include\n";

// Ahora tenemos acceso a las variables definidas en config.php
echo "Conectando a $db_host...\n";
```

El archivo incluido se evalúa como código PHP. Si tiene etiquetas
`<?php`, se ejecuta como PHP. Si es HTML puro, se envía al navegador.

```php
<?php

// archivo: encabezado.html
<!DOCTYPE html>
<html>
<head><title>Mi Sitio</title></head>
<body>

<?php

// archivo: index.php
include 'encabezado.html';
echo "<h1>Bienvenido</h1>";
include 'pie.html';
```

===============================================================================
2. COMPORTAMIENTO DE INCLUDE
===============================================================================

Comportamiento clave de `include`:

  1. **Evalúa el archivo incluido en el scope actual**: las variables,
     funciones y clases definidas en el archivo incluido están
     disponibles en el mismo ámbito desde el que se incluyó.

  2. **Genera SOLO una advertencia (warning)** si el archivo no existe.
     El script continúa ejecutándose.

  3. **Puede estar en cualquier expresión**: al ser una construcción
     del lenguaje, `include` puede usarse en condicionales, bucles, etc.

```php
<?php

// include en condicional
if ($modo === 'produccion') {
    include 'config.prod.php';
} else {
    include 'config.dev.php';
}

// include en bucle (raro, pero posible)
foreach ($modulos as $modulo) {
    include "modulos/$modulo.php";
}

// include como expresión (devuelve 1 en éxito, false en error)
$resultado = include 'archivo.php';
var_dump($resultado); // 1 (int) si tuvo éxito
```

Scope compartido:

```php
<?php

// archivo: mensaje.php
$texto = "Hola desde el archivo incluido";
function saludar(): string {
    return "¡Saludos!";
}

// archivo: principal.php
function probar_include(): void {
    include 'mensaje.php';
    echo $texto;        // "Hola desde el archivo incluido"
    echo saludar();     // "¡Saludos!"
}

probar_include();
// $texto NO está disponible aquí (scope local de probar_include)
// echo $texto; // Undefined
```

===============================================================================
3. RUTAS DE BÚSQUEDA DE ARCHIVOS
===============================================================================

PHP busca los archivos incluidos en el siguiente orden:

  1. La ruta especificada relativa al directorio del script actual.
  2. La ruta especificada relativa al `include_path` configurado en
     php.ini.
  3. El directorio del script actual (__DIR__).

```php
<?php

// Ruta relativa al directorio actual
include 'archivo.php';

// Ruta relativa con subdirectorio
include 'includes/funciones.php';

// Ruta absoluta (menos portátil, más explícita)
include '/var/www/html/config.php';

// Usando __DIR__ para rutas absolutas confiables
include __DIR__ . '/../includes/funciones.php';
```

Configurar `include_path` en php.ini:

```ini
; php.ini
include_path = ".:/usr/share/php:/var/www/lib"
```

O desde PHP:

```php
<?php

set_include_path('/ruta/a/mis/libs' . PATH_SEPARATOR . get_include_path());
include 'libreria.php'; // Busca primero en la nueva ruta
```

Buenas prácticas con rutas:

```php
<?php

// Usar __DIR__ para rutas absolutas basadas en la ubicación del archivo
define('ROOT_DIR', __DIR__);
define('INC_DIR', ROOT_DIR . '/includes');

// Incluir usando rutas absolutas
include INC_DIR . '/funciones.php';
include INC_DIR . '/conexion.php';
```

===============================================================================
4. INCLUDE ONCE
===============================================================================

`include_once` se comporta como `include`, pero verifica si el archivo
ya fue incluido. Si es así, NO lo incluye nuevamente.

```php
<?php

// archivo: funciones.php
function sumar(int $a, int $b): int {
    return $a + $b;
}

// archivo: main.php
include_once 'funciones.php'; // Se incluye
include_once 'funciones.php'; // No se incluye de nuevo (evita error)
include_once 'funciones.php'; // No se incluye

echo sumar(5, 3); // 8
```

¿Por qué usar `include_once`?

  1. **Evita redefinir funciones/clases**: si incluyes dos veces un
     archivo con funciones, PHP lanza error fatal.

  2. **Previene conflictos** en proyectos grandes donde múltiples
     archivos pueden incluir el mismo módulo.

  3. **Mejora el rendimiento** (evita leer y evaluar el mismo archivo).

```php
<?php

// SIN include_once — esto causaría error
include 'funciones.php'; // define sumar()
include 'funciones.php'; // ❌ Fatal error: Cannot redeclare sumar()

// CON include_once — seguro
include_once 'funciones.php'; // define sumar()
include_once 'funciones.php'; // No hace nada (ya incluido)
```

Sin embargo, `include_once` tiene un pequeño costo de rendimiento
porque PHP debe mantener un registro de los archivos ya incluidos.

===============================================================================
5. INCLUDE VS INCLUDE ONCE
===============================================================================

| Característica        | include              | include_once         |
|-----------------------|----------------------|----------------------|
| Incluye el archivo    | Siempre              | Solo si no fue incluido|
| Error si no existe    | Warning (continúa)   | Warning (continúa)   |
| Multiple inclusiones  | Sí (puede causar error)| No (seguro)         |
| Rendimiento           | Más rápido           | Mínimo overhead      |
| Uso recomendado       | HTML, plantillas     | Funciones, clases    |

Regla práctica:

  • Usa `include` para plantillas HTML, fragmentos de vista, contenido
    que puede repetirse.
  • Usa `include_once` para archivos con funciones, clases,
    configuraciones y definiciones que solo deben cargarse una vez.

```php
<?php

// include: plantillas que pueden repetirse
foreach ($articulos as $articulo) {
    include 'plantillas/tarjeta_articulo.php';
}

// include_once: definiciones que solo deben cargarse una vez
include_once 'includes/database.php';
include_once 'includes/validacion.php';
include_once 'includes/autenticacion.php';
```

===============================================================================
6. MANEJO DE ERRORES CON INCLUDE
===============================================================================

Si el archivo no existe, `include` emite un **warning** pero el script
continúa. `include_once` se comporta igual.

```php
<?php

// Esto solo genera un warning, el script continúa
include 'archivo_que_no_existe.php';
echo "El script sigue ejecutándose\n";
```

Puedes suprimir el warning con `@` (no recomendado):

```php
<?php

// Supresión de error (generalmente mala práctica)
$resultado = @include 'archivo_inexistente.php';

if ($resultado === false) {
    echo "No se pudo incluir el archivo\n";
}
```

Mejor: verificar que el archivo existe antes de incluirlo:

```php
<?php

$archivo = 'config.php';

if (file_exists($archivo)) {
    include $archivo;
} else {
    // Loggear el error o usar valores por defecto
    $db_host = 'localhost';
    $db_user = 'root';
    $db_pass = '';
    error_log("Advertencia: $archivo no encontrado, usando defaults");
}
```

Para control más fino, puedes usar `is_readable`:

```php
<?php

$archivo = __DIR__ . '/includes/secret.php';

if (is_readable($archivo)) {
    include $archivo;
} else {
    throw new RuntimeException("No se puede leer: $archivo");
}
```

===============================================================================
7. EJEMPLO PRÁCTICO: ORGANIZACIÓN CON INCLUDE
===============================================================================

```php
<?php

// ============================================
// archivo: includes/config.php
// ============================================
<?php

define('APP_NAME', 'Mi Aplicación');
define('APP_VERSION', '1.0.0');

$db_config = [
    'host' => 'localhost',
    'port' => 3306,
    'dbname' => 'mi_app',
    'user' => 'root',
    'pass' => '',
];

$app_config = [
    'debug' => true,
    'timezone' => 'America/Mexico_City',
    'locale' => 'es_MX',
];

// ============================================
// archivo: includes/funciones.php
// ============================================
<?php

function conectar_bd(): PDO
{
    global $db_config;
    $dsn = "mysql:host={$db_config['host']};port={$db_config['port']};dbname={$db_config['dbname']}";
    return new PDO($dsn, $db_config['user'], $db_config['pass']);
}

function obtener_usuario(int $id): ?array
{
    $db = conectar_bd();
    $stmt = $db->prepare("SELECT * FROM usuarios WHERE id = ?");
    $stmt->execute([$id]);
    $usuario = $stmt->fetch(PDO::FETCH_ASSOC);
    return $usuario ?: null;
}

function formatear_fecha(string $fecha): string
{
    return date('d/m/Y H:i', strtotime($fecha));
}

// ============================================
// archivo: index.php (punto de entrada)
// ============================================
<?php

// Cargar configuración y funciones
include_once __DIR__ . '/includes/config.php';
include_once __DIR__ . '/includes/funciones.php';

// Configurar zona horaria
date_default_timezone_set($app_config['timezone']);

// Cabecera
include __DIR__ . '/templates/header.php';

// Contenido principal
$usuario_id = $_GET['id'] ?? 1;
$usuario = obtener_usuario((int) $usuario_id);

if ($usuario) {
    ?>
    <h1>Perfil de <?= htmlspecialchars($usuario['nombre']) ?></h1>
    <p>Email: <?= htmlspecialchars($usuario['email']) ?></p>
    <p>Registrado: <?= formatear_fecha($usuario['fecha_creacion']) ?></p>
    <?php
} else {
    include __DIR__ . '/templates/usuario_no_encontrado.php';
}

// Pie de página
include __DIR__ . '/templates/footer.php';
```

===============================================================================
8. BUENAS PRÁCTICAS CON INCLUDE
===============================================================================

  • Usa rutas absolutas con `__DIR__` para evitar ambigüedades.
  • Organiza los includes al inicio del archivo (como imports).
  • Usa `include_once` para funciones, clases y configuraciones.
  • Usa `include` para plantillas y fragmentos repetibles.
  • Verifica la existencia del archivo si hay duda.
  • No incluyas archivos de usuario (nunca hagas `include $_GET['file']`).
  • Agrupa includes relacionados en carpetas temáticas.

```php
<?php

// ❌ PELIGRO: Inclusión dinámica de archivos de usuario
include $_GET['pagina'] . '.php'; // Vulnerability: Local File Inclusion (LFI)

// ✅ Seguro: usar un mapa de páginas permitidas
$paginas_permitidas = [
    'inicio' => 'pages/inicio.php',
    'acerca' => 'pages/acerca.php',
    'contacto' => 'pages/contacto.php',
];

$pagina = $_GET['pagina'] ?? 'inicio';
if (isset($paginas_permitidas[$pagina])) {
    include $paginas_permitidas[$pagina];
} else {
    include 'pages/404.php';
}
```

9. INCLUDE Y RUTAS DINÁMICAS
===============================================================================

A veces necesitas incluir archivos basados en datos dinámicos seguros:

```php
<?php

// Cargar helpers según el módulo activo
$modulo = $_SESSION['modulo'] ?? 'default';

$modulos_permitidos = ['ventas', 'compras', 'inventario', 'admin'];
if (in_array($modulo, $modulos_permitidos, true)) {
    include_once __DIR__ . "/modulos/$modulo/helpers.php";
}

// Cargar controlador basado en la ruta (enrutador simple)
$ruta = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);
$partes = array_values(array_filter(explode('/', $ruta)));

$controlador = $partes[0] ?? 'inicio';
$accion = $partes[1] ?? 'index';

$controladores_permitidos = ['inicio', 'usuarios', 'productos', 'pedidos'];
if (in_array($controlador, $controladores_permitidos, true)) {
    include __DIR__ . "/controladores/$controlador.php";
} else {
    include __DIR__ . '/controladores/error.php';
}
```

10. EJEMPLO COMPLETO: SITIO WEB MULTI-ARCHIVO
===============================================================================

```php
<?php

// ============================================
// Estructura de archivos:
//
// proyecto/
//   index.php
//   includes/
//     config.php
//     database.php
//     funciones.php
//     seguridad.php
//   templates/
//     header.php
//     footer.php
//     inicio.php
//     productos.php
//     contacto.php
//   assets/
//     css/
//     js/
// ============================================

// --- index.php ---
<?php

declare(strict_types=1);

// Bootstrap: cargar configuraciones fundamentales
require_once __DIR__ . '/includes/config.php'; // require (ver capítulo 28)
include_once __DIR__ . '/includes/database.php';
include_once __DIR__ . '/includes/funciones.php';
include_once __DIR__ . '/includes/seguridad.php';

// Iniciar sesión si no está iniciada
if (session_status() === PHP_SESSION_NONE) {
    session_start();
}

// Obtener la ruta solicitada
$ruta = obtener_ruta_segura();

// Array de páginas permitidas con sus archivos
$paginas = [
    'inicio' => 'templates/inicio.php',
    'productos' => 'templates/productos.php',
    'contacto' => 'templates/contacto.php',
    'perfil' => 'templates/perfil.php',
];

// Página por defecto
$pagina_actual = $paginas[$ruta] ?? 'templates/404.php';

// Renderizar
include_once __DIR__ . '/templates/header.php';
include __DIR__ . '/' . $pagina_actual;
include_once __DIR__ . '/templates/footer.php';

// --- includes/config.php ---
<?php

define('SITE_NAME', 'Mi Tienda Online');
define('SITE_URL', 'https://mitienda.com');
define('ADMIN_EMAIL', 'admin@mitienda.com');

// --- includes/funciones.php ---
<?php

function obtener_ruta_segura(): string
{
    $rutas_permitidas = ['inicio', 'productos', 'contacto', 'perfil'];
    $ruta = $_GET['pagina'] ?? 'inicio';

    return in_array($ruta, $rutas_permitidas, true) ? $ruta : 'inicio';
}

function precio_formateado(float $precio, string $moneda = 'MXN'): string
{
    return '$' . number_format($precio, 2) . " $moneda";
}

function resumir_texto(string $texto, int $max_caracteres = 150): string
{
    if (strlen($texto) <= $max_caracteres) return $texto;
    return substr($texto, 0, $max_caracteres) . '...';
}

// --- templates/productos.php ---
<?php
$productos = [
    ['id' => 1, 'nombre' => 'Laptop', 'precio' => 15000.00],
    ['id' => 2, 'nombre' => 'Mouse', 'precio' => 350.00],
    ['id' => 3, 'nombre' => 'Teclado', 'precio' => 850.00],
];
?>

<h2>Nuestros Productos</h2>
<div class="productos">
    <?php foreach ($productos as $producto): ?>
        <div class="producto">
            <h3><?= htmlspecialchars($producto['nombre']) ?></h3>
            <p class="precio"><?= precio_formateado($producto['precio']) ?></p>
            <a href="?pagina=producto&id=<?= $producto['id'] ?>">Ver detalle</a>
        </div>
    <?php endforeach; ?>
</div>
```

===============================================================================
ERRORES COMUNES CON INCLUDE
===============================================================================

Error 1: Ruta incorrecta (archivo no encontrado):

```php
<?php

// Si el script se ejecuta desde un directorio diferente, la ruta falla
include 'config.php'; // Puede no encontrarse

// Mejor usar __DIR__
include __DIR__ . '/includes/config.php';
```

Error 2: Incluir el mismo archivo dos veces con funciones:

```php
<?php

include 'funciones.php';
include 'funciones.php'; // ❌ Fatal error: Cannot redeclare function

// Usar include_once
include_once 'funciones.php';
include_once 'funciones.php'; // Seguro
```

Error 3: Confiar en el valor de retorno de include:

```php
<?php

$resultado = include 'archivo.php';
// Si archivo.php tiene un return, ese es el resultado
// Si no, devuelve 1 (int)

// Útil para archivos de configuración que retornan arrays:
// config.php:
// <?php
// return [
//     'host' => 'localhost',
//     ...
// ];
// 

$config = include 'config.php'; // $config recibe el array
```

Error 4: Incluir archivos de usuario sin validar:

```php
<?php

// ❌ MUY PELIGROSO: LFI (Local File Inclusion)
include $_GET['file'] . '.php';

// El atacante puede hacer:
// ?file=../../../etc/passwd%00
// ?file=../../wp-config
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- `include` inserta y evalúa el contenido de un archivo en el punto
  de inclusión, compartiendo el scope de variables.
- Si el archivo no existe, genera un warning pero el script continúa.
- `include_once` incluye el archivo solo si no ha sido incluido antes,
  previniendo errores de redefinición.
- Usa `include` para plantillas HTML y contenido repetible.
- Usa `include_once` para funciones, clases y configuraciones.
- Usa `__DIR__` para rutas absolutas confiables.
- Nunca incluyas archivos basados en entrada de usuario sin validar.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo usar include dentro de una función?**

Sí. El archivo se incluye en el scope de la función, y las variables
definidas en el archivo serán locales a esa función.

❓ **¿Include admite URLs remotas?**

Sí, si `allow_url_include` está habilitado en php.ini (deshabilitado
por defecto por seguridad). NO se recomienda por riesgos de seguridad.

❓ **¿Qué diferencia hay entre include y require?**

`include` genera warning si el archivo no existe; `require` genera
error fatal y detiene el script. (Lo veremos en el próximo capítulo).

❓ **¿Include funciona con archivos que no son PHP?**

Sí. Puedes incluir archivos HTML, texto, etc. El contenido se trata
como HTML (se imprime directamente) a menos que tenga etiquetas PHP.

❓ **¿Cuál es el máximo de archivos que puedo incluir?**

PHP no tiene un límite específico, pero está sujeto a la memoria
disponible y al tiempo de ejecución. Proyectos grandes pueden incluir
cientos de archivos sin problema.

DESAFÍO DEL ARQUITECTO

🏗️ **Sistema de Plugins con Include Dinámico**

Diseña un sistema de plugins que cargue dinámicamente funcionalidades
usando include, con las siguientes características:

  • `PluginManager` — clase que gestiona plugins usando include_once.
    Debe escanear un directorio `plugins/` y cargar cada plugin.

  • Cada plugin es un archivo PHP que:
    - Define una función `plugin_info(): array` (nombre, versión, autor).
    - Define una función `plugin_init(): void` (inicialización).
    - Puede definir hooks: `plugin_hooks(): array` con nombres de
      funciones a registrar.

  • `cargar_plugins(string $directorio): int` — escanea el directorio,
    incluye cada archivo .php y llama a plugin_init(). Devuelve
    número de plugins cargados.

  • `ejecutar_hook(string $nombre_hook, mixed ...$args): array` —
    ejecuta todas las funciones registradas para un hook y devuelve
    los resultados.

  • `desactivar_plugin(string $nombre): void` — elimina un plugin
    de la lista de activos (no puede "desincluir", pero puede
    evitar su inicialización futura).

  • Plugin de ejemplo: `plugins/auditoria.php` que registra cada
    hook ejecutado en un archivo de log.

Requisitos:
  - Usa include_once para cargar plugins.
  - Verifica que cada plugin tenga las funciones esperadas.
  - Usa is_readable() y file_exists() antes de incluir.
  - Maneja errores: si un plugin falla, continúa con los demás.
  - Implementa protección contra inclusión de archivos maliciosos
    (solo archivos .php, sin rutas externas).
  - Documenta con PHPDoc.

Este desafío integra include dinámico, verificación de archivos,
y un patrón de arquitectura extensible para aplicaciones modulares.
