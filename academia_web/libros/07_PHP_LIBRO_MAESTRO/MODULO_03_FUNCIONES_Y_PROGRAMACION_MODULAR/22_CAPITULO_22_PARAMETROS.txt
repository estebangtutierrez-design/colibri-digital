===============================================================================
1. PARÁMETROS VS ARGUMENTOS
===============================================================================

Es crucial entender la diferencia:

  • **Parámetro**: es la variable definida en la declaración de la función.
  • **Argumento**: es el valor real que se pasa al invocar la función.

```php
<?php

// $nombre es un PARÁMETRO
function saludar(string $nombre): string
{
    return "Hola, $nombre";
}

// "Juan" es un ARGUMENTO
echo saludar("Juan");
```

Los parámetros son como los "huecos" que dejamos en la función;
los argumentos son los "valores" que llenan esos huecos.

===============================================================================
2. PARÁMETROS REQUERIDOS
===============================================================================

Los parámetros requeridos son aquellos que deben ser proporcionados
obligatoriamente al invocar la función. Si faltan, PHP lanza un error.

```php
<?php

declare(strict_types=1);

function sumar(int $a, int $b): int
{
    return $a + $b;
}

// Correcto: se pasan ambos argumentos
echo sumar(5, 3); // 8

// Error: falta el segundo argumento
echo sumar(5); // ArgumentCountError
```

Puedes tener tantos parámetros requeridos como necesites, pero
recuerda el principio de mantener la firma de la función simple.
Si una función necesita más de 3 o 4 parámetros, considera usar
un array asociativo o un objeto DTO (Data Transfer Object).

```php
<?php

// Demasiados parámetros — difícil de recordar el orden
function crear_usuario(string $nombre, string $email, string $password,
                        string $telefono, string $direccion, string $ciudad,
                        string $pais, string $codigo_postal): void
{
    // ...
}

// Mejor: usar un array asociativo
function crear_usuario(array $datos): void
{
    $nombre = $datos['nombre'] ?? '';
    $email = $datos['email'] ?? '';
    $password = $datos['password'] ?? '';
    // ...
}
```

===============================================================================
3. PARÁMETROS OPCIONALES Y VALORES POR DEFECTO
===============================================================================

Los parámetros opcionales tienen un valor por defecto definido en la
declaración. Si no se proporciona un argumento, se usa el valor por
defecto.

```php
<?php

function saludar(string $nombre = "Invitado"): string
{
    return "Hola, $nombre";
}

echo saludar("María");  // Hola, María
echo saludar();          // Hola, Invitado
```

Reglas importantes:

  1. Los parámetros opcionales deben ir DESPUÉS de los requeridos.
  2. El valor por defecto debe ser una expresión constante (PHP 8+ permite
     expresiones más complejas como `new` y llamadas a funciones).
  3. Si no se especifica un tipo, el valor por defecto define el tipo
     esperado implícitamente.

```php
<?php

// ❌ Incorrecto: opcional antes de requerido
function ejemplo($opcional = 10, $requerido): void {}

// ✅ Correcto: requeridos primero, opcionales después
function ejemplo($requerido, $opcional = 10): void {}

// PHP 8+ — expresiones como valor por defecto
function ejemplo($fecha = new DateTimeImmutable()): void
{
    echo $fecha->format('Y-m-d');
}
```

Valores por defecto comunes:

```php
<?php

function configurar_coneccion(
    string $host = 'localhost',
    int $puerto = 3306,
    string $usuario = 'root',
    string $password = '',
    string $base_datos = 'mi_app'
): PDO {
    $dsn = "mysql:host=$host;port=$puerto;dbname=$base_datos";
    return new PDO($dsn, $usuario, $password);
}
```

===============================================================================
4. PARÁMETROS TIPADOS
===============================================================================

PHP 7+ permite declarar el tipo de cada parámetro:

```php
<?php

declare(strict_types=1);

function procesar_datos(
    string $nombre,
    int $edad,
    float $salario,
    bool $activo,
    array $etiquetas
): array {
    return [
        'nombre' => $nombre,
        'edad' => $edad,
        'salario' => $salario,
        'activo' => $activo,
        'etiquetas' => $etiquetas,
    ];
}
```

Tipos disponibles para parámetros:

  • `int` — número entero.
  • `float` — número de punto flotante.
  • `string` — cadena de texto.
  • `bool` — booleano.
  • `array` — arreglo.
  • `object` — objeto.
  • `callable` — función invocable.
  • `iterable` — array u objeto Traversable (PHP 7.1+).
  • `mixed` — cualquier tipo (PHP 8+).
  • `?tipo` — tipo o null (nullable).
  • `int|string` — unión de tipos (PHP 8+).

```php
<?php

// Nullable: el parámetro puede ser del tipo o null
function buscar_usuario(?int $id): ?array
{
    if ($id === null) {
        return null;
    }
    // lógica de búsqueda
    return ['id' => $id, 'nombre' => 'Usuario'];
}

// Unión de tipos (PHP 8+)
function formatear_valor(int|string $valor): string
{
    return (string) $valor;
}

// mixed (PHP 8+)
function depurar(mixed $dato): void
{
    var_dump($dato);
}
```

Modo estricto (strict_types):

Sin `declare(strict_types=1)`, PHP realiza coerción automática de tipos.
Por ejemplo, pasar un string "42" a un parámetro int funciona sin error.

Con `declare(strict_types=1)`, los tipos deben coincidir exactamente,
o se lanza un `TypeError`.

```php
<?php

// archivo_sin_estricto.php
function sumar(int $a, int $b): int
{
    return $a + $b;
}

echo sumar("5", "3"); // 8 (PHP convierte automáticamente)

// archivo_con_estricto.php
declare(strict_types=1);

function sumar_estricto(int $a, int $b): int
{
    return $a + $b;
}

echo sumar_estricto("5", "3"); // TypeError
```

===============================================================================
5. PASO POR VALOR VS PASO POR REFERENCIA
===============================================================================

Por defecto, los argumentos se pasan POR VALOR: la función recibe una
copia del valor original. Modificar el parámetro no afecta la variable
original.

```php
<?php

function agregar_exclamacion(string $texto): string
{
    $texto .= "!";
    return $texto;
}

$mensaje = "Hola";
echo agregar_exclamacion($mensaje); // "Hola!"
echo $mensaje; // "Hola" (sin cambios)
```

Paso por referencia (`&`): la función recibe una referencia a la
variable original. Modificar el parámetro afecta la variable original.

```php
<?php

function agregar_exclamacion_referencia(string &$texto): void
{
    $texto .= "!";
}

$mensaje = "Hola";
agregar_exclamacion_referencia($mensaje);
echo $mensaje; // "Hola!"
```

Precauciones con referencias:

  • Las referencias pueden hacer el código menos predecible.
  • Úsalas con moderación y solo cuando sea necesario.
  • Alternativa moderna: devolver un nuevo valor (inmutabilidad).
  • En PHP 8+, las referencias se usan principalmente en casos
    específicos como modificar arrays grandes o bucles.

Caso de uso legítimo: modificar un array grande sin copiarlo.

```php
<?php

function agregar_al_array(array &$array, mixed $elemento): void
{
    $array[] = $elemento;
}

$mi_array = [1, 2, 3];
agregar_al_array($mi_array, 4);
print_r($mi_array); // [1, 2, 3, 4]
```

===============================================================================
6. OPERADOR SPREAD (`...`)
===============================================================================

El operador spread (PHP 5.6+) permite capturar un número variable de
argumentos en un array dentro de la función.

```php
<?php

function sumar_todos(int ...$numeros): int
{
    return array_sum($numeros);
}

echo sumar_todos(1, 2, 3, 4, 5); // 15
echo sumar_todos(10, 20);         // 30
```

El operador spread debe ser el ÚLTIMO parámetro en la declaración:

```php
<?php

function hacer_pedido(string $cliente, string ...$productos): void
{
    echo "Cliente: $cliente\n";
    echo "Productos:\n";
    foreach ($productos as $producto) {
        echo "- $producto\n";
    }
}

hacer_pedido("Juan", "Laptop", "Mouse", "Teclado");
```

Spread para DESEMPAQUETAR argumentos (al invocar):

```php
<?php

function sumar_tres(int $a, int $b, int $c): int
{
    return $a + $b + $c;
}

$numeros = [10, 20, 30];
echo sumar_tres(...$numeros); // 60
```

Spread con arrays combinados:

```php
<?php

$parte1 = [1, 2, 3];
$parte2 = [4, 5, 6];
$combinado = [...$parte1, ...$parte2];
print_r($combinado); // [1, 2, 3, 4, 5, 6]

// También funciona con arrays asociativos (PHP 8.1+)
$defaults = ['host' => 'localhost', 'port' => 3306];
$custom = ['host' => 'prod.example.com'];
$configuracion = [...$defaults, ...$custom];
print_r($configuracion); // host sobrescrito, port preservado
```

Spread con parámetros tipados:

```php
<?php

declare(strict_types=1);

function promediar(float ...$valores): float
{
    if (empty($valores)) {
        throw new InvalidArgumentException("Se requiere al menos un valor");
    }
    return array_sum($valores) / count($valores);
}

echo promediar(10.5, 20.3, 30.8); // 20.5333...
echo promediar(100.0);             // 100.0
```

===============================================================================
7. PARÁMETROS NOMBRADOS (PHP 8+)
===============================================================================

PHP 8 introdujo los parámetros nombrados, que permiten pasar argumentos
especificando el nombre del parámetro, sin importar el orden:

```php
<?php

function crear_perfil(
    string $nombre,
    string $email,
    int $edad = 0,
    string $pais = "México",
    bool $activo = true
): string {
    return "$nombre ($email), $edad años, $pais, activo: $activo";
}

// Orden tradicional
echo crear_perfil("Ana", "ana@mail.com", 25, "Chile", true);

// Parámetros nombrados — solo especificamos lo que necesitamos
echo crear_perfil(
    nombre: "Ana",
    email: "ana@mail.com",
    pais: "Chile"
);
```

Ventajas de los parámetros nombrados:

  • Mejora la legibilidad en funciones con muchos parámetros.
  • Permite saltar parámetros opcionales intermedios.
  • No depende del orden de los parámetros.
  • Ideal para valores por defecto.

```php
<?php

// Saltar parámetros opcionales intermedios
echo crear_perfil(
    nombre: "Carlos",
    email: "carlos@mail.com",
    activo: false
);
// No necesitamos pasar $edad ni $pais
```

Limitaciones:

  • No se pueden mezclar argumentos posicionales y nombrados
    después del primer argumento nombrado.
  • Los argumentos nombrados no pueden usarse con funciones
    variádicas (spread) en el lado de la llamada.

```php
<?php

// ❌ Incorrecto
echo crear_perfil("Ana", email: "ana@mail.com", 25);

// ✅ Correcto: posicionales primero, luego nombrados
echo crear_perfil("Ana", "ana@mail.com", pais: "Chile");

// ✅ Correcto: todos nombrados
echo crear_perfil(nombre: "Ana", email: "ana@mail.com", edad: 25);
```

===============================================================================
8. VALIDACIÓN DE PARÁMETROS
===============================================================================

Siempre valida los parámetros al inicio de la función para garantizar
que cumplan con los requisitos:

```php
<?php

declare(strict_types=1);

function dividir(float $dividendo, float $divisor): float
{
    if ($divisor === 0.0) {
        throw new InvalidArgumentException("El divisor no puede ser cero");
    }

    if ($dividendo < 0) {
        throw new InvalidArgumentException("El dividendo no puede ser negativo");
    }

    return $dividendo / $divisor;
}

function crear_usuario(array $datos): array
{
    $campos_requeridos = ['nombre', 'email', 'password'];

    foreach ($campos_requeridos as $campo) {
        if (empty($datos[$campo])) {
            throw new InvalidArgumentException(
                "El campo '$campo' es obligatorio"
            );
        }
    }

    if (!filter_var($datos['email'], FILTER_VALIDATE_EMAIL)) {
        throw new InvalidArgumentException("Email inválido");
    }

    if (strlen($datos['password']) < 8) {
        throw new InvalidArgumentException(
            "La contraseña debe tener al menos 8 caracteres"
        );
    }

    // Procesar...
    return ['id' => 1, ...$datos];
}
```

9. EJEMPLO COMPLETO: PROCESADOR DE PEDIDOS
===============================================================================

```php
<?php

declare(strict_types=1);

/**
 * Calcula el subtotal de un pedido.
 */
function calcular_subtotal(array $productos): float
{
    $subtotal = 0.0;

    foreach ($productos as $producto) {
        if (!isset($producto['precio'], $producto['cantidad'])) {
            throw new InvalidArgumentException(
                "Cada producto requiere 'precio' y 'cantidad'"
            );
        }
        $subtotal += $producto['precio'] * $producto['cantidad'];
    }

    return $subtotal;
}

/**
 * Calcula el descuento basado en el total.
 */
function calcular_descuento(float $total, float $tasa_descuento = 0.0): float
{
    if ($tasa_descuento < 0 || $tasa_descuento > 1) {
        throw new InvalidArgumentException(
            "La tasa de descuento debe estar entre 0 y 1"
        );
    }

    return $total * $tasa_descuento;
}

/**
 * Calcula el impuesto.
 */
function calcular_impuesto(float $subtotal, float $tasa_impuesto = 0.16): float
{
    return $subtotal * $tasa_impuesto;
}

/**
 * Procesa un pedido completo.
 */
function procesar_pedido(
    array $productos,
    string $cliente,
    float $tasa_descuento = 0.0,
    float $tasa_impuesto = 0.16,
    string ...$notas
): array {
    $subtotal = calcular_subtotal($productos);
    $descuento = calcular_descuento($subtotal, $tasa_descuento);
    $subtotal_con_descuento = $subtotal - $descuento;
    $impuesto = calcular_impuesto($subtotal_con_descuento, $tasa_impuesto);
    $total = $subtotal_con_descuento + $impuesto;

    return [
        'cliente' => $cliente,
        'productos' => $productos,
        'subtotal' => $subtotal,
        'descuento' => $descuento,
        'impuesto' => $impuesto,
        'total' => $total,
        'notas' => $notas,
        'fecha' => date('Y-m-d H:i:s'),
    ];
}

// --- USO ---
$pedido = procesar_pedido(
    [
        ['nombre' => 'Laptop', 'precio' => 15000.00, 'cantidad' => 1],
        ['nombre' => 'Mouse', 'precio' => 350.00, 'cantidad' => 2],
        ['nombre' => 'Teclado', 'precio' => 850.00, 'cantidad' => 1],
    ],
    cliente: "Juan Pérez",
    tasa_descuento: 0.10,
    notas: "Envío urgente",
    "Facturar con RFC"
);

echo "Total a pagar: $" . number_format($pedido['total'], 2);
```

10. BUENAS PRÁCTICAS CON PARÁMETROS
===============================================================================

  • Limita el número de parámetros (idealmente ≤ 3).
  • Usa nombres descriptivos que indiquen el propósito.
  • Declara tipos siempre que sea posible.
  • Coloca los parámetros requeridos antes que los opcionales.
  • Usa valores por defecto sensatos.
  • Documenta los parámetros con PHPDoc.
  • Valida los parámetros al inicio de la función.
  • Prefiere inmutabilidad (paso por valor sobre referencia).
  • Considera arrays/objetos para grupos de parámetros relacionados.
  • Usa parámetros nombrados (PHP 8+) para mejorar legibilidad.

===============================================================================
ERRORES COMUNES CON PARÁMETROS
===============================================================================

Error 1: Olvidar el orden de los parámetros:

```php
<?php

function calcular_area(float $ancho, float $alto): float
{
    return $ancho * $alto;
}

// ❌ ¿Cuál es ancho y cuál alto?
echo calcular_area(10, 20);

// ✅ Con parámetros nombrados (PHP 8+)
echo calcular_area(ancho: 10, alto: 20);
```

Error 2: Pasar referencias cuando no es necesario:

```php
<?php

// ❌ Innecesario — la referencia confunde
function limpiar_texto(string &$texto): void
{
    $texto = trim($texto);
}

// ✅ Mejor — devolver el valor limpio
function limpiar_texto(string $texto): string
{
    return trim($texto);
}
```

Error 3: No validar valores por defecto:

```php
<?php

// ❌ Peligroso — $divisor puede ser 0
function dividir(float $a, float $b = 0): float
{
    return $a / $b; // DivisionByZeroError
}

// ✅ Seguro — validar antes de operar
function dividir(float $a, float $b = 1): float
{
    if ($b === 0.0) {
        throw new InvalidArgumentException("División por cero");
    }
    return $a / $b;
}
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Los parámetros son variables definidas en la función; los argumentos
  son los valores que se pasan al invocarla.
- Los parámetros requeridos son obligatorios; los opcionales tienen
  valores por defecto y deben ir después de los requeridos.
- Los tipos de parámetros (int, string, array, etc.) mejoran la
  seguridad y legibilidad del código.
- El paso por valor copia el dato; el paso por referencia (&) modifica
  la variable original.
- El operador spread (`...`) captura argumentos variables en un array.
- Los parámetros nombrados (PHP 8+) mejoran la legibilidad y permiten
  saltar opcionales.
- Valida siempre los parámetros al inicio de la función.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es el límite de parámetros que puede tener una función?**

PHP no impone un límite teórico, pero por buena práctica se recomienda
no superar los 3-4 parámetros. Si necesitas más, agrupa en un array
u objeto.

❓ **¿Puedo mezclar parámetros posicionales y nombrados?**

Sí, pero los posicionales deben ir antes que los nombrados.
Una vez que usas un parámetro nombrado, todos los siguientes
deben ser nombrados también.

❓ **¿Qué pasa si no uso strict_types?**

Sin strict_types, PHP realiza coerción automática. "5" se convierte
a 5, 1 a true, etc. Esto puede ocultar errores. En proyectos
profesionales, usa siempre strict_types.

❓ **¿El operador spread funciona con arrays asociativos?**

Sí, desde PHP 8.1 puedes usar spread con arrays asociativos.
Los elementos con mismas claves se sobrescriben (útil para merges).

❓ **¿Puedo tener un spread antes de parámetros requeridos?**

No. El operador spread debe ser el último parámetro en la
declaración de la función, después de todos los parámetros
requeridos y opcionales.
