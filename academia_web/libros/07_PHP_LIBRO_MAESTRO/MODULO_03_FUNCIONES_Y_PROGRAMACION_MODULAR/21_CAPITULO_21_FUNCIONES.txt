1. QUÉ ES UNA FUNCIÓN
===============================================================================

Una función es un bloque de código reutilizable que realiza una tarea
específica. Piensa en ella como una máquina: recibe una entrada (datos),
procesa esa entrada y produce una salida (resultado).

En PHP, las funciones son ciudadanos de primera clase, lo que significa
que pueden ser asignadas a variables, pasadas como argumentos y devueltas
desde otras funciones.

Beneficios de usar funciones:

  • Reutilización: escribes el código una vez y lo usas muchas veces.
  • Abstracción: ocultas los detalles complejos detrás de un nombre simple.
  • Organización: divides problemas grandes en partes pequeñas.
  • Mantenibilidad: cambios locales no afectan todo el sistema.
  • Legibilidad: el código se lee como una historia con verbos (funciones).

===============================================================================
2. SINTAXIS DE DECLARACIÓN DE FUNCIONES
===============================================================================

La sintaxis básica para declarar una función en PHP es la siguiente:

```php
<?php

function nombreDeLaFuncion(): tipoDeRetorno
{
    // Cuerpo de la función
    return valor;
}
```

Veamos un ejemplo completo:

```php
<?php

function saludar(): string
{
    return "¡Hola, mundo!";
}

echo saludar(); // ¡Hola, mundo!
```

Componentes de una declaración de función:

  • `function`: palabra reservada que indica que declaramos una función.
  • `nombreDeLaFuncion`: identificador que usaremos para invocarla.
  • `()`: paréntesis que pueden contener parámetros.
  • `{ }`: llaves que delimitan el cuerpo de la función.
  • `return`: palabra reservada que devuelve un valor (opcional).

Las funciones en PHP pueden ser declaradas en cualquier parte del script
y pueden ser invocadas antes de su declaración (gracias al parseo
temprano de PHP), aunque por convención se declaran al inicio o en
archivos separados.

```php
<?php

// Se puede invocar antes de declarar
echo sumar(5, 3); // 8

function sumar(int $a, int $b): int
{
    return $a + $b;
}
```

===============================================================================
3. CONVENCIONES DE NOMENCLATURA
===============================================================================

Nombrar funciones correctamente es crucial para la legibilidad del código.
PHP no impone un estilo único, pero la comunidad ha adoptado convenciones.

Reglas generales:

  • Usar snake_case (estilo PSR — recomendado por la comunidad PHP):
    `calcular_total`, `obtener_usuario`, `validar_email`.

  • El nombre debe ser descriptivo y revelar la intención:
    ❌ `hacer()` — demasiado vago.
    ✅ `calcular_impuesto()` — claro y específico.

  • Usar verbos o frases verbales:
    `obtenerUsuario()`, `eliminarRegistro()`, `procesarPago()`.

  • Ser consistente con el resto del proyecto: si el proyecto usa
    camelCase, úsalo también.

  • Prefijos útiles:
    - `get_` / `obtener_` para obtener datos.
    - `set_` / `establecer_` para asignar datos.
    - `is_` / `es_` para validaciones booleanas.
    - `has_` / `tiene_` para verificar existencia.
    - `process_` / `procesar_` para transformaciones.
    - `validate_` / `validar_` para validaciones.

Ejemplos de buenos nombres:

```php
<?php

function obtener_usuario_por_id(int $id): ?array
{
    // Lógica para obtener usuario
}

function es_email_valido(string $email): bool
{
    // Lógica de validación
}

function calcular_total_con_impuesto(float $subtotal, float $tasa): float
{
    return $subtotal * (1 + $tasa);
}

function formatear_fecha(string $fecha, string $formato): string
{
    return date($formato, strtotime($fecha));
}
```

===============================================================================
4. DECLARACIONES DE TIPO DE RETORNO
===============================================================================

PHP 7 introdujo las declaraciones de tipo de retorno, permitiendo
especificar qué tipo de dato devuelve una función.

Sintaxis:

```php
<?php

function nombre(): tipo
{
    // cuerpo
}
```

Tipos de retorno disponibles:

  • Tipos escalares: `int`, `float`, `string`, `bool`.
  • Tipos compuestos: `array`, `object`, `callable`, `iterable`.
  • Tipo especial: `void` (sin retorno).
  • Tipo nullable: `?tipo` (el tipo o null).
  • Tipos unión (PHP 8+): `int|string`, `array|false`.
  • Mixed (PHP 8+): `mixed` (cualquier tipo).

Ejemplos:

```php
<?php

function dividir(int $a, int $b): float
{
    return $a / $b;
}

function obtener_configuracion(string $clave): ?string
{
    $config = [
        'db_host' => 'localhost',
        'db_user' => 'root',
    ];

    return $config[$clave] ?? null;
}

function es_par(int $numero): bool
{
    return $numero % 2 === 0;
}

function loggear_mensaje(string $mensaje): void
{
    file_put_contents('app.log', $mensaje . PHP_EOL, FILE_APPEND);
}

// PHP 8.0+ — Unión de tipos
function obtener_valor(mixed $dato): int|float
{
    if (is_numeric($dato)) {
        return (float) $dato;
    }
    return 0;
}
```

Strict types:

PHP permite declarar el modo estricto de tipos al inicio del archivo:

```php
<?php

declare(strict_types=1);

function sumar(int $a, int $b): int
{
    return $a + $b;
}

echo sumar(5, 3);    // 8
// echo sumar("5", 3); // TypeError en strict_types
```

Sin strict_types, PHP intenta convertir automáticamente los tipos
(coerción). Con strict_types, se lanza un TypeError si los tipos
no coinciden exactamente.

===============================================================================
5. FUNCIONES INTERNAS VS FUNCIONES DE USUARIO
===============================================================================

PHP viene con cientos de funciones integradas (built-in functions).
Algunas de las más comunes:

```php
<?php

// Funciones de string
strlen("Hola");           // 4
strtoupper("hola");       // "HOLA"
str_replace("a", "o", "casa"); // "coso"

// Funciones de array
count([1, 2, 3]);         // 3
array_merge([1], [2]);    // [1, 2]
in_array(3, [1, 2, 3]);  // true

// Funciones de fecha
date("Y-m-d");            // 2026-07-01
time();                   // timestamp actual

// Funciones de archivo
file_get_contents("archivo.txt");
file_put_contents("archivo.txt", "contenido");
```

Las funciones definidas por el usuario son aquellas que creamos
nosotros para resolver problemas específicos de nuestra aplicación.

Recomendación: antes de escribir una función, verifica si PHP ya
tiene una función integrada que haga lo que necesitas. La documentación
oficial (php.net) es tu mejor aliada.

```php
<?php

// En lugar de escribir esto:
function a_mayusculas(string $texto): string
{
    return strtoupper($texto);
}

// Simplemente usa la función nativa:
echo strtoupper("hola");
```

===============================================================================
6. FUNCIONES CON EFECTOS SECUNDARIOS
===============================================================================

No todas las funciones deben devolver un valor. Algunas realizan
acciones conocidas como "efectos secundarios":

```php
<?php

function mostrar_bienvenida(string $nombre): void
{
    echo "<h1>Bienvenido, $nombre</h1>";
}

function enviar_correo(string $para, string $asunto, string $cuerpo): void
{
    mail($para, $asunto, $cuerpo);
}

function redirigir(string $url): never
{
    header("Location: $url");
    exit;
}
```

El tipo `never` (PHP 8.1+) se usa para funciones que nunca retornan
porque siempre lanzan una excepción o terminan la ejecución.

===============================================================================
7. DOCUMENTACIÓN DE FUNCIONES CON PHPDoc
===============================================================================

Una buena práctica es documentar tus funciones usando bloques PHPDoc:

```php
<?php

/**
 * Calcula el precio total incluyendo impuesto.
 *
 * @param float $subtotal El subtotal sin impuesto.
 * @param float $tasa_impuesto La tasa de impuesto (ej: 0.16 para 16%).
 * @return float El total con impuesto incluido.
 * @throws InvalidArgumentException Si la tasa es negativa.
 */
function calcular_total(float $subtotal, float $tasa_impuesto): float
{
    if ($tasa_impuesto < 0) {
        throw new InvalidArgumentException("La tasa no puede ser negativa");
    }

    return $subtotal * (1 + $tasa_impuesto);
}
```

La documentación PHPDoc ayuda a los IDE a proporcionar autocompletado
y advertencias, y facilita el trabajo en equipo.

===============================================================================
8. EJEMPLO COMPLETO: SISTEMA DE VALIDACIÓN
===============================================================================

Veamos un ejemplo práctico que integra funciones con tipo de retorno:

```php
<?php

declare(strict_types=1);

/**
 * Valida que un email tenga formato correcto.
 */
function validar_email(string $email): bool
{
    return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
}

/**
 * Valida que una contraseña cumpla los requisitos mínimos.
 */
function validar_contrasena(string $password): array
{
    $errores = [];

    if (strlen($password) < 8) {
        $errores[] = "Debe tener al menos 8 caracteres";
    }

    if (!preg_match('/[A-Z]/', $password)) {
        $errores[] = "Debe contener una mayúscula";
    }

    if (!preg_match('/[0-9]/', $password)) {
        $errores[] = "Debe contener un número";
    }

    return $errores;
}

/**
 * Procesa el registro de un nuevo usuario.
 */
function registrar_usuario(string $nombre, string $email, string $password): array
{
    $resultado = [
        'exito' => false,
        'errores' => [],
    ];

    if (empty($nombre)) {
        $resultado['errores'][] = "El nombre es obligatorio";
    }

    if (!validar_email($email)) {
        $resultado['errores'][] = "El email no es válido";
    }

    $errores_password = validar_contrasena($password);
    if (!empty($errores_password)) {
        $resultado['errores'] = array_merge($resultado['errores'], $errores_password);
    }

    if (empty($resultado['errores'])) {
        // Aquí iría la lógica para guardar en base de datos
        $resultado['exito'] = true;
        $resultado['mensaje'] = "Usuario registrado correctamente";
    }

    return $resultado;
}

// --- USO ---
$registro = registrar_usuario("Juan Pérez", "juan@ejemplo.com", "Segura123");

if ($registro['exito']) {
    echo $registro['mensaje'];
} else {
    echo "Errores encontrados:\n";
    foreach ($registro['errores'] as $error) {
        echo "- $error\n";
    }
}
```

9. BUENAS PRÁCTICAS CON FUNCIONES
===============================================================================

  • Una función debe hacer una sola cosa (principio de responsabilidad única).
  • Mantén las funciones pequeñas (idealmente menos de 20-30 líneas).
  • Usa nombres descriptivos que revelen la intención.
  • Declara tipos de retorno siempre que sea posible.
  • Documenta con PHPDoc cuando la lógica no sea obvia.
  • Evita efectos secundarios inesperados (modificar variables globales).
  • Prefiere devolver valores a imprimir directamente (testeabilidad).
  • Agrupa funciones relacionadas en archivos separados.

===============================================================================
ERRORES COMUNES AL DECLARAR FUNCIONES
===============================================================================

Error 1: Olvidar el punto y coma en `return`:

```php
<?php

function sumar(int $a, int $b): int
{
    return $a + $b  // ❌ Error: falta punto y coma
}
```

Error 2: Redefinir una función existente:

```php
<?php

function saludar(): string
{
    return "Hola";
}

function saludar(): string // ❌ Error: función ya definida
{
    return "Hello";
}
```

Error 3: Confundir parámetros con argumentos:

```php
<?php

function sumar(int $a, int $b): int  // $a y $b son parámetros
{
    return $a + $b;
}

sumar(5, 3); // 5 y 3 son argumentos
```

Error 4: No declarar strict_types cuando se necesita precisión:

```php
<?php

// Sin strict_types
function dividir(int $a, int $b): float
{
    return $a / $b;
}

echo dividir(5, 2); // 2.5 (funciona, pero PHP convierte implícitamente)

// Con strict_types
declare(strict_types=1);

function dividir_estricto(int $a, int $b): float
{
    return $a / $b;
}

echo dividir_estricto(5, 2); // 2.5
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Las funciones son bloques de código reutilizables que realizan tareas
  específicas y son la base de la programación modular.
- Se declaran con la palabra reservada `function`, seguida del nombre,
  paréntesis para parámetros y llaves para el cuerpo.
- Las convenciones de nomenclatura más usadas son snake_case (PSR)
  y camelCase; la consistencia es clave.
- Las declaraciones de tipo de retorno (`: tipo`) mejoran la claridad
  y seguridad del código, especialmente con `declare(strict_types=1)`.
- PHP ofrece cientos de funciones integradas que debemos conocer antes
  de reinventar la rueda.
- Documentar funciones con PHPDoc facilita el mantenimiento y el
  trabajo en equipo.
- Una buena función hace una sola cosa, tiene un nombre descriptivo
  y minimiza los efectos secundarios.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo declarar una función dentro de otra función?**

Sí, PHP permite declarar funciones dentro de funciones, pero la función
interna solo estará disponible después de que la función externa sea
invocada. No se recomienda por confusión en el scope.

❓ **¿Qué pasa si no uso `return` en una función?**

Si una función no tiene `return`, PHP devuelve `null` implícitamente.
Si declaraste un tipo de retorno no-nullable, PHP lanzará un TypeError.

❓ **¿Puedo tener dos funciones con el mismo nombre?**

No. PHP lanza un error fatal si intentas redefinir una función ya
declarada. Usa espacios de nombres (namespaces) para evitar colisiones.

❓ **¿Las funciones en PHP distinguen mayúsculas de minúsculas?**

No, los nombres de las funciones en PHP son case-insensitive.
`SALUDAR()` y `saludar()` llaman a la misma función. Sin embargo,
por convención siempre usamos la misma capitalización.

❓ **¿Debo siempre declarar el tipo de retorno?**

No es obligatorio, pero es una buena práctica. Mejora la legibilidad,
permite que el IDE te ayude y previene errores. En proyectos
profesionales, siempre declara tipos de retorno.

La función `generar_reporte_estadistico` debe devolver un array
asociativo con todas las estadísticas calculadas. Crea un script
que use tus funciones y genere un reporte formateado.

Este desafío pone a prueba tu capacidad para diseñar una API de
funciones coherente, tipada y documentada.
