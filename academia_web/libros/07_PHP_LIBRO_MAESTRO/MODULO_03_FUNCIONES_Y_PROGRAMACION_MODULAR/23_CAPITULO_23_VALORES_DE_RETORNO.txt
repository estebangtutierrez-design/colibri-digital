¡Hola de nuevo, valiente programador!

Hemos visto cómo las funciones reciben datos a través de parámetros.
Ahora exploraremos cómo las funciones devuelven resultados: los valores

El `return` es el mecanismo por el cual una función se comunica con

1. LA INSTRUCCIÓN RETURN
===============================================================================

La instrucción `return` finaliza inmediatamente la ejecución de una
función y devuelve un valor opcional a quien la invocó.

```php
<?php

function sumar(int $a, int $b): int
{
    return $a + $b;
    // Cualquier código después de return NO se ejecuta
    echo "Esto nunca se ejecutará";
}

$resultado = sumar(5, 3);
echo $resultado; // 8
```

Comportamiento de `return`:

  1. Finaliza la ejecución de la función inmediatamente.
  2. Devuelve el valor especificado a quien invocó la función.
  3. Solo puede devolver UN valor (pero ese valor puede ser un array).
  4. Si no se especifica un valor, la función devuelve `null`.

```php
<?php

function retorno_temprano(int $numero): string
{
    if ($numero < 0) {
        return "Negativo"; // Return temprano
    }

    if ($numero === 0) {
        return "Cero";
    }

    return "Positivo";
}

echo retorno_temprano(-5); // Negativo
echo retorno_temprano(0);  // Cero
echo retorno_temprano(10); // Positivo
```

Múltiples puntos de retorno: no hay nada malo con tener varios `return`
si el flujo es claro. El estilo "early return" (retorno temprano) es
una práctica recomendada que reduce la anidación.

```php
<?php

// ❌ Estilo profundamente anidado
function validar_usuario(array $usuario): array
{
    if (isset($usuario['nombre'])) {
        if (isset($usuario['email'])) {
            if (filter_var($usuario['email'], FILTER_VALIDATE_EMAIL)) {
                return ['valido' => true, 'usuario' => $usuario];
            } else {
                return ['valido' => false, 'error' => 'Email inválido'];
            }
        } else {
            return ['valido' => false, 'error' => 'Email requerido'];
        }
    } else {
        return ['valido' => false, 'error' => 'Nombre requerido'];
    }
}

// ✅ Estilo early return — más legible
function validar_usuario(array $usuario): array
{
    if (!isset($usuario['nombre'])) {
        return ['valido' => false, 'error' => 'Nombre requerido'];
    }

    if (!isset($usuario['email'])) {
        return ['valido' => false, 'error' => 'Email requerido'];
    }

    if (!filter_var($usuario['email'], FILTER_VALIDATE_EMAIL)) {
        return ['valido' => false, 'error' => 'Email inválido'];
    }

    return ['valido' => true, 'usuario' => $usuario];
}
```

===============================================================================
2. DECLARACIONES DE TIPO DE RETORNO
===============================================================================

PHP 7 introdujo la posibilidad de declarar el tipo de retorno esperado.
Esto permite al motor de PHP verificar que la función devuelva el tipo
correcto.

```php
<?php

declare(strict_types=1);

function obtener_edad(int $año_nacimiento): int
{
    return date('Y') - $año_nacimiento;
}

function obtener_precio_formateado(float $precio): string
{
    return '$' . number_format($precio, 2);
}

function obtener_usuario(int $id): array
{
    return [
        'id' => $id,
        'nombre' => 'Usuario ' . $id,
    ];
}

function obtener_config(string $clave): ?string
{
    $config = [
        'db_host' => 'localhost',
    ];
    return $config[$clave] ?? null;
}
```

Si una función declara un tipo de retorno pero devuelve un tipo
incorrecto, PHP lanza un `TypeError`:

```php
<?php

declare(strict_types=1);

function obtener_edad(int $año_nacimiento): int
{
    return "treinta"; // TypeError: string devuelto, se esperaba int
}
```

3. RETORNOS NULABLES (`?tipo`)
===============================================================================

Un tipo nullable permite que una función devuelva el tipo especificado
O `null`. Se declara con un signo `?` antes del tipo.

```php
<?php

function buscar_usuario(int $id): ?array
{
    $usuarios = [
        1 => ['nombre' => 'Ana'],
        2 => ['nombre' => 'Luis'],
    ];

    return $usuarios[$id] ?? null;
}

$usuario = buscar_usuario(1); // ['nombre' => 'Ana']
$usuario = buscar_usuario(99); // null

if ($usuario !== null) {
    echo $usuario['nombre'];
} else {
    echo "Usuario no encontrado";
}
```

En PHP 8+, también puedes usar la sintaxis de unión de tipos:

```php
<?php

// PHP 8+ — equivalente a ?array
function buscar_producto(int $id): array|null
{
    // ...
}
```

Casos de uso comunes para retornos nulables:

  • Búsquedas que pueden no encontrar resultados.
  • Configuraciones opcionales.
  • IDs que pueden ser null (relaciones opcionales).
  • Parsers que pueden fallar.

```php
<?php

function parsear_json(string $json): ?array
{
    $datos = json_decode($json, true);

    if (json_last_error() !== JSON_ERROR_NONE) {
        return null;
    }

    return $datos;
}

function obtener_primer_elemento(array $array): mixed
{
    if (empty($array)) {
        return null;
    }

    return reset($array);
}
```

===============================================================================
4. EL TIPO VOID
===============================================================================

El tipo `void` indica que la función no devuelve ningún valor útil.
Se usa para funciones que realizan acciones (efectos secundarios)
sin necesidad de retornar algo.

```php
<?php

function loggear(string $mensaje): void
{
    $archivo = 'app.log';
    $linea = '[' . date('Y-m-d H:i:s') . '] ' . $mensaje . PHP_EOL;
    file_put_contents($archivo, $linea, FILE_APPEND);
    // No hay return (o return; sin valor)
}

function enviar_correo_bienvenida(string $email): void
{
    $asunto = "¡Bienvenido a nuestra plataforma!";
    $cuerpo = "Gracias por registrarte.";
    mail($email, $asunto, $cuerpo);
    // Si no hay return, PHP retorna null implícitamente
}

// return sin valor está permitido en funciones void
function imprimir_encabezado(string $titulo): void
{
    if (empty($titulo)) {
        return; // Sale temprano, no devuelve valor
    }

    echo "<h1>$titulo</h1>";
}
```

Usar `void` en lugar de no declarar tipo hace explícita la intención
de que la función no retorna un valor significativo.

```php
<?php

// Explícito: otros desarrolladores saben que no deben esperar un valor
function actualizar_cache(string $clave, mixed $valor): void
{
    // Lógica de actualización de caché
}

// Si intentamos usar el retorno, PHP no lanza error pero es engañoso
$resultado = actualizar_cache('clave', 'valor');
var_dump($resultado); // null
```

===============================================================================
5. EL TIPO NEVER (PHP 8.1+)
===============================================================================

El tipo `never` indica que la función NUNCA retorna, porque siempre:

  • Lanza una excepción.
  • Termina la ejecución con `exit`/`die`.
  • Redirige con `header()` + `exit`.

```php
<?php

function abortar(int $codigo = 404): never
{
    http_response_code($codigo);
    require "error_$codigo.php";
    exit;
}

function lanzar_error(string $mensaje): never
{
    throw new RuntimeException($mensaje);
}

function redirigir(string $url): never
{
    header("Location: $url");
    exit;
}

// Uso
if (!$usuario_encontrado) {
    abortar(404);
}
```

Diferencia entre `void` y `never`:

  • `void`: la función termina normalmente pero no devuelve valor.
  • `never`: la función NUNCA termina normalmente (excepción o exit).

```php
<?php

function solo_log(): void
{
    // Termina normalmente, sin valor
}

function solo_error(): never
{
    // Nunca termina normalmente
    throw new Exception("Error fatal");
}
```

6. FUNCIONES PURAS VS FUNCIONES CON EFECTOS SECUNDARIOS
===============================================================================

Función pura:

  • Dados los mismos argumentos, siempre devuelve el mismo resultado.
  • No tiene efectos secundarios (no modifica estado externo).
  • No depende de estado externo (archivos, DB, variables globales).

```php
<?php

// ✅ Función pura
function sumar(int $a, int $b): int
{
    return $a + $b;
}

// ✅ Función pura
function calcular_iva(float $monto, float $tasa = 0.16): float
{
    return $monto * $tasa;
}

// ✅ Función pura
function convertir_celsius_a_fahrenheit(float $celsius): float
{
    return ($celsius * 9/5) + 32;
}
```

Función impura (con efectos secundarios):

  • Depende de o modifica estado externo.
  • Puede devolver resultados diferentes con los mismos argumentos.

```php
<?php

// ❌ Función impura: depende de la hora actual
function obtener_saludo(): string
{
    $hora = (int) date('H');
    return $hora < 12 ? "Buenos días" : "Buenas tardes";
}

// ❌ Función impura: modifica una variable externa
$contador = 0;
function incrementar_contador(): void
{
    global $contador;
    $contador++;
}

// ❌ Función impura: depende de archivo externo
function leer_configuracion(): array
{
    return json_decode(file_get_contents('config.json'), true);
}
```

Ventajas de las funciones puras:

  • Fáciles de probar (no requieren mockear estado externo).
  • Predecibles y deterministas.
  • Pueden ser cacheadas (misma entrada → misma salida).
  • Más fáciles de depurar.
  • Mejor composición (pueden combinarse sin sorpresas).

7. RETORNAR MÚLTIPLES VALORES
===============================================================================

PHP solo permite retornar un valor. Para devolver múltiples valores,
usa un array o un objeto:

```php
<?php

function dividir_con_residuo(int $dividendo, int $divisor): array
{
    if ($divisor === 0) {
        throw new InvalidArgumentException("División por cero");
    }

    return [
        'cociente' => intdiv($dividendo, $divisor),
        'residuo' => $dividendo % $divisor,
    ];
}

$resultado = dividir_con_residuo(10, 3);
echo $resultado['cociente']; // 3
echo $resultado['residuo'];  // 1

// PHP 8+ — también puedes usar match + array destructuring
function obtener_coordenadas(): array
{
    return ['x' => 10.5, 'y' => 20.3];
}

['x' => $x, 'y' => $y] = obtener_coordenadas();
echo "x=$x, y=$y";
```

Para retornar objetos:

```php
<?php

class ResultadoOperacion
{
    public function __construct(
        public readonly bool $exito,
        public readonly mixed $dato = null,
        public readonly ?string $error = null
    ) {}
}

function procesar_pago(float $monto, string $metodo): ResultadoOperacion
{
    if ($monto <= 0) {
        return new ResultadoOperacion(false, error: "Monto inválido");
    }

    // Lógica de pago...
    return new ResultadoOperacion(true, dato: ['id' => uniqid()]);
}

$resultado = procesar_pago(500, "tarjeta");
if ($resultado->exito) {
    print_r($resultado->dato);
} else {
    echo "Error: " . $resultado->error;
}
```

===============================================================================
8. RETORNAR FUNCIONES (CLOSURES)
===============================================================================

Las funciones pueden retornar otras funciones, lo que permite crear
fábricas de funciones (closures):

```php
<?php

function creador_saludo(string $idioma): callable
{
    return match ($idioma) {
        'es' => function(string $nombre): string {
            return "¡Hola, $nombre!";
        },
        'en' => function(string $nombre): string {
            return "Hello, $nombre!";
        },
        'fr' => function(string $nombre): string {
            return "Bonjour, $nombre!";
        },
        default => function(string $nombre): string {
            return "Hola, $nombre!";
        },
    };
}

$saludar_es = creador_saludo('es');
echo $saludar_es('Ana'); // ¡Hola, Ana!

$saludar_en = creador_saludo('en');
echo $saludar_en('John'); // Hello, John!
```

9. EJEMPLO COMPLETO: PROCESADOR DE TRANSFORMACIONES
===============================================================================

```php
<?php

declare(strict_types=1);

/**
 * Limpia y normaliza un texto.
 */
function limpiar_texto(string $texto, bool $minusculas = true): string
{
    $texto = trim($texto);

    if ($minusculas) {
        $texto = mb_strtolower($texto);
    }

    return $texto;
}

/**
 * Extrae palabras clave de un texto.
 */
function extraer_palabras_clave(string $texto, int $limite = 5): array
{
    $palabras = str_word_count($texto, 1);
    $palabras = array_unique($palabras);
    $palabras = array_filter($palabras, function(string $p): bool {
        return strlen($p) > 3;
    });

    return array_slice(array_values($palabras), 0, $limite);
}

/**
 * Calcula la similitud entre dos textos (Jaccard相似度 simplificado).
 */
function calcular_similitud(string $texto1, string $texto2): float
{
    $t1 = limpiar_texto($texto1);
    $t2 = limpiar_texto($texto2);

    $palabras1 = extraer_palabras_clave($t1, 100);
    $palabras2 = extraer_palabras_clave($t2, 100);

    if (empty($palabras1) && empty($palabras2)) {
        return 1.0;
    }

    $interseccion = array_intersect($palabras1, $palabras2);
    $union = array_unique(array_merge($palabras1, $palabras2));

    return count($interseccion) / count($union);
}

/**
 * Analiza un texto y devuelve un reporte completo.
 */
function analizar_texto(string $texto): array
{
    if (empty(trim($texto))) {
        return [
            'valido' => false,
            'error' => 'El texto no puede estar vacío',
        ];
    }

    $palabras_clave = extraer_palabras_clave($texto);

    return [
        'valido' => true,
        'longitud' => strlen($texto),
        'palabras' => str_word_count($texto),
        'palabras_clave' => $palabras_clave,
        'normalizado' => limpiar_texto($texto),
        'analisis' => [
            'tiene_mayusculas' => $texto !== mb_strtolower($texto),
            'tiene_numeros' => (bool) preg_match('/\d/', $texto),
            'similitud_referencia' => calcular_similitud(
                $texto,
                "PHP es un lenguaje de programación"
            ),
        ],
    ];
}

// --- USO ---
$reporte = analizar_texto("PHP MASTER BOOK es el mejor curso de programación PHP");

if ($reporte['valido']) {
    echo "Longitud: {$reporte['longitud']}\n";
    echo "Palabras: {$reporte['palabras']}\n";
    echo "Palabras clave: " . implode(', ', $reporte['palabras_clave']) . "\n";
    echo "Similitud: " . number_format($reporte['analisis']['similitud_referencia'] * 100, 2) . "%\n";
} else {
    echo "Error: {$reporte['error']}";
}
```

===============================================================================
10. BUENAS PRÁCTICAS CON VALORES DE RETORNO
===============================================================================

  • Declara siempre el tipo de retorno explícitamente.
  • Usa `void` para funciones sin retorno significativo.
  • Usa `never` para funciones que siempre lanzan excepción.
  • Prefiere funciones puras cuando sea posible.
  • Para errores, usa excepciones en lugar de códigos de error.
  • No uses `return` para múltiples valores no relacionados;
    usa arrays u objetos con nombres descriptivos.
  • Evita retornar tipos diferentes según el flujo (sé consistente).
  • Documenta el valor de retorno con PHPDoc `@return`.

===============================================================================
ERRORES COMUNES CON VALORES DE RETORNO
===============================================================================

Error 1: Olvidar el return:

```php
<?php

function obtener_precio(): float
{
    $precio = 100.50;
    // ❌ Olvidé return $precio;
}

echo obtener_precio(); // null (inesperado)
```

Error 2: Retornar tipo incorrecto:

```php
<?php

function buscar(int $id): array
{
    if ($id <= 0) {
        return null; // ❌ Debería ser array, no null
    }
    return ['id' => $id];
}
```

Error 3: No usar el valor de retorno:

```php
<?php

function limpiar_string(string $texto): string
{
    return trim($texto);
}

// ❌ Ignoramos el valor de retorno
limpiar_string("  Hola  ");
// El string original no se modifica
```

Error 4: Retornar null en lugar de lanzar excepción:

```php
<?php

// ❌ Ambiguo: ¿error o resultado válido?
function buscar_usuario(int $id): ?array
{
    //...
    return null; // ¿No existe? ¿Error de DB? ¿Credenciales?
}

// ✅ Mejor: excepción para errores, null para "no encontrado"
function buscar_usuario(int $id): ?array
{
    if ($id <= 0) {
        throw new InvalidArgumentException("ID inválido");
    }
    // null significa "no encontrado" (caso válido)
    return $usuarios[$id] ?? null;
}
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- `return` finaliza la función y devuelve un valor (o null).
- Las declaraciones de tipo de retorno (`: tipo`) hacen el código
  más seguro y autodocumentado.
- Los tipos nulables (`?tipo`) permiten retornar el tipo o null.
- `void` indica que la función no devuelve un valor significativo.
- `never` (PHP 8.1+) indica que la función nunca retorna normalmente.
- Las funciones puras son deterministas y sin efectos secundarios.
- Para múltiples valores, usa arrays asociativos u objetos DTO.
- Las funciones pueden retornar otras funciones (closures/fábricas).

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener múltiples return en una función?**

Sí, es válido y recomendable (early return). Múltiples returns
simplifican el flujo y reducen la anidación.

❓ **¿Qué diferencia hay entre `return` y `return null`?**

`return;` (sin valor) dentro de una función void es válido. En funciones
con tipo de retorno declarado, `return null;` solo es válido si el tipo
es nullable (`?tipo`).

❓ **¿Debo usar excepciones o códigos de error en el retorno?**

Usa excepciones para errores. El valor de retorno debe representar
el resultado exitoso de la operación. Los códigos de error como
valores de retorno son una práctica obsoleta.

❓ **¿Puedo retornar por referencia?**

Sí, pero es raro y generalmente no recomendado. Se declara con
`function &nombre(): tipo`. Úsalo solo cuando sepas exactamente
lo que haces.

❓ **¿Qué pasa si una función no tiene return?**

Si no hay `return`, la función devuelve `null`. Si el tipo de retorno
declarado no permite null, PHP lanza TypeError.
