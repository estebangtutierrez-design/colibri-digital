BIENVENIDA DEL PROFESOR BÚHO

Querido estudiante, ya sabes cómo guardar información en variables y
constantes. Pero, ¿qué tipo de información puedes almacenar? Números,
textos, booleanos, listas, objetos... cada tipo de dato tiene
características, usos y comportamientos específicos.

PHP es un lenguaje de tipado dinámico: no necesitas declarar el tipo de
una variable, el intérprete lo determina automáticamente según el valor
asignado. Sin embargo, desde PHP 7, puedes (y debes) declarar tipos
para hacer tu código más robusto y autodocumentado.

En este capítulo exploraremos todos los tipos de datos de PHP: los
escalares (int, float, string, bool), los compuestos (array, object), y
los especiales (null, resource, callable). También veremos el tipado
estricto, la conversión entre tipos y cómo aprovechar PHP 8 para
escribir código más seguro.

¡Abre tu mente a la diversidad de datos!

===============================================================================
1. TIPOS ESCALARES
===============================================================================

Los tipos escalares representan un solo valor. PHP tiene cuatro:

**1.1. int (Enteros)**

Números enteros positivos o negativos. El rango depende de la
arquitectura (64 bits: ±9.22 × 10¹⁸).

```php
<?php
$edad = 25;
$poblacion = 8_000_000_000;  // Separador visual (PHP 7.4+)
$temperatura = -10;
$hex = 0xFF;      // 255
$oct = 0777;      // 511
$bin = 0b1010;    // 10

echo PHP_INT_MAX;  // 9223372036854775807
echo PHP_INT_MIN;  // -9223372036854775808
echo PHP_INT_SIZE; // 8 (bytes)
?>
```

**1.2. float (Coma flotante)**

Números decimales. También llamados doubles.

```php
<?php
$precio = 19.99;
$pi = 3.14159265;
$iva = 0.16;
$notacion_cientifica = 1.5e3;  // 1500.0
$pequeno = 1.5e-3;             // 0.0015

// ⚠️ Precisión limitada
echo 0.1 + 0.2;  // 0.30000000000000004 (IEEE 754)
?>
```

**1.3. string (Cadenas de texto)**

Secuencias de caracteres. PHP soporta comillas simples y dobles.

```php
<?php
$nombre = 'Lucas';         // Comillas simples
$apellido = "EGA";         // Comillas dobles (interpreta variables)

// Comillas dobles: interpreta variables y caracteres de escape
$edad = 25;
echo "Me llamo $nombre y tengo $edad años";  // Interpolación
echo "Salto de línea:\nNueva línea";
echo "Tabulación:\tEspaciado";

// Comillas simples: todo literal
echo 'El nombre es $nombre';  // Muestra $nombre literal
echo 'Caracteres \\ n';        // No interpreta \n

// Heredoc (para bloques multilínea)
$texto = <<<TEXTO
Esto es un texto multilínea
con interpolación de $nombre
sin necesidad de concatenar.
TEXTO;

// Nowdoc (como heredoc pero sin interpolación)
$texto_literal = <<<'TEXTO'
Esto es un texto sin
interpolación de $nombre.
TEXTO;
?>
```

**1.4. bool (Booleanos)**

Representan verdadero o falso. Son clave para la lógica condicional.

```php
<?php
$activo = true;
$completado = false;

// Valores que se evalúan como false:
// 0, 0.0, "", "0", null, array(), false mismo
if (!$activo) {
    echo "Está inactivo";
}

// Comparación estricta (===) vs laxa (==)
var_dump(1 == true);    // bool(true)
var_dump(1 === true);   // bool(false) — tipos diferentes
?>
```

===============================================================================
2. TIPOS COMPUESTOS
===============================================================================

Los tipos compuestos pueden almacenar múltiples valores.

**2.1. array (Arreglos)**

Estructuras que almacenan colecciones de datos. PHP ofrece arrays
indexados y asociativos.

```php
<?php
// Array indexado (numérico)
$colores = ['rojo', 'verde', 'azul'];
echo $colores[0];  // rojo
echo $colores[2];  // azul

// Array asociativo (clave => valor)
$usuario = [
    'nombre' => 'Lucas',
    'edad' => 25,
    'email' => 'lucas@email.com'
];
echo $usuario['nombre'];  // Lucas

// Array multidimensional
$matriz = [
    ['nombre' => 'Ana', 'edad' => 30],
    ['nombre' => 'Luis', 'edad' => 28],
];
echo $matriz[0]['nombre'];  // Ana

// Funciones útiles para arrays
$numeros = [3, 1, 4, 1, 5, 9, 2, 6, 5];
echo count($numeros);    // 9
echo max($numeros);      // 9
echo min($numeros);      // 1
sort($numeros);          // Ordena el array
print_r($numeros);

// Array con spread operator (PHP 8.1+)
$parte1 = [1, 2, 3];
$parte2 = [4, 5, 6];
$completo = [...$parte1, ...$parte2];  // [1, 2, 3, 4, 5, 6]
?>
```

**2.2. object (Objetos)**

Instancias de clases. Los objetos son la base de la programación
orientada a objetos (POO).

```php
<?php
class Usuario {
    public function __construct(
        public string $nombre,
        public int $edad
    ) {}

    public function saludar(): string {
        return "Hola, soy {$this->nombre}";
    }
}

$user = new Usuario("Lucas", 25);
echo $user->saludar();  // Hola, soy Lucas

// Convertir array a objeto
$arr = ['nombre' => 'Ana', 'edad' => 30];
$obj = (object) $arr;
echo $obj->nombre;  // Ana
?>
```

===============================================================================
3. TIPOS ESPECIALES
===============================================================================

**3.1. null**

Representa la ausencia de valor. Es el valor por defecto de variables
no inicializadas.

```php
<?php
$variable = null;  // Asignación explícita

var_dump($variable);    // NULL
var_dump(is_null($variable));  // bool(true)
var_dump($variable === null);  // bool(true)

// Valores que retornan null
$resultado = buscarUsuario('id_inexistente');  // retorna null
$valor = $_GET['clave_inexistente'] ?? null;

// Nullable types (PHP 7.1+)
function procesar(?string $dato): void {
    if ($dato === null) {
        echo "No hay datos";
    } else {
        echo "Procesando: $dato";
    }
}

procesar(null);    // No hay datos
procesar("texto"); // Procesando: texto
?>
```

**3.2. resource**

Tipo especial que representa un recurso externo (archivo abierto, conexión
a base de datos). No es un tipo de dato que manipules directamente.

```php
<?php
$archivo = fopen('datos.txt', 'r');  // resource
var_dump($archivo);
// resource(5) of type (stream)

$conexion = mysqli_connect('localhost', 'root', '');  // resource
var_dump($conexion);

// Siempre cierra los recursos
fclose($archivo);
?>
```

**3.3. callable**

Tipo que representa una función, método o closure que puede ser invocada.

```php
<?php
function miFuncion(): void {
    echo "Función llamada";
}

$callable = 'miFuncion';
$callable();  // Función llamada

// Closure (función anónima)
$closure = function(string $nombre): string {
    return "Hola, $nombre";
};

echo $closure("Lucas");  // Hola, Lucas

// Callable con array (clase, método)
class Saludador {
    public static function saludar(string $nombre): string {
        return "¡Hola, $nombre!";
    }
}

$callable2 = ['Saludador', 'saludar'];
echo $callable2("Ana");  // ¡Hola, Ana!
?>
```

**3.4. mixed (PHP 8+)**

Tipo que acepta cualquier valor. Útil cuando realmente no puedes
restringir el tipo.

```php
<?php
function debug(mixed $dato): void {
    var_dump($dato);
}

debug("texto");
debug(123);
debug([1, 2, 3]);
debug(null);
?>
```

===============================================================================
4. TIPADO ESTRICTO (STRICT TYPES)
===============================================================================

PHP por defecto convierte tipos automáticamente (type juggling). Con
`declare(strict_types=1)`, forzas que los tipos coincidan exactamente.

**4.1. Sin tipos estrictos (modo coercitivo)**

```php
<?php
function sumar(int $a, int $b): int {
    return $a + $b;
}

echo sumar(5, "10");  // 15 (el string "10" se convierte a int)
?>
```

**4.2. Con tipos estrictos**

```php
<?php
declare(strict_types=1);

function sumar(int $a, int $b): int {
    return $a + $b;
}

echo sumar(5, "10");
// TypeError: sumar(): Argument #2 ($b) must be of type int, string given
?>
```

**4.3. Union types (PHP 8+)**

Permiten especificar múltiples tipos aceptables:

```php
<?php
declare(strict_types=1);

function procesar(int|string $dato): string {
    return match (true) {
        is_int($dato) => "Número: " . ($dato * 2),
        is_string($dato) => "Texto: " . strtoupper($dato),
    };
}

echo procesar(10);     // Número: 20
echo procesar("hola"); // Texto: HOLA
?>
```

===============================================================================
5. CONVERSIÓN DE TIPOS (CASTING)
===============================================================================

PHP permite convertir explícitamente un tipo a otro.

**5.1. Casting explícito**

```php
<?php
$numero = "42";

$entero = (int) $numero;        // 42
$decimal = (float) $numero;     // 42.0
$texto = (string) 42;           // "42"
$booleano = (bool) 1;           // true
$arreglo = (array) "texto";     // ["texto"]
$objeto = (object) ['clave' => 'valor'];  // object

// Uso de funciones intval, floatval, strval
$entero2 = intval("42");
$decimal2 = floatval("3.14");
$texto2 = strval(42);
?>
```

**5.2. Conversión implícita (type juggling)**

PHP convierte tipos automáticamente según el contexto:

```php
<?php
echo 5 + "10";          // 15 (string a int)
echo "5" * "2";         // 10 (strings a int)
echo 5 + 3.14;          // 8.14 (int a float)
echo "Hola" . 123;      // "Hola123" (int a string)
echo 5 + true;          // 6 (true = 1)
echo 5 + false;         // 5 (false = 0)

// ⚠️ Cuidado con las comparaciones laxas
var_dump(0 == "0");     // true
var_dump(0 == "");      // true (⚠️)
var_dump(0 == "texto"); // true (⚠️)
var_dump("" == false);  // true

// Siempre prefiere === (identidad)
var_dump(0 === "0");    // false
?>
```

===============================================================================
6. FUNCIONES DE VERIFICACIÓN DE TIPOS
===============================================================================

PHP ofrece funciones para verificar y obtener información de tipos:

```php
<?php
$valor = "Hello World";

echo gettype($valor);          // string
var_dump(is_int($valor));      // false
var_dump(is_string($valor));   // true
var_dump(is_numeric("42"));    // true (string numérico)
var_dump(is_numeric("abc"));   // false
var_dump(is_array([1,2]));     // true
var_dump(is_object(new stdClass())); // true
var_dump(is_null(null));       // true
var_dump(is_bool(true));       // true
var_dump(is_float(3.14));      // true
var_dump(is_callable('strlen')); // true

// PHP 8: get_debug_type() — más informativo
echo get_debug_type($valor);   // string (PHP 8)
?>
```

===============================================================================
7. ENUMERACIONES (PHP 8.1+)
===============================================================================

Las enumeraciones (Enums) son uno de los añadidos más importantes de
PHP 8.1. Permiten definir un conjunto fijo de valores posibles.

**7.1. Enums básicos (sin valor asociado)**

```php
<?php
enum Status
{
    case Pendiente;
    case EnProgreso;
    case Completada;
    case Cancelada;
}

function mostrarStatus(Status $status): string {
    return match ($status) {
        Status::Pendiente => '⏳ Pendiente',
        Status::EnProgreso => '🔄 En progreso',
        Status::Completada => '✅ Completada',
        Status::Cancelada => '❌ Cancelada',
    };
}

$tarea = Status::Pendiente;
echo mostrarStatus($tarea);  // ⏳ Pendiente
?>
```

**7.2. Enums con valor (backed enums)**

```php
<?php
enum Categoria: string
{
    case Trabajo = 'trabajo';
    case Personal = 'personal';
    case Estudio = 'estudio';
    case Salud = 'salud';
}

// Obtener el valor
echo Categoria::Trabajo->value;  // 'trabajo'

// Crear desde un valor
$cat = Categoria::from('personal');
echo $cat->name;  // Personal

// Listar todos los casos
$todas = Categoria::cases();
foreach ($todas as $categoria) {
    echo "$categoria->name: $categoria->value\n";
}
?>
```

**7.3. Enums con métodos**

```php
<?php
enum TipoUsuario: int
{
    case Admin = 1;
    case Editor = 2;
    case Usuario = 3;

    public function esAdmin(): bool {
        return $this === self::Admin;
    }

    public function label(): string {
        return match ($this) {
            self::Admin => 'Administrador',
            self::Editor => 'Editor de contenido',
            self::Usuario => 'Usuario estándar',
        };
    }
}

$rol = TipoUsuario::Admin;
var_dump($rol->esAdmin());  // true
echo $rol->label();  // Administrador
?>
```

===============================================================================
8. LA FUNCIÓN var_dump Y get_debug_type
===============================================================================

Para entender qué tipos tiene una variable, usa estas funciones:

```php
<?php
$datos = [
    'entero' => 42,
    'decimal' => 3.14,
    'texto' => 'PHP',
    'booleano' => true,
    'nulo' => null,
    'arreglo' => [1, 2, 3],
    'objeto' => (object) ['a' => 1],
];

foreach ($datos as $clave => $valor) {
    echo "$clave: ";
    echo "gettype → " . gettype($valor);
    echo " | get_debug_type → " . get_debug_type($valor);
    echo " | is_* → " . (is_int($valor) ? 'int' : '?');
    echo "\n";
}
?>
```

La diferencia entre `gettype()` y `get_debug_type()` (PHP 8+):
- `gettype()` retorna nombres genéricos (ej: "integer", "double").
- `get_debug_type()` retorna nombres más precisos (ej: "int", "float")
  y resuelve nombres de clase completos.

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- PHP tiene 10 tipos: int, float, string, bool (escalares); array,
  object (compuestos); null, resource, callable, mixed (especiales).
- Los arrays pueden ser indexados (numéricos) o asociativos (clave =>
  valor).
- Los objetos son instancias de clases y se pasan por referencia.
- `null` representa la ausencia de valor; los tipos nullable usan `?`.
- `declare(strict_types=1)` fuerza la coincidencia exacta de tipos.
- Union types (int|string) permiten múltiples tipos en parámetros.
- El casting explícito usa (int), (string), (bool), etc.
- Siempre prefiere `===` sobre `==` para evitar comparaciones peligrosas.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué 0.1 + 0.2 no es exactamente 0.3?**
Porque PHP sigue el estándar IEEE 754 para floats. Algunos decimales
no pueden representarse exactamente en binario. Usa `round()` o BC Math
para cálculos financieros.

❓ **¿Cuándo debo usar string vs heredoc?**
Usa heredoc/nowdoc para textos multilínea largos. Para líneas cortas,
usa comillas simples o dobles.

❓ **¿Qué significa `mixed` en PHP 8?**
`mixed` es un type alias que acepta cualquier tipo: int|float|string|
bool|null|array|object|callable|resource.

❓ **¿Debo siempre usar strict_types?**
Sí, en aplicaciones modernas. Hace el código más predecible y evita
errores sutiles de conversión automática.

❓ **¿Los objetos se pasan por valor o por referencia?**
Siempre por referencia. Al asignar un objeto a otra variable, ambas
apuntan al mismo objeto en memoria.

DESAFÍO DEL ARQUITECTO
===============================================================================

🏗️ **Validador Universal** — Diseña una clase o conjunto de funciones
que validen y conviertan datos de entrada según reglas configurables.
El sistema debe:

1. Recibir un array de datos (simulando $_POST).
2. Tener reglas de validación: required, type (int, float, string, bool),
   min, max, regex, email, enum.
3. Convertir automáticamente los valores al tipo especificado.
4. Retornar un array con los datos limpios y un array de errores.
5. Usar strict_types y union types.

Ejemplo de uso:
```php
$reglas = [
    'nombre' => 'required|string|min:3|max:50',
    'email' => 'required|email',
    'edad' => 'required|int|min:18|max:120',
    'activo' => 'bool',
];
$datos = validar($_POST, $reglas);
```
