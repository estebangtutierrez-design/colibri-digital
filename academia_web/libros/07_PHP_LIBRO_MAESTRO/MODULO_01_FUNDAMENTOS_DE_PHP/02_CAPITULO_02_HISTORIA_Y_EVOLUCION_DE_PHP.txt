1. LOS ORÍGENES: 1994-1995
===============================================================================

Todo comenzó con Rasmus Lerdorf, un programador danés que necesitaba
llevar el seguimiento de las visitas a su currículum vitae en línea.

**1.1. PHP/FI (Personal Home Page Tools)**

En 1994, Rasmus escribió un conjunto de scripts en Perl llamados
"Personal Home Page Tools". Estos scripts evolucionaron hasta convertirse
en un intérprete de plantillas que él llamó **PHP/FI** (Personal Home Page
/ Forms Interpreter).

```php
<!-- PHP/FI (1995) — El código más antiguo que existe -->
<!-- No ejecutes esto; es solo para apreciar la historia -->
<HTML>
<HEAD><TITLE>Mi página en 1995</TITLE></HEAD>
<BODY>
<?php
$visitas = 1;
echo "Has visitado esta página $visitas vez/veces.";
?>
</BODY>
</HTML>
```

PHP/FI ya incluía características básicas: soporte para formularios,
conexión a bases de datos (mSQL) y variables incrustadas en HTML.

**1.2. El nacimiento del nombre**

Originalmente, PHP significaba **P**ersonal **H**ome **P**age. En 1997,
cuando el lenguaje creció más allá de lo personal, se adoptó el acrónimo
recursivo **P**HP: **H**ypertext **P**reprocessor.

===============================================================================
2. PHP 3: EL PRIMER GRAN ÉXITO (1998)
===============================================================================

Zeev Suraski y Andi Gutmans, dos estudiantes israelíes, reescribieron el
núcleo de PHP/FI mientras estudiaban en el Technion. Nació **PHP 3**.

**2.1. La revolución del motor Zend**

Zeev y Andi crearon un nuevo motor de ejecución: **Zend Engine**
(acrónimo de Zeev + Andi). Este motor separó el análisis sintáctico de
la ejecución, sentando las bases para el crecimiento exponencial del
lenguaje.

PHP 3 fue el primero en parecerse al PHP que conocemos hoy:

```php
<?php
// PHP 3 (1998) — Sintaxis ya reconocible
function saludar($nombre) {
    echo "Hola, " . $nombre . "!";
}

saludar("Mundo");
?>
```

PHP 3 se distribuyó como código abierto y rápidamente fue adoptado por
miles de desarrolladores. Su integración con Apache mediante módulo
(mod_php) fue clave.

===============================================================================
3. PHP 4: MADUREZ Y ZEND ENGINE 1 (2000)
===============================================================================

PHP 4 trajo el **Zend Engine 1.0** y un salto significativo en
rendimiento y estabilidad.

**3.1. Características principales**

- Motor Zend Engine 1 dedicado.
- Soporte para sesiones nativas ($_SESSION).
- Variables superglobales ($_GET, $_POST, $_SERVER).
- Referencias y objetos más estables.
- Mejor soporte para programación orientada a objetos (aunque básico).

```php
<?php
// PHP 4 — Superglobales y sesiones
session_start();
$_SESSION['usuario'] = 'Lucas';

$nombre = $_GET['nombre'] ?? 'Invitado';
echo "Bienvenido, $nombre";
?>
```

PHP 4 fue el lenguaje que impulsó la primera oleada de blogs, foros y
sistemas de gestión de contenidos.

===============================================================================
4. PHP 5: LA ERA DE LOS OBJETOS (2004)
===============================================================================

PHP 5 fue una revolución. Incorporó el **Zend Engine 2** y un modelo de
objetos completamente renovado.

**4.1. Lo nuevo en PHP 5**

- Modelo de objetos moderno (clases, herencia, interfaces, abstractas).
- Excepciones (try/catch).
- PDO (PHP Data Objects) para acceso a bases de datos.
- MySQLi (mejor soporte para MySQL).
- Funciones anónimas (closures).
- Métodos mágicos (__construct, __destruct, __get, __set).

```php
<?php
// PHP 5 — Programación orientada a objetos
class Usuario {
    private string $nombre;

    public function __construct(string $nombre) {
        $this->nombre = $nombre;
    }

    public function saludar(): string {
        return "Hola, soy {$this->nombre}";
    }
}

$user = new Usuario("Ana");
echo $user->saludar();
?>
```

**4.2. El auge de los frameworks**

PHP 5 vio nacer a Symfony (2005), Zend Framework (2006) y CakePHP (2005).
Estos frameworks establecieron el patrón MVC (Modelo-Vista-Controlador)
como estándar de la industria.

===============================================================================
5. PHP 6: EL CAPÍTULO QUE NUNCA FUE (2005-2010)
===============================================================================

PHP 6 fue planeado como la versión que integraría soporte nativo para
Unicode. Sin embargo, el proyecto se volvió demasiado complejo y nunca
alcanzó un lanzamiento estable.

**5.1. Lecciones de PHP 6**

- La implementación de Unicode era demasiado intrusiva.
- Afectaba el rendimiento de todas las cadenas de texto.
- La comunidad se resistió al cambio radical.
- Finalmente, se abandonó el proyecto.

Las características de PHP 6 que sobrevivieron se integraron en PHP 5.3
y PHP 5.4. Este fracaso enseñó a los desarrolladores del núcleo que los
cambios deben ser incrementales, no revolucionarios.

```php
<?php
// Lo que sobrevivió de PHP 6 en PHP 5.3
// Namespaces
namespace App\Utils;

function fecha_actual(): string {
    return date('Y-m-d');
}
?>
```

===============================================================================
6. PHP 7: RENDIMIENTO Y NUEVO ZEND ENGINE 3 (2015)
===============================================================================

Después del fracaso de PHP 6, el equipo saltó directamente a PHP 7 con
el **Zend Engine 3** (reescrito por Dmitry Stogov). El resultado fue
espectacular.

**6.1. La revolución del rendimiento**

- Rendimiento hasta 2-3 veces más rápido que PHP 5.6.
- Reducción drástica del consumo de memoria.
- Declaración de tipos escalares (int, float, string, bool).
- Tipo de retorno para funciones.
- Operador de fusión null (??).
- Operador spaceship (<=>).
- Sentencias de devolución de tipo (declare(strict_types=1)).

```php
<?php
// PHP 7 — Tipos escalares y null coalescing
declare(strict_types=1);

function calcularIVA(float $precio): float {
    return $precio * 0.16;
}

$precio = $_GET['precio'] ?? 0;
echo "IVA: " . calcularIVA((float)$precio);
?>
```

PHP 7 devolvió la confianza en el lenguaje. Grandes empresas como
Facebook (con HHVM) comenzaron a migrar de vuelta a PHP nativo.

===============================================================================
7. PHP 8: LA ERA MODERNA (2020-2024)
===============================================================================

PHP 8, lanzado en noviembre de 2020, incorporó el **JIT (Just-In-Time
Compilation)** y decenas de características que modernizaron
radicalmente el lenguaje.

**7.1. Características destacadas de PHP 8.0**

- **JIT**: Compilación en tiempo real para código de uso intensivo.
- **Atributos (Attributes)**: Metadatos nativos para clases, métodos y
  propiedades.
- **Match expression**: Alternativa moderna a switch.
- **Constructor property promotion**: Menos código repetitivo.
- **Named arguments**: Argumentos con nombre en funciones.
- **Union types**: Tipos múltiples (int|float|string).
- **Nullsafe operator**: `?->` para llamadas seguras.

**7.2. PHP 8.1, 8.2, 8.3**

- **8.1**: Enumeraciones (Enums), Fibers (corrutinas), intersection types.
- **8.2**: Readonly classes, tipos disyuntivos puros, constantes en traits.
- **8.3**: Mejoras en JSON, métodos mágicos, serialización.

```php
<?php
// PHP 8 — Sintaxis moderna
declare(strict_types=1);

class Producto {
    public function __construct(
        private string $nombre,
        private float $precio
    ) {}

    public function aplicarDescuento(float $porcentaje): float {
        return $this->precio * (1 - $porcentaje / 100);
    }
}

enum Categoria: string {
    case ELECTRONICA = 'electrónica';
    case ROPA = 'ropa';
    case HOGAR = 'hogar';
}

$producto = new Producto('Laptop', 15000);
echo $producto->aplicarDescuento(10); // 13500
?>
```

9. PHP EN LA ACTUALIDAD: 2024-2026
===============================================================================

PHP 8.x ha revitalizado el lenguaje. Las estadísticas actuales muestran
un panorama sólido:

**9.1. Adopción de versiones**

- PHP 8.x: ~45% de los sitios web (creciendo rápido)
- PHP 7.x: ~40% (decreciendo, fin de soporte)
- PHP 5.x: ~15% (obsoleto, debe migrarse urgente)

**9.2. Frameworks más usados**

```php
<?php
$frameworks = [
    'Laravel' => ['creador' => 'Taylor Otwell', 'año' => 2011, 'versión' => 11],
    'Symfony' => ['creador' => 'Fabien Potencier', 'año' => 2005, 'versión' => 7],
    'CakePHP' => ['creador' => 'Larry E. Masters', 'año' => 2005, 'versión' => 5],
    'CodeIgniter' => ['creador' => 'EllisLab', 'año' => 2006, 'versión' => 4],
    'Yii' => ['creador' => 'Qiang Xue', 'año' => 2008, 'versión' => 2],
];

foreach ($frameworks as $nombre => $info) {
    echo "$nombre v{$info['versión']} por {$info['creador']} ({$info['año']})" . PHP_EOL;
}
?>
```

**9.3. Tendencias futuras**

- **PHP 8.4+**: Mejoras continuas en rendimiento JIT.
- **Fibers**: Programación asíncrona nativa sin librerías externas.
- **Type system**: Evolución hacia tipado más estricto y expresivo.
- **PHP + WASM**: Ejecutar PHP en el navegador vía WebAssembly.
- **PHP en IoT**: Uso creciente en dispositivos embebidos.

10. LECCIONES DE LA HISTORIA DE PHP PARA TU CARRERA
===============================================================================

La evolución de PHP nos deja enseñanzas valiosas para cualquier
desarrollador:

**10.1. La importancia de la comunidad**

PHP creció porque su comunidad contribuyó: documentación, extensiones,
frameworks y soporte en foros. Nunca subestimes el poder de compartir
conocimiento.

**10.2. La evolución incremental gana**

El fracaso de PHP 6 demostró que los cambios radicales no funcionan en
lenguajes establecidos. PHP 7 y 8 triunfaron porque introdujeron
mejoras graduales y compatibles hacia atrás.

**10.3. El rendimiento importa**

Zend Engine 3 (PHP 7) duplicó el rendimiento. JIT (PHP 8) lo acercó a
lenguajes compilados. PHP compite hoy en velocidad con Go y Java.

**10.4. No temas a los fracasos**

PHP 6 fue un fracaso, pero sus lecciones permitieron construir mejores
versiones. En tu carrera, los errores no son fracasos; son datos para
la siguiente iteración.

```php
<?php
// Las lecciones aplicadas a tu código
function leccionesHistoria(): array {
    return [
        'Se incremental, no radical' => 'Refactoriza poco a poco',
        'La comunidad es tu red' => 'Comparte y aprende',
        'El rendimiento importa' => 'Optimiza sin sacrificar claridad',
        'Aprende de los errores' => 'Cada bug es una lección',
    ];
}

print_r(leccionesHistoria());
?>
```

11. FIGURAS CLAVE EN LA HISTORIA DE PHP
===============================================================================

Detrás de PHP hay personas brillantes. Conócelas:

**11.1. Rasmus Lerdorf (1968-)**

Creador original de PHP. Nacido en Groenlandia, creció en Canadá y
estudió en la Universidad de Waterloo. En 1994 creó PHP/FI como un
conjunto de scripts Perl. Hoy trabaja en el desarrollo de PHP y da
conferencias sobre tecnología.

**11.2. Zeev Suraski (1975-)**

Cofundador de Zend Technologies. Israelí, estudiante del Technion,
reescribió PHP junto a Andi Gutmans. Creó el Zend Engine que impulsó
PHP 3 y 4. También es co-creador de PHP 7.

**11.3. Andi Gutmans (1975-)**

Cofundador de Zend Technologies. Compañero de Zeev en el Technion,
también reescribió PHP y creó el Zend Engine. Hoy es un referente en
la industria del software.

**11.4. Dmitry Stogov (1978-)**

Ingeniero ruso, principal desarrollador del Zend Engine 3 (PHP 7) y
del JIT de PHP 8. Sus contribuciones duplicaron el rendimiento del
lenguaje.

**11.5. Nikita Popov (1992-)**

Uno de los desarrolladores más prolíficos del núcleo de PHP.
Trabajó en PHP 7.4, 8.0 y 8.1 (incluyendo named arguments, match,
fibers, atributos). Luego se unió al equipo de LLVM.

**11.6. Taylor Otwell (1984-)**

Creador de Laravel, el framework PHP más popular. No contribuye al
núcleo de PHP, pero su trabajo ha definido cómo millones de
desarrolladores escriben PHP hoy.


12. EL LEGADO DE PHP Y SU IMPACTO CULTURAL
===============================================================================

PHP no solo ha sido importante técnicamente; también ha democratizado
la creación web:

**12.1. La democratización de la web**

Antes de PHP, crear un sitio web dinámico requería conocimientos
avanzados de Perl/CGI o costosos servidores comerciales (ASP de
Microsoft, ColdFusion de Adobe). PHP permitió que cualquier persona
con un hosting económico pudiera tener un sitio dinámico.

**12.2. La cultura del código abierto**

PHP fue uno de los primeros lenguajes en demostrar que el código
abierto podía ser profesional, seguro y escalable. Su éxito allanó
el camino para Python, Ruby, Node.js y otros.

**12.3. WordPress y la revolución de los CMS**

WordPress (PHP) impulsa el 43% de la web. Su facilidad de uso ha
permitido que millones de personas creen sitios web sin saber
programar, cambiando para siempre el panorama digital.

**12.4. Frameworks PHP que marcaron época**

- **CakePHP** (2005): Primer framework PHP con convenciones claras.
- **Symfony** (2005): Profesionalizó el desarrollo PHP empresarial.
- **Laravel** (2011): Hizo que PHP fuera "divertido" de nuevo.
- **Laminas** (antes Zend Framework): Estandarizó las prácticas
  corporativas.


PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué no existe PHP 6?**
Porque el proyecto de integración de Unicode fue abandonado. El equipo
decidió saltar a PHP 7 para marcar un nuevo comienzo.

❓ **¿Debo aprender PHP 5 si hay PHP 8?**
No. PHP 5 está obsoleto y sin soporte de seguridad. Aprende directamente
PHP 8.x. Este libro te enseñará sobre PHP 8.

❓ **¿Qué pasó con HHVM de Facebook?**
HHVM fue una máquina virtual alternativa para PHP. Con PHP 7 y sus
mejoras de rendimiento, Facebook migró de vuelta a PHP nativo.

❓ **¿Rasmus Lerdorf sigue desarrollando PHP?**
Ya no es el líder del proyecto. Hoy el desarrollo es gestionado por el
PHP Group y la comunidad.

❓ **¿Zend Engine y Zend Framework son lo mismo?**
No. Zend Engine es el motor de ejecución de PHP. Zend Framework es un
framework de desarrollo (hoy llamado Laminas).
