1. TU PRIMER PROGRAMA: "HOLA, MUNDO"
===============================================================================

Abre tu editor de texto favorito (VS Code, Sublime Text, PHPStorm) y
crea un archivo llamado `hola.php` dentro de la carpeta `htdocs` de
XAMPP (o `www` de Laragon).

```php
<?php
echo "¡Hola, Mundo desde PHP!";
?>
```

**1.1. Explicación línea por línea**

- `<?php`: Etiqueta de apertura que le dice al servidor "aquí comienza
  código PHP".
- `echo`: Instrucción (o constructo del lenguaje) que imprime texto.
- `"¡Hola, Mundo desde PHP!"`: Cadena de texto entre comillas.
- `;`: El punto y coma es obligatorio al final de cada instrucción.
- `?>`: Etiqueta de cierre (opcional al final del archivo).

**1.2. Ejecución**

Abre tu navegador y visita:
```
http://localhost/hola.php
```

Deberías ver:
```
¡Hola, Mundo desde PHP!
```

¡Felicidades! Acabas de ejecutar tu primer programa PHP.

2. LA ESTRUCTURA BÁSICA DE UN ARCHIVO PHP
===============================================================================

Un archivo PHP puede contener solo código PHP, o una mezcla de HTML y
PHP. La estructura general es:

```php
<?php
// Código PHP aquí
$nombre = "Lucas";
echo $nombre;
?>

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi página PHP</title>
</head>
<body>
    <h1>Bienvenido, <?php echo $nombre; ?></h1>
    <p>Esta página fue generada con PHP.</p>
</body>
</html>
```

**2.1. Reglas importantes**

1. Los archivos PHP deben tener extensión `.php`.
2. La etiqueta `<?php` puede ir al inicio o en medio del HTML.
3. La etiqueta de cierre `?>` es opcional. Si el archivo solo contiene
   PHP, se recomienda omitirla para evitar espacios en blanco no
   deseados.
4. Cada instrucción termina con punto y coma (`;`).

```php
<?php
// Sin etiqueta de cierre al final (recomendado)
echo "No necesito cerrar la etiqueta PHP";
```

===============================================================================
3. echo VS print
===============================================================================

Tanto `echo` como `print` sirven para mostrar texto en pantalla. Sin
embargo, tienen diferencias:

| Característica | echo | print |
|----------------|------|-------|
| Velocidad | Ligeramente más rápido | Un poco más lento |
| Valor de retorno | No retorna valor | Retorna 1 siempre |
| Sintaxis | echo "a", "b", "c"; | print "a"; |
| Uso común | 95% de los casos | 5% de los casos |

```php
<?php
// echo: puede recibir múltiples argumentos
echo "Hola ", "Mundo ", "PHP", "\n";

// print: solo un argumento, pero retorna 1
$resultado = print "Esto imprime y retorna 1";
echo "\nValor de retorno de print: " . $resultado;
?>
```

**Recomendación**: Usa `echo` siempre. Es más rápido, más flexible y es
el estándar en la comunidad.

===============================================================================
4. PHP DENTRO DE HTML
===============================================================================

PHP fue diseñado para incrustarse en HTML. Hay tres formas principales
de hacerlo:

**4.1. Bloques completos PHP + HTML**

```php
<!DOCTYPE html>
<html>
<head><title>Bloque PHP</title></head>
<body>
    <?php
    $hora = date('H');
    $saludo = $hora < 12 ? "Buenos días" : "Buenas tardes";
    ?>
    <h1><?php echo $saludo; ?></h1>
    <p>Son las <?php echo date('H:i:s'); ?></p>
</body>
</html>
```

**4.2. Echo abreviado (short tag)**

Si la directiva `short_open_tag` está habilitada, puedes usar `<?= ?>`
que es equivalente a `<?php echo ?>`:

```php
<!DOCTYPE html>
<html>
<head><title>Short Tag</title></head>
<body>
    <h1><?= "Bienvenido a PHP Master Book" ?></h1>
    <p>Versión de PHP: <?= phpversion() ?></p>
</body>
</html>
```

**Recomendación**: Usa `<?= ?>` solo para imprimir valores simples.
Para lógica compleja, usa bloques `<?php ... ?>`.

===============================================================================
5. RECIBIENDO DATOS DEL USUARIO CON $_GET
===============================================================================

Una de las tareas más comunes en PHP es recibir datos del usuario.
La variable superglobal `$_GET` contiene los parámetros enviados por
URL (método GET).

**5.1. Formulario con método GET**

Crea `saludo.php`:

```php
<!DOCTYPE html>
<html>
<head><title>Saludo Personalizado</title></head>
<body>
    <h1>Saludo PHP</h1>

    <form method="GET" action="">
        <label>¿Cómo te llamas?</label>
        <input type="text" name="nombre" required>
        <button type="submit">Saludar</button>
    </form>

    <?php
    if (isset($_GET['nombre'])) {
        $nombre = htmlspecialchars($_GET['nombre']);
        echo "<h2>¡Hola, $nombre! Bienvenido a PHP.</h2>";
    }
    ?>
</body>
</html>
```

**5.2. Explicación**

- `isset($_GET['nombre'])`: Verifica si el parámetro 'nombre' fue enviado.
- `htmlspecialchars()`: Convierte caracteres especiales a entidades HTML
  (medida de seguridad contra XSS).
- `$_GET['nombre']`: Contiene el valor enviado en la URL.

Si visitas: `http://localhost/saludo.php?nombre=Ana`
Verás: **¡Hola, Ana! Bienvenido a PHP.**

===============================================================================
6. LA VARIABLE SUPERGLOBAL $_SERVER
===============================================================================

`$_SERVER` es otra superglobal que contiene información del servidor y la
petición actual. Es una mina de oro para depuración y desarrollo.

```php
<?php
echo "<h1>Información del Servidor</h1>";
echo "<ul>";
echo "<li>Dirección IP del visitante: " . $_SERVER['REMOTE_ADDR'] . "</li>";
echo "<li>Navegador: " . $_SERVER['HTTP_USER_AGENT'] . "</li>";
echo "<li>Método de petición: " . $_SERVER['REQUEST_METHOD'] . "</li>";
echo "<li>Script actual: " . $_SERVER['SCRIPT_FILENAME'] . "</li>";
echo "<li>Protocolo: " . $_SERVER['SERVER_PROTOCOL'] . "</li>";
echo "</ul>";
?>
```

**6.1. Las superglobales más importantes**

| Variable | Contenido |
|----------|-----------|
| $_GET | Parámetros enviados por URL |
| $_POST | Datos enviados por formulario POST |
| $_SERVER | Información del servidor y petición |
| $_SESSION | Datos de sesión del usuario |
| $_COOKIE | Cookies almacenadas en el navegador |
| $_FILES | Archivos subidos por el usuario |

===============================================================================
7. DEPURACIÓN BÁSICA: ERRORES COMUNES
===============================================================================

Cuando algo sale mal, PHP te ayuda con mensajes de error. Los más
comunes son:

**7.1. Parse error (error de sintaxis)**

```php
<?php
echo "Falta el punto y coma"  // ⚠️ Error: missing semicolon
?>
```

**Solución**: Agrega `;` al final de la línea.

**7.2. Undefined variable**

```php
<?php
echo $nombre;  // ⚠️ Warning: Undefined variable
?>
```

**Solución**: Define la variable antes de usarla: `$nombre = "Lucas";`

**7.3. Headers already sent**

```php
<?php
echo "Texto";
header("Location: otra-pagina.php");  // ⚠️ Error
?>
```

**Solución**: No imprimas nada antes de `header()`. O usa `ob_start()`.

```php
<?php
// Reportar todos los errores (útil en desarrollo)
error_reporting(E_ALL);
ini_set('display_errors', '1');

// Script con errores adrede para practicar
$nombre = "Lucas";
echo "Hola, " . $nombre;

// Warning por división entre cero (controlada)
$resultado = @(10 / 0);  // El @ suprime el warning
?>
```

8. EJEMPLO COMPLETO: GENERADOR DE SALUDOS
===============================================================================

Vamos a crear un programa interactivo que combina todo lo aprendido:

```php
<?php
//--------------------------------------------------------------
// GENERADOR DE SALUDOS PERSONALIZADOS
//--------------------------------------------------------------
$titulo = "Generador de Saludos";
$ano_actual = date('Y');
$mensajes_bienvenida = [

    "El backend te espera"

<!DOCTYPE html>

        .mensaje { background: #f0f8ff; padding: 20px; border-radius: 8px; }

    </style>
</head>
<body>
    <h1><?= $titulo ?></h1>

    <form method="GET" action="">
        <label>Tu nombre:</label>
        <input type="text" name="nombre" value="<?= htmlspecialchars($nombre) ?>">

        <label>Tipo de saludo:</label>
        <select name="saludo">
            <option value="Hola" <?= $saludo === 'Hola' ? 'selected' : '' ?>>Hola</option>
            <option value="Bienvenido" <?= $saludo === 'Bienvenido' ? 'selected' : '' ?>>Bienvenido</option>

        </select>

        <div class="mensaje">

            <p>Tu IP: <?= $_SERVER['REMOTE_ADDR'] ?></p>
        </div>
    <?php endif; ?>

    <p><small>&copy; <?= $ano_actual ?> - PHP Master Book</small></p>

```

Este programa demuestra:
- Mezcla de PHP y HTML
- Uso de variables y arrays
- Formularios con método GET
- Seguridad básica con htmlspecialchars()
- Fechas y horas con date()
- Variables superglobales

8. EL CICLO PETICIÓN-RESPUESTA EN DETALLE
===============================================================================

Entender cómo fluye la información es clave para depurar y optimizar.

**8.1. Anatomía de una petición HTTP**

Cuando el navegador solicita `http://localhost/saludo.php?nombre=Ana`:

```php
<?php
// Veamos qué contiene $_SERVER durante una petición típica
echo "<pre>";
echo "Método: " . $_SERVER['REQUEST_METHOD'] . "\n";       // GET
echo "URI: " . $_SERVER['REQUEST_URI'] . "\n";              // /saludo.php?nombre=Ana
echo "Query String: " . ($_SERVER['QUERY_STRING'] ?? '') . "\n"; // nombre=Ana
echo "Script: " . $_SERVER['SCRIPT_NAME'] . "\n";            // /saludo.php
echo "Host: " . $_SERVER['HTTP_HOST'] . "\n";                // localhost
echo "User-Agent: " . $_SERVER['HTTP_USER_AGENT'] . "\n";    // Chrome/Firefox...
echo "IP Cliente: " . $_SERVER['REMOTE_ADDR'] . "\n";        // 127.0.0.1
echo "</pre>";
?>
```

**8.2. Respuesta del servidor**

PHP procesa el script y genera una respuesta que incluye:

1. **Headers HTTP**: Tipo de contenido, cookies, estado.
2. **Body**: El HTML generado (o JSON, XML, etc.).

```php
<?php
// Personalizar headers de respuesta
header('Content-Type: text/html; charset=utf-8');
header('X-Powered-By: PHP Master Book');

// Códigos de estado HTTP
// header('HTTP/1.1 404 Not Found');
// header('HTTP/1.1 403 Forbidden');
// header('HTTP/1.1 500 Internal Server Error');

echo "Respuesta generada con headers personalizados";
?>
```

===============================================================================
9. INCLUDES Y REUTILIZACIÓN DE CÓDIGO
===============================================================================

PHP permite incluir archivos dentro de otros, facilitando la
reutilización y organización.

**9.1. include y require**

```php
<?php
// header.php — Fragmento reutilizable
$titulo_sitio = "PHP Master Book";
echo "<header><h1>$titulo_sitio</h1></header>";
?>

<?php
// index.php — Página principal
// include: Si el archivo no existe, muestra warning y continúa
include 'header.php';

// require: Si el archivo no existe, lanza error fatal
require 'config.php';

// include_once/require_once: Solo incluye una vez (evita duplicados)
require_once 'funciones.php';
?>
```

**9.2. Estructura de proyecto con includes**

```php
<?php
// Una estructura típica con includes:
// index.php
require_once 'config/database.php';
require_once 'includes/funciones.php';
require_once 'includes/auth.php';
$pageTitle = "Inicio";
include 'templates/header.php';
include 'templates/nav.php';
include 'templates/content.php';
include 'templates/footer.php';
?>
```

10. EJEMPLO COMPLETO: CALCULADORA WEB
===============================================================================

Creemos una calculadora completa que integra formularios, operaciones
matemáticas y validación:

```php
<?php
// calculadora.php
$num1 = $_GET['num1'] ?? '';
$num2 = $_GET['num2'] ?? '';
$operacion = $_GET['operacion'] ?? 'suma';
$resultado = '';
$error = '';

if ($num1 !== '' && $num2 !== '') {
    if (is_numeric($num1) && is_numeric($num2)) {
        $n1 = (float) $num1;
        $n2 = (float) $num2;

        match ($operacion) {
            'suma' => $resultado = $n1 + $n2,
            'resta' => $resultado = $n1 - $n2,
            'multiplicacion' => $resultado = $n1 * $n2,
            'division' => $n2 != 0
                ? $resultado = $n1 / $n2
                : $error = "No se puede dividir entre cero",
            'potencia' => $resultado = $n1 ** $n2,
            default => $error = "Operación no válida",
        };

        if ($resultado !== '') {
            $resultado = round($resultado, 2);
        }
    } else {
        $error = "Ambos valores deben ser números";
    }
}
?>

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Calculadora PHP</title>
    <style>
        body { font-family: Arial; max-width: 500px; margin: 50px auto; }
        .calc { background: #f5f5f5; padding: 20px; border-radius: 8px; }
        input, select, button { padding: 8px; margin: 5px; font-size: 1em; }
        .resultado { background: #e3f2fd; padding: 15px; border-radius: 6px; margin-top: 15px; }
        .error { background: #ffebee; padding: 15px; border-radius: 6px; margin-top: 15px; color: #c62828; }
    </style>
</head>
<body>
    <div class="calc">
        <h1>🧮 Calculadora PHP</h1>
        <form method="GET" action="">
            <input type="number" step="any" name="num1"
                   value="<?= htmlspecialchars($num1) ?>" placeholder="Número 1" required>
            <input type="number" step="any" name="num2"
                   value="<?= htmlspecialchars($num2) ?>" placeholder="Número 2" required>
            <select name="operacion">
                <option value="suma" <?= $operacion === 'suma' ? 'selected' : '' ?>>+ Suma</option>
                <option value="resta" <?= $operacion === 'resta' ? 'selected' : '' ?>>- Resta</option>
                <option value="multiplicacion" <?= $operacion === 'multiplicacion' ? 'selected' : '' ?>>× Multiplicación</option>
                <option value="division" <?= $operacion === 'division' ? 'selected' : '' ?>>÷ División</option>
                <option value="potencia" <?= $operacion === 'potencia' ? 'selected' : '' ?>>^ Potencia</option>
            </select>
            <button type="submit">Calcular</button>
        </form>

        <?php if ($error): ?>
            <div class="error"><?= htmlspecialchars($error) ?></div>
        <?php elseif ($resultado !== ''): ?>
            <div class="resultado">
                <strong>Resultado:</strong>
                <?= htmlspecialchars($num1) ?>
                <?= match($operacion) {
                    'suma' => '+',
                    'resta' => '-',
                    'multiplicacion' => '×',
                    'division' => '÷',
                    'potencia' => '^',
                } ?>
                <?= htmlspecialchars($num2) ?>
                =
                <strong><?= htmlspecialchars((string)$resultado) ?></strong>
            </div>
        <?php endif; ?>
    </div>
    <p><small>Creada con PHP <?= phpversion() ?> — <?= date('Y') ?></small></p>
</body>
</html>
```

Este ejemplo integra:
- Formularios con método GET
- Validación de datos numéricos
- Match expression de PHP 8
- Manejo de errores (división entre cero)
- Formato de resultados con round()
- Persistencia de valores en el formulario
- Diseño básico con CSS embebido

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué veo el código PHP en el navegador en lugar de ejecutarse?**
Significa que Apache no está interpretando PHP. Verifica que Apache esté
corriendo y que el archivo tenga extensión `.php`.

❓ **¿Es obligatorio usar `htmlspecialchars()`?**
Siempre que muestres datos ingresados por el usuario, sí. Es la primera
línea de defensa contra ataques XSS.

❓ **¿Puedo usar `echo` con etiquetas HTML dentro de las comillas?**
Sí, `echo "<h1>Título</h1>"` es válido. Pero es mejor separar la lógica
PHP de la presentación HTML.

❓ **¿Qué significa `$_GET['nombre'] ?? ''` ?**
Es el operador de fusión null (PHP 7+). Devuelve el valor de
`$_GET['nombre']` si existe, o una cadena vacía si no.

❓ **¿Y si quiero enviar datos sin que se vean en la URL?**
Usa el método POST en el formulario y `$_POST` en PHP. Lo veremos en
capítulos posteriores.
