1. QUÉ ES PHP
===============================================================================

PHP (acrónimo recursivo de **PHP: Hypertext Preprocessor**) es un lenguaje
de programación interpretado, diseñado originalmente para el desarrollo web
del lado del servidor. Su código se ejecuta en el servidor web, generando
HTML dinámico que luego se envía al navegador del cliente.

A diferencia de JavaScript (que se ejecuta en el navegador), PHP trabaja
tras bambalinas: procesa formularios, accede a bases de datos, gestiona
sesiones de usuario y construye páginas completas antes de que el usuario
las vea.

```php
<?php
// Este es tu primer vistazo a PHP
echo "¡Hola, mundo desde el servidor!";
?>
```

El código PHP se incrusta dentro de HTML gracias a las etiquetas
`<?php ... ?>`. Cuando el servidor web recibe una petición, interpreta
todo lo que esté dentro de estas etiquetas y devuelve el resultado como
HTML puro.

===============================================================================
2. PHP EN EL ECOSISTEMA WEB
===============================================================================

Para entender PHP, primero debemos entender el modelo cliente-servidor:

- **Cliente**: El navegador (Chrome, Firefox, etc.) que solicita una página.
- **Servidor**: Una computadora remota que almacena archivos PHP y los
  procesa antes de enviar el resultado al cliente.

Cuando escribes una URL en tu navegador, ocurre lo siguiente:

1. El navegador envía una petición HTTP al servidor.
2. El servidor web (Apache o Nginx) recibe la solicitud.
3. Si el archivo es `.php`, el servidor invoca al intérprete de PHP.
4. PHP ejecuta el código, genera HTML puro y lo devuelve al servidor.
5. El servidor envía esa respuesta HTML al navegador.
6. El navegador renderiza la página.

Este modelo permite crear experiencias dinámicas: el contenido puede ser
diferente para cada usuario, cada visita o cada momento del día.

3. VENTAJAS DE PHP
===============================================================================

PHP ha sobrevivido y prosperado durante más de dos décadas por razones
sólidas:

**3.1. Curva de aprendizaje suave**
Su sintaxis es clara y está inspirada en C y Perl. Puedes escribir código
productivo desde tu primer día.

**3.2. Multiplataforma**
Funciona en Windows, Linux, macOS y la mayoría de servidores comerciales.

**3.3. Amplia documentación y comunidad**
La documentación oficial (php.net) es excelente, y comunidades como
Stack Overflow tienen millones de preguntas resueltas.

**3.4. Integración con bases de datos**
Soporta nativamente MySQL, PostgreSQL, SQLite, MongoDB y muchas más.

**3.5. Frameworks poderosos**
Laravel, Symfony, CodeIgniter, CakePHP — ecosistema maduro y productivo.

**3.6. Bajo costo de hosting**
Es el lenguaje más soportado por proveedores de hosting compartido.

**3.7. Rendimiento mejorado**
Con PHP 8.x, el rendimiento es comparable a lenguajes compilados gracias
a JIT (Just-In-Time compilation).

8. EL ECOSISTEMA PHP HOY
===============================================================================

PHP ha evolucionado enormemente desde su creación en 1994 por Rasmus
Lerdorf. Hoy, PHP 8.x incluye:

- **JIT (Just-In-Time Compilation)**: Compila código en tiempo real para
  acelerar la ejecución.
- **Tipos estrictos**: Declaración de tipos para parámetros y retornos.
- **Atributos (annotations)**: Metadatos nativos para clases y métodos.
- **Match expression**: Sintaxis moderna para condicionales múltiples.
- **Constructor property promotion**: Menos código boilerplate.

```php
<?php
// PHP 8: Sintaxis moderna con tipos estrictos
declare(strict_types=1);

function sumar(int $a, int $b): int {
    return $a + $b;
}

echo sumar(5, 3); // 8
?>
```

PHP moderno se escribe con tipado, orientación a objetos y frameworks
robustos. Olvida el "PHP espagueti" del pasado; hoy es un lenguaje
profesional y elegante.

11. PHP Y LA ARQUITECTURA MVC
===============================================================================

PHP moderno se escribe siguiendo patrones de diseño. El más importante
es **MVC (Modelo-Vista-Controlador)**:

**11.1. ¿Qué es MVC?**

- **Modelo**: Gestiona los datos y la lógica de negocio (base de datos,
  validación, reglas).
- **Vista**: Presenta la información al usuario (HTML, CSS, JSON).
- **Controlador**: Recibe las peticiones, coordina el modelo y la vista.

```php
<?php
// Controlador: Recibe la petición y decide qué hacer
$accion = $_GET['accion'] ?? 'inicio';

if ($accion === 'usuarios') {
    $controlador = new UsuarioController();
    $controlador->listar();
}

// Modelo: Maneja los datos
class UsuarioModelo {
    public function obtenerTodos(): array {
        // Aquí iría la consulta a la base de datos
        return [['id' => 1, 'nombre' => 'Lucas']];
    }
}

// Vista: Presenta los datos
function renderizarVista(string $nombre, array $datos): void {
    extract($datos);
    include "views/$nombre.php";
}
?>
```

12. PHP Y EL FUTURO DEL DESARROLLO WEB
===============================================================================

PHP no solo sigue vigente; está en una posición estratégica para el
futuro del desarrollo web:

**12.1. PHP + JavaScript: El dúo dinámico**

PHP se usa como backend de APIs que consumen frontends modernos en
React, Vue o Svelte. Es la combinación más común en la industria.

```php
<?php
// API REST en PHP que sirve JSON a un frontend React
header('Content-Type: application/json');
header('Access-Control-Allow-Origin: *');

    ['id' => 1, 'nombre' => 'Lucas', 'email' => 'lucas@email.com'],
    ['id' => 2, 'nombre' => 'Ana', 'email' => 'ana@email.com'],
];

echo json_encode($usuarios, JSON_PRETTY_PRINT);
?>
```

Laravel Livewire permite crear interfaces dinámicas escribiendo solo

**12.3. PHP + APIs: El backend universal**

PHP es excelente construyendo APIs que alimentan apps móviles, IoT y
servicios web:

```php
<?php
// API de clima simulada
function obtenerClima(string $ciudad): array {
    $datos = [
        'ciudad' => $ciudad,
        'temperatura' => rand(15, 35),
        'humedad' => rand(40, 90),
        'condicion' => ['Soleado', 'Nublado', 'Lluvia'][rand(0, 2)],
        'actualizado' => date('Y-m-d H:i:s'),
    ];
    return $datos;
}

header('Content-Type: application/json');
$ciudad = $_GET['ciudad'] ?? 'Ciudad de México';
echo json_encode(obtenerClima($ciudad), JSON_PRETTY_PRINT);
?>
```

13. CONFIGURACIÓN DE php.ini PARA EMPEZAR
===============================================================================

El archivo `php.ini` es el centro de configuración de PHP. Conocerlo
es parte de ser un buen desarrollador:

**13.1. Directivas importantes**

```ini
; php.ini — Configuraciones recomendadas para desarrollo
display_errors = On
error_reporting = E_ALL
memory_limit = 256M
upload_max_filesize = 64M
post_max_size = 68M
max_execution_time = 300
date.timezone = America/Mexico_City
short_open_tag = Off
```

**13.2. Verificar la configuración**

```php
<?php
// Mostrar configuraciones clave
$configs = [
    'display_errors' => ini_get('display_errors'),
    'memory_limit' => ini_get('memory_limit'),
    'upload_max_filesize' => ini_get('upload_max_filesize'),
    'post_max_size' => ini_get('post_max_size'),
    'max_execution_time' => ini_get('max_execution_time'),
    'date.timezone' => ini_get('date.timezone'),
    'short_open_tag' => ini_get('short_open_tag'),
];

echo "<table border='1'>";
echo "<tr><th>Directiva</th><th>Valor</th></tr>";
foreach ($configs as $directiva => $valor) {
    echo "<tr><td>$directiva</td><td>" . ($valor ?: 'No configurado') . "</td></tr>";
}
echo "</table>";
?>
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Necesito saber HTML antes de aprender PHP?**
Sí, es recomendable tener nociones básicas de HTML, ya que PHP genera
contenido HTML. Si no lo conoces, te sugieres estudiarlo paralelamente.

❓ **¿PHP está muriendo?**
No. PHP impulsa más del 75% de la web. WordPress, Laravel y Drupal
siguen siendo líderes. Lo que "muere" es el mal código, no el lenguaje.

❓ **¿Puedo aprender PHP sin un servidor web?**
Puedes usar PHP CLI (línea de comandos) para scripts simples, pero para
desarrollo web necesitarás Apache o Nginx. XAMPP te dará ambos.

❓ **¿Qué versión de PHP debo usar?**
Siempre la más reciente estable (PHP 8.3+). Las versiones antiguas (5.x,
7.x) pierden soporte de seguridad.

❓ **¿PHP sirve para aplicaciones móviles?**
PHP se usa como backend para apps móviles a través de APIs REST. La app
móvil se comunica con el servidor PHP mediante JSON.
