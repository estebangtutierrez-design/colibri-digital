1. VARIABLES EN PHP
===============================================================================

En PHP, las variables comienzan con el símbolo `$` seguido del nombre.
No necesitas declarar el tipo; PHP lo infiere automáticamente.

**1.1. Declaración y asignación básica**

```php
<?php
// Asignación básica
$nombre = "Lucas";
$edad = 25;
$altura = 1.75;
$es_estudiante = true;

echo "$nombre tiene $edad años y mide $altura metros.";
?>
```

**1.2. Reglas de nomenclatura**

- Comienzan con `$` seguido de una letra o guion bajo.
- Solo contienen letras, números y guiones bajos (a-z, A-Z, 0-9, _).
- No pueden comenzar con un número.
- Son case-sensitive (`$nombre` ≠ `$Nombre`).
- Deben ser descriptivas y significativas.

```php
<?php
// ✅ Válidos
$nombre = "Lucas";
$_edad = 25;
$miVariableLarga = "valor";
$nombreCompletoDelUsuario = "Lucas EGA";

// ❌ Inválidos
// $123nombre = "error";  // Comienza con número
// $mi-variable = "error"; // Guión no permitido
// $nombre completo = "error"; // Espacios no permitidos

// ⚠️ Válidos pero no recomendados
$n = "Lucas";           // No descriptivo
$var1 = "dato";         // No dice qué contiene
?>
```

===============================================================================
2. ASIGNACIÓN POR VALOR VS POR REFERENCIA
===============================================================================

PHP ofrece dos formas de asignar variables: por valor (copia) y por
referencia (apunta al mismo lugar en memoria).

**2.1. Asignación por valor**

Por defecto, PHP asigna por valor: se crea una copia independiente.

```php
<?php
$a = 10;
$b = $a;  // $b obtiene una copia del valor de $a
$b = 20;  // Cambiar $b no afecta a $a

echo "a = $a, b = $b";  // a = 10, b = 20
?>
```

**2.2. Asignación por referencia**

Con el operador `&`, ambas variables apuntan al mismo valor.

```php
<?php
$a = 10;
$b = &$a;  // $b es una referencia a $a
$b = 20;   // Cambiar $b también cambia $a

echo "a = $a, b = $b";  // a = 20, b = 20

// También funciona con objetos (siempre por referencia)
class Persona {
    public string $nombre;
}

$p1 = new Persona();
$p1->nombre = "Lucas";

$p2 = $p1;  // $p2 apunta al mismo objeto
$p2->nombre = "Ana";

echo $p1->nombre;  // Ana (se modificó el mismo objeto)
?>
```

===============================================================================
3. ÁMBITO DE LAS VARIABLES (SCOPE)
===============================================================================

El ámbito (scope) determina dónde una variable es accesible dentro del
código.

**3.1. Variables locales**

Declaradas dentro de una función, solo existen allí:

```php
<?php
function saludar(): void {
    $mensaje = "Hola desde la función";
    echo $mensaje;
}

saludar();         // ✅ Imprime el mensaje
// echo $mensaje;  // ❌ Undefined variable (fuera de la función)
?>
```

**3.2. Variables globales**

Declaradas fuera de funciones, accesibles con `global`:

```php
<?php
$nombre_global = "Lucas";

function mostrarNombre(): void {
    global $nombre_global;  // 👈 Declarar como global
    echo $nombre_global;
}

mostrarNombre();  // Lucas
?>
```

**3.3. El array $GLOBALS**

PHP almacena todas las variables globales en el array asociativo
`$GLOBALS`:

```php
<?php
$nombre = "Lucas";
$edad = 25;

function mostrarDatos(): void {
    echo $GLOBALS['nombre'] . " tiene " . $GLOBALS['edad'] . " años";
}

mostrarDatos();  // Lucas tiene 25 años
?>
```

**3.4. Variables estáticas**

Mantienen su valor entre llamadas a la función:

```php
<?php
function contador(): int {
    static $contador = 0;  // 👈 Se inicializa solo la primera vez
    $contador++;
    return $contador;
}

echo contador();  // 1
echo contador();  // 2
echo contador();  // 3
?>
```

===============================================================================
4. VARIABLES VARIABLES ($$)
===============================================================================

PHP permite usar el valor de una variable como nombre de otra variable:

```php
<?php
$nombre_variable = "mensaje";
$$nombre_variable = "Hola, esto es dinámico";  // Crea $mensaje

echo $mensaje;  // Hola, esto es dinámico
echo ${$nombre_variable};  // Misma salida

// Ejemplo práctico: configuración dinámica
$tipo = "db";
$$tipo = [
    'host' => 'localhost',
    'user' => 'root',
];

print_r($db);
?>
```

**¡Advertencia!** Las variables variables pueden hacer tu código difícil
de leer y depurar. Úsalas con moderación.

===============================================================================
5. CONSTANTES
===============================================================================

Las constantes son valores que no cambian durante la ejecución del
programa. En PHP hay dos formas de definirlas: `define()` y `const`.

**5.1. define() — La forma tradicional**

```php
<?php
define('NOMBRE_APP', 'PHP Master Book');
define('VERSION', '1.0.0');
define('MAX_INTENTOS', 3);
define('DEBUG_MODE', true);

echo NOMBRE_APP;  // PHP Master Book (sin $)
?>
```

**5.2. const — La forma moderna (PHP 7+)**

```php
<?php
// const solo puede usarse en el ámbito de clase o archivo
const NOMBRE_APP = 'PHP Master Book';
const VERSION = '1.0.0';
const MAX_INTENTOS = 3;

class Config {
    public const NOMBRE = 'PHP Master Book';
    public const VERSION = '1.0.0';
}

echo Config::NOMBRE;  // PHP Master Book
?>
```

**5.3. Diferencias entre define() y const**

| Característica | define() | const |
|----------------|----------|-------|
| Disponible desde | PHP 4 | PHP 5.3 (archivo), PHP 7 (expresiones) |
| Ámbito | En cualquier lugar | Solo en ámbito de clase o raíz |
| Expresiones dinámicas | Sí (define('X', 2+2)) | Sí (PHP 7+, const X = 2+2) |
| Arrays | Sí | Sí |
| Rendimiento | Ligeramente más lento | Ligeramente más rápido |

**5.4. Constantes predefinidas de PHP**

PHP incluye constantes útiles que puedes usar directamente:

```php
<?php
echo PHP_VERSION;       // 8.2.12
echo PHP_OS;            // Linux, WINNT, Darwin
echo PHP_INT_MAX;       // 9223372036854775807
echo PHP_INT_SIZE;      // 8
echo PHP_EOL;           // Caracter de nueva línea (\n)
echo DIRECTORY_SEPARATOR; // / (Linux) o \ (Windows)

// Constantes mágicas (cambian según el contexto)
echo __LINE__;           // Línea actual del archivo
echo __FILE__;           // Ruta completa del archivo actual
echo __DIR__;            // Directorio del archivo actual
echo __FUNCTION__;       // Nombre de la función actual
echo __CLASS__;          // Nombre de la clase actual
echo __METHOD__;         // Nombre del método actual
echo __NAMESPACE__;      // Namespace actual
?>
```

===============================================================================
6. SUPERGLOBALES
===============================================================================

Las superglobales son arrays asociativos que PHP pone a tu disposición
en todos los ámbitos. Están siempre disponibles, sin necesidad de
declararlas como globales.

**6.1. Las 9 superglobales de PHP**

```php
<?php
// $GLOBALS — Todas las variables globales
$GLOBALS['variable'] = "valor";

// $_SERVER — Información del servidor
echo $_SERVER['SERVER_NAME'];

// $_GET — Parámetros de la URL
echo $_GET['id'] ?? '';

// $_POST — Datos de formularios POST
echo $_POST['email'] ?? '';

// $_FILES — Archivos subidos
// $_FILES['archivo']['name'];

// $_COOKIE — Cookies del navegador
echo $_COOKIE['sesion'] ?? '';

// $_SESSION — Datos de sesión
session_start();
$_SESSION['usuario_id'] = 1;

// $_REQUEST — Combinación de GET, POST y COOKIE
echo $_REQUEST['nombre'] ?? '';

// $_ENV — Variables de entorno
echo $_ENV['APP_ENV'] ?? 'production';
?>
```

**6.2. Uso correcto de superglobales**

```php
<?php
// ✅ Siempre valida que existan
$nombre = isset($_GET['nombre']) ? $_GET['nombre'] : 'Invitado';

// ✅ Con null coalescing (PHP 7+)
$nombre = $_GET['nombre'] ?? 'Invitado';

// ✅ Sanitiza los datos
$email = filter_var($_POST['email'] ?? '', FILTER_SANITIZE_EMAIL);

// ⚠️ Nunca confíes en los datos del usuario
// ❌ echo $_GET['nombre'];  // Peligroso (XSS)
// ✅ echo htmlspecialchars($_GET['nombre']);  // Seguro
?>
```

7. BUENAS PRÁCTICAS CON VARIABLES Y CONSTANTES
===============================================================================

**7.1. Nombres descriptivos**

```php
<?php
// ❌ Malos nombres
$a = "Lucas";
$b = 25;
$c = true;

// ✅ Buenos nombres
$nombreUsuario = "Lucas";
$edadUsuario = 25;
$esActivo = true;
?>
```

**7.2. Inicialización antes de uso**

```php
<?php
// ❌ Peligroso: usar sin inicializar
$total += $precio;  // $total podría no existir

// ✅ Seguro: inicializar primero
$total = 0;
$total += $precio;
?>
```

**7.3. Constantes para valores fijos**

```php
<?php
// ❌ Números mágicos (difíciles de mantener)
if ($intentos > 3) {
    echo "Bloqueado";
}

// ✅ Constantes descriptivas
define('MAX_INTENTOS', 3);
define('TIEMPO_BLOQUEO', 900);  // 15 minutos en segundos

if ($intentos > MAX_INTENTOS) {
    echo "Cuenta bloqueada por " . TIEMPO_BLOQUEO . " segundos";
}
?>
```

**7.4. Usar type hinting con variables**

```php
<?php
// PHP 8: Typed properties en clases
class Usuario {
    public string $nombre;
    public int $edad;
    public ?string $email = null;  // nullable
}

$user = new Usuario();
$user->nombre = "Lucas";
$user->edad = 25;  // Si pones un string aquí, PHP 8 lanza error
?>
```

===============================================================================
8. DESTRUCTURACIÓN DE VARIABLES (PHP 8.1+)
===============================================================================

PHP 8.1 introdujo la destructuración de arrays como una forma más limpia
de extraer valores:

**8.1. Sintaxis básica**

```php
<?php
// Forma tradicional
$usuario = ['Lucas', 25, 'lucas@email.com'];
$nombre = $usuario[0];
$edad = $usuario[1];
$email = $usuario[2];

// Destructuración (PHP 8.1+)
[$nombre, $edad, $email] = $usuario;
echo "$nombre, $edad, $email";  // Lucas, 25, lucas@email.com

// Con arrays asociativos
$datos = ['nombre' => 'Ana', 'edad' => 30];
['nombre' => $nombre, 'edad' => $edad] = $datos;
echo "$nombre tiene $edad años";  // Ana tiene 30 años
?>
```

**8.2. Omitir elementos**

```php
<?php
$numeros = [1, 2, 3, 4, 5];
[$primero, , $tercero, , $quinto] = $numeros;
echo "$primero, $tercero, $quinto";  // 1, 3, 5
?>
```

9. VARIABLES Y MEMORIA EN PHP
===============================================================================

Entender cómo PHP maneja la memoria te ayuda a escribir código más
eficiente:

**9.1. Copy-on-write**

PHP usa una técnica llamada "copiar al escribir" (copy-on-write).
Mientras no modifiques una variable, múltiples variables pueden
compartir la misma región de memoria:

```php
<?php
$a = str_repeat('x', 1000000);  // 1 MB de datos
$b = $a;  // No se copia: ambas apuntan al mismo lugar
$b .= 'y'; // Solo aquí se copia (porque se modifica)

echo "Memoria usada: " . memory_get_usage() . " bytes\n";
?>
```

**9.2. Recolector de basura (Garbage Collector)**

PHP libera automáticamente la memoria de variables que ya no se usan.
El recolector de basura (GC) se activa cuando el ciclo de referencia
es detectado:

```php
<?php
// Liberar memoria explícitamente
$datosGrandes = str_repeat('x', 10000000);  // 10 MB
unset($datosGrandes);  // Libera la memoria inmediatamente
// o simplemente deja que el GC actúe al final del script
?>
```

**9.3. Consumo de memoria por tipo**

| Tipo | Tamaño aproximado (64 bits) |
|------|---------------------------|
| bool | 16 bytes |
| int | 16 bytes |
| float | 16 bytes |
| string (1 char) | 24 bytes + 1 |
| array (vacío) | 56 bytes |
| object (vacío) | 96 bytes + overhead |

RESUMEN DEL CAPÍTULO
===============================================================================

- Las variables en PHP comienzan con `$` y son case-sensitive.
- La asignación por valor copia el dato; por referencia (&) comparte la
  dirección de memoria.
- El ámbito local existe dentro de funciones; el global usa `global`.
- Las variables estáticas (`static`) persisten entre llamadas.
- `define()` y `const` crean constantes inmutables; `const` es más
  moderna y eficiente.
- PHP ofrece constantes mágicas (`__FILE__`, `__LINE__`, etc.) y
  predefinidas (`PHP_VERSION`, `PHP_OS`).
- Las 9 superglobales (`$_GET`, `$_POST`, `$_SERVER`, etc.) están
  siempre disponibles en cualquier ámbito.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo cambiar el valor de una constante?**
No. Las constantes son inmutables. Si intentas redefinir una constante,
PHP lanzará un warning y conservará el valor original.

❓ **¿Cuál es la diferencia entre `define()` y `const`?**
`define()` puede usarse en cualquier contexto (incluso dentro de
bucles). `const` es más rápida pero solo puede usarse en la raíz del
archivo o dentro de una clase.

❓ **¿Por qué no se recomienda usar `global`?**
Porque crea dependencias ocultas entre funciones y el estado global,
dificultando el mantenimiento y las pruebas unitarias.

❓ **¿Qué significa `?` antes del tipo (ej: `?string`)?**
Indica que la variable puede ser del tipo especificado o `null`. Es un
tipo nullable.

❓ **¿Debo usar `isset()` o `??` para verificar superglobales?**
Ambos son válidos, pero `??` (null coalescing) es más conciso.
`isset()` es útil cuando necesitas verificar múltiples variables.
