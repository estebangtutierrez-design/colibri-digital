Saludos, arquitecto backend. Has llegado a uno de los patrones más poderosos
en el desarrollo de aplicaciones web: el patrón Middleware.

El middleware es una capa de software que se sitúa entre el cliente y el
controlador final. Cada petición atraviesa una cadena de middlewares que
pueden: autenticar al usuario, registrar la petición en logs, validar datos,
comprimir respuestas, limitar la tasa de peticiones, y mucho más.

En lugar de tener código repetido de autenticación, logging y validación
en cada controlador, los middlewares centralizan estas responsabilidades
transversales. Esto hace tu código más limpio, mantenible y profesional.

implementarás middlewares concretos como autenticación, logging, CORS y
rate limiting, y entenderás cómo frameworks como Laravel y Slim manejan
este patrón.

1. EL PATRÓN MIDDLEWARE
===============================================================================

El middleware sigue el patrón de diseño **Chain of Responsibility** (Cadena
de Responsabilidad). Cada middleware decide si procesa la petición y la pasa
al siguiente, o si la interrumpe y devuelve una respuesta.

El flujo típico es:

```
PETICIÓN → Middleware1 → Middleware2 → Middleware3 → CONTROLADOR → RESPUESTA
               ↑             ↑             ↑              |
               |             |             |              |
               +—— Logging —+—— Auth ———+—— CORS ————---+
```

Cada middleware puede:

1. **Procesar la petición** (modificar headers, validar datos, etc.)
2. **Pasar al siguiente** middleware o controlador
3. **Interrumpir la cadena** y devolver una respuesta inmediata

```php
<?php
// Interface de Middleware
interface MiddlewareInterface
{
    public function handle(
        array $request,
        callable $next
    ): array;
}

// Implementación básica de un middleware
class LoggingMiddleware implements MiddlewareInterface
{
    public function handle(array $request, callable $next): array
    {
        // 1. Procesar antes de pasar al siguiente
        $inicio = microtime(true);
        $this->log("Inicio: {$request['method']} {$request['uri']}");

        // 2. Pasar al siguiente middleware/controlador
        $response = $next($request);

        // 3. Procesar después de la respuesta
        $duracion = (microtime(true) - $inicio) * 1000;
        $this->log("Fin: {$request['method']} {$request['uri']} - {$duracion}ms");

        // 4. Devolver la respuesta
        return $response;
    }

    private function log(string $mensaje): void
    {
        $fecha = date('Y-m-d H:i:s');
        file_put_contents(
            __DIR__ . '/../storage/logs/api.log',
            "[{$fecha}] {$mensaje}\n",
            FILE_APPEND
        );
    }
}
?>
```

2. PIPELINE DE MIDDLEWARES
===============================================================================

El pipeline es la estructura que encadena los middlewares en orden. Cada
middleware recibe una función `$next` que representa el resto de la cadena.

```php
<?php
// Pipeline.php — Ejecución de la cadena de middlewares
class Pipeline
{
    private array $middlewares = [];

    public function add(MiddlewareInterface $middleware): self
    {
        $this->middlewares[] = $middleware;
        return $this;
    }

    public function run(array $request, callable $finalHandler): array
    {
        if (empty($this->middlewares)) {
            return $finalHandler($request);
        }

        // Construir la cadena desde el último middleware hasta el primero
        $pipeline = array_reduce(
            array_reverse($this->middlewares),
            function (callable $next, MiddlewareInterface $middleware) {
                return function (array $request) use ($middleware, $next) {
                    return $middleware->handle($request, $next);
                };
            },
            $finalHandler
        );

        return $pipeline($request);
    }
}

// Uso del Pipeline
$pipeline = new Pipeline();
$pipeline
    ->add(new LoggingMiddleware())
    ->add(new AuthMiddleware())
    ->add(new CorsMiddleware());

$request = [
    'method' => $_SERVER['REQUEST_METHOD'],
    'uri'    => $_SERVER['REQUEST_URI'],
    'headers' => getallheaders(),
    'body'   => file_get_contents('php://input')
];

$response = $pipeline->run($request, function (array $request) {
    // Controlador final
    $router = new Router();
    return $router->dispatch($request['method'], $request['uri']);
});

// Enviar respuesta
if (is_array($response)) {
    header("HTTP/1.1 {$response['status']}");
    echo json_encode($response['body']);
}
?>
```

3. MIDDLEWARE DE AUTENTICACIÓN JWT
===============================================================================

El middleware de autenticación verifica el token JWT en cada petición.

```php
<?php
// src/Middleware/AuthMiddleware.php
namespace App\Middleware;

use App\Auth\JwtHandler;

class AuthMiddleware implements MiddlewareInterface
{
    private JwtHandler $jwtHandler;
    private array $rutasPublicas;

    public function __construct(array $rutasPublicas = [])
    {
        $this->jwtHandler = new JwtHandler();
        $this->rutasPublicas = $rutasPublicas;
    }

    public function handle(array $request, callable $next): array
    {
        // Verificar si la ruta es pública
        if ($this->esRutaPublica($request['uri'], $request['method'])) {
            return $next($request);
        }

        // Extraer token
        $token = $this->extraerToken($request['headers']);

        if ($token === null) {
            return [
                'status' => 401,
                'body'   => [
                    'success' => false,
                    'error'   => [
                        'codigo'  => 401,
                        'mensaje' => 'Token de autenticación requerido'
                    ]
                ]
            ];
        }

        try {
            // Verificar token
            $usuario = $this->jwtHandler->extraerUsuarioDeToken($token);

            // Inyectar usuario en la request
            $request['usuario'] = $usuario;

            return $next($request);

        } catch (\RuntimeException $e) {
            return [
                'status' => 401,
                'body'   => [
                    'success' => false,
                    'error'   => [
                        'codigo'  => 401,
                        'mensaje' => $e->getMessage()
                    ]
                ]
            ];
        }
    }

    private function extraerToken(array $headers): ?string
    {
        $authHeader = $headers['Authorization'] ?? '';

        if (preg_match('/^Bearer\s+(.+)$/i', $authHeader, $matches)) {
            return $matches[1];
        }

        return null;
    }

    private function esRutaPublica(string $uri, string $method): bool
    {
        foreach ($this->rutasPublicas as $rutaPublica) {
            if (str_starts_with($uri, $rutaPublica)) {
                return true;
            }
        }
        return false;
    }
}
?>
```

4. MIDDLEWARE DE LOGGING
===============================================================================

El middleware de logging registra cada petición y respuesta para
monitoreo y depuración.

```php
<?php
// src/Middleware/LoggingMiddleware.php
namespace App\Middleware;

class LoggingMiddleware implements MiddlewareInterface
{
    private string $logDir;
    private bool $logRespuestas;

    public function __construct(
        string $logDir = null,
        bool $logRespuestas = false
    ) {
        $this->logDir = $logDir ?? __DIR__ . '/../../storage/logs';
        $this->logRespuestas = $logRespuestas;

        if (!is_dir($this->logDir)) {
            mkdir($this->logDir, 0775, true);
        }
    }

    public function handle(array $request, callable $next): array
    {
        $inicio = microtime(true);
        $idPeticion = bin2hex(random_bytes(8));

        // Log de petición entrante
        $this->escribirLog($idPeticion, 'REQUEST', [
            'method' => $request['method'],
            'uri'    => $request['uri'],
            'query'  => $_GET,
            'ip'     => $_SERVER['REMOTE_ADDR'] ?? 'unknown',
            'user_agent' => $_SERVER['HTTP_USER_AGENT'] ?? 'unknown'
        ]);

        // Ejecutar la cadena
        try {
            $response = $next($request);
            $duracion = (microtime(true) - $inicio) * 1000;

            // Log de respuesta
            $logData = [
                'status'   => $response['status'] ?? 200,
                'duracion_ms' => round($duracion, 2)
            ];

            if ($this->logRespuestas) {
                $logData['body'] = $response['body'] ?? null;
            }

            $this->escribirLog($idPeticion, 'RESPONSE', $logData);

            // Agregar metadata a la respuesta
            if (isset($response['body']) && is_array($response['body'])) {
                if (!isset($response['body']['meta'])) {
                    $response['body']['meta'] = [];
                }
                $response['body']['meta']['tiempo_procesamiento_ms'] = round($duracion, 2);
                $response['body']['meta']['request_id'] = $idPeticion;
            }

        } catch (\Throwable $e) {
            $duracion = (microtime(true) - $inicio) * 1000;

            $this->escribirLog($idPeticion, 'ERROR', [
                'mensaje' => $e->getMessage(),
                'archivo' => $e->getFile(),
                'linea'   => $e->getLine(),
                'duracion_ms' => round($duracion, 2)
            ]);

            throw $e;
        }

        return $response;
    }

    private function escribirLog(string $id, string $tipo, array $datos): void
    {
        $fecha = date('Y-m-d H:i:s');
        $log = [
            'id'    => $id,
            'tipo'  => $tipo,
            'fecha' => $fecha,
            'datos' => $datos
        ];

        $archivo = $this->logDir . '/api-' . date('Y-m-d') . '.log';
        file_put_contents(
            $archivo,
            json_encode($log, JSON_UNESCAPED_UNICODE) . "\n",
            FILE_APPEND | LOCK_EX
        );
    }
}
?>
```

5. MIDDLEWARE CORS
===============================================================================

El middleware CORS maneja las políticas de intercambio de recursos entre
orígenes cruzados.

```php
<?php
// src/Middleware/CorsMiddleware.php
namespace App\Middleware;

class CorsMiddleware implements MiddlewareInterface
{
    private array $origenesPermitidos;
    private array $metodosPermitidos;
    private array $headersPermitidos;
    private int $maxAge;

    public function __construct(
        array $origenesPermitidos = ['*'],
        array $metodosPermitidos = ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'OPTIONS'],
        array $headersPermitidos = ['Content-Type', 'Authorization', 'X-Requested-With'],
        int $maxAge = 86400
    ) {
        $this->origenesPermitidos = $origenesPermitidos;
        $this->metodosPermitidos = $metodosPermitidos;
        $this->headersPermitidos = $headersPermitidos;
        $this->maxAge = $maxAge;
    }

    public function handle(array $request, callable $next): array
    {
        $origin = $request['headers']['Origin'] ?? '*';

        // Determinar el origen permitido
        $allowedOrigin = $this->getAllowedOrigin($origin);

        // Establecer headers CORS
        $headers = [];

        if ($allowedOrigin) {
            $headers['Access-Control-Allow-Origin'] = $allowedOrigin;
            $headers['Access-Control-Allow-Methods'] = implode(', ', $this->metodosPermitidos);
            $headers['Access-Control-Allow-Headers'] = implode(', ', $this->headersPermitidos);
            $headers['Access-Control-Max-Age'] = (string)$this->maxAge;

            if ($origin !== '*') {
                $headers['Vary'] = 'Origin';
            }
        }

        // Manejar peticiones preflight OPTIONS
        if ($request['method'] === 'OPTIONS') {
            return [
                'status'  => 204,
                'headers' => $headers,
                'body'    => null
            ];
        }

        // Ejecutar la cadena y agregar headers CORS
        $response = $next($request);

        if (isset($response['headers'])) {
            $response['headers'] = array_merge($response['headers'], $headers);
        } else {
            $response['headers'] = $headers;
        }

        return $response;
    }

    private function getAllowedOrigin(string $origin): ?string
    {
        if (in_array('*', $this->origenesPermitidos, true)) {
            return '*';
        }

        foreach ($this->origenesPermitidos as $permitido) {
            if ($origin === $permitido) {
                return $origin;
            }
            // Soporte para wildcard: https://*.example.com
            if (str_contains($permitido, '*')) {
                $pattern = '/^' . str_replace('\*', '.*', preg_quote($permitido, '/')) . '$/';
                if (preg_match($pattern, $origin)) {
                    return $origin;
                }
            }
        }

        return null;
    }
}
?>
```

6. MIDDLEWARE DE RATE LIMITING
===============================================================================

El rate limiting protege tu API contra abusos limitando el número de
peticiones por IP o por usuario en una ventana de tiempo.

```php
<?php
// src/Middleware/RateLimitMiddleware.php
namespace App\Middleware;

class RateLimitMiddleware implements MiddlewareInterface
{
    private int $maxPeticiones;
    private int $ventanaSegundos;
    private string $almacen;

    public function __construct(
        int $maxPeticiones = 100,
        int $ventanaSegundos = 60,
        string $almacen = 'file'
    ) {
        $this->maxPeticiones = $maxPeticiones;
        $this->ventanaSegundos = $ventanaSegundos;
        $this->almacen = $almacen;
    }

    public function handle(array $request, callable $next): array
    {
        $clave = $this->getClaveRateLimit($request);
        $limite = $this->verificarLimite($clave);

        if ($limite['excedido']) {
            $tiempoRestante = $limite['reset'] - time();

            return [
                'status'  => 429,
                'headers' => [
                    'Retry-After' => (string)$tiempoRestante,
                    'X-RateLimit-Limit' => (string)$this->maxPeticiones,
                    'X-RateLimit-Remaining' => '0',
                    'X-RateLimit-Reset' => (string)$limite['reset']
                ],
                'body'    => [
                    'success' => false,
                    'error'   => [
                        'codigo'  => 429,
                        'mensaje' => "Demasiadas peticiones. Intenta de nuevo en {$tiempoRestante} segundos."
                    ]
                ]
            ];
        }

        $response = $next($request);

        // Agregar headers de rate limit a la respuesta
        if (!isset($response['headers'])) {
            $response['headers'] = [];
        }

        $response['headers'] = array_merge($response['headers'], [
            'X-RateLimit-Limit' => (string)$this->maxPeticiones,
            'X-RateLimit-Remaining' => (string)$limite['restantes'],
            'X-RateLimit-Reset' => (string)$limite['reset']
        ]);

        return $response;
    }

    private function getClaveRateLimit(array $request): string
    {
        // Priorizar usuario autenticado sobre IP
        if (isset($request['usuario']['id'])) {
            return 'user:' . $request['usuario']['id'];
        }

        $ip = $_SERVER['REMOTE_ADDR'] ?? '127.0.0.1';
        return 'ip:' . $ip;
    }

    private function verificarLimite(string $clave): array
    {
        if ($this->almacen === 'file') {
            return $this->verificarLimiteFile($clave);
        }

        // Para producción, usar Redis o base de datos
        return $this->verificarLimiteFile($clave);
    }

    private function verificarLimiteFile(string $clave): array
    {
        $archivo = __DIR__ . '/../../storage/ratelimit/' .
                   md5($clave) . '.json';
        $directorio = dirname($archivo);

        if (!is_dir($directorio)) {
            mkdir($directorio, 0775, true);
        }

        $ahora = time();
        $datos = ['contador' => 0, 'reset' => $ahora + $this->ventanaSegundos];

        if (file_exists($archivo)) {
            $contenido = file_get_contents($archivo);
            $datos = json_decode($contenido, true) ?? $datos;

            // Reiniciar si la ventana expiró
            if ($datos['reset'] <= $ahora) {
                $datos = ['contador' => 0, 'reset' => $ahora + $this->ventanaSegundos];
            }
        }

        $datos['contador']++;

        file_put_contents($archivo, json_encode($datos), LOCK_EX);

        $restantes = max(0, $this->maxPeticiones - $datos['contador']);

        return [
            'excedido'  => $datos['contador'] > $this->maxPeticiones,
            'restantes' => $restantes,
            'reset'     => $datos['reset'],
            'limite'    => $this->maxPeticiones
        ];
    }
}
?>
```

7. MIDDLEWARE DE VALIDACIÓN
===============================================================================

Este middleware valida los datos de entrada antes de que lleguen al
controlador, evitando duplicar lógica de validación.

```php
<?php
// src/Middleware/ValidationMiddleware.php
namespace App\Middleware;

class ValidationMiddleware implements MiddlewareInterface
{
    private array $reglas;
    private array $mensajes;

    public function __construct(array $reglas, array $mensajes = [])
    {
        $this->reglas = $reglas;
        $this->mensajes = $mensajes;
    }

    public function handle(array $request, callable $next): array
    {
        if (empty($this->reglas)) {
            return $next($request);
        }

        $datos = json_decode($request['body'] ?? '{}', true) ?? [];
        $errores = [];

        foreach ($this->reglas as $campo => $reglasCampo) {
            $valor = $datos[$campo] ?? null;

            foreach ($reglasCampo as $regla => $parametro) {
                $error = $this->validarCampo($campo, $valor, $regla, $parametro);
                if ($error !== null) {
                    $errores[$campo][] = $error;
                }
            }
        }

        if (!empty($errores)) {
            return [
                'status' => 422,
                'body'   => [
                    'success' => false,
                    'error'   => [
                        'codigo'  => 422,
                        'mensaje' => 'Error de validación',
                        'detalles' => $errores
                    ]
                ]
            ];
        }

        // Inyectar datos validados en la request
        $request['datos_validados'] = $datos;

        return $next($request);
    }

    private function validarCampo(
        string $campo,
        mixed $valor,
        string $regla,
        mixed $parametro
    ): ?string {
        return match ($regla) {
            'required' => $valor === null || $valor === ''
                ? ($this->mensajes["{$campo}.required"] ?? "El campo {$campo} es requerido")
                : null,

            'string' => $valor !== null && !is_string($valor)
                ? ($this->mensajes["{$campo}.string"] ?? "El campo {$campo} debe ser texto")
                : null,

            'numeric' => $valor !== null && !is_numeric($valor)
                ? ($this->mensajes["{$campo}.numeric"] ?? "El campo {$campo} debe ser numérico")
                : null,

            'email' => $valor !== null && !filter_var($valor, FILTER_VALIDATE_EMAIL)
                ? ($this->mensajes["{$campo}.email"] ?? "El campo {$campo} no es un email válido")
                : null,

            'min' => is_string($valor) && strlen($valor) < $parametro
                ? ($this->mensajes["{$campo}.min"] ?? "El campo {$campo} debe tener al menos {$parametro} caracteres")
                : (is_numeric($valor) && $valor < $parametro
                    ? ($this->mensajes["{$campo}.min"] ?? "El campo {$campo} debe ser mayor o igual a {$parametro}")
                    : null),

            'max' => is_string($valor) && strlen($valor) > $parametro
                ? ($this->mensajes["{$campo}.max"] ?? "El campo {$campo} no debe exceder {$parametro} caracteres")
                : (is_numeric($valor) && $valor > $parametro
                    ? ($this->mensajes["{$campo}.max"] ?? "El campo {$campo} debe ser menor o igual a {$parametro}")
                    : null),

            'enum' => !in_array($valor, $parametro, true)
                ? ($this->mensajes["{$campo}.enum"] ?? "El campo {$campo} debe ser uno de: " . implode(', ', $parametro))
                : null,

            'regex' => is_string($valor) && !preg_match($parametro, $valor)
                ? ($this->mensajes["{$campo}.regex"] ?? "El campo {$campo} no tiene el formato requerido")
                : null,

            default => null
        };
    }
}

// Uso en rutas
$validationMiddleware = new ValidationMiddleware([
    'nombre'  => ['required' => true, 'string' => true, 'min' => 3, 'max' => 100],
    'email'   => ['required' => true, 'email' => true],
    'edad'    => ['required' => true, 'numeric' => true, 'min' => 18],
    'rol'     => ['required' => true, 'enum' => ['admin', 'editor', 'viewer']]
]);

$pipeline = new Pipeline();
$pipeline
    ->add(new CorsMiddleware())
    ->add(new LoggingMiddleware())
    ->add(new AuthMiddleware(['/api/v1/auth']))
    ->add($validationMiddleware)
    ->add(new RateLimitMiddleware(60, 60));
?>
```

8. MIDDLEWARE DE COMPRESIÓN
===============================================================================

Optimiza las respuestas comprimiéndolas con gzip para reducir el ancho
de banda.

```php
<?php
// src/Middleware/CompressionMiddleware.php
namespace App\Middleware;

class CompressionMiddleware implements MiddlewareInterface
{
    private int $nivelCompresion;
    private int $tamanoMinimo;

    public function __construct(
        int $nivelCompresion = 6,
        int $tamanoMinimo = 1024
    ) {
        $this->nivelCompresion = $nivelCompresion;
        $this->tamanoMinimo = $tamanoMinimo;
    }

    public function handle(array $request, callable $next): array
    {
        $response = $next($request);

        // Verificar si el cliente acepta gzip
        $acceptEncoding = $request['headers']['Accept-Encoding'] ?? '';

        if (!str_contains($acceptEncoding, 'gzip')) {
            return $response;
        }

        // Solo comprimir respuestas JSON grandes
        $body = $response['body'] ?? null;

        if ($body === null) {
            return $response;
        }

        $bodyStr = is_string($body) ? $body : json_encode($body);
        $tamano = strlen($bodyStr);

        if ($tamano < $this->tamanoMinimo) {
            return $response;
        }

        // Comprimir
        $comprimido = gzencode($bodyStr, $this->nivelCompresion);

        $response['body'] = $comprimido;

        if (!isset($response['headers'])) {
            $response['headers'] = [];
        }

        $response['headers']['Content-Encoding'] = 'gzip';
        $response['headers']['Vary'] = 'Accept-Encoding';

        return $response;
    }
}
?>
```

9. MIDDLEWARE DE SEGURIDAD
===============================================================================

Un conjunto de middlewares de seguridad que protegen contra ataques
comunes.

```php
<?php
// src/Middleware/SecurityMiddleware.php
namespace App\Middleware;

class SecurityMiddleware implements MiddlewareInterface
{
    public function handle(array $request, callable $next): array
    {
        // 1. Protección contra clickjacking
        header('X-Frame-Options: DENY');

        // 2. Protección contra MIME sniffing
        header('X-Content-Type-Options: nosniff');

        // 3. Protección contra XSS
        header('X-XSS-Protection: 1; mode=block');

        // 4. Política de referente
        header('Referrer-Policy: strict-origin-when-cross-origin');

        // 5. Content Security Policy básica
        header("Content-Security-Policy: default-src 'none'; script-src 'self'");

        // 6. Strict Transport Security (solo HTTPS)
        if (isset($_SERVER['HTTPS']) && $_SERVER['HTTPS'] === 'on') {
            header('Strict-Transport-Security: max-age=31536000; includeSubDomains');
        }

        // 7. Sanitizar query string
        $request['query'] = $this->sanitizarDatos($_GET);

        return $next($request);
    }

    private function sanitizarDatos(array $datos): array
    {
        $sanitizado = [];

        foreach ($datos as $clave => $valor) {
            if (is_array($valor)) {
                $sanitizado[$clave] = $this->sanitizarDatos($valor);
            } else {
                $sanitizado[$clave] = htmlspecialchars(
                    strip_tags($valor),
                    ENT_QUOTES,
                    'UTF-8'
                );
            }
        }

        return $sanitizado;
    }
}
?>
```

11. MIDDLEWARE POR RUTA ESPECÍFICA
===============================================================================

En APIs complejas, diferentes rutas necesitan diferentes middlewares.
Implementemos un sistema de asignación de middlewares por ruta.

```php
<?php
// Router avanzado con soporte de middlewares por ruta
class RouterAdvanced
{
    private array $rutas = [];
    private array $middlewaresGlobales = [];

    public function addMiddleware(MiddlewareInterface $middleware): void
    {
        $this->middlewaresGlobales[] = $middleware;
    }

    public function addRoute(
        string $method,
        string $pattern,
        callable $handler,
        array $middlewares = []
    ): void {
        $this->rutas[] = [
            'method'      => $method,
            'pattern'     => $pattern,
            'handler'     => $handler,
            'middlewares' => $middlewares
        ];
    }

    public function get(string $pattern, callable $handler, array $middlewares = []): void
    {
        $this->addRoute('GET', $pattern, $handler, $middlewares);
    }

    public function post(string $pattern, callable $handler, array $middlewares = []): void
    {
        $this->addRoute('POST', $pattern, $handler, $middlewares);
    }

    public function dispatch(string $method, string $uri): array
    {
        $uri = parse_url($uri, PHP_URL_PATH);
        $uri = rtrim($uri, '/') ?: '/';

        foreach ($this->rutas as $ruta) {
            if ($ruta['method'] !== $method) {
                continue;
            }

            $params = $this->match($ruta['pattern'], $uri);
            if ($params === false) {
                continue;
            }

            // Construir pipeline con middlewares globales + específicos
            $pipeline = new Pipeline();

            foreach ($this->middlewaresGlobales as $mw) {
                $pipeline->add($mw);
            }

            foreach ($ruta['middlewares'] as $mw) {
                $pipeline->add($mw);
            }

            // Ejecutar
            return $pipeline->run(
                ['params' => $params],
                function ($request) use ($ruta) {
                    return call_user_func(
                        $ruta['handler'],
                        $request['params']
                    );
                }
            );
        }

        return [
            'status' => 404,
            'body'   => ['error' => 'Ruta no encontrada']
        ];
    }

    private function match(string $pattern, string $uri): array|false
    {
        $regex = preg_replace('/\{(\w+)\}/', '(?P<$1>[^/]+)', $pattern);
        $regex = '#^' . $regex . '$#';

        if (preg_match($regex, $uri, $matches)) {
            return array_filter($matches, 'is_string', ARRAY_FILTER_USE_KEY);
        }

        return false;
    }
}

// Uso del router avanzado
$router = new RouterAdvanced();

// Middlewares globales
$router->addMiddleware(new SecurityMiddleware());
$router->addMiddleware(new CorsMiddleware());
$router->addMiddleware(new LoggingMiddleware());

// Rutas públicas
$router->post('/api/v1/auth/login', [AuthController::class, 'login']);

// Rutas protegidas con middlewares específicos
$router->get('/api/v1/usuarios', [UsuarioController::class, 'index'], [
    new AuthMiddleware(),
    new RateLimitMiddleware(30, 60)
]);

$router->post('/api/v1/productos', [ProductoController::class, 'store'], [
    new AuthMiddleware(),
    new ValidationMiddleware([
        'nombre' => ['required' => true, 'string' => true, 'min' => 2],
        'precio' => ['required' => true, 'numeric' => true, 'min' => 0.01]
    ])
]);

$router->delete('/api/v1/usuarios/{id}', [UsuarioController::class, 'destroy'], [
    new AuthMiddleware(),
    new RoleMiddleware(['admin'])
]);
?>
```

12. ORDEN DE LOS MIDDLEWARES
===============================================================================

El orden en que se ejecutan los middlewares es crítico. Un orden incorrecto
puede causar problemas de seguridad o rendimiento.

Orden recomendado para una API REST:

```php
<?php
// Orden de middlewares recomendado
$pipeline = new Pipeline();

// 1. Seguridad (siempre primero)
$pipeline->add(new SecurityMiddleware());

// 2. CORS (antes de cualquier procesamiento)
$pipeline->add(new CorsMiddleware());

// 3. Rate Limiting (proteger contra abusos antes de procesar)
$pipeline->add(new RateLimitMiddleware(100, 60));

// 4. Logging (registrar peticiones legítimas)
$pipeline->add(new LoggingMiddleware());

// 5. Autenticación (verificar identidad)
$pipeline->add(new AuthMiddleware([
    '/api/v1/auth/login',
    '/api/v1/auth/register'
]));

// 6. Autorización (verificar permisos)
$pipeline->add(new RoleMiddleware(['admin', 'editor']));

// 7. Validación (datos limpios al controlador)
$pipeline->add(new ValidationMiddleware($reglas));

// 8. Compresión (comprimir la respuesta final)
$pipeline->add(new CompressionMiddleware());

// 9. Controlador (último en la cadena)
$pipeline->add(new ControllerMiddleware($controlador));
?>
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- El patrón Middleware implementa la cadena de responsabilidad: cada
  middleware decide si procesa la petición y la pasa al siguiente.
- Un Pipeline encadena los middlewares usando `array_reduce` para construir
  la pila de ejecución en el orden correcto.
- Los middlewares transversales típicos son: autenticación JWT, logging,
  CORS, rate limiting, validación, compresión y seguridad.
- El orden de los middlewares importa: seguridad y CORS primero, luego
  rate limiting, logging, autenticación, autorización, validación, y
  finalmente compresión.
- Los middlewares pueden ser globales (se aplican a todas las rutas) o
  específicos (se asignan por ruta o grupo de rutas).
- El middleware debe devolver la respuesta en un formato estandarizado
  (status, headers, body) para que el pipeline pueda encadenarla.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuándo debo usar middleware en lugar de código en el controlador?**
Cuando la lógica es transversal a múltiples rutas (autenticación, logging,
CORS, rate limiting). Si solo afecta a un controlador, queda mejor ahí.

❓ **¿Puedo tener middlewares que se ejecuten después del controlador?** Sí.
Un middleware puede ejecutar código tanto antes como después de llamar a
`$next()`. El código después de `$next()` se ejecuta al regreso.

❓ **¿Los middlewares afectan el rendimiento?** Cada middleware añade una
capa de procesamiento, pero bien implementados (sin operaciones bloqueantes)
el impacto es mínimo. El rate limiting con archivos puede ser lento; para
producción usa Redis.

❓ **¿Debo usar la misma instancia de middleware para todas las rutas?** Sí,
los middlewares sin estado (stateless) pueden ser reutilizados. Si tienen
configuración específica por ruta, crea una nueva instancia.

❓ **¿Qué hago si un middleware necesita modificar la respuesta?** El
middleware recibe la respuesta del `$next()` y puede modificarla antes de
devolverla. El patrón "antes y después" es clave.
