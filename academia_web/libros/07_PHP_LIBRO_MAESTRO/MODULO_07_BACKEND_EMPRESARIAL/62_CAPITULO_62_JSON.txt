===============================================================================
1. FUNDAMENTOS DE JSON
===============================================================================

JSON es un formato de texto ligero para el intercambio de datos. Se basa
en dos estructuras fundamentales:

1. **Objeto**: Colección de pares clave/valor (similar a un array asociativo
   en PHP).
2. **Array**: Lista ordenada de valores (similar a un array indexado en PHP).

Los tipos de datos que soporta JSON son:

- **String**: Cadena de texto entre comillas dobles: `"Hola mundo"`
- **Number**: Número entero o decimal: `42`, `3.1416`
- **Boolean**: `true` o `false`
- **Null**: `null`
- **Object**: `{"clave": "valor"}`
- **Array**: `[1, 2, 3]`

Ejemplo de un objeto JSON completo:

```json
{
    "id": 1,
    "nombre": "Juan Pérez",
    "email": "juan@example.com",
    "activo": true,
    "saldo": 1500.50,
    "direccion": {
        "calle": "Av. Principal 123",
        "ciudad": "Ciudad de México"
    },
    "roles": ["admin", "editor"],
    "ultimo_acceso": null
}
```

Observa las reglas sintácticas:

- Las claves siempre van entre comillas dobles.
- Los strings siempre usan comillas dobles, no comillas simples.
- No se permiten trailing commas (coma después del último elemento).
- Los nombres de las propiedades son sensibles a mayúsculas.
- No se permiten comentarios en JSON puro.

===============================================================================
2. json_encode — DE PHP A JSON
===============================================================================

La función `json_encode()` convierte valores PHP a su representación JSON.
Es la herramienta principal para construir las respuestas de tu API.

```php
<?php
// Uso básico de json_encode
$datos = [
    'id'    => 1,
    'nombre' => 'Juan Pérez',
    'email'  => 'juan@example.com'
];

echo json_encode($datos);
// {"id":1,"nombre":"Juan Pérez","email":"juan@example.com"}
?>
```

La firma completa de la función es:

```php
json_encode(
    mixed $value,
    int   $flags = 0,
    int   $depth = 512
): string|false
```

- `$value`: El valor a codificar (arrays, objetos, tipos escalares).
- `$flags`: Constantes que modifican el comportamiento (bitmask).
- `$depth`: Profundidad máxima de anidamiento (default 512).

Valores de retorno:

```php
<?php
// Tipos de datos y su representación JSON
$datos = [
    'integer' => json_encode(42),            // 42
    'float'   => json_encode(3.14),          // 3.14
    'string'  => json_encode("Hola"),        // "Hola"
    'boolean' => json_encode(true),          // true
    'null'    => json_encode(null),          // null
    'array'   => json_encode([1, 2, 3]),     // [1,2,3]
    'object'  => json_encode(['a' => 1])     // {"a":1}
];

var_dump($datos);
?>
```

===============================================================================
3. BANDERAS (FLAGS) DE json_encode
===============================================================================

PHP ofrece múltiples constantes para controlar cómo se genera el JSON.
Estas se combinan usando el operador `|` (bitwise OR).

```php
<?php
// JSON_PRETTY_PRINT — Formato legible con sangrías
$datos = ['nombre' => 'Ana', 'edad' => 30];

echo json_encode($datos, JSON_PRETTY_PRINT);
/*
{
    "nombre": "Ana",
    "edad": 30
}
*/

// JSON_UNESCAPED_UNICODE — No escapar caracteres Unicode
$texto = ['mensaje' => '¡Hola, mundo! Café'];

echo json_encode($texto);
// {"mensaje":"\u00a1Hola, mundo! Caf\u00e9"}

echo json_encode($texto, JSON_UNESCAPED_UNICODE);
// {"mensaje":"¡Hola, mundo! Café"}

// JSON_UNESCAPED_SLASHES — No escapar barras /
$url = ['url' => 'https://api.example.com/usuarios'];

echo json_encode($url);
// {"url":"https:\/\/api.example.com\/usuarios"}

echo json_encode($url, JSON_UNESCAPED_SLASHES);
// {"url":"https://api.example.com/usuarios"}

// JSON_NUMERIC_CHECK — Convertir strings numéricos a números
$datos = ['id' => '42', 'precio' => '19.99'];

echo json_encode($datos, JSON_NUMERIC_CHECK);
// {"id":42,"precio":19.99}

// JSON_FORCE_OBJECT — Forzar arrays indexados a objetos
$array = ['rojo', 'verde', 'azul'];

echo json_encode($array);
// ["rojo","verde","azul"]

echo json_encode($array, JSON_FORCE_OBJECT);
// {"0":"rojo","1":"verde","2":"azul"}

// JSON_PRESERVE_ZERO_FRACTION — Preservar .0 en floats
$numero = 42.0;

echo json_encode($numero);
// 42

echo json_encode($numero, JSON_PRESERVE_ZERO_FRACTION);
// 42.0

// Combinación de flags para producción
$flags = JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES | JSON_NUMERIC_CHECK;
echo json_encode($datos, $flags);
?>
```

Flags para manejo de errores:

```php
<?php
// JSON_INVALID_UTF8_IGNORE — Ignorar caracteres UTF-8 inválidos
$datos = ['nombre' => "\xB1\x31"]; // UTF-8 inválido

echo json_encode($datos, JSON_INVALID_UTF8_IGNORE);
// {"nombre":""}

// JSON_INVALID_UTF8_SUBSTITUTE — Sustituir caracteres inválidos
echo json_encode($datos, JSON_INVALID_UTF8_SUBSTITUTE);
// {"nombre":"\ufffd1"}
?>
```

===============================================================================
4. json_decode — DE JSON A PHP
===============================================================================

La función `json_decode()` convierte una cadena JSON a un valor PHP.

```php
<?php
// Uso básico
$json = '{"id":1,"nombre":"Juan","activo":true}';

// Decodificar a objeto stdClass (por defecto)
$objeto = json_decode($json);
echo $objeto->nombre; // Juan

// Decodificar a array asociativo
$array = json_decode($json, true);
echo $array['nombre']; // Juan

// Decodificar a una clase específica
$usuario = json_decode($json, false, 512, JSON_OBJECT_AS_ARRAY);
?>
```

Firma completa:

```php
json_decode(
    string $json,
    ?bool  $associative = null,
    int    $depth = 512,
    int    $flags = 0
): mixed
```

- `$json`: La cadena JSON a decodificar.
- `$associative`: `true` devuelve arrays asociativos; `false` devuelve objetos
  `stdClass`; `null` usa el valor por defecto.
- `$depth`: Profundidad máxima de anidamiento.
- `$flags`: Bitmask de flags (JSON_BIGINT_AS_STRING, JSON_OBJECT_AS_ARRAY).

Ejemplos avanzados:

```php
<?php
// JSON anidado
$json = '{
    "usuario": {
        "nombre": "Ana",
        "direccion": {
            "calle": "Av. Reforma",
            "cp": "06600"
        },
        "hobbies": ["leer", "viajar", "fotografía"]
    }
}';

$datos = json_decode($json, true);
echo $datos['usuario']['direccion']['calle']; // Av. Reforma
echo $datos['usuario']['hobbies'][1];          // viajar

// JSON_BIGINT_AS_STRING — Evitar pérdida de precisión en enteros grandes
$json = '{"id": 12345678901234567890}';

$normal = json_decode($json, true);
var_dump($normal['id']);
// float(1.2345678901234568e19) — ¡Pérdida de precisión!

$bigint = json_decode($json, true, 512, JSON_BIGINT_AS_STRING);
var_dump($bigint['id']);
// string(20) "12345678901234567890" — Precisión intacta

// JSON_OBJECT_AS_ARRAY — Forzar objetos a arrays
$json = '{"a":1,"b":2}';

$resultado = json_decode($json, false, 512, JSON_OBJECT_AS_ARRAY);
var_dump($resultado);
// array(2) { ["a"]=> int(1) ["b"]=> int(2) }
?>
```

===============================================================================
5. MANEJO DE ERRORES EN JSON
===============================================================================

Las funciones JSON pueden fallar. Un manejo adecuado de errores es
fundamental para una API robusta.

```php
<?php
// Detectar errores en json_encode
$datos = ["nombre" => "\xB1\x31"]; // UTF-8 inválido

$resultado = json_encode($datos);
if ($resultado === false) {
    echo "Error json_encode: " . json_last_error_msg();
    // Error json_encode: Malformed UTF-8 characters, possibly incorrectly encoded
}

// json_last_error() devuelve el código del último error
switch (json_last_error()) {
    case JSON_ERROR_NONE:
        echo "Sin errores";
        break;
    case JSON_ERROR_DEPTH:
        echo "Profundidad máxima excedida";
        break;
    case JSON_ERROR_STATE_MISMATCH:
        echo "Desbordamiento o desajuste de modo";
        break;
    case JSON_ERROR_CTRL_CHAR:
        echo "Caracter de control inesperado";
        break;
    case JSON_ERROR_SYNTAX:
        echo "Error de sintaxis en JSON";
        break;
    case JSON_ERROR_UTF8:
        echo "Caracteres UTF-8 mal formados";
        break;
    case JSON_ERROR_RECURSION:
        echo "Recursión detectada";
        break;
    case JSON_ERROR_INF_OR_NAN:
        echo "Valor INF o NAN encontrado";
        break;
    case JSON_ERROR_UNSUPPORTED_TYPE:
        echo "Tipo no soportado para codificación";
        break;
    case JSON_ERROR_INVALID_PROPERTY_NAME:
        echo "Nombre de propiedad inválido";
        break;
    case JSON_ERROR_UTF16:
        echo "Caracter UTF-16 mal formado";
        break;
    default:
        echo "Error desconocido";
        break;
}

// Función segura para decodificar JSON
function jsonDecodeSeguro(string $json, bool $asArray = true): array|null
{
    $datos = json_decode($json, $asArray);

    if (json_last_error() !== JSON_ERROR_NONE) {
        throw new InvalidArgumentException(
            "JSON inválido: " . json_last_error_msg()
        );
    }

    return $datos;
}

// Uso
try {
    $datos = jsonDecodeSeguro('{"nombre": "Juan"}');
    echo "Decodificado exitosamente";
} catch (InvalidArgumentException $e) {
    echo "Error: " . $e->getMessage();
}
?>
```

===============================================================================
6. CABECERAS HTTP PARA JSON
===============================================================================

Para que el cliente interprete correctamente tu respuesta como JSON, debes
establecer la cabecera `Content-Type` adecuada.

```php
<?php
// Cabecera estándar para JSON
header('Content-Type: application/json; charset=utf-8');

// Cabecera para JSON con soporte CORS
header('Content-Type: application/json; charset=utf-8');
header('Access-Control-Allow-Origin: *');

// Cache control para respuestas JSON
header('Content-Type: application/json; charset=utf-8');
header('Cache-Control: no-store, no-cache, must-revalidate');

// Función completa para iniciar una respuesta JSON
function iniciarRespuestaJson(int $codigoHttp = 200): void
{
    http_response_code($codigoHttp);
    header('Content-Type: application/json; charset=utf-8');
    header('Access-Control-Allow-Origin: *');
    header('Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS');
    header('Access-Control-Allow-Headers: Content-Type, Authorization');
    header('Cache-Control: no-store, private');
}
?>
```

7. RESPUESTAS JSON CONSISTENTES
===============================================================================

Una API profesional define un formato de respuesta estándar que se usa en
todos los endpoints. Esto facilita el trabajo de los clientes que consumen
tu API.

Formato de respuesta exitosa:

```json
{
    "success": true,
    "data": { ... },
    "mensaje": "Operación exitosa",
    "meta": {
        "timestamp": "2025-01-15T10:30:00Z",
        "version": "1.0"
    }
}
```

Formato de respuesta con error:

```json
{
    "success": false,
    "error": {
        "codigo": 422,
        "mensaje": "Error de validación",
        "detalles": {
            "email": ["El email ya está registrado"],
            "edad": ["La edad debe ser mayor a 18"]
        }
    },
    "meta": {
        "timestamp": "2025-01-15T10:30:00Z",
        "version": "1.0"
    }
}
```

Implementación en PHP:

```php
<?php
// Response.php — Clase para respuestas JSON estandarizadas
class Response
{
    private const VERSION = '1.0';

    public static function success(
        mixed $data = null,
        string $mensaje = 'Operación exitosa',
        int $codigo = 200,
        array $extraMeta = []
    ): void
    {
        http_response_code($codigo);
        header('Content-Type: application/json; charset=utf-8');

        $respuesta = [
            'success' => true,
            'data'    => $data,
            'mensaje' => $mensaje,
            'meta'    => array_merge([
                'timestamp' => date('c'),
                'version'   => self::VERSION
            ], $extraMeta)
        ];

        echo self::encode($respuesta);
        exit;
    }

    public static function error(
        string $mensaje,
        int $codigo = 400,
        mixed $detalles = null
    ): void
    {
        http_response_code($codigo);
        header('Content-Type: application/json; charset=utf-8');

        $error = [
            'codigo'  => $codigo,
            'mensaje' => $mensaje
        ];

        if ($detalles !== null) {
            $error['detalles'] = $detalles;
        }

        $respuesta = [
            'success' => false,
            'error'   => $error,
            'meta'    => [
                'timestamp' => date('c'),
                'version'   => self::VERSION
            ]
        ];

        echo self::encode($respuesta);
        exit;
    }

    public static function paginated(
        array $data,
        int $total,
        int $page,
        int $perPage,
        string $mensaje = 'Operación exitosa'
    ): void
    {
        $totalPaginas = (int)ceil($total / $perPage);

        self::success($data, $mensaje, 200, [
            'pagination' => [
                'total'         => $total,
                'per_page'      => $perPage,
                'current_page'  => $page,
                'total_pages'   => $totalPaginas,
                'has_next'      => $page < $totalPaginas,
                'has_previous'  => $page > 1
            ]
        ]);
    }

    private static function encode(array $data): string
    {
        $flags = JSON_UNESCAPED_UNICODE |
                 JSON_UNESCAPED_SLASHES |
                 JSON_NUMERIC_CHECK;

        $json = json_encode($data, $flags);

        if ($json === false) {
            http_response_code(500);
            return json_encode([
                'success' => false,
                'error'   => [
                    'codigo'  => 500,
                    'mensaje' => 'Error interno al generar respuesta'
                ]
            ]);
        }

        return $json;
    }
}
?>
```

Uso en los controladores:

```php
<?php
// Uso de la clase Response
class ProductoController
{
    private PDO $db;

    public function __construct(PDO $db)
    {
        $this->db = $db;
    }

    public function index(): void
    {
        $page = max(1, (int)($_GET['page'] ?? 1));
        $perPage = min(100, max(1, (int)($_GET['per_page'] ?? 15)));

        $total = $this->db->query(
            "SELECT COUNT(*) FROM productos"
        )->fetchColumn();

        $offset = ($page - 1) * $perPage;
        $stmt = $this->db->prepare(
            "SELECT id, nombre, precio, stock
             FROM productos
             LIMIT ? OFFSET ?"
        );
        $stmt->execute([$perPage, $offset]);
        $productos = $stmt->fetchAll(PDO::FETCH_ASSOC);

        Response::paginated($productos, (int)$total, $page, $perPage);
    }

    public function show(int $id): void
    {
        $stmt = $this->db->prepare(
            "SELECT id, nombre, precio, stock FROM productos WHERE id = ?"
        );
        $stmt->execute([$id]);
        $producto = $stmt->fetch(PDO::FETCH_ASSOC);

        if (!$producto) {
            Response::error('Producto no encontrado', 404);
            return;
        }

        Response::success($producto);
    }

    public function store(): void
    {
        $datos = json_decode(file_get_contents('php://input'), true);

        if (!isset($datos['nombre']) || !isset($datos['precio'])) {
            Response::error(
                'Nombre y precio son requeridos',
                422,
                ['campos_faltantes' => ['nombre', 'precio']]
            );
            return;
        }

        // Guardar en base de datos...
        $nuevoId = 42;

        Response::success(
            ['id' => $nuevoId],
            'Producto creado exitosamente',
            201
        );
    }

    public function destroy(int $id): void
    {
        // Eliminar...
        http_response_code(204);
        exit;
    }
}
?>
```

8. SERIALIZACIÓN DE OBJETOS COMPLEJOS
===============================================================================

Cuando trabajas con objetos PHP, `json_encode` serializa automáticamente
las propiedades públicas. Para controlar este proceso, implementa la
interfaz `JsonSerializable`.

```php
<?php
// Serialización básica de objetos
class Usuario
{
    public function __construct(
        public int    $id,
        public string $nombre,
        public string $email,
        private string $password, // No debe exponerse
        public array  $roles,
        public ?string $ultimoAcceso = null
    ) {}
}

$usuario = new Usuario(1, 'Ana', 'ana@mail.com', 'secreta', ['admin']);
echo json_encode($usuario);
// {"id":1,"nombre":"Ana","email":"ana@mail.com","roles":["admin"],"ultimoAcceso":null}
// Nota: $password no se incluye porque es private

// Serialización personalizada con JsonSerializable
class UsuarioSeguro implements JsonSerializable
{
    public function __construct(
        public int     $id,
        public string  $nombre,
        public string  $email,
        private string $password,
        public array   $roles,
        public ?string $ultimoAcceso = null
    ) {}

    public function jsonSerialize(): array
    {
        return [
            'id'            => $this->id,
            'nombre'        => $this->nombre,
            'email'         => $this->email,
            'roles'         => $this->roles,
            'ultimo_acceso' => $this->ultimoAcceso,
            '_links'        => [
                'self' => "/api/usuarios/{$this->id}"
            ]
        ];
    }

    public function toArray(): array
    {
        return $this->jsonSerialize();
    }
}

$usuarioSeguro = new UsuarioSeguro(1, 'Ana', 'ana@mail.com', 'secreta', ['admin']);
echo json_encode($usuarioSeguro, JSON_PRETTY_PRINT);
/*
{
    "id": 1,
    "nombre": "Ana",
    "email": "ana@mail.com",
    "roles": ["admin"],
    "ultimo_acceso": null,
    "_links": {
        "self": "/api/usuarios/1"
    }
}
*/

// Serialización de colecciones
class ColeccionUsuarios implements JsonSerializable
{
    private array $usuarios = [];

    public function add(UsuarioSeguro $usuario): void
    {
        $this->usuarios[] = $usuario;
    }

    public function jsonSerialize(): array
    {
        return [
            'data'  => array_map(
                fn(UsuarioSeguro $u) => $u->jsonSerialize(),
                $this->usuarios
            ),
            'total' => count($this->usuarios)
        ];
    }
}

$coleccion = new ColeccionUsuarios();
$coleccion->add($usuarioSeguro);
echo json_encode($coleccion, JSON_PRETTY_PRINT);
?>
```

9. MANEJO DE FECHAS EN JSON
===============================================================================

Las fechas no tienen un tipo nativo en JSON. Debes definir un formato
consistente para toda tu API.

```php
<?php
// Opciones para serializar fechas

// Opción 1: ISO 8601 (recomendado)
$fechaISO = date('c'); // 2025-01-15T10:30:00-06:00

// Opción 2: Timestamp Unix
$fechaTimestamp = time(); // 1736953800

// Opción 3: Formato legible
$fechaLegible = date('Y-m-d H:i:s'); // 2025-01-15 10:30:00

// Clase para manejo consistente de fechas
class FechaJson
{
    public static function ahora(): string
    {
        return date('c');
    }

    public static function desdeTimestamp(int $timestamp): string
    {
        return date('c', $timestamp);
    }

    public static function desdeDB(string $fechaDB): string
    {
        return date('c', strtotime($fechaDB));
    }

    public static function aTimestamp(string $fechaJson): int
    {
        return strtotime($fechaJson);
    }
}

// Uso en respuestas API
$respuesta = [
    'success' => true,
    'data'    => [
        'id'          => 42,
        'nombre'      => 'Producto Ejemplo',
        'created_at'  => FechaJson::desdeDB('2025-01-15 10:30:00'),
        'updated_at'  => FechaJson::ahora()
    ]
];

echo json_encode($respuesta, JSON_PRETTY_PRINT);
/*
{
    "success": true,
    "data": {
        "id": 42,
        "nombre": "Producto Ejemplo",
        "created_at": "2025-01-15T10:30:00-06:00",
        "updated_at": "2025-01-15T12:00:00-06:00"
    }
}
*/
?>
```

10. VALIDACIÓN DE JSON DE ENTRADA
===============================================================================

Cuando tu API recibe JSON en el cuerpo de la petición, debes validar que
sea correcto antes de procesarlo.

```php
<?php
// Función para obtener y validar el cuerpo JSON de una petición
function obtenerCuerpoJson(): array
{
    $raw = file_get_contents('php://input');

    if (empty($raw)) {
        Response::error('El cuerpo de la petición está vacío', 400);
    }

    $datos = json_decode($raw, true);

    if (json_last_error() !== JSON_ERROR_NONE) {
        Response::error(
            'JSON inválido: ' . json_last_error_msg(),
            400
        );
    }

    if (!is_array($datos)) {
        Response::error(
            'El cuerpo debe ser un objeto JSON',
            400
        );
    }

    return $datos;
}

// Validador de esquema simple
function validarEsquema(array $datos, array $esquema): array
{
    $errores = [];

    foreach ($esquema as $campo => $reglas) {
        $valor = $datos[$campo] ?? null;

        foreach ($reglas as $regla => $parametro) {
            switch ($regla) {
                case 'required':
                    if ($valor === null || $valor === '') {
                        $errores[$campo][] = "El campo {$campo} es requerido";
                    }
                    break;

                case 'type':
                    $tipoValido = match ($parametro) {
                        'string' => is_string($valor),
                        'numeric' => is_numeric($valor),
                        'integer' => is_int($valor),
                        'array' => is_array($valor),
                        'boolean' => is_bool($valor),
                        default => true
                    };
                    if (!$tipoValido) {
                        $errores[$campo][] = "El campo {$campo} debe ser de tipo {$parametro}";
                    }
                    break;

                case 'min':
                    if (is_numeric($valor) && $valor < $parametro) {
                        $errores[$campo][] = "El campo {$campo} debe ser mayor o igual a {$parametro}";
                    }
                    break;

                case 'max':
                    if (is_numeric($valor) && $valor > $parametro) {
                        $errores[$campo][] = "El campo {$campo} debe ser menor o igual a {$parametro}";
                    }
                    break;

                case 'pattern':
                    if (is_string($valor) && !preg_match($parametro, $valor)) {
                        $errores[$campo][] = "El campo {$campo} no tiene el formato requerido";
                    }
                    break;

                case 'enum':
                    if (!in_array($valor, $parametro, true)) {
                        $errores[$campo][] = "El campo {$campo} debe ser uno de: " . implode(', ', $parametro);
                    }
                    break;
            }
        }
    }

    return $errores;
}

// Uso
$esquema = [
    'nombre'   => ['required' => true, 'type' => 'string', 'min' => 3, 'max' => 100],
    'email'    => ['required' => true, 'type' => 'string', 'pattern' => '/^[^\s@]+@[^\s@]+\.[^\s@]+$/'],
    'edad'     => ['required' => true, 'type' => 'integer', 'min' => 18, 'max' => 120],
    'rol'      => ['required' => true, 'enum' => ['admin', 'editor', 'viewer']],
    'activo'   => ['type' => 'boolean']
];

$datos = obtenerCuerpoJson();
$errores = validarEsquema($datos, $esquema);

if (!empty($errores)) {
    Response::error('Error de validación', 422, $errores);
}
?>
```

11. JSON Y BASE DE DATOS

MySQL y PostgreSQL tienen soporte nativo para JSON. Puedes almacenar y
consultar datos JSON directamente desde la base de datos.

<?php
// Almacenar datos JSON en MySQL
$stmt = $this->db->prepare(
    "INSERT INTO usuarios (nombre, metadata) VALUES (?, ?)"
);

$metadata = json_encode([
    'ultimo_login' => date('c'),
    'preferencias' => [
        'idioma'  => 'es',
        'tema'    => 'oscuro',
        'notificaciones' => true
    ],
    'direccion_envio' => [
        'calle'   => 'Av. Principal 123',
        'ciudad'  => 'Ciudad de México',
        'cp'      => '06600'
    ]
], JSON_UNESCAPED_UNICODE);

$stmt->execute(['Juan Pérez', $metadata]);

// Consultar campos JSON en MySQL (MySQL 5.7+)
$stmt = $this->db->query(
    "SELECT id, nombre,
            JSON_EXTRACT(metadata, '$.preferencias.idioma') as idioma
     FROM usuarios
     WHERE JSON_EXTRACT(metadata, '$.preferencias.tema') = 'oscuro'"
);

$usuarios = $stmt->fetchAll(PDO::FETCH_ASSOC);

// Decodificar el JSON al recuperarlo
foreach ($usuarios as &$usuario) {
    if (isset($usuario['metadata'])) {
        $usuario['metadata'] = json_decode($usuario['metadata'], true);
    }
}

// PostgreSQL tiene operadores específicos para JSON

12. JSON Y RESPUESTAS DE ERROR
===============================================================================

Nunca debes exponer errores internos de PHP en tus respuestas JSON. Usa
un manejador global que capture todo y devuelva JSON consistente.

```php
<?php
// Manejador de errores global para API JSON
function manejarErrorGlobal(): void
{
    set_exception_handler(function (Throwable $e) {
        $codigo = $e->getCode() >= 400 && $e->getCode() <= 599
            ? $e->getCode()
            : 500;

        $respuesta = [
            'success' => false,
            'error'   => [
                'codigo'  => $codigo,
                'mensaje' => $e->getMessage()
            ],
            'meta'    => [
                'timestamp' => date('c'),
                'version'   => '1.0'
            ]
        ];

        // En desarrollo, incluir depuración
        if (getenv('APP_ENV') === 'development') {
            $respuesta['error']['debug'] = [
                'archivo' => $e->getFile(),
                'linea'   => $e->getLine(),
                'trace'   => explode("\n", $e->getTraceAsString())
            ];
        }

        http_response_code($codigo);
        header('Content-Type: application/json; charset=utf-8');
        echo json_encode(
            $respuesta,
            JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES
        );
        exit;
    });

    set_error_handler(function ($nivel, $mensaje, $archivo, $linea) {
        throw new ErrorException($mensaje, 500, $nivel, $archivo, $linea);
    });
}

manejarErrorGlobal();
?>
```

13. JSON EN PETICIONES SALIENTES
===============================================================================

Cuando tu API necesita consumir otras APIs externas, también usarás JSON.

```php
<?php
// Consumir una API externa con file_get_contents
function consumirApiExterna(string $url, array $headers = []): array
{
    $context = stream_context_create([
        'http' => [
            'method' => 'GET',
            'header' => array_merge(
                ["Content-Type: application/json"],
                $headers
            ),
            'timeout' => 30,
            'ignore_errors' => true
        ]
    ]);

    $resultado = file_get_contents($url, false, $context);

    if ($resultado === false) {
        throw new RuntimeException("Error al conectar con {$url}");
    }

    return json_decode($resultado, true) ?? [];
}

// Consumir API con cURL
function consumirApiConCurl(
    string $url,
    string $metodo = 'GET',
    array $datos = [],
    array $headers = []
): array
{
    $ch = curl_init();
    curl_setopt_array($ch, [
        CURLOPT_URL            => $url,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_CUSTOMREQUEST  => $metodo,
        CURLOPT_TIMEOUT        => 30,
        CURLOPT_HTTPHEADER     => array_merge(
            ['Content-Type: application/json'],
            $headers
        )
    ]);

    if (!empty($datos) && in_array($metodo, ['POST', 'PUT', 'PATCH'])) {
        curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($datos));
    }

    $respuesta = curl_exec($ch);
    $info = curl_getinfo($ch);
    curl_close($ch);

    if ($respuesta === false) {
        throw new RuntimeException("cURL error: " . curl_error($ch));
    }

    return [
        'codigo' => $info['http_code'],
        'datos'  => json_decode($respuesta, true) ?? []
    ];
}

// Ejemplo: Consultar el clima desde una API externa
$resultado = consumirApiExterna(
    'https://api.openweathermap.org/data/2.5/weather?q=MexicoCity&appid=API_KEY'
);

// Ejemplo: Enviar datos a un webhook
$respuesta = consumirApiConCurl(
    'https://hooks.example.com/pedidos',
    'POST',
    [
        'pedido_id' => 1234,
        'total'     => 1500.00,
        'items'     => [
            ['producto' => 'Laptop', 'cantidad' => 1, 'precio' => 1500.00]
        ]
    ]
);
?>
```

RESUMEN DEL CAPÍTULO
===============================================================================

- `json_encode` convierte valores PHP a JSON con múltiples flags para
  controlar formato, escape de Unicode, barras y tipos numéricos.
- `json_decode` convierte JSON a PHP, pudiendo devolver arrays asociativos
  u objetos stdClass según el segundo parámetro.
- Siempre verifica errores con `json_last_error()` y `json_last_error_msg()`
  después de codificar o decodificar JSON.
- Establece la cabecera `Content-Type: application/json; charset=utf-8` en
  todas las respuestas de tu API.
- Define un formato de respuesta consistente con campos como `success`,
  `data`, `error` y `meta` para toda la API.
- Implementa `JsonSerializable` para controlar cómo se serializan tus
  objetos a JSON, ocultando datos sensibles como contraseñas.
- Las fechas deben usar formato ISO 8601 (`date('c')`) para consistencia
  entre sistemas.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Debo usar JSON_NUMERIC_CHECK siempre?** Con cuidado. Convierte strings
numéricos a números, lo cual es útil para IDs y precios, pero puede causar
problemas con códigos postales o números de teléfono que empiezan con cero.

❓ **¿Cómo manejo valores Infinity o NaN en PHP?** `json_encode` falla con
estos valores. Usa `JSON_PARTIAL_OUTPUT_ON_ERROR` o sanitiza los datos
antes de codificar reemplazando valores no finitos con `null`.

❓ **¿Cuál es la profundidad máxima de JSON que debo permitir?** El default
de 512 niveles es suficiente para el 99.9% de los casos. Aumentarlo podría
exponerte a ataques de profundidad (stack overflow).

❓ **¿Puedo incluir comentarios en JSON?** No en JSON estándar. Pero
existen formatos como JSONC (JSON with Comments) usados por VS Code. Si
necesitas comentarios, considera YAML.

❓ **¿Por qué `json_decode` devuelve null a veces?** `json_decode` devuelve
`null` tanto para la cadena "null" como para JSON inválido. Siempre verifica
con `json_last_error()`.
