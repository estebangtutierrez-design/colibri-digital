Saludos, arquitecto backend. Has construido una API REST robusta con
autenticación, autorización y middlewares. Pero hay un paso igual de
importante que el código mismo: la documentación.

Una API sin documentación es inutilizable para otros desarrolladores. Por
más intuitiva que sea, nadie podrá integrarse con ella sin saber qué
endpoints existen, qué parámetros esperan, qué formato tienen las respuestas
y qué errores pueden ocurrir.

OpenAPI (antes conocido como Swagger) es el estándar de la industria para
documentar APIs REST. Con una especificación OpenAPI, puedes generar
automáticamente: documentación interactiva, clientes SDK en múltiples
lenguajes, y tests de integración.

En este capítulo aprenderás a documentar tu API usando el formato OpenAPI
3.0, a generar documentación interactiva con Swagger UI, y a mantener tu
documentación sincronizada con el código real.

1. INTRODUCCIÓN A OPENAPI
===============================================================================

OpenAPI es una especificación estándar para describir APIs REST. Un
documento OpenAPI (en JSON o YAML) describe:

- Los endpoints disponibles y sus métodos HTTP
- Los parámetros de cada endpoint (path, query, header, cookie)
- Los cuerpos de petición y respuesta
- Los esquemas de datos (modelos)
- Los mecanismos de autenticación
- La información de contacto, licencia y versión

Una especificación OpenAPI se escribe como un archivo JSON o YAML. El
formato YAML es más legible y comúnmente usado.

Estructura básica de un documento OpenAPI 3.0:

```yaml
openapi: "3.0.3"
info:
  title: "API Empresarial EGA"
  description: "API REST para el sistema empresarial de EGA Academy"
  version: "1.0.0"
  contact:
    name: "Soporte EGA API"
    email: "api@egaenterprise.com"
  license:
    name: "MIT"
    url: "https://opensource.org/licenses/MIT"

servers:
  - url: https://api.egaenterprise.com/v1
    description: "Servidor de producción"
  - url: https://staging.egaenterprise.com/v1
    description: "Servidor de pruebas"

paths:
  /usuarios:
    get:
      summary: "Listar usuarios"
      description: "Obtiene una lista paginada de usuarios"
      tags:
        - Usuarios
      parameters:
        - name: page
          in: query
          schema:
            type: integer
            default: 1
        - name: per_page
          in: query
          schema:
            type: integer
            default: 15
      responses:
        "200":
          description: "Lista de usuarios obtenida exitosamente"
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: "#/components/schemas/Usuario"

components:
  schemas:
    Usuario:
      type: object
      properties:
        id:
          type: integer
        nombre:
          type: string
        email:
          type: string
          format: email
        rol:
          type: string
          enum: [admin, editor, usuario]
      required:
        - id
        - nombre
        - email

  securitySchemes:
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT

security:
  - BearerAuth: []
```

3. GENERACIÓN DE DOCUMENTACIÓN CON SWAGGER UI
===============================================================================

Swagger UI es una herramienta que renderiza la especificación OpenAPI como
una página web interactiva donde los desarrolladores pueden probar los
endpoints directamente desde el navegador.

```php
<?php
// Instalación de Swagger UI (descargar desde npm o CDN)
// Opción 1: Descargar archivos
//   wget https://github.com/swagger-api/swagger-ui/archive/refs/tags/v5.0.0.zip
//   unzip v5.0.0.zip -d public/swagger

// Opción 2: CDN (más simple)
//   Incluir en el HTML:
//   <link rel="stylesheet" href="https://unpkg.com/swagger-ui-dist@5/swagger-ui.css">
//   <script src="https://unpkg.com/swagger-ui-dist@5/swagger-ui-bundle.js"></script>
?>

<!-- public/docs/index.html — Página de documentación -->
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>API EGA Enterprise — Documentación</title>
    <link rel="stylesheet" href="https://unpkg.com/swagger-ui-dist@5/swagger-ui.css">
    <style>
        body { margin: 0; background: #fafafa; }
        .topbar { display: none; }
        .swagger-ui .info .title {
            color: #1a365d;

        }
        .swagger-ui .btn.authorize {

    </style>

<body>
    <div id="swagger-ui"></div>
    <script src="https://unpkg.com/swagger-ui-dist@5/swagger-ui-bundle.js"></script>
    <script>
        SwaggerUIBundle({
            url: "/docs/openapi.yaml",
            dom_id: "#swagger-ui",

            presets: [
                SwaggerUIBundle.presets.apis,

                theme: "monokai"
            }

```

Servir la documentación desde PHP:

```php
<?php
// public/docs/openapi.php — Servir YAML dinámicamente
header('Content-Type: application/x-yaml');
header('Access-Control-Allow-Origin: *');

// Cargar especificación base
$spec = yaml_parse_file(__DIR__ . '/../../docs/openapi.yaml');

// Personalizar según entorno
$env = getenv('APP_ENV') ?: 'development';

$spec['servers'] = match ($env) {
    'production' => [
        ['url' => 'https://api.egaenterprise.com/v1', 'description' => 'Producción']
    ],
    'staging' => [
        ['url' => 'https://staging.egaenterprise.com/v1', 'description' => 'Staging']
    ],
    default => [
        ['url' => 'http://localhost:8000/api/v1', 'description' => 'Desarrollo local']
    ]
};

echo yaml_emit($spec);
?>
```

4. GENERACIÓN AUTOMÁTICA DE DOCUMENTACIÓN DESDE CÓDIGO
===============================================================================

Mantener la documentación manualmente sincronizada con el código es un
desafío. Una solución es usar anotaciones PHPDoc para generar la
especificación automáticamente.

```php
<?php
// src/Annotations/OpenApi.php — Sistema simple de anotaciones

#[
    Route('/api/v1/usuarios', 'GET'),
    Summary('Listar usuarios'),
    Description('Obtiene lista paginada de usuarios del sistema'),
    Tag('Usuarios'),
    Parameter('page', 'query', 'integer', 'Número de página', false),
    Parameter('per_page', 'query', 'integer', 'Elementos por página', false),
    Response(200, 'Lista de usuarios obtenida exitosamente'),
    Response(401, 'No autenticado'),
    Security('BearerAuth')
]
class UsuarioController
{
    // ...
}

// Generador de OpenAPI desde anotaciones
class OpenApiGenerator
{
    private array $rutas = [];

    public function generarDesdeControladores(array $controladores): string
    {
        $spec = [
            'openapi' => '3.0.3',
            'info' => [
                'title' => 'API EGA Enterprise',
                'version' => '1.0.0'
            ],
            'paths' => [],
            'components' => [
                'schemas' => [],
                'securitySchemes' => [
                    'BearerAuth' => [
                        'type' => 'http',
                        'scheme' => 'bearer',
                        'bearerFormat' => 'JWT'
                    ]
                ]
            ]
        ];

        foreach ($controladores as $controlador) {
            $reflection = new ReflectionClass($controlador);
            $methods = $reflection->getMethods();

            foreach ($methods as $method) {
                $attributes = $method->getAttributes();

                if (empty($attributes)) {
                    continue;
                }

                $ruta = [];
                $metodoHTTP = 'GET';
                $path = '';

                foreach ($attributes as $attribute) {
                    $name = $attribute->getName();
                    $args = $attribute->getArguments();

                    match ($name) {
                        'Route' => [$path, $metodoHTTP] = $args,
                        'Summary' => $ruta['summary'] = $args[0],
                        'Description' => $ruta['description'] = $args[0],
                        'Tag' => $ruta['tags'] = [$args[0]],
                        'Parameter' => $this->agregarParametro($ruta, $args),
                        'Response' => $this->agregarRespuesta($ruta, $args),
                        'Security' => $ruta['security'] = [$args[0] => []],
                        default => null
                    };
                }

                if ($path) {
                    $spec['paths'][$path][strtolower($metodoHTTP)] = $ruta;
                }
            }
        }

        return yaml_emit($spec);
    }

    private function agregarParametro(array &$ruta, array $args): void
    {
        $ruta['parameters'][] = [
            'name' => $args[0],
            'in' => $args[1],
            'schema' => ['type' => $args[2]],
            'description' => $args[3] ?? '',
            'required' => $args[4] ?? false
        ];
    }

    private function agregarRespuesta(array &$ruta, array $args): void
    {
        $codigo = (string)$args[0];
        $ruta['responses'][$codigo] = [
            'description' => $args[1] ?? ''
        ];
    }
}
?>
```

5. INTEGRACIÓN CONTINUA DE DOCUMENTACIÓN
===============================================================================

Mantén la documentación actualizada automáticamente con tests que verifican
que la especificación OpenAPI coincida con la implementación real.

```php
<?php
// tests/ApiDocumentationTest.php
class ApiDocumentationTest
{
    private array $spec;
    private string $baseUrl;

    public function __construct(string $specPath, string $baseUrl)
    {
        $this->spec = yaml_parse_file($specPath);
        $this->baseUrl = $baseUrl;
    }

    public function verificarEndpointsDocumentados(): array
    {
        $errores = [];

        foreach ($this->spec['paths'] as $path => $methods) {
            foreach ($methods as $method => $detalles) {
                // Verificar que cada endpoint tenga summary
                if (!isset($detalles['summary'])) {
                    $errores[] = "{$method} {$path}: falta summary";
                }

                // Verificar que tenga tags
                if (empty($detalles['tags'])) {
                    $errores[] = "{$method} {$path}: falta tag";
                }

                // Verificar que tenga al menos una respuesta
                if (empty($detalles['responses'])) {
                    $errores[] = "{$method} {$path}: falta al menos una respuesta";
                }

                // Verificar que los endpoints protegidos tengan security
                if ($path !== '/auth/login' && $path !== '/auth/register') {
                    if (!isset($detalles['security'])) {
                        $errores[] = "{$method} {$path}: falta security (BearerAuth)";
                    }
                }
            }
        }

        return $errores;
    }

    public function probarEndpoints(): array
    {
        $resultados = [];

        foreach ($this->spec['paths'] as $path => $methods) {
            foreach ($methods as $method => $detalles) {
                $url = $this->baseUrl . $path;
                $ch = curl_init();

                curl_setopt_array($ch, [
                    CURLOPT_URL => $url,
                    CURLOPT_CUSTOMREQUEST => strtoupper($method),
                    CURLOPT_RETURNTRANSFER => true,
                    CURLOPT_HTTPHEADER => [
                        'Content-Type: application/json',
                        'Accept: application/json'
                    ]
                ]);

                $respuesta = curl_exec($ch);
                $codigo = curl_getinfo($ch, CURLINFO_HTTP_CODE);
                curl_close($ch);

                $resultados["{$method} {$path}"] = [
                    'codigo' => $codigo,
                    'documentado' => isset($detalles['responses'][(string)$codigo])
                ];
            }
        }

        return $resultados;
    }
}

// Ejecutar tests
$tester = new ApiDocumentationTest(
    __DIR__ . '/../docs/openapi.yaml',
    'http://localhost:8000/api/v1'
);

$errores = $tester->verificarEndpointsDocumentados();

if (!empty($errores)) {
    echo "Errores de documentación:\n";
    foreach ($errores as $error) {
        echo "  - {$error}\n";
    }
    exit(1);
}

echo "Documentación verificada exitosamente\n";
?>
```

OpenAPI permite generar automáticamente clientes SDK en múltiples lenguajes
usando herramientas como OpenAPI Generator.

```bash
# Instalar OpenAPI Generator
npm install @openapitools/openapi-generator-cli -g

# Generar cliente PHP
openapi-generator-cli generate \
  -i docs/openapi.yaml \
  -g php \
  -o clients/php

# Generar cliente JavaScript
openapi-generator-cli generate \
  -i docs/openapi.yaml \
  -g javascript \
  -o clients/javascript

# Generar cliente Python
openapi-generator-cli generate \
  -i docs/openapi.yaml \
  -g python \
  -o clients/python

# Generar documentación HTML estática
openapi-generator-cli generate \
  -i docs/openapi.yaml \
  -g html \
  -o docs/html
```

Uso del cliente generado:

```php
<?php
// Uso del SDK PHP generado
```

RESUMEN DEL CAPÍTULO
===============================================================================

- OpenAPI 3.0 es el estándar de la industria para documentar APIs REST
  usando archivos YAML o JSON estructurados.
- Un documento OpenAPI describe: servers, paths, parameters, request bodies,
  responses, schemas, security schemes y tags.
- Los componentes reutilizables (schemas, parameters, securitySchemes)
  centralizan definiciones que se referencian con `$ref`.
- Swagger UI renderiza la especificación como documentación interactiva
  donde los desarrolladores pueden probar endpoints en vivo.
- La autenticación se documenta con `securitySchemes` (Bearer JWT) y se
  aplica a endpoints específicos o globalmente.
- Los tests de documentación verifican que la especificación esté completa
  y sincronizada con la implementación real.
- OpenAPI Generator puede producir clientes SDK en PHP, JS, Python, etc.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Debo escribir OpenAPI a mano o usar herramientas?** Para APIs pequeñas,
escribir a mano es viable. Para APIs grandes, usa generadores desde
anotaciones o frameworks como Laravel que tienen paquetes de
auto-documentación.

❓ **¿OpenAPI 3.0 o 3.1?** 3.1 es la última versión y es compatible con
JSON Schema. Sin embargo, 3.0.3 tiene más soporte en herramientas. Usa
3.0.3 para máxima compatibilidad.

❓ **¿Cómo documento respuestas de error que pueden variar?** Usa
`oneOf` o `anyOf` en el schema para manejar múltiples formatos de
respuesta. También puedes documentar el código de estado y poner una
descripción textual.

❓ **¿La documentación debe estar en el mismo repositorio que el código?**
Sí, la documentación debe viajar con el código. Usa el mismo repositorio
y sincronízala con cada cambio de API.

❓ **¿Puedo tener múltiples versiones de la documentación?** Sí, crea
archivos separados como `openapi-v1.yaml` y `openapi-v2.yaml`, y sirve
cada uno en su ruta correspondiente.
