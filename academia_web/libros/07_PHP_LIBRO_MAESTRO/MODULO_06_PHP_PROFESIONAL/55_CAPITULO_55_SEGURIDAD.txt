BIENVENIDA DEL PROFESOR BÚHO
===============================================================================

La seguridad es el aspecto más importante (y más descuidado) del
desarrollo web profesional. PHP tiene una reputación histórica de
inseguridad, pero la realidad es que PHP es tan seguro como el
código que escribes.

Las vulnerabilidades web más comunes son:
  - XSS (Cross-Site Scripting): inyección de scripts en el HTML.
  - CSRF (Cross-Site Request Forgery): peticiones falsificadas.
  - SQL Injection: manipulación maliciosa de consultas SQL.
  - Insecure Direct Object Reference: acceso a recursos no autorizados.

En este capítulo aprenderás a prevenir cada una de estas amenazas
con técnicas concretas y código práctico. No es teoría: son
defensas que aplicarás en cada proyecto profesional.

===============================================================================
1. XSS (CROSS-SITE SCRIPTING)
===============================================================================

XSS ocurre cuando un atacante inyecta código JavaScript en páginas
web que otros usuarios visitan. El código malicioso puede robar
cookies, tokens de sesión, redirigir a sitios falsos, o modificar
el contenido de la página.

Tipos de XSS:

  XSS Reflejado:
    El código malicioso viaja en la URL o en un formulario.
    Ejemplo: ?q=<script>alert('XSS')</script>

  XSS Almacenado:
    El código se guarda en la base de datos y se muestra a otros
    usuarios. Es el más peligroso. Ejemplo: comentario con scripts.

  XSS DOM-based:
    El código malicioso modifica el DOM del lado del cliente.

Prevención: Siempre escapar la salida.

```php
<?php
// MAL - Vulnerable a XSS
echo $_GET['nombre'];

echo "<h1>Bienvenido, " . $_GET['nombre'] . "</h1>";

// BIEN - Escapado
echo htmlspecialchars($_GET['nombre'], ENT_QUOTES, 'UTF-8');

echo "<h1>Bienvenido, " . htmlspecialchars($_GET['nombre'], ENT_QUOTES, 'UTF-8') . "</h1>";
```

Función helper para escapar:

```php
<?php
function h(?string $value): string
{
    return htmlspecialchars($value ?? '', ENT_QUOTES | ENT_HTML5, 'UTF-8', false);
}

// En las vistas
<h1><?= h($titulo) ?></h1>
<p><?= h($usuario['nombre']) ?></p>
<textarea><?= h($comentario) ?></textarea>
<input type="text" value="<?= h($valor) ?>">
```

XSS en atributos HTML:

```php
<?php
// MAL - El atacante puede cerrar el atributo y agregar eventos
echo "<img src='" . $_GET['url'] . "'>";
// ?url=x onerror=alert(1) → <img src='x onerror=alert(1)'>

// BIEN - Escapar el valor
echo "<img src='" . h($_GET['url']) . "'>";
```

XSS en JavaScript:

```php
<?php
// MAL - Inyectar directamente en JS
echo "<script>var nombre = '" . $_GET['nombre'] . "';</script>";

// BIEN - JSON encode para JS
echo "<script>var nombre = " . json_encode($_GET['nombre']) . ";</script>";

// O escapar específico para JS
$jsEscaped = preg_replace('/[^\w\s]/', '', $_GET['nombre']);
```

La regla de oro: NUNCA confíes en la entrada del usuario.
Toda salida debe escaparse según el contexto (HTML, JS, CSS, URL).

===============================================================================
2. CSRF (CROSS-SITE REQUEST FORGERY)
===============================================================================

CSRF ocurre cuando un atacante engaña al navegador de un usuario
autenticado para que ejecute acciones no deseadas en una aplicación
web donde el usuario tiene sesión activa.

Ejemplo: Un atacante crea una imagen en su sitio:
```html
<img src="https://banco.com/transferir?cuenta=123&monto=10000">
```
Si el usuario está autenticado en el banco, el navegador envía
la cookie de sesión automáticamente y la transferencia se ejecuta.

Prevención: Tokens CSRF en cada formulario.

```php
<?php
// core/Security.php
class Security
{
    public static function generateCsrfToken(): string
    {
        if (empty($_SESSION['csrf_token'])) {
            $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
        }
        return $_SESSION['csrf_token'];
    }

    public static function verifyCsrfToken(string $token): bool
    {
        if (empty($_SESSION['csrf_token'])) {
            return false;
        }
        return hash_equals($_SESSION['csrf_token'], $token);
    }

    public static function csrfField(): string
    {
        $token = self::generateCsrfToken();
        return '<input type="hidden" name="_csrf_token" value="' . $token . '">';
    }
}
```

En el formulario:

```php
<form method="POST" action="/usuarios">
    <?= Security::csrfField() ?>
    <input type="text" name="nombre" required>
    <input type="email" name="email" required>
    <button type="submit">Guardar</button>
</form>
```

En el controlador:

```php
<?php
public function store(): void
{
    // Validar CSRF
    if (!Security::verifyCsrfToken($_POST['_csrf_token'] ?? '')) {
        http_response_code(419); // 419 Authentication Timeout
        die('Token CSRF inválido. Recarga la página.');
    }

    // Procesar formulario...
}
```

Middleware CSRF global:

```php
<?php
// core/Middleware/CsrfMiddleware.php
class CsrfMiddleware
{
    public function handle(): void
    {
        $excludedRoutes = ['/webhook/paypal', '/api/*'];

        if (in_array($_SERVER['REQUEST_URI'], $excludedRoutes)) {
            return;
        }

        if ($_SERVER['REQUEST_METHOD'] === 'POST') {
            $token = $_POST['_csrf_token'] ?? $_SERVER['HTTP_X_CSRF_TOKEN'] ?? '';

            if (!Security::verifyCsrfToken($token)) {
                http_response_code(419);
                echo json_encode(['error' => 'CSRF token inválido']);
                exit;
            }
        }
    }
}
```

Los tokens CSRF deben ser:
  - Únicos por sesión (o mejor: únicos por formulario).
  - Generados con random_bytes() (criptográficamente seguros).
  - Validados con hash_equals() para prevenir timing attacks.

===============================================================================
3. SQL INJECTION
===============================================================================

SQL Injection ocurre cuando un atacante manipula una consulta SQL
insertando comandos maliciosos a través de los datos de entrada.

Ejemplo vulnerable:
```php
<?php
// MAL - Vulnerable a SQL Injection
$id = $_GET['id'];
$sql = "SELECT * FROM usuarios WHERE id = $id";
$result = $db->query($sql);

// Si el atacante envía ?id=1 OR 1=1, la consulta devuelve TODOS los usuarios
// Si envía ?id=1; DROP TABLE usuarios; --, ¡borra la tabla!
```

Prevención: Siempre usar prepared statements.

```php
<?php
// BIEN - Prepared statement con PDO
$stmt = $db->prepare('SELECT * FROM usuarios WHERE id = :id AND activo = :activo');
$stmt->execute([
    ':id'     => $_GET['id'],
    ':activo' => true,
]);
$usuarios = $stmt->fetchAll();
```

```php
<?php
// BIEN - Prepared statement con MySQLi
$stmt = $mysqli->prepare('SELECT * FROM usuarios WHERE email = ?');
$stmt->bind_param('s', $_POST['email']);
$stmt->execute();
$result = $stmt->get_result();
```

Prepared statements con múltiples parámetros:

```php
<?php
// Inserción segura
$stmt = $db->prepare('
    INSERT INTO usuarios (nombre, email, password, rol, activo)
    VALUES (:nombre, :email, :password, :rol, :activo)
');
$stmt->execute([
    ':nombre'   => $_POST['nombre'],
    ':email'    => $_POST['email'],
    ':password' => password_hash($_POST['password'], PASSWORD_BCRYPT),
    ':rol'      => 'user',
    ':activo'   => true,
]);
```

```php
<?php
// Actualización segura
$stmt = $db->prepare('UPDATE usuarios SET nombre = :nombre WHERE id = :id');
$stmt->execute([
    ':nombre' => $_POST['nombre'],
    ':id'     => $id,
]);
```

```php
<?php
// DELETE seguro
$stmt = $db->prepare('DELETE FROM usuarios WHERE id = :id AND rol != :admin');
$stmt->execute([
    ':id'    => $id,
    ':admin' => 'admin',
]);
```

La regla: NUNCA concatenes variables en consultas SQL.
Siempre usa :named o ? como placeholders.

Para consultas dinámicas (ORDER BY, columnas variables):

```php
<?php
// ORDER BY no puede usar prepared statements directamente
$orderByWhitelist = ['nombre', 'email', 'creado_en', 'id'];
$orderBy = $_GET['order'] ?? 'id';

if (!in_array($orderBy, $orderByWhitelist)) {
    $orderBy = 'id';
}

// Dirección: ASC o DESC
$direction = strtoupper($_GET['dir'] ?? 'ASC');
if (!in_array($direction, ['ASC', 'DESC'])) {
    $direction = 'ASC';
}

$stmt = $db->query("SELECT * FROM usuarios ORDER BY $orderBy $direction");
```

Usa listas blancas (whitelist) para valores que no pueden ser
parametrizados.

4. VALIDACIÓN Y SANITIZACIÓN DE ENTRADAS
===============================================================================

Nunca confíes en datos del usuario. Siempre valida primero
(¿cumple el formato?) y sanitiza después (limpia el contenido).

```php
<?php
// core/Validator.php
class Validator
{
    private array $errors = [];

    public function required(string $field, mixed $value): static
    {
        if (empty($value) && $value !== '0') {
            $this->errors[$field][] = 'El campo es requerido';
        }
        return $this;
    }

    public function email(string $field, string $value): static
    {
        if (!filter_var($value, FILTER_VALIDATE_EMAIL)) {
            $this->errors[$field][] = 'Email inválido';
        }
        return $this;
    }

    public function minLength(string $field, string $value, int $min): static
    {
        if (strlen($value) < $min) {
            $this->errors[$field][] = "Mínimo $min caracteres";
        }
        return $this;
    }

    public function maxLength(string $field, string $value, int $max): static
    {
        if (strlen($value) > $max) {
            $this->errors[$field][] = "Máximo $max caracteres";
        }
        return $this;
    }

    public function numeric(string $field, mixed $value): static
    {
        if (!is_numeric($value)) {
            $this->errors[$field][] = 'Debe ser un número';
        }
        return $this;
    }

    public function inArray(string $field, mixed $value, array $options): static
    {
        if (!in_array($value, $options, true)) {
            $this->errors[$field][] = 'Valor no permitido';
        }
        return $this;
    }

    public function url(string $field, string $value): static
    {
        if (!filter_var($value, FILTER_VALIDATE_URL)) {
            $this->errors[$field][] = 'URL inválida';
        }
        return $this;
    }

    public function alphanumeric(string $field, string $value): static
    {
        if (!preg_match('/^[a-zA-Z0-9_]+$/', $value)) {
            $this->errors[$field][] = 'Solo letras, números y guión bajo';
        }
        return $this;
    }

    public function passes(): bool
    {
        return empty($this->errors);
    }

    public function getErrors(): array
    {
        return $this->errors;
    }

    public function getFirstError(): ?string
    {
        $first = reset($this->errors);
        return $first ? $first[0] : null;
    }
}
```

Uso en controlador:

```php
<?php
public function store(): void
{
    $validator = new Validator();

    $validator
        ->required('nombre', $_POST['nombre'])
        ->minLength('nombre', $_POST['nombre'], 3)
        ->maxLength('nombre', $_POST['nombre'], 100)
        ->required('email', $_POST['email'])
        ->email('email', $_POST['email'])
        ->required('password', $_POST['password'])
        ->minLength('password', $_POST['password'], 8);

    if (!$validator->passes()) {
        $this->render('usuarios/create', [
            'errors' => $validator->getErrors(),
            'old'    => $_POST,
        ]);
        return;
    }

    // Sanitizar antes de usar
    $nombre = strip_tags(trim($_POST['nombre']));
    $email  = filter_var(trim($_POST['email']), FILTER_SANITIZE_EMAIL);
    $password = $_POST['password']; // No sanitizar, se va a hashear

    // Guardar...
}
```

Sanitización por tipo de dato:

```php
<?php
// Strings
$nombre = strip_tags(trim($_POST['nombre']));
$nombre = htmlspecialchars($nombre, ENT_QUOTES, 'UTF-8');

// Emails
$email = filter_var($_POST['email'], FILTER_SANITIZE_EMAIL);

// URLs
$url = filter_var($_POST['url'], FILTER_SANITIZE_URL);

// Enteros
$edad = filter_var($_POST['edad'], FILTER_VALIDATE_INT, [
    'options' => ['min_range' => 1, 'max_range' => 120]
]);

// Booleanos
$activo = filter_var($_POST['activo'], FILTER_VALIDATE_BOOLEAN);

// Fechas
$fecha = date('Y-m-d', strtotime($_POST['fecha'])) ?: null;
```

5. CABECERAS HTTP DE SEGURIDAD
===============================================================================

Las cabeceras HTTP ayudan al navegador a proteger al usuario:

```php
<?php
// core/SecurityHeaders.php
class SecurityHeaders
{
    public static function apply(): void
    {
        // Prevenir MIME-type sniffing
        header('X-Content-Type-Options: nosniff');

        // Prevenir clickjacking
        header('X-Frame-Options: DENY');

        // Habilitar XSS filter en navegadores antiguos
        header('X-XSS-Protection: 1; mode=block');

        // Strict Transport Security (solo HTTPS)
        if (isset($_SERVER['HTTPS']) && $_SERVER['HTTPS'] === 'on') {
            header('Strict-Transport-Security: max-age=31536000; includeSubDomains');
        }

        // Content Security Policy
        header("Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'");

        // Referrer Policy
        header('Referrer-Policy: same-origin');

        // Permissions Policy
        header("Permissions-Policy: geolocation=(), camera=(), microphone=()");
    }
}
```

Aplicar las cabeceras en el Front Controller:

```php
<?php
// public/index.php
require_once __DIR__ . '/../vendor/autoload.php';

App\Core\SecurityHeaders::apply();

$app = new App\Core\Application();
$app->run();
```

Content-Security-Policy (CSP) es especialmente importante:
  - Evita XSS aunque haya un fallo en el escape.
  - Define qué recursos puede cargar el navegador.
  - Bloquea scripts inline no autorizados.

Política CSP recomendada para empezar:

```
default-src 'self';
script-src 'self';
style-src 'self' 'unsafe-inline';
img-src 'self' data:;
font-src 'self';
connect-src 'self';
frame-ancestors 'none';
```

Puedes usar https://csp-evaluator.withgoogle.com para probar
tu política CSP.

6. AUTENTICACIÓN SEGURA
===============================================================================

Sesiones seguras:

```php
<?php
// config/session.php

// Configuración de sesión segura
ini_set('session.use_strict_mode', '1');      // Solo aceptar IDs creados por PHP
ini_set('session.use_cookies', '1');           // Usar cookies (no URL)
ini_set('session.use_only_cookies', '1');      // Solo cookies, nunca URL
ini_set('session.cookie_httponly', '1');       // No accesible desde JS
ini_set('session.cookie_secure', '1');         // Solo HTTPS
ini_set('session.cookie_samesite', 'Lax');     // Protección CSRF parcial
ini_set('session.gc_maxlifetime', '7200');     // 2 horas de vida

// Regenerar ID de sesión periódicamente
session_start();

if (!isset($_SESSION['_created'])) {
    $_SESSION['_created'] = time();
    session_regenerate_id(true);
} elseif (time() - $_SESSION['_created'] > 1800) {
    // Regenerar cada 30 minutos
    session_regenerate_id(true);
    $_SESSION['_created'] = time();
}
```

Autenticación:

```php
<?php
// core/Auth.php
class Auth
{
    public static function attempt(string $email, string $password): bool
    {
        $db = Database::connect();

        $stmt = $db->prepare('SELECT * FROM usuarios WHERE email = :email AND activo = 1');
        $stmt->execute([':email' => $email]);
        $usuario = $stmt->fetch();

        if (!$usuario || !password_verify($password, $usuario['password'])) {
            return false;
        }

        // Regenerar sesión para prevenir session fixation
        session_regenerate_id(true);

        $_SESSION['user_id']   = $usuario['id'];
        $_SESSION['user_rol']  = $usuario['rol'];
        $_SESSION['user_nombre'] = $usuario['nombre'];

        return true;
    }

    public static function check(): bool
    {
        return isset($_SESSION['user_id']);
    }

    public static function user(): ?array
    {
        if (!self::check()) {
            return null;
        }

        $db = Database::connect();
        $stmt = $db->prepare('SELECT id, nombre, email, rol, creado_en FROM usuarios WHERE id = :id');
        $stmt->execute([':id' => $_SESSION['user_id']]);
        return $stmt->fetch() ?: null;
    }

    public static function logout(): void
    {
        $_SESSION = [];
        $params = session_get_cookie_params();
        setcookie(session_name(), '', time() - 3600, $params['path'], $params['domain'], $params['secure'], $params['httponly']);
        session_destroy();
    }

    public static function requireAuth(): void
    {
        if (!self::check()) {
            header('Location: /login');
            exit;
        }
    }

    public static function requireRol(string $rol): void
    {
        self::requireAuth();
        if ($_SESSION['user_rol'] !== $rol) {
            http_response_code(403);
            die('No tienes permiso para acceder a esta página.');
        }
    }
}
```

7. PROTECCIÓN CONTRA FUERZA BRUTA
===============================================================================

```php
<?php
// core/BruteForceProtection.php
class BruteForceProtection
{
    public static function check(string $ip, string $action = 'login'): bool
    {
        $db = Database::connect();
        $ventana = date('Y-m-d H:i:s', strtotime('-15 minutes'));

        $stmt = $db->prepare('
            SELECT COUNT(*) as intentos
            FROM intentos_fallidos
            WHERE ip = :ip
            AND action = :action
            AND created_at > :ventana
        ');
        $stmt->execute([
            ':ip'      => $ip,
            ':action'  => $action,
            ':ventana' => $ventana,
        ]);

        $intentos = $stmt->fetch()['intentos'];

        // Bloquear después de 5 intentos en 15 minutos
        return $intentos < 5;
    }

    public static function registerAttempt(string $ip, string $action = 'login'): void
    {
        $db = Database::connect();
        $stmt = $db->prepare('
            INSERT INTO intentos_fallidos (ip, action, created_at)
            VALUES (:ip, :action, NOW())
        ');
        $stmt->execute([
            ':ip'     => $ip,
            ':action' => $action,
        ]);
    }

    public static function clearAttempts(string $ip, string $action = 'login'): void
    {
        $db = Database::connect();
        $stmt = $db->prepare('
            DELETE FROM intentos_fallidos WHERE ip = :ip AND action = :action
        ');
        $stmt->execute([
            ':ip'     => $ip,
            ':action' => $action,
        ]);
    }
}
```

Uso en login:

```php
<?php
public function login(): void
{
    if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
        $this->render('auth/login');
        return;
    }

    $ip = $_SERVER['REMOTE_ADDR'];

    if (!BruteForceProtection::check($ip)) {
        $this->render('auth/login', [
            'error' => 'Demasiados intentos. Intenta en 15 minutos.'
        ]);
        return;
    }

    if (Auth::attempt($_POST['email'], $_POST['password'])) {
        BruteForceProtection::clearAttempts($ip);
        $this->redirect('/dashboard');
    } else {
        BruteForceProtection::registerAttempt($ip);
        $this->render('auth/login', [
            'error' => 'Credenciales inválidas',
            'email' => $_POST['email'],
        ]);
    }
}
```

8. CONTROL DE ACCESO POR ROLES (RBAC)
===============================================================================

```php
<?php
// core/Rbac.php
class Rbac
{
    private static array $roles = [
        'admin' => [
            'permissions' => ['*'], // Acceso total
        ],
        'editor' => [
            'permissions' => [
                'articles.create',
                'articles.edit',
                'articles.publish',
                'articles.delete',
            ],
        ],
        'author' => [
            'permissions' => [
                'articles.create',
                'articles.edit',
            ],
        ],
        'user' => [
            'permissions' => [
                'articles.view',
                'comments.create',
            ],
        ],
    ];

    public static function hasPermission(string $permission, ?string $rol = null): bool
    {
        $rol = $rol ?? ($_SESSION['user_rol'] ?? 'guest');

        if (!isset(self::$roles[$rol])) {
            return false;
        }

        $perms = self::$roles[$rol]['permissions'];

        // Admin tiene permiso para todo
        if (in_array('*', $perms)) {
            return true;
        }

        return in_array($permission, $perms);
    }

    public static function requirePermission(string $permission): void
    {
        if (!self::hasPermission($permission)) {
            http_response_code(403);
            die('Acceso denegado: no tienes permiso para esta acción.');
        }
    }

    public static function requireRol(string $rol): void
    {
        $userRol = $_SESSION['user_rol'] ?? 'guest';

        // Admin puede hacer todo
        if ($userRol === 'admin') {
            return;
        }

        if ($userRol !== $rol) {
            http_response_code(403);
            die('Acceso denegado: se requiere rol ' . $rol);
        }
    }
}
```

Middleware de autorización:

```php
<?php
// core/Middleware/AuthorizationMiddleware.php
class AuthorizationMiddleware
{
    public static function for(string $permission): callable
    {
        return function () use ($permission) {
            Auth::requireAuth();
            Rbac::requirePermission($permission);
        };
    }
}
```

Uso en rutas:

```php
<?php
$router->add('GET', '/admin/usuarios', 'Admin\UsuarioController@index')
    ->middleware(AuthorizationMiddleware::for('admin.access'));

$router->add('POST', '/articulos', 'ArticuloController@store')
    ->middleware(AuthorizationMiddleware::for('articles.create'));
```

9. PROTECCIÓN CONTRA ATAQUES COMUNES
===============================================================================

Directory Traversal:

```php
<?php
// MAL - El usuario puede acceder a /etc/passwd
$file = $_GET['file'];
$content = file_get_contents('uploads/' . $file);
// ?file=../../../etc/passwd ¡Acceso no autorizado!

// BIEN - Validar la ruta
$file = basename($_GET['file']); // Elimina ..
$file = preg_replace('/[^\w\.\-]/', '', $file); // Solo caracteres seguros
$path = realpath('uploads/' . $file);

if (!$path || strpos($path, realpath('uploads/')) !== 0) {
    http_response_code(403);
    die('Acceso denegado');
}
$content = file_get_contents($path);
```

File Upload:

```php
<?php
// Validación de archivos subidos
$allowedMimes = ['image/jpeg', 'image/png', 'image/gif', 'application/pdf'];
$maxSize = 5 * 1024 * 1024; // 5MB

if ($_FILES['archivo']['error'] !== UPLOAD_ERR_OK) {
    die('Error al subir archivo');
}

if ($_FILES['archivo']['size'] > $maxSize) {
    die('Archivo demasiado grande');
}

$finfo = new finfo(FILEINFO_MIME_TYPE);
$mime = $finfo->file($_FILES['archivo']['tmp_name']);

if (!in_array($mime, $allowedMimes)) {
    die('Tipo de archivo no permitido');
}

// Generar nombre seguro
$extension = pathinfo($_FILES['archivo']['name'], PATHINFO_EXTENSION);
$nuevoNombre = bin2hex(random_bytes(16)) . '.' . $extension;
move_uploaded_file($_FILES['archivo']['tmp_name'], 'uploads/' . $nuevoNombre);
```

10. HTTPS Y SSL
===============================================================================

Forzar HTTPS en producción:

```php
<?php
// config/https.php
function forceHttps(): void
{
    if (empty($_SERVER['HTTPS']) || $_SERVER['HTTPS'] === 'off') {
        $redirectUrl = 'https://' . $_SERVER['HTTP_HOST'] . $_SERVER['REQUEST_URI'];
        header('HTTP/1.1 301 Moved Permanently');
        header('Location: ' . $redirectUrl);
        exit;
    }
}
```

En servidores Apache, redirigir con .htaccess:

```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]
```

En servidores Nginx:

```nginx
server {
    listen 80;
    server_name mi-app.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name mi-app.com;

    ssl_certificate     /etc/ssl/certs/mi-app.crt;
    ssl_certificate_key /etc/ssl/private/mi-app.key;

    # Configuración SSL moderna
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
}
```

11. BUENAS PRÁCTICAS GENERALES DE SEGURIDAD
===============================================================================

Principio de mínimo privilegio:
  - El usuario de BD solo tiene permisos necesarios.
  - Los archivos tienen permisos 644 (dueño escribe, resto lee).
  - Las carpetas tienen permisos 755.

Mantener software actualizado:
  - PHP, librerías, servidor web, sistema operativo.
  - composer update regularmente.
  - npm audit si usas frontend.

No exponer información sensible:
  - archivos .env en .gitignore.
  - config/ fuera del document root.
  - No mostrar versiones de PHP en headers.

Prevenir Information Disclosure:
```php
<?php
// Ocultar versión de PHP en headers
header_remove('X-Powered-By');

// Deshabilitar exposición de PHP en php.ini
ini_set('expose_php', 'Off');
```

Logs de seguridad:
```php
<?php
function logSecurityEvent(string $event, array $context = []): void
{
    $logFile = __DIR__ . '/../logs/security.log';
    $entry = sprintf(
        "[%s] %s | IP: %s | User: %s | %s\n",
        date('Y-m-d H:i:s'),
        $event,
        $_SERVER['REMOTE_ADDR'] ?? 'CLI',
        $_SESSION['user_id'] ?? 'anonymous',
        json_encode($context, JSON_UNESCAPED_UNICODE)
    );
    file_put_contents($logFile, $entry, FILE_APPEND | LOCK_EX);
}

// Uso
logSecurityEvent('LOGIN_FAILED', ['email' => $_POST['email']]);
logSecurityEvent('PASSWORD_CHANGE', ['user_id' => $user['id']]);
logSecurityEvent('UNAUTHORIZED_ACCESS', ['url' => $_SERVER['REQUEST_URI']]);
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- XSS se previene escapando toda salida con htmlspecialchars().
- CSRF se previene con tokens únicos por sesión y hash_equals().
- SQL Injection se previene con prepared statements (:named o ?).
- La validación debe hacerse con whitelist (lo permitido).
- La sanitización limpia los datos según el contexto.
- Las cabeceras HTTP (CSP, HSTS, X-Frame-Options) protegen al
  navegador.
- Las sesiones seguras usan cookies httponly, secure, samesite.
- La autenticación debe regenerar el ID de sesión.
- La protección contra fuerza bruta limita intentos por IP.
- RBAC controla el acceso basado en roles y permisos.
- HTTPS debe forzarse en producción.
- El principio de mínimo privilegio aplica a BD, archivos y usuarios.
- Los logs de seguridad registran eventos críticos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Debo escapar la salida en el controlador o en la vista?**
   En la vista, justo antes de imprimir. El controlador pasa datos
   sin escapar, y la vista los escapa al mostrarlos. Esto permite
   usar los mismos datos en diferentes contextos (HTML, JSON, PDF).

❓ **¿Qué es más seguro: PDO o MySQLi?**
   Ambos son seguros si usas prepared statements. PDO es más
   recomendado porque soporta más motores de BD y tiene mejor API.

❓ **¿Cómo protejo APIs contra CSRF?**
   Las APIs que usan tokens (JWT, API keys) no son vulnerables a
   CSRF porque no dependen de cookies. Si tu API usa cookies,
   implementa SameSite=Strict y tokens CSRF en headers.

❓ **¿Qué hago si encuentro una vulnerabilidad en mi código?**
   1. Evalúa el impacto. 2. Corrige inmediatamente. 3. Despliega
   la corrección. 4. Notifica a usuarios afectados si es necesario.
   5. Documenta la lección aprendida.

❓ **¿HTML Purifier es necesario?**
   Para contenido HTML generado por usuarios (rich text), sí.
   Usa HTML Purifier o similar. Para texto plano, htmlspecialchars()
   es suficiente.

❓ **¿Debo usar filter_var o regex para validar?**
   filter_var es más seguro para casos estándar (email, URL, IP).
   Regex es más flexible para patrones personalizados. Usa ambos
   combinados.

❓ **¿Cómo manejo la seguridad en archivos .env?**
   El archivo .env nunca debe estar en el repositorio. En el
   servidor, los permisos deben ser 600 (solo el dueño lee).
   Nunca accedas a .env desde el navegador.

🏗️ **Sistema de Seguridad Integral**

   Construye un paquete de seguridad completo con:

   1. Firewall de aplicación: detecta y bloquea patrones de
      ataque comunes (SQL injection en URL, XSS en query strings).
   2. Rate limiter: limita peticiones por IP, ruta y tiempo
      (ej: 100 req/min por IP, 10 req/min por ruta).
   3. Sistema de 2FA (Two Factor Authentication) vía TOTP.
   4. Registro de auditoría: cada acción del usuario se registra
      con IP, user agent, timestamp y datos relevantes.
   5. Bloqueo automático de IPs después de múltiples ataques.
   6. Panel de seguridad en el admin: ver intentos fallidos,
      IPs bloqueadas, actividad sospechosa.
   7. Reporte semanal de seguridad enviado por email.

   Implementa una API REST para el panel de seguridad:
   ```
   GET /api/security/logs?page=1&type=login
   GET /api/security/blocked-ips
   POST /api/security/unblock-ip
   ```

   Este desafío integra todos los conceptos de seguridad del
   capítulo en un sistema profesional listo para producción.
