===============================================================================
1. INSTANCIACIÓN DE OBJETOS
===============================================================================

Un objeto es una instancia concreta de una clase. Mientras que la clase
es el modelo abstracto, el objeto ocupa memoria y tiene valores específicos
en sus propiedades. Para crear un objeto utilizamos la palabra clave `new`
seguida del nombre de la clase y paréntesis (con argumentos si el
constructor los requiere).

```php
<?php

declare(strict_types=1);

class Usuario
{
    public string $nombre;
    public string $email;
    public bool $activo = true;

    public function __construct(string $nombre, string $email)
    {
        $this->nombre = $nombre;
        $this->email = $email;
    }

    public function saludar(): string
    {
        return "Hola, soy {$this->nombre}";
    }
}

// Instanciación: creamos objetos (instancias) de la clase Usuario
$usuario1 = new Usuario('Ana García', 'ana@example.com');
$usuario2 = new Usuario('Carlos López', 'carlos@example.com');
$usuario3 = new Usuario('María Rodríguez', 'maria@example.com');

// Cada objeto es independiente y tiene su propio estado
echo $usuario1->saludar(); // Hola, soy Ana García
echo $usuario2->saludar(); // Hola, soy Carlos López
echo $usuario3->saludar(); // Hola, soy María Rodríguez
```

Los paréntesis después del nombre de la clase son opcionales si el
constructor no requiere parámetros:

```php
<?php

class Configuracion
{
    public string $entorno = 'desarrollo';
}

$config = new Configuracion();  // Válido
$config2 = new Configuracion;   // También válido, pero menos legible
```

Es recomendable usar siempre los paréntesis para mantener consistencia
y claridad. La diferencia es puramente sintáctica y no afecta el
rendimiento ni el comportamiento.

Puedes crear un objeto sin asignarlo a una variable si solo necesitas
acceder a un método o propiedad inmediatamente (aunque no es común):

```php
<?php

class Saludo
{
    public function decir(string $mensaje): string
    {
        return ">>> $mensaje <<<";
    }
}

echo (new Saludo())->decir('Hola mundo');
```

Desde PHP 8.0, puedes usar `new` en expresiones más complejas, como
dentro de argumentos de funciones:

```php
<?php

class Logger
{
    public function __construct(private string $prefix) {}
    public function log(string $msg): void { echo "[{$this->prefix}] $msg\n"; }
}

function procesar(Logger $logger): void {
    $logger->log('Procesando...');
}

procesar(new Logger('SISTEMA')); // PHP 8.0+ permite en argumentos directos
```

===============================================================================
2. ACCESO A PROPIEDADES Y MÉTODOS
===============================================================================

El operador `->` (flecha) se utiliza para acceder a propiedades y métodos
de un objeto desde fuera de la clase. La sintaxis es:

```php
$objeto->propiedad;  // Acceso a propiedad
$objeto->metodo();   // Llamada a método
```

Ejemplo completo de acceso:

```php
<?php

declare(strict_types=1);

class Coche
{
    public string $marca;
    public string $modelo;
    public int $velocidad = 0;

    public function __construct(string $marca, string $modelo)
    {
        $this->marca = $marca;
        $this->modelo = $modelo;
    }

    public function acelerar(int $incremento): void
    {
        $this->velocidad += $incremento;
    }

    public function frenar(int $decremento): void
    {
        $this->velocidad = max(0, $this->velocidad - $decremento);
    }

    public function velocidadActual(): int
    {
        return $this->velocidad;
    }
}

$coche = new Coche('Toyota', 'Corolla');

// Acceso a propiedades públicas (lectura)
echo $coche->marca;   // Toyota
echo $coche->modelo;  // Corolla

// Modificación de propiedades públicas (escritura)
$coche->marca = 'Honda';
echo $coche->marca;   // Honda

// Llamada a métodos
$coche->acelerar(50);
$coche->acelerar(30);
echo $coche->velocidadActual(); // 80

$coche->frenar(20);
echo $coche->velocidadActual(); // 60
```

Si intentas acceder a una propiedad privada o protegida desde fuera de
la clase, PHP lanzará un error fatal:

```php
<?php

class CuentaBancaria
{
    private float $saldo = 1000.00;
}

$cuenta = new CuentaBancaria();
// echo $cuenta->saldo; // Fatal error: Cannot access private property
```

El operador `->` también se puede usar en cadenas de acceso cuando una
propiedad contiene otro objeto:

```php
<?php

class Direccion
{
    public string $calle;
    public string $ciudad;
}

class Persona
{
    public string $nombre;
    public Direccion $direccion;

    public function __construct(string $nombre, Direccion $direccion)
    {
        $this->nombre = $nombre;
        $this->direccion = $direccion;
    }
}

$direccion = new Direccion();
$direccion->calle = 'Av. Principal 123';
$direccion->ciudad = 'Ciudad de México';

$persona = new Persona('Luisa', $direccion);

// Acceso encadenado
echo $persona->direccion->calle;  // Av. Principal 123
echo $persona->direccion->ciudad; // Ciudad de México
```

PHP 8.0 introdujo el operador `?->` (nullsafe) que permite acceder a
cadenas de propiedades sin lanzar errores si un elemento intermedio es
null:

```php
<?php

class Empresa
{
    public ?Direccion $direccion = null;
}

$empresa = new Empresa();
// Sin nullsafe: error si direccion es null
// echo $empresa->direccion->calle;

// Con nullsafe: devuelve null si cualquier elemento es null
echo $empresa->direccion?->calle ?? 'Sin dirección';
```

===============================================================================
3. REFERENCIAS VS COPIA DE OBJETOS
===============================================================================

Uno de los conceptos más importantes (y confusos) al trabajar con objetos
en PHP es la diferencia entre copia y referencia. En PHP, los objetos se
asignan por referencia de identidad, no por valor. Esto significa que
cuando asignas un objeto a otra variable, ambas apuntan al mismo objeto
en memoria.

```php
<?php

class Producto
{
    public string $nombre;
    public float $precio;

    public function __construct(string $nombre, float $precio)
    {
        $this->nombre = $nombre;
        $this->precio = $precio;
    }
}

$producto1 = new Producto('Laptop', 15000.00);
$producto2 = $producto1;  // Asignación por referencia (identidad)

$producto2->nombre = 'Laptop Gaming';
$producto2->precio = 22000.00;

echo $producto1->nombre; // Laptop Gaming (¡modificado!)
echo $producto1->precio; // 22000.00 (¡modificado!)

// Ambas variables apuntan al mismo objeto
var_dump($producto1 === $producto2); // bool(true)
```

Esto es fundamental para entender cómo funcionan los objetos como
parámetros de funciones:

```php
<?php

function aplicarDescuento(Producto $producto, float $porcentaje): void
{
    $producto->precio *= (1 - $porcentaje / 100);
    // No es necesario devolver el objeto, se modifica el original
}

$laptop = new Producto('Laptop', 15000.00);
aplicarDescuento($laptop, 10);
echo $laptop->precio; // 13500.00 (modificado dentro de la función)
```

Si realmente necesitas una copia independiente, debes usar la palabra
clave `clone`, que veremos en detalle en la siguiente sección.

Para verificar si dos variables apuntan al mismo objeto, se usa el
operador de identidad `===`:

```php
<?php

$a = new Producto('A', 100);
$b = $a;           // Misma referencia
$c = clone $a;     // Copia independiente
$d = new Producto('A', 100);  // Objeto diferente con mismos valores

var_dump($a === $b); // bool(true)  - misma identidad
var_dump($a === $c); // bool(false) - diferentes identidades
var_dump($a == $d);  // bool(true)  - misma igualdad (mismos valores)
```

Entender la diferencia entre `==` (igualdad de valores) y `===` (misma
identidad) es esencial para evitar bugs sutiles en tu código.

===============================================================================
4. CLONACIÓN DE OBJETOS
===============================================================================

Para crear una copia independiente de un objeto, utilizamos la palabra
clave `clone`. Esto crea un nuevo objeto con los mismos valores de
propiedades pero con una identidad diferente.

```php
<?php

$original = new Producto('Mouse', 500.00);
$copia = clone $original;

$copia->nombre = 'Mouse Inalámbrico';
$copia->precio = 750.00;

echo $original->nombre; // Mouse (sin cambios)
echo $original->precio; // 500.00 (sin cambios)
echo $copia->nombre;    // Mouse Inalámbrico
```

El problema con `clone` es que realiza una copia superficial (shallow
copy). Si el objeto contiene referencias a otros objetos, estas
referencias se copian, no los objetos en sí:

```php
<?php

class Pedido
{
    public array $productos = [];
    public string $cliente;

    public function agregarProducto(Producto $p): void
    {
        $this->productos[] = $p;
    }
}

$pedido1 = new Pedido();
$pedido1->cliente = 'Juan';
$pedido1->agregarProducto(new Producto('Laptop', 15000));

$pedido2 = clone $pedido1;
$pedido2->cliente = 'María';

// El producto dentro del array sigue siendo el mismo objeto
$pedido2->productos[0]->precio = 9999;

echo $pedido1->productos[0]->precio; // 9999 (modificado!)
```

Para solucionar esto, PHP provee el método mágico `__clone()` que se
ejecuta automáticamente después de la clonación. Dentro de este método
puedes implementar una copia profunda (deep copy):

```php
<?php

class PedidoMejorado
{
    public array $productos = [];
    public string $cliente;

    public function agregarProducto(Producto $p): void
    {
        $this->productos[] = $p;
    }

    public function __clone(): void
    {
        // Clonar cada producto del array para copia profunda
        foreach ($this->productos as $indice => $producto) {
            $this->productos[$indice] = clone $producto;
        }
    }
}

$pedido1 = new PedidoMejorado();
$pedido1->cliente = 'Juan';
$pedido1->agregarProducto(new Producto('Laptop', 15000));

$pedido2 = clone $pedido1;
$pedido2->productos[0]->precio = 9999;

echo $pedido1->productos[0]->precio; // 15000 (intacto!)
echo $pedido2->productos[0]->precio; // 9999
```

La clonación también respeta la visibilidad de las propiedades. Las
propiedades privadas se clonan correctamente. Sin embargo, hay casos
especiales como los recursos (conexiones de base de datos, archivos
abiertos) que pueden no clonarse adecuadamente.

===============================================================================
5. COMPARACIÓN DE OBJETOS
===============================================================================

PHP ofrece dos operadores para comparar objetos:

- `==` (Igualdad): Dos objetos son iguales si tienen los mismos atributos
  y valores, son instancias de la misma clase, y todas sus propiedades
  coinciden.
- `===` (Identidad): Dos objetos son idénticos si son la misma instancia
  (apuntan al mismo objeto en memoria).

```php
<?php

class Punto
{
    public function __construct(
        public int $x,
        public int $y
    ) {}
}

$p1 = new Punto(3, 4);
$p2 = new Punto(3, 4);
$p3 = new Punto(5, 6);
$p4 = $p1;

// Igualdad (==) - comparación de valores
var_dump($p1 == $p2);  // bool(true)  - mismos valores
var_dump($p1 == $p3);  // bool(false) - valores diferentes

// Identidad (===) - misma instancia
var_dump($p1 === $p2); // bool(false) - diferentes instancias
var_dump($p1 === $p4); // bool(true)  - misma instancia

// Comparación entre clases diferentes
class Punto2D
{
    public function __construct(
        public int $x,
        public int $y
    ) {}
}

$p5 = new Punto2D(3, 4);
var_dump($p1 == $p5);  // bool(false) - clases diferentes
```

Los objetos también pueden implementar la interfaz `Comparable` (no nativa
de PHP, pero puedes definirla) o el método mágico `__toString()` para
personalizar la comparación.

PHP 8.0 introdujo los objetos value (`Stringable` interface) y la
posibilidad de usar `match` con objetos:

```php
<?php

$punto = new Punto(1, 2);

$resultado = match (true) {
    $punto == new Punto(0, 0) => 'Origen',
    $punto->x > 0 && $punto->y > 0 => 'Primer cuadrante',
    default => 'Otra ubicación',
};
```

NOTA: La comparación con `==` no invoca ningún método mágico de manera
automática. Si necesitas lógica personalizada, implementa tu propio
método como `esIgualA(Punto $otro): bool`.

6. SERIALIZACIÓN DE OBJETOS
===============================================================================

La serialización convierte un objeto en una representación textual (string)
que puede ser almacenada o transmitida. La deserialización reconstruye el
objeto a partir de ese string.

PHP proporciona las funciones `serialize()` y `unserialize()`:

```php
<?php

class ConfiguracionUsuario
{
    public string $idioma = 'es';
    public string $tema = 'oscuro';
    public int $notificaciones = 1;
    private string $token = 'secreto';

    public function __serialize(): array
    {
        // Personalizar qué propiedades se serializan (PHP 7.4+)
        return [
            'idioma' => $this->idioma,
            'tema' => $this->tema,
            'notificaciones' => $this->notificaciones,
            // Excluimos token por seguridad
        ];
    }

    public function __unserialize(array $data): void
    {
        $this->idioma = $data['idioma'];
        $this->tema = $data['tema'];
        $this->notificaciones = $data['notificaciones'];
        $this->token = 'pendiente';
    }
}

$config = new ConfiguracionUsuario();
$serializado = serialize($config);
echo $serializado;
// Formato: O:22:"ConfiguracionUsuario":3:{...}

$restaurado = unserialize($serializado);
var_dump($restaurado);
```

Desde PHP 7.4, los métodos `__serialize()` y `__unserialize()` reemplazan
a los antiguos `__sleep()` y `__wakeup()`. Se recomienda usar los nuevos.

La serialización es útil para:

- Almacenar objetos en sesiones (PHP lo hace automáticamente).
- Cachear objetos complejos.
- Enviar objetos entre procesos.
- Almacenar estado en archivos o bases de datos.

Sin embargo, la serialización nativa de PHP tiene limitaciones de
seguridad: `unserialize()` puede ejecutar código arbitrario si se
alimenta con datos maliciosos. Nunca deserialices datos de fuentes
no confiables sin sanitización.

Una alternativa más segura para intercambio de datos es JSON:

```php
<?php

class Producto implements JsonSerializable
{
    public function __construct(
        public string $nombre,
        public float $precio,
        private string $codigoInterno
    ) {}

    public function jsonSerialize(): mixed
    {
        return [
            'nombre' => $this->nombre,
            'precio' => $this->precio,
            // Excluimos codigoInterno
        ];
    }
}

$producto = new Producto('Laptop', 15000, 'INT-001');
echo json_encode($producto);
// {"nombre":"Laptop","precio":15000}
```

La interfaz `JsonSerializable` permite controlar cómo se convierte un
objeto a JSON mediante la función `json_encode()`.

===============================================================================
7. RECORRIDO DE PROPIEDADES CON FOREACH
===============================================================================

PHP permite recorrer las propiedades públicas de un objeto con un bucle
`foreach`. También puedes implementar la interfaz `Iterator` o
`IteratorAggregate` para personalizar el recorrido.

```php
<?php

class Empleado
{
    public string $nombre;
    public string $puesto;
    public float $salario;
    private string $seguridadSocial;  // No se incluye en foreach

    public function __construct(
        string $nombre,
        string $puesto,
        float $salario,
        string $seguridadSocial
    ) {
        $this->nombre = $nombre;
        $this->puesto = $puesto;
        $this->salario = $salario;
        $this->seguridadSocial = $seguridadSocial;
    }
}

$empleado = new Empleado(
    'Roberto Méndez',
    'Desarrollador Senior',
    85000.00,
    'SS-1234-5678'
);

echo "Propiedades del empleado:\n";
foreach ($empleado as $propiedad => $valor) {
    echo "$propiedad: $valor\n";
}
// nombre: Roberto Méndez
// puesto: Desarrollador Senior
// salario: 85000.00
// (seguridadSocial NO aparece porque es privada)
```

Para controlar el recorrido de manera más precisa, implementa
`IteratorAggregate`:

```php
<?php

class DirectorioEmpleados implements IteratorAggregate
{
    private array $empleados = [];

    public function agregar(Empleado $e): void
    {
        $this->empleados[] = $e;
    }

    public function getIterator(): Traversable
    {
        return new ArrayIterator($this->empleados);
    }
}

$directorio = new DirectorioEmpleados();
$directorio->agregar(new Empleado('Ana', 'CEO', 150000, 'SS-0001'));
$directorio->agregar(new Empleado('Luis', 'CTO', 130000, 'SS-0002'));

foreach ($directorio as $empleado) {
    echo $empleado->nombre . "\n";
}
// Ana
// Luis
```

La implementación de `Iterator` (manual con current, next, key, valid,
rewind) ofrece control más detallado pero requiere más código:

```php
<?php

class RangoNumeros implements Iterator
{
    private int $actual;
    private int $inicio;
    private int $fin;

    public function __construct(int $inicio, int $fin)
    {
        $this->inicio = $inicio;
        $this->fin = $fin;
    }

    public function current(): int
    {
        return $this->actual;
    }

    public function key(): int
    {
        return $this->actual - $this->inicio;
    }

    public function next(): void
    {
        $this->actual++;
    }

    public function rewind(): void
    {
        $this->actual = $this->inicio;
    }

    public function valid(): bool
    {
        return $this->actual <= $this->fin;
    }
}

$rango = new RangoNumeros(1, 5);
foreach ($rango as $clave => $valor) {
    echo "$clave: $valor\n";
}
// 0: 1, 1: 2, 2: 3, 3: 4, 4: 5
```

El recorrido de objetos con `foreach` es una característica poderosa que
permite tratar objetos como si fueran colecciones.

8. TYPE HINTING CON OBJETOS
===============================================================================

PHP permite declarar tipos de objeto en parámetros de funciones, valores
de retorno y propiedades. Esto proporciona seguridad de tipos y permite
que el IDE ofrezca autocompletado.

```php
<?php

declare(strict_types=1);

interface Notificable
{
    public function enviar(string $mensaje): bool;
}

class EmailNotificacion implements Notificable
{
    public function enviar(string $mensaje): bool
    {
        echo "Enviando email: $mensaje\n";
        return true;
    }
}

class SMSNotificacion implements Notificable
{
    public function enviar(string $mensaje): bool
    {
        echo "Enviando SMS: $mensaje\n";
        return true;
    }
}

// Type hinting con interfaz (polimorfismo)
function procesarNotificacion(Notificable $notificador, string $msg): bool
{
    return $notificador->enviar($msg);
}

// Type hinting con clase concreta
class GestorNotificaciones
{
    private Notificable $canal;

    public function __construct(Notificable $canal)
    {
        $this->canal = $canal;
    }

    public function notificar(string $mensaje): bool
    {
        return $this->canal->enviar($mensaje);
    }
}

$email = new EmailNotificacion();
$sms = new SMSNotificacion();

procesarNotificacion($email, 'Bienvenido a PHP Master Book');
procesarNotificacion($sms, 'Tu código de verificación es 1234');

$gestor = new GestorNotificaciones($email);
$gestor->notificar('Notificación desde el gestor');
```

El type hinting con objetos permite:

- Validación automática en tiempo de ejecución.
- Polimorfismo (una función acepta cualquier objeto que implemente una
  interfaz o extienda una clase).
- Código autodocumentado.
- Detección temprana de errores.
- Mejor integración con herramientas de análisis estático.

PHP también soporta tipos unión con objetos desde PHP 8.0:

```php
<?php

class Resultado
{
    public function __construct(
        public Usuario|Cliente|null $entidad
    ) {}
}

// También en parámetros
function guardarDocumento(Documento|string $origen): void
{
    if ($origen instanceof Documento) {
        // Procesar como objeto
    } else {
        // Procesar como string (ruta de archivo)
    }
}
```

El operador `instanceof` verifica si un objeto es instancia de una clase
específica o implementa una interfaz:

```php
<?php

function analizarObjeto(object $obj): string
{
    return match (true) {
        $obj instanceof Producto => 'Es un producto',
        $obj instanceof Usuario => 'Es un usuario',
        $obj instanceof Notificable => 'Es notificable',
        default => 'Tipo desconocido'
    };
}
```

La combinación de type hinting y `instanceof` permite escribir código
seguro, expresivo y robusto.

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Los objetos son instancias de clases, creadas con `new NombreClase()`.
- Se accede a propiedades y métodos públicos con el operador `->`.
- Los objetos se asignan por referencia (identidad), no por valor.
- `clone` crea una copia superficial; `__clone()` permite copia profunda.
- `==` compara valores de propiedades; `===` compara identidad de objeto.
- `serialize()`/`unserialize()` convierten objetos a/desde strings.
- `__serialize()` y `__unserialize()` personalizan el proceso (PHP 7.4+).
- `foreach` recorre propiedades públicas; Iterator permite control total.
- Type hinting con interfaces y clases para código robusto y polimórfico.
- `instanceof` verifica el tipo de un objeto en tiempo de ejecución.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuántos objetos puedo crear de una misma clase?**
No hay límite teórico más allá de la memoria disponible. Puedes crear
tantos objetos como necesites. Cada objeto ocupa memoria para sus
propiedades. En aplicaciones web típicas, cientos o miles de objetos
son perfectamente manejables.

❓ **¿Qué significa "Fatal error: Cannot use object of type X as array"?**
Significa que estás intentando acceder a un objeto como si fuera un
array (usando corchetes $objeto['clave']). Los objetos no son arrays
a menos que implementen `ArrayAccess`. Si necesitas ese comportamiento,
implementa la interfaz `ArrayAccess` en tu clase.

❓ **¿Cómo libero la memoria de un objeto que ya no necesito?**
PHP tiene un recolector de basura (garbage collector) que libera
automáticamente la memoria cuando no hay más referencias al objeto.
Puedes ayudar asignando `null` a la variable: `$objeto = null;`. Esto
elimina la referencia y el GC puede recolectar el objeto.

❓ **¿Por qué al pasar un objeto a una función se modifica el original?**
Porque los objetos se pasan por referencia (handle). La función recibe
el identificador del objeto, no una copia. Cualquier modificación
dentro de la función afecta al objeto original. Si necesitas evitar
esto, clona el objeto antes de pasarlo.

❓ **¿Puedo crear objetos sin una clase definida?**
Sí, usando `stdClass` o creando un objeto con cast desde array:
`(object) ['nombre' => 'Juan']`. Sin embargo, esto es limitado y no
recomendado para código serio. stdClass no tiene métodos y no soporta
herencia. Siempre es mejor definir clases explícitamente.

❓ **¿Qué ocurre si serializo un objeto con recursos (conexiones BD)?**
Los recursos no se pueden serializar. Si un objeto contiene un recurso
(como una conexión de base de datos), la serialización fallará o el
recurso se perderá durante la deserialización. Debes reconectar
manualmente en `__unserialize()` o `__wakeup()`.
