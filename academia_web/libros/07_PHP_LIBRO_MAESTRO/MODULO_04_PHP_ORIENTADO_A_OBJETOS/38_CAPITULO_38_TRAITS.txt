===============================================================================
1. ¿QUÉ ES UN TRAIT?
===============================================================================

Un trait es un mecanismo de reutilización de código en PHP que permite
agrupar métodos (y propiedades) para ser incluidos en clases, sin usar
herencia. A diferencia de las clases, los traits no pueden ser
instanciados por sí mismos. Su único propósito es ser "inyectados" en
clases.

PHP introdujo los traits en la versión 5.4.0 como solución al problema
de la herencia única: proporcionan una forma de reutilizar código
horizontalmente.

```php
<?php

declare(strict_types=1);

// Definición de un trait
trait Logeable
{
    public function log(string $mensaje, string $nivel = 'INFO'): void
    {
        $fecha = date('Y-m-d H:i:s');
        echo "[$fecha] [$nivel] $mensaje\n";
    }

    public function logError(string $mensaje): void
    {
        $this->log($mensaje, 'ERROR');
    }

    public function logInfo(string $mensaje): void
    {
        $this->log($mensaje, 'INFO');
    }
}

// Clases no relacionadas que usan el mismo trait
class Usuario
{
    use Logeable;

    public function __construct(
        private string $nombre
    ) {}

    public function guardar(): void
    {
        // Lógica de guardado...
        $this->logInfo("Usuario '{$this->nombre}' guardado correctamente");
    }
}

class Pedido
{
    use Logeable;

    public function __construct(
        private string $id
    ) {}

    public function procesar(): void
    {
        try {
            // Lógica de procesamiento...
            $this->logInfo("Pedido {$this->id} procesado");
        } catch (\Exception $e) {
            $this->logError("Error procesando pedido {$this->id}: {$e->getMessage()}");
            throw $e;
        }
    }
}

class ProcesadorPagos
{
    use Logeable;

    public function cobrar(float $monto): void
    {
        $this->logInfo("Cobrando \${$monto}");
        // Lógica de cobro...
        $this->log("Transacción completada", 'DEBUG');
    }
}

$usuario = new Usuario('Ana');
$usuario->guardar(); // [2026-07-01 15:30:00] [INFO] Usuario 'Ana' guardado correctamente

$pedido = new Pedido('ORD-001');
$pedido->procesar(); // [2026-07-01 15:30:00] [INFO] Pedido ORD-001 procesado

$pagos = new ProcesadorPagos();
$pagos->cobrar(250.00);
```

Los traits permiten que clases que no comparten herencia (Usuario,
Pedido, ProcesadorPagos) compartan el mismo código de logging, evitando
duplicación y manteniendo una única fuente de verdad.

Características de los traits:

- Pueden tener métodos públicos, protegidos y privados.
- Pueden tener propiedades (desde PHP 5.4).
- Pueden tener métodos abstractos (las clases que usen el trait deben
  implementarlos).
- Pueden usar otros traits (composición de traits).
- No pueden ser instanciados directamente.
- No pueden tener constantes (hasta PHP 8.1, donde se añadió soporte
  para constantes en traits).

===============================================================================
2. DECLARACIÓN Y USO DE TRAITS
===============================================================================

Sintaxis de declaración:

```php
<?php

trait NombreTrait
{
    // Propiedades
    private array $datos = [];

    // Métodos
    public function metodoPublico(): void { /* ... */ }
    protected function metodoProtegido(): void { /* ... */ }
    private function metodoPrivado(): void { /* ... */ }

    // Métodos abstractos (las clases que usen el trait deben implementarlos)
    abstract public function metodoRequerido(): string;
}
```

Sintaxis de uso en una clase:

```php
<?php

trait Exportable
{
    public function exportarJSON(): string
    {
        return json_encode($this->getDatosExportables());
    }

    public function exportarArray(): array
    {
        return $this->getDatosExportables();
    }

    // Método abstracto: la clase debe implementarlo
    abstract protected function getDatosExportables(): array;
}

trait Validable
{
    public function validar(): bool
    {
        $errores = $this->validarCampos();
        return empty($errores);
    }

    public function getErrores(): array
    {
        return $this->validarCampos();
    }

    abstract protected function validarCampos(): array;
}

// Clase que usa múltiples traits
class FormularioContacto
{
    use Exportable, Validable;

    public function __construct(
        private string $nombre,
        private string $email,
        private string $mensaje
    ) {}

    // Implementación de métodos abstractos de los traits
    protected function getDatosExportables(): array
    {
        return [
            'nombre' => $this->nombre,
            'email' => $this->email,
            'mensaje' => $this->mensaje,
        ];
    }

    protected function validarCampos(): array
    {
        $errores = [];
        if (strlen($this->nombre) < 2) $errores[] = 'Nombre demasiado corto';
        if (!filter_var($this->email, FILTER_VALIDATE_EMAIL)) $errores[] = 'Email inválido';
        if (empty($this->mensaje)) $errores[] = 'Mensaje vacío';
        return $errores;
    }
}

$form = new FormularioContacto('Ana', 'ana@example.com', 'Hola, quiero información');
echo $form->exportarJSON(); // {"nombre":"Ana","email":"ana@example.com","mensaje":"..."}
var_dump($form->validar()); // bool(true)
```

Los traits pueden definir propiedades, pero con cuidado:

```php
<?php

trait ContadorTrait
{
    private int $contador = 0;

    public function incrementar(): void
    {
        $this->contador++;
    }

    public function obtenerContador(): int
    {
        return $this->contador;
    }
}

class ObjetoConContador
{
    use ContadorTrait;
}

$obj1 = new ObjetoConContador();
$obj2 = new ObjetoConContador();

$obj1->incrementar();
$obj1->incrementar();

echo $obj1->obtenerContador(); // 2
echo $obj2->obtenerContador(); // 0 (cada instancia tiene su propio contador)
```

Si dos traits definen la misma propiedad, se produce un error fatal a
menos que se resuelva con una declaración en la clase.

===============================================================================
3. RESOLUCIÓN DE CONFLICTOS
===============================================================================

Cuando una clase usa múltiples traits, o un trait y una clase padre tienen
métodos con el mismo nombre, PHP provee dos operadores para resolver
conflictos:

1. `insteadof`: Especifica qué implementación usar (reemplaza la otra).
2. `as`: Crea un alias para un método, permitiendo acceder a la versión
   reemplazada bajo otro nombre.

```php
<?php

trait A
{
    public function hablar(): string
    {
        return "Trait A dice Hola";
    }

    public function despedir(): string
    {
        return "Trait A dice Adiós";
    }
}

trait B
{
    public function hablar(): string
    {
        return "Trait B dice Hello";
    }

    public function despedir(): string
    {
        return "Trait B dice Goodbye";
    }
}

class Conflictiva
{
    use A, B {
        // Usar el método hablar() de A en lugar de B
        A::hablar insteadof B;

        // Usar el método despedir() de B en lugar de A
        B::despedir insteadof A;

        // Crear alias para acceder al método reemplazado
        B::hablar as hablarB;
        A::despedir as despedirA;
    }

    public function probar(): void
    {
        echo $this->hablar() . "\n";     // Trait A dice Hola
        echo $this->hablarB() . "\n";    // Trait B dice Hello (alias)
        echo $this->despedir() . "\n";   // Trait B dice Goodbye
        echo $this->despedirA() . "\n";  // Trait A dice Adiós (alias)
    }
}

$obj = new Conflictiva();
$obj->probar();
```

El operador `as` también puede cambiar la visibilidad de un método:

```php
<?php

trait MetodosPrivados
{
    public function metodoPublico(): string
    {
        return "público";
    }

    protected function metodoProtegido(): string
    {
        return "protegido";
    }

    private function metodoPrivado(): string
    {
        return "privado";
    }
}

class VisibilidadModificada
{
    use MetodosPrivados {
        // Cambiar visibilidad de métodos importados
        metodoPublico as private;
        metodoProtegido as public;
        // Crear alias con diferente visibilidad (manteniendo el original)
        metodoPrivado as public metodoPrivadoPublico;
    }

    public function probarInterno(): string
    {
        return $this->metodoPublico(); // Ahora es privado, pero funciona internamente
    }
}

$obj = new VisibilidadModificada();

// echo $obj->metodoPublico(); // Error: ahora es privado
echo $obj->metodoProtegido();   // Ahora es público: "protegido"
echo $obj->metodoPrivadoPublico(); // "privado" mediante alias público
echo $obj->probarInterno();     // "público" desde dentro
```

La sintaxis completa de resolución de conflictos es:

```php
use TraitA, TraitB {
    TraitA::metodo insteadof TraitB;  // Usar implementación de A
    TraitB::metodo as aliasMetodo;    // Alias para la versión de B
    TraitA::otroMetodo as protected;  // Cambiar visibilidad
    TraitB::otroMetodo as private aliasMetodoPrivado;  // Combinar
}
```

===============================================================================
4. COMPOSICIÓN DE TRAITS
===============================================================================

Los traits pueden usar otros traits, permitiendo construir comportamientos
complejos a partir de piezas más pequeñas.

```php
<?php

// Traits base (piezas pequeñas)
trait Timestampable
{
    private DateTime $creadoEn;
    private DateTime $actualizadoEn;

    public function inicializarTimestamps(): void
    {
        $this->creadoEn = new DateTime();
        $this->actualizadoEn = new DateTime();
    }

    public function getCreadoEn(): DateTime { return $this->creadoEn; }
    public function getActualizadoEn(): DateTime { return $this->actualizadoEn; }

    public function marcarActualizado(): void
    {
        $this->actualizadoEn = new DateTime();
    }
}

trait SoftDeletable
{
    private ?DateTime $eliminadoEn = null;

    public function eliminarSuave(): void
    {
        $this->eliminadoEn = new DateTime();
    }

    public function restaurar(): void
    {
        $this->eliminadoEn = null;
    }

    public function isEliminado(): bool
    {
        return $this->eliminadoEn !== null;
    }

    public function getEliminadoEn(): ?DateTime
    {
        return $this->eliminadoEn;
    }
}

trait Blameable
{
    private string $creadoPor = 'sistema';
    private string $actualizadoPor = 'sistema';

    public function setCreadoPor(string $usuario): void
    {
        $this->creadoPor = $usuario;
    }

    public function setActualizadoPor(string $usuario): void
    {
        $this->actualizadoPor = $usuario;
    }

    public function getCreadoPor(): string { return $this->creadoPor; }
    public function getActualizadoPor(): string { return $this->actualizadoPor; }
}

// Trait compuesto que agrupa funcionalidad común
trait Auditable
{
    use Timestampable, SoftDeletable, Blameable;

    // Método adicional que combina funcionalidad de los subtraits
    public function getAuditoriaCompleta(): array
    {
        return [
            'creado_en' => $this->getCreadoEn()->format('Y-m-d H:i:s'),
            'actualizado_en' => $this->getActualizadoEn()->format('Y-m-d H:i:s'),
            'creado_por' => $this->getCreadoPor(),
            'actualizado_por' => $this->getActualizadoPor(),
            'eliminado' => $this->isEliminado(),
            'eliminado_en' => $this->getEliminadoEn()?->format('Y-m-d H:i:s'),
        ];
    }
}

// Clase que usa el trait compuesto (hereda todos los métodos)
class EntidadBase
{
    use Auditable;

    public function __construct()
    {
        $this->inicializarTimestamps();
    }
}

class Producto extends EntidadBase
{
    public function __construct(
        private string $nombre,
        private float $precio
    ) {
        parent::__construct();
    }

    public function actualizarPrecio(float $nuevoPrecio): void
    {
        $this->precio = $nuevoPrecio;
        $this->marcarActualizado();
    }
}

$producto = new Producto('Laptop', 15000);
sleep(1);
$producto->actualizarPrecio(14000);
$producto->setActualizadoPor('admin@example.com');

print_r($producto->getAuditoriaCompleta());
// Array con timestamps, blame, y softdelete info

$producto->eliminarSuave();
var_dump($producto->isEliminado()); // true
$producto->restaurar();
var_dump($producto->isEliminado()); // false
```

La composición de traits permite construir "módulos de comportamiento"
reutilizables que pueden combinarse según las necesidades de cada clase,
siguiendo el principio de responsabilidad única (SRP).

===============================================================================
5. TRAITS VS HERENCIA VS INTERFACES VS COMPOSICIÓN
===============================================================================

Elegir el mecanismo correcto es crucial. Aquí tienes una guía:

```php
<?php

// HERENCIA: "es un" - relación jerárquica fuerte
class Animal { /* ... */ }
class Perro extends Animal { /* ... */ } // Un Perro ES UN Animal

// INTERFAZ: "puede hacer" - contrato de capacidades
interface Volador { public function volar(): void; }
class Avion implements Volador { public function volar(): void { /* ... */ } }

// TRAIT: "tiene capacidad de" - comportamiento reutilizable horizontal
trait Loggeable { public function log(string $m): void { /* ... */ } }
class Servicio { use Loggeable; }

// COMPOSICIÓN: "tiene un" - delega comportamiento a otro objeto
class Coche {
    private Motor $motor; // Coche TIENE UN Motor
    public function __construct(Motor $motor) { $this->motor = $motor; }
    public function acelerar(): void { $this->motor->encender(); }
}
```

**Cuándo usar cada uno:**

| Situación | Mecanismo |
|-----------|-----------|
| Relación "es un" con comportamiento base | Herencia (extends) |
| Contrato transversal de capacidades | Interfaz (implements) |
| Comportamiento compartido sin herencia | Trait (use) |
| Delegación de funcionalidad | Composición |
| Necesito estado + comportamiento base | Clase abstracta |
| Necesito múltiples fuentes de comportamiento | Traits o interfaces múltiples |

Limitaciones de traits a considerar:

1. **Conflicto de nombres**: Si dos traits definen el mismo método, debes
   resolverlo explícitamente con `insteadof` o `as`.

2. **Dependencia oculta**: Una clase que usa un trait adquiere dependencias
   de sus métodos abstractos, pero esto no siempre es evidente.

3. **Dificultad de depuración**: El comportamiento puede venir de múltiples
   traits, haciendo el flujo menos obvio.

4. **No son reemplazables en tiempo de ejecución**: A diferencia de la
   composición, no puedes cambiar el trait usado por una instancia.

**Mejores prácticas con traits:**

- Traits pequeños y enfocados (una sola responsabilidad).
- Nombres que terminen en "-able" o describan la capacidad.
- Documentar los métodos abstractos que requiere el trait.
- Evitar propiedades en traits (prefiere métodos que accedan a
  propiedades de la clase).
- No usar traits como "muletas" para evitar un diseño mejor.
- Preferir composición sobre traits cuando sea posible.

===============================================================================
6. TRAITS CON PROPIEDADES Y MÉTODOS ESTÁTICOS
===============================================================================

Los traits pueden definir propiedades estáticas y métodos estáticos:

```php
<?php

trait ContadorGlobal
{
    private static int $totalInstancias = 0;

    public static function getTotalInstancias(): int
    {
        return self::$totalInstancias;
    }
}

trait FactoryMethod
{
    public static function crear(array $datos): static
    {
        // static::class se refiere a la clase que usa el trait
        $instancia = new static();
        foreach ($datos as $propiedad => $valor) {
            if (property_exists($instancia, $propiedad)) {
                $instancia->$propiedad = $valor;
            }
        }
        return $instancia;
    }
}

class UsuarioEstatico
{
    use ContadorGlobal, FactoryMethod;

    public string $nombre;
    public string $email;

    public function __construct()
    {
        static::$totalInstancias++;
    }
}

class ProductoEstatico
{
    use ContadorGlobal, FactoryMethod;

    public string $nombre;
    public float $precio;

    public function __construct()
    {
        static::$totalInstancias++;
    }
}

$u1 = UsuarioEstatico::crear(['nombre' => 'Ana', 'email' => 'ana@mail.com']);
$u2 = UsuarioEstatico::crear(['nombre' => 'Luis', 'email' => 'luis@mail.com']);
$p1 = ProductoEstatico::crear(['nombre' => 'Laptop', 'precio' => 15000]);

echo UsuarioEstatico::getTotalInstancias();  // 2
echo ProductoEstatico::getTotalInstancias(); // 1
```

NOTA IMPORTANTE: `self::$totalInstancias` en el trait se refiere a la
clase que usa el trait, no al trait mismo. Esto se debe al late static
binding. Cada clase tiene su propia copia de la propiedad estática.

Si necesitas que el contador sea global (compartido entre todas las
clases que usen el trait), usa un nombre de propiedad calificado:

```php
<?php

trait ContadorGlobalCompartido
{
    private static array $contadores = [];

    public static function incrementarContador(): void
    {
        $clase = static::class;
        self::$contadores[$clase] = (self::$contadores[$clase] ?? 0) + 1;
    }

    public static function obtenerContador(): int
    {
        return self::$contadores[static::class] ?? 0;
    }
}
```

Los traits también pueden tener métodos abstractos que requieren que
la clase los implemente. Esto es útil para definir "puntos de extensión":

```php
<?php

trait CacheableTrait
{
    public function obtenerCacheado(string $key): mixed
    {
        $ttl = $this->getTTL();  // Método que la clase debe proveer
        // Lógica de caché usando $ttl...
        return null;
    }

    // La clase que use este trait DEBE implementar getTTL()
    abstract public function getTTL(): int;
}

class ServicioLento
{
    use CacheableTrait;

    public function getTTL(): int
    {
        return 3600; // 1 hora
    }
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Los traits son mecanismos de reutilización horizontal de código en PHP.
- Se declaran con `trait` y se incluyen en clases con `use`.
- Múltiples traits pueden usarse en una misma clase.
- `insteadof` resuelve conflictos eligiendo qué implementación usar.
- `as` crea alias o cambia visibilidad de métodos importados.
- Los traits pueden componerse de otros traits (composición).
- Traits vs Herencia: herencia para "es un", traits para "tiene
  capacidad de".
- Prefiere composición sobre traits cuando sea posible.
- Los traits pueden tener métodos abstractos, propiedades (estáticas y
  de instancia), y métodos estáticos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo instanciar un trait directamente?**
No. Los traits no pueden ser instanciados. Son solo un mecanismo para
agrupar código que se inyecta en clases. Si intentas `new MiTrait()`,
PHP lanza un error fatal.

❓ **¿Un trait puede implementar una interfaz?**
No directamente. Un trait no puede implementar interfaces. Pero una
clase que usa un trait puede implementar la interfaz, y los métodos
del trait pueden satisfacer los requisitos de la interfaz.

❓ **¿Puedo cambiar la visibilidad de un método importado de un trait?**
Sí, usando `as` con el nuevo modificador de visibilidad:
`use Trait { metodo as private; }` o `use Trait { metodo as public nuevoNombre; }`.

❓ **¿Qué pasa si dos traits tienen una propiedad con el mismo nombre?**
Si dos traits definen la misma propiedad con el mismo nombre, y la clase
no la redefine, PHP lanza un error fatal por conflicto. Puedes resolverlo
declarando la propiedad en la clase para sobrescribir ambas.

❓ **¿Los traits afectan el rendimiento?**
El impacto es mínimo. Los métodos de traits se compilan igual que los
métodos de clase. No hay penalización significativa en tiempo de
ejecución comparado con métodos definidos directamente en la clase.

❓ **¿Cuándo NO debo usar un trait?**
Evita traits cuando: el comportamiento se necesita en tiempo de ejecución
(usa composición/estrategia), la relación es claramente "es un" (usa
herencia), o solo necesitas un contrato (usa interfaz). También evita
traits muy grandes o con muchas responsabilidades.

===============================================================================
DESAFÍO DEL ARQUITECTO
===============================================================================

🏗️ **Sistema de ORM Sencillo con Traits** — Diseña un mini ORM (Object-
Relational Mapping) que demuestre todos los conceptos de traits:
definición, uso múltiple, composición, resolución de conflictos, métodos
abstractos en traits, y propiedades de traits.

Traits requeridos:

1. `HasTimestamps` — Proporciona `creadoEn`, `actualizadoEn` (DateTime).
   Métodos: `inicializarTimestamps()`, `marcarActualizado()`, `getCreadoEn()`,
   `getActualizadoEn()`. El constructor de la clase debe llamar a
   `inicializarTimestamps()`.

2. `HasSoftDelete` — Proporciona `eliminadoEn` (?DateTime). Métodos:
   `eliminar()`, `restaurar()`, `isEliminado(): bool`. La propiedad
   `eliminadoEn` debe ser inicializada como null.

3. `HasSlug` — Genera un slug único a partir del título/nombre.
   Propiedad `slug` (string). Métodos: `generarSlug(string $texto): string`,
   `getSlug(): string`. Implementa un algoritmo que convierta texto a
   slug (minúsculas, reemplazar espacios por guiones, eliminar
   caracteres especiales). Método abstracto: `getSlugSource(): string`
   (la clase dice qué campo usar para generar el slug).

4. `HasRelationships` — Proporciona métodos para definir relaciones.
   Propiedades: `$relaciones` (array). Métodos: `belongsTo(string $clase,
   string $foreignKey): static`, `hasMany(string $clase, string $foreignKey):
   static`, `getRelation(string $nombre): ?array`. Método abstracto:
   `getTableName(): string`.

5. `QueryBuilderTrait` — Proporciona métodos de consulta fluida.
   Métodos: `where(string $campo, string $operador, mixed $valor): static`,
   `orderBy(string $campo, string $direccion = 'ASC'): static`,
   `limit(int $limite): static`, `getQuery(): string` (genera SQL),
   `reset(): static`. Almacena internamente condiciones, orden y límite.

6. `ModelBase` — Trait compuesto que agrupa todos los traits anteriores
   y añade: `save(): bool`, `delete(): bool`, `find(int $id): ?static`,
   `all(): array`. Requiere que la clase implemente `getTableName(): string`
   y `getPrimaryKey(): string`. Resuelve conflictos entre `HasSoftDelete::eliminar`
   y el método `delete()` de ModelBase (usa `insteadof` para que `delete()`
   de ModelBase llame a `eliminar()` de HasSoftDelete suave).

Clases del modelo:

7. `Articulo` — Usa `ModelBase`. Propiedades: id, titulo, contenido,
   autorId, publicado. Implementa `getSlugSource()` devolviendo `titulo`.
   Define relación `belongsTo('Autor', 'autorId')`.

8. `Autor` — Usa `ModelBase`. Propiedades: id, nombre, email, bio.
   Define relación `hasMany('Articulo', 'autorId')`.

9. `Categoria` — Usa `ModelBase`. Propiedades: id, nombre, descripcion.

Requisitos adicionales:

- Resolver al menos un conflicto entre traits con `insteadof` y `as`.
- Demostrar que `Articulo` y `Autor` heredan comportamiento de
  timestamps, soft delete, slug, relaciones y query builder.
- Incluir métodos estáticos en al menos un trait.
- El trait compuesto `ModelBase` debe usar `insteadof` para resolver
  conflictos entre `HasSoftDelete::eliminar` y un método propio del trait.
- Script de demostración: crear artículos y autores, generar slugs,
  usar soft delete, construir consultas con QueryBuilderTrait, y mostrar
  relaciones.

Este desafío integra todos los conceptos de traits y demuestra cómo
pueden usarse para construir un sistema base reutilizable que evita la
herencia forzada y proporciona comportamiento transversal a múltiples
modelos.
