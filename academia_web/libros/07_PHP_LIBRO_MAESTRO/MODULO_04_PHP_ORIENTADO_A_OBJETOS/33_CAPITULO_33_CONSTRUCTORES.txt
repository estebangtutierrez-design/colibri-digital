===============================================================================
1. EL MÉTODO __construct()
===============================================================================

El método `__construct()` es un método mágico que PHP ejecuta
automáticamente cuando se crea un nuevo objeto con `new`. Es el lugar
ideal para inicializar propiedades, validar parámetros, establecer
conexiones y realizar cualquier configuración inicial.

Sintaxis básica:

```php
<?php

declare(strict_types=1);

class Usuario
{
    public string $nombre;
    public string $email;
    public bool $activo;
    public DateTime $fechaRegistro;

    // Constructor tradicional
    public function __construct(string $nombre, string $email)
    {
        $this->nombre = $nombre;
        $this->email = $email;
        $this->activo = true;
        $this->fechaRegistro = new DateTime();
    }
}

$usuario = new Usuario('Ana García', 'ana@example.com');
echo $usuario->nombre;         // Ana García
echo $usuario->activo;         // 1 (true)
echo $usuario->fechaRegistro->format('Y-m-d'); // Fecha actual
```

Características importantes de `__construct()`:

- No puede devolver un valor (excepto implícitamente void). PHP ignorará
  cualquier valor de retorno.
- Puede lanzar excepciones si la inicialización falla.
- Puede tener parámetros opcionales y valores por defecto.
- Solo puede haber un constructor por clase (no hay sobrecarga como
  en otros lenguajes, pero se pueden simular con parámetros opcionales
  o métodos de fábrica estáticos).
- Si no se define un constructor, PHP proporciona uno vacío por defecto.

Ejemplo con tipos unión y nullable:

```php
<?php

class Pedido
{
    public string $producto;
    public int $cantidad;
    public ?float $descuento;
    public string|int $identificador;

    public function __construct(
        string $producto,
        int $cantidad,
        ?float $descuento = null,
        string|int $identificador = 'SIN-ID'
    ) {
        $this->producto = $producto;
        $this->cantidad = $cantidad;
        $this->descuento = $descuento;
        $this->identificador = $identificador;
    }
}

$pedido1 = new Pedido('Laptop', 2, 10.5, 'PED-001');
$pedido2 = new Pedido('Mouse', 5); // usa valores por defecto
```

Los constructores también pueden definir lógica de validación:

```php
<?php

class CuentaBancaria
{
    public readonly string $numeroCuenta;
    public readonly string $titular;
    private float $saldo;

    public function __construct(string $numeroCuenta, string $titular, float $saldoInicial = 0.0)
    {
        // Validaciones
        if (strlen($numeroCuenta) < 10) {
            throw new InvalidArgumentException(
                'El número de cuenta debe tener al menos 10 dígitos.'
            );
        }

        if (empty(trim($titular))) {
            throw new InvalidArgumentException(
                'El titular no puede estar vacío.'
            );
        }

        if ($saldoInicial < 0) {
            throw new InvalidArgumentException(
                'El saldo inicial no puede ser negativo.'
            );
        }

        $this->numeroCuenta = $numeroCuenta;
        $this->titular = $titular;
        $this->saldo = $saldoInicial;
    }
}

try {
    $cuenta = new CuentaBancaria('123', 'Juan', -100);
} catch (InvalidArgumentException $e) {
    echo $e->getMessage(); // El número de cuenta debe tener al menos 10 dígitos.
}
```

La validación en el constructor es una aplicación del principio "fail
fast": si los datos son inválidos, es mejor fallar inmediatamente en
la creación del objeto que tener un objeto en estado inconsistente.

===============================================================================
2. CONSTRUCTOR PROPERTY PROMOTION (PHP 8.0+)
===============================================================================

PHP 8.0 introdujo una de las características más productivas del lenguaje:
la promoción de propiedades del constructor. Esta sintaxis permite declarar
y asignar propiedades directamente en la lista de parámetros del
constructor, eliminando la repetición tediosa de declarar la propiedad
y luego asignarla.

Sintaxis tradicional (antes de PHP 8.0):

```php
<?php

class Punto
{
    public int $x;
    public int $y;
    public int $z;

    public function __construct(int $x, int $y, int $z = 0)
    {
        $this->x = $x;
        $this->y = $y;
        $this->z = $z;
    }
}
```

Con property promotion (PHP 8.0+):

```php
<?php

class Punto
{
    public function __construct(
        public int $x,
        public int $y,
        public int $z = 0
    ) {}
}

// Uso idéntico
$punto = new Punto(3, 4, 5);
echo $punto->x; // 3
```

La promoción de propiedades reduce drásticamente el código repetitivo
(boilerplate). Las reglas son:

- El modificador de visibilidad (public, protected, private) debe estar
  presente en el parámetro para que se promocione.
- Se puede combinar con tipos, valores por defecto y atributos.
- Las propiedades promocionadas no pueden declararse también en el
  cuerpo de la clase.
- Se pueden mezclar parámetros promocionados y no promocionados.

Ejemplo más complejo:

```php
<?php

class Empleado
{
    public function __construct(
        public readonly string $nombre,
        public readonly string $email,
        private float $salario,
        public readonly DateTime $fechaIngreso = new DateTime(),
        private ?string $departamento = null,
        protected string|int $codigoEmpleado = 'SIN-ASIGNAR'
    ) {}

    public function obtenerSalario(): float
    {
        return $this->salario;
    }

    public function asignarDepartamento(string $departamento): void
    {
        $this->departamento = $departamento;
    }

    public function obtenerInfo(): string
    {
        return "{$this->nombre} ({$this->email}) - " .
               "{$this->departamento ?? 'Sin depto.'} - " .
               "\${$this->salario}";
    }
}

$empleado = new Empleado(
    nombre: 'María López',
    email: 'maria@example.com',
    salario: 75000.00,
    codigoEmpleado: 'EMP-001'
);

echo $empleado->nombre;                                    // María López
echo $empleado->obtenerInfo();                              // María López (maria@example.com) - Sin depto. - $75000
```

La combinación de property promotion con `readonly` (PHP 8.1) permite
crear objetos inmutables con mínima sintaxis:

```php
<?php

class ConfiguracionBaseDatos
{
    public function __construct(
        public readonly string $host,
        public readonly int $puerto,
        public readonly string $nombreBD,
        public readonly string $usuario,
        public readonly string $password,
        public readonly string $charset = 'utf8mb4'
    ) {}
}

$config = new ConfiguracionBaseDatos(
    host: 'localhost',
    puerto: 3306,
    nombreBD: 'tienda',
    usuario: 'root',
    password: 'secreto'
);

// Las propiedades son de solo lectura después de la construcción
// $config->host = 'otro'; // Error: Cannot modify readonly property
```

Desde PHP 8.1 también puedes usar `readonly` con promoción y el
modificador `mixed` o tipos unión. La promoción de propiedades es
considerada una buena práctica en PHP moderno y debe preferirse
siempre que sea posible.

===============================================================================
3. PARÁMETROS OPCIONALES Y VALORES POR DEFECTO
===============================================================================

Al igual que las funciones regulares, los constructores pueden tener
parámetros opcionales con valores por defecto. Esto permite crear
objetos con diferentes niveles de configuración sin necesidad de
múltiples constructores.

```php
<?php

class Logger
{
    public function __construct(
        public readonly string $nombre = 'app',
        public readonly string $nivel = 'debug',
        public readonly string $formato = '[{fecha}] [{nivel}] {mensaje}',
        public readonly string $archivo = '/var/log/app.log',
        public readonly int $maxTamano = 10485760  // 10MB
    ) {}
}

// Creación con valores por defecto
$logger1 = new Logger();

// Creación con algunos valores personalizados
$logger2 = new Logger(nombre: 'api', nivel: 'error');

// Creación con todos los valores
$logger3 = new Logger(
    nombre: 'payments',
    nivel: 'warning',
    formato: '{mensaje} - {fecha}',
    archivo: '/var/log/pagos.log',
    maxTamano: 52428800
);
```

Los valores por defecto pueden ser:

- Literales (números, strings, booleanos, null)
- Arrays y constantes
- Expresiones constantes (operaciones matemáticas simples)
- `new ClassName()` desde PHP 8.0+ (en parámetros promocionados también)

```php
<?php

class Factura
{
    public function __construct(
        public readonly string $cliente,
        public readonly DateTime $fechaEmision = new DateTime(),
        public readonly array $items = [],
        public readonly float $tasaIVA = 0.16,
        public readonly string $moneda = 'MXN'
    ) {}
}

// El valor por defecto new DateTime() se evalúa en tiempo de construcción
$factura = new Factura(cliente: 'Cliente SA de CV');
echo $factura->fechaEmision->format('Y-m-d'); // Fecha actual
```

Es importante recordar que los valores por defecto se evalúan una vez
en tiempo de compilación, excepto las expresiones con `new` que se
evalúan en tiempo de ejecución (desde PHP 8.0). Los valores por defecto
no pueden depender de otros parámetros del constructor.

Los parámetros opcionales deben ir siempre después de los obligatorios
en la declaración:

```php
<?php

// Correcto: obligatorios primero, opcionales después
class Producto
{
    public function __construct(
        public string $nombre,      // Obligatorio
        public float $precio,       // Obligatorio
        public string $categoria = 'General',  // Opcional
        public ?string $descripcion = null,    // Opcional
        public bool $disponible = true         // Opcional
    ) {}
}

// Incorrecto: opcional antes que obligatorio (error de sintaxis)
// class ProductoMal
// {
//     public function __construct(
//         public string $categoria = 'General',  // Opcional
//         public string $nombre                   // Obligatorio - ERROR
//     ) {}
// }
```

La flexibilidad de los parámetros opcionales combinada con named
arguments (PHP 8.0+) hace que los constructores sean extremadamente
versátiles.

===============================================================================
4. NAMED ARGUMENTS EN CONSTRUCTORES
===============================================================================

PHP 8.0 introdujo los named arguments (argumentos con nombre), que
permiten pasar parámetros a un constructor (y a cualquier función)
especificando el nombre del parámetro en lugar de su posición.

```php
<?php

class Usuario
{
    public function __construct(
        public string $nombre,
        public string $email,
        public int $edad = 0,
        public string $ciudad = '',
        public string $pais = '',
        public bool $activo = true
    ) {}
}

// Orden posicional (tradicional)
$u1 = new Usuario('Ana', 'ana@mail.com', 30, 'Madrid', 'España', true);

// Named arguments (más legible y flexible)
$u2 = new Usuario(
    nombre: 'Carlos',
    email: 'carlos@mail.com',
    pais: 'México',
    ciudad: 'CDMX'
);

// Named arguments permiten saltar opcionales intermedios
$u3 = new Usuario(
    nombre: 'Luisa',
    email: 'luisa@mail.com',
    activo: false
    // edad y ciudad quedan con valores por defecto
);
```

Ventajas de los named arguments en constructores:

1. **Legibilidad mejorada**: Queda claro qué valor corresponde a cada
   parámetro, especialmente en constructores con muchos parámetros.

2. **Flexibilidad**: Puedes saltar parámetros opcionales intermedios
   sin necesidad de pasar null explícitamente.

3. **Mantenibilidad**: Si agregas un nuevo parámetro opcional en medio
   de la lista, no tienes que actualizar todas las llamadas existentes.

4. **Autocompletado**: Los IDEs modernos muestran los nombres de los
   parámetros, facilitando el uso de clases con muchos parámetros.

Ejemplo con mezcla de argumentos posicionales y nombrados:

```php
<?php

// Mezcla: posicionales primero, nombrados después
$u4 = new Usuario(
    'Roberto',           // posicional para $nombre
    'roberto@mail.com',  // posicional para $email
    ciudad: 'Bogotá',
    pais: 'Colombia'
);

// Esto NO es válido (named antes que posicional):
// $u5 = new Usuario(
//     nombre: 'Test',
//     'test@mail.com'  // Error: posicional después de named
// );
```

Los named arguments son especialmente útiles con la promoción de
propiedades, donde los nombres de los parámetros son también los
nombres de las propiedades, haciendo el código muy autodocumentado.

===============================================================================
5. HERENCIA DE CONSTRUCTORES
===============================================================================

Cuando una clase extiende a otra, el constructor de la clase padre NO
se hereda automáticamente. Si la clase hija define su propio constructor,
este reemplaza al constructor padre. Para invocar el constructor de la
clase padre, se utiliza `parent::__construct()`.

```php
<?php

class Persona
{
    public function __construct(
        public readonly string $nombre,
        public readonly string $apellido
    ) {}
}

class Empleado extends Persona
{
    public function __construct(
        string $nombre,
        string $apellido,
        public readonly string $puesto,
        public readonly float $salario
    ) {
        // Llamada obligatoria al constructor padre si tiene parámetros
        parent::__construct($nombre, $apellido);
    }
}

// Con named arguments
class Cliente extends Persona
{
    public function __construct(
        string $nombre,
        string $apellido,
        public readonly string $email,
        public readonly string $tipoCliente = 'regular'
    ) {
        parent::__construct(nombre: $nombre, apellido: $apellido);
    }
}

$empleado = new Empleado(
    nombre: 'Ana',
    apellido: 'García',
    puesto: 'Desarrolladora',
    salario: 85000
);

echo $empleado->nombre;  // Ana
echo $empleado->puesto;  // Desarrolladora
```

Reglas de herencia de constructores:

- Si la clase hija NO define constructor, hereda el constructor del
  padre automáticamente.
- Si la clase hija SÍ define constructor, debe llamar a
  `parent::__construct()` explícitamente si la lógica del padre
  es necesaria.
- La llamada a `parent::__construct()` puede ir en cualquier lugar
  del constructor hijo, no necesariamente al inicio.
- PHP permite pasar parámetros al constructor padre, ya sean
  posicionales o nombrados.

Ejemplo con herencia y parámetros adicionales:

```php
<?php

class ConexionBD
{
    protected PDO $conexion;

    public function __construct(
        protected readonly string $host,
        protected readonly string $nombreBD,
        protected readonly string $usuario,
        protected readonly string $password
    ) {
        $dsn = "mysql:host={$this->host};dbname={$this->nombreBD};charset=utf8mb4";
        $this->conexion = new PDO($dsn, $this->usuario, $this->password, [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        ]);
    }
}

class ConexionMySQL extends ConexionBD
{
    public function __construct(
        string $host,
        string $nombreBD,
        string $usuario,
        string $password,
        public readonly int $puerto = 3306,
        public readonly string $charset = 'utf8mb4'
    ) {
        parent::__construct($host, $nombreBD, $usuario, $password);
    }

    public function getConexion(): PDO
    {
        return $this->conexion;
    }
}

$db = new ConexionMySQL(
    host: 'localhost',
    nombreBD: 'tienda',
    usuario: 'root',
    password: '',
    puerto: 3307
);
```

Es buena práctica que las clases hijas siempre llamen al constructor
padre, a menos que tengan una razón muy específica para no hacerlo.
Esto garantiza que la inicialización de la clase base se ejecute
correctamente.

===============================================================================
6. CONSTRUCTORES PRIVADOS Y PATRONES DE DISEÑO
===============================================================================

Los constructores pueden declararse como `private` o `protected`, lo que
restringe la creación de objetos desde fuera de la clase. Esto es útil
para implementar patrones de diseño como Singleton, Factory Method o
Builder.

Singleton (instancia única):

```php
<?php

class GestorBaseDatos
{
    private static ?GestorBaseDatos $instancia = null;

    // Constructor privado: nadie puede crear instancias directamente
    private function __construct(
        private readonly string $host,
        private readonly string $bd,
        private readonly string $usuario,
        private readonly string $password
    ) {
        echo "Inicializando conexión...\n";
    }

    // Método de fábrica estático que controla la instancia única
    public static function obtenerInstancia(
        string $host = 'localhost',
        string $bd = 'tienda',
        string $usuario = 'root',
        string $password = ''
    ): self {
        if (self::$instancia === null) {
            self::$instancia = new self($host, $bd, $usuario, $password);
        }
        return self::$instancia;
    }

    public function consultar(string $sql): array
    {
        echo "Ejecutando: $sql en {$this->host}/{$this->bd}\n";
        return ['resultado' => 'datos simulados'];
    }

    // Prevenir clonación
    private function __clone(): void {}

    // Prevenir deserialización que cree nuevas instancias
    public function __wakeup(): void
    {
        throw new \Exception("No se puede deserializar un singleton");
    }
}

// Uso del Singleton
$db1 = GestorBaseDatos::obtenerInstancia();
$db2 = GestorBaseDatos::obtenerInstancia();

var_dump($db1 === $db2); // bool(true) - misma instancia

// $db3 = new GestorBaseDatos(); // Error: constructor privado

$db1->consultar("SELECT * FROM usuarios");
```

Factory Method con constructor privado:

```php
<?php

class FormatoFecha
{
    // Constructor privado: solo la clase puede instanciarse
    private function __construct(
        public readonly string $formato,
        public readonly string $descripcion
    ) {}

    // Métodos de fábrica estáticos
    public static function ISO(): self
    {
        return new self('Y-m-d', 'Formato ISO internacional');
    }

    public static function Latino(): self
    {
        return new self('d/m/Y', 'Formato latino (día/mes/año)');
    }

    public static function EEUU(): self
    {
        return new self('m/d/Y', 'Formato estadounidense');
    }

    public static function Personalizado(string $formato, string $desc = ''): self
    {
        return new self($formato, $desc ?: "Formato personalizado: $formato");
    }
}

$iso = FormatoFecha::ISO();
$latino = FormatoFecha::Latino();
$personalizado = FormatoFecha::Personalizado('d-m-Y H:i:s');

echo (new DateTime())->format($iso->formato);        // 2026-07-01
echo (new DateTime())->format($latino->formato);     // 01/07/2026
echo (new DateTime())->format($personalizado->formato); // 01-07-2026 15:30:00
```

Builder pattern con constructor privado:

```php
<?php

class ConsultaSQL
{
    private function __construct(
        public readonly string $tabla,
        public readonly array $campos = ['*'],
        public readonly array $condiciones = [],
        public readonly ?string $orden = null,
        public readonly ?int $limite = null,
        public readonly ?int $offset = null
    ) {}

    public static function builder(string $tabla): ConsultaSQLBuilder
    {
        return new ConsultaSQLBuilder($tabla);
    }

    public function generar(): string
    {
        $campos = implode(', ', $this->campos);
        $sql = "SELECT $campos FROM {$this->tabla}";

        if (!empty($this->condiciones)) {
            $sql .= " WHERE " . implode(' AND ', $this->condiciones);
        }

        if ($this->orden) {
            $sql .= " ORDER BY {$this->orden}";
        }

        if ($this->limite) {
            $sql .= " LIMIT {$this->limite}";
            if ($this->offset) {
                $sql .= " OFFSET {$this->offset}";
            }
        }

        return $sql;
    }
}

class ConsultaSQLBuilder
{
    private array $campos = ['*'];
    private array $condiciones = [];
    private ?string $orden = null;
    private ?int $limite = null;
    private ?int $offset = null;

    public function __construct(
        private readonly string $tabla
    ) {}

    public function select(string ...$campos): self
    {
        $this->campos = $campos;
        return $this;
    }

    public function donde(string $condicion): self
    {
        $this->condiciones[] = $condicion;
        return $this;
    }

    public function ordenarPor(string $orden): self
    {
        $this->orden = $orden;
        return $this;
    }

    public function limite(int $limite, ?int $offset = null): self
    {
        $this->limite = $limite;
        $this->offset = $offset;
        return $this;
    }

    public function construir(): ConsultaSQL
    {
        return new ConsultaSQL(
            tabla: $this->tabla,
            campos: $this->campos,
            condiciones: $this->condiciones,
            orden: $this->orden,
            limite: $this->limite,
            offset: $this->offset
        );
    }
}

// Uso del Builder
$consulta = ConsultaSQL::builder('usuarios')
    ->select('id', 'nombre', 'email')
    ->donde('activo = 1')
    ->donde('rol = "admin"')
    ->ordenarPor('nombre ASC')
    ->limite(10, 20)
    ->construir();

echo $consulta->generar();
// SELECT id, nombre, email FROM usuarios WHERE activo = 1 AND rol = "admin" ORDER BY nombre ASC LIMIT 10 OFFSET 20
```

Los constructores privados y los patrones asociados son herramientas
poderosas para controlar cómo y cuándo se crean los objetos,
promoviendo un diseño más robusto y mantenible.

===============================================================================
7. CONSTRUCTORES EN CLASES ANÓNIMAS
===============================================================================

Las clases anónimas (PHP 7.0+) también pueden tener constructores. Son
útiles para objetos desechables, mocks en pruebas, o implementaciones
rápidas sin necesidad de definir una clase con nombre.

```php
<?php

interface LoggerInterface
{
    public function log(string $mensaje): void;
}

// Clase anónima con constructor
$logger = new class('/tmp/app.log', 'DEBUG') implements LoggerInterface {
    private string $archivo;
    private string $nivelMinimo;

    public function __construct(string $archivo, string $nivelMinimo)
    {
        $this->archivo = $archivo;
        $this->nivelMinimo = $nivelMinimo;
    }

    public function log(string $mensaje): void
    {
        $linea = "[" . date('Y-m-d H:i:s') . "] [{$this->nivelMinimo}] $mensaje\n";
        file_put_contents($this->archivo, $linea, FILE_APPEND);
    }
};

$logger->log('Sistema iniciado correctamente');
```

Las clases anónimas con constructores también soportan property
promotion desde PHP 8.0:

```php
<?php

$validador = new class(
    minLongitud: 8,
    requiereMayusculas: true,
    requiereNumeros: true
) {
    public function __construct(
        private int $minLongitud = 8,
        private bool $requiereMayusculas = true,
        private bool $requiereNumeros = true
    ) {}

    public function validar(string $password): bool
    {
        if (strlen($password) < $this->minLongitud) return false;
        if ($this->requiereMayusculas && !preg_match('/[A-Z]/', $password)) return false;
        if ($this->requiereNumeros && !preg_match('/[0-9]/', $password)) return false;
        return true;
    }
};

var_dump($validador->validar('Password1')); // bool(true)
var_dump($validador->validar('hola'));       // bool(false)
```

Las clases anónimas no pueden ser referenciadas por type hints fuera
del ámbito donde se crean. Sin embargo, como implementan interfaces,
pueden ser pasadas a funciones que esperen esa interfaz.

RESUMEN DEL CAPÍTULO
===============================================================================

- `__construct()` es el método que PHP ejecuta al instanciar un objeto,
  ideal para inicializar propiedades y validar estado.
- La promoción de propiedades del constructor (PHP 8.0+) elimina el
  código repetitivo declarando y asignando en un solo paso.
- Los parámetros opcionales y valores por defecto permiten constructores
  flexibles sin necesidad de sobrecarga.
- Los named arguments (PHP 8.0+) mejoran la legibilidad y flexibilidad
  al crear objetos, permitiendo saltar parámetros opcionales.
- En herencia, la clase hija debe llamar a `parent::__construct()` si
  necesita ejecutar la lógica del constructor padre.
- Los constructores privados permiten implementar patrones como Singleton,
  Factory Method y Builder.
- Las clases anónimas pueden tener constructores, soportando property
  promotion desde PHP 8.0.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener más de un constructor en una clase?**
No, PHP no soporta sobrecarga de métodos tradicional. Solo puede haber
un `__construct()` por clase. Sin embargo, puedes simular múltiples
constructores usando parámetros opcionales, tipos unión, o métodos de
fábrica estáticos.

❓ **¿Qué ocurre si no defino ningún constructor?**
PHP proporciona un constructor vacío por defecto. Las propiedades se
inicializan con sus valores por defecto declarados, o quedan sin
inicializar (null / undefined) si no tienen valor por defecto.

❓ **¿Puedo llamar a $this en el constructor antes de asignar propiedades?**
Sí, $this está disponible desde el momento en que se ejecuta el
constructor. Sin embargo, es buena práctica asignar propiedades antes
de llamar a métodos que las usen, para evitar acceder a valores no
inicializados.

❓ **¿Los constructores pueden devolver un valor?**
Técnicamente no pueden devolver un valor explícito (el valor de retorno
es ignorado). El constructor existe para inicializar el objeto, y su
resultado es la propia instancia creada por `new`. Si intentas usar
`return` en un constructor, solo puedes usar `return;` sin valor.

❓ **¿Qué diferencia hay entre constructor property promotion y la forma tradicional?**
La promoción de propiedades es solo azúcar sintáctico. El resultado
compilado es idéntico. La ventaja es menos código repetitivo, menos
posibilidad de errores, y mejor legibilidad. No hay diferencia en
rendimiento ni comportamiento.

❓ **¿Puedo tener propiedades promocionadas con tipos complejos?**
Sí, las propiedades promocionadas soportan todos los tipos de PHP,
incluyendo tipos unión (int|string), tipos intersección (PHP 8.1+),
tipos nullable (?type), y tipos de clase (DateTime, PDO, etc.).

🏗️ **Sistema de Configuración Multi-entorno con Builder y Singleton** —
Diseña un sistema completo de configuración para una aplicación web que
integre todos los conceptos de constructores vistos en el capítulo:
property promotion, named arguments, constructores privados, herencia
de constructores, valores por defecto, y patrones de diseño.

El sistema debe incluir las siguientes clases:

1. `ConfiguracionBase` — Clase base abstracta con constructor que recibe:
   entorno (string: 'dev', 'test', 'prod'), debug (bool), version (string).
   Usa property promotion con readonly. Implementa un método `esProduccion():
   bool`. El constructor debe validar que el entorno sea uno de los valores
   permitidos.

2. `ConfiguracionBD` — Extiende ConfiguracionBase. Añade: driver, host,
   puerto, nombreBD, usuario, password, charset, poolMin, poolMax. Todos
   con property promotion. El constructor llama a parent::__construct().
   Incluye el método `obtenerDSN(): string` que genere el string de conexión
   según el driver (mysql, pgsql, sqlite).

3. `ConfiguracionCache` — Extiende ConfiguracionBase. Añade: driver (redis,
   memcached, file), host, puerto, prefix, ttl (int, default 3600),
   serializador (string, default 'json'). Implementa `obtenerServidores():
   array` para configuraciones de caché distribuida.

4. `ConfiguracionApp` — Clase que agrupa las configuraciones anteriores
   y añade: nombreApp, urlBase, idiomas (array), zonaHoraria (string),
   maximoArchivosSubida (int), tamañanoMaximoSubida (int). Constructor
   privado con property promotion. Implementa el patrón Singleton con
   un método `cargar(string $archivo): self` que lea un archivo de
   configuración en formato JSON o YAML.

5. `ConfiguracionBuilder` — Builder para construir ConfiguracionApp
   paso a paso. Métodos: conBase(ConfiguracionBase $base), conBD
   (ConfiguracionBD $bd), conCache(ConfiguracionCache $cache),
   conNombreApp(string $nombre), conUrlBase(string $url), agregarIdioma
   (string $idioma), conZonaHoraria(string $zona), conLimitesSubida
   (int $maxArchivos, int $maxTamano), construir(): ConfiguracionApp.

Requisitos adicionales:

- Todos los constructores deben usar property promotion con readonly
  cuando sea posible.
- Los constructores deben validar sus parámetros y lanzar excepciones
  con mensajes descriptivos.
- Debe haber al menos 3 configuraciones predefinidas (desarrollo,
  pruebas, producción) creadas con métodos de fábrica estáticos.
- El builder debe validar que todos los componentes requeridos están
  presentes antes de construir.
- El Singleton debe ser thread-safe a nivel de aplicación.
- Demuestra el uso con named arguments en todas las creaciones.

Este desafío integra todos los conceptos de constructores y patrones,
preparándote para diseñar sistemas de configuración profesionales como
los utilizados en frameworks como Laravel, Symfony o aplicaciones
empresariales.
