===============================================================================
1. DEFINICIÓN DE INTERFACES
===============================================================================

Una interfaz en PHP es un tipo de referencia que define un conjunto de
firmas de métodos (sin implementación) que cualquier clase que implemente
la interfaz debe cumplir. Es como un contrato: "si implementas esta
interfaz, garantizas que tienes estos métodos".

Sintaxis básica:

```php
<?php

declare(strict_types=1);

interface Imprimible
{
    public function imprimir(): string;
    public function obtenerNombre(): string;
}

// Una clase que implementa la interfaz DEBE tener todos sus métodos
class Documento implements Imprimible
{
    public function __construct(
        private string $titulo,
        private string $contenido
    ) {}

    public function imprimir(): string
    {
        return "=== {$this->titulo} ===\n{$this->contenido}\n=== Fin ===";
    }

    public function obtenerNombre(): string
    {
        return $this->titulo;
    }
}

class Imagen implements Imprimible
{
    public function __construct(
        private string $nombreArchivo,
        private int $ancho,
        private int $alto
    ) {}

    public function imprimir(): string
    {
        return "[Imagen: {$this->nombreArchivo} ({$this->ancho}x{$this->alto})]";
    }

    public function obtenerNombre(): string
    {
        return $this->nombreArchivo;
    }
}

// Función polimórfica: acepta cualquier Imprimible
function generarSalida(Imprimible $elemento): void
{
    echo $elemento->obtenerNombre() . ":\n" . $elemento->imprimir() . "\n\n";
}

$elementos = [
    new Documento('Informe', 'Resultados del Q1 2026...'),
    new Imagen('foto.jpg', 1920, 1080),
];

foreach ($elementos as $elemento) {
    generarSalida($elemento);
}
```

Reglas para definir interfaces:

- Usan la palabra clave `interface` (no `class`).
- Todos los métodos son implícitamente `public` (no se puede poner otro
  modificador).
- Los métodos no tienen implementación (solo firma, terminan en `;`).
- No pueden tener propiedades (pero sí constantes desde PHP 7.0+).
- No pueden tener constructor (con implementación).
- Las interfaces pueden extender otras interfaces (herencia múltiple
  entre interfaces).

===============================================================================
2. IMPLEMENTACIÓN DE INTERFACES
===============================================================================

Una clase implementa una interfaz usando la palabra clave `implements`.
Debe proporcionar implementación para TODOS los métodos declarados en
la interfaz, con firmas compatibles.

```php
<?php

interface AnimalInterface
{
    public function hacerSonido(): string;
    public function getNombre(): string;
    public function getEdad(): int;
}

class Perro implements AnimalInterface
{
    public function __construct(
        private string $nombre,
        private int $edad
    ) {}

    public function hacerSonido(): string
    {
        return "¡Guau!";
    }

    public function getNombre(): string
    {
        return $this->nombre;
    }

    public function getEdad(): int
    {
        return $this->edad;
    }

    // Métodos adicionales (no requeridos por la interfaz)
    public function moverCola(): string
    {
        return "{$this->nombre} mueve la cola";
    }
}

class Gato implements AnimalInterface
{
    public function __construct(
        private string $nombre,
        private int $edad
    ) {}

    public function hacerSonido(): string
    {
        return "¡Miau!";
    }

    public function getNombre(): string
    {
        return $this->nombre;
    }

    public function getEdad(): int
    {
        return $this->edad;
    }
}

// Uso polimórfico
function describirAnimal(AnimalInterface $animal): string
{
    return "{$animal->getNombre()} ({$animal->getEdad()} años): {$animal->hacerSonido()}";
}

$perro = new Perro('Max', 3);
$gato = new Gato('Luna', 2);

echo describirAnimal($perro); // Max (3 años): ¡Guau!
echo describirAnimal($gato);  // Luna (2 años): ¡Miau!
```

Si una clase implementa una interfaz pero no proporciona todos los
métodos, PHP lanza un error fatal:

```php
<?php

interface NecesitoMetodo
{
    public function metodoRequerido(): void;
    public function otroMetodo(): void;
}

// Error fatal: Class contiene 1 método abstracto y debe declararse
// abstracta o implementar los métodos restantes
// class ClaseIncompleta implements NecesitoMetodo
// {
//     public function metodoRequerido(): void {}
// }
```

Una clase puede implementar múltiples interfaces separándolas con comas:

```php
<?php

interface ExportableJSON
{
    public function toJSON(): string;
}

interface ExportableXML
{
    public function toXML(): string;
}

interface Almacenable
{
    public function guardar(string $ruta): bool;
    public function cargar(string $ruta): self;
}

// Implementación múltiple
class Configuracion implements ExportableJSON, ExportableXML, Almacenable
{
    public function __construct(
        private array $datos = []
    ) {}

    public function toJSON(): string
    {
        return json_encode($this->datos, JSON_PRETTY_PRINT);
    }

    public function toXML(): string
    {
        $xml = new SimpleXMLElement('<configuracion/>');
        $this->arrayAXML($this->datos, $xml);
        return $xml->asXML();
    }

    private function arrayAXML(array $datos, SimpleXMLElement $xml): void
    {
        foreach ($datos as $key => $value) {
            if (is_array($value)) {
                $hijo = $xml->addChild($key);
                $this->arrayAXML($value, $hijo);
            } else {
                $xml->addChild($key, htmlspecialchars((string)$value));
            }
        }
    }

    public function guardar(string $ruta): bool
    {
        return file_put_contents($ruta, $this->toJSON()) !== false;
    }

    public function cargar(string $ruta): self
    {
        $contenido = file_get_contents($ruta);
        $datos = json_decode($contenido, true);
        return new self($datos ?? []);
    }
}

$config = new Configuracion([
    'app' => ['nombre' => 'MiApp', 'version' => '1.0'],
    'db' => ['host' => 'localhost', 'puerto' => 3306],
]);

echo $config->toJSON();
echo $config->toXML();
$config->guardar('/tmp/config.json');
```

La implementación de múltiples interfaces permite construir clases con
capacidades combinadas, compensando elegantemente la falta de herencia
múltiple en PHP.

===============================================================================
3. HERENCIA ENTRE INTERFACES
===============================================================================

Las interfaces pueden extender otras interfaces usando `extends`. Esto
permite construir jerarquías de contratos, donde una interfaz hija
hereda las firmas de métodos de la interfaz padre.

```php
<?php

// Interfaz base
interface VehiculoInterface
{
    public function encender(): void;
    public function apagar(): void;
    public function obtenerVelocidadMaxima(): int;
}

// Interfaz que extiende (hereda) de otra
interface VehiculoTerrestre extends VehiculoInterface
{
    public function obtenerNumeroRuedas(): int;
    public function tocarBocina(): string;
}

interface VehiculoAcuatico extends VehiculoInterface
{
    public function obtenerEslora(): float;
    public function fondejar(): void;
}

// Una interfaz puede extender múltiples interfaces
interface VehiculoAnfibio extends VehiculoTerrestre, VehiculoAcuatico
{
    public function cambiarModo(string $modo): void;
}

// Implementación de interfaz extendida
class Coche implements VehiculoTerrestre
{
    private bool $encendido = false;

    public function __construct(
        private int $velocidadMax,
        private int $numRuedas,
        private string $marca
    ) {}

    public function encender(): void { $this->encendido = true; }
    public function apagar(): void { $this->encendido = false; }
    public function obtenerVelocidadMaxima(): int { return $this->velocidadMax; }
    public function obtenerNumeroRuedas(): int { return $this->numRuedas; }
    public function tocarBocina(): string { return "¡Piiii! ({$this->marca})"; }
}

// Implementación de interfaz con herencia múltiple de interfaces
class VehiculoAnfibioImplementado implements VehiculoAnfibio
{
    private string $modo = 'terrestre';

    public function __construct(
        private int $velocidadMaxTerrestre,
        private int $velocidadMaxAcuatica,
        private float $eslora
    ) {}

    public function encender(): void { echo "Vehículo encendido\n"; }
    public function apagar(): void { echo "Vehículo apagado\n"; }
    public function obtenerVelocidadMaxima(): int {
        return $this->modo === 'terrestre'
            ? $this->velocidadMaxTerrestre
            : $this->velocidadMaxAcuatica;
    }
    public function obtenerNumeroRuedas(): int { return 4; }
    public function tocarBocina(): string { return "¡Cuac cuac!"; }
    public function obtenerEslora(): float { return $this->eslora; }
    public function fondejar(): void { echo "Ancla lanzada\n"; }
    public function cambiarModo(string $modo): void {
        $this->modo = $modo;
        echo "Modo cambiado a: $modo\n";
    }
}

function probarVehiculoTerrestre(VehiculoTerrestre $v): void
{
    $v->encender();
    echo $v->tocarBocina() . "\n";
    echo "Ruedas: {$v->obtenerNumeroRuedas()}\n";
    $v->apagar();
}

$coche = new Coche(200, 4, 'Toyota');
probarVehiculoTerrestre($coche);

$anfibio = new VehiculoAnfibioImplementado(120, 40, 6.5);
probarVehiculoTerrestre($anfibio); // También funciona como terrestre
$anfibio->fondejar(); // Y también como acuático
```

La herencia entre interfaces permite modelar relaciones complejas entre
capacidades. Una interfaz que extiende otra hereda todas sus firmas de
métodos, y la clase que la implementa debe proveer todos los métodos
de la cadena de herencia.

===============================================================================
4. CONSTANTES Y MÉTODOS ESTÁTICOS EN INTERFACES
===============================================================================

Desde PHP 8.1, las interfaces pueden contener constantes y métodos
estáticos. Las constantes son implícitamente públicas y no pueden ser
anuladas por las clases que implementan la interfaz.

```php
<?php

interface Configurable
{
    // Constantes de interfaz (PHP 7.0+)
    public const VERSION = '1.0.0';
    public const NOMBRE_APP = 'PHP Master System';

    // Método estático (PHP 8.1+)
    public static function getConfigDefault(): array;

    // Método de instancia
    public function configurar(array $opciones): void;
}

class ConfiguracionProduccion implements Configurable
{
    // Las constantes de la interfaz están disponibles
    // pero NO pueden ser redefinidas en la clase

    public static function getConfigDefault(): array
    {
        return [
            'debug' => false,
            'log_level' => 'error',
            'cache' => true,
            'version' => Configurable::VERSION, // Acceso a constante
        ];
    }

    public function configurar(array $opciones): void
    {
        echo "Configurando con: " . json_encode($opciones) . "\n";
    }
}

class ConfiguracionDesarrollo implements Configurable
{
    public static function getConfigDefault(): array
    {
        return [
            'debug' => true,
            'log_level' => 'debug',
            'cache' => false,
        ];
    }

    public function configurar(array $opciones): void
    {
        echo "[DEV] Configurando: " . json_encode($opciones) . "\n";
    }
}

// Las constantes se acceden desde la interfaz o desde la clase
echo Configurable::VERSION;           // 1.0.0
echo Configurable::NOMBRE_APP;        // PHP Master System

// Los métodos estáticos se llaman desde la clase concreta
$defaults = ConfiguracionProduccion::getConfigDefault();
print_r($defaults);

// Uso polimórfico
function inicializarAplicacion(Configurable $config): void
{
    echo "Iniciando " . Configurable::NOMBRE_APP . "\n";
    $defaults = $config::getConfigDefault(); // Llamada polimórfica estática
    $config->configurar($defaults);
}

inicializarAplicacion(new ConfiguracionProduccion());
inicializarAplicacion(new ConfiguracionDesarrollo());
```

Desde PHP 8.1 también puedes declarar métodos estáticos en interfaces,
lo que permite llamadas polimórficas a métodos estáticos mediante
`static::` desde la interfaz. Sin embargo, no puedes tener propiedades
(estáticas o de instancia) en interfaces.

Las constantes en interfaces son útiles para definir valores compartidos
que deben ser consistentes en todas las implementaciones:

```php
<?php

interface CodigosHTTP
{
    public const OK = 200;
    public const CREADO = 201;
    public const SIN_CONTENIDO = 204;
    public const NO_ENCONTRADO = 404;
    public const ERROR_INTERNO = 500;
}

interface RespuestaAPI extends CodigosHTTP
{
    public function responder(mixed $datos, int $codigo): string;
}

class RespuestaJSON implements RespuestaAPI
{
    public function responder(mixed $datos, int $codigo): string
    {
        http_response_code($codigo);
        return json_encode([
            'codigo' => $codigo,
            'datos' => $datos,
            'mensaje' => $this->getMensaje($codigo),
        ]);
    }

    private function getMensaje(int $codigo): string
    {
        return match ($codigo) {
            self::OK => 'Éxito',
            self::CREADO => 'Creado exitosamente',
            self::NO_ENCONTRADO => 'Recurso no encontrado',
            self::ERROR_INTERNO => 'Error interno del servidor',
            default => 'Respuesta',
        };
    }
}

$respuesta = new RespuestaJSON();
echo $respuesta->responder(['usuario' => 'Ana'], RespuestaAPI::OK);
```

===============================================================================
5. INTERFACES VS CLASES ABSTRACTAS
===============================================================================

Elegir entre interfaces y clases abstractas es una decisión de diseño
importante. Aquí tienes una comparación detallada:

| Característica            | Interfaz                    | Clase Abstracta            |
|---------------------------|-----------------------------|----------------------------|
| Métodos con implementación| No                          | Sí                         |
| Propiedades               | No (solo constantes)        | Sí                         |
| Constructor               | No                          | Sí                         |
| Múltiples padres          | Sí (extends múltiple)       | No (extends único)         |
| Visibilidad de métodos    | Siempre public              | public/protected/private   |
| Métodos finales           | No (todos abstractos)       | Sí                         |
| Constantes                | Sí (public)                 | Sí (con visibilidad)       |
| Métodos estáticos         | Sí (PHP 8.1+)               | Sí                         |

```php
<?php

// GUÍA PRÁCTICA:
// Usa INTERFAZ cuando:
// - Define un contrato de capacidades (qué puede hacer).
// - Clases no relacionadas necesitan compartir comportamiento.
// - Necesitas herencia múltiple de contratos.
// - No hay estado o implementación que compartir.

// Usa CLASE ABSTRACTA cuando:
// - Hay implementación base que compartir entre hijas.
// - Hay estado común (propiedades) entre hijas.
// - Quieres un constructor base.
// - Las hijas comparten lógica parcial.

// Ejemplo: Comparación de enfoques

// Enfoque con interfaces (contrato puro)
interface Pagable
{
    public function calcularPago(): float;
    public function getMetodoPago(): string;
}

// Enfoque con clase abstracta (base con implementación)
abstract class EmpleadoAbstracto
{
    protected string $nombre;
    protected float $salarioBase;

    public function __construct(string $nombre, float $salarioBase)
    {
        $this->nombre = $nombre;
        $this->salarioBase = $salarioBase;
    }

    abstract public function calcularPago(): float;

    public function getNombre(): string
    {
        return $this->nombre;
    }
}

// Mejor: combinación de ambos
interface Contratable
{
    public function calcularPago(): float;
    public function getNombre(): string;
}

abstract class Trabajador implements Contratable
{
    public function __construct(
        protected string $nombre,
        protected float $tarifaBase
    ) {}

    public function getNombre(): string
    {
        return $this->nombre;
    }
}

class EmpleadoFijo extends Trabajador
{
    public function calcularPago(): float
    {
        return $this->tarifaBase; // Salario fijo mensual
    }
}

class EmpleadoPorHoras extends Trabajador
{
    public function __construct(
        string $nombre,
        float $tarifaBase,
        private int $horasTrabajadas
    ) {
        parent::__construct($nombre, $tarifaBase);
    }

    public function calcularPago(): float
    {
        return $this->tarifaBase * $this->horasTrabajadas;
    }
}

function procesarNomina(Contratable $trabajador): void
{
    echo "{$trabajador->getNombre()}: \${$trabajador->calcularPago()}\n";
}

procesarNomina(new EmpleadoFijo('Ana', 25000));
procesarNomina(new EmpleadoPorHoras('Luis', 250, 80));
```

La regla general: **programa hacia una interfaz, no hacia una
implementación** (Design Patterns, GoF). Esto significa que las
dependencias deben ser interfaces/clases abstractas, no clases
concretas, para maximizar la flexibilidad.

===============================================================================
6. INTERFACES NATIVAS DE PHP
===============================================================================

PHP incluye muchas interfaces incorporadas que forman parte del núcleo
del lenguaje. Conocerlas es fundamental para escribir código PHP moderno.

```php
<?php

// ArrayAccess: permite que un objeto se acceda como arreglo
class ConfiguracionArray implements ArrayAccess
{
    private array $datos = [];

    public function offsetExists(mixed $offset): bool
    {
        return isset($this->datos[$offset]);
    }

    public function offsetGet(mixed $offset): mixed
    {
        return $this->datos[$offset] ?? null;
    }

    public function offsetSet(mixed $offset, mixed $valor): void
    {
        if ($offset === null) {
            $this->datos[] = $valor;
        } else {
            $this->datos[$offset] = $valor;
        }
    }

    public function offsetUnset(mixed $offset): void
    {
        unset($this->datos[$offset]);
    }
}

$config = new ConfiguracionArray();
$config['db_host'] = 'localhost';  // offsetSet
$config['db_port'] = 3306;
echo $config['db_host'];           // offsetGet
var_dump(isset($config['db_host'])); // offsetExists
unset($config['db_port']);         // offsetUnset

// Iterator: permite que un objeto sea iterable con foreach
class RangoIterator implements Iterator
{
    private int $posicion = 0;

    public function __construct(
        private array $items
    ) {}

    public function current(): mixed
    {
        return $this->items[$this->posicion];
    }

    public function key(): int
    {
        return $this->posicion;
    }

    public function next(): void
    {
        $this->posicion++;
    }

    public function rewind(): void
    {
        $this->posicion = 0;
    }

    public function valid(): bool
    {
        return isset($this->items[$this->posicion]);
    }
}

$rango = new RangoIterator(['a', 'b', 'c', 'd']);
foreach ($rango as $key => $value) {
    echo "$key: $value\n";
}

// Serializable (PHP 7.4+): controla la serialización
class UsuarioSerializable implements Serializable
{
    public function __construct(
        private string $nombre,
        private string $email,
        private string $passwordHash
    ) {}

    public function serialize(): string
    {
        // Solo serializar nombre y email, no password
        return serialize(['nombre' => $this->nombre, 'email' => $this->email]);
    }

    public function unserialize(string $data): void
    {
        $datos = unserialize($data);
        $this->nombre = $datos['nombre'];
        $this->email = $datos['email'];
        $this->passwordHash = 'pendiente';
    }
}

// Stringable (PHP 8.0+): objetos que pueden convertirse a string
class MensajeError implements Stringable
{
    public function __construct(
        private string $codigo,
        private string $mensaje
    ) {}

    public function __toString(): string
    {
        return "[{$this->codigo}] {$this->mensaje}";
    }
}

$error = new MensajeError('ERR-001', 'Archivo no encontrado');
echo $error; // [ERR-001] Archivo no encontrado

// JsonSerializable: controla la conversión a JSON
class ProductoJSON implements JsonSerializable
{
    public function __construct(
        private string $nombre,
        private float $precio,
        private string $codigoInterno
    ) {}

    public function jsonSerialize(): mixed
    {
        return [
            'nombre' => $this->nombre,
            'precio' => $this->precio,
            // Excluir codigoInterno
        ];
    }
}

echo json_encode(new ProductoJSON('Laptop', 15000, 'INT-001'));
// {"nombre":"Laptop","precio":15000}
```

Otras interfaces nativas importantes:

- `Countable`: `count()` en objetos.
- `Throwable`: Base para todas las excepciones y errores.
- `DateTimeInterface`: Base para DateTime y DateTimeImmutable.
- `SessionHandlerInterface`: Para manejo personalizado de sesiones.
- `UnitEnum` y `BackedEnum` (PHP 8.1+): Para enumeraciones.

===============================================================================
7. BUENAS PRÁCTICAS CON INTERFACES
===============================================================================

1. **Nombres descriptivos**: Los nombres de interfaces suelen terminar
   en "-able" (imprimible, exportable, cachéable) o describir una
   capacidad (LoggerInterface, NotificadorInterface).

2. **Interfaces pequeñas (Interface Segregation Principle)**: Es mejor
   tener muchas interfaces pequeñas y específicas que una interfaz
   grande y genérica. Una clase no debería implementar métodos que no
   necesita.

```php
<?php

// MAL: Interfaz grande (fat interface)
interface TodoEnUno
{
    public function leer(): string;
    public function escribir(string $data): void;
    public function comprimir(): string;
    public function descomprimir(string $data): string;
    public function enviarPorEmail(string $destino): bool;
}

// BIEN: Interfaces segregadas (ISP)
interface Leible { public function leer(): string; }
interface Escribible { public function escribir(string $data): void; }
interface Compresible { public function comprimir(): string; public function descomprimir(string $data): string; }
interface Enviable { public function enviarPorEmail(string $destino): bool; }

// Cada clase implementa solo lo que necesita
class ArchivoTexto implements Leible, Escribible { /* ... */ }
class CompresorZip implements Compresible { /* ... */ }
class Notificador implements Enviable { /* ... */ }
```

3. **Documenta el contrato**: Usa PHPDoc para describir qué hace cada
   método y qué se espera.

```php
<?php

/**
 * Define la capacidad de caché para un objeto.
 */
interface Cacheable
{
    /**
     * Obtiene el tiempo de vida del caché en segundos.
     *
     * @return int Tiempo en segundos (0 = sin expiración).
     */
    public function getTTL(): int;

    /**
     * Obtiene la clave única para el caché.
     *
     * @return string Clave para almacenar/recuperar del caché.
     */
    public function getCacheKey(): string;

    /**
     * Obtiene los datos a cachear.
     *
     * @return mixed Datos serializables.
     */
    public function getCacheData(): mixed;
}
```

4. **No añadas métodos a interfaces existentes en producción**: Si
   modificas una interfaz agregando un método, todas las clases que la
   implementan se rompen. Si necesitas extender, crea una nueva interfaz
   que extienda la original.

5. **Usa type hints con interfaces**: Siempre que sea posible, declara
   parámetros y retornos con interfaces, no con clases concretas.

6. **Interfaces como tipos de retorno**: Las funciones pueden devolver
   interfaces, ocultando la implementación concreta:
   `function crearLogger(): LoggerInterface { ... }`

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Una interfaz define un contrato de métodos que las clases deben
  implementar.
- `implements` vincula una clase a una interfaz; la clase debe proveer
  todos los métodos.
- Una clase puede implementar múltiples interfaces (separadas por comas).
- Las interfaces pueden extender otras interfaces con `extends`.
- PHP 8.1+ permite constantes y métodos estáticos en interfaces.
- Las interfaces nativas (ArrayAccess, Iterator, JsonSerializable,
  Stringable) añaden capacidades especiales a los objetos.
- Principio de Segregación de Interfaces: interfaces pequeñas y
  específicas.
- Programa hacia interfaces, no hacia implementaciones concretas.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puede una interfaz tener propiedades?**
No, las interfaces no pueden tener propiedades (variables de instancia).
Sí pueden tener constantes desde PHP 7.0. Si necesitas propiedades
compartidas, usa una clase abstracta.

❓ **¿Qué ocurre si dos interfaces que implemento tienen métodos con el mismo nombre?**
Las firmas deben ser compatibles (mismos tipos de parámetros y retorno).
Si son incompatibles, PHP lanza error. Si son compatibles, una sola
implementación satisface ambas interfaces.

❓ **¿Puedo cambiar la visibilidad de un método de interfaz?**
No. Todos los métodos de interfaz son implícitamente públicos. La clase
que los implementa debe declararlos como públicos (no se permite
protected o private).

❓ **¿Las interfaces pueden extender clases?**
No. Las interfaces solo extienden otras interfaces. Las clases solo
extienden clases. Los dos mundos son separados. Una clase implementa
interfaces y extiende (una) clase.

❓ **¿Puedo tener una interfaz sin métodos?**
Sí, se llaman interfaces de marcado (marker interfaces). Sirven para
etiquetar clases con una capacidad (ej: `JsonSerializable` sin métodos
no tendría sentido, pero una interface de marcado indica "esta clase
es serializable de alguna forma").

❓ **¿Debo crear interfaces para todas mis clases?**
No. Crea interfaces cuando tengas múltiples implementaciones posibles
del mismo concepto o cuando quieras desacoplar el contrato de la
implementación. Para clases que solo tienen una implementación y no
se espera que cambien, las interfaces pueden ser sobrada abstracción.

DESAFÍO DEL ARQUITECTO

🏗️ **Sistema de Gateway de Pagos con Interfaces** — Diseña un sistema
completo de gateway de pagos que demuestre todos los conceptos de
interfaces: definición, implementación múltiple, herencia de interfaces,
constantes de interfaz, y uso de interfaces nativas de PHP.

Interfaces requeridas:

1. `ProcesadorPago` — Interfaz base con métodos:
   `procesar(float $monto, Moneda $moneda): ResultadoPago`,
   `reembolsar(string $idTransaccion): ResultadoPago`,
   `getNombre(): string`, `getComision(): float`.
   Constantes: `ESTADO_EXITOSO = 'exitoso'`, `ESTADO_FALLIDO = 'fallido'`,
   `ESTADO_PENDIENTE = 'pendiente'`.

2. `ProcesadorPagoRecurrente extends ProcesadorPago` — Añade:
   `suscripcionCrear(DatosSuscripcion $datos): ResultadoSuscripcion`,
   `suscripcionCancelar(string $idSuscripcion): bool`,
   `suscripcionEstado(string $idSuscripcion): string`.

3. `ProcesadorPagoConVerificacion extends ProcesadorPago` — Añade:
   `verificarTransaccion(string $idTransaccion): array`,
   `getWebhookURL(): string`, `setWebhookURL(string $url): void`.

4. `Verificable` — Interfaz independiente con:
   `verificarIdentidad(array $datosCliente): bool`,
   `getNivelSeguridad(): string`.

Clases que implementan las interfaces:

5. `PagoStripe implements ProcesadorPago, ProcesadorPagoRecurrente,
   Verificable` — Simula integración con Stripe. Comisión 2.9% + 0.30
   por transacción. Soporta suscripciones y verificación 3D Secure.

6. `PagoPayPal implements ProcesadorPago, ProcesadorPagoConVerificacion`
   — Simula integración con PayPal. Comisión 3.5%. Webhooks para
   verificación. No soporta suscripciones recurrentes.

7. `PagoTransferencia implements ProcesadorPago` — Simula transferencia
   bancaria. Comisión 0%. No soporta suscripciones ni verificación web.
   Constante `DIAS_PROCESAMIENTO = 3`.

Value Objects:

8. `Moneda` — Clase readonly con `codigo` (MXN, USD, EUR) y
   `simbolo`. Implementa `Stringable`.

9. `ResultadoPago implements JsonSerializable` — Propiedades: idTransaccion,
   estado, monto, moneda, fecha, mensaje. Implementa JsonSerializable.

10. `DatosSuscripcion` — DTO con: planId, clienteEmail, monto, moneda,
    intervalo (monthly, yearly), trialDías.

11. `GestorPagos` — Clase final que gestiona múltiples procesadores.
    Métodos: registrarProcesador(ProcesadorPago $p), procesarPago(string
    $procesador, float $monto, Moneda $moneda): ResultadoPago,
    getProcesadoresDisponibles(): array, getProcesador(string $nombre):
    ?ProcesadorPago. Usa type hints con interfaces, no con clases
    concretas.

Requisitos adicionales:

- Demostrar que una clase puede implementar múltiples interfaces.
- Probar que GestorPagos funciona con cualquier ProcesadorPago.
- Demostrar `instanceof` para verificar capacidades adicionales
  (si un procesador es recurrente o verificable).
- El script de demostración debe procesar pagos con los 3 procesadores,
  crear una suscripción (Stripe), verificar una transacción (PayPal), y
- mostrar resultados como JSON (JsonSerializable).
- Incluir manejo de errores con excepciones personalizadas.

Este desafío integra todos los conceptos de interfaces y produce un
sistema extensible donde añadir un nuevo procesador de pagos solo
requiere implementar las interfaces necesarias, sin modificar el
GestorPagos ni el código existente.
