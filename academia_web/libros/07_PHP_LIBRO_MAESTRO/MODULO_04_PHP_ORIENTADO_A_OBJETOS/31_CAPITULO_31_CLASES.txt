===============================================================================
1. ¿QUÉ ES UNA CLASE?
===============================================================================

Una clase es una plantilla o modelo que define la estructura y el
comportamiento de un conjunto de objetos. En términos prácticos, una clase
agrupa datos (propiedades) y funciones (métodos) que operan sobre esos
datos. Imagina que quieres modelar un coche: la clase Coche define que
todo coche tiene un color, una marca, una velocidad máxima (propiedades)
y que puede acelerar, frenar y girar (métodos). El coche físico que ves
en la calle sería un objeto (una instancia) de esa clase.

PHP adoptó la POO de forma nativa desde su versión 5 y la ha refinado
enormemente en PHP 8.x. A día de hoy, PHP ofrece un sistema de tipos
sólido, promoción de propiedades en constructores, tipos unión, y un
sistema de herencia y polimorfismo maduro.

La sintaxis básica para declarar una clase es:

```php
<?php

class NombreClase
{
    // Propiedades (atributos)
    // Métodos (funciones)
}
```

La palabra reservada `class` seguida del nombre de la clase (en mayúscula
inicial, por convención UpperCamelCase) y un bloque de código delimitado
por llaves. Dentro de este bloque declaramos todo lo que la clase contiene.

Veamos un ejemplo completo:

```php
<?php

class Coche
{
    // Propiedades
    public string $marca;
    public string $modelo;
    public string $color;
    public int $velocidadMaxima;
    public bool $encendido = false;

    // Métodos
    public function encender(): void
    {
        $this->encendido = true;
        echo "El coche está encendido.\n";
    }

    public function apagar(): void
    {
        $this->encendido = false;
        echo "El coche está apagado.\n";
    }

    public function obtenerInformacion(): string
    {
        return "{$this->marca} {$this->modelo}, Color: {$this->color}, " .
               "Vel. Máx: {$this->velocidadMaxima} km/h";
    }
}
```

Las clases pueden contener propiedades con tipos explícitos (PHP 7.4+),
lo que nos permite escribir código más robusto y autodocumentado.

===============================================================================
2. PROPIEDADES DE CLASE
===============================================================================

Las propiedades (también llamadas atributos o campos) son variables que
pertenecen a una clase. Se declaran con un modificador de visibilidad
(public, protected o private), seguido de un tipo (opcional desde PHP 7.4,
obligatorio recomendado) y un nombre.

Sintaxis de declaración de propiedades:

```php
<?php

class Usuario
{
    // Propiedades con tipos explícitos (PHP 7.4+)
    public string $nombre;
    public string $email;
    public int $edad;
    public bool $activo = true;

    // Propiedad con tipo nullable (PHP 7.1+)
    public ?string $telefono = null;

    // Propiedad con tipo unión (PHP 8.0+)
    public int|string $identificador;

    // Propiedad con valor por defecto
    public string $rol = 'usuario';

    // Propiedad sin tipo (no recomendado)
    public $legacy;
}
```

Las propiedades pueden tener valores por defecto. Estos valores deben ser
expresiones constantes, es decir, no pueden depender de variables en tiempo
de ejecución. Si necesitas un valor dinámico, debes asignarlo en el
constructor.

PHP 8.1 introdujo las propiedades `readonly`, que solo pueden ser asignadas
una vez (normalmente en el constructor) y no pueden ser modificadas después:

```php
<?php

class Configuracion
{
    public readonly string $version;
    public readonly array $ajustes;

    public function __construct(string $version, array $ajustes)
    {
        $this->version = $version;
        $this->ajustes = $ajustes;
    }
}
```

Las propiedades declaradas con `readonly` no pueden tener modificadores
de visibilidad protected o private si también son estáticas (no tiene
sentido porque readonly se asocia a la instancia).

Desde PHP 8.0 también podemos usar propiedades con tipos mixtos:

```php
<?php

class Respuesta
{
    public mixed $datos;        // Puede ser cualquier tipo
    public int $codigo;
    public string $mensaje;
}
```

Las propiedades no declaradas explícitamente son creadas dinámicamente
al asignarlas, pero esta práctica está obsoleta y debe evitarse. Siempre
declara todas las propiedades de la clase.

===============================================================================
3. MÉTODOS DE CLASE
===============================================================================

Los métodos son funciones que pertenecen a una clase. Definen el
comportamiento de los objetos creados a partir de la clase. Al igual
que las propiedades, se declaran con un modificador de visibilidad,
seguido de la palabra `function`, el nombre del método, paréntesis con
los parámetros (opcionales) y el bloque de código.

Sintaxis de métodos:

```php
<?php

class Calculadora
{
    // Método público con tipo de retorno
    public function sumar(int $a, int $b): int
    {
        return $a + $b;
    }

    // Método público sin tipo de retorno
    public function mostrarResultado(int $resultado): void
    {
        echo "El resultado es: $resultado\n";
    }

    // Método con parámetros opcionales
    public function multiplicar(int $a, int $b = 1): int
    {
        return $a * $b;
    }

    // Método privado (solo accesible desde la misma clase)
    private function validarNumero(int $numero): bool
    {
        return $numero > 0;
    }

    // Método protegido (accesible desde clases hijas)
    protected function formatearSalida(string $mensaje): string
    {
        return ">>> $mensaje <<<";
    }
}
```

Los métodos pueden devolver cualquier tipo de dato. Es buena práctica
declarar siempre el tipo de retorno, incluso si es `void`. Esto facilita
la lectura del código y permite que el IDE te ofrezca autocompletado.

PHP soporta la sobrecarga de métodos mediante parámetros opcionales y
tipos unión, pero no soporta la sobrecarga tradicional (múltiples métodos
con el mismo nombre pero diferentes firmas).

===============================================================================
4. LA PSEUDO-VARIABLE $this
===============================================================================

`$this` es una pseudo-variable disponible dentro de los métodos de una
clase. Hace referencia al objeto actual (la instancia sobre la cual se
invocó el método). Sin `$this`, no podríamos acceder a las propiedades
ni a otros métodos de la misma instancia desde dentro de la clase.

Ejemplo fundamental:

```php
<?php

class Persona
{
    public string $nombre;
    public string $apellido;

    public function nombreCompleto(): string
    {
        // $this es indispensable aquí
        return $this->nombre . ' ' . $this->apellido;
    }

    public function saludar(): string
    {
        return "Hola, soy " . $this->nombreCompleto();
    }

    public function actualizarNombre(string $nombre): void
    {
        // Sin $this, PHP crearía una variable local $nombre
        $this->nombre = $nombre;
    }
}
```

Es importante notar que `$this` no está disponible en métodos estáticos.
Si intentas usarlo en un método declarado como `static`, PHP lanzará un
error fatal.

```php
<?php

class Ejemplo
{
    public string $nombre = 'Ejemplo';

    public static function metodoEstatico(): void
    {
        // Error fatal: Using $this when not in object context
        // echo $this->nombre;
    }
}
```

La pseudo-variable `$this` también permite encadenar métodos (fluent
interface) cuando devolvemos `$this` desde un método:

```php
<?php

class ConstructorConsulta
{
    private array $condiciones = [];
    private string $tabla = '';

    public function desde(string $tabla): self
    {
        $this->tabla = $tabla;
        return $this;
    }

    public function donde(string $campo, mixed $valor): self
    {
        $this->condiciones[] = "$campo = '$valor'";
        return $this;
    }

    public function obtener(): string
    {
        $sql = "SELECT * FROM {$this->tabla}";
        if (!empty($this->condiciones)) {
            $sql .= " WHERE " . implode(' AND ', $this->condiciones);
        }
        return $sql;
    }
}

// Uso del encadenamiento
$consulta = new ConstructorConsulta();
$sql = $consulta
    ->desde('usuarios')
    ->donde('activo', '1')
    ->donde('rol', 'admin')
    ->obtener();

echo $sql;
// SELECT * FROM usuarios WHERE activo = '1' AND rol = 'admin'
```

Este patrón es muy utilizado en bibliotecas de bases de datos, constructores
de consultas y frameworks modernos.

===============================================================================
5. MÉTODOS Y PROPIEDADES ESTÁTICAS
===============================================================================

Las propiedades y métodos estáticos pertenecen a la clase en sí misma, no
a las instancias. Se declaran con la palabra reservada `static` y se
acceden mediante el operador `::` (doble dos puntos) usando el nombre de
la clase o `self::` desde dentro de la clase.

```php
<?php

class Contador
{
    public static int $totalInstancias = 0;

    private string $nombre;

    public function __construct(string $nombre)
    {
        $this->nombre = $nombre;
        self::$totalInstancias++;
    }

    public static function obtenerTotal(): int
    {
        return self::$totalInstancias;
    }

    public static function reiniciar(): void
    {
        self::$totalInstancias = 0;
    }
}

$c1 = new Contador('A');
$c2 = new Contador('B');
$c3 = new Contador('C');

echo Contador::obtenerTotal(); // 3
echo Contador::$totalInstancias; // 3 (acceso directo a propiedad estática pública)
Contador::reiniciar();
echo Contador::obtenerTotal(); // 0
```

Los métodos estáticos son útiles para:

- Métodos de fábrica (factory methods) que crean instancias.
- Utilidades que no requieren estado de instancia.
- Implementación del patrón Singleton.
- Contadores globales o registro de instancias.
- Validaciones y formateo de datos.

Ejemplo de método de fábrica:

```php
<?php

class Usuario
{
    public function __construct(
        public readonly string $nombre,
        public readonly string $email
    ) {}

    public static function crearDesdeArray(array $datos): self
    {
        return new self(
            nombre: $datos['nombre'],
            email: $datos['email']
        );
    }

    public static function crearAdmin(string $nombre, string $email): self
    {
        return new self(nombre: $nombre, email: $email);
    }
}

// Creación mediante método de fábrica
$admin = Usuario::crearAdmin('Admin', 'admin@example.com');

// Creación desde arreglo asociativo
$datos = ['nombre' => 'Juan', 'email' => 'juan@example.com'];
$usuario = Usuario::crearDesdeArray($datos);
```

Desde PHP 8.0, podemos tener métodos estáticos en interfaces y usar
`new static()` para crear instancias de la clase concreta (late static
binding) en lugar de `new self()`.

===============================================================================
6. CONSTANTES DE CLASE
===============================================================================

Las clases pueden definir constantes con la palabra clave `const`. Las
constantes pertenecen a la clase, no a las instancias, y se accede a
ellas con `::` (igual que a los miembros estáticos).

```php
<?php

class ConfiguracionApp
{
    public const VERSION = '2.5.0';
    public const NOMBRE_APP = 'Gestor Empresarial';
    public const AUTOR = 'Lucas EGA Academy';
    public const ANIO_LANZAMIENTO = 2024;

    private const CLAVE_SECRETA = 's3cr3t0'; // Privada desde PHP 7.1+

    public static function obtenerInfoApp(): string
    {
        return self::NOMBRE_APP . ' v' . self::VERSION .
               ' © ' . self::ANIO_LANZAMIENTO . ' ' . self::AUTOR;
    }
}

// Acceso a constantes
echo ConfiguracionApp::VERSION;         // 2.5.0
echo ConfiguracionApp::obtenerInfoApp();
```

Desde PHP 7.1, las constantes de clase pueden tener modificadores de
visibilidad (public, protected, private). Antes de esa versión, todas
las constantes eran públicas.

Las constantes también pueden ser tipadas desde PHP 8.3:

```php
<?php

class ConstantesTipadas
{
    public const string NOMBRE = 'PHP Master Book';
    public const int MAX_INTENTOS = 5;
    public const array PERMISOS = ['lectura', 'escritura'];
}
```

7. AUTO-CARGA Y DECLARACIONES DE TIPO STRICT
===============================================================================

Al trabajar con clases, es fundamental declarar `strict_types` al inicio
de cada archivo PHP. Esto asegura que PHP respete los tipos declarados
en las firmas de los métodos y propiedades, lanzando errores fatales si
se pasan valores del tipo incorrecto.

```php
<?php

declare(strict_types=1);

class Validacion
{
    public function validarEdad(int $edad): bool
    {
        return $edad >= 18 && $edad <= 120;
    }
}

// Sin strict_types, PHP convertiría "25" a int automáticamente
// Con strict_types, pasar un string lanzará TypeError
$val = new Validacion();
$val->validarEdad("25"); // TypeError: Expected int, string given
```

La declaración `declare(strict_types=1)` debe ser la primera línea del
archivo (después de la etiqueta `<?php`). Afecta solo al archivo donde
se declara. Si llamas a una función definida en otro archivo, se aplica
la regla del archivo donde se hizo la llamada.

El auto-loading (carga automática de clases) es un mecanismo que permite
a PHP cargar automáticamente los archivos de las clases cuando son
necesitadas, sin necesidad de usar `require` o `include` manualmente.
La forma moderna es mediante Composer y el estándar PSR-4, que veremos
en detalle en el capítulo de Namespaces.

Sin embargo, es posible implementar un autoloader básico con la función
`spl_autoload_register`:

```php
<?php

spl_autoload_register(function (string $clase): void {
    $archivo = __DIR__ . '/src/' . str_replace('\\', '/', $clase) . '.php';
    if (file_exists($archivo)) {
        require_once $archivo;
    }
});
```

Este autoloader convierte el nombre de la clase (incluyendo su namespace)
en una ruta de archivo. Por ejemplo, `Modelo\Usuario` se convierte en
`src/Modelo/Usuario.php`. Esta es la base del estándar PSR-4.

8. BUENAS PRÁCTICAS AL DEFINIR CLASES
===============================================================================

Al definir clases en PHP, sigue estas recomendaciones para mantener un
código limpio, mantenible y profesional:

1. **Una clase por archivo**: Cada archivo debe contener una sola clase.
   El nombre del archivo debe coincidir con el nombre de la clase.

2. **UpperCamelCase**: Los nombres de clase deben usar UpperCamelCase
   (también conocido como PascalCase). Ejemplos: `Usuario`, `ConexionBD`,
   `ControladorArticulos`.

3. **Declaración explícita de tipos**: Siempre declara tipos en
   propiedades, parámetros y valores de retorno. Aprovecha el tipado
   fuerte de PHP 8.x.

4. **Visibilidad explícita**: Siempre declara public, protected o private
   en cada propiedad y método. Nunca omitas el modificador.

5. **Principio de responsabilidad única**: Cada clase debe tener una
   única responsabilidad bien definida. Si una clase hace demasiadas
   cosas, debes dividirla.

6. **Evita propiedades dinámicas**: No asignes propiedades que no hayan
   sido declaradas explícitamente. En PHP 8.2 esto genera un deprecation
   warning y en PHP 9.0 será un error.

7. **Documentación con PHPDoc**: Aunque los tipos están declarados, usa
   bloques PHPDoc para documentar el propósito de cada método y sus
   parámetros.

```php
<?php

/**
 * Servicio de autenticación de usuarios.
 *
 * Gestiona el proceso de login, logout y verificación
 * de credenciales contra la base de datos.
 */
class ServicioAutenticacion
{
    /**
     * Verifica las credenciales del usuario.
     *
     * @param string $email    Correo electrónico del usuario.
     * @param string $password Contraseña en texto plano.
     *
     * @return bool True si las credenciales son válidas.
     */
    public function verificarCredenciales(string $email, string $password): bool
    {
        // Lógica de verificación
        return true;
    }
}
```

8. **Nombres descriptivos**: Los nombres de clases, métodos y propiedades
   deben ser autodescriptivos. `$usuario->obtenerNombreCompleto()` es
   mejor que `$u->getNom()`.

RESUMEN DEL CAPÍTULO
===============================================================================

- Una clase es una plantilla que define la estructura (propiedades) y el
  comportamiento (métodos) de los objetos.
- Las propiedades se declaran con visibilidad (public, protected, private),
  tipo y nombre. PHP 7.4+ soporta propiedades tipadas.
- Los métodos son funciones dentro de una clase, con visibilidad y tipo
  de retorno.
- $this es la pseudo-variable que referencia la instancia actual dentro
  de un método no estático.
- Las propiedades y métodos estáticos pertenecen a la clase, no a las
  instancias, y se accede con :: (self:: o NombreClase::).
- Las constantes de clase se definen con `const` y pueden tener visibilidad
  desde PHP 7.1+.
- declare(strict_types=1) asegura el respeto estricto de los tipos.
- El autoloading mediante spl_autoload_register permite cargar clases
  automáticamente.
- Sigue las convenciones de nomenclatura (UpperCamelCase, una clase por
  archivo) para un código profesional.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Es obligatorio declarar el tipo de las propiedades?**
No es obligatorio, pero es altamente recomendado desde PHP 7.4. Las
propiedades sin tipo permiten cualquier valor, lo que puede provocar
errores difíciles de detectar. El tipado fuerte mejora la robustez
del código y facilita el mantenimiento.

❓ **¿Cuál es la diferencia entre self:: y $this->?**
self:: se usa para acceder a miembros estáticos (pertenecen a la clase)
y no requiere una instancia. $this-> accede a miembros de instancia y
solo está disponible dentro de métodos no estáticos. self:: no usa el
operador $ porque las constantes y miembros estáticos no tienen el
símbolo $.

❓ **¿Puedo tener una clase sin propiedades?**
Sí, una clase puede tener solo métodos sin propiedades. Esto es común
en clases de utilidades (utility classes) que agrupan métodos
relacionados, aunque en esos casos suele ser mejor usar funciones
independientes o métodos estáticos.

❓ **¿Qué ocurre si no pongo modificador de visibilidad?**
Antes de PHP 7.1, los métodos sin modificador eran implícitamente
públicos. Las propiedades debían declararse con var (obsoleto) o un
modificador específico. Desde PHP 7.1, la omisión del modificador
provoca un error fatal. Siempre debes declarar public, protected o
private.

❓ **¿Puedo tener métodos con el mismo nombre pero diferentes parámetros?**
PHP no soporta sobrecarga de métodos como Java o C++. Sin embargo,
puedes lograr comportamiento similar usando parámetros opcionales,
tipos unión, o el operador ... (variadic). También puedes usar el
método mágico __call para delegar llamadas dinámicamente.

❓ **¿Qué es una clase anónima?**
Desde PHP 7.0, puedes crear clases sin nombre, útiles para objetos
desechables o mocks en pruebas. Se declaran con `new class { ... }`.
Aunque tienen limitaciones, son prácticas en ciertos escenarios de
pruebas unitarias y mapeo rápido de datos.
