===============================================================================
1. CONCEPTO DE HERENCIA
===============================================================================

La herencia es un mecanismo de la POO que permite crear una nueva clase
basada en una clase existente. La clase hija (subclase) hereda todos los
métodos y propiedades públicas y protegidas de la clase padre (superclase),
y puede agregar sus propios miembros o sobrescribir los existentes.

La relación fundamental es "ES UN" (IS-A): un Perro ES UN Animal, un
Coche ES UN Vehículo, un Empleado ES UNA Persona.

Sintaxis básica:

```php
<?php

declare(strict_types=1);

// Clase padre (superclase)
class Animal
{
    public function __construct(
        public readonly string $nombre,
        public readonly int $edad
    ) {}

    public function hacerSonido(): string
    {
        return "El animal hace un sonido";
    }

    public function moverse(): string
    {
        return "{$this->nombre} se está moviendo";
    }

    public function dormir(): string
    {
        return "{$this->nombre} está durmiendo";
    }
}

// Clase hija (subclase) que hereda de Animal
class Perro extends Animal
{
    public function hacerSonido(): string
    {
        return "{$this->nombre} dice: ¡Guau guau!";
    }

    public function moverCola(): string
    {
        return "{$this->nombre} está moviendo la cola";
    }
}

$perro = new Perro('Max', 3);
echo $perro->hacerSonido();     // Max dice: ¡Guau guau!
echo $perro->moverse();         // Max se está moviendo (heredado)
echo $perro->dormir();          // Max está durmiendo (heredado)
echo $perro->moverCola();       // Max está moviendo la cola (propio)
```

La herencia en PHP es simple (solo herencia única): una clase puede
extender solo una clase padre. No hay herencia múltiple como en C++,
pero se puede lograr comportamiento similar con Traits (capítulo 38)
e Interfaces (capítulo 37).

La herencia crea una cadena: si B extiende A, y C extiende B, entonces
C tiene todo lo de A y B. Sin embargo, las cadenas muy profundas son
difíciles de mantener (se recomienda máximo 2-3 niveles de herencia).

===============================================================================
2. HERENCIA DE PROPIEDADES Y MÉTODOS
===============================================================================

La clase hija hereda todas las propiedades y métodos públicos y protegidos
del padre. Las propiedades y métodos privados NO se heredan (pero existen
en el objeto, solo no son accesibles desde la clase hija).

```php
<?php

class Vehiculo
{
    public string $marca;
    protected string $modelo;
    private string $numeroSerie;  // No se hereda

    public function __construct(string $marca, string $modelo, string $numeroSerie)
    {
        $this->marca = $marca;
        $this->modelo = $modelo;
        $this->numeroSerie = $numeroSerie;
    }

    public function obtenerInfo(): string
    {
        return "{$this->marca} {$this->modelo}";
    }

    protected function obtenerModelo(): string
    {
        return $this->modelo;
    }

    private function obtenerSerie(): string
    {
        return $this->numeroSerie;
    }
}

class Coche extends Vehiculo
{
    public int $numPuertas;

    public function __construct(string $marca, string $modelo, string $numeroSerie, int $numPuertas)
    {
        parent::__construct($marca, $modelo, $numeroSerie);
        $this->numPuertas = $numPuertas;
    }

    public function obtenerInfoCompleta(): string
    {
        // Puede acceder a $this->marca (public) y $this->modelo (protected)
        $info = $this->obtenerInfo(); // Método público heredado
        $modelo = $this->obtenerModelo(); // Método protected heredado

        // NO puede acceder a $this->numeroSerie (private)
        // NO puede llamar a $this->obtenerSerie() (private)

        return "$info - {$this->numPuertas} puertas";
    }
}

$coche = new Coche('Toyota', 'Corolla', 'SN-12345', 4);
echo $coche->marca;                 // Toyota (public, heredada)
echo $coche->obtenerInfo();         // Toyota Corolla (public, heredado)
echo $coche->obtenerInfoCompleta(); // Toyota Corolla - 4 puertas

// echo $coche->modelo;      // Error: protected
// echo $coche->numeroSerie; // Error: private
// echo $coche->obtenerSerie(); // Error: private
```

Puntos clave sobre herencia de miembros:

- `public`: se hereda y es accesible desde cualquier lugar.
- `protected`: se hereda pero solo es accesible desde la clase hija y
  el padre, no desde fuera.
- `private`: NO se hereda en absoluto. La clase hija no sabe que existe.
- Los métodos privados del padre pueden tener métodos con el mismo nombre
  en el hijo, y serán métodos diferentes (no es sobrescritura).

La herencia también aplica para constantes y propiedades estáticas:

```php
<?php

class ConfiguracionGlobal
{
    public const VERSION = '1.0.0';
    protected static int $contador = 0;

    public static function obtenerContador(): int
    {
        return self::$contador;
    }
}

class ConfiguracionAvanzada extends ConfiguracionGlobal
{
    public const VERSION = '2.0.0'; // Redefinir constante

    public static function incrementarContador(): void
    {
        static::$contador++; // Late static binding
    }
}

echo ConfiguracionAvanzada::VERSION; // 2.0.0
echo ConfiguracionGlobal::VERSION;   // 1.0.0
ConfiguracionAvanzada::incrementarContador();
echo ConfiguracionAvanzada::obtenerContador(); // 1
```

===============================================================================
3. SOBRESCRITURA DE MÉTODOS (OVERRIDE)
===============================================================================

La sobrescritura permite que una clase hija redefina un método heredado
del padre, proporcionando una implementación específica. Para que un
método pueda ser sobrescrito, debe ser declarado como `public` o
`protected` en el padre (no `private`).

```php
<?php

class Forma
{
    public function __construct(
        public readonly string $color = 'negro'
    ) {}

    public function area(): float
    {
        return 0.0;
    }

    public function descripcion(): string
    {
        return "Forma de color {$this->color}";
    }
}

class Circulo extends Forma
{
    public function __construct(
        string $color,
        public readonly float $radio
    ) {
        parent::__construct($color);
    }

    // Sobrescritura del método area()
    public function area(): float
    {
        return pi() * $this->radio ** 2;
    }

    // Sobrescritura del método descripcion()
    public function descripcion(): string
    {
        return "Círculo de radio {$this->radio} y color {$this->color}";
    }
}

class Rectangulo extends Forma
{
    public function __construct(
        string $color,
        public readonly float $base,
        public readonly float $altura
    ) {
        parent::__construct($color);
    }

    public function area(): float
    {
        return $this->base * $this->altura;
    }
}

$formas = [
    new Circulo('rojo', 5.0),
    new Rectangulo('azul', 4.0, 6.0),
    new Forma('verde'),
];

foreach ($formas as $forma) {
    echo $forma->descripcion() . " -> Área: " . $forma->area() . "\n";
}
// Círculo de radio 5 y color rojo -> Área: 78.539816339745
// Rectángulo de base 4 y altura 6 y color azul -> Área: 24
// Forma de color verde -> Área: 0
```

Reglas para la sobrescritura:

- La firma del método debe ser compatible (mismos tipos de parámetros
  y retorno, o covariante/contravariante en PHP 7.4+).
- El tipo de retorno puede ser más específico (covarianza) desde PHP 7.4.
- Los parámetros pueden ser menos específicos (contravarianza) desde
  PHP 7.4.
- La visibilidad no puede ser más restrictiva (puede ser igual o más
  permisiva: protected -> public es válido, public -> protected no).

Covarianza y contravarianza (PHP 7.4+):

```php
<?php

class Animal {}
class Perro extends Animal {}
class Gato extends Animal {}

class Refugio
{
    public function adoptar(): Animal
    {
        return new Animal();
    }

    public function recibir(Perro $perro): void
    {
        echo "Recibiendo un perro\n";
    }
}

// Covarianza: el tipo de retorno puede ser más específico
class RefugioPerros extends Refugio
{
    // Retorna un tipo más específico (Perro en lugar de Animal)
    public function adoptar(): Perro
    {
        return new Perro();
    }

    // Contravarianza: el parámetro puede ser más general
    public function recibir(Animal $animal): void
    {
        echo "Recibiendo un animal\n";
    }
}
```

La sobrescritura con `match` y tipos unión también es compatible:

```php
<?php

class Procesador
{
    public function procesar(int|string $dato): string
    {
        return match (true) {
            is_int($dato) => "Entero: $dato",
            is_string($dato) => "String: $dato",
        };
    }
}

class ProcesadorEspecifico extends Procesador
{
    public function procesar(int|string $dato): string
    {
        $resultado = parent::procesar($dato);
        return "[PROCESADO] $resultado";
    }
}
```

===============================================================================
4. EL OPERADOR parent::
===============================================================================

El operador `parent::` permite acceder a métodos y propiedades de la
clase padre desde la clase hija. Es especialmente útil cuando:

1. Se sobrescribe un método pero se quiere extender (no reemplazar por
   completo) la funcionalidad del padre.
2. Se necesita llamar al constructor del padre.
3. Se accede a miembros estáticos o constantes del padre.

```php
<?php

class PedidoBase
{
    protected float $subtotal = 0.0;
    protected float $impuesto = 0.0;
    protected const TASA_IVA = 0.16;

    public function __construct(
        protected readonly array $productos
    ) {
        $this->calcularSubtotal();
        $this->calcularImpuesto();
    }

    protected function calcularSubtotal(): void
    {
        $this->subtotal = array_sum(array_column($this->productos, 'precio'));
    }

    protected function calcularImpuesto(): void
    {
        $this->impuesto = $this->subtotal * self::TASA_IVA;
    }

    public function obtenerTotal(): float
    {
        return $this->subtotal + $this->impuesto;
    }

    public function obtenerResumen(): string
    {
        return "Subtotal: \${$this->subtotal}, IVA: \${$this->impuesto}, Total: \$" . $this->obtenerTotal();
    }
}

class PedidoInternacional extends PedidoBase
{
    protected const TASA_IVA = 0.0; // Sin IVA para exportación
    private float $impuestoAduana = 0.0;

    public function __construct(
        array $productos,
        private readonly string $paisDestino,
        private readonly float $tasaAduana = 0.05
    ) {
        parent::__construct($productos); // Llama al constructor del padre
    }

    // Extiende (no reemplaza) el método del padre
    protected function calcularImpuesto(): void
    {
        // Llama al método original del padre
        parent::calcularImpuesto();

        // Añade lógica adicional
        $this->impuestoAduana = $this->subtotal * $this->tasaAduana;
    }

    // Sobrescribe completamente
    public function obtenerTotal(): float
    {
        // Usa parent:: para obtener la lógica base y la extiende
        return parent::obtenerTotal() + $this->impuestoAduana;
    }

    // Usa parent:: para extender resumen
    public function obtenerResumen(): string
    {
        $resumenBase = parent::obtenerResumen();
        return "$resumenBase, Aduana: \${$this->impuestoAduana}, Destino: {$this->paisDestino}";
    }
}

$productos = [
    ['nombre' => 'Laptop', 'precio' => 15000.00],
    ['nombre' => 'Mouse', 'precio' => 500.00],
];

$pedidoNacional = new PedidoBase($productos);
$pedidoExportacion = new PedidoInternacional($productos, 'Estados Unidos', 0.08);

echo $pedidoNacional->obtenerResumen();
echo $pedidoExportacion->obtenerResumen();
```

Reglas de `parent::`:

- Solo se puede usar dentro de una clase hija.
- Debe referirse a la clase padre directa (no abuelos).
- Se puede usar para métodos estáticos: `parent::metodoEstatico()`.
- Se puede usar para constantes: `parent::CONSTANTE`.
- Se puede usar con el constructor: `parent::__construct()`.
- Si múltiples niveles de herencia, parent:: se refiere al padre directo.

`parent::` es fundamental para el patrón Template Method, donde el padre
define la estructura de un algoritmo y las hijas implementan pasos
específicos.

5. LA PALABRA CLAVE final
===============================================================================

La palabra clave `final` tiene dos usos en herencia:

1. **Clases final**: Una clase declarada como `final` no puede ser
   extendida (no puede tener hijas).
2. **Métodos final**: Un método declarado como `final` no puede ser
   sobrescrito en clases hijas.

```php
<?php

// Clase final: no puede tener hijas
final class ConstantesMatematicas
{
    public const PI = 3.1415926536;
    public const E = 2.7182818285;
}

// Error: Cannot extend final class
// class MisConstantes extends ConstantesMatematicas {}

// Método final: no puede ser sobrescrito
class ConfiguracionBase
{
    public string $nombreApp = 'App';

    // Método final: las hijas no pueden sobrescribirlo
    final public function obtenerVersion(): string
    {
        return '1.0.0';
    }

    // Método abierto a sobrescritura
    public function obtenerInfo(): string
    {
        return "{$this->nombreApp} v" . $this->obtenerVersion();
    }
}

class ConfiguracionAvanzada extends ConfiguracionBase
{
    // Esto funciona: el método no es final
    public function obtenerInfo(): string
    {
        return "[AVANZADO] " . parent::obtenerInfo();
    }

    // Error: Cannot override final method
    // public function obtenerVersion(): string { return '2.0.0'; }
}
```

Usos típicos de `final`:

- **Clases finales**: Para asegurar que una clase no sea extendida por
  razones de seguridad, inmutabilidad o diseño (Value Objects, DTOs,
  utility classes).
- **Métodos finales**: Para garantizar que un comportamiento crítico
  (validación, logging, transacciones) no sea alterado por subclases.
- **Template Method**: El método plantilla es final, los pasos que
  implementa son abstractos o sobrescribibles.

Ejemplo del patrón Template Method con final:

```php
<?php

abstract class ProcesadorArchivos
{
    // Template method: define el esqueleto del algoritmo
    final public function procesar(string $archivo): array
    {
        $this->validarArchivo($archivo);
        $contenido = $this->leerArchivo($archivo);
        $datos = $this->parsearContenido($contenido);
        $this->validarDatos($datos);
        $resultado = $this->transformarDatos($datos);
        $this->registrarProcesamiento($archivo);
        return $resultado;
    }

    // Pasos con implementación por defecto
    protected function validarArchivo(string $archivo): void
    {
        if (!file_exists($archivo)) {
            throw new RuntimeException("Archivo no encontrado: $archivo");
        }
    }

    // Pasos abstractos (deben ser implementados por las hijas)
    abstract protected function leerArchivo(string $archivo): string;
    abstract protected function parsearContenido(string $contenido): array;

    // Pasos con implementación por defecto sobrescribible
    protected function validarDatos(array $datos): void {}

    abstract protected function transformarDatos(array $datos): array;

    // Método final: no se puede sobrescribir
    final protected function registrarProcesamiento(string $archivo): void
    {
        echo "[LOG] Archivo procesado: $archivo\n";
    }
}

class ProcesadorJSON extends ProcesadorArchivos
{
    protected function leerArchivo(string $archivo): string
    {
        return file_get_contents($archivo);
    }

    protected function parsearContenido(string $contenido): array
    {
        $datos = json_decode($contenido, true);
        if (json_last_error() !== JSON_ERROR_NONE) {
            throw new RuntimeException("JSON inválido: " . json_last_error_msg());
        }
        return $datos;
    }

    protected function transformarDatos(array $datos): array
    {
        // Añadir metadatos
        $datos['procesado_en'] = date('Y-m-d H:i:s');
        return $datos;
    }
}

$procesador = new ProcesadorJSON();
$resultado = $procesador->procesar('/tmp/datos.json');
```

El uso de `final` es una decisión de diseño importante. Úsalo para
prevenir usos incorrectos de tu código, pero no abuses de él, ya que
reduce la flexibilidad y extensibilidad.

===============================================================================
6. BUENAS PRÁCTICAS EN HERENCIA
===============================================================================

La herencia es una herramienta poderosa pero a menudo mal utilizada.
Estos principios te ayudarán a diseñar jerarquías correctamente:

**1. Prefiere composición sobre herencia (Favor composition over inheritance)**

No forces una jerarquía de herencia donde no existe una relación "es un".
Si la relación es "tiene un" (has-a), usa composición (propiedades que
son objetos de otra clase).

```php
<?php

// MAL: Herencia forzada (un Coche NO es un Motor)
class CocheMal extends Motor { /* ... */ }

// BIEN: Composición (un Coche TIENE UN Motor)
class Coche {
    private Motor $motor;
    public function __construct(Motor $motor) {
        $this->motor = $motor;
    }
}
```

**2. No crees jerarquías muy profundas**

Más de 3 niveles de herencia es una señal de alerta. Las jerarquías
profundas son difíciles de entender, mantener y depurar.

**3. Liskov Substitution Principle (LSP)**

Una subclase debe poder sustituir a su clase padre sin alterar el
comportamiento correcto del programa. Si un método de la subclase
rompe las expectativas del padre, la herencia está mal diseñada.

```php
<?php

// Violación de LSP
class RectanguloLSP
{
    public function __construct(
        protected int $base,
        protected int $altura
    ) {}

    public function setBase(int $base): void { $this->base = $base; }
    public function setAltura(int $altura): void { $this->altura = $altura; }
    public function area(): int { return $this->base * $this->altura; }
}

class CuadradoLSP extends RectanguloLSP
{
    // Violación: cambiar base también cambia altura
    public function setBase(int $base): void {
        $this->base = $base;
        $this->altura = $base; // Comportamiento inesperado
    }
}
```

**4. Diseña para herencia o prohíbela**

Si una clase no está diseñada para ser extendida, declárala `final`.
Si esperas que sea extendida, documenta claramente qué métodos pueden
ser sobrescritos y cómo.

**5. Usa clases abstractas para bases parciales**

Las clases abstractas (capítulo 36) son ideales para definir
comportamiento base que requiere implementación en las hijas.

```php
<?php

abstract class Reporte
{
    // Plantilla de algoritmo
    final public function generar(): string
    {
        $encabezado = $this->encabezado();
        $cuerpo = $this->cuerpo();
        $pie = $this->pie();
        return $encabezado . "\n" . $cuerpo . "\n" . $pie;
    }

    abstract protected function encabezado(): string;
    abstract protected function cuerpo(): string;
    abstract protected function pie(): string;
}

class ReportePDF extends Reporte
{
    protected function encabezado(): string { return "PDF Header"; }
    protected function cuerpo(): string { return "PDF Body"; }
    protected function pie(): string { return "PDF Footer"; }
}
```

**6. No uses herencia solo para reutilizar código**

Si dos clases comparten código pero no tienen relación "es un", extrae
el código común a una clase de utilidad, trait o mediante composición,
no mediante herencia.

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- La herencia (`extends`) crea una relación "es un" entre clases,
  permitiendo que la hija herede propiedades y métodos públicos y
  protegidos del padre.
- La sobrescritura (override) permite redefinir métodos del padre en la
  hija, respetando reglas de firma compatible y visibilidad.
- `parent::` invoca métodos del padre desde la clase hija, útil para
  extender comportamiento en lugar de reemplazarlo.
- `final` en clases impide herencia; en métodos impide sobrescritura.
- Prefiere composición sobre herencia cuando la relación no es "es un".
- LSP: una subclase debe poder sustituir a su padre sin romper el programa.
- Las jerarquías profundas (>3 niveles) son difíciles de mantener.

$valor): bool`. `ValidadorEmail`, `ValidadorLongitud`, `ValidadorRango`
que hereden. Todos deben tener un método `getError(): string`. Crea un

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puede una clase hija tener el mismo nombre de propiedad que el padre?**
Sí, la clase hija puede redefinir propiedades. Si la propiedad del padre
es private, la hija puede crear una con el mismo nombre (son diferentes).
Si es protected o public, la hija la sobrescribe (cambia el valor inicial
o el tipo).

❓ **¿Qué es la herencia múltiple y por qué PHP no la soporta?**
La herencia múltiple permite que una clase herede de varias clases padres
simultáneamente. PHP no la soporta por los problemas del "diamante"
(conflicto cuando dos padres tienen métodos con el mismo nombre). PHP
ofrece Traits como alternativa.

❓ **¿Puedo cambiar la visibilidad de un método al sobrescribirlo?**
Puedes aumentar la visibilidad (protected -> public) pero no disminuirla
(public -> protected). Esto sigue el principio de que una subclase puede
ser más accesible pero no más restrictiva que su padre.

❓ **¿Qué diferencia hay entre self:: y parent::?**
self:: se refiere a la clase donde se escribe el código. parent:: se
refiere a la clase padre directa. En herencia, self:: puede apuntar a
la clase hija si se usa con late static binding (static:: en lugar de
self::).

❓ **¿Cómo sé si debo usar herencia o composición?**
Usa herencia si hay una relación "es un" (un Perro es un Animal). Usa
composición si hay una relación "tiene un" (un Coche tiene un Motor).
En caso de duda, prefiere composición: es más flexible y menos acoplada.

❓ **¿Puedo evitar que una clase sea instanciada pero permitir herencia?**
Sí. Declara la clase como abstracta. Las clases abstractas no pueden
instanciarse directamente pero pueden ser extendidas. Esto es ideal para
bases parciales.

DESAFÍO DEL ARQUITECTO

🏗️ **Sistema de Gestión de Empleados con Jerarquía Completa** — Diseña un
sistema completo de gestión de empleados que demuestre todos los conceptos
de herencia: extends, override, parent::, final, LSP, y jerarquía bien
diseñada.

Clases requeridas:

1. `Persona` (clase base) — Propiedades protegidas: nombre, apellidos,
   email, fechaNacimiento. Constructor con property promotion. Métodos:
   getNombreCompleto(): string, getEdad(): int, getEmail(): string,
   setEmail(string $email): void (con validación), esMayorDeEdad(): bool.

2. `Empleado` extends Persona — Añade: salario (protected float),
   fechaContratacion, puesto, activo (bool). Constructor llama a parent.
   Métodos: getSalarioAnual(): float, getSalarioMensual(): float,
   calcularBono(): float (método abierto a override, bono base = 0),
   trabajar(): string. Marca `setEmail` como final (no debe sobrescribirse).

3. `Gerente` extends Empleado — Añade: bonoGestion (float), equipo
   (array de Empleado). Sobrescribe calcularBono(): float = bonoGestion.
   Métodos: agregarAlEquipo(Empleado $e), getTamanoEquipo(): int,
   evaluarDesempeño(Empleado $e): string. Sobrescribe trabajar() para
   que diga "gestionando equipo de N personas".

4. `Desarrollador` extends Empleado — Añade: tecnologias (array),
   nivel (junior, mid, senior). Sobrescribe calcularBono(): float =
   10% salario si senior, 5% si mid, 0% si junior. Métodos: agregar
   Tecnologia(string $tech), getTecnologias(): array. trabajar() dice
   "desarrollando con [tecnologías]".

5. `Vendedor` extends Empleado — Añade: comisiones, metaVentas.
   Sobrescribe calcularBono(): float = 10% salario si comisiones >=
   metaVentas, sino 0%. trabajar() dice "realizando visitas comerciales".

6. `Empresa` (clase final) — Contiene array de Empleado. Métodos:
   contratar(Empleado $e), despedir(string $email), getTotalNomina():
   float, getTotalBonos(): float, listarPorPuesto(string $puesto):
   array, getEmpleadoDelMes(): Empleado (el que más bono generó).

Requisitos adicionales:

- La jerarquía debe ser coherente (Persona -> Empleado -> [Gerente,
  Desarrollador, Vendedor]).
- Los constructores deben usar property promotion con parent::__construct()
  donde corresponda.
- Al menos dos métodos deben ser final (uno en Persona, uno en Empleado).
- Demostrar LSP: una función que acepte Empleado debe funcionar con
  cualquier subclase.
- Demostrar que una clase Gerente no puede sobrescribir setEmail (final).
- Incluir un script de demostración con 6+ empleados de diferentes tipos,
  operaciones de nómina y reportes.

Este desafío integra todos los conceptos de herencia y produce un sistema
jerárquico bien diseñado que demuestra el poder y las limitaciones de la
herencia en PHP.
