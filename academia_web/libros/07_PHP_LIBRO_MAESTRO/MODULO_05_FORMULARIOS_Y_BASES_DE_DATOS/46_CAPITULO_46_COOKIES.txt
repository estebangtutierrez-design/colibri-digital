===============================================================================
1. ¿QUÉ SON LAS COOKIES?
===============================================================================

Una cookie es un pequeño archivo de texto (máximo 4 KB) que el servidor envía
al navegador, y que el navegador almacena en el equipo del usuario. En
peticiones posteriores al mismo dominio, el navegador devuelve la cookie al
servidor.

Flujo típico:

  1. El servidor PHP llama a setcookie('tema', 'oscuro').
  2. PHP envía un header HTTP: Set-Cookie: tema=oscuro; ...
  3. El navegador almacena la cookie.
  4. En la siguiente petición, el navegador envía: Cookie: tema=oscuro.
  5. PHP lo pone a disposición en $_COOKIE['tema'].

Las cookies son específicas de dominio, ruta y navegador. No se comparten entre
dominios ni entre navegadores diferentes en la misma máquina.

Limitaciones importantes:

  - Tamaño máximo: 4 KB por cookie.
  - Límite de ~50-100 cookies por dominio (según el navegador).
  - Límite total de ~4-10 KB para todas las cookies de un dominio.
  - Solo pueden almacenar strings.
  - El usuario puede eliminarlas o desactivarlas en cualquier momento.
  - No son adecuadas para datos sensibles (viajan en cada petición).

Usos apropiados de cookies:

  - Preferencias de idioma.
  - Tema visual (claro/oscuro).
  - Token de "recordar sesión" (no los datos de sesión).
  - Carrito de compras no autenticado (limitado).
  - Seguimiento de consentimiento (GDPR).

Usos inapropiados:

  - Contraseñas (inseguro).
  - Datos personales sensibles.
  - Grandes cantidades de datos.
  - Estado crítico de la aplicación.

2. SETCOOKIE() — CREAR COOKIES
===============================================================================

La función setcookie() define una cookie para ser enviada al navegador. Debe
llamarse antes de cualquier salida HTML (como session_start()).

```php
<?php
setcookie('nombre', 'Juan', [
    'expires' => time() + 86400 * 30, // 30 días
    'path' => '/',
    'domain' => '',
    'secure' => true,
    'httponly' => true,
    'samesite' => 'Lax',
]);
?>
<!DOCTYPE html>
<html>
...
</html>
```

Parámetros de setcookie():

```php
setcookie(
    string $name,           // Nombre de la cookie
    string $value = '',     // Valor
    int $expires_or_options = 0,  // Timestamp de expiración, o arreglo de opciones
    string $path = '',      // Ruta donde está disponible
    string $domain = '',    // Dominio
    bool $secure = false,   // Solo HTTPS
    bool $httponly = false  // No accesible por JavaScript
);
```

A partir de PHP 7.3, se recomienda usar un arreglo asociativo para mayor
claridad:

```php
<?php
setcookie('preferencia_tema', 'oscuro', [
    'expires' => time() + 86400 * 365, // 1 año
    'path' => '/',
    'domain' => 'ejemplo.com',
    'secure' => true,
    'httponly' => true,
    'samesite' => 'Lax',
]);
?>
```

Ejemplos de cookies:

```php
<?php
// Cookie de sesión (expira al cerrar el navegador)
setcookie('sesion_activa', '1');

// Cookie con expiración en 1 hora
setcookie('ultima_visita', date('Y-m-d H:i:s'), time() + 3600);

// Cookie de preferencia de idioma
setcookie('idioma', 'es', time() + 86400 * 30, '/');

// Cookie de "recordar sesión" (30 días)
$token = bin2hex(random_bytes(32));
setcookie('remember_token', $token, [
    'expires' => time() + 86400 * 30,
    'path' => '/',
    'secure' => true,
    'httponly' => true,
    'samesite' => 'Lax',
]);

// Cookie con array serializado
$preferencias = ['fuente' => 14, 'tema' => 'oscuro', 'idioma' => 'es'];
setcookie('preferencias', json_encode($preferencias), time() + 86400 * 30);
?>
```

Importante: setcookie() no afecta a $_COOKIE en la misma petición. La cookie
estará disponible en $_COOKIE solo a partir de la siguiente petición.

```php
<?php
setcookie('prueba', 'valor');
echo $_COOKIE['prueba'] ?? 'No disponible aún';
// Muestra "No disponible aún" hasta la siguiente recarga
?>
```

Para usar la cookie inmediatamente, debes asignarla manualmente a $_COOKIE:

```php
<?php
setcookie('prueba', 'valor');
$_COOKIE['prueba'] = 'valor'; // Forzar disponibilidad inmediata
echo $_COOKIE['prueba']; // "valor"
?>
```

===============================================================================
3. LEER COOKIES CON $_COOKIE
===============================================================================

$_COOKIE es una superglobal que contiene todas las cookies enviadas por el
navegador en la petición actual.

```php
<?php
if (isset($_COOKIE['idioma'])) {
    $idioma = $_COOKIE['idioma'];
} else {
    $idioma = 'es'; // Valor por defecto
}

// Aplicar preferencia de idioma
if ($idioma === 'en') {
    echo "Hello!";
} else {
    echo "¡Hola!";
}
?>
```

Cookies con JSON:

```php
<?php
if (isset($_COOKIE['preferencias'])) {
    $preferencias = json_decode($_COOKIE['preferencias'], true);
    if ($preferencias && isset($preferencias['tema'])) {
        $tema = $preferencias['tema'];
    }
}

$tema = $tema ?? 'claro';
echo "Tema seleccionado: " . htmlspecialchars($tema);
?>
```

Múltiples cookies del mismo nombre (técnicamente posible pero raro):

```php
<?php
// Si hay dos cookies 'id' para diferentes rutas, solo la más específica se envía
$id = $_COOKIE['id'] ?? '';
?>
```

Las cookies no tienen seguridad de tipo. Todo llega como string:

```php
<?php
setcookie('contador', 5); // Se almacena como "5"
$contador = (int)($_COOKIE['contador'] ?? 0); // Convertir a int
?>
```

Siempre trata los valores de cookies como datos no confiables. Sanitiza y
valida como lo harías con $_GET o $_POST.

===============================================================================
4. EXPIRACIÓN Y ELIMINACIÓN DE COOKIES
===============================================================================

Las cookies tienen una fecha de expiración. Cuando se alcanza, el navegador las
elimina automáticamente.

Cookie de sesión (sin expiración):

```php
<?php
// Expira cuando el usuario cierra el navegador
setcookie('temporal', 'valor');
?>
```

Cookie con expiración fija:

```php
<?php
// Expira en 30 días
setcookie('permanente', 'valor', time() + 86400 * 30);

// Expira en 1 hora
setcookie('hora', 'valor', time() + 3600);

// Expira al final de hoy (medianoche)
setcookie('hoy', 'valor', strtotime('tomorrow 00:00:00'));
?>
```

Eliminar una cookie: establece la expiración en el pasado.

```php
<?php
// Eliminar cookie
setcookie('nombre_cookie', '', time() - 3600);

// Eliminar con opciones (deben coincidir con las de creación)
setcookie('nombre_cookie', '', [
    'expires' => time() - 3600,
    'path' => '/',
    'domain' => 'ejemplo.com',
]);
?>
```

Para eliminar correctamente una cookie, los parámetros path, domain y secure
deben coincidir exactamente con los que se usaron al crearla. Si no coinciden,
el navegador no la eliminará.

```php
<?php
// Crear cookie con path específico
setcookie('app_session', 'abc123', time() + 86400, '/admin');

// Eliminar: debe usar el mismo path
setcookie('app_session', '', time() - 3600, '/admin');
?>
```

===============================================================================
5. PARÁMETROS DE SEGURIDAD
===============================================================================

Los parámetros de seguridad protegen las cookies contra accesos no autorizados.

Secure:

```php
<?php
// Solo se envía por conexiones HTTPS
setcookie('secreta', 'valor', [
    'secure' => true,
]);
?>
```

Sin secure, la cookie se envía también por HTTP, donde puede ser interceptada
(ataque man-in-the-middle).

HttpOnly:

```php
<?php
// No accesible por JavaScript (document.cookie)
setcookie('sesion_id', 'abc123', [
    'httponly' => true,
]);
?>
```

HttpOnly previene que scripts maliciosos (XSS) roben la cookie. Las cookies de
sesión de PHP (PHPSESSID) deberían tener httponly activado.

SameSite:

```php
<?php
// Strict: no se envía en peticiones de otros sitios
setcookie('csrf_token', 'token', [
    'samesite' => 'Strict',
]);

// Lax: se envía en navegación por GET desde otros sitios (default implícito)
setcookie('preferencia', 'valor', [
    'samesite' => 'Lax',
]);

// None: se envía siempre (requiere Secure)
setcookie('analytics', 'id', [
    'samesite' => 'None',
    'secure' => true,
]);
?>
```

SameSite previene ataques CSRF (Cross-Site Request Forgery):

  - Strict: La cookie no se envía en ninguna petición originada desde otro sitio.
    Máxima seguridad, pero puede romper funcionalidades como "pagar con" o
    enlaces externos.
  - Lax: La cookie se envía en peticiones GET de navegación (clic en enlaces),
    pero no en peticiones POST, iframes, imágenes, scripts, etc. Es el valor
    por defecto en navegadores modernos.
  - None: La cookie se envía en todas las peticiones. Solo debe usarse cuando
    Strict y Lax no son viables. Requiere Secure.

Configuración completa de una cookie segura:

```php
<?php
setcookie('token_sesion', $token, [
    'expires' => time() + 86400 * 7,
    'path' => '/',
    'domain' => 'ejemplo.com',
    'secure' => true,     // Solo HTTPS
    'httponly' => true,   // No JavaScript
    'samesite' => 'Lax',  // Protección CSRF básica
]);
?>
```

===============================================================================
6. COOKIES VS SESIONES
===============================================================================

¿Cuándo usar cookies y cuándo usar sesiones?

| Característica        | Cookies                          | Sesiones                    |
|-----------------------|----------------------------------|-----------------------------|
| Almacenamiento        | Cliente (navegador)              | Servidor                    |
| Capacidad             | 4 KB por cookie                  | Ilimitada (depende del medio) |
| Seguridad             | Datos visibles y modificables    | Datos seguros en servidor   |
| Persistencia          | Configurable (días, años)        | Expira al cerrar navegador  |
| Rendimiento           | Se envían en cada petición       | Solo se envía el ID         |
| Datos sensibles       | No recomendado                   | Sí                          |
| Control del usuario   | Puede eliminarlas                | No tiene acceso directo     |

Regla general:

  - Usa sesiones para datos que no deben ser manipulados por el usuario:
    estado de autenticación, roles, carrito de compras.
  - Usa cookies para preferencias no sensibles: idioma, tema, token de
    "recordar sesión".

Ejemplo combinado: sesiones para el estado de login, cookies para preferencias:

```php
<?php
session_start();

// Cookie: preferencia de idioma (no sensible)
$idioma = $_COOKIE['idioma'] ?? 'es';

// Sesión: estado de autenticación
if (isset($_SESSION['usuario_id'])) {
    echo "Bienvenido, usuario " . $_SESSION['usuario_id'];
    echo "<br>Idioma seleccionado: " . htmlspecialchars($idioma);
}

// Cambiar idioma
if (isset($_GET['cambiar_idioma'])) {
    $nuevoIdioma = $_GET['cambiar_idioma'];
    setcookie('idioma', $nuevoIdioma, time() + 86400 * 365, '/');
    $_COOKIE['idioma'] = $nuevoIdioma;
    header('Location: ' . str_replace('?cambiar_idioma=' . $nuevoIdioma, '', $_SERVER['REQUEST_URI']));
    exit;
}
?>
```

7. RECORDAR SESIÓN (REMEMBER ME)
===============================================================================

Un patrón común es "Recordar sesión": una cookie que permite al usuario
permanecer autenticado incluso después de cerrar el navegador.

Implementación segura:

  1. Al hacer login con "Recordarme", genera un token aleatorio.
  2. Guarda el token en la base de datos asociado al usuario.
  3. Guarda el token en una cookie con expiración larga (30 días).
  4. Al visitar el sitio, si no hay sesión activa pero hay cookie, verifica
     el token contra la BD.

```php
<?php
session_start();

// login.php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $usuario = $_POST['usuario'] ?? '';
    $password = $_POST['password'] ?? '';
    $recordar = isset($_POST['recordar']);

    if (verificarCredenciales($usuario, $password)) {
        // Iniciar sesión normal
        $_SESSION['usuario_id'] = obtenerId($usuario);
        $_SESSION['usuario'] = $usuario;

        if ($recordar) {
            // Generar token seguro
            $token = bin2hex(random_bytes(32));
            $tokenHash = hash('sha256', $token);

            // Guardar hash en BD
            guardarTokenRecordar($_SESSION['usuario_id'], $tokenHash);

            // Crear cookie (30 días)
            setcookie('recordar', $token, [
                'expires' => time() + 86400 * 30,
                'path' => '/',
                'secure' => true,
                'httponly' => true,
                'samesite' => 'Lax',
            ]);
        }

        header('Location: dashboard.php');
        exit;
    }
}

// En el inicio de cada página protegida
function verificarRecordar(): void
{
    // Si ya hay sesión, no hacer nada
    if (isset($_SESSION['usuario_id'])) {
        return;
    }

    // Si hay cookie de recordar, verificar token
    if (isset($_COOKIE['recordar'])) {
        $token = $_COOKIE['recordar'];
        $tokenHash = hash('sha256', $token);

        $usuarioId = verificarTokenRecordar($tokenHash);

        if ($usuarioId) {
            // Token válido: iniciar sesión
            session_regenerate_id(true);
            $_SESSION['usuario_id'] = $usuarioId;
            $_SESSION['usuario'] = obtenerNombre($usuarioId);

            // Renovar cookie
            $nuevoToken = bin2hex(random_bytes(32));
            $nuevoHash = hash('sha256', $nuevoToken);
            actualizarTokenRecordar($usuarioId, $nuevoHash);

            setcookie('recordar', $nuevoToken, [
                'expires' => time() + 86400 * 30,
                'path' => '/',
                'secure' => true,
                'httponly' => true,
                'samesite' => 'Lax',
            ]);
        } else {
            // Token inválido: eliminar cookie
            setcookie('recordar', '', time() - 3600, '/');
        }
    }
}
```

Seguridad del "Recordar sesión":

  - El token debe ser generado con random_bytes().
  - En la BD se almacena el hash del token, no el token mismo.
  - El token se renueva en cada uso (rotación).
  - La cookie tiene httponly, secure y samesite.
  - Siempre hay un límite de expiración (30 días es estándar).

Tabla SQL para tokens:

```sql
CREATE TABLE tokens_recordar (
    id INT AUTO_INCREMENT PRIMARY KEY,
    usuario_id INT NOT NULL,
    token_hash VARCHAR(64) NOT NULL UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP NOT NULL,
    FOREIGN KEY (usuario_id) REFERENCES usuarios(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

8. COOKIES Y EL REGLAMENTO GDPR / RGPD
===============================================================================

Si tu sitio web tiene audiencia en la Unión Europea, debes cumplir con el
Reglamento General de Protección de Datos (RGPD/GDPR).

Requisitos principales:

  1. Consentimiento: Debes obtener consentimiento explícito antes de
     establecer cookies no esenciales (analítica, publicidad, redes sociales).
  2. Información: Debes informar claramente qué cookies usas y para qué.
  3. Derecho de exclusión: El usuario debe poder rechazar cookies no
     esenciales.
  4. Transparencia: La política de cookies debe ser fácilmente accesible.

Implementación básica de consentimiento:

```php
<?php
// banner-cookies.php
$cookiesAceptadas = $_COOKIE['cookies_aceptadas'] ?? false;
$analiticaPermitida = $_COOKIE['analitica_permiso'] ?? false;
?>
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Política de Cookies</title>
</head>
<body>
    <?php if (!$cookiesAceptadas): ?>
        <div id="cookie-banner" style="
            position: fixed; bottom: 0; width: 100%;
            background: #333; color: #fff; padding: 20px;
            text-align: center;
        ">
            <p>
                Este sitio utiliza cookies propias y de terceros para mejorar
                tu experiencia. <a href="/politica-cookies.php">Más información</a>.
            </p>
            <button onclick="aceptarCookies()">Aceptar todas</button>
            <button onclick="rechazarCookies()">Solo necesarias</button>
        </div>
        <script>
        function aceptarCookies() {
            document.cookie = "cookies_aceptadas=1; path=/; max-age=" + (86400 * 365);
            document.cookie = "analitica_permiso=1; path=/; max-age=" + (86400 * 365);
            document.getElementById('cookie-banner').style.display = 'none';
            location.reload();
        }
        function rechazarCookies() {
            document.cookie = "cookies_aceptadas=1; path=/; max-age=" + (86400 * 365);
            document.cookie = "analitica_permiso=0; path=/; max-age=" + (86400 * 365);
            document.getElementById('cookie-banner').style.display = 'none';
        }
        </script>
    <?php endif; ?>

    <?php if ($analiticaPermitida): ?>
        <!-- Código de Google Analytics u otra herramienta -->
    <?php endif; ?>
</body>
</html>
```

Clasificación de cookies:

  - Esenciales: No requieren consentimiento. Son necesarias para el
    funcionamiento del sitio (PHPSESSID, cookies de autenticación).
  - Funcionales: Preferencias del usuario (idioma, tema). Consentimiento
    implícito.
  - Analítica: Estadísticas de uso. Requieren consentimiento explícito.
  - Publicitarias: Seguimiento de comportamiento. Requieren consentimiento
    explícito.

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Las cookies son datos almacenados en el navegador del cliente.
- setcookie() establece cookies; debe llamarse antes de cualquier salida HTML.
- $_COOKIE contiene las cookies recibidas en la petición actual.
- Las cookies expiran según el timestamp que se establezca; para eliminarlas,
  usa un timestamp en el pasado.
- Los parámetros secure, httponly y samesite son esenciales para la seguridad.
- Las cookies tienen límite de 4 KB por cookie y ~50 por dominio.
- Usa sesiones para datos sensibles y cookies para preferencias no críticas.
- El patrón "Recordar sesión" debe implementarse con tokens hash rotados.
- El RGPD/GDPR requiere consentimiento para cookies no esenciales.
- Siempre sanitiza los valores de cookies como datos no confiables.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo almacenar objetos en una cookie?** No directamente. Las cookies
solo almacenan strings. Puedes serializar con json_encode() y luego
json_decode().

❓ **¿Por qué mi cookie no se crea?** Posibles causas: setcookie() después de
HTML, la cookie ya existe con diferentes parámetros (path/domain), el
navegador tiene cookies desactivadas.

❓ **¿Las cookies se envían en peticiones AJAX?** Sí, si el dominio, ruta y
protocolo coinciden. SameSite puede bloquearlas según la configuración.

❓ **¿Puedo leer cookies de otros dominios?** No. Por seguridad, los
navegadores solo envían cookies al dominio que las creó.

❓ **¿Qué pasa si el usuario borra las cookies?** La aplicación debe manejar
la ausencia de cookies con valores por defecto o redirigiendo al login.

❓ **¿Las cookies caducan aunque el usuario no visite el sitio?** Sí, la
expiración se basa en la fecha absoluta, no en visitas. La cookie se elimina
al alcanzar la fecha de expiración.

❓ **¿Cuál es la diferencia entre session.use_cookies y setcookie()?**
session.use_cookies es una configuración que determina si PHP usa cookies
para el ID de sesión. setcookie() es una función para crear cookies
arbitrarias.

DESAFÍO DEL ARQUITECTO

🏗️ **Sistema de Personalización de Experiencia de Usuario con Cookies Seguras**

Diseña y construye un sistema completo de personalización basado en cookies
que cumpla con los siguientes requisitos:

  1. Panel de preferencias de usuario (/preferencias.php):
     - Tema: Claro, Oscuro, Automático (sigue al sistema).
     - Idioma: Español, Inglés, Portugués.
     - Zona horaria: selector con las 24 zonas horarias principales.
     - Formato de fecha: DD/MM/AAAA, MM/DD/AAAA, AAAA-MM-DD.
     - Resultados por página: 10, 20, 50, 100.
     - Notificaciones: Sonido, Vibración, Ninguna.
     - Cada preferencia debe guardarse en una cookie independiente (no una
       cookie gigante con todo).

  2. Aplicación de preferencias globales:
     - Archivo init.php que se incluye en todas las páginas.
     - Lee todas las cookies de preferencias al inicio.
     - Aplica el tema mediante una clase CSS en el <body>.
     - Configura el locale de PHP según el idioma.
     - Establece la zona horaria con date_default_timezone_set().

  3. Seguridad de cookies:
     - Todas las cookies de preferencias con httponly=false (para que JS
       pueda leerlas y aplicar el tema sin recargar), pero secure=true.
     - La cookie de "recordar sesión" con httponly=true, secure=true,
       samesite=Lax.
     - Validación de valores: solo aceptar valores predefinidos (lista
       blanca), nunca confiar ciegamente del valor de la cookie.

  4. Función de "Resetear preferencias":
     - Botón que elimina todas las cookies de preferencias.
