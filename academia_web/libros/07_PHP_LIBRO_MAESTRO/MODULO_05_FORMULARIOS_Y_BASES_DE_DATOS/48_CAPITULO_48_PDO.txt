===============================================================================
1. ¿QUÉ ES PDO Y POR QUÉ USARLO?
===============================================================================

PDO (PHP Data Objects) es una extensión de PHP que define una interfaz
uniforme para acceder a diferentes bases de datos.

Ventajas de PDO:

  - Abstracción de base de datos: el mismo código funciona con MySQL,
    PostgreSQL, SQLite, etc. Solo cambia el DSN (Data Source Name).
  - Prepared statements: separa el SQL de los datos, previniendo inyección
    SQL de forma nativa.
  - Manejo de errores: soporta excepciones (modo exception).
  - Transacciones: beginTransaction(), commit(), rollback().
  - Fetch modes: fetch() en diferentes formatos (asociativo, objeto, columna).
  - Binding: bindValue() y bindParam() para mayor control.
  - Rendimiento: las consultas preparadas se ejecutan más rápido en
    repeticiones.

Comparación con extensiones antiguas:

| Característica         | mysql_* (obsoleto)  | mysqli             | PDO                |
|------------------------|---------------------|--------------------|--------------------|
| Orientación a objetos | No                  | Sí                 | Sí                 |
| Prepared statements   | No                  | Sí                 | Sí                 |
| Excepciones           | No                  | Parcial            | Sí                 |
| Múltiples BD          | Solo MySQL          | Solo MySQL         | MySQL, PgSQL, etc. |
| Named parameters      | No                  | No                 | Sí                 |
| Activo                | Eliminado PHP 7     | Sí                 | Sí                 |

Regla: Siempre usa PDO para nuevos proyectos. mysqli solo tiene sentido en
proyectos legacy.

===============================================================================
2. CONEXIÓN A MYSQL CON PDO
===============================================================================

```php
<?php
// Configuración de la conexión
$host = 'localhost';
$dbname = 'tienda_online';
$usuario = 'root';
$password = '';
$charset = 'utf8mb4';

// DSN (Data Source Name)
$dsn = "mysql:host=$host;dbname=$dbname;charset=$charset";

// Opciones de PDO
$opciones = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES   => false,
    PDO::MYSQL_ATTR_INIT_COMMAND => "SET NAMES utf8mb4 COLLATE utf8mb4_unicode_ci",
];

try {
    $pdo = new PDO($dsn, $usuario, $password, $opciones);
    echo "Conexión exitosa a la base de datos.";
} catch (PDOException $e) {
    // En producción, registra el error y muestra un mensaje genérico
    error_log("Error de conexión: " . $e->getMessage());
    die("Error de conexión a la base de datos. Intente más tarde.");
}
?>
```

Explicación de las opciones:

  - ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION: Lanza excepciones en errores.
    Facilita el manejo de errores con try-catch.
  - ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC: Por defecto, fetch() devuelve
    arreglo asociativo.
  - ATTR_EMULATE_PREPARES => false: Usa prepared statements reales del
    servidor, no emulados. Más seguro.
  - MYSQL_ATTR_INIT_COMMAND: Comando ejecutado al conectar. Útil para
    establecer charset.

Conexión con diferentes bases de datos:

```php
<?php
// MySQL
$dsn = "mysql:host=localhost;dbname=tienda;charset=utf8mb4";

// PostgreSQL
$dsn = "pgsql:host=localhost;dbname=tienda;port=5432";

// SQLite (archivo local)
$dsn = "sqlite:/ruta/a/base_de_datos.sqlite";

// SQL Server
$dsn = "sqlsrv:Server=localhost;Database=tienda";

// Oracle
$dsn = "oci:dbname=localhost/XE";
?>
```

Conexión singleton (patrón de una sola conexión):

```php
<?php
class BaseDeDatos
{
    private static ?PDO $instancia = null;

    public static function conectar(): PDO
    {
        if (self::$instancia === null) {
            $dsn = "mysql:host=localhost;dbname=tienda;charset=utf8mb4";
            self::$instancia = new PDO($dsn, 'root', '', [
                PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
                PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
                PDO::ATTR_EMULATE_PREPARES => false,
            ]);
        }
        return self::$instancia;
    }
}

// Uso en cualquier parte del código:
$pdo = BaseDeDatos::conectar();
?>
```

===============================================================================
3. QUERY() Y EXEC() — CONSULTAS SIMPLES
===============================================================================

query() ejecuta una consulta SQL sin parámetros y devuelve un objeto
PDOStatement con los resultados.

```php
<?php
require 'conexion.php';

// SELECT con query()
$stmt = $pdo->query('SELECT * FROM usuarios');
$usuarios = $stmt->fetchAll();

foreach ($usuarios as $usuario) {
    echo htmlspecialchars($usuario['nombre']) . " - " . htmlspecialchars($usuario['email']) . "<br>";
}
?>
```

exec() ejecuta una consulta que no devuelve resultados (INSERT, UPDATE, DELETE)
y devuelve el número de filas afectadas.

```php
<?php
require 'conexion.php';

// INSERT con exec()
$afectadas = $pdo->exec(
    "INSERT INTO categorias (nombre, slug) VALUES ('Juguetes', 'juguetes')"
);
echo "Filas insertadas: $afectadas";

// UPDATE con exec()
$afectadas = $pdo->exec(
    "UPDATE productos SET precio = precio * 1.1 WHERE categoria_id = 1"
);
echo "Productos actualizados: $afectadas";

// DELETE con exec()
$afectadas = $pdo->exec(
    "DELETE FROM usuarios WHERE activo = 0"
);
echo "Usuarios eliminados: $afectadas";
?>
```

Advertencia: query() y exec() son peligrosas si concatenas valores del usuario
directamente. Siempre usa prepared statements cuando los datos provengan del
usuario.

```php
<?php
// PELIGROSO: inyección SQL
$pdo->query("SELECT * FROM usuarios WHERE email = '" . $_GET['email'] . "'");

// SEGURO: prepared statement
$stmt = $pdo->prepare('SELECT * FROM usuarios WHERE email = :email');
$stmt->execute([':email' => $_GET['email']]);
?>
```

===============================================================================
4. PREPARED STATEMENTS
===============================================================================

Los prepared statements separan el SQL de los datos. El SQL se prepara una vez
y se ejecuta con diferentes valores. Esto previene la inyección SQL y mejora
el rendimiento en consultas repetitivas.

Parámetros nombrados (más legibles):

```php
<?php
require 'conexion.php';

$sql = 'INSERT INTO usuarios (nombre, email, password, edad, pais)
        VALUES (:nombre, :email, :password, :edad, :pais)';

$stmt = $pdo->prepare($sql);

$stmt->execute([
    ':nombre' => 'Ana García',
    ':email' => 'ana@ejemplo.com',
    ':password' => password_hash('123456', PASSWORD_DEFAULT),
    ':edad' => 28,
    ':pais' => 'mx',
]);

echo "Usuario creado con ID: " . $pdo->lastInsertId();
?>
```

Parámetros posicionales (?):

```php
<?php
require 'conexion.php';

$sql = 'INSERT INTO usuarios (nombre, email, password, edad, pais)
        VALUES (?, ?, ?, ?, ?)';

$stmt = $pdo->prepare($sql);

$stmt->execute([
    'Carlos López',
    'carlos@ejemplo.com',
    password_hash('abcdef', PASSWORD_DEFAULT),
    35,
    'es',
]);

echo "Usuario creado con ID: " . $pdo->lastInsertId();
?>
```

SELECT con prepared statement:

```php
<?php
require 'conexion.php';

$email = $_GET['email'] ?? '';

$stmt = $pdo->prepare('SELECT * FROM usuarios WHERE email = :email');
$stmt->execute([':email' => $email]);
$usuario = $stmt->fetch();

if ($usuario) {
    echo "Nombre: " . htmlspecialchars($usuario['nombre']);
} else {
    echo "Usuario no encontrado.";
}
?>
```

Múltiples ejecuciones (eficiente para bucles):

```php
<?php
require 'conexion.php';

$sql = 'INSERT INTO productos (nombre, precio, stock) VALUES (:nombre, :precio, :stock)';
$stmt = $pdo->prepare($sql);

$productos = [
    ['nombre' => 'Laptop', 'precio' => 15000.00, 'stock' => 10],
    ['nombre' => 'Mouse', 'precio' => 250.50, 'stock' => 50],
    ['nombre' => 'Teclado', 'precio' => 450.00, 'stock' => 30],
];

foreach ($productos as $producto) {
    $stmt->execute($producto);
    echo "Insertado: " . $producto['nombre'] . "<br>";
}
?>
```

bindValue() y bindParam():

```php
<?php
require 'conexion.php';

// bindValue(): asigna el valor en el momento
$stmt = $pdo->prepare('SELECT * FROM usuarios WHERE edad > :edad');
$stmt->bindValue(':edad', 18, PDO::PARAM_INT);
$stmt->execute();

// bindParam(): asigna por referencia (el valor puede cambiar antes de ejecutar)
$limite = 10;
$stmt = $pdo->prepare('SELECT * FROM usuarios LIMIT :limite');
$stmt->bindParam(':limite', $limite, PDO::PARAM_INT);
$stmt->execute();

$limite = 20; // Cambiar antes de ejecutar
// $stmt->execute(); // Ahora usaría LIMIT 20
?>
```

La diferencia clave: bindValue() copia el valor en el momento; bindParam() usa
una referencia a la variable, por lo que el valor final depende del estado de
la variable al ejecutar execute().

===============================================================================
5. FETCH — RECUPERAR DATOS
===============================================================================

fetch() recupera la siguiente fila de resultados. fetchAll() recupera todas.

PDO::FETCH_ASSOC — Arreglo asociativo:

```php
<?php
require 'conexion.php';

$stmt = $pdo->query('SELECT id, nombre, email FROM usuarios');
$stmt->execute();

while ($fila = $stmt->fetch(PDO::FETCH_ASSOC)) {
    echo "{$fila['id']}: {$fila['nombre']} ({$fila['email']})<br>";
}
?>
```

PDO::FETCH_NUM — Arreglo numérico:

```php
<?php
$stmt = $pdo->query('SELECT nombre, email FROM usuarios');

while ($fila = $stmt->fetch(PDO::FETCH_NUM)) {
    echo "{$fila[0]} - {$fila[1]}<br>";
}
?>
```

PDO::FETCH_OBJ — Objeto anónimo:

```php
<?php
$stmt = $pdo->query('SELECT id, nombre, email FROM usuarios');

while ($usuario = $stmt->fetch(PDO::FETCH_OBJ)) {
    echo "{$usuario->id}: {$usuario->nombre} ({$usuario->email})<br>";
}
?>
```

PDO::FETCH_COLUMN — Columna única:

```php
<?php
$stmt = $pdo->query('SELECT email FROM usuarios');
$emails = $stmt->fetchAll(PDO::FETCH_COLUMN);
// $emails = ['ana@ejemplo.com', 'carlos@ejemplo.com', ...]
?>
```

PDO::FETCH_KEY_PAIR — Pares clave-valor (2 columnas):

```php
<?php
$stmt = $pdo->query('SELECT id, nombre FROM usuarios');
$nombres = $stmt->fetchAll(PDO::FETCH_KEY_PAIR);
// $nombres = [1 => 'Ana García', 2 => 'Carlos López', ...]
?>
```

PDO::FETCH_CLASS — Mapear a una clase:

```php
<?php
class Usuario
{
    public int $id;
    public string $nombre;
    public string $email;
    public int $edad;

    public function saludar(): string
    {
        return "Hola, soy {$this->nombre}";
    }
}

$stmt = $pdo->query('SELECT id, nombre, email, edad FROM usuarios');
$stmt->setFetchMode(PDO::FETCH_CLASS, 'Usuario');

while ($usuario = $stmt->fetch()) {
    echo $usuario->saludar() . "<br>";
}
?>
```

fetchAll() — Todas las filas de una vez:

```php
<?php
$stmt = $pdo->query('SELECT id, nombre, email FROM usuarios');
$usuarios = $stmt->fetchAll(PDO::FETCH_ASSOC);

// $usuarios es un arreglo de arreglos asociativos
foreach ($usuarios as $u) {
    echo htmlspecialchars($u['nombre']) . "<br>";
}

// Contar resultados
echo "Total: " . count($usuarios);
?>
```

fetchColumn() — Primer valor de la primera columna:

```php
<?php
$stmt = $pdo->query('SELECT COUNT(*) FROM usuarios');
$total = $stmt->fetchColumn();
echo "Total de usuarios: $total";

$stmt = $pdo->prepare('SELECT nombre FROM usuarios WHERE id = :id');
$stmt->execute([':id' => 1]);
$nombre = $stmt->fetchColumn();
echo "Nombre del usuario 1: " . htmlspecialchars($nombre);
?>
```

6. TRANSACCIONES
===============================================================================

Una transacción agrupa múltiples operaciones en una unidad atómica: todas se
completan exitosamente o ninguna se aplica.

```php
<?php
require 'conexion.php';

try {
    $pdo->beginTransaction();

    // Descontar stock del producto
    $stmt1 = $pdo->prepare(
        'UPDATE productos SET stock = stock - :cantidad WHERE id = :id AND stock >= :cantidad2'
    );
    $stmt1->execute([
        ':cantidad' => 2,
        ':id' => 5,
        ':cantidad2' => 2,
    ]);

    if ($stmt1->rowCount() === 0) {
        throw new Exception('Stock insuficiente para el producto 5.');
    }

    // Crear pedido
    $stmt2 = $pdo->prepare(
        'INSERT INTO pedidos (usuario_id, total, estado) VALUES (:usuario_id, :total, :estado)'
    );
    $stmt2->execute([
        ':usuario_id' => 1,
        ':total' => 599.99,
        ':estado' => 'confirmado',
    ]);

    $pedidoId = $pdo->lastInsertId();

    // Insertar detalle del pedido
    $stmt3 = $pdo->prepare(
        'INSERT INTO detalle_pedido (pedido_id, producto_id, cantidad, precio)
         VALUES (:pedido_id, :producto_id, :cantidad, :precio)'
    );
    $stmt3->execute([
        ':pedido_id' => $pedidoId,
        ':producto_id' => 5,
        ':cantidad' => 2,
        ':precio' => 299.99,
    ]);

    $pdo->commit();
    echo "Pedido #$pedidoId creado exitosamente.";
} catch (Exception $e) {
    $pdo->rollBack();
    error_log("Error al crear pedido: " . $e->getMessage());
    echo "Error al procesar el pedido. Intente de nuevo.";
}
?>
```

Beneficios de las transacciones:

  - Atomicidad: todo o nada.
  - Consistencia: los datos siempre están en estado válido.
  - Aislamiento: las operaciones no se interfieren entre sí.
  - Durabilidad: una vez confirmado, los datos persisten.

Uso típico: transferencias bancarias, pedidos, registros que afectan múltiples
tablas, cualquier operación que requiera consistencia.

Las transacciones solo funcionan con tablas InnoDB (no MyISAM).

7. MANEJO DE ERRORES
===============================================================================

PDO soporta tres modos de error:

```php
<?php
// Modo 1: Silencioso (PDO::ERRMODE_SILENT) — por defecto hasta PHP 7
// Debes verificar manualmente con errorCode() y errorInfo()
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_SILENT);
$stmt = $pdo->query('SELECT * FROM tabla_inexistente');
if ($stmt === false) {
    $error = $pdo->errorInfo();
    echo "Error SQL: {$error[2]}";
}

// Modo 2: Warning (PDO::ERRMODE_WARNING)
// Muestra un warning pero continúa la ejecución
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_WARNING);

// Modo 3: Excepción (PDO::ERRMODE_EXCEPTION) — RECOMENDADO
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
try {
    $stmt = $pdo->query('SELECT * FROM tabla_inexistente');
} catch (PDOException $e) {
    error_log("Error en BD: " . $e->getMessage());
    echo "Ocurrió un error. Intente más tarde.";
}
?>
```

Siempre usa el modo exception. Facilita el manejo centralizado de errores.

Obteniendo información del error:

```php
<?php
try {
    $stmt = $pdo->prepare('SELECT * FROM usuarios WHERE id = :id');
    $stmt->execute([':id' => 'texto']); // Error de tipo
} catch (PDOException $e) {
    echo "Código de error: " . $e->getCode() . "<br>";
    echo "Mensaje: " . $e->getMessage() . "<br>";
    echo "Archivo: " . $e->getFile() . ":" . $e->getLine() . "<br>";

    // Para depuración avanzada
    $errorInfo = $stmt->errorInfo();
    echo "SQLSTATE: {$errorInfo[0]}<br>";
    echo "Código driver: {$errorInfo[1]}<br>";
    echo "Mensaje driver: {$errorInfo[2]}<br>";
}
?>
```

Nunca muestres detalles del error al usuario en producción. Siempre registra
el error en un log y muestra un mensaje genérico.

8. CONSULTAS AVANZADAS CON PDO
===============================================================================

LIKE con prepared statement:

```php
<?php
$termino = $_GET['q'] ?? '';
$terminoBusqueda = "%{$termino}%";

$stmt = $pdo->prepare(
    'SELECT * FROM productos WHERE nombre LIKE :termino OR descripcion LIKE :termino2'
);
$stmt->execute([
    ':termino' => $terminoBusqueda,
    ':termino2' => $terminoBusqueda,
]);
$resultados = $stmt->fetchAll();
?>
```

IN con prepared statement:

```php
<?php
$ids = [1, 3, 5, 7];

// Generar placeholders dinámicamente
$placeholders = implode(',', array_fill(0, count($ids), '?'));

$stmt = $pdo->prepare(
    "SELECT * FROM productos WHERE id IN ($placeholders)"
);
$stmt->execute($ids);
$productos = $stmt->fetchAll();
?>
```

INSERT múltiple con prepared statement:

```php
<?php
$datos = [
    ['Ana', 'ana@mail.com', 28],
    ['Luis', 'luis@mail.com', 32],
    ['Eva', 'eva@mail.com', 25],
];

$sql = 'INSERT INTO usuarios (nombre, email, edad) VALUES (?, ?, ?)';
$stmt = $pdo->prepare($sql);

foreach ($datos as $fila) {
    $stmt->execute($fila);
}

echo "Se insertaron " . count($datos) . " usuarios.";
?>
```

UPDATE con JOIN:

```php
<?php
$stmt = $pdo->prepare(
    'UPDATE productos p
     JOIN categorias c ON p.categoria_id = c.id
     SET p.precio = p.precio * 1.15
     WHERE c.nombre = :categoria'
);
$stmt->execute([':categoria' => 'Electrónica']);
echo "Productos actualizados: " . $stmt->rowCount();
?>
```

RESUMEN DEL CAPÍTULO
===============================================================================

- PDO es una capa de abstracción para conectar PHP con múltiples bases de datos.
- La conexión se establece con new PDO($dsn, $usuario, $password, $opciones).
- Siempre usa PDO::ERRMODE_EXCEPTION y PDO::FETCH_ASSOC como opciones por
  defecto.
- query() ejecuta SELECT sin parámetros; exec() ejecuta INSERT/UPDATE/DELETE.
- Los prepared statements separan el SQL de los datos, previniendo inyección
  SQL.
- Usa parámetros nombrados (:nombre) o posicionales (?) en las consultas.
- fetch() recupera una fila; fetchAll() recupera todas.
- Las transacciones (beginTransaction/commit/rollback) garantizan atomicidad
  en operaciones múltiples.
- ATTR_EMULATE_PREPARES=false asegura prepared statements reales del servidor.
- Nunca concatenes valores del usuario en SQL; siempre usa prepared statements.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué significa PDO?** PHP Data Objects. Es una extensión que define una
interfaz ligera y consistente para acceder a bases de datos en PHP.

❓ **¿PDO es más lento que mysqli?** La diferencia es insignificante en
aplicaciones reales. La seguridad y portabilidad de PDO compensan cualquier
mínima diferencia de rendimiento.

❓ **¿Cómo obtengo el ID del último registro insertado?** Con
$pdo->lastInsertId().

❓ **¿Qué es el DSN?** Data Source Name. Es una cadena que describe la
ubicación y configuración de la base de datos (host, nombre, charset).

❓ **¿Qué hace ATTR_EMULATE_PREPARES?** Si es true, PDO emula prepared
statements en PHP. Si es false, usa los prepared statements nativos del
servidor. Usa false para máxima seguridad.

❓ **¿Puedo usar PDO con PostgreSQL?** Sí, PDO soporta MySQL, PostgreSQL,
SQLite, Oracle, SQL Server, Firebird, ODBC, etc.

❓ **¿Qué pasa si hago rollback sin haber empezado una transacción?** PDO
lanza una PDOException.

🏗️ **Capa de Abstracción de Base de Datos con PDO (Mini ORM)**

Diseña y construye una clase envoltorio para PDO que abstraiga las
operaciones más comunes de base de datos. Tu clase debe incluir:

  1. Conexión:
     - Constructor que acepte configuración (host, dbname, usuario, password,
       charset) o un arreglo de configuración.
     - Implementación singleton (una sola conexión en toda la ejecución).
     - Configuración segura: exception mode, fetch assoc,
       emulate_prepares=false, charset utf8mb4.

  2. Métodos CRUD genéricos:
     - obtenerTodos(string $tabla, string $orden = 'id ASC', ?int $limite = null): array
     - obtenerPorId(string $tabla, int $id, string $columnaId = 'id'): ?array
     - obtenerDonde(string $tabla, array $condiciones, string $orden = 'id ASC'): array
     - insertar(string $tabla, array $datos): int (devuelve el ID insertado)
     - actualizar(string $tabla, int $id, array $datos, string $columnaId = 'id'): int
     - eliminar(string $tabla, int $id, string $columnaId = 'id'): bool

  3. Características adicionales:
     - obtenerPorPagina(string $tabla, int $pagina, int $porPagina = 10,
       array $condiciones = []): arreglo con 'datos', 'total', 'paginas'.
     - contar(string $tabla, array $condiciones = []): int.
     - ejecutarConsulta(string $sql, array $params = []): PDOStatement.
     - transaccion(callable $callback): mixed (ejecuta $callback dentro de
       una transacción, hace commit si ok, rollback si excepción).

  4. Manejo de errores:
     - Los errores deben registrarse con error_log().
     - Lanza excepciones personalizadas (DatabaseException).
     - Nunca expone detalles de la base de datos al usuario.
