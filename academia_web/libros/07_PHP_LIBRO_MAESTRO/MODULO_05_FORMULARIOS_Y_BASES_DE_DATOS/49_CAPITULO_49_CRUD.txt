1. ARQUITECTURA DE UN SISTEMA CRUD
===============================================================================

Un sistema CRUD típico tiene esta estructura:

| Operación | Método HTTP | URL                        | Archivo PHP         |
|-----------|-------------|----------------------------|---------------------|
| Listar    | GET         | /productos.php             | productos.php       |
| Ver       | GET         | /productos.php?id=5        | productos.php       |
| Crear     | GET+POST   | /productos-crear.php       | productos-crear.php |
| Editar    | GET+POST   | /productos-editar.php?id=5 | productos-editar.php|
| Eliminar  | POST        | /productos-eliminar.php    | productos-eliminar.php |

Otra arquitectura común es tener un solo archivo por recurso que maneje todas
las operaciones según el parámetro action:

| URL                             | acción    | Descripción       |
|---------------------------------|-----------|-------------------|
| /productos.php                  | listar    | Lista de productos|
| /productos.php?action=crear     | crear     | Formulario crear  |
| /productos.php?action=editar&id=5 | editar | Formulario editar |
| /productos.php?action=eliminar&id=5 | eliminar | Confirmar+eliminar |

Usaremos la primera arquitectura (archivos separados) por claridad.

Estructura de archivos del proyecto CRUD:

```
crud-productos/
|-- config/
|   |-- database.php      # Conexión PDO
|   |-- session.php       # Sesión + mensajes flash
|-- layout/
|   |-- header.php        # Header HTML
|   |-- footer.php        # Footer HTML
|-- css/
|   |-- estilo.css        # Estilos
|-- productos.php         # Listado
|-- productos-crear.php   # Crear
|-- productos-editar.php  # Editar
|-- productos-eliminar.php# Eliminar
|-- index.php             # Redirección al listado
```

2. CONFIGURACIÓN Y BASE DE DATOS
===============================================================================

Base de datos:

```sql
CREATE DATABASE IF NOT EXISTS crud_productos
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

USE crud_productos;

CREATE TABLE IF NOT EXISTS productos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(200) NOT NULL,
    descripcion TEXT,
    precio DECIMAL(10, 2) NOT NULL,
    stock INT UNSIGNED NOT NULL DEFAULT 0,
    categoria VARCHAR(50) NOT NULL DEFAULT 'general',
    activo BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

config/database.php:

```php
<?php
$host     = 'localhost';
$dbname   = 'crud_productos';
$usuario  = 'root';
$password = '';
$charset  = 'utf8mb4';

$dsn = "mysql:host=$host;dbname=$dbname;charset=$charset";

$opciones = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES   => false,
];

try {
    $pdo = new PDO($dsn, $usuario, $password, $opciones);
} catch (PDOException $e) {
    error_log("Error de conexión: " . $e->getMessage());
    die("Error de conexión a la base de datos.");
}
?>
```

config/session.php:

```php
<?php
session_start();

// CSRF token
if (empty($_SESSION['csrf_token'])) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}

// Mensajes flash
function mensajeFlash(string $tipo, string $texto): void
{
    $_SESSION['flash'][] = ['tipo' => $tipo, 'texto' => $texto];
}

function obtenerFlash(): array
{
    $mensajes = $_SESSION['flash'] ?? [];
    unset($_SESSION['flash']);
    return $mensajes;
}
?>
```

productos-crear.php:

```php
<?php
require 'config/database.php';
require 'config/session.php';

$errores = [];
$datos = [
    'nombre' => '',
    'descripcion' => '',
    'precio' => '',
    'stock' => '',
    'categoria' => 'general',
    'activo' => 1,
];

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $tokenRecibido = $_POST['csrf_token'] ?? '';
    if (!hash_equals($_SESSION['csrf_token'], $tokenRecibido)) {
        die('Error de validación CSRF.');
    }

    $datos['nombre'] = trim($_POST['nombre'] ?? '');
    $datos['descripcion'] = trim($_POST['descripcion'] ?? '');
    $datos['precio'] = str_replace(',', '.', trim($_POST['precio'] ?? ''));
    $datos['stock'] = $_POST['stock'] ?? '';
    $datos['categoria'] = $_POST['categoria'] ?? 'general';
    $datos['activo'] = isset($_POST['activo']) ? 1 : 0;

    // Validaciones
    if ($datos['nombre'] === '') {
        $errores[] = 'El nombre del producto es obligatorio.';
    } elseif (strlen($datos['nombre']) > 200) {
        $errores[] = 'El nombre no debe exceder 200 caracteres.';
    }

    $precioValido = filter_var($datos['precio'], FILTER_VALIDATE_FLOAT);
    if ($precioValido === false || $precioValido < 0) {
        $errores[] = 'Ingrese un precio válido (mayor o igual a 0).';
    } else {
        $datos['precio'] = $precioValido;
    }

    $stockValido = filter_var($datos['stock'], FILTER_VALIDATE_INT, ['options' => ['min_range' => 0]]);
    if ($stockValido === false) {
        $errores[] = 'El stock debe ser un número entero mayor o igual a 0.';
    } else {
        $datos['stock'] = $stockValido;
    }

    $categoriasPermitidas = ['general', 'electronica', 'ropa', 'hogar', 'libros', 'deportes'];
    if (!in_array($datos['categoria'], $categoriasPermitidas)) {
        $errores[] = 'Seleccione una categoría válida.';
    }

    if (empty($errores)) {
        try {
            $stmt = $pdo->prepare(
                'INSERT INTO productos (nombre, descripcion, precio, stock, categoria, activo)
                 VALUES (:nombre, :descripcion, :precio, :stock, :categoria, :activo)'
            );
            $stmt->execute([
                ':nombre' => $datos['nombre'],
                ':descripcion' => $datos['descripcion'],
                ':precio' => $datos['precio'],
                ':stock' => $datos['stock'],
                ':categoria' => $datos['categoria'],
                ':activo' => $datos['activo'],
            ]);

            $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
            mensajeFlash('exito', 'Producto creado exitosamente.');
            header('Location: productos.php');
            exit;
        } catch (PDOException $e) {
            error_log("Error al crear producto: " . $e->getMessage());
            $errores[] = 'Error al guardar el producto. Intente de nuevo.';
        }
    }
}

require 'layout/header.php';
?>

<h1>Crear Producto</h1>

<?php if (!empty($errores)): ?>

        <ul>
            <?php foreach ($errores as $error): ?>

            <?php endforeach; ?>
        </ul>
    </div>
<?php endif; ?>

    </div>

    <div class="campo">
        <label for="descripcion">Descripción</label>
```

6. DELETE — ELIMINAR PRODUCTO
===============================================================================

productos-eliminar.php:

```php
<?php
require 'config/database.php';
require 'config/session.php';

if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    header('Location: productos.php');
    exit;
}

$tokenRecibido = $_POST['csrf_token'] ?? '';
if (!hash_equals($_SESSION['csrf_token'], $tokenRecibido)) {
    die('Error de validación CSRF.');
}

$id = (int)($_POST['id'] ?? 0);

if ($id <= 0) {
    mensajeFlash('error', 'ID de producto inválido.');
    header('Location: productos.php');
    exit;
}

try {
    $stmt = $pdo->prepare('DELETE FROM productos WHERE id = :id');
    $stmt->execute([':id' => $id]);

    if ($stmt->rowCount() > 0) {
        $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
        mensajeFlash('exito', 'Producto eliminado correctamente.');
    } else {
        mensajeFlash('error', 'El producto no existe o ya fue eliminado.');
    }
} catch (PDOException $e) {
    error_log("Error al eliminar producto: " . $e->getMessage());
    mensajeFlash('error', 'Error al eliminar el producto.');
}

header('Location: productos.php');
exit;
?>
```

8. FLUJO COMPLETO DEL CRUD
===============================================================================

Resumen del flujo de cada operación:

CREATE:
  1. GET productos-crear.php → muestra formulario vacío.
  2. Usuario llena campos y hace clic en "Guardar".
  3. POST productos-crear.php → valida, inserta en BD, redirige a listado.
  4. Listado muestra mensaje flash de éxito.

READ:
  1. GET productos.php → consulta BD con paginación y filtros.
  2. Muestra tabla con todos los productos.
  3. Enlaces a editar y formularios para eliminar.

UPDATE:
  1. Clic en "Editar" → GET productos-editar.php?id=X.
  2. Consulta producto por ID, llena formulario con datos actuales.
  3. Usuario modifica campos y hace clic en "Actualizar".
  4. POST productos-editar.php → valida, actualiza BD, redirige.
  5. Listado muestra mensaje flash de éxito.

DELETE:
  1. Clic en "Eliminar" → confirmación con JavaScript.
  2. POST productos-eliminar.php → verifica CSRF, elimina de BD.
  3. Redirige a listado con mensaje flash.

Puntos clave de seguridad:

  - CSRF: cada formulario POST tiene token CSRF.
  - SQL injection: todos los valores se pasan por prepared statements.
  - XSS: htmlspecialchars() en todas las salidas.
  - Validación: server-side en todas las operaciones.
  - Método HTTP: GET solo para lectura; POST para escritura.
  - ID: convertido a (int) antes de usar en consultas.

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- CRUD son las cuatro operaciones fundamentales: Create, Read, Update, Delete.
- Cada operación CRUD debe asociarse al método HTTP correcto.
- Los prepared statements de PDO protegen contra inyección SQL.
- Los tokens CSRF protegen los formularios de escritura.
- La paginación mejora la usabilidad en listados grandes.
- Los mensajes flash (con sesiones) dan feedback al usuario tras
  redirecciones.
- htmlspecialchars() previene XSS en todas las salidas.
- La validación server-side es obligatoria en crear y editar.
- DELETE debe usar POST (nunca GET) y confirmación del usuario.
- La estructura de archivos separados (crear, editar, eliminar) mantiene el
  código organizado.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Debo usar GET o POST para eliminar?** Siempre POST. GET es para
consultas. DELETE con GET podría ser pre-cargado por el navegador o un bot de
Google y eliminar datos accidentalmente.

❓ **¿Por qué usar mensajes flash en lugar de mostrar el mensaje directamente?**
Los mensajes flash sobreviven a una redirección HTTP. Si muestras el mensaje
directamente y luego rediriges, se pierde. Flash lo guarda en sesión.

❓ **¿Cómo manejo la paginación con filtros?** Incluye todos los parámetros de
filtro en los enlaces de paginación (q, categoria, etc.) para que los filtros
se preserven.

❓ **¿Qué es la regeneración del token CSRF?** Después de una operación
exitosa, generas un nuevo token CSRF para prevenir su reutilización.

❓ **¿Debo validar también en el cliente?** Sí, la validación HTML5 con
required, min, type="number" mejora la experiencia. Pero nunca confíes solo en
ella.

❓ **¿Cómo hago para que la tabla sea responsiva?** Agrega
overflow-x: auto al contenedor de la tabla, o usa un diseño de tarjetas en
pantallas pequeñas.

❓ **¿Qué hago si tengo muchas columnas?** Muestra las más importantes en el
listado y usa un botón "Ver detalle" para el resto. También puedes permitir
que el usuario seleccione qué columnas ver.

🏗️ **Sistema CRUD Completo para Gestión de Biblioteca con Relaciones y Roles**

Construye un sistema CRUD completo para una biblioteca que gestione libros,
autores, miembros y préstamos. El sistema debe cumplir con:

  1. Base de datos (MySQL con InnoDB):
     - autores: id, nombre, apellido, fecha_nacimiento, nacionalidad, biografia.
     - libros: id, titulo, isbn (UNIQUE), autor_id (FK), año, editorial, genero,
       sinopsis, ejemplares_totales, ejemplares_disponibles, created_at.
     - miembros: id, nombre, apellido, email (UNIQUE), telefono, direccion,
       fecha_registro, activo.
     - prestamos: id, libro_id (FK), miembro_id (FK), fecha_prestamo,
       fecha_devolucion_estimada, fecha_devolucion_real (NULL), estado.

  2. CRUD de Autores:
     - Listado con búsqueda por nombre.
     - Crear: nombre, apellido, fecha_nacimiento, nacionalidad (select), biografia.
     - Editar: mismos campos.
     - Eliminar: solo si no tiene libros asociados.

  3. CRUD de Libros:
     - Listado con búsqueda por título, autor, ISBN. Paginación.
     - Crear: todos los campos. Select de autores poblado desde BD.
     - Al crear, ejemplares_disponibles = ejemplares_totales.
     - Editar: mismos campos.
     - Eliminar: solo si no tiene préstamos activos.

  4. CRUD de Miembros:
     - Listado con búsqueda y paginación.
     - Crear/Editar con validación de email único.
     - Eliminar: desactivar (borrado lógico, no físico).

  5. Gestión de Préstamos:
     - Crear préstamo: seleccionar miembro (autocomplete), libro (solo
       disponibles), fecha_prestamo (hoy), fecha_devolucion_estimada (hoy+7).
     - Al crear, disminuir ejemplares_disponibles del libro.
     - Devolver libro: botón que actualiza fecha_devolucion_real y
       aumenta ejemplares_disponibles.
     - No permitir préstamo si ejemplares_disponibles = 0.
     - No permitir préstamo si el miembro tiene más de 3 préstamos activos.
     - Listado de préstamos activos y devueltos con filtros.

  6. Dashboard (index.php):
     - Total de libros, autores, miembros, préstamos activos.
     - Préstamos vencidos (fecha_devolucion_estimada pasada y no devueltos).
     - Top 5 libros más prestados.

  7. Seguridad y usabilidad:
     - Tokens CSRF en todos los formularios POST.
     - Prepared statements en todas las consultas.
     - htmlspecialchars() en todas las salidas.
     - Mensajes flash para éxito/error.
     - Confirmación antes de eliminar.
     - Validación server-side completa.
     - Paginación en todos los listados.

  8. Bonus: exportar listado de libros a CSV.

  Criterios de evaluación:
  - CRUD completo y funcional para las 4 entidades (autores, libros, miembros,
    préstamos).
  - Relaciones correctas entre tablas con integridad referencial.
  - Reglas de negocio implementadas correctamente (stock, límite de préstamos).
  - Código modular, seguro y bien estructurado.
  - Interfaz limpia y funcional.
