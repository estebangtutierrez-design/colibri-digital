1. ¿QUÉ ES MYSQL?
===============================================================================

MySQL es un sistema de gestión de bases de datos relacionales (RDBMS) de
código abierto. Almacena datos en tablas relacionadas entre sí mediante
claves.

Conceptos fundamentales:

  - Base de datos: Contenedor de tablas. Un servidor MySQL puede tener
    múltiples bases de datos.
  - Tabla: Estructura que almacena datos en filas y columnas. Similar a una
    hoja de cálculo.
  - Columna (campo): Un tipo de dato específico (nombre, email, edad).
  - Fila (registro): Un conjunto completo de datos (un usuario, un producto).
  - Clave primaria (PRIMARY KEY): Columna que identifica de forma única cada
    fila.
  - Clave foránea (FOREIGN KEY): Columna que relaciona una tabla con otra.
  - Índice (INDEX): Estructura que acelera las búsquedas.

PHP y MySQL trabajan juntos:

  - PHP recibe datos del usuario (formulario).
  - PHP procesa y valida los datos.
  - PHP envía consultas SQL a MySQL.
  - MySQL ejecuta las consultas y devuelve resultados.
  - PHP procesa los resultados y genera HTML.

Ejemplo conceptual:

```text
Tabla "usuarios" en MySQL:
+----+-----------+------------------+-----------+
| id | nombre    | email            | edad      |
+----+-----------+------------------+-----------+
| 1  | Ana       | ana@mail.com     | 28        |
| 2  | Carlos    | carlos@mail.com  | 35        |
| 3  | María     | maria@mail.com   | 22        |
+----+-----------+------------------+-----------+

PHP consulta: "SELECT * FROM usuarios WHERE edad > 25"
MySQL responde: filas 1 y 2
PHP muestra: "Ana (28) y Carlos (35)"
```

3. CREAR BASE DE DATOS Y TABLAS
===============================================================================

Una base de datos es el contenedor principal. Dentro de ella creamos las
tablas con sus columnas y tipos de datos.

```sql
-- Crear base de datos
CREATE DATABASE IF NOT EXISTS tienda_online
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

-- Usar la base de datos
USE tienda_online;

-- Crear tabla de usuarios
CREATE TABLE IF NOT EXISTS usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(150) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    edad TINYINT UNSIGNED NOT NULL,
    pais CHAR(2) NOT NULL DEFAULT 'mx',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

Tipos de datos comunes en MySQL:

| Tipo           | Descripción                          | Ejemplo                   |
|----------------|--------------------------------------|---------------------------|
| INT            | Número entero (4 bytes)              | 42, -5, 0                |
| TINYINT        | Entero pequeño (1 byte, -128 a 127)  | 18, 120                  |
| SMALLINT       | Entero mediano (2 bytes)             | 1000, -500               |
| BIGINT         | Entero grande (8 bytes)              | 9999999999               |
| FLOAT          | Decimal de precisión simple          | 3.14, -0.5               |
| DECIMAL(p,s)   | Decimal exacto para dinero           | 19.99, 1000.00           |
| VARCHAR(n)     | Cadena de longitud variable          | 'Ana García'             |
| CHAR(n)        | Cadena de longitud fija              | 'MX', 'ES'               |
| TEXT           | Texto largo (hasta 65 KB)            | 'Artículo completo...'   |
| LONGTEXT       | Texto muy largo (hasta 4 GB)         | 'Libro entero...'        |
| DATE           | Fecha YYYY-MM-DD                     | '2000-05-15'             |
| DATETIME       | Fecha y hora YYYY-MM-DD HH:MM:SS     | '2024-01-15 14:30:00'    |
| TIMESTAMP      | Fecha y hora (timezone-aware)         | '2024-01-15 14:30:00'    |
| BOOLEAN        | Sinónimo de TINYINT(1)               | 1 (true), 0 (false)      |
| ENUM('a','b')  | Selección de un valor de una lista    | 'pendiente', 'completado' |

Modificadores de columnas:

  - NOT NULL: La columna no puede estar vacía.
  - DEFAULT valor: Valor por defecto si no se especifica.
  - AUTO_INCREMENT: Se incrementa automáticamente (solo enteros).
  - UNIQUE: No puede haber valores duplicados.
  - PRIMARY KEY: Identificador único de la tabla.
  - INDEX: Acelera búsquedas en esta columna.

Crear tabla de productos:

```sql
CREATE TABLE IF NOT EXISTS productos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(200) NOT NULL,
    descripcion TEXT,
    precio DECIMAL(10, 2) NOT NULL,
    stock INT UNSIGNED NOT NULL DEFAULT 0,
    categoria_id INT NOT NULL,
    activo BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (categoria_id) REFERENCES categorias(id)
        ON DELETE RESTRICT ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

Crear tabla de categorías:

```sql
CREATE TABLE IF NOT EXISTS categorias (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    slug VARCHAR(100) NOT NULL UNIQUE,
    descripcion TEXT,
    activo BOOLEAN DEFAULT TRUE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

Ver todas las tablas:

```sql
SHOW TABLES;
```

Ver estructura de una tabla:

```sql
DESCRIBE usuarios;
-- o
SHOW COLUMNS FROM usuarios;
```

Modificar tabla existente:

```sql
-- Agregar columna
ALTER TABLE usuarios ADD COLUMN telefono VARCHAR(15) AFTER email;

-- Modificar columna
ALTER TABLE usuarios MODIFY COLUMN telefono VARCHAR(20);

-- Eliminar columna
ALTER TABLE usuarios DROP COLUMN telefono;

-- Agregar índice
ALTER TABLE usuarios ADD INDEX idx_email (email);
```

4. INSERTAR DATOS (INSERT)
===============================================================================

```sql
-- Insertar un registro
INSERT INTO usuarios (nombre, email, password, edad, pais)
VALUES ('Ana García', 'ana@ejemplo.com', '$2y$10$...hash...', 28, 'mx');

-- Insertar múltiples registros
INSERT INTO categorias (nombre, slug) VALUES
    ('Electrónica', 'electronica'),
    ('Ropa', 'ropa'),
    ('Hogar', 'hogar'),
    ('Libros', 'libros'),
    ('Deportes', 'deportes');
```

Si la tabla tiene AUTO_INCREMENT, puedes omitir la columna id:

```sql
INSERT INTO usuarios (nombre, email, password, edad)
VALUES ('Carlos López', 'carlos@ejemplo.com', 'hash123', 35);
```

Insertar con valores por defecto:

```sql
INSERT INTO usuarios (nombre, email, password)
VALUES ('María Martínez', 'maria@ejemplo.com', 'hash456');
-- edad usará su valor por defecto (NULL si no se definió)
-- pais usará DEFAULT 'mx'
-- created_at usará CURRENT_TIMESTAMP
```

Obtener el ID del último registro insertado:

```sql
-- En MySQL CLI
SELECT LAST_INSERT_ID();

-- En PHP (con PDO, como veremos en el siguiente capítulo)
-- $pdo->lastInsertId();
```

Insertar usando SET (sintaxis alternativa):

```sql
INSERT INTO usuarios SET
    nombre = 'Pedro Sánchez',
    email = 'pedro@ejemplo.com',
    password = 'hash789',
    edad = 40;
```

5. CONSULTAR DATOS (SELECT)
===============================================================================

SELECT es la instrucción más usada. Recupera datos de una o más tablas.

```sql
-- Seleccionar todas las columnas de todos los usuarios
SELECT * FROM usuarios;

-- Seleccionar columnas específicas
SELECT nombre, email, edad FROM usuarios;

-- Seleccionar con alias
SELECT nombre AS 'Nombre Completo', email AS 'Correo' FROM usuarios;

-- Seleccionar con condición WHERE
SELECT * FROM usuarios WHERE edad >= 18;
SELECT * FROM usuarios WHERE pais = 'mx';
SELECT * FROM usuarios WHERE edad BETWEEN 18 AND 30;
SELECT * FROM usuarios WHERE nombre LIKE '%ana%';
SELECT * FROM usuarios WHERE email IN ('ana@ejemplo.com', 'maria@ejemplo.com');

-- Seleccionar con múltiples condiciones
SELECT * FROM usuarios
WHERE edad >= 18 AND pais = 'mx'
ORDER BY nombre ASC;

-- ORDENAR resultados
SELECT * FROM usuarios ORDER BY edad DESC;
SELECT * FROM usuarios ORDER BY pais ASC, nombre DESC;

-- LIMITAR resultados (paginación)
SELECT * FROM usuarios LIMIT 10;           -- primeros 10
SELECT * FROM usuarios LIMIT 10 OFFSET 20; -- 10 registros, saltando 20
SELECT * FROM usuarios LIMIT 20, 10;       -- Sintaxis alternativa: OFFSET, LIMIT

-- Contar registros
SELECT COUNT(*) AS total FROM usuarios;
SELECT COUNT(*) AS total_mx FROM usuarios WHERE pais = 'mx';

-- Agrupar y contar
SELECT pais, COUNT(*) AS cantidad
FROM usuarios
GROUP BY pais
ORDER BY cantidad DESC;

-- Valores únicos
SELECT DISTINCT pais FROM usuarios;

-- Funciones de agregación
SELECT
    AVG(edad) AS edad_promedio,
    MIN(edad) AS edad_minima,
    MAX(edad) AS edad_maxima,
    SUM(edad) AS suma_edades
FROM usuarios;

-- Seleccionar con JOIN (relacionar tablas)
SELECT
    p.nombre AS producto,
    p.precio,
    c.nombre AS categoria
FROM productos p
JOIN categorias c ON p.categoria_id = c.id
WHERE p.activo = TRUE
ORDER BY p.precio DESC;
```

Operadores de comparación en WHERE:

| Operador | Descripción          | Ejemplo                        |
|----------|----------------------|--------------------------------|
| =        | Igual a              | WHERE nombre = 'Ana'          |
| !=       | Diferente de         | WHERE pais != 'mx'            |
| >        | Mayor que            | WHERE edad > 18               |
| <        | Menor que            | WHERE precio < 100            |
| >=       | Mayor o igual        | WHERE stock >= 5              |
| <=       | Menor o igual        | WHERE created_at <= NOW()     |
| BETWEEN  | Entre rango          | WHERE edad BETWEEN 18 AND 30  |
| LIKE     | Patrón (con %)       | WHERE nombre LIKE '%ana%'     |
| IN       | En lista             | WHERE pais IN ('mx','es')     |
| IS NULL  | Es nulo              | WHERE descripcion IS NULL     |
| IS NOT NULL | No es nulo        | WHERE descripcion IS NOT NULL |

6. ACTUALIZAR Y ELIMINAR DATOS
===============================================================================

UPDATE:

```sql
-- Actualizar un registro por ID
UPDATE usuarios
SET nombre = 'Ana García López', edad = 29
WHERE id = 1;

-- Actualizar múltiples registros
UPDATE productos
SET precio = precio * 1.1  -- aumentar 10%
WHERE categoria_id = 3;

-- Actualizar con condiciones
UPDATE usuarios
SET activo = 0
WHERE ultimo_acceso < '2023-01-01';

-- Sin WHERE: actualiza TODOS los registros
UPDATE productos SET activo = 1;
```

DELETE:

```sql
-- Eliminar un registro específico
DELETE FROM usuarios WHERE id = 5;

-- Eliminar múltiples registros
DELETE FROM productos WHERE stock = 0 AND activo = FALSE;

-- Eliminar todos los registros (mantiene la estructura)
DELETE FROM usuarios;

-- TRUNCATE: elimina todos los registros y resetea AUTO_INCREMENT
TRUNCATE TABLE usuarios;
```

Diferencias entre DELETE y TRUNCATE:

  - DELETE: Elimina filas una por una. Dispara triggers. Mantiene
    AUTO_INCREMENT. Puede tener WHERE.
  - TRUNCATE: Elimina todas las filas de golpe. Resetea AUTO_INCREMENT. Más
    rápido. No puede tener WHERE. No dispara triggers.

Siempre usa WHERE en DELETE y UPDATE a menos que quieras afectar toda la
tabla. Olvidar el WHERE puede ser catastrófico.

8. BUENAS PRÁCTICAS CON MYSQL
===============================================================================

1. Siempre define una PRIMARY KEY en cada tabla (normalmente id con
   AUTO_INCREMENT).
2. Usa utf8mb4 para soportar emojis y caracteres internacionales.
3. Define FOREIGN KEY para mantener la integridad referencial.
4. Usa índices en columnas usadas frecuentemente en WHERE y JOIN.
5. No almacenes contraseñas en texto plano; guarda el hash.
6. Usa DECIMAL para dinero, no FLOAT (evita errores de redondeo).
7. Prefiere TIMESTAMP sobre DATETIME si trabajas con múltiples zonas horarias.
8. Nombra las tablas en plural y las columnas en singular (usuarios.nombre).
9. Documenta la estructura de la base de datos con comentarios.

Comentarios en SQL:

```sql
CREATE TABLE productos (
    id INT AUTO_INCREMENT PRIMARY KEY, -- Identificador único
    nombre VARCHAR(200) NOT NULL COMMENT 'Nombre del producto para mostrar',
    precio DECIMAL(10, 2) NOT NULL COMMENT 'Precio en MXN sin IVA'
) ENGINE=InnoDB;
```

Respaldo de base de datos (mysqldump):

```bash
# Respaldar una base de datos
mysqldump -u root -p tienda_online > respaldo_tienda.sql

# Respaldar múltiples bases de datos
mysqldump -u root -p --databases tienda_online blog > respaldo.sql

# Respaldar todas las bases de datos
mysqldump -u root -p --all-databases > respaldo_total.sql

# Restaurar un respaldo
mysql -u root -p tienda_online < respaldo_tienda.sql
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué significa utf8mb4?** Es el conjunto de caracteres UTF-8 que soporta
caracteres de 4 bytes, incluyendo emojis y caracteres especiales. El utf8
estándar en MySQL solo soporta hasta 3 bytes.

❓ **¿Cuál es la diferencia entre MyISAM e InnoDB?** InnoDB soporta
transacciones, claves foráneas y bloqueo a nivel de fila. MyISAM es más rápido
en lecturas simples pero no tiene integridad referencial. Usa InnoDB.

❓ **¿Qué hace ON DELETE CASCADE?** Cuando se elimina un registro padre, los
registros hijos también se eliminan automáticamente. ON DELETE RESTRICT
impide eliminar si hay hijos. ON DELETE SET NULL pone NULL en la FK.

❓ **¿Cómo reinicio el AUTO_INCREMENT?** Con ALTER TABLE usuarios AUTO_INCREMENT = 1;.
O truncando la tabla: TRUNCATE TABLE usuarios;.

❓ **¿Qué es SQL injection?** Es un ataque donde el usuario ingresa código SQL
malicioso en un campo de formulario. Se previene con prepared statements.
Nunca concatenes valores directamente en SQL.

❓ **¿Puedo tener bases de datos en diferentes servidores?** Sí, MySQL permite
conexiones remotas. También puedes usar FEDERATED tables o replicación.

❓ **¿Cómo veo todas las bases de datos?** SHOW DATABASES;

🏗️ **Diseño e Implementación del Esquema de Base de Datos para un Sistema de Gestión de una Biblioteca**

Diseña y construye el esquema completo de base de datos para un sistema de
gestión de biblioteca que cumpla con los siguientes requisitos:

  1. Tablas requeridas:
     - autores: id, nombre, apellido, fecha_nacimiento, nacionalidad (CHAR(2)),
       biografia (TEXT), created_at.
     - libros: id, titulo, isbn (VARCHAR(13), UNIQUE), autor_id (FK), anio_publicacion
       (SMALLINT), editorial, paginas (SMALLINT UNSIGNED), genero, sinopsis (TEXT),
       ejemplares_totales (SMALLINT UNSIGNED), ejemplares_disponibles (SMALLINT UNSIGNED),
       created_at.
     - miembros: id, nombre, apellido, email (UNIQUE), telefono, direccion,
       fecha_registro (DATE), activo (BOOLEAN).
     - prestamos: id, miembro_id (FK), libro_id (FK), fecha_prestamo (DATE),
       fecha_devolucion_estimada (DATE), fecha_devolucion_real (DATE, NULLABLE),
       estado ENUM('activo', 'devuelto', 'vencido'), created_at.
     - multas: id, prestamo_id (FK), monto (DECIMAL(8,2)), fecha_generacion (DATE),
       pagada (BOOLEAN DEFAULT FALSE).

  2. Índices y restricciones:
     - ISBN debe ser único.
     - Un miembro no puede tener más de 3 préstamos activos simultáneos.
       (Implementa como regla de negocio en la aplicación, no en SQL).
     - La fecha_devolucion_estimada debe ser mayor que fecha_prestamo.
     - Un libro solo puede prestarse si ejemplares_disponibles > 0.
     - Al crear un préstamo, debe disminuir ejemplares_disponibles.
     - Al devolver un libro, debe aumentar ejemplares_disponibles.

  3. Inserta datos de prueba:
     - 5 autores.
     - 10 libros (al menos 2 por autor).
     - 5 miembros.
     - 8 préstamos (algunos activos, algunos devueltos, algunos vencidos).
     - 2 multas.

  4. Consultas que debes diseñar y ejecutar:
     - Listar todos los libros con su autor (JOIN).
     - Mostrar los préstamos activos con nombre del miembro y título del libro.
     - Encontrar los libros más prestados (COUNT en préstamos).
     - Listar miembros con multas pendientes.
     - Calcular cuántos préstamos tiene cada miembro.
     - Buscar libros por título o autor (LIKE).
     - Mostrar libros con ejemplares disponibles menores a 3 (reorden sugerido).

  5. Vistas útiles:
     - vista_libros_disponibles: libros con ejemplares_disponibles > 0.
     - vista_prestamos_activos: préstamos no devueltos con datos del miembro
       y libro.

  6. Exporta el esquema completo como archivo SQL con comentarios explicativos.

  Criterios de evaluación:
  - Diseño normalizado (3FN: sin redundancias, dependencias funcionales correctas).
  - Tipos de datos apropiados para cada columna.
  - Integridad referencial con claves foráneas y acciones ON DELETE adecuadas.
  - Índices correctamente ubicados (FKs, columnas de búsqueda frecuente).
  - Datos de prueba representativos y variados.
  - Consultas correctas y optimizadas.
