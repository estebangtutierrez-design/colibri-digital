1. PATRÓN MVC EN PROFUNDIDAD
===============================================================================

El patrón **Modelo-Vista-Controlador (MVC)** es la columna vertebral de la
mayoría de aplicaciones PHP profesionales. Separa la lógica en tres capas
interconectadas pero independientes:

- **Modelo (Model)**: Gestiona los datos y la lógica de negocio. Se comunica
  con la base de datos y contiene las reglas del dominio.
- **Vista (View)**: Presenta la información al usuario. En una SPA (Single
  Page Application), la vista es manejada por React, Vue o Angular. En una
  aplicación tradicional, son archivos PHP que generan HTML.
- **Controlador (Controller)**: Actúa como intermediario. Recibe peticiones
  del usuario, consulta al modelo, y envía los datos a la vista.

```php
<?php
// Ejemplo conceptual de MVC en PHP
declare(strict_types=1);

// === CONTROLADOR ===
class UsuarioController
{
    private UsuarioModel $model;
    private UsuarioView $view;

    public function __construct()
    {
        $this->model = new UsuarioModel();
        $this->view = new UsuarioView();
    }

    public function mostrar(int $id): void
    {
        $usuario = $this->model->obtenerPorId($id);
        $this->view->renderizar($usuario);
    }
}

// === MODELO ===
class UsuarioModel
{
    private PDO $db;

    public function __construct()
    {
        $this->db = new PDO(
            'mysql:host=localhost;dbname=mi_app',
            'root',
            '',
            [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION]
        );
    }

    public function obtenerPorId(int $id): array
    {
        $stmt = $this->db->prepare(
            'SELECT id, nombre, email FROM usuarios WHERE id = :id'
        );
        $stmt->execute(['id' => $id]);
        return $stmt->fetch(PDO::FETCH_ASSOC);
    }
}

// === VISTA ===
class UsuarioView
{
    public function renderizar(array $usuario): void
    {
        echo json_encode([
            'status' => 'success',
            'data' => $usuario
        ]);
    }
}
?>
```

El MVC bien implementado permite que múltiples desarrolladores trabajen en
paralelo: uno en el controlador, otro en el modelo y otro en la vista, sin
pisarse el código entre sí.

Cada directorio tiene una responsabilidad única y clara:

  capas.

3. FRONT CONTROLLER Y ENRUTAMIENTO
===============================================================================

El **Front Controller** es un patrón de diseño que centraliza todas las
peticiones HTTP en un único punto de entrada (index.php). Apache o Nginx
redirigen todo el tráfico a este archivo mediante reglas de reescritura.

```apache
# public/.htaccess - Reglas de reescritura para Apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^ index.php [QSA,L]
</IfModule>
```

El enrutador analiza la URL, la compara con las rutas definidas y ejecuta el
controlador correspondiente:

```php
<?php
// src/Core/Router.php
declare(strict_types=1);

namespace App\Core;

class Router
{
    private array $routes = [];

    public function get(string $path, array $handler): void
    {
        $this->routes['GET'][$path] = $handler;
    }

    public function post(string $path, array $handler): void
    {
        $this->routes['POST'][$path] = $handler;
    }

    public function put(string $path, array $handler): void
    {
        $this->routes['PUT'][$path] = $handler;
    }

    public function delete(string $path, array $handler): void
    {
        $this->routes['DELETE'][$path] = $handler;
    }

    public function dispatch(Request $request): Response
    {
        $method = $request->getMethod();
        $uri = $request->getUri();

        if (!isset($this->routes[$method])) {
            return new Response('Método no permitido', 405);
        }

        foreach ($this->routes[$method] as $pattern => $handler) {
            $pattern = preg_replace('/\{(\w+)\}/', '(?P<$1>[^/]+)', $pattern);
            $pattern = '#^' . $pattern . '$#';

            if (preg_match($pattern, $uri, $matches)) {
                $params = array_filter($matches, 'is_string', ARRAY_FILTER_USE_KEY);
                [$controllerClass, $methodName] = $handler;
                $controller = new $controllerClass();
                $result = $controller->$methodName($request, $params);
                return $result instanceof Response
                    ? $result
                    : new Response(json_encode($result), 200, ['Content-Type' => 'application/json']);
            }
        }

        return new Response('Ruta no encontrada', 404);
    }
}
?>
```

4. PATRÓN SINGLETON PARA BASE DE DATOS
===============================================================================

El patrón **Singleton** garantiza que una clase tenga una única instancia
y proporciona un punto de acceso global a ella. Es ideal para la conexión
a la base de datos, evitando crear múltiples conexiones innecesarias.

```php
<?php
// src/Core/Database.php
declare(strict_types=1);

namespace App\Core;

class Database
{
    private static ?Database $instance = null;
    private PDO $pdo;

    private function __construct()
    {
        $host = $_ENV['DB_HOST'] ?? 'localhost';
        $port = $_ENV['DB_PORT'] ?? '3306';
        $dbname = $_ENV['DB_NAME'] ?? 'mi_app';
        $username = $_ENV['DB_USER'] ?? 'root';
        $password = $_ENV['DB_PASS'] ?? '';
        $charset = $_ENV['DB_CHARSET'] ?? 'utf8mb4';

        $dsn = "mysql:host={$host};port={$port};dbname={$dbname};charset={$charset}";

        $options = [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
            PDO::ATTR_EMULATE_PREPARES => false,
            PDO::ATTR_STRINGIFY_FETCHES => false,
        ];

        $this->pdo = new PDO($dsn, $username, $password, $options);
    }

    public static function getInstance(): self
    {
        if (self::$instance === null) {
            self::$instance = new self();
        }
        return self::$instance;
    }

    public function getConnection(): PDO
    {
        return $this->pdo;
    }

    // Evitar clonación y deserialización
    private function __clone() {}
    public function __wakeup()
    {
        throw new \Exception('No se puede deserializar un Singleton');
    }
}
?>
```

6. MANEJO DE ERRORES Y EXCEPCIONES GLOBALES
===============================================================================

Un buen arquitecto anticipa los errores y los maneja con elegancia. PHP
proporciona mecanismos para capturar y gestionar errores de forma
centralizada.

```php
<?php
// src/Core/ErrorHandler.php
declare(strict_types=1);

namespace App\Core;

use Throwable;

class ErrorHandler
{
    public static function register(): void
    {
        set_error_handler([self::class, 'handleError']);
        set_exception_handler([self::class, 'handleException']);
    }

    public static function handleError(
        int $severity,
        string $message,
        string $file,
        int $line
    ): bool {
        throw new \ErrorException($message, 0, $severity, $file, $line);
    }

    public static function handleException(Throwable $exception): void
    {
        $isDebug = $_ENV['APP_DEBUG'] ?? false;

        $statusCode = 500;
        if (method_exists($exception, 'getStatusCode')) {
            $statusCode = $exception->getStatusCode();
        }

        $response = [
            'status' => 'error',
            'message' => $isDebug ? $exception->getMessage() : 'Error interno del servidor',
        ];

        if ($isDebug) {
            $response['file'] = $exception->getFile();
            $response['line'] = $exception->getLine();
            $response['trace'] = $exception->getTraceAsString();
        }

        // Registrar en el log
        self::logError($exception);

        http_response_code($statusCode);
        header('Content-Type: application/json');
        echo json_encode($response, JSON_PRETTY_PRINT);
        exit;
    }

    private static function logError(Throwable $exception): void
    {
        $logDir = __DIR__ . '/../../storage/logs';
        if (!is_dir($logDir)) {
            mkdir($logDir, 0775, true);
        }

        $logFile = $logDir . '/error_' . date('Y-m-d') . '.log';
        $message = sprintf(
            "[%s] %s en %s:%d\nStack trace:\n%s\n\n",
            date('Y-m-d H:i:s'),
            $exception->getMessage(),
            $exception->getFile(),
            $exception->getLine(),
            $exception->getTraceAsString()
        );

        file_put_contents($logFile, $message, FILE_APPEND);
    }
}
?>
```

7. MIDDLEWARE: CADENA DE RESPONSABILIDAD
===============================================================================

Los middleware son componentes que procesan las peticiones HTTP antes de que
lleguen al controlador. Implementan el patrón **Chain of Responsibility**
(Cadena de Responsabilidad), permitiendo añadir capas de procesamiento
de forma modular.

```php
<?php
// src/Middleware/AuthMiddleware.php
declare(strict_types=1);

namespace App\Middleware;

use App\Core\Request;
use App\Core\Response;

class AuthMiddleware
{
    public function handle(Request $request, callable $next): Response
    {
        $token = $request->getHeader('Authorization');

        if (!$token || !$this->validarToken($token)) {
            return new Response(
                json_encode(['status' => 'error', 'message' => 'No autorizado']),
                401,
                ['Content-Type' => 'application/json']
            );
        }

        $request->setAttribute('usuario_id', $this->decodificarToken($token));
        return $next($request);
    }

    private function validarToken(string $token): bool
    {
        // Verificar JWT o token de sesión
        $token = str_replace('Bearer ', '', $token);
        $secret = $_ENV['JWT_SECRET'] ?? 'default_secret';

        try {
            $payload = \Firebase\JWT\JWT::decode($token, new \Firebase\JWT\Key($secret, 'HS256'));
            return (bool)$payload;
        } catch (\Exception $e) {
            return false;
        }
    }

    private function decodificarToken(string $token): int
    {
        $token = str_replace('Bearer ', '', $token);
        $secret = $_ENV['JWT_SECRET'] ?? 'default_secret';
        $payload = \Firebase\JWT\JWT::decode($token, new \Firebase\JWT\Key($secret, 'HS256'));
        return (int)$payload->sub;
    }
}
?>
```

```php
<?php
// src/Middleware/CorsMiddleware.php
declare(strict_types=1);

namespace App\Middleware;

use App\Core\Request;
use App\Core\Response;

class CorsMiddleware
{
    private array $allowedOrigins;
    private array $allowedMethods;
    private array $allowedHeaders;

    public function __construct()
    {
        $config = require __DIR__ . '/../Config/app.php';
        $this->allowedOrigins = $config['cors']['allowed_origins'];
        $this->allowedMethods = $config['cors']['allowed_methods'];
        $this->allowedHeaders = $config['cors']['allowed_headers'];
    }

    public function handle(Request $request, callable $next): Response
    {
        $origin = $request->getHeader('Origin') ?? '*';

        if (in_array($origin, $this->allowedOrigins)) {
            header("Access-Control-Allow-Origin: {$origin}");
            header('Access-Control-Allow-Credentials: true');
        }

        header("Access-Control-Allow-Methods: " . implode(', ', $this->allowedMethods));
        header("Access-Control-Allow-Headers: " . implode(', ', $this->allowedHeaders));

        // Manejar preflight requests (OPTIONS)
        if ($request->getMethod() === 'OPTIONS') {
            return new Response('', 204);
        }

        return $next($request);
    }
}
?>
```

Composer es el gestor de dependencias de PHP. Además de instalar librerías

```json
{
    "name": "lucas-ega-academy/php-master-pro",

    "type": "project",

            "App\\": "src/"

        "psr-4": {
            "Tests\\": "tests/"
```

Para usar el autoloader de Composer, solo necesitas incluirlo en tu
front controller:

```php
<?php
require_once __DIR__ . '/../vendor/autoload.php';

// Ahora puedes usar cualquier clase de tu aplicación sin require explícitos
use App\Core\Router;
use App\Core\Database;

```

El estándar **PSR-4** (Autoloading) mapea el namespace `App\` al directorio
`src/`. La clase `App\Controllers\UsuarioController` debe estar en el archivo
`src/Controllers/UsuarioController.php`.

RESUMEN DEL CAPÍTULO
===============================================================================

- La arquitectura MVC separa la lógica en tres capas: Modelo, Vista y
  Controlador, facilitando el mantenimiento y la escalabilidad.
- La estructura de carpetas organiza el código en directorios con
  responsabilidades únicas: public/, src/, templates/, storage/, tests/.
- El Front Controller centraliza todas las peticiones en index.php usando
  reglas de reescritura de Apache o Nginx.
- El Router analiza la URL y ejecuta el controlador correspondiente,
  soportando rutas con parámetros dinámicos.
- El patrón Singleton asegura una única conexión a la base de datos,
  evitando la creación de múltiples conexiones innecesarias.
- Las variables de entorno (.env) almacenan credenciales y configuraciones
  sensibles fuera del código fuente.
- Los middleware procesan las peticiones en cadena antes de llegar al
  controlador, implementando autenticación, CORS y validaciones.
- Composer proporciona autoloading PSR-4 y gestión profesional de
  dependencias.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Es obligatorio usar el patrón MVC en PHP?**
No es obligatorio, pero es el estándar de la industria. Todos los frameworks
PHP modernos (Laravel, Symfony, CodeIgniter) implementan MVC. Te facilitará
la vida profesional.

❓ **¿Qué pasa si mi aplicación crece mucho?**
La arquitectura presentada escala verticalmente (mejor hardware) y
horizontalmente (más servidores). Para aplicaciones masivas, considera
microservicios con Laravel o Symfony.

❓ **¿Debo usar siempre Composer?**
En proyectos profesionales, sí. Composer no solo gestiona dependencias, sino
que proporciona autoloading profesional, scripts y control de versiones de
librerías.

❓ **¿Es seguro usar variables de entorno?**
Sí, siempre que el archivo .env esté fuera del directorio público y añadido
al .gitignore. Nunca subas .env a repositorios de Git.

❓ **¿Puedo usar Nginx en lugar de Apache?**
Sí. Nginx también soporta reglas de reescritura. La configuración cambia
ligeramente, pero el concepto de Front Controller es el mismo.
