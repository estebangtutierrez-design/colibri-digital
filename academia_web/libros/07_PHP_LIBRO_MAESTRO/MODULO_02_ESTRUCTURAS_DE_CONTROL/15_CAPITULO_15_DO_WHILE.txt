En el capítulo anterior aprendiste que while evalúa la condición antes de

Aquí entra do...while, el primo menos conocido de while. La diferencia es

validación de entrada donde primero pides el dato y luego verificas, o para
procesos donde la primera iteración debe ocurrir sí o sí.

1. INTRODUCCIÓN A DO...WHILE
===============================================================================

La estructura do...while es muy similar a while, con una diferencia crucial:
la condición se evalúa después de ejecutar el bloque de código, no antes.

Sintaxis:

```php
<?php
do {
    // Código que se ejecuta al menos una vez
} while (condición);
?>
```

Flujo de ejecución:
1. Se ejecuta el bloque de código.
2. Se evalúa la condición.
3. Si la condición es true, se repite desde el paso 1.
4. Si la condición es false, el bucle termina.

Esto garantiza que el bloque se ejecute siempre al menos una vez,
independientemente de si la condición es inicialmente true o false.

Ejemplo básico:

```php
<?php
$contador = 1;

do {
    echo "Iteración: $contador\n";
    $contador++;
} while ($contador <= 5);

echo "Bucle terminado.\n";
?>
```

Salida:
```
Iteración: 1
Iteración: 2
Iteración: 3
Iteración: 4
Iteración: 5
Bucle terminado.
```

Hasta aquí, parece igual que while. La diferencia se ve cuando la condición
es false desde el inicio:

```php
<?php
$contador = 10;

// While: nunca se ejecuta
while ($contador <= 5) {
    echo "While: $contador\n";
    $contador++;
}

// Do...while: se ejecuta al menos una vez
do {
    echo "Do...while: $contador\n";
    $contador++;
} while ($contador <= 5);
?>
```

Salida:
```
Do...while: 10
```

Mientras que while nunca ejecuta el bloque (la condición 10 <= 5 es false),
do...while lo ejecuta una vez y luego evalúa la condición.

Esta es la esencia de do...while: garantizar la primera ejecución.

===============================================================================
2. SINTAXIS DETALLADA
===============================================================================

La sintaxis de do...while tiene particularidades importantes.

Componentes:

```php
<?php
do:
    // Instrucciones
while (condición);
?>
```

Observaciones:
- do es una palabra reservada.
- El bloque va entre llaves { }.
- while va después del bloque, seguido de paréntesis con la condición.
- La estructura termina con punto y coma (;) después de while.

También existe una sintaxis alternativa con dos puntos:

```php
<?php
do:
    // Instrucciones
while (condición);
?>
```

Esta sintaxis alternativa es menos común pero válida.

El bloque puede contener cualquier código PHP:

```php
<?php
do {
    $dado = rand(1, 6);
    echo "Sacaste un $dado\n";
    
    if ($dado === 6) {
        echo "¡Sacaste un 6! ¡Ganaste!\n";
    }
} while ($dado !== 6);
?>
```

Puedes tener múltiples instrucciones dentro del bloque, incluyendo otras
estructuras de control:

```php
<?php
do {
    echo "Ingresa un número (0 para salir): ";
    $entrada = trim(fgets(STDIN));
    
    if (is_numeric($entrada)) {
        $numero = (float) $entrada;
        echo "Cuadrado: " . ($numero * $numero) . "\n";
        echo "Cubo: " . ($numero * $numero * $numero) . "\n";
    } elseif ($entrada !== "0") {
        echo "Error: No es un número válido.\n";
    }
} while ($entrada !== "0");

echo "Programa terminado.\n";
?>
```

La condición puede ser cualquier expresión que se evalúe a booleano,
incluyendo llamadas a funciones:

```php
<?php
do {
    $intento = rand(1, 10);
    echo "Intento: $intento\n";
} while ($intento !== 7);

echo "¡Encontramos el 7!\n";
?>
```

===============================================================================
3. DO...WHILE VS WHILE
===============================================================================

La elección entre while y do...while depende de si necesitas ejecución
garantizada al menos una vez.

Tabla comparativa:

| Aspecto              | while                          | do...while                     |
|----------------------|--------------------------------|--------------------------------|
| Evaluación condición | Antes del bloque               | Después del bloque             |
| Ejecución mínima     | 0 veces                        | 1 vez                          |
| Sintaxis             | while (cond) { }               | do { } while (cond);           |
| Terminación          | } sin punto y coma             | }; con punto y coma            |
| Ideal para           | Condiciones previas            | Acciones que deben ocurrir     |

Ejemplo: procesar un archivo

```php
<?php
// While: verificar que el archivo existe antes de procesar
$archivo = fopen("datos.txt", "r");
if ($archivo) {
    while (($linea = fgets($archivo)) !== false) {
        echo $linea;
    }
    fclose($archivo);
}

// Do...while: no tendría sentido aquí porque si el archivo no existe,
// la primera iteración fallaría.
?>
```

Ejemplo: menú interactivo

```php
<?php
// While: necesitas inicializar $opcion antes
$opcion = "";
while ($opcion !== "3") {
    echo "1. Opción 1\n";
    echo "2. Opción 2\n";
    echo "3. Salir\n";
    echo "Selecciona: ";
    $opcion = trim(fgets(STDIN));
}

// Do...while: no necesitas inicializar $opcion
do {
    echo "1. Opción 1\n";
    echo "2. Opción 2\n";
    echo "3. Salir\n";
    echo "Selecciona: ";
    $opcion = trim(fgets(STDIN));
} while ($opcion !== "3");
?>
```

En el caso del menú, do...while es más elegante porque no necesitas un
valor inicial artificial para $opcion.

Regla práctica:
- Usa while cuando **puede que no necesites** ejecutar el bloque.
- Usa do...while cuando **siempre necesitas** ejecutar el bloque al menos
  una vez.

Casos donde do...while es superior:

1. Menús interactivos (siempre se muestran al menos una vez).
2. Validación de entrada (primero pides, luego validas).
3. Juegos (al menos un turno).
4. Procesos de inicialización que deben ocurrir antes de verificar.

===============================================================================
4. DO...WHILE CON ENTRADA DE USUARIO
===============================================================================

Do...while es particularmente natural para validación de entrada porque
primero solicitas el dato y luego verificas si es válido.

Ejemplo: validar edad

```php
<?php
do {
    echo "Ingresa tu edad (0-120): ";
    $entrada = trim(fgets(STDIN));
    
    if (!is_numeric($entrada)) {
        echo "Error: Debes ingresar un número.\n";
        $edad = -1;
    } else {
        $edad = (int) $entrada;
        if ($edad < 0 || $edad > 120) {
            echo "Error: La edad debe estar entre 0 y 120.\n";
        }
    }
} while ($edad < 0 || $edad > 120);

echo "Edad registrada: $edad años.\n";
?>
```

Ejemplo: solicitar contraseña con confirmación

```php
<?php
do {
    echo "Crea una contraseña (mínimo 8 caracteres): ";
    $contrasena = trim(fgets(STDIN));
    
    if (strlen($contrasena) < 8) {
        echo "Error: La contraseña debe tener al menos 8 caracteres.\n";
        continue;
    }
    
    echo "Confirma tu contraseña: ";
    $confirmacion = trim(fgets(STDIN));
    
    if ($contrasena !== $confirmacion) {
        echo "Error: Las contraseñas no coinciden.\n";
    }
} while (strlen($contrasena) < 8 || $contrasena !== $confirmacion);

echo "¡Contraseña creada exitosamente!\n";
?>
```

Ejemplo: selección de opción válida

```php
<?php
do {
    echo "\nSelecciona un color:\n";
    echo "1. Rojo\n";
    echo "2. Verde\n";
    echo "3. Azul\n";
    echo "Opción: ";
    $opcion = trim(fgets(STDIN));
    
    if (!in_array($opcion, ["1", "2", "3"])) {
        echo "Opción no válida. Intenta de nuevo.\n";
    }
} while (!in_array($opcion, ["1", "2", "3"]));

$color = match ($opcion) {
    "1" => "Rojo",
    "2" => "Verde",
    "3" => "Azul",
};

echo "Has seleccionado: $color\n";
?>
```

Ejemplo: ingreso de múltiples valores hasta centinela

```php
<?php
$numeros = [];

echo "Ingresa números enteros (escribe 'fin' para terminar):\n";

do {
    echo "Número: ";
    $entrada = trim(fgets(STDIN));
    
    if ($entrada === "fin") {
        break;
    }
    
    if (!is_numeric($entrada)) {
        echo "Error: Ingresa un número válido o 'fin' para terminar.\n";
        continue;
    }
    
    $numeros[] = (int) $entrada;
} while (true);

echo "\nNúmeros ingresados: " . implode(", ", $numeros) . "\n";

if (!empty($numeros)) {
    $suma = array_sum($numeros);
    $promedio = $suma / count($numeros);
    echo "Suma: $suma\n";
    echo "Promedio: " . round($promedio, 2) . "\n";
    echo "Mínimo: " . min($numeros) . "\n";
    echo "Máximo: " . max($numeros) . "\n";
}
?>
```

La combinación do...while con entrada de usuario es natural y elegante.

5. DO...WHILE CON SWITCH Y MATCH

Do...while se combina excelentemente con switch/match para crear menús
interactivos.

Menú con switch:

```php
<?php
do {
```

Menú con match:

```php
<?php
do {
    echo "\n=== CALCULADORA AVANZADA ===\n";
    echo "1. Sumar\n";
    echo "2. Restar\n";
    echo "3. Multiplicar\n";
    echo "4. Dividir\n";
    echo "5. Potencia\n";
    echo "6. Salir\n";
    echo "Selecciona: ";
    $opcion = trim(fgets(STDIN));
    
    if ($opcion !== "6") {
        echo "Ingresa el primer número: ";
        $a = (float) trim(fgets(STDIN));
        echo "Ingresa el segundo número: ";
        $b = (float) trim(fgets(STDIN));
    }
    
    $resultado = match ($opcion) {
        "1" => "$a + $b = " . ($a + $b),
        "2" => "$a - $b = " . ($a - $b),

        "5" => "$a ^ $b = " . ($a ** $b),
        "6" => "Saliendo...",
        default => "Opción no válida",

    if ($opcion !== "6") {
        echo "\nResultado: $resultado\n";
    }
} while ($opcion !== "6");

echo "¡Hasta la próxima!\n";
?>
```

===============================================================================
6. DO...WHILE CON BREAK Y CONTINUE
===============================================================================

Break y continue funcionan en do...while igual que en while.

```php
<?php
$intentos = 0;
do {
    $intento = rand(1, 20);
    $intentos++;
    
    if ($intento % 5 === 0) {
        echo "($intentos) Múltiplo de 5: $intento - saltando...\n";
        continue;
    }
    
    echo "($intentos) Intento: $intento\n";
    
    if ($intento === 13) {
        echo "($intentos) ¡Encontramos el 13! Deteniendo.\n";
        break;
    }
    
} while ($intentos < 100);

echo "Total de intentos: $intentos\n";
?>
```

Break con niveles en do...while anidados:

```php
<?php
$contador = 0;
do {
    $contador++;
    $subcontador = 0;
    
    do {
        $subcontador++;
        echo "($contador, $subcontador) ";
        
        if ($contador >= 3 && $subcontador >= 3) {
            echo "¡Break 2!\n";
            break 2;
        }
        
        if ($subcontador >= 5) {
            echo "Break interno\n";
            break;
        }
    } while ($subcontador < 10);
    
    echo "Fin iteración $contador\n";
} while ($contador < 10);

echo "Bucle principal terminado en iteración $contador\n";
?>
```

===============================================================================
7. DO...WHILE VS OTROS BUCLES
===============================================================================

¿Cuándo usar do...while frente a las alternativas?

Do...while gana cuando:
- La primera iteración siempre debe ocurrir.
- La condición de salida se basa en el resultado de la primera ejecución.
- El código y la condición están lógicamente vinculados (primero haz, luego
  verifica).

While gana cuando:
- Puede que el bloque no deba ejecutarse nunca.
- La condición de salida es conocida antes de la primera iteración.
- Estás procesando datos que pueden estar vacíos.

For gana cuando:
- Sabes exactamente cuántas iteraciones necesitas.
- Tienes un contador claro con inicio y fin.

Foreach gana cuando:
- Iteras sobre todos los elementos de un array/colección.
- No te importa el índice explícito.

Ejemplo comparativo: procesar datos hasta que no haya más

```php
<?php
$datos = [1, 2, 3, 4, 5];

// Foreach: cuando sabes que hay datos y quieres todos
foreach ($datos as $dato) {
    echo "$dato ";
}

// While: cuando los datos pueden estar vacíos
$indice = 0;
while ($indice < count($datos)) {
    echo $datos[$indice] . " ";
    $indice++;
}

// Do...while: cuando necesitas procesar aunque sea vacío
$indice = 0;
do {
    if (isset($datos[$indice])) {
        echo $datos[$indice] . " ";
    }
    $indice++;
} while ($indice < count($datos));
?>
```

En la práctica, do...while rara vez es la mejor opción para arrays. Su
fuerte son los menús, juegos, procesos interactivos.

===============================================================================
8. ERRORES COMUNES Y BUENAS PRÁCTICAS
===============================================================================

Errores comunes con do...while:

1. Confundir la sintaxis (olvidar el punto y coma final):

```php
<?php
do {
    echo "Hola\n";
} while (true) // Error: falta punto y coma

// Correcto:
do {
    echo "Hola\n";
} while (true);
?>
```

2. Crear un bucle infinito sin intención:

```php
<?php
do {
    echo "Esto nunca terminará.\n";
} while (true); // while (true) sin break
?>
```

3. Condición que nunca se cumple (no afecta la primera iteración, pero
   las subsiguientes nunca ocurren):

```php
<?php
$i = 10;
do {
    echo "$i\n";
    $i++;
} while ($i < 5);
// Solo imprime 10 (una iteración)
?>
```

Esto no es un error per se, pero puede ser inesperado.

4. Usar do...while cuando while es más apropiado:

```php
<?php
// Mal uso: do...while cuando podría no haber datos
$archivo = fopen("datos.txt", "r");
do {
    $linea = fgets($archivo);
    if ($linea !== false) {
        echo $linea;
    }
} while ($linea !== false);
fclose($archivo);
// Esto imprimiría false como última línea

// Mejor: while porque el archivo podría estar vacío
while (($linea = fgets($archivo)) !== false) {
    echo $linea;
}
?>
```

Buenas prácticas:

1. Usa do...while cuando la primera iteración deba ocurrir siempre.
2. Limita el número de iteraciones con un contador de seguridad.
3. Para menús, do...while elimina la necesidad de inicializar la variable.
4. No uses do...while para iterar sobre arrays vacíos (usa foreach).
5. Siempre incluye una forma de salir del bucle (condición variable o break).
6. La condición debe ser simple y legible.
7. Para validación de entrada, do...while es la opción más natural.

Ejemplo 1: Simulador de dado hasta que salga 6

```php
<?php
$totalIntentos = 0;
$sumaTotal = 0;

do {
    $dado1 = rand(1, 6);
    $dado2 = rand(1, 6);
    $suma = $dado1 + $dado2;
    $totalIntentos++;
    $sumaTotal += $suma;
    
    echo "Intento #$totalIntentos: Dado 1 = $dado1, Dado 2 = $dado2, Suma = $suma\n";
    
    if ($dado1 === 6 && $dado2 === 6) {
        echo "¡Sacaste doble 6!\n";
        break;
    }
} while ($dado1 !== 6 && $dado2 !== 6);

echo "\n--- ESTADÍSTICAS ---\n";
echo "Intentos totales: $totalIntentos\n";
echo "Suma acumulada: $sumaTotal\n";
echo "Promedio por tirada: " . round($sumaTotal / $totalIntentos, 2) . "\n";
?>
```

Ejemplo 2: Cajero automático

```php
<?php
$saldo = 1000.00;

do {
    echo "\n" . str_repeat("=", 35) . "\n";
    echo "      CAJERO AUTOMÁTICO\n";
    echo str_repeat("=", 35) . "\n";
    echo "Saldo actual: \$" . number_format($saldo, 2) . "\n";
    echo str_repeat("-", 35) . "\n";
    echo "1. Depositar\n";
    echo "2. Retirar\n";
    echo "3. Consultar saldo\n";
    echo "4. Salir\n";
    echo str_repeat("=", 35) . "\n";
    echo "Opción: ";
    $opcion = trim(fgets(STDIN));
    
    switch ($opcion) {
        case "1":
            echo "Monto a depositar: \$";
            $monto = (float) trim(fgets(STDIN));
            if ($monto > 0) {
                $saldo += $monto;
                echo "Depósito exitoso. Nuevo saldo: \$" . number_format($saldo, 2) . "\n";
            } else {
                echo "Monto inválido.\n";
            }
            break;
            
        case "2":
            echo "Monto a retirar: \$";
            $monto = (float) trim(fgets(STDIN));

                echo "Monto inválido.\n";

                echo "Saldo insuficiente.\n";
            } elseif ($monto % 100 !== 0.0 && $monto % 100 !== 0) {
                echo "Solo puedes retirar múltiplos de \$100.\n";
            } else {
                $saldo -= $monto;
                echo "Retiro exitoso. Nuevo saldo: \$" . number_format($saldo, 2) . "\n";
            }
            break;
            
        case "3":
            echo "Saldo actual: \$" . number_format($saldo, 2) . "\n";
            break;
            
        case "4":
            echo "Gracias por usar nuestro cajero.\n";
            break;
            
        default:
            echo "Opción no válida.\n";
    }
} while ($opcion !== "4");
?>
```

Ejemplo 3: Juego de adivinanza con do...while

```php
<?php
do {
    $numeroSecreto = rand(1, 50);
    $intentos = 0;
    $adivinado = false;
    
    echo "\n=== NUEVA PARTIDA ===\n";
    echo "Adivina el número secreto (1-50).\n";
    
    do {
        $intentos++;
        echo "Intento #$intentos: ";
        $entrada = trim(fgets(STDIN));
        
        if (!is_numeric($entrada)) {
            echo "Ingresa un número válido.\n";
            $intentos--; // No contar intentos inválidos
            continue;
        }
        
        $intento = (int) $entrada;
        
        $mensaje = match (true) {
            $intento < 1 || $intento > 50 => "Fuera de rango (1-50).",
            $intento === $numeroSecreto => "¡Adivinaste en $intentos intentos!",
            $intento < $numeroSecreto => "El número es mayor.",
            default => "El número es menor.",
        };
        
        echo "$mensaje\n";
        
        if ($intento === $numeroSecreto) {
            $adivinado = true;
        }
    } while (!$adivinado);
    
    echo "\n¿Quieres jugar otra vez? (s/n): ";
    $otraVez = strtolower(trim(fgets(STDIN)));
} while ($otraVez === "s");

echo "¡Gracias por jugar!\n";
?>
```

Ejemplo 4: Procesador de transacciones bancarias

```php
<?php
$transacciones = [
    ["tipo" => "deposito", "monto" => 500],
    ["tipo" => "retiro", "monto" => 200],
    ["tipo" => "deposito", "monto" => 1000],
    ["tipo" => "retiro", "monto" => 150],
    ["tipo" => "retiro", "monto" => 700],
    ["tipo" => "deposito", "monto" => 300],
];

$saldo = 1000;
$indice = 0;
$transaccionesProcesadas = 0;

do {
    $tx = $transacciones[$indice];
    $monto = $tx["monto"];
    
    $resultado = match ($tx["tipo"]) {
        "deposito" => (function() use ($monto, &$saldo) {
            $saldo += $monto;
            return "Depósito de +\$$monto. Nuevo saldo: \$$saldo";
        })(),
        
        "retiro" => (function() use ($monto, &$saldo) {
            if ($monto > $saldo) {
                return "RETIRO FALLIDO: Saldo insuficiente para \$$monto";
            }
            $saldo -= $monto;
            return "Retiro de -\$$monto. Nuevo saldo: \$$saldo";
        })(),
        
        default => "Transacción desconocida",
    };
    
    echo "Tx #" . ($indice + 1) . ": $resultado\n";
    $transaccionesProcesadas++;
    $indice++;
} while ($indice < count($transacciones));

echo "\n--- RESUMEN ---\n";
echo "Transacciones procesadas: $transaccionesProcesadas\n";
echo "Saldo final: \$$saldo\n";
?>
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- do...while garantiza que el bloque se ejecute al menos una vez.
- La condición se evalúa después del bloque, no antes.
- La sintaxis termina con punto y coma después de while(condición).
- Es ideal para menús interactivos y validación de entrada.
- Mientras que while puede ejecutarse 0 veces, do...while siempre ejecuta 1.
- break y continue funcionan igual que en while.
- Combínalo con switch/match para menús completos.
- No es la mejor opción para iterar sobre arrays vacíos.
- Úsalo cuando la primera iteración deba ocurrir independientemente.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué significa el punto y coma al final de do...while?** Es parte de la
sintaxis. while (condición); marca el final del bucle. A diferencia de while
simple, do...while requiere este punto y coma.

❓ **¿Puedo tener un do...while sin llaves?** Sí, si el bloque tiene una sola
instrucción: do echo "hola\n"; while (false); Pero se recomienda usar llaves
para claridad.

❓ **¿Cuándo es mejor do...while que while?** Cuando necesitas que el bloque
se ejecute al menos una vez sin importar la condición. Menús interactivos,
validación de entrada, y procesos de inicialización son casos típicos.

❓ **¿Puedo usar do...while con for anidado?** Sí, cualquier combinación de
estructuras de control es posible. Puedes tener do...while dentro de for,
for dentro de do...while, etc.

❓ **¿El rendimiento de do...while vs while es diferente?** La diferencia es
insignificante. La elección debe basarse en la semántica (cuándo debe
ejecutarse el bloque), no en el rendimiento.

❓ **¿Puedo usar continue en do...while?** Sí, continue funciona igual que
en while: salta al final del bloque y evalúa la condición.

❓ **¿Qué es un bucle infinito en do...while?** Un bucle donde la condición
siempre es true o nunca cambia. Ejemplo: do { ... } while (true); sin break.
