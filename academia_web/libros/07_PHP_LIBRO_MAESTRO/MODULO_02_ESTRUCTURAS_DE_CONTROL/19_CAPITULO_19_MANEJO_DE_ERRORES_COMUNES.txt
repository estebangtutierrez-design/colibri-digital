===============================================================================
1. INTRODUCCIÓN A LOS ERRORES EN PHP
===============================================================================

PHP tiene varios tipos de errores, cada uno con su propósito y gravedad.

Tipos de errores en PHP:

- Error: Problemas graves como sintaxis incorrecta o errores fatales.
- Exception: Problemas recuperables que pueden capturarse con try/catch.
- Warning: Problemas no fatales (archivo no encontrado, variable no definida).
- Notice: Problemas menores (usar variable no definida).
- Deprecated: Código que funcionará en versiones futuras.

Niveles de error comunes:

```php
<?php
E_ERROR      // Error fatal (1)
E_WARNING    // Advertencia (2)
E_PARSE      // Error de sintaxis (4)
E_NOTICE     // Aviso (8)
E_DEPRECATED // Código obsoleto (8192)
E_ALL        // Todos los errores (32767)
?>
```

PHP 7+ introdujo el motor de errores basado en excepciones para errores
fatales:

```php
<?php
try {
    // Esto lanza un TypeError en PHP 7+
    function sumar(int $a, int $b): int {
        return $a + $b;
    }
    echo sumar("hola", 2);
} catch (TypeError $e) {
    echo "Error de tipo: " . $e->getMessage();
}
?>
```

===============================================================================
2. TRY/CATCH: CAPTURANDO EXCEPCIONES
===============================================================================

El bloque try/catch permite capturar y manejar excepciones de forma
controlada.

Sintaxis:

```php
<?php
try {
    // Código que puede lanzar una excepción
} catch (TipoDeExcepcion $e) {
    // Código que maneja la excepción
} catch (OtroTipo $e) {
    // Múltiples bloques catch
}
?>
```

Ejemplo básico:

```php
<?php
function dividir(int $a, int $b): float {
    if ($b === 0) {
        throw new DivisionByZeroError("No se puede dividir por cero.");
    }
    return $a / $b;
}

try {
    echo dividir(10, 0);
} catch (DivisionByZeroError $e) {
    echo "Error: " . $e->getMessage() . "\n";
    echo "En archivo: " . $e->getFile() . " línea " . $e->getLine() . "\n";
}

echo "El programa continúa...\n";
?>
```

Múltiples catch:

```php
<?php
function procesarEdad(int $edad): string {
    if ($edad < 0 || $edad > 150) {
        throw new OutOfRangeException("Edad fuera de rango: $edad");
    }
    if (!is_int($edad)) {
        throw new InvalidArgumentException("La edad debe ser un entero.");
    }
    return match (true) {
        $edad < 18 => "Menor de edad",
        $edad < 65 => "Adulto",
        default => "Adulto mayor",
    };
}

try {
    echo procesarEdad(-5);
} catch (OutOfRangeException $e) {
    echo "Rango inválido: " . $e->getMessage() . "\n";
} catch (InvalidArgumentException $e) {
    echo "Argumento inválido: " . $e->getMessage() . "\n";
} catch (Throwable $e) {
    // Captura cualquier otra excepción/error
    echo "Error inesperado: " . $e->getMessage() . "\n";
}
?>
```

El orden de los catch importa: las excepciones más específicas primero.

===============================================================================
3. FINALLY: EJECUCIÓN GARANTIZADA
===============================================================================

El bloque finally se ejecuta siempre, haya o no excepción. Es ideal para
limpiar recursos.

```php
<?php
function leerArchivo(string $ruta): string {
    $archivo = null;
    try {
        $archivo = fopen($ruta, "r");
        if (!$archivo) {
            throw new RuntimeException("No se pudo abrir: $ruta");
        }
        $contenido = fread($archivo, filesize($ruta));
        return $contenido;
    } catch (RuntimeException $e) {
        echo "Error: " . $e->getMessage() . "\n";
        throw $e; // Relanzar
    } finally {
        if ($archivo) {
            fclose($archivo);
            echo "Archivo cerrado.\n";
        }
    }
}

try {
    echo leerArchivo("inexistente.txt");
} catch (RuntimeException $e) {
    echo "Excepción capturada en el nivel superior.\n";
}
?>
```

Finally se ejecuta incluso con return en try/catch:

```php
<?php
function probarFinally(): string {
    try {
        echo "En try.\n";
        return "Valor de retorno.\n";
    } finally {
        echo "Finally se ejecuta incluso después del return.\n";
    }
}

echo probarFinally();
?>
```

Salida:
```
En try.
Finally se ejecuta incluso después del return.
Valor de retorno.
```

Finally es perfecto para cerrar conexiones de base de datos, archivos,
liberar memoria, etc.

4. EXCEPCIONES PERSONALIZADAS
===============================================================================

Puedes crear tus propias clases de excepción extendiendo Exception.

```php
<?php
class ErrorValidacion extends Exception {
    protected array $errores = [];

    public function __construct(string $mensaje = "", int $codigo = 0,
                                ?Throwable $previous = null,
                                array $errores = []) {
        parent::__construct($mensaje, $codigo, $previous);
        $this->errores = $errores;
    }

    public function getErrores(): array {
        return $this->errores;
    }
}

class ErrorBaseDatos extends Exception {
    private string $consulta;

    public function __construct(string $mensaje, string $consulta = "",
                                int $codigo = 0, ?Throwable $previous = null) {
        parent::__construct($mensaje, $codigo, $previous);
        $this->consulta = $consulta;
    }

    public function getConsulta(): string {
        return $this->consulta;
    }
}

class ErrorAutenticacion extends Exception {}
class ErrorRecursoNoEncontrado extends Exception {}
?>

Uso:

```php
<?php
require_once "excepciones.php";

function registrarUsuario(string $nombre, string $email, int $edad): void {
    $errores = [];

    if (strlen($nombre) < 3) {
        $errores[] = "Nombre debe tener al menos 3 caracteres.";
    }
    if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errores[] = "Email inválido.";
    }
    if ($edad < 18) {
        $errores[] = "Debes ser mayor de edad.";
    }

    if (!empty($errores)) {
        throw new ErrorValidacion("Error al registrar usuario.", 400, null, $errores);
    }

    echo "Usuario $nombre registrado.\n";
}

try {
    registrarUsuario("An", "email-invalido", 15);
} catch (ErrorValidacion $e) {
    echo "Error: " . $e->getMessage() . "\n";
    echo "Detalles:\n";
    foreach ($e->getErrores() as $error) {
        echo "  - $error\n";
    }
    echo "Código: " . $e->getCode() . "\n";
}
?>
```

Jerarquía de excepciones personalizadas:

```php
<?php
class ErrorAplicacion extends Exception {}
class ErrorPersistencia extends ErrorAplicacion {}
class ErrorBaseDatos extends ErrorPersistencia {}
class ErrorArchivo extends ErrorPersistencia {}
class ErrorLogica extends ErrorAplicacion {}
class ErrorNegocio extends ErrorLogica {}
class ErrorValidacion extends ErrorLogica {}
?>
```

Esta jerarquía permite capturar errores con diferentes niveles de
especificidad.

===============================================================================
5. THROW: LANZANDO EXCEPCIONES
===============================================================================

Cualquier código puede lanzar una excepción con throw.

```php
<?php
throw new Exception("Mensaje de error");
throw new InvalidArgumentException("Argumento inválido");
throw new RuntimeException("Error en tiempo de ejecución");
?>
```

Ejemplo: validación con lanzamiento temprano:

```php
<?php
function calcularRaiz(float $numero): float {
    if ($numero < 0) {
        throw new InvalidArgumentException(
            "No se puede calcular raíz cuadrada de un número negativo: $numero"
        );
    }
    return sqrt($numero);
}

try {
    echo calcularRaiz(-4);
} catch (InvalidArgumentException $e) {
    echo "Error: " . $e->getMessage() . "\n";
}
?>
```

Relanzar excepciones:

```php
<?php
function procesarPago(float $monto): void {
    try {
        if ($monto <= 0) {
            throw new InvalidArgumentException("Monto inválido: $monto");
        }
        if ($monto > 10000) {
            throw new RuntimeException("Monto excede el límite permitido.");
        }
        echo "Pago de \$$monto procesado.\n";
    } catch (InvalidArgumentException $e) {
        // Loggear el error
        error_log("Error de validación: " . $e->getMessage());
        throw $e; // Relanzar para que lo maneje el nivel superior
    }
}

try {
    procesarPago(-50);
} catch (InvalidArgumentException $e) {
    echo "No se pudo procesar: " . $e->getMessage() . "\n";
}
?>
```

Throw puede lanzar cualquier objeto que implemente Throwable:

```php
<?php
try {
    $condicion = true;
    if ($condicion) {
        throw new \RuntimeException("Error controlado");
    }
} catch (\Throwable $e) {
    echo "Throwable capturado: " . $e->getMessage() . "\n";
}
?>
```

Throwable es la interfaz base que implementan Exception y Error.

6. MANEJO DE ERRORES CON SET_ERROR_HANDLER
===============================================================================

Puedes personalizar el manejador de errores de PHP con set_error_handler.
Esto captura warnings, notices y otros errores no fatales.

```php
<?php
function manejadorPersonalizado(
    int $nivel, string $mensaje, string $archivo, int $linea
): bool {
    $niveles = [
        E_WARNING => "Warning",
        E_NOTICE => "Notice",
        E_DEPRECATED => "Deprecated",
        E_USER_ERROR => "User Error",
        E_USER_WARNING => "User Warning",
    ];

    $tipo = $niveles[$nivel] ?? "Desconocido";
    $fecha = date("Y-m-d H:i:s");

    $log = "[$fecha] $tipo: $mensaje en $archivo:$linea\n";
    error_log($log, 3, "/tmp/errores.log");
    echo "Error registrado: $tipo\n";

    return true; // Evitar que PHP maneje el error
}

set_error_handler("manejadorPersonalizado");

trigger_error("Esto es un warning", E_USER_WARNING);
trigger_error("Esto es un error de usuario", E_USER_ERROR);

echo $variableNoDefinida; // Notice capturado
?>
```

Restaurar el manejador original:

```php
<?php
restore_error_handler(); // Vuelve al manejador por defecto
?>
```

===============================================================================
7. ERRORES COMUNES EN ESTRUCTURAS DE CONTROL
===============================================================================

Repasemos los errores más frecuentes en las estructuras que aprendiste.

Error 1: Off-by-one en bucles for

```php
<?php
$numeros = [1, 2, 3, 4, 5];

// ERROR: <= count da índice fuera de rango
for ($i = 0; $i <= count($numeros); $i++) {
    echo $numeros[$i] . " "; // Notice: undefined offset 5
}

// CORRECTO: < count
for ($i = 0; $i < count($numeros); $i++) {
    echo $numeros[$i] . " ";
}
?>
```

Error 2: Olvidar break en switch

```php
<?php
// ERROR: fall-through no deseado
switch ($opcion) {
    case 1:
        echo "Opción 1\n";
        // Falta break aquí
    case 2:
        echo "Opción 2\n"; // También se ejecuta si opcion es 1
        break;
}
?>
```

Error 3: Bucle infinito en while

```php
<?php
// ERROR: condición nunca cambia
$i = 1;
while ($i <= 10) {
    echo "$i ";
    // Olvidaste $i++
}
?>
```

Error 4: Condición siempre verdadera

```php
<?php
// ERROR: = en lugar de ==
$edad = 18;
if ($edad = 20) { // Asignación, no comparación
    echo "Siempre se ejecuta.\n";
}
?>
```

Error 5: Modificar array durante foreach

```php
<?php
// ERROR: comportamiento impredecible
$items = [1, 2, 3, 4, 5];
foreach ($items as $clave => $valor) {
    if ($valor === 3) {
        unset($items[$clave]); // Eliminar durante iteración
    }
}

// CORRECTO: recolectar y eliminar después
$eliminar = [];
foreach ($items as $clave => $valor) {
    if ($valor === 3) $eliminar[] = $clave;
}
foreach ($eliminar as $clave) {
    unset($items[$clave]);
}
?>
```

Error 6: Continue mal ubicado en while

```php
<?php
// ERROR: bucle infinito
$i = 0;
while ($i < 10) {
    if ($i % 2 === 0) {
        continue; // $i nunca se incrementa
    }
    echo "$i ";
    $i++;
}

// CORRECTO: incrementar antes de continue
$i = 0;
while ($i < 10) {
    $i++;
    if ($i % 2 === 0) continue;
    echo "$i ";
}
?>
```

Error 7: No capturar excepciones específicas

```php
<?php
// ERROR: catch demasiado genérico
try {
    // código
} catch (\Exception $e) {
    // Captura todo, no diferencia
}

// MEJOR: capturar específicos
try {
    // código
} catch (\InvalidArgumentException $e) {
    // Manejar específicamente
} catch (\RuntimeException $e) {
    // Manejar otro tipo
} catch (\Throwable $e) {
    // Capturar cualquier otro error
}
?>
```

Error 8: No usar finally para limpiar recursos

```php
<?php
// ERROR: recurso no liberado si hay excepción
function leer(string $ruta): string {
    $f = fopen($ruta, "r");
    if (condicion()) {
        throw new \RuntimeException("Error");
        // $f nunca se cierra
    }
    fclose($f);
    return "ok";
}

// CORRECTO: finally asegura el cierre
function leerSeguro(string $ruta): string {
    $f = fopen($ruta, "r");
    try {
        if (condicion()) {
            throw new \RuntimeException("Error");
        }
        return "ok";
    } finally {
        fclose($f);
    }
}
?>
```

8. ESTRATEGIAS DE DEPURACIÓN
===============================================================================

Técnicas de depuración para encontrar errores.

1. var_dump y print_r:

```php
<?php
$datos = ["nombre" => "Ana", "edad" => 28];
var_dump($datos); // Muestra tipo y valor
print_r($datos); // Formato legible para humanos

// var_dump con formato (PHP 8+)
var_dump($datos, true); // Devuelve string en lugar de imprimir
?>
```

2. error_log:

```php
<?php
function procesar(array $datos): void {
    error_log("Iniciando procesamiento con " . count($datos) . " elementos.");

    foreach ($datos as $clave => $valor) {
        if (is_array($valor)) {
            error_log("Procesando elemento $clave (array anidado)");
            // Depuración más detallada
            error_log(print_r($valor, true));
        }
    }

    error_log("Procesamiento completado.");
}

procesar(["a" => 1, "b" => [2, 3]]);
?>
```

3. debug_backtrace y debug_print_backtrace:

```php
<?php
function funcionA() { funcionB(); }
function funcionB() { funcionC(); }
function funcionC() {
    echo "Trace:\n";
    debug_print_backtrace();
    echo "\n\n";

    $trace = debug_backtrace();
    foreach ($trace as $nivel => $info) {
        echo "#$nivel: {$info['function']}()";
        if (isset($info['file'])) {
            echo " en {$info['file']}:{$info['line']}";
        }
        echo "\n";
    }
}

funcionA();
?>
```

4. Aserción con assert:

```php
<?php
function dividir(int $a, int $b): float {
    assert($b !== 0, "Divisor no puede ser cero.");
    return $a / $b;
}

// Las aserciones se pueden desactivar en producción
ini_set('zend.assertions', 1);
assert_options(ASSERT_ACTIVE, true);
?>
```

5. Uso de breakpoints mentales con echo/die:

```php
<?php
$valor = calcularComplejo();
echo "DEBUG: valor antes del if es " . var_export($valor, true) . "\n";

if ($valor > 100) {
    echo "DEBUG: entrando al if\n";
    // ...
}
// die("DEBUG: detenido antes del foreach");
// exit;
?>
```

===============================================================================
9. EXCEPCIONES VS ERRORES
===============================================================================

¿Cuándo usar excepciones y cuándo usar errores?

Excepciones:
- Son recuperables (puedes capturarlas con try/catch).
- Son parte del flujo normal del programa.
- Deben usarse para condiciones excepcionales esperadas.
- Permiten propagar información detallada.

Errores:
- Son no recuperables (errores fatales, de parse).
- Representan bugs o condiciones que no deberían ocurrir.
- Deben usarse para problemas graves de infraestructura.
- En PHP 7+, los errores fatales lanzan \Error.

```php
<?php
// Excepción: problema recuperable esperado
try {
    $archivo = fopen("datos.txt", "r");
    if (!$archivo) {
        throw new RuntimeException("No se pudo abrir el archivo.");
    }
} catch (RuntimeException $e) {
    echo "Usando archivo de respaldo.\n";
    $archivo = fopen("respaldo.txt", "r");
}

// Error: problema grave inesperado
// Esto lanza TypeError si los argumentos son incorrectos
function sumar(int $a, int $b): int {
    return $a + $b;
}
// sumar("no", "strings"); // TypeError
?>
```

Jerarquía Throwable en PHP 7+:

```
Throwable (interface)
  ├── Exception
  │   ├── LogicException
  │   │   ├── InvalidArgumentException
  │   │   ├── LengthException
  │   │   └── OutOfRangeException
  │   └── RuntimeException
  │       ├── UnexpectedValueException
  │       ├── OutOfBoundsException
  │       ├── OverflowException
  │       └── UnderflowException
  └── Error
      ├── TypeError
      ├── ParseError
      ├── ArithmeticError
      └── DivisionByZeroError
```

Ejemplo: capturar tanto Exception como Error con Throwable:

```php
<?php
try {
    // Puede lanzar Exception o Error
    $resultado = 10 / 0; // DivisionByZeroError
} catch (\Throwable $e) {
    echo get_class($e) . ": " . $e->getMessage() . "\n";
}
?>
```

Ejemplo 1: Conexión a base de datos simulada

```php
<?php
class ConexionBD {
    private bool $conectado = false;

    public function conectar(string $host, string $user, string $pass): void {
        if (empty($host) || empty($user)) {
            throw new InvalidArgumentException("Host y usuario son obligatorios.");
        }

        if (rand(0, 3) === 0) { // Simular fallo aleatorio
            throw new RuntimeException("No se pudo conectar a $host.");
        }

        $this->conectado = true;
        echo "Conectado a $host como $user.\n";
    }

    public function consultar(string $sql): array {
        if (!$this->conectado) {
            throw new ErrorBaseDatos("No hay conexión activa.", $sql);
        }

        if (stripos($sql, "DROP") !== false) {
            throw new ErrorBaseDatos("Operación no permitida: DROP.", $sql, 403);
        }

        // Simular resultados
        return [["id" => 1, "nombre" => "Resultado simulado"]];
    }

    public function desconectar(): void {
        $this->conectado = false;
        echo "Desconectado.\n";
    }
}

$bd = new ConexionBD();

try {
    $bd->conectar("localhost", "root", "");
    $resultados = $bd->consultar("SELECT * FROM usuarios");
    print_r($resultados);
    $bd->consultar("DROP TABLE usuarios"); // Esto lanza excepción
} catch (InvalidArgumentException $e) {
    echo "Error de configuración: " . $e->getMessage() . "\n";
} catch (ErrorBaseDatos $e) {
    echo "Error BD [{$e->getCode()}]: " . $e->getMessage() . "\n";
    echo "Consulta: {$e->getConsulta()}\n";
} catch (RuntimeException $e) {
    echo "Error de conexión: " . $e->getMessage() . "\n";
    echo "Usando modo offline.\n";
} finally {
    $bd->desconectar();
}
?>
```

Ejemplo 2: Validador de formularios con excepciones

```php
<?php
class ErrorFormulario extends Exception {
    private array $errores;
    public function __construct(array $errores) {
        $this->errores = $errores;
        parent::__construct("Errores de validación: " . count($errores));
    }
    public function getErrores(): array { return $this->errores; }
}

function validarFormulario(array $datos): array {
    $errores = [];

    $nombre = trim($datos["nombre"] ?? "");
    if (empty($nombre)) {
        $errores["nombre"] = "El nombre es obligatorio.";
    } elseif (strlen($nombre) < 3) {
        $errores["nombre"] = "Mínimo 3 caracteres.";
    }

    $email = trim($datos["email"] ?? "");
    if (empty($email)) {
        $errores["email"] = "El email es obligatorio.";
    } elseif (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errores["email"] = "Formato de email inválido.";
    }

    $edad = (int)($datos["edad"] ?? 0);
    if ($edad < 18 || $edad > 120) {
        $errores["edad"] = "Edad debe estar entre 18 y 120.";
    }

    $password = $datos["password"] ?? "";
    if (strlen($password) < 8) {
        $errores["password"] = "Mínimo 8 caracteres.";
    } elseif (!preg_match('/[A-Z]/', $password)) {
        $errores["password"] = "Debe contener mayúscula.";
    } elseif (!preg_match('/[0-9]/', $password)) {
        $errores["password"] = "Debe contener número.";
    }

    if (!empty($errores)) {
        throw new ErrorFormulario($errores);
    }

    return [
        "nombre" => $nombre,
        "email" => $email,
        "edad" => $edad,
        "password_hash" => password_hash($password, PASSWORD_DEFAULT),
    ];
}

$datos = [
    "nombre" => "An",
    "email" => "invalido",
    "edad" => 15,
    "password" => "123",
];

try {
    $resultado = validarFormulario($datos);
    echo "Formulario válido. Datos procesados.\n";
} catch (ErrorFormulario $e) {
    echo "Errores de validación:\n";
    foreach ($e->getErrores() as $campo => $error) {
        echo "  - $campo: $error\n";
    }

Ejemplo 3: Procesamiento de archivos con garantía de cierre

```php
<?php
function procesarArchivo(string $ruta): array {
    $archivo = null;
    $lineas = [];

    try {
        $archivo = fopen($ruta, "r");
        if (!$archivo) {
            throw new RuntimeException("No se pudo abrir: $ruta");
        }

        $numLinea = 0;
        while (($linea = fgets($archivo)) !== false) {
            $numLinea++;
            $linea = trim($linea);

            if (empty($linea)) continue;

            $campos = str_getcsv($linea);
            if (count($campos) < 3) {
                throw new RuntimeException(
                    "Línea $numLinea: formato inválido (se esperan 3 campos)."
                );
            }

            $lineas[] = [
                "nombre" => $campos[0],
                "email" => filter_var($campos[1], FILTER_VALIDATE_EMAIL)
                    ? $campos[1] : throw new RuntimeException("Email inválido en línea $numLinea"),
                "edad" => (int)$campos[2],
            ];
        }

        return $lineas;

    } catch (RuntimeException $e) {
        error_log("Error procesando $ruta: " . $e->getMessage());
        throw $e;
    } finally {
        if ($archivo) {
            fclose($archivo);
            echo "Archivo $ruta cerrado.\n";
        }
    }
}

try {
    // Crear un archivo de prueba
    file_put_contents("/tmp/test.csv", "Ana,ana@test.com,28\nCarlos,carlos@test.com,35\n");
    $resultados = procesarArchivo("/tmp/test.csv");
    print_r($resultados);
} catch (RuntimeException $e) {
    echo "Error: " . $e->getMessage() . "\n";
}
?>
```

RESUMEN DEL CAPÍTULO
===============================================================================

- PHP tiene errores (E_ERROR, E_WARNING) y excepciones (Throwable).
- try/catch captura excepciones; finally se ejecuta siempre.
- Crea excepciones personalizadas extendiendo Exception.
- Usa throw para lanzar excepciones en condiciones de error.
- throw puede relanzar excepciones para manejo en niveles superiores.
- set_error_handler captura errores no fatales personalizados.
- Errores comunes: off-by-one, break olvidado, bucles infinitos.
- Depuración: var_dump, error_log, debug_backtrace, assert.
- Captura Throwable para cubrir tanto Exception como Error.
- Siempre libera recursos con finally o en un bloque catch.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre Exception y Error?** Exception es para
errores recuperables; Error (PHP 7+) es para errores fatales como TypeError.
Ambos implementan Throwable.

❓ **¿Debo capturar Exception o Throwable?** Captura Throwable si quieres
cubrir tanto excepciones como errores fatales. Captura Exception si solo
quieres errores recuperables.

❓ **¿Qué es finally?** Un bloque que se ejecuta siempre después de try/catch,
haya o no excepción, incluso si hay return en try.

❓ **¿Puedo tener try sin catch?** Sí, try { } finally { } es válido. El
finally se ejecuta después de salir del try (por return o excepción).

❓ **¿Cuándo debo crear una excepción personalizada?** Cuando necesitas
transportar información específica del error (códigos, campos, datos
adicionales) o cuando quieres que ciertos errores sean capturables
específicamente.

❓ **¿Qué es set_error_handler?** Una función que reemplaza el manejador de
errores de PHP. Puede capturar warnings, notices y convertirlos en
excepciones.

❓ **¿Cómo debugging sin xdebug?** Usa var_dump, error_log, print_r,
debug_backtrace, y asserts. Crea logs estructurados y usa archivos de log
para análisis posterior.
