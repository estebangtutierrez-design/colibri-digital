===============================================================================
1. LA ESTRUCTURA IF
===============================================================================

La estructura if es la más básica y fundamental de todas las estructuras de
control en PHP. Permite ejecutar un bloque de código únicamente si se cumple
una condición determinada.

Sintaxis básica:

```php
<?php
if (condición) {
    // Código que se ejecuta si la condición es verdadera
}
```

La condición debe ser una expresión que PHP pueda evaluar como true o false.
Si el resultado es true, el bloque de código dentro de las llaves se ejecuta.
Si es false, se omite.

Veamos un ejemplo sencillo:

```php
<?php
$edad = 18;

if ($edad >= 18) {
    echo "Eres mayor de edad. Puedes ingresar.";
}
?>
```

En este caso, la variable $edad es 18, la condición $edad >= 18 se evalúa
como true, y el mensaje se muestra en pantalla.

Es importante notar que las llaves { } delimitan el bloque de código que se
ejecuta cuando la condición es verdadera. Aunque PHP permite omitir las
llaves si hay una sola instrucción, siempre es recomendable usarlas para
evitar errores y mejorar la legibilidad.

```php
<?php
// Sin llaves (funciona pero no es recomendable)
if ($edad >= 18)
    echo "Eres mayor de edad.";

// Con llaves (recomendado)
if ($edad >= 18) {
    echo "Eres mayor de edad.";
}
?>
```

Si necesitas ejecutar varias instrucciones, las llaves son obligatorias para
agruparlas:

```php
<?php
$edad = 20;
$nombre = "Carlos";

if ($edad >= 18) {
    echo "Hola, $nombre. ";
    echo "Eres mayor de edad. ";
    echo "Bienvenido al sistema.";
}
?>
```

La condición puede ser cualquier expresión que devuelva un valor booleano. PHP
evalúa la expresión y la convierte implícitamente a booleano si es necesario.

```php
<?php
$tieneAcceso = true;

if ($tieneAcceso) {
    echo "Acceso concedido.";
}
?>
```

También puedes usar operadores lógicos para combinar condiciones:

```php
<?php
$edad = 25;
$tieneIdentificacion = true;

if ($edad >= 18 && $tieneIdentificacion) {
    echo "Puedes ingresar al evento.";
}
?>
```

El operador && (AND lógico) requiere que ambas condiciones sean verdaderas
para que el bloque se ejecute. Más adelante veremos otros operadores lógicos.

La estructura if es la base de la toma de decisiones en PHP. Sin ella, todos
los programas serían meras secuencias lineales de instrucciones sin capacidad
de adaptarse a diferentes situaciones.

Una buena práctica es mantener las condiciones simples y legibles. Si una
condición se vuelve demasiado compleja, considera asignarla a una variable
con un nombre descriptivo:

```php
<?php
$edad = 22;
$tieneLicencia = true;
$haCompletadoCurso = false;

$puedeConducir = $edad >= 18 && $tieneLicencia && $haCompletadoCurso;

if ($puedeConducir) {
    echo "Puedes solicitar tu licencia de conducir.";
}
?>
```

Esto hace que el código sea más fácil de leer y mantener.

===============================================================================
2. OPERADORES DE COMPARACIÓN
===============================================================================

Los operadores de comparación son herramientas fundamentales que se utilizan
dentro de las condiciones if. Permiten comparar dos valores y devolver un
resultado booleano (true o false).

PHP ofrece una variedad de operadores de comparación que debes conocer a
fondo:

Operador | Nombre           | Ejemplo       | Resultado
---------|------------------|---------------|----------
==       | Igualdad         | 5 == "5"      | true
===      | Identidad        | 5 === "5"     | false
!=       | Desigualdad      | 5 != "5"      | false
!==      | No identidad     | 5 !== "5"     | true
<        | Menor que        | 3 < 5         | true
>        | Mayor que        | 3 > 5         | false
<=       | Menor o igual    | 5 <= 5        | true
>=       | Mayor o igual    | 5 >= 4        | true
<=>      | Nave espacial    | 5 <=> 3       | 1

El operador == compara dos valores después de convertirlos al mismo tipo
(coerción de tipos). Esto puede llevar a resultados inesperados:

```php
<?php
var_dump(5 == "5");      // bool(true)  - "5" se convierte a 5
var_dump(0 == "hola");   // bool(true)  - "hola" se convierte a 0
var_dump("123" == 123);  // bool(true)
var_dump(null == false); // bool(true)
?>
```

El operador === (identidad) compara tanto el valor como el tipo. Es más
estricto y seguro:

```php
<?php
var_dump(5 === "5");      // bool(false) - int vs string
var_dump(5 === 5);        // bool(true)
var_dump("hola" === "hola"); // bool(true)
var_dump(null === false); // bool(false)
?>
```

Como regla general, siempre que sea posible, utiliza === en lugar de == para
evitar sorpresas con la coerción automática de tipos.

El operador != funciona como la negación de ==:

```php
<?php
var_dump(5 != "5");   // bool(false) - son iguales después de coerción
var_dump(5 != 6);     // bool(true)
?>
```

El operador !== funciona como la negación de ===:

```php
<?php
var_dump(5 !== "5");  // bool(true)  - diferente tipo
var_dump(5 !== 5);    // bool(false) - mismo valor y tipo
?>
```

Los operadores <, >, <=, >= funcionan con números y también con strings:

```php
<?php
var_dump(10 > 5);       // bool(true)
var_dump(3.14 <= 3.14); // bool(true)
var_dump("b" > "a");    // bool(true) - comparación alfabética
var_dump("10" < "5");   // bool(true) - comparación alfabética: "1" < "5"
var_dump(10 < 5);       // bool(false)
?>
```

El operador <=> (nave espacial, introducido en PHP 7) devuelve:
-1 si el valor izquierdo es menor
 0 si son iguales
 1 si el valor izquierdo es mayor

```php
<?php
echo 5 <=> 5;  // 0 (iguales)
echo 3 <=> 5;  // -1 (3 es menor)
echo 7 <=> 5;  // 1 (7 es mayor)
?>
```

La comparación de cadenas con <, >, etc. se basa en el orden lexicográfico
(alfabético). PHP compara carácter por carácter usando los valores ASCII.

```php
<?php
var_dump("manzana" < "pera");    // true (m viene antes que p)
var_dump("Zebra" < "abeja");     // true (Z tiene ASCII 90, a tiene 97)
var_dump("1000" < "2");          // true (comparación como strings: "1" < "2")
var_dump(1000 < 2);              // false (comparación numérica)
?>
```

Los operadores de comparación son el vocabulario básico de las condiciones.
Combinándolos con if, puedes expresar prácticamente cualquier lógica de
decisión.

===============================================================================
3. LA ESTRUCTURA ELSE
===============================================================================

El bloque else complementa a if y permite ejecutar código cuando la condición
no se cumple. Es la otra cara de la moneda: si la condición es true, se
ejecuta el bloque if; si es false, se ejecuta el bloque else.

Sintaxis:

```php
<?php
if (condición) {
    // Código si la condición es verdadera
} else {
    // Código si la condición es falsa
}
```

Ejemplo práctico:

```php
<?php
$edad = 16;

if ($edad >= 18) {
    echo "Eres mayor de edad.";
} else {
    echo "Eres menor de edad.";
}
?>
```

Aquí, como $edad es 16, la condición $edad >= 18 es false, por lo que se
ejecuta el bloque else.

Else permite manejar ambos caminos posibles de una decisión. Es ideal para
situaciones donde necesitas una respuesta para cada caso:

```php
<?php
$calificacion = 85;

if ($calificacion >= 60) {
    echo "Aprobaste el examen. ¡Felicidades!";
} else {
    echo "Reprobaste el examen. Debes estudiar más.";
}
?>
```

El bloque else también puede contener múltiples instrucciones:

```php
<?php
$usuarioAutenticado = false;

if ($usuarioAutenticado) {
    echo "Bienvenido al panel de control.";
    echo "Tu última visita fue ayer.";
    echo "Tienes 3 mensajes nuevos.";
} else {
    echo "Acceso denegado.";
    echo "Por favor, inicia sesión.";
    header("Location: login.php");
    exit;
}
?>
```

Else no puede existir sin un if que lo preceda. Siempre debe ir inmediatamente
después del bloque if correspondiente.

Else es útil para evitar la doble evaluación de condiciones:

```php
<?php
// Sin else (ineficiente)
if ($valor > 10) {
    $resultado = "Mayor que 10";
}
if ($valor <= 10) {
    $resultado = "Menor o igual a 10";
}

// Con else (eficiente y claro)
if ($valor > 10) {
    $resultado = "Mayor que 10";
} else {
    $resultado = "Menor o igual a 10";
}
?>
```

La versión con else es más eficiente porque solo evalúa la condición una vez
y expresa más claramente la intención: "si esto, entonces esto; si no, esto
otro".

Else sienta las bases para estructuras más complejas como elseif y ternarios.

===============================================================================
4. LA ESTRUCTURA ELSEIF
===============================================================================

Cuando necesitas evaluar múltiples condiciones en cadena, elseif es la
herramienta adecuada. Permite comprobar condiciones adicionales si la
anterior no se cumplió.

Sintaxis:

```php
<?php
if (condición1) {
    // Código si condición1 es true
} elseif (condición2) {
    // Código si condición2 es true (y condición1 fue false)
} elseif (condición3) {
    // Código si condición3 es true (y condición1 y condición2 fueron false)
} else {
    // Código si ninguna condición fue true
}
?>
```

Puedes tener tantos elseif como necesites. El bloque else final es opcional.

Ejemplo con calificaciones:

```php
<?php
$nota = 85;

if ($nota >= 90) {
    echo "Excelente (A)";
} elseif ($nota >= 80) {
    echo "Muy bien (B)";
} elseif ($nota >= 70) {
    echo "Bien (C)";
} elseif ($nota >= 60) {
    echo "Suficiente (D)";
} else {
    echo "Reprobado (F)";
}
?>
```

Con $nota = 85, la primera condición (>= 90) es false, la segunda (>= 80) es
true, por lo que se muestra "Muy bien (B)" y el resto de condiciones se
ignoran.

El orden de las condiciones es crucial. Coloca las condiciones más específicas
primero:

```php
<?php
$edad = 65;

// Correcto: condiciones específicas primero
if ($edad >= 65) {
    echo "Adulto mayor";
} elseif ($edad >= 18) {
    echo "Adulto";
} elseif ($edad >= 13) {
    echo "Adolescente";
} else {
    echo "Niño";
}

// Incorrecto: la condición general atrapa todos los casos antes
if ($edad >= 18) {
    echo "Adulto";  // Se ejecuta para $edad = 65 también
} elseif ($edad >= 65) {
    echo "Adulto mayor";  // ¡Nunca se alcanza!
}
?>
```

Elseif es diferente de tener if anidados. Con elseif, tan pronto como una
condición se cumple, las restantes no se evalúan. Con if anidados, todas las
condiciones se evalúan independientemente.

```php
<?php
// Versus elseif: solo una rama se ejecuta
if ($a > $b) {
    echo "a es mayor que b";
} elseif ($a == $b) {
    echo "a es igual a b";
} else {
    echo "a es menor que b";
}

// Versus if separados: múltiples ramas pueden ejecutarse
if ($a > $b) {
    echo "a es mayor que b";
}
if ($a == $b) {  // Este también se evalúa
    echo "a es igual a b";
}
if ($a < $b) {   // Este también se evalúa
    echo "a es menor que b";
}
?>
```

Con elseif, solo una rama se ejecuta. Con if separados, cada condición se
evalúa de forma independiente.

También puedes escribir "else if" como dos palabras. En PHP, else if y
elseif son equivalentes (aunque hay una diferencia semántica menor cuando
se usan llaves):

```php
<?php
// Son equivalentes en la práctica
if ($a > $b) {
    echo "a > b";
} elseif ($a == $b) {
    echo "a == b";
} else {
    echo "a < b";
}

// Esto también funciona
if ($a > $b) {
    echo "a > b";
} else if ($a == $b) {
    echo "a == b";
} else {
    echo "a < b";
}
?>
```

Elseif es ideal para jerarquías de decisión como rangos de edad, niveles de
acceso, categorías de productos, etc.

===============================================================================
5. CONDICIONES ANIDADAS
===============================================================================

Las condiciones anidadas (if dentro de if) permiten crear lógica de decisión
compleja donde una condición depende del resultado de otra.

Sintaxis:

```php
<?php
if (condición_externa) {
    if (condición_interna) {
        // Ambas condiciones se cumplen
    } else {
        // Solo la externa se cumple
    }
} else {
    // La externa no se cumple
}
?>
```

Ejemplo de la vida real:

```php
<?php
$esUsuarioPremium = true;
$diasDesdeUltimoPago = 15;

if ($esUsuarioPremium) {
    echo "Bienvenido, usuario premium. ";
    
    if ($diasDesdeUltimoPago > 30) {
        echo "Tu suscripción está vencida. Renueva para acceder.";
    } else {
        echo "Todo está al día. Disfruta del contenido exclusivo.";
    }
} else {
    echo "Usuario estándar. Considera actualizar a premium.";
}
?>
```

Las condiciones anidadas son poderosas pero pueden volverse difíciles de
leer si se abusan. Como regla general, no anides más de tres niveles. Si
necesitas más, considera refactorizar.

Ejemplo de múltiples niveles (evitar):

```php
<?php
$autenticado = true;
$rol = "editor";
$permiso = "escribir";
$articuloPublicado = false;

if ($autenticado) {
    if ($rol === "admin" || $rol === "editor") {
        if ($permiso === "escribir" || $permiso === "editar") {
            if (!$articuloPublicado) {
                echo "Puedes editar este artículo.";
            } else {
                echo "El artículo ya fue publicado.";
            }
        } else {
            echo "No tienes permiso de escritura.";
        }
    } else {
        echo "Tu rol no permite editar artículos.";
    }
} else {
    echo "Debes iniciar sesión.";
}
?>
```

Este código funciona, pero es difícil de seguir. Una mejor alternativa es
usar condiciones compuestas con operadores lógicos:

```php
<?php
if (!$autenticado) {
    echo "Debes iniciar sesión.";
} elseif ($rol !== "admin" && $rol !== "editor") {
    echo "Tu rol no permite editar artículos.";
} elseif ($permiso !== "escribir" && $permiso !== "editar") {
    echo "No tienes permiso de escritura.";
} elseif ($articuloPublicado) {
    echo "El artículo ya fue publicado.";
} else {
    echo "Puedes editar este artículo.";
}
?>
```

Otra técnica es usar early return dentro de funciones:

```php
<?php
function verificarAcceso($autenticado, $rol, $permiso, $articuloPublicado) {
    if (!$autenticado) {
        return "Debes iniciar sesión.";
    }
    if ($rol !== "admin" && $rol !== "editor") {
        return "Tu rol no permite editar artículos.";
    }
    if ($permiso !== "escribir" && $permiso !== "editar") {
        return "No tienes permiso de escritura.";
    }
    if ($articuloPublicado) {
        return "El artículo ya fue publicado.";
    }
    return "Puedes editar este artículo.";
}

echo verificarAcceso(true, "editor", "escribir", false);
?>
```

Esta técnica se conoce como "guard clauses" o "early return" y produce un
código más limpio y fácil de mantener.

Las condiciones anidadas son útiles cuando necesitas verificar una condición
que depende contextualmente de otra. Por ejemplo:

```php
<?php
$hora = 14;
$diaSemana = "sábado";

if ($diaSemana === "sábado" || $diaSemana === "domingo") {
    // Solo si es fin de semana, verificamos la hora
    if ($hora >= 10 && $hora <= 18) {
        echo "Horario de atención: abierto.";
    } else {
        echo "Horario de atención: cerrado.";
    }
} else {
    echo "Es día laboral. Horario normal.";
}
?>
```

En este caso, tiene sentido anidar porque la verificación de la hora solo
tiene sentido en el contexto del fin de semana.

===============================================================================
6. OPERADORES LÓGICOS
===============================================================================

Los operadores lógicos permiten combinar múltiples condiciones en una sola
expresión. Son esenciales para construir condiciones complejas.

PHP soporta los siguientes operadores lógicos:

Operador | Nombre  | Ejemplo               | Resultado
---------|---------|-----------------------|----------
&&       | AND     | true && false         | false
||       | OR      | true || false         | true
!        | NOT     | !true                 | false
and      | AND     | true and false        | false
or       | OR      | true or false         | true
xor      | XOR     | true xor false        | true

AND (&&): Devuelve true solo si ambas condiciones son true.

```php
<?php
$edad = 25;
$tieneINE = true;

if ($edad >= 18 && $tieneINE) {
    echo "Puedes votar en las elecciones.";
}
?>
```

OR (||): Devuelve true si al menos una de las condiciones es true.

```php
<?php
$efectivo = 50;
$tarjeta = true;

if ($efectivo >= 100 || $tarjeta) {
    echo "Puedes realizar la compra.";
} else {
    echo "No tienes suficiente saldo disponible.";
}
?>
```

NOT (!): Invierte el valor booleano de una condición.

```php
<?php
$puertaAbierta = false;

if (!$puertaAbierta) {
    echo "La puerta está cerrada. Ábrela para pasar.";
}
?>
```

XOR: Devuelve true si exactamente una de las condiciones es true (no ambas).

```php
<?php
$metodoPago = "efectivo";
$usuarioVip = true;

// Descuento: si paga en efectivo O es VIP, pero no ambos
if (($metodoPago === "efectivo") xor ($usuarioVip)) {
    echo "Tienes derecho a un descuento especial.";
}
?>
```

Diferencia entre && y and, || y or: La diferencia radica en la precedencia.
&& y || tienen mayor precedencia que and y or.

```php
<?php
$resultado1 = true && false;   // false (&& se evalúa primero)
$resultado2 = true and false;  // true (and tiene menor precedencia)
// Esto se interpreta como: ($resultado2 = true) and false

echo ($resultado1 ? "true" : "false") . "\n";  // false
echo ($resultado2 ? "true" : "false") . "\n";  // true
?>
```

Por esta razón, es recomendable usar && y || en lugar de and y or para
evitar confusiones.

Combinando múltiples operadores:

```php
<?php
$edad = 22;
$tieneLicencia = true;
$haPagado = true;
$esEmpleado = false;

// Combinación de AND y OR
if (($edad >= 18 && $tieneLicencia) || $esEmpleado) {
    if ($haPagado) {
        echo "Puedes recoger tu pase.";
    } else {
        echo "Debes pagar la cuota primero.";
    }
} else {
    echo "No cumples con los requisitos.";
}
?>
```

Usa paréntesis para agrupar condiciones y hacer explícita la precedencia:

```php
<?php
// Sin paréntesis (ambiguo)
if ($edad >= 18 && $tieneLicencia || $esEmpleado && $haPagado) {
    // ...
}

// Con paréntesis (claro)
if (($edad >= 18 && $tieneLicencia) || ($esEmpleado && $haPagado)) {
    // ...
}
?>
```

Los operadores lógicos también tienen "cortocircuito" (short-circuit):
- Con &&, si la primera condición es false, la segunda no se evalúa.
- Con ||, si la primera condición es true, la segunda no se evalúa.

```php
<?php
function registrarIntento() {
    echo "Intento registrado.\n";
    return true;
}

// El cortocircuito evita llamadas innecesarias
$autenticado = false;

// registrarIntento() nunca se llama porque $autenticado ya es false
if ($autenticado && registrarIntento()) {
    echo "Acceso concedido.";
}
?>
```

Aprovecha el cortocircuito para optimizar tu código y evitar errores:

```php
<?php
// Si $archivo es null, file_get_contents nunca se llama (evita error)
if ($archivo !== null && file_get_contents($archivo)) {
    echo "Archivo leído correctamente.";
}
?>
```

Conocer los operadores lógicos y el cortocircuito te permitirá escribir
condiciones más elegantes y eficientes.

===============================================================================
7. VALORES TRUTHY Y FALSY
===============================================================================

En PHP, cualquier valor puede ser evaluado como booleano en un contexto
condicional. Los valores que se evalúan como true se llaman "truthy" y los
que se evalúan como false se llaman "falsy".

Valores que se evalúan como false (falsy):

```php
<?php
// false explícito
if (false) { }

// 0 (entero)
if (0) { }

// 0.0 (float)
if (0.0) { }

// Cadena vacía ""
if ("") { }

// Cadena "0"
if ("0") { }

// Array vacío []
if ([]) { }

// null
if (null) { }

// La constante null
if (NULL) { }

// Variable no definida (genera warning pero se evalúa como false)
if ($variableNoDefinida) { }

// Objeto SimpleXML vacío (caso especial)
?>
```

Todos los demás valores se evalúan como true (truthy):

```php
<?php
// Números distintos de cero
if (1) { }
if (-1) { }
if (3.14) { }

// Cadenas no vacías (excepto "0")
if ("hola") { }
if ("0.0") { }     // ¡Esto es true! "0.0" no es "0"
if ("false") { }   // ¡Esto es true! Es una cadena no vacía

// Arrays con elementos
if ([1, 2, 3]) { }

// Objetos
if (new stdClass()) { }

// true explícito
if (true) { }
?>
```

Este comportamiento es muy útil para escribir código conciso:

```php
<?php
$nombre = "Ana";

// Forma explícita
if ($nombre !== "" && $nombre !== null) {
    echo "Hola, $nombre";
}

// Forma concisa usando truthy
if ($nombre) {
    echo "Hola, $nombre";
}
?>
```

Ejemplos prácticos de truthy/falsy:

```php
<?php
// Verificar si un array tiene elementos
$productos = ["laptop", "mouse", "teclado"];
if ($productos) {
    echo "Hay " . count($productos) . " productos en el carrito.";
} else {
    echo "El carrito está vacío.";
}

// Verificar si una cadena no está vacía
$nombre = "Carlos";
if ($nombre) {
    echo "Bienvenido, $nombre";
} else {
    echo "El nombre es obligatorio.";
}

// Verificar si un número es diferente de cero
$descuento = 15;
if ($descuento) {
    $total -= $total * ($descuento / 100);
    echo "Descuento aplicado: $descuento%";
}
?>
```

Sin embargo, ten cuidado con casos ambiguos:

```php
<?php
// ¿Qué esperas aquí?
$cantidad = 0;

if ($cantidad) {
    echo "Hay $cantidad unidades.";
} else {
    echo "No hay unidades.";  // Se ejecuta esto porque 0 es falsy
}

// Si quieres diferenciar entre "0" y "no hay valor":
if ($cantidad !== null) {
    echo "Cantidad especificada: $cantidad";
} else {
    echo "Cantidad no especificada.";
}
?>
```

Un caso especialmente engañoso es la cadena "0":

```php
<?php
$opcion = "0";

if ($opcion) {
    echo "Opción seleccionada.";
} else {
    echo "No se seleccionó ninguna opción.";  // "0" es falsy
}

// Para verificar si una cadena tiene contenido realmente:
if (strlen($opcion) > 0) {
    echo "La cadena tiene " . strlen($opcion) . " carácter(es).";
}
?>
```

Comprender qué valores son truthy y cuáles falsy te ayudará a:
- Escribir condiciones más concisas y legibles.
- Evitar errores sutiles cuando un valor inesperado se evalúa como true o false.
- Entender por qué ciertas condiciones funcionan o fallan.

La regla de oro: usa truthy/falsy para verificaciones rápidas de existencia,
pero usa comparaciones explícitas (===, !==) cuando necesites precisión.

```php
<?php
// Concisa: verificar si hay algo
if ($nombre) { }

// Precisa: verificar valores específicos
if ($estado === "activo") { }
if ($cantidad !== 0) { }
if ($opcion !== "0") { }
?>
```

Domina los valores truthy y falsy y tu código PHP será más elegante y
expresivo.

===============================================================================
8. EL OPERADOR TERNARIO
===============================================================================

El operador ternario es una forma abreviada de if/else que devuelve un valor.
Es útil para asignaciones condicionales simples.

Sintaxis:

```php
<?php
$variable = (condición) ? valor_si_true : valor_si_false;
?>
```

Ejemplo básico:

```php
<?php
$edad = 20;
$mensaje = ($edad >= 18) ? "Mayor de edad" : "Menor de edad";
echo $mensaje; // "Mayor de edad"

// Equivalente con if/else
if ($edad >= 18) {
    $mensaje = "Mayor de edad";
} else {
    $mensaje = "Menor de edad";
}
?>
```

El ternario es ideal para asignaciones condicionales simples:

```php
<?php
$hora = 14;
$saludo = ($hora < 12) ? "Buenos días" : "Buenas tardes";
echo $saludo;

$nota = 85;
$resultado = ($nota >= 60) ? "Aprobado" : "Reprobado";

$autenticado = true;
$clase = $autenticado ? "visible" : "oculto";
?>
```

Puedes anidar ternarios, pero hazlo con moderación:

```php
<?php
$nota = 85;
$calificacion = ($nota >= 90) ? "A" : (($nota >= 80) ? "B" : (($nota >= 70) ? "C" : "F"));
echo $calificacion; // "B"
?>
```

Los ternarios anidados son difíciles de leer. En estos casos, es mejor usar
if/else o elseif.

PHP 8 introdujo el operador nullsafe (??) que es una variante del ternario:

```php
<?php
// ?? verifica si el valor es null
$nombre = $usuario ?? "Invitado";

// Equivalente a:
$nombre = isset($usuario) ? $usuario : "Invitado";
?>
```

El operador ternario es una herramienta poderosa para escribir código más
compacto. Úsalo cuando la lógica sea simple y la legibilidad no se vea
comprometida.

===============================================================================
9. ERRORES COMUNES Y BUENAS PRÁCTICAS
===============================================================================

Incluso los programadores experimentados cometen errores con if/else.
Conocer los errores más comunes te ahorrará horas de depuración.

Error 1: Usar = en lugar de == o ===

```php
<?php
$edad = 18;

// Error: asignación en lugar de comparación
if ($edad = 18) {   // Asigna 18 a $edad, luego evalúa 18 (truthy)
    echo "Tienes 18 años.";
}

// Siempre será true porque la asignación devuelve el valor asignado
// El código original $edad se pierde
?>
```

Para evitar este error, muchos programadores colocan el valor literal a la
izquierda (estilo Yoda):

```php
<?php
if (18 === $edad) {  // Si olvidas el segundo =, causa error de sintaxis
    echo "Tienes 18 años.";
}
?>
```

Error 2: Olvidar las llaves con múltiples instrucciones

```php
<?php
$edad = 20;

// Sin llaves: solo la primera instrucción está en el if
if ($edad >= 18)
    echo "Mayor de edad.\n";
    echo "Puedes ingresar.\n";  // ¡Esto siempre se ejecuta!

// Con llaves: ambas instrucciones están condicionadas
if ($edad >= 18) {
    echo "Mayor de edad.\n";
    echo "Puedes ingresar.\n";
}
?>
```

Error 3: Punto y coma después de if

```php
<?php
if ($edad >= 18); {  // El ; termina el if, el bloque siempre se ejecuta
    echo "Mayor de edad.";
}
// Es equivalente a:
if ($edad >= 18) { /* nada */ }
{ echo "Mayor de edad."; }  // Bloque independiente
?>
```

Error 4: Condiciones redundantes

```php
<?php
$activo = true;

// Redundante
if ($activo === true) { }

// Correcto
if ($activo) { }

// Otro ejemplo
if ($activo == false) { }
// Mejor
if (!$activo) { }
?>
```

Error 5: No considerar el orden de las condiciones en elseif

```php
<?php
$edad = 70;

if ($edad >= 18) {
    echo "Adulto";
} elseif ($edad >= 65) {
    echo "Adulto mayor";  // Nunca se ejecuta porque >=18 atrapa todos los >=65
}
?>
```

Buenas prácticas:

1. Siempre usa llaves {} aunque el bloque tenga una sola línea.

2. Usa === y !== en lugar de == y != a menos que necesites explícitamente
   la coerción de tipos.

3. Mantén las condiciones simples. Si una condición es muy compleja,
   asígnala a una variable.

4. Evita más de tres niveles de anidamiento.

5. Usa elseif en lugar de if anidados cuando sea posible.

6. Comprueba primero las condiciones más específicas.

7. Usa ternarios solo para asignaciones simples.

8. Considera early return en funciones para evitar anidamiento.

9. Sé consistente con el estilo de llaves (K&R vs Allman).

10. Escribe condiciones legibles para otros humanos, no solo para la máquina.

Ejemplo 1: Sistema de validación de formulario


RESUMEN DEL CAPÍTULO
===============================================================================

- if ejecuta un bloque de código si una condición es verdadera.
- else proporciona una alternativa cuando la condición es false.
- elseif permite encadenar múltiples condiciones en secuencia.
- Los operadores de comparación (==, ===, !=, !==, <, >, <=, >=) comparan
  valores y devuelven booleanos.
- === compara valor y tipo; == solo compara valor (con coerción).
- Los operadores lógicos (&&, ||, !) combinan condiciones booleanas.
- Los valores truthy y falsy determinan cómo se evalúan los tipos no
  booleanos en contextos condicionales.
- El operador ternario ofrece una sintaxis compacta para if/else simple.
- Las condiciones anidadas permiten lógica jerárquica pero deben usarse
  con moderación.
- Buenas prácticas: usar ===, llaves siempre, evitar anidamiento profundo.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre = y ==?** = es el operador de asignación,
asigna un valor a una variable. == es el operador de comparación, compara
dos valores. Confundirlos es uno de los errores más comunes en PHP.

❓ **¿Debo usar siempre llaves en if aunque tenga una sola línea?** Sí, es
altamente recomendable. Las llaves previenen errores cuando agregas más
líneas al bloque y hacen el código más legible y mantenible.

❓ **¿Cuándo debería usar elseif en lugar de if anidados?** Usa elseif cuando
las condiciones son mutuamente excluyentes y quieres que solo una se ejecute.
Usa if anidados cuando una condición solo tiene sentido en el contexto de otra.

❓ **¿Qué operador de comparación debo usar por defecto?** Usa === y !== por
defecto. Solo usa == y != cuando necesites explícitamente la coerción de
tipos (poco común).

❓ **¿Puedo usar if sin else?** Sí, else es opcional. Un if sin else es
perfectamente válido cuando solo necesitas ejecutar código en un caso
específico y no hay acción alternativa.

❓ **¿Cuántos elseif puedo encadenar?** PHP no tiene un límite teórico, pero
por legibilidad se recomienda no usar más de 3-4 elseif. Para muchos casos,
considera usar switch o un array de mapeo.

❓ **Qué es el operador navegador espacial? Es el operador <=> (introducido en
PHP 7). Devuelve -1, 0 o 1 según si el valor izquierdo es menor, igual o
mayor que el derecho. Es útil para funciones de ordenamiento (usort).
