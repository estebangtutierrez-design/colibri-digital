===============================================================================
1. INTRODUCCIÓN A SWITCH
===============================================================================

La estructura switch es una alternativa elegante cuando necesitas comparar
una misma expresión contra múltiples valores posibles. En lugar de escribir
una larga cadena de elseif, usas switch para listar los casos de forma clara.

Imagina que tienes una variable $dia y quieres mostrar un mensaje según el
día de la semana. Con elseif se vería así:

```php
<?php
$dia = "lunes";

if ($dia === "lunes") {
    echo "Inicio de semana. ¡Ánimo!";
} elseif ($dia === "martes") {
    echo "Martes, sigue adelante.";
} elseif ($dia === "miércoles") {
    echo "Mitad de semana.";
} elseif ($dia === "jueves") {
    echo "Casi viernes.";
} elseif ($dia === "viernes") {
    echo "¡Por fin viernes!";
} elseif ($dia === "sábado") {
    echo "Día de descanso.";
} elseif ($dia === "domingo") {
    echo "Día de recargar energías.";
} else {
    echo "Día no válido.";
}
?>
```

Con switch, el mismo código es más limpio:

```php
<?php
$dia = "lunes";

switch ($dia) {
    case "lunes":
        echo "Inicio de semana. ¡Ánimo!";
        break;
    case "martes":
        echo "Martes, sigue adelante.";
        break;
    case "miércoles":
        echo "Mitad de semana.";
        break;
    case "jueves":
        echo "Casi viernes.";
        break;
    case "viernes":
        echo "¡Por fin viernes!";
        break;
    case "sábado":
        echo "Día de descanso.";
        break;
    case "domingo":
        echo "Día de recargar energías.";
        break;
    default:
        echo "Día no válido.";
}
?>
```

La estructura switch hace evidente que todas las opciones dependen del mismo
valor ($dia). Es más legible y más fácil de mantener.

Sintaxis general:

```php
switch (expresión) {
    case valor1:
        // Código si expresión === valor1
        break;
    case valor2:
        // Código si expresión === valor2
        break;
    // Más casos...
    default:
        // Código si ningún caso coincide
}
```

Importante: switch usa comparación no estricta (==), no identidad (===). Esto
significa que switch ("42") y case 42 coincidirán porque "42" == 42 es true.

Desde PHP 8, puedes usar switch con comparación estricta si usas match.
Pero switch tradicional sigue usando ==.

Switch es ideal cuando:
- Comparas una sola expresión contra muchos valores.
- Los valores de comparación son constantes conocidas.
- La lógica para cada valor es independiente.
- La legibilidad es una prioridad.

No es ideal cuando:
- Las condiciones son complejas (rangos, combinaciones lógicas).
- Necesitas comparar múltiples variables.
- Las condiciones implican operadores <, >, etc.

===============================================================================
2. SINTAXIS DETALLADA
===============================================================================

La sintaxis de switch merece un estudio detallado para evitar errores.

Componentes:

1. La palabra reservada switch seguida de paréntesis con la expresión a
   evaluar.

2. Un bloque de llaves { } que contiene todos los casos.

3. Cada case seguido del valor a comparar y dos puntos (:).

4. El código a ejecutar para cada caso.

5. break para salir del switch (opcional pero crucial).

6. default para el caso por defecto (opcional).

Ejemplo completo:

```php
<?php
$opcion = 2;

switch ($opcion) {
    case 1:
        echo "Seleccionaste la opción 1";
        break;
    case 2:
        echo "Seleccionaste la opción 2";
        break;
    case 3:
        echo "Seleccionaste la opción 3";
        break;
    default:
        echo "Opción no válida";
}
?>
```

La expresión en switch puede ser cualquier valor que devuelva un tipo
comparable: int, float, string, bool, o incluso null.

```php
<?php
// Con string
$color = "rojo";
switch ($color) {
    case "rojo":
        echo "Color primario";
        break;
    case "verde":
        echo "Color secundario";
        break;
}

// Con int
$codigo = 404;
switch ($codigo) {
    case 200:
        echo "OK";
        break;
    case 404:
        echo "Not Found";
        break;
}

// Con bool
$activo = true;
switch ($activo) {
    case true:
        echo "Activo";
        break;
    case false:
        echo "Inactivo";
        break;
}
?>
```

Puedes usar switch dentro de bucles o funciones sin problemas:

```php
<?php
function obtenerMensaje(int $codigo): string {
    switch ($codigo) {
        case 200:
            return "OK";
        case 301:
            return "Moved Permanently";
        case 404:
            return "Not Found";
        case 500:
            return "Internal Server Error";
        default:
            return "Código desconocido";
    }
}

echo obtenerMensaje(404); // "Not Found"
?>
```

En funciones, puedes usar return en lugar de break para salir del switch
y de la función simultáneamente.

Una sintaxis alternativa permite usar switch con dos puntos y endswitch:

```php
<?php
$dia = 3;

switch ($dia):
    case 1:
        echo "Lunes";
        break;
    case 2:
        echo "Martes";
        break;
    case 3:
        echo "Miércoles";
        break;
    default:
        echo "Día desconocido";
endswitch;
?>
```

Esta sintaxis es menos común pero útil cuando mezclas switch con HTML.

===============================================================================
3. LA IMPORTANCIA DE BREAK
===============================================================================

break es probablemente el elemento más importante (y traicionero) de switch.
Sin break, la ejecución continúa al siguiente case, lo que se conoce como
"fall-through" (caída).

Veamos qué sucede sin break:

```php
<?php
$numero = 1;

switch ($numero) {
    case 1:
        echo "Uno\n";
    case 2:
        echo "Dos\n";
    case 3:
        echo "Tres\n";
}

/* Salida:
Uno
Dos
Tres
*/
?>
```

Sin break, después de ejecutar case 1, PHP continúa ejecutando case 2 y
case 3. Esto rara vez es lo que queremos.

El comportamiento correcto requiere break:

```php
<?php
$numero = 1;

switch ($numero) {
    case 1:
        echo "Uno\n";
        break;
    case 2:
        echo "Dos\n";
        break;
    case 3:
        echo "Tres\n";
        break;
}

/* Salida:
Uno
*/
?>
```

Recuerda siempre incluir break al final de cada case, a menos que
intencionalmente quieras fall-through.

break también puede ir acompañado de un número para salir de múltiples
niveles de switch anidados:

```php
<?php
switch ($opcionExterna) {
    case "menu":
        switch ($opcionInterna) {
            case 1:
                echo "Opción 1 del menú";
                break 2; // Sale de ambos switch
            case 2:
                echo "Opción 2 del menú";
                break;   // Solo sale del switch interno
        }
        break;
}
?>
```

break 2 sale de dos niveles: el switch interno y el externo.

===============================================================================
4. FALL-THROUGH INTENCIONAL
===============================================================================

El fall-through (caída entre casos) no siempre es un error. A veces es
exactamente lo que necesitas. Puedes agrupar varios casos que ejecuten el
mismo código.

Ejemplo clásico: días de la semana

```php
<?php
$dia = "sábado";

switch ($dia) {
    case "lunes":
    case "martes":
    case "miércoles":
    case "jueves":
    case "viernes":
        echo "Es día laboral.";
        break;
    case "sábado":
    case "domingo":
        echo "Es fin de semana. ¡A descansar!";
        break;
    default:
        echo "Día no válido.";
}
?>
```

Aquí, los casos lunes a viernes se agrupan porque comparten el mismo código.
El fall-through es intencional y beneficioso.

Otro ejemplo: estaciones del año

```php
<?php
$mes = 3;

switch ($mes) {
    case 12:
    case 1:
    case 2:
        echo "Verano";
        break;
    case 3:
    case 4:
    case 5:
        echo "Otoño";
        break;
    case 6:
    case 7:
    case 8:
        echo "Invierno";
        break;
    case 9:
    case 10:
    case 11:
        echo "Primavera";
        break;
    default:
        echo "Mes no válido";
}
?>
```

El fall-through intencional es limpio porque evita repetir código:

```php
<?php
// Sin fall-through (código repetido)
switch ($rol) {
    case "admin":
        echo "Acceso total.";
        echo "Puedes gestionar usuarios.";
        break;
    case "editor":
        echo "Acceso limitado.";
        echo "Puedes gestionar contenido.";
        break;
    case "suscriptor":
        echo "Acceso limitado.";
        echo "Puedes ver contenido premium.";
        break;
}

// Con fall-through
switch ($rol) {
    case "admin":
        echo "Acceso total.\n";
        echo "Puedes gestionar usuarios.\n";
        break;
    case "editor":
    case "suscriptor":
        echo "Acceso limitado.\n";
        if ($rol === "editor") {
            echo "Puedes gestionar contenido.\n";
        } else {
            echo "Puedes ver contenido premium.\n";
        }
        break;
}
?>
```

El fall-through debe usarse con cuidado y siempre documentarlo con un
comentario si no es obvio:

```php
<?php
switch ($codigoError) {
    case E_WARNING:
    case E_NOTICE:
        // Intencional: ambos son errores no fatales
        $tipo = "No fatal";
        break;
    case E_ERROR:
    case E_PARSE:
        // Intencional: ambos son errores fatales
        $tipo = "Fatal";
        break;
}
?>
```

Un error común es olvidar break y causar fall-through no deseado. Siempre
revisa tu código para asegurarte de que cada case termine adecuadamente.

===============================================================================
5. EL CASO DEFAULT
===============================================================================

El caso default maneja cualquier valor que no coincida con ningún case
especificado. Es equivalente al else en if/else.

```php
<?php
$rol = "visitante";

switch ($rol) {
    case "admin":
        echo "Bienvenido, administrador.";
        break;
    case "editor":
        echo "Bienvenido, editor.";
        break;
    case "usuario":
        echo "Bienvenido, usuario.";
        break;
    default:
        echo "Rol no reconocido. Acceso denegado.";
}
?>
```

Default es opcional pero recomendado. Sin él, si ningún case coincide,
switch simplemente no hace nada, lo cual puede ocultar errores.

Posición de default: tradicionalmente va al final, pero PHP permite
colocarlo en cualquier posición dentro del switch.

```php
<?php
$accion = "salir";

// Default al inicio (válido pero poco común)
switch ($accion) {
    default:
        echo "Acción no reconocida.\n";
        break;
    case "entrar":
        echo "Iniciando sesión.\n";
        break;
    case "salir":
        echo "Cerrando sesión.\n";
        break;
}

// Default al final (convencional)
switch ($accion) {
    case "entrar":
        echo "Iniciando sesión.\n";
        break;
    case "salir":
        echo "Cerrando sesión.\n";
        break;
    default:
        echo "Acción no reconocida.\n";
        break;
}
?>
```

Si colocas default en medio, no olvides el break para evitar fall-through:

```php
<?php
$valor = "especial";

switch ($valor) {
    case "normal":
        echo "Procesando valor normal.\n";
        break;
    default:
        echo "Valor especial o desconocido.\n";
        // Sin break aquí, caería al siguiente caso
        break;
    case "urgente":
        echo "Procesando valor urgente.\n";
        break;
}
?>
```

Default también puede ser usado en conjunto con fall-through:

```php
<?php
$usuario = "invitado";

switch ($usuario) {
    case "admin":
        $permisos = "total";
        break;
    case "editor":
    case "autor":
        $permisos = "escritura";
        break;
    case "suscriptor":
        $permisos = "lectura";
        break;
    default:
        $permisos = "ninguno";
        echo "Usuario no autenticado. Asignando permisos básicos.\n";
}
?>
```

Siempre incluye un default significativo. Un buen default registra el caso
inesperado o lanza una excepción para depuración.

===============================================================================
6. SWITCH ANIDADOS
===============================================================================

Aunque no es común, switch puede anidarse dentro de otro switch para manejar
decisiones jerárquicas.

```php
<?php
$categoria = "electrónica";
$subcategoria = "laptop";

switch ($categoria) {
    case "electrónica":
        echo "Categoría: Electrónica\n";
        switch ($subcategoria) {
            case "laptop":
                echo "Subcategoría: Laptops\n";
                echo "Productos: Dell, HP, Lenovo\n";
                break;
            case "celular":
                echo "Subcategoría: Celulares\n";
                echo "Productos: iPhone, Samsung, Xiaomi\n";
                break;
            case "audio":
                echo "Subcategoría: Audio\n";
                echo "Productos: Auriculares, Parlantes\n";
                break;
            default:
                echo "Subcategoría no encontrada.\n";
        }
        break;
    case "ropa":
        echo "Categoría: Ropa\n";
        switch ($subcategoria) {
            case "hombre":
                echo "Sección: Caballeros\n";
                break;
            case "mujer":
                echo "Sección: Damas\n";
                break;
            default:
                echo "Subcategoría no encontrada.\n";
        }
        break;
    default:
        echo "Categoría no encontrada.\n";
}
?>
```

Los switch anidados pueden volverse difíciles de leer. Si necesitas más de
dos niveles, considera usar funciones para encapsular la lógica:

```php
<?php
function manejarElectronica(string $subcategoria): void {
    switch ($subcategoria) {
        case "laptop":
            echo "Subcategoría: Laptops\n";
            break;
        case "celular":
            echo "Subcategoría: Celulares\n";
            break;
        default:
            echo "Subcategoría no encontrada.\n";
    }
}

function manejarRopa(string $subcategoria): void {
    switch ($subcategoria) {
        case "hombre":
            echo "Sección: Caballeros\n";
            break;
        case "mujer":
            echo "Sección: Damas\n";
            break;
        default:
            echo "Sección no encontrada.\n";
    }
}

$categoria = "electrónica";
$subcategoria = "laptop";

switch ($categoria) {
    case "electrónica":
        manejarElectronica($subcategoria);
        break;
    case "ropa":
        manejarRopa($subcategoria);
        break;
    default:
        echo "Categoría no encontrada.\n";
}
?>
```

Esta aproximación mantiene el código modular y legible.

===============================================================================
7. SWITCH VS IF/ELSEIF
===============================================================================

¿Cuándo usar switch y cuándo usar if/elseif? No hay una regla absoluta,
pero estas guías te ayudarán a decidir.

Usa switch cuando:
- Comparas una sola expresión contra múltiples valores constantes.
- Los valores son conocidos y limitados.
- La legibilidad se beneficia de la estructura tabular de switch.

```php
<?php
// Buen candidato para switch
$codigoPais = "+1";
switch ($codigoPais) {
    case "+1": echo "USA/Canadá"; break;
    case "+52": echo "México"; break;
    case "+34": echo "España"; break;
    case "+54": echo "Argentina"; break;
    default: echo "País desconocido";
}
?>
```

Usa if/elseif cuando:
- Las condiciones implican rangos (<, >, <=, >=).
- Necesitas combinar condiciones con &&, ||.
- Comparas múltiples variables diferentes.

```php
<?php
// Mal candidato para switch (rangos)
// NO se puede hacer: case ($edad >= 18):
$edad = 25;
if ($edad >= 65) {
    echo "Adulto mayor";
} elseif ($edad >= 18) {
    echo "Adulto";
} elseif ($edad >= 13) {
    echo "Adolescente";
} else {
    echo "Niño";
}
?>
```

Consideraciones de rendimiento:
- switch suele ser más rápido que elseif (PHP optimiza switch con una tabla
  de búsqueda cuando los casos son valores consecutivos).
- La diferencia es mínima para la mayoría de aplicaciones.
- Prioriza la legibilidad sobre micro-optimizaciones.

Ejemplo: ambos códigos hacen lo mismo, ¿cuál prefieres?

```php
<?php
// Versión con elseif
$estado = "pendiente";
if ($estado === "activo") {
    $accion = "mostrar";
} elseif ($estado === "inactivo") {
    $accion = "ocultar";
} elseif ($estado === "pendiente") {
    $accion = "revisar";
} elseif ($estado === "archivado") {
    $accion = "eliminar";
} else {
    $accion = "ignorar";
}

// Versión con switch
switch ($estado) {
    case "activo":
        $accion = "mostrar";
        break;
    case "inactivo":
        $accion = "ocultar";
        break;
    case "pendiente":
        $accion = "revisar";
        break;
    case "archivado":
        $accion = "eliminar";
        break;
    default:
        $accion = "ignorar";
}
?>
```

Ambos son válidos. Switch es más verboso pero más legible cuando hay muchos
casos. Elseif es más flexible pero puede ser más difícil de leer con muchas
condiciones.

Una alternativa interesante para switch con valores es usar un array de
mapeo:

```php
<?php
// Array de mapeo (alternativa a switch)
$acciones = [
    "activo" => "mostrar",
    "inactivo" => "ocultar",
    "pendiente" => "revisar",
    "archivado" => "eliminar",
];

$estado = "pendiente";
$accion = $acciones[$estado] ?? "ignorar";
echo $accion; // "revisar"
?>
```

Esta técnica es muy útil y a menudo más elegante que switch.

===============================================================================
8. SWITCH Y TIPOS DE DATOS
===============================================================================

Switch usa comparación no estricta (==). Esto tiene implicaciones
importantes con los diferentes tipos de datos.

```php
<?php
$valor = "42";

switch ($valor) {
    case 42:
        echo "Coincide con 42 (int)";
        break;
    case "42":
        echo "Coincide con '42' (string)";
        break;
}
// Salida: Coincide con 42 (int) porque "42" == 42 es true
?>
```

Esto significa que con switch, el orden de los cases importa cuando hay
valores que pueden ser iguales con ==:

```php
<?php
$valor = 0;

switch ($valor) {
    case "abc":
        echo "abc";
        break;
    case 0:
        echo "cero";
        break;
}
// Salida: cero (correcto)

// Pero si cambiamos el orden:
switch ($valor) {
    case "abc":
        echo "abc";  // "abc" == 0 es TRUE en PHP 7
        break;
    case 0:
        echo "cero";
        break;
}
// Salida: abc (¡sorpresa!)
?>
```

Este comportamiento es una fuente conocida de bugs. En PHP 8, se modificó
la comparación para evitar algunos casos extremos, pero la lección es:
ten cuidado con la comparación no estricta en switch.

Switch con booleanos:

```php
<?php
$activo = true;

switch ($activo) {
    case true:
        echo "Está activo";
        break;
    case false:
        echo "No está activo";
        break;
}
?>
```

Switch con null:

```php
<?php
$valor = null;

switch ($valor) {
    case null:
        echo "Es null";
        break;
    case 0:
        echo "Es cero";
        break;
    case "":
        echo "Es cadena vacía";
        break;
}
// Salida: Es null (null == 0 es false, null == "" es true en PHP 7, cuidado)
?>
```

En PHP 8, null == "" sigue siendo true, pero null == 0 es false (cambio
con respecto a PHP 7). Esto hace que switch sea más predecible en PHP 8.

Para evitar problemas con tipos mixtos, puedes forzar el tipo de la
expresión antes de pasarla a switch:

```php
<?php
$entrada = "42";
$valor = (int) $entrada; // Forzar a entero

switch ($valor) {
    case 42:
        echo "Cuarenta y dos";
        break;
    case 0:
        echo "Cero";
        break;
}
?>
```

O usar match (PHP 8+), que usa comparación estricta (===).

9. EJEMPLOS PRÁCTICOS
===============================================================================

Ejemplo 1: Menú interactivo

```php
<?php
echo "=== MENÚ PRINCIPAL ===\n";
echo "1. Ver perfil\n";
echo "2. Editar configuración\n";
echo "3. Ver mensajes\n";
echo "4. Cerrar sesión\n";
echo "Selecciona una opción: ";

$opcion = trim(fgets(STDIN));

switch ($opcion) {
    case "1":
        echo "Mostrando perfil del usuario...\n";
        echo "Nombre: Juan Pérez\n";
        echo "Email: juan@example.com\n";
        break;
    case "2":
        echo "Opciones de configuración:\n";
        echo "- Cambiar contraseña\n";
        echo "- Notificaciones\n";
        echo "- Privacidad\n";
        break;
    case "3":
        echo "Bandeja de entrada:\n";
        echo "- 3 mensajes sin leer\n";
        echo "- 12 mensajes leídos\n";
        break;
    case "4":
        echo "Cerrando sesión...\n";
        echo "¡Hasta pronto!";
        break;
    default:
        echo "Opción no válida. Intenta de nuevo.\n";
}
?>
```

Ejemplo 2: Procesador de eventos

```php
<?php
function procesarEvento(string $tipo, array $datos): void {
    switch ($tipo) {
        case "login":
            echo "Usuario {$datos['usuario']} inició sesión.\n";
            echo "IP: {$datos['ip']}\n";
            echo "Fecha: {$datos['fecha']}\n";
            break;
            
        case "compra":
            $total = array_sum($datos['productos']);
            echo "Compra realizada por {$datos['usuario']}.\n";
            echo "Total: \${$total}\n";
            echo "Productos: " . count($datos['productos']) . "\n";
            break;
            
        case "registro":
            echo "Nuevo usuario registrado: {$datos['email']}\n";
            echo "Plan: {$datos['plan']}\n";
            break;
            
        case "error":
            echo "Error: {$datos['mensaje']}\n";
            echo "Código: {$datos['codigo']}\n";
            echo "Archivo: {$datos['archivo']}:{$datos['linea']}\n";
            break;
            
        default:
            echo "Evento desconocido: $tipo\n";
            error_log("Evento no manejado: $tipo");
    }
}

procesarEvento("login", [
    "usuario" => "ana123",
    "ip" => "192.168.1.1",
    "fecha" => date("Y-m-d H:i:s"),
]);
?>
```

Ejemplo 3: Calculadora básica

```php
<?php
function calculadora(float $a, float $b, string $operacion): string {
    switch ($operacion) {
        case "+":
        case "suma":
            $resultado = $a + $b;
            $simbolo = "+";
            break;
        case "-":
        case "resta":
            $resultado = $a - $b;
            $simbolo = "-";
            break;
        case "*":
        case "x":
        case "multiplicacion":
            $resultado = $a * $b;
            $simbolo = "×";
            break;
        case "/":
        case "division":
            if ($b == 0) {
                return "Error: División entre cero";
            }
            $resultado = $a / $b;
            $simbolo = "÷";
            break;
        case "%":
        case "modulo":
            if ($b == 0) {
                return "Error: Módulo entre cero";
            }
            $resultado = $a % $b;
            $simbolo = "%";
            break;
        case "**":
        case "potencia":
            $resultado = $a ** $b;
            $simbolo = "^";
            break;
        default:
            return "Error: Operación '$operacion' no soportada";
    }
    return "$a $simbolo $b = $resultado";
}

echo calculadora(10, 3, "+") . "\n";       // 10 + 3 = 13
echo calculadora(10, 3, "-") . "\n";       // 10 - 3 = 7
echo calculadora(10, 3, "*") . "\n";       // 10 × 3 = 30
echo calculadora(10, 0, "/") . "\n";       // Error: División entre cero
echo calculadora(10, 3, "%") . "\n";       // 10 % 3 = 1
echo calculadora(2, 8, "potencia") . "\n"; // 2 ^ 8 = 256
?>
```

        case "exit":

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- switch evalúa una expresión y ejecuta el case que coincide usando
  comparación no estricta (==).
- Cada case debe terminar con break para evitar fall-through no deseado.
- El fall-through intencional permite agrupar múltiples cases con el
  mismo código.
- default maneja cualquier valor que no coincida con ningún case.
- Switch es ideal para comparar una sola expresión contra múltiples
  valores constantes.
- Switch usa comparación ==, no === (a diferencia de match en PHP 8+).
- Los switch anidados deben usarse con moderación.
- Un array de mapeo puede ser una alternativa elegante a switch.
- Siempre incluye un default significativo.
- switch con switch (true) permite usar expresiones booleanas.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué se llama "switch"?** Porque "cambias" (switch) entre diferentes
caminos de ejecución según el valor de una expresión. Es como un selector
de ferrocarril que dirige el tren a diferentes vías.

❓ **¿Es obligatorio usar break en cada case?** No es obligatorio, pero
olvidarlo casi siempre es un error. Sin break, la ejecución continúa al
siguiente case (fall-through). Úsalo siempre excepto cuando el
fall-through sea intencional.

❓ **¿Puedo usar switch con rangos (case > 5)?** No directamente. Switch solo
compara valores exactos con ==. Para rangos, debes usar if/elseif o usar
switch (true) con expresiones booleanas en cada case.

❓ **¿Qué diferencia hay entre switch y match en PHP 8?** Match usa
comparación estricta (===), devuelve un valor (es una expresión), no
necesita break, y lanza UnhandledMatchError si no hay coincidencia y
no hay default.

❓ **¿default siempre debe ir al final?** No, default puede ir en cualquier
posición. Por convención va al final, pero si lo colocas en otra posición,
recuerda incluir break.

❓ **¿Puedo usar switch con objetos?** Switch no funciona directamente con
objetos. La expresión se evalúa, pero la comparación == con objetos
compara identidad, no propiedades. Para objetos, usa if/elseif o match.

❓ **¿Cuántos cases puede tener un switch?** PHP no impone un límite, pero
switch con más de 10-15 casos suele ser señal de que necesitas una
estructura de datos diferente (arrays, polimorfismo, etc.).
