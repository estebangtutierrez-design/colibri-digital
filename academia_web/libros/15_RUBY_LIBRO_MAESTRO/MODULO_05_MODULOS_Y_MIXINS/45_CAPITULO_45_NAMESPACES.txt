────────────────────────────────────────────────────────────────────────────────────
  BIENVENIDA
────────────────────────────────────────────────────────────────────────────────────

Bienvenido al Capítulo 45 del Ruby Master Book. Hemos visto los módulos como
contenedores de métodos y como mixins. Ahora exploraremos su tercer gran uso:
los espacios de nombres (namespaces).

Conforme un proyecto crece, el riesgo de colisiones entre nombres de clases,
constantes y métodos aumenta. Dos gemas diferentes pueden definir una clase
`Cliente` o una constante `VERSION`. Los namespaces en Ruby, implementados
mediante módulos, resuelven este problema elegantemente.

En este capítulo veremos el anidamiento de módulos, el operador `::` para
resolución, cómo Ruby busca constantes, la pila de módulos con
`Module.nesting`, y las convenciones de organización que usan frameworks
como Rails. También exploraremos cómo cargar archivos automáticamente
usando la convención de nombres de módulos.

────────────────────────────────────────────────────────────────────────────────────
  FRASE DEL CAPÍTULO
────────────────────────────────────────────────────────────────────────────────────

"Los espacios de nombres son como apellidos para tus clases: evitan
  confusiones y dejan claro de quién está hablando cada uno."
  — David Heinemeier Hansson (DHH), creador de Ruby on Rails

────────────────────────────────────────────────────────────────────────────────────
  DESARROLLO DEL CAPÍTULO — NAMESPACES
────────────────────────────────────────────────────────────────────────────────────

En Ruby, los módulos actúan como espacios de nombres. Puedes anidar módulos
y clases dentro de otros módulos para crear una jerarquía organizativa clara.

    module Empresa
      module Ventas
        class Cliente
          # ...
        end
      end

      module Finanzas
        class Cliente
          # ...
        end
      end
    end

# Las dos clases Cliente son diferentes:
# Empresa::Ventas::Cliente  ≠  Empresa::Finanzas::Cliente

La resolución de constantes sigue reglas específicas que combinan el
anidamiento léxico (dónde está definido el código) y la cadena de herencia.

Ejemplo 1: Definición de namespaces anidados.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 45_01_namespaces_basico.rb
# Definición básica de espacios de nombres

module RedSocial
  VERSION = "2.0.0"

  class Usuario
    attr_reader :nombre, :email

    def initialize(nombre, email)
      @nombre = nombre
      @email = email
    end

    def to_s
      "RS::Usuario: #{@nombre} <#{@email}>"
    end
  end

  class Publicacion
    attr_reader :autor, :contenido, :fecha

    def initialize(autor, contenido)
      @autor = autor
      @contenido = contenido
      @fecha = Time.now
    end

    def to_s
      "#{@autor.nombre} dijo: #{@contenido[0..30]}..."
    end
  end

  module Privacidad
    NIVEL_PUBLICO = "publico"
    NIVEL_PRIVADO = "privado"
    NIVEL_AMIGOS = "amigos"
  end

  module Moderacion
    class Reporte
      attr_reader :motivo, :reportado_por

      def initialize(motivo, reportado_por)
        @motivo = motivo
        @reportado_por = reportado_por
      end
    end

    class Bloqueo
      attr_reader :usuario, :duracion_horas

      def initialize(usuario, duracion_horas)
        @usuario = usuario
        @duracion_horas = duracion_horas
      end
    end
  end
end

module SistemaInterno
  VERSION = "1.0.0"

  class Usuario
    attr_reader :nombre, :rol, :ultimo_acceso

    def initialize(nombre, rol)
      @nombre = nombre
      @rol = rol
      @ultimo_acceso = Time.now
    end

    def to_s
      "SI::Usuario: #{@nombre} (#{@rol})"
    end
  end

  module Auditoria
    class Log
      attr_reader :accion, :usuario, :tiempo

      def initialize(accion, usuario)
        @accion = accion
        @usuario = usuario
        @tiempo = Time.now
      end

      def to_s
        "[#{@tiempo}] #{@usuario}: #{@accion}"
      end
    end
  end
end

# Acceso a clases dentro de namespaces
usuario_rs = RedSocial::Usuario.new("Alice", "alice@rs.com")
usuario_si = SistemaInterno::Usuario.new("admin", "administrador")

puts usuario_rs
puts usuario_si

puts "RedSocial versión: #{RedSocial::VERSION}"
puts "SistemaInterno versión: #{SistemaInterno::VERSION}"
puts "Nivel privacidad: #{RedSocial::Privacidad::NIVEL_PUBLICO}"

reporte = RedSocial::Moderacion::Reporte.new("Spam", usuario_rs)
puts "Reporte: #{reporte.motivo} por #{reporte.reportado_por.nombre}"

log = SistemaInterno::Auditoria::Log.new("Login exitoso", "admin")
puts log
```

Ejemplo 2: El operador `::` de resolución.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 45_02_operador_resolucion.rb
# El operador :: para resolver constantes

module A
  VALOR = "A"
  PI = 3.14

  module B
    VALOR = "B"

    module C
      VALOR = "C"

      def self.mostrar
        puts "VALOR: #{VALOR}"
        puts "A::VALOR: #{A::VALOR}"
        puts "A::B::VALOR: #{A::B::VALOR}"
        puts "A::B::C::VALOR: #{A::B::C::VALOR}"
        puts "::A::VALOR: #{::A::VALOR}"
        puts "PI (desde A): #{A::PI}"
        puts "PI (global): #{::Math::PI}"
      end
    end
  end
end

A::B::C.mostrar

# :: al inicio se refiere al ámbito global/top-level
puts "Objeto global: #{::Object.new}"
puts "String global: #{::String.new("hola")}"
```

```ruby
# Cómo Ruby resuelve las constantes según el anidamiento léxico

module Externo
  VALOR = "externo"

  module Interno
    VALOR = "interno"

    def self.mostrar
      puts "VALOR desde Interno: #{VALOR}"
    end
  end

  # Crear un método que muestre la resolución
  def self.mostrar
    puts "VALOR desde Externo: #{VALOR}"
  end
end

Externo.mostrar
Externo::Interno.mostrar

# ¿Qué pasa si una constante no está definida?
module Prueba
  puts "Module.nesting dentro de Prueba: #{Module.nesting}"
end

# La búsqueda de constantes va desde el nivel más interno hacia afuera
module Nivel1
  VALOR = "N1"

  module Nivel2
    VALOR = "N2"

    module Nivel3
      # VALOR no definida aquí, busca en Nivel2, luego en Nivel1
      def self.mostrar
        puts "VALOR desde N3: #{VALOR}"
        puts "Module.nesting: #{Module.nesting}"
      end
    end
  end
end

Nivel1::Nivel2::Nivel3.mostrar
```

Ejemplo 4: `Module.nesting` y cómo influye en la resolución.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 45_04_module_nesting.rb
# Module.nesting muestra la pila de módulos anidados

module A
  module B
    module C
      def self.mostrar_nesting
        puts "Nesting actual: #{Module.nesting.inspect}"
      end
    end
  end
end

A::B::C.mostrar_nesting

# Ejemplo de cómo el nesting afecta la resolución
module X
  CONSTANTE = "X"

  module Y
    CONSTANTE = "Y"

    # Aquí Module.nesting = [X::Y, X]
    def self.desde_y
      CONSTANTE  # Encuentra X::Y::CONSTANTE
    end

    # Esto busca en X::Y, luego en X
    def self.desde_y_con_absoluto
      ::X::CONSTANTE  # Fuerza búsqueda desde el nivel global
    end
  end
end

puts "Desde Y: #{X::Y.desde_y}"
puts "Desde Y (absoluto): #{X::Y.desde_y_con_absoluto}"

# Mostrar nesting en diferentes contextos
puts "\nNesting en main: #{Module.nesting.inspect}"

module M1
  puts "Nesting en M1: #{Module.nesting.inspect}"

  module M2
    puts "Nesting en M2: #{Module.nesting.inspect}"

    class C1
      puts "Nesting en M1::M2::C1: #{Module.nesting.inspect}"
    end
  end
end
```

Ejemplo 5: Convención de nombres y autoloading.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 45_05_convencion_nombres.rb
# Convención de nombres: módulos ↔ directorios

# En Rails, la convención es:
#   app/models/usuario.rb        → class Usuario
#   app/models/admin/usuario.rb  → class Admin::Usuario
#   app/models/admin/permiso.rb  → class Admin::Permiso
#
# El nombre del archivo debe coincidir con el nombre de la clase/módulo
# en snake_case, y los módulos anidados se reflejan en directorios.

module Administracion
  VERSION = "1.0.0"

  # Esto simula la carga de un archivo basada en convención
  module Cargador
    def self.cargar_clase(ruta_completa)
      # Convierte "admin/usuarios" → Admin::Usuario
      partes = ruta_completa.split('/')
      nombre_clase = partes.pop
      modulo = partes.map(&:capitalize).join('::')
      clase = (modulo.empty? ? "" : modulo + "::") + nombre_clase.capitalize
      puts "Cargando: #{clase}"
      clase
    end
  end

  class Usuario
    attr_reader :nombre, :rol

    def initialize(nombre, rol)
      @nombre = nombre
      @rol = rol
    end

    def to_s
      "Admin::Usuario: #{@nombre} (#{@rol})"
    end
  end

  module Reportes
    class Ventas
      def generar
        "Reporte de ventas"
      end
    end

    class Usuarios
      def generar
        "Reporte de usuarios"
      end
    end
  end
end

# Simulación de cómo Rails resuelve nombres
puts "Cargador: #{Administracion::Cargador.cargar_clase("admin/usuarios")}"
puts "Cargador: #{Administracion::Cargador.cargar_clase("admin/reportes/ventas")}"

# Uso normal
admin_user = Administracion::Usuario.new("root", "superadmin")
reporte = Administracion::Reportes::Ventas.new
puts admin_user
puts reporte.generar
```

Ejemplo 7: Resolución de constantes con herencia.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 45_07_resolucion_herencia.rb
# La resolución de constantes también considera la herencia

module M
  CONSTANTE = "desde M"
end

module N
  CONSTANTE = "desde N"
end

class A
  CONSTANTE = "desde A"
end

class B < A
  CONSTANTE = "desde B"
end

class C < B
  include M
  prepend N

  def mostrar
    puts "CONSTANTE: #{CONSTANTE}"
    # La constante se busca en: C, B, A, Object, Kernel, BasicObject
  end
end

C.new.mostrar

# La búsqueda es: herencia + lexica
module X
  CONST = "X"

  module Y
    CONST = "Y"

    class Z
      def mostrar
        puts "CONST: #{CONST}"
        # Busca: Z -> Y -> X -> (herencia de Z) -> Object...
      end
    end
  end
end

X::Y::Z.new.mostrar
```

```ruby
# archivo: 45_09_anidamiento_profundo.rb
# Anidamiento profundo de módulos

module Empresa
  module Ventas
    module CRM
      module Contactos
        VERSION = "1.0"

        class Cliente
          attr_reader :nombre, :email, :telefono

          def initialize(nombre, email, telefono)
            @nombre = nombre
            @email = email
            @telefono = telefono
          end

          def to_s
            "Cliente: #{@nombre} (#{@email})"
          end

          def version_modulo
            VERSION
          end
        end

        class Proveedor
          attr_reader :nombre, :contacto

          def initialize(nombre, contacto)
            @nombre = nombre
            @contacto = contacto
          end

          def to_s
            "Proveedor: #{@nombre}"
          end
        end
      end

      module Oportunidades
        class Oportunidad
          attr_reader :cliente, :valor, :etapa

          def initialize(cliente, valor)
            @cliente = cliente
            @valor = valor
            @etapa = :nueva
          end

          def avanzar
            etapas = [:nueva, :calificada, :propuesta, :negociacion, :cerrada]
            idx = etapas.index(@etapa)
            @etapa = etapas[idx + 1] if idx && idx < etapas.length - 1
          end

          def to_s
            "Oportunidad: #{@cliente.nombre} - $#{@valor} (#{@etapa})"
          end
        end
      end
    end

    module Pedidos
      class Pedido
        attr_reader :numero, :cliente, :total, :fecha

        def initialize(numero, cliente, total)
          @numero = numero
          @cliente = cliente
          @total = total
          @fecha = Time.now
        end

        def to_s
          "Pedido ##{@numero}: #{@cliente.nombre} - $#{@total}"
        end
      end
    end
  end

  module Finanzas
    VERSION = "2.0"

    module Contabilidad
      class Factura
        attr_reader :numero, :monto, :cliente

        def initialize(numero, monto, cliente)
          @numero = numero
          @monto = monto
          @cliente = cliente
        end

        def to_s
          "Factura ##{@numero}: #{@cliente} - $#{@monto}"
        end
      end
    end

    module Nominas
      class Empleado
        attr_reader :nombre, :salario, :puesto

        def initialize(nombre, salario, puesto)
          @nombre = nombre
          @salario = salario
          @puesto = puesto
        end

        def to_s
          "Empleado: #{@nombre} (#{@puesto}) - $#{@salario}/año"
        end
      end
    end
  end
end

# Crear objetos en namespaces profundos

  "ana@empresa.com",

  "Juan Pérez"
)

oportunidad = Empresa::Ventas::CRM::Oportunidades::Oportunidad.new(cliente, 50000)
oportunidad.avanzar
oportunidad.avanzar
puts oportunidad

pedido = Empresa::Ventas::Pedidos::Pedido.new("PED-2024-001", cliente, 1250)
puts pedido

factura = Empresa::Finanzas::Contabilidad::Factura.new("FAC-001", 1250, cliente.nombre)
puts factura

empleado = Empresa::Finanzas::Nominas::Empleado.new("Carlos López", 85000, "Ingeniero")

puts "\nAnidamiento:"
```

Ejemplo 10: `include` y `extend` con namespaces.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 45_10_namespace_mixins.rb
# Incluyendo y extendiendo módulos desde namespaces

module MiApp
  module Utils
    module Formateable
      def formatear_fecha(tiempo)
        tiempo.strftime("%d/%m/%Y %H:%M:%S")
      end

      def formatear_moneda(monto)
        "$#{format('%.2f', monto)}"
      end
    end

    module Loggeable
      def loggear(mensaje, nivel = :info)
        timestamp = Time.now.strftime("%Y-%m-%d %H:%M:%S")
        puts "[#{timestamp}] [#{nivel.upcase}] #{mensaje}"
      end
    end

    module Serializacion
      def to_hash
        instance_variables.each_with_object({}) do |iv, h|
          h[iv.to_s.delete('@')] = instance_variable_get(iv)
        end
      end
    end
  end

  module Models
    class Usuario
      include Utils::Formateable
      include Utils::Loggeable
      include Utils::Serializacion

      attr_reader :nombre, :email, :creado_en

      def initialize(nombre, email)
        @nombre = nombre
        @email = email
        @creado_en = Time.now
        loggear("Usuario #{@nombre} creado")
      end

      def to_s
        "Usuario: #{@nombre} (#{@email}) - creado: #{formatear_fecha(@creado_en)}"
      end
    end

    class Factura
      include Utils::Formateable
      include Utils::Loggeable
      include Utils::Serializacion

      attr_reader :numero, :total, :fecha

      def initialize(numero, total)
        @numero = numero
        @total = total
        @fecha = Time.now
        loggear("Factura ##{@numero} creada por #{formatear_moneda(@total)}")
      end

      def to_s
        "Factura ##{@numero}: #{formatear_moneda(@total)} - #{formatear_fecha(@fecha)}"
      end
    end
  end

  module Services
    class Facturador
      extend Utils::Loggeable

      def self.procesar(factura)
        loggear("Procesando factura ##{factura.numero}")
        "Factura ##{factura.numero} procesada"
      end
    end
  end
end

usuario = MiApp::Models::Usuario.new("Ana", "ana@example.com")
puts usuario
puts "Hash: #{usuario.to_hash}"

factura = MiApp::Models::Factura.new("F-001", 1500.50)
puts factura

puts MiApp::Services::Facturador.procesar(factura)
```

```ruby
# archivo: 45_11_autoload.rb
# Carga perezosa con autoload en módulos

module MiLibreria
  autoload :Parser, "mi_libreria/parser"
  autoload :Validator, "mi_libreria/validator"
  autoload :Formatter, "mi_libreria/formatter"
  autoload :Serializer, "mi_libreria/serializer"

  module Models
    autoload :User, "mi_libreria/models/user"
    autoload :Post, "mi_libreria/models/post"
    autoload :Comment, "mi_libreria/models/comment"
  end

  module Services
    autoload :Mailer, "mi_libreria/services/mailer"
    autoload :Cache, "mi_libreria/services/cache"
    autoload :Logger, "mi_libreria/services/logger"
  end

  VERSION = "1.0.0"

  def self.version
    VERSION
  end

  def self.configure
    yield Configuration.instance if block_given?
    Configuration.instance
  end

  class Configuration
    include Singleton
```

```ruby
# archivo: 45_17_module_function_namespace.rb
# module_function dentro de namespaces

module Utilidades
  module Texto
    def capitalizar_palabras(oracion)
      oracion.split.map(&:capitalize).join(' ')
    end

    def invertir(oracion)
      oracion.split.reverse.join(' ')
    end

    def contar_palabras(oracion)
      oracion.split.length
    end

    def truncar(oracion, longitud = 30)
      oracion.length > longitud ? oracion[0..longitud - 3] + "..." : oracion
    end

    def a_snake_case(camel)
      camel.gsub(/([A-Z])/) { "_#{$1}" }.downcase.gsub(/^_/, '')
    end

    def a_camel_case(snake)
      snake.split('_').map(&:capitalize).join
    end

    module_function :capitalizar_palabras, :invertir, :contar_palabras,
                   :truncar, :a_snake_case, :a_camel_case
  end

  module Numerica
    def redondear(valor, decimales = 2)
      format("%.#{decimales}f", valor).to_f
    end

    def porcentaje(valor, total)
      total > 0 ? (valor.to_f / total * 100).round(1) : 0
    end

    def rango_porcentaje(valor, min, max)
      porcentaje(valor - min, max - min)
    end

    module_function :redondear, :porcentaje, :rango_porcentaje
  end
end

puts Utilidades::Texto.a_snake_case("MiClaseRubista")
puts Utilidades::Texto.a_camel_case("mi_clase_rubista")
```

```ruby
# Construyendo un DSL interno usando namespaces

module ConfigDSL
  module Definiciones
    def configurar(&bloque)
      dsl = DSL.new
      dsl.instance_eval(&bloque)
      dsl.to_h
    end
  end

  class DSL
    def initialize
      @config = {}
    end

    def servidor(nombre, &bloque)
      @config[:servidores] ||= {}
      sv = ServidorDSL.new(nombre)
      sv.instance_eval(&bloque) if bloque
      @config[:servidores][nombre] = sv.to_h
    end

    def base_de_datos(&bloque)
      db = DatabaseDSL.new
      db.instance_eval(&bloque) if bloque
      @config[:base_de_datos] = db.to_h
    end

    def logging(&bloque)
      log = LoggingDSL.new
      log.instance_eval(&bloque) if bloque
      @config[:logging] = log.to_h
    end

    def to_h
      @config
    end
  end

  class ServidorDSL
    attr_reader :nombre

    def initialize(nombre)
      @nombre = nombre
      @config = { nombre: nombre }
    end

    def puerto(p)
      @config[:puerto] = p
    end

    def ssl(activo = true)
      @config[:ssl] = activo
    end

    def timeout(t)
      @config[:timeout] = t
    end

    def max_conexiones(n)
      @config[:max_conexiones] = n
    end

    def to_h
      @config
    end
  end

  class DatabaseDSL

    end

    def host(h)

      @config[:port] = p

    end

      @config

  end
```

────────────────────────────────────────────────────────────────────────────────────

  RESUMEN DEL CAPÍTULO
────────────────────────────────────────────────────────────────────────────────────

Los namespaces mediante módulos son esenciales para la organización del código:

  1. Los módulos actúan como espacios de nombres para clases y constantes.
  2. El operador `::` resuelve constantes en la jerarquía de módulos.
  3. `Module.nesting` muestra la pila de anidamiento léxico actual.
  4. La resolución de constantes busca primero en el nesting léxico y luego
     en la cadena de herencia.
  5. Los namespaces evitan colisiones entre nombres en proyectos grandes.
  6. La convención de directorios (Rails) refleja namespaces en el sistema
     de archivos.
  7. `autoload` permite carga perezosa de módulos y clases dentro de namespaces.
  8. Las constantes privadas por convención empiezan con `_`.

────────────────────────────────────────────────────────────────────────────────────
  FAQ — PREGUNTAS FRECUENTES
────────────────────────────────────────────────────────────────────────────────────

P: ¿Cuál es la diferencia entre `::` y `.`?
R: `::` es para acceder a constantes (clases, módulos, constantes). `.` es
   para llamar métodos. `String::NEW` vs `string.new`.

P: ¿Qué pasa si uso `::` al principio del nombre?
R: `::MiModulo` busca desde el ámbito global (top-level), ignorando el
   anidamiento actual.

P: ¿Cómo Rails sabe qué archivo cargar para `Admin::Usuario`?
R: Rails usa `const_missing` y busca `app/models/admin/usuario.rb`
   siguiendo la convención de nombres.

P: ¿Puedo tener un módulo y una clase con el mismo nombre?
R: Técnicamente no, porque una clase es un módulo. Pero puedes tener
   `MiModulo::MiClase` y `Otra::MiClase`.

P: ¿Es buena práctica anidar muy profundamente?
R: Generalmente no más de 3-4 niveles. La profundidad excesiva hace el
   código difícil de leer y mantener.

P: ¿Las constantes en módulos son accesibles globalmente?
R: Sí, cualquier constante definida dentro de un módulo es accesible
   desde fuera usando el operador `::`.

  DESAFÍO DEL ARQUITECTO
────────────────────────────────────────────────────────────────────────────────────

Diseña e implementa un sistema completo de "API REST Framework" usando
namespaces como estructura organizativa principal. El sistema debe tener:

  1. `API::Router` — enruta peticiones a controladores
  2. `API::Controllers::Base` — clase base para controladores
  3. `API::Controllers::V1::*` — controladores de versión 1
  4. `API::Controllers::V2::*` — controladores de versión 2
  5. `API::Models::*` — modelos de datos
  6. `API::Serializers::*` — serializadores para cada modelo
  7. `API::Middleware::*` — middleware (auth, logging, cors)
  8. `API::Config::*` — configuración por entorno
  9. `API::Errors::*` — clases de error personalizadas
  10. `API::Validators::*` — validadores de parámetros

Cada namespace debe tener sus propias constantes y clases, sin colisiones
con otros namespaces. Implementa al menos 3 endpoints funcionales que
demuestren cómo los namespaces organizan el código.

El sistema debe demostrar:
- Organización jerárquica clara
- Separación de responsabilidades por namespace
- Convención de nombres consistente
- Facilidad de navegación y extensión

Este desafío integra todos los conceptos de namespaces: anidamiento,
resolución de constantes, organización de archivos y diseño de APIs.

────────────────────────────────────────────────────────────────────────────────────
  CIERRE DEL CAPÍTULO
────────────────────────────────────────────────────────────────────────────────────

Los namespaces son la herramienta organizativa definitiva en Ruby. Con ellos
puedes estructurar grandes codebases de manera lógica, evitar colisiones de
nombres y seguir convenciones que hacen tu código predecible y navegable.

Ya sea que estés construyendo una gema, una aplicación Rails, o un sistema
empresarial, los namespaces te permiten mantener el orden a medida que tu
código crece. La combinación de módulos anidados, el operador `::` y la
convención de archivos es la base de la organización en el mundo Ruby.
