  BIENVENIDA
────────────────────────────────────────────────────────────────────────────────────

Bienvenido al Capítulo 43 del Ruby Master Book. Si en el capítulo anterior
aprendiste a usar `include` para añadir métodos de instancia a una clase,
ahora exploraremos `extend`, su contraparte que añade métodos de módulo
directamente a un objeto.

La forma más común de usar `extend` es dentro de una clase para añadir métodos
de clase. Pero su verdadero poder está en que `extend` funciona sobre cualquier
objeto, permitiéndote añadir comportamiento singleton a instancias individuales.

En este capítulo veremos la diferencia fundamental entre `include` y `extend`,
cómo usar `extend` para crear métodos de clase, cómo extender objetos
individuales, y patrones avanzados como fábricas, decoradores y el mixin
de clase auto-contenido.

────────────────────────────────────────────────────────────────────────────────────
  DESARROLLO DEL CAPÍTULO — EXTEND
────────────────────────────────────────────────────────────────────────────────────

La directiva `extend` añade los métodos de un módulo como métodos singleton
del objeto receptor. Cuando se usa dentro de una definición de clase, los
métodos del módulo se convierten en métodos de clase. Cuando se usa sobre
una instancia, los métodos se convierten en métodos singleton de esa instancia.

Sintaxis:

    # Como declaración dentro de una clase
    class MiClase
      extend MiModulo    # métodos de módulo → métodos de clase
    end

    # Sobre un objeto cualquiera
    objeto.extend(MiModulo)  # métodos de módulo → métodos singleton

Internamente, `extend` es equivalente a `objeto.singleton_class.include(modulo)`.

```ruby
# archivo: 43_01_extend_clase.rb
# Usando extend para añadir métodos de clase

module MetodosDeClase
  def version
    "1.0.0"
  end

  def autor
    "Ruby Master Book"
  end

  def descripcion
    "Clase manejada por #{name}"
  end

  def configurar(&bloque)
    @configuracion ||= {}
    bloque.call(@configuracion) if bloque
    @configuracion
  end

  def configuracion
    @configuracion || {}
  end
end

class Producto
  extend MetodosDeClase

  attr_reader :nombre, :precio

  def initialize(nombre, precio)
    @nombre = nombre
    @precio = precio
  end

  def to_s
    "#{@nombre}: $#{@precio}"
  end
end

# Llamando métodos de clase definidos por el módulo
puts Producto.version
puts Producto.autor
puts Producto.descripcion

Producto.configurar do |conf|
  conf[:impuesto] = 0.16
  conf[:descuento_max] = 0.5
end

puts "Configuración: #{Producto.configuracion}"
```

```ruby
    eventos.last
```

```ruby
# archivo: 43_04_include_extend.rb
# El patrón clásico: include para instancia, extend para clase

module Persistible
  def self.included(base)
    base.extend(ClassMethods)
  end

  module ClassMethods
    def buscar_por_id(id)
      puts "#{name}: Buscando por id=#{id}"
      # Simulación de búsqueda
      new(id: id, nombre: "Auto-generado")
    end

    def todos
      puts "#{name}: Obteniendo todos los registros"
      []
    end

    def count
      puts "#{name}: Contando registros"
      0
    end

    def crear(atributos = {})
      nuevo = new(atributos)
      puts "#{name}: Objeto creado: #{nuevo}"
      nuevo
    end

    def schema
      @schema || {}
    end

    def field(nombre, tipo)
      @schema ||= {}
      @schema[nombre] = tipo
      attr_accessor nombre
    end
  end

  def guardar
    puts "#{self.class}: Objeto guardado: #{self}"
    true
  end

  def eliminar
    puts "#{self.class}: Objeto eliminado: #{self}"
    true
  end

  def valido?
    true
  end

  def to_h
    instance_variables.each_with_object({}) do |iv, h|
      h[iv.to_s.delete('@')] = instance_variable_get(iv)
    end
  end
end

class Usuario
  include Persistible

  field :nombre, String
  field :email, String
  field :edad, Integer

  def initialize(atributos = {})
    @nombre = atributos[:nombre]
    @email = atributos[:email]
    @edad = atributos[:edad]
  end

  def to_s
    "Usuario(#{@nombre}, #{@email}, #{@edad})"
  end
end

class Articulo
  include Persistible

  field :titulo, String
  field :contenido, String
  field :publicado, [TrueClass, FalseClass]

  def initialize(atributos = {})
    @titulo = atributos[:titulo]
    @contenido = atributos[:contenido]
    @publicado = atributos[:publicado] || false
  end

  def to_s
    "Artículo(#{@titulo}, publicado=#{@publicado})"
  end
end

# Métodos de clase vía extend
u = Usuario.crear(nombre: "Alice", email: "alice@test.com", edad: 30)
Usuario.buscar_por_id(42)
Usuario.todos

puts ""
a = Articulo.crear(titulo: "Ruby Extend", contenido: "Contenido...", publicado: true)
puts "Schema Usuario: #{Usuario.schema}"

# Métodos de instancia vía include
puts u.to_h
```

Ejemplo 5: Fábrica con `extend`.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 43_05_fabrica_extend.rb
# Patrón fábrica usando extend para crear objetos especializados

module ConLog
  def loggear(mensaje)
    puts "[#{Time.now.strftime('%H:%M:%S')}] #{self.class}: #{mensaje}"
  end
end

module ConValidacion
  def validar_presencia(atributo)
    valor = instance_variable_get("@#{atributo}")
    unless valor && !valor.to_s.empty?
      raise ArgumentError, "#{atributo} no puede estar vacío"
    end
  end

  def validar_numerico(atributo)
    valor = instance_variable_get("@#{atributo}")
    unless valor.is_a?(Numeric)
      raise TypeError, "#{atributo} debe ser numérico"
    end
  end
end

module ConSerializacion
  def to_json
    require 'json'
    JSON.generate(to_h)
  end

  def to_h
    instance_variables.each_with_object({}) do |iv, h|
      h[iv.to_s.delete('@')] = instance_variable_get(iv)
    end
  end

  def to_yaml
    require 'yaml'
    YAML.dump(to_h)
  end
end

class FabricaDeObjetos
  def self.crear(tipo, atributos = {})
    case tipo
    when :usuario
      obj = ClaseBase.new(atributos)
      obj.extend(ConLog)
      obj.extend(ConValidacion)
      obj.extend(ConSerializacion)
      obj
    when :producto
      obj = ClaseBase.new(atributos)
      obj.extend(ConLog)
      obj.extend(ConSerializacion)
      obj
    when :evento
      obj = ClaseBase.new(atributos)
      obj.extend(ConLog)
      obj
    else
      raise ArgumentError, "Tipo desconocido: #{tipo}"
    end
  end
end

class ClaseBase
  attr_reader :atributos

  def initialize(atributos = {})
    @atributos = atributos
    atributos.each do |k, v|
      instance_variable_set("@#{k}", v)
    end
  end

  def method_missing(name, *args, &block)
    attr = name.to_s.delete('@')
    if instance_variable_defined?("@#{attr}")
      instance_variable_get("@#{attr}")
    else
      super
    end
  end

  def respond_to_missing?(name, include_private = false)
    instance_variable_defined?("@#{name.to_s.delete('@')}") || super
  end

  def to_s
    "Objeto(#{@atributos})"
  end
end

usuario = FabricaDeObjetos.crear(:usuario, nombre: "Ana", email: "ana@test.com", edad: 28)
producto = FabricaDeObjetos.crear(:producto, codigo: "PROD-001", precio: 99.99)
evento = FabricaDeObjetos.crear(:evento, tipo: "login", usuario_id: 1)

usuario.loggear("Creado")
puts usuario.to_json

producto.loggear("Creado")
puts producto.to_h

evento.loggear("Evento registrado")

begin
  usuario.validar_presencia(:nombre)
  usuario.validar_numerico(:edad)
  puts "Validaciones pasadas"
rescue => e
  puts "Error: #{e.message}"
end
```

Ejemplo 6: Decorador dinámico con `extend`.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 43_06_decorador_extend.rb
# Patrón decorador usando extend para añadir comportamiento dinámico

module ConTiempo
  def ejecutar(*args)
    inicio = Time.now
    resultado = super(*args)
    duracion = Time.now - inicio
    puts "[TIMER] #{duracion * 1000}ms"
    resultado
  end
end

module ConLog
  def ejecutar(*args)
    puts "[LOG] Llamando a #{self.class}##{__method__} con #{args.inspect}"
    super(*args)
  rescue => e
    puts "[LOG] Error: #{e.message}"
    raise
  end
end

module ConCache
  def ejecutar(*args)
    @cache ||= {}
    clave = args.hash
    if @cache.key?(clave)
      puts "[CACHE] Hit para #{args.inspect}"
      @cache[clave]
    else
      puts "[CACHE] Miss para #{args.inspect}"
      @cache[clave] = super(*args)
    end
  end
end

module ConValidacion
  def ejecutar(*args)
    raise ArgumentError, "Se requieren argumentos" if args.empty?
    super(*args)
  end
end

class Calculadora
  def ejecutar(operacion, a, b)
    case operacion
    when :suma then a + b
    when :resta then a - b
    when :multiplicacion then a * b
    when :division then raise("División por cero") if b == 0; a / b
    else raise "Operación desconocida"
    end
  end

  def status
    "Calculadora básica"
  end
end

calc = Calculadora.new
calc.extend(ConValidacion)
calc.extend(ConLog)
calc.extend(ConTiempo)
calc.extend(ConCache)

puts "Resultado: #{calc.ejecutar(:suma, 5, 3)}"
puts "Resultado (cacheado): #{calc.ejecutar(:suma, 5, 3)}"
puts "Resultado: #{calc.ejecutar(:multiplicacion, 4, 7)}"

begin
  calc.ejecutar(:division, 10, 0)
rescue => e
  puts "Error capturado: #{e.message}"
end

begin
  calc.ejecutar()
rescue => e
  puts "Error de validación: #{e.message}"
end
```

Ejemplo 7: `extend` con módulos de clase para DSL.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 43_07_extend_dsl.rb
# Usando extend para construir un DSL interno

module DSLBuilder
  def attribute(nombre, tipo = nil, &bloque)
    @attributes ||= []
    @attributes << { nombre: nombre, tipo: tipo }

    attr_reader nombre

    define_method("#{nombre}=") do |valor|
      if tipo && !valor.is_a?(tipo)
        raise TypeError, "#{nombre} debe ser #{tipo}"
      end
      instance_variable_set("@#{nombre}", valor)
    end

    if bloque
      define_method(nombre) do |*args|
        if args.empty?
          instance_variable_get("@#{nombre}")
        else
          send("#{nombre}=", args.first)
        end
      end
    end
  end

  def has_many(nombre, clase = nil, &bloque)
    @associations ||= []
    @associations << { nombre: nombre, clase: clase }

    define_method(nombre) do
      instance_variable_get("@#{nombre}") || []
    end

    define_method("#{nombre}=") do |valor|
      instance_variable_set("@#{nombre}", valor)
    end

    define_method("agregar_#{nombre.to_s.singularize}") do |elemento|
      coleccion = send(nombre)
      coleccion << elemento
      instance_variable_set("@#{nombre}", coleccion)
    end
  end

  def validates(nombre, opciones = {})
    @validaciones ||= []
    @validaciones << { campo: nombre, opciones: opciones }

    define_method("validar_#{nombre}") do
      valor = send(nombre)
      errores = []
      errores << "#{nombre} requerido" if opciones[:presence] && (valor.nil? || valor.to_s.empty?)
      if opciones[:length]
        min = opciones[:length][:min] || 0
        max = opciones[:length][:max] || Float::INFINITY
        if valor && (valor.length < min || valor.length > max)
          errores << "#{nombre} longitud entre #{min} y #{max}"
        end
      end
      errores
    end
  end

  def attributes_list
    @attributes || []
  end

  def associations
    @associations || []
  end

  def validaciones
    @validaciones || []
  end
end

class Post
  extend DSLBuilder

  attribute :titulo, String
  attribute :contenido, String
  attribute :visitas, Integer
  has_many :comentarios
  validates :titulo, presence: true, length: { min: 3, max: 100 }
  validates :contenido, presence: true
end

post = Post.new
post.titulo = "Ruby Extend es Poderoso"
post.contenido = "Contenido del artículo..."
post.visitas = 42

puts "Título: #{post.titulo}"
puts "Contenido: #{post.contenido}"
puts "Visitas: #{post.visitas}"

com1 = "¡Gran artículo!"
com2 = "Muy útil, gracias."
post.agregar_comentario(com1)
post.agregar_comentario(com2)
puts "Comentarios: #{post.comentarios.inspect}"

puts "\nAttributes: #{Post.attributes_list}"
puts "Validaciones: #{Post.validaciones}"
puts "Associations: #{Post.associations}"
```

```ruby
# Creando clases anónimas con comportamiento específico vía extend

module Autenticable
  def autenticar(usuario, password)
    usuario == @usuario && password == @password
  end

  def cambiar_password(viejo, nuevo)
    if autenticar(@usuario, viejo)
      @password = nuevo
      true
    else
      false
    end
  end
end

module Auditable
  def auditar(accion)
    @auditoria ||= []
    @auditoria << { accion: accion, tiempo: Time.now, usuario: @usuario }
  end

  def auditoria
    @auditoria || []
  end

  def ultima_auditoria
    auditoria.last
  end
end

# Crear un objeto singleton con múltiples comportamientos
servicio = Object.new
servicio.extend(Autenticable)
servicio.extend(Auditable)

# Configurar como si fuera un objeto de clase
```

Ejemplo 9: `extend` en lugar de herencia.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 43_09_extend_vs_herencia.rb
# Usando extend como alternativa a herencia para objetos únicos

module Exportable
  def exportar(formato = :csv)
    case formato
    when :csv then exportar_csv
    when :json then exportar_json
    when :xml then exportar_xml
    else raise "Formato no soportado: #{formato}"
    end
  end

  def exportar_csv
    headers = instance_variables.map { |iv| iv.to_s.delete('@') }
    values = instance_variables.map { |iv| instance_variable_get(iv) }
    "#{headers.join(',')}\n#{values.join(',')}"
  end

  def exportar_json
    require 'json'
    hash = instance_variables.each_with_object({}) do |iv, h|
      h[iv.to_s.delete('@')] = instance_variable_get(iv)
    end
    JSON.generate(hash)
  end

  def exportar_xml
    buffer = "<#{self.class.name}>\n"
    instance_variables.each do |iv|
      nombre = iv.to_s.delete('@')
      valor = instance_variable_get(iv)
      buffer += "  <#{nombre}>#{valor}</#{nombre}>\n"
    end
    buffer += "</#{self.class.name}>"
    buffer
  end
end

# En lugar de crear una clase hija para cada tipo de exportación,
# extendemos objetos individuales
class Reporte
  def initialize(nombre, datos)
    @nombre = nombre
    @datos = datos
    @creado_en = Time.now
  end

  def mostrar
    puts "Reporte: #{@nombre}"
    puts "Datos: #{@datos}"
    puts "Creado: #{@creado_en}"
  end
end

reporte_csv = Reporte.new("Ventas", [100, 200, 150])
reporte_csv.extend(Exportable)

reporte_json = Reporte.new("Usuarios", { activos: 50, total: 100 })
reporte_json.extend(Exportable)

puts "=== CSV ==="
puts reporte_csv.exportar(:csv)

puts "\n=== JSON ==="
puts reporte_json.exportar(:json)

puts "\n=== XML ==="
puts reporte_json.exportar(:xml)
```

Ejemplo 10: `extend` con `self.included` para herencia de clase completa.

────────────────────────────────────────────────────────────────────────────────────

```ruby
# archivo: 43_10_extend_completo.rb
# Módulo que añade métodos de instancia Y de clase simultáneamente

module ModeloCompleto
  def self.included(base)
    base.extend(ClassMethods)
    base.instance_variable_set(:@registros, [])
  end

  module ClassMethods
    def registros
      @registros || []
    end

    def crear(atributos)
      nuevo = new(atributos)
      @registros << nuevo
      nuevo.guardar if nuevo.respond_to?(:guardar)
      nuevo
    end

    def buscar_por(campo, valor)
      registros.find { |r| r.send(campo) == valor }
    end

    def buscar_todos_por(campo, valor)
      registros.select { |r| r.send(campo) == valor }
    end

    def count
      registros.length
    end

    def primero
      registros.first
    end

    def ultimo
      registros.last
    end

    def sample
      registros.sample
    end

    def cada(&bloque)
      registros.each(&bloque)
    end

    def map(&bloque)
      registros.map(&bloque)
    end

    def select(&bloque)
      registros.select(&bloque)
    end

    def where(**condiciones)
      registros.select do |r|
        condiciones.all? { |campo, valor| r.send(campo) == valor }
      end
    end

    def ordenar_por(campo, direccion = :asc)
      registros.sort do |a, b|
        compare = a.send(campo) <=> b.send(campo)
        direccion == :asc ? compare : -compare
      end
    end

    def to_s

  end

    end
  end

  def inspect
    "#{self.class}(#{atributos})"
  end
end
```

```ruby
  def profundidad

  def fusionar(*otros)
    otros.each { |otro| merge!(otro) }
    self
```

────────────────────────────────────────────────────────────────────────────────────

  RESUMEN DEL CAPÍTULO
────────────────────────────────────────────────────────────────────────────────────

`extend` es una herramienta versátil para añadir comportamiento en Ruby:

  1. `extend` en una clase convierte métodos de módulo en métodos de clase.
  2. `extend` en una instancia añade métodos singleton a ese objeto.
  3. `extend` es equivalente a `singleton_class.include(modulo)`.
  4. El patrón `include + extend` con `included` proporciona métodos de
     instancia y de clase desde un solo módulo.
  5. `extend` permite decorar objetos individuales sin afectar a la clase.
  6. Es una alternativa limpia a la herencia para objetos únicos.
  7. No se puede "des-extender" un objeto, pero se puede limpiar estado.

────────────────────────────────────────────────────────────────────────────────────
  FAQ — PREGUNTAS FRECUENTES
────────────────────────────────────────────────────────────────────────────────────

P: ¿Puedo usar `include` y `extend` con el mismo módulo?
R: Sí, y es un patrón muy común. El módulo se incluye para métodos de
   instancia y se extiende (o se usa el callback `included`) para clase.

P: ¿Puedo extender un objeto después de crearlo?
R: Sí, esa es una de las características más poderosas de Ruby. Cualquier
   objeto puede ser extendido en cualquier momento.

P: ¿Extender un objeto afecta a otros objetos de la misma clase?
R: No. `extend` sobre una instancia solo afecta a esa instancia en particular.

P: ¿Cómo "des-extiendo" un objeto?
R: No hay una forma directa. Ruby no tiene `unextend`. Una alternativa es
   usar delegación o crear un nuevo objeto.

P: ¿Cuál es la diferencia entre `extend` y `include`?
R: `include` añade métodos de instancia. `extend` añade métodos singleton
   (de clase cuando se usa en la definición de una clase).

P: ¿`extend` funciona con objetos frozen?
R: No. Los objetos frozen no pueden ser modificados, incluyendo extend.

  3. La clase `PluginManager` debe:
     - Mantener un registro de plugins cargados
     - Proveer `load_plugin(plugin_module, target)`
     - Proveer `unload_plugin(plugin_module, target)` (simulado)
     - Proveer `plugins_activos` y `plugin_status`

  4. Cada plugin debe poder configurarse con opciones al cargarse.

  5. Implementa el patrón `include + extend` donde el plugin aporta tanto
     métodos de instancia como de clase al target.

  6. El sistema debe soportar que un target tenga múltiples plugins
     activos simultáneamente.

Este desafío explora `extend` en profundidad, combinándolo con `include`,
manejo de estado, y diseño de sistemas extensibles. Es similar a cómo
funcionan los sistemas de plugins en frameworks reales como Rails.

────────────────────────────────────────────────────────────────────────────────────
  CIERRE DEL CAPÍTULO
────────────────────────────────────────────────────────────────────────────────────

`extend` te ha mostrado cómo añadir comportamiento donde y cuando lo necesites.
Ya sea para dotar a una clase de métodos de utilidad, para decorar objetos
individuales, o para construir sistemas extensibles de plugins, `extend` es
una herramienta elegante y poderosa.

Recuerda: `include` es para comportamiento compartido entre instancias,
`extend` es para comportamiento específico de un objeto o clase. Domina
ambos y tendrás un control total sobre la organización de tu código.

  PREVIEW DEL PRÓXIMO CAPÍTULO
────────────────────────────────────────────────────────────────────────────────────

En el Capítulo 44: Prepend, exploraremos la tercera directiva de mezcla de
módulos en Ruby. `prepend` es como `include` pero invierte el orden en la
cadena de ancestros: el módulo se inserta antes que la clase, permitiendo
que los métodos del módulo sobrescriban a los de la clase. Esto abre la
puerta a patrones avanzados como Aspect-Oriented Programming, alias_method
chain, y monkey-patching controlado.
