================================================================================

  BIENVENIDA

  Hasta ahora hemos trabajado con métodos de instancia: comportamientos
  que pertenecen a objetos individuales. Pero hay comportamientos que
  pertenecen a la clase misma, no a sus instancias. Por ejemplo: crear
  objetos (new es un método de clase), contar instancias, o proporcionar
  puntos de acceso globales.

  Los métodos de clase en Ruby se pueden definir de varias formas:
  self.metodo, class << self, o extendiendo módulos. También existen
  las variables de clase @@var, que se comparten entre la clase, sus
  instancias y sus subclases.

  En este capítulo exploraremos todas las formas de definir métodos de
  clase, cuándo usarlos, y cómo las variables de clase pueden ser útiles
  (y peligrosas). También veremos patrones como el Singleton Pattern
  y los Class Methods Factory.

  FRASE DEL CAPÍTULO

  1. ¿QUÉ SON LOS MÉTODOS DE CLASE?

  Los métodos de clase son métodos que se llaman sobre la clase misma,
  no sobre una instancia. Se definen añadiendo self. al nombre del
  método, o usando la sintaxis class << self.

  Como las clases son objetos (instancias de Class), los métodos de
  clase son simplemente métodos de instancia de la clase Class, o más
  precisamente, métodos almacenados en la singleton_class de la clase.

  ------------------------------------------------------------------
  Ejemplo 1: Métodos de clase básicos con self.
  ------------------------------------------------------------------

  class Matemáticas
    def self.sumar(a, b)
      a + b
    end

    def self.restar(a, b)
      a - b
    end

    def self.pi
      3.141592653589793
    end
  end

  # Llamada: se invocan sobre la clase, no sobre una instancia
  puts Matemáticas.sumar(5, 3)    # => 8
  puts Matemáticas.restar(10, 4)  # => 6
  puts Matemáticas.pi             # => 3.141592653589793

  # No se puede llamar sobre una instancia:
  # m = Matemáticas.new
  # m.sumar(1, 2)  # => NoMethodError

  2. MÚLTIPLES FORMAS DE DEFINIR MÉTODOS DE CLASE

  Ruby ofrece varias sintaxis para definir métodos de clase. Todas son
  equivalentes, pero cada una tiene sus ventajas contextuales.

  ------------------------------------------------------------------
  Ejemplo 2: Las 5 formas de definir métodos de clase
  ------------------------------------------------------------------

  class Ejemplo
    # Forma 1: self.nombre (la más común)
    def self.metodo1
      "Forma 1: self.metodo"
    end

    # Forma 2: class << self (bloque de clase singleton)
    class << self
      def metodo2
        "Forma 2: class << self"
      end

      def metodo3
        "Forma 3: otro dentro de class << self"
      end
    end

    # Forma 4: definición separada con el nombre completo
    def Ejemplo.metodo4
      "Forma 4: NombreClase.metodo"
    end
  end

  # Forma 5: después de la clase, usando class << self
  class << Ejemplo
    def metodo5
      "Forma 5: class << NombreClase externo"
    end
  end

  puts Ejemplo.metodo1  # => Forma 1: self.metodo
  puts Ejemplo.metodo2  # => Forma 2: class << self
  puts Ejemplo.metodo3  # => Forma 3: otro dentro de class << self
  puts Ejemplo.metodo4  # => Forma 4: NombreClase.metodo
  puts Ejemplo.metodo5  # => Forma 5: class << NombreClase externo

  La forma 1 (self.metodo) es la más recomendada por claridad. La
  forma 2 (class << self) es útil cuando tienes muchos métodos de clase.

================================================================================

  3. CLASS << SELF EN PROFUNDIDAD

  class << self abre la clase singleton (o clase eigen) de la clase
  actual. Todo lo que definas dentro se convierte en métodos de clase.

  ------------------------------------------------------------------
  Ejemplo 3: class << self en detalle
  ------------------------------------------------------------------

  class LoggerSistema
    class << self
      def nivel=(nivel)
        @nivel = nivel
      end

      def nivel
        @nivel || "INFO"
      end

      def log(mensaje)
        puts "[#{nivel}] #{Time.now}: #{mensaje}"
      end

      def info(mensaje)
        log("INFO", mensaje)
      end

      def error(mensaje)
        log("ERROR", mensaje)
      end

      def warn(mensaje)
        log("WARN", mensaje)
      end

      private

      def log(nivel, mensaje)
        puts "[#{nivel}] #{Time.now.strftime('%H:%M:%S')}: #{mensaje}"
      end
    end
  end

  LoggerSistema.nivel = "DEBUG"
  LoggerSistema.info("Sistema iniciado")
  LoggerSistema.error("Error de conexión")
  # LoggerSistema.log("INFO", "test")  # Error! private

  Dentro de class << self:
  - self es la clase misma
  - Los métodos definidos son métodos de clase
  - public/private/protected funcionan igual

  4. NEW COMO MÉTODO DE CLASE

  El método new que hemos usado desde el capítulo 31 es un método de
  clase definido por Class. Podemos sobreescribirlo o decorarlo.

  ------------------------------------------------------------------
  Ejemplo 4: Personalizando new
  ------------------------------------------------------------------

  class Usuario
    attr_reader :nombre, :email

    def initialize(nombre, email)
      @nombre = nombre
      @email = email
    end

    # Decoramos new para añadir logging
    def self.new(nombre, email)
      puts "Creando usuario: #{nombre}"
      instance = allocate  # Asigna memoria sin llamar a initialize
      instance.send(:initialize, nombre, email)
      instance
    end
  end

  # u = Usuario.new("Ana", "ana@ejemplo.com")
  # => Creando usuario: Ana

  # Otra forma: hook after_initialize
  class Producto
    attr_reader :nombre

    def initialize(nombre)
      @nombre = nombre
      after_initialize
    end

    def self.new(*args)
      instance = allocate
      instance.send(:initialize, *args)
      instance
    end

    private

    def after_initialize
      puts "Producto '#{@nombre}' creado"
    end
  end

  5. MÉTODOS DE CLASE PRIVADOS

  Los métodos de clase también pueden ser privados, accesibles solo
  desde otros métodos de clase.

  ------------------------------------------------------------------
  Ejemplo 5: Métodos de clase privados
  ------------------------------------------------------------------

  class ConexionBD
    class << self
      def conectar
        config = cargar_configuracion
        puts "Conectando a #{config[:host]}:#{config[:puerto]}..."
        new(config)
      end

      def config
        @config ||= cargar_configuracion
      end

      private

      def cargar_configuracion
        puts "Cargando configuración..."
        { host: "localhost", puerto: 5432, usuario: "admin", password: "secret" }
      end
    end

    def initialize(config)
      @config = config
      puts "Conexión establecida"
    end
  end

  db = ConexionBD.conectar
  # => Cargando configuración...
  # => Conectando a localhost:5432...
  # => Conexión establecida

  # ConexionBD.cargar_configuracion  # Error! private method

  6. VARIABLES DE CLASE (@@)

  Las variables de clase (prefijo @@) se comparten entre la clase, todas
  sus instancias y todas sus subclases. Son útiles para datos que
  pertenecen a la jerarquía completa, como contadores de instancias.

  ------------------------------------------------------------------
  Ejemplo 6: Variables de clase para contador de instancias
  ------------------------------------------------------------------

  class Persona
    @@contador = 0

    def initialize(nombre)
      @nombre = nombre
      @@contador += 1
      puts "Persona ##{@@contador}: #{@nombre}"
    end

    def self.total_personas
      @@contador
    end
  end

  p1 = Persona.new("Ana")   # => Persona #1: Ana
  p2 = Persona.new("Luis")  # => Persona #2: Luis
  p3 = Persona.new("Carla") # => Persona #3: Carla

  puts Persona.total_personas  # => 3

  ------------------------------------------------------------------
  Ejemplo 7: Variables de clase y herencia (cuidado!)
  ------------------------------------------------------------------

  class Vehiculo
    @@count = 0

    def initialize
      @@count += 1
    end

    def self.count
      @@count
    end
  end

  class Coche < Vehiculo
  end

  class Moto < Vehiculo
  end

  Vehiculo.new
  Coche.new
  Moto.new
  Coche.new

  puts Vehiculo.count  # => 4 (no 1!)
  puts Coche.count     # => 4
  puts Moto.count      # => 4

  @@count se comparte en toda la jerarquía. Si queremos contadores
  separados por clase, necesitamos variables de instancia de clase.

================================================================================

  7. VARIABLES DE INSTANCIA DE CLASE

  Las variables de instancia de clase (@) se definen a nivel de clase
  (no dentro de métodos de instancia). Pertenecen exclusivamente a la
  clase, no se comparten con subclases.

  ------------------------------------------------------------------
  Ejemplo 8: Variables de instancia de clase
  ------------------------------------------------------------------

  class Vehiculo
    @count = 0

    class << self
      attr_accessor :count
    end

    def initialize
      self.class.count += 1
    end
  end

  class Coche < Vehiculo
    @count = 0

    class << self
      attr_accessor :count
    end
  end

  class Moto < Vehiculo
    @count = 0

    class << self
      attr_accessor :count
    end
  end

  Vehiculo.new
  Coche.new
  Moto.new
  Coche.new

  puts Vehiculo.count  # => 1
  puts Coche.count     # => 2
  puts Moto.count      # => 1

  Cada clase tiene su propio @count. Esto es más seguro que @@.

  ------------------------------------------------------------------
  Ejemplo 9: Acceso a variables de instancia de clase
  ------------------------------------------------------------------

  class ConfiguracionApp
    @valores = {}

    class << self
      def set(key, value)
        @valores[key] = value
      end

      def get(key)
        @valores[key]
      end

      def valores
        @valores.dup
      end
    end
  end

  class ConfiguracionHija < ConfiguracionApp
    @valores = {}  # Su propio hash, independiente
  end

  ConfiguracionApp.set(:host, "localhost")
  ConfiguracionApp.set(:puerto, 8080)

  ConfiguracionHija.set(:host, "192.168.1.1")

  puts ConfiguracionApp.get(:host)     # => localhost
  puts ConfiguracionHija.get(:host)    # => 192.168.1.1

  8. PATRÓN SINGLETON CON MÉTODOS DE CLASE

  El patrón Singleton asegura que una clase tenga solo una instancia y
  proporciona un punto de acceso global a ella. Ruby lo simplifica con
  el módulo Singleton de la stdlib.

  ------------------------------------------------------------------
  Ejemplo 10: Singleton manual con métodos de clase
  ------------------------------------------------------------------

  class ConfiguracionGlobal
    @instancia = nil

    class << self
      def instance
        @instancia ||= new
      end

      private :new, :allocate
    end

    def initialize
      @valores = {
        host: "localhost",
        puerto: 3000,
        debug: false
      }
    end

    def get(key)
      @valores[key]
    end

    def set(key, value)
      @valores[key] = value
    end

    def to_s
      @valores.inspect
    end
  end

  c1 = ConfiguracionGlobal.instance
  c2 = ConfiguracionGlobal.instance

  puts c1 == c2       # => true (mismo objeto)
  c1.set(:debug, true)
  puts c2.get(:debug) # => true

  # ConfiguracionGlobal.new  # Error! private method new

  ------------------------------------------------------------------
  Ejemplo 11: Singleton con la gema estándar
  ------------------------------------------------------------------

  require 'singleton'

  class LoggerGlobal
    include Singleton

    def initialize
      @log = []
    end

    def info(mensaje)
      escribir("[INFO]", mensaje)
    end

    def error(mensaje)
      escribir("[ERROR]", mensaje)
    end

    def leer
      @log.dup
    end

    private

    def escribir(nivel, mensaje)
      entrada = "#{Time.now.strftime('%H:%M:%S')} #{nivel}: #{mensaje}"
      @log << entrada
      puts entrada
    end
  end

  log = LoggerGlobal.instance
  log.info("Sistema iniciado")
  log.error("Fallo crítico")

  log2 = LoggerGlobal.instance
  puts log2.leer.size  # => 2

  # include Singleton hace que new sea privado automáticamente

  9. MÉTODOS DE CLASE HEREDADOS

  Los métodos de clase se heredan como cualquier otro método. Si
  defines un método de clase en una superclase, la subclase puede
  llamarlo.

  ------------------------------------------------------------------
  Ejemplo 12: Herencia de métodos de clase
  ------------------------------------------------------------------

  class Animal
    def self.especies
      @especies ||= []
    end

    def self.heredero
      especies << self
    end

    def self.total_especies
      especies.size
    end
  end

  class Perro < Animal
    heredero
  end

  class Gato < Animal
    heredero
  end

  class Pajaro < Animal
    heredero
  end

  puts Animal.total_especies  # => 3
  puts Animal.especies.inspect
  # => [Perro, Gato, Pajaro]

  ------------------------------------------------------------------
  Ejemplo 13: self.nombre en subclases
  ------------------------------------------------------------------

  class Documento
    def self.formato
      :texto
    end

    def self.crear(titulo, contenido)
      new(titulo, contenido)
    end
  end

  class PDF < Documento
    def self.formato
      :pdf
    end
  end

  class HTML < Documento
    def self.formato
      :html
    end
  end

  puts Documento.formato  # => texto
  puts PDF.formato        # => pdf
  puts HTML.formato       # => html

  # Cada subclase puede personalizar sus propios métodos de clase.

  10. MÉTODOS DE CLASE Y GLOBALES

  Los métodos de clase pueden actuar como funciones globales organizadas.
  Es una práctica común agrupar funciones relacionadas como métodos de
  clase de un módulo o clase.

  ------------------------------------------------------------------
  Ejemplo 14: Utilidades como métodos de clase
  ------------------------------------------------------------------

  class Formateador
    def self.numero(numero, decimales: 2)
      format("%.#{decimales}f", numero)
    end

    def self.moneda(monto, moneda: "$")
      "#{moneda}#{numero(monto, decimales: 2)}"
    end

    def self.porcentaje(valor, total)
      return "0%" if total == 0
      "#{numero((valor.to_f / total) * 100)}%"
    end

    def self.fecha(fecha, formato: "%d/%m/%Y")
      fecha.strftime(formato)
    end

    def self.rut(numero)
      return "" if numero.nil?
      cuerpo = numero.to_s.reverse.gsub(/(\d{3})(?=\d)/, '\\1.').reverse
      cuerpo
    end
  end

  puts Formateador.numero(1234.5678)    # => 1234.57
  puts Formateador.moneda(1234.5)       # => $1234.50
  puts Formateador.porcentaje(25, 100)  # => 25.00%
  puts Formateador.fecha(Time.now)      # => 02/07/2026
  puts Formateador.rut(12345678)        # => 12.345.678

  11. LA CLASE COMO OBJETO: SELF FUERA DE MÉTODOS

  Fuera de cualquier método, self es la clase misma. Esto permite
  ejecutar código en el contexto de la clase al cargar el archivo.

  ------------------------------------------------------------------
  Ejemplo 15: self fuera de métodos
  ------------------------------------------------------------------

  class Prueba
    puts "Dentro de la clase, self es: #{self}"  # => Prueba

    @valor = 42

    puts "Variable de instancia de clase: #{@valor}"

    def self.mostrar_valor
      @valor
    end
  end

  puts Prueba.mostrar_valor  # => 42

  # Esto es código ejecutable en el contexto de la clase.

  12. MÉTODOS DE CLASE VS MÓDULOS (EXTEND)

  Podemos usar extend para añadir métodos de clase desde un módulo.
  Esto permite compartir métodos de clase entre clases no relacionadas.

  ------------------------------------------------------------------
  Ejemplo 16: extend para métodos de clase
  ------------------------------------------------------------------

  module ContadorInstancias
    def self.included(base)
      base.extend(ClassMethods)
    end

    module ClassMethods
      attr_accessor :count

      def new(*args, &block)
        self.count ||= 0
        self.count += 1
        super
      end
    end
  end

  class Producto
    include ContadorInstancias
  end

  class Pedido
    include ContadorInstancias
  end

  Producto.new
  Producto.new
  Pedido.new

  puts Producto.count  # => 2
  puts Pedido.count    # => 1

  extend añade los métodos del módulo como métodos de clase.

  ------------------------------------------------------------------
  Ejemplo 17: extend directo
  ------------------------------------------------------------------

  module MetodosClase
    def hola
      "Hola desde #{self}"
    end

    def adios
      "Adiós desde #{self}"
    end
  end

  class Persona
    extend MetodosClase
  end

  puts Persona.hola   # => Hola desde Persona
  puts Persona.adios  # => Adiós desde Persona

  # extend es como incluir pero a nivel de clase (singleton_class).

  13. EJEMPLO COMPLETO: GESTIÓN DE EMPLEADOS

  ------------------------------------------------------------------
  Ejemplo 18: Sistema de empleados con métodos de clase
  ------------------------------------------------------------------

  class Empleado
    @empleados = []
    @@id_contador = 0

    class << self
      attr_reader :empleados

      def crear(nombre, puesto, salario)
        empleado = new(nombre, puesto, salario)
        @empleados << empleado
        empleado
      end

      def buscar_por_id(id)
        @empleados.find { |e| e.id == id }
      end

      def buscar_por_nombre(nombre)
        @empleados.select { |e| e.nombre.downcase.include?(nombre.downcase) }
      end

      def total_empleados
        @empleados.size
      end

      def salario_promedio
        return 0 if @empleados.empty?
        @empleados.sum(&:salario) / @empleados.size.to_f
      end

      def empleados_por_puesto(puesto)
        @empleados.select { |e| e.puesto == puesto }
      end

      def top_salarios(n = 5)
        @empleados.sort_by(&:salario).reverse.first(n)
      end

      def reporte
        puts "=== REPORTE DE EMPLEADOS ==="
        puts "Total: #{total_empleados}"
        puts "Salario promedio: $#{salario_promedio.round(2)}"
        puts "Puestos: #{@empleados.map(&:puesto).uniq.join(', ')}"
        @empleados.each { |e| puts "  #{e}" }
      end

      private

      def nuevo_id
        @@id_contador += 1
      end
    end

    attr_reader :id, :nombre, :puesto, :salario

    def initialize(nombre, puesto, salario)
      @id = self.class.send(:nuevo_id)
      @nombre = nombre
      @puesto = puesto
      @salario = salario
      @fecha_contratacion = Time.now
    end

    def to_s
      "##{@id}: #{@nombre} (#{@puesto}) — $#{@salario}"
    end
  end

  Empleado.crear("Ana García", "Ingeniera", 75000)
  Empleado.crear("Carlos López", "Diseñador", 65000)
  Empleado.crear("María Pérez", "Ingeniera", 82000)
  Empleado.crear("Luis Martínez", "Gerente", 95000)

  Empleado.reporte
  # => === REPORTE DE EMPLEADOS ===
  # => Total: 4
  # => ...

  puts "Mejor pagado: #{Empleado.top_salarios(1).first}"
  puts "Ingenieras: #{Empleado.empleados_por_puesto('Ingeniera').size}"

  14. TIPS Y BUENAS PRÁCTICAS

  1. Usa self.metodo para un método de clase ocasional.
  2. Usa class << self para grupos de métodos de clase relacionados.
  3. Prefiere variables de instancia de clase (@) sobre @@ para datos
     que no deben compartirse con subclases.
  4. Las variables de clase (@@) se comparten en toda la jerarquía.
  5. No abuses de los métodos de clase; pertenecen a la clase, no a
     las instancias. Úsalos para crear objetos o consultar datos globales.
  6. Los métodos de clase privados son útiles para lógica interna.
  7. El patrón Singleton es útil pero controversial. Úsalo con moderación.
  8. extend es la forma correcta de añadir métodos de clase desde
     módulos reutilizables.

  RESUMEN DEL CAPÍTULO

  • Los métodos de clase se definen con self.metodo, class << self,
    o NombreClase.metodo.
  • Las variables de clase @@var se comparten entre clase, instancias
    y subclases.
  • Las variables de instancia de clase @var pertenecen solo a la clase.
  • Los métodos de clase se heredan por las subclases.
  • El patrón Singleton restringe a una única instancia.
  • extend importa módulos como métodos de clase.
  • self fuera de métodos es la clase misma.

  PREGUNTAS FRECUENTES (FAQ)

  P: ¿Por qué los métodos de clase se definen con self?
  R: Porque dentro de la definición de clase, self es la propia clase.
     self.metodo define un método en la singleton_class de la clase, lo
     que lo convierte en un método de clase.

  P: ¿Cuál es la diferencia entre @@var y @var a nivel de clase?
  R: @@var (variable de clase) se comparte entre la clase, sus instancias
     y todas las subclases. @var (variable de instancia de clase) solo
     pertenece a esa clase específica, no a sus subclases.

  P: ¿Puedo llamar a un método de instancia desde un método de clase?
  R: No directamente, porque un método de clase no tiene una instancia
     (self es la clase). Necesitas crear una instancia o recibir una
     como argumento.

  P: ¿Los métodos de clase son más lentos que los de instancia?
  R: No hay diferencia apreciable. Ambos son métodos; la diferencia es
     solo en qué objeto los recibe (la clase vs. la instancia).

  P: ¿Cuándo debería usar class << self en lugar de self.metodo?
  R: Cuando tienes varios métodos de clase, class << self los agrupa
     y evita repetir self. cada vez. También permite usar private,
     protected dentro del bloque.

  P: ¿Qué significa "abrir la clase singleton"?
  R: class << self entra en el contexto de la clase singleton (o eigenclass)
     del objeto actual. Para una clase, su clase singleton es donde
     viven los métodos de clase.

  P: ¿Puedo tener attr_accessor a nivel de clase?
  R: Sí, pero debes hacerlo dentro de class << self:
     class << self; attr_accessor :count; end
     Esto crea getters/setters para variables de instancia de clase.

  P: ¿Es malo usar el patrón Singleton?
  R: No es malo en sí mismo, pero debe usarse con moderación. Los
     singletons son difíciles de probar (testing) y crean acoplamiento
     global. Úsalos solo cuando realmente necesites una única instancia.

  DESAFÍO DEL ARQUITECTO

  Construye un sistema completo de gestión de conexiones a bases de
  datos utilizando métodos de clase, variables de instancia de clase,
  y el patrón Singleton.

  1. Clase ConexionBD (Singleton usando require 'singleton'):
     - Mantiene una conexión única (simulada con un hash)
     - Métodos: conectar(config), desconectar, ejecutar(sql),
       conectado?, configuracion
     - La configuración se pasa una vez y se mantiene

  2. Clase PoolConexiones (NO Singleton, múltiples conexiones):
     - @conexiones (variable de instancia de clase, array)
     - Métodos de clase:
       - crear(config): crea y registra una nueva conexión
       - todas: lista de conexiones activas
       - buscar_por_host(host): filtrar conexiones
       - total_conexiones
       - cerrar_todas
       - estadisticas: hash con resumen de todas las conexiones

  3. Clase Base con método de clase abstracto:
     - conectar_clase: cada subclase implementa su propia conexión
     - Las subclases: MySQL, PostgreSQL, SQLite

  4. Variables de clase:
     - @@version_global en ConexionBD que se actualiza con cada
       conexión exitosa
     - @@pool_total en PoolConexiones que cuenta todas las conexiones
       de todos los pools

  5. Métodos de clase privados:
     - validar_configuracion(config) en PoolConexiones
     - generar_id_unico en PoolConexiones

  6. Extend con un módulo Estadisticas que añada a las clases los
     métodos de clase: total_sistema, registrar_evento, reporte_global

  Este desafío integra: self.metodo, class << self, variables de clase
  @@, variables de instancia de clase @, Singleton, extend, métodos de
  clase privados, y herencia de métodos de clase.

  CIERRE DEL CAPÍTULO

  Has dominado los métodos de clase en Ruby. Ahora sabes que las clases
  son objetos y que los métodos de clase son simplemente métodos en la
  singleton_class de la clase. Conoces las diferencias entre @@var y
  @var, y sabes cuándo usar cada una.

  Los métodos de clase son herramientas poderosas para organizar
  comportamiento que pertenece a la clase como concepto, no a sus
  instancias individuales. Factory methods, contadores, consultas
  globales, y singletons son patrones que enriquecen tu caja de
  herramientas.

  En el próximo capítulo exploraremos la herencia: cómo las clases
  pueden extender otras clases, compartir comportamiento, y cómo
  super y la cadena de ancestros permiten composición y reutilización.
