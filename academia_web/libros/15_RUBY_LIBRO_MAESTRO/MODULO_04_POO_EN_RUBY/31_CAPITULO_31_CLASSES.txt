  1. ¿QUÉ ES UNA CLASE?

  Una clase es una plantilla o molde a partir del cual se crean objetos.
  Define un conjunto de atributos (datos) y métodos (comportamientos) que
  todos los objetos de esa clase compartirán.

  Imagina que una clase es el plano arquitectónico de una casa. El plano
  define cómo será la casa: cuántas habitaciones tendrá, dónde estarán
  las puertas, qué tipo de techo tendrá. Pero el plano no es la casa. A
  partir de ese plano puedes construir muchas casas. Cada casa construida
  es una instancia (un objeto) de esa clase.

  En Ruby, las clases se definen con la palabra reservada class, seguida
  del nombre de la clase (que debe comenzar con mayúscula) y finalizan
  con la palabra reservada end.

  ------------------------------------------------------------------
  Ejemplo 1: Definición básica de una clase
  ------------------------------------------------------------------

  class Perro
  end

  Esta es la clase más simple posible. No tiene atributos ni métodos,
  pero ya es una clase válida. Podemos crear objetos a partir de ella.

  ------------------------------------------------------------------
  Ejemplo 2: Crear instancias con new
  ------------------------------------------------------------------

  class Perro
  end

  perro1 = Perro.new
  perro2 = Perro.new

  puts perro1.class   # => Perro
  puts perro2.class   # => Perro
  puts perro1         # => #<Perro:0x00007f...>
  puts perro2         # => #<Perro:0x00007f...>

  Cada objeto tiene un identificador único en memoria. El método new
  asigna memoria para el objeto, lo inicializa y devuelve una referencia
  al mismo.

  ------------------------------------------------------------------
  Ejemplo 3: Las clases son objetos
  ------------------------------------------------------------------

  puts Perro.class          # => Class
  puts Perro.superclass     # => Object
  puts Perro.object_id      # => 123456 (algún número)

  Perro es una constante que referencia a un objeto de la clase Class.
  Esto significa que podemos enviar mensajes a Perro como a cualquier
  otro objeto. Perro.class nos dice que Perro es una instancia de Class.

  2. VARIABLES DE INSTANCIA (@)

  Las variables de instancia son variables que pertenecen a cada objeto
  individualmente. Se identifican por el prefijo @ (arroba). Cada objeto
  tiene su propia copia de las variables de instancia, independiente de
  las demás instancias.

  Las variables de instancia se definen la primera vez que se asignan.
  Si se accede a una variable de instancia que no ha sido definida, su
  valor es nil.

  ------------------------------------------------------------------
  Ejemplo 4: Variables de instancia básicas
  ------------------------------------------------------------------

  class Perro
    def asignar_nombre(nombre)
      @nombre = nombre
    end

    def obtener_nombre
      @nombre
    end
  end

  fido = Perro.new
  lassie = Perro.new

  fido.asignar_nombre("Fido")
  lassie.asignar_nombre("Lassie")

  puts fido.obtener_nombre    # => Fido
  puts lassie.obtener_nombre  # => Lassie

  Observa que @nombre en fido vale "Fido" y @nombre en lassie vale
  "Lassie". Son variables distintas aunque tengan el mismo nombre.

  ------------------------------------------------------------------
  Ejemplo 5: Variables de instancia no inicializadas
  ------------------------------------------------------------------

  class Perro
    def mostrar_edad
      puts @edad
    end
  end

  fido = Perro.new
  fido.mostrar_edad   # => nil (no imprime nada, es nil)

  A diferencia de las variables locales, las variables de instancia
  no inicializadas valen nil, no lanzan un error.

  3. LA DEFINICIÓN DE CLASE ES CÓDIGO EJECUTABLE

  Una de las características más singulares de Ruby es que la definición
  de una clase no es meramente declarativa: es código que se ejecuta en
  tiempo de interpretación. Cada línea dentro de una definición de clase
  se ejecuta cuando el intérprete carga el archivo.

  ------------------------------------------------------------------
  Ejemplo 6: Código ejecutable dentro de la clase
  ------------------------------------------------------------------

  class Prueba
    puts "¡Esto se ejecuta cuando se define la clase!"
    puts "El self aquí es: #{self}"

    @variable_de_clase = "Soy una variable de instancia de clase"

    def metodo
      puts "Esto se ejecuta cuando llamas al método"
    end

    puts "Fin de la definición de la clase"
  end

  # => ¡Esto se ejecuta cuando se define la clase!
  # => El self aquí es: Prueba
  # => Fin de la definición de la clase

  Cuando Ruby ve la palabra class, entra en el contexto de esa clase y
  ejecuta todo el código hasta el end correspondiente. Las definiciones
  de métodos (def...end) se almacenan para ser ejecutadas cuando se
  invoquen, pero el resto del código se ejecuta inmediatamente.

  ------------------------------------------------------------------
  Ejemplo 7: Condicionales dentro de la clase
  ------------------------------------------------------------------

  class TipoDeReporte
    if RUBY_VERSION >= "3.0"
      puts "Usando características modernas de Ruby"
    else
      puts "Usando modo de compatibilidad"
    end
  end

  Esto permite cargar diferentes implementaciones según el entorno,
  una técnica utilizada por muchas gemas para mantener compatibilidad.

  ------------------------------------------------------------------
  Ejemplo 8: Métodos definidos condicionalmente
  ------------------------------------------------------------------

  class Plataforma
    if RUBY_PLATFORM =~ /linux/
      def sistema
        "Linux"
      end
    elsif RUBY_PLATFORM =~ /darwin/
      def sistema
        "macOS"
      end
    else
      def sistema
        "Windows"
      end
    end
  end

  plat = Plataforma.new
  puts plat.sistema   # => Depende del sistema operativo

  4. EL MÉTODO new POR DENTRO

  Cuando llamamos a Perro.new, ocurre lo siguiente:

  1. Ruby llama al método new de la clase Class.
  2. new asigna memoria para el nuevo objeto.
  3. new llama al método initialize (si está definido) para inicializar
     el objeto.
  4. new devuelve la referencia al objeto creado.

  Si la clase no define initialize, los objetos se crean sin inicialización
  adicional, y todas las variables de instancia serán nil hasta que se
  asignen.

  ------------------------------------------------------------------
  Ejemplo 9: El ciclo de vida de new
  ------------------------------------------------------------------

  class Demo
    def initialize
      puts "¡El objeto está siendo inicializado!"
    end
  end

  puts "Antes de new"
  obj = Demo.new
  puts "Después de new"

  # => Antes de new
  # => ¡El objeto está siendo inicializado!
  # => Después de new

  ------------------------------------------------------------------
  Ejemplo 10: Verificando el object_id
  ------------------------------------------------------------------

  class Persona
    def initialize(nombre)
      @nombre = nombre
      puts "Creada persona: #{@nombre} con object_id = #{self.object_id}"
    end
  end

  p1 = Persona.new("Ana")
  p2 = Persona.new("Luis")

  puts "p1 object_id: #{p1.object_id}"
  puts "p2 object_id: #{p2.object_id}"

  # Observa que self.object_id y p1.object_id son el mismo.

  5. OBJETOS Y MENSAJES

  En la POO clásica (Smalltalk), la computación se modela como objetos
  que se envían mensajes entre sí. Ruby adopta esta filosofía.

  Cuando escribimos:

    fido.obtener_nombre

  Lo que realmente ocurre es que enviamos el mensaje :obtener_nombre al
  objeto fido. El objeto fido busca en su clase (Perro) un método que
  responda a ese mensaje y, si lo encuentra, lo ejecuta.

  ------------------------------------------------------------------
  Ejemplo 11: Enviando mensajes con send
  ------------------------------------------------------------------

  class Persona
    def initialize(nombre)
      @nombre = nombre
    end

    def saludar
      "¡Hola! Soy #{@nombre}"
    end
  end

  p = Persona.new("Carlos")

  puts p.saludar           # Forma tradicional
  puts p.send(:saludar)    # Enviando el mensaje explícitamente
  puts p.public_send(:saludar)  # Versión pública

  El método send permite invocar métodos dinámicamente, algo muy útil
  en metaprogramación.

  6. SINGLETON METHODS (MÉTODOS DE OBJETO ÚNICO)

  Ruby permite definir métodos que pertenecen exclusivamente a un objeto
  en particular, no a toda la clase. Esto se hace con la sintaxis:

    def objeto.metodo
      # ...
    end

  ------------------------------------------------------------------
  Ejemplo 12: Métodos singleton
  ------------------------------------------------------------------

  class Perro
    def ladrar
      "¡Guau!"
    end
  end

  fido = Perro.new
  lassie = Perro.new

  def fido.bailar
    "Fido está bailando"
  end

  def lassie.saltar
    "Lassie está saltando"
  end

  puts fido.ladrar    # => ¡Guau!
  puts fido.bailar    # => Fido está bailando
  puts lassie.ladrar  # => ¡Guau!
  puts lassie.saltar  # => Lassie está saltando
  # puts fido.saltar  # => Error! NoMethodError

  Cada objeto puede tener su propio comportamiento único. Esto es posible
  gracias a que en Ruby cada objeto tiene su propia clase singleton
  (clase anónima) que se interpone en la cadena de herencia.

  ------------------------------------------------------------------
  Ejemplo 13: La clase singleton
  ------------------------------------------------------------------

  obj = Object.new

  def obj.mi_metodo
    "Soy único"
  end

  puts obj.singleton_class        # => #<Class:#<Object:0x...>>
  puts obj.singleton_methods      # => [:mi_metodo]

  Cada objeto tiene una singleton_class que almacena los métodos que son
  exclusivos de ese objeto.

  7. ATRIBUTOS BÁSICOS (GETTERS Y SETTERS MANUALES)

  Para exponer las variables de instancia al exterior, necesitamos definir
  métodos getter (lectura) y setter (escritura). Aunque en capítulos
  posteriores veremos attr_accessor, aquí los haremos manualmente para
  entender qué sucede internamente.

  ------------------------------------------------------------------
  Ejemplo 14: Getters y setters manuales
  ------------------------------------------------------------------

  class Perro
    def initialize(nombre, raza)
      @nombre = nombre
      @raza = raza
    end

    # Getter para @nombre
    def nombre
      @nombre
    end

    # Setter para @nombre
    def nombre=(nuevo_nombre)
      @nombre = nuevo_nombre
    end

    # Getter para @raza
    def raza
      @raza
    end

    # Setter para @raza
    def raza=(nueva_raza)
      @raza = nueva_raza
    end
  end

  fido = Perro.new("Fido", "Labrador")

  puts fido.nombre      # => Fido (getter)
  puts fido.raza        # => Labrador (getter)

  fido.nombre = "Firulais"  # Setter
  fido.raza = "Pastor Alemán"

  puts fido.nombre      # => Firulais
  puts fido.raza        # => Pastor Alemán

  La sintaxis fido.nombre = "Firulais" es azúcar sintáctica. Ruby lo
  interpreta como: fido.nombre=("Firulais")

  8. TO_S Y LA REPRESENTACIÓN DE OBJETOS

  Cada objeto en Ruby tiene el método to_s, que devuelve una representación
  en forma de cadena. Por defecto, to_s devuelve algo como:

    #<Perro:0x00007f8b1a8b3d60>

  Podemos sobreescribir to_s para proporcionar una representación más
  significativa.

  ------------------------------------------------------------------
  Ejemplo 15: Sobreescribir to_s
  ------------------------------------------------------------------

  class Perro
    def initialize(nombre, raza)
      @nombre = nombre
      @raza = raza
    end

    def to_s
      "Perro: #{@nombre} (#{@raza})"
    end
  end

  fido = Perro.new("Fido", "Labrador")
  puts fido                # => Perro: Fido (Labrador)
  puts "Mi perro: #{fido}" # => Mi perro: Perro: Fido (Labrador)

  También existe inspect, que se usa en la consola (IRB/Pry) para mostrar
  objetos. Por defecto, inspect usa la representación con #<...>.

  ------------------------------------------------------------------
  Ejemplo 16: inspect
  ------------------------------------------------------------------

  class Perro
    def initialize(nombre, raza)
      @nombre = nombre
      @raza = raza
    end

    def inspect
      "<Perro: @nombre=#{@nombre.inspect}, @raza=#{@raza.inspect}>"
    end
  end

  fido = Perro.new("Fido", "Labrador")
  p fido  # Usa inspect internamente
  # => <Perro: @nombre="Fido", @raza="Labrador">

  9. COMPARACIÓN DE OBJETOS

  Por defecto, Ruby compara objetos por identidad (si son el mismo objeto
  en memoria). Dos objetos pueden tener exactamente el mismo contenido
  pero ser considerados diferentes.

  ------------------------------------------------------------------
  Ejemplo 17: Comparación por defecto vs. personalizada
  ------------------------------------------------------------------

  class Punto
    def initialize(x, y)
      @x = x
      @y = y
    end
  end

  p1 = Punto.new(1, 2)
  p2 = Punto.new(1, 2)
  p3 = p1

  puts p1 == p2   # => false (distintos objetos)
  puts p1 == p3   # => true (mismo objeto)
  puts p1.equal?(p2)  # => false (equal? compara identidad)
  puts p1.equal?(p3)  # => true

  Para comparar por valor, debemos definir el método ==:

  class Punto
    def initialize(x, y)
      @x = x
      @y = y
    end

    def ==(otro)
      return false unless otro.is_a?(Punto)
      @x == otro.x && @y == otro.y
    end

    # Necesitamos getters para acceder desde otro objeto
    def x
      @x
    end

    def y
      @y
    end
  end

  p1 = Punto.new(1, 2)
  p2 = Punto.new(1, 2)

  puts p1 == p2   # => true

  10. CLASES ANIDADAS

  Ruby permite definir clases dentro de otras clases. Esto es útil para
  organizar código y crear namespaces.

  ------------------------------------------------------------------
  Ejemplo 18: Clases anidadas
  ------------------------------------------------------------------

  class Universidad
    class Estudiante
      def initialize(nombre)
        @nombre = nombre
      end

      def to_s
        "Estudiante: #{@nombre}"
      end
    end

    class Profesor
      def initialize(nombre, materia)
        @nombre = nombre
        @materia = materia
      end

      def to_s
        "Profesor: #{@nombre} (#{@materia})"
      end
    end
  end

  est = Universidad::Estudiante.new("Ana Pérez")
  prof = Universidad::Profesor.new("Dr. García", "Matemáticas")

  puts est   # => Estudiante: Ana Pérez
  puts prof  # => Profesor: Dr. García (Matemáticas)

  Las clases anidadas se referencian con el operador :: (resolución de
  ámbito).

  11. LA CLASE COMO CIUDADANO DE PRIMERA CLASE

  En Ruby, las clases son objetos como cualquier otro. Esto significa
  que podemos:

  • Asignar una clase a una variable
  • Pasar una clase como argumento a un método
  • Devolver una clase desde un método
  • Preguntar a una clase por su nombre, superclase, etc.

  ------------------------------------------------------------------
  Ejemplo 19: Clases como objetos
  ------------------------------------------------------------------

  class Animal; end
  class Perro < Animal; end

  mi_clase = Perro
  puts mi_clase.name          # => "Perro"
  puts mi_clase.superclass    # => Animal
  puts mi_clase.ancestors.inspect  # => [Perro, Animal, Object, ...]

  def crear_objeto(clase, *args)
    clase.new(*args)
  end

  obj = crear_objeto(Perro)  # Equivalente a Perro.new
  puts obj.class             # => Perro

  12. CONSTANTES DENTRO DE LA CLASE

  Las clases pueden definir constantes que están disponibles dentro del
  ámbito de la clase.

  ------------------------------------------------------------------
  Ejemplo 20: Constantes en clases
  ------------------------------------------------------------------

  class Circulo
    PI = 3.141592653589793

    def initialize(radio)
      @radio = radio
    end

    def area
      PI * @radio ** 2
    end

    def perimetro
      2 * PI * @radio
    end
  end

  c = Circulo.new(5)
  puts c.area        # => 78.53981633974483
  puts c.perimetro   # => 31.41592653589793

  puts Circulo::PI   # Acceso desde fuera de la clase

  Las constantes comienzan con mayúscula y, por convención, se escriben
  en MAYÚSCULAS.

  13. EJEMPLO COMPLETO: SISTEMA DE BIBLIOTECA

  ------------------------------------------------------------------
  Ejemplo 21: Sistema simple de biblioteca
  ------------------------------------------------------------------

  class Biblioteca
    def initialize
      @libros = []
    end

    def agregar_libro(libro)
      @libros << libro
      puts "Libro agregado: #{libro}"
    end

    def mostrar_libros
      if @libros.empty?
        puts "La biblioteca está vacía"
      else
        puts "Catálogo de la biblioteca:"
        @libros.each_with_index do |libro, i|
          puts "#{i + 1}. #{libro}"
        end
      end
    end

    def buscar_por_titulo(titulo)
      encontrados = @libros.select { |l| l.titulo.downcase.include?(titulo.downcase) }
      if encontrados.empty?
        puts "No se encontraron libros con '#{titulo}'"
      else
        puts "Resultados de búsqueda:"
        encontrados.each { |l| puts "- #{l}" }
      end
    end

    def total_libros
      @libros.size
    end

    def to_s
      "Biblioteca con #{@libros.size} libros"
    end
  end

  class Libro
    def initialize(titulo, autor, año)
      @titulo = titulo
      @autor = autor
      @año = año
    end

    def titulo
      @titulo
    end

    def autor
      @autor
    end

    def año
      @año
    end

    def to_s
      "#{@titulo} por #{@autor} (#{@año})"
    end
  end

  # Uso del sistema
  biblio = Biblioteca.new

  libro1 = Libro.new("Cien Años de Soledad", "Gabriel García Márquez", 1967)
  libro2 = Libro.new("El Quijote", "Miguel de Cervantes", 1605)
  libro3 = Libro.new("Rayuela", "Julio Cortázar", 1963)

  biblio.agregar_libro(libro1)
  biblio.agregar_libro(libro2)
  biblio.agregar_libro(libro3)

  biblio.mostrar_libros
  biblio.buscar_por_titulo("quijote")

  puts "Total de libros: #{biblio.total_libros}"
  puts biblio

  14. TIPS Y BUENAS PRÁCTICAS

  1. Nombres de clase en CamelCase: class MiClase, no class mi_clase.
  2. Un archivo debe contener idealmente una sola clase.
  3. El nombre del archivo debe ser en snake_case y reflejar la clase:
     class MiClase → mi_clase.rb
  4. No definas demasiada lógica en el cuerpo de la clase (código
     ejecutable). Úsalo con moderación para configuraciones.
  5. Las variables de instancia (@var) siempre deben inicializarse en
     initialize, no en otros métodos.
  6. Usa to_s para dar representación legible a tus objetos.
  7. No abusues de los métodos singleton; si muchos objetos necesitan
     el mismo comportamiento, defínelo en la clase.

  RESUMEN DEL CAPÍTULO

  • class es la palabra reservada para definir clases en Ruby.
  • Las clases son objetos de la clase Class.
  • new es el método que crea instancias de una clase.
  • Las variables de instancia (@var) pertenecen a cada objeto y
    almacenan su estado interno.
  • Si no se inicializan, las variables de instancia valen nil.
  • La definición de clase es código ejecutable; Ruby ejecuta las
    líneas dentro de class...end al cargar el archivo.
  • Podemos definir métodos singleton para objetos individuales con
    def objeto.metodo.
  • Cada objeto tiene una singleton_class que contiene sus métodos
    exclusivos.
  • Podemos sobreescribir to_s e inspect para personalizar la
    representación de objetos.
  • La comparación por defecto (==) compara identidad; podemos
    sobreescribir == para comparar por valor.
  • Las clases pueden anidarse para organización y namespace.

  PREGUNTAS FRECUENTES (FAQ)

  P: ¿Por qué el nombre de la clase debe comenzar con mayúscula?
  R: En Ruby, los identificadores que comienzan con mayúscula son
     constantes. class MiClase define una constante MiClase que referencia
     al objeto de clase Class. Por convención, los nombres de clases y
     módulos se escriben en CamelCase.

  P: ¿Cuál es la diferencia entre una clase y un objeto?
  R: Una clase es una plantilla o molde. Un objeto (o instancia) es una
     materialización concreta de esa plantilla. La clase define qué
     atributos y métodos tendrán los objetos; cada objeto tiene su propio
     conjunto de valores para esos atributos.

  P: ¿Qué ocurre si llamo a new sin definir initialize?
  R: No ocurre ningún error. Ruby proporciona un initialize implícito que
     no hace nada. El objeto se crea igualmente, y todas las variables de
     instancia serán nil hasta que se asignen.

  P: ¿Puedo tener dos clases con el mismo nombre?
  R: Técnicamente no, porque el nombre de la clase es una constante y
     Ruby mostrará una advertencia si redefines una constante. Sin embargo,
     si ya existe una clase con ese nombre, Ruby la reabre y puedes añadir
     o redefinir métodos (open classes). Esto se conoce como "monkey
     patching" y debe usarse con cuidado.

  P: ¿Qué es la singleton_class?
  R: Es una clase anónima (sin nombre) que Ruby crea automáticamente para
     cada objeto. Los métodos singleton se almacenan en esta clase, que
     se inserta en la cadena de herencia del objeto justo encima de su
     clase original.

  P: ¿Por qué la comparación con == da false entre objetos idénticos?
  R: Porque el == por defecto (heredado de BasicObject) compara identidad
     de objeto (si son el mismo objeto en memoria), no valor. Para comparar
     por valor debes sobreescribir == en tu clase.

  P: ¿Es obligatorio definir initialize en una clase?
  R: No, pero es altamente recomendable si tu clase tiene variables de
     instancia que deben inicializarse con valores específicos. Si no
     necesitas inicialización especial, puedes omitirlo.

  P: ¿Puedo llamar a un método antes de definirlo?
  R: Depende. Si el método está definido en la misma clase, sí, siempre
     que el método exista en el momento de la llamada. Ruby no tiene un
     paso de compilación separado; los métodos se definen cuando se
     ejecuta la definición. Sin embargo, si llamas a un método que no
     existe, obtendrás NoMethodError.
