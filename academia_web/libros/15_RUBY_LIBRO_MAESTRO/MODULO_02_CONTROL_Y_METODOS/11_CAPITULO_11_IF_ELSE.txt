  11.1  EL IF MÁS SENCILLO
  --------------------------------------------------

  La estructura if más básica evalúa una condición y ejecuta un bloque de
  código si dicha condición es verdadera.  En Ruby, la condición puede ser
  cualquier expresión que devuelva true o false.  Recuerda que en Ruby solo
  nil y false se evalúan como falso; cualquier otro valor (0, "", [], {})
  es considerado verdadero.

  ------------------------------------------------------------------

  # Ejemplo 1: if simple
  edad = 18

  if edad >= 18
    puts "Eres mayor de edad. Puedes votar."
  end

  # Ejemplo 2: if con múltiples líneas
  temperatura = 30

  if temperatura > 25
    puts "Hace calor."
    puts "Recomendación: usa protector solar."
    puts "Toma agua frecuentemente."
  end

  # Ejemplo 3: condición con método
  nombre = "Alex"

  if nombre.empty?
    puts "El nombre está vacío. Por favor ingresa un nombre válido."
  end

  ------------------------------------------------------------------

  - La palabra reservada if inicia la estructura.
  - La condición se escribe a continuación (sin paréntesis obligatorios).
  - El bloque de código se sangra (por convención con 2 espacios).
  - end cierra la estructura.
  - Si la condición es false o nil, el bloque simplemente se salta.

================================================================================

  11.2  IF / ELSE — EL CAMINO BIFURCADO
  --------------------------------------------------

  Cuando necesitas ejecutar un bloque si la condición es verdadera y otro
  bloque si es falsa, usas if/else.  Es la estructura de decisión binaria
  más común.

  ------------------------------------------------------------------

  # Ejemplo 1: validación de acceso
  usuario_autenticado = false

  if usuario_autenticado
    puts "Bienvenido al sistema."
    puts "Panel de control:"
    puts "  1. Ver perfil"
    puts "  2. Configuración"
  else
    puts "Acceso denegado."
    puts "Por favor inicia sesión."
  end

  # Ejemplo 2: número par o impar
  numero = 7

  if numero.even?
    puts "#{numero} es par."
  else
    puts "#{numero} es impar."
  end

  # Ejemplo 3: validación de nota
  nota = 85

  if nota >= 60
    puts "Aprobado. Felicidades."
  else
    puts "Reprobado. Debes estudiar más."
  end

  ------------------------------------------------------------------

  - if ejecuta el primer bloque si la condición es true.
  - else ejecuta el segundo bloque si la condición es false.
  - Solo uno de los dos bloques se ejecuta, nunca ambos.
  - end cierra la estructura completa.
  - No puede haber más de un else por estructura if.

================================================================================

  11.3  IF / ELSIF / ELSE — MÚLTIPLES CAMINOS
  --------------------------------------------------

  Cuando tienes más de dos caminos posibles, usas elsif (nota: NO es
  "elseif" ni "else if" como en otros lenguajes; Ruby usa "elsif").
  Puedes tener tantos elsif como necesites.

  ------------------------------------------------------------------

  # Ejemplo 1: calificación con letras
  calificacion = 85

  if calificacion >= 90
    letra = "A"
  elsif calificacion >= 80
    letra = "B"
  elsif calificacion >= 70
    letra = "C"
  elsif calificacion >= 60
    letra = "D"
  else
    letra = "F"
  end

  puts "Tu calificación es: #{letra}"

  # Ejemplo 2: menú de opciones
  opcion = 2

  if opcion == 1
    puts "Seleccionaste: Ver perfil"
  elsif opcion == 2
    puts "Seleccionaste: Editar configuración"
  elsif opcion == 3
    puts "Seleccionaste: Cerrar sesión"
  elsif opcion == 4
    puts "Seleccionaste: Salir"
  else
    puts "Opción inválida. Por favor elige 1-4."
  end

  # Ejemplo 3: estación del año
  mes = 7

  if [12, 1, 2].include?(mes)
    estacion = "Verano"
  elsif [3, 4, 5].include?(mes)
    estacion = "Otoño"
  elsif [6, 7, 8].include?(mes)
    estacion = "Invierno"
  elsif [9, 10, 11].include?(mes)
    estacion = "Primavera"
  end

  puts "El mes #{mes} corresponde a: #{estacion}"

  # Ejemplo 4: tipo de dato
  valor = 42

  if valor.is_a?(String)
    puts "Es una cadena de texto."
  elsif valor.is_a?(Integer)
    puts "Es un número entero."
  elsif valor.is_a?(Float)
    puts "Es un número decimal."
  elsif valor.is_a?(Array)
    puts "Es un arreglo."
  elsif valor.is_a?(Hash)
    puts "Es un hash."
  else
    puts "Tipo de dato desconocido."
  end

  ------------------------------------------------------------------

  - if evalúa la primera condición.
  - Si es false, pasa al primer elsif, luego al siguiente, etc.
  - Si ningún elsif coincide, ejecuta else (si existe).
  - Las condiciones se evalúan en orden; la primera que sea true
    ejecuta su bloque y luego sale de toda la estructura.
  - elsif debe escribirse exactamente así (falta una 'e' respecto
    a otros lenguajes).

================================================================================

  11.4  VALOR DE RETORNO DE IF
  --------------------------------------------------

  En Ruby, if no es solo una estructura de control: es una expresión.
  Esto significa que if devuelve un valor.  El valor devuelto es el
  resultado de la última expresión evaluada dentro del bloque que se
  ejecutó.

  ------------------------------------------------------------------

  # Ejemplo 1: if devuelve valor
  edad = 20

  mensaje = if edad >= 18
              "Mayor de edad"
            else
              "Menor de edad"
            end

  puts mensaje  # => "Mayor de edad"

  # Ejemplo 2: asignación condicional
  usuario = "admin"
  rol = if usuario == "admin"
          "Administrador"
        elsif usuario == "mod"
          "Moderador"
        else
          "Usuario"
        end

  puts "Rol: #{rol}"

  # Ejemplo 3: if sin else devuelve nil
  llueve = false
  resultado = if llueve
                "Lleva paraguas"
              end

  puts resultado.inspect  # => nil

  # Ejemplo 4: condicionales anidados con retorno
  x = 10
  y = 20

  mayor = if x > y
            x
          else
            y
          end

  puts "El mayor es: #{mayor}"  # => 20

  ------------------------------------------------------------------

  - Puedes asignar el resultado de if directamente a una variable.
  - Esto elimina la necesidad de asignar dentro de cada rama.
  - Si no hay else y la condición es false, el valor retornado es nil.
  - Esta característica hace el código más funcional y conciso.

================================================================================

  11.5  EL MODIFICADOR IF (IF EN UNA LÍNEA)
  --------------------------------------------------

  Ruby permite escribir if al final de una línea como modificador.
  Es una forma elegante y concisa de ejecutar una acción simple
  cuando se cumple una condición.

  ------------------------------------------------------------------

  # Ejemplo 1: modificador if básico
  puts "Es fin de semana" if Time.now.saturday? || Time.now.sunday?

  # Ejemplo 2: guard clause en una línea
  saldo = 100
  puts "Fondos insuficientes" if saldo < 0

  # Ejemplo 3: validación rápida
  usuario = "admin"
  puts "Acceso total concedido" if usuario == "admin"

  # Ejemplo 4: log condicional
  debug = true
  puts "Valor de x: #{x}" if debug

  # Ejemplo 5: asignación condicional
  archivo = nil
  puts "Archivo no encontrado" if archivo.nil?

  # Ejemplo 6: múltiples condiciones
  edad = 25
  tiene_licencia = true
  puts "Puede conducir" if edad >= 18 && tiene_licencia

  # Ejemplo 7: con métodos que retornan boolean
  def puede_votar?(persona)
    persona[:edad] >= 18
  end

  persona = { nombre: "Ana", edad: 22 }
  puts "#{persona[:nombre]} puede votar" if puede_votar?(persona)

  ------------------------------------------------------------------

  - La acción se ejecuta solo si la condición es verdadera.
  - Es ideal para una sola instrucción (no para bloques grandes).
  - Mejora la legibilidad cuando se usa con moderación.
  - No puede llevar else.
  - Es conocido como "conditional modifier" o "statement modifier".

================================================================================

  11.6  EL OPERADOR TERNARIO ( ? : )
  --------------------------------------------------

  El operador ternario es una forma compacta de if/else que devuelve
  un valor.  Su sintaxis es: condición ? valor_si_true : valor_si_false.
  En Ruby, el ternario es especialmente útil para asignaciones rápidas.

  ------------------------------------------------------------------

  # Ejemplo 1: ternario básico
  edad = 17
  estado = edad >= 18 ? "Mayor" : "Menor"
  puts estado  # => "Menor"

  # Ejemplo 2: ternario en interpolación
  nota = 85
  puts "Resultado: #{nota >= 60 ? 'Aprobado' : 'Reprobado'}"

  # Ejemplo 3: ternario con método
  def tipo_numero(n)
    n.positive? ? "Positivo" : (n.negative? ? "Negativo" : "Cero")
  end

  puts tipo_numero(5)   # => Positivo
  puts tipo_numero(-3)  # => Negativo
  puts tipo_numero(0)   # => Cero

  # Ejemplo 4: ternario vs if/else
  # Con if:
  if user.admin?
    mensaje = "Bienvenido admin"
  else
    mensaje = "Bienvenido usuario"
  end

  # Con ternario (más compacto):
  mensaje = user.admin? ? "Bienvenido admin" : "Bienvenido usuario"

  # Ejemplo 5: ternario para valores por defecto
  nombre = nil
  saludo = nombre ? "Hola, #{nombre}" : "Hola, invitado"
  puts saludo  # => "Hola, invitado"

  # Ejemplo 6: ternario para estilo visual
  modo_oscuro = true
  fondo = modo_oscuro ? "#000000" : "#FFFFFF"
  texto = modo_oscuro ? "#FFFFFF" : "#000000"
  puts "Fondo: #{fondo}, Texto: #{texto}"

  # Ejemplo 7: ternario en return
  def maximo(a, b)
    a > b ? a : b
  end

  puts maximo(10, 20)  # => 20

  # Ejemplo 8: evitar ternarios anidados complejos
  # ❌ Malo (difícil de leer):
  # resultado = a > b ? (a > c ? a : c) : (b > c ? b : c)

  # ✅ Bueno (usa método o if normal):
  def maximo_de_tres(a, b, c)
    if a > b && a > c
      a
    elsif b > c
      b
    else
      c
    end
  end

  ------------------------------------------------------------------

  - El ternario SIEMPRE devuelve un valor.
  - Es ideal para asignaciones condicionales simples.
  - No abuses del ternario: si la lógica es compleja, usa if/else.
  - Los ternarios anidados son difíciles de leer; evítalos.
  - En Ruby, el ternario es una expresión, no una sentencia.

  11.7  IF ANIDADOS
  --------------------------------------------------

  Puedes colocar un if dentro de otro if.  Esto se conoce como
  anidamiento.  Aunque es posible, debes usarlo con moderación,
  ya que demasiados niveles de anidamiento reducen la legibilidad.

  ------------------------------------------------------------------

  # Ejemplo 1: if anidado simple
  usuario = "admin"
  contrasena_valida = true

  if usuario == "admin"
    if contrasena_valida
      puts "Acceso concedido al panel de administración."
    else
      puts "Contraseña incorrecta."
    end
  else
    puts "Usuario no reconocido."
  end

  # Ejemplo 2: tres niveles (evitar si es posible)
  edad = 25
  tiene_licencia = true
  tiene_seguro = true

  if edad >= 18
    if tiene_licencia
      if tiene_seguro
        puts "Puedes conducir legalmente."
      else
        puts "Necesitas seguro para conducir."
      end
    else
      puts "Necesitas licencia para conducir."
    end
  else
    puts "Debes ser mayor de edad para conducir."
  end

  # Ejemplo 3: refactorizando con && (mejor práctica)
  if edad >= 18 && tiene_licencia && tiene_seguro
    puts "Puedes conducir legalmente."
  elsif edad >= 18 && !tiene_licencia
    puts "Necesitas licencia para conducir."
  elsif edad >= 18 && !tiene_seguro
    puts "Necesitas seguro para conducir."
  else
    puts "Debes ser mayor de edad para conducir."
  end

  # Ejemplo 4: early return para evitar anidamiento
  def validar_acceso(usuario, contrasena)
    return "Usuario no encontrado" unless usuario
    return "Contraseña requerida" if contrasena.nil? || contrasena.empty?
    return "Credenciales inválidas" if usuario != "admin" || contrasena != "1234"

    "Acceso concedido"
  end

  puts validar_acceso("admin", "1234")   # => Acceso concedido
  puts validar_acceso(nil, "1234")       # => Usuario no encontrado
  puts validar_acceso("admin", "")       # => Contraseña requerida
  puts validar_acceso("user", "1234")    # => Credenciales inválidas

  ------------------------------------------------------------------

  - Los if anidados son necesarios a veces, pero intenta limitarlos.
  - Con más de 2 niveles de anidamiento, considera refactorizar.
  - Usa operadores lógicos (&&, ||) para combinar condiciones.
  - El patrón "early return" reduce drásticamente el anidamiento.
  - Un código con menos anidamiento es más fácil de leer y mantener.

  11.8  IF CON RANGOS Y REGEX
  --------------------------------------------------

  Ruby permite usar rangos y expresiones regulares dentro de
  condiciones if, lo que hace el código muy expresivo.

  ------------------------------------------------------------------

  # Ejemplo 1: if con rango (case equality con ===)
  nota = 75

  if (0..59) === nota
    puts "Calificación: F"
  elsif (60..69) === nota
    puts "Calificación: D"
  elsif (70..79) === nota
    puts "Calificación: C"
  elsif (80..89) === nota
    puts "Calificación: B"
  elsif (90..100) === nota
    puts "Calificación: A"
  end

  # Ejemplo 2: if con expresión regular
  email = "usuario@ejemplo.com"

  if email =~ /\A[\w+\-.]+@[a-z\d\-]+(\.[a-z\d\-]+)*\.[a-z]+\z/i
    puts "Email válido."
  else
    puts "Email inválido."
  end

  # Ejemplo 3: match con regex (devuelve MatchData o nil)
  texto = "La fecha es 2024-12-25"
  if (match = texto.match(/(\d{4})-(\d{2})-(\d{2})/))
    puts "Año: #{match[1]}, Mes: #{match[2]}, Día: #{match[3]}"
  else
    puts "No se encontró fecha."
  end

  # Ejemplo 4: verificar tipo con ===
  valor = [1, 2, 3]

  if Array === valor
    puts "Es un arreglo con #{valor.length} elementos."
  elsif Hash === valor
    puts "Es un hash."
  elsif String === valor
    puts "Es un string."
  end

  # Ejemplo 5: include? con array en condición
  vocales = %w[a e i o u]
  letra = "e"

  if vocales.include?(letra)
    puts "#{letra} es una vocal."
  else
    puts "#{letra} es una consonante."
  end

  ------------------------------------------------------------------

  - El operador === es "case equality" y es usado por rangos y clases.
  - Puedes usar regex con =~ para validar patrones.
  - .match devuelve un objeto MatchData (verdadero) o nil (falso).
  - Puedes asignar el resultado de match dentro de la condición.
  - Los rangos son muy útiles para validar intervalos numéricos.

  11.9  BUENAS PRÁCTICAS CON IF
  --------------------------------------------------

  Escribir condicionales limpios es una habilidad fundamental.
  Aquí tienes las mejores prácticas para usar if en Ruby.

  ------------------------------------------------------------------

  # 1. Usa paréntesis solo cuando sea necesario para claridad
  if (x > 5 && y < 10) || z == 0
    puts "Condición compleja"
  end

  # 2. Prefiere el modificador if para guard clauses simples
  return if usuario.nil?

  # 3. Evita comparar con true / false explícitamente
  # ❌ Malo:
  if activo == true
    puts "Activo"
  end
  # ✅ Bueno:
  if activo
    puts "Activo"
  end

  # ❌ Malo:
  if activo == false
    puts "Inactivo"
  end
  # ✅ Bueno:
  if !activo
    puts "Inactivo"
  end
  # O mejor aún:
  unless activo
    puts "Inactivo"
  end

  # 4. No compares nil explícitamente si puedes evitarlo
  # ❌ Malo:
  if valor != nil
    puts valor
  end
  # ✅ Bueno:
  if valor
    puts valor
  end

  # 5. Usa el ternario solo para asignaciones simples
  tipo = edad >= 18 ? "adulto" : "menor"

  # 6. No anides más de dos niveles; refactoriza
  # ❌ Malo:
  if a
    if b
      if c
        # demasiado profundo
      end
    end
  end
  # ✅ Bueno: usa early return o combina condiciones
  return unless a && b && c

  # 7. Usa unless para condiciones negativas (próximo capítulo)
  # if !algo  =>  unless algo

  # 8. Extrae condiciones complejas a métodos
  # ❌ Malo:
  if usuario.edad >= 18 && usuario.activo? && !usuario.suspendido? && usuario.plan == "premium"
  end
  # ✅ Bueno:
  def acceso_premium?(usuario)
    usuario.edad >= 18 && usuario.activo? && !usuario.suspendido? && usuario.plan == "premium"
  end
  if acceso_premium?(usuario)
  end

  # 9. Usa case/when cuando haya múltiples valores fijos
  # 10. Sé consistente con el estilo de tu equipo

  ------------------------------------------------------------------

  11.10  ANTIPATRONES COMUNES
  --------------------------------------------------

  ------------------------------------------------------------------

  # Antipatrón 1: Asignación dentro de condición
  # ❌ Malo (asigna y evalúa):
  # if x = 5  # Siempre true!
  #   puts "Siempre se ejecuta"
  # end

  # ✅ Bueno:
  if x == 5
    puts "x es 5"
  end

  # Antipatrón 2: if innecesario para booleanos
  # ❌ Malo:
  if es_valido? == true
  end
  # ✅ Bueno:
  if es_valido?
  end

  # Antipatrón 3: if/else que devuelve boolean
  # ❌ Malo:
  if x > 5
    resultado = true
  else
    resultado = false
  end
  # ✅ Bueno:
  resultado = x > 5

  # Antipatrón 4: Cadenas de elsif para un solo valor
  # ❌ Malo:
  if valor == "a"
  elsif valor == "b"
  elsif valor == "c"
  end
  # ✅ Bueno (usa case/when):
  case valor
  when "a" then # ...
  when "b" then # ...
  when "c" then # ...
  end

  # Antipatrón 5: if vacío con else
  # ❌ Malo:
  if condicion
    # no hacer nada
  else
    puts "Condición falsa"
  end
  # ✅ Bueno:
  unless condicion
    puts "Condición falsa"
  end

  # Antipatrón 6: Condición negada (usa unless)
  # ❌ Malo:
  if !usuario.activo?
    puts "Usuario inactivo"
  end
  # ✅ Bueno:
  unless usuario.activo?
    puts "Usuario inactivo"
  end

  # Antipatrón 7: Comparar con nil usando ==
  # ❌ Malo:
  if nombre == nil
    puts "Sin nombre"
  end
  # ✅ Bueno:
  if nombre.nil?
    puts "Sin nombre"
  end

  ------------------------------------------------------------------

  11.11  EJEMPLO INTEGRADOR: SISTEMA DE AUTENTICACIÓN
  --------------------------------------------------

  ------------------------------------------------------------------

  # Sistema que combina if/else, elsif, ternario, modificador if

  usuarios = {
    "admin" => { password: "1234", rol: "admin", activo: true },
    "user1" => { password: "abcd", rol: "editor", activo: true },
    "user2" => { password: "pass", rol: "lector", activo: false }
  }

  def autenticar(usuarios, usuario, password)
    return { exito: false, mensaje: "Usuario no encontrado" } unless usuarios.key?(usuario)
    datos = usuarios[usuario]
    return { exito: false, mensaje: "Cuenta desactivada" } unless datos[:activo]
    return { exito: false, mensaje: "Contraseña incorrecta" } if datos[:password] != password
    { exito: true, mensaje: "Bienvenido, #{usuario}", rol: datos[:rol] }
  end

  def mostrar_menu(rol)
    puts "=== MENÚ PRINCIPAL ==="
    if rol == "admin"
      puts "1. Gestionar usuarios"
      puts "2. Ver logs del sistema"
      puts "3. Configuración global"
    elsif rol == "editor"
      puts "1. Crear artículo"
      puts "2. Editar artículo"
      puts "3. Publicar artículo"
    elsif rol == "lector"
      puts "1. Ver artículos"
      puts "2. Buscar contenido"
    end
  end

  intentos = [
    { usuario: "admin", password: "1234" },
    { usuario: "user1", password: "incorrecta" },
    { usuario: "user2", password: "pass" }
  ]

  intentos.each do |intento|
    puts "\n--- Intentando: #{intento[:usuario]} ---"
    resultado = autenticar(usuarios, intento[:usuario], intento[:password])
    puts resultado[:mensaje]
    if resultado[:exito]
      mostrar_menu(resultado[:rol])
      acceso_total = resultado[:rol] == "admin" ? "ACCESO TOTAL" : "Acceso limitado"
      puts "Nivel: #{acceso_total}"
    end
  end

  puts "\n¿Hay usuarios administradores?" if usuarios.values.any? { |u| u[:rol] == "admin" }

  RESUMEN DEL CAPÍTULO
  ====================

  1. if ejecuta un bloque si la condición es verdadera.
  2. if/else proporciona dos caminos: verdadero y falso.
  3. if/elsif/else maneja múltiples condiciones mutuamente excluyentes.
  4. if es una expresión en Ruby y devuelve un valor.
  5. El modificador if (acción if condición) es ideal para una línea.
  6. El operador ternario (cond ? val_true : val_false) es compacto.
  7. Los if anidados deben minimizarse (máx 2 niveles).
  8. Usa early returns para reducir anidamiento.
  9. Extrae condiciones complejas a métodos con nombres descriptivos.
  10. Usa unless para condiciones negativas (próximo capítulo).

  PREGUNTAS FRECUENTES (FAQ)
  ==========================

  P: ¿Cuál es la diferencia entre if y unless?
  R: if ejecuta el bloque cuando la condición es verdadera; unless lo
     ejecuta cuando la condición es falsa. unless es if !condicion.

  P: ¿Puedo usar return dentro de un if?
  R: Sí, y es una práctica común para "early returns" dentro de métodos.

  P: ¿Cuándo debo usar el modificador if?
  R: Cuando tienes una sola acción que ejecutar si se cumple una
     condición. Ej: puts "Error" if error.

  P: ¿Es malo anidar if?
  R: No es malo per se, pero más de 2 niveles de anidamiento dificulta
     la lectura. Prefiere early returns o combinar condiciones.

  P: ¿El operador ternario es más rápido que if/else?
  R: No hay diferencia de rendimiento significativa. La elección es
     puramente estilística.

  P: ¿Puedo tener un if sin else?
  R: Sí, es perfectamente válido. Si la condición es false, simplemente
     no se ejecuta nada y el valor de retorno es nil.

  P: ¿Cómo funciona la precedencia del modificador if?
  R: El modificador if tiene menor precedencia que la mayoría de
     operadores. La acción se evalúa primero, luego la condición.

  P: ¿Cuándo debo usar elsif vs case/when?
  R: Usa elsif para condiciones complejas (rangos, múltiples
     expresiones). Usa case/when para comparar un solo valor contra
     múltiples posibilidades.

  P: ¿Puedo usar unless con elsif?
  R: No. unless solo soporta else, no elsif. Para múltiples condiciones
     usa if/elsif/else.

  P: ¿Qué significa "if as a modifier"?
  R: Es cuando colocas if al final de una línea: acción if condición.
     Es equivalente a: if condición; acción; end.

RESUMEN TÉCNICO RÁPIDO
======================

if condición
  # código si true
end

if condición
  # código si true
else
  # código si false
end

if condición1
  # código
elsif condición2
  # código
else
  # código
end

# Modificador
acción if condición

# Ternario
variable = condición ? valor_true : valor_false

# If como expresión (devuelve valor)
resultado = if condición
              "valor1"
            else
              "valor2"
            end
