  BIENVENIDA
  ----------
  Bienvenido al Capítulo 17.  Has aprendido a definir métodos y a pasarles
  argumentos básicos.  Pero Ruby ofrece un sistema de argumentos rico y
  flexible que te permite escribir métodos más expresivos y adaptables.

  En este capítulo exploraremos:
  - Argumentos posicionales (los clásicos).
  - Valores por defecto.
  - Splat (*args) para argumentos variables.
  - Keyword arguments (clave: valor).
  - Doble splat (**kwargs) para keyword arguments variables.
  - El bloque como argumento (&block).
  - Todas estas variantes combinadas.

  Entender los argumentos en profundidad te permitirá diseñar APIs
  (interfaces de métodos) claras y flexibles.

  17.1  ARGUMENTOS POSICIONALES BÁSICOS
  --------------------------------------------------

  Los argumentos posicionales se asignan en el mismo orden en que se
  definen los parámetros.

  ------------------------------------------------------------------

  # Ejemplo 1: argumentos posicionales básicos
  def conectar(host, puerto, base_datos)
    "Conectando a #{host}:#{puerto}/#{base_datos}"
  end

  puts conectar("localhost", 3306, "tienda")
  # => Conectando a localhost:3306/tienda

  # Ejemplo 2: orden importa
  def restar(a, b)
    a - b
  end

  puts restar(10, 3)  # => 7
  puts restar(3, 10)  # => -7

  # Ejemplo 3: obligatoriedad
  def metodo_requerido(x, y)
    x + y
  end

  # metodo_requerido(5)       # ArgumentError (faltan argumentos)
  # metodo_requerido(1, 2, 3) # ArgumentError (sobran argumentos)

  # Ejemplo 4: paréntesis opcionales
  puts conectar "localhost", 5432, "test"
  # => Conectando a localhost:5432/test

  ------------------------------------------------------------------

================================================================================

  17.2  VALORES POR DEFECTO
  --------------------------------------------------

  Los parámetros pueden tener valores por defecto, haciéndolos
  opcionales en la llamada.

  ------------------------------------------------------------------

  # Ejemplo 1: valores por defecto básicos
  def crear_usuario(nombre, rol = "lector", activo = true)
    { nombre: nombre, rol: rol, activo: activo }
  end

  puts crear_usuario("Ana")
  # => {nombre: "Ana", rol: "lector", activo: true}

  puts crear_usuario("Luis", "admin")
  # => {nombre: "Luis", rol: "admin", activo: true}

  puts crear_usuario("Carlos", "editor", false)
  # => {nombre: "Carlos", rol: "editor", activo: false}

  # Ejemplo 2: valores por defecto con expresiones
  def conectar(host = "localhost", puerto = 3306, base = "db_#{host}")
    "#{host}:#{puerto}/#{base}"
  end

  puts conectar                      # => localhost:3306/db_localhost
  puts conectar("servidor")          # => servidor:3306/db_servidor
  puts conectar("srv", 5432)         # => srv:5432/db_srv
  puts conectar("srv", 5432, "admin") # => srv:5432/admin

  # Ejemplo 3: nil como valor por defecto (patrón común)
  def procesar(archivo, encoding = nil)
    encoding ||= "UTF-8"
    "Procesando #{archivo} con encoding #{encoding}"
  end

  puts procesar("datos.txt")             # => UTF-8
  puts procesar("datos.txt", "ISO-8859") # => ISO-8859

  # Ejemplo 4: valor por defecto que es una llamada a método
  def generar_id
    rand(1000..9999)
  end

  def registrar(nombre, id = generar_id)
    "Usuario: #{nombre}, ID: #{id}"
  end

  puts registrar("Ana")    # => ID aleatorio
  puts registrar("Luis", 1234)

================================================================================

  17.3  SPLAT: *ARGS (ARGUMENTOS VARIABLES)
  --------------------------------------------------

  El operador splat (*) captura múltiples argumentos en un array.
  Permite métodos con número variable de argumentos.

  ------------------------------------------------------------------

  # Ejemplo 1: splat básico
  def sumar_todos(*numeros)
    numeros.sum
  end

  puts sumar_todos(1, 2, 3)        # => 6
  puts sumar_todos(10, 20)         # => 30
  puts sumar_todos(1, 2, 3, 4, 5) # => 15
  puts sumar_todos                 # => 0 (array vacío)

  # Ejemplo 2: splat con parámetros fijos
  def presentar(saludo, *nombres)
    nombres.map { |n| "#{saludo}, #{n}!" }.join(" ")
  end

  puts presentar("Hola", "Ana", "Luis", "Carlos")
  # => Hola, Ana! Hola, Luis! Hola, Carlos!

  # Ejemplo 3: splat para delegar argumentos
  def envoltorio(*args)
    puts "Llamando a otro método con: #{args.inspect}"
    sumar_todos(*args)  # Expande el array de vuelta
  end

  puts envoltorio(1, 2, 3, 4)  # => 10

  # Ejemplo 4: splat para ignorar argumentos extra
  def solo_primero(primero, *resto)
    "Primero: #{primero}, ignorados: #{resto.length}"
  end

  puts solo_primero(1, 2, 3, 4, 5)  # => Primero: 1, ignorados: 4

  # Ejemplo 5: splat en la llamada (expande arrays)
  def saludar(nombre, edad, ciudad)
    "Soy #{nombre}, tengo #{edad} años y vivo en #{ciudad}"
  end

  datos = ["Ana", 28, "Madrid"]
  puts saludar(*datos)  # Expande el array en argumentos

  # Ejemplo 6: múltiples splat (solo uno por método)
  def combinar(*a, *b)  # Error! No puedes tener dos splats
  end

  # Pero puedes usar un solo splat:
  def listar(primero, *medios, ultimo)
    "Primero: #{primero}, medios: #{medios}, ultimo: #{ultimo}"
  end

  puts listar(1, 2, 3, 4, 5)  # => Primero: 1, medios: [2, 3, 4], ultimo: 5

  ------------------------------------------------------------------

  - *args captura 0 o más argumentos en un array.
- Solo un splat por lista de parámetros.
  - Puede ir al inicio, medio o final.
  - En la llamada, *array expande el array en argumentos individuales.

================================================================================

  17.4  KEYWORD ARGUMENTS (ARGUMENTOS NOMBRADOS)
  --------------------------------------------------

  Los keyword arguments permiten pasar argumentos por nombre, no por
  posición.  Esto hace las llamadas más claras y el orden no importa.

  ------------------------------------------------------------------

  # Ejemplo 1: keyword arguments básicos
  def conectar(host:, puerto:, base_datos:)
    "Conectando a #{host}:#{puerto}/#{base_datos}"
  end

  puts conectar(host: "localhost", puerto: 3306, base_datos: "tienda")
  puts conectar(base_datos: "test", host: "servidor", puerto: 5432)
  # El orden no importa!

  # Ejemplo 2: keyword arguments con valores por defecto
  def crear_usuario(nombre:, email:, rol: "lector", activo: true)
    { nombre: nombre, email: email, rol: rol, activo: activo }
  end

  puts crear_usuario(nombre: "Ana", email: "ana@test.com")
  # => {nombre: "Ana", email: "ana@test.com", rol: "lector", activo: true}

  puts crear_usuario(nombre: "Luis", email: "luis@test.com", rol: "admin")
  # => {nombre: "Luis", email: "luis@test.com", rol: "admin", activo: true}

  # Ejemplo 3: keyword arguments requeridos (sin default)
  def configurar(host:, puerto:, ssl: false, timeout: 30)
    "Config: #{host}:#{puerto} (SSL: #{ssl}, timeout: #{timeout}s)"
  end

  puts configurar(host: "api.com", puerto: 443)
  # => Config: api.com:443 (SSL: false, timeout: 30s)

  # puts configurar(host: "api.com")  # Error! Falta puerto:

  # Ejemplo 4: keyword arguments para claridad en métodos con muchos params
  # ❌ Difícil de leer:
  actualizar_usuario("Ana", 28, "Madrid", true, false, 3)

  # ✅ Fácil de leer:
  actualizar_usuario(nombre: "Ana", edad: 28, ciudad: "Madrid",
                     activo: true, admin: false, nivel: 3)

  def actualizar_usuario(nombre:, edad: nil, ciudad: nil,
                         activo: nil, admin: false, nivel: 1)
    "Actualizado: #{nombre}, nivel #{nivel}"
  end

  # Ejemplo 5: keyword arguments con hash (la vieja escuela)
  # Antes de Ruby 2.0, se usaban hashes como keyword arguments:
  def vieja_escuela(opciones = {})
    nombre = opciones[:nombre] || "Invitado"
    edad = opciones[:edad] || 0
    "#{nombre} (#{edad})"
  end

  puts vieja_escuela(nombre: "Ana", edad: 28)
  # Ahora mejor usar keyword arguments reales.

  ------------------------------------------------------------------

  - Los keyword arguments hacen las llamadas más autodocumentadas.
  - El orden no importa en la llamada.
  - Pueden ser requeridos (sin default) u opcionales (con default).
  - Son ideales para métodos con 3+ parámetros.

  17.5  DOBLE SPLAT: **KWARGS
  --------------------------------------------------

  El doble splat (**) captura keyword arguments adicionales como un
  hash.

  ------------------------------------------------------------------

  # Ejemplo 1: **kwargs básico
  def configurar(**opciones)
    puts "Opciones recibidas: #{opciones.inspect}"
    opciones.each { |k, v| puts "  #{k}: #{v}" }
  end

  configurar(host: "localhost", puerto: 3306, ssl: true, debug: false)

  # Ejemplo 2: **kwargs con keyword arguments fijos
  def conectar(host:, puerto:, **extra)
    puts "Conectando a #{host}:#{puerto}"
    puts "Opciones extra: #{extra}" unless extra.empty?
  end

  conectar(host: "srv", puerto: 5432, ssl: true, timeout: 30, pool: 5)
  # host y puerto se consumen, el resto va a **extra

  # Ejemplo 3: **kwargs para delegar opciones
  def validar(nombre:, **resto)
    puts "Validando #{nombre}..."
    procesar(nombre: nombre, **resto)  # Pasa el resto a otro método
  end

  def procesar(nombre:, email: nil, edad: nil)
    puts "Procesando: #{nombre}, #{email}, #{edad}"
  end

  validar(nombre: "Ana", email: "ana@test.com", edad: 28)

  # Ejemplo 4: Combinar *args y **kwargs
  def metodo_complejo(requerido, *opcionales, clave:, **extra)
    puts "Requerido: #{requerido}"
    puts "Opcionales: #{opcionales}"
    puts "Keyword: #{clave}"
    puts "Extra: #{extra}"
  end

  metodo_complejo("A", 1, 2, 3, clave: "X", otra: "Y", debug: true)

  # Ejemplo 5: **kwargs para "opciones passthrough"
  def ejecutar_comando(comando, **opciones)
  base = { verbose: false, timout: 30 }
  opciones_finales = base.merge(opciones)

    "Ejecutando #{comando} con #{opciones_finales}"
  end

  puts ejecutar_comando("ls -la", verbose: true, timout: 10)

  ------------------------------------------------------------------

  - **kwargs captura keyword arguments adicionales en un hash.
  - Solo un **kwargs por lista de parámetros.
  - Debe ir después de los parámetros regulares y keyword.
  - Útil para "pasar a través" opciones a otros métodos.

================================================================================

  17.6  EL BLOQUE COMO ARGUMENTO: &BLOCK
  --------------------------------------------------

  El operador & captura un bloque como un Proc.  Permite almacenar
  el bloque y ejecutarlo después.

  ------------------------------------------------------------------

  # Ejemplo 1: &block para capturar bloque
  def ejecutar_bloque(&bloque)
    puts "Ejecutando bloque..."
    bloque.call
    puts "Bloque ejecutado!"
  end

  ejecutar_bloque { puts "  Soy el bloque!" }

  # Ejemplo 2: &block con argumentos regulares
  def con_temporizador(nombre, &bloque)
    inicio = Time.now
    puts "Iniciando: #{nombre}"
    bloque.call
    duracion = Time.now - inicio
    puts "Finalizado: #{nombre} (#{duracion.round(2)}s)"
  end

  con_temporizador("Tarea 1") do
    sleep(0.1)  # Simula trabajo
    puts "  Trabajando..."
  end

  # Ejemplo 3: &block explícito vs bloque implícito
  # Método con bloque implícito (yield):
  def con_yield
    yield if block_given?
  end

  # Método con bloque explícito (&block):
  def con_block(&bloque)
    bloque.call if bloque
  end

  # Son equivalentes en funcionalidad, pero &block permite:
  # - Guardar el bloque en una variable.
  # - Pasar el bloque a otro método.
  # - Ejecutarlo múltiples veces.
  # - Verificar su existencia con bloque.nil?

  # Ejemplo 4: pasar el bloque a otro método
  def envoltorio(&bloque)
    puts "Envoltorio: antes"
    procesar(&bloque)  # Pasa el bloque a otro método
    puts "Envoltorio: después"
  end

  def procesar(&bloque)
    puts "  Procesar: antes del bloque"
    bloque.call if bloque
    puts "  Procesar: después del bloque"
  end

  envoltorio { puts "    Bloque ejecutado!" }

  # Ejemplo 5: convertir bloque a Proc y pasarlo
  def crear_proc(&bloque)
    bloque  # Devuelve el bloque como Proc
  end

  mi_proc = crear_proc { |x| puts "Recibí: #{x}" }
  mi_proc.call(42)  # => Recibí: 42

  # Ejemplo 6: & en la llamada para convertir Proc a bloque
  mi_lambda = ->(nombre) { puts "Hola #{nombre}" }
  metodo_que_espera_bloque(&mi_lambda)

  def metodo_que_espera_bloque(&bloque)
    bloque.call("Ana")
  end

  ------------------------------------------------------------------

  - &block captura el bloque como un objeto Proc.
  - &block debe ser el ÚLTIMO parámetro.
  - block.call ejecuta el bloque.
  - Puedes pasar el bloque a otros métodos con &.
  - Convierte Proc/Lambda a bloque con & en la llamada.

  17.7  ORDEN CORRECTO DE LOS PARÁMETROS
  --------------------------------------------------

  Ruby tiene un orden estricto para los parámetros en la definición.

  ------------------------------------------------------------------

  # Orden correcto:
  def metodo(requeridos, opcionales = [], *splat, keyword:, keyword_opcional: nil, **kwargs, &bloque)
    # cuerpo
  end

  # 1. Parámetros requeridos (posicionales)
  # 2. Parámetros con valor por defecto (opcionales)
  # 3. Splat (*args)
  # 4. Keyword arguments requeridos (nombre:)
  # 5. Keyword arguments opcionales (nombre: valor)
  # 6. Doble splat (**kwargs)
  # 7. Bloque (&block)

  # Ejemplo completo:
  def todo(a, b = 1, *c, d:, e: 2, **f, &g)
    puts "a=#{a}, b=#{b}, c=#{c}, d=#{d}, e=#{e}, f=#{f}"
    g.call if g
  end

  todo("A", "B", "C1", "C2", "C3", d: "D", extra1: "X", extra2: "Y") do
    puts "Bloque!"
  end
  # a=A, b=B, c=["C1", "C2", "C3"], d=D, e=2, f={extra1: "X", extra2: "Y"}
  # Bloque!

  # Error común: orden incorrecto
  # def mal(*a, b)  # Error? No, esto SÍ funciona (splat absorbe, b toma el último)
  # end

  # Pero es confuso:
  def mal(*a, b)
    "a=#{a}, b=#{b}"
  end

  puts mal(1, 2, 3, 4, 5)  # => a=[1, 2, 3, 4], b=5

  17.8  PATRONES AVANZADOS CON ARGUMENTOS
  --------------------------------------------------

  ------------------------------------------------------------------

  # Patrón 1: "Options Hash" (keyword arguments como hash)
  # Cuando necesitas pasar muchas opciones, usa keyword arguments.
  def conectar_bd(**opts)
    host = opts[:host] || "localhost"
    puerto = opts[:puerto] || 3306
    usuario = opts[:usuario] || "root"
    password = opts[:password] || ""
    base = opts[:base] || "test"

    "MySQL://#{usuario}@#{host}:#{puerto}/#{base}"
  end

  puts conectar_bd(usuario: "admin", base: "tienda")

  # Patrón 2: Forwarding de argumentos (delegación)
  def logger(accion, *args, **kwargs, &bloque)
    puts "[LOG] #{accion}: args=#{args}, kwargs=#{kwargs}"
    procesar(accion, *args, **kwargs, &bloque)
  end

  def procesar(accion, *args, **kwargs)
    "Procesando #{accion} con #{args} y #{kwargs}"
  end

  puts logger("calcular", 1, 2, 3, operacion: "suma")

  # Patrón 3: Argumentos requeridos con nil como default
  def conectar!(host: nil, puerto: nil, base: nil)
    raise ArgumentError, "host es requerido" if host.nil?
    raise ArgumentError, "puerto es requerido" if puerto.nil?
    "#{host}:#{puerto}/#{base}"
  end

  puts conectar!(host: "localhost", puerto: 3306, base: "test")
  # conectar!(host: "localhost")  # ArgumentError

  # Patrón 4: Método con argumentos mixtos para flexibilidad
  def buscar(termino, *campos, pagina: 1, por_pagina: 20, **filtros, &bloque)
    resultado = "Buscando '#{termino}' en campos: #{campos}, "
    resultado += "página #{pagina}, #{por_pagina} por página, "
    resultado += "filtros: #{filtros}"
    bloque.call(resultado) if bloque
    resultado
  end

  buscar("ruby", "titulo", "descripcion", pagina: 2, por_pagina: 10,
         año: 2024, idioma: "es") { |r| puts "Resultado: #{r}" }

  # "Patrón 5: Factory method con keyword arguments"
  class Punto
    attr_reader :x, :y

    def initialize(x:, y:)
      @x = x
      @y = y
    end

    def self.crear(x, y)
      new(x: x, y: y)
    end

    def to_s
      "(#{@x}, #{@y})"
    end
  end

  p = Punto.new(x: 3, y: 4)
  puts p  # => (3, 4)

  17.9  EJEMPLO INTEGRADOR: API DE CONFIGURACIÓN
  --------------------------------------------------

  ------------------------------------------------------------------

  class ConfiguracionAPI
    def initialize(**opciones_iniciales)
      @config = {
        host: "localhost",
        puerto: 3000,
        ssl: false,
        timeout: 30,
        max_requests: 100,
        debug: false
      }.merge(opciones_iniciales)
    end

    def configurar(**nuevas_opciones)
      @config.merge!(nuevas_opciones)
      self
    end

    def mostrar
      @config.each { |k, v| puts "  #{k}: #{v}" }
    end

    def ejecutar(ruta, *args, **params, &bloque)
      url = construir_url(ruta, **params)
      puts "[#{@config[:host]}:#{@config[:puerto]}] #{url}"

      respuesta = simular_peticion(url, *args)
      bloque.call(respuesta) if bloque
      respuesta
    end

    private

    def construir_url(ruta, **parametros)
      base = "#{@config[:ssl] ? 'https' : 'http'}://"
      base += "#{@config[:host]}:#{@config[:puerto]}/#{ruta}"
      unless parametros.empty?
        query = parametros.map { |k, v| "#{k}=#{v}" }.join("&")
        base += "?#{query}"
      end
      base
    end

    def simular_peticion(url, *args)
      { status: 200, body: "Respuesta de #{url}", args: args }
    end
  end

  api = ConfiguracionAPI.new(host: "api.miservicio.com", ssl: true, debug: true)
  api.configurar(timeout: 60, max_requests: 500)

  puts "Configuración actual:"
  api.mostrar

  puts "\nEjecutando peticiones:"
  api.ejecutar("usuarios", "admin", pagina: 1, limite: 10) do |resp|
    puts "  Código: #{resp[:status]}"
    puts "  Body: #{resp[:body]}"
  end

  api.ejecutar("productos", "busqueda", q: "ruby", orden: "precio")

  # Uso con bloque como argumento explícito
  bloque_personalizado = ->(resp) {
    puts "  [Callback] Status: #{resp[:status]}"
  }

  api.ejecutar("dashboard", &bloque_personalizado)

  17.10  BUENAS PRÁCTICAS CON ARGUMENTOS
  --------------------------------------------------

  ------------------------------------------------------------------

  # 1. Usa keyword arguments para métodos con 3+ parámetros
  # ✅ Bueno:
  def crear( nombre:, email:, edad: nil, rol: "usuario")
  end

  # 2. Usa valores por defecto para hacer parámetros opcionales
  # ✅ Bueno:
  def conectar(host = "localhost", puerto = 3306)
  end

  # 3. Usa *args para métodos que aceptan cualquier número de args
  # ✅ Bueno:
  def sumar(*numeros)
  end

  # 4. Usa **kwargs para opciones adicionales no definidas
  # ✅ Bueno:
  def enviar_email(para:, asunto:, **opciones)
  end

  # 5. Usa &block cuando necesites manipular el bloque
  # ✅ Bueno:
  def con_manejo_errores(&bloque)
    begin
      bloque.call
    rescue => e
      puts "Error: #{e.message}"
    end
  end

  # 6. Prefiere keyword arguments sobre hashes simples
  # ❌ Vieja escuela:
  def crear_usuario(opciones = {})
    nombre = opciones[:nombre]
  end

  # ✅ Moderno:
  def crear_usuario(nombre:, email:, rol: "lector")
  end

  # 7. Sé consistente: no mezcles estilos sin razón
  # 8. Documenta los parámetros cuando sea necesario
  # 9. Limita el número de parámetros (ideal: <= 4)
  # 10. Usa nombres descriptivos para los parámetros

  ------------------------------------------------------------------

================================================================================

  RESUMEN DEL CAPÍTULO
  ====================

  1. Argumentos posicionales: orden importa, pueden ser obligatorios.
  2. Valores por defecto: hacen parámetros opcionales.
  3. *args (splat): argumentos variables en un array.
  4. Keyword arguments (nombre:): claridad, orden no importa.
  5. **kwargs (doble splat): keyword arguments variables como hash.
  6. &block: captura el bloque como Proc.
  7. Orden: requeridos, default, *splat, keywords, **kwargs, &block.
  8. En la llamada, *array expande, **hash expande, &proc convierte.
  9. Keyword arguments son preferibles para APIs claras.
  10. El bloque es un argumento especial y poderoso.

  PREGUNTAS FRECUENTES (FAQ)
  ==========================

  P: ¿Cuál es la diferencia entre *args y **kwargs?
  R: *args captura argumentos posicionales extra en un array. **kwargs
     captura keyword arguments extra en un hash.

  P: ¿Puedo tener un método sin paréntesis en los parámetros?
  R: La definición siempre lleva paréntesis, pero la llamada no los
     requiere.

  P: ¿Qué pasa si paso más argumentos de los que el método espera?
  R: Depende. Si no hay *args, obtienes ArgumentError. Si hay *args,
     los extra van allí.

  P: ¿Puedo mezclar argumentos posicionales y keyword?
  R: Sí, pero los keyword deben ir después de los posicionales.

  P: ¿Los keyword arguments son más lentos?
  R: Mínimamente. La claridad compensa cualquier diferencia.

  P: ¿Debo siempre usar keyword arguments?
  R: No. Para 1-2 parámetros simples, los posicionales están bien.

  P: ¿Cómo hago un parámetro requerido con keyword?
  R: Solo pon nombre: sin valor por defecto.

  P: ¿Puedo pasar un hash como keyword arguments?
  R: Sí, con **hash en la llamada: metodo(**hash)

  P: ¿Cuál es la diferencia entre &block y yield?
  R: yield ejecuta el bloque implícito. &block captura el bloque como
     objeto. &block es más flexible (puedes guardarlo, pasarlo, etc.)

  P: ¿Puedo tener más de un &block?
  R: No, solo un bloque por método.

RESUMEN TÉCNICO RÁPIDO
======================

# Posicionales
def metodo(a, b, c)

# Valores por defecto
def metodo(a, b = 1, c = 2)

# Splat
def metodo(*args)
def metodo(primero, *resto, ultimo)

# Keyword arguments
def metodo(nombre:, edad: 0, activo: true)

# Doble splat
def metodo(**kwargs)

# Bloque
def metodo(&block)

# Orden completo
def metodo(req, opt = 1, *splat, kw:, kw_opt: 2, **kwargs, &block)

# En la llamada
metodo(*array)    # Expande array a posicionales
metodo(**hash)    # Expande hash a keywords
metodo(&proc)     # Convierte proc a bloque
