  16.1  DEF / END — DEFINICIÓN BÁSICA
  --------------------------------------------------

  La forma más simple de definir un método es con def seguido del
  nombre del método y end.

  ------------------------------------------------------------------

  # Ejemplo 1: método sin parámetros
  def saludar
    puts "Hola, bienvenido a Ruby!"
  end

  # Llamada al método
  saludar  # => Hola, bienvenido a Ruby!

  # Ejemplo 2: método que devuelve un valor
  def suma_dos
    2 + 2
  end

  resultado = suma_dos
  puts resultado  # => 4

  # Ejemplo 3: método con parámetros
  def saludar_persona(nombre)
    "Hola, #{nombre}!"
  end

  puts saludar_persona("Ana")  # => Hola, Ana!
  puts saludar_persona("Luis") # => Hola, Luis!

  # Ejemplo 4: método con múltiples parámetros
  def sumar(a, b)
    a + b
  end

  puts sumar(3, 4)  # => 7

  # Ejemplo 5: método que llama a otro método
  def cuadrado(n)
    n * n
  end

  def suma_de_cuadrados(a, b)
    cuadrado(a) + cuadrado(b)
  end

  puts suma_de_cuadrados(3, 4)  # => 25

  # Ejemplo 6: método con lógica condicional
  def mayor_edad(edad)
    if edad >= 18
      "Mayor de edad"
    else
      "Menor de edad"
    end
  end

  puts mayor_edad(20)  # => Mayor de edad
  puts mayor_edad(15)  # => Menor de edad

  ------------------------------------------------------------------

  - def inicia la definición, end la termina.
  - El nombre del método debe empezar con letra minúscula o _.
  - Los métodos deben definirse antes de ser llamados (Ruby evalúa
    el archivo de arriba a abajo).
  - Los paréntesis en la llamada son opcionales: sumar 3, 4 funciona.

  16.2  RETURN — IMPLÍCITO Y EXPLÍCITO
  --------------------------------------------------

  Ruby es único: el valor de retorno de un método es la ÚLTIMA
  expresión evaluada.  No necesitas escribir return explícitamente
  a menos que quieras salir antes.

  ------------------------------------------------------------------

  # Return implícito (última expresión)
  def multiplicar(a, b)
    a * b  # Esta es la última expresión, es el valor de retorno
  end

  puts multiplicar(4, 5)  # => 20

  # Return explícito
  def dividir(a, b)
    return "Error: división entre cero" if b == 0
    a / b
  end

  puts dividir(10, 2)   # => 5
  puts dividir(10, 0)   # => Error: división entre cero

  # Return explícito en múltiples puntos
  def clasificar_numero(n)
    return "Cero" if n == 0
    return "Positivo" if n > 0
    "Negativo"
  end

  puts clasificar_numero(0)   # => Cero
  puts clasificar_numero(5)   # => Positivo
  puts clasificar_numero(-3)  # => Negativo

  # Return sin valor (devuelve nil)
  def mostrar_mensaje(msg)
    puts msg
    return  # Opcional: sale devolviendo nil
  end

  resultado = mostrar_mensaje("Hola")
  puts resultado.inspect  # => nil

  # Return con múltiples valores (devuelve array)
  def dividir_con_residuo(a, b)
    [a / b, a % b]
  end

  cociente, residuo = dividir_con_residuo(10, 3)
  puts "Cociente: #{cociente}, Residuo: #{residuo}"

  # Return implícito con if (cuidado!)
  def maximo(a, b)
    if a > b
      a
    else
      b
    end
  end
  # El if completo es la última expresión

  # Versión más concisa con ternario:
  def maximo(a, b)
    a > b ? a : b
  end

  ------------------------------------------------------------------

  - RETURN IMPLÍCITO: Ruby devuelve la última expresión.
  - RETURN EXPLÍCITO: Usa return para salir antes o para claridad.
  - return sin argumentos devuelve nil.
  - return puede devolver múltiples valores como array.
  - El estilo Ruby prefiere return implícito para métodos simples.

  16.3  CONVENCIONES DE NOMBRES: ? Y !
  --------------------------------------------------

  Ruby tiene convenciones de nomenclatura que hacen el código más
  expresivo.  Los métodos pueden terminar en ? o !.

  ------------------------------------------------------------------

  # Métodos que terminan en ? — devuelven true/false (predicados)
  def par?(n)
    n % 2 == 0
  end

  puts par?(4)   # => true
  puts par?(7)   # => false

  def vacio?(array)
    array.empty?
  end

  puts vacio?([])      # => true
  puts vacio?([1, 2])  # => false

  def puede_votar?(persona)
    persona[:edad] >= 18 && persona[:nacionalidad] == "mexicana"
  end

  persona = { nombre: "Ana", edad: 22, nacionalidad: "mexicana" }
  puts puede_votar?(persona)  # => true

  # Métodos que terminan en ! — versión peligrosa o modificadora
  # Por convención: ! indica que el método MODIFICA el objeto receptor
  # o que es una versión más "peligrosa" que la versión sin !

  # Ejemplo con String:
  texto = "  hola  "
  puts texto.strip   # => "hola" (sin cambiar texto original)
  puts texto         # => "  hola  "
  puts texto.strip!  # => "hola" (modifica texto)
  puts texto         # => "hola"

  # Nuestro propio método con !
  def invertir!(array)
    array.replace(array.reverse)
  end

  arr = [1, 2, 3]
  invertir!(arr)
  puts arr.inspect  # => [3, 2, 1]

  # Método sin ! (no modifica)
  def invertir(array)
    array.reverse
  end

  arr2 = [1, 2, 3]
  resultado = invertir(arr2)
  puts arr2.inspect       # => [1, 2, 3] (sin cambios)
  puts resultado.inspect  # => [3, 2, 1]

  ------------------------------------------------------------------

  - ?: indica que el método devuelve un booleano (predicado).
  - !: indica que el método modifica el objeto receptor o es "peligroso".
  - Son CONVENCIONES, no reglas del lenguaje.
  - Siempre define la versión ! junto con la versión sin !.

  16.4  PARÁMETROS POR DEFECTO
  --------------------------------------------------

  Puedes asignar valores por defecto a los parámetros.  Si el llamador
  no proporciona ese argumento, se usa el valor por defecto.

  ------------------------------------------------------------------

  # Ejemplo 1: parámetro por defecto simple
  def saludar(nombre = "invitado")
    "Hola, #{nombre}!"
  end

  puts saludar("Ana")   # => Hola, Ana!
  puts saludar          # => Hola, invitado!

  # Ejemplo 2: múltiples parámetros por defecto
  def crear_email(nombre, dominio = "example.com", extension = "com")
    "#{nombre.downcase}@#{dominio}.#{extension}"
  end

  puts crear_email("Ana")                         # => ana@example.com
  puts crear_email("Luis", "corp.com")             # => luis@corp.com.com
  puts crear_email("Carlos", "gmail", "com")       # => carlos@gmail.com

  # Ejemplo 3: parámetros por defecto con expresiones
  def calcular_precio(base, impuesto = base * 0.16, descuento = 0)
    base + impuesto - descuento
  end

  puts calcular_precio(100)              # => 116.0
  puts calcular_precio(100, 10)          # => 110.0
  puts calcular_precio(100, 10, 5)       # => 105.0

  # Ejemplo 4: parámetros por defecto que dependen de otros
  def conectar(host, puerto = 3306, base_datos = "db_#{host}")
    "Conectando a #{host}:#{puerto}/#{base_datos}"
  end

  puts conectar("localhost")           # => Conectando a localhost:3306/db_localhost
  puts conectar("servidor", 5432)      # => Conectando a servidor:5432/db_servidor
  puts conectar("srv", 27017, "admin") # => Conectando a srv:27017/admin

  ------------------------------------------------------------------

  - Los valores por defecto se evalúan en cada llamada.
  - Si el valor por defecto es una expresión, se evalúa cuando se
    llama al método, no cuando se define.
  - Los parámetros con valores por defecto pueden ir antes o después
    de los requeridos, pero es más claro colocarlos al final.

  16.5  MÉTODOS QUE LLAMAN A OTROS MÉTODOS
  --------------------------------------------------

  La composición de métodos es la base de la programación modular.

  ------------------------------------------------------------------

  # Ejemplo 1: composición simple
  def cuadrado(n)
    n * n
  end

  def suma_cuadrados(a, b)
    cuadrado(a) + cuadrado(b)
  end

  def hipotenusa(a, b)
    Math.sqrt(suma_cuadrados(a, b))
  end

  puts hipotenusa(3, 4)  # => 5.0

  # Ejemplo 2: validación compuesta
  def email_valido?(email)
    email =~ /\A[\w+\-.]+@[a-z\d\-]+(\.[a-z\d\-]+)*\.[a-z]+\z/i
  end

  def edad_valida?(edad)
    edad.is_a?(Integer) && edad >= 0 && edad <= 150
  end

  def usuario_valido?(usuario)
    email_valido?(usuario[:email]) && edad_valida?(usuario[:edad])
  end

  usuario = { nombre: "Ana", email: "ana@test.com", edad: 28 }
  puts usuario_valido?(usuario)  # => true

  # Ejemplo 3: métodos de ayuda (helpers)
  def formatear_fecha(dia, mes, anio)
    "#{dia.to_s.rjust(2, '0')}/#{mes.to_s.rjust(2, '0')}/#{anio}"
  end

  def formatear_hora(hora, minuto)
    "#{hora.to_s.rjust(2, '0')}:#{minuto.to_s.rjust(2, '0')}"
  end

  def formatear_datetime(dia, mes, anio, hora, minuto)
    "#{formatear_fecha(dia, mes, anio)} #{formatear_hora(hora, minuto)}"
  end

  puts formatear_datetime(5, 12, 2024, 14, 30)
  # => 05/12/2024 14:30

  16.6  SCOPE Y VARIABLES EN MÉTODOS
  --------------------------------------------------

  Los métodos crean su propio ámbito (scope).  Las variables definidas
  dentro de un método no son accesibles fuera.

  ------------------------------------------------------------------

  # Variables locales dentro de métodos
  def metodo_uno
    x = 10
    puts "Dentro: x = #{x}"
  end

  metodo_uno  # => Dentro: x = 10
  # puts x    # Error: undefined local variable x

  # Variables externas no afectadas por el método
  total = 100

  def calcular(descuento)
    total = 50         # Esta es una NUEVA variable local
    total - descuento
  end

  puts calcular(10)   # => 40
  puts total          # => 100 (la variable externa no cambió)

  # Para modificar una variable externa, pásala como argumento
  # y devuelve el nuevo valor, o usa una variable de instancia.

  def aplicar_descuento(total, descuento)
    total - descuento
  end

  total = 100
  total = aplicar_descuento(total, 15)
  puts total  # => 85

  # Variables de instancia (@) son accesibles desde cualquier método
  # de la misma instancia.
  class Calculadora
    def initialize(valor)
      @valor = valor
    end

    def sumar(n)
      @valor += n
    end

    def resultado
      @valor
    end
  end

  calc = Calculadora.new(10)
  calc.sumar(5)
  puts calc.resultado  # => 15

  ------------------------------------------------------------------

  - Las variables creadas dentro de un método son locales.
  - Las variables externas no se modifican desde dentro (a menos que
    sean de instancia @, de clase @@, o globales $).
  - Para devolver un valor modificado, usa return.
  - Las variables de instancia (@) persisten entre llamadas de métodos
    de la misma instancia.

  16.7  MÉTODOS PREDICADO (BOOLEANOS)
  --------------------------------------------------

  Los métodos que devuelven true/false se llaman predicados y por
  convención terminan en ?.

  ------------------------------------------------------------------

  # Ejemplo 1: predicados matemáticos
  def par?(n)
    n % 2 == 0
  end

  def primo?(n)
    return false if n <= 1
    (2..Math.sqrt(n)).none? { |i| n % i == 0 }
  end

  def perfecto?(n)
    return false if n <= 1
    (1...n).select { |i| n % i == 0 }.sum == n
  end

  puts par?(6)       # => true
  puts primo?(7)     # => true
  puts perfecto?(6)  # => true (1+2+3=6)
  puts perfecto?(7)  # => false

  # Ejemplo 2: predicados de validación
  def mayuscula?(letra)
    letra =~ /[A-Z]/
  end

  def minuscula?(letra)
    letra =~ /[a-z]/
  end

  def digito?(c)
    c =~ /\d/
  end

  puts mayuscula?("A")   # => 0 (truthy)
  puts mayuscula?("a")   # => nil (falsy)
  puts digito?("5")      # => 0 (truthy)

  # Ejemplo 3: predicados de colecciones
  def todos_pares?(array)
    array.all? { |n| n % 2 == 0 }
  end

  def algun_par?(array)
    array.any? { |n| n % 2 == 0 }
  end

  def ningun_par?(array)
    array.none? { |n| n % 2 == 0 }
  end

  puts todos_pares?([2, 4, 6])  # => true
  puts algun_par?([1, 3, 4])    # => true
  puts ningun_par?([1, 3, 5])   # => true

  # Ejemplo 4: predicado compuesto
  def acceso_permitido?(usuario, accion)
    return false unless usuario[:activo]
    return true if usuario[:rol] == "admin"

    case accion
    when :leer then true
    when :escribir then usuario[:rol] == "editor"
    when :borrar then usuario[:rol] == "superadmin"
    else false
    end
  end

  usuario = { nombre: "Ana", rol: "editor", activo: true }
  puts acceso_permitido?(usuario, :leer)     # => true
  puts acceso_permitido?(usuario, :borrar)   # => false

  ------------------------------------------------------------------

  16.8  MÉTODOS MODIFICADORES (!)
  --------------------------------------------------

  Los métodos que terminan en ! modifican el objeto receptor.  La
  versión sin ! devuelve una copia modificada.

  ------------------------------------------------------------------

  # Ejemplo 1: Implementación de método con y sin !
  class Array
    def ordenar_asc!
      self.sort!
    end

    def ordenar_asc
      self.sort
    end
  end

  # Nota: sort y sort! ya existen en Array. Este es un ejemplo conceptual.

  # Ejemplo 2: método personalizado con !
  class GestorTareas
    attr_reader :tareas

    def initialize
      @tareas = []
    end

    def agregar(tarea)
      @tareas << { tarea: tarea, completada: false }
    end

    def completar!(indice)
      return unless indice >= 0 && indice < @tareas.length
      @tareas[indice][:completada] = true
      @tareas[indice]
    end

    def completar(indice)
      return unless indice >= 0 && indice < @tareas.length
      tarea_modificada = @tareas[indice].dup
      tarea_modificada[:completada] = true
      tarea_modificada
    end
  end

  gestor = GestorTareas.new
  gestor.agregar("Estudiar Ruby")
  gestor.agregar("Hacer ejercicios")

  # Versión ! (modifica el original):
  gestor.completar!(0)
  puts gestor.tareas.inspect

  # Versión sin ! (no modifica):
  resultado = gestor.completar(1)
  puts resultado.inspect         # Tarea completada temporalmente
  puts gestor.tareas.inspect     # Tarea original sin modificar

  # Ejemplo 3: ! para métodos "peligrosos"
  def borrar_archivo!(ruta)
    return "Archivo no existe" unless File.exist?(ruta)
    File.delete(ruta)
    "Archivo eliminado"
  end

  # def borrar_archivo(ruta)  # Versión segura: simula la operación
  #   return "Archivo no existe" unless File.exist?(ruta)
  #   "Se eliminaría: #{ruta}"
  # end

  ------------------------------------------------------------------

  - Los métodos ! deben modificar el objeto receptor.
  - Siempre ofrece una versión sin ! junto con la versión con !.
  - El ! es una advertencia: "cuidado, esto modifica el original".

  16.9  PRINCIPIO DE RESPONSABILIDAD ÚNICA
  --------------------------------------------------

  Uno de los principios más importantes de diseño: cada método debe
  hacer UNA sola cosa.

  ------------------------------------------------------------------

  # ❌ MAL: método que hace demasiadas cosas
  def procesar_usuario(nombre, email, edad)
    # Validar
    return "Nombre inválido" if nombre.nil? || nombre.empty?
    return "Email inválido" unless email =~ /\A[\w+\-.]+@[a-z\d\-]+(\.[a-z\d\-]+)*\.[a-z]+\z/i
    return "Edad inválida" unless edad.is_a?(Integer) && edad >= 0

    # Transformar
    nombre_formateado = nombre.strip.capitalize
    email_formateado = email.downcase.strip

    # Guardar
    usuario = { nombre: nombre_formateado, email: email_formateado, edad: edad }
    # DB.save(usuario)  # simulación
    "Usuario creado exitosamente"
  end

  # ✅ BIEN: cada método hace una sola cosa
  def validar_nombre(nombre)
    return "Nombre inválido" if nombre.nil? || nombre.empty?
    true
  end

  def validar_email(email)
    return "Email inválido" unless email =~ /\A[\w+\-.]+@[a-z\d\-]+(\.[a-z\d\-]+)*\.[a-z]+\z/i
    true
  end

  def validar_edad(edad)
    return "Edad inválida" unless edad.is_a?(Integer) && edad >= 0
    true
  end

  def formatear_nombre(nombre)
    nombre.strip.capitalize
  end

  def formatear_email(email)
    email.downcase.strip
  end

  def crear_usuario(nombre, email, edad)
    # Validaciones
    return validar_nombre(nombre) unless validar_nombre(nombre) == true
    return validar_email(email) unless validar_email(email) == true
    return validar_edad(edad) unless validar_edad(edad) == true

    # Transformación
    nombre_f = formatear_nombre(nombre)
    email_f = formatear_email(email)

    # Creación
    { nombre: nombre_f, email: email_f, edad: edad }
  end

  resultado = crear_usuario("  ana  ", "Ana@Test.com", 28)
  puts resultado.inspect  # => {nombre: "Ana", email: "ana@test.com", edad: 28}

  # Beneficios de métodos pequeños:
  # 1. Fáciles de probar individualmente.
  # 2. Fáciles de entender.
  # 3. Reutilizables en otros contextos.
  # 4. Fáciles de modificar sin efectos secundarios.

  ------------------------------------------------------------------

  16.10  MÉTODOS DE CLASE VS DE INSTANCIA
  --------------------------------------------------

  Si bien profundizaremos en POO más adelante, es importante distinguir
  los tipos de métodos.

  ------------------------------------------------------------------

  # Métodos de instancia (se llaman sobre una instancia)
  class Persona
    def initialize(nombre)
      @nombre = nombre
    end

    # Método de instancia
    def saludar
      "Hola, soy #{@nombre}"
    end

    # Método de clase (se llama sobre la clase misma)
    def self.especie
      "Homo sapiens"
    end

    # Otra forma de definir método de clase
    class << self
      def crear_anonimo
        new("Anónimo")
      end
    end
  end

  persona = Persona.new("Ana")
  puts persona.saludar       # => Hola, soy Ana
  puts Persona.especie        # => Homo sapiens
  puts Persona.crear_anonimo.saludar  # => Hola, soy Anónimo

  # Diferencia clave:
  # - Métodos de instancia: operan sobre una instancia específica.
  # - Métodos de clase: operan sobre la clase en general.

  ------------------------------------------------------------------

  16.11  EJEMPLO INTEGRADOR: CALCULADORA MODULAR
  --------------------------------------------------

  ------------------------------------------------------------------

  # Calculadora modular con métodos de responsabilidad única

  class Calculadora
    # Validaciones
    def self.numero_valido?(n)
      n.is_a?(Numeric)
    end

    def self.validar_operacion(a, b, operador)
      return "Ambos deben ser números" unless numero_valido?(a) && numero_valido?(b)
      return "División entre cero" if operador == '/' && b == 0
      nil  # Sin error
    end

    # Operaciones básicas
    def self.sumar(a, b)
      a + b
    end

    def self.restar(a, b)
      a - b
    end

    def self.multiplicar(a, b)
      a * b
    end

    def self.dividir(a, b)
      a / b
    end

    def self.potencia(base, exp)
      base ** exp
    end

    def self.raiz_cuadrada(n)
      Math.sqrt(n)
    end

    # Operador general
    def self.calcular(a, b, operador)
      error = validar_operacion(a, b, operador)
      return error if error

      case operador
      when '+' then sumar(a, b)
      when '-' then restar(a, b)
      when '*' then multiplicar(a, b)
      when '/' then dividir(a.to_f, b.to_f)
      when '**' then potencia(a, b)
      when 'sqrt' then raiz_cuadrada(a)
      else "Operador no soportado"
      end
    end

    # Métodos de conveniencia
    def self.sumar_array(numbers)
      return 0 if numbers.empty?
      numbers.sum
    end

    def self.promedio(numbers)
      return 0 if numbers.empty?
      sumar_array(numbers).to_f / numbers.length
    end

    def self.factorial(n)
      return "Debe ser entero positivo" unless n.is_a?(Integer) && n >= 0
      return 1 if n <= 1
      (1..n).reduce(:*)
    end

    # Estadísticas
    def self.minimo(array)
      array.min
    end

    def self.maximo(array)
      array.max
    end

    def self.mediana(array)
      sorted = array.sort
      mid = sorted.length / 2
      sorted.length.odd? ? sorted[mid] : (sorted[mid - 1] + sorted[mid]) / 2.0
    end
  end

  puts Calculadora.calcular(10, 5, '+')     # => 15
  puts Calculadora.calcular(10, 5, '-')     # => 5
  puts Calculadora.calcular(10, 5, '*')     # => 50
  puts Calculadora.calcular(10, 5, '/')     # => 2.0
  puts Calculadora.calcular(10, 0, '/')     # => División entre cero
  puts Calculadora.calcular(2, 8, '**')     # => 256
  puts Calculadora.calcular(25, nil, 'sqrt') # => 5.0
  puts Calculadora.factorial(5)             # => 120
  puts Calculadora.sumar_array([1,2,3,4,5]) # => 15
  puts Calculadora.promedio([10, 20, 30])   # => 20.0
  puts Calculadora.mediana([1, 3, 5, 7, 9]) # => 5

  PREGUNTAS FRECUENTES (FAQ)
  ==========================

  P: ¿Cuál es la diferencia entre def y def self?
  R: def define métodos de instancia; def self.method define métodos de
     clase.

  P: ¿Debo usar return siempre?
  R: No. Ruby tiene return implícito. Usa return explícito solo para
     early returns o claridad.

  P: ¿Puedo tener un método vacío?
  R: Sí, devuelve nil. A veces útil como placeholder.

  P: ¿Los paréntesis son obligatorios?
  R: No. puts "hola" es válido. Los paréntesis son opcionales.

  P: ¿Puedo definir un método dentro de otro método?
  R: Sí, Ruby permite métodos anidados, pero no se comportan como
     closures. No es una práctica común.

  P: ¿Cuál es la diferencia entre método y función?
  R: En Ruby, todo es método (incluso los "métodos" globales son
     métodos privados de Object). No hay funciones independientes.

  P: ¿Cómo documentar métodos?
  R: Comentarios arriba del método, o usando RDoc/YARD.

  P: ¿Qué es un método predicado?
  R: Método que termina en ? y devuelve true o false.

  P: ¿Qué significa el ! al final de un método?
  R: Indica que el método modifica el objeto receptor o es "peligroso".

  P: ¿Puedo tener métodos con el mismo nombre pero diferentes parámetros?
  R: Ruby no soporta sobrecarga (overloading). Si defines un método dos
     veces, el último reemplaza al anterior.
