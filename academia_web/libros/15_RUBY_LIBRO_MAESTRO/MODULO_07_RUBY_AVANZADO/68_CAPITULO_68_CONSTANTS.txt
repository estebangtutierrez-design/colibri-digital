================================================================================
                        BIENVENIDA AL CAPÍTULO 68
================================================================================

  Bienvenido al estudio de las constantes en Ruby. A primera vista,
  una constante parece simple: un valor que no cambia. Pero en Ruby,
  las constantes tienen un comportamiento único que las hace mucho más
  interesantes (y a veces sorprendentes).

  En Ruby, las constantes comienzan con mayúscula, pero no son
  completamente inmutables. Ruby te advierte cuando reasignas una
  constante, pero no te lo impide. Además, las constantes en Ruby
  son más parecidas a "nombres que se espera que no cambien" que a
  verdaderas constantes inmutables.

  También exploraremos la jerarquía de búsqueda de constantes, cómo
  congelar objetos con freeze, y los métodos dinámicos const_get y
  const_set.

-------------------------------------------------------------------------------
  1. ¿QUÉ ES UNA CONSTANTE EN RUBY?
-------------------------------------------------------------------------------

  Una constante en Ruby es cualquier identificador que comienza con
  mayúscula:

    PI = 3.14159
    NOMBRE_APP = "Ruby Master"
    MAX_INTENTOS = 3

  A diferencia de otros lenguajes, las constantes en Ruby pueden ser
  reasignadas (con una advertencia):

    PI = 3.14159
    PI = 3.14  # warning: already initialized constant PI

  La convención es usar MAYÚSCULAS con guiones bajos.

-------------------------------------------------------------------------------
  2. CONSTANTES VS VARIABLES
-------------------------------------------------------------------------------

  Diferencia clave: Las constantes comienzan con mayúscula, las
  variables con minúscula.

    nombre = "Ana"       # variable local
    Nombre = "Ana"       # constante
    NOMBRE = "Ana"       # constante (convención)

  Las constantes tienen visibilidad: son accesibles dentro de la clase
  o módulo donde se definen, y a través de la jerarquía.

-------------------------------------------------------------------------------
  3. ÁMBITO DE LAS CONSTANTES
-------------------------------------------------------------------------------

  Las constantes se buscan primero en el ámbito léxico actual, luego
  en la jerarquía de herencia, y finalmente en el ámbito superior:

    module ModuloA
      VALOR = 10

      module ModuloB
        VALOR = 20

        def self.mostrar
          puts VALOR  # 20 (ámbito léxico)
        end
      end
    end

    puts ModuloA::VALOR         # 10
    puts ModuloA::ModuloB::VALOR # 20

-------------------------------------------------------------------------------
  4. BÚSQUEDA DE CONSTANTES (CONSTANT LOOKUP)
-------------------------------------------------------------------------------

  Ruby busca constantes en este orden:
    1. Ámbito léxico actual (clase/módulo donde se hace referencia)
    2. Ámbito léxico superior
    3. Jerarquía de herencia (superclases y módulos incluidos)
    4. Ámbito superior (Object)

    class Padre
      CONST = "Padre"
    end

    module Mixin
      CONST = "Mixin"
    end

    class Hijo < Padre
      include Mixin
      CONST = "Hijo"

      def mostrar
        CONST  # "Hijo"
      end
    end

-------------------------------------------------------------------------------
  5. REASIGNACIÓN VS MUTACIÓN
-------------------------------------------------------------------------------

  Reasignar una constante produce una advertencia, pero mutar el objeto
  al que apunta no:

    LISTA = [1, 2, 3]
    LISTA << 4         # OK, mutación (sin advertencia)
    puts LISTA.inspect  # [1, 2, 3, 4]

    LISTA = [5, 6, 7]  # warning: already initialized constant

  La constante es una referencia al objeto, no el objeto en sí.

-------------------------------------------------------------------------------
  6. FREEZE: INMUTABILIDAD
-------------------------------------------------------------------------------

  freeze evita la modificación de un objeto:

    VALORES = [1, 2, 3].freeze
    # VALORES << 4  # FrozenError: can't modify frozen array

    CONFIG = { host: "localhost" }.freeze
    # CONFIG[:host] = "otro"  # FrozenError

  freeze solo congela el objeto inmediato, no objetos anidados:

    DATOS = { lista: [1, 2, 3] }.freeze
    DATOS[:lista] << 4  # Funciona (el array interno no está frozen)

  Para congelar profundamente, necesitas deep_freeze.

-------------------------------------------------------------------------------
  7. DEEP FREEZE
-------------------------------------------------------------------------------

  Congelar recursivamente:

    module DeepFreezable
      refine Object do
        def deep_freeze
          freeze
          instance_variables.each do |var|
            instance_variable_get(var).deep_freeze
          end
          self
        end

        def deep_frozen?
          frozen? && instance_variables.all? do |var|
            instance_variable_get(var).deep_frozen?
          end
        end
      end

      refine Array do
        def deep_freeze
          each(&:deep_freeze)
          freeze
        end
      end

      refine Hash do
        def deep_freeze
          each_value(&:deep_freeze)
          freeze
        end
      end
    end

    using DeepFreezable

    CONFIG = { db: { host: "localhost", port: 5432 } }.deep_freeze

-------------------------------------------------------------------------------
  8. CONST_GET Y CONST_SET
-------------------------------------------------------------------------------

  Acceso dinámico a constantes:

    class Config
      API_KEY = "sk-12345"
      DB_URL = "postgres://localhost/dev"
      DEBUG = true
    end

    # Acceso dinámico
    [:API_KEY, :DB_URL, :DEBUG].each do |const|
      puts "#{const}: #{Config.const_get(const)}"
    end

    # Definición dinámica
    Config.const_set(:VERSION, "2.0.0")
    puts Config.const_get(:VERSION)

    # Verificar existencia
    puts Config.const_defined?(:API_KEY)   # true
    puts Config.const_defined?(:FAKE)       # false

-------------------------------------------------------------------------------
  9. CONSTANTS MÉTODO
-------------------------------------------------------------------------------

  El método .constants lista todas las constantes de un módulo/clase:

    puts Math.constants.inspect  # [:E, :PI, ...]
    puts Object.constants.first(5).inspect

  También existe .const_defined? para verificar existencia:

    puts Object.const_defined?(:String)  # true
    puts Object.const_defined?(:FooBar)  # false

-------------------------------------------------------------------------------
  10. CONSTANTES Y CLASES ANIDADAS
-------------------------------------------------------------------------------

  Las clases y módulos son constantes:

    class Usuario
      class Direccion
      end
    end

    # Usuario::Direccion es una constante de tipo Class
    puts Usuario.constants.inspect  # [:Direccion]

  Por eso las clases se nombran con mayúscula: son constantes que
  apuntan a objetos Class.

-------------------------------------------------------------------------------
  11. CONST_MISSING
-------------------------------------------------------------------------------

  Similar a method_missing pero para constantes:

    class AutoLoader
      def self.const_missing(nombre)
        archivo = nombre.to_s.downcase
        if File.exist?("#{archivo}.rb")
          require_relative archivo
          const_get(nombre)
        else
          super
        end
      end
    end

    # Si MiModulo no existe, intenta cargar mi_modulo.rb

-------------------------------------------------------------------------------
  12. CONSTANTES EN BLOQUES Y CLOSURES
-------------------------------------------------------------------------------

  Las constantes dentro de bloques se resuelven en el ámbito donde
  se define el bloque, no donde se ejecuta:

    module A
      VALOR = 10
    end

    module B
      VALOR = 20

      def self.obtener_bloque
        -> { VALOR }
      end
    end

    bloque = B.obtener_bloque
    puts bloque.call  # 20 (ámbito léxico de B)

-------------------------------------------------------------------------------
  13. MUTABILIDAD DE CONSTANTES: EJEMPLOS
-------------------------------------------------------------------------------

  String (mutable por defecto):

    NOMBRE = "Ruby"
    NOMBRE.upcase!  # Modifica el objeto (sin advertencia)
    puts NOMBRE     # "RUBY"

  Array:

    LISTA = [1, 2, 3]
    LISTA.pop       # Modifica el array
    puts LISTA.inspect  # [1, 2]

  Hash:

    MAPA = { a: 1 }
    MAPA[:b] = 2    # Modifica el hash
    puts MAPA.inspect   # { a: 1, b: 2 }

  Para evitar mutación: usa literal.freeze.

-------------------------------------------------------------------------------
  14. CONSTANTES Y MÓDULOS
-------------------------------------------------------------------------------

  Las constantes definidas en módulos se acceden con :::

    module Configuracion
      HOST = "localhost"
      PORT = 3000

      module Database
        URL = "postgres://#{HOST}/db"
      end
    end

    puts Configuracion::HOST
    puts Configuracion::Database::URL

-------------------------------------------------------------------------------
  15. CONSTANTES Y FREEZE EN PRÁCTICA
-------------------------------------------------------------------------------

  Patrón común: constantes congeladas para configuración:

    class AppConfig
      SETTINGS = {
        db: { adapter: "postgres", host: "localhost" },
        cache: { type: "redis" },
        logging: { level: :debug }
      }.freeze

      def self.get(*path)
        path.reduce(SETTINGS) { |hash, key| hash[key] }
      end
    end

    puts AppConfig.get(:db, :adapter)

-------------------------------------------------------------------------------
  16. COMPARATIVA: CONSTANTE EN DIFERENTES LENGUAJES
-------------------------------------------------------------------------------

  Ruby:  PI = 3.14 (reasignable con warning)
  Java:  final double PI = 3.14 (no reasignable)
  C++:   const double PI = 3.14 (no reasignable)
  JS:    const PI = 3.14 (no reasignable, pero objeto mutable)
  Python: PI = 3.14 (convención, nada impide cambio)

  Ruby y JS comparten el mismo approach: la referencia es constante,
  el objeto puede mutar.

-------------------------------------------------------------------------------
  17. PATRÓN: MÓDULO CON CONSTANTES
-------------------------------------------------------------------------------

  Módulos como namespaces de constantes:

    module HTTP
      METHODS = [:get, :post, :put, :delete, :patch].freeze
      HEADERS = {
        json: "application/json",
        html: "text/html",
        xml:  "application/xml"
      }.freeze
      VERSION = "1.1".freeze
    end

    puts HTTP::METHODS.inspect
    puts HTTP::HEADERS[:json]

-------------------------------------------------------------------------------
  18. CONSTANTES CONGELADAS COMO ENUMS
-------------------------------------------------------------------------------

  Simulación de enums con constantes:

    class Estado
      PENDIENTE = :pendiente.freeze
      EN_PROGRESO = :en_progreso.freeze
      COMPLETADO = :completado.freeze
      CANCELADO = :cancelado.freeze

      VALIDOS = [PENDIENTE, EN_PROGRESO, COMPLETADO, CANCELADO].freeze

      def initialize(estado)
        raise "Estado inválido" unless VALIDOS.include?(estado)
        @estado = estado
      end
    end

    e = Estado.new(Estado::PENDIENTE)
    puts e.inspect

-------------------------------------------------------------------------------
  19. PELIGROS COMUNES CON CONSTANTES
-------------------------------------------------------------------------------

  1. Mutación inadvertida: Modificar una constante sin darte cuenta.
  2. Reasignación accidental: Usar el mismo nombre en otra parte.
  3. Dependencia de orden: Las constantes se evalúan en orden.
  4. Ámbito incorrecto: Acceder a una constante que no está visible.
  5. Falta de freeze: No congelar lleva a mutaciones accidentales.

  Buenas prácticas:
    • Usa .freeze en toda constante que sea colección o string.
    • Usa mayúsculas sostenidas (SCREAMING_SNAKE_CASE).
    • Agrupa constantes relacionadas en módulos.
    • Prefiere const_get para acceso dinámico.
    • Documenta las constantes públicas.

================================================================================

  En este capítulo hemos explorado las constantes en Ruby:

  • Las constantes comienzan con mayúscula y se nombran en MAYÚSCULAS.
  • Ruby advierte al reasignar pero no lo impide (salvo FrozenError).
  • Las constantes son referencias, no valores: el objeto puede mutar.
  • freeze hace un objeto inmutable (superficialmente).
  • const_get y const_set permiten acceso dinámico.
  • const_missing maneja acceso a constantes inexistentes.
  • La búsqueda de constantes sigue ámbito léxico primero, luego
    herencia, luego jerarquía superior.
  • Las constantes congeladas y los módulos como namespace son
    buenas prácticas.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Las constantes en Ruby son realmente constantes?
  R: No en el sentido estricto. Ruby advierte al reasignar pero no lo
     impide. El objeto referenciado puede mutar a menos que se use
     freeze.

  P: ¿Por qué las clases se nombran con mayúscula?
  R: Porque las clases son constantes: asignaciones a objetos Class.
     class Usuario es equivalente a Usuario = Class.new.

  P: ¿Cómo hago una constante realmente inmutable?
  R: Usa freeze en el objeto y deep_freeze para objetos anidados.
     También usa # frozen_string_literal: true para strings.

  P: ¿Qué es const_missing?
  R: Es un callback que se ejecuta cuando se accede a una constante
     que no existe. Similar a method_missing.

  P: ¿Se heredan las constantes?
  R: Sí, las subclases pueden acceder a las constantes de la superclase
     a través de la jerarquía de búsqueda.

  P: ¿Cuál es la diferencia entre CONST y :CONST?
  R: CONST es una constante (identificador). :CONST es un símbolo. Se
     usa :CONST con const_get porque espera un símbolo o string.
