================================================================================
                        BIENVENIDA AL CAPÍTULO 66
================================================================================

  Bienvenido a uno de los conceptos más singulares (nunca mejor dicho)
  de Ruby: los métodos singleton. En Ruby, no solo las clases tienen
  métodos: los objetos individuales también pueden tener métodos
  exclusivos.

  Un método singleton es un método que pertenece a un único objeto.
  Ningún otro objeto de la misma clase lo tiene. Esto es posible gracias
  a la clase singleton (también llamada eigenclass o metaclass), una
  clase oculta que Ruby crea automáticamente para cada objeto.

  Este mecanismo es la base de los métodos de clase, los métodos de
  objeto y gran parte de la magia de la metaprogramación Ruby.

  FRASE DEL CAPÍTULO

-------------------------------------------------------------------------------
  1. ¿QUÉ ES UN MÉTODO SINGLETON?
-------------------------------------------------------------------------------

  Un método singleton es un método definido para un solo objeto:

    obj = "Hola"
    def obj.metodo_especial
      "Solo este string tiene este método"
    end

    puts obj.metodo_especial  # "Solo este string tiene este método"

    otro = "Mundo"
    # otro.metodo_especial  # NoMethodError

  El método solo existe en obj, no en ningún otro String.

-------------------------------------------------------------------------------
  2. LA CLASE SINGLETON (EIGENCLASS)
-------------------------------------------------------------------------------

  Cuando defines un método singleton, Ruby crea una clase anónima
  especial llamada clase singleton (o eigenclass) y la inserta entre
  el objeto y su clase original en la cadena de herencia.

  Diagrama conceptual:

    obj (String)
      ↓ método_especial se almacena aquí
    #<Class:#<String:0x...>>  ← clase singleton (eigenclass)
      ↓
    String  ← clase original
      ↓
    Object
      ↓
    BasicObject

  Podemos acceder a la clase singleton con singleton_class:

    puts obj.singleton_class  # => #<Class:#<String:0x...>>

-------------------------------------------------------------------------------
  3. SINTAXIS: def obj.metodo
-------------------------------------------------------------------------------

  La forma más directa de definir un método singleton:

    coche = "Ford"
    def coche.arrancar
      "El coche arranca"
    end

    def coche.acelerar(kmh)
      "Acelerando a #{kmh} km/h"
    end

    puts coche.arrancar
    puts coche.acelerar(120)

  Es simple y legible para pocos métodos.

-------------------------------------------------------------------------------
  4. SINTAXIS: class << obj
-------------------------------------------------------------------------------

  Para definir múltiples métodos singleton, abre la eigenclass con
  class << objeto:

    coche = "Ford"

    class << coche
      def arrancar
        "Arrancando..."
      end

      def acelerar(kmh)
        "Acelerando a #{kmh} km/h"
      end

      def frenar
        "Frenando"
      end
    end

    puts coche.arrancar
    puts coche.acelerar(100)

  Dentro de class << obj, self es la eigenclass.

-------------------------------------------------------------------------------
  5. class << self (MÉTODOS DE CLASE)
-------------------------------------------------------------------------------

  Cuando usas class << self dentro de una clase, abres la eigenclass
  de la propia clase, lo que te permite definir métodos de clase:

    class Vehiculo
      class << self
        def tipo
          "Vehículo genérico"
        end

        def crear
          new
        end
      end
    end

    puts Vehiculo.tipo    # "Vehículo genérico"
    puts Vehiculo.crear.class  # Vehiculo

  Esta es la forma más común de definir métodos de clase en Ruby.

-------------------------------------------------------------------------------
  6. MÉTODOS DE CLASE CON DEF self.metodo
-------------------------------------------------------------------------------

  Otra forma de definir métodos singleton en la clase (métodos de clase):

    class Vehiculo
      def self.tipo
        "Vehículo"
      end

      def self.crear
        new
      end
    end

  def self.metodo es equivalente a:
    • def Vehiculo.metodo (si el nombre de la clase es conocido)
    • class << self; def metodo; end; end

-------------------------------------------------------------------------------
  7. LA CADENA DE HERENCIA DE LA EIGENCLASS
-------------------------------------------------------------------------------

  La eigenclass también tiene una jerarquía de herencia:

    class Padre
      def self.clase_metodo
        "padre"
      end
    end

    class Hijo < Padre
    end

    # La eigenclass de Hijo hereda de la eigenclass de Padre
    puts Hijo.singleton_class.superclass  # #<Class:Padre>

  Esto explica por qué los métodos de clase se heredan:

    puts Hijo.clase_metodo  # "padre"

-------------------------------------------------------------------------------
  8. SINGLETON_CLASS Y OBJETOS INDIVIDUALES
-------------------------------------------------------------------------------

  Cada objeto tiene su propia singleton_class:

    a = "hola"
    b = "mundo"

    def a.metodo_a; "A"; end

    puts a.singleton_class  # #<Class:#<String:0x...>>
    puts b.singleton_class  # #<Class:#<String:0x...>> (diferente)

    puts a.singleton_class == b.singleton_class  # false

-------------------------------------------------------------------------------
  9. SINGLETON_CLASS Y ATRIBUTOS DE CLASE
-------------------------------------------------------------------------------

  Podemos definir attr_accessor en la eigenclass:

    class Configuracion
      class << self
        attr_accessor :db_host, :db_port, :debug

        def setup
          self.db_host = "localhost"
          self.db_port = 5432
          self.debug = false
        end
      end
    end

    Configuracion.setup
    puts Configuracion.db_host  # "localhost"

  Estos son atributos de la clase (no de instancia).

-------------------------------------------------------------------------------
  10. SINGLETON METHODS CON DEFINE_METHOD
-------------------------------------------------------------------------------

  Podemos usar define_method en la eigenclass:

    obj = "test"

    obj.singleton_class.define_method(:nuevo_metodo) do
      "definido dinámicamente"
    end

    puts obj.nuevo_metodo

  Esto es equivalente a def obj.nuevo_metodo; end.

-------------------------------------------------------------------------------
  11. EXTEND: AGREGAR MÉTODOS SINGLETON DESDE MÓDULOS
-------------------------------------------------------------------------------

  extend agrega métodos de un módulo como singleton methods del objeto:

    module Loggable
      def log(mensaje)
        puts "[LOG] #{mensaje}"
      end
    end

    servidor = "Server-1"
    servidor.extend(Loggable)
    servidor.log("Iniciando")  # [LOG] Iniciando

  extend en una clase agrega métodos de clase:

    class Servicio
      extend Loggable
    end

    Servicio.log("Cargado")  # [LOG] Cargado

-------------------------------------------------------------------------------
  12. INSPECCIONAR LA EIGENCLASS
-------------------------------------------------------------------------------

  Métodos útiles para inspeccionar la eigenclass:

    obj = "prueba"

    def obj.mi_metodo; end

    puts obj.singleton_class            # #<Class:#<String:0x...>>
    puts obj.singleton_methods          # [:mi_metodo]
    puts obj.singleton_class.instance_methods(false)  # solo métodos
    puts obj.singleton_class.superclass # String

  singleton_methods muestra solo los métodos singleton del objeto.

-------------------------------------------------------------------------------
  13. SINGLETON METHODS EN CLASES (MÉTODOS DE CLASE)
-------------------------------------------------------------------------------

  Todas las clases son objetos, por lo que pueden tener métodos
  singleton (que son los métodos de clase):

    puts String.singleton_methods  # [:try_convert, ...]

    class MiClase
      def self.metodo_clase; end
    end

    puts MiClase.singleton_methods  # [:metodo_clase]

-------------------------------------------------------------------------------
  14. HERENCIA DE MÉTODOS SINGLETON
-------------------------------------------------------------------------------

  Los métodos singleton de un objeto NO se heredan. Pero los métodos
  singleton de una clase (métodos de clase) SÍ se heredan en la
  eigenclass de la subclase:

    class A
      def self.metodo_a; "A"; end
    end

    class B < A
      def self.metodo_b; "B"; end
    end

    puts B.metodo_a  # "A" (heredado vía eigenclass)
    puts B.metodo_b  # "B"

    obj_a = A.new
    def obj_a.unico; "único"; end
    # obj_b = B.new; obj_b.unico  # NoMethodError

-------------------------------------------------------------------------------
  15. APLICACIONES PRÁCTICAS
-------------------------------------------------------------------------------

  1. Métodos de clase: def self.metodo o class << self.
  2. Singleton objects: objetos con estado global (como config).
  3. Mocking en tests: agregar métodos a objetos mock.
  4. Extensiones de objetos: extender con módulos.
  5. Metaprogramación: definir métodos dinámicamente en un objeto.

  Ejemplo: Objeto singleton:

    class AppConfig
      class << self
        attr_accessor :database_url, :api_key, :debug

        def load!
          self.database_url = ENV["DATABASE_URL"]
          self.api_key = ENV["API_KEY"]
          self.debug = ENV["DEBUG"] == "true"
        end
      end
    end

    AppConfig.load!
    puts AppConfig.database_url

-------------------------------------------------------------------------------
  16. CLASS VARIABLES VIA EIGENCLASS
-------------------------------------------------------------------------------

  Podemos usar la eigenclass para crear variables de clase limpias:

    class Vehiculo
      class << self
        attr_accessor :contador
      end

      self.contador = 0

      def initialize
        self.class.contador += 1
      end
    end

    Vehiculo.new
    Vehiculo.new
    puts Vehiculo.contador  # 2

  Esto es preferible a @@class_variables por su claridad.

-------------------------------------------------------------------------------
  17. SINGLETON METHODS Y LA CADENA DE MÉTODOS
-------------------------------------------------------------------------------

  Cuando llamas a un método en un objeto, Ruby busca en:
    1. La eigenclass del objeto
    2. La clase del objeto
    3. Los módulos incluidos
    4. La superclase
    5. ... hasta BasicObject

    obj = "test"
    def obj.length
      "NO! #{super}"
    end

    puts obj.length  # "NO! 4"

  El método singleton tiene prioridad sobre el de la clase.

-------------------------------------------------------------------------------
  18. EVALUACIÓN EN LA EIGENCLASS
-------------------------------------------------------------------------------

  Podemos evaluar código directamente en la eigenclass con instance_eval:

    obj = "hola"
    obj.singleton_class.instance_eval do
      define_method(:saludo) { "Hola desde singleton" }
    end

    puts obj.saludo  # "Hola desde singleton"

  Esto es útil para metaprogramación avanzada.

-------------------------------------------------------------------------------
  19. PATRÓN: OBJETO CON COMPORTAMIENTO DINÁMICO
-------------------------------------------------------------------------------

  Podemos crear objetos con comportamiento único en tiempo de ejecución:

    def crear_personaje(tipo)
      personaje = { nombre: "Desconocido", vida: 100 }

      class << personaje
        attr_accessor :nombre, :vida
      end

      personaje.nombre = "Héroe"

      case tipo
      when :guerrero
        def personaje.atacar
          "Ataque con espada!"
        end
      when :mago
        def personaje.atacar
          "Bola de fuego!"
        end
      when :arquero
        def personaje.atacar
          "Disparo de flecha!"
        end
      end

      personaje
    end

    guerrero = crear_personaje(:guerrero)
    mago = crear_personaje(:mago)

    puts guerrero.atacar  # "Ataque con espada!"
    puts mago.atacar      # "Bola de fuego!"

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  En este capítulo hemos explorado los métodos singleton:

  • Un método singleton pertenece a un único objeto, no a su clase.
  • Ruby almacena los métodos singleton en la clase singleton (eigenclass).
  • class << obj abre la eigenclass para definir múltiples métodos.
  • class << self dentro de una clase define métodos de clase.
  • def self.metodo es equivalente a class << self; def metodo; end.
  • extend agrega módulos como métodos singleton.
  • La eigenclass tiene su propia jerarquía de herencia.
  • Los métodos singleton tienen prioridad sobre métodos de instancia.
  • singleton_class permite acceder a la eigenclass de cualquier objeto.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Cuál es la diferencia entre clase singleton y clase regular?
  R: La clase singleton es anónima y se crea automáticamente para cada
     objeto. No tiene nombre y no se define explícitamente.

  P: ¿Los métodos singleton se heredan?
  R: Los métodos singleton de un objeto NO se heredan. Los métodos
     singleton de una clase (métodos de clase) SÍ se heredan mediante
     la jerarquía de eigenclasses.

  P: ¿Cuándo usar class << self vs def self.metodo?
  R: Usa def self.metodo para métodos individuales. Usa class << self
     cuando necesites definir varios métodos de clase, attr_accessor
     de clase, o métodos privados de clase.

  P: ¿Qué es la eigenclass?
  R: Es otro nombre para la clase singleton. También se llama metaclass.
     Es la clase oculta que contiene los métodos singleton de un objeto.

  P: ¿Puedo acceder a la clase singleton de cualquier objeto?
  R: Sí, con obj.singleton_class. Todos los objetos tienen una.

  P: ¿Qué relación hay entre extend y singleton methods?
  R: extend agrega los métodos de un módulo como métodos singleton
     del objeto receptor.

  Los métodos singleton son una de las características más únicas de
  Ruby. Revelan la naturaleza profundamente orientada a objetos del

  identidad, sus propios métodos, su propia clase secreta.

  objeto se manifiesta en código. Los métodos de clase, los patrones
  de factoría, los roles dinámicos y gran parte de la magia de Ruby
  se sostienen sobre este mecanismo.
