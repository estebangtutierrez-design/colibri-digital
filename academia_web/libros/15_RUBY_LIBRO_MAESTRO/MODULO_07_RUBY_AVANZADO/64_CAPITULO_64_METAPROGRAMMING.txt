                        BIENVENIDA AL CAPÍTULO 64
================================================================================

  Bienvenido al capítulo donde el código cobra conciencia. La meta-
  programación en Ruby es la capacidad de escribir código que escribe,
  modifica o inspecciona código en tiempo de ejecución.

  Este es uno de los poderes más distintivos de Ruby: un programa puede
  definirse a sí mismo, responder a mensajes que no existen en tiempo
  de compilación y adaptar su comportamiento dinámicamente. Es lo que
  hace posible Rails, RSpec, ActiveRecord y prácticamente todas las
  gemas que amas.

  Pero con gran poder viene gran responsabilidad. La metaprogramación
  mal utilizada puede producir código imposible de depurar. Este
  capítulo te enseñará a usar este poder con sabiduría.

-------------------------------------------------------------------------------
  1. ¿QUÉ ES LA METAPROGRAMACIÖN?
-------------------------------------------------------------------------------

  Metaprogramación es la técnica de escribir programas que:
    • Escriben otros programas (código que genera código).
    • Se modifican a sí mismos en tiempo de ejecución.
    • Inspeccionan su propia estructura (reflexión/introspección).

  Ruby facilita la metaprogramación porque:
    • Todo es un objeto (clases, métodos, constantes).
    • Las clases son abiertas (open classes).
    • Los métodos se pueden definir en tiempo de ejecución.
    • Existe eval, method_missing, define_method, etc.

-------------------------------------------------------------------------------
  2. INTROSPECCIÓN (REFLEXIÓN)
-------------------------------------------------------------------------------

  Antes de modificar, debemos inspeccionar:

    obj = "Hola"
    puts obj.class          # => String
    puts obj.methods        # => [:upcase, :downcase, ...]
    puts obj.instance_variables  # => []
    puts obj.respond_to?(:upcase)  # => true

    puts String.instance_methods  # métodos de instancia
    puts String.class_methods     # métodos de clase

  La introspección es el primer paso de la metaprogramación.

-------------------------------------------------------------------------------
  3. define_method
-------------------------------------------------------------------------------

  define_method crea métodos dinámicamente en una clase:

    class Persona
      [:nombre, :edad, :email].each do |atributo|
        define_method(atributo) do
          instance_variable_get("@#{atributo}")
        end

        define_method("#{atributo}=") do |valor|
          instance_variable_set("@#{atributo}", valor)
        end
      end
    end

    p = Persona.new
    p.nombre = "Ana"
    puts p.nombre  # => Ana

  define_method es un método privado de Module/Class.

-------------------------------------------------------------------------------
  4. method_missing
-------------------------------------------------------------------------------

  Cuando llamas a un método que no existe, Ruby llama a method_missing:

    class Proxy
      def initialize(target)
        @target = target
      end

      def method_missing(nombre, *args, &bloque)
        if @target.respond_to?(nombre)
          @target.send(nombre, *args, &bloque)
        else
          super
        end
      end

      def respond_to_missing?(nombre, include_private = false)
        @target.respond_to?(nombre) || super
      end
    end

    proxy = Proxy.new([1, 2, 3])
    puts proxy.length  # => 3
    puts proxy.map { |x| x * 2 }.inspect  # => [2, 4, 6]

  Siempre sobrescribe respond_to_missing? junto con method_missing.

-------------------------------------------------------------------------------
  5. const_missing
-------------------------------------------------------------------------------

  Similar a method_missing pero para constantes:

    class Modulo
      def self.const_missing(nombre)
        puts "Constante #{nombre} no encontrada, creándola..."
        const_set(nombre, "auto_#{nombre.downcase}")
      end
    end

    puts Modulo::ALGO  # => "auto_algo"

  Útil para autocarga de módulos (autoloading).

-------------------------------------------------------------------------------
  6. eval, class_eval, instance_eval
-------------------------------------------------------------------------------

  eval: Evalúa código Ruby en el contexto actual.

    x = 10
    eval("puts x * 2")  # => 20

  class_eval (también module_eval): Evalúa código en el contexto de
  una clase (define métodos de instancia).

    String.class_eval do
      def gritar
        upcase + "!"
      end
    end

    puts "hola".gritar  # => "HOLA!"

  instance_eval: Evalúa código en el contexto de una instancia
  (define métodos de singletón, accede a variables de instancia).

    obj = "mundo"
    obj.instance_eval do
      def longitud
        size
      end
    end
    puts obj.longitud  # => 5

-------------------------------------------------------------------------------
  7. SEND: LLAMADA DINÁMICA DE MÉTODOS
-------------------------------------------------------------------------------

  send invoca métodos usando su nombre como símbolo o string:

    class Calculadora
      def sumar(a, b)  a + b end
      def restar(a, b) a - b end
    end

    calc = Calculadora.new
    puts calc.send(:sumar, 5, 3)  # => 8
    puts calc.send("restar", 5, 3) # => 2

  public_send es la versión segura (respeta métodos privados).

-------------------------------------------------------------------------------
  8. DSL CON METAPROGRAMACIÖN
-------------------------------------------------------------------------------

  La metaprogramación permite crear lenguajes específicos de dominio:

    class Configuracion
      def initialize
        @valores = {}
      end

      def method_missing(nombre, *args, &bloque)
        if bloque
          @valores[nombre] = bloque
        elsif args.any?
          @valores[nombre] = args.first
        elsif @valores.key?(nombre)
          @valores[nombre]
        else
          super
        end
      end

      def respond_to_missing?(nombre, _)
        @valores.key?(nombre) || super
      end
    end

    config = Configuracion.new
    config.db "postgres"
    config.debug true
    config.on_start { puts "Iniciando..." }

    puts config.db    # => "postgres"
    puts config.debug # => true
    config.on_start.call

-------------------------------------------------------------------------------
  9. ATTR_ACCESSOR DINÁMICO
-------------------------------------------------------------------------------

  Podemos crear nuestros propios attr_accessor:

    class Object
      def self.mi_attr_accessor(*nombres)
        nombres.each do |nombre|
          define_method(nombre) do
            instance_variable_get("@#{nombre}")
          end

          define_method("#{nombre}=") do |valor|
            instance_variable_set("@#{nombre}", valor)
          end
        end
      end
    end

    class Punto
      mi_attr_accessor :x, :y
    end

    p = Punto.new
    p.x = 10
    p.y = 20
    puts "(#{p.x}, #{p.y})"  # => (10, 20)

-------------------------------------------------------------------------------
  10. SINGLETON METHODS (MÉTODOS DE OBJETO)
-------------------------------------------------------------------------------

  Puedes definir métodos en un solo objeto:

    obj = "texto"
    def obj.metodo_especial
      "solo yo tengo este método"
    end

    puts obj.metodo_especial
    # "solo yo tengo este método"

    otro = "otro texto"
    # otro.metodo_especial  # NoMethodError

  Esto se logra mediante la clase singletón (eigenclass).

-------------------------------------------------------------------------------
  11. INCLUSIÓN DINÁMICA DE MÓDULOS
-------------------------------------------------------------------------------

  Podemos incluir módulos dinámicamente:

    module Loggable
      def log(mensaje)
        puts "[LOG] #{mensaje}"
      end
    end

    class Servicio
    end

    s = Servicio.new
    s.extend(Loggable)
    s.log("Hola")  # => [LOG] Hola

  extend agrega métodos del módulo como singleton methods.

-------------------------------------------------------------------------------
  12. REMOVER MÉTODOS DINÁMICAMENTE
-------------------------------------------------------------------------------

  Podemos eliminar métodos en tiempo de ejecución:

    class Ejemplo
      def metodo_normal
        "existo"
      end
    end

    e = Ejemplo.new
    puts e.metodo_normal  # => "existo"

    Ejemplo.remove_method(:metodo_normal)
    # e.metodo_normal  # NoMethodError

  remove_method elimina el método de esa clase.
  undef_method evita que el método sea llamado (incluso heredado).

-------------------------------------------------------------------------------
  13. CONSTANTES DINÁMICAS
-------------------------------------------------------------------------------

  Podemos acceder y definir constantes dinámicamente:

    class Modulo
    end

    Modulo.const_set(:VERSION, "1.0.0")
    puts Modulo.const_get(:VERSION)  # => "1.0.0"
    puts Modulo.constants             # => [:VERSION]

  const_defined? verifica si una constante existe.

-------------------------------------------------------------------------------
  14. BINDING Y EVAL AVANZADO
-------------------------------------------------------------------------------

  Podemos capturar y reutilizar contextos:

    def crear_contexto
      x = 10
      y = 20
      binding
    end

    ctx = crear_contexto
    puts eval("x + y", ctx)  # => 30

  Esto es la base de las plantillas ERB y otros sistemas de
  evaluación dinámica.

-------------------------------------------------------------------------------
  15. DEFINE_METHOD CON BLOQUE
-------------------------------------------------------------------------------

  define_method con bloque accede a variables externas (closure):

    class CreadorDeMetodos
      def self.crear_metodo(nombre, operacion)
        define_method(nombre) do |a, b|
          a.send(operacion, b)
        end
      end
    end

    class Calculadora < CreadorDeMetodos
      crear_metodo(:suma, :+)
      crear_metodo(:resta, :-)
      crear_metodo(:multiplica, :*)
    end

    c = Calculadora.new
    puts c.suma(5, 3)        # => 8
    puts c.multiplica(4, 3)  # => 12

-------------------------------------------------------------------------------
  16. CLASS_EVAL VS INSTANCE_EVAL
-------------------------------------------------------------------------------

  class_eval abre la clase como si estuvieras dentro de ella:

    String.class_eval do
      def reverso
        reverse
      end
    end
    puts "Ruby".reverso  # "ybuR"

  instance_eval cambia self al objeto:

    class Example
      def initialize
        @valor = 42
      end
    end

    Example.new.instance_eval { puts @valor }  # 42

  instance_eval con clase define métodos de clase:

    Example.instance_eval do
      def metodo_de_clase
        "método de clase"
      end
    end
    puts Example.metodo_de_clase

-------------------------------------------------------------------------------
  17. PELIGROS DE LA METAPROGRAMACIÖN
-------------------------------------------------------------------------------

  1. Seguridad: eval puede ejecutar código malicioso.
  2. Depuración: los métodos generados dinámicamente no aparecen
     en el stack trace de forma clara.
  3. Rendimiento: define_method es más lento que def.
  4. Complejidad: el código metaprogramado es difícil de entender.
  5. Colisiones: puedes sobrescribir métodos existentes sin saberlo.

  Buenas prácticas:
    • Siempre sobrescribe respond_to_missing? con method_missing.
    • Usa public_send en lugar de send cuando sea posible.
    • Documenta claramente los métodos generados dinámicamente.
    • Privilegia define_method sobre eval.
    • No uses eval con entrada de usuario.

-------------------------------------------------------------------------------
  18. CASOS DE USO REALES
-------------------------------------------------------------------------------

  ActiveRecord: define attributes y associations dinámicamente.
  RSpec: crea métodos describe/it con metaprogramación.
  Rails Routing: genera rutas dinámicamente.
  Dry-types: define tipos con DSLs.
  Grape API: construye APIs con DSL.

-------------------------------------------------------------------------------
  19. COMPARATIVA: MÉTODOS DE METAPROGRAMACIÖN
-------------------------------------------------------------------------------

  Método              | Contexto          | Define métodos de
  --------------------|-------------------|-------------------
  define_method       | Clase (self)      | Instancia
  class_eval          | Clase (self)      | Instancia
  instance_eval       | Objeto (self)     | Singleton
  module_eval         | Módulo (self)     | Instancia
  eval                | Ámbito actual     | Variable
  send                | Objeto receptor   | N/A (invoca)
  method_missing      | Receptor          | N/A (intercepta)

    true
  end

    @rutas.each do |r|

    end
  end
end

class ProxySeguro
  def initialize(objeto, solo_metodos = [])
    @objeto = objeto
    @permitidos = solo_metodos
  end

  def method_missing(nombre, *args, &bloque)
    if @permitidos.empty? || @permitidos.include?(nombre)
      if @objeto.respond_to?(nombre)
        @objeto.public_send(nombre, *args, &bloque)
      else
        super
      end
    else
      raise "Método #{nombre} no permitido"
    end
  end

  def respond_to_missing?(nombre, incl_priv = false)
    (@permitidos.empty? || @permitidos.include?(nombre)) &&
      @objeto.respond_to?(nombre) || super
  end
end

class ConfigApp
  attr_reader :config

  def initialize
    @config = {}
  end

  def self.configurar(&bloque)
    app = ConfigApp.new
    app.instance_eval(&bloque)
    app.config
  end

  private

  def database(adaptador, hash = {})
    @config[:database] = { adaptador: adaptador }.merge(hash)
  end

  def cache(tipo, opciones = {})
    @config[:cache] = { tipo: tipo }.merge(opciones)
  end

  def middleware(nombre, *args)
    @config[:middleware] ||= []
    @config[:middleware] << { nombre: nombre, args: args }
  end
end

def crear_clase(nombre, atributos = [])
  clase = Class.new do
    atributos.each do |attr|
      attr_accessor attr
    end

    define_method(:initialize) do |*valores|
      atributos.zip(valores).each do |attr, valor|
        send("#{attr}=", valor) if valor
      end
    end

    define_method(:to_s) do
      atributos.map { |a| "#{a}: #{send(a)}" }.join(", ")
    end
  end

  Object.const_set(nombre, clase)
  clase
end

class CacheDinamico
  def initialize
    @cache = {}
  end

  def method_missing(nombre, *args)
    clave = [nombre, *args]
    if @cache.key?(clave)
      @cache[clave]
    elsif nombre.to_s.start_with?("calc_")
      @cache[clave] = yield(*args) if block_given?
    else
      super
    end
  end

  def respond_to_missing?(nombre, *)
    nombre.to_s.start_with?("calc_") || super
  end
end

class Repositorio
  def initialize
    @datos = []
    @indices = {}
  end

  def agregar(registro)
    @datos << registro
  end

  def method_missing(nombre, *args)
    if nombre.to_s =~ /^find_by_(\w+)$/
      campo = $1.to_sym
      @datos.select { |r| r[campo] == args.first }
    else
      super
    end
  end

  def respond_to_missing?(nombre, *)
    nombre.to_s.start_with?("find_by_") || super
  end
end

class TemplateEngine
  def initialize(template)
    @template = template
    @variables = {}
  end

  def asignar(nombre, valor)
    @variables[nombre] = valor
  end

  def render
    b = binding
    @variables.each { |k, v| b.local_variable_set(k, v) }
    eval("\"#{@template}\"", b)
  end
end

  En este capítulo hemos explorado la metaprogramación en Ruby:

  • define_method crea métodos dinámicamente en una clase.
  • method_missing captura llamadas a métodos inexistentes.
  • const_missing captura acceso a constantes inexistentes.
  • eval, class_eval, instance_eval evalúan código en contextos.
  • send y public_send invocan métodos dinámicamente.
  • La metaprogramación permite crear DSLs potentes.
  • Siempre se debe sobrescribir respond_to_missing? con method_missing.
  • La metaprogramación conlleva riesgos de seguridad y complejidad.

  La metaprogramación es el superpoder de Ruby. Úsalo con sabiduría.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Cuál es la diferencia entre class_eval e instance_eval?
  R: class_eval abre el contexto de la clase (self es la clase) y
     define métodos de instancia. instance_eval abre el contexto de
     un objeto (self es el objeto) y define métodos singleton.

  P: ¿method_missing afecta el rendimiento?
  R: Sí, porque cada llamada "missing" debe recorrer la jerarquía de
     clases antes de llegar a method_missing. Es más lento que un
     método real. Para alto rendimiento, define métodos dinámicamente
     con define_method.

  P: ¿Es seguro usar eval con entrada de usuario?
  R: No, nunca. eval ejecuta código Ruby arbitrario. Si necesitas
     evaluar expresiones de usuario, usa un parser seguro o sandbox.

  P: ¿Qué es respond_to_missing? y por qué es necesario?
  R: Sobrescribe el comportamiento de respond_to? para que los métodos
     capturados por method_missing aparezcan como existentes. Sin esto,
     respond_to? devolverá false para métodos delegados.

  P: ¿define_method es más lento que def?
  R: Sí, ligeramente. define_method es una llamada de método que define
     un método; es más lento en definición, no en ejecución. Para
     métodos que se definen una vez y se llaman muchas veces, la
     diferencia es despreciable.
