  Bienvenido al proyecto final del Módulo VII: Ruby Avanzado. A lo largo
  de 9 capítulos has recorrido un camino que va desde los lambdas y
  closures hasta la metaprogramación, las clases singleton y las fibras.
  Ahora es momento de integrarlo todo en un proyecto completo.

  Este proyecto consiste en construir un mini-framework de desarrollo
  web similar a Sinatra, pero extremadamente ligero, usando exclusiva-
  mente los conceptos de metaprogramación, closures, lambdas, y fibras
  que has aprendido.

  Llamaremos a este proyecto "MicroWeb" y será un enrutador web con
  DSL, middleware, pipeline de procesamiento y generación de respuestas.

  El objetivo no es crear un framework listo para producción, sino
  demostrar cómo todos los conceptos del módulo se combinan para
  construir herramientas reales.

-------------------------------------------------------------------------------
  4. ROUTER (METHOD_MISSING + DEFINE_METHOD)
-------------------------------------------------------------------------------

  El router usa method_missing para capturar definiciones de rutas:

    module MicroWeb
      class Router
        HTTP_METHODS = [:get, :post, :put, :delete, :patch].freeze

        attr_reader :routes

        def initialize
          @routes = []
        end

        def method_missing(metodo, *args, &bloque)
          if HTTP_METHODS.include?(metodo)
            path = args.first
            @routes << {
              method: metodo,
              path: path,
              pattern: compile_path(path),
              params: extract_params(path),
              handler: bloque
            }
          else
            super
          end
        end

        def respond_to_missing?(metodo, *)
          HTTP_METHODS.include?(metodo) || super
        end

        def match(method, path)
          @routes.find do |route|
            route[:method] == method && route[:pattern].match(path)
          end
        end

        private

        def compile_path(path)
          regex = path.gsub(/:(\w+)/, '(?<\1>[^/]+)')
          Regexp.new("\\A#{regex}\\z")
        end

        def extract_params(path)
          path.scan(/:(\w+)/).flatten.map(&:to_sym)
        end
      end
    end

  5. DSL DE LA APP (INSTANCE_EVAL)
-------------------------------------------------------------------------------

  El DSL usa class_eval e instance_eval para la definición de rutas:

    module MicroWeb
      class App
        def self.run!
          app = new
          app.instance_eval(&@setup) if @setup
          app.start
        end

        def self.define(&bloque)
          @setup = bloque
        end

        def initialize
          @router = Router.new
          @middleware = MiddlewareStack.new
          setup_default_middleware
        end

        def start
          puts "MicroWeb iniciado en #{Config.get(:server, :host)}:#{Config.get(:server, :port)}"
          # En un proyecto real, aquí iría el servidor
        end

        private

        def setup_default_middleware
          @middleware.use :logger
          @middleware.use :static
        end

        def router(&bloque)
          @router.instance_eval(&bloque)
        end

        def middleware(nombre, &bloque)
          @middleware.use(nombre, &bloque)
        end

        def method_missing(metodo, *args, &bloque)
          if Router::HTTP_METHODS.include?(metodo)
            @router.public_send(metodo, *args, &bloque)
          else
            super
          end
        end

        def respond_to_missing?(metodo, *)
          Router::HTTP_METHODS.include?(metodo) || super
        end
      end
    end

-------------------------------------------------------------------------------
  6. MIDDLEWARE CON FIBRAS Y CLOSURES
-------------------------------------------------------------------------------

  El middleware usa fibras para el pipeline:

    module MicroWeb
      class MiddlewareStack
        def initialize
          @middlewares = []
        end

        def use(nombre, &bloque)
          if bloque
            @middlewares << { nombre: nombre, handler: bloque }
          else
            @middlewares << { nombre: nombre, handler: default_handler(nombre) }
          end
        end

        def process(env)
          pipeline = build_pipeline(env)
          pipeline.resume while pipeline.alive?
        end

        private

        def build_pipeline(env)
          Fiber.new do
            result = env
            @middlewares.each do |mw|
              result = mw[:handler].call(result)
              Fiber.yield
            end
            result
          end
        end

        def default_handler(nombre)
          case nombre
          when :logger
            ->(env) {
              puts "  [LOG] #{env[:method]} #{env[:path]}"
              env
            }
          when :static
            ->(env) {
              if env[:path] =~ /\.(css|js|png|jpg)$/
                env[:response] = { static: true, path: env[:path] }
              end
              env
            }
          when :json
            ->(env) {
              env[:content_type] = "application/json"
              env
            }
          else
            ->(env) { env }
          end
        end
      end
    end

    module MicroWeb
      module Response
        FORMATS = {
          html: ->(body, status = 200) {
            [status, { "Content-Type" => "text/html" }, [body]]
          },
          json: ->(data, status = 200) {
            require "json" unless defined?(JSON)
            [status, { "Content-Type" => "application/json" }, [data.to_json]]
          },
          text: ->(body, status = 200) {
            [status, { "Content-Type" => "text/plain" }, [body]]
          },
          redirect: ->(url, status = 302) {
            [status, { "Location" => url }, []]
          },
          error: ->(code = 404, message = "Not Found") {
            [code, { "Content-Type" => "text/plain" }, [message]]
          }
        }.freeze

        def self.format(tipo, *args)
          FORMATS.fetch(tipo) { FORMATS[:text] }.call(*args)
        end

        # Respuestas predefinidas con currying
        OK = FORMATS[:text].curry.call("OK")
        NOT_FOUND = FORMATS[:error].call(404, "Not Found")
        SERVER_ERROR = FORMATS[:error].call(500, "Internal Server Error")
      end
    end

  9. SERVIDOR SIMULADO CON FIBERS
-------------------------------------------------------------------------------

  Para demostrar el pipeline completo, simulamos peticiones con fibras:

    module MicroWeb
      class Server
        def initialize(app)
          @app = app
        end

        def simulate(requests)
          requests.each do |req|
            Fiber.new do
              env = {
                method: req[:method],
                path: req[:path],
                params: {},
                headers: req[:headers] || {},
                body: req[:body] || ""
              }

              # Pasar por middleware
              env = @app.instance_variable_get(:@middleware).process(env)

              # Buscar ruta
              route = @app.instance_variable_get(:@router).match(
                req[:method], req[:path]
              )

              if route
                # Extraer parámetros de la URL
                match_data = route[:pattern].match(req[:path])
                env[:params] = match_data&.named_captures&.transform_keys(&:to_sym) || {}

                # Ejecutar handler
                status, headers, body = route[:handler].call(env)
                puts "  Respuesta: #{status} #{body.first rescue ""}"
              else
                puts "  Ruta no encontrada: #{req[:method]} #{req[:path]}"
              end
            end.resume
          end
        end
      end
    end

          method: metodo,
          path: path,
          pattern: compile_path(path),

      end
    end

    def list_routes
      @routes.map { |r| "#{r[:method].upcase} #{r[:path]}" }

module MicroWeb
  class App
    def self.define(&bloque)
      @setup = bloque
    end

    def self.run!
      return unless @setup

      app = new
      app.instance_eval(&@setup)
      app.start
    end

    def initialize
      @router = Router.new
      @middleware = MiddlewareStack.new
      @config_overrides = {}
    end

    def start
      host = @config_overrides.dig(:server, :host) || Config.get(:server, :host)
      port = @config_overrides.dig(:server, :port) || Config.get(:server, :port)

      puts "=" * 50
      puts "  MicroWeb v1.0.0"
      puts "  Servidor en #{host}:#{port}"
      puts "  Rutas registradas:"
      @router.list_routes.each { |r| puts "    #{r}" }
      puts "=" * 50
    end

    private

    def config(opts = {})
      @config_overrides.merge!(opts)
    end

    def router(&bloque)
      @router.instance_eval(&bloque)
    end

    def middleware(nombre, &bloque)
      @middleware.use(nombre, &bloque)
    end

    def method_missing(metodo, *args, &bloque)
      if Router::HTTP_METHODS.include?(metodo)
        @router.public_send(metodo, *args, &bloque)
      else
        super
      end
    end

    def respond_to_missing?(metodo, *)
      Router::HTTP_METHODS.include?(metodo) || super
    end
  end
end

module MicroWeb
  class Server
    def initialize(app)
      @app = app
      @router = app.instance_variable_get(:@router)
      @middleware = app.instance_variable_get(:@middleware)
    end

    def simulate(requests)
      puts
      puts "  Procesando #{requests.size} peticiones..."
      puts

      requests.each do |req|
        Fiber.new do
          env = {
            method: req[:method],
            path: req[:path],
            params: {},
            headers: req[:headers] || {},
            body: req[:body] || "",
            query: req[:query] || {}
          }

          puts "  → #{env[:method]} #{env[:path]}"

          env = @middleware.process(env)

          route = @router.match(req[:method], req[:path])

          if route
            match_data = route[:pattern].match(req[:path])
            env[:params] = match_data&.named_captures&.transform_keys(&:to_sym) || {}

            status, headers, body = route[:handler].call(env)
            puts "  ← #{status} #{headers['Content-Type']} | #{body.first[0..50]}..."
          else
            status, headers, body = Response::NOT_FOUND
            puts "  ← #{status} Not Found"
          end
        end.resume
      end
    end
  end
end

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  En este proyecto hemos construido MicroWeb, un mini-framework web
  que integra todos los conceptos del Módulo VII:

  • Lambdas y Procs: Los formatos de respuesta en Response son lambdas
    almacenadas en un hash congelado. Los middlewares son procs/lambdas
    que procesan el entorno.

  • Closures: Los handlers de rutas y middlewares capturan variables
    del ámbito donde se definen.

  • Currying: Response::OK usa currying para crear respuestas
    preconfiguradas.

  • Metaprogramación: Router usa method_missing para capturar métodos
    HTTP dinámicos. define_method podría usarse para optimización.

  • class_eval / instance_eval: El DSL de rutas usa instance_eval
    para evaluar bloques en el contexto del Router y la App.

  • Singleton methods: Los métodos HTTP están definidos como constantes
    congeladas en arrays. Los middlewares se asignan con lambdas.

  • Open Classes: Router reabre su comportamiento con method_missing.

  • Constantes congeladas: Config, Response::FORMATS y HTTP_METHODS
    usan freeze para inmutabilidad.

  • Fibras: Server usa fibras para simular peticiones concurrentes
    cooperativas.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Este framework es funcional?
  R: Es una demostración educativa. Para producción usa Sinatra, Rails
     o Roda. MicroWeb muestra cómo se construyen estas herramientas.

  P: ¿Por qué usar fibras en lugar de hilos?
  R: Las fibras permiten control explícito del flujo y son más ligeras.
     Para este ejemplo demostrativo, ilustran perfectamente el concepto.

  P: ¿Cómo añadiría un servidor HTTP real?
  R: Usarías la gema `webrick` (incluida) o `puma` con un socket TCP
     que procese peticiones HTTP reales.

  P: ¿Por qué usar method_missing en lugar de definir los métodos?
  R: Para demostrar metaprogramación. En producción, define_method
     sería más eficiente porque evita el overhead de method_missing
     en cada llamada.

  P: ¿Puedo extender este proyecto para mi uso personal?
  R: Por supuesto. El código es educativo pero funcional. Puedes
     agregar servidor real, base de datos, etc.
