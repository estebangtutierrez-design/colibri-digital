                    BIENVENIDA A LOS HASHES
═══════════════════════════════════════════════════════════════════════════════════

Bienvenido, maestro organizador de datos.

Si los arrays son listas ordenadas, los hashes son diccionarios: asociaciones
entre una clave (key) y un valor (value). Mientras que en un array accedes
por índice numérico, en un hash accedes por una clave que puede ser cualquier
objeto.

Los hashes son omnipresentes en Ruby. Los verás en configuración, en datos
provenientes de APIs (JSON), en parámetros de métodos, en opciones, y en
incontables situaciones más. Dominar los hashes es dominar la organización
de información en Ruby.

En este capítulo aprenderás a crear hashes, usar símbolos como claves,
establecer valores por defecto, acceder de forma segura con fetch y dig,
fusionar hashes con merge, y transformar claves con transform_keys.

                    1. ¿QUÉ ES UN HASH?
═══════════════════════════════════════════════════════════════════════════════════

Un hash es una colección de pares clave-valor. Cada clave es única (no puede
haber dos claves iguales). Los valores pueden ser cualquier objeto.

Características clave:

  • Asociativo: cada clave se asocia a un valor.
  • Claves únicas: no se permiten duplicados (la última prevalece).
  • Ordenado: desde Ruby 1.9, los hashes preservan el orden de inserción.
  • Flexible: claves y valores pueden ser de cualquier tipo.

Ejemplo conceptual:

  persona = {
    "nombre" => "Ana",
    "edad"   => 30,
    "ciudad" => "Lima"
  }

═══════════════════════════════════════════════════════════════════════════════════
                    2. CREACIÓN DE HASHES — SINTAXIS LITERAL
═══════════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 1 — Hash literal con => (hash rocket)

```ruby
# Hash vacio
vacio = {}
puts vacio.class  # => Hash
puts vacio.length # => 0

# Hash con strings como claves
persona = {
  "nombre" => "Ana",
  "edad" => 30,
  "ciudad" => "Lima"
}
puts persona.inspect
# => {"nombre"=>"Ana", "edad"=>30, "ciudad"=>"Lima"}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 2 — Hash con símbolos como claves (sintaxis moderna)

```ruby
# Simbolos como claves (recomendado)
persona = {
  nombre: "Ana",
  edad: 30,
  ciudad: "Lima"
}
puts persona.inspect
# => {:nombre=>"Ana", :edad=>30, :ciudad=>"Lima"}

# Sintaxis alternativa con hash rocket
persona2 = {
  :nombre => "Ana",
  :edad => 30,
  :ciudad => "Lima"
}
puts persona2.inspect
# => {:nombre=>"Ana", :edad=>30, :ciudad=>"Lima"}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 3 — Otras formas de crear hashes

```ruby
# Hash.new con valor por defecto
contador = Hash.new(0)
puts contador["ruby"]  # => 0 (no existe, devuelve default)

# Hash.new con bloque
hash_autovivificado = Hash.new { |h, k| h[k] = [] }
hash_autovivificado["frutas"] << "manzana"
hash_autovivificado["frutas"] << "banana"
puts hash_autovivificado.inspect
# => {"frutas"=>["manzana", "banana"]}

# Convertir array de pares a hash
pares = [[:a, 1], [:b, 2], [:c, 3]]
hash = pares.to_h
puts hash.inspect  # => {:a=>1, :b=>2, :c=>3}
```

                    3. ACCEDIENDO A ELEMENTOS
═══════════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 4 — Acceso básico

```ruby
persona = { nombre: "Ana", edad: 30, ciudad: "Lima" }

puts persona[:nombre]  # => "Ana"
puts persona[:edad]    # => 30
puts persona[:pais]    # => nil (clave no existe)

# Acceso con string (no funciona si la clave es simbolo)
puts persona["nombre"] # => nil (clave no existe)
```

═══════════════════════════════════════════════════════════════════════════════════
                    4. SÍMBOLOS COMO CLAVES
═══════════════════════════════════════════════════════════════════════════════════

Los símbolos son las claves más comunes en Ruby por su eficiencia y legibilidad.
Un símbolo es una cadena inmutable que se almacena una sola vez en memoria.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 5 — Por que usar símbolos como claves

```ruby
# Ventajas de símbolos:
# 1. Inmutables
# 2. Más rápidos que strings
# 3. Más legibles
# 4. Sintaxis mas limpia

# Con strings
h1 = { "nombre" => "Ana", "edad" => 30 }

# Con simbolos
h2 = { nombre: "Ana", edad: 30 }

# Acceder
puts h2[:nombre]  # => "Ana"
puts h2[:edad]    # => 30
```

═══════════════════════════════════════════════════════════════════════════════════
                    5. VALOR POR DEFECTO
═══════════════════════════════════════════════════════════════════════════════════

Cuando accedes a una clave que no existe, por defecto devuelve nil.
Puedes cambiar este comportamiento.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 6 — Valor por defecto

```ruby
# Sin valor por defecto (nil)
h = {}
puts h[:no_existe]  # => nil

# Con valor por defecto fijo
contador = Hash.new(0)
puts contador[:visitas]  # => 0
contador[:visitas] += 1
puts contador[:visitas]  # => 1

# Con bloque (valor por defecto dinamico)
auto = Hash.new { |hash, key| hash[key] = key.to_s.upcase }
puts auto[:hola]  # => "HOLA"
puts auto[:ruby]  # => "RUBY"
puts auto.inspect  # => {:hola=>"HOLA", :ruby=>"RUBY"}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 7 — Default mutable vs inmutable

```ruby
# PELIGRO: con default fijo, se comparte el mismo objeto
contador = Hash.new(0)
contador[:a] += 1  # 0 + 1 = 1, se asigna 1
puts contador[:a]  # => 1 (correcto)

# PELIGRO: con objetos mutables como default
hash = Hash.new([])
hash[:a] << 1
hash[:b] << 2
puts hash.inspect  # => {} (no ha asignado nada a :a ni :b!)
# El problema: hash[:a] devuelve el array default, lo modifica,
# pero no lo asigna a :a.

# Solucion: usar bloque
hash2 = Hash.new { |h, k| h[k] = [] }
hash2[:a] << 1
hash2[:b] << 2
puts hash2.inspect  # => {:a=>[1], :b=>[2]}
```

═══════════════════════════════════════════════════════════════════════════════════
                    6. FETCH — ACCESO SEGURO
═══════════════════════════════════════════════════════════════════════════════════

fetch es similar a [] pero lanza excepción si la clave no existe,
a menos que le proporciones un valor por defecto.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 8 — fetch

```ruby
persona = { nombre: "Ana", edad: 30 }

# fetch exitoso
puts persona.fetch(:nombre)  # => "Ana"

# fetch con clave inexistente
# puts persona.fetch(:pais)  # KeyError! (comentado)

# fetch con valor por defecto
puts persona.fetch(:pais, "No especificado")  # => "No especificado"

# fetch con bloque
puts persona.fetch(:pais) { |clave| "La clave #{clave} no existe" }
# => "La clave pais no existe"
```

═══════════════════════════════════════════════════════════════════════════════════
                    7. DIG — ACCESO ANIDADO SEGURO
═══════════════════════════════════════════════════════════════════════════════════

dig (Ruby 2.3+) permite acceder a claves anidadas sin preocuparte por
errores de nil intermedios.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 9 — dig

```ruby
data = {
  usuario: {
    nombre: "Ana",
    direccion: {
      ciudad: "Lima",
      pais: "Peru"
    },
    preferencias: {
      color: "azul"
    }
  }
}

# Acceso anidado tradicional
puts data[:usuario][:direccion][:ciudad]  # => "Lima"

# Si algo es nil, se rompe
# puts data[:usuario][:telefono][:codigo]  # NoMethodError!

# Con dig, seguro
puts data.dig(:usuario, :direccion, :ciudad)  # => "Lima"
puts data.dig(:usuario, :telefono, :codigo)    # => nil (seguro)

# Encadenamiento con dig
puts data.dig(:usuario, :preferencias, :color)  # => "azul"
puts data.dig(:usuario, :preferencias, :tema)   # => nil
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 10 — dig con arrays

```ruby
datos = {
  estudiantes: [
    { nombre: "Ana", notas: [85, 92, 78] },
    { nombre: "Bob", notas: [73, 88] },
    { nombre: "Carlos", notas: [95] }
  ]
}

# Acceder a la segunda nota del primer estudiante
puts datos.dig(:estudiantes, 0, :notas, 1)  # => 92

# Acceso seguro a profundidad variable
puts datos.dig(:estudiantes, 5, :notas, 0)  # => nil (no hay indice 5)
```

═══════════════════════════════════════════════════════════════════════════════════
                    8. MERGE — FUSIÓN DE HASHES
═══════════════════════════════════════════════════════════════════════════════════

merge combina dos hashes. Si hay claves duplicadas, el valor del segundo
hash prevalece (a menos que uses un bloque).

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 11 — merge básico

```ruby
predeterminado = { nombre: "Invitado", edad: 0, ciudad: "Desconocida" }
personalizado = { nombre: "Ana", edad: 30 }

# Merge: personalizado prevalece sobre predeterminado
usuario = predeterminado.merge(personalizado)
puts usuario.inspect
# => {:nombre=>"Ana", :edad=>30, :ciudad=>"Desconocida"}

# El original no se modifica
puts predeterminado.inspect
# => {:nombre=>"Invitado", :edad=>0, :ciudad=>"Desconocida"}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 12 — merge con bloque

```ruby
a = { nombre: "Ana", edad: 30, ciudad: "Lima" }
b = { nombre: "Bob", edad: 25, pais: "Peru" }

# Con bloque para resolver conflictos
fusion = a.merge(b) do |clave, valor_a, valor_b|
  # En caso de conflicto, concatenar
  "#{valor_a} / #{valor_b}"
end
puts fusion.inspect
# => {:nombre=>"Ana / Bob", :edad=>"30 / 25", :ciudad=>"Lima", :pais=>"Peru"}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 13 — merge! (modifica el original)

```ruby
config = { host: "localhost", port: 3000 }
nuevos = { port: 8080, debug: true }

config.merge!(nuevos)
puts config.inspect
# => {:host=>"localhost", :port=>8080, :debug=>true}
```

═══════════════════════════════════════════════════════════════════════════════════
                    9. TRANSFORM_KEYS — TRANSFORMAR CLAVES
═══════════════════════════════════════════════════════════════════════════════════

transform_keys (Ruby 2.5+) transforma las claves del hash según un bloque.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 14 — transform_keys

```ruby
hash = { "nombre" => "Ana", "edad" => 30, "ciudad" => "Lima" }

# Convertir claves a simbolos
simbolizado = hash.transform_keys { |k| k.to_sym }
puts simbolizado.inspect
# => {:nombre=>"Ana", :edad=>30, :ciudad=>"Lima"}

# Convertir claves a strings
hash2 = { nombre: "Ana", edad: 30 }
stringeado = hash2.transform_keys(&:to_s)
puts stringeado.inspect
# => {"nombre"=>"Ana", "edad"=>30}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 15 — transform_keys con mapeo

```ruby
hash = { "first_name" => "Ana", "last_name" => "Lopez", "age" => 30 }

# Mapear claves especificas
mapeo = hash.transform_keys do |k|
  case k
  when "first_name" then :nombre
  when "last_name"  then :apellido
  when "age"        then :edad
  else k.to_sym
  end
end
puts mapeo.inspect
# => {:nombre=>"Ana", :apellido=>"Lopez", :edad=>30}
```

═══════════════════════════════════════════════════════════════════════════════════
                    10. TRANSFORM_VALUES — TRANSFORMAR VALORES
═══════════════════════════════════════════════════════════════════════════════════

transform_values (Ruby 2.4+) transforma los valores del hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 16 — transform_values

```ruby
hash = { a: 1, b: 2, c: 3 }

# Duplicar valores
dobles = hash.transform_values { |v| v * 2 }
puts dobles.inspect  # => {a: 2, b: 4, c: 6}

# Convertir valores a strings
strs = hash.transform_values(&:to_s)
puts strs.inspect  # => {a: "1", b: "2", c: "3"}

# transform_values! (modifica in-place)
h = { x: 10, y: 20 }
h.transform_values! { |v| v + 1 }
puts h.inspect  # => {x: 11, y: 21}
```

═══════════════════════════════════════════════════════════════════════════════════
                    11. OTRAS OPERACIONES ESENCIALES
═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 17 — length, empty?, keys, values

```ruby
h = { a: 1, b: 2, c: 3 }

puts h.length  # => 3
puts h.size    # => 3 (alias)
puts h.empty?  # => false
puts {}.empty? # => true

puts h.keys.inspect   # => [:a, :b, :c]
puts h.values.inspect # => [1, 2, 3]
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 18 — key? / has_key? / include? / member?

```ruby
h = { nombre: "Ana", edad: 30 }

puts h.key?(:nombre)     # => true
puts h.has_key?(:edad)   # => true (alias)
puts h.include?(:ciudad) # => false (alias)
puts h.member?(:nombre)  # => true (alias)

puts h.value?(30)   # => true
puts h.has_value?(25)  # => false
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 19 — fetch_values

```ruby
h = { a: 1, b: 2, c: 3, d: 4 }

# Extraer valores de multiples claves
puts h.fetch_values(:a, :c).inspect  # => [1, 3]

# Si una clave no existe, lanza KeyError
# h.fetch_values(:a, :z)  # KeyError!

# Con bloque para claves faltantes
puts h.fetch_values(:a, :z) { |k| "no encontrado: #{k}" }.inspect
# => [1, "no encontrado: z"]
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 20 — slice

```ruby
h = { a: 1, b: 2, c: 3, d: 4, e: 5 }

# Extraer un subconjunto del hash
subset = h.slice(:a, :c, :e)
puts subset.inspect  # => {a: 1, c: 3, e: 5}

# Si una clave no existe, se omite
puts h.slice(:a, :z).inspect  # => {a: 1}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 21 — except (Rails / ActiveSupport), pero podemos simularlo

```ruby
# Ruby puro no tiene except, pero podemos hacerlo:
h = { a: 1, b: 2, c: 3, d: 4 }

# Excluir ciertas claves
sin_b_c = h.reject { |k, _| k == :b || k == :c }
puts sin_b_c.inspect  # => {a: 1, d: 4}

# Tambien con slice + keys
excluir = [:b, :c]
resultado = h.reject { |k, _| excluir.include?(k) }
puts resultado.inspect  # => {a: 1, d: 4}
```

                    12. ITERACIÓN SOBRE HASHES
═══════════════════════════════════════════════════════════════════════════════════

Existen múltiples formas de iterar sobre un hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 22 — each con hash

```ruby
persona = { nombre: "Ana", edad: 30, ciudad: "Lima" }

persona.each do |clave, valor|
  puts "#{clave.capitalize}: #{valor}"
end
# => Nombre: Ana
# => Edad: 30
# => Ciudad: Lima

# each_key
persona.each_key { |k| puts "Clave: #{k}" }

# each_value
persona.each_value { |v| puts "Valor: #{v}" }
```

═══════════════════════════════════════════════════════════════════════════════════
                    13. COMPARACIÓN DE HASHES
═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 23 — Comparando hashes

```ruby
a = { nombre: "Ana", edad: 30 }
b = { edad: 30, nombre: "Ana" }
c = { nombre: "Ana", edad: 25 }

# Son iguales si tienen las mismas claves y valores
puts a == b  # => true (el orden no importa desde Ruby 1.9? en realidad sí importa)
puts a == c  # => false

# eql? es mas estricto (compara tipo)
puts a.eql?(b)   # => true
puts a.eql?({})  # => false

# Con strict: claves y valores exactos
```

                    14. HASHES ANIDADOS
═══════════════════════════════════════════════════════════════════════════════════

Los hashes pueden contener otros hashes (y arrays) para formar estructuras
complejas.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 24 — Hashes anidados

```ruby
empresa = {
  nombre: "TechCorp",
  empleados: [
    { nombre: "Ana", puesto: "Ingeniera", salario: 5000 },
    { nombre: "Bob", puesto: "Disenador", salario: 4500 },
    { nombre: "Carlos", puesto: "Gerente", salario: 7000 }
  ],
  direccion: {
    calle: "Av. Principal 123",
    ciudad: "Lima",
    pais: "Peru"
  }
}

puts empresa[:direccion][:ciudad]           # => "Lima"
puts empresa[:empleados][0][:nombre]        # => "Ana"
puts empresa.dig(:empleados, 1, :puesto)    # => "Disenador"
```

═══════════════════════════════════════════════════════════════════════════════════
                    15. HASH COMO ARGUMENTO DE MÉTODO
═══════════════════════════════════════════════════════════════════════════════════

Una de las características mas utiles de Ruby es el uso de hashes como
argumentos con nombre simulados.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 25 — Hash como argumento

```ruby
def configurar_servidor(opciones = {})
  host = opciones.fetch(:host, "localhost")
  port = opciones.fetch(:port, 3000)
  debug = opciones.fetch(:debug, false)

  puts "Servidor iniciado en #{host}:#{port} (debug=#{debug})"
end

configurar_servidor(host: "example.com", port: 8080, debug: true)
# => Servidor iniciado en example.com:8080 (debug=true)

configurar_servidor
# => Servidor iniciado en localhost:3000 (debug=false)
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 26 — Keyword arguments (Ruby 2.0+)

```ruby
def saludar(nombre:, saludo: "Hola", puntuacion: "!")
  puts "#{saludo}, #{nombre}#{puntuacion}"
end

saludar(nombre: "Ana")                  # => Hola, Ana!
saludar(nombre: "Bob", saludo: "Hey")   # => Hey, Bob!
```

                    16. INVERT — INVERSIÓN DE CLAVE-VALOR
═══════════════════════════════════════════════════════════════════════════════════

invert intercambia claves y valores.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 27 — invert

```ruby
original = { a: 1, b: 2, c: 3 }

invertido = original.invert
puts invertido.inspect  # => {1=>:a, 2=>:b, 3=>:c}

# Cuidado: si hay valores duplicados, uno prevalece
h = { a: 1, b: 1, c: 2 }
puts h.invert.inspect  # => {1=>:b, 2=>:c} (b prevalece sobre a)
```

═══════════════════════════════════════════════════════════════════════════════════
                    17. SELECT / REJECT EN HASHES
═══════════════════════════════════════════════════════════════════════════════════

select y reject también funcionan en hashes, devolviendo un nuevo hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 28 — select y reject en hash

```ruby
calificaciones = { Ana: 85, Bob: 73, Carlos: 95, Diana: 60, Eva: 88 }

# Seleccionar aprobados (> 70)
aprobados = calificaciones.select { |nombre, nota| nota >= 70 }
puts aprobados.inspect
# => {:Ana=>85, :Bob=>73, :Carlos=>95, :Eva=>88}

# Rechazar reprobados
reprobados = calificaciones.reject { |_, nota| nota >= 70 }
puts reprobados.inspect  # => {:Diana=>60}
```

═══════════════════════════════════════════════════════════════════════════════════
                    18. SORT_BY EN HASHES
═══════════════════════════════════════════════════════════════════════════════════

sort_by en hash devuelve un array de pares ordenados.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 29 — sort_by en hash

```ruby
calificaciones = { Ana: 85, Bob: 73, Carlos: 95, Diana: 60, Eva: 88 }

# Ordenar por nota ascendente
ordenado = calificaciones.sort_by { |_, nota| nota }
puts ordenado.inspect
# => [[:Diana, 60], [:Bob, 73], [:Ana, 85], [:Eva, 88], [:Carlos, 95]]

# Convertir de vuelta a hash
puts ordenado.to_h.inspect
# => {:Diana=>60, :Bob=>73, :Ana=>85, :Eva=>88, :Carlos=>95}

# Ordenar por nombre
por_nombre = calificaciones.sort_by { |nombre, _| nombre }
puts por_nombre.to_h.inspect
# => {:Ana=>85, :Bob=>73, :Carlos=>95, :Diana=>60, :Eva=>88}
```

En este capítulo has aprendido:

  • Los hashes son colecciones clave-valor, creados con {}.
  • Los símbolos son las claves preferidas por su eficiencia.
  • Se puede establecer un valor por defecto con Hash.new.
  • fetch proporciona acceso seguro con opción de default o bloque.
  • dig permite acceso anidado seguro sin errores nil.
  • merge fusiona dos hashes, con bloque para resolver conflictos.
  • transform_keys y transform_values modifican claves y valores.
  • keys, values, key?, value?, length, empty? son métodos esenciales.
  • select, reject, sort_by funcionan también en hashes.
  • invert intercambia claves y valores.
  • Los hashes son ideales como argumentos de métodos.

Los hashes son la navaja suiza del desarrollo Ruby. Los usaras en cada
aplicacion, cada script, cada programa que escribas.

                     FAQ — PREGUNTAS FRECUENTES
═══════════════════════════════════════════════════════════════════════════════════

P: ¿Cual es la diferencia entre usar simbolos y strings como claves?
R: Los simbolos son inmutables, se almacenan una sola vez en memoria,
y son mas rapidos para comparar. Los strings son mutables y cada vez
que los usas como clave se crea un nuevo objeto. Siempre prefiere simbolos.

P: ¿Por que mi hash con default [] no funciona correctamente?
R: Porque el mismo array se comparte entre todas las claves. Usa un bloque
en Hash.new para crear un nuevo array cada vez: Hash.new { |h, k| h[k] = [] }

P: ¿Cual es la diferencia entre fetch y []?
R: [] devuelve nil si la clave no existe. fetch lanza KeyError, a menos
que le pases un valor por defecto o bloque. Usa fetch cuando quieras
asegurar que la clave existe.

P: ¿Los hashes preservan el orden de insercion?
R: Si, desde Ruby 1.9. Cuando iteras con each, obtienes los elementos
en el orden en que fueron insertados.

P: ¿Como convierto un array en un hash?
R: Usa to_h en un array de pares: [[:a, 1], [:b, 2]].to_h => {:a=>1, :b=>2}

P: ¿Que hace dig si una clave intermedia no existe?
R: Devuelve nil sin lanzar error. Es la forma segura de navegar estructuras
anidadas.

P: ¿Como elimino una clave de un hash?
R: Usa delete: hash.delete(:clave). Devuelve el valor eliminado o nil.

P: ¿Como se si dos hashes son iguales?
R: Usa ==. Compara que tengan las mismas claves y valores, sin importar
el orden (desde Ruby 1.9 el orden se preserva pero no afecta la igualdad).
