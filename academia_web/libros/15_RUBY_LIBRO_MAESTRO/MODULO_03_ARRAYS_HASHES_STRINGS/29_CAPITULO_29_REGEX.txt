                    BIENVENIDA A LAS EXPRESIONES REGULARES
═══════════════════════════════════════════════════════════════════════════════════

Bienvenido, maestro de los patrones.

Las expresiones regulares (regex) son un lenguaje de patrones para buscar,
extraer y reemplazar texto. Son una herramienta increíblemente poderosa que
todo programador debe dominar.

En Ruby, las regex son ciudadanos de primera clase. Se crean con la sintaxis
/pattern/ y soportan opciones como i (ignore case), m (multiline), y x
(extended). Ruby tiene una implementación de regex particularmente potente
y expresiva.

Este capítulo cubre:
  • Creación de regex con /pattern/ y Regexp.new.
  • Opciones: i, m, x.
  • El operador =~ para coincidencias.
  • El método match y el objeto MatchData.
  • Grupos de captura y backreferences.
  • gsub con backreferences para reemplazos avanzados.
  • Aplicaciones prácticas de regex.

Dominar las regex es dominar el procesamiento de texto. No hay herramienta
más versátil para buscar y manipular información textual.

                    1. ¿QUÉ ES UNA EXPRESIÓN REGULAR?
═══════════════════════════════════════════════════════════════════════════════════

Una expresión regular es una secuencia de caracteres que define un patrón
de búsqueda. Se usa para:

  • Validar formatos (email, teléfono, código postal)
  • Buscar texto específico
  • Extraer partes de un texto
  • Reemplazar patrones

En Ruby, las regex son instancias de la clase Regexp.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 1 — Creación de regex

```ruby
# Sintaxis literal (más común)
patron = /ruby/
puts patron.class  # => Regexp

# Con Regexp.new
patron2 = Regexp.new("ruby")
puts patron2  # => /ruby/

# Con %r (alternativa, útil para patrones con /)
patron3 = %r{ruby}
puts patron3  # => /ruby/

# Son equivalentes
puts /ruby/ == Regexp.new("ruby")  # => true
```

═══════════════════════════════════════════════════════════════════════════════════
                    2. SINTAXIS BÁSICA DE PATRONES
═══════════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 2 — Caracteres literales y metacaracteres

```ruby
# Caracteres literales: coinciden exactamente
/ruby/  # coincide con "ruby"

# Metacaracteres: tienen significado especial
# . ^ $ * + ? { } [ ] \ | ( )

# .  -> cualquier caracter excepto nueva línea
/h.lla/  # coincide con "holla", "halla", "hulla", etc.

# *  -> 0 o más repeticiones del caracter anterior
/ho*la/  # coincide con "hla", "hola", "hoola", etc.

# +  -> 1 o más repeticiones
/ho+la/  # coincide con "hola", "hoola", pero NO con "hla"

# ?  -> 0 o 1 repetición
/ho?la/  # coincide con "hla", "hola"

# \  -> escapa metacaracteres o crea secuencias especiales
/ruby\.com/  # el punto es literal, no metacaracter
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 3 — Clases de caracteres []

```ruby
# [abc] -> un caracter: a, b, o c
/r[aeiou]by/  # coincide con "raby", "reby", "riby", "roby", "ruby"

# [a-z] -> rango: cualquier letra minúscula
/[a-z]/       # cualquier letra minúscula

# [0-9] -> cualquier dígito
/[0-9]{3}/    # exactamente 3 dígitos

# [^abc] -> negación: cualquier caracter EXCEPTO a, b, c
/[^0-9]/      # cualquier caracter que no sea dígito

# Combinación
/[a-zA-Z0-9_]/  # caracter alfanumérico o guión bajo
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 4 — Secuencias abreviadas

```ruby
# \d -> cualquier dígito (equivalente a [0-9])
/\d{3}-\d{4}/  # coincide con "123-4567"

# \w -> caracter de palabra: [a-zA-Z0-9_]
/\w+/  # una o más letras/dígitos/_

# \s -> espacio en blanco (espacio, tab, nueva línea)
/\s+/  # uno o más espacios

# \D -> NO dígito
# \W -> NO caracter de palabra
# \S -> NO espacio en blanco

# \b -> límite de palabra
/\bruby\b/  # "ruby" como palabra completa, no "rubyist"

# ^ -> inicio de línea
/^ruby/  # linea que empieza con "ruby"

# $ -> fin de línea
/ruby$/  # linea que termina con "ruby"
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 5 — Cuantificadores

```ruby
# {n}    -> exactamente n veces
/\d{3}/     # 3 dígitos

# {n,}   -> n o más veces
/\d{3,}/    # 3 o más dígitos

# {n,m}  -> entre n y m veces
/\d{2,4}/   # 2, 3, o 4 dígitos

# *      -> 0 o más (equivalente a {0,})
# +      -> 1 o más (equivalente a {1,})
# ?      -> 0 o 1  (equivalente a {0,1})

# Cuantificadores perezosos (lazy): *? +? ?? {n,}?
# Por defecto son greedy (toman lo máximo posible)
texto = "<h1>Título</h1><p>Párrafo</p>"
puts texto.scan(/<.*>/).inspect    # greedy: ["<h1>Título</h1><p>Párrafo</p>"]
puts texto.scan(/<.*?>/).inspect   # lazy: ["<h1>", "</h1>", "<p>", "</p>"]
```

═══════════════════════════════════════════════════════════════════════════════════
                    3. OPCIONES (FLAGS)
═══════════════════════════════════════════════════════════════════════════════════

Las opciones se colocan después del patrón: /patron/opciones.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 6 — Opción i (insensible a mayúsculas)

```ruby
# Sin i: sensible a mayúsculas
puts /ruby/.match?("Ruby")  # => false
puts /ruby/.match?("ruby")  # => true

# Con i: insensible
puts /ruby/i.match?("Ruby")  # => true
puts /ruby/i.match?("RUBY")  # => true
puts /ruby/i.match?("ruby")  # => true
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 7 — Opción m (multilínea)

```ruby
# Sin m: . no coincide con nueva línea
texto = "hola\nmundo"
puts /hola.mundo/.match?(texto)  # => false (el . no incluye \n)

# Con m: . coincide con cualquier caracter (incluyendo \n)
puts /hola.mundo/m.match?(texto)  # => true

# Nota: en Ruby, /m hace que . coincida con \n
# (diferente de otros lenguajes donde /m cambia el comportamiento de ^ y $)
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 8 — Opción x (extendido — permite espacios y comentarios)

```ruby
# Sin x: los espacios son literales
# /H o l a/  # busca "H o l a"

# Con x: los espacios se ignoran, puedes usar comentarios
patron = /
  \d{3}     # código de área
  [-.\s]    # separador
  \d{4}     # número
/x

puts patron.match?("123-4567")  # => true
puts patron.match?("123.4567")  # => true
puts patron.match?("123 4567")  # => true
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 9 — Combinación de opciones

```ruby
# Múltiples opciones
texto = "HOLA\nMUNDO"

# i + m
puts /hola.mundo/im.match?(texto)  # => true

# En Regexp.new
patron = Regexp.new("hola.mundo", Regexp::IGNORECASE | Regexp::MULTILINE)
puts patron.match?(texto)  # => true
```

                    4. EL OPERADOR =~
═══════════════════════════════════════════════════════════════════════════════════

=~ es el operador de coincidencia de patrones. Devuelve la posición (índice)
donde comienza la coincidencia, o nil si no hay coincidencia.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 10 — =~ básico

```ruby
texto = "Hola Mundo"

# patrón =~ texto
posicion = /Mundo/ =~ texto
puts posicion  # => 5 (la "M" está en el índice 5)

# texto =~ patrón (también funciona al revés)
posicion2 = texto =~ /Mundo/
puts posicion2  # => 5

# nil si no hay coincidencia
puts texto =~ /Ruby/  # => nil

# Uso en condicionales
if texto =~ /Mundo/
  puts "Encontrado en posición #{texto =~ /Mundo/}"
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 11 — =~ y variables especiales

```ruby
"Mi email es ana@email.com" =~ /(\w+)@(\w+)\.(\w+)/

# Variables especiales (obsoletas pero aún funcionales)
puts $&  # => "ana@email.com" (coincidencia completa)
puts $1  # => "ana" (primer grupo)
puts $2  # => "email" (segundo grupo)
puts $3  # => "com" (tercer grupo)
puts $`  # => "Mi email es " (texto antes)
puts $'  # => "" (texto después)
```

═══════════════════════════════════════════════════════════════════════════════════
                    5. EL MÉTODO MATCH
═══════════════════════════════════════════════════════════════════════════════════

match devuelve un objeto MatchData con información detallada, o nil.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 12 — match y MatchData

```ruby
texto = "Teléfono: 123-4567"

matched = texto.match(/(\d{3})-(\d{4})/)
if matched
  puts matched[0]      # => "123-4567" (coincidencia completa)
  puts matched[1]      # => "123" (primer grupo)
  puts matched[2]      # => "4567" (segundo grupo)

  puts matched.pre_match  # => "Teléfono: "
  puts matched.post_match # => ""

  puts matched.captures.inspect  # => ["123", "4567"]

  puts matched.begin(0)  # => 10 (posición de inicio)
  puts matched.end(0)    # => 18 (posición final)
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 13 — Named captures (grupos con nombre)

```ruby
texto = "Nombre: Ana, Edad: 30"

# Grupos con nombre: (?<nombre>patrón)
patron = /Nombre: (?<nombre>\w+), Edad: (?<edad>\d+)/
matched = texto.match(patron)

if matched
  puts matched[:nombre]  # => "Ana"
  puts matched[:edad]    # => "30"

  # También accesible como string
  puts matched["nombre"]  # => "Ana"

  # Con nombres de variables locales (Ruby 2.0+)
  # if patron =~ texto
  #   puts nombre  # => "Ana" (variables locales automáticas)
  # end
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 14 — match? (Ruby 2.4+)

```ruby
texto = "Hola Mundo"

# match? es más rápido que match si solo necesitas boolean
puts texto.match?(/Mundo/)  # => true
puts texto.match?(/Ruby/)   # => false

# No crea MatchData ni asigna variables globales
```

═══════════════════════════════════════════════════════════════════════════════════
                    6. GRUPOS DE CAPTURA
═══════════════════════════════════════════════════════════════════════════════════

Los grupos se definen con paréntesis () y capturan el texto que coincide.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 15 — Grupos de captura básicos

```ruby
texto = "2026-07-02"

# Grupos de captura
patron = /(\d{4})-(\d{2})-(\d{2})/
matched = texto.match(patron)

if matched
  anio, mes, dia = matched.captures
  puts "Año: #{anio}, Mes: #{mes}, Día: #{dia}"
  # => Año: 2026, Mes: 07, Día: 02
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 16 — Grupos no capturantes (?: )

```ruby
# (?:patrón) agrupa pero no captura
texto = "ruby ruby python ruby"

# Sin grupo no capturante
resultado = texto.scan(/(ruby|python)\s*/)
puts resultado.inspect  # => [["ruby"], ["ruby"], ["python"], ["ruby"]]

# Con grupo no capturante (solo capturamos la palabra completa)
resultado2 = texto.scan(/(?:ruby|python)\s*/)
puts resultado2.inspect  # => ["ruby ", "ruby ", "python ", "ruby"]
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 17 — Lookahead y Lookbehind

```ruby
# Lookahead positivo (?=patrón): sigue si el patrón sigue
# Extraer dígitos seguidos de "px"
texto = "200px 300px 150em 100px"
pixeles = texto.scan(/\d+(?=px)/)
puts pixeles.inspect  # => ["200", "300", "100"]

# Lookahead negativo (?!patrón): sigue si el patrón NO sigue
no_pixeles = texto.scan(/\d+(?!px)/)
puts no_pixeles.inspect  # => ["15", "0"] (de "150em")

# Lookbehind positivo (?<=patrón): sigue si precede el patrón
# Extraer dígitos después de "precio: "
precios = "precio: 100, descuento: 20, precio: 50"
puts precios.scan(/(?<=precio: )\d+/).inspect  # => ["100", "50"]
```

                    7. GSUB CON BACKREFERENCES
═══════════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 18 — Backreferences básicas

```ruby
# \1, \2, etc. se refieren a los grupos capturados
texto = "John Doe"

# Intercambiar nombre y apellido
intercambiado = texto.gsub(/(\w+)\s+(\w+)/, '\2, \1')
puts intercambiado  # => "Doe, John"
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 19 — Backreferences en casos reales

```ruby
# Formatear números de teléfono
telefonos = "1234567 9876543 5551234"
formateados = telefonos.gsub(/(\d{3})(\d{4})/, '(\1) \2')
puts formateados  # => "(123) 4567 (987) 6543 (555) 1234"

# Ocultar datos sensibles
data = "Tarjeta: 1234-5678-9012-3456"
oculto = data.gsub(/(\d{4}-){3}\d{4}/, '****-****-****-\3')
puts oculto  # => "Tarjeta: ****-****-****-3456"
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 20 — gsub con bloque y backreferences

```ruby
texto = "El precio es $25 y el descuento es $5"

# Incrementar precios en 10%
resultado = texto.gsub(/\$(\d+)/) do |match|
  numero = $1.to_i
  "$#{numero + 10}"
end
puts resultado  # => "El precio es $35 y el descuento es $15"
```

                    8. REGEXP.NEW — CREACIÓN DINÁMICA
═══════════════════════════════════════════════════════════════════════════════════

A veces necesitas crear regex dinámicamente a partir de strings.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 21 — Regexp.new

```ruby
# Crear regex a partir de un string
patron_str = "ruby"
patron = Regexp.new(patron_str, Regexp::IGNORECASE)
puts patron  # => /ruby/i

# Con opciones
Regexp.new("pattern")                       # sin opciones
Regexp.new("pattern", "i")                  # ignore case
Regexp.new("pattern", Regexp::IGNORECASE)   # usando constantes
Regexp.new("pattern", Regexp::IGNORECASE | Regexp::MULTILINE)  # múltiples

# Escapar caracteres especiales
texto_buscar = "ruby.com"
patron_escapado = Regexp.new(Regexp.escape(texto_buscar))
puts patron_escapado  # => /ruby\.com/
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 22 — Regexp.union

```ruby
# Unir múltiples patrones en uno
palabras = ["ruby", "python", "java"]
patron = Regexp.union(palabras)
puts patron  # => /ruby|python|java/

texto = "Me gusta ruby y java"
encontradas = texto.scan(patron)
puts encontradas.inspect  # => ["ruby", "java"]
```

Ejemplo 23 — Validación de email

```ruby
def email_valido?(email)
  patron = /\A[\w+\-.]+@[a-z\d\-]+(\.[a-z\d\-]+)*\.[a-z]+\z/i
  email.match?(patron)
end

puts email_valido?("user@example.com")   # => true
puts email_valido?("user.name+tag@example.co.uk")  # => true
puts email_valido?("user@example")       # => false
puts email_valido?("@example.com")       # => false
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 24 — Validación de URL

```ruby
def url_valida?(url)
  patron = %r{\Ahttps?://[\w\-]+(\.[\w\-]+)+[/#?]?.*\z}
  url.match?(patron)
end

puts url_valida?("https://ruby-lang.org")          # => true
puts url_valida?("http://example.com/path?q=1")     # => true
puts url_valida?("ftp://example.com")               # => false
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 25 — Extraer URLs de un texto

```ruby
texto = "Visita https://ruby-lang.org y http://example.com"
urls = texto.scan(%r{https?://[^\s]+})
puts urls.inspect  # => ["https://ruby-lang.org", "http://example.com"]
```

Ejemplo 26 — Filtrar líneas de log

```ruby
logs = <<~LOGS
  2026-07-02 INFO  Usuario inició sesión
  2026-07-02 ERROR Conexión fallida
  2026-07-02 WARN  Intento de acceso no autorizado
  2026-07-02 ERROR Timeout en servidor
LOGS

# Extraer solo errores
errores = logs.scan(/^.*ERROR.*$/)
puts errores
# => 2026-07-02 ERROR Conexión fallida
# => 2026-07-02 ERROR Timeout en servidor

# Extraer fecha y mensaje de errores
logs.scan(/^(\d{4}-\d{2}-\d{2})\s+ERROR\s+(.*)$/) do |fecha, msg|
  puts "[#{fecha}] #{msg}"
end
# => [2026-07-02] Conexión fallida
# => [2026-07-02] Timeout en servidor
```

Ejemplo 27 — Sanitizar texto

```ruby
texto_sucio = "Hola <b>Mundo</b>, visita mi <a href='x'>link</a>!"

# Eliminar etiquetas HTML
limpio = texto_sucio.gsub(/<[^>]*>/, "")
puts limpio  # => "Hola Mundo, visita mi link!"

# Eliminar caracteres no alfanuméricos (excepto espacios)
solo_texto = limpio.gsub(/[^a-zA-Záéíóúüñ\s]/, "")
puts solo_texto  # => "Hola Mundo visita mi link"
```

Ejemplo 28 — Parsing de fechas

```ruby
def parsear_fechas(texto)
  patron = /(\d{4})[-\/](\d{2})[-\/](\d{2})/
  texto.scan(patron).map do |anio, mes, dia|
    { anio: anio.to_i, mes: mes.to_i, dia: dia.to_i }
  end
end

texto = "Eventos: 2026-07-02, 2026/12/25, 2027-01-15"
fechas = parsear_fechas(texto)
fechas.each { |f| puts "#{f[:dia]}/#{f[:mes]}/#{f[:anio]}" }
# => 2/7/2026
# => 25/12/2026
# => 15/1/2027
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 29 — Contar palabras con regex

```ruby
def frecuencia_palabras(texto)
  texto.downcase
       .scan(/\b[a-záéíóúüñ]{3,}\b/)
       .each_with_object(Hash.new(0)) { |p, h| h[p] += 1 }
       .sort_by { |_, v| -v }
       .first(10)
       .to_h
end

texto = "Ruby es un lenguaje hermoso y poderoso. Ruby es divertido."
puts frecuencia_palabras(texto).inspect
# => {"ruby"=>2, "es"=>2, "lenguaje"=>1, "hermoso"=>1, ...}
```

  • Backreferences \1-\9 en gsub.
  • Regexp.union y Regexp.escape para creación dinámica.
  • Aplicaciones: validación, parsing, sanitización.

Las expresiones regulares son una habilidad que se desarrolla con la práctica.
Úsalas con moderación y siempre prioriza la legibilidad.

                     FAQ — PREGUNTAS FRECUENTES
═══════════════════════════════════════════════════════════════════════════════════

P: ¿Cuál es la diferencia entre =~ y match?
R: =~ devuelve la posición (entero) o nil. match devuelve un objeto MatchData
o nil. match es más informativo.

P: ¿Qué son los grupos no capturantes (?:...)?
R: Agrupan sin capturar. Útiles para aplicar cuantificadores sin almacenar
el resultado.

P: ¿Cómo hago una regex insensible a mayúsculas?
R: Añade la opción i: /ruby/i.

P: ¿Qué significa \b en una regex?
R: \b es un límite de palabra. Coincide entre un caracter de palabra (\w)
y uno que no lo es.

P: ¿Cómo creo una regex dinámicamente?
R: Usa Regexp.new(string). Si el string contiene metacaracteres, usa
Regexp.escape primero.

P: ¿Qué es un cuantificador perezoso?
R: Añade ? después del cuantificador: *?, +?, ??, {n,}?. Toma la menor
cantidad posible en lugar de la mayor.

P: ¿Cómo uso múltiples opciones en una regex?
R: /patron/im (combina i y m) o Regexp.new("patron", Regexp::IGNORECASE | Regexp::MULTILINE).

P: ¿Qué hace Regexp.union?
R: Combina varios patrones en uno usando |. Útil para buscar cualquiera de
varias palabras.
