Bienvenido, maestro de la asociación clave-valor.

En el capítulo anterior aprendiste los fundamentos de los hashes: crearlos,
acceder a ellos y transformarlos. Ahora es momento de profundizar en los

                    1. EACH_KEY — ITERACIÓN SOBRE CLAVES
═══════════════════════════════════════════════════════════════════════════════════

each_key itera sobre cada clave del hash, ignorando los valores.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 1 — each_key básico

```ruby
persona = { nombre: "Ana", edad: 30, ciudad: "Lima" }

puts "Claves del hash:"
persona.each_key { |clave| puts "  > #{clave}" }
# => Claves del hash:
#   > nombre
#   > edad
#   > ciudad
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 2 — each_key para validación

```ruby
opciones = { host: "localhost", port: 3000, debug: true }

# Verificar que solo existan claves permitidas
permitidas = [:host, :port, :debug, :ssl]
opciones.each_key do |k|
  unless permitidas.include?(k)
    puts "ADVERTENCIA: Clave desconocida '#{k}'"
  end
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 3 — each_key para transformación

```ruby
hash = { "nombre" => "Ana", "edad" => 30, "activo" => true }

# Convertir todas las claves a mayúsculas
nuevo_hash = {}
hash.each_key { |k| nuevo_hash[k.upcase] = hash[k] }
puts nuevo_hash.inspect
# => {"NOMBRE"=>"Ana", "EDAD"=>30, "ACTIVO"=>true}
```

═══════════════════════════════════════════════════════════════════════════════════
                    2. EACH_VALUE — ITERACIÓN SOBRE VALORES
═══════════════════════════════════════════════════════════════════════════════════

each_value itera sobre cada valor del hash, ignorando las claves.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 4 — each_value básico

```ruby
calificaciones = { Ana: 85, Bob: 73, Carlos: 95, Diana: 60 }

puts "Calificaciones:"
calificaciones.each_value { |v| puts "  #{v}" }
# => 85
# => 73
# => 95
# => 60

# Calcular promedio
suma = 0
calificaciones.each_value { |v| suma += v }
promedio = suma / calificaciones.length.to_f
puts "Promedio: #{promedio.round(2)}"  # => Promedio: 78.25
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 5 — each_value para verificación

```ruby
config = { host: "localhost", port: 3000, debug: true, ssl: false }

# Verificar que todos los valores sean del tipo esperado
config.each_value do |v|
  unless [String, Integer, TrueClass, FalseClass].include?(v.class)
    puts "Valor con tipo inesperado: #{v.class}"
  end
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 6 — each_value con transformación

```ruby
precios = { laptop: 1000, mouse: 25, teclado: 80, monitor: 300 }

# Aplicar IVA (18%)
precios.each_value { |p| puts "Precio con IVA: $#{p * 1.18}" }
# => Precio con IVA: $1180.0
# => Precio con IVA: $29.5
# => Precio con IVA: $94.4
# => Precio con IVA: $354.0
```

═══════════════════════════════════════════════════════════════════════════════════
                    3. HAS_KEY? / KEY? — ¿EXISTE LA CLAVE?
═══════════════════════════════════════════════════════════════════════════════════

has_key? (alias key?) verifica si una clave existe en el hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 7 — has_key? y key?

```ruby
usuario = { nombre: "Ana", email: "ana@email.com", rol: "admin" }

puts usuario.has_key?(:nombre)  # => true
puts usuario.key?(:email)       # => true
puts usuario.key?(:telefono)    # => false

# Alias adicionales
puts usuario.include?(:rol)     # => true
puts usuario.member?(:nombre)   # => true
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 8 — has_key? en condicionales

```ruby
def procesar_usuario(datos)
  if datos.key?(:email)
    puts "Enviando email a #{datos[:email]}"
  else
    puts "Error: email requerido"
  end
end

procesar_usuario({ nombre: "Ana", email: "ana@email.com" })
# => Enviando email a ana@email.com

procesar_usuario({ nombre: "Bob" })
# => Error: email requerido
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 9 — has_value? / value?

```ruby
hash = { a: 1, b: 2, c: 3 }

puts hash.has_value?(2)  # => true
puts hash.value?(5)      # => false
```

═══════════════════════════════════════════════════════════════════════════════════
                    4. KEYS — OBTENER TODAS LAS CLAVES
═══════════════════════════════════════════════════════════════════════════════════

keys devuelve un array con todas las claves del hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 10 — keys

```ruby
persona = { nombre: "Ana", edad: 30, ciudad: "Lima", profesion: "Ingeniera" }

claves = persona.keys
puts claves.inspect  # => [:nombre, :edad, :ciudad, :profesion]
puts claves.class    # => Array

# keys es útil para iterar o validar
persona.keys.each { |k| puts "Propiedad: #{k}" }
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 11 — keys para validación

```ruby
def validar_objeto(objeto, requeridas)
  faltantes = requeridas - objeto.keys
  if faltantes.any?
    puts "Faltan las propiedades: #{faltantes.join(', ')}"
  else
    puts "Objeto valido"
  end
end

req = [:nombre, :email, :edad]
validar_objeto({ nombre: "Ana", email: "a@a.com", edad: 30 }, req)
# => Objeto valido

validar_objeto({ nombre: "Bob" }, req)
# => Faltan las propiedades: email, edad
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 12 — keys.length para conteo

```ruby
config = { a: 1, b: 2, c: 3, d: 4, e: 5 }

puts "El hash tiene #{config.keys.length} claves"  # => 5
puts "Equivalente a: #{config.length}"              # => 5
```

═══════════════════════════════════════════════════════════════════════════════════
                    5. VALUES — OBTENER TODOS LOS VALORES
═══════════════════════════════════════════════════════════════════════════════════

values devuelve un array con todos los valores del hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 13 — values

```ruby
calificaciones = { Ana: 85, Bob: 73, Carlos: 95, Diana: 60, Eva: 88 }

notas = calificaciones.values
puts notas.inspect  # => [85, 73, 95, 60, 88]

# Calcular estadisticas
puts "Nota maxima: #{notas.max}"    # => 95
puts "Nota minima: #{notas.min}"    # => 60
puts "Promedio: #{notas.sum / notas.length.to_f}"  # => 80.2
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 14 — values_at

```ruby
hash = { a: 10, b: 20, c: 30, d: 40, e: 50 }

# Extraer valores de claves específicas
puts hash.values_at(:a, :c, :e).inspect  # => [10, 30, 50]
puts hash.values_at(:b, :d).inspect      # => [20, 40]

# Si una clave no existe, devuelve nil
puts hash.values_at(:a, :x, :c).inspect  # => [10, nil, 30]
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 15 — values con select

```ruby
precios = { laptop: 1000, mouse: 25, teclado: 80, monitor: 300, webcam: 45 }

# Productos que cuestan menos de 100
baratos = precios.select { |_, p| p < 100 }
puts baratos.inspect  # => {:mouse=>25, :teclado=>80, :webcam=>45}

# Solo los valores baratos
puts baratos.values.inspect  # => [25, 80, 45]
```

═══════════════════════════════════════════════════════════════════════════════════
                    6. SORT_BY — ORDENAR HASHES
═══════════════════════════════════════════════════════════════════════════════════

sort_by en un hash devuelve un array de pares [clave, valor] ordenados.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 16 — sort_by por valor

```ruby
poblacion = {
  "Peru" => 33,
  "Colombia" => 51,
  "Chile" => 19,
  "Argentina" => 45,
  "Brasil" => 213
}

# Ordenar por poblacion ascendente
ordenado = poblacion.sort_by { |_, poblacion| poblacion }
puts ordenado.inspect
# => [["Chile", 19], ["Peru", 33], ["Argentina", 45], ["Colombia", 51], ["Brasil", 213]]

# Convertir de vuelta a hash
puts ordenado.to_h.inspect
# => {"Chile"=>19, "Peru"=>33, "Argentina"=>45, "Colombia"=>51, "Brasil"=>213}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 17 — sort_by por clave

```ruby
hash = { c: 3, a: 1, b: 2, e: 5, d: 4 }

# Ordenar alfabeticamente por clave
ordenado = hash.sort_by { |clave, _| clave }
puts ordenado.to_h.inspect  # => {a: 1, b: 2, c: 3, d: 4, e: 5}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 18 — sort_by descendente

```ruby
ventas = { enero: 1500, febrero: 2300, marzo: 1800, abril: 2100 }

# Mayor a menor
top = ventas.sort_by { |_, v| -v }
puts top.to_h.inspect  # => {:febrero=>2300, :abril=>2100, :marzo=>1800, :enero=>1500}

# O usando reverse
top2 = ventas.sort_by { |_, v| v }.reverse
puts top2.to_h.inspect  # => {:febrero=>2300, :abril=>2100, :marzo=>1800, :enero=>1500}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 19 — sort_by con múltiples criterios

```ruby
personas = {
  ana: { edad: 30, salario: 5000 },
  bob: { edad: 25, salario: 6000 },
  carlos: { edad: 35, salario: 4500 },
  diana: { edad: 25, salario: 5500 }
}

# Ordenar por edad, luego por salario
ordenado = personas.sort_by { |_, datos| [datos[:edad], datos[:salario]] }
ordenado.each { |nombre, datos| puts "#{nombre}: #{datos[:edad]} anios, $#{datos[:salario]}" }
# bob: 25, $6000
# diana: 25, $5500
# ana: 30, $5000
# carlos: 35, $4500
```

═══════════════════════════════════════════════════════════════════════════════════
                    7. TO_A — CONVERSIÓN A ARRAY
═══════════════════════════════════════════════════════════════════════════════════

to_a convierte el hash en un array de arrays (pares clave-valor).

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 20 — to_a

```ruby
hash = { a: 1, b: 2, c: 3 }

array = hash.to_a
puts array.inspect  # => [[:a, 1], [:b, 2], [:c, 3]]
puts array.class    # => Array

# to_a es util para operaciones que requieren arrays
pares = hash.to_a
pares.each { |clave, valor| puts "#{clave} => #{valor}" }
# => a => 1
# => b => 2
# => c => 3
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 21 — to_a para manipulación

```ruby
hash = { nombre: "Ana", edad: 30, ciudad: "Lima" }

# Convertir a array, agregar un par, volver a hash
pares = hash.to_a
pares << [:pais, "Peru"]
nuevo_hash = pares.to_h
puts nuevo_hash.inspect
# => {:nombre=>"Ana", :edad=>30, :ciudad=>"Lima", :pais=>"Peru"}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 22 — flatten en hash

```ruby
hash = { a: [1, 2], b: [3, 4] }

# to_a + flatten
plano = hash.to_a.flatten
puts plano.inspect  # => [:a, [1, 2], :b, [3, 4]]

# flatten(1) solo un nivel
puts hash.to_a.flatten(1).inspect  # => [:a, [1, 2], :b, [3, 4]]
```

═══════════════════════════════════════════════════════════════════════════════════
                    8. INVERT — INVERSIÓN CLAVE-VALOR
═══════════════════════════════════════════════════════════════════════════════════

invert intercambia claves y valores. Los valores se convierten en claves
y las claves en valores.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 23 — invert básico

```ruby
original = { a: 1, b: 2, c: 3 }

invertido = original.invert
puts invertido.inspect  # => {1=>:a, 2=>:b, 3=>:c}

# Acceso inverso
puts invertido[2]  # => :b
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 24 — invert con valores duplicados

```ruby
# Si hay valores duplicados, el ultimo prevalece
h = { a: 1, b: 1, c: 2, d: 3 }
puts h.invert.inspect  # => {1=>:b, 2=>:c, 3=>:d} (b sobreescribe a)

# Para manejar duplicados, podemos usar group_by
h2 = { a: 1, b: 1, c: 2, d: 3 }
invertido_seguro = h2.group_by { |_, v| v }.transform_values { |pares| pares.map(&:first) }
puts invertido_seguro.inspect
# => {1=>[:a, :b], 2=>[:c], 3=>[:d]}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 25 — invert para lookup inverso

```ruby
codigos = { peru: "PE", chile: "CL", argentina: "AR", brasil: "BR" }

# Lookup directo: pais a codigo
puts codigos[:peru]  # => "PE"

# Lookup inverso: codigo a pais
codigos_por_codigo = codigos.invert
puts codigos_por_codigo["PE"]  # => :peru
puts codigos_por_codigo["BR"]  # => :brasil
```

═══════════════════════════════════════════════════════════════════════════════════
                    9. SELECT / REJECT EN HASHES
═══════════════════════════════════════════════════════════════════════════════════

select y reject filtran pares clave-valor según una condición.
Devuelven un nuevo hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 26 — select en hash

```ruby
calificaciones = { Ana: 85, Bob: 73, Carlos: 95, Diana: 60, Eva: 88 }

# Aprobados (>= 70)
aprobados = calificaciones.select { |nombre, nota| nota >= 70 }
puts aprobados.inspect
# => {:Ana=>85, :Bob=>73, :Carlos=>95, :Eva=>88}

# Solo los que tienen A (>= 90)
sobresalientes = calificaciones.select { |_, n| n >= 90 }
puts sobresalientes.inspect  # => {:Carlos=>95}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 27 — reject en hash

```ruby
inventario = {
  laptop: { cantidad: 5, precio: 1000 },
  mouse: { cantidad: 0, precio: 25 },
  teclado: { cantidad: 10, precio: 80 },
  monitor: { cantidad: 0, precio: 300 }
}

# Productos sin stock
sin_stock = inventario.reject { |_, datos| datos[:cantidad] > 0 }
puts sin_stock.inspect
# => {:mouse=>{:cantidad=>0, :precio=>25}, :monitor=>{:cantidad=>0, :precio=>300}}

# Productos con stock
con_stock = inventario.select { |_, datos| datos[:cantidad] > 0 }
puts con_stock.keys.inspect  # => [:laptop, :teclado]
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 28 — select! y reject! (versiones bang)

```ruby
hash = { a: 1, b: 2, c: 3, d: 4, e: 5 }

# select! modifica el original
hash.select! { |_, v| v.even? }
puts hash.inspect  # => {:b=>2, :d=>4}

# reject!
hash2 = { a: 1, b: 2, c: 3, d: 4, e: 5 }
hash2.reject! { |_, v| v < 3 }
puts hash2.inspect  # => {:c=>3, :d=>4, :e=>5}
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 29 — select con claves

```ruby
hash = { nombre: "Ana", edad: 30, email: "ana@mail.com", password: "secreto", rol: "admin" }

# Seleccionar solo claves permitidas (whitelist)
permitidas = [:nombre, :email, :rol]
filtrado = hash.select { |k, _| permitidas.include?(k) }
puts filtrado.inspect
# => {:nombre=>"Ana", :email=>"ana@mail.com", :rol=>"admin"}

# Excluir claves sensibles (blacklist)
sensibles = [:password]
seguro = hash.reject { |k, _| sensibles.include?(k) }
puts seguro.inspect
# => {:nombre=>"Ana", :edad=>30, :email=>"ana@mail.com", :rol=>"admin"}
```

                    10. OTROS MÉTODOS ESENCIALES
═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 30 — delete

```ruby
hash = { a: 1, b: 2, c: 3 }

# delete devuelve el valor eliminado
eliminado = hash.delete(:b)
puts eliminado   # => 2
puts hash.inspect  # => {a: 1, c: 3}

# delete con bloque (si la clave no existe)
resultado = hash.delete(:z) { |k| "Clave #{k} no encontrada" }
puts resultado  # => "Clave z no encontrada"
```

# shift elimina y devuelve el primer par
par = hash.shift
puts par.inspect   # => [:a, 1]
puts hash.inspect  # => {b: 2, c: 3}
```

Ejemplo 35 — assoc y rassoc

```

Ejemplo 39 — merge con bloque (repaso avanzado)

```ruby
a = { a: 1, b: 2, c: 3 }
b = { a: 10, c: 30, d: 40 }

# Sumar valores en conflicto
suma = a.merge(b) { |_, v1, v2| v1 + v2 }
puts suma.inspect  # => {a: 11, b: 2, c: 33, d: 40}
```

                     FAQ — PREGUNTAS FRECUENTES
═══════════════════════════════════════════════════════════════════════════════════

P: ¿Cuándo usaría each_key en lugar de each?
R: Cuando solo te interesan las claves. Es más explícito y evita crear
variables no utilizadas en el bloque.

P: ¿Cuál es la diferencia entre keys y to_a?
R: keys devuelve un array de claves solamente. to_a devuelve un array de
pares [clave, valor].

P: ¿Qué sucede con invert si hay valores duplicados?
R: La última clave prevalece. Si necesitas preservar todas las claves,
usa group_by seguido de transform_values.

P: ¿Cómo ordeno un hash por valor numérico?
R: Usa sort_by { |_, valor| valor }. Si quieres descendente, usa -valor
o .reverse al final.

P: ¿select en hash devuelve un hash o un array?
R: Devuelve un hash. El bloque recibe |clave, valor| y debe devolver
true/false.

P: ¿Cómo elimino una clave de un hash?
R: Usa delete(:clave). Si la clave existe, devuelve el valor. Si no,
devuelve nil (o el resultado del bloque si se proporciona).

P: ¿Cuál es la diferencia entre assoc y rassoc?
R: assoc busca por clave, rassoc busca por valor. Ambos devuelven
el par [clave, valor] completo o nil.

P: ¿Cómo extraigo múltiples valores específicos de un hash?
R: Usa values_at(:clave1, :clave2, ...) o fetch_values(:clave1, :clave2).
