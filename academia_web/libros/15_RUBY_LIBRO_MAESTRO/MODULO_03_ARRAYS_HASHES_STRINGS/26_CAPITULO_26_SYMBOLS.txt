                    BIENVENIDA AL MUNDO DE LOS SÍMBOLOS
═══════════════════════════════════════════════════════════════════════════════════

Bienvenido, maestro de la eficiencia.

Los símbolos son una de las características más distintivas de Ruby. A simple
vista, parecen strings: :nombre, :edad, :ciudad. Pero bajo el capó, son mucho
más eficientes y especializados.

Un símbolo es esencialmente un nombre — una cadena inmutable que se almacena
una sola vez en memoria. Mientras que dos strings con el mismo contenido son
objetos diferentes, dos símbolos con el mismo nombre SON el mismo objeto.

Esta distinción tiene implicaciones profundas en rendimiento, uso de memoria,
y semántica del código. Los símbolos son los ciudadanos de primera clase para
claves de hash, nombres de métodos, y cualquier situación donde el identificador
importe más que el contenido textual.

═══════════════════════════════════════════════════════════════════════════════════
    ✦ "Symbols are not strings. They are names." — Yukihiro Matsumoto

═══════════════════════════════════════════════════════════════════════════════════
                    1. ¿QUÉ ES UN SÍMBOLO?
═══════════════════════════════════════════════════════════════════════════════════

Un símbolo es un objeto que representa un nombre o identificador. Se crea
anteponiendo dos puntos a un nombre.

Características clave:

  • Inmutable: no puedes modificar un símbolo.
  • Único en memoria: :nombre es siempre el mismo objeto en toda la app.
  • Ligero: la comparación entre símbolos es extremadamente rápida.
  • Ideal como identificador: para claves, estados, opciones.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 1 — Creación de símbolos

```ruby
# Sintaxis básica
s1 = :nombre
s2 = :edad
s3 = :ciudad

puts s1.class   # => Symbol
puts s1         # => :nombre

# Símbolos con espacios (usando :"")
s4 = :"hola mundo"
puts s4         # => :"hola mundo"

# Símbolos creados con %s
s5 = %s(saludo)
puts s5         # => :saludo
```

═══════════════════════════════════════════════════════════════════════════════════
                    2. SYMBOL vs STRING — DIFERENCIAS CLAVE
═══════════════════════════════════════════════════════════════════════════════════

La diferencia fundamental: dos strings con el mismo contenido son objetos
distintos; dos símbolos con el mismo nombre son el mismo objeto.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 2 — Comparación de identidad

```ruby
# Strings: objetos diferentes aunque tengan el mismo contenido
s1 = "hola"
s2 = "hola"
puts s1 == s2                    # => true (igualdad de contenido)
puts s1.object_id == s2.object_id  # => false (objetos diferentes)

# Símbolos: el mismo objeto
sym1 = :hola
sym2 = :hola
puts sym1 == sym2                     # => true
puts sym1.object_id == sym2.object_id # => true (MISMO objeto)
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 3 — Comparación de rendimiento

```ruby
require "benchmark"

n = 10_000_000

Benchmark.bm do |x|
  x.report("símbolos") do
    n.times { :nombre == :nombre }
  end

  x.report("strings ") do
    n.times { "nombre" == "nombre" }
  end
end

# Los símbolos son más rápidos porque comparan object_id
# mientras que strings comparan caracter por caracter
```

═══════════════════════════════════════════════════════════════════════════════════
                    3. INMUTABILIDAD
═══════════════════════════════════════════════════════════════════════════════════

Los símbolos son inmutables: no puedes modificar su contenido una vez creados.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 4 — Inmutabilidad

```ruby
# Los strings son mutables
saludo = "hola"
saludo << " mundo"
puts saludo  # => "hola mundo" (modificado)

# Los símbolos son inmutables
sym = :hola
# sym << :mundo  # NoMethodError: Symbol no tiene <<

# No hay manera de modificar un símbolo
# Lo que puedes hacer es crear uno nuevo
sym2 = :"#{sym}_mundo"
puts sym2  # => :hola_mundo
```

═══════════════════════════════════════════════════════════════════════════════════
                    4. MEMORIA Y EFICIENCIA
═══════════════════════════════════════════════════════════════════════════════════

Los símbolos se almacenan en una tabla global de símbolos. Cada símbolo
existe una sola vez, lo que ahorra memoria.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 5 — Memoria compartida

```ruby
# Strings: cada uno ocupa su propio espacio
10.times { "nombre".object_id }
# => 10 objetos distintos en memoria

# Símbolos: todos apuntan al mismo objeto
10.times { :nombre.object_id }
# => Mismo object_id las 10 veces

# Demostración
puts :ruby.object_id   # => 1054428 (ejemplo)
puts :ruby.object_id   # => 1054428 (el mismo)
puts "ruby".object_id  # => 70345234567890 (cambiará cada vez)
puts "ruby".object_id  # => 70345234512345 (diferente)
```

═══════════════════════════════════════════════════════════════════════════════════
                    5. SÍMBOLOS COMO CLAVES DE HASH
═══════════════════════════════════════════════════════════════════════════════════

La aplicación más común de los símbolos es como claves de hash.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 6 — Símbolos como claves

```ruby
# Con strings (menos eficiente)
persona_strings = {
  "nombre" => "Ana",
  "edad"   => 30,
  "ciudad" => "Lima"
}

# Con símbolos (más eficiente y legible)
persona_símbolos = {
  nombre: "Ana",
  edad: 30,
  ciudad: "Lima"
}

# Acceso
puts persona_símbolos[:nombre]  # => "Ana"

# Comparación de object_id de las claves
puts persona_strings.keys.map(&:object_id).inspect
# => [70345..., 70346..., 70347...] (diferentes)

puts persona_símbolos.keys.map(&:object_id).inspect
# => [1054428, 1054448, 1054478] (fijos)
```

═══════════════════════════════════════════════════════════════════════════════════
                    6. TO_SYM — CONVERSIÓN DE STRING A SÍMBOLO
═══════════════════════════════════════════════════════════════════════════════════

to_sym (alias intern) convierte un string a símbolo.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 7 — to_sym

```ruby
# Convertir string a símbolo
str = "nombre"
sym = str.to_sym
puts sym.class  # => Symbol
puts sym        # => :nombre

# Uso común: entrada de usuario a clave de hash
entrada = "email"
datos = { nombre: "Ana", email: "ana@mail.com" }

if datos.key?(entrada.to_sym)
  puts datos[entrada.to_sym]  # => "ana@mail.com"
end
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 8 — to_sym con entrada dinámica

```ruby
def buscar_en_hash(hash, clave_string)
  clave_sym = clave_string.to_sym
  hash.fetch(clave_sym, "No encontrado")
end

config = { host: "localhost", port: 3000, debug: true }

puts buscar_en_hash(config, "host")  # => "localhost"
puts buscar_en_hash(config, "port")  # => 3000
puts buscar_en_hash(config, "ssl")   # => "No encontrado"
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 9 — intern (alias de to_sym)

```ruby
str = "edad"
sym = str.intern
puts sym  # => :edad

# Ambos son equivalentes
puts "hola".to_sym == "hola".intern  # => true
```

═══════════════════════════════════════════════════════════════════════════════════
                    7. TO_S — CONVERSIÓN DE SÍMBOLO A STRING
═══════════════════════════════════════════════════════════════════════════════════

to_s convierte un símbolo a string.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 10 — to_s

```ruby
sym = :nombre
str = sym.to_s
puts str.class  # => String
puts str        # => "nombre"

# Útil para construir mensajes o nombres de métodos
accion = :crear
puts "#{accion.to_s.capitalize} usuario"  # => "Crear usuario"
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 11 — Interpolación de símbolos

```ruby
# Los símbolos también se pueden interpolar en strings
tipo = :admin
puts "El rol es #{tipo}"  # => "El rol es admin"

# Con interpolación dinámica
nombre = "usuario"
sym = :"#{nombre}"
puts sym  # => :usuario
```

═══════════════════════════════════════════════════════════════════════════════════
                    8. SYMBOL.ALL_SYMBOLS — TABLA GLOBAL
═══════════════════════════════════════════════════════════════════════════════════

Symbol.all_symbols devuelve un array con todos los símbolos actualmente
en la tabla global de símbolos de Ruby.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 12 — Symbol.all_symbols

```ruby
# Todos los símbolos actuales
todos = Symbol.all_symbols
puts "Total de símbolos: #{todos.length}"

# Verificar si un símbolo existe
puts Symbol.all_symbols.include?(:nombre)  # => true

# Encontrar símbolos que contienen "ruby"
ruby_syms = Symbol.all_symbols.select { |s| s.to_s.include?("ruby") }
puts ruby_syms.inspect
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 13 — Observar la tabla de símbolos

```ruby
puts "Símbolos antes: #{Symbol.all_symbols.length}"

# Crear algunos símbolos nuevos
:mi_nuevo_símbolo_abc123
:otro_símbolo_xyz789

puts "Símbolos después: #{Symbol.all_symbols.length}"
# Notar: los símbolos permanecen incluso si no se usan

# Esto puede ser un problema de memoria si creas muchos símbolos
# dinámicamente (LOS SÍMBOLOS NUNCA SE LIBERAN)
```

═══════════════════════════════════════════════════════════════════════════════════
                    9. SÍMBOLOS COMO MENSAJES (METHOD NAMES)
═══════════════════════════════════════════════════════════════════════════════════

Internamente, Ruby representa los nombres de métodos como símbolos.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 14 — Símbolos y métodos

```ruby
# Enviar un método usando send
obj = "Hola Mundo"
método = :length
puts obj.send(método)  # => 10

# Listar métodos de un objeto como símbolos
puts "Hola".methods.include?(:length)  # => true
puts "Hola".methods.include?(:upcase)  # => true

# respond_to? usa símbolos
puts "Hola".respond_to?(:downcase)  # => true
puts "Hola".respond_to?(:volar)     # => false
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 15 — define_method con símbolos

```ruby
class Saludo
  [:hola, :adios, :buenas].each do |saludo|
    define_method(saludo) do |nombre|
      "#{saludo.capitalize}, #{nombre}!"
    end
  end
end

s = Saludo.new
puts s.hola("Ana")     # => "Hola, Ana!"
puts s.adios("Bob")    # => "Adiós, Bob!"
puts s.buenas("Carlos") # => "Buenas, Carlos!"
```

═══════════════════════════════════════════════════════════════════════════════════
                    10. %i Y %I — ARRAYS DE SÍMBOLOS
═══════════════════════════════════════════════════════════════════════════════════

Ruby proporciona atajos para crear arrays de símbolos.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 16 — %i y %I

```ruby
# %i() crea un array de símbolos (sin interpolación)
claves = %i(nombre edad email ciudad)
puts claves.inspect  # => [:nombre, :edad, :email, :ciudad]

# %I() crea un array de símbolos (CON interpolación)
prefijo = "user"
campos = %I(#{prefijo}_name #{prefijo}_email)
puts campos.inspect  # => [:user_name, :user_email]
```

═══════════════════════════════════════════════════════════════════════════════════
                    11. SÍMBOLOS Y PROCS (&:method)
═══════════════════════════════════════════════════════════════════════════════════

La notación &:símbolo convierte un símbolo en un Proc.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 17 — &:method shorthand

```ruby
numeros = [1, 2, 3, 4, 5]

# Sin shorthand
strs1 = numeros.map { |n| n.to_s }

# Con &:method symbol shorthand
strs2 = numeros.map(&:to_s)

puts strs1.inspect  # => ["1", "2", "3", "4", "5"]
puts strs2.inspect  # => ["1", "2", "3", "4", "5"]

# Otros ejemplos
puts numeros.select(&:even?).inspect     # => [2, 4]
puts numeros.map(&:next).inspect         # => [2, 3, 4, 5, 6]

# Cómo funciona internamente:
# &:to_s  ->  Proc.new { |obj| obj.to_s }
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 18 — &:method con métodos con argumentos

```ruby
# &:method solo funciona para métodos sin argumentos
# Para métodos con argumentos, usa bloque explícito

palabras = %w(ruby python java)
# &:upcase funciona (sin argumentos)
puts palabras.map(&:upcase).inspect  # => ["RUBY", "PYTHON", "JAVA"]

# Para include?("r") necesitas bloque
con_r = palabras.select { |p| p.include?("r") }
puts con_r.inspect  # => ["ruby"]
```

═══════════════════════════════════════════════════════════════════════════════════
                    12. CUÁNDO USAR SÍMBOLOS vs STRINGS
═══════════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 19 — Guía de uso

```ruby
# USA SÍMBOLOS cuando:
#   • Son claves de hash
#   • Son nombres de métodos (send, define_method, respond_to?)
#   • Representan estados o categorías fijas
#   :pendiente, :completado, :fallido
#   • Son identificadores en configuraciones

# USA STRINGS cuando:
#   • Necesitas modificar el contenido
#   • Vienen de entrada del usuario
#   • Necesitas interpolación frecuente
#   • Son datos para mostrar al usuario

# Ejemplo de estados con símbolos
def procesar_pedido(estado)
  case estado
  when :pendiente
    puts "Pedido pendiente de revisión"
  when :enviado
    puts "Pedido enviado al cliente"
  when :entregado
    puts "Pedido entregado con éxito"
  when :cancelado
    puts "Pedido cancelado"
  else
    puts "Estado desconocido"
  end
end

procesar_pedido(:pendiente)  # => Pedido pendiente de revisión
procesar_pedido(:entregado)  # => Pedido entregado con éxito
```

                    13. PELIGRO: CREACIÓN DINÁMICA DE SÍMBOLOS
═══════════════════════════════════════════════════════════════════════════════════

NUNCA crees símbolos dinámicamente a partir de entrada de usuario.

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 20 — Peligro de símbolos dinámicos

```ruby
# PELIGROSO: creación dinámica de símbolos
entrada_usuario = gets.chomp  # Usuario escribe: "user_1"
sym = entrada_usuario.to_sym

# Esto crea un nuevo símbolo que NUNCA será liberado
# Si el usuario puede enviar muchas entradas diferentes,
# los símbolos se acumulan en memoria para siempre

# SEGURO: usa strings para datos de usuario
# Solo usa símbolos para valores fijos y controlados

# Ejemplo de ataque DoS por símbolos:
# def vulnerable
#   params = params_from_user
#   # params.each { |k, v| instance_variable_set(:"@#{k}", v) }
#   # ¡Crea símbolos ilimitados!
# end
```

                    14. SÍMBOLOS EN LA PRÁCTICA
═══════════════════════════════════════════════════════════════════════════════════

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 21 — Enums con símbolos

```ruby
# Simular enums con símbolos
module EstadoPedido
  PENDIENTE  = :pendiente
  CONFIRMADO = :confirmado
  ENVIADO    = :enviado
  ENTREGADO  = :entregado
  CANCELADO  = :cancelado

  VALIDOS = [PENDIENTE, CONFIRMADO, ENVIADO, ENTREGADO, CANCELADO].freeze
end

# Uso
pedido_estado = EstadoPedido::PENDIENTE
puts EstadoPedido::VALIDOS.include?(pedido_estado)  # => true
```

═══════════════════════════════════════════════════════════════════════════════════

Ejemplo 22 — Símbolos como opciones de método

```ruby
def ordenar(array, direccion: :asc, metodo: :length)
  case direccion
  when :asc
    array.sort_by { |e| e.send(metodo) }
  when :desc
    array.sort_by { |e| e.send(metodo) }.reverse
  else
    raise ArgumentError, "Dirección desconocida: #{direccion}"
  end
end

palabras = %w(Ruby Python Java C JavaScript)
puts ordenar(palabras, direccion: :asc, metodo: :length).inspect
# => ["C", "Java", "Ruby", "Python", "JavaScript"]

puts ordenar(palabras, direccion: :desc, metodo: :length).inspect
# => ["JavaScript", "Python", "Ruby", "Java", "C"]
```

En este capítulo has aprendido:

  • Los símbolos son nombres inmutables, eficientes y únicos en memoria.
  • Se crean con :nombre o %s(nombre).
  • Son diferentes de los strings: :hola y :hola son el mismo objeto.
  • Son inmutables: no se pueden modificar una vez creados.
  • Son la opción preferida para claves de hash.
  • to_sym convierte string a símbolo; to_s convierte símbolo a string.
  • Symbol.all_symbols muestra la tabla global de símbolos.
  • &:símbolo es atajo para crear Procs.
  • Nunca crees símbolos dinámicamente de entrada de usuario.

                     FAQ — PREGUNTAS FRECUENTES
═══════════════════════════════════════════════════════════════════════════════════

P: ¿Por qué son más rápidos los símbolos que los strings?
R: Porque la comparación de símbolos es comparación de object_id (enteros),
mientras que los strings se comparan caracter por caracter.

P: ¿Los símbolos se liberan de memoria?
R: No. Una vez creado, un símbolo permanece en la tabla global para siempre.
Por eso es peligroso crearlos dinámicamente.

P: ¿Cuándo debo usar un símbolo en lugar de un string?
R: Usa símbolos para identificadores, claves de hash, nombres de métodos,
y estados. Usa strings para datos que se muestran al usuario o se modifican.

P: ¿Cómo convierto un símbolo a string?
R: Usa to_s: :nombre.to_s => "nombre".

P: ¿Cómo convierto un string a símbolo?
R: Usa to_sym o intern: "nombre".to_sym => :nombre.

P: ¿Qué significa &:upcase en Ruby?
R: Es un atajo para { |x| x.upcase }. Convierte el símbolo en un Proc
que llama al método representado por el símbolo.

P: ¿Qué es Symbol.all_symbols?
R: Es un método de clase que devuelve un array con todos los símbolos
actualmente registrados en la tabla global de símbolos.

P: ¿Puedo usar un símbolo como nombre de método?
R: Sí, internamente los nombres de métodos son símbolos. Puedes usar
send(:método) para invocar métodos dinámicamente.

  │  El método debe:                                                     │
  │    • Si recibe un hash, transformar todas sus claves a símbolos      │
  │    • Si un valor es un hash, transformarlo también recursivamente    │
  │    • Si un valor es un array, transformar cada elemento del array    │
  │    • Devolver el objeto transformado (nuevo, no modificar original)  │
  │                                                                      │
  │  Ejemplo:                                                            │
  │    datos = {                                                         │
  │      "nombre" => "Ana",                                              │
  │      "direccion" => { "ciudad" => "Lima", "pais" => "Perú" },        │
  │      "hobbies" => [                                                  │
  │        { "nombre" => "Leer", "nivel" => "alto" },                    │
  │        { "nombre" => "Correr", "nivel" => "medio" }                  │
  │      ]                                                               │
  │    }                                                                 │
  │    simbolizar(datos)                                                 │
  │    # => { nombre: "Ana", dirección: { ciudad: "Lima", pais: "Perú" },│
  │    #      hobbies: [{ nombre: "Leer", nivel: "alto" },               │
  │    #                { nombre: "Correr", nivel: "medio" }] }          │
  │                                                                      │
  │  Pista: usa transform_keys para hashes y llamada recursiva           │
  │  para valores que sean arrays o hashes.                              │
  └──────────────────────────────────────────────────────────────────────┘

Los símbolos son una de esas características que hacen que Ruby sea Ruby.
Son pequeños, eficientes, y perfectos para el trabajo que hacen. Una vez
que entiendes la diferencia entre símbolos y strings, empiezas a verlos
por todas partes en el código Ruby.

Recuerda siempre: los símbolos son nombres, no textos.

En el próximo capítulo exploraremos los Rangos: una forma elegante de
representar secuencias y rangos de valores.
