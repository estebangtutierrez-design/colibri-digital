1. OPERADORES ARITMÉTICOS
===============================================================================

Los operadores aritméticos realizan operaciones matemáticas básicas.

--- 1.1 OPERADORES BÁSICOS ---

```ruby
# Suma
puts 10 + 5        # 15
puts 3.14 + 2.86   # 6.0

# Resta
puts 10 - 5        # 5
puts 3.14 - 1.14   # 2.0

# Multiplicación
puts 10 * 5        # 50
puts 3.14 * 2      # 6.28

# División
puts 10 / 3        # 3 (división entera)
puts 10 / 3.0      # 3.3333333333333335
puts 10.0 / 3      # 3.3333333333333335

# Módulo (resto de división)
puts 10 % 3        # 1
puts 10 % 5        # 0
puts 10 % 7        # 3
puts 10.5 % 3      # 1.5

# Potencia
puts 2 ** 3        # 8
puts 2 ** 10       # 1024
puts 5 ** 2        # 25
puts 9 ** 0.5      # 3.0 (raíz cuadrada)
puts 2 ** -1       # 0 (entero) / 0.5 (con float)
puts 2.0 ** -1     # 0.5
```

--- 1.2 PRECEDENCIA DE OPERADORES ---

```ruby
# La precedencia determina el orden de evaluación
puts 2 + 3 * 4      # 14 (multiplicación primero)
puts (2 + 3) * 4    # 20 (paréntesis primero)
puts 10 - 3 + 2     # 9 (izquierda a derecha)
puts 2 ** 3 ** 2    # 512 (de derecha a izquierda!)
puts 2 ** (3 ** 2)  # 512
puts (2 ** 3) ** 2  # 64

# Tabla de precedencia (de mayor a menor)
# 1. **
# 2. ! ~ + (unario)
# 3. * / %
# 4. + - (binario)
# 5. << >>
# 6. &
# 7. | ^
# 8. < > <= >=
# 9. == === != =~ !~
# 10. && (and)
# 11. || (or)
# 12. .. ...
# 13. ?: (ternario)
# 14. = += -= (asignación)
# 15. not
# 16. or and
```

--- 1.3 OPERADORES ARITMÉTICOS EN STRINGS ---

```ruby
# El operador + en strings (concatenación)
puts "Hola " + "Mundo"     # Hola Mundo

# El operador * en strings (repetición)
puts "Ruby! " * 3          # Ruby! Ruby! Ruby!

# El operador << (append)
saludo = "Hola "
saludo << "Mundo"
puts saludo                # Hola Mundo

# Comparación de strings con <=>
puts "a" <=> "b"           # -1 (a < b)
puts "b" <=> "a"           # 1 (b > a)
puts "a" <=> "a"           # 0 (iguales)
```

--- 1.4 LOS OPERADORES SON MÉTODOS ---

```ruby
# En Ruby, los operadores son métodos
puts 2 + 3        # Es equivalente a:
puts 2.+(3)       # 5

puts 10 - 4       # Es equivalente a:
puts 10.-(4)      # 6

puts 3 * 4        # Es equivalente a:
puts 3.*(4)       # 12

puts 10 / 3       # Es equivalente a:
puts 10./(3)      # 3

puts 2 ** 10      # Es equivalente a:
puts 2.**(10)     # 1024

puts 10 % 3       # Es equivalente a:
puts 10.%(3)      # 1

# Todos los objetos tienen estos métodos
puts 42.methods.include?(:+)   # true
puts 42.methods.include?(:**)  # true
puts "abc".methods.include?(:+)  # true
```

--- 1.5 SOBRESCRIBIENDO OPERADORES ---

```ruby
# Puedes definir operadores personalizados en tus clases
class Punto
  attr_reader :x, :y

  def initialize(x, y)
    @x = x
    @y = y
  end

  # Sobrescribir el operador +
  def +(otro)
    Punto.new(@x + otro.x, @y + otro.y)
  end

  # Sobrescribir el operador -
  def -(otro)
    Punto.new(@x - otro.x, @y - otro.y)
  end

  # Sobrescribir el operador *
  def *(escalar)
    Punto.new(@x * escalar, @y * escalar)
  end

  # Sobrescribir el operador ==
  def ==(otro)
    @x == otro.x && @y == otro.y
  end

  def to_s
    "(#{@x}, #{@y})"
  end
end

p1 = Punto.new(3, 4)
p2 = Punto.new(1, 2)

puts p1 + p2    # (4, 6)
puts p1 - p2    # (2, 2)
puts p1 * 2     # (6, 8)
puts p1 == Punto.new(3, 4)  # true
```

===============================================================================
2. OPERADORES DE COMPARACIÓN
===============================================================================

Los operadores de comparación devuelven true o false.

--- 2.1 OPERADORES BÁSICOS ---

```ruby
# Igualdad
puts 5 == 5        # true
puts 5 == 6        # false
puts "hola" == "hola"  # true
puts 5 == 5.0      # true (conversión implícita)

# Desigualdad
puts 5 != 6        # true
puts 5 != 5        # false
puts "a" != "b"    # true

# Mayor que
puts 10 > 5        # true
puts 5 > 10        # false
puts 5 > 5         # false

# Menor que
puts 5 < 10        # true
puts 10 < 5        # false

# Mayor o igual
puts 5 >= 5        # true
puts 10 >= 5       # true
puts 3 >= 5        # false

# Menor o igual
puts 5 <= 5        # true
puts 3 <= 5        # true
puts 10 <= 5       # false
```

--- 2.2 COMPARADOR DE ESPACIO (<=>) ---

El operador `<=>` (spaceship) devuelve -1, 0, o 1.

```ruby
# Spaceship operator
puts 5 <=> 10      # -1 (menor que)
puts 10 <=> 5      # 1 (mayor que)
puts 5 <=> 5       # 0 (igual)

# Útil para ordenar
puts [5, 3, 1, 4, 2].sort { |a, b| a <=> b }  # [1, 2, 3, 4, 5]
puts [5, 3, 1, 4, 2].sort                     # [1, 2, 3, 4, 5]
puts [5, 3, 1, 4, 2].sort { |a, b| b <=> a }  # [5, 4, 3, 2, 1]

# Implementando <=> en clases personalizadas
class Persona
  include Comparable
  attr_reader :nombre, :edad

  def initialize(nombre, edad)
    @nombre = nombre
    @edad = edad
  end

  def <=>(otra)
    @edad <=> otra.edad
  end

  def to_s
    "#{@nombre} (#{@edad})"
  end
end

personas = [
  Persona.new("Ana", 30),
  Persona.new("Luis", 25),
  Persona.new("Carlos", 35)
]

puts personas.sort.map(&:to_s)  # Luis (25), Ana (30), Carlos (35)
```

--- 2.3 OPERADOR === (CASE EQUALITY) ---

```ruby
# === es usado en case/when
case 5
when 1..10 then "Entre 1 y 10"
when 11..20 then "Entre 11 y 20"
end

# === con clases
puts String === "hola"   # true
puts Integer === 42      # true
puts Array === [1, 2]    # true

# === con rangos
puts (1..10) === 5       # true
puts (1..10) === 15      # false

# === con regex
puts /ruby/ === "me gusta ruby"  # true
puts /python/ === "me gusta ruby"  # false

# === con lambdas
par = ->(n) { n.even? }
puts par === 4    # true (aplicación de lambda)
puts par === 5    # false
```

--- 2.4 OPERADOR =~ (MATCH) ---

```ruby
# =~ para coincidencia de patrones (regex)
puts "Hola Ruby" =~ /Ruby/    # 5 (posición donde encuentra)
puts "Hola Ruby" =~ /Python/  # nil (no encuentra)
puts /Ruby/ =~ "Hola Ruby"    # 5 (también funciona al revés)

# !~ para negación
puts "Hola Ruby" !~ /Python/  # true (no coincide)
puts "Hola Ruby" !~ /Ruby/    # false (sí coincide)

# MatchData
match = "Hola Ruby" =~ /(Ruby)/
puts $&  # "Ruby" (texto completo del match)
puts $1  # "Ruby" (primer grupo capturado)
puts $`  # "Hola " (texto antes del match)
puts $'  # "" (texto después del match)
```

3. OPERADORES DE ASIGNACIÓN
===============================================================================

--- 3.1 ASIGNACIÓN SIMPLE ---

```ruby
# Asignación simple
x = 10
nombre = "Ana"
activo = true
```

--- 3.2 ASIGNACIÓN COMPUESTA ---

```ruby
x = 10

x += 5    # x = x + 5     => 15
x -= 3    # x = x - 3     => 12
x *= 2    # x = x * 2     => 24
x /= 4    # x = x / 4     => 6
x %= 4    # x = x % 4     => 2
x **= 3   # x = x ** 3    => 8

puts x  # 8

# Asignación compuesta con strings
saludo = "Hola"
saludo += " "     # "Hola "
saludo += "Mundo" # "Hola Mundo"
puts saludo

# Asignación compuesta con arrays
arr = [1, 2]
arr += [3, 4]     # [1, 2, 3, 4]
puts arr.inspect
```

--- 3.3 ASIGNACIÓN CONDICIONAL (||= Y &&=) ---

```ruby
# ||= asigna solo si la variable es nil o false
cache = nil
cache ||= "Calculado"
puts cache  # "Calculado"

cache ||= "No se recalcula"
puts cache  # "Calculado" (no cambia)

# &&= asigna solo si la variable es truthy
valor = "original"
valor &&= "modificado"
puts valor  # "modificado"

valor = nil
valor &&= "nunca se asigna"
puts valor  # nil

# Patrón común: memoización
def datos_costosos
  @datos ||= calcular_datos_costosos
end

def calcular_datos_costosos
  sleep(2)  # Simula operación costosa
  [1, 2, 3, 4, 5]
end

puts datos_costosos  # Tarda 2 segundos
puts datos_costosos  # Instantáneo (cacheado)
```

--- 3.4 ASIGNACIÓN ABREVIADA CON << ---

```ruby
# << para strings
texto = "Hola"
texto << " Mundo"
puts texto  # "Hola Mundo"

# << para arrays
numeros = [1, 2]
numeros << 3
numeros << 4 << 5
puts numeros.inspect  # [1, 2, 3, 4, 5]
```

4. OPERADORES LÓGICOS
===============================================================================

Los operadores lógicos combinan expresiones booleanas.

--- 4.1 && (AND) ---

```ruby
# && devuelve true si ambas expresiones son truthy
puts true && true     # true
puts true && false    # false
puts false && true    # false
puts false && false   # false

# && con valores no booleanos
puts "hola" && "mundo"  # "mundo" (último truthy)
puts "hola" && nil      # nil (encuentra falsy)
puts nil && "hola"      # nil (falsy, cortocircuito)

# Cortocircuito: si la primera es false, no evalúa la segunda
def peligrosa
  puts "No debería ejecutarse"
  true
end

resultado = false && peligrosa  # peligrosa nunca se ejecuta
puts resultado  # false

# Uso común de &&
usuario = { nombre: "Ana", admin: true }
if usuario[:nombre] && usuario[:admin]
  puts "Acceso concedido a #{usuario[:nombre]}"
end
```

--- 4.2 || (OR) ---

```ruby
# || devuelve true si al menos una expresión es truthy
puts true || true     # true
puts true || false    # true
puts false || true    # true
puts false || false   # false

# || con valores no booleanos
puts nil || "default"       # "default"
puts false || "alternativo" # "alternativo"
puts "hola" || "mundo"      # "hola" (primer truthy)
puts "hola" || nil          # "hola"

# Cortocircuito: si la primera es true, no evalúa la segunda
def costosa
  puts "Operación costosa"
  true
end

resultado = true || costosa  # costosa nunca se ejecuta
puts resultado  # true

# Uso común de || para valores por defecto
nombre = nil
nombre_mostrar = nombre || "Invitado"
puts nombre_mostrar  # "Invitado"

config = { timeout: nil }
timeout = config[:timeout] || 30  # Valor por defecto
puts timeout  # 30
```

--- 4.3 ! (NOT) ---

```ruby
# ! invierte el valor de verdad
puts !true        # false
puts !false       # true
puts !nil         # true
puts !"hola"      # false
puts !42          # false

# Doble negación (!!) para obtener valor booleano exacto
puts !!true       # true
puts !!false      # false
puts !!nil        # false
puts !!0          # true (0 es truthy en Ruby)
puts !!""         # true ("" es truthy en Ruby)
puts !!"hola"     # true

# not - palabra reservada (menor precedencia)
puts not true     # false
puts not false    # true
```

--- 4.4 AND, OR, NOT (PALABRAS RESERVADAS) ---

Ruby tiene versiones en palabras de los operadores lógicos, con MENOR
precedencia.

```ruby
# Diferencia de precedencia
# && > || > and > or

# Con && y || (mayor precedencia)
resultado1 = true || false && false
puts resultado1  # true (evalúa como true || (false && false))

# Con or (menor precedencia)
resultado2 = true || false and false
puts resultado2  # false? No, es true (evalúa como (true) and false, da false)
# Espera: en realidad es (true || false) and false => true and false => false

# Convención: usa && y || para lógica, and/or para control de flujo
# Uso común de and/or:
file = File.open("archivo.txt", "w") and file.write("Hola")
# Abre el archivo y escribe si se pudo abrir

# open_result = File.open("archivo.txt", "w") or raise "No se pudo abrir"
```

5. OPERADORES DE RANGO
===============================================================================

Ruby tiene dos operadores de rango para crear secuencias.

--- 5.1 RANGO INCLUSIVO (..) ---

```ruby
# Rango inclusivo: incluye el último elemento
rango = 1..5
puts rango.to_a  # [1, 2, 3, 4, 5]
puts rango.include?(5)  # true

# Rangos con letras
puts ("a".."e").to_a  # ["a", "b", "c", "d", "e"]

# Iterando sobre rangos
(1..5).each { |i| print "#{i} " }  # 1 2 3 4 5

# Rangos en case/when
nota = 85
case nota
when 90..100 then "Excelente"
when 80...90 then "Muy bien"
when 70...80 then "Bien"
when 60...70 then "Suficiente"
else "Reprobado"
end
```

--- 5.2 RANGO EXCLUSIVO (...) ---

```ruby
# Rango exclusivo: excluye el último elemento
rango = 1...5
puts rango.to_a  # [1, 2, 3, 4]

# Diferencia práctica
puts (1..10).to_a.length   # 10 (incluye 10)
puts (1...10).to_a.length  # 9 (excluye 10)

# Rangos en arrays
array = [0, 1, 2, 3, 4, 5]
puts array[1..3].inspect   # [1, 2, 3]
puts array[1...3].inspect  # [1, 2]
puts array[3..].inspect    # [3, 4, 5] (Ruby 2.6+)
```

--- 5.3 RANGOS INFINITOS ---

```ruby
# Rangos sin fin (beginless range)
mayores_a_3 = 3..
puts mayores_a_3.include?(5)    # true
puts mayores_a_3.include?(1)    # false

# Rangos sin principio (endless range)
menores_a_5 = ..5
puts menores_a_5.include?(3)    # true
puts menores_a_5.include?(7)    # false

# Usos prácticos
numeros = [1, 2, 3, 4, 5, 6]
puts numeros[3..].inspect   # [4, 5, 6]
puts numeros[..2].inspect   # [1, 2, 3]

# Rangos infinitos en case
puntuacion = 95
case puntuacion
when 90.. then "Sobre 90"
when 80..90 then "Entre 80 y 90"
else "Menos de 80"
end
```

--- 5.4 RANGOS CON FECHAS ---

```ruby
require 'date'

# Rangos con fechas
rango_fechas = Date.new(2024, 1, 1)..Date.new(2024, 1, 7)
rango_fechas.each { |fecha| puts fecha }
# 2024-01-01
# 2024-01-02
# ...
# 2024-01-07

# Verificar pertenencia
fecha = Date.new(2024, 1, 15)
puts rango_fechas.include?(fecha)  # false
```

6. OPERADOR DEFINED?
===============================================================================

`defined?` verifica si una expresión está definida.

--- 6.1 USO BÁSICO ---

```ruby
# defined? devuelve nil o un string describiendo la expresión

# Variables
x = 10
puts defined? x      # "local-variable"
puts defined? y      # nil (no definida)

# Constantes
PI = 3.14159
puts defined? PI     # "constant"

# Métodos
puts defined? puts   # "method"
puts defined? inexistente  # nil

# Asignaciones
puts defined? x = 5  # "assignment"

# Expresiones
puts defined? 2 + 2  # "expression"

# Self
puts defined? self   # "self"

# nil
puts defined? nil    # "nil"
puts defined? true   # "true"
puts defined? false  # "false"

# Super
puts defined? super  # nil (si no hay superclase con ese método)
```

--- 6.2 USO PRÁCTICO ---

```ruby
# Verificar si un método existe antes de llamarlo
if defined? some_undefined_method
  some_undefined_method
else
  puts "Método no disponible"
end

# Verificar constantes de módulos
module Config
  TIMEOUT = 30
end

if defined? Config::TIMEOUT
  puts "Timeout: #{Config::TIMEOUT}"
else
  puts "Timeout no configurado"
end

# Asignación condicional segura
x = defined?(y) ? y : "valor por defecto"
puts x  # "valor por defecto"

# Verificar variables de instancia
@nombre = "Ana"
if defined? @nombre
  puts @nombre.upcase
end
```

7. OPERADORES ESPECIALES
===============================================================================

--- 7.1 OPERADOR TERNARIO (? :) ---

```ruby
# Ternario: condición ? valor_si_true : valor_si_false
edad = 20
categoria = edad >= 18 ? "Adulto" : "Menor"
puts categoria  # "Adulto"

# Anidación (no recomendada)
nota = 85
resultado = nota >= 90 ? "A" : nota >= 80 ? "B" : nota >= 70 ? "C" : "D"
puts resultado  # "B"

# Mejor usar if/else para lógica compleja
# El ternario es ideal para asignaciones simples
```

--- 7.2 OPERADOR DE RESOLUCIÓN (::) ---

```ruby
# :: para acceder a constantes, clases y módulos
puts Math::PI       # 3.141592653589793
puts Time::HTTP_DATE  # Constante de Time

# Clases anidadas
module A
  class B
    C = 42
  end
end

puts A::B::C  # 42

# Llamar a métodos de clase
puts Array.new(3, "ruby")
```

--- 7.3 OPERADOR SPLAT (*) ---

```ruby
# Splat: expandir arrays en argumentos
def sumar(a, b, c)
  a + b + c
end

numeros = [1, 2, 3]
puts sumar(*numeros)  # 6

# Splat en asignación
head, *tail = [1, 2, 3, 4, 5]
puts head.inspect  # 1
puts tail.inspect  # [2, 3, 4, 5]

# Double splat (**) para hashes
def metodo_hash(a:, b:)
  "#{a} #{b}"
end

datos = { a: "Hola", b: "Mundo" }
puts metodo_hash(**datos)  # "Hola Mundo"
```

--- 7.4 OPERADOR &. (SAFE NAVIGATION) ---

```ruby
# &. - safe navigation operator (Ruby 2.3+)
# Evita errores cuando un objeto es nil

persona = nil
puts persona&.nombre   # nil (no lanza error)
puts persona&.nombre&.upcase  # nil

# Sin &., esto lanzaría NoMethodError:
# puts persona.nombre

# Con objetos anidados
datos = { usuario: { perfil: { nombre: "Ana" } } }
puts datos[:usuario]&.dig(:perfil)&.dig(:nombre)  # "Ana"

datos_invalidos = { usuario: nil }
puts datos_invalidos[:usuario]&.dig(:perfil)  # nil

# Asignación condicional con &.
persona = Persona.new("Ana")
persona&.nombre = "Luis"  # Asigna si persona no es nil
```

--- 7.5 OPERADOR DE ASOCIACIÓN DE RANGO FLIP-FLOP ---

```ruby
# El operador flip-flop (.. en condición) es único de Ruby
# Se activa cuando la primera condición es true y se desactiva
# cuando la segunda condición es true

(1..10).each do |n|
  puts n if n == 3..n == 7
end
# Imprime: 3, 4, 5, 6, 7

# Útil para procesar secciones de archivos
File.readlines("archivo.txt").each do |linea|
  puts linea if linea =~ /INICIO/..linea =~ /FIN/
end
```

8. PRECEDENCIA COMPLETA
==============================================================================---

--- 8.1 TABLA DE PRECEDENCIA (DE MAYOR A MENOR) ---

```
Nivel  Operadores                 Asociatividad
────────────────────────────────────────────────────
1       ! ~ + (unario)            Derecha
2       **                         Derecha
3       - (unario)                 Derecha
4       * / %                      Izquierda
5       + - (binario)              Izquierda
6       << >>                      Izquierda
7       &                          Izquierda
8       | ^                        Izquierda
9       < > <= >=                  Izquierda
10      == === != =~ !~ <=>        Izquierda
11      &&                         Izquierda
12      ||                         Izquierda
13      .. ...                     Izquierda
14      ?: (ternario)              Derecha
15      = += -= etc.               Derecha
16      not                        Derecha
17      and or                     Izquierda
```

--- 8.2 EJEMPLOS DE PRECEDENCIA ---

```ruby
# Ejemplos prácticos
puts 2 + 3 * 4       # 14 (no 20)
puts (2 + 3) * 4     # 20

puts 2 ** 3 ** 2     # 512 (derecha a izquierda)
puts (2 ** 3) ** 2   # 64

puts -5 ** 2         # -25 ((-5) ** 2 sería 25)
puts (-5) ** 2       # 25

# Operadores lógicos
puts true or false && false  # true (or tiene baja precedencia)
puts (true or false) && false  # false

# Asignación en condicional
```

```ruby
# calculadora_avanzada.rb - Usando todos los operadores
class CalculadoraAvanzada
  attr_reader :historial

  def initialize
    @historial = []
  end

  def operar(a, b, operacion)
    resultado = case operacion
                when :+ then a + b
                when :- then a - b
                when :* then a * b
                when :/ then dividir(a, b)
                when :** then a ** b
                when :% then a % b
                else nil
                end

    registrar("#{a} #{operacion} #{b} = #{resultado}") if resultado
    resultado
  end

  def comparar(a, b)
    {
      igual: a == b,
      diferente: a != b,
      mayor: a > b,
      menor: a < b,
      mayor_igual: a >= b,
      menor_igual: a <= b,
      spaceship: a <=> b
    }
  end

  def rango(valor, inicio, fin)
    inclusivo = (inicio..fin).include?(valor)
    exclusivo = (inicio...fin).include?(valor)
    { inclusivo: inclusivo, exclusivo: exclusivo }
  end

  def evaluar_logica(a, b)
    {
      "a && b": a && b,
      "a || b": a || b,
      "!a": !a,
      "!b": !b
    }
  end

  def mostrar_historial
    @historial.each_with_index { |h, i| puts "#{i + 1}. #{h}" }
  end

  private

  def dividir(a, b)
    if b == 0
      "Error: División por cero"
    else
      a / b.to_f
    end
  end

  def registrar(operacion)
    @historial << operacion
  end
end

calc = CalculadoraAvanzada.new
puts "Suma: #{calc.operar(10, 5, :+)}"
puts "Resta: #{calc.operar(10, 5, :-)}"
puts "Multiplicación: #{calc.operar(10, 5, :*)}"
puts "División: #{calc.operar(10, 5, :/)}"
puts "Potencia: #{calc.operar(2, 10, :**)}"
puts "Módulo: #{calc.operar(10, 3, :%)}"
puts ""
puts "Comparación 10 vs 5:"
calc.comparar(10, 5).each { |k, v| puts "  #{k}: #{v}" }
puts ""
puts "Historial de operaciones:"
calc.mostrar_historial
```

- Los operadores aritméticos son: +, -, *, /, %, **
- Los operadores de comparación son: ==, !=, >, <, >=, <=, <=>, ===, =~
- Los operadores de asignación son: =, +=, -=, *=, /=, %=, **=, ||=, &&=

- En Ruby, los operadores son métodos y se pueden sobrescribir

- :: es el operador de resolución de ámbito
- * (splat) expande arrays, ** (double splat) expande hashes
- &. (safe navigation) evita errores con nil
- <=> (spaceship) devuelve -1, 0, 1
- La precedencia determina el orden de evaluación de las expresiones
- Usa paréntesis para hacer la precedencia explícita y legible
- ||= es el operador de asignación condicional (memoización)
- Los operadores lógicos tienen versiones como métodos: &, |, ^, ~

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué 10 / 3 da 3 y no 3.333?**
Porque ambos operandos son Integer, Ruby hace división entera. Si al menos uno
es Float, Ruby hace división decimal.

❓ **¿Qué significa "cortocircuito" en operadores lógicos?**
Significa que si el resultado se puede determinar solo con el primer operando,
el segundo no se evalúa. En `false && algo`, `algo` nunca se evalúa.

❓ **¿Cuál es la diferencia entre and y &&?**
and tiene menor precedencia que &&. Usa && para lógica normal y and para
control de flujo estilo "haz esto y luego esto otro".

❓ **¿Puedo sobrescribir cualquier operador?**
Puedes sobrescribir la mayoría, pero no todos. Los operadores de asignación
(=, +=, etc.) y operadores lógicos (&&, ||) no se pueden sobrescribir
directamente.

❓ **¿Para qué sirve el operador <=>?**
Devuelve -1, 0, o 1. Es útil para ordenar colecciones. Si incluyes el módulo
Comparable en tu clase, solo necesitas implementar <=>.

❓ **¿Qué hace defined??**
Verifica si una expresión (variable, método, constante) está definida.
Devuelve un string descriptivo o nil si no está definida.

❓ **¿Cómo puedo evitar errores con nil al encadenar métodos?**
Usa el operador &. (safe navigation): `objeto&.metodo&.otro_metodo`.

❓ **¿Por qué 2 ** 3 ** 2 es 512 y no 64?**
Porque ** tiene asociatividad derecha: se evalúa como 2 ** (3 ** 2) = 2 ** 9 = 512.
