1. PREPARACIÓN DEL ENTORNO
===============================================================================

Antes de instalar Ruby, es importante entender qué herramientas necesitamos
y para qué sirve cada una.

--- 1.1 HERRAMIENTAS ESENCIALES ---

| Herramienta | Propósito |
|------------|-----------|
| Ruby | El lenguaje de programación en sí (intérprete MRI) |
| RVM/rbenv | Gestor de versiones de Ruby (múltiples versiones) |
| gem | Sistema de paquetes de Ruby (RubyGems) |
| bundler | Gestor de dependencias para proyectos Ruby |
| irb | Intérprete interactivo de Ruby |
| VS Code | Editor de código recomendado |

```ruby
# Verificación de herramientas (código conceptual)
module HerramientasRuby
  def self.herramientas
    {
      ruby: "Intérprete principal (MRI - Matz's Ruby Interpreter)",
      rvm: "Ruby Version Manager (gestor de versiones)",
      gem: "RubyGems - Sistema de paquetes",
      bundler: "Gestor de dependencias",
      irb: "Interactive Ruby Shell",
      vs_code: "Visual Studio Code (editor recomendado)"
    }
  end

  def self.mostrar_herramientas
    herramientas.each { |k, v| puts "  #{k} - #{v}" }
  end
end

HerramientasRuby.mostrar_herramientas
```

--- 1.2 REQUISITOS DEL SISTEMA ---

Ruby es compatible con los principales sistemas operativos:

- **Linux** (Ubuntu, Debian, Fedora, Arch, etc.)
- **macOS** (versiones recientes)
- **Windows** (a través de WSL2 recomendado, o instalador nativo)

Requisitos mínimos:
- 512 MB de RAM (1 GB+ recomendado)
- 500 MB de espacio en disco
- Conexión a internet (para descargar gemas)
- Compilador C (gcc/clang) para compilar Ruby desde fuente

```ruby
# Verificación de requisitos (guía)
module RequisitosSistema
  def self.verificar
    requisitos = {
      sistema: RUBY_PLATFORM,
      ram: "512 MB mínimo, 1 GB recomendado",
      disco: "500 MB mínimo",
      internet: "Necesario para gemas",
      compilador: "gcc o clang (para compilar desde fuente)"
    }

    puts "=== REQUISITOS DEL SISTEMA ==="
    requisitos.each do |k, v|
      puts "  #{k.to_s.capitalize}: #{v}"
    end
  end
end

RequisitosSistema.verificar
```

2. INSTALACIÓN EN LINUX
===============================================================================

Linux es el sistema operativo donde Ruby se siente más nativo. La mayoría de
los servidores Ruby corren en Linux.

--- 2.1 INSTALACIÓN CON RVM (RECOMENDADA) ---

RVM (Ruby Version Manager) es la herramienta más popular para instalar y
gestionar versiones de Ruby en Linux y macOS.

```bash
# Paso 1: Instalar dependencias necesarias
# Ubuntu/Debian:
# sudo apt-get update
# sudo apt-get install -y curl g++ gcc autoconf automake bison libc6-dev \
#   libffi-dev libgdbm-dev libncurses5-dev libsqlite3-dev libtool \
#   libyaml-dev make pkg-config sqlite3 zlib1g-dev libgmp-dev \
#   libreadline-dev libssl-dev

# Paso 2: Instalar RVM
# gpg --keyserver keyserver.ubuntu.com --recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3
# \curl -sSL https://get.rvm.io | bash -s stable

# Paso 3: Cargar RVM en la terminal
# source ~/.rvm/scripts/rvm

# Paso 4: Verificar la instalación de RVM
# rvm --version
```

```ruby
# Verificar que RVM está instalado
module RVMCkeck
  def self.instalado?
    system("which rvm") ? "RVM está instalado" : "RVM no está instalado"
  end

  def self.version
    `rvm --version 2>/dev/null`.lines.first || "No disponible"
  end
end

puts "RVM: #{RVMCkeck.instalado?}"
```

--- 2.2 INSTALAR RUBY CON RVM ---

```bash
# Paso 1: Listar versiones disponibles de Ruby
# rvm list known

# Paso 2: Instalar la última versión estable
# rvm install ruby

# Paso 3: Instalar una versión específica
# rvm install ruby-3.3.0

# Paso 4: Usar una versión específica
# rvm use ruby-3.3.0 --default

# Paso 5: Verificar la instalación
# ruby --version
# which ruby

# Paso 6: Instalar gemas básicas
# gem update --system
# gem install bundler
# gem install rails
```

--- 2.3 INSTALACIÓN CON RBENV (ALTERNATIVA) ---

rbenv es una alternativa más ligera a RVM. No carga scripts en cada terminal,
lo que lo hace más rápido.

```bash
# Instalación de rbenv y ruby-build
# git clone https://github.com/rbenv/rbenv.git ~/.rbenv
# echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bashrc
# echo 'eval "$(rbenv init -)"' >> ~/.bashrc
# source ~/.bashrc

# Instalar ruby-build (plugin para instalar versiones)
# git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build

# Instalar Ruby con rbenv
# rbenv install 3.3.0
# rbenv global 3.3.0

# Verificar
# ruby --version
```

```ruby
# Comprobación de instalación
module CheckInstalacion
  def self.verificar_todo
    comandos = ["ruby", "gem", "irb", "bundler"]
    comandos.each do |cmd|
      existe = system("which #{cmd} > /dev/null 2>&1")
      puts "#{cmd}: #{existe ? '✓' : '✗'}"
    end
  end

  def self.version_ruby
    "Ruby #{RUBY_VERSION} (plataforma: #{RUBY_PLATFORM})"
  end
end

puts CheckInstalacion.version_ruby
CheckInstalacion.verificar_todo
```

--- 2.4 INSTALACIÓN DIRECTA CON APT (UBUNTU/DEBIAN) ---

Si prefieres no usar un gestor de versiones, puedes instalar Ruby directamente
desde los repositorios del sistema. No recomendado porque obtienes versiones
antiguas.

```bash
# Instalación directa (NO RECOMENDADA - versiones antiguas)
# sudo apt-get install ruby-full

# Para obtener versiones más recientes, usa el PPA de Brightbox
# sudo apt-get install software-properties-common
# sudo apt-add-repository ppa:brightbox/ruby-ng
# sudo apt-get update
# sudo apt-get install ruby3.3 ruby3.3-dev
```

===============================================================================

macOS incluye una versión de Ruby preinstalada (generalmente antigua), pero

--- 3.1 INSTALACIÓN CON HOMEBREW ---

```bash
# Paso 3: Agregar al PATH
# echo 'export PATH="/usr/local/opt/ruby/bin:$PATH"' >> ~/.zshrc

# Paso 4: Verificar
# ruby --version
```

--- 3.2 INSTALACIÓN CON RVM EN MACOS ---

El proceso es similar a Linux:

```bash
# Instalar dependencias (Xcode Command Line Tools)
# xcode-select --install

# Instalar RVM
# gpg --keyserver keyserver.ubuntu.com --recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3
# \curl -sSL https://get.rvm.io | bash -s stable

# Instalar Ruby
# rvm install ruby
# rvm use ruby --default

# Verificar
# ruby --version
```

```ruby
# Código para verificar macOS
module MacOSCheck
  def self.es_macos?
    RUBY_PLATFORM.include?("darwin")
  end

  def self.informacion
    if es_macos?
      puts "Estás en macOS. Ruby funciona excelente aquí."
      puts "Ruby preinstalado: #{`/usr/bin/ruby --version`.strip}"
      puts "Ruby actual: #{RUBY_VERSION}"
    else
      puts "No estás en macOS"
    end
  end
end

MacOSCheck.informacion
```

4. INSTALACIÓN EN WINDOWS
===============================================================================

Windows tiene varias opciones para ejecutar Ruby. La más recomendada es usar
WSL2 (Windows Subsystem for Linux).

--- 4.1 OPCIÓN 1: WSL2 (RECOMENDADA) ---

WSL2 te permite ejecutar Linux dentro de Windows, dándote el entorno Ruby más
compatible y probado.

```bash
# Paso 1: Instalar WSL2
# Abrir PowerShell como Administrador y ejecutar:
# wsl --install

# Paso 2: Instalar una distribución Linux (Ubuntu recomendada)
# wsl --install -d Ubuntu

# Paso 3: Abrir Ubuntu desde el menú de inicio
# Paso 4: Dentro de Ubuntu, instalar RVM y Ruby (ver sección de Linux)
# sudo apt-get update
# sudo apt-get install -y curl g++ gcc autoconf automake bison
# \curl -sSL https://get.rvm.io | bash -s stable
# rvm install ruby
# rvm use ruby --default

# Paso 5: Verificar
# ruby --version
```

--- 4.2 OPCIÓN 2: RUBYINSTALLER (NATIVO) ---

Si prefieres no usar WSL, RubyInstaller es la opción nativa para Windows.

```bash
# 1. Descargar RubyInstaller desde https://rubyinstaller.org/
# 2. Ejecutar el instalador (elegir versión estable con DevKit)
# 3. Durante la instalación, marcar "Add Ruby executables to PATH"
# 4. Al final, ejecutar 'ridk install' para instalar MSYS2 DevKit
# 5. Abrir una nueva terminal y verificar:
# ruby --version
# gem --version
```

```ruby
# Verificación de Windows
module WindowsCheck
  def self.es_windows?
    RUBY_PLATFORM.include?("mingw") || RUBY_PLATFORM.include?("mswin")
  end

  def self.consejo_wsl
    if es_windows?
      puts "Estás en Windows. Recomiendo usar WSL2 para Ruby."
      puts "El desarrollo web con Rails funciona mejor en entornos Unix."
    end
  end
end

WindowsCheck.consejo_wsl
```

```ruby
# Verificación programática
module VerificacionFinal
  def self.ejecutar
    puts "=== VERIFICACIÓN DE INSTALACIÓN ==="
    puts ""

    checks = {
      "Ruby versión" => RUBY_VERSION,
      "Ruby plataforma" => RUBY_PLATFORM,
      "Ruby patchlevel" => RUBY_PATCHLEVEL.to_s,
      "Ruby release date" => RUBY_RELEASE_DATE,
      "Ruby engine" => RUBY_ENGINE,
      "Ruby description" => RUBY_DESCRIPTION
    }

    checks.each do |name, value|
      puts "  #{name}: #{value}"
    end

    puts ""
    puts "✓ Ruby está correctamente instalado"

    # Probar funcionalidad básica
    puts "  Prueba de código: #{2 + 2 == 4 ? '✓' : '✗'}"
    puts "  Prueba de objetos: #{'hola'.upcase == 'HOLA' ? '✓' : '✗'}"
    puts "  Prueba de bloques: #{[1,2,3].map { |n| n * 2 } == [2,4,6] ? '✓' : '✗'}"
  end
end

VerificacionFinal.ejecutar
```

--- 5.2 PRIMER PROGRAMA DE PRUEBA ---

Crea un archivo llamado `verificar.rb`:

```ruby
# verificar.rb - Programa para verificar la instalación
puts "╔════════════════════════════════════╗"

  puts "  ✓ 10 * 5 = #{10 * 5}"

def prueba_strings
  puts "  ✓ 'ruby'.upcase = #{'ruby'.upcase}"
  puts "  ✓ 'RUBY'.downcase = #{'RUBY'.downcase}"
  puts "  ✓ ' ruby '.strip = '#{' ruby '.strip}'"
end

def prueba_arrays
  puts "  ✓ [1,2,3].length = #{[1,2,3].length}"
  puts "  ✓ [1,2,3].reverse = #{[1,2,3].reverse}"

puts "Pruebas de strings:"
```

Estas extensiones transforman VS Code en un IDE completo para Ruby:

| Extensión | ID | Propósito |
|-----------|-----|-----------|
| Ruby | `shopify.ruby-extensions-pack` | Pack completo de Ruby |
| Ruby LSP | `Shopify.ruby-lsp` | Servidor de lenguaje Ruby |
| Ruby Solargraph | `castwide.solargraph` | Autocompletado avanzado |
| Endwise | `kaiwood.endwise` | Cierra automáticamente end |
| VSCode Ruby | `staff0rd. ruby` | Sintaxis y snippets |
| Ruby on Rails | `hridoy.rails-snippets` | Snippets para Rails |
| ERB Support | `aki77.erb-support` | Soporte para ERB |
| RuboCop | `shanoor.vscode-rubocop` | Linter y formateador |

```ruby
# Las extensiones en código (concepto)
module ExtensionesVSCode
  def self.esenciales
    [
      { nombre: "Ruby LSP", funcion: "Autocompletado y análisis" },
      { nombre: "Endwise", funcion: "Cierra end automáticamente" },
      { nombre: "RuboCop", funcion: "Linter y formateador" },
      { nombre: "Solargraph", funcion: "Documentación y navegación" }
    ]
  end

  def self.mostrar
    puts "Extensiones esenciales para VS Code:"
    esenciales.each do |ext|
      puts "  - #{ext[:nombre]}: #{ext[:funcion]}"
    end
    puts ""
    puts "Instálalas desde el marketplace de VS Code"
    puts "o con: code --install-extension <id-ext>"
  end
end

ExtensionesVSCode.mostrar
```

8. SOLUCIÓN DE PROBLEMAS COMUNES
===============================================================================

--- 8.1 PROBLEMAS DE INSTALACIÓN ---

```ruby
module ProblemasComunes
  def self.soluciones
    {
      "ruby: command not found" => {
        causa: "Ruby no está en el PATH",
        solucion: "Revisa la instalación. source ~/.rvm/scripts/rvm"
      },
      "Error al compilar Ruby" => {
        causa: "Faltan dependencias de compilación",
        solucion: "Instala build-essential, libssl-dev, libreadline-dev"
      },
      "Permission denied al instalar gemas" => {
        causa: "Gemas instaladas como root",
        solucion: "Usa RVM/rbenv para entornos de usuario"
      },
      "Gemas no encontradas" => {
        causa: "Gemfile.lock desactualizado",
        solucion: "Ejecuta bundle install o bundle update"
      },
      "Versión incorrecta de Ruby" => {
        causa: "Múltiples versiones instaladas",
        solucion: "rvm use 3.3.0 --default"
      },
      "Encoding problems" => {
        causa: "Configuración regional incorrecta",
        solucion: "export LANG=en_US.UTF-8"
      },
      "Segmentation fault" => {
        causa: "Gema nativa mal compilada",
        solucion: "Reinstalar la gema: gem uninstall && gem install"
      }
    }
  end

  def self.mostrar_ayuda
    puts "=== SOLUCIÓN DE PROBLEMAS COMUNES ==="
    ProblemasComunes.soluciones.each do |problema, info|
      puts ""
```

```ruby
  def self.ejecutar

    puts "  IRB instalado: #{pruebas.last ? '✓' : '✗'}"

      eval("2 + 2")
```

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué versión de Ruby debo instalar?**
La más reciente estable (Ruby 3.3+). Las versiones impares (2.1, 2.3, 2.5) son
versiones de desarrollo; las pares (2.0, 2.2, 2.4) son estables. Desde Ruby 3.0,
el versionado es semántico.

❓ **¿RVM o rbenv? ¿Cuál es mejor?**
Ambos son excelentes. RVM tiene más funcionalidades integradas. rbenv es más
ligero y minimalista. Para principiantes, RVM es más fácil de usar.

❓ **¿Por qué no usar la versión del sistema?**
Las versiones del sistema (apt, brew) suelen estar desactualizadas. RVM/rbenv
te permiten tener versiones modernas y múltiples versiones simultáneas.

❓ **¿Cómo cambio entre versiones de Ruby?**
Con RVM: `rvm use 3.3.0`. Con rbenv: `rbenv global 3.3.0`. Puedes tener
versiones diferentes por proyecto.

❓ **¿Necesito instalar Rails ahora?**
No. Primero aprende Ruby. Rails se instalará cuando lo necesites. Es mejor
no mezclar el aprendizaje de Ruby y Rails al mismo tiempo.

❓ **¿Qué hago si la instalación falla?**
Revisa los logs de error. Generalmente falta alguna dependencia (build-essential,
libssl-dev, etc.). Consulta la sección de solución de problemas de este
capítulo o busca en Stack Overflow.

❓ **¿Puedo usar Ruby online sin instalarlo?**
Sí. Prueba `try.ruby-lang.org` o `replit.com`. Pero para aprendizaje serio,
necesitas un entorno local instalado.

❓ **¿Debo usar WSL2 en Windows o RubyInstaller?**
WSL2 es muy recomendado porque te da un entorno Linux completo. RubyInstaller
es bueno si prefieres mantenerte completamente en Windows.

❓ **¿Cómo sé que Ruby está correctamente instalado?**
Ejecuta `ruby --version`, `gem --version`, `irb --version`. Si los tres
responden, estás listo.

❓ **¿Qué hago si VS Code no reconoce Ruby?**
Asegúrate de tener las extensiones Ruby LSP y Solargraph instaladas. Verifica
que la extensión está usando el Ruby correcto (RVM/rbenv).
