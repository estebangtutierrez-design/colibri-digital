1. EL ORIGEN: YUKIHIRO "MATZ" MATSUMOTO
===============================================================================

Yukihiro Matsumoto, conocido mundialmente como "Matz", nació el 14 de abril de
1965 en Osaka, Japón. Desde joven mostró un gran interés por la programación,
aprendiendo lenguajes como Perl, Python, Smalltalk, Eiffel, Ada, y Lisp.

Matz se graduó en Ciencias de la Computación en la Universidad de Tsukuba.
Trabajó como programador y arquitecto de software, y fue durante este tiempo
que comenzó a concebir la idea de un nuevo lenguaje de programación.

```ruby
# Un homenaje a Matz en código
module RubyHistory
  CREADOR = "Yukihiro 'Matz' Matsumoto"
  ANIO_CREACION = 1993
  PAIS = "Japón"

  def self.filosofia
    {
      principio: "Principio de Menor Sorpresa",
      objetivo: "Hacer felices a los programadores",
      inspiracion: ["Perl", "Smalltalk", "Python", "Lisp", "Eiffel", "Ada"]
    }
  end

  def self.mostrar_info
    info = <<~INFO
      Creador: #{CREADOR}
      Año de creación: #{ANIO_CREACION}
      País: #{PAIS}
      Principio: #{filosofia[:principio]}
    INFO
    puts info
  end
end

RubyHistory.mostrar_info
```

La motivación de Matz para crear Ruby fue su insatisfacción con los lenguajes
existentes. Encontró que Perl era poderoso pero demasiado "sucio" en su
sintaxis. Python era elegante pero no lo suficientemente orientado a objetos
para su gusto. Smalltalk era puramente orientado a objetos pero demasiado
radical. Matz quería un lenguaje que combinara lo mejor de todos estos mundos.

Matz dijo una vez: "Había muchos lenguajes que me gustaban, pero ninguno que
realmente amara. Así que decidí crear mi propio lenguaje." Ruby nació de la
frustración y el deseo de crear algo mejor.

```ruby
# La inspiración de Ruby en código
module InspiracionRuby
  # De Perl: expresividad y utilidad
  def self.desde_perl
    "Expresividad, regex integradas, variables especiales"
  end

  # De Smalltalk: orientación a objetos pura
  def self.desde_smalltalk
    "Todo es un objeto, mensajes entre objetos, bloques"
  end

  # De Python: legibilidad y facilidad de uso
  def self.desde_python
    "Código limpio, legible, y fácil de aprender"
  end

  # De Lisp: poder expresivo
  def self.desde_lisp
    "Bloques como funciones anónimas, metaprogramación"
  end

  # De Eiffel: diseño por contrato
  def self.desde_eiffel
    "Diseño elegante de clases y herencia"
  end
end
```

2. EL NACIMIENTO DE RUBY (1993 - 1995)
===============================================================================

En 1993, durante una conversación con un compañero de trabajo, Matz mencionó
que estaba buscando un nombre para su nuevo lenguaje. Inicialmente pensó en
llamarlo "Coral", pero luego se decidió por "Ruby" (Rubí en inglés).

¿Por qué Ruby? La piedra preciosa rubí está asociada con el fuego, la pasión
y la energía. Además, era un guiño al lenguaje Perl (nacimiento de perla),
ya que el rubí es una piedra preciosa más valiosa. El nombre refleja la
ambición de Matz: crear algo más valioso que Perl.

```ruby
# El nombre del lenguaje
module NombreRuby
  def self.significado
    {
      nombre: "Ruby",
      tipo: "Lenguaje de programación",
      significado: "Rubí (piedra preciosa)",
      inspiracion: "Perl -> Ruby (una gema más valiosa)",
      año_nacimiento: 1993,
      primera_publicacion: 1995
    }
  end

  def self.historia_nombre
    <<~HISTORIA
      En 1993, Matz buscaba un nombre.
      'Coral' fue una opción temprana.
      'Ruby' fue la elegida: una gema más valiosa que Perl.
      El fuego del rubí representa la pasión por programar.
    HISTORIA
  end
end

puts NombreRuby.historia_nombre
```

El desarrollo de Ruby comenzó en serio en 1993. Matz trabajaba en el lenguaje
durante su tiempo libre, escribiendo el intérprete en C. Su objetivo era crear
un lenguaje que fuera más poderoso que Perl y más orientado a objetos que
Python.

El 21 de diciembre de 1995, Matz lanzó la primera versión pública de Ruby
(Ruby 0.95) en el grupo de noticias japonés `fj.comp.lang`. Esto marcó el
nacimiento oficial del lenguaje.

```ruby
# Timeline temprano de Ruby
module TimelineRuby
  EVENTOS = [
    { año: 1993, evento: "Matz comienza a desarrollar Ruby" },
    { año: 1994, evento: "Primera implementación básica del intérprete" },
    { año: 1995, evento: "Ruby 0.95: primera versión pública (21 Dic)" },
    { año: 1996, evento: "Ruby 1.0: primera versión estable" },
  ]

  def self.linea_tiempo
    EVENTOS.each do |evento|
      puts "  #{evento[:año]} - #{evento[:evento]}"
    end
  end

puts "Timeline inicial de Ruby:"
TimelineRuby.linea_tiempo
```

--- 2.1 EL PRIMER CÓDIGO RUBY ---

No se conserva el primer programa Ruby que Matz escribió, pero podemos
imaginar que fue algo simple como esto:

```ruby
# El primer programa Ruby (conceptualmente)
puts "Hello, world"

# O quizás una prueba de la orientación a objetos
class Hola
  def initialize(mensaje)
    @mensaje = mensaje
  end

  def mostrar
    puts @mensaje
  end
end

saludo = Hola.new("¡Ruby ha nacido!")
saludo.mostrar
```

--- 2.2 PRIMERA LISTA DE CORREO ---

En 1997 se creó la primera lista de correo de Ruby, permitiendo que la
comunidad comenzara a formarse. Esta lista fue crucial para el desarrollo
temprano de Ruby, ya que Matz recibía sugerencias y reportes de errores
de los primeros usuarios.

```ruby
# Metáfora de la comunidad temprana
class Comunidad
  attr_reader :miembros, :año

  def initialize(nombre, año)
    @nombre = nombre
    @año = año
    @miembros = []
    @mensajes = []
  end

  def agregar_miembro(nombre)
    @miembros << nombre
    puts "#{nombre} se unió a #{@nombre}"
  end

  def publicar_mensaje(autor, contenido)
    @mensajes << { autor: autor, contenido: contenido }
    puts "[#{@nombre}] #{autor}: #{contenido[0..50]}..."
  end
end

comunidad = Comunidad.new("Lista de Correo Ruby", 1997)
comunidad.agregar_miembro("Matz")
comunidad.agregar_miembro("Usuarios pioneros")
comunidad.publicar_mensaje("Matz", "¡Bienvenidos a la primera lista de Ruby!")
```

3. LOS PRIMEROS AÑOS: RUBY 1.0 A 1.6 (1996 - 2002)
===============================================================================

--- 3.1 RUBY 1.0 (1996) ---

Ruby 1.0 fue la primera versión considerada estable y completa. Incluía:

- Sintaxis básica del lenguaje (casi completa)
- Orientación a objetos pura
- Manejo de strings y arrays
- Expresiones regulares integradas
- Módulos y mixins

```ruby
# Características de Ruby 1.0
modulo = Module.new do
  def metodo_de_modulo
    "Un mixin desde Ruby 1.0"
  end
end

class MiClase
  include modulo
end

objeto = MiClase.new
puts objeto.metodo_de_modulo
```

--- 3.2 RUBY 1.6 (2000) ---

Ruby 1.6 trajo mejoras significativas:

- Soporte para Windows (Cygwin)
- Mejoras en la internacionalización
- Nuevos métodos para strings y arrays
- Mejor manejo de excepciones
- La biblioteca estándar comenzó a crecer

En el año 2000 se publicó el primer libro de Ruby en inglés: "Programming
Ruby" (conocido como el libro "Pickaxe" por el pico en la portada), escrito
por David Thomas y Andrew Hunt. Este libro fue fundamental para la
popularización de Ruby fuera de Japón.

4. LA REVOLUCIÓN: RUBY ON RAILS (2004)
===============================================================================

El 2004 marcó un antes y un después en la historia de Ruby. David Heinemeier
Hansson (DHH) lanzó Ruby on Rails, un framework web construido en Ruby que
revolucionó el desarrollo de aplicaciones web.

Rails introdujo conceptos innovadores:
- Convención sobre Configuración (CoC)
- No te repitas (DRY)
- Scaffolding automático
- Migraciones de base de datos
- ActiveRecord (ORM)
- MVC (Modelo-Vista-Controlador)

```ruby
# El impacto de Rails en código
module RubyOnRails
  CREADOR = "David Heinemeier Hansson (DHH)"
  ANIO_LANZAMIENTO = 2004
  EMPRESA = "Basecamp (37signals)"

  def self.filosofia
    {
      coc: "Convention over Configuration",
      dry: "Don't Repeat Yourself",
      mvc: "Model-View-Controller",
      scaffolding: "Generación automática de código",
      orm: "ActiveRecord - Mapeo Objeto-Relacional"
    }
  end

  def self.ejemplo_activo
    puts "Con ActiveRecord (solo 1 línea define la tabla):"
  end
end

# ActiveRecord simplifica el acceso a BD
# class Usuario < ApplicationRecord
#   has_many :publicaciones
#   validates :email, presence: true
# end

# En Rails, crear una tabla es tan simple como:
# rails generate model Usuario nombre:string email:string
```

El éxito de Rails fue explosivo. Sitios como Basecamp, Twitter (inicialmente),
GitHub, Shopify y muchos más fueron construidos con Rails. Esto atrajo a miles
de desarrolladores a Ruby, que de ser un lenguaje de nicho pasó a ser uno de
los más populares del mundo.

```ruby
# Rails vs Desarrollo tradicional
class DesarrolloWeb
  def self.sin_rails
    "Configurar todo desde cero: rutas, BD, sesiones, etc. - SEMANAS"
  end

  def self.con_rails
    "rails new app -> rails generate scaffold -> ¡LISTO! - MINUTOS"
  end

  def self.comparacion
    puts "Sin Rails: #{sin_rails}"
    puts "Con Rails: #{con_rails}"
    puts "Rails te da superpoderes de productividad"
  end
end

DesarrolloWeb.comparacion
```

--- 4.1 EL EFECTO RAILS ---

El "efecto Rails" transformó Ruby de tres maneras fundamentales:

1. **Explosión de popularidad**: Ruby pasó de ser un lenguaje de nicho a
   estar en el top 10 de los lenguajes más populares.
2. **Inversión en la comunidad**: Más desarrolladores significaron más gemas,
   más conferencias, más tutoriales, más documentación.
3. **Madurez del ecosistema**: El éxito de Rails forzó mejoras en Ruby mismo
   (rendimiento, empaquetado, despliegue).

```ruby
# Estadísticas del impacto de Rails
module EfectoRails
  def self.impacto
    {
      nuevas_gemas_2004_2010: "De 100 a +10,000",
      conferencias: "RubyConf, RailsConf, Euruko",
      desarrolladores: "Crecimiento exponencial",
      empresas: "GitHub, Shopify, Basecamp, Airbnb, Twitch",
      libros_publicados: "De 2 a +50 en 5 años"
    }
  end

  def self.mostrar_crecimiento
    impacto = EfectoRails.impacto
    impacto.each do |k, v|
      puts "  #{k.to_s.gsub('_', ' ').capitalize}: #{v}"
    end
  end
end

EfectoRails.mostrar_crecimiento
```

5. RUBY 1.9 Y 2.0 (2007 - 2013)
===============================================================================

--- 5.1 RUBY 1.9 (2007) ---

Ruby 1.9 fue una versión revolucionaria. No era simplemente una actualización,
sino una reescritura significativa del intérprete. Cambios importantes:

- Nuevo motor de ejecución (YARV - Yet Another Ruby VM)
- Mejoras dramáticas en rendimiento
- Nuevas características del lenguaje
- Cambios en la sintaxis (nueva notación para hashes)
- Bloques con variables locales
- Nuevo encoding por defecto (UTF-8)

```ruby
# Ruby 1.9 trajo cambios importantes

# Nueva sintaxis de hashes
antiguo = { :nombre => "Ruby" }  # 1.8 style
nuevo = { nombre: "Ruby" }       # 1.9 style (JSON-like)

# Bloques con variables locales
[1, 2, 3].each do |n|
  puts "Ruby 1.9: #{n}"
end

# Nuevo encoding UTF-8 por defecto
puts "ñandú".length  # 5 (correcto en 1.9, antes daba 7)

# Lambdas con nueva sintaxis
lambda_vieja = lambda { |x| x * 2 }
lambda_nueva = ->(x) { x * 2 }

puts lambda_nueva.call(5)  # 10

# El splat operator en args
def metodo(*args)
  args.join(", ")
end

puts metodo(1, 2, 3)  # "1, 2, 3"
```

--- 5.2 RUBY 2.0 (2013) ---

Ruby 2.0 fue lanzado el 24 de febrero de 2013, exactamente en el 20 aniversario
de Ruby. Fue una versión centrada en rendimiento y nuevas características:

- Keyword arguments (argumentos con nombre)
- lazy enumerators (enumeradores perezosos)
- Module#prepend (para módulos)
- Refinements (refinamientos de clases)
- %i() para arrays de símbolos
- Encoding mejorado
- GC mejorado

```ruby
# Ruby 2.0 - Nuevas características

# Keyword arguments
def saludar(nombre:, saludo: "Hola")
  "#{saludo}, #{nombre}!"
end

puts saludar(nombre: "Ana")               # Hola, Ana!
puts saludar(nombre: "Luis", saludo: "Hey") # Hey, Luis!

# Lazy enumerators
datos = (1..Float::INFINITY).lazy
  .select { |n| n % 2 == 0 }
  .map { |n| n * 2 }
  .first(5)

puts datos.inspect  # [4, 8, 12, 16, 20]

# Module#prepend
module Logger
  def initialize(*args)
    puts "Creando instancia de #{self.class}"
    super
  end
end

class Persona
  prepend Logger

  def initialize(nombre)
    @nombre = nombre
  end
end

p = Persona.new("Ana")  # "Creando instancia de Persona"

# %i() para arrays de símbolos
colores = %i(rojo verde azul)
puts colores.inspect  # [:rojo, :verde, :azul]

# Refinements
module StringExtensions
  refine String do
    def exclamar
      self.upcase + "!"
    end
  end
end

using StringExtensions
puts "hola".exclamar  # "HOLA!"
```

6. LA ERA MODERNA: RUBY 2.1 A 2.7 (2013 - 2019)
===============================================================================

Cada versión de Ruby 2.x trajo mejoras incrementales que mantuvieron el
lenguaje actualizado y competitivo.

--- 6.1 RUBY 2.1 (2013) ---

- Numeración semántica de versiones
- Mejoras en el GC generacional
- Refinements por defecto (experimental)
- def de método con valores por defecto
- Mejoras en BigDecimal

--- 6.2 RUBY 2.2 (2014) ---

- GC simbólico
- Mejoras en el manejo de memoria
- Incremento en el rendimiento de Float
- Bundler incluido por defecto

--- 6.3 RUBY 2.3 (2015) ---

- Operador &. (safe navigation / lonely operator)
- Digest::UUID
- Frozen string literals pragma
- Array#dig y Hash#dig
- DidYouMean (sugerencias de métodos)

```ruby
# Ruby 2.3 - Características destacadas

# Safe navigation operator (&.)
persona = nil
puts persona&.nombre  # nil en lugar de NoMethodError

# Hash#dig
datos = { usuario: { nombre: "Ana", direccion: { ciudad: "Madrid" } } }
puts datos.dig(:usuario, :direccion, :ciudad)  # "Madrid"

# Array#dig
matriz = [[1, [2, 3]], 4]
puts matriz.dig(0, 1, 0)  # 2

# DidYouMean - Cuando escribes mal un método
# "hola".lenght  # Sugiere: Did you mean? length
```

--- 6.4 RUBY 2.4 (2016) ---

- Unificación de Fixnum y Bignum en Integer
- Regexp#match? (más rápido que match)
- Suma de enumerables
- Nuevo método Integer#digits

--- 6.5 RUBY 2.5 (2017) ---

- yield_self (luego renombrado a then)
- rescue anidado en bloques
- Mejoras en ERB
- Coveralls para cobertura de código

--- 6.6 RUBY 2.6 (2018) ---

- JIT experimental (Just-In-Time compiler)
- Bundler integrado completamente
- Keyword arguments y posiciónales pueden mezclarse
- endless ranges (.. y ...)

--- 6.7 RUBY 2.7 (2019) ---

- Pattern Matching (experimental)
- Numbered parameters (numbered block parameters)
- Mejoras en keyword arguments
- GC compactación

```ruby
# Ruby 2.6 y 2.7 - Características destacadas

# Endless ranges (2.6+)
puts [1, 2, 3, 4].slice(2..)  # [3, 4]
puts [1, 2, 3, 4].slice(1...3)  # [2, 3]

# Numbered parameters (2.7+)
[1, 2, 3].each { puts _1 * 2 }  # 2, 4, 6

# yield_self / then (2.5+)
resultado = "hola"
  .then { |s| s.upcase }
  .then { |s| s.reverse }
puts resultado  # "ALOH"

# Pattern Matching experimental (2.7)
case [1, 2, 3]
in [a, b, c]
  puts "a=#{a}, b=#{b}, c=#{c}"
end
```

7. RUBY 3.0: EL FUTURO LLEGÓ (2020)
===============================================================================

Ruby 3.0 fue lanzado el 25 de diciembre de 2020 (Navidad), siguiendo la
tradición de lanzamientos navideños de Ruby. Esta versión cumplió la promesa
de "Ruby 3x3" - ser 3 veces más rápido que Ruby 2.0.

--- 7.1 PRINCIPALES INNOVACIONES DE RUBY 3.0 ---

1. **Ractors**: modelo de concurrencia sin GIL
2. **Fiber Scheduler**: E/S concurrente ligera
3. **Pattern Matching**: maduro (ya no experimental)
4. **Type Profiler**: análisis de tipos estático
5. **Find Pattern**: búsqueda en estructuras
6. **Hash#except**: exclusión de claves
7. **Endless method definition**: métodos en una línea

```ruby
# Ruby 3.0 - Revolución en concurrencia

# Ractors (concurrencia real sin GIL)
ractor = Ractor.new do
  suma = 0
  100.times { |i| suma += i }
  suma
end

puts ractor.take  # 4950

# Múltiples ractors en paralelo
ractors = (1..4).map do |i|
  Ractor.new(i) do |id|
    "Ractor #{id} completado"
  end
end

puts ractors.map(&:take)

# Fiber Scheduler
require 'io/nonblock'

class EchoServer
  def initialize(host, port)
    @host = host
    @port = port
  end
end

# Endless method definition
def cuadrado(x) = x * x
puts cuadrado(5)  # 25

def saludar(nombre) = "¡Hola, #{nombre}!"
puts saludar("Ruby")  # ¡Hola, Ruby!
```

--- 7.2 RUBY 3.1 (2021) ---

- YJIT (Yet Another JIT) - compilador JIT en Rust
- Debug gem incluido por defecto
- Hash#shorthand para valores
- Mejoras en Pattern Matching (array/in hash)
- Prism parser (nuevo parser)

```ruby
# Ruby 3.1 - Mejoras

# Hash shorthand (cuando key y variable tienen el mismo nombre)
nombre = "Ana"
edad = 30
persona = { nombre:, edad: }
puts persona  # {:nombre=>"Ana", :edad=>"30"}

# Pattern Matching avanzado
case { a: 1, b: 2, c: 3 }
in { a:, b:, **rest }
  puts "a=#{a}, b=#{b}, rest=#{rest}"
end
```

--- 7.3 RUBY 3.2 (2022) ---

- YJIT mejorado (producción-ready)
- Syntax Sugaring (mejoras en sintaxis)
- Data.define para objetos valor
- Mejoras en Regexp y Time
- WASI (WebAssembly)

--- 7.4 RUBY 3.3 (2023) ---

- Prism parser como parser por defecto
- YJIT más rápido
- Mejoras en el GC
- LSP nativo (Language Server Protocol)
- RJIT (nuevo compilador JIT en Ruby)

--- 7.5 RUBY 3.4 (2024) ---

- Mejoras continuas en rendimiento
- YJIT aún más rápido
- Nuevas características del lenguaje
- Mejoras en concurrencia
- Deprecaciones y limpieza

```ruby
# Ruby 3.3+ - Estado del arte

# Data.define - Objetos valor inmutables
Persona = Data.define(:nombre, :edad)
ana = Persona.new("Ana", 30)
puts ana.nombre  # Ana
puts ana.edad    # 30

# YJIT para máximo rendimiento
# Ejecuta: ruby --yjit tu_archivo.rb
```

9. LA FILOSOFÍA DETRÁS DE LA HISTORIA
===============================================================================

La historia de Ruby no es solo una cronología de versiones. Es la historia de
una filosofía de diseño consistente que ha guiado el lenguaje durante más de
30 años.

--- 9.1 LOS PRINCIPIOS INMUTABLES ---

A través de todas las versiones, Matz ha mantenido estos principios:

1. **Optimizar para la felicidad del programador**
2. **Principio de Menor Sorpresa**
3. **Naturalidad sobre formalidad**
4. **Orientación a objetos pura**
5. **Flexibilidad y expresividad**

```ruby
# La filosofía de Matz en código
module FilosofiaMatz
  def self.principios
    {
      felicidad: "El programador es lo primero",
      menor_sorpresa: "Ruby hace lo que esperas",
      naturalidad: "El código debe fluir como lenguaje natural",
      poo_pura: "Todo es un objeto, sin excepciones",
      flexibilidad: "Múltiples formas de hacer lo mismo"
    }
  end

  def self.ejemplificar_principio(principio)
    case principio
    when :felicidad
      puts "Código expresivo que es un placer leer"
      puts "5.times { print 'Feliz ' }"
    when :menor_sorpresa
      puts "nil.to_s => '' en lugar de lanzar error"
      puts "''.empty? => true"
    when :naturalidad
      puts "unless, until, if modificador"
      puts "puts 'Hola' if amigo?"
    end
  end
end

FilosofiaMatz.ejemplificar_principio(:felicidad)
```

--- 9.2 LA EVOLUCIÓN SIN PERDER LA ESENCIA ---

Una de las cosas más notables de Ruby es cómo ha evolucionado sin perder su
esencia. Ruby 3.x sigue sintiéndose como Ruby, a pesar de los cambios masivos
en su implementación interna.

```ruby
# Ruby 1995 vs Ruby 2024 - Misma esencia
class Comparacion
  def self.ruby_1995
    <<~CODIGO
      # Habría funcionado en 1995
      def saludar(nombre)
        puts "Hola, #{nombre}"
      end
      saludar("Mundo")
    CODIGO
  end

  def self.ruby_2024
    <<~CODIGO
      # Funciona en 2024 igual que en 1995
      def saludar(nombre)
        puts "Hola, #{nombre}"
      end
      saludar("Mundo")
    CODIGO
  end

  def self.conclusion
    puts "Ruby ha mantenido su sintaxis básica por décadas"
    puts "La compatibilidad hacia atrás es sagrada para Matz"
  end
end
```

Ruby ha influido en muchos lenguajes y tecnologías modernas:

- **Elixir**: tomó los bloques y la filosofía de Ruby para la BEAM

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué Matz creó Ruby?**
Estaba insatisfecho con los lenguajes existentes. Quería un lenguaje que fuera
más poderoso que Perl y más orientado a objetos que Python. Quería un lenguaje
que hiciera felices a los programadores.

❓ **¿Qué significa que Ruby es "una gema más valiosa"?**
Ruby (rubí) es una piedra preciosa más valiosa que Perl (perla). El nombre es
un juego de palabras que refleja la ambición de Matz de crear algo superior.

❓ **¿Por qué las versiones de Ruby se lanzan en Navidad?**
Tradición iniciada por Matz. Ruby 1.0 se lanzó cerca de Navidad de 1996, y
desde entonces muchos lanzamientos mayores han ocurrido el 25 de diciembre.

❓ **¿Qué pasó con Ruby 1.9? ¿Por qué no Ruby 2.0 directo?**
Ruby 1.9 fue un cambio tan grande (YARV, encoding, etc.) que merecía su propio
número de versión. Fue el puente hacia Ruby 2.0.

❓ **¿Cuál fue la versión más popular de Ruby?**
Ruby 1.8 fue la más longeva y popular por muchos años. Ruby 2.7 y 3.0 también
han sido muy adoptadas.

❓ **¿Quién toma las decisiones sobre el futuro de Ruby?**
Matz es el BDFL (Benevolent Dictator for Life), pero el core team de Ruby
(colaboradores clave) discute y propone cambios. Las decisiones se toman por
consenso.

❓ **¿Ruby sigue siendo relevante en 2024+?**
Absolutamente. Ruby sigue evolucionando con nuevas versiones cada año. YJIT ha
mejorado drásticamente el rendimiento. Grandes empresas siguen invirtiendo en
Ruby y Rails.

❓ **¿Qué es Ruby 3x3?**
La promesa de Matz de hacer Ruby 3.0 tres veces más rápido que Ruby 2.0. Se
logró mediante YARV, JIT, y mejoras en el GC.

❓ **¿Rails sigue siendo relevante?**
Sí. Rails 8 ha sido lanzado recientemente con nuevas características. GitHub,
Shopify, GitLab y Basecamp siguen usándolo y mejorándolo.

❓ **¿Cuánto tiempo ha existido Ruby?**
Ruby comenzó en 1993, lo que significa que tiene más de 30 años. Es uno de los
lenguajes dinámicos más maduros y estables.

2. Devolver eventos por década (1990s, 2000s, 2010s, 2020s)
3. Devolver eventos por categoría
4. Encontrar el evento más cercano a un año dado

```ruby
# Esqueleto del desafío
class Evento
  attr_reader :año, :titulo, :descripcion, :categoria

  def initialize(año, titulo, descripcion, categoria)
    @año = año
    @titulo = titulo
    @descripcion = descripcion
    @categoria = categoria
  end

  def to_s
    "[#{@año}] [#{@categoria}] #{@titulo}: #{@descripcion}"
  end
end

class TimelineRuby
  def initialize
    @eventos = []
  end

  def agregar_evento(evento)
    @eventos << evento
  end

  def eventos_por_decada(decada)
    # decada: 1990, 2000, 2010, 2020
    @eventos.select { |e| e.año >= decada && e.año < decada + 10 }
  end

  def eventos_por_categoria(categoria)
    @eventos.select { |e| e.categoria == categoria }
  end

  def evento_mas_cercano(año)
    @eventos.min_by { |e| (e.año - año).abs }
  end

  def generar_reporte
    @eventos.sort_by(&:año).each do |evento|
      puts evento
    end
  end
end
```
