1. ¿QUÉ ES RUBY?
===============================================================================

Ruby es un lenguaje de programación dinámico, interpretado, de alto nivel y
multipropósito. Fue creado en 1995 por Yukihiro "Matz" Matsumoto en Japón.
Ruby combina lo mejor de varios lenguajes: toma la elegancia de Perl, la
orientación a objetos de Smalltalk, y la facilidad de uso de Python.

La filosofía central de Ruby se resume en una frase de Matz: "Ruby está
diseñado para hacer felices a los programadores". Esto significa que la
experiencia del desarrollador es lo más importante. Ruby prioriza la
legibilidad, la expresividad y la facilidad de mantenimiento del código.

```ruby
# Este es un ejemplo clásico de la expresividad de Ruby
puts "¡Hola, Mundo!"

# Ruby se lee casi como inglés
5.times do
  puts "Ruby es elegante"
end

# La orientación a objetos es pura en Ruby
puts 3.class          # Integer
puts "texto".class    # String
puts true.class       # TrueClass
puts nil.class        # NilClass
```

Ruby es conocido por ser el lenguaje subyacente de Ruby on Rails, uno de los
frameworks web más influyentes de la historia. Sin embargo, Ruby es mucho más
que Rails: se utiliza en automatización, análisis de datos, scripting,
desarrollo de juegos, inteligencia artificial y mucho más.

2. CARACTERÍSTICAS PRINCIPALES DE RUBY
===============================================================================

Ruby posee una serie de características que lo distinguen de otros lenguajes:

--- 2.1 LENGUAJE INTERPRETADO ---

Ruby es un lenguaje interpretado, no compilado. Esto significa que el código
fuente se ejecuta línea por línea a través de un intérprete (llamado MRI -
Matz's Ruby Interpreter) sin necesidad de un paso de compilación previo.

```ruby
# Puedes ejecutar Ruby directamente desde la línea de comandos
# ruby -e 'puts "Esto se ejecuta sin archivo .rb"'

# O crear un archivo .rb y ejecutarlo
# ruby mi_programa.rb

# El intérprete lee y ejecuta en tiempo real
puts "El intérprete está ejecutando esta línea ahora mismo"
```

Ventajas de ser interpretado:
- Desarrollo rápido: prueba ideas sin compilar
- Portabilidad: el mismo código corre en diferentes plataformas
- Facilidad de depuración: puedes inspeccionar el estado en tiempo de ejecución

--- 2.2 TODO ES UN OBJETO ---

En Ruby, absolutamente todo es un objeto. No hay tipos primitivos separados.
Los números, strings, booleanos, incluso las clases y los métodos son objetos.

```ruby
# Todo tiene métodos porque todo es un objeto
puts -42.abs                  # 42
puts "ruby".capitalize        # Ruby
puts [1, 2, 3].length         # 3
puts 3.14.round               # 3
puts true.to_s                # "true"

# Incluso los números tienen métodos
puts 5.times { print "Hola " }  # Hola Hola Hola Hola Hola

# Las clases también son objetos (de Class)
puts String.class             # Class
puts Class.class              # Class

# Los métodos pueden ser llamados en cualquier objeto
objeto = "Soy un objeto"
puts objeto.object_id         # Un número único que identifica al objeto
puts objeto.respond_to?(:length)  # true
```

--- 2.3 SINTAXIS ELEGANTE Y EXPRESIVA ---

Ruby tiene una sintaxis limpia que se lee casi como lenguaje natural. Está
diseñada para minimizar la cantidad de código boilerplate y maximizar la
claridad.

```ruby
# Ruby evita símbolos innecesarios
# No uses paréntesis si no los necesitas
puts "Hola"

# Los bloques hacen el código más legible
[1, 2, 3].each { |n| puts n * 2 }

# Convenciones de nombres descriptivos
nombre_usuario = "Ana"
edad_usuario = 28

# Métodos con signos de interrogación para booleanos
puts "ruby".start_with?("r")    # true
puts [].empty?                  # true

# Métodos con signos de exclamación para versiones destructivas
texto = "hola"
puts texto.upcase               # "HOLA" (no modifica original)
puts texto                      # "hola"
puts texto.upcase!              # "HOLA" (modifica original)
puts texto                      # "HOLA"
```

--- 2.4 ORIENTACIÓN A OBJETOS PURA ---

Ruby implementa la programación orientada a objetos de manera completa y
consistente. Todo el código se escribe dentro de clases y objetos.

```ruby
# Definición de una clase en Ruby
class Persona
  # Constructor
  def initialize(nombre, edad)
    @nombre = nombre
    @edad = edad
  end

  # Método de instancia
  def saludar
    "Hola, soy #{@nombre}"
  end

  # Método de clase
  def self.especie
    "Humano"
  end
end

# Creación de objetos
persona1 = Persona.new("Ana", 30)
persona2 = Persona.new("Luis", 25)

puts persona1.saludar          # Hola, soy Ana
puts Persona.especie            # Humano

# Herencia
class Estudiante < Persona
  def initialize(nombre, edad, curso)
    super(nombre, edad)
    @curso = curso
  end

  def saludar
    "#{super} y estudio #{@curso}"
  end
end

estudiante = Estudiante.new("Carlos", 20, "Ruby")
puts estudiante.saludar         # Hola, soy Carlos y estudio Ruby
```

--- 2.5 TIPADO DINÁMICO ---

Ruby es dinámicamente tipado. Las variables no tienen tipo fijo; pueden
contener cualquier tipo de objeto y cambiar su tipo en tiempo de ejecución.

```ruby
# Las variables en Ruby no declaran su tipo
variable = 42
puts variable.class            # Integer

variable = "Ahora soy un texto"
puts variable.class            # String

variable = [1, 2, 3]
puts variable.class            # Array

variable = ->(x) { x * 2 }
puts variable.class            # Proc (lambda)

# Esto permite gran flexibilidad pero requiere cuidado
def procesar(dato)
  if dato.respond_to?(:each)
    dato.each { |e| puts e }
  elsif dato.respond_to?(:to_f)
    puts dato.to_f * 2
  else
    puts "No sé cómo procesar #{dato.class}"
  end
end

procesar([1, 2, 3])            # 1 2 3
procesar("5")                  # 10.0
procesar(:simbolo)             # No sé cómo procesar Symbol
```

--- 2.6 GESTIÓN AUTOMÁTICA DE MEMORIA ---

Ruby cuenta con un recolector de basura (Garbage Collector) que gestiona la
memoria automáticamente. No necesitas liberar memoria manualmente.

```ruby
# Ruby crea objetos y los destruye cuando ya no se usan
def crear_objetos_temporales
  1000.times do
    # Estos objetos se crean y destruyen automáticamente
    temp = "objeto #{rand(1000)}"
    temp.upcase
  end
  puts "Los objetos temporales ya fueron recolectados"
end

crear_objetos_temporales

# El GC se ejecuta periódicamente
puts "Iniciando recolección manual..."
GC.start  # Forzar recolección (rara vez necesario)
puts "Recolección completada"
```

--- 2.7 BLOQUES E ITERADORES ---

Una de las características más distintivas de Ruby son los bloques. Los bloques
son fragmentos de código anónimos que se pueden pasar a métodos.

```ruby
# Los bloques son el corazón de Ruby
# Se definen con do...end o con llaves {}

# Iteración con bloque
[1, 2, 3].each do |numero|
  puts "Número: #{numero}"
end

# Forma compacta con llaves
[1, 2, 3].each { |n| puts "Número: #{n}" }

# Bloques con yield
def ejecuta_bloque
  puts "Antes del bloque"
  yield if block_given?
  puts "Después del bloque"
end

ejecuta_bloque { puts "Dentro del bloque" }

# Los bloques capturan variables del entorno
nombre = "Ana"
[1, 2].each { puts "Hola #{nombre}" }

# Iteradores poderosos
puts [1, 2, 3, 4, 5].map { |n| n ** 2 }      # [1, 4, 9, 16, 25]
puts [1, 2, 3, 4, 5].select { |n| n.even? }   # [2, 4]
puts [1, 2, 3, 4, 5].reduce { |sum, n| sum + n }  # 15
```

4. FILOSOFÍA DE RUBY: EL PRINCIPIO DE MENOR SORPRESA
===============================================================================

El principio fundamental de Ruby es el "Principio de Menor Sorpresa" (POLS -
Principle of Least Surprise). Ruby se comporta de la manera que el
programador espera intuitivamente.

```ruby
# Ruby hace lo que esperas
puts "Ruby" + " " + "es" + " " + "genial"   # Ruby es genial

# Los métodos tienen nombres descriptivos
texto = "  Hola Mundo  "
puts texto.strip                # "Hola Mundo"
puts texto.downcase             # "  hola mundo  "
puts texto.swapcase             # "  hOLA mUNDO  "

# Las convenciones son consistentes
# ? para métodos booleanos
puts "".empty?                  # true
puts [1, 2].include?(2)        # true

# ! para métodos destructivos
fruta = "manzana"
fruta.capitalize!
puts fruta                      # "Manzana"

# Rangos intuitivos
puts (1..5).to_a                # [1, 2, 3, 4, 5]
puts (1...5).to_a               # [1, 2, 3, 4] (excluye el último)
```

Matz dice: "Ruby está diseñado para hacer felices a los programadores". Esto
significa que cuando escribes código Ruby, el lenguaje se adapta a tu forma de
pensar, no al revés.

```ruby
# Ruby tiene múltiples formas de hacer lo mismo
# Eliges la que te parezca más natural

# Forma 1: if tradicional
if x > 0
  puts "Positivo"
end

# Forma 2: if en una línea
puts "Positivo" if x > 0

# Forma 3: a menos que (unless)
unless x <= 0
  puts "Positivo"
end

# Forma 4: operador ternario
puts x > 0 ? "Positivo" : "No positivo"
```

5. COMPARATIVA CON OTROS LENGUAJES
===============================================================================

Ruby no existe en el vacío. Entender cómo se compara con otros lenguajes te
ayudará a apreciar sus fortalezas únicas.

--- 5.1 RUBY VS PYTHON ---

Ruby y Python son quizás los lenguajes más comparados. Ambos son dinámicos,
interpretados y de alto nivel. Sin embargo, tienen diferencias filosóficas
importantes.

| Aspecto          | Ruby                        | Python                      |
|-----------------|-----------------------------|-----------------------------|
| Filosofía       | Felicidad del programador   | Una forma de hacerlo        |
| Orientación a objetos | Pura (todo es objeto)  | Híbrida (tipos primitivos) |
| Bloques         | Sí (do/end, llaves)         | No (lambda limitado)        |
| Paradigma       | Multiparadigma              | Multiparadigma              |
| Gems/PyPI       | RubyGems                    | pip                         |
| Framework web   | Ruby on Rails               | Django, Flask               |

```ruby
# Ruby: Todo es un objeto
puts 5.class                   # Integer
puts 5 + 5                     # 10 (en realidad es 5.+(5))
puts 5.send(:+, 5)            # 10 (llamada a método explícita)

# Ruby: Bloques
[1, 2, 3].map { |n| n * 2 }   # [2, 4, 6]

# Ruby: Sintaxis flexible
puts "Hola" if true
puts "Adios" unless false
```

--- 5.2 RUBY VS JAVASCRIPT ---

JavaScript y Ruby comparten algunas similitudes (tipado dinámico, funciones de
primera clase, orientación a objetos basada en prototipos/clases), pero difieren
en su sintaxis y modelo de objetos.

```ruby
# Ruby: todo es un objeto
# JavaScript: primitivos no son objetos

# Ruby: clases tradicionales
class Animal
  def hablar
    puts "..."
  end
end

# Ruby: mixins (múltiple herencia limitada)
module Volador
  def volar
    puts "Volando!"
  end
end

class Pajaro < Animal
  include Volador
end
```

--- 5.3 RUBY VS JAVA ---

Java y Ruby representan dos filosofías muy diferentes. Java es estático y
verboso; Ruby es dinámico y conciso.

```ruby
# Java requiere mucho código boilerplate
# Ruby hace lo mismo con menos líneas

# Java:
# public class Persona {
#   private String nombre;
#   public Persona(String nombre) {
#     this.nombre = nombre;
#   }
#   public String getNombre() {
#     return nombre;
#   }
# }

# Ruby:
class Persona
  attr_accessor :nombre

  def initialize(nombre)
    @nombre = nombre
  end
end

p = Persona.new("Ana")
puts p.nombre        # Ana
p.nombre = "Luis"
puts p.nombre        # Luis
```

```ruby
    @nombre = nombre
    @productos = []
  end
```

--- 6.3 LIMITACIONES DE RUBY ---

Ningún lenguaje es perfecto. Ruby tiene limitaciones que debes conocer:

- **Rendimiento**: Ruby no es el lenguaje más rápido (MRI)
- **Concurrencia**: el GIL (Global Interpreter Lock) limita el paralelismo
- **Tipado dinámico**: puede causar errores en tiempo de ejecución
- **Consumo de memoria**: mayor que lenguajes compilados
- **Arranque lento**: la carga de Rails puede ser lenta

Sin embargo, para la mayoría de aplicaciones web y scripting, estas
limitaciones no son problemas reales. La productividad del desarrollador
compensa con creces estas diferencias.

```ruby

# JRuby: Ruby en la JVM (sin GIL, verdadero paralelismo)
# TruffleRuby: implementación de alto rendimiento
# Ruby 3.x: mejoras significativas de velocidad (Ruby 3x3)

ractor = Ractor.new do
  "Ejecutando en paralelo"
end

puts ractor.take
```

--- 7.2 MENTALIDAD RUBY ---

Para programar bien en Ruby, necesitas adoptar cierta mentalidad:

1. **Escribe código legible**: el código se lee más veces de las que se escribe
2. **Sé expresivo**: elige nombres descriptivos para variables y métodos
3. **Usa bloques**: los bloques son el alma de Ruby
4. **Aprovecha la orientación a objetos**: todo es un objeto
5. **Sigue las convenciones**: snake_case, métodos con ?, métodos con !
6. **DRY (Don't Repeat Yourself)**: no te repitas
7. **Prueba tu código**: Ruby tiene excelentes herramientas de testing

```ruby
# Código Ruby expresivo y limpio
class Calculadora
  def initialize
    @historial = []
  end

  def sumar(a, b)
    resultado = a + b
    @historial << "sumar(#{a}, #{b}) = #{resultado}"
    resultado
  end

  def restar(a, b)
    resultado = a - b
    @historial << "restar(#{a}, #{b}) = #{resultado}"
    resultado
  end

  def historial
    @historial.join("\n")
  end
end

calc = Calculadora.new
puts calc.sumar(10, 5)    # 15
puts calc.restar(10, 5)   # 5
puts calc.historial
```

8. EL FUTURO DE RUBY
===============================================================================

Ruby continúa evolucionando. Con la llegada de Ruby 3.x, el lenguaje ha dado
pasos importantes en rendimiento, concurrencia y tipado estático opcional.

--- 8.1 RUBY 3.0 Y MÁS ALLÁ ---

Ruby 3.0 (Navidad 2020) introdujo:

- **Ruby 3x3**: 3 veces más rápido que Ruby 2.0
- **Ractors**: modelo de concurrencia sin GIL
- **Fiber Scheduler**: fibra ligera para E/S concurrente
- **Type Profiler**: análisis de tipos estático opcional
- **Pattern Matching**: coincidencia de patrones
- **Find pattern**: búsqueda en estructuras de datos

```ruby
# Pattern Matching en Ruby 3
case [1, 2, 3]
in [a, b, c]
  puts "Array de 3 elementos: #{a}, #{b}, #{c}"
end

# Pattern Matching con estructuras
case { nombre: "Ana", edad: 30 }
in { nombre:, edad: }
  puts "#{nombre} tiene #{edad} años"
end

# Ractors (concurrencia sin GIL)
ractor = Ractor.new do
  resultado = 0
  1000000.times { resultado += 1 }
  resultado
end

puts ractor.take
```

--- 8.2 RUBY 3.3 Y 3.4 ---

Las versiones más recientes continúan mejorando:

- Mejoras en el GC (Generational + Incremental GC)
- RJIT y YJIT: compiladores JIT para mejor rendimiento
- Mejoras en Prism parser (nuevo parser oficial)
- LSP nativo (Language Server Protocol)
- Mejoras en seguridad y rendimiento

```ruby
# YJIT compila Ruby a código máquina nativo
# Actívalo con: ruby --yjit mi_archivo.rb

# Prism parser (Ruby 3.3+)
# Más rápido y compatible hacia atrás

# Mejoras en debugging
require 'debug'
binding.break  # Punto de interrupción (similar a byebug/pry)
```

9. CONCEPTOS ERRÓNEOS COMUNES SOBRE RUBY
===============================================================================

Aclaremos algunos mitos comunes sobre Ruby:

--- 9.1 "RUBY ES SOLO RAILS" ---

Falso. Ruby es un lenguaje de propósito general que existía 9 años antes de
Rails. Se usa para scripting, automatización, análisis de datos, etc.

--- 9.2 "RUBY ES MUY LENTO" ---

Ruby no es el más rápido, pero con Ruby 3.x y YJIT, el rendimiento ha mejorado
drásticamente. Para el 99% de las aplicaciones, el rendimiento no es un cuello
de botella. La productividad del desarrollador suele ser más importante.

--- 9.3 "RUBY ESTÁ MUERTO" ---

Ruby no está ni cerca de estar muerto. Tiene una comunidad activa, grandes
empresas que lo usan, y versiones lanzadas cada año. GitHub, Shopify, y
GitLab son prueba de su vigencia.

--- 9.4 "RUBY ES DIFÍCIL DE APRENDER" ---

Ruby es considerado uno de los lenguajes más amigables para principiantes. Su
sintaxis natural y su comunidad acogedora lo hacen ideal para aprender a
programar.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Necesito saber programación para aprender Ruby?**
No, Ruby es un excelente primer lenguaje. Su sintaxis natural y su comunidad
amigable lo hacen ideal para principiantes. Este libro comienza desde cero.

❓ **¿Cuánto tiempo toma aprender Ruby?**
Puedes aprender los fundamentos en 2-4 semanas. Dominar el lenguaje y Rails
puede tomar de 3 a 6 meses de práctica constante. El viaje nunca termina.

❓ **¿Ruby es solo para desarrollo web?**
No. Aunque Ruby on Rails es muy popular, Ruby se usa para scripting,
automatización, análisis de datos, DevOps, juegos y más.

❓ **¿Debo aprender Ruby o Ruby on Rails primero?**
Aprende Ruby primero. Rails es un framework que usa Ruby. Sin una base sólida
de Ruby, te será difícil entender Rails profundamente.

❓ **¿Ruby es un lenguaje que paga bien?**
Sí. Los desarrolladores Ruby/Rails están entre los mejor pagados. Empresas
como GitHub, Shopify y Basecamp pagan salarios competitivos.

❓ **¿Cuál es la diferencia entre Ruby y Ruby on Rails?**
Ruby es el lenguaje de programación. Ruby on Rails es un framework web
construido en Ruby. Es como la diferencia entre madera y una casa de madera.

❓ **¿Qué versión de Ruby debo instalar?**
La versión más reciente estable (Ruby 3.x). En el Capítulo 3 aprenderás a
instalar y gestionar versiones con RVM o rbenv.

❓ **¿Ruby es adecuado para microservicios?**
Sí. Combinado con Sinatra, Hanami o Rails API, Ruby es excelente para
microservicios. Su productividad permite iterar rápidamente.

❓ **¿Debo preocuparme por el rendimiento de Ruby?**
Para el 99% de las aplicaciones, el rendimiento de Ruby es más que suficiente.
Si necesitas máximo rendimiento, existen implementaciones como JRuby.

❓ **¿Puedo conseguir trabajo sabiendo solo Ruby?**
Sí, pero es recomendable complementar con HTML, CSS, JavaScript y SQL si
quieres desarrollo web. También es valioso conocer Rails y bases de datos.
