===============================================================================
1. COMENTARIOS DE UNA LÍNEA
===============================================================================

Los comentarios de una línea comienzan con `#`. Todo lo que sigue en esa
línea es ignorado por el intérprete.

--- 1.1 SINTAXIS BÁSICA ---

```ruby
# Esto es un comentario de una línea
puts "Hola Mundo"  # Esto también es un comentario (inline)

# Los comentarios pueden estar al inicio de la línea
# O después del código
# O en líneas intermedias

# Los comentarios son ignorados completamente por Ruby
# No afectan la ejecución ni el rendimiento
```

--- 1.2 COMENTARIOS INLINE ---

Los comentarios inline van al final de una línea de código.

```ruby
edad = 25          # Edad del usuario en años
nombre = "Ana"     # Nombre completo del usuario
activo = true      # Estado de la cuenta del usuario

total = subtotal * 1.16  # Aplica IVA del 16%

# Los comentarios inline deben estar separados del código
# con al menos 2 espacios para buena legibilidad
```

--- 1.3 COMENTARIOS DE BLOQUE CON # ---

Puedes crear bloques de comentarios usando múltiples líneas con #.

```ruby
#
# Este es un bloque de comentarios
# que abarca múltiples líneas
# usando el símbolo # al inicio
#

# También puedes hacerlo así:
#
# Sección: Configuración inicial
# Autor: Profesor Búho AI
# Fecha: 2024
#
```

--- 1.4 COMENTARIOS PARA DEBUGGING ---

```ruby
def calcular_complejo(x, y)
  # puts "DEBUG: x = #{x}, y = #{y}"  # Comentado para debugging

  resultado = x * y

  # if resultado > 1000
  #   puts "Resultado grande: #{resultado}"
  # end

  resultado
end

# También puedes comentar código temporalmente
# while condicion
#   puts "Esto está comentado"
#   proceso_importante
# end
```

===============================================================================
2. COMENTARIOS MULTILÍNEA (=BEGIN / =END)
===============================================================================

Ruby tiene una sintaxis especial para comentarios multilínea usando
`=begin` y `=end`.

--- 2.1 SINTAXIS BÁSICA ---

```ruby
=begin
Este es un comentario
multilínea en Ruby.
Todo lo que está entre
=begin y =end es ignorado.
=end

puts "Este código sí se ejecuta"

=begin
Puedes escribir párrafos enteros
sin tener que poner # al inicio
de cada línea.
Muy útil para documentación extensa.
=end
```

--- 2.2 REGLAS IMPORTANTES ---

```ruby
# =begin y =end DEBEN estar al inicio de la línea
# No puede haber espacios antes de =begin

=begin
  Esto funciona correctamente
  porque =begin está al inicio
=end

# Esto NO funciona:
#   =begin
#   Esto tiene espacios antes
#   =end

# =begin y =end deben estar en sus propias líneas
# No pueden compartir línea con código:

# =begin Esto no funciona =end
```

--- 2.3 USOS APROPIADOS ---

```ruby
# =begin/=end es útil para:
# 1. Comentar grandes bloques de código durante debugging

=begin
def metodo_complejo
  # ... muchas líneas de código
  # que quieres desactivar temporalmente
end
=end

# 2. Documentación larga al inicio de archivos

=begin
  Ruby Master Book - Módulo I
  Capítulo 9: Comentarios

  Este archivo contiene ejemplos de todos los
  tipos de comentarios en Ruby.

  Autor: Profesor Búho AI
  Versión: 1.0
=end

# 3. Notas extensas para desarrolladores

=begin
  NOTA DE IMPLEMENTACIÓN:
  Este algoritmo utiliza búsqueda binaria en lugar
  de búsqueda lineal porque los datos están ordenados
  y el conjunto es grande (> 10000 elementos).

  Complejidad: O(log n) en tiempo, O(1) en espacio.
=end
```

--- 2.4 =BEGIN/=END VS MÚLTIPLES # ---

```ruby
# Para la mayoría de casos, los comentarios con #
# son preferibles porque:

# 1. Son más visibles y comunes en Ruby
# 2. Algunos editores no resaltan bien =begin/=end
# 3. Son más fáciles de alternar con atajos de teclado

# La convención de Ruby es usar # para casi todo
# =begin/=end se usa principalmente para:
# - Documentación al inicio de archivos
# - Grandes bloques de documentación
# - Desactivar código temporalmente (pero preferible usar control de versiones)
```

===============================================================================
3. COMENTARIOS DE DOCUMENTACIÓN (RDoc/YARD)
===============================================================================

Ruby tiene potentes herramientas de documentación: RDoc y YARD. Estas
herramientas extraen comentarios especiales para generar documentación HTML.

--- 3.1 RDOC ---

RDoc es la herramienta de documentación estándar de Ruby. Usa comentarios
con formato especial antes de clases, módulos y métodos.

```ruby
# == RDoc Documentation
#
# Esta clase calcula operaciones matemáticas básicas.
# Proporciona métodos para suma, resta, multiplicación y división.
#
# == Ejemplos
#
#   calc = Calculadora.new
#   calc.sumar(10, 5)  #=> 15
#
class Calculadora
  # Inicializa la calculadora con un valor opcional
  #
  # === Parámetros
  #
  # * +valor_inicial+ - valor inicial de la calculadora (por defecto 0)
  #
  # === Ejemplo
  #
  #   calc = Calculadora.new(100)
  #
  def initialize(valor_inicial = 0)
    @valor = valor_inicial
  end

  # Suma un número al valor actual
  #
  # === Parámetros
  #
  # * +numero+ - el número a sumar
  #
  # === Retorna
  #
  # El resultado de la suma
  #
  def sumar(numero)
    @valor += numero
  end

  # Resta un número del valor actual
  #
  # === Parámetros
  # [numero] el número a restar
  #
  # === Retorna
  # El resultado de la resta
  #
  def restar(numero)
    @valor -= numero
  end
end
```

--- 3.2 YARD ---

YARD es una alternativa moderna a RDoc con sintaxis más expresiva.

```ruby
# YARD utiliza @tag para marcar elementos de documentación

# Clase para representar un usuario del sistema
#
# @author Profesor Búho AI
# @since 1.0.0
#
# @example Crear un usuario
#   usuario = Usuario.new("Ana", "ana@email.com")
#   usuario.saludar  #=> "Hola, soy Ana"
#
class Usuario
  # @return [String] el nombre del usuario
  attr_reader :nombre

  # @return [String] el email del usuario
  attr_reader :email

  # Crea un nuevo usuario
  #
  # @param nombre [String] el nombre del usuario
  # @param email [String] el email del usuario
  # @param edad [Integer] la edad del usuario (opcional)
  #
  # @raise [ArgumentError] si el email no es válido
  #
  def initialize(nombre, email, edad = nil)
    @nombre = nombre
    @email = email
    @edad = edad
  end

  # Genera un saludo personalizado
  #
  # @example
  #   usuario.saludar  #=> "Hola, soy Ana"
  #
  # @return [String] el saludo
  #
  def saludar
    "Hola, soy #{@nombre}"
  end

  # Verifica si el usuario es mayor de edad
  #
  # @return [Boolean] true si tiene 18 años o más, false en caso contrario
  #
  def mayor_edad?
    @edad && @edad >= 18
  end

  # Actualiza el email del usuario
  #
  # @param nuevo_email [String] el nuevo email
  #
  # @return [Boolean] true si se actualizó correctamente
  #
  # @raise [ArgumentError] si el formato del email es inválido
  #
  def actualizar_email(nuevo_email)
    unless nuevo_email.match?(/\A[\w.+-]+@[\w-]+\.[\w.]+\z/)
      raise ArgumentError, "Email inválido: #{nuevo_email}"
    end
    @email = nuevo_email
    true
  end
end
```

4. MEJORES PRÁCTICAS PARA COMENTARIOS
===============================================================================

--- 4.1 EL QUÉ VS EL POR QUÉ ---

El código dice QUÉ hace. Los comentarios deben explicar POR QUÉ lo hace.

```ruby
# MAL comentario (dice lo que el código ya muestra):
x = x + 1  # Incrementa x en 1

# BUEN comentario (explica por qué):
# Incrementamos el contador para evitar el bucle infinito
# que ocurriría si el usuario no presiona una tecla
x = x + 1

# MAL:
total = subtotal * 1.16  # Multiplica por 1.16

# BUEN:
total = subtotal * 1.16  # Aplica IVA del 16% (tasa vigente en México)
```

--- 4.2 COMENTARIOS DE SECCIÓN ---

Organiza tu código con comentarios de sección claros.

```ruby
# ============================================

# ============================================

# ============================================

# ============================================
# MÉTODOS DE AUTENTICACIÓN
# ============================================

# ============================================
```

```ruby
# TODO: Implementar validación de email

def procesar_datos(datos)
  # TODO: Agregar manejo de errores aquí
```

6. COMENTARIOS PARA GENERACIÓN DE DOCUMENTACIÓN

--- 6.1 DOCUMENTACIÓN DE MÓDULOS ---

```ruby
# Documentación de módulo con RDoc

# =Módulo de utilidades matemáticas
#
# Este módulo proporciona funciones matemáticas adicionales
# que no están incluidas en el módulo Math de Ruby.
#
# == Métodos
#
# * +promedio+ - calcula el promedio de un array de números
# * +mediana+ - calcula la mediana de un array de números
# * +moda+ - encuentra el valor más frecuente
#
# == Ejemplo
#
#   include UtilidadesMatematicas
#   promedio([1, 2, 3, 4, 5])  #=> 3.0
#
module UtilidadesMatematicas
  # Calcula el promedio de un array de números
  def promedio(numeros)
    numeros.sum.to_f / numeros.length
  end

  # Calcula la mediana de un array de números
  def mediana(numeros)
    sorted = numeros.sort
    medio = sorted.length / 2
    sorted.length.odd? ? sorted[medio] : (sorted[medio - 1] + sorted[medio]) / 2.0
  end
end
```

--- 6.2 DOCUMENTACIÓN DE EXCEPCIONES ---

```ruby
# Documentación completa de errores personalizados

# Error base para la aplicación
class AppError < StandardError; end

# Error cuando un usuario no es encontrado
#
# Se lanza cuando se intenta acceder a un usuario que no existe
# en la base de datos.
#
# @example
#   raise UsuarioNoEncontrado, "ID: 42"
#
class UsuarioNoEncontrado < AppError
  def initialize(mensaje = "Usuario no encontrado")
    super(mensaje)
  end
end

# Error cuando hay un problema de validación
#
# @attr_reader [Array<String>] errores lista de errores de validación
#
class ErrorValidacion < AppError
  attr_reader :errores

  def initialize(errores)
    @errores = errores
    super("Error de validación: #{errores.join(', ')}")
  end
end
```

--- 6.3 DOCUMENTACIÓN DE EJEMPLOS ---

```ruby
# La documentación debe incluir ejemplos de uso

# Convierte una cadena a formato título (cada palabra capitalizada)
#
# @param texto [String] el texto a convertir
#
# @return [String] el texto en formato título
#
# @example Texto simple
#   a_titulo("hola mundo")  #=> "Hola Mundo"
#
# @example Texto con palabras cortas
#   a_titulo("el gato en el sombrero")  #=> "El Gato en el Sombrero"
#
# @example Texto vacío
#   a_titulo("")  #=> ""
#
def a_titulo(texto)
  palabras_excluidas = %w[el la los las y e o de del en un una con por para]
  texto.split.each_with_index.map do |palabra, i|
    if i == 0 || !palabras_excluidas.include?(palabra.downcase)
      palabra.capitalize
    else
      palabra.downcase
    end
  end.join(" ")
end
```

8. PROYECTO: DOCUMENTADOR AUTOMÁTICO
===============================================================================

```ruby
# documentador.rb - Genera documentación YARD para tus clases
class DocumentadorAutomatico
  def initialize
    @clases = {}
  end

  def analizar(archivo)
    contenido = File.read(archivo)
    codigo = contenido.gsub(/=begin.*?=end/m, "")  # Elimina comentarios =begin
    codigo = codigo.gsub(/#.*$/, "")               # Elimina comentarios #

    clases = codigo.scan(/class\s+(\w+)/).flatten
    metodos = codigo.scan(/def\s+(self\.)?(\w+[?!]?)/).flatten.compact
    modulos = codigo.scan(/module\s+(\w+)/).flatten

    {
      archivo: archivo,
      clases: clases,
      metodos: metodos,
      modulos: modulos
    }
  end

  def generar_plantilla_yard(clase, metodos)
    plantilla = []
    plantilla << "# #{clase} - Descripción pendiente"
    plantilla << "#"
    plantilla << "# @author Documentador Automático"
    plantilla << "# @since 1.0.0"
    plantilla << "#"
    plantilla << "# @example"
    plantilla << "#   #{clase.downcase} = #{clase}.new"
    plantilla << "#   #{clase.downcase}.metodo"
    plantilla << "#"
    plantilla << "class #{clase}"
    plantilla << ""

    metodos.each do |metodo|
      plantilla << "  # Descripción de ##{metodo}"
      plantilla << "  #"
      plantilla << "  # @return [void]"
      plantilla << "  #"
      plantilla << "  # @example"
      plantilla << "  #   #{clase.downcase}.#{metodo}"
      plantilla << "  #"
      plantilla << "  def #{metodo}"
      plantilla << "    # TODO: Implementar"
      plantilla << "  end"
      plantilla << ""
    end

    plantilla << "end"
    plantilla.join("\n")
  end

  def reporte(archivos)
    puts "=== REPORTE DE DOCUMENTACIÓN ==="
    puts ""
    archivos.each do |archivo|
      info = analizar(archivo)
      puts "Archivo: #{info[:archivo]}"
      puts "  Clases: #{info[:clases].join(', ')}" unless info[:clases].empty?
      puts "  Módulos: #{info[:modulos].join(', ')}" unless info[:modulos].empty?
      puts "  Métodos: #{info[:metodos].length}"
      puts ""
    end
  end
end
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Los comentarios de una línea comienzan con # y van hasta el final de la línea
- =begin / =end crean comentarios multilínea (deben ir al inicio de línea)
- RDoc y YARD generan documentación HTML a partir de comentarios especiales
- Los buenos comentarios explican el POR QUÉ, no el QUÉ (el código ya muestra qué hace)
- Usa nombres descriptivos para reducir la necesidad de comentarios
- Los comentarios inline deben ir con 2 espacios de separación del código
- Los magic comments (# frozen_string_literal, # encoding) afectan al intérprete
- YARD usa etiquetas como @param, @return, @example, @raise
- TODO, FIXME, XXX, HACK son marcadores útiles para tareas pendientes
- Los comentarios desactualizados son peor que ningún comentario
- Documenta siempre las APIs públicas con YARD o RDoc
- Los comentarios de depuración (código comentado) deben eliminarse antes de producción
- La mejor documentación es el código limpio y bien estructurado
- Los comentarios no afectan el rendimiento (son ignorados por el intérprete)
- Usa shebang (#!) al inicio para scripts ejecutables
- Escribe comentarios pensando en quien leerá tu código en el futuro

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Los comentarios afectan el rendimiento?**
No. Los comentarios son ignorados completamente por el intérprete. No tienen
ningún impacto en el rendimiento.

❓ **¿Debo comentar todo mi código?**
No. Comenta el POR QUÉ, no el QUÉ. El código bien escrito es auto-explicativo.
Los comentarios sobran cuando el código es claro.

❓ **¿Qué es mejor: muchos comentarios o código limpio?**
Código limpio siempre es mejor. Los comentarios deben complementar, no
substituir, un código claro y bien estructurado.

❓ **¿Debo eliminar el código comentado antes de hacer commit?**
Sí. El código comentado es ruido. Si necesitas recuperarlo, usa el control
de versiones (git). No dejes código muerto.

❓ **¿Qué es frozen_string_literal?**
Un magic comment que congela todos los strings literales del archivo,
haciéndolos inmutables. Mejora rendimiento y seguridad.

❓ **¿Puedo usar acentos en los comentarios?**
Sí, siempre que el archivo esté en UTF-8 (por defecto en Ruby). Los comentarios
pueden contener cualquier carácter Unicode.

❓ **¿Cómo documento mi código para otros desarrolladores?**
Usa YARD (estándar moderno) o RDoc (estándar histórico). Ambos generan
documentación HTML profesional a partir de comentarios.

❓ **¿Qué significan TODO, FIXME, HACK?**
Son marcadores estándar: TODO (por hacer), FIXME (hay un bug), HACK (solución
temporal), OPTIMIZE (mejorable), XXX (peligro).

🏗️ **Arquitecto de Software** — Diseña un "Sistema de Gestión de Documentación"

Diseña una clase `GestorDocumentacion` que:

1. Escanee un directorio de archivos Ruby en busca de comentarios
2. Extraiga clases, módulos y métodos documentados
3. Cuantifique la cobertura de documentación (% de métodos documentados)
4. Genere un reporte JSON con la documentación encontrada
5. Identifique elementos no documentados que deberían tener documentación

```ruby
class GestorDocumentacion
  attr_reader :reporte

  def initialize(directorio)
    @directorio = directorio
    @reporte = { archivos: [], total: { clases: 0, metodos: 0, documentados: 0 } }
  end

  def escanear
    Dir.glob("#{@directorio}/**/*.rb").each do |archivo|
      info = analizar_archivo(archivo)
      @reporte[:archivos] << info if info[:clases] > 0 || info[:metodos] > 0
    end
    calcular_totales
    self
  end

  def analizar_archivo(archivo)
    contenido = File.read(archivo)
    lineas = contenido.lines

    info = {
      archivo: archivo,
      clases: [],
      metodos: [],
      documentacion: { completa: 0, parcial: 0, ninguna: 0 }
    }

    # Analizar clases
    lineas.each_with_index do |linea, i|
      if linea.match?(/^\s*class\s+(\w+)/)
        clase = $1
        tiene_doc = i > 0 && lineas[i - 1].match?(/^\s*#/)
        info[:clases] << { nombre: clase, documentada: tiene_doc }
      end

      if linea.match?(/^\s*def\s+(self\.)?(\w+[?!]?)/)
        metodo = $2
        tiene_doc = i > 0 && lineas[i - 1].match?(/^\s*#/)
        info[:metodos] << { nombre: metodo, documentada: tiene_doc }
      end
    end

    info
  end

  def calcular_totales
    @reporte[:archivos].each do |info|
      @reporte[:total][:clases] += info[:clases].length
      @reporte[:total][:metodos] += info[:metodos].length
      @reporte[:total][:documentados] += info[:metodos].count { |m| m[:documentada] }
    end
  end

  def cobertura
    total = @reporte[:total][:metodos]
    documentados = @reporte[:total][:documentados]
    total > 0 ? (documentados.to_f / total * 100).round(1) : 0
  end

  def elementos_no_documentados
    no_doc = []
    @reporte[:archivos].each do |info|
      info[:metodos].select { |m| !m[:documentada] }.each do |m|
        no_doc << "#{info[:archivo]}##{m[:nombre]}"
      end
    end
    no_doc
  end

  def generar_reporte
    puts "=== REPORTE DE DOCUMENTACIÓN ==="
    puts "Directorio: #{@directorio}"
    puts "Archivos analizados: #{@reporte[:archivos].length}"
    puts "Clases encontradas: #{@reporte[:total][:clases]}"
    puts "Métodos encontrados: #{@reporte[:total][:metodos]}"
    puts "Métodos documentados: #{@reporte[:total][:documentados]}"
    puts "Cobertura: #{cobertura}%"
    puts ""
    puts "Elementos no documentados:"
    elementos_no_documentados.each { |e| puts "  - #{e}" }
  end
end

```
