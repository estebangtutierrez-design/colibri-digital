1. INTRODUCCIÓN A DIR
----------------------------------------------------------------------

Dir es la clase de Ruby para trabajar con directorios. No requiere
require (es parte del core). Ofrece métodos de clase para operaciones
comunes y se puede instanciar para iterar sobre entradas.

Métodos de clase principales:
  Dir.pwd          # directorio de trabajo actual
  Dir.getwd        # sinónimo de pwd
  Dir.chdir(ruta)  # cambiar directorio
  Dir.glob(patron) # buscar archivos con patrones
  Dir.entries(ruta)# listar entradas (incluye . y ..)
  Dir.children(ruta)# listar entradas (sin . y ..)
  Dir.mkdir(ruta)  # crear directorio
  Dir.delete(ruta) # eliminar directorio
  Dir.exist?(ruta) # verificar si existe
  Dir.home         # directorio home del usuario
  Dir.tmpdir       # directorio temporal

También se puede instanciar:
  d = Dir.new("ruta")
  d.each { |entrada| puts entrada }
  d.close

================================================================================
EJEMPLO 1: Introducción a Dir
================================================================================

#------ archivo: ej01_intro_dir.rb ------
puts "=== INTRODUCCIÓN A DIR ==="

# Directorio actual
puts "Directorio actual: #{Dir.pwd}"
puts "También: #{Dir.getwd}"

# Directorio home
puts "Directorio home: #{Dir.home}"

# Directorio temporal
puts "Directorio tmp: #{Dir.tmpdir}"

# Verificar existencia
puts "¿Existe /tmp? #{Dir.exist?('/tmp')}"
puts "¿Existe /no_existe? #{Dir.exist?('/no_existe')}"
puts "¿Es directorio? #{File.directory?('/tmp')}"

# Listar entradas
puts "\nEntradas de /tmp (primeras 10):"
Dir.entries("/tmp").first(10).each { |e| puts "  #{e}" }

puts "\nChildren de /tmp (sin . y .., primeras 10):"
Dir.children("/tmp").first(10).each { |e| puts "  #{e}" }

#------ fin del ejemplo ------

2. DIR.GLOB — BÚSQUEDA CON PATRONES
----------------------------------------------------------------------

Dir.glob es el método más poderoso para listar archivos. Acepta
patrones similares a los de la terminal (shell globbing).

Patrones comunes:
  *          - cualquier secuencia de caracteres (excepto /)
  ?          - un solo carácter
  [abc]      - un carácter del conjunto
  [a-z]      - un carácter en el rango
  {a,b,c}    - expansión de alternativas
  **/        - cualquier nivel de subdirectorios
  .*         - archivos ocultos (que empiezan con .)

Ejemplos:
  Dir.glob("*.rb")           # todos los .rb del directorio actual
  Dir.glob("**/*.txt")       # todos los .txt recursivamente
  Dir.glob("src/**/*.rb")    # todos los .rb dentro de src/
  Dir.glob("*.{rb,py,js}")   # .rb, .py, .js
  Dir.glob(".*")             # archivos ocultos

================================================================================
EJEMPLO 2: Dir.glob en acción
================================================================================

#------ archivo: ej02_glob.rb ------
puts "=== DIR.GLOB ==="

Dir.mkdir("glob_test") unless Dir.exist?("glob_test")
%w[main.rb app.rb helper.rb config.yml README.md data.csv].each do |f|
  File.write("glob_test/#{f}", "")
end
Dir.mkdir("glob_test/lib") unless Dir.exist?("glob_test/lib")
%w[core.rb utils.rb].each { |f| File.write("glob_test/lib/#{f}", "") }

puts "Patrones glob en acción:"

puts "\n* (todos):"
Dir.glob("glob_test/*").each { |f| puts "  #{f}" }

puts "\n*.rb (Ruby):"
Dir.glob("glob_test/*.rb").each { |f| puts "  #{f}" }

puts "\n*.{rb,yml} (Ruby o YAML):"
Dir.glob("glob_test/*.{rb,yml}").each { |f| puts "  #{f}" }

puts "\n**/* (recursivo):"
Dir.glob("glob_test/**/*").each { |f| puts "  #{f}" }

puts "\n**/*.rb (Ruby recursivo):"
Dir.glob("glob_test/**/*.rb").each { |f| puts "  #{f}" }

puts "\n?ain.rb (? = un carácter):"
Dir.glob("glob_test/?ain.rb").each { |f| puts "  #{f}" }

puts "\n*[aeiou]*.rb (vocales):"
Dir.glob("glob_test/*[aeiou]*.rb").each { |f| puts "  #{f}" }

puts "\nFile::FNM_CASEFOLD:"
Dir.glob("glob_test/*.TXT", File::FNM_CASEFOLD).each { |f| puts "  #{f}" }

puts "\nDir[] (atajo):"
Dir["glob_test/*.rb"].each { |f| puts "  #{f}" }

FileUtils.rm_rf("glob_test") if defined?(FileUtils)

#------ fin del ejemplo ------

----------------------------------------------------------------------
3. DIR.ENTRIES Y DIR.CHILDREN
----------------------------------------------------------------------

Dir.entries lista todas las entradas incluyendo "." y "..".
Dir.children las excluye.

  entries = Dir.entries("ruta")
  children = Dir.children("ruta")

================================================================================
EJEMPLO 3: Dir.entries y Dir.children
================================================================================

#------ archivo: ej03_entries_children.rb ------
puts "=== DIR.ENTRIES VS DIR.CHILDREN ==="

Dir.mkdir("test_dir") unless Dir.exist?("test_dir")
%w[a.txt b.txt c.txt].each { |f| File.write("test_dir/#{f}", "") }
Dir.mkdir("test_dir/subdir") unless Dir.exist?("test_dir/subdir")

puts "Dir.entries (incluye . y ..):"
Dir.entries("test_dir").each { |e| puts "  #{e.inspect}" }

puts "\nDir.children (sin . y ..):"
Dir.children("test_dir").each { |e| puts "  #{e.inspect}" }

puts "\nSolo archivos:"
Dir.entries("test_dir")
   .reject { |e| e.start_with?('.') }
   .select { |e| File.file?("test_dir/#{e}") }
   .each { |f| puts "  #{f}" }

puts "\nSolo directorios:"
Dir.children("test_dir")
   .select { |e| File.directory?("test_dir/#{e}") }
   .each { |d| puts "  #{d}" }

FileUtils.rm_rf("test_dir") if defined?(FileUtils)

#------ fin del ejemplo ------

4. DIR.MKDIR, DIR.DELETE, DIR.RMDIR
----------------------------------------------------------------------

  Dir.mkdir("nombre")         # crear directorio
  Dir.mkdir("nombre", 0755)   # con permisos
  Dir.delete("nombre")        # eliminar (vacío)
  Dir.rmdir("nombre")         # sinónimo
  FileUtils.mkdir_p("a/b/c")  # mkdir -p

================================================================================
EJEMPLO 4: Crear y eliminar directorios
================================================================================

#------ archivo: ej04_mkdir_delete.rb ------
require 'fileutils'

puts "=== CREAR Y ELIMINAR DIRECTORIOS ==="

Dir.mkdir("nuevo_directorio") unless Dir.exist?("nuevo_directorio")
puts "Creado: nuevo_directorio"
puts "  ¿Existe? #{Dir.exist?('nuevo_directorio')}"

Dir.mkdir("directorio_privado", 0700) unless Dir.exist?("directorio_privado")
puts "\nCreado: directorio_privado (0700)"
stat = File.stat("directorio_privado")
puts "  Permisos: #{stat.mode.to_s(8)}"

FileUtils.mkdir_p("a/b/c/d")
puts "\nCreado: a/b/c/d"
puts "  ¿Existe a/b/c/d? #{Dir.exist?('a/b/c/d')}"

begin
  Dir.mkdir("x/y/z")
rescue Errno::ENOENT => e
  puts "\nError mkdir simple: #{e.message}"
end

Dir.delete("nuevo_directorio")
puts "\nEliminado: nuevo_directorio"

Dir.delete("a/b/c/d"); Dir.delete("a/b/c"); Dir.delete("a/b"); Dir.delete("a")

Dir.mkdir("no_vacio")
File.write("no_vacio/archivo.txt", "contenido")
begin
  Dir.delete("no_vacio")
rescue Errno::ENOTEMPTY => e
  puts "\nError dir no vacío: #{e.message}"
end
FileUtils.rm_rf("no_vacio")
FileUtils.rm_rf("directorio_privado")

#------ fin del ejemplo ------

================================================================================
EJEMPLO 5: Dir.chdir

7. PATHNAME — MANEJO ELEGANTE DE RUTAS
----------------------------------------------------------------------

Pathname es una clase de la librería estándar que unifica File, Dir
y FileUtils en una interfaz coherente.

  require 'pathname'
  pn = Pathname.new("/usr/local/bin/ruby")
  pn.basename     # => #<Pathname:ruby>
  pn.dirname      # => #<Pathname:/usr/local/bin>
  pn.exist?       # => true
  pn.file?        # => true
  pn.read         # leer
  pn.write("datos") # escribir

Pathname es encadenable:
  Pathname.new("/usr").join("local", "bin").join("ruby")

================================================================================
EJEMPLO 7: Pathname básico
================================================================================

#------ archivo: ej07_pathname.rb ------
require 'pathname'

puts "=== PATHNAME BÁSICO ==="

p = Pathname.new("/usr/local/bin/ruby")
puts "Ruta: #{p}"
puts "  class: #{p.class}"

puts "\nComponentes:"
puts "  basename:  #{p.basename}"
puts "  dirname:   #{p.dirname}"
puts "  extname:   #{p.extname}"
puts "  parent:    #{p.parent}"

puts "\nConsultas:"
puts "  exist?: #{p.exist?}"
puts "  file?:  #{p.file?}"

# join
p2 = Pathname.new("/usr").join("local", "bin")
puts "\njoin: #{p2}"

# Operaciones con archivos
pn = Pathname.new("pathname_test.txt")
pn.write("Contenido desde Pathname\n")
puts "\nEscrito: #{pn.read}"

puts "  absolute?: #{pn.absolute?}"
puts "  relative?: #{pn.relative?}"

# relative_path_from
base = Pathname.new("/usr/local")
target = Pathname.new("/usr/local/bin/ruby")
puts "\nrelative_path_from: #{target.relative_path_from(base)}"

# Glob desde Pathname
puts "\nGlob con Pathname:"
Pathname.glob("/etc/host*").each { |f| puts "  #{f}" }

pn.delete if pn.exist?

#------ fin del ejemplo ------

8. PATHNAME AVANZADO
----------------------------------------------------------------------

Pathname ofrece métodos para casi todo:

  # Navegación
  ascend { |path| ... }   # subir hacia la raíz
  descend { |path| ... }  # bajar desde la raíz

  # Operaciones de archivo
  read, write, binread, binwrite
  each_line, each_char, foreach
  delete, rename, stat, ftype

  # Operaciones de directorio
  mkdir, rmdir, children, entries
  find { |path| ... }     # recorrido recursivo

================================================================================
EJEMPLO 8: Pathname avanzado
================================================================================

#------ archivo: ej08_pathname_avanzado.rb ------
require 'pathname'

puts "=== PATHNAME AVANZADO ==="

# Crear estructura
base = Pathname.new("pathname_demo")
base.mkdir unless base.exist?
(base + "src").mkdir unless (base + "src").exist?
(base + "lib").mkdir unless (base + "lib").exist?
(base + "src" + "main.rb").write("puts 'Hello'\n")
(base + "lib" + "core.rb").write("module Core; end\n")

# Recorrido recursivo con find
puts "Recorrido recursivo con find:"
base.find do |entry|
  next if entry == base
  tipo = entry.directory? ? "[DIR]" : "[FILE]"
  puts "  #{tipo} #{entry.relative_path_from(base)}"
end

# ascend
puts "\nAscend desde #{base + 'src' + 'main.rb'}:"
(base + "src" + "main.rb").ascend do |path|
  puts "  #{path}"
  break if path == base
end

# Basename con extensión específica
pn = Pathname.new("documento.txt.bak")
puts "\nVariantes de basename:"
puts "  basename: #{pn.basename}"
puts "  basename('.*'): #{pn.basename('.*')}"
puts "  basename('.bak'): #{pn.basename('.bak')}"

# Subir niveles
ruta = Pathname.new("/a/b/c/d/e.txt")
puts "\nSubir niveles:"
puts "  0: #{ruta}"
puts "  1: #{ruta.parent}"
puts "  2: #{ruta.parent.parent}"
puts "  3: #{ruta.parent.parent.parent}"

# Limpieza
base.rmtree if base.exist?

#------ fin del ejemplo ------

9. FILEUTILS — OPERACIONES DE ALTO NIVEL
----------------------------------------------------------------------

FileUtils es una librería estándar que ofrece operaciones Unix-like
para archivos y directorios:

  cp(origen, destino)      # copiar archivo
  cp_r(origen, destino)    # copiar recursivo
  mv(origen, destino)      # mover/renombrar
  rm(ruta)                 # eliminar archivo
  rm_f(ruta)               # eliminar forzado (no preguntar)
  rm_rf(ruta)              # eliminar recursivo forzado
  mkdir(ruta)              # crear directorio
  mkdir_p(ruta)            # mkdir -p
  touch(ruta)              # crear archivo vacío
  chmod(modo, ruta)        # cambiar permisos
  chown(usuario, grupo, ruta) # cambiar propietario
  compare_file(a, b)       # comparar archivos
  compare_stream(a, b)     # comparar streams

También ofrece "verbosidad" y "no-op":

  FileUtils.verbose(true)  # mostrar comandos
  FileUtils.noop(true)     # solo mostrar, no ejecutar

Operaciones en lotes:
  FileUtils.cp(lista, directorio)  # copiar varios

================================================================================
EJEMPLO 9: FileUtils avanzado
================================================================================

#------ archivo: ej09_fileutils.rb ------
require 'fileutils'

puts "=== FILEUTILS AVANZADO ==="

# Crear estructura de prueba
FileUtils.mkdir_p("fuente/sub1")
FileUtils.mkdir_p("fuente/sub2")
File.write("fuente/a.txt", "Contenido A")
File.write("fuente/b.txt", "Contenido B")
File.write("fuente/sub1/c.txt", "Contenido C")

# cp_r (copia recursiva)
FileUtils.cp_r("fuente", "destino")
puts "Copia recursiva realizada"
puts "  destino/a.txt: #{File.exist?('destino/a.txt')}"
puts "  destino/sub1/c.txt: #{File.exist?('destino/sub1/c.txt')}"

# touch
FileUtils.touch("destino/nuevo.txt")
puts "\ntouch: #{File.exist?('destino/nuevo.txt')}"

# compare_file
iguales = FileUtils.compare_file("fuente/a.txt", "destino/a.txt")
puts "\n¿Archivos iguales? #{iguales}"

# mv (mover)
FileUtils.mv("destino/nuevo.txt", "destino/renombrado.txt")
puts "\nMovido: ¿nuevo.txt? #{File.exist?('destino/nuevo.txt')}"
puts "         ¿renombrado.txt? #{File.exist?('destino/renombrado.txt')}"

# verbose mode
puts "\n--- Modo verbose ---"
FileUtils.verbose(true) do
  FileUtils.touch("destino/verbose_test.txt")
end

# chmod
FileUtils.chmod(0644, "destino/a.txt")
stat = File.stat("destino/a.txt")
puts "\nPermisos de a.txt: #{stat.mode.to_s(8)[-4..]}"

# Limpieza
FileUtils.rm_rf("fuente")
FileUtils.rm_rf("destino")

#------ fin del ejemplo ------

10. RECORRIDO RECURSIVO DE DIRECTORIOS
----------------------------------------------------------------------

A veces necesitas recorrer un árbol de directorios manualmente. Ruby
ofrece varias estrategias:

  1. Dir.glob("**/*")
  2. Pathname#find
  3. Find.find (librería estándar 'find')
  4. Recursión manual

================================================================================
EJEMPLO 10: Recorrido recursivo
================================================================================

#------ archivo: ej10_recorrido_recursivo.rb ------
require 'find'
require 'fileutils'

puts "=== RECORRIDO RECURSIVO ==="

# Crear estructura
FileUtils.mkdir_p("arbol/a1")
FileUtils.mkdir_p("arbol/a2/b1")
FileUtils.mkdir_p("arbol/a2/b2")
File.write("arbol/root.txt", "root")
File.write("arbol/a1/x.txt", "x")
File.write("arbol/a2/b1/y.txt", "y")
File.write("arbol/a2/b2/z.txt", "z")

# Método 1: Dir.glob("**/*")
puts "--- Dir.glob('**/*') ---"
Dir.glob("arbol/**/*").each { |e| puts "  #{e}" }

# Método 2: Pathname#find
puts "\n--- Pathname#find ---"
require 'pathname'
Pathname.new("arbol").find do |entry|
  next if entry.to_s == "arbol"
  puts "  #{entry}"
end

# Método 3: Find.find
puts "\n--- Find.find ---"
Find.find("arbol") do |entry|
  puts "  #{entry}"
end

# Método 4: Recursión manual
puts "\n--- Recursión manual ---"
def recorrer(ruta, nivel = 0)
  Dir.children(ruta).each do |entry|
    full = File.join(ruta, entry)
    puts "#{'  ' * nivel}#{entry}#{File.directory?(full) ? '/' : ''}"
    recorrer(full, nivel + 1) if File.directory?(full)
  end
end
recorrer("arbol")

FileUtils.rm_rf("arbol")

#------ fin del ejemplo ------

12. DIR.COUNT Y DIR.SIZE
----------------------------------------------------------------------

No hay métodos directos para contar archivos, pero es fácil:

  count = Dir.glob("**/*").count { |f| File.file?(f) }
  size = Dir.glob("**/*").sum { |f| File.file?(f) ? File.size(f) : 0 }

================================================================================
EJEMPLO 12: Contar y medir directorios
================================================================================

#------ archivo: ej12_count_size.rb ------
puts "=== CONTAR Y MEDIR DIRECTORIOS ==="

target = "/etc"

if Dir.exist?(target)
  archivos = Dir.glob("#{target}/**/*").select { |f| File.file?(f) }
  dirs = Dir.glob("#{target}/**/*").select { |f| File.directory?(f) }

  puts "Estadísticas de #{target}:"
  puts "  Total entradas: #{archivos.size + dirs.size}"
  puts "  Archivos: #{archivos.size}"
  puts "  Directorios: #{dirs.size}"

  if archivos.any?
    total_bytes = archivos.sum { |f| File.size(f) rescue 0 }
    puts "  Tamaño total: #{total_bytes} bytes " +
         "(#{format('%.2f', total_bytes / 1024.0)} KB)"
  end
end

13. DIR EN EL MUNDO REAL
----------------------------------------------------------------------

Casos de uso comunes de directorios en aplicaciones Ruby:

  a) Rails: Recorrer app/models/**/*.rb para cargar modelos
  b) Gemas: Buscar .gemspec o Gemfile
  c) Logs: Rotar logs en /var/log/
  d) Uploads: Organizar archivos por fecha (2024/01/15/)
  e) Backup: Recorrer y comprimir directorios
  f) Tests: Descubrir *_test.rb o *_spec.rb
  g) Config: Leer todos los YAML en config/

================================================================================
EJEMPLO 13: Caso real - organizador de archivos
================================================================================

#------ archivo: ej13_organizador.rb ------
require 'fileutils'

puts "=== ORGANIZADOR DE ARCHIVOS ==="

# Crear directorio de prueba
src = "downloads_demo"
FileUtils.mkdir_p(src)
%w[foto.jpg imagen.png documento.pdf nota.txt script.rb datos.csv].each do |f|
  File.write("#{src}/#{f}", "contenido")
end

# Organizar por extensión
extensiones = Dir.glob("#{src}/*").each_with_object({}) do |archivo, hash|
  ext = File.extname(archivo).downcase
  (hash[ext] ||= []) << archivo
end

puts "Archivos organizados por extensión:"
extensiones.each do |ext, archivos|
  puts "  #{ext.empty? ? '(sin ext)' : ext}: #{archivos.size} archivo(s)"
end

# Mover a subdirectorios
extensiones.each do |ext, archivos|
  dir_name = ext.empty? ? "sin_extension" : ext.gsub('.', '')
  dest_dir = File.join(src, dir_name)
  FileUtils.mkdir_p(dest_dir)
  archivos.each { |a| FileUtils.mv(a, dest_dir) }
end

puts "\nEstructura resultante:"
Dir.glob("#{src}/**/*").each { |e| puts "  #{e}" }

FileUtils.rm_rf(src)

#------ fin del ejemplo ------

================================================================================
PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Cuál es la diferencia entre Dir.entries y Dir.glob?
R: Dir.entries lista todas las entradas de un directorio específico.
   Dir.glob busca archivos con patrones en múltiples directorios.

P: ¿Cómo elimino un directorio que no está vacío?
R: Usa FileUtils.rm_rf("directorio"). Dir.delete solo funciona en
   directorios vacíos.

P: ¿Qué es Pathname y por qué usarlo?
R: Pathname es una envoltura OO para rutas. Ofrece métodos como
   .join, .basename, .find, .ascend. Es más legible y seguro que
   manipular strings de rutas.

P: ¿Cómo creo directorios anidados?
R: Dir.mkdir crea un nivel. FileUtils.mkdir_p("a/b/c") crea todos
   los niveles intermedios (como mkdir -p).

P: ¿Dir.chdir con bloque es thread-safe?
R: No. Cambia el directorio del proceso globalmente. Si usas hilos,
   usa rutas absolutas en lugar de chdir.

P: ¿Cómo obtengo la ruta absoluta de un directorio?
R: File.expand_path("ruta") o Pathname.new("ruta").realpath.

P: ¿Cómo listo solo archivos (no directorios)?
R: Dir.glob("*").select { |f| File.file?(f) }

P: ¿Cómo busco archivos recursivamente?
R: Dir.glob("**/*.rb") o Pathname.new(".").find.select(&:file?)

P: ¿FileUtils.rm_rf es peligroso?
R: Sí, elimina todo sin confirmación. Ten cuidado al usarlo,
   especialmente con rutas generadas dinámicamente.

P: ¿Cómo cambio los permisos de un directorio?
R: FileUtils.chmod(0755, "directorio")

DESAFÍO DEL ARQUITECTO

Construye una herramienta llamada "DirSync" que sincronice dos
directorios (origen y destino) como un backup inteligente.

Requisitos funcionales:

1. DirSync.sync(origen, destino, opciones = {})
   - Copia archivos nuevos de origen a destino.
   - Sobrescribe archivos modificados (comparando mtime y tamaño).
   - Elimina en destino los archivos que ya no están en origen (si
     opciones[:delete] es true).
   - Respeta la estructura de directorios.

2. DirSync.dry_run(origen, destino)
   - Igual que sync pero no ejecuta nada, solo muestra lo que haría.

3. DirSync.stats(origen, destino)
   - Devuelve Hash con: archivos_origen, archivos_destino,
     archivos_nuevos, archivos_actualizados, archivos_eliminados,
     bytes_origen, bytes_destino, bytes_a_copiar.

4. Opciones:
   - :verbose (Boolean) - muestra cada operación.
   - :delete (Boolean) - elimina archivos en destino no presentes
     en origen.
   - :exclude (Array) - patrones glob a excluir (ej: ["*.log"]).
   - :include (Array) - patrones glob a incluir (por defecto todos).

5. Logging:
   - Escribe un log de la operación en destino/dirsync.log.

Requisitos técnicos:
  - Usa Pathname para todas las rutas.
  - Usa FileUtils para copiar y eliminar.
  - Usa Dir.glob con ** para listar archivos.
  - Usa File.stat para comparar mtime y tamaño.
  - Maneja excepciones (permisos, archivos en uso, etc.).
  - Implementa pruebas unitarias con directorios temporales.

Este desafío integra todos los conceptos del capítulo y simula una
herramienta real de administración de sistemas.
