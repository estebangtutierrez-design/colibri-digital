1. INTRODUCCIÓN A LA CLASE FILE
----------------------------------------------------------------------

En Ruby, todo es un objeto. Los archivos no son la excepción. La clase
File pertenece a la librería estándar y no requiere require alguno para
ser utilizada. File es una subclase de IO, que a su vez es la clase
base para todas las operaciones de entrada y salida.

Cuando trabajamos con archivos, básicamente estamos creando un puente
entre nuestro programa Ruby y el sistema de archivos del sistema operativo.
Este puente nos permite leer, escribir, modificar y eliminar archivos.

El ciclo de vida típico de un archivo en Ruby es:

  1. Apertura (File.open o File.new)
  2. Operaciones (leer, escribir, posicionarse)
  3. Cierre (File#close)

Ruby nos ofrece una forma abreviada y segura mediante un bloque que
automatiza el paso 3.

Veamos nuestro primer ejemplo:

================================================================================
EJEMPLO 1: Apertura básica con File.open
================================================================================

#------ archivo: ej01_apertura_basica.rb ------
puts "=== EJEMPLO 1: File.open básico ==="

# Abrimos un archivo en modo lectura
archivo = File.open("mi_archivo.txt", "r")

# Verificamos que el archivo está abierto
puts "El archivo está abierto: #{!archivo.closed?}"

# Cerramos el archivo manualmente
archivo.close

puts "El archivo está cerrado: #{archivo.closed?}"

#------ fin del ejemplo ------

----------------------------------------------------------------------
EXPLICACIÓN:
----------------------------------------------------------------------

File.open recibe dos argumentos principales:
  - El nombre o ruta del archivo (String o Pathname).
  - El modo de apertura (String o Integer).

Devuelve una instancia de File. Es responsabilidad del programador
cerrar el archivo. Si olvidamos cerrarlo, podríamos tener fugas de
recursos (file descriptors).

----------------------------------------------------------------------
2. MODOS DE APERTURA
----------------------------------------------------------------------

Los modos de apertura determinan qué operaciones podemos realizar
sobre el archivo. Se especifican como una cadena de texto:

  Modo  |  Descripción
  ------+------------------------------------------------------------
  "r"   |  Lectura (debe existir el archivo)
  "w"   |  Escritura (crea o sobrescribe)
  "a"   |  Append (escribe al final, crea si no existe)
  "r+"  |  Lectura y escritura (debe existir)
  "w+"  |  Lectura y escritura (crea o sobrescribe)
  "a+"  |  Lectura y append (crea si no existe)

Podemos agregar 'b' para modo binario: "rb", "wb", etc.
Podemos agregar ':' para especificar codificación: "r:UTF-8".

================================================================================
EJEMPLO 2: Demostración de todos los modos
================================================================================

#------ archivo: ej02_modos_apertura.rb ------
puts "=== DEMOSTRACIÓN DE MODOS DE APERTURA ==="
puts

# Creamos un archivo de prueba
File.write("prueba_modos.txt", "Contenido original\n")

# Modo "r" - lectura
begin
  f = File.open("prueba_modos.txt", "r")
  puts "Modo 'r': #{f.read}"
  f.close
rescue => e
  puts "Error en modo 'r': #{e.message}"
end

# Modo "w" - escritura (sobrescribe)
f = File.open("prueba_modos.txt", "w")
f.puts "Escritura con modo w"
f.close
puts "Modo 'w': archivo sobrescrito"

# Modo "a" - append
f = File.open("prueba_modos.txt", "a")
f.puts "Línea agregada con modo a"
f.close
puts "Modo 'a': línea agregada al final"

# Modo "r+" - lectura/escritura
f = File.open("prueba_modos.txt", "r+")
contenido = f.read
f.puts "Agregado con r+"
f.close
puts "Modo 'r+': leído y escrito"

# Modo "w+" - lectura/escritura (trunca)
f = File.open("prueba_modos.txt", "w+")
f.puts "Nuevo contenido con w+"
f.rewind
puts "Modo 'w+': #{f.read}"
f.close

# Modo "a+" - lectura/append
f = File.open("prueba_modos.txt", "a+")
f.puts "Línea con a+"
f.rewind
puts "Modo 'a+': #{f.read}"
f.close

# Limpieza
File.delete("prueba_modos.txt") if File.exist?("prueba_modos.txt")

#------ fin del ejemplo ------

----------------------------------------------------------------------
EXPLICACIÓN DETALLADA DE CADA MODO:
----------------------------------------------------------------------

-- MODO "r" (READ) --
El modo más seguro. Solo permite lectura. El archivo DEBE existir,
de lo contrario Ruby lanzará Errno::ENOENT. Es el modo predeterminado
si no se especifica uno. Ideal para archivos de configuración, datos
de entrada, plantillas, etc.

-- MODO "w" (WRITE) --
Crea un archivo nuevo o sobrescribe uno existente. Si el archivo ya
existía, su contenido se trunca a cero bytes inmediatamente. Ten
cuidado: no hay confirmación ni papelera de reciclaje. Una vez que
abres con "w", el contenido anterior se pierde para siempre.

-- MODO "a" (APPEND) --
Escribe al final del archivo. Si el archivo no existe, lo crea. El
cursor de escritura siempre se posiciona al final, justo antes del
EOF. No importa si mueves el cursor con #seek; las escrituras siempre
van al final. Ideal para logs y bitácoras.

-- MODO "r+" (READ-WRITE) --
Permite leer y escribir. El archivo DEBE existir. NO trunca el
archivo. El cursor se posiciona al inicio. Puedes leer y luego
escribir, o viceversa. Útil para actualizar archivos existentes
sin perder contenido.

-- MODO "w+" (READ-WRITE TRUNCATE) --
Como "w" pero también permite lectura. Trunca el archivo si existe.
Útil cuando necesitas crear un archivo desde cero y luego leerlo
para verificar su contenido.

-- MODO "a+" (READ-APPEND) --
Permite lectura en cualquier posición, pero la escritura siempre
va al final (como "a"). El archivo se crea si no existe. Útil para
logs que también necesitas consultar.

----------------------------------------------------------------------
3. FILE.OPEN CON BLOQUE (FORMA SEGURA)
----------------------------------------------------------------------

La forma más común y recomendada de trabajar con archivos en Ruby
es usando File.open con un bloque. Al final del bloque, Ruby cierra
el archivo automáticamente, incluso si ocurre una excepción.

Esta es una de las características que hacen que Ruby sea tan
elegante para el manejo de archivos.

================================================================================
EJEMPLO 3: File.open con bloque automático
================================================================================

#------ archivo: ej03_bloque_automatico.rb ------
puts "=== FILE.OPEN CON BLOQUE ==="

File.open("saludo.txt", "w") do |archivo|
  archivo.puts "¡Hola desde Ruby!"
  archivo.puts "Este archivo se cerrará solo."
  archivo.puts "Incluso si ocurre un error."
  puts "Escribiendo dentro del bloque..."
end

# El archivo ya está cerrado aquí
puts "¿Archivo cerrado? No podemos saberlo desde fuera, pero lo está."

# Verificamos que se creó
if File.exist?("saludo.txt")
  contenido = File.read("saludo.txt")
  puts "Contenido del archivo:"
  puts contenido
end

File.delete("saludo.txt") if File.exist?("saludo.txt")

#------ fin del ejemplo ------

----------------------------------------------------------------------
EXPLICACIÓN:
----------------------------------------------------------------------

File.open recibe un bloque. El bloque recibe el objeto File como
argumento. Al terminar el bloque (incluso por raise o return), Ruby
ejecuta automáticamente archivo.close.

Esto es equivalente a:

  archivo = File.open("ruta", "w")
  begin
    archivo.puts "datos"
  ensure
    archivo.close
  end

Pero mucho más conciso. Siempre que puedas, usa la forma con bloque.

----------------------------------------------------------------------
4. FILE.NEW VS FILE.OPEN
----------------------------------------------------------------------

File.new es el constructor tradicional. File.open puede funcionar
como File.new (sin bloque) o con la versión automática (con bloque).

La diferencia principal:
  - File.new: siempre devuelve un objeto File. Debes cerrarlo tú.
  - File.open sin bloque: igual que File.new.
  - File.open con bloque: cierra automáticamente, devuelve nil a
    menos que el bloque tenga un valor de retorno explícito.

================================================================================
EJEMPLO 4: File.new vs File.open
================================================================================

#------ archivo: ej04_new_vs_open.rb ------
puts "=== FILE.NEW VS FILE.OPEN ==="

# File.new
archivo1 = File.new("ejemplo_new.txt", "w")
archivo1.puts "Creado con File.new"
archivo1.close
puts "File.new: archivo creado y cerrado manualmente."

# File.open sin bloque
archivo2 = File.open("ejemplo_open.txt", "w")
archivo2.puts "Creado con File.open sin bloque"
archivo2.close
puts "File.open sin bloque: igual que File.new."

# File.open con bloque
resultado = File.open("ejemplo_bloque.txt", "w") do |f|
  f.puts "Creado con File.open y bloque"
  :completado
end
puts "File.open con bloque: devuelve #{resultado.inspect}"

# Limpieza
%w[ejemplo_new.txt ejemplo_open.txt ejemplo_bloque.txt].each do |f|
  File.delete(f) if File.exist?(f)
end

#------ fin del ejemplo ------

----------------------------------------------------------------------
5. VERIFICACIÓN DE EXISTENCIA
----------------------------------------------------------------------

Antes de operar sobre un archivo, es buena práctica verificar si
existe. Ruby ofrece varios métodos en File y FileTest:

  Método           |  Descripción
  -----------------+---------------------------------------------------
  File.exist?(ruta)|  ¿Existe el archivo o directorio?
  File.file?(ruta) |  ¿Es un archivo regular?
  File.directory?  |  ¿Es un directorio?
  File.readable?   |  ¿Tiene permiso de lectura?
  File.writable?   |  ¿Tiene permiso de escritura?
  File.executable? |  ¿Tiene permiso de ejecución?
  File.size(ruta)  |  Tamaño en bytes
  File.zero?(ruta) |  ¿Está vacío (tamaño cero)?
  File.symlink?    |  ¿Es un enlace simbólico?

================================================================================
EJEMPLO 5: File.exist? y métodos de verificación
================================================================================

#------ archivo: ej05_verificacion.rb ------
puts "=== VERIFICACIÓN DE ARCHIVOS ==="

archivos = ["/etc/passwd", "/etc/hosts", "/no/existe/archivo.txt"]

archivos.each do |ruta|
  puts "--- #{ruta} ---"
  puts "  ¿Existe?      #{File.exist?(ruta)}"
  puts "  ¿Es archivo?  #{File.file?(ruta)}"
  puts "  ¿Es directorio? #{File.directory?(ruta)}"
  puts "  ¿Lectura?     #{File.readable?(ruta)}"
  puts "  ¿Escritura?   #{File.writable?(ruta)}"
  puts "  Tamaño:       #{File.size(ruta)} bytes" if File.exist?(ruta)
  puts
end

#------ fin del ejemplo ------

6. FILE.STAT Y METADATOS
----------------------------------------------------------------------

File.stat devuelve un objeto File::Stat con información detallada:

  stat = File.stat("ruta")
  stat.size       # tamaño en bytes
  stat.mtime      # última modificación (Time)
  stat.atime      # último acceso (Time)
  stat.ctime      # último cambio de metadatos (Time)
  stat.mode       # permisos (integer)
  stat.uid        # ID del usuario propietario
  stat.gid        # ID del grupo propietario
  stat.ftype      # tipo: "file", "directory", "link", etc.
  stat.owned?     # ¿es nuestro?
  stat.file?      # ¿es archivo regular?

================================================================================
EJEMPLO 6: File.stat en acción
================================================================================

#------ archivo: ej06_file_stat.rb ------
puts "=== FILE.STAT ==="

# Creamos un archivo temporal
File.open("stat_ejemplo.txt", "w") { |f| f.puts "Contenido de prueba" }

stat = File.stat("stat_ejemplo.txt")

puts "Metadatos del archivo:"
puts "  Tamaño:     #{stat.size} bytes"
puts "  Tipo:       #{stat.ftype}"
puts "  Modificado: #{stat.mtime}"
puts "  Accedido:   #{stat.atime}"
puts "  Creado:     #{stat.ctime}"
puts "  Permisos:   #{stat.mode.to_s(8)}"
puts "  Propietario UID: #{stat.uid}"
puts "  Grupo GID:      #{stat.gid}"
puts "  ¿Es archivo?    #{stat.file?}"
puts "  ¿Es directorio? #{stat.directory?}"
puts "  ¿Es readable?   #{stat.readable?}"
puts "  ¿Es writable?   #{stat.writable?}"

File.delete("stat_ejemplo.txt") if File.exist?("stat_ejemplo.txt")

#------ fin del ejemplo ------

7. FILE.DELETE Y FILE.RENAME
----------------------------------------------------------------------

Para eliminar y renombrar archivos:

  File.delete("ruta")    # elimina el archivo
  File.unlink("ruta")    # sinónimo de delete
  File.rename("orig", "dest")  # renombra o mueve

================================================================================
EJEMPLO 7: Eliminar y renombrar
================================================================================

#------ archivo: ej07_delete_rename.rb ------

File.rename("original.txt", "renombrado.txt")

end

if File.exist?("renombrado.txt")
  puts "'renombrado.txt' ahora existe"
end

# Eliminar
File.delete("para_borrar.txt")

File.delete("renombrado.txt") if File.exist?("renombrado.txt")

#------ fin del ejemplo ------

----------------------------------------------------------------------
8. RUTAS ABSOLUTAS Y RELATIVAS
----------------------------------------------------------------------

Ruby maneja rutas como strings. Es importante entender la diferencia:

  Ruta absoluta: "/home/usuario/documentos/archivo.txt"
  Ruta relativa: "documentos/archivo.txt"
  Ruta relativa: "./documentos/archivo.txt"
  Directorio actual: "."
  Directorio padre: ".."
  Directorio HOME: "~" (NO funciona directamente, usar Dir.home o ENV["HOME"])

Para trabajar con rutas de forma más segura, existe File.join:

  ruta = File.join("usr", "local", "bin", "ruby")
  # => "usr/local/bin"

Y File.expand_path para convertir relativas en absolutas:

  abs = File.expand_path("documentos/archivo.txt")
  # => "/home/usuario/documentos/archivo.txt"

  abs = File.expand_path("archivo.txt", "/tmp")
  # => "/tmp/archivo.txt"

================================================================================
EJEMPLO 8: Trabajando con rutas
================================================================================

#------ archivo: ej08_rutas.rb ------
puts "=== TRABAJANDO CON RUTAS ==="

# File.join
puts "File.join:"
puts "  #{File.join('usr', 'local', 'bin', 'ruby')}"
puts "  #{File.join('/home', 'usuario', 'docs', 'notas.txt')}"

# File.expand_path
puts "\nFile.expand_path:"
puts "  #{File.expand_path('archivo.txt')}"
puts "  #{File.expand_path('~/docs')}"
puts "  #{File.expand_path('archivo.txt', '/tmp')}"

# Dir.pwd (directorio actual)
puts "\nDirectorio actual: #{Dir.pwd}"

# Dir.home
puts "Directorio home: #{Dir.home}"

# Construir ruta absoluta segura
ruta_base = Dir.home
subcarpeta = "proyectos"
archivo = "config.yml"
ruta_completa = File.join(ruta_base, subcarpeta, archivo)
puts "\nRuta construida: #{ruta_completa}"

# Obtener partes de una ruta
ruta = "/home/usuario/documentos/archivo.txt"
puts "\nPartes de '#{ruta}':"
puts "  Dirname:  #{File.dirname(ruta)}"
puts "  Basename: #{File.basename(ruta)}"
puts "  Extname:  #{File.extname(ruta)}"
puts "  Basename sin ext: #{File.basename(ruta, File.extname(ruta))}"

#------ fin del ejemplo ------

9. FILE::CONSTANTS
----------------------------------------------------------------------

La clase File define constantes para modos numéricos:

  File::RDONLY   # 0 - solo lectura
  File::WRONLY   # 1 - solo escritura
  File::RDWR     # 2 - lectura y escritura
  File::APPEND   # 8 - append
  File::CREAT    # 64 - crear si no existe
  File::TRUNC    # 512 - truncar si existe
  File::EXCL     # 2048 - fallar si existe
  File::NOCTTY   # no asignar terminal
  File::BINARY   # modo binario

Se combinan con operador binario OR (|):

  modo = File::RDWR | File::CREAT | File::TRUNC
  File.open("archivo", modo)

Pero en la práctica, casi siempre usarás los modos como strings.

================================================================================
EJEMPLO 9: Constantes de File
================================================================================

#------ archivo: ej09_constantes.rb ------
puts "=== CONSTANTES DE FILE ==="

puts "File::RDONLY = #{File::RDONLY}"
puts "File::WRONLY = #{File::WRONLY}"
puts "File::RDWR   = #{File::RDWR}"
puts "File::APPEND = #{File::APPEND}"
puts "File::CREAT  = #{File::CREAT}"
puts "File::TRUNC  = #{File::TRUNC}"

# Equivalencias
puts "\nEquivalencia de modos string vs constantes:"
puts "  'w'  = WRONLY | CREAT | TRUNC  = #{File::WRONLY | File::CREAT | File::TRUNC}"
puts "  'a'  = WRONLY | CREAT | APPEND  = #{File::WRONLY | File::CREAT | File::APPEND}"
puts "  'r'  = RDONLY                    = #{File::RDONLY}"
puts "  'w+' = RDWR | CREAT | TRUNC     = #{File::RDWR | File::CREAT | File::TRUNC}"
puts "  'a+' = RDWR | CREAT | APPEND     = #{File::RDWR | File::CREAT | File::APPEND}"
puts "  'r+' = RDWR                      = #{File::RDWR}"

#------ fin del ejemplo ------

10. CODIFICACIÓN Y ARCHIVOS BINARIOS
----------------------------------------------------------------------

Ruby 3+ maneja codificaciones de forma nativa. Puedes especificar la
codificación al abrir el archivo:

  File.open("archivo.txt", "r:UTF-8")
  File.open("archivo.txt", "r:UTF-8:ISO-8859-1") # de UTF-8 a Latin-1

Para archivos binarios:

  File.open("imagen.jpg", "rb")
  File.open("imagen.jpg", "wb")

La 'b' en el modo evita que el sistema operativo haga traducciones
de fin de línea (importante en Windows).

================================================================================
EJEMPLO 10: Codificación y binarios
================================================================================

#------ archivo: ej10_codificacion.rb ------
puts "=== CODIFICACIÓN Y ARCHIVOS BINARIOS ==="

# Archivo con codificación UTF-8
File.open("utf8_ejemplo.txt", "w:UTF-8") do |f|
  f.puts "Caracteres especiales: ñ, á, é, í, ó, ú, ü, ¿, ¡"
  f.puts "Emojis: 🚀 Ruby es increíble"
end

# Leer con codificación específica
contenido = File.read("utf8_ejemplo.txt", encoding: "UTF-8")
puts "Contenido leído: #{contenido}"

# Verificar codificación
puts "Codificación del string: #{contenido.encoding}"

# Archivo binario - simulación
File.open("datos.bin", "wb") do |f|
  # Escribir bytes
  f.write([0x48, 0x65, 0x6C, 0x6C, 0x6F].pack("C*"))
end

datos = File.open("datos.bin", "rb") { |f| f.read }
puts "Datos binarios: #{datos.inspect}"
puts "Bytes: #{datos.bytes.inspect}"

# Limpieza
File.delete("utf8_ejemplo.txt") if File.exist?("utf8_ejemplo.txt")
File.delete("datos.bin") if File.exist?("datos.bin")

#------ fin del ejemplo ------

----------------------------------------------------------------------
11. ERRORS COMUNES
----------------------------------------------------------------------

Trabajar con archivos implica lidiar con errores del sistema operativo.
Ruby define excepciones en el módulo Errno:

  Errno::ENOENT     # "No such file or directory"
  Errno::EACCES     # "Permission denied"
  Errno::EISDIR     # "Is a directory"
  Errno::ENOSPC     # "No space left on device"
  Errno::EINVAL     # "Invalid argument"
  Errno::EEXIST     # "File exists"

================================================================================
EJEMPLO 11: Manejo de errores en archivos
================================================================================

#------ archivo: ej11_errores.rb ------
puts "=== MANEJO DE ERRORES EN ARCHIVOS ==="

# Error típico: archivo no existe
begin
  File.open("/no/existe/archivo.txt", "r")
rescue Errno::ENOENT => e
  puts "Error ENOENT: #{e.message}"
  puts "  Código: #{e.errno}"
end

# Error típico: permisos
begin
  File.open("/etc/shadow", "r")
rescue Errno::EACCES => e
  puts "Error EACCES: #{e.message}"
rescue Errno::ENOENT => e
  puts "Error ENOENT (en algunos sistemas): #{e.message}"
end

# Error: archivo ya existe al crear
begin
  File.open("/tmp/test_exist.txt", "w") { |f| f.puts "test" }
  File.open("/tmp/test_exist.txt", File::RDWR | File::CREAT | File::EXCL)
rescue Errno::EEXIST => e
  puts "Error EEXIST: #{e.message}"
ensure
  File.delete("/tmp/test_exist.txt") if File.exist?("/tmp/test_exist.txt")
end

#------ fin del ejemplo ------

12. FILE.TRUNCATE Y FILE.SIZE
----------------------------------------------------------------------

Puedes truncar un archivo a un tamaño específico:

  File.truncate("ruta", longitud)

Y obtener su tamaño:

  File.size("ruta")
  File.size?("ruta")  # nil si está vacío, tamaño si no

================================================================================
EJEMPLO 12: Truncar y medir
================================================================================

#------ archivo: ej12_truncate.rb ------
puts "=== TRUNCATE Y SIZE ==="

File.open("truncar_ejemplo.txt", "w") do |f|
  f.puts "Línea 1: Este es un archivo de prueba"
  f.puts "Línea 2: Vamos a truncarlo"
  f.puts "Línea 3: Solo quedará la primera parte"
  f.puts "Línea 4: Adiós mundo cruel"
end

puts "Tamaño original: #{File.size("truncar_ejemplo.txt")} bytes"
puts "File.size?: #{File.size?("truncar_ejemplo.txt")}"

# Truncar a 20 bytes
File.truncate("truncar_ejemplo.txt", 20)
puts "Tamaño después de truncar: #{File.size("truncar_ejemplo.txt")} bytes"

contenido = File.read("truncar_ejemplo.txt")
puts "Contenido truncado: #{contenido.inspect}"

File.delete("truncar_ejemplo.txt") if File.exist?("truncar_ejemplo.txt")

#------ fin del ejemplo ------

#------ archivo: ej13_fileutils.rb ------
require 'fileutils'

puts "=== COPIA Y MOVIMIENTO CON FILEUTILS ==="

# Crear archivo original
File.write("original.txt", "Contenido importante")

# Copiar
FileUtils.cp("original.txt", "copia.txt")
puts "Archivo copiado: 'copia.txt'"

# Mover (renombrar)
FileUtils.mv("copia.txt", "movido.txt")
puts "Archivo movido: 'movido.txt'"

# Copia segura (no sobrescribe)
begin
  FileUtils.cp("original.txt", "movido.txt")
rescue Errno::EEXIST => e
  puts "Error al copiar: #{e.message}"
end

# cp con opciones
FileUtils.cp("original.txt", "movido.txt", preserve: true)
puts "Copiado con preserve: true"

# Múltiples archivos
FileUtils.cp(["original.txt", "movido.txt"], "/tmp/")
puts "Archivos copiados a /tmp/"

# Limpieza
File.delete("original.txt") if File.exist?("original.txt")
File.delete("movido.txt") if File.exist?("movido.txt")
File.delete("/tmp/original.txt") if File.exist?("/tmp/original.txt")
File.delete("/tmp/movido.txt") if File.exist?("/tmp/movido.txt")

================================================================================
PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Cuál es la diferencia entre File.new y File.open?
R: File.new siempre devuelve un objeto File. File.open sin bloque
   también, pero con bloque cierra automáticamente. La forma con bloque
   es la recomendada.

P: ¿Qué pasa si olvido cerrar un archivo?
R: Los file descriptors no se liberan hasta que el objeto se recolecta
   como basura o el programa termina. En sistemas Unix hay un límite
   (ulimit -n). Siempre usa bloque o ensure.

P: ¿File.exist? y File.file? son lo mismo?
R: No. File.exist? devuelve true para archivos y directorios.
   File.file? solo devuelve true para archivos regulares.

P: ¿Cómo abro un archivo en modo binario en Windows?
R: Agrega 'b' al modo: "rb", "wb", "ab". En Unix/Linux/Mac la 'b' es
   ignorada porque no hay distinción entre texto y binario.

P: ¿Puedo leer un archivo mientras otro proceso lo escribe?
R: Sí, pero puedes obtener datos inconsistentes. Usa bloqueos de archivo
   (File.flock) para acceso concurrente seguro.

P: ¿Cuál es el límite de tamaño de archivo que puede manejar Ruby?
R: Depende de la memoria disponible. File.read carga todo el archivo en
   memoria. Para archivos grandes, usa IO#each_line o read parcial.

P: ¿Qué significa Errno::ENOENT?
R: Es un error del sistema operativo: "No such file or directory".
   ENOENT = Error NO ENTry (entrada no encontrada).

P: ¿File.open es thread-safe?
R: No intrínsecamente. Cada thread puede tener su propio file descriptor.
   Para acceso compartido, usa mutex o flock.

implementar un módulo llamado BackupManager que ofrezca las siguientes

  1. backup_file(origen, destino): copia un archivo de origen a destino,
     con las siguientes características:
     - Si destino existe, lo renombra añadiendo .bak antes de sobrescribir.
     - Preserva los metadatos (mtime, atime) en la copia.

  2. safe_delete(ruta): elimina un archivo de forma segura:
     - Si el archivo existe, lo mueve a ~/.trash/ con un timestamp.
     - Si ~/.trash/ no existe, lo crea.

  - Usa Digest::MD5 para el checksum.

  - Implementa al menos 3 pruebas unitarias (comentadas al final).
