► 1. EL PANORAMA DE SEGURIDAD EN C++

  C++ ocupa un lugar especial en seguridad: da control total sobre
  la memoria, pero también total responsabilidad. Vulnerabilidades
  comunes:

  • Buffer Overflow: Escribir más allá de los límites de un array.
  • Format String: Usar entrada del usuario como formato de printf.
  • Integer Overflow: Operaciones aritméticas que desbordan.
  • Use-After-Free: Usar memoria ya liberada.
  • Double Free: Liberar la misma memoria dos veces.
  • Null Pointer Dereference: Acceder a memoria nullptr.
  • Type Confusion: Tratar un tipo como si fuera otro.
  • Race Conditions: Acceso concurrente sin sincronización.

  C++ moderno mitiga muchas de estas con:
  • RAII y smart pointers (eliminan use-after-free y leaks).
  • std::array y std::vector (bounds checking en debug).
  • std::string (sin buffer overflow).
  • static_cast/dynamic_cast (type safety).
  • constexpr (validación en compilación).

*/

/*
  Un buffer overflow ocurre cuando se escriben más datos de los
  que el buffer puede contener. En C clásico, esto no se detecta.
  En C++ moderno, usamos contenedores seguros.
*/

// VULNERABLE: En C, este código puede ser explotado
void vulnerable_gets() {
    char buffer[16];
    // gets(buffer);  // ¡NUNCA USAR! No hay límite

    // fgets es más seguro, pero hay que calcular el tamaño
    // fgets(buffer, sizeof(buffer), stdin);

    // En C++ moderno: std::string directamente
    std::string input;
    std::getline(std::cin, input);
}

// VULNERABLE: strcpy sin verificar tamaño
void vulnerable_strcpy(const char* entrada) {
    char buffer[32];
    // strcpy(buffer, entrada);  // Si entrada > 32, overflow!

    // Seguro:
    strncpy(buffer, entrada, sizeof(buffer) - 1);
    buffer[sizeof(buffer) - 1] = '\0';
}

// VULNERABLE: sprintf sin límite
void vulnerable_sprintf(const char* nombre, int edad) {
    char buffer[64];
    // sprintf(buffer, "Nombre: %s, Edad: %d", nombre, edad);

    // Seguro:
    snprintf(buffer, sizeof(buffer), "Nombre: %s, Edad: %d",
             nombre, edad);

    // En C++:
    std::string seguro = std::format("Nombre: {}, Edad: {}",
                                      nombre, edad);
}

// VULNERABLE: acceso a array sin límite
int vulnerable_array_access(const std::vector<int>& v, int idx) {
    // return v[idx];  // Si idx < 0 o >= v.size(), undefined behavior

    // Seguro:
    if (idx < 0 || static_cast<size_t>(idx) >= v.size()) {
        throw std::out_of_range("Índice fuera de rango");
    }
    return v[idx];

    // O usar v.at(idx) que lanza excepción
}

/*
  Los enteros con signo tienen UB al desbordar. Los unsigned
  wraparound, pero el resultado puede ser inesperado.
*/

void integer_overflow_ejemplos() {
    // Caso 1: Signed overflow (UB)
    int max_int = std::numeric_limits<int>::max();
    // int resultado = max_int + 1;  // UB!

    // Caso 2: Unsigned wraparound (definido pero peligroso)
    unsigned int u_max = std::numeric_limits<unsigned int>::max();
    unsigned int u_result = u_max + 1;  // u_result = 0

    // Caso 3: En cálculos de tamaño
    size_t tamano = 0;
    // size_t buffer_size = tamano - 1;  // wraparound: número enorme!

    // Caso 4: Asignación a tipo más pequeño
    int grande = 100000;
    short chico = static_cast<short>(grande);  // Pérdida de datos
}

// Función segura para sumar con detección de overflow
template<typename T>
bool sumar_seguro(T a, T b, T& resultado) {
    if constexpr (std::is_signed_v<T>) {
        if ((b > 0) && (a > std::numeric_limits<T>::max() - b)) {
            return false; // Overflow positivo
        }
        if ((b < 0) && (a < std::numeric_limits<T>::min() - b)) {
            return false; // Overflow negativo
        }
    } else {
        if (a > std::numeric_limits<T>::max() - b) {
            return false; // Overflow unsigned
        }
    }
    resultado = a + b;
    return true;
}

void ejemplo_sumar_seguro() {
    int a = 2'000'000'000;
    int b = 500'000'000;
    int resultado;

    if (sumar_seguro(a, b, resultado)) {
        std::cout << "Suma: " << resultado << "\n";
    } else {
        std::cout << "Overflow detectado!\n";
    }

    // C++20: std::add_sat, std::add overflow checking
    // #include <numeric>

// EJEMPLO 4: SMART POINTERS vs RAW POINTERS
//==============================================================================

/*
  Los smart pointers son la defensa principal contra:
  • Memory leaks (olvidar delete)
  • Double free (dos delete)
  • Use-after-free (usar tras delete)
  • Excepciones que saltan el delete

  REGLA: No uses new/delete explícitos. Usa:
  • std::unique_ptr: Propiedad exclusiva (no copiable, movible).
  • std::shared_ptr: Propiedad compartida (reference counting).
  • std::weak_ptr: Observador sin propiedad (rompe ciclos).
*/

// MAL: Raw pointers
class GestorRaw {
    int* datos_;
    int* buffer_;
public:
    GestorRaw() {
        datos_ = new int[100];
        buffer_ = new int[500];
        // Si new int[500] lanza, datos_ se fuga
    }

    ~GestorRaw() {
        delete[] datos_;
        delete[] buffer_;
    }

    // Regla de 3/5 violada: si se copia, doble delete!
};

// BIEN: Smart pointers
class GestorSmart {
    std::unique_ptr<int[]> datos_;
    std::unique_ptr<int[]> buffer_;
public:
    GestorSmart()
        : datos_(std::make_unique<int[]>(100))
        , buffer_(std::make_unique<int[]>(500))
    {}
    // Destructor automático, move semantics, sin fugas
};

// shared_ptr vs unique_ptr
class Recurso {
public:
    explicit Recurso(int id) : id_(id) {
        std::cout << "Recurso " << id << " creado\n";
    }
    ~Recurso() {
        std::cout << "Recurso " << id_ << " destruido\n";
    }
    int id() const { return id_; }
private:
    int id_;
};

void ejemplo_smart_pointers() {
    // unique_ptr: propiedad única
    auto ptr1 = std::make_unique<Recurso>(1);
    // auto ptr2 = ptr1;  // Error: no copiable
    auto ptr2 = std::move(ptr1);  // Transferencia de propiedad

    // shared_ptr: propiedad compartida
    auto sp1 = std::make_shared<Recurso>(2);
    {
        auto sp2 = sp1;  // Referencia contada
        std::cout << "Usos: " << sp1.use_count() << "\n";  // 2
    }
    std::cout << "Usos: " << sp1.use_count() << "\n";  // 1

    // weak_ptr: observador
    std::weak_ptr<Recurso> wp = sp1;
    if (auto locked = wp.lock()) {
        std::cout << "Recurso aún vivo: " << locked->id() << "\n";
    }
}

void bounds_checking() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // operator[] sin check (rápido, peligroso si no verificas)
    int x = v[2];  // OK si sabes que 2 está en rango

    // at() con check (lanza std::out_of_range)
    try {
        int y = v.at(10);  // Lanza excepción
    } catch (const std::out_of_range& e) {
        std::cerr << "Error: " << e.what() << "\n";
    }

    // Iteradores seguros
    for (auto it = v.begin(); it != v.end(); ++it) {
        // *it es seguro
    }

    // Range-for: siempre seguro
    for (int val : v) {
        std::cout << val;
    }

    // C++20: std::span con bounds checking
    std::span<int> s(v);
    // s[10];  // undefined behavior sin checks
    // s.at(10);  // Esto no existe en span

    // std::array: tamaño fijo en compilación
    std::array<int, 5> arr = {1, 2, 3, 4, 5};
    // arr.at(10);  // Lanza out_of_range
}

    std::string a = "Hello ";
    std::string b = "World";
    std::string c = a + b;  // Sin riesgo de overflow

    // Substring con check
    try {
        std::string sub = cpp_str.substr(0, 100);  // Trunca al tamaño real
    } catch (const std::out_of_range& e) {
        // Esto no pasa porque substr trunca, no lanza
    }
}

#include <fstream>

class GestorArchivoSeguro {
public:
    void escribir(const std::string& ruta,
                  const std::vector<char>& datos) {
        // Validar la ruta (path traversal)
        if (ruta.find("..") != std::string::npos) {
            throw std::runtime_error("Path traversal detectado");
        }

        // Validar tamaño máximo
        if (datos.size() > MAX_TAMANO) {
            throw std::runtime_error("Archivo demasiado grande");
        }

        std::ofstream archivo(ruta, std::ios::binary);
        if (!archivo) {
            throw std::runtime_error(
                "No se pudo abrir el archivo");
        }

        archivo.write(datos.data(), datos.size());
        if (!archivo) {
            throw std::runtime_error(
                "Error al escribir en el archivo");
        }
    }

    std::vector<char> leer(const std::string& ruta) {
        // Validar ruta
        if (ruta.find("..") != std::string::npos) {
            throw std::runtime_error("Path traversal detectado");
        }

        std::ifstream archivo(ruta, std::ios::binary | std::ios::ate);
        if (!archivo) {
            throw std::runtime_error("Archivo no encontrado");
        }

        auto tamano = archivo.tellg();
        if (tamano > MAX_TAMANO) {
            throw std::runtime_error("Archivo demasiado grande");
        }

        archivo.seekg(0);
        std::vector<char> buffer(tamano);
        archivo.read(buffer.data(), tamano);

        if (!archivo) {
            throw std::runtime_error("Error de lectura");
        }

        return buffer;
    }

private:
    static constexpr size_t MAX_TAMANO = 10 * 1024 * 1024; // 10MB
};

// EJEMPLO 9: SANITIZACIÓN DE ENTRADA

/*
  Nunca confíes en la entrada del usuario. Valida y sanitiza.
*/

class Sanitizador {
public:
    // Eliminar caracteres peligrosos para HTML
    static std::string sanitizar_html(const std::string& input) {
        std::string resultado;
        resultado.reserve(input.size());

        for (char c : input) {
            switch (c) {
                case '<': resultado += "&lt;"; break;
                case '>': resultado += "&gt;"; break;
                case '&': resultado += "&amp;"; break;
                case '"': resultado += "&quot;"; break;
                case '\'': resultado += "&#39;"; break;
                default: resultado += c;
            }
        }
        return resultado;
    }

    // Validar que sea un correo electrónico válido
    static bool validar_email(const std::string& email) {
        // Verificar @
        auto arroba = email.find('@');
        if (arroba == std::string::npos || arroba == 0
            || arroba == email.size() - 1) {
            return false;
        }

        // Verificar punto después de @
        auto punto = email.find('.', arroba);
        if (punto == std::string::npos
            || punto == arroba + 1
            || punto == email.size() - 1) {
            return false;
        }

        // Verificar caracteres válidos
        for (char c : email) {
            if (!std::isalnum(c) && c != '@' && c != '.'
                && c != '_' && c != '-') {
                return false;
            }
        }

        return true;
    }

    // Validar que un entero esté en rango
    template<typename T>
    static bool validar_rango(T valor, T min, T max) {
        return valor >= min && valor <= max;
    }
};

/*
  C++ ofrece 4 casts, cada uno con un propósito específico.
  Evita los C-style casts (type*)expr.
*/

void casting_seguro() {
    double d = 3.14;

    // static_cast: conversiones relacionadas (seguro en compilación)
    int i = static_cast<int>(d);  // Trunca, pero es explícito

    // dynamic_cast: downcasting polimórfico (seguro en runtime)
    class Base { virtual void f() {} };
    class Derivada : public Base { void f() override {} };

    Base* b = new Derivada();
    Derivada* d_ptr = dynamic_cast<Derivada*>(b);
    if (d_ptr) {
        // Cast exitoso
    } else {
        // Cast falló (b no apuntaba a Derivada)
    }

    // const_cast: quitar const (peligroso, usado para APIs legacy)
    const int constante = 42;
    int& ref = const_cast<int&>(constante);  // No modificar!

    // reinterpret_cast: convertir entre tipos no relacionados
    uintptr_t dir = reinterpret_cast<uintptr_t>(b);  // Puntero a entero
}

// EJEMPLO 13: EVITAR CICLOS DE REFERENCIA

/*
  shared_ptr puede crear ciclos de referencia que nunca se liberan.
  weak_ptr rompe estos ciclos.
*/

// MAL: Ciclo de shared_ptr
struct NodeMal {
    std::shared_ptr<NodeMal> next;
    ~NodeMal() { std::cout << "Node destruido\n"; }
};

// BIEN: weak_ptr para evitar ciclo
struct NodeBien {
    std::shared_ptr<NodeBien> next;
    std::weak_ptr<NodeBien> prev;  // Observa, no propietario
    ~NodeBien() { std::cout << "NodeBien destruido\n"; }
};

void ejemplo_ciclos() {
    // Ciclo MAL:
    auto a = std::make_shared<NodeMal>();
    auto b = std::make_shared<NodeMal>();
    a->next = b;
    b->next = a;  // Ciclo: nunca se destruyen!
    // Fuga de memoria silenciosa

    // Ciclo BIEN:
    auto c = std::make_shared<NodeBien>();
    auto d = std::make_shared<NodeBien>();
    c->next = d;
    d->prev = c;  // weak_ptr, no hay ciclo
}

/*

/*
  P: ¿C++ es inseguro por naturaleza?
  R: C++ da herramientas para ser seguro (RAII, smart pointers,
     contenedores). El peligro está en usar APIs de C y raw
     pointers. C++ moderno es tan seguro como cualquier lenguaje.

  P: ¿Debo evitar completamente los raw pointers?
  R: En la medida de lo posible. Úsalos solo para observar
     (non-owning) y en interfaces de C legacy. unique_ptr/shared_ptr
     para ownership.

  P: ¿std::string es siempre seguro contra buffer overflow?
  R: Sí, std::string crece automáticamente. Sin embargo, ten
     cuidado al usar .data() con APIs de C que esperan buffers
     de tamaño fijo.

  P: ¿Los smart pointers tienen overhead?
  R: unique_ptr tiene CERO overhead vs raw pointer. shared_ptr
     tiene overhead por el reference count (generalmente 2 palabras
     adicionales). Es mínimo para la seguridad que ofrece.

  P: ¿Cómo protejo mi código de inyección SQL?
  R: Usando prepared statements (parámetros, nunca concatenar
     strings). En C++, librerías como sqlpp11, SOCI, o el driver
     de PostgreSQL/MySQL con parametrización.

  P: ¿Qué es ASLR y cómo me ayuda?
  R: Address Space Layout Randomization. Aleatoriza las direcciones
     de memoria para dificultar exploits de buffer overflow.
     El sistema operativo lo maneja, no el código.

  P: ¿Debo usar std::optional para evitar nullptr?
  R: Sí. std::optional<T> indica claramente que un valor puede
     estar ausente, evitando la necesidad de punteros nulos.

  P: ¿Qué herramientas de seguridad debo usar?
  R: Además del análisis estático (capítulo 74):
     • -fstack-protector-strong (protección contra stack smashing)
     • -D_FORTIFY_SOURCE=2 (fortificación de funciones)
     • -fsanitize=address (detección de memory errors)
     • Valgrind (análisis dinámico)
*/
