► 1. EL VIAJE — RESUMEN DE LOS 8 MÓDULOS

  MÓDULO I: FUNDAMENTOS
  ---------------------
  • Variables, tipos, operadores.
  • Flujo de control: if, switch, for, while.
  • Funciones, parámetros, sobrecarga.
  • Arrays, strings, punteros básicos.

  Lo que sabías entonces: "int x = 5;"
  Lo que sabes ahora: "template<typename T, size_t N> struct Array {...}"

  MÓDULO II: PROGRAMACIÓN ORIENTADA A OBJETOS
  --------------------------------------------
  • Clases, objetos, encapsulación.
  • Herencia, polimorfismo, virtual.
  • Constructores, destructores, RAII.
  • Friend, static, const.

  Lo que sabías entonces: "class Perro { ... };"
  Lo que sabes ahora: "template<typename Derived> class AnimalCRTP {...}"

  MÓDULO III: STL Y GENÉRICOS
  ----------------------------
  • Templates, especialización, variádicos.
  • Contenedores: vector, map, set, unordered_map.
  • Iteradores, algoritmos, ranges.
  • std::string, string_view.

  Lo que sabías entonces: "std::vector<int> v = {1,2,3};"
  Lo que sabes ahora: "std::ranges::sort(v, std::greater{});"

  MÓDULO IV: TÉCNICAS AVANZADAS
  ------------------------------
  • Move semantics, forwarding, perfect forwarding.
  • Lambdas, std::function, std::bind.
  • Smart pointers: unique, shared, weak.
  • Type erasure, any, variant, optional.

  Lo que sabías entonces: "void f(int x) { ... }"
  Lo que sabes ahora: "auto f = [](auto&&... args) -> decltype(auto) {...}"

  MÓDULO V: MANEJO DE ERRORES Y CONCURRENCIA
  -------------------------------------------
  • Excepciones, stack unwinding, noexcept.
  • Hilos, mutex, condition_variable.
  • async, future, promise.
  • Programación concurrente segura.

  Lo que sabías entonces: "¡El programa crasheó!"
  Lo que sabes ahora: "std::scoped_lock lock(m1, m2); // sin deadlock"

  MÓDULO VI: HERRAMIENTAS DEL OFICIO
  -----------------------------------
  • CMake, Makefile, compilación.
  • Git, GitHub, CI/CD.
  • Testing con Google Test.
  • LSP, editores, depuradores.

  Lo que sabías entonces: "g++ programa.cpp"
  Lo que sabes ahora: "cmake -B build && cmake --build build && ctest"

  MÓDULO VII: TEMPLATES AVANZADOS Y METAPROGRAMACIÓN
  ---------------------------------------------------
  • Traits, SFINAE, concepts (C++20).
  • Variadic templates, fold expressions.
  • constexpr, consteval, constinit.
  • Metaprogramación práctica.

  Lo que sabías entonces: "template<typename T>"
  Lo que sabes ahora: "template<Container C> requires std::ranges::range<C>"

  MÓDULO VIII: PROYECTO PROFESIONAL
  ----------------------------------
  • Arquitectura (capas, hexagonal, limpia).
  • Patrones de diseño (Singleton, Factory, Observer, Strategy, CRTP).
  • Debugging con GDB.
  • Análisis estático (cppcheck, clang-tidy).
  • Optimización (-O2, constexpr, move, cache-friendly).
  • Seguridad (smart pointers, bounds checking, sanitizers).
  • Buenas prácticas (estilo, Doxygen, const-correctness).
  • Proyecto final y graduación.

  Lo que eras entonces: Un estudiante de C++.
  Lo que eres ahora: ¡UN ARQUITECTO C++!

► 4. HOJA DE RUTA — ¿AHORA QUÉ?

  Felicitaciones, Arquitecto. Has completado el curso. Pero el
  aprendizaje en C++ nunca termina. Aquí tienes una guía de
  los siguientes pasos:

  MES 1-2: CONSOLIDACIÓN
  -----------------------
  • Implementa 3 proyectos personales pequeños pero completos.
  • Lee "Effective Modern C++" de Scott Meyers.
  • Contribuye a un proyecto open source (busca issues "good first issue").
  • Practica code review con otros desarrolladores.

  MES 3-4: ESPECIALIZACIÓN
  -------------------------
  Elige un área y profudiza:

  ▸ GAME DEVELOPMENT:
    • Aprende Unreal Engine (C++).
    • Lee "Game Programming Patterns" de Robert Nystrom.
    • Proyecto: Clon de un juego clásico con SFML/SDL.

  ▸ SISTEMAS EMBEBIDOS/IoT:
    • Aprende Arduino o ESP32 en C++.
    • Lee "Real-Time C++" de Christopher Kormanyos.
    • Proyecto: Sistema de monitoreo con sensores.

  ▸ FINANZAS/TRADING:
    • Aprende sobre latencia y optimización extrema.
    • Lee "Developing High-Frequency Trading Systems".
    • Proyecto: Backtester de estrategias.

  ▸ MOTORES GRÁFICOS:
    • Aprende Vulkan o DirectX 12.
    • Lee "Ray Tracing in One Weekend".
    • Proyecto: Renderizador 3D básico.

  ▸ COMPILADORES:
    • Aprende LLVM/Clang internals.
    • Lee "Engineering a Compiler".
    • Proyecto: Frontend para un lenguaje pequeño.

  MES 5-6: MAESTRÍA
  ------------------
  • Estudia en detalle C++23/26 nuevas features.
  • Escribe tu propia librería y publícala (header-only).
  • Prepara entrevistas técnicas (LeetCode en C++).
  • Da una charla técnica (meetup, conferencia).
  • Certifícate (si hay certificaciones disponibles).

  MES 7-12: CONTRIBUCIÓN
  -----------------------
  • Conviértete en mantenedor de un proyecto open source.
  • Escribe artículos técnicos (blog, Medium, Dev.to).
  • Mentoriza a otros desarrolladores.
  • Asiste a CppCon o Meeting C++.
  • Considera contribuir al estándar de C++ (papeles, propuestas).

  LECTURAS RECOMENDADAS (POST-CURSO):
  ------------------------------------
  • "Effective Modern C++" — Scott Meyers
  • "Effective C++" — Scott Meyers
  • "C++ Templates: The Complete Guide" — Vandevoorde, Josuttis, Gregor
  • "The C++ Programming Language" — Bjarne Stroustrup
  • "C++ Concurrency in Action" — Anthony Williams
  • "Design Patterns: Elements of Reusable OO Software" — GoF
  • "Clean Code" — Robert C. Martin
  • "Large-Scale C++ Software Design" — John Lakos
  • "Beautiful Architecture" — Diomidis Spinellis
  • "A Tour of C++" — Bjarne Stroustrup

  PROYECTOS OPEN SOURCE PARA CONTRIBUIR:
  --------------------------------------
  • LLVM/Clang — El compilador
  • Catch2 — Testing framework
  • fmtlib — String formatting
  • spdlog — Logging
  • nlohmann/json — JSON parser
  • Boost — La librería de librerías
  • TensorFlow C++ — Machine Learning
  • Godot Engine — Game engine
  • SerenityOS — Operating system en C++
  • SQLite — Base de datos
