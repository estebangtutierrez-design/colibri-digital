Bienvenido al capítulo sobre std::deque (double-ended queue). deque
es un contenedor secuencial que combina las ventajas de vector y list:
acceso aleatorio O(1) e inserción/eliminación O(1) en ambos extremos.
Es ideal para colas, buffers circulares y algoritmos que trabajan
en ambos frentes.

std::deque (double-ended queue) es un contenedor secuencial que
ofrece inserción y eliminación en tiempo constante O(1) tanto al
inicio como al final, además de acceso aleatorio O(1). Internamente,
deque se implementa como una secuencia de bloques contiguos de
memoria (un arreglo de arreglos), lo que permite crecimiento en
ambas direcciones sin reubicar todos los elementos.

Estructura interna simplificada:
  deque almacena elementos en bloques de tamaño fijo. Un mapa
  central (arreglo de punteros) apunta a cada bloque. Cuando se
  necesita crecer por un extremo, se asigna un nuevo bloque y se
  agrega al mapa. Esto evita la reasignación masiva de vector.

Características principales:
- Inserción/eliminación O(1) en ambos extremos.
- Acceso aleatorio O(1) mediante operator[].
- Sin necesidad de memoria contigua para todos los elementos.
- Más lento que vector en acceso aleatorio (una indirección extra).
- Inserción en medio O(n).

La cabecera necesaria es <deque>.

Declaración básica:
  std::deque<int> d;              // deque vacía
  std::deque<int> d(10);          // 10 enteros inicializados a 0
  std::deque<int> d(10, 5);       // 10 enteros con valor 5
  std::deque<int> copia = d;      // copia
  std::deque<std::string> palabras {"Hola", "Mundo", "C++"};

push_front y push_back:
  std::deque<int> d;
  d.push_back(10);     // {10}
  d.push_front(20);    // {20, 10}
  d.push_back(30);     // {20, 10, 30}
  d.push_front(40);    // {40, 20, 10, 30}

pop_front y pop_back:
  d.pop_front();  // elimina 40 → {20, 10, 30}
  d.pop_back();   // elimina 30 → {20, 10}

Acceso a elementos:
  d[0] = 100;          // acceso directo sin verificación
  d.at(1) = 200;       // acceso con verificación de límites
  int primero = d.front();
  int ultimo  = d.back();

Iteradores:
  std::deque<int> d {10, 20, 30, 40, 50};

  for (auto it = d.begin(); it != d.end(); ++it)
      std::cout << *it << ' ';

  for (auto it = d.rbegin(); it != d.rend(); ++it)
      std::cout << *it << ' ';

  for (const auto& x : d)
      std::cout << x << ' ';

  // Modificar con range-based for
  for (auto& x : d)
      x += 5;

Inserción y eliminación (en medio, O(n)):
  auto it = d.begin() + 2;
  d.insert(it, 99);    // {10, 20, 99, 30, 40, 50}
  d.erase(d.begin() + 3);  // elimina el 30

Capacidad y tamaño:
  d.size();        // número de elementos
  d.max_size();    // máximo teórico
  d.empty();       // true si está vacía
  d.resize(10);    // cambia tamaño (nuevos = 0)
  d.resize(10, 5); // cambia tamaño con valor por defecto
  d.shrink_to_fit(); // libera memoria no usada (C++11)

A diferencia de vector, deque NO tiene capacity() ni reserve().
La memoria se gestiona en bloques y no hay una capacidad total
fácil de determinar.

Comparación deque vs vector vs list:

  Característica          deque               vector              list
  ──────────────────────────────────────────────────────────────────────
  Acceso aleatorio        O(1)                O(1)                No
  Inserción/elim. inicio  O(1)                O(n)                O(1)
  Inserción/elim. final   O(1)                O(1) amortizado     O(1)
  Inserción/elim. medio   O(n)                O(n)                O(1)
  Memoria contigua        No (bloques)        Sí                  No
  capacity/reserve        No                  Sí                  No
  Overhead por operación  Bajo                Mínimo              Alto (caché)
  Invalidación iteradores Parcial (ver abajo) Sí (reasignación)   Solo borrado

Invalidación de iteradores en deque:
- insert/erase en medio: invalida todos los iteradores.
- push_front/push_back: invalida algunos iteradores (los de gestión).
- pop_front/pop_back: invalida solo el iterador al elemento eliminado.
- Sin reasignación global (a diferencia de vector).

Uso de emplace:
  struct Punto { int x, y; Punto(int a, int b) : x(a), y(b) {} };

  std::deque<Punto> puntos;
  puntos.emplace_front(10, 20);
  puntos.emplace_back(30, 40);
  puntos.emplace(puntos.begin() + 1, 0, 0);

Asignación:
  d.assign(5, 100);    // 5 elementos con valor 100
  d.assign({1,2,3});   // initializer list

Intercambio:
  std::deque<int> d1 {1,2,3};
  std::deque<int> d2 {4,5,6};
  d1.swap(d2);  // d1 tiene 4,5,6 y d2 tiene 1,2,3

Operaciones relacionales (comparación lexicográfica):
  std::deque<int> a {1,2,3};
  std::deque<int> b {1,2,4};
  if (a < b)
      std::cout << "a es menor que b\n";

Aplicaciones comunes de deque:

1. Cola de doble extremo (buffer bidireccional):
  - push_back + pop_front = cola FIFO.
  - push_front + pop_back = pila LIFO (aunque stack es más claro).
  - push_front + pop_front = cola LIFO inversa.
  - push_back + pop_back = pila.

2. Ventana deslizante (sliding window):
   Mantener una ventana de N elementos donde se agregan nuevos
   al final y se eliminan viejos del inicio.

  std::deque<int> ventana;
  for (int i = 0; i < 10; ++i) {
      ventana.push_back(i);
      if (ventana.size() > 5)
          ventana.pop_front();
      // ventana contiene los últimos 5 elementos
  }

3. Algoritmo de cola para BFS (Breadth-First Search):
   Aunque queue es más apropiado, deque puede usarse para BFS
   con capacidad de agregar al inicio (como en un deque bidireccional).

4. Undo/Redo con capacidad limitada:
   Almacenar los últimos N estados, descartando los más antiguos.

  std::deque<std::string> historial;
  historial.push_back(estadoActual);
  if (historial.size() > 100)
      historial.pop_front();

5. Implementación de un buffer circular:
   Usar deque como buffer circular con capacidad máxima fija.

Ejemplo completo: Sistema de cola de atención al cliente.

  #include <iostream>
  #include <deque>
  #include <string>
  #include <ctime>

  struct Cliente {
      int numero;
      std::string nombre;
      std::time_t horaLLegada;
      bool esVip;
  };

  class SistemaColas {
  private:
      std::deque<Cliente> colaNormal;
      std::deque<Cliente> colaVip;
      int contador;

  public:
      SistemaColas() : contador(0) {}

      void llegarCliente(const std::string& nombre, bool vip) {
          Cliente c {++contador, nombre, std::time(nullptr), vip};
          if (vip)
              colaVip.push_back(c);
          else
              colaNormal.push_back(c);
          std::cout << "Cliente " << contador << " (" << nombre
                    << ") en cola " << (vip ? "VIP" : "Normal") << '\n';
      }

      void atenderSiguiente() {
          if (!colaVip.empty()) {
              atender(colaVip.front());
              colaVip.pop_front();
          } else if (!colaNormal.empty()) {
              atender(colaNormal.front());
              colaNormal.pop_front();
          } else {
              std::cout << "No hay clientes en espera.\n";
          }
      }

      void atenderVipPrimero() {
          if (!colaVip.empty()) {
              // Mover VIP al frente de la cola normal
              auto vip = colaVip.front();
              colaVip.pop_front();
              colaNormal.push_front(vip);
              std::cout << "VIP " << vip.numero
                        << " movido al frente de cola normal.\n";
          } else {
              std::cout << "No hay clientes VIP.\n";
          }
      }

      void expulsarCliente(int numero) {
          auto it = colaNormal.begin();
          while (it != colaNormal.end()) {
              if (it->numero == numero) {
                  colaNormal.erase(it);
                  std::cout << "Cliente " << numero << " expulsado.\n";
                  return;
              }
              ++it;
          }
          // Buscar en VIP
          it = colaVip.begin();
          while (it != colaVip.end()) {
              if (it->numero == numero) {
                  colaVip.erase(it);
                  std::cout << "Cliente VIP " << numero
                            << " expulsado.\n";
                  return;
              }
              ++it;
          }
          std::cout << "Cliente no encontrado.\n";
      }

      void mostrarEstado() const {
          std::cout << "\n=== ESTADO DE COLAS ===\n";
          std::cout << "VIP (" << colaVip.size() << "): ";
          for (const auto& c : colaVip)
              std::cout << c.numero << " ";
          std::cout << "\nNormal (" << colaNormal.size() << "): ";
          for (const auto& c : colaNormal)
              std::cout << c.numero << " ";
          std::cout << '\n';
      }

      void rotarPrioridad() {
          // Mover el primer cliente normal al final
          if (!colaNormal.empty()) {
              colaNormal.push_back(colaNormal.front());
              colaNormal.pop_front();
              std::cout << "Cola rotada.\n";
          }
      }

  private:
      void atender(const Cliente& c) {
          double espera = std::difftime(std::time(nullptr), c.horaLLegada);
          std::cout << "Atendiendo cliente " << c.numero
                    << " (" << c.nombre << ")"
                    << " - Esperó " << espera << " segundos\n";
      }
  };

Rendimiento:

  std::deque<int> d;
  // push_front/push_back son O(1)
  for (int i = 0; i < 100000; ++i) {
      d.push_front(i);
  }

  // Acceso aleatorio: O(1) pero más lento que vector
  int x = d[50000];  // requiere cálculo de bloque + offset

  // Iteración: similar a vector en velocidad
  for (auto& val : d)
      val *= 2;

Comparación práctica:
- vector: mejor para acceso aleatorio frecuente y operaciones al final.
- deque: mejor cuando necesitas push/pop en ambos extremos.
- list: mejor cuando necesitas inserción/eliminación en medio.

Una regla empírica: si necesitas push_back y pop_front (cola FIFO),
usa deque. Si solo push_back, usa vector.

Conversión entre contenedores:

  std::vector<int> vec(d.begin(), d.end());  // deque a vector
  std::list<int> lst(d.begin(), d.end());     // deque a list
  std::deque<int> dq(vec.begin(), vec.end()); // vector a deque

Uso con algoritmos STL:
  #include <algorithm>
  std::deque<int> d {3, 1, 4, 1, 5, 9};

  std::sort(d.begin(), d.end());
  auto it = std::find(d.begin(), d.end(), 4);
  std::reverse(d.begin(), d.end());
  std::random_shuffle(d.begin(), d.end());

  // O(n) para inserción en medio
  auto pos = d.begin() + 3;
  d.insert(pos, 99);  // desplaza elementos desde el inicio o final,
                      // el que tenga menos desplazamiento

Este es un detalle interesante de implementación: deque puede
elegir si desplazar desde el inicio o desde el final para insertar
en medio, minimizando el número de elementos movidos.

Ejemplo de buffer fijo con deque:

  template<typename T>
  class CircularBuffer {
  private:
      std::deque<T> buffer;
      size_t maxSize;

  public:
      explicit CircularBuffer(size_t max) : maxSize(max) {}

      void push(const T& item) {
          if (buffer.size() == maxSize)
              buffer.pop_front();
          buffer.push_back(item);
      }

      const T& operator[](size_t index) const {
          return buffer[index];
      }

      size_t size() const { return buffer.size(); }
      bool empty() const { return buffer.empty(); }
      const T& front() const { return buffer.front(); }
      const T& back() const { return buffer.back(); }
      void clear() { buffer.clear(); }

      auto begin() const { return buffer.begin(); }
      auto end() const { return buffer.end(); }
  };

  // Uso:
  CircularBuffer<int> cb(5);
  for (int i = 0; i < 10; ++i) {
      cb.push(i);
      // Contenido: {i-4, i-3, i-2, i-1, i} cuando i >= 4
  }

Ejemplo de ventana deslizante para promedio móvil:

  class PromedioMovil {
  private:
      std::deque<double> ventana;
      int tamano;
      double suma;

  public:
      explicit PromedioMovil(int n) : tamano(n), suma(0.0) {}

      double agregar(double valor) {
          ventana.push_back(valor);
          suma += valor;
          if (ventana.size() > tamano) {
              suma -= ventana.front();
              ventana.pop_front();
          }
          return suma / ventana.size();
      }

      double promedio() const {
          return ventana.empty() ? 0.0 : suma / ventana.size();
      }
  };

  // Uso:
  PromedioMovil pm(3);
  pm.agregar(10);  // promedio = 10
  pm.agregar(20);  // promedio = 15
  pm.agregar(30);  // promedio = 20
  pm.agregar(40);  // promedio = 30 (descarta 10)

//==============================================================
// EJEMPLO COMPLETO: SISTEMA DE MENSAJERÍA CON DEQUE
//==============================================================
#include <iostream>
#include <deque>
#include <string>
#include <algorithm>
#include <ctime>
#include <iomanip>

enum class TipoMensaje { TEXTO, IMAGEN, VIDEO, SISTEMA };

struct Mensaje {
    int id;
    std::string remitente;
    std::string contenido;
    TipoMensaje tipo;
    std::time_t timestamp;
    bool leido;
    int prioridad;

    Mensaje(int i, std::string r, std::string c, TipoMensaje t, int p = 0)
        : id(i), remitente(std::move(r)), contenido(std::move(c)),
          tipo(t), timestamp(std::time(nullptr)), leido(false),
          prioridad(p) {}
};

std::string tipoToString(TipoMensaje t) {
    switch (t) {
        case TipoMensaje::TEXTO:  return "TEXTO";
        case TipoMensaje::IMAGEN: return "IMAGEN";
        case TipoMensaje::VIDEO:  return "VIDEO";
        case TipoMensaje::SISTEMA:return "SISTEMA";
    }
    return "?";
}

class BuzonMensajes {
private:
    std::deque<Mensaje> mensajes;
    int nextId;
    static const size_t MAX_MENSAJES = 100;

public:
    BuzonMensajes() : nextId(1) {}

    void recibirMensaje(const std::string& remitente,
                        const std::string& contenido,
                        TipoMensaje tipo, int prioridad = 0) {
        if (mensajes.size() >= MAX_MENSAJES) {
            mensajes.pop_front();  // eliminar el más antiguo
        }
        mensajes.push_back(
            Mensaje(nextId++, remitente, contenido, tipo, prioridad));
        std::cout << "Mensaje " << (nextId - 1) << " recibido.\n";
    }

    void recibirMensajeUrgente(const std::string& remitente,
                               const std::string& contenido,
                               TipoMensaje tipo) {
        if (mensajes.size() >= MAX_MENSAJES)
            mensajes.pop_back();  // eliminar el menos urgente
        mensajes.push_front(
            Mensaje(nextId++, remitente, contenido, tipo, 10));
        std::cout << "MENSAJE URGENTE " << (nextId - 1)
                  << " recibido.\n";
    }

    Mensaje* siguienteNoLeido() {
        for (auto& m : mensajes) {
            if (!m.leido)
                return &m;
        }
        return nullptr;
    }

    void marcarLeido(int id) {
        auto it = std::find_if(mensajes.begin(), mensajes.end(),
            [id](const Mensaje& m) { return m.id == id; });
        if (it != mensajes.end()) {
            it->leido = true;
            // Mover a la posición de leídos (final)
            Mensaje copia = *it;
            mensajes.erase(it);
            mensajes.push_back(copia);
        }
    }

    void eliminarMensaje(int id) {
        auto it = std::find_if(mensajes.begin(), mensajes.end(),
            [id](const Mensaje& m) { return m.id == id; });
        if (it != mensajes.end()) {
            mensajes.erase(it);
            std::cout << "Mensaje " << id << " eliminado.\n";
        }
    }

    void eliminarLeidos() {
        auto it = std::remove_if(mensajes.begin(), mensajes.end(),
            [](const Mensaje& m) { return m.leido; });
        size_t eliminados = std::distance(it, mensajes.end());
        mensajes.erase(it, mensajes.end());
        std::cout << eliminados << " mensajes leídos eliminados.\n";
    }

    void mostrarNoLeidos() const {
        std::cout << "\n=== MENSAJES NO LEÍDOS ===\n";
        for (const auto& m : mensajes) {
            if (!m.leido) {
                std::cout << "[" << m.id << "] "
                          << tipoToString(m.tipo) << " | "
                          << m.remitente << ": "
                          << m.contenido.substr(0, 50)
                          << (m.contenido.size() > 50 ? "..." : "")
                          << " (prioridad: " << m.prioridad << ")\n";
            }
        }
    }

    void mostrarTodos() const {
        std::cout << "\n=== TODOS LOS MENSAJES (" << mensajes.size()
                  << ") ===\n";
        int i = 1;
        for (const auto& m : mensajes) {
            std::cout << i++ << ". [" << m.id << "]"
                      << (m.leido ? " [LEIDO]" : " [NO LEIDO]")
                      << " " << tipoToString(m.tipo)
                      << " " << m.remitente << ": "
                      << m.contenido.substr(0, 40)
                      << (m.contenido.size() > 40 ? "..." : "")
                      << '\n';
        }
    }

    void ordenarPorPrioridad() {
        // Usar stable_partition para mover urgentes al frente
        std::stable_partition(mensajes.begin(), mensajes.end(),
            [](const Mensaje& m) {
                return m.prioridad >= 5 && !m.leido;
            });
        std::cout << "Mensajes reordenados por prioridad.\n";
    }

    void limpiarViejos(int dias) {
        std::time_t ahora = std::time(nullptr);
        double segundos = dias * 24 * 3600;
        auto it = std::remove_if(mensajes.begin(), mensajes.end(),
            [ahora, segundos](const Mensaje& m) {
                return std::difftime(ahora, m.timestamp) > segundos;
            });
        mensajes.erase(it, mensajes.end());
    }

    size_t contarNoLeidos() const {
        return std::count_if(mensajes.begin(), mensajes.end(),
            [](const Mensaje& m) { return !m.leido; });
    }

    void mostrarEstadisticas() const {
        int texto = 0, imagen = 0, video = 0, sistema = 0;
        for (const auto& m : mensajes) {
            switch (m.tipo) {
                case TipoMensaje::TEXTO:  ++texto; break;
                case TipoMensaje::IMAGEN: ++imagen; break;
                case TipoMensaje::VIDEO:  ++video; break;
                case TipoMensaje::SISTEMA:++sistema; break;
            }
        }
        std::cout << "\n=== ESTADÍSTICAS ===\n";
        std::cout << "Total: " << mensajes.size() << '\n';
        std::cout << "No leídos: " << contarNoLeidos() << '\n';
        std::cout << "Texto: " << texto << '\n';
        std::cout << "Imagen: " << imagen << '\n';
        std::cout << "Video: " << video << '\n';
        std::cout << "Sistema: " << sistema << '\n';
    }

    // Iteradores públicos
    auto begin() { return mensajes.begin(); }
    auto end() { return mensajes.end(); }
};

int main() {
    BuzonMensajes buzon;

    buzon.recibirMensaje("Ana", "¿Qué tal el proyecto?", TipoMensaje::TEXTO);
    buzon.recibirMensaje("Luis", "Foto del diagrama", TipoMensaje::IMAGEN);
    buzon.recibirMensajeUrgente("SISTEMA", "¡Servidor caído!",
                                TipoMensaje::SISTEMA);
    buzon.recibirMensaje("Carlos", "Video de la demo", TipoMensaje::VIDEO, 3);
    buzon.recibirMensaje("Ana", "Revisa el documento adjunto",
                         TipoMensaje::TEXTO);
    buzon.recibirMensajeUrgente("Jefe", "Reunión inmediata",
                                TipoMensaje::TEXTO);

    buzon.mostrarTodos();
    buzon.mostrarNoLeidos();

    std::cout << "\nLeyendo mensaje 1...\n";
    buzon.marcarLeido(1);
    buzon.mostrarTodos();

    std::cout << "\nReordenando por prioridad...\n";
    buzon.ordenarPorPrioridad();
    buzon.mostrarTodos();

    std::cout << "\nEliminando mensaje 3...\n";
    buzon.eliminarMensaje(3);
    buzon.mostrarTodos();

    std::cout << "\nEliminando leídos...\n";
    buzon.eliminarLeidos();
    buzon.mostrarTodos();

    buzon.mostrarEstadisticas();

    return 0;
}

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

std::deque es un contenedor secuencial que ofrece:
- Inserción/eliminación O(1) en ambos extremos.
- Acceso aleatorio O(1).
- No tiene capacity() ni reserve().
- Estructura interna basada en bloques.

Es ideal para:
- Colas FIFO (push_back + pop_front).
- Buffers circulares y ventanas deslizantes.
- Situaciones donde necesitas push/pop en ambos extremos.
- Alternativa a vector cuando también haces push_front.

No es ideal para:
- Inserción/eliminación en medio (O(n)).
- Cuando solo necesitas push_back (vector es mejor).
- Cuando la memoria contigua es necesaria (vector).

La elección entre vector, deque y list depende del patrón de acceso:
- vector: acceso aleatorio + operaciones al final.
- deque: acceso aleatorio + operaciones en ambos extremos.
- list: inserción/eliminación en medio + sin acceso aleatorio.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿deque tiene capacity() como vector?
R: No. deque no expone una función capacity(). Su memoria se
   gestiona en bloques y no hay una capacidad total significativa.

P: ¿Es deque más lento que vector en acceso aleatorio?
R: Sí, ligeramente. deque requiere una indirección adicional
   (calcular en qué bloque está el elemento). En la práctica,
   la diferencia es pequeña para la mayoría de aplicaciones.

P: ¿Cuándo debo elegir deque sobre vector?
R: Cuando necesites push_front y pop_front además de push_back
   y pop_back. deque optimiza ambos extremos.

P: ¿Los iteradores de deque se invalidan?
R: Depende de la operación:
   - push_front/push_back: invalida algunos iteradores.
   - insert/erase en medio: invalida todos.
   - pop_front/pop_back: solo invalida el iterador al elemento
     eliminado.

P: ¿Puedo usar deque con algoritmos que requieren acceso aleatorio?
R: Sí, std::sort, std::binary_search, etc. funcionan con deque
   porque tiene iteradores de acceso aleatorio.

P: ¿Debo usar deque o queue para una cola FIFO?
R: queue es un adaptador que usa deque por defecto. Si necesitas
   solo operaciones FIFO básicas, queue es más simple. Si necesitas
   acceso aleatorio o insertar en medio, usa deque directamente.

  DESAFÍO DEL ARQUITECTO
================================================================================

Diseña e implementa un "Sistema de Gestión de Pedidos en Línea"
usando std::deque como estructura principal. El sistema debe:

1. Gestionar pedidos con: ID, cliente, lista de productos (vector),
   total, prioridad (1-5), estado (pendiente, preparación, listo,
   entregado), timestamp.

2. Dos colas de pedidos:
   - Cola normal (push_back).
   - Pedidos exprés (push_front) para clientes premium o pedidos
     con prioridad ≥ 4.

3. Operaciones:
   - nuevo_pedido(): agrega al final de la cola normal.
   - nuevo_pedido_expres(): agrega al frente de la cola exprés.
   - procesar_siguiente(): extrae de cola exprés primero, luego
     normal.
   - cancelar_pedido(id): busca y elimina de cualquier cola.
   - urgir_pedido(id): mueve un pedido normal al frente de su cola.
   - mostrar_colas(): imprime ambas colas con detalles.
   - estadisticas(): totales, pendientes, tiempo promedio.

4. Los pedidos exprés tienen prioridad absoluta: siempre se procesan
   antes que los normales.

5. Implementar un límite máximo de 50 pedidos en cada cola. Si se
   excede, rechazar el nuevo pedido y notificar.

6. El sistema debe ejecutarse en modo interactivo con menú de opciones
   (1-8) y ciclo continuo.

Este desafío integra push_back, push_front, pop_front, erase,
find_if y el patrón de cola doble con deque.
