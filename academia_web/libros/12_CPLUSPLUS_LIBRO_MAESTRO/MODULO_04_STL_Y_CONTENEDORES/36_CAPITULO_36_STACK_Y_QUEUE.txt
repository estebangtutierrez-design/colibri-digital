Los adaptadores de contenedor son wrappers que proporcionan una
interfaz restringida sobre un contenedor subyacente. No son
contenedores completos, sino que limitan las operaciones para
garantizar un comportamiento específico.

STACK (LIFO - Last In, First Out)

std::stack es un adaptador que opera bajo el principio LIFO:
el último elemento insertado es el primero en ser extraído.

Contenedor subyacente por defecto: std::deque.
También puede usar std::vector o std::list.

  #include <stack>

  std::stack<int> pila;           // usa deque por defecto
  std::stack<int, std::vector<int>> pila_vec;  // usa vector
  std::stack<int, std::list<int>> pila_list;   // usa list

Operaciones básicas:
  pila.push(10);     // inserta elemento en el tope
  pila.push(20);
  pila.push(30);     // pila: [10, 20, 30] (30 es el tope)

  int tope = pila.top();  // acceso al tope (30)
  pila.pop();             // elimina el tope (30)
  // pila: [10, 20]

  pila.size();   // número de elementos
  pila.empty();  // true si está vacía
  pila.emplace(40);  // construye in-situ (C++11)

  // No tiene begin/end ni iteradores
  // No tiene clear() (hay que popear todo)

Ejemplo: Revertir una palabra con stack.

  std::string palabra = "HOLA";
  std::stack<char> pila;

  for (char c : palabra)
      pila.push(c);

  std::string invertida;
  while (!pila.empty()) {
      invertida += pila.top();
      pila.pop();
  }
  // invertida = "ALOH"

Ejemplo: Evaluación de expresión postfija.

  #include <stack>
  #include <sstream>
  #include <cctype>

  double evaluarPostfija(const std::string& expr) {
      std::stack<double> pila;
      std::istringstream iss(expr);
      std::string token;

      while (iss >> token) {
          if (isdigit(token[0]) || (token[0] == '-' && token.size() > 1)) {
              pila.push(std::stod(token));
          } else {
              double b = pila.top(); pila.pop();
              double a = pila.top(); pila.pop();
              if (token == "+") pila.push(a + b);
              else if (token == "-") pila.push(a - b);
              else if (token == "*") pila.push(a * b);
              else if (token == "/") pila.push(a / b);
          }
      }
      return pila.top();
  }

  // evaluarPostfija("3 4 + 5 *") = 35

Aplicaciones clásicas de stack:
- Evaluación de expresiones (postfija, infija).
- Conversión de infija a postfija (Shunting Yard).
- Algoritmo de backtracking (laberintos, búsqueda DFS).
- Verificación de paréntesis balanceados.
- Deshacer (undo) en editores.
- Recorrido de árboles (DFS iterativo).

Verificación de paréntesis:

  bool parentesisBalanceados(const std::string& s) {
      std::stack<char> pila;
      for (char c : s) {
          if (c == '(' || c == '[' || c == '{')
              pila.push(c);
          else if (c == ')' || c == ']' || c == '}') {
              if (pila.empty()) return false;
              char tope = pila.top(); pila.pop();
              if ((c == ')' && tope != '(') ||
                  (c == ']' && tope != '[') ||
                  (c == '}' && tope != '{'))
                  return false;
          }
      }
      return pila.empty();
  }

QUEUE (FIFO - First In, First Out)

std::queue es un adaptador FIFO: el primer elemento insertado
es el primero en ser extraído.

Contenedor subyacente por defecto: std::deque.
También puede usar std::list.

  #include <queue>

  std::queue<int> cola;           // usa deque por defecto
  std::queue<int, std::list<int>> cola_list;  // usa list

Operaciones básicas:
  cola.push(10);     // inserta al final
  cola.push(20);
  cola.push(30);     // cola: [10, 20, 30]

  int frente = cola.front();  // primer elemento (10)
  int final = cola.back();    // último elemento (30)
  cola.pop();                 // elimina el frente (10)
  // cola: [20, 30]

  cola.size();
  cola.empty();
  cola.emplace(40);

  // No tiene iteradores ni clear()

Ejemplo: Simulación de cola de banco.

  #include <queue>
  #include <string>

  std::queue<std::string> colaBanco;

  void llegarCliente(const std::string& nombre) {
      colaBanco.push(nombre);
      std::cout << nombre << " llegó. En cola: "
                << colaBanco.size() << '\n';
  }

  void atenderCliente() {
      if (colaBanco.empty()) {
          std::cout << "No hay clientes\n";
          return;
      }
      std::cout << "Atendiendo a: " << colaBanco.front() << '\n';
      colaBanco.pop();
  }

Aplicaciones de queue:
- Colas de impresión.
- Buffers de datos (streams).
- BFS (Breadth-First Search) en grafos.
- Colas de tareas (task scheduling).
- Sistemas de atención al cliente.

BFS con queue:

  void bfs(const std::vector<std::vector<int>>& grafo,
           int inicio) {
      std::queue<int> cola;
      std::vector<bool> visitado(grafo.size(), false);

      visitado[inicio] = true;
      cola.push(inicio);

      while (!cola.empty()) {
          int nodo = cola.front();
          cola.pop();
          std::cout << "Visitando: " << nodo << '\n';

          for (int vecino : grafo[nodo]) {
              if (!visitado[vecino]) {
                  visitado[vecino] = true;
                  cola.push(vecino);
              }
          }
      }
  }

PRIORITY_QUEUE

std::priority_queue es un adaptador que siempre extrae el elemento
de mayor prioridad (por defecto, el más grande). Internamente usa
un heap (estructura de montículo).

Contenedor subyacente por defecto: std::vector.

  #include <queue>

  std::priority_queue<int> pq;  // máximo elemento en el tope
  pq.push(30);
  pq.push(10);
  pq.push(50);
  pq.push(20);
  // top() = 50, luego 30, 20, 10

  int max = pq.top();  // 50
  pq.pop();            // elimina 50, top = 30

  pq.size();
  pq.empty();
  pq.emplace(60);

Prioridad inversa (mínimo elemento):

  std::priority_queue<int, std::vector<int>,
                      std::greater<int>> pq_min;
  pq_min.push(30);
  pq_min.push(10);
  pq_min.push(50);
  // top() = 10

Comparador personalizado:

  struct Tarea {
      int prioridad;
      std::string nombre;
  };

  struct CompararTarea {
      bool operator()(const Tarea& a, const Tarea& b) const {
          return a.prioridad < b.prioridad;  // mayor prioridad primero
      }
  };

  std::priority_queue<Tarea, std::vector<Tarea>,
                      CompararTarea> colaTareas;

  colaTareas.push({5, "Crítica"});
  colaTareas.push({1, "Baja"});
  colaTareas.push({3, "Media"});
  // top() = Crítica (prioridad 5)

  // Con lambda (C++14)
  auto cmp = [](int a, int b) { return a > b; };
  std::priority_queue<int, std::vector<int>,
                      decltype(cmp)> pq_lambda(cmp);

Aplicaciones de priority_queue:
- Algoritmo de Dijkstra (camino más corto).
- Planificación de procesos (scheduling).
- Colas de prioridad en sistemas operativos.
- Algoritmo Huffman (compresión).
- Top K elementos.
- Mediana dinámica.

Top K elementos con priority_queue:

  std::vector<int> encontrarTopK(const std::vector<int>& datos, int k) {
      std::priority_queue<int, std::vector<int>, std::greater<int>> minHeap;

      for (int x : datos) {
          minHeap.push(x);
          if (minHeap.size() > k)
              minHeap.pop();  // elimina el más pequeño
      }

      std::vector<int> resultado;
      while (!minHeap.empty()) {
          resultado.push_back(minHeap.top());
          minHeap.pop();
      }
      return resultado;  // top K elementos
  }

Mediana dinámica con dos priority_queues:

  class MedianFinder {
  private:
      std::priority_queue<int> maxHeap;  // mitad inferior
      std::priority_queue<int, std::vector<int>,
                          std::greater<int>> minHeap;  // mitad superior

  public:
      void addNum(int num) {
          if (maxHeap.empty() || num <= maxHeap.top())
              maxHeap.push(num);
          else
              minHeap.push(num);

          // Balancear
          if (maxHeap.size() > minHeap.size() + 1) {
              minHeap.push(maxHeap.top());
              maxHeap.pop();
          } else if (minHeap.size() > maxHeap.size()) {
              maxHeap.push(minHeap.top());
              minHeap.pop();
          }
      }

      double findMedian() {
          if (maxHeap.size() == minHeap.size())
              return (maxHeap.top() + minHeap.top()) / 2.0;
          return maxHeap.top();
      }
  };

Comparación de adaptadores:

  Característica      stack           queue           priority_queue
  ────────────────────────────────────────────────────────────────
  Principio           LIFO            FIFO            Por prioridad
  Elemento tope       top()           front()         top()
  Inserción           push()          push()          push()
  Eliminación         pop()           pop()           pop()
  Contenedor default  deque           deque           vector
  Iteradores          No              No              No
  Orden               No              No             Sí (heap)

Consideraciones de rendimiento:
- stack: todas las operaciones O(1).
- queue: todas las operaciones O(1).
- priority_queue: push/pop O(log n), top O(1).

```cpp
//==============================================================
// EJEMPLO COMPLETO: SISTEMA DE PROCESAMIENTO DE TAREAS
//==============================================================
#include <iostream>
#include <stack>
#include <queue>
#include <string>
#include <vector>
#include <cstdlib>
#include <ctime>
#include <iomanip>

enum class TipoTarea { SIMPLE, COMPLEJA, CRITICA, REPORTE };
enum class Estado { PENDIENTE, EN_PROCESO, COMPLETADA, CANCELADA };

struct Tarea {
    int id;
    std::string descripcion;
    TipoTarea tipo;
    Estado estado;
    int prioridad;
    int tiempoEstimado;
    int tiempoReal;

    Tarea(int i, std::string d, TipoTarea t, int p, int te)
        : id(i), descripcion(std::move(d)), tipo(t),
          estado(Estado::PENDIENTE), prioridad(p),
          tiempoEstimado(te), tiempoReal(0) {}
};

std::string tipoToString(TipoTarea t) {
    switch (t) {
        case TipoTarea::SIMPLE:   return "SIMPLE";
        case TipoTarea::COMPLEJA: return "COMPLEJA";
        case TipoTarea::CRITICA:  return "CRITICA";
        case TipoTarea::REPORTE:  return "REPORTE";
    }
    return "?";
}

std::string estadoToString(Estado e) {
    switch (e) {
        case Estado::PENDIENTE:   return "PENDIENTE";
        case Estado::EN_PROCESO:  return "EN PROCESO";
        case Estado::COMPLETADA:  return "COMPLETADA";
        case Estado::CANCELADA:   return "CANCELADA";
    }
    return "?";
}

class SistemaTareas {
private:
    std::queue<Tarea> colaNormal;          // FIFO para tareas normales
    std::stack<Tarea> pilaUrgente;         // LIFO para tareas urgentes
    std::priority_queue<Tarea> colaPrioridad;  // por prioridad

    // Función para comparar prioridad (mayor prioridad primero)
    struct ComparePrioridad {
        bool operator()(const Tarea& a, const Tarea& b) const {
            if (a.prioridad != b.prioridad)
                return a.prioridad < b.prioridad;
            return a.id > b.id;
        }
    };

    std::priority_queue<Tarea, std::vector<Tarea>,
                        ComparePrioridad> colaCritica;

    std::vector<Tarea> completadas;
    std::vector<Tarea> canceladas;
    int nextId;
    const size_t MAX_COLA = 100;

public:
    SistemaTareas() : nextId(1) {}

    void agregarTareaNormal(const std::string& desc,
                            TipoTarea tipo, int tiempo) {
        if (colaNormal.size() >= MAX_COLA) {
            std::cout << "Cola normal llena. Rechazando tarea.\n";
            return;
        }
        colaNormal.push(Tarea(nextId++, desc, tipo, 1, tiempo));
        std::cout << "Tarea normal " << (nextId - 1)
                  << " agregada a la cola.\n";
    }

    void agregarTareaUrgente(const std::string& desc,
                             TipoTarea tipo, int tiempo) {
        pilaUrgente.push(Tarea(nextId++, desc, tipo, 5, tiempo));
        std::cout << "Tarea URGENTE " << (nextId - 1)
                  << " agregada a la pila.\n";
    }

    void agregarTareaCritica(const std::string& desc,
                             int prioridad, int tiempo) {
        colaCritica.push(
            Tarea(nextId++, desc, TipoTarea::CRITICA,
                  prioridad, tiempo));
        std::cout << "Tarea CRITICA " << (nextId - 1)
                  << " agregada (prioridad " << prioridad << ").\n";
    }

    Tarea procesarSiguiente() {
        // Prioridad: stack urgente > priority queue crítica > queue normal
        if (!pilaUrgente.empty()) {
            Tarea t = pilaUrgente.top();
            pilaUrgente.pop();
            t.estado = Estado::EN_PROCESO;
            t.tiempoReal = t.tiempoEstimado;
            std::cout << "Procesando URGENTE: " << t.descripcion << '\n';
            return t;
        }

        if (!colaCritica.empty()) {
            Tarea t = colaCritica.top();
            colaCritica.pop();
            t.estado = Estado::EN_PROCESO;
            t.tiempoReal = t.tiempoEstimado;
            std::cout << "Procesando CRITICA: " << t.descripcion
                      << " (prioridad " << t.prioridad << ")\n";
            return t;
        }

        if (!colaNormal.empty()) {
            Tarea t = colaNormal.front();
            colaNormal.pop();
            t.estado = Estado::EN_PROCESO;
            t.tiempoReal = t.tiempoEstimado;
            std::cout << "Procesando NORMAL: " << t.descripcion << '\n';
            return t;
        }

        std::cout << "No hay tareas pendientes.\n";
        return Tarea(-1, "", TipoTarea::SIMPLE, 0, 0);
    }

    void completarTarea(Tarea t) {
        t.estado = Estado::COMPLETADA;
        completadas.push_back(t);
        std::cout << "Tarea " << t.id << " completada: "
                  << t.descripcion << '\n';
    }

    void cancelarTarea(Tarea t) {
        t.estado = Estado::CANCELADA;
        canceladas.push_back(t);
        std::cout << "Tarea " << t.id << " cancelada: "
                  << t.descripcion << '\n';
    }

    void procesarTodo() {
        while (!pilaUrgente.empty() || !colaCritica.empty()
               || !colaNormal.empty()) {
            Tarea t = procesarSiguiente();
            if (t.id == -1) break;
            completarTarea(t);
        }
        std::cout << "\nTodas las tareas procesadas.\n";
    }

    void mostrarEstado() const {
        std::cout << "\n=== ESTADO DEL SISTEMA ===\n";
        std::cout << "Cola Normal (FIFO): " << colaNormal.size() << '\n';
        std::cout << "Pila Urgente (LIFO): " << pilaUrgente.size() << '\n';
        std::cout << "Cola Crítica (Prioridad): "
                  << colaCritica.size() << '\n';
        std::cout << "Completadas: " << completadas.size() << '\n';
        std::cout << "Canceladas: " << canceladas.size() << '\n';
    }

    void mostrarCompletadas() const {
        std::cout << "\n=== TAREAS COMPLETADAS ===\n";
        for (const auto& t : completadas) {
            std::cout << "  [" << t.id << "] "
                      << t.descripcion << " | "
                      << tipoToString(t.tipo) << " | "
                      << t.tiempoReal << " min\n";
        }
    }

    void procesarEnLote(const std::vector<std::string>& lotes) {
        for (const auto& desc : lotes) {
            agregarTareaNormal(desc, TipoTarea::SIMPLE,
                               5 + rand() % 20);
        }
    }

    void simularDeshacer() {
        // Usar stack para deshacer la última tarea completada
        std::stack<Tarea> pilaDeshacer;
        for (const auto& t : completadas)
            pilaDeshacer.push(t);

        if (!pilaDeshacer.empty()) {
            Tarea ultima = pilaDeshacer.top();
            pilaDeshacer.pop();
            std::cout << "Deshaciendo tarea: " << ultima.descripcion
                      << " (vuelve a la cola)\n";
            // En un sistema real, la reinsertaríamos
        }
    }
};

// Demostración de uso de stack para verificar paréntesis
bool verificarParentesis(const std::string& expr) {
    std::stack<char> pila;
    for (char c : expr) {
        if (c == '(' || c == '[' || c == '{') {
            pila.push(c);
        } else if (c == ')' || c == ']' || c == '}') {
            if (pila.empty()) return false;
            char top = pila.top(); pila.pop();
            if ((c == ')' && top != '(') ||
                (c == ']' && top != '[') ||
                (c == '}' && top != '{'))
                return false;
        }
    }
    return pila.empty();
}

int main() {
    std::srand(std::time(nullptr));

    SistemaTareas st;

    st.agregarTareaNormal("Procesar facturas", TipoTarea::REPORTE, 15);
    st.agregarTareaNormal("Actualizar inventario", TipoTarea::SIMPLE, 10);
    st.agregarTareaUrgente("REVISAR SERVIDOR", TipoTarea::CRITICA, 5);
    st.agregarTareaNormal("Generar reporte mensual", TipoTarea::REPORTE, 30);
    st.agregarTareaCritica("Corregir bug crítico", 10, 20);
    st.agregarTareaCritica("Optimizar consulta", 7, 15);
    st.agregarTareaUrgente("Incidente de seguridad", TipoTarea::CRITICA, 10);

    st.mostrarEstado();

    std::cout << "\n--- Procesando tareas ---\n";
    for (int i = 0; i < 5; ++i) {
        Tarea t = st.procesarSiguiente();
        if (t.id != -1)
            st.completarTarea(t);
    }

    st.mostrarEstado();
    st.mostrarCompletadas();

    std::cout << "\n--- Verificación de paréntesis ---\n";
    std::string expr1 = "((a+b)*(c-d))";
    std::string expr2 = "((a+b)*(c-d)";
    std::cout << expr1 << " -> "
              << (verificarParentesis(expr1) ? "BALANCEADO" : "NO BALANCEADO")
              << '\n';
    std::cout << expr2 << " -> "
              << (verificarParentesis(expr2) ? "BALANCEADO" : "NO BALANCEADO")
              << '\n';

    return 0;
}
```

Este ejemplo muestra stack (LIFO), queue (FIFO) y priority_queue
con sus operaciones push, pop, top, front, back, size, empty.

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

Puntos clave de stack:
- LIFO: Last-In, First-Out.
- Operaciones: push, pop, top, size, empty.
- Contenedor subyacente: deque (por defecto), vector, list.
- Aplicaciones: paréntesis, undo, DFS, notación postfija.

Puntos clave de queue:
- FIFO: First-In, First-Out.
- Operaciones: push, pop, front, back, size, empty.
- Contenedor subyacente: deque (por defecto), list.
- Aplicaciones: colas de atención, BFS, buffers.

Puntos clave de priority_queue:
- Extrae el elemento de mayor prioridad.
- Operaciones: push, pop, top, size, empty.
- Contenedor subyacente: vector (por defecto).
- Orden por defecto: std::less (máximo).
- Para mínimo: std::greater<T>.
- Aplicaciones: Dijkstra, Huffman, top K, scheduling.

Los adaptadores no tienen iteradores ni begin/end. Son interfaces
restringidas intencionalmente para garantizar el comportamiento
deseado.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Por qué stack y queue se llaman adaptadores?
R: Porque adaptan un contenedor existente (deque, vector, list)
   para proporcionar una interfaz restringida.

P: ¿Puedo iterar sobre un stack o queue?
R: No directamente. Debes extraer los elementos (pop) para
   recorrerlos. Si necesitas iterar, usa el contenedor subyacente
   directamente.

P: ¿Qué contenedor subyacente debo usar?
R: Por defecto deque es buena opción. stack puede usar vector
   para mejor localidad de caché. queue no debe usar vector
   porque no tiene pop_front eficiente.

P: ¿priority_queue es estable?
R: No. Elementos con igual prioridad no tienen orden garantizado.

P: ¿Cómo hago una priority_queue que extraiga el mínimo?
R: Usa std::greater<T> como comparador:
   priority_queue<int, vector<int>, greater<int>> pq;

P: ¿Los adaptadores tienen clear()?
R: No. Debes hacer pop() hasta vaciar, o reemplazar el adaptador:
   pila = std::stack<int>();

P: ¿Cuándo usar stack vs vector?
R: stack cuando quieres forzar el comportamiento LIFO y evitar
   accesos indebidos. vector cuando necesitas acceso aleatorio.

usando stack, queue y priority_queue. El sistema debe:

1. Convertir expresiones infijas a postfijas (Shunting Yard):
   - Entrada: "3 + 4 * 2 / (1 - 5)"
   - Usar stack para operadores y paréntesis.
   - Usar queue para la salida.

2. Evaluar expresiones postfijas con stack:
   - "3 4 2 * 1 5 - / +"
   - Manejar +, -, *, /, ^ (potencia con **).

3. Soporte para funciones: sin, cos, sqrt, log.
   - Almacenar en un map<string, function<double(double)>>.

4. Variables y asignaciones:
   - stack<map<string,double>> para ámbitos anidados.
   - queue para comandos de asignación.

5. Historial de operaciones:
   - Usar stack<string> para deshacer (undo).
   - Cada operación se almacena en el stack de historial.
   - Deshacer restaura el estado anterior.

6. Cola de prioridad para operaciones largas:
   - Las expresiones grandes (>20 tokens) tienen baja prioridad.
   - Las expresiones con funciones tienen alta prioridad.
   - priority_queue<Expresion> para procesar en orden.

7. El programa debe funcionar en modo REPL (Read-Eval-Print-Loop)
   interactivo, aceptando entrada del usuario y mostrando resultados.

Requisitos técnicos:
- stack<char> para convertir infija a postfija.
- queue<string> para salida postfija.
- stack<double> para evaluar postfija.
- priority_queue para ordenar expresiones.
- map<string, function> para funciones matemáticas.

Este desafío integra todos los conceptos del capítulo: stack LIFO,
queue FIFO, priority_queue, y cómo combinarlos para crear un
sistema completo.
