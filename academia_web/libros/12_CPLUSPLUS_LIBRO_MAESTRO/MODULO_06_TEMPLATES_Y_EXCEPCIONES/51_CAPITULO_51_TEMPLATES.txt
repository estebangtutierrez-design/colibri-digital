¡Bienvenido al Módulo VI de C++ Master Book! Hasta ahora hemos dominado la

Los templates son el mecanismo de C++ para programación genérica. Te permiten
escribir código que funciona con cualquier tipo de dato, sin sacrificar
rendimiento. A diferencia de los genéricos en otros lenguajes (Java, C#),
los templates de C++ se resuelven completamente en tiempo de compilación,
generando código optimizado para cada tipo concreto.

En este capítulo sentaremos las bases: qué son, por qué existen, cómo se
diferencian de la sobrecarga de funciones, y cómo el compilador genera
código a partir de ellos. Prepárate para uno de los conceptos más poderosos
(y complejos) de C++.

1. ¿QUÉ SON LOS TEMPLATES?
--------------------------

Un template es una plantilla que describe cómo generar una función o una
clase para un tipo específico. Piensa en un template como un molde: el molde
no es la pieza final, sino que defines el molde una vez y produces tantas
piezas (instancias) como necesites.

El compilador toma el template y lo "rellena" con el tipo que le
proporciones, generando código máquina específico para ese tipo. Esto se
conoce como "instanciación de template".

SINTAXIS BÁSICA:

  template <typename T>
  T funcion(T a, T b) {
      return a + b;
  }

  template <class T>
  class MiClase {
      T dato;
  };

Nota: 'typename' y 'class' son equivalentes en este contexto.

2. GENERACIÓN DE CÓDIGO EN TIEMPO DE COMPILACIÓN
-------------------------------------------------

A diferencia de otras características de C++, los templates se evalúan
completamente en tiempo de compilación. Esto significa:

  - El compilador genera una versión separada de la función/clase para cada
    tipo con el que se use.
  - No hay overhead en tiempo de ejecución por el uso de templates.
  - El código resultante es tan eficiente como si hubieras escrito la
    versión específica a mano.
  - Los errores de tipo se detectan en compilación, no en ejecución.

Esto contrasta con el polimorfismo dinámico (virtual functions), donde las
decisiones se toman en tiempo de ejecución y hay overhead por la tabla V.

3. TEMPLATES VS SOBRECARGA DE FUNCIONES
----------------------------------------

Antes de los templates, la única forma de tener una función que operara
sobre múltiples tipos era mediante la sobrecarga:

  int maximo(int a, int b) { return (a > b) ? a : b; }
  double maximo(double a, double b) { return (a > b) ? a : b; }
  char maximo(char a, char b) { return (a > b) ? a : b; }

Esto funciona, pero es repetitivo y propenso a errores. Con templates:

  template <typename T>
  T maximo(T a, T b) { return (a > b) ? a : b; }

Una sola definición sirve para cualquier tipo que soporte el operador >.

4. EL PROCESO DE COMPILACIÓN DE TEMPLATES
------------------------------------------

La compilación de templates ocurre en dos fases:

  Fase 1: Cuando el compilador encuentra la definición del template,
          verifica la sintaxis básica pero NO genera código.

  Fase 2: Cuando encuentra una instanciación (uso) del template, genera
          el código reemplazando T por el tipo concreto.

Esto tiene implicaciones importantes:
  - La definición completa del template debe estar disponible en el punto
    de uso (típicamente en archivos .h, no .cpp).
  - Los errores pueden aparecer en cualquiera de las dos fases.

5. PARÁMETROS DE TIPO Y PARÁMETROS NO-TYPE
-------------------------------------------

Los templates pueden recibir dos tipos de parámetros:

  Parámetros de tipo (type parameters):
      template <typename T, typename U>
      T convertir(U valor) { return static_cast<T>(valor); }

  Parámetros no-type (non-type parameters):
      template <typename T, int N>
      class Array {
          T datos[N];
      };

      Array<int, 10> arr;  // Array de 10 enteros

Los parámetros no-type deben ser constantes en tiempo de compilación:
  - enteros, enums, punteros, referencias, o std::nullptr_t.

6. INSTANCIACIÓN IMPLÍCITA Y EXPLÍCITA
---------------------------------------

  Instanciación implícita:
      template <typename T>
      T suma(T a, T b) { return a + b; }

      int r = suma(3, 4);       // T se deduce como int
      double r2 = suma(3.5, 2.0); // T se deduce como double

  Instanciación explícita:
      template <typename T>
      T producto(T a, T b) { return a * b; }

      template int producto<int>(int, int); // Fuerza la instanciación

  Instanciación explícita con tipo:
      auto r = producto<double>(3.0, 4.0); // T = double, explícito

7. DEDUCCIÓN DE ARGUMENTOS DE TEMPLATE
----------------------------------------

El compilador puede deducir los tipos de los parámetros del template a
partir de los argumentos de la función:

  template <typename T>
  void print(T valor) { std::cout << valor; }

  print(42);      // T = int
  print(3.14);    // T = double
  print("hola");  // T = const char*

Sin embargo, hay limitaciones:
  - La deducción no funciona para tipos de retorno (debes especificarlos).
  - La deducción falla si hay ambigüedad.

8. ESPECIALIZACIÓN Y SOBRECARGA DE TEMPLATES
---------------------------------------------

Los templates se pueden especializar para tipos específicos:

  template <typename T>
  T maximo(T a, T b) { return (a > b) ? a : b; }

  // Especialización para const char*
  template <>
  const char* maximo<const char*>(const char* a, const char* b) {
      return (strcmp(a, b) > 0) ? a : b;
  }

La sobrecarga de funciones también puede coexistir con templates:

  template <typename T> void func(T a) { }
  void func(int a) { }  // Sobrecarga no-template

Las reglas de resolución favorecen las funciones no-template cuando hay
coincidencia exacta.

9. VENTAJAS Y DESVENTAJAS DE LOS TEMPLATES
-------------------------------------------

  VENTAJAS:
  - Código reusable para cualquier tipo.
  - Rendimiento óptimo (sin overhead en runtime).
  - Detección de errores en compilación.
  - Permite bibliotecas completas (STL, Boost) basadas en templates.

  DESVENTAJAS:
  - Mayor tiempo de compilación.
  - Código binario más grande (inflación de código).
  - Mensajes de error crípticos y extensos.
  - Mayor complejidad conceptual.
  - La definición debe estar en headers.

```cpp
// EJEMPLO 1: Template básico - Función maximo
//==============================================================================
#include <iostream>
#include <string>
#include <cstring>

// Definición del template
template <typename T>
T maximo(T a, T b) {
    return (a > b) ? a : b;
}

// Especialización para const char*
template <>
const char* maximo<const char*>(const char* a, const char* b) {
    return (std::strcmp(a, b) > 0) ? a : b;
}

int main() {
    int a = 10, b = 20;
    std::cout << "maximo(10, 20) = " << maximo(a, b) << "\n";

    double x = 3.14, y = 2.71;
    std::cout << "maximo(3.14, 2.71) = " << maximo(x, y) << "\n";

    std::string s1 = "hola", s2 = "mundo";
    std::cout << "maximo(hola, mundo) = " << maximo(s1, s2) << "\n";

    const char* c1 = "alpha";
    const char* c2 = "beta";
    std::cout << "maximo(alpha, beta) = " << maximo(c1, c2) << "\n";

    return 0;
}
```

```cpp
// EJEMPLO 2: Parámetros no-type
//==============================================================================
#include <iostream>

template <typename T, int N>
class ArrayEstatico {
private:
    T datos[N];

public:
    ArrayEstatico() {
        for (int i = 0; i < N; ++i)
            datos[i] = T();
    }

    T& operator[](int index) {
        return datos[index];
    }

    const T& operator[](int index) const {
        return datos[index];
    }

    int tamaño() const { return N; }

    void llenar(const T& valor) {
        for (int i = 0; i < N; ++i)
            datos[i] = valor;
    }

    void imprimir() const {
        for (int i = 0; i < N; ++i)
            std::cout << datos[i] << " ";
        std::cout << "\n";
    }
};

int main() {
    ArrayEstatico<int, 5> arr;
    arr.llenar(42);
    arr[2] = 99;
    arr.imprimir();

    ArrayEstatico<double, 3> arr2;
    arr2[0] = 1.1;
    arr2[1] = 2.2;
    arr2[2] = 3.3;
    arr2.imprimir();

    std::cout << "Tamaño arr: " << arr.tamaño() << "\n";
    std::cout << "Tamaño arr2: " << arr2.tamaño() << "\n";

    return 0;
}
```

```cpp
//==============================================================================
// EJEMPLO 3: Múltiples parámetros de tipo
//==============================================================================
#include <iostream>
#include <typeinfo>

template <typename T, typename U>
class Par {
private:
    T primero;
    U segundo;

public:
    Par() : primero(T()), segundo(U()) {}
    Par(const T& p, const U& s) : primero(p), segundo(s) {}

    T& getPrimero() { return primero; }
    const T& getPrimero() const { return primero; }

    U& getSegundo() { return segundo; }
    const U& getSegundo() const { return segundo; }

    void imprimir() const {
        std::cout << "(" << primero << ", " << segundo << ")\n";
    }

    // Template de función miembro
    template <typename V, typename W>
    Par<V, W> convertir() const {
        return Par<V, W>(static_cast<V>(primero), static_cast<W>(segundo));
    }
};

int main() {
    Par<int, double> p1(10, 3.14);
    p1.imprimir();

    Par<std::string, int> p2("edad", 25);
    p2.imprimir();

    // Conversión de tipos usando template miembro
    auto p3 = p1.convertir<double, int>();
    p3.imprimir();

    // Uso del operador typeid
    std::cout << "Tipo de p1: (" << typeid(p1.getPrimero()).name()
              << ", " << typeid(p1.getSegundo()).name() << ")\n";

    return 0;
}
```

```cpp
// EJEMPLO 5: Compilación en dos fases y errores comunes

#include <iostream>
#include <vector>

template <typename T>
class DemoCompilacion {
private:
    T dato;

public:
    DemoCompilacion(const T& d) : dato(d) {}

    // Este método solo funciona si T soporta el operador +
    T duplicar() const {
        return dato + dato;
    }

    // Este método solo funciona si T tiene .toString()
    void imprimir() const {
        // Error FASE 2 si T no tiene toString()
        // std::cout << dato.toString();
        // Solución: usar operator<<
        std::cout << dato << "\n";
    }
};

// Clase que NO tiene operator+
struct SinSuma {
    int valor;
    explicit SinSuma(int v) : valor(v) {}
};

// Necesitamos operator<< para imprimir
std::ostream& operator<<(std::ostream& os, const SinSuma& s) {
    os << "SinSuma(" << s.valor << ")";
    return os;
}

int main() {
    DemoCompilacion<int> di(10);
    std::cout << "Duplicado de 10: " << di.duplicar() << "\n";

    DemoCompilacion<double> dd(3.5);
    std::cout << "Duplicado de 3.5: " << dd.duplicar() << "\n";

    DemoCompilacion<std::string> ds("Hola ");
    std::cout << "Duplicado de Hola: " << ds.duplicar() << "\n";

    // DemoCompilacion<SinSuma> dss(SinSuma(5));
    // std::cout << dss.duplicar();  // ERROR: SinSuma no tiene operator+

    DemoCompilacion<SinSuma> dss(SinSuma(5));
    dss.imprimir();  // Funciona porque SinSuma tiene operator<<
```

```cpp
//==============================================================================
// EJEMPLO 6: Concepto de polimorfismo en tiempo de compilación
//==============================================================================
#include <iostream>

// Polimorfismo en tiempo de compilación (static polymorphism) via templates
template <typename T>
class Operaciones {
public:
    static T suma(const T& a, const T& b) { return a + b; }
    static T resta(const T& a, const T& b) { return a - b; }
    static T multiplica(const T& a, const T& b) { return a * b; }
    static T divide(const T& a, const T& b) { return a / b; }
};

// Versión sin template (para comparación)
class OperacionesInt {
public:
    static int suma(int a, int b) { return a + b; }
    static int resta(int a, int b) { return a - b; }
    static int multiplica(int a, int b) { return a * b; }
    static int divide(int a, int b) { return a / b; }
};

// Función template que usa el concepto de "tipo con operadores aritméticos"
template <typename T>
T calcularMedia(const T& a, const T& b) {
    return Operaciones<T>::suma(a, b) / T(2);
}

int main() {
    std::cout << "Media de 10 y 20: " << calcularMedia(10, 20) << "\n";
    std::cout << "Media de 3.5 y 4.5: " << calcularMedia(3.5, 4.5) << "\n";

    // Comparación: el código generado es idéntico en eficiencia
    int r1 = OperacionesInt::suma(10, 20);
    int r2 = Operaciones<int>::suma(10, 20);
    std::cout << "r1 = " << r1 << ", r2 = " << r2 << " (mismo resultado)\n";

    return 0;
}
```

```cpp
//==============================================================================
// EJEMPLO 9: Problemas comunes con templates y soluciones
//==============================================================================
#include <iostream>
#include <vector>
#include <type_traits>

// PROBLEMA 1: Código duplicado - Solución: templates
// En lugar de escribir:

int sumarInts(int a, int b, int c) { return a + b + c; }
double sumarDoubles(double a, double b, double c) { return a + b + c; }

// Escribimos:

template <typename T>
T sumar(T a, T b, T c) { return a + b + c; }

// PROBLEMA 2: Dependencia de tipo (typename vs class)
template <class T>
void func1(T a) { }

template <typename T>
void func2(T a) { }

// Son equivalentes, pero typename es más claro semánticamente

// PROBLEMA 3: La palabra clave 'template' para miembros dependientes
template <typename T>
class Outer {
public:
    template <typename U>
    class Inner {
    public:
        U valor;
    };

    Inner<T> crear() {
        return Inner<T>();
    }
};

template <typename T>
void procesar() {
    Outer<int> o;
    // Si Inner fuera dependiente, necesitaríamos:
    // typename Outer<T>::template Inner<T> ...
    typename Outer<int>::Inner<int> obj = o.crear();
    obj.valor = 42;
}

// PROBLEMA 4: typename para tipos dependientes
template <typename T>
void ejemplo() {
    // T::AlgunTipo* ptr;  // ERROR: el compilador no sabe si AlgunTipo es un tipo
    // Solución:
    typename T::AlgunTipo* ptr;
    // Con typename, decimos al compilador que AlgunTipo ES un tipo
}

struct EjemploConTipo {
    using AlgunTipo = int;
    AlgunTipo dato = 10;
};

int main() {
    procesar<int>();

    // Uso de sumar con tipos deducidos
    std::cout << "sumar(1,2,3): " << sumar(1, 2, 3) << "\n";
    std::cout << "sumar(1.1,2.2,3.3): " << sumar(1.1, 2.2, 3.3) << "\n";

    // Verificar typename con tipos dependientes
    Outer<int> o;
    auto inner = o.crear();
    inner.valor = 100;
    std::cout << "Inner valor: " << inner.valor << "\n";

    return 0;
}
```

```cpp
//==============================================================================
// EJEMPLO 10: Template y rendimiento - Comparación con punteros a función
//==============================================================================
#include <iostream>
#include <functional>
#include <chrono>

// Usando template (polimorfismo estático / compile-time)
template <typename Operacion>
int ejecutarTemplate(int a, int b, Operacion op) {
    return op(a, b);
}

// Usando std::function (polimorfismo dinámico / runtime)
int ejecutarFunction(int a, int b, std::function<int(int,int)> op) {
    return op(a, b);
}

// Usando puntero a función
int ejecutarPtr(int a, int b, int (*op)(int, int)) {
    return op(a, b);
}

int suma(int a, int b) { return a + b; }
int resta(int a, int b) { return a - b; }
int multi(int a, int b) { return a * b; }

// Lambdas para template
auto lambdaSuma = [](int a, int b) { return a + b; };
auto lambdaResta = [](int a, int b) { return a - b; };
auto lambdaMulti = [](int a, int b) { return a * b; };

int main() {
    constexpr int ITERACIONES = 10000000;

    // Con template (compile-time, sin overhead)
    long long total1 = 0;
    for (int i = 0; i < ITERACIONES; ++i) {
        total1 += ejecutarTemplate(i, i+1, lambdaSuma);
    }
    std::cout << "Template result: " << total1 << "\n";

    // Con std::function (runtime, con overhead)
    long long total2 = 0;
    std::function<int(int,int)> f = lambdaSuma;
    for (int i = 0; i < ITERACIONES; ++i) {
        total2 += ejecutarFunction(i, i+1, f);
    }
    std::cout << "Function result: " << total2 << "\n";

    // Con puntero a función
    long long total3 = 0;
    for (int i = 0; i < ITERACIONES; ++i) {
        total3 += ejecutarPtr(i, i+1, suma);
    }
    std::cout << "Ptr result: " << total3 << "\n";

    std::cout << "\nNota: El template genera codigo inline optimizado.\n";
    std::cout << "std::function tiene overhead por type erasure.\n";
    std::cout << "Puntero a funcion tiene overhead por indireccion.\n";

    return 0;
}
```

```cpp
//==============================================================================
// EJEMPLO 11: Template SFINAE (Substitution Failure Is Not An Error)
//==============================================================================
#include <iostream>
#include <type_traits>

// SFINAE: si la sustitución de un tipo en un template falla,
// el compilador no lo considera un error, simplemente descarta
// esa sobrecarga y busca otra.

// Versión para tipos enteros
template <typename T,
          typename = std::enable_if_t<std::is_integral_v<T>>>
T mitad(T valor) {
    std::cout << "Version entera: ";
    return valor / 2;
}

// Versión para tipos de punto flotante
template <typename T,
          typename = std::enable_if_t<std::is_floating_point_v<T>>,
          typename = void>  // dummy para diferenciar
T mitad(T valor) {
    std::cout << "Version flotante: ";
    return valor / 2.0;
}

// Función que imprime el tipo
template <typename T>
void imprimirTipo(const T&) {
    if constexpr (std::is_integral_v<T>) {
        std::cout << "Tipo entero\n";
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << "Tipo flotante\n";
    } else {
        std::cout << "Otro tipo\n";
    }
}

int main() {
    std::cout << mitad(10) << "\n";         // entero: 5
    std::cout << mitad(10.0) << "\n";       // flotante: 5
    // mitad("texto");  // Error: no hay sobrecarga para const char*

    imprimirTipo(42);
    imprimirTipo(3.14);
    imprimirTipo("hola");

    return 0;
}
```

```cpp
// EJEMPLO 12: if constexpr con templates (C++17)
//==============================================================================
#include <iostream>
#include <type_traits>

template <typename T>
void procesarValor(const T& valor) {
    // if constexpr evalúa en tiempo de compilación
    if constexpr (std::is_pointer_v<T>) {
        std::cout << "Es puntero: " << *valor << "\n";
    } else if constexpr (std::is_integral_v<T>) {
        std::cout << "Es entero: " << valor << "\n";
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << "Es flotante: " << valor << "\n";
    } else {
        std::cout << "Es otro tipo: " << valor << "\n";
    }
}

// Versión tradicional (pre-C++17) con sobrecarga
template <typename T, std::enable_if_t<std::is_integral_v<T>, int> = 0>
void tradicional(T v) {
    std::cout << "Entero tradicional: " << v << "\n";
}

template <typename T, std::enable_if_t<std::is_floating_point_v<T>, int> = 0>
void tradicional(T v) {
    std::cout << "Flotante tradicional: " << v << "\n";
}

int main() {
    procesarValor(42);
    procesarValor(3.14);
    procesarValor("Hola mundo");
    int x = 100;
    procesarValor(&x);

    tradicional(42);
    tradicional(3.14);

    return 0;
}
```

RESUMEN DEL CAPÍTULO
--------------------

En este capítulo hemos aprendido:

  1. Los templates son plantillas que generan código en tiempo de
     compilación para tipos específicos.

  2. La sintaxis básica es template <typename T> o template <class T>.

  3. A diferencia de la sobrecarga, un template es una definición única
     que funciona para cualquier tipo.

  4. La compilación ocurre en dos fases: verificación sintáctica y
     generación de código.

  5. Existen parámetros de tipo (typename) y parámetros no-type (int, etc.).

  6. La instanciación puede ser implícita (deducción) o explícita.

  7. Los templates permiten polimorfismo en tiempo de compilación (static
     polymorphism) sin overhead en runtime.

  8. La especialización permite adaptar el comportamiento para tipos
     específicos.

  9. Las ventajas incluyen reusabilidad y rendimiento; las desventajas
     incluyen mayor tiempo de compilación y código binario más grande.

  10. Técnicas avanzadas como SFINAE y if constexpr permiten escribir
      código genérico sofisticado.

FAQ - PREGUNTAS FRECUENTES
---------------------------

  P: ¿Cuál es la diferencia entre typename y class en la declaración de
     template?
  R: Ninguna. Son completamente equivalentes. Se recomienda usar typename
  para parámetros de tipo genérico y class cuando se espera que el
  parámetro sea un tipo de clase, pero ambos son intercambiables.

  P: ¿Por qué los templates deben definirse en archivos .h y no en .cpp?
  R: Porque el compilador necesita la definición completa en el punto de
  instanciación. Si la definición está en un .cpp, otros archivos no
  podrán instanciar el template. Las soluciones incluyen definición en
  headers, instanciación explícita, o usar el patrón de inclusión (.inl).

  P: ¿Qué es la inflación de código (code bloat) en templates?
  R: Es el aumento del tamaño del binario porque el compilador genera
  código separado para cada combinación de tipos. Si usas
  vector<int>, vector<double>, vector<string>, tendrás tres copias
  completas del código de vector.

  P: ¿Cómo puedo reducir la inflación de código?
  R: Usando instanciación explícita en un solo .cpp, factorizando código
  común en clases base no-template, o usando técnicas como el "thin
  template" pattern.

  P: ¿Los templates hacen que el programa sea más lento?
  R: No. De hecho, los templates pueden ser más rápidos que las
  alternativas (punteros a función, virtual functions, std::function)
  porque todo se resuelve en compilación y el compilador puede inline
  el código generado.

  P: ¿Qué es CTAD (Class Template Argument Deduction)?
  R: Es una característica de C++17 que permite deducir los argumentos
  de un template de clase a partir del constructor. Ejemplo:
  std::vector v = {1, 2, 3};  // deduce vector<int>

  P: ¿Qué significa "dependient name" en templates?
  R: Es un nombre que depende de un parámetro del template. Por ejemplo,
  T::iterator es un nombre dependiente. El compilador no puede saber qué
  es hasta la instanciación. Se debe usar 'typename' para indicar que es
  un tipo y 'template' para indicar que es un template.

DESAFÍO DEL ARQUITECTO

  Implementa un sistema de "serialización genérica" usando templates que
  pueda convertir cualquier tipo de dato a string y viceversa.

  Requisitos:
    - Template de función 'serializar' que convierta un valor a std::string.
    - Template de función 'deserializar' que convierta de std::string al tipo.
    - Debe funcionar para tipos primitivos (int, double, bool, etc.).
    - Debe funcionar para std::string (identidad).
    - Debe funcionar para pares std::pair<T,U> (serializando ambos elementos).
    - Debe funcionar para contenedores STL (vector, list, map) serializando
      todos los elementos.
    - Manejar errores de formato en deserialización (lanzar excepción).
    - Usar SFINAE o if constexpr para seleccionar la implementación correcta
      según el tipo.

  Pistas:
    - Para tipos primitivos, usa std::to_string y std::istringstream.
    - Para contenedores, usa iteradores y serializa cada elemento.
    - Define separadores para listas y pares.
    - El formato podría ser: "int:42" o "pair:(int:1,double:3.14)".
    - Usa std::is_same_v, std::is_arithmetic_v para detectar tipos.

  Este desafío te preparará para los capítulos avanzados de templates
  y metaprogramación.
