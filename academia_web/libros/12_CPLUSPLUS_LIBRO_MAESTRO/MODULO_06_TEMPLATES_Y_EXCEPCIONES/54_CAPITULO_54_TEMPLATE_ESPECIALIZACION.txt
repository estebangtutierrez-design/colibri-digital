BIENVENIDA
----------

Bienvenido a uno de los temas más avanzados y poderosos de los templates
de C++: la especialización. Hasta ahora hemos escrito templates que
funcionan para cualquier tipo. Pero ¿qué pasa cuando necesitas un
comportamiento diferente para un tipo específico?

La especialización de templates te permite proporcionar implementaciones
alternativas para tipos concretos, manteniendo el template general para
el resto. Esto es fundamental para optimizar código, manejar casos
especiales, y construir bibliotecas como la STL.

Además, exploraremos los type traits: herramientas que nos permiten
interrogar y manipular tipos en tiempo de compilación. Conceptos como
std::is_integral, std::enable_if, y std::conditional son la base de la
metaprogramación en C++.

FRASE DEL CAPÍTULO
------------------

  "La especialización es donde los templates dejan de ser genéricos

1. ¿QUÉ ES LA ESPECIALIZACIÓN DE TEMPLATES?
--------------------------------------------

Especializar un template significa proporcionar una implementación
diferente para uno o más parámetros de template específicos.

  template <typename T>
  struct MiStruct {
      static constexpr const char* nombre = "Generico";
  };

  // Especialización total para int
  template <>
  struct MiStruct<int> {
      static constexpr const char* nombre = "Especializado para int";
  };

2. ESPECIALIZACIÓN TOTAL (FULL SPECIALIZATION)
-----------------------------------------------

Ocurre cuando TODOS los parámetros del template están especificados:

  template <typename T, typename U>
  class Par { /* ... */ };

  template <>  // Todos los parámetros especificados
  class Par<int, double> { /* ... */ };

3. ESPECIALIZACIÓN PARCIAL (PARTIAL SPECIALIZATION)
----------------------------------------------------

Ocurre cuando SOLO ALGUNOS parámetros están especificados:

  template <typename T, typename U>
  class Par { /* ... */ };

  template <typename T>  // U está fijo como int
  class Par<T, int> { /* ... */ };

  template <typename U>  // T está fijo como double
  class Par<double, U> { /* ... */ };

Solo las CLASES pueden tener especialización parcial. Las funciones
deben usar sobrecarga en su lugar.

4. ESPECIALIZACIÓN DE FUNCIONES TEMPLATE
------------------------------------------

  template <typename T>
  T maximo(T a, T b) {
      return (a > b) ? a : b;
  }

  // Especialización total para const char*
  template <>
  const char* maximo<const char*>(const char* a, const char* b) {
      return (strcmp(a, b) > 0) ? a : b;
  }

Nota: Para funciones, la especialización total es posible pero a menudo
se prefiere la sobrecarga.

5. TYPE TRAITS: INTROSPECCIÓN DE TIPOS
---------------------------------------

Los type traits son templates que proporcionan información sobre tipos
en tiempo de compilación.

  #include <type_traits>

  std::is_integral<int>::value           // true
  std::is_integral<double>::value        // false
  std::is_floating_point<float>::value   // true
  std::is_pointer<int*>::value           // true
  std::is_const<const int>::value        // true
  std::is_same<int, int>::value          // true

  // Helper variables (C++17):
  std::is_integral_v<int>                // true

6. enable_if: HABILITACIÓN CONDICIONAL
---------------------------------------

enable_if es un type trait que habilita o deshabilita una sobrecarga
dependiendo de una condición en tiempo de compilación.

  template <typename T>
  std::enable_if_t<std::is_integral_v<T>, T>
  funcion(T valor) {
      return valor * 2;
  }

  template <typename T>
  std::enable_if_t<std::is_floating_point_v<T>, T>
  funcion(T valor) {
      return valor * 2.5;
  }

7. SFINAE EN PROFUNDIDAD
--------------------------

SFINAE (Substitution Failure Is Not An Error) es el principio por el
cual, si la sustitución de tipos en un template falla, el compilador
no reporta un error sino que simplemente descarta esa sobrecarga.

  template <typename T>
  auto incremento(T valor) -> decltype(valor + 1) {
      return valor + 1;
  }

  // Si T no soporta operator+, esta sobrecarga se descarta silenciosamente

8. CONDITIONAL Y OTROS TRAITS
-------------------------------

  template <bool Condicion, typename T, typename U>
  struct conditional;

  // Uso:
  typename std::conditional<true, int, double>::type  // int
  typename std::conditional<false, int, double>::type // double

  // Otros traits útiles:
  std::remove_const<const int>::type       // int
  std::add_pointer<int>::type              // int*
  std::decay_t<int&>                        // int
  std::common_type_t<int, double>          // double

9. ESPECIALIZACIÓN PARCIAL CON PATRONES
------------------------------------------

  template <typename T>
  class MiClase { /* generico */ };

  template <typename T>
  class MiClase<T*> { /* especializacion para punteros */ };

  template <typename T>
  class MiClase<std::vector<T>> { /* especializacion para vector */ };

  template <typename T, typename U>
  class MiClase<std::pair<T, U>> { /* especializacion para pair */ };

10. METAPROGRAMACIóN CON TRAITS
--------------------------------

Los type traits permiten escribir código que se ejecuta en tiempo de
compilación:

  template <typename T>
  struct TipoInfo {
      static constexpr bool esEntero = std::is_integral_v<T>;
      static constexpr bool esFlotante = std::is_floating_point_v<T>;
      static constexpr bool esAritmetico = esEntero || esFlotante;
      static constexpr size_t tamano = sizeof(T);
  };

//==============================================================================
#include <iostream>
#include <cstring>
#include <string>

T maximo(T a, T b) {
    std::cout << "Template generico: ";

    std::cout << "Especializacion const char*: ";

}

// Especialización total para std::string (convierte a string)
template <>

    std::cout << "Especializacion string: ";
    return (a > b) ? a : b;

    std::cout << maximo(10, 20) << "\n";
    std::cout << maximo(3.14, 2.71) << "\n";
    std::cout << maximo("alfa", "beta") << "\n";
    std::cout << maximo<std::string>("hola", "mundo") << "\n";

    return 0;

#include <iostream>
#include <string>

// Template general
template <typename T>
class Procesador {
public:
    void procesar(const T& valor) {
        std::cout << "Procesador generico: " << valor << "\n";
    }
};

// Especialización total para int
template <>
class Procesador<int> {
public:
    void procesar(const int& valor) {
        std::cout << "Procesador especializado para int: "

    }
};

template <>
class Procesador<std::string> {
public:
    void procesar(const std::string& valor) {
        std::cout << "Procesador especializado para string: "
                  << "\"" << valor << "\" (longitud="
                  << valor.length() << ")\n";
    }
};

// Especialización total para bool
template <>
class Procesador<bool> {
public:
    void procesar(const bool& valor) {
        std::cout << "Procesador especializado para bool: "
                  << (valor ? "VERDADERO" : "FALSO") << "\n";
    }
};

int main() {
    Procesador<double> pd;
    pd.procesar(3.14);

    Procesador<int> pi;
    pi.procesar(42);

    Procesador<std::string> ps;
    ps.procesar("Hola mundo");

    Procesador<bool> pb;
    pb.procesar(true);
    pb.procesar(false);

    return 0;
}

// EJEMPLO 3: Especialización parcial de clases
//==============================================================================
#include <iostream>

// Template general con dos parámetros
template <typename T, typename U>
class Par {
public:
    void imprimir() const {
        std::cout << "Par generico<T, U>\n";
    }
};

// Especialización parcial: ambos tipos son iguales
template <typename T>
class Par<T, T> {
public:
    void imprimir() const {
        std::cout << "Par<T, T> - mismo tipo\n";
    }
};

// Especialización parcial: T es puntero
template <typename T, typename U>
class Par<T*, U*> {
public:
    void imprimir() const {
        std::cout << "Par<T*, U*> - ambos punteros\n";
    }
};

// Especialización parcial: U es int
template <typename T>
class Par<T, int> {
public:
    void imprimir() const {
        std::cout << "Par<T, int> - U es int\n";
    }
};

// Especialización parcial: T y U son punteros al mismo tipo
template <typename T>
class Par<T*, T*> {
public:
    void imprimir() const {
        std::cout << "Par<T*, T*> - ambos punteros al mismo tipo\n";
    }
};

int main() {
    Par<double, std::string> p1;
    p1.imprimir();  // generico

    Par<int, int> p2;
    p2.imprimir();  // Par<T,T>

    Par<int*, double*> p3;
    p3.imprimir();  // Par<T*,U*>

    Par<double, int> p4;
    p4.imprimir();  // Par<T,int>

    Par<int*, int*> p5;
    p5.imprimir();  // Par<T*,T*>

    return 0;
}

// EJEMPLO 6: enable_if y SFINAE en acción
//==============================================================================
#include <iostream>
#include <type_traits>

// Versión para tipos integrales
template <typename T>
std::enable_if_t<std::is_integral_v<T>, T>
procesar(T valor) {
    std::cout << "Entero: ";
    return valor * 2;
}

// Versión para tipos de punto flotante
template <typename T>
std::enable_if_t<std::is_floating_point_v<T>, T>
procesar(T valor) {
    std::cout << "Flotante: ";
    return valor / 2.0;
}

// Versión para punteros
template <typename T>
std::enable_if_t<std::is_pointer_v<T>, T>
procesar(T ptr) {
    std::cout << "Puntero: ";
    *ptr *= 2;
    return ptr;
}

// Uso con clase
struct MiTipo {};

template <typename T>
std::enable_if_t<std::is_class_v<T>, std::string>
procesar(const T&) {
    return "Tipo clase (no soporta operaciones aritmeticas)";
}

int main() {
    std::cout << procesar(10) << "\n";
    std::cout << procesar(3.14) << "\n";

    int x = 5;
    procesar(&x);
    std::cout << "x despues de procesar: " << x << "\n";

    MiTipo mt;
    std::cout << procesar(mt) << "\n";

    // char también es integral
    std::cout << procesar('A') << "\n";

    return 0;
}

// EJEMPLO 7: enable_if con múltiples condiciones
//==============================================================================
#include <iostream>
#include <type_traits>
#include <string>

// Solo para tipos numéricos (enteros o flotantes)
template <typename T>
std::enable_if_t<std::is_arithmetic_v<T>, T>
suma(T a, T b) {
    return a + b;
}

// Solo para tipos que NO son aritméticos pero tienen operator+
template <typename T>
std::enable_if_t<!std::is_arithmetic_v<T>, T>
suma(const T& a, const T& b) {
    return a + b;
}

// Versión para strings
std::string suma(const std::string& a, const std::string& b) {
    return a + b;
}

// enable_if con tipo de retorno void
template <typename T>
std::enable_if_t<std::is_integral_v<T>>
imprimirDetalle(T valor) {
    std::cout << "Entero con valor: " << valor << "\n";
}

template <typename T>
std::enable_if_t<std::is_floating_point_v<T>>
imprimirDetalle(T valor) {
    std::cout.precision(6);
    std::cout << "Flotante con valor: " << valor << "\n";
}

// Uso de enable_if en parámetros (más legible)
template <typename T>
T duplicar(T valor, std::enable_if_t<std::is_arithmetic_v<T>, int> = 0) {
    return valor * 2;
}

int main() {
    std::cout << "suma(10, 20) = " << suma(10, 20) << "\n";
    std::cout << "suma(3.14, 2.71) = " << suma(3.14, 2.71) << "\n";
    std::cout << "suma(string, string) = " << suma(std::string("hola "), std::string("mundo")) << "\n";

    imprimirDetalle(42);
    imprimirDetalle(3.14159);

    std::cout << "duplicar(5) = " << duplicar(5) << "\n";

//==============================================================================
// EJEMPLO 8: conditional, common_type, y otros traits de transformación
//==============================================================================
#include <iostream>
#include <type_traits>

// Uso de conditional para elegir tipo en tiempo de compilación
template <typename T>
using TipoAdecuado = std::conditional_t<
    std::is_integral_v<T>,
    double,      // Si es entero, usar double
    T            // Si no, mantener el tipo original
>;

template <typename T>
TipoAdecuado<T> mejorar(T valor) {
    if constexpr (std::is_integral_v<T>) {
        return static_cast<double>(valor) * 1.5;
    } else {
        return valor * 1.5;
    }
}

// Uso de common_type para encontrar el tipo común
template <typename T, typename U>
using TipoComun = std::common_type_t<T, U>;

template <typename T, typename U>
TipoComun<T, U> multiplicar(T a, U b) {
    return a * b;
}

// Uso de remove_const, add_pointer, decay
template <typename T>
void mostrarTransformaciones() {
    std::cout << "Original: " << typeid(T).name() << "\n";
    std::cout << "  remove_const: " << typeid(std::remove_const_t<T>).name() << "\n";
    std::cout << "  add_pointer: " << typeid(std::add_pointer_t<T>).name() << "\n";
    std::cout << "  add_reference: " << typeid(std::add_lvalue_reference_t<T>).name() << "\n";
    std::cout << "  decay: " << typeid(std::decay_t<T>).name() << "\n";
    std::cout << "  make_unsigned: " << typeid(std::make_unsigned_t<T>).name() << "\n";
    std::cout << "  make_signed: " << typeid(std::make_signed_t<T>).name() << "\n";
}

int main() {
    std::cout << "mejorar(5) = " << mejorar(5) << " ("
              << typeid(decltype(mejorar(5))).name() << ")\n";

    std::cout << "mejorar(3.5) = " << mejorar(3.5) << " ("
              << typeid(decltype(mejorar(3.5))).name() << ")\n";

    std::cout << "multiplicar(3, 4.5) = " << multiplicar(3, 4.5) << " ("
              << typeid(decltype(multiplicar(3, 4.5))).name() << ")\n";

    std::cout << "\nTransformaciones de const int:\n";
    mostrarTransformaciones<const int>();

    std::cout << "\nTransformaciones de int&:\n";
    mostrarTransformaciones<int&>();

    return 0;
}

//==============================================================================
// EJEMPLO 9: Metaprogramación - calcular en tiempo de compilación
//==============================================================================
#include <iostream>
#include <type_traits>

// Factorial en tiempo de compilación usando templates (clásico)
template <unsigned int N>
struct Factorial {
    static constexpr unsigned int valor = N * Factorial<N - 1>::valor;
};

template <>
struct Factorial<0> {
    static constexpr unsigned int valor = 1;
};

// Potencia en tiempo de compilación
template <unsigned int Base, unsigned int Exp>
struct Potencia {
    static constexpr unsigned long long valor = Base * Potencia<Base, Exp - 1>::valor;
};

template <unsigned int Base>
struct Potencia<Base, 0> {
    static constexpr unsigned long long valor = 1;
};

// Fibonnaci en tiempo de compilación
template <unsigned int N>
struct Fibonacci {
    static constexpr unsigned long long valor = Fibonacci<N - 1>::valor + Fibonacci<N - 2>::valor;
};

template <>
struct Fibonacci<0> {
    static constexpr unsigned long long valor = 0;
};

template <>
struct Fibonacci<1> {
    static constexpr unsigned long long valor = 1;
};

// Type trait personalizado: is_pointer_compatible
template <typename T>
struct is_pointer_compatible : std::integral_constant<bool,
    std::is_trivially_copyable_v<T> && !std::is_pointer_v<T>
> {};

// Type trait: is_numeric
template <typename T>
struct is_numeric : std::integral_constant<bool,
    std::is_integral_v<T> || std::is_floating_point_v<T>
> {};

int main() {
    std::cout << "Factorial<5> = " << Factorial<5>::valor << "\n";
    std::cout << "Factorial<10> = " << Factorial<10>::valor << "\n";

    std::cout << "Potencia<2, 10> = " << Potencia<2, 10>::valor << "\n";
    std::cout << "Potencia<3, 4> = " << Potencia<3, 4>::valor << "\n";

    std::cout << "Fibonacci<10> = " << Fibonacci<10>::valor << "\n";
    std::cout << "Fibonacci<20> = " << Fibonacci<20>::valor << "\n";

    std::cout << "is_pointer_compatible<int>: "
              << is_pointer_compatible<int>::value << "\n";
    std::cout << "is_pointer_compatible<int*>: "
              << is_pointer_compatible<int*>::value << "\n";

    std::cout << "is_numeric<int>: " << is_numeric<int>::value << "\n";
    std::cout << "is_numeric<double>: " << is_numeric<double>::value << "\n";
    std::cout << "is_numeric<std::string>: " << is_numeric<std::string>::value << "\n";

    return 0;
}

//==============================================================================
// EJEMPLO 10: Tag dispatch - alternativa a enable_if
//==============================================================================
#include <iostream>
#include <type_traits>

// Tag dispatch usa sobrecarga basada en tags (tipos vacíos)
struct EnteroTag {};
struct FlotanteTag {};
struct OtroTag {};

// Función auxiliar con tags
template <typename T>
void procesarImpl(T valor, EnteroTag) {
    std::cout << "Entero: " << valor * 2 << "\n";
}

template <typename T>
void procesarImpl(T valor, FlotanteTag) {
    std::cout << "Flotante: " << valor / 2.0 << "\n";
}

template <typename T>
void procesarImpl(T valor, OtroTag) {
    std::cout << "Otro: " << valor << "\n";
}

// Función principal que selecciona el tag
template <typename T>
void procesar(T valor) {
    using Tag = std::conditional_t<
        std::is_integral_v<T>,
        EnteroTag,
        std::conditional_t<
            std::is_floating_point_v<T>,
            FlotanteTag,
            OtroTag
        >
    >;
    procesarImpl(valor, Tag{});
}

// Ejemplo más práctico: avanzar iteradores
struct InputIteratorTag {};
struct ForwardIteratorTag : InputIteratorTag {};
struct BidirectionalIteratorTag : ForwardIteratorTag {};
struct RandomAccessIteratorTag : BidirectionalIteratorTag {};

// Versión para RandomAccess (O(1))
template <typename Iter>
void avanzarImpl(Iter& it, int n, RandomAccessIteratorTag) {
    it += n;
}

// Versión para otros (O(n))
template <typename Iter>
void avanzarImpl(Iter& it, int n, InputIteratorTag) {
    while (n-- > 0) ++it;
}

template <typename Iter>
void avanzar(Iter& it, int n) {
    // En la vida real usarías std::iterator_traits<Iter>::iterator_category
    avanzarImpl(it, n, RandomAccessIteratorTag{});
}

int main() {
    procesar(42);

//==============================================================================
// EJEMPLO 12: Especialización práctica - Serializador genérico
//==============================================================================
#include <iostream>
#include <string>
#include <sstream>
#include <vector>
#include <type_traits>

// Serializador base (usando operator<<)
template <typename T, typename = void>
struct Serializador {
    static std::string serializar(const T& valor) {
        std::ostringstream oss;
        oss << valor;
        return oss.str();
    }
};

// Especialización para tipos integrales
template <typename T>
struct Serializador<T, std::enable_if_t<std::is_arithmetic_v<T>>> {
    static std::string serializar(const T& valor) {
        return std::to_string(valor);
    }
};

// Especialización para bool
template <>
struct Serializador<bool> {
    static std::string serializar(const bool& valor) {
        return valor ? "true" : "false";
    }
};

// Especialización para std::string
template <>
struct Serializador<std::string> {
    static std::string serializar(const std::string& valor) {
        return "\"" + valor + "\"";
    }
};

// Especialización para const char*
template <>
struct Serializador<const char*> {
    static std::string serializar(const char* valor) {
        return "\"" + std::string(valor) + "\"";
    }
};

// Especialización para std::vector
template <typename T>
struct Serializador<std::vector<T>> {
    static std::string serializar(const std::vector<T>& vec) {
        std::string resultado = "[";
        for (size_t i = 0; i < vec.size(); ++i) {
            if (i > 0) resultado += ", ";
            resultado += Serializador<T>::serializar(vec[i]);
        }
        resultado += "]";
        return resultado;
    }
};

// Función de ayuda
template <typename T>
std::string aString(const T& valor) {
    return Serializador<T>::serializar(valor);
}

int main() {
    std::cout << aString(42) << "\n";
    std::cout << aString(3.14159) << "\n";
    std::cout << aString(true) << "\n";
    std::cout << aString(false) << "\n";
    std::cout << aString(std::string("Hola mundo")) << "\n";
    std::cout << aString("Texto literal") << "\n";

    std::vector<int> vec = {1, 2, 3, 4, 5};
    std::cout << aString(vec) << "\n";

    std::vector<std::string> strVec = {"a", "bb", "ccc"};
    std::cout << aString(strVec) << "\n";

    std::vector<double> dVec = {1.1, 2.2, 3.3};
    std::cout << aString(dVec) << "\n";

    return 0;
}

FAQ - PREGUNTAS FRECUENTES
---------------------------

  P: ¿Cuál es la diferencia entre especialización y sobrecarga?
  R: La especialización es para templates: proporcionas una
  implementación alternativa para tipos específicos. La sobrecarga es
  para funciones normales: múltiples funciones con el mismo nombre pero
  diferentes parámetros. Las funciones template prefieren la sobrecarga
  sobre la especialización.

  P: ¿Por qué no se puede especializar parcialmente una función template?
  R: Porque las funciones template ya soportan sobrecarga, que es más
  flexible que la especialización parcial. La sobrecarga permite
  diferenciar por tipos de parámetros, no solo por tipos de template.

  P: ¿Qué es SFINAE?
  R: "Substitution Failure Is Not An Error". Si al sustituir tipos en un
  template se produce una expresión inválida, el compilador no da error
  sino que descarta esa sobrecarga y busca otra. Esto permite escribir
  código que se adapta a las capacidades del tipo.

  P: ¿Qué son los type traits?
  R: Son templates que proporcionan información sobre tipos en tiempo
  de compilación. Por ejemplo, is_integral<T>::value es true si T es
  un tipo entero. Desde C++17, se pueden usar _v: is_integral_v<T>.

  P: ¿Cuándo debo usar enable_if vs if constexpr?
  R: enable_if es necesario cuando necesitas descartar una sobrecarga
  completa (SFINAE). if constexpr es más legible y se evalúa dentro
  del cuerpo de una función template. Prefiere if constexpr cuando sea
  posible (C++17+).

  P: ¿Qué es void_t?
  R: Es un alias de template definido como template<typename...> using
  void_t = void; Se usa en SFINAE para detectar si una expresión es
  válida: si la expresión es válida, void_t se evalúa a void; si no,
  hay SFINAE.

  P: ¿Puedo crear mis propios type traits?
  R: Sí. Los type traits son simplemente templates que heredan de
  std::integral_constant<bool, valor> o std::integral_constant<T, v>.

CIERRE DEL CAPÍTULO
-------------------

La especialización de templates y los type traits representan la
frontera entre la programación genérica básica y la metaprogramación
avanzada en C++. Con estas herramientas, puedes escribir código que se
adapta inteligentemente a las capacidades de cada tipo, optimizando
casos especiales sin perder la generalidad.

Hemos visto cómo SFINAE, enable_if, conditional, y void_t nos permiten
tomar decisiones en tiempo de compilación. Estas técnicas son la base
de bibliotecas como la STL, Boost, y gran parte del ecosistema moderno
de C++.
