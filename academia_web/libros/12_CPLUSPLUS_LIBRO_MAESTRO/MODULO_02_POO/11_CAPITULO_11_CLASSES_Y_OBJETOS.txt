  arreglos, punteros, funciones y estructuras. En este módulo daremos el salto
  más importante en la carrera de todo programador C++: la Programación
  Orientada a Objetos (POO).

  La POO no es solo una característica del lenguaje; es una forma de pensar y
  organizar el código. En lugar de tener funciones dispersas que manipulan
  datos, agrupamos datos y funciones en entidades llamadas clases, y a partir
  de ellas creamos objetos. Este enfoque permite modelar el mundo real de
  manera más natural, facilita el mantenimiento y promueve la reutilización.

  En este capítulo sentaremos las bases: qué es una clase, cómo se define,
  cómo se crean objetos, qué son los miembros (datos y funciones), y cómo
  se organiza la memoria cuando trabajamos con objetos. Dominar estos
  conceptos es esencial antes de avanzar a constructores, destructores,
  herencia y polimorfismo.

  Prepárate para transformar tu forma de programar. La POO en C++ es
  profunda, elegante y poderosa. Comencemos.

  ------------------------------------------------------------------
  1. ¿QUÉ ES UNA CLASE?
  ------------------------------------------------------------------

  Una clase es un tipo de dato definido por el programador que agrupa:

    • Datos miembro (atributos o propiedades) → variables.
    • Funciones miembro (métodos o comportamientos) → funciones.

  Una vez definida la clase, podemos crear variables de ese tipo. A esas
  variables se les llama objetos o instancias de la clase.

  La sintaxis básica es:

  class NombreClase {
      // especificador de acceso:
      // public:   accesible desde fuera de la clase.
      // private:  accesible solo desde dentro de la clase (por defecto).
      // protected: similar a private pero accesible por clases derivadas.

      tipo_dato miembro1;
      tipo_dato miembro2;

      tipo_retorno metodo1(parametros);
      tipo_retorno metodo2(parametros);
  };  // <-- punto y coma obligatorio

  ------------------------------------------------------------------
  2. PUBLIC VS PRIVATE
  ------------------------------------------------------------------

  Por defecto, todos los miembros de una clase son private. Esto significa
  que solo las funciones miembro de la misma clase pueden acceder a ellos.
  Para permitir acceso desde fuera, usamos la etiqueta public:.

  class Rectangulo {
  private:
      double ancho_;
      double alto_;

  public:
      void setDimensiones(double a, double h);
      double calcularArea() const;
      double calcularPerimetro() const;
  };

  Regla general: los datos miembro deben ser private y el acceso a ellos
  se controla mediante funciones públicas (getters/setters), que veremos
  en el capítulo 15.

  ------------------------------------------------------------------
  3. OBJETOS (INSTANCIAS)
  ------------------------------------------------------------------

  Crear un objeto es tan simple como declarar una variable del tipo de la
  clase:

  Rectangulo rect1;           // instancia en la pila (stack)
  Rectangulo* rect2 = new Rectangulo();  // instancia en el heap

  Para acceder a los miembros públicos usamos el operador punto (.) con
  objetos directos, y la flecha (->) con punteros:

  rect1.setDimensiones(10.0, 5.0);    // objeto directo → .
  rect2->setDimensiones(3.0, 4.0);    // puntero → ->

  double area = rect1.calcularArea();

  delete rect2;  // liberar memoria del heap

  ------------------------------------------------------------------
  4. FUNCIONES MIEMBRO (MÉTODOS)
  ------------------------------------------------------------------

  Las funciones miembro se declaran dentro de la clase y se definen de
  dos formas:

  a) Definición inline (dentro de la clase):

     class Circulo {
     public:
         double calcularArea() const {
             return 3.14159 * radio_ * radio_;
         }
     private:
         double radio_;
     };

  b) Definición fuera de la clase (recomendado para proyectos grandes):

     // En el .hpp:
     class Circulo {
     public:
         double calcularArea() const;
     private:
         double radio_;
     };

     // En el .cpp:
     double Circulo::calcularArea() const {
         return 3.14159 * radio_ * radio_;
     }

  El operador :: (scope resolution) indica a qué clase pertenece el método.

  ------------------------------------------------------------------
  5. CALIFICADOR CONST EN FUNCIONES MIEMBRO
  ------------------------------------------------------------------

  Cuando añadimos const después de la lista de parámetros de una función
  miembro, le prometemos al compilador que esa función no modificará ningún
  dato miembro de la clase (a menos que sea mutable).

  double getRadio() const { return radio_; }

  Esto es crucial para poder llamar a métodos sobre objetos constantes
  (const Circulo c).

  ------------------------------------------------------------------
  6. MODELO DE MEMORIA DE LOS OBJETOS
  ------------------------------------------------------------------

  Cuando declaras un objeto, el compilador reserva memoria para todos sus
  datos miembro en el orden en que fueron declarados. Las funciones miembro
  NO ocupan espacio en cada objeto; viven una sola vez en el código y
  reciben un puntero oculto llamado this (capítulo 18) que apunta al objeto
  que las invocó.

  class Punto {
      double x_;   // 8 bytes
      double y_;   // 8 bytes
      double z_;   // 8 bytes → total 24 bytes por objeto
  public:
      void mostrar() const;  // no ocupa espacio en el objeto
  };

  Podemos verificar con sizeof():

  Punto p;
  std::cout << sizeof(p);  // 24 (depende del alineamiento)

  El tamaño de un objeto puede tener padding (relleno) debido al
  alineamiento de memoria. El compilador alinea los miembros para
  optimizar el acceso.

  ------------------------------------------------------------------
  7. SEPARACIÓN DE INTERFAZ E IMPLEMENTACIÓN
  ------------------------------------------------------------------

  En proyectos profesionales, la declaración de la clase va en un archivo
  de encabezado (.hpp o .h) y la implementación en un archivo .cpp.

  circulo.hpp:
      #ifndef CIRCULO_HPP
      #define CIRCULO_HPP

      class Circulo {
      public:
          Circulo(double radio);
          double calcularArea() const;
          double getRadio() const;
      private:
          double radio_;
      };

      #endif

  circulo.cpp:
      #include "circulo.hpp"
      #include <cmath>

      Circulo::Circulo(double radio) : radio_(radio) {}

      double Circulo::calcularArea() const {
          return M_PI * radio_ * radio_;
      }

      double Circulo::getRadio() const {
          return radio_;
      }

  ------------------------------------------------------------------
  8. CREACIÓN DE OBJETOS EN EL HEAP
  ------------------------------------------------------------------

  Podemos crear objetos en el heap con new y debemos liberarlos con delete:

  Circulo* c = new Circulo(5.0);
  std::cout << c->calcularArea();
  delete c;  // liberar memoria

  Si usamos new[], liberamos con delete[]:

  Circulo* arr = new Circulo[3]{1.0, 2.0, 3.0};
  // ...
  delete[] arr;

  ------------------------------------------------------------------
  9. ARREGLOS DE OBJETOS
  ------------------------------------------------------------------

  Podemos crear arreglos de objetos:

  Circulo circulos[3] = {Circulo(1.0), Circulo(2.0), Circulo(3.0)};

  Si la clase no tiene constructor por defecto, debemos inicializar cada
  elemento del arreglo explícitamente.

  ------------------------------------------------------------------
  10. CLASES CON PUNTEROS MIEMBRO
  ------------------------------------------------------------------

  Cuando una clase contiene punteros, debemos manejar la memoria con
  cuidado (regla de tres/cinco, que veremos más adelante).

  class Texto {
  private:
      char* buffer_;
      size_t longitud_;
  public:
      Texto(const char* str);
      ~Texto();                     // destructor
      Texto(const Texto& otro);     // copia
      Texto& operator=(const Texto& otro);  // asignación
  };

  ------------------------------------------------------------------
  11. TYPEDEF Y USING CON CLASES
  ------------------------------------------------------------------

  Podemos crear alias para clases:

  typedef Circulo Circ;
  Circ c(3.0);

  using Circ = Circulo;  // moderno (C++11)
  Circ c2(4.0);

  ------------------------------------------------------------------
  12. CLASES ANIDADAS
  ------------------------------------------------------------------

  Una clase puede definirse dentro de otra:

  class Contenedor {
  public:
      class Anidada {
      public:
          void mostrar() const;
      };
  private:
      int valor_;
  };

  void Contenedor::Anidada::mostrar() const {
      // ...
  }

  Contenedor::Anidada obj;

  ------------------------------------------------------------------
  13. FORWARD DECLARATION
  ------------------------------------------------------------------

  Podemos declarar una clase antes de definirla para usarla como referencia:

  class ClaseB;  // forward declaration

  class ClaseA {
  private:
      ClaseB* ptrB_;
  };

  class ClaseB {
  private:
      ClaseA* ptrA_;
  };

  Esto permite referencias circulares entre clases.

  ------------------------------------------------------------------
  14. MIEMBROS MUTABLES
  ------------------------------------------------------------------

  Un miembro mutable puede ser modificado incluso dentro de funciones
  miembro const:

  class Contador {
  private:
      mutable int accesos_ = 0;
      int valor_;
  public:
      int getValor() const {
          ++accesos_;            // permitido por mutable
          return valor_;
      }
  };

------------[ archivo: circulo.cpp ]------------
#include "circulo.hpp"

Circulo::Circulo(double radio) : radio_(radio) {}

double Circulo::calcularArea() const {
    return M_PI * radio_ * radio_;
}

double Circulo::calcularCircunferencia() const {
    return 2.0 * M_PI * radio_;
}

double Circulo::getRadio() const {
    return radio_;
}

void Circulo::setRadio(double radio) {
    if (radio >= 0.0) {
        radio_ = radio;
    }
}

void Circulo::mostrarInfo() const {
    std::cout << "=== INFORMACIÓN DEL CÍRCULO ===" << std::endl;
    std::cout << "Radio: " << radio_ << std::endl;
    std::cout << "Área: " << calcularArea() << std::endl;
    std::cout << "Circunferencia: " << calcularCircunferencia() << std::endl;
}

------------[ archivo: rectangulo.cpp ]------------
#include "rectangulo.hpp"
#include <iostream>

Rectangulo::Rectangulo(double ancho, double alto)
    : ancho_(ancho), alto_(alto) {}

double Rectangulo::calcularArea() const {
    return ancho_ * alto_;
}

double Rectangulo::calcularPerimetro() const {
    return 2.0 * (ancho_ + alto_);
}

double Rectangulo::getAncho() const { return ancho_; }
double Rectangulo::getAlto() const { return alto_; }

void Rectangulo::setAncho(double ancho) {
    if (ancho > 0.0) ancho_ = ancho;
}

void Rectangulo::setAlto(double alto) {
    if (alto > 0.0) alto_ = alto;
}

void Rectangulo::redimensionar(double factor) {
    if (factor > 0.0) {
        ancho_ *= factor;
        alto_ *= factor;
    }
}

void Rectangulo::mostrar() const {
    std::cout << "Rectángulo [" << ancho_ << " x " << alto_ << "]" << std::endl;
    std::cout << "  Área: " << calcularArea() << std::endl;
    std::cout << "  Perímetro: " << calcularPerimetro() << std::endl;
}

------------[ archivo: main.cpp ]------------
#include <iostream>
#include "circulo.hpp"
#include "rectangulo.hpp"

int main() {
    // === Creación de objetos en la pila ===
    std::cout << "=== OBJETOS EN LA PILA (STACK) ===" << std::endl;

    Circulo c1(5.0);
    c1.mostrarInfo();
    std::cout << std::endl;

    Rectangulo r1(4.0, 7.0);
    r1.mostrar();
    std::cout << std::endl;

    // === Creación de objetos en el heap ===
    std::cout << "=== OBJETOS EN EL HEAP ===" << std::endl;

    Circulo* c2 = new Circulo(2.5);
    c2->mostrarInfo();

    Rectangulo* r2 = new Rectangulo(3.0, 6.0);
    r2->mostrar();

    delete c2;
    delete r2;

    // === Arreglo de objetos ===
    std::cout << "=== ARREGLO DE OBJETOS ===" << std::endl;

    Circulo arr[3] = {Circulo(1.0), Circulo(2.0), Circulo(3.0)};

    for (int i = 0; i < 3; ++i) {
        std::cout << "Círculo " << i + 1 << ": área = "
                  << arr[i].calcularArea() << std::endl;
    }

    // === Demostración de tamaño de objetos ===
    std::cout << "=== TAMAÑO DE OBJETOS ===" << std::endl;
    std::cout << "sizeof(Circulo): " << sizeof(Circulo) << " bytes" << std::endl;
    std::cout << "sizeof(Rectangulo): " << sizeof(Rectangulo) << " bytes" << std::endl;
    std::cout << "sizeof(c1): " << sizeof(c1) << " bytes" << std::endl;

    // === Uso de setters ===
    std::cout << "=== MODIFICACIÓN CON SETTERS ===" << std::endl;
    r1.setAncho(10.0);
    r1.setAlto(20.0);
    r1.mostrar();

    r1.redimensionar(0.5);
    std::cout << "Después de redimensionar al 50%:" << std::endl;
    r1.mostrar();

    return 0;
}

==========================================================================
  RESUMEN DEL CAPÍTULO
==========================================================================

  • Una clase es un tipo de dato definido por el usuario que encapsula
    datos y funciones.
  • Los miembros privados (private) solo son accesibles desde dentro de
    la clase; los públicos (public) desde cualquier parte.
  • Un objeto es una instancia de una clase. Se crea como cualquier
    variable.
  • Usamos . para acceder a miembros de un objeto directo y -> para
    punteros a objetos.
  • Las funciones miembro definidas fuera de la clase usan el operador
    :: (scope resolution).
  • El calificador const después de una función miembro indica que no
    modifica el objeto.
  • Cada objeto tiene su propia copia de los datos miembro; las funciones
    miembro son compartidas por todos los objetos de la clase.
  • El tamaño de un objeto es la suma de sus datos miembro (más posible
    padding por alineamiento).
  • Buena práctica: separar declaración (.hpp) e implementación (.cpp).
  • Los objetos en el heap deben liberarse con delete.

==========================================================================
  PREGUNTAS FRECUENTES (FAQ)
==========================================================================

  P: ¿Cuál es la diferencia entre struct y class en C++?
  R: En C++, las structs tienen todos sus miembros públicos por defecto,
     mientras que las clases los tienen privados. Por lo demás, son
     equivalentes. Las structs se usan típicamente para agrupar datos
     simples (POD), mientras que las clases se usan para encapsulamiento
     completo.

  P: ¿Puedo tener una función miembro con el mismo nombre que un dato
     miembro?
  R: Sí, pero no es recomendable por ambigüedad. La convención es usar
     un sufijo como _ (guion bajo) para los datos miembro y el nombre
     sin guion para los métodos (ej: radio_ y getRadio()).

  P: ¿Qué pasa si no pongo public: antes de mis miembros?
  R: Por defecto todo es private. El compilador generará error si
     intentas acceder desde fuera.

  P: ¿Las funciones miembro inline son más rápidas?
  R: Pueden serlo porque evitan el overhead de llamada a función, pero
     el compilador decide finalmente. Para funciones pequeñas es una
     buena optimización.

  P: ¿Puedo inicializar datos miembro directamente en la clase?
  R: Sí, desde C++11: int contador_ = 0; Esto se llama in-class
     member initializer.

  P: ¿Por qué separar .hpp y .cpp?
  R: Para mejorar tiempos de compilación, ocultar implementación y
     facilitar el mantenimiento en proyectos grandes.

  P: ¿Qué es el padding y cómo afecta al tamaño?
  R: El compilador alinea los datos en direcciones que son múltiplos
     de su tamaño. Esto puede dejar huecos (padding). Por ejemplo, un
     char seguido de un int puede ocupar 8 bytes en lugar de 5.

  P: ¿Cómo accedo a un miembro privado desde fuera de la clase?
  R: Solo a través de métodos públicos (getters y setters). Esa es la
     esencia del encapsulamiento.
