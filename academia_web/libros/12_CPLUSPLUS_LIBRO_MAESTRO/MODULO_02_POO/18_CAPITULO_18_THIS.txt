  Capitulo 18: THIS
================================================================================

  "this no es solo un puntero: es la conciencia del objeto, su capacidad
   de referirse a si mismo dentro de sus propios metodos."

================================================================================
  BIENVENIDA
================================================================================

  Bienvenido al Capitulo 18 de C++ Master Book. Cada vez que llamamos a
  una funcion miembro, el compilador pasa un parametro oculto: un puntero
  al objeto sobre el cual se invoca la funcion. Ese puntero se llama this.

  this es una palabra clave fundamental en C++. Permite que los metodos
  accedan al objeto actual, resuelvan conflictos de nombres, implementen
  encadenamiento de metodos (method chaining), y comparen objetos consigo
  mismos.

  Aunque muchas veces no necesitamos escribir this explicitamente (es
  implicito), entenderlo a fondo es esencial para dominar C++ y escribir
  codigo mas expresivo y seguro.

  ------------------------------------------------------------------
  1. QUE ES THIS?
  ------------------------------------------------------------------

  this es una palabra clave (keyword) de C++ que es un puntero al
  objeto para el cual se invoco la funcion miembro.

  class Ejemplo {
  private:
      int valor_;
  public:
      void setValor(int valor) {
          this->valor_ = valor;  // this-> miembro = parametro
      }

      int getValor() const {
          return this->valor_;   // implicito, se puede omitir
      }
  };

  Cuando llamamos a obj.setValor(5), el compilador pasa la direccion
  de obj como this dentro del metodo.

  ------------------------------------------------------------------
  2. THIS ES IMPLICITO (PERO PUEDE SER EXPLICITO)
  ------------------------------------------------------------------

  Normalmente no necesitamos escribir this-> porque el compilador
  resuelve los nombres de los miembros automaticamente.

  class Punto {
  private:
      int x_, y_;
  public:
      // Sin this (implicito, mas comun):
      void setX(int x) { x_ = x; }

      // Con this (explicito, para claridad):
      void setY(int y) { this->y_ = y; }
  };

  El estilo con this explicito es util cuando hay conflictos o para
  ser mas legible en ciertos contextos.

  ------------------------------------------------------------------
  3. RESOLUCION DE CONFLICTOS DE NOMBRES
  ------------------------------------------------------------------

  El uso mas comun de this es cuando el parametro del metodo tiene
  el mismo nombre que el miembro de la clase:

  class Persona {
  private:
      std::string nombre_;
      int edad_;
  public:
      // Sin this: no se puede, nombre del parametro igual al miembro
      Persona(const std::string& nombre, int edad) {
          // nombre = nombre;  // ERROR: asigna parametro a si mismo
          this->nombre_ = nombre;
          this->edad_ = edad;
      }

      // Mejor solucion: usar inicializacion o nombres diferentes
      Persona(const std::string& n, int e)
          : nombre_(n), edad_(e) {}
  };

  En el constructor con asignacion (no recomendado), this-> es
  necesario para distinguir el miembro del parametro.

  ------------------------------------------------------------------
  4. TYPE OF THIS
  ------------------------------------------------------------------

  El tipo de this depende de si el metodo es const o no:

  En un metodo NO-const:
      this es de tipo Clase* const
      (puntero constante a Clase: no se puede cambiar a donde apunta,
       pero si modificar el objeto).

  En un metodo const:
      this es de tipo const Clase* const
      (puntero constante a Clase constante: no se puede cambiar a
       donde apunta ni modificar el objeto).

  class Demo {
  public:
      void noConst() {
          // this es Demo* const
          this->dato_ = 5;  // OK
          // this = &otro;  // ERROR: this es const*
      }

      void siConst() const {
          // this es const Demo* const
          // this->dato_ = 5;  // ERROR: objeto const
          int v = this->dato_;  // OK: solo lectura
      }
  private:
      int dato_;
  };

  ------------------------------------------------------------------
  5. METHOD CHAINING (ENCADENAMIENTO DE METODOS)
  ------------------------------------------------------------------

  Devolviendo *this desde los setters, podemos encadenar llamadas:

  class StringBuilder {
  private:
      std::string buffer_;
  public:
      StringBuilder& append(const std::string& texto) {
          buffer_ += texto;
          return *this;
      }

      StringBuilder& appendLine(const std::string& texto) {
          buffer_ += texto + "\n";
          return *this;
      }

      StringBuilder& clear() {
          buffer_.clear();
          return *this;
      }

      std::string toString() const { return buffer_; }
  };

  // Uso con chaining:
  StringBuilder sb;
  sb.append("Hola")
    .append(" ")
    .appendLine("Mundo!")
    .append("Esto es ")
    .append("C++!")
    .clear()
    .append("Nuevo comienzo");

  std::cout << sb.toString();

  Este patron es muy usado en:
  - std::cout << "a" << "b" << "c";  (operator<< devuelve ostream&)
  - Qt, jQuery, y muchas APIs modernas.

  ------------------------------------------------------------------
  6. COMPARACION CON SELF (AUTOCOMPARACION)
  ------------------------------------------------------------------

  En operadores de asignacion y copia, debemos verificar que no nos
  asignemos a nosotros mismos:

  class Texto {
  private:
      char* datos_;
  public:
      Texto& operator=(const Texto& otro) {
          if (this != &otro) {  // auto-comprobacion
              delete[] datos_;
              // copiar datos de otro...
          }
          return *this;
      }
  };

  Tambien en comparaciones:

  class Punto {
  private:
      double x_, y_;
  public:
      bool operator==(const Punto& otro) const {
          return (this == &otro) ||  // mismo objeto?
                 (x_ == otro.x_ && y_ == otro.y_);
      }
  };

  ------------------------------------------------------------------
  7. DEVOLVER *THIS DESDE OPERADORES
  ------------------------------------------------------------------

  Los operadores de asignacion (copia y movimiento) devuelven *this
  para permitir asignacion multiple:

  class Contador {
  private:
      int valor_;
  public:
      Contador& operator=(const Contador& otro) {
          if (this != &otro) {
              valor_ = otro.valor_;
          }
          return *this;
      }

      Contador& operator++() {  // pre-incremento
          ++valor_;
          return *this;
      }

      Contador operator++(int) {  // post-incremento
          Contador temp = *this;
          ++(*this);
          return temp;
      }
  };

  // Uso:
  Contador a, b, c;
  a = b = c;       // a = (b = c) -> retorna *this
  ++a;             // retorna *this

  ------------------------------------------------------------------
  8. THIS EN CLASES CON HERENCIA
  ------------------------------------------------------------------

  En herencia, this tiene el tipo de la clase actual, pero puede ser
  upcasteado implicitamente:

  class Base {
  public:
      void mostrarTipo() {
          std::cout << "this es de tipo Base*\n";
      }
  };

  class Derivada : public Base {
  public:
      void mostrarTipo() {
          std::cout << "this es de tipo Derivada*\n";
          Base::mostrarTipo();  // llamada a version base
          // Base* ptr = this;  // upcasting implicito
      }
  };

  ------------------------------------------------------------------
  9. THIS EN LAMBDAS (C++17)
  ------------------------------------------------------------------

  En C++17, podemos capturar *this por copia en lambdas dentro de
  funciones miembro:

  class Computadora {
  private:
      std::string nombre_;
      std::vector<std::string> tareas_;
  public:
      void procesar() {
          auto lambda = [*this]() {  // captura copia del objeto
              // trabajar con una copia de this
              std::cout << "Procesando: " << nombre_ << std::endl;
          };
          lambda();
      }
  };

  Sin *this, capturariamos this por referencia (peligroso si la
  lambda sobrevive al objeto).

  ------------------------------------------------------------------
  10. THIS EN FUNCIONES MIEMBRO STATIC
  ------------------------------------------------------------------

  En funciones miembro static NO existe this. Las funciones static
  no se invocan sobre un objeto en particular, sino sobre la clase.

  class Util {
  public:
      static void helper() {
          // this->algo();  // ERROR: no existe this en static
      }
  };

  ------------------------------------------------------------------
  11. THIS EN TEMPLATES
  ------------------------------------------------------------------

  En clases template, this-> es necesario para acceder a miembros
  heredados (dependientes del template):

  template <typename T>
  class Base {
  protected:
      T valor_;
  };

  template <typename T>
  class Derivada : public Base<T> {
  public:
      void setValor(T v) {
          // valor_ = v;         // ERROR: no dependiente?
          this->valor_ = v;       // OK: this-> hace la busqueda
          Base<T>::valor_ = v;    // OK: calificacion completa
      }
  };

  ------------------------------------------------------------------
  12. FUNCIONES MIEMBRO QUE DEVUELVEN REFERENCIA A THIS
  ------------------------------------------------------------------

  Ademas de setters, podemos usar *this en otros contextos:

  class Nodo {
  private:
      int dato_;
      Nodo* siguiente_;
  public:
      Nodo(int d) : dato_(d), siguiente_(nullptr) {}

      Nodo& setDato(int d) { dato_ = d; return *this; }

      // Metodo que devuelve el objeto mismo
      Nodo& conectarCon(Nodo& otro) {
          siguiente_ = &otro;
          return *this;
      }

      // Getter que devuelve referencia al siguiente
      Nodo& getSiguiente() {
          return *siguiente_;
      }

      void mostrar() const {
          std::cout << dato_;
      }
  };

  Nodo n1(1), n2(2), n3(3);
  n1.conectarCon(n2).conectarCon(n3);

  ------------------------------------------------------------------
  13. THIS Y EL OPERADOR DE INDICE []
  ------------------------------------------------------------------

  Sobrecarga del operador [] usando this:

  class Arreglo {
  private:
      int datos_[10];
  public:
      int& operator[](size_t i) {
          return this->datos_[i];
      }

      const int& operator[](size_t i) const {
          return this->datos_[i];
      }
  };

  ------------------------------------------------------------------
  14. PATRONES DE DISENO CON THIS
  ------------------------------------------------------------------

  Varios patrones de diseno usan this explicitamente:

  - Builder Pattern: devuelve *this para construir objetos paso a paso.
  - State Pattern: los estados cambian usando this en transiciones.
  - Observer Pattern: los sujetos pasan *this a los observadores.
  - Prototype Pattern: clone() devuelve new Tipo(*this).

#ifndef PUNTO_HPP
#define PUNTO_HPP

#include <iostream>
#include <cmath>

class Punto {
public:
    Punto(double x = 0.0, double y = 0.0);

    // Setters con chaining
    Punto& setX(double x);
    Punto& setY(double y);

    // Getters
    double getX() const;
    double getY() const;

    // Operadores
    bool operator==(const Punto& otro) const;
    bool operator!=(const Punto& otro) const;

    // Distancia
    double distancia(const Punto& otro) const;

    // Mostrar
    void mostrar() const;

    // Metodo que devuelve *this
    Punto& mover(double dx, double dy);
    Punto clonar() const;

private:
    double x_;
    double y_;
};

#endif

------------[ archivo: punto.cpp ]------------
#include "punto.hpp"

Punto::Punto(double x, double y) : x_(x), y_(y) {}

Punto& Punto::setX(double x) {
    this->x_ = x;
    return *this;
}

Punto& Punto::setY(double y) {
    this->y_ = y;
    return *this;
}

double Punto::getX() const { return x_; }
double Punto::getY() const { return y_; }

bool Punto::operator==(const Punto& otro) const {
    if (this == &otro) return true;  // mismo objeto
    return x_ == otro.x_ && y_ == otro.y_;
}

bool Punto::operator!=(const Punto& otro) const {
    return !(*this == otro);
}

double Punto::distancia(const Punto& otro) const {
    double dx = x_ - otro.x_;
    double dy = y_ - otro.y_;
    return std::sqrt(dx * dx + dy * dy);
}

void Punto::mostrar() const {
    std::cout << "(" << x_ << ", " << y_ << ")";
}

Punto& Punto::mover(double dx, double dy) {
    x_ += dx;
    y_ += dy;
    return *this;
}

Punto Punto::clonar() const {
    return *this;  // usa constructor de copia
}

#include "punto.hpp"
#include "builder.hpp"

int main() {
    std::cout << "=== METHOD CHAINING CON THIS ===\n";

    Punto p;
    p.setX(3.0).setY(4.0).mover(1.0, 2.0);
    std::cout << "Punto: ";
    p.mostrar();
    std::cout << " | Distancia al origen: " << p.distancia(Punto()) << "\n";

    std::cout << "\n=== AUTOCOMPARACION ===\n";
    Punto p1(5, 5), p2(5, 5), p3(10, 10);
    std::cout << "p1 == p2: " << (p1 == p2) << std::endl;
    std::cout << "p1 == p3: " << (p1 == p3) << std::endl;

    Punto& refP1 = p1;
    std::cout << "p1 == refP1 (mismo objeto): " << (p1 == refP1) << std::endl;

    std::cout << "\n=== CLONAR CON *THIS ===\n";
    Punto clone = p1.clonar();
    std::cout << "Clone: ";
    clone.mostrar();
    std::cout << std::endl;

==========================================================================
  RESUMEN DEL CAPÍTULO
==========================================================================

  this es un puntero al objeto actual dentro de funciones miembro.
  En metodos no-const: this es Tipo* const.
  En metodos const: this es const Tipo* const.
  this-> permite acceder a miembros cuando hay conflictos de nombres.
  Devolviendo *this se implementa method chaining (encadenamiento).
  this != &otro verifica auto-asignacion en operator=.
  this permite a un objeto clonarse (return *this;).
  En funciones static no existe this.
  En templates, this-> es necesario para miembros dependientes.

==========================================================================
  PREGUNTAS FRECUENTES (FAQ)
==========================================================================

  P: Necesito escribir this-> siempre?
  R: No. El compilador lo resuelve implicitamente. Solo es necesario
     cuando hay conflictos (parametro con mismo nombre que miembro) o
     en templates para miembros dependientes.

  P: Que devuelve *this exactamente?
  R: Una referencia (T&) al objeto actual. Es decir, el objeto mismo,
     no una copia.

  P: Puedo cambiar la direccion de this?
  R: No. this es un prvalue (puro rvalue) y no se puede reasignar.
     Siempre apunta al objeto actual.

  P: Cual es la alternativa a this cuando hay conflicto de nombres?
  R: Usar nombres diferentes para parametros (convencion: parametros
     sin guion, miembros con _). O usar lista de inicializacion.

  P: Por que los operadores de flujo (<<, >>) no pueden ser miembros?
  R: Porque el primer operando es std::ostream, no nuestra clase.
     Si son miembros, el objeto seria el primer operando (izquierdo),
     lo que no es natural. Se implementan como funciones friend.

  P: Que pasa si capturo this en una lambda y el objeto se destruye?
  R: La lambda tiene una copia del puntero, pero el objeto puede haber
     sido destruido. Usar [*this] (C++17) captura una copia segura.

  --- FRAMEWORK DE CONSULTAS A BASE DE DATOS CON CHAINING ---

  Disena un framework de consultas SQL usando method chaining con this.

  Clases:
  1. QueryBuilder: la clase principal.
     - Metodos que devuelven *this:
       select(const std::vector<std::string>& campos)
       from(const std::string& tabla)
       where(const std::string& condicion)
       andWhere(const std::string& condicion)
       orWhere(const std::string& condicion)
       orderBy(const std::string& campo, bool asc = true)
       groupBy(const std::string& campo)
       having(const std::string& condicion)
       limit(int n)
       offset(int n)
       join(const std::string& tabla, const std::string& on)
       leftJoin(...), rightJoin(...)
     - Metodo final: ejecutar() o construir() que genera el SQL string.

  2. ResultSet: almacena resultados simulados.

  3. Database: ejecuta consultas y devuelve ResultSet.

  Requisitos:
  - Todos los metodos intermedios devuelven QueryBuilder&.
  - El metodo construir() devuelve std::string con la consulta.
  - El metodo ejecutar() devuelve ResultSet.
  - Cada metodo debe validar sus parametros (no vacio, etc.).
  - Usar this para auto-comprobacion si es necesario.

  Ejemplo de uso:
  Database db;
  auto resultado = db.query()
      .select({"nombre", "edad"})
      .from("usuarios")
      .where("edad > 18")
      .andWhere("activo = 1")
      .orderBy("nombre")
      .limit(10)
      .ejecutar();
