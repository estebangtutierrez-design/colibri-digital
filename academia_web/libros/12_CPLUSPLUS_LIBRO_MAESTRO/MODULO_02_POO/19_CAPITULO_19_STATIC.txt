  Capitulo 19: STATIC
================================================================================

  "static no es una simple palabra: es el puente entre la instancia y
   la esencia compartida de la clase."

================================================================================
  BIENVENIDA
================================================================================

  Bienvenido al Capitulo 19 de C++ Master Book. Hasta ahora, todos los
  miembros de una clase pertenecian a cada objeto individual: cada objeto
  tenia su propia copia de los datos miembro.

  Pero a veces necesitamos datos o funciones que pertenezcan a la clase
  en si misma, no a objetos particulares. Ahi entran los miembros static.

  Un miembro static es compartido por todas las instancias de la clase.
  Existe aunque no haya ningun objeto creado. Es como una variable global
  pero con el ambito y acceso de la clase.

  En este capitulo exploraremos: datos miembro static, funciones miembro
  static, variables static locales a funciones, constantes static, y el
  patron de diseno Singleton.

================================================================================
  FRASE DEL CAPÍTULO
================================================================================

  "Lo static pertenece a la clase, no al objeto. Es la memoria colectiva
   de todas las instancias."

  ------------------------------------------------------------------
  1. DATOS MIEMBRO STATIC
  ------------------------------------------------------------------

  Un dato miembro static es compartido por TODOS los objetos de la clase.
  Solo hay una copia, exista o no exista ningun objeto.

  class Contador {
  public:
      static int totalInstancias;  // declaracion
  };

  // Definicion e inicializacion (en un .cpp, fuera de la clase):
  int Contador::totalInstancias = 0;

  // Acceso:
  Contador::totalInstancias = 5;  // sin objeto
  Contador obj;
  obj.totalInstancias = 10;       // con objeto (tambien funciona)

  ------------------------------------------------------------------
  2. DECLARACION VS DEFINICION
  ------------------------------------------------------------------

  Los datos static se DECLARAN dentro de la clase y se DEFINEN fuera.
  La definicion debe ir en un archivo .cpp (no en el .hpp) para evitar
  violaciones de la regla de una definicion (ODR).

  // contador.hpp
  class Contador {
  public:
      static int total;
      static const int MAX = 100;  // excepcion: static const puede
                                   // inicializarse in-class
  };

  // contador.cpp
  int Contador::total = 0;

  Desde C++17, inline static permite definir e inicializar en la clase:

  class Ejemplo {
  public:
      inline static int contador = 0;  // C++17: definicion in-class
  };

  ------------------------------------------------------------------
  3. FUNCIONES MIEMBRO STATIC
  ------------------------------------------------------------------

  Las funciones static:
  - No reciben el puntero this (no se invocan sobre un objeto).
  - Solo pueden acceder a miembros static (no a los de instancia).
  - Se pueden llamar sin objeto: Clase::funcion().
  - No pueden ser const, virtual, ni volatile.

  class MathUtils {
  public:
      static int sumar(int a, int b) { return a + b; }
      static int factorial(int n);
      static constexpr double PI = 3.1415926535;
  private:
      static int contadorLlamadas_;
  };

  int MathUtils::contadorLlamadas_ = 0;

  int MathUtils::factorial(int n) {
      ++contadorLlamadas_;  // OK: static
      return (n <= 1) ? 1 : n * factorial(n - 1);
  }

  // Uso:
  int r = MathUtils::sumar(5, 3);
  int f = MathUtils::factorial(6);

  ------------------------------------------------------------------
  4. CONTAR INSTANCIAS
  ------------------------------------------------------------------

  Uno de los usos mas comunes de static: llevar la cuenta de cuantos
  objetos de una clase se han creado.

  class Usuario {
  private:
      inline static int contador_ = 0;  // C++17
      int id_;
      std::string nombre_;
  public:
      Usuario(const std::string& nombre) : nombre_(nombre) {
          ++contador_;
          id_ = contador_;  // ID unico autoincremental
      }

      ~Usuario() {
          --contador_;
      }

      int getId() const { return id_; }

      static int getTotalUsuarios() {
          return contador_;
      }
  };

  // Uso:
  Usuario u1("Ana"), u2("Luis");
  std::cout << Usuario::getTotalUsuarios();  // 2
  {
      Usuario u3("Pedro");
      std::cout << Usuario::getTotalUsuarios();  // 3
  }
  std::cout << Usuario::getTotalUsuarios();  // 2

  ------------------------------------------------------------------
  5. VARIABLES STATIC LOCALES A FUNCIONES
  ------------------------------------------------------------------

  Dentro de una funcion, static mantiene su valor entre llamadas:

  void contadorLlamadas() {
      static int llamadas = 0;  // se inicializa UNA sola vez
      ++llamadas;
      std::cout << "Llamada #" << llamadas << std::endl;
  }

  // Uso:
  contadorLlamadas();  // Llamada #1
  contadorLlamadas();  // Llamada #2
  contadorLlamadas();  // Llamada #3

  La variable static local se inicializa la primera vez que se ejecuta
  la funcion y persiste hasta el final del programa.

  ------------------------------------------------------------------
  6. STATIC EN EL PATRON SINGLETON
  ------------------------------------------------------------------

  Singleton: garantiza que una clase tenga una unica instancia.

  class Singleton {
  private:
      Singleton() = default;
      ~Singleton() = default;

      Singleton(const Singleton&) = delete;
      Singleton& operator=(const Singleton&) = delete;

  public:
      static Singleton& getInstance() {
          static Singleton instancia;  // inicializacion perezosa
          return instancia;            // segura para hilos (C++11)
      }

      void hacerAlgo() {
          std::cout << "Singleton haciendo algo\n";
      }
  };

  // Uso:
  Singleton::getInstance().hacerAlgo();

  La variable static local en getInstance() se inicializa solo una vez,
  incluso en entornos multi-hilo (garantizado por C++11).

  ------------------------------------------------------------------
  7. CONSTEXPR STATIC
  ------------------------------------------------------------------

  Los miembros static constexpr pueden definirse dentro de la clase:

  class Config {
  public:
      static constexpr int VERSION_MAYOR = 2;
      static constexpr int VERSION_MENOR = 1;
      static constexpr const char* NOMBRE = "SistemaX";
      static constexpr double PI = 3.141592653589793;

      static std::string getVersion() {
          return std::to_string(VERSION_MAYOR) + "."
               + std::to_string(VERSION_MENOR);
      }
  };

  std::cout << Config::getVersion();  // "2.1"
  std::cout << Config::PI;

  ------------------------------------------------------------------
  8. STATIC EN CLASES TEMPLATE
  ------------------------------------------------------------------

  Cada especializacion de un template tiene su propia copia de los
  miembros static:

  template <typename T>
  class Almacen {
  public:
      inline static int contador = 0;
  };

  Almacen<int> a1, a2;
  Almacen<double> b1;

  ++Almacen<int>::contador;     // contador de int = 1
  ++Almacen<double>::contador;  // contador de double = 1

  // a1.contador == 1, a2.contador == 1 (comparten)
  // b1.contador == 1 (independiente)

  ------------------------------------------------------------------
  9. FUNCIONES STATIC VS FUNCIONES GLOBALES
  ------------------------------------------------------------------

  Comparacion: funcion static vs funcion global vs funcion en
  namespace:

  // Funcion static miembro:
  class Util {
  public:
      static int doble(int x) { return x * 2; }
  };
  // Llamada: Util::doble(5);

  // Funcion global:
  int doble(int x) { return x * 2; }
  // Llamada: doble(5);

  // Funcion en namespace:
  namespace util {
      int doble(int x) { return x * 2; }
  }
  // Llamada: util::doble(5);

  Las funciones static miembro tienen acceso a miembros privados
  static de la clase.

  ------------------------------------------------------------------
  10. LIMITACIONES DE STATIC
  ------------------------------------------------------------------

  this no existe en funciones miembro static.
  No pueden ser virtuales ni override.
  No pueden ser const (no hay this a quien aplicar const).
  No pueden acceder a miembros no-static directamente.
  El orden de inicializacion de static entre diferentes archivos
  no esta definido (static initialization order fiasco).

  Para evitar el fiasco de inicializacion, usar:
  - Inicializacion perezosa con static local (como Singleton).
  - Constexpr cuando sea posible.

  ------------------------------------------------------------------
  11. STATIC EN ESTRUCTURAS
  ------------------------------------------------------------------

  Las structs tambien pueden tener miembros static:

  struct Punto {
      double x, y;

      static Punto origen() {
          return Punto{0.0, 0.0};
      }

      static constexpr Punto* NULO = nullptr;
  };

  ------------------------------------------------------------------
  12. STATIC Y THREADS (C++11)
  ------------------------------------------------------------------

  Desde C++11, las variables static locales se inicializan de forma
  segura para hilos. El compilador garantiza que solo un hilo ejecute
  la inicializacion.

  class Logger {
  public:
      static Logger& getInstance() {
          static Logger instancia;  // segura para hilos
          return instancia;
      }

      void log(const std::string& msg) {
          std::cout << "[LOG] " << msg << std::endl;
      }
  };

  // Varios hilos pueden llamar a getInstance() simultaneamente
  // sin riesgo de doble inicializacion.

  ------------------------------------------------------------------
  13. ALMACENAMIENTO DE MIEMBROS STATIC
  ------------------------------------------------------------------

  Los miembros static NO forman parte del sizeof de los objetos.
  Existen en un segmento de datos separado (como las variables globales).

  class Demo {
  private:
      int a_;       // 4 bytes
      double b_;    // 8 bytes
      static int c_;  // NO cuenta en sizeof
      static constexpr int d_ = 10;  // tampoco cuenta
  };

  // sizeof(Demo) = 12 (mas padding)
  // La memoria para c_ y d_ esta aparte

  ------------------------------------------------------------------
  14. EJEMPLO: ID AUTOINCREMENTAL
  ------------------------------------------------------------------

  class Producto {
  private:
      inline static int nextId_ = 1;
      int id_;
      std::string nombre_;
  public:
      Producto(const std::string& nombre) : nombre_(nombre) {
          id_ = nextId_++;
      }

      int getId() const { return id_; }

      static int getSiguienteId() {
          return nextId_;
      }

      static void resetearContador(int inicio = 1) {
          nextId_ = inicio;
      }
  };

    static double getTotalSalarios();
    static double getSalarioPromedio();

    inline static int contador_ = 0;
    inline static double totalSalarios_ = 0.0;

    int id_;
    std::string nombre_;
    double salario_;

------------[ archivo: empresa.cpp ]------------
#include "empresa.hpp"

Empleado::Empleado(const std::string& nombre, double salario)
    : nombre_(nombre), salario_(salario) {
    id_ = ++contador_;
    totalSalarios_ += salario;
}

Empleado::~Empleado() {
    totalSalarios_ -= salario_;
    // Nota: no decrementamos contador_ porque el ID es unico
    // y permanente. Pero si queremos contar activos, si.
}

int Empleado::getId() const { return id_; }
std::string Empleado::getNombre() const { return nombre_; }
double Empleado::getSalario() const { return salario_; }

int Empleado::getTotalEmpleados() { return contador_; }
double Empleado::getTotalSalarios() { return totalSalarios_; }

double Empleado::getSalarioPromedio() {
    if (contador_ == 0) return 0.0;
    return totalSalarios_ / contador_;
}

void Empleado::resetearContador() { contador_ = 0; }

void Empleado::mostrar() const {
    std::cout << "Empleado #" << id_
              << ": " << nombre_
              << " - $" << salario_ << std::endl;
}

#include <iostream>
#include "empresa.hpp"
#include "config.hpp"

int main() {
    std::cout << "=== MIEMBROS STATIC ===\n";

    std::cout << "\n-- Contador de instancias --\n";
    std::cout << "Total empleados (antes): "
              << Empleado::getTotalEmpleados() << std::endl;

    Empleado e1("Ana Lopez", 3500.0);
    Empleado e2("Carlos Ruiz", 4200.0);
    Empleado e3("Maria Garcia", 3800.0);

    e1.mostrar();
    e2.mostrar();
    e3.mostrar();

    std::cout << "\n-- Datos static de la clase --\n";
    std::cout << "Total empleados: "
              << Empleado::getTotalEmpleados() << std::endl;
    std::cout << "Total salarios: $"
              << Empleado::getTotalSalarios() << std::endl;
    std::cout << "Salario promedio: $"
              << Empleado::getSalarioPromedio() << std::endl;

    std::cout << "\nAcceso a ID via objeto: " << e1.getId() << std::endl;
    std::cout << "Acceso a total via objeto: "
              << e1.getTotalEmpleados() << std::endl;

    std::cout << "\n-- Singleton --\n";
    Configuracion& cfg = Configuracion::getInstance();
    cfg.mostrar();

    cfg.setNombreApp("SuperApp");
    cfg.setVersion("2.5.1");
    cfg.setModoDebug(true);

    Configuracion::getInstance().mostrar();  // misma instancia

    std::cout << "\nMax usuarios constantes: "
              << Configuracion::MAX_USUARIOS << std::endl;

    std::cout << "\n-- Variable static local --\n";
    for (int i = 0; i < 3; ++i) {
        static int contador = 0;
        ++contador;
        std::cout << "Iteracion " << i + 1
                  << ", contador static: " << contador << std::endl;
    }

    std::cout << "\n-- Tamano de objetos --\n";
    std::cout << "sizeof(Empleado): " << sizeof(Empleado)
              << " bytes (miembros static NO cuentan)\n";

    return 0;
}

==========================================================================
  RESUMEN DEL CAPÍTULO
==========================================================================

  Miembros static: pertenecen a la clase, no a las instancias.
  Datos static: se declaran en la clase, se definen en el .cpp.
  C++17: inline static permite definicion in-class.
  Funciones static: no tienen this, solo acceden a miembros static.
  Variables static locales: mantienen valor entre llamadas,
  inicializacion perezosa y segura para hilos (C++11).
  Singleton: patron que usa static para garantizar una unica instancia.
  Constexpr static: constantes de clase definidas in-class.
  sizeof: los miembros static no contribuyen al tamano del objeto.

==========================================================================
  PREGUNTAS FRECUENTES (FAQ)
==========================================================================

  P: Puedo acceder a un miembro static desde un metodo no-static?
  R: Si, perfectamente. Los metodos no-static ven todos los miembros,
     tanto static como de instancia.

  P: Y al reves? Puede un metodo static acceder a miembros no-static?
  R: No. Un metodo static no tiene this, por lo que no puede acceder
     a miembros de instancia. Solo a miembros static.

  P: Donde se almacenan los miembros static en memoria?
  R: En el segmento de datos (como las variables globales), no en el
     heap ni en el stack de los objetos.

  P: Que es el "static initialization order fiasco"?
  R: Cuando dos archivos .cpp tienen variables static que dependen una
     de la otra, el orden de inicializacion no esta definido. Solucion:
     usar static local (inicializacion perezosa) o constexpr.

  P: Son los miembros static seguros para hilos?
  R: Su inicializacion si (C++11 garantiza inicializacion segura para
     static locales). Pero su modificacion requiere sincronizacion
     explicita (mutex, atomic, etc.).

  P: static afecta el polimorfismo?
  R: Las funciones static no pueden ser virtuales, por lo que no
     participan en el polimorfismo dinamico.

  --- SISTEMA DE MONITOREO CON ESTADISTICAS GLOBALES ---

  Disena un sistema de monitoreo que use miembros static para
  recolectar estadisticas globales de la aplicacion.

  Clases:
  1. Transaccion: cada operacion realizada.
     - Atributos: id (auto-increment), tipo, monto, timestamp.
     - Constructor incrementa contador static global.
     - Destructor no decrementa (historial permanente).

  2. EstadisticasGlobales:
     - Miembros static: totalTransacciones, totalMonto,
       transaccionesPorTipo (map<string, int>),
       montoMaximo, montoMinimo.
     - Metodos static: getTotal(), getPromedio(), getMax(),
       getMin(), getReporte().
     - Los constructores de Transaccion actualizan estas
       estadisticas automaticamente.

  3. Monitor:
     - Singleton que periodicamente muestra estadisticas.
     - Metodo iniciar(), detener(), mostrarReporte().
     - Metodo static getTiempoActivo().

  4. Simulador:
     - Genera transacciones aleatorias.
     - Muestra estadisticas en tiempo real.

  Requisitos:
  - Todos los contadores y acumuladores static.
  - Metodos static para acceder a estadisticas.
  - Singleton para Monitor y Config.
  - Constexpr para constantes de configuracion.
  - Variables static locales para tiempos.

  EXTRA: Hacerlo thread-safe con std::atomic y std::mutex.
