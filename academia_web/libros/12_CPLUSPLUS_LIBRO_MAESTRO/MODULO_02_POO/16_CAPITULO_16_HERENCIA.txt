  Capitulo 16: HERENCIA

  "La herencia no es solo reutilizar codigo: es modelar la jerarquia
   natural del dominio, donde lo especifico extiende lo general."

================================================================================
  BIENVENIDA
================================================================================

  Bienvenido al Capitulo 16 de C++ Master Book. Hemos cubierto clases,
  objetos, constructores, destructores, encapsulacion y getters/setters.
  Ahora abordamos uno de los pilares mas potentes de la POO: la herencia.

  La herencia permite crear una clase nueva (derivada) a partir de una
  clase existente (base), heredando sus miembros y comportamientos.
  La clase derivada puede agregar nuevos miembros, modificar el
  comportamiento de los existentes (override) y extender la funcionalidad.

  En C++, la herencia puede ser public, protected o private, y soporta
  herencia simple (una base) y multiple (varias bases, capitulo 17).
  Aprenderemos la sintaxis, el orden de construccion/destruccion, el
  ocultamiento de nombres, y como disenar jerarquias de clases correctas.

================================================================================
  FRASE DEL CAPÍTULO
================================================================================

  "La clase base es el tronco; la derivada, las ramas. Juntas forman
   el arbol de la jerarquia."

  ------------------------------------------------------------------
  1. SINTAXIS DE HERENCIA
  ------------------------------------------------------------------

  class Derivada : [especificador] Base {
      // nuevos miembros
  };

  El especificador puede ser: public (mas comun), protected, o private.

  class Animal {
  public:
      void comer() { std::cout << "Comiendo\n"; }
  };

  class Perro : public Animal {
  public:
      void ladrar() { std::cout << "Guau!\n"; }
  };

  Perro p;
  p.comer();   // heredado de Animal
  p.ladrar();  // propio de Perro

  ------------------------------------------------------------------
  2. RELACION "ES-UN" (IS-A)
  ------------------------------------------------------------------

  La herencia public modela una relacion "es-un":
  - Un Perro es un Animal.
  - Un Circulo es una Figura.
  - Un Empleado es una Persona.

  Si la relacion no es "es-un", probablemente deberias usar composicion
  ("tiene-un") en lugar de herencia.

  ------------------------------------------------------------------
  3. TIPOS DE HERENCIA
  ------------------------------------------------------------------

  Herencia public:
  - Los miembros public de la base siguen siendo public en la derivada.
  - Los miembros protected de la base siguen siendo protected.
  - Modela "es-un". Es la mas usada.

  Herencia protected:
  - Los miembros public y protected de la base se vuelven protected.
  - Oculta la relacion de herencia al mundo exterior.

  Herencia private:
  - Todos los miembros de la base se vuelven private.
  - La herencia es un detalle de implementacion, no de interfaz.

  class A {
  public:    int x;
  protected: int y;
  private:   int z;
  };

  class B : public A     { /* x:public, y:protected, z:inaccesible */ };
  class C : protected A  { /* x:protected, y:protected, z:inaccesible */ };
  class D : private A    { /* x:private, y:private, z:inaccesible */ };

  ------------------------------------------------------------------
  4. CONSTRUCTORES EN HERENCIA
  ------------------------------------------------------------------

  Orden de construccion:
  1. Constructor de la clase base.
  2. Constructores de los miembros (en orden de declaracion).
  3. Cuerpo del constructor de la derivada.

  La clase derivada debe llamar al constructor de la base en su lista
  de inicializacion si la base no tiene constructor por defecto.

  class Base {
  public:
      Base(int valor) : valor_(valor) {
          std::cout << "Base(" << valor << ")\n";
      }
  protected:
      int valor_;
  };

  class Derivada : public Base {
  public:
      Derivada(int valor, const std::string& nombre)
          : Base(valor), nombre_(nombre) {  // llamada a base
          std::cout << "Derivada(" << valor << ", " << nombre << ")\n";
      }
  private:
      std::string nombre_;
  };

  Si la base tiene constructor por defecto, se llama implicitamente.
  Si no, es obligatorio llamarlo explicitamente.

  ------------------------------------------------------------------
  5. DESTRUCTORES EN HERENCIA
  ------------------------------------------------------------------

  Orden de destruccion (inverso a la construccion):
  1. Cuerpo del destructor de la derivada.
  2. Destructores de los miembros (orden inverso).
  3. Destructor de la clase base.

  IMPORTANTE: si piensas usar polimorfismo, el destructor de la base
  debe ser virtual. De lo contrario, al eliminar un objeto derivado
  a traves de un puntero base, solo se llamara al destructor de la base.

  class Base {
  public:
      virtual ~Base() = default;
  };

  class Derivada : public Base {
  private:
      int* datos_;
  public:
      Derivada() : datos_(new int[100]) {}
      ~Derivada() override { delete[] datos_; }
  };

  ------------------------------------------------------------------
  6. REDEFINICION DE FUNCIONES MIEMBRO (OVERRIDE)
  ------------------------------------------------------------------

  La clase derivada puede redefinir funciones de la base:

  class Empleado {
  public:
      double calcularPago() const {
          return salarioBase_;
      }
  protected:
      double salarioBase_ = 2000.0;
  };

  class Gerente : public Empleado {
  public:
      double calcularPago() const {   // redefinicion
          return salarioBase_ + bono_;
      }
  private:
      double bono_ = 1000.0;
  };

  Para llamar explicitamente a la version de la base:
  double Empleado::calcularPago() const {
      return salarioBase_;
  }

  double Gerente::calcularPago() const {
      return Empleado::calcularPago() + bono_;  // llama a la base
  }

  Con virtual y override (lo veremos en Modulo III) se logra
  polimorfismo en tiempo de ejecucion.

  ------------------------------------------------------------------
  7. OCULTAMIENTO DE NOMBRES (NAME HIDING)
  ------------------------------------------------------------------

  Cuando la clase derivada declara una funcion con el mismo nombre
  que una funcion de la base, oculta TODAS las funciones de la base
  con ese nombre (no solo las que coinciden en firma).

  class Base {
  public:
      void f(int) { std::cout << "Base::f(int)\n"; }
      void f(double) { std::cout << "Base::f(double)\n"; }
      void g() { std::cout << "Base::g()\n"; }
  };

  class Derivada : public Base {
  public:
      void f(int) { std::cout << "Derivada::f(int)\n"; }  // oculta AMBAS f de Base
      void h() { std::cout << "Derivada::h()\n"; }
  };

  Derivada d;
  d.f(5);     // Derivada::f(int)
  // d.f(3.14); // ERROR: f(double) ocultada
  d.Base::f(3.14);  // OK: llamada explicita

  Para evitar el ocultamiento, podemos usar using:

  class Derivada : public Base {
  public:
      using Base::f;   // trae TODAS las f de Base al ambito
      void f(int) { /* ... */ }
  };

  ------------------------------------------------------------------
  8. HERENCIA Y ACCESO A MIEMBROS
  ------------------------------------------------------------------

  class Base {
  private:
      int privado_;
  protected:
      int protegido_;
  public:
      int publico_;
  };

  class Derivada : public Base {
  public:
      void ejemplo() {
          // privado_ = 1;  // ERROR: no accesible
          protegido_ = 2;    // OK: accesible desde derivada
          publico_ = 3;      // OK: publico
      }
  };

  // Desde fuera:
  Derivada d;
  d.publico_ = 10;     // OK
  // d.protegido_ = 20; // ERROR: protegido no accesible fuera

  ------------------------------------------------------------------
  9. CONSTRUCTORES DE COPIA Y MOVIMIENTO EN HERENCIA
  ------------------------------------------------------------------

  class Base {
  public:
      Base(const Base& otro) : valor_(otro.valor_) {}
  private:
      int valor_;
  };

  class Derivada : public Base {
  public:
      Derivada(const Derivada& otro)
          : Base(otro), nombre_(otro.nombre_) {}  // llama a copia de Base
  private:
      std::string nombre_;
  };

  El constructor de copia de la derivada debe llamar al de la base.
  Si no se especifica, se llama al constructor por defecto de la base.

  ------------------------------------------------------------------
  10. HERENCIA VS COMPOSICION
  ------------------------------------------------------------------

  Preferir composicion sobre herencia cuando la relacion es
  "tiene-un" en lugar de "es-un".

  Herencia (es-un):
  class Coche : public Vehiculo { ... };

  Composicion (tiene-un):
  class Coche {
  private:
      Motor motor_;
      Rueda ruedas_[4];
  };

  La composicion es mas flexible, menos acoplada y mas facil de
  mantener. Usa herencia solo cuando haya una verdadera relacion
  jerarquica y polimorfismo.

  ------------------------------------------------------------------
  11. CLASES BASE ABSTRACTAS
  ------------------------------------------------------------------

  (Avance del modulo III) Una clase con al menos una funcion virtual
  pura es abstracta y no se puede instanciar.

  class Figura {
  public:
      virtual double area() const = 0;  // pura virtual
      virtual ~Figura() = default;
  };

  class Circulo : public Figura {
  private:
      double radio_;
  public:
      Circulo(double r) : radio_(r) {}
      double area() const override { return 3.14159 * radio_ * radio_; }
  };

  // Figura f;      // ERROR: clase abstracta
  Circulo c(5.0);    // OK

  ------------------------------------------------------------------
  12. HERENCIA Y FUNCIONES AMIGAS
  ------------------------------------------------------------------

  La amistad no se hereda. Si la clase Base declara a Func como amiga,
  Func NO tiene acceso automatico a los miembros privados de Derivada.

  class Base {
      friend class Amiga;
  private:
      int secreto_;
  };

  class Derivada : public Base {
  private:
      int otroSecreto_;
  };

  class Amiga {
      void f(Base& b) { b.secreto_ = 1; }       // OK
      void g(Derivada& d) { d.secreto_ = 2; }   // OK? Solo si es parte de Base
      // d.otroSecreto_ = 3;                     // ERROR: no es amiga de Derivada
  };

  ------------------------------------------------------------------
  13. UPCASTING Y DOWNCASTING
  ------------------------------------------------------------------

  Upcasting: convertir puntero/referencia de derivada a base (seguro).

  Downcasting: convertir de base a derivada (requiere verificacion).

  class Base { };
  class Derivada : public Base { };

  Derivada d;
  Base* ptrBase = &d;        // upcasting: implicito, seguro
  Base& refBase = d;         // upcasting

  Derivada* ptrDer = dynamic_cast<Derivada*>(ptrBase);  // downcasting

  dynamic_cast requiere RTTI y destructor virtual en la base.

  ------------------------------------------------------------------
  14. HERENCIA Y ATRIBUTOS STATIC
  ------------------------------------------------------------------

  Los atributos static se heredan pero son compartidos entre todas
  las clases de la jerarquia (hay una sola instancia).

  class Base {
  public:
      static int contador;
      Base() { ++contador; }
  };
  int Base::contador = 0;

  class Derivada : public Base { };

  Base b1;
  Derivada d1, d2;
  std::cout << Base::contador;    // 3
  std::cout << Derivada::contador; // 3 (misma variable)

------------[ archivo: main.cpp ]------------
#include <iostream>
#include <vector>
#include "coche.hpp"
#include "moto.hpp"

int main() {
    std::cout << "=== CONSTRUCCION ===\n";
    Coche coche("Toyota", 2022, 4);
    Moto moto("Honda", 2023, false);

    std::cout << "\n=== INFORMACION ===\n";
    coche.mostrarInfo();
    std::cout << std::endl;
    moto.mostrarInfo();
    std::cout << std::endl;

    std::cout << "\n=== COMPORTAMIENTO POLIMORFICO ===\n";
    std::vector<Vehiculo*> vehiculos;
    vehiculos.push_back(&coche);
    vehiculos.push_back(&moto);

    for (const auto& v : vehiculos) {
        v->arrancar();
    }

    std::cout << "\n=== HERENCIA Y ACCESO ===\n";
    std::cout << "Marca del coche: " << coche.getMarca() << std::endl;
    std::cout << "Puertas del coche: " << coche.getNumPuertas() << std::endl;
    coche.abrirPuertas();

    std::cout << "\n=== UPCASTING ===\n";
    Vehiculo* ptr = &coche;
    ptr->arrancar();  // llama a Coche::arrancar() por virtual
    ptr->mostrarInfo();
    std::cout << std::endl;

    std::cout << "\n=== DESTRUCCION ===\n";
    return 0;
}

  RESUMEN DEL CAPÍTULO
==========================================================================

  Herencia: permite crear clases derivadas a partir de bases.
  Relacion "es-un": la herencia public modela "es-un".
  Orden de construccion: base -> miembros -> derivada.
  Orden de destruccion: inverso.
  Redefinicion: la derivada puede redefinir funciones de la base.
  Ocultamiento: una funcion en la derivada oculta TODAS las del mismo
  nombre en la base (usar using para evitarlo).
  Destructor virtual: necesario para eliminacion polimorfica correcta.
  Clase abstracta: al menos una funcion virtual pura (= 0).
  La amistad no se hereda.
  Preferir composicion sobre herencia cuando la relacion sea "tiene-un".

==========================================================================
  PREGUNTAS FRECUENTES (FAQ)
==========================================================================

  P: Puedo heredar de una clase sin constructor por defecto?
  R: Si, pero debes llamar explicitamente al constructor de la base
     en la lista de inicializacion de la derivada.

  P: Que pasa si no llamo al constructor de la base?
  R: El compilador intenta llamar al constructor por defecto de la
     base. Si no existe, error de compilacion.

  P: La herencia private se usa alguna vez?
  R: Si, para implementar "se implementa en terminos de" (implemented-
     in-terms-of). Es un detalle interno. Menos comun.

  P: Cual es la diferencia entre redefinicion y override?
  R: Redefinicion: la derivada define una funcion con el mismo nombre
     que la base (con o sin virtual). Override: la derivada redefine
     una funcion virtual de la base con la misma firma (C++11: palabra
     override explicita).

  P: Puedo heredar de una clase amiga?
  R: Si, pero la amistad no se hereda. La derivada no es automaticamente
     amiga de lo que la base era amiga.

  P: Que es LSP (Liskov Substitution Principle)?
  R: Si S es subtipo de T, entonces los objetos de T pueden ser
     reemplazados por objetos de S sin alterar las propiedades del
     programa. La herencia debe respetar este principio.
