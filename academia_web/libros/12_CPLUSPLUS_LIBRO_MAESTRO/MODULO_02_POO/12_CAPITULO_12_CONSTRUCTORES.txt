  Capítulo 12: CONSTRUCTORES
================================================================================

  "El constructor no solo crea el objeto: le da identidad, lo inicializa
   y lo prepara para cumplir su propósito."

================================================================================
  BIENVENIDA
================================================================================

  Bienvenido al Capítulo 12 de C++ Master Book. En el capítulo anterior
  aprendimos a definir clases y crear objetos. Sin embargo, había una
  pregunta pendiente: ¿cómo inicializamos los objetos de manera controlada?

  La respuesta son los constructores: funciones miembro especiales que se
  invocan automáticamente cuando se crea un objeto. Su propósito es
  inicializar los datos miembro, asignar recursos y dejar el objeto en
  un estado válido.

  C++ ofrece varios tipos de constructores: por defecto, con parámetros,
  de copia, de movimiento (C++11), delegados (C++11), y explícitos.
  Dominarlos es esencial para escribir código robusto y eficiente.

  En este capítulo exploraremos cada tipo de constructor en detalle,
  incluyendo la lista de inicialización (initialization list), el
  constructor por defecto generado por el compilador, el constructor de
  copia y la palabra clave explicit.

  ------------------------------------------------------------------
  1. ¿QUÉ ES UN CONSTRUCTOR?
  ------------------------------------------------------------------

  Un constructor es una función miembro especial que:

    • Tiene el mismo nombre que la clase.
    • No tiene tipo de retorno (ni siquiera void).
    • Se invoca automáticamente al crear un objeto.
    • Puede estar sobrecargado (varios constructores con diferentes
      parámetros).
    • No puede ser virtual (hasta C++11, pero puede ser delegado).

  Sintaxis:

  class NombreClase {
  public:
      NombreClase();               // constructor por defecto
      NombreClase(parametros);     // constructor con parámetros
      NombreClase(const NombreClase& otro);  // constructor de copia
  };

  ------------------------------------------------------------------
  2. CONSTRUCTOR POR DEFECTO (DEFAULT CONSTRUCTOR)
  ------------------------------------------------------------------

  Es el constructor que no requiere argumentos. Si no declaramos ningún
  constructor, el compilador genera uno implícitamente (que no hace nada).

  class Punto {
  public:
      Punto();  // declaración
  private:
      double x_, y_;
  };

  // Implementación:
  Punto::Punto() : x_(0.0), y_(0.0) {}

  Si declaramos al menos un constructor, el compilador NO genera el
  constructor por defecto implícito. Podemos forzarlo con = default:

  class Ejemplo {
  public:
      Ejemplo() = default;  // genera el constructor por defecto
      Ejemplo(int v);
  };

  También podemos eliminarlo con = delete:

  class NoDefault {
  public:
      NoDefault() = delete;   // no se puede crear sin parámetros
      NoDefault(int v);
  };

  ------------------------------------------------------------------
  3. CONSTRUCTOR CON PARÁMETROS
  ------------------------------------------------------------------

  Permite inicializar el objeto con valores específicos:

  class Rectangulo {
  public:
      Rectangulo(double ancho, double alto);
  private:
      double ancho_;
      double alto_;
  };

  Rectangulo::Rectangulo(double ancho, double alto)
      : ancho_(ancho), alto_(alto) {}

  // Uso:
  Rectangulo r1(10.0, 5.0);       // directo
  Rectangulo r2 = Rectangulo(3.0, 4.0);  // copia directa (C++11)
  auto r3 = Rectangulo{7.0, 2.0};        // uniform initialization

  ------------------------------------------------------------------
  4. LISTA DE INICIALIZACIÓN (INITIALIZATION LIST)
  ------------------------------------------------------------------

  La lista de inicialización es la forma correcta y eficiente de
  inicializar los datos miembro de un constructor. Va después de los
  parámetros, separada por dos puntos (:).

  class Persona {
  private:
      std::string nombre_;
      int edad_;
      const int id_;        // constante → debe inicializarse en lista
      int& referencia_;     // referencia → debe inicializarse en lista
  public:
      Persona(const std::string& nombre, int edad, int id, int& ref)
          : nombre_(nombre), edad_(edad), id_(id), referencia_(ref) {}
  };

  Ventajas de la lista de inicialización:
  • Inicializa directamente (no asigna después de construir por defecto).
  • Obligatoria para miembros const y referencias.
  • Más eficiente: evita llamadas dobles a constructores.
  • Única forma de inicializar clases base (herencia, capítulo 16).

  Sin lista de inicialización (ineficiente):
  Persona::Persona(const std::string& nombre, int edad) {
      nombre_ = nombre;  // primero construye nombre_ vacío, luego asigna
      edad_ = edad;
  }

  Con lista de inicialización (eficiente):
  Persona::Persona(const std::string& nombre, int edad)
      : nombre_(nombre), edad_(edad) {}

  ------------------------------------------------------------------
  5. INICIALIZACIÓN UNIFORME (C++11)
  ------------------------------------------------------------------

  C++11 introdujo la sintaxis con llaves {} para inicialización uniforme:

  class Circulo {
  public:
      Circulo(double radio) : radio_(radio) {}
  private:
      double radio_;
  };

  Circulo c1{5.0};     // uniform initialization
  Circulo c2 = {5.0};  // copy-list-initialization

  Ventajas: evita la narrowing conversion (conversión estrecha) y
  funciona con agregados (arrays, structs).

  ------------------------------------------------------------------
  6. CONSTRUCTOR DE COPIA (COPY CONSTRUCTOR)
  ------------------------------------------------------------------

  El constructor de copia crea un nuevo objeto como copia de otro existente.

  class Punto {
  public:
      Punto(double x, double y) : x_(x), y_(y) {}
      Punto(const Punto& otro) : x_(otro.x_), y_(otro.y_) {}
  private:
      double x_, y_;
  };

  // Uso:
  Punto p1(3.0, 4.0);
  Punto p2(p1);         // constructor de copia
  Punto p3 = p1;        // también constructor de copia
  Punto p4{p1};         // uniform copy

  El compilador genera un constructor de copia por defecto que copia
  miembro a miembro (shallow copy). Cuando la clase gestiona recursos
  (memoria dinámica, archivos), debemos implementarlo para hacer una
  copia profunda (deep copy).

  ------------------------------------------------------------------
  7. COPIA PROFUNDA (DEEP COPY) VS COPIA SUPERFICIAL (SHALLOW COPY)
  ------------------------------------------------------------------

  Shallow copy: copia el puntero, ambos objetos apuntan a la misma
  memoria. Peligroso: doble delete.

  Deep copy: copia el contenido al que apunta el puntero. Cada objeto
  tiene su propia copia de los datos.

  class Texto {
  private:
      char* buffer_;
      size_t longitud_;
  public:
      // Constructor
      Texto(const char* str) {
          longitud_ = strlen(str);
          buffer_ = new char[longitud_ + 1];
          strcpy(buffer_, str);
      }

      // Constructor de copia (deep copy)
      Texto(const Texto& otro)
          : longitud_(otro.longitud_) {
          buffer_ = new char[longitud_ + 1];
          strcpy(buffer_, otro.buffer_);
      }

      // Destructor
      ~Texto() {
          delete[] buffer_;
      }

      // Asignación (operator=)
      Texto& operator=(const Texto& otro) {
          if (this != &otro) {
              delete[] buffer_;
              longitud_ = otro.longitud_;
              buffer_ = new char[longitud_ + 1];
              strcpy(buffer_, otro.buffer_);
          }
          return *this;
      }
  };

  Esta es la Regla de Tres: si necesitas destructor, constructor de
  copia u operator= personalizados, generalmente necesitas los tres.

  ------------------------------------------------------------------
  8. CONSTRUCTOR POR DEFECTO GENERADO POR EL COMPILADOR
  ------------------------------------------------------------------

  El compilador genera un constructor por defecto si y solo si no hemos
  declarado ningún constructor. Ese constructor generado:
  • Tiene la firma ClassName() = default;
  • Llama al constructor por defecto de cada base y de cada miembro.
  • Para tipos primitivos (int, double, etc.), no inicializa (valor
    indeterminado).

  class A {
      int x;      // no inicializado por el constructor por defecto
      std::string s;  // inicializado vacío por su constructor por defecto
  };

  Podemos forzar la generación: A() = default;
  Podemos eliminar:        A() = delete;

  ------------------------------------------------------------------
  9. EXPLICIT KEYWORD
  ------------------------------------------------------------------

  La palabra clave explicit impide que el compilador use el constructor
  para conversiones implícitas.

  class Vector2D {
  public:
      explicit Vector2D(double magnitud) : x_(magnitud), y_(magnitud) {}
      Vector2D(double x, double y) : x_(x), y_(y) {}
  private:
      double x_, y_;
  };

  void procesar(Vector2D v);

  procesar(5.0);  // ERROR si el constructor es explicit
  procesar(Vector2D(5.0));  // OK: conversión explícita

  Sin explicit, el compilador podría convertir implícitamente un double
  a Vector2D, lo que puede causar bugs sutiles.

  Recomendación: marca todos los constructores de un solo parámetro
  como explicit a menos que tengas una buena razón para no hacerlo.

  ------------------------------------------------------------------
  10. CONSTRUCTORES DELEGADOS (C++11)
  ------------------------------------------------------------------

  Un constructor puede delegar en otro constructor de la misma clase:

  class Empleado {
  private:
      std::string nombre_;
      int edad_;
      double salario_;
  public:
      Empleado(const std::string& nombre, int edad, double salario)
          : nombre_(nombre), edad_(edad), salario_(salario) {}

      // Delegación: llama al constructor de arriba
      Empleado(const std::string& nombre)
          : Empleado(nombre, 18, 0.0) {}

      Empleado()
          : Empleado("Sin nombre", 0, 0.0) {}
  };

  La delegación evita duplicación de código de inicialización.

  ------------------------------------------------------------------
  11. CONSTRUCTOR DE MOVIMIENTO (MOVE CONSTRUCTOR) (C++11)
  ------------------------------------------------------------------

  El constructor de movimiento transfiere recursos de un objeto temporal
  a otro, evitando copias costosas.

  class Texto {
  public:
      // Move constructor
      Texto(Texto&& otro) noexcept
          : buffer_(otro.buffer_), longitud_(otro.longitud_) {
          otro.buffer_ = nullptr;  // dejar al otro en estado válido
          otro.longitud_ = 0;
      }
  };

  Se invoca cuando pasamos un objeto temporal o usamos std::move():

  Texto crearTexto() {
      return Texto("Hola");  // mueve (no copia) desde C++11
  }

  Texto t1("Mundo");
  Texto t2(std::move(t1));  // mueve t1 a t2

  ------------------------------------------------------------------
  12. CONSTRUCTORES Y ARREGLOS
  ------------------------------------------------------------------

  Si una clase no tiene constructor por defecto, no podemos crear un
  arreglo de objetos sin inicializarlos explícitamente.

  class Punto {
  public:
      Punto(double x, double y) : x_(x), y_(y) {}
  private:
      double x_, y_;
  };

  Punto arr[3];  // ERROR: no hay constructor por defecto
  Punto arr2[3] = {Punto(1,2), Punto(3,4), Punto(5,6)};  // OK

  Solución: proporcionar constructor por defecto o usar std::vector.

  ------------------------------------------------------------------
  13. CONSTRUCTOR PRIVADO
  ------------------------------------------------------------------

  Podemos tener constructores privados para evitar que el usuario cree
  objetos directamente. Útil para patrones como Singleton o Factory.

  class Singleton {
  private:
      Singleton() = default;
  public:
      static Singleton& getInstance() {
          static Singleton instancia;
          return instancia;
      }
      Singleton(const Singleton&) = delete;
      void operator=(const Singleton&) = delete;
  };

  ------------------------------------------------------------------
  14. CONSTRUCTOR Y HERENCIA
  ------------------------------------------------------------------

  En herencia, el constructor de la clase base se llama antes que el
  constructor de la clase derivada. Veremos esto en detalle en el
  capítulo 16.

  class Base {
  public:
      Base(int v) : valor_(v) {}
  private:
      int valor_;
  };

  class Derivada : public Base {
  public:
      Derivada(int v, const std::string& n)
          : Base(v), nombre_(n) {}  // inicializar base primero
  private:
      std::string nombre_;
  };

#ifndef PUNTO_HPP
#define PUNTO_HPP

#include <iostream>
#include <cmath>

class Punto {
public:
    // Constructor por defecto (con valores por defecto)
    Punto();

    // Constructor con parámetros
    Punto(double x, double y);

    // Constructor de copia
    Punto(const Punto& otro);

    // Constructor de movimiento
    Punto(Punto&& otro) noexcept;

    // Constructor delegado (usando un solo parámetro)
    explicit Punto(double valor);

    // Destructor
    ~Punto();

    // Métodos
    double distanciaAlOrigen() const;
    double distanciaA(const Punto& otro) const;
    Punto operator+(const Punto& otro) const;
    void mostrar() const;

    // Getters
    double getX() const { return x_; }
    double getY() const { return y_; }

private:
    double x_;
    double y_;
};

#endif

#include "punto.hpp"

Punto::Punto()
    : x_(0.0), y_(0.0) {
    std::cout << "Punto() por defecto: (" << x_ << ", " << y_ << ")\n";
}

Punto::Punto(double x, double y)
    : x_(x), y_(y) {
    std::cout << "Punto(" << x << ", " << y << ") creado\n";
}

Punto::Punto(const Punto& otro)
    : x_(otro.x_), y_(otro.y_) {
    std::cout << "Punto copiado desde (" << otro.x_ << ", " << otro.y_ << ")\n";
}

Punto::Punto(Punto&& otro) noexcept
    : x_(otro.x_), y_(otro.y_) {
    otro.x_ = 0.0;
    otro.y_ = 0.0;
    std::cout << "Punto movido\n";
}

Punto::Punto(double valor)
    : Punto(valor, valor) {
    std::cout << "Punto delegado con valor " << valor << "\n";
}

Punto::~Punto() {
    std::cout << "~Punto() destruyendo (" << x_ << ", " << y_ << ")\n";
}

double Punto::distanciaAlOrigen() const {
    return std::sqrt(x_ * x_ + y_ * y_);
}

double Punto::distanciaA(const Punto& otro) const {
    double dx = x_ - otro.x_;
    double dy = y_ - otro.y_;
    return std::sqrt(dx * dx + dy * dy);
}

Punto Punto::operator+(const Punto& otro) const {
    return Punto(x_ + otro.x_, y_ + otro.y_);
}

void Punto::mostrar() const {
    std::cout << "(" << x_ << ", " << y_ << ")" << std::endl;
}

#include "texto.hpp"

void Texto::copiarCadena(const char* str) {
    longitud_ = std::strlen(str);
    buffer_ = new char[longitud_ + 1];
    std::strcpy(buffer_, str);
}

Texto::Texto()
    : buffer_(nullptr), longitud_(0) {
    buffer_ = new char[1];
    buffer_[0] = '\0';
    std::cout << "Texto() por defecto\n";
}

Texto::Texto(const char* str) {
    copiarCadena(str);
    std::cout << "Texto(\"" << str << "\") creado\n";
}

Texto::Texto(const Texto& otro)
    : buffer_(nullptr), longitud_(otro.longitud_) {
    buffer_ = new char[longitud_ + 1];
    std::strcpy(buffer_, otro.buffer_);
    std::cout << "Texto copiado: \"" << otro.buffer_ << "\"\n";
}

Texto::Texto(Texto&& otro) noexcept
    : buffer_(otro.buffer_), longitud_(otro.longitud_) {
    otro.buffer_ = nullptr;
    otro.longitud_ = 0;
    std::cout << "Texto movido\n";
}

Texto::~Texto() {
    delete[] buffer_;
    std::cout << "~Texto() destructor\n";
}

Texto& Texto::operator=(const Texto& otro) {
    if (this != &otro) {
        delete[] buffer_;
        longitud_ = otro.longitud_;
        buffer_ = new char[longitud_ + 1];
        std::strcpy(buffer_, otro.buffer_);
        std::cout << "Texto asignado por copia\n";
    }
    return *this;
}

Texto& Texto::operator=(Texto&& otro) noexcept {
    if (this != &otro) {
        delete[] buffer_;
        buffer_ = otro.buffer_;
        longitud_ = otro.longitud_;
        otro.buffer_ = nullptr;
        otro.longitud_ = 0;
        std::cout << "Texto asignado por movimiento\n";
    }
    return *this;
}

const char* Texto::c_str() const { return buffer_; }
size_t Texto::longitud() const { return longitud_; }

void Texto::mostrar() const {
    std::cout << (buffer_ ? buffer_ : "(null)") << std::endl;
}

#include <iostream>
#include "punto.hpp"
#include "texto.hpp"

// Función que recibe por valor (prueba copia/movimiento)
void procesarPunto(Punto p) {
    p.mostrar();
}

int main() {
    std::cout << "=== CONSTRUCTOR POR DEFECTO ===" << std::endl;
    Punto p1;
    p1.mostrar();

    std::cout << "\n=== CONSTRUCTOR CON PARÁMETROS ===" << std::endl;
    Punto p2(3.0, 4.0);
    Punto p3(7.0, 1.0);

    std::cout << "\n=== CONSTRUCTOR DE COPIA ===" << std::endl;
    Punto p4(p2);
    Punto p5 = p2;

    std::cout << "\n=== CONSTRUCTOR DELEGADO ===" << std::endl;
    Punto p6(9.0);

    std::cout << "\n=== OPERACIONES ===" << std::endl;
    std::cout << "Distancia de p2 al origen: " << p2.distanciaAlOrigen() << std::endl;
    std::cout << "Distancia entre p2 y p3: " << p2.distanciaA(p3) << std::endl;

    Punto suma = p2 + p3;
    std::cout << "Suma p2 + p3: ";
    suma.mostrar();

    std::cout << "\n=== PASE POR VALOR (COPIA) ===" << std::endl;
    procesarPunto(p2);

    std::cout << "\n=== CLASE TEXTO (DEEP COPY) ===" << std::endl;
    Texto t1("Hola Mundo");
    Texto t2(t1);         // deep copy
    Texto t3 = t1;        // deep copy

    std::cout << "t1: "; t1.mostrar();
    std::cout << "t2: "; t2.mostrar();
    std::cout << "t3: "; t3.mostrar();

    std::cout << "\n=== ASIGNACIÓN ===" << std::endl;
    Texto t4("Temporal");
    t4 = t1;
    std::cout << "t4 después de asignación: "; t4.mostrar();

    std::cout << "\n=== FIN DEL PROGRAMA ===" << std::endl;
    return 0;
}

==========================================================================
  RESUMEN DEL CAPÍTULO
==========================================================================

  • Constructor: función miembro especial sin tipo de retorno, mismo
    nombre que la clase, se invoca al crear un objeto.
  • Constructor por defecto: no requiere argumentos. Se genera
    automáticamente si no declaramos ningún constructor.
  • Lista de inicialización: forma correcta y eficiente de inicializar
    miembros. Obligatoria para const y referencias.
  • Constructor de copia: crea un objeto como copia de otro. Cuidado
    con la deep copy vs shallow copy.
  • Constructor de movimiento (C++11): transfiere recursos de un objeto
    temporal, evitando copias costosas.
  • explicit: evita conversiones implícitas en constructores de un
    solo parámetro.
  • Constructores delegados (C++11): un constructor puede llamar a
    otro de la misma clase.
  • = default: fuerza la generación del constructor por defecto.
  • = delete: elimina el constructor (objeto no se puede construir de
    esa forma).
  • Regla de Tres: destructor, copia y asignación suelen ir juntos.
    Con movimiento (C++11), Regla de Cinco.

==========================================================================
  PREGUNTAS FRECUENTES (FAQ)
==========================================================================

  P: ¿Puedo llamar a un constructor manualmente después de crear el
     objeto?
  R: No directamente. Los constructores solo se invocan al crear el
     objeto. Puedes usar operator= para reasignar valores, o usar
     placement new en casos avanzados.

  P: ¿Qué es la narrowing conversion y cómo la evitan las llaves {}?
  R: Narrowing conversion es cuando se pierde precisión (ej: double a
     int). La inicialización uniforme con {} genera error en estos casos:
     int x{3.14}; // ERROR: narrowing conversion.

  P: ¿Por qué el constructor de copia recibe const referencia?
  R: Para evitar copias infinitas. Si fuera por valor, al pasar el
     argumento se llamaría al constructor de copia, creando un ciclo.

  P: ¿Qué diferencia hay entre Punto p = 5 y Punto p(5)?
  R: Con explicit, Punto p = 5 no compila (conversión implícita).
     Punto p(5) siempre funciona porque es inicialización directa.

  P: ¿Cuándo se genera un constructor de copia automáticamente?
  R: Cuando no lo declaramos ni lo eliminamos (= delete). El generado
     copia miembro a miembro (shallow copy).

  P: ¿Qué es noexcept en el constructor de movimiento?
  R: Indica que el constructor no lanza excepciones. Es importante para
     optimizaciones (std::vector lo usa para decidir si mover o copiar).

  P: ¿Puedo tener un constructor virtual?
  R: No. Los constructores no pueden ser virtuales porque la tabla
     virtual (vtable) aún no está configurada cuando se ejecuta el
     constructor de la base.

  CIERRE DEL CAPÍTULO

  Los constructores son la puerta de entrada a la vida de un objeto. Has
  aprendido las distintas formas de inicializar objetos: por defecto,
  con parámetros, por copia, por movimiento, y cómo controlar el proceso
  con explicit y listas de inicialización.

  Este conocimiento es fundamental porque un objeto mal inicializado es
  fuente de bugs difíciles de rastrear. La lista de inicialización, en
  particular, es una de las características más importantes de C++ que
  debes usar siempre.

  En el próximo capítulo abordaremos el complemento natural del constructor:
  el destructor. Veremos cómo liberar recursos y aplicar RAII, uno de los
  patrones más poderosos de C++.
