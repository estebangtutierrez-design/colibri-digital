  Pero entonces surge una pregunta: como accedemos a los datos privados
  de forma controlada?

  La respuesta son los getters (obtener) y setters (establecer). Son
  funciones miembro publicas que permiten leer y modificar atributos
  privados respectivamente, aplicando validaciones y reglas de negocio.

  En este capitulo profundizaremos en el diseno correcto de getters y
  setters, el uso del calificador const, la validacion de datos, la
  devolucion de referencias vs copias, y las propiedades computadas.

  ------------------------------------------------------------------
  1. QUE SON GETTERS Y SETTERS?
  ------------------------------------------------------------------

  Getter: funcion miembro que devuelve el valor de un atributo privado.
  Setter: funcion miembro que modifica el valor de un atributo privado.

  class Persona {
  private:
      std::string nombre_;
      int edad_;
  public:
      // Getters
      std::string getNombre() const { return nombre_; }
      int getEdad() const { return edad_; }

      // Setters
      void setNombre(const std::string& nombre) { nombre_ = nombre; }
      void setEdad(int edad) {
          if (edad >= 0 && edad <= 150) {
              edad_ = edad;
          }
      }
  };

  ------------------------------------------------------------------
  2. POR QUE USARLOS?
  ------------------------------------------------------------------

  Control de acceso: podemos decidir que atributos son solo lectura
  (getter sin setter), solo escritura (setter sin getter) o ambos.

  Validacion: los setters pueden rechazar valores invalidos.

  Mantenibilidad: podemos cambiar la representacion interna sin
  afectar a los usuarios de la clase.

  Efectos secundarios: un setter puede notificar cambios, actualizar
  logs, o invalidar cache.

  ------------------------------------------------------------------
  3. CONST EN GETTERS
  ------------------------------------------------------------------

  Todos los getters deben ser const (a menos que haya una razon muy
  especifica para no serlo). Esto permite:

  - Llamar a getters sobre objetos constantes.
  - Usar objectos const en parametros de funciones.
  - Expresar claramente que el metodo no modifica el objeto.

  class Circulo {
  private:
      double radio_;
  public:
      double getRadio() const { return radio_; }  // const correcto
      void setRadio(double r) { radio_ = r; }      // no const
  };

  void mostrarRadio(const Circulo& c) {
      std::cout << c.getRadio();  // necesita getter const
  }

  ------------------------------------------------------------------
  4. DEVOLVER POR VALOR VS REFERENCIA
  ------------------------------------------------------------------

  Por valor (recomendado para tipos pequenos):
  int getEdad() const { return edad_; }

  Por referencia const (para objetos grandes):
  const std::string& getNombre() const { return nombre_; }

  Por referencia (solo si se necesita modificar, poco comun):
  int& getEdad() { return edad_; }  // peligroso, rompe encapsulacion

  Buenas practicas:
  - Tipos primitivos (int, double, char): devolver por valor.
  - std::string, vectores, objetos grandes: devolver por ref const.
  - Nunca devolver referencias no-const a datos internos.

  ------------------------------------------------------------------
  5. SETTERS CON VALIDACION
  ------------------------------------------------------------------

  Los setters son el lugar ideal para validar datos:

  class CuentaBancaria {
  private:
      double saldo_;
      std::string titular_;
  public:
      void setSaldo(double saldo) {
          if (saldo < 0) {
              throw std::invalid_argument("El saldo no puede ser negativo");
          }
          saldo_ = saldo;
      }

      void setTitular(const std::string& titular) {
          if (titular.empty()) {
              throw std::invalid_argument("El titular no puede estar vacio");
          }
          titular_ = titular;
      }
  };

  Tambien podemos registrar cambios o notificar a otros componentes.

  ------------------------------------------------------------------
  6. SETTERS QUE RETORNAN REFERENCIA A THIS
  ------------------------------------------------------------------

  Para permitir encadenamiento (method chaining):

  class Punto {
  private:
      int x_, y_;
  public:
      Punto& setX(int x) { x_ = x; return *this; }
      Punto& setY(int y) { y_ = y; return *this; }

      void mostrar() const {
          std::cout << "(" << x_ << ", " << y_ << ")\n";
      }
  };

  // Uso:
  Punto p;
  p.setX(10).setY(20).mostrar();

  Veremos mas sobre this en el capitulo 18.

  ------------------------------------------------------------------
  7. PROPIEDADES COMPUTADAS (GETTERS SIN RESPALDO DIRECTO)
  ------------------------------------------------------------------

  No todos los getters corresponden a un atributo directo:

  class Rectangulo {
  private:
      double ancho_;
      double alto_;
  public:
      double getAncho() const { return ancho_; }
      double getAlto() const { return alto_; }

      // Propiedades computadas:
      double getArea() const { return ancho_ * alto_; }
      double getPerimetro() const { return 2.0 * (ancho_ + alto_); }
  };

  class Persona {
  private:
      std::string nombre_;
      int edad_;
  public:
      std::string getNombreCompleto() const {
          return nombre_ + " (" + std::to_string(edad_) + " anos)";
      }
  };

  ------------------------------------------------------------------
  8. GETTERS QUE DEVUELVEN OBJETOS POR COPIA
  ------------------------------------------------------------------

  Cuidado al devolver objetos grandes por copia:

  // Ineficiente: copia todo el vector cada vez
  std::vector<int> getDatos() const { return datos_; }

  // Eficiente: devuelve referencia const
  const std::vector<int>& getDatos() const { return datos_; }

  Si necesitas devolver una copia (para evitar que el usuario modifique
  tus datos internos a traves de una referencia), devuelve por valor.

  ------------------------------------------------------------------
  9. SETTERS CON EFECTOS SECUNDARIOS
  ------------------------------------------------------------------

  class Sensor {
  private:
      double temperatura_;
      bool limiteSuperado_;
      double limite_;
  public:
      void setTemperatura(double temp) {
          temperatura_ = temp;
          limiteSuperado_ = (temp > limite_);  // efecto secundario
          if (limiteSuperado_) {
              notificarAlarma();  // otro efecto
          }
      }

      bool getLimiteSuperado() const { return limiteSuperado_; }

  private:
      void notificarAlarma() const {
          std::cout << "ALARMA: temperatura supera el limite!\n";
      }
  };

  ------------------------------------------------------------------
  10. INMUTABILIDAD: SOLO GETTERS SIN SETTERS
  ------------------------------------------------------------------

  A veces queremos que un atributo sea publico en lectura pero no
  modificable despues de la construccion:

  class Producto {
  private:
      const int id_;         // constante: no cambia jamas
      std::string nombre_;
      double precio_;
  public:
      Producto(int id, const std::string& nombre, double precio)
          : id_(id), nombre_(nombre), precio_(precio) {}

      int getId() const { return id_; }        // solo lectura
      std::string getNombre() const { return nombre_; }
      double getPrecio() const { return precio_; }

      void setNombre(const std::string& nombre) { nombre_ = nombre; }
      void setPrecio(double precio) { precio_ = precio; }
      // NO hay setId: el ID es inmutable
  };

  ------------------------------------------------------------------
  11. GETTERS Y SETTERS CON STL
  ------------------------------------------------------------------

  Con contenedores STL, los getters/setters requieren atencion extra:

  class Equipo {
  private:
      std::vector<std::string> miembros_;
  public:
      // Getter seguro: devuelve copia
      std::vector<std::string> getMiembros() const {
          return miembros_;
      }

      // Getter eficiente: devuelve ref const
      const std::vector<std::string>& getMiembrosRef() const {
          return miembros_;
      }

      // Setter que reemplaza todo
      void setMiembros(const std::vector<std::string>& m) {
          miembros_ = m;
      }

      // Acceso a elementos individuales
      std::string getMiembro(size_t index) const {
          if (index < miembros_.size()) {
              return miembros_[index];
          }
          throw std::out_of_range("Indice fuera de rango");
      }

      void agregarMiembro(const std::string& nombre) {
          miembros_.push_back(nombre);
      }
  };

  ------------------------------------------------------------------
  12. BOOLEAN GETTERS: PREFIJO IS O HAS
  ------------------------------------------------------------------

  Para atributos booleanos, los getters suelen usar prefijo is o has:

  class Pedido {
  private:
      bool enviado_;
      bool pagado_;
      bool tieneDescuento_;
  public:
      bool isEnviado() const { return enviado_; }
      bool isPagado() const { return pagado_; }
      bool hasDescuento() const { return tieneDescuento_; }

      void setEnviado(bool v) { enviado_ = v; }
      void setPagado(bool v) { pagado_ = v; }
      void setDescuento(bool v) { tieneDescuento_ = v; }
  };

  ------------------------------------------------------------------
  13. GETTERS QUE DEVUELVEN REFERENCIA A MIEMBRO INTERNO
  ------------------------------------------------------------------

  A veces es necesario pero peligroso:

  class Arreglo {
  private:
      int datos_[100];
  public:
      // Permite modificar el elemento, rompe encapsulacion
      int& operator[](size_t i) { return datos_[i]; }

      // Solo lectura, mantiene encapsulacion
      const int& operator[](size_t i) const { return datos_[i]; }
  };

  Sobrecargar con & y const& es una tecnica comun en contenedores.

  ------------------------------------------------------------------
  14. GETTERS Y SETTERS EN CLASES CON HERENCIA
  ------------------------------------------------------------------

  class Base {
  private:
      int datoPrivado_;
  protected:
      int datoProtegido_;
  public:
      int getDatoPrivado() const { return datoPrivado_; }
      void setDatoPrivado(int v) { datoPrivado_ = v; }
  };

  class Derivada : public Base {
  public:
      void ejemplo() {
          // datoPrivado_ = 5;       // ERROR: private
          datoProtegido_ = 10;        // OK: protected
          setDatoPrivado(15);         // OK: getter/setter publicos
      }
  };

  Las derivadas acceden a los datos privados de la base solo a traves
  de getters/setters publicos o protected.

#include "empleado.hpp"

Empleado::Empleado(int id, const std::string& nombre, double salarioBase)
    : id_(id), activo_(true) {
    validarNombre(nombre);
    validarSalario(salarioBase);
    nombre_ = nombre;
    salarioBase_ = salarioBase;
}

void Empleado::validarNombre(const std::string& nombre) const {
    if (nombre.empty()) {
        throw std::invalid_argument("El nombre no puede estar vacio");
    }
}

void Empleado::validarSalario(double salario) const {
    if (salario < 0.0) {
        throw std::invalid_argument("El salario no puede ser negativo");
    }
    if (salario > 1000000.0) {
        throw std::invalid_argument("El salario excede el maximo permitido");
    }
}

int Empleado::getId() const { return id_; }

std::string Empleado::getNombre() const { return nombre_; }

double Empleado::getSalarioBase() const { return salarioBase_; }

double Empleado::getSalarioNeto() const {
    // Propiedad computada: aplica impuestos y deducciones
    double impuesto = 0.0;
    if (salarioBase_ > 3000.0) {
        impuesto = salarioBase_ * 0.15;
    } else if (salarioBase_ > 1500.0) {
        impuesto = salarioBase_ * 0.10;
    }
    return salarioBase_ - impuesto;
}

bool Empleado::isActivo() const { return activo_; }

std::vector<std::string> Empleado::getHabilidades() const {
    return habilidades_;  // copia intencional
}

void Empleado::setNombre(const std::string& nombre) {
    validarNombre(nombre);
    nombre_ = nombre;
}

void Empleado::setSalarioBase(double salario) {
    validarSalario(salario);
    salarioBase_ = salario;
}

void Empleado::setActivo(bool activo) { activo_ = activo; }

void Empleado::agregarHabilidad(const std::string& habilidad) {
    if (!habilidad.empty()) {
        habilidades_.push_back(habilidad);
    }
}

Empleado& Empleado::setDepartamento(const std::string& dept) {
    departamento_ = dept;
    return *this;
}

void Empleado::mostrar() const {
    std::cout << "=== EMPLEADO ===" << std::endl;
    std::cout << "ID: " << id_ << std::endl;
    std::cout << "Nombre: " << nombre_ << std::endl;
    std::cout << "Salario Base: $" << salarioBase_ << std::endl;
    std::cout << "Salario Neto: $" << getSalarioNeto() << std::endl;
    std::cout << "Activo: " << (activo_ ? "Si" : "No") << std::endl;
    std::cout << "Departamento: " << departamento_ << std::endl;
    std::cout << "Habilidades: ";
    for (const auto& h : habilidades_) {
        std::cout << h << " ";
    }
    std::cout << std::endl;
}

#include "termostato.hpp"

Termostato::Termostato(double temperaturaInicial, double min, double max)
    : minimo_(min), maximo_(max) {
    temperatura_ = std::clamp(temperaturaInicial, minimo_, maximo_);
    actualizarActuadores();
}

double Termostato::getTemperatura() const { return temperatura_; }
double Termostato::getMinimo() const { return minimo_; }
double Termostato::getMaximo() const { return maximo_; }
bool Termostato::isCalefaccionEncendida() const { return calefaccion_; }
bool Termostato::isAireAcondicionadoEncendido() const { return aireAcondicionado_; }

void Termostato::setTemperatura(double temp) {
    temperatura_ = std::clamp(temp, minimo_, maximo_);
    actualizarActuadores();
}

void Termostato::setMinimo(double min) {
    minimo_ = min;
    actualizarActuadores();
}

void Termostato::setMaximo(double max) {
    maximo_ = max;
    actualizarActuadores();
}

void Termostato::actualizarActuadores() {
    if (temperatura_ < minimo_) {
        calefaccion_ = true;
        aireAcondicionado_ = false;
    } else if (temperatura_ > maximo_) {
        calefaccion_ = false;
        aireAcondicionado_ = true;
    } else {
        calefaccion_ = false;
        aireAcondicionado_ = false;
    }
}

void Termostato::mostrarEstado() const {
    std::cout << "=== TERMOSTATO ===" << std::endl;
    std::cout << "Temperatura: " << temperatura_ << " C" << std::endl;
    std::cout << "Rango: [" << minimo_ << " - " << maximo_ << "] C" << std::endl;
    std::cout << "Calefaccion: " << (calefaccion_ ? "ENCENDIDA" : "APAGADA") << std::endl;
    std::cout << "A/A: " << (aireAcondicionado_ ? "ENCENDIDO" : "APAGADO") << std::endl;
}

==========================================================================
  RESUMEN DEL CAPÍTULO
==========================================================================

  Los getters proporcionan acceso de lectura controlado a atributos
  privados. Deben ser const y devolver por valor (tipos pequenos) o
  referencia const (objetos grandes).

  Los setters proporcionan acceso de escritura controlado. Deben
  validar los datos y mantener los invariantes de la clase.

  Las propiedades computadas son getters que calculan un valor en
  lugar de devolver un atributo directamente.

  Los getters booleanos usan prefijo is o has.

  El encadenamiento de setters se logra devolviendo *this (referencia).

  Los getters/setters son la puerta de entrada a la encapsulacion.

==========================================================================
  PREGUNTAS FRECUENTES (FAQ)
==========================================================================

  P: Debo crear getters y setters para todos los atributos?
  R: No. Solo para los que necesiten acceso externo. Algunos atributos
     pueden ser puramente internos y no necesitar getter ni setter.

  P: Es mejor devolver por valor o por referencia?
  R: Para int, double, char, bool -> por valor. Para string, vectores,
     objetos grandes -> por referencia const.

  P: Puedo tener un getter que modifique el objeto?
  R: Si, usando mutable (visto en cap 11) o舍弃 getters no-const, pero
     rompe la semantica. Mejor redisenar.

  P: Como hago para que un setter valide y notifique?
  R: Ejecuta la validacion primero, luego modifica, luego ejecuta
     notificaciones o efectos secundarios.

  P: Que es method chaining?
  R: Es cuando los setters devuelven *this para permitir llamadas
     encadenadas: obj.setX(1).setY(2).setZ(3);
