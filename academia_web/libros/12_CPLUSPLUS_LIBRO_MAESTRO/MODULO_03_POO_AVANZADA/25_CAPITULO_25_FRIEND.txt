  ------------------------------------------------------------------
  1. ¿QUÉ ES FRIEND?
  ------------------------------------------------------------------

  La palabra clave 'friend' permite que una función o clase externa
  acceda a los miembros privados y protegidos de una clase.

  Es una excepción controlada a la encapsulación. La clase propietaria
  decide quién es su amigo y qué acceso tiene.

  Características fundamentales:
  • friend NO es simétrico: si A es friend de B, B NO es friend de A.
  • friend NO es transitivo: si A es friend de B y B es friend de C,
    A NO es friend de C.
  • friend NO se hereda: si B es friend de A, las derivadas de B NO
    son friend de A.
  • La declaración friend se hace DENTRO de la clase que otorga acceso.

  ------------------------------------------------------------------
  2. FUNCIONES FRIEND
  ------------------------------------------------------------------

  Una función friend puede ser:
  a) Una función libre (no miembro de ninguna clase).
  b) Una función miembro de otra clase.

  Sintaxis:

    class Clase {
        friend void funcionLibre(Clase& obj);
        friend void OtraClase::metodoAmigo(Clase& obj);
        //...
    };

  La función friend recibe típicamente un objeto de la clase por
  referencia o puntero y accede a sus miembros privados.

  ------------------------------------------------------------------
  3. CLASES FRIEND
  ------------------------------------------------------------------

  Cuando toda una clase es declarada friend, TODAS sus funciones
  miembro tienen acceso a los miembros privados de la clase amiga.

  Sintaxis:

    class Clase {
        friend class ClaseAmiga;
        //...
    };

  Útil cuando dos clases están fuertemente acopladas por diseño
  (ej: un contenedor y su iterador).

  ------------------------------------------------------------------
  4. FORWARD DECLARATION CON FRIEND
  ------------------------------------------------------------------

  A veces necesitas declarar una friend de una clase que aún no está
  definida. Usas forward declaration:

    class ClaseB;  // Forward declaration

    class ClaseA {
        friend class ClaseB;
        //...
    };

    class ClaseB {
        // Puede acceder a miembros privados de ClaseA
    };

  ------------------------------------------------------------------
  5. USOS COMUNES DE FRIEND
  ------------------------------------------------------------------

  • Sobrecarga de operadores (<<, >>, etc.) que necesitan acceso
    a miembros privados.
  • Clases iterador que necesitan acceso interno del contenedor.
  • Clases de prueba (testing) que verifican estado interno.
  • Fábricas que construyen objetos con estado complejo.
  • Clases en un mismo módulo con acoplamiento fuerte intencional.

  ------------------------------------------------------------------
  6. PROS Y CONTRAS DE FRIEND
  ------------------------------------------------------------------

  VENTAJAS:
  • Permite mantener la interfaz pública limpia.
  • Evita exponer getters/setters solo para casos específicos.
  • Útil para sobrecarga de operadores binarios.
  • Mejora rendimiento al evitar llamadas indirectas.

  DESVENTAJAS:
  • Viola la encapsulación.
  • Aumenta el acoplamiento.
  • Dificulta el mantenimiento (más dependencias).
  • Puede ser abusado.
  • No es compatible con algunos patrones de diseño.

  ------------------------------------------------------------------
  7. ALTERNATIVAS A FRIEND
  ------------------------------------------------------------------

  • Métodos públicos bien diseñados.
  • Getters/setters (aunque exponen detalles).
  • Interfaces que definan el acceso necesario.
  • Patrón Visitor (más complejo pero más limpio).
  • Métodos protected con herencia.

  ```cpp
  //====================================================================
  //  EJEMPLO 1: FUNCIÓN LIBRE FRIEND
  //====================================================================
  #include <iostream>
  #include <string>

  class CuentaBancaria {
  private:
      std::string titular_;
      double saldo_;
      std::string numeroCuenta_;

  public:
      CuentaBancaria(const std::string& t, double s, const std::string& n)
          : titular_(t), saldo_(s), numeroCuenta_(n) {}

      // Función libre friend
      friend void mostrarDetalleCuenta(const CuentaBancaria& c);

      // Clase friend
      friend class Auditor;

      // Método público normal
      double obtenerSaldo() const { return saldo_; }
  };

  // Función friend: puede acceder a miembros privados
  void mostrarDetalleCuenta(const CuentaBancaria& c) {
      std::cout << "=== DETALLE DE CUENTA ===" << std::endl;
      std::cout << "Titular: " << c.titular_ << std::endl;       // Privado
      std::cout << "Saldo: $" << c.saldo_ << std::endl;          // Privado
      std::cout << "Número: ****" << c.numeroCuenta_.substr(4) << std::endl;  // Privado
  }

  class Auditor {
  public:
      void auditar(const CuentaBancaria& c) {
          std::cout << "AUDITORÍA: Titular=" << c.titular_
                    << ", Saldo=$" << c.saldo_
                    << ", Nro=" << c.numeroCuenta_ << std::endl;
      }

      bool esSospechosa(const CuentaBancaria& c, double limite) {
          return c.saldo_ > limite;
      }
  };

  int main() {
      CuentaBancaria cuenta("Juan Pérez", 15000.50, "1234567890");

      mostrarDetalleCuenta(cuenta);

      Auditor aud;
      aud.auditar(cuenta);
      std::cout << "¿Sospechosa? " << (aud.esSospechosa(cuenta, 10000) ? "Sí" : "No") << std::endl;

      return 0;
  }
  ```

  ```
  SALIDA:
  === DETALLE DE CUENTA ===
  Titular: Juan Pérez
  Saldo: $15000.5
  Número: ****567890
  AUDITORÍA: Titular=Juan Pérez, Saldo=$15000.5, Nro=1234567890
  ¿Sospechosa? Sí
  ```

  //====================================================================
  //  EJEMPLO 2: FUNCIÓN MIEMBRO FRIEND
  //====================================================================
  #include <iostream>
  #include <string>

  class Ventana;  // Forward declaration

  class Boton {
  public:
      void click(Ventana& v);
      void hover(Ventana& v);
  };

  class Ventana {
  private:
      int ancho_;
      int alto_;
      std::string titulo_;
      bool visible_;

  public:
      Ventana(int w, int h, const std::string& t)
          : ancho_(w), alto_(h), titulo_(t), visible_(false) {}

      // Solo Boton::click es friend, NO Boton::hover
      friend void Boton::click(Ventana& v);

      void mostrar() { visible_ = true; }
  };

  void Boton::click(Ventana& v) {
      std::cout << "Botón clickeado en ventana "" << v.titulo_ << """ << std::endl;
      v.visible_ = !v.visible_;
      std::cout << "  Ventana ahora " << (v.visible_ ? "visible" : "oculta") << std::endl;
  }

  void Boton::hover(Ventana& v) {
      // No puede acceder a miembros privados - solo público
      std::cout << "Hover sobre ventana: " << sizeof(v) << " bytes" << std::endl;
      v.mostrar();  // Método público
  }

  int main() {
      Ventana vent(800, 600, "Mi Ventana");
      Boton btn;

      btn.hover(vent);
      btn.click(vent);
      btn.click(vent);

      return 0;
  }

  ```cpp
  //====================================================================
  //  EJEMPLO 3: CLASE FRIEND COMPLETA
  //====================================================================
  #include <iostream>
  #include <string>
  #include <vector>

  class Empresa;  // Forward

  class Empleado {
  private:
      std::string nombre_;
      double salario_;
      int id_;
      Empresa* empresa_;

  public:
      Empleado(const std::string& n, double s, int id)
          : nombre_(n), salario_(s), id_(id), empresa_(nullptr) {}

      friend class Empresa;  // Empresa completa es friend

      void mostrarInfo() const {
          std::cout << id_ << ": " << nombre_ << " ($" << salario_ << ")";
          if (empresa_) {
              // Empleado NO es friend de Empresa, solo puede usar públic
              std::cout << " en Empresa";
          }
          std::cout << std::endl;
      }
  };

  class Empresa {
  private:
      std::string nombre_;
      std::vector<Empleado*> empleados_;

  public:
      Empresa(const std::string& n) : nombre_(n) {}

      void contratar(Empleado* e) {
          empleados_.push_back(e);
          e->empresa_ = this;  // Acceso privado de Empleado
          e->id_ = empleados_.size() * 100;  // Reasigna ID interno
      }

      void darAumento(Empleado* e, double cantidad) {
          e->salario_ += cantidad;  // Acceso privado
          std::cout << "Aumento de $" << cantidad << " a "
                    << e->nombre_ << std::endl;
      }

      void despedir(Empleado* e) {
          e->empresa_ = nullptr;
          std::cout << e->nombre_ << " ha sido despedido." << std::endl;
      }

      void listarEmpleados() const {
          std::cout << "
=== Empleados de " << nombre_ << " ===" << std::endl;
          for (const auto* e : empleados_) {
              // Empresa puede acceder a miembros privados de Empleado
              std::cout << "  " << e->id_ << ": " << e->nombre_
                        << " ($" << e->salario_ << ")" << std::endl;
          }
      }
  };

  int main() {
      Empleado e1("Ana López", 3000, 1);
      Empleado e2("Carlos Ruiz", 4500, 2);
      Empleado e3("Laura Díaz", 5000, 3);

      Empresa emp("TechCorp");
      emp.contratar(&e1);
      emp.contratar(&e2);
      emp.contratar(&e3);

      emp.listarEmpleados();
      emp.darAumento(&e1, 500);
      emp.darAumento(&e3, 1000);

      emp.listarEmpleados();
      emp.despedir(&e2);
      emp.listarEmpleados();

      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 4: FRIEND NO ES TRANSITIVO NI HEREDABLE
  //====================================================================
  #include <iostream>
  #include <string>

  class A {
  private:
      int secretoA_ = 10;
      friend class B;  // B es friend de A
  };

  class B {
  private:
      int secretoB_ = 20;
      friend class C;  // C es friend de B
  public:
      void accederA(A& a) {
          std::cout << "B accede a A::secretoA_ = " << a.secretoA_ << std::endl;
      }
  };

  class C {
  public:
      void accederA(A& a) {
          // ERROR: C NO es friend de A (no transitivo)
          // std::cout << a.secretoA_;
          std::cout << "C NO puede acceder a A::secretoA_ (no transitivo)" << std::endl;
      }

      void accederB(B& b) {
          std::cout << "C accede a B::secretoB_ = " << b.secretoB_ << std::endl;
      }
  };

  class DerivadaB : public B {
  public:
      void accederA(A& a) {
          // ERROR: friend no se hereda
          // std::cout << a.secretoA_;
          std::cout << "DerivadaB NO puede acceder a A::secretoA_ (no hereda friend)" << std::endl;
      }
  };

  int main() {
      std::cout << "=== FRIEND NO ES TRANSITIVO ===" << std::endl;
      A a; B b; C c; DerivadaB db;

      b.accederA(a);
      c.accederA(a);
      c.accederB(b);
      db.accederA(a);

      return 0;
  }
  ```

  SALIDA:
  === FRIEND NO ES TRANSITIVO ===
  B accede a A::secretoA_ = 10
  C NO puede acceder a A::secretoA_ (no transitivo)
  C accede a B::secretoB_ = 20
  DerivadaB NO puede acceder a A::secretoA_ (no hereda friend)

  ```cpp
  //====================================================================
  //  EJEMPLO 5: FRIEND CON SOBRECARGA DE OPERADORES
  //====================================================================
  #include <iostream>
  #include <string>

  class Vector2D {
  private:
      double x_, y_;

  public:
      Vector2D(double x = 0, double y = 0) : x_(x), y_(y) {}

      // friend necesario para operador << (ostream)
      friend std::ostream& operator<<(std::ostream& os, const Vector2D& v);

      // friend para operador >> (istream)
      friend std::istream& operator>>(std::istream& is, Vector2D& v);

      // friend para operador binario que necesita acceso privado
      friend Vector2D operator*(double escalar, const Vector2D& v);
      friend Vector2D operator*(const Vector2D& v, double escalar);

      // Métodos públicos para suma (no necesita friend)
      Vector2D operator+(const Vector2D& otro) const {
          return Vector2D(x_ + otro.x_, y_ + otro.y_);
      }
  };

  std::ostream& operator<<(std::ostream& os, const Vector2D& v) {
      os << "(" << v.x_ << ", " << v.y_ << ")";
      return os;
  }

  std::istream& operator>>(std::istream& is, Vector2D& v) {
      std::cout << "Ingrese x e y: ";
      is >> v.x_ >> v.y_;
      return is;
  }

  Vector2D operator*(double escalar, const Vector2D& v) {
      return Vector2D(escalar * v.x_, escalar * v.y_);
  }

  Vector2D operator*(const Vector2D& v, double escalar) {
      return escalar * v;  // Reutiliza el otro operador
  }

  int main() {
      Vector2D v1(3, 4);
      Vector2D v2(1, 2);

      std::cout << "v1 = " << v1 << std::endl;
      std::cout << "v2 = " << v2 << std::endl;
      std::cout << "v1 + v2 = " << (v1 + v2) << std::endl;
      std::cout << "3 * v1 = " << (3 * v1) << std::endl;
      std::cout << "v1 * 2 = " << (v1 * 2) << std::endl;

      return 0;
  }
  ```

  //====================================================================
  //  EJEMPLO 6: FRIEND ENTRE CLASES ANIDADAS
  //====================================================================
  #include <iostream>
  #include <string>
  #include <vector>

  class ListaTareas {
  private:
      struct Tarea {
          std::string descripcion;
          bool completada;
          int prioridad;
      };

      std::vector<Tarea> tareas_;

  public:
      class Iterador {
      private:
          ListaTareas& lista_;
          size_t indice_;
      public:
          Iterador(ListaTareas& l, size_t i) : lista_(l), indice_(i) {}

          Tarea& operator*() { return lista_.tareas_[indice_]; }
          // Necesita friend o acceso público limitado

          void completar() {
              lista_.tareas_[indice_].completada = true;
          }

          void mostrar() const {
              const auto& t = lista_.tareas_[indice_];
              std::cout << (t.completada ? "[✓]" : "[ ]") << " "
                        << t.descripcion << " (p:" << t.prioridad << ")"
                        << std::endl;
          }
      };

      // La clase Iterador es friend para acceder a tareas_ privada
      friend class Iterador;

      void agregar(const std::string& desc, int pri = 5) {
          tareas_.push_back({desc, false, pri});
      }

      Iterador begin() { return Iterador(*this, 0); }
  };

  int main() {
      ListaTareas lista;
      lista.agregar("Comprar leche", 3);
      lista.agregar("Estudiar C++", 1);
      lista.agregar("Hacer ejercicio", 4);

      auto it = lista.begin();
      it.mostrar();
      it.completar();
      it.mostrar();

      return 0;
  }

================================================================================
  RESUMEN
================================================================================

  • friend permite que funciones o clases externas accedan a miembros
    privados/protegidos de una clase.
  • friend NO es simétrico, NO es transitivo y NO se hereda.
  • Forward declaration resuelve dependencias circulares con friend.
  • Usos comunes: sobrecarga de operadores, clases iterador, testing.
  • friend es controvertido: viola encapsulación pero evita exponer
    getters/setters innecesarios.
  • Alternativas: getters públicos, interfaces, patrón Visitor.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿friend rompe la encapsulación?
  R: Sí, pero es una violación controlada y explícita. La clase decide
    quién es su amigo.

  P: ¿Cuándo DEBO usar friend?
  R: Principalmente para sobrecarga de operadores binarios (<<, >>),
    para clases fuertemente acopladas (contenedor/iterador), y testing.

  P: ¿Cuándo NO debo usar friend?
  R: Cuando puedes lograr lo mismo con interfaz pública limpia. Si dos
    clases no están naturalmente acopladas, no uses friend.

  P: ¿friend se hereda?
  R: No. Si B es friend de A, las clases derivadas de B NO son friend.

  P: ¿friend es simétrico?
  R: No. Si A declara a B como friend, B accede a miembros privados de
    A, pero A NO accede a los de B.

  P: ¿Puedo declarar una plantilla como friend?
  R: Sí. template<typename T> friend class MiClase; es válido.

    - Verifica invariantes antes de guardar

  • Demostrar:
    - Entidad creada solo por Repositorio
    - Serialización sin getters públicos
    - Validación de invariantes internos
    - Caché de repositorio

  - Manejo de transacciones
