  ------------------------------------------------------------------
  1. ¿QUÉ ES UNA CLASE ABSTRACTA?
  ------------------------------------------------------------------

  Una clase abstracta es una clase que no puede ser instanciada
  directamente. Sirve como base para otras clases, definiendo una
  interfaz común que las clases derivadas deben implementar.

  En C++, una clase se vuelve abstracta cuando tiene al menos una
  función virtual pura (= 0).

    class ClaseAbstracta {
    public:
        virtual void metodoPuro() = 0;
        virtual ~ClaseAbstracta() = default;
    };

  No se puede crear un objeto de ClaseAbstracta directamente, pero sí
  se puede usar un puntero o referencia a ella.

  ------------------------------------------------------------------
  2. CARACTERÍSTICAS DE LAS CLASES ABSTRACTAS
  ------------------------------------------------------------------

  • Pueden tener datos miembro (atributos).
  • Pueden tener funciones concretas (con implementación).
  • Pueden tener constructores (llamados por clases derivadas).
  • Deben tener destructor virtual.
  • No se pueden instanciar directamente.
  • Una clase derivada DEBE implementar TODAS las funciones virtuales
    puras para ser concreta.
  • Si no implementa alguna, la derivada también es abstracta.

  ------------------------------------------------------------------
  3. CLASE ABSTRACTA VS INTERFAZ
  ------------------------------------------------------------------

  En C++ no existe el keyword 'interface' como en Java o C#. Las
  interfaces se simulan con clases abstractas que tienen solo funciones
  virtuales puras y ningún dato miembro.

    // Interfaz pura (estilo Java)
    class IImprimible {
    public:
        virtual ~IImprimible() = default;
        virtual void imprimir() = 0;
        virtual void obtenerPaginas() const = 0;
    };

  Diferencias:
  • Interfaz: solo métodos públicos virtuales puros, sin datos.
  • Clase abstracta: puede tener datos, métodos concretos, constructores.

  ------------------------------------------------------------------
  4. PATRÓN TEMPLATE METHOD CON CLASES ABSTRACTAS
  ------------------------------------------------------------------

  El patrón Template Method define el esqueleto de un algoritmo en la
  clase base abstracta. Las clases derivadas implementan los pasos
  específicos sin cambiar la estructura general del algoritmo.

  Estructura:
    class Base {
    public:
        void templateMethod() {
            paso1();
            paso2();
            paso3();
        }
    protected:
        virtual void paso1() = 0;
        virtual void paso2() = 0;
        void paso3() { /* implementación por defecto */ }
    };

  ------------------------------------------------------------------
  5. ABSTRACT FACTORY PATTERN
  ------------------------------------------------------------------

  El patrón Abstract Factory proporciona una interfaz para crear
  familias de objetos relacionados sin especificar sus clases concretas.

  En C++, se implementa típicamente con:
  • Una clase abstracta de fábrica con métodos virtuales puros.
  • Clases concretas de fábrica que crean productos concretos.
  • Productos que también heredan de clases abstractas.

  ------------------------------------------------------------------
  6. CLASES ABSTRACTAS COMO CONTRATO
  ------------------------------------------------------------------

  "Programar por contrato" significa que la clase base abstracta define
  qué hacer, y las derivadas definen cómo hacerlo. El contrato incluye:

  • Precondiciones: lo que debe cumplirse antes de llamar al método.
  • Postcondiciones: lo que el método garantiza después de ejecutarse.
  • Invariantes: condiciones que siempre se mantienen verdaderas.

  ------------------------------------------------------------------
  7. LIMITACIONES
  ------------------------------------------------------------------

  • No se pueden instanciar directamente.
  • No se pueden usar como tipos de parámetros por valor.
  • No se puede tener una función virtual pura en el constructor.
  • El operador de asignación no puede ser virtual puro.

  ```cpp
  //====================================================================
  //  EJEMPLO 1: CLASE ABSTRACTA BÁSICA
  //====================================================================
  #include <iostream>
  #include <string>

  class Animal {
  protected:
      std::string nombre_;
      int edad_;
  public:
      Animal(const std::string& nombre, int edad)
          : nombre_(nombre), edad_(edad) {}
      virtual ~Animal() = default;

      virtual void hacerSonido() const = 0;
      virtual std::string obtenerEspecie() const = 0;

      void mostrarInfo() const {
          std::cout << "[" << obtenerEspecie() << "] "
                    << nombre_ << ", " << edad_ << " años >> ";
          hacerSonido();
      }
  };

  class Perro : public Animal {
  public:
      Perro(const std::string& nombre, int edad) : Animal(nombre, edad) {}
      void hacerSonido() const override { std::cout << "¡Guau guau!" << std::endl; }
      std::string obtenerEspecie() const override { return "Perro"; }
  };

  class Gato : public Animal {
  public:
      Gato(const std::string& nombre, int edad) : Animal(nombre, edad) {}
      void hacerSonido() const override { std::cout << "¡Miau!" << std::endl; }
      std::string obtenerEspecie() const override { return "Gato"; }
  };

  int main() {
      std::cout << "=== CLASES ABSTRACTAS ===" << std::endl;
      Perro perro("Rex", 5);
      Gato gato("Misi", 3);

      perro.mostrarInfo();
      gato.mostrarInfo();

      Animal* animales[] = {&perro, &gato};
      for (auto* a : animales) a->mostrarInfo();
      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 2: TEMPLATE METHOD
  //====================================================================
  #include <iostream>
  #include <string>

  class Juego {
  public:
      virtual ~Juego() = default;
      void jugar() {
          if (!inicializar()) return;
          while (!juegoTerminado()) {
              realizarTurno();
              if (hayGanador()) { anunciarGanador(); break; }
          }
          finalizar();
      }
  protected:
      virtual bool inicializar() { std::cout << "Inicializando...\n"; return true; }
      virtual bool juegoTerminado() const = 0;
      virtual void realizarTurno() = 0;
      virtual bool hayGanador() const = 0;
      virtual void anunciarGanador() const = 0;
      virtual void finalizar() { std::cout << "Finalizado.\n"; }
  };

  class JuegoAdivinanza : public Juego {
      int numeroSecreto_, intentos_, maxIntentos_;
      bool acertado_;
  public:
      JuegoAdivinanza(int max = 5) : maxIntentos_(max), intentos_(0), acertado_(false) {}
  protected:
      bool inicializar() override {
          Juego::inicializar();
          numeroSecreto_ = 42; intentos_ = 0; acertado_ = false;
          std::cout << "Adivina el numero.\n";
          return true;
      }
      bool juegoTerminado() const override { return intentos_ >= maxIntentos_ || acertado_; }
      void realizarTurno() override {
          ++intentos_;
          if (intentos_ == 3) { acertado_ = true; std::cout << "!Acertaste!\n"; }
          else std::cout << "Intento " << intentos_ << " fallido.\n";
      }
      bool hayGanador() const override { return acertado_; }
      void anunciarGanador() const override { std::cout << "!Ganaste!\n"; }
  };

  int main() {
      JuegoAdivinanza juego;
      juego.jugar();
      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 3: ABSTRACT FACTORY
  //====================================================================
  #include <iostream>
  #include <memory>

  class Boton {
  public:
      virtual ~Boton() = default;
      virtual void renderizar() const = 0;
  };

  class Ventana {
  public:
      virtual ~Ventana() = default;
      virtual void mostrar() const = 0;
  };

  class BotonWin : public Boton {
  public:
      void renderizar() const override { std::cout << "Boton Windows\n"; }
  };

  class VentanaWin : public Ventana {
  public:
      void mostrar() const override { std::cout << "Ventana Windows\n"; }
  };

  class BotonMac : public Boton {
  public:
      void renderizar() const override { std::cout << "Boton Mac\n"; }
  };

  class VentanaMac : public Ventana {
  public:
      void mostrar() const override { std::cout << "Ventana Mac\n"; }
  };

  class GUIFactory {
  public:
      virtual ~GUIFactory() = default;
      virtual std::unique_ptr<Boton> crearBoton() = 0;
      virtual std::unique_ptr<Ventana> crearVentana() = 0;
  };

  class WinFactory : public GUIFactory {
  public:
      std::unique_ptr<Boton> crearBoton() override { return std::make_unique<BotonWin>(); }
      std::unique_ptr<Ventana> crearVentana() override { return std::make_unique<VentanaWin>(); }
  };

  class MacFactory : public GUIFactory {
  public:
      std::unique_ptr<Boton> crearBoton() override { return std::make_unique<BotonMac>(); }
      std::unique_ptr<Ventana> crearVentana() override { return std::make_unique<VentanaMac>(); }
  };

  void construirGUI(GUIFactory& f) {
      auto btn = f.crearBoton();
      auto vent = f.crearVentana();
      vent->mostrar();
      btn->renderizar();
  }

  int main() {
      WinFactory wf; MacFactory mf;
      construirGUI(wf); construirGUI(mf);
      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 4: CLASE ABSTRACTA CON ESTADO
  //====================================================================
  #include <iostream>
  #include <string>
  #include <vector>

  class Tarea {
  protected:
      std::string titulo_;
      bool completada_ = false;
      int prioridad_;
  public:
      Tarea(const std::string& t, int p = 5) : titulo_(t), prioridad_(p) {}
      virtual ~Tarea() = default;

      void completar() { completada_ = true; }
      virtual void ejecutar() = 0;
      virtual int tiempoEstimado() const = 0;

      void mostrarInfo() const {
          std::cout << (completada_ ? "[✓]" : "[ ]") << " "
                    << titulo_ << " (" << tiempoEstimado() << " min)\n";
      }
  };

  class TareaSimple : public Tarea {
  public:
      TareaSimple(const std::string& t, int p = 5) : Tarea(t, p) {}
      void ejecutar() override { std::cout << "Ejecutando: " << titulo_ << "\n"; completar(); }
      int tiempoEstimado() const override { return 15; }
  };

  class TareaCompuesta : public Tarea {
      std::vector<Tarea*> subs;
  public:
      TareaCompuesta(const std::string& t, int p = 5) : Tarea(t, p) {}
      void add(Tarea* st) { subs.push_back(st); }
      void ejecutar() override {
          for (auto* s : subs) if (!s->estaCompletada()) s->ejecutar();
          completar();
      }
      int tiempoEstimado() const override {
          int total = 30;
          for (const auto* s : subs) total += s->tiempoEstimado();
          return total;
      }
  };

  int main() {
      TareaSimple t1("Comprar leche", 6);
      TareaCompuesta t2("Preparar informe", 2);
      TareaSimple st1("Recolectar datos");
      TareaSimple st2("Redactar");
      t2.add(&st1); t2.add(&st2);

      t1.mostrarInfo();
      t2.mostrarInfo();
      t2.ejecutar();
      t2.mostrarInfo();
      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 5: VALIDACIÓN CON TEMPLATE METHOD
  //====================================================================
  #include <iostream>
  #include <string>
  #include <cctype>

  class Validador {
  public:
      virtual ~Validador() = default;
      bool validar(const std::string& entrada) {
          if (entrada.empty()) return false;
          if (!preValidar(entrada)) return false;
          bool r = validarImpl(entrada);
          postValidar(entrada, r);
          return r;
      }
  protected:
      virtual bool preValidar(const std::string&) { return true; }
      virtual void postValidar(const std::string& e, bool r) {
          std::cout << "  "" << e << "": " << (r ? "OK" : "FAIL") << "\n";
      }
      virtual bool validarImpl(const std::string&) = 0;
  };

  class ValidadorEmail : public Validador {
  protected:
      bool validarImpl(const std::string& email) override {
          auto arroba = email.find('@');
          if (arroba == std::string::npos || arroba == 0) return false;
          auto punto = email.find('.', arroba + 1);
          return punto != std::string::npos && punto < email.size() - 1;
      }
  };

  class ValidadorPassword : public Validador {
  protected:
      bool preValidar(const std::string& pwd) override {
          if (pwd.size() < 8) { std::cout << "  Demasiado corta.\n"; return false; }
          return true;
      }
      bool validarImpl(const std::string& pwd) override {
          bool may = false, min = false, dig = false;
          for (char c : pwd) {
              if (std::isupper(c)) may = true;
              else if (std::islower(c)) min = true;
              else if (std::isdigit(c)) dig = true;
          }
          return may && min && dig;
      }
  };

  int main() {
      ValidadorEmail ve; ValidadorPassword vp;
      std::cout << (ve.validar("user@example.com") ? "valido" : "invalido") << "\n";
      std::cout << (ve.validar("invalido") ? "valido" : "invalido") << "\n";
      std::cout << (vp.validar("Abc123!.") ? "valida" : "invalida") << "\n";
      std::cout << (vp.validar("corta") ? "valida" : "invalida") << "\n";
      return 0;
  }
  ```

  RESUMEN
================================================================================

  • Una clase abstracta tiene al menos una función virtual pura (= 0).
  • No se puede instanciar directamente; sirve como contrato.
  • Puede tener datos miembro, funciones concretas y constructores.
  • En C++, las interfaces son clases con solo virtuales puras.
  • Template Method define el esqueleto de un algoritmo en la base.
  • Abstract Factory crea familias de objetos relacionados.
  • Programar por contrato: precondiciones, postcondiciones, invariantes.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Puedo instanciar una clase con virtuales puras aunque tengan
     implementación?
  R: No. Basta UNA virtual pura para que sea abstracta.

  P: ¿Una clase abstracta puede tener constructor?
  R: Sí. Los constructores son llamados por las derivadas.

  P: ¿Clase abstracta vs interfaz?
  R: Interfaz = solo virtuales puras. Abstracta = puede tener estado.

  P: ¿Vector de objetos abstractos?
  R: No directamente. Usa vector<Abstracta*> o smart pointers.

  P: ¿Destructor de clase abstracta?
  R: Debe ser virtual siempre.

  P: ¿Si una derivada no implementa todas las puras?
  R: También es abstracta.

  NOTAS ADICIONALES

  Resumen visual de clases abstractas vs interfaces:

    CARACTERÍSTICA         | CLASE ABSTRACTA       | INTERFAZ PURA
    -----------------------|-----------------------|----------------------
    Función virtual pura   | Sí (al menos 1)       | Sí (todas)
    Datos miembro          | Sí (protected/private)| No (solo static)
    Métodos concretos      | Sí                    | No
    Constructores          | Sí (protected)        | No (o default)
    Destructor virtual     | Sí (obligatorio)      | Sí (obligatorio)
    Herencia múltiple      | Posible (riesgosa)    | Recomendada
    Propósito              | Base con estado       | Contrato puro

  Esta tabla te guiará en la decisión de diseño: si necesitas compartir
  estado o implementación, usa clase abstracta. Si solo necesitas
  definir un contrato, usa interfaz.
