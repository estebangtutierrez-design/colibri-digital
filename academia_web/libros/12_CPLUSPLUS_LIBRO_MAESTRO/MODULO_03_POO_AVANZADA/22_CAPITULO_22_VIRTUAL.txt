  ------------------------------------------------------------------
  1. LA PALABRA CLAVE VIRTUAL
  ------------------------------------------------------------------

  La palabra clave 'virtual' en C++ es un especificador que se aplica a
  funciones miembro para habilitar el enlace dinámico (dynamic dispatch).

  Cuando declaras una función como virtual:

    class Base {
    public:
        virtual void mostrar() const;
    };

  El compilador genera código para que la llamada a 'mostrar()' se resuelva
  en tiempo de ejecución basándose en el tipo real del objeto, no en el tipo
  del puntero o referencia.

  Una vez que una función se declara virtual en una clase base, permanece
  virtual en todas las clases derivadas, incluso si no se repite la palabra
  clave 'virtual'. Sin embargo, es buena práctica repetirla (o usar
  'override') por claridad.

  ------------------------------------------------------------------
  2. FUNCIONES VIRTUALES PURAS
  ------------------------------------------------------------------

  Una función virtual pura es una función virtual que no tiene
  implementación en la clase base. Se declara asignándole = 0:

    class Interface {
    public:
        virtual void metodo() = 0;  // Virtual pura
    };

  Características:
  • La clase que contiene al menos una función virtual pura se vuelve
    abstracta (no se puede instanciar directamente).
  • Las clases derivadas DEBEN implementar todas las funciones virtuales
    puras para poder ser instanciadas.
  • Las funciones virtuales puras pueden tener implementación en la clase
    base, aunque es poco común. Se invocan con Base::metodo().

  ------------------------------------------------------------------
  3. DESTRUCTORES VIRTUALES
  ------------------------------------------------------------------

  Regla fundamental: Si una clase tiene al menos una función virtual, su
  destructor debe ser virtual.

  ¿Por qué? Porque cuando eliminas un objeto derivado a través de un puntero
  a la clase base, el compilador debe saber qué destructor llamar. Sin un
  destructor virtual, solo se llamará al destructor de la clase base,
  causando potenciales fugas de memoria.

  Sintaxis correcta:

    class Base {
    public:
        virtual ~Base() = default;  // C++11: defaulted virtual destructor
        virtual void hacerAlgo() = 0;
    };

  ------------------------------------------------------------------
  4. COMPORTAMIENTO DE VIRTUAL EN CONSTRUCTORES
  ------------------------------------------------------------------

  Durante la construcción de un objeto, el comportamiento de las funciones
  virtuales cambia:

  • Cuando se ejecuta el constructor de la clase base, el vptr apunta a la
    vtable de la base, NO de la derivada.
  • Si el constructor de la base llama a una función virtual, se ejecuta la
    versión de la base, aunque la derivada la haya sobreescrito.
  • Lo mismo ocurre en sentido inverso durante la destrucción.

  Esto significa que NO debes llamar a funciones virtuales desde
  constructores o destructores si esperas comportamiento polimórfico.

  ------------------------------------------------------------------
  5. HERENCIA VIRTUAL
  ------------------------------------------------------------------

  La herencia virtual resuelve el problema del diamante, donde una clase
  derivada hereda de dos clases que comparten una misma clase base común.

  Sin herencia virtual, la clase base común se replica dos veces en la
  clase derivada final, causando ambigüedad.

  Con herencia virtual, la clase base común se comparte, eliminando la
  ambigüedad y el desperdicio de memoria.

  Sintaxis:

    class A { ... };
    class B : virtual public A { ... };
    class C : virtual public A { ... };
    class D : public B, public C { ... };  // Una sola copia de A

  ------------------------------------------------------------------
  6. LA TABLA VIRTUAL EN HERENCIA MÚLTIPLE
  ------------------------------------------------------------------

  Cuando se usa herencia múltiple, el objeto puede tener múltiples vptr,
  uno por cada base que tenga funciones virtuales. Además, pueden existir
  ajustes (thunks) que modifican el puntero this para que apunte a la
  subclase correcta antes de llamar a la función.

  El compilador gestiona automáticamente estas complejidades, pero es
  importante entender que la herencia múltiple con funciones virtuales
  tiene un costo adicional en memoria y rendimiento.

  ------------------------------------------------------------------
  7. VIRTUAL INLINE
  ------------------------------------------------------------------

  Las funciones virtuales no pueden ser inline en el sentido tradicional
  porque su resolución es dinámica. Sin embargo, el compilador puede
  inlinearlas cuando conoce el tipo exacto en tiempo de compilación
  (por ejemplo, cuando se llama directamente sobre un objeto, no a través
  de un puntero).

  ------------------------------------------------------------------
  8. VIRTUAL Y ESTÁTICO
  ------------------------------------------------------------------

  Una función no puede ser simultáneamente virtual y estática. Las
  funciones estáticas pertenecen a la clase, no a los objetos, y no
  participan en el enlace dinámico.

  ```cpp
  //====================================================================
  //  EJEMPLO 1: FUNCIÓN VIRTUAL VS FUNCIÓN VIRTUAL PURA
  //====================================================================
  #include <iostream>
  #include <string>

  class Animal {
  public:
      virtual ~Animal() = default;

      // Función virtual (tiene implementación)
      virtual void saludar() const {
          std::cout << "Animal saluda genéricamente." << std::endl;
      }

      // Función virtual pura (NO tiene implementación en la base)
      virtual void hacerSonido() const = 0;

      // Función virtual pura CON implementación (caso especial)
      virtual void dormir() const = 0;
  };

  // Implementación de la virtual pura (raro, pero posible)
  void Animal::dormir() const {
      std::cout << "Zzz... (sueño genérico de animal)" << std::endl;
  }

  class Perro : public Animal {
  public:
      void hacerSonido() const override {
          std::cout << "¡Guau guau!" << std::endl;
      }

      void dormir() const override {
          // Llamamos a la implementación base
          Animal::dormir();
          std::cout << "(El perro sueña con huesos)" << std::endl;
      }

      // Opcional: sobreescribir saludar
      void saludar() const override {
          std::cout << "Perro mueve la cola." << std::endl;
      }
  };

  class Gato : public Animal {
  public:
      void hacerSonido() const override {
          std::cout << "¡Miau!" << std::endl;
      }

      void dormir() const override {
          Animal::dormir();
          std::cout << "(El gato sueña con ratones)" << std::endl;
      }
      // NO sobreescribe saludar() -> se usa la versión de Animal
  };

  int main() {
      // Animal a;  // ERROR: no se puede instanciar clase abstracta

      Perro perro;
      Gato gato;

      std::cout << "=== FUNCIONES VIRTUALES ===" << std::endl;

      Animal* ptr = &perro;
      ptr->saludar();      // Virtual con implementación base
      ptr->hacerSonido();  // Virtual pura implementada en Perro
      ptr->dormir();       // Virtual pura con llamada a base

      std::cout << "---" << std::endl;

      ptr = &gato;
      ptr->saludar();      // NO override -> usa Animal::saludar()
      ptr->hacerSonido();  // Gato::hacerSonido()
      ptr->dormir();       // Gato::dormir() que llama a Animal::dormir()

      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 2: DESTRUCTORES VIRTUALES EN ACCIÓN
  //====================================================================
  #include <iostream>
  #include <vector>

  class Recurso {
  public:
      Recurso(int id) : id_(id) {
          std::cout << "  [+] Recurso " << id_ << " adquirido" << std::endl;
      }

      virtual ~Recurso() {
          std::cout << "  [-] Recurso " << id_ << " liberado" << std::endl;
      }

      virtual void usar() const {
          std::cout << "  Usando Recurso " << id_ << std::endl;
      }

  private:
      int id_;
  };

  class RecursoRed : public Recurso {
  public:
      RecursoRed(int id, const std::string& url)
          : Recurso(id), url_(new std::string(url)) {
          std::cout << "  [+] Conexión de red a " << *url_ << std::endl;
      }

      ~RecursoRed() override {
          std::cout << "  [-] Cerrando conexión a " << *url_ << std::endl;
          delete url_;
      }

      void usar() const override {
          std::cout << "  Transfiriendo datos desde " << *url_ << std::endl;
      }

  private:
      std::string* url_;
  };

  class RecursoArchivo : public Recurso {
  public:
      RecursoArchivo(int id, const std::string& path)
          : Recurso(id), path_(new std::string(path)) {
          std::cout << "  [+] Archivo abierto: " << *path_ << std::endl;
      }

      ~RecursoArchivo() override {
          std::cout << "  [-] Cerrando archivo: " << *path_ << std::endl;
          delete path_;
      }

      void usar() const override {
          std::cout << "  Leyendo archivo " << *path_ << std::endl;
      }

  private:
      std::string* path_;
  };

  class GestorRecursos {
  public:
      void agregar(Recurso* r) {
          recursos_.push_back(r);
      }

      void usarTodos() const {
          for (auto* r : recursos_) {
              r->usar();
          }
      }

      ~GestorRecursos() {
          std::cout << "GestorRecursos liberando recursos:" << std::endl;
          for (auto* r : recursos_) {
              delete r;  // Gracias a destructor virtual, todo ok
          }
      }

  private:
      std::vector<Recurso*> recursos_;
  };

  int main() {
      std::cout << "=== DESTRUCTORES VIRTUALES ===" << std::endl;
      {
          GestorRecursos gestor;
          gestor.agregar(new RecursoRed(1, "https://api.example.com"));
          gestor.agregar(new RecursoArchivo(2, "/datos/informe.pdf"));
          gestor.agregar(new Recurso(3));

          std::cout << "\nUsando todos los recursos:" << std::endl;
          gestor.usarTodos();
          std::cout << "\nSaliendo del ámbito..." << std::endl;
      }
      std::cout << "Todos los recursos liberados correctamente." << std::endl;

      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 3: PROBLEMA DEL DIAMANTE Y HERENCIA VIRTUAL
  //====================================================================
  #include <iostream>
  #include <string>

  // =========================================================
  //  SIN herencia virtual — PROBLEMA DEL DIAMANTE
  // =========================================================
  namespace SinVirtual {
      class Animal {
      public:
          Animal(const std::string& nombre) : nombre_(nombre) {
              std::cout << "  Animal(" << nombre_ << ") construido" << std::endl;
          }
          virtual ~Animal() = default;
          virtual void hablar() const {
              std::cout << nombre_ << " dice: ..." << std::endl;
          }
          std::string nombre_;
      };

      class Caballo : public Animal {
      public:
          Caballo(const std::string& nombre) : Animal(nombre) {
              std::cout << "  Caballo(" << nombre << ") construido" << std::endl;
          }
          void hablar() const override {
              std::cout << nombre_ << " relincha: ¡Hiiii!" << std::endl;
          }
      };

      class Delfin : public Animal {
      public:
          Delfin(const std::string& nombre) : Animal(nombre) {
              std::cout << "  Delfin(" << nombre << ") construido" << std::endl;
          }
          void hablar() const override {
              std::cout << nombre_ << " chilla: ¡Eeee!" << std::endl;
          }
      };

      // CaballoDelfin hereda de Caballo Y Delfin (dos copias de Animal)
      class CaballoDelfin : public Caballo, public Delfin {
      public:
          CaballoDelfin(const std::string& nomCaballo,
                        const std::string& nomDelfin)
              : Animal("ANIMAL_BASE_?"),  // ERROR: ¿cuál Animal?
                Caballo(nomCaballo),
                Delfin(nomDelfin) {
              std::cout << "  CaballoDelfin construido (¡PROBLEMA!)" << std::endl;
          }
          // Ambigüedad: ¿hablar() de Caballo o de Delfin?
      };
  }

  // =========================================================
  //  CON herencia virtual — SOLUCIÓN
  // =========================================================
  namespace ConVirtual {
      class Animal {
      public:
          Animal(const std::string& nombre) : nombre_(nombre) {
              std::cout << "  [V] Animal(" << nombre_ << ") construido" << std::endl;
          }
          virtual ~Animal() = default;
          virtual void hablar() const {
              std::cout << nombre_ << " dice: ..." << std::endl;
          }
          std::string nombre_;
      };

      // Herencia virtual: Caballo y Delfin comparten la misma instancia de Animal
      class Caballo : virtual public Animal {
      public:
          Caballo(const std::string& nombre) : Animal(nombre) {
              std::cout << "  [V] Caballo(" << nombre << ") construido" << std::endl;
          }
          void hablar() const override {
              std::cout << nombre_ << " relincha: ¡Hiiii!" << std::endl;
          }
      };

      class Delfin : virtual public Animal {
      public:
          Delfin(const std::string& nombre) : Animal(nombre) {
              std::cout << "  [V] Delfin(" << nombre << ") construido" << std::endl;
          }
          void hablar() const override {
              std::cout << nombre_ << " chilla: ¡Eeee!" << std::endl;
          }
      };

      // CaballoDelfin con herencia virtual: una sola copia de Animal
      class CaballoDelfin : public Caballo, public Delfin {
      public:
          // AHORA podemos (y debemos) inicializar Animal directamente
          CaballoDelfin(const std::string& nombre)
              : Animal(nombre),      // Clase virtual base inicializada aquí
                Caballo(nombre),
                Delfin(nombre) {
              std::cout << "  [V] CaballoDelfin(" << nombre
                        << ") construido" << std::endl;
          }

          // Debemos resolver la ambigüedad de hablar()
          void hablar() const override {
              std::cout << nombre_
                        << " es un CaballoDelfin: ¡Hiiii-Eeee!" << std::endl;
          }
      };
  }

  int main() {
      std::cout << "=== SIN HERENCIA VIRTUAL (PROBLEMA) ===" << std::endl;
      // La siguiente línea NO compilaría por la ambigüedad
      // SinVirtual::CaballoDelfin cd("Pegaso", "Flash");
      std::cout << "Ni siquiera compila debido a la ambigüedad." << std::endl;

      std::cout << "\n=== CON HERENCIA VIRTUAL (SOLUCIÓN) ===" << std::endl;
      ConVirtual::CaballoDelfin cd("AquaPegaso");

      std::cout << "\nComportamiento:" << std::endl;
      ConVirtual::Animal* a1 = &cd;
      ConVirtual::Caballo* c1 = &cd;
      ConVirtual::Delfin* d1 = &cd;

      a1->hablar();  // CaballoDelfin::hablar()
      c1->hablar();  // misma función
      d1->hablar();  // misma función

      std::cout << "\nINFORMACIÓN DE MEMORIA:" << std::endl;
      std::cout << "Dirección de Animal: " << &cd << std::endl;
      std::cout << "Todos los punteros apuntan al MISMO objeto Animal."
                << std::endl;

      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 4: VIRTUAL EN CONSTRUCTORES Y DESTRUCTORES
  //====================================================================
  #include <iostream>

  class Base {
  public:
      Base() {
          std::cout << "Constructor de Base - vptr apunta a vtable de Base\n";
          llamarVirtual();  // PELIGRO: llama a Base::mostrar(), no a Derivada
      }

      virtual ~Base() {
          std::cout << "Destructor de Base - vptr apunta a vtable de Base\n";
          llamarVirtual();  // PELIGRO: llama a Base::mostrar()
      }

      virtual void mostrar() const {
          std::cout << "  Base::mostrar()" << std::endl;
      }

      void llamarVirtual() const {
          mostrar();  // Llamada virtual (peligrosa en ctor/dtor)
      }
  };

  class Derivada : public Base {
  public:
      Derivada() {
          std::cout << "Constructor de Derivada - vptr cambia a vtable de Derivada\n";
      }

      ~Derivada() override {
          std::cout << "Destructor de Derivada - vptr aún apunta a vtable de Derivada\n";
      }

      void mostrar() const override {
          std::cout << "  Derivada::mostrar()" << std::endl;
      }
  };

  int main() {
      std::cout << "=== VIRTUAL EN CONSTRUCTORES/DESTRUCTORES ===" << std::endl;
      std::cout << "\nCreando objeto Derivado:" << std::endl;
      Derivada d;

      std::cout << "\nLlamada normal (fuera de ctor/dtor):" << std::endl;
      Base& ref = d;
      ref.mostrar();  // Derivada::mostrar() - correcto

      std::cout << "\nDestruyendo objeto:" << std::endl;
      // (la destrucción ocurre al salir del ámbito)

      std::cout << "\nConclusión: NO llamar a funciones virtuales desde";
      std::cout << " constructores o destructores." << std::endl;

      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 5: VIRTUAL CON HERENCIA MÚLTIPLE Y TABLAS VIRTUALES
  //====================================================================
  #include <iostream>

  // Interfaces con funciones virtuales
  class IImprimible {
  public:
      virtual ~IImprimible() = default;
      virtual void imprimir() const = 0;
      virtual void obtenerTamanio() const {
          std::cout << "  Tamaño genérico" << std::endl;
      }
  };

  class IEscaneable {
  public:
      virtual ~IEscaneable() = default;
      virtual void escanear() const = 0;
      virtual void obtenerResolucion() const {
          std::cout << "  Resolución genérica" << std::endl;
      }
  };

  class IFax {
  public:
      virtual ~IFax() = default;
      virtual void enviarFax() const = 0;
      virtual void recibirFax() const = 0;
  };

  // Clase concreta que implementa múltiples interfaces
  class ImpresoraMultifuncional : public IImprimible,
                                  public IEscaneable,
                                  public IFax {
  public:
      void imprimir() const override {
          std::cout << "  Imprimiendo documento..." << std::endl;
      }

      void escanear() const override {
          std::cout << "  Escaneando documento..." << std::endl;
      }

      void enviarFax() const override {
          std::cout << "  Enviando fax..." << std::endl;
      }

      void recibirFax() const override {
          std::cout << "  Fax recibido." << std::endl;
      }

      void obtenerTamanio() const override {
          std::cout << "  Tamaño: A4, Carta, Legal" << std::endl;
      }

      void obtenerResolucion() const override {
          std::cout << "  Resolución: 1200 DPI" << std::endl;
      }

      // Función exclusiva de esta clase
      void hacerCopia() const {
          std::cout << "  Copiando documento (imprimir + escanear)..." << std::endl;
      }
  };

  int main() {
      std::cout << "=== HERENCIA MÚLTIPLE CON VIRTUAL ===" << std::endl;

      ImpresoraMultifuncional hp;

      // Usando diferentes interfaces
      IImprimible* pImprimir = &hp;
      IEscaneable* pEscaneo = &hp;
      IFax* pFax = &hp;

      std::cout << "\nComo IImprimible:" << std::endl;
      pImprimir->imprimir();
      pImprimir->obtenerTamanio();

      std::cout << "\nComo IEscaneable:" << std::endl;
      pEscaneo->escanear();
      pEscaneo->obtenerResolucion();

      std::cout << "\nComo IFax:" << std::endl;
      pFax->enviarFax();
      pFax->recibirFax();

      // Los punteros pueden tener diferentes direcciones
      std::cout << "\nPUNTEROS (pueden diferir debido a ajuste de vtable):"
                << std::endl;
      std::cout << "  IImprimible*:  " << pImprimir << std::endl;
      std::cout << "  IEscaneable*:  " << pEscaneo << std::endl;
      std::cout << "  IFax*:         " << pFax << std::endl;
      std::cout << "  Impresora*:    " << &hp << std::endl;

      // dynamic_cast puede convertir entre interfaces
      if (auto* copiadora = dynamic_cast<ImpresoraMultifuncional*>(pFax)) {
          std::cout << "\n dynamic_cast exitoso: pFax → ImpresoraMultifuncional*"
                    << std::endl;
          copiadora->hacerCopia();
      }

      return 0;
  }
  ```

  ```cpp
  //====================================================================
  //  EJEMPLO 6: virtual = 0 CON IMPLEMENTACIÓN (USO AVANZADO)
  //====================================================================
  #include <iostream>
  #include <string>
  #include <vector>

  class Empleado {
  protected:
      std::string nombre_;
      int id_;
      double salarioBase_;
  public:
      Empleado(const std::string& nom, int id, double sal)
          : nombre_(nom), id_(id), salarioBase_(sal) {}

      virtual ~Empleado() = default;

      // Virtual pura SIN implementación (la derivada debe implementarla)
      virtual double calcularBono() const = 0;

      // Virtual pura CON implementación (la derivada puede usarla o no)
      virtual std::string obtenerDescripcion() const = 0;

      // Función no virtual
      void mostrarInfo() const {
          std::cout << "ID: " << id_ << " | "
                    << nombre_ << " | "
                    << "Salario Base: $" << salarioBase_ << " | "
                    << "Bono: $" << calcularBono() << " | "
                    << "Total: $" << (salarioBase_ + calcularBono())
                    << std::endl;
      }

      int obtenerId() const { return id_; }
  };

  // Implementación de la virtual pura obtenerDescripcion()
  // (sirve como base para que las derivadas la llamen)
  std::string Empleado::obtenerDescripcion() const {
      return "Empleado #" + std::to_string(id_) + ": " + nombre_;
  }

  class Ingeniero : public Empleado {
      int proyectos_;
  public:
      Ingeniero(const std::string& nom, int id, double sal, int proy)
          : Empleado(nom, id, sal), proyectos_(proy) {}

      double calcularBono() const override {
          return salarioBase_ * 0.10 + proyectos_ * 500;
      }

      std::string obtenerDescripcion() const override {
          return Empleado::obtenerDescripcion() + " [Ingeniero, "
               + std::to_string(proyectos_) + " proyectos]";
      }
  };

  class Vendedor : public Empleado {
      double ventas_;
  public:
      Vendedor(const std::string& nom, int id, double sal, double ventas)
          : Empleado(nom, id, sal), ventas_(ventas) {}

      double calcularBono() const override {
          return ventas_ * 0.05;  // 5% de comisión
      }

      std::string obtenerDescripcion() const override {
          return Empleado::obtenerDescripcion() + " [Vendedor, ventas: $"
               + std::to_string(ventas_) + "]";
      }
  };

  class Gerente : public Empleado {
      double bonoFijo_;
  public:
      Gerente(const std::string& nom, int id, double sal, double bonoFijo)
          : Empleado(nom, id, sal), bonoFijo_(bonoFijo) {}

      double calcularBono() const override {
          return bonoFijo_ + salarioBase_ * 0.20;
      }

      std::string obtenerDescripcion() const override {
          return Empleado::obtenerDescripcion() + " [Gerente]";
      }
  };

  class Nomina {
      std::vector<Empleado*> empleados_;
  public:
      void agregar(Empleado* e) { empleados_.push_back(e); }

      void generarReporte() const {
          std::cout << "\n=== REPORTE DE NÓMINA ===" << std::endl;
          double total = 0;
          for (const auto* e : empleados_) {
              std::cout << e->obtenerDescripcion() << std::endl;
              e->mostrarInfo();
              total += salarioBase_ + e->calcularBono();
              // Nota: salarioBase_ es protected, no accesible desde aquí
              // En un caso real agregaríamos un getter
              std::cout << "---" << std::endl;
          }
      }

      void limpiar() {
          for (auto* e : empleados_) delete e;
          empleados_.clear();
      }
  };

  int main() {
      std::cout << "=== VIRTUAL PURA CON IMPLEMENTACIÓN ===" << std::endl;

      Nomina nomina;
      nomina.agregar(new Ingeniero("Ana López", 101, 5000, 3));
      nomina.agregar(new Vendedor("Carlos Ruiz", 102, 3000, 25000));
      nomina.agregar(new Gerente("Laura Díaz", 103, 8000, 2000));

      // Reporte
      for (int id : {101, 102, 103}) {
          // Podríamos buscar empleado por id, etc.
          std::cout << "Procesando empleado ID " << id << "..." << std::endl;
      }

      nomina.generarReporte();
      nomina.limpiar();

      return 0;
  }
  ```

================================================================================
  RESUMEN
================================================================================

  • 'virtual' es el especificador que habilita el enlace dinámico.
  • Una función virtual pura (= 0) convierte la clase en abstracta.
  • Los destructores virtuales son obligatorios en clases base polimórficas.
  • No llamar a funciones virtuales desde constructores/destructores; el
    vptr aún no apunta a la vtable de la clase derivada.
  • La herencia virtual (virtual public Base) resuelve el problema del
    diamante compartiendo una única instancia de la clase base.
  • En herencia múltiple, un objeto puede tener múltiples vptr y el
    compilador ajusta los punteros con thunks.
  • Una función virtual pura puede tener implementación en la clase base,
    aunque la derivada debe sobreescribirla para ser concreta.
  • 'override' no es obligatorio, pero verifica la firma y evita errores.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Puedo tener una función virtual estática?
  R: No. static y virtual son mutuamente excluyentes. Las funciones
     estáticas pertenecen a la clase, no a los objetos, y no participan
     en el enlace dinámico.

  P: ¿Cuándo debo usar herencia virtual?
  R: Cuando existe un "diamante" en la jerarquía de herencia y quieres
     una sola copia de la clase base compartida. Sin herencia virtual,
     la base se replica.

  P: ¿Es obligatorio implementar todas las virtuales puras?
  R: Sí, para que una clase derivada sea concreta (instanciable). Si no
     implementa alguna, la derivada también es abstracta.

  P: ¿Puedo llamar a la implementación base de una virtual desde la
     derivada?
  R: Sí, con el operador de ámbito: Base::funcion(). Esto es útil para
     extender el comportamiento base, no reemplazarlo completamente.

  P: ¿El destructor virtual aumenta el tamaño del objeto?
  R: Sí, añade un vptr (8 bytes en 64-bit). Pero el costo es mínimo
     comparado con los beneficios de seguridad.

  P: ¿Qué es 'final' en C++11?
  R: 'final' impide que una clase sea base de otras, o que una función
     virtual sea sobreescrita en clases derivadas.

  P: ¿Cómo sé si mi clase tiene vtable?
  R: Si tiene al menos una función virtual (incluyendo destructor virtual),
     el compilador generará una vtable para ella.
