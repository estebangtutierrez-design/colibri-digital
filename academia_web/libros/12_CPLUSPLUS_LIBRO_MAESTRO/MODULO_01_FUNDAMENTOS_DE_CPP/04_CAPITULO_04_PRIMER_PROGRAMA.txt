2.1. COMENTARIOS

```cpp
// Primer programa en C++  ← El compilador ignora esto
// Muestra "¡Hola, Mundo!" ← Son para nosotros, los humanos
```

Los comentarios son texto que el compilador ignora completamente.
Sirven para documentar el código. Hay dos tipos:

```cpp
// Comentario de una línea (desde // hasta el final de línea)

/*
   Comentario de múltiples líneas
   Todo lo que está aquí dentro es ignorado

```

2.2. DIRECTIVA #include

```cpp
#include <iostream>
```

`#include` es una directiva de preprocesador. Le dice al
preprocesador que incluya el contenido del archivo <iostream>
antes de la compilación.

<iostream> (Input/Output Stream) es una biblioteca estándar
que proporciona las herramientas para entrada y salida de datos:
- `std::cout` para imprimir en pantalla.
- `std::cin` para leer del teclado.
- `std::cerr` para mensajes de error.
- `std::endl` para salto de línea.

El preprocesador expande esta directiva reemplazándola con el
contenido real del archivo de cabecera. Puedes ver el resultado
usando:

```bash
g++ -E hola_mundo.cpp -o hola_mundo.i
```

```cpp
// Otras cabeceras comunes que veremos:
#include <string>    // Para std::string
#include <vector>    // Para std::vector
#include <algorithm> // Para algoritmos STL
#include <cmath>     // Para funciones matemáticas
```

2.3. LA FUNCIÓN main

```cpp
int main() {
    // ...
    return 0;
}
```

`main` es la función principal de todo programa C++. Es el
punto de entrada: cuando ejecutas el programa, el sistema
operativo llama a `main`.

Características de main:
- **Debe existir exactamente una** en cada programa.
- **Devuelve un int**: 0 indica éxito, otro valor indica error.
- **Puede tener parámetros**: para recibir argumentos de línea
  de comandos (ver más adelante).
- **El cuerpo va entre llaves {}**: contiene las instrucciones.

```cpp
// Las dos formas estándar de main:

// Forma 1: Sin argumentos (la que usaremos ahora)
int main() {
    // código
    return 0;
}

// Forma 2: Con argumentos de línea de comandos
int main(int argc, char* argv[]) {
    // argc = número de argumentos
    // argv = arreglo de strings con los argumentos
    return 0;
}
```

2.4. cout Y EL OPERADOR DE INSERCIÓN <<

```cpp
std::cout << "¡Hola, Mundo!" << std::endl;
```

- **std::cout**: Objeto de flujo de salida estándar (consola).
- **<<**: Operador de inserción. Envía datos al flujo de salida.
- **"¡Hola, Mundo!"**: Literal de cadena (string).
- **std::endl**: Manipulador que inserta un salto de línea y
  vacía el buffer (flush).

Puedes encadenar múltiples inserciones:

```cpp
std::cout << "Hola" << " " << "Mundo" << "!" << std::endl;
std::cout << "La respuesta es " << 42 << std::endl;
```

2.5. LA INSTRUCCIÓN return

```cpp
return 0;
```

`return` finaliza la función `main` y devuelve un valor al
sistema operativo. Por convención:
- **0**: Ejecución exitosa.
- **Distinto de 0**: Error (el valor indica el tipo de error).

```cpp
// Formas equivalentes de indicar éxito:
return 0;
return EXIT_SUCCESS;  // Necesita #include <cstdlib>

// Indicar error:
return 1;
return EXIT_FAILURE;  // Necesita #include <cstdlib>
```

Puedes omitir `return 0` en main (desde C++99). El compilador
asume automáticamente `return 0` al final:

```cpp
int main() {
    std::cout << "Sin return explícito" << std::endl;
    // return 0; // Implícito
}
```

===============================================================================
3. USO DE using namespace std
===============================================================================

Escribir `std::` antes de cada elemento de la biblioteca
estándar puede ser tedioso. Podemos evitarlo con:

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "¡Sin std::!" << endl;
    return 0;
}
```

Sin embargo, esta práctica tiene controversia:

**¿Cuándo usarla?**
- En archivos .cpp pequeños.
- En código de prueba o aprendizaje.
- En funciones específicas (con alcance limitado).

**¿Cuándo evitarla?**
- En archivos de cabecera (.h, .hpp) - NUNCA.
- En proyectos grandes donde puede haber colisiones.
- En código profesional.

Alternativa recomendada: usar solo lo que necesitas.

```cpp
using std::cout;
using std::endl;
using std::string;
// Solo importa cout, endl y string
```

A lo largo de este curso usaremos `std::` explícitamente para
claridad, excepto cuando queramos simplificar ejemplos. En tu
código personal, elige el estilo que prefieras.

5. ENTRADA BÁSICA CON cin
===============================================================================

`cin` (character input) es el objeto de flujo de entrada estándar.

```cpp
#include <iostream>

int main() {
    int edad;
    double peso;
    std::string nombre;

    std::cout << "Ingresa tu nombre: ";
    std::cin >> nombre;

    std::cout << "Ingresa tu edad: ";
    std::cin >> edad;

    std::cout << "Ingresa tu peso: ";
    std::cin >> peso;

    std::cout << "\n--- DATOS ---" << std::endl;
    std::cout << "Nombre: " << nombre << std::endl;
    std::cout << "Edad: " << edad << std::endl;
    std::cout << "Peso: " << peso << std::endl;

    return 0;
}
```

5.1. PROBLEMA CON cin Y ESPACIOS

`cin >>` se detiene al encontrar un espacio, tabulador o nueva
línea. Para leer líneas completas usa `getline`:

```cpp
#include <iostream>
#include <string>

int main() {
    std::string nombreCompleto;

    std::cout << "Ingresa tu nombre completo: ";

    // Importante: ignorar el salto de línea pendiente
    std::cin.ignore();

    std::getline(std::cin, nombreCompleto);

    std::cout << "Hola, " << nombreCompleto << "!" << std::endl;

    return 0;
}
```

5.2. LIMPIEZA DEL BUFFER DE ENTRADA

Cuando mezclas `cin >>` con `getline`:

```cpp
#include <iostream>
#include <string>
#include <limits>

int main() {
    int edad;
    std::string nombre;

    std::cout << "Edad: ";
    std::cin >> edad;

    // Limpiar el buffer de entrada
    std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');

    std::cout << "Nombre: ";
    std::getline(std::cin, nombre);

    std::cout << edad << " - " << nombre << std::endl;

    return 0;
}
```

8. CICLO BÁSICO DE TRABAJO
===============================================================================

El ciclo que repetirás constantemente:

```
1. EDITAR → 2. COMPILAR → 3. EJECUTAR → 4. DEPURAR → (volver a 1)
```

8.1. FLUJO COMPLETO

```bash
# 1. Crear/editar el archivo
code programa.cpp

# 2. Compilar
g++ -std=c++20 -Wall programa.cpp -o programa

# Si hay errores → volver a editar
# Si no hay errores → continuar

# 3. Ejecutar
./programa

# 4. Si el resultado no es el esperado → depurar
g++ -g programa.cpp -o programa
gdb ./programa
```

8.2. ATAJOS ÚTILES

```bash
# Compilar y ejecutar en un solo paso (Linux/macOS):
g++ programa.cpp -o programa && ./programa

# Compilar con warnings y ejecutar:
g++ -std=c++20 -Wall -Wextra programa.cpp -o programa && ./programa

# Historial de comandos (terminal):
# ↑ flecha arriba: recuperar comando anterior
```

```cpp
#include <iostream>

int main() {
    double a, b;

    std::cout << "=== CALCULADORA SIMPLE ===" << std::endl;
    std::cout << "Ingresa el primer número: ";
    std::cin >> a;
    std::cout << "Ingresa el segundo número: ";

    std::cout << a << " + " << b << " = " << (a + b) << std::endl;

    } else {
        std::cout << "División por cero no permitida" << std::endl;
    }

}
```

9.3. JUEGO DE ADIVINANZA

```cpp
#include <iostream>
#include <cstdlib>  // Para rand() y srand()
#include <ctime>    // Para time()

int main() {
    // Inicializar generador de números aleatorios
    std::srand(std::time(0));
    int secreto = std::rand() % 100 + 1;
    int intento;
    int intentos = 0;

    std::cout << "=== ADIVINA EL NÚMERO ===" << std::endl;
    std::cout << "Estoy pensando en un número entre 1 y 100."
              << std::endl;

    do {
        std::cout << "Tu intento: ";
        std::cin >> intento;
        intentos++;

        if (intento < secreto) {
            std::cout << "Muy bajo!" << std::endl;
        } else if (intento > secreto) {
            std::cout << "Muy alto!" << std::endl;
        } else {
            std::cout << "¡Correcto! Lo lograste en "
                      << intentos << " intentos." << std::endl;
        }
    } while (intento != secreto);

    return 0;
}
```

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué main devuelve int y no void?** El estándar de C++
define que main debe devolver int para comunicar el estado de
salida al sistema operativo. Algunos compiladores aceptan
void main, pero no es estándar y no debe usarse.

❓ **¿Qué pasa si no pongo return 0 en main?** El compilador
asume automáticamente return 0 al final de main. Es una
excepción especial de C++ solo para main.

❓ **¿Por qué se llama "iostream"?** Es la contracción de
Input/Output Stream (flujo de entrada/salida). El nombre
describe exactamente su propósito.

❓ **¿Puedo tener múltiples funciones main en un proyecto?**
No. Cada programa debe tener exactamente una función main.
Si tienes varios .cpp con main, obtendrás un error de
"multiple definition of main".

❓ **¿Qué significa el error 'undeclared identifier'?**
Significa que estás usando un nombre (variable, función, tipo)
que el compilador no conoce. Posibles causas: falta de #include,
error ortográfico, o falta de declaración.
