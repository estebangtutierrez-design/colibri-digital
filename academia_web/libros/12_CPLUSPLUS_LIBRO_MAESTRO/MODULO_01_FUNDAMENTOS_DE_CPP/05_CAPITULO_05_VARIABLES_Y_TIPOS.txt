1. TIPOS DE DATOS PRIMITIVOS
===============================================================================

Los tipos primitivos son los bloques fundamentales con los que
construimos datos más complejos.

1.1. TIPOS ENTEROS

Representan números sin parte fraccionaria.

| TIPO            | TAMAÑO   | RANGO                               |
|-----------------|----------|-------------------------------------|
| short           | 2 bytes  | -32,768 a 32,767                    |
| int             | 4 bytes  | -2,147,483,648 a 2,147,483,647      |
| long            | 4 u 8    | Depende de la plataforma            |
| long long       | 8 bytes  | -9e18 a 9e18 aprox.                 |
| unsigned short  | 2 bytes  | 0 a 65,535                          |
| unsigned int    | 4 bytes  | 0 a 4,294,967,295                   |
| unsigned long   | 4 u 8    | 0 a ...                             |
| unsigned long long | 8 bytes| 0 a 1.8e19 aprox.                   |

```cpp
#include <iostream>

int main() {
    short s = 32000;
    int i = -2000000;
    long l = 100000L;
    long long ll = 10000000000LL;

    unsigned int u = 4000000000U;

    std::cout << "short: " << s << std::endl;
    std::cout << "int: " << i << std::endl;
    std::cout << "long: " << l << std::endl;
    std::cout << "long long: " << ll << std::endl;
    std::cout << "unsigned: " << u << std::endl;

    return 0;
}
```

1.2. TIPOS DE PUNTO FLOTANTE

Representan números con parte fraccionaria.

| TIPO      | TAMAÑO   | PRECISIÓN          |
|-----------|----------|--------------------|
| float     | 4 bytes  | ~7 dígitos decimales |
| double    | 8 bytes  | ~15 dígitos decimales |
| long double | 8-16    | ~19+ dígitos decimales |

```cpp
#include <iostream>
#include <iomanip>

int main() {
    float f = 3.14159265358979323846f;
    double d = 3.14159265358979323846;
    long double ld = 3.14159265358979323846L;

    std::cout << std::setprecision(20);
    std::cout << "float:       " << f << std::endl;
    std::cout << "double:      " << d << std::endl;
    std::cout << "long double: " << ld << std::endl;

    return 0;
}
```

1.3. TIPO BOOLEANO

Representa valores de verdad: true o false.

```cpp
#include <iostream>

int main() {
    bool verdad = true;
    bool falso = false;

    std::cout << "true:  " << verdad << std::endl;
    std::cout << "false: " << falso << std::endl;

    bool b1 = 0;      // false
    bool b2 = 1;      // true
    bool b3 = 42;     // true
    bool b4 = -1;     // true
    bool b5 = 0.0;    // false
    bool b6 = 3.14;   // true

    return 0;
}
```

1.4. TIPO CARACTER

Representa un solo caracter. Internamente es un entero pequeno.

```cpp
#include <iostream>

int main() {
    char letra = 'A';
    char digito = '9';
    char simbolo = '$';

    std::cout << "Letra: " << letra << std::endl;
    std::cout << "Digito: " << digito << std::endl;
    std::cout << "Simbolo: " << simbolo << std::endl;

    // Los char son numericos internamente
    std::cout << "Codigo ASCII de A: " << (int)letra << std::endl;
    std::cout << "Codigo ASCII de 9: " << (int)digito << std::endl;

    // Operaciones aritmeticas con char
    char siguiente = letra + 1;
    std::cout << "Siguiente de A: " << siguiente << std::endl;

    unsigned char uc = 200;
    signed char sc = -100;

    std::cout << "unsigned char: " << (int)uc << std::endl;
    std::cout << "signed char: " << (int)sc << std::endl;

    return 0;
}
```

2. DECLARACION E INICIALIZACION
===============================================================================

2.1. DECLARACION VS DEFINICION

- Declaracion: Anuncia que existe una variable.
- Definicion: Reserva espacio y opcionalmente asigna valor.

```cpp
int x;              // Definicion: reserva espacio (valor indeterminado)
int y = 42;         // Definicion con inicializacion
extern int z;       // Solo declaracion (no reserva espacio)
```

2.2. FORMAS DE INICIALIZACION

```cpp
#include <iostream>

int main() {
    // 1. Asignacion (C-style)
    int a = 10;

    // 2. Constructor (C++ style)
    int b(20);

    // 3. Uniform initialization (C++11, preferida)
    int c{30};
    int d{};  // Inicializacion por defecto (0 para ints)

    // 4. Auto con inicializador
    auto e = 40;

    std::cout << a << " " << b << " " << c << " "
              << d << " " << e << std::endl;

    return 0;
}
```

2.3. REGLAS DE INICIALIZACION

```cpp
#include <iostream>

int global;  // Inicializada a 0 (zero-initialization)

int main() {
    static int estatica;  // Inicializada a 0
    int local;            // NO inicializada (valor basura!)

    std::cout << "global: " << global << std::endl;
    std::cout << "estatica: " << estatica << std::endl;

    // Siempre inicializa las variables locales
    int segura = 0;
    std::cout << "segura: " << segura << std::endl;

    return 0;
}
```

3. CALIFICADORES DE TIPO
===============================================================================

3.1. const

Objetos que no pueden modificarse despues de su inicializacion.

```cpp
#include <iostream>

int main() {
    const double PI = 3.1415926535;
    const int DIAS_SEMANA = 7;

    // PI = 3.14;  // ERROR: no se puede modificar

    // Punteros y const
    int valor = 42;
    const int* ptr1 = &valor;  // Puntero a const int
    int* const ptr2 = &valor;  // Puntero const a int
    const int* const ptr3 = &valor;  // Puntero const a const int

    std::cout << "const es seguridad semantica" << std::endl;
    return 0;
}
```

3.2. signed Y unsigned

```cpp
#include <iostream>

int main() {
    signed int si = -100;
    unsigned int ui = 100;

    std::cout << "signed: " << si << std::endl;
    std::cout << "unsigned: " << ui << std::endl;

    // Cuidado! Comportamiento inesperado con unsigned
    unsigned int u = 0;
    u = u - 1;  // Underflow: se convierte en 4294967295

    std::cout << "0 - 1 = " << u << std::endl;

    // Comparacion signed/unsigned
    int a = -1;
    unsigned int b = 1;

    if (a < b) {
        std::cout << "Esto NO se imprime" << std::endl;
    } else {
        std::cout << "a se convierte a unsigned!" << std::endl;
    }

    return 0;
}
```

3.3. static

```cpp
#include <iostream>

void contador() {
    static int llamadas = 0;  // Solo se inicializa una vez
    llamadas++;
    std::cout << "Llamada #" << llamadas << std::endl;
}

int main() {
    contador();  // #1
    contador();  // #2
    contador();  // #3
    contador();  // #4
    return 0;
}
```

3.4. extern

```cpp
// archivo1.cpp

// archivo2.cpp

    std::cout << compartida << std::endl;  // 100

```

===============================================================================
4. DEDUCCION DE TIPOS: auto Y decltype
===============================================================================

4.1. auto (C++11)

El compilador deduce el tipo desde el inicializador.

```cpp
#include <iostream>
#include <vector>
#include <typeinfo>

int main() {
    auto entero = 42;           // int
    auto flotante = 3.14;       // double
    auto booleano = true;       // bool
    auto caracter = 'A';        // char
    auto texto = "Hola";        // const char*

    std::vector<int> vec = {1, 2, 3, 4, 5};
    auto it = vec.begin();      // std::vector<int>::iterator

    // auto elimina referencias y const por defecto
    int x = 10;
    int& ref = x;
    auto copia = ref;           // int (no int&)
    const int cx = 20;
    auto copia_cx = cx;         // int (no const int)

    // Para mantener referencialidad y const:
    const auto& ref_auto = x;   // const int&
    auto& ref_auto2 = ref;      // int&

    std::cout << "auto simplifica el codigo moderno" << std::endl;
    return 0;
}
```

4.2. decltype (C++11)

Obtiene el tipo de una expresion sin evaluarla.

```cpp
#include <iostream>

int funcion() { return 42; }

int main() {
    int x = 10;
    const int& rx = x;

    decltype(x) a = 5;          // int
    decltype(rx) b = x;         // const int&
    decltype(funcion()) c = 0;  // int

    std::cout << "decltype da el tipo exacto" << std::endl;
    return 0;
}
```

5. TAMANO DE TIPOS: sizeof
===============================================================================

sizeof devuelve el tamano en bytes de un tipo o variable.

```cpp
#include <iostream>

int main() {
    std::cout << "=== TAMANO DE TIPOS (bytes) ===" << std::endl;
    std::cout << "bool:        " << sizeof(bool) << std::endl;
    std::cout << "char:        " << sizeof(char) << std::endl;
    std::cout << "short:       " << sizeof(short) << std::endl;
    std::cout << "int:         " << sizeof(int) << std::endl;
    std::cout << "long:        " << sizeof(long) << std::endl;
    std::cout << "long long:   " << sizeof(long long) << std::endl;
    std::cout << "float:       " << sizeof(float) << std::endl;
    std::cout << "double:      " << sizeof(double) << std::endl;
    std::cout << "long double: " << sizeof(long double) << std::endl;

    int arreglo[10];
    struct Punto { int x; int y; };

    std::cout << "int[10]:     " << sizeof(arreglo) << std::endl;
    std::cout << "Punto:       " << sizeof(Punto) << std::endl;

    int elementos = sizeof(arreglo) / sizeof(arreglo[0]);
    std::cout << "Elementos en arreglo: " << elementos << std::endl;

    return 0;
}
```

===============================================================================
6. INSPECCION DE TIPOS: typeid
===============================================================================

typeid (parte de RTTI) devuelve informacion del tipo.

```cpp
#include <iostream>
#include <typeinfo>

int main() {
    int a = 10;
    double b = 3.14;
    char c = 'X';
    const char* texto = "Hola";

    std::cout << "Tipo de a: " << typeid(a).name() << std::endl;
    std::cout << "Tipo de b: " << typeid(b).name() << std::endl;
    std::cout << "Tipo de c: " << typeid(c).name() << std::endl;
    std::cout << "Tipo de texto: " << typeid(texto).name() << std::endl;

    if (typeid(a) == typeid(int)) {
        std::cout << "a es int" << std::endl;
    }

    auto x = 42;
    std::cout << "Tipo de auto: " << typeid(x).name() << std::endl;

    return 0;
}
```

===============================================================================
7. CONVERSION DE TIPOS (CASTING)
===============================================================================

7.1. CONVERSION IMPLICITA

```cpp
#include <iostream>

int main() {
    int a = 10;
    double b = a;      // int -> double (promocion segura)

    double c = 3.14;
    int d = c;         // double -> int (perdida de precision)

    char ch = 'A';
    int codigo = ch;   // char -> int (promocion)

    bool flag = 42;    // int -> bool

    std::cout << "b: " << b << std::endl;
    std::cout << "d: " << d << std::endl;
    std::cout << "codigo: " << codigo << std::endl;
    std::cout << "flag: " << flag << std::endl;

    return 0;
}
```

7.2. CONVERSION EXPLICITA (CASTING)

```cpp
#include <iostream>

int main() {
    double pi = 3.14159265;

    // C-style cast (NO recomendado en C++)
    int a = (int)pi;

    // Function-style cast
    int b = int(pi);

    // static_cast (preferido para conversiones seguras)
    int c = static_cast<int>(pi);

    std::cout << "C-style: " << a << std::endl;
    std::cout << "function-style: " << b << std::endl;
    std::cout << "static_cast: " << c << std::endl;

    char ch = static_cast<char>(65);
    std::cout << "char(65): " << ch << std::endl;

    return 0;
}
```

7.3. OTROS CASTS ESPECIALIZADOS

```cpp
#include <iostream>

int main() {
    // dynamic_cast: para jerarquias de clases (polimorfismo)
    class Base { virtual void f() {} };
    class Derivada : public Base { void f() override {} };

    Base* b = new Derivada;
    Derivada* d = dynamic_cast<Derivada*>(b);
    if (d) {
        std::cout << "Cast dinamico exitoso" << std::endl;
    }

    // const_cast: elimina/anade const
    const int constante = 10;
    int* modificable = const_cast<int*>(&constante);

    // reinterpret_cast: conversion de tipos no relacionada
    int valor = 0x3F800000;
    float* flotante = reinterpret_cast<float*>(&valor);
    std::cout << "reinterpret: " << *flotante << std::endl;

    return 0;
}
```

10. VARIABLES GLOBALES, LOCALES Y ESTATICAS

```cpp
#include <iostream>

// Variable global (accesible desde cualquier funcion)
int global = 100;

// Variable global constante
const double TASA_IVA = 0.21;

// Variable global estatica (solo visible en este archivo)
static int contadorGlobal = 0;

void ejemplo() {
    // Variable local (solo existe dentro de esta funcion)
    int local = 200;

    // Variable estatica local (persiste entre llamadas)
    static int estaticaLocal = 0;
    estaticaLocal++;

    global++;
    contadorGlobal++;

    std::cout << "estaticaLocal: " << estaticaLocal << std::endl;
    std::cout << "global: " << global << std::endl;
}

int main() {
    ejemplo();  // estaticaLocal = 1
    ejemplo();  // estaticaLocal = 2
    ejemplo();  // estaticaLocal = 3

    std::cout << "contadorGlobal: " << contadorGlobal << std::endl;

    return 0;
}
```

RESUMEN DEL CAPITULO
===============================================================================

- C++ tiene tipos primitivos: int, char, bool, float, double.
- Los enteros tienen variantes signed/unsigned y short/long.
- Los tipos tienen tamanos fijos que sizeof puede medir.
- Siempre inicializa las variables locales para evitar UB.
- const protege variables de modificaciones no deseadas.
- auto deduce el tipo del inicializador (C++11).
- decltype obtiene el tipo de una expresion sin evaluarla.
- typeid permite inspeccionar tipos en tiempo de ejecucion.
- static_cast es la forma preferida de conversion explicita.
- enum class es mas seguro que enum tradicional.
- struct agrupa datos relacionados; union comparte memoria.
- Usa using en lugar de typedef para alias de tipos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

P: Que pasa si uso una variable sin inicializar?
R: Comportamiento indefinido (UB). El programa puede funcionar,
dar resultados incorrectos, crashear o comportarse erraticamente.
Siempre inicializa.

P: Cuando usar float vs double?
R: Usa double por defecto. Solo usa float cuando necesites
ahorrar memoria (arrays grandes, GPU, sistemas embebidos).

P: Por que auto no deduce const o &?
R: Por diseno: auto deduce el tipo "valor". Si quieres const
o referencia, debes especificarlos: const auto&, auto&.

P: Que diferencia hay entre int y long?
R: En sistemas de 32 bits ambos son 4 bytes. En 64 bits, long
puede ser 4 u 8 segun la plataforma. long long siempre es 8 bytes.

P: Debo usar enum class siempre?
R: Si, a menos que tengas una razon muy especifica para usar
enum tradicional. enum class es type-safe y con alcance limitado.
