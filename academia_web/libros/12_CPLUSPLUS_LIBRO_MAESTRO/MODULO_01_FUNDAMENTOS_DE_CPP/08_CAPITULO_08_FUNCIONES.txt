!Bienvenido al capitulo de funciones! Este es uno de los temas
mas importantes del curso. Las funciones son el mecanismo
fundamental para organizar, reutilizar y abstraer codigo.

En C++, las funciones pueden ser simples o complejas, pueden
tener parametros por valor o por referencia, pueden sobrecargarse,
ser inline, tener argumentos por defecto, y mucho mas.

Dominar las funciones es dominar la estructura misma de tus
programas. Sin funciones, el codigo es una sopa de letras.
Con funciones, es una obra de ingenieria bien disenada.

5. SOBRECARGA DE FUNCIONES
===============================================================================

La sobrecarga permite multiples funciones con el mismo nombre
pero diferentes parametros.

# 0 "<stdin>"
# 0 "<built-in>"
# 0 "<command-line>"
# 1 "/usr/include/stdc-predef.h" 1 3 4
# 0 "<command-line>" 2
# 1 "<stdin>"

5.1. REGLAS DE SOBRECARGA

NO son sobrecargas validas:

- Solo difieren en tipo de retorno (int f(int); double f(int);)
- Solo difieren en nombre de parametro (void f(int x); void f(int y);)

SOBRECARGAS validas:

- Diferentes tipos de parametros: void f(int); void f(double);
- Diferente numero de parametros: void f(int); void f(int, int);

Usar solo para funciones muy pequenas (1-3 lineas).

Con optimizacion (-O2), el compilador inlinea automaticamente.

15. BUENAS PRACTICAS CON FUNCIONES
===============================================================================

1. Una funcion debe hacer UNA sola cosa.
2. Nombres descriptivos: calcularArea() y no f().
3. Maximo 3-4 parametros; si hay mas, agrupar en struct.
4. Usar const para parametros de solo lectura.
5. Evitar variables globales.
6. Documentar funciones complejas con comentarios.
7. Mantener funciones cortas (< 30 lineas ideal).
8. Usar prototipos o archivos de cabecera.

===============================================================================
RESUMEN DEL CAPITULO
===============================================================================

- Una funcion tiene: tipo retorno, nombre, parametros y cuerpo.
- Declaracion (prototipo) antes del uso; definicion despues.
- Parametros: valor (copia), referencia (alias), const ref (lectura).
- Valores por defecto: deben ir al final de la lista.
- Sobrecarga: mismo nombre, diferentes parametros.
- inline: sugerencia de insercion directa.
- Variables: locales, globales y estaticas.
- Lambda: funcion anonima con captura [=] o [&].
- Recursion: necesita caso base y llamada recursiva.
- Una funcion = una responsabilidad.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

P: Diferencia entre pasar por valor y por referencia?
R: Por valor se copia (seguro, costoso para objetos grandes).
Por referencia se pasa un alias (eficiente, modificable).
Usa const& para solo lectura.

P: Que es decaimiento de arreglo a puntero?
R: Al pasar un arreglo a una funcion, este se convierte a puntero
al primer elemento. La funcion no conoce el tamano real.

P: Puedo tener main recursiva?
R: Legal pero no recomendado. main es especial y no deberia
llamarse a si misma.

P: Que es inline?
R: Sugiere al compilador insertar el codigo directamente donde
se llama, eliminando el overhead de llamada. Solo util para
funciones muy pequenas.

P: Las lambdas pueden modificar variables capturadas?
R: Por defecto no (captura por valor es const). Con mutable si.
Con captura por referencia, modifican el original.

16. FUNCIONES TEMPLATE
===============================================================================

Las funciones template permiten escribir codigo generico que
funciona con cualquier tipo de dato.

```cpp
#include <iostream>
#include <string>

template <typename T>
T maximo(T a, T b) {
    return (a > b) ? a : b;
}

template <typename T>
void imprimirArreglo(const T* arr, int n) {
    for (int i = 0; i < n; ++i) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;
}

int main() {
    std::cout << maximo(3, 7) << std::endl;          // int
    std::cout << maximo(3.14, 2.71) << std::endl;    // double
    std::cout << maximo(std::string("Ana"),
                        std::string("Luis")) << std::endl;  // string

    int enteros[] = {1, 2, 3, 4, 5};
    double reales[] = {1.1, 2.2, 3.3};
    imprimirArreglo(enteros, 5);
    imprimirArreglo(reales, 3);

    return 0;
}
```

16.1. MULTIPLES PARAMETROS TEMPLATE

```cpp
#include <iostream>

template <typename T, typename U>
auto producto(T a, U b) -> decltype(a * b) {
    return a * b;
}

template <typename retorno, typename T, typename U>
retorno convertir(T a, U b) {
    return static_cast<retorno>(a + b);
}

int main() {
    auto r1 = producto(3, 4.5);  // double = int * double
    auto r2 = producto(2.5, 3);   // double = double * int
    int r3 = convertir<int>(3.7, 2.3);  // 6
    std::cout << r1 << " " << r2 << " " << r3 << std::endl;
    return 0;
}
```

17. FUNCIONES CONSTEXPR (C++11)
===============================================================================

constexpr indica que una funcion puede evaluarse en tiempo
de compilacion si sus argumentos son constantes.

```cpp
#include <iostream>

constexpr long long factorialConstexpr(int n) {
    return (n <= 1) ? 1 : n * factorialConstexpr(n - 1);
}

constexpr double cuadradoConstexpr(double x) {
    return x * x;
}

int main() {
    // Evaluado en tiempo de compilacion
    constexpr long long fact10 = factorialConstexpr(10);
    std::cout << "Factorial 10: " << fact10 << std::endl;

    // Evaluado en tiempo de ejecucion
    int n = 7;
    long long factN = factorialConstexpr(n);
    std::cout << "Factorial " << n << ": " << factN << std::endl;

    // Uso en contextos de constantes
    constexpr double lado = 5.0;
    constexpr double area = cuadradoConstexpr(lado);
    std::cout << "Area cuadrado: " << area << std::endl;

    return 0;
}
```

18. FUNCIONES CON NOEXCEPT
===============================================================================

noexcept indica que una funcion no lanza excepciones.

```cpp
#include <iostream>
#include <vector>

int suma(int a, int b) noexcept {
    return a + b;
}

void procesar(const std::vector<int>& datos) noexcept {
    for (int x : datos) {
        std::cout << x << " ";
    }
    std::cout << std::endl;
}

int main() {
    std::cout << std::boolalpha;
    std::cout << "suma es noexcept: "
              << noexcept(suma(1, 2)) << std::endl;

    std::vector<int> v = {1, 2, 3};
    std::cout << "procesar es noexcept: "
              << noexcept(procesar(v)) << std::endl;

    return 0;
}
```

```cpp
#include <iostream>

    std::cout << "Obsoleta" << std::endl;
}

// [[maybe_unused]] (C++17): suprime warning si no se usa
[[maybe_unused]] void funcionNoUsada() {
    std::cout << "Aun no implementada" << std::endl;
}

void nuevaFuncion() {
    std::cout << "Nueva version" << std::endl;
}

int main() {
    nuevaFuncion();
    // obtenerValorImportante();  // Warning: nodiscard!
    int val = obtenerValorImportante();
    std::cout << "Valor: " << val << std::endl;
    return 0;
}
```

20. FUNCIONES COMO CIUDADANOS DE PRIMERA CLASE
===============================================================================

En C++ moderno, las funciones pueden pasarse, almacenarse y
devolverse como cualquier otro valor.

```cpp
#include <iostream>
#include <functional>
#include <map>
#include <string>

// std::function: envoltura generica para funciones, lambdas, etc.
void ejecutar(const std::string& nombre,
              std::function<int(int, int)> operacion,
              int a, int b) {
    std::cout << nombre << ": " << operacion(a, b) << std::endl;
}

int main() {
    // Mapa de operaciones
    std::map<std::string, std::function<int(int, int)>> ops;
    ops["sumar"] = [](int a, int b) { return a + b; };
    ops["restar"] = [](int a, int b) { return a - b; };
    ops["multiplicar"] = [](int a, int b) { return a * b; };
    ops["dividir"] = [](int a, int b) {
        return (b != 0) ? a / b : 0;
    };

    for (const auto& [nombre, op] : ops) {
        ejecutar(nombre, op, 10, 5);
    }

    // std::bind: crear funciones parciales
    auto alCuadrado = std::bind(
        [](int x, int y) { return x * y; },
        std::placeholders::_1,
        std::placeholders::_1
    );
    std::cout << "5 al cuadrado: " << alCuadrado(5) << std::endl;

    return 0;
}
```

21. EJEMPLO INTEGRADOR: CALCULADORA CIENTIFICA CON FUNCIONES
===============================================================================

```cpp
#include <iostream>
#include <functional>
#include <map>
#include <cmath>
#include <iomanip>

using Operacion = std::function<double(double, double)>;

double suma(double a, double b) { return a + b; }
double resta(double a, double b) { return a - b; }
double mult(double a, double b) { return a * b; }
double div(double a, double b) {
    if (b == 0) throw std::runtime_error("Division por cero");
    return a / b;
}
double pot(double a, double b) { return std::pow(a, b); }
double raiz(double a, double) { return std::sqrt(a); }

int main() {
    std::map<char, Operacion> ops = {
        {'+', suma}, {'-', resta}, {'*', mult},
        {'/', div}, {'^', pot}, {'r', raiz}
    };

    std::cout << "=== CALCULADORA FUNCIONAL ===" << std::endl;
    std::cout << std::fixed << std::setprecision(4);

    double a, b;
    char op;

    std::cout << "Ingresa: numero operacion numero: ";
    std::cin >> a >> op >> b;

    auto it = ops.find(op);
    if (it != ops.end()) {
        try {
            double resultado = it->second(a, b);
            std::cout << a << " " << op << " " << b << " = "
                      << resultado << std::endl;
        } catch (const std::exception& e) {
            std::cerr << "Error: " << e.what() << std::endl;
        }
    } else {
        std::cout << "Operacion no valida" << std::endl;
    }

    return 0;
}
```

22. DEPURACION DE FUNCIONES
===============================================================================

Tecnicas para depurar funciones:

```cpp
#include <iostream>
#include <cassert>

// assert: verifica condiciones en debug
int factorialDebug(int n) {
    assert(n >= 0);  // Se activa solo en debug (sin -DNDEBUG)
    if (n <= 1) return 1;
    return n * factorialDebug(n - 1);
}

// static_assert: verifica en tiempo de compilacion
template <int N>
struct Factorial {
    static_assert(N >= 0, "N no puede ser negativo");
    static constexpr long long valor = N * Factorial<N - 1>::valor;
};

template <>
struct Factorial<0> {
    static constexpr long long valor = 1;
};

int main() {
    // En debug, assert verifica
    std::cout << factorialDebug(5) << std::endl;

    // static_assert en tiempo de compilacion
    std::cout << Factorial<10>::valor << std::endl;

    // Esto no compila:
    // std::cout << Factorial<-5>::valor << std::endl;

    return 0;
}
```
