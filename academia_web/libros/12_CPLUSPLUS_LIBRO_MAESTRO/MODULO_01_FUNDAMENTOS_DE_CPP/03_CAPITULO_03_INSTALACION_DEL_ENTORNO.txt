===============================================================================
1. INSTALACIÓN DEL COMPILADOR G++
===============================================================================

G++ (GNU C++ Compiler) es parte del proyecto GNU Compiler
Collection (GCC). Es el compilador más utilizado en sistemas
Linux y macOS, y también está disponible en Windows a través
de MinGW o WSL.

1.1. LINUX (UBUNTU/DEBIAN)

```bash
# Actualizar el índice de paquetes
sudo apt update

# Instalar el compilador G++ y herramientas relacionadas
sudo apt install g++ build-essential gdb make

# Verificar la instalación
g++ --version

# Salida esperada:
# g++ (Ubuntu 13.2.0-4ubuntu3) 13.2.0
```

Para otras distribuciones:

```bash
# Fedora/RHEL/CentOS
sudo dnf install gcc-c++ gdb make

# Arch Linux
sudo pacman -S gcc gdb make

# openSUSE
sudo zypper install gcc-c++ gdb make
```

1.2. macOS

```bash
# Opción 1: Xcode Command Line Tools (recomendada)
xcode-select --install

# Esto instala clang++ (compilador por defecto en macOS)
# Es compatible con g++ en la mayoría de los casos
clang++ --version

# Opción 2: Homebrew (instala gcc real)
brew install gcc

# Verificar
g++-14 --version
```

1.3. WINDOWS

**Opción 1: MinGW-w64 (Recomendada)**

```bash
# Descargar MinGW-w64 de:
# https://www.mingw-w64.org/

# O usando winget (Windows 10/11)
winget install -e --id MSYS2.MSYS2

# Luego en MSYS2:
pacman -S mingw-w64-x86_64-gcc
```

**Opción 2: WSL (Windows Subsystem for Linux)**

```bash
# En PowerShell (como administrador)
wsl --install -d Ubuntu

# Luego dentro de Ubuntu:
sudo apt update
sudo apt install g++ build-essential
```

**Opción 3: Visual Studio con MSVC**

```bash
# Descargar Visual Studio Community desde:
# https://visualstudio.microsoft.com/

# Durante la instalación, seleccionar:
# "Desarrollo de escritorio con C++"
```

===============================================================================
2. COMPILACIÓN DESDE LA TERMINAL
===============================================================================

2.1. COMANDOS BÁSICOS DE G++

```bash
# Sintaxis general
g++ [opciones] archivos_fuente -o nombre_ejecutable

# Compilación simple de un archivo
g++ programa.cpp -o programa

# Compilar con optimización
g++ -O2 programa.cpp -o programa

# Compilar con información de depuración
g++ -g programa.cpp -o programa

# Compilar con todos los warnings
g++ -Wall -Wextra programa.cpp -o programa

# Compilar con un estándar específico
g++ -std=c++20 programa.cpp -o programa

# Compilar múltiples archivos fuente
g++ main.cpp auxiliar.cpp -o programa

# Compilar a código objeto (sin enlazar)
g++ -c modulo.cpp -o modulo.o
```

2.2. OPCIONES IMPORTANTES DE G++

| OPCIÓN          | DESCRIPCIÓN                                 |
|-----------------|---------------------------------------------|
| -o <archivo>    | Nombre del archivo de salida                |
| -c              | Compilar solo a objeto (no enlazar)         |
| -S              | Compilar a ensamblador                      |
| -E              | Solo preprocesar                            |
| -O0, -O1, -O2, -O3 | Niveles de optimización                   |
| -Os             | Optimizar por tamaño                        |
| -g              | Información de depuración (GDB)             |
| -Wall           | Activar warnings principales                |
| -Wextra         | Activar warnings adicionales                |
| -Wpedantic      | Warnings del estándar estricto              |
| -Werror         | Tratar warnings como errores                |
| -std=c++20      | Especificar estándar                        |
| -I<directorio>  | Añadir directorio de include                |
| -L<directorio>  | Añadir directorio de bibliotecas            |
| -l<biblioteca>  | Enlazar biblioteca (ej: -lm para math)      |
| -pthread        | Soporte para hilos POSIX                    |

2.3. EJEMPLO COMPLETO

```cpp
// calculadora.cpp
#include <iostream>

int suma(int a, int b) { return a + b; }
int resta(int a, int b) { return a - b; }
int multiplica(int a, int b) { return a * b; }
int divide(int a, int b) { return b != 0 ? a / b : 0; }

int main() {
    int x = 10, y = 5;
    std::cout << "Suma: " << suma(x, y) << std::endl;
    std::cout << "Resta: " << resta(x, y) << std::endl;
    std::cout << "Multiplica: " << multiplica(x, y) << std::endl;
    std::cout << "Divide: " << divide(x, y) << std::endl;
    return 0;
}
```

```bash
# Compilar con todas las opciones recomendadas
g++ -std=c++20 -Wall -Wextra -Wpedantic -O2 -g calculadora.cpp -o calculadora

# Ejecutar
./calculadora
```

3.3. CONFIGURACIÓN DE VS CODE PARA C++

Crear el archivo `.vscode/tasks.json` para compilar con un clic:

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Compilar C++",
            "type": "shell",
            "command": "g++",
            "args": [
                "-std=c++20",
                "-Wall",
                "-Wextra",
                "-Wpedantic",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "problemMatcher": ["$gcc"],
            "detail": "Compilar archivo C++ activo"
        }
    ]
}
```

Crear `launch.json` para depuración:

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Depurar C++",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${fileDirname}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "miDebuggerPath": "/usr/bin/gdb",
            "preLaunchTask": "Compilar C++"
        }
    ]
}
```

4. CMake: EL GESTOR DE PROYECTOS
===============================================================================

CMake es la herramienta estándar para gestionar proyectos C++.
Genera archivos de compilación para múltiples plataformas.

4.1. INSTALACIÓN DE CMAKE

```bash
# Linux (Ubuntu/Debian)
sudo apt install cmake cmake-gui

# macOS
brew install cmake

# Windows (con MSYS2)
pacman -S mingw-w64-x86_64-cmake

# Verificar
cmake --version
```

4.2. ESTRUCTURA BÁSICA DE UN PROYECTO CMAKE

```
mi_proyecto/
├── CMakeLists.txt
├── src/
│   ├── main.cpp
│   └── funciones.cpp
└── include/
    └── funciones.h
```

4.3. CMakeLists.txt BÁSICO

```cmake
# CMakeLists.txt
cmake_minimum_required(VERSION 3.20)

# Nombre del proyecto
project(MiProyecto VERSION 1.0.0 LANGUAGES CXX)

# Especificar el estándar C++
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# Warnings
if(CMAKE_COMPILER_IS_GNUCC OR CMAKE_COMPILER_IS_GNUCXX)
    add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# Incluir directorios de cabeceras
include_directories(${PROJECT_SOURCE_DIR}/include)

# Crear el ejecutable
add_executable(mi_programa
    src/main.cpp
    src/funciones.cpp
)
```

4.4. COMPILAR CON CMAKE

```bash
# Desde la raíz del proyecto
mkdir build
cd build

# Configurar
cmake ..

# Compilar
cmake --build .

# Ejecutar
./mi_programa

# Liberar (Release)
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build .

# Depuración (Debug)
cmake .. -DCMAKE_BUILD_TYPE=Debug
cmake --build .
```

5. DEPURACIÓN CON GDB
===============================================================================

5.1. COMPILAR PARA DEPURACIÓN

```bash
g++ -g programa.cpp -o programa
```

5.2. COMANDOS BÁSICOS DE GDB

```bash
# Iniciar depurador
gdb ./programa

# Comandos dentro de GDB:
run          # Ejecutar el programa
break main   # Poner punto de ruptura en main
break 25     # Poner punto de ruptura en línea 25
next         # Ejecutar siguiente línea (sin entrar en función)
step         # Ejecutar siguiente línea (entrando en función)
continue     # Continuar hasta siguiente breakpoint
print x      # Mostrar valor de variable x
list         # Mostrar código fuente
quit         # Salir de GDB
backtrace    # Mostrar pila de llamadas
info locals  # Mostrar variables locales
```

```cpp
// programa_depuracion.cpp
#include <iostream>

int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

int main() {
    int numero = 5;
    int resultado = factorial(numero);

    std::cout << "Factorial de " << numero
              << " es " << resultado << std::endl;

    return 0;
}
```

```bash
# Depurar
g++ -g programa_depuracion.cpp -o programa_depuracion
gdb ./programa_depuracion

# Dentro de GDB:
# (gdb) break main
# (gdb) run
# (gdb) next
# (gdb) print numero
# (gdb) step
# (gdb) continue
```

6. COMPILACIÓN DE MÚLTIPLES ARCHIVOS
===============================================================================

En proyectos reales, el código se organiza en múltiples archivos.

6.1. ORGANIZACIÓN POR ARCHIVOS

```
proyecto/
├── main.cpp
├── operaciones.h
├── operaciones.cpp
├── utilidades.h
└── utilidades.cpp
```

```cpp
// operaciones.h
#ifndef OPERACIONES_H
#define OPERACIONES_H

namespace Operaciones {
    int suma(int a, int b);
    int resta(int a, int b);
    int multiplicacion(int a, int b);
    double division(double a, double b);
}

#endif
```

```cpp
// operaciones.cpp
#include "operaciones.h"

namespace Operaciones {
    int suma(int a, int b) {
        return a + b;
    }

    int resta(int a, int b) {
        return a - b;
    }

    int multiplicacion(int a, int b) {
        return a * b;
    }

    double division(double a, double b) {
        if (b == 0.0) return 0.0;
        return a / b;
    }
}
```

```cpp
// utilidades.h
#ifndef UTILIDADES_H
#define UTILIDADES_H

#include <string>
#include <vector>

namespace Utilidades {
    void imprimirResultado(const std::string& operacion, double valor);
    void limpiarPantalla();
    void pausar();
}

#endif
```

```cpp
// utilidades.cpp
#include "utilidades.h"
#include <iostream>
#include <cstdlib>

namespace Utilidades {
    void imprimirResultado(const std::string& operacion, double valor) {
        std::cout << operacion << ": " << valor << std::endl;
    }

    void limpiarPantalla() {
        #ifdef _WIN32
            system("cls");
        #else
            system("clear");
        #endif
    }

    void pausar() {
        std::cout << "Presiona Enter para continuar...";
        std::cin.get();
    }
}
```

```cpp
// main.cpp
#include <iostream>
#include "operaciones.h"
#include "utilidades.h"

int main() {
    int a = 15, b = 4;

    Utilidades::imprimirResultado("Suma",
        Operaciones::suma(a, b));
    Utilidades::imprimirResultado("Resta",
        Operaciones::resta(a, b));
    Utilidades::imprimirResultado("Multiplicación",
        Operaciones::multiplicacion(a, b));
    Utilidades::imprimirResultado("División",
        Operaciones::division(a, b));

    return 0;
}
```

```bash
# Forma 1: Compilar todo junto
g++ -std=c++20 -Wall main.cpp operaciones.cpp utilidades.cpp -o programa

# Forma 2: Compilar por partes (recomendado para proyectos grandes)
g++ -std=c++20 -Wall -c main.cpp -o main.o
g++ -std=c++20 -Wall -c operaciones.cpp -o operaciones.o
g++ -std=c++20 -Wall -c utilidades.cpp -o utilidades.o
g++ main.o operaciones.o utilidades.o -o programa
```

===============================================================================
7. MAKE Y MAKEFILE
===============================================================================

Make es una herramienta clásica de automatización de compilación.

```makefile
# Makefile
CXX = g++
CXXFLAGS = -std=c++20 -Wall -Wextra -Wpedantic -O2
OBJETOS = main.o operaciones.o utilidades.o
EJECUTABLE = programa

all: $(EJECUTABLE)

$(EJECUTABLE): $(OBJETOS)
	$(CXX) $(OBJETOS) -o $(EJECUTABLE)

main.o: main.cpp operaciones.h utilidades.h
	$(CXX) $(CXXFLAGS) -c main.cpp

operaciones.o: operaciones.cpp operaciones.h
	$(CXX) $(CXXFLAGS) -c operaciones.cpp

utilidades.o: utilidades.cpp utilidades.h
	$(CXX) $(CXXFLAGS) -c utilidades.cpp

clean:
	rm -f *.o $(EJECUTABLE)

run: $(EJECUTABLE)
	./$(EJECUTABLE)

.PHONY: all clean run
```

```bash
# Usar Make
make        # Compilar todo
make run    # Compilar y ejecutar
make clean  # Limpiar archivos objeto
```

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué compilador debo usar, g++ o clang++?** Ambos son
excelentes. G++ tiene soporte más completo de estándares
recientes. Clang++ tiene mejores mensajes de error. Usa el
que tengas disponible. Este curso usa g++.

❓ **¿Necesito un IDE o basta con un editor?** Puedes empezar
con cualquier editor. VS Code es un buen punto medio entre
simplicidad y potencia. Más adelante puedes probar CLion.

❓ **¿Qué hago si la compilación da error 'undefined reference'?**
Significa que no estás enlazando todos los archivos objeto.
Asegúrate de compilar y enlazar todos los .cpp de tu proyecto.

❓ **¿Por qué usar CMake en lugar de compilar a mano?** Para
proyectos pequeños, compilar a mano basta. Para proyectos
grandes, CMake automatiza todo y es multiplataforma.

❓ **¿Qué significa el error 'command not found: g++'?**
Que G++ no está instalado o no está en el PATH. Revisa la
instalación según tu sistema operativo.
