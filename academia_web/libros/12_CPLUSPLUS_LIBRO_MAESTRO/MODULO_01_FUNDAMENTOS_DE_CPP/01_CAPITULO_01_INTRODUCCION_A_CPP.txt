1. ¿QUÉ ES C++?
===============================================================================

C++ es un lenguaje de programación de propósito general, compilado,
de tipado estático y multiparadigma. Fue creado por Bjarne Stroustrup
en los laboratorios Bell a partir de 1979. Originalmente llamado
"C with Classes", fue renombrado a C++ en 1983.

El nombre C++ proviene del operador de incremento en C (++), indicando
que es "el siguiente paso de C". El lenguaje hereda la eficiencia de C
y añade abstracciones de alto nivel como clases, herencia, polimorfismo
y programación genérica.

```cpp
// Un primer vistazo a C++
#include <iostream>
#include <string>

int main() {
    std::string saludo = "¡Hola, C++!";
    std::cout << saludo << std::endl;
    return 0;
}
```

C++ ha pasado por múltiples estándares: C++98, C++11, C++14, C++17,
C++20 y C++23. Cada estándar introduce nuevas características que
hacen el lenguaje más expresivo, seguro y eficiente.

===============================================================================
2. CARACTERÍSTICAS PRINCIPALES DE C++
===============================================================================

C++ es único porque combina programación de bajo y alto nivel. Puedes
manejar memoria manualmente y también usar abstracciones modernas.

2.1. MULTIPARADIGMA

C++ soporta múltiples estilos de programación:

- **Programación procedural**: funciones y procedimientos al estilo C.
- **Programación orientada a objetos**: clases, herencia, polimorfismo.
- **Programación genérica**: templates, STL.
- **Programación funcional**: lambdas, algorithms (desde C++11).

```cpp
// Programación procedural
int suma(int a, int b) {
    return a + b;
}

// Programación orientada a objetos
class Calculadora {
public:
    int suma(int a, int b) { return a + b; }
};

// Programación genérica
template <typename T>
T suma(T a, T b) {
    return a + b;
}

// Programación funcional (lambda, C++11)
auto sumador = [](int a, int b) { return a + b; };
```

2.2. EFICIENCIA Y RENDIMIENTO

C++ proporciona control total sobre los recursos del sistema:

- Gestión manual de memoria (new/delete).
- Asignación en stack y heap.
- Sin runtime pesado ni garbage collector.
- Compilación directa a código máquina.
- Optimizaciones agresivas del compilador.

```cpp
// Control de memoria
int* arreglo = new int[1000];
// ... trabajar con el arreglo ...
delete[] arreglo;
```

2.3. TIPADO ESTÁTICO Y FUERTE

Las variables tienen tipos definidos en tiempo de compilación.
El compilador verifica la coherencia de tipos antes de ejecutar.

```cpp
int entero = 42;          // Correcto
double decimal = 3.14;    // Correcto
// entero = "texto";      // Error de compilación
```

2.4. ABSTRACCIÓN SIN COSTE

Uno de los principios más importantes de C++: no pagas por lo que
no usas. Las abstracciones de alto nivel no introducen overhead
si no se utilizan.

```cpp
// Clase sin overhead respecto a struct C
class Punto {
public:
    int x, y;
};

// Equivalente funcional a struct { int x, y; }
```

3. C++ VS C
===============================================================================

C++ es un superconjunto de C en la práctica, aunque no oficialmente.
Todo programa C válido debería compilar en C++ (con pocas excepciones).

| CARACTERÍSTICA       | C                      | C++                         |
|----------------------|------------------------|-----------------------------|
| Paradigma            | Procedural             | Multiparadigma              |
| Clases y objetos     | No                     | Sí                          |
| STL                  | No                     | Sí                          |
| Sobrecarga           | No                     | Sí                          |
| Templates            | No                     | Sí                          |
| Excepciones          | No                     | Sí                          |
| Polimorfismo         | Manual (function ptrs) | Virtual functions           |
| Gestión de memoria   | malloc/free            | new/delete + smart pointers |
| Namespaces           | No                     | Sí                          |
| Referencias          | No                     | Sí                          |
| Entrada/Salida       | printf/scanf           | cout/cin                    |
| Strings              | char*                  | std::string                 |

```cpp
// C: Manejo manual con funciones
// C++: Abstracción con clases
#include <string>

std::string nombre = "Lucas";
size_t longitud = nombre.length();
```

7. LA STL (STANDARD TEMPLATE LIBRARY)
===============================================================================

La STL es una de las bibliotecas más influyentes de la historia
de la programación. Fue desarrollada por Alexander Stepanov y
adoptada en el estándar C++98.

La STL consta de cuatro componentes fundamentales:

7.1. CONTENEDORES

Estructuras de datos predefinidas:

- **Secuenciales**: vector, list, deque, array.
- **Asociativos**: set, map, multiset, multimap.
- **No ordenados**: unordered_set, unordered_map.

```cpp
#include <vector>
#include <map>
#include <string>

std::vector<int> numeros = {1, 2, 3, 4, 5};
std::map<std::string, int> edades;
edades["Ana"] = 25;
edades["Luis"] = 30;
```

7.2. ALGORITMOS

Funciones genéricas para operar sobre contenedores:

```cpp
#include <algorithm>
#include <vector>

std::vector<int> v = {5, 2, 8, 1, 9};
std::sort(v.begin(), v.end());
auto it = std::find(v.begin(), v.end(), 8);
int suma = std::accumulate(v.begin(), v.end(), 0);
```

7.3. ITERADORES

Abstracción para recorrer contenedores de forma uniforme:

```cpp
std::vector<int> v = {10, 20, 30};
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it << " ";
}
```

7.4. FUNCTORES Y ADAPTADORES

Objetos que se comportan como funciones y componentes
que adaptan otros componentes:

```cpp
struct MayorQue {
    int valor;
    MayorQue(int v) : valor(v) {}
    bool operator()(int x) { return x > valor; }
};

std::vector<int> v = {1, 5, 8, 3, 10};
auto it = std::find_if(v.begin(), v.end(), MayorQue(5));
```

8. COMPILACIÓN VS INTERPRETACIÓN
===============================================================================

C++ es un lenguaje compilado. Esto significa que el código fuente
se traduce a código máquina antes de la ejecución.

8.1. PROCESO DE COMPILACIÓN

```
[main.cpp]  →  PREPROCESADOR  →  [main.i]
   ↓
COMPILADOR  →  [main.s]  (ensamblador)
   ↓
ENSAMBLADOR →  [main.o]  (código objeto)
   ↓
ENLAZADOR   →  [programa]  (ejecutable)
```

8.2. ETAPAS EN DETALLE

1. **Preprocesamiento**: Expande #include, #define, #ifdef, etc.
2. **Compilación**: Traduce C++ a código ensamblador.
3. **Ensamblado**: Convierte ensamblador a código máquina (.o).
4. **Enlace**: Une todos los .o y bibliotecas en un ejecutable.

```bash
# Compilación paso a paso
g++ -E main.cpp -o main.i    # Solo preprocesador
g++ -S main.i -o main.s      # Solo compilación a ASM
g++ -c main.s -o main.o      # Solo ensamblado
g++ main.o -o programa        # Solo enlace

# Todo en uno
g++ main.cpp -o programa
```

10. MITOS Y VERDADES SOBRE C++
===============================================================================

Mito 1: "C++ es un lenguaje muerto"
Verdad: C++ esta mas vivo que nunca. Se actualiza cada 3 anyos
con nuevos estandares. Es esencial en sistemas de alto rendimiento.

Mito 2: "C++ es solo C con clases"
Verdad: C++ ha evolucionado mucho mas alla. Programacion generica,
funcional, metaprogramacion, y caracteristicas que C no tiene.

Mito 3: "C++ es demasiado dificil"
Verdad: C++ moderno (C++11 en adelante) es mas seguro y expresivo.
Puedes empezar con un subconjunto simple e ir aprendiendo mas.

Mito 4: "Rust va a reemplazar a C++"
Verdad: Rust es un gran lenguaje, pero C++ tiene demasiado codigo
legado y ecosistema. Ambos coexistiran por mucho tiempo.

Mito 5: "C++ no es seguro"
Verdad: C++ moderno con smart pointers, RAII y const correctness
puede ser tan seguro como cualquier otro lenguaje.

14. CONSEJOS PARA APRENDER C++ EFECTIVAMENTE
===============================================================================

1. **Practica a diario**: 30 minutos de codigo al dia > 5 horas
   un solo dia a la semana.

2. **Escribe codigo, no solo leas**: La programacion es una
   habilidad practica, no teorica.

3. **Usa C++ moderno**: Ignora los tutoriales que ensenyan C++98.
   Aprende C++17/20 desde el principio.

4. **Lee codigo de otros**: Estudia proyectos open source en C++
   (LLVM, Boost, spdlog, fmt).

5. **Compila con -Wall -Wextra**: Trata los warnings como errores.

6. **Usa un depurador**: Aprende GDB o el depurador de tu IDE.

7. **Haz preguntas**: Stack Overflow, Reddit, Discord de C++.

8. **Construye proyectos personales**: La mejor forma de aprender
   es construyendo algo que te interese.

9. **Entiende el "por que"**: No solo copies codigo; entiende
   por que funciona asi.

10. **Se paciente**: Dominar C++ lleva anyos, pero cada mes
    veras progreso.
