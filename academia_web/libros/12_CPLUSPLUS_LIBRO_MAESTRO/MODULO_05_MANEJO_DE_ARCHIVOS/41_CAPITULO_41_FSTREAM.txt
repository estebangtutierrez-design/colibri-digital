1. INTRODUCCIÓN A FSTREAM

La biblioteca <fstream> proporciona tres clases fundamentales para el manejo
de archivos en C++:

  - ifstream  (input file stream)  → lectura desde archivos
  - ofstream  (output file stream) → escritura hacia archivos
  - fstream   (file stream)        → lectura y escritura combinadas

Estas clases heredan de istream, ostream e iostream respectivamente, lo que
significa que comparten la misma interfaz que cin y cout.

2. LA CLASE OFSTREAM (ESCRITURA)

La clase ofstream se utiliza para crear y escribir datos en archivos. Cuando
abrimos un archivo con ofstream, por defecto se crea si no existe y se trunca
(su contenido anterior se pierde).

  ┌─────────────────────────────────────────────────────┐
  │ Modos comunes de apertura (ofstream):                │
  │                                                      │
  │  ios::out   → escritura (predeterminado)            │
  │  ios::app   → append (agregar al final)             │
  │  ios::trunc → truncar si existe (predeterminado)    │
  │  ios::binary → modo binario                         │
  └─────────────────────────────────────────────────────┘

3. LA CLASE IFSTREAM (LECTURA)

ifstream permite leer datos desde archivos. Requiere que el archivo exista, de
lo contrario el flujo entrará en estado de error.

  ┌─────────────────────────────────────────────────────┐
  │ Modos comunes de apertura (ifstream):                │
  │                                                      │
  │  ios::in    → lectura (predeterminado)              │
  │  ios::binary → modo binario                         │
  └─────────────────────────────────────────────────────┘

4. LA CLASE FSTREAM (LECTURA + ESCRITURA)

fstream combina ambas capacidades. Se utiliza cuando necesitamos leer y
escribir en el mismo archivo sin cerrarlo y volverlo a abrir.

5. RAII CON ARCHIVOS

RAII (Resource Acquisition Is Initialization) es una técnica donde el recurso
(archivo) se adquiere en el constructor y se libera en el destructor. En C++,
las clases fstream ya implementan RAII: el archivo se cierra automáticamente
al salir del ámbito.

6. APERTURA EXPLÍCITA VS IMPLÍCITA

  - Implícita:  ofstream archivo("datos.txt");
  - Explícita:  ofstream archivo; archivo.open("datos.txt");

La forma explícita es útil cuando queremos reutilizar el objeto para abrir
diferentes archivos.

#include <iostream>
#include <fstream>
#include <string>
#include <stdexcept>

using namespace std;

//==============================================================================
// EJEMPLO 1: OFSTREAM BÁSICO
//==============================================================================
/*
  Demostración de ofstream para escritura básica. El archivo se crea
  automáticamente; si ya existía, se trunca.
*/
void ejemplo_ofstream_basico()
{
    cout << "\n--- EJEMPLO 1: ofstream básico ---\n";

    ofstream archivo("ejemplo1.txt");

    if (!archivo.is_open())
    {
        cerr << "Error: no se pudo crear el archivo.\n";
        return;
    }

    archivo << "Línea 1: Hola, mundo!\n";
    archivo << "Línea 2: Esto es C++ Master Book.\n";
    archivo << "Línea 3: Módulo V - Manejo de Archivos.\n";
    archivo << 42 << " también se pueden escribir números.\n";

    archivo.close();
    cout << "Archivo 'ejemplo1.txt' creado con éxito.\n";
}

//==============================================================================
// EJEMPLO 2: IFSTREAM BÁSICO
//==============================================================================
/*
  Demostración de ifstream para lectura básica.
*/
void ejemplo_ifstream_basico()
{
    cout << "\n--- EJEMPLO 2: ifstream básico ---\n";

    ifstream archivo("ejemplo1.txt");

    if (!archivo)
    {
        cerr << "Error: no se pudo abrir el archivo.\n";
        return;
    }

    string linea;
    while (getline(archivo, linea))
    {
        cout << "Leído: " << linea << "\n";
    }

    archivo.close();
}

//==============================================================================
// EJEMPLO 3: MODO APPEND (ios::app)
//==============================================================================
/*
  ios::app permite agregar contenido al final del archivo sin truncar.
*/
void ejemplo_append()
{
    cout << "\n--- EJEMPLO 3: Modo append (ios::app) ---\n";

    ofstream archivo("ejemplo1.txt", ios::app);

    if (!archivo)
    {
        cerr << "Error al abrir para append.\n";
        return;
    }

    archivo << "Línea agregada con ios::app\n";
    archivo << "Otra línea al final.\n";

    archivo.close();
    cout << "Contenido agregado al final.\n";
}

//==============================================================================
// EJEMPLO 4: FSTREAM LECTURA + ESCRITURA
//==============================================================================
/*
  fstream permite leer y escribir en el mismo archivo.
  Importante: al cambiar de lectura a escritura debe usarse seek.
*/
void ejemplo_fstream()
{
    cout << "\n--- EJEMPLO 4: fstream lectura/escritura ---\n";

    // Primero creamos un archivo base
    fstream archivo("ejemplo_fstream.txt", ios::out | ios::trunc);
    archivo << "Primera línea\n";
    archivo << "Segunda línea\n";
    archivo << "Tercera línea\n";
    archivo.close();

    // Ahora abrimos para lectura y escritura
    archivo.open("ejemplo_fstream.txt", ios::in | ios::out);

    if (!archivo)
    {
        cerr << "Error al abrir fstream.\n";
        return;
    }

    string linea;
    // Leemos la primera línea
    getline(archivo, linea);
    cout << "Leído: " << linea << "\n";

    // Volvemos al inicio y escribimos
    archivo.seekp(0, ios::beg);
    archivo << "Línea MODIFICADA\n";

    archivo.close();

    // Verificamos el resultado
    archivo.open("ejemplo_fstream.txt", ios::in);
    cout << "Contenido final:\n";
    while (getline(archivo, linea))
        cout << "  " << linea << "\n";
    archivo.close();
}

//==============================================================================
// EJEMPLO 5: RAII — CIERRE AUTOMÁTICO
//==============================================================================
/*
  RAII: el destructor de ofstream/ifstream cierra el archivo automáticamente.
  No es necesario llamar a close() explícitamente, aunque es buena práctica.
*/
void ejemplo_raii()
{
    cout << "\n--- EJEMPLO 5: RAII con archivos ---\n";
    {
        ofstream archivo("raii.txt");
        archivo << "Este archivo se cerrará automáticamente.\n";
        archivo << "Al salir de este bloque, el destructor cierra el recurso.\n";
        cout << "Escrito dentro del bloque. Saliendo...\n";
    } // El destructor de ofstream se llama aquí

    // Verificamos que el archivo existe y está completo
    ifstream archivo("raii.txt");
    if (archivo.good())
        cout << "El archivo se cerró correctamente via RAII.\n";
}

//==============================================================================
// EJEMPLO 6: MODO BINARIO
//==============================================================================
/*
  ios::binary evita la traducción de caracteres de nueva línea (CR+LF a LF).
*/
void ejemplo_binario()
{
    cout << "\n--- EJEMPLO 6: Modo binario ---\n";

    ofstream archivo("binario.bin", ios::out | ios::binary);
    if (!archivo)
    {
        cerr << "Error al crear binario.\n";
        return;
    }

    int numeros[] = {100, 200, 300, 400, 500};
    archivo.write(reinterpret_cast<char*>(numeros), sizeof(numeros));
    archivo.close();
    cout << "Datos binarios escritos (5 enteros).\n";

    ifstream archivo_in("binario.bin", ios::in | ios::binary);
    if (!archivo_in)
    {
        cerr << "Error al leer binario.\n";
        return;
    }

    int leido[5];
    archivo_in.read(reinterpret_cast<char*>(leido), sizeof(leido));
    archivo_in.close();

    for (int i = 0; i < 5; ++i)
        cout << "  Número " << i + 1 << ": " << leido[i] << "\n";
}

// EJEMPLO 7: APERTURA EXPLÍCITA CON OPEN()
//==============================================================================
/*
  open() permite reutilizar el mismo objeto stream para varios archivos.
*/
void ejemplo_open_explicito()
{
    cout << "\n--- EJEMPLO 7: open() explícito ---\n";

    ofstream archivo;

    archivo.open("archivo_a.txt");
    if (archivo)
        archivo << "Este es el archivo A.\n";
    archivo.close();

    archivo.open("archivo_b.txt");
    if (archivo)
        archivo << "Este es el archivo B.\n";
    archivo.close();

    cout << "Se escribieron dos archivos con el mismo objeto ofstream.\n";
}

//==============================================================================
// EJEMPLO 8: VERIFICACIÓN DE APERTURA
//==============================================================================
/*
  Múltiples formas de verificar si un archivo se abrió correctamente.
*/
void ejemplo_verificacion()
{
    cout << "\n--- EJEMPLO 8: Verificación de apertura ---\n";

    ifstream archivo;

    // Método 1: is_open()
    archivo.open("archivo_inexistente.txt");
    if (!archivo.is_open())
        cout << "Método 1: is_open() detectó que no se pudo abrir.\n";

    // Método 2: operador bool()
    if (!archivo)
        cout << "Método 2: operator bool() detectó error.\n";

    // Método 3: fail()
    if (archivo.fail())
        cout << "Método 3: fail() true.\n";

    // Método 4: good()
    if (!archivo.good())
        cout << "Método 4: good() false.\n";
}

//==============================================================================
// EJEMPLO 9: COMBINACIÓN DE MODOS
//==============================================================================
/*
  Podemos combinar múltiples flags con el operador OR (|).
*/
void ejemplo_modos_combinados()
{
    cout << "\n--- EJEMPLO 9: Combinación de modos ---\n";

    // out | app | binary → escritura al final en modo binario
    ofstream archivo("combinado.bin", ios::out | ios::app | ios::binary);
    if (archivo)
    {
        int dato = 999;
        archivo.write(reinterpret_cast<char*>(&dato), sizeof(dato));
        archivo.close();
        cout << "Modo combinado: out | app | binary funcionó.\n";
    }

    // in | out | trunc → lectura/escritura con truncado
    fstream archivo2("combinado2.txt", ios::in | ios::out | ios::trunc);
    if (archivo2)
    {
        archivo2 << "Combinado con in | out | trunc.\n";
        archivo2.close();
        cout << "Modo combinado: in | out | trunc funcionó.\n";
    }
}

// EJEMPLO 10: TIPOS DE APERTURA Y SUS EFECTOS
//==============================================================================
/*
  Tabla práctica de modos de apertura y sus comportamientos.
*/
void ejemplo_tabla_modos()
{
    cout << "\n--- EJEMPLO 10: Comportamiento de modos ---\n";

    // Modo trunc (por defecto en ofstream)
    ofstream("test_trunc.txt", ios::out | ios::trunc) << "Truncado\n";
    cout << "  ios::trunc: archivo creado/vaciado.\n";

    // Modo ate (seek al final al abrir, pero se puede mover el puntero)
    ofstream("test_ate.txt", ios::out | ios::ate) << "Al final\n";
    cout << "  ios::ate: puntero al final tras apertura.\n";

    // Modo noreplace (C++23) — evita sobrescribir archivos existentes
    // Nota: no disponible en todos los compiladores.
    #if __cplusplus >= 202302L
    ofstream("test_noreplace.txt", ios::out | ios::noreplace);
    cout << "  ios::noreplace: no sobrescribe si existe.\n";
    #else
    cout << "  ios::noreplace: requiere C++23.\n";
    #endif
}

//==============================================================================
// EJEMPLO 11: EXCEPCIONES CON FLUJOS
//==============================================================================
/*
  Se pueden habilitar excepciones en los flujos de archivo.
*/
void ejemplo_excepciones()
{
    cout << "\n--- EJEMPLO 11: Excepciones en flujos ---\n";

    ifstream archivo;
    archivo.exceptions(ifstream::failbit | ifstream::badbit);

    try
    {
        archivo.open("archivo_que_no_existe.txt");
    }
    catch (const ifstream::failure& e)
    {
        cout << "Excepción capturada: " << e.what() << "\n";
        cout << "  Código: " << e.code() << "\n";
    }
}

// EJEMPLO 12: VERIFICAR SI UN ARCHIVO EXISTE
//==============================================================================
/*
  Función utilitaria para comprobar existencia sin abrirlo para escritura.
*/
bool archivo_existe(const string& nombre)
{
    ifstream f(nombre);
    return f.good();
}

void ejemplo_existe()
{
    cout << "\n--- EJEMPLO 12: Verificar existencia ---\n";

    cout << "  ¿ejemplo1.txt existe? " << (archivo_existe("ejemplo1.txt") ? "Sí" : "No") << "\n";
    cout << "  ¿no_existe.txt existe? " << (archivo_existe("no_existe.txt") ? "Sí" : "No") << "\n";
}

//==============================================================================
// EJEMPLO 13: BÚSQUEDA Y POSICIONAMIENTO
//==============================================================================
/*
  seekg() para posicionar el cursor de lectura (get pointer).
  seekp() para posicionar el cursor de escritura (put pointer).
*/
void ejemplo_posicionamiento()
{
    cout << "\n--- EJEMPLO 13: Posicionamiento ---\n";

    ofstream("pos.txt", ios::trunc) << "ABCDEFGHIJKLMNOPQRSTUVWXYZ\n";

    fstream archivo("pos.txt", ios::in | ios::out);
    if (!archivo) return;

    // seekp: coloca el puntero de escritura en la posición 5
    archivo.seekp(5, ios::beg);
    archivo << "12345";

    // seekg: coloca el puntero de lectura
    archivo.seekg(0, ios::end);
    streampos tamano = archivo.tellg();
    cout << "  Tamaño del archivo: " << tamano << " bytes.\n";

    archivo.seekg(0, ios::beg);
    string contenido;
    getline(archivo, contenido);
    cout << "  Contenido: " << contenido << "\n";

    archivo.close();
}

// EJEMPLO 14: MÚLTIPLES ARCHIVOS SIMULTÁNEOS
//==============================================================================
/*
  Se pueden tener varios flujos abiertos al mismo tiempo.
*/
void ejemplo_multiple()
{
    cout << "\n--- EJEMPLO 14: Múltiples archivos ---\n";

    ofstream log("log.txt");
    ofstream datos("datos.txt");
    ifstream entrada("ejemplo1.txt");

    if (log && datos && entrada)
    {
        log << "Inicio de sesión: " << __DATE__ << "\n";

        string linea;
        int cont = 0;
        while (getline(entrada, linea))
        {
            datos << "Dato_" << ++cont << ": " << linea << "\n";
            log << "Procesada línea " << cont << "\n";
        }

        log << "Fin de sesión.\n";
        cout << "  Archivos log.txt y datos.txt generados.\n";
    }
}

PREGUNTAS FRECUENTES (FAQ)
--------------------------
P: ¿Es necesario llamar a close() o el destructor lo hace?
R: El destructor cierra el archivo automáticamente (RAII). Sin embargo, es
   buena práctica llamar a close() si necesitas verificar errores de cierre
   (close() puede fallar en algunos sistemas).

P: ¿Cuál es la diferencia entre ios::trunc y ios::app?
R: trunc borra el contenido del archivo al abrirlo; app no borra nada y
   todas las escrituras van al final del archivo. trunc es el modo por
   defecto en ofstream; app requiere especificarlo explícitamente.

P: ¿Qué significa "if (!archivo)" en C++?
R: Las clases de flujo tienen un operador de conversión a bool que devuelve
   false si el flujo está en estado de error. Es equivalente a
   "if (archivo.fail())".

P: ¿Puedo usar un mismo objeto stream para leer y escribir?
R: Sí, usando fstream. Debes tener cuidado de posicionar correctamente los
   punteros seekg (lectura) y seekp (escritura) al cambiar de operación.

P: ¿Qué modo debo usar para archivos binarios?
R: ios::binary. Es importante en Windows para evitar que los caracteres
   0x0A (LF) se conviertan a 0x0D 0x0A (CRLF). En Linux no hay diferencia.

P: ¿Qué es el modo noreplace?
R: Disponible desde C++23, noreplace hace que la apertura falle si el
   archivo ya existe. Útil para evitar sobrescrituras accidentales.

P: ¿Puedo abrir múltiples archivos a la vez?
R: Sí, cada objeto stream maneja un archivo independiente. Puedes tener
   tantos abiertos como permita el sistema operativo.
