--- 1. AUTO: DEDUCCIÓN DE TIPOS ---

La palabra clave auto le indica al compilador que deduzca el tipo de una
variable a partir de su inicializador. Esto reduce la verbosidad y mejora
la mantenibilidad del código.

```cpp
#include <iostream>
#include <vector>
#include <map>
#include <string>
#include <typeinfo>

int main() {
    // Auto deduce el tipo del inicializador
    auto entero = 42;           // int
    auto flotante = 3.14;       // double
    auto caracter = 'A';        // char
    auto cadena = "Hola";       // const char*
    auto booleano = true;       // bool

    std::cout << "entero: " << entero << "\n";
    std::cout << "flotante: " << flotante << "\n";
    std::cout << "caracter: " << caracter << "\n";
    std::cout << "cadena: " << cadena << "\n";
    std::cout << "booleano: " << booleano << "\n";

    // Auto con expresiones complejas
    std::vector<int> numeros = {1, 2, 3, 4, 5};
    auto iterador = numeros.begin();  // std::vector<int>::iterator
    std::cout << "Primer elemento: " << *iterador << "\n";

    // Auto con mapas
    std::map<std::string, int> edades;
    edades["Ana"] = 25;
    edades["Luis"] = 30;
    for (auto par = edades.begin(); par != edades.end(); ++par) {
        std::cout << par->first << ": " << par->second << "\n";
    }

    return 0;
}
```

Reglas de deducción de auto:
- auto sin referencia ni calificador: descarta referencias y cv-qualifiers.
- auto& mantiene la referencia.
- const auto mantiene la const-cualificación.
- auto&& es una referencia universal (forwarding reference).

```cpp
#include <iostream>
#include <type_traits>

int main() {
    int x = 10;
    int& ref = x;
    const int cx = 20;
    const int& cref = cx;

    // Regla 1: auto sin calificadores descarta referencias y const
    auto a1 = ref;   // int (referencia descartada)
    auto a2 = cx;    // int (const descartado)
    auto a3 = cref;  // int (referencia y const descartados)

    // Regla 2: auto& mantiene calificadores
    auto& a4 = ref;   // int&
    const auto& a5 = cref;  // const int&

    // Verificación de tipos
    std::cout << std::is_same<decltype(a1), int>::value << "\n";       // 1
    std::cout << std::is_same<decltype(a4), int&>::value << "\n";      // 1
    std::cout << std::is_same<decltype(a5), const int&>::value << "\n"; // 1

    return 0;
}
```

Auto en funciones (C++14 en adelante permite auto en return):

```cpp
#include <iostream>

// C++11: auto en trailing return type
auto suma(int a, int b) -> int {
    return a + b;
}

// C++14: auto deduce el tipo de retorno
auto producto(double a, double b) {
    return a * b;
}

int main() {
    std::cout << "Suma: " << suma(3, 4) << "\n";
    std::cout << "Producto: " << producto(2.5, 4.0) << "\n";
    return 0;
}
```

Auto con arrays:

```cpp
#include <iostream>

int main() {
    int arr[] = {1, 2, 3, 4, 5};
    auto arr2 = arr;  // int* (decae a puntero)

    // Para evitar la decadencia, usar referencia
    auto& arr3 = arr;  // int(&)[5]

    std::cout << "arr2[0]: " << arr2[0] << "\n";
    std::cout << "sizeof(arr3): " << sizeof(arr3) << "\n";

    return 0;
}
```

--- 2. DECLTYPE: CONSULTA DE TIPOS ---

decltype inspecciona el tipo de una expresión sin evaluarla. Es útil en
plantillas y cuando se necesita declarar tipos basados en expresiones.

```cpp
#include <iostream>
#include <vector>
#include <typeinfo>

int funcion() { return 42; }

int main() {
    int x = 10;
    const int y = 20;
    int& ref = x;
    const int& cref = y;

    // decltype de variables
    decltype(x) a = 5;        // int
    decltype(y) b = 30;       // const int
    decltype(ref) c = x;      // int&
    decltype(cref) d = y;     // const int&

    // decltype de expresiones
    decltype(funcion()) e = funcion();  // int
    decltype((x)) f = x;                // int& (paréntesis doble)

    // decltype con auto en trailing return type
    auto suma = [](auto a, auto b) -> decltype(a + b) {
        return a + b;
    };

    std::cout << suma(5, 3) << "\n";
    std::cout << suma(2.5, 3.7) << "\n";

    // Usando decltype con std::is_same
    std::cout << std::is_same<decltype(a), int>::value << "\n";

    return 0;
}
```

Reglas de decltype:
1. Si la expresión es un nombre de variable sin paréntesis, decltype devuelve
   el tipo declarado de esa variable.
2. Si la expresión es un llamado a función, devuelve el tipo de retorno.
3. Si la expresión está entre paréntesis (x), es una expresión lvalue y
   devuelve T&.
4. Si la expresión es un rvalue, devuelve T.

```cpp
#include <iostream>
#include <type_traits>

struct Widget {
    int valor;
};

int main() {
    Widget w{42};
    const Widget& cw = w;

    // Caso 1: nombre de variable
    decltype(w) w2{10};   // Widget
    decltype(cw) cw2 = w; // const Widget&

    // Caso 2: acceso a miembro
    decltype(w.valor) v = 5;  // int

    // Caso 3: paréntesis doble → lvalue
    decltype((w.valor)) ref_v = w.valor;  // int&

    // decltype en contexto de plantillas
    auto comparar = [](auto a, auto b) -> decltype(a < b) {
        return a < b;
    };

    std::cout << std::boolalpha;
    std::cout << comparar(5, 3) << "\n";      // false
    std::cout << comparar(2.5, 3.7) << "\n"; // true

    return 0;
}
```

decltype(auto) (C++14):

```cpp
#include <iostream>

int x = 42;
int& getRef() { return x; }

// decltype(auto) preserva referencias exactamente
decltype(auto) getRef2() { return getRef(); }  // int&

auto getRef3() { return getRef(); }  // int (pierde referencia)

int main() {
    std::cout << "x antes: " << x << "\n";
    getRef2() = 100;
    std::cout << "x despues: " << x << "\n";

    // getRef3() = 200;  // Error: rvalue
    return 0;
}
```

--- 3. NULLPTR: EL PUNTERO NULO SEGURO ---

nullptr reemplaza NULL y (void*)0 con un tipo seguro que no se convierte
implícitamente a tipos enteros.

```cpp
#include <iostream>
#include <type_traits>

void funcion(int) {
    std::cout << " llamado con int\n";
}

void funcion(void*) {
    std::cout << " llamado con void*\n";
}

void funcion(std::nullptr_t) {
    std::cout << " llamado con nullptr\n";
}

int main() {
    // nullptr es de tipo std::nullptr_t
    auto p = nullptr;
    std::cout << "Tipo de nullptr: "
              << std::is_same<decltype(p), std::nullptr_t>::value << "\n";

    // nullptr resuelve ambigüedades que NULL no puede
    funcion(0);        // llama a funcion(int)
    funcion(NULL);     // llama a funcion(int) — ¡puede ser confuso!
    funcion(nullptr);  // llama a funcion(std::nullptr_t)

    // nullptr se puede asignar a cualquier puntero
    int* pi = nullptr;
    double* pd = nullptr;
    char* pc = nullptr;
    void* pv = nullptr;

    // nullptr se puede comparar
    if (pi == nullptr) {
        std::cout << "pi es nulo\n";
    }

    // nullptr no se convierte a entero
    // bool b = nullptr;  // Error en algunos compiladores
    // int i = nullptr;   // Error

    return 0;
}
```

Ventajas de nullptr sobre NULL:

```cpp
#include <iostream>
#include <cstring>
#include <memory>

class String {
    char* data;
public:
    // Constructor con puntero
    String(const char* str) {
        if (str == nullptr) {
            data = new char[1];
            data[0] = '\0';
        } else {
            data = new char[std::strlen(str) + 1];
            std::strcpy(data, str);
        }
        std::cout << "Constructor: " << (data ? data : "null") << "\n";
    }

    // Constructor con nullptr_t evita ambigüedades
    String(std::nullptr_t) : String(static_cast<const char*>(nullptr)) {}

    ~String() { delete[] data; }
};

void buscar(const std::string& text) {
    std::cout << "Buscando texto...\n";
}

void buscar(int* ptr) {
    std::cout << "Buscando con puntero...\n";
}

int main() {
    // Sin ambigüedad
    String s1("Hola");
    String s2(nullptr);

    // Llamadas claras
    buscar("Hola");     // string
    buscar(nullptr);    // int* (mejor que NULL que sería ambiguo)

    // nullptr en smart pointers
    std::unique_ptr<int> up = nullptr;
    std::shared_ptr<double> sp = nullptr;

    if (!up) {
        std::cout << "unique_ptr es nulo\n";
    }

    return 0;
}
```

--- 4. RANGE-BASED FOR LOOP ---

El bucle for basado en rango simplifica la iteración sobre contenedores.

```cpp
#include <iostream>
#include <vector>
#include <map>
#include <string>
#include <array>
#include <initializer_list>

int main() {
    // Con vector
    std::vector<int> numeros = {10, 20, 30, 40, 50};
    std::cout << "Vector: ";
    for (int n : numeros) {
        std::cout << n << " ";
    }
    std::cout << "\n";

    // Con array nativo
    int arr[] = {1, 2, 3, 4, 5};
    std::cout << "Array: ";
    for (int x : arr) {
        std::cout << x << " ";
    }
    std::cout << "\n";

    // Con initializer_list
    std::cout << "Lista: ";
    for (auto x : {1, 2, 3, 4, 5}) {
        std::cout << x << " ";
    }
    std::cout << "\n";

    // Con referencia para evitar copias
    std::vector<std::string> palabras = {"hola", "mundo", "c++"};
    for (const auto& p : palabras) {
        std::cout << p << " ";
    }
    std::cout << "\n";

    // Modificar elementos con referencia
    for (auto& n : numeros) {
        n *= 2;
    }
    std::cout << "Duplicado: ";
    for (const auto& n : numeros) {
        std::cout << n << " ";
    }
    std::cout << "\n";

    // Con mapas
    std::map<std::string, int> edades = {
        {"Ana", 25}, {"Luis", 30}, {"Pedro", 35}
    };
    for (const auto& [nombre, edad] : edades) {  // C++17 structured bindings
        std::cout << nombre << ": " << edad << "\n";
    }

    // Con arrays de std::array
    std::array<double, 4> valores = {1.1, 2.2, 3.3, 4.4};
    for (double v : valores) {
        std::cout << v << " ";
    }
    std::cout << "\n";

    return 0;
}
```

Cómo funciona range-based for internamente:

```cpp
#include <iostream>
#include <vector>

// Equivalente interno de:
//   for (auto x : container) { ... }
// se transforma a:
//   auto __begin = begin(container);
//   auto __end = end(container);
//   for (; __begin != __end; ++__begin) {
//       auto x = *__begin;
//       ...
//   }

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // Lo que el compilador genera internamente
    auto __begin = std::begin(v);
    auto __end = std::end(v);
    for (; __begin != __end; ++__begin) {
        auto x = *__begin;
        std::cout << x << " ";
    }
    std::cout << "\n";

    // Versión simplificada con range-based for
    for (auto x : v) {
        std::cout << x << " ";
    }
    std::cout << "\n";

    return 0;
}
```

--- 5. OVERRIDE Y FINAL ---

override asegura que una función virtual está realmente sobrescribiendo una
función base. final evita que una clase sea heredada o que un método virtual
sea sobrescrito.

```cpp
#include <iostream>
#include <memory>

class Base {
public:
    virtual void mostrar() const {
        std::cout << "Base::mostrar()\n";
    }

    virtual void calcular(int x) {
        std::cout << "Base::calcular(" << x << ")\n";
    }

    virtual ~Base() = default;
};

class Derivada : public Base {
public:
    // override verifica que existe la función base
    void mostrar() const override {
        std::cout << "Derivada::mostrar()\n";
    }

    // Sin override, esto sería una sobrecarga (nueva función)
    void calcular(double x) {
        std::cout << "Derivada::calcular(" << x << ")\n";
    }

    // Error si la firma no coincide con la base:
    // void calcular(int x) override { }  // OK
    // void calcular(short x) override { }  // Error: no sobrescribe
};

class BaseFinal {
public:
    virtual void metodoFinal() final {
        std::cout << "BaseFinal::metodoFinal()\n";
    }
};

class IntentaSobrescribir : public BaseFinal {
public:
    // Error: metodoFinal es final en BaseFinal
    // void metodoFinal() override { }
};

class ClaseFinal final {
    // Esta clase no puede ser heredada
};

// Error: no se puede heredar de ClaseFinal
// class IntentaHeredar : public ClaseFinal { };

int main() {
    std::unique_ptr<Base> ptr = std::make_unique<Derivada>();
    ptr->mostrar();  // Derivada::mostrar()
    ptr->calcular(5);  // Base::calcular(5)

    Derivada d;
    d.mostrar();     // Derivada::mostrar()
    d.calcular(3.14); // Derivada::calcular(double)
    d.calcular(5);    // ¿cuál se llama? Derivada::calcular(double) por promoción

    return 0;
}
```

--- 6. DEFAULT Y DELETE ---

default fuerza la generación del constructor/destructor por defecto.
delete elimina funciones miembro específicas.

```cpp
#include <iostream>
#include <memory>

class GestorRecursos {
public:
    int* datos;
    size_t tamano;

    // Constructor por defecto explícito
    GestorRecursos() = default;

    // Constructor con parámetros
    GestorRecursos(size_t n) : tamano(n) {
        datos = new int[tamano]();
        std::cout << "Recursos asignados: " << tamano << "\n";
    }

    // Copy constructor eliminado — no se pueden copiar recursos
    GestorRecursos(const GestorRecursos&) = delete;

    // Copy assignment eliminado
    GestorRecursos& operator=(const GestorRecursos&) = delete;

    // Move constructor (C++11)
    GestorRecursos(GestorRecursos&& otro) noexcept
        : datos(otro.datos), tamano(otro.tamano) {
        otro.datos = nullptr;
        otro.tamano = 0;
        std::cout << "Recursos movidos\n";
    }

    // Move assignment
    GestorRecursos& operator=(GestorRecursos&& otro) noexcept {
        if (this != &otro) {
            delete[] datos;
            datos = otro.datos;
            tamano = otro.tamano;
            otro.datos = nullptr;
            otro.tamano = 0;
            std::cout << "Recursos movidos (asignacion)\n";
        }
        return *this;
    }

    // Destructor por defecto explícito
    ~GestorRecursos() {
        delete[] datos;
        std::cout << "Recursos liberados\n";
    }
};

// Clase con成员特化
class SoloMovible {
public:
    SoloMovible() = default;
    SoloMovible(const SoloMovible&) = delete;
    SoloMovible& operator=(const SoloMovible&) = delete;
    SoloMovible(SoloMovible&&) = default;
    SoloMovible& operator=(SoloMovible&&) = default;
};

// Eliminar conversiones implícitas
class Entero {
    int valor;
public:
    explicit Entero(int v) : valor(v) {}

    // Eliminar conversiones de double
    Entero(double) = delete;
};

void soloInt(int x) {
    std::cout << "Entero: " << x << "\n";
}

void soloDouble(double) = delete;  // Error de enlace si se usa

int main() {
    GestorRecursos g1(100);
    GestorRecursos g2;  // Default constructor

    // g2 = g1;  // Error: operador de copia eliminado
    // GestorRecursos g3(g1);  // Error: copy constructor eliminado

    g2 = std::move(g1);  // OK: move assignment

    Entero e1(42);
    // Entero e2(3.14);  // Error: constructor con double eliminado

    return 0;
}
```

Más ejemplos de default/delete:

```cpp
#include <iostream>
#include <string>

class Archivo {
    std::string nombre;

    Archivo& operator=(Archivo&&) = default;

    ~Archivo() = default;

        std::cout << "Abriendo archivo: " << nombre << "\n";

// Eliminar new y delete personalizados
class SoloStack {

public:

    static void* operator new[](std::size_t) = delete;
    static void operator delete(void*) = delete;
    static void operator delete[](void*) = delete;

// Evitar copia implícita de parámetros
class Logger {
public:
    explicit Logger(const std::string& name) : nombre(name) {}

        std::cout << "[" << nombre << "] " << msg << "\n";

private:
    std::string nombre;
};

int main() {
    Archivo a1("datos.txt");
    Archivo a2 = std::move(a1);  // Move OK
    a2.abrir();

    SoloStack obj(42);
    // SoloStack* p = new SoloStack(10);  // Error: new eliminado

    // Logger log("test");
    // log.log("mensaje");  // Error: log(const char*) eliminado
    // log.log(std::string("mensaje"));  // OK

    return 0;
}
```

--- 7. CARACTERÍSTICAS ADICIONALES DE C++11 ---

static_assert y noexcept:

```cpp
#include <iostream>
#include <type_traits>

template<typename T>
class ContenedorSeguro {
    static_assert(std::is_default_constructible<T>::value,
                  "T debe ser default-constructible");

    T dato;
public:
    ContenedorSeguro() : dato() {}
};

// noexcept
void funcionSegura() noexcept {
    std::cout << "Esta funcion no lanza excepciones\n";
}

void funcionTalVez() noexcept(false) {
    std::cout << "Esta funcion puede lanzar excepciones\n";
}

int main() {
    static_assert(sizeof(int) == 4, "int debe ser de 4 bytes");
    static_assert(true, "Esto siempre pasa");

    ContenedorSeguro<int> c1;
    // ContenedorSeguro<int&> c2;  // Error: las referencias no son default-constructibles

    std::cout << std::boolalpha;
    std::cout << "noexcept(funcionSegura): " << noexcept(funcionSegura()) << "\n";
    std::cout << "noexcept(funcionTalVez): " << noexcept(funcionTalVez()) << "\n";

    return 0;
}
```

Enumeraciones fuertemente tipadas (enum class):

```cpp
#include <iostream>

enum class Color { Rojo, Verde, Azul };
enum class Estado { Activo, Inactivo, Pausado };

// enum tradicional tiene alcance global
enum Antiguo { OpcionA, OpcionB };

int main() {
    Color c = Color::Rojo;
    Estado e = Estado::Activo;

    // No hay conversión implícita a entero
    // int x = c;  // Error
    int x = static_cast<int>(c);  // OK

    // No hay conflicto de nombres (como con enum tradicional)
    std::cout << "Color: " << static_cast<int>(c) << "\n";
    std::cout << "Estado: " << static_cast<int>(e) << "\n";

    // enum class puede especificar el tipo subyacente
    enum class Byte : unsigned char { Min = 0, Max = 255 };
    std::cout << "Sizeof(Byte): " << sizeof(Byte) << "\n";

    return 0;
}
```

constexpr (C++11 básico):

```cpp
#include <iostream>

// Función constexpr evaluable en compilación
constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}

// C++11: constexpr limitado a una sola sentencia return
constexpr int cuadrado(int x) {
    return x * x;
}

int main() {
    // Evaluado en tiempo de compilación
    constexpr int valor = factorial(5);
    static_assert(valor == 120, "factorial(5) debe ser 120");

    // También funciona en runtime
    int n = 5;
    std::cout << "factorial(" << n << ") = " << factorial(n) << "\n";
    std::cout << "cuadrado(10) = " << cuadrado(10) << "\n";

    // Array con tamaño constexpr
    int arr[factorial(4)];  // arr[24]
    std::cout << "sizeof(arr): " << sizeof(arr) / sizeof(arr[0]) << "\n";

    return 0;
}
```

--- 8. EJEMPLO INTEGRADOR ---

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <memory>
#include <algorithm>

class Empleado final {
    std::string nombre;
    int id;
    double salario;

public:
    Empleado() = delete;  // No default construction

    Empleado(std::string nom, int id, double sal)
        : nombre(std::move(nom)), id(id), salario(sal) {}

    // Rule of five con default/delete
    Empleado(const Empleado&) = default;
    Empleado& operator=(const Empleado&) = default;
    Empleado(Empleado&&) = default;
    Empleado& operator=(Empleado&&) = default;
    ~Empleado() = default;

    void mostrar() const {
        std::cout << "ID: " << id << ", Nombre: " << nombre
                  << ", Salario: $" << salario << "\n";
    }

    double getSalario() const { return salario; }
    void setSalario(double s) { salario = s; }
    int getID() const { return id; }
};

class Departamento final {
    std::string nombre;
    std::vector<std::unique_ptr<Empleado>> empleados;

public:
    explicit Departamento(std::string nom) : nombre(std::move(nom)) {}

    Departamento(const Departamento&) = delete;
    Departamento& operator=(const Departamento&) = delete;
    Departamento(Departamento&&) = default;
    Departamento& operator=(Departamento&&) = default;

    void contratar(std::string nom, int id, double sal) {
        empleados.push_back(
            std::make_unique<Empleado>(std::move(nom), id, sal));
    }

    void mostrarEmpleados() const {
        std::cout << "\n=== " << nombre << " ===\n";
        for (const auto& emp : empleados) {
            emp->mostrar();
        }
    }

    void aplicarAumento(double porcentaje) {
        for (auto& emp : empleados) {
            double nuevoSal = emp->getSalario() * (1.0 + porcentaje / 100.0);
            emp->setSalario(nuevoSal);
        }
    }

    auto begin() const { return empleados.begin(); }
    auto end() const { return empleados.end(); }
};

int main() {
    Departamento dept("Ingenieria");

    dept.contratar("Ana Garcia", 1001, 75000);
    dept.contratar("Luis Perez", 1002, 82000);
    dept.contratar("Carlos Ruiz", 1003, 68000);

    dept.mostrarEmpleados();

    std::cout << "\nAplicando aumento del 10%...\n";
    dept.aplicarAumento(10.0);
    dept.mostrarEmpleados();

    // nullptr en lugar de NULL
    Empleado* ptr = nullptr;
    if (ptr == nullptr) {
        std::cout << "\nPuntero nulo (seguro con nullptr)\n";
    }

    return 0;
}
```

--- 9. COMPARATIVA C++03 vs C++11 ---

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <algorithm>

// C++03 style
class VectorOld {
    int* data;
    size_t sz;
public:
    VectorOld() : data(NULL), sz(0) {}  // NULL inseguro
    explicit VectorOld(size_t n) : data(new int[n]), sz(n) {}
    ~VectorOld() { delete[] data; }
    // Sin move semantics, sin delete
};

// C++11 style
class VectorNew {
    std::unique_ptr<int[]> data;
    size_t sz;
public:
    VectorNew() : data(nullptr), sz(0) {}  // nullptr seguro
    explicit VectorNew(size_t n)
        : data(std::make_unique<int[]>(n)), sz(n) {}

    void fill(int v) {
        for (size_t i = 0; i < sz; ++i) data[i] = v;
    }

    void mostrar() const {
        for (size_t i = 0; i < sz; ++i) std::cout << data[i] << " ";
        std::cout << "\n";
    }
};

int main() {
    std::cout << "=== C++11 vs C++03 ===\n";

    // C++03: iteración verbosa
    std::vector<int> vec = {1, 2, 3, 4, 5};
    std::cout << "C++03 style: ";
    for (std::vector<int>::iterator it = vec.begin();
         it != vec.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << "\n";

    // C++11: range-based for con auto
    std::cout << "C++11 style: ";
    for (const auto& v : vec) {
        std::cout << v << " ";
    }
    std::cout << "\n";

    // nullptr vs NULL
    int* p1 = NULL;      // C++03: puede causar ambigüedad
    int* p2 = nullptr;   // C++11: seguro y explícito

    VectorNew vn(5);
    vn.fill(42);
    vn.mostrar();

    return 0;
}
```

Hemos cubierto las características fundamentales de C++11:

• auto: Deducción automática de tipos que reduce verbosidad y mejora
  mantenibilidad. Sigue reglas específicas de decaimiento de tipos.

• decltype: Consulta de tipos en tiempo de compilación sin evaluar
  expresiones. Esencial en metaprogramación y plantillas.

• nullptr: Reemplazo type-safe para NULL y (void*)0. Evita ambigüedades
  en sobrecarga de funciones y conversiones implícitas a enteros.

• Range-based for: Iteración simplificada sobre contenedores usando
  begin/end. Soporta arrays, contenedores STL e initializer_lists.

• override: Verifica en compilación que un método virtual realmente
  sobrescribe una función base, previniendo errores de firma.

• final: Impide la sobrescritura de métodos virtuales o la herencia
  de clases, permitiendo diseñar jerarquías cerradas.

• default/delete: Control explícito sobre la generación de miembros
  especiales (constructores, destructores, operadores de copia/movimiento).

Estas características forman la base del C++ Moderno y son prerrequisito
para los temas avanzados de los próximos capítulos.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿auto deduce siempre el tipo exacto del inicializador?
R: No. auto sin calificadores descarta referencias y cv-qualifiers
(const/volatile). Para preservarlos, usa auto&, const auto, etc.

P: ¿Cuándo debo usar nullptr en lugar de NULL?
R: Siempre. nullptr es type-safe, no se convierte implícitamente a
entero, y resuelve correctamente las sobrecargas de funciones.

P: ¿override es obligatorio en C++11?
R: No es obligatorio, pero es altamente recomendado. El compilador
genera un error si el método marcado con override no sobrescribe
ninguna función base, previniendo errores sutiles.

P: ¿Qué diferencia hay entre delete y default?
R: default fuerza al compilador a generar la implementación por defecto
de un miembro especial. delete elimina una función, causando error de
compilación si se intenta usar.

P: ¿decltype y auto son intercambiables?
R: No. auto deduce el tipo de una variable. decltype inspecciona el tipo
de una expresión. decltype(auto) combina ambos (C++14).

P: ¿Range-based for funciona con cualquier tipo?
R: Funciona con tipos que tienen begin()/end() (miembros o funciones
libres) o con arrays de tamaño conocido.
