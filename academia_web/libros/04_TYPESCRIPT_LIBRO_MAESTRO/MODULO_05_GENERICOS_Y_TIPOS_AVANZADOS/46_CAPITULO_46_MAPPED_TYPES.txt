BIENVENIDA DEL PROFESOR BÚHO

Querido estudiante,

Hasta ahora hemos trabajado con tipos estáticos. Pero que pasaria si te
dijera que TypeScript permite transformar tipos programaticamente, igual
que `Array.map()` transforma arreglos?

Los **Mapped Types** son exactamente eso: una sintaxis que itera sobre las
claves de un tipo y produce un nuevo tipo basado en esas claves. Son el
motor interno de utility types como `Partial<T>`, `Readonly<T>`, `Pick<T,K>`,
y `Record<K,V>`.

La sintaxis `{ [K in keyof T]: T[K] }` puede parecer critica al principio,
pero en este capitulo la desglosaremos pieza por pieza. Una vez que
domines los Mapped Types, podras crear tus propios utility types,
transformar tipos de forma declarativa y construir abstracciones que antes
requerian bibliotecas externas.

===============================================================================
1. SINTAXIS FUNDAMENTAL
===============================================================================

Un Mapped Type itera sobre las claves de un tipo existente y construye
un nuevo tipo. La sintaxis base es:

```typescript
type MappedType<T> = {
  [K in keyof T]: T[K];
};
```

**Desglose:**

- `[K in keyof T]`: Para cada clave K dentro de las claves de T...
- `T[K]`: El valor de esa clave (se mantiene igual)
- El resultado es un tipo identico al original

```typescript
interface Persona {
  nombre: string;
  edad: number;
  activo: boolean;
}

type PersonaMapeada = {
  [K in keyof Persona]: Persona[K];
};
// Equivalente a Persona original:
// { nombre: string; edad: number; activo: boolean }
```

**Con genericos:**

```typescript
type Copia<T> = {
  [K in keyof T]: T[K];
};

type CopiaPersona = Copia<Persona>;
// { nombre: string; edad: number; activo: boolean }
```

**Analogia:** Si `T` es un edificio con habitaciones, `keyof T` te da
los nombres de las habitaciones, y `T[K]` te da el contenido de cada una.
Un Mapped Type recorre todas las habitaciones y decide que hacer con cada una.

2. ANIADIENDO MODIFICADORES: OPCIONAL Y READONLY
===============================================================================

Podemos anadir los modificadores `?` (opcional) y `readonly`:

```typescript
// Hacer todas las propiedades opcionales
type Opcional<T> = {
  [K in keyof T]?: T[K];
};

type PersonaOpcional = Opcional<Persona>;
// { nombre?: string; edad?: number; activo?: boolean }

// Hacer todas las propiedades de solo lectura
type SoloLectura<T> = {
  readonly [K in keyof T]: T[K];
};

type PersonaLectura = SoloLectura<Persona>;
// { readonly nombre: string; readonly edad: number; readonly activo: boolean }

// Combinar ambos
type OpcionalLectura<T> = {
  readonly [K in keyof T]?: T[K];
};

type PersonaOpLectura = OpcionalLectura<Persona>;
// { readonly nombre?: string; readonly edad?: number; readonly activo?: boolean }
```

**Eliminar modificadores con `-?` y `-readonly`:**

```typescript
// Requerir todas las propiedades (eliminar opcional)
type Requerido<T> = {
  [K in keyof T]-?: T[K];
};

// Modificable (eliminar readonly)
type Modificable<T> = {
  -readonly [K in keyof T]: T[K];
};

// Ambos
type Normalizado<T> = {
  -readonly [K in keyof T]-?: T[K];
};

interface ConfigOpcional {
  url?: string;
  readonly timeout?: number;
}

type ConfigNormal = Normalizado<ConfigOpcional>;
// { url: string; timeout: number }
```

**Nota:** `Partial<T>` y `Required<T>` de TypeScript son exactamente
implementados asi internamente.

===============================================================================
3. MODIFICANDO TIPOS DE PROPIEDADES
===============================================================================

Podemos transformar el tipo de cada propiedad a algo diferente:

```typescript
// Convertir todas las propiedades a strings
type ToString<T> = {
  [K in keyof T]: string;
};

type PersonaString = ToString<Persona>;
// { nombre: string; edad: string; activo: string }

// Convertir todas las propiedades a nullables
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};

type PersonaNullable = Nullable<Persona>;
// { nombre: string | null; edad: number | null; activo: boolean | null }

// Envolver en Promise
type Asyncronico<T> = {
  [K in keyof T]: Promise<T[K]>;
};

type PersonaAsync = Asyncronico<Persona>;
// { nombre: Promise<string>; edad: Promise<number>; activo: Promise<boolean> }

// Envolver en arreglo
type ArregloDe<T> = {
  [K in keyof T]: T[K][];
};

type PersonaArreglo = ArregloDe<Persona>;
// { nombre: string[]; edad: number[]; activo: boolean[] }

// Hacer getters (funciones que devuelven el valor)
type Getters<T> = {
  [K in keyof T]: () => T[K];
};

type PersonaGetters = Getters<Persona>;
// { nombre: () => string; edad: () => number; activo: () => boolean }
```

**Transformacion condicional por tipo de valor:**

```typescript
type TransformarStrings<T> = {
  [K in keyof T]: T[K] extends string ? Date : T[K];
};

interface Evento {
  titulo: string;
  fecha: string;
  asistentes: number;
}

type EventoTransformado = TransformarStrings<Evento>;
// { titulo: string; fecha: Date; asistentes: number }
```

4. USANDO AS PARA RENOMBRAR CLAVES
===============================================================================

Desde TypeScript 4.1, podemos usar `as` para renombrar claves:

```typescript
// Anadir prefijo a todas las propiedades
type ConPrefijo<T, Prefijo extends string> = {
  [K in keyof T as `${Prefijo}${Capitalize<string & K>}`]: T[K];
};

interface Usuario {
  nombre: string;
  edad: number;
}

type UsuarioAPI = ConPrefijo<Usuario, "api">;
// { apiNombre: string; apiEdad: number }
```

**Renombrar con sufijo:**

```typescript
type ConSufijo<T, Sufijo extends string> = {
  [K in keyof T as `${string & K}${Capitalize<Sufijo>}`]: T[K];
};

type UsuarioDB = ConSufijo<Usuario, "DB">;
// { nombreDB: string; edadDB: number }
```

**Mapear claves a un formato especifico:**

```typescript
type ToGetter<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

type UsuarioGetters = ToGetter<Usuario>;
// { getNombre: () => string; getEdad: () => number }

type ToSetter<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (valor: T[K]) => void;
};

type UsuarioSetters = ToSetter<Usuario>;
// { setNombre: (valor: string) => void; setEdad: (valor: number) => void }
```

===============================================================================
5. FILTRANDO CLAVES CON AS Y CONDICIONALES
===============================================================================

Podemos usar `as` junto con tipos condicionales para filtrar claves:

```typescript
// Solo propiedades que sean string
type PropiedadesString<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

interface Mezcla {
  id: number;
  nombre: string;
  activo: boolean;
  email: string;
}

type SoloStrings = PropiedadesString<Mezcla>;
// { nombre: string; email: string }

// Solo propiedades que sean funciones
type PropiedadesFuncion<T> = {
  [K in keyof T as T[K] extends (...args: any[]) => any ? K : never]: T[K];
};

interface Servicio {
  nombre: string;
  iniciar(): void;
  detener(): void;
  version: number;
}

type Metodos = PropiedadesFuncion<Servicio>;
// { iniciar: () => void; detener: () => void }
```

**Filtrar por nombre de clave:**

```typescript
// Solo propiedades que comienzan con "get"
type PropiedadesGet<T> = {
  [K in keyof T as K extends `get${string}` ? K : never]: T[K];
};

interface Controlador {
  getUsuario(): void;
  setUsuario(): void;
  getProducto(): void;
  eliminar(): void;
}

type SoloGetters = PropiedadesGet<Controlador>;
// { getUsuario: () => void; getProducto: () => void }
```

===============================================================================
6. MAPEANDO SOBRE UNIONES Y OTROS TIPOS
===============================================================================

Los Mapped Types no solo funcionan con objetos, tambien con uniones:

```typescript
// Mapear una union a un objeto
type EventoMap = {
  [K in "click" | "focus" | "blur"]: { tipo: K; timestamp: number };
};

// Equivalente a:
// {
//   click: { tipo: "click"; timestamp: number };
//   focus: { tipo: "focus"; timestamp: number };
//   blur: { tipo: "blur"; timestamp: number };
// }
```

**Record<K, V> es un Mapped Type:**

```typescript
type Record<K extends keyof any, V> = {
  [P in K]: V;
};

type Roles = "admin" | "user" | "guest";
type Permisos = Record<Roles, string[]>;
// { admin: string[]; user: string[]; guest: string[] }
```

**Mapped Types sobre tuplas:**

```typescript
type NullableTupla<T extends any[]> = {
  [K in keyof T]: T[K] | null;
};

type TuplaOriginal = [string, number, boolean];
type TuplaNullable = NullableTupla<TuplaOriginal>;
// [string | null, number | null, boolean | null]
```

===============================================================================
7. DEEP MAPPED TYPES (RECURSIVOS)
===============================================================================

Los Mapped Types pueden ser recursivos para tipos anidados:

```typescript
// DeepPartial: hace opcionales todas las propiedades recursivamente
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object ? DeepPartial<T[K]> : T[K];
};

interface Arbol {
  valor: number;
  hijo?: Arbol;
  datos: {
    nombre: string;
    etiquetas: string[];
  };
}

type ArbolParcial = DeepPartial<Arbol>;
// {
//   valor?: number;
//   hijo?: DeepPartial<Arbol>;
//   datos?: {
//     nombre?: string;
//     etiquetas?: string[];
//   };
// }

// DeepReadonly: hace readonly todas las propiedades recursivamente
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepReadonly<T[K]>
    : T[K];
};

// DeepRequired: quita opcionales recursivamente
type DeepRequired<T> = {
  [K in keyof T]-?: T[K] extends object ? DeepRequired<T[K]> : T[K];
};
```

===============================================================================
8. MAPEANDO CON MODIFICADORES EN CLASES ESPECIFICAS
===============================================================================

Podemos marcar/desmarcar propiedades especificas:

```typescript
type SoloLecturaKeys<T, K extends keyof T> = {
  readonly [P in keyof T]: P extends K ? T[P] : T[P];
};

interface Documento {
  id: number;
  titulo: string;
  contenido: string;
  autor: string;
}

type DocumentoSeguro = SoloLecturaKeys<Documento, "id" | "autor">;
// {
//   readonly id: number;
//   titulo: string;
//   contenido: string;
//   readonly autor: string;
// }
```

**Opcionalizar solo algunas propiedades:**

```typescript
type OpcionalKeys<T, K extends keyof T> = {
  [P in keyof T]: P extends K ? T[P] | undefined : T[P];
} & {};

interface Config {
  url: string;
  timeout: number;
  retries: number;
}

type ConfigConOpcionales = OpcionalKeys<Config, "timeout" | "retries">;
// { url: string; timeout: number | undefined; retries: number | undefined; }
```

9. MAPEANDO CON TIPOS CONDICIONALES AVANZADOS
===============================================================================

```typescript
// Extraer tipos de promesas
type UnwrapPromises<T> = {
  [K in keyof T]: T[K] extends Promise<infer V> ? V : T[K];
};

interface Respuestas {
  usuario: Promise<{ id: number }>;
  productos: Promise<{ items: string[] }>;
  config: { version: string };
}

type RespuestasResueltas = UnwrapPromises<Respuestas>;
// {
//   usuario: { id: number };
//   productos: { items: string[] };
//   config: { version: string };
// }
```

**Anadir metadata a cada propiedad:**

```typescript
type ConMetadata<T> = {
  [K in keyof T]: {
    valor: T[K];
    metadata: {
      clave: string;
      tipo: string;
      opcional: K extends keyof T ? false : true;
    };
  };
};

interface Producto {
  nombre: string;
  precio: number;
}

type ProductoMeta = ConMetadata<Producto>;
// {
//   nombre: { valor: string; metadata: { clave: string; tipo: string; opcional: false } };
//   precio: { valor: number; metadata: { clave: string; tipo: string; opcional: false } };
// }
```

10. IMPLEMENTACION DE UTILITY TYPES DESDE CERO
===============================================================================

Ahora que entiendes Mapped Types, puedes implementar los utility types
de TypeScript por ti mismo:

```typescript
// Partial<T>
type MiPartial<T> = {
  [P in keyof T]?: T[P];
};

// Required<T>
type MiRequired<T> = {
  [P in keyof T]-?: T[P];
};

// Readonly<T>
type MiReadonly<T> = {
  readonly [P in keyof T]: T[P];
};

// Pick<T, K>
type MiPick<T, K extends keyof T> = {
  [P in K]: T[P];
};

// Record<K, V>
type MiRecord<K extends keyof any, V> = {
  [P in K]: V;
};

// Ejemplo de uso de implementacion propia
interface Tarea {
  id: number;
  titulo: string;
  completada: boolean;
}

type TareaOpcional = MiPartial<Tarea>;
type SoloIdTitulo = MiPick<Tarea, "id" | "titulo">;
type MapaRoles = MiRecord<"admin" | "user", Tarea[]>;
```

11. EJEMPLO INTEGRADOR: SISTEMA DE VALIDACION AUTOMATICO
===============================================================================

```typescript
// === Definicion base ===
interface ReglasValidacion<T> {
  [K in keyof T]?: (valor: T[K]) => string | null;
}

type ErroresValidacion<T> = {
  [K in keyof T]?: string;
};

type ResultadoValidacion<T> = {
  valido: boolean;
  errores: ErroresValidacion<T>;
};

// === Validador generico ===
class Validador<T extends object> {
  private reglas: ReglasValidacion<T>;

  constructor(reglas: ReglasValidacion<T>) {
    this.reglas = reglas;
  }

  validar(objeto: T): ResultadoValidacion<T> {
    const errores: ErroresValidacion<T> = {};

    for (const clave in this.reglas) {
      if (Object.prototype.hasOwnProperty.call(this.reglas, clave)) {
        const validador = this.reglas[clave] as (valor: any) => string | null;
        if (validador) {
          const error = validador(objeto[clave as keyof T]);
          if (error) {
            errores[clave as keyof T] = error;
          }
        }
      }
    }

    return {
      valido: Object.keys(errores).length === 0,
      errores
    };
  }

  // Metodo para obtener solo las propiedades con error
  obtenerErrores<K extends keyof T>(objeto: T): ErroresValidacion<T> {
    return this.validar(objeto).errores;
  }
}

// === Uso ===
interface FormularioContacto {
  nombre: string;
  email: string;
  mensaje: string;
  edad: number;
}

const reglasFormulario: ReglasValidacion<FormularioContacto> = {
  nombre: (v) => !v || v.length < 2 ? "Nombre muy corto" : null,
  email: (v) => !v || !v.includes("@") ? "Email invalido" : null,
  mensaje: (v) => !v || v.length < 10 ? "Mensaje muy corto" : null,
  edad: (v) => v < 18 ? "Debe ser mayor de edad" : null
};

const validador = new Validador<FormularioContacto>(reglasFormulario);

const resultado = validador.validar({
  nombre: "An",
  email: "invalido",
  mensaje: "Corto",
  edad: 15
});

console.log(resultado.valido);    // false
console.log(resultado.errores);
// {
//   nombre: "Nombre muy corto",
//   email: "Email invalido",
//   mensaje: "Mensaje muy corto",
//   edad: "Debe ser mayor de edad"
// }
```

12. MAPEANDO TIPOS DE FUNCIONES
===============================================================================

Podemos usar Mapped Types para transformar firmas de funciones:

```typescript
// Convertir metodos sincronos a asincronos
type AsyncMethods<T> = {
  [K in keyof T]: T[K] extends (...args: infer A) => infer R
    ? (...args: A) => Promise<R>
    : T[K];
};

interface ServicioLocal {
  obtenerUsuario(id: number): { nombre: string };
  guardarUsuario(usuario: { nombre: string }): boolean;
  version: string;
}

type ServicioRemoto = AsyncMethods<ServicioLocal>;
// {
//   obtenerUsuario: (id: number) => Promise<{ nombre: string }>;
//   guardarUsuario: (usuario: { nombre: string }) => Promise<boolean>;
//   version: string;
// }
```

**Convertir a observables (patron RxJS):**

```typescript
type ObservableMethods<T> = {
  [K in keyof T]: T[K] extends (...args: infer A) => infer R
    ? (...args: A) => { subscribe: (cb: (valor: R) => void) => void }
    : T[K];
};
```

13. PICK Y OMIT MANUALES CON MAPEADOS
===============================================================================

**Pick manual:**

```typescript
type MiPick<T, K extends keyof T> = {
  [P in K]: T[P];
};

// Omit manual (usando Pick + Exclude)
type MiOmit<T, K extends keyof T> = {
  [P in Exclude<keyof T, K>]: T[P];
};

// Otra forma de Omit con as
type MiOmitV2<T, K extends keyof T> = {
  [P in keyof T as P extends K ? never : P]: T[P];
};

interface Producto {
  id: number;
  nombre: string;
  precio: number;
  stock: number;
}

type SinStock = MiOmit<Producto, "stock">;
// { id: number; nombre: string; precio: number }
```

14. MAPEANDO CON MULTIPLES TIPOS DE ENTRADA
===============================================================================

Podemos combinar dos tipos en uno:

```typescript
// Fusionar dos tipos, con preferencia del segundo
type Fusionar<T, U> = {
  [K in keyof (T & U)]: K extends keyof U ? U[K] : K extends keyof T ? T[K] : never;
};

interface Base {
  id: number;
  creado: Date;
}

interface Formulario {
  nombre: string;
  email: string;
}

type UsuarioCompleto = Fusionar<Base, Formulario>;
// { id: number; creado: Date; nombre: string; email: string }
```

RESUMEN DEL CAPITULO
===============================================================================

- Los Mapped Types iteran sobre las claves de un tipo con `[K in keyof T]`.
- Los modificadores `?`, `readonly`, `-?`, `-readonly` controlan
  opcionalidad y mutabilidad.
- `as` permite renombrar y filtrar claves dentro del mapeo.
- Los Mapped Types pueden ser recursivos para tipos anidados.
- Son la base de todos los utility types de TypeScript.
- Combinados con tipos condicionales, permiten filtrado sofisticado.
- Patrones como `DeepPartial`, `Getters`, `AsyncMethods` son posibles
  gracias a los Mapped Types.

PREGUNTAS FRECUENTES
===============================================================================

**Que es `[K in keyof T]`?**
Es la sintaxis que itera sobre cada clave K dentro del conjunto de claves
de T. Es analogo a `for (const K in T)` pero en el sistema de tipos.

**Cual es la diferencia entre `{ [K in keyof T]: T[K] }` y `T`?**
Funcionalmente son identicos. El Mapped Type simplemente copia el tipo.
Pero la sintaxis de Mapped Type permite anadir modificadores y
transformaciones en el proceso de copia.

**Puedo mapear sobre cualquier tipo?**
Mapped Types funcionan mejor con tipos de objeto. Con uniones, el
comportamiento es diferente. Con primitivos, no producen nada util.

**Que significa `as` en un Mapped Type?**
`as` permite renombrar la clave actual durante la iteracion. Se usa
con template literals para crear nuevos nombres. Si produces `never`,
la clave se excluye del resultado.
