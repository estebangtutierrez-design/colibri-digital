Querido estudiante,

Hemos visto cómo `keyof` extrae las claves de un tipo. Ahora toca el
operador complementario: `typeof`. Mientras que `keyof` opera sobre
tipos, `typeof` opera sobre **valores** y produce su tipo en tiempo de
compilación.

¿Por qué es importante? Porque no siempre definimos tipos primero.
Muchas veces creamos objetos, funciones o clases en JavaScript, y luego
queremos extraer su tipo para usarlo en otras partes. `typeof` nos permite
hacer eso: obtener el tipo TypeScript de un valor existente.

En este capítulo veremos:
- `typeof` en el ámbito de tipos (type context)
- `typeof` combinado con `keyof` para máxima flexibilidad
- `ReturnType` y `Parameters` para extraer firmas de funciones
- `InstanceType` para obtener tipos de instancias de clases
- Patrones avanzados como `satisfies` y extracción de literales

Al final, serás capaz de navegar entre el mundo de los valores y el mundo
de los tipos con total fluidez.

1. TYPEOF EN JAVASCRIPT VS TYPEOF EN TYPESCRIPT
===============================================================================

Es crucial entender que `typeof` existe en dos contextos diferentes:

**En JavaScript (runtime):**

```javascript
console.log(typeof "hola");   // "string"
console.log(typeof 42);       // "number"
console.log(typeof {});       // "object"
console.log(typeof undefined); // "undefined"
```

Devuelve un **string** que indica el tipo en tiempo de ejecución.

**En TypeScript (compile-time / type context):**

```typescript
const mensaje = "hola mundo";
type TipoMensaje = typeof mensaje;
//    ^? type TipoMensaje = string

const numero = 42;
type TipoNumero = typeof numero;
//    ^? type TipoNumero = number
```

Devuelve el **tipo TypeScript** del valor en tiempo de compilación.

```typescript
// En JavaScript typeof devuelve un string
const jsType = typeof "hola";
//    ^? const jsType: string (en TS, el valor es "string")

// En TypeScript typeof devuelve un tipo
type tsType = typeof "hola";
//    ^? type tsType = "hola" (literal)
```

===============================================================================
2. TYPEOF CON VARIABLES Y OBJETOS
===============================================================================

```typescript
const nombre = "Ana";
type NombreType = typeof nombre;
//    ^? type NombreType = string

const edad = 30;
type EdadType = typeof edad;
//    ^? type EdadType = number

const usuario = {
  id: 1,
  nombre: "Carlos",
  activo: true,
  roles: ["admin", "user"]
};

type UsuarioType = typeof usuario;
//    ^? type UsuarioType = {
//         id: number;
//         nombre: string;
//         activo: boolean;
//         roles: string[];
//       }
```

**typeof con const assertions:**

```typescript
const config = {
  url: "https://api.com",
  puerto: 443,
  timeout: 5000
} as const;

type ConfigType = typeof config;
//    ^? type ConfigType = {
//         readonly url: "https://api.com";
//         readonly puerto: 443;
//         readonly timeout: 5000;
//       }

// Con "as const", los tipos son literales exactos, no ampliados
```

Esta es una de las combinaciones más poderosas: `typeof` + `as const`
permite crear tipos a partir de valores exactos.

3. TYPEOF CON ARREGLOS
===============================================================================

```typescript
const numeros = [1, 2, 3];
type TipoNumeros = typeof numeros;
//    ^? type TipoNumeros = number[]

const mezclado = [1, "dos", true];
type TipoMezclado = typeof mezclado;
//    ^? type TipoMezclado = (string | number | boolean)[]

const tupla = [1, "hola", true] as const;
type TipoTupla = typeof tupla;
//    ^? type TipoTupla = readonly [1, "hola", true]

// Acceder a elementos
type PrimerElemento = typeof tupla[0];
//    ^? type PrimerElemento = 1

type TipoElementos = typeof tupla[number];
//    ^? type TipoElementos = 1 | "hola" | true
```

**Patrón: extraer tipo de elementos de un arreglo**

```typescript
const colores = ["rojo", "verde", "azul"] as const;
type Color = typeof colores[number];
//    ^? type Color = "rojo" | "verde" | "azul"

function pintar(color: Color): void {
  console.log(`Pintando de ${color}`);
}

pintar("rojo");   // OK
pintar("verde");  // OK
// pintar("negro"); // Error
```

4. TYPEOF CON FUNCIONES
===============================================================================

```typescript
function sumar(a: number, b: number): number {
  return a + b;
}

type TipoFuncion = typeof sumar;
//    ^? type TipoFuncion = (a: number, b: number) => number

// Con funciones flecha
const multiplicar = (a: number, b: number): number => a * b;
type TipoMultiplicar = typeof multiplicar;
//    ^? type TipoMultiplicar = (a: number, b: number) => number
```

**typeof con funciones async:**

```typescript
async function fetchUser(id: number): Promise<{ name: string }> {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
}

type TipoFetchUser = typeof fetchUser;
//    ^? type TipoFetchUser = (id: number) => Promise<{ name: string }>
```

5. RETURNTYPE Y PARAMETERS
===============================================================================

Estos dos utility types, que ya vimos en el capítulo 43, se usan casi
siempre con `typeof`:

```typescript
function procesarDatos(
  id: number,
  config: { verbose: boolean }
): Promise<string[]> {
  return Promise.resolve(["dato1", "dato2"]);
}

type ParametrosProcesar = Parameters<typeof procesarDatos>;
//    ^? type ParametrosProcesar = [id: number, config: { verbose: boolean }]

type PrimerParam = Parameters<typeof procesarDatos>[0];
//    ^? type PrimerParam = number

type RetornoProcesar = ReturnType<typeof procesarDatos>;
//    ^? type RetornoProcesar = Promise<string[]>

type RetornoResuelto = Awaited<ReturnType<typeof procesarDatos>>;
//    ^? type RetornoResuelto = string[]
```

**Caso de uso: wrapper genérico de funciones**

```typescript
function crearLogger<T extends (...args: any[]) => any>(
  fn: T,
  nombre: string
): (...args: Parameters<T>) => ReturnType<T> {
  return (...args: Parameters<T>): ReturnType<T> => {
    console.log(`[${nombre}] Llamada con:`, args);
    const resultado = fn(...args);
    console.log(`[${nombre}] Resultado:`, resultado);
    return resultado;
  };
}

const sumaLogeada = crearLogger(sumar, "suma");
console.log(sumaLogeada(3, 4));
// Consola:
// [suma] Llamada con: [3, 4]
// [suma] Resultado: 7
// 7
```

===============================================================================
6. INSTANCETYPE Y CONSTRUCTOR PARAMETERS
===============================================================================

**InstanceType<T>**: Obtiene el tipo de instancia de una clase (o constructor).

```typescript
class Usuario {
  constructor(
    public id: number,
    public nombre: string
  ) {}
}

type InstanciaUsuario = InstanceType<typeof Usuario>;
//    ^? type InstanciaUsuario = Usuario

// Equivalente a:
type InstanciaUsuario2 = Usuario;
```

**ConstructorParameters<T>**: Obtiene los parámetros del constructor.

```typescript
type ParametrosConstructor = ConstructorParameters<typeof Usuario>;
//    ^? type ParametrosConstructor = [id: number, nombre: string]
```

**Ejemplo práctico: fábrica tipada**

```typescript
interface Constructor<T> {
  new (...args: any[]): T;
}

function fabricar<T>(
  Clase: Constructor<T>,
  ...args: ConstructorParameters<typeof Clase>
): T {
  return new Clase(...args);
}

// Pero esto requiere un tipo más preciso:
function fabricarMejor<T extends new (...args: any[]) => any>(
  Clase: T,
  ...args: ConstructorParameters<T>
): InstanceType<T> {
  return new Clase(...args);
}

const usuario = fabricarMejor(Usuario, 1, "Ana");
//    ^? const usuario: Usuario
```

7. TYPEOF CON CLASES
===============================================================================

```typescript
class Producto {
  constructor(
    public nombre: string,
    public precio: number
  ) {}

  calcularIVA(tasa: number = 0.21): number {
    return this.precio * tasa;
  }
}

// typeof Producto da el tipo del constructor (no instancia)
type TipoProductoClass = typeof Producto;
//    ^? typeof Producto
// (no vemos la firma exacta porque es la clase)

// Para la instancia:
type ProductoInstance = Producto;
// O:
type ProductoInstance2 = InstanceType<typeof Producto>;
```

**typeof con miembros estáticos:**

```typescript
class ConfiguracionGlobal {
  static VERSION = "1.0.0";
  static MAX_INTENTOS = 3;

  static cargar(): void {
    console.log("Cargando configuración...");
  }
}

type TipoEstaticos = typeof ConfiguracionGlobal;
// Contiene VERSION, MAX_INTENTOS, cargar (miembros estáticos)
// No contiene miembros de instancia

type VersionType = typeof ConfiguracionGlobal.VERSION;
//    ^? type VersionType = string
```

8. COMBINANDO TYPEOF CON KEYOF
===============================================================================

Esta combinación es extremadamente común:

```typescript
const usuario = {
  id: 1,
  nombre: "Ana",
  email: "ana@test.com",
  direccion: {
    calle: "Av. Principal",
    ciudad: "Madrid"
  }
};

// typeof + keyof
type ClavesUsuario = keyof typeof usuario;
//    ^? type ClavesUsuario = "id" | "nombre" | "email" | "direccion"

// typeof + keyof + indexed access
type TipoDireccion = (typeof usuario)["direccion"];
//    ^? type TipoDireccion = { calle: string; ciudad: string }

// Función genérica con typeof
function obtenerValor<T extends Record<string, any>>(
  obj: T,
  key: keyof T
): T[keyof T] {
  return obj[key];
}
```

**Patrón: constante como fuente de verdad de tipos**

```typescript
export const ROLES = {
  ADMIN: "admin",
  USER: "user",
  GUEST: "guest"
} as const;

export type Rol = typeof ROLES[keyof typeof ROLES];
//    ^? type Rol = "admin" | "user" | "guest"

// Ahora Rol está sincronizado con ROLES automáticamente
function tienePermiso(rol: Rol, accion: string): boolean {
  if (rol === ROLES.ADMIN) return true;
  if (rol === ROLES.USER && accion === "leer") return true;
  return false;
}
```

Si agregamos un nuevo rol en `ROLES`, el tipo `Rol` se actualiza solo.

9. PATRÓN: EVENTOS TIPADOS DESDE CONSTANTES
===============================================================================

```typescript
const EVENTOS = {
  USUARIO_LOGIN: "usuario:login",
  USUARIO_LOGOUT: "usuario:logout",
  PRODUCTO_CREADO: "producto:creado",
  PRODUCTO_ELIMINADO: "producto:eliminado",
  UI_CLICK: "ui:click",
  UI_NAVIGATE: "ui:navigate"
} as const;

type EventoKey = keyof typeof EVENTOS;
type EventoValue = typeof EVENTOS[EventoKey];
//    ^? type EventoValue = "usuario:login" | "usuario:logout"
//                       | "producto:creado" | "producto:eliminado"
//                       | "ui:click" | "ui:navigate"

interface PayloadMap {
  "usuario:login": { userId: number };
  "usuario:logout": { userId: number };
  "producto:creado": { productId: number; nombre: string };
  "producto:eliminado": { productId: number };
  "ui:click": { x: number; y: number };
  "ui:navigate": { ruta: string };
}

class EventBus {
  private handlers = new Map<string, Set<Function>>();

  on<E extends EventoValue>(
    evento: E,
    handler: (payload: PayloadMap[E]) => void
  ): void {
    if (!this.handlers.has(evento)) {
      this.handlers.set(evento, new Set());
    }
    this.handlers.get(evento)!.add(handler);
  }

  emit<E extends EventoValue>(evento: E, payload: PayloadMap[E]): void {
    this.handlers.get(evento)?.forEach(h => h(payload));
  }
}

const bus = new EventBus();

bus.on("usuario:login", (payload) => {
  console.log(`Login: ${payload.userId}`);
});

bus.emit("usuario:login", { userId: 1 });
```

10. PATRÓN: CONST ASSERTION + TYPEOF PARA CONFIGURACIONES
===============================================================================

```typescript
const CONFIG_APP = {
  api: {
    url: "https://api.miapp.com",
    version: "v2",
    timeout: 10000
  },
  features: {
    darkMode: true,
    analytics: false,
    exportPDF: true
  },
  limits: {
    maxFileSize: 10 * 1024 * 1024,
    maxUsers: 100
  }
} as const;

// Extraer tipos de la configuración
type AppConfig = typeof CONFIG_APP;
//    ^? Tipo completo con readonly en todas partes

type ApiConfig = typeof CONFIG_APP.api;
//    ^? { readonly url: "https://api.miapp.com"; readonly version: "v2"; ... }

type FeatureKey = keyof typeof CONFIG_APP.features;
//    ^? "darkMode" | "analytics" | "exportPDF"

type FeatureValue = typeof CONFIG_APP.features[FeatureKey];
//    ^? true | false

// Función que accede a features de forma tipada
function isFeatureEnabled(feature: FeatureKey): boolean {
  return CONFIG_APP.features[feature];
}

console.log(isFeatureEnabled("darkMode")); // true
// isFeatureEnabled("inexistente"); // Error
```

```typescript
enum Direccion {

  SUR = "S",
  ESTE = "E",
  OESTE = "O"

type TipoDireccion = typeof Direccion;
//    ^? { NORTE: Direccion.NORTE; SUR: Direccion.SUR; ... }

//    ^? "NORTE" | "SUR" | "ESTE" | "OESTE"

//    ^? "NORTE" | "SUR" | "ESTE" | "OESTE"
// (no confundir con los valores del enum)
```

12. EJEMPLO INTEGRADOR: API ENDPOINT CONFIGURATION
===============================================================================

```typescript
// === Constantes de configuración ===
const ENDPOINTS = {
  usuarios: {
    listar: { metodo: "GET", ruta: "/api/usuarios" },
    obtener: { metodo: "GET", ruta: "/api/usuarios/:id" },
    crear: { metodo: "POST", ruta: "/api/usuarios" },
    actualizar: { metodo: "PUT", ruta: "/api/usuarios/:id" },
    eliminar: { metodo: "DELETE", ruta: "/api/usuarios/:id" }
  },
  productos: {
    listar: { metodo: "GET", ruta: "/api/productos" },
    obtener: { metodo: "GET", ruta: "/api/productos/:id" },
    crear: { metodo: "POST", ruta: "/api/productos" }
  }
} as const;

// === Tipos derivados ===
type EndpointGroup = keyof typeof ENDPOINTS;
type EndpointName<G extends EndpointGroup> = keyof typeof ENDPOINTS[G];
type EndpointConfig<
  G extends EndpointGroup,
  N extends EndpointName<G>
> = typeof ENDPOINTS[G][N];

// === Payloads por endpoint ===
interface Payloads {
  "usuarios:listar": { page?: number };
  "usuarios:obtener": { id: number };
  "usuarios:crear": { nombre: string; email: string };
  "usuarios:actualizar": { id: number; datos: Partial<{ nombre: string; email: string }> };
  "usuarios:eliminar": { id: number };
  "productos:listar": { categoria?: string };
  "productos:obtener": { id: number };
  "productos:crear": { nombre: string; precio: number };
}

type EndpointKey = `${EndpointGroup}:${EndpointName<EndpointGroup>}`;

// === Cliente API ===
class ApiClient {
  private baseUrl: string;

  constructor(baseUrl: string) {
    this.baseUrl = baseUrl;
  }

  async request<
    G extends EndpointGroup,
    N extends EndpointName<G>
  >(
    group: G,
    name: N,
    payload: Payloads[`${G}:${N}`]
  ): Promise<unknown> {
    const config = ENDPOINTS[group][name];
    const url = `${this.baseUrl}${config.ruta}`;

    console.log(`[${config.metodo}] ${url}`, payload);

    if (config.metodo === "GET") {
      // Simular GET
      return { data: [], status: 200 };
    }

    return { data: null, status: 201 };
  }
}

// === Uso tipado ===
async function main() {
  const api = new ApiClient("https://api.com");

  const usuarios = await api.request("usuarios", "listar", { page: 1 });
  const usuario = await api.request("usuarios", "obtener", { id: 1 });
  const nuevo = await api.request("usuarios", "crear", {
    nombre: "Ana",
    email: "ana@test.com"
  });

  // Errores en compilación:
  // api.request("usuarios", "crear", { id: 1 });
  // Error: falta "nombre" y "email"

  // api.request("usuarios", "inexistente", {});
  // Error: "inexistente" no es un endpoint válido
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

- `typeof` en contexto de tipo extrae el tipo TypeScript de un valor.
- Combinado con `as const`, produce tipos literales exactos.
- `typeof` + `keyof` permite extraer claves y valores de objetos reales.
- `ReturnType<T>` y `Parameters<T>` se usan con `typeof funcion`.
- `InstanceType<T>` y `ConstructorParameters<T>` se usan con clases.
- Es la herramienta ideal para sincronizar tipos con constantes runtime.
- Patrón `const X = {...} as const; type T = typeof X[keyof typeof X]`
  es uno de los más usados en TypeScript profesional.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿`typeof` en TypeScript es igual que en JavaScript?**
No. En JavaScript, `typeof` es un operador en runtime que devuelve un
string. En TypeScript, en contexto de tipo, devuelve un tipo TypeScript
en tiempo de compilación.

❓ **¿Necesito usar `as const` siempre con typeof?**
No es obligatorio, pero sin `as const` los tipos se amplían
(string en lugar de "hola", number en lugar de 42). `as const`
preserva los literales exactos.

❓ **¿Puedo usar `typeof` con tipos que vienen de librerías?**
Sí. `typeof express` o `typeof fs` funcionan para obtener los tipos
exportados por esos módulos.

❓ **¿Cómo obtengo el tipo de un elemento de un arreglo con typeof?**
`typeof arreglo[number]` te da el tipo de los elementos.
Si usaste `as const`, `typeof arreglo[number]` te da la unión de
literales.

❓ **¿typeof funciona con null o undefined?**
`typeof null` en contexto de tipo da `typeof null` (literal null).
`typeof undefined` da `undefined`.
