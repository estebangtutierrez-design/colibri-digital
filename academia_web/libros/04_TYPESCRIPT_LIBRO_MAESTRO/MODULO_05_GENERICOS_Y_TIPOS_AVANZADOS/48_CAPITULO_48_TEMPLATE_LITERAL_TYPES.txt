En TypeScript 4.1 se introdujo una de las caracteristicas mas innovadoras
del lenguaje: los **Template Literal Types**. Si has usado template
literals de JavaScript (las comillas invertidas con `${}`), entonces ya

tipos**, no en runtime.

- Crear tipos de string literales mediante concatenacion
- Extraer partes de strings con `infer`

con la misma soltura con que lo haces en JavaScript.

===============================================================================
1. SINTAXIS BASICA
===============================================================================

Un Template Literal Type usa la misma sintaxis que los template literals
de JavaScript, pero en el sistema de tipos:

```typescript
type Saludo = "Hola, " + "mundo";  // Esto no funciona

type Saludo = `Hola, ${"mundo"}`;
//    ^? type Saludo = "Hola, mundo"

type Saludar<T extends string> = `Hola, ${T}!`;

type SaludoAna = Saludar<"Ana">;
//    ^? type SaludoAna = "Hola, Ana!"

type SaludoCarlos = Saludar<"Carlos">;
//    ^? type SaludoCarlos = "Hola, Carlos!"
```

**Con tipos literales y uniones:**

```typescript
type Color = "rojo" | "verde" | "azul";
type Tonalidad = "claro" | "oscuro";

type ColorCompleto = `${Color}-${Tonalidad}`;
//    ^? type ColorCompleto = "rojo-claro" | "rojo-oscuro"
//                        | "verde-claro" | "verde-oscuro"
//                        | "azul-claro" | "azul-oscuro"

// Con numeros
type Anio = `20${"20" | "21" | "22" | "23" | "24"}`;
// "2020" | "2021" | "2022" | "2023" | "2024"
```

===============================================================================
2. COMBINACIONES CARTESIANAS
===============================================================================

Cuando usas uniones dentro de un template literal, TypeScript genera
el producto cartesiano de todas las combinaciones:

```typescript
type Prefijo = "get" | "set" | "delete";
type Recurso = "usuario" | "producto" | "orden";

type MetodosAPI = `${Prefijo}${Capitalize<Recurso>}`;
//    ^? type MetodosAPI =
//     "getUsuario" | "getProducto" | "getOrden"
//     | "setUsuario" | "setProducto" | "setOrden"
//     | "deleteUsuario" | "deleteProducto" | "deleteOrden"

type HTTPMethod = "GET" | "POST" | "PUT" | "DELETE";
type Path = "/users" | "/products" | "/orders";

type Endpoint = `${HTTPMethod} ${Path}`;
//    ^? type Endpoint = "GET /users" | "GET /products" | "GET /orders"
//                     | "POST /users" | "POST /products" | "POST /orders"
//                     | "PUT /users" | "PUT /products" | "PUT /orders"
//                     | "DELETE /users" | "DELETE /products" | "DELETE /orders"
```

===============================================================================
3. STRING MANIPULATION UTILITY TYPES
===============================================================================

TypeScript incluye 4 tipos para manipular strings en el sistema de tipos:

```typescript
// Uppercase: convierte todo a mayusculas
type Gritar = Uppercase<"hola">;
//    ^? type Gritar = "HOLA"

// Lowercase: convierte todo a minusculas
type Susurrar = Lowercase<"HOLA">;
//    ^? type Susurrar = "hola"

// Capitalize: primera letra mayuscula
type Capitalizar = Capitalize<"hola mundo">;
//    ^? type Capitalizar = "Hola mundo"

// Uncapitalize: primera letra minuscula
type Descopitalizar = Uncapitalize<"Hola">;
//    ^? type Descopitalizar = "hola"
```

**Combinacion con template literals:**

```typescript
type GetterName<T extends string> = `get${Capitalize<T>}`;
type SetterName<T extends string> = `set${Capitalize<T>}`;

type GetNombre = GetterName<"nombre">;
//    ^? type GetNombre = "getNombre"

type SetEdad = SetterName<"edad">;
//    ^? type SetEdad = "setEdad"
```

===============================================================================
4. TEMPLATE LITERALS EN MAPEADOS (RENOMBRAR CLAVES)
===============================================================================

Esta es una de las aplicaciones mas poderosas:

```typescript
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

type Setters<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (valor: T[K]) => void;
};

interface Persona {
  nombre: string;
  edad: number;
}

type PersonaGetters = Getters<Persona>;
// { getNombre: () => string; getEdad: () => number }

type PersonaSetters = Setters<Persona>;
// { setNombre: (valor: string) => void; setEdad: (valor: number) => void }

// Combinar ambos
type PropiedadesCompletas<T> = Getters<T> & Setters<T>;

type PersonaCompleta = PropiedadesCompletas<Persona>;
// { getNombre: () => string; getEdad: () => number }
// & { setNombre: (valor: string) => void; setEdad: (valor: number) => void }
```

**Renombrar con prefijo personalizado:**

```typescript
type ConPrefijo<T, P extends string> = {
  [K in keyof T as `${P}${Capitalize<string & K>}`]: T[K];
};

interface Usuario {
  id: number;
  nombre: string;
}

type UsuarioAPI = ConPrefijo<Usuario, "api">;
// { apiId: number; apiNombre: string }

type UsuarioDB = ConPrefijo<Usuario, "db">;
// { dbId: number; dbNombre: string }
```

===============================================================================
5. INFER EN TEMPLATE LITERAL TYPES
===============================================================================

Podemos usar `infer` dentro de template literals para extraer partes de
un string:

```typescript
// Extraer el primer caracter
type PrimerCaracter<T extends string> = T extends `${infer P}${string}`
  ? P
  : never;

type A = PrimerCaracter<"Hola">;
//    ^? type A = "H"

type B = PrimerCaracter<"TypeScript">;
//    ^? type B = "T"

// Extraer el ultimo caracter
type UltimoCaracter<T extends string> = T extends `${string}${infer U}`
  ? U extends `${string}${infer}`
    ? UltimoCaracter<U>
    : U
  : never;

type C = UltimoCaracter<"Hola">;
//    ^? type C = "a"
```

**Separar en partes:**

```typescript
// Dividir por un separador
type AntesDelGuion<T extends string> = T extends `${infer Antes}-${string}`
  ? Antes
  : T;

type DespuesDelGuion<T extends string> = T extends `${string}-${infer Despues}`
  ? Despues
  : T;

type Prefijo = AntesDelGuion<"usuario:login">;
//    ^? type Prefijo = "usuario"

type Sufijo = DespuesDelGuion<"usuario:login">;
//    ^? type Sufijo = "login"
```

**Validacion de patrones:**

```typescript
// Validar que un string comience con "get"
type EsGetter<T extends string> = T extends `get${string}` ? true : false;

type A = EsGetter<"getNombre">;
//    ^? type A = true

type B = EsGetter<"setNombre">;
//    ^? type B = false

// Validar formato de email (basico)
type EsEmail<T extends string> = T extends `${string}@${string}.${string}`
  ? true
  : false;

type C = EsEmail<"user@test.com">;
//    ^? type C = true

type D = EsEmail<"invalido">;
//    ^? type D = false
```

6. PATRON: EVENTOS TIPADOS CON TEMPLATE LITERALS
===============================================================================

```typescript
// Definir dominio de eventos
type Dominio = "usuario" | "producto" | "orden";
type Accion = "creado" | "actualizado" | "eliminado" | "obtenido";

type EventoDominio = `${Dominio}:${Accion}`;
//    ^? "usuario:creado" | "usuario:actualizado" | ...

// Extraer dominio y accion de un evento
type ExtraerDominio<T extends string> = T extends `${infer D}:${string}`
  ? D
  : never;

type ExtraerAccion<T extends string> = T extends `${string}:${infer A}`
  ? A
  : never;

type Dom = ExtraerDominio<"usuario:creado">;
//    ^? type Dom = "usuario"

type Act = ExtraerAccion<"usuario:creado">;
//    ^? type Act = "creado"

// Mapa de eventos a sus payloads
interface EventPayloadMap {
  "usuario:creado": { id: number; nombre: string };
  "usuario:eliminado": { id: number };
  "producto:creado": { id: number; nombre: string; precio: number };
  "producto:actualizado": { id: number; cambios: string[] };
  "orden:creado": { ordenId: number; total: number };
  "orden:obtenido": { ordenId: number };
}

// Handler tipado
type EventHandler<E extends keyof EventPayloadMap> = (
  payload: EventPayloadMap[E]
) => void;

// Sistema de eventos
class EventBus {
  private handlers = new Map<string, Set<Function>>();

  on<E extends keyof EventPayloadMap>(
    evento: E,
    handler: EventHandler<E>
  ): void {
    if (!this.handlers.has(evento)) {
      this.handlers.set(evento, new Set());
    }
    this.handlers.get(evento)!.add(handler);
  }

  emit<E extends keyof EventPayloadMap>(
    evento: E,
    payload: EventPayloadMap[E]
  ): void {
    this.handlers.get(evento)?.forEach((h) => h(payload));
  }
}

// Uso
const bus = new EventBus();

bus.on("usuario:creado", (payload) => {
  console.log(`Usuario creado: ${payload.nombre}`);
});

bus.emit("usuario:creado", { id: 1, nombre: "Ana" });
```

7. PATRON: CSS PROPIEDADES TIPADAS
===============================================================================

```typescript
// Valores CSS comunes
type Unit = "px" | "em" | "rem" | "%" | "vw" | "vh";

type ValorCSS = `${number}${Unit}`;

type Ancho = ValorCSS;
//    ^? `${number}px` | `${number}em` | `${number}rem` | ...

// Validar margen completo
type Margen = `${ValorCSS} ${ValorCSS} ${ValorCSS} ${ValorCSS}`;

// Colores
type ColorNombre = "red" | "blue" | "green" | "white" | "black";
type HexColor = `#${string}`;
type RGBColor = `rgb(${number}, ${number}, ${number})`;
type ColorCSS = ColorNombre | HexColor | RGBColor;

const bg: ColorCSS = "#ff0000";
const fg: ColorCSS = "rgb(255, 0, 0)";
// const invalido: ColorCSS = "not-a-color"; // Error

// Border shorthand
type BorderStyle = "solid" | "dashed" | "dotted" | "none";
type Border = `${string}px ${BorderStyle} ${ColorCSS}`;

const borde: Border = "2px solid red";
// const malBorde: Border = "2px wavy red"; // Error
```

8. PATRON: NOMBRES DE RUTAS API
===============================================================================

```typescript
// Parametros de ruta
type RutaConParametros = "/api/:recurso/:id";

// Generar rutas tipadas
type Recurso = "users" | "products" | "orders";
type AccionRuta = "" | "/:id" | "/:id/edit";

type Ruta = `/api/${Recurso}${AccionRuta}`;
//    ^? "/api/users" | "/api/users/:id" | "/api/users/:id/edit"
//     | "/api/products" | "/api/products/:id" | ...

// Extraer parametros de ruta
type ParametrosRuta<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ParametrosRuta<`/${Rest}`>
    : T extends `${string}:${infer Param}`
      ? Param
      : never;

type Params = ParametrosRuta<"/api/users/:id/posts/:postId">;
//    ^? type Params = "id" | "postId"

// Construir objeto de parametros
type ObjetoParams<Ruta extends string> = {
  [K in ParametrosRuta<Ruta>]: string;
};

type UsersParams = ObjetoParams<"/api/users/:id">;
// { id: string }

type PostsParams = ObjetoParams<"/api/users/:id/posts/:postId">;
// { id: string; postId: string }
```

9. TEMPLATE LITERALS CON CONDICIONALES
===============================================================================

```typescript
// Añadir prefijo opcional
type ConOPrefijo<T extends string, P extends string | undefined> =
  P extends string ? `${P}${Capitalize<T>}` : T;

type SinPrefijo = ConOPrefijo<"nombre", undefined>;
//    ^? type SinPrefijo = "nombre"

type ConPrefijo = ConOPrefijo<"nombre", "api">;
//    ^? type ConPrefijo = "apiNombre"

// Pluralizar palabras (basico)
type Plural<T extends string> = T extends `${infer Ultimo}`
  ? Ultimo extends "s"
    ? T
    : Ultimo extends "y"
      ? `${Extract<T, string>}ies`
      : `${T}s`
  : never;

type Pl = Plural<"usuario">;
//    ^? type Pl = "usuarios"

type Pl2 = Plural<"clase">;
//    ^? type Pl2 = "clases"
```

10. PATRON: NOMBRES DE METODOS CRUD

```typescript
type Entidad = "user" | "product" | "order";
type Metodo = "get" | "create" | "update" | "delete";

type MetodoCRUD = `${Metodo}${Capitalize<Entidad>}`;
//    ^? "getUser" | "createUser" | "updateUser" | "deleteUser"

//     | "getOrder" | "createOrder" | "updateOrder" | "deleteOrder"

type PluralEntidad = {
  user: "users";
  product: "products";
  order: "orders";

type RutaCRUD = {
  [E in Entidad]: {
    get: `/api/${PluralEntidad[E]}`;
    create: `/api/${PluralEntidad[E]}`;
    update: `/api/${PluralEntidad[E]}/:id`;
    delete: `/api/${PluralEntidad[E]}/:id`;
  };
};

type UserRoutes = RutaCRUD["user"];
// { get: "/api/users"; create: "/api/users"; update: "/api/users/:id"; delete: "/api/users/:id" }
```

11. TEMPLATE LITERALS CON CONDICIONALES AVANZADOS

```typescript
// Encontrar y reemplazar en tipos (simplificado)
type Reemplazar<
  T extends string,
  Buscar extends string,
  Remplazo extends string
> = T extends `${infer Inicio}${Buscar}${infer Fin}`
  ? `${Inicio}${Remplazo}${Reemplazar<Fin, Buscar, Remplazo>}`
  : T;

type ConGuiones = Reemplazar<"nombreDeUsuario", "De", "-">;
//    ^? type ConGuiones = "nombre-De-Usuario"

// Convertir camelCase a kebab-case
type CamelToKebab<T extends string> =
  T extends `${infer C}${infer Rest}`
    ? C extends Uppercase<C>
      ? `-${Lowercase<C>}${CamelToKebab<Rest>}`
      : `${C}${CamelToKebab<Rest>}`
    : T;

type Kebab = CamelToKebab<"miNombreDeVariable">;
//    ^? type Kebab = "mi-nombre-de-variable"

// Convertir camelCase a snake_case
type CamelToSnake<T extends string> =
  T extends `${infer C}${infer Rest}`
    ? C extends Uppercase<C>
      ? `_${Lowercase<C>}${CamelToSnake<Rest>}`
      : `${C}${CamelToSnake<Rest>}`
    : T;

type Snake = CamelToSnake<"miNombreDeVariable">;
//    ^? type Snake = "mi_nombre_de_variable"
```

12. EJEMPLO INTEGRADOR: SISTEMA DE CONFIGURACION CON VARIABLES DE ENTORNO
===============================================================================

```typescript
// === Definicion de variables de entorno ===
type EnvVar =
  | "DATABASE_URL"
  | "API_KEY"
  | "PORT"
  | "NODE_ENV"
  | "LOG_LEVEL";

type NamingConvention = "camel" | "snake" | "kebab";

// Convertir variable de entorno a diferentes formatos
type EnvToCamel<T extends string> =
  T extends `${infer Part1}_${infer Part2}`
    ? `${Lowercase<Part1>}${Capitalize<EnvToCamel<Part2>>}`
    : Lowercase<T>;

type EnvToSnake<T extends string> = Lowercase<T>;

type EnvToKebab<T extends string> =
  T extends `${infer Part1}_${infer Part2}`
    ? `${Lowercase<Part1>}-${EnvToKebab<Part2>}`
    : Lowercase<T>;

type ConfigCamel = {
  [K in EnvVar as EnvToCamel<K>]: string;
};

type ConfigSnake = {
  [K in EnvVar as EnvToSnake<K>]: string;
};

// === Clase de configuracion ===
class ConfigManager<T extends EnvVar> {
  private config: Record<string, string> = {};

  cargar(vars: T[]): void {
    for (const v of vars) {
      const valor = process.env[v];
      if (valor) {
        this.config[v] = valor;
      } else {
        console.warn(`Variable ${v} no definida`);
      }
    }
  }

  obtener<K extends T>(varName: K): string {
    return this.config[varName] ?? "";
  }
}

// === Ejemplo de uso ===
const config = new ConfigManager();
config.cargar(["DATABASE_URL", "PORT", "NODE_ENV"]);
const dbUrl = config.obtener("DATABASE_URL");

// Funcion de tipo para construir nombres de propiedades
type PropiedadConfig = `config${Capitalize<EnvToCamel<EnvVar>>}`;
//    ^? type PropiedadConfig =
//     "configDatabaseUrl" | "configApiKey" | "configPort"
//     | "configNodeEnv" | "configLogLevel"

// === Clase de configuracion tipada avanzada ===
class ConfigSistema {
  private valores = new Map<string, string>();

  constructor() {
    this.cargarVariables();
  }

  private cargarVariables(): void {
    const mapeo: Record<EnvToCamel<EnvVar>, EnvVar> = {
      databaseUrl: "DATABASE_URL",
      apiKey: "API_KEY",
      port: "PORT",
      nodeEnv: "NODE_ENV",
      logLevel: "LOG_LEVEL"
    };

    for (const [key, env] of Object.entries(mapeo)) {
      const valor = process.env[env];
      if (valor) {
        this.valores.set(key, valor);
      }
    }
  }

  get<T extends EnvToCamel<EnvVar>>(clave: T): string | undefined {
    return this.valores.get(clave);
  }
}
```

RESUMEN DEL CAPITULO
===============================================================================

- Los Template Literal Types usan `${}` en el sistema de tipos para
  construir strings en tiempo de compilacion.
- Combinados con uniones, generan el producto cartesiano de todas las
  combinaciones posibles.
- Los utility types `Uppercase`, `Lowercase`, `Capitalize`, `Uncapitalize`
  manipulan strings en el sistema de tipos.
- `infer` dentro de template literals permite extraer partes de strings.
- Combinados con Mapped Types, permiten renombrar propiedades.
- Son ideales para tipar eventos, rutas, CSS, y configuraciones.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

**Puedo usar template literals con tipos que no son literales?**
Si, puedes usar cualquier tipo que extienda `string | number | boolean |
bigint`. Los tipos no literales se amplian a su tipo base (string, number).

**Que pasa si uso uniones muy grandes en template literals?**
TypeScript tiene un limite en la cantidad de combinaciones que genera.
Si la union es demasiado grande (cientos de combinaciones), puede
afectar el rendimiento del compilador.

**Puedo usar template literals con simbolos?**
No, los simbolos no son admisibles en template literals.

**Son los template literals utiles para librerias?**
Extremadamente utiles. Librerias como styled-components, tRPC, y Zod
los usan para generar tipos a partir de strings de configuracion.
