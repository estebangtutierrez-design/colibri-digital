Querido estudiante,

En los capítulos anteriores hemos visto cómo los genéricos y los utility
types nos permiten manipular tipos. Pero hay un operador que es tan
fundamental que merece su propio capítulo: `keyof`.

`keyof` es un operador de tipo que toma un tipo y devuelve la unión de
sus claves (nombres de propiedades). Parece simple, pero su poder es
inmenso. Es la base de `Pick`, `Omit`, los Mapped Types, y prácticamente
cada patrón avanzado de TypeScript.

¿Quieres acceder a propiedades de forma segura? `keyof`. ¿Quieres crear
tipos derivados de otros? `keyof`. ¿Quieres restringir valores a las
claves existentes de un objeto? `keyof`.

En este capítulo no solo aprenderás su sintaxis, sino que entenderás
cómo combinarlo con Indexed Access Types (`T[K]`) para construir
sistemas de tipos verdaderamente dinámicos y seguros.

===============================================================================
1. ¿QUÉ ES KEYOF?
===============================================================================

`keyof` es un operador de tipo que toma un tipo existente y produce una
unión de sus nombres de propiedad (claves).

```typescript
interface Persona {
  nombre: string;
  edad: number;
  direccion: string;
}

type ClavesPersona = keyof Persona;
//    ^? type ClavesPersona = "nombre" | "edad" | "direccion"
```

**Analogía mental:** Si un objeto es un edificio, `keyof` te da la lista
de todas las puertas con sus nombres.

**Con tipos primitivos:**

```typescript
type ClavesString = keyof string;
//    ^? type ClavesString = number | typeof Symbol.iterator
//         | "toString" | "charAt" | "charCodeAt" | "concat" | ...
// String tiene muchos métodos heredados

type ClavesNumber = keyof number;
//    ^? type ClavesNumber = "toString" | "toFixed" | "toExponential" | ...

type ClavesBoolean = keyof boolean;
//    ^? type ClavesBoolean = "valueOf"
```

**Con arreglos:**

```typescript
type ClavesArray = keyof number[];
//    ^? type ClavesArray = number | "length" | "toString" | "push" | ...

// Más útil con tuplas
type Tupla = [string, number];
type ClavesTupla = keyof Tupla;
//    ^? type ClavesTupla = "0" | "1" | "length" | ...
```

===============================================================================
2. KEYOF COMO CONSTRAINT GENÉRICO
===============================================================================

El uso más común de `keyof` es como constraint para parámetros de tipo:

```typescript
function obtenerPropiedad<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const persona = { nombre: "Ana", edad: 30, ciudad: "Madrid" };

const nombre = obtenerPropiedad(persona, "nombre");
//    ^? const nombre: string

const edad = obtenerPropiedad(persona, "edad");
//    ^? const edad: number

// Error en tiempo de compilación:
// obtenerPropiedad(persona, "email");
// Error: Argument of type '"email"' is not assignable to
// parameter of type '"nombre" | "edad" | "ciudad"'
```

**¿Qué está pasando aquí?**

1. `K extends keyof T` restringe K a ser solo una de las claves de T.
2. `T[K]` es el tipo del valor en esa clave (Indexed Access Type).
3. El compilador verifica estáticamente que la clave existe.

**Sin keyof, tendríamos que hacer esto:**

```typescript
// ❌ Inseguro
function obtenerPropiedadInseguro(obj: any, key: string): any {
  return obj[key];
}

// ❌ Verboso y frágil
function obtenerPropiedadManual(obj: {
  nombre?: string;
  edad?: number;
  ciudad?: string;
}, key: string): unknown {
  return (obj as any)[key];
}
```

===============================================================================
3. INDEXED ACCESS TYPES (T[K])
===============================================================================

El operador `T[K]` accede al tipo de la propiedad `K` dentro de `T`.

```typescript
interface Usuario {
  id: number;
  nombre: string;
  direccion: {
    calle: string;
    numero: number;
    ciudad: string;
  };
  telefonos: string[];
}

type TipoId = Usuario["id"];
//    ^? type TipoId = number

type TipoNombre = Usuario["nombre"];
//    ^? type TipoNombre = string

type TipoDireccion = Usuario["direccion"];
//    ^? type TipoDireccion = { calle: string; numero: number; ciudad: string }

type TipoTelefonos = Usuario["telefonos"];
//    ^? type TipoTelefonos = string[]
```

**Acceso con uniones de claves:**

```typescript
type IdONombre = Usuario["id" | "nombre"];
//    ^? type IdONombre = number | string

type Todo = Usuario[keyof Usuario];
//    ^? type Todo = number | string | { calle: string; ... } | string[]
```

**Acceso anidado:**

```typescript
type Calle = Usuario["direccion"]["calle"];
//    ^? type Calle = string

type TipoTelefono = Usuario["telefonos"][number];
//    ^? type TipoTelefono = string
// [number] accede al tipo de elemento de un arreglo

// Con tuplas
type Tupla = [string, number, boolean];
type Primero = Tupla[0];
//    ^? type Primero = string

type Segundo = Tupla[1];
//    ^? type Segundo = number
```

===============================================================================
4. KEYOF CON INTERFACES ANIDADAS
===============================================================================

```typescript
interface Producto {
  id: number;
  nombre: string;
  precio: {
    base: number;
    descuento: number;
    moneda: "EUR" | "USD";
  };
  inventario: {
    disponible: boolean;
    stock: number;
    almacenes: string[];
  };
}

// Acceder a propiedades anidadas con genéricos
function obtenerAnidado<T, K1 extends keyof T, K2 extends keyof T[K1]>(
  obj: T,
  k1: K1,
  k2: K2
): T[K1][K2] {
  return obj[k1][k2];
}

const producto: Producto = {
  id: 1,
  nombre: "Laptop",
  precio: { base: 1000, descuento: 0.1, moneda: "EUR" },
  inventario: { disponible: true, stock: 50, almacenes: ["Madrid"] }
};

const moneda = obtenerAnidado(producto, "precio", "moneda");
//    ^? const moneda: "EUR" | "USD"

const disponible = obtenerAnidado(producto, "inventario", "disponible");
//    ^? const disponible: boolean
```

**Versión con múltiples niveles (recursiva):**

```typescript
type TipoAnidado<T, K extends (keyof T | (string & {}))[]> = K extends [
  infer Primero,
  ...infer Resto
]
  ? Primero extends keyof T
    ? Resto extends []
      ? T[Primero]
      : Resto extends (keyof T[Primero] | (string & {}))[]
        ? TipoAnidado<T[Primero], Resto>
        : never
    : never
  : never;
```

Aunque esta versión recursiva es compleja, demuestra el poder de combinar
keyof con tipos condicionales.

===============================================================================
5. KEYOF CON TYPE ALIAS Y UNIONES
===============================================================================

`keyof` también funciona con type alias, uniones e intersecciones:

```typescript
// Con type alias
type Config = {
  url: string;
  timeout: number;
  retries: number;
};

type ClavesConfig = keyof Config;
//    ^? type ClavesConfig = "url" | "timeout" | "retries"
```

**keyof sobre una intersección:**

```typescript
interface A {
  a: string;
  b: number;
}

interface B {
  b: number;
  c: boolean;
}

type ClavesInterseccion = keyof (A & B);
//    ^? type ClavesInterseccion = "a" | "b" | "c"
// La intersección combina todas las propiedades
```

**keyof sobre una unión:**

```typescript
type Union = A | B;

type ClavesUnion = keyof Union;
//    ^? type ClavesUnion = "b"
// Solo las propiedades comunes a todos los miembros de la unión
```

**keyof con tipos condicionales:**

```typescript
interface Exitoso {
  estado: "exito";
  datos: string[];
}

interface Fallo {
  estado: "error";
  mensaje: string;
}

type Resultado = Exitoso | Fallo;

type ClavesComunes = keyof Resultado;
//    ^? type ClavesComunes = "estado"
// Solo "estado" existe en ambos
```

6. KEYOF CON CLASES
===============================================================================

```typescript
class Vehiculo {
  constructor(
    public marca: string,
    public modelo: string,
    public año: number
  ) {}

  arrancar(): void {
    console.log("Arrancando...");
  }

  detener(): void {
    console.log("Deteniendo...");
  }
}

type ClavesVehiculo = keyof Vehiculo;
// "marca" | "modelo" | "año" | "arrancar" | "detener"
// Incluye métodos y propiedades públicas

// Para excluir métodos:
type PropiedadesVehiculo = {
  [K in keyof Vehiculo]: Vehiculo[K] extends (...args: any[]) => any ? never : K;
}[keyof Vehiculo];
// "marca" | "modelo" | "año"
```

**Método genérico en clase:**

```typescript
class Gestor<T extends object> {
  private datos: T;

  constructor(datos: T) {
    this.datos = datos;
  }

  obtener<K extends keyof T>(clave: K): T[K] {
    return this.datos[clave];
  }

  establecer<K extends keyof T>(clave: K, valor: T[K]): void {
    this.datos[clave] = valor;
  }

  obtenerMultiples<K extends keyof T>(...claves: K[]): Pick<T, K> {
    const resultado = {} as Pick<T, K>;
    for (const c of claves) {
      resultado[c] = this.datos[c];
    }
    return resultado;
  }
}

const gestor = new Gestor({ nombre: "Ana", edad: 30, activo: true });
const nombre = gestor.obtener("nombre");
//    ^? const nombre: string

const parcial = gestor.obtenerMultiples("nombre", "activo");
//    ^? const parcial: { nombre: string; activo: boolean }
```

7. PATRÓN: SELECTOR DINÁMICO DE PROPIEDADES
===============================================================================

```typescript
function seleccionar<T, K extends keyof T>(
  objeto: T,
  ...claves: K[]
): Pick<T, K> {
  const resultado = {} as Pick<T, K>;
  for (const clave of claves) {
    if (clave in objeto) {
      resultado[clave] = objeto[clave];
    }
  }
  return resultado;
}

interface Empleado {
  id: number;
  nombre: string;
  email: string;
  salario: number;
  departamento: string;
  fechaContratacion: Date;
}

const empleado: Empleado = {
  id: 1, nombre: "Carlos", email: "carlos@empresa.com",
  salario: 50000, departamento: "IT", fechaContratacion: new Date()
};

// Selección dinámica
const datosPublicos = seleccionar(empleado, "nombre", "departamento");
//    ^? { nombre: string; departamento: string }

const datosSensible = seleccionar(empleado, "salario");
//    ^? { salario: number }
```

8. PATRÓN: VALIDACIÓN DE CAMPOS
===============================================================================

```typescript
interface ReglasValidacion<T> {
  [K in keyof T]?: (valor: T[K]) => string | null;
}

function validar<T extends object>(
  objeto: T,
  reglas: ReglasValidacion<T>
): Partial<Record<keyof T, string>> {
  const errores: Partial<Record<keyof T, string>> = {};

  for (const clave in reglas) {
    const validador = reglas[clave];
    if (validador) {
      const error = validador(objeto[clave as keyof T]);
      if (error) {
        errores[clave as keyof T] = error;
      }
    }
  }

  return errores;
}

interface FormularioRegistro {
  nombre: string;
  email: string;
  edad: number;
  password: string;
}

const reglas: ReglasValidacion<FormularioRegistro> = {
  nombre: (v) => v.length < 2 ? "Nombre muy corto" : null,
  email: (v) => !v.includes("@") ? "Email inválido" : null,
  edad: (v) => v < 18 ? "Debe ser mayor de edad" : null,
  password: (v) => v.length < 6 ? "Contraseña muy corta" : null
};

const errores = validar({
  nombre: "A",
  email: "invalido",
  edad: 15,
  password: "123"
}, reglas);
// errores tiene los mensajes de validación
```

9. PATRÓN: RENOMBRAR PROPIEDADES (MAPEADO CON KEYOF)
===============================================================================

```typescript
type Renombrar<T, Mapa extends Partial<Record<keyof T, string>>> = {
  [K in keyof T as K extends keyof Mapa
    ? Mapa[K] extends string
      ? Mapa[K]
      : K
    : K
  ]: T[K];
};

interface UsuarioViejo {
  nombre: string;
  apellido: string;
  correo: string;
}

type UsuarioNuevo = Renombrar<UsuarioViejo, {
  nombre: "firstName";
  apellido: "lastName";
  correo: "email";
}>;

// Equivalente a:
// {
//   firstName: string;
//   lastName: string;
//   email: string;
// }
```

10. KEYOF CON TIPOS MAPEADOS (AVANCE DEL CAPÍTULO 46)
===============================================================================

Una breve introducción a cómo `keyof` se usa en mapped types:

```typescript
type Opcional<T> = {
  [K in keyof T]?: T[K];
};

type SoloLectura<T> = {
  readonly [K in keyof T]: T[K];
};

// Combinando
type OpcionalYLectura<T> = {
  readonly [K in keyof T]?: T[K];
};

interface Datos {
  a: string;
  b: number;
  c: boolean;
}

type DatosOpcionales = Opcional<Datos>;
// { a?: string; b?: number; c?: boolean }
```

**Filtrar propiedades por tipo:**

```typescript
type PropiedadesString<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

interface Mezcla {
  id: number;
  nombre: string;
  activo: boolean;
  email: string;
}

type SoloStrings = PropiedadesString<Mezcla>;
// { nombre: string; email: string }
```

11. KEYOF CON TIPOS AVANZADOS: SATISFIES
===============================================================================

TypeScript 4.9 introdujo `satisfies`, que se beneficia de `keyof`:

```typescript
const config = {
  url: "https://api.com",
  puerto: 443,
  opciones: {
    timeout: 5000,
    retries: 3
  }
} satisfies Record<string, any>;

type ClavesConfig = keyof typeof config;
// "url" | "puerto" | "opciones"

// Verificación de que ciertas claves existen
type ClavesRequeridas = "url" | "puerto";
type Verificacion = ClavesRequeridas extends keyof typeof config ? true : false;
//    ^? type Verificacion = true
```

12. EJEMPLO INTEGRADOR: SISTEMA DE EVENTOS
===============================================================================

```typescript
// === Definición de eventos ===
interface EventosApp {
  usuario: {
    login: { userId: number; token: string };
    logout: { userId: number };
    registro: { userId: number; email: string };
  };
  producto: {
    creado: { productId: number; nombre: string };
    actualizado: { productId: number; cambios: string[] };
    eliminado: { productId: number };
  };
  ui: {
    click: { x: number; y: number };
    navigate: { ruta: string };
    error: { mensaje: string; codigo: number };
  };
}

// === Manejador de eventos ===
type CategoriaEvento = keyof EventosApp;
type NombreEvento<C extends CategoriaEvento> = keyof EventosApp[C];
type PayloadEvento<C extends CategoriaEvento, N extends NombreEvento<C>> =
  EventosApp[C][N];

class EmisorEventos {
  private listeners: Map<string, Set<Function>> = new Map();

  on<C extends CategoriaEvento, N extends NombreEvento<C>>(
    categoria: C,
    nombre: N,
    callback: (payload: PayloadEvento<C, N>) => void
  ): void {
    const clave = `${categoria}:${String(nombre)}`;
    if (!this.listeners.has(clave)) {
      this.listeners.set(clave, new Set());
    }
    this.listeners.get(clave)!.add(callback);
  }

  emit<C extends CategoriaEvento, N extends NombreEvento<C>>(
    categoria: C,
    nombre: N,
    payload: PayloadEvento<C, N>
  ): void {
    const clave = `${categoria}:${String(nombre)}`;
    console.log(`[Evento] ${clave}`, payload);
    this.listeners.get(clave)?.forEach(cb => cb(payload));
  }

  off<C extends CategoriaEvento, N extends NombreEvento<C>>(
    categoria: C,
    nombre: N,
    callback: (payload: PayloadEvento<C, N>) => void
  ): void {
    const clave = `${categoria}:${String(nombre)}`;
    this.listeners.get(clave)?.delete(callback);
  }
}

// === Uso ===
const emisor = new EmisorEventos();

emisor.on("usuario", "login", (payload) => {
  console.log(`Usuario ${payload.userId} logueado con token ${payload.token}`);
});

emisor.on("ui", "click", (payload) => {
  console.log(`Click en (${payload.x}, ${payload.y})`);
});

emisor.emit("usuario", "login", { userId: 1, token: "abc123" });
emisor.emit("ui", "click", { x: 100, y: 200 });

// Errores en compilación:
// emisor.emit("usuario", "click", { x: 1, y: 2 });
// Error: "click" no es un evento de "usuario"

// emisor.emit("producto", "creado", { productId: 1, nombre: "Laptop" });
// OK
```

Este ejemplo demuestra el poder de `keyof` combinado con Indexed Access
Types para crear sistemas de eventos totalmente tipados.

RESUMEN DEL CAPÍTULO
===============================================================================

- `keyof T` produce la unión de nombres de propiedad de `T`.
- `T[K]` (Indexed Access Type) obtiene el tipo de la propiedad K en T.
- `K extends keyof T` es un constraint que restringe K a claves válidas.
- `keyof` funciona con interfaces, type alias, clases, uniones e intersecciones.
- Combinado con genéricos, permite crear APIs que conocen la estructura
  de los objetos que reciben.
- `keyof` es la base de los Mapped Types y los tipos condicionales.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿`keyof` funciona con cualquier tipo?**
Sí, pero el resultado varía. Con objetos produce nombres de propiedad.
Con primitivos produce métodos heredados. Con arreglos produce índices
y miembros de Array.

❓ **¿Qué diferencia hay entre `keyof T` y `"nombre" | "edad"`?**
`keyof T` se adapta automáticamente al tipo T. Si T cambia, `keyof T`
se actualiza. Los literales manuales requieren mantenimiento manual.

❓ **¿Puedo usar `keyof` en tipos genéricos?**
Sí, es uno de sus usos principales. `keyof T` dentro de un genérico
permite escribir código que se adapta a cualquier tipo de objeto.

❓ **¿`T[K]` funciona con cualquier tipo de K?**
K debe ser un subtipo de `keyof T`. Puede ser una clave literal o una
unión de claves.

❓ **¿Qué significa `T[keyof T]`?**
Es el tipo de todos los valores de T (la unión de todos los tipos de
las propiedades de T).
