1. SINTAXIS BASICA
===============================================================================

Un tipo condicional tiene la forma:

```typescript
type Condicion<T> = T extends U ? TipoSi : TipoNo;
```

Donde:
- `T extends U` es la condicion (T es asignable a U?)
- `TipoSi` es el resultado si la condicion es verdadera
- `TipoNo` es el resultado si la condicion es falsa

**Ejemplo basico:**

```typescript
type EsString<T> = T extends string ? "si" : "no";

type A = EsString<string>;    // "si"
type B = EsString<number>;    // "no"
type C = EsString<"hola">;    // "si" ("hola" extiende string)
type D = EsString<any>;       // "si" (any extiende todo)
type E = EsString<never>;     // "si" de forma especial
```

**Ejemplo practico: filtro basico**

```typescript
type TipoPermitido<T> = T extends string | number ? T : never;

type Filtrado = TipoPermitido<string | number | boolean>;
//    ^? type Filtrado = string | number
// boolean se convierte en never (se elimina)
```

===============================================================================
2. DISTRIBUCION SOBRE UNIONES
===============================================================================

Esta es la caracteristica mas importante y confusa al mismo tiempo.

Cuando un tipo condicional se aplica a una **union**, TypeScript
**distribuye** la condicion sobre cada miembro de la union:

```typescript
type EsStringDistribuido<T> = T extends string ? "si" : "no";

// Con union:
type Resultado = EsStringDistribuido<string | number>;
// Paso 1: EsStringDistribuido<string> -> "si"
// Paso 2: EsStringDistribuido<number> -> "no"
// Paso 3: "si" | "no"
// Resultado: "si" | "no"
```

**Sin distribucion:**

Para evitar la distribucion, envuelve T y U en corchetes `[T]`:

```typescript
type EsStringNoDistribuido<T> = [T] extends [string] ? "si" : "no";

type Resultado2 = EsStringNoDistribuido<string | number>;
// [string | number] extends [string] ? "si" : "no"
// string | number NO es asignable a string
// Resultado: "no"
```

**Ejemplo practico: filtrar tipos de una union**

```typescript
// Exclude<T, U> implementado manualmente
type MiExclude<T, U> = T extends U ? never : T;

type Primitivos = string | number | boolean | null | undefined;
type SoloPrimitivos = MiExclude<Primitivos, null | undefined>;
// string | number | boolean

type SoloString = MiExclude<Primitivos, number | boolean | null | undefined>;
// string
```

===============================================================================
3. EXTRACT Y EXCLUDE CON TIPOS CONDICIONALES
===============================================================================

Los utility types `Extract` y `Exclude` son tipos condicionales:

```typescript
// Extract<T, U>: extrae de T lo que es asignable a U
type Extract<T, U> = T extends U ? T : never;

// Exclude<T, U>: excluye de T lo que es asignable a U
type Exclude<T, U> = T extends U ? never : T;

// Ejemplos
type Eventos = "click" | "focus" | "blur" | "resize" | "scroll";
type EventosMouse = Extract<Eventos, "click" | "focus" | "blur">;
// "click" | "focus" | "blur"

type SinScrollNiResize = Exclude<Eventos, "scroll" | "resize">;
// "click" | "focus" | "blur"
```

**Con tipos de objeto:**

```typescript
type Exitoso = { estado: "exito"; datos: any[] };
type Error = { estado: "error"; mensaje: string };
type Cargando = { estado: "cargando" };

type ResultadoApi = Exitoso | Error | Cargando;

type SoloExitoso = Extract<ResultadoApi, { estado: "exito" }>;
// { estado: "exito"; datos: any[] }

type NoExitoso = Exclude<ResultadoApi, { estado: "exito" }>;
// { estado: "error"; mensaje: string } | { estado: "cargando" }
```

===============================================================================
4. NONNULLABLE CON TIPOS CONDICIONALES
===============================================================================

```typescript
type NonNullable<T> = T extends null | undefined ? never : T;

type Valor = string | number | null | undefined;
type ValorDefinido = NonNullable<Valor>;
// string | number
```

**Implementacion de extraer definidos:**

```typescript
type NoNulo<T> = T extends null | undefined ? never : T;

function filtrarNulos<T>(arreglo: (T | null | undefined)[]): NoNulo<T>[] {
  return arreglo.filter(
    (item): item is NoNulo<T> => item !== null && item !== undefined
  );
}

const valores = [1, null, 2, undefined, 3];
const definidos = filtrarNulos(valores);
//    ^? const definidos: number[]
```

===============================================================================
5. LA PALABRA CLAVE INFER
===============================================================================

`infer` permite **extraer** un tipo desde dentro de otro tipo. Solo puede
usarse dentro de un tipo condicional en la parte `extends`:

```typescript
type TipoElementoArreglo<T> = T extends (infer U)[] ? U : never;

type A = TipoElementoArreglo<string[]>;
//    ^? type A = string

type B = TipoElementoArreglo<number[]>;
//    ^? type B = number

type C = TipoElementoArreglo<(string | boolean)[]>;
//    ^? type C = string | boolean

type D = TipoElementoArreglo<string>;
//    ^? type D = never (string no es arreglo)
```

**Extraer tipo de retorno de una funcion (ReturnType manual):**

```typescript
type MiReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type Funcion = (a: string) => number;
type Retorno = MiReturnType<Funcion>;
//    ^? type Retorno = number
```

**Extraer parametros (Parameters manual):**

```typescript
type MiParameters<T> = T extends (...args: infer P) => any ? P : never;

type Parametros = MiParameters<(a: string, b: number) => boolean>;
//    ^? type Parametros = [a: string, b: number]
```

**Extraer tipo de instancia de clase (InstanceType manual):**

```typescript
type MiInstanceType<T> = T extends new (...args: any[]) => infer R ? R : never;

class Usuario {
  constructor(public nombre: string) {}
}

type Instancia = MiInstanceType<typeof Usuario>;
//    ^? type Instancia = Usuario
```

===============================================================================
6. INFER EN ANIDADOS
===============================================================================

Podemos usar `infer` en estructuras anidadas:

```typescript
// Extraer tipo de promesa resuelta
type DesenvolverPromesa<T> = T extends Promise<infer U> ? U : T;

type A = DesenvolverPromesa<Promise<string>>;
//    ^? type A = string

type B = DesenvolverPromesa<Promise<Promise<number>>>;
//    ^? type B = Promise<number> (solo un nivel)

// Version recursiva
type AwaitedManual<T> = T extends Promise<infer U>
  ? AwaitedManual<U>
  : T;

type C = AwaitedManual<Promise<Promise<Promise<number>>>>;
//    ^? type C = number
```

**Extraer tipo de una funcion dentro de un objeto:**

```typescript
type TipoMetodo<T, K extends keyof T> = T[K] extends (...args: infer A) => infer R
  ? { args: A; retorno: R }
  : never;

interface Servicio {
  obtener(id: string): Promise<{ nombre: string }>;
  guardar(datos: { nombre: string }): void;
}

type MetodoObtener = TipoMetodo<Servicio, "obtener">;
// { args: [id: string]; retorno: Promise<{ nombre: string }> }
```

**Extraer de funciones con this:**

```typescript
type TipoThis<T> = T extends (this: infer This, ...args: any[]) => any
  ? This
  : unknown;

type ConThis = TipoThis<(this: Window, x: number) => void>;
//    ^? type ConThis = Window
```

===============================================================================
7. TIPOS CONDICIONALES RECURSIVOS
===============================================================================

Los tipos condicionales pueden referenciarse a si mismos:

```typescript
// Aplanar arreglos anidados recursivamente
type Aplanar<T> = T extends (infer U)[]
  ? U extends any[]
    ? Aplanar<U>
    : U
  : T;

type Plano = Aplanar<number[][][]>;
//    ^? type Plano = number

type Mezclado = Aplanar<(string | number)[][]>;
//    ^? type Mezclado = string | number
```

**DeepReadonly con tipos condicionales recursivos:**

```typescript
type DeepReadonly<T> = T extends (...args: any[]) => any
  ? T
  : T extends object
    ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
    : T;

interface Perfil {
  nombre: string;
  direccion: {
    calle: string;
    ciudad: string;
  };
  hobbies: string[];
}

type PerfilInmutable = DeepReadonly<Perfil>;
// {
//   readonly nombre: string;
//   readonly direccion: {
//     readonly calle: string;
//     readonly ciudad: string;
//   };
//   readonly hobbies: readonly string[];
// }
```

**Union a interseccion (avanzado):**

```typescript
type UnionToIntersection<U> = (
  U extends any ? (k: U) => void : never
) extends (k: infer I) => void
  ? I
  : never;

type A = { a: string };
type B = { b: number };
type Union = A | B;
type Interseccion = UnionToIntersection<Union>;
//    ^? type Interseccion = A & B
```

8. TIPOS CONDICIONALES CON MAPEADOS
===============================================================================

La combinacion de tipos condicionales y mapped types es extraordinaria:

```typescript
// Filtrar propiedades por tipo de valor
type PropiedadesDeTipo<T, TipoDeseado> = {
  [K in keyof T as T[K] extends TipoDeseado ? K : never]: T[K];
};

interface Documento {
  titulo: string;
  contenido: string;
  version: number;
  createdAt: Date;
  actualizado: Date;
}

type Strings = PropiedadesDeTipo<Documento, string>;
// { titulo: string; contenido: string }

type Dates = PropiedadesDeTipo<Documento, Date>;
// { createdAt: Date; actualizado: Date }

// Transformar condicionalmente tipos de propiedades
type TransformarFechas<T> = {
  [K in keyof T]: T[K] extends Date ? string : T[K];
};

type DocSerializado = TransformarFechas<Documento>;
// { titulo: string; contenido: string; version: number; createdAt: string; actualizado: string }
```

9. PATRON: DISCRIMINATED UNIONS CON TIPOS CONDICIONALES
===============================================================================

```typescript
type Evento =
  | { tipo: "click"; x: number; y: number }
  | { tipo: "keypress"; tecla: string }
  | { tipo: "focus"; elemento: string }
  | { tipo: "blur"; elemento: string };

// Extraer payload de un tipo de evento especifico
type PayloadEvento<T extends Evento["tipo"]> = Extract<
  Evento,
  { tipo: T }
>;

type ClickPayload = PayloadEvento<"click">;
// { tipo: "click"; x: number; y: number }

// Handler generico con condicional
type Handler<T extends Evento["tipo"]> = (
  payload: PayloadEvento<T>
) => void;

const handleClick: Handler<"click"> = (event) => {
  console.log(`Click en (${event.x}, ${event.y})`);
};

// Mapa de handlers
type HandlerMap = {
  [E in Evento as E["tipo"]]: (payload: E) => void;
};

const handlers: HandlerMap = {
  click: (e) => console.log(`Click en (${e.x}, ${e.y})`),
  keypress: (e) => console.log(`Tecla: ${e.tecla}`),
  focus: (e) => console.log(`Focus en ${e.elemento}`),
  blur: (e) => console.log(`Blur en ${e.elemento}`)
};
```

10. PATRON: TIPOS CONDICIONALES PARA FUNCIONES SOBRECARGADAS
===============================================================================

```typescript
// Extraer el tipo de una funcion sobrecargada
type UltimaFirma<T> = T extends (...args: any[]) => infer R ? R : never;

// Extraer sobrecarga especifica
type SobrecargaPorArgs<T, Args extends any[]> = T extends (
  ...args: Args
) => infer R
  ? R
  : never;

function procesar(valor: string): number;
function procesar(valor: number): string;
function procesar(valor: string | number): string | number {
  return typeof valor === "string" ? valor.length : String(valor);
}

type RetornoConString = SobrecargaPorArgs<typeof procesar, [string]>;
//    ^? type RetornoConString = number
```

11. IMPLEMENTACION DE UTILITY TYPES AVANZADOS
===============================================================================

```typescript
// ReturnType con infer
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : any;

// Parameters con infer
type Parameters<T> = T extends (...args: infer P) => any ? P : never;

// ConstructorParameters con infer
type ConstructorParameters<T> = T extends new (...args: infer P) => any
  ? P
  : never;

// InstanceType con infer
type InstanceType<T> = T extends new (...args: any[]) => infer R ? R : any;

// ThisParameterType con infer
type ThisParameterType<T> = T extends (this: infer U, ...args: any[]) => any
  ? U
  : unknown;

// OmitThisParameter con infer
type OmitThisParameter<T> = T extends (this: any, ...args: infer P) => infer R
  ? (...args: P) => R
  : T;

// Awaited con infer recursivo
type Awaited<T> = T extends Promise<infer U> ? Awaited<U> : T;
```

12. EJEMPLO INTEGRADOR: API CLIENTE CON TIPOS CONDICIONALES
===============================================================================

```typescript
// === Definiciones base ===
interface ApiRequest {
  method: "GET" | "POST" | "PUT" | "DELETE";
  path: string;
  params?: Record<string, string>;
  body?: unknown;
}

interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

// === Tipos condicionales para metodos HTTP ===
type GetResponse<T> = T extends { getData: infer D } ? D : never;
type PostResponse<T> = T extends { postData: infer D } ? D : never;

// === Cliente con tipos condicionales ===
class ApiClient {
  private baseUrl: string;

  constructor(baseUrl: string) {
    this.baseUrl = baseUrl;
  }

  async request<T>(
    config: ApiRequest
  ): Promise<ApiResponse<T>> {
    const url = new URL(`${this.baseUrl}${config.path}`);
    if (config.params) {
      Object.entries(config.params).forEach(([k, v]) =>
        url.searchParams.set(k, v)
      );
    }

    const response = await fetch(url.toString(), {
      method: config.method,
      headers: { "Content-Type": "application/json" },
      body: config.body ? JSON.stringify(config.body) : undefined
    });

    const data: T = await response.json();
    return {
      data,
      status: response.status,
      message: response.ok ? "OK" : "Error"
    };
  }
}

// === Endpoints con tipos derivados ===
type EndpointMap = {
  "/users": { getData: User[]; postData: User };
  "/users/:id": { getData: User; putData: User };
  "/products": { getData: Product[]; postData: Product };
};

interface User {
  id: number;
  name: string;
  email: string;
}

interface Product {
  id: number;
  name: string;
  price: number;
}

// === Tipos condicionales por endpoint ===
type ExtractEndpointData<
  T extends keyof EndpointMap,
  M extends string
> = EndpointMap[T] extends Record<`${M}Data`, infer D> ? D : never;

type GetEndpointData<T extends keyof EndpointMap> =
  EndpointMap[T] extends { getData: infer D } ? D : never;

type PostEndpointData<T extends keyof EndpointMap> =
  EndpointMap[T] extends { postData: infer D } ? D : never;

// === Funcion de consulta tipada ===
async function get<T extends keyof EndpointMap>(
  client: ApiClient,
  endpoint: T
): Promise<ApiResponse<GetEndpointData<T>>> {
  return client.request<GetEndpointData<T>>({
    method: "GET",
    path: endpoint as string
  });
}

async function post<T extends keyof EndpointMap>(
  client: ApiClient,
  endpoint: T,
  body: PostEndpointData<T>
): Promise<ApiResponse<PostEndpointData<T>>> {
  return client.request<PostEndpointData<T>>({
    method: "POST",
    path: endpoint as string,
    body
  });
}

// === Uso ===
async function main() {
  const api = new ApiClient("https://api.com");

  const users = await get(api, "/users");
  // users.data es User[] (inferido)

  const newUser = await post(api, "/users", {
    id: 1,
    name: "Ana",
    email: "ana@test.com"
  });
  // newUser.data es User (inferido)
  // newUser.data.id, .name, .email disponibles
}
```

RESUMEN DEL CAPITULO
===============================================================================

- Los tipos condicionales usan `T extends U ? X : Y` para elegir entre
  dos tipos basados en una condicion.
- La distribucion sobre uniones aplica la condicion a cada miembro.
- `infer` permite extraer y nombrar tipos desde dentro de otros tipos.
- Los tipos condicionales pueden ser recursivos.
- Utility types como `Extract`, `Exclude`, `NonNullable`, `ReturnType`,
  `Parameters`, `InstanceType` se implementan con tipos condicionales.
- La combinacion de tipos condicionales con mapped types permite
  filtrado y transformacion avanzada de propiedades.

PREGUNTAS FRECUENTES
===============================================================================

**Que significa que un tipo condicional se distribuye sobre una union?**
Significa que TypeScript aplica la condicion a cada miembro de la union
individualmente y luego une los resultados. Sin distribucion, la
condicion evalua toda la union como un solo tipo.

**Donde se puede usar `infer`?**
Solo dentro de la parte `extends` de un tipo condicional, y siempre
dentro de un patron que TypeScript pueda desestructurar.

**Hay limites en la recursion de tipos condicionales?**
TypeScript tiene un limite de profundidad de recursion (~50 niveles)
para evitar bucles infinitos. Para estructuras muy profundas, considera
disenos alternativos.

**Puedo usar `infer` multiples veces?**
Si, puedes usar varios `infer` en un mismo patron:
`T extends (infer A)[] | (infer B)[] ? A | B : never`.
