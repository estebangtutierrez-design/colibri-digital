1. ¿QUÉ ES TSC?
===============================================================================

`tsc` (TypeScript Compiler) es el programa oficial que compila TypeScript a
JavaScript. Está escrito en TypeScript (compilado a JavaScript) y se ejecuta
en Node.js.

Funciones principales de tsc:
1. **Analizar** el código TypeScript en busca de errores de tipo y sintaxis
2. **Transpilar** el código a JavaScript, eliminando las anotaciones de tipo
3. **Generar** archivos de salida (JS, mapas, declaraciones)

El proceso completo es:
```
┌─────────┐    ┌──────────────┐    ┌──────────┐    ┌─────────┐
│ .ts     │───>│ Análisis     │───>│          │───>│ .js     │
│ archivos│    │ léxico/      │    │ Emisión  │    │ archivos│
│ fuente  │    │ sintáctico/  │    │ de código│    │ salida  │
└─────────┘    │ semántico    │    └──────────┘    └─────────┘
               └──────────────┘
                      │
                      ▼
               ┌──────────────┐
               │ Errores de   │
               │ tipo y       │
               │ sintaxis     │
               └──────────────┘
```

---

📌 **tsc hace dos cosas distintas: verifica tipos Y transpila. Puedes hacer
una sin la otra usando flags específicos.**

```bash
# Verificar tipos sin generar archivos
npx tsc --noEmit

# Transpilar sin verificar tipos (no recomendado)
# npx tsc --transpileOnly  # Requiere ts-node o similar
```

---

🔵 **Ejemplo 1: tsc en su forma más simple**

```bash
# Compilar un solo archivo
npx tsc archivo.ts

# Esto produce archivo.js en el mismo directorio

# Compilar múltiples archivos
npx tsc archivo1.ts archivo2.ts

# Compilar usando tsconfig.json (proyecto completo)
npx tsc
```

2. COMPILACIÓN DE ARCHIVOS INDIVIDUALES
===============================================================================

Aunque lo más común es usar tsconfig.json, también podemos compilar archivos
individuales pasándolos como argumentos a tsc.

```bash
# Compilar un archivo específico
npx tsc mi-archivo.ts

# Especificar el archivo de salida
npx tsc mi-archivo.ts --outFile resultado.js

# Compilar con un target específico
npx tsc mi-archivo.ts --target ES2022
```

---

📌 **Cuando compilas archivos individuales, tsc usa valores por defecto en
lugar de tsconfig.json. Es mejor usar siempre tsconfig.json.**

```typescript
// archivo: saludar.ts
function saludar(nombre: string, edad: number): string {
  return `Hola, soy ${nombre} y tengo ${edad} años.`;
}

console.log(saludar("Ana", 25));
```

```bash
# Compilar archivo individual
$ npx tsc saludar.ts

# Ver el resultado
$ cat saludar.js
function saludar(nombre, edad) {
    return "Hola, soy " + nombre + " y tengo " + edad + " años.";
}
console.log(saludar("Ana", 25));
```

---

🔵 **Ejemplo 2: Compilación con diferentes targets**

```bash
# Compilar para ES5 (máxima compatibilidad)
$ npx tsc saludar.ts --target ES5
$ cat saludar.js
function saludar(nombre, edad) {
    return "Hola, soy " + nombre + " y tengo " + edad + " a\u00F1os.";
}
console.log(saludar("Ana", 25));
```

```bash
# Compilar para ES2022 (mínima transformación)
$ npx tsc saludar.ts --target ES2022
$ cat saludar.js
function saludar(nombre, edad) {
    return `Hola, soy ${nombre} y tengo ${edad} años.`;
}
console.log(saludar("Ana", 25));
# ¡Los template literals se conservan!
```

===============================================================================
3. COMPILACIÓN DE PROYECTOS CON TSCONFIG.JSON
===============================================================================

Cuando tienes un tsconfig.json en la raíz, simplemente ejecutar `npx tsc`
compila todo el proyecto según la configuración definida.

```bash
# Compilar el proyecto completo (usa tsconfig.json)
npx tsc

# Compilar usando un archivo de configuración específico
npx tsc --project tsconfig.production.json

# Compilar solo la verificación de tipos
npx tsc --noEmit
```

---

📌 **Usar `--noEmit` es la forma más rápida de verificar errores, ya que
no genera archivos de salida.**

```bash
# Flujo de trabajo típico
# 1. Escribes código
# 2. Verificas errores (rápido)
npx tsc --noEmit

# 3. Solo cuando estás listo, compilas
npx tsc
```

```json
// tsconfig.production.json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "outDir": "./dist",
    "sourceMap": false,
    "removeComments": true
  },
  "exclude": ["**/*.test.ts", "**/*.spec.ts"]
}
```

---

🔵 **Ejemplo 3: Compilación con perfiles**

```bash
# Desarrollando: modo rápido
$ npx tsc --noEmit --watch
# Verifica tipos continuamente

# Build de desarrollo: con sourceMaps
$ npx tsc --sourceMap

# Build de producción: optimizado
$ npx tsc --project tsconfig.production.json
```

4. MODO WATCH (OBSERVADOR)
===============================================================================

El modo `--watch` (o `-w`) hace que tsc observe los archivos del proyecto y
recompile automáticamente cuando detecta cambios.

```bash
# Iniciar modo watch
npx tsc --watch

# O abreviado
npx tsc -w
```

Cuando ejecutas `npx tsc --watch`:
1. tsc compila el proyecto una vez
2. Permanece en ejecución, observando los archivos
3. Ante cualquier cambio, recompila automáticamente
4. Muestra errores en tiempo real

---

📌 **El modo watch es esencial para el desarrollo. Te da feedback inmediato
mientras escribes código.**

```bash
$ npx tsc --watch
[12:00:00] Starting compilation in watch mode...
[12:00:00] Found 0 errors. Watching for file changes.

# ... haces un cambio en un archivo .ts ...
[12:05:23] File change detected. Starting incremental compilation...
[12:05:23] Found 2 errors. Watching for file changes.

  src/index.ts:10:5 - error TS2322: Type 'string' is not assignable to type 'number'.
```

---

🔵 **Ejemplo 4: Usando watch en desarrollo**

```bash
# Terminal 1: Modo watch
$ npx tsc --watch --noEmit
[12:00:00] Starting compilation in watch mode...
[12:00:00] Found 0 errors. Watching for file changes.

# Terminal 2: Ejecutas el código tras compilar
$ node dist/index.js
```

```json
// En package.json, añade scripts para facilitar
{
  "scripts": {
    "build": "tsc",
    "watch": "tsc --watch",
    "check": "tsc --noEmit",
    "dev": "tsc --watch & nodemon dist/index.js"
  }
}
```

```bash
# Ahora puedes usar:
npm run build    # Compilar una vez
npm run watch    # Compilar continuamente
npm run check    # Solo verificar tipos
```

===============================================================================
5. BANDERAS (FLAGS) IMPORTANTES DE TSC
===============================================================================

| Flag | Descripción | Ejemplo |
|------|-------------|---------|
| `--noEmit` | Verificar tipos sin generar archivos | `tsc --noEmit` |
| `--watch` | Modo observador | `tsc --watch` |
| `--project` | Usar tsconfig específico | `tsc --project tsconfig.prod.json` |
| `--outDir` | Directorio de salida | `tsc --outDir ./build` |
| `--rootDir` | Directorio raíz | `tsc --rootDir ./source` |
| `--target` | Versión JS de salida | `tsc --target ES2022` |
| `--module` | Sistema de módulos | `tsc --module esnext` |
| `--strict` | Modo estricto | `tsc --strict` |
| `--declaration` | Generar .d.ts | `tsc --declaration` |
| `--sourceMap` | Generar .js.map | `tsc --sourceMap` |
| `--removeComments` | Quitar comentarios | `tsc --removeComments` |
| `--listFiles` | Listar archivos compilados | `tsc --listFiles` |
| `--showConfig` | Mostrar configuración efectiva | `tsc --showConfig` |
| `--help` | Mostrar ayuda completa | `tsc --help` |
| `--version` | Mostrar versión | `tsc --version` |

---

📌 **Las flags de CLI tienen prioridad sobre tsconfig.json. Son útiles para
sobrescribir configuraciones temporalmente.**

```bash
# Sobrescribir outDir desde la línea de comandos
npx tsc --outDir ./temp-build

# Combinar flags
npx tsc --noEmit --strict --target ES2022

# Depuración: ver qué archivos se están compilando
npx tsc --listFiles | grep -v node_modules
```

---

🔵 **Ejemplo 5: Usando flags en combinación**

```bash
# Escenario: quieres verificar errores en modo estricto
# pero tu tsconfig no tiene strict activado
npx tsc --noEmit --strict

# Escenario: quieres ver la configuración efectiva
npx tsc --showConfig

# Escenario: build rápido para pruebas
npx tsc --outDir ./temp --declaration false --sourceMap false
```

6. MENSAJES DE ERROR Y SU INTERPRETACIÓN
===============================================================================

Los errores de TypeScript tienen un formato específico y muy informativo:

```
src/archivo.ts:10:5 - error TS2323: Type 'X' is not assignable to type 'Y'.
              │    │       │                                    │
              │    │       │                                    └── Mensaje descriptivo
              │    │       └── Código de error único
              │    └── Columna
              └── Línea
```

**Errores comunes:**

| Código | Significado |
|--------|-------------|
| TS2322 | Tipo no asignable |
| TS2531 | Objeto posiblemente 'null' |
| TS2339 | Propiedad no existe en tipo |
| TS2345 | Argumento no asignable |
| TS2554 | Número incorrecto de parámetros |
| TS2304 | Nombre no encontrado |

---

📌 **El código de error (TS2322, TS2531, etc.) es tu mejor amigo. Búscalo en
la documentación oficial para entenderlo a fondo.**

```typescript
// Error TS2531: Object is possibly 'null'
function obtenerNombre(usuario?: { nombre: string }) {
  return usuario.nombre;  // ❌ TS2531: Object is possibly 'null'
}

// Solución 1: Verificación
function obtenerNombre(usuario?: { nombre: string }) {
  if (usuario) {
    return usuario.nombre;
  }
  return "Anónimo";
}

// Solución 2: Optional chaining
function obtenerNombre(usuario?: { nombre: string }) {
  return usuario?.nombre ?? "Anónimo";
}
```

---

🔵 **Ejemplo 6: Depurando errores comunes**

```typescript
// Error TS2322: Type 'number' is not assignable to type 'string'
let nombre: string = 42;
// ❌ error TS2322: Type 'number' is not assignable to type 'string'.

// Error TS2339: Property 'toUpperCase' does not exist on type 'number'
const edad: number = 25;
console.log(edad.toUpperCase());
// ❌ error TS2339: Property 'toUpperCase' does not exist on type 'number'.

// Error TS2554: Expected 2 arguments, but got 1
function sumar(a: number, b: number): number {
  return a + b;
}
sumar(5);
// ❌ error TS2554: Expected 2 arguments, but got 1.
```

```bash
# Salida de error en terminal
$ npx tsc --noEmit
src/index.ts:5:12 - error TS2322: Type 'number' is not assignable to type 'string'.

5 let nombre: string = 42;
             ~~~~~~
  Found 1 error.
```

7. COMPILACIÓN INCREMENTAL Y RENDIMIENTO
===============================================================================

TypeScript 3.4 introdujo la compilación incremental, que acelera las
compilaciones subsiguientes al guardar información de proyectos anteriores.

```json
{
  "compilerOptions": {
    "incremental": true,
    "tsBuildInfoFile": "./.tsbuildinfo"
  }
}
```

**Estrategias para mejorar el rendimiento del compilador:**
- Usa `--noEmit` durante el desarrollo (solo verificar tipos)
- Usa `--watch` solo cuando sea necesario
- Divide proyectos grandes en proyectos más pequeños (project references)
- Usa `skipLibCheck: true` para no verificar tipos de bibliotecas
- Usa `isolatedModules: true` para compilación por archivo

---

📌 **Project references permiten dividir un proyecto grande en subproyectos
más pequeños que se compilan de forma independiente.**

```bash
# Verificar tipos es más rápido que compilar + verificar
# Porque no genera archivos
time npx tsc --noEmit    # ~1 segundo
time npx tsc             # ~3 segundos
```

```json
// Estructura de project references
{
  "compilerOptions": {
    "composite": true
  },
  "references": [
    { "path": "./shared" },
    { "path": "./server" },
    { "path": "./client" }
  ]
}
```

---

🔵 **Ejemplo 7: Configuración de compilación incremental**

```bash
# Primera compilación: más lenta
$ npx tsc --incremental
[12:00:00] Starting compilation...
[12:00:03] Found 0 errors. (3.2s)

# Segunda compilación (sin cambios): mucho más rápida
$ npx tsc --incremental
[12:05:00] Starting compilation...
[12:05:00] Found 0 errors. (0.3s)
# ¡10x más rápido!
```

```json
// tsconfig.json con optimizaciones de rendimiento
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "commonjs",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "incremental": true,
    "tsBuildInfoFile": "./.tsbuildinfo",
    "skipLibCheck": true,
    "skipDefaultLibCheck": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

- `tsc` es el compilador oficial de TypeScript, escrito en TypeScript mismo
- tsc realiza dos funciones: verificación de tipos y transpilación a JS
- `--noEmit` verifica tipos sin generar archivos de salida
- `--watch` (o -w) observa cambios y recompila automáticamente
- Las flags de CLI tienen prioridad sobre las opciones en tsconfig.json
- Los errores tienen formato `archivo:línea:columna - código: mensaje`
- La compilación incremental (`--incremental`) acelera compilaciones repetidas
- Project references mejoran el rendimiento en proyectos grandes
- `--showConfig` y `--listFiles` son herramientas esenciales de depuración

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué es más rápido: --noEmit o compilación completa?**
`--noEmit` es más rápido porque solo analiza el código y verifica tipos,
sin generar archivos de salida.

❓ **¿Puedo usar tsc sin Node.js?**
No. tsc está escrito en TypeScript compilado a JavaScript y necesita Node.js
para ejecutarse.

❓ **¿El modo watch consume muchos recursos?**
tsc--watch está optimizado para ser eficiente. Usa compilación incremental
por defecto, por lo que solo reprocesa los archivos modificados.

❓ **¿Qué hago si tsc es muy lento?**
Activa incremental: true, skipLibCheck: true, usa --noEmit en desarrollo y
considera dividir el proyecto con project references.
