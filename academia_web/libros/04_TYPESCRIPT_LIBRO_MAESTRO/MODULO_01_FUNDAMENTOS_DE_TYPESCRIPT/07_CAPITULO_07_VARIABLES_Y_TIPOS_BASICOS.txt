1. DECLARACIÓN DE VARIABLES: LET, CONST Y VAR
===============================================================================

TypeScript soporta tres formas de declarar variables, heredadas de JavaScript
pero con supervisión de tipos.

**`let`**: Variable que puede reasignarse. Ámbito de bloque.
**`const`**: Variable constante (no reasignable). Ámbito de bloque.
**`var`**: Variable con ámbito de función (obsoleto, evitar).

---

📌 **En TypeScript moderno, usa SIEMPRE `let` o `const`. Nunca `var`.**

```typescript
// let: puede reasignarse
let edad: number = 25;
edad = 26; // ✅ Permitido

// const: no puede reasignarse
const nombre: string = "Ana";
// nombre = "María"; // ❌ Error

// const no significa inmutable, solo la referencia no se reasigna
const numeros: number[] = [1, 2, 3];
numeros.push(4); // ✅ El arreglo puede modificarse
```

---

🔵 **Ejemplo 1: Ámbito de bloque (let vs var)**

```typescript
// var tiene ámbito de función, NO de bloque
function ejemploVar() {
  var x = 1;
  if (true) {
    var x = 2; // ¡La misma variable!
    console.log(x); // 2
  }
  console.log(x); // 2 (pisó el valor exterior)
}

// let tiene ámbito de bloque
function ejemploLet() {
  let x = 1;
  if (true) {
    let x = 2; // Variable diferente
    console.log(x); // 2
  }
  console.log(x); // 1 (la variable exterior no se modificó)
}
```

2. TIPO STRING
===============================================================================

El tipo `string` representa texto. Puede escribirse con comillas simples,
comillas dobles o backticks (template literals).

```typescript
let saludo: string = "Hola";
let despedida: string = 'Adiós';
let mensaje: string = `Hola, tengo ${25} años.`;

let texto: string = "TypeScript";
console.log(texto.length);          // 10
console.log(texto.toUpperCase());   // "TYPESCRIPT"
console.log(texto.includes("Type")); // true
console.log(texto.slice(0, 4));     // "Type"
```

---

📌 **Los template literals son la forma más moderna de concatenar strings.**

```typescript
// Tradicional
let mensaje = "Hola, me llamo " + nombre + " y tengo " + edad + " años.";

// Template literal (recomendado)
let mensajeTS = `Hola, me llamo ${nombre} y tengo ${edad} años.`;
```

---

🔵 **Ejemplo 2: Funciones con strings**

```typescript
function formatearNombre(nombre: string, apellido: string): string {
  return `${nombre} ${apellido}`.trim();
}

function obtenerIniciales(nombre: string, apellido: string): string {
  return `${nombre[0].toUpperCase()}${apellido[0].toUpperCase()}`;
}

function esPalindromo(texto: string): boolean {
  const limpio = texto.toLowerCase().replace(/[^a-z]/g, "");
  return limpio === limpio.split("").reverse().join("");
}

console.log(formatearNombre("  Ana  ", "  García  "));  // "Ana García"
console.log(obtenerIniciales("maría", "lópez"));         // "ML"
console.log(esPalindromo("anitalavalatina"));            // true
```

3. TIPO NUMBER
===============================================================================

El tipo `number` representa números, tanto enteros como decimales.

```typescript
let entero: number = 42;
let decimal: number = 3.1416;
let negativo: number = -10;
let hexadecimal: number = 0xFF;  // 255
let binario: number = 0b1010;    // 10
let notacion: number = 1.5e3;    // 1500
```

---

📌 **TypeScript no diferencia entre enteros y decimales. Todo es `number`.**

```typescript
let a: number = 10;
let b: number = 3;

console.log(a + b);  // 13
console.log(a - b);  // 7
console.log(a * b);  // 30
console.log(a / b);  // 3.3333...
console.log(a % b);  // 1
console.log(a ** b); // 1000

let precio: number = 123.456;
console.log(precio.toFixed(2)); // "123.46"
```

---

🔵 **Ejemplo 3: Cálculos con tipos**

```typescript
function calcularIMC(peso: number, altura: number): number {
  if (peso <= 0 || altura <= 0) {
    throw new Error("Peso y altura deben ser positivos.");
  }
  return peso / (altura * altura);
}

function clasificarIMC(imc: number): string {
  if (imc < 18.5) return "Bajo peso";
  if (imc < 25) return "Normal";
  if (imc < 30) return "Sobrepeso";
  return "Obesidad";
}

const peso = 70;
const altura = 1.75;
const imc = calcularIMC(peso, altura);
console.log(`IMC: ${imc.toFixed(2)}`); // IMC: 22.86
console.log(clasificarIMC(imc));       // Normal
```

===============================================================================
4. TIPO BOOLEAN
===============================================================================

El tipo `boolean` representa valores de verdad: `true` o `false`.

```typescript
let activo: boolean = true;
let completado: boolean = false;

let a: boolean = true;
let b: boolean = false;

console.log(a && b);  // false
console.log(a || b);  // true
console.log(!a);      // false

let edad: number = 18;
let esMayor: boolean = edad >= 18;  // true
```

---

📌 **Usa SIEMPRE `===` y `!==`. `==` hace coerción de tipos y es peligroso.**

```typescript
console.log(0 == false);   // true (coerción)
console.log(0 === false);  // false (correcto)
console.log("" == false);  // true (coerción)
console.log("" === false); // false (correcto)
```

---

🔵 **Ejemplo 4: Booleanos en lógica de negocio**

```typescript
interface Usuario {
  nombre: string;
  edad: number;
  activo: boolean;
  premium: boolean;
}

function tieneAccesoCompleto(usuario: Usuario): boolean {
  return usuario.activo && (usuario.premium || usuario.edad >= 18);
}

const usuario1: Usuario = {
  nombre: "Ana", edad: 25, activo: true, premium: false,
};
const usuario2: Usuario = {
  nombre: "Luis", edad: 15, activo: true, premium: true,
};

console.log(tieneAccesoCompleto(usuario1)); // true
console.log(tieneAccesoCompleto(usuario2)); // true
```

5. NULL Y UNDEFINED
===============================================================================

`null` y `undefined` son tipos que representan la ausencia de valor.

- **`null`**: Ausencia intencional de valor
- **`undefined`**: Variable declarada pero no inicializada

Con `strictNullChecks` activado (recomendado), no puedes asignar `null` o
`undefined` a variables de otros tipos sin declararlo explícitamente.

```typescript
// Con strict: true
let nombre: string = null;     // ❌ Error
let edad: number = undefined;  // ❌ Error

// Forma correcta: tipos unión
let nombre2: string | null = null;    // ✅
let edad2: number | undefined = undefined; // ✅
let dato: string | null | undefined = null; // ✅
```

---

📌 **Con `strict: true`, TypeScript te obliga a manejar null/undefined explícitamente.**

```typescript
function buscarUsuario(id: number): { nombre: string } | null {
  if (id <= 0) return null;
  return { nombre: "Ana" };
}

const resultado = buscarUsuario(1);
// console.log(resultado.nombre); // ❌ Error: Object is possibly 'null'

// Solución: verificar
if (resultado !== null) {
  console.log(resultado.nombre); // ✅ "Ana"
}

// O usar optional chaining
console.log(resultado?.nombre ?? "No encontrado");
```

---

🔵 **Ejemplo 5: Manejo seguro de null**

```typescript
function dividir(a: number, b: number): number | null {
  if (b === 0) return null;
  return a / b;
}

function mostrarDivision(a: number, b: number): void {
  const resultado = dividir(a, b);
  if (resultado === null) {
    console.log("Error: No se puede dividir por cero.");
  } else {
    console.log(`${a} / ${b} = ${resultado}`);
  }
}

mostrarDivision(10, 2);  // 10 / 2 = 5
mostrarDivision(10, 0);  // Error: No se puede dividir por cero.
```

6. VOID, NEVER Y UNKNOWN
===============================================================================

**`void`**: Ausencia de valor de retorno en funciones.

```typescript
function logger(mensaje: string): void {
  console.log(mensaje);
  // No necesita return
}
```

**`never`**: Indica que una función nunca termina o siempre lanza error.

```typescript
function error(mensaje: string): never {
  throw new Error(mensaje);
  // Nunca llega al final
}

function bucleInfinito(): never {
  while (true) {
    // Nunca termina
  }
}
```

**`unknown`**: Similar a `any`, pero seguro. No puedes operar con él sin
verificar el tipo primero.

```typescript
let dato: unknown = "Hola";
dato = 42;
dato = true;

// console.log(dato.toUpperCase()); // ❌ Error: Object is of type 'unknown'

// Debes verificar el tipo primero
if (typeof dato === "string") {
  console.log(dato.toUpperCase()); // ✅ Ahora TypeScript sabe que es string
}
```

---

📌 **`unknown` es la alternativa segura a `any`. Siempre que no sepas un tipo,
usa `unknown` en lugar de `any`.**

```typescript
// ❌ any: permite cualquier operación (inseguro)
function procesarAny(valor: any): string {
  return valor.toUpperCase(); // Sin verificación, puede fallar en ejecución
}

// ✅ unknown: obliga a verificar (seguro)
function procesarUnknown(valor: unknown): string {
  if (typeof valor === "string") {
    return valor.toUpperCase();
  }
  return String(valor);
}
```

---

🔵 **Ejemplo 6: Comparativa any vs unknown**

```typescript
// any: desactiva toda verificación de tipos
let valorAny: any = "texto";
valorAny = 42;              // ✅
valorAny.toUpperCase();     // ✅ Error en ejecución, no en compilación
valorAny.noExiste();        // ✅ Error en ejecución
valorAny.propiedad.otra;    // ✅ Error en ejecución

// unknown: obliga a verificar
let valorUnknown: unknown = "texto";
valorUnknown = 42;          // ✅
// valorUnknown.toUpperCase(); // ❌ Error de compilación

// Uso correcto de unknown
function procesarEntrada(entrada: unknown): string {
  if (typeof entrada === "string") {
    return entrada.toUpperCase();
  }
  if (typeof entrada === "number") {
    return entrada.toFixed(2);
  }
  return String(entrada);
}

console.log(procesarEntrada("hola")); // "HOLA"
console.log(procesarEntrada(42));     // "42.00"
```

7. EL TIPO ANY (Y POR QUÉ EVITARLO)
===============================================================================

`any` es el tipo que desactiva la verificación de tipos para una variable.
Es útil durante migraciones de JS a TS, pero debe evitarse en código nuevo.

```typescript
// Usar any desactiva los beneficios de TypeScript
let datos: any = "Hola";
datos = 42;
datos = true;
datos.hacerAlgo();  // Sin error de compilación (pero fallará en ejecución)
```

---

📌 **Piensa en `any` como "renunciar a TypeScript para esta variable". Úsalo
solo como último recurso.**

```typescript
// ❌ Mal: abusar de any
function procesar(items: any[]): any {
  return items.map((item: any) => item.valor);
}

// ✅ Bien: usar genéricos o tipos específicos
function procesarBien<T extends { valor: number }>(items: T[]): number[] {
  return items.map((item) => item.valor);
}

// Alternativa: unknown si no sabes el tipo
function procesarSeguro(items: unknown[]): number[] {
  return items.filter((item): item is { valor: number } => {
    return typeof item === "object" && item !== null && "valor" in item;
  }).map((item) => item.valor);
}
```

---

🔵 **Ejemplo 7: Migración gradual sin any**

```typescript
// Código JS que migramos a TS
// Version original (JS):
// function calcularTotal(items) {
//   return items.reduce((acc, item) => acc + item.precio, 0);
// }

// Paso 1: Usar unknown para parámetros
function calcularTotal(items: unknown[]): number {
  return items.reduce((acc: number, item: unknown) => {
    if (typeof item === "object" && item !== null && "precio" in item) {
      const producto = item as { precio: number };
      return acc + producto.precio;
    }
    return acc;
  }, 0);
}

// Paso 2: Crear interfaz y refinar
interface Item {
  precio: number;
  nombre: string;
}

function calcularTotalTipado(items: Item[]): number {
  return items.reduce((acc, item) => acc + item.precio, 0);
}

// Paso 3: Uso correcto
const productos: Item[] = [
  { precio: 100, nombre: "Camisa" },
  { precio: 200, nombre: "Pantalón" },
];

console.log(calcularTotalTipado(productos)); // 300
```

8. TYPE ALIAS: CREANDO TIPOS PERSONALIZADOS
===============================================================================

TypeScript permite crear alias para tipos usando `type`.

```typescript
// Alias para tipos primitivos
type ID = number;
type Nombre = string;

let usuarioId: ID = 123;
let nombreUsuario: Nombre = "Ana";

// Alias para tipos unión
type Estado = "activo" | "inactivo" | "pendiente";
let estadoActual: Estado = "activo";
// estadoActual = "eliminado"; // ❌ Error

// Alias para tipos más complejos
type Coordenada = { x: number; y: number };
typeCallback = (valor: number) => boolean;

function mover(punto: Coordenada, callback: Callback): void {
  if (callback(punto.x)) {
    console.log(`Moviendo a (${punto.x}, ${punto.y})`);
  }
}
```

---

📌 **Usa `type` para crear nombres significativos para tipos complejos.
Mejora la legibilidad y el mantenimiento del código.**

```typescript
// Sistema de tipos para una aplicación de comercio electrónico
type ProductoID = string;
type Cantidad = number;
type Precio = number;
type Moneda = "USD" | "EUR" | "MXN" | "CLP";

type Producto = {
  id: ProductoID;
  nombre: string;
  precio: Precio;
  moneda: Moneda;
  stock: Cantidad;
};

type Carrito = {
  productos: Producto[];
  total: Precio;
  moneda: Moneda;
};

function calcularTotal(carrito: Carrito): Precio {
  return carrito.productos.reduce(
    (acc, prod) => acc + prod.precio,
    0
  );
}
```

---

🔵 **Ejemplo 8: Type alias para funciones**

```typescript
// Definir tipos de funciones reutilizables
type OperacionBinaria = (a: number, b: number) => number;
type Transformacion<T> = (valor: T) => T;
type Predicado<T> = (valor: T) => boolean;

// Usar los alias
const sumar: OperacionBinaria = (a, b) => a + b;
const multiplicar: OperacionBinaria = (a, b) => a * b;

const duplicar: Transformacion<number> = (n) => n * 2;
const mayusculas: Transformacion<string> = (s) => s.toUpperCase();

const esPar: Predicado<number> = (n) => n % 2 === 0;
const esLargo: Predicado<string> = (s) => s.length > 5;

function ejecutarOperacion(a: number, b: number, op: OperacionBinaria): number {
  return op(a, b);
}

console.log(ejecutarOperacion(10, 5, sumar));  // 15
console.log(ejecutarOperacion(10, 5, multiplicar)); // 50
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Usa `let` para variables que cambian y `const` para constantes. Nunca `var`
- `string`: texto, con template literals para interpolación
- `number`: enteros y decimales (no hay int/float separados)
- `boolean`: true/false, usa siempre `===` y `!==`
- `null` y `undefined`: requieren manejo explícito con strictNullChecks
- `void`: funciones que no retornan valor
- `never`: funciones que nunca terminan
- `unknown`: alternativa segura a `any`, requiere verificación de tipo
- Evita `any` siempre que sea posible
- `type` permite crear alias para tipos complejos y reutilizables

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué evitar var?**
`var` tiene ámbito de función en lugar de bloque, lo que causa bugs difíciles
de detectar. `let` y `const` tienen ámbito de bloque y son más predecibles.

❓ **¿Cuándo usar null vs undefined?**
Convención: `undefined` significa "no asignado", `null` significa "ausencia
intencional de valor". TypeScript no impone una regla, pero la comunidad
tiende a usar `null` para valores vacíos explícitos.

❓ **¿Puedo mezclar tipos en un arreglo?**
Sí, con tipos unión: `let datos: (string | number)[] = [1, "dos", 3];`

❓ **¿Debo anotar el tipo en cada variable?**
No. TypeScript infiere tipos automáticamente. Anota solo cuando la inferencia
no sea suficiente o cuando quieras ser explícito en fronteras de funciones.
