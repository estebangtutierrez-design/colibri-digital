1. ¿QUÉ ES LA INFERENCIA DE TIPOS?
===============================================================================

La **inferencia de tipos** es la capacidad de TypeScript para determinar
automáticamente el tipo de una expresión basándose en su valor o contexto.
No necesitas decirle a TypeScript "esto es un número"; él lo deduce al ver
que le asignas `42`.

```typescript
// TypeScript INFIERE el tipo automáticamente
let edad = 25;        // TypeScript infiere: number
let nombre = "Ana";   // TypeScript infiere: string
let activo = true;    // TypeScript infiere: boolean

// El tipo inferido es exactamente el mismo que si lo anotaras
let edadExplicita: number = 25;   // Igual que arriba
let nombreExplicito: string = "Ana";  // Igual que arriba
```

---

📌 **La inferencia de tipos es el "superpoder silencioso" de TypeScript.
Hace que el código sea seguro sin necesidad de anotaciones constantes.**

```typescript
// La inferencia funciona con operaciones
let a = 10;
let b = 20;
let suma = a + b;  // TypeScript infiere: number

let saludo = "Hola, ";
let nombre = "Mundo";
let mensaje = saludo + nombre;  // TypeScript infiere: string

// No necesitamos anotar estas variables
// TypeScript ya sabe que 'suma' es number y 'mensaje' es string
```

```typescript
// Inferencia con typeof
let x = 42;
let tipoDeX = typeof x;  // TypeScript infiere: "string" | "number" | "bigint" | ...
// Pero más importante: sabe que en tiempo de ejecución será "number"
```

---

🔵 **Ejemplo 1: Inferencia en acción**

```typescript
const PI = 3.14159;
// PI es del tipo: 3.14159 (literal type, no solo number)

let radio = 5;
// radio es del tipo: number

let area = PI * radio * radio;
// area es del tipo: number
// TypeScript infiere todo sin una sola anotación

console.log(`El área es: ${area}`);
```

2. INFERENCIA EN VARIABLES LOCALES
===============================================================================

En variables locales, la inferencia es muy potente y casi siempre suficiente.

```typescript
function ejemplosInferencia() {
  // Tipos primitivos
  const nombre = "Ana";         // "Ana" (literal type)
  let edad = 30;                // number
  let esActivo = true;          // boolean
  let fecha = new Date();       // Date
  let ids = [1, 2, 3];          // number[]

  // TypeScript infiere correctamente los tipos
  console.log(nombre.toUpperCase());  // ✅ "ANA"
  console.log(edad.toFixed(2));       // ✅ "30.00"
  console.log(ids.map(id => id * 2)); // ✅ [2, 4, 6]
}
```

---

📌 **Con `const`, TypeScript infiere el tipo literal más específico. Con
`let`, infiere el tipo general.**

```typescript
// Diferencia clave entre const y let en inferencia
const color = "rojo";
// color es del tipo "rojo" (literal type)
// No puede ser reasignado, entonces TypeScript usa el valor exacto

let color2 = "rojo";
// color2 es del tipo string
// Puede ser reasignado, entonces TypeScript usa el tipo general

// Esto es útil para:
function pintar(color: "rojo" | "azul" | "verde") {
  console.log(`Pintando de ${color}`);
}

const miColor = "rojo";
pintar(miColor); // ✅ "rojo" es asignable a "rojo" | "azul" | "verde"

let miColor2 = "rojo";
// pintar(miColor2); // ❌ Error: string no es asignable a "rojo" | "azul" | "verde"
```

---

🔵 **Ejemplo 2: const vs let en inferencia**

```typescript
function configurarServidor() {
  // Con const: tipo literal exacto
  const PUERTO = 3000;           // Tipo: 3000
  const HOST = "localhost";      // Tipo: "localhost"
  const PROTOCOLO = "https";     // Tipo: "https"

  // Con let: tipo general
  let puerto = 3000;             // Tipo: number
  let host = "localhost";        // Tipo: string

  // Los literales con const permiten uniones estrechas
  type Esquemas = "http" | "https";

  const esquema = "https";       // Tipo: "https"
  // let esquema2: Esquemas = "https"; // Anotación necesaria para union

  return `${PROTOCOLO}://${HOST}:${PUERTO}`;
}
```

3. INFERENCIA EN FUNCIONES: PARÁMETROS Y RETORNO
===============================================================================

TypeScript puede inferir el tipo de retorno de una función automáticamente.

```typescript
// Inferencia del tipo de retorno
function sumar(a: number, b: number) {
  return a + b;  // TypeScript infiere: number
}

function crearSaludo(nombre: string) {
  return `¡Hola, ${nombre}!`;  // TypeScript infiere: string
}

function esPar(n: number) {
  return n % 2 === 0;  // TypeScript infiere: boolean
}
```

---

📌 **Los parámetros SIEMPRE deben anotarse. El retorno puede inferirse,
pero anotarlo explícitamente mejora la documentación y previene errores.**

```typescript
// ❌ Parámetros sin tipo: cualquier cosa
function sumarMalo(a, b) {
  return a + b;  // a y b son any implícito (error con noImplicitAny)
}

// ✅ Parámetros tipados, retorno inferido
function sumarBien(a: number, b: number) {
  return a + b;
}

// ✅ Todo explícito (recomendado para funciones públicas)
function sumarExplicito(a: number, b: number): number {
  return a + b;
}
```

---

🔵 **Ejemplo 3: Inferencia de retorno en funciones complejas**

```typescript
function procesarUsuario(nombre: string, edad: number) {
  // TypeScript infiere la estructura completa del objeto retornado
  return {
    nombre: nombre.toUpperCase(),
    edad: edad,
    esMayor: edad >= 18,
    saludo: `Soy ${nombre} y tengo ${edad} años.`,
  };
  // Tipo inferido: { nombre: string; edad: number; esMayor: boolean; saludo: string }
}

const usuario = procesarUsuario("Ana", 25);
console.log(usuario.nombre);     // "ANA"
console.log(usuario.esMayor);    // true
console.log(usuario.saludo);     // "Soy Ana y tengo 25 años."
```

```typescript
// Inferencia con condicionales
function obtenerDescripcion(valor: number | string) {
  if (typeof valor === "number") {
    return `El número es ${valor}`;
  }
  return `El texto es: ${valor}`;
}
// Tipo de retorno inferido: string (ambas ramas devuelven string)

function obtenerValor(valor: number | string) {
  if (typeof valor === "number") {
    return valor * 2;
  }
  return valor.toUpperCase();
}
// Tipo de retorno inferido: string | number
```

4. INFERENCIA EN ARREGLOS Y OBJETOS
===============================================================================

TypeScript infiere los tipos de los elementos de arreglos y objetos basándose
en los valores proporcionados.

```typescript
// Arreglos homogéneos
let numeros = [1, 2, 3, 4, 5];         // number[]
let nombres = ["Ana", "Luis", "María"]; // string[]
let mixto = [1, "dos", true];           // (string | number | boolean)[]

// Objetos
let punto = { x: 10, y: 20 };
// Tipo inferido: { x: number; y: number }

let usuario = {
  id: 1,
  nombre: "Ana",
  activo: true,
};
// Tipo inferido: { id: number; nombre: string; activo: boolean }
```

---

📌 **TypeScript infiere los tipos de arreglos y objetos con gran precisión,
incluyendo tipos unión cuando hay heterogeneidad.**

```typescript
// Inferencia en arreglos con operaciones
let numeros = [1, 2, 3];
let duplicados = numeros.map(n => n * 2);  // number[] (inferido)
let pares = numeros.filter(n => n % 2 === 0); // number[] (inferido)
let primero = numeros[0]; // number (inferido)

// Inferencia en objetos con métodos
const calculadora = {
  sumar(a: number, b: number) {
    return a + b;
  },
  restar(a: number, b: number) {
    return a - b;
  },
};
// Tipo inferido: { sumar(a: number, b: number): number; restar(a: number, b: number): number }
```

---

🔵 **Ejemplo 4: Inferencia con objetos anidados**

```typescript
function crearConfiguracion() {
  return {
    servidor: {
      host: "localhost",
      puerto: 3000,
      ssl: true,
    },
    baseDeDatos: {
      host: "db.local",
      puerto: 5432,
      usuario: "admin",
    },
    debug: false,
  };
}

const config = crearConfiguracion();
// config.servidor.host → string
// config.servidor.puerto → number
// config.baseDeDatos.usuario → string

console.log(config.servidor.host);  // "localhost"
// config.debug = "true";  // ❌ Error: string no asignable a boolean
```

5. INFERENCIA CONTEXTUAL
===============================================================================

La **inferencia contextual** ocurre cuando TypeScript infiere tipos basándose
en el contexto donde se usa una expresión, no solo en su valor.

```typescript
// Inferencia contextual en callbacks
const numeros = [1, 2, 3];

// TypeScript sabe que 'n' es number porque está en un arreglo number[]
numeros.forEach((n) => {
  console.log(n.toFixed(2));  // ✅ TypeScript sabe que n es number
});

// Inferencia contextual en event listeners (en el navegador)
// document.addEventListener("click", (event) => {
//   console.log(event.clientX);  // ✅ TypeScript sabe que event es MouseEvent
// });
```

---

📌 **La inferencia contextual es especialmente útil en callbacks y funciones
que se pasan como argumentos.**

```typescript
// Inferencia contextual con tipos definidos
type Operacion = (a: number, b: number) => number;

function ejecutar(a: number, b: number, operacion: Operacion): number {
  return operacion(a, b);
}

// Los parámetros 'x' e 'y' se infieren como number por el contexto
const resultado = ejecutar(10, 5, (x, y) => {
  return x * y;  // ✅ x e y son number por inferencia contextual
});

// Inferencia contextual con arreglos de objetos
interface Producto {
  id: number;
  nombre: string;
  precio: number;
}

const productos: Producto[] = [
  { id: 1, nombre: "Camisa", precio: 500 },
  { id: 2, nombre: "Pantalón", precio: 800 },
];

// 'p' se infiere como Producto por el contexto
const nombresProductos = productos.map((p) => p.nombre);
// Tipo inferido: string[]
```

---

🔵 **Ejemplo 5: Inferencia contextual en React (simulado)**

```typescript
// Simulación de patrón React con inferencia contextual
interface Props<T> {
  items: T[];
  renderItem: (item: T, index: number) => string;
}

function Lista<T>(props: Props<T>): void {
  props.items.forEach((item, index) => {
    console.log(props.renderItem(item, index));
  });
}

// Inferencia contextual: TypeScript sabe que 'item' es { nombre: string; edad: number }
Lista(
  { nombre: "Ana", edad: 25 },
  { nombre: "Luis", edad: 30 },
], (item, index) => {
  return `${index + 1}. ${item.nombre} (${item.edad})`;
});

// Inferencia contextual: TypeScript sabe que 'item' es number
Lista([10, 20, 30], (item, index) => {
  return `#${index + 1}: ${item.toFixed(2)}`;
});
```

6. CUÁNDO ANOTAR VS CUÁNDO INFERIR
===============================================================================

Una de las habilidades más importantes es saber cuándo dejar que TypeScript
infiera y cuándo anotar explícitamente.

**Anotar explícitamente cuando:**
1. Parámetros de funciones (SIEMPRE)
2. Tipos de retorno de funciones públicas (recomendado)
3. Variables donde la inferencia no da el tipo deseado
4. Cuando quieres restringir un tipo a un subtipo específico
5. En fronteras de módulos (export)

**Dejar inferir cuando:**
1. Variables locales con inicialización clara
2. `const` con valores literales
3. Variables de retorno de funciones (si ya anotaste el retorno)
4. Cuando el tipo es obvio por el contexto

---

📌 **Regla general: sé explícito en las fronteras (parámetros, retornos
públicos) y deja que la inferencia trabaje dentro de las funciones.**

```typescript
// 🟢 BUENAS PRÁCTICAS

// Anotar parámetros y retorno
function encontrarMaximo(numeros: number[]): number | null {
  if (numeros.length === 0) return null;
  return Math.max(...numeros);
}

// Dejar inferir variables locales
const resultado = encontrarMaximo([1, 5, 3, 9, 2]);
// resultado es: number | null (inferido del retorno de la función)

if (resultado !== null) {
  console.log(`El máximo es: ${resultado}`);
}

// 🔴 MALAS PRÁCTICAS

// Anotación redundante en variable local con inicialización clara
let nombre: string = "Ana";  // ❌ Redundante: TypeScript ya infiere string

// No anotar parámetros
function procesar(dato) {  // ❌ 'dato' es any implícito
  return dato * 2;
}
```

---

🔵 **Ejemplo 6: Decisión de anotación vs inferencia**

```typescript
// Caso 1: Variable local con inicialización → INFERIR (correcto)
const API_URL = "https://api.example.com";
let timeout = 5000;

// Caso 2: Variable sin inicialización → ANOTAR (necesario)
let email: string;
// ... más código ...
email = "usuario@example.com";

// Caso 3: Quieres un tipo más amplio que el inferido → ANOTAR
const button = document.createElement("button");
// button es HTMLButtonElement (inferido)
// Si quieres HTMLElement:
// const button: HTMLElement = document.createElement("button");

// Caso 4: Arreglo vacío → ANOTAR
const items: string[] = [];
items.push("nuevo");

// Sin anotación: items sería any[]
// const items = []; // ❌ any[] (no queremos esto)

// Caso 5: Parámetros de función → SIEMPRE ANOTAR
function saludar(nombre: string): string {
  return `¡Hola, ${nombre}!`;
}

// Caso 6: Retorno de función → ANOTAR (recomendado)
function obtenerUsuario(id: number): Usuario {
  // ... lógica ...
  return { id, nombre: "Ana", activo: true };
}
```

7. INFERENCIA CON DESTRUCTURACIÓN
===============================================================================

TypeScript infiere tipos también al destructurar objetos y arreglos.

```typescript
// Destructuración de objetos
const usuario = { id: 1, nombre: "Ana", edad: 25 };

const { id, nombre, edad } = usuario;
// id: number, nombre: string, edad: number (inferidos)

// Destructuración con renombre
const { id: userId, nombre: userName } = usuario;
// userId: number, userName: string

// Destructuración de arreglos
const coordenadas = [10, 20, 30];
const [x, y, z] = coordenadas;
// x: number, y: number, z: number (inferidos)

// Rest operator
const [primero, ...resto] = [1, 2, 3, 4, 5];
// primero: number, resto: number[]
```

---

📌 **La destructuración con tipos explícitos es útil en parámetros de función.**

```typescript
// Sin destructuración
function mostrarUsuario(usuario: { nombre: string; edad: number }): void {
  console.log(`${usuario.nombre}, ${usuario.edad} años`);
}

// Con destructuración en parámetros (más limpio)
function mostrarUsuarioV2({ nombre, edad }: { nombre: string; edad: number }): void {
  console.log(`${nombre}, ${edad} años`);
}

// Con interface (mejor para tipos reutilizables)
interface Usuario {
  nombre: string;
  edad: number;
  email?: string;
}

function mostrarUsuarioV3({ nombre, edad, email }: Usuario): void {
  const contacto = email ?? "sin email";
  console.log(`${nombre} (${edad}) - ${contacto}`);
}
```

---

🔵 **Ejemplo 7: Inferencia con destructuración anidada**

```typescript
interface Direccion {
  calle: string;
  ciudad: string;
  codigoPostal: string;
}

interface Empleado {
  nombre: string;
  edad: number;
  direccion: Direccion;
  habilidades: string[];
}

const empleado: Empleado = {
  nombre: "Carlos",
  edad: 35,
  direccion: {
    calle: "Av. Siempre Viva 742",
    ciudad: "Springfield",
    codigoPostal: "12345",
  },
  habilidades: ["TypeScript", "React", "Node.js"],
};

// Destructuración anidada con tipos inferidos
const {
  nombre,
  direccion: { ciudad, codigoPostal },
  habilidades: [principal, ...secundarias],
} = empleado;

console.log(`${nombre} vive en ${ciudad} (CP: ${codigoPostal})`);
console.log(`Habilidad principal: ${principal}`);
console.log(`Otras habilidades: ${secundarias.join(", ")}`);
```

8. CAPTURA DE INFERENCIA CON TYPEOF
===============================================================================

TypeScript ofrece `typeof` para capturar el tipo inferido de una variable.

```typescript
const config = {
  url: "https://api.example.com",
  puerto: 443,
  ssl: true,
};

// Capturar el tipo de config
type TipoConfig = typeof config;
// { url: string; puerto: number; ssl: boolean }

// Usar el tipo capturado
function clonarConfig(original: TipoConfig): TipoConfig {
  return { ...original };
}

const copia = clonarConfig(config);
```

---

📌 **`typeof` en TypeScript no es el operador de JavaScript; es una consulta
de tipo en tiempo de compilación.**

```typescript
// typeof en tiempo de compilación (TypeScript)
const circulo = { radio: 5, color: "rojo" as const };
type CirculoType = typeof circulo;
// { radio: number; readonly color: "rojo" }

// typeof en tiempo de ejecución (JavaScript)
console.log(typeof 42);        // "number"
console.log(typeof "hola");    // "string"
console.log(typeof {});        // "object"
```

```typescript
// Uso práctico de typeof para reutilizar tipos
function crearUsuario() {
  return {
    id: Math.random(),
    nombre: "Usuario",
    creado: new Date(),
  };
}

type UsuarioRetornado = ReturnType<typeof crearUsuario>;
// { id: number; nombre: string; creado: Date }

// Ahora podemos usar el tipo inferido
function actualizarUsuario(usuario: UsuarioRetornado): UsuarioRetornado {
  return { ...usuario, nombre: "Actualizado" };
}
```

---

🔵 **Ejemplo 8: typeof para constantes complejas**

```typescript
const ERROR_CODES = {
  NOT_FOUND: 404,
  UNAUTHORIZED: 401,
  INTERNAL_ERROR: 500,
} as const;

type ErrorCode = typeof ERROR_CODES[keyof typeof ERROR_CODES];
// 404 | 401 | 500

function manejarError(codigo: ErrorCode): string {
  switch (codigo) {
    case 404: return "No encontrado";
    case 401: return "No autorizado";
    case 500: return "Error interno";
    default: return "Error desconocido";
  }
}

console.log(manejarError(ERROR_CODES.NOT_FOUND)); // "No encontrado"
// manejarError(403); // ❌ Error: 403 no está en ErrorCode
```

RESUMEN DEL CAPÍTULO
===============================================================================

- TypeScript infiere tipos automáticamente a partir de los valores asignados
- Con `const` infiere tipos literales exactos; con `let` infiere tipos generales
- Los parámetros de función SIEMPRE deben anotarse explícitamente
- El tipo de retorno puede inferirse, pero anotarlo es buena práctica
- La inferencia contextual ayuda en callbacks y funciones pasadas como argumento
- Anota explícitamente en fronteras (parámetros, exports); infiere en locales
- Usa destructuración con tipos para código más limpio
- `typeof` captura tipos inferidos para reutilización
- La inferencia bien usada hace el código más conciso sin perder seguridad

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Debo siempre anotar los tipos de retorno?**
Es una buena práctica para funciones públicas o exports. Para funciones
privadas internas, la inferencia suele ser suficiente.

❓ **¿La inferencia de tipos afecta el rendimiento?**
No. La inferencia ocurre en tiempo de compilación. No tiene ningún impacto
en el rendimiento en ejecución.

❓ **¿Qué pasa si TypeScript infiere un tipo incorrecto?**
Si la inferencia no produce el tipo que deseas, puedes anotarlo
explícitamente. La anotación siempre tiene prioridad sobre la inferencia.

❓ **¿Puedo desactivar la inferencia?**
No se puede desactivar, pero no deberías querer hacerlo. La inferencia es
una característica que hace que TypeScript sea más productivo.
