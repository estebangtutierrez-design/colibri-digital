1. ¿QUÉ NECESITAMOS INSTALAR?
===============================================================================

Para trabajar con TypeScript necesitamos tres componentes principales:

1. **Node.js**: Entorno de ejecución de JavaScript fuera del navegador.
   TypeScript se compila a JavaScript, y ese JavaScript se ejecuta con Node.js.

2. **npm** (Node Package Manager): Gestor de paquetes que viene con Node.js.
   Lo usaremos para instalar TypeScript y otras herramientas.

3. **TypeScript Compiler** (`tsc`): El compilador que transforma archivos `.ts`
   a `.js`. Se instala mediante npm.

---

📌 **Node.js y npm son los cimientos sobre los que se construye todo el
ecosistema TypeScript moderno.**

```bash
# Al final de la instalación, podremos ejecutar estos comandos:
node --version   # v22.x.x o superior
npm --version    # 10.x.x o superior
tsc --version    # Version 5.x.x o superior
```

---

```

```

2. INSTALACIÓN DE NODE.JS
===============================================================================

Node.js está disponible para Windows, macOS y Linux. Recomendamos instalar
la versión **LTS (Long Term Support)**, que es la más estable.

**Opción 1: Instalador oficial (recomendado para principiantes)**
Visita https://nodejs.org y descarga el instalador LTS para tu sistema
operativo. Ejecútalo y sigue los pasos.

**Opción 2: nvm (Node Version Manager) - recomendado para desarrolladores**
`nvm` permite instalar y cambiar entre múltiples versiones de Node.js.

---

📌 **Usar nvm te permite cambiar fácilmente entre versiones de Node.js para
diferentes proyectos.**

```bash
# Linux/macOS: Instalar nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Instalar la versión LTS más reciente
nvm install --lts

# Usar la versión LTS
nvm use --lts

# Verificar
node --version
```

```powershell
# Windows: Usar nvm-windows
# Descargar de: https://github.com/coreybutler/nvm-windows/releases
nvm install lts
nvm use lts
```

---

🔵 **Ejemplo 2: Verificación de instalación de Node.js**

```bash
# Abre tu terminal y ejecuta:
$ node --version
v22.12.0

$ npm --version
10.9.0

# Si ves números de versión, ¡Node.js está instalado correctamente!
```

===============================================================================
3. COMPRENDIENDO NODE.JS Y NPM
===============================================================================

Antes de instalar TypeScript, es importante entender qué son Node.js y npm.

**Node.js** es un entorno de ejecución de JavaScript basado en el motor V8 de
Chrome. Fue creado por Ryan Dahl en 2009 y permite ejecutar JavaScript fuera
del navegador.

**npm** es el gestor de paquetes más grande del mundo. Viene incluido con
Node.js y nos permite instalar bibliotecas y herramientas como TypeScript.

Conceptos clave de npm:
- **Package**: Una biblioteca o herramienta reutilizable
- **Dependency**: Un paquete del que depende tu proyecto
- **node_modules**: Carpeta donde npm instala los paquetes
- **package.json**: Archivo que describe tu proyecto y sus dependencias

---

📌 **npm no es el único gestor de paquetes. También existen yarn y pnpm,
pero npm es el estándar y viene con Node.js.**

```bash
# Comandos básicos de npm
npm init -y                    # Crear package.json rápidamente
npm install <paquete>          # Instalar un paquete
npm install -g <paquete>       # Instalar globalmente
npm uninstall <paquete>        # Desinstalar
npm update                     # Actualizar paquetes
npm list                       # Listar paquetes instalados
```

---

🔵 **Ejemplo 3: Creando tu primer package.json**

```bash
# Crea una carpeta para tu proyecto
mkdir mi-primer-proyecto-ts
cd mi-primer-proyecto-ts

# Inicializa el proyecto (crea package.json)
npm init -y

# Ver el contenido del archivo creado
cat package.json
```

```json
// Resultado: package.json
{
  "name": "mi-primer-proyecto-ts",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {},
  "keywords": [],
  "author": "",
  "license": "ISC"
}
```

4. INSTALACIÓN DE TYPESCRIPT GLOBAL
===============================================================================

TypeScript se instala como un paquete de npm. Puedes instalarlo de forma
**global** (disponible en todo el sistema) o **local** (solo en el proyecto
actual).

**Instalación global:**

```bash
# Instalar TypeScript globalmente
npm install -g typescript

# Verificar la instalación
tsc --version
# Output esperado: Version 5.x.x
```

---

📌 **La instalación global es útil para tener `tsc` disponible en cualquier
carpeta. Sin embargo, para proyectos serios se recomienda la instalación local.**

```bash
# La instalación global coloca tsc en el PATH del sistema
which tsc
# Linux/macOS: /usr/local/bin/tsc
# Windows: C:\Users\TuUsuario\AppData\Roaming\npm\tsc
```

---

🔵 **Ejemplo 4: Instalación global paso a paso**

```bash
# Paso 1: Instalar TypeScript globalmente
$ npm install -g typescript

added 1 package in 2s

# Paso 2: Verificar la instalación
$ tsc --version
Version 5.5.4

# Paso 3: Ver la ayuda del comando
$ tsc --help
# (Muestra todas las opciones del compilador)
```

===============================================================================
5. INSTALACIÓN DE TYPESCRIPT LOCAL
===============================================================================

Para proyectos reales, se recomienda instalar TypeScript como dependencia de
desarrollo local. Esto asegura que todos los miembros del equipo usen la misma
versión.

```bash
# Crear proyecto e instalar TypeScript localmente
mkdir mi-proyecto-ts
cd mi-proyecto-ts
npm init -y
npm install --save-dev typescript

# Para ejecutar tsc local, usa npx
npx tsc --version
```

---

📌 **La instalación local es la práctica recomendada en la industria. Cada
proyecto tiene su propia versión de TypeScript.**

```bash
# Diferencia entre global y local:
# Global:  tsc --version
# Local:   npx tsc --version
#         ./node_modules/.bin/tsc --version
```

---

🔵 **Ejemplo 5: Verificación de instalación local**

```bash
# Ver package.json después de instalar localmente
$ cat package.json
{
  "name": "mi-proyecto-ts",
  "version": "1.0.0",
  "devDependencies": {
    "typescript": "^5.5.4"
  }
}

# Ver que tsc existe en node_modules
$ ls node_modules/.bin/tsc
node_modules/.bin/tsc

# Ejecutar tsc local con npx
$ npx tsc --version
Version 5.5.4
```

6. HERRAMIENTAS ALTERNATIVAS: TS-NODE, TSX, DENO, BUN
===============================================================================

Además del flujo tradicional (tsc + node), existen herramientas modernas que
simplifican la ejecución de TypeScript:

**ts-node**: Ejecuta TypeScript directamente sin compilación previa
**tsx**: Alternativa moderna a ts-node (más rápida, basada en esbuild)
**Deno**: Runtime que soporta TypeScript nativamente
**Bun**: Runtime todo-en-uno con soporte nativo de TypeScript

```bash
# Instalar ts-node globalmente
npm install -g ts-node

# Ejecutar TypeScript directamente
ts-node mi-archivo.ts

# Instalar tsx
npm install -g tsx
tsx mi-archivo.ts
```

---

📌 **tsx (TypeScript Execute) es la alternativa moderna a ts-node: es más
rápido porque usa esbuild para la transpilación.**

```typescript
// archivo: saludo.ts
function saludar(nombre: string): string {
  return `¡Hola, ${nombre}!`;
}

console.log(saludar("Mundo"));
```

```bash
# Ejecutar directamente con tsx
$ tsx saludo.ts
¡Hola, Mundo!
```

---

🔵 **Ejemplo 6: TypeScript en Deno y Bun**

```bash
# Deno (instalación)
curl -fsSL https://deno.land/install.sh | sh
deno run --allow-read mi-archivo.ts

# Bun (instalación)
curl -fsSL https://bun.sh/install | bash
bun run mi-archivo.ts
```

```typescript
// Deno tiene soporte nativo de TypeScript
// No necesita tsconfig.json ni instalación de tipos
const url = "https://api.github.com/users/octocat";
const response = await fetch(url);
const data = await response.json();
console.log(data.name); // "The Octocat"
```

7. SOLUCIÓN DE PROBLEMAS COMUNES
===============================================================================

**Problema 1: "comando no encontrado" después de instalar globalmente**

```bash
# Solución: Verificar el PATH
npm config get prefix
# Añadir la carpeta bin al PATH en ~/.bashrc o ~/.zshrc
export PATH="$PATH:$(npm config get prefix)/bin"
```

**Problema 2: Permisos al instalar globalmente en Linux/macOS**

```bash
# Solución: Usar nvm para evitar problemas de permisos
# O configurar npm para instalar en un directorio local
npm config set prefix ~/.npm-global
```

**Problema 3: Versión incorrecta de Node.js**

```bash
# Solución: Usar nvm para cambiar de versión
nvm install --lts
nvm use --lts
```

**Problema 4: Error EACCES en macOS/Linux**

```bash
# Solución temporal (no recomendada para producción)
sudo npm install -g typescript
# Solución recomendada: usar nvm
```

---

📌 **La mayoría de los problemas de instalación se resuelven usando nvm y
evitando la instalación global de paquetes.**

```bash
# Comprobación rápida de salud del entorno
echo "Node: $(node --version)"
echo "npm: $(npm --version)"
echo "npx: $(npx --version)"
echo "tsc: $(npx tsc --version 2>/dev/null || echo 'No instalado')"
```

---

🔵 **Ejemplo 7: Script de verificación completa**

```bash
#!/bin/bash
# check-ts-env.sh
echo "=== Verificación del Entorno TypeScript ==="

if command -v node &> /dev/null; then
    echo "✅ Node.js: $(node --version)"
else
    echo "❌ Node.js no encontrado"
fi

if command -v npm &> /dev/null; then
    echo "✅ npm: $(npm --version)"
else
    echo "❌ npm no encontrado"
fi

if npx tsc --version &> /dev/null; then
    echo "✅ TypeScript: $(npx tsc --version)"
else
    echo "❌ TypeScript no instalado"
fi
```

```bash
# Ejecutar el script
$ bash check-ts-env.sh
=== Verificación del Entorno TypeScript ===
✅ Node.js: v22.12.0
✅ npm: 10.9.0
✅ TypeScript: Version 5.5.4
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Necesito instalar Node.js para usar TypeScript?**
Sí, porque el compilador tsc está escrito en TypeScript (que se compila a
JavaScript) y se distribuye como paquete de npm.

❓ **¿Qué versión de Node.js debo usar?**
Siempre la versión LTS (Long Term Support) más reciente. Al momento de
escribir este libro, es Node.js 22 LTS.

❓ **¿Puedo tener múltiples versiones de TypeScript?**
Sí. Puedes tener una versión global y versiones locales diferentes en cada
proyecto. Es una práctica común y recomendada.

❓ **¿Qué hago si tsc --version da un error?**
Verifica que TypeScript esté instalado con `npm list -g typescript` (global)
o `npm list typescript` (local). Si no aparece, instálalo.
