1. ¿QUÉ ES TYPESCRIPT?
===============================================================================

TypeScript es un **superconjunto de JavaScript** creado por Microsoft en 2012.
Esto significa que todo código JavaScript válido también es código TypeScript
válido. TypeScript añade **tipado estático opcional**, lo que nos permite
detectar errores antes de que el código se ejecute, directamente en el editor
o durante la compilación.

El código TypeScript no se ejecuta directamente en el navegador o en Node.js;
primero debe ser **transpilado** (compilado) a JavaScript mediante el compilador
oficial `tsc` (TypeScript Compiler). El resultado es JavaScript puro que puede
ejecutarse en cualquier entorno que soporte JS.

---

📌 **TypeScript es JavaScript con tipos.** No reinventa la rueda; la hace más
segura.

```typescript
// JavaScript válido → también es TypeScript válido
let mensaje = "Hola, mundo";
console.log(mensaje);
```

```typescript
// TypeScript añade tipos opcionales
let mensaje: string = "Hola, mundo";
// mensaje = 42; // ❌ Error: Type 'number' not assignable to type 'string'
```

---

🔵 **Ejemplo 1: JavaScript vs TypeScript**

```typescript
// JavaScript (sin tipos)
function sumar(a, b) {
  return a + b;
}
console.log(sumar(5, 10));      // 15
console.log(sumar("5", 10));    // "510" → ¡error silencioso!

// TypeScript (con tipos)
function sumarTS(a: number, b: number): number {
  return a + b;
}
console.log(sumarTS(5, 10));    // 15
// console.log(sumarTS("5", 10)); // ❌ Error en compilación
```

2. PROBLEMAS QUE RESUELVE TYPESCRIPT
===============================================================================

JavaScript es un lenguaje increíblemente flexible, pero esa flexibilidad tiene
un costo: muchos errores solo se descubren en tiempo de ejecución. TypeScript
traslada la detección de errores al momento de escribir el código.

**Problemas comunes que TypeScript ayuda a evitar:**

| Problema | JavaScript | TypeScript |
|----------|-----------|------------|
| Llamar a funciones con argumentos incorrectos | Solo falla en ejecución | Error en el editor |
| Acceder a propiedades inexistentes | Devuelve `undefined` | Error de compilación |
| Errores tipográficos en nombres | Silencioso | Error inmediato |
| Refactorización insegura | Hay que revisar manualmente | El compilador guía |

---

📌 **El 15% de los errores en producción de código JavaScript se podrían haber
evitado con TypeScript.** (Fuente: Microsoft Research)

```typescript
// Acceso a propiedades inexistentes
interface Usuario {
  nombre: string;
  edad: number;
}

const usuario: Usuario = { nombre: "Ana", edad: 30 };
// console.log(usuario.email); // ❌ Error: Property 'email' does not exist
```

---

🔵 **Ejemplo 2: Refactorización segura**

```typescript
// Supón que cambias el nombre de una propiedad
interface Producto {
  id: number;
  // nombre: string;  ← renombramos a "titulo"
  titulo: string;
  precio: number;
}

function mostrarProducto(p: Producto) {
  // console.log(p.nombre); // ❌ TypeScript marca el error al instante
  console.log(p.titulo);     // ✅ Correcto
}
```

3. TIPADO ESTÁTICO VS TIPADO DINÁMICO
===============================================================================

Para entender TypeScript, primero debemos entender la diferencia fundamental
entre tipado estático y dinámico.

**JavaScript** usa tipado dinámico: una variable puede contener cualquier tipo
de valor y puede cambiar de tipo durante la ejecución.

**TypeScript** añade tipado estático opcional: declaramos el tipo de una
variable y el compilador verifica que solo reciba valores de ese tipo.

---

📌 **El tipado estático no te quita flexibilidad; te da seguridad.**

```typescript
// Tipado dinámico (JS)
let dato = "texto";
dato = 42;        // ✅ Válido en JS, pero propenso a errores
dato = true;      // ✅ También válido

// Tipado estático (TS)
let datoTS: string = "texto";
// datoTS = 42;   // ❌ Error: Type 'number' not assignable to type 'string'
```

---

🔵 **Ejemplo 3: Beneficios en funciones complejas**

```typescript
// Sin TypeScript — ¿qué espera esta función? ¿qué devuelve?
function procesar(data, config) {
  return data.filter(config.criterio).map(config.transformacion);
}

// Con TypeScript — todo está documentado en el código
interface Config<T> {
  criterio: (item: T) => boolean;
  transformacion: (item: T) => unknown;
}

function procesarTS<T>(data: T[], config: Config<T>): unknown[] {
  return data.filter(config.criterio).map(config.transformacion);
}
```

4. ARQUITECTURA DE TYPESCRIPT
===============================================================================

TypeScript no es solo un lenguaje; es un ecosistema completo que incluye:

1. **Lenguaje**: La sintaxis extendida de JavaScript con tipos
2. **Compilador** (`tsc`): Transpila TypeScript a JavaScript
3. **Servicio de lenguaje**: Potencia editores con autocompletado, navegación,
   refactorización y diagnóstico en tiempo real
4. **Lenguaje de definición de tipos** (`.d.ts`): Describe tipos de bibliotecas
   JavaScript existentes

El flujo de trabajo típico es:

```
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  Escribes TS │ ───> │  Compilador  │ ───> │  JavaScript  │
│  (.ts files) │      │  (tsc)       │      │  (.js files) │
└──────────────┘      └──────────────┘      └──────────────┘
                              │
                              ▼
                      ┌──────────────┐
                      │  Errores en  │
                      │  tiempo de   │
                      │  compilación │
                      └──────────────┘
```

---

📌 **TypeScript no afecta el rendimiento en ejecución.** Los tipos se eliminan
durante la compilación. El JavaScript generado no contiene ninguna anotación
de tipo.

```typescript
// Código TypeScript original
function saludar(nombre: string): string {
  return `¡Hola, ${nombre}!`;
}

// JavaScript generado (los tipos desaparecen)
function saludar(nombre) {
  return `¡Hola, ${nombre}!`;
}
```

---

🔵 **Ejemplo 4: El servicio de lenguaje en acción**

```typescript
// Al escribir esto en VS Code...
const numeros = [1, 2, 3, 4, 5];

// TypeScript infiere el tipo y ofrece autocompletado
numeros.map((n) => n.toFixed(2));   // ✅ toFixed aparece en autocompletado
// numeros.map((n) => n.toLowerCase()); // ❌ TS sabe que 'n' es number, no string
```

5. TYPESCRIPT EN EL ECOSISTEMA MODERNO
===============================================================================

TypeScript se ha convertido en el estándar de facto para el desarrollo web
profesional. Frameworks como Angular, React (con Create React App), Vue 3,
Next.js, NestJS, Deno y Bun tienen soporte nativo o excelente integración
con TypeScript.

**Empresas que usan TypeScript en producción:**
- Microsoft (lógico, ¿verdad?)
- Google (Angular está escrito en TS)
- Airbnb (migró su código base a TypeScript)
- Slack, Asana, Lyft, y cientos más

Las estadísticas de Stack Overflow 2024 muestran que TypeScript es el segundo
lenguaje más querido entre los desarrolladores, solo superado por Rust.

---

📌 **Aprender TypeScript hoy es invertir en tu futuro profesional.**

```typescript
// Ejemplo real: definición de tipos en una API
interface ApiResponse<T> {
  success: boolean;
  data: T;
  error?: string;
  timestamp: number;
}

async function fetchUser(id: number): Promise<ApiResponse<Usuario>> {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
  // ✅ Tipado completo desde el servidor hasta la UI
}
```

---

🔵 **Ejemplo 5: TypeScript en el frontend moderno**

```typescript
// React con TypeScript
interface Props {
  titulo: string;
  items: string[];
  onSelect: (item: string) => void;
}

function ListaComponente({ titulo, items, onSelect }: Props) {
  return (
    <div>
      <h2>{titulo}</h2>
      <ul>
        {items.map((item) => (
          <li key={item} onClick={() => onSelect(item)}>
            {item}
          </li>
        ))}
      </ul>
    </div>
  );
}
```

6. MITOS COMUNES SOBRE TYPESCRIPT
===============================================================================

**Mito 1: "TypeScript es un lenguaje completamente diferente a JavaScript"**
Realidad: Es un superconjunto. Cualquier código JS es código TS válido.

**Mito 2: "TypeScript hace el código más lento"**
Realidad: TypeScript no afecta el rendimiento en ejecución; los tipos se
eliminan durante la compilación.

**Mito 3: "TypeScript solo sirve para proyectos grandes"**
Realidad: Incluso en proyectos pequeños, TypeScript mejora la productividad
con autocompletado y detección temprana de errores.

**Mito 4: "Aprender TypeScript es muy difícil"**
Realidad: Si sabes JavaScript, ya sabes la mayor parte. Los tipos son una
capa adicional que puedes ir adoptando gradualmente.

---

📌 **TypeScript es gradual: puedes empezar con `any` y añadir tipos poco a poco.**

```typescript
// Migración gradual: empieza así
let dato: any = "texto";
dato = 42;       // ✅ Permitido con 'any'

// Y refinas progresivamente
let dato2: string | number = "texto";
dato2 = 42;      // ✅ Permitido (union type)
// dato2 = true; // ❌ Error: boolean no está en la unión
```

---

🔵 **Ejemplo 6: Progresividad en la práctica**

```typescript
// Paso 1: JS puro (ya es TS válido)
function sumar(a, b) { return a + b; }

// Paso 2: Añadir tipos a los parámetros
function sumar(a: number, b: number) { return a + b; }

// Paso 3: Añadir tipo de retorno
function sumar(a: number, b: number): number { return a + b; }

// Paso 4: Usar interfaces para objetos más complejos
interface Operacion {
  a: number;
  b: number;
  operador: "suma" | "resta";
}
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿TypeScript reemplazará a JavaScript?**
No. TypeScript depende completamente de JavaScript para ejecutarse. Siempre se
compila a JavaScript. Es más probable que TypeScript se convierta en la forma
predeterminada de escribir JavaScript.

❓ **¿Puedo usar bibliotecas de JavaScript con TypeScript?**
Sí. TypeScript puede usar cualquier biblioteca JavaScript. Para tener
autocompletado y verificación de tipos, necesitas archivos de declaración
(`.d.ts`), que la comunidad mantiene en DefinitelyTyped.

❓ **¿TypeScript es solo para aplicaciones web?**
No. TypeScript se usa también en backend (Node.js, Deno, Bun), aplicaciones
móviles (React Native, NativeScript) e incluso escritorio (Electron).

❓ **¿Cuánto tiempo toma aprender TypeScript?**
Si ya sabes JavaScript, puedes sentirte productivo en TypeScript en una o dos
semanas. Dominar el sistema de tipos avanzado puede llevar meses, pero no es
necesario para empezar.
