1. ¿QUÉ ES LA ABSTRACCIÓN?
===============================================================================

La abstracción es el proceso de ocultar los detalles de implementación y
mostrar solo la funcionalidad esencial al usuario. En términos prácticos,
es como el panel de control de un coche: no necesitas saber cómo funciona
el motor para conducir; solo necesitas el volante, los pedales y el
tablero.

```typescript
// Abstracción: definimos QUÉ hace, no CÓMO lo hace
interface ReproductorMusica {
  reproducir(): void;
  pausar(): void;
  detener(): void;
  siguiente(): void;
  anterior(): void;
  obtenerCancionActual(): string;
}

// Implementación concreta: aquí está el CÓMO
class ReproductorSpotify implements ReproductorMusica {
  private canciones: string[] = [];
  private indiceActual: number = 0;
  private reproduciendo: boolean = false;

  constructor(canciones: string[]) {
    this.canciones = canciones;
  }

  reproducir(): void {
    this.reproduciendo = true;
    console.log(`▶️ Reproduciendo: ${this.canciones[this.indiceActual]}`);
  }

  pausar(): void {
    this.reproduciendo = false;
    console.log("⏸️ Reproducción pausada");
  }

  detener(): void {
    this.reproduciendo = false;
    this.indiceActual = 0;
    console.log("⏹️ Reproducción detenida");
  }

  siguiente(): void {
    this.indiceActual = (this.indiceActual + 1) % this.canciones.length;
    this.reproducir();
  }

  anterior(): void {
    this.indiceActual = (this.indiceActual - 1 + this.canciones.length)
      % this.canciones.length;
    this.reproducir();
  }

  obtenerCancionActual(): string {
    return this.canciones[this.indiceActual];
  }
}

// Quien usa el reproductor solo conoce la abstracción
function usarReproductor(reproductor: ReproductorMusica): void {
  reproductor.reproducir();
  reproductor.siguiente();
  reproductor.pausar();
}

const spotify = new ReproductorSpotify([
  "Bohemian Rhapsody",
  "Stairway to Heaven",
  "Hotel California"
]);

usarReproductor(spotify);
```

La abstracción permite que quien usa el código no dependa de los detalles
de implementación. Esto facilita cambiar la implementación sin afectar
a los consumidores.

2. ABSTRACCIÓN CON CLASES ABSTRACTAS
===============================================================================

Una clase abstracta es una clase que no puede ser instanciada directamente.
Sirve como base para otras clases y puede contener tanto métodos concretos
(con implementación) como métodos abstractos (sin implementación).

```typescript
abstract class Animal {
  constructor(protected nombre: string) {}

  // Método concreto: compartido por todos
  dormir(): void {
    console.log(`${this.nombre} está durmiendo... 😴`);
  }

  // Método abstracto: cada subclase debe implementarlo
  abstract hacerSonido(): void;

  // Método concreto que usa el abstracto
  presentarse(): void {
    console.log(`Soy ${this.nombre}`);
    this.hacerSonido();
  }
}

// const animal = new Animal("Genérico"); // ❌ Error: no se puede instanciar

class Perro extends Animal {
  constructor(nombre: string) {
    super(nombre);
  }

  hacerSonido(): void {
    console.log(`${this.nombre} dice: ¡Guau! 🐕`);
  }
}

class Gato extends Animal {
  constructor(nombre: string) {
    super(nombre);
  }

  hacerSonido(): void {
    console.log(`${this.nombre} dice: ¡Miau! 🐱`);
  }
}

const mascotas: Animal[] = [
  new Perro("Rex"),
  new Gato("Luna")
];

mascotas.forEach(m => m.presentarse());
// "Soy Rex"
// "Rex dice: ¡Guau!"
// "Soy Luna"
// "Luna dice: ¡Miau!"
```

Las clases abstractas permiten compartir implementación común (como
`dormir` y `presentarse`) mientras se fuerza a las subclases a
implementar ciertos métodos (como `hacerSonido`).

===============================================================================
3. ABSTRACCIÓN CON INTERFACES
===============================================================================

Las interfaces definen contratos puros: solo declaran la forma que debe
tener un objeto, sin ninguna implementación. Son el máximo nivel de
abstracción.

```typescript
interface Volador {
  volar(): void;
  aterrizar(): void;
}

interface Nadador {
  nadar(): void;
}

// Una clase puede implementar múltiples interfaces
class Pato implements Volador, Nadador {
  volar(): void {
    console.log("El pato vuela en formación 🦆");
  }

  aterrizar(): void {
    console.log("El pato aterriza en el agua");
  }

  nadar(): void {
    console.log("El pato nada elegantemente");
  }
}

class Avion implements Volador {
  volar(): void {
    console.log("El avión despega ✈️");
  }

  aterrizar(): void {
    console.log("El avión aterriza");
  }
}

class Pez implements Nadador {
  nadar(): void {
    console.log("El pez nada 🐟");
  }
}

function lanzarVoladores(voladores: Volador[]): void {
  voladores.forEach(v => v.volar());
}

const voladores: Volador[] = [
  new Pato(),
  new Avion()
];

lanzarVoladores(voladores);
```

Las interfaces permiten polimorfismo transversal: clases que no
comparten una jerarquía de herencia pueden implementar la misma
interfaz.

===============================================================================
4. CLASE ABSTRACTA VS INTERFAZ
===============================================================================

Esta es una de las preguntas más importantes en TypeScript. ¿Cuándo
usar cada una?

| Característica           | Clase Abstracta         | Interfaz               |
|--------------------------|-------------------------|------------------------|
| Implementación           | Puede tenerla           | No (solo declaración)  |
| Métodos concretos        | ✅ Sí                   | ❌ No                  |
| Propiedades             | ✅ Sí (con valores)     | ✅ Sí (solo firma)     |
| Constructores           | ✅ Sí                   | ❌ No                  |
| Modificadores de acceso | ✅ public, protected... | ❌ Solo public         |
| Herencia múltiple       | ❌ (solo una)           | ✅ (múltiples)         |
| Instanciable            | ❌                      | ❌                     |
| Uso típico              | Base con código común   | Contrato puro          |

```typescript
// Usa INTERFAZ cuando quieras definir un contrato puro
interface Enviable {
  enviar(destino: string): Promise<boolean>;
}

// Usa CLASE ABSTRACTA cuando tengas implementación que compartir
abstract class ServicioEnvio {
  protected abstract validarDestino(destino: string): boolean;

  abstract enviar(destino: string): Promise<boolean>;

  // Implementación compartida
  protected log(mensaje: string): void {
    console.log(`[${new Date().toISOString()}] ${mensaje}`);
  }
}

class ServicioCorreo extends ServicioEnvio {
  protected validarDestino(destino: string): boolean {
    return destino.includes("@") && destino.includes(".");
  }

  async enviar(destino: string): Promise<boolean> {
    if (!this.validarDestino(destino)) {
      this.log(`Destino inválido: ${destino}`);
      return false;
    }
    this.log(`Enviando correo a ${destino}`);
    return true;
  }
}

// Múltiples interfaces: una clase puede ser Enviable y Exportable
interface Exportable {
  exportar(formato: string): string;
}

class Factura implements Enviable, Exportable {
  constructor(private numero: string, private total: number) {}

  async enviar(destino: string): Promise<boolean> {
    console.log(`Enviando factura ${this.numero} a ${destino}`);
    return true;
  }

  exportar(formato: string): string {
    return `Factura #${this.numero}: $${this.total} [${formato}]`;
  }
}
```

5. PRINCIPIO DE INVERSIÓN DE DEPENDENCIAS (DIP)
===============================================================================

La abstracción es fundamental para cumplir con el Principio de Inversión
de Dependencias (DIP), el último de los principios SOLID. DIP dice:

1. Los módulos de alto nivel no deben depender de módulos de bajo nivel.
   Ambos deben depender de abstracciones.
2. Las abstracciones no deben depender de detalles. Los detalles deben
   depender de las abstracciones.

```typescript
// ❌ Violación de DIP: el módulo de alto nivel depende de detalles
class BaseDatosMySQL {
  conectar(): void { console.log("Conectando a MySQL"); }
  guardarUsuario(datos: any): void { console.log("Guardando en MySQL"); }
}

class ServicioUsuarios {
  private db: BaseDatosMySQL;

  constructor() {
    this.db = new BaseDatosMySQL(); // Dependencia directa
  }

  registrarUsuario(nombre: string, email: string): void {
    this.db.conectar();
    this.db.guardarUsuario({ nombre, email });
  }
}

// ✅ Correcto: ambos dependen de abstracciones
interface BaseDatos {
  conectar(): void;
  guardar(coleccion: string, datos: any): void;
}

class MySQL implements BaseDatos {
  conectar(): void { console.log("Conectando a MySQL"); }
  guardar(coleccion: string, datos: any): void {
    console.log(`Guardando en MySQL.${coleccion}`);
  }
}

class MongoDB implements BaseDatos {
  conectar(): void { console.log("Conectando a MongoDB"); }
  guardar(coleccion: string, datos: any): void {
    console.log(`Guardando en MongoDB.${coleccion}`);
  }
}

class ServicioUsuarios {
  constructor(private db: BaseDatos) {} // ✅ Inyección de dependencia

  registrarUsuario(nombre: string, email: string): void {
    this.db.conectar();
    this.db.guardar("usuarios", { nombre, email });
  }
}

// Podemos usar cualquier implementación de BaseDatos
const mysql = new MySQL();
const mongo = new MongoDB();

const servicio = new ServicioUsuarios(mysql);
servicio.registrarUsuario("Ana", "ana@mail.com");
```

La abstracción combinada con la inyección de dependencias hace que tu
código sea desacoplado y fácil de probar.

6. ABSTRACTION EN LA VIDA REAL: REPOSITORIOS
===============================================================================

Un patrón muy común que usa abstracción es el patrón Repositorio.
Separa la lógica de acceso a datos de la lógica de negocio.

```typescript
// Abstracción del repositorio
interface Repositorio<T extends { id: string }> {
  obtenerPorId(id: string): Promise<T | null>;
  obtenerTodos(): Promise<T[]>;
  guardar(elemento: T): Promise<T>;
  eliminar(id: string): Promise<boolean>;
}

// Entidad
interface Usuario {
  id: string;
  nombre: string;
  email: string;
  fechaRegistro: Date;
}

// Implementación concreta: en memoria
class RepositorioUsuariosMemoria implements Repositorio<Usuario> {
  private usuarios: Map<string, Usuario> = new Map();

  async obtenerPorId(id: string): Promise<Usuario | null> {
    return this.usuarios.get(id) ?? null;
  }

  async obtenerTodos(): Promise<Usuario[]> {
    return Array.from(this.usuarios.values());
  }

  async guardar(usuario: Usuario): Promise<Usuario> {
    this.usuarios.set(usuario.id, usuario);
    return usuario;
  }

  async eliminar(id: string): Promise<boolean> {
    return this.usuarios.delete(id);
  }
}

// Servicio que depende de la abstracción
class ServicioUsuarios {
  constructor(private repo: Repositorio<Usuario>) {}

  async obtenerPerfil(id: string): Promise<Usuario | null> {
    return this.repo.obtenerPorId(id);
  }

  async crearUsuario(nombre: string, email: string): Promise<Usuario> {
    const usuario: Usuario = {
      id: crypto.randomUUID(),
      nombre,
      email,
      fechaRegistro: new Date()
    };
    return this.repo.guardar(usuario);
  }
}

// Podemos cambiar la implementación sin tocar el servicio
const repoMemoria = new RepositorioUsuariosMemoria();
const servicio = new ServicioUsuarios(repoMemoria);
```

La abstracción del repositorio permite cambiar la fuente de datos
(memoria, archivos, base de datos, API) sin modificar el servicio.

7. NIVELES DE ABSTRACCIÓN
===============================================================================

Un buen diseño en capas tiene múltiples niveles de abstracción, cada
uno ocultando los detalles del nivel inferior.

```typescript
// Nivel 1: Abstracción de más alto nivel
interface ServicioPagos {
  cobrar(usuarioId: string, monto: number): Promise<boolean>;
  reembolsar(transaccionId: string): Promise<boolean>;
}

// Nivel 2: Abstracción intermedia
interface ProcesadorPagos {
  autorizar(monto: number, moneda: string): Promise<string>;
  capturar(autorizacionId: string): Promise<boolean>;
  reversar(transaccionId: string): Promise<boolean>;
}

interface ValidadorPagos {
  validarLimites(usuarioId: string, monto: number): boolean;
  validarMetodoPago(usuarioId: string): boolean;
}

interface NotificadorPagos {
  notificarExito(usuarioId: string, monto: number): void;
  notificarFallo(usuarioId: string, error: string): void;
}

// Nivel 3: Implementaciones concretas
class ProcesadorStripe implements ProcesadorPagos {
  async autorizar(monto: number, moneda: string): Promise<string> {
    console.log(`Stripe: autorizando ${monto} ${moneda}`);
    return "auth_" + Date.now();
  }

  async capturar(autorizacionId: string): Promise<boolean> {
    console.log(`Stripe: capturando ${autorizacionId}`);
    return true;
  }

  async reversar(transaccionId: string): Promise<boolean> {
    console.log(`Stripe: revirtiendo ${transaccionId}`);
    return true;
  }
}

class ValidadorSimple implements ValidadorPagos {
  validarLimites(usuarioId: string, monto: number): boolean {
    return monto > 0 && monto <= 10000;
  }

  validarMetodoPago(usuarioId: string): boolean {
    return true; // Simplificado
  }
}

class NotificadorEmail implements NotificadorPagos {
  notificarExito(usuarioId: string, monto: number): void {
    console.log(`Email a ${usuarioId}: pago de $${monto} exitoso`);
  }

  notificarFallo(usuarioId: string, error: string): void {
    console.log(`Email a ${usuarioId}: pago falló - ${error}`);
  }
}

// Implementación del servicio de alto nivel
class ServicioPagosImpl implements ServicioPagos {
  constructor(
    private procesador: ProcesadorPagos,
    private validador: ValidadorPagos,
    private notificador: NotificadorPagos
  ) {}

  async cobrar(usuarioId: string, monto: number): Promise<boolean> {
    try {
      if (!this.validador.validarLimites(usuarioId, monto)) {
        throw new Error("Límite excedido");
      }

      const authId = await this.procesador.autorizar(monto, "USD");
      const exitoso = await this.procesador.capturar(authId);

      if (exitoso) {
        this.notificador.notificarExito(usuarioId, monto);
      }
      return exitoso;
    } catch (error) {
      this.notificador.notificarFallo(usuarioId, String(error));
      return false;
    }
  }

  async reembolsar(transaccionId: string): Promise<boolean> {
    return this.procesador.reversar(transaccionId);
  }
}
```

8. ABSTRACCIÓN CON TIPOS AVANZADOS
===============================================================================

TypeScript permite crear abstracciones combinando interfaces con genéricos:

```typescript
interface Mapper<TSource, TTarget> {
  mapear(origen: TSource): TTarget;
}

interface UsuarioDTO {
  id: string;
  nombreCompleto: string;
  email: string;
}

interface Usuario {
  id: string;
  nombre: string;
  apellido: string;
  email: string;
  fechaRegistro: Date;
}

class UsuarioMapper implements Mapper<UsuarioDTO, Usuario> {
  mapear(origen: UsuarioDTO): Usuario {
    const [nombre, apellido = ""] = origen.nombreCompleto.split(" ");
    return { id: origen.id, nombre, apellido, email: origen.email, fechaRegistro: new Date() };
  }
}
```

9. ABSTRACCIÓN Y TESTEABILIDAD
===============================================================================

La abstracción facilita las pruebas unitarias. Puedes crear mocks de
las abstracciones para aislar el código bajo prueba.

```typescript
interface ServicioClima {
  obtenerTemperatura(ciudad: string): Promise<number>;
}

class ServicioClimaMock implements ServicioClima {
  async obtenerTemperatura(ciudad: string): Promise<number> {
    const temps: Record<string, number> = { Madrid: 28, Tokyo: 18, NY: 15 };
    return temps[ciudad] ?? 20;
  }
}

async function recomendarActividad(clima: ServicioClima, ciudad: string): Promise<string> {
  const temp = await clima.obtenerTemperatura(ciudad);
  if (temp > 30) return "Ir a la playa";
  if (temp > 20) return "Picnic";
  if (temp > 10) return "Museo";
  return "Leer en casa";
}
```

10. BUENAS PRÁCTICAS DE ABSTRACCIÓN
===============================================================================

1. **Empieza con interfaces, no con clases concretas**: Diseña primero
   el contrato, luego la implementación. Esto te obliga a pensar en
   términos de comportamiento.

2. **No abstraigas prematuramente**: La abstracción tiene un costo.
   Espera a tener al menos dos implementaciones concretas antes de
   crear una abstracción.

3. **Manten las abstracciones pequeñas (ISP)**: El Principio de
   Segregación de Interfaces dice que es mejor tener interfaces
   pequeñas y específicas que una grande y genérica.

4. **Nombra las abstracciones con claridad**: Usa nombres que describan
   el comportamiento, no la implementación.

5. **Documenta el contrato**: Explica qué se espera de cada método de
   la abstracción, incluyendo precondiciones y postcondiciones.

```typescript
// ❌ Mala abstracción: demasiado grande
interface Procesador {
  procesarPago(monto: number): boolean;
  enviarEmail(para: string, asunto: string, cuerpo: string): void;
  generarReporte(fechaInicio: Date, fechaFin: Date): string;
  autenticarUsuario(usuario: string, password: string): boolean;
}

// ✅ Buena abstracción: interfaces pequeñas y enfocadas
interface ProcesadorPagos {
  procesar(monto: number, moneda: string): ResultadoPago;
}

interface ServicioEmail {
  enviar(para: string, asunto: string, cuerpo: string): Promise<void>;
}

interface GeneradorReportes {
  generar(periodo: RangoFecha): Reporte;
}

interface Autenticador {
  autenticar(credenciales: Credenciales): Usuario | null;
}
```

11. ERRORES COMUNES CON ABSTRACCIÓN
===============================================================================

**Error 1: Abstracción demasiado genérica**
```typescript
// ❌ Demasiado genérica
interface Procesador {
  procesar(entrada: any): any;
}

// ✅ Específica y útil
interface ProcesadorArchivos {
  procesar(ruta: string): Promise<ResultadoProcesamiento>;
}
```

**Error 2: Abstracción filter (debilita la interfaz)**
```typescript
// ❌ Filtrada: añade excepciones
interface BaseDatos {
  guardar(datos: any): void;
  obtener(id: string): any;
}

// Pero algunas implementaciones no soportan obtener()
// Esto viola el principio de sustitución
```

**Error 3: Abstracción que cambia constantemente**
```typescript
// ❌ Cambia cada vez que se añade una implementación
interface Almacenamiento {
  guardar(clave: string, valor: string): void;
  leer(clave: string): string | null;
  listarPrefijo(prefijo: string): string[];
  eliminarPorPrefijo(prefijo: string): void;
  copiar(origen: string, destino: string): void;
  mover(origen: string, destino: string): void;
  obtenerMetadatos(clave: string): Metadatos;
}
```

12. CASO PRÁCTICO: SISTEMA DE PLUGINS
===============================================================================

Un caso clásico de abstracción bien diseñada es un sistema de plugins:

```typescript
interface Plugin {
  readonly nombre: string;
  readonly version: string;
  inicializar(contexto: ContextoPlugin): Promise<void>;
  ejecutar(entrada: any): Promise<any>;
  limpiar(): Promise<void>;
}

interface ContextoPlugin {
  logger: Logger;
  config: Configuracion;
  baseDatos: BaseDatos;
}

class Logger {
  info(msg: string): void { console.log(`[INFO] ${msg}`); }
  error(msg: string): void { console.error(`[ERROR] ${msg}`); }
}

class Configuracion {
  private valores: Map<string, any> = new Map();

  obtener<T>(clave: string, valorDefecto?: T): T | undefined {
    return this.valores.get(clave) ?? valorDefecto;
  }

  establecer(clave: string, valor: any): void {
    this.valores.set(clave, valor);
  }
}

// Plugin concreto: compresión
class PluginCompresion implements Plugin {
  readonly nombre = "Compresión ZIP";
  readonly version = "1.0.0";

  async inicializar(contexto: ContextoPlugin): Promise<void> {
    contexto.logger.info("Plugin de compresión inicializado");
  }

  async ejecutar(entrada: any): Promise<any> {
    if (typeof entrada === "string") {
      return `[COMPRIMIDO:${entrada}]`;
    }
    return entrada;
  }

  async limpiar(): Promise<void> {
    console.log("Plugin de compresión: limpiando recursos");
  }
}

// Plugin concreto: encriptación
class PluginEncriptacion implements Plugin {
  readonly nombre = "Encriptación AES";
  readonly version = "2.1.0";

  async inicializar(contexto: ContextoPlugin): Promise<void> {
    contexto.logger.info("Plugin de encriptación inicializado");
  }

  async ejecutar(entrada: any): Promise<any> {
    if (typeof entrada === "string") {
      return `[ENCRIPTADO:${entrada}]`;
    }
    return entrada;
  }

  async limpiar(): Promise<void> {
    console.log("Plugin de encriptación: limpiando recursos");
  }
}

// Motor de plugins
class MotorPlugins {
  private plugins: Map<string, Plugin> = new Map();
  private contexto: ContextoPlugin;

  constructor() {
    this.contexto = {
      logger: new Logger(),
      config: new Configuracion(),
      baseDatos: {} as BaseDatos
    };
  }

  registrar(plugin: Plugin): void {
    if (this.plugins.has(plugin.nombre)) {
      throw new Error(`Plugin "${plugin.nombre}" ya registrado`);
    }
    this.plugins.set(plugin.nombre, plugin);
  }

  async iniciarTodos(): Promise<void> {
    for (const plugin of this.plugins.values()) {
      await plugin.inicializar(this.contexto);
    }
  }

  async ejecutarPlugins(entrada: any): Promise<any> {
    let resultado = entrada;
    for (const plugin of this.plugins.values()) {
      resultado = await plugin.ejecutar(resultado);
    }
    return resultado;
  }

  async detenerTodos(): Promise<void> {
    for (const plugin of this.plugins.values()) {
      await plugin.limpiar();
    }
  }
}

// Uso
async function main() {
  const motor = new MotorPlugins();
  motor.registrar(new PluginCompresion());
  motor.registrar(new PluginEncriptacion());

  await motor.iniciarTodos();
  const resultado = await motor.ejecutarPlugins("Hola Mundo");
  console.log(resultado);
  await motor.detenerTodos();
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

- La abstracción oculta los detalles de implementación y expone solo la
  funcionalidad esencial.
- Las clases abstractas pueden tener implementación concreta y métodos
  abstractos. No se pueden instanciar.
- Las interfaces definen contratos puros sin implementación.
- Usa interfaces para contratos puros; usa clases abstractas cuando
  necesites compartir implementación.
- El Principio de Inversión de Dependencias (DIP) establece que tanto
  módulos de alto como bajo nivel deben depender de abstracciones.
- La abstracción facilita las pruebas unitarias (mocks).
- Un buen diseño tiene múltiples niveles de abstracción.
- Las abstracciones deben ser pequeñas, enfocadas y estables.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuándo debo usar clase abstracta vs interfaz?**
Usa interfaz para contratos puros que múltiples clases no relacionadas
puedan implementar. Usa clase abstracta cuando haya implementación
común que compartir entre clases relacionadas.

❓ **¿Puedo tener una interfaz que extienda una clase?**
No. Las interfaces extienden otras interfaces. Las clases implementan
interfaces. Una clase abstracta puede implementar una interfaz.

❓ **¿La abstracción siempre es necesaria?**
No. Para scripts pequeños o prototipos, la abstracción puede ser
excesiva. Añádela cuando el código crezca y necesites flexibilidad,
testeabilidad o desacoplamiento.

❓ **¿Cómo sé si mi abstracción es buena?**
Una buena abstracción es estable (no cambia frecuentemente), enfocada
(un solo propósito), y tiene al menos dos implementaciones concretas
(o una real y una mock para pruebas).

❓ **¿La abstracción tiene costo de rendimiento?**
Mínimo. Las llamadas a métodos virtuales tienen un costo ínfimo.
El mayor costo es la complejidad cognitiva: no abstraigas sin una
necesidad real.
