===============================================================================
1. INTRODUCCIÓN A LOS MODIFICADORES DE ACCESO
===============================================================================

Los modificadores de acceso controlan la visibilidad de propiedades y
métodos dentro de una clase. TypeScript ofrece cuatro modificadores:

- **`public`** (por defecto): Accesible desde cualquier lugar.
- **`private`**: Solo accesible dentro de la clase donde se define.
- **`protected`**: Accesible dentro de la clase y sus subclases.
- **`readonly`**: Solo lectura (puede ser combinado con los anteriores).

```typescript
class Demo {
  public    propiedadPublica: string = "visible para todos";
  private   propiedadPrivada: string = "solo en esta clase";
  protected propiedadProtegida: string = "esta clase y subclases";
  readonly  propiedadSoloLectura: string = "no puede cambiar";
  private readonly privadaLectura: string = "privada y de solo lectura";
}
```

Si no especificas un modificador, TypeScript asume `public`:

```typescript
class Ejemplo {
  nombre: string = "público";           // implícitamente public
  public edad: number = 30;              // explícitamente public
  private _saldo: number = 0;            // privado
  protected readonly ID: string = "x";   // protegido + readonly
}
```

===============================================================================
2. PUBLIC: ACCESO TOTAL
===============================================================================

`public` es el modificador por defecto. Las propiedades y métodos
públicos son accesibles desde:
- Dentro de la clase.
- Subclases.
- Instancias.
- Cualquier código externo.

```typescript
class Vehiculo {
  public marca: string;
  public modelo: string;

  constructor(marca: string, modelo: string) {
    this.marca = marca;
    this.modelo = modelo;
  }

  public obtenerInfo(): string {
    return `${this.marca} ${this.modelo}`;
  }
}

class Coche extends Vehiculo {
  public numPuertas: number;

  constructor(marca: string, modelo: string, numPuertas: number) {
    super(marca, modelo);
    this.numPuertas = numPuertas;
  }

  public obtenerInfoCompleta(): string {
    return `${super.obtenerInfo()} - ${this.numPuertas} puertas`;
  }
}

const coche = new Coche("Toyota", "Corolla", 4);
console.log(coche.marca);       // ✅ "Toyota"
console.log(coche.obtenerInfo()); // ✅ "Toyota Corolla"
```

Al ser el modificador por defecto, raramente necesitas escribir `public`
explícitamente, aunque algunos equipos prefieren hacerlo por claridad.

===============================================================================
3. PRIVATE: SOLO INTERNO
===============================================================================

`private` restringe el acceso solo a la clase donde se define. Ni las
subclases ni el código externo pueden acceder.

```typescript
class CuentaBancaria {
  private _saldo: number;
  private _numeroCuenta: string;

  constructor(numeroCuenta: string, saldoInicial: number = 0) {
    this._numeroCuenta = numeroCuenta;
    this._saldo = saldoInicial;
  }

  public depositar(monto: number): void {
    if (monto > 0) {
      this._saldo += monto;
      this._registrarTransaccion("DEPÓSITO", monto);
    }
  }

  public obtenerSaldo(): number {
    return this._saldo;
  }

  private _registrarTransaccion(tipo: string, monto: number): void {
    console.log(`[${new Date().toISOString()}] ${tipo}: $${monto}`);
  }
}

class CuentaPremium extends CuentaBancaria {
  constructor(numeroCuenta: string, saldoInicial: number) {
    super(numeroCuenta, saldoInicial);
  }

  public aplicarBono(): void {
    // this._saldo += 100;  // ❌ Error: _saldo es private
    // this._registrarTransaccion("BONO", 100); // ❌ Error: private
  }
}

const cuenta = new CuentaBancaria("123-456", 5000);
console.log(cuenta.obtenerSaldo());  // ✅ 5000
// console.log(cuenta._saldo);       // ❌ Error: private
// cuenta._registrarTransaccion("TEST", 0); // ❌ Error: private
```

**Importante**: La privacidad de TypeScript es solo en tiempo de
compilación. En JavaScript compilado, las propiedades `private` siguen
siendo públicas.

```typescript
// TypeScript
class Secreto {
  private contrasena: string = "123";
}

// JavaScript compilado
class Secreto {
  constructor() {
    this.contrasena = "123"; // ¡Sigue siendo público en JS!
  }
}
```

Si necesitas privacidad real en tiempo de ejecución, usa el modificador
`#` de JavaScript (ECMAScript privado).

===============================================================================
4. PROTECTED: FAMILIA Y SUBCLASES
===============================================================================

`protected` permite acceso desde la clase donde se define y desde
cualquier subclase, pero no desde fuera.

```typescript
class Empleado {
  constructor(
    public nombre: string,
    protected salario: number,
    private _seguroSocial: string
  ) {}

  protected obtenerSalario(): number {
    return this.salario;
  }
}

class Gerente extends Empleado {
  private bono: number;

  constructor(nombre: string, salario: number, seguroSocial: string, bono: number) {
    super(nombre, salario, seguroSocial);
    this.bono = bono;
  }

  public calcularCompensacion(): number {
    return this.salario + this.bono; // ✅ protected: accesible
  }

  public mostrarDetalles(): void {
    console.log(`Gerente: ${this.nombre}`);
    console.log(`Salario base: ${this.obtenerSalario()}`); // ✅ protected
    // console.log(this._seguroSocial); // ❌ Error: private
  }
}

class Director extends Gerente {
  public compensacionTotal(): number {
    return this.calcularCompensacion() + 10000; // ✅ todo accesible
  }
}

const gerente = new Gerente("Ana", 50000, "SS-123", 10000);
console.log(gerente.nombre);    // ✅ "Ana"
// console.log(gerente.salario); // ❌ Error: protected
// console.log(gerente._seguroSocial); // ❌ Error: private
gerente.calcularCompensacion(); // ✅ público
```

Jerarquía de acceso de `protected`:
- `Empleado`: define `salario` como protected.
- `Gerente` (extends Empleado): puede acceder a `salario`.
- `Director` (extends Gerente): puede acceder a `salario`.
- Código externo: NO puede acceder a `salario`.

===============================================================================
5. READONLY: SOLO LECTURA
===============================================================================

`readonly` marca una propiedad que solo puede asignarse en su declaración
o en el constructor. Cualquier intento de modificación posterior es un
error en tiempo de compilación.

```typescript
class TarjetaIdentidad {
  readonly id: string;
  readonly fechaEmision: Date;
  readonly tipoSangre: string;

  constructor(id: string, tipoSangre: string) {
    this.id = id;
    this.fechaEmision = new Date();
    this.tipoSangre = tipoSangre;
  }

  renovar(): void {
    // this.fechaEmision = new Date(); // ❌ Error: readonly
  }
}

const tarjeta = new TarjetaIdentidad("ID-001", "O+");
console.log(tarjeta.id);           // ✅ "ID-001"
// tarjeta.id = "ID-002";          // ❌ Error: readonly
```

`readonly` se puede combinar con otros modificadores:

```typescript
class Documento {
  constructor(
    public readonly id: string,            // público + readonly
    private readonly _contenido: string,   // privado + readonly
    protected readonly _version: number = 1 // protegido + readonly
  ) {}

  public obtenerContenido(): string {
    return this._contenido; // ✅ Acceso interno permitido
  }
}

class DocumentoVersionado extends Documento {
  public mostrarVersion(): void {
    console.log(`Versión: ${this._version}`); // ✅ protected + readonly
  }
}

const doc = new Documento("DOC-1", "Contenido...");
console.log(doc.id);           // ✅
// console.log(doc._contenido); // ❌ Error: private
// console.log(doc._version);   // ❌ Error: protected
```

===============================================================================
6. COMBINACIONES COMUNES
===============================================================================

Algunas combinaciones de modificadores son especialmente útiles:

**public readonly**: Propiedad pública que no puede modificarse.

```typescript
class Producto {
  constructor(
    public readonly codigo: string,
    public readonly nombre: string
  ) {}
}

// Ideal para DTOs inmutables
```

**private readonly**: Constante interna de la clase.

```typescript
class ServicioAPI {
  private readonly URL_BASE = "https://api.example.com/v1";
  private readonly TIMEOUT = 5000;
  private readonly headers: Record<string, string>;

  constructor(private readonly apiKey: string) {
    this.headers = {
      "Authorization": `Bearer ${this.apiKey}`,
      "Content-Type": "application/json"
    };
  }
}
```

**protected readonly**: Constante heredable.

```typescript
abstract class ConfiguracionBase {
  protected readonly VERSION = "1.0.0";
  protected readonly MAX_INTENTOS = 3;

  abstract configurar(): void;
}

class ConfiguracionApp extends ConfiguracionBase {
  configurar(): void {
    console.log(`Versión: ${this.VERSION}`); // ✅
    console.log(`Intentos: ${this.MAX_INTENTOS}`); // ✅
  }
}
```

===============================================================================
7. PROPIEDADES PRIVADAS REALES CON #
===============================================================================

TypeScript también soporta las propiedades privadas reales de JavaScript
ECMAScript usando el prefijo `#`. A diferencia de `private` de TS, estas
son verdaderamente privadas también en tiempo de ejecución.

```typescript
class CuentaRealmentePrivada {
  #saldo: number;     // Privada de verdad (runtime)
  #titular: string;   // Privada de verdad (runtime)
  public numeroCuenta: string;

  constructor(numeroCuenta: string, titular: string, saldo: number) {
    this.numeroCuenta = numeroCuenta;
    this.#titular = titular;
    this.#saldo = saldo;
  }

  #registrarTransaccion(tipo: string, monto: number): void {
    console.log(`[${tipo}] $${monto}`);
  }

  depositar(monto: number): void {
    if (monto > 0) {
      this.#saldo += monto;
      this.#registrarTransaccion("DEPÓSITO", monto);
    }
  }

  obtenerSaldo(): number {
    return this.#saldo;
  }

  obtenerTitular(): string {
    return this.#titular;
  }
}

const cuenta = new CuentaRealmentePrivada("123", "Ana", 5000);
console.log(cuenta.obtenerSaldo()); // ✅ 5000
console.log(cuenta.numeroCuenta);   // ✅ "123"
// console.log(cuenta.#saldo);      // ❌ Error de sintaxis (incluso en JS)
```

Diferencias `private` vs `#`:

| Característica              | private (TS)        | # (JS)               |
|-----------------------------|---------------------|----------------------|
| Privacidad en compilación   | ✅                  | ✅                   |
| Privacidad en runtime       | ❌ (sigue siendo pública) | ✅ (realmente privada) |
| Acceso en subclases         | ❌                  | ❌                   |
| Acceso desde fuera          | ❌ (error TS)       | ❌ (error sintaxis)  |
| Compatibilidad con versiones| ES3+                | ES2021+              |
| Se puede usar en interfaces | ❌                  | ❌                   |

Recomendación: usa `private` de TypeScript para la mayoría de los casos.
Usa `#` solo cuando necesites privacidad real en tiempo de ejecución
(bibliotecas publicadas, código que será usado desde JavaScript).

8. MODIFICADORES EN PARAMETER PROPERTIES
===============================================================================

Los modificadores de acceso son una parte esencial de las parameter
properties. Cada parámetro con modificador crea automáticamente la
propiedad con ese nivel de acceso.

```typescript
class Usuario {
  constructor(
    public readonly id: number,           // Público + solo lectura
    public nombre: string,                // Público
    private _email: string,               // Privado
    protected readonly _fechaRegistro: Date, // Protegido + solo lectura
    private _puntaje: number = 0           // Privado con valor por defecto
  ) {}

  get email(): string { return this._email; }

  set email(valor: string) {
    if (!valor.includes("@")) throw new Error("Email inválido");
    this._email = valor;
  }

  protected incrementarPuntaje(puntos: number): void {
    this._puntaje += puntos;
  }
}

class UsuarioVIP extends Usuario {
  constructor(
    id: number,
    nombre: string,
    email: string,
    public nivel: number
  ) {
    super(id, nombre, email, new Date());
  }

  darBono(): void {
    this.incrementarPuntaje(100); // ✅ protected
    // this._puntaje += 100;      // ❌ private
    console.log(`Nivel VIP: ${this._fechaRegistro}`); // ✅ protected readonly
  }
}
```

Regla: Usa el modificador más restrictivo que sea necesario. Empieza
con `private` y solo sube a `protected` o `public` cuando sea necesario.

9. COMPARATIVA DE ACCESO
===============================================================================

```typescript
class Base {
  public a = "público";
  protected b = "protegido";
  private c = "privado";
  #d = "privado JS";

  test(): void {
    console.log(this.a); // ✅
    console.log(this.b); // ✅
    console.log(this.c); // ✅
    console.log(this.#d); // ✅
  }
}

class Subclase extends Base {
  test(): void {
    console.log(this.a); // ✅ público
    console.log(this.b); // ✅ protegido
    // console.log(this.c); // ❌ privado TS
    // console.log(this.#d); // ❌ privado JS
  }
}

const instancia = new Subclase();
console.log(instancia.a); // ✅ público
// console.log(instancia.b); // ❌ protegido
// console.log(instancia.c); // ❌ privado TS
// console.log(instancia.#d); // ❌ privado JS
```

| Acceso desde               | public | protected | private (TS) | # (JS) |
|----------------------------|--------|-----------|--------------|--------|
| Misma clase                | ✅     | ✅        | ✅           | ✅     |
| Subclase                   | ✅     | ✅        | ❌           | ❌     |
| Instancia                  | ✅     | ❌        | ❌           | ❌     |
| Código externo             | ✅     | ❌        | ❌ (error)   | ❌ (sintaxis) |

10. ERRORES COMUNES CON MODIFICADORES DE ACCESO
===============================================================================

**Error 1: Confundir private de TS con privacidad real**
```typescript
class Usuario {
  private password: string = "secreto";
}

// En JavaScript compilado:
// const u = new Usuario();
// console.log(u.password); // "secreto" - ¡sigue siendo accesible!
```

**Error 2: Usar private cuando debería ser protected**
```typescript
class Base {
  private valor = 10;
}

class Derivada extends Base {
  mostrar(): void {
    // console.log(this.valor); // ❌ private, no accesible
  }
}
// Si las subclases necesitan acceder, usa protected
```

**Error 3: Olvidar que readonly no es inmutable**
```typescript
class Lista {
  readonly items: number[] = [1, 2, 3];
}

const lista = new Lista();
// lista.items = [4, 5, 6]; // ❌ Error: readonly
lista.items.push(4); // ✅ ¡Esto funciona! La referencia es readonly, no el contenido
```

Para inmutabilidad real, usa `ReadonlyArray` o `Object.freeze`:

```typescript
class ListaSegura {
  readonly items: ReadonlyArray<number> = [1, 2, 3];
}

const lista = new ListaSegura();
// lista.items.push(4); // ❌ Error: no existe push en ReadonlyArray
```

11. BUENAS PRÁCTICAS
===============================================================================

1. **Principio de mínimo privilegio**: Usa el modificador más restrictivo
   posible. Empieza con `private` y solo amplía el acceso si es necesario.

2. **Sé explícito con public**: Aunque es el predeterminado, algunos
   equipos prefieren escribirlo explícitamente para claridad.

3. **Usa el prefijo `_` para propiedades privadas**: Es una convención
   ampliamente aceptada que mejora la legibilidad.

4. **Diferencia entre private TS y # JS**:
   - `private`: para la mayoría de casos.
   - `#`: para APIs públicas/bibliotecas donde la privacidad en runtime
     importa.

5. **Documenta el porqué de los modificadores**: Si una propiedad es
   `protected` en lugar de `private`, explica por qué las subclases
   necesitan acceder a ella.

6. **Considera getters/setters para acceso controlado** en lugar de
   hacer propiedades públicas directamente.

```typescript
class EjemploFinal {
  // ✅ Privado hasta que se necesite más acceso
  private _datosInternos: Map<string, any> = new Map();

  // ✅ Público porque es parte de la API
  constructor(
    public readonly id: string,
    public nombre: string
  ) {}

  // ✅ Protegido para que subclases puedan extender
  protected realizarValidacion(): boolean {
    return this._datosInternos.size > 0;
  }

  // ✅ Público porque expone funcionalidad controlada
  public procesar(datos: any): void {
    this._datosInternos.set("ultimo", datos);
  }
}
```

12. CASO PRÁCTICO: SISTEMA DE SEGURIDAD
===============================================================================

```typescript
class SistemaSeguro {
  private static instancia: SistemaSeguro;
  private readonly _logs: string[] = [];
  private _sesionesActivas: Map<string, Sesion> = new Map();
  protected static readonly VERSION_API = "2.0.0";

  private constructor(
    private readonly _claveMaestra: string,
    private _nivelSeguridad: number = 3
  ) {}

  static obtenerInstancia(clave?: string): SistemaSeguro {
    if (!SistemaSeguro.instancia) {
      if (!clave) throw new Error("Se requiere clave maestra");
      SistemaSeguro.instancia = new SistemaSeguro(clave);
    }
    return SistemaSeguro.instancia;
  }

  get nivelSeguridad(): number {
    return this._nivelSeguridad;
  }

  set nivelSeguridad(valor: number) {
    if (valor < 1 || valor > 5) {
      throw new Error("Nivel de seguridad debe ser 1-5");
    }
    this._nivelSeguridad = valor;
  }

  get logs(): ReadonlyArray<string> {
    return [...this._logs];
  }

  get sesionesActivas(): number {
    return this._sesionesActivas.size;
  }

  iniciarSesion(usuarioId: string, token: string): void {
    this.#validarToken(token);
    this._sesionesActivas.set(usuarioId, {
      id: usuarioId,
      inicio: new Date(),
      token: token
    });
    this.#agregarLog(`Sesión iniciada: ${usuarioId}`);
  }

  cerrarSesion(usuarioId: string): void {
    this._sesionesActivas.delete(usuarioId);
    this.#agregarLog(`Sesión cerrada: ${usuarioId}`);
  }

  #validarToken(token: string): void {
    if (token.length < 10) {
      throw new Error("Token inválido");
    }
  }

  #agregarLog(mensaje: string): void {
    this._logs.push(`[${new Date().toISOString()}] ${mensaje}`);
  }
}

interface Sesion {
  id: string;
  inicio: Date;
  token: string;
}

const sistema = SistemaSeguro.obtenerInstancia("clave-super-secreta");
sistema.iniciarSesion("user-1", "token-seguro-12345");
sistema.iniciarSesion("user-2", "otro-token-67890");
console.log(sistema.sesionesActivas); // 2
console.log(sistema.logs); // Copia de los logs
```

RESUMEN DEL CAPÍTULO
===============================================================================

- `public` (por defecto): acceso desde cualquier lugar.
- `private`: solo dentro de la clase donde se define.
- `protected`: dentro de la clase y sus subclases.
- `readonly`: solo asignación en declaración o constructor.
- Los modificadores se pueden combinar: `public readonly`, `private readonly`.
- La privacidad de TypeScript es solo en compilación; para runtime usa `#`.
- Las parameter properties usan modificadores para crear propiedades
  automáticamente.
- Principio de mínimo privilegio: usa el modificador más restrictivo.
- `readonly` no hace inmutable el contenido de objetos o arrays.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre private y #?**
`private` de TypeScript solo protege en tiempo de compilación. En el
JavaScript generado, la propiedad sigue siendo pública. `#` es una
característica de JavaScript que proporciona verdadera privacidad en
tiempo de ejecución.

❓ **¿Debo usar # siempre?**
No. `#` requiere ES2021 o superior. Para la mayoría de proyectos, el
`private` de TypeScript es suficiente. Usa `#` solo si publicas una
biblioteca o necesitas protección en runtime.

❓ **¿Puedo acceder a propiedades privadas desde una subclase?**
No. Las propiedades `private` solo son accesibles en la clase que las
define. Si una subclase necesita acceder, usa `protected`.

❓ **¿Qué pasa si no pongo ningún modificador?**
TypeScript asume `public` por defecto. Todas las propiedades y métodos
serán públicos a menos que especifiques lo contrario.

❓ **¿Puedo cambiar el modificador en una subclase?**
No puedes debilitar el acceso (ej: cambiar protected a public), pero
en TypeScript esto no se controla estrictamente. Es una mala práctica.
