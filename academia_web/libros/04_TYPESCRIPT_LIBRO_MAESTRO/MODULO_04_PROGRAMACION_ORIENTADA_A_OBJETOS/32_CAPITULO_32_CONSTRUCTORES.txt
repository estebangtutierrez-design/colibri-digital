1. EL MÉTODO CONSTRUCTOR
===============================================================================

El constructor es un método especial que se invoca automáticamente cuando
se usa la palabra clave `new` para crear una instancia de una clase. Su
propósito principal es inicializar las propiedades del objeto y realizar
cualquier configuración necesaria.

En TypeScript, el constructor se define con la palabra clave `constructor`
seguida de paréntesis y un bloque de código. No puede tener un tipo de
retorno explícito (siempre retorna la instancia de la clase).

```typescript
class Usuario {
  nombre: string;
  email: string;

  constructor(nombre: string, email: string) {
    this.nombre = nombre;
    this.email = email;
    console.log(`Usuario ${nombre} creado exitosamente`);
  }
}

const usuario = new Usuario("María", "maria@example.com");
```

Características importantes del constructor:
- Se ejecuta una sola vez por instancia (al crearla).
- No puede ser llamado directamente (solo con `new`).
- Puede recibir parámetros con tipos, valores por defecto y opcionales.
- No puede tener un tipo de retorno explícito.
- En TypeScript, todas las propiedades deben inicializarse en el
  constructor (o tener valores por defecto).

===============================================================================
2. PARÁMETROS OPCIONALES Y VALORES POR DEFECTO
===============================================================================

Al igual que en las funciones regulares, los parámetros del constructor
pueden ser opcionales o tener valores por defecto. Esto proporciona
flexibilidad al crear instancias.

```typescript
class Producto {
  nombre: string;
  precio: number;
  descuento: number;
  categoria?: string;  // Opcional

  constructor(
    nombre: string,
    precio: number,
    descuento: number = 0,
    categoria?: string
  ) {
    this.nombre = nombre;
    this.precio = precio;
    this.descuento = descuento;
    this.categoria = categoria;
  }

  obtenerPrecioFinal(): number {
    return this.precio - (this.precio * this.descuento) / 100;
  }
}

const producto1 = new Producto("Laptop", 15000);
const producto2 = new Producto("Monitor", 5000, 10);
const producto3 = new Producto("Mouse", 800, 5, "Periféricos");
```

Reglas importantes:
- Los parámetros opcionales deben ir después de los obligatorios.
- Los parámetros con valor por defecto pueden ir antes que los opcionales.
- Un parámetro con valor por defecto es implícitamente opcional.

===============================================================================
3. PROPIEDADES DE PARÁMETROS DEL CONSTRUCTOR (PARAMETER PROPERTIES)
===============================================================================

Esta es una de las características más poderosas y distintivas de
TypeScript. Las "parameter properties" permiten declarar e inicializar
una propiedad en un solo paso, directamente en los parámetros del
constructor.

La sintaxis consiste en anteponer un modificador de acceso (`public`,
`private`, `protected` o `readonly`) al parámetro del constructor.

```typescript
// ❌ Sin parameter properties
class UsuarioVerboso {
  nombre: string;
  email: string;
  edad: number;

  constructor(nombre: string, email: string, edad: number) {
    this.nombre = nombre;
    this.email = email;
    this.edad = edad;
  }
}

// ✅ Con parameter properties
class UsuarioConciso {
  constructor(
    public nombre: string,
    public email: string,
    public edad: number
  ) {}
}
```

Ambas clases son exactamente equivalentes, pero la segunda es mucho más
concisa. TypeScript genera automáticamente:
1. La declaración de la propiedad con el tipo correspondiente.
2. La asignación en el cuerpo del constructor.

¿Qué modificadores se pueden usar?
- `public`: Propiedad pública (accesible desde cualquier lugar).
- `private`: Propiedad privada (solo accesible dentro de la clase).
- `protected`: Propiedad protegida (accesible en la clase y subclases).
- `readonly`: Propiedad de solo lectura.
- Combinaciones: `public readonly`, `private readonly`, etc.

```typescript
class Empleado {
  constructor(
    public readonly id: number,
    public nombre: string,
    private salario: number,
    protected departamento: string
  ) {}
}

const emp = new Empleado(1, "Carlos", 50000, "IT");
console.log(emp.id);          // ✅  1
console.log(emp.nombre);      // ✅  "Carlos"
console.log(emp.salario);     // ❌  Error: salario es private
console.log(emp.departamento);// ❌  Error: departamento es protected
```

===============================================================================
4. PARAMETER PROPERTIES CON VALORES POR DEFECTO
===============================================================================

Las parameter properties también pueden combinarse con valores por defecto
y parámetros opcionales:

```typescript
class Configuracion {
  constructor(
    public host: string = "localhost",
    public puerto: number = 3000,
    public ssl: boolean = false,
    public timeout?: number
  ) {}
}

const config1 = new Configuracion();
// config1.host = "localhost", config1.puerto = 3000, config1.ssl = false

const config2 = new Configuracion("api.example.com", 443, true, 5000);
// config2.host = "api.example.com", config2.puerto = 443, config2.ssl = true
// config2.timeout = 5000
```

Esto permite crear objetos con configuraciones sensatas por defecto,
pero con la flexibilidad de personalizar cada valor cuando sea necesario.

===============================================================================
5. VALIDACIÓN EN EL CONSTRUCTOR
===============================================================================

El constructor es el lugar ideal para validar los datos antes de
inicializar el objeto. Puedes lanzar errores si los datos no cumplen
con los requisitos establecidos:

```typescript
class CuentaBancaria {
  constructor(
    public readonly numeroCuenta: string,
    public titular: string,
    private _saldo: number
  ) {
    if (!numeroCuenta || numeroCuenta.length < 10) {
      throw new Error("El número de cuenta debe tener al menos 10 dígitos");
    }
    if (!titular || titular.trim().length === 0) {
      throw new Error("El titular no puede estar vacío");
    }
    if (_saldo < 0) {
      throw new Error("El saldo inicial no puede ser negativo");
    }
  }

  obtenerSaldo(): number {
    return this._saldo;
  }
}

// ✅ Válido
const cuenta1 = new CuentaBancaria("1234567890", "Ana López", 1000);

// ❌ Lanza error
const cuenta2 = new CuentaBancaria("12", "", -500);
```

También puedes realizar transformaciones de datos en el constructor:

```typescript
class FechaPersonalizada {
  constructor(
    public readonly fecha: Date
  ) {
    // Normalizar a medianoche para comparaciones precisas
    this.fecha = new Date(
      fecha.getFullYear(),
      fecha.getMonth(),
      fecha.getDate()
    );
  }
}
```

===============================================================================
6. SOBRECARGA DE CONSTRUCTORES
===============================================================================

TypeScript soporta la sobrecarga de constructores, permitiendo definir
múltiples firmas para el constructor. Sin embargo, a diferencia de otros
lenguajes, solo puede haber una implementación real.

La sintaxis consiste en escribir múltiples declaraciones de constructor
(solo la firma, sin cuerpo) seguidas de una implementación única que
maneje todos los casos.

```typescript
class Punto {
  x: number;
  y: number;

  // Sobrecarga: sin parámetros
  constructor();
  // Sobrecarga: con coordenadas individuales
  constructor(x: number, y: number);
  // Sobrecarga: con objeto
  constructor(coord: { x: number; y: number });
  // Implementación única
  constructor(xOrCoord?: number | { x: number; y: number }, y?: number) {
    if (typeof xOrCoord === "undefined") {
      this.x = 0;
      this.y = 0;
    } else if (typeof xOrCoord === "object") {
      this.x = xOrCoord.x;
      this.y = xOrCoord.y;
    } else {
      this.x = xOrCoord;
      this.y = y ?? 0;
    }
  }
}

const origen = new Punto();
const punto1 = new Punto(3, 4);
const punto2 = new Punto({ x: 1, y: 2 });
```

Otro ejemplo común es una clase `Vector`:

```typescript
class Vector {
  constructor(
    public x: number = 0,
    public y: number = 0
  ) {}

  static desdeAngulo(magnitud: number, angulo: number): Vector {
    return new Vector(
      magnitud * Math.cos(angulo),
      magnitud * Math.sin(angulo)
    );
  }
}

// Sobrecarga con métodos estáticos de fábrica (factory methods)
class Usuario {
  constructor(
    public nombre: string,
    public email: string
  ) {}

  static desdeJSON(json: string): Usuario {
    const data = JSON.parse(json);
    return new Usuario(data.nombre, data.email);
  }

  static desdeFormulario(formData: FormData): Usuario {
    return new Usuario(
      formData.get("nombre") as string,
      formData.get("email") as string
    );
  }
}
```

7. CONSTRUCTOR PRIVADO (PATRÓN SINGLETON)
===============================================================================

Un constructor privado impide que se creen instancias desde fuera de la
clase. Esto es útil para implementar el patrón Singleton, donde solo
debe existir una única instancia de la clase.

```typescript
class ConexionBaseDatos {
  private static instancia: ConexionBaseDatos;
  private conexion: boolean = false;

  // Constructor privado: no se puede llamar desde fuera
  private constructor() {}

  static obtenerInstancia(): ConexionBaseDatos {
    if (!ConexionBaseDatos.instancia) {
      ConexionBaseDatos.instancia = new ConexionBaseDatos();
    }
    return ConexionBaseDatos.instancia;
  }

  conectar(): void {
    this.conexion = true;
    console.log("Conectado a la base de datos");
  }

  desconectar(): void {
    this.conexion = false;
    console.log("Desconectado de la base de datos");
  }
}

// const db = new ConexionBaseDatos(); // ❌ Error: constructor privado

const db1 = ConexionBaseDatos.obtenerInstancia();
const db2 = ConexionBaseDatos.obtenerInstancia();
console.log(db1 === db2); // true (misma instancia)
```

Otro patrón que usa constructores privados es el patrón Builder:

```typescript
class PersonaBuilder {
  private constructor(
    public readonly nombre: string,
    public readonly edad: number,
    public readonly email?: string,
    public readonly telefono?: string
  ) {}

  static crearConNombre(nombre: string): PersonaBuilder {
    return new PersonaBuilder(nombre, 0);
  }

  conEdad(edad: number): PersonaBuilder {
    return new PersonaBuilder(this.nombre, edad, this.email, this.telefono);
  }

  conEmail(email: string): PersonaBuilder {
    return new PersonaBuilder(this.nombre, this.edad, email, this.telefono);
  }

  conTelefono(telefono: string): PersonaBuilder {
    return new PersonaBuilder(this.nombre, this.edad, this.email, telefono);
  }

  construir(): Persona {
    return new Persona(
      this.nombre,
      this.edad,
      this.email,
      this.telefono
    );
  }
}
```

8. FLUJO DE INICIALIZACIÓN
===============================================================================

Es importante entender el orden en que se inicializan los elementos de una
clase cuando se crea una instancia:

1. Se asignan los valores por defecto de las propiedades declaradas en la
   clase (en el orden en que aparecen).
2. Se ejecuta el constructor.
3. Si hay herencia, primero se ejecuta el constructor de la clase padre
   (mediante `super()`).

```typescript
class Demostracion {
  paso1: string = this.mostrarPaso("1: Inicialización directa");
  paso2: string;

  constructor() {
    this.paso2 = this.mostrarPaso("2: Inicialización en constructor");
  }

  private mostrarPaso(mensaje: string): string {
    console.log(mensaje);
    return mensaje;
  }
}

const demo = new Demostracion();
// Output:
// 1: Inicialización directa
// 2: Inicialización en constructor
```

Este flujo es importante porque significa que las propiedades con valores
por defecto se inicializan antes de que el constructor acceda a ellas,
pero también significa que el constructor puede sobrescribir esos valores.

9. PARÁMETROS REST EN CONSTRUCTORES
===============================================================================

Los constructores también pueden usar parámetros rest para aceptar un
número variable de argumentos:

```typescript
class ListaNumeros {
  private numeros: number[];

  constructor(...numeros: number[]) {
    this.numeros = numeros;
  }

  agregar(...valores: number[]): void {
    this.numeros.push(...valores);
  }

  obtenerNumeros(): number[] {
    return [...this.numeros];
  }

  sumar(): number {
    return this.numeros.reduce((acc, n) => acc + n, 0);
  }
}

const lista = new ListaNumeros(1, 2, 3, 4, 5);
lista.agregar(6, 7, 8);
console.log(lista.sumar()); // 36
```

10. CONSTRUCTORES Y READONLY CON PARAMETER PROPERTIES
===============================================================================

Una combinación muy potente es usar `readonly` con parameter properties.
Esto crea propiedades que solo pueden asignarse durante la construcción:

```typescript
class Documento {
  constructor(
    public readonly id: string,
    public readonly fechaCreacion: Date = new Date(),
    public titulo: string,
    private _contenido: string
  ) {
    if (!id) {
      throw new Error("El ID es obligatorio");
    }
  }
}

const doc = new Documento("DOC-001", undefined, "Mi Documento", "Contenido...");
// doc.id = "DOC-002";         // ❌ Error: readonly
// doc.fechaCreacion = new Date(); // ❌ Error: readonly
doc.titulo = "Nuevo título";  // ✅ Válido
```

Pero ten cuidado: si usas `readonly` sin modificador de acceso, la
propiedad no se crea automáticamente como parameter property. Solo se
crea cuando hay un modificador de acceso (`public`, `private`, `protected`).

```typescript
class Ejemplo {
  constructor(
    readonly id: string,           // ❌ NO es parameter property (solo readonly)
    public readonly codigo: string // ✅ Es parameter property (public readonly)
  ) {
    // this.id necesita declaración explícita arriba
    // this.codigo se declara automáticamente
  }
}
```

===============================================================================
11. COMPARACIÓN CON JAVASCRIPT
===============================================================================

Es importante entender las diferencias entre los constructores de
TypeScript y JavaScript:

| Característica            | JavaScript          | TypeScript            |
|---------------------------|---------------------|-----------------------|
| Tipado en parámetros      | No                  | Sí                    |
| Parameter properties      | No                  | Sí                    |
| Varias firmas             | No                  | Sí (sobrecarga)       |
| Constructor privado       | No nativamente      | Sí                    |
| Validación en compile     | No                  | Sí                    |
| Valores por defecto       | ES6+                | Sí                    |
| Parámetros opcionales     | No nativamente      | Sí (con ?)            |

```typescript
// JavaScript
class UsuarioJS {
  constructor(nombre) {
    this.nombre = nombre;
  }
}

// TypeScript
class UsuarioTS {
  constructor(public nombre: string) {}
}
```

12. BUENAS PRÁCTICAS CON CONSTRUCTORES
===============================================================================

1. **Usa parameter properties siempre que sea posible**: Reduce el código
   boilerplate y mejora la legibilidad.

2. **Mantén los constructores simples**: Si el constructor necesita mucha
   lógica, considera usar un método estático de fábrica o el patrón Builder.

3. **Valida los datos de entrada**: El constructor es el guardián de tu
   objeto. No permitas estados inválidos.

4. **No hagas trabajo pesado en el constructor**: Si necesitas hacer
   operaciones costosas (llamadas a API, lectura de archivos), considera
   usar inicialización asíncrona con métodos estáticos.

5. **Usa sobrecarga con moderación**: La sobrecarga de constructores puede
   ser confusa. A veces es mejor usar métodos estáticos con nombres
   descriptivos (factory methods).

6. **Constructor privado para Singletons**: Cuando necesites garantizar
   una única instancia, combina constructor privado con un método estático.

```typescript
class RepositorioUsuarios {
  private constructor(
    private usuarios: Map<string, Usuario>
  ) {}

  static async crear(): Promise<RepositorioUsuarios> {
    const datos = await fetch("/api/usuarios").then(r => r.json());
    const usuarios = new Map(datos.map((u: any) => [u.id, u]));
    return new RepositorioUsuarios(usuarios);
  }
}
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- El constructor es un método especial que inicializa las propiedades
  cuando se crea una instancia con `new`.
- Los parámetros pueden ser obligatorios, opcionales (`?`) o tener valores
  por defecto, siguiendo las mismas reglas que las funciones.
- Las parameter properties permiten declarar e inicializar propiedades en
  un solo paso anteponiendo `public`, `private`, `protected` o `readonly`
  al parámetro del constructor.
- La sobrecarga de constructores permite múltiples firmas de entrada con
  una única implementación que discrimina los casos.
- Los constructores privados habilitan patrones como Singleton y Builder.
- El flujo de inicialización primero asigna valores por defecto y luego
  ejecuta el cuerpo del constructor.
- Las validaciones en el constructor protegen la integridad del objeto.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener múltiples constructores como en Java?**
TypeScript permite declarar múltiples firmas (sobrecarga), pero solo puede
haber una implementación. Esa implementación debe discriminar entre los
diferentes casos usando condicionales.

❓ **¿Qué pasa si no defino un constructor?**
Si no defines un constructor, TypeScript genera uno vacío por defecto. Si
extiendes de otra clase, el constructor por defecto llamará a `super()`.

❓ **¿Puedo usar async/await en un constructor?**
No, los constructores no pueden ser asíncronos. Si necesitas inicialización
asíncrona, usa un método estático factory que devuelva una Promise.

❓ **¿Cuándo debo usar parameter properties vs. declaración explícita?**
Usa parameter properties para propiedades que se asignan directamente desde
los parámetros del constructor. Usa declaración explícita si necesitas
lógica adicional, valores calculados o si la propiedad no viene de un
parámetro.

❓ **¿Los parámetros con valores por defecto afectan el tipo?**
Sí, un parámetro con valor por defecto es opcional en la práctica. TypeScript
infiere el tipo del valor por defecto, pero puedes anotarlo explícitamente.
