1. INTRODUCCIÓN A LAS CLASES
===============================================================================

La Programación Orientada a Objetos (POO) es un paradigma que organiza el
código en torno a "objetos" que contienen datos (propiedades) y
comportamiento (métodos). TypeScript implementa este paradigma mediante la
palabra reservada `class`, introducida en ECMAScript 2015 (ES6) y potenciada
con el sistema de tipos del lenguaje.

Una clase en TypeScript actúa como un "molde" o "plantilla" para crear
objetos que comparten una misma estructura y comportamiento. A diferencia
de JavaScript vanilla, TypeScript nos permite declarar los tipos de cada
propiedad y método directamente en la definición de la clase.

```typescript
class Persona {
  nombre: string;
  edad: number;

  constructor(nombre: string, edad: number) {
    this.nombre = nombre;
    this.edad = edad;
  }

  saludar(): string {
    return `Hola, soy ${this.nombre} y tengo ${this.edad} años.`;
  }
}
```

En el ejemplo anterior:
- La clase `Persona` tiene dos propiedades: `nombre` (string) y `edad` (number).
- El constructor inicializa estas propiedades cuando se crea una instancia.
- El método `saludar` devuelve un string con un mensaje personalizado.

Presta atención a que las propiedades se declaran con su tipo antes de ser
usadas en el constructor. Esto es una diferencia clave con JavaScript, donde
las propiedades se crean dinámicamente.

===============================================================================
2. DECLARACIÓN DE PROPIEDADES
===============================================================================

En TypeScript, todas las propiedades de una clase deben ser declaradas
explícitamente, a menos que se utilicen propiedades de parámetros del
constructor (lo veremos en el Capítulo 32). Cada propiedad puede tener:

- Un nombre
- Un tipo (obligatorio en modo strict)
- Un valor inicial opcional
- Modificadores como `readonly`, `public`, `private`, `protected`

```typescript
class Usuario {
  // Propiedad con tipo explícito y valor inicial
  id: number = 0;
  // Propiedad con tipo explícito sin valor inicial
  nombre: string;
  // Propiedad readonly: no puede modificarse después de la inicialización
  readonly fechaCreacion: Date;
  // Propiedad con tipo union
  activo: boolean | null;

  constructor(nombre: string) {
    this.nombre = nombre;
    this.fechaCreacion = new Date();
    this.activo = true;
  }
}

const usuario = new Usuario("Ana");
usuario.id = 10;          // ✅ Válido
usuario.fechaCreacion = new Date(); // ❌ Error: readonly
```

Es importante notar que TypeScript no obliga a inicializar todas las
propiedades en el constructor si se utiliza el operador `!` (definite
assignment assertion):

```typescript
class Configuracion {
  tema!: string; // Afirmamos que será inicializada antes de su uso
  idioma!: string;
}
```

Sin embargo, abusar de esta característica puede llevar a errores en
tiempo de ejecución. Es mejor inicializar las propiedades adecuadamente.

===============================================================================
3. MÉTODOS DE INSTANCIA
===============================================================================

Los métodos de instancia son funciones definidas dentro de una clase que
operan sobre los datos de una instancia específica. Se definen sin la
palabra `function` y pueden acceder a `this` para referirse a la instancia.

```typescript
class Calculadora {
  valor: number = 0;

  sumar(n: number): this {
    this.valor += n;
    return this;
  }

  restar(n: number): this {
    this.valor -= n;
    return this;
  }

  obtenerResultado(): number {
    return this.valor;
  }
}

const calc = new Calculadora();
calc.sumar(10).restar(3);
console.log(calc.obtenerResultado()); // 7
```

Presta atención al retorno `this` en los métodos `sumar` y `restar`. Esto
permite el encadenamiento de métodos (method chaining), un patrón muy útil.

Los métodos también aceptan parámetros con tipos, valores por defecto y
parámetros opcionales, igual que las funciones regulares:

```typescript
class Saludo {
  saludar(nombre: string, titulo?: string): string {
    if (titulo) {
      return `Bienvenido, ${titulo} ${nombre}`;
    }
    return `Bienvenido, ${nombre}`;
  }
}
```

===============================================================================
4. PROPIEDADES Y MÉTODOS ESTÁTICOS
===============================================================================

Las propiedades y métodos estáticos pertenecen a la clase en sí misma, no a
las instancias individuales. Se definen con la palabra clave `static` y se
acceden usando el nombre de la clase, no `this`.

```typescript
class ConfiguracionGlobal {
  static readonly VERSION: string = "1.0.0";
  static idioma: string = "es";
  private static instancias: number = 0;

  static obtenerIdiomaActual(): string {
    return this.idioma;
  }

  static incrementarInstancias(): void {
    this.instancias++;
  }
}

console.log(ConfiguracionGlobal.VERSION);            // "1.0.0"
console.log(ConfiguracionGlobal.obtenerIdiomaActual()); // "es"
```

Los miembros estáticos son ideales para:
- Constantes relacionadas con la clase
- Métodos de utilidad o fábrica (factory)
- Contadores o registros compartidos entre todas las instancias

Las propiedades estáticas también pueden ser privadas, protegiendo así
datos internos de la clase.

===============================================================================
5. READONLY Y PROPIEDADES DE SOLO LECTURA
===============================================================================

El modificador `readonly` permite definir propiedades que solo pueden ser
asignadas durante la inicialización (directamente en su declaración o
dentro del constructor). Cualquier intento de modificación posterior
genera un error en tiempo de compilación.

```typescript
class TarjetaCredito {
  readonly numero: string;
  readonly titular: string;
  readonly fechaExpiracion: Date;

  constructor(numero: string, titular: string, fechaExpiracion: Date) {
    this.numero = numero;
    this.titular = titular;
    this.fechaExpiracion = fechaExpiracion;
  }

  mostrarInfo(): string {
    return `Tarjeta de ${this.titular}`;
  }
}

const tarjeta = new TarjetaCredito("1234", "Luis", new Date("2028-12-31"));
tarjeta.numero = "5678"; // ❌ Error: No se puede asignar a readonly
```

`readonly` es especialmente útil para:
- Identificadores únicos que no deben cambiar
- Fechas de creación
- Constantes de configuración
- Datos inmutables

A diferencia de `const`, que se usa para variables, `readonly` se aplica a
propiedades de clases e interfaces.

===============================================================================
6. CLASES COMO TIPOS
===============================================================================

Una particularidad de TypeScript es que las clases generan tanto un valor
(en JavaScript) como un tipo (en el sistema de tipos). Esto significa que
puedes usar una clase como tipo de una variable, parámetro o retorno.

```typescript
class Producto {
  constructor(
    public nombre: string,
    public precio: number
  ) {}
}

function aplicarDescuento(producto: Producto, porcentaje: number): Producto {
  const descuento = producto.precio * (porcentaje / 100);
  return new Producto(producto.nombre, producto.precio - descuento);
}

const camisa = new Producto("Camisa", 500);
const camisaOferta = aplicarDescuento(camisa, 20);
```

Esto significa que toda clase define implícitamente una interfaz con la
misma estructura. TypeScript utiliza el tipado estructural (duck typing),
por lo que cualquier objeto que tenga la misma forma será compatible con
el tipo de la clase, aunque no sea una instancia de ella.

```typescript
class Perro {
  constructor(public nombre: string, public raza: string) {}
}

function presentar(mascota: Perro): void {
  console.log(`${mascota.nombre} es un ${mascota.raza}`);
}

// ✅ Un objeto literal con la misma estructura es compatible
presentar({ nombre: "Rex", raza: "Pastor Alemán" });
```

===============================================================================
7. COMPARACIÓN CON OBJETOS LITERALES
===============================================================================

Antes de las clases, era común usar objetos literales para agrupar datos
y funciones. Sin embargo, las clases ofrecen ventajas importantes:

| Característica        | Objeto Literal      | Clase               |
|-----------------------|---------------------|---------------------|
| Reutilización         | Manual (copiar)     | Con new             |
| Tipado                | Inferido            | Explícito           |
| Métodos              | En el objeto        | En el prototipo     |
| Herencia             | Object.assign       | extends             |
| Encapsulación        | No                  | Sí (modificadores)  |

```typescript
// Objeto literal
const usuarioLiteral = {
  nombre: "Carlos",
  edad: 30,
  saludar: function() {
    return `Hola, soy ${this.nombre}`;
  }
};

// Clase equivalente
class UsuarioClase {
  constructor(
    public nombre: string,
    public edad: number
  ) {}

  saludar(): string {
    return `Hola, soy ${this.nombre}`;
  }
}

const usuarioClase = new UsuarioClase("Carlos", 30);
```

Las clases son más adecuadas cuando necesitas crear múltiples objetos con
la misma estructura, aplicar herencia, o utilizar modificadores de acceso.

===============================================================================
8. THIS EN CLASES
===============================================================================

El manejo de `this` en TypeScript es más seguro que en JavaScript gracias
al análisis estático. TypeScript puede detectar usos incorrectos de `this`
y prevenir errores comunes.

```typescript
class Boton {
  private texto: string;

  constructor(texto: string) {
    this.texto = texto;
  }

  // Arrow function: captura el this del contexto circundante
  onClick = (): void => {
    console.log(`Botón "${this.texto}" clickeado`);
  };
}

// Uso correcto
const boton = new Boton("Enviar");
boton.onClick(); // ✅ "Botón Enviar clickeado"

// Esto también funciona porque la arrow function preserva el this
const clickHandler = boton.onClick;
clickHandler(); // ✅ "Botón Enviar clickeado"
```

Si usas una función regular en lugar de arrow function, el `this` dependerá
de cómo se invoque la función, lo que puede causar errores:

```typescript
class BotonInseguro {
  private texto: string;

  constructor(texto: string) {
    this.texto = texto;
  }

  onClick(): void {
    console.log(`Botón "${this.texto}" clickeado`);
  }
}

const boton2 = new BotonInseguro("Cancelar");
const handler = boton2.onClick;
handler(); // ❌ undefined en tiempo de ejecución (this perdido)
```

TypeScript te advertirá sobre estos casos si configuras `noImplicitThis`
en tu tsconfig.json.

===============================================================================
9. PROPIEDADES OPCIONALES Y CON VALORES POR DEFECTO
===============================================================================

Al igual que en interfaces y tipos, las propiedades de una clase pueden ser
opcionales (con `?`) o tener valores por defecto.

```typescript
class Empleado {
  nombre: string;
  apellido?: string;         // Propiedad opcional
  salario: number = 1000;    // Valor por defecto
  departamento: string;

  constructor(nombre: string, departamento: string) {
    this.nombre = nombre;
    this.departamento = departamento;
  }

  obtenerNombreCompleto(): string {
    if (this.apellido) {
      return `${this.nombre} ${this.apellido}`;
    }
    return this.nombre;
  }
}

const empleado1 = new Empleado("Ana", "Ventas");
console.log(empleado1.salario); // 1000 (valor por defecto)

const empleado2 = new Empleado("Luis", "IT");
empleado2.apellido = "Pérez";
empleado2.salario = 2500;
```

Las propiedades opcionales deben verificarse antes de usarse, igual que
con cualquier tipo nullable. TypeScript no te obliga a inicializarlas en
el constructor.

10. BUENAS PRÁCTICAS CON CLASES
===============================================================================

Al trabajar con clases en TypeScript, es importante seguir ciertas buenas
prácticas que harán tu código más mantenible y profesional:

1. **Declara siempre los tipos de las propiedades**: Nunca confíes solo en
   la inferencia para las propiedades de clase. Sé explícito.

2. **Inicializa las propiedades en el constructor o en la declaración**:
   Evita propiedades que queden sin inicializar. Usa el operador `!` solo
   cuando sea estrictamente necesario.

3. **Prefiere propiedades de parámetros del constructor** para clases
   simples (lo veremos en el Capítulo 32).

4. **Usa readonly para propiedades que no deben cambiar** después de la
   inicialización. Esto comunica intención y previene errores.

5. **Mantén las clases pequeñas y con una sola responsabilidad**: Una clase
   debe tener una única razón para cambiar (Principio de Responsabilidad
   Única).

6. **Encadena métodos cuando tenga sentido**: Esto mejora la legibilidad
   y fluidez del código.

7. **Usa métodos estáticos para operaciones que no dependen del estado**
   de una instancia.

```typescript
class Validador {
  static esEmail(valor: string): boolean {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(valor);
  }

  static esTelefono(valor: string): boolean {
    return /^\+\d{1,3}\d{7,10}$/.test(valor);
  }
}

console.log(Validador.esEmail("usuario@example.com")); // true
```

===============================================================================
11. ERRORES COMUNES Y CÓMO EVITARLOS
===============================================================================

Al empezar con clases en TypeScript, es fácil caer en ciertos errores.
Conocerlos te ayudará a evitarlos:

**Error 1: Olvidar declarar propiedades**
```typescript
// ❌ Incorrecto
class Usuario {
  constructor(nombre: string) {
    this.nombre = nombre; // Error: 'nombre' no está declarada
  }
}

// ✅ Correcto
class Usuario {
  nombre: string;
  constructor(nombre: string) {
    this.nombre = nombre;
  }
}
```

**Error 2: No inicializar propiedades obligatorias**
```typescript
// ❌ Incorrecto
class Usuario {
  nombre: string; // Error en strict mode si no se inicializa
}

// ✅ Correcto
class Usuario {
  nombre: string = "Invitado";
}
```

**Error 3: Asignar a propiedades readonly fuera del constructor**
```typescript
class Pedido {
  readonly id: string;
  constructor() {
    this.id = "temp"; // ✅ Válido
  }

  actualizarId(nuevoId: string): void {
    this.id = nuevoId; // ❌ Error: readonly
  }
}
```

===============================================================================
12. CLASES Y EL COMPILADOR DE TYPESCRIPT
===============================================================================

Es importante entender cómo TypeScript compila las clases a JavaScript.
Dependiendo de la versión de ECMAScript que configures en tu tsconfig.json,
el código generado será diferente.

Para ES2015 o superior, TypeScript genera código `class` nativo:

```typescript
// TypeScript
class Persona {
  nombre: string;
  constructor(nombre: string) {
    this.nombre = nombre;
  }
}

// JavaScript (target ES2015+)
class Persona {
  constructor(nombre) {
    this.nombre = nombre;
  }
}
```

Para ES5 o inferior, TypeScript genera código basado en funciones
constructoras y prototipos:

```javascript
// JavaScript (target ES5)
var Persona = (function () {
  function Persona(nombre) {
    this.nombre = nombre;
  }
  return Persona;
}());
```

Esta compatibilidad hacia atrás permite usar clases de TypeScript en
entornos que no soportan ES2015, como navegadores antiguos.

RESUMEN DEL CAPÍTULO
===============================================================================

- Las clases en TypeScript son una mejora sobre las clases de ES6 con
  tipado estático, modificadores y nuevas capacidades.
- Las propiedades deben declararse explícitamente con su tipo, ya sea en
  la declaración o mediante propiedades de parámetros del constructor.
- Los métodos de instancia operan sobre datos de una instancia específica
  y pueden encadenarse si retornan `this`.
- Las propiedades y métodos estáticos pertenecen a la clase y se accede
  a ellos sin instanciar.
- `readonly` previene la modificación de propiedades después de la
  inicialización, ideal para datos inmutables.
- Las clases generan tanto un valor (en JS) como un tipo (en TS).
- El manejo de `this` es más seguro con TypeScript, especialmente usando
  arrow functions para métodos callback.
- Las propiedades pueden ser opcionales (`?`) o tener valores por defecto.
- TypeScript compila clases a JavaScript nativo o funciones constructoras
  según la configuración de salida.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Es obligatorio declarar las propiedades antes del constructor?**
Sí, en TypeScript todas las propiedades deben declararse explícitamente,
a menos que uses propiedades de parámetros del constructor (lo veremos en
el Capítulo 32). En JavaScript esto no es necesario porque las propiedades
se crean dinámicamente.

❓ **¿Cuál es la diferencia entre class y type?**
Una clase genera tanto un valor (código JavaScript ejecutable) como un tipo
(para el sistema de tipos). Un `type` solo existe en tiempo de compilación
y desaparece al compilar. Las clases también proporcionan implementación
concreta (métodos, constructores), mientras que los tipos son solo
estructuras.

❓ **¿Puedo tener propiedades sin inicializar?**
Sí, pero TypeScript en modo strict te obligará a inicializarlas en el
constructor. Puedes usar el operador `!` (definite assignment assertion)
para indicar que confías en que la propiedad será inicializada antes de
su uso, pero no es recomendable abusar de esta característica.

❓ **¿Las clases de TypeScript son iguales a las de Java o C#?**
Son muy similares en sintaxis, pero internamente siguen siendo prototipos
de JavaScript. TypeScript añade tipado, modificadores de acceso y otras
características, pero la base sigue siendo el sistema de prototipos de JS.

❓ **¿Puedo usar una clase como interfaz?**
Sí, toda clase define implícitamente un tipo con la misma forma que la
clase. Puedes usar una clase como tipo de un parámetro, variable o retorno.
TypeScript usa tipado estructural, por lo que cualquier objeto compatible
será aceptado.
