1. ¿QUÉ ES EL POLIMORFISMO?
===============================================================================

El polimorfismo es la capacidad de un objeto de comportarse de diferentes
maneras según el contexto, manteniendo una misma interfaz. En términos
prácticos, significa que puedes escribir código que trabaje con tipos
abstractos y, en tiempo de ejecución, los objetos concretos ejecutarán
su propia versión del comportamiento.

```typescript
// Interfaz común (abstracta)
interface Animal {
  hacerSonido(): void;
}

// Implementaciones concretas
class Perro implements Animal {
  hacerSonido(): void {
    console.log("¡Guau!");
  }
}

class Gato implements Animal {
  hacerSonido(): void {
    console.log("¡Miau!");
  }
}

class Vaca implements Animal {
  hacerSonido(): void {
    console.log("¡Muuu!");
  }
}

// Código polimórfico: funciona con cualquier Animal
function hacerQueSuene(animal: Animal): void {
  animal.hacerSonido();
}

const animales: Animal[] = [
  new Perro(),
  new Gato(),
  new Vaca()
];

animales.forEach(hacerQueSuene);
// "¡Guau!"
// "¡Miau!"
// "¡Muuu!"
```

La función `hacerQueSuene` no sabe ni le importa qué tipo concreto de
Animal recibe. Solo sabe que responde al método `hacerSonido()`. Esto es
polimorfismo en acción.

===============================================================================
2. POLIMORFISMO POR HERENCIA
===============================================================================

La forma más clásica de polimorfismo es mediante herencia y sobrescritura
de métodos. Una clase base define el contrato y las subclases proporcionan
implementaciones específicas.

```typescript
class Figura {
  constructor(public nombre: string) {}

  calcularArea(): number {
    return 0; // Implementación por defecto
  }

  describir(): string {
    return `Soy un ${this.nombre} con área ${this.calcularArea()}`;
  }
}

class Circulo extends Figura {
  constructor(public radio: number) {
    super("Círculo");
  }

  calcularArea(): number {
    return Math.PI * this.radio ** 2;
  }
}

class Rectangulo extends Figura {
  constructor(
    public base: number,
    public altura: number
  ) {
    super("Rectángulo");
  }

  calcularArea(): number {
    return this.base * this.altura;
  }
}

class Triangulo extends Figura {
  constructor(
    public base: number,
    public altura: number
  ) {
    super("Triángulo");
  }

  calcularArea(): number {
    return (this.base * this.altura) / 2;
  }
}

// Código polimórfico
function mostrarAreas(figuras: Figura[]): void {
  figuras.forEach(figura => {
    console.log(figura.describir());
  });
}

const figuras: Figura[] = [
  new Circulo(5),
  new Rectangulo(10, 4),
  new Triangulo(6, 3)
];

mostrarAreas(figuras);
// "Soy un Círculo con área 78.539..."
// "Soy un Rectángulo con área 40"
// "Soy un Triángulo con área 9"
```

Cada subclase sobrescribe `calcularArea()` con su propia fórmula. La
función `mostrarAreas` trata a todas las figuras como `Figura`, pero
cada una se comporta según su tipo real.

===============================================================================
3. POLIMORFISMO POR INTERFACES
===============================================================================

En TypeScript, las interfaces ofrecen una forma más flexible de
polimorfismo que no requiere una jerarquía de herencia. Cualquier clase
que implemente una interfaz puede ser tratada polimórficamente.

```typescript
interface Imprimible {
  imprimir(): string;
}

interface Exportable {
  exportar(formato: string): void;
}

class Documento implements Imprimible, Exportable {
  constructor(private contenido: string) {}

  imprimir(): string {
    return `Documento: ${this.contenido}`;
  }

  exportar(formato: string): void {
    console.log(`Exportando documento a ${formato}`);
  }
}

class Imagen implements Imprimible {
  constructor(private ruta: string) {}

  imprimir(): string {
    return `Imagen desde: ${this.ruta}`;
  }
}

class Factura implements Imprimible, Exportable {
  constructor(
    private numero: string,
    private total: number
  ) {}

  imprimir(): string {
    return `Factura #${this.numero} - Total: $${this.total}`;
  }

  exportar(formato: string): void {
    console.log(`Exportando factura a ${formato}`);
  }
}

function procesarImpresion(elementos: Imprimible[]): void {
  elementos.forEach(e => console.log(e.imprimir()));
}

const elementos: Imprimible[] = [
  new Documento("Contenido secreto"),
  new Imagen("/fotos/logo.png"),
  new Factura("F-001", 1500)
];

procesarImpresion(elementos);
```

Las interfaces permiten polimorfismo transversal: clases no relacionadas
por herencia pueden compartir comportamientos.

===============================================================================
4. POLIMORFISMO ESTRUCTURAL (DUCK TYPING)
===============================================================================

TypeScript tiene tipado estructural, lo que significa que el polimorfismo
no requiere una declaración explícita de `implements`. Si un objeto tiene
la forma adecuada, es compatible:

```typescript
interface Volador {
  volar(): void;
}

class Aguila {
  volar(): void {
    console.log("El águila vuela alto");
  }
}

class Avion {
  volar(): void {
    console.log("El avión despega");
  }
}

// La clase Aguila no implementa Volador explícitamente,
// pero es compatible estructuralmente
const voladores: Volador[] = [
  new Aguila(),     // ✅ Compatible por su estructura
  new Avion(),      // ✅ Compatible por su estructura
  {
    volar: () => console.log("Objeto volador anónimo")
  }
];

function lanzar(volador: Volador): void {
  volador.volar();
}

voladores.forEach(lanzar);
```

Esto es una gran ventaja de TypeScript sobre lenguajes como Java o C#:
no necesitas acoplar tus clases a interfaces específicas. Si la forma
coincide, el polimorfismo funciona.

5. POLIMORFISMO CON GENÉRICOS
===============================================================================

Los genéricos permiten una forma de polimorfismo paramétrico, donde una
función o clase puede trabajar con cualquier tipo de manera segura.

```typescript
// Polimorfismo con genéricos
interface Comparable<T> {
  comparar(otro: T): number;
}

class Numero implements Comparable<Numero> {
  constructor(public valor: number) {}

  comparar(otro: Numero): number {
    if (this.valor < otro.valor) return -1;
    if (this.valor > otro.valor) return 1;
    return 0;
  }
}

class Persona implements Comparable<Persona> {
  constructor(public nombre: string) {}

  comparar(otro: Persona): number {
    return this.nombre.localeCompare(otro.nombre);
  }
}

function encontrarMaximo<T extends Comparable<T>>(elementos: T[]): T {
  if (elementos.length === 0) {
    throw new Error("Array vacío");
  }

  let maximo = elementos[0];
  for (let i = 1; i < elementos.length; i++) {
    if (elementos[i].comparar(maximo) > 0) {
      maximo = elementos[i];
    }
  }
  return maximo;
}

const numeros = [new Numero(3), new Numero(7), new Numero(1)];
console.log(encontrarMaximo(numeros).valor); // 7

const personas = [
  new Persona("Carlos"),
  new Persona("Ana"),
  new Persona("Beatriz")
];
console.log(encontrarMaximo(personas).nombre); // "Carlos"
```

El polimorfismo genérico permite que una misma función trabaje con
cualquier tipo que cumpla con el contrato especificado.

6. POLIMORFISMO VS SOBRECARGA
===============================================================================

Es importante no confundir polimorfismo con sobrecarga de métodos.
La sobrecarga permite que un mismo método tenga diferentes firmas
(múltiples versiones) dentro de la misma clase. El polimorfismo se
refiere a que diferentes objetos respondan a la misma firma de
método de formas distintas.

```typescript
// Sobrecarga (compile-time polymorphism)
class Calculadora {
  sumar(a: number, b: number): number;
  sumar(a: string, b: string): string;
  sumar(a: number, b: number, c: number): number;
  sumar(a: any, b: any, c?: any): any {
    if (c !== undefined) return a + b + c;
    return a + b;
  }
}

// Polimorfismo (runtime polymorphism)
interface Operacion {
  ejecutar(a: number, b: number): number;
}

class Suma implements Operacion {
  ejecutar(a: number, b: number): number {
    return a + b;
  }
}

class Resta implements Operacion {
  ejecutar(a: number, b: number): number {
    return a - b;
  }
}

class Multiplicacion implements Operacion {
  ejecutar(a: number, b: number): number {
    return a * b;
  }
}

function calcular(operacion: Operacion, a: number, b: number): number {
  return operacion.ejecutar(a, b);
}

console.log(calcular(new Suma(), 5, 3));          // 8
console.log(calcular(new Resta(), 5, 3));         // 2
console.log(calcular(new Multiplicacion(), 5, 3)); // 15
```

La sobrecarga es polimorfismo en tiempo de compilación (una función,
múltiples firmas). La sobrescritura es polimorfismo en tiempo de
ejecución (múltiples clases, misma firma, diferentes implementaciones).

7. POLIMORFISMO CON CLASES ABSTRACTAS
===============================================================================

Las clases abstractas combinan lo mejor de la herencia y las interfaces.
Pueden definir implementaciones parciales y métodos abstractos que las
subclases deben implementar.

```typescript
abstract class Empleado {
  constructor(
    public nombre: string,
    public salarioBase: number
  ) {}

  abstract calcularSalario(): number;

  obtenerInfo(): string {
    return `${this.nombre}: $${this.calcularSalario()}`;
  }
}

class EmpleadoFijo extends Empleado {
  calcularSalario(): number {
    return this.salarioBase;
  }
}

class EmpleadoPorHoras extends Empleado {
  constructor(
    nombre: string,
    salarioBase: number,
    public horasTrabajadas: number,
    public tarifaPorHora: number
  ) {
    super(nombre, salarioBase);
  }

  calcularSalario(): number {
    return this.horasTrabajadas * this.tarifaPorHora;
  }
}

class EmpleadoComision extends Empleado {
  constructor(
    nombre: string,
    salarioBase: number,
    public ventas: number,
    public porcentajeComision: number
  ) {
    super(nombre, salarioBase);
  }

  calcularSalario(): number {
    return this.salarioBase + (this.ventas * this.porcentajeComision / 100);
  }
}

function procesarNomina(empleados: Empleado[]): number {
  let total = 0;
  empleados.forEach(e => {
    console.log(e.obtenerInfo());
    total += e.calcularSalario();
  });
  return total;
}

const empleados: Empleado[] = [
  new EmpleadoFijo("Ana", 3000),
  new EmpleadoPorHoras("Luis", 0, 160, 25),
  new EmpleadoComision("Carlos", 2000, 50000, 5)
];

console.log(`Total nómina: $${procesarNomina(empleados)}`);
```

8. EL PRINCIPIO ABIERTO/CERRADO (OCP)
===============================================================================

El polimorfismo es la clave para cumplir con el Principio Abierto/Cerrado
(Open/Closed Principle): las entidades deben estar abiertas para extensión
pero cerradas para modificación.

```typescript
// ❌ Sin polimorfismo: difícil de extender
class ProcesadorPagos {
  procesar(tipo: string, monto: number): void {
    if (tipo === "tarjeta") {
      console.log(`Procesando pago con tarjeta: $${monto}`);
    } else if (tipo === "paypal") {
      console.log(`Procesando pago con PayPal: $${monto}`);
    } else if (tipo === "cripto") {
      console.log(`Procesando pago con cripto: $${monto}`);
    }
    // Para añadir un nuevo método, hay que modificar esta clase 😱
  }
}

// ✅ Con polimorfismo: abierto a extensión
interface MetodoPago {
  procesar(monto: number): void;
  obtenerNombre(): string;
}

class PagoTarjeta implements MetodoPago {
  obtenerNombre(): string { return "Tarjeta"; }
  procesar(monto: number): void {
    console.log(`Procesando pago con tarjeta: $${monto}`);
  }
}

class PagoPayPal implements MetodoPago {
  obtenerNombre(): string { return "PayPal"; }
  procesar(monto: number): void {
    console.log(`Procesando pago con PayPal: $${monto}`);
  }
}

class PagoCripto implements MetodoPago {
  obtenerNombre(): string { return "Cripto"; }
  procesar(monto: number): void {
    console.log(`Procesando pago con cripto: $${monto}`);
  }
}

// Nuevo método sin modificar código existente:
class PagoTransferencia implements MetodoPago {
  obtenerNombre(): string { return "Transferencia"; }
  procesar(monto: number): void {
    console.log(`Procesando transferencia: $${monto}`);
  }
}

class ProcesadorPagos {
  constructor(private metodos: MetodoPago[]) {}

  procesarTodos(monto: number): void {
    this.metodos.forEach(m => m.procesar(monto));
  }
}
```

El polimorfismo permite que tu código sea extensible sin tener que
modificar las clases existentes.

9. TYPE GUARDS Y POLIMORFISMO
===============================================================================

A veces necesitas saber el tipo concreto de un objeto polimórfico para
realizar operaciones específicas. TypeScript ofrece varias herramientas:

```typescript
interface Vehiculo {
  marca: string;
  modelo: string;
  acelerar(): void;
}

class Coche implements Vehiculo {
  constructor(
    public marca: string,
    public modelo: string,
    public numPuertas: number
  ) {}

  acelerar(): void {
    console.log("Coche acelerando con motor de combustión");
  }

  abrirMaletero(): void {
    console.log("Maletero abierto");
  }
}

class Moto implements Vehiculo {
  constructor(
    public marca: string,
    public modelo: string,
    public tieneSidecar: boolean
  ) {}

  acelerar(): void {
    console.log("Moto acelerando");
  }

  hacerCaballito(): void {
    console.log("¡Haciendo caballito!");
  }
}

// Type guard con instanceof
function inspeccionarVehiculo(v: Vehiculo): void {
  v.acelerar();

  if (v instanceof Coche) {
    v.abrirMaletero(); // ✅ TypeScript sabe que es Coche
  }

  if (v instanceof Moto) {
    v.hacerCaballito(); // ✅ TypeScript sabe que es Moto
  }
}

// Type guard con propiedad discriminante
interface Triangulo {
  tipo: "triangulo";
  base: number;
  altura: number;
}

interface Rectangulo {
  tipo: "rectangulo";
  ancho: number;
  alto: number;
}

type FiguraGeometrica = Triangulo | Rectangulo;

function calcularArea(figura: FiguraGeometrica): number {
  switch (figura.tipo) {
    case "triangulo":
      return (figura.base * figura.altura) / 2;
    case "rectangulo":
      return figura.ancho * figura.alto;
  }
}
```

10. CASO PRÁCTICO: SISTEMA DE NOTIFICACIONES
===============================================================================

Un ejemplo completo que integra todos los aspectos del polimorfismo:

```typescript
interface Notificacion {
  enviar(mensaje: string): void;
  obtenerTipo(): string;
}

class NotificacionEmail implements Notificacion {
  private static readonly PREFIJO_ASUNTO = "[NOTIFICACIÓN]";

  constructor(
    private destinatario: string,
    private remitente: string = "noreply@system.com"
  ) {}

  obtenerTipo(): string {
    return "Email";
  }

  enviar(mensaje: string): void {
    console.log(`
      De: ${this.remitente}
      Para: ${this.destinatario}
      Asunto: ${NotificacionEmail.PREFIJO_ASUNTO}
      Mensaje: ${mensaje}
    `);
  }
}

class NotificacionSMS implements Notificacion {
  constructor(private telefono: string) {}

  obtenerTipo(): string {
    return "SMS";
  }

  enviar(mensaje: string): void {
    const mensajeCorto = mensaje.slice(0, 160);
    console.log(`Enviando SMS a ${this.telefono}: ${mensajeCorto}`);
  }
}

class NotificacionPush implements Notificacion {
  constructor(
    private tokenDispositivo: string,
    private titulo: string = "Nueva notificación"
  ) {}

  obtenerTipo(): string {
    return "Push";
  }

  enviar(mensaje: string): void {
    console.log(`
      Push a dispositivo ${this.tokenDispositivo.slice(0, 8)}...
      Título: ${this.titulo}
      Cuerpo: ${mensaje}
    `);
  }
}

class NotificacionCompuesta implements Notificacion {
  private notificaciones: Notificacion[] = [];

  constructor(private nombre: string) {}

  obtenerTipo(): string {
    return `Compuesta (${this.nombre})`;
  }

  agregar(notificacion: Notificacion): void {
    this.notificaciones.push(notificacion);
  }

  enviar(mensaje: string): void {
    console.log(`=== Enviando notificación compuesta: ${this.nombre} ===`);
    this.notificaciones.forEach(n => {
      console.log(`[${n.obtenerTipo()}]`);
      n.enviar(mensaje);
    });
    console.log(`=== Fin de notificación compuesta ===`);
  }
}

class SistemaNotificaciones {
  constructor(private canales: Notificacion[]) {}

  agregarCanal(canal: Notificacion): void {
    this.canales.push(canal);
  }

  notificar(mensaje: string): void {
    this.canales.forEach(canal => {
      try {
        canal.enviar(mensaje);
        console.log(`✅ ${canal.obtenerTipo()}: enviado`);
      } catch (error) {
        console.error(`❌ ${canal.obtenerTipo()}: error - ${error}`);
      }
    });
  }
}

// Uso del sistema
const email = new NotificacionEmail("usuario@mail.com");
const sms = new NotificacionSMS("+521234567890");
const push = new NotificacionPush("device-token-123", "Alerta");

const compuesta = new NotificacionCompuesta("Bienvenida");
compuesta.agregar(email);
compuesta.agregar(push);

const sistema = new SistemaNotificaciones([email, sms, push]);
sistema.agregarCanal(compuesta);

sistema.notificar("¡Bienvenido al sistema!");
```

11. POLIMORFISMO Y PATRONES DE DISEÑO
===============================================================================

El polimorfismo es la base de muchos patrones de diseño. Algunos
ejemplos:

**Strategy Pattern**: Permite cambiar el comportamiento de un algoritmo
en tiempo de ejecución.

```typescript
interface EstrategiaCompresion {
  comprimir(datos: string): string;
}

class CompresionZip implements EstrategiaCompresion {
  comprimir(datos: string): string {
    return `[ZIP:${datos}]`;
  }
}

class CompresionGzip implements EstrategiaCompresion {
  comprimir(datos: string): string {
    return `[GZIP:${datos}]`;
  }
}

class Compresor {
  constructor(private estrategia: EstrategiaCompresion) {}

  cambiarEstrategia(estrategia: EstrategiaCompresion): void {
    this.estrategia = estrategia;
  }

  comprimir(datos: string): string {
    return this.estrategia.comprimir(datos);
  }
}
```

**Observer Pattern**: Objetos se suscriben a eventos de otros objetos.

```typescript
interface Observador {
  actualizar(evento: string, datos: any): void;
}

class Logger implements Observador {
  actualizar(evento: string, datos: any): void {
    console.log(`[LOG] Evento: ${evento}`, datos);
  }
}

class NotificadorUI implements Observador {
  actualizar(evento: string, datos: any): void {
    alert(`Evento: ${evento}`);
  }
}
```

12. BUENAS PRÁCTICAS CON POLIMORFISMO
===============================================================================

1. **Programa para una interfaz, no para una implementación**: Este es el
   principio fundamental. Usa tipos abstractos (interfaces o clases
   abstractas) en lugar de tipos concretos.

2. **No preguntes por el tipo; solo llama al método**: Si necesitas usar
   `instanceof` o `typeof` constantemente, probablemente estás usando mal
   el polimorfismo.

3. **Mantén las interfaces pequeñas y enfocadas**: El principio de
   segregación de interfaces (ISP) dice que es mejor tener interfaces
   pequeñas y específicas.

4. **Usa el polimorfismo para eliminar condicionales**: Si tienes un
   `switch` o `if-else` que depende del tipo de un objeto, considera
   usar polimorfismo.

5. **No fuerces el polimorfismo donde no existe una relación natural**:
   No todas las clases necesitan compartir una interfaz solo por
   conveniencia.

```typescript
// ❌ Mal: forzar polimorfismo
interface Persistible {
  guardar(): void;
}

class Usuario implements Persistible {
  guardar(): void { /* guarda en BD */ }
}

class Configuracion implements Persistible {
  guardar(): void { /* guarda en archivo */ }
}

// ✅ Bien: polimorfismo natural
interface Reproducible {
  reproducir(): void;
  pausar(): void;
  detener(): void;
}

class Cancion implements Reproducible {
  reproducir(): void { /* reproduce canción */ }
  pausar(): void { /* pausa canción */ }
  detener(): void { /* detiene canción */ }
}

class Video implements Reproducible {
  reproducir(): void { /* reproduce video */ }
  pausar(): void { /* pausa video */ }
  detener(): void { /* detiene video */ }
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

- El polimorfismo permite que diferentes objetos respondan al mismo
  mensaje de formas distintas.
- Se puede implementar mediante herencia (sobrescritura), interfaces o
  tipado estructural.
- El tipado estructural de TypeScript permite polimorfismo sin
  declaraciones explícitas de implementación.
- Los genéricos proporcionan polimorfismo paramétrico.
- El Principio Abierto/Cerrado se logra mediante polimorfismo.
- Los type guards (`instanceof`, discriminantes) permiten manejar casos
  específicos dentro de código polimórfico.
- El polimorfismo es la base de patrones como Strategy, Observer,
  Command, y muchos más.
- Programa para interfaces, no para implementaciones concretas.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre polimorfismo y herencia?**
La herencia es un mecanismo para compartir código entre clases. El
polimorfismo es la capacidad de usar objetos de diferentes tipos a
través de una interfaz común. La herencia es una forma de lograr
polimorfismo, pero no la única.

❓ **¿Puedo tener polimorfismo sin herencia?**
Sí, en TypeScript puedes tener polimorfismo mediante interfaces y tipado
estructural. Un objeto es compatible con una interfaz si tiene la misma
forma, aunque no la implemente explícitamente.

❓ **¿Cuándo debo usar polimorfismo?**
Cuando tienes comportamientos que varían según el tipo de objeto, pero
quieres tratarlos de manera uniforme. Por ejemplo, diferentes formas de
calcular área, diferentes métodos de pago, diferentes formatos de
exportación.

❓ **¿El polimorfismo afecta el rendimiento?**
Ligeramente, porque las llamadas a métodos virtuales requieren una
indirección (vtable). Sin embargo, en la práctica, el impacto es mínimo
y los beneficios en mantenibilidad y flexibilidad superan con creces
el costo.

❓ **¿Cómo evito el abuso de instanceof?**
Si te encuentras usando `instanceof` frecuentemente, es posible que
estés violando el principio de polimorfismo. Pregúntate: ¿puedo mover
esta lógica condicional a un método polimórfico?
