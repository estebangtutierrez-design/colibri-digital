3.3 CONSTRUYE TU PORTAFOLIO

Tu portafolio debe incluir:

1. **LibroFlow** (el proyecto integrador del Capítulo 78) — demo en vivo y código
2. **TaskFlow** (el proyecto final del Capítulo 75) — con pruebas incluidas
3. **Tus propios proyectos** — aplicaciones que resuelvan problemas reales
4. **Contribuciones open source** — PRs a proyectos que uses
5. **Blog técnico** — artículos explicando conceptos de TypeScript

Cada proyecto debe tener:
- README.md con descripción, tecnologías y cómo ejecutarlo
- Tests que demuestren calidad
- Despliegue en vivo (Vercel, Netlify, Render)
- Código limpio y bien organizado

El aprendizaje nunca termina. Aquí tienes una lista curada de recursos
para continuar tu desarrollo profesional:

4.1 LIBROS RECOMENDADOS

- **"Programming TypeScript"** — Boris Cherny (O'Reilly)
  La guía definitiva para dominar el sistema de tipos a fondo.
- **"Effective TypeScript"** — Dan Vanderkam
  62 técnicas para mejorar tu código TypeScript.
- **"Clean Code"** — Robert C. Martin
  Principios universales de código limpio, aplicables a cualquier lenguaje.
- **"Design Patterns"** — Gang of Four
  El libro clásico de patrones de diseño, ahora aplicables con TypeScript.
- **"Domain-Driven Design"** — Eric Evans
  Para entender cómo modelar dominios complejos con tipos.
