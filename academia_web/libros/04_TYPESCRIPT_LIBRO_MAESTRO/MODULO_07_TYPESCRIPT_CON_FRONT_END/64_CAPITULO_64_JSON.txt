1. JSON.parse Y EL PELIGRO DE any
===============================================================================

`JSON.parse` devuelve `any`. Esto significa que TypeScript pierde todo
control de tipos cuando parseamos JSON.

```typescript
const datos = JSON.parse('{"name": "Ana", "age": 30}');
// Tipo: any — ¡peligro!

console.log(datos.name);    // ✅ funciona, pero sin seguridad
console.log(datos.edad);    // ❌ error de escritura, pero TS no lo captura
console.log(datos.xyz.no.existe); // 💥 runtime error!
```

Solución: declarar el tipo explícitamente.

```typescript
interface Persona {
  name: string;
  age: number;
}

const datos: Persona = JSON.parse('{"name": "Ana", "age": 30}');
console.log(datos.name);  // ✅ string
console.log(datos.age);   // ✅ number
```

Pero esto solo es una aserción. Si el JSON no coincide, tendremos un error
en runtime. Necesitamos algo más robusto.

===============================================================================
2. PARSEAR CON unknown Y VALIDAR
===============================================================================

Lo más seguro es parsear como `unknown` y luego validar:

```typescript
function parseJSON<T>(texto: string): T | null {
  try {
    const datos: unknown = JSON.parse(texto);
    return datos as T; // Todavía es una aserción
  } catch {
    return null;
  }
}

// Uso
const usuario = parseJSON<Persona>('{"name": "Ana", "age": 30}');
if (usuario) {
  console.log(usuario.name);
}
```

Pero la verdadera seguridad viene con la validación en runtime.

===============================================================================
3. TYPE GUARDS PARA JSON
===============================================================================

Un type guard es una función que verifica en runtime si un valor cumple
con una estructura esperada.

```typescript
interface Usuario {
  id: number;
  nombre: string;
  email: string;
  activo: boolean;
}

function esUsuario(dato: unknown): dato is Usuario {
  if (!dato || typeof dato !== "object") return false;

  const obj = dato as Record<string, unknown>;

  return (
    typeof obj.id === "number" &&
    typeof obj.nombre === "string" &&
    typeof obj.email === "string" &&
    typeof obj.activo === "boolean"
  );
}

// Uso
const raw = JSON.parse('{"id": 1, "nombre": "Ana", "email": "a@b.com", "activo": true}');

if (esUsuario(raw)) {
  console.log(raw.nombre.toUpperCase()); // ✅ TS sabe que es string
} else {
  console.error("Datos de usuario inválidos");
}
```

El type guard `dato is Usuario` le dice a TypeScript: "Si esta función
devuelve true, el dato es de tipo Usuario".

===============================================================================
4. VALIDACIÓN ESTRUCTURAL AVANZADA
===============================================================================

Para objetos más complejos:

```typescript
interface Direccion {
  calle: string;
  ciudad: string;
  codigoPostal: string;
}

interface UsuarioCompleto {
  id: number;
  nombre: string;
  direccion: Direccion;
  etiquetas: string[];
  puntuacion: number | null;
}

function esDireccion(dato: unknown): dato is Direccion {
  if (!dato || typeof dato !== "object") return false;
  const obj = dato as Record<string, unknown>;
  return (
    typeof obj.calle === "string" &&
    typeof obj.ciudad === "string" &&
    typeof obj.codigoPostal === "string"
  );
}

function esUsuarioCompleto(dato: unknown): dato is UsuarioCompleto {
  if (!dato || typeof dato !== "object") return false;
  const obj = dato as Record<string, unknown>;

  return (
    typeof obj.id === "number" &&
    typeof obj.nombre === "string" &&
    esDireccion(obj.direccion) &&
    Array.isArray(obj.etiquetas) &&
    obj.etiquetas.every(e => typeof e === "string") &&
    (obj.puntuacion === null || typeof obj.puntuacion === "number")
  );
}
```

===============================================================================
5. FUNCIONES DE ASERCIÓN (ASSERTION FUNCTIONS)
===============================================================================

TypeScript permite funciones que lanzan error si la validación falla:

```typescript
function asegurarUsuario(dato: unknown): asserts dato is Usuario {
  if (!esUsuario(dato)) {
    throw new Error("El dato no es un Usuario válido");
  }
}

// Uso — si pasa, dato es Usuario; si no, lanza error
const raw = JSON.parse('{"id": 1, "nombre": "Ana", "email": "a@b.com", "activo": true}');

asegurarUsuario(raw);
console.log(raw.nombre); // ✅ TS sabe que es string
```

Las assertion functions son ideales para puntos de entrada de datos
(APIs, archivos, localStorage).

6. JSON.STRINGIFY CON TIPOS
===============================================================================

`JSON.stringify` también tiene particularidades con tipos:

```typescript
interface Configuracion {
  tema: "claro" | "oscuro";
  fontSize: number;
  notificaciones: boolean;
}

const config: Configuracion = {
  tema: "oscuro",
  fontSize: 16,
  notificaciones: true
};

const json = JSON.stringify(config);
// Tipo: string

// Con pretty print
const jsonBonito = JSON.stringify(config, null, 2);
```

`JSON.stringify` acepta un "replacer" que también puede ser tipado:

```typescript
function replacer(clave: string, valor: unknown): unknown {
  if (clave === "password") return undefined; // omitir
  return valor;
}

const usuario = { id: 1, nombre: "Ana", password: "secreta" };
const json = JSON.stringify(usuario, replacer);
// {"id":1,"nombre":"Ana"}
```

7. UNIONES DISCRIMINADAS EN JSON
===============================================================================

Las APIs a menudo devuelven formas diferentes según el resultado:

```typescript
// Respuesta exitosa vs error
type RespuestaAPI<T> =
  | { status: "success"; data: T }
  | { status: "error"; error: string; codigo: number };

// Type guard para cada caso
function esExito<T>(respuesta: RespuestaAPI<T>): respuesta is { status: "success"; data: T } {
  return respuesta.status === "success";
}

// Uso con fetch
async function obtenerDatos<T>(url: string): Promise<T> {
  const raw: unknown = await fetch(url).then(r => r.json());
  const respuesta = raw as RespuestaAPI<T>;

  if (!esExito(respuesta)) {
    throw new Error(`Error ${respuesta.codigo}: ${respuesta.error}`);
  }

  return respuesta.data;
}
```

La unión discriminada por `status` permite a TypeScript saber exactamente
qué propiedades están disponibles en cada caso.

8. JSON ANIDADO Y TIPOS COMPLEJOS

```typescript
interface ApiResponse {
  page: number;
  perPage: number;
  total: number;
  data: {
    users: Usuario[];
    metadata: {
      lastUpdated: string;
      version: number;
    };
  };
}

// Validación recursiva
function esApiResponse(dato: unknown): dato is ApiResponse {
  if (!dato || typeof dato !== "object") return false;
  const obj = dato as Record<string, unknown>;

  return (
    typeof obj.page === "number" &&
    typeof obj.perPage === "number" &&
    typeof obj.total === "number" &&
    obj.data !== null && typeof obj.data === "object"
  );
}
```

9. TRANSFORMACIÓN DE JSON A OBJETOS TIPADOS
===============================================================================

A veces el JSON viene con formatos que no coinciden exactamente con
nuestras interfaces. Creamos funciones de transformación:

```typescript
interface UsuarioApp {
  id: number;
  nombreCompleto: string;
  fechaRegistro: Date;      // Date, no string
  activo: boolean;
}

// Datos crudos de la API (snake_case, fechas como string)
interface UsuarioAPI {
  id: number;
  full_name: string;
  registration_date: string; // ISO date
  is_active: number;         // 0 | 1
}

function transformarUsuario(api: UsuarioAPI): UsuarioApp {
  return {
    id: api.id,
    nombreCompleto: api.full_name,
    fechaRegistro: new Date(api.registration_date),
    activo: api.is_active === 1
  };
}

// Uso
const raw = JSON.parse(`{
  "id": 1,
  "full_name": "Ana López",
  "registration_date": "2025-01-15",
  "is_active": 1
}`) as UsuarioAPI;

const usuario = transformarUsuario(raw);
console.log(usuario.fechaRegistro.getFullYear()); // ✅ Date real
```

10. MAPEO DE ARRAYS JSON CON TIPOS
===============================================================================

```typescript
interface Post {
  userId: number;
  id: number;
  title: string;
  body: string;
}

// Fetch y transformación de array
async function obtenerPosts(): Promise<Post[]> {
  const respuesta = await fetch("https://jsonplaceholder.typicode.com/posts");
  const datos = await respuesta.json() as Post[];

  // Validación simple: asegurar que es un array
  if (!Array.isArray(datos)) {
    throw new Error("Se esperaba un array de posts");
  }

  return datos;
}

// Filtrado tipado
async function postsRecientes(): Promise<Post[]> {
  const posts = await obtenerPosts();
  return posts.filter(p => p.id > 50);
}
```

11. LOCAL STORAGE CON JSON TIPADO
===============================================================================

```typescript
class StorageTipado<T> {
  constructor(private key: string) {}

  guardar(datos: T): void {
    try {
      const json = JSON.stringify(datos);
      localStorage.setItem(this.key, json);
    } catch (error) {
      console.error("Error al guardar en localStorage:", error);
    }
  }

  obtener(): T | null {
    try {
      const raw = localStorage.getItem(this.key);
      if (!raw) return null;

      const datos: unknown = JSON.parse(raw);
      return datos as T;
    } catch {
      return null;
    }
  }

  eliminar(): void {
    localStorage.removeItem(this.key);
  }
}

// Uso
interface PreferenciasUsuario {
  tema: "claro" | "oscuro";
  fontSize: number;
  idioma: string;
}

const storagePrefs = new StorageTipado<PreferenciasUsuario>("preferencias");

storagePrefs.guardar({ tema: "oscuro", fontSize: 18, idioma: "es" });
const prefs = storagePrefs.obtener();
if (prefs) {
  console.log(`Tema: ${prefs.tema}`);
}
```

12. LIBRERÍAS DE VALIDACIÓN (OPCIONAL)
===============================================================================

Aunque este libro se centra en TypeScript puro, es bueno saber que existen
librerías como Zod, Yup o io-ts que validan JSON en runtime y derivan
tipos automáticamente.

```typescript
// Ejemplo conceptual con Zod (no incluido en el proyecto)
import { z } from "zod";

const UsuarioSchema = z.object({
  id: z.number(),
  nombre: z.string(),
  email: z.string().email(),
  activo: z.boolean()
});

// El tipo se infiere del schema
type UsuarioZod = z.infer<typeof UsuarioSchema>;

// Validación con parse
const resultado = UsuarioSchema.parse(JSON.parse(rawJson));
// resultado tiene tipo UsuarioZod
// Si falla, lanza ZodError con detalles
```

Pero para la mayoría de los casos, los type guards manuales son suficientes
y no añaden dependencias.

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- `JSON.parse` devuelve `any` — debemos tipar explícitamente.
- Parsear como `unknown` y validar es el enfoque más seguro.
- Los type guards (`dato is Tipo`) verifican tipos en runtime.
- Las assertion functions (`asserts dato is Tipo`) lanzan error si falla.
- `JSON.stringify` acepta un replacer para filtrar datos.
- Las uniones discriminadas modelan respuestas API variables.
- Las funciones de transformación convierten JSON crudo a objetos tipados.
- `StorageTipado<T>` encapsula localStorage con genéricos.
- Zod y otras librerías ofrecen validación + inferencia de tipos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué no basta con `as Tipo` después de JSON.parse?**

Porque `as Tipo` es una aserción de tipos, no una validación. Si el JSON
no coincide, tendrás un error en runtime. Los type guards protegen contra
eso.

❓ **¿Necesito una librería como Zod sí o sí?**

No. Para proyectos pequeños, los type guards manuales son suficientes.
Para proyectos grandes o críticos, Zod añade seguridad adicional.

❓ **¿Cómo valido arrays de objetos?**

Usa `Array.isArray(dato)` y luego `.every(item => esMiTipo(item))`.
El type guard se aplica elemento por elemento.

❓ **¿Qué hago con fechas en JSON?**

JSON no tiene tipo Date. Las fechas vienen como strings ISO. Parsea con
`new Date()` o librerías como date-fns después de validar.
