1. LA MAGIA DE async/await CON TIPOS
===============================================================================

Cuando una función lleva `async`, TypeScript automáticamente envuelve su
retorno en una `Promise<T>`.

```typescript
// El tipo de retorno se infiere como Promise<number>
async function suma(a: number, b: number) {
  return a + b;
}

// Explícitamente: Promise<string>
async function saludar(nombre: string): Promise<string> {
  return `Hola, ${nombre}`;
}

// Promise<void>
async function logger(mensaje: string): Promise<void> {
  console.log(mensaje);
}
```

Y cuando usamos `await`, TypeScript "desenvuelve" la promesa y nos da el
tipo interno:

```typescript
async function main(): Promise<void> {
  const resultado = await suma(3, 4);
  // resultado: number (no Promise<number>)

  const saludo = await saludar("Ana");
  // saludo: string (no Promise<string>)
}
```

Esta extracción de tipos es esencial: `await` convierte `Promise<T>` en `T`.

2. ERRORES COMUNES Y CÓMO EVITARLOS
===============================================================================

2.1. Olvidar el await

```typescript
async function obtenerDatos(): Promise<string> {
  return "datos";
}

async function main() {
  const datos = obtenerDatos(); // ❌ datos es Promise<string>, no string
  console.log(datos.length);    // ❌ Error: Promise no tiene .length

  const datosOk = await obtenerDatos(); // ✅ datosOk: string
  console.log(datosOk.length);          // ✅ funciona
}
```

2.2. Olvidar async en la función

```typescript
// ❌ Esto NO funciona
function malaFuncion() {
  const datos = await fetch("url"); // Error: await solo en async
  return datos.json();
}

// ✅ Correcto
async function buenaFuncion() {
  const datos = await fetch("url");
  return datos.json();
}
```

2.3. Devolver Promise<Promise<T>>

```typescript
async function obtenerNumero(): Promise<number> {
  return 42;
}

// ❌ Devolver una promesa dentro de async (redundante)
async function redundante(): Promise<number> {
  return Promise.resolve(42); // Esto devuelve Promise<number> envuelto en otra Promise
}

// ✅ Correcto — el async ya envuelve
async function correcta(): Promise<number> {
  return 42;
}
```

3. FLUJOS SECUENCIALES CON await
===============================================================================

```typescript
interface Usuario {
  id: number;
  name: string;
  email: string;
}

interface Publicacion {
  id: number;
  title: string;
  body: string;
}

async function obtenerPerfilCompleto(usuarioId: number): Promise<{
  usuario: Usuario;
  publicaciones: Publicacion[];
}> {
  // Secuencial: primero usuario, luego sus publicaciones
  const usuario = await fetchJSON<Usuario>(
    `https://jsonplaceholder.typicode.com/users/${usuarioId}`
  );

  const publicaciones = await fetchJSON<Publicacion[]>(
    `https://jsonplaceholder.typicode.com/users/${usuarioId}/posts`
  );

  return { usuario, publicaciones };
}
```

Cada `await` espera a que la promesa anterior se resuelva. Esto es
secuencial.

4. FLUJOS PARALELOS CON await + Promise.all
===============================================================================

```typescript
async function obtenerDashboard(usuarioId: number): Promise<{
  usuario: Usuario;
  publicaciones: Publicacion[];
  tareas: any[]; // simplificado
}> {
  // Paralelo: todas las peticiones al mismo tiempo
  const [usuario, publicaciones, tareas] = await Promise.all([
    fetchJSON<Usuario>(`https://jsonplaceholder.typicode.com/users/${usuarioId}`),
    fetchJSON<Publicacion[]>(`https://jsonplaceholder.typicode.com/users/${usuarioId}/posts`),
    fetchJSON<any[]>(`https://jsonplaceholder.typicode.com/users/${usuarioId}/todos`)
  ]);

  return { usuario, publicaciones, tareas };
}
```

Regla de oro: si las operaciones son independientes, hazlas en paralelo
con `Promise.all`. Si dependen una de otra, hazlas secuenciales.

5. MANEJO DE ERRORES CON TRY/CATCH
===============================================================================

```typescript
class ApiError extends Error {
  constructor(
    public status: number,
    message: string
  ) {
    super(message);
    this.name = "ApiError";
  }
}

async function obtenerUsuarioSeguro(id: number): Promise<Usuario | null> {
  try {
    const usuario = await fetchJSON<Usuario>(
      `https://jsonplaceholder.typicode.com/users/${id}`
    );
    return usuario;
  } catch (error) {
    if (error instanceof ApiError) {
      console.error(`Error API ${error.status}: ${error.message}`);
    } else if (error instanceof TypeError) {
      console.error("Error de red:", error.message);
    } else {
      console.error("Error desconocido:", error);
    }
    return null;
  }
}
```

El `try/catch` con `async/await` captura tanto errores de la promesa como
errores síncronos dentro de la función.

7. BUCLES CON ASYNC/AWAIT
===============================================================================

```typescript
interface Producto {
  id: number;
  nombre: string;
  precio: number;
}

// Secuencial — uno tras otro
async function procesarProductosSecuencial(productos: Producto[]): Promise<void> {
  for (const producto of productos) {
    await actualizarInventario(producto);
    console.log(`Producto ${producto.id} actualizado`);
  }
}

// Paralelo — todos a la vez
async function procesarProductosParalelo(productos: Producto[]): Promise<void> {
  const promesas = productos.map(p => actualizarInventario(p));
  await Promise.all(promesas);
  console.log("Todos los productos actualizados");
}

async function actualizarInventario(producto: Producto): Promise<void> {
  await fetch(`https://api.example.com/inventario/${producto.id}`, {
    method: "PUT",
    body: JSON.stringify(producto)
  });
}

// Paralelo con límite de concurrencia
async function procesarEnLotes<T>(
  items: T[],
  fn: (item: T) => Promise<void>,
  tamanoLote: number = 5
): Promise<void> {
  for (let i = 0; i < items.length; i += tamanoLote) {
    const lote = items.slice(i, i + tamanoLote);
    await Promise.all(lote.map(fn));
    console.log(`Lote ${i / tamanoLote + 1} completado`);
  }
}
```

8. PATRÓN RETRY CON ASYNC/AWAIT
===============================================================================

```typescript
async function fetchConReintento<T>(
  url: string,
  opciones?: {
    maxIntentos?: number;
    delayMs?: number;
    factorBackoff?: number;
  }
): Promise<T> {
  const {
    maxIntentos = 3,
    delayMs = 1000,
    factorBackoff = 2
  } = opciones ?? {};

  let ultimoError: unknown;

  for (let intento = 1; intento <= maxIntentos; intento++) {
    try {
      console.log(`Intento ${intento}/${maxIntentos}: ${url}`);
      return await fetchJSON<T>(url);
    } catch (error) {
      ultimoError = error;

      if (intento < maxIntentos) {
        const espera = delayMs * Math.pow(factorBackoff, intento - 1);
        console.log(`Fallo. Reintentando en ${espera}ms...`);
        await new Promise(resolve => setTimeout(resolve, espera));
      }
    }
  }

  throw new Error(`Fallaron todos los intentos: ${ultimoError}`);
}

// Uso
const datos = await fetchConReintento<Usuario>(
  "https://api.example.com/users/1",
  { maxIntentos: 5, delayMs: 500 }
);

console.log(datos.name);
```

9. ASYNC/AWAIT CON EVENTOS (PATRÓN)
===============================================================================

```typescript
// Convertir un evento en una promesa
function esperarEvento<T extends Event>(
  elemento: HTMLElement,
  nombreEvento: string
): Promise<T> {
  return new Promise(resolve => {
    elemento.addEventListener(nombreEvento, (event: Event) => {
      resolve(event as T);
    }, { once: true });
  });
}

async function esperarClic(): Promise<void> {
  const boton = document.querySelector("#miBoton") as HTMLButtonElement;
  console.log("Esperando clic...");

  const evento = await esperarEvento<MouseEvent>(boton, "click");
  console.log(`Clic en (${evento.clientX}, ${evento.clientY})`);
}
```

10. TIPADO DE FUNCIONES QUE RECIBEN CALLBACKS ASÍNCRONOS
===============================================================================

```typescript
// Función que ejecuta un callback asíncrono para cada item
async function mapAsync<T, U>(
  items: T[],
  callback: (item: T, index: number) => Promise<U>
): Promise<U[]> {
  const resultados: U[] = [];

  for (let i = 0; i < items.length; i++) {
    const resultado = await callback(items[i], i);
    resultados.push(resultado);
  }

  return resultados;
}

// Uso
const numeros = [1, 2, 3, 4, 5];

const cuadrados = await mapAsync(numeros, async (n) => {
  await esperar(100); // simula operación asíncrona
  return n * n;
});

console.log(cuadrados); // [1, 4, 9, 16, 25]
// Tipo: number[]
```

11. PROMESAS COLGADAS (DANGLING PROMISES)
===============================================================================

Un error común es no esperar una promesa dentro de una función async:

```typescript
// ❌ MAL: promesa colgada
async function guardarUsuario(datos: Usuario): Promise<void> {
  fetch("https://api.example.com/users", {
    method: "POST",
    body: JSON.stringify(datos)
  });
  // La promesa de fetch no se espera — el error se traga
}

// ✅ BIEN: esperar la promesa
async function guardarUsuarioBien(datos: Usuario): Promise<void> {
  await fetch("https://api.example.com/users", {
    method: "POST",
    body: JSON.stringify(datos)
  });
}
```

TypeScript no marca las promesas colgadas por defecto, pero ESLint con
la regla `@typescript-eslint/no-floating-promises` sí lo hace.

12. ASYNC/AWAIT EN CLASES
===============================================================================

```typescript
class RepositorioUsuarios {
  private baseUrl = "https://jsonplaceholder.typicode.com";

  async obtenerPorId(id: number): Promise<Usuario> {
    const url = `${this.baseUrl}/users/${id}`;
    return fetchJSON<Usuario>(url);
  }

  async obtenerTodos(): Promise<Usuario[]> {
    const url = `${this.baseUrl}/users`;
    return fetchJSON<Usuario[]>(url);
  }

  async crear(datos: Omit<Usuario, "id">): Promise<Usuario> {
    const url = `${this.baseUrl}/users`;
    return fetchJSON<Usuario>(url, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(datos)
    });
  }
}

// Uso
const repo = new RepositorioUsuarios();
const usuario = await repo.obtenerPorId(1);
console.log(usuario.name);
```

13. EJEMPLO COMPLETO: PROCESAMIENTO DE PEDIDOS
===============================================================================

```typescript
interface Pedido {
  id: string;
  producto: string;
  cantidad: number;
  precioUnitario: number;
}

interface PedidoProcesado extends Pedido {
  total: number;
  impuesto: number;
  fechaProcesamiento: string;
}

async function procesarPedido(pedido: Pedido): Promise<PedidoProcesado> {
  // Paso 1: validar (simulado)
  await esperar(100);
  if (pedido.cantidad <= 0) {
    throw new Error("Cantidad inválida");
  }

  // Paso 2: calcular total
  const total = pedido.cantidad * pedido.precioUnitario;
  const impuesto = total * 0.16;

  // Paso 3: guardar en base de datos (simulado)
  await esperar(200);

  return {
    ...pedido,
    total,
    impuesto,
    fechaProcesamiento: new Date().toISOString()
  };
}

async function procesarPedidos(pedidos: Pedido[]): Promise<PedidoProcesado[]> {
  const resultados: PedidoProcesado[] = [];

  for (const pedido of pedidos) {
    try {
      const procesado = await procesarPedido(pedido);
      resultados.push(procesado);
      console.log(`Pedido ${pedido.id} procesado correctamente`);
    } catch (error) {
      console.error(`Error procesando pedido ${pedido.id}:`, error);
    }
  }

  return resultados;
}

// Uso
const pedidos: Pedido[] = [
  { id: "P001", producto: "Laptop", cantidad: 1, precioUnitario: 15000 },
  { id: "P002", producto: "Mouse", cantidad: 2, precioUnitario: 500 },
];

const procesados = await procesarPedidos(pedidos);
console.log(`Procesados ${procesados.length} pedidos`);
```

RESUMEN DEL CAPÍTULO
===============================================================================

- `async` convierte el retorno de la función en `Promise<T>` automáticamente.
- `await` extrae el tipo `T` de una `Promise<T>`.
- El try/catch captura errores de promesas y errores síncronos.
- Operaciones independientes → paralelo con `Promise.all`.
- Operaciones dependientes → secuencial con `await`.
- Los bucles `for` funcionan naturalmente con `await`.
- Evita promesas colgadas: toda promesa debe esperarse o manejarse.
- El patrón retry con async/await es limpio y legible.
- Las clases pueden tener métodos async perfectamente tipados.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo usar await fuera de una función async?**

No. `await` solo es válido dentro de funciones declaradas con `async`.
En módulos ES modernos existe "top-level await" (await en el módulo sin
envolver en async function).

❓ **¿Qué es mejor, .then() o async/await?**

Async/await es más legible para flujos complejos. .then() es útil para
transformaciones rápidas. La decisión es más de estilo que técnica.

❓ **¿Cómo manejo múltiples errores en Promise.all?**

Envuelve cada promesa individual en un try/catch, o usa
`Promise.allSettled` para obtener el estado de cada una.

❓ **¿Por qué mi función async devuelve Promise<void> aunque no tenga return?**

Porque todas las funciones async devuelven una promesa. Si no hay return,
la promesa resuelve con `undefined` (que es `void`).
