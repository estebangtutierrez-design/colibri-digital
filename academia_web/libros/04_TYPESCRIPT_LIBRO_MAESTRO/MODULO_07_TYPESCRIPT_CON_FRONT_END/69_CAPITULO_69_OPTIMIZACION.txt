1. TREE SHAKING CON TYPESCRIPT
===============================================================================

Tree shaking es la eliminación de código muerto (no usado) durante el
empaquetado. TypeScript y los bundlers modernos (Webpack, Vite, esbuild)
lo soportan nativamente.

```typescript
// utils.ts
export function suma(a: number, b: number): number {
  return a + b;
}

export function resta(a: number, b: number): number {
  return a - b;
}

export function multiplicacion(a: number, b: number): number {
  return a * b;
}

// app.ts — solo importamos suma
import { suma } from "./utils";

console.log(suma(2, 3));
// Tree shaking eliminará 'resta' y 'multiplicacion' del bundle final
```

Para que el tree shaking funcione:

1. Usa módulos ES (`import`/`export`), no CommonJS
2. Evita efectos secundarios en los módulos
3. Marca el package.json con `"sideEffects": false`

```json
// package.json
{
  "sideEffects": false
}
```

Si algún archivo tiene efectos secundarios (ej. estilos), especifícalos:

```json
{
  "sideEffects": ["*.css", "*.scss"]
}
```

2. CODE SPLITTING CON IMPORTACIONES DINÁMICAS
===============================================================================

El code splitting divide el bundle en trozos más pequeños que se cargan
bajo demanda.

```typescript
// En lugar de importación estática
// import { PanelDashboard } from "./PanelDashboard";

// Importación dinámica — se carga solo cuando se necesita
async function cargarPanel(): Promise<void> {
  const { PanelDashboard } = await import("./PanelDashboard");
  const panel = new PanelDashboard();
  panel.montar("#contenedor");
}

// Con tipos explícitos
type ModuloGrafico = typeof import("./graficos");
async function cargarGraficos(): Promise<ModuloGrafico> {
  return import("./graficos");
}
```

Las importaciones dinámicas devuelven `Promise<typeof módulo>`. TypeScript
infiere los tipos correctamente.

```typescript
// lazy.ts
interface ModuloAdmin {
  iniciar(): void;
  cargarUsuarios(): Promise<Usuario[]>;
}

async function cargarAdmin(): Promise<void> {
  try {
    const modulo: ModuloAdmin = await import("./admin");
    await modulo.cargarUsuarios();
    modulo.iniciar();
  } catch (error) {
    console.error("Error al cargar módulo admin:", error);
  }
}
```

3. LAZY LOADING DE COMPONENTES
===============================================================================

```typescript
// En un patrón de fábrica
type ConstructorComponente = new (contenedor: HTMLElement) => ComponenteBase;

interface ComponenteBase {
  montar(): void;
  desmontar(): void;
}

class LazyCargador {
  private cache = new Map<string, ConstructorComponente>();

  async obtenerComponente(ruta: string): Promise<ConstructorComponente> {
    if (this.cache.has(ruta)) {
      return this.cache.get(ruta)!;
    }

    const modulo = await import(/* webpackChunkName: "componente" */ `./componentes/${ruta}`);
    const Componente = Object.values(modulo)[0] as ConstructorComponente;

    this.cache.set(ruta, Componente);
    return Componente;
  }
}

// Uso
const cargador = new LazyCargador();
const contenedor = document.getElementById("app") as HTMLDivElement;

const Componente = await cargador.obtenerComponente("PanelUsuario");
const instancia = new Componente(contenedor);
instancia.montar();
```

4. MEMORIZACIÓN CON TIPOS
===============================================================================

La memorización evita recalcular valores que ya se calcularon antes.

```typescript
// Memoizador genérico tipado
function memoizar<T extends (...args: any[]) => any>(
  fn: T
): (...args: Parameters<T>) => ReturnType<T> {
  const cache = new Map<string, ReturnType<T>>();

  return (...args: Parameters<T>): ReturnType<T> => {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      console.log("Cache hit:", key);
      return cache.get(key)!;
    }

    console.log("Cache miss:", key);
    const resultado = fn(...args);
    cache.set(key, resultado);
    return resultado;
  };
}

// Uso
const calcularPrecioConImpuesto = memoizar(
  (precio: number, tasa: number): number => {
    // Simulación de cálculo costoso
    console.log("Calculando...");
    return precio + precio * tasa;
  }
);

console.log(calcularPrecioConImpuesto(100, 0.16)); // Calculando... 116
console.log(calcularPrecioConImpuesto(100, 0.16)); // Cache hit! 116
console.log(calcularPrecioConImpuesto(200, 0.16)); // Calculando... 232
```

Con clave personalizada:

```typescript
function memoizarConClave<T extends (...args: any[]) => any>(
  fn: T,
  obtenerClave: (...args: Parameters<T>) => string = (...args) => JSON.stringify(args)
): (...args: Parameters<T>) => ReturnType<T> {
  const cache = new Map<string, ReturnType<T>>();
  return (...args) => {
    const key = obtenerClave(...args);
    if (cache.has(key)) return cache.get(key)!;
    const resultado = fn(...args);
    cache.set(key, resultado);
    return resultado;
  };
}
```

5.2. Eliminar console.log en producción

```typescript
// utils/logger.ts
const LOG_LEVELS = ["debug", "info", "warn", "error"] as const;
type LogLevel = (typeof LOG_LEVELS)[number];

function crearLogger(nivelMinimo: LogLevel = "warn") {
  const niveles = { debug: 0, info: 1, warn: 2, error: 3 };

  return {
    debug: (...args: unknown[]) => niveles[nivelMinimo] <= 0 && console.debug(...args),
    info: (...args: unknown[]) => niveles[nivelMinimo] <= 1 && console.info(...args),
    warn: (...args: unknown[]) => niveles[nivelMinimo] <= 2 && console.warn(...args),
    error: (...args: unknown[]) => console.error(...args),
  };
}

export const logger = crearLogger(
  process.env.NODE_ENV === "production" ? "warn" : "debug"
);
```

6. VIRTUAL SCROLLING CON TIPOS
===============================================================================

Para listas grandes, renderiza solo los elementos visibles:

```typescript
interface VirtualItem<T> {
  index: number;
  data: T;
  style: { position: string; top: number; height: number };
}

class VirtualScroller<T> {
  private container: HTMLElement;
  private items: T[] = [];
  private itemHeight: number;
  private visibleItems: VirtualItem<T>[] = [];

  constructor(
    selector: string,
    itemHeight: number = 50
  ) {
    this.container = document.querySelector(selector) as HTMLElement;
    this.itemHeight = itemHeight;
    this.container.style.position = "relative";
    this.container.style.overflowY = "auto";

    this.container.addEventListener("scroll", () => this.render());
  }

  setItems(items: T[]): void {
    this.items = items;
    this.render();
  }

  private render(): void {
    const scrollTop = this.container.scrollTop;
    const viewportHeight = this.container.clientHeight;

    const startIndex = Math.floor(scrollTop / this.itemHeight);
    const endIndex = Math.min(
      startIndex + Math.ceil(viewportHeight / this.itemHeight) + 1,
      this.items.length
    );

    this.visibleItems = [];
    for (let i = startIndex; i < endIndex; i++) {
      this.visibleItems.push({
        index: i,
        data: this.items[i],
        style: {
          position: "absolute",
          top: i * this.itemHeight,
          height: this.itemHeight
        }
      });
    }

    this.actualizarDOM();
  }

  private actualizarDOM(): void {
    this.container.innerHTML = this.visibleItems
      .map(item => `<div style="position:absolute;top:${item.style.top}px;height:${item.style.height}px">
        ${String(item.data)}
      </div>`)
      .join("");
  }
}

// Uso
interface UsuarioItem {
  id: number;
  nombre: string;
}

const scroller = new VirtualScroller<UsuarioItem>("#lista-usuarios", 60);
scroller.setItems(
  Array.from({ length: 10000 }, (_, i) => ({
    id: i + 1,
    nombre: `Usuario ${i + 1}`
  }))
);
```

7.2. Debouncing con tipos

```typescript
function debounce<T extends (...args: any[]) => void>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timeoutId: ReturnType<typeof setTimeout>;

  return (...args: Parameters<T>): void => {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => fn(...args), delay);
  };
}

// Uso
const buscar = debounce(
  (query: string) => {
    console.log("Buscando:", query);
    // fetch a la API
  },
  300
);

const input = document.querySelector("#busqueda") as HTMLInputElement;
input.addEventListener("input", (e: Event) => {
  buscar((e.target as HTMLInputElement).value);
});
```

7.3. Throttling

```typescript
function throttle<T extends (...args: any[]) => void>(
  fn: T,
  limit: number
): (...args: Parameters<T>) => void {
  let enEspera = false;

  return (...args: Parameters<T>): void => {
    if (!enEspera) {
      fn(...args);
      enEspera = true;
      setTimeout(() => { enEspera = false; }, limit);
    }
  };
}

const manejarScroll = throttle(() => {
  console.log("Scroll position:", window.scrollY);
}, 100);

window.addEventListener("scroll", manejarScroll);
```

Los tipos mismos ayudan a escribir código más eficiente:

```typescript
// Tipos estrictos evitan comparaciones innecesarias
type Status = "idle" | "loading" | "success" | "error";

function procesarStatus(status: Status): string {
  switch (status) {
    case "idle": return "Inactivo";
    case "loading": return "Cargando...";
    case "success": return "Éxito";
    case "error": return "Error";
    // TypeScript avisa si falta un caso
  }
}

// Tipos de unión excluyente evitan validaciones extra
type Forma =
  | { tipo: "circulo"; radio: number }
  | { tipo: "rectangulo"; ancho: number; alto: number };

function area(forma: Forma): number {
  switch (forma.tipo) {
    case "circulo":
      return Math.PI * forma.radio ** 2; // solo radio disponible
    case "rectangulo":
      return forma.ancho * forma.alto; // solo ancho y alto disponibles
  }
}
```

```typescript
async function cargarEstilos(ruta: string): Promise<void> {
  const link = document.createElement("link");
  link.rel = "stylesheet";
  link.href = ruta;

  return new Promise((resolve, reject) => {
    link.onload = () => resolve();
    link.onerror = () => reject(new Error(`Error al cargar ${ruta}`));
    document.head.appendChild(link);
  });
}

// Uso para paneles que se muestran condicionalmente
async function mostrarPanelAdmin(): Promise<void> {
  await cargarEstilos("/estilos/admin.css");
  const { PanelAdmin } = await import("./PanelAdmin");
  new PanelAdmin().montar();
}
```

```typescript
// app.ts — Aplicación optimizada con lazy loading + code splitting
type Pagina = "dashboard" | "usuarios" | "reportes" | "configuracion";

interface ModuloPagina {
  default: new (contenedor: HTMLElement) => { montar: () => void; desmontar: () => void };
}

class App {
  private contenedor: HTMLElement;
  private moduloActual: { desmontar: () => void } | null = null;

  constructor() {
    this.contenedor = document.getElementById("app") as HTMLElement;
    this.inicializarNavegacion();
  }

  private inicializarNavegacion(): void {
    document.querySelectorAll("[data-pagina]").forEach(enlace => {
      enlace.addEventListener("click", async (e: Event) => {
        e.preventDefault();
        const pagina = (e.currentTarget as HTMLElement).dataset.pagina as Pagina;
        await this.navegar(pagina);
      });
    });
  }

  async navegar(pagina: Pagina): Promise<void> {
    this.moduloActual?.desmontar();
    this.contenedor.innerHTML = "<div class='cargando'>Cargando...</div>";

    try {
      const { default: Modulo } = await import(
        /* webpackChunkName: "pagina-[request]" */
        `./paginas/${pagina}`
      ) as ModuloPagina;

      this.moduloActual = new Modulo(this.contenedor);
      this.moduloActual.montar();
    } catch (error) {
      this.contenedor.innerHTML = `<div class='error'>Error al cargar ${pagina}</div>`;
    }
  }
}

const app = new App();
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿TypeScript hace más lento mi bundle?**

No. TypeScript solo es un transpilador. El bundle final es JavaScript
puro. De hecho, TypeScript puede ayudar a reducir el bundle eliminando
ramas muertas con tipos de unión.

❓ **¿Qué es mejor para bundle, Webpack o Vite?**

Vite (con esbuild + Rollup) es significativamente más rápido en desarrollo
y produce bundles competitivos en producción. Es la opción recomendada hoy.

❓ **¿Debo optimizar desde el principio o al final?**

No optimices prematuramente. Escribe código claro primero, luego mide y
optimiza los cuellos de botella reales. Pero sí aplica buenas prácticas
desde el inicio (importaciones selectivas, code splitting básico).

❓ **¿Cómo sé qué parte de mi bundle es más pesada?**

Usa herramientas de análisis: `vite-bundle-analyzer`,
`webpack-bundle-analyzer`, o `source-map-explorer`.
