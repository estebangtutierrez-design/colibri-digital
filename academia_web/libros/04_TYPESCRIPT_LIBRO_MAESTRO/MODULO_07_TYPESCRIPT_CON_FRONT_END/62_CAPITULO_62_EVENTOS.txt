===============================================================================
1. LA JERARQUÍA DE EVENTOS EN TYPESCRIPT
===============================================================================

TypeScript define una jerarquía completa de tipos de eventos. Todos
extienden de la clase base `Event`.

```
Event
├── UIEvent
│   ├── MouseEvent
│   │   ├── WheelEvent
│   │   └── DragEvent
│   ├── KeyboardEvent
│   ├── FocusEvent
│   ├── TouchEvent
│   ├── InputEvent
│   └── PointerEvent
├── ClipboardEvent
├── FormEvent (genérico en React, en DOM es Event)
├── ProgressEvent<T>
├── StorageEvent
└── ... muchos más
```

Cada nivel añade propiedades específicas:

```typescript
// Event — propiedades base
event.type          // string
event.target        // EventTarget | null
event.currentTarget // EventTarget | null
event.preventDefault()  // void
event.stopPropagation() // void

// MouseEvent (hereda de UIEvent, que hereda de Event)
event.clientX       // number
event.clientY       // number
event.button        // number (0=izquierdo, 1=medio, 2=derecho)
event.altKey        // boolean
event.ctrlKey       // boolean

// KeyboardEvent
event.key           // string
event.code          // string
event.repeat        // boolean

// FocusEvent
event.relatedTarget // EventTarget | null
```

===============================================================================
2. addEventListener CON TIPOS INFERIDOS
===============================================================================

Cuando usamos `addEventListener`, TypeScript infiere automáticamente el tipo
de evento según el nombre del evento y el elemento.

```typescript
const boton = document.querySelector("button") as HTMLButtonElement;

// TypeScript sabe que 'click' → MouseEvent
boton.addEventListener("click", (event) => {
  // event: MouseEvent
  console.log(event.clientX, event.clientY);
});

// 'keydown' → KeyboardEvent
const input = document.querySelector("input") as HTMLInputElement;
input.addEventListener("keydown", (event) => {
  // event: KeyboardEvent
  if (event.key === "Enter") {
    console.log("¡Enter presionado!");
  }
});

// 'focus' → FocusEvent
input.addEventListener("focus", (event) => {
  // event: FocusEvent
  console.log("Elemento enfocado:", event.target);
});
```

===============================================================================
3. EL TIPO GENERIC EN addEventListener
===============================================================================

El método `addEventListener` tiene esta firma genérica:

```typescript
// addEventListener<K extends keyof HTMLElementEventMap>(
//   type: K,
//   listener: (this: HTMLElement, ev: HTMLElementEventMap[K]) => void
// ): void
```

`HTMLElementEventMap` es un tipo que mapea nombres de eventos a sus tipos:

```typescript
interface HTMLElementEventMap {
  "click": MouseEvent;
  "dblclick": MouseEvent;
  "keydown": KeyboardEvent;
  "keyup": KeyboardEvent;
  "focus": FocusEvent;
  "blur": FocusEvent;
  "submit": SubmitEvent;
  "input": InputEvent;
  "change": Event;
  "mouseenter": MouseEvent;
  "mouseleave": MouseEvent;
  "scroll": Event;
  "resize": UIEvent;
  // ... muchos más
}
```

Gracias a este mapa, no necesitas declarar el tipo del evento manualmente.

4. MANEJADORES CON TIPO EXPLÍCITO
===============================================================================

A veces necesitas escribir la función por separado. Entonces debes declarar
el tipo explícitamente.

```typescript
function manejadorClick(event: MouseEvent): void {
  console.log(`Clic en (${event.clientX}, ${event.clientY})`);
}

document.querySelector("button")?.addEventListener("click", manejadorClick);

// También con arrow function guardada
const manejadorTecla = (event: KeyboardEvent): void => {
  if (event.key === "Escape") {
    cerrarModal();
  }
};

document.addEventListener("keydown", manejadorTecla);
```

===============================================================================
5. PREVENIR COMPORTAMIENTO POR DEFECTO
===============================================================================

Los eventos tienen `preventDefault()` y `stopPropagation()`. Ambos están en
la interfaz base `Event`.

```typescript
const formulario = document.querySelector("form") as HTMLFormElement;

formulario.addEventListener("submit", (event: SubmitEvent) => {
  event.preventDefault(); // ✅ evitar recarga de página

  const formData = new FormData(formulario);
  console.log("Datos del formulario:", Object.fromEntries(formData));
});

// En un enlace
const enlace = document.querySelector("a") as HTMLAnchorElement;
enlace.addEventListener("click", (event: MouseEvent) => {
  event.preventDefault();
  console.log("Navegación bloqueada");
});
```

6. EVENTOS DE TECLADO: KeyboardEvent
===============================================================================

```typescript
const campoTexto = document.querySelector("#inputBusqueda") as HTMLInputElement;

campoTexto.addEventListener("keydown", (event: KeyboardEvent) => {
  // Propiedades útiles
  console.log("Tecla:", event.key);       // "a", "Enter", "ArrowUp"
  console.log("Código:", event.code);     // "KeyA", "Enter", "ArrowUp"
  console.log("Alt:", event.altKey);
  console.log("Ctrl:", event.ctrlKey);
  console.log("Shift:", event.shiftKey);

  // Filtrar solo teclas alfanuméricas
  if (event.key.length === 1) {
    console.log("Carácter escrito:", event.key);
  }

  // Atajo: Ctrl + Enter
  if (event.ctrlKey && event.key === "Enter") {
    enviarFormulario();
  }
});
```

7. EVENTOS DE MOUSE: MouseEvent
===============================================================================

```typescript
const area = document.querySelector(".drawing-area") as HTMLElement;

area.addEventListener("mousedown", (event: MouseEvent) => {
  console.log(`Ratón presionado en (${event.clientX}, ${event.clientY})`);
  console.log(`Botón: ${event.button}`);  // 0=izq, 1=medio, 2=der
});

area.addEventListener("mousemove", (event: MouseEvent) => {
  // Solo actualizar cada 50ms para rendimiento
  console.log(`Ratón moviéndose: (${event.clientX}, ${event.clientY})`);
});

area.addEventListener("mouseup", (event: MouseEvent) => {
  console.log("Ratón liberado");
});

area.addEventListener("wheel", (event: WheelEvent) => {
  console.log(`Scroll: ${event.deltaY}`);
});
```

===============================================================================
8. EVENTOS DE FORMULARIO
===============================================================================

```typescript
const form = document.querySelector("form") as HTMLFormElement;
const emailInput = document.querySelector("#email") as HTMLInputElement;

// 'input' — cada vez que el valor cambia
emailInput.addEventListener("input", (event: InputEvent) => {
  const input = event.target as HTMLInputElement;
  console.log("Valor actual:", input.value);
});

// 'change' — cuando pierde el foco y el valor cambió
emailInput.addEventListener("change", (event: Event) => {
  const input = event.target as HTMLInputElement;
  if (!input.validity.valid) {
    input.style.borderColor = "red";
  }
});

// 'focus' y 'blur'
emailInput.addEventListener("focus", (event: FocusEvent) => {
  (event.target as HTMLInputElement).style.borderColor = "blue";
});

emailInput.addEventListener("blur", (event: FocusEvent) => {
  (event.target as HTMLInputElement).style.borderColor = "";
});
```

9. DELEGACIÓN DE EVENTOS CON TIPOS
===============================================================================

La delegación de eventos aprovecha la propagación (burbujeo) para manejar
eventos de múltiples hijos desde un solo padre.

```typescript
interface Tarea {
  id: number;
  texto: string;
  completada: boolean;
}

const lista = document.querySelector("#lista-tareas") as HTMLUListElement;

lista.addEventListener("click", (event: MouseEvent) => {
  const target = event.target as HTMLElement;

  // Delegación: detectamos si el clic fue en un <li> o en un botón dentro
  if (target.tagName === "LI") {
    console.log("Tarea seleccionada:", target.textContent);
  }

  if (target.classList.contains("btn-eliminar")) {
    const li = target.closest("li") as HTMLLIElement;
    li.remove();
    console.log("Tarea eliminada");
  }
});

// También con data attributes
lista.addEventListener("click", (event: MouseEvent) => {
  const boton = (event.target as HTMLElement).closest("[data-accion]");
  if (!boton) return;

  const accion = boton.getAttribute("data-accion");
  const id = Number(boton.getAttribute("data-id"));

  switch (accion) {
    case "completar":
      console.log(`Completar tarea ${id}`);
      break;
    case "eliminar":
      console.log(`Eliminar tarea ${id}`);
      break;
  }
});
```

10. REMOVER EVENTOS Y ABORTAR CONTRALORES
===============================================================================

Para remover un evento, necesitamos la referencia a la función.

```typescript
function manejadorScroll(this: HTMLElement, event: Event): void {
  console.log("Scrolling...", (event.target as HTMLElement).scrollTop);
}

const contenedor = document.querySelector(".contenedor") as HTMLElement;
contenedor.addEventListener("scroll", manejadorScroll);

// Remover después de 5 segundos
setTimeout(() => {
  contenedor.removeEventListener("scroll", manejadorScroll);
  console.log("Evento de scroll removido");
}, 5000);
```

Usando `AbortController` (moderno):

```typescript
const controlador = new AbortController();
const { signal } = controlador;

document.querySelector("button")?.addEventListener("click", (event: MouseEvent) => {
  console.log("Click registrado");
}, { signal });

// Cancelar todos los eventos asociados a esta signal
controlador.abort();
```

11. EVENTOS PERSONALIZADOS (CustomEvent)
===============================================================================

TypeScript también tipa eventos creados por nosotros.

```typescript
interface DatosUsuario {
  id: number;
  nombre: string;
}

// Crear evento personalizado
const eventoUsuario = new CustomEvent<DatosUsuario>("usuario-login", {
  detail: { id: 1, nombre: "Ana" }
});

// Escuchar
document.addEventListener("usuario-login", (event: CustomEvent<DatosUsuario>) => {
  const usuario = event.detail;  // Tipo: DatosUsuario
  console.log(`Bienvenido ${usuario.nombre}`);
});

// Disparar
document.dispatchEvent(eventoUsuario);
```

Podemos extender la interfaz `HTMLElementEventMap` para incluir nuestros
eventos personalizados:

```typescript
declare global {
  interface HTMLElementEventMap {
    "usuario-login": CustomEvent<DatosUsuario>;
  }
}

// Ahora addEventListener infiere el tipo correcto
document.addEventListener("usuario-login", (event) => {
  // event: CustomEvent<DatosUsuario>
  console.log(event.detail.nombre);
});
```

13. EVENTOS EN ELEMENTOS ESPECÍFICOS
===============================================================================

Cada tipo de elemento tiene su propio mapa de eventos:

const input = document.querySelector("input") as HTMLInputElement;

  const submitter = event.submitter;  // HTMLElement | null

// HTMLMediaElement tiene eventos multimedia

14. EVENTOS TIPADOS EN CLASES
===============================================================================

```typescript
class ComponenteUI {
  private elemento: HTMLElement;

  constructor(selector: string) {
    this.elemento = document.querySelector(selector) as HTMLElement;
    this.inicializarEventos();
  }

  private inicializarEventos(): void {
    this.elemento.addEventListener("click", this.manejarClick);
    this.elemento.addEventListener("mouseenter", this.manejarMouseEnter);
  }

  private manejarClick = (event: MouseEvent): void => {
    console.log(`Componente clickeado en (${event.clientX}, ${event.clientY})`);
  };

  private manejarMouseEnter = (event: MouseEvent): void => {
    this.elemento.style.opacity = "0.8";
  };

  public destruir(): void {
    this.elemento.removeEventListener("click", this.manejarClick);
    this.elemento.removeEventListener("mouseenter", this.manejarMouseEnter);
  }
}

const header = new ComponenteUI("#header");
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Los eventos en TypeScript siguen una jerarquía: Event → UIEvent → MouseEvent, etc.
- `addEventListener` infiere el tipo según el nombre del evento gracias a
  `HTMLElementEventMap`.
- Puedes escribir manejadores separados declarando el tipo explícito.
- `preventDefault()` y `stopPropagation()` son métodos tipados de Event.
- `MouseEvent` tiene `clientX`, `clientY`, `button`, etc.
- `KeyboardEvent` tiene `key`, `code`, `ctrlKey`, etc.
- `FocusEvent` tiene `relatedTarget`.
- `CustomEvent<T>` permite eventos personalizados con tipo en `detail`.
- La delegación de eventos requiere castear `event.target` al tipo esperado.
- `AbortController` permite cancelar eventos limpiamente.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué a veces event.target es EventTarget | null y no el elemento?**

Porque `event.target` puede ser cualquier nodo (texto, comentario, etc.).
Casi siempre debes castearlo: `event.target as HTMLElement`.

❓ **¿Cómo sé qué tipo de evento usar para cada nombre de evento?**

TypeScript lo infiere automáticamente. Si usas `"click"`, será `MouseEvent`.
Si usas `"keydown"`, será `KeyboardEvent`. Confía en `HTMLElementEventMap`.

❓ **¿Qué hago con eventos que no están en el mapa?**

Puedes extender `HTMLElementEventMap` usando `declare global`. O usar el
tipo genérico `Event` y castear dentro del manejador.

❓ **¿removeEventListener necesita la misma referencia de función?**

Sí, exactamente la misma. Por eso es útil guardar la función en una
variable o usar métodos de clase con arrow function.
