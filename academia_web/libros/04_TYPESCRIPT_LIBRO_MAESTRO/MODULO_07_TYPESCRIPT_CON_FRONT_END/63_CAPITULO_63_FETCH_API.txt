1. LA FIRMA DE FETCH
===============================================================================

La función `fetch` tiene esta firma en TypeScript:

```typescript
declare function fetch(
  input: RequestInfo | URL,
  init?: RequestInit
): Promise<Response>;
```

- `RequestInfo` es `string | Request`
- `RequestInit` es un objeto con opciones como `method`, `headers`, `body`
- La respuesta es `Promise<Response>`

El tipo `Response` tiene métodos importantes:

```typescript
interface Response {
  ok: boolean;           // status 200-299
  status: number;        // código HTTP
  statusText: string;    // mensaje
  headers: Headers;      // cabeceras
  url: string;           // URL final

  // Métodos para leer el cuerpo
  json(): Promise<any>;          // ❌ any!
  text(): Promise<string>;
  blob(): Promise<Blob>;
  formData(): Promise<FormData>;
  arrayBuffer(): Promise<ArrayBuffer>;
}
```

El problema: `response.json()` devuelve `Promise<any>`. Aquí entra
TypeScript a salvar el día.

2. PETICIÓN GET BÁSICA CON TIPO
===============================================================================

```typescript
interface Usuario {
  id: number;
  name: string;
  email: string;
}

async function obtenerUsuario(id: number): Promise<Usuario> {
  const respuesta = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);

  if (!respuesta.ok) {
    throw new Error(`Error HTTP: ${respuesta.status} ${respuesta.statusText}`);
  }

  const usuario: Usuario = await respuesta.json();
  return usuario;
}

// Uso
obtenerUsuario(1).then(usuario => {
  console.log(usuario.name);  // ✅ string
  console.log(usuario.email); // ✅ string
});
```

El truco está en declarar `const usuario: Usuario = await respuesta.json()`.
Le decimos a TypeScript: "Confía en mí, esta respuesta tiene esta forma".

3. TIPADO GENÉRICO PARA FETCH
===============================================================================

Podemos crear una función genérica que tipifique automáticamente la respuesta:

```typescript
async function fetchJSON<T>(url: string, options?: RequestInit): Promise<T> {
  const respuesta = await fetch(url, options);

  if (!respuesta.ok) {
    throw new Error(`Error HTTP: ${respuesta.status} ${respuesta.statusText}`);
  }

  const datos: T = await respuesta.json();
  return datos;
}

// Uso con inferencia de tipo
interface Producto {
  id: number;
  title: string;
  price: number;
}

const producto = await fetchJSON<Producto>("https://api.example.com/products/1");
console.log(producto.title); // ✅ string
console.log(producto.price); // ✅ number
```

El genérico `<T>` se convierte en el tipo de retorno. El compilador
verificará que uses las propiedades correctas.

4. REQUESTINIT: CONFIGURACIÓN TIPADA
===============================================================================

El objeto `RequestInit` tiene todas las opciones de la petición:

```typescript
interface RequestInit {
  method?: string;
  headers?: HeadersInit;       // Headers | Record<string, string> | string[][]
  body?: BodyInit | null;      // string | FormData | Blob | etc.
  mode?: RequestMode;
  credentials?: RequestCredentials;
  cache?: RequestCache;
  redirect?: RequestRedirect;
  referrer?: string;
  referrerPolicy?: ReferrerPolicy;
  integrity?: string;
  signal?: AbortSignal | null;
}
```

```typescript
const options: RequestInit = {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer token123"
  },
  body: JSON.stringify({ name: "Ana", email: "ana@example.com" })
};

fetchJSON<UsuarioNuevo>("https://api.example.com/users", options);
```

6. MANEJO DE ERRORES HTTP ESPECÍFICOS
===============================================================================

```typescript
class ApiError extends Error {
  public readonly status: number;
  public readonly statusText: string;
  public readonly data?: unknown;

  constructor(status: number, statusText: string, data?: unknown) {
    super(`API Error: ${status} ${statusText}`);
    this.name = "ApiError";
    this.status = status;
    this.statusText = statusText;
    this.data = data;
  }
}

async function fetchConError<T>(url: string, options?: RequestInit): Promise<T> {
  const respuesta = await fetch(url, options);

  if (!respuesta.ok) {
    let datosError: unknown;
    try {
      datosError = await respuesta.json();
    } catch {
      datosError = null;
    }
    throw new ApiError(respuesta.status, respuesta.statusText, datosError);
  }

  const datos: T = await respuesta.json();
  return datos;
}

// Uso con try/catch
try {
  const usuario = await fetchConError<Usuario>("https://api.example.com/users/999");
} catch (error) {
  if (error instanceof ApiError) {
    console.error(`Error ${error.status}: ${error.message}`);
    if (error.status === 404) {
      console.log("Usuario no encontrado");
    }
    if (error.status === 500) {
      console.log("Error del servidor, intenta más tarde");
    }
  }
}
```

7. ABORTAR PETICIONES CON ABORTCONTROLLER
===============================================================================

```typescript
function fetchConTimeout<T>(url: string, ms: number = 5000): Promise<T> {
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), ms);

  return fetchJSON<T>(url, { signal: controller.signal })
    .finally(() => clearTimeout(timeoutId));
}

// Uso
try {
  const data = await fetchConTimeout<Usuario[]>("https://api.example.com/users", 3000);
  console.log(`Recibidos ${data.length} usuarios`);
} catch (error) {
  if (error instanceof DOMException && error.name === "AbortError") {
    console.error("La petición tardó demasiado");
  }
}
```

8. CABECERAS TIPADAS
===============================================================================

El tipo `Headers` permite manipular cabeceras con métodos tipados:

```typescript
async function fetchConCabeceras(url: string): Promise<Response> {
  const headers = new Headers();
  headers.append("Content-Type", "application/json");
  headers.append("Accept", "application/json");
  headers.append("X-API-Key", "mi-clave-secreta");

  // También se puede inicializar con un objeto
  const headers2 = new Headers({
    "Content-Type": "application/json",
    "Authorization": "Bearer token"
  });

  return fetch(url, { headers: headers });
}
```

Métodos de `Headers`:

```typescript
const h = new Headers();
h.set("Content-Type", "application/json");
h.get("Content-Type");       // string | null
h.has("Authorization");      // boolean
h.delete("X-Custom");
h.forEach((value, key) => {});
```

9. TRABAJANDO CON LA RESPUESTA
===============================================================================

```typescript
async function inspeccionarRespuesta(url: string): Promise<void> {
  const respuesta = await fetch(url);

  // Propiedades tipadas de Response
  console.log("Status:", respuesta.status);           // number
  console.log("Status Text:", respuesta.statusText);  // string
  console.log("OK:", respuesta.ok);                    // boolean
  console.log("URL final:", respuesta.url);            // string
  console.log("Tipo:", respuesta.type);                // ResponseType

  // Cabeceras de respuesta
  const contentType = respuesta.headers.get("content-type");
  console.log("Content-Type:", contentType);

  // Leer el cuerpo
  if (contentType?.includes("application/json")) {
    const datos = await respuesta.json() as unknown;
    console.log("Datos:", datos);
  } else {
    const texto = await respuesta.text();
    console.log("Texto:", texto);
  }
}
```

10. MÚLTIPLES PETICIONES EN PARALELO
===============================================================================

```typescript
interface Post {
  userId: number;
  id: number;
  title: string;
  body: string;
}

async function obtenerMultiplesPosts(): Promise<Post[]> {
  const urls = [
    "https://jsonplaceholder.typicode.com/posts/1",
    "https://jsonplaceholder.typicode.com/posts/2",
    "https://jsonplaceholder.typicode.com/posts/3"
  ];

  const promesas = urls.map(url => fetchJSON<Post>(url));
  const posts = await Promise.all(promesas);

  return posts;
}

// Promise.allSettled para tolerancia a fallos
async function obtenerPostsTolerante(): Promise<(Post | null)[]> {
  const urls = [
    "https://jsonplaceholder.typicode.com/posts/1",
    "https://jsonplaceholder.typicode.com/posts/invalid", // fallará
    "https://jsonplaceholder.typicode.com/posts/3"
  ];

  const resultados = await Promise.allSettled(
    urls.map(url => fetchJSON<Post>(url))
  );

  return resultados.map(r =>
    r.status === "fulfilled" ? r.value : null
  );
}
```

async function enviarFormulario(datos: Record<string, string>): Promise<Response> {
  const formData = new FormData();

  for (const [clave, valor] of Object.entries(datos)) {
    formData.append(clave, valor);
  }

  return fetch("https://api.example.com/submit", {
    method: "POST",
    body: formData  // El Content-Type se pone automáticamente
  });
}

// Para archivos
async function subirArchivo(archivo: File): Promise<{ url: string }> {
  const formData = new FormData();
  formData.append("file", archivo);

  return fetchJSON<{ url: string }>("https://api.example.com/upload", {
    method: "POST",
    body: formData
  });
}

12. FETCH TIPADO CON CLASES
===============================================================================

```typescript
class ApiCliente {
  private baseUrl: string;
  private headers: Headers;

  constructor(baseUrl: string, token?: string) {
    this.baseUrl = baseUrl;
    this.headers = new Headers({
      "Content-Type": "application/json"
    });

    if (token) {
      this.headers.set("Authorization", `Bearer ${token}`);
    }
  }

  async get<T>(endpoint: string): Promise<T> {
    const url = `${this.baseUrl}${endpoint}`;
    return fetchJSON<T>(url, { headers: this.headers });
  }

  async post<T, U>(endpoint: string, datos: U): Promise<T> {
    const url = `${this.baseUrl}${endpoint}`;
    return fetchJSON<T>(url, {
      method: "POST",
      headers: this.headers,
      body: JSON.stringify(datos)
    });
  }

  async put<T, U>(endpoint: string, datos: U): Promise<T> {
    const url = `${this.baseUrl}${endpoint}`;
    return fetchJSON<T>(url, {
      method: "PUT",
      headers: this.headers,
      body: JSON.stringify(datos)
    });
  }

  async delete<T>(endpoint: string): Promise<T> {
    const url = `${this.baseUrl}${endpoint}`;
    return fetchJSON<T>(url, {
      method: "DELETE",
      headers: this.headers
    });
  }
}

// Uso
const api = new ApiCliente("https://api.example.com", "mi-token");

api.get<Usuario[]>("/users").then(users => {
  users.forEach(u => console.log(u.name));
});

api.post<UsuarioCreado, NuevoUsuario>("/users", {
  name: "Ana",
  email: "ana@example.com"
}).then(nuevo => {
  console.log("Creado:", nuevo.id);
});
```

RESUMEN DEL CAPÍTULO
===============================================================================

- `fetch` devuelve `Promise<Response>`, donde `Response` es un tipo completo.
- `response.json()` devuelve `Promise<any>` — debemos tiparlo explícitamente.
- Las funciones genéricas `fetchJSON<T>` centralizan el tipado de respuestas.
- `RequestInit` tipa todas las opciones de la petición.
- `ApiError` personalizado permite manejar errores HTTP específicos.
- `AbortController` + `signal` para cancelar peticiones.
- `Headers` tiene métodos tipados para manipular cabeceras.
- `Promise.all` y `Promise.allSettled` para peticiones paralelas.
- Una clase `ApiCliente` encapsula toda la lógica de fetch con tipos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué `json()` devuelve `Promise<any>`?**

Porque TypeScript no puede saber la forma del JSON que devuelve el servidor.
Es nuestra responsabilidad declarar el tipo con una interfaz.

❓ **¿Qué pasa si la respuesta no coincide con mi interfaz?**

No habrá error de compilación (los tipos no existen en runtime). Usa
validación en runtime con type guards o librerías como Zod.

❓ **¿Cómo manejo errores de red (sin conexión)?**

`fetch` solo rechaza la promesa por errores de red. Los errores HTTP
(404, 500) son respuestas exitosas a nivel de fetch. Debes verificar
`response.ok` manualmente.

❓ **¿Es necesario usar `async/await` con fetch?**

No, puedes usar `.then()` y `.catch()`. Pero `async/await` con tipos
suele ser más legible y fácil de tipar.
