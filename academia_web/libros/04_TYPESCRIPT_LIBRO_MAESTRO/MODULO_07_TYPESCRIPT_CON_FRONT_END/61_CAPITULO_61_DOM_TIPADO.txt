1. ¿QUÉ ES EL DOM Y POR QUÉ LE IMPORTAN LOS TIPOS A TYPESCRIPT?
===============================================================================

El DOM es la representación en memoria de la estructura HTML de una página
web. Cada etiqueta HTML se convierte en un objeto que podemos manipular con
JavaScript — y por supuesto, con TypeScript.

Cuando escribimos JavaScript vanilla, accedemos al DOM con métodos como
`getElementById` y el resultado es... un tipo genérico `Element` o `HTMLElement`.
TypeScript refina estos tipos para que sepamos exactamente con qué estamos
trabajando.

Tipos fundamentales del DOM:

- `Node`       → clase base abstracta
- `Element`    → nodo elemento genérico
- `HTMLElement` → elemento HTML genérico
- `HTMLDivElement` → <div>
- `HTMLInputElement` → <input>
- `HTMLButtonElement` → <button>
- `HTMLAnchorElement` → <a>
- `HTMLParagraphElement` → <p>
- `HTMLHeadingElement` → <h1>...<h6>

Cada tipo específico expone propiedades y métodos propios de esa etiqueta.

```typescript
// El navegador ya tiene tipos incluidos

// Sin tipo específico — devuelve HTMLElement | null
const elemento = document.getElementById("app");

// Con tipo específico — hacemos casteo
const input = document.getElementById("email") as HTMLInputElement;
input.value;            // ✅ propiedad existente en input
input.src;              // ❌ Error: src no existe en HTMLInputElement
```

El casteo `as HTMLInputElement` le dice a TypeScript: "confía en mí, sé lo
que estoy haciendo". Pero es mejor cuando podemos evitarlo.

===============================================================================
2. SELECCIÓN DE ELEMENTOS CON TIPOS
===============================================================================

2.1. getElementById

Devuelve `HTMLElement | null`. Debemos manejar el null.

```typescript
const contenedor = document.getElementById("root");
// Tipo: HTMLElement | null

if (contenedor) {
  contenedor.style.backgroundColor = "#f0f0f0";
}

// Con casteo si estamos seguros
const nav = document.getElementById("navbar") as HTMLElement;
nav.style.display = "flex";
```

2.2. querySelector

Devuelve `Element | null`. Es más genérico, así que el casteo es común.

```typescript
const boton = document.querySelector(".btn-primary") as HTMLButtonElement;
boton.disabled = true;

const primerInput = document.querySelector("input") as HTMLInputElement;
primerInput.focus();
```

2.3. querySelectorAll

Devuelve `NodeListOf<Element>`. Podemos refinarlo.

```typescript
const items = document.querySelectorAll(".list-item") as NodeListOf<HTMLLIElement>;
items.forEach(item => {
  item.style.color = "blue";
});
```

2.4. getElementsByClassName / getElementsByTagName

Devuelven `HTMLCollectionOf<T>`.

```typescript
const divs = document.getElementsByTagName("div");
// Tipo: HTMLCollectionOf<HTMLDivElement>

const parrafos = document.getElementsByClassName("texto");
// Tipo: HTMLCollectionOf<Element>  — más genérico
```

===============================================================================
3. MANIPULACIÓN DE ATRIBUTOS Y PROPIEDADES
===============================================================================

3.1. Atributos con getAttribute / setAttribute

```typescript
const enlace = document.querySelector("a") as HTMLAnchorElement;

const href = enlace.getAttribute("href");  // string | null
enlace.setAttribute("target", "_blank");

// Propiedades directas (mejor)
enlace.href = "https://lucasega.academy";
enlace.target = "_blank";
```

3.2. Clases con classList

```typescript
const card = document.querySelector(".card") as HTMLElement;

card.classList.add("active");
card.classList.remove("inactive");
card.classList.toggle("visible");
const tieneClase = card.classList.contains("active"); // boolean
```

3.3. Estilos inline

```typescript
const titulo = document.querySelector("h1") as HTMLHeadingElement;

titulo.style.color = "#ff6600";
titulo.style.fontSize = "2rem";
titulo.style.marginTop = "20px";
// Todas las propiedades CSS están tipadas
// fontSize → propiedad camelCase
```

===============================================================================
4. CREACIÓN E INSERCIÓN DE ELEMENTOS
===============================================================================

4.1. createElement

```typescript
const nuevoDiv = document.createElement("div");
// Tipo: HTMLDivElement

const nuevoParrafo = document.createElement("p");
// Tipo: HTMLParagraphElement

const nuevoInput = document.createElement("input");
// Tipo: HTMLInputElement
```

4.2. createTextNode

```typescript
const texto = document.createTextNode("Hola TypeScript");
// Tipo: Text

nuevoParrafo.appendChild(texto);
```

4.3. Inserción en el DOM

```typescript
const contenedor = document.getElementById("app") as HTMLDivElement;

const tarjeta = document.createElement("div");
tarjeta.className = "tarjeta";
tarjeta.innerHTML = `
  <h3>Título</h3>
  <p>Descripción</p>
`;

contenedor.appendChild(tarjeta);
contenedor.insertBefore(tarjeta, contenedor.firstChild);
```

===============================================================================
5. NAVEGACIÓN DEL ÁRBOL DOM
===============================================================================

Propiedades de navegación tipadas:

```typescript
const section = document.querySelector("section") as HTMLElement;

// Hijos
const hijos: HTMLCollection = section.children;        // solo elementos
const nodosHijos: NodeList = section.childNodes;        // incluye texto

// Padre
const padre: HTMLElement | null = section.parentElement;

// Hermanos
const prev: Element | null = section.previousElementSibling;
const next: Element | null = section.nextElementSibling;

// Primer / último hijo elemento
const primero: Element | null = section.firstElementChild;
const ultimo: Element | null = section.lastElementChild;
```

Cuidado: `children` devuelve `HTMLCollection`, que es una colección viva.
`querySelectorAll` devuelve `NodeList`, que es estática.

6. EVENTOS BÁSICOS SOBRE EL DOM (VISTA PREVIA)
===============================================================================

Aunque el Capítulo 62 está dedicado a eventos, veamos un adelanto con tipos:

```typescript
const boton = document.querySelector("button") as HTMLButtonElement;

boton.addEventListener("click", (event: MouseEvent) => {
  console.log("Coordenadas:", event.clientX, event.clientY);
});
```

TypeScript infiere el tipo del evento según el elemento y el event name.
`MouseEvent`, `KeyboardEvent`, `FocusEvent` — cada uno con sus propiedades
específicas.

===============================================================================
7. EL TIPO GENERIC EN querySelector
===============================================================================

`querySelector` y `querySelectorAll` tienen versiones genéricas:

```typescript
// querySelector<K extends keyof HTMLElementTagNameMap>
const img = document.querySelector<HTMLImageElement>("img");
// Tipo: HTMLImageElement | null

const inputs = document.querySelectorAll<HTMLInputElement>("input");
// Tipo: NodeListOf<HTMLInputElement>

// También funciona con elementos SVG
const svg = document.querySelector<SVGSVGElement>("svg");
```

El `HTMLElementTagNameMap` es un tipo de utilidad que mapea nombres de
etiquetas a sus tipos específicos.

```typescript
// Internamente TypeScript tiene:
interface HTMLElementTagNameMap {
  "a": HTMLAnchorElement;
  "button": HTMLButtonElement;
  "div": HTMLDivElement;
  "input": HTMLInputElement;
  // ...etc
}
```

===============================================================================
8. MANEJO SEGURO DE NULL
===============================================================================

Siempre que seleccionamos un elemento, puede ser `null`. Estrategias:

8.1. if guard

```typescript
const header = document.getElementById("header");
if (header) {
  header.textContent = "Bienvenido";
}
```

8.2. Optional chaining

```typescript
document.getElementById("stats")?.textContent = "Cargando...";
```

8.3. Non-null assertion (solo cuando estés 100% seguro)

```typescript
const footer = document.getElementById("footer")!;
footer.style.borderTop = "1px solid #ccc";
```

8.4. Aserción temprana con as

```typescript
const main = document.getElementById("main") as HTMLElement;
main.innerHTML = "<p>Contenido principal</p>";
```

9. EJEMPLO COMPLETO: CONSTRUCCIÓN DE UNA TARJETA DE PERFIL
===============================================================================

```typescript
interface DatosPerfil {
  nombre: string;
  edad: number;
  ocupacion: string;
}

function crearTarjetaPerfil(datos: DatosPerfil): HTMLDivElement {
  const contenedor = document.getElementById("perfiles") as HTMLDivElement;

  const tarjeta = document.createElement("div");
  tarjeta.className = "perfil-tarjeta";

  const nombre = document.createElement("h2");
  nombre.textContent = datos.nombre;

  const edad = document.createElement("p");
  edad.textContent = `Edad: ${datos.edad}`;

  const ocupacion = document.createElement("p");
  ocupacion.textContent = `Ocupación: ${datos.ocupacion}`;

  tarjeta.appendChild(nombre);
  tarjeta.appendChild(edad);
  tarjeta.appendChild(ocupacion);

  contenedor.appendChild(tarjeta);
  return tarjeta;
}

// Uso
crearTarjetaPerfil({
  nombre: "Ana López",
  edad: 28,
  ocupacion: "Ingeniera de Software"
});
```

===============================================================================
10. EL TIPO Document Y Window
===============================================================================

Además de los elementos, el propio `document` y `window` tienen tipos:

```typescript
// Document
const doc: Document = document;
doc.title = "Mi Página";
doc.body.style.margin = "0";

// Window
const ventana: Window = window;
ventana.alert("Hola");
ventana.innerWidth;  // number
ventana.location.href;  // string
```

`Window` también tiene `document` y todos los métodos globales del navegador.
TypeScript incluye estos tipos gracias al archivo `lib.dom.d.ts`.

===============================================================================
RESUMEN DEL CAPÍNULO
===============================================================================

- El DOM se compone de tipos como HTMLElement, HTMLDivElement, etc.
- `getElementById` devuelve `HTMLElement | null`, siempre maneja el null.
- `querySelector` con generic `<T>` permite seleccionar con tipo específico.
- El casteo `as Tipo` refina el tipo cuando TypeScript no puede inferirlo.
- `classList` y `style` están completamente tipados.
- `createElement` infiere el tipo según el nombre de etiqueta.
- Navegar el DOM con `children`, `parentElement`, `nextElementSibling` etc.
- El `HTMLElementTagNameMap` asocia etiquetas con sus tipos.
- Usa optional chaining `?.` y non-null assertion `!` con cuidado.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué querySelector devuelve Element y no HTMLElement?**

Porque `querySelector` puede devolver también elementos SVG (SVGElement).
Al usar el generic `<T>` le indicamos el tipo esperado.

❓ **¿Es malo usar `as Tipo` siempre?**

No es malo, pero reduce la seguridad. Prefiere el generic
`querySelector<HTMLInputElement>("input")` sobre el casteo directo.

❓ **¿Cómo sé qué tipo usar para un elemento personalizado?**

Crea una interfaz que extienda `HTMLElement` y usa el casteo. O declara
el elemento en `HTMLElementTagNameMap` si es global.

❓ **¿Qué pasa si el elemento no existe en el DOM?**

Los métodos de selección devuelven `null`. TypeScript te obliga a
manejarlo, lo cual es una ventaja, no un problema.
