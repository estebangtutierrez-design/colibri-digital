1. INTRODUCCIÓN AL TIPO BOOLEAN
===============================================================================

El tipo boolean en TypeScript solo puede tener dos valores: true o false.
Es el tipo de dato más simple, pero también uno de los más importantes.

```typescript
let activo: boolean = true;
let completado: boolean = false;

// TypeScript infiere el tipo
let esValido = true;  // type: boolean
let esMayor = 10 > 5; // type: boolean

console.log(activo);       // true
console.log(completado);   // false
console.log(esMayor);      // true
```

Los booleanos son el resultado natural de las comparaciones y operaciones
lógicas. Cada vez que usas if, while, o el operador ternario, estás
trabajando con valores booleanos.

---

📌 **Boolean es un tipo primitivo, no un objeto**

En TypeScript, `boolean` (minúscula) es el tipo primitivo. `Boolean`
(mayúscula) es el constructor wrapper. Siempre usa el tipo primitivo.

```typescript
let correcto: boolean = true;    // ✅ Correcto: tipo primitivo
let incorrecto: Boolean = true;  // ❌ Evitar: tipo objeto wrapper
```

---

🔵 **Ejemplo 1: Declaración y uso básico**

```typescript
// Variables booleanas
let estaAutenticado: boolean = false;
let esAdministrador = true;

// Booleanos a partir de expresiones
let edad = 18;
let esMayorDeEdad = edad >= 18;
let puedeVotar = esMayorDeEdad && estaAutenticado;

console.log(esMayorDeEdad);   // true
console.log(puedeVotar);      // false (no autenticado)

// Uso en condicionales
if (estaAutenticado) {
  console.log("Usuario autenticado");
} else {
  console.log("Usuario no autenticado");
}
```

🔵 **Ejemplo 2: Funciones que retornan boolean**

```typescript
function esPar(numero: number): boolean {
  return numero % 2 === 0;
}

function esPositivo(numero: number): boolean {
  return numero > 0;
}

function tieneLongitudValida(texto: string, min: number, max: number): boolean {
  return texto.length >= min && texto.length <= max;
}

console.log(esPar(4));               // true
console.log(esPar(7));               // false
console.log(esPositivo(-5));         // false
console.log(tieneLongitudValida("Hola", 2, 10));  // true
```

🔵 **Ejemplo 3: Booleanos en parámetros opcionales**

```typescript
function crearUsuario(
  nombre: string,
  activo: boolean = true,
  notificar: boolean = false
): object {
  return {
    nombre,
    activo,
    notificar,
    createdAt: new Date()
  };
}

console.log(crearUsuario("Lucas"));
// { nombre: "Lucas", activo: true, notificar: false, ... }

console.log(crearUsuario("Ana", false, true));
// { nombre: "Ana", activo: false, notificar: true, ... }
```

2. OPERADORES LÓGICOS CON BOOLEANOS
===============================================================================

TypeScript hereda los tres operadores lógicos de JavaScript: AND (&&),
OR (||) y NOT (!). Estos operadores trabajan con valores booleanos y
siguen las reglas clásicas de la lógica proposicional.

---

📌 **Tabla de verdad de los operadores lógicos**

```typescript
// AND (&&) — true solo si ambos son true
console.log(true && true);    // true
console.log(true && false);   // false
console.log(false && true);   // false
console.log(false && false);  // false

// OR (||) — true si al menos uno es true
console.log(true || true);    // true
console.log(true || false);   // true
console.log(false || true);   // true
console.log(false || false);  // false

// NOT (!) — invierte el valor
console.log(!true);           // false
console.log(!false);          // true
console.log(!!true);          // true (doble negación)
```

---

🔵 **Ejemplo 4: Evaluación de cortocircuito (short-circuit)**

```typescript
// AND (&&) — si el primero es false, no evalúa el segundo
function riesgoAlto(): boolean {
  console.log("Evaluando riesgo...");
  return true;
}

let usuarioActivo = false;
let resultado = usuarioActivo && riesgoAlto();
console.log(resultado);   // false (riesgoAlto no se ejecuta)

// OR (||) — si el primero es true, no evalúa el segundo
let usuarioAdmin = true;
let permiso = usuarioAdmin || riesgoAlto();
console.log(permiso);     // true (riesgoAlto no se ejecuta)
```

🔵 **Ejemplo 5: Patrones comunes con operadores lógicos**

```typescript
// Valor por defecto con ||
function obtenerNombre(nombre?: string): string {
  return nombre || "Invitado";
}

console.log(obtenerNombre("Lucas")); // "Lucas"
console.log(obtenerNombre());        // "Invitado"

// Ejecución condicional con &&
let esAdmin = true;
esAdmin && console.log("Acceso concedido");  // Se ejecuta

let esUsuario = false;
esUsuario && console.log("Bienvenido");       // No se ejecuta

// Operador ternario (?:)
let edad = 20;
let tipo = edad >= 18 ? "Adulto" : "Menor";
console.log(tipo);  // "Adulto"
```

🔵 **Ejemplo 6: Funciones con validación lógica**

```typescript
function puedeAcceder(
  esAutenticado: boolean,
  tienePermiso: boolean,
  esDueno: boolean
): boolean {
  return esAutenticado && (tienePermiso || esDueno);
}

console.log(puedeAcceder(true, false, true));   // true (es dueño)
console.log(puedeAcceder(true, false, false));  // false
console.log(puedeAcceder(false, true, true));   // false (no autenticado)

// Versión más legible con variables descriptivas
function validarAcceso(
  usuario: { autenticado: boolean; admin: boolean; dueno: boolean }
): boolean {
  let autenticacionValida = usuario.autenticado;
  let autorizacionValida = usuario.admin || usuario.dueno;
  return autenticacionValida && autorizacionValida;
}
```

3. TRUTHY Y FALSY
===============================================================================

En TypeScript, todo valor puede ser evaluado como booleano en un contexto
lógico. Los valores que se evalúan como true se llaman truthy, y los que
se evalúan como false se llaman falsy.

---

📌 **Valores falsy en TypeScript**

Solo hay 7 valores falsy en todo el lenguaje:

```typescript
if (false) {}        // false (el único booleano falsy)
if (0) {}            // 0 (cero)
if (-0) {}           // -0 (cero negativo)
if (0n) {}           // 0n (bigint cero)
if ("") {}           // "" (string vacío)
if (null) {}         // null
if (undefined) {}    // undefined
if (NaN) {}          // NaN

// Todos los demás valores son truthy
if ("false") {}      // truthy (string no vacío)
if ("0") {}          // truthy
if ([]) {}           // truthy (array vacío)
if ({}) {}           // truthy (objeto vacío)
if (Infinity) {}     // truthy
```

---

🔵 **Ejemplo 7: Identificando truthy y falsy**

```typescript
function esFalsy(valor: unknown): boolean {
  return !valor;
}

function esTruthy(valor: unknown): boolean {
  return !!valor;  // doble negación → booleano
}

console.log(esFalsy(0));          // true
console.log(esFalsy(""));         // true
console.log(esFalsy(null));       // true
console.log(esFalsy("Hola"));     // false
console.log(esTruthy([]));        // true
console.log(esTruthy("false"));   // true (string, no booleano)
console.log(esTruthy("0"));       // true (string, no número)
```

🔵 **Ejemplo 8: Peligros de la coerción implícita**

```typescript
// Problema: el número 0 es falsy
function procesarDescuento(descuento: number): string {
  if (!descuento) {
    return "Sin descuento";
  }
  return `Descuento: ${descuento}%`;
}

console.log(procesarDescuento(10));  // "Descuento: 10%"
console.log(procesarDescuento(0));   // "Sin descuento" (¡incorrecto!)

// Solución: verificación explícita
function procesarDescuentoSeguro(descuento: number): string {
  if (descuento === 0 || descuento == null) {
    return "Sin descuento";
  }
  return `Descuento: ${descuento}%`;
}

console.log(procesarDescuentoSeguro(0)); // "Sin descuento" (correcto)
```

🔵 **Ejemplo 9: Casos engañosos con truthy/falsy**

```typescript
// Caso 1: El string "false" es truthy
let dato: string = "false";
if (dato) {
  console.log("Esto se ejecuta aunque el string diga 'false'");
}

// Caso 2: Arrays vacíos son truthy
let items: string[] = [];
if (items) {
  console.log("Un array vacío es truthy");
}
// Forma correcta:
if (items.length > 0) {
  console.log("El array tiene elementos");
}

// Caso 3: NaN es falsy pero typeof NaN es "number"
let valor = NaN;
if (!valor) {
  console.log("NaN es falsy");
}
console.log(typeof valor);  // "number"
```

4. EL CONSTRUCTOR BOOLEAN()
===============================================================================

El constructor Boolean() puede usarse como función para convertir cualquier
valor a su representación booleana explícita.

---

📌 **Boolean() vs doble negación !!**

Ambos producen el mismo resultado. La diferencia es puramente estilística.

```typescript
console.log(Boolean(1));         // true
console.log(Boolean(0));         // false
console.log(Boolean(""));        // false
console.log(Boolean("Hola"));    // true
console.log(Boolean(null));      // false
console.log(Boolean(undefined)); // false
console.log(Boolean([]));        // true
console.log(Boolean({}));        // true

// Equivalente con doble negación
console.log(!!1);         // true
console.log(!!0);         // false
```

---

🔵 **Ejemplo 10: Uso práctico de Boolean()**

```typescript
// Filtrar valores falsy de un array
let valores = [0, 1, "", "Hola", null, undefined, false, true, NaN];
let valoresValidos = valores.filter(Boolean);
console.log(valoresValidos);  // [1, "Hola", true]

// Equivalente sin Boolean
let equivalentes = valores.filter(v => !!v);
console.log(equivalentes);    // [1, "Hola", true]

// Verificar si un valor existe
function mostrarMensaje(mensaje?: string): void {
  if (Boolean(mensaje)) {
    console.log(mensaje);
  } else {
    console.log("No hay mensaje");
  }
}
```

🔵 **Ejemplo 11: Boolean() con new (no recomendado)**

```typescript
// Usar Boolean como función (correcto)
let comoFuncion: boolean = Boolean("true");
console.log(comoFuncion);          // true
console.log(typeof comoFuncion);   // "boolean"

// Usar Boolean como constructor (evitar)
let comoObjeto: Boolean = new Boolean("true");
console.log(comoObjeto);           // [Boolean: true]
console.log(typeof comoObjeto);    // "object" (¡no es boolean primitivo!)

// Peligro: los objetos Boolean siempre son truthy
let falsoObjeto = new Boolean(false);
if (falsoObjeto) {
  console.log("¡Esto se ejecuta aunque el valor sea false!");
}
```

🔵 **Ejemplo 12: Conversión explícita vs implícita**

```typescript
// Explícita (clara y predecible)
function puedePublicar(articulo: { titulo?: string; contenido?: string }): boolean {
  let tieneTitulo = Boolean(articulo.titulo);
  let tieneContenido = Boolean(articulo.contenido);
  return tieneTitulo && tieneContenido;
}

// Implícita (menos clara)
function puedePublicarImplicito(articulo: { titulo?: string; contenido?: string }): boolean {
  return !!(articulo.titulo && articulo.contenido);
}

console.log(puedePublicar({ titulo: "Hola", contenido: "Mundo" }));   // true
console.log(puedePublicar({ titulo: "", contenido: "Mundo" }));       // false
```

🔵 **Ejemplo 13: Nombres descriptivos para booleanos**

```typescript
// ❌ Malos nombres
let flag = true;
let estado = false;
let check = true;

// ✅ Buenos nombres (responden sí/no)
let estaActivo = true;
let esAdministrador = false;
let tienePermiso = true;
let puedeEditar = false;
let notificacionEnviada = true;

// Uso en condicionales legibles
if (estaActivo && esAdministrador) {
  console.log("Admin activo");
}
// Se lee como: "si está activo Y es administrador"
```

🔵 **Ejemplo 14: Evitar comparaciones innecesarias**

```typescript
let activo = true;

// ❌ Redundante
if (activo === true) { ... }
if (activo === false) { ... }

// ✅ Correcto
if (activo) { ... }
if (!activo) { ... }

// ❌ Redundante en ternarios
let texto = activo === true ? "Sí" : "No";

// ✅ Correcto
let texto = activo ? "Sí" : "No";

// ❌ Retornar booleano de forma verbosa
function esMayorEdad(edad: number): boolean {
  if (edad >= 18) {
    return true;
  } else {
    return false;
  }
}

// ✅ Correcto (directo)
function esMayorEdad(edad: number): boolean {
  return edad >= 18;
}
```

🔵 **Ejemplo 15: Booleanos en objetos de configuración**

```typescript
interface ConfiguracionApp {
  darkMode: boolean;
  notificaciones: boolean;
  sonido: boolean;
  vibracion: boolean;
  modoAvion: boolean;
}

function iniciarApp(config: ConfiguracionApp): void {
  if (config.darkMode) {
    console.log("Modo oscuro activado");
  }
  if (config.notificaciones) {
    console.log("Notificaciones activadas");
  }
}

const config: ConfiguracionApp = {
  darkMode: true,
  notificaciones: false,
  sonido: true,
  vibracion: false,
  modoAvion: false
};

iniciarApp(config);
```

RESUMEN DEL CAPÍTULO
===============================================================================

- El tipo boolean solo tiene dos valores: true y false, y es la base de
  todo flujo de control en TypeScript.
- Los operadores lógicos (&&, ||, !) siguen las reglas de la lógica
  proposicional y utilizan evaluación de cortocircuito.
- Los valores truthy son aquellos que se evalúan como true en un contexto
  booleano; los falsy son solo 7: false, 0, -0, 0n, "", null, undefined, NaN.
- El constructor Boolean() y la doble negación (!!) convierten cualquier
  valor a su representación booleana explícita.
- Evita la coerción implícita cuando el 0 o el string vacío sean valores
  válidos en tu dominio.
- Nombra tus booleanos con prefijos como es, esta, tiene, puede para que
  las condiciones se lean de forma natural.
- Siempre usa el tipo primitivo boolean, no el objeto wrapper Boolean.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre boolean y Boolean?**
boolean (minúscula) es el tipo primitivo de TypeScript. Boolean (mayúscula)
es el constructor wrapper de JavaScript. Siempre debes usar boolean como tipo.
Usar Boolean como función (sin new) es útil para conversiones explícitas,
pero new Boolean() crea un objeto que siempre es truthy, incluso con false.

❓ **¿Por qué if([]) se ejecuta si el array está vacío?**
Porque los arrays vacíos son objetos, y todos los objetos son truthy en
TypeScript. Para verificar si un array está vacío, usa array.length > 0
en lugar de confiar en la coerción booleana.

❓ **¿Cuándo debo usar Boolean() vs !!?**
Ambos son equivalentes. Boolean() es más explícito y legible para quienes
no conocen el patrón de doble negación. !! es más compacto y común en
código existente. La elección depende de la convención de tu equipo.

❓ **¿Es seguro usar !variable para verificar existencia?**
No siempre. !variable devuelve true para cualquier valor falsy, incluyendo
0 y "", que pueden ser valores válidos. Usa comprobaciones explícitas como
variable === null || variable === undefined cuando necesites verificar
existencia real.
