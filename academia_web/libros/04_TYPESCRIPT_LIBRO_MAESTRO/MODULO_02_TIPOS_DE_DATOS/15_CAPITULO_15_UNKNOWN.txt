1. INTRODUCCIÓN AL TIPO UNKNOWN
===============================================================================

El tipo unknown es el counterpart seguro de any. Fue introducido en
TypeScript 3.0 como una alternativa type-safe para representar valores
cuyo tipo no conocemos en tiempo de compilación.

```typescript
let valor: unknown;

valor = 42;                    // OK
valor = "Hola";                // OK
valor = true;                  // OK
valor = { nombre: "Lucas" };   // OK
valor = [1, 2, 3];            // OK

// A diferencia de any, no puedes operar directamente
// valor.toUpperCase();        // Error: Object is of type 'unknown'
// valor + 10;                 // Error: Object is of type 'unknown'
// valor.propiedad;            // Error: Object is of type 'unknown'
```

La regla de oro: **no puedes hacer nada con un valor unknown sin antes
verificar su tipo**. TypeScript te obliga a demostrar que sabes lo que
estás haciendo.

---

📌 **Comparación: any vs unknown**

```typescript
let valorAny: any;
let valorUnknown: unknown;

// Asignación
valorAny = "cualquier cosa";     // OK
valorUnknown = "cualquier cosa"; // OK

// Re-asignación a otros tipos
let texto: string = "hola";
texto = valorAny;                // OK (any asignable a cualquier tipo)
// texto = valorUnknown;        // Error: Type 'unknown' is not assignable to 'string'

// Operaciones
valorAny.metodo();              // OK (sin verificación)
// valorUnknown.metodo();       // Error

// Type narrowing requerido
if (typeof valorUnknown === "string") {
  console.log(valorUnknown.toUpperCase());  // OK
}
```

---

🔵 **Ejemplo 1: unknown en la práctica**

```typescript
// Función que recibe un valor desconocido
function procesarEntrada(entrada: unknown): string {
  // No podemos usar entrada directamente
  // return entrada.toUpperCase(); // Error

  // Debemos verificar el tipo primero
  if (typeof entrada === "string") {
    return entrada.toUpperCase();
  }

  if (typeof entrada === "number") {
    return entrada.toFixed(2);
  }

  if (typeof entrada === "boolean") {
    return entrada ? "Verdadero" : "Falso";
  }

  return "Tipo no soportado";
}

console.log(procesarEntrada("hola"));   // "HOLA"
console.log(procesarEntrada(3.1415));   // "3.14"
console.log(procesarEntrada(true));     // "Verdadero"
console.log(procesarEntrada({}));       // "Tipo no soportado"
```

🔵 **Ejemplo 2: unknown con JSON.parse**

```typescript
// JSON.parse devuelve any por defecto
const datosAny = JSON.parse('{"nombre": "Lucas"}');
// datosAny es any: podemos hacer cualquier cosa (peligroso)

// Envoltorio seguro que devuelve unknown
function parseJsonSeguro(texto: string): unknown {
  try {
    return JSON.parse(texto);
  } catch {
    return null;
  }
}

const datos = parseJsonSeguro('{"nombre": "Lucas"}');
// datos es unknown: necesitamos verificar

if (typeof datos === "object" && datos !== null) {
  // Ahora TypeScript sabe que datos es un objeto (no null)
  // Pero aún necesitamos verificar las propiedades
  if ("nombre" in datos) {
    console.log(datos.nombre);  // OK gracias a "in"
  }
}
```

===============================================================================
2. TYPE NARROWING CON UNKNOWN
===============================================================================

El type narrowing es el proceso de reducir un tipo amplio (como unknown)
a un tipo más específico mediante verificaciones en runtime.

---

📌 **Técnicas de type narrowing para unknown**

```typescript
let valor: unknown = obtenerValorExterno();

// 1. typeof
if (typeof valor === "string") { /* valor: string */ }
if (typeof valor === "number") { /* valor: number */ }
if (typeof valor === "boolean") { /* valor: boolean */ }
if (typeof valor === "function") { /* valor: Function */ }
if (typeof valor === "object") { /* valor: object | null */ }

// 2. instanceof
if (valor instanceof Date) { /* valor: Date */ }
if (valor instanceof Array) { /* valor: any[] */ }
if (valor instanceof Map) { /* valor: Map<any, any> */ }

// 3. Verificación de propiedades (in)
if (typeof valor === "object" && valor !== null) {
  if ("id" in valor) { /* valor: object & { id: any } */ }
}

// 4. Type guards personalizados
if (esUsuario(valor)) { /* valor: Usuario */ }
```

---

🔵 **Ejemplo 3: Type narrowing con typeof**

```typescript
function analizar(valor: unknown): string {
  // typeof es la primera línea de defensa
  switch (typeof valor) {
    case "string":
      return `String de longitud ${valor.length}`;
    case "number":
      return `Número: ${isNaN(valor) ? "NaN" : valor.toFixed(2)}`;
    case "boolean":
      return `Booleano: ${valor ? "true" : "false"}`;
    case "undefined":
      return "Indefinido";
    case "function":
      return `Función con ${valor.length} parámetros`;
    case "bigint":
      return `BigInt: ${valor.toString()}`;
    case "symbol":
      return "Symbol";
    case "object":
      if (valor === null) return "Null";
      if (Array.isArray(valor)) return `Array de ${valor.length} elementos`;
      return "Objeto";
    default:
      return "Tipo desconocido";
  }
}

console.log(analizar("Hola"));           // String de longitud 4
console.log(analizar([1, 2, 3]));        // Array de 3 elementos
console.log(analizar(null));             // Null
```

🔵 **Ejemplo 4: Type guards personalizados para unknown**

```typescript
interface Usuario {
  id: number;
  nombre: string;
  email: string;
  activo: boolean;
}

function esUsuario(valor: unknown): valor is Usuario {
  if (typeof valor !== "object" || valor === null) return false;

  const obj = valor as Record<string, unknown>;

  return (
    typeof obj.id === "number"
    && typeof obj.nombre === "string"
    && typeof obj.email === "string"
    && typeof obj.activo === "boolean"
  );
}

function procesarUsuario(datos: unknown): Usuario {
  if (!esUsuario(datos)) {
    throw new Error("Datos de usuario inválidos");
  }
  // Aquí datos es Usuario gracias al type guard
  console.log(`Procesando: ${datos.nombre} (${datos.email})`);
  return datos;
}

const entrada1 = parseJsonSeguro('{"id": 1, "nombre": "Lucas", "email": "l@e.com", "activo": true}');
const entrada2 = parseJsonSeguro('{"nombre": "Incompleto"}');

console.log(procesarUsuario(entrada1));  // OK
// console.log(procesarUsuario(entrada2));  // Error: Datos inválidos
```

🔵 **Ejemplo 5: Validación anidada con unknown**

```typescript
interface RespuestaAPI<T> {
  exito: boolean;
  datos: T;
  error?: string;
}

function validarRespuesta<T>(raw: unknown, validador: (d: unknown) => d is T): RespuestaAPI<T> {
  if (typeof raw !== "object" || raw === null) {
    return { exito: false, datos: null as T, error: "Respuesta no es objeto" };
  }

  const obj = raw as Record<string, unknown>;

  if (typeof obj.exito !== "boolean") {
    return { exito: false, datos: null as T, error: "Campo 'exito' inválido" };
  }

  if (!obj.exito) {
    return {
      exito: false,
      datos: null as T,
      error: typeof obj.error === "string" ? obj.error : "Error desconocido"
    };
  }

  if (!validador(obj.datos)) {
    return { exito: false, datos: null as T, error: "Datos inválidos" };
  }

  return { exito: true, datos: obj.datos };
}

// Uso
interface Producto {
  id: number;
  nombre: string;
  precio: number;
}

function esProducto(v: unknown): v is Producto {
  if (typeof v !== "object" || v === null) return false;
  const o = v as Record<string, unknown>;
  return typeof o.id === "number"
    && typeof o.nombre === "string"
    && typeof o.precio === "number";
}

const respuesta = parseJsonSeguro('{"exito": true, "datos": {"id": 1, "nombre": "Laptop", "precio": 999}}');
const resultado = validarRespuesta(respuesta, esProducto);
console.log(resultado.exito);  // true
```

3. UNKNOWN EN ESTRUCTURAS DE DATOS
===============================================================================

unknown es ideal para estructuras de datos que contienen valores de
diferentes tipos.

🔵 **Ejemplo 6: Map con valores unknown**

```typescript
// Almacén seguro de valores de diferentes tipos
class AlmacenSeguro {
  private items = new Map<string, unknown>();

  setItem<T>(clave: string, valor: T): void {
    this.items.set(clave, valor);
  }

  getItem<T>(clave: string, validador: (v: unknown) => v is T): T | null {
    const valor = this.items.get(clave);
    if (valor === undefined) return null;
    if (validador(valor)) return valor;
    return null;
  }
}

const almacen = new AlmacenSeguro();

almacen.setItem("usuario", { id: 1, nombre: "Lucas" });
almacen.setItem("numeros", [1, 2, 3]);
almacen.setItem("activo", true);

// Recuperación segura con validación
function esArrayNumber(v: unknown): v is number[] {
  return Array.isArray(v) && v.every(item => typeof item === "number");
}

const nums = almacen.getItem("numeros", esArrayNumber);
console.log(nums);  // [1, 2, 3]
```

🔵 **Ejemplo 7: unknown vs any en arrays heterogéneos**

```typescript
// ❌ Array any (poco seguro)
const datosAny: any[] = [1, "texto", true, {}];
datosAny.forEach(item => {
  console.log(item.toUpperCase());  // Sin error de compilación, error en runtime
});

// ✅ Array unknown (obliga a verificar)
const datosUnknown: unknown[] = [1, "texto", true, {}];
datosUnknown.forEach(item => {
  if (typeof item === "string") {
    console.log(item.toUpperCase());  // ✅ Seguro
  } else if (typeof item === "number") {
    console.log(item.toFixed(2));      // ✅ Seguro
  }
});
```

4. UNKNOWN EN FUNCIONES Y PARÁMETROS
===============================================================================

🔵 **Ejemplo 8: Parámetros rest con unknown**

```typescript
function loguear(...args: unknown[]): void {
  args.forEach((arg, index) => {
    if (typeof arg === "string") {
      console.log(`[${index}] String: ${arg}`);
    } else if (typeof arg === "number") {
      console.log(`[${index}] Number: ${arg}`);
    } else if (typeof arg === "object" && arg !== null) {
      console.log(`[${index}] Object: ${JSON.stringify(arg)}`);
    } else {
      console.log(`[${index}] ${String(arg)}`);
    }
  });
}

loguear("Hola", 42, { a: 1 }, true);
// [0] String: Hola
// [1] Number: 42
// [2] Object: {"a":1}
// [3] true
```

🔵 **Ejemplo 9: Genéricos con unknown como constraint**

```typescript
// Restringir un genérico a tipos que no sean null/undefined
type NoNulo<T> = T extends null | undefined ? never : T;

function procesar<T>(valor: NoNulo<T>): void {
  console.log("Procesando:", valor);
}

procesar("hola");        // OK
procesar(42);            // OK
// procesar(null);        // Error
// procesar(undefined);   // Error

// Alternativa con unknown
function ejecutarSeguro<T extends unknown>(fn: () => T): T | null {
  try {
    return fn();
  } catch {
    return null;
  }
}
```

5. UNKNOWN VS OTHER TYPES
===============================================================================

🔵 **Ejemplo 10: unknown, any, never — la jerarquía**

```typescript
// Jerarquía de tipos en TypeScript:
//
//       unknown (super tipo de todos)
//        /    \
//    any       string | number | ...
//        \    /
//        never (sub tipo de todos)

let unknown: unknown = "texto";
let any: any = "texto";
let string: string = "texto";

string = any;           // ✅ any es asignable a string
// string = unknown;    // ❌ unknown no es asignable a string

unknown = string;        // ✅ string es asignable a unknown
unknown = any;           // ✅ any es asignable a unknown

// never es asignable a todos
function error(): never { throw new Error(); }
unknown = error();       // ✅
string = error();        // ✅ (nunca se ejecuta)
```

🔵 **Ejemplo 11: Patrón de función segura con unknown**

```typescript
// Función segura para acceder a propiedades anidadas
function obtenerRutaSegura<T>(
  obj: unknown,
  ruta: string[],
  validador: (v: unknown) => v is T
): T | undefined {
  let actual: unknown = obj;

  for (const clave of ruta) {
    if (typeof actual !== "object" || actual === null) {
      return undefined;
    }
    const objActual = actual as Record<string, unknown>;
    if (!(clave in objActual)) {
      return undefined;
    }
    actual = objActual[clave];
  }

  return validador(actual) ? actual : undefined;
}

const datos = {
  usuario: {
    perfil: {
      nombre: "Lucas",
      edad: 30
    }
  }
};

const nombre = obtenerRutaSegura(datos, ["usuario", "perfil", "nombre"], (v): v is string => typeof v === "string");
console.log(nombre);  // "Lucas"

const edad = obtenerRutaSegura(datos, ["usuario", "perfil", "edad"], (v): v is number => typeof v === "number");
console.log(edad);  // 30
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- unknown es la alternativa segura a any: representa un valor desconocido
  que requiere verificación de tipo antes de usarlo.
- A diferencia de any, unknown no permite ninguna operación sin type
  narrowing previo (typeof, instanceof, type guards).
- Las técnicas principales de type narrowing son: typeof, instanceof,
  verificación de propiedades con "in", y type guards personalizados.
- unknown es ideal para APIs externas, JSON.parse, entrada de usuario
  y cualquier fuente de datos no confiable.
- unknown es asignable solo a los tipos any y unknown; a diferencia de
  any que es asignable a cualquier tipo.
- Los type guards personalizados (valor is Tipo) permiten transformar
  un unknown en un tipo específico de forma segura.
- Usa unknown siempre que no puedas determinar el tipo exacto en tiempo
  de compilación. Nunca defaults a any sin antes considerar unknown.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué debería usar unknown en lugar de any?**
Porque unknown te obliga a verificar el tipo antes de usar el valor,
eliminando errores silenciosos en runtime. any desactiva completamente
el sistema de tipos, reintroduciendo los mismos problemas que TypeScript
busca prevenir. Como regla general: nunca uses any donde unknown pueda
funcionar.

❓ **¿Puedo asignar un valor unknown a una variable de otro tipo?**
No directamente. Si tienes `let valor: unknown = "texto"`, no puedes
asignarlo a `let texto: string = valor`. Debes usar type narrowing
primero: `if (typeof valor === "string") { texto = valor; }`.

❓ **¿Cómo valido que un objeto unknown tiene una estructura específica?**
Usa type guards personalizados. Verifica que el valor sea un objeto no
nulo, luego accede a sus propiedades usando type assertions y valida
cada una con typeof u otras verificaciones. El patrón típico es:
`if (typeof v !== "object" || v === null) return false;`

❓ **¿Funciona unknown con clases y instanceof?**
Sí, instanceof funciona perfectamente con unknown. Si tienes una clase
Persona, puedes hacer: `if (valor instanceof Persona) { /* valor: Persona */ }`.
Esto es especialmente útil para validar instancias de clases.
