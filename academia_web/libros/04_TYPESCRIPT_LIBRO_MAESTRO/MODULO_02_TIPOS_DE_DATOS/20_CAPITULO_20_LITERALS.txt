1. INTRODUCCIÓN A LOS TIPOS LITERALES
===============================================================================

Un tipo literal es un tipo que representa exactamente un valor específico,
no una categoría de valores. Usando un valor como tipo, dices: "esto solo
puede ser exactamente este valor".

```typescript
// Tipos literales de string
let direccion: "norte";
direccion = "norte";  // ✅ OK
// direccion = "sur"; // ❌ Error: Type '"sur"' is not assignable to type '"norte"'

// Tipos literales de number
let respuesta: 42;
respuesta = 42;   // ✅ OK
// respuesta = 43; // ❌ Error

// Tipos literales de boolean
let verdad: true;
verdad = true;   // ✅ OK
// verdad = false; // ❌ Error
```

Los tipos literales por sí solos son limitados. Su verdadero poder surge
cuando los combinas con union types para crear conjuntos de valores
permitidos.

---

📌 **Combinando literales con uniones**

```typescript
type Direccion = "norte" | "sur" | "este" | "oeste";
type RespuestaHTTP = 200 | 201 | 400 | 404 | 500;
type BooleanoExtendido = true | false | "tal vez";

function mover(direccion: Direccion, pasos: number): void {
  console.log(`Moviendo ${pasos} pasos al ${direccion}`);
}

mover("norte", 5);  // ✅ OK
// mover("noroeste", 3); // ❌ Error
```

---

🔵 **Ejemplo 1: Tipos literales básicos**

```typescript
type EstadoCivil = "soltero" | "casado" | "divorciado" | "viudo";
type DiaSemana = 1 | 2 | 3 | 4 | 5 | 6 | 7;

let estado: EstadoCivil = "soltero";
let dia: DiaSemana = 3;  // Miércoles

function esFinDeSemana(dia: DiaSemana): boolean {
  return dia === 6 || dia === 7;
}

console.log(esFinDeSemana(1));  // false
console.log(esFinDeSemana(7));  // true

// Los literales boolean son menos comunes pero existen
type SiNo = true | false | "quizas";

function responder(respuesta: SiNo): string {
  if (respuesta === true) return "Sí";
  if (respuesta === false) return "No";
  return "Quizás";
}
```

🔵 **Ejemplo 2: Tipos literales en funciones**

```typescript
// Parámetros con valores exactos
function configurarModo(modo: "oscuro" | "claro" | "auto"): void {
  console.log(`Modo configurado: ${modo}`);
  // Aplicar configuración
}

configurarModo("oscuro");  // ✅
configurarModo("claro");   // ✅
// configurarModo("rojo"); // ❌ Error

// Retorno de tipo literal
function obtenerEstadoServidor(): "online" | "offline" | "mantenimiento" {
  return "online";
}

// Overloads con literales
function transformar(texto: string, formato: "mayusculas"): string;
function transformar(texto: string, formato: "minusculas"): string;
function transformar(texto: string, formato: "capitalizar"): string;
function transformar(texto: string, formato: string): string {
  switch (formato) {
    case "mayusculas": return texto.toUpperCase();
    case "minusculas": return texto.toLowerCase();
    case "capitalizar":
      return texto.charAt(0).toUpperCase() + texto.slice(1).toLowerCase();
    default:
      return texto;
  }
}

console.log(transformar("hola mundo", "mayusculas"));  // "HOLA MUNDO"
console.log(transformar("HOLA MUNDO", "minusculas"));  // "hola mundo"
console.log(transformar("hOLA mUNDO", "capitalizar")); // "Hola mundo"
```

2. TIPOS LITERALES EN OBJETOS E INTERFACES
===============================================================================

Las propiedades literales en objetos e interfaces permiten modelar
dominios con precisión quirúrgica.

🔵 **Ejemplo 3: Propiedades literales en interfaces**

```typescript
interface ConfiguracionApp {
  tema: "claro" | "oscuro" | "contraste";
  idioma: "es" | "en" | "fr" | "de" | "pt";
  notificaciones: "todas" | "importantes" | "ninguna";
  autoGuardado: 5 | 10 | 15 | 30 | 60;  // minutos
}

function aplicarConfiguracion(config: ConfiguracionApp): void {
  console.log(`Tema: ${config.tema}`);
  console.log(`Idioma: ${config.idioma}`);
  console.log(`Notificaciones: ${config.notificaciones}`);
  console.log(`Auto-guardado cada ${config.autoGuardado} min`);
}

const configValida: ConfiguracionApp = {
  tema: "oscuro",
  idioma: "es",
  notificaciones: "importantes",
  autoGuardado: 15
};

aplicarConfiguracion(configValida);
```

🔵 **Ejemplo 4: Literales para estados de UI**

```typescript
type EstadoBoton = "normal" | "hover" | "active" | "disabled" | "loading";
type TamanoBoton = "xs" | "sm" | "md" | "lg" | "xl";
type VarianteBoton = "primario" | "secundario" | "peligro" | "exito" | "warning";

interface BotonProps {
  estado?: EstadoBoton;
  tamano?: TamanoBoton;
  variante: VarianteBoton;
  etiqueta: string;
  icono?: string;
  deshabilitado?: boolean;
}

function renderizarBoton(props: BotonProps): string {
  const clases = [
    `btn-${props.variante}`,
    `btn-${props.tamano ?? "md"}`,
    props.estado === "disabled" ? "btn-disabled" : ""
  ].filter(Boolean).join(" ");

  return `<button class="${clases}">${props.etiqueta}</button>`;
}

const boton: BotonProps = {
  variante: "primario",
  tamano: "lg",
  etiqueta: "Guardar",
  estado: "loading"
};

console.log(renderizarBoton(boton));
```

🔵 **Ejemplo 5: Discriminated unions con tipos literales**

```typescript
type Pago =
  | { metodo: "tarjeta"; numero: string; cvv: string; vencimiento: string }
  | { metodo: "paypal"; email: string }
  | { metodo: "transferencia"; banco: string; cuenta: string }
  | { metodo: "cripto"; direccion: string; moneda: "BTC" | "ETH" | "USDT" };

function procesarPago(pago: Pago): string {
  switch (pago.metodo) {
    case "tarjeta":
      return `Pagando con tarjeta ****${pago.numero.slice(-4)}`;
    case "paypal":
      return `Pagando con PayPal (${pago.email})`;
    case "transferencia":
      return `Transferencia bancaria: ${pago.banco}`;
    case "cripto":
      return `Pagando ${pago.moneda} desde ${pago.direccion.slice(0, 6)}...`;
    default:
      const _exhaustivo: never = pago;
      return _exhaustivo;
  }
}
```

===============================================================================
3. CONST ASSERTIONS (AS CONST)
===============================================================================

Las const assertions (as const) le indican a TypeScript que infiera el
tipo más estricto posible para un valor, incluyendo tipos literales.

---

📌 **as const convierte valores en tipos literales**

```typescript
// Sin as const: TypeScript infiere tipos amplios
let a = { nombre: "Lucas", edad: 30 };
// typeof a: { nombre: string; edad: number }

// Con as const: TypeScript infiere tipos literales
let b = { nombre: "Lucas", edad: 30 } as const;
// typeof b: { readonly nombre: "Lucas"; readonly edad: 30 }

// Arrays con as const
let c = [1, 2, 3] as const;
// typeof c: readonly [1, 2, 3] (tupla literal)
```

---

🔵 **Ejemplo 6: as const con objetos**

```typescript
// Sin as const
const config1 = {
  url: "https://api.example.com",
  puerto: 443,
  ssl: true
};
// typeof: { url: string; puerto: number; ssl: boolean }

// Con as const
const config2 = {
  url: "https://api.example.com",
  puerto: 443,
  ssl: true
} as const;
// typeof: { readonly url: "https://api.example.com"; readonly puerto: 443; readonly ssl: true }

// Beneficio: no puedes reasignar accidentalmente
// config2.url = "otro";  // ❌ Error: read-only

// Útil para valores constantes
const COLORES = {
  primario: "#FF0000",
  secundario: "#00FF00",
  fondo: "#FFFFFF",
  texto: "#000000"
} as const;

type Color = typeof COLORES[keyof typeof COLORES];
// Color = "#FF0000" | "#00FF00" | "#FFFFFF" | "#000000"
```

🔵 **Ejemplo 7: as const con arrays**

```typescript
// Sin as const: el array es mutable y los tipos son amplios
const frutas1 = ["manzana", "pera", "uva"];
// typeof: string[]

// Con as const: el array es readonly y los valores son literales
const frutas2 = ["manzana", "pera", "uva"] as const;
// typeof: readonly ["manzana", "pera", "uva"]

type Fruta = typeof frutas2[number];
// Fruta = "manzana" | "pera" | "uva"

// Extraer tipo de un elemento específico
type PrimeraFruta = typeof frutas2[0];
// PrimeraFruta = "manzana"

// Ejemplo práctico: enumeraciones sin enum
const DIRECCIONES = ["norte", "sur", "este", "oeste"] as const;
type Direccion = typeof DIRECCIONES[number];

function mover(d: Direccion): void {
  console.log(`Moviendo al ${d}`);
}

mover("norte");  // ✅
// mover("noroeste");  // ❌ Error
```

🔵 **Ejemplo 8: Combinando as const con funciones**

```typescript
// Extraer tipos de constantes para parámetros de funciones
const NIVELES_ACCESO = ["lectura", "escritura", "admin", "superadmin"] as const;
type NivelAcceso = typeof NIVELES_ACCESO[number];

const ROLES_USUARIO = ["invitado", "usuario", "moderador", "admin"] as const;
type RolUsuario = typeof ROLES_USUARIO[number];

function asignarPermiso(usuario: string, rol: RolUsuario, nivel: NivelAcceso): void {
  console.log(`${usuario} tiene rol ${rol} con nivel ${nivel}`);
}

asignarPermiso("Lucas", "admin", "escritura");

// También funciona con objetos anidados
const PLANES_SUSCRIPCION = {
  basico: { precio: 9.99, almacenamiento: "5GB" },
  pro: { precio: 19.99, almacenamiento: "50GB" },
  enterprise: { precio: 99.99, almacenamiento: "1TB" }
} as const;

type Plan = keyof typeof PLANES_SUSCRIPCION;
// Plan = "basico" | "pro" | "enterprise"
```

🔵 **Ejemplo 9: Sistema de notificaciones con literales**

```typescript
// Constantes del sistema
const CANALES_NOTIFICACION = ["email", "sms", "push", "inApp"] as const;
type CanalNotificacion = typeof CANALES_NOTIFICACION[number];

const PRIORIDADES = ["baja", "media", "alta", "critica"] as const;
type Prioridad = typeof PRIORIDADES[number];

const ESTADOS_NOTIFICACION = ["pendiente", "enviando", "entregada", "fallida"] as const;
type EstadoNotificacion = typeof ESTADOS_NOTIFICACION[number];

// Configuración de canales
const CONFIG_CANALES = {
  email: { maxDestinatarios: 50, permitirHTML: true, tasaLimite: 100 },
  sms: { maxCaracteres: 160, permitirHTML: false, tasaLimite: 10 },
  push: { maxTitulo: 50, maxCuerpo: 200, tasaLimite: 500 },
  inApp: { maxTitulo: 100, maxCuerpo: 500, tasaLimite: 1000 }
} as const;

// Sistema de notificaciones
interface Notificacion {
  id: string;
  canal: CanalNotificacion;
  destino: string;
  titulo: string;
  cuerpo: string;
  prioridad: Prioridad;
  estado: EstadoNotificacion;
}

function validarNotificacion(notif: Notificacion): string[] {
  const errores: string[] = [];
  const config = CONFIG_CANALES[notif.canal];

  if (!CANALES_NOTIFICACION.includes(notif.canal)) {
    errores.push(`Canal inválido: ${notif.canal}`);
  }

  if (notif.titulo.length > config.maxTitulo) {
    errores.push(`Título muy largo (máx ${config.maxTitulo})`);
  }

  if (notif.canal === "sms" && notif.cuerpo.length > config.maxCaracteres) {
    errores.push(`SMS muy largo (máx ${config.maxCaracteres} caracteres)`);
  }

  if (!PRIORIDADES.includes(notif.prioridad)) {
    errores.push(`Prioridad inválida: ${notif.prioridad}`);
  }

  return errores;
}

function enviarNotificacion(notif: Notificacion): EstadoNotificacion {
  const errores = validarNotificacion(notif);
  if (errores.length > 0) {
    console.error("Errores de validación:", errores);
    return "fallida";
  }

  console.log(`Enviando ${notif.canal} a ${notif.destino}: ${notif.titulo}`);
  return "entregada";
}

// Uso
const notificacion: Notificacion = {
  id: "notif-001",
  canal: "email",
  destino: "usuario@email.com",
  titulo: "Bienvenido",
  cuerpo: "Gracias por registrarte",
  prioridad: "alta",
  estado: "pendiente"
};

const resultado = enviarNotificacion(notificacion);
console.log(`Estado final: ${resultado}`);
```

🔵 **Ejemplo 10: Proyecto final — API de configuración tipada**

```typescript
// Sistema completo de configuración de aplicación

const ENTORNOS = ["desarrollo", "pruebas", "produccion"] as const;
type Entorno = typeof ENTORNOS[number];

const NIVELES_LOG = ["debug", "info", "warn", "error"] as const;
type NivelLog = typeof NIVELES_LOG[number];

const BASES_DATOS = ["postgresql", "mysql", "sqlite", "mongodb"] as const;
type BaseDatos = typeof BASES_DATOS[number];

interface ConfigBaseDatos {
  motor: BaseDatos;
  host: string;
  puerto: number;
  nombre: string;
  usuario: string;
  password: string;
  pool: 5 | 10 | 20 | 50;
  ssl: boolean;
}

interface ConfigServidor {
  puerto: number;
  host: string;
  cors: boolean;
  rateLimit: 100 | 500 | 1000 | 5000;
}

interface ConfigAppCompleta {
  nombre: string;
  version: string;
  entorno: Entorno;
  servidor: ConfigServidor;
  baseDatos: ConfigBaseDatos;
  logging: {
    nivel: NivelLog;
    archivo: boolean;
    consola: boolean;
    formato: "json" | "texto" | "simple";
  };
  caracteristicas: {
    autenticacion: boolean;
    cache: boolean;
    colaMensajes: boolean;
    webSockets: boolean;
  };
}

// Valores por defecto según entorno
const CONFIG_POR_DEFECTO: Record<Entorno, Partial<ConfigAppCompleta>> = {
  desarrollo: {
    servidor: { puerto: 3000, host: "localhost", cors: true, rateLimit: 5000 },
    logging: { nivel: "debug", archivo: false, consola: true, formato: "simple" }
  },
  pruebas: {
    servidor: { puerto: 4000, host: "localhost", cors: true, rateLimit: 1000 },
    logging: { nivel: "info", archivo: true, consola: true, formato: "json" }
  },
  produccion: {
    servidor: { puerto: 80, host: "0.0.0.0", cors: false, rateLimit: 100 },
    logging: { nivel: "warn", archivo: true, consola: false, formato: "json" }
  }
} as const;

function crearConfiguracion(entorno: Entorno, override?: Partial<ConfigAppCompleta>): ConfigAppCompleta {
  const defaults = CONFIG_POR_DEFECTO[entorno];

  const config: ConfigAppCompleta = {
    nombre: "MiApp",
    version: "1.0.0",
    entorno,
    servidor: {
      puerto: 3000,
      host: "localhost",
      cors: false,
      rateLimit: 1000,
      ...defaults.servidor,
      ...override?.servidor
    },
    baseDatos: {
      motor: "postgresql",
      host: "localhost",
      puerto: 5432,
      nombre: "miapp",
      usuario: "admin",
      password: "secreto",
      pool: 10,
      ssl: true,
      ...override?.baseDatos
    },
    logging: {
      nivel: "info",
      archivo: true,
      consola: true,
      formato: "json",
      ...defaults.logging,
      ...override?.logging
    },
    caracteristicas: {
      autenticacion: true,
      cache: true,
      colaMensajes: false,
      webSockets: false,
      ...override?.caracteristicas
    }
  };

  return config;
}

function validarConfiguracion(config: ConfigAppCompleta): boolean {
  if (!ENTORNOS.includes(config.entorno)) {
    console.error(`Entorno inválido: ${config.entorno}`);
    return false;
  }

  if (!NIVELES_LOG.includes(config.logging.nivel)) {
    console.error(`Nivel de log inválido: ${config.logging.nivel}`);
    return false;
  }

  if (!BASES_DATOS.includes(config.baseDatos.motor)) {
    console.error(`Motor de BD inválido: ${config.baseDatos.motor}`);
    return false;
  }

  return true;
}

// Inicializar aplicación
function iniciarAplicacion(entorno: Entorno): void {
  console.log(`=== Iniciando aplicación en entorno: ${entorno} ===`);

  const config = crearConfiguracion(entorno, {
    nombre: "TypeScriptMasterApp",
    baseDatos: { motor: "postgresql", pool: 20 }
  });

  if (!validarConfiguracion(config)) {
    console.error("Configuración inválida. Abortando.");
    process.exit(1);
  }

  console.log(`Aplicación "${config.nombre}" v${config.version} iniciada`);
  console.log(`Servidor en ${config.servidor.host}:${config.servidor.puerto}`);
  console.log(`Base de datos: ${config.baseDatos.motor}`);
  console.log(`Log nivel: ${config.logging.nivel}`);
}

// Ejecutar
const entornoActual: Entorno = "desarrollo";
iniciarAplicacion(entornoActual);
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Los tipos literales representan valores exactos (no categorías): "norte",
  42, true, etc.
- Combinados con union types, crean conjuntos precisos de valores
  permitidos: type Direccion = "norte" | "sur" | "este" | "oeste".
- Los tipos literales son ideales para modelar dominios con opciones
  limitadas: estados, configuraciones, roles, etc.
- Las const assertions (as const) le indican a TypeScript que infiera
  el tipo literal más estricto para un valor constante, incluyendo
  objetos y arrays completos.
- as const convierte objetos en readonly con valores literales y
  arrays en tuplas readonly de tipos literales.
- Los tipos literales mejoran la auto-documentación del código, reducen
  errores en runtime y habilitan el autocompletado en editores.
- Combinados con discriminated unions, crean patrones de diseño
  extremadamente seguros y expresivos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre un tipo literal y un enum?**
Los tipos literales son más flexibles y ligeros que los enum. Mientras
que los enums crean un nuevo tipo con un objeto en runtime, los tipos
literales son puramente en tiempo de compilación. Los tipos literales
también se integran mejor con el sistema de tipos existente y son más
fáciles de combinar con uniones y genéricos.

❓ **¿Debo usar as const en todas mis constantes?**
as const es útil cuando necesitas que TypeScript preserve los valores
exactos para usarlos como tipos. Es especialmente útil para constantes
que definen conjuntos de valores permitidos (como arrays de opciones)
o configuraciones inmutables. No es necesario para constantes simples
donde el tipo amplio es suficiente.

❓ **¿Puedo mezclar tipos literales con tipos normales?**
Sí, y es una práctica común. Por ejemplo:
`type Resultado = "éxito" | "error" | string;` — aunque en este caso
"éxito" y "error" ya están incluidos en string, el tipo literal los
hace explícitos para el autocompletado y la documentación.

❓ **¿Los tipos literales afectan el rendimiento?**
No. Los tipos literales son puramente una característica de tiempo de
compilación. No generan código adicional en JavaScript y no tienen
ningún impacto en el rendimiento en runtime. Son completamente gratis
en términos de rendimiento.
