1. INTRODUCCIÓN AL TIPO NUMBER
===============================================================================

El tipo number en TypeScript representa tanto números enteros como decimales
(coma flotante). Internamente, todos los números siguen el estándar IEEE 754
de doble precisión (64 bits).

TypeScript infiere el tipo number automáticamente cuando asignas un valor
numérico a una variable.

```typescript
let edad = 25;          // type: number (inferido)
let precio = 19.99;     // type: number (inferido)
let negativo = -10;     // type: number (inferido)

// Declaración explícita
let cantidad: number = 100;
let altura: number = 1.75;
```

No hay diferencia sintáctica entre enteros y flotantes en TypeScript. Ambos
son del mismo tipo number. Esto simplifica el lenguaje, pero requiere que
como desarrollador seas consciente de las limitaciones de la aritmética de
punto flotante.

---

📌 **Los números en TypeScript siguen el estándar IEEE 754**

Esto significa que ciertos decimales no pueden representarse con precisión
absoluta. Por ejemplo, 0.1 + 0.2 no es exactamente 0.3.

```typescript
console.log(0.1 + 0.2); // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3); // false
```

Para comparaciones precisas usa una tolerancia (epsilon):

```typescript
function sonIguales(a: number, b: number, epsilon = 1e-10): boolean {
  return Math.abs(a - b) < epsilon;
}

console.log(sonIguales(0.1 + 0.2, 0.3)); // true
```

---

🔵 **Ejemplo 1: Declaración y uso básico**

```typescript
// Números enteros
let entero: number = 42;
let binario: number = 0b1010;   // 10 en decimal
let octal: number = 0o52;       // 42 en decimal
let hex: number = 0x2A;         // 42 en decimal

// Números decimales
let decimal: number = 3.1416;
let notacionCientifica: number = 1.5e3; // 1500

console.log(entero);               // 42
console.log(binario);              // 10
console.log(hex);                  // 42
console.log(decimal);              // 3.1416
console.log(notacionCientifica);   // 1500
```

🔵 **Ejemplo 2: Separadores numéricos**

TypeScript (ES2021+) permite usar guiones bajos para mejorar la legibilidad
de números grandes.

```typescript
let millones: number = 1_000_000;
let presupuesto: number = 45_678_912.50;
let binarioLegible: number = 0b1010_0011_1101;
let hexLegible: number = 0xFF_EE_DD_CC;

console.log(millones);      // 1000000
console.log(presupuesto);   // 45678912.5
```

Los separadores no afectan el valor numérico, solo mejoran la lectura.

🔵 **Ejemplo 3: typeof y tipo inferido**

```typescript
let x = 10;
let y = 3.14;
let z = NaN;

console.log(typeof x); // "number"
console.log(typeof y); // "number"
console.log(typeof z); // "number" (¡NaN sigue siendo number!)

// TypeScript infiere el tipo automáticamente
// let x: number
// let y: number
```

===============================================================================
2. ENTEROS Y FLOTANTES EN TYPESCRIPT
===============================================================================

Aunque TypeScript no tiene tipos separados para enteros y flotantes, existen
técnicas para trabajar con cada uno de forma específica.

---

📌 **TypeScript no tiene int ni float como tipos separados**

Todo es number. Sin embargo, puedes usar type aliases para documentar
tu intención:

```typescript
type Entero = number;
type Flotante = number;

let edad: Entero = 25;
let precio: Flotante = 19.99;
```

Estos alias no imponen validación en tiempo de compilación, pero mejoran
la legibilidad del código.

---

🔵 **Ejemplo 4: Verificar si un número es entero**

```typescript
function esEntero(valor: number): boolean {
  return Number.isInteger(valor);
}

console.log(esEntero(10));     // true
console.log(esEntero(10.5));   // false
console.log(esEntero(NaN));    // false
console.log(esEntero(Infinity)); // false
```

🔵 **Ejemplo 5: Redondeo y truncado**

```typescript
let valor = 3.14159;

console.log(Math.round(valor));    // 3
console.log(Math.floor(valor));    // 3
console.log(Math.ceil(valor));     // 4
console.log(Math.trunc(valor));    // 3 (elimina decimales)
console.log(valor.toFixed(2));     // "3.14" (devuelve string)
console.log(parseInt("42"));       // 42
console.log(parseFloat("3.14"));   // 3.14
```

🔵 **Ejemplo 6: Rango seguro de enteros**

JavaScript puede representar enteros de forma segura entre -(2⁵³-1) y
(2⁵³-1). Fuera de este rango, la precisión se pierde.

```typescript
console.log(Number.MAX_SAFE_INTEGER);  // 9007199254740991
console.log(Number.MIN_SAFE_INTEGER);  // -9007199254740991

let seguro = 9007199254740991;
let inseguro = 9007199254740993;

console.log(seguro + 1 === seguro + 2);  // true (¡pérdida de precisión!)

function esEnteroSeguro(valor: number): boolean {
  return Number.isSafeInteger(valor);
}

console.log(esEnteroSeguro(9007199254740991));  // true
console.log(esEnteroSeguro(9007199254740993));  // false
```

===============================================================================
3. VALORES ESPECIALES: NAN E INFINITY
===============================================================================

El tipo number incluye valores especiales que representan situaciones
matemáticas excepcionales. Es crucial entenderlos para evitar errores.

---

📌 **NaN (Not a Number) es el único valor en JavaScript que no es igual
a sí mismo**

```typescript
console.log(NaN === NaN); // false

// La forma correcta de verificar NaN
console.log(Number.isNaN(NaN));  // true
console.log(isNaN("texto"));     // true (coerción)
console.log(Number.isNaN("texto")); // false (sin coerción)
```

---

🔵 **Ejemplo 7: Operaciones que producen NaN**

```typescript
let resultado1 = 0 / 0;                // NaN
let resultado2 = Math.sqrt(-1);        // NaN
let resultado3 = parseInt("hola");     // NaN
let resultado4 = undefined + 1;        // NaN
let resultado5 = NaN + 5;              // NaN (contagio)

console.log(resultado1);  // NaN
console.log(resultado2);  // NaN
console.log(resultado3);  // NaN
console.log(resultado4);  // NaN
console.log(resultado5);  // NaN
```

🔵 **Ejemplo 8: Infinity y -Infinity**

```typescript
let infinitoPositivo = Infinity;
let infinitoNegativo = -Infinity;

let div1 = 1 / 0;           // Infinity
let div2 = -1 / 0;          // -Infinity
let max = Number.MAX_VALUE * 2; // Infinity

console.log(infinitoPositivo);  // Infinity
console.log(infinitoNegativo);  // -Infinity
console.log(div1);              // Infinity
console.log(div2);              // -Infinity
console.log(max);               // Infinity

// Operaciones con Infinity
console.log(Infinity + 1);      // Infinity
console.log(Infinity * 0);      // NaN
console.log(Infinity / Infinity); // NaN
console.log(isFinite(Infinity)); // false
console.log(isFinite(42));       // true
```

🔵 **Ejemplo 9: Función segura para división**

```typescript
function dividirSeguro(a: number, b: number): number {
  if (b === 0) {
    console.warn("División por cero detectada");
    return 0;
  }
  let resultado = a / b;
  if (!Number.isFinite(resultado)) {
    console.warn("Resultado no finito");
    return 0;
  }
  return resultado;
}

console.log(dividirSeguro(10, 2));  // 5
console.log(dividirSeguro(10, 0));  // 0 (con warning)
console.log(dividirSeguro(Infinity, 2)); // 0 (con warning)
```

4. COMPARACIÓN ENTRE NUMBER Y BIGINT
===============================================================================

TypeScript introdujo el tipo bigint para representar enteros de tamaño
arbitrario. Aunque es un tipo diferente, es importante entender cómo se
relaciona con number.

---

📌 **No puedes mezclar number y bigint en operaciones aritméticas**

```typescript
let num: number = 42;
let big: bigint = 42n;

// Error: Operator '+' cannot be applied to types 'number' and 'bigint'
// let suma = num + big;
```

Debes convertir explícitamente entre tipos.

---

🔵 **Ejemplo 10: Comparación de valores**

```typescript
let numero: number = 42;
let grande: bigint = 42n;

// Comparación abstracta (==) permite comparar tipos diferentes
console.log(numero == grande);   // true (compara valor)

// Comparación estricta (===) revisa tipo y valor
console.log(numero === grande);  // false (tipos diferentes)

// Conversión explícita
console.log(BigInt(numero) === grande);  // true
console.log(Number(grande) === numero);  // true
```

🔵 **Ejemplo 11: Cuándo usar bigint en lugar de number**

```typescript
// Números que exceden MAX_SAFE_INTEGER
let numeroInseguro: number = 9007199254740993;
let bigintSeguro: bigint = 9007199254740993n;

console.log(numeroInseguro);           // 9007199254740992 (pérdida)
console.log(bigintSeguro);             // 9007199254740993n (preciso)

// Operaciones aritméticas con bigint
let a = 1000000000000000000n;
let b = 2000000000000000000n;
let suma = a + b;
console.log(suma);  // 3000000000000000000n

// División con bigint trunca a entero
console.log(10n / 3n);  // 3n
```

🔵 **Ejemplo 12: Type guards para number y bigint**

```typescript
function procesarValor(valor: number | bigint): string {
  if (typeof valor === "number") {
    return `Número: ${valor.toFixed(2)}`;
  } else {
    return `BigInt: ${valor.toString()}`;
  }
}

console.log(procesarValor(3.1415));  // "Número: 3.14"
console.log(procesarValor(42n));     // "BigInt: 42"
```

5. OPERACIONES MATEMÁTICAS Y MÉTODOS ÚTILES
===============================================================================

TypeScript expone el objeto Math con funciones matemáticas y Number con
propiedades y métodos útiles para trabajar con números.

---

📌 **El objeto Math y Number son tus aliados**

```typescript
// Propiedades de Number
console.log(Number.EPSILON);          // 2.220446049250313e-16
console.log(Number.MAX_VALUE);        // 1.7976931348623157e+308
console.log(Number.MIN_VALUE);        // 5e-324

// Métodos estáticos de Number
console.log(Number.isInteger(10));    // true
console.log(Number.isNaN(NaN));       // true
console.log(Number.isFinite(Infinity)); // false
console.log(Number.parseFloat("3.14")); // 3.14
console.log(Number.parseInt("42"));   // 42
```

---

🔵 **Ejemplo 13: Métodos del objeto Math**

```typescript
let angulo = Math.PI / 4; // 45 grados en radianes

console.log(Math.abs(-5));          // 5
console.log(Math.pow(2, 10));       // 1024
console.log(Math.sqrt(25));         // 5
console.log(Math.sin(angulo));      // 0.707...
console.log(Math.cos(angulo));      // 0.707...
console.log(Math.tan(angulo));      // 1.0
console.log(Math.log(10));          // Logaritmo natural
console.log(Math.log10(100));       // Logaritmo base 10 = 2
console.log(Math.max(1, 5, 3, 9));  // 9
console.log(Math.min(1, 5, 3, 9));  // 1
console.log(Math.random());         // Aleatorio entre 0 y 1
```

🔵 **Ejemplo 14: Números aleatorios en rangos específicos**

```typescript
function aleatorioEntre(min: number, max: number): number {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

function aleatorioDecimal(min: number, max: number): number {
  return Math.random() * (max - min) + min;
}

console.log(aleatorioEntre(1, 6));         // Simula un dado
console.log(aleatorioDecimal(0, 100));     // 0.00 a 99.99
```

🔵 **Ejemplo 15: Formateo de números**

```typescript
let ganancia = 1234567.8901;

// toFixed - decimales fijos
console.log(ganancia.toFixed(2));        // "1234567.89"

// toPrecision - cifras significativas
console.log(ganancia.toPrecision(4));    // "1.235e+6"

// toLocaleString - formato local
console.log(ganancia.toLocaleString("es-ES")); // "1.234.567,89"
console.log(ganancia.toLocaleString("en-US")); // "1,234,567.89"

// Formato moneda
console.log(ganancia.toLocaleString("es-ES", {
  style: "currency",
  currency: "EUR"
})); // "1.234.567,89 €"
```

6. BUENAS PRÁCTICAS Y ERRORES COMUNES
===============================================================================

Trabajar con números en TypeScript requiere atención a ciertos detalles
para evitar bugs sutiles.

🔵 **Ejemplo 16: Errores comunes con punto flotante**

```typescript
// Problema: suma de decimales imprecisa
let total = 0.0;
for (let i = 0; i < 10; i++) {
  total += 0.1;
}
console.log(total);          // 0.9999999999999999
console.log(total === 1.0);  // false

// Solución: trabajar con enteros (centavos)
function calcularTotalCentavos(preciosCentavos: number[]): number {
  return preciosCentavos.reduce((a, b) => a + b, 0);
}

let preciosCentavos = [100, 50, 75]; // $1.00, $0.50, $0.75
console.log(calcularTotalCentavos(preciosCentavos)); // 225 centavos = $2.25
```

🔵 **Ejemplo 17: Validación de entrada numérica**

```typescript
function procesarEdad(input: string): number | null {
  let edad = Number(input);

  if (input.trim() === "" || Number.isNaN(edad)) {
    console.error("Entrada inválida: no es un número");
    return null;
  }

  if (!Number.isInteger(edad)) {
    console.error("La edad debe ser un número entero");
    return null;
  }

  if (edad < 0 || edad > 150) {
    console.error("Edad fuera de rango válido");
    return null;
  }

  return edad;
}

console.log(procesarEdad("25"));    // 25
console.log(procesarEdad("abc"));   // null (NaN)
console.log(procesarEdad("200"));   // null (fuera de rango)
```

🔵 **Ejemplo 18: Tipos literales numéricos**

TypeScript permite definir tipos literales con números específicos.

```typescript
type Edad = 0 | 1 | 2 | 3 | 4 | 5;
type Porcentaje = 0 | 25 | 50 | 75 | 100;

function avanzarNivel(experiencia: Porcentaje): void {
  console.log(`Progreso: ${experiencia}%`);
}

avanzarNivel(50);  // OK
// avanzarNivel(33); // Error: Type '33' is not assignable to type 'Porcentaje'
```

RESUMEN DEL CAPÍTULO
===============================================================================

- El tipo number en TypeScript representa tanto enteros como decimales
  siguiendo el estándar IEEE 754 de doble precisión (64 bits).
- No existe distinción entre int y float; todo es number, pero puedes usar
  Number.isInteger() para verificar si un valor es entero.
- NaN es un valor especial que no es igual a sí mismo; usa Number.isNaN()
  para detectarlo de forma segura.
- Infinity representa valores fuera del rango finito; usa isFinite() para
  verificar si un número es finito.
- bigint complementa a number para enteros de tamaño arbitrario, pero no
  pueden mezclarse en operaciones aritméticas sin conversión explícita.
- Los separadores numéricos (_) mejoran la legibilidad de números grandes.
- El rango seguro para enteros es hasta 2⁵³-1; fuera de él, usa bigint.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué 0.1 + 0.2 no es exactamente 0.3 en TypeScript?**
Porque JavaScript (y por tanto TypeScript) sigue el estándar IEEE 754 para
números de punto flotante. Algunos decimales no tienen representación binaria
exacta, lo que causa pequeñas imprecisiones. Usa una tolerancia (epsilon)
para comparar números decimales.

❓ **¿Cuándo debería usar bigint en lugar de number?**
Usa bigint cuando necesites trabajar con enteros mayores a 2⁵³-1
(Number.MAX_SAFE_INTEGER) o cuando requieras precisión absoluta en
operaciones con enteros muy grandes. Para la mayoría de aplicaciones
cotidianas, number es suficiente.

❓ **¿Cómo verifico si una variable es de tipo number?**
Usa `typeof variable === "number"`. Ten en cuenta que NaN también devuelve
"number", así que si necesitas excluir NaN, combínalo con
`!Number.isNaN(variable)`.

❓ **¿Puedo usar parseInt y parseFloat con TypeScript?**
Sí, están disponibles globalmente. TypeScript los tipifica correctamente:
ambos devuelven number. parseInt acepta un segundo parámetro opcional para
la base (radix). Siempre especifica el radix para evitar comportamientos
inesperados: parseInt("42", 10).
