Querido estudiante,

En capítulos anteriores hemos visto funciones tipadas, parámetros opcionales,
y valores por defecto. Ahora es momento de formalizar un concepto central:
la firma de una función (call signature). Una firma describe el tipo completo
de una función: qué parámetros acepta y qué valor retorna. Podemos definir
firmas en interfaces, en type aliases, y usarlas como tipos independientes.
Este conocimiento es esencial para escribir código que reciba y devuelva
funciones correctamente, creando abstracciones poderosas y reutilizables.

1. CALL SIGNATURES EN INTERFACES
===============================================================================

Una call signature define el tipo de una función dentro de una interfaz.
La sintaxis usa `:` en lugar de `=>`:

```typescript
interface OperacionMatematica {
  (a: number, b: number): number;
}

const sumar: OperacionMatematica = (x, y) => x + y;
const restar: OperacionMatematica = (x, y) => x - y;

function ejecutarOperacion(a: number, b: number, op: OperacionMatematica): number {
  return op(a, b);
}

console.log(ejecutarOperacion(10, 5, sumar));  // 15
console.log(ejecutarOperacion(10, 5, restar)); // 5
```

1.1. CALL SIGNATURE VS METHOD SIGNATURE

En interfaces, hay dos formas de declarar métodos:

```typescript
interface Calculadora {
  // Call signature (función sin nombre)
  (a: number, b: number): number;

  // Method signature (propiedad que es función)
  sumar(a: number, b: number): number;
}

// Un objeto que implementa Calculadora debe tener ambas
const calc: Calculadora = Object.assign(
  (a: number, b: number) => a * b, // call signature
  { sumar: (a: number, b: number) => a + b } // método
);

console.log(calc(5, 3));      // 15 (usa la call signature)
console.log(calc.sumar(5, 3)); // 8 (usa el método)
```

===============================================================================
2. FIRMAS EN TYPE ALIASES
===============================================================================

También podemos definir firmas de funciones con `type`:

```typescript
type Comparador<T> = (a: T, b: T) => number;
type Transformador<T, R> = (valor: T) => R;
type Predicado<T> = (valor: T) => boolean;
type Proveedor<T> = () => T;
type Consumidor<T> = (valor: T) => void;

// Uso
const compararNumeros: Comparador<number> = (a, b) => a - b;
const aMayusculas: Transformador<string, string> = (s) => s.toUpperCase();
const esPar: Predicado<number> = (n) => n % 2 === 0;
const obtenerId: Proveedor<number> = () => Math.floor(Math.random() * 1000);
const imprimir: Consumidor<string> = (s) => console.log(s);

console.log([3, 1, 4, 1, 5].sort(compararNumeros)); // [1, 1, 3, 4, 5]
```

2.1. FIRMAS CON PARÁMETROS OPCIONALES

```typescript
type Logger = (mensaje: string, nivel?: "info" | "warn" | "error") => void;

const consolaLogger: Logger = (mensaje, nivel = "info") => {
  const prefix = nivel === "error" ? "❌" : nivel === "warn" ? "⚠️" : "ℹ️";
  console.log(`${prefix} ${mensaje}`);
};

consolaLogger("Inicio del programa");
consolaLogger("Algo salió mal", "error");
```

===============================================================================
3. FIRMAS CON PROPIEDADES
===============================================================================

Una interfaz puede combinar una call signature con propiedades adicionales:

```typescript
interface Contador {
  // Call signature
  (incremento?: number): number;

  // Propiedades
  valor: number;
  reset(): void;
}

function crearContador(valorInicial: number = 0): Contador {
  const contador = ((incremento: number = 1) => {
    contador.valor += incremento;
    return contador.valor;
  }) as Contador;

  contador.valor = valorInicial;
  contador.reset = () => {
    contador.valor = 0;
  };

  return contador;
}

const miContador = crearContador(10);
console.log(miContador());     // 11 (incrementa en 1)
console.log(miContador(5));    // 16 (incrementa en 5)
console.log(miContador.valor); // 16
miContador.reset();
console.log(miContador.valor); // 0
```

===============================================================================
4. FIRMAS CON GENERICOS
===============================================================================

Las firmas genéricas permiten crear funciones flexibles pero seguras:

```typescript
interface Identidad<T> {
  (valor: T): T;
}

const identidad: Identidad<number> = (n) => n;
const identidadStr: Identidad<string> = (s) => s;

console.log(identidad(42));       // 42
console.log(identidadStr("hola")); // "hola"

// Con múltiples genéricos
interface TransformadorPar<T, U> {
  (primero: T, segundo: U): [U, T];
}

const invertir: TransformadorPar<string, number> = (s, n) => [n, s];
console.log(invertir("edad", 28)); // [28, "edad"]
```

4.1. RESTRICCIONES EN FIRMAS GENÉRICAS

```typescript
interface Procesador<T extends { id: number }> {
  (item: T): void;
}

type Entidad = { id: number; nombre: string };

const procesar: Procesador<Entidad> = (item) => {
  console.log(`Procesando entidad ${item.id}: ${item.nombre}`);
};

procesar({ id: 1, nombre: "Ejemplo" });
```

===============================================================================
5. CONSTRUCT SIGNATURES
===============================================================================

Además de las call signatures (para funciones normales), TypeScript tiene
construct signatures para clases/constructores. Se denotan con `new`:

```typescript
interface ConstructorPunto {
  new (x: number, y: number): { x: number; y: number };
}

function crearInstancia(
  Constructor: ConstructorPunto,
  x: number,
  y: number
): { x: number; y: number } {
  return new Constructor(x, y);
}

class Punto {
  constructor(
    public x: number,
    public y: number
  ) {}
}

const punto = crearInstancia(Punto, 3, 4);
console.log(punto); // Punto { x: 3, y: 4 }
```

5.1. COMBINAR CALL Y CONSTRUCT SIGNATURES

```typescript
interface FuncionFecha {
  (timestamp?: number): Date;
  new (timestamp?: number): Date;
}

// Date en JavaScript cumple con esta interfaz
const ahora1: Date = new Date();     // constructor
const ahora2: Date = Date();         // función (aunque deprecated)
```

6. COMPATIBILIDAD DE FIRMAS
===============================================================================

TypeScript verifica la compatibilidad entre firmas de funciones. Las reglas
son:

```typescript
type FirmaEstricta = (x: number) => number;
type FirmaRelajada = (x: number) => number | string;

// Una función más específica es asignable a un tipo más general
const f1: FirmaEstricta = (x) => x * 2;
const f2: FirmaRelajada = f1; // ✅ Una función que retorna number se puede asignar

// Parámetros: menos parámetros es compatible
type FirmaTresParam = (a: number, b: number, c: number) => void;
type FirmaDosParam = (a: number, b: number) => void;

const funcionDosParam: FirmaDosParam = (a, b) => console.log(a, b);
const funcionTresParam: FirmaTresParam = funcionDosParam; // ✅
// Esto funciona porque JavaScript permite ignorar parámetros extra
```

6.1. CONTRAVARIANZA DE PARÁMETROS

```typescript
type TipoAnimal = (animal: { nombre: string }) => void;
type TipoPerro = (perro: { nombre: string; raza: string }) => void;

const fnAnimal: TipoAnimal = (animal) => console.log(animal.nombre);
const fnPerro: TipoPerro = fnAnimal; // ✅ (seguro)

// En modo strictFunctionTypes, los parámetros son contravariantes
// fnAnimal no puede ser asignado a fnPerro porque espera menos propiedades
// const fnPerro2: TipoPerro = fnAnimal; // ❌ con strictFunctionTypes
```

7. CASO PRÁCTICO: SISTEMA DE EVENTOS
===============================================================================

```typescript
type EscuchadorEvento<T> = (datos: T) => void;

interface EmisorEventos {
  <T>(evento: string, datos: T): void;
  on<T>(evento: string, escuchador: EscuchadorEvento<T>): void;
  off<T>(evento: string, escuchador: EscuchadorEvento<T>): void;
}

function crearEmisorEventos(): EmisorEventos {
  const escuchadores = new Map<string, Set<Function>>();

  const emisor = ((evento: string, datos: unknown) => {
    const listeners = escuchadores.get(evento);
    if (listeners) {
      listeners.forEach(fn => fn(datos));
    }
  }) as EmisorEventos;

  emisor.on = (evento, escuchador) => {
    if (!escuchadores.has(evento)) {
      escuchadores.set(evento, new Set());
    }
    escuchadores.get(evento)!.add(escuchador);
  };

  emisor.off = (evento, escuchador) => {
    escuchadores.get(evento)?.delete(escuchador);
  };

  return emisor;
}

const emisor = crearEmisorEventos();

emisor.on<string>("saludo", (datos) => {
  console.log(`Recibido: ${datos}`);
});

emisor("saludo", "¡Hola, mundo!");
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Las call signatures describen el tipo de una función usando `(param: T): R`
- Pueden definirse en interfaces y en type aliases
- Las interfaces pueden combinar call signatures con propiedades y métodos
- Las construct signatures usan `new` para describir constructores
- Los genéricos en firmas crean funciones reutilizables y seguras
- La compatibilidad de firmas sigue reglas de covarianza/contravarianza
- Menos parámetros es compatible con más parámetros (por JS)
- Las firmas son esenciales para tipar callbacks y funciones de orden superior

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre `(x: number) => void` y `(x: number): void`?**
La primera es la sintaxis de tipo de función (arrow). La segunda es la call
signature dentro de una interfaz. Son equivalentes.

❓ **¿Puedo tener overloads en una call signature?**
No directamente en la firma, pero puedes usar sobrecarga de funciones (lo
veremos en el próximo capítulo).

❓ **¿Qué es una construct signature?**
Es una firma que describe cómo se usa una función con `new`. Se declara
como `new (param: T): R`.

❓ **¿Puedo mezclar propiedades y call signatures?**
Sí, en interfaces. Es útil para objetos que son "llamables" pero también
tienen estado, como las funciones con propiedades estáticas.
