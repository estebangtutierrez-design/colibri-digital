===============================================================================
1. SINTAXIS BÁSICA DE TYPE ALIASES
===============================================================================

Un type alias crea un nombre para cualquier tipo:

```typescript
type ID = number;
type Nombre = string;
type Activo = boolean;

let usuarioId: ID = 123;
let nombreUsuario: Nombre = "Ana";
let estaActivo: Activo = true;
```

1.1. TIPOS DE OBJETO CON TYPE

```typescript
type Punto = {
  x: number;
  y: number;
};

const origen: Punto = { x: 0, y: 0 };

function distancia(p1: Punto, p2: Punto): number {
  const dx = p2.x - p1.x;
  const dy = p2.y - p1.y;
  return Math.sqrt(dx * dx + dy * dy);
}

console.log(distancia({ x: 0, y: 0 }, { x: 3, y: 4 })); // 5
```

===============================================================================
2. TYPE ALIASES PARA UNIONES
===============================================================================

Una de las ventajas más importantes de type sobre interface es la capacidad
de crear uniones:

```typescript
type Status = "activo" | "inactivo" | "pendiente";
type Resultado = "éxito" | "error" | "cancelado";
type Id = string | number;

function procesarEstado(estado: Status): string {
  switch (estado) {
    case "activo":
      return "El usuario está activo";
    case "inactivo":
      return "El usuario está inactivo";
    case "pendiente":
      return "El usuario está pendiente de verificación";
  }
}

console.log(procesarEstado("activo")); // "El usuario está activo"

let id: Id = "usr_001";
id = 42; // ✅ Válido
```

2.1. UNIONES DE OBJETOS (DISCRIMINATED UNIONS)

```typescript
type Exitoso = {
  estado: "éxito";
  datos: string[];
};

type ErrorApi = {
  estado: "error";
  mensaje: string;
};

type RespuestaApi = Exitoso | ErrorApi;

function manejarRespuesta(respuesta: RespuestaApi): void {
  switch (respuesta.estado) {
    case "éxito":
      console.log(`Datos: ${respuesta.datos.join(", ")}`);
      break;
    case "error":
      console.error(`Error: ${respuesta.mensaje}`);
      break;
  }
}

manejarRespuesta({ estado: "éxito", datos: ["a", "b", "c"] });
manejarRespuesta({ estado: "error", mensaje: "Algo salió mal" });
```

===============================================================================
3. TYPE ALIASES PARA FUNCIONES
===============================================================================

Los type aliases pueden describir firmas de funciones de manera limpia:

```typescript
type OperacionBinaria = (a: number, b: number) => number;

const sumar: OperacionBinaria = (a, b) => a + b;
const restar: OperacionBinaria = (a, b) => a - b;
const multiplicar: OperacionBinaria = (a, b) => a * b;
const dividir: OperacionBinaria = (a, b) => a / b;

// Con múltiples parámetros y genéricos
type Transformador<T, R> = (valor: T) => R;

const toString: Transformador<number, string> = (n) => n.toString();
const toNumber: Transformador<string, number> = (s) => parseFloat(s);

console.log(toString(42));        // "42"
console.log(toNumber("3.14"));    // 3.14
```

===============================================================================
4. TYPE ALIASES PARA TUPLAS
===============================================================================

Las tuplas son un tipo específico de array con longitud y tipos fijos:

```typescript
type Coordenada = [number, number];
type RGB = [number, number, number];
type PersonaInfo = [string, number, boolean?];

const punto: Coordenada = [10, 20];
const color: RGB = [255, 0, 0];
const persona: PersonaInfo = ["Ana", 28, true];

function calcularDistancia([x1, y1]: Coordenada, [x2, y2]: Coordenada): number {
  return Math.sqrt((x2 - x1) ** 2 + (y2 - y1) ** 2);
}

console.log(calcularDistancia([0, 0], [3, 4])); // 5
```

===============================================================================
5. INTERSECCIÓN DE TIPOS CON TYPE
===============================================================================

Mientras que las interfaces se extienden con `extends`, los types usan `&`:

```typescript
type Identificable = {
  id: number;
};

type Nombrable = {
  nombre: string;
};

type Fechable = {
  creadoEn: Date;
};

// Intersección: el objeto debe tener todas las propiedades
type Entidad = Identificable & Nombrable & Fechable;

const entidad: Entidad = {
  id: 1,
  nombre: "Ejemplo",
  creadoEn: new Date(),
};

// Con tipos más complejos
type ConEmail = {
  email: string;
};

type ConDireccion = {
  direccion: string;
};

type UsuarioCompleto = Identificable & Nombrable & ConEmail & ConDireccion;

const usuario: UsuarioCompleto = {
  id: 1,
  nombre: "Ana García",
  email: "ana@email.com",
  direccion: "Calle 123",
};
```

5.1. INTERSECCIÓN VS EXTENDS

```typescript
// Con interfaces
interface A { a: string }
interface B extends A { b: number }

// Con types (equivalente)
type TA = { a: string }
type TB = TA & { b: number }
```

===============================================================================
6. TYPE VS INTERFACE: COMPARATIVA DETALLADA
===============================================================================

6.1. CARACTERÍSTICAS EXCLUSIVAS DE TYPE

Los type aliases pueden representar conceptos que las interfaces no pueden:

```typescript
// Uniones de tipos primitivos (no posible con interface)
type Estado = "activo" | "inactivo" | "suspendido";

// Tipos condicionales
type EsString<T> = T extends string ? "sí" : "no";

// Mapped types
type Opcional<T> = {
  [K in keyof T]?: T[K];
};

// Tuplas
type Par = [string, number];
```

6.2. CARACTERÍSTICAS EXCLUSIVAS DE INTERFACE

Las interfaces tienen declaration merging:

```typescript
interface Usuario {
  nombre: string;
}

interface Usuario {
  edad: number;
}

// Fusionado automáticamente
const u: Usuario = { nombre: "Ana", edad: 28 }; // ✅

// No se puede hacer con type (error: Duplicate identifier)
// type Usuario2 = { nombre: string };
// type Usuario2 = { edad: number }; // ❌ Error
```

6.3. GUÍA DE ELECCIÓN

| Situación                                | Recomendación |
|------------------------------------------|---------------|
| Definir forma de objetos/API pública     | Interface     |
| Uniones y tipos complejos                | Type          |
| Necesitas herencia simple                | Interface     |
| Necesitas intersección                   | Type          |
| Declaration merging                      | Interface     |
| Tipos condicionales/mapped types         | Type          |
| Librerías/API pública                    | Interface     |
| Tipos internos/privados                  | Type          |

===============================================================================
7. TYPE ALIASES CON GENÉRICOS
===============================================================================

Los type aliases también pueden ser genéricos:

```typescript
type Resultado<T> = {
  exito: boolean;
  datos?: T;
  error?: string;
};

function obtenerUsuario(id: number): Resultado<{ nombre: string; email: string }> {
  try {
    // Simulación de operación
    return {
      exito: true,
      datos: { nombre: "Ana", email: "ana@email.com" },
    };
  } catch (error) {
    return {
      exito: false,
      error: "Usuario no encontrado",
    };
  }
}

type ParClaveValor<K, V> = {
  clave: K;
  valor: V;
};

const entrada: ParClaveValor<string, number> = {
  clave: "edad",
  valor: 28,
};
```

===============================================================================
8. TYPE ALIASES LITERALES Y TEMPLATE LITERAL TYPES
===============================================================================

TypeScript 4.1+ permite tipos literales con templates:

```typescript
type Evento = "click" | "hover" | "focus";
type Manejador = `on${Capitalize<Evento>}`;
// "onClick" | "onHover" | "onFocus"

type CSSUnidad = "px" | "rem" | "em" | "%";
type Medida = `${number}${CSSUnidad}`;

const tamaño1: Medida = "16px";
const tamaño2: Medida = "2rem";
// const tamaño3: Medida = "16"; // ❌ Error: falta unidad
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Los type aliases se declaran con `type Nombre = Tipo`
- Son ideales para uniones, intersecciones, tuplas y tipos complejos
- Las interfaces soportan declaration merging; los type no
- Los type aliases pueden ser genéricos y usar mapped types
- Usa interface para APIs públicas y objetos; type para uniones y transformaciones
- Los template literal types permiten crear patrones de strings como tipos
- Las intersecciones (`&`) son equivalentes a `extends` en interfaces

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo extender un type alias como si fuera una interfaz?**
No con `extends`, pero puedes usar intersección: `type B = A & { ... }`.

❓ **¿Los type aliases son más lentos que las interfaces?**
No hay diferencia en rendimiento en tiempo de ejecución. Ambos desaparecen
al compilar.

❓ **¿Cuándo debería preferir type sobre interface?**
Para uniones, tuplas, tipos condicionales, mapped types, o cuando trabajas
con tipos primitivos.

❓ **¿Puedo usar un type alias en una clase con implements?**
Sí, si el type describe la forma de un objeto.

❓ **¿Declaration merging es útil?**
Sí, especialmente para extender tipos de librerías externas sin modificar
sus archivos fuente.
