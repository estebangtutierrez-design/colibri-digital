1. TIPOS DE OBJETOS EN LÍNEA
===============================================================================

TypeScript permite definir la forma de un objeto directamente donde se usa,
sin necesidad de una interfaz o type alias separado:

```typescript
function imprimirCoordenada(punto: { x: number; y: number }): string {
  return `(${punto.x}, ${punto.y})`;
}

console.log(imprimirCoordenada({ x: 10, y: 20 })); // "(10, 20)"

// También en variables
const centro: { x: number; y: number; z?: number } = { x: 0, y: 0 };
```

1.1. CUÁNDO USAR TIPOS EN LÍNEA VS INTERFACES

- Tipo en línea: para usos únicos o simples
- Interface/Type: para tipos reutilizables o complejos

```typescript
// ✅ Aceptable para una función simple
function mostrarPrecio(producto: { nombre: string; precio: number }): void {
  console.log(`${producto.nombre}: $${producto.precio}`);
}

// ✅ Mejor con interfaz si se repite
interface Producto {
  nombre: string;
  precio: number;
}

function aplicarDescuento(producto: Producto, descuento: number): Producto {
  return {
    nombre: producto.nombre,
    precio: producto.precio * (1 - descuento),
  };
}
```

2. OBJETOS ANIDADOS
===============================================================================

Los objetos pueden contener otros objetos, y TypeScript los tipa
perfectamente:

```typescript
interface Direccion {
  calle: string;
  numero: number;
  ciudad: string;
  pais: string;
}

interface Persona {
  nombre: string;
  edad: number;
  direccion: Direccion;
  contacto: {
    email: string;
    telefono?: string;
  };
}

const persona: Persona = {
  nombre: "María López",
  edad: 32,
  direccion: {
    calle: "Av. Reforma",
    numero: 123,
    ciudad: "Ciudad de México",
    pais: "México",
  },
  contacto: {
    email: "maria@example.com",
    telefono: "+525512345678",
  },
};

console.log(persona.direccion.ciudad); // "Ciudad de México"
console.log(persona.contacto.email);   // "maria@example.com"
```

2.1. OBJETOS ANIDADOS COMO TIPOS EN LÍNEA

```typescript
function procesarPedido(pedido: {
  id: number;
  cliente: { nombre: string; email: string };
  items: { producto: string; cantidad: number; precio: number }[];
  total: number;
}): void {
  console.log(`Pedido #${pedido.id} para ${pedido.cliente.nombre}`);
  pedido.items.forEach(item => {
    console.log(`  - ${item.producto} x${item.cantidad}: $${item.precio}`);
  });
}
```

3. INDEX SIGNATURES (OBJETOS DINÁMICOS)
===============================================================================

Cuando no conocemos las propiedades exactas de un objeto, pero sabemos el
tipo de sus valores, usamos index signatures:

```typescript
interface DiccionarioString {
  [key: string]: string;
}

const colores: DiccionarioString = {
  rojo: "#FF0000",
  verde: "#00FF00",
  azul: "#0000FF",
};

// Podemos añadir propiedades dinámicamente
colores["amarillo"] = "#FFFF00";
colores.blanco = "#FFFFFF";

console.log(colores.rojo); // "#FF0000"
```

3.1. COMBINAR PROPIEDADES FIJAS Y DINÁMICAS

```typescript
interface Estudiante {
  nombre: string;
  edad: number;
  [materia: string]: string | number; // propiedades dinámicas
}

const estudiante: Estudiante = {
  nombre: "Luis",
  edad: 22,
  matematicas: 95,
  historia: 88,
  ciencias: 92,
};

console.log(`${estudiante.nombre}: Matemáticas ${estudiante.matematicas}`);
```

3.2. INDEX SIGNATURES CON READONLY

```typescript
interface ConfigReadonly {
  readonly [key: string]: string;
}

const config: ConfigReadonly = {
  host: "localhost",
  puerto: "8080",
};

// config.host = "otro"; // ❌ Error: Index signature in type 'ConfigReadonly' only permits reading
```

4. ARRAYS DE OBJETOS
===============================================================================

Los arrays de objetos son ubicuos en el desarrollo real:

```typescript
interface Tarea {
  id: number;
  titulo: string;
  completada: boolean;
  prioridad: "alta" | "media" | "baja";
}

const tareas: Tarea[] = [
  { id: 1, titulo: "Comprar leche", completada: false, prioridad: "alta" },
  { id: 2, titulo: "Estudiar TypeScript", completada: true, prioridad: "alta" },
  { id: 3, titulo: "Hacer ejercicio", completada: false, prioridad: "media" },
];

// Filtrar tareas pendientes
const pendientes = tareas.filter(t => !t.completada);

// Buscar por id
const tarea = tareas.find(t => t.id === 2);

// Mapear a otro formato
const resumen = tareas.map(t => ({
  titulo: t.titulo,
  estado: t.completada ? "✅" : "⏳",
}));

console.log(resumen);
```

4.1. ARRAYS MULTIDIMENSIONALES DE OBJETOS

```typescript
interface Celda {
  valor: string;

type Tabla = Celda[][];

const hojaCalculo: Tabla = [

  ],

```

5. OBJETOS CON MÉTODOS
===============================================================================

Los objetos pueden contener funciones como propiedades (métodos):

```typescript
interface Contador {
  valor: number;
  incrementar(): void;
  decrementar(): void;
  obtenerValor(): number;
}

function crearContador(valorInicial: number = 0): Contador {
  return {
    valor: valorInicial,
    incrementar() {
      this.valor++;
    },
    decrementar() {
      this.valor--;
    },
    obtenerValor() {
      return this.valor;
    },
  };
}

const contador = crearContador(10);
contador.incrementar();
contador.incrementar();
contador.decrementar();
console.log(contador.obtenerValor()); // 11
```

6. OBJETOS CON PROPIEDADES COMPUTADAS
===============================================================================

TypeScript soporta propiedades computadas con tipos específicos:

```typescript
const prefijo = "prop_";

interface ObjetoDinamico {
  [key: `${string}_${string}`]: string;
}

const obj: ObjetoDinamico = {
  prop_nombre: "valor",
  // key_normal: "error", // ❌ No cumple el patrón
};

// Con Template Literal Types (TS 4.1+)
type EventoNombre = `on${Capitalize<string>}`;

interface ManejadorEventos {
  [K: EventoNombre]: (datos: unknown) => void;
}

// No es posible indexar con template literal patterns directamente,
// pero es útil con mapped types
```

7. UTILIDADES PARA OBJETOS (PARTIAL, PICK, OMIT)
===============================================================================

TypeScript proporciona tipos utilitarios que manipulan objetos:

```typescript
interface Usuario {
  id: number;
  nombre: string;
  email: string;
  password: string;
  activo: boolean;
}

// Partial: todas las propiedades opcionales
function actualizarUsuario(id: number, cambios: Partial<Usuario>): void {
  console.log(`Actualizando usuario ${id} con:`, cambios);
}

actualizarUsuario(1, { nombre: "Nuevo nombre" });
actualizarUsuario(1, { email: "nuevo@email.com", activo: false });

// Pick: seleccionar propiedades específicas
type UsuarioPublico = Pick<Usuario, "id" | "nombre" | "email">;
const publico: UsuarioPublico = { id: 1, nombre: "Ana", email: "ana@email.com" };

// Omit: excluir propiedades
type UsuarioSinPassword = Omit<Usuario, "password">;
const seguro: UsuarioSinPassword = {
  id: 1,
  nombre: "Ana",
  email: "ana@email.com",
  activo: true,
};

// Required: todas obligatorias
type UsuarioCompleto = Required<Partial<Usuario>>;
```

8. DESESTRUCTURACIÓN DE OBJETOS CON TIPADO
===============================================================================

Combinar desestructuración con TypeScript:

```typescript
interface Producto {
  id: number;
  nombre: string;
  precio: number;
  categoria: string;
  stock: number;
}

function mostrarProducto({ nombre, precio, stock }: Producto): string {
  return `${nombre}: $${precio} (${stock} unidades)`;
}

// Desestructuración con renombrado
function mostrarProductoRenombrado({ nombre: titulo, precio: costo }: Producto): string {
  return `${titulo}: $${costo}`;
}

// Desestructuración anidada
interface Pedido {
  id: number;
  cliente: { nombre: string; email: string };
  total: number;
}

function resumenPedido({ id, cliente: { nombre, email }, total }: Pedido): void {
  console.log(`Pedido #${id}: ${nombre} (${email}) - $${total}`);
}
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener un objeto con propiedades de diferentes tipos en index signature?**
No directamente. Debes usar una unión: `[key: string]: string | number`.

❓ **¿Qué es excess property checking?**
TypeScript verifica que los objetos literales no tengan propiedades no
declaradas en el tipo. Esto no aplica cuando se asigna una variable.

❓ **¿Cómo hago para que TODAS las propiedades de un tipo sean opcionales?**
Usa el utilitario `Partial<T>`.

❓ **¿Puedo usar desestructuración con tipos en línea?**
Sí: `function f({ nombre, edad }: { nombre: string; edad: number })`.
