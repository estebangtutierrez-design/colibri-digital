1. TIPADO BÁSICO DE FUNCIONES
===============================================================================

En TypeScript, cada función puede especificar el tipo de sus parámetros y
el tipo de su valor de retorno. La sintaxis más simple es:

```typescript
function sumar(a: number, b: number): number {
  return a + b;
}

const resultado = sumar(5, 3); // tipo inferido: number
```

Si intentamos llamar a la función con argumentos de tipo incorrecto,
TypeScript lo rechazará en tiempo de compilación:

```typescript
// Error: Argument of type 'string' is not assignable to parameter of type 'number'
// sumar("5", 3);
```

También podemos asignar funciones tipadas a variables:

```typescript
const multiplicar: (x: number, y: number) => number = (x, y) => {
  return x * y;
};
```

La parte `(x: number, y: number) => number` es la firma de la función escrita
como tipo. Veremos más detalles sobre firmas en capítulos posteriores.

1.1. FUNCIONES CON RETORNOS EXPLÍCITOS

Siempre es buena práctica anotar el tipo de retorno de una función, aunque
TypeScript pueda inferirlo. Esto actúa como una red de seguridad:

```typescript
function obtenerSaludo(nombre: string): string {
  return `¡Hola, ${nombre}!`;
}

function calcularEdad(anioNacimiento: number, anioActual: number): number {
  return anioActual - anioNacimiento;
}
```

1.2. FUNCIONES SIN RETORNO: TIPO VOID

Cuando una función no devuelve ningún valor, usamos el tipo `void`:

```typescript
function mostrarMensaje(mensaje: string): void {
  console.log(mensaje);
  // No hay return
}

function registrarUsuario(nombre: string, edad: number): void {
  console.log(`Usuario ${nombre} registrado con edad ${edad}`);
  // Podemos tener un return sin valor
  return;
}
```

El tipo `void` indica que la función no produce un valor útil.

1.3. FUNCIONES QUE NUNCA TERMINAN: TIPO NEVER

El tipo `never` representa funciones que nunca finalizan con éxito, ya sea
porque lanzan una excepción o porque entran en un bucle infinito:

```typescript
function lanzarError(mensaje: string): never {
  throw new Error(mensaje);
}

function bucleInfinito(): never {
  while (true) {
    console.log("Esto nunca termina...");
  }
}

function validarEdad(edad: number): string | never {
  if (edad < 0) {
    return lanzarError("La edad no puede ser negativa");
  }
  return `Edad válida: ${edad}`;
}
```

===============================================================================
2. INFERENCIA DE TIPOS EN FUNCIONES
===============================================================================

TypeScript es inteligente: a menudo puede inferir el tipo de retorno sin
necesidad de anotación explícita:

```typescript
function dividir(a: number, b: number) {
  return a / b; // TypeScript infiere: number
}

function concatenar(a: string, b: string) {
  return a + b; // TypeScript infiere: string
}
```

Sin embargo, la inferencia tiene límites. En funciones complejas la anotación
explícita mejora la legibilidad y la seguridad:

```typescript
function procesarDatos(datos: (string | number)[]): string[] {
  return datos.map(item => String(item).toUpperCase());
}
```

2.1. INFERENCIA EN ARROW FUNCTIONS

Las funciones flecha también se benefician de la inferencia:

```typescript
const numeros = [1, 2, 3, 4, 5];

const dobles = numeros.map(n => n * 2);
// TypeScript infiere que 'n' es number y que el retorno es number[]

const pares = numeros.filter(n => n % 2 === 0);
// TypeScript infiere: number[]
```

===============================================================================
3. FUNCIONES FLECHA TIPADAS
===============================================================================

Las arrow functions pueden tiparse de forma explícita:

```typescript
const sumar = (a: number, b: number): number => a + b;

const saludar = (nombre: string): string => `Hola, ${nombre}`;

const imprimir = (mensaje: string): void => console.log(mensaje);
```

Cuando se usan como callbacks, el tipado puede ser inferido del contexto:

```typescript
const frutas = ["manzana", "pera", "uva"];

// El tipo de 'fruta' se infiere como string
const frutasUpper = frutas.map((fruta) => fruta.toUpperCase());
```

Para mayor claridad, podemos anotar el callback explícitamente:

```typescript
const frutasUpper2 = frutas.map((fruta: string): string => fruta.toUpperCase());
```

===============================================================================
4. PARÁMETROS DE FUNCIONES
===============================================================================

Los parámetros pueden tener cualquier tipo, incluyendo tipos unión,
literales, o incluso otras funciones:

```typescript
function operacion(
  a: number,
  b: number,
  tipo: "suma" | "resta" | "multiplica"
): number {
  switch (tipo) {
    case "suma":
      return a + b;
    case "resta":
      return a - b;
    case "multiplica":
      return a * b;
  }
}

console.log(operacion(10, 5, "suma"));      // 15
console.log(operacion(10, 5, "resta"));     // 5
console.log(operacion(10, 5, "multiplica")); // 50
```

4.1. FUNCIONES QUE RECIBEN FUNCIONES (CALLBACKS)

Una función puede recibir otra función como parámetro. Tipamos el callback
para garantizar su correcto uso:

```typescript
function ejecutarOperacion(
  a: number,
  b: number,
  callback: (resultado: number) => void
): void {
  const resultado = a + b;
  callback(resultado);
}

ejecutarOperacion(10, 20, (r) => {
  console.log(`El resultado es: ${r}`);
});

// También podemos pasar una función nombrada
function mostrarResultado(r: number): void {
  console.log(`Resultado: ${r}`);
}

ejecutarOperacion(100, 200, mostrarResultado);
```

4.2. FUNCIONES QUE DEVUELVEN FUNCIONES

TypeScript permite que una función devuelva otra función, y podemos tipar
tanto la función externa como la interna:

```typescript
function crearMultiplicador(factor: number): (valor: number) => number {
  return (valor: number) => valor * factor;
}

const duplicar = crearMultiplicador(2);
const triplicar = crearMultiplicador(3);

console.log(duplicar(5));  // 10
console.log(triplicar(5)); // 15
```

===============================================================================
5. FUNCIONES CON PARÁMETROS DESTRUCTURADOS
===============================================================================

Podemos aplicar TypeScript a la desestructuración de objetos y arreglos
en los parámetros de una función:

```typescript
interface Usuario {
  nombre: string;
  edad: number;
  email: string;
}

function mostrarUsuario({ nombre, edad, email }: Usuario): void {
  console.log(`${nombre} (${edad}) - ${email}`);
}

const usuario: Usuario = {
  nombre: "Ana",
  edad: 28,
  email: "ana@example.com",
};

mostrarUsuario(usuario);
```

También con arreglos:

```typescript
function sumarPares([a, b]: [number, number]): number {
  return a + b;
}

console.log(sumarPares([10, 20])); // 30
```

6. FUNCIONES ASÍNCRONAS TIPADAS
===============================================================================

Las funciones asíncronas en TypeScript devuelven una Promesa. El tipo del
valor resuelto se especifica como generic de Promise:

```typescript
async function obtenerDatos(url: string): Promise<object> {
  const respuesta = await fetch(url);
  const datos = await respuesta.json();
  return datos;
}

async function sumarAsync(a: number, b: number): Promise<number> {
  // Simulamos una operación asíncrona
  return new Promise((resolve) => {
    setTimeout(() => resolve(a + b), 1000);
  });
}

// Uso
async function main(): Promise<void> {
  const resultado = await sumarAsync(5, 7);
  console.log(`Resultado asíncrono: ${resultado}`);
}

main();
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Las funciones en TypeScript pueden tipar tanto sus parámetros como su retorno
- El tipo `void` indica que una función no retorna un valor útil
- El tipo `never` indica que una función nunca finaliza normalmente
- TypeScript infiere tipos de retorno, pero la anotación explícita mejora la seguridad
- Las arrow functions se benefician del tipado contextual
- Las funciones pueden recibir y devolver otras funciones con tipos específicos
- La desestructuración tipada hace que las funciones sean más legibles
- Las funciones async devuelven `Promise<T>` donde T es el tipo resuelto

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Es obligatorio anotar el tipo de retorno?**
No, TypeScript infiere el tipo. Pero anotarlo explícitamente evita errores
y sirve como documentación.

❓ **¿Cuál es la diferencia entre void y undefined como tipo de retorno?**
void significa que el valor de retorno no se usará. undefined es un valor
real. Una función con retorno void puede devolver undefined, pero no al revés.

❓ **¿Puedo mezclar funciones tipadas con JavaScript plano?**
Sí, TypeScript es un superconjunto de JavaScript. Puedes tener funciones sin
tipar junto a funciones tipadas, aunque no es recomendable.

❓ **¿Qué pasa si no pongo tipo a un parámetro?**
TypeScript lo infiere como `any`, perdiendo la seguridad de tipos. Es mejor
siempre tipar los parámetros.
