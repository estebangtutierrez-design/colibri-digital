En el capítulo anterior aprendiste a declarar funciones con parámetros

===============================================================================
1. SINTAXIS DE PARÁMETROS OPCIONALES
===============================================================================

Un parámetro opcional se declara añadiendo `?` justo después del nombre del
parámetro, antes de los dos puntos que indican el tipo:

```typescript
function saludar(nombre: string, saludo?: string): string {
  if (saludo) {
    return `${saludo}, ${nombre}!`;
  }
  return `Hola, ${nombre}!`;
}

console.log(saludar("Ana"));               // "Hola, Ana!"
console.log(saludar("Ana", "Buenos días")); // "Buenos días, Ana!"
```

El parámetro `saludo` es opcional. Si no se proporciona, su valor será
`undefined`. TypeScript obliga a verificar su existencia antes de usarlo.

1.1. TIPO INTERNO DE UN PARÁMETRO OPCIONAL

Internamente, un parámetro opcional es una unión con `undefined`:

```typescript
// Estas dos declaraciones son equivalentes:
function ejemplo1(x?: number): void {}
function ejemplo2(x: number | undefined): void {}
```

La diferencia es que en el primer caso puedes omitir el argumento, mientras
que en el segundo debes pasar explícitamente `undefined`:

```typescript
ejemplo1();        // ✅ Válido
ejemplo1(5);       // ✅ Válido
ejemplo2(undefined); // ✅ Válido
// ejemplo2();     // ❌ Error: Expected 1 argument
```

===============================================================================
2. REGLA FUNDAMENTAL: OPCIONALES AL FINAL
===============================================================================

Los parámetros opcionales deben ir siempre después de los obligatorios:

```typescript
// ✅ Correcto
function configurar(host: string, puerto?: number, timeout?: number): void {
  console.log(`Conectando a ${host}:${puerto ?? 5432}`);
}

// ❌ Incorrecto: parámetro obligatorio después de opcional
// function incorrecto(opcional?: string, obligatorio: number): void {
//   // Error: Required parameter after optional parameter
// }
```

Si necesitas un orden diferente, usa objetos de configuración:

```typescript
interface ConfigConexion {
  host: string;
  puerto?: number;
  timeout?: number;
}

function conectar(config: ConfigConexion): void {
  const { host, puerto = 5432, timeout = 5000 } = config;
  console.log(`Conectando a ${host}:${puerto} (timeout: ${timeout}ms)`);
}

conectar({ host: "localhost" });
conectar({ host: "localhost", puerto: 3306 });
```

===============================================================================
3. VALIDACIÓN DE PARÁMETROS OPCIONALES
===============================================================================

Dentro de la función debemos asegurarnos de que el parámetro existe antes
de usarlo. Hay varias estrategias:

3.1. USO DE CONDICIONALES

```typescript
function mostrarPerfil(
  nombre: string,
  edad?: number,
  email?: string
): string {
  let perfil = `Nombre: ${nombre}`;

  if (edad !== undefined) {
    perfil += `, Edad: ${edad}`;
  }

  if (email) {
    perfil += `, Email: ${email}`;
  }

  return perfil;
}

console.log(mostrarPerfil("Carlos"));
console.log(mostrarPerfil("Carlos", 30));
console.log(mostrarPerfil("Carlos", 30, "carlos@email.com"));
```

3.2. OPERADOR DE ENCADENAMIENTO OPCIONAL (?. )

Para acceder a propiedades de un objeto opcional:

```typescript
interface Direccion {
  calle?: string;
  ciudad?: string;
}

function mostrarDireccion(dir?: Direccion): string {
  const calle = dir?.calle ?? "Calle no especificada";
  const ciudad = dir?.ciudad ?? "Ciudad no especificada";
  return `${calle}, ${ciudad}`;
}

console.log(mostrarDireccion()); // "Calle no especificada, Ciudad no especificada"
console.log(mostrarDireccion({ calle: "Av. Siempre Viva" }));
```

3.3. VALOR POR DEFECTO CON OPERADOR ??

```typescript
function crearMensaje(
  texto: string,
  prefijo?: string,
  sufijo?: string
): string {
  const p = prefijo ?? ">>";
  const s = sufijo ?? "<<";
  return `${p} ${texto} ${s}`;
}

console.log(crearMensaje("Hola"));            // ">> Hola <<"
console.log(crearMensaje("Hola", "***"));     // "*** Hola <<"
console.log(crearMensaje("Hola", "***", "***")); // "*** Hola ***"
```

===============================================================================
4. PARÁMETROS OPCIONALES EN ARROW FUNCTIONS
===============================================================================

Las funciones flecha también admiten parámetros opcionales:

```typescript
const sumar = (a: number, b?: number): number => {
  return a + (b ?? 0);
};

console.log(sumar(5));   // 5
console.log(sumar(5, 3)); // 8

// Con un callback opcional
const procesarNumeros = (
  numeros: number[],
  transformador?: (n: number) => number
): number[] => {
  if (transformador) {
    return numeros.map(transformador);
  }
  return numeros;
};

console.log(procesarNumeros([1, 2, 3]));         // [1, 2, 3]
console.log(procesarNumeros([1, 2, 3], (n) => n * 2)); // [2, 4, 6]
```

===============================================================================
5. PARÁMETROS OPCIONALES EN CALLBACKS
===============================================================================

Cuando definimos una función que recibe un callback opcional, debemos
manejar adecuadamente la ausencia:

```typescript
function leerArchivo(
  ruta: string,
  callback?: (contenido: string) => void
): void {
  const contenidoSimulado = `Contenido de ${ruta}`;

  if (callback) {
    callback(contenidoSimulado);
  } else {
    console.log(contenidoSimulado);
  }
}

leerArchivo("datos.txt");
leerArchivo("datos.txt", (c) => console.log(`Callback: ${c}`));
```

===============================================================================
6. PARÁMETROS OPCIONALES CON DESTRUCTURACIÓN
===============================================================================

Al desestructurar objetos en parámetros, podemos hacer opcionales tanto
el objeto completo como sus propiedades:

```typescript
interface Opciones {
  color?: string;
  tamano?: number;
  borde?: boolean;
}

function crearCaja(
  contenido: string,
  opciones?: Opciones
): string {
  const color = opciones?.color ?? "negro";
  const tamano = opciones?.tamano ?? 100;
  const borde = opciones?.borde ?? false;

  return `Caja ${color} de ${tamano}px${borde ? " con borde" : ""}: ${contenido}`;
}

console.log(crearCaja("Zapatos"));
console.log(crearCaja("Zapatos", { color: "rojo" }));
console.log(crearCaja("Zapatos", { color: "azul", tamano: 200, borde: true }));
```

Desestructuración con valor por defecto:

```typescript
function crearCaja2(
  contenido: string,
  { color = "negro", tamano = 100, borde = false }: Opciones = {}
): string {
  return `Caja ${color} de ${tamano}px${borde ? " con borde" : ""}: ${contenido}`;
}
```

Nótese el `= {}` al final: si no se pasa el objeto, se usa un objeto vacío
y luego las propiedades toman sus valores por defecto.

===============================================================================
7. ERRORES COMUNES Y BUENAS PRÁCTICAS
===============================================================================

7.1. NO VERIFICAR EL PARÁMETRO OPCIONAL

```typescript
function imprimirLongitud(texto?: string): void {
  // ❌ Error: 'texto' is possibly 'undefined'
  // console.log(texto.length);

  // ✅ Correcto
  console.log(texto?.length ?? 0);
}
```

7.2. COLOCAR OPCIONALES ANTES DE OBLIGATORIOS

```typescript
// ❌ Incorrecto
// function ejemplo(opcional?: string, obligatorio: number) {}

// ✅ Correcto
function ejemplo(obligatorio: number, opcional?: string) {}
```

7.3. USAR PARÁMETROS OPCIONALES CUANDO DEBERÍAN SER POR DEFECTO

Si siempre usas un mismo valor cuando el parámetro no se pasa, considera
usar un parámetro por defecto (lo veremos en el próximo capítulo).

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Los parámetros opcionales se declaran con `?` después del nombre del parámetro
- Internamente son `T | undefined`
- Los parámetros opcionales deben ir al final de la lista de parámetros
- Siempre verifica la existencia del parámetro antes de usarlo
- Usa `?.` y `??` para manejar opcionales de forma segura
- Los callbacks opcionales requieren verificación antes de invocarlos
- En desestructuración, combina `?` con valores por defecto para mayor seguridad

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener todos los parámetros opcionales?**
Sí, pero entonces cualquier llamada sin argumentos sería válida. Úsalo con
cuidado, pues puede hacer la función ambigua.

❓ **¿Cuál es la diferencia entre `param?: string` y `param: string | undefined`?**
Con `?` puedes omitir el argumento al llamar la función. Con `| undefined`
debes pasar explícitamente `undefined`.

❓ **¿Puedo usar parámetros opcionales en métodos de clases?**
Sí, exactamente con la misma sintaxis que en funciones normales.

❓ **¿Qué pasa si uso `!` (non-null assertion) en un parámetro opcional?**
El operador `!` le dice a TypeScript que confíes en que el valor no es
undefined, pero si el argumento no se pasa, el programa fallará en
tiempo de ejecución. Es mejor usar validaciones seguras.
