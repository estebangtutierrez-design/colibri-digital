1. SINTAXIS DE PARÁMETROS POR DEFECTO
===============================================================================

Un parámetro por defecto se declara asignando un valor con `=` en la
declaración del parámetro:

```typescript
function saludar(nombre: string, saludo: string = "Hola"): string {
  return `${saludo}, ${nombre}!`;
}

console.log(saludar("Ana"));           // "Hola, Ana!"
console.log(saludar("Ana", "Buenos días")); // "Buenos días, Ana!"
```

Cuando no se pasa el argumento, o se pasa `undefined`, se usa el valor
por defecto:

```typescript
console.log(saludar("Pedro", undefined)); // "Hola, Pedro!"
// undefined dispara el valor por defecto
```

1.1. TIPO DE UN PARÁMETRO POR DEFECTO

TypeScript infiere el tipo del valor por defecto. La anotación de tipo es
opcional si el valor por defecto es suficientemente claro:

```typescript
function conectar(
  host: string = "localhost",
  puerto: number = 5432
): string {
  return `${host}:${puerto}`;
}

// El tipo se infiere de los valores por defecto
```

1.2. DIFERENCIA CON PARÁMETROS OPCIONALES

Un parámetro por defecto es implícitamente opcional. No necesitas el `?`:

```typescript
// Con ? (opcional)
function opcional(nombre?: string): string {
  return `Hola, ${nombre ?? "invitado"}`;
}

// Con valor por defecto (más limpio)
function porDefecto(nombre: string = "invitado"): string {
  return `Hola, ${nombre}`;
}

console.log(opcional());     // "Hola, invitado"
console.log(porDefecto());   // "Hola, invitado"
```

===============================================================================
2. VALORES POR DEFECTO VS undefined Y null
===============================================================================

Es crucial entender el comportamiento con `undefined` y `null`:

```typescript
function mostrar(mensaje: string = "Sin mensaje"): string {
  return mensaje;
}

console.log(mostrar());              // "Sin mensaje"  (usa defecto)
console.log(mostrar("Hola"));        // "Hola"         (usa argumento)
console.log(mostrar(undefined));     // "Sin mensaje"  (usa defecto)
console.log(mostrar(null));          // null           (NO usa defecto)
```

⚠️ **Importante:** `null` NO dispara el valor por defecto. Solo `undefined`
lo hace. Este es un comportamiento heredado de JavaScript.

===============================================================================
3. EXPRESIONES COMO VALORES POR DEFECTO
===============================================================================

Los valores por defecto pueden ser expresiones, incluyendo llamadas a
funciones:

```typescript
function generarId(): number {
  return Math.floor(Math.random() * 10000);
}

function crearUsuario(nombre: string, id: number = generarId()): object {
  return { nombre, id };
}

console.log(crearUsuario("Ana"));
console.log(crearUsuario("Luis"));
console.log(crearUsuario("Carlos", 999));
```

3.1. USO DE FUNCIONES ANIDADAS

```typescript
function obtenerFechaPorDefecto(): string {
  return new Date().toISOString().split("T")[0];
}

function registrarEvento(
  evento: string,
  fecha: string = obtenerFechaPorDefecto()
): string {
  return `[${fecha}] ${evento}`;
}

console.log(registrarEvento("Inicio del sistema"));
console.log(registrarEvento("Error crítico", "2025-01-01"));
```

3.2. EXPRESIONES COMPLEJAS

```typescript
function calcularIVA(precio: number, porcentaje: number = 0.21): number {
  return precio * (1 + porcentaje);
}

function crearMensaje(
  texto: string,
  prefijo: string = ">>",
  sufijo: string = prefijo + " "
): string {
  return `${prefijo} ${texto} ${sufijo}`;
}

console.log(crearMensaje("Hola"));     // ">> Hola >> "
console.log(crearMensaje("Hola", "*")); // "* Hola * "
```

Nótese que `sufijo` usa `prefijo` en su expresión. El orden importa:
solo puedes usar parámetros anteriores como parte de la expresión.

===============================================================================
4. PARÁMETROS POR DEFECTO Y DESTRUCTURACIÓN
===============================================================================

Una de las combinaciones más potentes es usar valores por defecto en
parámetros desestructurados:

```typescript
interface Config {
  host?: string;
  puerto?: number;
  timeout?: number;
}

function conectar({
  host = "localhost",
  puerto = 5432,
  timeout = 5000,
}: Config = {}): string {
  return `Conectando a ${host}:${puerto} (timeout: ${timeout}ms)`;
}

console.log(conectar());
console.log(conectar({ host: "servidor.com" }));
console.log(conectar({ host: "servidor.com", puerto: 3306, timeout: 10000 }));
```

El `= {}` al final es crucial para que la función funcione sin argumentos.

4.1. DESESTRUCTURACIÓN DE ARREGLOS CON VALORES POR DEFECTO

```typescript
function crearPunto(
  [x, y]: [number, number] = [0, 0]
): string {
  return `(${x}, ${y})`;
}

console.log(crearPunto());        // "(0, 0)"
console.log(crearPunto([10, 20])); // "(10, 20)"
```

===============================================================================
5. ORDEN DE PARÁMETROS
===============================================================================

Los parámetros con valor por defecto deben ir después de los obligatorios,
al igual que los opcionales:

```typescript
// ✅ Correcto
function configurar(host: string, puerto: number = 5432, timeout?: number): void {
  console.log(`${host}:${puerto}`);
}

// También se puede mezclar con opcionales
function ejemplo(a: string, b: number = 10, c?: boolean): void {}
```

Sin embargo, a diferencia de los opcionales, los parámetros por defecto
pueden ir antes de parámetros obligatorios si les pasamos `undefined`:

```typescript
function ejemplo(a: string = "default", b: number): string {
  return `${a} - ${b}`;
}

// ❌ Esto no funciona: falta el segundo argumento
// ejemplo("Hola");

// ✅ Funciona pasando undefined para el primero
console.log(ejemplo(undefined, 42)); // "default - 42"
```

Aunque esto es posible, no es recomendable por legibilidad.

===============================================================================
6. PARÁMETROS POR DEFECTO EN ARROW FUNCTIONS
===============================================================================

Las arrow functions también aceptan valores por defecto:

```typescript
const sumar = (a: number, b: number = 0): number => a + b;

const crearSaludo = (nombre: string, saludo: string = "Hola"): string =>
  `${saludo}, ${nombre}!`;

const mapearNumeros = (
  numeros: number[],
  factor: number = 1
): number[] => numeros.map(n => n * factor);

console.log(sumar(5));            // 5
console.log(sumar(5, 3));         // 8
console.log(crearSaludo("Ana"));  // "Hola, Ana!"
console.log(mapearNumeros([1, 2, 3], 2)); // [2, 4, 6]
```

===============================================================================
7. VALORES POR DEFECTO EN MÉTODOS DE OBJETOS
===============================================================================

```typescript
const calculadora = {
  sumar(a: number, b: number = 0): number {
    return a + b;
  },
  multiplicar(a: number, b: number = 1): number {
    return a * b;
  },
  potencia(base: number, exponente: number = 2): number {
    return base ** exponente;
  },
};

console.log(calculadora.sumar(5));         // 5
console.log(calculadora.multiplicar(4));    // 4
console.log(calculadora.potencia(3));       // 9
console.log(calculadora.potencia(3, 3));    // 27
```

8. CASO PRÁCTICO: FUNCIÓN DE CONFIGURACIÓN ROBUSTA
===============================================================================

```typescript
interface ConexionBD {
  host?: string;
  puerto?: number;
  usuario?: string;
  password?: string;
  baseDatos?: string;
  ssl?: boolean;
}

function conectarBD(
  config: ConexionBD = {}
): void {
  const {
    host = "localhost",
    puerto = 3306,
    usuario = "root",
    password = "",
    baseDatos = "test",
    ssl = false,
  } = config;

  console.log(`
    Conectando a:
    - Host: ${host}
    - Puerto: ${puerto}
    - Usuario: ${usuario}
    - Base de datos: ${baseDatos}
    - SSL: ${ssl ? "Activado" : "Desactivado"}
  `);
}

conectarBD();
conectarBD({ host: "produccion.com", ssl: true });
conectarBD({
  host: "localhost",
  usuario: "admin",
  password: "secreto",
  baseDatos: "produccion",
});
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Los parámetros por defecto se declaran con `= valor` en la firma de la función
- Son implícitamente opcionales: no necesitan `?`
- Solo `undefined` dispara el valor por defecto, `null` no
- Los valores por defecto pueden ser expresiones y llamadas a funciones
- Se pueden combinar con desestructuración para crear configuraciones flexibles
- En desestructuración de objetos, el `= {}` permite llamar sin argumentos
- Los valores por defecto mejoran la robustez y la legibilidad del código

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo usar valores por defecto en funciones con overloads?**
Sí, pero ten cuidado: los valores por defecto cuentan como una firma
implícita que debe ser compatible con todas las sobrecargas.

❓ **¿Los valores por defecto afectan el tipo de retorno inferido?**
No directamente. El tipo de retorno se infiere del cuerpo, no de los
valores por defecto de los parámetros.

❓ **¿Puedo usar un objeto como valor por defecto?**
Sí, pero recuerda que el mismo objeto se reutiliza en todas las llamadas
que no proporcionen el argumento. Para objetos mutables, es mejor usar
una función fábrica como valor por defecto.

❓ **¿Es mejor usar opcionales con ?? o valores por defecto?**
Generalmente, los valores por defecto son más limpios. Usa opcionales
con ?? solo cuando necesites distinguir entre "no pasado" y "pasado
como null".
