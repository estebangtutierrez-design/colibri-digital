===============================================================================
1. ¿QUÉ ES UNA INTERFAZ?
===============================================================================

Una interfaz en TypeScript es un contrato que define la estructura de un
objeto. No se compila a JavaScript: existe solo en tiempo de compilación
para verificar tipos.

```typescript
interface Usuario {
  nombre: string;
  edad: number;
  email: string;
}

function registrarUsuario(usuario: Usuario): void {
  console.log(`${usuario.nombre} registrado con éxito`);
}

// ✅ Correcto: el objeto cumple con la interfaz
const nuevoUsuario: Usuario = {
  nombre: "Ana García",
  edad: 28,
  email: "ana@example.com",
};

registrarUsuario(nuevoUsuario);

// ❌ Error: falta la propiedad 'email'
// const usuarioInvalido: Usuario = {
//   nombre: "Carlos",
//   edad: 35,
// };
```

1.1. TYPE CHECKING ESTRUCTURAL

TypeScript usa tipado estructural (duck typing): si un objeto tiene la forma
correcta, es compatible con la interfaz, aunque no se declare explícitamente:

```typescript
interface Punto {
  x: number;
  y: number;
}

function imprimirPunto(p: Punto): void {
  console.log(`(${p.x}, ${p.y})`);
}

// El objeto no se declara como Punto, pero es compatible
imprimirPunto({ x: 10, y: 20 }); // ✅

// El objeto con propiedades extra aún puede ser compatible
const puntoExtra = { x: 5, y: 10, z: 15 };
imprimirPunto(puntoExtra); // ✅ (solo en paso por referencia)
```

⚠️ Nota: Pasar un objeto literal con propiedades extra directamente causa
error por *excess property checking*:

```typescript
// ❌ Error: Object literal may only specify known properties
// imprimirPunto({ x: 5, y: 10, z: 15 });
```

===============================================================================
2. PROPIEDADES OPCIONALES EN INTERFACES
===============================================================================

Al igual que los parámetros de funciones, las propiedades de una interfaz
pueden ser opcionales usando `?`:

```typescript
interface Configuracion {
  host: string;
  puerto?: number;
  timeout?: number;
  ssl?: boolean;
}

function iniciarServidor(config: Configuracion): string {
  const puerto = config.puerto ?? 8080;
  const timeout = config.timeout ?? 5000;
  return `Servidor en ${config.host}:${puerto} (timeout: ${timeout}ms)`;
}

console.log(iniciarServidor({ host: "localhost" }));
console.log(iniciarServidor({ host: "localhost", puerto: 3000, ssl: true }));
```

===============================================================================
3. PROPIEDADES READONLY
===============================================================================

El modificador `readonly` impide que una propiedad sea reasignada después
de la inicialización:

```typescript
interface Punto {
  readonly x: number;
  readonly y: number;
}

const origen: Punto = { x: 0, y: 0 };
console.log(origen.x); // 0

// ❌ Error: Cannot assign to 'x' because it is a read-only property
// origen.x = 10;
```

3.1. READONLY VS CONST

- `const` impide reasignar la variable completa
- `readonly` impide modificar esa propiedad específica

```typescript
const punto: Punto = { x: 5, y: 10 };
// punto = { x: 0, y: 0 }; // ❌ Error (const)
punto.x = 5;                // ❌ Error (readonly)

// Se pueden usar juntos para máxima protección
```

3.2. ARRAYS READONLY

Podemos usar `ReadonlyArray<T>` o el shorthand `readonly T[]`:

```typescript
interface Datos {
  readonly id: number;
  readonly items: ReadonlyArray<string>;
}

const datos: Datos = {
  id: 1,
  items: ["a", "b", "c"],
};

// datos.id = 2;       // ❌ Error
// datos.items.push("d"); // ❌ Error (no hay push en ReadonlyArray)
```

===============================================================================
4. INTERFACES CON MÉTODOS
===============================================================================

Las interfaces también pueden definir métodos que los objetos deben
implementar:

```typescript
interface Reproducible {
  titulo: string;
  duracion: number;
  reproducir(): void;
  pausar(): void;
  obtenerInfo(): string;
}

class Cancion implements Reproducible {
  constructor(
    public titulo: string,
    public duracion: number,
    private artista: string
  ) {}

  reproducir(): void {
    console.log(`▶️ Reproduciendo: ${this.titulo} - ${this.artista}`);
  }

  pausar(): void {
    console.log(`⏸️ Pausado: ${this.titulo}`);
  }

  obtenerInfo(): string {
    return `${this.titulo} (${this.duracion}s) - ${this.artista}`;
  }
}

const cancion = new Cancion("Bohemian Rhapsody", 354, "Queen");
cancion.reproducir();
console.log(cancion.obtenerInfo());
```

===============================================================================
5. EXTENSIÓN DE INTERFACES
===============================================================================

Las interfaces pueden extender otras interfaces, heredando sus propiedades:

```typescript
interface Animal {
  nombre: string;
  edad: number;
}

interface Mascota extends Animal {
  dueno: string;
  vacunado: boolean;
}

interface Perro extends Mascota {
  raza: string;
  ladrar(): void;
}

const miPerro: Perro = {
  nombre: "Rex",
  edad: 5,
  dueno: "Carlos",
  vacunado: true,
  raza: "Pastor Alemán",
  ladrar: () => console.log("¡Guau!"),
};

miPerro.ladrar();
```

5.1. EXTENSIÓN MÚLTIPLE

Una interfaz puede extender múltiples interfaces:

```typescript
interface Volador {
  volar(): void;
}

interface Nadador {
  nadar(): void;
}

interface Pato extends Volador, Nadador {
  nombre: string;
  hacerGraznido(): void;
}

const pato: Pato = {
  nombre: "Donald",
  volar: () => console.log("Volando..."),
  nadar: () => console.log("Nadando..."),
  hacerGraznido: () => console.log("¡Cuac!"),
};

pato.volar();
pato.nadar();
pato.hacerGraznido();
```

===============================================================================
6. INTERFACES CON INDEX SIGNATURES
===============================================================================

Las interfaces pueden definir objetos con propiedades dinámicas usando
index signatures:

```typescript
interface Diccionario {
  [clave: string]: string;
}

const traducciones: Diccionario = {
  hola: "hello",
  adios: "goodbye",
  casa: "house",
};

console.log(traducciones.hola);    // "hello"
console.log(traducciones["casa"]); // "house"

// Los valores deben ser strings
// traducciones.numero = 42; // ❌ Error
```

6.1. COMBINAR PROPIEDADES FIJAS Y DINÁMICAS

```typescript
interface UsuarioDinamico {
  nombre: string;
  email: string;
  [key: string]: string; // Todas las propiedades extra deben ser string
}

const usuario: UsuarioDinamico = {
  nombre: "Ana",
  email: "ana@email.com",
  telefono: "+521234567890",
  direccion: "Calle 123",
};
```

===============================================================================
7. INTERFACES PARA FUNCIONES
===============================================================================

Las interfaces también pueden describir la firma de una función:

```typescript
interface OperacionMatematica {
  (a: number, b: number): number;
}

const sumarFn: OperacionMatematica = (a, b) => a + b;
const restarFn: OperacionMatematica = (a, b) => a - b;
const multiplicarFn: OperacionMatematica = (a, b) => a * b;

console.log(sumarFn(5, 3));       // 8
console.log(restarFn(10, 4));     // 6
console.log(multiplicarFn(6, 7)); // 42
```

===============================================================================
8. INTERFACES PARA ARRAYS
===============================================================================

Podemos describir arrays con interfaces usando index signatures:

```typescript
interface ListaNumeros {
  [index: number]: number;
  length: number;
}

const numeros: ListaNumeros = [1, 2, 3, 4, 5];
console.log(numeros[0]); // 1

interface Cola<T> {
  [index: number]: T;
  push(item: T): void;
  pop(): T | undefined;
}
```

===============================================================================
9. INTERFACES VS TYPE ALIASES (AVANCE)
===============================================================================

Aunque lo veremos en detalle en el capítulo 26, es bueno saber que las
interfaces y los type aliases comparten muchas capacidades:

| Característica         | Interface | Type |
|------------------------|-----------|------|
| Extender otras         | ✅ extends | ✅ & |
| Uniones                | ❌         | ✅   |
| Propiedades opcionales | ✅         | ✅   |
| Readonly               | ✅         | ✅   |
| Declaration merging    | ✅         | ❌   |
| Mapped types           | ❌         | ✅   |

RESUMEN DEL CAPÍTULO
===============================================================================

- Las interfaces definen contratos estructurales para objetos
- Usa `?` para propiedades opcionales y `readonly` para propiedades inmutables
- Las interfaces pueden contener métodos y propiedades
- Las interfaces se extienden con `extends` (herencia simple y múltiple)
- Las index signatures permiten propiedades dinámicas
- Las interfaces describen firmas de funciones y arrays
- TypeScript usa tipado estructural, no nominal
- Las interfaces no existen en tiempo de ejecución

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener una interfaz sin propiedades (vacía)?**
Sí, aunque raramente útil. Una interfaz vacía es compatible con cualquier
objeto.

❓ **¿Las interfaces soportan genéricos?**
Sí, completamente. Lo veremos en módulos avanzados.

❓ **¿Puedo reabrir una interfaz para añadir propiedades?**
Sí, esto se llama *declaration merging*. Si declaras dos interfaces con
el mismo nombre, se fusionan.

❓ **¿Las interfaces pueden tener propiedades privadas o protegidas?**
No. Las interfaces son públicas por naturaleza. Para miembros privados
se usan clases.
