BIENVENIDA DEL PROFESOR BÚHO

Querido estudiante…

¿Puede una función llamarse a sí misma?

Sí. Se llama RECURSIÓN.

Es una técnica donde una función se llama a sí misma
para resolver un problema dividiéndolo en versiones más pequeñas
del mismo problema.

Útil para: árboles, factoriales, Fibonacci, búsquedas, y más.

✓ Evitar el desbordamiento de pila.

===============================================================================
1. ¿CÓMO FUNCIONA LA RECURSIÓN?
===============================================================================

Toda función recursiva necesita:

1. Caso base: condición que DETIENE la recursión.
2. Caso recursivo: la función se llama a sí misma con un problema menor.

```javascript
function cuentaRegresiva(n) {
    // Caso base
    if (n <= 0) {
        console.log("¡Despegue!");
        return;
    }
    // Caso recursivo
    console.log(n);
    cuentaRegresiva(n - 1);
}

cuentaRegresiva(5);
// 5, 4, 3, 2, 1, ¡Despegue!
```

===============================================================================
2. FACTORIAL CON RECURSIÓN
===============================================================================

```javascript
function factorial(n) {
    // Caso base
    if (n === 0 || n === 1) {
        return 1;
    }
    // Caso recursivo
    return n * factorial(n - 1);
}

console.log(factorial(5));  // 120
// 5 * factorial(4)
// 5 * 4 * factorial(3)
// 5 * 4 * 3 * factorial(2)
// 5 * 4 * 3 * 2 * factorial(1)
// 5 * 4 * 3 * 2 * 1 = 120
```

===============================================================================
3. FIBONACCI
===============================================================================

```javascript
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

console.log(fibonacci(0));  // 0
console.log(fibonacci(1));  // 1
console.log(fibonacci(6));  // 8 (0,1,1,2,3,5,8)
```

⚠️ Fibonacci recursivo es lento para n > 30.
Solución: usar iteración o memoización.

===============================================================================
4. RECURSIÓN VS ITERACIÓN
===============================================================================

```javascript
// Factorial ITERATIVO
function factorialIterativo(n) {
    let resultado = 1;
    for (let i = 2; i <= n; i++) {
        resultado *= i;
    }
    return resultado;
}

// Factorial RECURSIVO
function factorialRecursivo(n) {
    if (n <= 1) return 1;
    return n * factorialRecursivo(n - 1);
}

// Iterativo: más eficiente en memoria.
// Recursivo: más elegante y legible para problemas recursivos naturales.
```

===============================================================================
5. RIESGO: STACK OVERFLOW
===============================================================================

Cada llamada recursiva ocupa memoria en la pila.
Demasiadas llamadas → desbordamiento de pila.

```javascript
function recursioInfinita() {
    return recursioInfinita();  // ❌ RangeError: Maximum call stack size exceeded
}
```

Siempre asegúrate de tener un caso base que se alcance.

MINI PROYECTO: EXPLORADOR DE DIRECTORIOS
===============================================================================

```javascript
let sistemaArchivos = {
    nombre: "/",
    tipo: "directorio",
    hijos: [
        {
            nombre: "Documentos",
            tipo: "directorio",
            hijos: [
                { nombre: "carta.txt", tipo: "archivo", tamano: 10 },
                { nombre: "foto.jpg", tipo: "archivo", tamano: 500 }
            ]
        },
        {
            nombre: "Música",
            tipo: "directorio",
            hijos: [
                { nombre: "cancion.mp3", tipo: "archivo", tamano: 3000 }
            ]
        },
        { nombre: "README.md", tipo: "archivo", tamano: 2 }
    ]
};

function explorar(nodo, nivel = 0) {
    let indentacion = "  ".repeat(nivel);

    if (nodo.tipo === "archivo") {
        console.log(indentacion + "📄 " + nodo.nombre + " (" + nodo.tamano + "KB)");
    } else {
        console.log(indentacion + "📁 " + nodo.nombre);
        for (let i = 0; i < nodo.hijos.length; i++) {
            explorar(nodo.hijos[i], nivel + 1);
        }
    }
}

explorar(sistemaArchivos);
// 📁 /
//   📁 Documentos
//     📄 carta.txt (10KB)
//     📄 foto.jpg (500KB)
//   📁 Música
//     📄 cancion.mp3 (3000KB)
//   📄 README.md (2KB)
```

===============================================================================
RESUMEN
===============================================================================

✔ Recursión: función que se llama a sí misma.
✔ Caso base: condición que detiene la recursión.
✔ Caso recursivo: llamada con problema menor.
✔ Útil para: árboles, factorial, Fibonacci, búsquedas.
✔ Peligro: stack overflow si no hay caso base o es muy profundo.
