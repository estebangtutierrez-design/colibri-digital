1. ¿QUÉ ES UNA FUNCIÓN?
===============================================================================

Una función es un bloque de código con un nombre,
que puedes ejecutar cuando quieras, cuantas veces quieras.

📌 **Analogía del microondas**

Un microondas tiene botones.
No necesitas saber cómo funciona internamente.
Solo pones la comida, presionas un botón y obtienes el resultado.

Las funciones son como esos botones.
Tú defines qué hace cada botón una vez,
y luego solo presionas (llamas) la función cuando la necesitas.

---

📌 **Sintaxis de declaración**

```javascript
function nombreDeLaFuncion() {
    // Código que ejecuta la función
}

---

🔵 **Ejemplo 1: Tu primera función**

```javascript
// Declaración
function saludar() {
    console.log("¡Hola desde una función!");
}

// Llamada
saludar();
saludar();
saludar();

// Resultado:
// ¡Hola desde una función!
// ¡Hola desde una función!
// ¡Hola desde una función!

---

📌 **Declarar vs Llamar**

Declarar: defines la función (el código no se ejecuta todavía).
Llamar: ejecutas la función (el código se ejecuta).

```javascript
function mensaje() {
    console.log("Esto solo se ejecuta cuando llamas a la función");
}

// Si solo declaras, no pasa nada.
// Tienes que LLAMAR:
mensaje();

===============================================================================
PARTES DE UNA FUNCIÓN
===============================================================================

📌 **Estructura completa**

```javascript
function nombre(parametro1, parametro2) {
    // Cuerpo de la función
    return resultado;
}

1. function: palabra clave para declarar.
2. nombre: identificador único (como las variables).
3. parámetros: valores de entrada (opcional).
4. cuerpo: código que ejecuta.
5. return: valor de salida (opcional).

---

🔵 **Ejemplo 2: Función con parámetros y return**

```javascript
function sumar(a, b) {
    let resultado = a + b;
    return resultado;
}

let total = sumar(5, 3);
console.log("Total: " + total);  // Total: 8

Desglose:
- a y b son parámetros (reciben valores cuando llamas).
- 5 y 3 son argumentos (los valores que pasas).
- return devuelve el resultado para usarlo fuera.

===============================================================================
TIPOS DE FUNCIONES
===============================================================================

📌 **Función que no recibe parámetros ni devuelve nada**

```javascript
function mostrarHora() {
    let ahora = new Date();
    console.log("Son las: " + ahora.toLocaleTimeString());
}

mostrarHora();  // Muestra la hora actual

📌 **Función que recibe parámetros pero no devuelve nada**

```javascript
function saludarPersona(nombre) {
    console.log("Hola " + nombre + "!");
}

saludarPersona("Ana");   // Hola Ana!
saludarPersona("Luis");  // Hola Luis!

📌 **Función que recibe parámetros y devuelve un valor**

```javascript
function calcularIVA(precio) {
    return precio * 0.21;
}

let iva = calcularIVA(100);
console.log("IVA: $" + iva);  // IVA: $21

📌 **Función que no recibe parámetros pero devuelve un valor**

```javascript
function obtenerSaludo() {
    return "¡Bienvenido al sistema!";
}

let mensaje = obtenerSaludo();
console.log(mensaje);  // ¡Bienvenido al sistema!

===============================================================================
RETURN
===============================================================================

return es la palabra clave que devuelve un valor desde la función.

📌 **Reglas del return**

1. Termina la ejecución de la función.
2. Devuelve el valor especificado.
3. Si no hay return, la función devuelve undefined.

---

🔵 **Ejemplo 3: Return termina la función**

```javascript
function probarReturn() {
    console.log("Antes del return");
    return "Valor devuelto";
    console.log("Después del return");  // ❌ Esto nunca se ejecuta
}

let resultado = probarReturn();
console.log(resultado);

// Resultado:
// Antes del return
// Valor devuelto

---

🔵 **Ejemplo 4: Función sin return**

```javascript
function sinReturn() {
    let x = 10;
    // No hay return
}

let valor = sinReturn();
console.log(valor);  // undefined

===============================================================================
ÁMBITO (SCOPE) DE FUNCIONES
===============================================================================

Las variables declaradas dentro de una función
solo existen dentro de esa función.

📌 **Variables locales vs globales**

```javascript
let global = "Soy global";  // Fuera de funciones

function miFuncion() {
    let local = "Soy local";  // Solo dentro de la función
    console.log(global);      // ✅ Puede acceder a global
    console.log(local);       // ✅ Puede acceder a local
}

miFuncion();
console.log(global);  // ✅ Puede acceder a global
console.log(local);   // ❌ Error: local is not defined

🦉 **Profesor Búho dice:**
"Siempre que puedas, usa variables locales dentro de funciones.
Así evitas conflictos y mantienes tu código organizado."

LA ANALOGÍA DEL PROFESOR BÚHO: LA FÁBRICA DE JUEGOS
===============================================================================

Imagina una fábrica de videojuegos.

Cada estación de trabajo es una función:
- Estación "Diseñar Personaje": recibe ideas, devuelve un personaje.
- Estación "Crear Escenario": recibe un personaje, devuelve un nivel.
- Estación "Probar Juego": recibe un nivel, devuelve un informe.

```javascript
function disenarPersonaje(nombre, poder) {
    return {
        nombre: nombre,
        poder: poder,
        vida: 100
    };
}

function crearEscenario(personaje, dificultad) {
    console.log("Creando escenario para " + personaje.nombre);
    return {
        enemigos: dificultad * 3,
        personaje: personaje
    };
}

function probarJuego(escenario) {
    if (escenario.enemigos > 10) {
        return "❌ Demasiados enemigos. Rediseñar.";
    }
    return "✅ Juego equilibrado. Apto para lanzamiento.";
}

// Usamos las funciones
let heroe = disenarPersonaje("Lucas", 85);
let nivel1 = crearEscenario(heroe, 2);
let resultado = probarJuego(nivel1);

console.log(resultado);  // ✅ Juego equilibrado. Apto para lanzamiento.

Cada función hace una cosa.
Cada función se puede reutilizar.
Juntas, construyen algo más grande.

===============================================================================
ERRORES COMUNES CON FUNCIONES
===============================================================================

❌ **Error 1: Llamar la función antes de declararla**

Con function declaration, no hay problema (hoisting):

```javascript
miFuncion();  // ✅ Funciona por hoisting

function miFuncion() {
    console.log("Hola");
}
```

Pero con function expression, sí hay problema:

```javascript
miFuncion();  // ❌ Error: miFuncion is not a function

let miFuncion = function() {
    console.log("Hola");
};
```

❌ **Error 2: Olvidar los paréntesis al llamar**

```javascript
function saludar() {
    console.log("Hola");
}

// ❌ Sin paréntesis: no ejecuta la función, devuelve la función misma
console.log(saludar);  // [Function: saludar]

// ✅ Con paréntesis: ejecuta la función
saludar();  // Hola

❌ **Error 3: Confundir parámetros con argumentos**

Parámetros: las variables que defines en la declaración.
Argumentos: los valores que pasas al llamar.

```javascript
// a y b son PARÁMETROS
function restar(a, b) {
    return a - b;
}

// 10 y 3 son ARGUMENTOS
console.log(restar(10, 3));  // 7
```

❌ **Error 4: No usar return cuando es necesario**

```javascript
function duplicar(n) {
    n * 2;  // ❌ Falta return
}

let resultado = duplicar(5);
console.log(resultado);  // undefined
```

```javascript
// ============================================================
// BIBLIOTECA DE FUNCIONES MATEMÁTICAS
// Mini proyecto del Capítulo 21
// ============================================================

function sumar(a, b) {
    return a + b;
}

function restar(a, b) {
    return a - b;
}

function multiplicar(a, b) {
    return a * b;
}

function dividir(a, b) {
    if (b === 0) {
        return "Error: No se puede dividir por cero";
    }
    return a / b;
}

function potencia(base, exponente) {
    return base ** exponente;
}

function factorial(n) {
    if (n < 0) return "Error: No existe factorial de negativos";
    if (n === 0 || n === 1) return 1;
    let resultado = 1;
    for (let i = 2; i <= n; i++) {
        resultado *= i;
    }
    return resultado;
}

function esPar(n) {
    return n % 2 === 0;
}

function esPrimo(n) {
    if (n <= 1) return false;
    for (let i = 2; i < n; i++) {
        if (n % i === 0) return false;
    }
    return true;
}

// Probamos las funciones
console.log("5 + 3 = " + sumar(5, 3));
console.log("10 - 4 = " + restar(10, 4));
console.log("6 x 7 = " + multiplicar(6, 7));
console.log("20 / 4 = " + dividir(20, 4));
console.log("2^10 = " + potencia(2, 10));
console.log("5! = " + factorial(5));
console.log("7 es par? " + esPar(7));
console.log("17 es primo? " + esPrimo(17));
```

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre declarar y llamar una función?**

Declarar es definirla (no ejecuta el código).
Llamar es ejecutarla (usa paréntesis).

❓ **¿Puedo tener una función sin nombre?**

Sí, se llaman funciones anónimas. Las verás en el próximo capítulo.

❓ **¿Cuántos parámetros puede tener una función?**

Los que necesites. No hay límite técnico.

❓ **¿Qué pasa si llamo una función con menos argumentos que parámetros?**

Los parámetros sin argumento son undefined.

❓ **¿Puedo llamar una función dentro de otra función?**

Sí. Es una práctica común y recomendada.
