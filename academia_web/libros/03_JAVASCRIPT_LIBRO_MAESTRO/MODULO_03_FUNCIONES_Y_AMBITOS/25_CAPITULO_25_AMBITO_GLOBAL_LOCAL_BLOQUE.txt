BIENVENIDA DEL PROFESOR BÚHO
===============================================================================

Querido estudiante…

¿Dónde vive una variable? ¿En toda la casa o solo en una habitación?

El ÁMBITO (scope) define dónde puede accederse a una variable.

Hay tres tipos:
- Global: en toda la casa.
- Local (función): solo en esa función.
- Bloque: solo dentro de { } (con let y const).

Hoy vas a entender cómo se organizan tus variables.

===============================================================================
OBJETIVOS DEL CAPÍTULO
===============================================================================

✓ Diferenciar ámbito global, local y de bloque.
✓ Entender cómo let y const respetan el bloque.
✓ Saber cómo var ignora el bloque.
✓ Evinar conflictos entre variables.

===============================================================================
1. ÁMBITO GLOBAL
===============================================================================

Variables declaradas fuera de cualquier función o bloque.

```javascript
let nombre = "Ana";  // Global

function saludar() {
    console.log("Hola " + nombre);  // ✅ Accede a global
}

saludar();     // Hola Ana
console.log(nombre);  // ✅ Ana
```

Las variables globales son accesibles desde cualquier parte.

===============================================================================
2. ÁMBITO LOCAL (DE FUNCIÓN)
===============================================================================

Variables declaradas DENTRO de una función.

```javascript
function despedir() {
    let mensaje = "Adiós";  // Local
    console.log(mensaje);   // ✅ Adiós
}

despedir();
console.log(mensaje);  // ❌ Error: mensaje is not defined
```

Las variables locales no existen fuera de la función.

===============================================================================
3. ÁMBITO DE BLOQUE (let y const)
===============================================================================

let y const solo existen dentro del bloque { } donde se declaran.

```javascript
if (true) {
    let x = 10;
    const y = 20;
    var z = 30;  // var ignora el bloque
    console.log(x);  // ✅ 10
    console.log(y);  // ✅ 20
}

console.log(x);  // ❌ Error
console.log(y);  // ❌ Error
console.log(z);  // ✅ 30 (var filtró fuera del bloque)
```

===============================================================================
4. VAR VS LET vs CONST
===============================================================================

```javascript
function ejemploVar() {
    if (true) {
        var x = 10;  // "Filtra" fuera del bloque
    }
    console.log(x);  // ✅ 10 (var no respeta bloque)
}

function ejemploLet() {
    if (true) {
        let x = 10;  // Solo vive en el bloque
    }
    console.log(x);  // ❌ Error
}
```

Regla: Siempre usa let y const. Nunca var (a menos que sepas por qué).

===============================================================================
5. ÁMBITO ANIDADO
===============================================================================

Los bloques pueden estar anidados. El interior puede acceder al exterior.

```javascript
let global = "🌍";

function exterior() {
    let externa = "🌳";

    function interior() {
        let interna = "🌱";
        console.log(global);   // ✅ 🌍
        console.log(externa);  // ✅ 🌳
        console.log(interna);  // ✅ 🌱
    }

    interior();
    console.log(interna);  // ❌ Error
}

exterior();
```

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: EL EDIFICIO
===============================================================================

Imagina un edificio de departamentos.

- Global: la calle (todos pueden verla).
- Función exterior: el edificio.
- Función interior: un departamento.

Desde el departamento ves la calle y el edificio.
Desde el edificio ves la calle, pero no dentro del departamento.
Desde la calle no ves ni el edificio ni los departamentos.

```javascript
let calle = "Av. Principal";  // Global

function edificio() {
    let nombreEdificio = "Torre A";  // Local de edificio

    function departamento() {
        let numDepto = "101";  // Local de departamento
        console.log(calle);         // ✅
        console.log(nombreEdificio); // ✅
        console.log(numDepto);      // ✅
    }

    departamento();
    console.log(numDepto);  // ❌
}

edificio();
console.log(nombreEdificio);  // ❌
console.log(numDepto);        // ❌
```

===============================================================================
ERRORES COMUNES
===============================================================================

❌ Error 1: Asumir que var respeta bloques
```javascript
for (var i = 0; i < 5; i++) { }
console.log(i);  // 5  ❌ i debería haber muerto en el for
```

❌ Error 2: Redeclarar sin querer
```javascript
let x = 10;
let x = 20;  // ❌ Error: ya declarada
```

❌ Error 3: Usar variable global cuando debería ser local
```javascript
let contador = 0;  // Global innecesaria
function incrementar() {
    contador++;
}
// Mejor: que la función reciba y devuelva el valor
```

📝 Ejercicio 1: ¿Qué imprime?
```javascript
let a = 1;
function prueba() {
    let a = 2;
    console.log(a);
}
prueba();
console.log(a);
// Respuesta: 2, 1 (la local no afecta a la global)
```

📝 Ejercicio 2: Bloque con let
```javascript
let x = 5;
{
    let x = 10;
    console.log(x);
}
console.log(x);
// Respuesta: 10, 5
```

===============================================================================
MINI PROYECTO: CONTADOR SEGURO
===============================================================================

```javascript
function crearContador() {
    let contador = 0;  // Solo vive dentro de crearContador

    return {
        incrementar: function() { contador++; },
        decrementar: function() { contador--; },
        obtenerValor: function() { return contador; }
    };
}

let miContador = crearContador();
miContador.incrementar();
miContador.incrementar();
console.log(miContador.obtenerValor());  // 2
// No hay forma de modificar contador desde fuera directamente.
```

===============================================================================
RESUMEN
===============================================================================

✔ Global: fuera de funciones, accesible desde cualquier parte.
✔ Local (función): dentro de una función, solo allí.
✔ Bloque: dentro de { }, solo con let/const.
✔ var ignora el bloque (ámbito de función).
✔ Los bloques internos acceden a externos, no al revés.
