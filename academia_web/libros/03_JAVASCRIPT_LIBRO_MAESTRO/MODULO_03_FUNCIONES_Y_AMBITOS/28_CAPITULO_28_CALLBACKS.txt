BIENVENIDA DEL PROFESOR BÚHO
===============================================================================

Querido estudiante…

Los callbacks son funciones que se pasan como argumentos
para que otra función las ejecute cuando sea necesario.

Son la base de la asincronía en JavaScript:
- Eventos del usuario (clic, teclado).
- Peticiones a servidores.
- Lectura de archivos.
- Temporizadores.

Hoy aprenderás a usarlos.

===============================================================================
OBJETIVOS DEL CAPÍTULO
===============================================================================

✓ Entender qué es un callback.
✓ Escribir funciones que acepten callbacks.
✓ Usar callbacks con temporizadores y eventos.
✓ Evitar el callback hell.

===============================================================================
1. ¿QUÉ ES UN CALLBACK?
===============================================================================

```javascript
function hacerAlgo(callback) {
    console.log("Haciendo algo...");
    callback();  // Ejecuta el callback
}

function cuandoTermine() {
    console.log("¡Terminé!");
}

hacerAlgo(cuandoTermine);
// Haciendo algo...
// ¡Terminé!
```

===============================================================================
2. CALLBACKS CON PARÁMETROS
===============================================================================

```javascript
function procesarUsuario(nombre, callback) {
    let usuario = nombre.toUpperCase();
    callback(usuario);
}

procesarUsuario("Ana", function(u) {
    console.log("Usuario procesado: " + u);
});
// Usuario procesado: ANA
```

===============================================================================
3. CALLBACKS EN TEMPORIZADORES
===============================================================================

```javascript
console.log("Inicio");

setTimeout(function() {
    console.log("Esto se ejecuta después de 2 segundos");
}, 2000);

console.log("Fin");

// Orden: Inicio → Fin → (2s después) Esto se ejecuta...
```

===============================================================================
4. CALLBACKS EN EVENTOS
===============================================================================

```javascript
// Simulación de evento clic
function simularClick(elemento, callback) {
    console.log("🔘 Usuario hizo clic en " + elemento);
    callback(elemento);
}

simularClick("Botón Guardar", function(el) {
    console.log("✅ Guardando datos desde: " + el);
});
```

===============================================================================
5. CALLBACK HELL
===============================================================================

Cuando anidas muchos callbacks, el código se vuelve ilegible:

```javascript
// 🚫 Callback Hell
obtenerUsuario(1, function(usuario) {
    obtenerPedidos(usuario.id, function(pedidos) {
        obtenerDetalle(pedidos[0].id, function(detalle) {
            console.log(detalle);
        });
    });
});
```

Solución: Promesas y async/await (lo verás más adelante).

===============================================================================
MINI PROYECTO: SIMULADOR DE DESCARGA
===============================================================================

```javascript
function simularDescarga(archivo, callback) {
    console.log("⏳ Descargando " + archivo + "...");

    setTimeout(function() {
        let tamano = Math.floor(Math.random() * 100) + 10;
        let resultado = {
            nombre: archivo,
            tamano: tamano + "MB",
            exito: true
        };
        callback(resultado);
    }, 2000);
}

function mostrarResultado(archivo, callbackExito, callbackError) {
    simularDescarga(archivo, function(resultado) {
        if (resultado.exito) {
            callbackExito(resultado);
        } else {
            callbackError(resultado);
        }
    });
}

mostrarResultado("foto.jpg",
    function(r) {
        console.log("✅ " + r.nombre + " descargado (" + r.tamano + ")");
    },
    function(r) {
        console.log("❌ Error descargando " + r.nombre);
    }
);
```

===============================================================================
RESUMEN
===============================================================================

✔ Callback: función pasada como argumento para ejecutar después.
✔ Útiles para: asincronía, eventos, procesamiento.
✔ Temporizadores: setTimeout(callback, ms).
✔ Callback Hell: múltiples callbacks anidados (difícil de leer).
✔ La solución moderna: Promesas y async/await.
