BIENVENIDA DEL PROFESOR BÚHO

Querido estudiante…

JavaScript ejecuta UNA cosa a la vez (single thread).
Pero muchas operaciones toman tiempo:
- Leer un archivo.
- Pedir datos a un servidor.
- Esperar un temporizador.

Si JS esperara bloqueado, la página se congelaría.
La solución: ASINCRONÍA.

"Empieza esto, no esperes, te aviso cuando termine."

===============================================================================
1. SÍNCRONO VS ASÍNCRONO
===============================================================================

```javascript
// SÍNCRONO: bloquea hasta terminar
console.log("Paso 1");
for (let i = 0; i < 1000000000; i++) { /* operación pesada */ }
console.log("Paso 2");  // Espera a que termine el bucle

// ASÍNCRONO: no bloquea
console.log("Inicio");

setTimeout(function() {
    console.log("Esto tarda 2 segundos");
}, 2000);

console.log("Fin (no espera al setTimeout)");

// Orden: Inicio → Fin → (2s) Esto tarda 2 segundos
```

===============================================================================
2. EL EVENT LOOP
===============================================================================

El event loop es el mecanismo que permite la asincronía:

1. Pila (Call Stack): ejecuta código síncrono.
2. APIs del navegador: setTimeout, fetch, etc.
3. Cola de tareas (Task Queue): callbacks esperando.
4. Event Loop: mueve tareas de la cola a la pila cuando está vacía.

```javascript
console.log("1");  // Pila

setTimeout(() => console.log("2"), 0);  // Va a la cola

console.log("3");  // Pila

// Orden: 1, 3, 2
// Aunque setTimeout es 0ms, va a la cola, no a la pila directo
```

===============================================================================
3. OPERACIONES ASÍNCRONAS COMUNES
===============================================================================

```javascript
// 1. Temporizadores
setTimeout(() => {}, 1000);
setInterval(() => {}, 1000);

// 2. Eventos del DOM (siempre asíncronos)
boton.addEventListener("click", () => {});

// 3. Peticiones de red
fetch("https://api.ejemplo.com/datos");

// 4. Almacenamiento
localStorage.getItem("clave");  // Síncrono
// IndexedDB, FileReader → asíncronos

// 5. Geolocalización
navigator.geolocation.getCurrentPosition(() => {});
```

===============================================================================
4. PILA DE LLAMADAS
===============================================================================

```javascript
function a() {
    console.log("a");
    b();
}

function b() {
    console.log("b");
    c();
}

function c() {
    console.log("c");
    // Pila: a → b → c
    // Se vacía: c → b → a
}

a();
// a, b, c

// Si pones un console.trace() dentro de c:
// c → b → a → (anonymous)
```

===============================================================================
5. EJEMPLO VISUAL DEL EVENT LOOP
===============================================================================

```javascript
console.log("🥇 Primero");

setTimeout(function cb1() {
    console.log("🥈 setTimeout 0ms");
}, 0);

setTimeout(function cb2() {
    console.log("🥉 setTimeout 100ms");
}, 100);

Promise.resolve().then(function() {
    console.log("🏅 Promise (microtask)");
});

console.log("🥇 Último síncrono");

// Orden:
// 1. "🥇 Primero"
// 2. "🥇 Último síncrono"
// 3. "🏅 Promise (microtask)"  ← Las microtasks van antes
// 4. "🥈 setTimeout 0ms"
// 5. "🥉 setTimeout 100ms"
```

===============================================================================
RESUMEN
===============================================================================

✔ JavaScript: single thread (una cosa a la vez).
✔ Asincronía: no bloquear mientras esperas.
✔ Event Loop: mueve tareas de la cola a la pila.
✔ Microtasks (Promesas) van antes que macrotasks (setTimeout).
✔ Operaciones asíncronas: timers, eventos, fetch, archivos.
