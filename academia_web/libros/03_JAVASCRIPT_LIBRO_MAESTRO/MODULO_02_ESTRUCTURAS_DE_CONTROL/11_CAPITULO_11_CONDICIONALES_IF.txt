1. ¿QUÉ ES UNA ESTRUCTURA CONDICIONAL?
===============================================================================

Una estructura condicional permite que tu programa tome decisiones.

No es magia.
Es una pregunta que tu código se hace a sí mismo:

"Si pasa esto, haz esto otro. Si no, haz esto otro."

---

📌 **Sintaxis básica del if**

```javascript
if (condicion) {
    // Código que se ejecuta si la condición es verdadera
}

La condición siempre va entre paréntesis ( ).
El código que se ejecuta va entre llaves { }.

Si la condición es true, el código se ejecuta.
Si es false, el código se salta.

---

🔵 **Ejemplo 1: El primer if de tu vida**

```javascript
let edad = 18;

if (edad >= 18) {
    console.log("Eres mayor de edad");
}

// Resultado en consola:
// Eres mayor de edad

¿Por qué se ejecutó?
Porque edad >= 18 es true (18 es mayor o igual que 18).

---

🔵 **Ejemplo 2: Cuando no se cumple**

```javascript
let edad = 15;

if (edad >= 18) {
    console.log("Eres mayor de edad");
}

// No pasa nada. No hay salida en consola.

¿Por qué no se ejecutó?
Porque edad >= 18 es false (15 NO es mayor o igual que 18).

🦉 **Profesor Búho dice:**
"Este es el error más común del principiante: esperar que algo pase
cuando la condición es false. Si quieres que pase algo en ambos casos,
necesitas if...else."

===============================================================================
IF...ELSE
===============================================================================

¿Qué pasa si quieres hacer algo cuando la condición es verdadera
y otra cosa cuando es falsa?

Ahí entra else.

📌 **Sintaxis de if...else**

```javascript
if (condicion) {
    // Código si la condición es true
} else {
    // Código si la condición es false
}

---

🔵 **Ejemplo 3: ¿Es par o impar?**

```javascript
let numero = 7;

if (numero % 2 === 0) {
    console.log(numero + " es par");
} else {
    console.log(numero + " es impar");
}

// Resultado:
// 7 es impar

Explicación:
- El operador % (módulo) devuelve el resto de una división.
- Si un número dividido por 2 da resto 0, es par.
- Si da resto 1, es impar.

---

🔵 **Ejemplo 4: Validación de acceso (caso real)**

Imagina que estás construyendo un sistema de login.

```javascript
let usuarioIngresado = "lucas";
let passwordIngresado = "1234";

let usuarioCorrecto = "lucas";
let passwordCorrecto = "1234";

if (usuarioIngresado === usuarioCorrecto && passwordIngresado === passwordCorrecto) {
    console.log("✅ Acceso concedido. Bienvenido " + usuarioIngresado);
} else {
    console.log("❌ Usuario o contraseña incorrectos");
}

// Resultado:
// ✅ Acceso concedido. Bienvenido lucas

Este es un ejemplo real. Los sistemas de login del mundo real
funcionan exactamente así, solo que con más seguridad.

¿Y si tienes más de dos opciones?

📌 **Sintaxis de if...else if...else**

```javascript
if (condicion1) {
    // Código si condicion1 es true
} else if (condicion2) {
    // Código si condicion2 es true (y condicion1 era false)
} else if (condicion3) {
    // Código si condicion3 es true (y las anteriores eran false)
} else {
    // Código si todas las condiciones anteriores son false
}

---

🔵 **Ejemplo 5: Calculadora de calificaciones**

```javascript
let nota = 85;

if (nota >= 90) {
    console.log("Calificación: A (Excelente)");
} else if (nota >= 80) {
    console.log("Calificación: B (Bueno)");
} else if (nota >= 70) {
    console.log("Calificación: C (Suficiente)");
} else if (nota >= 60) {
    console.log("Calificación: D (Insuficiente)");
} else {
    console.log("Calificación: F (Reprobado)");
}

// Resultado:
// Calificación: B (Bueno)

---

📌 **Regla importante: el orden importa**

Las condiciones se evalúan de arriba a abajo.
En cuanto una da true, las demás se ignoran.

```javascript
let nota = 95;

if (nota >= 60) {
    console.log("Aprobado");
} else if (nota >= 90) {
    console.log("Excelente"); // ❌ Esto nunca se ejecutará
}

// Resultado:
// Aprobado

¿El problema?
95 >= 60 es true, entonces entra ahí y nunca evalúa 95 >= 90.

Solución: ordenar de la más específica a la más general.

```javascript
let nota = 95;

if (nota >= 90) {
    console.log("Excelente");
} else if (nota >= 60) {
    console.log("Aprobado");
} else {
    console.log("Reprobado");
}

// Resultado:
// Excelente ✅

🦉 **Profesor Búho dice:**
"Este es uno de los errores más comunes incluso entre programadores
con experiencia. En las escaleras if-else if, el orden lo es todo.
Pon siempre las condiciones más restrictivas primero."

===============================================================================
OPERADORES DE COMPARACIÓN
===============================================================================

Para escribir condiciones necesitas conocer las preguntas que puedes hacer.

📌 **Tabla de operadores de comparación**

Operador	Significado	            Ejemplo true	    Ejemplo false
===	        Igualdad estricta	    5 === 5	            5 === "5"
!==	        Desigualdad estricta	5 !== "5"	        5 !== 5
>	        Mayor que	            5 > 3	            3 > 5
<	        Menor que	            3 < 5	            5 < 3
>=	        Mayor o igual	        5 >= 5	            4 >= 5
<=	        Menor o igual	        3 <= 5	            5 <= 3

📌 **¿Por qué "=== " y no "=="?**

JavaScript tiene dos tipos de igualdad:

- ==  : igualdad débil (compara solo el valor, hace conversión automática)
- === : igualdad estricta (compara valor Y tipo)

```javascript
console.log(5 == "5");   // true  (JavaScript convierte "5" a número)
console.log(5 === "5");  // false (número vs string, no son iguales)

🦉 **Profesor Búho dice:**
"Usa siempre === y !==. Son más predecibles y evitan errores extraños.
El == es una reliquia del pasado. No le des cuerda."

---

🔵 **Ejemplo 6: Comparaciones en acción**

```javascript
let a = 10;
let b = "10";
let c = 20;

console.log(a === b);   // false (número vs string)
console.log(a == b);    // true  (conversión automática, no recomendado)
console.log(a !== c);   // true  (10 es diferente de 20)
console.log(a > c);     // false (10 no es mayor que 20)
console.log(a <= c);    // true  (10 es menor o igual que 20)

TRUTHY Y FALSY
===============================================================================

En JavaScript, cualquier valor puede ser usado como condición.

Algunos valores se comportan como true (truthy).
Otros se comportan como false (falsy).

📌 **Valores falsy (se comportan como false)**

```javascript
false
0
""        (string vacío)
null
undefined
NaN

📌 **Valores truthy (se comportan como true)**

Todo lo demás:
```javascript
true
1
"hola"
[]
{}
Infinity
-1
"false"   (el string "false" NO es igual a false)

---

🔵 **Ejemplo 7: Truthy y falsy en acción**

```javascript
let nombre = "";

if (nombre) {
    console.log("El nombre es: " + nombre);
} else {
    console.log("No ingresaste ningún nombre");
}

// Resultado:
// No ingresaste ningún nombre

¿Por qué? Porque "" es falsy.

```javascript
let nombre = "Lucas";

if (nombre) {
    console.log("El nombre es: " + nombre);
} else {
    console.log("No ingresaste ningún nombre");
}

// Resultado:
// El nombre es: Lucas

¿Por qué? Porque "Lucas" es truthy.

---

```javascript
let email = prompt("Ingresa tu correo electrónico:");

===============================================================================
ERRORES COMUNES CON CONDICIONALES
===============================================================================

❌ **Error 1: Usar = en lugar de ===**

```javascript
let edad = 18;

// ❌ MAL
if (edad = 18) {
    console.log("Eres mayor de edad");
}

// ¿Qué pasó? edad = 18 no está COMPARANDO, está ASIGNANDO.
// La expresión edad = 18 devuelve 18, que es truthy.
// El if siempre se ejecuta, sin importar el valor original de edad.

// ✅ BIEN
if (edad === 18) {
    console.log("Eres mayor de edad");
}

Solución: Siempre usa === para comparar. Reserva = solo para asignar.

❌ **Error 2: Olvidar las llaves { }**

```javascript
let edad = 20;

// ❌ MAL (sin llaves, solo la primera línea pertenece al if)
if (edad >= 18)
    console.log("Eres mayor de edad");
    console.log("Puedes votar"); // ❌ Esta línea siempre se ejecuta

// ✅ BIEN
if (edad >= 18) {
    console.log("Eres mayor de edad");
    console.log("Puedes votar");
}

Solución: Siempre usa llaves { }, aunque el if tenga solo una línea.

❌ **Error 3: Punto y coma después del if**

```javascript
let edad = 18;

// ❌ MAL
if (edad >= 18); {  // El punto y coma TERMINA el if
    console.log("Eres mayor de edad");
}

// El if no hace nada. El bloque de código siempre se ejecuta.

// ✅ BIEN
if (edad >= 18) {
    console.log("Eres mayor de edad");
}

Solución: Nunca pongas punto y coma después del if.

❌ **Error 4: Condiciones muertas (dead code)**

```javascript
let numero = 10;

if (numero > 5) {
    console.log("El número es mayor que 5");
} else if (numero > 0) {
    // ❌ Este código NUNCA se ejecutará
    // Si numero > 5 es true, ya entró en el primer if.
    // Si numero > 5 es false, entonces numero <= 5,
    // y por lo tanto numero > 0 también es false (para 0 o menos).
    console.log("El número es positivo pero pequeño");
}

Solución: Revisa siempre la lógica de tus condiciones.
Asegúrate de que cada rama sea alcanzable.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre = y ===?**

= es para asignar un valor.
=== es para comparar dos valores.

❓ **¿Puedo tener un if sin llaves?**

Sí, pero solo se ejecuta la primera línea después del if.
Siempre usa llaves para evitar errores.

❓ **¿Cuántos else if puedo tener?**

Los que necesites. No hay límite técnico.

❓ **¿Qué pasa si ninguna condición se cumple y no hay else?**

El programa simplemente continúa. No pasa nada.

❓ **¿El if puede ir sin else?**

Sí. El else es opcional.

❓ **¿Qué son los valores truthy y falsy?**

Valores que se comportan como true o false cuando se usan en un if.
