===============================================================================
1. ¿QUÉ ES EL BUCLE FOR?
===============================================================================

El bucle for es una estructura de repetición que concentra
las tres partes del control del bucle en una sola línea:

1. **Inicialización**: dónde empieza el contador.
2. **Condición**: hasta cuándo se repite.
3. **Incremento**: cómo avanza el contador.

📌 **Sintaxis**

```javascript
for (inicializacion; condicion; incremento) {
    // Código que se repite
}

---

🔵 **Ejemplo 1: For básico**

```javascript
for (let i = 1; i <= 5; i++) {
    console.log("Número: " + i);
}

// Resultado:
// Número: 1
// Número: 2
// Número: 3
// Número: 4
// Número: 5

Desglose:
- let i = 1    → empieza en 1 (se ejecuta UNA vez al inicio)
- i <= 5       → se repite mientras i sea menor o igual a 5
- i++          → después de cada repetición, suma 1 a i

===============================================================================
LAS TRES PARTES DEL FOR
===============================================================================

📌 **Inicialización**

Se ejecuta UNA SOLA VEZ, al entrar al bucle.

```javascript
// Normalmente declaras una variable
for (let i = 0; ...)

// Puedes tener múltiples inicializaciones
for (let i = 0, j = 10; ...)

// La inicialización puede estar fuera
let i = 0;
for (; i < 5; i++) { ... }

📌 **Condición**

Se evalúa ANTES de cada iteración.
Si es true, se ejecuta el código.
Si es false, el bucle termina.

```javascript
for (let i = 0; i < 10; i++) { ... }  // Mientras i < 10
for (let i = 10; i > 0; i--) { ... }  // Mientras i > 0
for (let i = 0; i <= 100; i += 2) { ... }  // Pares

📌 **Incremento**

Se ejecuta DESPUÉS de cada iteración.

```javascript
i++      // Incremento de 1 en 1
i += 2   // Incremento de 2 en 2
i--      // Decremento de 1 en 1
i *= 2   // Multiplicación (crecimiento exponencial)

===============================================================================
FOR CON ARRAYS
===============================================================================

El uso más común de for es recorrer arrays.

📌 **Sintaxis para recorrer un array**

```javascript
for (let i = 0; i < array.length; i++) {
    console.log(array[i]);
}

---

🔵 **Ejemplo 2: Lista de estudiantes**

```javascript
let estudiantes = ["Ana", "Luis", "María", "Pedro", "Sofía"];

console.log("=== Lista de estudiantes ===");

for (let i = 0; i < estudiantes.length; i++) {
    console.log("Estudiante " + (i + 1) + ": " + estudiantes[i]);
}

// Resultado:
// Estudiante 1: Ana
// Estudiante 2: Luis
// Estudiante 3: María
// Estudiante 4: Pedro
// Estudiante 5: Sofía

---

🔵 **Ejemplo 3: Sumar elementos de un array**

```javascript
let numeros = [10, 20, 30, 40, 50];
let suma = 0;

for (let i = 0; i < numeros.length; i++) {
    suma += numeros[i];
}

console.log("La suma es: " + suma);

// Resultado:
// La suma es: 150

---

🔵 **Ejemplo 4: Buscar el número mayor**

```javascript
let valores = [45, 23, 89, 12, 67, 34];
let mayor = valores[0];  // Asumimos que el primero es el mayor

for (let i = 1; i < valores.length; i++) {
    if (valores[i] > mayor) {
        mayor = valores[i];
    }
}

console.log("El número mayor es: " + mayor);

// Resultado:
// El número mayor es: 89

===============================================================================
FOR VS WHILE
===============================================================================

📌 **Tabla comparativa**

Situación	                For	                    While
Contar del 1 al 10	        ✅ Natural	            ❌ Más verbose
Recorrer un array	        ✅ Natural	            ❌ Más verbose
Validar entrada del usuario	❌ Incómodo	            ✅ Natural
Número indefinido de reps	❌ Incómodo	            ✅ Natural
Legibilidad en conteos	    ✅ Excelente	            ❌ Regular

---

🔵 **Regla práctica**

Usa for cuando SABES cuántas veces quieres repetir.
Usa while cuando NO SABES cuántas veces (depende del usuario).

```javascript
// FOR: sabemos que son 10 números
for (let i = 0; i < 10; i++) {
    console.log(i);
}

// WHILE: no sabemos cuántos números ingresará el usuario
let numero;
do {
    numero = Number(prompt("Ingresa un número (0 para salir):"));
    if (numero !== 0) {
        console.log("Ingresaste: " + numero);
    }
} while (numero !== 0);

LA ANALOGÍA DEL PROFESOR BÚHO: LA FÁBRICA
===============================================================================

Imagina una fábrica de botellas.

For es como una cinta transportadora programada:
- Empieza con la botella 1 (inicialización).
- Sigue mientras haya botellas (condición).
- Avanza a la siguiente botella (incremento).

```javascript
let botellas = ["vidrio", "plástico", "aluminio", "vidrio", "plástico"];

console.log("=== Fábrica de botellas ===");

for (let i = 0; i < botellas.length; i++) {
    console.log("Procesando botella " + (i + 1) + ": " + botellas[i]);

    // Simulamos el proceso
    if (botellas[i] === "vidrio") {
        console.log("   → Lavando botella de vidrio");
    } else if (botellas[i] === "plástico") {
        console.log("   → Triturando botella de plástico");
    } else {
        console.log("   → Fundiendo botella de aluminio");
    }
}

console.log("=== Producción finalizada ===");

Ahora imagina una máquina que cuenta piezas.

Con while necesitas tres instrucciones separadas.
Con for, todo está junto, como un panel de control compacto.

===============================================================================
ERRORES COMUNES CON FOR
===============================================================================

❌ **Error 1: Usar <= cuando debería ser < (o viceversa)**

```javascript
let arr = ["a", "b", "c"];

// ❌ MAL: arr.length es 3, pero los índices van de 0 a 2
for (let i = 0; i <= arr.length; i++) {
    console.log(arr[i]);  // arr[3] es undefined
}

// Resultado: a, b, c, undefined

// ✅ BIEN: i < arr.length
for (let i = 0; i < arr.length; i++) {
    console.log(arr[i]);
}

// Resultado: a, b, c

Solución: Los índices de arrays van de 0 a length-1.
Usa i < array.length, no i <= array.length.

❌ **Error 2: Olvidar el incremento**

```javascript
// ❌ MAL: bucle infinito
for (let i = 0; i < 10;) {
    console.log(i);
    // Falta: i++
}

Solución: Nunca olvides la tercera parte del for.
Si necesitas omitirla, considera usar while.

❌ **Error 3: Punto y coma después del for**

```javascript
// ❌ MAL: el for no tiene cuerpo
for (let i = 0; i < 5; i++); {
    console.log("Hola");
}

// El for itera 5 veces sin hacer nada.
// "Hola" se imprime UNA sola vez (fuera del bucle).

Solución: Nunca pongas punto y coma después del for.

❌ **Error 4: Declarar la variable sin let**

```javascript
// ❌ MAL: i es global
for (i = 0; i < 5; i++) {
    console.log(i);
}

// ✅ BIEN: i está limitada al bucle
for (let i = 0; i < 5; i++) {
    console.log(i);
}

Solución: Siempre usa let en la inicialización del for.

MINI PROYECTO: ANALIZADOR DE CALIFICACIONES

```javascript
// ============================================================
// ANALIZADOR DE CALIFICACIONES
// Mini proyecto del Capítulo 15
// ============================================================

console.log("=== ANALIZADOR DE CALIFICACIONES ===");

// Pedimos cuántas calificaciones ingresará
let numCalificaciones = Number(prompt("¿Cuántas calificaciones vas a ingresar?"));

// Validamos
while (isNaN(numCalificaciones) || numCalificaciones <= 0) {
    numCalificaciones = Number(prompt("❌ Ingresa un número válido:"));
}

let calificaciones = [];
let suma = 0;

// Pedimos cada calificación con for
for (let i = 0; i < numCalificaciones; i++) {
    let calif = Number(prompt("Calificación " + (i + 1) + " de " + numCalificaciones + ":"));

    // Validación de cada calificación
    while (isNaN(calif) || calif < 0 || calif > 100) {
        calif = Number(prompt("❌ Ingresa una calificación válida (0-100):"));
    }

    calificaciones.push(calif);
    suma += calif;

    console.log("Nota " + (i + 1) + ": " + calif);
}

// Calculamos estadísticas
let promedio = suma / numCalificaciones;
let mayor = calificaciones[0];
let menor = calificaciones[0];
let aprobados = 0;

for (let i = 0; i < calificaciones.length; i++) {
    if (calificaciones[i] > mayor) mayor = calificaciones[i];
    if (calificaciones[i] < menor) menor = calificaciones[i];
    if (calificaciones[i] >= 60) aprobados++;
}

console.log("========== RESULTADOS ==========");
console.log("📊 Calificaciones: " + calificaciones.join(", "));
console.log("📈 Promedio: " + promedio.toFixed(2));
console.log("⬆️ Mayor: " + mayor);
console.log("⬇️ Menor: " + menor);
console.log("✅ Aprobados: " + aprobados + " de " + numCalificaciones);

let porcentajeAprobados = (aprobados / numCalificaciones) * 100;

if (porcentajeAprobados >= 80) {
    console.log("🏆 ¡Excelente rendimiento grupal!");
} else if (porcentajeAprobados >= 60) {
    console.log("👍 Buen rendimiento grupal");
} else {
    console.log("📚 Necesitan mejorar");
}

console.log("=== FIN DEL PROGRAMA ===");

---

- Resultado: Promedio 78.00, Mayor 100, Menor 45, Aprobados 4

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuándo usar for en lugar de while?**

Cuando sabes cuántas veces quieres repetir.
For es más compacto y legible para conteos.

❓ **¿Puedo tener múltiples variables en el for?**

Sí:
```javascript
for (let i = 0, j = 10; i < j; i++, j--) { ... }
```

❓ **¿Qué pasa si la condición del for nunca es true?**

El cuerpo del for no se ejecuta ni una vez.

❓ **¿Puedo omitir partes del for?**

Sí, pero los puntos y coma son obligatorios.
```javascript
let i = 0;
for (; i < 10; i++) { ... }  // Inicialización fuera
```

❓ **¿Por qué los arrays empiezan en 0?**

Por herencia del lenguaje C. El índice representa
el desplazamiento desde la dirección base de memoria.
