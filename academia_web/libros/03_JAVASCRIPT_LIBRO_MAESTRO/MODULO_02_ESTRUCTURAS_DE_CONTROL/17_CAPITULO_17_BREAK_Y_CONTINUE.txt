===============================================================================
1. BREAK: SALIR DEL BUCLE
===============================================================================

break termina el bucle inmediatamente.
El código continúa ejecutándose después del bucle.

📌 **Analogía del mapa del tesoro**

Imagina que estás buscando un tesoro en una isla.
Tienes un mapa con 100 ubicaciones.

Empiezas a cavar en la ubicación 1, luego 2, luego 3...
En la ubicación 7, encuentras el tesoro.

¿Sigues cavando en las 93 ubicaciones restantes?
No. Paras. Has encontrado lo que buscabas.

Eso es break.

---

🔵 **Ejemplo 1: Buscar un número en un array**

```javascript
let numeros = [10, 25, 7, 43, 89, 12, 56, 3];
let buscado = 43;
let encontrado = false;

for (let i = 0; i < numeros.length; i++) {
    console.log("Revisando posición " + i + ": " + numeros[i]);

    if (numeros[i] === buscado) {
        console.log("✅ ¡Encontrado " + buscado + " en posición " + i + "!");
        encontrado = true;
        break;  // Salimos del bucle
    }
}

if (!encontrado) {
    console.log("❌ " + buscado + " no está en la lista");
}

// Resultado:
// Revisando posición 0: 10
// Revisando posición 1: 25
// Revisando posición 2: 7
// Revisando posición 3: 43
// ✅ ¡Encontrado 43 en posición 3!

Sin break, seguiría revisando hasta el final.
Con break, nos detenemos en cuanto encontramos lo que buscamos.

---

🔵 **Ejemplo 2: Break en while (validación mejorada)**

```javascript
let intentos = 0;
let maxIntentos = 3;

while (intentos < maxIntentos) {
    let password = prompt("Ingresa la contraseña:");

    if (password === "secreta") {
        console.log("✅ Acceso concedido");
        break;  // Salimos del bucle aunque queden intentos
    }

    intentos++;
    console.log("❌ Intento " + intentos + " de " + maxIntentos);
}

if (intentos >= maxIntentos) {
    console.log("🔒 Cuenta bloqueada");
}

// Si acierta en el intento 1, no sigue preguntando.

===============================================================================
CONTINUE: SALTAR A LA SIGUIENTE ITERACIÓN
===============================================================================

continue salta el resto del código en la iteración actual
y pasa directamente a la siguiente.

📌 **Analogía del control de calidad**

Imagina una fábrica de frascos de mermelada.

Los frascos pasan por una cinta transportadora.
Si un frasco está roto, no lo empacas.
Lo sacas de la línea y sigues con el siguiente.

Eso es continue: "este elemento no sirve, pasemos al siguiente".

---

🔵 **Ejemplo 3: Mostrar solo números pares**

```javascript
for (let i = 1; i <= 10; i++) {
    if (i % 2 !== 0) {
        continue;  // Saltamos los impares
    }
    console.log("Par: " + i);
}

// Resultado:
// Par: 2
// Par: 4
// Par: 6
// Par: 8
// Par: 10

El continue salta la iteración si i es impar.
Solo se ejecuta console.log para los pares.

---

🔵 **Ejemplo 4: Filtrar datos inválidos**

```javascript
let datos = [10, "abc", 25, "", 30, null, 45, undefined, 50];

console.log("=== PROCESANDO DATOS ===");

for (let i = 0; i < datos.length; i++) {
    if (datos[i] === null || datos[i] === undefined || datos[i] === "") {
        console.log("⚠️ Dato inválido en posición " + i + ". Omitiendo...");
        continue;
    }

    if (typeof datos[i] !== "number") {
        console.log("⚠️ Dato no numérico en posición " + i + ". Omitiendo...");
        continue;
    }

    console.log("✅ Dato válido: " + (datos[i] * 2));
}

// Resultado:
// ✅ Dato válido: 20
// ⚠️ Dato no numérico en posición 1. Omitiendo...
// ✅ Dato válido: 50
// ⚠️ Dato inválido en posición 3. Omitiendo...
// ✅ Dato válido: 60
// ⚠️ Dato inválido en posición 5. Omitiendo...
// ✅ Dato válido: 90
// ⚠️ Dato inválido en posición 7. Omitiendo...
// ✅ Dato válido: 100

===============================================================================
BREAK VS CONTINUE
===============================================================================

📌 **Tabla comparativa**

Instrucción	Efecto	                            ¿Sale del bucle?
break	    Termina el bucle completamente	    Sí
continue	Salta a la siguiente iteración	    No

---

🔵 **Ejemplo 5: Visualizar la diferencia**

```javascript
console.log("=== CON BREAK ===");
for (let i = 1; i <= 5; i++) {
    if (i === 3) break;
    console.log(i);
}
// 1, 2

console.log("=== CON CONTINUE ===");
for (let i = 1; i <= 5; i++) {
    if (i === 3) continue;
    console.log(i);
}
// 1, 2, 4, 5

Break: cuando llega a 3, TODO el bucle termina.
Continue: cuando llega a 3, solo salta esa iteración.

===============================================================================
BREAK CON BUCLES ANIDADOS
===============================================================================

Por defecto, break solo afecta al bucle más interno donde se encuentra.

Para salir de un bucle exterior, necesitas ETIQUETAS (labels).

📌 **Sintaxis de etiquetas**

```javascript
nombreEtiqueta: for (...) {
    for (...) {
        break nombreEtiqueta;  // Sale del bucle con esa etiqueta
    }
}

---

🔵 **Ejemplo 6: Buscar en matriz con break etiquetado**

```javascript
let matriz = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
];

let buscado = 5;
let encontrado = false;

busqueda: for (let i = 0; i < matriz.length; i++) {
    for (let j = 0; j < matriz[i].length; j++) {
        console.log("Revisando [" + i + "][" + j + "]: " + matriz[i][j]);

        if (matriz[i][j] === buscado) {
            console.log("✅ ¡Encontrado!");
            encontrado = true;
            break busqueda;  // Sale de AMBOS bucles
        }
    }
}

// Resultado:
// Revisando [0][0]: 1
// Revisando [0][1]: 2
// Revisando [0][2]: 3
// Revisando [1][0]: 4
// Revisando [1][1]: 5
// ✅ ¡Encontrado!

Sin la etiqueta, break solo saldría del bucle interior.

LA ANALOGÍA DEL PROFESOR BÚHO: EL ASCENSOR
===============================================================================

Imagina un edificio de 10 pisos.

Break es como el botón de "PISO DESTINO".
En cuanto llegas a tu piso, te bajas.
No sigues subiendo hasta el piso 10.

```javascript
let miPiso = 5;

for (let piso = 1; piso <= 10; piso++) {
    console.log("Pasando por piso " + piso);

    if (piso === miPiso) {
        console.log("🚪 Llegué a mi piso. Saliendo...");
        break;
    }
}
// Resultado: 1, 2, 3, 4, 5 y para.

Continue es como saltarse un piso en mantenimiento.

```javascript
let pisoEnMantenimiento = 4;

for (let piso = 1; piso <= 10; piso++) {
    if (piso === pisoEnMantenimiento) {
        console.log("🔧 Piso " + piso + " en mantenimiento. Saltando...");
        continue;
    }
    console.log("Deteniéndose en piso " + piso);
}
// Se detiene en todos los pisos excepto el 4.

===============================================================================
ERRORES COMUNES CON BREAK Y CONTINUE
===============================================================================

❌ **Error 1: Usar break fuera de un bucle o switch**

```javascript
// ❌ MAL: break solo funciona en bucles y switch
if (true) {
    break;  // Error: break no está en un bucle ni switch
}

Solución: break solo se usa dentro de for, while, do...while o switch.

❌ **Error 2: Usar continue en switch**

```javascript
switch (opcion) {
    case 1:
        if (condicion) continue;  // ❌ continue no funciona en switch
        break;
}

Solución: continue solo funciona dentro de bucles.

❌ **Error 3: Olvidar que break solo afecta al bucle más interno**

```javascript
for (let i = 0; i < 3; i++) {
    console.log("Exterior: " + i);
    for (let j = 0; j < 3; j++) {
        if (j === 1) break;  // ❌ Solo sale del bucle interior
        console.log("  Interior: " + j);
    }
}

// Resultado:
// Exterior: 0
//   Interior: 0
// Exterior: 1
//   Interior: 0
// Exterior: 2
//   Interior: 0

Solución: Si necesitas salir de todos, usa etiquetas.

❌ **Error 4: Abusar de break y continue**

A veces, un simple if es más claro.

```javascript
// ❌ Complicado
for (let i = 0; i < 10; i++) {
    if (i === 5) continue;
    if (i === 8) break;
    console.log(i);
}

// ✅ Más claro
for (let i = 0; i < 8; i++) {
    if (i === 5) continue;  // Solo este tiene sentido
    console.log(i);
}

Solución: No uses break/continue si un cambio en la condición lo resuelve.

Suma solo los números positivos de un array. Usa continue
para saltar los negativos o cero.

Solución:
```javascript
let valores = [10, -5, 20, -8, 0, 15, -3, 25];
let suma = 0;

for (let i = 0; i < valores.length; i++) {
    if (valores[i] <= 0) continue;
    suma += valores[i];
}

📝 **Ejercicio 3: Salir al encontrar un string vacío**

Pide nombres hasta que el usuario ingrese un nombre vacío.
Usa break para salir.

Solución:
```javascript
let nombres = [];

while (true) {
    let nombre = prompt("Ingresa un nombre (vacío para terminar):");

    if (nombre === null || nombre === "") {
        console.log("👋 Terminando...");
        break;
    }

    nombres.push(nombre);
    console.log("✅ " + nombre + " agregado a la lista");
}

console.log("Nombres ingresados: " + nombres.join(", "));

📝 **Ejercicio 4: Imprimir solo los que empiezan con "A"**

Dado un array de palabras, imprime solo las que empiezan con "A"
(o "a"), saltando las demás con continue.

Solución:
```javascript
let palabras = ["Ana", "casa", "auto", "perro", "avión", "gato"];

for (let i = 0; i < palabras.length; i++) {
    if (!palabras[i].toLowerCase().startsWith("a")) continue;
    console.log(palabras[i]);
}

// Ana, auto, avión

📝 **Ejercicio 5: Break en while con centinela**

Usa while(true) con break para crear un menú interactivo.

Solución:
```javascript
while (true) {
    let opcion = prompt(
        "1. Saludar\n" +
        "2. Mostrar hora\n" +
        "3. Salir\n" +
        "Elige:"
    );

    switch (opcion) {
        case "1":
            alert("¡Hola!");
            break;  // break del switch, no del while
        case "2":
            alert(new Date().toLocaleTimeString());
            break;
        case "3":
            alert("👋 Adiós");
            break;  // break del switch
        default:
            alert("Opción inválida");
    }

    if (opcion === "3") break;  // break del while
}

MINI PROYECTO: PROCESADOR DE CALIFICACIONES

```javascript
// ============================================================
// PROCESADOR DE CALIFICACIONES
// Mini proyecto del Capítulo 17
// ============================================================

console.log("=== PROCESADOR DE CALIFICACIONES ===");

let calificaciones = [];

// Pedimos calificaciones hasta que el usuario ingrese -1
console.log("Ingresa calificaciones (0-100). Escribe -1 para terminar.");

while (true) {
    let entrada = prompt("Calificación:");

    if (entrada === null) {
        console.log("⚠️ Entrada cancelada.");
        break;
    }

    let calif = Number(entrada);

    if (calif === -1) {
        console.log("📊 Procesando calificaciones...");
        break;
    }

    if (isNaN(calif) || calif < 0 || calif > 100) {
        console.log("❌ Calificación inválida. Ingresa un número entre 0 y 100.");
        continue;  // Saltamos esta iteración
    }

    calificaciones.push(calif);
    console.log("✅ Calificación " + calif + " registrada.");
}

console.log("\n========== RESULTADOS ==========");

if (calificaciones.length === 0) {
    console.log("No se ingresaron calificaciones.");
} else {
    // Procesamos con continue para saltar valores atípicos
    let suma = 0;
    let contadorValidos = 0;
    let aprobados = 0;
    let reprobados = 0;

    for (let i = 0; i < calificaciones.length; i++) {
        let c = calificaciones[i];

        // Filtramos valores atípicos (menos de 0 o más de 100)
        if (c < 0 || c > 100) {
            console.log("⚠️ Dato atípico: " + c + ". Omitiendo...");
            continue;
        }

        suma += c;
        contadorValidos++;

        if (c >= 60) {
            aprobados++;
        } else {
            reprobados++;
        }
    }

    let promedio = suma / contadorValidos;

    console.log("📊 Total de calificaciones: " + calificaciones.length);
    console.log("📈 Promedio: " + promedio.toFixed(2));
    console.log("✅ Aprobados: " + aprobados);
    console.log("❌ Reprobados: " + reprobados);

    // Buscamos si alguien tiene calificación perfecta con break
    for (let i = 0; i < calificaciones.length; i++) {
        if (calificaciones[i] === 100) {
            console.log("🏆 ¡Calificación perfecta encontrada!");
            break;
        }
    }
}

console.log("=== FIN DEL PROGRAMA ===");

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Break funciona en if?**

No. break solo funciona en bucles (for, while, do...while) y switch.

❓ **¿Continue funciona en switch?**

No. continue solo funciona en bucles.

❓ **¿Puedo tener múltiples breaks en un bucle?**

Sí. Puedes tener tantos breaks como necesites.

❓ **¿Qué es while(true)?**

Un bucle que nunca termina por sí solo.
Necesitas un break dentro para salir.

❓ **¿Break es mala práctica?**

No, pero úsalo con moderación.
Si ves que abusas de break, reconsidera la lógica.
