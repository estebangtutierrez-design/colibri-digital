===============================================================================
1. ¿QUÉ ES EL MANEJO DE ERRORES?
===============================================================================

try...catch es una estructura que permite "intentar" ejecutar código
y "atrapar" cualquier error que ocurra.

📌 **Sintaxis básica**

```javascript
try {
    // Código que puede fallar
} catch (error) {
    // Código que se ejecuta si hay error
}

---

📌 **Analogía de la red de seguridad**

Imagina que eres un trapecista en un circo.

Intestas un salto mortal (try).
Si fallas, la red de seguridad te atrapa (catch).
No importa si caes bien o mal, el show continúa.

---

🔵 **Ejemplo 1: Primer try...catch**

```javascript
console.log("Inicio del programa");

try {
    console.log("Intentando ejecutar código riesgoso...");
    let resultado = 10 / variableNoDefinida;
    console.log("Esto no se ejecuta si hay error");
} catch (error) {
    console.log("❌ ¡Error atrapado!");
    console.log("Mensaje: " + error.message);
}

console.log("El programa continúa normalmente");

// Resultado:
// Inicio del programa
// Intentando ejecutar código riesgoso...
// ❌ ¡Error atrapado!
// Mensaje: variableNoDefinida is not defined
// El programa continúa normalmente

Sin try...catch, el programa se habría detenido.
Con try...catch, el programa continúa.

===============================================================================
TRY...CATCH EN PROFUNDIDAD
===============================================================================

📌 **El bloque try**

Contiene el código que PUEDE fallar.
Si falla, el resto del try se salta y se ejecuta el catch.

```javascript
try {
    let x = JSON.parse("texto inválido");  // Error aquí
    console.log("Esto nunca se ejecuta");  // Saltado
} catch (e) {
    console.log("Error de parsing");
}

📌 **El bloque catch**

Recibe el error como parámetro.
Puedes nombrarlo como quieras (error, e, err, etc.).

```javascript
try {
    let x = JSON.parse("{'mal': 'formato'}");
} catch (error) {
    console.log(error.name);     // Tipo de error
    console.log(error.message);  // Mensaje descriptivo
    console.log(error.stack);    // Pila de llamadas (debug)
}

---

🔵 **Ejemplo 2: Parseo de JSON (caso real)**

```javascript
let jsonInvalido = "{nombre: 'Lucas'}";  // Falta comillas en la clave

try {
    let usuario = JSON.parse(jsonInvalido);
    console.log("Usuario:", usuario);
} catch (error) {
    console.log("❌ Error al procesar los datos del usuario");
    console.log("Detalle: " + error.message);
    console.log("Usando datos por defecto...");

    // Valor por defecto
    let usuario = { nombre: "Invitado", edad: 0 };
    console.log("Usuario por defecto:", usuario);
}

// Resultado:
// ❌ Error al procesar los datos del usuario
// Detalle: Unexpected identifier 'nombre'
// Usando datos por defecto...
// Usuario por defecto: { nombre: "Invitado", edad: 0 }

===============================================================================
FINALLY: SIEMPRE SE EJECUTA
===============================================================================

finally se ejecute SIEMPRE, haya o no error.

📌 **Sintaxis con finally**

```javascript
try {
    // Código que puede fallar
} catch (error) {
    // Código si hay error
} finally {
    // Código que SIEMPRE se ejecuta
}

---

🔵 **Ejemplo 3: Limpieza con finally**

```javascript
function procesarArchivo() {
    console.log("📂 Abriendo archivo...");

    try {
        console.log("📝 Procesando archivo...");

        // Simulamos un error
        if (Math.random() > 0.5) {
            throw new Error("Archivo corrupto");
        }

        console.log("✅ Archivo procesado exitosamente");
    } catch (error) {
        console.log("❌ Error: " + error.message);
    } finally {
        console.log("🔒 Cerrando archivo...");
        console.log("---");
    }
}

// Ejecutamos 3 veces
procesarArchivo();
procesarArchivo();
procesarArchivo();

// finally siempre cierra el archivo, haya o no error.

🦉 **Profesor Búho dice:**
"finally es perfecto para limpieza: cerrar archivos,
liberar memoria, cerrar conexiones. Siempre se ejecuta."

===============================================================================
THROW: LANZAR TUS PROPIOS ERRORES
===============================================================================

No solo puedes atrapar errores: también puedes lanzarlos.

📌 **Sintaxis de throw**

```javascript
throw new Error("Mensaje de error");

---

🔵 **Ejemplo 4: Validación con throw**

```javascript
function calcularRaizCuadrada(numero) {
    if (numero < 0) {
        throw new Error("No se puede calcular la raíz cuadrada de un número negativo");
    }
    return Math.sqrt(numero);
}

// Usamos la función con try...catch
try {
    let resultado = calcularRaizCuadrada(-9);
    console.log("Resultado: " + resultado);
} catch (error) {
    console.log("❌ " + error.message);
}

// Resultado:
// ❌ No se puede calcular la raíz cuadrada de un número negativo

---

🔵 **Ejemplo 5: Validación de formulario con throw**

```javascript
function validarEdad(edad) {
    if (edad === undefined || edad === null) {
        throw new Error("La edad es obligatoria");
    }
    if (typeof edad !== "number") {
        throw new Error("La edad debe ser un número");
    }
    if (edad < 0 || edad > 120) {
        throw new Error("La edad debe estar entre 0 y 120");
    }
    if (!Number.isInteger(edad)) {
        throw new Error("La edad debe ser un número entero");
    }
    return true;
}

// Probamos
try {
    validarEdad(25.5);  // No es entero
    console.log("✅ Edad válida");
} catch (error) {
    console.log("❌ " + error.message);
}

// Resultado:
// ❌ La edad debe ser un número entero

===============================================================================
TIPOS DE ERROR EN JAVASCRIPT
===============================================================================

JavaScript tiene varios tipos de error incorporados.

📌 **Tipos principales**

Tipo	                    Cuándo ocurre
ReferenceError	            Acceder a variable no definida
TypeError	                Usar un valor de forma incorrecta
SyntaxError	                Código con sintaxis inválida
RangeError	                Número fuera de rango permitido
URIError	                Funciones URI mal formadas

---

🔵 **Ejemplo 6: Diferentes tipos de error**

```javascript
// ReferenceError
try {
    console.log(variableNoDefinida);
} catch (e) {
    console.log(e.name + ": " + e.message);
}

// TypeError
try {
    null.toUpperCase();
} catch (e) {
    console.log(e.name + ": " + e.message);
}

// RangeError
try {
    let arr = new Array(-1);
} catch (e) {
    console.log(e.name + ": " + e.message);
}

// Resultado:
// ReferenceError: variableNoDefinida is not defined
// TypeError: Cannot read properties of null
// RangeError: Invalid array length

===============================================================================
TRY...CATCH VS VALIDACIÓN
===============================================================================

A veces puedes prevenir errores con validación (if).
A veces necesitas try...catch.

📌 **Cuándo usar cada uno**

Situación	                            Validación (if)	    try...catch
El usuario ingresa un número	            ✅ Sí	            ❌ Excesivo
Parsear JSON de origen externo	            ❌ No siempre	    ✅ Necesario
Acceder a una variable que existe	        ✅ Sí	            ❌ Excesivo
Leer datos de una API	                    ❌ Impredecible	    ✅ Necesario
Dividir por cero	                        ✅ Sí	            ❌ Excesivo
Convertir string a número	                ✅ Sí	            ❌ Excesivo

---

🔵 **Ejemplo 7: Prevenir con if vs atrapar con catch**

```javascript
// ✅ Mejor: validar con if
function dividirIf(a, b) {
    if (b === 0) {
        return "Error: No se puede dividir por cero";
    }
    return a / b;
}

// ❌ Peor: esperar el error
function dividirCatch(a, b) {
    try {
        return a / b;
    } catch (e) {
        return "Error: " + e.message;
    }
}

// En este caso, el if es más claro.
// b === 0 lo puedes saber antes de ejecutar la división.

🦉 **Profesor Búho dice:**
"Usa validación cuando puedas predecir el error.
Usa try...catch cuando el error sea impredecible
(API externa, archivos, entrada del usuario compleja)."

LA ANALOGÍA DEL PROFESOR BÚHO: EL RESTAURANTE

Imagina un restaurante.

try     = El cocinero intenta preparar el plato.
catch   = Si algo sale mal, el mesero se disculpa y ofrece alternativas.
finally = Siempre lavan los platos al final, haya salido bien o mal.
throw   = El cocinero detecta un ingrediente en mal estado y lo notifica.

```javascript
function prepararPlato(ingrediente) {
    console.log("👨‍🍳 Cocinero: Intentando preparar el plato...");

    try {
        if (!ingrediente) {
            throw new Error("No hay ingredientes");

        if (ingrediente === "pescado" && Math.random() > 0.7) {
            throw new Error("El pescado está en mal estado");
        }
        console.log("✅ Plato preparado: " + ingrediente);
    } catch (error) {
        console.log("❌ " + error.message);
        console.log("   Ofreciendo plato alternativo...");
    } finally {
        console.log("🧹 Limpiando la cocina...");
        console.log("---");
    }
}

prepararPlato("carne");

```

===============================================================================
ERRORES COMUNES CON TRY...CATCH
===============================================================================

❌ **Error 1: Atrapar errores que no deberías**

```javascript
// ❌ MAL: atrapar un error que puedes prevenir
try {
    let edad = Number(prompt("Edad:"));
    if (edad < 0) throw new Error("Edad negativa");
    console.log(edad);
} catch (e) {
    console.log("Error");
}

// ✅ BIEN: validar antes
let edad = Number(prompt("Edad:"));
if (edad < 0) {
    console.log("La edad no puede ser negativa");
} else {
    console.log(edad);
}

Solución: No uses try...catch para control de flujo normal.

❌ **Error 2: Catch vacío**

```javascript
// ❌ MAL: tragar el error sin hacer nada
try {
    JSON.parse(datos);
} catch (e) {
    // No haces nada. El error desaparece.
}

Solución: Siempre registra el error (console.log)
o muestra algo al usuario.

❌ **Error 3: No usar el parámetro de error**

```javascript
// ❌ MAL: no usas la información del error
try {
    JSON.parse(datos);
} catch {
    console.log("Error");
}

// ✅ BIEN: usas el error para dar contexto
try {
    JSON.parse(datos);
} catch (error) {
    console.log("Error al procesar datos: " + error.message);
}

❌ **Error 4: Finally sin try**

```javascript
// ❌ MAL: finally no funciona sin try
finally {
    console.log("Esto no funciona solo");
}

Solución: finally siempre va después de try o try...catch.

try {
    let num1 = Number(prompt("Primer número:"));

    let operacion = prompt("Operación (+, -, *, /):");

MINI PROYECTO: PROCESADOR DE DATOS SEGURO
===============================================================================

```javascript
// ============================================================
// PROCESADOR DE DATOS SEGURO
// Mini proyecto del Capítulo 18
// ============================================================

console.log("=== PROCESADOR DE DATOS SEGURO ===");

function procesarUsuario(datosJSON) {
    let usuario;

    // Paso 1: Parsear JSON
    try {
        usuario = JSON.parse(datosJSON);
        console.log("✅ JSON parseado correctamente");
    } catch (error) {
        console.log("❌ Error al parsear JSON: " + error.message);
        console.log("   Usando datos por defecto...");
        usuario = { nombre: "Invitado", edad: 0 };
    }

    // Paso 2: Validar datos (con finally para registrar)
    let resultado;
    try {
        resultado = validarUsuario(usuario);
        console.log("✅ Usuario válido:", usuario.nombre);
    } catch (error) {
        console.log("❌ Validación fallida: " + error.message);
        resultado = { nombre: "Invitado", edad: 0 };
    } finally {
        console.log("📝 Registro de procesamiento completado");
    }

    return resultado;
}

function validarUsuario(usuario) {
    if (!usuario.nombre || typeof usuario.nombre !== "string") {
        throw new Error("Nombre inválido o faltante");
    }
    if (!usuario.edad || typeof usuario.edad !== "number") {
        throw new Error("Edad inválida o faltante");
    }
    if (usuario.edad < 0 || usuario.edad > 120) {
        throw new Error("Edad fuera de rango");
    }
    if (!usuario.email || !usuario.email.includes("@")) {
        throw new Error("Email inválido");
    }
    return usuario;
}

// Pruebas
let jsonValido = '{"nombre": "Lucas", "edad": 30, "email": "lucas@email.com"}';
let jsonInvalido = '{"nombre": "Ana", "edad": -5, "email": "sin-arroba"}';
let jsonMalFormado = "{nombre:}";

console.log("\n--- Prueba 1: JSON válido ---");
procesarUsuario(jsonValido);

console.log("\n--- Prueba 2: JSON inválido (validación) ---");
procesarUsuario(jsonInvalido);

console.log("\n--- Prueba 3: JSON mal formado ---");
procesarUsuario(jsonMalFormado);

console.log("\n=== FIN DEL PROGRAMA ===");
```

---

📌 **Prueba del proyecto**

Prueba 1: JSON válido → todo funciona.
Prueba 2: JSON válido pero datos inválidos → catch de validación.
Prueba 3: JSON mal formado → catch de parsing.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Qué pasa si no uso catch?**

Si hay error en try, el programa se rompe aunque haya finally.

❓ **¿Puedo tener varios catch?**

No en JavaScript. Solo un catch por try.
Puedes usar instanceof dentro del catch para diferenciar tipos.

❓ **¿Finally es obligatorio?**

No. Puedes tener try...catch sin finally.

❓ **¿Qué es error.stack?**

Es la pila de llamadas (call stack): muestra dónde ocurrió el error.
Útil para debugging.

❓ **¿Puedo lanzar cualquier tipo de valor?**

Sí, pero es mejor lanzar objetos Error.
throw "mensaje" funciona, pero pierdes información.
