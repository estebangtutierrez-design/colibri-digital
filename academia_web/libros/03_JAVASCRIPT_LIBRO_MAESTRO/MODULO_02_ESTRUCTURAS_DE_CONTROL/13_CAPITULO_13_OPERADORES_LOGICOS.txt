===============================================================================
1. ¿QUÉ SON LOS OPERADORES LÓGICOS?
===============================================================================

Los operadores lógicos combinan condiciones booleanas.

Trabajan con valores true y false, y devuelven true o false
(o a veces, el valor original, como veremos más adelante).

📌 **Los tres operadores lógicos**

Operador	Nombre	        Significado
&&	        AND (Y)	        Todas las condiciones deben ser true
||	        OR (O)	        Al menos una condición debe ser true
!	        NOT (NO)	    Invierte el valor (true → false, false → true)

---

🔵 **Ejemplo 1: AND en la vida real**

"Para entrar al cine necesitas tener boleto Y tener edad suficiente."

```javascript
let tieneBoleto = true;
let edadSuficiente = true;

if (tieneBoleto && edadSuficiente) {
    console.log("Puedes entrar al cine");
} else {
    console.log("No puedes entrar al cine");
}

// Resultado:
// Puedes entrar al cine

Si cualquiera de las dos condiciones es false, no puedes entrar.

```javascript
let tieneBoleto = true;
let edadSuficiente = false;  // Es menor de edad

if (tieneBoleto && edadSuficiente) {
    console.log("Puedes entrar al cine");
} else {
    console.log("No puedes entrar al cine");
}

// Resultado:
// No puedes entrar al cine

---

🔵 **Ejemplo 2: OR en la vida real**

"Puedes pagar con tarjeta O puedes pagar en efectivo."

```javascript
let pagaConTarjeta = false;
let pagaEnEfectivo = true;

if (pagaConTarjeta || pagaEnEfectivo) {
    console.log("Pago aceptado");
} else {
    console.log("No tienes método de pago válido");
}

// Resultado:
// Pago aceptado

Con OR, basta con que UNA condición sea true.

---

🔵 **Ejemplo 3: NOT en la vida real**

"Si NO está lloviendo, podemos salir a caminar."

```javascript
let estaLloviendo = false;

if (!estaLloviendo) {
    console.log("Podemos salir a caminar");
} else {
    console.log("Mejor quedémonos en casa");
}

// Resultado:
// Podemos salir a caminar

! invierte el valor.
Si estaLloviendo es false, !estaLloviendo es true.

===============================================================================
TABLA DE VERDAD
===============================================================================

Las tablas de verdad muestran todas las combinaciones posibles.

📌 **Tabla de AND (&&)**

Condición 1	Condición 2	Resultado
true	    true	    true
true	    false	    false
false	    true	    false
false	    false	    false

AND solo da true cuando AMBAS son true.

📌 **Tabla de OR (||)**

Condición 1	Condición 2	Resultado
true	    true	    true
true	    false	    true
false	    true	    true
false	    false	    false

OR da true cuando AL MENOS UNA es true.

📌 **Tabla de NOT (!)**

Condición	Resultado
true	    false
false	    true

NOT invierte el valor.

===============================================================================
COMBINACIÓN DE OPERADORES LÓGICOS
===============================================================================

Puedes combinar &&, || y ! en una misma condición.

📌 **Reglas de precedencia**

1. ! (NOT) se evalúa primero.
2. && (AND) se evalúa después.
3. || (OR) se evalúa al final.

```javascript
// Orden de evaluación:
// 1. !pendiente → false
// 2. true && false → false
// 3. false || false → false

let esAdmin = true;
let esEditor = false;
let pendiente = true;

if (esAdmin || (esEditor && !pendiente)) {
    console.log("Acceso concedido");
} else {
    console.log("Acceso denegado");
}

// Resultado:
// Acceso concedido

¿Por qué?
1. !pendiente → false
2. esEditor && false → false && false → false
3. esAdmin || false → true || false → true

===============================================================================
EVALUACIÓN DE CORTOCIRCUITO
===============================================================================

JavaScript evalúa las condiciones de izquierda a derecha
y se detiene en cuanto sabe el resultado.

Esto se llama evaluación de cortocircuito (short-circuit evaluation).

📌 **Cortocircuito con &&**

Si la primera condición es false, JavaScript NI SIQUIERA evalúa la segunda.

```javascript
let usuario = null;

// ❌ Esto daría error si usuario fuera null
// console.log(usuario.nombre);

// ✅ Esto es seguro gracias al cortocircuito
if (usuario && usuario.nombre) {
    console.log("El nombre del usuario es: " + usuario.nombre);
} else {
    console.log("Usuario no encontrado");
}

// Resultado:
// Usuario no encontrado

¿Por qué es seguro?
usuario es null (falsy).
AND se detiene en la primera condición false.
Nunca evalúa usuario.nombre. No hay error.

---

📌 **Cortocircuito con ||**

Si la primera condición es true, JavaScript NI SIQUIERA evalúa la segunda.

```javascript
let nombre = "";
let nombrePorDefecto = nombre || "Invitado";

console.log(nombrePorDefecto);

// Resultado:
// Invitado

¿Por qué?
nombre es "" (falsy).
OR continúa evaluando.
"Invitatdo" es truthy.
OR devuelve "Invitado".

---

🔵 **Ejemplo 11: Valor por defecto (patrón profesional)**

```javascript
let nombreUsuario = prompt("Ingresa tu nombre:");
let nombreFinal = nombreUsuario || "Usuario anónimo";

console.log("Bienvenido, " + nombreFinal);

// Si el usuario ingresa "Lucas" → "Bienvenido, Lucas"
// Si el usuario cancela o deja vacío → "Bienvenido, Usuario anónimo"

Imagina que eres el guardia de una biblioteca secreta.

Para entrar necesitas:
- Tener una llave dorada (&&).
- Tener una llave plateada (&&).

Si tienes ambas, entras.
Si te falta una, no entras.

Eso es AND (&&).

---

Ahora imagina que es una fiesta.

Puedes entrar si:
- Eres miembro del club (||).
- Vienes invitado por un miembro (||).

Si cumples UNA de las dos, entras.

Eso es OR (||).

---

Ahora imagina que hay una lista negra.

Si NO estás en la lista negra, puedes entrar.
Si estás en la lista negra, no puedes entrar.

Eso es NOT (!).

---

```javascript
let llaveDorada = true;
let llavePlateada = false;
let esMiembro = true;
let esInvitado = false;
let enListaNegra = false;

// AND: necesita ambas llaves
if (llaveDorada && llavePlateada) {
    console.log("Entras a la biblioteca secreta");
} else {
    console.log("No puedes entrar a la biblioteca");
}

// OR: necesita al menos una
if (esMiembro || esInvitado) {
    console.log("Entras a la fiesta");
}

// NOT: inversión
if (!enListaNegra) {
    console.log("Puedes pasar, no estás en la lista negra");
}

ERRORES COMUNES CON OPERADORES LÓGICOS
===============================================================================

❌ **Error 1: Usar | o & en lugar de || o &&**

```javascript
let a = true;
let b = false;

// ❌ MAL: | y & son operadores de bits, no lógicos
if (a | b) { ... }

// ✅ BIEN: || y && son operadores lógicos
if (a || b) { ... }

Solución: || y && son dos caracteres. | y & son uno solo.

❌ **Error 2: Olvidar los paréntesis en combinaciones**

```javascript
let edad = 25;
let tienePermiso = true;

// ❌ MAL: ¿qué se evalúa primero?
if (edad >= 18 && tienePermiso || edad >= 65) { ... }

// ✅ BIEN: paréntesis explícitos
if ((edad >= 18 && tienePermiso) || edad >= 65) { ... }

Solución: Siempre usa paréntesis en combinaciones.

❌ **Error 3: Confundir la lógica de negación**

```javascript
let edad = 20;

// ❌ MAL: "Si NO es mayor de edad O NO tiene permiso"
// Quiere decir: si NO cumple con AMBOS requisitos
if (!(edad >= 18 && tienePermiso)) { ... }

// ✅ BIEN: Usa la lógica de De Morgan
// !(A && B) es equivalente a !A || !B
if (edad < 18 || !tienePermiso) { ... }

Solución: Recuerda las leyes de De Morgan:
- !(A && B) es igual a !A || !B
- !(A || B) es igual a !A && !B

❌ **Error 4: Asumir que todas las condiciones se evalúan**

JavaScript usa cortocircuito.
No asumas que la segunda condición siempre se ejecuta.

```javascript
let contador = 0;

// ❌ Peligro: si la primera condición es false, la segunda no se ejecuta
if (contador > 0 && ++contador > 0) {
    console.log(contador);
}

// contador sigue siendo 0

Solución: No pongas efectos secundarios (++contador) dentro de condiciones.

MINI PROYECTO: SISTEMA DE AUTENTICACIÓN
===============================================================================

Vamos a construir un sistema que combine AND, OR y NOT.

```javascript
// ============================================================
// SISTEMA DE AUTENTICACIÓN
// Mini proyecto del Capítulo 13
// ============================================================

console.log("=== SISTEMA DE AUTENTICACIÓN ===");

// Datos simulados
let usuarioCorrecto = "admin";
let passwordCorrecto = "1234";
let codigo2FA = "7890";

let intentos = 0;
let maxIntentos = 3;
let accesoConcedido = false;

while (intentos < maxIntentos && !accesoConcedido) {
    console.log("Intento " + (intentos + 1) + " de " + maxIntentos);

    let usuario = prompt("Usuario:");
    let password = prompt("Contraseña:");

    if (usuario === null || password === null) {
        console.log("⚠️ Autenticación cancelada.");
        break;
    }

    // Primera validación: usuario Y contraseña
    if (usuario === usuarioCorrecto && password === passwordCorrecto) {
        console.log("✅ Usuario y contraseña correctos");

        // Segundo factor: código 2FA
        let codigo = prompt("Ingresa el código de verificación:");

        if (codigo === codigo2FA) {
            console.log("✅ Código correcto");
            accesoConcedido = true;
            console.log("🎉 Acceso concedido. Bienvenido " + usuario);
            alert("🎉 ¡Bienvenido " + usuario + "!");
        } else {
            console.log("❌ Código incorrecto");
            alert("❌ Código de verificación incorrecto");
            intentos++;
        }
    } else {
        console.log("❌ Usuario o contraseña incorrectos");

        // NOT: mostrar mensaje si el usuario NO está bloqueado
        if (intentos < maxIntentos - 1) {
            alert("❌ Credenciales incorrectas. Te quedan " +
                  (maxIntentos - intentos - 1) + " intentos.");
        }
        intentos++;
    }
}

if (!accesoConcedido && intentos >= maxIntentos) {
    console.log("🔒 Cuenta bloqueada por múltiples intentos fallidos.");
    alert("🔒 Cuenta bloqueada. Demasiados intentos fallidos.");
}

console.log("=== FIN DEL PROGRAMA ===");

Escenario 2: admin / 1234 / 0000

Escenario 3: admin / incorrecto
- Resultado: Credenciales incorrectas ❌

Escenario 4: 3 intentos fallidos
- Resultado: Cuenta bloqueada 🔒

GLOSARIO DEL CAPÍTULO
===============================================================================

Operador lógico: Operador que combina condiciones booleanas.

AND (&&): Devuelve true solo si todas las condiciones son true.

OR (||): Devuelve true si al menos una condición es true.

NOT (!): Invierte el valor de una condición.

Cortocircuito: Evaluación que se detiene al conocer el resultado.

Precedencia: Orden en que se evalúan los operadores.

De Morgan: Leyes que relacionan AND, OR y NOT.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre & y &&?**

& es operador de bits (trabaja con binarios).
&& es operador lógico (trabaja con booleanos).
Usa siempre && para condiciones.

❓ **¿Qué significa || en asignaciones?**

```javascript
let nombre = usuario || "Invitado";
// Si usuario es truthy, nombre = usuario.
// Si usuario es falsy, nombre = "Invitado".
```

Es un patrón profesional para valores por defecto.

❓ **¿Puedo usar más de dos condiciones?**

Sí. Puedes encadenar tantas como necesites:
```javascript
if (a && b && c && d) { ... }
```

❓ **¿Qué es la evaluación de cortocircuito?**

JavaScript evalúa de izquierda a derecha y se detiene
en cuanto sabe el resultado final.

❓ **¿Cómo funcionan las leyes de De Morgan?**

- !(A && B) es igual a !A || !B
- !(A || B) es igual a !A && !B

Útil para simplificar negaciones complejas.
