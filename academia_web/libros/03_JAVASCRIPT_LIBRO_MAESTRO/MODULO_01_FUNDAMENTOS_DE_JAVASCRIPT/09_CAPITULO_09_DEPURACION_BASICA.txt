===============================================================================
1. TIPOS DE ERRORES EN JAVASCRIPT
===============================================================================

Hay varios tipos de errores en JavaScript.

---

📌 **Error de sintaxis (SyntaxError)**

Ocurre cuando el código no sigue las reglas del lenguaje.

```javascript
// ❌ Error de sintaxis
console.log("Hola"   // Falta cerrar paréntesis
Mensaje: Uncaught SyntaxError: missing ) after argument list

📌 Error de referencia (ReferenceError)

Ocurre cuando usas una variable que no existe.

javascript
// ❌ Error de referencia
console.log(edad);   // edad no está definida
Mensaje: Uncaught ReferenceError: edad is not defined

📌 Error de tipo (TypeError)

Ocurre cuando usas un valor de tipo incorrecto.

javascript
// ❌ Error de tipo
let x = null;
x.toString();   // null no tiene método toString
Mensaje: Uncaught TypeError: Cannot read property 'toString' of null

📌 Error de rango (RangeError)

Ocurre cuando usas un valor fuera de rango.

javascript
// ❌ Error de rango
let array = new Array(-1);   // Tamaño negativo no permitido
📌 Error de lógica (no da error, pero no funciona)

El código se ejecuta, pero no hace lo que esperabas.

javascript
// ❌ Error de lógica
let total = 10 + "5";   // total = "105" (concatenación, no suma)
===============================================================================

LEER MENSAJES DE ERROR
===============================================================================

Los mensajes de error te dicen:

Qué pasó (tipo de error).

Dónde pasó (archivo y línea).

Por qué pasó (descripción).

🔵 Ejemplo de mensaje de error

text
Uncaught TypeError: Cannot read property 'length' of undefined
    at miFuncion (script.js:15:23)
    at script.js:20:5
Tipo: TypeError

Descripción: Cannot read property 'length' of undefined

Archivo: script.js

Línea: 15

Columna: 23

Pila de llamadas: dónde se llamó la función.

📌 Cómo leer la pila de llamadas (call stack)

La pila de llamadas muestra el camino que tomó el código hasta el error.
El error está en la parte superior de la pila.

CONSOLE.LOG() PARA DEPURAR
===============================================================================

console.log() es tu herramienta principal para depurar.

🔵 Ejemplo: depurar con console.log()

javascript
function calcularPromedio(numeros) {
    console.log("Numeros recibidos:", numeros);
    let suma = 0;
    for (let i = 0; i < numeros.length; i++) {
        suma += numeros[i];
        console.log(`Paso ${i}: suma = ${suma}`);
    }
    let promedio = suma / numeros.length;
    console.log("Promedio calculado:", promedio);
    return promedio;
}
📌 Técnicas con console.log()

Mostrar valores de variables en diferentes puntos.

Mostrar el tipo de una variable con typeof.

Mostrar mensajes para saber si se ejecutó una parte del código.

javascript
console.log("Entrando a la función");
console.log("Valor de x:", x, "tipo:", typeof x);
console.log("Saliendo de la función");
📌 Otros métodos de consola

Método	Descripción
console.log()	Mensaje normal
console.warn()	Mensaje de advertencia (amarillo)
console.error()	Mensaje de error (rojo)
console.table()	Muestra datos en tabla
console.time() / console.timeEnd()	Medir tiempo de ejecución
javascript
console.warn("Esto es una advertencia");
console.error("Esto es un error");
console.table([{nombre: "Ana", edad: 25}, {nombre: "Luis", edad: 30}]);

EL INSPECTOR DEL NAVEGADOR (HERRAMIENTAS DE DESARROLLO)
===============================================================================

Las herramientas de desarrollo son tu mejor amigo para depurar.

📌 Abrir herramientas de desarrollo

Chrome: F12 o Ctrl + Shift + I (Win/Linux) / Cmd + Option + I (Mac)

Firefox: F12 o Ctrl + Shift + I (Win/Linux) / Cmd + Option + I (Mac)

📌 Pestaña "Sources" (Fuentes)

Aquí puedes ver y depurar tu código JavaScript.

Abrir tu archivo .js en la pestaña Sources.

Hacer clic en el número de línea para poner un breakpoint.

Recargar la página para que el código se detenga en el breakpoint.

📌 Pestaña "Console" (Consola)

Aquí puedes:

Ver mensajes de console.log().

Ver errores.

Ejecutar código al instante.

📌 Breakpoints (Puntos de interrupción)

Un breakpoint detiene la ejecución del código para que puedas inspeccionarlo.

Abre Sources.

Haz clic en el número de línea donde quieres detenerte (aparecerá un marcador azul).

Recarga la página.

El código se detendrá en ese punto.

Inspecciona variables en el panel lateral.

Presiona "Resume" (▶️) para continuar.

📌 Botones de depuración

Botón	Función
Resume (▶️)	Continúa la ejecución
Step Over (⏭️)	Ejecuta la siguiente línea (sin entrar en funciones)
Step Into (⏬)	Entra en la función que se está ejecutando
Step Out (⏫)	Sale de la función actual
Deactivate breakpoints	Desactiva todos los breakpoints

TÉCNICAS DE DEPURACIÓN
===============================================================================

📌 Técnica 1: Divide y vencerás

Si un programa no funciona, divide el problema en partes más pequeñas.

javascript
// En lugar de depurar todo el programa...
function programaCompleto() { /* ... */ }

// Prueba cada parte por separado
function parte1() { /* ... */ }
function parte2() { /* ... */ }
function parte3() { /* ... */ }
📌 Técnica 2: Imprime todo

Usa console.log() para mostrar el valor de las variables en cada paso.

javascript
let resultado = 10 + 5 * 2;
console.log("Resultado:", resultado);   // 20
📌 Técnica 3: Aísla el error

Comenta partes del código hasta que encuentres la que causa el error.

javascript
// función1();
// función2();
funcion3();   // ← El error está aquí
📌 Técnica 4: Verifica los casos límite

Prueba con valores extremos (0, negativo, muy grande).

javascript
function dividir(a, b) {
    if (b === 0) {
        console.warn("División por cero");
        return Infinity;
    }
    return a / b;
}

ERRORES COMUNES Y CÓMO SOLUCIONARLOS
===============================================================================

❌ "Uncaught TypeError: Cannot read property 'X' of undefined"

Causa: estás intentando acceder a una propiedad de undefined.

Solución: verifica que la variable tenga un valor antes de usarla.

javascript
if (usuario && usuario.nombre) {
    console.log(usuario.nombre);
}
❌ "Uncaught ReferenceError: X is not defined"

Causa: usas una variable que no existe.

Solución: declara la variable antes de usarla.

❌ "Uncaught SyntaxError: Unexpected token"

Causa: hay un error de sintaxis (falta un paréntesis, una llave, etc.).

Solución: revisa la línea que indica el error.

❌ El código no hace nada (sin errores)

Causa: error de lógica.

Solución: usa console.log() para ver si el código se está ejecutando
y qué valores tienen las variables.

ERRORES COMUNES CON DEPURACIÓN
===============================================================================

❌ Error 1: Ignorar los mensajes de error

Los mensajes de error te dicen exactamente qué pasa.
Solución: léelos y entiéndelos.

❌ Error 2: No usar console.log() suficiente

Sin información, es difícil encontrar el error.
Solución: imprime variables y mensajes.

❌ Error 3: Cambiar el código sin entender el error

Puedes introducir nuevos errores.
Solución: entiende el error antes de cambiarlo.

❌ Error 4: No usar breakpoints

console.log() es útil, pero los breakpoints son más poderosos.
Solución: aprende a usar breakpoints.

GLOSARIO DEL CAPÍTULO
===============================================================================

Depuración (debugging): Proceso de encontrar y corregir errores.

Breakpoint: Punto donde el código se detiene para ser inspeccionado.

Pila de llamadas (call stack): Lista de funciones que se llamaron para llegar al error.

Inspeccionar: Ver el estado de las variables en un punto del código.
