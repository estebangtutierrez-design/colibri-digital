===============================================================================
1. SALIDA: CONSOLE.LOG() (PARA DESARROLLADORES)
===============================================================================

`console.log()` muestra mensajes en la consola del navegador.

---

🔵 **Ejemplo**

```javascript
console.log("Hola desde la consola");
console.log("El resultado es:", 42);
console.log("Variable:", nombre);
📌 Ventajas de console.log()

No interrumpe al usuario (solo visible para el desarrollador).

Puede mostrar cualquier tipo de dato (objetos, arrays, números, strings).

Es la herramienta principal de depuración.

📌 Cuándo usar console.log()

Para depurar código.

Para verificar el valor de variables.

Para mostrar mensajes de desarrollo.

===============================================================================

SALIDA: ALERT() (VENTANA EMERGENTE)
===============================================================================

alert() muestra un mensaje en una ventana emergente que el usuario debe cerrar.

🔵 Ejemplo

javascript
alert("¡Bienvenido a mi programa!");
alert("El resultado es: " + 42);
📌 Características de alert()

Bloquea la ejecución: el código se detiene hasta que el usuario hace clic en "Aceptar".

Simple y directa: no requiere interacción compleja.

Mensajes informativos: para avisar al usuario.

📌 Cuándo usar alert()

Para mostrar mensajes importantes.

Para avisar al usuario de algo.

Para depuración simple (no recomendado).

⚠️ Advertencia

alert() es molesto si se usa en exceso. Úsala con moderación.

===============================================================================

SALIDA: DOCUMENT.WRITE() (ESCRIBIR EN LA PÁGINA)
===============================================================================

document.write() escribe contenido directamente en la página HTML.

🔵 Ejemplo

javascript
document.write("<h1>Bienvenido</h1>");
document.write("<p>Este texto fue escrito con JavaScript</p>");
📌 Características de document.write()

Escribe en la página (visible para el usuario).

Acepta HTML (puedes escribir etiquetas).

Sobrescribe la página si se usa después de cargada.

📌 Cuándo usar document.write()

Para escribir contenido dinámico en la página.

Para pruebas rápidas.

⚠️ Advertencia

document.write() no se recomienda para proyectos grandes.
Si se usa después de que la página esté cargada, sobrescribe todo.

===============================================================================

ENTRADA: PROMPT() (PEDIR INFORMACIÓN)
===============================================================================

prompt() muestra un cuadro de diálogo que pide al usuario que ingrese texto.

📌 Sintaxis

javascript
let nombre = prompt("¿Cuál es tu nombre?");
console.log("Hola " + nombre);
🔵 Ejemplo completo

javascript
let nombre = prompt("¿Cómo te llamas?");
alert("¡Hola " + nombre + "!");
console.log("El usuario se llama:", nombre);
📌 Características de prompt()

Recibe entrada del usuario (texto).

Devuelve un string (siempre, incluso si el usuario ingresa un número).

Permite un valor predeterminado (opcional).

Bloquea la ejecución hasta que el usuario responde.

📌 Sintaxis con valor predeterminado

javascript
let edad = prompt("¿Cuántos años tienes?", "18");
🔵 Ejemplo: prompt con conversión a número

javascript
let edadTexto = prompt("¿Cuántos años tienes?");
let edad = Number(edadTexto);   // Convertir a número
console.log(edad + 5);          // Edad + 5
⚠️ Importante

prompt() siempre devuelve un string.
Si necesitas un número, debes convertirlo con Number().

===============================================================================

ENTRADA/SALIDA: CONFIRM() (PREGUNTAS SÍ/NO)
===============================================================================

confirm() muestra un cuadro de diálogo con "Aceptar" y "Cancelar".
Devuelve true si el usuario hace clic en "Aceptar" y false si hace clic en "Cancelar".

📌 Sintaxis

javascript
let respuesta = confirm("¿Estás seguro?");
console.log(respuesta);   // true o false
🔵 Ejemplo

javascript
let confirmacion = confirm("¿Deseas continuar?");
if (confirmacion) {
    console.log("El usuario continuó");
} else {
    console.log("El usuario canceló");
}
📌 Características de confirm()

Devuelve un booleano (true o false).

Bloquea la ejecución hasta que el usuario responde.

Útil para confirmar acciones (eliminar, salir, etc.).

🔵 Ejemplo: confirmación antes de eliminar

javascript
let eliminar = confirm("¿Estás seguro de que quieres eliminar este archivo?");
if (eliminar) {
    console.log("Archivo eliminado");
} else {
    console.log("Operación cancelada");
}
===============================================================================

EJEMPLOS COMPLETOS DE ENTRADA Y SALIDA
===============================================================================

🔵 Ejemplo 1: Saludo personalizado

javascript
// Pedir el nombre
let nombre = prompt("¿Cómo te llamas?");

// Mostrar saludo
alert("¡Hola " + nombre + "!");
console.log("Usuario:", nombre);

// Escribir en la página
document.write("<h1>Bienvenido, " + nombre + "</h1>");
document.write("<p>Este es tu primer programa interactivo.</p>");
🔵 Ejemplo 2: Calculadora simple con entrada

javascript
// Pedir dos números
let num1 = Number(prompt("Ingresa el primer número:"));
let num2 = Number(prompt("Ingresa el segundo número:"));

// Calcular
let suma = num1 + num2;

// Mostrar resultado
alert("La suma es: " + suma);
console.log("Suma:", suma);
document.write("<p>La suma de " + num1 + " y " + num2 + " es " + suma + "</p>");
🔵 Ejemplo 3: Confirmación de acción

javascript
let continuar = confirm("¿Quieres continuar con el programa?");

if (continuar) {
    alert("Continuando...");
    document.write("<p>El programa continúa.</p>");
} else {
    alert("Programa cancelado.");
    document.write("<p>El programa ha sido cancelado.</p>");
}
===============================================================================

CONVERSIÓN DE ENTRADA
===============================================================================

prompt() siempre devuelve un string. A veces necesitas convertirlo.

📌 Conversión a número

javascript
let edad = Number(prompt("¿Cuántos años tienes?"));
console.log(edad + 5);   // Edad + 5 (suma matemática)
📌 Conversión a booleano

javascript
let respuesta = prompt("¿Eres estudiante? (si/no)");
let esEstudiante = respuesta === "si";
console.log(esEstudiante);   // true o false
📌 Validación básica

javascript
let edad = prompt("¿Cuántos años tienes?");
if (edad === "" || edad === null) {
    alert("No ingresaste una edad.");
} else {
    alert("Tu edad es: " + edad);
}
📌 Conversión con parseInt y parseFloat

javascript
let entero = parseInt(prompt("Ingresa un número entero:"));
let decimal = parseFloat(prompt("Ingresa un número decimal:"));
===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: ENTRADA Y SALIDA COMO UNA CONVERSACIÓN
===============================================================================

Imagina que tu programa es una persona.

Salida es cuando la persona habla (alert(), console.log(), document.write()).

Entrada es cuando la persona escucha (prompt(), confirm()).

console.log() → habla en voz baja (solo para ti).

alert() → habla en voz alta a todos.

document.write() → escribe en la pizarra.

prompt() → te pregunta algo.

confirm() → te pide que decidas entre sí o no.

Un buen programa sabe hablar y escuchar.

===============================================================================
ERRORES COMUNES CON ENTRADA Y SALIDA
===============================================================================

❌ Error 1: No convertir prompt a número

javascript
let edad = prompt("¿Edad?");
console.log(edad + 5);   // "255" (concatenación, no suma)
Solución: convierte con Number().

❌ Error 2: Usar document.write después de cargar la página

javascript
document.write("Hola");   // Si la página ya está cargada, la sobrescribe
Solución: usa console.log() o manipula el DOM.

❌ Error 3: No manejar el valor null de prompt

Si el usuario cancela, prompt() devuelve null.

javascript
let nombre = prompt("Nombre:");
if (nombre === null) {
    console.log("El usuario canceló");
}
❌ Error 4: Usar alert en exceso

Muchas alertas seguidas son molestas.
Solución: usa console.log() para depuración y alert() solo para mensajes importantes.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Cuál es la diferencia entre alert y console.log?

alert() muestra un mensaje al usuario (visible para todos).
console.log() muestra un mensaje en la consola (solo visible para el desarrollador).

❓ ¿Por qué prompt devuelve un string?

Porque siempre devuelve texto, incluso si el usuario ingresa números.
Debes convertirlo con Number() si necesitas un número.

❓ ¿Qué hace confirm?

Muestra un cuadro de diálogo con "Aceptar" y "Cancelar".
Devuelve true si el usuario hace clic en "Aceptar".

❓ ¿Cuándo usar document.write?

Para escribir contenido directamente en la página HTML.
No se recomienda para proyectos grandes.

❓ ¿Qué pasa si el usuario cancela un prompt?

prompt() devuelve null. Debes manejar este caso.
