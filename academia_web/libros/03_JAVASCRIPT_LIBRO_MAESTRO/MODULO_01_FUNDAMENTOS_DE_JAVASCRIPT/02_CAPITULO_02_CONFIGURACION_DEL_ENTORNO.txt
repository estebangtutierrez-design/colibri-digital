1. ELIGE TU EDITOR DE CÓDIGO
===============================================================================

Un editor de código es donde escribirás todo tu JavaScript.
Existen muchos, pero hay uno que destaca sobre los demás.

---

📌 **Visual Studio Code (VS Code)**

| Editor | Características |

| **Notepad++** | Ligero, ideal para Windows. |

📌 **Recomendación del Profesor Búho**

Si no tienes un editor favorito, **elige Visual Studio Code**.

CREA TU PRIMER ARCHIVO JAVASCRIPT
===============================================================================

Ahora vas a crear tu primer archivo .js.

📌 Paso 1: Crear una carpeta para tu proyecto

Crea una carpeta en tu computadora, por ejemplo:

text
mi-primer-proyecto-js/
📌 Paso 2: Abrir la carpeta en VS Code

Abre VS Code.

Ve a Archivo → Abrir carpeta.

Selecciona la carpeta que creaste.

📌 Paso 3: Crear un archivo HTML

Dentro de la carpeta, crea un archivo llamado index.html:

index.html

<!DOCTYPE html><html lang="es"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Mi primer JavaScript</title> </head> <body> <h1>¡Hola desde HTML!</h1> <p>Abre la consola para ver el mensaje de JavaScript.</p> <!-- Vinculamos nuestro archivo JavaScript --> <script src="script.js"></script></body> </html> ------------------------------------------------------------
📌 Paso 4: Crear un archivo JavaScript

En la misma carpeta, crea un archivo llamado script.js:

script.js

// Mi primer archivo JavaScript
console.log("¡Hola desde JavaScript!");

📌 Paso 5: Abrir el HTML en el navegador

Haz clic derecho en index.html en VS Code.

Selecciona "Abrir con Live Server" (si tienes la extensión instalada).

O: abre el archivo directamente en el navegador.

📌 Paso 6: Ver el resultado

Abre la consola del navegador (Ctrl + Shift + J en Chrome).
Verás el mensaje: ¡Hola desde JavaScript!

¡Acabas de crear tu primer archivo JavaScript!

VINCULAR JAVASCRIPT CON HTML
===============================================================================

Hay tres formas de vincular JavaScript con HTML.

📌 1. JavaScript externo (recomendado)

Se escribe en un archivo .js y se vincula con la etiqueta <script>.

HTML

<script src="script.js"></script>
📌 2. JavaScript interno (dentro del HTML)

Se escribe dentro de la etiqueta <script> en el HTML.

HTML

<script> console.log("JavaScript interno"); </script>
📌 3. JavaScript en línea (directamente en un elemento)

Se escribe dentro del atributo onclick, onload, etc.
(NO RECOMENDADO para código complejo).

HTML

<button onclick="alert('Hola')">Clic</button>

📌 Buenas prácticas

Usa JavaScript externo (archivos .js) para mantener el código limpio.

Coloca la etiqueta <script> al final del <body> para que el HTML se cargue primero.

Usa defer o async para controlar la carga de scripts.

🔵 Ejemplo con defer

HTML

<script src="script.js" defer></script>
defer asegura que el script se ejecute después de que el HTML se haya cargado por completo.

===============================================================================

LIVE SERVER: RECARGA AUTOMÁTICA
===============================================================================

Live Server es una extensión de VS Code que recarga automáticamente
tu página cada vez que guardas un cambio.

📌 Cómo instalar Live Server

Abre VS Code.

Ve a la sección de extensiones (cuadrados en la barra lateral).

Busca "Live Server".

Haz clic en "Instalar".

📌 Cómo usar Live Server

Abre tu archivo index.html en VS Code.

Haz clic derecho → "Open with Live Server".

O: haz clic en "Go Live" en la barra inferior de VS Code.

Se abrirá tu navegador en http://127.0.0.1:5500/.
Cada vez que guardes cambios, la página se recargará automáticamente.

===============================================================================
ERRORES COMUNES AL CONFIGURAR EL ENTORNO
===============================================================================

❌ Error 1: No instalar Live Server

Sin Live Server, tienes que recargar la página manualmente cada vez.
Solución: instala la extensión Live Server.

❌ Error 2: Vincular JavaScript en el <head> sin defer

Si el script se carga antes de que el HTML esté listo, puede dar errores.
Solución: coloca el script al final del <body> o usa defer.

❌ Error 3: No abrir la consola para ver errores

Los errores de JavaScript se muestran en la consola.
Solución: siempre ten la consola abierta mientras desarrollas.

❌ Error 4: Usar extensiones incorrectas

Los archivos JavaScript deben tener extensión .js.
No uses .txt o .html para código JavaScript.

❌ Error 5: No guardar los cambios antes de probar

Si no guardas, los cambios no se reflejarán.
Solución: usa Ctrl + S y confirma que Live Server recargue la página.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Qué editor de código es mejor para JavaScript?

Visual Studio Code es el más recomendado por su facilidad de uso,
extensiones y soporte de la comunidad.

❓ ¿Qué es Live Server?

Es una extensión de VS Code que recarga automáticamente la página
en el navegador cada vez que guardas cambios.

❓ ¿Cómo veo los errores de JavaScript?

Abre la consola del navegador (F12 → Consola).
Los errores aparecerán en rojo.

❓ ¿Debo usar defer en mis scripts?

Sí, es buena práctica para asegurar que el HTML se cargue antes de ejecutar JavaScript.

❓ ¿Puedo usar otro navegador además de Chrome?

Sí, Firefox y Edge también tienen excelentes herramientas de desarrollo.
