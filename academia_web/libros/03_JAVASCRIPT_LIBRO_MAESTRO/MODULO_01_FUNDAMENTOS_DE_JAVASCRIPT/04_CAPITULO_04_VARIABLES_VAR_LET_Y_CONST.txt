INTRODUCCIÓN: ¿QUÉ ES UNA VARIABLE?
===============================================================================

Una **variable** es un espacio en la memoria del ordenador donde almacenamos un valor.

Imagina que las variables son cajas con etiquetas.

- La **etiqueta** es el nombre de la variable.
- El **contenido** de la caja es el valor que guarda.

---

📌 **Ejemplo en la vida real**
Caja "nombre" → contiene "Ana"
Caja "edad" → contiene 25
Caja "esEstudiante" → contiene true

text

---

📌 **Sintaxis básica**

```javascript
let nombre = "Ana";
let edad = 25;
let esEstudiante = true;
let → palabra clave para declarar la variable.

nombre → el nombre de la variable.

= → operador de asignación (guarda el valor).

"Ana" → el valor que se guarda.

===============================================================================

LET: LA FORMA MODERNA (VALOR CAMBIANTE)
===============================================================================

let es la forma moderna de declarar variables que pueden cambiar de valor.

📌 Sintaxis

javascript
let nombre = "valor";
🔵 Ejemplo

javascript
let edad = 25;
console.log(edad);   // Muestra: 25

edad = 26;           // Cambiamos el valor
console.log(edad);   // Muestra: 26
📌 Características de let

Puede reasignarse: puedes cambiar el valor cuantas veces quieras.

Ámbito de bloque: la variable solo existe dentro del bloque donde se declara.

No se puede redeclarar: no puedes declarar la misma variable dos veces con let.

Hoisting con TDZ: la variable existe pero no se puede usar antes de declararla.

🔵 Ejemplo: reasignar con let

javascript
let mensaje = "Hola";
console.log(mensaje);   // Hola

mensaje = "Hola Mundo";
console.log(mensaje);   // Hola Mundo

mensaje = "¡Adiós!";
console.log(mensaje);   // ¡Adiós!
🔵 Ejemplo: ámbito de bloque con let

javascript
if (true) {
    let x = 10;          // x solo existe dentro de este bloque
    console.log(x);      // 10
}
console.log(x);          // Error: x no está definida
===============================================================================

CONST: VALOR QUE NO CAMBIA
===============================================================================

const es la forma moderna de declarar variables cuyo valor no cambia.

📌 Sintaxis

javascript
const nombre = "valor";
🔵 Ejemplo

javascript
const PI = 3.1416;
console.log(PI);   // 3.1416

PI = 3.15;         // ❌ Error: no se puede reasignar una constante
📌 Características de const

No se puede reasignar: una vez asignado, el valor no puede cambiar.

Debe inicializarse al declararse: no puedes declarar const sin valor.

Ámbito de bloque: igual que let.

No se puede redeclarar: igual que let.

🔵 Ejemplo: constante correcta

javascript
const NOMBRE_SITIO = "Lucas EGA Academy";
console.log(NOMBRE_SITIO);   // Lucas EGA Academy
🔵 Ejemplo: error con const

javascript
const fecha;         // ❌ Error: falta asignar un valor
fecha = "2026-06-30";
📌 ¿Cuándo usar const?

Cuando el valor no va a cambiar nunca.

Para valores fijos como PI, URL de API, nombres de sitios.

Siempre que sea posible, usa const. Es más seguro y claro.

⚠️ Excepción importante

Si const guarda un objeto o un array, el contenido del objeto puede cambiar, pero la variable no puede apuntar a otro objeto.

javascript
const persona = { nombre: "Ana" };
persona.nombre = "Carlos";   // ✅ Permitido (cambia el contenido)
console.log(persona.nombre); // Carlos

persona = { nombre: "Luis" }; // ❌ Error (reasignar la variable)
===============================================================================

VAR: LA FORMA ANTIGUA (YA NO SE RECOMIENDA)
===============================================================================

var era la forma original de declarar variables en JavaScript.
Ya no se recomienda porque tiene comportamientos confusos.

📌 Problemas de var

Ámbito de función: no respeta los bloques (if, for, etc.).

Se puede redeclarar: puedes declarar la misma variable varias veces.

Hoisting: la variable se mueve al principio, pero el valor no.

🔵 Ejemplo: var en un bloque

javascript
if (true) {
    var x = 10;
}
console.log(x);   // 10 (var no respeta el bloque, esto es confuso)
🔵 Ejemplo: redeclarar var

javascript
var nombre = "Ana";
var nombre = "Carlos";   // ✅ Permitido (pero confuso)
console.log(nombre);     // Carlos
📌 Regla de oro

No uses var en proyectos nuevos.
Usa let para valores que cambian y const para valores fijos.

===============================================================================

NOMBRES DE VARIABLES (REGLAS Y BUENAS PRÁCTICAS)
===============================================================================

📌 Reglas obligatorias

Pueden contener letras, números, _ (guión bajo) y $ (dólar).

No pueden empezar con un número.

No pueden ser palabras reservadas (como let, const, if, for).

Son sensibles a mayúsculas (nombre es diferente de Nombre).

🔵 Ejemplos correctos

javascript
let nombre;
let edad2;
let _precio;
let $id;
let usuario_nombre;
let miVariable;
🔵 Ejemplos incorrectos

javascript
let 2edad;       // ❌ No puede empezar con número
let let;         // ❌ Palabra reservada
let mi-variable; // ❌ No puede contener guiones (-)
let nombre!;     // ❌ No puede contener caracteres especiales
📌 Buenas prácticas (convenciones)

Usa camelCase → empieza en minúscula, las siguientes palabras en mayúscula.

javascript
let nombreUsuario;
let edadPersona;
let esMayorDeEdad;
Usa nombres descriptivos.

javascript
let x;           // ❌ No dice nada
let contador;    // ✅ Mejor
let numeroDeVisitas; // ✅ Excelente
Usa mayúsculas para constantes (opcional, pero común).

javascript
const PI = 3.1416;
const URL_API = "https://api.ejemplo.com";
No uses nombres muy largos.

javascript
let elNumeroDeVecesQueElUsuarioHaClickeado; // ❌ Demasiado largo
let contadorClics; // ✅
===============================================================================

ÁMBITO (SCOPE): DÓNDE VIVEN LAS VARIABLES
===============================================================================

El ámbito es la región del código donde una variable existe y es accesible.

📌 Tipos de ámbito

Tipo	Descripción
Global	La variable existe en todo el programa.
Local (de función)	La variable solo existe dentro de la función.
De bloque	La variable solo existe dentro de un bloque {} (con let y const).
🔵 Ejemplo: ámbito global

javascript
let global = "Soy global";

function mostrar() {
    console.log(global);   // ✅ Accede a la variable global
}
mostrar();                 // "Soy global"
🔵 Ejemplo: ámbito local

javascript
function miFuncion() {
    let local = "Soy local";
    console.log(local);     // ✅ "Soy local"
}
miFuncion();
console.log(local);         // ❌ Error: local no está definida
🔵 Ejemplo: ámbito de bloque (let vs var)

javascript
if (true) {
    let conLet = "Solo en el bloque";
    var conVar = "Soy de función";
}
console.log(conLet);   // ❌ Error
console.log(conVar);   // ✅ "Soy de función"
===============================================================================

HOISTING: EL ELEVADOR DE VARIABLES
===============================================================================

El hoisting es un comportamiento de JavaScript donde las declaraciones de variables
se mueven al principio de su ámbito.

📌 Hoisting con var

javascript
console.log(x);   // undefined (no error)
var x = 5;
Explicación: JavaScript interpreta esto como:

javascript
var x;
console.log(x);   // undefined
x = 5;
📌 Hoisting con let y const

javascript
console.log(y);   // ❌ Error: no se puede acceder antes de declarar
let y = 5;
let y const sí tienen hoisting, pero están en una "zona muerta temporal"
(TDZ - Temporal Dead Zone) hasta que se declaran.

Regla: siempre declara las variables al principio del bloque.

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: LAS VARIABLES COMO CAJAS
===============================================================================

Imagina que las variables son cajas en un almacén.

let → una caja que puedes abrir, cambiar el contenido y cerrar.

const → una caja sellada. Una vez que pones algo dentro, no puedes cambiarlo.

var → una caja antigua que puede estar en cualquier lugar del almacén
y puede ser reetiquetada varias veces. Es confusa.

El nombre de la variable es la etiqueta de la caja.

El valor es lo que hay dentro de la caja.

El ámbito es la parte del almacén donde está la caja.

===============================================================================
ERRORES COMUNES CON VARIABLES
===============================================================================

❌ Error 1: Usar var en lugar de let o const

var tiene comportamientos confusos que pueden causar errores.
Solución: usa let para valores que cambian y const para valores fijos.

❌ Error 2: Usar const para valores que cambian

Si intentas cambiar una constante, JavaScript dará un error.
Solución: usa let si el valor va a cambiar.

❌ Error 3: Declarar una variable sin inicializar y luego usarla

javascript
let nombre;
console.log(nombre);   // undefined (no error, pero no es un valor útil)
Solución: inicializa siempre las variables con un valor.

❌ Error 4: Usar nombres de variables no descriptivos

javascript
let a;   // ❌ No dice nada
let contador; // ✅
Solución: usa nombres que describan el propósito de la variable.

❌ Error 5: Intentar usar una variable antes de declararla

javascript
console.log(edad);   // ❌ Error con let/const
let edad = 25;
Solución: siempre declara antes de usar.

PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Cuál es la diferencia entre let y const?

let permite reasignar valores. const no permite reasignar.

❓ ¿Puedo cambiar el valor de una constante si es un objeto?

Sí, puedes cambiar las propiedades del objeto, pero no reasignar la variable.

❓ ¿Por qué ya no se usa var?

var tiene comportamientos confusos (ámbito de función, redeclaración, hoisting).
let y const son más predecibles.

❓ ¿Qué significa "undefined"?

undefined es un valor especial que significa que una variable ha sido declarada
pero no tiene valor asignado.

❓ ¿Qué es el ámbito de bloque?

Es la región delimitada por {} donde let y const solo existen dentro de ese bloque.

===============================================================================
INTRODUCCIÓN: ¿QUÉ SON LOS TIPOS DE DATOS?
===============================================================================

En JavaScript, cada valor tiene un **tipo de dato** que define qué tipo de información es.

---

📌 **Analogía: tipos de datos como etiquetas de frascos**

Imagina que tienes varios frascos en una despensa.

- Un frasco con **sal** (texto).
- Un frasco con **azúcar** (número).
- Un frasco con **café** (booleano).
- Un frasco **vacío** (undefined).
- Un frasco que has dejado vacío a propósito (null).

Cada frasco tiene una etiqueta que dice qué contiene.
Esa etiqueta es el **tipo de dato**.

---

📌 **Tipos primitivos en JavaScript**

| Tipo | Descripción | Ejemplo |
|------|-------------|---------|
| **string** | Texto | `"Hola"`, `'Ana'`, `"123"` |
| **number** | Números | `10`, `3.14`, `-5` |
| **boolean** | Verdadero/Falso | `true`, `false` |
| **undefined** | Sin valor | `undefined` |
| **null** | Vacío intencional | `null` |
| **symbol** | Valor único | `Symbol("id")` |
| **bigint** | Números muy grandes | `9007199254740991n` |

===============================================================================
1. STRING (TEXTO)
===============================================================================

El tipo **string** representa texto.

---

📌 **Sintaxis**

```javascript
let nombre = "Ana";        // Comillas dobles
let apellido = 'García';   // Comillas simples
let mensaje = "Hola \"mundo\""; // Comillas escapadas
🔵 Ejemplos

javascript
let nombre = "Carlos";
let saludo = "Hola " + nombre;
console.log(saludo);   // "Hola Carlos"
📌 Operaciones con strings

Operación	Ejemplo	Resultado
Concatenación	"Hola " + "Mundo"	"Hola Mundo"
Longitud	"Hola".length	4
Acceso	"Hola"[0]	"H"
Mayúsculas	"hola".toUpperCase()	"HOLA"
Minúsculas	"HOLA".toLowerCase()	"hola"
📌 Template strings (plantillas de texto)

Son una forma moderna de crear strings con variables.

javascript
let nombre = "Ana";
let edad = 25;
let mensaje = `Hola ${nombre}, tienes ${edad} años.`;
console.log(mensaje);   // "Hola Ana, tienes 25 años."
Las template strings usan comillas invertidas (`) y ${} para insertar variables.

===============================================================================

NUMBER (NÚMEROS)
===============================================================================

El tipo number representa números enteros y decimales.

📌 Sintaxis

javascript
let edad = 25;          // Entero
let precio = 19.99;     // Decimal
let temperatura = -5;   // Negativo
🔵 Ejemplos

javascript
let suma = 10 + 5;
console.log(suma);   // 15

let resta = 20 - 8;
console.log(resta);  // 12

let multiplicacion = 4 * 3;
console.log(multiplicacion); // 12

let division = 15 / 3;
console.log(division); // 5
📌 Números especiales

Valor	Descripción
Infinity	Número infinito (resultado de dividir por 0)
-Infinity	Número negativo infinito
NaN	Not a Number (no es un número)
javascript
console.log(1 / 0);   // Infinity
console.log(0 / 0);   // NaN
console.log("hola" * 2); // NaN
📌 Operaciones matemáticas comunes

Operación	Ejemplo	Resultado
Suma	5 + 3	8
Resta	5 - 3	2
Multiplicación	5 * 3	15
División	15 / 3	5
Módulo (resto)	10 % 3	1
Potencia	2 ** 3	8
===============================================================================

BOOLEAN (VERDADERO/FALSO)
===============================================================================

El tipo boolean representa un valor lógico: verdadero o falso.

📌 Sintaxis

javascript
let esMayor = true;

🔵 Ejemplos

javascript
let edad = 25;
let esMayorDeEdad = edad >= 18;
console.log(esMayorDeEdad);   // true

let temperatura = 30;
let haceCalor = temperatura > 25;
console.log(haceCalor);   // true

let tienePermiso = false;
📌 Valores que son "falsy" (se comportan como false)

false

0

"" (string vacío)

null

undefined

NaN

Todo lo demás es truthy (se comporta como true).

===============================================================================

UNDEFINED (SIN VALOR)
===============================================================================

undefined significa que una variable ha sido declarada pero no tiene valor asignado.

🔵 Ejemplos

javascript
let nombre;
console.log(nombre);   // undefined

let edad = undefined;  // Asignar undefined explícitamente (no recomendado)
console.log(edad);     // undefined
📌 Cuándo aparece undefined

Variable declarada sin valor → let x;

Propiedad que no existe en un objeto → objeto.propiedadInexistente

Función que no retorna nada → function() {}

Elemento de array que no existe → array[10]

===============================================================================

NULL (VACÍO INTENCIONAL)
===============================================================================

null representa un valor vacío o nulo intencional.

🔵 Ejemplos

javascript
let usuario = null;   // El usuario no existe intencionalmente
console.log(usuario); // null
📌 Diferencia entre null y undefined

Característica	undefined	null
Significado	No tiene valor	Vacío intencional
Asignado por	JavaScript	El programador
Tipo (typeof)	"undefined"	"object" (error histórico)
javascript
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" (¡es un error de JavaScript!)
===============================================================================

TYPEOF: IDENTIFICAR EL TIPO DE DATO
===============================================================================

typeof es un operador que te dice qué tipo de dato tiene una variable.

🔵 Ejemplos

javascript
console.log(typeof "Hola");      // "string"
console.log(typeof 42);          // "number"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" (error histórico)
console.log(typeof Symbol());    // "symbol"

1. OPERADORES ARITMÉTICOS
===============================================================================

Los operadores aritméticos realizan operaciones matemáticas.

---

📌 **Tabla de operadores aritméticos**

| Operador | Descripción | Ejemplo | Resultado |
|----------|-------------|---------|-----------|
| `+` | Suma | `5 + 3` | `8` |
| `-` | Resta | `5 - 3` | `2` |
| `*` | Multiplicación | `5 * 3` | `15` |
| `/` | División | `15 / 3` | `5` |
| `%` | Módulo (resto) | `10 % 3` | `1` |
| `**` | Potencia | `2 ** 3` | `8` |

---

🔵 **Ejemplos**

```javascript
let a = 10;
let b = 3;

console.log(a + b);   // 13
console.log(a - b);   // 7
console.log(a * b);   // 30
console.log(a / b);   // 3.333...
console.log(a % b);   // 1
console.log(a ** b);  // 1000
📌 Módulo (%): El resto de una división.

javascript
console.log(10 % 3);   // 1 (porque 10 ÷ 3 = 3, sobra 1)
console.log(10 % 2);   // 0 (porque 10 ÷ 2 = 5, sobra 0)
console.log(7 % 3);    // 1
📌 Potencia ()**: Eleva un número a la potencia de otro.

javascript
console.log(2 ** 3);   // 8 (2 * 2 * 2)
console.log(3 ** 2);   // 9 (3 * 3)
console.log(10 ** 2);  // 100

OPERADORES DE ASIGNACIÓN
===============================================================================

Los operadores de asignación guardan valores en variables.

📌 Tabla de operadores de asignación

Operador	Descripción	Ejemplo	Equivalente
=	Asignación simple	x = 5	x = 5
+=	Suma y asigna	x += 3	x = x + 3
-=	Resta y asigna	x -= 3	x = x - 3
*=	Multiplica y asigna	x *= 3	x = x * 3
/=	Divide y asigna	x /= 3	x = x / 3
%=	Módulo y asigna	x %= 3	x = x % 3
**=	Potencia y asigna	x **= 3	x = x ** 3
🔵 Ejemplos

javascript
let x = 10;

x += 5;    // x = 15

x *= 2;    // x = 24

x %= 4;    // x = 2
x **= 3;   // x = 8
📌 Incremento y decremento

Operador	Descripción	Ejemplo
++	Incremento (suma 1)	x++ o ++x
--	Decremento (resta 1)	x-- o --x

let contador = 0;

OPERADORES DE COMPARACIÓN
===============================================================================

Los operadores de comparación comparan dos valores y devuelven true o false.

📌 Tabla de operadores de comparación

Operador	Descripción	Ejemplo	Resultado
===	Igualdad estricta	5 === 5	true
!==	Desigualdad estricta	5 !== 3	true
>	Mayor que	5 > 3	true
<	Menor que	5 < 3	false
>=	Mayor o igual que	5 >= 5	true
<=	Menor o igual que	5 <= 3	false
⚠️ Importante: == vs ===

== compara valor con conversión automática.

=== compara valor Y tipo (más seguro).

javascript
console.log(10 == "10");   // true (conversión automática)
console.log(10 === "10");  // false (tipo diferente)
Regla: SIEMPRE usa === y !==.

🔵 Ejemplos

javascript
let edad = 25;

console.log(edad === 25);   // true
console.log(edad !== 25);   // false
console.log(edad > 18);     // true
console.log(edad < 18);     // false
console.log(edad >= 25);    // true
console.log(edad <= 20);    // false

OPERADORES LÓGICOS
===============================================================================

Los operadores lógicos combinan valores booleanos.

📌 Tabla de operadores lógicos

Operador	Descripción	Ejemplo	Resultado
&&	Y (AND)	true && true	true
||	O (OR)	true && false	false
!	NO (NOT)	!true	false
🔵 Ejemplos

javascript
let esMayor = true;
let tienePermiso = false;

console.log(esMayor && tienePermiso);   // false
console.log(esMayor || tienePermiso);   // true
console.log(!esMayor);                   // false
📌 Tabla de verdad de AND (&&)

A	B	A && B
true	true	true
true	false	false
false	true	false
false	false	false
📌 Tabla de verdad de OR (||)

A	B	A || B
true	true	true
true	false	true
false	true	true
false	false	false
📌 Tabla de verdad de NOT (!)

A	!A
true	false
false	true
🔵 Ejemplos prácticos

javascript
let edad = 25;

// Si es mayor de 18 Y tiene licencia

    console.log("Puede conducir");

PRECEDENCIA DE OPERADORES
===============================================================================

La precedencia determina el orden en que se evalúan los operadores.

📌 Orden de precedencia (de mayor a menor)

() → paréntesis

** → potencia

*, /, % → multiplicación, división, módulo

+, - → suma, resta

===, !==, >, <, >=, <= → comparación

&& → AND lógico

|| → OR lógico

🔵 Ejemplo

javascript
let resultado = 10 + 5 * 2;
console.log(resultado);   // 20 (primero multiplica: 5*2=10, luego suma 10+10=20)

let resultado2 = (10 + 5) * 2;
console.log(resultado2);  // 30 (primero el paréntesis: 10+5=15, luego *2=30)

ERRORES COMUNES CON OPERADORES
===============================================================================

❌ Error 1: Usar = en lugar de ===

javascript
if (x = 5) {   // ❌ Asignación, no comparación
    // ...
}
Solución: usa === para comparar.

❌ Error 2: Confundir + (suma) con + (concatenación)

javascript
console.log("10" + 5);   // "105" (concatenación)
console.log("10" - 5);   // 5 (resta convierte a número)
❌ Error 3: No usar paréntesis cuando es necesario

javascript
let resultado = 10 + 5 * 2;   // 20 (multiplicación primero)
Solución: usa paréntesis para aclarar.

❌ Error 4: Confundir && con ||

javascript
if (edad >= 18 && tieneLicencia)   // Ambas condiciones deben ser true
if (edad >= 18 || tienePermiso)    // Una condición debe ser true

alert() es molesto si se usa en exceso. Úsala con moderación.

===============================================================================

SALIDA: DOCUMENT.WRITE() (ESCRIBIR EN LA PÁGINA)
===============================================================================

document.write() escribe contenido directamente en la página HTML.

🔵 Ejemplo

javascript
document.write("<h1>Bienvenido</h1>");
document.write("<p>Este texto fue escrito con JavaScript</p>");
📌 Características de document.write()

Escribe en la página (visible para el usuario).

Acepta HTML (puedes escribir etiquetas).

Sobrescribe la página si se usa después de cargada.

📌 Cuándo usar document.write()

Para escribir contenido dinámico en la página.

Para pruebas rápidas.

⚠️ Advertencia

document.write() no se recomienda para proyectos grandes.
Si se usa después de que la página esté cargada, sobrescribe todo.

===============================================================================

ENTRADA: PROMPT() (PEDIR INFORMACIÓN)
===============================================================================

prompt() muestra un cuadro de diálogo que pide al usuario que ingrese texto.

📌 Sintaxis

javascript
let nombre = prompt("¿Cuál es tu nombre?");
console.log("Hola " + nombre);
🔵 Ejemplo completo

javascript
let nombre = prompt("¿Cómo te llamas?");
alert("¡Hola " + nombre + "!");
console.log("El usuario se llama:", nombre);
📌 Características de prompt()

Recibe entrada del usuario (texto).

Devuelve un string (siempre, incluso si el usuario ingresa un número).

Permite un valor predeterminado (opcional).

Bloquea la ejecución hasta que el usuario responde.

📌 Sintaxis con valor predeterminado

javascript
let edad = prompt("¿Cuántos años tienes?", "18");
🔵 Ejemplo: prompt con conversión a número

javascript
let edadTexto = prompt("¿Cuántos años tienes?");
let edad = Number(edadTexto);   // Convertir a número
console.log(edad + 5);          // Edad + 5
⚠️ Importante

prompt() siempre devuelve un string.
Si necesitas un número, debes convertirlo con Number().

===============================================================================

ENTRADA/SALIDA: CONFIRM() (PREGUNTAS SÍ/NO)
===============================================================================

confirm() muestra un cuadro de diálogo con "Aceptar" y "Cancelar".
Devuelve true si el usuario hace clic en "Aceptar" y false si hace clic en "Cancelar".

📌 Sintaxis

javascript
let respuesta = confirm("¿Estás seguro?");
console.log(respuesta);   // true o false
🔵 Ejemplo

javascript
let confirmacion = confirm("¿Deseas continuar?");
if (confirmacion) {
    console.log("El usuario continuó");
} else {
    console.log("El usuario canceló");
}
📌 Características de confirm()

Devuelve un booleano (true o false).

Bloquea la ejecución hasta que el usuario responde.

Útil para confirmar acciones (eliminar, salir, etc.).

🔵 Ejemplo: confirmación antes de eliminar

javascript
let eliminar = confirm("¿Estás seguro de que quieres eliminar este archivo?");
if (eliminar) {
    console.log("Archivo eliminado");
} else {
    console.log("Operación cancelada");
}
===============================================================================

EJEMPLOS COMPLETOS DE ENTRADA Y SALIDA
===============================================================================

🔵 Ejemplo 1: Saludo personalizado

javascript
// Pedir el nombre
let nombre = prompt("¿Cómo te llamas?");

// Mostrar saludo
alert("¡Hola " + nombre + "!");
console.log("Usuario:", nombre);

// Escribir en la página
document.write("<h1>Bienvenido, " + nombre + "</h1>");
document.write("<p>Este es tu primer programa interactivo.</p>");
🔵 Ejemplo 2: Calculadora simple con entrada

javascript
// Pedir dos números
let num1 = Number(prompt("Ingresa el primer número:"));
let num2 = Number(prompt("Ingresa el segundo número:"));

// Calcular
let suma = num1 + num2;

// Mostrar resultado
alert("La suma es: " + suma);
console.log("Suma:", suma);
document.write("<p>La suma de " + num1 + " y " + num2 + " es " + suma + "</p>");
🔵 Ejemplo 3: Confirmación de acción

javascript
let continuar = confirm("¿Quieres continuar con el programa?");

if (continuar) {
    alert("Continuando...");
    document.write("<p>El programa continúa.</p>");
} else {
    alert("Programa cancelado.");
    document.write("<p>El programa ha sido cancelado.</p>");
}
===============================================================================

CONVERSIÓN DE ENTRADA
===============================================================================

prompt() siempre devuelve un string. A veces necesitas convertirlo.

📌 Conversión a número

javascript
let edad = Number(prompt("¿Cuántos años tienes?"));
console.log(edad + 5);   // Edad + 5 (suma matemática)
📌 Conversión a booleano

javascript
let respuesta = prompt("¿Eres estudiante? (si/no)");
let esEstudiante = respuesta === "si";
console.log(esEstudiante);   // true o false
📌 Validación básica

javascript
let edad = prompt("¿Cuántos años tienes?");
if (edad === "" || edad === null) {
    alert("No ingresaste una edad.");
} else {
    alert("Tu edad es: " + edad);
}
📌 Conversión con parseInt y parseFloat

javascript
let entero = parseInt(prompt("Ingresa un número entero:"));
let decimal = parseFloat(prompt("Ingresa un número decimal:"));
===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: ENTRADA Y SALIDA COMO UNA CONVERSACIÓN
===============================================================================

Imagina que tu programa es una persona.

Salida es cuando la persona habla (alert(), console.log(), document.write()).

Entrada es cuando la persona escucha (prompt(), confirm()).

console.log() → habla en voz baja (solo para ti).

alert() → habla en voz alta a todos.

document.write() → escribe en la pizarra.

prompt() → te pregunta algo.

confirm() → te pide que decidas entre sí o no.

Un buen programa sabe hablar y escuchar.

===============================================================================
ERRORES COMUNES CON ENTRADA Y SALIDA
===============================================================================

❌ Error 1: No convertir prompt a número

javascript
let edad = prompt("¿Edad?");
console.log(edad + 5);   // "255" (concatenación, no suma)
Solución: convierte con Number().

❌ Error 2: Usar document.write después de cargar la página

javascript
document.write("Hola");   // Si la página ya está cargada, la sobrescribe
Solución: usa console.log() o manipula el DOM.

❌ Error 3: No manejar el valor null de prompt

Si el usuario cancela, prompt() devuelve null.

javascript
let nombre = prompt("Nombre:");
if (nombre === null) {
    console.log("El usuario canceló");
}
❌ Error 4: Usar alert en exceso

Muchas alertas seguidas son molestas.
Solución: usa console.log() para depuración y alert() solo para mensajes importantes.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Cuál es la diferencia entre alert y console.log?

alert() muestra un mensaje al usuario (visible para todos).
console.log() muestra un mensaje en la consola (solo visible para el desarrollador).

❓ ¿Por qué prompt devuelve un string?

Porque siempre devuelve texto, incluso si el usuario ingresa números.
Debes convertirlo con Number() si necesitas un número.

❓ ¿Qué hace confirm?

Muestra un cuadro de diálogo con "Aceptar" y "Cancelar".
Devuelve true si el usuario hace clic en "Aceptar".

❓ ¿Cuándo usar document.write?

Para escribir contenido directamente en la página HTML.
No se recomienda para proyectos grandes.

❓ ¿Qué pasa si el usuario cancela un prompt?

prompt() devuelve null. Debes manejar este caso.
