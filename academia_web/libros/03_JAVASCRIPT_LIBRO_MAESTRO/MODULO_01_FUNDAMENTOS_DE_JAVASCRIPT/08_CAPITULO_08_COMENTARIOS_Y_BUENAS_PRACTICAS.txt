1. COMENTARIOS: LA DOCUMENTACIÓN DEL CÓDIGO
===============================================================================

Los comentarios son texto que el navegador ignora.
Sirven para explicar qué hace el código.

---

📌 **Comentarios de una línea (`//`)**

```javascript
// Esto es un comentario de una línea
let nombre = "Ana";  // Comentario al final de la línea
📌 Comentarios de bloque (/* */)

javascript
/*
   Esto es un comentario de bloque.
   Puede ocupar varias líneas.
   El navegador ignora todo lo que está dentro.
*/
📌 Cuándo comentar

✅ Hazlo cuando:

El código es complejo y necesita explicación.

Hay una decisión importante que justificar.

Se usa un algoritmo no obvio.

Se documenta una función o módulo.

❌ No lo hagas cuando:

El código es obvio (no comentes let x = 5;).

El comentario repite lo que dice el código.

El comentario es incorrecto o desactualizado.

🔵 Ejemplo de buenos comentarios

javascript
// ============================================================
// CONFIGURACIÓN DEL SISTEMA
// ============================================================

// URL de la API de clima (cambiar a producción)
const API_URL = "https://api.clima.com/v1/";

// Tiempo máximo de espera para peticiones (en milisegundos)
const TIMEOUT = 5000;

// ============================================================
// FUNCIONES PRINCIPALES
// ============================================================

/**
 * Calcula el área de un círculo
 * @param {number} radio - El radio del círculo
 * @returns {number} El área calculada
 */
function calcularAreaCirculo(radio) {
    return Math.PI * radio * radio;
}
📌 Malos comentarios (evitar)

javascript
// Asignar 5 a x
let x = 5;   // ❌ Obvio, no hace falta

// Si el usuario es mayor de edad, imprimir mensaje
if (edad >= 18) {   // ❌ El código ya dice esto
    console.log("Mayor de edad");
}
===============================================================================

NOMBRES DE VARIABLES DESCRIPTIVOS
===============================================================================

El nombre de una variable debe decir qué contiene.

📌 Reglas

Usa nombres descriptivos.

Usa camelCase para variables y funciones.

Usa MAYUSCULAS para constantes fijas.

Usa nombres de verbos para funciones (hacer algo).

🔵 Ejemplos

javascript
// ❌ Malos nombres
let a = "Ana";
let x = 25;
let d = true;

// ✅ Buenos nombres
let nombreUsuario = "Ana";
let edadPersona = 25;
let esMayorDeEdad = true;
🔵 Ejemplos con funciones

javascript
// ❌ Mal
function calc() { /* ... */ }

// ✅ Bueno
function calcularPromedio() { /* ... */ }
📌 Convenciones comunes

Tipo	Ejemplo
Variable	nombreUsuario, contadorVisitas
Constante	PI, URL_API, MAXIMO_INTENTOS
Función	calcularTotal(), obtenerUsuario()
Booleano	esActivo, tienePermiso, estaAutenticado
===============================================================================

SANGRÍA Y ESPACIADO
===============================================================================

El código debe ser visualmente limpio y fácil de leer.

📌 Sangría (indentación)

Cada nivel de anidamiento debe tener una sangría (2 o 4 espacios).

javascript
// ❌ Sin sangría
if (edad >= 18) {
console.log("Mayor de edad");
if (tieneLicencia) {
console.log("Puede conducir");
}
}

// ✅ Con sangría
if (edad >= 18) {
    console.log("Mayor de edad");
    if (tieneLicencia) {
        console.log("Puede conducir");
    }
}
📌 Espacios entre operadores

javascript
// ❌ Sin espacios
let suma=5+3;
let resta=10-2;

// ✅ Con espacios
let suma = 5 + 3;
let resta = 10 - 2;
📌 Líneas en blanco

Usa líneas en blanco para separar secciones lógicas.

javascript
// ============================================================
// DECLARACIÓN DE CONSTANTES
// ============================================================

const NOMBRE_SITIO = "Lucas EGA Academy";
const URL_API = "https://api.ejemplo.com";

// ============================================================
// FUNCIONES
// ============================================================

function saludar(nombre) {
    console.log("Hola " + nombre);
}

function despedir(nombre) {
    console.log("Adiós " + nombre);
}
===============================================================================

PRINCIPIO DRY (DON'T REPEAT YOURSELF)
===============================================================================

"No te repitas a ti mismo."

Si escribes el mismo código en varios lugares, es mejor extraerlo a una función.

🔵 Ejemplo: código repetido

javascript
// ❌ Código repetido
let area1 = 3.14 * 5 * 5;
let area2 = 3.14 * 3 * 3;
let area3 = 3.14 * 7 * 7;
javascript
// ✅ Código DRY
function areaCirculo(radio) {
    return Math.PI * radio * radio;
}

let area1 = areaCirculo(5);
let area2 = areaCirculo(3);
let area3 = areaCirculo(7);
===============================================================================

NOMBRES DE CONSTANTES EN MAYÚSCULAS
===============================================================================

Los valores fijos que nunca cambian se escriben en mayúsculas con guiones bajos.

javascript
// ✅ Constantes
const PI = 3.1416;
const URL_API = "https://api.ejemplo.com";
const MAXIMO_CARACTERES = 255;
const COLOR_PRINCIPAL = "#3498DB";
===============================================================================

EVITAR NÚMEROS MÁGICOS
===============================================================================

Un número mágico es un número literal sin significado claro.

javascript
// ❌ Números mágicos
if (edad >= 18) { /* ... */ }  // ¿Qué significa 18?
let total = precio * 1.21;     // ¿Qué significa 1.21?
javascript
// ✅ Constantes con nombre
const EDAD_MINIMA = 18;
const IVA = 1.21;

if (edad >= EDAD_MINIMA) { /* ... */ }
let total = precio * IVA;
===============================================================================

USAR TIPOS DE DATOS CORRECTAMENTE
===============================================================================

Usa el tipo de dato adecuado para cada valor.

javascript
// ❌ Confuso
let edad = "25";      // string pero es un número
let esMayor = "si";   // string en lugar de booleano

// ✅ Correcto
let edad = 25;        // number
let esMayor = true;   // boolean
===============================================================================

USAR PLANTILLAS DE TEXTO (TEMPLATE STRINGS)
===============================================================================

En lugar de concatenar con +, usa template strings.

javascript
// ❌ Concatenación
let mensaje = "Hola " + nombre + ", tienes " + edad + " años.";

// ✅ Template string
let mensaje = `Hola ${nombre}, tienes ${edad} años.`;
===============================================================================

EVITAR VAR
===============================================================================

Usa let y const. var tiene comportamientos confusos.

javascript
// ❌ Mal
var nombre = "Ana";

// ✅ Bien
let nombre = "Ana";
const PI = 3.1416;
===============================================================================
ERRORES COMUNES CON BUENAS PRÁCTICAS
===============================================================================

❌ Error 1: Comentar demasiado (código sobrecomentado)

Comentar cosas obvias ensucia el código.
Solución: comenta el "por qué", no el "qué".

❌ Error 2: Nombres de variables poco descriptivos

let x; no dice nada. Usa let contador;.

❌ Error 3: No usar sangría

El código sin sangría es ilegible.
Solución: usa sangría de 2 o 4 espacios.

❌ Error 4: Código repetido

Si copias y pegas código, estás violando DRY.
Solución: extrae el código repetido a una función.

❌ Error 5: Números mágicos

if (edad >= 18) no explica qué significa 18.
Solución: usa const EDAD_MINIMA = 18;.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Cuándo debo comentar mi código?

Cuando es complejo, cuando hay una decisión importante,
cuando el código no es obvio. No comentes lo obvio.

❓ ¿Qué es el principio DRY?

Es una práctica que dice: "No te repitas a ti mismo".
Si escribes el mismo código en varios lugares, extraelo a una función.

❓ ¿Por qué no usar var?

var tiene comportamientos confusos (ámbito de función, redeclaración).
let y const son más predecibles.

❓ ¿Qué son los números mágicos?

Son números literales que aparecen en el código sin explicación.
Deben reemplazarse por constantes con nombre.

❓ ¿Cómo sé si mi código es limpio?

Si otra persona puede leerlo y entenderlo sin esfuerzo.
Si tú puedes leerlo dentro de 6 meses y entenderlo.
