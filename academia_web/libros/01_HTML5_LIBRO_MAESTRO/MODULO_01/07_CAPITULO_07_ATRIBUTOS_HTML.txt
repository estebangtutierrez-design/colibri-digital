En los capítulos anteriores aprendiste a escribir etiquetas y a estructurar elementos.

Pero hay algo que hace que una etiqueta sea realmente poderosa:

Son como los ajustes de un instrumento.

===============================================================================
PARTE 1: ¿QUÉ ES UN ATRIBUTO HTML?
===============================================================================

Un atributo es información adicional que se coloca dentro de la etiqueta de apertura de un elemento.

Sintaxis:

<etiqueta atributo="valor">Contenido</etiqueta>

───────────────────────────────────────────────────────────
ANATOMÍA DE UN ATRIBUTO
───────────────────────────────────────────────────────────

+----------------------------------------------------------+
|  <a href="https://ejemplo.com">Click aquí</a>              |
|     ↑     ↑                      ↑                        |
|  Etiqueta Atributo            Valor                       |
+----------------------------------------------------------+

- El atributo siempre va en la etiqueta de apertura.
- Se escribe después del nombre de la etiqueta.
- Lleva un signo igual (=).
- El valor va entre comillas (simples o dobles, aunque se recomienda dobles).

───────────────────────────────────────────────────────────
EJEMPLOS BÁSICOS
───────────────────────────────────────────────────────────

<a href="https://google.com">Ir a Google</a>
<img src="foto.jpg" alt="Descripción de la foto">
<p title="Este es un párrafo con información extra">Texto</p>

===============================================================================
PARTE 2: ATRIBUTOS GLOBALES (LOS QUE PUEDES USAR EN TODAS LAS ETIQUETAS)
===============================================================================

Hay atributos que funcionan en prácticamente cualquier etiqueta HTML.
Se llaman atributos globales.

───────────────────────────────────────────────────────────
id → IDENTIFICADOR ÚNICO
───────────────────────────────────────────────────────────

El atributo id asigna un nombre único a un elemento.
No puede repetirse en la misma página.

Sintaxis:
<etiqueta id="nombre-unico">Contenido</etiqueta>

Ejemplo:
<h1 id="titulo-principal">Bienvenido</h1>
<p id="intro">Este es el primer párrafo.</p>

¿Para qué sirve?
- Para identificar un elemento específico con CSS.
- Para enlazar a una sección desde un enlace interno.
- Para manipular el elemento con JavaScript.

───────────────────────────────────────────────────────────
class → CLASE (AGRUPACIÓN)
───────────────────────────────────────────────────────────

El atributo class asigna una o varias clases a un elemento.
Puede repetirse en varios elementos.

Sintaxis:
<etiqueta class="nombre-clase">Contenido</etiqueta>
<etiqueta class="nombre1 nombre2">Contenido</etiqueta>

Ejemplo:
<p class="destacado">Este párrafo está destacado.</p>
<p class="destacado importante">Este también, pero con más importancia.</p>

¿Para qué sirve?
- Para agrupar elementos que comparten el mismo estilo con CSS.
- Para seleccionar grupos de elementos con JavaScript.

───────────────────────────────────────────────────────────
style → ESTILO EN LÍNEA
───────────────────────────────────────────────────────────

El atributo style permite aplicar estilos CSS directamente al elemento.

Sintaxis:
<etiqueta style="propiedad: valor;">Contenido</etiqueta>

Ejemplo:
<p style="color: red; font-size: 20px;">Texto rojo y grande.</p>

¿Cuándo usarlo?
- Para cambios rápidos y puntuales.
- Para pruebas o prototipos.
- Cuando necesitas un estilo muy específico y no quieres crear una clase.

Nota: Se recomienda usar CSS externo o interno en lugar de style para mantener el código limpio.

───────────────────────────────────────────────────────────
title → TEXTO EMERGENTE
───────────────────────────────────────────────────────────

El atributo title muestra un texto emergente cuando el usuario pasa el cursor sobre el elemento.

Sintaxis:
<etiqueta title="texto emergente">Contenido</etiqueta>

Ejemplo:
<p title="Este es un dato adicional">Pasa el mouse aquí.</p>
<a href="#" title="Ir a la página de inicio">Inicio</a>

¿Para qué sirve?
- Para dar información adicional sin saturar la página.
- Para explicar el propósito de un enlace o botón.
- Para ofrecer ayuda contextual.

───────────────────────────────────────────────────────────
lang → IDIOMA
───────────────────────────────────────────────────────────

El atributo lang indica el idioma del contenido del elemento.

Sintaxis:
<etiqueta lang="código-idioma">Contenido</etiqueta>

Ejemplo:
<p lang="es">Este párrafo está en español.</p>
<p lang="en">This paragraph is in English.</p>

¿Para qué sirve?
- Para que los lectores de pantalla pronuncien correctamente.
- Para que los motores de búsqueda identifiquen el idioma del contenido.

───────────────────────────────────────────────────────────
data-* → DATOS PERSONALIZADOS
───────────────────────────────────────────────────────────

El atributo data-* permite almacenar información personalizada en un elemento.

Sintaxis:
<etiqueta data-nombre="valor">Contenido</etiqueta>

Ejemplo:
<button data-usuario="juan" data-rol="admin">Perfil</button>
<div data-producto="123" data-precio="99.99">Producto</div>

¿Para qué sirve?
- Para guardar información que JavaScript puede leer y usar.
- Para pasar datos entre HTML y JavaScript sin necesidad de usar variables globales.

───────────────────────────────────────────────────────────
tabindex → ORDEN DE TABULACIÓN
───────────────────────────────────────────────────────────

El atributo tabindex controla el orden en que los elementos reciben el foco al presionar la tecla Tab.

Sintaxis:
<etiqueta tabindex="numero">Contenido</etiqueta>

Ejemplo:
<input type="text" tabindex="1">
<input type="text" tabindex="2">
<button tabindex="3">Enviar</button>

¿Para qué sirve?
- Para mejorar la accesibilidad de personas que navegan con teclado.
- Para establecer un orden lógico de navegación.

Nota: Solo usa tabindex cuando sea necesario. El orden natural del HTML suele ser suficiente.

PARTE 3: ATRIBUTOS ESPECÍFICOS (POR ETIQUETA)
===============================================================================

Ciertos atributos solo tienen sentido en etiquetas específicas.

───────────────────────────────────────────────────────────
EN ENLACES (<a>)
───────────────────────────────────────────────────────────

href → URL de destino
target → Dónde abrir el enlace (_self, _blank, _parent, _top)
rel → Relación con la página de destino (nofollow, noopener, noreferrer)

Ejemplo:
<a href="https://google.com" target="_blank" rel="noopener noreferrer">Google</a>

───────────────────────────────────────────────────────────
EN IMÁGENES (<img>)
───────────────────────────────────────────────────────────

src → Ruta de la imagen
alt → Texto alternativo (obligatorio)
width → Ancho en píxeles
height → Alto en píxeles
loading → Carga perezosa (lazy, eager)

Ejemplo:
<img src="logo.png" alt="Logo de la empresa" width="200" height="100" loading="lazy">

───────────────────────────────────────────────────────────
EN FORMULARIOS (<input>)
───────────────────────────────────────────────────────────

type → Tipo de input (text, password, email, number, date, etc.)
name → Nombre del campo (se envía al servidor)
value → Valor por defecto
placeholder → Texto de ejemplo dentro del campo
required → Campo obligatorio
disabled → Campo deshabilitado
readonly → Campo de solo lectura
min → Valor mínimo (para números)
max → Valor máximo (para números)
pattern → Expresión regular para validación

Ejemplo:
<input type="email" name="correo" placeholder="tu@email.com" required>

───────────────────────────────────────────────────────────
EN BOTONES (<button>)
───────────────────────────────────────────────────────────

type → submit, reset, button
disabled → Botón deshabilitado
name → Nombre del botón
value → Valor enviado

Ejemplo:
<button type="submit" name="accion" value="enviar">Enviar</button>

───────────────────────────────────────────────────────────
EN TABLAS (<table>, <td>, <th>)
───────────────────────────────────────────────────────────

colspan → Número de columnas que ocupa una celda
rowspan → Número de filas que ocupa una celda
scope → Indica si es encabezado de columna o fila (col, row)

Ejemplo:
<th colspan="2">Datos personales</th>
<th scope="col">Nombre</th>
<th scope="row">Ana</th>

PARTE 4: ATRIBUTOS BOOLEANOS (SÍ/NO)
===============================================================================

Los atributos booleanos no necesitan un valor.
Su presencia indica "verdadero" y su ausencia "falso".

Sintaxis:
<etiqueta atributo>Contenido</etiqueta>

───────────────────────────────────────────────────────────
EJEMPLOS DE ATRIBUTOS BOOLEANOS
───────────────────────────────────────────────────────────

required → El campo es obligatorio.
<input type="text" required>

disabled → El campo no se puede usar.
<input type="text" disabled>

checked → El checkbox o radio está marcado.
<input type="checkbox" checked>

readonly → El campo es de solo lectura.
<input type="text" readonly>

autofocus → El campo recibe el foco al cargar la página.
<input type="text" autofocus>

hidden → El elemento está oculto.
<p hidden>Este párrafo no se ve.</p>

───────────────────────────────────────────────────────────
REPRESENTACIÓN VISUAL
───────────────────────────────────────────────────────────

Con atributo    →  <input type="text" required>
Sin atributo    →  <input type="text">

El primero será obligatorio; el segundo, no.

PARTE 5: ATRIBUTOS DE ACCESIBILIDAD (ARIA)
===============================================================================

WAI-ARIA (Accessible Rich Internet Applications) es un conjunto de atributos que mejoran la accesibilidad.

───────────────────────────────────────────────────────────
PRINCIPALES ATRIBUTOS ARIA
───────────────────────────────────────────────────────────

aria-label → Proporciona una etiqueta accesible.
<button aria-label="Cerrar ventana">X</button>

aria-hidden → Oculta el elemento de los lectores de pantalla.
<div aria-hidden="true">Decoración</div>

aria-expanded → Indica si un elemento desplegable está abierto.
<button aria-expanded="false">Menú</button>

aria-controls → Asocia un control con el elemento que controla.
<button aria-controls="menu">Abrir menú</button>

aria-live → Anuncia cambios en tiempo real.
<div aria-live="polite">El mensaje de confirmación.</div>

aria-describedby → Asocia una descripción adicional.
<input aria-describedby="ayuda">
<p id="ayuda">Introduce tu correo electrónico.</p>

───────────────────────────────────────────────────────────
CUÁNDO USAR ARIA
───────────────────────────────────────────────────────────

La regla de oro:
"Usa el elemento HTML correcto antes que ARIA."

Ejemplo:
- Usa <button> en lugar de <div role="button">.
- Usa <nav> en lugar de <div role="navigation">.

Usa ARIA solo cuando el elemento nativo no exista o necesites información adicional.

Se separan por espacios.

CÓDIGO HTML

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">

    <title>Atributos HTML - Ejemplo Completo</title>

        <p lang="es" style="font-size: 18px;">Bienvenido a mi página de prueba con atributos.</p>
    </header>

    <nav aria-label="Navegación principal">
        <ul>
            <li><a href="#inicio" class="enlace" title="Ir a Inicio">Inicio</a></li>
            <li><a href="#blog" class="enlace" target="_blank">Blog</a></li>
            <li><a href="#contacto" class="enlace" rel="nofollow">Contacto</a></li>
        </ul>
    </nav>

    <main id="contenido-principal">
        <section id="inicio">
            <h2>Sección de Bienvenida</h2>
            <p data-autor="Esteban" data-fecha="2026-06-25">
                Este párrafo tiene atributos <strong>data-*</strong> personalizados.
            </p>
            <p>
                Aquí hay un enlace a 
                <a href="https://developer.mozilla.org/es/" target="_blank" rel="noopener noreferrer">
                    MDN Web Docs
                </a>
                (abre en nueva pestaña con seguridad).
            </p>
        </section>

        <section>
            <h2>Formulario de Contacto</h2>
            <form action="/enviar" method="POST">
                <label for="nombre">Nombre:</label>
                <input type="text" id="nombre" name="nombre" placeholder="Escribe tu nombre" required autofocus>

                <label for="email">Correo:</label>
                <input type="email" id="email" name="email" placeholder="tu@email.com" required>

                <label for="edad">Edad:</label>
                <input type="number" id="edad" name="edad" min="18" max="99" value="18">

                <label>
                    <input type="checkbox" name="terminos" required checked>
                    Acepto los términos
                </label>

                <button type="submit" disabled>Enviar (deshabilitado)</button>
                <button type="reset">Limpiar</button>
            </form>
        </section>

        <article>
            <h3>Artículo de ejemplo</h3>
            <p>Este artículo tiene un <span title="Texto con información extra">span con título</span>.</p>
        </article>
    </main>

    <aside aria-labelledby="aside-titulo">
        <h3 id="aside-titulo">Dato curioso</h3>
        <p>Los atributos <code>data-*</code> son muy útiles para guardar información.</p>
    </aside>

    <footer>
        <p>&copy; 2026 Lucas EGA Academy</p>
        <p hidden>Este párrafo está oculto</p>
    </footer>

</body>
</html>

PARTE 9: ¿QUÉ PASA DENTRO DEL NAVEGADOR CON LOS ATRIBUTOS?
===============================================================================

Cuando el navegador encuentra un atributo:

1. Lee el nombre del atributo.
2. Lee su valor.
3. Aplica el comportamiento correspondiente:

   - href → Prepara el enlace.
   - src → Prepara la carga de la imagen.
   - alt → Guarda el texto alternativo.
   - required → Marca el campo como obligatorio.
   - disabled → Deshabilita el elemento.
   - style → Aplica los estilos directamente.
   - class → Asocia el elemento a una clase (para CSS y JS).
   - id → Asigna un identificador único.

4. Todo esto ocurre durante la construcción del DOM.

PARTE 10: ERRORES COMUNES CON ATRIBUTOS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR LAS COMILLAS
───────────────────────────────────────────────────────────

Incorrecto: <a href=https://google.com>Google</a>
Correcto:   <a href="https://google.com">Google</a>

Solución: Siempre usa comillas dobles o simples.

───────────────────────────────────────────────────────────
ERROR 2: USAR NOMBRES DE ATRIBUTOS MAL ESCRITOS
───────────────────────────────────────────────────────────

Incorrecto: <img scr="foto.jpg" alt="Foto">
Correcto:   <img src="foto.jpg" alt="Foto">

Solución: Revisa la ortografía del atributo. Los más comunes:
- src (no scr)
- href (no herf)
- alt (no alr)
- type (no tipo)

───────────────────────────────────────────────────────────
ERROR 3: ATRIBUTOS SIN VALOR (CUANDO DEBEN TENERLO)
───────────────────────────────────────────────────────────

Incorrecto: <a href>Enlace</a>
Correcto:   <a href="https://ejemplo.com">Enlace</a>

Solución: La mayoría de los atributos necesitan un valor. Solo los booleanos pueden ir sin valor (required, disabled, checked).

───────────────────────────────────────────────────────────
ERROR 4: REPETIR EL MISMO ID
───────────────────────────────────────────────────────────

Incorrecto:
<h1 id="titulo">Título 1</h1>
<h2 id="titulo">Título 2</h2>

Correcto:
<h1 id="titulo-1">Título 1</h1>
<h2 id="titulo-2">Título 2</h2>

Solución: Los id deben ser únicos en toda la página.

───────────────────────────────────────────────────────────
ERROR 5: OLVIDAR EL ALT EN IMÁGENES
───────────────────────────────────────────────────────────

Incorrecto: <img src="foto.jpg">
Correcto:   <img src="foto.jpg" alt="Descripción de la foto">

Solución: Siempre añade alt a las imágenes. Es obligatorio por accesibilidad y SEO.

===============================================================================
PARTE 11: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Usa un editor de código que coloree los atributos y los valores (Bluefish, VS Code, etc.).
2. Si ves que un enlace no funciona, revisa que href tenga una URL válida y esté entre comillas.
3. Si una imagen no se muestra, revisa que src tenga la ruta correcta y que alt esté presente.
4. Si un formulario no valida, revisa que los atributos required, type, etc., estén bien escritos.
5. Usa el validador del W3C: https://validator.w3.org/

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- Atributo: Información adicional que se coloca en la etiqueta de apertura.
- Atributo global: Válido en cualquier etiqueta HTML (id, class, style, title, lang, data-*).
- Atributo específico: Solo válido en ciertas etiquetas (href en <a>, src en <img>, type en <input>).
- Atributo booleano: No necesita valor; su presencia indica "verdadero" (required, disabled, checked).
- Valor: El contenido del atributo, entre comillas.
- ARIA: Conjunto de atributos para mejorar la accesibilidad.
- data-*: Atributo personalizado para guardar información.
