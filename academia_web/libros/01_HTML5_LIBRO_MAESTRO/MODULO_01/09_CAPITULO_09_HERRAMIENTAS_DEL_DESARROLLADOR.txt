PARTE 1: ¿QUÉ SON LAS HERRAMIENTAS DEL DESARROLLADOR?
===============================================================================

Las Herramientas del Desarrollador (DevTools) son un conjunto de utilidades integradas en los navegadores modernos.

Permiten:

- Inspeccionar y editar el HTML y CSS en tiempo real.
- Ejecutar JavaScript y ver resultados en la consola.
- Depurar errores y encontrar problemas de código.
- Analizar el rendimiento y la velocidad de carga.
- Ver las peticiones de red que hace la página.
- Y mucho más.

───────────────────────────────────────────────────────────
NAVEGADORES QUE LAS INCLUYEN
───────────────────────────────────────────────────────────

- Google Chrome → DevTools (el más usado, recomendado).
- Mozilla Firefox → Developer Tools.
- Microsoft Edge → DevTools (igual que Chrome).
- Safari → Web Inspector (en Mac).

En este capítulo usaremos Google Chrome, pero la mayoría de los conceptos son iguales en todos.

PARTE 3: LAS PESTAÑAS PRINCIPALES (VISIÓN GENERAL)
===============================================================================

Cuando abres DevTools, verás varias pestañas en la parte superior.

───────────────────────────────────────────────────────────
DESCRIPCIÓN DE CADA PESTAÑA
───────────────────────────────────────────────────────────

1. Elements (Elementos)
   → Muestra el HTML de la página.
   → Puedes editarlo en tiempo real.
   → Es la pestaña que más usarás.

2. Console (Consola)
   → Muestra mensajes de error y advertencias.
   → Permite ejecutar código JavaScript.
   → Es el "cuaderno de bitácora" del navegador.

3. Sources (Fuentes)
   → Muestra los archivos originales (HTML, CSS, JS).
   → Permite depurar código paso a paso.
   → Ideal para encontrar errores en JavaScript.

4. Network (Red)
   → Muestra todas las peticiones que hace la página.
   → Archivos HTML, CSS, JS, imágenes, etc.
   → Útil para ver qué se carga y cuánto tarda.

5. Performance (Rendimiento)
   → Mide la velocidad de carga y ejecución.
   → Identifica qué hace lenta a tu página.

6. Application (Aplicación)
   → Muestra el almacenamiento local, cookies, etc.
   → Útil para ver datos guardados.

7. Lighthouse
   → Audita tu página en: Rendimiento, Accesibilidad, SEO, Buenas prácticas.
   → Te da una puntuación y recomendaciones.

PARTE 4: PESTAÑA ELEMENTS (LA MÁS IMPORTANTE AL PRINCIPIO)
===============================================================================

La pestaña "Elements" muestra el HTML de la página tal como lo ve el navegador.

───────────────────────────────────────────────────────────
¿QUÉ PUEDES HACER?
───────────────────────────────────────────────────────────

1. VER EL HTML:
   → Verás la estructura completa de la página.
   → Puedes expandir y colapsar elementos.

2. EDITAR EL HTML EN VIVO:
   → Haz doble clic en el texto de un elemento.
   → Modifica su contenido.
   → Los cambios se ven instantáneamente en la página.
   → (Nota: los cambios son temporales, no se guardan en el archivo).

3. COPIAR UN ELEMENTO:
   → Haz clic derecho sobre un elemento.
   → Selecciona "Copiar" → "Copiar elemento".
   → Así puedes copiar estructuras complejas.

4. ELIMINAR UN ELEMENTO:
   → Haz clic derecho sobre un elemento.
   → Selecciona "Eliminar".
   → El elemento desaparece de la página.

5. VER LOS ESTILOS CSS:
   → En la parte derecha, verás los estilos aplicados al elemento seleccionado.
   → Puedes ver qué reglas CSS se están aplicando.

6. EDITAR ESTILOS CSS EN VIVO:
   → En la parte derecha, puedes añadir o modificar estilos.
   → Los cambios se ven en tiempo real.

7. VER EL MODELO DE CAJA:
   → En la parte derecha, verás el "Box Model" del elemento.
   → Muestra el contenido, padding, borde y margen.

───────────────────────────────────────────────────────────
EJEMPLO PRÁCTICO (PRUEBA AHORA MISMO)
───────────────────────────────────────────────────────────

1. Abre DevTools (F12).
2. Ve a la pestaña "Elements".
3. Encuentra el <h1> de esta página.
4. Haz doble clic en el texto y cámbialo.
5. Observa cómo cambia la página en tiempo real.

PARTE 5: PESTAÑA CONSOLE (EL DIARIO DEL NAVEGADOR)
===============================================================================

La consola es donde el navegador escribe mensajes.
Es donde verás los errores, advertencias y mensajes que tú mismo puedes añadir.

───────────────────────────────────────────────────────────
¿QUÉ PUEDES HACER?
───────────────────────────────────────────────────────────

1. VER ERRORES:
   → Si hay errores en tu código, aparecerán aquí en rojo.
   → Incluyen el nombre del archivo y la línea donde ocurrió.

2. EJECUTAR CÓDIGO JAVASCRIPT:
   → Escribe código JavaScript directamente en la consola.
   → Presiona Enter y se ejecutará.

3. PROBAR FUNCIONES:
   → Si tienes una función en tu código, puedes llamarla desde la consola.

4. MOSTRAR MENSAJES (PARA DEPURAR):
   → Puedes añadir console.log("mensaje") en tu código JavaScript.
   → Los mensajes aparecerán en la consola.

───────────────────────────────────────────────────────────
EJEMPLO PRÁCTICO (PRUEBA AHORA MISMO)
───────────────────────────────────────────────────────────

1. Abre DevTools (F12).
2. Ve a la pestaña "Console".
3. Escribe: console.log("Hola, Profesor Búho");
4. Presiona Enter.
5. Verás el mensaje en la consola.

───────────────────────────────────────────────────────────
ERRORES COMUNES EN LA CONSOLA
───────────────────────────────────────────────────────────

- Uncaught SyntaxError → Error de sintaxis (falta un paréntesis, comilla, etc.).
- Uncaught ReferenceError → Variable no definida.
- Uncaught TypeError → Error de tipo (ej: intentar llamar a una función que no existe).
- 404 Not Found → Recurso no encontrado (archivo no existe).

PARTE 6: PESTAÑA SOURCES (DEPURACIÓN DE CÓDIGO)
===============================================================================

La pestaña "Sources" muestra los archivos originales de tu página.

───────────────────────────────────────────────────────────
¿QUÉ PUEDES HACER?
───────────────────────────────────────────────────────────

1. VER ARCHIVOS:
   → Verás la lista de archivos HTML, CSS, JS.
   → Puedes abrirlos y ver su contenido.

2. DEPURAR JAVASCRIPT:
   → Puedes establecer "puntos de interrupción" (breakpoints).
   → El código se detendrá en esos puntos para que puedas inspeccionar variables.

3. VER VARIABLES:
   → Cuando el código está detenido, puedes ver los valores de las variables.

4. EJECUTAR PASO A PASO:
   → Puedes ejecutar el código línea por línea para ver qué hace.

───────────────────────────────────────────────────────────
EJEMPLO PRÁCTICO (AVANZADO)
───────────────────────────────────────────────────────────

1. Abre DevTools (F12).
2. Ve a la pestaña "Sources".
3. Busca tu archivo JavaScript.
4. Haz clic en el número de una línea para poner un "breakpoint" (punto rojo).
5. Recarga la página.
6. El código se detendrá en ese punto.
7. Puedes inspeccionar variables y ejecutar paso a paso.

===============================================================================
PARTE 7: PESTAÑA NETWORK (ANÁLISIS DE RED)
===============================================================================

La pestaña "Network" muestra todas las peticiones que hace la página al cargar.

───────────────────────────────────────────────────────────
¿QUÉ PUEDES HACER?
───────────────────────────────────────────────────────────

1. VER TODAS LAS PETICIONES:
   → HTML, CSS, JS, imágenes, fuentes, etc.
   → Cada petición muestra: nombre, tipo, tamaño, tiempo de carga.

2. VER EL TIEMPO DE CARGA:
   → Cuánto tarda en cargar cada archivo.
   → Identifica qué archivos son más lentos.

3. VER EL ESTADO DE LA PETICIÓN:
   → 200 (OK) → Todo bien.
   → 404 (Not Found) → Archivo no encontrado.
   → 500 (Internal Server Error) → Error en el servidor.

4. VER EL CONTENIDO:
   → Puedes hacer clic en una petición y ver su contenido (respuesta).

───────────────────────────────────────────────────────────
EJEMPLO PRÁCTICO (PRUEBA AHORA MISMO)
───────────────────────────────────────────────────────────

1. Abre DevTools (F12).
2. Ve a la pestaña "Network".
3. Recarga la página (F5).
4. Verás todas las peticiones que se hicieron.
5. Haz clic en una petición para ver su contenido.

===============================================================================
PARTE 8: PESTAÑA PERFORMANCE (RENDIMIENTO)
===============================================================================

La pestaña "Performance" mide la velocidad y eficiencia de tu página.

───────────────────────────────────────────────────────────
¿QUÉ PUEDES HACER?
───────────────────────────────────────────────────────────

1. GRABAR LA CARGA DE LA PÁGINA:
   → Presiona "Record" y recarga la página.
   → DevTools graba todo lo que ocurre durante la carga.

2. ANALIZAR EL TIEMPO DE CARGA:
   → Puedes ver cuánto tarda cada parte:
     - Parseo del HTML.
     - Aplicación de estilos.
     - Ejecución de JavaScript.
     - Renderizado.

3. IDENTIFICAR CUELLOS DE BOTELLA:
   → Ver qué partes son más lentas.
   → Optimizar esas partes.

───────────────────────────────────────────────────────────
CONSEJO DEL PROFESOR BÚHO
───────────────────────────────────────────────────────────

"Si tu página es lenta, no adivines la razón.
Usa Performance para medir exactamente qué está causando la lentitud.
Las suposiciones son el enemigo de la optimización."

===============================================================================
PARTE 9: PESTAÑA APPLICATION (ALMACENAMIENTO)
===============================================================================

La pestaña "Application" muestra los datos guardados por la página.

───────────────────────────────────────────────────────────
¿QUÉ PUEDES VER?
───────────────────────────────────────────────────────────

1. Local Storage:
   → Datos guardados con localStorage.setItem().
   → Puedes ver, añadir, modificar y eliminar datos.

2. Session Storage:
   → Datos guardados con sessionStorage.setItem().
   → Se borran al cerrar la pestaña.

3. Cookies:
   → Las cookies de la página.

4. Base de datos IndexedDB:
   → Bases de datos locales (más avanzado).

5. Cache Storage:
   → Archivos guardados en caché.

───────────────────────────────────────────────────────────
EJEMPLO PRÁCTICO
───────────────────────────────────────────────────────────

1. Abre DevTools (F12).
2. Ve a la pestaña "Application".
3. En el menú de la izquierda, busca "Local Storage".
4. Verás los datos guardados por la página.

PARTE 10: PESTAÑA LIGHTHOUSE (AUDITORÍA AUTOMÁTICA)
===============================================================================

Lighthouse es una herramienta que audita tu página automáticamente.

───────────────────────────────────────────────────────────
¿QUÉ MIDE LIGHTHOUSE?
───────────────────────────────────────────────────────────

1. Performance (Rendimiento): ¿Qué tan rápida es tu página?
2. Accessibility (Accesibilidad): ¿Es usable para personas con discapacidad?
3. Best Practices (Buenas prácticas): ¿Usas código moderno y seguro?
4. SEO (Posicionamiento): ¿Está optimizada para buscadores?
5. Progressive Web App: ¿Puede funcionar como una app?

───────────────────────────────────────────────────────────
¿CÓMO USARLO?
───────────────────────────────────────────────────────────

1. Abre DevTools (F12).
2. Ve a la pestaña "Lighthouse" (puede estar en "Más pestañas").
3. Selecciona las categorías que quieres auditar.
4. Haz clic en "Generate report".
5. Lighthouse te dará una puntuación (0-100) en cada categoría.
6. También te dará recomendaciones para mejorar.

───────────────────────────────────────────────────────────
EJEMPLO DE RESULTADOS
───────────────────────────────────────────────────────────

- Performance: 85 → Bueno, pero se puede optimizar imágenes.
- Accessibility: 95 → Excelente, muy accesible.
- SEO: 90 → Bueno, pero falta meta description.
- Best Practices: 88 → Bueno, pero usa HTTP en lugar de HTTPS.

PARTE 12: EJEMPLO PRÁCTICO COMPLETO (DEPURACIÓN DE UNA PÁGINA)
===============================================================================

Vamos a usar DevTools para depurar una página con errores.

───────────────────────────────────────────────────────────
PASO 1: CREAR UNA PÁGINA CON ERRORES (INTENCIONALMENTE)
───────────────────────────────────────────────────────────

CÓDIGO HTML

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Depuración con DevTools</title>
    <style>
        /* Error intencional: estilo no cerrado */
        .destacado {
            color: red;
            font-weight: bold;
        /* Falta el cierre */
    </style>
</head>
<body>
    <h1>Bienvenido</h1>
    <p class="destacado">Este párrafo debería ser rojo y negrita.</p>
    <script>
        // Error intencional: variable no definida
        console.log(mensaje);
    </script>
</body>
</html>

FIN DEL CÓDIGO

───────────────────────────────────────────────────────────
PASO 2: ABRIR DEVTOOLS Y BUSCAR ERRORES
───────────────────────────────────────────────────────────

1. Abre el archivo en el navegador.
2. Presiona F12.
3. Ve a la pestaña "Console".
   → Verás un error: "Uncaught ReferenceError: mensaje is not defined"
   → Esto indica que la variable "mensaje" no está definida.

4. Ve a la pestaña "Elements".
   → Busca el párrafo con clase "destacado".
   → En la parte derecha, verás los estilos aplicados.
   → Notarás que el estilo no se está aplicando correctamente.
   → El error en el CSS (falta el cierre) causó que el estilo no se aplique.

5. Ve a la pestaña "Network".
   → Recarga la página.
   → Verás todas las peticiones (HTML, CSS, JS).
   → Si hubiera archivos externos, verías su estado.

───────────────────────────────────────────────────────────
PASO 3: CORREGIR LOS ERRORES
───────────────────────────────────────────────────────────

1. Cierra el CSS con el cierre correcto:
   .destacado {
       color: red;
       font-weight: bold;
   }

2. Define la variable "mensaje" o elimina el console.log:
   let mensaje = "Hola, Profesor Búho";
   console.log(mensaje);

3. Guarda y recarga la página.
   → Los errores deberían desaparecer.

PARTE 13: ERRORES COMUNES AL USAR DEVTOOLS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: NO ACTUALIZAR LA PÁGINA DESPUÉS DE CAMBIOS
───────────────────────────────────────────────────────────

Si modificas el código en el editor, recuerda guardar (Ctrl+S) y actualizar el navegador (F5).

───────────────────────────────────────────────────────────
ERROR 2: EDITAR EN DEVTOOLS Y PENSAR QUE SE GUARDÓ
───────────────────────────────────────────────────────────

Los cambios en DevTools son temporales.
No se guardan en el archivo original.
Si quieres guardar cambios, cópialos y pégalos en tu editor.

───────────────────────────────────────────────────────────
ERROR 3: OLVÍDATE DE QUE DEVTOOLS ESTÁ ABIERTO
───────────────────────────────────────────────────────────

DevTools puede cambiar el comportamiento de la página (ej: puede hacerla más lenta al grabar).
Si notas comportamientos extraños, cierra DevTools y vuelve a probar.

───────────────────────────────────────────────────────────
ERROR 4: NO SABER QUÉ BUSCAR EN LA CONSOLA
───────────────────────────────────────────────────────────

La consola puede mostrar muchos mensajes.
Busca errores en rojo (estos son los más importantes).
Las advertencias en amarillo son menos graves pero también se deben revisar.

───────────────────────────────────────────────────────────
ERROR 5: NO USAR EL FILTRO EN NETWORK
───────────────────────────────────────────────────────────

La pestaña Network puede mostrar muchas peticiones (imágenes, fuentes, etc.).
Usa el filtro para ver solo lo que te interesa:
- All (todas)
- HTML
- CSS
- JS
- Images
- Media

PARTE 17: MINI PROYECTO

PROYECTO: AUDITORÍA DE TU PROPIA PÁGINA

Toma la página que creaste en el capítulo anterior (la plantilla profesional) o cualquier página que hayas creado.
Realiza una auditoría completa usando DevTools.

PASOS:

1. Abre tu página en el navegador.
2. Abre DevTools (F12).
3. Ve a la pestaña "Lighthouse".
4. Ejecuta una auditoría completa (todas las categorías).
5. Anota las puntuaciones en:
   - Performance
   - Accessibility
   - Best Practices
   - SEO

6. Lee las recomendaciones de Lighthouse.
7. Corrige TODOS los problemas que puedas:
   - Añade meta description (si falta).
   - Añade alt a las imágenes (si falta).
   - Optimiza el rendimiento (si es posible).
   - Corrige cualquier error de accesibilidad.

8. Vuelve a ejecutar Lighthouse y compara las nuevas puntuaciones.
9. Escribe un informe breve con:
   - Las puntuaciones antes y después.
   - Los cambios que hiciste.
   - El impacto de esos cambios.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- DevTools: Herramientas del desarrollador integradas en el navegador.
- Elements: Pestaña que muestra el HTML de la página.
- Console: Pestaña que muestra mensajes, errores y permite ejecutar código.
- Sources: Pestaña que muestra los archivos originales.
- Network: Pestaña que muestra las peticiones de red.
- Performance: Pestaña que mide el rendimiento de la página.
- Lighthouse: Herramienta que audita la página y da recomendaciones.
- Breakpoint: Punto de interrupción en JavaScript (detiene la ejecución).
- Depurar: Encontrar y corregir errores en el código.
- Auditoría: Evaluación de la calidad de una página web.
- 404: Código de error cuando un recurso no se encuentra.

¿SABÍAS QUE...?
===============================================================================

Las herramientas del desarrollador de Chrome fueron creadas originalmente por un ingeniero de Google llamado Pavel Feldman.

Se inspiraron en las herramientas de Firebug, un complemento de Firefox que fue muy popular en los primeros años de la web.

Hoy, DevTools es una de las herramientas más poderosas y utilizadas por desarrolladores de todo el mundo.

Errores frecuentes:
- No actualizar la página después de cambios en el editor.
- Pensar que los cambios en DevTools son permanentes.
- No saber qué buscar en la consola.
- No usar filtros en la pestaña Network.

Preguntas frecuentes:
- ¿Los cambios en DevTools se guardan? No, son temporales.
- ¿Qué hago si veo un error en la consola? Busca el nombre del archivo y la línea, ve a ese archivo y corrige el error.
- ¿Cómo sé si mi página es rápida? Usa la pestaña Performance o Lighthouse.
- ¿Cómo sé si mi página es accesible? Usa Lighthouse y revisa la puntuación de Accesibilidad.
- ¿Qué es un breakpoint? Un punto donde el código se detiene para que puedas inspeccionar variables.

Analogías utilizadas:
- DevTools = Rayos X de la página web.
- Consola = Diario del navegador.
- Elements = Caja de herramientas para editar.
- Network = Registro de todas las peticiones.
- Lighthouse = Auditor de calidad.

Ejemplos utilizados:
- Depuración de una página con errores (CSS incompleto, variable no definida).
- Edición en vivo de HTML y CSS.
- Uso de la consola para ejecutar JavaScript.
- Auditoría con Lighthouse.

Mini proyecto:

- Capaz de encontrar y corregir errores usando la consola.
