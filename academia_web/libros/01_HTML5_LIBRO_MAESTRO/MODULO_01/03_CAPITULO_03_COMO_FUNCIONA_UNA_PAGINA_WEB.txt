===============================================================================
PASO A PASO
¿QUÉ OCURRE CUANDO ESCRIBES UNA DIRECCIÓN WEB?
===============================================================================

Supongamos que escribes:

https://www.lucasegaacademy.com

Entonces ocurre lo siguiente.

PASO 1

Abres el navegador.

Puede ser:

Google Chrome

Firefox

Edge

Brave

Opera

Safari

El navegador espera una dirección.

---------------------------------------------------

PASO 2

Escribes la dirección.

https://www.lucasegaacademy.com

y presionas ENTER.

---------------------------------------------------

PASO 3

El navegador busca dónde está ese sitio.

Utiliza el sistema DNS.

El DNS funciona como una agenda telefónica.

Convierte:

www.lucasegaacademy.com

en

192.xxx.xxx.xxx

(es decir, una dirección IP)

---------------------------------------------------

PASO 4

El navegador encuentra el servidor.

Ahora envía una solicitud.

La petición dice algo parecido a:

"Hola.

Necesito el archivo principal de esta página."

---------------------------------------------------

PASO 5

El servidor recibe la solicitud.

Busca el archivo.

Generalmente:

index.html

Después responde.

¿QUÉ ENVÍA EL SERVIDOR?
===============================================================================

Normalmente envía:

HTML

CSS

JavaScript

Imágenes

Videos

Fuentes

Archivos adicionales

Todo eso viaja por Internet.

===============================================================================
¿QUÉ HACE EL NAVEGADOR?
===============================================================================

Cuando recibe el HTML comienza a leerlo.

De arriba hacia abajo.

Como si leyera un libro.

Encuentra algo parecido a esto.

<!DOCTYPE html>

<html>

<head>

<title>Mi página</title>

</head>

<body>

<h1>Hola Mundo</h1>

</body>

</html>

Entonces comienza a construir la página.

===============================================================================
EL TRABAJO DE HTML
===============================================================================

HTML solamente describe la estructura.

Ejemplo:

Aquí habrá un título.

Aquí habrá un párrafo.

Aquí habrá una imagen.

Aquí habrá un botón.

No aplica colores.

No mueve elementos.

No crea animaciones.

Sólo organiza.

===============================================================================
EL TRABAJO DE CSS
===============================================================================

Después aparece CSS.

CSS dice:

Este título será azul.

El fondo será blanco.

La letra será grande.

El botón será verde.

Los márgenes serán de 20 píxeles.

Ahora la página comienza a verse bonita.

===============================================================================
EL TRABAJO DE JAVASCRIPT
===============================================================================

Después entra JavaScript.

JavaScript agrega vida.

Puede hacer:

Abrir ventanas.

Validar formularios.

Mover objetos.

Mostrar menús.

Crear juegos.

Actualizar datos.

Animar elementos.

Sin JavaScript la mayoría de las páginas modernas serían estáticas.

===============================================================================
EL RENDERIZADO
===============================================================================

Cuando el navegador junta:

HTML

+

CSS

+

JavaScript

Comienza el proceso llamado:

RENDERIZADO

Renderizar significa:

Construir la página que finalmente ves en la pantalla.

===============================================================================
EJEMPLO COMPLETO
===============================================================================

index.html

<html>

<h1>Lucas EGA Academy</h1>

<p>Bienvenido.</p>

</body>

</html>

CSS

}

JavaScript

alert("Bienvenido");

Resultado.

El navegador muestra:

Lucas EGA Academy

Bienvenido

y aparece un mensaje de bienvenida.

===============================================================================
ERRORES COMUNES
===============================================================================

Pensar que HTML crea páginas bonitas.

No.

Eso lo hace CSS.

---------------------------------------------------

Pensar que JavaScript reemplaza HTML.

No.

JavaScript necesita HTML.

---------------------------------------------------

Pensar que el servidor muestra la página.

No.

Quien realmente la muestra es el navegador.
