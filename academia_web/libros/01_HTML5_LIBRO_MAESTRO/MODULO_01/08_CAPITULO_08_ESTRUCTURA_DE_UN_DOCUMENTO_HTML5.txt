+----------------------------------------------------------+

|  |                                                         |

|  |  |  <meta name="viewport" content="...">               |

|  |  |  <link rel="stylesheet" href="style.css">          |
|  |  |  <script src="app.js" defer></script>              |

===============================================================================
PARTE 2: EL DOCTYPE Y EL ATRIBUTO lang
===============================================================================

───────────────────────────────────────────────────────────
<!DOCTYPE html> → EL PERMISO DE CONSTRUCCIÓN
───────────────────────────────────────────────────────────

Esta línea le dice al navegador:
"Usa el modo estándar de HTML5 para interpretar esta página."

Sin ella, el navegador podría entrar en "modo quirks" (modo de compatibilidad con navegadores antiguos), lo que causaría errores de renderizado.

Siempre debe ser la PRIMERA línea del documento.

───────────────────────────────────────────────────────────
<html lang="es"> → EL TERRENO
───────────────────────────────────────────────────────────

El atributo lang="es" indica el idioma principal de la página.

¿Por qué es importante?
- Los lectores de pantalla lo usan para pronunciar correctamente.
- Google lo usa para mostrar la página en los resultados de búsqueda correctos.
- Los navegadores lo usan para ofrecer traducciones automáticas.

Si tu página está en español, usa lang="es".
Si está en inglés, usa lang="en".
Si mezclas idiomas, puedes usar lang en elementos específicos.

PARTE 3: EL ELEMENTO <head> (LOS PLANOS Y LAS ESPECIFICACIONES)
===============================================================================

El <head> contiene información que NO se ve en la página, pero es esencial para el navegador, los buscadores y los lectores de pantalla.

───────────────────────────────────────────────────────────
METADATOS OBLIGATORIOS
───────────────────────────────────────────────────────────

1. <meta charset="UTF-8">
   Define la codificación de caracteres.
   Permite usar tildes, ñ, emojis y caracteres especiales.
   SIN ESTO, tu página mostrará caracteres extraños (�, Ã±, etc.).

2. <meta name="viewport" content="width=device-width, initial-scale=1.0">
   Hace que tu página sea responsive en dispositivos móviles.
   SIN ESTO, tu página se verá como si estuviera en una computadora de escritorio, pero miniaturizada en el celular.

3. <title>Nombre de la página</title>
   Aparece en la pestaña del navegador.
   Es el factor más importante para el SEO.
   Debe ser descriptivo y contener las palabras clave principales.

───────────────────────────────────────────────────────────
METADATOS RECOMENDADOS
───────────────────────────────────────────────────────────

4. <meta name="description" content="Descripción de la página (150-160 caracteres)">
   Es el texto que aparece debajo del título en los resultados de búsqueda de Google.
   Atrae clics. Debe ser atractivo y resumir el contenido.

5. <meta name="robots" content="index, follow">
   Indica a los buscadores qué hacer:
   - index: Permite que la página sea indexada.
   - follow: Permite que se sigan los enlaces.
   - noindex: No indexar.
   - nofollow: No seguir enlaces.

───────────────────────────────────────────────────────────
METADATOS PARA REDES SOCIALES (OPEN GRAPH)
───────────────────────────────────────────────────────────

6. <meta property="og:title" content="Título para redes sociales">
7. <meta property="og:description" content="Descripción para redes sociales">
8. <meta property="og:image" content="URL de la imagen (1200x630px)">
9. <meta property="og:url" content="URL de la página">

Estos metadatos controlan cómo se ve tu página cuando se comparte en Facebook, LinkedIn, WhatsApp, etc.

───────────────────────────────────────────────────────────
RECURSOS EXTERNOS
───────────────────────────────────────────────────────────

10. <link rel="stylesheet" href="ruta/estilos.css">
    Enlaza una hoja de estilos CSS.

11. <link rel="icon" href="favicon.ico" type="image/x-icon">
    Coloca el ícono que aparece en la pestaña del navegador.

12. <script src="ruta/script.js" defer></script>
    Enlaza un archivo de JavaScript.
    El atributo defer retrasa la ejecución hasta que la página esté cargada.

PARTE 4: EL ELEMENTO <body> (LA CASA EN SÍ)
===============================================================================

El <body> contiene TODO el contenido visible de la página.

Se organiza en una jerarquía de elementos semánticos que describen el propósito de cada parte.

───────────────────────────────────────────────────────────
LA JERARQUÍA SEMÁNTICA DEL <body>
───────────────────────────────────────────────────────────

<body>
    │
    ├── <header> → Cabecera (logo, título, navegación principal).
    │
    ├── <nav> → Menú de navegación principal (suele ir dentro del header o justo después).
    │
    ├── <main> → Contenido principal (UNO SOLO por página).
    │   │
    │   ├── <section> → Agrupa contenido temático.
    │   │   └── <h2> a <h6> (encabezados de la sección)
    │   │
    │   ├── <article> → Contenido independiente (blog, noticia, post).
    │   │   └── <h2> a <h6> (encabezados del artículo)
    │   │
    │   └── (puedes tener múltiples secciones y artículos)
    │
    ├── <aside> → Contenido complementario (barra lateral, publicidad, enlaces relacionados).
    │
    └── <footer> → Pie de página (copyright, contacto, enlaces legales).

───────────────────────────────────────────────────────────
REGLAS IMPORTANTES DEL <body>
───────────────────────────────────────────────────────────

1. Solo debe haber UN <main> por página.
2. Todo el contenido único de la página debe estar dentro del <main>.
3. El <header> y <footer> pueden repetirse dentro de secciones y artículos.
4. La jerarquía de encabezados debe ser lógica: h1 (título principal), h2 (subtítulos), h3 (subsecciones), etc.
5. Usa la etiqueta que mejor describa el contenido (semántica).

===============================================================================
PARTE 5: EL ESQUEMA DE CONTENIDO (OUTLINE) – JERARQUÍA DE ENCABEZADOS
===============================================================================

El "outline" es la estructura de títulos de tu página.
Es como el índice de un libro.

───────────────────────────────────────────────────────────
REGLAS DEL OUTLINE
───────────────────────────────────────────────────────────

1. Solo un <h1> por página.
   Es el título principal de toda la página.
   Debe describir el tema central.

2. Los <h2> son los capítulos principales.
   Representan las secciones grandes del contenido.

3. Los <h3> son subcapítulos dentro de los <h2>.

4. Los <h4> a <h6> son niveles más profundos.

5. No saltes niveles.
   Incorrecto: h1 → h3 (sin h2 entre medio).
   Correcto:   h1 → h2 → h3 → h4.

───────────────────────────────────────────────────────────
EJEMPLO DE ESQUEMA CORRECTO
───────────────────────────────────────────────────────────

<h1>Guía de Estudio 2026</h1>          ← Título principal

    <h2>Matemáticas</h2>               ← Capítulo 1
        <h3>Álgebra</h3>               ← Subcapítulo 1.1
        <h3>Geometría</h3>             ← Subcapítulo 1.2

    <h2>Programación</h2>              ← Capítulo 2
        <h3>HTML</h3>                  ← Subcapítulo 2.1
            <h4>Etiquetas</h4>         ← Subcapítulo 2.1.1
            <h4>Atributos</h4>         ← Subcapítulo 2.1.2

        <h3>CSS</h3>                   ← Subcapítulo 2.2

    <h2>Idiomas</h2>                   ← Capítulo 3

Este esquema es lógico, claro y fácil de seguir.

PARTE 9: ¿QUÉ PASA DENTRO DEL NAVEGADOR CON LA ESTRUCTURA?
===============================================================================

Cuando el navegador recibe un documento HTML5:

1. Lee el <!DOCTYPE html> y entra en modo estándar.

2. Construye el DOM (Document Object Model):
   - Cada etiqueta se convierte en un nodo.
   - La jerarquía del DOM refleja la anidación del HTML.

3. Procesa el <head>:
   - Aplica el charset (UTF-8).
   - Aplica el viewport.
   - Carga el título.
   - Descarga los recursos (CSS, JS) en paralelo.

4. Descarga y aplica el CSS (si hay enlaces externos).

5. Construye el CSSOM (CSS Object Model).

6. Combina DOM + CSSOM = Render Tree.

7. Calcula el layout (posición y tamaño de cada elemento).

8. Pinta los píxeles en la pantalla.

9. Ejecuta el JavaScript (si está cargado).

Todo esto ocurre en milisegundos.

PARTE 10: ERRORES COMUNES EN LA ESTRUCTURA DEL DOCUMENTO
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR EL DOCTYPE O EL CHARSET
───────────────────────────────────────────────────────────

Consecuencia: La página se ve mal o muestra caracteres extraños.

Solución: Siempre empieza con <!DOCTYPE html> y <meta charset="UTF-8">.

───────────────────────────────────────────────────────────
ERROR 2: OLVIDAR EL VIEWPORT
───────────────────────────────────────────────────────────

Consecuencia: La página no es responsive en móviles (se ve miniaturizada).

Solución: Siempre incluye <meta name="viewport" content="width=device-width, initial-scale=1.0">.

───────────────────────────────────────────────────────────
ERROR 3: TENER MÁS DE UN <main>
───────────────────────────────────────────────────────────

Consecuencia: Confunde a los lectores de pantalla y a Google.

Solución: Solo UN <main> por página.

───────────────────────────────────────────────────────────
ERROR 4: COLOCAR CONTENIDO REPETIDO DENTRO DEL <main>
───────────────────────────────────────────────────────────

Incorrecto: Poner el menú de navegación o el footer dentro del <main>.

Correcto: El <main> solo debe contener el contenido único de esa página.

───────────────────────────────────────────────────────────
ERROR 5: JERARQUÍA DE ENCABEZADOS INCORRECTA
───────────────────────────────────────────────────────────

Incorrecto: h1 → h3 (saltando h2).
Incorrecto: Múltiples h1 en la misma página.

Correcto: h1 → h2 → h3 → h4.

───────────────────────────────────────────────────────────
ERROR 6: NO USAR SEMÁNTICA (TODO CON <div>)
───────────────────────────────────────────────────────────

Consecuencia: El código no comunica su propósito.

Solución: Usa <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- DOCTYPE: Declaración que indica al navegador la versión de HTML.
- Metadato: Información en el <head> que describe la página.
- Charset: Codificación de caracteres (UTF-8 incluye tildes, ñ y emojis).
- Viewport: Configuración para la visualización en dispositivos móviles.
- Open Graph: Protocolo de Facebook para mejorar la vista previa en redes sociales.
- SEMÁNTICA: Uso de etiquetas que describen el propósito del contenido.
- OUTLINE: Estructura jerárquica de encabezados de la página.
- DOM: Document Object Model, la representación en árbol del documento HTML.

¿SABÍAS QUE...?
===============================================================================

Google utiliza la estructura semántica de tu página para entender de qué trata.

El <main> le dice a Google: "Aquí está el contenido más importante".
El <nav> le dice: "Aquí está la navegación".
El <article> le dice: "Este contenido es independiente".

Usar estas etiquetas no es opcional si quieres que tu página sea encontrada fácilmente.
