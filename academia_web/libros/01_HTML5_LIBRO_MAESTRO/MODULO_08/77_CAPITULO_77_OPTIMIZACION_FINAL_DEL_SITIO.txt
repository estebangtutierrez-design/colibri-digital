PARTE 1: LIMPIEZA DE CÓDIGO
===============================================================================

El código limpio es más rápido de cargar y más fácil de mantener.

───────────────────────────────────────────────────────────
1. ELIMINAR COMENTARIOS DE PRODUCCIÓN
───────────────────────────────────────────────────────────

Los comentarios son útiles en desarrollo, pero no son necesarios en producción.

```html
<!-- ❌ Antes de optimizar -->
<!-- ===== HEADER ===== -->
<header>...</header>
<!-- ===== SECCIÓN HERO ===== -->
<section class="hero">...</section>
<!-- ===== SECCIÓN CURSOS ===== -->
<section id="cursos">...</section>

<!-- ✅ Después de optimizar -->
<header>...</header>
<section class="hero">...</section>
<section id="cursos">...</section>
Herramientas:

HTMLMinifier: Elimina comentarios y espacios.

Minify: Herramientas online para minimizar código.

───────────────────────────────────────────────────────────

ELIMINAR CÓDIGO NO UTILIZADO
───────────────────────────────────────────────────────────

Revisa tu código y elimina lo que no se usa.

html
<!-- ❌ Antes de optimizar -->
<!-- Clase no utilizada -->
<div class="antiguo">...</div>

<!-- ✅ Después de optimizar -->
<!-- Eliminar la clase "antiguo" si no se usa -->
css
/* ❌ Antes de optimizar */
.antiguo {
    color: red;
}

/* ✅ Después de optimizar */
/* Eliminar la regla si no se usa */
javascript
// ❌ Antes de optimizar
function funcionAntigua() {
    console.log('No se usa');
}

// ✅ Después de optimizar
// Eliminar la función si no se usa
───────────────────────────────────────────────────────────

MINIMIZAR HTML, CSS Y JAVASCRIPT
───────────────────────────────────────────────────────────

La minimización elimina espacios, saltos de línea y comentarios innecesarios.

Antes de minimizar:

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Bienvenido</h1>
</body>
</html>
Después de minimizar:

html
<!DOCTYPE html><html lang="es"><head><meta charset="UTF-8"><title>Mi Página</title><link rel="stylesheet" href="style.css"></head><body><h1>Bienvenido</h1></body></html>
Herramientas:

HTML: HTMLMinifier.

CSS: CSSNano.

JavaScript: Terser, UglifyJS.

PARTE 2: OPTIMIZACIÓN DE IMÁGENES
===============================================================================

Las imágenes suelen ser los recursos más pesados de un sitio.

───────────────────────────────────────────────────────────

CONVERTIR A FORMATOS MODERNOS
───────────────────────────────────────────────────────────

Formato	Mejor para	Tamaño
WebP	Todo (fotos, gráficos)	Muy pequeño
AVIF	Todo (nuevo, mejor compresión)	Extremadamente pequeño
JPEG	Fotos, imágenes con gradientes	Pequeño
PNG	Imágenes con transparencia	Grande
SVG	Logos, iconos, gráficos vectoriales	Muy pequeño
Recomendación: Usa WebP o AVIF para la mayoría de las imágenes.

Herramientas:

Squoosh.app (online, gratuito)

TinyPNG (online, gratuito)

ImageOptim (Mac)

FFmpeg (línea de comandos)

───────────────────────────────────────────────────────────

COMPRIMIR IMÁGENES
───────────────────────────────────────────────────────────

La compresión reduce el tamaño de las imágenes sin perder calidad visible.

Ejemplo:

Original: 2 MB

Comprimida: 200 KB (90% menos)

Herramientas:

Squoosh.app (online, gratuito)

TinyPNG (online, gratuito)

ImageOptim (Mac)

───────────────────────────────────────────────────────────

REDIMENSIONAR IMÁGENES
───────────────────────────────────────────────────────────

No uses una imagen de 4000px de ancho si la vas a mostrar a 400px.

html
<!-- ❌ Incorrecto -->
<img src="imagen-4000px.jpg" alt="Foto" width="400" height="300">

<!-- ✅ Correcto -->
<img src="imagen-800px.webp" alt="Foto" width="800" height="600" loading="lazy">
Tamaños recomendados:

Hero: 1200-1920px de ancho.

Contenido: 800-1200px de ancho.

Miniaturas: 400-600px de ancho.

───────────────────────────────────────────────────────────

USAR LAZY LOADING
───────────────────────────────────────────────────────────

La carga perezosa retrasa la carga de imágenes hasta que son necesarias.

html
<img src="foto.webp" alt="Foto" loading="lazy" decoding="async">
Beneficios:

Página más rápida.

Menos ancho de banda.

Mejor experiencia de usuario.

PARTE 3: OPTIMIZACIÓN DEL RENDIMIENTO
===============================================================================

El rendimiento es la velocidad con la que carga y responde tu sitio.

───────────────────────────────────────────────────────────

COMPRIMIR RECURSOS (GZIP / BROTLI)
───────────────────────────────────────────────────────────

La compresión reduce el tamaño de los archivos que se envían al navegador.

Gzip:

Compatible con todos los navegadores.

Reduce el tamaño de HTML, CSS y JS entre un 60-80%.

Brotli:

Más moderno y eficiente que Gzip.

Mejor compresión (hasta un 20% más).

Configuración:

Apache: mod_deflate para Gzip.

Nginx: gzip on; para Gzip, brotli on; para Brotli.

Netlify/Vercel: Activado por defecto.

───────────────────────────────────────────────────────────

USAR LAZY LOADING
───────────────────────────────────────────────────────────

La carga perezosa retrasa la carga de imágenes y iframes hasta que son necesarios.

html
<img src="foto.webp" alt="Foto" loading="lazy" decoding="async">
<iframe src="mapa.html" loading="lazy"></iframe>
───────────────────────────────────────────────────────────

USAR PRELOAD PARA RECURSOS CRÍTICOS
───────────────────────────────────────────────────────────

La precarga (preload) indica al navegador que cargue recursos críticos con prioridad.

html
<link rel="preload" href="fonts/OpenSans.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="images/hero.webp" as="image">
<link rel="preconnect" href="https://fonts.googleapis.com">
───────────────────────────────────────────────────────────

REDUCIR PETICIONES HTTP
───────────────────────────────────────────────────────────

Cada archivo requiere una petición HTTP. Reduce el número de archivos combinándolos.

html
<!-- ❌ Malo: muchas peticiones -->
<link rel="stylesheet" href="reset.css">
<link rel="stylesheet" href="style.css">
<link rel="stylesheet" href="responsive.css">
<script src="lib1.js"></script>
<script src="lib2.js"></script>
<script src="app.js"></script>

<!-- ✅ Bueno: pocas peticiones -->
<link rel="stylesheet" href="style.min.css">
<script src="app.min.js"></script>

PARTE 4: OPTIMIZACIÓN DE CSS
===============================================================================

───────────────────────────────────────────────────────────

MINIMIZAR CSS
───────────────────────────────────────────────────────────

Elimina espacios, saltos de línea y comentarios innecesarios.

Antes de minimizar:

css
/* ===== HEADER ===== */
header {
    background: #1a202c;
    padding: 20px;
}

/* ===== SECCIÓN HERO ===== */
.hero {
    background: #2b6cb0;
    color: white;
    padding: 60px 20px;

Después de minimizar:

css
header{background:#1a202c;padding:20px}.hero{background:#2b6cb0;color:#fff;padding:60px 20px}
Herramientas:

CSSNano.

CleanCSS.

───────────────────────────────────────────────────────────

COMBINAR ARCHIVOS CSS
───────────────────────────────────────────────────────────

Combina varios archivos CSS en uno solo.

html
<!-- ❌ Malo -->
<link rel="stylesheet" href="reset.css">
<link rel="stylesheet" href="style.css">
<link rel="stylesheet" href="responsive.css">

<!-- ✅ Bueno -->
<link rel="stylesheet" href="style.min.css">
───────────────────────────────────────────────────────────

ELIMINAR CSS NO UTILIZADO
───────────────────────────────────────────────────────────

Revisa tu CSS y elimina las reglas que no se usan.

Herramientas:

PurgeCSS: Elimina CSS no utilizado.

UnCSS: Elimina CSS no utilizado.

PARTE 9: ERRORES COMUNES EN LA OPTIMIZACIÓN
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: NO OPTIMIZAR IMÁGENES
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<img src="foto-4000px.jpg" alt="Foto">
<!-- 5 MB, 4000px de ancho -->
✅ Correcto:

html
<img src="foto-800px.webp" alt="Foto" loading="lazy" width="800" height="600">
<!-- 200 KB, 800px de ancho -->
───────────────────────────────────────────────────────────
ERROR 2: NO MINIMIZAR CÓDIGO
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<link rel="stylesheet" href="style.css">
<script src="app.js"></script>
✅ Correcto:

html
<link rel="stylesheet" href="style.min.css">
<script src="app.min.js" defer></script>
───────────────────────────────────────────────────────────
ERROR 3: NO USAR LAZY LOADING
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<img src="foto.webp" alt="Foto">
<!-- Se carga aunque esté fuera de pantalla -->
✅ Correcto:

html
<img src="foto.webp" alt="Foto" loading="lazy">
<!-- Solo se carga cuando está cerca del viewport -->
───────────────────────────────────────────────────────────
ERROR 4: NO COMPRIMIR RECURSOS
───────────────────────────────────────────────────────────

❌ Incorrecto:

text
No usar Gzip o Brotli.
✅ Correcto:

text
Activar Gzip o Brotli en el servidor.
───────────────────────────────────────────────────────────
ERROR 5: DEJAR CÓDIGO NO UTILIZADO
───────────────────────────────────────────────────────────

❌ Incorrecto:

css
/* Clase no utilizada */
.antiguo {
    color: red;
}
✅ Correcto:

css
/* Eliminar código no utilizado */

===============================================================================

Usa loading="lazy" en imágenes fuera de pantalla.

Activa Gzip o Brotli en el servidor.

Elimina código no utilizado.

Usa herramientas como Lighthouse para medir el rendimiento.
