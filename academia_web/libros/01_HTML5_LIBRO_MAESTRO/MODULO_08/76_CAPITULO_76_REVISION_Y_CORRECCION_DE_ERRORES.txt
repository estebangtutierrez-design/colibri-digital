PARTE 1: TIPOS DE ERRORES COMUNES EN HTML
===============================================================================

───────────────────────────────────────────────────────────
1. ERRORES DE ESTRUCTURA
───────────────────────────────────────────────────────────

**Etiquetas sin cerrar:**
```html
<!-- ❌ Incorrecto -->
<h1>Título

<!-- ✅ Correcto -->
<h1>Título</h1>
Etiquetas mal anidadas:

html
<!-- ❌ Incorrecto -->
<p><strong>Texto</p></strong>

<!-- ✅ Correcto -->
<p><strong>Texto</strong></p>
Atributos sin comillas:

html
<!-- ❌ Incorrecto -->
<img src=foto.jpg alt=foto>

<!-- ✅ Correcto -->
<img src="foto.jpg" alt="Foto">
───────────────────────────────────────────────────────────

ERRORES DE SEMÁNTICA
───────────────────────────────────────────────────────────

Múltiples <h1>:

html
<!-- ❌ Incorrecto -->
<h1>Título 1</h1>
<h1>Título 2</h1>

<!-- ✅ Correcto -->
<h1>Título 1</h1>
<h2>Título 2</h2>
Saltos de nivel en encabezados:

html
<!-- ❌ Incorrecto -->
<h1>Título</h1>
<h3>Subtítulo</h3>

<!-- ✅ Correcto -->
<h1>Título</h1>
<h2>Subtítulo</h2>
Uso incorrecto de etiquetas semánticas:

html
<!-- ❌ Incorrecto -->
<div class="header">...</div>
<div class="main">...</div>

<!-- ✅ Correcto -->
<header>...</header>
<main>...</main>
───────────────────────────────────────────────────────────

ERRORES DE ENLACES E IMÁGENES
───────────────────────────────────────────────────────────

Enlaces rotos (404):

html
<!-- ❌ Incorrecto -->
<a href="acerca.html">Acerca de</a>
<!-- "acerca.html" no existe -->

<!-- ✅ Correcto -->
<a href="sobre.html">Acerca de</a>
<!-- "sobre.html" existe -->
Imágenes sin alt:

html
<!-- ❌ Incorrecto -->
<img src="logo.png">

<!-- ✅ Correcto -->
<img src="logo.png" alt="Logo de la empresa">
Rutas incorrectas:

html
<!-- ❌ Incorrecto -->
<img src="images/logo.png">
<!-- La imagen está en "img/logo.png" -->

<!-- ✅ Correcto -->
<img src="img/logo.png">
───────────────────────────────────────────────────────────

ERRORES DE FORMULARIOS
───────────────────────────────────────────────────────────

Campos sin label:

html
<!-- ❌ Incorrecto -->
<input type="text" placeholder="Nombre">

<!-- ✅ Correcto -->
<label for="nombre">Nombre:</label>
<input type="text" id="nombre" placeholder="Nombre">
Falta de validación:

html
<!-- ❌ Incorrecto -->
<input type="text" placeholder="Email">

<!-- ✅ Correcto -->
<input type="email" placeholder="Email" required>
Action o method incorrectos:

html
<!-- ❌ Incorrecto -->
<form action="/enviar" method="GET">
<!-- Para datos sensibles debería ser POST -->

<!-- ✅ Correcto -->
<form action="/enviar" method="POST">
───────────────────────────────────────────────────────────

ERRORES DE ACCESIBILIDAD
───────────────────────────────────────────────────────────

Contraste insuficiente:

html
<!-- ❌ Incorrecto -->
<p style="color: #999; background: #fff;">Texto gris claro sobre blanco</p>

<!-- ✅ Correcto -->
<p style="color: #333; background: #fff;">Texto oscuro sobre blanco</p>
Sin skip link:

html
<!-- ❌ Incorrecto -->
<body>
    <header>...</header>
    <main>...</main>
</body>

<!-- ✅ Correcto -->
<body>
    <a href="#main" class="skip-link">Saltar al contenido principal</a>
    <header>...</header>
    <main id="main">...</main>
</body>
Falta de aria-label:

html
<!-- ❌ Incorrecto -->
<button>✕</button>

<!-- ✅ Correcto -->
<button aria-label="Cerrar">✕</button>

PARTE 2: HERRAMIENTAS DE DEPURACIÓN
===============================================================================

───────────────────────────────────────────────────────────

HERRAMIENTAS DE DESARROLLO (DEVTOOLS)
───────────────────────────────────────────────────────────

Cómo abrir DevTools:

Windows/Linux: F12 o Ctrl+Shift+I

Mac: Cmd+Option+I

Pestañas principales:

Pestaña	Función
Elements	Ver y editar HTML/CSS en vivo
Console	Ver errores de JavaScript
Sources	Depurar JavaScript paso a paso
Network	Ver peticiones y recursos
Performance	Medir rendimiento
Lighthouse	Auditar el sitio
Uso práctico:

Abre DevTools (F12).

Ve a la pestaña "Console".

Busca errores en rojo.

Haz clic en el error para ver el archivo y la línea.

Corrige el error.

Recarga la página y verifica.

───────────────────────────────────────────────────────────

VALIDADOR DEL W3C
───────────────────────────────────────────────────────────

URL: https://validator.w3.org/

Cómo usarlo:

Ve a validator.w3.org.

Elige una opción:

"Validate by URI" (URL en línea).

"Validate by File Upload" (subir archivo).

"Validate by Direct Input" (pegar código).

Haz clic en "Check".

Revisa los errores y advertencias.

Corrige los errores.

Vuelve a validar.

Errores comunes detectados:

Etiquetas sin cerrar.

Atributos incorrectos.

Etiquetas obsoletas.

Estructura inválida.

───────────────────────────────────────────────────────────

CONSOLA DEL NAVEGADOR
───────────────────────────────────────────────────────────

Cómo abrir la consola:

DevTools → Pestaña "Console"

Qué buscar:

Errores en rojo (JavaScript, CSS).

Advertencias en amarillo.

Errores 404 (recursos no encontrados).

Mensajes comunes:

text
Uncaught TypeError: ... → Error de tipo en JavaScript.
Uncaught ReferenceError: ... → Variable no definida.
404 Not Found → Recurso no encontrado.
Failed to load resource → Imagen o archivo no cargado.
───────────────────────────────────────────────────────────

PESTAÑA NETWORK
───────────────────────────────────────────────────────────

Cómo usar:

Abre DevTools (F12).

Ve a la pestaña "Network".

Recarga la página (F5).

Verifica que todos los recursos tengan código 200.

Códigos de estado:

200 → OK (todo bien).

301 → Redirigido permanentemente.

404 → No encontrado.

500 → Error del servidor.

Qué buscar:

Archivos con código 404 (no encontrados).

Archivos con código 500 (error del servidor).

Archivos que tardan mucho en cargar.

El proceso de depuración es sistemático y organizado.

───────────────────────────────────────────────────────────
PASO 1: IDENTIFICAR EL PROBLEMA
───────────────────────────────────────────────────────────

¿Qué esperabas que pasara?

¿Qué está pasando realmente?

¿Cuándo ocurre el problema?

¿En qué página o sección?

───────────────────────────────────────────────────────────
PASO 2: AISLAR EL PROBLEMA
───────────────────────────────────────────────────────────

¿El problema ocurre en todas las páginas o solo en una?

¿El problema ocurre en todos los navegadores o solo en uno?

¿El problema ocurre con todos los datos o solo con algunos?

───────────────────────────────────────────────────────────
PASO 3: REPRODUCIR EL PROBLEMA
───────────────────────────────────────────────────────────

¿Puedes reproducir el problema siempre?

¿Hay pasos específicos para que ocurra?

───────────────────────────────────────────────────────────
PASO 4: USAR HERRAMIENTAS
───────────────────────────────────────────────────────────

Abre DevTools (F12).

Ve a la pestaña "Console" para ver errores.

Ve a la pestaña "Network" para ver recursos.

Usa el validador del W3C.

───────────────────────────────────────────────────────────
PASO 5: CORREGIR EL PROBLEMA
───────────────────────────────────────────────────────────

Identifica la causa raíz.

Corrige el error.

Prueba que funciona.

───────────────────────────────────────────────────────────
PASO 6: VERIFICAR
───────────────────────────────────────────────────────────

Recarga la página.

Prueba que el problema se haya solucionado.

Verifica que no se hayan introducido nuevos errores.

───────────────────────────────────────────────────────────

ERRORES DE SINTAXIS
───────────────────────────────────────────────────────────

javascript
// ❌ Incorrecto
const nombre = 'Ana' // Falta punto y coma

// ✅ Correcto
const nombre = 'Ana';

EVENTOS SIN FUNCIÓN
───────────────────────────────────────────────────────────

javascript

button.onclick = saludar; // "saludar" no está definida

// ✅ Correcto
function saludar() {
    alert('Hola');
}
button.onclick = saludar;

PARTE 7: EJEMPLO PRÁCTICO – DEPURACIÓN DE UN SITIO
===============================================================================

Imagina que tienes un sitio con los siguientes problemas:

───────────────────────────────────────────────────────────
PROBLEMA 1: IMAGEN QUE NO CARGA
───────────────────────────────────────────────────────────

Síntoma: Una imagen no se muestra en la página.

Depuración:

Abre DevTools (F12).

Ve a la pestaña "Network".

Recarga la página.

Busca la imagen en la lista.

Verifica el código de estado (404 = no encontrada).

Causa: La ruta de la imagen es incorrecta.

Solución:

html
<!-- ❌ Incorrecto -->
<img src="images/logo.png">

<!-- ✅ Correcto (verificar la ruta real) -->
<img src="img/logo.png">
───────────────────────────────────────────────────────────
PROBLEMA 2: BOTÓN QUE NO FUNCIONA
───────────────────────────────────────────────────────────

Síntoma: Un botón no responde al hacer clic.

Depuración:

Abre DevTools (F12).

Ve a la pestaña "Console".

Busca errores en rojo.

El error dice: "Uncaught ReferenceError: saludar is not defined".

Causa: La función "saludar" no está definida.

Solución:

javascript
// ❌ Incorrecto
button.onclick = saludar; // "saludar" no existe

// ✅ Correcto
function saludar() {
    alert('Hola');
}
button.onclick = saludar;
───────────────────────────────────────────────────────────
PROBLEMA 3: FORMULARIO QUE NO VALIDA
───────────────────────────────────────────────────────────

Síntoma: El formulario se envía aunque los campos estén vacíos.

Depuración:

Revisa el código HTML del formulario.

Verifica que los campos tengan required.

Causa: Faltan los atributos required.

Solución:

html
<!-- ❌ Incorrecto -->
<input type="text" placeholder="Nombre">

<!-- ✅ Correcto -->
<input type="text" placeholder="Nombre" required>
───────────────────────────────────────────────────────────
PROBLEMA 4: ESTILOS QUE NO SE APLICAN
───────────────────────────────────────────────────────────

Síntoma: Un elemento no tiene el estilo esperado.

Depuración:

Abre DevTools (F12).

Ve a la pestaña "Elements".

Selecciona el elemento.

En la parte derecha, ve los estilos aplicados.

Busca el estilo que esperas ver.

Si no aparece, el selector puede ser incorrecto.

Causa: El selector CSS no coincide con el elemento.

Solución:

css
/* ❌ Incorrecto */
.heroe { color: blue; } /* La clase es "hero" */

/* ✅ Correcto */
.hero { color: blue; }
