PARTE 1: EL ELEMENTO <input> – REPASO RÁPIDO
===============================================================================

<input> es la etiqueta más utilizada en formularios.
Es una etiqueta VACÍA (no tiene cierre).

───────────────────────────────────────────────────────────
SINTAXIS GENERAL
───────────────────────────────────────────────────────────

<input type="tipo" name="nombre" id="id" atributos>

───────────────────────────────────────────────────────────
ATRIBUTOS COMUNES A TODOS LOS INPUTS
───────────────────────────────────────────────────────────

- id → Identificador único (para asociar con label).
- name → Nombre del campo (se envía al servidor).
- value → Valor por defecto.
- placeholder → Texto de ejemplo (no todos los tipos lo soportan).
- required → Campo obligatorio.
- disabled → Campo deshabilitado.
- readonly → Campo de solo lectura.
- autofocus → Recibe el foco al cargar.
- autocomplete → Sugerencias automáticas (on/off).
- aria-* → Atributos de accesibilidad.

PARTE 2: INPUTS DE TEXTO (type="text", type="search", type="url", type="email", type="tel", type="password")
===============================================================================

───────────────────────────────────────────────────────────
type="text" – TEXTO CORTO (EL MÁS COMÚN)
───────────────────────────────────────────────────────────

<input type="text" id="nombre" name="nombre" placeholder="Ej: Ana Pérez" required>

Usos:
- Nombres, apellidos.
- Direcciones.
- Palabras clave.
- Texto corto.

───────────────────────────────────────────────────────────
type="password" – CONTRASEÑA (OCULTA EL TEXTO)
───────────────────────────────────────────────────────────

<input type="password" id="clave" name="clave" placeholder="Mínimo 8 caracteres" required minlength="8">

Usos:
- Contraseñas.
- Claves de acceso.
- Códigos PIN.

───────────────────────────────────────────────────────────
type="email" – CORREO ELECTRÓNICO (CON VALIDACIÓN)
───────────────────────────────────────────────────────────

<input type="email" id="email" name="email" placeholder="tu@email.com" required>

Usos:
- Correos electrónicos.
- Registros de usuarios.
- Recuperación de contraseñas.

Validación: El navegador verifica que el formato sea válido (usuario@dominio.com).

───────────────────────────────────────────────────────────
type="tel" – TELÉFONO (TECLADO NUMÉRICO EN MÓVILES)
───────────────────────────────────────────────────────────

<input type="tel" id="telefono" name="telefono" placeholder="55 1234 5678">

Usos:
- Números de teléfono.
- Móviles o fijos.

No valida formato automáticamente. Puedes usar pattern para validar.

───────────────────────────────────────────────────────────
type="url" – URL (CON VALIDACIÓN)
───────────────────────────────────────────────────────────

<input type="url" id="sitio" name="sitio" placeholder="https://ejemplo.com">

Usos:
- Enlaces a sitios web.
- Perfiles de redes sociales.
- Portafolios.

Validación: El navegador verifica que el formato sea una URL válida.

───────────────────────────────────────────────────────────
type="search" – BÚSQUEDA (CON BOTÓN DE LIMPIAR)
───────────────────────────────────────────────────────────

<input type="search" id="buscar" name="buscar" placeholder="Buscar...">

Usos:
- Barras de búsqueda.
- Filtros de contenido.
- Encontrar elementos en una página.

En algunos navegadores, muestra una "X" para borrar el contenido.

PARTE 3: INPUTS NUMÉRICOS (type="number", type="range")
===============================================================================

───────────────────────────────────────────────────────────
type="number" – NÚMERO (CON FLECHAS DE INCREMENTO)
───────────────────────────────────────────────────────────

<input type="number" id="edad" name="edad" min="18" max="99" step="1" value="25">

Usos:
- Edades.
- Cantidades.
- Calificaciones.
- Precios.
- Años.

Atributos:
- min → Valor mínimo.
- max → Valor máximo.
- step → Incremento (1, 0.5, 0.01, etc.).
- value → Valor por defecto.

───────────────────────────────────────────────────────────
type="range" – RANGO DESLIZANTE (SLIDER)
───────────────────────────────────────────────────────────

<input type="range" id="volumen" name="volumen" min="0" max="100" step="5" value="50">

Usos:
- Volumen de audio.
- Nivel de brillo.
- Control de zoom.
- Satisfacción del usuario (1-10).
- Cualquier cosa que se controle deslizando.

Atributos:
- min → Valor mínimo (default 0).
- max → Valor máximo (default 100).
- step → Incremento (default 1).
- value → Valor por defecto.

No muestra el valor actual; puedes mostrarlo con JavaScript o con un <output>.

PARTE 4: INPUTS DE FECHA Y HORA (type="date", type="time", type="datetime-local", type="month", type="week")
===============================================================================

───────────────────────────────────────────────────────────
type="date" – FECHA (CALENDARIO)
───────────────────────────────────────────────────────────

<input type="date" id="nacimiento" name="nacimiento" min="1900-01-01" max="2026-12-31" value="2000-01-01">

Usos:
- Fechas de nacimiento.
- Fechas de eventos.
- Plazos de entrega.
- Reservas.

Formato: YYYY-MM-DD (ej: 2000-01-01).
Solo fecha, sin hora.

───────────────────────────────────────────────────────────
type="time" – HORA
───────────────────────────────────────────────────────────

<input type="time" id="hora" name="hora" min="08:00" max="18:00" step="900" value="12:00">

Usos:
- Horarios de citas.
- Horas de apertura.
- Tiempos de duración.

Formato: HH:MM (ej: 14:30) o HH:MM:SS.
No incluye fecha.

───────────────────────────────────────────────────────────
type="datetime-local" – FECHA Y HORA (LOCAL)
───────────────────────────────────────────────────────────

<input type="datetime-local" id="cita" name="cita" min="2026-01-01T08:00" max="2026-12-31T18:00" value="2026-06-25T10:30">

Usos:
- Eventos con fecha y hora exacta.
- Citas programadas.
- Lanzamientos.

Formato: YYYY-MM-DDTHH:MM (ej: 2026-06-25T10:30).
No incluye la zona horaria.

───────────────────────────────────────────────────────────
type="month" – MES Y AÑO
───────────────────────────────────────────────────────────

<input type="month" id="mes" name="mes" min="2024-01" max="2027-12" value="2026-06">

Usos:
- Períodos de facturación.
- Programación de tareas mensuales.
- Pagos recurrentes.

Formato: YYYY-MM (ej: 2026-06).

───────────────────────────────────────────────────────────
type="week" – SEMANA DEL AÑO
───────────────────────────────────────────────────────────

<input type="week" id="semana" name="semana" min="2026-W01" max="2026-W52" value="2026-W25">

Usos:
- Planificación semanal.
- Proyectos de duración semanal.
- Asignación de tareas.

Formato: YYYY-WNN (ej: 2026-W25, donde W25 es la semana 25 del año).

===============================================================================
PARTE 5: INPUTS DE SELECCIÓN (type="checkbox", type="radio")
===============================================================================

───────────────────────────────────────────────────────────
type="checkbox" – CASILLA DE VERIFICACIÓN (MÚLTIPLE SELECCIÓN)
───────────────────────────────────────────────────────────

<input type="checkbox" id="terminos" name="terminos" required>
<label for="terminos">Acepto los términos y condiciones</label>

<input type="checkbox" id="interes1" name="interes" value="deporte">
<label for="interes1">Deporte</label>

<input type="checkbox" id="interes2" name="interes" value="musica">
<label for="interes2">Música</label>

<input type="checkbox" id="interes3" name="interes" value="lectura">
<label for="interes3">Lectura</label>

Usos:
- Aceptación de términos.
- Selección de intereses.
- Preferencias múltiples.
- Confirmación de acciones.

Características:
- Se pueden seleccionar múltiples opciones.
- Cada checkbox es independiente.
- El atributo checked lo marca por defecto.
- El value se envía si está marcado.

───────────────────────────────────────────────────────────
type="radio" – BOTÓN RADIAL (SELECCIÓN ÚNICA)
───────────────────────────────────────────────────────────

<label>
    <input type="radio" name="genero" value="masculino" checked>
    Masculino
</label>
<label>
    <input type="radio" name="genero" value="femenino">
    Femenino
</label>
<label>
    <input type="radio" name="genero" value="otro">
    Otro
</label>

Usos:
- Selección de género.
- Elección de método de pago.
- Selección de talla.
- Opciones mutuamente excluyentes.

Características:
- Solo se puede seleccionar UNO por grupo.
- El grupo se define por el mismo name.
- El atributo checked marca la opción por defecto.
- Siempre debe tener un valor (value).

===============================================================================
PARTE 6: INPUTS ESPECIALES (type="color", type="file", type="hidden")
===============================================================================

───────────────────────────────────────────────────────────
type="color" – SELECTOR DE COLOR (PALETA)
───────────────────────────────────────────────────────────

<input type="color" id="color" name="color" value="#ff0000">

Usos:
- Elección de colores.
- Personalización de temas.
- Diseño de interfaces.

Formato: #rrggbb (hexadecimal, ej: #ff0000 para rojo).
Value por defecto: #000000 (negro).

───────────────────────────────────────────────────────────
type="file" – SUBIDA DE ARCHIVOS
───────────────────────────────────────────────────────────

<input type="file" id="archivo" name="archivo" accept=".pdf,.docx,image/*" multiple>

Usos:
- Subir documentos.
- Cargar imágenes.
- Enviar archivos adjuntos.
- Compartir documentos.

Atributos:
- accept → Tipos de archivo permitidos (ej: image/*, .pdf, .docx).
- multiple → Permite seleccionar varios archivos.
- required → Obligatorio.

Nota: Para subir archivos, el formulario debe tener:
<form action="/upload" method="POST" enctype="multipart/form-data">

───────────────────────────────────────────────────────────
type="hidden" – CAMPO OCULTO (NO VISIBLE PARA EL USUARIO)
───────────────────────────────────────────────────────────

<input type="hidden" id="usuario_id" name="usuario_id" value="12345">

Usos:
- Enviar datos sin que el usuario los vea.
- Identificadores de sesión.
- Datos de configuración.
- Información de estado.

Características:
- No se muestra en la página.
- Se envía al servidor.
- No se puede modificar (a menos que uses JavaScript).

PARTE 7: INPUTS MODERNOS Y AVANZADOS (type="image", type="button", type="reset", type="submit")
===============================================================================

───────────────────────────────────────────────────────────
type="image" – BOTÓN CON IMAGEN (ENVÍA EL FORMULARIO)
───────────────────────────────────────────────────────────

<input type="image" src="boton-enviar.png" alt="Enviar" name="enviar">

Usos:
- Botones de envío personalizados con imágenes.
- Mapas de imagen.
- Botones con iconos.

Funciona como submit, pero usa una imagen en lugar de texto.

───────────────────────────────────────────────────────────
type="button" – BOTÓN GENÉRICO (SIN ACCIÓN PREDETERMINADA)
───────────────────────────────────────────────────────────

<input type="button" id="btn" value="Clic aquí" onclick="alert('Hola')">

Usos:
- Botones con funcionalidad JavaScript.
- Acciones personalizadas.
- Disparadores de eventos.

No envía el formulario a menos que se programe con JavaScript.

───────────────────────────────────────────────────────────
type="reset" – BOTÓN DE REINICIO (LIMPIAR EL FORMULARIO)
───────────────────────────────────────────────────────────

<input type="reset" value="Limpiar">

Usos:
- Restablecer todos los campos a sus valores iniciales.
- Botones de limpieza.

───────────────────────────────────────────────────────────
type="submit" – BOTÓN DE ENVÍO (ENVÍA EL FORMULARIO)
───────────────────────────────────────────────────────────

<input type="submit" value="Enviar">

Usos:
- Enviar el formulario al servidor.
- Acción principal del formulario.

PARTE 8: VALIDACIONES HTML5 (ATRIBUTOS DE VALIDACIÓN)
===============================================================================

HTML5 incluye validaciones nativas sin necesidad de JavaScript.

───────────────────────────────────────────────────────────
required → CAMPO OBLIGATORIO
───────────────────────────────────────────────────────────

<input type="text" name="nombre" required>

El campo no puede estar vacío.

───────────────────────────────────────────────────────────
minlength y maxlength → LONGITUD MÍNIMA Y MÁXIMA
───────────────────────────────────────────────────────────

<input type="text" name="usuario" minlength="3" maxlength="20">
<input type="password" name="clave" minlength="8">

El texto debe tener entre minlength y maxlength caracteres.

───────────────────────────────────────────────────────────
pattern → EXPRESIÓN REGULAR (VALIDACIÓN CUSTOM)
───────────────────────────────────────────────────────────

<input type="text" name="codigo" pattern="[A-Z]{3}-[0-9]{4}" placeholder="ABC-1234">

Usa una expresión regular para validar el formato.

Ejemplos:
- Código postal: pattern="[0-9]{5}"
- Teléfono: pattern="[0-9]{10}"
- Letras mayúsculas: pattern="[A-Z]+"
- Email (mejor usar type="email")

───────────────────────────────────────────────────────────
min y max → VALORES MÍNIMO Y MÁXIMO (numérico, fecha)
───────────────────────────────────────────────────────────

<input type="number" name="edad" min="18" max="99">
<input type="date" name="fecha" min="2000-01-01" max="2026-12-31">

El valor debe estar dentro del rango.

───────────────────────────────────────────────────────────
step → INCREMENTO (numérico, fecha)
───────────────────────────────────────────────────────────

<input type="number" name="cantidad" step="5">
<input type="range" name="volumen" step="2">

Los valores deben ser múltiplos del step.

───────────────────────────────────────────────────────────
VALIDACIÓN DE CORREO (type="email")
───────────────────────────────────────────────────────────

<input type="email" name="email">

Valida que el formato sea usuario@dominio.com.

───────────────────────────────────────────────────────────
VALIDACIÓN DE URL (type="url")
───────────────────────────────────────────────────────────

<input type="url" name="sitio">

Valida que sea una URL válida.

───────────────────────────────────────────────────────────
MENSAJES DE ERROR PERSONALIZADOS
───────────────────────────────────────────────────────────

Puedes personalizar los mensajes de error con JavaScript:

<input type="text" id="nombre" required
       oninvalid="this.setCustomValidity('El nombre es obligatorio')"
       oninput="this.setCustomValidity('')">

PARTE 9: ATRIBUTOS AVANZADOS DE INPUTS
===============================================================================

───────────────────────────────────────────────────────────
autocomplete → SUGERENCIAS AUTOMÁTICAS
───────────────────────────────────────────────────────────

<input type="text" name="nombre" autocomplete="name">
<input type="email" name="email" autocomplete="email">
<input type="tel" name="telefono" autocomplete="tel">

Valores comunes:
- name → Nombre completo.
- given-name → Nombre de pila.
- family-name → Apellido.
- email → Correo electrónico.
- tel → Teléfono.
- address → Dirección.
- username → Nombre de usuario.
- current-password → Contraseña actual.
- new-password → Nueva contraseña.
- off → Deshabilitar autocompletado.

───────────────────────────────────────────────────────────
autofocus → FOCUS AUTOMÁTICO
───────────────────────────────────────────────────────────

<input type="text" name="nombre" autofocus>

El campo recibe el foco al cargar la página.

───────────────────────────────────────────────────────────
readonly → SOLO LECTURA (NO SE PUEDE MODIFICAR)
───────────────────────────────────────────────────────────

<input type="text" name="codigo" value="ABC-123" readonly>

El campo no se puede modificar, pero se envía al servidor.

───────────────────────────────────────────────────────────
disabled → DESHABILITADO (NO SE PUEDE USAR, NO SE ENVÍA)
───────────────────────────────────────────────────────────

<input type="text" name="codigo" value="ABC-123" disabled>

El campo está gris, no se puede modificar y NO se envía.

───────────────────────────────────────────────────────────
placeholder → TEXTO DE EJEMPLO
───────────────────────────────────────────────────────────

<input type="text" name="nombre" placeholder="Ej: Ana Pérez">

Texto de ejemplo dentro del campo, desaparece al escribir.

───────────────────────────────────────────────────────────
multiple → SELECCIÓN MÚLTIPLE (archivos, email)
───────────────────────────────────────────────────────────

<input type="file" name="archivos" multiple>
<input type="email" name="emails" multiple>

Permite seleccionar varios archivos o ingresar varios correos separados por coma.

───────────────────────────────────────────────────────────
size → TAMAÑO VISIBLE (para selects y text inputs)
───────────────────────────────────────────────────────────

<input type="text" name="codigo" size="10">

Ancho del campo en caracteres (no confundir con maxlength).

PARTE 13: ERRORES COMUNES CON INPUTS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: USAR EL TIPO INCORRECTO
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" name="email">

Consecuencia: No hay validación de correo.

Solución: Usa <input type="email" name="email">

───────────────────────────────────────────────────────────
ERROR 2: OLVIDAR EL ATRIBUTO name
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" id="nombre">

Consecuencia: El campo no se envía al servidor.

Solución: Añade name="nombre"

───────────────────────────────────────────────────────────
ERROR 3: NO ASOCIAR LABEL CON INPUT (accesibilidad)
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" id="nombre">

Solución: Añade <label for="nombre">Nombre:</label>

───────────────────────────────────────────────────────────
ERROR 4: USAR PATTERN INCORRECTO
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" pattern="[A-Z]{3}" placeholder="3 letras mayúsculas">

El usuario puede escribir 3 letras mayúsculas, pero no hay validación si no se llena el campo (a menos que se use required).

Solución: Añade required para que sea obligatorio.

───────────────────────────────────────────────────────────
ERROR 5: NO INCLUIR enctype PARA ARCHIVOS
───────────────────────────────────────────────────────────

Incorrecto:
<form action="/upload" method="POST">
    <input type="file" name="archivo">
</form>

Consecuencia: El archivo no se sube correctamente.

Solución: Añade enctype="multipart/form-data"

───────────────────────────────────────────────────────────
ERROR 6: NO USAR EL TIPO CORRECTO PARA FECHAS
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" name="fecha" placeholder="DD/MM/YYYY">

Consecuencia: El usuario puede ingresar cualquier formato.

Solución: Usa <input type="date" name="fecha">

===============================================================================
PARTE 14: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre elige el tipo de input más específico para cada dato.
2. Siempre añade name a todos los inputs.
3. Siempre asocia label con input usando for/id.
4. Usa pattern con required para validar formatos.
5. Para subir archivos, siempre añade enctype="multipart/form-data".
6. Usa type="date" y type="time" para fechas y horas.
7. Prueba cada input en diferentes navegadores.
8. Usa las herramientas de desarrollador para depurar.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- Input: Campo de entrada de datos en un formulario.
- type: Atributo que define el tipo de input.
- placeholder: Texto de ejemplo dentro del campo.
- required: Campo obligatorio.
- pattern: Expresión regular para validar el formato.
- min: Valor mínimo (numérico, fecha).
- max: Valor máximo (numérico, fecha).
- step: Incremento.
- autocomplete: Sugerencias automáticas.
- autofocus: Foco automático al cargar.
- readonly: Solo lectura.
- disabled: Deshabilitado.
- multiple: Selección múltiple (archivos, email).
- accept: Tipos de archivo permitidos.
- enctype: Tipo de codificación para archivos (multipart/form-data).
