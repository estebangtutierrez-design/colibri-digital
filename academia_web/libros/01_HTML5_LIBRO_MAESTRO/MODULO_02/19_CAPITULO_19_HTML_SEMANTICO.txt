PARTE 1: ¿QUÉ ES LA SEMÁNTICA EN HTML?
===============================================================================

La semántica es el estudio del significado.

En HTML, la semántica se refiere al significado de las etiquetas.

───────────────────────────────────────────────────────────
EJEMPLO DE CÓDIGO NO SEMÁNTICO
───────────────────────────────────────────────────────────

<div id="header">Logo y título</div>
<div id="nav">Menú de navegación</div>
<div id="content">Contenido principal</div>
<div id="sidebar">Barra lateral</div>
<div id="footer">Pie de página</div>

Este código funciona.
Pero no comunica.
Solo dice: "aquí hay un contenedor".
No dice qué contiene.

───────────────────────────────────────────────────────────
EJEMPLO DE CÓDIGO SEMÁNTICO
───────────────────────────────────────────────────────────

<header>Logo y título</header>
<nav>Menú de navegación</nav>
<main>Contenido principal</main>
<aside>Barra lateral</aside>
<footer>Pie de página</footer>

Este código también funciona.
Pero además comunica.
Dice exactamente qué contiene cada parte.

───────────────────────────────────────────────────────────
¿POR QUÉ ES IMPORTANTE LA SEMÁNTICA?
───────────────────────────────────────────────────────────

1. SEO: Google entiende mejor la estructura de tu página.
2. Accesibilidad: Los lectores de pantalla navegan más fácilmente.
3. Mantenimiento: El código es más legible para otros programadores.
4. Estándares: Sigue las buenas prácticas de la web moderna.

===============================================================================
PARTE 2: ETIQUETAS SEMÁNTICAS DE HTML5 (VISIÓN GENERAL)
===============================================================================

HTML5 introdujo un conjunto de etiquetas semánticas que describen la estructura de una página.

+----------------------------------------------------------+
|  <header>   |  Cabecera de la página o sección.           |
|  <nav>      |  Menú de navegación.                        |
|  <main>     |  Contenido principal (único por página).    |
|  <section>  |  Agrupación temática de contenido.          |
|  <article>  |  Contenido independiente (post, noticia).   |
|  <aside>    |  Contenido complementario (barra lateral).  |
|  <footer>   |  Pie de página o sección.                   |
+----------------------------------------------------------+

Cada una de estas etiquetas tiene un propósito específico.
Vamos a explorarlas una por una.

===============================================================================
PARTE 3: <header> – LA CABECERA
===============================================================================

<header> representa la cabecera de una página o sección.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- Logo de la página.
- Título del sitio.
- Eslogan o descripción.
- Navegación principal (nav).
- Elementos introductorios.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<header>
    <h1>Lucas EGA Academy</h1>
    <p>Aprende programación con el Profesor Búho AI</p>
    <nav>
        <a href="#inicio">Inicio</a>
        <a href="#cursos">Cursos</a>
        <a href="#contacto">Contacto</a>
    </nav>
</header>

───────────────────────────────────────────────────────────
REGLA DE ORO
───────────────────────────────────────────────────────────

Puede haber MÚLTIPLES <header> en una página.
Uno para la cabecera principal.
Y otros para cabeceras de secciones o artículos.

===============================================================================
PARTE 4: <nav> – NAVEGACIÓN
===============================================================================

<nav> representa un conjunto de enlaces de navegación.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- Menú principal del sitio.
- Enlaces a secciones importantes.
- Enlaces a páginas clave.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<nav aria-label="Navegación principal">
    <ul>
        <li><a href="index.html">Inicio</a></li>
        <li><a href="servicios.html">Servicios</a></li>
        <li><a href="blog.html">Blog</a></li>
        <li><a href="contacto.html">Contacto</a></li>
    </ul>
</nav>

───────────────────────────────────────────────────────────
REGLA DE ORO
───────────────────────────────────────────────────────────

No uses <nav> para todos los enlaces.
Solo para los bloques de navegación PRINCIPALES.
Los enlaces en el footer (legales) no necesitan nav.

===============================================================================
PARTE 5: <main> – CONTENIDO PRINCIPAL
===============================================================================

<main> representa el contenido principal y único de la página.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- El contenido específico de la página.
- Artículos, secciones, formularios.
- TODO lo que NO se repite en otras páginas.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<main>
    <h1>Bienvenido a mi portafolio</h1>
    <p>Aquí encontrarás mis proyectos y experiencia.</p>

    <section id="proyectos">
        <h2>Proyectos</h2>
        <article>...</article>
        <article>...</article>
    </section>

    <section id="contacto">
        <h2>Contacto</h2>
        <form>...</form>
    </section>
</main>

───────────────────────────────────────────────────────────
REGLA DE ORO
───────────────────────────────────────────────────────────

Solo debe haber UN <main> por página.
Es el contenido principal, no se repite.

===============================================================================
PARTE 6: <section> – SECCIÓN TEMÁTICA
===============================================================================

<section> agrupa contenido que comparte un tema común.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- Un grupo de contenido temáticamente relacionado.
- Siempre debe tener un encabezado (h1-h6).

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<section id="servicios">
    <h2>Nuestros servicios</h2>
    <p>Ofrecemos desarrollo web, diseño y consultoría.</p>
</section>

<section id="testimonios">
    <h2>Testimonios</h2>
    <blockquote>...</blockquote>
    <blockquote>...</blockquote>
</section>

───────────────────────────────────────────────────────────
DIFERENCIA ENTRE <section> Y <div>
───────────────────────────────────────────────────────────

<section> → Tiene significado semántico (tema).
<div> → No tiene significado (solo agrupación visual).

Usa <section> cuando el contenido tenga un tema.
Usa <div> cuando solo necesites un contenedor para CSS o JavaScript.

===============================================================================
PARTE 7: <article> – CONTENIDO INDEPENDIENTE
===============================================================================

<article> representa contenido que es independiente y autocontenido.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- Un post de blog.
- Una noticia.
- Un comentario.
- Un producto.
- Cualquier contenido que pueda leerse por separado.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<article>
    <header>
        <h2>Cómo aprender HTML en 10 días</h2>
        <p>Publicado el 25 de junio de 2026</p>
    </header>
    <p>Contenido del artículo...</p>
    <footer>
        <p>Autor: Esteban</p>
    </footer>
</article>

<article>
    <header>
        <h2>Los 10 errores más comunes en HTML</h2>
        <p>Publicado el 20 de junio de 2026</p>
    </header>
    <p>Contenido del artículo...</p>
</article>

───────────────────────────────────────────────────────────
DIFERENCIA ENTRE <section> Y <article>
───────────────────────────────────────────────────────────

<article> → Contenido independiente (puede leerse solo).
<section> → Contenido temático (parte de un todo).

Ejemplo:
- Un blog tiene varios <article> (cada post).
- Dentro de un <article>, puedes tener <section> (introducción, desarrollo, conclusión).

===============================================================================
PARTE 8: <aside> – CONTENIDO COMPLEMENTARIO
===============================================================================

<aside> representa contenido complementario al contenido principal.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- Barras laterales.
- Publicidad.
- Enlaces relacionados.
- Biografía del autor.
- Notas al margen.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<aside>
    <h3>Artículos relacionados</h3>
    <ul>
        <li><a href="#">HTML semántico</a></li>
        <li><a href="#">CSS moderno</a></li>
        <li><a href="#">JavaScript básico</a></li>
    </ul>
</aside>

<aside>
    <h3>Sobre el autor</h3>
    <p>Esteban es desarrollador web y fundador de Lucas EGA Academy.</p>
</aside>

───────────────────────────────────────────────────────────
REGLA DE ORO
───────────────────────────────────────────────────────────

El contenido de <aside> debe estar RELACIONADO con el contenido principal, pero no ser esencial.
Si quitas el aside, el contenido principal sigue teniendo sentido.

===============================================================================
PARTE 9: <footer> – PIE DE PÁGINA
===============================================================================

<footer> representa el pie de página de una página o sección.

───────────────────────────────────────────────────────────
¿QUÉ CONTIENE?
───────────────────────────────────────────────────────────

- Derechos de autor (copyright).
- Información de contacto.
- Enlaces legales.
- Mapa del sitio.
- Redes sociales.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<footer>
    <p>&copy; 2026 Lucas EGA Academy. Todos los derechos reservados.</p>
    <nav aria-label="Enlaces legales">
        <a href="#">Aviso legal</a> |
        <a href="#">Política de privacidad</a> |
        <a href="#">Política de cookies</a>
    </nav>
    <div>
        <a href="#">YouTube</a> |
        <a href="#">Twitter</a> |
        <a href="#">GitHub</a>
    </div>
</footer>

───────────────────────────────────────────────────────────
REGLA DE ORO
───────────────────────────────────────────────────────────

Puede haber MÚLTIPLES <footer>.
Uno para el pie de página principal.
Y otros para pies de artículos o secciones.

PARTE 13: SEMÁNTICA Y SEO
===============================================================================

Google utiliza la estructura semántica para entender tu página.

───────────────────────────────────────────────────────────
CÓMO GOOGLE INTERPRETA LAS ETIQUETAS SEMÁNTICAS
───────────────────────────────────────────────────────────

<header> → "Aquí está la cabecera del sitio."
<nav> → "Aquí está la navegación principal."
<main> → "Aquí está el contenido más importante."
<section> → "Aquí hay un tema."
<article> → "Aquí hay contenido independiente."
<aside> → "Aquí hay contenido complementario."
<footer> → "Aquí está el pie de página."

───────────────────────────────────────────────────────────
BENEFICIOS PARA EL SEO
───────────────────────────────────────────────────────────

1. Google indexa mejor el contenido principal.
2. Google entiende la jerarquía de tu contenido.
3. Los fragmentos enriquecidos (rich snippets) mejoran el CTR.
4. Mejora la experiencia de usuario, lo que Google premia.

===============================================================================
PARTE 14: SEMÁNTICA Y ACCESIBILIDAD
===============================================================================

Las etiquetas semánticas son fundamentales para la accesibilidad.

───────────────────────────────────────────────────────────
CÓMO AYUDAN A LAS PERSONAS CON DISCAPACIDAD
───────────────────────────────────────────────────────────

Los lectores de pantalla (como NVDA o VoiceOver) utilizan las etiquetas semánticas para navegar.

- Los usuarios pueden saltar directamente al <main>.
- Los usuarios pueden navegar por los <nav>.
- Los usuarios pueden entender la estructura de la página.

───────────────────────────────────────────────────────────
EJEMPLO DE NAVEGACIÓN CON LECTOR DE PANTALLA
───────────────────────────────────────────────────────────

Un usuario con discapacidad visual puede:
1. Presionar una tecla para ir al <main> (saltarse el header y nav).
2. Presionar otra tecla para ir al siguiente <section>.
3. Escuchar la estructura de la página.

Sin etiquetas semánticas, todo el contenido es un bloque sin estructura.

PARTE 15: BUENAS PRÁCTICAS CON SEMÁNTICA
===============================================================================

───────────────────────────────────────────────────────────
1. USA LA ETIQUETA CORRECTA PARA CADA PROPÓSITO
───────────────────────────────────────────────────────────

- Cabecera → <header>
- Navegación → <nav>
- Contenido principal → <main>
- Secciones temáticas → <section>
- Contenido independiente → <article>
- Contenido complementario → <aside>
- Pie de página → <footer>

───────────────────────────────────────────────────────────
2. NO USES <div> CUANDO PUEDAS USAR UNA ETIQUETA SEMÁNTICA
───────────────────────────────────────────────────────────

❌ Malo:
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>

✅ Bueno:
<header>...</header>
<nav>...</nav>
<main>...</main>

───────────────────────────────────────────────────────────
3. USA <section> CON ENCABEZADO
───────────────────────────────────────────────────────────

Cada <section> debe tener un encabezado (h2-h6).

❌ Malo:
<section>
    <p>Contenido sin título</p>
</section>

✅ Bueno:
<section>
    <h2>Título de la sección</h2>
    <p>Contenido...</p>
</section>

───────────────────────────────────────────────────────────
4. USA <article> PARA CONTENIDO AUTOCONTENIDO
───────────────────────────────────────────────────────────

Si el contenido puede leerse por separado, es un <article>.

───────────────────────────────────────────────────────────
5. USA <aside> SOLO PARA CONTENIDO COMPLEMENTARIO
───────────────────────────────────────────────────────────

El <aside> debe estar relacionado con el contenido principal, pero no ser esencial.

===============================================================================
PARTE 16: ERRORES COMUNES CON SEMÁNTICA
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: USAR <section> COMO CONTENEDOR GENÉRICO
───────────────────────────────────────────────────────────

Incorrecto:
<section style="color: red;">Contenido sin tema</section>

Consecuencia: La sección no tiene significado semántico.

Solución: Usa <div> para contenedores genéricos.

───────────────────────────────────────────────────────────
ERROR 2: NO USAR <main> O TENER MÚLTIPLES
───────────────────────────────────────────────────────────

Incorrecto:
<body>
    <main>Contenido 1</main>
    <main>Contenido 2</main>
</body>

Consecuencia: Confunde a los lectores de pantalla.

Solución: Solo UN <main> por página.

───────────────────────────────────────────────────────────
ERROR 3: USAR <article> PARA CONTENIDO QUE NO ES INDEPENDIENTE
───────────────────────────────────────────────────────────

Incorrecto:
<article>
    <p>Un párrafo dentro de un artículo sin sentido</p>
</article>

Consecuencia: El <article> pierde su propósito.

Solución: Usa <section> para contenido temático que no es independiente.

───────────────────────────────────────────────────────────
ERROR 4: USAR <aside> PARA CONTENIDO NO RELACIONADO
───────────────────────────────────────────────────────────

Incorrecto:
<aside>
    <p>Información completamente ajena al contenido principal</p>
</aside>

Consecuencia: El aside no tiene relación con el contenido principal.

Solución: El aside debe estar relacionado (aunque sea tangencialmente).

───────────────────────────────────────────────────────────
ERROR 5: OLVIDAR ENCABEZADOS DENTRO DE <section>
───────────────────────────────────────────────────────────

Incorrecto:
<section>
    <p>Contenido de la sección</p>
</section>

Consecuencia: La sección no tiene un tema declarado.

Solución: Añade un encabezado (h2-h6).

===============================================================================
PARTE 17: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Usa <section> solo cuando el contenido tenga un tema.
2. Usa <div> para contenedores genéricos sin significado.
3. Siempre usa UN <main> por página.
4. Usa <article> para contenido independiente (post, noticia, producto).
5. Usa <aside> para contenido complementario relacionado.
6. Siempre añade encabezados a las <section>.
7. Prueba tu página con un lector de pantalla para verificar la semántica.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- Semántica: Estudio del significado. En HTML, etiquetas que describen su propósito.
- header: Cabecera de página o sección.
- nav: Navegación principal.
- main: Contenido principal (único por página).
- section: Sección temática de contenido.
- article: Contenido independiente y autocontenido.
- aside: Contenido complementario.
- footer: Pie de página o sección.
- SEO: Search Engine Optimization (optimización para motores de búsqueda).
- Accesibilidad: Capacidad de una página para ser usada por personas con discapacidad.
- Lector de pantalla: Software que lee el contenido de una página en voz alta.

¿SABÍAS QUE...?
===============================================================================

Antes de HTML5 (1991-2014), los desarrolladores usaban <div> para todo.
La navegación era <div id="nav">.
El contenido principal era <div id="content">.
El footer era <div id="footer">.

HTML5 simplificó todo al introducir etiquetas semánticas.
Hoy, el código es más limpio, más legible y más accesible.
