PARTE 1: LA ESTRUCTURA BÁSICA DE UNA TABLA
===============================================================================

Una tabla HTML se construye con tres elementos principales:

───────────────────────────────────────────────────────────
<table> → EL CONTENEDOR
───────────────────────────────────────────────────────────

La etiqueta <table> envuelve toda la tabla.

───────────────────────────────────────────────────────────
<tr> → TABLE ROW (FILA)
───────────────────────────────────────────────────────────

Cada <tr> define una fila de la tabla.

───────────────────────────────────────────────────────────
<td> → TABLE DATA (CELDA DE DATOS)
───────────────────────────────────────────────────────────

Cada <td> define una celda con datos.

───────────────────────────────────────────────────────────
<th> → TABLE HEADER (CELDA DE ENCABEZADO)
───────────────────────────────────────────────────────────

<th> es como <td>, pero se usa para encabezados.
Se muestra en negrita y centrada por defecto.

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<table>
    <tr>
        <th>Producto</th>
        <th>Precio</th>
        <th>Stock</th>
    </tr>
    <tr>
        <td>Manzana</td>
        <td>$1.00</td>
        <td>50</td>
    </tr>
    <tr>
        <td>Pan</td>
        <td>$2.50</td>
        <td>30</td>
    </tr>
    <tr>
        <td>Leche</td>
        <td>$1.50</td>
        <td>20</td>
    </tr>
</table>

───────────────────────────────────────────────────────────
RESULTADO VISUAL
───────────────────────────────────────────────────────────

+----------+--------+-------+
| Producto | Precio | Stock |
+----------+--------+-------+
| Manzana  | $1.00  | 50    |
| Pan      | $2.50  | 30    |
| Leche    | $1.50  | 20    |
+----------+--------+-------+

===============================================================================
PARTE 2: ENCABEZADOS CON <th> (MÁS QUE NEGRITA)
===============================================================================

<th> no es solo una celda en negrita.
Es una celda que describe los datos de una columna o fila.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<tr>
    <th>Encabezado 1</th>
    <th>Encabezado 2</th>
    <th>Encabezado 3</th>
</tr>

───────────────────────────────────────────────────────────
EJEMPLO CON ENCABEZADOS DE FILA
───────────────────────────────────────────────────────────

<table>
    <tr>
        <th>Producto</th>
        <td>Manzana</td>
        <td>Pan</td>
    </tr>
    <tr>
        <th>Precio</th>
        <td>$1.00</td>
        <td>$2.50</td>
    </tr>
</table>

Resultado:
Producto | Manzana | Pan
Precio   | $1.00   | $2.50

Los <th> en la primera columna describen cada fila.

───────────────────────────────────────────────────────────
scope → INDICA EL ALCANCE DEL ENCABEZADO
───────────────────────────────────────────────────────────

Para mejorar la accesibilidad, usa el atributo scope:

- scope="col" → Encabezado de columna.
- scope="row" → Encabezado de fila.

<th scope="col">Producto</th>
<th scope="row">Manzana</th>

Esto ayuda a los lectores de pantalla a entender la estructura de la tabla.

===============================================================================
PARTE 3: ESTRUCTURA AVANZADA (<thead>, <tbody>, <tfoot>)
===============================================================================

Para tablas grandes, es recomendable dividir el contenido en secciones.

───────────────────────────────────────────────────────────
<thead> → CABECERA DE LA TABLA
───────────────────────────────────────────────────────────

Agrupa las filas de encabezado (generalmente la primera fila).

───────────────────────────────────────────────────────────
<tbody> → CUERPO DE LA TABLA
───────────────────────────────────────────────────────────

Agrupa las filas de datos (el contenido principal).

───────────────────────────────────────────────────────────
<tfoot> → PIE DE LA TABLA
───────────────────────────────────────────────────────────

Agrupa las filas de resumen (totales, promedios, etc.).

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<table>
    <thead>
        <tr>
            <th>Producto</th>
            <th>Precio</th>
            <th>Stock</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Manzana</td>
            <td>$1.00</td>
            <td>50</td>
        </tr>
        <tr>
            <td>Pan</td>
            <td>$2.50</td>
            <td>30</td>
        </tr>
        <tr>
            <td>Leche</td>
            <td>$1.50</td>
            <td>20</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td><strong>Total</strong></td>
            <td>$5.00</td>
            <td>100</td>
        </tr>
    </tfoot>
</table>

───────────────────────────────────────────────────────────
VENTAJAS DE USAR thead, tbody, tfoot
───────────────────────────────────────────────────────────

1. Semántica: El código describe claramente cada parte de la tabla.
2. Estilo: Puedes aplicar estilos diferentes a cada sección.
3. Accesibilidad: Los lectores de pantalla pueden navegar mejor.
4. Mantenimiento: Más fácil de leer y modificar.

===============================================================================
PARTE 4: FUSIÓN DE CELDAS (colspan Y rowspan)
===============================================================================

A veces necesitas que una celda ocupe varias columnas o filas.

───────────────────────────────────────────────────────────
colspan → FUSIÓN HORIZONTAL
───────────────────────────────────────────────────────────

colspan="n" → La celda ocupa n columnas.

Ejemplo:
<th colspan="2">Datos Personales</th>

Esto crea un encabezado que abarca dos columnas.

───────────────────────────────────────────────────────────
rowspan → FUSIÓN VERTICAL
───────────────────────────────────────────────────────────

rowspan="n" → La celda ocupa n filas.

Ejemplo:
<td rowspan="3">Contacto</td>

Esto crea una celda que abarca tres filas.

───────────────────────────────────────────────────────────
EJEMPLO COMPLETO
───────────────────────────────────────────────────────────

<table>
    <thead>
        <tr>
            <th colspan="2">Datos Personales</th>
            <th colspan="2">Contacto</th>
        </tr>
        <tr>
            <th>Nombre</th>
            <th>Edad</th>
            <th>Email</th>
            <th>Teléfono</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Ana</td>
            <td>25</td>
            <td>ana@email.com</td>
            <td>555-1111</td>
        </tr>
        <tr>
            <td>Luis</td>
            <td>30</td>
            <td>luis@email.com</td>
            <td>555-2222</td>
        </tr>
        <tr>
            <td rowspan="2">Carlos</td>
            <td>28</td>
            <td>carlos@email.com</td>
            <td>555-3333</td>
        </tr>
        <tr>
            <td colspan="3">Estudiante de programación</td>
        </tr>
    </tbody>
</table>

───────────────────────────────────────────────────────────
RESULTADO VISUAL
───────────────────────────────────────────────────────────

+------------------+--------------+-------------------------+
|  Datos Personales |    Contacto                            |
+----------+-------+----------+----------------------------+
| Nombre   | Edad  | Email    | Teléfono                   |
+----------+-------+----------+----------------------------+
| Ana      | 25    | ana@...  | 555-1111                   |
| Luis     | 30    | luis@... | 555-2222                   |
| Carlos   | 28    | carlos@  | 555-3333                   |
|          +-------+----------+----------------------------+
|          | Estudiante de programación (colspan="3")      |

===============================================================================
PARTE 5: <caption> – EL TÍTULO DE LA TABLA
===============================================================================

<caption> añade un título descriptivo a la tabla.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<table>
    <caption>Ventas mensuales por producto</caption>
    ...
</table>

El <caption> debe ser el PRIMER elemento dentro de <table>.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<table>
    <caption>Tabla de precios de frutas - 2026</caption>
    <thead>
        <tr>
            <th>Fruta</th>
            <th>Precio</th>
            <th>Stock</th>
        </tr>
    </thead>
    <tbody>
        ...
    </tbody>
</table>

───────────────────────────────────────────────────────────
VENTAJAS
───────────────────────────────────────────────────────────

- Mejora la accesibilidad (los lectores de pantalla anuncian el caption).
- Mejora el SEO (Google entiende el contexto de la tabla).
- Da contexto al usuario sobre lo que muestra la tabla.

PARTE 9: BUENAS PRÁCTICAS CON TABLAS
===============================================================================

───────────────────────────────────────────────────────────
1. USA TABLAS SOLO PARA DATOS
───────────────────────────────────────────────────────────

✅ Correcto: Horarios, precios, inventarios, resultados.
❌ Incorrecto: Maquetación de páginas (usa CSS para diseño).

───────────────────────────────────────────────────────────
2. SIEMPRE USA <caption>
───────────────────────────────────────────────────────────

Ayuda a todos a entender de qué trata la tabla.

───────────────────────────────────────────────────────────
3. USA <thead>, <tbody>, <tfoot>
───────────────────────────────────────────────────────────

Organiza la tabla en secciones lógicas.

───────────────────────────────────────────────────────────
4. USA scope EN <th>
───────────────────────────────────────────────────────────

<th scope="col"> → Encabezado de columna.
<th scope="row"> → Encabezado de fila.

───────────────────────────────────────────────────────────
5. EVITA colspan Y rowspan COMPLEJOS
───────────────────────────────────────────────────────────

Si puedes evitarlos, hazlo. Son difíciles de mantener y leer.

───────────────────────────────────────────────────────────
6. NO USES TABLAS PARA DISEÑO
───────────────────────────────────────────────────────────

Las tablas son para datos, no para maquetación.
Usa CSS Grid o Flexbox para diseño.

PARTE 10: ACCESIBILIDAD EN TABLAS
===============================================================================

Las tablas pueden ser difíciles de entender para personas con discapacidad visual.

───────────────────────────────────────────────────────────
CONSEJOS PARA TABLAS ACCESIBLES
───────────────────────────────────────────────────────────

1. Usa <caption> para describir la tabla.
2. Usa <th> para encabezados, no <td>.
3. Usa scope="col" y scope="row".
4. Evita estructuras complejas con colspan y rowspan.
5. No uses tablas para diseño.
6. Si la tabla es muy compleja, considera dividirla en varias tablas más simples.

───────────────────────────────────────────────────────────
EJEMPLO DE TABLA ACCESIBLE
───────────────────────────────────────────────────────────

<table>
    <caption>Lista de productos y precios</caption>
    <thead>
        <tr>
            <th scope="col">Producto</th>
            <th scope="col">Precio</th>
            <th scope="col">Stock</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">Manzana</th>
            <td>$1.00</td>
            <td>50</td>
        </tr>
        <tr>
            <th scope="row">Pan</th>
            <td>$2.50</td>
            <td>30</td>
        </tr>
    </tbody>
</table>

PARTE 11: ERRORES COMUNES CON TABLAS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: USAR TABLAS PARA MAQUETACIÓN
───────────────────────────────────────────────────────────

Incorrecto:
<table>
    <tr>
        <td>Menú</td>
        <td>Contenido</td>
    </tr>
</table>

Consecuencia: Malo para SEO, malo para accesibilidad, difícil de mantener.

Solución: Usa CSS Grid o Flexbox para diseño.

───────────────────────────────────────────────────────────
ERROR 2: OLVIDAR EL <caption>
───────────────────────────────────────────────────────────

Incorrecto:
<table>...</table>

Solución: Siempre añade un <caption> descriptivo.

───────────────────────────────────────────────────────────
ERROR 3: NO USAR <th> PARA ENCABEZADOS
───────────────────────────────────────────────────────────

Incorrecto:
<tr>
    <td>Producto</td>
    <td>Precio</td>
</tr>

Solución: Usa <th> para encabezados.

───────────────────────────────────────────────────────────
ERROR 4: NÚMERO DESIGUAL DE CELDAS EN LAS FILAS
───────────────────────────────────────────────────────────

Incorrecto:
<tr>
    <td>Dato 1</td>
    <td>Dato 2</td>
</tr>
<tr>
    <td>Dato 3</td>
</tr>

Consecuencia: La tabla se ve desordenada.

Solución: Todas las filas deben tener el mismo número de celdas (o usar colspan).

───────────────────────────────────────────────────────────
ERROR 5: ANIDAR TABLAS
───────────────────────────────────────────────────────────

Incorrecto:
<table>
    <tr>
        <td>
            <table>...</table>
        </td>
    </tr>
</table>

Consecuencia: Muy difícil de leer y mantener.

Solución: Nunca anides tablas. Si necesitas datos complejos, usa CSS.

===============================================================================
PARTE 12: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa tablas para DATOS, no para diseño.
2. Siempre añade <caption>.
3. Siempre usa <th> para encabezados.
4. Asegúrate de que todas las filas tengan el mismo número de celdas.
5. Nunca anides tablas.
6. Usa herramientas de validación (validator.w3.org).
7. Prueba tu tabla con un lector de pantalla (o la extensión de Chrome) para verificar accesibilidad.

GLOSARIO DE TÉRMINOS
===============================================================================

- Tabla: Estructura de datos organizada en filas y columnas.
- table: Contenedor de la tabla.
- tr: Table Row (fila).
- td: Table Data (celda de datos).
- th: Table Header (celda de encabezado).
- thead: Cabecera de la tabla.
- tbody: Cuerpo de la tabla.
- tfoot: Pie de la tabla.
- caption: Título de la tabla.
- colspan: Fusión horizontal (columnas).
- rowspan: Fusión vertical (filas).
- scope: Alcance del encabezado (col o row).
