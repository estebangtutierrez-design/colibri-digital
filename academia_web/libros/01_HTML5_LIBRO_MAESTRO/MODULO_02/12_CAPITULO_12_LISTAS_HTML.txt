===============================================================================
BIENVENIDA DEL PROFESOR BÚHO
===============================================================================

Hoy aprendemos a organizar información.

La información desordenada es como un cuarto lleno de ropa sin doblar:
todo está ahí, pero no encuentras nada.

La información ordenada es como un armario bien organizado:
cada cosa en su lugar, fácil de encontrar, fácil de usar.

Las listas HTML son el armario de tu página web.
Te permiten organizar elementos de manera clara y estructurada.

En este capítulo aprenderás tres tipos de listas, cómo anidarlas y cómo usarlas en situaciones reales como menús de navegación.

ANTES DE EMPEZAR...
===============================================================================

Cierra los ojos por un momento.

Imagina que estás escribiendo una lista de compras.
¿Qué harías?

• Escribirías cada elemento en una línea separada.
• Pondrías puntos o números al inicio de cada línea.
• Agruparías elementos relacionados (frutas, verduras, lácteos).

Ahora imagina que estás escribiendo una receta de cocina.
¿Qué harías?

• Numerarías los pasos en orden.
• Cada paso debe ejecutarse en el orden correcto.

Ahora imagina que estás escribiendo un glosario.
¿Qué harías?

• Pondrías el término y luego su definición.
• Agruparías términos y definiciones.

HTML tiene una etiqueta para cada una de esas situaciones.
Eso es lo que vamos a aprender hoy.

===============================================================================
PARTE 1: LISTAS DESORDENADAS (<ul>) – VIÑETAS
===============================================================================

La lista desordenada se usa cuando el orden NO importa.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
    <li>Elemento 3</li>
</ul>

- <ul> = Unordered List (lista desordenada)
- <li> = List Item (elemento de la lista)

───────────────────────────────────────────────────────────
EJEMPLO VISUAL
───────────────────────────────────────────────────────────

<ul>
    <li>Manzanas</li>
    <li>Plátanos</li>
    <li>Naranjas</li>
</ul>

Resultado en el navegador:
• Manzanas
• Plátanos
• Naranjas

───────────────────────────────────────────────────────────
¿CUÁNDO USAR UNA LISTA DESORDENADA?
───────────────────────────────────────────────────────────

- Lista de compras
- Ingredientes de una receta
- Habilidades o características
- Opciones de un menú
- Cualquier lista donde el orden no sea importante

===============================================================================
PARTE 2: LISTAS ORDENADAS (<ol>) – NÚMEROS
===============================================================================

La lista ordenada se usa cuando el orden SÍ importa.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<ol>
    <li>Primer paso</li>
    <li>Segundo paso</li>
    <li>Tercer paso</li>
</ol>

- <ol> = Ordered List (lista ordenada)
- <li> = List Item (elemento de la lista)

───────────────────────────────────────────────────────────
EJEMPLO VISUAL
───────────────────────────────────────────────────────────

<ol>
    <li>Precalentar el horno</li>
    <li>Mezclar los ingredientes</li>
    <li>Hornear durante 20 minutos</li>
</ol>

Resultado en el navegador:
1. Precalentar el horno
2. Mezclar los ingredientes
3. Hornear durante 20 minutos

───────────────────────────────────────────────────────────
ATRIBUTOS DE <ol>
───────────────────────────────────────────────────────────

type → Cambia el tipo de numeración
- type="1" → Números (default)
- type="A" → Letras mayúsculas (A, B, C)
- type="a" → Letras minúsculas (a, b, c)
- type="I" → Números romanos mayúsculos (I, II, III)
- type="i" → Números romanos minúsculos (i, ii, iii)

start → Cambia el número inicial
- start="5" → Comienza en el número 5

reversed → Invierte el orden (HTML5)
- <ol reversed>

───────────────────────────────────────────────────────────
EJEMPLO CON ATRIBUTOS
───────────────────────────────────────────────────────────

<ol type="A" start="3">
    <li>Opción C</li>
    <li>Opción D</li>
    <li>Opción E</li>
</ol>

Resultado:
C. Opción C
D. Opción D
E. Opción E

───────────────────────────────────────────────────────────
¿CUÁNDO USAR UNA LISTA ORDENADA?
───────────────────────────────────────────────────────────

- Pasos de una receta
- Instrucciones de un proceso
- Ranking de algo (top 5)
- Cronología de eventos
- Cualquier lista donde el orden sea importante

===============================================================================
PARTE 3: LISTAS DE DEFINICIÓN (<dl>) – GLOSARIOS
===============================================================================

La lista de definición se usa para pares término-definición.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<dl>
    <dt>Término</dt>
    <dd>Definición del término</dd>

    <dt>Otro término</dt>
    <dd>Definición del otro término</dd>
</dl>

- <dl> = Definition List (lista de definición)
- <dt> = Definition Term (término)
- <dd> = Definition Description (definición)

───────────────────────────────────────────────────────────
EJEMPLO VISUAL
───────────────────────────────────────────────────────────

<dl>
    <dt>HTML</dt>
    <dd>Lenguaje de marcado para crear páginas web.</dd>

    <dt>CSS</dt>
    <dd>Lenguaje de estilos para diseñar páginas web.</dd>

    <dt>JavaScript</dt>
    <dd>Lenguaje de programación para dar interactividad.</dd>
</dl>

Resultado en el navegador:
HTML
    Lenguaje de marcado para crear páginas web.
CSS
    Lenguaje de estilos para diseñar páginas web.
JavaScript
    Lenguaje de programación para dar interactividad.

───────────────────────────────────────────────────────────
¿CUÁNDO USAR UNA LISTA DE DEFINICIÓN?
───────────────────────────────────────────────────────────

- Glosarios
- Diccionarios
- Preguntas frecuentes (FAQ)
- Términos técnicos con sus explicaciones
- Metadatos estructurados

===============================================================================
PARTE 4: LISTAS ANIDADAS (LISTAS DENTRO DE LISTAS)
===============================================================================

Las listas anidadas permiten crear estructuras jerárquicas.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<ul>
    <li>Fruta
        <ul>
            <li>Manzana</li>
            <li>Plátano</li>
            <li>Naranja</li>
        </ul>
    </li>
    <li>Verdura
        <ul>
            <li>Lechuga</li>
            <li>Zanahoria</li>
            <li>Brócoli</li>
        </ul>
    </li>
</ul>

───────────────────────────────────────────────────────────
EJEMPLO VISUAL
───────────────────────────────────────────────────────────

Resultado en el navegador:
• Fruta
    • Manzana
    • Plátano
    • Naranja
• Verdura
    • Lechuga
    • Zanahoria
    • Brócoli

───────────────────────────────────────────────────────────
REGLAS DE ANIDACIÓN
───────────────────────────────────────────────────────────

1. La lista anidada debe ir DENTRO del <li> de la lista padre.
2. La lista anidada debe cerrarse antes de cerrar el <li> padre.
3. Puedes anidar cualquier tipo de lista dentro de cualquier otra.
   (ul dentro de ol, ol dentro de ul, etc.)

───────────────────────────────────────────────────────────
EJEMPLO CORRECTO
───────────────────────────────────────────────────────────

<ul>
    <li>
        Fruta
        <ul>
            <li>Manzana</li>
            <li>Plátano</li>
        </ul>
    </li>
</ul>

───────────────────────────────────────────────────────────
EJEMPLO INCORRECTO
───────────────────────────────────────────────────────────

<ul>
    <li>Fruta</li>   ← Error: aquí se cierra el li antes de la sublista
    <ul>
        <li>Manzana</li>
    </ul>
</ul>

La sublista debe estar DENTRO del <li> padre.

PARTE 5: LISTAS PARA MENÚS DE NAVEGACIÓN
===============================================================================

Una de las aplicaciones más importantes de las listas es la navegación.

───────────────────────────────────────────────────────────
MENÚ DE NAVEGACIÓN BÁSICO
───────────────────────────────────────────────────────────

<nav>
    <ul>
        <li><a href="index.html">Inicio</a></li>
        <li><a href="servicios.html">Servicios</a></li>
        <li><a href="blog.html">Blog</a></li>
        <li><a href="contacto.html">Contacto</a></li>
    </ul>
</nav>

Resultado:
Inicio   Servicios   Blog   Contacto

(Con CSS, estos elementos se pueden poner en horizontal)

───────────────────────────────────────────────────────────
MENÚ CON SUBMENÚ (ANIDADO)
───────────────────────────────────────────────────────────

<nav>
    <ul>
        <li><a href="index.html">Inicio</a></li>
        <li>
            <a href="servicios.html">Servicios</a>
            <ul>
                <li><a href="diseno.html">Diseño web</a></li>
                <li><a href="desarrollo.html">Desarrollo</a></li>
                <li><a href="consultoria.html">Consultoría</a></li>
            </ul>
        </li>
        <li><a href="blog.html">Blog</a></li>
        <li><a href="contacto.html">Contacto</a></li>
    </ul>
</nav>

Resultado:
Inicio   Servicios ▼   Blog   Contacto
         • Diseño web
         • Desarrollo
         • Consultoría

PARTE 9: ERRORES COMUNES CON LISTAS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: NO USAR <li> DENTRO DE <ul> O <ol>
───────────────────────────────────────────────────────────

Incorrecto:
<ul>
    Elemento 1
    Elemento 2
</ul>

Correcto:
<ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
</ul>

Solución: Siempre usa <li> para cada elemento de la lista.

───────────────────────────────────────────────────────────
ERROR 2: CERRAR EL <li> ANTES DE LA SUBLISTA
───────────────────────────────────────────────────────────

Incorrecto:
<ul>
    <li>Fruta</li>   ← Error: cerró el li antes de la sublista
    <ul>
        <li>Manzana</li>
    </ul>
</ul>

Correcto:
<ul>
    <li>
        Fruta
        <ul>
            <li>Manzana</li>
        </ul>
    </li>
</ul>

Solución: La sublista va DENTRO del <li> padre.

───────────────────────────────────────────────────────────
ERROR 3: MEZCLAR TIPOS DE LISTAS INCORRECTAMENTE
───────────────────────────────────────────────────────────

Incorrecto:
<dl>
    <ul>
        <li>Elemento</li>
    </ul>
</dl>

Correcto:
<dl>
    <dt>Término</dt>
    <dd>Definición</dd>
</dl>

Solución: <dl> solo contiene <dt> y <dd>.
<ul> y <ol> solo contienen <li>.

───────────────────────────────────────────────────────────
ERROR 4: NO CERRAR ETIQUETAS
───────────────────────────────────────────────────────────

Incorrecto:
<ul>
    <li>Elemento 1
    <li>Elemento 2
</ul>

Correcto:
<ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
</ul>

Solución: Siempre cierra todas las etiquetas.

===============================================================================
PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa <li> dentro de <ul> o <ol>.
2. Siempre coloca las sublistas DENTRO de un <li> padre.
3. Usa el validador del W3C (validator.w3.org).
4. Usa un editor de código que coloree las etiquetas.
5. Mantén la indentación correcta: los hijos deben estar sangrados respecto a los padres.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- Lista desordenada (ul): Lista con viñetas (•) donde el orden no importa.
- Lista ordenada (ol): Lista con números donde el orden importa.
- Lista de definición (dl): Lista de términos y definiciones.
- Elemento de lista (li): Cada elemento dentro de una lista ul u ol.
- Término (dt): El término en una lista de definición.
- Definición (dd): La definición en una lista de definición.
- Anidación: Colocar una lista dentro de otra.
- Menú de navegación: Lista de enlaces para navegar por un sitio web.

Preguntas frecuentes:
- ¿Cuándo usar ul y cuándo ol? Usa ul cuando el orden no importa; usa ol cuando el orden sí importa.
- ¿Puedo anidar ol dentro de ul? Sí, puedes anidar cualquier tipo de lista dentro de cualquier otra.
- ¿Las listas se pueden estilizar con CSS? Sí, con CSS puedes cambiar viñetas, colores, espaciado, etc.
- ¿Por qué usar listas para menús? Porque son semánticas, accesibles y fáciles de estilizar.
- ¿Puedo tener una lista de definición sin definición? Técnicamente sí, pero no tiene sentido semántico.

Analogías utilizadas:
- Lista desordenada = lista de compras (sin orden).
- Lista ordenada = receta de cocina (pasos en orden).
- Lista de definición = diccionario (término y definición).
- Lista anidada = categorías y subcategorías.

- Lista de materiales.
