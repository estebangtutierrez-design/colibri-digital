PARTE 1: LA ETIQUETA <form> (EL CONTENEDOR DE FORMULARIOS)
===============================================================================

La etiqueta <form> envuelve todos los elementos del formulario.

───────────────────────────────────────────────────────────
SINTAXIS BÁSICA
───────────────────────────────────────────────────────────

<form action="/enviar" method="POST">
    <!-- Campos del formulario -->
</form>

───────────────────────────────────────────────────────────
ATRIBUTOS PRINCIPALES DE <form>
───────────────────────────────────────────────────────────

action → URL donde se enviarán los datos.
method → Cómo se enviarán los datos (GET o POST).
name → Nombre del formulario (para JavaScript).
enctype → Tipo de codificación (para archivos: multipart/form-data).
target → Dónde mostrar la respuesta (_self, _blank, etc.).

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<form action="/contacto" method="POST">
    <!-- Aquí van los campos -->
</form>

===============================================================================
PARTE 2: method – GET vs POST (LA DIFERENCIA CLAVE)
===============================================================================

El atributo method define cómo se envían los datos al servidor.

───────────────────────────────────────────────────────────
GET → DATOS VISIBLES EN LA URL
───────────────────────────────────────────────────────────

<form action="/buscar" method="GET">
    <input type="text" name="q">
    <button type="submit">Buscar</button>
</form>

Cuando el usuario envía el formulario, los datos aparecen en la URL:
/buscar?q=html

Ventajas:
- Los datos son visibles (se pueden compartir, marcar como favorito).
- Se puede usar en buscadores y filtros.

Desventajas:
- Los datos son VISIBLES (no usar para contraseñas).
- Límite de tamaño (aproximadamente 2048 caracteres).

Cuándo usar GET:
- Búsquedas.
- Filtros.
- Cualquier cosa que quieras compartir por URL.

───────────────────────────────────────────────────────────
POST → DATOS OCULTOS EN EL CUERPO DE LA PETICIÓN
───────────────────────────────────────────────────────────

<form action="/registro" method="POST">
    <input type="text" name="nombre">
    <input type="password" name="clave">
    <button type="submit">Registrar</button>
</form>

Los datos NO aparecen en la URL.
Van en el cuerpo de la petición HTTP.

Ventajas:
- Los datos son OCULTOS (seguridad).
- Sin límite de tamaño (puedes enviar archivos grandes).

Desventajas:
- No se pueden marcar como favorito.
- No se pueden compartir por URL.

Cuándo usar POST:
- Formularios de contacto.
- Registros de usuario.
- Inicios de sesión.
- Envío de archivos.
- Cualquier dato sensible.

PARTE 3: <label> – LA ETIQUETA DE LOS CAMPOS
===============================================================================

<label> asocia un texto descriptivo a un campo de formulario.

───────────────────────────────────────────────────────────
SINTAXIS (MÉTODO 1: CON for)
───────────────────────────────────────────────────────────

<label for="nombre">Nombre:</label>
<input type="text" id="nombre" name="nombre">

El atributo for del label debe coincidir con el id del input.

───────────────────────────────────────────────────────────
SINTAXIS (MÉTODO 2: ANIDADO)
───────────────────────────────────────────────────────────

<label>
    Nombre:
    <input type="text" name="nombre">
</label>

Ambos métodos son válidos, pero el primero es más claro y recomendado.

───────────────────────────────────────────────────────────
¿POR QUÉ USAR <label>?
───────────────────────────────────────────────────────────

1. ACCESIBILIDAD: Los lectores de pantalla anuncian el label.
2. USABILIDAD: Al hacer clic en el label, el campo recibe el foco.
3. SEO: Mejora la comprensión del formulario.

───────────────────────────────────────────────────────────
EJEMPLO COMPLETO
───────────────────────────────────────────────────────────

<form action="/enviar" method="POST">
    <label for="nombre">Nombre completo:</label>
    <input type="text" id="nombre" name="nombre">

    <label for="email">Correo electrónico:</label>
    <input type="email" id="email" name="email">

    <button type="submit">Enviar</button>
</form>

PARTE 4: <input> – LOS CAMPOS DE ENTRADA (TIPOS BÁSICOS)
===============================================================================

<input> es la etiqueta más común en formularios.

───────────────────────────────────────────────────────────
type="text" → TEXTO CORTO
───────────────────────────────────────────────────────────

<input type="text" id="nombre" name="nombre" placeholder="Escribe tu nombre">

───────────────────────────────────────────────────────────
type="email" → CORREO ELECTRÓNICO
───────────────────────────────────────────────────────────

<input type="email" id="email" name="email" placeholder="tu@email.com">

Valida automáticamente que sea un correo válido.

───────────────────────────────────────────────────────────
type="password" → CONTRASEÑA
───────────────────────────────────────────────────────────

<input type="password" id="clave" name="clave" placeholder="Mínimo 8 caracteres">

Oculta los caracteres con asteriscos o puntos.

───────────────────────────────────────────────────────────
type="tel" → TELÉFONO
───────────────────────────────────────────────────────────

<input type="tel" id="telefono" name="telefono" placeholder="55 1234 5678">

En móviles, muestra el teclado numérico.

───────────────────────────────────────────────────────────
type="number" → NÚMERO
───────────────────────────────────────────────────────────

<input type="number" id="edad" name="edad" min="18" max="99" value="18">

Muestra flechas para subir/bajar.

───────────────────────────────────────────────────────────
type="search" → BÚSQUEDA
───────────────────────────────────────────────────────────

<input type="search" id="buscar" name="buscar" placeholder="Buscar...">

En algunos navegadores, muestra una "X" para borrar.

───────────────────────────────────────────────────────────
type="url" → URL
───────────────────────────────────────────────────────────

<input type="url" id="sitio" name="sitio" placeholder="https://ejemplo.com">

Valida que sea una URL válida.

===============================================================================
PARTE 5: ATRIBUTOS COMUNES DE <input>
===============================================================================

───────────────────────────────────────────────────────────
id → IDENTIFICADOR ÚNICO (para asociar con label)
───────────────────────────────────────────────────────────

<input type="text" id="nombre" name="nombre">

───────────────────────────────────────────────────────────
name → NOMBRE DEL CAMPO (se envía al servidor)
───────────────────────────────────────────────────────────

<input type="text" name="nombre">

El servidor recibe: nombre = valor.

───────────────────────────────────────────────────────────
placeholder → TEXTO DE EJEMPLO
───────────────────────────────────────────────────────────

<input type="text" placeholder="Escribe tu nombre">

Aparece dentro del campo, desaparece al escribir.

───────────────────────────────────────────────────────────
value → VALOR POR DEFECTO
───────────────────────────────────────────────────────────

<input type="text" value="Valor predeterminado">

Aparece dentro del campo y se puede modificar.

───────────────────────────────────────────────────────────
required → CAMPO OBLIGATORIO
───────────────────────────────────────────────────────────

<input type="text" required>

El navegador no permite enviar el formulario si está vacío.

───────────────────────────────────────────────────────────
readonly → SOLO LECTURA (no se puede modificar)
───────────────────────────────────────────────────────────

<input type="text" value="No editable" readonly>

El valor se envía, pero no se puede cambiar.

───────────────────────────────────────────────────────────
disabled → DESHABILITADO (no se puede usar, no se envía)
───────────────────────────────────────────────────────────

<input type="text" value="Deshabilitado" disabled>

El campo aparece gris, no se puede interactuar, no se envía.

───────────────────────────────────────────────────────────
autofocus → FOCUS AUTOMÁTICO
───────────────────────────────────────────────────────────

<input type="text" autofocus>

El campo recibe el foco al cargar la página.

───────────────────────────────────────────────────────────
min y max → VALORES MÍNIMO Y MÁXIMO
───────────────────────────────────────────────────────────

<input type="number" min="18" max="99">

Funciona con number, date, range, etc.

───────────────────────────────────────────────────────────
step → INCREMENTO
───────────────────────────────────────────────────────────

<input type="number" step="5">

Los valores aumentan de 5 en 5.

PARTE 6: <textarea> – ÁREA DE TEXTO MULTILÍNEA
===============================================================================

<textarea> permite escribir texto largo, como mensajes o comentarios.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<textarea id="mensaje" name="mensaje" rows="5" cols="40">
    Texto inicial (opcional)
</textarea>

───────────────────────────────────────────────────────────
ATRIBUTOS DE <textarea>
───────────────────────────────────────────────────────────

rows → Número de líneas visibles.
cols → Ancho en caracteres.
placeholder → Texto de ejemplo.
maxlength → Número máximo de caracteres.
required → Campo obligatorio.
readonly → Solo lectura.
disabled → Deshabilitado.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<label for="mensaje">Mensaje:</label>
<textarea id="mensaje" name="mensaje" rows="6" cols="50" placeholder="Escribe tu mensaje aquí..." required></textarea>

===============================================================================
PARTE 7: <select> y <option> – MENÚ DESPLEGABLE
===============================================================================

<select> crea un menú desplegable con opciones.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<select id="pais" name="pais" required>
    <option value="">Selecciona un país</option>
    <option value="mx">México</option>
    <option value="us">Estados Unidos</option>
    <option value="es">España</option>
    <option value="ar">Argentina</option>
</select>

───────────────────────────────────────────────────────────
ATRIBUTOS DE <select>
───────────────────────────────────────────────────────────

- required → Obligatorio.
- multiple → Permite seleccionar múltiples opciones.
- size → Número de opciones visibles.
- disabled → Deshabilitado.

───────────────────────────────────────────────────────────
ATRIBUTOS DE <option>
───────────────────────────────────────────────────────────

- value → Valor que se envía al servidor.
- selected → Opción seleccionada por defecto.

───────────────────────────────────────────────────────────
EJEMPLO CON SELECTED
───────────────────────────────────────────────────────────

<select id="pais" name="pais">
    <option value="mx">México</option>
    <option value="us" selected>Estados Unidos</option>
    <option value="es">España</option>
</select>

───────────────────────────────────────────────────────────
EJEMPLO CON OPTGROUP (AGRUPACIÓN)
───────────────────────────────────────────────────────────

<select id="ciudad" name="ciudad">
    <optgroup label="México">
        <option value="cdmx">Ciudad de México</option>
        <option value="gdl">Guadalajara</option>
        <option value="mty">Monterrey</option>
    </optgroup>
    <optgroup label="España">
        <option value="mad">Madrid</option>
        <option value="bcn">Barcelona</option>
        <option value="sev">Sevilla</option>
    </optgroup>
</select>

PARTE 8: <button> – BOTONES DE ACCIÓN
===============================================================================

<button> crea botones para acciones como enviar o reiniciar.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<button type="submit">Enviar</button>
<button type="reset">Limpiar</button>
<button type="button">Clic aquí</button>

───────────────────────────────────────────────────────────
ATRIBUTOS DE <button>
───────────────────────────────────────────────────────────

type → submit, reset, button
name → Nombre del botón.
value → Valor que se envía con submit.
disabled → Deshabilitado.

───────────────────────────────────────────────────────────
DIFERENCIA ENTRE <button> Y <input type="submit">
───────────────────────────────────────────────────────────

<button> permite contenido HTML dentro (iconos, emojis, imágenes).
<input type="submit"> solo permite texto.

───────────────────────────────────────────────────────────
EJEMPLO CON ICONO
───────────────────────────────────────────────────────────

<button type="submit">📨 Enviar mensaje</button>

===============================================================================
PARTE 9: <fieldset> y <legend> – AGRUPACIÓN DE CAMPOS
===============================================================================

<fieldset> agrupa campos relacionados.
<legend> da un título al grupo.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<fieldset>
    <legend>Datos personales</legend>
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    <label for="email">Correo:</label>
    <input type="email" id="email" name="email">
</fieldset>

<fieldset>
    <legend>Mensaje</legend>
    <label for="mensaje">Escribe tu mensaje:</label>
    <textarea id="mensaje" name="mensaje" rows="5"></textarea>
</fieldset>

───────────────────────────────────────────────────────────
VENTAJAS
───────────────────────────────────────────────────────────

- Organiza visualmente el formulario.
- Mejora la accesibilidad.
- Ayuda al usuario a entender la estructura.

PARTE 13: ERRORES COMUNES CON FORMULARIOS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR EL ATRIBUTO name EN LOS INPUTS
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" id="nombre">

Consecuencia: El campo no se envía al servidor.

Solución: Siempre añade name a los inputs.

───────────────────────────────────────────────────────────
ERROR 2: NO ASOCIAR <label> CON <input> (usando for/id)
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" id="nombre">

Solución: Añade <label for="nombre">Nombre:</label>

───────────────────────────────────────────────────────────
ERROR 3: USAR GET PARA DATOS SENSIBLES
───────────────────────────────────────────────────────────

Incorrecto:
<form method="GET">
    <input type="password" name="clave">
</form>

Consecuencia: La contraseña aparece en la URL.

Solución: Usa POST para contraseñas y datos sensibles.

───────────────────────────────────────────────────────────
ERROR 4: OLVIDAR ACTION EN EL FORMULARIO
───────────────────────────────────────────────────────────

Incorrecto:
<form method="POST">

Consecuencia: Los datos se envían a la misma página (no a un procesador).

Solución: Siempre añade action (puede ser "" para la misma página).

───────────────────────────────────────────────────────────
ERROR 5: NO USAR REQUIRED EN CAMPOS OBLIGATORIOS
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" name="nombre">

Consecuencia: El usuario puede enviar el formulario vacío.

Solución: Añade required a los campos que deben llenarse.

───────────────────────────────────────────────────────────
ERROR 6: OLVIDAR EL ATRIBUTO type EN <button>
───────────────────────────────────────────────────────────

Incorrecto:
<button>Enviar</button>

Consecuencia: Por defecto, type="submit", puede funcionar mal si no quieres enviar.

Solución: Siempre especifica type="submit" o type="button".

===============================================================================
PARTE 14: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre añade name a todos los inputs, selects y textareas.
2. Siempre asocia label con input usando for/id.
3. Usa POST para datos sensibles, GET para búsquedas.
4. Siempre añade action (puede ser "" para la misma página).
5. Usa required en todos los campos obligatorios.
6. Siempre especifica type en los botones.
7. Prueba tu formulario enviándolo y revisa que los datos lleguen correctamente.
8. Usa validación HTML5 (required, type, pattern) antes de JavaScript.

GLOSARIO DE TÉRMINOS
===============================================================================

- Formulario: Conjunto de campos que permiten al usuario enviar datos.
- form: Contenedor del formulario.
- action: URL donde se envían los datos.
- method: GET (visible) o POST (oculto).
- input: Campo de entrada de datos.
- label: Etiqueta descriptiva de un campo.
- placeholder: Texto de ejemplo dentro del campo.
- required: Campo obligatorio.
- fieldset: Agrupación de campos.
- legend: Título de un fieldset.
- textarea: Área de texto multilínea.
- select: Menú desplegable.
- option: Opción dentro de un select.
- button: Botón de acción.

- Seguridad web (GET vs POST)
- Accesibilidad
- Validación de datos

Errores frecuentes:
- Olvidar el atributo name
- No asociar label con input
- Usar GET para datos sensibles
- Olvidar action
- No usar required
- Olvidar type en button

Preguntas frecuentes:
- ¿GET o POST? POST para datos sensibles, GET para búsquedas.
- ¿Por qué usar label? Por accesibilidad y usabilidad.
- ¿Qué pasa si no uso name? El campo no se envía.
- ¿Cómo valido un campo? Usa required, type, min, max, pattern.
- ¿Qué es fieldset? Agrupa campos relacionados.

Analogías utilizadas:
- Formulario = hoja de solicitud.
- action = dirección de envío.
- method = sobre abierto (GET) o sellado (POST).
- label = etiqueta del campo.
- placeholder = ejemplo dentro del campo.
- fieldset = carpeta que agrupa documentos.

Ejemplos utilizados:
- Formulario de contacto completo.
- Formulario de registro con validaciones.
- Menú desplegable con optgroup.
