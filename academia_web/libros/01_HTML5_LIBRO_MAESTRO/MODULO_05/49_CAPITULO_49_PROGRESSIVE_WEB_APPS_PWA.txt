===============================================================================
PARTE 1: ¿QUÉ SON LAS PWA?
===============================================================================

Una Progressive Web App (PWA) es una aplicación web que utiliza tecnologías modernas para ofrecer una experiencia similar a la de una aplicación nativa.

───────────────────────────────────────────────────────────
CARACTERÍSTICAS PRINCIPALES
───────────────────────────────────────────────────────────

1. INSTALABLE: Se puede instalar en el dispositivo del usuario.
2. OFFLINE: Funciona sin conexión a Internet.
3. RÁPIDA: Carga instantáneamente (incluso en redes lentas).
4. SEGURA: Se sirve a través de HTTPS.
5. ENGANCHE: Envía notificaciones push.
6. RESPONSIVE: Funciona en cualquier dispositivo.
7. ACTUALIZABLE: Se actualiza automáticamente como una web.

───────────────────────────────────────────────────────────
PWA VS. APPS NATIVAS
───────────────────────────────────────────────────────────

| Característica | PWA | App Nativa |
|----------------|-----|------------|
| Instalación | Desde el navegador | Desde la tienda de apps |
| Actualización | Automática (web) | Manual (tienda) |
| Coste de desarrollo | Bajo | Alto (iOS + Android) |
| Distribución | URL | App Store / Google Play |
| Acceso a dispositivos | Limitado | Completo |
| Tamaño | Pequeño (KB-MB) | Grande (MB-GB) |
| Sin conexión | Sí (con Service Worker) | Sí |

───────────────────────────────────────────────────────────
EJEMPLOS DE PWA FAMOSAS
───────────────────────────────────────────────────────────

- **Twitter Lite**: Versión PWA de Twitter.
- **Spotify Web**: Reproductor de música PWA.
- **Uber Web**: Versión PWA de Uber.
- **Pinterest**: PWA con instalación en pantalla de inicio.
- **Tinder Web**: Versión PWA de Tinder.
- **Google Maps Go**: Versión PWA de Google Maps.
- **Starbucks**: PWA para pedidos sin conexión.

PARTE 4: SERVICE WORKER (INTRODUCCIÓN)
===============================================================================

Un Service Worker es un script que se ejecuta en segundo plano, separado de la página web.

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

SEPARADO: No tiene acceso al DOM.

PERSISTENTE: Permanece activo incluso después de cerrar la página.

INTERCEPTOR: Puede interceptar peticiones de red.

CACHÉ: Puede almacenar recursos para uso offline.

PUSH: Puede recibir notificaciones push.

───────────────────────────────────────────────────────────
CICLO DE VIDA DEL SERVICE WORKER
───────────────────────────────────────────────────────────

REGISTRO: El navegador registra el Service Worker.

INSTALACIÓN: Se dispara el evento install.

ACTIVACIÓN: Se dispara el evento activate.

INACTIVO: El worker está esperando eventos.

FETCH: Intercepta peticiones de red.

TERMINADO: El navegador lo termina para ahorrar recursos.

───────────────────────────────────────────────────────────
REGISTRAR UN SERVICE WORKER
───────────────────────────────────────────────────────────

javascript
// En tu página principal (ej: index.html)
if ('serviceWorker' in navigator) {
    window.addEventListener('load', function() {
        navigator.serviceWorker.register('/sw.js')
            .then(function(registration) {
                console.log('✅ Service Worker registrado:', registration);
            })
            .catch(function(error) {
                console.log('❌ Error al registrar Service Worker:', error);
            });
    });
}
───────────────────────────────────────────────────────────
SERVICE WORKER BÁSICO (sw.js)
───────────────────────────────────────────────────────────

javascript
// sw.js - Service Worker básico

const CACHE_NAME = 'mi-pwa-v1';
const URLS_CACHE = [
    '/',
    '/index.html',
    '/css/style.css',
    '/js/app.js',
    '/images/icon-192.png',
    '/images/icon-512.png'
];

// ============================================================ */
// INSTALACIÓN                                                  */
// ============================================================ */

self.addEventListener('install', function(event) {
    console.log('🔄 Service Worker instalando...');

    event.waitUntil(
        caches.open(CACHE_NAME).then(function(cache) {
            console.log('📦 Cacheando recursos...');
            return cache.addAll(URLS_CACHE);
        })
    );
});

// ============================================================ */
// ACTIVACIÓN                                                   */
// ============================================================ */

self.addEventListener('activate', function(event) {
    console.log('✅ Service Worker activado');

    // Limpiar caches antiguos
    event.waitUntil(
        caches.keys().then(function(cacheNames) {
            return Promise.all(
                cacheNames.map(function(cacheName) {
                    if (cacheName !== CACHE_NAME) {
                        console.log('🗑️ Eliminando cache antiguo:', cacheName);
                        return caches.delete(cacheName);
                    }
                })
            );
        })
    );
});

// ============================================================ */
// FETCH (INTERCEPTAR PETICIONES)                               */
// ============================================================ */

self.addEventListener('fetch', function(event) {
    console.log('🌐 Interceptando:', event.request.url);

    event.respondWith(
        caches.match(event.request).then(function(response) {
            // Si está en cache, devolverlo
            if (response) {
                return response;
            }

            // Si no, ir a la red
            return fetch(event.request).then(function(response) {
                // Si la respuesta es válida, guardarla en cache
                if (response && response.status === 200) {
                    const responseClone = response.clone();
                    caches.open(CACHE_NAME).then(function(cache) {
                        cache.put(event.request, responseClone);
                    });
                }
                return response;
            });
        })
    );
});

PARTE 5: INSTALAR LA PWA
===============================================================================

Cuando todos los requisitos están cumplidos, el navegador muestra un botón de instalación.

───────────────────────────────────────────────────────────
CÓMO SE INSTALA
───────────────────────────────────────────────────────────

El usuario visita la página web.

El navegador detecta que es una PWA.

Aparece un botón de instalación (en la barra de direcciones o un mensaje).

El usuario hace clic en "Instalar".

La PWA se instala en el dispositivo.

Aparece un icono en la pantalla de inicio.

───────────────────────────────────────────────────────────
DETECTAR EL EVENTO DE INSTALACIÓN
───────────────────────────────────────────────────────────

javascript
// En tu página principal
let deferredPrompt;

window.addEventListener('beforeinstallprompt', function(event) {
    console.log('🔔 Evento de instalación detectado');
    event.preventDefault();
    deferredPrompt = event;

    // Mostrar un botón personalizado de instalación
    const botonInstalar = document.getElementById('btn-instalar');
    if (botonInstalar) {
        botonInstalar.style.display = 'block';
        botonInstalar.onclick = function() {
            if (deferredPrompt) {
                deferredPrompt.prompt();
                deferredPrompt.userChoice.then(function(choiceResult) {
                    if (choiceResult.outcome === 'accepted') {
                        console.log('✅ Usuario aceptó la instalación');
                    } else {
                        console.log('❌ Usuario rechazó la instalación');
                    }
                    deferredPrompt = null;
                });
            }
        };
    }
});

// ============================================================ */
// GESTIÓN DE NOTAS                                             */
// ============================================================ */

let notas = JSON.parse(localStorage.getItem('notas') || '[]');

function renderizarNotas() {
    const lista = document.getElementById('lista-notas');
    if (notas.length === 0) {
        lista.innerHTML = '<p style="color: #8b949e;">📭 No hay notas. Agrega una.</p>';
        return;
    }

    let html = '';
    notas.forEach((nota, index) => {
        html += `
            <div class="nota">
                <span>${nota}</span>
                <button onclick="eliminarNota(${index})">✕</button>
            </div>
        `;
    });
    lista.innerHTML = html;
}

function agregarNota() {
    const input = document.getElementById('nota-input');
    const texto = input.value.trim();
    if (texto === '') return;

    notas.push(texto);
    localStorage.setItem('notas', JSON.stringify(notas));
    input.value = '';
    renderizarNotas();
}

function eliminarNota(index) {
    notas.splice(index, 1);
    localStorage.setItem('notas', JSON.stringify(notas));
    renderizarNotas();
}

// ============================================================ */
// EVENTOS                                                      */
// ============================================================ */

document.getElementById('btn-agregar').addEventListener('click', agregarNota);
document.getElementById('nota-input').addEventListener('keypress', function(e) {
    if (e.key === 'Enter') agregarNota();
});

// ============================================================ */
// ESTADO DE CONEXIÓN                                           */
// ============================================================ */

window.addEventListener('online', function() {
    document.getElementById('estado-offline').textContent = '🟢 Conectado';
});

window.addEventListener('offline', function() {
    document.getElementById('estado-offline').textContent = '🔴 Sin conexión (modo offline)';
});

// ============================================================ */
// INSTALACIÓN PWA                                              */
// ============================================================ */

let deferredPrompt;

window.addEventListener('beforeinstallprompt', function(event) {
    event.preventDefault();
    deferredPrompt = event;

    const btn = document.getElementById('btn-instalar');
    btn.style.display = 'inline-block';
    btn.onclick = function() {
        if (deferredPrompt) {
            deferredPrompt.prompt();
            deferredPrompt.userChoice.then(function(result) {
                if (result.outcome === 'accepted') {
                    console.log('✅ PWA instalada');
                }
                deferredPrompt = null;
                btn.style.display = 'none';
            });
        }
    };
});

window.addEventListener('appinstalled', function() {
    console.log('📲 PWA instalada correctamente');
});

PARTE 7: BUENAS PRÁCTICAS DE PWA
===============================================================================

───────────────────────────────────────────────────────────

USA HTTPS
───────────────────────────────────────────────────────────

Las PWA solo funcionan en HTTPS (o localhost).

───────────────────────────────────────────────────────────

ICONOS EN MÚLTIPLES TAMAÑOS
───────────────────────────────────────────────────────────

Proporciona iconos en varios tamaños (72, 96, 128, 144, 152, 192, 384, 512).

───────────────────────────────────────────────────────────

CACHEA RECURSOS ESTRATÉGICAMENTE
───────────────────────────────────────────────────────────

Cachea el HTML, CSS y JS principal.

Cachea iconos y recursos estáticos.

No cachees datos dinámicos (a menos que sea necesario).

───────────────────────────────────────────────────────────

PARTE 9: ERRORES COMUNES EN PWA
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: NO USAR HTTPS
───────────────────────────────────────────────────────────

Consecuencia: La PWA no funciona en producción.

Solución: Usa HTTPS (o localhost en desarrollo).

───────────────────────────────────────────────────────────
ERROR 2: SERVICE WORKER NO REGISTRADO
───────────────────────────────────────────────────────────

Consecuencia: La app no funciona sin conexión.

Solución: Verifica que sw.js esté en la ruta correcta.

───────────────────────────────────────────────────────────
ERROR 3: ICONOS FALTANTES
───────────────────────────────────────────────────────────

Consecuencia: La instalación falla o el icono no se muestra.

Solución: Asegúrate de que los iconos existan en las rutas especificadas.

───────────────────────────────────────────────────────────
ERROR 4: start_url INCORRECTA
───────────────────────────────────────────────────────────

Consecuencia: La app abre en la página incorrecta.

Solución: Verifica que start_url sea correcta.

───────────────────────────────────────────────────────────
ERROR 5: CACHE DEMASIADO GRANDE
───────────────────────────────────────────────────────────

Consecuencia: La instalación es lenta o falla.

Solución: Cachea solo los recursos esenciales.

===============================================================================
PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

Usa HTTPS (certificado SSL o localhost).

Verifica que sw.js esté en la raíz del sitio.

Asegúrate de que todos los iconos existan.

Verifica que start_url sea correcta.

Limpia el caché en el evento activate.

Experimento: Registrar un Service Worker desde la consola

Abre la consola del navegador (F12 → Console) y prueba:

javascript
// Verificar soporte
if ('serviceWorker' in navigator) {
    console.log('✅ Service Worker soportado');

    // Registrar Service Worker
    navigator.serviceWorker.register('/sw.js')
        .then(function(reg) {
            console.log('✅ Service Worker registrado:', reg);
        })
        .catch(function(err) {
            console.log('❌ Error:', err);
        });
} else {
    console.log('❌ Service Worker no soportado');
}

GLOSARIO DE TÉRMINOS
===============================================================================

PWA: Progressive Web App.

Manifest: Archivo JSON que configura la PWA.

Service Worker: Script que se ejecuta en segundo plano.

Cache: Almacenamiento temporal de recursos.

Instalable: Capacidad de ser instalada en el dispositivo.

Offline: Funcionamiento sin conexión a Internet.

HTTPS: Protocolo seguro (requerido para PWA).

Lighthouse: Herramienta de Google para auditar PWA.
