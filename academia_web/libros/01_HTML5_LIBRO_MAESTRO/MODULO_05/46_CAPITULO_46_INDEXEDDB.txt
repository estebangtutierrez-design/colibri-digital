PARTE 1: ¿QUÉ ES INDEXEDDB?
===============================================================================

IndexedDB es una API de HTML5 para almacenar grandes cantidades de datos estructurados en el navegador.

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

1. BASE DE DATOS NoSQL: Almacena objetos, no tablas.
2. ASÍNCRONA: Las operaciones no bloquean la interfaz.
3. GRAN CAPACIDAD: Cientos de MB o incluso GB.
4. BÚSQUEDA AVANZADA: Usa índices para buscar rápidamente.
5. TRANSACCIONES: Asegura la integridad de los datos.
6. COMPATIBLE: Funciona en todos los navegadores modernos.
7. PERSISTENTE: Los datos permanecen hasta que se eliminan.

───────────────────────────────────────────────────────────
INDEXEDDB VS. LOCALSTORAGE
───────────────────────────────────────────────────────────

| Característica | IndexedDB | LocalStorage |
|----------------|-----------|--------------|
| Capacidad | Cientos de MB/GB | 5-10 MB |
| Tipos de datos | Cualquier objeto | Solo texto |
| Búsqueda | Con índices | Solo por clave |
| Asíncrono | Sí (no bloquea) | No (bloquea) |
| Transacciones | Sí | No |
| Complejidad | Media-alta | Muy baja |
| Uso | Datos grandes, offline | Preferencias, configuraciones |

───────────────────────────────────────────────────────────
ESTRUCTURA DE INDEXEDDB
───────────────────────────────────────────────────────────
+----------------------------------------------------------+
| BASE DE DATOS |
| (ej: "MiBaseDeDatos") |
| +--------------------------------------------------+ |
| | ALMACÉN DE OBJETOS 1 | |
| | (ej: "notas") | |
| | - Clave: id (auto-increment) | |
| | - Datos: {titulo, contenido, fecha} | |
| | - Índices: titulo, fecha | |
| +--------------------------------------------------+ |
| +--------------------------------------------------+ |
| | ALMACÉN DE OBJETOS 2 | |
| | (ej: "usuarios") | |
| | - Clave: email | |
| | - Datos: {nombre, email, edad} | |
| +--------------------------------------------------+ |
+----------------------------------------------------------+

PARTE 2: ABRIR UNA BASE DE DATOS
===============================================================================

Para usar IndexedDB, primero debes abrir (o crear) una base de datos.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

```javascript
const request = indexedDB.open('MiBaseDatos', 1);

request.onsuccess = function(event) {
    const db = event.target.result;
    console.log('Base de datos abierta:', db);
};

request.onerror = function(event) {
    console.error('Error al abrir la base de datos:', event.target.error);
};

request.onupgradeneeded = function(event) {
    const db = event.target.result;
    // Crear almacenes de objetos (solo cuando la base de datos se crea o actualiza)
    console.log('Actualización de la base de datos necesaria');
};
───────────────────────────────────────────────────────────
PARÁMETROS DE indexedDB.open()
───────────────────────────────────────────────────────────

nombre → Nombre de la base de datos.

version → Número de versión (número entero).

───────────────────────────────────────────────────────────
EVENTOS DE open()
───────────────────────────────────────────────────────────

onupgradeneeded → Se dispara cuando la base de datos se crea o la versión cambia.

onsuccess → Se dispara cuando la base de datos se abre correctamente.

onerror → Se dispara si ocurre un error.

            document.getElementById('estado').textContent = '❌ Error: ' + event.target.error.message;

        };

PARTE 3: CREAR ALMACENES DE OBJETOS (OBJECT STORES)
===============================================================================

Los almacenes de objetos son como las "tablas" de IndexedDB.

───────────────────────────────────────────────────────────
CREAR UN ALMACÉN DE OBJETOS
───────────────────────────────────────────────────────────

javascript
request.onupgradeneeded = function(event) {
    const db = event.target.result;

    // Crear almacén "notas" con clave auto-increment
    const store = db.createObjectStore('notas', {
        keyPath: 'id',
        autoIncrement: true
    });

    // Crear almacén "usuarios" con clave personalizada
    const store2 = db.createObjectStore('usuarios', {
        keyPath: 'email'
    });
};
───────────────────────────────────────────────────────────
OPCIONES DE createObjectStore()
───────────────────────────────────────────────────────────

keyPath → La propiedad del objeto que se usará como clave.

autoIncrement → Si la clave se genera automáticamente (true/false).

───────────────────────────────────────────────────────────
EJEMPLO: CREAR MÚLTIPLES ALMACENES
───────────────────────────────────────────────────────────

javascript
request.onupgradeneeded = function(event) {
    const db = event.target.result;

    // Almacén: notas
    if (!db.objectStoreNames.contains('notas')) {
        const store = db.createObjectStore('notas', {
            keyPath: 'id',
            autoIncrement: true
        });
        // Crear índices
        store.createIndex('por_titulo', 'titulo', { unique: false });
        store.createIndex('por_fecha', 'fecha', { unique: false });
    }

    // Almacén: usuarios
    if (!db.objectStoreNames.contains('usuarios')) {
        const store = db.createObjectStore('usuarios', {
            keyPath: 'email'
        });
        store.createIndex('por_nombre', 'nombre', { unique: false });
    }
};

PARTE 4: OPERACIONES CRUD – CREAR, LEER, ACTUALIZAR, ELIMINAR
===============================================================================

───────────────────────────────────────────────────────────
GUARDAR DATOS – add() (AÑADIR)
───────────────────────────────────────────────────────────

javascript
function agregarNota(db, nota) {
    const transaction = db.transaction(['notas'], 'readwrite');
    const store = transaction.objectStore('notas');

    const request = store.add(nota);

    request.onsuccess = function() {
        console.log('Nota agregada correctamente');
    };

    request.onerror = function() {
        console.error('Error al agregar nota:', request.error);
    };
}

// Uso
const nota = {
    titulo: 'Mi primera nota',
    contenido: 'Este es el contenido de la nota',
    fecha: new Date().toISOString()
};
agregarNota(db, nota);
───────────────────────────────────────────────────────────
LEER DATOS – get() (OBTENER POR CLAVE)
───────────────────────────────────────────────────────────

javascript
function obtenerNota(db, id) {
    const transaction = db.transaction(['notas'], 'readonly');
    const store = transaction.objectStore('notas');

    const request = store.get(id);

    request.onsuccess = function() {
        if (request.result) {
            console.log('Nota encontrada:', request.result);
            return request.result;
        } else {
            console.log('Nota no encontrada');
            return null;
        }
    };

    request.onerror = function() {
        console.error('Error al obtener nota:', request.error);
    };
}

// Uso
const nota = obtenerNota(db, 1);
───────────────────────────────────────────────────────────
ACTUALIZAR DATOS – put() (GUARDAR O ACTUALIZAR)
───────────────────────────────────────────────────────────

javascript
function actualizarNota(db, nota) {
    const transaction = db.transaction(['notas'], 'readwrite');
    const store = transaction.objectStore('notas');

    const request = store.put(nota);

        console.log('Nota actualizada correctamente');
    };

    request.onerror = function() {
        console.error('Error al actualizar nota:', request.error);
    };
}

// Uso
const notaActualizada = {
    id: 1,
    titulo: 'Mi primera nota (actualizada)',

    fecha: new Date().toISOString()

actualizarNota(db, notaActualizada);

PARTE 6: ÍNDICES EN INDEXEDDB
===============================================================================

Los índices permiten buscar datos de forma eficiente sin usar la clave principal.

───────────────────────────────────────────────────────────
CREAR UN ÍNDICE
───────────────────────────────────────────────────────────

javascript
request.onupgradeneeded = function(event) {
    const db = event.target.result;
    const store = db.createObjectStore('notas', {
        keyPath: 'id',
        autoIncrement: true
    });

    // Crear un índice en el campo 'titulo'
    store.createIndex('por_titulo', 'titulo', { unique: false });

    // Crear un índice en el campo 'fecha'
    store.createIndex('por_fecha', 'fecha', { unique: false });

    // Crear un índice único en el campo 'email'
    // store.createIndex('por_email', 'email', { unique: true });
};
───────────────────────────────────────────────────────────
BUSCAR CON ÍNDICES
───────────────────────────────────────────────────────────

javascript
function buscarPorTitulo(db, texto) {
    const transaction = db.transaction(['notas'], 'readonly');
    const store = transaction.objectStore('notas');
    const index = store.index('por_titulo');

    const range = IDBKeyRange.bound(texto, texto + '\uffff');
    const request = index.getAll(range);

    request.onsuccess = function() {
        console.log('Notas encontradas:', request.result);
    };
}
───────────────────────────────────────────────────────────
TIPOS DE RANGOS
───────────────────────────────────────────────────────────

IDBKeyRange.only(valor) → Exactamente igual a valor.

IDBKeyRange.lowerBound(valor) → Mayor o igual que valor.

IDBKeyRange.upperBound(valor) → Menor o igual que valor.

IDBKeyRange.bound(inicio, fin) → Entre inicio y fin.

PARTE 10: ERRORES COMUNES CON INDEXEDDB
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: NO ESPERAR A QUE LA BASE DE DATOS SE ABRA
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
const request = indexedDB.open('MiBase', 1);
// Intentar usar la base de datos inmediatamente
✅ Correcto:

javascript
request.onsuccess = function(event) {
    const db = event.target.result;
    // Usar la base de datos aquí
};
───────────────────────────────────────────────────────────
ERROR 2: INTENTAR CREAR UN ALMACÉN FUERA DE onupgradeneeded
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
request.onsuccess = function(event) {
    const db = event.target.result;
    db.createObjectStore('nuevo', { keyPath: 'id' }); // Error
};
✅ Correcto:

javascript
request.onupgradeneeded = function(event) {
    const db = event.target.result;
    db.createObjectStore('nuevo', { keyPath: 'id' });
};
───────────────────────────────────────────────────────────
ERROR 3: USAR TRANSACCIONES DESPUÉS DE QUE TERMINEN
───────────────────────────────────────────────────────────

Las transacciones se cierran automáticamente cuando terminan.

javascript
const transaction = db.transaction(['notas'], 'readonly');
const store = transaction.objectStore('notas');
const request = store.get(1);

// No usar la transacción después de que request termine
───────────────────────────────────────────────────────────
ERROR 4: NO MANEJAR ERRORES
───────────────────────────────────────────────────────────

Siempre maneja onerror para capturar problemas.

───────────────────────────────────────────────────────────
ERROR 5: GUARDAR DATOS NO SERIALIZABLES
───────────────────────────────────────────────────────────

Los datos deben ser serializables con Structured Clone Algorithm.

PARTE 11: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

Siempre espera a onsuccess antes de usar la base de datos.

Solo crea almacenes en onupgradeneeded.

Usa las transacciones dentro de su propio alcance.

Siempre maneja onerror.

Guarda solo datos serializables (objetos, arrays, números, strings).

Consejo del desarrollador

IndexedDB es una herramienta poderosa, pero no es mágica. Para grandes volúmenes de datos, usa índices para búsquedas eficientes. Para datos muy grandes, usa almacenamiento en trozos (chunks). Y siempre maneja errores correctamente.

GLOSARIO DE TÉRMINOS
===============================================================================

IndexedDB: Base de datos NoSQL en el navegador.

Object Store: Almacén de objetos (similar a una tabla).

Transaction: Operación que agrupa varias acciones.

KeyPath: Propiedad que se usa como clave.

AutoIncrement: Generación automática de claves.

Index: Estructura para búsquedas eficientes.

CRUD: Create, Read, Update, Delete.

Asíncrono: Las operaciones no bloquean la ejecución.
