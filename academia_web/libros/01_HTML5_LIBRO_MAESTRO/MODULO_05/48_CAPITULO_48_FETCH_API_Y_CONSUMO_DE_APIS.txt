PARTE 1: ¿QUÉ ES FETCH API?
===============================================================================

Fetch API es una interfaz de JavaScript para realizar solicitudes HTTP y obtener recursos.

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

1. BASADA EN PROMESAS: Usa promesas para manejar respuestas.
2. MODERNA: Reemplaza a XMLHttpRequest (más antigua).
3. FLEXIBLE: Soporta GET, POST, PUT, DELETE y otros métodos.
4. NATIVA: Está integrada en todos los navegadores modernos.
5. SIMPLE: Sintaxis limpia y fácil de usar.

───────────────────────────────────────────────────────────
SINTAXIS BÁSICA
───────────────────────────────────────────────────────────

```javascript
fetch(url)
    .then(response => response.json())
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error('Error:', error);
    });
───────────────────────────────────────────────────────────
USANDO async/await
───────────────────────────────────────────────────────────

javascript
async function obtenerDatos() {
    try {
        const response = await fetch(url);
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Error:', error);
    }
}

PARTE 2: SOLICITUDES GET – OBTENER DATOS
===============================================================================

GET es el método más común. Se usa para obtener datos de un servidor.

───────────────────────────────────────────────────────────
EJEMPLO: OBTENER DATOS DE UNA API
───────────────────────────────────────────────────────────

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Fetch API - GET</title>
</head>
<body>
    <h1>📡 Consumiendo una API</h1>

    <button onclick="obtenerUsuarios()">👥 Obtener usuarios</button>

    <div id="resultado"></div>

    <script>
        function obtenerUsuarios() {
            const resultado = document.getElementById('resultado');
            resultado.innerHTML = '🔄 Cargando usuarios...';

            // API pública de prueba (JSONPlaceholder)
            const url = 'https://jsonplaceholder.typicode.com/users';

            fetch(url)
                .then(response => {
                    if (!response.ok) {
                        throw new Error('Error en la respuesta: ' + response.status);
                    }
                    return response.json();
                })
                .then(usuarios => {
                    let html = '<h2>👥 Lista de usuarios</h2><ul>';
                    usuarios.forEach(usuario => {
                        html += `<li><strong>${usuario.name}</strong> - ${usuario.email}</li>`;
                    });
                    html += '</ul>';
                    resultado.innerHTML = html;
                })
                .catch(error => {
                    resultado.innerHTML = '❌ Error: ' + error.message;
                });
        }
    </script>
</body>
</html>
───────────────────────────────────────────────────────────
EJEMPLO CON async/await
───────────────────────────────────────────────────────────

javascript
async function obtenerUsuariosAsync() {
    const resultado = document.getElementById('resultado');
    resultado.innerHTML = '🔄 Cargando usuarios...';

    try {
        const url = 'https://jsonplaceholder.typicode.com/users';
        const response = await fetch(url);

            throw new Error('Error: ' + response.status);

        usuarios.forEach(usuario => {
            html += `<li><strong>${usuario.name}</strong> - ${usuario.email}</li>`;
        });
        html += '</ul>';
        resultado.innerHTML = html;

    } catch (error) {
        resultado.innerHTML = '❌ Error: ' + error.message;
    }
}

PARTE 3: MANEJO DE ERRORES EN FETCH
===============================================================================

Es importante manejar errores correctamente.

───────────────────────────────────────────────────────────
TIPOS DE ERRORES
───────────────────────────────────────────────────────────

Error de red: No hay conexión a Internet.

Error HTTP: 404 (No encontrado), 500 (Error del servidor), etc.

Error de parseo: La respuesta no es JSON válido.

───────────────────────────────────────────────────────────
MANEJO COMPLETO DE ERRORES
───────────────────────────────────────────────────────────

javascript
async function obtenerDatosSeguro(url) {
    try {
        const response = await fetch(url);

        // Verificar si la respuesta es exitosa (200-299)
        if (!response.ok) {
            throw new Error(`HTTP Error: ${response.status} - ${response.statusText}`);
        }

        const data = await response.json();
        return data;

    } catch (error) {
        if (error.name === 'TypeError' && error.message.includes('Failed to fetch')) {
            console.error('❌ Error de red: No hay conexión a Internet');
        } else {
            console.error('❌ Error:', error.message);
        }
        throw error; // Re-lanzar para que el llamador lo maneje
    }
}
───────────────────────────────────────────────────────────
CÓDIGOS DE ESTADO HTTP COMUNES
───────────────────────────────────────────────────────────

Código	Significado
200	OK (éxito)
201	Created (creado)
204	No Content (eliminado)
400	Bad Request (solicitud incorrecta)
401	Unauthorized (no autorizado)
403	Forbidden (prohibido)
404	Not Found (no encontrado)
500	Internal Server Error (error del servidor)

PARTE 4: SOLICITUDES POST – ENVIAR DATOS
===============================================================================

POST se usa para enviar datos al servidor (crear recursos).

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

javascript
fetch(url, {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify(datos)
})
.then(response => response.json())
.then(data => console.log('Creado:', data))
.catch(error => console.error('Error:', error));
───────────────────────────────────────────────────────────
EJEMPLO: CREAR UN NUEVO USUARIO
───────────────────────────────────────────────────────────

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Fetch API - POST</title>
</head>
<body>
    <h1>📤 Enviar datos con POST</h1>

    <input type="text" id="nombre" placeholder="Nombre">
    <input type="text" id="email" placeholder="Email">
    <button onclick="crearUsuario()">➕ Crear usuario</button>

    <div id="resultado"></div>

    <script>
        async function crearUsuario() {
            const nombre = document.getElementById('nombre').value.trim();
            const email = document.getElementById('email').value.trim();

            if (!nombre || !email) {
                alert('Completa todos los campos');
                return;
            }

            const resultado = document.getElementById('resultado');
            resultado.innerHTML = '🔄 Enviando datos...';

            const datos = {
                name: nombre,
                email: email,
                username: nombre.toLowerCase().replace(' ', '')
            };

            try {
                const response = await fetch('https://jsonplaceholder.typicode.com/users', {
                    method: 'POST',
                    headers: {
                        'Content-Type': 'application/json'
                    },
                    body: JSON.stringify(datos)
                });

                if (!response.ok) {
                    throw new Error('Error: ' + response.status);
                }

                const data = await response.json();
                resultado.innerHTML = `
                    ✅ Usuario creado correctamente<br>
                    <strong>ID:</strong> ${data.id}<br>
                    <strong>Nombre:</strong> ${data.name}<br>
                    <strong>Email:</strong> ${data.email}
                `;

                document.getElementById('nombre').value = '';
                document.getElementById('email').value = '';

            } catch (error) {
                resultado.innerHTML = '❌ Error: ' + error.message;
            }
        }
    </script>
</body>
</html>

───────────────────────────────────────────────────────────
EJEMPLO: ACTUALIZAR UN USUARIO
───────────────────────────────────────────────────────────

javascript
async function actualizarUsuario(id, nombre, email) {
    const datos = { name: nombre, email: email };

    try {
        const response = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`, {
            method: 'PUT',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(datos)
        });

        if (!response.ok) {
            throw new Error('Error: ' + response.status);
        }

        const data = await response.json();
        console.log('✅ Usuario actualizado:', data);
        return data;

    } catch (error) {
        console.error('❌ Error:', error.message);
    }
}

async function eliminarUsuario(id) {
    try {
        const response = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`, {
            method: 'DELETE'
        });

        if (response.ok) {
            console.log(`✅ Usuario ${id} eliminado correctamente`);
            return true;
        } else {
            throw new Error('Error: ' + response.status);
        }

    } catch (error) {
        console.error('❌ Error:', error.message);
        return false;
    }
}

<body>

    <h1>🌤️ Clima en tu ciudad</h1>

    <div>
        <input type="text" id="ciudad" placeholder="Ej: Madrid, Londres, México" value="Madrid">
        <button onclick="obtenerClima()">🔍 Buscar clima</button>
    </div>

    <div id="resultado"></div>

    <script>
        // NOTA: Necesitas una API Key de OpenWeatherMap (gratuita)
        // Regístrate en https://openweathermap.org/api
        const API_KEY = 'TU_API_KEY_AQUI'; // Reemplaza con tu API Key

        async function obtenerClima() {
            const ciudad = document.getElementById('ciudad').value.trim();
            if (!ciudad) {
                alert('Escribe una ciudad');
                return;
            }

            const resultado = document.getElementById('resultado');
            resultado.innerHTML = '🔄 Buscando clima...';

            try {
                const url = `https://api.openweathermap.org/data/2.5/weather?q=${encodeURIComponent(ciudad)}&appid=${API_KEY}&units=metric&lang=es`;

                const response = await fetch(url);

                if (!response.ok) {
                    if (response.status === 404) {
                        throw new Error('Ciudad no encontrada. Verifica el nombre.');
                    }
                    throw new Error('Error: ' + response.status);
                }

                const data = await response.json();

                const temp = Math.round(data.main.temp);
                const descripcion = data.weather[0].description;
                const icono = data.weather[0].icon;
                const ciudadNombre = data.name;
                const pais = data.sys.country;

                resultado.innerHTML = `
                    <div class="clima">
                        <h2>${ciudadNombre}, ${pais}</h2>
                        <img src="https://openweathermap.org/img/wn/${icono}@2x.png" alt="${descripcion}">
                        <div class="temperatura">${temp}°C</div>
                        <div class="descripcion">${descripcion}</div>
                        <p class="ciudad">Humedad: ${data.main.humidity}% | Viento: ${data.wind.speed} m/s</p>
                    </div>
                `;

            } catch (error) {
                resultado.innerHTML = `<div class="error">❌ ${error.message}</div>`;
            }
        }

        // Cargar clima al abrir la página
        window.onload = obtenerClima;
    </script>

</body>
</html>

javascript
if (!response.ok) {

}

MANEJA ERRORES CON try/catch
───────────────────────────────────────────────────────────

javascript
try {
    const response = await fetch(url);
    // ...
} catch (error) {
    // Manejar error de red o de parseo
}
───────────────────────────────────────────────────────────

USA async/await PARA CÓDIGO MÁS LIMPIO
───────────────────────────────────────────────────────────

javascript
async function obtenerDatos() {
    // Código más legible que con .then()
}
───────────────────────────────────────────────────────────

ENVÍA HEADERS CORRECTOS
───────────────────────────────────────────────────────────

javascript
const response = await fetch(url, {
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer token123'
    }
});
───────────────────────────────────────────────────────────

USA encodeURIComponent() PARA PARÁMETROS EN URL
───────────────────────────────────────────────────────────

javascript
const ciudad = 'Nueva York';
const url = `https://api.com/weather?q=${encodeURIComponent(ciudad)}`;
───────────────────────────────────────────────────────────

NO EXPONGAS API KEYS EN EL CÓDIGO FRONTEND
───────────────────────────────────────────────────────────

Las API Keys deben estar en el servidor (backend).

Para pruebas, puedes usarlas, pero no las subas a repositorios públicos.

PARTE 11: ERRORES COMUNES CON FETCH
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR AWAIT EN async/await
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
const response = fetch(url); // Falta await
✅ Correcto:

javascript
const response = await fetch(url);
───────────────────────────────────────────────────────────
ERROR 2: NO VERIFICAR response.ok
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
const data = await response.json(); // Si falla, error
✅ Correcto:

javascript
if (!response.ok) {
    throw new Error('Error: ' + response.status);
}
const data = await response.json();
───────────────────────────────────────────────────────────
ERROR 3: NO MANEJAR ERRORES DE RED
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
const data = await fetch(url).then(r => r.json());
✅ Correcto:

javascript
try {
    const response = await fetch(url);
    // ...
} catch (error) {
    // Manejar error de red o parseo
}
───────────────────────────────────────────────────────────
ERROR 4: ENVIAR DATOS SIN JSON.stringify()
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
body: datos // Error: no es texto
✅ Correcto:

javascript
body: JSON.stringify(datos)
───────────────────────────────────────────────────────────
ERROR 5: OLVIDAR HEADERS EN POST/PUT
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
fetch(url, {
    method: 'POST',
    body: JSON.stringify(datos)
});

Pregunta del Profesor Búho

¿Qué crees que pasaría si una API cambia la estructura de sus datos? ¿Cómo debería manejar tu aplicación ese cambio?

Consejo del desarrollador

Las APIs pueden cambiar. Siempre maneja la posibilidad de que la respuesta no tenga el formato esperado. Usa try/catch y valida los datos antes de usarlos. Además, siempre usa HTTPS para las solicitudes y nunca expongas API keys en el código frontend.
