PARTE 1: ¿QUÉ SON LOS WEB WORKERS?
===============================================================================

Un Web Worker es un script que se ejecuta en un hilo separado del hilo principal de JavaScript.

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

1. MULTI-HILO: Se ejecuta en un hilo separado.
2. NO BLOQUEA: La interfaz sigue siendo receptiva.
3. SIN DOM: No puede acceder al DOM (document, window, etc.).
4. COMUNICACIÓN: Se comunica con el hilo principal mediante mensajes.
5. INDEPENDIENTE: Cada worker tiene su propio contexto.
6. PESADO: Ideal para tareas que consumen mucho tiempo.

───────────────────────────────────────────────────────────
USOS COMUNES
───────────────────────────────────────────────────────────

- Cálculos matemáticos complejos.
- Procesamiento de imágenes (filtros, redimensionado).
- Cifrado y descifrado de datos.
- Procesamiento de grandes cantidades de datos (JSON, CSV).
- Renderizado de gráficos complejos.
- Operaciones de inteligencia artificial (inferencia).

───────────────────────────────────────────────────────────
TIPOS DE WORKERS
───────────────────────────────────────────────────────────

1. **Dedicated Worker**: Worker dedicado a una sola página.
2. **Shared Worker**: Worker compartido entre varias páginas del mismo origen.
3. **Service Worker**: Worker que actúa como proxy de red (para PWA).

En este capítulo nos enfocaremos en los **Dedicated Workers**, los más comunes.

PARTE 2: CREAR UN WEB WORKER
===============================================================================

Para crear un Web Worker, necesitas un archivo JavaScript separado.

───────────────────────────────────────────────────────────
ARCHIVO DEL WORKER (worker.js)
───────────────────────────────────────────────────────────

```javascript
// worker.js - Este código se ejecuta en un hilo separado

// Escuchar mensajes del hilo principal
self.onmessage = function(event) {
    const dato = event.data;
    console.log('Worker recibió:', dato);

    // Procesar el dato (simular trabajo pesado)
    const resultado = dato * 2;

    // Enviar resultado al hilo principal
    self.postMessage(resultado);
};
───────────────────────────────────────────────────────────
CÓDIGO PRINCIPAL (index.html)
───────────────────────────────────────────────────────────

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Web Worker - Básico</title>
</head>
<body>
    <h1>🧠 Web Worker - Básico</h1>

    <p>El worker procesa datos en segundo plano.</p>

    <button onclick="enviarAlWorker()">Enviar dato al worker</button>
    <button onclick="terminarWorker()">Terminar worker</button>

    <pre id="resultado"></pre>

    <script>
        // Crear el worker (debe estar en un archivo separado)
        const worker = new Worker('worker.js');

        // Escuchar mensajes del worker
        worker.onmessage = function(event) {
            document.getElementById('resultado').textContent = '✅ Resultado del worker: ' + event.data;
        };

        // Manejar errores del worker
        worker.onerror = function(error) {
            document.getElementById('resultado').textContent = '❌ Error en worker: ' + error.message;
        };

        function enviarAlWorker() {
            const numero = Math.floor(Math.random() * 100) + 1;
            document.getElementById('resultado').textContent = '⏳ Enviando: ' + numero + '...';
            worker.postMessage(numero);
        }

        function terminarWorker() {
            worker.terminate();
            document.getElementById('resultado').textContent = '⏹ Worker terminado';
        }
    </script>
</body>
</html>

PARTE 3: COMUNICACIÓN ENTRE EL HILO PRINCIPAL Y EL WORKER
===============================================================================

La comunicación entre el hilo principal y el worker se hace mediante mensajes.

───────────────────────────────────────────────────────────
ENVIAR MENSAJES – postMessage()
───────────────────────────────────────────────────────────

javascript
// Desde el hilo principal al worker
worker.postMessage('Hola worker');

// Desde el worker al hilo principal
self.postMessage('Hola principal');
───────────────────────────────────────────────────────────
RECIBIR MENSAJES – onmessage
───────────────────────────────────────────────────────────

javascript
// En el hilo principal
worker.onmessage = function(event) {
    console.log('Mensaje del worker:', event.data);
};

// En el worker
self.onmessage = function(event) {
    console.log('Mensaje del principal:', event.data);
};
───────────────────────────────────────────────────────────
EJEMPLO: ENVÍO Y RECEPCIÓN DE DATOS
───────────────────────────────────────────────────────────

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Web Worker - Comunicación</title>
</head>
<body>
    <h1>📨 Comunicación con Web Worker</h1>

    <input type="text" id="mensaje" placeholder="Escribe un mensaje...">
    <button onclick="enviarMensaje()">Enviar</button>

    <div id="respuestas"></div>

    <script>
        const worker = new Worker('worker.js');

        worker.onmessage = function(event) {
            const div = document.createElement('div');
            div.textContent = '🦉 Worker respondió: ' + event.data;
            document.getElementById('respuestas').appendChild(div);
        };

        function enviarMensaje() {
            const input = document.getElementById('mensaje');
            const mensaje = input.value.trim();
            if (mensaje === '') return;

            worker.postMessage(mensaje);
            input.value = '';
        }
    </script>
</body>
</html>
javascript
// worker.js
self.onmessage = function(event) {
    const mensaje = event.data;

    // Procesar el mensaje (simular trabajo)
    const respuesta = 'Recibí: "' + mensaje + '" (procesado en worker)';

    // Enviar respuesta
    self.postMessage(respuesta);
};

PARTE 4: TRABAJO PESADO – CALCULADORA EN SEGUNDO PLANO
===============================================================================

Este es un ejemplo de una tarea pesada que no bloquea la interfaz.

───────────────────────────────────────────────────────────
CÓDIGO COMPLETO
───────────────────────────────────────────────────────────

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Web Worker - Calculadora pesada</title>
    <style>
        body { font-family: Arial; max-width: 600px; margin: 40px auto; padding: 20px; background: #0d1117; color: #c9d1d9; }
        h1 { color: #58a6ff; }
        button { padding: 10px 20px; border: none; border-radius: 6px; background: #238636; color: white; cursor: pointer; margin: 5px; }
        button:hover { background: #2ea043; }
        button.danger { background: #e53e3e; }
        button.danger:hover { background: #c53030; }
        .resultado { background: #161b22; border: 1px solid #30363d; padding: 15px; border-radius: 8px; margin: 15px 0; }
        .estado { color: #8b949e; font-size: 0.9em; }
        .contador { background: #1a202c; padding: 10px; border-radius: 8px; text-align: center; font-size: 1.2em; }
    </style>
</head>
<body>

    <h1>🧮 Calculadora en segundo plano</h1>

    <div class="estado" id="estado">🟢 Worker listo</div>

    <div class="contador" id="contador">Clics en la interfaz: 0</div>

    <button onclick="iniciarCalculo()">🔢 Calcular (con worker)</button>
    <button onclick="calcularSinWorker()">🔢 Calcular (sin worker)</button>
    <button onclick="incrementarContador()">🖱️ Hacer clic</button>
    <button class="danger" onclick="terminarWorker()">⏹ Terminar worker</button>

    <div class="resultado" id="resultado">
        Esperando cálculo...
    </div>

    <script>
        let contadorClics = 0;
        let worker = null;

        // ============================================================ */
        // CREAR WORKER                                                  */
        // ============================================================ */

        function crearWorker() {
            if (worker) return;
            worker = new Worker('calculadora-worker.js');

            worker.onmessage = function(event) {
                document.getElementById('resultado').textContent = '✅ Resultado: ' + event.data;
                document.getElementById('estado').textContent = '🟢 Worker listo';
            };

            worker.onerror = function(error) {
                document.getElementById('resultado').textContent = '❌ Error: ' + error.message;
                document.getElementById('estado').textContent = '🔴 Error en worker';
            };
        }

        // ============================================================ */
        // CALCULAR CON WORKER                                          */
        // ============================================================ */

        function iniciarCalculo() {
            crearWorker();

            if (!worker) {
                document.getElementById('resultado').textContent = '❌ No se pudo crear el worker';
                return;
            }

            document.getElementById('resultado').textContent = '⏳ Calculando en segundo plano...';
            document.getElementById('estado').textContent = '🟡 Worker trabajando';

            // Enviar el cálculo al worker
            worker.postMessage({
                tipo: 'calcular',
                valor: 100000000 // Número grande para simular trabajo pesado
            });
        }

        // ============================================================ */
        // CALCULAR SIN WORKER (BLOQUEA LA INTERFAZ)                    */
        // ============================================================ */

        function calcularSinWorker() {
            document.getElementById('resultado').textContent = '⏳ Calculando (sin worker)...';

            // Esto bloquea la interfaz
            let suma = 0;
            for (let i = 0; i < 1000000000; i++) {
                suma += i;
            }

            document.getElementById('resultado').textContent = '✅ Resultado (sin worker): ' + suma;
        }

        // ============================================================ */
        // CONTADOR DE INTERACCIÓN                                      */
        // ============================================================ */

        function incrementarContador() {
            contadorClics++;
            document.getElementById('contador').textContent = 'Clics en la interfaz: ' + contadorClics;
        }

        // ============================================================ */
        // TERMINAR WORKER                                              */
        // ============================================================ */

        function terminarWorker() {
            if (worker) {
                worker.terminate();
                worker = null;
                document.getElementById('estado').textContent = '⏹ Worker terminado';
                document.getElementById('resultado').textContent = '⏹ Worker terminado manualmente';
            }
        }

        // Crear worker al cargar la página
        crearWorker();
    </script>

</body>
</html>
javascript
// calculadora-worker.js
self.onmessage = function(event) {
    const datos = event.data;

    if (datos.tipo === 'calcular') {
        const n = datos.valor;

        // Simular trabajo pesado (calcular la suma de 1 a n)
        let suma = 0;
        for (let i = 0; i < n; i++) {
            suma += i;
        }

        // Enviar el resultado al hilo principal
        self.postMessage(suma);
    }
};
───────────────────────────────────────────────────────────
EXPLICACIÓN DEL EJEMPLO
───────────────────────────────────────────────────────────

Con worker: El cálculo se ejecuta en segundo plano. La interfaz sigue respondiendo. Puedes hacer clic en "Hacer clic" mientras se calcula.

Sin worker: El cálculo bloquea la interfaz. No puedes hacer clic en nada hasta que termine.

Contador: Muestra que la interfaz sigue respondiendo mientras el worker trabaja.

PARTE 5: MANEJO DE ERRORES EN WORKERS
===============================================================================

Los workers pueden fallar. Debes manejar los errores.

───────────────────────────────────────────────────────────
onerror – CAPTURAR ERRORES
───────────────────────────────────────────────────────────

javascript
worker.onerror = function(error) {
    console.error('Error en worker:', error.message);
    console.error('Archivo:', error.filename);
    console.error('Línea:', error.lineno);
};
───────────────────────────────────────────────────────────
EJEMPLO COMPLETO DE MANEJO DE ERRORES
───────────────────────────────────────────────────────────

javascript
// En el hilo principal
const worker = new Worker('worker.js');

worker.onmessage = function(event) {
    console.log('Resultado:', event.data);
};

worker.onerror = function(error) {
    console.error('❌ Error en el worker:');
    console.error('  Mensaje:', error.message);
    console.error('  Archivo:', error.filename);
    console.error('  Línea:', error.lineno);

    // Mostrar mensaje al usuario
    document.getElementById('error').textContent = '⚠️ El worker tuvo un error. Intenta de nuevo.';
};
javascript
// worker.js - con error intencional
self.onmessage = function(event) {
    const dato = event.data;

    // Error: intentar acceder a una variable no definida
    const resultado = dato + variableNoDefinida; // Esto genera un error

    self.postMessage(resultado);
};

PARTE 6: TERMINAR WORKERS – terminate()
===============================================================================

Cuando ya no necesitas un worker, debes terminarlo para liberar recursos.

───────────────────────────────────────────────────────────
TERMINAR UN WORKER
───────────────────────────────────────────────────────────

javascript
// En el hilo principal
worker.terminate();
worker = null; // Liberar la referencia
───────────────────────────────────────────────────────────
EL WORKER PUEDE TERMINARSE A SÍ MISMO
───────────────────────────────────────────────────────────

javascript
// En el worker
self.close();
───────────────────────────────────────────────────────────
EJEMPLO DE TERMINADO AUTOMÁTICO
───────────────────────────────────────────────────────────

javascript
// worker.js
self.onmessage = function(event) {
    const resultado = event.data * 2;

    // Enviar resultado
    self.postMessage(resultado);

    // Terminar el worker después de enviar
    self.close();
};

PARTE 7: CASOS DE USO REALES DE WEB WORKERS
===============================================================================

───────────────────────────────────────────────────────────

PROCESAMIENTO DE IMÁGENES
───────────────────────────────────────────────────────────

Aplicar filtros, redimensionar o convertir imágenes sin bloquear la interfaz.

javascript
// worker.js
self.onmessage = function(event) {
    const imageData = event.data;
    // Aplicar filtro (ej: blanco y negro)
    const data = imageData.data;
    for (let i = 0; i < data.length; i += 4) {
        const avg = (data[i] + data[i+1] + data[i+2]) / 3;
        data[i] = avg;
        data[i+1] = avg;
        data[i+2] = avg;
    }
    self.postMessage(imageData);
};
───────────────────────────────────────────────────────────

PROCESAMIENTO DE DATOS GRANDES
───────────────────────────────────────────────────────────

Analizar grandes archivos JSON o CSV sin bloquear la interfaz.

javascript
// worker.js
self.onmessage = function(event) {
    const datos = event.data;
    // Procesar miles o millones de registros
    const resultado = datos.filter(item => item.valor > 100);
    self.postMessage(resultado);
};
───────────────────────────────────────────────────────────

CIFRADO Y SEGURIDAD
───────────────────────────────────────────────────────────

Cifrar o descifrar datos (contraseñas, archivos) en segundo plano.

javascript
// worker.js
self.onmessage = async function(event) {
    const texto = event.data;
    // Cifrar con Web Crypto API (en el worker)
    const encoder = new TextEncoder();
    const data = encoder.encode(texto);
    const key = await crypto.subtle.generateKey(...);
    const cifrado = await crypto.subtle.encrypt(...);
    self.postMessage(cifrado);
};
───────────────────────────────────────────────────────────

COMPRESIÓN Y DESCOMPRESIÓN
───────────────────────────────────────────────────────────

Comprimir archivos grandes antes de subirlos al servidor.

javascript
// worker.js (usando una librería de compresión)
self.onmessage = function(event) {
    const datos = event.data;
    // Comprimir con pako.js (librería de compresión)
    const comprimido = pako.deflate(datos);
    self.postMessage(comprimido);
};

PARTE 8: BUENAS PRÁCTICAS CON WEB WORKERS
===============================================================================

───────────────────────────────────────────────────────────

USA WORKERS PARA TAREAS PESADAS
───────────────────────────────────────────────────────────

No uses workers para tareas ligeras. El overhead de comunicación puede ser mayor que el beneficio.

───────────────────────────────────────────────────────────

TERMINA LOS WORKERS CUANDO NO LOS NECESITAS
───────────────────────────────────────────────────────────

javascript
// Cuando el usuario cierra la página o ya no necesita el worker
worker.terminate();
───────────────────────────────────────────────────────────

MANEJA ERRORES
───────────────────────────────────────────────────────────

javascript
worker.onerror = function(error) {
    // Informar al usuario
    // Intentar recuperar o reiniciar el worker
};
───────────────────────────────────────────────────────────

NO USES WORKERS PARA ACCEDER AL DOM
───────────────────────────────────────────────────────────

Los workers NO pueden acceder al DOM. Si necesitas actualizar la interfaz, envía un mensaje al hilo principal.

───────────────────────────────────────────────────────────

ENVÍA DATOS EFICIENTEMENTE
───────────────────────────────────────────────────────────

Los datos enviados con postMessage() se copian (estructura clonada). Para datos grandes, usa Transferable Objects.

javascript
// Transferir datos (más eficiente)
const buffer = new ArrayBuffer(1024);
worker.postMessage(buffer, [buffer]); // El buffer se transfiere, no se copia

PARTE 10: ERRORES COMUNES CON WEB WORKERS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: INTENTAR ACCEDER AL DOM DESDE EL WORKER
───────────────────────────────────────────────────────────

❌ Incorrecto (en worker.js):

javascript
document.getElementById('resultado').textContent = 'Hola';
// Error: document is not defined
✅ Correcto (en worker.js):

javascript
self.postMessage('Resultado procesado');
// El hilo principal actualiza el DOM
───────────────────────────────────────────────────────────
ERROR 2: NO MANEJAR ERRORES
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
const worker = new Worker('worker.js');
// Sin manejo de errores
✅ Correcto:

javascript
const worker = new Worker('worker.js');
worker.onerror = function(error) {
    console.error('Error en worker:', error);
};
───────────────────────────────────────────────────────────
ERROR 3: NO TERMINAR LOS WORKERS
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
// Crear worker pero nunca terminarlo
const worker = new Worker('worker.js');
✅ Correcto:

javascript
// Terminar cuando ya no se necesita
worker.terminate();
───────────────────────────────────────────────────────────
ERROR 4: ENVIAR DATOS NO SERIALIZABLES
───────────────────────────────────────────────────────────

❌ Incorrecto:

javascript
// Enviar una función o un elemento DOM
worker.postMessage(document.getElementById('elemento'));
// Error: No se puede clonar un elemento DOM
✅ Correcto:

javascript
// Enviar solo datos serializables (números, strings, objetos, arrays)
worker.postMessage({ id: 123, nombre: 'Ana' });
───────────────────────────────────────────────────────────
ERROR 5: USAR WORKERS EN HTTP (NO HTTPS)
───────────────────────────────────────────────────────────

En algunos navegadores, los workers solo funcionan en HTTPS (o localhost).

===============================================================================
PARTE 11: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

Los workers no pueden acceder al DOM. Envía mensajes al hilo principal.

Siempre maneja errores con onerror.

Termina los workers con terminate() cuando ya no los necesites.

Solo envía datos serializables (números, strings, arrays, objetos simples).

Usa HTTPS en producción (o localhost en desarrollo).

Experimento: Worker en la consola

Abre la consola del navegador (F12 → Console) y prueba:

javascript
// Crear un worker desde la consola
const workerCode = `
    self.onmessage = function(e) {
        const resultado = e.data * 2;
        self.postMessage(resultado);
    };
`;

const blob = new Blob([workerCode], { type: 'application/javascript' });
const url = URL.createObjectURL(blob);
const worker = new Worker(url);

worker.onmessage = function(e) {
    console.log('Resultado:', e.data);
};

// Enviar un número al worker
worker.postMessage(42);
// Resultado: 84

Pregunta del Profesor Búho

¿Qué crees que pasaría si usaras un Web Worker para cada tarea pesada en tu aplicación? ¿Podrías tener demasiados workers? ¿Cuántos son demasiados?

Consejo del desarrollador

Los Web Workers son una herramienta poderosa, pero no son una bala de plata. Usa workers para tareas que realmente necesitan procesamiento pesado. Para tareas ligeras, el overhead de la comunicación puede ser mayor que el beneficio.

Además, no crees más workers de los necesarios. Cada worker consume memoria y recursos del sistema.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

Web Worker: Hilo de ejecución en segundo plano.

Dedicated Worker: Worker dedicado a una sola página.

Shared Worker: Worker compartido entre páginas.

Service Worker: Worker para redes y caché.

Hilo principal: El hilo que maneja la interfaz de usuario.

postMessage(): Método para enviar mensajes.

onmessage: Evento para recibir mensajes.

terminate(): Método para terminar un worker.

close(): Método para que un worker se termine a sí mismo.

Transferable Objects: Objetos que se transfieren (no se copian) para mayor eficiencia.

¿SABÍAS QUE...?
===============================================================================

Los Web Workers fueron introducidos en HTML5 en 2010.

Antes de los workers, los desarrolladores usaban técnicas como setTimeout() o setInterval() para simular multitarea, pero estas técnicas no eran verdaderamente paralelas.

Los workers trajeron verdadera concurrencia a JavaScript.
