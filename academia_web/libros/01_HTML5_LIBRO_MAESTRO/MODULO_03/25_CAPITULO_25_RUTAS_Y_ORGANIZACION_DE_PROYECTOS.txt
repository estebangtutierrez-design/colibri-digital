PARTE 1: RUTAS ABSOLUTAS VS. RUTAS RELATIVAS
===============================================================================

Una ruta es la dirección de un archivo.
Hay dos tipos: absolutas y relativas.

───────────────────────────────────────────────────────────
RUTAS ABSOLUTAS (ABSOLUTE PATHS)
───────────────────────────────────────────────────────────

Una ruta absoluta es la dirección COMPLETA de un archivo.
Incluye el protocolo, el dominio y la ruta completa.

EJEMPLOS:
https://misitio.com/imagenes/logo.png
https://misitio.com/css/estilos.css
https://www.google.com/images/logo.png

Características:
- Comienza con http:// o https://.
- Incluye el dominio completo.
- Funciona desde cualquier lugar.
- Especifica la ubicación exacta.

Cuándo usar rutas absolutas:
- Enlaces a sitios externos (Google, Wikipedia, etc.).
- Recursos de CDN (Content Delivery Network).
- Cuando necesitas compartir una URL exacta.

───────────────────────────────────────────────────────────
RUTAS RELATIVAS (RELATIVE PATHS)
───────────────────────────────────────────────────────────

Una ruta relativa es la dirección de un archivo RELATIVA al archivo actual.

EJEMPLOS:
imagenes/logo.png          (Misma carpeta, entrando a imagenes/)
../css/estilos.css         (Subir una carpeta y entrar a css/)
../../index.html           (Subir dos carpetas)

Características:
- No incluye el dominio.
- Se basa en la ubicación del archivo actual.
- Es más portable (puedes mover el proyecto sin romper enlaces).
- Es más corta y limpia.

Cuándo usar rutas relativas:
- TODOS los recursos internos de tu proyecto.
- Imágenes, CSS, JavaScript, documentos.
- Enlaces entre páginas de tu sitio.

───────────────────────────────────────────────────────────
LA REGLA DE ORO
───────────────────────────────────────────────────────────

Siempre usa RUTAS RELATIVAS para los recursos de tu propio proyecto.

Usa RUTAS ABSOLUTAS solo para recursos externos (otros sitios web, CDN, etc.).

===============================================================================
PARTE 2: SINTAXIS DE RUTAS RELATIVAS
===============================================================================

───────────────────────────────────────────────────────────
./ → MISMA CARPETA (MISMA QUE EL ARCHIVO ACTUAL)
───────────────────────────────────────────────────────────

./ es opcional, pero ayuda a ser explícito.

EJEMPLOS:
<img src="./imagenes/logo.png" alt="Logo">
<link rel="stylesheet" href="./css/estilos.css">

Es equivalente a:
<img src="imagenes/logo.png" alt="Logo">
<link rel="stylesheet" href="css/estilos.css">

───────────────────────────────────────────────────────────
../ → SUBIR UNA CARPETA (CARPETA PADRE)
───────────────────────────────────────────────────────────

../ significa "sube un nivel".

EJEMPLOS:
<!-- Desde una página en /carpeta/pagina.html -->
<img src="../imagenes/logo.png" alt="Logo">
<!-- Va a /imagenes/logo.png -->

<link rel="stylesheet" href="../css/estilos.css">
<!-- Va a /css/estilos.css -->

───────────────────────────────────────────────────────────
../../ → SUBIR DOS CARPETAS
───────────────────────────────────────────────────────────

../../ significa "sube dos niveles".

EJEMPLOS:
<!-- Desde una página en /carpeta/subcarpeta/pagina.html -->
<img src="../../imagenes/logo.png" alt="Logo">
<!-- Va a /imagenes/logo.png -->

───────────────────────────────────────────────────────────
/ → RUTA ABSOLUTA DESDE LA RAÍZ DEL SITIO
───────────────────────────────────────────────────────────

/ al inicio significa "desde la raíz del sitio".

EJEMPLOS:
<img src="/imagenes/logo.png" alt="Logo">
<!-- Va a http://misitio.com/imagenes/logo.png -->

<link rel="stylesheet" href="/css/estilos.css">
<!-- Va a http://misitio.com/css/estilos.css -->

Esto es técnicamente una ruta absoluta (desde la raíz del dominio).
Funciona bien, pero no es portable si mueves el sitio a una subcarpeta.

───────────────────────────────────────────────────────────
RECOMENDACIÓN
───────────────────────────────────────────────────────────

Usa rutas relativas sin / al inicio para máxima portabilidad.

✅ Bueno: <img src="imagenes/logo.png" alt="Logo">
⚠️ Aceptable: <img src="/imagenes/logo.png" alt="Logo">
❌ Malo (para internos): <img src="https://misitio.com/imagenes/logo.png" alt="Logo">

PARTE 3: ESTRUCTURA DE CARPETAS PROFESIONAL
===============================================================================

Una estructura de carpetas profesional es la base de un proyecto web bien organizado.

───────────────────────────────────────────────────────────
ESTRUCTURA RECOMENDADA (PARA UN SITIO WEB)
───────────────────────────────────────────────────────────

mi-sitio-web/
│
├── index.html                 ← Página principal
├── acerca.html                ← Página "Acerca de"
├── servicios.html             ← Página "Servicios"
├── blog.html                  ← Página "Blog"
├── contacto.html              ← Página "Contacto"
│
├── css/                       ← TODOS los archivos CSS
│   └── style.css
│
├── js/                        ← TODOS los archivos JavaScript
│   └── app.js
│
├── images/                    ← TODAS las imágenes
│   ├── logo.png
│   ├── banner.jpg
│   ├── hero/
│   │   └── hero-bg.jpg
│   └── galeria/
│       ├── proyecto1.jpg
│       ├── proyecto2.jpg
│       └── proyecto3.jpg
│
├── fonts/                     ← Fuentes personalizadas
│   ├── fuente.woff2
│   └── fuente.woff
│
├── media/                     ← Audio y video
│   ├── cancion.mp3
│   └── video.mp4
│
├── docs/                      ← Documentación (opcional)
│   └── manual.pdf
│
├── assets/                    ← Recursos adicionales (logo, iconos, etc.)
│   ├── logo.svg
│   └── favicon.ico
│
└── pages/                     ← Páginas secundarias (opcional)
    ├── pagina1.html
    └── pagina2.html

ESTRUCTURA PARA PROYECTOS MÁS GRANDES (CON SISTEMA DE PLANTILLAS)

PARTE 4: ORGANIZACIÓN DE ARCHIVOS EN EL PROYECTO
===============================================================================

───────────────────────────────────────────────────────────
1. ARCHIVOS HTML (EN LA RAÍZ)
───────────────────────────────────────────────────────────

Los archivos HTML principales van en la raíz del proyecto.

✅ BUENO:
index.html
acerca.html
servicios.html
blog.html
contacto.html

❌ MALO:
paginas/index.html
paginas/acerca.html
(no es recomendable para sitios pequeños)

───────────────────────────────────────────────────────────
2. CSS (EN carpeta css/)
───────────────────────────────────────────────────────────

Todos los archivos CSS van en la carpeta css/.

✅ BUENO:
css/style.css
css/responsive.css
css/theme.css

❌ MALO:
css/ (vacío, con estilos en el HTML)
style.css en la raíz (desordenado)

───────────────────────────────────────────────────────────
3. JAVASCRIPT (EN carpeta js/)
───────────────────────────────────────────────────────────

Todos los archivos JavaScript van en la carpeta js/.

✅ BUENO:
js/app.js
js/validacion.js
js/animaciones.js

❌ MALO:
js/ (vacío, con scripts en el HTML)
app.js en la raíz (desordenado)

───────────────────────────────────────────────────────────
4. IMÁGENES (EN carpeta images/)
───────────────────────────────────────────────────────────

Todas las imágenes van en la carpeta images/.
Puedes crear subcarpetas para organizar por categorías.

✅ BUENO:
images/logo.png
images/banner.jpg
images/galeria/
images/hero/
images/icons/

❌ MALO:
images/ (sin subcarpetas, con 100 archivos desordenados)
Imágenes en la raíz del proyecto.

───────────────────────────────────────────────────────────
5. FUENTES (EN carpeta fonts/)
───────────────────────────────────────────────────────────

Las fuentes personalizadas van en fonts/.

✅ BUENO:
fonts/OpenSans.woff2
fonts/OpenSans.woff

───────────────────────────────────────────────────────────
6. MULTIMEDIA (EN carpeta media/)
───────────────────────────────────────────────────────────

Archivos de audio y video van en media/.

✅ BUENO:
media/cancion.mp3
media/video.mp4

PARTE 5: CÓMO ENLAZAR ARCHIVOS DESDE EL HTML
===============================================================================

Una vez que tienes la estructura de carpetas, debes enlazar los archivos desde el HTML usando rutas relativas.

───────────────────────────────────────────────────────────
ENLAZAR CSS
───────────────────────────────────────────────────────────

<!-- Desde index.html (en la raíz) -->
<link rel="stylesheet" href="css/style.css">

<!-- Desde pages/pagina.html (en una subcarpeta) -->
<link rel="stylesheet" href="../css/style.css">

───────────────────────────────────────────────────────────
ENLAZAR JAVASCRIPT
───────────────────────────────────────────────────────────

<!-- Desde index.html (en la raíz) -->
<script src="js/app.js" defer></script>

<!-- Desde pages/pagina.html (en una subcarpeta) -->
<script src="../js/app.js" defer></script>

───────────────────────────────────────────────────────────
ENLAZAR IMÁGENES
───────────────────────────────────────────────────────────

<!-- Desde index.html (en la raíz) -->
<img src="images/logo.png" alt="Logo">

<!-- Desde pages/pagina.html (en una subcarpeta) -->
<img src="../images/logo.png" alt="Logo">

───────────────────────────────────────────────────────────
ENLAZAR OTRAS PÁGINAS (ENLACES ENTRE HTML)
───────────────────────────────────────────────────────────

<!-- Desde index.html (en la raíz) a otra página en la raíz -->
<a href="acerca.html">Acerca de</a>
<a href="blog.html">Blog</a>

<!-- Desde index.html (en la raíz) a una página en subcarpeta -->
<a href="pages/articulo.html">Artículo</a>

<!-- Desde pages/articulo.html a una página en la raíz -->
<a href="../index.html">Inicio</a>

───────────────────────────────────────────────────────────
ARCHIVO: index.html (en la raíz)
───────────────────────────────────────────────────────────

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Sitio Web</title>

    <!-- CSS (ruta relativa desde la raíz) -->
    <link rel="stylesheet" href="css/style.css">

    <!-- Favicon -->
    <link rel="icon" href="assets/favicon.ico" type="image/x-icon">
</head>
<body>

    <header>
        <!-- Logo (ruta relativa desde la raíz) -->
        <img src="images/logo.png" alt="Logo de mi sitio" width="200" height="100">

        <nav>
            <ul>
                <!-- Enlaces a otras páginas en la raíz -->
                <li><a href="index.html">Inicio</a></li>
                <li><a href="about.html">Acerca de</a></li>
                <li><a href="contacto.html">Contacto</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section class="hero">
            <img src="images/hero/hero-bg.jpg" alt="Fondo hero" width="1200" height="400" loading="lazy">
            <h1>Bienvenido a mi sitio</h1>
        </section>

        <section>
            <h2>Proyectos</h2>
            <div class="galeria">
                <img src="images/galeria/proyecto1.jpg" alt="Proyecto 1" width="300" height="200" loading="lazy">
                <img src="images/galeria/proyecto2.jpg" alt="Proyecto 2" width="300" height="200" loading="lazy">
            </div>
        </section>

        <!-- Video -->
        <section>
            <h2>Video introductorio</h2>
            <video controls width="640" height="360">
                <source src="media/intro.mp4" type="video/mp4">
                Tu navegador no soporta video.
            </video>
        </section>
    </main>

    <!-- JavaScript (ruta relativa desde la raíz) -->
    <script src="js/app.js" defer></script>
</body>
</html>

PARTE 9: ERRORES COMUNES CON RUTAS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: RUTA INCORRECTA (ARCHIVO NO ENCONTRADO)
───────────────────────────────────────────────────────────

Incorrecto:
<img src="images/logo.png" alt="Logo">
<!-- Pero la imagen está en imagenes/ (con a) -->

Consecuencia: La imagen no se carga (error 404).

Solución: Verifica la ortografía y la ruta exacta.

───────────────────────────────────────────────────────────
ERROR 2: USAR RUTAS ABSOLUTAS PARA RECURSOS INTERNOS
───────────────────────────────────────────────────────────

Incorrecto:
<img src="https://misitio.com/images/logo.png" alt="Logo">

Consecuencia: Si el dominio cambia, la imagen se rompe.

Solución: Usa rutas relativas para recursos internos.

───────────────────────────────────────────────────────────
ERROR 3: OLVIDAR ../ EN SUBCARPETAS
───────────────────────────────────────────────────────────

Incorrecto (desde pages/articulo.html):
<link rel="stylesheet" href="css/style.css">

Consecuencia: El navegador busca css/style.css en pages/css/, no en la raíz.

Solución: Usa ../css/style.css.

───────────────────────────────────────────────────────────
ERROR 4: NOMBRES DE ARCHIVOS CON ESPACIOS O MAYÚSCULAS
───────────────────────────────────────────────────────────

Incorrecto:
<img src="images/Mi Foto.jpg" alt="Foto">

Consecuencia: Algunos servidores no reconocen espacios o mayúsculas.

Solución: Usa minúsculas y guiones: mi-foto.jpg.

───────────────────────────────────────────────────────────
ERROR 5: RUTA CON \ (BARRA INVERTIDA) EN LUGAR DE / (BARRA)
───────────────────────────────────────────────────────────

Incorrecto:
<img src="images\logo.png" alt="Logo">

Consecuencia: Las barras invertidas no funcionan en URLs.

Solución: Siempre usa / en las rutas web.

===============================================================================
PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Verifica la ortografía de las carpetas y archivos.
2. Usa rutas relativas para recursos internos.
3. Usa ../ para subir niveles desde subcarpetas.
4. Usa minúsculas y guiones en nombres de archivos.
5. Siempre usa / en las rutas web.
6. Prueba tus rutas en el navegador (abre las herramientas de desarrollo y ve a la pestaña Network para ver qué archivos fallan).
7. Usa una estructura de carpetas consistente desde el principio.

GLOSARIO DE TÉRMINOS
===============================================================================

- Ruta absoluta: Dirección completa de un archivo (incluye protocolo y dominio).
- Ruta relativa: Dirección de un archivo relativa al archivo actual.
- Raíz del proyecto: Carpeta principal del proyecto.
- Estructura de carpetas: Organización de archivos en carpetas.
- Assets: Recursos del proyecto (imágenes, fuentes, etc.).
- CDN: Content Delivery Network (red de distribución de contenido).
- 404: Error cuando un archivo no se encuentra.
- Portabilidad: Capacidad de mover un proyecto sin romper enlaces.
