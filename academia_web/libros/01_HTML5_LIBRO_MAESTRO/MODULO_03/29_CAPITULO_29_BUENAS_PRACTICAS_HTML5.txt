PARTE 1: ¿QUÉ SON LAS BUENAS PRÁCTICAS?
===============================================================================

Las buenas prácticas son un conjunto de convenciones y hábitos que mejoran la calidad del código.

───────────────────────────────────────────────────────────
¿POR QUÉ SON IMPORTANTES?
───────────────────────────────────────────────────────────

1. LEGIBILIDAD: El código es más fácil de leer.
2. MANTENIBILIDAD: Es más fácil de modificar y corregir.
3. ESCALABILIDAD: Es más fácil de expandir y añadir funcionalidades.
4. COLABORACIÓN: Es más fácil trabajar en equipo.
5. CALIDAD: Menos errores y mejor rendimiento.
6. PROFESIONALISMO: Demuestra que eres un desarrollador serio.

───────────────────────────────────────────────────────────
ÁREAS DE LAS BUENAS PRÁCTICAS
───────────────────────────────────────────────────────────

1. Estructura y organización del código.
2. Semántica y significado.
3. Accesibilidad.
4. Rendimiento y optimización.
5. Compatibilidad entre navegadores.
6. SEO (optimización para buscadores).
7. Seguridad (básica).
8. Documentación y comentarios.

En este capítulo nos enfocaremos en las más importantes para HTML5.

PARTE 2: ESTRUCTURA Y ORGANIZACIÓN
===============================================================================

───────────────────────────────────────────────────────────
1. INDENTACIÓN CONSISTENTE
───────────────────────────────────────────────────────────

La indentación (sangría) hace que el código sea más legible.

✅ BUENO:
<article>
    <header>
        <h2>Título del artículo</h2>
        <p>Publicado el <time datetime="2026-06-25">25 de junio</time></p>
    </header>
    <section>
        <p>Contenido del artículo...</p>
    </section>
</article>

❌ MALO:
<article>
<header>
<h2>Título del artículo</h2>
<p>Publicado el <time datetime="2026-06-25">25 de junio</time></p>
</header>
<section>
<p>Contenido del artículo...</p>
</section>
</article>

REGLAS:
- Usa 2 o 4 espacios por nivel (¡no tabs mezclados!).
- Sé consistente en todo el proyecto.

───────────────────────────────────────────────────────────
2. ELEMENTOS EN LÍNEA Y BLOQUE
───────────────────────────────────────────────────────────

No pongas elementos de bloque dentro de elementos en línea.

❌ MALO:
<p>Este es un párrafo con un <div>bloque dentro</div>.</p>

✅ BUENO:
<p>Este es un párrafo con un <span>elemento en línea</span>.</p>

───────────────────────────────────────────────────────────
3. CIERRE DE ETIQUETAS
───────────────────────────────────────────────────────────

Siempre cierra todas las etiquetas que lo requieran.

❌ MALO:
<p>Texto sin cerrar

✅ BUENO:
<p>Texto correctamente cerrado</p>

───────────────────────────────────────────────────────────
4. NOMBRES EN MINÚSCULAS
───────────────────────────────────────────────────────────

Usa minúsculas para nombres de archivos, carpetas, etiquetas y atributos.

❌ MALO:
<IMG SRC="Imagen.JPG" ALT="Foto">

✅ BUENO:
<img src="imagen.jpg" alt="Foto">

───────────────────────────────────────────────────────────
5. ATRIBUTOS CON COMILLAS
───────────────────────────────────────────────────────────

Siempre usa comillas dobles para los valores de los atributos.

❌ MALO:
<img src=imagen.jpg alt=Foto>

✅ BUENO:
<img src="imagen.jpg" alt="Foto">

===============================================================================
PARTE 3: SEMÁNTICA Y SIGNIFICADO
===============================================================================

───────────────────────────────────────────────────────────
1. USA LAS ETIQUETAS SEMÁNTICAS CORRECTAS
───────────────────────────────────────────────────────────

Usa la etiqueta que mejor describa el contenido.

❌ MALO:
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>

✅ BUENO:
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

───────────────────────────────────────────────────────────
2. JERARQUÍA DE ENCABEZADOS
───────────────────────────────────────────────────────────

Usa los encabezados en orden jerárquico.

❌ MALO:
<h1>Título</h1>
<h4>Subtítulo</h4>
<h3>Otro título</h3>

✅ BUENO:
<h1>Título</h1>
<h2>Subtítulo</h2>
<h3>Otro título</h3>

───────────────────────────────────────────────────────────
3. UN SOLO <h1> POR PÁGINA
───────────────────────────────────────────────────────────

El <h1> debe ser único y describir el tema principal.

❌ MALO:
<h1>Bienvenido</h1>
<h1>Servicios</h1>
<h1>Contacto</h1>

✅ BUENO:
<h1>Mi Sitio Web</h1>
<h2>Servicios</h2>
<h2>Contacto</h2>

───────────────────────────────────────────────────────────
4. EVITA <div> INNECESARIOS
───────────────────────────────────────────────────────────

No uses <div> cuando una etiqueta semántica es más apropiada.

❌ MALO:
<div class="articulo">...</div>

✅ BUENO:
<article>...</article>

===============================================================================
PARTE 4: ACCESIBILIDAD
===============================================================================

───────────────────────────────────────────────────────────
1. SIEMPRE USA alt EN IMÁGENES
───────────────────────────────────────────────────────────

El atributo alt es obligatorio para la accesibilidad.

❌ MALO:
<img src="foto.jpg">

✅ BUENO:
<img src="foto.jpg" alt="Atardecer en la playa">

───────────────────────────────────────────────────────────
2. ASOCIA LABEL CON INPUT
───────────────────────────────────────────────────────────

Cada campo de formulario debe tener un label asociado.

❌ MALO:
<input type="text" id="nombre" placeholder="Nombre">

✅ BUENO:
<label for="nombre">Nombre:</label>
<input type="text" id="nombre" placeholder="Nombre">

───────────────────────────────────────────────────────────
3. USA ESTRUCTURA SEMÁNTICA
───────────────────────────────────────────────────────────

Los lectores de pantalla usan la semántica para navegar.

✅ BUENO:
<header>...</header>
<nav>...</nav>
<main>...</main>
<section>...</section>
<article>...</article>
<aside>...</aside>
<footer>...</footer>

───────────────────────────────────────────────────────────
4. USA scope EN TABLAS
───────────────────────────────────────────────────────────

<th scope="col"> y <th scope="row"> ayudan a los lectores de pantalla.

✅ BUENO:
<th scope="col">Producto</th>
<th scope="row">Manzana</th>

───────────────────────────────────────────────────────────
5. CONTRASTE Y LEGIBILIDAD
───────────────────────────────────────────────────────────

Asegura que el texto sea legible (contraste, tamaño de fuente).

✅ BUENO:
- Texto oscuro sobre fondo claro.
- Tamaño de fuente mínimo de 16px.
- Espaciado entre líneas de 1.5x.

===============================================================================
PARTE 5: RENDIMIENTO Y OPTIMIZACIÓN
===============================================================================

───────────────────────────────────────────────────────────
1. OPTIMIZA IMÁGENES
───────────────────────────────────────────────────────────

- Usa formatos modernos (WebP, AVIF).
- Comprime las imágenes.
- Redimensiona al tamaño necesario.
- Usa loading="lazy".

✅ BUENO:
<img src="imagen.webp" alt="Foto" loading="lazy" width="800" height="600">

───────────────────────────────────────────────────────────
2. CARGA CSS EN EL <head>
───────────────────────────────────────────────────────────

Esto evita un "flash de contenido sin estilo" (FOUC).

✅ BUENO:
<head>
    <link rel="stylesheet" href="css/style.css">
</head>

❌ MALO:
<body>
    <!-- contenido -->
    <link rel="stylesheet" href="css/style.css">
</body>

───────────────────────────────────────────────────────────
3. CARGA JAVASCRIPT CON defer O async
───────────────────────────────────────────────────────────

Esto evita que el JS bloquee el renderizado.

✅ BUENO:
<script src="js/app.js" defer></script>
<script src="js/analytics.js" async></script>

❌ MALO:
<script src="js/app.js"></script>  (bloquea el renderizado)

───────────────────────────────────────────────────────────
4. EVITA RECURSOS INNECESARIOS
───────────────────────────────────────────────────────────

Solo carga los recursos que realmente necesitas.

- No cargues bibliotecas enteras si solo usas una función.
- No cargues fuentes que no uses.
- Elimina código muerto.

───────────────────────────────────────────────────────────
5. COMPRIME ARCHIVOS
───────────────────────────────────────────────────────────

- Minimiza CSS y JavaScript (quita espacios y comentarios en producción).
- Comprime imágenes y otros recursos.

===============================================================================
PARTE 6: SEO (OPTIMIZACIÓN PARA BUSCADORES)
===============================================================================

───────────────────────────────────────────────────────────
1. USA TÍTULOS DESCRIPTIVOS
───────────────────────────────────────────────────────────

<title> debe ser descriptivo y tener entre 50-60 caracteres.

✅ BUENO:
<title>Curso de HTML5 - Aprende desde cero | Lucas EGA Academy</title>

❌ MALO:
<title>Página 1</title>

───────────────────────────────────────────────────────────
2. USA META DESCRIPTION
───────────────────────────────────────────────────────────

<meta name="description"> debe ser atractiva y tener entre 150-160 caracteres.

✅ BUENO:
<meta name="description" content="Aprende HTML5 desde cero con el Profesor Búho AI. Cursos prácticos, ejercicios y proyectos. ¡Empieza hoy!">

───────────────────────────────────────────────────────────
3. USA OPEN GRAPH (OG:)
───────────────────────────────────────────────────────────

Controla la vista previa en redes sociales.

✅ BUENO:
<meta property="og:title" content="Curso de HTML5">
<meta property="og:description" content="Aprende HTML5 desde cero">
<meta property="og:image" content="https://.../og-image.jpg">

───────────────────────────────────────────────────────────
4. ESTRUCTURA DE URLS AMIGABLES
───────────────────────────────────────────────────────────

Usa URLs descriptivas y legibles.

✅ BUENO:
https://misitio.com/curso-html5
https://misitio.com/blog/que-es-html

❌ MALO:
https://misitio.com/p=123
https://misitio.com/index.php?id=45

───────────────────────────────────────────────────────────

───────────────────────────────────────────────────────────

- Minúsculas.
- Sin espacios.
- Nombres descriptivos.

✅ BUENO:
css/
js/
images/
assets/
fonts/
media/

❌ MALO:
CSS/
JavaScript/
Imagenes/
Recursos/

───────────────────────────────────────────────────────────
3. CLASES E IDS EN HTML
───────────────────────────────────────────────────────────

- Minúsculas.
- Usa guiones (-) para separar palabras.
- Nombres descriptivos.

✅ BUENO:
class="contenedor-principal"
class="articulo-destacado"
id="hero"
class="btn-primario"

class="ContenedorPrincipal"
class="articulo_destacado"
id="Hero"
class="btnPrimario"

PARTE 9: ERRORES COMUNES EN BUENAS PRÁCTICAS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: MEZCLAR TABS Y ESPACIOS
───────────────────────────────────────────────────────────

Consecuencia: El código se ve desordenado en diferentes editores.

Solución: Usa solo espacios (2 o 4) en todo el proyecto.

───────────────────────────────────────────────────────────
ERROR 2: NO INDENTAR EL CÓDIGO
───────────────────────────────────────────────────────────

Consecuencia: Es difícil leer la estructura.

Solución: Siempre identa los elementos anidados.

───────────────────────────────────────────────────────────
ERROR 3: NOMBRES DE ARCHIVOS CON ESPACIOS
───────────────────────────────────────────────────────────

Consecuencia: Pueden causar errores en servidores y URLs.

Solución: Usa minúsculas y guiones.

───────────────────────────────────────────────────────────
ERROR 4: NO CERRAR ETIQUETAS
───────────────────────────────────────────────────────────

Consecuencia: Puede romper el diseño y causar errores.

Solución: Siempre cierra todas las etiquetas.

───────────────────────────────────────────────────────────
ERROR 5: OLVIDAR alt EN IMÁGENES
───────────────────────────────────────────────────────────

Consecuencia: Problemas de accesibilidad y SEO.

Solución: Siempre añade alt a las imágenes.

───────────────────────────────────────────────────────────
ERROR 6: NO USAR SEMÁNTICA
───────────────────────────────────────────────────────────

Consecuencia: Problemas de SEO y accesibilidad.

Solución: Usa las etiquetas semánticas correctas.

───────────────────────────────────────────────────────────
ERROR 7: CARGAR SCRIPTS EN EL <head> SIN defer/async
───────────────────────────────────────────────────────────

Consecuencia: Bloqueo del renderizado.

Solución: Usa defer para scripts que necesitan el DOM.

PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Configura tu editor para usar espacios en lugar de tabs.
2. Usa un formateador automático (Prettier, Beautify).
3. Revisa los nombres de archivos y carpetas.
4. Usa un validador HTML (validator.w3.org).
5. Prueba tu página con un lector de pantalla.
6. Usa Lighthouse para auditar rendimiento, SEO y accesibilidad.
7. Revisa las buenas prácticas periódicamente.

GLOSARIO DE TÉRMINOS
===============================================================================

- Buena práctica: Convención o hábito que mejora la calidad del código.
- Código limpio: Código fácil de leer, entender y mantener.
- Indentación: Sangría para mostrar la jerarquía del código.
- Semántica: Uso de etiquetas con significado.
- Accesibilidad: Capacidad de ser usado por personas con discapacidad.
- Rendimiento: Velocidad de carga y ejecución.
- SEO: Optimización para motores de búsqueda.
- Refactorización: Mejorar el código sin cambiar su comportamiento.
- Convención: Regla o estándar acordado.
