PARTE 1: ¿QUÉ ES UN `<div>`?
===============================================================================

`<div>` es una etiqueta HTML que significa "división" (division).

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<div>
    Contenido agrupado aquí
</div>

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

1. Es un elemento de BLOQUE.
   → Ocupa todo el ancho disponible.
   → Empieza en una nueva línea.
   → Se apila verticalmente.

2. No tiene significado semántico.
   → Solo sirve como contenedor.
   → No describe el contenido.

3. Es el contenedor más versátil.
   → Puede contener cualquier otro elemento HTML.
   → Puede ser anidado infinitamente.

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<div>
    <h1>Bienvenido a mi sitio</h1>
    <p>Este es un párrafo dentro de un div.</p>
</div>

El `<div>` agrupa el título y el párrafo en un solo bloque.

===============================================================================
PARTE 2: ELEMENTOS DE BLOQUE VS. ELEMENTOS EN LÍNEA
===============================================================================

Para entender `<div>`, primero debes entender la diferencia entre bloques y líneas.

───────────────────────────────────────────────────────────
ELEMENTOS DE BLOQUE (BLOCK)
───────────────────────────────────────────────────────────

- Ocupan todo el ancho disponible.
- Siempre comienzan en una nueva línea.
- Se apilan verticalmente.

EJEMPLOS:
`<div>`, `<h1>` a `<h6>`, `<p>`, `<ul>`, `<ol>`, `<li>`, `<table>`, `<form>`, `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`

───────────────────────────────────────────────────────────
ELEMENTOS EN LÍNEA (INLINE)
───────────────────────────────────────────────────────────

- Ocupan solo el espacio necesario.
- No comienzan en una nueva línea.
- Se colocan uno al lado del otro.

EJEMPLOS:
`<span>`, `<a>`, `<strong>`, `<em>`, `<mark>`, `<img>`, `<br>`, `<code>`, `<label>`, `<button>`, `<input>`

───────────────────────────────────────────────────────────
LA DIFERENCIA CLAVE
───────────────────────────────────────────────────────────

<div> es un elemento de BLOQUE.
<span> es un elemento en LÍNEA.

Esta diferencia es fundamental para saber cuándo usar cada uno.

===============================================================================
PARTE 3: ¿CUÁNDO USAR `<div>`?
===============================================================================

`<div>` se usa cuando necesitas agrupar elementos sin un significado semántico específico.

───────────────────────────────────────────────────────────
CASOS DE USO COMUNES
───────────────────────────────────────────────────────────

1. AGRUPAR ELEMENTOS PARA CSS:
   → Cuando quieras aplicar estilos a un grupo de elementos.
   → `<div class="contenedor">...</div>`

2. AGRUPAR ELEMENTOS PARA JAVASCRIPT:
   → Cuando quieras manipular un grupo de elementos con JS.
   → `<div id="seccion-dinamica">...</div>`

3. CREAR ESTRUCTURAS DE MAQUETACIÓN:
   → Cuando necesites dividir la página en filas y columnas.
   → `<div class="fila"><div class="columna">...</div></div>`

4. CONTENEDORES TEMPORALES:
   → Cuando estés desarrollando y necesites un contenedor rápido.

───────────────────────────────────────────────────────────
EJEMPLO DE USO PROFESIONAL
───────────────────────────────────────────────────────────

<div class="contenedor-principal">
    <header>
        <div class="logo">
            <img src="logo.png" alt="Logo">
        </div>
        <nav>
            <ul>
                <li><a href="#">Inicio</a></li>
                <li><a href="#">Servicios</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <div class="hero">
            <h1>Bienvenido</h1>
            <p>Contenido principal</p>
        </div>

        <div class="servicios">
            <div class="servicio">
                <h3>Servicio 1</h3>
                <p>Descripción...</p>
            </div>
            <div class="servicio">
                <h3>Servicio 2</h3>
                <p>Descripción...</p>
            </div>
        </div>
    </main>

    <footer>
        <div class="footer-contenido">
            <p>&copy; 2026</p>
        </div>
    </footer>
</div>

===============================================================================
PARTE 4: ¿CUÁNDO NO USAR `<div>`?
===============================================================================

Tan importante como saber cuándo usar `<div>` es saber cuándo NO usarlo.

───────────────────────────────────────────────────────────
NO USES `<div>` CUANDO HAYA UNA ETIQUETA SEMÁNTICA DISPONIBLE
───────────────────────────────────────────────────────────

❌ MALO:
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="section">...</div>
<div class="article">...</div>
<div class="aside">...</div>
<div class="footer">...</div>

✅ BUENO:
<header>...</header>
<nav>...</nav>
<main>...</main>
<section>...</section>
<article>...</article>
<aside>...</aside>
<footer>...</footer>

───────────────────────────────────────────────────────────
REGLAS DE ORO
───────────────────────────────────────────────────────────

1. Si el contenido tiene un significado semántico, usa la etiqueta semántica.
2. Si solo necesitas un contenedor para estilos o JavaScript, usa `<div>`.
3. `<div>` es para ESTRUCTURA. Las etiquetas semánticas son para SIGNIFICADO.

===============================================================================
PARTE 9: ERRORES COMUNES CON `<div>`
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: USAR `<div>` CUANDO HAY UNA ETIQUETA SEMÁNTICA
───────────────────────────────────────────────────────────

Incorrecto:
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>

Correcto:
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

───────────────────────────────────────────────────────────
ERROR 2: ANIDACIÓN EXCESIVA (DIV-ITIS)
───────────────────────────────────────────────────────────

Incorrecto:
<div><div><div><div><div>Contenido</div></div></div></div></div>

Consecuencia: Código difícil de leer y mantener.

Solución: Solo usa la cantidad necesaria de divs.

───────────────────────────────────────────────────────────
ERROR 3: NO USAR CLASES O IDS PARA IDENTIFICAR DIVS
───────────────────────────────────────────────────────────

Incorrecto:
<div>Contenido</div>
<div>Contenido</div>

Correcto:
<div class="contenido-principal">Contenido</div>
<div class="contenido-secundario">Contenido</div>

───────────────────────────────────────────────────────────
ERROR 4: NO CERRAR LAS ETIQUETAS DE DIV
───────────────────────────────────────────────────────────

Incorrecto:
<div>
    <p>Contenido</p>

Consecuencia: El navegador intenta corregir, pero puede causar errores.

Solución: Siempre cierra cada div que abres.

───────────────────────────────────────────────────────────
ERROR 5: PONER CONTENIDO DIRECTAMENTE DENTRO DE DIV SIN ESTRUCTURA
───────────────────────────────────────────────────────────

Incorrecto:
<div>
    Texto suelto aquí
    <p>Párrafo</p>
    Más texto suelto
</div>

Correcto:
<div>
    <p>Texto organizado</p>
    <p>Más texto organizado</p>
</div>

===============================================================================
PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa etiquetas semánticas cuando existan.
2. Mantén la anidación simple (máximo 3-4 niveles).
3. Usa clases o ids para identificar divs.
4. Siempre cierra tus divs.
5. Estructura el contenido dentro de los divs con etiquetas adecuadas.
6. Usa un editor de código que coloree las etiquetas para detectar errores de cierre.

GLOSARIO DE TÉRMINOS
===============================================================================

- div: Etiqueta HTML para agrupar contenido (división).
- Contenedor: Elemento que agrupa otros elementos.
- Bloque: Elemento que ocupa todo el ancho y empieza en nueva línea.
- Línea: Elemento que ocupa solo el espacio necesario y no empieza en nueva línea.
- Anidación: Colocar un elemento dentro de otro.
- Clase: Atributo para identificar un grupo de elementos.
- ID: Atributo para identificar un elemento único.
- Maquetación: Organización y distribución de elementos en una página.

No uses divs para todo.
No tengas miedo de usarlos cuando sea necesario.

La clave está en el equilibrio.
Una página bien estructurada usa tanto divs como etiquetas semánticas,
cada una en su lugar correcto."
