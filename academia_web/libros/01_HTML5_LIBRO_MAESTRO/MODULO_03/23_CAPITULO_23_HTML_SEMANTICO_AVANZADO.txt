PARTE 1: REPASO DE LAS ETIQUETAS SEMÁNTICAS BÁSICAS
===============================================================================

Antes de profundizar, repasemos las etiquetas semánticas que ya conoces.

───────────────────────────────────────────────────────────
ETIQUETAS BÁSICAS
───────────────────────────────────────────────────────────

<header>   → Cabecera de página o sección.
<nav>      → Navegación principal.
<main>     → Contenido principal (único por página).
<section>  → Agrupación temática de contenido.
<article>  → Contenido independiente y autocontenido.
<aside>    → Contenido complementario (barra lateral).
<footer>   → Pie de página o sección.

───────────────────────────────────────────────────────────
REGLAS DE ORO (REPASO)
───────────────────────────────────────────────────────────

1. Solo un <main> por página.
2. <section> siempre debe tener un encabezado.
3. <article> debe ser contenido independiente.
4. <aside> es complementario, no principal.
5. Usa estas etiquetas antes que <div> siempre que sea posible.

PARTE 2: <figure> Y <figcaption> – CONTENIDO CON CONTEXTO
===============================================================================

<figure> agrupa contenido multimedia (imágenes, diagramas, código, etc.) con su descripción.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<figure>
    <img src="imagen.jpg" alt="Descripción">
    <figcaption>Pie de foto explicativo.</figcaption>
</figure>

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

- El contenido dentro de <figure> debe ser autónomo.
- Puede contener: imágenes, videos, audios, código, tablas, etc.
- <figcaption> es opcional pero recomendado.
- <figcaption> debe ser el primero o el último hijo de <figure>.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Imagen con pie de foto -->
<figure>
    <img src="paisaje.jpg" alt="Atardecer en la montaña" width="600" height="400">
    <figcaption>Atardecer en los Andes, tomado en enero de 2026.</figcaption>
</figure>

<!-- Múltiples imágenes en una figura -->
<figure>
    <img src="grafico1.jpg" alt="Gráfico de ventas 2025">
    <img src="grafico2.jpg" alt="Gráfico de ventas 2026">
    <figcaption>Evolución de las ventas anuales.</figcaption>
</figure>

<!-- Código con figura -->
<figure>
    <pre><code>
        function saludar() {
            console.log("Hola, mundo");
        }
    </code></pre>
    <figcaption>Código de ejemplo en JavaScript.</figcaption>
</figure>

<!-- Video con figura -->
<figure>
    <video controls width="640" height="360">
        <source src="tutorial.mp4" type="video/mp4">
    </video>
    <figcaption>Tutorial de HTML5 para principiantes.</figcaption>
</figure>

PARTE 3: <details> Y <summary> – CONTENIDO DESPLEGABLE
===============================================================================

<details> crea un elemento que el usuario puede expandir o contraer.
<summary> proporciona el título visible del elemento desplegable.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<details>
    <summary>Título del acordeón</summary>
    <p>Contenido que se muestra al expandir.</p>
</details>

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

- Por defecto, el contenido está oculto (solo se ve el summary).
- El usuario hace clic en el summary para expandir/contraer.
- Se puede usar el atributo open para que esté expandido por defecto.
- No necesita JavaScript (es nativo de HTML5).

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Básico -->
<details>
    <summary>Más información</summary>
    <p>Este es el contenido oculto que se muestra al hacer clic.</p>
</details>

<!-- Con open -->
<details open>
    <summary>Preguntas frecuentes</summary>
    <p>Aquí puedes encontrar las respuestas a las preguntas más comunes.</p>
</details>

<!-- Múltiples details -->
<details>
    <summary>¿Qué es HTML?</summary>
    <p>HTML es el lenguaje de marcado que da estructura a las páginas web.</p>
</details>

<details>
    <summary>¿Qué es CSS?</summary>
    <p>CSS es el lenguaje que da estilo a las páginas web.</p>
</details>

<details>
    <summary>¿Qué es JavaScript?</summary>
    <p>JavaScript es el lenguaje que da interactividad a las páginas web.</p>
</details>

<!-- Anidación de details -->
<details>
    <summary>HTML5</summary>
    <details>
        <summary>Etiquetas semánticas</summary>
        <p>Las etiquetas semánticas dan significado al contenido.</p>
    </details>
    <details>
        <summary>Formularios</summary>
        <p>Los formularios permiten la interacción del usuario.</p>
    </details>
</details>

PARTE 4: <time> – FECHAS Y HORAS SEMÁNTICAS
===============================================================================

<time> representa una fecha, hora o duración en formato semántico.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<time datetime="2026-06-25">25 de junio de 2026</time>
<time datetime="14:30">2:30 PM</time>
<time datetime="2026-06-25T14:30">25 de junio de 2026, 2:30 PM</time>

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

datetime → Fecha y/o hora en formato estándar (YYYY-MM-DD, HH:MM, etc.)

───────────────────────────────────────────────────────────
FORMATOS DE DATETIME
───────────────────────────────────────────────────────────

- Fecha: 2026-06-25
- Hora: 14:30
- Fecha y hora: 2026-06-25T14:30
- Fecha y hora con zona horaria: 2026-06-25T14:30-06:00
- Duración: P2D (2 días), PT1H30M (1 hora 30 minutos)

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Fecha -->
<p>Artículo publicado el <time datetime="2026-06-25">25 de junio de 2026</time>.</p>

<!-- Hora -->
<p>La reunión es a las <time datetime="14:30">2:30 PM</time>.</p>

<!-- Fecha y hora -->
<p>Evento: <time datetime="2026-06-25T14:30">25 de junio de 2026, 2:30 PM</time>.</p>

<!-- Fecha con formato legible -->
<p>El <time datetime="2026-06-25">25/06/2026</time> es una fecha importante.</p>

<!-- Duración -->
<p>Tiempo estimado: <time datetime="PT1H30M">1 hora 30 minutos</time>.</p>

<!-- En un artículo -->
<article>
    <header>
        <h2>Título del artículo</h2>
        <p>Publicado el <time datetime="2026-06-20">20 de junio de 2026</time></p>
    </header>
    <p>Contenido del artículo...</p>
</article>

PARTE 5: <address> – INFORMACIÓN DE CONTACTO
===============================================================================

<address> representa información de contacto del autor o propietario de la página.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<address>
    <a href="mailto:info@ejemplo.com">info@ejemplo.com</a><br>
    <a href="tel:+521234567890">+52 123 456 7890</a><br>
    Calle Principal #123, Ciudad, País
</address>

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

- Se usa para información de contacto del autor.
- Puede contener: nombre, correo, teléfono, dirección física.
- Generalmente se coloca en el <footer> de la página o del artículo.
- Los navegadores lo muestran en cursiva por defecto.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- En el footer de la página -->
<footer>
    <address>
        Escrito por <a href="mailto:esteban@lucasEGA.com">Esteban</a>.<br>
        Lucas EGA Academy<br>
        Calle Principal #123, Ciudad de México
    </address>
</footer>

<!-- En un artículo -->
<article>
    <h2>Título del artículo</h2>
    <p>Contenido...</p>
    <address>
        Autor: <a href="mailto:ana@ejemplo.com">Ana Pérez</a>
    </address>
</article>

PARTE 6: <dialog> – VENTANA MODAL
===============================================================================

<dialog> crea una ventana de diálogo o modal.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<dialog id="miDialogo">
    <p>Este es un diálogo modal.</p>
    <button onclick="document.getElementById('miDialogo').close()">Cerrar</button>
</dialog>

<button onclick="document.getElementById('miDialogo').showModal()">
    Abrir diálogo
</button>

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

- Crea una ventana superpuesta.
- Se abre con showModal() o show().
- Se cierra con close().
- El atributo open lo muestra por defecto.
- Requiere JavaScript para abrir/cerrar.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<!-- El diálogo -->
<dialog id="modal">
    <h2>¡Importante!</h2>
    <p>Este es un mensaje importante para todos los usuarios.</p>
    <button onclick="document.getElementById('modal').close()">Cerrar</button>
</dialog>

<!-- Botón para abrir -->
<button onclick="document.getElementById('modal').showModal()">
    Abrir mensaje importante
</button>

PARTE 7: <progress> Y <meter> – INDICADORES VISUALES
===============================================================================

───────────────────────────────────────────────────────────
<progress> → BARRA DE PROGRESO
───────────────────────────────────────────────────────────

Indica el progreso de una tarea.

<progress value="75" max="100">75%</progress>

Atributos:
- value → Valor actual.
- max → Valor máximo (por defecto 1).

───────────────────────────────────────────────────────────
<meter> → MEDIDOR
───────────────────────────────────────────────────────────

Indica un valor dentro de un rango conocido.

<meter value="0.6" min="0" max="1" low="0.3" high="0.8" optimum="0.7">60%</meter>

Atributos:
- value → Valor actual.
- min → Valor mínimo (por defecto 0).
- max → Valor máximo (por defecto 1).
- low → Valor considerado bajo.
- high → Valor considerado alto.
- optimum → Valor óptimo.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Progreso de una tarea -->
<label for="progreso">Progreso de la instalación:</label>
<progress id="progreso" value="75" max="100">75%</progress>

<!-- Nivel de conocimiento -->
<label for="nivel">Nivel de HTML:</label>
<meter id="nivel" value="0.8" min="0" max="1" low="0.3" high="0.8">80%</meter>

<!-- Uso de disco -->
<label for="disco">Uso de disco:</label>
<meter id="disco" value="70" min="0" max="100" low="50" high="90">70%</meter>

<!-- Calificación -->
<label for="calif">Calificación:</label>
<meter id="calif" value="8.5" min="0" max="10" low="5" high="9">8.5</meter>

PARTE 8: OTRAS ETIQUETAS SEMÁNTICAS
===============================================================================

───────────────────────────────────────────────────────────
<mark> – TEXTO RESALTADO
───────────────────────────────────────────────────────────

<mark> representa texto resaltado (como un marcador de texto).

<p>La <mark>programación</mark> es el arte de resolver problemas.</p>

───────────────────────────────────────────────────────────
<small> – TEXTO PEQUEÑO (DESCARGO DE RESPONSABILIDAD)
───────────────────────────────────────────────────────────

<small> representa texto pequeño, como descargos de responsabilidad.

<p><small>© 2026 Todos los derechos reservados.</small></p>

───────────────────────────────────────────────────────────
<cite> – TÍTULO DE UNA OBRA
───────────────────────────────────────────────────────────

<cite> representa el título de una obra (libro, película, artículo).

<blockquote>
    <p>"La programación es el arte de resolver problemas."</p>
    <cite>— Profesor Búho AI</cite>
</blockquote>

───────────────────────────────────────────────────────────
<abbr> – ABREVIATURA
───────────────────────────────────────────────────────────

<abbr> representa una abreviatura o acrónimo.

<p>El <abbr title="HyperText Markup Language">HTML</abbr> es fundamental.</p>

───────────────────────────────────────────────────────────
<kbd> – TEXTO QUE EL USUARIO DEBE TECLEAR
───────────────────────────────────────────────────────────

<kbd> representa texto que el usuario debe ingresar.

<p>Presiona <kbd>Ctrl+S</kbd> para guardar.</p>

───────────────────────────────────────────────────────────
<code> – CÓDIGO FUENTE
───────────────────────────────────────────────────────────

<code> representa fragmentos de código.

<p>El comando <code>console.log()</code> muestra en consola.</p>

───────────────────────────────────────────────────────────
<pre> – TEXTO PREFORMATEADO
───────────────────────────────────────────────────────────

<pre> respeta espacios y saltos de línea.

<pre>
    function saludar() {
        console.log("Hola");
    }
</pre>

───────────────────────────────────────────────────────────
<var> – VARIABLE
───────────────────────────────────────────────────────────

<var> representa una variable matemática o de programación.

<p>El resultado es <var>x</var> = 10.</p>

───────────────────────────────────────────────────────────
<samp> – SALIDA DE PROGRAMA
───────────────────────────────────────────────────────────

<samp> representa la salida de un programa.

<p>El programa mostró: <samp>Error 404</samp></p>

PARTE 12: SEMÁNTICA, SEO Y ACCESIBILIDAD (LA TRINIDAD)
===============================================================================

La semántica no existe en el vacío.
Está íntimamente relacionada con el SEO y la accesibilidad.

───────────────────────────────────────────────────────────
SEMÁNTICA Y SEO
───────────────────────────────────────────────────────────

Google utiliza la semántica para entender tu contenido.

- <h1> → Tema principal de la página.
- <article> → Contenido independiente.
- <time> → Fecha de publicación.
- <address> → Información de contacto.
- <figure> → Contenido multimedia con contexto.

Una página con buena semántica tiene mejor posicionamiento en buscadores.

───────────────────────────────────────────────────────────
SEMÁNTICA Y ACCESIBILIDAD
───────────────────────────────────────────────────────────

Los lectores de pantalla utilizan la semántica para navegar.

- <main> → El usuario puede saltar directamente al contenido principal.
- <nav> → El usuario puede saltar a la navegación.
- <article> → El usuario puede navegar entre artículos.
- <aside> → El usuario sabe que es contenido complementario.
- <details> → El usuario puede expandir/contraer contenido.

Una página con buena semántica es más accesible.

───────────────────────────────────────────────────────────
LA TRINIDAD
───────────────────────────────────────────────────────────

SEMÁNTICA → SEO → ACCESIBILIDAD

Cuando escribes código semántico:
- Google te entiende mejor.
- Los lectores de pantalla te entienden mejor.
- Otros desarrolladores te entienden mejor.

Es un triple beneficio.

PARTE 13: BUENAS PRÁCTICAS EN SEMÁNTICA AVANZADA
===============================================================================

───────────────────────────────────────────────────────────
1. SIEMPRE USA LA ETIQUETA MÁS DESCRIPTIVA
───────────────────────────────────────────────────────────

❌ Malo: <div class="fecha">25/06/2026</div>
✅ Bueno: <time datetime="2026-06-25">25/06/2026</time>

❌ Malo: <div class="pie-foto">Descripción</div>
✅ Bueno: <figcaption>Descripción</figcaption>

❌ Malo: <div class="contacto">info@email.com</div>
✅ Bueno: <address>info@email.com</address>

───────────────────────────────────────────────────────────
2. USA <details> PARA CONTENIDO OPCIONAL
───────────────────────────────────────────────────────────

Usa details para:
- Preguntas frecuentes (FAQ).
- Información adicional que no es esencial.
- Ayuda contextual.
- Contenido que el usuario puede ocultar/mostrar.

───────────────────────────────────────────────────────────
3. USA <time> PARA TODAS LAS FECHAS
───────────────────────────────────────────────────────────

Siempre usa <time> con datetime para fechas y horas.
Esto ayuda a Google y mejora la accesibilidad.

───────────────────────────────────────────────────────────
4. USA <address> EN EL FOOTER
───────────────────────────────────────────────────────────

El address debe estar en el footer de la página o del artículo.
Contiene información de contacto del autor.

───────────────────────────────────────────────────────────
5. USA <figure> Y <figcaption> PARA MULTIMEDIA
───────────────────────────────────────────────────────────

Siempre que tengas una imagen, diagrama, video o código con contexto, usa figure y figcaption.

===============================================================================
PARTE 14: ERRORES COMUNES EN SEMÁNTICA AVANZADA
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: NO USAR <time> EN FECHAS
───────────────────────────────────────────────────────────

Incorrecto:
<p>Publicado el 25 de junio de 2026</p>

Correcto:
<p>Publicado el <time datetime="2026-06-25">25 de junio de 2026</time></p>

───────────────────────────────────────────────────────────
ERROR 2: USAR <div> EN LUGAR DE <address>
───────────────────────────────────────────────────────────

Incorrecto:
<div class="contacto">info@email.com</div>

Correcto:
<address>info@email.com</address>

───────────────────────────────────────────────────────────
ERROR 3: NO USAR <figcaption> DENTRO DE <figure>
───────────────────────────────────────────────────────────

Incorrecto:
<figure>
    <img src="imagen.jpg" alt="...">
    <p>Pie de foto</p>
</figure>

Correcto:
<figure>
    <img src="imagen.jpg" alt="...">
    <figcaption>Pie de foto</figcaption>
</figure>

───────────────────────────────────────────────────────────
ERROR 4: USAR <details> SIN <summary>
───────────────────────────────────────────────────────────

Incorrecto:
<details>
    <p>Contenido oculto</p>
</details>

Correcto:
<details>
    <summary>Título del acordeón</summary>
    <p>Contenido oculto</p>
</details>

───────────────────────────────────────────────────────────
ERROR 5: NO USAR <main> O TENER MÚLTIPLES
───────────────────────────────────────────────────────────

Incorrecto:
<body>
    <main>Contenido 1</main>
    <main>Contenido 2</main>
</body>

Correcto:
<body>
    <main>Contenido único</main>
</body>

PARTE 15: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa <time> con datetime para fechas y horas.
2. Usa <address> para información de contacto del autor.
3. Dentro de <figure>, usa <figcaption> para el pie de foto.
4. <details> siempre debe tener un <summary>.
5. Solo un <main> por página.
6. Valida tu código con el validador del W3C.
7. Prueba tu página con un lector de pantalla para verificar la accesibilidad.

Errores frecuentes:
- No usar <time> para fechas
- No usar <figcaption> dentro de <figure>
- Usar <details> sin <summary>
- No usar <address> para contacto
- No usar <main> o usar múltiples

Preguntas frecuentes:
- ¿Cuándo usar <figure> y cuándo <div>? Usa figure para contenido multimedia con contexto; div para contenedores genéricos.
- ¿Puedo usar <details> sin <summary>? No, debe tener un summary.
- ¿<address> solo puede ir en el footer? Generalmente sí, pero también puede ir dentro de un article.
- ¿Por qué usar <time>? Porque Google y los lectores de pantalla interpretan mejor las fechas.
- ¿Cuál es la diferencia entre <progress> y <meter>? progress indica progreso de una tarea; meter indica un valor dentro de un rango.

Analogías utilizadas:
- figure = marco con etiqueta.
- details = acordeón desplegable.
- time = fecha con etiqueta semántica.
- address = tarjeta de visita.
- progress = termómetro de carga.
- meter = velocímetro.

- Artículo completo con todas las etiquetas semánticas avanzadas.
