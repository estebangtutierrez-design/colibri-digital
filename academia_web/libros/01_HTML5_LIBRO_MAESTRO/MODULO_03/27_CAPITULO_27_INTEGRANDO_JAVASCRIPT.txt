PARTE 1: ¿QUÉ ES JAVASCRIPT?
===============================================================================

JavaScript es un lenguaje de programación que permite añadir interactividad a las páginas web.

───────────────────────────────────────────────────────────
DEFINICIÓN
───────────────────────────────────────────────────────────

JavaScript es un lenguaje de programación interpretado que se ejecuta en el navegador.
Permite:
- Responder a eventos del usuario (clics, teclas, movimientos).
- Manipular el DOM (cambiar contenido, estilos, estructura).
- Comunicarse con servidores (enviar y recibir datos).
- Crear animaciones y efectos visuales.
- Validar formularios.
- Y mucho más.

───────────────────────────────────────────────────────────
JAVASCRIPT VS. JAVA
───────────────────────────────────────────────────────────

NO son lo mismo.
- Java es un lenguaje de programación general (para aplicaciones, servidores, etc.).
- JavaScript es un lenguaje para la web (se ejecuta en navegadores).

El nombre es confuso, pero no tienen relación.

PARTE 2: FORMAS DE INTEGRAR JAVASCRIPT EN HTML
===============================================================================

Al igual que CSS, JavaScript se puede integrar de tres formas.

───────────────────────────────────────────────────────────
1. JAVASCRIPT EN LÍNEA (INLINE)
───────────────────────────────────────────────────────────

Se escribe directamente en el HTML, dentro del atributo de un evento.

EJEMPLO:
<button onclick="alert('¡Hola!')">Haz clic</button>

Ventajas:
- Rápido y directo.
- Útil para pruebas rápidas.

Desventajas:
- Mezcla HTML con JavaScript.
- Difícil de mantener.
- No es reutilizable.

⚠️ RECOMENDACIÓN: No uses JavaScript en línea en proyectos profesionales.

───────────────────────────────────────────────────────────
2. JAVASCRIPT INTERNO (INTERNAL)
───────────────────────────────────────────────────────────

Se escribe dentro del HTML, usando la etiqueta `<script>`.

EJEMPLO:
<script>
    alert('¡Hola desde JavaScript interno!');
</script>

Ubicación:
- Puede ir en el `<head>`.
- Puede ir al final del `<body>` (recomendado).

Ventajas:
- No mezcla tanto HTML con JavaScript.
- Útil para páginas individuales.

Desventajas:
- No se reutiliza entre páginas.
- Aumenta el tamaño del HTML.
- No aprovecha la caché del navegador.

⚠️ RECOMENDACIÓN: Usa JavaScript interno para pruebas o páginas muy sencillas.

───────────────────────────────────────────────────────────
3. JAVASCRIPT EXTERNO (EXTERNAL)
───────────────────────────────────────────────────────────

Se escribe en un archivo separado con extensión `.js`, y se enlaza desde el HTML usando `<script src="...">`.

EJEMPLO:
<!-- index.html -->
<script src="js/app.js"></script>

// app.js
alert('¡Hola desde JavaScript externo!');

Ventajas:
- Separa completamente el contenido (HTML) del comportamiento (JavaScript).
- El JavaScript es reutilizable en múltiples páginas.
- Los archivos JS se almacenan en caché (carga más rápida).
- Fácil de mantener (cambios en un lugar afectan a todas las páginas).
- Código más limpio y profesional.

Desventajas:
- Requiere un archivo adicional.
- El navegador debe hacer una petición extra para cargar el JS.

⚠️ RECOMENDACIÓN: Usa SIEMPRE JavaScript externo en proyectos profesionales.

PARTE 3: LA ETIQUETA `<script>` – TODO LO QUE NECESITAS SABER
===============================================================================

La etiqueta `<script>` se usa para definir o enlazar JavaScript.

───────────────────────────────────────────────────────────
SINTAXIS BÁSICA
───────────────────────────────────────────────────────────

<!-- JavaScript interno -->
<script>
    // Código JavaScript aquí
    console.log('Hola');
</script>

<!-- JavaScript externo -->
<script src="ruta/archivo.js"></script>

───────────────────────────────────────────────────────────
ATRIBUTOS PRINCIPALES DE `<script>`
───────────────────────────────────────────────────────────

src → Ruta del archivo JavaScript (para externo).
defer → Retrasa la ejecución hasta que el HTML esté cargado.
async → Carga el script de forma asíncrona.
type → Tipo de contenido (opcional, por defecto es "text/javascript").
integrity → Verifica la integridad del script (seguridad).

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Script externo básico (sin optimización) -->
<script src="js/app.js"></script>

<!-- Script externo con defer (recomendado) -->
<script src="js/app.js" defer></script>

<!-- Script externo con async -->
<script src="js/analytics.js" async></script>

<!-- Múltiples scripts -->
<script src="js/validacion.js" defer></script>
<script src="js/animaciones.js" defer></script>
<script src="js/app.js" defer></script>

<!-- Script interno -->
<script>
    console.log('Hola desde JavaScript interno');
</script>

PARTE 4: `defer` VS. `async` – CÓMO OPTIMIZAR LA CARGA
===============================================================================

La ubicación y los atributos `defer` y `async` afectan el rendimiento de carga.

───────────────────────────────────────────────────────────
EL PROBLEMA SIN `defer` O `async`
───────────────────────────────────────────────────────────

<script src="js/app.js"></script>

Consecuencia:
- El navegador DETIENE el procesamiento del HTML.
- Descarga y ejecuta el script COMPLETO.
- Luego continúa procesando el HTML.

Si el script es grande, la página tarda en mostrarse.

───────────────────────────────────────────────────────────
SOLUCIÓN 1: `defer` (RECOMENDADO)
───────────────────────────────────────────────────────────

<script src="js/app.js" defer></script>

Comportamiento:
- El navegador CARGA el script en segundo plano (sin detener el HTML).
- El script se EJECUTA después de que el HTML esté completamente cargado.
- Mantiene el orden de los scripts.

Ventajas:
- No bloquea el renderizado.
- Los scripts se ejecutan en orden.
- Es la mejor opción para la mayoría de los casos.

Cuándo usar `defer`:
- Para scripts que necesitan el DOM completo (ej: manipulación de elementos).
- Para la mayoría de tus scripts.

───────────────────────────────────────────────────────────
SOLUCIÓN 2: `async` (PARA SCRIPTS INDEPENDIENTES)
───────────────────────────────────────────────────────────

<script src="js/analytics.js" async></script>

Comportamiento:
- El navegador CARGA el script en segundo plano.
- El script se EJECUTA TAN PRONTO como se descarga.
- No mantiene el orden de los scripts.

Ventajas:
- No bloquea el renderizado.
- Ejecución inmediata.

Desventajas:
- No garantiza el orden de ejecución.
- No espera a que el DOM esté listo.

Cuándo usar `async`:
- Para scripts que no dependen del DOM.
- Para scripts que no dependen de otros scripts.
- Ejemplos: Google Analytics, scripts de terceros.

───────────────────────────────────────────────────────────
CUADRO COMPARATIVO
───────────────────────────────────────────────────────────

+----------+------------------+------------------+------------------+
| Atributo | Bloquea HTML    | Orden garantizado| DOM disponible   |
+----------+------------------+------------------+------------------+
| (ninguno)| ✅ Sí (bloquea)  | ✅ Sí            | ❌ No            |
| defer    | ❌ No            | ✅ Sí            | ✅ Sí            |
| async    | ❌ No            | ❌ No            | ❌ No            |
+----------+------------------+------------------+------------------+

PARTE 5: ¿DÓNDE COLOCAR LOS SCRIPTS?
===============================================================================

La ubicación de los scripts afecta el rendimiento.

───────────────────────────────────────────────────────────
OPCIÓN 1: EN EL `<HEAD>` (NO RECOMENDADO)
───────────────────────────────────────────────────────────

<head>
    <script src="js/app.js"></script>
</head>

Consecuencia:
- El navegador procesa el script ANTES de renderizar el contenido.
- La página se muestra más lenta.
- No es recomendado (a menos que uses defer).

───────────────────────────────────────────────────────────
OPCIÓN 2: AL FINAL DEL `<BODY>` (RECOMENDADO SIN `defer`)
───────────────────────────────────────────────────────────

<body>
    <!-- Contenido HTML -->
    <script src="js/app.js"></script>
</body>

Ventajas:
- El navegador procesa todo el HTML antes de cargar el script.
- La página se muestra más rápido.
- Es la forma tradicional de hacerlo.

Desventajas:
- El script se ejecuta después de que el DOM esté cargado, pero el navegador puede bloquearse.

───────────────────────────────────────────────────────────
OPCIÓN 3: EN EL `<HEAD>` CON `defer` (RECOMENDADO)
───────────────────────────────────────────────────────────

<head>
    <script src="js/app.js" defer></script>
</head>

Ventajas:
- El script se carga en segundo plano.
- El HTML se procesa sin bloqueos.
- La página se muestra más rápido.
- Es la forma MÁS PROFESIONAL de hacerlo.

───────────────────────────────────────────────────────────
RECOMENDACIÓN FINAL
───────────────────────────────────────────────────────────

✅ Usa `<script src="js/app.js" defer></script>` en el `<head>`.

O

✅ Usa `<script src="js/app.js"></script>` al final del `<body>`.

Ambas son buenas prácticas.
Pero `defer` en el `<head>` es la más moderna y profesional.

PARTE 7: EJEMPLO PRÁCTICO COMPLETO
===============================================================================

CÓDIGO HTML (index.html)

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Integrando JavaScript</title>

    <!-- CSS -->
    <link rel="stylesheet" href="css/style.css">

    <!-- JavaScript con defer (RECOMENDADO) -->
    <script src="js/app.js" defer></script>

    <!-- JavaScript interno (para cosas muy específicas) -->
    <script>
        console.log('Hola desde JavaScript interno');
    </script>
</head>
<body>

    <h1>Integrando JavaScript en HTML</h1>

    <div id="mensaje">
        <p>Haz clic en el botón para cambiar este texto.</p>
    </div>

    <button id="boton-cambiar">Cambiar texto</button>

    <!-- Botón con JavaScript en línea (NO RECOMENDADO) -->
    <button onclick="alert('¡Hola desde JavaScript en línea!')">
        Saludar (en línea)
    </button>

    <!-- Botón que usa el script externo -->
    <button id="boton-saludar">Saludar (externo)</button>

</body>
</html>

───────────────────────────────────────────────────────────
CÓDIGO JAVASCRIPT EXTERNO (js/app.js)

/* ===== app.js - JavaScript principal ===== */

console.log('Hola desde JavaScript externo');

// 1. Cambiar texto con un botón
const botonCambiar = document.getElementById('boton-cambiar');
const mensaje = document.getElementById('mensaje');

if (botonCambiar) {
    botonCambiar.addEventListener('click', function() {
        mensaje.innerHTML = '<p>¡El texto ha cambiado gracias a JavaScript!</p>';
    });
}

// 2. Mostrar alerta con botón
const botonSaludar = document.getElementById('boton-saludar');

if (botonSaludar) {
    botonSaludar.addEventListener('click', function() {
        alert('¡Hola desde JavaScript externo!');
    });
}

// 3. Mensaje en consola
console.log('¡JavaScript está funcionando correctamente!');

/* ===== FIN DE app.js ===== */

PARTE 9: ERRORES COMUNES AL INTEGRAR JAVASCRIPT
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: RUTA INCORRECTA AL ARCHIVO JS
───────────────────────────────────────────────────────────

Incorrecto:
<script src="js/app.js">
<!-- Pero app.js está en otra carpeta -->

Consecuencia: El JavaScript no se carga (error 404).

Solución: Usa la ruta correcta.

───────────────────────────────────────────────────────────
ERROR 2: OLVIDAR EL ATRIBUTO src O LA ETIQUETA DE CIERRE
───────────────────────────────────────────────────────────

Incorrecto:
<script src="js/app.js">

Correcto:
<script src="js/app.js"></script>

Solución: Siempre usa `</script>`.

───────────────────────────────────────────────────────────
ERROR 3: EJECUTAR JS ANTES DE QUE EL DOM ESTÉ LISTO
───────────────────────────────────────────────────────────

Incorrecto (en el <head> sin defer):
<script src="js/app.js"></script>

Consecuencia: El script intenta acceder a elementos que aún no existen en el DOM.

Solución: Usa `defer` o coloca el script al final del `<body>`.

───────────────────────────────────────────────────────────
ERROR 4: ERRORES DE SINTAXIS EN JAVASCRIPT
───────────────────────────────────────────────────────────

Incorrecto:
console.log('Hola'  ← falta paréntesis de cierre

Consecuencia: El script no se ejecuta completamente.

Solución: Revisa la sintaxis de tu JavaScript.

───────────────────────────────────────────────────────────
ERROR 5: NO USAR `defer` CON SCRIPTS QUE MANIPULAN EL DOM
───────────────────────────────────────────────────────────

Incorrecto:
<script src="js/app.js"></script>

Consecuencia: El script puede ejecutarse antes de que el DOM esté listo.

Solución: Usa `defer` en el `<head>` o coloca el script al final del `<body>`.

PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Verifica que la ruta del JS sea correcta (usa rutas relativas).
2. Siempre cierra la etiqueta `<script>` con `</script>`.
3. Usa `defer` en el `<head>` para asegurar que el DOM esté listo.
4. Revisa la consola del navegador (F12 → Console) para ver errores de JavaScript.
5. Usa `console.log()` para depurar tu código.
6. Verifica que el archivo JS se cargue correctamente (F12 → Network).

GLOSARIO DE TÉRMINOS
===============================================================================

- JavaScript: Lenguaje de programación que da interactividad a las páginas web.
- DOM: Document Object Model (representación del HTML en JavaScript).
- Event Listener: Función que escucha eventos del usuario (clics, teclas, etc.).

- src: Atributo que define la ruta del archivo JavaScript.
- Script: Conjunto de instrucciones en JavaScript.
- Console: Herramienta de depuración en el navegador.

RESUMEN DEL CAPÍTULO
===============================================================================

En este capítulo aprendiste:

- JavaScript en línea: `onclick="..."` (no recomendado).
- JavaScript interno: `<script>` en HTML (útil para pruebas).
- JavaScript externo: `<script src="ruta.js">` (recomendado).
- `defer`: El script se ejecuta después de que el HTML esté cargado (recomendado).
- `async`: El script se ejecuta tan pronto como se descarga (para scripts independientes).
- Ubicación: en `<head>` con `defer` o al final del `<body>`.
- Organización: carpeta `js/` para archivos JavaScript.

Ahora puedes integrar JavaScript en HTML como un profesional.

BASE DE CONOCIMIENTO PARA EL PROFESOR BÚHO AI
===============================================================================

Tema: Integrando JavaScript en HTML
Nivel: Intermedio (Nivel 3/10)
Tiempo estimado: 60 minutos

Palabras clave:
JavaScript, script, defer, async, src, interno, externo, en línea, DOM, event listener, interactividad

Conceptos relacionados:
- JavaScript (lenguaje)
- DOM (Document Object Model)
- Eventos del navegador
- Rendimiento web
- Buenas prácticas de desarrollo

Errores frecuentes:
- Ruta incorrecta al archivo JS
- Olvidar cerrar `<script>`
- Ejecutar JS antes de que el DOM esté listo
- Errores de sintaxis en JavaScript
- No usar `defer` o `async`

Preguntas frecuentes:
- ¿Cuál es la mejor forma de integrar JavaScript? JavaScript externo con `defer`.
- ¿Cuándo usar `async`? Para scripts que no dependen del DOM (ej: Google Analytics).
- ¿Cuándo usar `defer`? Para scripts que manipulan el DOM (la mayoría de los casos).
- ¿Qué pasa si no uso `defer` o `async`? El script bloquea el renderizado.
- ¿Cómo sé si el JavaScript se carga correctamente? Usa la consola (F12) y la pestaña Network.

Analogías utilizadas:
- JavaScript = vida que mueve el esqueleto (HTML) y la ropa (CSS).
- defer = esperar a que la casa esté lista antes de poner los muebles.
- async = poner los muebles tan pronto como llegan, sin esperar.

Ejemplos utilizados:
- Página con las tres formas de JavaScript.
- Botón que cambia texto.
- Botón con alerta.
- Event listeners.

Mini proyecto:
- Página con JavaScript externo profesional.

Nivel alcanzado por el alumno:
- Capaz de integrar JavaScript usando las tres formas.
- Capaz de elegir la forma adecuada para cada situación.
- Capaz de usar `defer` y `async` correctamente.
- Capaz de estructurar un proyecto con JavaScript externo.
