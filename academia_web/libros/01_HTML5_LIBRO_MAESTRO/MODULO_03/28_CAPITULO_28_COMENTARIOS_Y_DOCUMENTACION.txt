ANTES DE EMPEZAR...
===============================================================================

Cierra los ojos por un momento.

Imagina que estás leyendo un libro.
El libro tiene capítulos, títulos, subtítulos.
Tiene notas al margen que explican conceptos.
Tiene un índice que te guía.

Ahora imagina que el mismo libro está escrito sin capítulos, sin títulos, sin notas.
Es un bloque de texto continuo de 300 páginas.

¿Cuál de los dos libros prefieres leer?
¿Cuál de los dos entiendes mejor?

El código es igual.

Un código con comentarios y documentación es como un libro bien organizado.
Un código sin comentarios es como un bloque de texto sin estructura.

Los comentarios son las notas al margen de tu código.
La documentación es el índice y los capítulos.

===============================================================================
PARTE 1: ¿QUÉ SON LOS COMENTARIOS EN HTML?
===============================================================================

Los comentarios son fragmentos de texto que el navegador IGNORA.
Son solo para las personas que leen el código.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<!-- Esto es un comentario en HTML -->

El navegador no muestra esto en la página.
Solo es visible para quien vea el código fuente.

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<!-- Bienvenida del sitio -->
<h1>Bienvenido a Lucas EGA Academy</h1>

<!-- Sección de cursos -->
<section id="cursos">
    <h2>Nuestros cursos</h2>
    <!-- Aquí van los cursos -->
</section>

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

1. El navegador los IGNORA completamente.
2. No se ven en la página web.
3. Son visibles al inspeccionar el código (F12).
4. Sirven para EXPLICAR el código a otros desarrolladores.
5. Sirven para RECORDARTE a ti mismo lo que hiciste.

PARTE 2: SINTAXIS Y REGLAS DE LOS COMENTARIOS
===============================================================================

───────────────────────────────────────────────────────────
SINTAXIS CORRECTA
───────────────────────────────────────────────────────────

<!-- Comentario de una línea -->

    Comentario de

───────────────────────────────────────────────────────────
REGLAS IMPORTANTES
───────────────────────────────────────────────────────────

1. Un comentario comienza con `<!--`.
2. Un comentario termina con `-->`.
3. NO se pueden anidar comentarios dentro de comentarios.

───────────────────────────────────────────────────────────
❌ INCORRECTO (ANIDACIÓN)
───────────────────────────────────────────────────────────

<!-- Esto es un comentario <!-- con otro comentario dentro --> -->

Consecuencia: El navegador se confunde.

───────────────────────────────────────────────────────────
✅ CORRECTO
───────────────────────────────────────────────────────────

<!-- Esto es un comentario -->
<!-- Este es otro comentario -->

───────────────────────────────────────────────────────────

contador++;

✅ BUENO (explica el por qué):
<!--
    Este contador se usa para rastrear cuántas veces
    el usuario ha hecho clic en el botón "Me gusta".
    Se reinicia al recargar la página.
-->

───────────────────────────────────────────────────────────

Un comentario desactualizado es peor que no tener comentario.

===============================================================================
PARTE 5: COMENTARIOS VS. DOCUMENTACIÓN
===============================================================================

───────────────────────────────────────────────────────────
COMENTARIOS (DENTRO DEL CÓDIGO)
───────────────────────────────────────────────────────────

- Están dentro del archivo HTML.
- Explican partes específicas del código.
- Son para desarrolladores que leen el código.

EJEMPLO:
<!--
    Esta sección muestra los cursos disponibles.
    Cada curso es un article con título, descripción y enlace.
-->

───────────────────────────────────────────────────────────
DOCUMENTACIÓN (ARCHIVOS SEPARADOS)
───────────────────────────────────────────────────────────

- Está en archivos separados (README.md, manuales, guías).
- Explica el proyecto en general.
- Es para nuevos desarrolladores, usuarios, etc.

EJEMPLO (README.md):

# Lucas EGA Academy

## Descripción
Academia de programación con el Profesor Búho AI.

## Estructura del proyecto
- css/ → Estilos
- js/ → JavaScript
- images/ → Imágenes

## Cómo ejecutar
Abrir index.html en el navegador.

───────────────────────────────────────────────────────────
LA DIFERENCIA
───────────────────────────────────────────────────────────

- Los comentarios explican el CÓDIGO.
- La documentación explica el PROYECTO.

Ambos son necesarios.

===============================================================================

ERROR 1: ANIDAR COMENTARIOS
───────────────────────────────────────────────────────────

Incorrecto:
<!-- Comentario exterior <!-- Comentario interior --> -->

Consecuencia: El navegador se confunde.

Solución: No anides comentarios.

───────────────────────────────────────────────────────────
ERROR 2: COMENTARIOS DESACTUALIZADOS
───────────────────────────────────────────────────────────

Incorrecto:
<!-- Esta sección contiene 3 cursos -->
<section id="cursos">
    <!-- Solo 2 cursos actualmente -->
    <article>Curso 1</article>
    <article>Curso 2</article>
</section>

Consecuencia: El comentario dice 3, pero hay 2.

Solución: Actualiza los comentarios cuando cambies el código.

GLOSARIO DE TÉRMINOS
===============================================================================

- Comentario: Texto en el código que el navegador ignora, solo para humanos.
- Documentación: Información sobre el proyecto en archivos separados.
- Código limpio: Código fácil de leer, entender y mantener.
- TODO: Marca de tarea pendiente en el código.
- Cabecera: Comentario al inicio del archivo con información del proyecto.
- Refactorizar: Mejorar el código sin cambiar su comportamiento.
- Mantenibilidad: Capacidad de un código para ser modificado fácilmente.
