PARTE 1: ¿QUÉ ES CSS?
===============================================================================

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada).

───────────────────────────────────────────────────────────
DEFINICIÓN
───────────────────────────────────────────────────────────

CSS es un lenguaje que describe la presentación de un documento HTML.
Controla:
- Colores.
- Fuentes.
- Tamaños.
- Espaciados.
- Posicionamiento.
- Animaciones.
- Diseño responsive.
- Y mucho más.

───────────────────────────────────────────────────────────
SINTAXIS BÁSICA DE CSS
───────────────────────────────────────────────────────────

selector {
    propiedad: valor;
    propiedad: valor;
}

EJEMPLO:
h1 {
    color: blue;
    font-size: 24px;
}

- selector → El elemento al que se aplica el estilo (h1).
- propiedad → Lo que quieres cambiar (color, font-size).
- valor → El nuevo valor (blue, 24px).

===============================================================================
PARTE 2: CSS EN LÍNEA (INLINE CSS)
===============================================================================

El CSS en línea se aplica directamente a un elemento HTML usando el atributo style.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<etiqueta style="propiedad: valor; propiedad: valor;">
    Contenido
</etiqueta>

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<h1 style="color: blue; font-size: 32px;">Título en azul</h1>
<p style="color: red; font-weight: bold;">Este párrafo es rojo y negrita.</p>

───────────────────────────────────────────────────────────
VENTAJAS
───────────────────────────────────────────────────────────

- Rápido y directo.
- Útil para pruebas y prototipos.
- Útil para estilos muy específicos y únicos.
- Tiene la máxima prioridad (sobrescribe otros estilos).

───────────────────────────────────────────────────────────
DESVENTAJAS
───────────────────────────────────────────────────────────

- Mezcla contenido (HTML) con presentación (CSS).
- Difícil de mantener (cambios uno por uno).
- No es reutilizable (no se comparte entre elementos).
- Aumenta el tamaño del HTML.
- No aprovecha la caché del navegador.

───────────────────────────────────────────────────────────
CUÁNDO USAR CSS EN LÍNEA
───────────────────────────────────────────────────────────

- Solo en casos muy específicos.
- Para pruebas rápidas.
- Para estilos que son únicos y no se repiten.
- En correos electrónicos HTML (donde el CSS externo no funciona bien).

⚠️ RECOMENDACIÓN: Evita el CSS en línea en proyectos profesionales.
Usa CSS externo siempre que sea posible.

===============================================================================
PARTE 3: CSS INTERNO (INTERNAL CSS)
===============================================================================

El CSS interno se escribe dentro del <head> de la página, usando la etiqueta <style>.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<head>
    <style>
        selector {
            propiedad: valor;
        }
        selector {
            propiedad: valor;
        }
    </style>
</head>

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>CSS Interno</title>
    <style>
        h1 {
            color: blue;
            font-size: 32px;
        }
        p {
            color: red;
            font-weight: bold;
        }
    </style>
</head>
<body>
    <h1>Título en azul</h1>
    <p>Este párrafo es rojo y negrita.</p>
</body>
</html>

───────────────────────────────────────────────────────────
VENTAJAS
───────────────────────────────────────────────────────────

- Mantiene el CSS separado del contenido (aunque en la misma página).
- Permite aplicar estilos a múltiples elementos.
- Es útil para páginas individuales.
- No requiere archivos adicionales.

───────────────────────────────────────────────────────────
DESVENTAJAS
───────────────────────────────────────────────────────────

- El CSS no se reutiliza entre páginas.
- Aumenta el tamaño de cada página.
- No aprovecha la caché del navegador.
- Si el CSS es grande, ralentiza la carga.

───────────────────────────────────────────────────────────
CUÁNDO USAR CSS INTERNO
───────────────────────────────────────────────────────────

- Páginas individuales sin necesidad de compartir estilos.
- Páginas de destino (landing pages) con estilos únicos.
- Cuando no se puede usar un archivo externo.
- Para pruebas y prototipos.

⚠️ RECOMENDACIÓN: Usa CSS interno solo para páginas muy específicas.
Para la mayoría de los proyectos, usa CSS externo.

===============================================================================
PARTE 4: CSS EXTERNO (EXTERNAL CSS)
===============================================================================

El CSS externo se escribe en un archivo separado con extensión .css, y se enlaza desde el HTML usando la etiqueta <link>.

───────────────────────────────────────────────────────────
SINTAXIS (HTML)
───────────────────────────────────────────────────────────

<head>
    <link rel="stylesheet" href="ruta/estilos.css">
</head>

───────────────────────────────────────────────────────────
SINTAXIS (CSS – archivo style.css)
───────────────────────────────────────────────────────────

h1 {
    color: blue;
    font-size: 32px;
}
p {
    color: red;
    font-weight: bold;
}

───────────────────────────────────────────────────────────
EJEMPLO COMPLETO
───────────────────────────────────────────────────────────

<!-- index.html -->
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>CSS Externo</title>
    <link rel="stylesheet" href="css/style.css">
</head>
<body>
    <h1>Título en azul</h1>
    <p>Este párrafo es rojo y negrita.</p>
</body>
</html>

/* style.css */
h1 {
    color: blue;
    font-size: 32px;
}
p {
    color: red;
    font-weight: bold;
}

───────────────────────────────────────────────────────────
VENTAJAS
───────────────────────────────────────────────────────────

- Separa completamente el contenido (HTML) de la presentación (CSS).
- El CSS es reutilizable en múltiples páginas.
- Los archivos CSS se almacenan en caché (carga más rápida).
- Fácil de mantener (cambios en un lugar afectan a todas las páginas).
- Código más limpio y profesional.

───────────────────────────────────────────────────────────
DESVENTAJAS
───────────────────────────────────────────────────────────

- Requiere un archivo adicional.
- El navegador debe hacer una petición extra para cargar el CSS.
- Si el archivo CSS no se carga, la página se ve sin estilos.

───────────────────────────────────────────────────────────
CUÁNDO USAR CSS EXTERNO
───────────────────────────────────────────────────────────

- ¡SIEMPRE!
- Es la forma PROFESIONAL de integrar CSS.
- Úsalo en todos tus proyectos.

===============================================================================
PARTE 5: LA ETIQUETA <link> – ENLAZANDO CSS
===============================================================================

La etiqueta <link> se usa en el <head> para enlazar recursos externos.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<link rel="stylesheet" href="ruta/archivo.css" type="text/css">

───────────────────────────────────────────────────────────
ATRIBUTOS PRINCIPALES
───────────────────────────────────────────────────────────

rel → Relación con el documento.
- "stylesheet" → Para hojas de estilo.
- "icon" → Para favicon.
- "preconnect" → Para precargar conexiones.

href → Ruta del archivo (absoluta o relativa).

type → Tipo de archivo (opcional en HTML5).
- "text/css" → Para CSS.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Estilo principal -->
<link rel="stylesheet" href="css/style.css">

<!-- Estilo para impresión -->
<link rel="stylesheet" href="css/print.css" media="print">

<!-- Fuente de Google Fonts -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Open+Sans&display=swap">

<!-- Favicon -->
<link rel="icon" href="assets/favicon.ico" type="image/x-icon">

<!-- Preconectar para mejorar rendimiento -->
<link rel="preconnect" href="https://fonts.googleapis.com">

───────────────────────────────────────────────────────────
ATRIBUTO media (para diferentes dispositivos)
───────────────────────────────────────────────────────────

<!-- Para pantallas -->
<link rel="stylesheet" href="css/style.css" media="screen">

<!-- Para impresión -->
<link rel="stylesheet" href="css/print.css" media="print">

<!-- Para dispositivos móviles -->
<link rel="stylesheet" href="css/mobile.css" media="screen and (max-width: 768px)">

PARTE 6: PRIORIDADES Y CASCADA (¿CUÁL GANA?)
===============================================================================

Cuando hay múltiples estilos aplicados al mismo elemento, se aplican reglas de prioridad.

───────────────────────────────────────────────────────────
ORDEN DE PRIORIDAD (DE MAYOR A MENOR)
───────────────────────────────────────────────────────────

1. CSS EN LÍNEA (style="" en el elemento)
2. CSS INTERNO (<style> en el <head>)
3. CSS EXTERNO (<link>)

───────────────────────────────────────────────────────────
REGLAS ADICIONALES
───────────────────────────────────────────────────────────

- Si hay estilos con la misma prioridad, gana el ÚLTIMO que se declara.
- El selector más específico tiene prioridad sobre uno menos específico.
- El atributo !important sobrescribe todo (se desaconseja usar).

───────────────────────────────────────────────────────────
EJEMPLO DE PRIORIDAD
───────────────────────────────────────────────────────────

<!-- CSS EXTERNO (style.css) -->
p {
    color: red;
}

<!-- CSS INTERNO -->
<style>
    p {
        color: blue;
    }
</style>

<!-- CSS EN LÍNEA -->
<p style="color: green;">Este texto será verde</p>

Resultado: El párrafo será VERDE (prioridad máxima).

───────────────────────────────────────────────────────────
EJEMPLO DE CASCADA (MISMA PRIORIDAD)
───────────────────────────────────────────────────────────

/* style.css */
p {
    color: red;
}

/* style.css (más abajo) */
p {
    color: blue;
}

Resultado: El párrafo será AZUL (el último declarado gana).

PARTE 7: BUENAS PRÁCTICAS PARA INTEGRAR CSS
===============================================================================

───────────────────────────────────────────────────────────
1. USA SIEMPRE CSS EXTERNO PARA PROYECTOS PROFESIONALES
───────────────────────────────────────────────────────────

Separa el contenido de la presentación.
Es más limpio, más mantenible y más profesional.

───────────────────────────────────────────────────────────
2. ENLAZA EL CSS EN EL <head>
───────────────────────────────────────────────────────────

Siempre enlaza el CSS en el <head> para que se cargue antes de que el contenido se renderice.
Esto evita un "flash de contenido sin estilo" (FOUC).

───────────────────────────────────────────────────────────
3. USA RUTAS RELATIVAS PARA EL CSS
───────────────────────────────────────────────────────────

<link rel="stylesheet" href="css/style.css">
<link rel="stylesheet" href="../css/style.css">

No uses rutas absolutas para recursos internos.

───────────────────────────────────────────────────────────
4. ORGANIZA EL CSS EN ARCHIVOS POR PROPÓSITO
───────────────────────────────────────────────────────────

- style.css → Estilos principales.
- responsive.css → Estilos responsivos.
- theme.css → Tema o colores.
- print.css → Estilos para impresión.

───────────────────────────────────────────────────────────
5. EVITA EL CSS EN LÍNEA
───────────────────────────────────────────────────────────

Solo úsalo para pruebas rápidas o estilos únicos.
En proyectos profesionales, evítalo.

───────────────────────────────────────────────────────────
6. EVITA EL CSS INTERNO EN PROYECTOS GRANDES
───────────────────────────────────────────────────────────

Úsalo solo para páginas individuales o prototipos.
Para proyectos grandes, siempre externo.

───────────────────────────────────────────────────────────
7. USA COMENTARIOS EN EL CSS
───────────────────────────────────────────────────────────

/* ===== HEADER ===== */
header {
    background: #1a202c;
}

/* ===== NAVEGACIÓN ===== */
nav a {
    color: white;
}

PARTE 11: ERRORES COMUNES AL INTEGRAR CSS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: RUTA INCORRECTA AL ARCHIVO CSS
───────────────────────────────────────────────────────────

Incorrecto:
<link rel="stylesheet" href="style.css">
<!-- Pero style.css está en la carpeta css/ -->

Consecuencia: El CSS no se carga.

Solución: Usa la ruta correcta: href="css/style.css"

───────────────────────────────────────────────────────────
ERROR 2: OLVIDAR EL ATRIBUTO rel="stylesheet"
───────────────────────────────────────────────────────────

Incorrecto:
<link href="style.css">

Consecuencia: El navegador no reconoce que es una hoja de estilo.

Solución: Siempre incluye rel="stylesheet".

───────────────────────────────────────────────────────────
ERROR 3: NO CERRAR LA ETIQUETA <style>
───────────────────────────────────────────────────────────

Incorrecto:
<style>
    h1 { color: blue; }

Consecuencia: El navegador no interpreta correctamente el CSS.

Solución: Siempre cierra </style>.

───────────────────────────────────────────────────────────
ERROR 4: CSS EN LÍNEA CON ERROR DE SINTAXIS
───────────────────────────────────────────────────────────

Incorrecto:
<p style="color: blue font-size: 18px;">Texto</p>

Consecuencia: Falta el punto y coma entre propiedades.

Solución: Usa ; para separar propiedades: "color: blue; font-size: 18px;"

───────────────────────────────────────────────────────────
ERROR 5: NO USAR LA RUTA CORRECTA DESDE SUBCARPETAS
───────────────────────────────────────────────────────────

Incorrecto (desde pages/articulo.html):
<link rel="stylesheet" href="css/style.css">

Consecuencia: Busca css/style.css en pages/css/.

Solución: Usa href="../css/style.css".

===============================================================================
PARTE 12: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Verifica que la ruta del CSS sea correcta (usa rutas relativas).
2. Siempre incluye rel="stylesheet" en <link>.
3. Siempre cierra la etiqueta </style>.
4. Usa ; para separar propiedades en CSS en línea.
5. Desde subcarpetas, usa ../ para subir niveles.
6. Abre las herramientas de desarrollo (F12) y ve a la pestaña "Network" para ver si el CSS se carga correctamente.
7. Usa el validador del W3C para verificar tu código.

PARTE 13: PRÁCTICA GUIADA
===============================================================================

1. Crea una carpeta "mi-proyecto-css".
2. Crea la siguiente estructura:
   - index.html
   - css/
     - style.css
3. En index.html, escribe la estructura básica de HTML5.
4. En <head>, enlaza el CSS externo: <link rel="stylesheet" href="css/style.css">.
5. En <head>, añade un <style> con una clase "interno".
6. En <body>, crea un <p> con CSS en línea (style="color: red;").
7. En <body>, crea un <p> con la clase "interno".
8. En <body>, crea un <p> con la clase "externo" (definida en style.css).
9. En style.css, define la clase "externo".
10. Guarda y abre index.html en el navegador.

GLOSARIO DE TÉRMINOS
===============================================================================

- CSS: Cascading Style Sheets (Hojas de Estilo en Cascada).
- CSS en línea: Estilos aplicados directamente en el elemento (style="").
- CSS interno: Estilos dentro de <style> en el <head>.
- CSS externo: Estilos en un archivo .css separado, enlazado con <link>.
- <link> : Etiqueta para enlazar recursos externos.
- <style> : Etiqueta para definir estilos internos.
- Cascada: Mecanismo por el cual se aplican los estilos según prioridad.
- Prioridad: Orden de importancia de los estilos (en línea > interno > externo).
- Selector: Elemento, clase o id al que se aplican estilos CSS.
