───────────────────────────────────────────────────────────
1. ESTRUCTURA DE CARPETAS
───────────────────────────────────────────────────────────
mi-proyecto/

│ └── app.js

│ └── favicon.ico

text

───────────────────────────────────────────────────────────
2. SEPARACIÓN DE RESPONSABILIDADES
───────────────────────────────────────────────────────────

- **HTML:** Estructura del contenido.
- **CSS:** Presentación y diseño.
- **JavaScript:** Comportamiento e interactividad.

❌ Malo:
```html
<!-- Todo mezclado -->
<div style="color: red;" onclick="alert('Hola')">Texto</div>
✅ Bueno:

html
<!-- Separado -->
<div class="destacado" id="miElemento">Texto</div>
css
/* style.css */
.destacado {
    color: red;
}
javascript
// app.js
document.getElementById('miElemento').addEventListener('click', function() {
    alert('Hola');
});
───────────────────────────────────────────────────────────

INDENTACIÓN CONSISTENTE
───────────────────────────────────────────────────────────

Usa 2 o 4 espacios (¡no tabs mezclados!).

✅ Bueno:

html
<article>
    <header>
        <h2>Título</h2>
    </header>
    <section>
        <p>Contenido...</p>
    </section>
</article>
❌ Malo:

html
<article>
<header>
<h2>Título</h2>
</header>
<section>
<p>Contenido...</p>
</section>
</article>
===============================================================================
PARTE 2: NOMENCLATURA Y CONVENCIONES
===============================================================================

Los nombres deben ser descriptivos y consistentes.

───────────────────────────────────────────────────────────

NOMBRES DE ARCHIVOS Y CARPETAS
───────────────────────────────────────────────────────────

Minúsculas.

Sin espacios.

Usa guiones (-) en lugar de guiones bajos (_).

✅ Bueno:

text
index.html
acerca-de.html
mi-sitio-web.html
style.css
app.js
❌ Malo:

text
Index.HTML
Acerca De.html
mi_sitio_web.html
style.CSS
app.JS
───────────────────────────────────────────────────────────

NOMBRES DE CLASES E IDS EN HTML
───────────────────────────────────────────────────────────

Minúsculas.

Usa guiones (-) para separar palabras.

Nombres descriptivos.

✅ Bueno:

html
<div class="contenedor-principal">
<div id="hero">
<button class="btn-primario">
❌ Malo:

html
<div class="ContenedorPrincipal">
<div id="Hero">
<button class="btnPrimario">
───────────────────────────────────────────────────────────

NOMBRES DE VARIABLES EN JAVASCRIPT
───────────────────────────────────────────────────────────

Usa camelCase para variables y funciones.

Usa UPPER_CASE para constantes.

✅ Bueno:

javascript
const nombreUsuario = 'Ana';
const MAX_INTENTOS = 3;
function obtenerDatos() { ... }
❌ Malo:

javascript
const nombre_usuario = 'Ana';
const max_intentos = 3;
function obtener_datos() { ... }

PARTE 3: CÓDIGO LIMPIO (CLEAN CODE)
===============================================================================

El código limpio es fácil de leer, entender y modificar.

───────────────────────────────────────────────────────────

NOMBRES DESCRIPTIVOS
───────────────────────────────────────────────────────────

✅ Bueno:

javascript
const edadUsuario = 25;
const precioTotal = calcularPrecio(productos);
❌ Malo:

javascript
const a = 25;
const b = calcularPrecio(c);
───────────────────────────────────────────────────────────

FUNCIONES PEQUEÑAS Y ESPECÍFICAS
───────────────────────────────────────────────────────────

✅ Bueno:

javascript
function calcularTotal(productos) {
    let total = 0;
    productos.forEach(p => total += p.precio);
    return total;
}

function aplicarDescuento(total, descuento) {
    return total * (1 - descuento / 100);
}
❌ Malo:

javascript
function calcularTotalYDescuento(productos, descuento) {
    // Muchas líneas haciendo varias cosas
}
───────────────────────────────────────────────────────────

EVITA CÓDIGO DUPLICADO
───────────────────────────────────────────────────────────

✅ Bueno:

javascript
function saludar(nombre) {
    console.log('Hola, ' + nombre);
}
saludar('Ana');
saludar('Carlos');
❌ Malo:

javascript
console.log('Hola, Ana');
console.log('Hola, Carlos');
───────────────────────────────────────────────────────────

USA COMENTARIOS CON PROPÓSITO
───────────────────────────────────────────────────────────

Explica el "por qué", no el "qué".

✅ Bueno:

javascript
// Descuento especial para usuarios premium
const precioFinal = precio * 0.9;
❌ Malo:

javascript
// Multiplicar por 0.9
const precioFinal = precio * 0.9;

PARTE 6: ESTÁNDARES WEB
===============================================================================

Los estándares web aseguran que tu código funcione en diferentes navegadores y dispositivos.

───────────────────────────────────────────────────────────

HTML5
───────────────────────────────────────────────────────────

Usa <!DOCTYPE html>.

Usa lang="es".

Usa etiquetas semánticas.

Valida tu HTML con el validador del W3C.

───────────────────────────────────────────────────────────

CSS3
───────────────────────────────────────────────────────────

Usa CSS3 estándar.

Usa prefijos cuando sea necesario (-webkit-, -moz-, -ms-).

Usa unidades relativas (rem, em, %).

───────────────────────────────────────────────────────────

JAVASCRIPT
───────────────────────────────────────────────────────────

Usa ES6+ (ECMAScript 2015+).

Usa 'use strict'.

Usa let y const en lugar de var.

Usa async/await para manejar promesas.

───────────────────────────────────────────────────────────

ACCESIBILIDAD (WCAG)
───────────────────────────────────────────────────────────

Usa alt en imágenes.

Usa label en formularios.

Usa roles ARIA.

Usa contraste suficiente.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

Buenas prácticas: Conjunto de hábitos y convenciones que mejoran la calidad del código.

Código limpio: Código fácil de leer, entender y modificar.

Indentación: Sangría para mostrar la jerarquía del código.

Refactorización: Mejorar el código sin cambiar su comportamiento.

Documentación: Información sobre el código y el proyecto.

Control de versiones: Sistema para gestionar cambios en el código (Git).

Estándares web: HTML5, CSS3, ES6+, WCAG.

DRY: Don't Repeat Yourself (No te repitas).
