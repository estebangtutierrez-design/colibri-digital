PARTE 1: ¿QUÉ ES EL RENDIMIENTO WEB?
===============================================================================

El rendimiento web es la velocidad con la que una página web carga y responde a las interacciones del usuario.

───────────────────────────────────────────────────────────
¿POR QUÉ ES IMPORTANTE?
───────────────────────────────────────────────────────────

1. **Experiencia del usuario:** Las páginas rápidas son más agradables.
2. **Conversión:** Las páginas rápidas convierten mejor.
3. **SEO:** Google usa la velocidad como factor de ranking.
4. **Retención:** Los usuarios vuelven a sitios rápidos.
5. **Accesibilidad:** Las páginas rápidas son más accesibles.

───────────────────────────────────────────────────────────
ESTADÍSTICAS CLAVE
───────────────────────────────────────────────────────────

- El 53% de los usuarios abandona una página que tarda más de 3 segundos en cargar.
- Un retraso de 1 segundo en la carga reduce las conversiones un 7%.
- El 79% de los usuarios no vuelve a un sitio que consideran lento.
- Las páginas rápidas tienen un 30% más de tráfico.

PARTE 2: CORE WEB VITALS
===============================================================================

Core Web Vitals es un conjunto de métricas que Google considera esenciales para la experiencia del usuario.

───────────────────────────────────────────────────────────
1. LCP (Largest Contentful Paint) – CARGA VISUAL
───────────────────────────────────────────────────────────

Mide el tiempo que tarda en cargar el elemento más grande de la página (imagen, bloque de texto, video).

- **Bueno:** < 2.5 segundos.
- **Necesita mejora:** 2.5 - 4.0 segundos.
- **Malo:** > 4.0 segundos.

Cómo mejorar LCP:
- Optimiza imágenes (compresión, formato WebP).
- Elimina recursos que bloquean el renderizado.
- Usa un CDN (Content Delivery Network).
- Prioriza la carga del contenido visible.

───────────────────────────────────────────────────────────
2. FID (First Input Delay) – INTERACTIVIDAD
───────────────────────────────────────────────────────────

Mide el tiempo que tarda la página en responder a la primera interacción del usuario (clic, toque, tecla).

- **Bueno:** < 100 milisegundos.
- **Necesita mejora:** 100 - 300 ms.
- **Malo:** > 300 ms.

Cómo mejorar FID:
- Minimiza el JavaScript en el hilo principal.
- Usa `defer` o `async` para scripts no críticos.
- Divide el código en partes más pequeñas.
- Usa Web Workers para tareas pesadas.

───────────────────────────────────────────────────────────
3. CLS (Cumulative Layout Shift) – ESTABILIDAD VISUAL
───────────────────────────────────────────────────────────

Mide cuánto se mueve el contenido de la página mientras se carga.

- **Bueno:** < 0.1.
- **Necesita mejora:** 0.1 - 0.25.
- **Malo:** > 0.25.

Cómo mejorar CLS:
- Especifica width y height en imágenes y videos.
- Reserva espacio para anuncios y elementos dinámicos.
- Evita insertar contenido por encima del contenido existente.
- Usa `transform` en lugar de animaciones que cambian el layout.

───────────────────────────────────────────────────────────
RESUMEN DE CORE WEB VITALS
───────────────────────────────────────────────────────────

| Métrica | Lo que mide | Bueno | Malo |
|---------|-------------|-------|------|
| LCP | Carga visual | < 2.5s | > 4.0s |
| FID | Interactividad | < 100ms | > 300ms |
| CLS | Estabilidad visual | < 0.1 | > 0.25 |

PARTE 3: OPTIMIZACIÓN DE IMÁGENES
===============================================================================

Las imágenes suelen ser los recursos más pesados de una página.

───────────────────────────────────────────────────────────
1. ELIGE EL FORMATO CORRECTO
───────────────────────────────────────────────────────────

| Formato | Mejor para | Tamaño |
|---------|------------|--------|
| WebP | Todo (fotos, gráficos) | Muy pequeño |
| AVIF | Todo (nuevo, mejor compresión) | Extremadamente pequeño |
| JPEG | Fotos, imágenes con gradientes | Pequeño |
| PNG | Imágenes con transparencia | Grande |
| SVG | Logos, iconos, gráficos vectoriales | Muy pequeño |

Recomendación: Usa WebP o AVIF para la mayoría de las imágenes.

───────────────────────────────────────────────────────────
2. COMPRIME LAS IMÁGENES
───────────────────────────────────────────────────────────

Herramientas de compresión:
- **Squoosh.app** (online, gratuito)
- **TinyPNG** (online, gratuito)
- **ImageOptim** (Mac)
- **FFmpeg** (línea de comandos)
- **Sharp** (Node.js)

Ejemplo con Squoosh:
1. Sube una imagen.
2. Elige WebP.
3. Ajusta la calidad (80-90% es un buen equilibrio).
4. Descarga la imagen optimizada.

───────────────────────────────────────────────────────────
3. REDIMENSIONA LAS IMÁGENES
───────────────────────────────────────────────────────────

No uses una imagen de 4000px de ancho si la vas a mostrar a 400px.

```html
<!-- ❌ Malo: imagen grande, mostrada pequeña -->
<img src="foto-4000px.jpg" width="400" height="300">

<!-- ✅ Bueno: imagen del tamaño correcto -->
<img src="foto-400px.jpg" width="400" height="300">
───────────────────────────────────────────────────────────

USA LAZY LOADING
───────────────────────────────────────────────────────────

html
<img src="foto.jpg" alt="Descripción" loading="lazy">
Las imágenes fuera de pantalla no se cargan hasta que el usuario se desplaza cerca de ellas.

───────────────────────────────────────────────────────────

USA SRCset PARA RESOLUCIONES RESPONSIVAS
───────────────────────────────────────────────────────────

html
<img src="foto-pequeña.jpg"
     srcset="foto-pequeña.jpg 400w, foto-mediana.jpg 800w, foto-grande.jpg 1200w"
     sizes="(max-width: 600px) 100vw, 50vw"
     alt="Descripción"
     loading="lazy">
El navegador elige la imagen adecuada según el tamaño de la pantalla.

PARTE 4: OPTIMIZACIÓN DE HTML
===============================================================================

───────────────────────────────────────────────────────────

MINIMIZA EL HTML
───────────────────────────────────────────────────────────

Elimina espacios, saltos de línea y comentarios innecesarios.

Antes:

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
    <!-- Esto es un comentario -->
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Bienvenido</h1>
    <p>Este es un párrafo.</p>
</body>
</html>
Después (minimizado):

html
<!DOCTYPE html><html lang="es"><head><meta charset="UTF-8"><title>Mi Página</title><link rel="stylesheet" href="style.css"></head><body><h1>Bienvenido</h1><p>Este es un párrafo.</p></body></html>
Herramientas:

HTMLMinifier (online)

Prettier (con opción de minificar)

───────────────────────────────────────────────────────────

ELIMINA CÓDIGO INNECESARIO
───────────────────────────────────────────────────────────

Elimina comentarios de producción.

Elimina elementos no utilizados.

Elimina atributos innecesarios.

───────────────────────────────────────────────────────────

USA ELEMENTOS SEMÁNTICOS
───────────────────────────────────────────────────────────

Los elementos semánticos ayudan al navegador a entender la estructura y mejorar el rendimiento.

html
<!-- ❌ Malo -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>

<!-- ✅ Bueno -->
<header>...</header>
<nav>...</nav>
<main>...</main>

PARTE 5: OPTIMIZACIÓN DE CSS Y JAVASCRIPT
===============================================================================

───────────────────────────────────────────────────────────

MINIMIZA CSS Y JAVASCRIPT
───────────────────────────────────────────────────────────

Elimina espacios, saltos de línea y comentarios.

Herramientas:

CSSNano (CSS)

UglifyJS (JavaScript)

Terser (JavaScript)

───────────────────────────────────────────────────────────

COMBINA ARCHIVOS CSS Y JS
───────────────────────────────────────────────────────────

Reduce el número de peticiones HTTP combinando varios archivos en uno solo.

html
<!-- ❌ Malo: muchas peticiones -->
<link rel="stylesheet" href="reset.css">
<link rel="stylesheet" href="style.css">
<link rel="stylesheet" href="responsive.css">
<script src="lib1.js"></script>
<script src="lib2.js"></script>
<script src="app.js"></script>

<!-- ✅ Bueno: pocas peticiones -->
<link rel="stylesheet" href="style.min.css">
<script src="app.min.js"></script>
───────────────────────────────────────────────────────────

CARGA CSS EN EL <head>
───────────────────────────────────────────────────────────

El CSS debe cargarse en el <head> para evitar un "flash of unstyled content" (FOUC).

html
<head>
    <link rel="stylesheet" href="style.css">
</head>
───────────────────────────────────────────────────────────

CARGA JAVASCRIPT CON defer O async
───────────────────────────────────────────────────────────

html
<!-- No bloquea el renderizado, ejecuta después del DOM -->
<script src="app.js" defer></script>

<!-- No bloquea el renderizado, ejecuta tan pronto como se descarga -->
<script src="analytics.js" async></script>
───────────────────────────────────────────────────────────

USA PRELOAD PARA RECURSOS CRÍTICOS
───────────────────────────────────────────────────────────

html
<head>
    <link rel="preload" href="fuente.woff2" as="font" type="font/woff2" crossorigin>
    <link rel="preload" href="hero.jpg" as="image">
    <link rel="preconnect" href="https://fonts.googleapis.com">
</head>

PARTE 6: COMPRESIÓN DE RECURSOS
===============================================================================

La compresión reduce el tamaño de los archivos que se envían al navegador.

───────────────────────────────────────────────────────────

GZIP
───────────────────────────────────────────────────────────

El formato de compresión más común en la web.

Compatible con todos los navegadores.

Reduce el tamaño de HTML, CSS y JS entre un 60-80%.

Configuración en servidores:

Apache: mod_deflate.

Nginx: gzip on;.

Netlify/Vercel: activado por defecto.

───────────────────────────────────────────────────────────

BROTLI
───────────────────────────────────────────────────────────

Formato de compresión más moderno y eficiente que Gzip.

Mejor compresión que Gzip (hasta un 20% más).

Compatible con navegadores modernos.

Recomendado para sitios que usan HTTPS.

Configuración:

Nginx: brotli on;.

CDN: Cloudflare, Fastly, etc.
