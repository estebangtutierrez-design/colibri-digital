PARTE 1: WCAG – PAUTAS DE ACCESIBILIDAD
===============================================================================

WCAG (Web Content Accessibility Guidelines) es el estándar internacional para la accesibilidad web.

───────────────────────────────────────────────────────────
LOS 4 PRINCIPIOS DE WCAG
───────────────────────────────────────────────────────────

1. PERCEPTIBLE:
   → La información debe ser presentada de forma que todos puedan percibirla.
   → Ejemplo: Texto alternativo en imágenes, subtítulos en videos.

2. OPERABLE:
   → La interfaz debe ser operable por todos.
   → Ejemplo: Navegación por teclado, tiempo suficiente para leer.

3. COMPRENSIBLE:
   → La información y la interfaz deben ser comprensibles.
   → Ejemplo: Lenguaje claro, instrucciones, ayuda para corregir errores.

4. ROBUSTO:
   → El contenido debe ser compatible con diferentes tecnologías de asistencia.
   → Ejemplo: Código HTML válido, uso correcto de ARIA.

───────────────────────────────────────────────────────────
NIVELES DE CONFORMIDAD
───────────────────────────────────────────────────────────

- **Nivel A (Mínimo):** Los requisitos más básicos de accesibilidad.
- **Nivel AA (Recomendado):** El nivel que la mayoría de las organizaciones deben cumplir.
- **Nivel AAA (Óptimo):** El nivel más alto de accesibilidad.

───────────────────────────────────────────────────────────
CRITERIOS DE ÉXITO (EJEMPLOS)
───────────────────────────────────────────────────────────

| Criterio | Nivel | Descripción |
|----------|-------|-------------|
| 1.1.1 | A | Texto alternativo en imágenes. |
| 1.2.2 | A | Subtítulos en videos. |
| 1.4.3 | AA | Contraste mínimo 4.5:1. |
| 2.1.1 | A | Todo debe ser operable con teclado. |
| 2.4.7 | AA | Foco visible. |
| 3.3.2 | A | Etiquetas en formularios. |
| 4.1.2 | A | El código debe ser válido y accesible. |

PARTE 2: ARIA AVANZADO
===============================================================================

ARIA (Accessible Rich Internet Applications) proporciona atributos para mejorar la accesibilidad.

───────────────────────────────────────────────────────────
ROLES DE LANDMARKS (REPASO Y PROFUNDIZACIÓN)
───────────────────────────────────────────────────────────

```html
<!-- Roles de landmarks (regiones principales) -->
<header role="banner">...</header>
<nav role="navigation">...</nav>
<main role="main">...</main>
<aside role="complementary">...</aside>
<footer role="contentinfo">...</footer>

<!-- Otros roles importantes -->
<div role="search">...</div>
<div role="form">...</div>
<div role="region" aria-label="Área de contenido">...</div>
───────────────────────────────────────────────────────────
ROLES DE WIDGETS (ELEMENTOS INTERACTIVOS)
───────────────────────────────────────────────────────────

html
<!-- Botón personalizado -->
<div role="button" tabindex="0" onclick="..." onkeydown="...">
    Botón personalizado
</div>

<!-- Menú desplegable -->
<div role="menu">
    <div role="menuitem">Opción 1</div>
    <div role="menuitem">Opción 2</div>
</div>

<!-- Barra de progreso -->
<div role="progressbar" aria-valuenow="50" aria-valuemin="0" aria-valuemax="100">
    50%
</div>

<!-- Diálogo / Modal -->
<div role="dialog" aria-labelledby="dialog-titulo" aria-modal="true">
    <h2 id="dialog-titulo">Confirmación</h2>
    <p>¿Estás seguro?</p>
</div>
───────────────────────────────────────────────────────────
PROPIEDADES ARIA (ESTADOS Y PROPIEDADES)
───────────────────────────────────────────────────────────

aria-label → Etiqueta accesible

html
<button aria-label="Cerrar ventana">✕</button>
aria-labelledby → Etiqueta referenciada

html
<h2 id="modal-titulo">Confirmar eliminación</h2>
<div role="dialog" aria-labelledby="modal-titulo">...</div>
aria-describedby → Descripción adicional

html
<input type="text" aria-describedby="email-ayuda">
<span id="email-ayuda">Introduce un correo válido</span>
aria-hidden → Ocultar de lectores de pantalla

html
<div aria-hidden="true">🎨</div>
aria-expanded → Estado de expansión

html
<button aria-expanded="false" aria-controls="menu">Menú</button>
<ul id="menu" hidden>...</ul>
aria-controls → Controla otro elemento

html
<button aria-controls="contenido">Mostrar</button>
<div id="contenido">...</div>
aria-live → Actualizaciones en tiempo real

html
<div aria-live="polite">
    <p id="mensaje">El formulario se ha enviado</p>
</div>
aria-required → Campo obligatorio

html
<input type="text" required aria-required="true">
aria-invalid → Campo inválido

html
<input type="email" aria-invalid="true">
aria-current → Elemento actual (página, paso)

html
<a href="index.html" aria-current="page">Inicio</a>
aria-modal → Diálogo modal

html
<div role="dialog" aria-modal="true">...</div>

PARTE 3: NAVEGACIÓN POR TECLADO
===============================================================================

La navegación por teclado es esencial para personas con discapacidad motora.

───────────────────────────────────────────────────────────
REGLAS DE ORO PARA EL TECLADO
───────────────────────────────────────────────────────────

Todo debe ser accesible con Tab y Enter.

El orden de tabulación debe ser lógico.

El foco debe ser siempre visible.

Los elementos interactivos deben ser enfocables.

Nunca uses tabindex positivo (1, 2, 3...).

───────────────────────────────────────────────────────────
tabindex – CONTROL DEL FOCO
───────────────────────────────────────────────────────────

html
<!-- Enfocable en orden natural -->
<div tabindex="0">Elemento enfocable</div>

<!-- Solo enfocable con JavaScript -->
<div tabindex="-1">Elemento con foco programático</div>

<!-- ❌ MALO (NUNCA USAR) -->
<div tabindex="1">Primero</div>
<div tabindex="2">Segundo</div>
───────────────────────────────────────────────────────────
FOCUS VISIBLE (SIEMPRE DEBE SER VISIBLE)
───────────────────────────────────────────────────────────

css
/* ❌ MALO (nunca quites el outline sin alternativa) */
*:focus {
    outline: none;
}

/* ✅ BUENO (estilo personalizado pero visible) */
*:focus {
    outline: 3px solid #58a6ff;
    outline-offset: 2px;
}

/* ✅ BUENO (para elementos con fondo oscuro) */
*:focus-visible {
    outline: 3px solid #58a6ff;
    outline-offset: 2px;
}
───────────────────────────────────────────────────────────
EJEMPLO: MENÚ DESPLEGABLE ACCESIBLE
───────────────────────────────────────────────────────────

html
<button id="btn-menu" aria-expanded="false" aria-controls="menu" aria-haspopup="true">
    ☰ Menú
</button>
<ul id="menu" role="menu" hidden>
    <li role="menuitem"><a href="#">Inicio</a></li>
    <li role="menuitem"><a href="#">Servicios</a></li>
    <li role="menuitem"><a href="#">Contacto</a></li>
</ul>

<script>
    const btn = document.getElementById('btn-menu');
    const menu = document.getElementById('menu');

    btn.addEventListener('click', function() {
        const expanded = this.getAttribute('aria-expanded') === 'true' ? false : true;
        this.setAttribute('aria-expanded', expanded);
        menu.hidden = !expanded;
    });

    btn.addEventListener('keydown', function(event) {
        if (event.key === 'Enter' || event.key === ' ') {
            event.preventDefault();
            this.click();
        }
        if (event.key === 'Escape') {
            this.setAttribute('aria-expanded', 'false');
            menu.hidden = true;
            this.focus();
        }
    });
</script>

PARTE 4: LECTORES DE PANTALLA
===============================================================================

Los lectores de pantalla convierten el contenido web en voz o braille.

───────────────────────────────────────────────────────────
LECTORES DE PANTALLA POPULARES
───────────────────────────────────────────────────────────

NVDA (Windows) → Gratuito, muy utilizado.

VoiceOver (macOS, iOS) → Integrado en Apple.

Narrator (Windows) → Integrado en Windows.

JAWS (Windows) → Comercial, muy completo.

TalkBack (Android) → Integrado en Android.

───────────────────────────────────────────────────────────
CÓMO PRUEBA UN LECTOR DE PANTALLA
───────────────────────────────────────────────────────────

El usuario navega con teclas (Tab, Enter, flechas).

El lector lee el contenido en voz alta.

El lector anuncia el tipo de elemento (encabezado, enlace, botón).

El lector anuncia el rol y el estado (expanded, selected, etc.).

───────────────────────────────────────────────────────────
QUÉ ANUNCIA UN LECTOR DE PANTALLA
───────────────────────────────────────────────────────────

Elemento	Anuncio
<h1>Título</h1>	"Título, nivel 1, Título"
<a href="#">Enlace</a>	"Enlace, Enlace"
<button>Botón</button>	"Botón, Botón"
<img src="foto.jpg" alt="Foto">	"Foto"
<input type="text" placeholder="Nombre">	"Editar, Nombre"
<div role="button">...</div>	"Botón, ..."

PARTE 5: INCLUSIÓN DIGITAL
===============================================================================

La inclusión digital va más allá de la técnica. Es una filosofía de diseño.

───────────────────────────────────────────────────────────
PRINCIPIOS DE DISEÑO INCLUSIVO
───────────────────────────────────────────────────────────

Diseña para todos: No diseñes para una persona "promedio". Diseña para la diversidad.

Flexibilidad: Ofrece múltiples formas de hacer lo mismo (teclado, mouse, voz).

Simplicidad: El contenido debe ser claro y fácil de entender.

Perdona errores: Ayuda al usuario a corregir errores.

Consistencia: Las interacciones deben ser predecibles.

───────────────────────────────────────────────────────────
EJEMPLOS DE INCLUSIÓN DIGITAL
───────────────────────────────────────────────────────────

Color:

No uses el color como única forma de transmitir información.

Asegura un contraste suficiente.

Texto:

Usa un lenguaje claro y sencillo.

Ofrece resúmenes para contenido largo.

Usa tamaños de fuente legibles.

Navegación:

Ofrece múltiples formas de navegar (menú, buscador, mapa del sitio).

Asegura que la navegación sea predecible.

Formularios:

Ofrece instrucciones claras.

Muestra mensajes de error útiles.

Permite corregir errores fácilmente.

                const mensaje = document.getElementById('mensaje').value.trim();

                const nombreError = document.getElementById('nombre-error');
                const emailError = document.getElementById('email-error');
                const mensajeError = document.getElementById('mensaje-error');

                let valido = true;

                // Validar nombre
                if (nombre.length < 3) {
                    nombreError.textContent = '❌ El nombre debe tener al menos 3 caracteres.';
                    valido = false;
                } else {
                    nombreError.textContent = '';
                }

                // Validar email
                if (!email.includes('@') || !email.includes('.')) {
                    emailError.textContent = '❌ Introduce un correo válido.';
                    valido = false;
                } else {
                    emailError.textContent = '';
                }

                // Validar mensaje
                if (mensaje.length < 10) {
                    mensajeError.textContent = '❌ El mensaje debe tener al menos 10 caracteres.';
                    valido = false;
                } else {
                    mensajeError.textContent = '';
                }

                if (valido) {
                    notificacion.textContent = '✅ ¡Formulario enviado correctamente! Gracias por tu mensaje.';
                    notificacion.style.borderLeftColor = '#238636';
                } else {
                    notificacion.textContent = '❌ Por favor, corrige los errores marcados.';
                    notificacion.style.borderLeftColor = '#e53e3e';
                }
            });
        }

PARTE 8: ERRORES COMUNES EN ACCESIBILIDAD
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR alt EN IMÁGENES
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<img src="foto.jpg">
✅ Correcto:

html
<img src="foto.jpg" alt="Descripción de la imagen">
───────────────────────────────────────────────────────────
ERROR 2: NO ASOCIAR LABEL CON INPUT
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<input type="text" placeholder="Nombre">
✅ Correcto:

html
<label for="nombre">Nombre:</label>
<input type="text" id="nombre" placeholder="Nombre">
───────────────────────────────────────────────────────────
ERROR 3: QUITAR EL OUTLINE DEL FOCO
───────────────────────────────────────────────────────────

❌ Incorrecto:

css
*:focus { outline: none; }
✅ Correcto:

css
*:focus-visible { outline: 3px solid #58a6ff; }
───────────────────────────────────────────────────────────
ERROR 4: NO USAR roles DE LANDMARKS
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<div>Contenido principal</div>
✅ Correcto:

html
<main role="main">Contenido principal</main>
───────────────────────────────────────────────────────────
ERROR 5: NO USAR aria-label EN BOTONES SIN TEXTO
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<button>✕</button>
✅ Correcto:

html
<button aria-label="Cerrar">✕</button>
───────────────────────────────────────────────────────────
ERROR 6: USAR COLOR COMO ÚNICA FORMA DE INFORMACIÓN
───────────────────────────────────────────────────────────

❌ Incorrecto:

html
<span style="color: red;">Error</span>
✅ Correcto:

html
<span role="alert" style="color: red;">❌ Error: El campo es obligatorio</span>

PARTE 9: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

Siempre añade alt a todas las imágenes (con alt="" para decorativas).

Siempre asocia label con input usando for/id.

Nunca quites el outline del foco. Si lo personalizas, asegura que sea visible.

Usa roles de landmarks (banner, navigation, main, complementary, contentinfo).

Usa aria-label para botones sin texto visible.

No uses el color como única forma de transmitir información.

Prueba con lectores de pantalla (NVDA, VoiceOver).

Prueba la navegación por teclado (Tab, Enter, Espacio).
