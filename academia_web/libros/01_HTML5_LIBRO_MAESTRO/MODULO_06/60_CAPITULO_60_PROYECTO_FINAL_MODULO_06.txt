===============================================================================
PARTE 2: REQUISITOS DEL PROYECTO
===============================================================================

Tu sitio debe cumplir con TODOS los siguientes requisitos:

───────────────────────────────────────────────────────────
ESTRUCTURA Y ORGANIZACIÓN
───────────────────────────────────────────────────────────

✅ Estructura de carpetas profesional.
✅ Rutas relativas para todos los recursos internos.
✅ Indentación consistente.
✅ Etiquetas siempre cerradas.
✅ Atributos con comillas dobles.
✅ Nombres de archivos en minúscula con guiones.

───────────────────────────────────────────────────────────
SEMÁNTICA
───────────────────────────────────────────────────────────

✅ <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.
✅ Solo un <main> por página.
✅ Solo un <h1> por página.
✅ Jerarquía de encabezados correcta (h1 → h2 → h3).
✅ <time> para fechas.
✅ <address> para contacto.

───────────────────────────────────────────────────────────
SEO (Capítulo 52)
───────────────────────────────────────────────────────────

✅ <title> optimizado (50-60 caracteres).
✅ <meta name="description"> optimizada (150-160 caracteres).
✅ <meta name="robots" content="index, follow">.
✅ Jerarquía de encabezados correcta.
✅ URLs amigables.
✅ Imágenes con alt descriptivo.
✅ Contenido original y bien estructurado.

───────────────────────────────────────────────────────────
ACCESIBILIDAD (Capítulo 51)
───────────────────────────────────────────────────────────

✅ Roles de landmarks (banner, navigation, main, complementary, contentinfo).
✅ alt en todas las imágenes.
✅ label asociado con input en formularios.
✅ aria-label en botones sin texto.
✅ aria-describedby en campos de formulario.
✅ aria-required en campos obligatorios.
✅ Focus visible.
✅ Contraste suficiente (ratio ≥ 4.5:1).
✅ Skip link al contenido principal.
✅ Navegación por teclado funcional.

───────────────────────────────────────────────────────────
METADATOS Y OPEN GRAPH (Capítulo 55)
───────────────────────────────────────────────────────────

✅ Open Graph (og:title, og:description, og:image, og:url, og:type, og:site_name, og:locale).
✅ Twitter Cards (twitter:card, twitter:title, twitter:description, twitter:image).
✅ theme-color.
✅ apple-mobile-web-app-capable.

───────────────────────────────────────────────────────────
RENDIMIENTO Y OPTIMIZACIÓN (Capítulo 53 y 54)
───────────────────────────────────────────────────────────

✅ Imágenes en WebP con compresión.
✅ `loading="lazy"` en imágenes fuera de pantalla.
✅ `decoding="async"` en imágenes.
✅ `width` y `height` en todas las imágenes.
✅ `srcset` para imágenes responsive (opcional).
✅ CSS minimizado (style.min.css).
✅ JavaScript minimizado (app.min.js).
✅ `preload` para recursos críticos (fuentes, hero).
✅ `preconnect` para dominios externos.
✅ Core Web Vitals en verde (LCP < 2.5s, FID < 100ms, CLS < 0.1).

───────────────────────────────────────────────────────────
SEGURIDAD (Capítulo 56)
───────────────────────────────────────────────────────────

✅ HTTPS (certificado SSL/TLS).
✅ CSP (Content Security Policy) configurado.
✅ Formularios seguros (POST, validación en servidor).
✅ `rel="noopener noreferrer"` en enlaces externos.
✅ Escape de salida (textContent en lugar de innerHTML).
✅ Política de privacidad visible.

───────────────────────────────────────────────────────────
COMPATIBILIDAD (Capítulo 57)
───────────────────────────────────────────────────────────

✅ Funciona en Chrome, Firefox, Safari, Edge.
✅ Fallbacks para características no soportadas.
✅ Polyfills para funcionalidades faltantes (opcional).

───────────────────────────────────────────────────────────
BUENAS PRÁCTICAS (Capítulo 58)
───────────────────────────────────────────────────────────

✅ Código limpio y bien organizado.
✅ Documentación (comentarios en el código).
✅ README.md.
✅ Control de versiones (Git).

───────────────────────────────────────────────────────────
AUDITORÍA (Capítulo 59)
───────────────────────────────────────────────────────────

✅ Informe de auditoría (docs/informe-auditoria.md).
✅ Puntuación Lighthouse ≥ 90 en todas las categorías.
✅ Plan de acción con mejoras.

document.addEventListener('DOMContentLoaded', function() {

    const menuToggle = document.querySelector('.menu-toggle');
    const navPrincipal = document.querySelector('.nav-principal');

    if (menuToggle && navPrincipal) {
        menuToggle.addEventListener('click', function() {
            navPrincipal.classList.toggle('abierto');
            menuToggle.classList.toggle('activo');
            const isOpen = navPrincipal.classList.contains('abierto');
            menuToggle.setAttribute('aria-expanded', isOpen);
        });
    }

    // ============================================================ */
    // BOTÓN "VOLVER ARRIBA"                                       */
    // ============================================================ */
    const btnVolverArriba = document.getElementById('btn-volver-arriba');

    if (btnVolverArriba) {
        window.addEventListener('scroll', function() {
            if (window.scrollY > 300) {
                btnVolverArriba.classList.add('visible');
            } else {
                btnVolverArriba.classList.remove('visible');
            }
        });

        btnVolverArriba.addEventListener('click', function() {
            window.scrollTo({ top: 0, behavior: 'smooth' });
        });
    }

    // ============================================================ */
