PARTE 1: PRINCIPIOS BÁSICOS DE SEGURIDAD WEB
===============================================================================

───────────────────────────────────────────────────────────
LA TRÍADA CIA (CONFIDENCIALIDAD, INTEGRIDAD, DISPONIBILIDAD)
───────────────────────────────────────────────────────────

1. **Confidencialidad:** Los datos solo deben ser accesibles para quienes estén autorizados.
2. **Integridad:** Los datos no deben ser alterados sin autorización.
3. **Disponibilidad:** Los datos y servicios deben estar disponibles cuando se necesiten.

───────────────────────────────────────────────────────────
PRINCIPIOS DE SEGURIDAD PARA DESARROLLADORES
───────────────────────────────────────────────────────────

1. **Nunca confíes en la entrada del usuario.**
   → Siempre valida y sanitiza los datos que el usuario envía.

2. **Usa HTTPS siempre.**
   → Protege la comunicación entre el navegador y el servidor.

3. **Mantén el software actualizado.**
   → Las vulnerabilidades se descubren y se corrigen constantemente.

4. **Mínimo privilegio.**
   → Los usuarios y procesos solo deben tener los permisos necesarios.

5. **Defensa en profundidad.**
   → Usa múltiples capas de seguridad.

PARTE 2: XSS (CROSS-SITE SCRIPTING)
===============================================================================

XSS es un ataque en el que un atacante inyecta código malicioso en una página web.

───────────────────────────────────────────────────────────
TIPOS DE XSS
───────────────────────────────────────────────────────────

1. **XSS Reflejado (Reflected XSS):**
   → El código malicioso se inyecta a través de una URL.
   → Ejemplo: `https://sitio.com/buscar?q=<script>alert('XSS')</script>`

2. **XSS Almacenado (Stored XSS):**
   → El código malicioso se guarda en el servidor (base de datos, comentarios, etc.).
   → Cuando otro usuario carga la página, el código se ejecuta.

3. **XSS Basado en DOM (DOM-based XSS):**
   → El código malicioso se ejecuta a través de manipulación del DOM.
   → No requiere comunicación con el servidor.

───────────────────────────────────────────────────────────
EJEMPLO DE XSS ALMACENADO (PELIGROSO)
───────────────────────────────────────────────────────────

```html
<!-- ❌ MALO: Código vulnerable a XSS -->
<div>
    <p>Comentario del usuario:</p>
    <p><?php echo $comentario; ?></p>
    <!-- Si el comentario contiene <script>alert('XSS')</script>, se ejecutará -->
</div>
html
<!-- ✅ BUENO: Escapar la salida -->
<div>
    <p>Comentario del usuario:</p>
    <p><?php echo htmlspecialchars($comentario, ENT_QUOTES, 'UTF-8'); ?></p>
    <!-- Los caracteres especiales se convierten en entidades HTML -->
</div>
───────────────────────────────────────────────────────────
CÓMO PREVENIR XSS EN HTML
───────────────────────────────────────────────────────────

Escapar la salida: Usa htmlspecialchars() en PHP, o textContent en JavaScript.

javascript
// ❌ MALO (XSS)
element.innerHTML = datosUsuario;

// ✅ BUENO (escapado)
element.textContent = datosUsuario;
Usar CSP (Content Security Policy): Restringe qué recursos puede cargar la página.

Validar la entrada: Filtra y valida los datos del usuario.

Usar atributos rel="noopener noreferrer" en enlaces externos.

html
<a href="https://ejemplo.com" target="_blank" rel="noopener noreferrer">Enlace</a>

PARTE 3: FORMULARIOS SEGUROS
===============================================================================

Los formularios son la puerta de entrada a los ataques.

───────────────────────────────────────────────────────────
BUENAS PRÁCTICAS PARA FORMULARIOS SEGUROS
───────────────────────────────────────────────────────────

Usa method="POST" para datos sensibles.
→ Los datos no aparecen en la URL.

Usa HTTPS siempre.
→ La comunicación está cifrada.

Valida los datos en el servidor.
→ La validación en el cliente es para la experiencia del usuario, no para la seguridad.

Escapa la salida.
→ Siempre escapa los datos que muestras en la página.

Usa tokens CSRF.
→ Protege contra ataques de falsificación de solicitudes.

───────────────────────────────────────────────────────────
EJEMPLO DE FORMULARIO SEGURO
───────────────────────────────────────────────────────────

html
<!-- FORMULARIO SEGURO -->
<form action="/enviar" method="POST" enctype="application/x-www-form-urlencoded">

    <!-- Token CSRF (protección contra CSRF) -->
    <input type="hidden" name="csrf_token" value="token_aleatorio_seguro">

    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre" required>

    <label for="email">Correo:</label>
    <input type="email" id="email" name="email" required>

    <label for="mensaje">Mensaje:</label>
    <textarea id="mensaje" name="mensaje" required></textarea>

    <button type="submit">Enviar</button>
</form>
javascript
// Validación en el lado del cliente (experiencia de usuario)
function validarFormulario() {
    const nombre = document.getElementById('nombre').value.trim();
    const email = document.getElementById('email').value.trim();
    const mensaje = document.getElementById('mensaje').value.trim();

    if (nombre.length < 3) {
        alert('El nombre debe tener al menos 3 caracteres');
        return false;
    }

    if (!email.includes('@')) {
        alert('Introduce un correo válido');
        return false;
    }

    if (mensaje.length < 10) {
        alert('El mensaje debe tener al menos 10 caracteres');
        return false;
    }

    return true;
}

PARTE 4: HTTPS (PROTOCOLO SEGURO)
===============================================================================

HTTPS es la versión segura de HTTP.
Cifra la comunicación entre el navegador y el servidor.

───────────────────────────────────────────────────────────
¿POR QUÉ USAR HTTPS?
───────────────────────────────────────────────────────────

Seguridad: Protege los datos del usuario (contraseñas, información personal).

Confianza: Los usuarios confían más en sitios con HTTPS.

SEO: Google da preferencia a los sitios con HTTPS.

Requisito: Muchas APIs requieren HTTPS (Geolocalización, Service Workers, etc.).

───────────────────────────────────────────────────────────
CÓMO IMPLEMENTAR HTTPS
───────────────────────────────────────────────────────────

Obtén un certificado SSL/TLS (de pago o gratuito como Let's Encrypt).

Configura tu servidor para usar HTTPS.

Redirige todo el tráfico HTTP a HTTPS.

───────────────────────────────────────────────────────────
REDIRECCIÓN DE HTTP A HTTPS (Apache)
───────────────────────────────────────────────────────────

apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]
───────────────────────────────────────────────────────────
REDIRECCIÓN DE HTTP A HTTPS (Nginx)
───────────────────────────────────────────────────────────

nginx
server {
    listen 80;
    server_name ejemplo.com;
    return 301 https://ejemplo.com$request_uri;
}

PARTE 5: CONTENT SECURITY POLICY (CSP)
===============================================================================

CSP es un mecanismo que protege contra ataques XSS y otros ataques de inyección.

───────────────────────────────────────────────────────────
¿CÓMO FUNCIONA?
───────────────────────────────────────────────────────────

CSP permite definir qué recursos puede cargar la página (scripts, estilos, imágenes, etc.).
Los recursos no permitidos son bloqueados.

───────────────────────────────────────────────────────────
EJEMPLO DE CSP
───────────────────────────────────────────────────────────

html
<meta http-equiv="Content-Security-Policy" content="
    default-src 'self';
    script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net;
    style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
    font-src 'self' https://fonts.gstatic.com;
    img-src 'self' data: https://images.unsplash.com;
    connect-src 'self' https://api.ejemplo.com;
">
───────────────────────────────────────────────────────────
DIRECTIVAS DE CSP
───────────────────────────────────────────────────────────

Directiva	Controla
default-src	Fuentes por defecto para todos los tipos.
script-src	Scripts JavaScript.
style-src	Hojas de estilo CSS.
img-src	Imágenes.
font-src	Fuentes.
connect-src	Conexiones (API, WebSockets).
frame-src	Iframes.
media-src	Audio y video.
object-src	Plugins (Flash, etc.).
form-action	Destinos de formularios.
───────────────────────────────────────────────────────────
EJEMPLO COMPLETO DE CSP
───────────────────────────────────────────────────────────

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- CSP para proteger contra XSS -->
    <meta http-equiv="Content-Security-Policy" content="
        default-src 'self';
        script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net;
        style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
        font-src 'self' https://fonts.gstatic.com;
        img-src 'self' data: https://images.unsplash.com;
        connect-src 'self' https://api.ejemplo.com;
        frame-src 'self' https://www.youtube.com;
        form-action 'self';
    ">

    <title>Página con CSP</title>
</head>
<body>
    <!-- Contenido -->
</body>
</html>

PARTE 6: VALIDACIÓN Y SANITIZACIÓN DE ENTRADA
===============================================================================

La regla de oro de la seguridad: Nunca confíes en la entrada del usuario.

───────────────────────────────────────────────────────────
VALIDACIÓN (Asegurar que los datos cumplen con el formato esperado)
───────────────────────────────────────────────────────────

javascript
// Validación del lado del cliente (experiencia de usuario)
function validarEmail(email) {
    const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return regex.test(email);
}

// Validación de número
function validarEdad(edad) {
    return !isNaN(edad) && edad >= 18 && edad <= 120;
}
───────────────────────────────────────────────────────────
SANITIZACIÓN (Limpiar los datos para eliminar contenido malicioso)
───────────────────────────────────────────────────────────

javascript
// Sanitizar para prevenir XSS (remueve etiquetas HTML)
function sanitizarHTML(texto) {
    const temp = document.createElement('div');
    temp.textContent = texto;
    return temp.innerHTML;
}

// Sanitizar para URLs
function sanitizarURL(url) {
    const regex = /^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$/;
    return regex.test(url) ? url : '#';
}
───────────────────────────────────────────────────────────
EJEMPLO EN FORMULARIO
───────────────────────────────────────────────────────────

javascript
// En el servidor (PHP)
$nombre = htmlspecialchars($_POST['nombre'], ENT_QUOTES, 'UTF-8');
$email = filter_var($_POST['email'], FILTER_SANITIZE_EMAIL);
$mensaje = htmlspecialchars($_POST['mensaje'], ENT_QUOTES, 'UTF-8');

PARTE 7: PRIVACIDAD Y PROTECCIÓN DE DATOS
===============================================================================

La privacidad es un derecho fundamental.

───────────────────────────────────────────────────────────
PRINCIPIOS DE PRIVACIDAD
───────────────────────────────────────────────────────────

Minimización de datos: Solo recoge los datos que necesitas.

Consentimiento: Pide permiso antes de recoger datos.

Transparencia: Informa al usuario sobre qué datos recoges y por qué.

Seguridad: Protege los datos que recoges.

Eliminación: Elimina los datos cuando ya no los necesites.

───────────────────────────────────────────────────────────
EJEMPLO: AVISO DE PRIVACIDAD
───────────────────────────────────────────────────────────

html
<div class="aviso-privacidad">
    <p>Este sitio web utiliza cookies para mejorar la experiencia del usuario.
       Al continuar navegando, aceptas nuestra
       <a href="/politica-privacidad">Política de privacidad</a>.</p>
    <button onclick="aceptarCookies()">Aceptar</button>
</div>
───────────────────────────────────────────────────────────
EJEMPLO: SOLICITAR CONSENTIMIENTO PARA GEOLOCALIZACIÓN
───────────────────────────────────────────────────────────

javascript
// Solicitar permiso de geolocalización
if ("geolocation" in navigator) {
    // Siempre pide permiso explícito
    const permiso = await navigator.permissions.query({ name: "geolocation" });

    if (permiso.state === "prompt") {
        // El usuario no ha decidido aún
        // Mostrar explicación de por qué necesitas la ubicación
        mostrarExplicacion("Necesitamos tu ubicación para mostrarte el clima de tu ciudad.");
    }

    // Solo después de la explicación, solicitar la ubicación
    navigator.geolocation.getCurrentPosition(...);
}

PARTE 8: BUENAS PRÁCTICAS DE SEGURIDAD EN HTML
===============================================================================

───────────────────────────────────────────────────────────

USA HTTPS SIEMPRE
───────────────────────────────────────────────────────────

html
<!-- ✅ Bueno -->
<link rel="stylesheet" href="https://ejemplo.com/style.css">
<script src="https://ejemplo.com/app.js" defer></script>
<img src="https://ejemplo.com/foto.jpg" alt="Foto">

<!-- ❌ Malo (no seguro) -->
<link rel="stylesheet" href="http://ejemplo.com/style.css">
───────────────────────────────────────────────────────────

USA rel="noopener noreferrer" EN ENLACES EXTERNOS
───────────────────────────────────────────────────────────

html
<a href="https://externo.com" target="_blank" rel="noopener noreferrer">
    Enlace externo
</a>
───────────────────────────────────────────────────────────

NO USES eval() EN JAVASCRIPT
───────────────────────────────────────────────────────────

javascript
// ❌ Malo (permite ejecución de código arbitrario)
eval(datosUsuario);

// ✅ Bueno (usa alternativas seguras)
JSON.parse(datosUsuario);
───────────────────────────────────────────────────────────

USA textContent EN LUGAR DE innerHTML
───────────────────────────────────────────────────────────

javascript
// ❌ Malo (vulnerable a XSS)
element.innerHTML = datosUsuario;

// ✅ Bueno (seguro)
element.textContent = datosUsuario;
───────────────────────────────────────────────────────────

VALIDA Y SANITIZA TODA LA ENTRADA DEL USUARIO
───────────────────────────────────────────────────────────

javascript
// Ejemplo de validación y sanitización
function procesarDatos(datos) {
    // Validar
    if (!datos || typeof datos !== 'string') {
        throw new Error('Datos inválidos');
    }

    // Sanitizar
    return datos.replace(/[<>]/g, '');
}

GLOSARIO DE TÉRMINOS
===============================================================================

XSS: Cross-Site Scripting (ataque de inyección de código).

HTTPS: HTTP Seguro (cifrado).

CSP: Content Security Policy (política de seguridad de contenido).

CSRF: Cross-Site Request Forgery (falsificación de solicitud entre sitios).

Sanitización: Limpieza de datos para eliminar contenido malicioso.

Validación: Verificación de que los datos cumplen con el formato esperado.

Escapado: Conversión de caracteres especiales a entidades HTML.

Token CSRF: Token aleatorio para prevenir ataques CSRF.
