PARTE 1: ¿POR QUÉ SON IMPORTANTES LAS CONVENCIONES?
===============================================================================

Las convenciones son reglas acordadas para mantener consistencia en el código.

───────────────────────────────────────────────────────────
BENEFICIOS DE LAS CONVENCIONES
───────────────────────────────────────────────────────────

LEGIBILIDAD: El código es más fácil de leer.

CONSISTENCIA: Todos los archivos siguen el mismo patrón.

MANTENIBILIDAD: Es más fácil encontrar y modificar archivos.

COLABORACIÓN: Es más fácil trabajar en equipo.

PROFESIONALISMO: Demuestra que eres un desarrollador serio.

───────────────────────────────────────────────────────────
CONSECUENCIAS DE NO USAR CONVENCIONES
───────────────────────────────────────────────────────────

Código desordenado: Diferentes estilos en diferentes archivos.

Confusión: No sabes qué nombre usar.

Errores: Es más fácil cometer errores.

Dificultad de mantenimiento: Es difícil modificar el código.

Mala impresión: Da una imagen amateur.

───────────────────────────────────────────────────────────

Usa guiones (-) en lugar de espacios.

contacto.html
❌ MALO:

text
mi sitio web.html
acerca de.html

GUIONES (-) EN LUGAR DE GUIONES BAJOS (_)
───────────────────────────────────────────────────────────

Usa guiones (-) en lugar de guiones bajos (_) para separar palabras.

✅ BUENO:

text
mi-sitio-web.html
curso-html5.html
blog-post-1.html
❌ MALO:

text
mi_sitio_web.html
curso_html5.html
blog_post_1.html

───────────────────────────────────────────────────────────

El nombre debe decir qué contiene el archivo.

text
index.html
acerca-de.html
curso-html5.html
style.css
validacion.js
❌ MALO:

text
p1.html
p2.html
p3.html
style1.css
script.js

===============================================================================

───────────────────────────────────────────────────────────

CLASES EN HTML
───────────────────────────────────────────────────────────

Minúsculas.

───────────────────────────────────────────────────────────

IDS EN HTML
───────────────────────────────────────────────────────────

Minúsculas.

Usa guiones (-) para separar palabras.

Nombres descriptivos.

Únicos en la página.

✅ BUENO:

html
<div id="hero">
<div id="contenido-principal">
<button id="btn-enviar">
❌ MALO:

html
<div id="Hero">
<div id="contenido_principal">
<button id="btnEnviar">
───────────────────────────────────────────────────────────

CLASES EN CSS
───────────────────────────────────────────────────────────

Usa la misma nomenclatura que en HTML.

✅ BUENO:

css
.contenedor-principal {
    display: grid;
}
.articulo-destacado {
    background: #2b6cb0;
}
.btn-primario {
    background: #238636;
}
❌ MALO:

css
.ContenedorPrincipal {
    display: grid;
}
.articulo_destacado {
    background: #2b6cb0;
}
.btnPrimario {
    background: #238636;
}
───────────────────────────────────────────────────────────

NOMBRES SEMÁNTICOS
───────────────────────────────────────────────────────────

El nombre debe describir el propósito, no la apariencia.

✅ BUENO:

css
.articulo-destacado { /* Un artículo destacado */ }
.mensaje-error { /* Un mensaje de error */ }
.btn-primario { /* Botón principal */ }
❌ MALO:

css
.azul { /* Un elemento azul */ }
.rojo { /* Un elemento rojo */ }
.grande { /* Un elemento grande */ }

PARTE 4: CONVENCIONES PARA COMENTARIOS
===============================================================================

Los comentarios deben ser útiles y consistentes.

───────────────────────────────────────────────────────────

COMENTARIOS EN HTML
───────────────────────────────────────────────────────────

html
<!-- ===== HEADER ===== -->
<header>
    <!-- Logo -->
    <div class="logo">...</div>
    <!-- Navegación principal -->
    <nav>...</nav>
</header>

<!-- ===== CONTENIDO PRINCIPAL ===== -->
<main>
    <!-- Sección de bienvenida -->
    <section class="hero">...</section>
    <!-- Sección de cursos -->
    <section id="cursos">...</section>
</main>

<!-- ===== FOOTER ===== -->
<footer>...</footer>
───────────────────────────────────────────────────────────

COMENTARIOS EN CSS
───────────────────────────────────────────────────────────

css
/* ================================================================ */
/* HEADER                                                           */
/* ================================================================ */
header {
    background: #1a202c;
    padding: 20px;
}

/* ================================================================ */
/* SECCIÓN HERO                                                     */
/* ================================================================ */
.hero {
    background: #2b6cb0;
    color: white;
    padding: 60px 20px;
}
───────────────────────────────────────────────────────────

COMENTARIOS EN JAVASCRIPT
───────────────────────────────────────────────────────────

javascript
// ============================================================ */
// FUNCIÓN SALUDAR                                               */
// ============================================================ */

    // Muestra un mensaje de bienvenida en la consola
    console.log('Hola, ' + nombre);

// ============================================================ */
// INICIALIZACIÓN                                                */
// ============================================================ */
document.addEventListener('DOMContentLoaded', function() {
    saludar('Esteban');

PARTE 5: CONVENCIONES PARA VARIABLES EN JAVASCRIPT
===============================================================================

───────────────────────────────────────────────────────────

CAMELCASE (VARIABLES Y FUNCIONES)
───────────────────────────────────────────────────────────

Primera palabra en minúscula.

Palabras siguientes con mayúscula inicial.

✅ BUENO:

javascript
const nombreUsuario = 'Ana';
let contadorClics = 0;
function obtenerDatos() { ... }
function calcularTotal() { ... }
❌ MALO:

javascript
const nombre_usuario = 'Ana';
let contador_clics = 0;
function obtener_datos() { ... }
function CalcularTotal() { ... }
───────────────────────────────────────────────────────────

UPPER_CASE (CONSTANTES)
───────────────────────────────────────────────────────────

Todas mayúsculas.

Guiones bajos (_) para separar palabras.

✅ BUENO:

javascript
const MAX_INTENTOS = 3;
const URL_API = 'https://api.ejemplo.com';
const COLOR_PRIMARIO = '#2b6cb0';
❌ MALO:

javascript
const maxIntentos = 3;
const urlApi = 'https://api.ejemplo.com';
const colorPrimario = '#2b6cb0';
───────────────────────────────────────────────────────────

NOMBRES DESCRIPTIVOS
───────────────────────────────────────────────────────────

✅ BUENO:

javascript
const edadUsuario = 25;
const precioTotal = calcularPrecio(productos);
function enviarMensaje() { ... }
❌ MALO:

javascript
const a = 25;
const b = calcularPrecio(c);
function enviar() { ... }

PARTE 6: ESTÁNDARES DE CODIFICACIÓN
===============================================================================

───────────────────────────────────────────────────────────

HTML5
───────────────────────────────────────────────────────────

Usa <!DOCTYPE html>.

Usa lang="es".

Usa etiquetas semánticas.

Indenta con 2 o 4 espacios.

✅ BUENO:

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
</head>
<body>
    <h1>Bienvenido</h1>
</body>
</html>
❌ MALO:

html
<!DOCTYPE HTML>
<HTML>
<HEAD>
<TITLE>Mi pagina</TITLE>
</HEAD>
<BODY>
<H1>Bienvenido</H1>
</BODY>
</HTML>
───────────────────────────────────────────────────────────

CSS3
───────────────────────────────────────────────────────────

Usa minúsculas.

Usa propiedades CSS estándar.

Indenta correctamente.

✅ BUENO:

css
.hero {
    background: #2b6cb0;
    color: white;
    padding: 20px;
}
❌ MALO:

css
.HERO {
    BACKGROUND: #2b6cb0;
    COLOR: white;
    padding: 20px;
}
───────────────────────────────────────────────────────────

JAVASCRIPT (ES6+)
───────────────────────────────────────────────────────────

Usa let y const en lugar de var.

Usa arrow functions cuando sea apropiado.

Usa 'use strict'.

✅ BUENO:

javascript
'use strict';
const nombre = 'Ana';
const saludar = () => console.log('Hola');
❌ MALO:

javascript
var nombre = 'Ana';
function saludar() { console.log('Hola'); }
