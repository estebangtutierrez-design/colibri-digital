PARTE 1: ¿POR QUÉ ES IMPORTANTE LA PREPARACIÓN?
===============================================================================

La preparación para publicación es el paso final antes de que el mundo vea tu sitio.

───────────────────────────────────────────────────────────
¿QUÉ PUEDE SALIR MAL SIN PREPARACIÓN?
───────────────────────────────────────────────────────────

1. **Enlaces rotos:** El usuario hace clic y no encuentra nada.
2. **Imágenes que no cargan:** La página se ve incompleta.
3. **Errores de validación:** La página se muestra mal.
4. **Problemas de rendimiento:** La página carga lento.
5. **Mala experiencia de usuario:** El usuario se frustra y se va.

───────────────────────────────────────────────────────────
BENEFICIOS DE UNA BUENA PREPARACIÓN
───────────────────────────────────────────────────────────

1. **Profesionalismo:** Tu sitio da una buena impresión.
2. **Funcionalidad:** Todo funciona como debería.
3. **Rendimiento:** Tu sitio es rápido.
4. **Accesibilidad:** Todos pueden usarlo.
5. **Confianza:** El usuario confía en tu sitio.

PARTE 2: REVISIÓN FINAL DEL CÓDIGO
===============================================================================

───────────────────────────────────────────────────────────
1. VALIDAR HTML
───────────────────────────────────────────────────────────

Usa el validador del W3C para verificar que tu HTML es correcto.

**Herramienta:** https://validator.w3.org/

**Qué verifica:**
- Etiquetas sin cerrar.
- Atributos incorrectos.
- Estructura inválida.
- Etiquetas obsoletas.

**Pasos:**
1. Ve a validator.w3.org.
2. Sube tu archivo o pega la URL.
3. Revisa los errores y advertencias.
4. Corrige todos los errores.

**Ejemplo de error:**
Error: Elemento <h1> no cerrado.

text

**Solución:**
```html
<!-- ❌ Incorrecto -->
<h1>Bienvenido

<!-- ✅ Correcto -->
<h1>Bienvenido</h1>
───────────────────────────────────────────────────────────

VALIDAR CSS
───────────────────────────────────────────────────────────

Usa el validador de CSS para verificar que tu CSS es correcto.

Herramienta: https://jigsaw.w3.org/css-validator/

Qué verifica:

Propiedades incorrectas.

Valores inválidos.

Errores de sintaxis.

Ejemplo de error:

text
Error: Propiedad "colorr" no es válida.
Solución:

css
/* ❌ Incorrecto */
.hero {
    colorr: blue;
}

/* ✅ Correcto */
.hero {
    color: blue;
}
───────────────────────────────────────────────────────────

REVISAR INDENTACIÓN Y ORGANIZACIÓN
───────────────────────────────────────────────────────────

Asegúrate de que tu código esté bien organizado.

Qué revisar:

Indentación consistente (2 o 4 espacios).

Comentarios útiles.

Estructura clara.

Ejemplo de código limpio:

html
<!-- ===== HEADER ===== -->
<header>
    <div class="logo">
        <img src="logo.png" alt="Logo">
    </div>
    <nav>
        <ul>
            <li><a href="#">Inicio</a></li>
            <li><a href="#">Servicios</a></li>
        </ul>
    </nav>
</header>

PARTE 3: VERIFICACIÓN DE ENLACES
===============================================================================

Los enlaces rotos son uno de los problemas más comunes en sitios web.

───────────────────────────────────────────────────────────

VERIFICAR ENLACES INTERNOS
───────────────────────────────────────────────────────────

Los enlaces internos son los que van a otras páginas de tu sitio.

Qué verificar:

Que todos los enlaces apunten a páginas que existen.

Que las rutas sean correctas.

Ejemplo de enlace correcto:

html
<a href="acerca.html">Acerca de</a>
<a href="cursos.html">Cursos</a>
<a href="contacto.html">Contacto</a>
Ejemplo de enlace roto:

html
<a href="acerca-de.html">Acerca de</a>
<!-- Si "acerca-de.html" no existe, es un enlace roto -->
───────────────────────────────────────────────────────────

VERIFICAR ENLACES EXTERNOS
───────────────────────────────────────────────────────────

Los enlaces externos son los que van a otros sitios.

Qué verificar:

Que las URLs sean correctas.

Que los sitios existan.

Que usen HTTPS.

Ejemplo de enlace externo correcto:

html
<a href="https://www.google.com" target="_blank" rel="noopener noreferrer">Google</a>
Ejemplo de enlace externo incorrecto:

html
<a href="http://google.com" target="_blank">Google</a>
<!-- HTTP no es seguro, y falta rel="noopener noreferrer" -->
───────────────────────────────────────────────────────────

VERIFICAR ENLACES DE ANCLA (ANCLAS INTERNAS)
───────────────────────────────────────────────────────────

Los enlaces de ancla saltan a una sección de la misma página.

Qué verificar:

Que el ID exista.

Que el ID esté escrito correctamente.

Ejemplo correcto:

html
<!-- Enlace -->
<a href="#contacto">Ir a contacto</a>

<!-- Destino -->
<h2 id="contacto">Contacto</h2>
Ejemplo incorrecto:

html
<!-- Enlace -->
<a href="#contacto">Ir a contacto</a>

<!-- Destino (error: el ID no coincide) -->
<h2 id="contact">Contacto</h2>
===============================================================================
PARTE 4: REVISIÓN DE IMÁGENES
===============================================================================

Las imágenes son una parte crucial de cualquier sitio web.

───────────────────────────────────────────────────────────

VERIFICAR RUTAS DE IMÁGENES
───────────────────────────────────────────────────────────

Asegúrate de que todas las imágenes se carguen correctamente.

Qué verificar:

Que las rutas sean correctas.

Que los archivos existan.

Ejemplo correcto:

html
<img src="images/logo.png" alt="Logo">
<img src="images/hero/hero-bg.jpg" alt="Hero">
Ejemplo incorrecto:

html
<img src="images/logo.png" alt="Logo">
<!-- Si "logo.png" no está en la carpeta images, no cargará -->
───────────────────────────────────────────────────────────

VERIFICAR ALT EN IMÁGENES
───────────────────────────────────────────────────────────

Todas las imágenes deben tener alt.

Qué verificar:

Que todas las imágenes tengan alt.

Que el alt sea descriptivo.

Ejemplo correcto:

html
<img src="logo.png" alt="Logo de Lucas EGA Academy">
<img src="hero.jpg" alt="Profesor Búho AI enseñando programación">
Ejemplo incorrecto:

html
<img src="logo.png">
<!-- Falta alt -->
───────────────────────────────────────────────────────────

VERIFICAR TAMAÑO Y OPTIMIZACIÓN DE IMÁGENES
───────────────────────────────────────────────────────────

Las imágenes deben ser optimizadas para la web.

Qué verificar:

Tamaño de archivo (debe ser pequeño).

Formato (WebP recomendado).

Dimensiones (no demasiado grandes).

Herramientas de optimización:

Squoosh.app

TinyPNG

ImageOptim

Ejemplo optimizado:

html
<img src="foto.webp" alt="Descripción" loading="lazy" width="800" height="600">

PARTE 5: PRUEBAS DE RENDIMIENTO
===============================================================================

Un sitio rápido es un sitio profesional.

───────────────────────────────────────────────────────────

PRUEBA DE VELOCIDAD DE CARGA
───────────────────────────────────────────────────────────

Usa herramientas para medir la velocidad de carga.

Herramientas:

PageSpeed Insights: https://pagespeed.web.dev/

GTmetrix: https://gtmetrix.com/

Lighthouse: Integrado en Chrome DevTools.

Qué medir:

Tiempo de carga (debe ser < 3 segundos).

Core Web Vitals (LCP, FID, CLS).

───────────────────────────────────────────────────────────

PRUEBA DE RESPONSIVE
───────────────────────────────────────────────────────────

Asegúrate de que tu sitio se vea bien en todos los dispositivos.

Cómo probar:

Usa las herramientas de desarrollo (DevTools) y el modo "Responsive".

Prueba en diferentes tamaños de pantalla.

Prueba en dispositivos reales si es posible.

Tamaños a probar:

Desktop (1920x1080)

Tablet (768x1024)

Móvil (375x812)

───────────────────────────────────────────────────────────

PRUEBA DE ACCESIBILIDAD
───────────────────────────────────────────────────────────

Asegúrate de que tu sitio sea accesible.

Herramientas:

Lighthouse: Audita accesibilidad.

WAVE: https://wave.webaim.org/

Qué verificar:

Contraste suficiente.

Alt en imágenes.

Labels en formularios.

Navegación por teclado.

PARTE 6: CHECKLIST DE LANZAMIENTO
===============================================================================

Antes de publicar tu sitio, revisa esta lista.

───────────────────────────────────────────────────────────
CHECKLIST PRE-LANZAMIENTO
───────────────────────────────────────────────────────────

CÓDIGO:
□ ¿El HTML está validado (sin errores)?
□ ¿El CSS está validado (sin errores)?
□ ¿El código está bien indentado?
□ ¿Hay comentarios útiles?
□ ¿No hay código muerto (comentado o sin usar)?

ENLACES:
□ ¿Todos los enlaces internos funcionan?
□ ¿Todos los enlaces externos funcionan?
□ ¿Los enlaces de ancla (internos) funcionan?
□ ¿Los enlaces externos tienen rel="noopener noreferrer"?

IMÁGENES:
□ ¿Todas las imágenes tienen alt?
□ ¿Todas las imágenes se cargan?
□ ¿Las imágenes están optimizadas (formato, tamaño)?
□ ¿Las imágenes tienen loading="lazy" (fuera de pantalla)?

RENDIMIENTO:
□ ¿El sitio carga en menos de 3 segundos?
□ ¿Core Web Vitals están en verde?
□ ¿El sitio es responsive?
□ ¿El sitio es accesible?

SEO:
□ ¿Tiene título (title) optimizado?
□ ¿Tiene meta descripción?
□ ¿Tiene Open Graph (para redes sociales)?
□ ¿Tiene Twitter Cards?

SEGURIDAD:
□ ¿Usa HTTPS?
□ ¿Tiene CSP (Content Security Policy)?
□ ¿Los formularios son seguros?
□ ¿Los enlaces externos son seguros?

GENERAL:
□ ¿El favicon está configurado?
□ ¿El sitio funciona en diferentes navegadores?
□ ¿La página de error 404 está configurada?
□ ¿El sitio está en control de versiones (Git)?
