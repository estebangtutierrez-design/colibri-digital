PARTE 1: REPASO DEL MÓDULO I – FUNDAMENTOS DE HTML
===============================================================================

El Módulo I es la base de todo. Sin estos conceptos, nada más tiene sentido.

───────────────────────────────────────────────────────────
CONCEPTOS CLAVE
───────────────────────────────────────────────────────────

1. **¿Qué es HTML?** HyperText Markup Language.
2. **Historia de HTML:** Tim Berners-Lee, 1991 → HTML5, 2014.
3. **Cliente y servidor:** El navegador (cliente) pide, el servidor responde.
4. **Estructura básica:** `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`.
5. **Etiquetas y elementos:** Apertura, cierre, contenido, vacías.
6. **Atributos:** Información adicional en etiquetas.
7. **Comentarios:** `<!-- comentario -->`.

───────────────────────────────────────────────────────────
ERRORES COMUNES
───────────────────────────────────────────────────────────

❌ Olvidar cerrar etiquetas.
❌ Olvidar el `<!DOCTYPE html>`.
❌ Olvidar `lang="es"` en `<html>`.
❌ Múltiples `<h1>` por página.
❌ No usar indentación.

───────────────────────────────────────────────────────────
EJEMPLO DE ESTRUCTURA CORRECTA
───────────────────────────────────────────────────────────

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Página</title>
</head>
<body>
    <h1>Título principal</h1>
    <p>Contenido...</p>
</body>
</html>

PARTE 2: REPASO DEL MÓDULO II – CONSTRUYENDO PÁGINAS WEB
===============================================================================

El Módulo II te enseñó a construir páginas web con contenido real.

───────────────────────────────────────────────────────────
CONCEPTOS CLAVE
───────────────────────────────────────────────────────────

Encabezados: h1 a h6 (jerarquía).

Párrafos: <p>.

Saltos y separadores: <br>, <hr>.

Formato de texto: <strong>, <em>, <mark>, <del>, <ins>, <sub>, <sup>.

Citas: <blockquote>, <q>, <cite>, <address>.

Código fuente: <code>, <pre>, <kbd>, <samp>, <var>.

Caracteres especiales: &lt;, &gt;, &amp;, &copy;, &nbsp;.

Listas: <ul>, <ol>, <dl>, <li>, <dt>, <dd>.

Enlaces: <a href="">, target="_blank", rel="noopener noreferrer".

Imágenes: <img src="" alt="">, loading="lazy".

Audio y video: <audio>, <video>, <source>, <track>.

Tablas: <table>, <tr>, <td>, <th>, <thead>, <tbody>, <tfoot>, colspan, rowspan.

Formularios: <form>, <input>, <textarea>, <select>, <option>, <button>, <fieldset>, <legend>, <label>.

HTML semántico: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.

───────────────────────────────────────────────────────────
ERRORES COMUNES
───────────────────────────────────────────────────────────

❌ No usar alt en imágenes.
❌ No asociar label con input.
❌ Confundir <section> y <article>.
❌ Usar tablas para maquetación.
❌ Olvidar action y method en formularios.

PARTE 3: REPASO DEL MÓDULO III – ESTRUCTURA PROFESIONAL
===============================================================================

El Módulo III te enseñó a organizar proyectos como un profesional.

───────────────────────────────────────────────────────────
CONCEPTOS CLAVE
───────────────────────────────────────────────────────────

<div> y <span>: Contenedores de bloque y línea.

Semántica avanzada: <figure>, <figcaption>, <details>, <summary>, <time>, <address>, <dialog>, <progress>, <meter>.

Metaetiquetas: <meta charset>, <meta name="viewport">, <meta name="description">, <meta name="robots">.

Open Graph: og:title, og:description, og:image, og:url, og:type, og:site_name, og:locale.

Twitter Cards: twitter:card, twitter:title, twitter:description, twitter:image.

Rutas: Absolutas vs. relativas.

Estructura de carpetas: css/, js/, images/, fonts/, assets/, docs/.

Integración CSS: Inline, interno, externo.

Integración JavaScript: defer, async, ubicación.

Comentarios: <!-- -->, /* */, //.

Buenas prácticas: Indentación, nomenclatura, organización.

───────────────────────────────────────────────────────────
ERRORES COMUNES
───────────────────────────────────────────────────────────

❌ Usar <div> cuando hay una etiqueta semántica.
❌ Rutas relativas incorrectas.
❌ No usar defer en scripts.
❌ No documentar el código.
❌ No organizar archivos en carpetas.

PARTE 5: REPASO DEL MÓDULO V – HTML5 PROFESIONAL Y APIS
===============================================================================

El Módulo V te enseñó a usar APIs modernas del navegador.

───────────────────────────────────────────────────────────
CONCEPTOS CLAVE
───────────────────────────────────────────────────────────

LocalStorage: setItem(), getItem(), removeItem(), clear(), JSON.stringify(), JSON.parse().

SessionStorage: sessionStorage, diferencias con LocalStorage.

Geolocalización: navigator.geolocation.getCurrentPosition(), watchPosition(), clearWatch().

Drag and Drop: draggable, dataTransfer, dragstart, dragend, dragover, drop.

Web Workers: new Worker(), postMessage(), onmessage, onerror, terminate().

IndexedDB: indexedDB.open(), createObjectStore(), transaction(), add(), get(), put(), delete().

Notificaciones: Notification.requestPermission(), new Notification(), onclick, onclose, onaction.

Fetch API: fetch(), response.json(), async/await, try/catch.

PWA: manifest.json, Service Worker (install, activate, fetch).

───────────────────────────────────────────────────────────
ERRORES COMUNES
───────────────────────────────────────────────────────────

❌ No usar JSON.stringify() para objetos en LocalStorage.
❌ No manejar errores en Fetch.
❌ No verificar soporte de geolocalización.
❌ No usar event.preventDefault() en dragover.
❌ No manejar onerror en Web Workers.
❌ No usar HTTPS para PWA.

PARTE 10: PREGUNTAS FRECUENTES DEL LIBRO HTML5
===============================================================================

───────────────────────────────────────────────────────────
¿Qué es HTML5?
───────────────────────────────────────────────────────────

HTML5 es la quinta versión del lenguaje de marcado HTML. Es el estándar actual para crear páginas web.

───────────────────────────────────────────────────────────
¿Cuál es la diferencia entre HTML, CSS y JavaScript?
───────────────────────────────────────────────────────────

HTML → Estructura.

CSS → Estilo.

JavaScript → Interactividad.

───────────────────────────────────────────────────────────
¿Por qué usar etiquetas semánticas?
───────────────────────────────────────────────────────────

Por SEO, accesibilidad y mantenibilidad.

───────────────────────────────────────────────────────────
¿Qué es la accesibilidad web?
───────────────────────────────────────────────────────────

Es la práctica de hacer que las páginas web sean utilizables por personas con discapacidad.

───────────────────────────────────────────────────────────
¿Qué es el SEO?
───────────────────────────────────────────────────────────

Search Engine Optimization. Es la práctica de optimizar una página para motores de búsqueda.

───────────────────────────────────────────────────────────
¿Qué son las Core Web Vitals?
───────────────────────────────────────────────────────────

Son métricas de Google para medir la experiencia del usuario: LCP (carga), FID (interactividad), CLS (estabilidad).

───────────────────────────────────────────────────────────
¿Qué es una PWA?
───────────────────────────────────────────────────────────

Progressive Web App. Es una página web que se comporta como una aplicación nativa.

───────────────────────────────────────────────────────────
¿Qué es Git?
───────────────────────────────────────────────────────────

Un sistema de control de versiones para gestionar cambios en el código.

───────────────────────────────────────────────────────────
¿Qué es GitHub?
───────────────────────────────────────────────────────────

Una plataforma para alojar repositorios Git.
