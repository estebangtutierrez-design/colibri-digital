PARTE 1: ¿QUÉ ES UN DOMINIO?
===============================================================================

Un dominio es la dirección que las personas escriben en su navegador para visitar tu sitio web.

───────────────────────────────────────────────────────────
EJEMPLOS DE DOMINIOS
───────────────────────────────────────────────────────────
lucasEGA.com
google.com
wikipedia.org
github.io

text

───────────────────────────────────────────────────────────
ESTRUCTURA DE UN DOMINIO
───────────────────────────────────────────────────────────

Un dominio tiene varias partes:
www.ejemplo.com
│ │ │
│ │ └── Extensión (.com, .org, .net, .es, etc.)
│ └─────────── Dominio de segundo nivel (ejemplo)
└─────────────── Subdominio (www, blog, tienda, etc.)

text

───────────────────────────────────────────────────────────
TIPOS DE EXTENSIONES DE DOMINIO
───────────────────────────────────────────────────────────

| Extensión | Propósito | Ejemplo |
|-----------|-----------|---------|
| .com | Comercial (general) | google.com |
| .org | Organizaciones | wikipedia.org |
| .net | Redes | example.net |
| .es | España | lucasEGA.es |
| .mx | México | ejemplo.mx |
| .io | Tecnología | github.io |
| .edu | Educación | harvard.edu |
| .gov | Gobierno | usa.gov |

───────────────────────────────────────────────────────────
¿CÓMO FUNCIONA UN DOMINIO?
───────────────────────────────────────────────────────────

1. El usuario escribe el dominio en el navegador.
2. El navegador consulta los servidores DNS.
3. Los servidores DNS traducen el dominio a una dirección IP.
4. El navegador se conecta al servidor con esa IP.
5. El servidor envía la página web.

**Analogía:** Un dominio es como el nombre de una persona.
La IP es como su número de teléfono.
El DNS es como la guía telefónica que asocia el nombre con el número.

PARTE 2: ¿QUÉ ES UN HOSTING?
===============================================================================

El hosting es un servicio que almacena los archivos de tu sitio web en un servidor conectado a Internet.

───────────────────────────────────────────────────────────
TIPOS DE HOSTING
───────────────────────────────────────────────────────────

1. **HOSTING COMPARTIDO (SHARED HOSTING)**
   - Varios sitios web en el mismo servidor.
   - Barato y fácil de usar.
   - Ideal para sitios pequeños y principiantes.

   **Ejemplos:** HostGator, GoDaddy, SiteGround.

2. **HOSTING VPS (VIRTUAL PRIVATE SERVER)**
   - Un servidor virtual dedicado a ti.
   - Más rendimiento y control.
   - Ideal para sitios medianos.

3. **HOSTING DEDICADO (DEDICATED SERVER)**
   - Un servidor físico solo para ti.
   - Máximo rendimiento y control.
   - Ideal para sitios grandes.

4. **HOSTING EN LA NUBE (CLOUD HOSTING)**
   - Escalable y flexible.
   - Pagas por lo que usas.
   - Ideal para sitios con tráfico variable.

   **Ejemplos:** AWS, Google Cloud, DigitalOcean.

5. **HOSTING ESTÁTICO (STATIC HOSTING)**
   - Solo para archivos estáticos (HTML, CSS, JS).
   - Muy rápido y económico.
   - Ideal para sitios sin backend.

   **Ejemplos:** GitHub Pages, Netlify, Vercel.

───────────────────────────────────────────────────────────
COMPARATIVA DE TIPOS DE HOSTING
───────────────────────────────────────────────────────────

| Tipo | Precio | Rendimiento | Control | Ideal para |
|------|--------|-------------|---------|------------|
| Compartido | $ | Bajo | Bajo | Principiantes |
| VPS | $$ | Medio | Medio | Sitios medianos |
| Dedicado | $$$ | Alto | Alto | Sitios grandes |
| Nube | $$-$$$ | Alto | Alto | Tráfico variable |
| Estático | $-$$ | Muy alto | Medio | Sitios estáticos |

───────────────────────────────────────────────────────────
QUÉ BUSCAR EN UN HOSTING
───────────────────────────────────────────────────────────

1. **UPTIME:** El servidor debe estar funcionando siempre (99.9%).
2. **VELOCIDAD:** El servidor debe ser rápido.
3. **SOPORTE:** Debe tener soporte técnico disponible.
4. **SEGURIDAD:** Debe tener medidas de seguridad (SSL, backups).
5. **PRECIO:** Debe ajustarse a tu presupuesto.
6. **FACILIDAD DE USO:** Debe ser fácil de administrar.

PARTE 3: ¿CÓMO FUNCIONA EL DNS?
===============================================================================

DNS significa Domain Name System (Sistema de Nombres de Dominio).
Es el sistema que traduce los nombres de dominio en direcciones IP.

───────────────────────────────────────────────────────────
¿POR QUÉ ES NECESARIO EL DNS?
───────────────────────────────────────────────────────────

Las computadoras se comunican usando direcciones IP.
Los humanos usamos nombres de dominio.

El DNS traduce nombres a IPs para que los humanos puedan usar nombres fáciles de recordar.

───────────────────────────────────────────────────────────
EJEMPLO DE FUNCIONAMIENTO DEL DNS
───────────────────────────────────────────────────────────

1. El usuario escribe: `lucasEGA.com`
2. El navegador consulta el DNS.
3. El DNS devuelve la IP: `192.168.1.1`
4. El navegador se conecta a esa IP.
5. El servidor envía la página web.

───────────────────────────────────────────────────────────
TIPOS DE REGISTROS DNS
───────────────────────────────────────────────────────────

| Registro | Propósito | Ejemplo |
|----------|-----------|---------|
| A | Asocia un dominio a una IP | lucasEGA.com → 192.168.1.1 |
| CNAME | Asocia un dominio a otro dominio | www.lucasEGA.com → lucasEGA.com |
| MX | Asocia un dominio a un servidor de correo | lucasEGA.com → mail.google.com |
| TXT | Almacena texto (verificaciones) | lucasEGA.com → "verification=..." |
| NS | Define los servidores DNS | lucasEGA.com → ns1.hosting.com |

PARTE 4: CÓMO PUBLICAR UN SITIO WEB
===============================================================================

───────────────────────────────────────────────────────────
PASOS PARA PUBLICAR UN SITIO WEB
───────────────────────────────────────────────────────────

1. **ELEGIR UN HOSTING:** Selecciona un proveedor de hosting.
2. **REGISTRAR UN DOMINIO:** Elige y registra un dominio.
3. **CONFIGURAR DNS:** Apunta el dominio al hosting.
4. **SUBIR ARCHIVOS:** Sube tus archivos al hosting.
5. **VERIFICAR:** Asegúrate de que todo funciona.

───────────────────────────────────────────────────────────
PUBLICAR CON GITHUB PAGES (GRATUITO)
───────────────────────────────────────────────────────────

**Paso 1:** Crea un repositorio en GitHub.
**Paso 2:** Sube tus archivos HTML, CSS y JS.
**Paso 3:** Activa GitHub Pages en la configuración.
**Paso 4:** Tu sitio estará disponible en `usuario.github.io/repositorio`.

**Ejemplo de URL:** `estebanm10.github.io/mi-sitio`

───────────────────────────────────────────────────────────
PUBLICAR CON NETLIFY (GRATUITO)
───────────────────────────────────────────────────────────

**Paso 1:** Crea una cuenta en Netlify.
**Paso 2:** Arrastra tu carpeta de proyecto.
**Paso 3:** Netlify genera una URL automáticamente.
**Paso 4:** Tu sitio estará disponible en `nombre-sitio.netlify.app`.

**Ejemplo de URL:** `lucas-ega.netlify.app`

───────────────────────────────────────────────────────────
PUBLICAR CON HOSTING DE PAGO
───────────────────────────────────────────────────────────

**Paso 1:** Compra un hosting y un dominio.
**Paso 2:** Conéctate al hosting vía FTP.
**Paso 3:** Sube tus archivos a la carpeta `public_html`.
**Paso 4:** Tu sitio estará disponible en tu dominio.

**Ejemplo de URL:** `www.lucasEGA.com`

PARTE 5: HERRAMIENTAS PARA PUBLICAR
===============================================================================

───────────────────────────────────────────────────────────
1. FTP (FILE TRANSFER PROTOCOL)
───────────────────────────────────────────────────────────

FTP es un protocolo para transferir archivos entre tu computadora y el servidor.

**Clientes FTP populares:**
- FileZilla (gratuito)
- Cyberduck (gratuito)
- WinSCP (gratuito)

**Cómo usar FTP:**
1. Abre el cliente FTP.
2. Introduce las credenciales del hosting.
3. Conéctate al servidor.
4. Arrastra los archivos a la carpeta `public_html`.

───────────────────────────────────────────────────────────
2. GIT
───────────────────────────────────────────────────────────

Git también se puede usar para publicar sitios web.

**Plataformas que usan Git:**
- GitHub Pages
- GitLab Pages
- Netlify (con Git)
- Vercel (con Git)

**Cómo usar Git para publicar:**
1. Sube tu código a un repositorio Git.
2. Configura la plataforma para desplegar automáticamente.
3. Cada `git push` actualiza el sitio.

───────────────────────────────────────────────────────────
3. PANEL DE CONTROL (CPANEL)
───────────────────────────────────────────────────────────

La mayoría de los hostings de pago incluyen cPanel.

**cPanel permite:**
- Gestionar archivos (File Manager).
- Gestionar bases de datos.
- Gestionar dominios.
- Gestionar correos electrónicos.
- Ver estadísticas.

───────────────────────────────────────────────────────────
CASO: PUBLICAR LUCAS EGA ACADEMY

PARTE 8: ERRORES COMUNES EN HOSTING Y DOMINIOS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: REGISTRO DE DOMINIO SIN HOSTING
───────────────────────────────────────────────────────────

❌ Incorrecto:
Registrar un dominio sin contratar hosting.

text
Consecuencia: El dominio no tiene a dónde apuntar.

Solución: Contrata hosting al mismo tiempo o apunta el dominio a un servicio como GitHub Pages.

───────────────────────────────────────────────────────────
ERROR 2: DNS INCORRECTO
───────────────────────────────────────────────────────────

❌ Incorrecto:
Configurar DNS incorrectamente.

text
Consecuencia: El dominio no apunta al hosting.

Solución: Verifica los registros DNS y espera a que se propaguen (puede tardar 24-48 horas).

───────────────────────────────────────────────────────────
ERROR 3: SUBIR ARCHIVOS A LA CARPETA INCORRECTA
───────────────────────────────────────────────────────────

❌ Incorrecto:
Subir archivos a la carpeta raíz en lugar de public_html.

text
Consecuencia: Los archivos no son accesibles desde el navegador.

Solución: Sube los archivos a `public_html` o `www` (dependiendo del hosting).

───────────────────────────────────────────────────────────
ERROR 4: NO USAR HTTPS
───────────────────────────────────────────────────────────

❌ Incorrecto:
Publicar sin SSL/HTTPS.

text
Consecuencia: El sitio no es seguro y los navegadores muestran advertencias.

Solución: Activa SSL (Let's Encrypt es gratuito).

───────────────────────────────────────────────────────────
ERROR 5: NO HACER BACKUP
───────────────────────────────────────────────────────────

❌ Incorrecto:
No tener copia de seguridad.

text
Consecuencia: Si algo falla, pierdes todo.

Solución: Siempre mantén una copia de seguridad de tus archivos.

===============================================================================
PARTE 9: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Contrata hosting y dominio al mismo tiempo (o apunta el dominio a un servicio gratuito).
2. Verifica la configuración de DNS.
3. Sube los archivos a la carpeta correcta (public_html o www).
4. Activa SSL (Let's Encrypt es gratuito).
5. Mantén copias de seguridad (backups).
