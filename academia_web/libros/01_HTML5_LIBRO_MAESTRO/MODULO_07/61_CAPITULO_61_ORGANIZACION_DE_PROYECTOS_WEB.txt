PARTE 1: ¿POR QUÉ ES IMPORTANTE LA ORGANIZACIÓN?
===============================================================================

La organización de un proyecto web es la base de la productividad.

───────────────────────────────────────────────────────────
BENEFICIOS DE UN PROYECTO ORGANIZADO
───────────────────────────────────────────────────────────

1. **MANTENIBILIDAD:** Es fácil encontrar y modificar archivos.
2. **ESCALABILIDAD:** Es fácil añadir nuevas funcionalidades.
3. **COLABORACIÓN:** Es fácil trabajar en equipo.
4. **PROFESIONALISMO:** Demuestra que eres un desarrollador serio.
5. **EFICIENCIA:** Ahorras tiempo buscando archivos.
6. **SEGURIDAD:** Es más fácil auditar y proteger.

───────────────────────────────────────────────────────────
CONSECUENCIAS DE UN PROYECTO DESORGANIZADO
───────────────────────────────────────────────────────────

1. **Pérdida de tiempo:** No encuentras lo que necesitas.
2. **Errores:** Es más fácil cometer errores.
3. **Duplicación:** Puedes tener archivos duplicados.
4. **Dificultad de mantenimiento:** Es difícil modificar el código.
5. **Mala impresión:** Da una imagen amateur.

PARTE 4: CÓMO ENLAZAR ARCHIVOS DESDE EL HTML
===============================================================================

Una vez que tienes la estructura de carpetas, debes enlazar los archivos desde el HTML usando rutas relativas.

───────────────────────────────────────────────────────────
ENLAZAR CSS
───────────────────────────────────────────────────────────

```html
<!-- Desde index.html (en la raíz) -->
<link rel="stylesheet" href="css/style.css">

<!-- Desde pages/pagina.html (en una subcarpeta) -->
<link rel="stylesheet" href="../css/style.css">
───────────────────────────────────────────────────────────
ENLAZAR JAVASCRIPT
───────────────────────────────────────────────────────────

html
<!-- Desde index.html (en la raíz) -->
<script src="js/app.js" defer></script>

<!-- Desde pages/pagina.html (en una subcarpeta) -->
<script src="../js/app.js" defer></script>
───────────────────────────────────────────────────────────
ENLAZAR IMÁGENES
───────────────────────────────────────────────────────────

html
<!-- Desde index.html (en la raíz) -->
<img src="images/logo.png" alt="Logo">

<!-- Desde pages/pagina.html (en una subcarpeta) -->
<img src="../images/logo.png" alt="Logo">
───────────────────────────────────────────────────────────
ENLAZAR OTRAS PÁGINAS (ENLACES ENTRE HTML)
───────────────────────────────────────────────────────────

html
<!-- Desde index.html (en la raíz) -->
<a href="acerca.html">Acerca de</a>
<a href="cursos.html">Cursos</a>

<!-- Desde pages/articulo.html a una página en la raíz -->
<a href="../index.html">Inicio</a>

CÓDIGO HTML (index.html)

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Lucas EGA Academy</title>

    <!-- CSS (en carpeta css/) -->
    <link rel="stylesheet" href="css/style.css">

    <!-- Favicon (en carpeta assets/) -->
    <link rel="icon" href="assets/favicon.ico" type="image/x-icon">

    <!-- JavaScript (en carpeta js/, con defer) -->
    <script src="js/app.js" defer></script>
</head>
<body>

    <!-- Logo (en carpeta images/) -->
    <header>
        <img src="images/logo.png" alt="Logo de Lucas EGA Academy" width="200" height="100">
        <nav>
            <ul>
                <li><a href="index.html">Inicio</a></li>
                <li><a href="cursos.html">Cursos</a></li>
                <li><a href="contacto.html">Contacto</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <h1>Bienvenido a Lucas EGA Academy</h1>
        <p>Aprende programación con el Profesor Búho AI.</p>
        <img src="images/hero/hero-bg.jpg" alt="Profesor Búho AI" width="600" height="400">

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

Carpeta: Contenedor de archivos en un sistema de archivos.

Estructura de carpetas: Organización de archivos en carpetas.

Assets: Recursos del proyecto (imágenes, fuentes, etc.).

Raíz del proyecto: Carpeta principal del proyecto.

Ruta relativa: Dirección de un archivo desde la ubicación actual.

Mantenibilidad: Capacidad de un código para ser modificado fácilmente.

Escalabilidad: Capacidad de un proyecto para crecer.

RESUMEN DEL CAPÍTULO
===============================================================================

En este capítulo aprendiste:

Estructura de carpetas profesional: css/, js/, images/, fonts/, assets/, docs/.

Organización de recursos: CSS, JavaScript, imágenes, fuentes, assets.

Ejemplo real: estructura de Lucas EGA Academy.

Rutas relativas: ./, ../ para enlazar archivos.

Buenas prácticas: minúsculas, sin espacios, guiones en lugar de guiones bajos.

Subdivisión de imágenes por categorías.

Ahora puedes organizar cualquier proyecto web como un profesional.

organización, carpetas, estructura, assets, css, js, images, fonts, docs, rutas, mantenibilidad, escalabilidad

Errores frecuentes:

Todos los archivos en la raíz

Nombres de carpetas inconsistentes

Rutas incorrectas

Sin subdivisión de imágenes

Archivos de recursos en la raíz

Preguntas frecuentes:

¿Cuál es la mejor estructura de carpetas? La que mantiene separados CSS, JS, imágenes y recursos.

¿Por qué usar rutas relativas? Porque son portables (puedes mover el proyecto sin romper enlaces).

¿Cómo sé si una ruta está correcta? Abre las herramientas de desarrollo (F12) y ve a la pestaña Network.

¿Por qué usar minúsculas en nombres de carpetas? Para evitar problemas en servidores.
