PARTE 1: ¿QUÉ ES LA ACCESIBILIDAD WEB?
===============================================================================

La accesibilidad web es la práctica de hacer que los sitios web sean utilizables por todas las personas, independientemente de sus capacidades o discapacidades.

───────────────────────────────────────────────────────────
TIPOS DE DISCAPACIDADES CONSIDERADAS
───────────────────────────────────────────────────────────

1. VISUAL:
   - Ceguera total.
   - Baja visión.
   - Daltonismo (incapacidad para distinguir colores).

2. AUDITIVA:
   - Sordera total.
   - Hipoacusia (pérdida parcial de audición).

3. MOTORA:
   - Parálisis.
   - Temblor.
   - Limitaciones de movimiento.
   - Uso de solo teclado o dispositivos especiales.

4. COGNITIVA:
   - Dificultades de aprendizaje.
   - Trastornos de atención.
   - Dislexia.
   - Problemas de memoria.

───────────────────────────────────────────────────────────
WCAG – PAUTAS DE ACCESIBILIDAD
───────────────────────────────────────────────────────────

Las WCAG (Web Content Accessibility Guidelines) son el estándar internacional para la accesibilidad web.

4 PRINCIPIOS:

1. PERCEPTIBLE: La información debe ser presentada de forma que todos puedan percibirla.
   - Texto alternativo en imágenes.
   - Subtítulos en videos.
   - Contraste adecuado.

2. OPERABLE: La interfaz debe ser operable por todos.
   - Navegación por teclado.
   - Tiempos suficientes para leer y usar.
   - No usar contenido que cause convulsiones.

3. COMPRENSIBLE: La información y la interfaz deben ser comprensibles.
   - Lenguaje claro y simple.
   - Instrucciones claras.
   - Ayuda para corregir errores.

4. ROBUSTO: El contenido debe ser compatible con diferentes tecnologías de asistencia.
   - Código HTML válido.
   - Uso correcto de ARIA.
   - Compatibilidad con lectores de pantalla.

PARTE 2: ROLES ARIA – DEFINIR LA ESTRUCTURA
===============================================================================

Los roles ARIA definen el propósito de un elemento.

───────────────────────────────────────────────────────────
ROLES DE LANDMARKS (REGIÓN)
───────────────────────────────────────────────────────────

Los roles de landmarks definen las regiones principales de una página.

- `role="banner"` → Cabecera (equivalente a `<header>`).
- `role="navigation"` → Navegación (equivalente a `<nav>`).
- `role="main"` → Contenido principal (equivalente a `<main>`).
- `role="complementary"` → Contenido complementario (equivalente a `<aside>`).
- `role="contentinfo"` → Pie de página (equivalente a `<footer>`).
- `role="search"` → Área de búsqueda.
- `role="form"` → Formulario.

───────────────────────────────────────────────────────────
EJEMPLO DE ROLES DE LANDMARKS
───────────────────────────────────────────────────────────

<header role="banner">
    <h1>Mi Sitio Web</h1>
</header>

<nav role="navigation" aria-label="Navegación principal">
    <ul>
        <li><a href="index.html">Inicio</a></li>
        <li><a href="servicios.html">Servicios</a></li>
    </ul>
</nav>

<main role="main">
    <h2>Contenido principal</h2>
    <p>Este es el contenido más importante de la página.</p>
</main>

<aside role="complementary">
    <h3>Artículos relacionados</h3>
    <ul>
        <li><a href="#">Enlace 1</a></li>
        <li><a href="#">Enlace 2</a></li>
    </ul>
</aside>

<footer role="contentinfo">
    <p>&copy; 2026</p>
</footer>

───────────────────────────────────────────────────────────
OTROS ROLES ÚTILES
───────────────────────────────────────────────────────────

- `role="button"` → Botón (si usas un div en lugar de button).
- `role="link"` → Enlace (si usas un div en lugar de a).
- `role="heading"` → Encabezado (si usas un div en lugar de h1-h6).
- `role="list"` → Lista (para ul/ol no estándar).
- `role="listitem"` → Elemento de lista.
- `role="dialog"` → Ventana modal.
- `role="alert"` → Mensaje de alerta.
- `role="progressbar"` → Barra de progreso.

───────────────────────────────────────────────────────────
REGLAS DE ORO DE ARIA
───────────────────────────────────────────────────────────

1. No uses ARIA si puedes usar HTML semántico.
   ✅ <button> en lugar de <div role="button">.
   ✅ <nav> en lugar de <div role="navigation">.

2. Si usas ARIA, úsalo correctamente.
   - No uses `role="button"` si no tiene funcionalidad de botón.

3. Siempre prueba con un lector de pantalla.
   - No asumas que tu código funciona.

PARTE 3: ATRIBUTOS ARIA – INFORMACIÓN ADICIONAL
===============================================================================

Los atributos ARIA proporcionan información adicional sobre el estado y las propiedades de los elementos.

───────────────────────────────────────────────────────────
aria-label → ETIQUETA ACCESIBLE
───────────────────────────────────────────────────────────

Proporciona una etiqueta textual para elementos que no tienen texto visible.

<button aria-label="Cerrar ventana">✕</button>

El lector de pantalla lee: "Cerrar ventana" en lugar de "✕".

───────────────────────────────────────────────────────────
aria-labelledby → ETIQUETA REFERENCIADA
───────────────────────────────────────────────────────────

Asocia un elemento con otro que contiene su etiqueta.

<h2 id="titulo-modal">Confirmación</h2>
<div role="dialog" aria-labelledby="titulo-modal">
    <p>¿Estás seguro de que quieres eliminar este elemento?</p>
</div>

───────────────────────────────────────────────────────────
aria-describedby → DESCRIPCIÓN ADICIONAL
───────────────────────────────────────────────────────────

Asocia un elemento con una descripción adicional.

<label for="email">Correo electrónico</label>
<input type="email" id="email" aria-describedby="email-ayuda">
<p id="email-ayuda">Introduce un correo válido, ej: usuario@dominio.com</p>

El lector de pantalla lee el label y luego la descripción.

───────────────────────────────────────────────────────────
aria-hidden → OCULTAR DE LECTORES DE PANTALLA
───────────────────────────────────────────────────────────

Oculta un elemento de los lectores de pantalla.

<div aria-hidden="true">🎨</div>

Útil para elementos decorativos que no aportan información.

───────────────────────────────────────────────────────────
aria-expanded → INDICADOR DE EXPANSIÓN
───────────────────────────────────────────────────────────

Indica si un elemento desplegable está expandido o contraído.

<button aria-expanded="false" aria-controls="menu">
    Menú
</button>
<ul id="menu" hidden>
    <li><a href="#">Opción 1</a></li>
    <li><a href="#">Opción 2</a></li>
</ul>

Valores: `true` (expandido), `false` (contraído).

───────────────────────────────────────────────────────────
aria-controls → CONTROL DE ELEMENTOS
───────────────────────────────────────────────────────────

Asocia un control con el elemento que controla.

<button aria-controls="contenido" aria-expanded="false">
    Mostrar contenido
</button>
<div id="contenido" hidden>
    <p>Contenido oculto que se muestra al hacer clic.</p>
</div>

───────────────────────────────────────────────────────────
aria-live → ACTUALIZACIONES EN TIEMPO REAL
───────────────────────────────────────────────────────────

Anuncia cambios en el contenido en tiempo real.

<div aria-live="polite">
    <p id="mensaje">El formulario se ha enviado correctamente.</p>
</div>

Valores:
- `polite` → Espera a que el usuario termine de leer.
- `assertive` → Interrumpe inmediatamente.

───────────────────────────────────────────────────────────
aria-required → CAMPO OBLIGATORIO (SEMÁNTICO)
───────────────────────────────────────────────────────────

Indica que un campo es obligatorio (además de `required`).

<input type="text" id="nombre" name="nombre" required aria-required="true">

───────────────────────────────────────────────────────────
aria-invalid → CAMPO INVÁLIDO
───────────────────────────────────────────────────────────

Indica que un campo tiene un valor inválido.

<input type="email" id="email" aria-invalid="true">

Valores: `true` (inválido), `false` (válido).

PARTE 4: tabindex – NAVEGACIÓN POR TECLADO
===============================================================================

`tabindex` controla el orden de navegación por teclado (tecla Tab).

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<elemento tabindex="valor">

───────────────────────────────────────────────────────────
VALORES
───────────────────────────────────────────────────────────

- `tabindex="0"` → El elemento es enfocable en el orden natural del DOM.
- `tabindex="-1"` → El elemento NO es enfocable con Tab, pero sí con JavaScript.
- `tabindex="1"`, `tabindex="2"`, etc. → El orden se define numéricamente (NO RECOMENDADO).

───────────────────────────────────────────────────────────
REGLAS DE ORO DE tabindex
───────────────────────────────────────────────────────────

1. NO uses tabindex positivo (1, 2, 3, ...).
   → Rompe el orden natural del DOM.
   → Confunde a los usuarios de teclado.

2. Usa `tabindex="0"` para elementos que no son naturalmente enfocables.
   → Divs, spans, etc. que funcionan como botones.

3. Usa `tabindex="-1"` para elementos que deben ser enfocables solo con JavaScript.

───────────────────────────────────────────────────────────
EJEMPLO DE USO
───────────────────────────────────────────────────────────

<!-- Elemento naturalmente enfocable (ya tiene tabindex implícito) -->
<button>Botón</button>

<!-- Elemento que necesita ser enfocable -->
<div role="button" tabindex="0" onclick="alert('Clic')">
    Botón personalizado
</div>

<!-- Elemento que solo se enfoca con JavaScript -->
<div id="modal" role="dialog" tabindex="-1">
    <p>Contenido del modal</p>
</div>

PARTE 5: ACCESIBILIDAD EN IMÁGENES
===============================================================================

───────────────────────────────────────────────────────────
alt – TEXTO ALTERNATIVO
───────────────────────────────────────────────────────────

El atributo `alt` es OBLIGATORIO en todas las imágenes.

<!-- Imagen informativa -->
<img src="grafico.jpg" alt="Gráfico de barras que muestra un aumento de ventas del 20%">

<!-- Imagen decorativa -->
<img src="adorno.png" alt="">

<!-- Imagen con texto -->
<img src="banner.jpg" alt="Oferta especial: 50% de descuento en todos los cursos">

───────────────────────────────────────────────────────────
REGLAS DE ORO PARA alt
───────────────────────────────────────────────────────────

1. Describe la imagen de forma clara y concisa.
2. Si la imagen contiene texto, repite ese texto en alt.
3. Si la imagen es decorativa, usa alt="".
4. No uses "imagen de" o "foto de" (es redundante).
5. Incluye información relevante para el contexto.

===============================================================================
PARTE 6: ACCESIBILIDAD EN FORMULARIOS
===============================================================================

───────────────────────────────────────────────────────────
LABEL – ETIQUETA DE CAMPOS
───────────────────────────────────────────────────────────

Cada campo de formulario debe tener un `<label>` asociado.

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre" required>

───────────────────────────────────────────────────────────
FIELDSET Y LEGEND – AGRUPACIÓN DE CAMPOS
───────────────────────────────────────────────────────────

Agrupa campos relacionados.

<fieldset>
    <legend>Datos personales</legend>
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    <label for="email">Correo:</label>
    <input type="email" id="email" name="email">
</fieldset>

───────────────────────────────────────────────────────────
ARIA-REQUIRED – CAMPO OBLIGATORIO
───────────────────────────────────────────────────────────

<input type="text" id="nombre" name="nombre" required aria-required="true">

───────────────────────────────────────────────────────────
ARIA-DESCRIBEDBY – DESCRIPCIÓN DEL CAMPO
───────────────────────────────────────────────────────────

<label for="password">Contraseña:</label>
<input type="password" id="password" aria-describedby="password-ayuda">
<p id="password-ayuda">La contraseña debe tener al menos 8 caracteres.</p>

PARTE 7: ACCESIBILIDAD EN ENLACES Y NAVEGACIÓN
===============================================================================

───────────────────────────────────────────────────────────
TEXTO DE ENLACE DESCRIPTIVO
───────────────────────────────────────────────────────────

❌ Malo:
<a href="#">Clic aquí</a>

✅ Bueno:
<a href="cursos.html">Ver todos los cursos disponibles</a>

───────────────────────────────────────────────────────────
ARIA-LABEL EN ENLACES
───────────────────────────────────────────────────────────

<a href="contacto.html" aria-label="Ir a la página de contacto">📧</a>

───────────────────────────────────────────────────────────
ARIA-CURRENT – PÁGINA ACTUAL
───────────────────────────────────────────────────────────

<a href="index.html" aria-current="page">Inicio</a>

Valores: `page`, `step`, `location`, `date`, `time`.

───────────────────────────────────────────────────────────
NAVEGACIÓN POR TECLADO
───────────────────────────────────────────────────────────

Asegúrate de que todos los elementos interactivos sean accesibles con el teclado.

- Los enlaces (`<a>`) ya son accesibles con Tab y Enter.
- Los botones (`<button>`) ya son accesibles con Tab y Enter/Espacio.
- Los campos de formulario (`<input>`, `<textarea>`, `<select>`) ya son accesibles.
- Los elementos personalizados (divs como botones) necesitan `tabindex="0"` y eventos de teclado.

CÓDIGO HTML

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Accesibilidad - Lucas EGA Academy</title>
</head>
<body>

    <!-- ===== HEADER CON ROL BANNER ===== -->
    <header role="banner">
        <h1>Lucas EGA Academy</h1>
        <p>Aprende programación con el Profesor Búho AI</p>

        <!-- ===== NAVEGACIÓN CON ROL NAVIGATION ===== -->
        <nav role="navigation" aria-label="Navegación principal">
            <ul>
                <li><a href="index.html" aria-current="page">Inicio</a></li>
                <li><a href="cursos.html">Cursos</a></li>
                <li><a href="blog.html">Blog</a></li>
                <li><a href="contacto.html">Contacto</a></li>
            </ul>
        </nav>
    </header>

    <!-- ===== CONTENIDO PRINCIPAL CON ROL MAIN ===== -->
    <main role="main">

        <!-- ===== SECCIÓN CON IMAGEN ACCESIBLE ===== -->
        <section>
            <h2>Bienvenido</h2>
            <!-- Imagen con alt descriptivo -->
            <img src="profesor-buho.jpg" alt="Profesor Búho AI, un búho con lentes y una pizarra" width="300" height="300">
            <p>El Profesor Búho AI te guiará en tu aprendizaje de programación.</p>
        </section>

        <!-- ===== SECCIÓN CON MENÚ DESPLEGABLE ACCESIBLE ===== -->
        <section>
            <h2>Preguntas frecuentes</h2>

            <!-- Menú desplegable con ARIA -->
            <button aria-expanded="false" aria-controls="faq1" id="btn-faq1">
                ¿Qué es HTML5?
            </button>
            <div id="faq1" hidden>
                <p>HTML5 es la quinta versión del lenguaje de marcado HTML.</p>
            </div>

            <button aria-expanded="false" aria-controls="faq2" id="btn-faq2">
                ¿Qué es el Profesor Búho AI?
            </button>
            <div id="faq2" hidden>
                <p>El Profesor Búho AI es un mentor virtual de programación.</p>
            </div>
        </section>

        <!-- ===== SECCIÓN CON FORMULARIO ACCESIBLE ===== -->
        <section>
            <h2>Contacto</h2>

            <form action="/enviar" method="POST">
                <fieldset>
                    <legend>Datos personales</legend>

                    <!-- Campo con label y aria-describedby -->
                    <div>
                        <label for="nombre">Nombre completo *</label>
                        <input type="text" id="nombre" name="nombre"
                               required aria-required="true"
                               aria-describedby="nombre-ayuda">
                        <span id="nombre-ayuda">Introduce tu nombre completo.</span>
                    </div>

                    <!-- Campo con label y aria-describedby -->
                    <div>
                        <label for="email">Correo electrónico *</label>
                        <input type="email" id="email" name="email"
                               required aria-required="true"
                               aria-describedby="email-ayuda">
                        <span id="email-ayuda">Introduce un correo válido, ej: usuario@dominio.com</span>
                    </div>
                </fieldset>

                <!-- Botón accesible -->
                <button type="submit" aria-label="Enviar formulario de contacto">
                    Enviar mensaje
                </button>
            </form>
        </section>

        <!-- ===== SECCIÓN CON MENSAJES EN VIVO ===== -->
        <section>
            <h2>Notificaciones</h2>

            <!-- Área de mensajes en vivo -->
            <div aria-live="polite" role="status">
                <p id="mensaje-notificacion">No hay notificaciones.</p>
            </div>

            <button onclick="mostrarNotificacion()">
                Mostrar notificación de prueba
            </button>
        </section>

    </main>

    <!-- ===== BARRA LATERAL CON ROLE COMPLEMENTARY ===== -->
    <aside role="complementary">
        <h3>Artículos relacionados</h3>
        <ul>
            <li><a href="#">Qué es HTML5</a></li>
            <li><a href="#">Cómo funciona CSS</a></li>
            <li><a href="#">Introducción a JavaScript</a></li>
        </ul>
    </aside>

    <!-- ===== FOOTER CON ROLE CONTENTINFO ===== -->
    <footer role="contentinfo">
        <p>&copy; 2026 Lucas EGA Academy. Todos los derechos reservados.</p>
    </footer>

    <script>
        // ============================================================ */
        // MENÚ DESPLEGABLE ACCESIBLE                                    */
        // ============================================================ */
        document.querySelectorAll('[aria-expanded]').forEach(button => {
            button.addEventListener('click', function() {
                const expanded = this.getAttribute('aria-expanded') === 'true' ? false : true;
                const controls = document.getElementById(this.getAttribute('aria-controls'));

                this.setAttribute('aria-expanded', expanded);
                controls.hidden = !expanded;
            });
        });

        // ============================================================ */
        // NOTIFICACIÓN EN VIVO                                           */
        // ============================================================ */
        function mostrarNotificacion() {
            const mensaje = document.getElementById('mensaje-notificacion');
            mensaje.textContent = '¡Nuevo mensaje recibido!';
            // Después de 3 segundos, volver al mensaje original
            setTimeout(() => {
                mensaje.textContent = 'No hay notificaciones.';
            }, 3000);
        }
    </script>

</body>
</html>

    <p>HTML5 es la quinta versión del lenguaje de marcado HTML.</p>

- `aria-expanded` → Indica si el contenido está expandido o contraído.
- `aria-controls` → Asocia el botón con el contenido que controla.

───────────────────────────────────────────────────────────
ARIA-DESCRIBEDBY
───────────────────────────────────────────────────────────

<label for="nombre">Nombre completo *</label>
<input type="text" id="nombre" name="nombre"
       required aria-required="true"
       aria-describedby="nombre-ayuda">
<span id="nombre-ayuda">Introduce tu nombre completo.</span>

El lector de pantalla lee: "Nombre completo, campo obligatorio, Introduce tu nombre completo."

───────────────────────────────────────────────────────────
ARIA-LIVE Y ROLE="STATUS"
───────────────────────────────────────────────────────────

<div aria-live="polite" role="status">
    <p id="mensaje-notificacion">No hay notificaciones.</p>
</div>

- `aria-live="polite"` → Anuncia cambios sin interrumpir.
- `role="status"` → Indica que es un mensaje de estado.

PARTE 12: ERRORES COMUNES EN ACCESIBILIDAD
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR alt EN IMÁGENES
───────────────────────────────────────────────────────────

Incorrecto:
<img src="foto.jpg">

Consecuencia: El lector de pantalla dice "imagen" y el usuario no sabe qué es.

Solución: Siempre añade alt.

───────────────────────────────────────────────────────────
ERROR 2: NO ASOCIAR LABEL CON INPUT
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" id="nombre" placeholder="Nombre">

Consecuencia: El lector de pantalla no sabe qué es el campo.

Solución: Usa `<label for="nombre">Nombre:</label>`.

───────────────────────────────────────────────────────────
ERROR 3: USAR DIVS COMO BOTONES SIN ARIA
───────────────────────────────────────────────────────────

Incorrecto:
<div onclick="alert('Hola')">Clic</div>

Consecuencia: El lector de pantalla no sabe que es un botón.

Solución: Usa `<button>` o añade `role="button"` y `tabindex="0"`.

───────────────────────────────────────────────────────────
ERROR 4: NO USAR ROLES DE LANDMARKS
───────────────────────────────────────────────────────────

Consecuencia: El lector de pantalla no puede saltar entre secciones.

Solución: Usa roles de landmarks o etiquetas semánticas.

───────────────────────────────────────────────────────────
ERROR 5: TEXTO DE ENLACE GENÉRICO
───────────────────────────────────────────────────────────

Incorrecto:
<a href="#">Clic aquí</a>

Consecuencia: El usuario no sabe a dónde va.

Solución: Usa texto descriptivo.

===============================================================================
PARTE 13: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre añade alt a todas las imágenes (con alt="" para decorativas).
2. Siempre asocia label con input usando for/id.
3. Usa etiquetas semánticas (button, a, input, etc.) en lugar de divs para interactividad.
4. Usa roles de landmarks o etiquetas semánticas para estructurar la página.
5. Usa texto descriptivo en los enlaces.
6. Prueba con un lector de pantalla.
7. Usa Lighthouse o WAVE para auditar la accesibilidad.

GLOSARIO DE TÉRMINOS
===============================================================================

- Accesibilidad: Práctica de hacer que los sitios web sean utilizables por todas las personas.
- WCAG: Web Content Accessibility Guidelines (Pautas de Accesibilidad para el Contenido Web).
- ARIA: Accessible Rich Internet Applications.
- Rol: Define el propósito de un elemento (banner, navigation, main, etc.).
- aria-label: Etiqueta accesible para elementos sin texto visible.
- aria-hidden: Oculta un elemento de los lectores de pantalla.
- aria-expanded: Indica si un elemento desplegable está expandido.
- aria-controls: Asocia un control con el elemento que controla.
- aria-live: Anuncia cambios en tiempo real.
- tabindex: Controla el orden de navegación por teclado.
- Lector de pantalla: Software que lee el contenido de la pantalla en voz alta.
