PARTE 1: ¿QUÉ SON LOS MICRODATOS Y SCHEMA.ORG?
===============================================================================

Los microdatos son un conjunto de atributos HTML que permiten estructurar la información para que los motores de búsqueda la entiendan mejor.

───────────────────────────────────────────────────────────
SCHEMA.ORG
───────────────────────────────────────────────────────────

Schema.org es un vocabulario común creado por Google, Bing, Yahoo y Yandex en 2011.

Proporciona un conjunto de tipos (person, product, article, event, etc.) y propiedades (name, description, price, etc.) que puedes usar para marcar tu contenido.

───────────────────────────────────────────────────────────
ATRIBUTOS DE MICRODATOS
───────────────────────────────────────────────────────────

1. `itemscope` → Indica que un elemento contiene microdatos.
2. `itemtype` → Define el tipo de microdatos (ej: "https://schema.org/Article").
3. `itemprop` → Define una propiedad del tipo (ej: "name", "description", "author").

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<div itemscope itemtype="https://schema.org/Person">
    <span itemprop="name">Ana Pérez</span>
    <span itemprop="jobTitle">Desarrolladora web</span>
    <span itemprop="email">ana@email.com</span>
</div>

Google entiende: "Hay una persona llamada Ana Pérez, que es desarrolladora web y su correo es ana@email.com."

===============================================================================
PARTE 2: TIPOS PRINCIPALES DE SCHEMA.ORG
===============================================================================

───────────────────────────────────────────────────────────
1. ARTICLE – ARTÍCULO
───────────────────────────────────────────────────────────

Para artículos de blog, noticias, publicaciones.

Propiedades clave:
- `headline` → Título del artículo.
- `description` → Resumen.
- `author` → Autor (Person u Organization).
- `datePublished` → Fecha de publicación.
- `dateModified` → Fecha de modificación.
- `image` → Imagen principal.
- `publisher` → Publicador (Organization).

───────────────────────────────────────────────────────────
2. PRODUCT – PRODUCTO
───────────────────────────────────────────────────────────

Para productos en tiendas en línea.

Propiedades clave:
- `name` → Nombre del producto.
- `description` → Descripción.
- `image` → Imagen del producto.
- `price` → Precio.
- `priceCurrency` → Moneda (USD, EUR, MXN, etc.).
- `availability` → Disponibilidad (InStock, OutOfStock).
- `brand` → Marca (Brand).
- `review` → Reseñas (Review).

───────────────────────────────────────────────────────────
3. PERSON – PERSONA
───────────────────────────────────────────────────────────

Para personas (autores, perfiles, etc.).

Propiedades clave:
- `name` → Nombre.
- `jobTitle` → Título profesional.
- `email` → Correo.
- `image` → Foto.
- `url` → Sitio web.
- `affiliation` → Organización a la que pertenece.

───────────────────────────────────────────────────────────
4. ORGANIZATION – ORGANIZACIÓN
───────────────────────────────────────────────────────────

Para empresas, organizaciones, academias.

Propiedades clave:
- `name` → Nombre de la organización.
- `description` → Descripción.
- `url` → Sitio web.
- `logo` → Logo.
- `email` → Correo de contacto.
- `address` → Dirección (PostalAddress).

───────────────────────────────────────────────────────────
5. EVENT – EVENTO
───────────────────────────────────────────────────────────

Para eventos (conciertos, conferencias, clases, etc.).

Propiedades clave:
- `name` → Nombre del evento.
- `description` → Descripción.
- `startDate` → Fecha de inicio.
- `endDate` → Fecha de fin.
- `location` → Ubicación (Place).
- `image` → Imagen del evento.

───────────────────────────────────────────────────────────
6. RECIPE – RECETA
───────────────────────────────────────────────────────────

Para recetas de cocina.

Propiedades clave:
- `name` → Nombre de la receta.
- `description` → Descripción.
- `image` → Imagen.
- `recipeIngredient` → Ingredientes.
- `recipeInstructions` → Instrucciones.
- `prepTime` → Tiempo de preparación.
- `cookTime` → Tiempo de cocción.
- `nutrition` → Información nutricional.

───────────────────────────────────────────────────────────
7. REVIEW – RESEÑA
───────────────────────────────────────────────────────────

Para reseñas de productos o servicios.

Propiedades clave:
- `itemReviewed` → Elemento reseñado (Product, Service, etc.).
- `author` → Autor de la reseña.
- `reviewRating` → Calificación (Rating).
- `reviewBody` → Texto de la reseña.

───────────────────────────────────────────────────────────
8. RATING – CALIFICACIÓN
───────────────────────────────────────────────────────────

Para calificaciones (estrellas).

Propiedades clave:
- `ratingValue` → Valor (ej: 4.5).
- `bestRating` → Mejor valor posible (ej: 5).
- `worstRating` → Peor valor posible (ej: 1).

PARTE 3: SINTAXIS DE MICRODATOS
===============================================================================

───────────────────────────────────────────────────────────
itemscope – CONTENEDOR DE MICRODATOS
───────────────────────────────────────────────────────────

<div itemscope>
    <!-- Contenido con microdatos -->
</div>

───────────────────────────────────────────────────────────
itemtype – TIPO DE MICRODATOS
───────────────────────────────────────────────────────────

<div itemscope itemtype="https://schema.org/Article">
    <!-- Contenido de un artículo -->
</div>

───────────────────────────────────────────────────────────
itemprop – PROPIEDADES
───────────────────────────────────────────────────────────

<div itemscope itemtype="https://schema.org/Article">
    <h1 itemprop="headline">Cómo aprender HTML en 10 días</h1>
    <p itemprop="description">Una guía práctica para dominar HTML rápidamente.</p>
    <p itemprop="author">Esteban</p>
    <time itemprop="datePublished" datetime="2026-06-25">25 de junio de 2026</time>
</div>

───────────────────────────────────────────────────────────
ANIDACIÓN DE MICRODATOS
───────────────────────────────────────────────────────────

Los microdatos pueden anidarse para crear estructuras más complejas.

<div itemscope itemtype="https://schema.org/Article">
    <h1 itemprop="headline">Título del artículo</h1>

    <!-- Autor anidado (Person) -->
    <div itemprop="author" itemscope itemtype="https://schema.org/Person">
        <span itemprop="name">Ana Pérez</span>
        <span itemprop="jobTitle">Desarrolladora web</span>
    </div>

    <!-- Publicador anidado (Organization) -->
    <div itemprop="publisher" itemscope itemtype="https://schema.org/Organization">
        <span itemprop="name">Lucas EGA Academy</span>
        <img itemprop="logo" src="logo.png" alt="Logo">
    </div>
</div>

PARTE 5: MICRODATOS + OPEN GRAPH + TWITTER CARDS (EL TRÍO PERFECTO)
===============================================================================

Para maximizar la visibilidad, combina los tres.

───────────────────────────────────────────────────────────
MICRODATOS (SCHEMA.ORG)
───────────────────────────────────────────────────────────

Para motores de búsqueda (Google, Bing, Yahoo).
→ Rich snippets en los resultados de búsqueda.

───────────────────────────────────────────────────────────
OPEN GRAPH (og:)
───────────────────────────────────────────────────────────

Para redes sociales (Facebook, LinkedIn, WhatsApp).
→ Vista previa al compartir.

───────────────────────────────────────────────────────────
TWITTER CARDS
───────────────────────────────────────────────────────────

Para Twitter.
→ Vista previa en Twitter.

───────────────────────────────────────────────────────────
EJEMPLO INTEGRADO
───────────────────────────────────────────────────────────

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- SEO BÁSICO -->
    <title>Curso de HTML5 - Aprende desde cero</title>
    <meta name="description" content="Aprende HTML5 desde cero con el Profesor Búho AI. Curso práctico de 8 semanas con certificado.">

    <!-- OPEN GRAPH -->
    <meta property="og:title" content="Curso de HTML5 - Aprende desde cero">

PARTE 8: ERRORES COMUNES EN MICRODATOS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR ITEMSCOPE O ITEMTYPE
───────────────────────────────────────────────────────────

Incorrecto:
<div itemtype="https://schema.org/Article">
    <span itemprop="headline">Título</span>
</div>

Solución: Añade `itemscope`.

───────────────────────────────────────────────────────────
ERROR 2: ITEMTYPE INCORRECTO
───────────────────────────────────────────────────────────

Incorrecto:
<div itemscope itemtype="https://schema.org/Product">
    <span itemprop="headline">Título</span>  <!-- headline no es propiedad de Product -->
</div>

Solución: Usa propiedades correctas para el tipo.

───────────────────────────────────────────────────────────
ERROR 3: ANIDACIÓN INCORRECTA
───────────────────────────────────────────────────────────

Incorrecto:
<div itemscope itemtype="https://schema.org/Article">
    <div itemprop="author" itemscope itemtype="https://schema.org/Person">
        <span itemprop="name">Ana</span>
    </div>
    <span itemprop="author">Carlos</span>  <!-- Error: duplicado o fuera de lugar -->
</div>

Solución: Mantén la anidación clara.

───────────────────────────────────────────────────────────
ERROR 4: FALTAN PROPIEDADES OBLIGATORIAS
───────────────────────────────────────────────────────────

Algunos tipos requieren propiedades mínimas.

Ejemplo: Article necesita `headline` y `author`.

Solución: Revisa la documentación de Schema.org.

───────────────────────────────────────────────────────────
ERROR 5: CONFLICTOS CON OTROS MICRODATOS
───────────────────────────────────────────────────────────

No uses múltiples sistemas de marcado de datos (microdatos + JSON-LD) de forma inconsistente.

Solución: Elige un sistema y sé consistente. JSON-LD es más fácil y moderno (aunque en este capítulo usamos microdatos).

===============================================================================
PARTE 9: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa `itemscope` e `itemtype` juntos.
2. Consulta la documentación de Schema.org para propiedades correctas.
3. Mantén la anidación clara y lógica.
4. Asegúrate de incluir las propiedades obligatorias.
5. Sé consistente en el sistema de marcado que usas.
6. Usa los validadores de Google y Schema.org.
7. Prueba los rich snippets con la herramienta de Google.

extre in formacion
Los microdatos son un conjunto de etiquetas HTML que estructuran la información de una página web para que los motores de búsqueda comprendan su significado semántico preciso. En el SEO avanzado, el marcado de datos estructurados no se limita a conseguir fragmentos enriquecidos (rich snippets); su verdadera función es alimentar el Grafo de Conocimiento de Google y establecer conexiones explícitas entre entidades.A continuación, se detalla una guía técnica de nivel intermedio-avanzado (Nivel 4/10) estructurada para dominar su implementación y optimización.1. Vocabularios y Sintaxis: Schema.org y JSON-LDPara implementar datos estructurados con éxito, debes diferenciar el vocabulario del formato técnico en el que se codifica.Schema.org: Es el vocabulario estándar global unificado por Google, Bing, Yahoo y Yandex. Define las propiedades y tipos de datos (como Product, Article u Organization).JSON-LD (JavaScript Object Notation for Linked Data): Es la sintaxis recomendada por Google por encima de los microdatos tradicionales incrustados en etiquetas HTML (itemscope, itemtype). Se implementa dentro de un bloque <script> independiente en el código, facilitando el mantenimiento.2.
