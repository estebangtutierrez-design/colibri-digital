PARTE 1: <datalist> – AUTOCOMPLETADO INTELIGENTE
===============================================================================

`<datalist>` proporciona una lista de opciones sugeridas para un campo de entrada.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<input type="text" name="pais" list="paises">
<datalist id="paises">
    <option value="México">
    <option value="España">
    <option value="Argentina">
    <option value="Colombia">
    <option value="Chile">
</datalist>

El atributo `list` del input debe coincidir con el `id` del datalist.

───────────────────────────────────────────────────────────
CARACTERÍSTICAS
───────────────────────────────────────────────────────────

1. El usuario puede escribir libremente o seleccionar una opción.
2. Las sugerencias aparecen mientras el usuario escribe.
3. Es nativo de HTML5 (no necesita JavaScript).
4. Funciona con `type="text"`, `type="search"`, `type="url"`, `type="email"`, etc.

───────────────────────────────────────────────────────────
EJEMPLOS PRÁCTICOS
───────────────────────────────────────────────────────────

<!-- Países -->
<label for="pais">País:</label>
<input type="text" id="pais" name="pais" list="paises">
<datalist id="paises">
    <option value="México">
    <option value="Estados Unidos">
    <option value="Canadá">
    <option value="España">
    <option value="Argentina">
    <option value="Chile">
    <option value="Colombia">
    <option value="Perú">
    <option value="Uruguay">
</datalist>

<!-- Lenguajes de programación -->
<label for="lenguaje">Lenguaje favorito:</label>
<input type="text" id="lenguaje" name="lenguaje" list="lenguajes">
<datalist id="lenguajes">
    <option value="HTML">
    <option value="CSS">
    <option value="JavaScript">
    <option value="Python">
    <option value="Java">
    <option value="C#">
    <option value="C++">
    <option value="PHP">
    <option value="Ruby">
    <option value="Go">
</datalist>

<!-- Ciudades (con búsqueda) -->
<label for="ciudad">Ciudad:</label>
<input type="search" id="ciudad" name="ciudad" list="ciudades" placeholder="Busca tu ciudad...">
<datalist id="ciudades">
    <option value="Ciudad de México">
    <option value="Guadalajara">
    <option value="Monterrey">
    <option value="Madrid">
    <option value="Barcelona">
    <option value="Buenos Aires">
    <option value="Santiago">
    <option value="Bogotá">
</datalist>

───────────────────────────────────────────────────────────
datalist VS. select
───────────────────────────────────────────────────────────

- `<select>` → El usuario solo puede elegir entre las opciones disponibles.
- `<datalist>` → El usuario puede elegir una opción o escribir libremente.

Usa `<select>` cuando las opciones son fijas y limitadas.
Usa `<datalist>` cuando quieras sugerencias pero también libertad.

PARTE 2: <output> – MOSTRAR RESULTADOS DE CÁLCULOS
===============================================================================

`<output>` muestra el resultado de un cálculo o interacción del usuario.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<output id="resultado">0</output>

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `for` → Asocia el output con los campos que lo afectan (separados por espacio).
- `name` → Nombre para enviar al servidor (opcional).

───────────────────────────────────────────────────────────
EJEMPLO: CALCULADORA DE SUMA
───────────────────────────────────────────────────────────

<form oninput="suma.value = parseInt(a.value) + parseInt(b.value)">
    <input type="number" id="a" name="a" value="5">
    +
    <input type="number" id="b" name="b" value="3">
    =
    <output id="suma" name="suma" for="a b">8</output>
</form>

───────────────────────────────────────────────────────────
EJEMPLO: RANGO CON OUTPUT
───────────────────────────────────────────────────────────

<form>
    <label for="volumen">Volumen: <output id="valor-volumen">50</output></label>
    <input type="range" id="volumen" name="volumen"
           min="0" max="100" step="1" value="50"
           oninput="document.getElementById('valor-volumen').value = this.value">
</form>

───────────────────────────────────────────────────────────
EJEMPLO: CALCULADORA DE PRECIO CON DESCUENTO
───────────────────────────────────────────────────────────

<form oninput="total.value = (precio.value * cantidad.value) * (1 - descuento.value / 100)">
    <label>Precio unitario: <input type="number" id="precio" value="100"></label>
    <label>Cantidad: <input type="number" id="cantidad" value="1"></label>
    <label>Descuento (%): <input type="number" id="descuento" value="10" min="0" max="100"></label>
    <p>Total: <output id="total" for="precio cantidad descuento">90</output></p>
</form>

───────────────────────────────────────────────────────────
EJEMPLO: CONVERSOR DE TEMPERATURA
───────────────────────────────────────────────────────────

<form>
    <label>Celsius: <input type="number" id="celsius" value="0"
           oninput="fahrenheit.value = celsius.value * 9/5 + 32"></label>
    <label>Fahrenheit: <output id="fahrenheit" for="celsius">32</output></label>
</form>

PARTE 3: <progress> – BARRA DE PROGRESO
===============================================================================

`<progress>` muestra el progreso de una tarea.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<progress value="75" max="100">75%</progress>

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `value` → Valor actual (progreso alcanzado).
- `max` → Valor máximo (100 por defecto).

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<!-- Progreso simple -->
<progress value="75" max="100">75%</progress>

<!-- Progreso sin valor (animación indeterminada) -->
<progress></progress>

───────────────────────────────────────────────────────────
EJEMPLO CON ETIQUETA
───────────────────────────────────────────────────────────

<label for="progreso-curso">Progreso del curso:</label>
<progress id="progreso-curso" value="60" max="100">60%</progress>
<span>60%</span>

───────────────────────────────────────────────────────────
EJEMPLO CON JAVASCRIPT (SIMULACIÓN)
───────────────────────────────────────────────────────────

<label for="carga">Cargando archivo:</label>
<progress id="carga" value="0" max="100">0%</progress>
<button onclick="simularCarga()">Iniciar carga</button>

<script>
    function simularCarga() {
        const progress = document.getElementById('carga');
        let valor = 0;
        const intervalo = setInterval(() => {
            valor += 5;
            if (valor > 100) {
                clearInterval(intervalo);
                valor = 100;
            }
            progress.value = valor;
            progress.textContent = valor + '%';
        }, 200);
    }
</script>

PARTE 4: <meter> – MEDIDOR DE VALORES
===============================================================================

`<meter>` representa un valor dentro de un rango conocido.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<meter value="60" min="0" max="100">60%</meter>

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `value` → Valor actual.
- `min` → Valor mínimo (0 por defecto).
- `max` → Valor máximo (1 por defecto).
- `low` → Valor considerado bajo.
- `high` → Valor considerado alto.
- `optimum` → Valor óptimo.

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<!-- Nivel de batería -->
<label>Batería:</label>
<meter value="75" min="0" max="100">75%</meter>

<!-- Nivel de conocimiento -->
<label>Nivel de HTML:</label>
<meter value="0.8" min="0" max="1" low="0.3" high="0.8">80%</meter>

<!-- Uso de disco -->
<label>Uso de disco:</label>
<meter value="70" min="0" max="100" low="50" high="90">70%</meter>

───────────────────────────────────────────────────────────
EJEMPLO CON COLORES AUTOMÁTICOS
───────────────────────────────────────────────────────────

<!-- Verde (bueno) -->
<meter value="90" min="0" max="100" low="40" high="80" optimum="90">90%</meter>

<!-- Amarillo (medio) -->
<meter value="60" min="0" max="100" low="40" high="80" optimum="90">60%</meter>

<!-- Rojo (bajo) -->
<meter value="20" min="0" max="100" low="40" high="80" optimum="90">20%</meter>

Los navegadores cambian el color automáticamente:
- Verde si el valor está en el rango óptimo.
- Amarillo si está en el rango medio.
- Rojo si está bajo.

───────────────────────────────────────────────────────────
EJEMPLO: INDICADOR DE SATISFACCIÓN
───────────────────────────────────────────────────────────

<label for="satisfaccion">Satisfacción del usuario:</label>
<meter id="satisfaccion" value="0.85" min="0" max="1"
       low="0.4" high="0.8" optimum="0.9">
    85%
</meter>
<span>85%</span>

───────────────────────────────────────────────────────────
progress VS. meter
───────────────────────────────────────────────────────────

- `<progress>` → Progreso de una tarea (puede cambiar con el tiempo).
- `<meter>` → Valor estático dentro de un rango (no cambia con el tiempo).

Usa `<progress>` para tareas en curso (carga, descarga, instalación).
Usa `<meter>` para indicadores fijos (nivel de batería, uso de disco, calificación).

PARTE 5: <details> Y <summary> – CONTENIDO DESPLEGABLE
===============================================================================

`<details>` crea un elemento que el usuario puede expandir o contraer.
`<summary>` proporciona el título visible.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<details>
    <summary>Título visible</summary>
    <p>Contenido oculto que se muestra al hacer clic.</p>
</details>

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `open` → El contenido está expandido por defecto.

───────────────────────────────────────────────────────────
EJEMPLO BÁSICO
───────────────────────────────────────────────────────────

<details>
    <summary>Más información sobre HTML5</summary>
    <p>HTML5 es la quinta versión del lenguaje de marcado HTML.</p>
    <p>Fue lanzado en 2014 y revolucionó la web.</p>
</details>

───────────────────────────────────────────────────────────
EJEMPLO CON OPEN
───────────────────────────────────────────────────────────

<details open>
    <summary>Sección visible por defecto</summary>
    <p>Este contenido ya está visible al cargar la página.</p>
</details>

        // Simulación de carga
        function simularCarga() {
            const progress = document.getElementById('carga');
            if (!progress) return;

            if (progress.value === 100) {
                progress.value = 0;
                progress.textContent = '0%';
            }

            let valor = progress.value || 0;
            const intervalo = setInterval(() => {
                valor += Math.floor(Math.random() * 10) + 5;
                if (valor > 100) {
                    clearInterval(intervalo);
                    valor = 100;
                }
                progress.value = valor;
                progress.textContent = valor + '%';
                if (valor === 100) {
                    clearInterval(intervalo);
                }
            }, 300);
        }

PARTE 8: EXPLICACIÓN LÍNEA POR LÍNEA

───────────────────────────────────────────────────────────
SECCIÓN 1: DATALIST
───────────────────────────────────────────────────────────

<input type="text" id="ciudad" name="ciudad" list="ciudades">
<datalist id="ciudades">
    <option value="Ciudad de México">
    <option value="Guadalajara">
    ...
</datalist>

- El atributo `list` del input coincide con el `id` del datalist.
- El usuario ve sugerencias mientras escribe.
- Puede seleccionar una o escribir libremente.

───────────────────────────────────────────────────────────
SECCIÓN 2: OUTPUT
───────────────────────────────────────────────────────────

<form oninput="total.value = (precio.value * cantidad.value) - descuento.value">
    <input type="number" id="precio" value="100">
    <input type="number" id="cantidad" value="1">
    <input type="number" id="descuento" value="0">
    <output id="total" for="precio cantidad descuento">100</output>
</form>

- `oninput` recalcula cada vez que un campo cambia.
- `output` muestra el resultado.
- `for` asocia el output con los campos que lo afectan.

───────────────────────────────────────────────────────────
SECCIÓN 3: PROGRESS Y METER
───────────────────────────────────────────────────────────

<progress value="65" max="100">65%</progress>
<meter value="0.8" min="0" max="1" low="0.3" high="0.8">80%</meter>

- `progress` muestra progreso (puede cambiar).
- `meter` muestra un valor en un rango (estático).
- Los colores de `meter` cambian automáticamente según el valor y los atributos low/high/optimum.

───────────────────────────────────────────────────────────
SECCIÓN 4: DETAILS Y SUMMARY
───────────────────────────────────────────────────────────

<details>
    <summary>¿Qué es HTML5?</summary>
    <p>HTML5 es la quinta versión...</p>
</details>

- `details` crea el contenedor desplegable.
- `summary` es el título visible.
- El contenido se oculta/muestra al hacer clic en `summary`.

PARTE 9: ERRORES COMUNES CON CONTROLES AVANZADOS
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR EL ATRIBUTO list EN EL INPUT
───────────────────────────────────────────────────────────

Incorrecto:
<input type="text" name="pais">
<datalist id="paises">...</datalist>

Consecuencia: No aparecen sugerencias.

Solución: Añade `list="paises"` al input.

───────────────────────────────────────────────────────────
ERROR 2: OLVIDAR EL ATRIBUTO for EN OUTPUT
───────────────────────────────────────────────────────────

Incorrecto:
<output id="resultado">0</output>

Consecuencia: No hay relación semántica entre el output y los inputs.

Solución: Añade `for="campo1 campo2"`.

───────────────────────────────────────────────────────────
ERROR 3: PROGRESS SIN VALOR (USARLO COMO METER)
───────────────────────────────────────────────────────────

Incorrecto:
<progress value="60" min="0" max="100">60%</progress>

Consecuencia: No es el uso correcto (progress no usa min, usa max).

Solución: `min` no es necesario en progress. Usa `max="100"`.

───────────────────────────────────────────────────────────
ERROR 4: METER SIN MIN Y MAX
───────────────────────────────────────────────────────────

Incorrecto:
<meter value="60">60%</meter>

Consecuencia: El rango por defecto es 0-1, así que 60 se interpreta como 60 (no 60%).

Solución: Usa `min="0" max="100"` para porcentajes.

───────────────────────────────────────────────────────────
ERROR 5: DETAILS SIN SUMMARY
───────────────────────────────────────────────────────────

Incorrecto:
<details>
    <p>Contenido</p>
</details>

Consecuencia: No hay título visible, el usuario no sabe qué contiene.

Solución: Siempre usa `<summary>` dentro de `<details>`.

===============================================================================
PARTE 10: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa `list` en el input que coincide con el `id` del datalist.
2. Usa `for` en el output para asociarlo con los campos que lo afectan.
3. En progress, no uses `min`; solo `value` y `max`.
4. En meter, usa `min`, `max`, `low`, `high` y `optimum` para un medidor completo.
5. Siempre usa `<summary>` dentro de `<details>`.
6. Prueba los controles en diferentes navegadores.
7. Usa las herramientas de desarrollo (F12) para depurar.

GLOSARIO DE TÉRMINOS
===============================================================================

- datalist: Lista de opciones sugeridas para un input.
- output: Muestra resultados de cálculos o interacciones.
- progress: Barra de progreso para tareas en curso.
- meter: Medidor de valores en un rango conocido.
- details: Contenedor de contenido desplegable.
- summary: Título visible de un elemento details.
- value: Valor actual (en progress, meter, output).
- max: Valor máximo (en progress, meter).
- min: Valor mínimo (en meter).
- low: Valor considerado bajo (en meter).
- high: Valor considerado alto (en meter).
- optimum: Valor óptimo (en meter).

¿SABÍAS QUE...?
===============================================================================

Los controles `<progress>` y `<meter>` son visualmente diferentes en cada navegador.
Chrome, Firefox, Safari y Edge los muestran con estilos ligeramente distintos.

Puedes personalizar su apariencia con CSS usando pseudo-clases como `::-webkit-progress-bar` y `::-webkit-progress-value`.
