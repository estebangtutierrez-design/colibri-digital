PARTE 1: ¿QUÉ ES SVG?
===============================================================================

SVG significa Scalable Vector Graphics (Gráficos Vectoriales Escalables).

───────────────────────────────────────────────────────────
CARACTERÍSTICAS PRINCIPALES
───────────────────────────────────────────────────────────

1. ESCALABLE: Se ve perfecto en cualquier tamaño.
2. VECTORIAL: Usa fórmulas matemáticas (no píxeles).
3. BASADO EN XML: Puedes escribirlo y editarlo con código.
4. INTERACTIVO: Puedes animarlo con CSS y JavaScript.
5. LIGERO: Los archivos son muy pequeños comparados con imágenes rasterizadas.
6. ACCESIBLE: Puedes añadir texto y etiquetas semánticas.

───────────────────────────────────────────────────────────
SVG VS. RASTER (JPEG, PNG, GIF)
───────────────────────────────────────────────────────────

| Característica | SVG (Vectorial) | JPEG/PNG (Raster) |
|----------------|-----------------|-------------------|
| Escalabilidad  | ✅ Siempre perfecto | ❌ Se pixela |
| Tamaño de archivo | Pequeño | Grande (especialmente PNG) |
| Editable con código | ✅ Sí | ❌ No |
| Animación | ✅ Sí (CSS/JS) | ❌ Limitada |
| Transparencia | ✅ Sí | PNG sí, JPEG no |
| Ideal para | Logos, iconos, ilustraciones | Fotos, imágenes complejas |

===============================================================================
PARTE 2: LA ETIQUETA `<svg>` – EL CONTENEDOR
===============================================================================

La etiqueta `<svg>` es el contenedor de todos los gráficos vectoriales.

───────────────────────────────────────────────────────────
SINTAXIS BÁSICA
───────────────────────────────────────────────────────────

<svg width="200" height="200" xmlns="http://www.w3.org/2000/svg">
    <!-- Elementos gráficos aquí -->
</svg>

───────────────────────────────────────────────────────────
ATRIBUTOS PRINCIPALES
───────────────────────────────────────────────────────────

- `width` → Ancho del lienzo en píxeles.
- `height` → Alto del lienzo en píxeles.
- `viewBox` → Define el sistema de coordenadas (ver Parte 8).
- `xmlns` → Namespace (necesario para documentos SVG independientes, no para HTML5).

───────────────────────────────────────────────────────────
EJEMPLO DE SVG EN HTML
───────────────────────────────────────────────────────────

<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="red" />
</svg>

Esto dibuja un círculo rojo en un lienzo de 100x100.

===============================================================================
PARTE 3: `<circle>` – CÍRCULOS
===============================================================================

`<circle>` dibuja un círculo.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<circle cx="centro_x" cy="centro_y" r="radio" fill="color" stroke="color" stroke-width="ancho" />

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `cx` → Coordenada X del centro.
- `cy` → Coordenada Y del centro.
- `r` → Radio del círculo.
- `fill` → Color de relleno.
- `stroke` → Color del borde.
- `stroke-width` → Ancho del borde.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Círculo básico -->
<circle cx="50" cy="50" r="40" fill="blue" />

<!-- Círculo con borde -->
<circle cx="50" cy="50" r="40" fill="yellow" stroke="black" stroke-width="3" />

<!-- Círculo sin relleno -->
<circle cx="50" cy="50" r="40" fill="none" stroke="red" stroke-width="2" />

<!-- Círculo rojo pequeño -->
<circle cx="20" cy="20" r="15" fill="red" />

===============================================================================
PARTE 4: `<rect>` – RECTÁNGULOS
===============================================================================

`<rect>` dibuja un rectángulo.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<rect x="pos_x" y="pos_y" width="ancho" height="alto" rx="radio_borde" ry="radio_borde" />

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `x` → Posición X de la esquina superior izquierda.
- `y` → Posición Y de la esquina superior izquierda.
- `width` → Ancho del rectángulo.
- `height` → Alto del rectángulo.
- `rx` → Radio de las esquinas redondeadas (horizontal).
- `ry` → Radio de las esquinas redondeadas (vertical).
- `fill` → Color de relleno.
- `stroke` → Color del borde.
- `stroke-width` → Ancho del borde.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Rectángulo básico -->
<rect x="20" y="20" width="160" height="80" fill="blue" />

<!-- Rectángulo con borde -->
<rect x="20" y="20" width="160" height="80" fill="green" stroke="black" stroke-width="2" />

<!-- Rectángulo con esquinas redondeadas -->
<rect x="20" y="20" width="160" height="80" fill="orange" rx="10" ry="10" />

<!-- Rectángulo sin relleno -->
<rect x="20" y="20" width="160" height="80" fill="none" stroke="red" stroke-width="3" />

===============================================================================
PARTE 5: `<line>` – LÍNEAS
===============================================================================

`<line>` dibuja una línea recta entre dos puntos.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<line x1="inicio_x" y1="inicio_y" x2="fin_x" y2="fin_y" stroke="color" stroke-width="ancho" />

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `x1` → Coordenada X del punto inicial.
- `y1` → Coordenada Y del punto inicial.
- `x2` → Coordenada X del punto final.
- `y2` → Coordenada Y del punto final.
- `stroke` → Color de la línea (obligatorio).
- `stroke-width` → Ancho de la línea.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Línea básica -->
<line x1="20" y1="20" x2="180" y2="180" stroke="red" stroke-width="2" />

<!-- Línea horizontal -->
<line x1="20" y1="50" x2="180" y2="50" stroke="blue" stroke-width="4" />

<!-- Línea vertical -->
<line x1="100" y1="20" x2="100" y2="180" stroke="green" stroke-width="3" />

<!-- Línea discontinua (con stroke-dasharray) -->
<line x1="20" y1="100" x2="180" y2="100" stroke="purple" stroke-width="2" stroke-dasharray="5,5" />

===============================================================================
PARTE 6: `<polygon>` – POLÍGONOS
===============================================================================

`<polygon>` dibuja una forma cerrada con múltiples lados.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<polygon points="x1,y1 x2,y2 x3,y3 ..." fill="color" stroke="color" />

───────────────────────────────────────────────────────────
ATRIBUTOS
───────────────────────────────────────────────────────────

- `points` → Lista de pares de coordenadas (x,y) separados por espacios.
- `fill` → Color de relleno.
- `stroke` → Color del borde.
- `stroke-width` → Ancho del borde.

───────────────────────────────────────────────────────────
EJEMPLOS
───────────────────────────────────────────────────────────

<!-- Triángulo -->
<polygon points="100,20 180,180 20,180" fill="yellow" stroke="black" stroke-width="2" />

<!-- Cuadrado -->
<polygon points="20,20 180,20 180,180 20,180" fill="blue" />

<!-- Pentágono -->
<polygon points="100,20 180,80 150,160 50,160 20,80" fill="green" stroke="white" stroke-width="2" />

<!-- Estrella -->
<polygon points="100,10 120,70 180,70 130,110 150,170 100,140 50,170 70,110 20,70 80,70"
         fill="gold" stroke="orange" stroke-width="2" />

===============================================================================
PARTE 7: `<path>` – TRAZADOS COMPLEJOS (EL MÁS PODEROSO)
===============================================================================

`<path>` dibuja trazados complejos usando comandos de dibujo.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<path d="comandos" fill="color" stroke="color" />

───────────────────────────────────────────────────────────
COMANDOS DE PATH
───────────────────────────────────────────────────────────

- `M x y` → Mover a la posición (x, y) (sin dibujar).
- `L x y` → Dibujar línea hasta (x, y).
- `H x` → Dibujar línea horizontal hasta x.
- `V y` → Dibujar línea vertical hasta y.
- `Z` → Cerrar el trazado (volver al punto inicial).
- `C x1 y1, x2 y2, x y` → Curva de Bézier cúbica.
- `Q x1 y1, x y` → Curva de Bézier cuadrática.
- `A rx ry x-axis-rotation large-arc-flag sweep-flag x y` → Arco.

───────────────────────────────────────────────────────────
EJEMPLOS DE PATH
───────────────────────────────────────────────────────────

<!-- Triángulo con path -->
<path d="M 100 20 L 180 180 L 20 180 Z" fill="red" />

<!-- Corazón -->
<path d="M 100 150 C 100 100, 20 80, 20 120 C 20 160, 100 180, 100 180 C 100 180, 180 160, 180 120 C 180 80, 100 100, 100 150 Z"
      fill="red" stroke="darkred" stroke-width="2" />

<!-- Casa -->
<path d="M 20 120 L 20 180 L 80 180 L 80 140 L 120 140 L 120 180 L 180 180 L 180 120 L 100 60 L 20 120 Z"
      fill="lightblue" stroke="black" stroke-width="2" />

<!-- Arco (ejemplo) -->
<path d="M 20 120 A 80 80 0 0 1 180 120" fill="none" stroke="red" stroke-width="3" />

===============================================================================
PARTE 8: viewBox – EL SISTEMA DE COORDENADAS
===============================================================================

`viewBox` define el sistema de coordenadas del SVG.
Permite que el gráfico se escale automáticamente.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<svg viewBox="minX minY width height" width="100%" height="100%">

───────────────────────────────────────────────────────────
EJEMPLO
───────────────────────────────────────────────────────────

<!-- viewBox 0 0 200 200, mostrado en 400x400 -->
<svg viewBox="0 0 200 200" width="400" height="400">
    <circle cx="100" cy="100" r="80" fill="blue" />
</svg>

- `viewBox="0 0 200 200"` → El área de dibujo es de 200x200.
- `width="400" height="400"` → Se muestra en 400x400.
- El círculo se escala automáticamente.

───────────────────────────────────────────────────────────
VENTAJAS DE viewBox
───────────────────────────────────────────────────────────

1. El gráfico se adapta a cualquier tamaño.
2. Puedes usar `width="100%"` y `height="100%"` para que sea responsive.
3. Las coordenadas siempre se refieren al sistema de viewBox, no al tamaño de visualización.

PARTE 9: COLORES Y ESTILOS EN SVG
===============================================================================

───────────────────────────────────────────────────────────
FILL – RELLENO
───────────────────────────────────────────────────────────

<circle cx="50" cy="50" r="40" fill="red" />
<circle cx="50" cy="50" r="40" fill="#ff0000" />
<circle cx="50" cy="50" r="40" fill="rgb(255,0,0)" />
<circle cx="50" cy="50" r="40" fill="none" />

───────────────────────────────────────────────────────────
STROKE – BORDE
───────────────────────────────────────────────────────────

<circle cx="50" cy="50" r="40" fill="blue" stroke="black" stroke-width="3" />

───────────────────────────────────────────────────────────
OPACIDAD
───────────────────────────────────────────────────────────

<circle cx="50" cy="50" r="40" fill="red" opacity="0.5" />

───────────────────────────────────────────────────────────
stroke-dasharray – BORDE DISCONTINUO
───────────────────────────────────────────────────────────

<line x1="20" y1="100" x2="180" y2="100" stroke="red" stroke-width="2" stroke-dasharray="10,5" />

- `10,5` → 10px de línea, 5px de espacio.

───────────────────────────────────────────────────────────
GRADIENTES Y SOMBRAS (AVANZADO)
───────────────────────────────────────────────────────────

SVG también soporta gradientes (`<linearGradient>`, `<radialGradient>`) y sombras (`<filter>`), pero son temas avanzados que se cubrirán en capítulos posteriores.

PARTE 12: EXPLICACIÓN LÍNEA POR LÍNEA
===============================================================================

───────────────────────────────────────────────────────────
SECCIÓN 1: FIGURAS BÁSICAS
───────────────────────────────────────────────────────────

<svg viewBox="0 0 400 200" width="400" height="200">
    <circle cx="60" cy="100" r="50" fill="#e53e3e" stroke="#333" stroke-width="3" />
</svg>

- `viewBox="0 0 400 200"` → Sistema de coordenadas 400x200.
- `circle cx="60" cy="100" r="50"` → Círculo en (60,100) con radio 50.
- `fill="#e53e3e"` → Color rojo.
- `stroke="#333"` → Borde gris oscuro.
- `stroke-width="3"` → Borde de 3px.

───────────────────────────────────────────────────────────
SECCIÓN 2: POLÍGONOS
───────────────────────────────────────────────────────────

<polygon points="100,20 180,180 20,180" fill="#ecc94b" stroke="#333" stroke-width="3" />

- `points="100,20 180,180 20,180"` → Tres vértices: (100,20), (180,180), (20,180).
- Dibuja un triángulo.

───────────────────────────────────────────────────────────
SECCIÓN 3: PATH – CORAZÓN
───────────────────────────────────────────────────────────

<path d="M 200 180 C 200 130, 100 100, 100 140 C 100 180, 200 210, 200 210 C 200 210, 300 180, 300 140 C 300 100, 200 130, 200 180 Z" ...>

- `M 200 180` → Mover a (200,180) (punto inicial).
- `C` → Curvas de Bézier cúbicas que forman el corazón.
- `Z` → Cerrar el trazado.

───────────────────────────────────────────────────────────
SECCIÓN 4: LOGOTIPO
───────────────────────────────────────────────────────────

<rect x="20" y="20" width="360" height="110" rx="15" fill="#1a202c" />
<circle cx="80" cy="75" r="40" fill="#2b6cb0" />
<text x="80" y="85" text-anchor="middle" font-size="48" font-weight="bold" fill="white">L</text>
<text x="140" y="65" font-size="24" font-weight="bold" fill="#58a6ff">Lucas EGA</text>
<text x="140" y="95" font-size="16" fill="#c9d1d9">Academy</text>

- `rect` → Fondo redondeado.
- `circle` → Círculo azul para la L.
- `text` → Texto con diferentes tamaños y colores.

───────────────────────────────────────────────────────────
SECCIÓN 5: SVG RESPONSIVE
───────────────────────────────────────────────────────────

<svg viewBox="0 0 400 200" width="100%" height="auto">

- `viewBox` define el sistema de coordenadas.
- `width="100%"` → Ocupa todo el ancho del contenedor.
- `height="auto"` → Mantiene la proporción.

===============================================================================
PARTE 13: SVG CON TEXTO (ADELANTO)
===============================================================================

SVG también permite añadir texto con `<text>`.

───────────────────────────────────────────────────────────
SINTAXIS
───────────────────────────────────────────────────────────

<text x="100" y="150" font-family="Arial" font-size="24" fill="black" text-anchor="middle">
    Texto en SVG
</text>

───────────────────────────────────────────────────────────
ATRIBUTOS DE TEXTO
───────────────────────────────────────────────────────────

- `x` → Posición X.
- `y` → Posición Y (base del texto).
- `font-family` → Fuente.
- `font-size` → Tamaño.
- `font-weight` → Grosor (bold, normal).
- `fill` → Color.
- `text-anchor` → Alineación (start, middle, end).

PARTE 14: ERRORES COMUNES CON SVG
===============================================================================

───────────────────────────────────────────────────────────
ERROR 1: OLVIDAR EL viewBox
───────────────────────────────────────────────────────────

Incorrecto:
<svg width="200" height="200">
    <circle cx="100" cy="100" r="80" fill="red" />
</svg>

Consecuencia: El círculo se muestra correctamente, pero el SVG no escala bien.

Solución: Usa viewBox.

───────────────────────────────────────────────────────────
ERROR 2: COORDENADAS FUERA DEL viewBox
───────────────────────────────────────────────────────────

Incorrecto:
<svg viewBox="0 0 200 200">
    <circle cx="300" cy="100" r="50" fill="red" />
</svg>

Consecuencia: El círculo no se ve porque está fuera del área visible.

Solución: Asegúrate de que las coordenadas estén dentro del viewBox.

───────────────────────────────────────────────────────────
ERROR 3: NO CERRAR LAS ETIQUETAS (SELF-CLOSING)
───────────────────────────────────────────────────────────

Incorrecto:
<circle cx="50" cy="50" r="40" fill="red">

Correcto:
<circle cx="50" cy="50" r="40" fill="red" />

Solución: SVG es XML, las etiquetas vacías deben cerrarse con />.

───────────────────────────────────────────────────────────
ERROR 4: NO ESPECIFICAR STROKE EN LÍNEAS
───────────────────────────────────────────────────────────

Incorrecto:
<line x1="20" y1="20" x2="180" y2="180" />

Consecuencia: La línea no se ve (sin stroke).

Solución: Añade stroke="color".

───────────────────────────────────────────────────────────
ERROR 5: USAR `</svg>` SIN ABRIR LA ETIQUETA
───────────────────────────────────────────────────────────

Solución: Siempre abre y cierra correctamente.

===============================================================================
PARTE 15: CÓMO SOLUCIONAR LOS ERRORES
===============================================================================

1. Siempre usa viewBox para escalabilidad.
2. Verifica que las coordenadas estén dentro del viewBox.
3. Cierra las etiquetas vacías con />.
4. Añade stroke a las líneas (es obligatorio).
5. Usa el validador de SVG (https://validator.w3.org/).
6. Prueba el SVG en diferentes tamaños de pantalla.

===============================================================================
GLOSARIO DE TÉRMINOS
===============================================================================

- SVG: Scalable Vector Graphics (Gráficos Vectoriales Escalables).
- Vectorial: Imagen basada en fórmulas matemáticas (no píxeles).
- viewBox: Sistema de coordenadas del SVG.
- fill: Color de relleno.
- stroke: Color del borde.
- stroke-width: Ancho del borde.
- cx: Centro X de un círculo.
- cy: Centro Y de un círculo.
- r: Radio de un círculo.
- rx, ry: Radio de esquinas redondeadas.
- points: Lista de puntos para un polígono.
- path: Trazado complejo con comandos de dibujo.
- M: Mover a (Move).
- L: Línea (Line).
- Z: Cerrar (Close).
