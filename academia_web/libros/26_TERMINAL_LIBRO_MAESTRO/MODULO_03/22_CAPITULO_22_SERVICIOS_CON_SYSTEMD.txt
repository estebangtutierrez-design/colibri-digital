CONTENIDO DEL CAPÍTULO
===============================================================================

1. SYSTEMD

Sistema de init moderno. systemctl manage servicios. journalctl logs.

systemd-analyze analiza tiempo de arranque.



2. SYSTEMCTL

systemctl start|stop|restart|status servicio. systemctl enable|disable

activa/desactiva inicio automático. systemctl is-active verifica estado.

systemctl list-units --type=service lista servicios.



3. UNIT FILES

Archivos .service en /etc/systemd/system/. Sección [Unit], [Service],

[Install]. ExecStart, Restart, User, WorkingDirectory, Environment.



4. JOURNALCTL

journalctl -u servicio logs de servicio. journalctl -f sigue en tiempo real.

journalctl --since '1 hour ago' desde hace 1 hora. -p err solo errores.

EJEMPLO DE CÓDIGO
----------------------------------------

[Unit]
Description=Mi Servicio
After=network.target

[Service]
Type=simple
User=appuser
WorkingDirectory=/opt/app
ExecStart=/usr/bin/node /opt/app/server.js
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

# systemctl daemon-reload
# systemctl enable myservice
# systemctl start myservice
