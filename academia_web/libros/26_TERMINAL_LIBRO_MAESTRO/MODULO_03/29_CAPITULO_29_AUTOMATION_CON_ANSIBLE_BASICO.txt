CONTENIDO DEL CAPÍTULO
===============================================================================

1. ANSIBLE

Herramienta de automatización: configuración, despliegue, orquestación.

Sin agente (usa SSH). Playbooks en YAML. Idempotente.



2. INVENTORY

[grupo]
host1 ansible_host=192.168.1.10 ansible_user=root
host2

[all:vars]
ansible_python_interpreter=/usr/bin/python3



3. PLAYBOOKS

Playbooks YAML con hosts, tasks, modules. Módulos: apt, copy, service,

template, command. Handlers para acciones en cambios (restart service).



4. COMANDOS

ansible all -m ping test conexión. ansible-playbook playbook.yml ejecuta.

ansible all -a 'comando' ad-hoc. --check dry-run. --diff muestra cambios.

EJEMPLO DE CÓDIGO
----------------------------------------

---
- name: Configurar servidor web
  hosts: webservers
  become: yes
  tasks:
    - name: Instalar nginx
      apt:
        name: nginx
        state: present

    - name: Copiar configuracion
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/sites-available/default
      notify: restart nginx

  handlers:
    - name: restart nginx
      service:
        name: nginx
        state: restarted

# ansible-playbook -i inventory playbook.yml
