CONTENIDO DEL CAPÍTULO
===============================================================================

1. SANITIZACIÓN DE INPUT

Validar todo input del usuario. Usar [[ ]] en vez de [ ] para más seguridad.

Escapar variables en comandos: "$var" en vez de $var. Evitar eval.



2. PERMISOS SEGUROS

Scripts ejecutables: chmod 700 o 750. No usar chmod 777. Propietario

adecuado. Scripts en $PATH protegido (no confiar en PATH).



3. EJECUCIÓN SEGURA

set -euo pipefail. Verificar existencia de comandos: command -v cmd.

No usar `...` para comandos (usar $()). Tener cuidado con /tmp.



4. SECRETOS Y CONTRASEÑAS

No hardcodear contraseñas. Usar variables de entorno o archivos .env.

Restringir permisos de archivos con secretos. Usar trap para limpiar.

EJEMPLO DE CÓDIGO
----------------------------------------

#!/bin/bash
set -euo pipefail

# Validacion de comandos
for cmd in curl jq git; do
  if ! command -v $cmd &>/dev/null; then
    echo "Error: $cmd no instalado"
    exit 1
  fi
done

# Lectura segura de contrasena
read -s -p "Contrasena: " PASSWORD
echo ""

# No usar eval nunca
eval "$INPUT"  # PELIGRO

# Archivos temporales seguros
TMPFILE=$(mktemp)
trap "rm -f $TMPFILE" EXIT
