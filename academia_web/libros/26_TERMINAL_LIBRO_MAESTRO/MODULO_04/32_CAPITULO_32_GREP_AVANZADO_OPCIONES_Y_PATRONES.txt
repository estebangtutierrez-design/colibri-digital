CONTENIDO DEL CAPÍTULO
===============================================================================

1. GREP RECURSIVO

grep -rn 'patron' . busca recursivo con números de línea. grep -Rl 'patron'

solo archivos que coinciden. grep --include='*.py' 'import' solo .py.



2. GREP CONTEXTUAL

grep -B 5 'error' 5 líneas antes. grep -A 5 'error' 5 después.

grep -C 3 'error' 3 alrededor. Útil para entender contexto del error.



3. GREP CON REGEX

grep -E 'error|fatal|panic' múltiples patrones. grep -P '\d{3}' perl regex.

grep -v '^#' archivo excluye comentarios. grep '^$' líneas vacías.



4. GREP OUTPUT

grep -c 'patron' cuenta. grep -n 'patron' números de línea. grep -H archivo.

grep -o 'patron' solo texto coincidente. grep --color=always colores.

EJEMPLO DE CÓDIGO
----------------------------------------

# Encontrar TODOs en codigo
grep -rn 'TODO\|FIXME\|HACK' src/ --include='*.{py,js,ts}'

# Analizar logs de errores
grep -E '^(ERROR|FATAL)' app.log | tail -50

# Lineas que no son comentarios
grep -v '^#' /etc/nginx/nginx.conf | grep -v '^$'

# IPs en logs
grep -Eo '([0-9]{1,3}\.){3}[0-9]{1,3}' access.log | sort | uniq -c | sort -rn
