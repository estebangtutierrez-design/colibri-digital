CONTENIDO DEL CAPÍTULO
===============================================================================

1. ESTRATEGIA DE BACKUP

Full semanal, incremental diario, WAL cada hora (PostgreSQL). Retención:

30 días diarios, 6 meses semanales, 1 año mensual.



2. BACKUP BASE DE DATOS

pg_dump -Fc para full. pg_basebackup para streaming. WAL archiving.

Prueba de restauración mensual. Backup cifrado. A múltiples destinos.



3. BACKUP ARCHIVOS

rsync a servidor de backup. rclone a S3/B2/Wasabi. Tar + gzip.

Snapshot LVM antes de cambios grandes. Versionado de archivos.



4. DRP (DISASTER RECOVERY)

RPO (Recovery Point Objetive): 1 hora. RTO (Recovery Time Objetive): 4 horas.

Procedimiento documentado. Prueba de DR trimestral. Infraestructura en

otra región. DNS failover.

EJEMPLO DE CÓDIGO
----------------------------------------

#!/bin/bash
# Backup PostgreSQL
BACKUP_DIR="/backup/postgres/$(date +%Y-%m-%d)"
mkdir -p $BACKUP_DIR
pg_dump -Fc mydb > $BACKUP_DIR/mydb.dump
rclone copy $BACKUP_DIR remote:backups/postgres/

# Restaurar
pg_restore -d mydb mydb.dump

# Backup archivos
rsync -avz /var/www/ backup@storage:/backups/www/
rclone sync /var/www/ s3:my-bucket/www/

# Limpiar backups viejos
find /backup -type d -mtime +30 -exec rm -rf {} +
