CONTENIDO DEL CAPÍTULO
===============================================================================

1. DEPLOYMENT SCRIPT

Script de deploy: git pull, build, migraciones, restart. Rollback en caso

de error. Logging de cada paso. Notificaciones (Slack/email).



2. BACKUP SCRIPT

Backup de BD (pg_dump), archivos (rsync), configuración. Rotación.

Backup a S3/object storage. Prueba de restauración periódica.



3. MONITORING SCRIPT

Health checks (HTTP, puertos, procesos). Alertas por email/Slack.

Chequeo de disco, memoria, CPU. Logs de estado diario.



4. MAINTENANCE SCRIPT

Limpieza de logs. Rotación. Actualización de paquetes. Renovación SSL.

Reboot programado. Reporte semanal de estado.

EJEMPLO DE CÓDIGO
----------------------------------------

#!/bin/bash
set -euo pipefail

deploy() {
  echo "[$(date)] Iniciando deploy..."
  cd /opt/app
  git pull origin main
  docker-compose build --no-cache app
  docker-compose up -d app
  echo "[$(date)] Health check..."
  sleep 5
  curl -f http://localhost:3000/health || {
    echo "[$(date)] Deploy fallo, haciendo rollback..."
    docker-compose stop app
    docker-compose up -d app_previous
    exit 1
  }
  echo "[$(date)] Deploy exitoso"
}

deploy
