CONTENIDO DEL CAPÍTULO
===============================================================================

1. DISEÑO

Requerimientos: número de usuarios, tráfico esperado, disponibilidad.

Presupuesto. Stack tecnológico. Diagrama de red. Plan de crecimiento.



2. COMPONENTES

Servidor web (nginx), aplicación (Node/Python/Java), base de datos

(PostgreSQL), caché (Redis), almacenamiento (archivos/object storage).



3. ALTA DISPONIBILIDAD

Load balancer, múltiples instancias, base de datos replicada, failover.

Health checks, monitorización, alertas. DRP (Disaster Recovery Plan).



4. SEGURIDAD

Firewall, SSL/TLS, autenticación, backups, logs de auditoría.

Actualizaciones automáticas, escaneo de vulnerabilidades.
