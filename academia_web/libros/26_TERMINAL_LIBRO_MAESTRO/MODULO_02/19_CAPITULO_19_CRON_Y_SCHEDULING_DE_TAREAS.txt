CONTENIDO DEL CAPÍTULO
===============================================================================

1. CRON

crontab -e edita tareas. crontab -l lista. crontab -r elimina. Formato:

minuto hora día mes día_semana comando.



2. FORMATO CRON

* * * * * comando cada minuto. 0 * * * * cada hora. 0 0 * * * cada día.

0 0 * * 0 cada semana. */5 * * * * cada 5 minutos. 0 9-17 * * 1-5 laboral.



3. CRON ABREVIATURAS

@reboot al iniciar. @daily (0 0 * * *). @hourly (0 * * * *). @weekly.

@monthly. @yearly. No disponibles en todos los sistemas.



4. BUENAS PRÁCTICAS

Usar rutas absolutas. Redirigir salida: >> /var/log/cron.log 2>&1.

Variables de entorno limitadas en cron. Scripts autocontenidos.

Logging: registrar ejecución con fecha y resultado.

EJEMPLO DE CÓDIGO
----------------------------------------

# crontab -e
# Min Hora Dia Mes DiaSem Comando
*/5 * * * * /home/user/scripts/monitor.sh >> /var/log/monitor.log 2>&1
0 3 * * * /usr/bin/backup.sh
@daily /home/user/scripts/cleanup.sh
0 9-17 * * 1-5 /home/user/scripts/check.sh

# Explicacion:
# */5 = cada 5 minutos
# 0 3 = a las 3 AM
# @daily = una vez al dia
# 0 9-17 * * 1-5 = cada hora en horario laboral
