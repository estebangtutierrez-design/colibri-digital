CONTENIDO DEL CAPÍTULO
===============================================================================

1. IaC CONCEPTOS

Infrastructure as Code: gestionar infraestructura mediante código.

Versionado, reproducible, automatizado. Herramientas: Ansible, Terraform,

Pulumi, CloudFormation.



2. ANSIBLE PLAYBOOKS

YAML declarativo. Idempotente. Sin agente (SSH). Módulos para todo:

paquetes, servicios, archivos, usuarios, redes.



3. TERRAFORM

HashiCorp Terraform. Declarativo con HCL. Plan y apply. State management.

Providers: AWS, Azure, GCP, Proxmox, Docker. Módulos reutilizables.



4. GIT-OPS

Infraestructura en Git. CI/CD para cambios. Revisión de código para

infraestructura. Rollback mediante git revert. Compliance y auditoría.

EJEMPLO DE CÓDIGO
----------------------------------------

# Ansible playbook basico
---
- name: Web server setup
  hosts: all
  become: yes
  tasks:
    - apt: name=nginx state=present
    - service: name=nginx state=started enabled=yes
    - copy: src=index.html dest=/var/www/html/

# Terraform basico
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  tags = { Name = "web-server" }
}
