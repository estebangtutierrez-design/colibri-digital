1. EL PRIMER PROGRAMA
===============================================================================

Crea un archivo llamado `main.go` con el siguiente contenido:

```go
package main

import "fmt"

func main() {
    fmt.Println("¡Hola, mundo!")
}
```

Este programa, aunque diminuto, ya es un programa Go completo y funcional.
Vamos a desglosar cada parte.

1.1 Ejecutar el programa

```bash
go run main.go
```

Salida:
```
¡Hola, mundo!
```

1.2 Compilar el programa

```bash
go build main.go
```

Esto genera un binario llamado `main` (o `main.exe` en Windows).
Puedes ejecutarlo directamente:

```bash
./main
```

Salida:
```
¡Hola, mundo!
```

===============================================================================
2. ANATOMÍA DEL PROGRAMA
===============================================================================

Analicemos línea por línea:

```go
package main
```

Todo archivo Go comienza con una declaración de paquete. El paquete
`main` es especial: indica que este es un programa ejecutable, no
una biblioteca. Cuando compilas un paquete `main`, produces un binario.

```go
import "fmt"
```

La declaración `import` trae funcionalidad de otros paquetes. `fmt`
es un paquete de la biblioteca estándar que proporciona funciones
para formato de entrada y salida.

Podemos importar múltiples paquetes:

```go
import (
    "fmt"
    "math"
    "os"
)
```

```go
func main() {
```

`func main()` es el punto de entrada del programa. Cuando ejecutas
el binario, Go busca y ejecuta la función `main` del paquete `main`.

```go
    fmt.Println("¡Hola, mundo!")
```

`fmt.Println` imprime una línea en la salida estándar. El nombre
completo es `fmt.Println`: el paquete `fmt` seguido de la función
`Println`.

```go
}
```

La llave de cierre. En Go, las llaves van en la misma línea que
la declaración (obligatorio, el formateador lo fuerza).

===============================================================================
3. VARIACIONES DEL HOLA MUNDO
===============================================================================

3.1 Con formato

```go
package main

import "fmt"

func main() {
    nombre := "Mundo"
    fmt.Printf("¡Hola, %s!\n", nombre)
    fmt.Printf("Hoy es un gran día para programar en Go.\n")
}
```

3.2 Saludando al usuario

```go
package main

import (
    "bufio"
    "fmt"
    "os"
    "strings"
)

func main() {
    fmt.Print("¿Cómo te llamas? ")
    lector := bufio.NewReader(os.Stdin)
    nombre, _ := lector.ReadString('\n')
    nombre = strings.TrimSpace(nombre)
    fmt.Printf("¡Hola, %s! Bienvenido a Go.\n", nombre)
}
```

3.3 Con emojis y unicode

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, 世界!")       // Chino
    fmt.Println("Привет, мир!")       // Ruso
    fmt.Println("مرحبا بالعالم!")     // Árabe
    fmt.Println("¡Hola, 🌍!")         // Con emoji
}
```

Go soporta UTF-8 nativamente, por lo que puedes usar cualquier
carácter Unicode sin configuración especial.

3.4 Múltiples mensajes

```go
package main

import "fmt"

func main() {
    fmt.Println("Go es:")
    fmt.Println("  ✅ Rápido")
    fmt.Println("  ✅ Simple")
    fmt.Println("  ✅ Concurrente")
    fmt.Println("  ✅ Productivo")
}
```

3.5 Con variables

```go
package main

import "fmt"

func main() {
    lenguaje := "Go"
    año := 2007
    creador := "Google"
    fmt.Printf("%s fue creado en %d por %s.\n", lenguaje, año, creador)
}
```

===============================================================================
4. GO RUN VS GO BUILD
===============================================================================

4.1 go run
Compila y ejecuta en un solo paso. No deja un binario en el disco.

```bash
go run main.go        # Ejecuta un archivo
go run .              # Ejecuta todo el paquete
go run main.go util.go # Ejecuta múltiples archivos
```

Ventajas:
- Rápido para desarrollo
- No genera archivos temporales visibles
- Ideal para probar cambios rápidos

Desventajas:
- Más lento en ejecución (incluye compilación)
- No produce un binario distribuible

4.2 go build
Compila y genera un binario estático.

```bash
go build main.go      # Genera ./main
go build -o programa  # Genera ./programa
go build .            # Compila el paquete actual
```

Ventajas:
- Produce un binario autocontenido
- El binario se puede distribuir sin dependencias
- Más rápido en ejecución (ya está compilado)

Desventajas:
- Requiere un paso adicional
- El binario puede ser grande (incluye el runtime)

4.3 go install
Compila e instala el binario en `$GOPATH/bin`.

```bash
go install .
# El binario queda en $GOPATH/bin/
```

Esto permite ejecutar el programa desde cualquier lugar si
`$GOPATH/bin` está en el PATH.

===============================================================================
5. GO FMT: FORMATEO AUTOMÁTICO
===============================================================================

Una de las características más valoradas de Go es `go fmt`, que
formatea el código automáticamente según las convenciones oficiales.

5.1 Ejecutar go fmt

```bash
go fmt archivo.go     # Formatea un archivo
go fmt ./...          # Formatea todo el proyecto
```

5.2 Ejemplo: código desordenado

Código sin formatear (`desordenado.go`):

```go
package main
import "fmt"
func main() {
    fmt.Println("¡Hola!")
}
```

Después de `go fmt`:

```go
package main

import "fmt"

func main() {
    fmt.Println("¡Hola!")
}
```

5.3 Reglas de go fmt

- Tabulaciones para indentación (no espacios).
- Llaves en la misma línea que la declaración.
- Espacio después de `//` en comentarios.
- Importaciones agrupadas y ordenadas.
- Espacios alrededor de operadores.
- Líneas en blanco entre bloques lógicos.

5.4 Beneficios

- Elimina debates sobre estilo de código.
- Todo el código Go se ve igual.
- Facilita la revisión de código (diferencias limpias).
- Las herramientas pueden modificar el código sin preocuparse.

===============================================================================
6. PAQUETES EN GO
===============================================================================

Un paquete es la unidad de organización en Go. Cada archivo.go
pertenece a un paquete.

6.1 Declaración de paquete

```go
package main
package fmt
package http
```

6.2 Paquete main
Especial: genera un binario ejecutable. Solo el paquete `main`
necesita una función `main()`.

6.3 Importación de paquetes

```go
import "fmt"
import "os"
import "net/http"

// O agrupado:
import (
    "fmt"
    "os"
    "net/http"
)
```

6.4 Importación con alias

```go
import (
    f "fmt"
    "os"
)

func main() {
    f.Println("Usando alias")
    os.Exit(0)
}
```

6.5 Importación "en blanco" (blank import)

```go
import _ "image/png"
// Solo ejecuta init() del paquete
```

6.6 Crear tu propio paquete

```go
// saludos/saludos.go
package saludos

func Saludar(nombre string) string {
    return "¡Hola, " + nombre + "!"
}
```

```go
// main.go
package main

import (
    "fmt"
    "mi-proyecto/saludos"
)

func main() {
    mensaje := saludos.Saludar("Lucas")
    fmt.Println(mensaje)
}
```

===============================================================================
7. LA FUNCIÓN MAIN
===============================================================================

La función `main` es especial en Go:

- Es el punto de entrada del programa.
- No recibe parámetros (a diferencia de C).
- No retorna valores (usa `os.Exit` para códigos de salida).
- Solo una por programa (en el paquete main).
- Es opcional en paquetes que no son main.

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    if len(os.Args) < 2 {
        fmt.Println("Uso: programa <nombre>")
        os.Exit(1)
    }
    nombre := os.Args[1]
    fmt.Printf("¡Hola, %s!\n", nombre)
}
```

7.1 os.Args: argumentos de línea de comandos

`os.Args` es un slice de strings que contiene los argumentos
pasados al programa. El primer elemento (`os.Args[0]`) es
el nombre del programa.

```bash
go run main.go Lucas
# Salida: ¡Hola, Lucas!
```

===============================================================================
8. LA FUNCIÓN INIT
===============================================================================

Go permite una función especial `init()` que se ejecuta automáticamente
antes de `main()`, en el orden de inicialización de paquetes.

```go
package main

import "fmt"

func init() {
    fmt.Println("Inicializando...")
}

func init() {
    fmt.Println("Más inicialización...")
}

func main() {
    fmt.Println("Ejecutando programa principal")
}
```

Salida:
```
Inicializando...
Más inicialización...
Ejecutando programa principal
```

Características:
- Se puede tener múltiples `init()` en el mismo paquete.
- Se ejecutan en el orden de declaración.
- No reciben parámetros ni retornan valores.
- Útiles para configurar conexiones, variables globales, etc.

===============================================================================
9. EL PROGRAMA MÁS PEQUEÑO POSIBLE
===============================================================================

El programa Go más pequeño posible es:

```go
package main
func main() {}
```

Este programa compila y ejecuta sin errores, aunque no hace nada útil.
Podemos verificarlo:

```bash
cat > minimo.go << 'EOF'
package main
func main() {}
EOF
go build minimo.go
./minimo
echo "Código de salida: $?"
```

===============================================================================
10. ERRORES COMUNES EN EL PRIMER PROGRAMA
===============================================================================

10.1 Paquete incorrecto

```go
package fmt   // Error: debe ser "main"

func main() {
    fmt.Println("Hola")
}
```

Error: `main() must be in package main`

10.2 Función main mal escrita

```go
package main

import "fmt"

func Main() {   // Error: mayúscula, debe ser minúscula
    fmt.Println("Hola")
}
```

Compila, pero no se ejecuta la función (no es el punto de entrada).

10.3 Import no usado

```go
package main

import (
    "fmt"
    "os"    // Error: importado pero no usado
)

func main() {
    fmt.Println("Hola")
}
```

Go no permite imports no usados (error de compilación).

10.4 Variables no usadas

```go
package main

import "fmt"

func main() {
    x := 42   // Error: variable declarada pero no usada
    fmt.Println("Hola")
}
```

Go no permite variables no usadas (error de compilación).

10.5 Llaves en línea incorrecta

```go
package main

import "fmt"

func main()   // Error: la llave debe ir en esta línea
{
    fmt.Println("Hola")
}
```

Error: `syntax error: unexpected newline, expecting {`

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- Todo programa Go ejecutable necesita `package main`, `func main()`.
- `import "fmt"` trae el paquete de formato de la biblioteca estándar.
- `fmt.Println` imprime texto con salto de línea.
- `go run` compila y ejecuta en un paso; `go build` genera un binario.
- `go fmt` formatea el código automáticamente según las convenciones.
- Los paquetes organizan el código y pueden ser ejecutables (main)
  o bibliotecas (cualquier otro nombre).
- La función `init()` se ejecuta automáticamente antes de `main()`.
- Go es estricto: no permite imports no usados ni variables no usadas.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué Go requiere `package main` para programas ejecutables?**
Porque Go distingue entre programas (binarios) y bibliotecas
(paquetes reutilizables). `package main` genera un binario,
mientras que cualquier otro nombre genera una biblioteca.

❓ **¿Qué pasa si no pongo `import "fmt"`?**
El compilador dará error porque `fmt.Println` no está definido.
Debes importar explícitamente todos los paquetes que uses.

❓ **¿Puedo tener `main()` en otro archivo?**
Sí, pero solo un `main()` por programa. Si tienes varios archivos
en el mismo paquete `main`, la función `main()` debe estar en
solo uno de ellos.

❓ **¿Por qué `go run` es más lento que ejecutar el binario?**
`go run` compila el código primero, mientras que el binario
ya está compilado. Para programas pequeños la diferencia es
imperceptible. Para proyectos grandes, `go build` primero
y luego ejecutar el binario es más rápido.

❓ **¿Qué es la función `init()` y para qué sirve?**
`init()` se ejecuta automáticamente al cargar un paquete,
antes de `main()`. Sirve para inicializar variables,
establecer conexiones, registrar drivers, etc. Se puede
tener múltiples `init()` en un mismo archivo o paquete.

❓ **¿Go tiene punto y coma?**
Sintácticamente no, pero el lexer de Go inserta punto y coma
automáticamente al final de cada línea que no pueda ser
continuación de la anterior. Por eso las llaves deben ir
en la misma línea que la declaración.

Diseña una herramienta CLI llamada `gonew` que genere la
estructura inicial de un proyecto Go con:

1. Estructura de directorios estándar (cmd/, internal/,
   pkg/, etc).

4. Archivo `README.md` básico.

argumento y crear todo automáticamente.

```
mi-api/
├── cmd/
│   └── server/
│       └── main.go
├── internal/
├── pkg/
├── go.mod
├── README.md
├── .gitignore
└── .vscode/
    └── settings.json
```

Este desafío integra todo lo aprendido: módulos, paquetes,
estructura de proyectos y argumentos de línea de comandos.
