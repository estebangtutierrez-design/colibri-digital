1. DECLARACIÓN DE CONSTANTES
===============================================================================

```go
package main

import "fmt"

func main() {
    // Constante simple
    const Pi = 3.14159
    const Nombre = "Lucas"
    const Activo = true

    fmt.Println(Pi, Nombre, Activo)

    // Constante con tipo explícito
    const Edad int = 30
    const Precio float64 = 99.99
    const Inicial rune = 'A'

    fmt.Printf("Edad: %d, Precio: %.2f, Inicial: %c\n", Edad, Precio, Inicial)

    // Las constantes no pueden modificarse
    // Pi = 3.14  // Error: no se puede modificar una constante

    // Múltiples constantes
    const X, Y = 10, 20
    const (
        A = 1
        B = 2
        C = 3
    )
    fmt.Println(X, Y)
    fmt.Println(A, B, C)
}
```

1.1 Reglas de las constantes

- Se declaran con `const`, no con `:=`.
- Deben ser valores conocidos en tiempo de compilación.
- Solo tipos básicos: números (int, float, complex), string, bool.
- No pueden ser slices, maps, structs, funciones, etc.
- Se evalúan en tiempo de compilación.
- Pueden ser tipadas o no tipadas.

```go
package main

import "fmt"

func main() {
    // Esto es válido
    const X = 2 + 3  // Se evalúa en compilación
    fmt.Println(X)

    // Esto NO es válido
    // const Y = time.Now().Unix()  // Error: no es constante en compilación
    // const Z = rand.Int()         // Error: no es constante

    // Esto tampoco
    // const S []int = []int{1, 2, 3}  // Error: no es tipo básico
    // const M map[string]int = {"a": 1} // Error: no es tipo básico
}
```

2. CONSTANTES TIPADAS VS NO TIPADAS
===============================================================================

```go
package main

import "fmt"

func main() {
    // Constante no tipada (tipo implícito, flexible)
    const Valor = 42          // Sin tipo explícito
    const Texto = "Hola"      // Sin tipo explícito

    // Pueden usarse en diferentes contextos
    var entero int = Valor
    var flotante float64 = Valor
    fmt.Println(entero, flotante)

    // Constante tipada (tipo fijo, estricto)
    const ValorTipado int = 42
    const TextoTipado string = "Hola"

    // Solo puede usarse con su tipo específico
    var entero2 int = ValorTipado
    // var flotante2 float64 = ValorTipado // Error: int no es float64
    _ = entero2

    // Las constantes no tipadas tienen "precisión arbitraria"
    const MuyGrande = 1 << 200  // Válido como constante no tipada
    fmt.Printf("Tipo: %T, Valor: %v (truncado a int)\n", MuyGrande, MuyGrande)

    // Pero al asignar a una variable, debe caber
    // var x int = MuyGrande  // Error: overflow
}
```

2.1 ¿Por qué existen constantes no tipadas?

```go
package main

import "fmt"

func main() {
    // Sin constantes no tipadas, esto sería engorroso:
    const MinutosEnHora = 60

    // Con constante no tipada, funciona en cualquier contexto numérico
    var duracionMinutos int = 120
    var horas float64 = float64(duracionMinutos) / MinutosEnHora
    fmt.Println(horas) // 2.0

    // Si MinutosEnHora fuera int, la división sería entera
    const MinutosEnHoraInt int = 60
    var horasEnteras int = duracionMinutos / MinutosEnHoraInt
    fmt.Println(horasEnteras) // 2
}
```

3. INTRODUCCIÓN A IOTA
===============================================================================

`iota` es un identificador predeclarado que representa enteros
consecutivos dentro de una declaración `const`.

```go
package main

import "fmt"

func main() {
    const (
        LUNES = iota  // 0
        MARTES       // 1
        MIERCOLES    // 2
        JUEVES       // 3
        VIERNES      // 4
        SABADO       // 5
        DOMINGO      // 6
    )

    fmt.Println("LUNES:", LUNES)
    fmt.Println("MARTES:", MARTES)
    fmt.Println("MIERCOLES:", MIERCOLES)
    fmt.Println("JUEVES:", JUEVES)
    fmt.Println("VIERNES:", VIERNES)
    fmt.Println("SABADO:", SABADO)
    fmt.Println("DOMINGO:", DOMINGO)
}
```

3.1 iota se reinicia en cada bloque const

```go
package main

import "fmt"

func main() {
    const (
        A = iota // 0
        B        // 1
        C        // 2
    )

    const (
        D = iota // 0 (se reinicia)
        E        // 1
        F        // 2
    )

    fmt.Println(A, B, C) // 0 1 2
    fmt.Println(D, E, F) // 0 1 2
}
```

4. PATRONES COMUNES CON IOTA
===============================================================================

4.1 Comenzar desde 1

```go
package main

import "fmt"

func main() {
    const (
        _  = iota          // ignorar 0
        ENERO = iota       // 1
        FEBRERO            // 2
        MARZO              // 3
        ABRIL              // 4
        MAYO               // 5
        JUNIO              // 6
        JULIO              // 7
        AGOSTO             // 8
        SEPTIEMBRE         // 9
        OCTUBRE            // 10
        NOVIEMBRE          // 11
        DICIEMBRE          // 12
    )

    fmt.Println("Enero:", ENERO)
    fmt.Println("Diciembre:", DICIEMBRE)
}
```

4.2 Saltar valores con guion bajo

```go
package main

import "fmt"

func main() {
    const (
        _  = iota      // 0
        UNO            // 1

        _              // saltar 2

        TRES           // 3
        CUATRO         // 4

        _              // saltar 5
        SEIS           // 6
    )

    fmt.Println(UNO)    // 1
    fmt.Println(TRES)   // 3
    fmt.Println(CUATRO) // 4
    fmt.Println(SEIS)   // 6
}
```

4.3 iota con expresiones

```go
package main

import "fmt"

func main() {
    const (
        _  = iota                    // ignorar 0
        KB = 1 << (10 * iota)       // 1 << 10 = 1024
        MB                           // 1 << 20 = 1048576
        GB                           // 1 << 30 = 1073741824
        TB                           // 1 << 40 = 1099511627776
        PB                           // 1 << 50 = 1125899906842624
    )

    fmt.Println("KB:", KB)
    fmt.Println("MB:", MB)
    fmt.Println("GB:", GB)
    fmt.Println("TB:", TB)
    fmt.Println("PB:", PB)

    // Uso práctico
    var archivoBytes int64 = 2 * GB
    fmt.Printf("Archivo: %d bytes (%.2f GB)\n", archivoBytes,
        float64(archivoBytes)/float64(GB))
}
```

4.4 Flags con iota y bitwise

```go
package main

import "fmt"

func main() {
    const (
        Lectura  = 1 << iota // 001
        Escritura            // 010
        Ejecucion            // 100
    )

    // Combinar permisos
    permisos := Lectura | Escritura | Ejecucion
    fmt.Printf("Permisos completos: %03b\n", permisos)

    permisosLecturaEscritura := Lectura | Escritura
    fmt.Printf("Lectura y escritura: %03b\n", permisosLecturaEscritura)

    // Verificar permisos
    tieneLectura := permisos&Lectura != 0
    tieneEscritura := permisos&Escritura != 0
    tieneEjecucion := permisos&Ejecucion != 0

    fmt.Println("¿Tiene lectura?", tieneLectura)
    fmt.Println("¿Tiene escritura?", tieneEscritura)
    fmt.Println("¿Tiene ejecución?", tieneEjecucion)
}
```

4.5 Estados con iota

```go
package main

import "fmt"

func main() {
    const (
        EstadoPendiente = iota
        EstadoEnProgreso
        EstadoCompletado
        EstadoCancelado
        EstadoFallido
    )

    estado := EstadoEnProgreso

    switch estado {
    case EstadoPendiente:
        fmt.Println("La tarea está pendiente")
    case EstadoEnProgreso:
        fmt.Println("La tarea está en progreso")
    case EstadoCompletado:
        fmt.Println("La tarea está completada")
    case EstadoCancelado:
        fmt.Println("La tarea fue cancelada")
    case EstadoFallido:
        fmt.Println("La tarea falló")
    default:
        fmt.Println("Estado desconocido")
    }

    // Función helper para nombre del estado
    fmt.Println("Nombre del estado:", nombreEstado(estado))
}

func nombreEstado(estado int) string {
    nombres := []string{
        "Pendiente",
        "En Progreso",
        "Completado",
        "Cancelado",
        "Fallido",
    }
    if estado >= 0 && estado < len(nombres) {
        return nombres[estado]
    }
    return "Desconocido"
}
```

5. IOTA AVANZADO
===============================================================================

5.1 iota con múltiples constantes por línea

```go
package main

import "fmt"

func main() {
    const (
        A, B = iota, iota * 10  // 0, 0
        C, D                    // 1, 10
        E, F                    // 2, 20
    )

    fmt.Println(A, B) // 0 0
    fmt.Println(C, D) // 1 10
    fmt.Println(E, F) // 2 20
}
```

5.2 iota con operaciones complejas

```go
package main

import "fmt"

func main() {
    const (
        _  = iota
        Rojo   = iota + 1   // 2
        Verde  = iota * 2   // 4
        Azul   = iota * 3   // 6
    )

    fmt.Println("Rojo:", Rojo)
    fmt.Println("Verde:", Verde)
    fmt.Println("Azul:", Azul)
}
```

5.3 iota con stringers automáticos

```go
package main

import "fmt"

type DiaSemana int

const (
    Lunes DiaSemana = iota
    Martes
    Miercoles
    Jueves
    Viernes
    Sabado
    Domingo
)

func (d DiaSemana) String() string {
    dias := [...]string{
        "Lunes", "Martes", "Miércoles",
        "Jueves", "Viernes", "Sábado", "Domingo",
    }
    if int(d) < len(dias) {
        return dias[d]
    }
    return "Desconocido"
}

func main() {
    dia := Miercoles
    fmt.Println("Hoy es:", dia)
    fmt.Println("Mañana es:", Miercoles+1)
}
```

```go
    fmt.Println("Pi:", 3.141592653589793)
    fmt.Println("E:", 2.718281828459045)
}
```

10. ERRORES COMUNES CON CONSTANTES
===============================================================================

```go
package main

import "fmt"

func main() {
    // Error 1: Modificar una constante
    // const x = 10
    // x = 20  // Error: no se puede modificar

    // Error 2: Constante sin usar (esto SÍ compila, a diferencia de variables)
    const noUsada = 42  // Válido, las constantes no usadas no dan error
    _ = noUsada

    // Error 3: Asignar valor no constante
    // const tiempo = time.Now()  // Error: no es constante de compilación

    // Error 4: Constante con tipo complejo
    // const mapa = map[string]int{"a": 1}  // Error

    // Error 5: Desbordamiento
    // const grande int8 = 200  // Error: overflow

    // Esto sí funciona (constante no tipada)
    const grande = 200
    // var x int8 = grande  // Error: overflow (en asignación)

    // Error 6: iota fuera de const
    // var x = iota  // Error: iota solo en const

    // Error 7: Duplicar iota en el mismo bloque
    const (
        A = iota  // 0
        B = iota  // 1 (explícito, innecesario pero válido)
        C = iota  // 2
    )
    fmt.Println(A, B, C)

    // Mejor (implícito):
    const (
        X = iota  // 0
        Y         // 1 (implícito)
        Z         // 2 (implícito)
    )
    fmt.Println(X, Y, Z)
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Las constantes se declaran con `const` y deben tener valores conocidos
  en tiempo de compilación.
- Las constantes no tipadas son flexibles y se adaptan al contexto.
- Las constantes tipadas tienen un tipo fijo y estricto.
- `iota` genera secuencias de enteros dentro de bloques `const`.
- Patrones comunes: días de la semana, flags bitwise, estados,
  tamaños de datos (KB, MB, GB).
- Las constantes no pueden ser el resultado de llamadas a funciones.
- Usa constantes para evitar magic numbers en tu código.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre const y var?**
`const` declara valores inmutables conocidos en compilación.
`var` declara variables mutables. Las constantes solo pueden
ser tipos básicos (números, strings, bools). Las variables
pueden ser cualquier tipo.

❓ **¿Qué es iota y cómo funciona?**
`iota` es un contador entero que comienza en 0 dentro de cada
bloque `const`. Se incrementa automáticamente en cada línea
nueva. Se usa para crear enumeraciones y secuencias de
valores constantes.

❓ **¿Puedo tener constantes de tipo float o string?**
Sí, Go soporta constantes de tipo float64, string y bool,
además de enteros. Todas deben ser valores literales o
expresiones evaluables en tiempo de compilación.

❓ **¿Las constantes ocupan memoria?**
Las constantes no tipadas no ocupan memoria en tiempo de
ejecución; se sustituyen por su valor literal en el código
compilado. Las constantes tipadas se comportan como variables
inmutables.

❓ **¿Puedo usar iota fuera de un bloque const?**
No. `iota` solo tiene significado dentro de un bloque `const`.
Fuera de él, no está definido.

❓ **¿Cómo reinicio iota?**
Crea un nuevo bloque `const`. `iota` se reinicia a 0 en
cada bloque `const` nuevo.
