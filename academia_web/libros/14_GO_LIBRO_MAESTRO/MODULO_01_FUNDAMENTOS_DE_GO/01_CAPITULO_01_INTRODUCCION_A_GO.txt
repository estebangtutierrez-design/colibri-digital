1. ¿QUÉ ES GO?
===============================================================================

Go, también conocido como Golang, es un lenguaje de programación compilado,
tipado estáticamente y concurrente, diseñado por ingenieros de Google.
Su sintaxis es limpia y minimalista, inspirada en C pero con mejoras
significativas en seguridad, legibilidad y productividad.

Go fue creado para abordar problemas que los equipos de Google enfrentaban
al desarrollar software a gran escala: compilaciones lentas, dependencias
enredadas, falta de concurrencia nativa y código difícil de mantener.

El resultado es un lenguaje que compila rápido, ejecuta rápido y permite
escribir programas claros y eficientes.

```go
package main

import "fmt"

func main() {
    fmt.Println("¡Bienvenido al mundo de Go!")
}
```

Este pequeño programa, aunque sencillo, muestra la esencia de Go:
un paquete principal (`package main`), una importación (`import "fmt"`),
una función principal (`func main()`) y una llamada a la biblioteca
estándar para imprimir en pantalla.

Go no necesita punto y coma al final de cada línea (aunque el lexer los
inserta automáticamente). Las llaves van en la misma línea que la
declaración de la función. El lenguaje fuerza un estilo consistente,
lo que elimina debates sobre formato dentro del equipo.

2. CARACTERÍSTICAS PRINCIPALES DE GO
===============================================================================

Go posee una serie de características que lo hacen único:

2.1 Compilado y rápido
Go compila directamente a código máquina, sin necesidad de una máquina
virtual. Esto da como resultado ejecutables binarios estáticos que se
pueden copiar a cualquier servidor sin instalar dependencias.

```go
// Compilar y ejecutar
// $ go build programa.go
// $ ./programa
```

Un binario típico de Go tiene entre 5 y 20 MB, e incluye todo lo
necesario para ejecutarse, incluyendo el runtime y el recolector
de basura.

2.2 Tipado estático
Las variables en Go tienen un tipo conocido en tiempo de compilación.
Esto permite que el compilador detecte errores de tipo antes de que
el programa llegue a producción.

```go
var edad int = 30
nombre := "Lucas"
// edad = "treinta" // Error de compilación: no se puede asignar string a int
```

2.3 Concurrencia nativa
Go incluye gorutinas y canales como parte del lenguaje, no como una
biblioteca externa. Esto permite escribir código concurrente de forma
sencilla y segura.

```go
go func() {
    fmt.Println("Esto se ejecuta en una gorutina")
}()
```

2.4 Recolector de basura
Go maneja la memoria automáticamente mediante un garbage collector
eficiente y de baja latencia. El programador no necesita liberar
memoria explícitamente como en C o C++.

2.5 Sintaxis minimalista
Go tiene muy pocas palabras clave (25 en total). No tiene clases,
herencia, genéricos (antes de 1.18), excepciones, ni muchas de las
características que otros lenguajes consideran esenciales.

```go
// No hay while, solo for
for i := 0; i < 10; i++ {
    fmt.Println(i)
}

// for como while
x := 0
for x < 10 {
    fmt.Println(x)
    x++
}

// for infinito
for {
    break
}
```

2.6 Bibliotecas estándar poderosas
Go incluye bibliotecas para HTTP, JSON, criptografía, compresión,
bases de datos, pruebas, formato de texto, y mucho más.

```go
import (
    "encoding/json"
    "net/http"
    "os"
)
```

2.7 Herramientas integradas
El comando `go` incluye herramientas para formatear código (`go fmt`),
gestionar dependencias (`go mod`), ejecutar pruebas (`go test`),
analizar código (`go vet`), y generar documentación (`go doc`).

2.8 Compatibilidad hacia atrás
Desde Go 1.0, el lenguaje garantiza que el código escrito para una
versión 1.x seguirá compilando en versiones 1.y posteriores (siendo
y > x). Esta promesa de compatibilidad es única y muy valorada.

3. CASOS DE USO DE GO
===============================================================================

Go se utiliza en una amplia variedad de dominios:

3.1 Servicios web y APIs REST
Go es ideal para construir microservicios gracias a su rendimiento,
bajo uso de memoria y concurrencia nativa.

```go
package main

import (
    "fmt"
    "net/http"
)

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hola, %s!", r.URL.Path[1:])
}

func main() {
    http.HandleFunc("/", handler)
    http.ListenAndServe(":8080", nil)
}
```

3.2 Herramientas de línea de comandos (CLI)
Go produce binarios estáticos que no necesitan runtime externo,
perfectos para herramientas CLI portables.

Herramientas famosas escritas en Go:
- Docker: Contenedores de software
- Kubernetes: Orquestación de contenedores
- Terraform: Infraestructura como código
- Hugo: Generador de sitios estáticos
- Prometheus: Sistema de monitorización

3.3 Infraestructura en la nube
La mayoría de las plataformas cloud modernas tienen componentes
escritos en Go: Docker, Kubernetes, etcd, containerd, Traefik, Caddy.

3.4 DevOps y automatización
Go se usa extensivamente para escribir pipelines, herramientas de
despliegue, y automatización de infraestructura.

3.5 Procesamiento de datos y streaming
Bibliotecas como GoStream y el soporte nativo para concurrencia
hacen de Go una excelente opción para pipelines de datos.

3.6 Aplicaciones de red
Servidores proxy, balanceadores de carga, cortafuegos, y cualquier
aplicación que maneje muchas conexiones simultáneas se benefician
del modelo de concurrencia de Go.

===============================================================================
4. GO VS OTROS LENGUAJES
===============================================================================

4.1 Go vs Python
Python es más lento (interpretado), Go es rápido (compilado).
Python tiene tipado dinámico, Go tiene tipado estático.
Python tiene una sintaxis más expresiva, Go es más explícito.
Go tiene concurrencia nativa (gorutinas), Python tiene GIL.

4.2 Go vs Java
Java tiene una máquina virtual (JVM), Go compila a binario nativo.
Java requiere más código boilerplate, Go es más conciso.
Java tiene herencia de clases, Go usa composición e interfaces.
Java tiene un ecosistema enorme, Go tiene un ecosistema más joven.

```go
// Go
type Animal struct {
    Nombre string
}

func (a Animal) Hablar() string {
    return "..."
}
```

```go
// Equivalente en Java (mucho más código)
// public class Animal {
//     private String nombre;
//     public Animal(String nombre) {
//         this.nombre = nombre;
//     }
//     public String hablar() {
//         return "...";
//     }
// }
```

4.3 Go vs C++
Go tiene recolector de basura, C++ requiere gestión manual de memoria.
Go compila más rápido, C++ tiene compilaciones más lentas.
Go no tiene punteros aritméticos, C++ sí.
Go tiene canales y gorutinas, C++ tiene threads más pesados.

4.4 Go vs Rust
Ambos son compilados y estáticos. Rust no tiene GC (gestión de memoria
por ownership), Go sí tiene GC. Rust es más complejo de aprender,
Go es más simple. Rust tiene mejor control de bajo nivel,
Go prioriza la productividad del desarrollador.

6. EL ECOSISTEMA DE GO
===============================================================================

6.1 El comando `go`
El binario `go` incluye múltiples herramientas:
- `go build`: Compila paquetes y dependencias
- `go run`: Compila y ejecuta
- `go test`: Ejecuta pruebas unitarias
- `go fmt`: Formatea código
- `go vet`: Analiza código en busca de errores sospechosos
- `go doc`: Muestra documentación
- `go mod`: Gestiona módulos y dependencias
- `go get`: Descarga dependencias
- `go install`: Compila e instala paquetes

6.2 Go Modules
Desde Go 1.11, el sistema oficial de gestión de dependencias.

6.3 pkg.go.dev
El repositorio oficial de paquetes de Go, con documentación y
ejemplos de uso.

6.4 GOPATH
El directorio de trabajo tradicional de Go (anterior a los módulos).
Aunque hoy los módulos son el estándar, muchos tutoriales antiguos
aún lo usan.

6.5 gopls
El servidor de lenguaje LSP para Go, que proporciona autocompletado,
navegación, refactorización, detección de errores y más en editores
como VS Code, Vim, Emacs y otros.

===============================================================================
7. FILOSOFÍA DE GO
===============================================================================

La filosofía de Go se resume en varios principios:

- Simplicidad sobre complejidad
- Claridad sobre brevedad
- Composición sobre herencia
- Concurrencia como ciudadano de primera clase
- Interfaces implícitas (duck typing estático)
- Paquetes como unidad de organización
- Pruebas integradas en el lenguaje
- Herramientas unificadas

El equipo de Go dijo una vez: "Go es el lenguaje que escribimos para
escribir el software que escribimos." Fue diseñado para resolver
problemas reales, no para ser el lenguaje más innovador.

Go deliberadamente omite características que sus creadores consideran
problemáticas:
- Sin herencia de clases (solo composición)
- Sin excepciones (solo errores como valores)
- Sin genéricos (añadidos en Go 1.18)
- Sin sobrecarga de operadores
- Sin punteros aritméticos
- Sin tipos de datos dependientes

```go
// Manejo de errores como valores en Go
f, err := os.Open("archivo.txt")
if err != nil {
    fmt.Println("Error:", err)
    return
}
defer f.Close()
```

8. COMPILACIÓN Y EJECUCIÓN
===============================================================================

Go es un lenguaje compilado, pero el proceso es extremadamente rápido
comparado con C++ o Rust. La compilación incluye varias etapas:

1. Análisis léxico: El código fuente se convierte en tokens.
2. Análisis sintáctico: Los tokens se organizan en un AST.
3. Análisis semántico: Se verifica la corrección de tipos.
4. Generación de código: Se produce el binario nativo.

```go
// hello.go
package main

import "fmt"

func main() {
    fmt.Println("Hola, Go!")
}
```

Para compilar:
```
$ go build hello.go
$ ./hello
Hola, Go!
```

Para compilar y ejecutar en un solo paso:
```
$ go run hello.go
Hola, Go!
```

El binario generado es estático y autocontenido. Puedes copiarlo
a cualquier máquina Linux (o del mismo sistema operativo y
arquitectura) y ejecutarlo sin instalar nada más.

Para compilar para otra plataforma, usa las variables de entorno
GOOS y GOARCH:

```
$ GOOS=windows GOARCH=amd64 go build hello.go
$ GOOS=linux GOARCH=arm64 go build hello.go
$ GOOS=darwin GOARCH=amd64 go build hello.go
```

9. EL RUNTIME DE GO
===============================================================================

El runtime de Go es la parte del lenguaje que maneja:
- La ejecución de gorutinas (scheduler M:N)
- El recolector de basura (GC)
- El manejo de memoria (asignación y liberación)
- Los canales y operaciones de concurrencia
- El mapeo de tipos (interfaces, reflexión)

El scheduler de Go implementa un modelo M:N: M gorutinas se ejecutan
en N hilos del sistema operativo. Esto permite que miles o millones
de gorutinas coexistan con un costo mínimo.

```go
// El runtime maneja automáticamente la concurrencia
package main

import (
    "fmt"
    "runtime"
)

func main() {
    fmt.Println("Núcleos disponibles:", runtime.NumCPU())
    fmt.Println("GOROOT:", runtime.GOROOT())
    fmt.Println("Versión:", runtime.Version())
    fmt.Println("Arquitectura:", runtime.GOARCH)
    fmt.Println("OS:", runtime.GOOS)
}
```

El recolector de basura es de tipo concurrente, mark-and-sweep
con compactación. Está diseñado para tener pausas de menos de
500 microsegundos en la mayoría de los casos.

10. ¿POR QUÉ APRENDER GO EN 2026?
===============================================================================

Razones para aprender Go hoy:

1. Demanda laboral creciente: Go está entre los lenguajes mejor
pagados y con mayor demanda de la industria.

2. Rendimiento excelente: Cerca de C++ en velocidad, pero con
la facilidad de Python en la mayoría de los casos.

3. Código mantenible: Las restricciones del lenguaje fuerzan
código consistente y legible.

4. Puerta al cloud computing: Docker, Kubernetes, Terraform...
todas las herramientas cloud están en Go.

5. Comunidad activa y en crecimiento: Miles de paquetes,
conferencias (GopherCon), meetups y recursos.

6. Ideal para equipos grandes: La simplicidad de Go hace que
el código sea fácil de entender por cualquier miembro del equipo.

7. Binarios estáticos: Despliegue simple y sin dependencias.

8. Puerta de entrada a sistemas: Entender Go ayuda a entender
cómo funcionan los sistemas operativos y las redes.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Go es un lenguaje orientado a objetos?**
Técnicamente no, aunque soporta interfaces y métodos. Go no tiene
clases ni herencia. Usa composición en lugar de herencia, y las
interfaces se implementan implícitamente. Se dice que Go es un
lenguaje orientado a objetos sin jerarquía de tipos.

❓ **¿Go es más rápido que Python?**
Sí, Go es significativamente más rápido (entre 10x y 100x) porque
es compilado a código máquina, mientras que Python es interpretado.
Además, la concurrencia en Go es mucho más eficiente.

❓ **¿Necesito instalar algo además de Go?**
Solo necesitas el SDK de Go. Para editar código, VS Code con la
extensión de Go es la opción más popular, pero puedes usar
cualquier editor de texto.

❓ **¿Qué es GOPATH?**
GOPATH es la variable de entorno que define el directorio raíz
del workspace de Go. Con Go Modules (desde 1.11), ya no es
necesario usar GOPATH para la mayoría de proyectos.

❓ **¿Puedo usar Go para desarrollo web?**
Sí, Go incluye el paquete `net/http` para crear servidores web,
y hay frameworks populares como Gin, Echo, Fiber y Chi.

❓ **¿Go tiene excepciones?**
No, Go no tiene excepciones al estilo try-catch. En su lugar,
usa el patrón de retornar errores como valores. Esto hace que
el manejo de errores sea explícito y visible.

❓ **¿Cuánto tiempo toma aprender Go?**
Un programador con experiencia puede aprender los fundamentos
de Go en 1-2 semanas. La sintaxis es pequeña y consistente.
Dominar la concurrencia y los patrones avanzados puede tomar
varios meses.

❓ **¿Go reemplazará a C++ o Java?**
No, cada lenguaje tiene su nicho. Go es excelente para servicios
web, herramientas CLI y cloud computing, pero no es ideal para
sistemas de tiempo real, kernels, o aplicaciones que requieran
máximo control sobre la memoria.
