1. INSTALACIÓN DEL SDK DE GO
===============================================================================

El primer paso es instalar el compilador y las herramientas de Go.
El SDK (Software Development Kit) de Go incluye:

- El compilador (go build)
- El gestor de paquetes (go mod)
- El formateador (go fmt)
- Las herramientas de prueba (go test)
- El servidor LSP (gopls)
- La documentación (go doc)
- Y muchas más herramientas

1.1 Descarga
Ve a https://go.dev/dl y descarga la versión más reciente para
tu sistema operativo y arquitectura.

Opciones disponibles:
- Linux: .tar.gz (para distribuciones basadas en Linux)
- macOS: .pkg (instalador) o .tar.gz
- Windows: .msi (instalador) o .zip

Siempre elige la versión más reciente (al momento de escribir,
Go 1.24.x).

1.2 Instalación en Linux

```bash
# Descargar Go (sustituir con la versión más reciente)
wget https://go.dev/dl/go1.24.0.linux-amd64.tar.gz

# Verificar integridad (opcional pero recomendado)
sha256sum go1.24.0.linux-amd64.tar.gz

# Extraer en /usr/local (requiere sudo)
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.24.0.linux-amd64.tar.gz

# Añadir Go al PATH
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPATH=$HOME/go' >> ~/.bashrc
echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.bashrc

# Recargar configuración
source ~/.bashrc

# Verificar instalación
go version
```

Salida esperada:
```
go version go1.24.0 linux/amd64
```

1.3 Instalación en macOS

```bash
# Opción 1: Descargar el .pkg desde go.dev
# Simplemente haz doble clic y sigue el asistente.

# Opción 2: Usando Homebrew
brew install go

# Verificar
go version
```

1.4 Instalación en Windows

```bash
# Opción 1: Descargar el .msi desde go.dev
# Ejecuta el instalador y sigue los pasos.
# El instalador configura automáticamente el PATH.

# Opción 2: Con winget (Windows Package Manager)
winget install GoLang.Go

# Verificar en PowerShell
go version
```

1.5 Verificación completa

```bash
# Verificar versión
go version

# Ver entorno
go env

# Ver GOROOT (dónde está instalado Go)
go env GOROOT

# Ver GOPATH (dónde irán tus paquetes descargados)
go env GOPATH

# Ver GOOS y GOARCH (sistema operativo y arquitectura)
go env GOOS
go env GOARCH
```

2. VARIABLES DE ENTORNO IMPORTANTES
===============================================================================

2.1 GOROOT
Directorio donde está instalado el SDK de Go. Normalmente:
- Linux/macOS: `/usr/local/go`
- Windows: `C:\Go`

No necesitas tocar esto a menos que tengas una instalación
personalizada.

```go
package main

import (
    "fmt"
    "runtime"
)

func main() {
    fmt.Println("GOROOT:", runtime.GOROOT())
    fmt.Println("Versión:", runtime.Version())
    fmt.Println("OS/Arch:", runtime.GOOS+"/"+runtime.GOARCH)
}
```

2.2 GOPATH
Directorio de trabajo para los proyectos Go. Por defecto:
- Linux: `$HOME/go`
- macOS: `$HOME/go`
- Windows: `%USERPROFILE%\go`

Estructura típica de GOPATH:
```
$HOME/go/
├── bin/       # Binarios compilados
├── pkg/       # Paquetes compilados
└── src/       # Código fuente (obsoleto con Go Modules)
```

Con Go Modules, ya no es necesario que tu código esté dentro de
GOPATH/src. Puedes trabajar en cualquier directorio.

2.3 GOBIN
Directorio donde `go install` coloca los binarios compilados.
Si no se define, por defecto es `$GOPATH/bin`.

Se recomienda añadir `$GOPATH/bin` al PATH:
```bash
export PATH=$PATH:$(go env GOPATH)/bin
```

2.4 GOOS y GOARCH
Variables para compilar para otras plataformas:

```bash
# Compilar para Windows desde Linux
GOOS=windows GOARCH=amd64 go build programa.go

# Compilar para ARM (Raspberry Pi)
GOOS=linux GOARCH=arm64 go build programa.go

# Compilar para macOS
GOOS=darwin GOARCH=amd64 go build programa.go
```

```go
package main

import (
    "fmt"
    "runtime"
)

func main() {
    fmt.Printf("Compilando para: %s/%s\n", runtime.GOOS, runtime.GOARCH)
    fmt.Printf("Versión Go: %s\n", runtime.Version())
}
```

3. VS CODE CONFIGURACIÓN
===============================================================================

VS Code es el editor más popular para Go, aunque no es el único.
Las alternativas incluyen:
- Vim/Neovim + vim-go
- Emacs + go-mode
- GoLand (IDE de JetBrains, de pago)
- Sublime Text con paquetes Go

3.1 Instalar VS Code
Descarga desde https://code.visualstudio.com

3.2 Extensiones necesarias

- **Go** (golang.go): La extensión oficial de Go para VS Code.
  Proporciona:
  - Autocompletado inteligente (con gopls)
  - Formateo automático (go fmt o gofumpt)
  - Navegación de código (ir a definición, referencias)
  - Refactorización (renombrar, extraer)
  - Depuración (con delve)
  - Pruebas integradas
  - Análisis de código en tiempo real

3.3 Instalación de la extensión Go

1. Abre VS Code
2. Ve a Extensiones (Ctrl+Shift+X)
3. Busca "Go" de Google
4. Haz clic en Instalar
5. VS Code te pedirá instalar herramientas adicionales (gopls,
   delve, staticcheck, etc). Acepta todo.

3.4 Verificar gopls

```bash
# gopls (Go Protocol Language Server) debe estar instalado
gopls version
```

Si no está instalado:
```bash
go install golang.org/x/tools/gopls@latest
```

3.5 Otras herramientas útiles

```bash
# Delve (depurador)
go install github.com/go-delve/delve/cmd/dlv@latest

# Staticcheck (lint avanzado)
go install honnef.co/go/tools/cmd/staticcheck@latest

# Go fumpt (formato más estricto)
go install mvdan.cc/gofumpt@latest

# Go imports (gestión de imports)
go install golang.org/x/tools/cmd/goimports@latest
```

3.6 Configuración recomendada de VS Code

Crea o edita `.vscode/settings.json` en tu proyecto:

```json
{
    "go.goroot": "/usr/local/go",
    "go.gopath": "/home/tu-usuario/go",
    "go.useLanguageServer": true,
    "go.languageServerFlags": [],
    "go.formatTool": "gofumpt",
    "go.lintTool": "staticcheck",
    "go.lintFlags": ["-checks", "all"],
    "go.testFlags": ["-v"],
    "editor.formatOnSave": true,
    "editor.codeActionsOnSave": {
        "source.organizeImports": true
    },
    "[go]": {
        "editor.tabSize": 4,
        "editor.insertSpaces": true
    }
}
```

4. GO MODULES
===============================================================================

Go Modules es el sistema oficial de gestión de dependencias,
introducido en Go 1.11 (2018) y estabilizado en Go 1.14 (2020).

Un módulo es una colección de paquetes Go con un archivo
`go.mod` en la raíz.

4.1 Inicializar un módulo

```bash
# Crear directorio del proyecto
mkdir mi-proyecto
cd mi-proyecto

# Inicializar módulo
go mod init github.com/tu-usuario/mi-proyecto
```

Esto crea un archivo `go.mod` con:

```
module github.com/tu-usuario/mi-proyecto

go 1.24
```

4.2 Añadir dependencias

```go
package main

import (
    "fmt"
    "rsc.io/quote"
)

func main() {
    fmt.Println(quote.Go())
}
```

```bash
# Go descarga automáticamente las dependencias al compilar
go mod tidy
```

Esto actualiza `go.mod` y crea `go.sum` con los checksums.

4.3 Estructura típica de un proyecto Go

```
mi-proyecto/
├── go.mod
├── go.sum
├── main.go
├── cmd/
│   └── server/
│       └── main.go
├── internal/
│   ├── api/
│   │   └── handler.go
│   └── db/
│       └── repository.go
├── pkg/
│   └── models/
│       └── user.go
├── tests/
│   └── integration_test.go
└── README.md
```

4.4 Comandos de módulos

```bash
# Añadir dependencia
go get github.com/gin-gonic/gin

# Actualizar todas las dependencias
go get -u ./...

# Limpiar dependencias no usadas
go mod tidy

# Verificar dependencias
go mod verify

# Mostrar dependencias
go list -m all

# Mostrar el gráfico de dependencias
go mod graph
```

===============================================================================
5. COMANDOS ESENCIALES DE GO
===============================================================================

5.1 Compilación y ejecución

```bash
# Compilar paquete actual
go build .

# Compilar y generar binario con nombre específico
go build -o mi-programa .

# Compilar y ejecutar
go run main.go

# Compilar varios archivos
go run *.go

# Compilar sin optimizaciones (más rápido para desarrollo)
go build -gcflags="-N -l" .
```

5.2 Formateo de código

```bash
# Formatear archivo
go fmt archivo.go

# Formatear todos los archivos del proyecto
go fmt ./...

# Ver qué archivos cambiarían (sin modificar)
gofmt -l .
```

5.3 Pruebas

```bash
# Ejecutar pruebas
go test .

# Con verbose
go test -v .

# Con cobertura
go test -cover .

# Ver cobertura en detalle
go test -coverprofile=coverage.out
go tool cover -html=coverage.out
```

5.4 Análisis de código

```bash
# Analizar código sospechoso
go vet ./...

# Con staticcheck (más exhaustivo)
staticcheck ./...
```

5.5 Documentación

```bash
# Ver documentación de un paquete
go doc fmt

# Ver documentación de una función
go doc fmt.Println

# Iniciar servidor web de documentación
godoc -http=:6060
# Abrir en navegador: http://localhost:6060
```

7. SOLUCIÓN DE PROBLEMAS COMUNES
===============================================================================

7.1 "go: command not found"
Causa: Go no está en el PATH.
Solución: Verifica que has añadido `/usr/local/go/bin` al PATH.
```bash
export PATH=$PATH:/usr/local/go/bin
```

7.2 "go: cannot find main module"
Causa: No has inicializado el módulo.
Solución: Ejecuta `go mod init nombre-del-modulo`.

7.3 "package main is not in GOROOT"
Causa: Estás dentro de GOPATH/src con Go Modules desactivado.
Solución:
```bash
export GO111MODULE=on
go mod init nombre
```

7.4 "gopls: command not found"
Causa: gopls no está instalado.
Solución:
```bash
go install golang.org/x/tools/gopls@latest
```

7.5 VS Code no autocompleta
Causa: gopls no está funcionando.
Solución:
1. Abre VS Code
2. Ctrl+Shift+P → "Go: Locate Configured Go Tools"
3. Verifica que gopls aparece
4. Si no, instala las herramientas recomendadas

7.6 Error de permisos al instalar
Causa: El directorio GOPATH/bin no tiene permisos de escritura.
Solución:
```bash
chmod -R 755 $GOPATH
```

9. GO EN LA NUBE (PLAYGROUND)
===============================================================================

Si no puedes instalar Go ahora mismo, puedes usar el Playground
oficial: https://play.golang.org

Características:
- Escribe y ejecuta Go en el navegador.
- Comparte código con enlaces permanentes.
- Incluye ejemplos de la biblioteca estándar.
- Soporta la mayoría de paquetes estándar (sin E/S de red ni archivos).
- Ideal para probar fragmentos de código rápidamente.

```go
package main

import (
    "fmt"
    "time"
)

func main() {
    fmt.Println("¡Hola desde el Playground de Go!")
    fmt.Println("La hora es:", time.Now())
}
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Necesito instalar gopls por separado?**
Normalmente VS Code lo instala automáticamente cuando
instalas la extensión Go. Si no, puedes hacerlo manualmente
con `go install golang.org/x/tools/gopls@latest`.

❓ **¿Qué es GO111MODULE?**
Es una variable de entorno que controla si se usan módulos
(desde Go 1.11). Los valores posibles: `off`, `on`, `auto`.
En `auto` (por defecto desde Go 1.16), los módulos se activan
cuando hay un archivo go.mod.

❓ **¿Debo usar GOPATH o Go Modules?**
Siempre usa Go Modules. GOPATH es el sistema antiguo y solo
se usa para proyectos heredados o casos muy específicos.
Desde Go 1.16, Go Modules está activado por defecto.

❓ **¿Puedo instalar Go sin permisos de administrador?**
Sí, puedes descargar el .tar.gz y extraerlo en cualquier
directorio donde tengas permisos de escritura. Luego
configura el PATH para que apunte a ese directorio.

❓ **¿Qué hago si `go build` produce un binario muy grande?**
Usa `go build -ldflags="-s -w"` para eliminar símbolos de
depuración y reducir el tamaño. Para aún más reducción,
puedes usar `upx` (compresor de binarios).

❓ **¿Cómo desinstalo Go?**
En Linux/macOS: `sudo rm -rf /usr/local/go` y elimina
las líneas de Go del archivo ~/.bashrc. En Windows:
ve a Agregar o quitar programas y desinstala Go.
