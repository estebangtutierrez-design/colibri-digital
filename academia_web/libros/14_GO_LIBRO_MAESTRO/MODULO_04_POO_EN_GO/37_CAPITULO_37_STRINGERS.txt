Bienvenido al capitulo 37. Imagina que creas un tipo Persona y quieres
imprimirlo con fmt.Println. Sin hacer nada especial, Go imprime algo
como {Lucas 30}. No muy bonito, verdad?

Para eso existe el metodo String(). Si tu tipo implementa el metodo
String() string, automagicamente fmt.Println, fmt.Sprintf, %s, %v
y cualquier funcion que use fmt usaran TU representacion.

El metodo String() es la puerta de entrada a la interfaz fmt.Stringer,
una de las interfaces mas utiles y usadas de la biblioteca estandar.

En este capitulo aprenderas a crear representaciones textuales
claras, informativas y hermosas para tus tipos.

--- 1. LA INTERFAZ fmt.Stringer ----------------------------------------------

La interfaz Stringer se define en el paquete fmt:

    type Stringer interface {
        String() string
    }

Cualquier tipo que implemente String() obtiene representacion textual
automatica en:

  * fmt.Println, fmt.Print, fmt.Printf con %s y %v
  * fmt.Sprintf
  * fmt.Fprint, fmt.Fprintf, fmt.Fprintln
  * log.Println, log.Printf
  * Y cualquier funcion que use fmt

--- 2. PRIMER EJEMPLO: STRING() BASICO ---------------------------------------

--------------------------------------------------------------------------------
package main

import "fmt"

type Persona struct {
    Nombre string
    Edad   int
}

func (p Persona) String() string {
    return fmt.Sprintf("%s (%d anios)", p.Nombre, p.Edad)
}

func main() {
    p := Persona{"Lucas", 30}
    fmt.Println(p)         // Lucas (30 anios)
    fmt.Printf("%s\n", p)  // Lucas (30 anios)
    fmt.Printf("%v\n", p)  // Lucas (30 anios)
}
--------------------------------------------------------------------------------

--- 3. STRING() CON FORMATO DETALLADO ----------------------------------------

Puedes decidir que informacion incluir y como presentarla:

--------------------------------------------------------------------------------
package main

import "fmt"

type Producto struct {
    ID       int
    Nombre   string
    Precio   float64
    Stock    int
    Categoria string
}

func (p Producto) String() string {
    estado := "En stock"
    if p.Stock == 0 {
        estado = "Agotado"
    } else if p.Stock < 10 {
        estado = "Stock bajo"
    }
    return fmt.Sprintf("[%d] %-20s $%7.2f (%s) - %s",
        p.ID, p.Nombre, p.Precio, estado, p.Categoria)
}

func main() {
    productos := []Producto{
        {1, "Laptop Gamer", 1299.99, 5, "Electronica"},
        {2, "Mouse Pad", 19.99, 0, "Accesorios"},
        {3, "Teclado Mecanico", 89.99, 50, "Perifericos"},
    }

    fmt.Println("Catalogo de productos:")
    fmt.Println("======================")
    for _, p := range productos {
        fmt.Println(p)
    }
}

--- 4. STRING() Y RECURSION INFINITA -----------------------------------------

CUIDADO: No uses fmt.Sprintf dentro de String() con el mismo formato
que llama a String() de nuevo. Causa recursion infinita.

MAL:

    func (p Persona) String() string {
        return fmt.Sprintf("%v", p) // RECURSION INFINITA!
    }

BIEN:

    func (p Persona) String() string {
        return fmt.Sprintf("%s (%d)", p.Nombre, p.Edad)
    }

Si necesitas usar %v sin recursion, convierte a tipo base:

    func (p Persona) String() string {
        return fmt.Sprintf("%v", struct {
            Nombre string
            Edad   int
        }{p.Nombre, p.Edad})
    }

--- 5. STRING() CON PUNTERO RECEIVER ----------------------------------------

Puedes usar puntero receiver en String():

--------------------------------------------------------------------------------
package main

import "fmt"

type Punto struct {
    X, Y float64
}

func (p *Punto) String() string {
    if p == nil {
        return "nil"
    }
    return fmt.Sprintf("(%.1f, %.1f)", p.X, p.Y)
}

func main() {
    p1 := &Punto{3.5, 4.2}
    fmt.Println(p1) // (3.5, 4.2)

    var p2 *Punto
    fmt.Println(p2) // nil
}

--- 6. %V VS %+V VS %#V ------------------------------------------------

Ademas de %v que usa String(), Go ofrece otros verbos:

  * %v  -> usa String() si existe
  * %+v -> incluye nombres de campos en structs
  * %#v -> representacion como codigo Go (sintaxis de literales)
  * %T  -> tipo del valor

--------------------------------------------------------------------------------
package main

import "fmt"

type Direccion struct {
    Calle  string
    Numero int
}

type Usuario struct {
    Nombre   string
    Edad     int
    Direccion
}

func (u Usuario) String() string {
    return fmt.Sprintf("%s (%d)", u.Nombre, u.Edad)
}

func main() {
    u := Usuario{
        Nombre: "Lucas",
        Edad:   30,
        Direccion: Direccion{
            Calle:  "Av. Siempre Viva",
            Numero: 742,
        },
    }

    fmt.Printf("String():  %v\n", u)
    fmt.Printf("%%+v:      %+v\n", u)
    fmt.Printf("%%#v:      %#v\n", u)
    fmt.Printf("%%T:       %T\n", u)
}

--- 8. STRINGER PARA ENUMS (CONSTANTES IOTA) ---------------------------------

String() es ideal para dar nombres a constantes numericas:

--------------------------------------------------------------------------------
package main

import "fmt"

type DiaSemana int

const (
    Lunes DiaSemana = iota
    Martes
    Miercoles
    Jueves
    Viernes
    Sabado
    Domingo
)

func (d DiaSemana) String() string {
    nombres := []string{
        "Lunes", "Martes", "Miercoles",
        "Jueves", "Viernes", "Sabado", "Domingo",
    }
    if d < Lunes || d > Domingo {
        return fmt.Sprintf("DiaSemana(%d)", int(d))
    }
    return nombres[d]
}

type Mes int

const (
    Enero Mes = iota + 1
    Febrero
    Marzo
    Abril
    Mayo
    Junio
    Julio
    Agosto
    Septiembre
    Octubre
    Noviembre
    Diciembre
)

func (m Mes) String() string {
    nombres := []string{
        "Enero", "Febrero", "Marzo", "Abril", "Mayo", "Junio",
        "Julio", "Agosto", "Septiembre", "Octubre", "Noviembre", "Diciembre",
    }
    if m < Enero || m > Diciembre {
        return fmt.Sprintf("Mes(%d)", int(m))
    }
    return nombres[m-1]
}

func main() {
    fmt.Println("Hoy es:", Miercoles)
    fmt.Println("Mes:", Julio)

    for d := Lunes; d <= Domingo; d++ {
        fmt.Print(d, " ")
    }
    fmt.Println()
}

--- 9. STRINGER PARA ERRORES PERSONALIZADOS ----------------------------------

Todo error en Go implementa la interfaz error (con Error() string).
Si ademas implementas String(), obtienes dos representaciones:

--------------------------------------------------------------------------------
package main

import "fmt"

type ValidationError struct {
    Campo   string
    Valor   any
    Motivo  string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("error de validacion: %s", e.Motivo)
}

func (e *ValidationError) String() string {
    return fmt.Sprintf("ValidationError{Campo:%s, Valor:%v, Motivo:%s}",
        e.Campo, e.Valor, e.Motivo)
}

func validarEdad(edad int) error {
    if edad < 0 || edad > 150 {
        return &ValidationError{
            Campo:  "edad",
            Valor:  edad,
            Motivo: "debe estar entre 0 y 150",
        }
    }
    return nil
}

func main() {
    err := validarEdad(200)
    if err != nil {
        fmt.Println("Error:", err)        // usa Error()
        fmt.Printf("Debug: %v\n", err)    // usa String()
        fmt.Printf("Detalle: %#v\n", err) // struct completo
    }
}

--- 10. STRING() PARA TIPOS COMPLEJOS ----------------------------------------

Para tipos con muchos campos, String() puede ser multi-linea:

--------------------------------------------------------------------------------
package main

import (
    "fmt"
    "strings"
)

type Servidor struct {
    Host      string
    Puerto    int
    Protocolo string
    TLS       bool
    Timeout   int
    MaxConns  int
    Status    string
}

func (s Servidor) String() string {
    var b strings.Builder
    b.WriteString(fmt.Sprintf("Servidor %s:%d [%s]\n", s.Host, s.Puerto, s.Protocolo))
    b.WriteString(fmt.Sprintf("  TLS: %v\n", s.TLS))
    b.WriteString(fmt.Sprintf("  Timeout: %ds\n", s.Timeout))
    b.WriteString(fmt.Sprintf("  Max Conexiones: %d\n", s.MaxConns))
    b.WriteString(fmt.Sprintf("  Status: %s\n", s.Status))
    return b.String()
}

func main() {
    svr := Servidor{
        Host:      "api.example.com",
        Puerto:    443,
        Protocolo: "HTTPS",
        TLS:       true,
        Timeout:   30,
        MaxConns:  1000,
        Status:    "Operativo",
    }

    fmt.Println(svr)
}

--- 11. STRINGER EN MAPAS Y SLICES -------------------------------------------

Puedes definir String() para tipos mapa y slice:

--------------------------------------------------------------------------------
package main

import (
    "fmt"
    "strings"
)

type Configuracion map[string]string

func (c Configuracion) String() string {
    if c == nil {
        return "Configuracion(nil)"
    }
    var b strings.Builder
    b.WriteString("Configuracion:\n")
    for k, v := range c {
        b.WriteString(fmt.Sprintf("  %s = %s\n", k, v))
    }
    return b.String()
}

type ListaCompras []string

func (l ListaCompras) String() string {
    if l == nil {
        return "ListaCompras(nil)"
    }
    if len(l) == 0 {
        return "ListaCompras(vacia)"
    }
    var b strings.Builder
    b.WriteString("Lista de Compras:\n")
    for i, item := range l {
        b.WriteString(fmt.Sprintf("  %d. %s\n", i+1, item))
    }
    return b.String()
}

func main() {
    cfg := Configuracion{
        "host": "localhost",
        "port": "8080",
        "env":  "development",
    }
    fmt.Println(cfg)

    lista := ListaCompras{"Leche", "Pan", "Huevos"}
    fmt.Println(lista)
}

--- 13. DIFERENCIA ENTRE ERROR() Y STRING() ----------------------------------

Un tipo puede implementar ambas interfaces (error y Stringer):

  * Error() string -> interfaz error (para manejo de errores)
  * String() string -> interfaz Stringer (para representacion)

Usa Error() para mensajes cortos de error.
Usa String() para representacion detallada/debug.

--- 14. STRING() Y JSON ------------------------------------------------------

String() NO afecta la serializacion JSON. json.Marshal usa los
tags json o los nombres de campos, no String().

Si quieres controlar la salida JSON, implementa json.Marshaler.

--- 15. STRING() EN PRUEBAS --------------------------------------------------

String() mejora los mensajes de error en pruebas:

--------------------------------------------------------------------------------
package main

import "fmt"

type Resultado struct {
    Operacion string
    Esperado  any
    Obtenido  any
}

func (r Resultado) String() string {
    return fmt.Sprintf("%s: esperado=%v, obtenido=%v",
        r.Operacion, r.Esperado, r.Obtenido)
}

func TestSuma() {
    resultados := []Resultado{
        {"suma(1,2)", 3, 1 + 2},
        {"suma(-1,1)", 0, -1 + 1},
        {"suma(0,0)", 0, 0 + 0},
        {"suma(100,200)", 300, 100 + 200},
    }

    for _, r := range resultados {
        if r.Esperado != r.Obtenido {
            fmt.Printf("FALLO: %s\n", r)
        } else {
            fmt.Printf("PASO: %s\n", r)
        }
    }
}

func main() {
    TestSuma()
}

--- 16. STRING() PARA ARBOLES Y ESTRUCTURAS ANIDADAS -------------------------

Un String() que muestre la jerarquia:

--------------------------------------------------------------------------------
package main

import (
    "fmt"
    "strings"
)

type Nodo struct {
    Valor    string
    Hijos    []*Nodo
}

func (n *Nodo) String() string {
    return n.stringArbol(0)
}

func (n *Nodo) stringArbol(nivel int) string {
    if n == nil {
        return ""
    }
    prefijo := strings.Repeat("  ", nivel)
    var b strings.Builder
    b.WriteString(fmt.Sprintf("%s- %s\n", prefijo, n.Valor))
    for _, hijo := range n.Hijos {
        b.WriteString(hijo.stringArbol(nivel + 1))
    }
    return b.String()
}

func main() {
    raiz := &Nodo{
        Valor: "Raiz",
        Hijos: []*Nodo{
            {Valor: "Hijo 1", Hijos: []*Nodo{
                {Valor: "Nieto 1.1"},
                {Valor: "Nieto 1.2"},
            }},
            {Valor: "Hijo 2"},
            {Valor: "Hijo 3", Hijos: []*Nodo{
                {Valor: "Nieto 3.1"},
            }},
        },
    }

    fmt.Println("Arbol:")
    fmt.Println(raiz)
}

--- 18. EJEMPLO COMPLETO: SISTEMA DE LOGS ------------------------------------

--------------------------------------------------------------------------------
package main

import (
    "fmt"
    "strings"
    "time"
)

type NivelLog int

const (
    DEBUG NivelLog = iota
    INFO
    WARN
    ERROR
    FATAL
)

func (n NivelLog) String() string {
    switch n {
    case DEBUG:
        return "DEBUG"
    case INFO:
        return "INFO"
    case WARN:
        return "WARN"
    case ERROR:
        return "ERROR"
    case FATAL:
        return "FATAL"
    default:
        return "UNKNOWN"
    }
}

type EntradaLog struct {
    Momento   time.Time
    Nivel     NivelLog
    Mensaje   string
    Modulo    string
    UsuarioID int
    Datos     map[string]string
}

func (e EntradaLog) String() string {
    var b strings.Builder
    b.WriteString(fmt.Sprintf("[%s] %-5s | %s | %s",
        e.Momento.Format("2006-01-02 15:04:05.000"),
        e.Nivel,
        e.Modulo,
        e.Mensaje,
    ))

    if e.UsuarioID != 0 {
        b.WriteString(fmt.Sprintf(" | user:%d", e.UsuarioID))
    }

    if len(e.Datos) > 0 {
        b.WriteString(" | {")
        first := true
        for k, v := range e.Datos {
            if !first {
                b.WriteString(", ")
            }
            b.WriteString(fmt.Sprintf("%s: %s", k, v))
            first = false
        }
        b.WriteString("}")
    }

    return b.String()
}

type Logger struct {
    entradas []EntradaLog
}

func (l *Logger) Log(nivel NivelLog, modulo, mensaje string, datos map[string]string) {
    entrada := EntradaLog{
        Momento: time.Now(),
        Nivel:   nivel,
        Mensaje: mensaje,
        Modulo:  modulo,
        Datos:   datos,
    }
    l.entradas = append(l.entradas, entrada)
    fmt.Println(entrada)
}

func (l *Logger) Resumen() string {
    var b strings.Builder
    b.WriteString("=== RESUMEN DE LOGS ===\n")
    for _, e := range l.entradas {
        b.WriteString(fmt.Sprintf("  %s\n", e))
    }
    b.WriteString("========================")
    return b.String()
}

func main() {
    log := &Logger{}

    log.Log(INFO, "main", "Servicio iniciado", map[string]string{
        "version": "1.0.0",
        "puerto":  "8080",
    })

    log.Log(WARN, "db", "Conexion lenta", map[string]string{
        "tiempo_ms": "3500",
    })

    log.Log(ERROR, "api", "Error al procesar pago", map[string]string{
        "codigo": "TIMEOUT",
        "pedido": "1234",
    })

    log.Log(INFO, "auth", "Usuario autenticado", map[string]string{
        "email": "lucas@example.com",
    })

    fmt.Println("\n" + log.Resumen())
}

--- 19. STRING() PARA SLICE DE BYTES -----------------------------------------

Si creas un tipo basado en []byte, puedes darle representacion:

--------------------------------------------------------------------------------
package main

import "fmt"

type HexBytes []byte

func (h HexBytes) String() string {
    if h == nil {
        return "nil"
    }
    return fmt.Sprintf("%x", []byte(h))
}

type HumanBytes int64

func (b HumanBytes) String() string {
    const unit = 1024
    if b < unit {
        return fmt.Sprintf("%d B", b)
    }
    div, exp := int64(unit), 0
    for n := b / unit; n >= unit; n /= unit {
        div *= unit
        exp++
    }
    unidades := []string{"B", "KB", "MB", "GB", "TB", "PB"}
    return fmt.Sprintf("%.2f %s", float64(b)/float64(div), unidades[exp])
}

func main() {
    datos := HexBytes{0x48, 0x65, 0x6C, 0x6C, 0x6F}
    fmt.Println("Hex:", datos)

    var size HumanBytes = 2 * 1024 * 1024 * 1024
    fmt.Println("Tamano:", size)

    fmt.Println("1KB:", HumanBytes(1024))
    fmt.Println("1.5MB:", HumanBytes(1.5*1024*1024))
}

--- 20. FORMATO GO CON %#V ---------------------------------------------------

El verbo %#v imprime la representacion Go del valor. Puedes
personalizarlo implementando GoStringer:

    type GoStringer interface {
        GoString() string
    }

--------------------------------------------------------------------------------
package main

import "fmt"

type Punto struct {
    X, Y int
}

func (p Punto) String() string {
    return fmt.Sprintf("(%d, %d)", p.X, p.Y)
}

func (p Punto) GoString() string {
    return fmt.Sprintf("Punto{X: %d, Y: %d}", p.X, p.Y)
}

func main() {
    p := Punto{3, 4}
    fmt.Printf("String:  %v\n", p)
    fmt.Printf("GoString: %#v\n", p)
}

--- 21. STRING() PARA CODIGOS DE ERROR HTTP ----------------------------------

--------------------------------------------------------------------------------
package main

import "fmt"

type HTTPStatus int

const (
    StatusOK                  HTTPStatus = 200
    StatusCreated             HTTPStatus = 201
    StatusBadRequest          HTTPStatus = 400
    StatusUnauthorized        HTTPStatus = 401
    StatusNotFound            HTTPStatus = 404
    StatusInternalServerError HTTPStatus = 500
    StatusServiceUnavailable  HTTPStatus = 503
)

func (s HTTPStatus) String() string {
    nombres := map[HTTPStatus]string{
        200: "OK",
        201: "Created",
        400: "Bad Request",
        401: "Unauthorized",
        404: "Not Found",
        500: "Internal Server Error",
        503: "Service Unavailable",
    }
    if nombre, ok := nombres[s]; ok {
        return fmt.Sprintf("%d %s", int(s), nombre)
    }
    return fmt.Sprintf("%d Unknown", int(s))
}

func main() {
    fmt.Println(StatusOK)
    fmt.Println(StatusNotFound)
    fmt.Println(StatusInternalServerError)
}

--- 23. STRING() COMPARANDO TIPOS --------------------------------------------

Cuando implementas String(), afecta como se muestran las
comparaciones en pruebas:

--------------------------------------------------------------------------------
package main

import (
    "fmt"
    "reflect"
)

type Punto struct {
    X, Y int
}

func (p Punto) String() string {
    return fmt.Sprintf("(%d, %d)", p.X, p.Y)
}

func main() {
    a := Punto{1, 2}
    b := Punto{1, 3}

    if !reflect.DeepEqual(a, b) {
        fmt.Printf("Diferencia: %v != %v\n", a, b)
    }
}

--------------------------------------------------------------------------------
  RESUMEN DEL CAPITULO
--------------------------------------------------------------------------------

  * String() string implementa la interfaz fmt.Stringer.
  * fmt.Println, %s, %v usan String() si existe.
  * Evita recursion infinita: no uses %v o %s en String().
  * String() puede ser multi-linea con strings.Builder.
  * %+v muestra campos con nombre; %#v muestra representacion Go.
  * String() mejora debugging, logging y pruebas.
  * Tambien existe GoStringer para %#v.
  * String() NO afecta JSON ni serializacion binaria.

--------------------------------------------------------------------------------
  FAQ - PREGUNTAS FRECUENTES
--------------------------------------------------------------------------------

P: String() afecta el rendimiento?
R: Si, porque se llama cada vez que imprimes. Para logging critico,
   usa formatos condicionales.

P: Que pasa si String() paniquea?
R: fmt.Println captura el panic y muestra un mensaje de error,
   pero deberias evitarlo.

P: String() se hereda con embedding?
R: Si. Si el tipo embebido tiene String(), se promociona. Si el
   contenedor tiene String(), oculta al embebido.

P: Como formateo numeros grandes en String()?
R: Usa fmt.Sprintf con verbos como %d, %f, o separadores manuales.

P: String() funciona con JSON?
R: No. JSON usa json.Marshaler, no Stringer.

P: Puedo tener String() con parametros?
R: No, String() debe ser exactamente func (T) String() string.

P: Como muestro un struct sin String()?
R: Con %+v para campos con nombre, %#v para sintaxis Go.

Sistema de Visualizacion de Datos:

Construye una libreria que tome datos estructurados y los
visualice en diferentes formatos usando String().

  Tipo base:
    type Visualizable interface {
        Encabezados() []string
        Filas() [][]string
    }

  Tipos de visualizacion (cada uno con String()):
    1. TablaSimple - columnas separadas por |
    2. TablaMarcada - con bordes +---+---+
    3. TablaMarkdown - formato para Markdown
    4. TablaCSV - valores separados por comas
    5. TablaHTML - tabla HTML basica

  type Reporte struct {
      Titulo     string
      Visualizable
  }

  Implementaciones de Visualizable:
    - VentasReport: Encabezados: Producto, Cantidad, Precio, Total
    - UsuariosReport: Encabezados: ID, Nombre, Email, Rol
    - InventarioReport: Encabezados: Codigo, Descripcion, Stock, Ubicacion

  Requisitos:
    1. Cada tipo de tabla tiene su propio String().
    2. Los datos se pasan como Visualizable.
    3. Las tablas manejan anchos variables de columna.
    4. Si una celda es muy larga, se trunca con "...".
    5. Reporte agrega el titulo antes de la tabla.

  Extra:
    - TablaHTML con colores alternados en filas.
    - TablaJSON que serialice a JSON.
    - Exportar a archivo con metodo Exportar(ruta string).

Este desafio integra Stringer, interfaces, formato y diseno
de presentacion de datos en Go.
