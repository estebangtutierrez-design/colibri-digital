  DESARROLLO DEL PROYECTO — WEB CRAWLER CONCURRENTE
══════════════════════════════════════════════════════════════════════════════

----------------------------------------------------------------------
  1. DESCRIPCIÓN DEL PROYECTO
----------------------------------------------------------------------

Construiremos un rastreador web concurrente que:

  1. Recibe una URL inicial y un número máximo de páginas a rastrear.
  2. Descarga páginas web concurrentemente (simulado con time.Sleep).
  3. Extrae enlaces de cada página.
  4. Rastrea los nuevos enlaces encontrados.
  5. Evita rastrear la misma URL dos veces.
  6. Genera un reporte con estadísticas y el mapa del sitio.
  7. Se detiene ordenadamente al alcanzar el límite o por timeout.

----------------------------------------------------------------------
  2. ARQUITECTURA DEL SISTEMA
----------------------------------------------------------------------

El sistema está compuesto por:

  • Fetcher: Simula la descarga de una página web.
  • Parser: Extrae enlaces del contenido HTML simulado.
  • Crawler: Coordina el rastreo usando un worker pool.
  • URLStore: Almacena URLs visitadas y pendientes (protegido con Mutex).
  • StatsCollector: Recolecta estadísticas (con Atomic).
  • Reporter: Genera el reporte final.

Flujo de datos:
  seedURL → [Crawler] → worker pool (fan-out)
    → fetcher → parser → extraer enlaces
    → agregar nuevas URLs a la cola
    → repetir hasta límite o cola vacía
    → generar reporte

----------------------------------------------------------------------
  3. COMPONENTES DEL SISTEMA
----------------------------------------------------------------------

COMPONENTE 1: URLStore
  • Almacena URLs visitadas y pendientes.
  • Protegido con sync.RWMutex.
  • Métodos: Add, IsVisited, Pendientes, Len.

COMPONENTE 2: Fetcher
  • Recibe una URL y devuelve el contenido simulado.
  • Tiempo de respuesta simulado entre 50-200ms.
  • Puede fallar aleatoriamente (10% de las veces).
  • Usa context para soportar cancelación.

COMPONENTE 3: Parser
  • Recibe contenido HTML simulado.
  • Extrae enlaces (hrefs) usando expresiones regulares.
  • Convierte URLs relativas a absolutas.
  • Filtra enlaces externos (solo mismo dominio).

COMPONENTE 4: Worker (Crawler Worker)
  • Recibe una URL del canal de tareas.
  • Llama a Fetcher para descargar.
  • Llama a Parser para extraer enlaces.
  • Envía nuevos enlaces al canal de descubrimiento.
  • Reporta estadísticas (URLs procesadas, errores, tiempo).

COMPONENTE 5: StatsCollector
  • Contadores atómicos: pagesFetched, pagesFailed, totalLinks, totalTime.
  • Métodos: Increment, Snapshot.

COMPONENTE 6: Reporter
  • Recibe el snapshot de estadísticas.
  • Genera reporte formateado con:
    - URLs visitadas y pendientes
    - Páginas exitosas y fallidas
    - Total enlaces encontrados
    - Tiempo promedio por página
    - Throughput (páginas/segundo)
    - Mapa del sitio (árbol de URLs)

package main

import (
    "context"
    "fmt"
    "math/rand"
    "regexp"
    "strings"
    "sync"
    "sync/atomic"
    "time"
)

// ============================================================================
// URLStore: Almacén seguro de URLs
// ============================================================================

type URLStore struct {
    mu        sync.RWMutex
    visitados map[string]bool
    pendientes []string
    limite    int
}

func NewURLStore(seedURL string, limite int) *URLStore {
    return &URLStore{
        visitados:  make(map[string]bool),
        pendientes: []string{seedURL},
        limite:     limite,
    }
}

func (s *URLStore) Siguiente() (string, bool) {
    s.mu.Lock()
    defer s.mu.Unlock()

    if len(s.pendientes) == 0 {
        return "", false
    }

    url := s.pendientes[0]
    s.pendientes = s.pendientes[1:]
    s.visitados[url] = true
    return url, true
}

func (s *URLStore) Agregar(urls []string) {
    s.mu.Lock()
    defer s.mu.Unlock()

    for _, url := range urls {
        if !s.visitados[url] && !s.contienePendiente(url) {
            if len(s.visitados)+len(s.pendientes) < s.limite {
                s.pendientes = append(s.pendientes, url)
            }
        }
    }
}

func (s *URLStore) contienePendiente(url string) bool {
    for _, p := range s.pendientes {
        if p == url {
            return true
        }
    }
    return false
}

func (s *URLStore) Visitados() int {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return len(s.visitados)
}

func (s *URLStore) PendientesCount() int {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return len(s.pendientes)
}

func (s *URLStore) Total() int {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return len(s.visitados) + len(s.pendientes)
}

func (s *URLStore) ListaVisitados() []string {
    s.mu.RLock()
    defer s.mu.RUnlock()
    urls := make([]string, 0, len(s.visitados))
    for url := range s.visitados {
        urls = append(urls, url)
    }
    return urls
}

// ============================================================================
// StatsCollector: Estadísticas con Atomic
// ============================================================================

type StatsCollector struct {
    pagesFetched int64
    pagesFailed  int64
    totalLinks   int64
    totalTimeNs  int64
}

func (s *StatsCollector) PageFetched(duracion time.Duration) {
    atomic.AddInt64(&s.pagesFetched, 1)
    atomic.AddInt64(&s.totalTimeNs, int64(duracion))
}

func (s *StatsCollector) PageFailed() {
    atomic.AddInt64(&s.pagesFailed, 1)
}

func (s *StatsCollector) LinksFound(count int) {
    atomic.AddInt64(&s.totalLinks, int64(count))
}

func (s *StatsCollector) Snapshot() StatsSnapshot {
    return StatsSnapshot{
        PagesFetched: atomic.LoadInt64(&s.pagesFetched),
        PagesFailed:  atomic.LoadInt64(&s.pagesFailed),
        TotalLinks:   atomic.LoadInt64(&s.totalLinks),
        TotalTimeNs:  atomic.LoadInt64(&s.totalTimeNs),
    }
}

type StatsSnapshot struct {
    PagesFetched int64
    PagesFailed  int64
    TotalLinks   int64
    TotalTimeNs  int64
}

func (s StatsSnapshot) TiempoPromedio() time.Duration {
    if s.PagesFetched == 0 {
        return 0
    }
    return time.Duration(s.TotalTimeNs / s.PagesFetched)
}

func (s StatsSnapshot) TasaError() float64 {
    total := s.PagesFetched + s.PagesFailed
    if total == 0 {
        return 0
    }
    return float64(s.PagesFailed) / float64(total) * 100
}

// ============================================================================
// Fetcher: Descarga simulada de páginas
// ============================================================================

type Fetcher struct {
    dominio string
    client  *http.Client // En un crawler real, usaríamos http
}

func NewFetcher(dominio string) *Fetcher {
    return &Fetcher{
        dominio: dominio,
    }
}

func (f *Fetcher) Fetch(ctx context.Context, url string) (string, error) {
    // Simular latencia de red
    select {
    case <-time.After(time.Duration(50+rand.Intn(150)) * time.Millisecond):
    case <-ctx.Done():
        return "", ctx.Err()
    }

    // Simular error aleatorio (10%)
    if rand.Float64() < 0.1 {
        return "", fmt.Errorf("error simulado al descargar %s", url)
    }

    // Generar HTML simulado con enlaces
    html := f.generarHTML(url)
    return html, nil
}

func (f *Fetcher) generarHTML(url string) string {
    var links []string
    numLinks := rand.Intn(5) + 1

    for i := 0; i < numLinks; i++ {
        path := fmt.Sprintf("/pagina-%d-%d.html", rand.Intn(100), i)
        links = append(links, f.dominio+path)
    }

    var sb strings.Builder
    sb.WriteString("<html><body>\n")
    sb.WriteString(fmt.Sprintf("<h1>Página: %s</h1>\n", url))
    for _, link := range links {
        sb.WriteString(fmt.Sprintf("<a href=\"%s\">Enlace</a>\n", link))
    }
    sb.WriteString("<p>Contenido simulado de página web.</p>\n")
    sb.WriteString("</body></html>")
    return sb.String()
}

// ============================================================================
// Parser: Extracción de enlaces
// ============================================================================

var linkRegex = regexp.MustCompile(`<a\s+(?:[^>]*?\s+)?href="([^"]*)"`)

func ParseLinks(html, baseURL string) []string {
    matches := linkRegex.FindAllStringSubmatch(html, -1)

    var links []string
    seen := make(map[string]bool)

    for _, match := range matches {
        if len(match) < 2 {
            continue
        }
        link := match[1]

        // Resolver URLs relativas
        if strings.HasPrefix(link, "/") {
            link = baseURL + link
        }

        // Solo URLs del mismo dominio
        if strings.HasPrefix(link, baseURL) {
            if !seen[link] {
                links = append(links, link)
                seen[link] = true
            }
        }
    }

    return links
}

// ============================================================================
// Reporter: Generación de reportes
// ============================================================================

type Reporter struct {
    inicio time.Time
}

func NewReporter() *Reporter {
    return &Reporter{inicio: time.Now()}
}

func (r *Reporter) Generar(stats StatsSnapshot, urls []string) {
    duracion := time.Since(r.inicio)
    pagesTotal := stats.PagesFetched + stats.PagesFailed
    throughput := float64(pagesTotal) / duracion.Seconds()

    fmt.Println("\n" + strings.Repeat("=", 60))
    fmt.Println("  REPORTE DEL WEB CRAWLER")
    fmt.Println(strings.Repeat("=", 60))
    fmt.Printf("  Duración:                %v\n", duracion.Round(time.Millisecond))
    fmt.Printf("  URLs visitadas:          %d\n", len(urls))
    fmt.Printf("  Páginas exitosas:        %d\n", stats.PagesFetched)
    fmt.Printf("  Páginas fallidas:        %d\n", stats.PagesFailed)
    fmt.Printf("  Tasa de error:           %.2f%%\n", stats.TasaError())
    fmt.Printf("  Total enlaces:           %d\n", stats.TotalLinks)
    fmt.Printf("  Tiempo promedio/página:  %v\n", stats.TiempoPromedio().Round(time.Millisecond))
    fmt.Printf("  Throughput:              %.1f páginas/segundo\n", throughput)
    fmt.Println(strings.Repeat("-", 60))
    fmt.Println("  MAPA DEL SITIO:")
    for i, url := range urls {
        fmt.Printf("  %3d. %s\n", i+1, url)
    }
    fmt.Println(strings.Repeat("=", 60))
}

// ============================================================================
// Monitor: Estadísticas en tiempo real
// ============================================================================

type Monitor struct {
    stats     *StatsCollector
    store     *URLStore
    ticker    *time.Ticker
    done      chan struct{}
}

func NewMonitor(stats *StatsCollector, store *URLStore, intervalo time.Duration) *Monitor {
    return &Monitor{
        stats:  stats,
        store:  store,
        ticker: time.NewTicker(intervalo),
        done:   make(chan struct{}),
    }
}

func (m *Monitor) Start() {
    go func() {
        for {
            select {
            case <-m.ticker.C:
                m.mostrar()
            case <-m.done:
                m.ticker.Stop()
                return
            }
        }
    }()
}

func (m *Monitor) Stop() {
    close(m.done)
}

func (m *Monitor) mostrar() {
    stats := m.stats.Snapshot()
    fmt.Printf("[MONITOR] Visitadas: %d | Pendientes: %d | Éxito: %d | Fallo: %d | Enlaces: %d\n",
        m.store.Visitados(),
        m.store.PendientesCount(),
        stats.PagesFetched,
        stats.PagesFailed,
        stats.TotalLinks)
}

// ============================================================================
// Crawler: Coordinador principal
// ============================================================================

type Crawler struct {
    store    *URLStore
    fetcher  *Fetcher
    stats    *StatsCollector
    monitor  *Monitor
    reporter *Reporter
    numWorkers int
    limite     int
    timeout    time.Duration
}

func NewCrawler(seedURL string, limite, numWorkers int, timeout time.Duration) *Crawler {
    dominio := extraerDominio(seedURL)
    store := NewURLStore(seedURL, limite)
    stats := &StatsCollector{}
    fetcher := NewFetcher(dominio)
    monitor := NewMonitor(stats, store, 1*time.Second)
    reporter := NewReporter()

    return &Crawler{
        store:      store,
        fetcher:    fetcher,
        stats:      stats,
        monitor:    monitor,
        reporter:   reporter,
        numWorkers: numWorkers,
        limite:     limite,
        timeout:    timeout,
    }
}

func extraerDominio(url string) string {
    // Simulación: extraemos la base
    parts := strings.Split(url, "/")
    if len(parts) >= 3 {
        return parts[0] + "//" + parts[2]
    }
    return url
}

func (c *Crawler) Start(ctx context.Context) {
    fmt.Println("=== INICIANDO WEB CRAWLER ===")
    fmt.Printf("URL inicial: %s\n", c.store.pendientes[0])
    fmt.Printf("Workers: %d\n", c.numWorkers)
    fmt.Printf("Límite: %d páginas\n", c.limite)
    fmt.Printf("Timeout: %v\n", c.timeout)
    fmt.Println()

    c.monitor.Start()
    defer c.monitor.Stop()

    // Canales
    tareas := make(chan string, c.numWorkers*10)
    descubrimientos := make(chan []string, c.numWorkers*10)

    var wg sync.WaitGroup

    // Fan-Out: Workers
    for i := 0; i < c.numWorkers; i++ {
        wg.Add(1)
        go c.worker(ctx, i, tareas, descubrimientos, &wg)
    }

    // Gorutina que alimenta el canal de tareas
    go func() {
        defer close(tareas)
        for {
            url, ok := c.store.Siguiente()
            if !ok {
                return
            }
            select {
            case tareas <- url:
            case <-ctx.Done():
                return
            }
        }
    }()

    // Gorutina que procesa descubrimientos
    go func() {
        for {
            select {
            case links := <-descubrimientos:
                c.store.Agregar(links)
            case <-ctx.Done():
                return
            }
        }
    }()

    // Esperar workers o timeout
    done := make(chan struct{})
    go func() {
        wg.Wait()
        close(descubrimientos)
        close(done)
    }()

    select {
    case <-done:
        fmt.Println("\nCrawler completado: todas las URLs procesadas")
    case <-ctx.Done():
        fmt.Println("\nCrawler cancelado por timeout o señal externa")
    }

    // Pequeña pausa para que se procesen los últimos descubrimientos
    time.Sleep(100 * time.Millisecond)

    // Generar reporte
    stats := c.stats.Snapshot()
    urls := c.store.ListaVisitados()
    c.reporter.Generar(stats, urls)
}

func (c *Crawler) worker(ctx context.Context, id int, tareas <-chan string, descubrimientos chan<- []string, wg *sync.WaitGroup) {
    defer wg.Done()

    for url := range tareas {
        select {
        case <-ctx.Done():
            return
        default:
        }

        inicio := time.Now()
        fmt.Printf("[Worker %d] Descargando: %s\n", id, url)

        // Fetch
        html, err := c.fetcher.Fetch(ctx, url)
        if err != nil {
            c.stats.PageFailed()
            fmt.Printf("[Worker %d] ERROR: %v\n", id, err)
            continue
        }

        c.stats.PageFetched(time.Since(inicio))

        // Parsear enlaces
        dominio := extraerDominio(url)
        links := ParseLinks(html, dominio)
        c.stats.LinksFound(len(links))

        fmt.Printf("[Worker %d] OK: %s (%d enlaces)\n", id, url, len(links))

        // Enviar descubrimientos
        select {
        case descubrimientos <- links:
        case <-ctx.Done():
            return
        }
    }
}

func main() {
    rand.Seed(time.Now().UnixNano())

    // Configuración
    seedURL := "https://ejemplo.com"
    limite := 50
    numWorkers := 5
    timeout := 30 * time.Second

    // Context con timeout
    ctx, cancel := context.WithTimeout(context.Background(), timeout)
    defer cancel()

    // Crear y ejecutar crawler
    crawler := NewCrawler(seedURL, limite, numWorkers, timeout)
    crawler.Start(ctx)

    fmt.Println("Proyecto Web Crawler completado exitosamente")
}

══════════════════════════════════════════════════════════════════════════════
  EJERCICIOS DE EXTENSIÓN DEL PROYECTO
══════════════════════════════════════════════════════════════════════════════

1. AÑADIR PROFUNDIDAD — Añade un límite de profundidad al rastreo. La URL
   inicial tiene profundidad 0. Cada enlace extraído incrementa en 1.

2. PERSISTENCIA — Guarda el mapa del sitio en un archivo JSON al finalizar
   el rastreo.

3. REPORTE EN VIVO — Mejora el monitor para que muestre una barra de
   progreso y ETA (tiempo estimado de finalización).

4. PRIORIDAD — Implementa URLs con prioridad. Las URLs con más enlaces
   entrantes tienen mayor prioridad.

5. ROBOTS.TXT — Simula un archivo robots.txt que impide rastrear ciertas
   rutas. Respeta las exclusiones.

6. ALMACENAMIENTO — Almacena el contenido de cada página visitada en
   archivos individuales en disco.

7. ESTADÍSTICAS AVANZADAS — Añade: páginas más rápidas/lentas, histograma
   de tiempos, distribución de profundidad.

8. RECOLECTOR DE BASURA — Implementa un mecanismo que descarte URLs de
   la cola si llevan demasiado tiempo esperando.

9. WEB REAL — Reemplaza el Fetcher simulado por uno real usando el
   paquete net/http. (Cuidado: respeta el archivo robots.txt).

10. INTERFAZ WEB — Crea una interfaz web simple usando net/http que
    muestre el progreso del crawler en tiempo real.

══════════════════════════════════════════════════════════════════════════════
  FAQ — PREGUNTAS FRECUENTES DEL PROYECTO
══════════════════════════════════════════════════════════════════════════════

❓ ¿Por qué usamos RWMutex en URLStore en lugar de Mutex?
R: Porque las lecturas (Visitados, PendientesCount) son más frecuentes que
las escrituras (Agregar, Siguiente). RWMutex permite múltiples lectores
simultáneos.

❓ ¿Por qué los canales tienen buffer?
R: Para desacoplar a productores y consumidores. Si un worker es lento,
los demás pueden seguir enviando tareas al buffer sin bloquearse.

❓ ¿Cómo se evita rastrear la misma URL dos veces?
R: URLStore mantiene un mapa de URLs visitadas. Antes de agregar una URL,
se verifica que no esté en visitados ni en pendientes.

❓ ¿Qué pasa si todos los workers fallan?
R: Los workers individuales pueden fallar, pero el worker pool sigue
funcionando. Si todas las URLs fallan, no se agregan nuevas URLs y el
crawler termina naturalmente.

❓ ¿Cómo se implementa el timeout?
R: Usamos context.WithTimeout. Cuando el contexto expira, ctx.Done() se
cierra, y todas las gorutinas que usan select detectan la cancelación.

❓ ¿Por qué hay una gorutina separada para procesar descubrimientos?
R: Para evitar que los workers se bloqueen al agregar URLs al store.
El envío al canal de descubrimientos es no bloqueante (gracias al buffer).

❓ ¿El sistema maneja correctamente el shutdown?
R: Sí. Cuando se alcanza el límite o el timeout, se cierran los canales,
los workers terminan, y se espera a todos con WaitGroup.

❓ ¿Cuántas gorutinas se crean en total?
R: N workers + 1 monitor + 1 alimentador + 1 procesador descubrimientos
+ 1 gorutina de espera = N+4 gorutinas aproximadamente.

❓ ¿Puedo usar este crawler en producción?
R: El Fetcher es simulado. Para uso real, reemplázalo con net/http y
añade respeto a robots.txt, rate limiting, y manejo de redirecciones.

❓ ¿Cómo puedo verificar que no hay fugas de gorutinas?
R: Usa runtime.NumGoroutine() antes y después de la ejecución. También
puedes usar el endpoint /debug/pprof/goroutine de net/http/pprof.
