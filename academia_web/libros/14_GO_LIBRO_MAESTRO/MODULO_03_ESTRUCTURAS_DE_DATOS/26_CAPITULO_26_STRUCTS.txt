  1. ¿Qué es un Struct?
  ─────────────────────

  Un struct es una colección de campos (fields) con nombre y tipo. Es como
  una "clase" ligera sin métodos (aunque los métodos se pueden añadir por
  separado). Se define con la palabra clave type:

      type Persona struct {
          Nombre string
          Edad   int
      }

  2. Declaración e Inicialización
  ───────────────────────────────

  ================================================================================
  EJEMPLO 1: Declaración básica de struct
  ================================================================================

  package main

  import "fmt"

  type Persona struct {
      Nombre string
      Edad   int
  }

  func main() {
      // Inicialización con campos en orden
      p1 := Persona{"Alice", 30}
      fmt.Println("p1:", p1)

      // Inicialización con campos nombrados (recomendado)
      p2 := Persona{Nombre: "Bob", Edad: 25}
      fmt.Println("p2:", p2)

      // Inicialización parcial (el resto con valor cero)
      p3 := Persona{Nombre: "Charlie"}
      fmt.Println("p3:", p3)

      // Struct vacío (todos los campos a cero)
      p4 := Persona{}
      fmt.Println("p4:", p4)

      // Declaración e inicialización posterior
      var p5 Persona
      p5.Nombre = "Diana"
      p5.Edad = 28
      fmt.Println("p5:", p5)
  }

  ================================================================================
  EJEMPLO 2: Acceso a campos
  ================================================================================

  package main

  import "fmt"

  type Libro struct {
      Titulo  string
      Autor   string
      Paginas int
  }

  func main() {
      libro := Libro{
          Titulo:  "Go Master Book",
          Autor:   "Lucas Ega",
          Paginas: 9999,
      }

      fmt.Println("Título:", libro.Titulo)
      fmt.Println("Autor:", libro.Autor)
      fmt.Println("Páginas:", libro.Paginas)

      // Modificar campos
      libro.Paginas = 10000
      fmt.Println("Páginas actualizadas:", libro.Paginas)
  }

  3. Struct Pass por Valor vs Puntero
  ────────────────────────────────────

  ================================================================================
  EJEMPLO 3: Struct por valor (copia)
  ================================================================================

  package main

  import "fmt"

  type Punto struct {
      X, Y int
  }

  func mover(p Punto, dx, dy int) Punto {
      p.X += dx
      p.Y += dy
      return p
  }

  func main() {
      p := Punto{1, 2}
      resultado := mover(p, 10, 20)

      fmt.Println("Original:", p)          // {1, 2}
      fmt.Println("Resultado:", resultado) // {11, 22}
  }

  ================================================================================
  EJEMPLO 4: Struct por puntero (eficiente y modificable)
  ================================================================================

  package main

  import "fmt"

  type Punto struct {
      X, Y int
  }

  func mover(p *Punto, dx, dy int) {
      p.X += dx
      p.Y += dy
  }

  func main() {
      p := Punto{1, 2}
      mover(&p, 10, 20)

      fmt.Println("Modificado:", p) // {11, 22}
  }

  4. Métodos sobre Structs
  ────────────────────────

  ================================================================================
  EJEMPLO 5: Métodos con valor
  ================================================================================

  package main

  import "fmt"

  type Rectangulo struct {
      Ancho, Alto float64
  }

  func (r Rectangulo) Area() float64 {
      return r.Ancho * r.Alto
  }

  func (r Rectangulo) Perimetro() float64 {
      return 2 * (r.Ancho + r.Alto)
  }

  func main() {
      rect := Rectangulo{10, 5}
      fmt.Println("Área:", rect.Area())
      fmt.Println("Perímetro:", rect.Perimetro())
  }

  ================================================================================
  EJEMPLO 6: Métodos con puntero (modifican el struct)
  ================================================================================

  package main

  import "fmt"

  type Contador struct {
      Valor int
  }

  func (c *Contador) Incrementar() {
      c.Valor++
  }

  func (c *Contador) Reset() {
      c.Valor = 0
  }

  func main() {
      c := Contador{}
      c.Incrementar()
      c.Incrementar()
      c.Incrementar()
      fmt.Println("Valor:", c.Valor) // 3

      c.Reset()
      fmt.Println("Después de reset:", c.Valor) // 0
  }

  5. Struct Embedding (Composición)
  ────────────────────────────────

  Go no tiene herencia. En su lugar, usa COMPOSICIÓN mediante embedding:
  incluir un struct anónimo dentro de otro. Los campos y métodos del struct
  embebido se "promueven" al struct contenedor.

  ================================================================================
  EJEMPLO 7: Embedding básico
  ================================================================================

  package main

  import "fmt"

  type Direccion struct {
      Calle string
      Ciudad string
      Pais   string
  }

  type Persona struct {
      Nombre string
      Edad   int
      Direccion // Embedding (campo anónimo)
  }

  func main() {
      p := Persona{
          Nombre: "Alice",
          Edad:   30,
          Direccion: Direccion{
              Calle:  "Av. Siempre Viva 123",
              Ciudad: "Springfield",
              Pais:   "USA",
          },
      }

      // Acceso directo a campos promovidos
      fmt.Println("Nombre:", p.Nombre)
      fmt.Println("Calle:", p.Calle)   // Promovido desde Direccion
      fmt.Println("Ciudad:", p.Ciudad) // Promovido
      fmt.Println("País:", p.Pais)     // Promovido

      // También puedes acceder explícitamente
      fmt.Println("Dirección completa:", p.Direccion)
  }

  ================================================================================
  EJEMPLO 8: Embedding con colisión de campos
  ================================================================================

  package main

  import "fmt"

  type A struct {
      Valor int
  }

  type B struct {
      Valor int
  }

  type C struct {
      A
      B
      Valor int // Campo propio
  }

  func main() {
      c := C{
          A:     A{Valor: 1},
          B:     B{Valor: 2},
          Valor: 3,
      }

      fmt.Println("c.Valor:", c.Valor)   // 3 (propio)
      fmt.Println("c.A.Valor:", c.A.Valor) // 1
      fmt.Println("c.B.Valor:", c.B.Valor) // 2
  }

  6. Structs con Tags
  ───────────────────

  Las tags son metadatos en forma de string que acompañan a los campos. Se usan
  para serialización (JSON, XML), validación, ORMs, etc.

  ================================================================================
  EJEMPLO 9: Tags para JSON
  ================================================================================

  package main

  import (
      "fmt"
      "encoding/json"
  )

  type Usuario struct {
      ID        int    `json:"id,omitempty"`
      Nombre    string `json:"nombre"`
      Email     string `json:"email,omitempty"`
      Password  string `json:"-"`
      Activo    bool   `json:"activo"`
  }

  func main() {
      u := Usuario{
          ID:       1,
          Nombre:   "Alice",
          Email:    "alice@example.com",
          Password: "secreto123",
          Activo:   true,
      }

      jsonBytes, _ := json.MarshalIndent(u, "", "  ")
      fmt.Println(string(jsonBytes))
      // Password no aparece (tag "-")
      // Email aparece (omitempty no aplica porque hay valor)
  }

  ================================================================================
  EJEMPLO 10: omitempty en acción
  ================================================================================

  package main

  import (
      "fmt"
      "encoding/json"
  )

  type Config struct {
      Host string `json:"host"`
      Port int    `json:"port,omitempty"`
      Debug bool  `json:"debug,omitempty"`
  }

  func main() {
      c1 := Config{Host: "localhost", Port: 8080, Debug: true}
      c2 := Config{Host: "localhost"} // Port=0, Debug=false

      j1, _ := json.Marshal(c1)
      j2, _ := json.Marshal(c2)

      fmt.Println("c1:", string(j1))
      fmt.Println("c2:", string(j2)) // Port y Debug omitidos
  }

  7. Structs Anónimos
  ───────────────────

  Puedes declarar un struct sin nombre de tipo (inline). Útil para datos
  temporales o one-off.

  ================================================================================
  EJEMPLO 11: Struct anónimo
  ================================================================================

  package main

  import "fmt"

  func main() {
      // Struct anónimo
      persona := struct {
          Nombre string
          Edad   int
      }{
          Nombre: "Bob",
          Edad:   25,
      }

      fmt.Println("Persona:", persona)

      // Slice de structs anónimos
      personas := []struct {
          Nombre string
          Edad   int
      }{
          {"Alice", 30},
          {"Charlie", 35},
      }

      fmt.Println("Personas:", personas)
  }

  ================================================================================
  EJEMPLO 12: Struct anónimo para JSON temporal
  ================================================================================

  package main

  import (
      "fmt"
      "encoding/json"
  )

  func main() {
      data := struct {
          Nombre  string `json:"nombre"`
          Version int    `json:"version"`
      }{
          Nombre:  "API v2",
          Version: 2,
      }

      jsonBytes, _ := json.Marshal(data)
      fmt.Println(string(jsonBytes))
  }

  8. Struct Constructores (Factory Functions)
  ──────────────────────────────────────────

  ================================================================================
  EJEMPLO 13: Función constructora
  ================================================================================

  package main

  import "fmt"

  type Cuenta struct {
      Titular  string
      Saldo    float64
      activa   bool // campo privado (minúscula)
  }

  func NuevaCuenta(titular string, depositoInicial float64) *Cuenta {
      if depositoInicial < 0 {
          depositoInicial = 0
      }
      return &Cuenta{
          Titular: titular,
          Saldo:   depositoInicial,
          activa:  true,
      }
  }

  func main() {
      c := NuevaCuenta("Alice", 1000)
      fmt.Printf("Cuenta: %+v\n", c)
  }

      fmt.Println("Precio:", p.Precio)
      fmt.Println("Código:", p.codigo)        // Accesible mismo paquete
      fmt.Println("Inventario:", p.inventario) // Accesible mismo paquete
  }

  11. Struct Inicialización con new
  ────────────────────────────────

  ================================================================================
  EJEMPLO 16: new vs literal
  ================================================================================

  package main

  import "fmt"

  type Punto struct {
      X, Y int
  }

  func main() {
      p1 := new(Punto)  // *Punto (puntero) con campos a cero
      p2 := &Punto{}    // *Punto (puntero) equivalente
      p3 := Punto{}     // Punto (valor)

      fmt.Printf("p1: %#v\n", p1)
      fmt.Printf("p2: %#v\n", p2)
      fmt.Printf("p3: %#v\n", p3)

      p1.X = 10
      p2.Y = 20
      fmt.Println("p1:", p1)
  }

  12. Structs con Métodos y Campos Privados
  ─────────────────────────────────────────

  ================================================================================
  EJEMPLO 17: Getters y Setters
  ================================================================================

  package main

  import "fmt"

  type Temperatura struct {
      celsius float64 // privado
  }

  func NewTemperatura(c float64) Temperatura {
      return Temperatura{celsius: c}
  }

  func (t Temperatura) Celsius() float64 {
      return t.celsius
  }

  func (t Temperatura) Fahrenheit() float64 {
      return t.celsius*9/5 + 32
  }

  func (t *Temperatura) SetCelsius(c float64) {
      t.celsius = c
  }

  func main() {
      t := NewTemperatura(25)
      fmt.Printf("%.1f°C = %.1f°F\n", t.Celsius(), t.Fahrenheit())

      t.SetCelsius(30)
      fmt.Printf("%.1f°C = %.1f°F\n", t.Celsius(), t.Fahrenheit())
  }

  13. Struct Recursivo

  ================================================================================
  EJEMPLO 18: Lista enlazada con struct
  ================================================================================

  package main

  import "fmt"

  type Nodo struct {
      Valor int
      Siguiente *Nodo
  }

  func main() {
      n1 := &Nodo{Valor: 1}
      n2 := &Nodo{Valor: 2}
      n3 := &Nodo{Valor: 3}

      n1.Siguiente = n2
      n2.Siguiente = n3

      // Recorrer la lista
      for n := n1; n != nil; n = n.Siguiente {
          fmt.Println(n.Valor)
      }
  }

  15. Struct Tags para Validación
  ──────────────────────────────

  ================================================================================
  EJEMPLO 20: Tags personalizados
  ================================================================================

  package main

  import (
      "fmt"
      "reflect"
  )

  type Usuario struct {
      Nombre string `validacion:"requerido"`
      Edad   int    `validacion:"minimo=18"`
      Email  string `validacion:"email"`
  }

  func main() {
      u := Usuario{}
      t := reflect.TypeOf(u)

      for i := 0; i < t.NumField(); i++ {
          campo := t.Field(i)
          fmt.Printf("%s: tag = %s\n", campo.Name, campo.Tag.Get("validacion"))
      }
  }

  20. Struct Copy (Shallow)
  ─────────────────────────

  ================================================================================
  EJEMPLO 25: Copia superficial
  ================================================================================

  package main

  import "fmt"

  type Direccion struct {
      Calle string
  }

  type Persona struct {
      Nombre   string
      Direccion *Direccion
  }

  func main() {
      dir := &Direccion{Calle: "Av. Principal"}
      p1 := Persona{Nombre: "Alice", Direccion: dir}
      p2 := p1 // Copia superficial

      p2.Nombre = "Bob"
      p2.Direccion.Calle = "Calle Secundaria"

      fmt.Println("p1.Nombre:", p1.Nombre)     // Alice
      fmt.Println("p1.Direccion:", p1.Direccion.Calle) // Calle Secundaria
  }

  ================================================================================
  RESUMEN DEL CAPÍTULO
  ================================================================================

  • type Nombre struct { campos } define un struct.
  • Inicialización: Nombre{campo: valor} o Nombre{valor1, valor2}.
  • Los campos se acceden con ".", ej: p.Nombre.
  • Los structs se pasan POR VALOR (copia). Usa *Struct para referencia.
  • Los métodos se definen fuera del struct con func (r Tipo) Metodo().
  • Tags: metadatos entre `backticks` para JSON, validación, etc.
  • Embedding: composición mediante campos anónimos (herencia simulada).
  • Los campos exportados empiezan con mayúscula; privados con minúscula.
  • Structs anónimos: struct { ... } {...} para uso temporal.
  • Zero value: todos los campos con valor cero de su tipo.
  • Los structs son comparables con == si todos sus campos lo son.

  ================================================================================
  PREGUNTAS FRECUENTES (FAQ)
  ================================================================================

  ❓ ¿Struct o map? ¿Cuándo usar cuál?
  → Struct para datos con campos fijos conocidos en compilación. Map para datos
    dinámicos o cuando las claves no se conocen de antemano.

  ❓ ¿Cuál es la diferencia entre new(Struct) y &Struct{}?
  → Son equivalentes. Ambos retornan *Struct con campos inicializados a cero.

  ❓ ¿Los structs se pasan por valor o por referencia?
  → Por valor. Para evitar copias, pasa un puntero *Struct.

  ❓ ¿Qué es el embedding en Go?
  → Es composición: incluir un struct anónimo dentro de otro. Sus campos y
    métodos se promueven al struct contenedor.

  ❓ ¿Puedo tener herencia en Go?
  → No hay herencia de tipos. Go usa composición mediante embedding.

  ❓ ¿Para qué sirven las tags en los campos?
  → Para metadatos: serialización JSON/XML, validación, ORM, etc.

  ❓ ¿Cómo convierto un struct a JSON?
  → Con json.Marshal() y json.MarshalIndent().

  ❓ ¿Un struct vacío ocupa memoria?
  → struct{} ocupa 0 bytes. Útil para sets y canales de señalización.

  ❓ ¿Puedo comparar dos structs con ==?
  → Sí, si todos los campos son comparables (no slices, maps).

  ❓ ¿Cómo encapsulo campos privados?
  → Usa minúscula para el nombre del campo. Solo accesible dentro del paquete.
