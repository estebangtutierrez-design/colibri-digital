  1. ¿Qué es un Map?
  ──────────────────

  Un map asocia claves únicas con valores. La sintaxis es:

      map[K]V

  donde K es el tipo de clave (debe ser comparable) y V es el tipo de valor.

  Características:
  • No ordenado: el orden de iteración no está garantizado.
  • Claves únicas: no puede haber duplicados.
  • Acceso O(1) promedio.
  • Tipo referencia: múltiples variables pueden apuntar al mismo map.
  • Valor cero: nil (no se puede escribir en un map nil).

  2. Declaración e Inicialización
  ───────────────────────────────

  ================================================================================
  EJEMPLO 1: Declaración básica
  ================================================================================

  package main

  import "fmt"

  func main() {
      // Map nil (valor cero)
      var m1 map[string]int
      fmt.Println("m1:", m1, "nil:", m1 == nil)

      // Map vacío con make
      m2 := make(map[string]int)
      fmt.Println("m2:", m2, "nil:", m2 == nil)

      // Map con capacidad inicial (opcional)
      m3 := make(map[string]int, 100)
      fmt.Println("m3 (cap 100):", m3)

      // Map literal
      m4 := map[string]int{
          "a": 1,
          "b": 2,
          "c": 3,
      }
      fmt.Println("m4:", m4)
  }

  ================================================================================
  EJEMPLO 2: Diferencia nil vs vacío
  ================================================================================

  package main

  import "fmt"

  func main() {
      var nilMap map[string]int
      emptyMap := make(map[string]int)

      // nilMap no se puede escribir
      // nilMap["clave"] = 42 // PANIC: assignment to entry in nil map

      // emptyMap sí se puede
      emptyMap["clave"] = 42
      fmt.Println("emptyMap:", emptyMap)

      // Ambas se pueden leer (nilMap retorna valor cero)
      fmt.Println("nilMap[\"x\"]:", nilMap["x"])    // 0
      fmt.Println("emptyMap[\"x\"]:", emptyMap["x"]) // 0
  }

  3. Operaciones Básicas
  ──────────────────────

  ================================================================================
  EJEMPLO 3: Set, Get, Delete
  ================================================================================

  package main

  import "fmt"

  func main() {
      edades := make(map[string]int)

      // Set (insertar/actualizar)
      edades["Alice"] = 30
      edades["Bob"] = 25
      edades["Charlie"] = 35

      fmt.Println("Map:", edades)

      // Get (obtener)
      fmt.Println("Edad de Alice:", edades["Alice"])

      // Delete (eliminar)
      delete(edades, "Bob")
      fmt.Println("Después de eliminar Bob:", edades)
  }

  4. La Expresión "Coma OK"
  ────────────────────────

  ================================================================================
  EJEMPLO 5: Coma OK
  ================================================================================

  package main

  import "fmt"

  func main() {
      m := map[string]int{"x": 100, "y": 200}

      valor, ok := m["x"]
      fmt.Printf("m[\"x\"]: valor=%d, existe=%t\n", valor, ok)

      valor, ok = m["z"]
      fmt.Printf("m[\"z\"]: valor=%d, existe=%t\n", valor, ok)

      if v, existe := m["y"]; existe {
          fmt.Println("y existe con valor:", v)
      } else {
          fmt.Println("y no existe")
      }
  }

  ================================================================================
  EJEMPLO 6: Verificar existencia antes de operar
  ================================================================================

  package main

  import "fmt"

  func main() {
      inventario := map[string]int{
          "manzanas": 10,
          "peras":     5,
      }

      if _, ok := inventario["naranjas"]; ok {
          fmt.Println("Hay naranjas")
      } else {
          fmt.Println("No hay naranjas")
      }

      // Incrementar solo si existe
      if val, ok := inventario["manzanas"]; ok {
          inventario["manzanas"] = val + 1
      }
      fmt.Println("manzanas:", inventario["manzanas"])
  }

  ================================================================================
  EJEMPLO 10: Map de slices
  ================================================================================

  package main

  import "fmt"

  func main() {
      // Map de lista de compras por categoría
      lista := map[string][]string{
          "frutas":     {"manzana", "pera", "uva"},

      fmt.Println("Frutas actualizado:", lista["frutas"])
  }

  11. Iteración Ordenada (simulada)
  ─────────────────────────────────

  ================================================================================
  EJEMPLO 20: Map ordenado manualmente
  ================================================================================

  package main

  import (
      "fmt"
      "sort"
  )

  func main() {
      notas := map[string]float64{
          "Juan": 85.5, "María": 92.0,
          "Pedro": 78.3, "Ana": 95.7,
      }

      // Extraer claves, ordenar, iterar
      nombres := make([]string, 0, len(notas))
      for n := range notas {
          nombres = append(nombres, n)
      }
      sort.Strings(nombres)

      fmt.Println("Notas ordenadas por nombre:")
      for _, n := range nombres {
          fmt.Printf("  %s: %.1f\n", n, notas[n])
      }
  }

  ================================================================================
  EJEMPLO 21: Map ordenado por valor
  ================================================================================

  package main

  import (
      "fmt"
      "sort"
  )

  func main() {
      notas := map[string]float64{
          "Juan": 85.5, "María": 92.0,
          "Pedro": 78.3, "Ana": 95.7,
      }

      type kv struct {
          K string
          V float64
      }
      var sorted []kv
      for k, v := range notas {
          sorted = append(sorted, kv{k, v})
      }
      sort.Slice(sorted, func(i, j int) bool {
          return sorted[i].V > sorted[j].V // Mayor a menor
      })

      for _, p := range sorted {
          fmt.Printf("  %s: %.1f\n", p.K, p.V)
      }
  }

  12. Merge de Maps
  ─────────────────

  ================================================================================
  EJEMPLO 22: Fusionar maps
  ================================================================================

  package main

  import "fmt"

  func merge[K comparable, V any](dest, src map[K]V) {
      for k, v := range src {
          dest[k] = v
      }
  }

  func main() {
      a := map[string]int{"a": 1, "b": 2}
      b := map[string]int{"c": 3, "d": 4, "b": 99}

      merge(a, b)
      fmt.Println("Merge:", a) // {a:1, b:99, c:3, d:4}
  }

  13. Copiar un Map
  ─────────────────

  ================================================================================
  EJEMPLO 23: Copia profunda de map
  ================================================================================

  package main

  import "fmt"

  func copiarMap(original map[string]int) map[string]int {
      copia := make(map[string]int, len(original))
      for k, v := range original {
          copia[k] = v
      }
      return copia
  }

  func main() {
      original := map[string]int{"a": 1, "b": 2, "c": 3}
      copia := copiarMap(original)

      copia["a"] = 999
      fmt.Println("Original:", original)
      fmt.Println("Copia:", copia)
  }

  RESUMEN DEL CAPÍTULO
  ================================================================================

  • map[K]V asocia claves únicas con valores.
  • Las claves deben ser COMPARABLES (no slices, maps ni functions).
  • Valor cero: nil (no se puede escribir, solo leer).
  • Inicialización: make(map[K]V) o literal map[K]V{k: v}.
  • Operaciones: m[k] = v (set/update), v := m[k] (get), delete(m, k).
  • "Coma OK": v, ok := m[k] para verificar existencia.
  • Iteración: for k, v := range m (orden NO garantizado).
  • len(m) para número de entradas.
  • Los maps son tipos REFERENCIA (se pasan por referencia).
  • No son seguros para concurrencia (usar sync.Map).

  ================================================================================
  PREGUNTAS FRECUENTES (FAQ)
  ================================================================================

  ❓ ¿Por qué un map nil no se puede escribir?
  → Porque no tiene memoria asignada. make() la asigna.

  ❓ ¿Cómo sé si una clave existe en un map?
  → Con la expresión "coma ok": valor, existe := m[clave].

  ❓ ¿Los maps mantienen el orden de inserción?
  → No. El orden de iteración es impredecible. Si necesitas orden, usa un slice
    adicional.

  ❓ ¿Puedo usar un slice como clave de map?
  → No. Los slices no son comparables. Usa string o array.

  ❓ ¿Qué es mejor: map o switch?
  → map es mejor para datos dinámicos; switch para opciones fijas en compilación.

  ❓ ¿Cuándo debo preasignar capacidad en make(map[K]V, n)?
  → Cuando sabes aproximadamente cuántas entradas tendrá. Mejora rendimiento.

  ❓ ¿Cómo elimino todas las entradas de un map?
  → Puedes usar delete en un bucle o reasignar: m = make(map[K]V).

  ❓ ¿Los maps son seguros para concurrencia?
  → No. Para acceso concurrente usa sync.Map o mutex.

  ❓ ¿Qué pasa si modifico un map durante la iteración?
  → Puedes modificar valores, pero añadir/eliminar claves puede dar resultados
    impredecibles.

  ❓ ¿Puedo tener un map con clave string y valor map[string]int?
  → Sí: map[string]map[string]int.
