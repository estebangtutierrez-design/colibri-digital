================================================================================
  11.1 INTRODUCCIÓN TEÓRICA
================================================================================

  La estructura de control if/else es probablemente la más fundamental en
  cualquier lenguaje de programación. Permite ejecutar bloques de código
  condicionalmente, basándose en el valor de verdad de una expresión booleana.

  En Go, el if/else se destaca por tres características principales:

  1. NO se usan paréntesis alrededor de la condición.
     En C, Java o JavaScript, escribimos:
       if (x > 0) { ... }
     En Go, escribimos:
       if x > 0 { ... }

     Esto elimina ruido visual y hace que el código sea más legible.

  2. Soporta una declaración corta (short statement) antes de la condición.
     Podemos declarar una variable cuyo ámbito (scope) queda limitado al
     bloque if/else. Esto es extremadamente útil para operaciones que
     pueden fallar, como veremos más adelante con funciones.

  3. Las llaves { } son obligatorias, incluso para una sola línea.
     Esto previene errores comunes como el "dangling else" y mejora la
     legibilidad.

  En este capítulo exploraremos cada una de estas características a fondo,
  con decenas de ejemplos prácticos que van desde lo más básico hasta
  patrones profesionales.

================================================================================
  11.2 SINTAXIS BÁSICA DE IF
================================================================================

  La forma más simple de if en Go es:

    if condición {
        // código a ejecutar si la condición es verdadera
    }

  Notas importantes:
  - La condición debe ser una expresión booleana (bool).
  - Go NO convierte implícitamente enteros a booleanos.
  - Las llaves son obligatorias.
  - La llave de apertura va en la MISMA línea que el if.

  Ejemplo básico:

  ```go
  package main

  import "fmt"

  func main() {
      edad := 18

      if edad >= 18 {
          fmt.Println("Eres mayor de edad")
      }

      fmt.Println("Esto se ejecuta siempre")
  }
  ```

  Output:
    Eres mayor de edad
    Esto se ejecuta siempre

  Veamos otro ejemplo con una variable booleana directamente:

  ```go
  package main

  import "fmt"

  func main() {
      esMayor := true

      if esMayor {
          fmt.Println("Acceso concedido")
      }
  }
  ```

  Output:
    Acceso concedido

  En Go, no puedes hacer esto:

  ```go
  // INCORRECTO — No compila
  x := 10
  if x {
      fmt.Println("Esto no funciona")
  }
  ```

  La razón es que x es un int, no un bool. Go es estrictamente tipado y
  no realiza conversiones automáticas. Debes escribir explícitamente:

  ```go
  if x != 0 {
      fmt.Println("x no es cero")
  }
  ```

  Esta decisión de diseño hace que el código sea más predecible y menos
  propenso a errores.

================================================================================
  11.3 OPERADORES DE COMPARACIÓN
================================================================================

  Los operadores de comparación en Go son los siguientes:

    ==  igual
    !=  diferente (no igual)
    <   menor que
    >   mayor que
    <=  menor o igual que
    >=  mayor o igual que

  Todos estos operadores producen un valor bool.

  Ejemplos:

  ```go
  package main

  import "fmt"

  func main() {
      a := 10
      b := 20
      c := 10

      fmt.Println("a == b:", a == b)   // false
      fmt.Println("a == c:", a == c)   // true
      fmt.Println("a != b:", a != b)   // true
      fmt.Println("a < b:", a < b)     // true
      fmt.Println("a > b:", a > b)     // false
      fmt.Println("a <= c:", a <= c)   // true
      fmt.Println("a >= c:", a >= c)   // true

      // Comparación de strings
      fmt.Println("\"abc\" == \"abc\":", "abc" == "abc")  // true
      fmt.Println("\"abc\" < \"abd\":", "abc" < "abd")   // true (comparación lexicográfica)
  }
  ```

  Output:
    a == b: false
    a == c: true
    a != b: true
    a < b: true
    a > b: false
    a <= c: true
    a >= c: true
    "abc" == "abc": true
    "abc" < "abd": true

  Los operadores de comparación se pueden usar con cualquier tipo que
  sea comparable: tipos numéricos, strings, booleanos (== y !=), punteros,
  canales, structs (si todos sus campos son comparables), arrays (si el
  elemento es comparable).

  IMPORTANTE: Los slices, maps y funciones NO son comparables con ==
  (excepto con nil).

  ```go
  // INCORRECTO — No compila
  // a := []int{1, 2, 3}
  // b := []int{1, 2, 3}
  // if a == b { ... }
  ```

================================================================================
  11.4 OPERADORES LÓGICOS (&&, ||, !)
================================================================================

  Go tiene tres operadores lógicos:

    &&  AND (y)      — verdadero si AMBAS condiciones son verdaderas
    ||  OR (o)       — verdadero si AL MENOS UNA condición es verdadera
    !   NOT (no)     — invierte el valor de verdad

  Estos operadores funcionan con valores bool y siguen las reglas de
  evaluación de cortocircuito (short-circuit evaluation):

  - En a && b, si a es false, b NO se evalúa.
  - En a || b, si a es true, b NO se evalúa.

  Ejemplo:

  ```go
  package main

  import "fmt"

  func main() {
      edad := 25
      tieneLicencia := true

      // AND
      if edad >= 18 && tieneLicencia {
          fmt.Println("Puedes conducir")
      }

      // OR
      if edad < 12 || edad > 65 {
          fmt.Println("Tienes descuento especial")
      }

      // NOT
      if !tieneLicencia {
          fmt.Println("No tienes licencia")
      } else {
          fmt.Println("Tienes licencia")
      }

      // Combinación de operadores
      if (edad >= 18 && tieneLicencia) || edad > 70 {
          fmt.Println("Puedes conducir (con precaución si eres mayor)")
      }
  }
  ```

  Output:
    Puedes conducir
    Tienes licencia
    Puedes conducir (con precaución si eres mayor)

  Cortocircuito práctico:

  ```go
  package main

  import (
      "fmt"
      "strings"
  )

  func main() {
      nombre := ""

      // Gracias al cortocircuito, no llamamos strings.ToUpper si nombre está vacío
      if nombre != "" && strings.ToUpper(nombre) == "ADMIN" {
          fmt.Println("Acceso de administrador")
      } else {
          fmt.Println("Acceso denegado o nombre vacío")
      }
  }
  ```

  Output:
    Acceso denegado o nombre vacío

  La tabla de verdad de &&:

    true  && true   = true
    true  && false  = false
    false && true   = false
    false && false  = false

  La tabla de verdad de ||:

    true  || true   = true
    true  || false  = true
    false || true   = true
    false || false  = false

  La tabla de verdad de !:

    !true  = false
    !false = true

================================================================================
  11.5 IF CON SHORT STATEMENT
================================================================================

  Una de las características más distintivas de Go es el "if con short
  statement" (declaración corta). La sintaxis es:

    if declaración; condición {
        // código
    }

  La declaración se ejecuta antes de evaluar la condición. La variable
  (o variables) declaradas en esta declaración tienen un ámbito (scope)
  que abarca todo el bloque if y sus bloques else asociados.

  Ejemplo básico:

  ```go
  package main

  import "fmt"

  func main() {
      x := 10

      if v := x * 2; v > 15 {
          fmt.Println("v es mayor que 15:", v)
      } else {
          fmt.Println("v es menor o igual a 15:", v)
      }

      // fmt.Println(v) // ERROR: v no está definido aquí
  }
  ```

  Output:
    v es mayor que 15: 20

  El uso más común es con funciones que retornan un valor y un posible
  error:

  ```go
  package main

  import (
      "fmt"
      "strconv"
  )

  func main() {
      valor := "123"

      if num, err := strconv.Atoi(valor); err != nil {
          fmt.Println("Error de conversión:", err)
      } else {
          fmt.Println("Número convertido:", num)
          // num está disponible aquí
      }

      // fmt.Println(num) // ERROR: num no está definido aquí
  }
  ```

  Output:
    Número convertido: 123

  Este patrón es ubicuo en Go. Lo verás en todas partes: al abrir archivos,
  hacer peticiones HTTP, convertir tipos, etc.

  Podemos declarar múltiples variables:

  ```go
  package main

  import (
      "fmt"
      "math"
  )

  func main() {
      x := -10.0

      if y, z := math.Abs(x), math.Sqrt(x); y > 5 {
          fmt.Printf("y = %.2f, pero z no es real\n", y)
      } else {
          fmt.Printf("y = %.2f, z no está definido\n", y)
      }
  }
  ```

  El short statement es especialmente útil cuando quieres limitar el
  ámbito de una variable a solo donde se necesita. Esto hace el código
  más legible y seguro.

  Otro ejemplo más elaborado:

  ```go
  package main

  import (
      "fmt"
      "os"
  )

  func main() {
      // Simula un mapa que puede tener o no una clave
      config := map[string]string{
          "timeout": "30",
          "host":    "localhost",
      }

      if value, ok := config["timeout"]; ok {
          fmt.Println("Timeout encontrado:", value)
      } else {
          fmt.Println("Timeout no configurado, usando valor por defecto")
      }
  }
  ```

  Output:
    Timeout encontrado: 30

================================================================================
  11.6 IF-ELSE
================================================================================

  La estructura if-else permite ejecutar un bloque de código cuando la
  condición es verdadera y otro cuando es falsa:

    if condición {
        // código si condición es true
    } else {
        // código si condición es false
    }

  Ejemplo:

  ```go
  package main

  import "fmt"

  func main() {
      numero := 7

      if numero%2 == 0 {
          fmt.Println(numero, "es par")
      } else {
          fmt.Println(numero, "es impar")
      }
  }
  ```

  Output:
    7 es impar

  Ejemplo con short statement:

  ```go
  package main

  import (
      "fmt"
      "math"
  )

  func main() {
      numero := -25.0

      if raiz, err := math.Sqrt(numero); err != nil {
          // En Go estándar, math.Sqrt no retorna error (retorna NaN para negativos)
          // pero simulamos el concepto
          fmt.Println("No se puede calcular la raíz de un número negativo")
      } else {
          fmt.Printf("La raíz cuadrada de %.0f es %.2f\n", numero, raiz)
      }

      // Versión corregida:
      if numero >= 0 {
          raiz := math.Sqrt(numero)
          fmt.Printf("La raíz cuadrada de %.0f es %.2f\n", numero, raiz)
      } else {
          fmt.Printf("No se puede calcular la raíz de %.0f (negativo)\n", numero)
      }
  }
  ```

  Output:
    No se puede calcular la raíz de -25 (negativo)

================================================================================
  11.7 IF-ELSE IF-ELSE
================================================================================

  Podemos encadenar múltiples condiciones con if-else if-else:

    if condición1 {
        // código si condición1 es true
    } else if condición2 {
        // código si condición1 es false y condición2 es true
    } else if condición3 {
        // código si condición1 y condición2 son false y condición3 es true
    } else {
        // código si todas las condiciones son false
    }

  Ejemplo:

  ```go
  package main

  import "fmt"

  func main() {
      nota := 85

      if nota >= 90 {
          fmt.Println("Calificación: A")
      } else if nota >= 80 {
          fmt.Println("Calificación: B")
      } else if nota >= 70 {
          fmt.Println("Calificación: C")
      } else if nota >= 60 {
          fmt.Println("Calificación: D")
      } else {
          fmt.Println("Calificación: F")
      }
  }
  ```

  Output:
    Calificación: B

  Ejemplo con short statement:

  ```go
  package main

  import (
      "fmt"
      "time"
  )

  func main() {
      hora := time.Now().Hour()

      if hora := 14; hora < 12 {
          fmt.Println("Buenos días")
      } else if hora < 18 {
          fmt.Println("Buenas tardes")
      } else {
          fmt.Println("Buenas noches")
      }

      // También podemos combinar short statement en cada rama
      // aunque esto es poco común
  }
  ```

  Output:
    Buenas tardes

  Las escaleras de if-else if pueden volverse difíciles de leer cuando
  hay muchas condiciones. En esos casos, switch es una mejor alternativa
  (Capítulo 12).

================================================================================
  11.8 IF ANIDADOS
================================================================================

  Podemos colocar un if dentro de otro if. Esto se llama anidamiento.

  ```go
  package main

  import "fmt"

  func main() {
      edad := 20
      tieneLicencia := true

      if edad >= 18 {
          fmt.Println("Eres mayor de edad")

          if tieneLicencia {
              fmt.Println("Y tienes licencia para conducir")
          } else {
              fmt.Println("Pero no tienes licencia")
          }
      } else {
          fmt.Println("Eres menor de edad")
      }
  }
  ```

  Output:
    Eres mayor de edad
    Y tienes licencia para conducir

  Aunque el anidamiento es posible, se recomienda evitar más de 2-3
  niveles de anidación. El código profundamente anidado es difícil de
  leer y mantener.

  Una técnica para reducir anidamiento es usar early returns (en funciones)
  o reorganizar la lógica:

  ```go
  package main

  import "fmt"

  func procesarEdad(edad int, tieneLicencia bool) {
      // Early return: si la condición inicial no se cumple, salimos
      if edad < 18 {
          fmt.Println("Acceso denegado: menor de edad")
          return
      }

      if !tieneLicencia {
          fmt.Println("Acceso denegado: sin licencia")
          return
      }

      fmt.Println("Acceso concedido: puedes conducir")
  }

  func main() {
      procesarEdad(20, true)
      procesarEdad(15, true)
      procesarEdad(20, false)
  }
  ```

  Output:
    Acceso concedido: puedes conducir
    Acceso denegado: menor de edad
    Acceso denegado: sin licencia

  Esta técnica (early return) es muy común en Go y produce código más
  lineal y legible.

================================================================================
  11.9 IF SIN PARÉNTESIS
================================================================================

  Como mencionamos al inicio, Go NO requiere (ni permite) paréntesis
  alrededor de la condición del if.

  Correcto:
    if x > 0 { ... }
    if v := f(); v > 0 { ... }

  Incorrecto (pero compila en algunos casos por compatibilidad):
    if (x > 0) { ... }   // Compila pero es considerado mala práctica

  La recomendación oficial es NO usar paréntesis. El equipo de Go diseñó
  el lenguaje para que sea simple y limpio.

  ```go
  package main

  import "fmt"

  func main() {
      x := 5

      // Estilo Go (recomendado)
      if x > 0 {
          fmt.Println("x es positivo")
      }

      // Estilo C-like (no recomendado en Go)
      if (x > 0) {
          fmt.Println("Esto también funciona, pero no es idiomático")
      }
  }
  ```

  Output:
    x es positivo
    Esto también funciona, pero no es idiomático

  La ausencia de paréntesis hace que el código sea más limpio, pero
  requiere que te acostumbres si vienes de C, Java o JavaScript.

================================================================================
  11.10 BUENAS PRÁCTICAS
================================================================================

  1. NO uses paréntesis alrededor de la condición.
  2. Siempre usa llaves { }, incluso para una sola línea.
  3. Prefiere early returns para evitar anidamiento profundo.
  4. Usa el short statement para limitar el ámbito de variables.
  5. Evalúa primero las condiciones más específicas o más probables.
  6. Usa else if con moderación; considera switch si hay muchas ramas.
  7. Mantén las condiciones simples; si son complejas, usa variables
     auxiliares.
  8. No anides más de 2-3 niveles.
  9. Aprovecha el cortocircuito para evitar operaciones innecesarias.
  10. En condiciones compuestas, ordena las subcondiciones de la más
      barata a la más cara (en términos de rendimiento).

  Ejemplo de condición compleja refactorizada:

  ```go
  // En lugar de esto:
  if (usuario != nil && usuario.Rol == "admin" && len(usuario.Permisos) > 0 && usuario.Permisos["escritura"]) {
      // ...
  }

  // Haz esto:
  esAdmin := usuario != nil && usuario.Rol == "admin"
  tienePermiso := esAdmin && len(usuario.Permisos) > 0 && usuario.Permisos["escritura"]

  if tienePermiso {
      // ...
  }
  ```

================================================================================
  11.11 ERRORES COMUNES
================================================================================

  Error 1: Usar paréntesis como en C/Java
  ```go
  // Estilo C (no idiomático en Go)
  if (x > 5) {
      fmt.Println("x > 5")
  }
  ```

  Error 2: Olvidar las llaves
  ```go
  // INCORRECTO - No compila
  if x > 5
      fmt.Println("x > 5")
  ```

  Error 3: Condición con tipo incorrecto
  ```go
  // INCORRECTO - No compila
  x := 10
  if x {
      fmt.Println("x es verdadero")
  }
  // Debes usar: if x != 0 { ... }
  ```

  Error 4: Else en nueva línea
  ```go
  // INCORRECTO - No compila
  if x > 5 {
      fmt.Println("x > 5")
  }
  else {
      fmt.Println("x <= 5")
  }
  // La llave de cierre y else deben estar en la misma línea
  ```

  Error 5: Dos condiciones con coma
  ```go
  // INCORRECTO - No compila
  if x > 5, y < 10 {
      // ...
  }
  // Debes usar operador lógico: if x > 5 && y < 10 { ... }
  ```

  Error 6: Shadowing no intencional con short statement
  ```go
  x := 10
  if x := 20; x > 15 {  // Esto crea una nueva x dentro del if
      fmt.Println(x)    // Imprime 20, no 10
  }
  fmt.Println(x)        // Imprime 10
  ```

  Error 7: Asignar en lugar de comparar
  ```go
  // INCORRECTO - No compila (en Go no se puede usar asignación como expresión)
  if x = 5 {
      // ...
  }
  ```

================================================================================
  11.12 CÓDIGO PROFESIONAL: EJEMPLOS COMPLETOS
================================================================================

  Ejemplo 1: Validador de contraseña

  ```go
  package main

  import (
      "fmt"
      "strings"
      "unicode"
  )

  func validarContrasena(password string) bool {
      if len(password) < 8 {
          fmt.Println("La contraseña debe tener al menos 8 caracteres")
          return false
      }

      mayuscula := false
      minuscula := false
      numero := false
      especial := false

      for _, char := range password {
          switch {
          case unicode.IsUpper(char):
              mayuscula = true
          case unicode.IsLower(char):
              minuscula = true
          case unicode.IsDigit(char):
              numero = true
          case unicode.IsPunct(char) || unicode.IsSymbol(char):
              especial = true
          }
      }

      if !mayuscula {
          fmt.Println("Debe contener al menos una mayúscula")
          return false
      }
      if !minuscula {
          fmt.Println("Debe contener al menos una minúscula")
          return false
      }
      if !numero {
          fmt.Println("Debe contener al menos un número")
          return false
      }
      if !especial {
          fmt.Println("Debe contener al menos un carácter especial")
          return false
      }

      return true
  }

  func main() {
      passwords := []string{
          "hola",
          "SOLOMAYUSCULAS",
          "solominusculas",
          "SinNumeros!",
          "Val1d0!",
      }

      for _, pwd := range passwords {
          fmt.Printf("Validando: %s\n", pwd)
          if validarContrasena(pwd) {
              fmt.Println("  ✓ VÁLIDA")
          } else {
              fmt.Println("  ✖ INVÁLIDA")
          }
          fmt.Println()
      }
  }
  ```

  Ejemplo 2: Clasificador de números

  ```go
  package main

  import "fmt"

  func clasificarNumero(n int) string {
      if n == 0 {
          return "cero"
      }

      tipo := ""
      if n > 0 {
          tipo = "positivo"
      } else {
          tipo = "negativo"
      }

      paridad := ""
      if n%2 == 0 {
          paridad = "par"
      } else {
          paridad = "impar"
      }

      return fmt.Sprintf("%s y %s", tipo, paridad)
  }

  func main() {
      numeros := []int{-5, -2, 0, 3, 8, 11, -1, 100}

      for _, n := range numeros {
          fmt.Printf("%4d → %s\n", n, clasificarNumero(n))
      }
  }
  ```

  Ejemplo 3: Calculadora de descuentos

  ```go
  package main

  import "fmt"

  func calcularDescuento(precio float64, esMiembro bool, mesesDesdeCompra int) float64 {
      descuento := 0.0

      if esMiembro {
          descuento = 0.10 // 10% para miembros
      }

      if mesesDesdeCompra > 12 {
          descuento += 0.05 // 5% adicional por fidelidad
      }

      if precio > 100 {
          descuento += 0.05 // 5% adicional por compra grande
      }

      if descuento > 0.25 {
          descuento = 0.25 // Máximo 25% de descuento
      }

      return precio * (1 - descuento)
  }

  func main() {
      tipoCliente := "miembro"
      precioOriginal := 200.0
      esMiembro := tipoCliente == "miembro"
      meses := 15

      precioFinal := calcularDescuento(precioOriginal, esMiembro, meses)
      ahorro := precioOriginal - precioFinal

      fmt.Printf("Precio original:  $%.2f\n", precioOriginal)
      fmt.Printf("Descuento aplicado: $%.2f (%.0f%%)\n", ahorro, ahorro/precioOriginal*100)
      fmt.Printf("Precio final:     $%.2f\n", precioFinal)
  }
  ```

================================================================================
  11.15 PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Por qué Go no permite if (x) cuando x es un entero?
  R: Go es estrictamente tipado. La condición debe ser explícitamente
     bool. Esto elimina ambigüedades y errores sutiles. Debes escribir
     if x != 0 { ... }.

  P: ¿Puedo poner el else en la línea siguiente?
  R: No. En Go, el else debe ir en la misma línea que la llave de
     cierre del if. Si no, el compilador inserta un punto y coma
     automáticamente y el código no compila.

  P: ¿Cuál es la diferencia entre el short statement de Go y declarar
     la variable antes del if?
  R: La variable del short statement solo existe dentro del bloque
     if/else. Esto la hace más segura y evita contaminar el ámbito
     superior.

  P: ¿Puedo usar else con un short statement?
  R: Sí. La variable declarada en el short statement está disponible
     tanto en el bloque if como en el else.

  P: ¿Cómo manejo múltiples condiciones?
  R: Usa los operadores lógicos && y ||. Si las condiciones son muchas,
     considera usar switch.

  P: ¿Existe el operador ternario en Go?
  R: No. Go no tiene operador ternario (cond ? val1 : val2). Se
     considera que if-else es más legible. Usa if-else directamente.

  P: ¿Puedo asignar en la condición del if?
  R: No como en otros lenguajes. En Go, la asignación simple no es una
     expresión. Pero puedes usar el short statement con :=.

  P: ¿Qué pasa si la condición del if es una llamada a función?
  R: La función se evalúa primero y su resultado (bool) se usa como
     condición. Puedes usar short statement para capturar el resultado.
