  ¡Bienvenido, arquitecto del código!

  En el capítulo anterior aprendiste los fundamentos de las funciones.
  Ahora exploraremos una de las características que hace único a Go:
  la capacidad de retornar múltiples valores desde una función.

  Mientras que en otros lenguajes necesitas crear un objeto, struct o
  tupla para devolver más de un valor, en Go simplemente los listas
  en la declaración de retorno y los devuelves separados por comas.

  (valor, error), que se ha convertido en el estándar para el manejo
  de errores en Go. En lugar de lanzar excepciones (como en Java o
  Python), las funciones retornan un valor y un posible error.

  También exploraremos el identificador _ (blank identifier) para
  ignorar valores que no necesitas, y los retornos nombrados.

================================================================================
  15.1 INTRODUCCIÓN TEÓRICA
================================================================================

  Los múltiples retornos son una característica distintiva de Go.
  Permiten que una función devuelva más de un valor simultáneamente.

  Esto es útil para:
  1. Devolver un resultado y un error (el patrón más común).
  2. Devolver varios resultados relacionados (como cociente y resto).
  3. Devolver valores calculados y metadatos.
  4. Devolver un valor y un indicador de éxito (ok).

  La filosofía de Go es que las funciones deben ser honestas sobre
  lo que pueden producir. Si una función puede fallar, debería
  indicarlo explícitamente en su firma, no mediante excepciones
  ocultas.

  El patrón (valor, error) es ubicuo en Go. Lo encontrarás en:
  - Apertura de archivos (os.Open).
  - Conversión de tipos (strconv.Atoi).
  - Lectura de entrada (bufio.Reader.ReadString).
  - Operaciones HTTP (http.Get).
  - Operaciones de base de datos.
  - Cualquier operación que pueda fallar.

  A diferencia de otros lenguajes, Go no tiene try-catch. Los errores
  se manejan explícitamente donde ocurren.

================================================================================
  15.2 SINTAXIS DE MÚLTIPLES RETORNOS
================================================================================

  Para declarar una función que retorna múltiples valores, se
  especifican los tipos entre paréntesis:

    func nombre(parametros) (tipo1, tipo2, tipo3) {
        return valor1, valor2, valor3
    }

  Ejemplo:

  ```go
  package main

  import "fmt"

  // Retorna cociente y resto
  func dividir(a, b int) (int, int) {
      cociente := a / b
      resto := a % b
      return cociente, resto
  }

  func main() {
      q, r := dividir(17, 5)
      fmt.Printf("17 / 5 = %d, resto = %d\n", q, r)
  }
  ```

  Output:
    17 / 5 = 3, resto = 2

  Puedes retornar cualquier combinación de tipos:

  ```go
  package main

  import (
      "fmt"
      "strings"
  )

  func procesarNombre(nombre string) (string, int, bool) {
      nombre = strings.TrimSpace(nombre)
      longitud := len(nombre)
      esValido := longitud > 0
      return nombre, longitud, esValido
  }

  func main() {
      nombre, long, valido := procesarNombre("  Ana  ")
      fmt.Printf("Nombre: '%s', Long: %d, Válido: %t\n", nombre, long, valido)

      nombre2, long2, valido2 := procesarNombre("")
      fmt.Printf("Nombre: '%s', Long: %d, Válido: %t\n", nombre2, long2, valido2)
  }
  ```

  Output:
    Nombre: 'Ana', Long: 3, Válido: true
    Nombre: '', Long: 0, Válido: false

  También puedes nombrar los retornos:

  ```go
  func dividir(a, b int) (cociente, resto int) {
      cociente = a / b
      resto = a % b
      return
  }
  ```

================================================================================
  15.3 CAPTURA DE MÚLTIPLES VALORES
================================================================================

  Para capturar los valores retornados, usa asignación múltiple:

  ```go
  valor1, valor2, valor3 := funcion()
  ```

  Si no necesitas todos los valores, usa _ (blank identifier):

  ```go
  valor1, _, valor3 := funcion()
  ```

  Ejemplos:

  ```go
  package main

  import "fmt"

  func estadisticas(nums []int) (int, float64, int, int) {
      if len(nums) == 0 {
          return 0, 0, 0, 0
      }

      suma := 0
      min := nums[0]
      max := nums[0]

      for _, v := range nums {
          suma += v
          if v < min { min = v }
          if v > max { max = v }
      }

      promedio := float64(suma) / float64(len(nums))
      return suma, promedio, min, max
  }

  func main() {
      valores := []int{8, 3, 12, 5, 7, 9}

      // Captura todos los valores
      suma, prom, min, max := estadisticas(valores)
      fmt.Printf("Suma: %d, Prom: %.1f, Min: %d, Max: %d\n", suma, prom, min, max)

      // Captura solo algunos valores
      s, p, _, _ := estadisticas(valores)
      fmt.Printf("Suma: %d, Promedio: %.1f\n", s, p)

      _, _, mn, mx := estadisticas(valores)
      fmt.Printf("Min: %d, Max: %d\n", mn, mx)
  }
  ```

  Output:
    Suma: 44, Prom: 7.3, Min: 3, Max: 12
    Suma: 44, Promedio: 7.3
    Min: 3, Max: 12

  La asignación múltiple también funciona con funciones que retornan
  un solo valor, por supuesto:

  ```go
  resultado := funcion()
  ```

  Pero si una función retorna 3 valores, debes proporcionar 3
  variables (o usar _):

  ```go
  // INCORRECTO - No compila
  resultado := funcionQueRetorna3() // Error: assignment mismatch

  // CORRECTO
  a, b, c := funcionQueRetorna3()
  ```

================================================================================
  15.4 EL IDENTIFICADOR _ (BLANK IDENTIFIER)
================================================================================

  El identificador _ (blank identifier, identificador en blanco) se
  usa para ignorar valores que no necesitas. Es como un "agujero
  negro" para asignaciones.

  ```go
  package main

  import (
      "fmt"
      "strconv"
  )

  func main() {
      // Ignorar el error (no recomendado en producción)
      num, _ := strconv.Atoi("42")
      fmt.Println("Número:", num)

      // Ignorar el valor, solo nos interesa si hay error
      _, err := strconv.Atoi("no-es-numero")
      if err != nil {
          fmt.Println("Error:", err)
      }

      // Ignorar índice en for range
      palabras := []string{"hola", "mundo", "go"}
      for _, palabra := range palabras {
          fmt.Println(palabra)
      }

      // Ignorar valor en for range de map
      edades := map[string]int{"Ana": 25, "Juan": 30}
      for nombre, _ := range edades {
          fmt.Println(nombre)
      }
      // También se puede: for nombre := range edades
  }
  ```

  Output:
    Número: 42
    Error: strconv.Atoi: parsing "no-es-numero": invalid syntax
    hola
    mundo
    go
    Ana
    Juan

  Usos comunes de _:
  1. Ignorar el error cuando estás seguro de que no fallará.
  2. Ignorar el índice en for range.
  3. Ignorar el valor en for range (cuando solo interesa la clave).
  4. Forzar la importación de un paquete (aunque no se use) en
     desarrollo.
  5. Forzar que un tipo implemente una interfaz (uso avanzado).

  ADVERTENCIA: Ignorar errores con _ puede ser peligroso. Úsalo solo
  cuando estés absolutamente seguro de que la operación no fallará
  (ej: strconv.Atoi en una constante conocida).

================================================================================
  15.5 EL PATRÓN (VALOR, ERROR)
================================================================================

  Este es EL patrón más importante de Go. Casi todas las funciones
  que pueden fallar retornan (valor, error). Si el error es nil, la
  operación fue exitosa. Si no, el valor es el cero del tipo.

  ```go
  package main

  import (
      "fmt"
      "os"
      "strconv"
  )

  func main() {
      // strconv.Atoi retorna (int, error)
      valor := "123"
      numero, err := strconv.Atoi(valor)
      if err != nil {
          fmt.Println("Error de conversión:", err)
          return
      }
      fmt.Println("Número:", numero)

      // Apertura de archivo
      archivo, err := os.Open("archivo.txt")
      if err != nil {
          fmt.Println("Error al abrir:", err)
          // No es fatal, el programa puede continuar
      } else {
          fmt.Println("Archivo abierto:", archivo.Name())
          archivo.Close()
      }
  }
  ```

  Output:
    Número: 123
    Error al abrir: open archivo.txt: no such file or directory

  Siempre debes verificar err ANTES de usar el valor:

  ```go
  // CORRECTO
  archivo, err := os.Open("archivo.txt")
  if err != nil {
      // Manejar error
      return
  }
  // Usar archivo (seguro porque err == nil)
  defer archivo.Close()

  // INCORRECTO - No verificar error
  archivo, _ := os.Open("archivo.txt")
  archivo.Close() // ¡Panic si el archivo no existe!
  ```

  Puedes crear tus propias funciones que retornen error:

  ```go
  package main

  import (
      "fmt"
      "math"
  )

  func sqrtConError(x float64) (float64, error) {
      if x < 0 {
          return 0, fmt.Errorf("no se puede calcular raíz de %v (negativo)", x)
      }
      return math.Sqrt(x), nil
  }

  func main() {
      valores := []float64{9.0, 16.0, -4.0, 25.0}

      for _, v := range valores {
          raiz, err := sqrtConError(v)
          if err != nil {
              fmt.Printf("Error: %v\n", err)
          } else {
              fmt.Printf("sqrt(%.0f) = %.2f\n", v, raiz)
          }
      }
  }
  ```

  Output:
    sqrt(9) = 3.00
    sqrt(16) = 4.00
    Error: no se puede calcular raíz de -4 (negativo)
    sqrt(25) = 5.00

  El tipo error es una interfaz incorporada (built-in):

  ```go
  type error interface {
      Error() string
  }
  ```

  Cualquier tipo que tenga un método Error() string implementa la
  interfaz error.

================================================================================
  15.6 RETORNOS NOMBRADOS CON MÚLTIPLES VALORES
================================================================================

  Los retornos nombrados funcionan perfectamente con múltiples valores:

  ```go
  package main

  import "fmt"

  func rectangulo(ancho, alto float64) (area, perimetro float64, err error) {
      if ancho <= 0 || alto <= 0 {
          err = fmt.Errorf("dimensiones inválidas: %.2f x %.2f", ancho, alto)
          return
      }

      area = ancho * alto
      perimetro = 2 * (ancho + alto)
      return // return desnudo: retorna area, perimetro, err
  }

  func main() {
      dimensiones := [][]float64{
          {5.0, 3.0},
          {0.0, 4.0},
          {-2.0, 6.0},
          {10.0, 10.0},
      }

      for _, dim := range dimensiones {
          a, p, err := rectangulo(dim[0], dim[1])
          if err != nil {
              fmt.Printf("Error: %v\n", err)
          } else {
              fmt.Printf("Rectángulo %.0fx%.0f → Área: %.1f, Perímetro: %.1f\n",
                  dim[0], dim[1], a, p)
          }
      }
  }
  ```

  Output:
    Rectángulo 5x3 → Área: 15.0, Perímetro: 16.0
    Error: dimensiones inválidas: 0.0 x 4.0
    Error: dimensiones inválidas: -2.0 x 6.0
    Rectángulo 10x10 → Área: 100.0, Perímetro: 40.0

  Con retornos nombrados, el return desnudo retorna los valores
  actuales de las variables de retorno. Esto es especialmente útil
  cuando hay múltiples retornos.

  Pero cuidado: el return desnudo puede hacer el código menos legible
  si la función es larga. En funciones cortas y claras, es aceptable.

  ```go
  // ESTILO RECOMENDADO para funciones largas:
  func procesarArchivo(ruta string) (contenido string, lineas int, err error) {
      datos, err := os.ReadFile(ruta)
      if err != nil {
          return "", 0, err  // Explícito
      }
      contenido = string(datos)
      lineas = strings.Count(contenido, "\n") + 1
      return contenido, lineas, nil  // Explícito
  }
  ```

================================================================================
  15.7 MÚLTIPLES RETORNOS EN FUNCIONES VARIÁDICAS
================================================================================

  Las funciones variádicas también pueden tener múltiples retornos:

  ```go
  package main

  import "fmt"

  func minMax(numeros ...int) (int, int, error) {
      if len(numeros) == 0 {
          return 0, 0, fmt.Errorf("se requiere al menos un número")
      }

      min := numeros[0]
      max := numeros[0]

      for _, v := range numeros[1:] {
          if v < min { min = v }
          if v > max { max = v }
      }

      return min, max, nil
  }

  func main() {
      // Probar con varios argumentos
      min, max, err := minMax(3, 7, 1, 9, 4, 2, 8)
      if err != nil {
          fmt.Println("Error:", err)
      } else {
          fmt.Printf("Min: %d, Max: %d\n", min, max)
      }

      // Probar con un solo argumento
      min2, max2, err2 := minMax(5)
      if err2 != nil {
          fmt.Println("Error:", err2)
      } else {
          fmt.Printf("Min: %d, Max: %d\n", min2, max2)
      }

      // Probar sin argumentos (error)
      _, _, err3 := minMax()
      if err3 != nil {
          fmt.Println("Error:", err3)
      }
  }
  ```

  Output:
    Min: 1, Max: 9
    Min: 5, Max: 5
    Error: se requiere al menos un número

================================================================================
  15.8 MÚLTIPLES RETORNOS CON DEFER
================================================================================

  defer se ejecuta después del return pero antes de que la función
  termine. Esto permite modificar los valores de retorno nombrados.

  ```go
  package main

  import "fmt"

  func contar() (cantidad int) {
      defer func() {
          cantidad += 10 // Modifica el retorno nombrado
      }()

      cantidad = 5
      return // Retorna 5, pero defer la modifica a 15
  }

  func main() {
      resultado := contar()
      fmt.Println("Resultado:", resultado) // 15
  }
  ```

  Output:
    Resultado: 15

  Esto es útil para tareas como:
  - Registrar el tiempo de ejecución de una función.
  - Asegurar que un recurso se cierre correctamente.
  - Modificar el error retornado.

  ```go
  package main

  import (
      "fmt"
      "time"
  )

  func medirTiempo(nombre string) func() {
      inicio := time.Now()
      return func() {
          duracion := time.Since(inicio)
          fmt.Printf("%s tardó %v\n", nombre, duracion)
      }
  }

  func operacionLenta() (resultado int, err error) {
      defer medirTiempo("operacionLenta")()

      // Simula trabajo
      time.Sleep(100 * time.Millisecond)

      resultado = 42
      return resultado, nil
  }

  func main() {
      r, err := operacionLenta()
      if err != nil {
          fmt.Println("Error:", err)
      } else {
          fmt.Println("Resultado:", r)
      }
  }
  ```

  Output:
    operacionLenta tardó 100.5ms
    Resultado: 42

================================================================================
  15.9 BUENAS PRÁCTICAS
================================================================================

  1. Siempre verifica el error antes de usar el valor. Este es el
     patrón más importante de Go.

  2. Usa el patrón (valor, error) para cualquier función que pueda
     fallar. No uses excepciones ni valores especiales (-1, nil) para
     indicar errores.

  3. Prefiere retornos nombrados para autodocumentar el significado
     de cada valor.

  4. No abuses de los múltiples retornos. Si necesitas retornar más
     de 3-4 valores, considera usar un struct.

  5. Usa _ para ignorar valores deliberadamente, pero nunca ignores
     errores a menos que estés 100% seguro.

  6. En el patrón (valor, error), si hay error, el valor debe ser el
     valor cero del tipo. Esto es una convención, no una regla del
     compilador.

  7. El return desnudo está bien en funciones cortas, pero prefiere
     return explícito en funciones largas.

  8. Si una función retorna un bool como indicador de éxito, nombra
     el retorno como "ok" para seguir la convención.

  9. Documenta qué retorna cada valor, especialmente en funciones
     exportadas.

  10. Usa errores descriptivos: incluye contexto sobre qué operación
      falló y con qué valores.

================================================================================
  15.10 ERRORES COMUNES
================================================================================

  Error 1: No verificar el error
  ```go
  archivo, _ := os.Open("archivo.txt") // Ignora el error
  archivo.Close() // ¡PANIC si archivo.txt no existe!
  ```

  Error 2: Asignación incorrecta
  ```go
  // INCORRECTO - assignment mismatch: 1 variable but 2 values
  resultado := dividir(10, 3)

  // CORRECTO
  q, r := dividir(10, 3)
  ```

  Error 3: Usar valor cuando hay error
  ```go
  num, err := strconv.Atoi("abc")
  if err != nil {
      fmt.Println(num) // num = 0, pero podría ser un valor válido
      // No deberías usar num aquí
  }
  ```

  Error 4: Olvidar retornar error cuando corresponde
  ```go
  func dividir(a, b int) (int, error) {
      if b == 0 {
          return 0 // FALTA retornar el error
      }
      return a / b, nil
  }
  ```

  Error 5: No usar _ cuando es necesario
  ```go
  // Si solo te interesa un valor, usa _
  suma, _, _, _ := estadisticas(valores)

  // En lugar de declarar variables que no usas
  suma, prom, min, max := estadisticas(valores)
  _ = prom
  _ = min
  _ = max
  ```

  Error 6: Confundir el orden de los retornos
  ```go
  // Siempre verifica la firma de la función
  // os.Open retorna (*File, error), no (error, *File)
  archivo, err := os.Open("archivo.txt")
  ```

  Error 7: Return desnudo confuso
  ```go
  func ejemplo() (a, b int) {
      a = 5
      // Olvidé asignar b
      return // Retorna (5, 0) porque b se inicializa en cero
      // Esto puede ser confuso
  }
  ```

  Ejemplo 1: Parseo de configuración con (valor, error)

  ```go
  package main

  import (
      "fmt"
      "strconv"
      "strings"
  )

  type Config struct {
      Host    string
      Puerto  int
      Debug   bool
      Timeout int
  }

  func parsearLineaConfig(linea string) (clave string, valor string, err error) {
      linea = strings.TrimSpace(linea)

      if len(linea) == 0 || strings.HasPrefix(linea, "#") {
          return "", "", nil // Línea vacía o comentario
      }

      partes := strings.SplitN(linea, "=", 2)
      if len(partes) != 2 {
          return "", "", fmt.Errorf("formato inválido: %s", linea)
      }

      clave = strings.TrimSpace(partes[0])
      valor = strings.TrimSpace(partes[1])

      if clave == "" {
          return "", "", fmt.Errorf("clave vacía en: %s", linea)
      }

      return clave, valor, nil
  }

  func cargarConfig(lineas []string) (Config, error) {
      cfg := Config{
          Host:    "localhost",
          Puerto:  8080,
          Debug:   false,
          Timeout: 30,
      }

      for _, linea := range lineas {
          clave, valor, err := parsearLineaConfig(linea)
          if err != nil {
              return cfg, err
          }
          if clave == "" {
              continue
          }

          switch clave {
          case "host":
              cfg.Host = valor
          case "puerto":
              p, err := strconv.Atoi(valor)
              if err != nil {
                  return cfg, fmt.Errorf("puerto inválido: %s", valor)
              }
              cfg.Puerto = p
          case "debug":
              cfg.Debug = valor == "true" || valor == "1" || valor == "yes"
          case "timeout":
              t, err := strconv.Atoi(valor)
              if err != nil {
                  return cfg, fmt.Errorf("timeout inválido: %s", valor)
              }
              cfg.Timeout = t
          default:
              fmt.Printf("Advertencia: clave desconocida '%s'\n", clave)
          }
      }

      return cfg, nil
  }

  func main() {
      configTexto := []string{
          "host=  servidor01",
          "puerto=9090",
          "# esto es un comentario",
          "debug=true",
          "timeout=60",
          "extra=ignorado",
      }

      cfg, err := cargarConfig(configTexto)
      if err != nil {
          fmt.Println("Error cargando configuración:", err)
          return
      }

      fmt.Printf("Config: %+v\n", cfg)
  }
  ```

  Ejemplo 2: API de base de datos simulada

  ```go
  package main

  import (
      "fmt"
      "math/rand"
      "time"
  )

  type Usuario struct {
      ID        int
      Nombre    string
      Email     string
      Activo    bool
      CreatedAt time.Time
  }

  type DB struct {
      usuarios map[int]Usuario
      nextID   int
  }

  func NuevaDB() *DB {
      return &DB{
          usuarios: make(map[int]Usuario),
          nextID:   1,
      }
  }

  func (db *DB) CrearUsuario(nombre, email string) (Usuario, error) {
      if nombre == "" {
          return Usuario{}, fmt.Errorf("el nombre es requerido")
      }
      if email == "" {
          return Usuario{}, fmt.Errorf("el email es requerido")
      }

      usuario := Usuario{
          ID:        db.nextID,
          Nombre:    nombre,
          Email:     email,
          Activo:    true,
          CreatedAt: time.Now(),
      }
      db.nextID++
      db.usuarios[usuario.ID] = usuario

      return usuario, nil
  }

  func (db *DB) ObtenerUsuario(id int) (Usuario, error) {
      usuario, ok := db.usuarios[id]
      if !ok {
          return Usuario{}, fmt.Errorf("usuario %d no encontrado", id)
      }
      return usuario, nil
  }

  func (db *DB) ActualizarUsuario(id int, nombre, email string) (Usuario, error) {
      usuario, err := db.ObtenerUsuario(id)
      if err != nil {
          return Usuario{}, err
      }

      if nombre != "" {
          usuario.Nombre = nombre
      }
      if email != "" {
          usuario.Email = email
      }

      db.usuarios[id] = usuario
      return usuario, nil
  }

  func (db *DB) EliminarUsuario(id int) (bool, error) {
      _, ok := db.usuarios[id]
      if !ok {
          return false, fmt.Errorf("usuario %d no encontrado", id)
      }
      delete(db.usuarios, id)
      return true, nil
  }

  func main() {
      rand.Seed(time.Now().UnixNano())
      db := NuevaDB()

      // Crear usuarios
      nombres := []string{"Ana", "Juan", "Luis", "Marta"}
      emails := []string{"ana@mail.com", "juan@mail.com", "luis@mail.com", "marta@mail.com"}

      for i := 0; i < len(nombres); i++ {
          usuario, err := db.CrearUsuario(nombres[i], emails[i])
          if err != nil {
              fmt.Printf("Error creando usuario: %v\n", err)
          } else {
              fmt.Printf("Creado: %+v\n", usuario)
          }
      }

      // Obtener usuario
      usuario, err := db.ObtenerUsuario(2)
      if err != nil {
          fmt.Println("Error:", err)
      } else {
          fmt.Printf("Obtenido: %+v\n", usuario)
      }

      // Actualizar usuario
      usuario, err = db.ActualizarUsuario(2, "Juan Carlos", "")
      if err != nil {
          fmt.Println("Error:", err)
      } else {
          fmt.Printf("Actualizado: %+v\n", usuario)
      }

      // Eliminar usuario
      ok, err := db.EliminarUsuario(3)
      if err != nil {
          fmt.Println("Error:", err)
      } else {
          fmt.Printf("Eliminado: %t\n", ok)
      }

      // Intentar obtener usuario eliminado
      _, err = db.ObtenerUsuario(3)
      if err != nil {
          fmt.Println("Error esperado:", err)
      }
  }
  ```

  Ejemplo 3: Operaciones matemáticas seguras

  ```go
  package main

  import (
      "fmt"
      "math"
  )

  func dividirSeguro(a, b float64) (float64, error) {
      if b == 0 {
          return 0, fmt.Errorf("división entre cero")
      }
      return a / b, nil
  }

  func raizSegura(x float64) (float64, error) {
      if x < 0 {
          return 0, fmt.Errorf("raíz de número negativo: %.2f", x)
      }
      return math.Sqrt(x), nil
  }

  func logSeguro(x float64) (float64, error) {
      if x <= 0 {
          return 0, fmt.Errorf("logaritmo de número no positivo: %.2f", x)
      }
      return math.Log(x), nil
  }

  func operacionCompleja(x, y float64) (float64, error) {
      suma, err := dividirSeguro(x, y)
      if err != nil {
          return 0, fmt.Errorf("división falló: %v", err)
      }

      raiz, err := raizSegura(suma)
      if err != nil {
          return 0, fmt.Errorf("raíz falló: %v", err)
      }

      log, err := logSeguro(raiz + 1)
      if err != nil {
          return 0, fmt.Errorf("log falló: %v", err)
      }

      return log, nil
  }

  func main() {
      casos := [][]float64{
          {10, 2},
          {10, 0},
          {-5, 2},
          {0.5, 2},
      }

      for _, c := range casos {
          resultado, err := operacionCompleja(c[0], c[1])
          if err != nil {
              fmt.Printf("operacionCompleja(%.1f, %.1f) → Error: %v\n", c[0], c[1], err)
          } else {
              fmt.Printf("operacionCompleja(%.1f, %.1f) → %.4f\n", c[0], c[1], resultado)
          }
      }
  }
  ```

  Ejemplo 4: Procesamiento de pedidos

  ```go
  package main

  import (
      "fmt"
      "time"
  )

  type Item struct {
      Nombre   string
      Precio   float64
      Cantidad int
  }

  type Pedido struct {
      ID         string
      Items      []Item
      Total      float64
      Descuento  float64
      Final      float64
      CreadoEn   time.Time
  }

  func validarItem(item Item) error {
      if item.Nombre == "" {
          return fmt.Errorf("el nombre del item es requerido")
      }
      if item.Precio <= 0 {
          return fmt.Errorf("el precio de '%s' debe ser positivo", item.Nombre)
      }
      if item.Cantidad <= 0 {
          return fmt.Errorf("la cantidad de '%s' debe ser positiva", item.Nombre)
      }
      return nil
  }

  func calcularTotal(items []Item) (total float64, err error) {
      for _, item := range items {
          if err := validarItem(item); err != nil {
              return 0, err
          }
          total += item.Precio * float64(item.Cantidad)
      }
      return total, nil
  }

  func aplicarDescuento(total float64, codigoDescuento string) (descuento float64, final float64, err error) {
      switch codigoDescuento {
      case "":
          descuento = 0
      case "BIENVENIDO10":
          descuento = total * 0.10
      case "VIP20":
          descuento = total * 0.20
      case "MITAD":
          descuento = total * 0.50
      default:
          return 0, 0, fmt.Errorf("código de descuento inválido: %s", codigoDescuento)
      }

      if descuento > total {
          descuento = total
      }

      final = total - descuento
      return descuento, final, nil
  }

  func crearPedido(id string, items []Item, codigoDescuento string) (Pedido, error) {
      if id == "" {
          return Pedido{}, fmt.Errorf("el ID del pedido es requerido")
      }

      total, err := calcularTotal(items)
      if err != nil {
          return Pedido{}, fmt.Errorf("error calculando total: %v", err)
      }

      descuento, final, err := aplicarDescuento(total, codigoDescuento)
      if err != nil {
          return Pedido{}, fmt.Errorf("error aplicando descuento: %v", err)
      }

      return Pedido{
          ID:        id,
          Items:     items,
          Total:     total,
          Descuento: descuento,
          Final:     final,
          CreadoEn:  time.Now(),
      }, nil
  }

  func main() {
      items := []Item{
          {"Laptop", 1200.00, 1},
          {"Mouse", 25.50, 2},
          {"Teclado", 80.00, 1},
      }

      pedido, err := crearPedido("PED-001", items, "VIP20")
      if err != nil {
          fmt.Println("Error creando pedido:", err)
          return
      }

      fmt.Printf("Pedido: %s\n", pedido.ID)
      fmt.Printf("Items: %d\n", len(pedido.Items))
      fmt.Printf("Total: $%.2f\n", pedido.Total)
      fmt.Printf("Descuento: -$%.2f\n", pedido.Descuento)
      fmt.Printf("Final: $%.2f\n", pedido.Final)
      fmt.Printf("Creado: %s\n", pedido.CreadoEn.Format("15:04:05"))
  }
  ```

  15.14 PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Por qué Go usa múltiples retornos en lugar de excepciones?
  R: Go prioriza la simplicidad y el manejo explícito de errores.
     Las excepciones ocultan el flujo de error; los múltiples retornos
     lo hacen visible y obligan al programador a manejarlo.

  P: ¿Cuántos valores puede retornar una función?
  R: Técnicamente no hay límite en el compilador, pero por claridad
     se recomienda no retornar más de 3-4 valores. Para más, usa un
     struct.

  P: ¿Qué pasa si no capturo todos los retornos?
  R: El código no compila. Debes capturar todos o usar _ para
     ignorar los que no necesitas.

  P: ¿Puedo usar retornos nombrados solo para algunos valores?
  R: No. O nombras todos o ninguno. La sintaxis es (a, b tipo) para
     anónimos o (a, b tipo) para nombrados.

  P: ¿Debo siempre verificar el error primero?
  R: Sí. Primero verifica err != nil, luego usa el valor. Esta es
     una convención estándar en Go.

  P: ¿Puedo retornar un error siendo nil?
  R: Sí. err = nil indica que la operación fue exitosa.

  P: ¿Cuándo debo usar retornos nombrados vs anónimos?
  R: Usa nombrados cuando el significado de cada retorno no sea
     obvio, o cuando uses return desnudo. Usa anónimos en funciones
     simples.

  P: ¿Existe alguna función en la stdlib que retorne más de 3 valores?
  R: Sí, por ejemplo strings.Cut retorna (before, after, found).
     Pero es raro ver más de 3 valores en la stdlib.
