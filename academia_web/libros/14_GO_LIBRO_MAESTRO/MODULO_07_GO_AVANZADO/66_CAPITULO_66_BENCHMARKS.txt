  ── 1. INTRODUCCIÓN A BENCHMARKS ──

  Un benchmark en Go es una función especial en un archivo _test.go:

  func BenchmarkXxx(b *testing.B) {
      for i := 0; i < b.N; i++ {
          // Código a medir
      }
  }

  El bucle for con b.N es esencial. El framework de testing ajusta b.N
  automáticamente para obtener una medición estable (típicamente ~1s).

  ── 2. EJECUTAR BENCHMARKS ──

  # Ejecutar todos los benchmarks del paquete
  go test -bench=.

  # Benchmark específico
  go test -bench=BenchmarkSuma

  # Verbose
  go test -bench=. -v

  # Múltiples benchmarks
  go test -bench="Suma|Resta"

  # Sin tests (solo benchmarks)
  go test -bench=. -run=^

  ── 3. EJEMPLO BÁSICO ──

  // --- calc.go ---
  package calc

  func Suma(n int) int {
      total := 0
      for i := 0; i < n; i++ {
          total += i
      }
      return total
  }

  func SumaGauss(n int) int {
      return n * (n + 1) / 2
  }

  // --- calc_test.go ---
  package calc

  import "testing"

  func BenchmarkSuma(b *testing.B) {
      for i := 0; i < b.N; i++ {
          Suma(1000)
      }
  }

  func BenchmarkSumaGauss(b *testing.B) {
      for i := 0; i < b.N; i++ {
          SumaGauss(1000)
      }
  }

  // Salida típica:
  // BenchmarkSuma-8         1000000     1250 ns/op
  // BenchmarkSumaGauss-8   1000000000   0.33 ns/op

  ── 4. B.N: AJUSTE AUTOMÁTICO ──

  El framework de testing ajusta b.N automáticamente:

  1. Comienza con un valor pequeño (ej. 1).
  2. Mide cuánto tarda una iteración.
  3. Calcula cuántas iteraciones caben en ~1 segundo.
  4. Ejecuta el benchmark completo con ese b.N.

  Puedes ver el valor de b.N con -v:

  go test -bench=. -v

  // Output:
  // BenchmarkSuma-8   1000000   1250 ns/op
  //        ^ b.N = 1,000,000

  ── 5. RESET Y START/STOP TIMER ──

  Para evitar que el setup contamine la medición:

  func BenchmarkProcess(b *testing.B) {
      // Setup costoso
      data := generateLargeData(10000)

      b.ResetTimer() // Reinicia el contador, ignorando el setup

      for i := 0; i < b.N; i++ {
          process(data)
      }
  }

  Para pausar/reanudar el timer (útil si hay setup en cada iteración):

  func BenchmarkWithSetup(b *testing.B) {
      for i := 0; i < b.N; i++ {
          b.StopTimer()
          data := generateData()
          b.StartTimer()

          process(data)
      }
  }

  ── 6. BENCHMEM: MEDIR MEMORIA ──

  El flag -benchmem muestra asignaciones de memoria:

  go test -bench=. -benchmem

  // Output:
  // BenchmarkConcat-8    500000    5120 ns/op    1024 B/op    1 allocs/op
  //              ^iter    ^ns/op    ^bytes/op     ^allocs/op

  B/op: bytes asignados por operación.
  allocs/op: número de asignaciones por operación.

  Ejemplo comparando concatenación de strings:

  func BenchmarkConcatPlus(b *testing.B) {
      s := ""
      for i := 0; i < b.N; i++ {
          s += "a"
      }
      _ = s
  }

  func BenchmarkConcatBuilder(b *testing.B) {
      var sb strings.Builder
      for i := 0; i < b.N; i++ {
          sb.WriteString("a")
      }
      _ = sb.String()
  }

  // Resultados típicos:
  // ConcatPlus:   alta memoria, muchas allocs
  // ConcatBuilder: baja memoria, pocas allocs

  ── 7. BENCHTIME: CONTROLAR DURACIÓN ──

  Por defecto, cada benchmark dura ~1 segundo. Puedes cambiarlo:

  go test -bench=. -benchtime=5s   # 5 segundos
  go test -bench=. -benchtime=100x # Exactamente 100 iteraciones

  Útil cuando:
  ● Quieres más precisión (benchtime mayor).
  ● El benchmark es muy lento (benchtime menor o iteraciones fijas).

  ── 8. COMPARAR BENCHMARKS ──

  Para comparar dos algoritmos, ejecuta ambos y calcula la diferencia:

  func BenchmarkFast(b *testing.B) {
      for i := 0; i < b.N; i++ {
          FastAlgorithm()
      }
  }

  func BenchmarkSlow(b *testing.B) {
      for i := 0; i < b.N; i++ {
          SlowAlgorithm()
      }
  }

  go test -bench=. -benchtime=1000x

  También puedes usar benchstat (herramienta externa) para análisis
  estadístico:

  go get golang.org/x/perf/cmd/benchstat
  go test -bench=. -count=10 > results.txt
  benchstat results.txt

  benchstat calcula medias, desviaciones, y detecta regresiones.

  ── 10. EVITAR OPTIMIZACIONES DEL COMPILADOR ──

  El compilador puede optimizar código no usado, sesgando benchmarks:

  // ❌ El compilador puede eliminar esto si no se usa el resultado
  func BenchmarkBad(b *testing.B) {
      for i := 0; i < b.N; i++ {
          Suma(1000) // Resultado descartado
      }
  }

  // ✅ Forzar uso del resultado
  func BenchmarkGood(b *testing.B) {
      var result int
      for i := 0; i < b.N; i++ {
          result = Suma(1000)
      }
      _ = result // El compilador no eliminará la asignación
  }

  // ✅ O mejor: asignar a variable global (más seguro)
  var globalResult int

  func BenchmarkBest(b *testing.B) {
      var result int
      for i := 0; i < b.N; i++ {
          result = Suma(1000)
      }
      globalResult = result
  }

  Para funciones que devuelven slices, usa:

  var globalSlice []byte

  func BenchmarkAlloc(b *testing.B) {
      for i := 0; i < b.N; i++ {
          s := make([]byte, 1024)
          globalSlice = s
      }
  }

  ── 11. PROFILING DESDE BENCHMARKS ──

  Puedes generar perfiles de CPU y memoria desde benchmarks:

  # Perfil de CPU
  go test -bench=. -cpuprofile=cpu.out

  # Perfil de memoria (heap)
  go test -bench=. -memprofile=mem.out

  # Perfil de bloqueo
  go test -bench=. -blockprofile=block.out

  # Perfil de goroutines
  go test -bench=. -goroutineprofile=goroutine.out

  # Ver el perfil
  go tool pprof cpu.out
  (pprof) top        # Funciones más costosas
  (pprof) web        # Visualizar gráfico
  (pprof) list Func  # Ver código línea por línea

  # Perfil de memoria interactivo
  go tool pprof -alloc_space mem.out
  go tool pprof -inuse_space mem.out

  ── 12. EJEMPLO COMPLETO: COMPARACIÓN DE ALGORITMOS ──

  // --- sort.go ---
  package sortalgo

  func BubbleSort(arr []int) {
      n := len(arr)
      for i := 0; i < n-1; i++ {
          for j := 0; j < n-i-1; j++ {
              if arr[j] > arr[j+1] {
                  arr[j], arr[j+1] = arr[j+1], arr[j]
              }
          }
      }
  }

  func QuickSort(arr []int) {
      if len(arr) < 2 {
          return
      }
      pivot := arr[0]
      left, right := 0, len(arr)-1
      for i := 1; i <= right; {
          if arr[i] < pivot {
              arr[i], arr[left] = arr[left], arr[i]
              left++
              i++
          } else if arr[i] > pivot {
              arr[i], arr[right] = arr[right], arr[i]
              right--
          } else {
              i++
          }
      }
      QuickSort(arr[:left])
      QuickSort(arr[right+1:])
  }

  // --- sort_test.go ---
  package sortalgo

  import (
      "math/rand"
      "testing"
  )

  func generateSlice(n int) []int {
      s := make([]int, n)
      for i := range s {
          s[i] = rand.Intn(n)
      }
      return s
  }

  func BenchmarkBubbleSort(b *testing.B) {
      sizes := []int{10, 100, 1000}
      for _, size := range sizes {
          b.Run(fmt.Sprintf("n=%d", size), func(b *testing.B) {
              for i := 0; i < b.N; i++ {
                  b.StopTimer()
                  data := generateSlice(size)
                  b.StartTimer()
                  BubbleSort(data)
              }
          })
      }
  }

  func BenchmarkQuickSort(b *testing.B) {
      sizes := []int{10, 100, 1000, 10000}
      for _, size := range sizes {
          b.Run(fmt.Sprintf("n=%d", size), func(b *testing.B) {
              for i := 0; i < b.N; i++ {
                  b.StopTimer()
                  data := generateSlice(size)
                  b.StartTimer()
                  QuickSort(data)
              }
          })
      }
  }

  ── 13. BENCHMARK DE CONCURRENCIA ──

  func BenchmarkParallel(b *testing.B) {
      b.RunParallel(func(pb *testing.PB) {
          for pb.Next() {
              // Código ejecutado en paralelo
              expensiveOperation()
          }
      })
  }

  b.RunParallel crea múltiples goroutines (tantas como GOMAXPROCS) y
  distribuye las iteraciones entre ellas. Es ideal para medir
  rendimiento de operaciones concurrentes.

  func BenchmarkParallelSum(b *testing.B) {
      data := generateSlice(1000)
      b.ResetTimer()

      b.RunParallel(func(pb *testing.PB) {
          for pb.Next() {
              sum := 0
              for _, v := range data {
                  sum += v
              }
              globalResult = sum
          }
      })
  }

  ── 15. COMPARATIVA: STRING CONCATENATION ──

  // --- strconcat.go ---
  package strconcat

  func ConcatPlus(parts []string) string {
      result := ""
      for _, p := range parts {
          result += p
      }
      return result
  }

  func ConcatBuilder(parts []string) string {
      var sb strings.Builder
      for _, p := range parts {
          sb.WriteString(p)
      }
      return sb.String()
  }

  func ConcatJoin(parts []string) string {
      return strings.Join(parts, "")
  }

  func ConcatSprintf(parts []string) string {
      result := ""
      for _, p := range parts {
          result = fmt.Sprintf("%s%s", result, p)
      }
      return result
  }

  // --- strconcat_test.go ---
  package strconcat

  import (
      "testing"
  )

  func BenchmarkConcat(b *testing.B) {
      parts := []string{"hello", " ", "world", " ", "from", " ", "Go"}
      sizes := []int{10, 100, 1000}

      methods := []struct {
          name string
          fn   func([]string) string
      }{
          {"Plus", ConcatPlus},
          {"Builder", ConcatBuilder},
          {"Join", ConcatJoin},
          {"Sprintf", ConcatSprintf},
      }

      for _, size := range sizes {
          for _, m := range methods {
              b.Run(fmt.Sprintf("%s_%d", m.name, size), func(b *testing.B) {
                  // Preparar datos del tamaño correcto
                  data := make([]string, size)
                  for i := range data {
                      data[i] = parts[i%len(parts)]
                  }
                  b.ResetTimer()

                  for i := 0; i < b.N; i++ {
                      m.fn(data)
                  }
              })
          }
      }
  }

  ── 21. COMPARATIVA CON benchstat ──

  # Instalar benchstat
  go install golang.org/x/perf/cmd/benchstat@latest

  # Ejecutar benchmarks 10 veces para obtener datos significativos
  go test -bench=. -count=10 -benchmem > bench.txt

  # Analizar con benchstat
  benchstat bench.txt

  # Comparar dos versiones del código
  git checkout main
  go test -bench=. -count=10 > old.txt
  git checkout feature
  go test -bench=. -count=10 > new.txt
  benchstat old.txt new.txt

  benchstat muestra:
  ● Media y desviación de cada benchmark.
  ● Diferencia porcentual entre versiones (p-value).
  ● Señalización de cambios significativos.

  ── 22. BUENAS PRÁCTICAS DE BENCHMARKS ──

  1. Siempre usa el bucle for con b.N (no pongas un número fijo).
  2. Reinicia el timer después del setup con b.ResetTimer().
  3. Asigna resultados a variables globales para evitar optimizaciones.
  4. Usa b.RunParallel para medir rendimiento concurrente.
  5. Ejecuta benchmarks varias veces con -count para datos estables.
  6. Mide memoria con -benchmem para entender asignaciones.
  7. Usa benchstat para comparar resultados estadísticamente.
  8. No optimices basado en una sola ejecución; busca tendencias.
  9. Aísla el benchmark: minimiza trabajo fuera del bucle.
  10. Documenta qué estás midiendo con nombres descriptivos.

  PREGUNTAS FRECUENTES (FAQ)
  ──────────────────────────
  P: ¿Por qué mi benchmark da resultados inconsistentes?
  R: Posibles causas: recolector de basura, throttling térmico, otros
     procesos. Usa -count=10 para obtener una muestra grande.

  P: ¿Debo ejecutar benchmarks con -race?
  R: No. -race añade overhead y distorsiona las mediciones. Ejecuta
     benchmarks sin -race.

  P: ¿Cómo mido solo el benchmark, sin tests?
  R: go test -bench=. -run=^$

  P: ¿Qué significa ns/op, B/op, allocs/op?
  R: ns/op = nanosegundos por operación (menor es mejor). B/op = bytes
     asignados por operación. allocs/op = asignaciones por operación.

  P: ¿benchmem afecta el rendimiento?
  R: Muy poco. La medición de memoria es precisa y el overhead es
     mínimo.
