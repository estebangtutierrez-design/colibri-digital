1. EFFECTIVE GO
───────────────

Effective Go (go.dev/doc/effective_go) es la guía oficial de estilo.
Sus principios fundamentales:

1.1. Formato — gofmt

gofmt es obligatorio. No hay discusión. No hay configuración.
gofmt es la ley.

```bash
# Formatear un archivo
gofmt -w main.go

# Formatear todo el proyecto
gofmt -w ./...

# Ver diferencias sin aplicar
gofmt -d main.go

# goimports también ordena imports
goimports -w ./...
```

Integración con VS Code: settings.json

```json
{
    "go.formatTool": "gofmt",
    "go.formatOnSave": true,
    "editor.formatOnSave": true,
    "[go]": {
        "editor.codeActionsOnSave": {
            "source.organizeImports": true
        }
    }
}
```

1.2. Comentarios

```go
// Package math proporciona constantes y funciones matemáticas
// básicas. Sigue la notación estándar de IEEE 754.
package math

// MaxInt64 es el valor máximo de un int64.
const MaxInt64 = 1<<63 - 1

// Sqrt devuelve la raíz cuadrada de x.
// Para valores negativos, devuelve NaN.
func Sqrt(x float64) float64 {
    // implementación
    return 0
}
```

Reglas de comentarios:
- Los comentarios de paquete empiezan con "Package <nombre>".
- Los comentarios de función/documento empiezan con el nombre.
- Los comentarios son oraciones completas terminadas en punto.
- Explicar el "qué" y el "por qué", no el "cómo".

1.3. Nombres

| Elemento       | Convención          | Ejemplo         |
|---------------|---------------------|-----------------|
| Paquetes      | minúsculas, cortos  | http, fmt, strutil |
| Tipos         | CamelCase           | UserService, Config |
| Funciones     | CamelCase           | CreateUser, ParseJSON |
| Métodos       | CamelCase           | (u *User) Validate() |
| Variables     | camelCase           | userName, configMap |
| Constantes    | CamelCase           | StatusOK, MaxRetries |
| Interfaces    | -er/-able           | Reader, Writer, Sortable |
| Acrónimos     | Mayúsculas          | HTTP, URL, DB, API |

MAL:

```go

var userId int64   // correcto
var user_id int64  // incorrecto en Go
```

```go

f, err := os.Open(filename)
```

2. GOLANGCI-LINT
────────────────

La herramienta definitiva de linting para Go.

Instalación:

```bash
# Instalar
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest

# Verificar
golangci-lint --version

# Ejecutar
golangci-lint run ./...
```

Configuración (.golangci.yml):

```yaml
# .golangci.yml
linters:
  enable:
    - errcheck
    - gosimple
    - govet
    - ineffassign
    - staticcheck
    - typecheck
    - unused
    - bodyclose
    - goconst
    - gocritic
    - gofmt
    - goimports
    - gosec
    - misspell
    - prealloc
    - revive
    - tenv
    - thelper
    - unconvert
    - whitespace

linters-settings:
  errcheck:
    check-type-assertions: true
    check-blank: true
  gocritic:
    enabled-tags:
      - diagnostic
      - experimental
      - opinionated
      - performance
      - style
  revive:
    rules:
      - name: exported
        severity: warning
  gosec:
    excludes:
      - G101  # Potential hardcoded credentials

issues:
  exclude-rules:
    - path: _test\.go
      linters:
        - errcheck
        - gosec
  max-issues-per-linter: 0
  max-same-issues: 0
```

Ejecutar en CI:

```yaml
# GitHub Actions
name: lint
on: [push, pull_request]
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.22'
      - name: golangci-lint
        uses: golangci/golangci-lint-action@v3
        with:
          version: latest
```

```go

func (u *User) SetName(n string) { u.name = n }
```

5. DOCUMENTACIÓN CON GODOC

```go
// Package calculator provee operaciones aritméticas básicas.
//
// La package soporta suma, resta, multiplicación y división.
// Todas las operaciones trabajan con float64.
package calculator

import "errors"

// ErrDivisionByZero se devuelve cuando se intenta dividir por cero.
var ErrDivisionByZero = errors.New("division by zero")

// Sum retorna la suma de dos números.
func Sum(a, b float64) float64 {
    return a + b
}

// Divide retorna la división de a entre b.
// Retorna ErrDivisionByZero si b es cero.
func Divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, ErrDivisionByZero
    }
    return a / b, nil
}
```


11. ERROR HANDLING AVANZADO
───────────────────────────

```go
package service

import (
    "errors"
    "fmt"
)

// Sentinel errors
var ErrNotFound = errors.New("entity not found")

// Custom error type
type ValidationError struct {
    Field   string
    Reason  string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation error: %s: %s", e.Field, e.Reason)
}

// Error wrapping
func (s *Service) GetUser(id int) (*User, error) {
    user, err := s.repo.FindByID(id)
    if err != nil {
        return nil, fmt.Errorf("service: getting user %d: %w", id, err)
    }
    return user, nil
}

// Error checking
func main() {
    _, err := svc.GetUser(42)
    if errors.Is(err, ErrNotFound) {
        fmt.Println("user not found")
    }
    var valErr *ValidationError
    if errors.As(err, &valErr) {
        fmt.Printf("validation: %s: %s\n", valErr.Field, valErr.Reason)
    }
}
```

12.1. Functonal Options (ya visto, pero recordatorio)

```go
type Server struct {
    host    string
    port    int
    timeout time.Duration
}

type Option func(*Server)

func WithHost(host string) Option {
    return func(s *Server) { s.host = host }
}

func WithPort(port int) Option {
    return func(s *Server) { s.port = port }
}

func NewServer(opts ...Option) *Server {
    s := &Server{host: "localhost", port: 8080, timeout: 30 * time.Second}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

12.2. Decorator pattern

```go
type Notifier interface {
    Send(ctx context.Context, msg string) error
}

type LoggingNotifier struct {
    next   Notifier
    logger *slog.Logger
}

func (n *LoggingNotifier) Send(ctx context.Context, msg string) error {
    n.logger.Info("sending notification", "msg", msg)
    err := n.next.Send(ctx, msg)
    if err != nil {
        n.logger.Error("notification failed", "error", err)
    }
    return err
}

type RetryNotifier struct {
    next    Notifier
    retries int
}

func (n *RetryNotifier) Send(ctx context.Context, msg string) error {
    var err error
    for i := 0; i <= n.retries; i++ {
        if err = n.next.Send(ctx, msg); err == nil {
            return nil
        }
    }
    return fmt.Errorf("after %d retries: %w", n.retries, err)
}
```

12.3. Fan-out/Fan-in

```go
func fanOut(ctx context.Context, source <-chan int, n int) []<-chan int {
    channels := make([]<-chan int, n)
    for i := 0; i < n; i++ {
        ch := make(chan int)
        channels[i] = ch
        go func() {
            defer close(ch)
            for v := range source {
                select {
                case ch <- v:
                case <-ctx.Done():
                    return
                }
            }
        }()
    }
    return channels
}

func fanIn(ctx context.Context, channels ...<-chan int) <-chan int {
    out := make(chan int)
    var wg sync.WaitGroup
    for _, ch := range channels {
        wg.Add(1)
        go func(c <-chan int) {
            defer wg.Done()
            for v := range c {
                select {
                case out <- v:
                case <-ctx.Done():
                    return
                }
            }
        }(ch)
    }
    go func() {
        wg.Wait()
        close(out)
    }()
    return out
}
```

13. MANEJO DE CONFIGURACIÓN
───────────────────────────

```go
package config

import (
    "os"
    "strconv"
    "time"
)

type Config struct {
    Port    int           `env:"PORT" default:"8080"`
    Timeout time.Duration `env:"TIMEOUT" default:"30s"`
    DB      DBConfig
}

type DBConfig struct {
    DSN         string `env:"DB_DSN" required:"true"`
    MaxConns    int    `env:"DB_MAX_CONNS" default:"25"`
    MaxIdle     int    `env:"DB_MAX_IDLE" default:"5"`
}

func Load() *Config {
    return &Config{
        Port:    getEnvInt("PORT", 8080),
        Timeout: getEnvDuration("TIMEOUT", 30*time.Second),
        DB: DBConfig{
            DSN:      getEnv("DB_DSN", "postgres://localhost:5432/db"),
            MaxConns: getEnvInt("DB_MAX_CONNS", 25),
            MaxIdle:  getEnvInt("DB_MAX_IDLE", 5),
        },
    }
}

func getEnv(key, fallback string) string {
    if v := os.Getenv(key); v != "" {
        return v
    }
    return fallback
}

func getEnvInt(key string, fallback int) int {
    if v := os.Getenv(key); v != "" {
        if n, err := strconv.Atoi(v); err == nil {
            return n
        }
    }
    return fallback
}

func getEnvDuration(key string, fallback time.Duration) time.Duration {
    if v := os.Getenv(key); v != "" {
        if d, err := time.ParseDuration(v); err == nil {
            return d
        }
    }
    return fallback
}
```

16. GRACEFUL SHUTDOWN
─────────────────────

```go
package main

import (
    "context"
    "log/slog"
    "net/http"
    "os"
    "os/signal"
    "syscall"
    "time"
)

func main() {
    srv := &http.Server{Addr: ":8080"}

    go func() {
        if err := srv.ListenAndServe(); err != nil &&
            err != http.ErrServerClosed {
            slog.Error("server error", "error", err)
            os.Exit(1)
        }
    }()

    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit

    slog.Info("shutting down gracefully...")
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()

    if err := srv.Shutdown(ctx); err != nil {
        slog.Error("shutdown error", "error", err)
    }
}
```

18.2. strings.Builder

```go
// MAL
s := ""
for _, w := range words {
    s += w + " "
}

// BIEN
var b strings.Builder
b.Grow(len(words) * 10)
for _, w := range words {
    b.WriteString(w)
    b.WriteByte(' ')
}
s := b.String()
```

PREGUNTAS FRECUENTES (FAQ)
──────────────────────────

P: ¿Debo usar gofmt o goimports?
R: goimports (incluye gofmt + ordena imports). Es el estándar.

P: ¿golangci-lint ralentiza mi IDE?
R: La configuración por defecto puede ser lenta. Usa
   --fast para desarrollo y el full análisis en CI.

P: ¿Debo escribir comentarios en cada función?
R: Solo para funciones exportadas (públicas). Las funciones
   privadas pueden documentarse si la lógica no es obvia.

P: ¿Cuánto testing es suficiente?
R: Mínimo 70% de cobertura. Pero la calidad importa más que
   el número: prioriza casos borde y error paths.

P: ¿Debo seguir todas las reglas de Effective Go al pie
   de la letra?
R: Sí, a menos que tengas una razón muy convincente. La
   consistencia del ecosistema Go es su superpoder.
