2. RESUMEN DEL VIAJE — 80 CAPÍTULOS
────────────────────────────────────

MÓDULO I: FUNDAMENTOS (Capítulos 1-10)
• Instalación, GOPATH, módulos, tipos básicos.
• Variables, constantes, control de flujo.
• Funciones, múltiples retornos, defer.
• Paquetes, exports, init().
• Structs, métodos, punteros vs valores.

MÓDULO II: ESTRUCTURAS DE DATOS (Capítulos 11-20)
• Arrays, slices, maps internos.
• Strings, runas, encoding.
• Interfaces, type assertion, empty interface.
• Genéricos (Go 1.18+), constraints.
• JSON, XML, serialización.

MÓDULO III: POO EN GO (Capítulos 21-30)
• Composición vs herencia.
• Embedding de structs e interfaces.
• Polimorfismo con interfaces.
• Manejo de errores, panic/recover.
• Paquete io, interfaces Reader/Writer.

MÓDULO IV: CONCURRENCIA (Capítulos 31-40)
• Goroutines, scheduler, GOMAXPROCS.
• Channels, buffer, select.
• sync.Mutex, RWMutex, WaitGroup.
• sync.Once, sync.Pool, atomic.
• Patrones de concurrencia, context.

MÓDULO V: TESTING (Capítulos 41-50)
• testing.T, table-driven tests.
• Subtests, parallel tests.
• Benchmarks, profiling.
• Fuzzing, coverage.
• Test doubles, mocking, httptest.

MÓDULO VI: APIs Y REDES (Capítulos 51-60)
• net/http, servidores, clientes.
• Middleware, routers.
• gRPC, Protocol Buffers.
• WebSockets, SSE.
• Microservicios, service mesh.

MÓDULO VII: PATRONES (Capítulos 61-70)
• Creacionales: singleton, factory, builder.
• Estructurales: adapter, decorator, proxy.
• Comportamiento: strategy, observer, command.
• Arquitectura: clean, hexagonal, CQRS.
• Domain-Driven Design en Go.

MÓDULO VIII: PROYECTO (Capítulos 71-80)
• Arquitectura en Go, DI, Wire.
• SOLID, diseño, table-driven tests.
• Debugging con Delve.
• Logging: log, slog, zap.
• Optimización: pprof, escape analysis.
• Seguridad: bcrypt, JWT, OWASP.
• Examen profesional.
• Proyecto final: Task Manager Pro.
• Buenas prácticas, golangci-lint.
• Graduación y carta del profesor.
