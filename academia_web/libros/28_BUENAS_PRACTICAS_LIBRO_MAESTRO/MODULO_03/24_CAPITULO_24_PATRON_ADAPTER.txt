CONTENIDO DEL CAPÍTULO
===============================================================================

1. ADAPTER

Convierte interfaz de una clase en otra interfaz que el cliente espera.

Permite que clases trabajen juntas aunque tengan interfaces incompatibles.

Wrapper que traduce llamadas.



2. ESTRUCTURA

Target: interfaz que cliente usa. Adaptee: interfaz existente incompatible.

Adapter: implementa Target y envuelve Adaptee. Puede ser de objeto (composición)

o de clase (herencia múltiple).



3. CUÁNDO USAR

Integrar biblioteca externa con interfaz diferente. Unificar diferentes

sistemas (pagos, notificaciones). Legacy code con interfaz antigua.

Reutilizar código existente con interfaz incompatible.



4. EJEMPLO

Sistema de notificaciones con interfaz send(title, body). Integrar Pushover

(sendPush(apiKey, message, title)). Adapter traduce llamadas.

EJEMPLO DE CÓDIGO
----------------------------------------

# Adapter Pattern
# Target: interfaz que esperamos
class Notification:
  def send(self, title, body): pass

# Adaptee: biblioteca externa
class PushoverAPI:
  def send_push(self, api_key, message, title):
    print(f"Pushover: {title} - {message}")

# Adapter
class PushoverAdapter(Notification):
  def __init__(self, api_key):
    self.api_key = api_key
    self.pushover = PushoverAPI()
  
  def send(self, title, body):
    self.pushover.send_push(self.api_key, body, title)

# Otro adaptee
class EmailAPI:
  def send_email(self, to, subject, html):
    print(f"Email to {to}: {subject}")

class EmailAdapter(Notification):
  def __init__(self, to):
    self.to = to
    self.email = EmailAPI()
  
  def send(self, title, body):
    self.email.send_email(self.to, title, body)

# Uso unificado
notifications = [PushoverAdapter('key'), EmailAdapter('user@test.com')]
for n in notifications:
  n.send('Alerta', 'Servidor caido')
