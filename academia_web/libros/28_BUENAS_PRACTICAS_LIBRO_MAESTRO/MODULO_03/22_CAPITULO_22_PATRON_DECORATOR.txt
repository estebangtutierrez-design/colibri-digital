1. DECORATOR

Añade responsabilidades a un objeto dinámicamente. Alternativa flexible a

la herencia para extender funcionalidad. Envuelve objeto original.



2. ESTRUCTURA

Component: interfaz común. ConcreteComponent: objeto base. Decorator: envuelve

Component, delega y añade comportamiento. ConcreteDecorator: implementaciones.

Múltiples decorators apilables.



3. CUÁNDO USAR

Necesidad de añadir funcionalidad a objetos individuales dinámicamente.

Evitar explosión de subclases. Funcionalidades que se pueden combinar.

Ej: logging, caché, compresión, cifrado, validación.



4. EJEMPLO

DataService que lee archivos. Decorators añaden: compresión, cifrado,

caché. Se pueden apilar en cualquier orden. Sin modificar DataService.

# Decorator Pattern
from functools import wraps

def log_execution(func):
  @wraps(func)
  def wrapper(*args, **kwargs):
    print(f"Ejecutando {func.__name__}")
    result = func(*args, **kwargs)
    print(f"Completado {func.__name__}")
    return result
  return wrapper

def measure_time(func):
  @wraps(func)
  def wrapper(*args, **kwargs):
    import time
    start = time.time()
    result = func(*args, **kwargs)
    print(f"{func.__name__}: {time.time()-start:.2f}s")
    return result
  return wrapper

@log_execution
@measure_time
def process_data(data):
  return [x * 2 for x in data]

# Clase Decorator
class DataSource:
  def read(self): pass

class FileDataSource(DataSource):
  def read(self): return "data"

class CompressionDecorator(DataSource):
  def __init__(self, source): self.source = source
  def read(self):
    return f"compressed({self.source.read()})"
