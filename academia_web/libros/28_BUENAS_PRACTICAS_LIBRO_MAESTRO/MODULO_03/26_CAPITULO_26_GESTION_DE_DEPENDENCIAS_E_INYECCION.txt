CONTENIDO DEL CAPÍTULO
===============================================================================

1. INYECCIÓN DE DEPENDENCIAS

Pasar dependencias desde fuera. Constructor injection: parámetros en

constructor. Setter injection: método setter. Interface injection: interfaz

que recibe dependencia. Favorece bajo acoplamiento y testabilidad.



2. CONTENEDORES DI

Spring (Java), Guice (Java), .NET Core DI, Dagger (Android), InversifyJS

(Node), FastAPI (Python). Configuran y resuelven dependencias automáticamente.



3. SERVICE LOCATOR (ANTI-PATRÓN)

Centraliza obtención de servicios. Anti-patrón porque oculta dependencias.

Dificulta testing (estado global). Preferir DI explícita.



4. PRINCIPIO DE HOLLYWOOD

'No nos llames, nosotros te llamamos'. El framework controla el flujo.

Inversión de control (IoC). DI es una forma de IoC.

EJEMPLO DE CÓDIGO
----------------------------------------

# Sin DI
class EmailService:
  def send(self, msg):
    return smtplib.SMTP().send(msg)

class UserService:
  def __init__(self):
    self.email = EmailService()  # acoplado

# Con DI
class UserService:
  def __init__(self, email_service):  # inyectado
    self.email = email_service

# Testing con DI
def test_register():
  mock_email = MagicMock()
  service = UserService(mock_email)
  service.register('test')
  mock_email.send.assert_called_once()

# Contenedor DI (FastAPI)
from fastapi import Depends, FastAPI

def get_db():
  return Database()

@app.get("/users")
def get_users(db = Depends(get_db)):
  return db.get_users()
