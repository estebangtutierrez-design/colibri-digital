CONTENIDO DEL CAPÍTULO
===============================================================================

1. FACTORY METHOD

Define interfaz para crear objeto, pero permite a subclases decidir qué

clase instanciar. La creación se delega a subclases. Útil cuando no se

sabe la clase exacta a crear hasta runtime.



2. ABSTRACT FACTORY

Proporciona interfaz para crear familias de objetos relacionados sin

especificar sus clases concretas. Ej: UI Factory para diferentes SO

(Windows, macOS, Linux) con botones, inputs, menús.



3. CUÁNDO USAR

Clases no se conocen en tiempo de compilación. Subclases deciden qué crear.

Creación compleja que merece centralizarse. Familias de objetos relacionados.



4. EJEMPLO

Procesador de pagos: PaymentFactory.crear('tarjeta'), crear('paypal'),

crear('transferencia'). Cada método crea procesador específico.

EJEMPLO DE CÓDIGO
----------------------------------------

# Factory Method
def create_payment_processor(type):
  processors = {
    'card': CardProcessor(),
    'paypal': PayPalProcessor(),
    'transfer': TransferProcessor(),
  }
  return processors.get(type)

# Uso
processor = create_payment_processor('card')
processor.process(amount=100)

# Abstract Factory
class UIFactory(ABC):
  @abstractmethod
  def create_button(self): pass
  @abstractmethod
  def create_input(self): pass

class WindowsFactory(UIFactory):
  def create_button(self): return WindowsButton()
  def create_input(self): return WindowsInput()

class MacFactory(UIFactory):
  def create_button(self): return MacButton()
  def create_input(self): return MacInput()

# Uso
factory = WindowsFactory() if os.name == 'nt' else MacFactory()
btn = factory.create_button()
