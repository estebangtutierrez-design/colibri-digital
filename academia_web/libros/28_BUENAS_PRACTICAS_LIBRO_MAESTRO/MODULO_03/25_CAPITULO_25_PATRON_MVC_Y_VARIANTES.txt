1. MVC (MODEL-VIEW-CONTROLLER)

Separa datos (Model), presentación (View) y lógica (Controller). Model:

datos y reglas de negocio. View: interfaz de usuario. Controller: maneja

input del usuario y actualiza Model/View.



2. MVP (MODEL-VIEW-PRESENTER)

Presenter en lugar de Controller. Presenter maneja toda la lógica de UI.

View es pasiva (solo muestra datos). Fácil de testear (View mockeable).



3. MVVM (MODEL-VIEW-VIEWMODEL)

ViewModel expone datos y comandos como propiedades. Data binding automático.

View se suscribe a cambios. Angular, Vue, WPF usan este patrón.

React: similar con estado + props.



4. FLUX/REDUX

Arquitectura unidireccional. Action → Dispatcher → Store → View. Estado

centralizado inmutable. Acciones describen cambios. Reducers calculan

nuevo estado. Predecible y trazable.

EJEMPLO DE CÓDIGO
----------------------------------------

# MVC
# Model: datos + logica negocio
class User:
  def __init__(self, name):
    self.name = name

# View: presentacion
class UserView:
  def show(self, user):
    print(f"Usuario: {user.name}")

# Controller: gestiona interaccion
class UserController:
  def __init__(self, model, view):
    self.model = model
    self.view = view
  
  def update_name(self, new_name):
    self.model.name = new_name
    self.view.show(self.model)

# Redux (JS)
// Action
{ type: 'ADD_TODO', payload: { text: 'Learn Redux' } }
// Reducer
const todos = (state = [], action) => {
  switch (action.type) {
    case 'ADD_TODO':
      return [...state, action.payload]
    default: return state
  }
}
