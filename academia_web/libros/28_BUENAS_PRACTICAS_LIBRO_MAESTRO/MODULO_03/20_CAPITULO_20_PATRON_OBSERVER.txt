CONTENIDO DEL CAPÍTULO
===============================================================================

1. OBSERVER

Define dependencia uno-a-muchos. Cuando objeto cambia estado, notifica a

sus dependientes automáticamente. Event-driven programming.



2. ESTRUCTURA

Subject: mantiene lista de observers, métodos attach/detach/notify.

Observer: interfaz con update(). ConcreteObserver: implementa update().

Subject conoce Observer solo por interfaz (bajo acoplamiento).



3. CUÁNDO USAR

Cambios en un objeto requieren cambios en otros (sin conocerlos).

Sistema de eventos: UI, notificaciones, mensajería. Publicación/suscripción.



4. EJEMPLO

Notificaciones de eventos del sistema: usuario registrado, orden creada.

EmailNotification, SMSNotification, PushNotification como observers.

Nuevos canales de notificación sin modificar UserService.

EJEMPLO DE CÓDIGO
----------------------------------------

# Observer Pattern
class EventManager:
  def __init__(self):
    self.listeners = {}
  
  def subscribe(self, event_type, listener):
    if event_type not in self.listeners:
      self.listeners[event_type] = []
    self.listeners[event_type].append(listener)
  
  def unsubscribe(self, event_type, listener):
    self.listeners[event_type].remove(listener)
  
  def notify(self, event_type, data):
    for listener in self.listeners.get(event_type, []):
      listener.update(data)

class EmailListener:
  def update(self, data):
    print(f"Email: {data}")

class SMSListener:
  def update(self, data):
    print(f"SMS: {data}")

# Uso
events = EventManager()
events.subscribe('user.created', EmailListener())
events.subscribe('user.created', SMSListener())
events.notify('user.created', {'email': 'user@test.com'})
