CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES DEUDA TÉCNICA?

Ward Cunningham: código que se apresura en entregar, como deuda financiera.

Intereses: más difícil añadir features, bugs frecuentes, onboarding lento.

Tipos: deliberada (consciente) vs inadvertida (sin saber).



2. CAUSAS COMUNES

Presión de tiempo. Falta de conocimiento. Procesos inmaduros. Falta de

code review. Tests insuficientes. Stack legacy. Monolitos gigantes.



3. PRIORIZACIÓN

Deuda crítica: bugs en producción. Alta: impide features nuevas. Media:

ralentiza desarrollo. Baja: mejora estética. Cuadrante impacto/esfuerzo.



4. ESTRATEGIAS

Boy Scout Rule: deja el código mejor de lo que lo encontraste. 20% time

para refactorización. Sprint de deuda técnica. Refactorizar al tocar

(cuando modificas, mejora). Automatizar con linters y formateadores.
