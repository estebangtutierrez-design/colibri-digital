CONTENIDO DEL CAPÍTULO
===============================================================================

1. CODE SMELLS COMUNES

Funciones largas (>30 líneas). Clases grandes (>200 líneas). Muchos

parámetros (>4). Duplicación. Código muerto. Comentarios excesivos.

Envidia de características (Feature Envy). Switch/if por tipo.



2. OLOR A CÓDIGO

Shotgun Surgery: un cambio afecta muchas clases. Divergent Change: una

clase cambia por múltiples razones (viola SRP). Parallel Inheritance

Hierarchies: jerarquías paralelas. Middle Man: delegación excesiva.



3. DATA CLUMPS

Grupos de datos que aparecen juntos repetidamente. Crear clase para el

grupo. Ej: dirección (calle, ciudad, CP). Rango (inicio, fin).



4. PRIMITIVE OBSESSION

Obsesión por tipos primitivos. Usar objetos de valor en vez de strings

y números. Email, PhoneNumber, Money, Color, DateRange.

EJEMPLO DE CÓDIGO
----------------------------------------

# Code smells comunes

# Funcion larga: >30 lineas
# Clase grande: >200 lineas
# Muchos parametros: >4
# Switch por tipo: polimorfismo
# Feature Envy: metodo usa mas datos de otra clase

# Data clump
# Mal:
def create_order(customer_name, customer_email, customer_phone):
  pass

# Bien:
def create_order(customer: Customer):
  pass

# Primitive obsession
# Mal:
email = "user@example.com"
if '@' in email:
  send_email(email)

# Bien:
email = Email("user@example.com")
email.validate()
notifier.send(email)
