CONTENIDO DEL CAPÍTULO
===============================================================================

1. IDENTIFICAR SECCIONES

Funciones largas tienen múltiples secciones lógicas. Cada sección hace

una cosa: validar, procesar, formatear, enviar. Extraer cada sección.



2. EXTRAER CONDICIONALES

Condicionales complejos a funciones con nombre: if is_valid_user(user).

if has_permission(user, 'edit'). en vez de if user and user.active and...



3. REDUCIR PARÁMETROS

Más de 4 parámetros → crear objeto parámetro. Introducir Parameter Object.

Usar objetos en vez de listas de parámetros. Config objects.



4. ELIMINAR FLAGS

Flags booleanos en parámetros indican múltiples responsabilidades. Dividir

función: sendEmail() y sendEmailWithAttachment(). En vez de sendEmail(attach).

EJEMPLO DE CÓDIGO
----------------------------------------

# Reducir parametros
# Antes
def create_user(name, email, phone, address, city, country, zip_code):
  pass

# Despues
class Address:
  def __init__(self, street, city, country, zip_code):
    pass

def create_user(name, email, phone, address: Address):
  pass

# Eliminar flags
# Antes
def send_email(to, subject, body, attach_report=False):
  if attach_report:
    # ...

# Despues
def send_email(to, subject, body):
  pass

def send_email_with_report(to, subject, body, report):
  send_email(to, subject, body)
  # attach report

# Funciones de menos de 20 lineas
# 1 funcion = 1 responsabilidad
