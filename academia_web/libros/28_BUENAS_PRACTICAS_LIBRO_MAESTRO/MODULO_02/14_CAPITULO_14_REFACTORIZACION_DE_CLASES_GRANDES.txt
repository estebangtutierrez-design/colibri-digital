CONTENIDO DEL CAPÍTULO
===============================================================================

1. EXTRAER CLASE

Grupo de métodos/datos que pueden separarse. Crear nueva clase con

responsabilidad específica. Componer en lugar de acumular.



2. EXTRAER INTERFAZ

Si múltiples clientes usan diferentes subconjuntos. Crear interfaces

específicas (ISP). Clientes dependen de interfaces pequeñas.



3. DIVIDIR MÉTODOS POR RESPONSABILIDAD

Métodos de validación → UserValidator. Métodos de persistencia →

UserRepository. Métodos de notificación → NotificationService.

UserService coordina pero delega.



4. STRATEGY PATTERN

Algoritmos intercambiables dentro de clase grande. Estrategias para

cálculos de precios, descuentos, impuestos. Fácil de extender sin

modificar.
