1. PIRÁMIDE DE TESTS

Base: tests unitarios (muchos, rápidos). Medio: tests de integración.

Arriba: tests E2E (pocos, lentos, frágiles). Proporción: 70/20/10.



2. TDD (TEST-DRIVEN DEVELOPMENT)

1. Escribir test que falla. 2. Escribir código mínimo para pasar test.

3. Refactorizar. Ciclo red-green-refactor. Garantiza código testeable.



3. UNIT TESTING

Testean una unidad aislada (función, clase). Mocks para dependencias.

Rápidos (<100ms). Sin red/BD/IO. Frameworks: Jest (JS), pytest (Python).



4. INTEGRATION TESTING

Testean interacción entre componentes. BD real, API real, sin mocks.

Más lentos pero más fiables. Prueban que componentes funcionan juntos.

Contract tests entre servicios.

EJEMPLO DE CÓDIGO
----------------------------------------

# TDD: Red-Green-Refactor
# 1. RED: Escribir test que falla
def test_calculate_total():
  assert calculate_total([10, 20]) == 30

# 2. GREEN: Codigo minimo
def calculate_total(items):
  return sum(items)

# 3. REFACTOR: Mejorar sin romper tests
# (codigo ya funciona, puede optimizarse)

# Piramide de tests
#     /
#    / E2E (5-10%)
#   / ---------
#  / Integration (20-30%)
# / ---------------
# Unit tests (60-70%)
# --------------------
#
# Unit: < 100ms, sin IO
# Integration: < 2s, BD real
# E2E: < 30s, browser real
