CONTENIDO DEL CAPÍTULO
===============================================================================

1. TESTING DE APIs REST

Probar endpoints: GET, POST, PUT, DELETE. Códigos de estado. Body response.

Headers. Autenticación. Validación de entrada. Paginación. Errores.



2. HERRAMIENTAS

Supertest (Node), pytest + requests (Python), REST Assured (Java).

Postman/Newman para tests de integración. Schemas de respuesta (JSON Schema).



3. CONTRACT TESTING

Pact: consumer-driven contracts. Consumidor define expectativas. Provider

verifica contrato. Sin necesidad de integración real. Evolución segura.



4. TESTING DE SEGURIDAD

SQL injection: enviar ' OR 1=1. XSS: <script>alert(1)</script>. Auth:

token inválido/expirado. Rate limiting: muchas peticiones. Input malformed.

EJEMPLO DE CÓDIGO
----------------------------------------

# Testing con Supertest (Node)
describe('GET /api/users', () => {
  it('should return users list', async () => {
    const res = await request(app)
      .get('/api/users')
      .expect(200)
    
    expect(res.body).toHaveProperty('data')
    expect(Array.isArray(res.body.data)).toBe(true)
  })
  
  it('should require auth', async () => {
    await request(app)
      .get('/api/users')
      .expect(401)
  })
})

# Testing con pytest + requests
def test_create_user():
  response = client.post("/api/users", json={
    "name": "Test",
    "email": "test@test.com"
  })
  assert response.status_code == 201
  assert response.json()["name"] == "Test"
