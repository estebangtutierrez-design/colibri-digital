CONTENIDO DEL CAPÍTULO
===============================================================================

1. UNIT TESTING FRONTEND

Testear funciones puras: utilidades, helpers, selectores. Sin DOM.

Rápidos y simples. Jest + Vitest para lógica.



2. COMPONENT TESTING

Testear componentes de UI. Renderizar, interactuar, verificar output.

React Testing Library, Vue Test Utils, Angular TestBed. Enfocado en

comportamiento de usuario, no implementación.



3. E2E TESTING

Playwright, Cypress. Browser real. Flujos completos: login, navegación,

CRUD. Assertion en UI real. Video de ejecución. Screenshots.



4. VISUAL REGRESSION

Comparar screenshots de componentes antes/después. Detectar cambios

visuales no intencionados. Percy, Chromatic, Loki. Automatizado en CI.

EJEMPLO DE CÓDIGO
----------------------------------------

# Component testing (React Testing Library)
import { render, screen, fireEvent } from '@testing-library/react'

test('button click triggers callback', () => {
  const handleClick = jest.fn()
  render(<Button onClick={handleClick}>Click me</Button>)
  
  fireEvent.click(screen.getByText('Click me'))
  expect(handleClick).toHaveBeenCalledTimes(1)
})

# E2E con Playwright
test('login flow', async ({ page }) => {
  await page.goto('http://localhost:3000/login')
  await page.fill('[name="email"]', 'user@test.com')
  await page.fill('[name="password"]', 'password123')
  await page.click('button[type="submit"]')
  await expect(page).toHaveURL(/dashboard/)
  await expect(page.locator('.welcome')).toContainText('Welcome')
})
