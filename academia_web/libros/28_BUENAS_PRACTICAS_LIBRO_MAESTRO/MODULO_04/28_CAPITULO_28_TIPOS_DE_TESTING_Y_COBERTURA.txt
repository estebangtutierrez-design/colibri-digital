CONTENIDO DEL CAPÍTULO
===============================================================================

1. UNITARIOS

Testean unidades individuales. Sin IO/BD/red. Mocks para dependencias.

Rápidos (<100ms). Framework: Jest, pytest, JUnit, NUnit.



2. INTEGRACIÓN

Testean interacción entre componentes. BD real o testcontainers.

Más lentos. Validan que componentes funcionan juntos.



3. E2E (END-TO-END)

Testean flujo completo del sistema. Browser real (Cypress, Playwright).

Lentos y frágiles. Pocos pero críticos. Validan requisitos de negocio.



4. COBERTURA

Porcentaje de código ejercitado por tests. Líneas, ramas, funciones.

Mínimo 80%. No obsesionarse con 100%. Priorizar lógica crítica.

EJEMPLO DE CÓDIGO
----------------------------------------

# Cobertura de codigo
def calcular_precio(base, descuento, impuesto):
  # Linea 1
  precio = base - (base * descuento)  # Linea 2
  total = precio + (precio * impuesto)  # Linea 3
  return total  # Linea 4

# Test cubre solo linea 1 y 4?
test calcular_precio_sin_descuento()  # no ejecuta lineas 2-3
# Cobertura: 50% (2/4 lineas)

# Mejor:
test calcular_precio_con_descuento()
test calcular_precio_con_impuesto()
test calcular_precio_completo()
# Cobertura: 100%

# Reportes de cobertura
# pytest --cov=src tests/
# Jest --coverage
# Los reportes muestran lineas no cubiertas
