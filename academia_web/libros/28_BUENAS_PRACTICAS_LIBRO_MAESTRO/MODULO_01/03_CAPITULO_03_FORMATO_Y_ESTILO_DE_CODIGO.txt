CONTENIDO DEL CAPÍTULO
===============================================================================

1. FORMATEADORES

Prettier (JS/TS). Black (Python). gofmt (Go). rustfmt (Rust). ESLint +

Prettier para JS/TS. Ruff para Python. EditorConfig para consistencia.



2. INDENTACIÓN Y ESPACIOS

2 espacios (JS/TS/Go), 4 espacios (Python/Java). No mezclar tabs y espacios.

Espacios alrededor de operadores. Coma después de último elemento en listas.

Línea máxima: 80-120 caracteres (depende del lenguaje).



3. LÍNEAS EN BLANCO

2 líneas entre funciones. 1 línea entre métodos. Separar secciones lógicas.

No dejar líneas en blanco al inicio/fin de bloques. Agrupar imports:

estándar, terceros, locales (separados por línea en blanco).



4. CONFIGURACIÓN .editorconfig

root = true. [*] charset = utf-8, end_of_line = lf, insert_final_newline = true.

[*.{js,ts}] indent_style = space, indent_size = 2. [*.py] indent_size = 4.

EJEMPLO DE CÓDIGO
----------------------------------------

# .editorconfig
root = true

[*]
charset = utf-8
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true

[*.{js,ts,jsx,tsx,css}]
indent_style = space
indent_size = 2

[*.py]
indent_style = space
indent_size = 4

[*.md]
trim_trailing_whitespace = false

# Orden de imports
# 1. Modulos estandar
import os
import sys
# 2. Terceros
import flask
import requests
# 3. Locales
from models import User
