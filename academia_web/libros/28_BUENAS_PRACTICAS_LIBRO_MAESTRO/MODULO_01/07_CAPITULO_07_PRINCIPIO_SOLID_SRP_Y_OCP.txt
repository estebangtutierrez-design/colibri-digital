1. SRP (SINGLE RESPONSIBILITY PRINCIPLE)

Una clase debe tener una única razón para cambiar. Cada clase/módulo debe

tener una responsabilidad bien definida. No mezclar lógica de negocio con

infraestructura. Separar concerns.



2. EJEMPLO SRP

UserService no debe enviar emails ni generar PDFs. EmailService envía emails.

PDFService genera PDFs. UserService solo gestiona usuarios. Composición.



3. OCP (OPEN/CLOSED PRINCIPLE)

Abierto para extensión, cerrado para modificación. Añadir funcionalidad sin

modificar código existente. Usar herencia, interfaces, composición.

Strategy pattern para algoritmos intercambiables.



4. EJEMPLO OCP

Shape con método area(). Square, Circle heredan y implementan area().

Nuevas formas se añaden sin modificar Shape. No hay switch/if por tipo.

EJEMPLO DE CÓDIGO
----------------------------------------

# SRP: Separar responsabilidades
class UserManager:
  def create_user(self, data):
    # Solo crea usuario
    return User(**data)

class EmailService:
  def send_welcome(self, user):
    # Solo envia email
    pass

# OCP: Abierto a extension
from abc import ABC, abstractmethod
class Shape(ABC):
  @abstractmethod
  def area(self): pass

class Square(Shape):
  def __init__(self, side): self.side = side
  def area(self): return self.side ** 2

class Circle(Shape):
  def __init__(self, radius): self.radius = radius
  def area(self): return math.pi * self.radius ** 2

# Nueva forma no modifica Shape
class Triangle(Shape):
  def __init__(self, b, h): self.b = b; self.h = h
  def area(self): return 0.5 * self.b * self.h
