CONTENIDO DEL CAPÍTULO
===============================================================================

1. KISS

La simplicidad es la máxima sofisticación. La mayoría de los sistemas

funcionan mejor cuando se mantienen simples. Las soluciones simples son más

fáciles de entender, probar, mantener y extender.



2. SEÑALES DE COMPLEJIDAD

Muchas capas de abstracción. Patrones innecesarios. Configuración excesiva.

Framework sobrecargado. Muchas dependencias. Código difícil de seguir.

Funciones de 100+ líneas. Clases gigantes.



3. SIMPLICIDAD EN LA PRÁCTICA

YAGNI: no añadas funcionalidad hasta que sea necesaria. Menos código es

mejor código. La solución más simple es la que tiene menos partes móviles.

Preferir bibliotecas estándar sobre frameworks complejos.



4. BALANCE

Simplicidad no es simplismo. No sacrificar calidad por simplicidad.

Una abstracción bien diseñada reduce la complejidad general. El código

simple se escribe rápido y se lee para siempre.

EJEMPLO DE CÓDIGO
----------------------------------------

# Complejo (mal)
def process(data, opts):
  class ProcessorFactory:
    @staticmethod
    def create(type):
      if type == 'a':
        return AProcessor()
      elif type == 'b':
        return BProcessor()
  factory = ProcessorFactory()
  processor = factory.create(opts.get('type', 'a'))
  return processor.process(data)

# Simple (bien)
def process(data, type='a'):
  processors = {
    'a': lambda d: d * 2,
    'b': lambda d: d ** 2,
  }
  return processors[type](data)

# No uses un framework cuando una funcion basta
# No hagas una clase cuando una funcion basta
