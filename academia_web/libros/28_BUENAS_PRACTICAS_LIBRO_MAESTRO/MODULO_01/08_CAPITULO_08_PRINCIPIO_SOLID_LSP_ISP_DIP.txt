CONTENIDO DEL CAPÍTULO
===============================================================================

1. LSP (LISKOV SUBSTITUTION PRINCIPLE)

Una clase derivada debe poder sustituir a su clase base sin alterar el

comportamiento esperado. No heredar si la subclase no puede hacer todo

lo que hace la superclase. Preferir composición sobre herencia.



2. ISP (INTERFACE SEGREGATION PRINCIPLE)

Muchas interfaces específicas son mejores que una interfaz general. Los

clientes no deben depender de interfaces que no usan. Interfaces pequeñas

y cohesionadas. Mejor tener interfaces pequeñas separadas.



3. DIP (DEPENDENCY INVERSION PRINCIPLE)

Módulos de alto nivel no deben depender de módulos de bajo nivel. Ambos

deben depender de abstracciones. Las abstracciones no deben depender de

detalles. Los detalles deben depender de abstracciones.



4. INYECCIÓN DE DEPENDENCIAS

Pasar dependencias desde fuera (constructor, método, setter). No crear

dentro de la clase. Facilita testing (mocks). Promueve bajo acoplamiento.

EJEMPLO DE CÓDIGO
----------------------------------------

# LSP: Incorrecto
class Bird:
  def fly(self): pass
class Penguin(Bird):
  def fly(self): raise Exception("No vuela")  # Viola LSP

# ISP: Correcto
class Workable(ABC):
  @abstractmethod
  def work(self): pass
class Eatable(ABC):
  @abstractmethod
  def eat(self): pass
class Human(Workable, Eatable): pass
class Robot(Workable): pass  # Robot no necesita eat()

# DIP + DI
class UserService:
  def __init__(self, db: DatabaseInterface):
    self.db = db  # Depende de abstraccion, no de implementacion

class PostgresDB(DatabaseInterface): pass
class MySQLDB(DatabaseInterface): pass
