CONTENIDO DEL CAPÍTULO
===============================================================================

1. DRY

Evitar duplicación de conocimiento. Cada pieza debe tener una representación

única, sin ambigüedad. Código duplicado = mantenimiento duplicado.



2. SEÑALES DE DUPLICACIÓN

Copy-paste de código. Lógica similar en múltiples lugares. Validaciones

repetidas. Strings hardcodeados repetidos. Constantes mágicas repetidas.



3. CÓMO EVITAR DRY

Extraer funciones: misma lógica en múltiples lugares. Parámetrizar: hacer

función genérica con parámetros. Herencia/composición para clases similares.

Constantes para valores repetidos. Templates para HTML repetitivo.



4. CUIDADO CON OVER-DRY

No forzar abstracciones prematuras. A veces 2 copias son mejor que una

abstracción incorrecta (Rule of Three: abstraelo solo al 3er uso).

La duplicación accidental es diferente de la duplicación esencial.

EJEMPLO DE CÓDIGO
----------------------------------------

# MAL: codigo duplicado
if user.role == 'admin':
  canEdit = True
  canDelete = True
  canCreate = True
if user.role == 'editor':
  canEdit = True
  canDelete = False
  canCreate = True

# BIEN: extraer a funcion
def get_permissions(role):
  permissions = {
    'admin': {'edit': True, 'delete': True, 'create': True},
    'editor': {'edit': True, 'delete': False, 'create': True},
    'viewer': {'edit': False, 'delete': False, 'create': False},
  }
  return permissions.get(role, {})

# Rule of Three: espera a 3 repeticiones antes de abstraer
# Las primeras 2 veces, simplemente copia
