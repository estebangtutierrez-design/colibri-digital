CONTENIDO DEL CAPÍTULO
===============================================================================

1. CLEAN CODE

Código limpio es código fácil de leer, entender y modificar. Robert C. Martin

(Uncle Bob) popularizó el concepto. El código se escribe una vez pero se lee

muchas veces. La legibilidad es la prioridad número uno.



2. NOMBRES SIGNIFICATIVOS

Los nombres deben revelar intención. int d; // dias transcurridos (mal).

int daysSinceLastUpdate; (bien). Evitar abreviaturas: usr → user, txt → text.

bool flag → bool isActive. Nombres buscables y pronunciables.



3. FUNCIONES PEQUEÑAS

Funciones deben hacer una sola cosa (Single Responsibility). Máximo 20-30

líneas. Un nivel de abstracción por función. Nombres verbales: createUser(),

sendEmail(). Sin efectos secundarios. Sin parámetros de salida.



4. COMENTARIOS

El buen código se documenta solo. Comentarios explican 'por qué', no 'qué'.

TODO, FIXME para deudas técnicas. Evitar comentarios obvios: i++ // incrementa i.

Documentar APIs públicas con JSDoc, docstrings, etc.

EJEMPLO DE CÓDIGO

# Mal codigo
def p(d):
  # calcula algo
  r = d * 3.1416
  return r

# Buen codigo
def calcular_area_circulo(radio):
  return math.pi * radio ** 2
