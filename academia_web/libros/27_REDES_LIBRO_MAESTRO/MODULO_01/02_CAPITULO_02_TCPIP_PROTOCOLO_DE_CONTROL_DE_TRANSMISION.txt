CONTENIDO DEL CAPÍTULO
===============================================================================

1. TCP (TRANSMISSION CONTROL PROTOCOL)

Protocolo orientado a conexión. Garantiza entrega ordenada y sin errores.

Three-way handshake: SYN → SYN-ACK → ACK. Control de flujo y congestión.



2. CABECERA TCP

Puerto origen (16 bits), puerto destino (16 bits), número de secuencia,

número de ACK, flags (SYN, ACK, FIN, RST, PSH, URG), checksum.



3. UDP (USER DATAGRAM PROTOCOL)

Protocolo no orientado a conexión. Sin garantía de entrega. Menor latencia.

Usado para streaming, DNS, DHCP, VoIP. Cabecera simple: 8 bytes.



4. PUERTOS

0-1023: Well-known (HTTP 80, HTTPS 443, SSH 22, DNS 53, SMTP 25).

1024-49151: Registered. 49152-65535: Dynamic/Private.

SMTP: 25/TCP
POP3: 110/TCP
IMAP: 143/TCP
MySQL: 3306/TCP
PostgreSQL: 5432/TCP
Redis: 6379/TCP
MongoDB: 27017/TCP
