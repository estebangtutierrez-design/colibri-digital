CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES DNS?

Traduce nombres de dominio (google.com) a direcciones IP (142.250.184.46).

Sistema jerárquico y distribuido. Esencial para el funcionamiento de Internet.



2. JERARQUÍA DNS

Raíz (.) → TLD (.com, .org, .es) → Dominio (google) → Subdominio (mail).

13 servidores raíz. TLD servers. Authoritative name servers.



3. TIPOS DE REGISTROS

A: IPv4. AAAA: IPv6. CNAME: alias. MX: servidores de correo. TXT: texto.

NS: nameservers. SOA: Start of Authority. PTR: reverse DNS.



4. RESOLUCIÓN DNS

1. Consulta a resolver local. 2. Root server. 3. TLD server. 4. Authoritative.

5. Cachea resultado. TTL (Time To Live) controla tiempo de cache.

EJEMPLO DE CÓDIGO
----------------------------------------

# Consultas DNS
dig google.com ANY
nslookup google.com
host google.com

# Tipos de registro
dig google.com A
dig google.com MX
dig google.com TXT

# Ver servidores DNS del sistema
cat /etc/resolv.conf

# Archivo hosts locales
cat /etc/hosts
# 127.0.0.1 localhost
# 192.168.1.10 mi-servidor.local
