CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES VPN?

Red privada virtual. Extiende una red privada a través de una red pública.

Cifra el tráfico entre extremos. Autenticación mutua. Tipos: Site-to-Site,

Remote Access, SSL VPN.



2. PROTOCOLOS VPN

IPsec: estándar, modo túnel y transporte. IKEv2: moderno, móvil.

OpenVPN: SSL/TLS, flexible, multiplataforma. WireGuard: moderno, rápido,

simple. L2TP: combinado con IPsec. PPTP: obsoleto, inseguro.



3. WIREGUARD

Moderno, rápido, simple. Criptografía moderna (Curve25519, ChaCha20).

Menos líneas de código que OpenVPN. Integrado en kernel Linux 5.6+.

Configuración simple: clave privada/pública, endpoint, peers.



4. CASOS DE USO

Acceso remoto a red corporativa. Privacidad en WiFi público. Bypass de

censura. Conexión entre datacenters. Acceso a recursos internos.

EJEMPLO DE CÓDIGO
----------------------------------------

# Wireguard config servidor
# /etc/wireguard/wg0.conf
[Interface]
Address = 10.0.0.1/24
PrivateKey = servidor_private_key
ListenPort = 51820

[Peer]
PublicKey = cliente_public_key
AllowedIPs = 10.0.0.2/32

# Cliente
[Interface]
Address = 10.0.0.2/24
PrivateKey = cliente_private_key

[Peer]
PublicKey = servidor_public_key
Endpoint = server.com:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
