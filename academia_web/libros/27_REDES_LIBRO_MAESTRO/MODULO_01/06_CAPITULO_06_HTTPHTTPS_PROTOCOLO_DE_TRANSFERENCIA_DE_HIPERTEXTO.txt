CONTENIDO DEL CAPÍTULO
===============================================================================

1. HTTP

Protocolo de capa de aplicación. Modelo request-response. Sin estado

(stateless). Versiones: 1.0, 1.1, 2.0, 3.0 (QUIC).



2. MÉTODOS HTTP

GET: obtener recurso. POST: crear recurso. PUT: actualizar/reemplazar.

PATCH: actualización parcial. DELETE: eliminar. HEAD: solo cabeceras.

OPTIONS: métodos disponibles. TRACE: diagnóstico.



3. CÓDIGOS DE ESTADO

1xx: Informativos. 2xx: Éxito (200 OK, 201 Created, 204 No Content).

3xx: Redirección (301 Moved, 304 Not Modified). 4xx: Error cliente

(400 Bad, 401 Unauthorized, 403 Forbidden, 404 Not Found). 5xx: Error

servidor (500 Internal, 502 Bad Gateway, 503 Service Unavailable).



4. HTTPS

HTTP sobre TLS. Cifra la comunicación. Certificados digitales. Confianza

mediante CA (Certificate Authority). TLS 1.3 es el estándar actual.

EJEMPLO DE CÓDIGO
----------------------------------------

# Metodos HTTP
GET /api/users
POST /api/users  {"name": "Juan"}
PUT /api/users/1  {"name": "Juan"}
PATCH /api/users/1  {"name": "Juan"}
DELETE /api/users/1

# Codigos comunes
200 OK
201 Created
301 Moved Permanently
400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found
500 Internal Server Error
502 Bad Gateway
503 Service Unavailable
