CONTENIDO DEL CAPÍTULO
===============================================================================

1. RADIUS

Remote Authentication Dial-In User Service. AAA para redes. Puerto 1812

(auth) y 1813 (accounting). Cliente/servidor. Secreto compartido.



2. TACACS+

Protocolo Cisco. Separa autenticación, autorización y auditoría. Puerto

49 TCP. Cifra todo el tráfico. Más flexible que RADIUS.



3. LDAP

Lightweight Directory Access Protocol. Directorio de usuarios. OpenLDAP,

389 Directory Server. Active Directory usa LDAP + Kerberos.



4. 802.1X

Autenticación a nivel de puerto de switch. EAP (Extensible Authentication

Protocol). RADIUS como servidor backend. WPA2-Enterprise usa 802.1X.

EJEMPLO DE CÓDIGO
----------------------------------------

# FreeRADIUS config
# /etc/freeradius/3.0/users
"user" Cleartext-Password := "password"
  Reply-Message = "Hello, user"

# /etc/freeradius/3.0/clients.conf
client localhost {
  ipaddr = 127.0.0.1
  secret = testing123
}

# 802.1X con hostapd
# /etc/hostapd/hostapd.conf
auth_algs=1
eapol_version=2
ieee8021x=1
auth_server_addr=127.0.0.1
auth_server_port=1812
auth_server_shared_secret=testing123
