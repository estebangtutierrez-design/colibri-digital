CONTENIDO DEL CAPÍTULO
===============================================================================

1. DESAFÍOS IoT

Dispositivos con recursos limitados. Falta de actualizaciones. Protocolos

inseguros (MQTT sin TLS). Gran superficie de ataque. Botnets (Mirai).



2. PROTOCOLOS IoT

MQTT: publicación/suscripción ligero. CoAP: UDP para dispositivos.

AMQP: mensajería empresarial. HTTP/2 para dispositivos más capaces.



3. SEGURIDAD IoT

TLS mutual (mTLS). Certificados de dispositivo. Firmas de firmware.

Secure boot. TPM (Trusted Platform Module). Actualizaciones OTA.

Segmentación de red IoT (VLAN separada).



4. REDES IoT

LPWAN: LoRaWAN, NB-IoT (bajo consumo, largo alcance). Zigbee, Z-Wave

(hogar). Bluetooth Low Energy (proximidad). 5G (alta velocidad).

EJEMPLO DE CÓDIGO
----------------------------------------

# MQTT con TLS
mosquitto_pub -h broker.example.com -p 8883 --cafile ca.crt \
  --cert client.crt --key client.key -t "sensor/temp" -m "25.5"

# Segmentacion IoT
# VLAN separada para IoT
interface VLAN300
 description IoT
 ip address 192.168.300.1 255.255.255.0

# Reglas firewall: IoT solo puede hablar con servidor MQTT
iptables -A FORWARD -i vlan300 -o vlan200 -p tcp --dport 8883 -j ACCEPT
iptables -A FORWARD -i vlan300 -o any -j DROP

# MQTT ACLs
$ mosquitto_passwd -c /etc/mosquitto/passwd sensor1
$ cat /etc/mosquitto/acl
user sensor1
topic write sensor/+/temp
topic read server/+/command
