CONTENIDO DEL CAPÍTULO
===============================================================================

1. IDS/IPS

Intrusion Detection System: detecta y alerta. Intrusion Prevention System:

detecta y bloquea. Snort y Suricata son los más populares.



2. FIRMAS Y ANOMALÍAS

Signature-based: detecta patrones conocidos. Anomaly-based: detecta

comportamiento anómalo. Ambos tienen fortalezas y debilidades.



3. SNORT/SURICATA

Reglas de detección: action protocol src_ip src_port → dst_ip dst_port.

alert tcp $HOME_NET any -> $EXTERNAL_NET 80 (msg: "HTTP"; sid:1;).

Suricata: multihilo, GPU acceleration, HTTP/2, TLS inspection.



4. ZEEK (BRO)

Framework de análisis de tráfico. No solo alertas, sino generación de

logs estructurados: conexiones, DNS, HTTP, SSL. Análisis posterior.

EJEMPLO DE CÓDIGO
----------------------------------------

# Regla Snort/Suricata
alert tcp $EXTERNAL_NET any -> $HOME_NET 22 (
  msg:"SSH Brute Force Attempt";
  flow:to_server;
  detection_filter:track by_src, count 5, seconds 30;
  sid:1000001;
  rev:1;)

# Suricata config basica
af-packet:
  - interface: eth0

# Ejecutar Suricata
suricata -c /etc/suricata/suricata.yaml -i eth0

# Ver alertas
tail -f /var/log/suricata/fast.log

# Zeek (Bro)
zeek -i eth0
cat /var/log/zeek/conn.log
