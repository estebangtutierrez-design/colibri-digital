CONTENIDO DEL CAPÍTULO
===============================================================================

1. SSL/TLS

Secure Sockets Layer / Transport Layer Security. Cifra comunicación entre

cliente y servidor. TLS 1.3 (RFC 8446) es el estándar actual. Handshake

autenticado y cifrado.



2. CERTIFICADOS

X.509: estándar de certificados. CA (Certificate Authority) emite y firma.

Chain: Root CA → Intermediate CA → Certificado. Auto-firmados para desarrollo.



3. LET'S ENCRYPT

CA gratuita y automatizada. Certbot para obtener y renovar certificados.

Validación HTTP-01 (archivo en servidor web) o DNS-01 (registro TXT en DNS).

Renovación automática cada 90 días.



4. OPENSSL

openssl genrsa -b 2048 genera clave. openssl req -new -x509 auto-firmado.

openssl s_client -connect host:443 verifica certificado remoto.

openssl x509 -text -in cert.pem inspecciona certificado.

EJEMPLO DE CÓDIGO
----------------------------------------

# Obtener certificado Let's Encrypt
certbot certonly --standalone -d example.com -d www.example.com

# Renovacion automatica
certbot renew --dry-run
# Anadir a crontab: 0 3 * * * certbot renew --quiet

# Inspeccionar certificado
openssl x509 -in /etc/letsencrypt/live/example.com/fullchain.pem -text -noout

# Ver conexion TLS
openssl s_client -connect google.com:443 -tls1_3

# Generar self-signed para desarrollo
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes
