CONTENIDO DEL CAPÍTULO
===============================================================================

1. LDAP (LIGHTWEIGHT DIRECTORY ACCESS PROTOCOL)

Protocolo para acceder a directorios de información. Jerárquico. Árbol DIT

(Directory Information Tree). Entradas con atributos. Estándar X.500.



2. ESTRUCTURA

dc=dominio,dc=com (base). ou=people, ou=groups (unidades organizativas).

cn=usuario (common name). uid=user (user id). Atributos: mail, uid, sn.



3. OPENLDAP

slapd: servidor LDAP. ldapsearch: consultas. ldapadd: agregar entradas.

ldappasswd: cambiar contraseña. ACLs para control de acceso.



4. ACTIVE DIRECTORY

Microsoft AD: LDAP + Kerberos + DNS + DHCP. Dominios y bosques.

Políticas de grupo (GPO). SSO con Kerberos. Integración con Linux (SSSD).

EJEMPLO DE CÓDIGO
----------------------------------------

# OpenLDAP: agregar entrada
cat << EOF | ldapadd -x -D cn=admin,dc=example,dc=com -W
dn: cn=Juan Perez,ou=people,dc=example,dc=com
objectClass: inetOrgPerson
cn: Juan Perez
sn: Perez
mail: juan@example.com
uid: jperez
telephoneNumber: +123456789
EOF

# Buscar usuarios
ldapsearch -x -b dc=example,dc=com '(uid=jperez)'

# SSSD para Linux + AD
# /etc/sssd/sssd.conf
[domain/ad.example.com]
id_provider = ad
auth_provider = ad
access_provider = ad

# Autenticar con AD
realm join ad.example.com
