CONTENIDO DEL CAPÍTULO
===============================================================================

1. TIPOS DE FIREWALL

Packet Filter (capa 3/4): filtra por IP, puerto, protocolo. Stateful:

mantiene estado de conexiones. Application (capa 7): inspecciona contenido.

Next-gen: IDS/IPS, antivirus, filtrado URL.



2. POLÍTICAS DE FIREWALL

Whitelist: permitir solo lo explícito (más seguro). Blacklist: bloquear

lo explícito. Default deny: todo bloqueado por defecto. Default allow:

todo permitido (menos seguro).



3. DMZ (ZONA DESMILITARIZADA)

Red separada para servidores públicos. Firewall de 3 patas: WAN, LAN, DMZ.

Servidores web/email en DMZ. Base de datos en LAN. Acceso controlado.



4. FW EN LINUX

ufw: frontend simple para iptables. firewalld: zones y servicios.

iptables/nftables: control granular. Fail2ban: bloqueo por intentos.

EJEMPLO DE CÓDIGO
----------------------------------------

# ufw basico
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable

# firewalld
firewall-cmd --zone=public --add-service=http --permanent
firewall-cmd --zone=public --add-port=3000/tcp --permanent
firewall-cmd --reload

# Fail2ban
# /etc/fail2ban/jail.local
[sshd]
enabled = true
port = 22
maxretry = 3
bantime = 3600
