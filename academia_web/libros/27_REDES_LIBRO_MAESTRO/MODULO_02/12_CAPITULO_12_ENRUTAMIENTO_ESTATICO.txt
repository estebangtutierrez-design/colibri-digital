CONTENIDO DEL CAPÍTULO
===============================================================================

1. TABLA DE RUTAS

ip route show o netstat -rn. Destino, gateway, máscara, interfaz, métrica.

Ruta por defecto: 0.0.0.0/0 → gateway. Rutas específicas para redes.



2. AÑADIR RUTAS

ip route add 10.0.0.0/24 via 192.168.1.1 dev eth0. route add -net

10.0.0.0 netmask 255.255.255.0 gw 192.168.1.1. Ruta por defecto:

ip route add default via 192.168.1.1.



3. RUTEO ENTRE INTERFACES

Router necesita IP en cada red. ip_forward=1. iptables para NAT si

es necesario. RIP, OSPF para enrutamiento dinámico.



4. METRICAS Y MÚLTIPLES RUTAS

ip route add 10.0.0.0/24 via 192.168.1.1 metric 100. Menor métrica =

preferida. Balanceo de carga ECMP con múltiples rutas misma métrica.

EJEMPLO DE CÓDIGO
----------------------------------------

# Tabla de rutas
$ ip route show
default via 192.168.1.1 dev eth0
10.0.0.0/24 via 10.0.1.1 dev eth1
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.10

# Añadir rutas
ip route add 10.0.0.0/24 via 10.0.1.1
ip route add 172.16.0.0/16 via 10.0.1.1 metric 200
ip route add default via 192.168.1.1

# Activar IP forwarding
echo 1 > /proc/sys/net/ipv4/ip_forward
# O persistente: /etc/sysctl.conf
net.ipv4.ip_forward=1
