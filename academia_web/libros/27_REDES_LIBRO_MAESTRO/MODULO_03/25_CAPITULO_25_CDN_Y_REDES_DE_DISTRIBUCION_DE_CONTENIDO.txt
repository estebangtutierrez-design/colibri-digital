CONTENIDO DEL CAPÍTULO
===============================================================================

1. CDN (CONTENT DELIVERY NETWORK)

Red distribuida de servidores que entregan contenido desde ubicaciones

cercanas al usuario. Reduce latencia, descarga servidor origen, mejora

disponibilidad. Ejemplos: CloudFlare, Akamai, Fastly.



2. CÓMO FUNCIONA

DNS resuelve a IP del servidor CDN más cercano. Content caching en edge.

Anycast: misma IP desde múltiples ubicaciones. Pull: CDN obtiene contenido

del origen. Push: contenido subido directamente a CDN.



3. BENEFICIOS

Menor latencia (TTFB). Mayor throughput. Protección DDoS. SSL/TLS.

Optimización de imágenes. Minificación. Gzip/Brotli. HTTP/2 y HTTP/3.



4. CONFIGURACIÓN

Registrar dominio en CDN. Configurar DNS (CNAME). Reglas de caché.

Purge cache. Reglas de seguridad (WAF). Analytics de tráfico.

EJEMPLO DE CÓDIGO
----------------------------------------

# CloudFlare: configurar DNS
# Tipo: CNAME
# Nombre: www
# Destino: www.ejemplo.com
# Proxy: Activado (nube naranja)

# nginx + CDN
# Confiar en CDN para IP real
set_real_ip_from 103.21.244.0/22;
real_ip_header CF-Connecting-IP;

# Reglas de cache en CDN
# Cache static: 30 dias
# Cache HTML: 0 (no cachear)
# Cache API: 1 minuto

# Edge Workers (CloudFlare Workers)
addEventListener('fetch', event => {
  event.respondWith(handleRequest(event.request))
})
