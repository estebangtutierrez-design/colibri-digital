CONTENIDO DEL CAPÍTULO
===============================================================================

1. SDN (SOFTWARE-DEFINED NETWORKING)

Separa plano de control (software) de plano de datos (hardware).

Controlador centralizado. Programable. Flexible. OpenFlow protocolo.



2. COMPONENTES

Aplicaciones SDN, controlador SDN, dispositivos de red (OpenFlow).

Controladores: OpenDaylight, ONOS, Ryu, POX. Northbound APIs (REST).

Southbound APIs (OpenFlow, NETCONF, OVSDB).



3. OPEN VSWITCH (OVS)

Switch virtual open source. Usado en KVM, Docker, Kubernetes. OpenFlow.

VLANs, túneles (VXLAN, GRE, GENEVE). Open vSwitch Database (OVSDB).



4. NFV (NETWORK FUNCTION VIRTUALIZATION)

Virtualizar funciones de red: firewall, router, load balancer, IDS/IPS.

Ejecutan en hardware estándar. Escalables. Menor CAPEX.

EJEMPLO DE CÓDIGO
----------------------------------------

# Open vSwitch
ovs-vsctl add-br ovs-br0
ovs-vsctl add-port ovs-br0 eth0
ovs-vsctl add-port ovs-br0 veth0 -- set interface veth0 type=internal
ovs-vsctl set-controller ovs-br0 tcp:192.168.1.10:6633

# VXLAN con OVS
ovs-vsctl add-port ovs-br0 vxlan0 -- set interface vxlan0 type=vxlan \
  options:remote_ip=10.0.0.2 options:key=100

# Ver estado
ovs-vsctl show
ovs-ofctl dump-flows ovs-br0
