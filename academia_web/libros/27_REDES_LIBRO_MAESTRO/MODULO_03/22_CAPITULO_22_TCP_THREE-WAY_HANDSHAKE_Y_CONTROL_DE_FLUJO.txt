CONTENIDO DEL CAPÍTULO
===============================================================================

1. THREE-WAY HANDSHAKE

SYN: cliente envía número de secuencia inicial (ISN). SYN-ACK: servidor

responde con su ISN y ACK del cliente. ACK: cliente confirma.

Establece conexión bidireccional.



2. CONTROL DE FLUJO

Ventana deslizante (window size). Tamaño de ventana: cuántos bytes puede

enviar sin ACK. Window scaling para ventanas grandes (alto rendimiento).



3. CONTROL DE CONGESTIÓN

Slow Start: aumenta ventana exponencialmente hasta threshold. Congestion

Avoidance: aumento lineal. Fast Retransmit: 3 ACKs duplicados = pérdida.

Fast Recovery. Algoritmos: Reno, Cubic, BBR.



4. CONEXIÓN TCP

ESTABLISHED: conexión activa. TIME_WAIT: espera antes de cerrar (2*MSL).

CLOSE_WAIT: espera cierre de app. FIN-WAIT: espera FIN del otro lado.

EJEMPLO DE CÓDIGO
----------------------------------------

# Ver conexiones TCP
$ ss -tan  # todas las conexiones
$ ss -tan state established
$ ss -tan state time-wait
$ ss -tln  # listening

# Observar handshake con tcpdump
$ tcpdump -i any port 80 -S
# 1. [SYN] Seq=100
# 2. [SYN, ACK] Seq=200 Ack=101
# 3. [ACK] Seq=101 Ack=201

# Ver estadisticas TCP
$ netstat -s | grep -i tcp
$ nstat -az | grep Tcp
