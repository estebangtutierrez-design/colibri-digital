CONTENIDO DEL CAPÍTULO
===============================================================================

1. ICMP (INTERNET CONTROL MESSAGE PROTOCOL)

Protocolo de control y diagnóstico. No para datos de aplicación. Mensajes:

Echo Request/Reply (ping), Destination Unreachable, Time Exceeded (traceroute).



2. PING

Envía ICMP Echo Request. Mide RTT (Round Trip Time). Verifica conectividad.

ping -c 4 google.com. ping -i 0.5 intervalo. ping -s 1472 tamaño.



3. TRACEROUTE

Rastro de routers hasta destino. Usa TTL (Time To Live). Incrementa TTL

desde 1 hasta destino. cada salto responde Time Exceeded. mtr combina

ping + traceroute en tiempo real.



4. OTRAS HERRAMIENTAS

netstat -tulpn puertos escuchando. ss -tulpn moderno. nmap escaneo.

tcpdump captura paquetes. wireshark análisis gráfico. iperf pruebas

de ancho de banda.

EJEMPLO DE CÓDIGO
----------------------------------------

$ ping -c 4 google.com
$ traceroute google.com
$ mtr google.com
$ nmap -sS 192.168.1.1
$ netstat -tulpn
$ ss -tulpn
$ tcpdump -i eth0 port 80
$ iperf3 -c server -t 10
