CONTENIDO DEL CAPÍTULO
===============================================================================

1. UDP CARACTERÍSTICAS

No orientado a conexión. Sin garantía de entrega. Sin orden. Sin control

de congestión. Baja latencia. Usos: DNS, DHCP, VoIP, streaming, gaming.



2. CABECERA UDP

Puerto origen (16 bits), puerto destino (16 bits), longitud (16 bits),

checksum (16 bits). Total: 8 bytes. Sin opciones ni flags.



3. APLICACIONES DE TIEMPO REAL

VoIP (SIP, RTP): audio/video en tiempo real. Streaming: video bajo demanda.

Gaming online: baja latencia crítica. DNS: consultas rápidas.



4. CALIDAD DE SERVICIO (QoS)

Priorización de tráfico sensible. DSCP (Differentiated Services Code Point).

Colas: FIFO, PQ, WFQ, CBQ. Limitación de ancho de banda. Evitar congestión.

EJEMPLO DE CÓDIGO
----------------------------------------

# UDP puertos comunes
DNS: 53
DHCP: 67, 68
SNMP: 161, 162
NTP: 123
TFTP: 69
RTP: 16384-32786 (VoIP)

# Ver sockets UDP
$ ss -uan
$ netstat -uan

# Prueba de rendimiento UDP
iperf3 -c server -u -b 100M

# QoS con tc (traffic control)
tc qdisc add dev eth0 root handle 1: htb default 30
tc class add dev eth0 parent 1: classid 1:1 htb rate 100mbit
tc class add dev eth0 parent 1:1 classid 1:10 htb rate 10mbit ceil 20mbit prio 0
