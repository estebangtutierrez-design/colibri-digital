CONTENIDO DEL CAPÍTULO
===============================================================================

1. PROXY

Servidor intermediario entre cliente y destino. Forward proxy: cliente →

proxy → internet (anónimato, caché, filtrado). Reverse proxy: internet →

proxy → servidor interno (SSL termination, load balancing, caching).



2. REVERSE PROXY (NGINX)

nginx como proxy inverso: proxy_pass, load balancing, SSL termination,

caching, compresión, seguridad. Headers: X-Forwarded-For, X-Real-IP.



3. BALANCEO DE CARGA

Round Robin: distribución circular. Least Connections: al de menos conexiones.

IP Hash: misma IP siempre mismo servidor (sticky sessions). Weighted: con

pesos. Health checks: servidores caídos se retiran.



4. HAPROXY

Balanceador de alto rendimiento. TCP y HTTP. Estadísticas. ACLs para

routing avanzado. Keepalived + VRRP para alta disponibilidad.

EJEMPLO DE CÓDIGO
----------------------------------------

# nginx reverse proxy
upstream backend {
  server 10.0.0.1:3000 weight=3;
  server 10.0.0.2:3000;
  server 10.0.0.3:3000 backup;
}

server {
  listen 80;
  location / {
    proxy_pass http://backend;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}

# HAProxy
frontend http-in
  bind *:80
  default_backend servers

backend servers
  balance roundrobin
  server web1 10.0.0.1:3000 check
  server web2 10.0.0.2:3000 check
