CAPITULO 1: INTRODUCCIÓN A TYPESCRIPT
================================================================================

¿Qué es TypeScript?
-------------------
TypeScript es un superset de JavaScript que añade tipado estático, clases,
interfaces, y otras características orientadas a objetos. Fue desarrollado por
Microsoft y lanzado en 2012. Se compila a JavaScript puro y es ampliamente usado
en aplicaciones grandes y complejas.

Características principales:
- Tipado estático (detecta errores en tiempo de compilación).
- Mejora la autocompletación y documentación (IntelliSense).
- Facilita el mantenimiento y refactorización en proyectos grandes.
- Compatible con todo el ecosistema de JavaScript (se compila a JS).
- Soporte para las últimas características de ECMAScript.
- Amplia adopción en frameworks como Angular, React, Vue, NestJS, etc.

Filosofía: "JavaScript that scales" - JavaScript que escala para grandes equipos
y proyectos.

Versión actual: TypeScript 5.5 (2024). Se actualiza frecuentemente.

CAPITULO 3: TIPOS BÁSICOS Y ANOTACIONES
================================================================================

Tipos primitivos:
-----------------
let nombre: string = "Esteban";
let edad: number = 30;
let activo: boolean = true;
let nulo: null = null;
let indefinido: undefined = undefined;
let simbolo: symbol = Symbol("id");
let big: bigint = 100n;

Any (evitar si es posible - desactiva el tipado):
-------------------------------------------------
let algo: any = "cualquier cosa";
algo = 10;      // OK
algo = true;    // OK

Unknown (alternativa segura a any - requiere verificación):
-----------------------------------------------------------
let valor: unknown = 10;
// valor.toUpperCase(); // Error! Hay que verificar el tipo

if (typeof valor === "string") {
    console.log(valor.toUpperCase());
}

Void (para funciones que no retornan):
--------------------------------------
function saludar(): void {
    console.log("Hola");
}

Never (función que nunca retorna, lanza excepción o bucle infinito):
---------------------------------------------------------------------
function error(mensaje: string): never {
    throw new Error(mensaje);
}

Arrays:
-------
let numeros: number[] = [1, 2, 3];
let letras: Array<string> = ["a", "b", "c"];
let mixto: (string | number)[] = [1, "hola", 2];

Tuplas (array de longitud y tipos fijos):
-----------------------------------------
let par: [number, string] = [1, "uno"];
let usuario: [number, string, boolean] = [1, "Juan", true];

Unión de tipos (Union Types):
-----------------------------
let id: number | string;
id = 10;
id = "diez";

Literal Types (valores específicos):
------------------------------------
let direccion: "norte" | "sur" | "este" | "oeste";
direccion = "norte";
// direccion = "noroeste"; // Error!

Tipo object:
------------
let persona: { nombre: string; edad: number } = {
    nombre: "Ana",
    edad: 25
};

Tipo array de objetos:
-----------------------
let personas: { nombre: string; edad: number }[] = [
    { nombre: "Ana", edad: 25 },
    { nombre: "Luis", edad: 30 }
];

CAPITULO 4: INTERFACES Y TYPE ALIASES
================================================================================

Interfaces (describen la forma de un objeto):
---------------------------------------------
interface Usuario {
    id: number;
    nombre: string;
    email: string;
    telefono?: string;   // Opcional
    readonly fechaRegistro: Date; // Solo lectura
    // Método:
    saludar(): void;
}

function mostrarUsuario(usr: Usuario): void {
    console.log(usr.nombre);
}

const usuario: Usuario = {
    id: 1,
    nombre: "Juan",
    email: "juan@mail.com",
    fechaRegistro: new Date(),
    saludar() {
        console.log(`Hola, soy ${this.nombre}`);
    }
};

Extensión de interfaces (herencia):
-----------------------------------
interface Administrador extends Usuario {
    permisos: string[];
}

Type Aliases (alias de tipos - más flexibles):
-----------------------------------------------
type Punto = { x: number; y: number };
function distancia(p1: Punto, p2: Punto): number {
    return Math.sqrt((p2.x - p1.x) ** 2 + (p2.y - p1.y) ** 2);
}

type ID = number | string;  // Unión
type Evento = "click" | "hover" | "submit"; // Literal union

Interfaz vs Type:
-----------------
- Interface: extensible (se puede reabrir), mejor para objetos y clases.
- Type: puede definir uniones, intersecciones, mapeos, etc.

Index Signature (propiedades dinámicas):
----------------------------------------
interface Diccionario {
    [key: string]: string;
}
const dict: Diccionario = {
    "nombre": "Esteban",
    "ciudad": "Houston"
};

CAPITULO 5: CLASES Y HERENCIA
================================================================================

Clase básica:
-------------
class Persona {
    // Propiedades con modificadores
    public nombre: string;
    private edad: number;      // Solo dentro de la clase
    protected id: number;      // Accesible en subclases
    readonly fechaCreacion: Date = new Date(); // Solo lectura

    constructor(nombre: string, edad: number) {
        this.nombre = nombre;
        this.edad = edad;
        this.id = Math.random();
    }

    saludar(): void {
        console.log(`Hola, soy ${this.nombre}`);
    }

    public getEdad(): number {
        return this.edad;
    }

    // Getter/Setter
    get Edad(): number {
        return this.edad;
    }
    set Edad(valor: number) {
        if (valor > 0) {
            this.edad = valor;
        }
    }
}

Herencia:
---------
class Empleado extends Persona {
    public salario: number;

    constructor(nombre: string, edad: number, salario: number) {
        super(nombre, edad);   // Llamada al constructor padre
        this.salario = salario;
    }

    // Sobrescritura de método
    saludar(): void {
        super.saludar();
        console.log(`Mi salario es ${this.salario}`);
    }

    // Método adicional
    trabajar(): void {
        console.log("Trabajando...");
    }
}

Modificador readonly (solo lectura):
-------------------------------------
class Rectangulo {
    constructor(
        public readonly ancho: number,
        public readonly alto: number
    ) {}
}

Propiedades y métodos estáticos:
--------------------------------
class Contador {
    static total: number = 0;
    static incrementar(): void {
        Contador.total++;
    }
}
Contador.incrementar();
console.log(Contador.total); // 1

Clases abstractas (no se pueden instanciar):
--------------------------------------------
abstract class Figura {
    abstract area(): number;
    getNombre(): string {
        return "Figura";
    }
}

class Circulo extends Figura {
    constructor(private radio: number) {
        super();
    }
    area(): number {
        return Math.PI * this.radio ** 2;
    }
}

CAPITULO 6: GENÉRICOS (GENERICS)
================================================================================

Permiten crear componentes reutilizables que funcionan con varios tipos.

Función genérica:
-----------------
function identidad<T>(valor: T): T {
    return valor;
}
let numero = identidad<number>(5);
let texto = identidad("hola"); // Inferencia de tipo

Clase genérica:
---------------
class Caja<T> {
    private contenido: T;
    constructor(contenido: T) {
        this.contenido = contenido;
    }
    obtener(): T {
        return this.contenido;
    }
    establecer(nuevo: T): void {
        this.contenido = nuevo;
    }
}

let cajaNumeros = new Caja<number>(100);
let cajaString = new Caja("texto"); // inferido como string

Interfaz genérica:
------------------
interface Par<K, V> {
    clave: K;
    valor: V;
}
let par: Par<string, number> = { clave: "edad", valor: 30 };

Restricciones (extends):
------------------------
function longitud<T extends { length: number }>(item: T): number {
    return item.length;
}
longitud("hola");        // OK (string tiene length)
longitud([1, 2, 3]);     // OK (array tiene length)
// longitud(10);         // Error! number no tiene length

Función con clave genérica:
---------------------------
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}
const p = { nombre: "Ana", edad: 25 };
console.log(getProperty(p, "nombre")); // "Ana"

CAPITULO 8: FUNCIONES Y TIPOS DE FUNCIÓN
================================================================================

Tipado de funciones:
--------------------
function sumar(a: number, b: number): number {
    return a + b;
}

Parámetros opcionales:
----------------------
function saludar(nombre: string, saludo?: string): string {
    return saludo ? `${saludo}, ${nombre}` : `Hola, ${nombre}`;
}

Parámetros por defecto:
-----------------------
function multiplicar(a: number, b: number = 2): number {
    return a * b;
}

Parámetros rest (resto de argumentos):
--------------------------------------
function sumarTodos(...numeros: number[]): number {
    return numeros.reduce((acc, n) => acc + n, 0);
}

Tipo de función (callback):
---------------------------
type Operacion = (a: number, b: number) => number;
function ejecutar(operacion: Operacion, a: number, b: number): number {
    return operacion(a, b);
}
ejecutar((a, b) => a + b, 5, 3); // 8

Function overloads (sobrecarga):
--------------------------------
function obtenerInfo(id: number): string;
function obtenerInfo(id: string): string;
function obtenerInfo(id: any): any {
    if (typeof id === "number") {
        return `ID numérico: ${id}`;
    } else {
        return `ID string: ${id}`;
    }
}

Funciones con genéricos:
------------------------
function fusionar<T, U>(a: T, b: U): T & U {
    return { ...a, ...b };
}
const resultado = fusionar({ nombre: "Ana" }, { edad: 25 });

CAPITULO 10: DECORADORES (DECORATORS)
================================================================================

Los decoradores son funciones que modifican clases, métodos, propiedades o
parámetros. (Requieren "experimentalDecorators": true en tsconfig.json).

Decorador de clase:
-------------------
function logClase(constructor: Function) {
    console.log(`Clase creada: ${constructor.name}`);
    // Agregar método estático
    constructor.prototype.nuevoMetodo = function() {
        console.log("Método agregado por decorador");
    };
}

@logClase
class MiClase {
    constructor() {
        console.log("Instancia creada");
    }
}

Decorador de método:
--------------------
function logMetodo(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    const original = descriptor.value;
    descriptor.value = function(...args: any[]) {
        console.log(`Llamando a ${propertyKey} con args: ${args}`);
        const resultado = original.apply(this, args);
        console.log(`Resultado: ${resultado}`);
        return resultado;
    };
    return descriptor;
}

class Ejemplo {
    @logMetodo
    sumar(a: number, b: number): number {
        return a + b;
    }
}

Decorador de propiedad:
-----------------------
function readonly(target: any, propertyKey: string) {
    Object.defineProperty(target, propertyKey, {
        writable: false
    });
}

class Config {
    @readonly
    version: string = "1.0.0";
}

Decorador de parámetro:
-----------------------
function logParam(target: any, propertyKey: string, parameterIndex: number) {
    console.log(`Parámetro ${parameterIndex} de ${propertyKey}`);
}

CAPITULO 11: TYPE GUARDS Y NARROWING
================================================================================

Narrowing es reducir el tipo de una variable a un tipo más específico.

Typeof:
-------
function procesar(valor: string | number): string {
    if (typeof valor === "string") {
        return valor.toUpperCase();
    } else {
        return valor.toFixed(2);
    }
}

Instanceof:
-----------
if (objeto instanceof Date) {
    console.log(objeto.getFullYear());
}

In (verificar propiedad):
-------------------------
if ("nombre" in objeto) {
    console.log(objeto.nombre);
}

User-defined type guards:
-------------------------
function esString(valor: any): valor is string {
    return typeof valor === "string";
}
if (esString(dato)) {
    console.log(dato.toUpperCase());
}

function esArray<T>(valor: any): valor is T[] {
    return Array.isArray(valor);
}

Discriminated unions (uniones discriminadas):
---------------------------------------------
interface Cuadrado {
    tipo: "cuadrado";
    lado: number;
}
interface Circulo {
    tipo: "circulo";
    radio: number;
}
type Figura = Cuadrado | Circulo;

function area(figura: Figura): number {
    if (figura.tipo === "cuadrado") {
        return figura.lado ** 2;
    } else {
        return Math.PI * figura.radio ** 2;
    }
}

El operador as (type assertion - solo compilación):
---------------------------------------------------
let valor = "hola" as any;
let longitud = (valor as string).length;

    // código

        case "decrementar": return { count: state.count - 1 };

const [estado, dispatch] = useReducer(reducer, { count: 0 });

Código (src/index.ts):
----------------------
import express, { Request, Response } from 'express';
import cors from 'cors';

const app = express();
const port = process.env.PORT || 3000;

app.use(cors());
app.use(express.json());

// Interfaces
interface Producto {
    id: number;
    nombre: string;
    precio: number;
}

// Base de datos en memoria
let productos: Producto[] = [
    { id: 1, nombre: "Laptop", precio: 1200 },
    { id: 2, nombre: "Mouse", precio: 25 }
];

// Rutas
app.get('/productos', (req: Request, res: Response) => {
    res.json(productos);
});

app.get('/productos/:id', (req: Request, res: Response) => {
    const id = parseInt(req.params.id);
    const producto = productos.find(p => p.id === id);
    if (!producto) {
        return res.status(404).json({ error: "Producto no encontrado" });
    }
    res.json(producto);
});

app.post('/productos', (req: Request, res: Response) => {
    const { nombre, precio } = req.body;
    const nuevo: Producto = {
        id: productos.length + 1,
        nombre,
        precio
    };
    productos.push(nuevo);
    res.status(201).json(nuevo);
});

app.put('/productos/:id', (req: Request, res: Response) => {
    const id = parseInt(req.params.id);
    const { nombre, precio } = req.body;
    const producto = productos.find(p => p.id === id);
    if (!producto) {
        return res.status(404).json({ error: "Producto no encontrado" });
    }
    producto.nombre = nombre || producto.nombre;
    producto.precio = precio || producto.precio;
    res.json(producto);
});

app.delete('/productos/:id', (req: Request, res: Response) => {
    const id = parseInt(req.params.id);
    const index = productos.findIndex(p => p.id === id);
    if (index === -1) {
        return res.status(404).json({ error: "Producto no encontrado" });
    }
    productos.splice(index, 1);
    res.status(204).send();
});

app.listen(port, () => {
    console.log(`Servidor en http://localhost:${port}`);
});

CAPITULO 14: BUENAS PRÁCTICAS Y PATRONES
================================================================================

1. Usa `strict: true` en tsconfig.json para el máximo nivel de seguridad.
2. Prefiere interfaces sobre type para objetos (por extensibilidad).
3. Evita `any`; usa `unknown` si es necesario.
4. Utiliza tipos de unión y narrowing en lugar de `any`.
5. Nombra tipos con PascalCase, variables y funciones en camelCase.
6. Documenta funciones con JSDoc o comentarios (especialmente las exportadas).
7. Organiza código en módulos (import/export).
8. Usa `readonly` para propiedades que no deben cambiar.
9. Aprovecha los genéricos para código reutilizable.
10. Usa `as const` para objetos que deben ser literales:
    const colores = ["rojo", "verde", "azul"] as const;

11. Usa `satisfies` para verificar que un objeto cumple un tipo sin perder inferencia:
    const config = { host: "localhost", port: 3000 } satisfies Config;

12. Usa `NonNullable<T>` para eliminar null/undefined de un tipo.
13. Usa `Partial<T>` y `Required<T>` para hacer propiedades opcionales u obligatorias.
14. Usa `Omit<T, K>` y `Pick<T, K>` para seleccionar/omitir propiedades.

Utilidades de tipo comunes:
---------------------------

Middleware de autenticación:
----------------------------
// middleware/auth.ts
import { Request, Response, NextFunction } from 'express';
import jwt from 'jsonwebtoken';

export interface AuthRequest extends Request {
    user?: any;
}

export const verificarToken = (req: AuthRequest, res: Response, next: NextFunction) => {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) {
        return res.status(401).json({ error: 'Token no proporcionado' });
    }
    try {
        const decoded = jwt.verify(token, process.env.JWT_SECRET!);
        req.user = decoded;
        next();
    } catch (error) {
        return res.status(403).json({ error: 'Token inválido' });
    }
};
