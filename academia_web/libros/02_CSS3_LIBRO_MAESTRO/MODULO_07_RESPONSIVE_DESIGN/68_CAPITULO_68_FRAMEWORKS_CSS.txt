===============================================================================
¿QUÉ SON LOS FRAMEWORKS CSS?
===============================================================================

Los frameworks CSS son colecciones de estilos predefinidos que
aceleran el desarrollo. Incluyen:

  • Sistema de rejilla (grid)
  • Componentes (botones, cards, modales)
  • Utilidades (colores, espaciados)
  • Normalización de estilos entre navegadores

===============================================================================
FRAMEWORKS POPULARES
===============================================================================

  Bootstrap (2011)
    → El más usado. Incluye todo: grid, componentes, JS.
    → ventaja: gran ecosistema, documentación, comunidad.
    → desventaja: sitios se ven iguales, mucho CSS que no usas.

  Tailwind CSS (2017)
    → Utility-first. No tiene componentes, solo clases atómicas.
    → ventaja: máximo control, bundle pequeño (con purga).
    → desventaja: HTML verboso, curva de aprendizaje.

  Bulma (2016)
    → Basado en Flexbox, solo CSS (sin JS).
    → ventaja: limpio, moderno, fácil de aprender.
    → desventaja: menos componentes que Bootstrap.

  Foundation (2011)
    → Framework profesional de Zurb.
    → ventaja: muy flexible, accesibilidad integrada.
    → desventaja: curva de aprendizaje alta.

===============================================================================
CUÁNDO USAR UN FRAMEWORK
===============================================================================

  ✓ Prototipos rápidos y MVP.
  ✓ Equipos grandes que necesitan consistencia.
  ✓ Proyectos con plazos cortos.
  ✓ Cuando el framework se alinea con el diseño deseado.

===============================================================================
CUÁNDO EVITAR UN FRAMEWORK
===============================================================================

  ✗ Sitios con diseño muy personalizado.
  ✗ Proyectos pequeños (el framework pesa más que la solución).
  ✗ Cuando entiendes bien CSS y no necesitas la abstracción.
  ✗ Cuando el rendimiento es crítico (frameworks añaden peso).

===============================================================================
FRAMEWORKS Y RESPONSIVE DESIGN
===============================================================================

  La mayoría de frameworks incluyen un sistema responsivo:

  <!-- Bootstrap: clases responsivas -->
  <div class="col-12 col-md-6 col-lg-4">
    <!-- 1 col móvil, 2 tablet, 3 desktop -->
  </div>

  <!-- Tailwind: utility responsivo -->
  <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3">

  Ventaja: no escribes media queries.
  Desventaja: dependes del breakpoint del framework.

===============================================================================
CÓDIGO PRÁCTICO: COMPARATIVA
===============================================================================

  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { font-family: Arial; padding: 20px; background: #f0f2f5; }

    h1 { margin-bottom: 20px; }

    .comparacion {
      display: grid;
      grid-template-columns: 1fr;
      gap: 20px;
      max-width: 1000px;
      margin: 0 auto;
    }

    .lado {
      background: white;
      border-radius: 10px;
      padding: 24px;
      box-shadow: 0 2px 8px rgba(0,0,0,0.06);
    }

    .lado h2 {
      font-size: 1.2rem; margin-bottom: 12px; color: #2c3e50;
    }

    .lado pre {
      background: #f8f9fa; padding: 12px; border-radius: 6px;
      font-size: 0.85rem; overflow-x: auto;
    }

    code { background: #ecf0f1; padding: 2px 6px; border-radius: 3px; }

    .grid-demo {
      display: grid;
      grid-template-columns: 1fr;
      gap: 8px;
      margin-top: 12px;
    }

    .grid-demo > div {
      background: #3498db; color: white;
      padding: 12px; text-align: center; border-radius: 4px;
    }

    @media (min-width: 600px) { .comparacion { grid-template-columns: 1fr 1fr; } }
    @media (min-width: 768px) { .grid-demo { grid-template-columns: 1fr 1fr; } }
    @media (min-width: 1024px) { .grid-demo { grid-template-columns: 1fr 1fr 1fr; } }
  </style>

  <h1>Frameworks CSS</h1>

  <div class="comparacion">
    <div class="lado">
      <h2>Con framework (Bootstrap)</h2>
      <pre>&lt;div class="container"&gt;
  &lt;div class="row"&gt;
    &lt;div class="col-12 col-md-6 col-lg-4"&gt;
      Item
    &lt;/div&gt;
  &lt;/div&gt;
&lt;/div&gt;</pre>
    </div>

    <div class="lado">
      <h2>CSS nativo</h2>
      <pre>@media (min-width: 768px) {
  .grid { grid-template-columns: 1fr 1fr; }
}
@media (min-width: 1024px) {
  .grid { grid-template-columns: 1fr 1fr 1fr; }
}</pre>
    </div>
  </div>

  <div class="lado" style="margin-top:20px;max-width:1000px;margin-left:auto;margin-right:auto;">
    <h2>Grid responsivo con CSS nativo</h2>
    <div class="grid-demo">
      <div>1</div>
      <div>2</div>
      <div>3</div>
    </div>
  </div>

===============================================================================
RECOMENDACIÓN
===============================================================================

  Aprende CSS puro antes de usar frameworks. Cuando entiendas
  grid, flexbox, media queries y variables CSS, los frameworks
  serán herramientas, no muletas.

  Para proyectos personales o de aprendizaje: CSS nativo.
  Para proyectos profesionales con equipo: evalúa el framework
  según las necesidades del proyecto.

===============================================================================
EJERCICIOS
===============================================================================

1. Crea un grid responsivo con CSS nativo (sin framework).
2. Replica el mismo grid usando Bootstrap o Tailwind.
3. Compara el peso del CSS: tu código nativo vs un framework CDN.
4. Decide para tu proyecto: ¿framework o nativo? Justifica.

===============================================================================
GLOSARIO
===============================================================================

Framework:      Conjunto de estilos y herramientas predefinidas.
Utility-first:  Enfoque con clases atómicas (Tailwind).
Bootstrap:      Framework CSS más popular.
Tailwind:       Framework utility-first moderno.
CSS nativo:     CSS sin librerías externas.
purge:          Eliminar CSS no usado en producción.
