===============================================================================
¿QUÉ ES MOBILE FIRST?
===============================================================================

Mobile First es una estrategia donde se diseña primero la versión
móvil y luego se añaden estilos para pantallas más grandes.

  Desktop-first:   Escribes CSS para desktop y luego rompes con
                   @media (max-width: 768px) { ... }

  Mobile-first:    Escribes CSS para móvil y luego expandes con
                   @media (min-width: 768px) { ... }

===============================================================================
VENTAJAS
===============================================================================

  • Rendimiento: menos CSS en móvil (donde más importa).
  • Progresivo: empiezas simple, añades complejidad gradual.
  • Priorización: fuerzas pensar qué es realmente importante.
  • Google: Mobile-First Indexing desde 2019.
  • Futuro: cada vez más tráfico viene de dispositivos pequeños.

===============================================================================
PATRÓN MOBILE FIRST
===============================================================================

  /* BASE: MÓVIL — 100% del ancho, una columna */
  .layout { display: block; }
  .sidebar { display: none; }
  .nav { flex-direction: column; }
  body { font-size: 16px; }

  /* TABLET — dos columnas, sidebar visible */
  @media (min-width: 768px) {
    .layout { display: grid; grid-template-columns: 1fr 3fr; }
    .sidebar { display: block; }
    .nav { flex-direction: row; }
  }

  /* DESKTOP — layout más amplio, más espaciado */
  @media (min-width: 1024px) {
    .layout { grid-template-columns: 1fr 4fr; gap: 32px; }
    body { font-size: 18px; }
  }

El CSS base es el móvil. Cada media query AÑADE, no reemplaza.

===============================================================================
MOBILE FIRST VS DESKTOP FIRST
===============================================================================

  Mobile First:
    Base: móvil
    @media (min-width: 600px) { ... }
    @media (min-width: 900px) { ... }

  Desktop First:
    Base: desktop
    @media (max-width: 900px) { ... }
    @media (max-width: 600px) { ... }

Mobile First es más limpio: siempre añades estilos, nunca
los cancelas con max-width.

CÓDIGO PRÁCTICO
===============================================================================

  <!DOCTYPE html>
  <html lang="es">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mobile First</title>
    <style>
      * { margin: 0; padding: 0; box-sizing: border-box; }

      body { font-family: Arial; color: #333; background: #f8f9fa; }

      /* BASE: MÓVIL */
      .contenedor {
        padding: 16px;
        max-width: 1200px;
        margin: 0 auto;
      }

      header {
        background: #2c3e50; color: white;
        padding: 24px 16px; text-align: center;
        border-radius: 0 0 12px 12px;
        margin: -16px -16px 16px;
      }

      nav {
        display: flex;
        flex-direction: column;
        gap: 4px;
        margin-bottom: 16px;
      }

      nav a {
        display: block; padding: 12px; text-align: center;
        background: #ecf0f1; color: #2c3e50;
        text-decoration: none; border-radius: 6px;
        font-weight: bold;
      }

      main { margin-bottom: 16px; }

      main article {
        background: white; padding: 20px; border-radius: 8px;
        box-shadow: 0 2px 8px rgba(0,0,0,0.06);
        margin-bottom: 16px;
      }

      aside {
        background: #ecf0f1; padding: 20px; border-radius: 8px;
        margin-bottom: 16px;
      }

      footer {
        background: #34495e; color: white;
        padding: 16px; text-align: center; border-radius: 8px;
      }

      /* TABLET (≥600px) */
      @media (min-width: 600px) {
        nav { flex-direction: row; }
        nav a { flex: 1; }
        .layout { display: grid; grid-template-columns: 1fr 2fr; gap: 16px; }
        aside { margin-bottom: 0; }

      /* DESKTOP (≥900px) */
      @media (min-width: 900px) {
        .contenedor { padding: 32px; }
        header { padding: 40px 32px; border-radius: 12px; margin: 0 0 24px; }

        .layout {
          grid-template-columns: 1fr 3fr;
          gap: 24px;
        }

        main article { padding: 32px; }

        aside { padding: 32px; }
      }
    </style>
  </head>
  <body>
    <div class="contenedor">
      <header>
        <h1>Mobile First</h1>
        <p>Diseñado desde lo esencial</p>
      </header>

        <a href="#">Inicio</a>

        <a href="#">Blog</a>

      </nav>

        <aside>
          <h3>Sidebar</h3>
          <p>En móvil está abajo del contenido. En tablet+ al lado.</p>
        </aside>
        <main>
          <article>
            <h2>¿Por qué Mobile First?</h2>
            <p>El 60% del tráfico web es móvil. Google indexa
            primero la versión móvil. Los usuarios esperan que
            tu sitio funcione en su teléfono.</p>
          </article>
          <article>
            <h2>Beneficios</h2>
            <p>CSS más pequeño y rápido. Diseño más enfocado.
            Mejor experiencia en todos los dispositivos.</p>
          </article>
        </main>
      </div>

      <footer>© 2026 — Lucas EGA Academy</footer>
    </div>
  </body>

===============================================================================
CUÁNDO NO USAR MOBILE FIRST
===============================================================================

  • Sitios exclusivamente internos (intranet desktop).
  • Aplicaciones muy complejas con dashboard grande.
  • Cuando el público objetivo es 100% desktop.

En cualquier otro caso: Mobile First es la mejor opción.

===============================================================================
EJERCICIOS
===============================================================================

1. Convierte un diseño desktop-first a mobile-first.
2. Crea una página donde el menú de navegación sea vertical
   en móvil y horizontal en desktop.
3. Añade una sidebar que solo aparezca en tablet+.
4. Usa min-width: 576px, 768px, 1024px para tres breakpoints.

===============================================================================
GLOSARIO
===============================================================================

Mobile First:  Estrategia de diseño priorizando móvil.
Desktop First: Estrategia que prioriza desktop.
min-width:     Punto de expansión en mobile-first.
breakpoint:    Valor donde se activan estilos adicionales.
progressive enhancement: Añadir funcionalidad progresivamente.
