===============================================================================
¿POR QUÉ OPTIMIZAR?
===============================================================================

  Un CSS optimizado significa:

  • Carga más rápida — menos bytes = menos tiempo.
  • Menos datos móviles — importante en 3G/4G.
  • Mejor rendimiento — menos reglas = menos cálculo.
  • Mejor mantenibilidad — menos código = menos bugs.

===============================================================================
REGLAS DE ORO
===============================================================================

  1. No escribas CSS que no necesitas.
  2. Minimiza el CSS en producción.
  3. Elimina CSS no usado (uncss, purgecss).
  4. Comprime con Gzip o Brotli en el servidor.
  5. Carga el CSS crítico inline en el <head>.

===============================================================================
CSS CRÍTICO
===============================================================================

  El CSS crítico es el mínimo necesario para renderizar la
  parte visible de la página (above the fold).

  <head>
    <style>
      /* CSS crítico: header, hero, navegación */
      body { margin: 0; font-family: Arial; }
      header { background: #2c3e50; color: white; padding: 20px; }
      /* ... solo lo que se ve al cargar */
    </style>
    <link rel="stylesheet" href="estilos.css" media="print" onload="this.media='all'">
  </head>

  El CSS completo se carga de forma diferida (no bloqueante).

===============================================================================
MINIMIZACIÓN
===============================================================================

  Original:

  .tarjeta {
    background-color: #ffffff;
    border-radius: 8px;
    padding: 20px;
    margin-bottom: 16px;
  }

  Minificado:

  .tarjeta{background-color:#fff;border-radius:8px;padding:20px;margin-bottom:16px}

  Herramientas: cssnano, clean-css, UglifyCSS.

  En frameworks modernos (Vite, Webpack) la minificación es
  automática en producción.

===============================================================================
PURGAR CSS NO USADO
===============================================================================

  Frameworks como Tailwind o Bootstrap incluyen miles de reglas
  que quizás no usas. Purgar elimina las que no están en tu HTML.

  <!-- Tailwind + PurgeCSS -->
  module.exports = {
    purge: ['./src/**/*.html'],
    // solo incluye clases que aparecen en esos archivos
  }

  El resultado: de 3MB de CSS a 10KB.

===============================================================================
BUENAS PRÁCTICAS DE ESCRITURA
===============================================================================

  Selectores específicos pero no excesivos:

    /* Mal */
    body div.container ul li a { color: blue; }

    /* Bien */
    .nav-link { color: blue; }

  Evita !important (rompe la cascada):

    /* Mal */
    .btn { background: red !important; }

    /* Bien */
    .btn-primary { background: red; }

  Usa shorthand:

    /* Mal */
    margin-top: 10px;
    margin-right: 20px;
    margin-bottom: 10px;
    margin-left: 20px;

    /* Bien */
    margin: 10px 20px;

HERRAMIENTAS DE OPTIMIZACIÓN
===============================================================================

  Lighthouse → Auditoría de rendimiento (Chrome DevTools).
  PurgeCSS   → Elimina CSS no usado.
  cssnano    → Minificador de CSS.
  critical   → Genera CSS crítico automáticamente.
  PageSpeed Insights → Recomendaciones de Google.

===============================================================================
RESUMEN
===============================================================================

  ✓ Elimina CSS muerto (purge).
  ✓ Minifica en producción.
  ✓ Usa CSS crítico inline.
  ✓ Comprime con Gzip/Brotli.
  ✓ Escribe selectores simples y específicos.
  ✓ Usa shorthand y variables CSS.
  ✓ Evita !important.
  ✓ Mide con Lighthouse antes y después.

===============================================================================
GLOSARIO
===============================================================================

CSS crítico:   CSS mínimo para el contenido visible.
minificación:  Eliminar espacios, saltos y comentarios.
purge:         Eliminar reglas CSS no usadas.
shorthand:     Propiedad abreviada (margin, padding, background).
Gzip/Brotli:   Algoritmos de compresión del servidor.
Lighthouse:    Herramienta de auditoría de rendimiento.
