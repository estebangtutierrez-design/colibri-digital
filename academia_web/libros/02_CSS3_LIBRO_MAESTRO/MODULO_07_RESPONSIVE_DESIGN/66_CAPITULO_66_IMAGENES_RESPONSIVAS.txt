"Una imagen que no se adapta pesa más que mil palabras mal cargadas."
                         — Profesor Búho AI

===============================================================================
EL PROBLEMA
===============================================================================

  • Cargar una imagen de 1920px en un móvil de 375px = 5× más datos.
  • 50% del peso de una página suele ser imágenes.
  • Imágenes mal dimensionadas afectan el rendimiento y el SEO.

===============================================================================
SOLUCIÓN 1: CSS BÁSICO
===============================================================================

  img {
    max-width: 100%;
    height: auto;
    display: block;
  }

  Esto evita que cualquier imagen se desborde de su contenedor.
  Es la regla mínima obligatoria.

  Imágenes de fondo:

  .hero {
    background-image: url('fondo.jpg');
    background-size: cover;
    background-position: center;
  }

===============================================================================
SOLUCIÓN 2: SRCSET Y SIZES
===============================================================================

  El navegador elige la mejor imagen según la pantalla:

  <img
    src="foto-800.jpg"
    srcset="
      foto-400.jpg  400w,
      foto-800.jpg  800w,
      foto-1200.jpg 1200w,
      foto-1600.jpg 1600w
    "
    sizes="
      (max-width: 600px) 100vw,
      (max-width: 1200px) 50vw,
      33vw
    "
    alt="Descripción"
  >

  srcset: imágenes disponibles con su ancho en píxeles (w).
  sizes:  espacio que ocupará la imagen en cada breakpoint.

  El navegador descarga solo la imagen adecuada.

===============================================================================
SOLUCIÓN 3: PICTURE + FORMATOS MODERNOS
===============================================================================

  <picture>
    <source srcset="foto.avif" type="image/avif">
    <source srcset="foto.webp" type="image/webp">
    <img src="foto.jpg" alt="Descripción">
  </picture>

  El navegador usa el primer formato que soporte:
    • AVIF → 50% más ligero que JPEG
    • WebP → 25–35% más ligero que JPEG
    • JPEG → fallback universal

===============================================================================
SOLUCIÓN 4: ART DIRECTION
===============================================================================

  Cambiar la composición según el tamaño:

  <picture>
    <source
      media="(min-width: 768px)"
      srcset="paisaje-completo.jpg"
    >
    <img src="recorte-movil.jpg" alt="Paisaje">
  </picture>

  Desktop → foto horizontal completa.
  Móvil   → recorte vertical centrado en el sujeto.

===============================================================================
SOLUCIÓN 5: LAZY LOADING
===============================================================================

  loading="lazy" → las imágenes se cargan solo cuando están cerca
  del viewport. Mejora el rendimiento inicial.

  <img src="foto.jpg" loading="lazy" alt="...">

  También evita CLS (Cumulative Layout Shift) especificando
  dimensiones:

  <img src="foto.jpg" width="800" height="600" loading="lazy" alt="...">

===============================================================================
RESUMEN DE BUENAS PRÁCTICAS
===============================================================================

  ✓ max-width: 100% en todas las imágenes.
  ✓ srcset + sizes para servir el tamaño correcto.
  ✓ <picture> + type para formatos modernos (WebP, AVIF).
  ✓ <picture> + media para art direction.
  ✓ loading="lazy" para rendimiento.
  ✓ width + height en HTML para evitar CLS.
  ✓ SVG para iconos, logos y gráficos vectoriales.

===============================================================================
EJERCICIOS
===============================================================================

1. Añade max-width: 100% a todas las imágenes de un proyecto.
2. Configura srcset con 3 resoluciones para una imagen.
3. Crea un <picture> que sirva WebP con fallback JPEG.
4. Implementa art direction: imagen horizontal en desktop,
   recorte vertical en móvil.
5. Añade loading="lazy" a imágenes debajo del fold.

===============================================================================
GLOSARIO
===============================================================================

srcset:      Lista de imágenes candidatas con sus anchos.
sizes:       Reglas de ancho ocupado por la imagen.
picture:     Elemento HTML para control de fuentes de imagen.
WebP:        Formato Google, 25–35% más ligero que JPEG.
AVIF:        Formato abierto, ~50% más ligero que JPEG.
art direction: Cambiar composición según pantalla.
CLS:         Cumulative Layout Shift (estabilidad visual).
lazy loading: Carga diferida de imágenes.
