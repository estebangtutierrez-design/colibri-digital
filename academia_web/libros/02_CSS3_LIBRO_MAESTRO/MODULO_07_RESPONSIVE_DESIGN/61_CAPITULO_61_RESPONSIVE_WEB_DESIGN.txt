===============================================================================
¿QUÉ ES RESPONSIVE WEB DESIGN?
===============================================================================

El Responsive Web Design (RWD) es un enfoque de diseño y desarrollo
web que hace que las páginas se vean y funcionen bien en cualquier
dispositivo: desktop, tablet, móvil.

No es un framework ni una librería. Es una forma de construir.

  Principios originales (Ethan Marcotte, 2010):

  1. Rejilla fluida — usar % en lugar de px.
  2. Imágenes flexibles — max-width: 100%.
  3. Media queries — puntos de quiebre según el dispositivo.

Hoy añadimos:

  4. Viewport meta tag — escala correcta en móviles.
  5. Mobile first — construir desde lo pequeño hacia lo grande.

===============================================================================
¿POR QUÉ ES IMPORTANTE?
===============================================================================

  • 60%+ del tráfico web mundial es móvil.
  • Google indexa primero la versión móvil (Mobile-First Index).
  • Los usuarios abandonan un sitio si no se ve bien en su móvil.
  • Una sola base de código para todos los dispositivos.

ESTRUCTURA DE UN SITIO RESPONSIVO
===============================================================================

  ┌─────────────────────────────────────────────┐
  │   <!DOCTYPE html>                           │
  │   <html>                                    │
  │   <head>                                    │
  │     <meta name="viewport" ...>              │  ← Obligatorio
  │     <link rel="stylesheet" href="estilos.css">│
  │   </head>                                   │
  │   <body>                                    │
  │     <div class="contenedor">                │
  │       <header>...</header>                  │
  │       <main>...</main>                      │
  │       <footer>...</footer>                  │
  │     </div>                                  │
  │   </body>                                   │
  │   </html>                                   │
  └─────────────────────────────────────────────┘

===============================================================================
PRIMEROS PASOS
===============================================================================

  1. Añade la meta viewport:

     <meta name="viewport" content="width=device-width, initial-scale=1.0">

  2. Usa un contenedor fluido:

     .contenedor {
       max-width: 1200px;
       margin: 0 auto;
       padding: 0 16px;
     }

  3. Haz que las imágenes nunca se desborden:

     img { max-width: 100%; height: auto; }

  4. Piensa en columnas flexibles, no fijas:

     .columna { width: 100%; }           /* móvil */
     @media (min-width: 768px) {
       .columna { width: 50%; }          /* tablet */
     }

===============================================================================
CÓDIGO PRÁCTICO
===============================================================================

  <!DOCTYPE html>
  <html lang="es">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>RWD — Fundamentos</title>
    <style>
      * { margin: 0; padding: 0; box-sizing: border-box; }

      body {
        font-family: Arial, sans-serif;
        background: #f0f2f5;
        color: #333;
      }

      .contenedor {
        max-width: 1200px;
        margin: 0 auto;
        padding: 16px;
      }

      header {
        background: #2c3e50;
        color: white;
        padding: 24px;
        text-align: center;
        border-radius: 8px;
        margin-bottom: 16px;
      }

      .grid {
        display: grid;
        grid-template-columns: 1fr;
        gap: 16px;
        margin-bottom: 16px;
      }

      .card {
        background: white;
        padding: 24px;
        border-radius: 8px;
        box-shadow: 0 2px 8px rgba(0,0,0,0.06);
      }

      .card h3 { margin-bottom: 8px; color: #2c3e50; }
      .card p  { color: #666; line-height: 1.5; }

      footer {
        background: #34495e;
        color: white;
        padding: 16px;
        text-align: center;
        border-radius: 8px;
      }

      /* Tablet */
      @media (min-width: 600px) {
        .grid { grid-template-columns: 1fr 1fr; }
      }

      /* Desktop */
      @media (min-width: 900px) {
        .grid { grid-template-columns: 1fr 1fr 1fr; }
      }
    </style>
  </head>
  <body>
    <div class="contenedor">
      <header>
        <h1>Responsive Web Design</h1>
        <p>Redimensiona esta ventana</p>
      </header>
      <div class="grid">
        <div class="card">
          <h3>Fluido</h3>
          <p>Usa porcentajes y max-width. El layout respira.</p>
        </div>
        <div class="card">
          <h3>Flexible</h3>
          <p>Imágenes y medios se escalan solos.</p>
        </div>
        <div class="card">
          <h3>Adaptable</h3>
          <p>Media queries ajustan el diseño por pantalla.</p>
        </div>
      </div>
      <footer>© 2026 — Lucas EGA Academy</footer>
    </div>
  </body>
  </html>

===============================================================================
RESUMEN
===============================================================================

  ✓ RWD = diseño que funciona en cualquier pantalla.
  ✓ Tres pilares: rejilla fluida, imágenes flexibles, media queries.
  ✓ Meta viewport es obligatorio.
  ✓ max-width: 100% en imágenes.
  ✓ Diseña para el contenido, no para dispositivos.

===============================================================================
EJERCICIOS
===============================================================================

1. Toma un sitio con ancho fijo y conviértelo a responsivo.
2. Añade la meta viewport a todos tus proyectos.
3. Crea una página con 3 columnas en desktop, 2 en tablet, 1 en móvil.
4. Prueba tu sitio en Chrome DevTools (Ctrl+Shift+M).

===============================================================================
GLOSARIO
===============================================================================

RWD:          Responsive Web Design.
Viewport:     Área visible del navegador.
Fluido:       Diseño basado en % en lugar de px.
Media Query:  Regla CSS condicional por tamaño de pantalla.
Mobile First: Diseñar primero para móvil y escalar hacia arriba.
