"La tablet es el punto medio: ni la urgencia del móvil ni el
 espacio del desktop. Es el diseño del equilibrio."
                         — Profesor Búho AI

===============================================================================
EL RETO DE LA TABLET
===============================================================================

La tablet (768px–1024px) presenta un desafío único:

  • Más ancha que un móvil → no apilar todo.
  • Menos ancha que desktop → no caben 3 columnas.
  • Interacción táctil → botones y enlaces deben ser grandes.
  • Orientación variable → vertical u horizontal.

===============================================================================
BREAKPOINTS TÍPICOS PARA TABLET
===============================================================================

  Tablet vertical:   768px – 834px  (iPad, iPad Air)
  Tablet horizontal: 1024px – 1112px (iPad horizontal)

  /* Mobile first: tablet = primer breakpoint */
  @media (min-width: 768px) {
    .grid { grid-template-columns: 1fr 1fr; }
    .sidebar { display: block; }
  }

  @media (min-width: 1024px) {
    .grid { grid-template-columns: 1fr 1fr 1fr; }
  }

===============================================================================
LAYOUTS PARA TABLET
===============================================================================

  2 columnas simétricas:

    @media (min-width: 768px) {
      .grid { grid-template-columns: 1fr 1fr; }
    }

  1 + 2 (sidebar + contenido doble):

    @media (min-width: 768px) {
      .layout { grid-template-columns: 1fr 2fr; }
    }

  Menú horizontal:

    @media (min-width: 768px) {
      .nav { flex-direction: row; }
      .nav a { padding: 12px 24px; }
    }

===============================================================================
INTERACCIÓN TÁCTIL EN TABLET
===============================================================================

  En tablet (y móvil) no hay hover. Los estados deben funcionar
  con :focus y :active.

  /* hover solo para dispositivos con puntero */
  @media (hover: hover) {
    .card:hover { transform: translateY(-4px); }
  }

  @media (hover: none) {
    .card:active { transform: scale(0.97); }
  }

  Tamaños mínimos táctiles:
    • Botones: 44×44px
    • Enlaces: 44px de alto
    • Espaciado: 8px entre elementos táctiles

===============================================================================
CÓDIGO PRÁCTICO
===============================================================================

  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { font-family: Arial; background: #f0f2f5; padding: 20px; }

    h1 { margin-bottom: 8px; }
    .sub { color: #666; margin-bottom: 20px; }

    /* MÓVIL: 1 columna */
    .grid {
      display: grid;
      grid-template-columns: 1fr;
      gap: 16px;
    }

    .card {
      background: white;
      border-radius: 12px;
      padding: 24px;
      box-shadow: 0 2px 8px rgba(0,0,0,0.06);
      transition: transform 0.2s, box-shadow 0.2s;
    }

    .card h3 { color: #2c3e50; margin-bottom: 8px; }
    .card p { color: #666; line-height: 1.5; }
    .card .icono {
      font-size: 2rem; margin-bottom: 12px; display: block;
    }

    .card:active { transform: scale(0.97); }

    /* hover solo en dispositivos con mouse */
    @media (hover: hover) {
      .card:hover {
        transform: translateY(-4px);
        box-shadow: 0 8px 24px rgba(0,0,0,0.1);
      }
    }

    /* TABLET: 2 columnas */
    @media (min-width: 768px) {
      body { padding: 32px; }
      .grid { grid-template-columns: 1fr 1fr; }
    }

    /* DESKTOP: 3 columnas */
    @media (min-width: 1024px) {
      .grid {
        grid-template-columns: 1fr 1fr 1fr;
        max-width: 1100px; margin: 0 auto;
      }
    }

    /* TABLET HORIZONTAL: ajuste fino */
    @media (min-width: 768px) and (orientation: landscape) {
      .card { padding: 32px; }
      .card .icono { font-size: 2.5rem; }
    }
  </style>

  <h1>Diseño para Tablet</h1>
  <p class="sub">Redimensiona: móvil → tablet vertical → horizontal → desktop</p>

  <div class="grid">
    <div class="card">
      <span class="icono">📱</span>
      <h3>Móvil</h3>
      <p>1 columna, toque directo, sin hover.</p>
    </div>
    <div class="card">
      <span class="icono">📟</span>
      <h3>Tablet</h3>
      <p>2 columnas, táctil, orientación variable.</p>
    </div>
    <div class="card">
      <span class="icono">🖥</span>
      <h3>Desktop</h3>
      <p>3 columnas, hover, más espaciado.</p>
    </div>
  </div>

===============================================================================
TABLET EN ORIENTACIÓN HORIZONTAL
===============================================================================

  Cuando la tablet está en horizontal, el ancho es similar al de un
  desktop pequeño (~1024px). Puedes adelantar el breakpoint de desktop:

  @media (min-width: 1024px) {
    .grid { grid-template-columns: repeat(3, 1fr); }
  }

  O tratar la horizontal como un caso aparte:

  @media (orientation: landscape) and (max-width: 1024px) {
    .grid { grid-template-columns: repeat(2, 1fr); }
  }

===============================================================================
EJERCICIOS
===============================================================================

1. Crea un layout que muestre 1 columna en móvil, 2 en tablet, 3 en desktop.
2. Añade una media query para hover: none y cambia los efectos.
3. Diseña un menú de navegación que siempre tenga botones ≥44px.
4. Prueba tu diseño en orientación vertical y horizontal en tablet.

===============================================================================
GLOSARIO
===============================================================================

Tablet:       Dispositivo entre 768px y 1024px de ancho.
landscape:    Orientación horizontal.
portrait:     Orientación vertical.
hover: none:  Media query para detectar tacto.
interacción táctil: Toque directo sin cursor intermedio.
