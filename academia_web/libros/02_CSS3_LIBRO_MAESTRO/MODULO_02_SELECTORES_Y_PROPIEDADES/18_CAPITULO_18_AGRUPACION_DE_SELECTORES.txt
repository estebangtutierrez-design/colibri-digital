===============================================================================
INTRODUCCIÓN: ¿QUÉ ES LA AGRUPACIÓN DE SELECTORES?
===============================================================================

La agrupación de selectores es el proceso de combinar varios selectores
en una sola regla CSS, separándolos con una coma (`,`).

---

📌 **Sintaxis**
selector1, selector2, selector3 {
propiedad: valor;
propiedad: valor;
}

text

Todos los selectores en la lista reciben los mismos estilos.

---

🔵 **Ejemplo básico**

Sin agrupar:
------------------------------------------------------------
h1 {
    color: blue;
    font-family: Arial, sans-serif;
}

h2 {
    color: blue;
    font-family: Arial, sans-serif;
}

h3 {
    color: blue;
    font-family: Arial, sans-serif;
}
------------------------------------------------------------

Agrupado:
------------------------------------------------------------
h1, h2, h3 {
    color: blue;
    font-family: Arial, sans-serif;
}
------------------------------------------------------------

**Resultado:** El mismo estilo, pero con mucho menos código.

---

📌 **Principio DRY (Don't Repeat Yourself)**

La agrupación es una aplicación directa del principio DRY:
"No te repitas."

Si tienes el mismo bloque de estilos en varios lugares,
agrupa los selectores en una sola regla.

===============================================================================
AGRUPANDO DIFERENTES TIPOS DE SELECTORES
===============================================================================

Puedes agrupar cualquier tipo de selector:
elementos, clases, IDs, selectores descendientes, pseudo-clases, etc.

---

🔵 **Ejemplo: Elementos + Clases**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1, .titulo-principal, #encabezado {
    font-size: 2rem;
    font-weight: bold;
    color: #2C3E50;
}
------------------------------------------------------------

**Resultado:** Los elementos h1, los que tienen clase `.titulo-principal`
y el elemento con ID `#encabezado` comparten los mismos estilos.

---

🔵 **Ejemplo: Selectores descendientes**

------------------------------------------------------------
CSS
------------------------------------------------------------
nav a, .menu a, footer a {
    color: #007BFF;
    text-decoration: none;
}

nav a:hover, .menu a:hover, footer a:hover {
    text-decoration: underline;
}
------------------------------------------------------------

**Resultado:** Todos los enlaces dentro de `nav`, `.menu` y `footer`
tienen el mismo color y subrayado en hover.

---

🔵 **Ejemplo: Mezclando tipos**

------------------------------------------------------------
CSS
------------------------------------------------------------
.boton, button, input[type="submit"] {
    background-color: #007BFF;
    color: white;
    border: none;
    padding: 10px 20px;
    border-radius: 5px;
    cursor: pointer;
}

.boton:hover, button:hover, input[type="submit"]:hover {
    background-color: #0056b3;
}
------------------------------------------------------------

**Resultado:** Todos los elementos que actúan como botones comparten el mismo estilo.

---

📌 **Ventajas de agrupar diferentes tipos**

- **Consistencia visual**: Los elementos que deben verse igual, se ven igual.
- **Mantenibilidad**: Un cambio en el estilo se aplica a todos.
- **Menos código**: Menos líneas que escribir y mantener.

===============================================================================
AGRUPANDO PSEUDO-CLASES Y PSEUDO-ELEMENTOS
===============================================================================

También puedes agrupar pseudo-clases y pseudo-elementos.

---

🔵 **Ejemplo: Agrupando pseudo-clases**

------------------------------------------------------------
CSS
------------------------------------------------------------
.boton:hover, .boton:focus, .boton:active {
    background-color: #0056b3;
    transform: scale(0.98);
}
------------------------------------------------------------

**Resultado:** El botón reacciona de la misma manera en hover, focus y active.

---

🔵 **Ejemplo: Agrupando pseudo-elementos**

------------------------------------------------------------
CSS
------------------------------------------------------------
.cita::before, .cita::after {
    color: #3498DB;
    font-size: 1.5rem;
}
------------------------------------------------------------

---

🔵 **Ejemplo: Selectores con pseudo-clases mixtas**

------------------------------------------------------------
CSS
------------------------------------------------------------
a:link, a:visited, .enlace-no-visitado {
    color: #007BFF;
    text-decoration: none;
}

a:hover, .enlace:hover {
    color: #0056b3;
    text-decoration: underline;
}
------------------------------------------------------------

===============================================================================
ESPECIFICIDAD EN SELECTORES AGRUPADOS
===============================================================================

Es importante entender cómo funciona la especificidad cuando agrupas selectores.

---

📌 **Regla importante**

**Cada selector en la agrupación mantiene su propia especificidad.**
La especificidad no se "suma" ni se "promedia".

---

🔵 **Ejemplo**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1, .titulo, #principal {
    color: blue;
}
------------------------------------------------------------

En este caso:
- `h1` tiene especificidad (0,0,0,1) = 1
- `.titulo` tiene especificidad (0,0,1,0) = 10
- `#principal` tiene especificidad (0,1,0,0) = 100

Si hay un conflicto con otra regla, el navegador evaluará
cada selector por separado.

---

🔴 **Ojo con esto**

Si después tienes:

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: red;
}

h1, .titulo, #principal {
    color: blue;
}
------------------------------------------------------------

El color será **rojo** para el `h1`, porque la regla `h1 { color: red; }`
tiene la misma especificidad que `h1` en la agrupación, pero aparece después
(orden de cascada), o puede tener la misma especificidad.

**El orden de la cascada** decide en caso de empate.

---

📌 **Especificidad de la agrupación**

La agrupación en sí misma **no** tiene especificidad.
Cada selector dentro de la agrupación se evalúa de forma independiente.
/* Esto es un error conceptual: la agrupación no tiene especificidad propia /
h1, .titulo, #principal {
color: blue; / Cada selector tiene su propia especificidad */
}

===============================================================================
ERRORES COMUNES AL AGRUPAR SELECTORES
===============================================================================

❌ **Error 1: Olvidar la coma entre selectores**

   ❌ `h1 h2 h3 { color: blue; }` → Esto es un selector descendiente (h1 h2 h3), no una agrupación.

   ✅ `h1, h2, h3 { color: blue; }` → Agrupación correcta.

---

❌ **Error 2: Poner coma al final**

   ❌ `h1, h2, h3, { color: blue; }` → La coma extra puede causar errores en navegadores antiguos.

   ✅ `h1, h2, h3 { color: blue; }`

---

❌ **Error 3: Agrupar selectores que no deberían compartir estilos**

   A veces, aunque dos selectores compartan algunos estilos, pueden necesitar otros diferentes.
   En ese caso, agrupa los estilos comunes y luego aplica los específicos por separado.

   ✅ Buena práctica:
h1, h2, h3 {
font-family: Arial, sans-serif;
color: #333;
}

h1 {
font-size: 2.5rem;
}

h2 {
font-size: 2rem;
}

h3 {
font-size: 1.5rem;
}

text

---

❌ **Error 4: No considerar la especificidad**

Si agrupas un selector de tipo (baja especificidad) con una clase (mayor especificidad),
el navegador aplicará el estilo según la especificidad de cada selector cuando haya conflictos.

---

❌ **Error 5: Agrupar demasiados selectores**

Agrupar decenas de selectores en una sola regla puede hacer que el código
sea difícil de leer y mantener. Agrupa con criterio.

EJEMPLOS PRÁCTICOS DE AGRUPACIÓN
===============================================================================

---

🔵 **Ejemplo 1: Estilos para títulos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1, h2, h3, h4, h5, h6 {
 font-family: 'Georgia', serif;
 color: #2C3E50;
 margin-bottom: 0.5rem;
 line-height: 1.2;
}

h1 { font-size: 2.5rem; }
h2 { font-size: 2rem; }
h3 { font-size: 1.75rem; }
h4 { font-size: 1.5rem; }
h5 { font-size: 1.25rem; }
h6 { font-size: 1rem; }
------------------------------------------------------------

**Ventaja:** Todos los títulos comparten la fuente, el color y el margen,
pero cada uno tiene su propio tamaño.

---

🔵 **Ejemplo 2: Estilos para inputs**

------------------------------------------------------------
CSS
------------------------------------------------------------
input[type="text"], input[type="email"], input[type="tel"], textarea {
 width: 100%;
 padding: 10px;
 border: 2px solid #CED4DA;
 border-radius: 5px;
 font-size: 1rem;
 transition: border-color 0.3s;
}

input[type="text"]:focus, input[type="email"]:focus, input[type="tel"]:focus, textarea:focus {
 border-color: #007BFF;
 outline: none;
}
------------------------------------------------------------

---

🔵 **Ejemplo 3: Clases de utilidad**

------------------------------------------------------------
CSS
------------------------------------------------------------
.text-center, .texto-centrado, .centrar {
 text-align: center;
}

.text-left, .texto-izquierda {
 text-align: left;
}

.text-right, .texto-derecha {
 text-align: right;
}
------------------------------------------------------------

---

🔵 **Ejemplo 4: Agrupación con pseudo-clases**

------------------------------------------------------------
CSS
------------------------------------------------------------

 padding: 10px 20px;
 border: none;
 border-radius: 5px;
 font-weight: bold;
 cursor: pointer;
 transition: background-color 0.3s;

 background-color: #007BFF;
 color: white;
}

.boton-primario:hover {
 background-color: #0056b3;
}

.boton-secundario {
 background-color: #6C757D;

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: AGRUPACIÓN COMO UN EQUIPO
===============================================================================

Imagina que estás formando un equipo de trabajo.

- Cada persona (selector) tiene habilidades diferentes.
- Pero todas deben seguir las mismas reglas básicas: horario, código de vestimenta, etc.

En lugar de dar las reglas a cada persona por separado,
las reúnes a todas y les dices las reglas una sola vez.

Eso es la agrupación de selectores.

- `h1, h2, h3` son como tres personas diferentes.
- Pero todas comparten el mismo `color` y `font-family`.
- Les das el estilo una sola vez y todas lo aplican.

Luego, si `h1` necesita un tamaño diferente, se lo das de forma individual.

GLOSARIO DEL CAPÍTULO
===============================================================================

- **Agrupación de selectores**: Técnica que combina varios selectores en una sola regla, separándolos con comas.
- **Coma (,)**: El símbolo que separa los selectores en una agrupación.
- **DRY (Don't Repeat Yourself)**: Principio de desarrollo que promueve evitar la duplicación de código.
- **Especificidad en agrupación**: Cada selector en la agrupación mantiene su propia especificidad.
- **Código limpio**: Código que es fácil de leer, entender y mantener.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Puedo agrupar selectores de diferentes tipos?**

   Sí, puedes agrupar elementos, clases, IDs, pseudo-clases, pseudo-elementos y combinarlos.
   Ejemplo: `h1, .titulo, #header, a:hover { ... }`

---

❓ **¿La agrupación afecta la especificidad?**

   No. Cada selector en la agrupación mantiene su propia especificidad.
   La agrupación en sí misma no tiene especificidad.

---

❓ **¿Cuántos selectores puedo agrupar?**

   No hay límite, pero es mejor mantener la agrupación razonable.
   Si tienes más de 10 selectores, considera si realmente comparten todos los estilos.

---

❓ **¿Qué pasa si un selector en la agrupación no existe en el HTML?**

   No pasa nada. El navegador simplemente lo ignora.
   Los selectores que no tienen coincidencias no afectan al CSS.

---

❓ **¿Es mejor agrupar o usar una clase común?**

   Depende. Si los elementos comparten estilos y tienen una relación semántica,
   la agrupación es adecuada. Si puedes añadir una clase a todos, también es válido.
