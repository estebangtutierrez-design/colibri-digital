BIENVENIDA DEL PROFESOR BÚHO

Querido estudiante…

Hasta ahora has aprendido a seleccionar elementos por su nombre, clase, ID o su relación con otros elementos.

Pero hay una pregunta que aún no hemos respondido:

**¿Cómo selecciono un elemento cuando el usuario pasa el mouse por encima?**
**¿Cómo selecciono el primer elemento de una lista?**
**¿Cómo selecciono un enlace que ya ha sido visitado?**

La respuesta son las **pseudo-clases**.

Las pseudo-clases son selectores especiales que se activan en función del **estado** de un elemento, no de su estructura.

- `:hover` → cuando el mouse está encima.
- `:focus` → cuando el elemento tiene el foco (teclado).
- `:first-child` → el primer hijo de un contenedor.
- `:nth-child()` → el enésimo hijo.
- `:not()` → todo excepto algo.
- Y muchas más.

Las pseudo-clases son una de las herramientas más poderosas de CSS.
Te permiten crear interfaces interactivas y dinámicas sin escribir ni una línea de JavaScript.

Hoy vas a aprender las más importantes.

===============================================================================
INTRODUCCIÓN: ¿QUÉ ES UNA PSEUDO-CLASE?
===============================================================================

Una pseudo-clase es un **estado** de un elemento HTML que no está definido
por su estructura, sino por su condición dinámica.

---

📌 **Sintaxis**
selector:pseudo-clase {
propiedad: valor;
}

text

O en combinación con otros selectores:
selector.pseudo-clase:hover {
propiedad: valor;
}

text

---

📌 **Tipos de pseudo-clases**

| Categoría | Descripción | Ejemplos |
|-----------|-------------|----------|
| **Enlaces** | Estados de los enlaces | :link, :visited, :hover, :active |
| **Interacción** | Estados del usuario | :hover, :focus, :active |
| **Formularios** | Estados de inputs | :checked, :disabled, :enabled, :required |
| **Estructurales** | Posición en el DOM | :first-child, :last-child, :nth-child, :nth-of-type |
| **Negación** | Exclusión | :not() |
| **Contenido** | Relacionado con contenido | :empty, :lang() |
| **URL** | Navegación | :target, :link, :visited |

Vamos a explorar las más comunes y útiles.

===============================================================================
1. PSEUDO-CLASES PARA ENLACES (:link, :visited, :hover, :active)
===============================================================================

Estas son las pseudo-clases más conocidas. Se usan para estilizar enlaces
según su estado.

---

📌 **Orden recomendado (LVHA)**

El orden correcto para aplicar estilos a enlaces es:
:link → :visited → :hover → :active

text

(Se puede recordar con la regla mnemotécnica: **L**o **V**eo **H**oy **A**ctivo o **L**V **H**A)

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Enlace no visitado */
a:link {
    color: #007BFF;
    text-decoration: none;
}

/* Enlace ya visitado */
a:visited {
    color: #6C757D;
}

/* Al pasar el mouse */
a:hover {
    color: #0056b3;
    text-decoration: underline;
}

/* Al hacer clic (momento de la pulsación) */
a:active {
    color: #DC3545;
}
------------------------------------------------------------

---

📌 **Explicación de cada estado**

- **`:link`**: Enlace que el usuario **no ha visitado** aún.
- **`:visited`**: Enlace que el usuario **ya ha visitado**.
- **`:hover`**: Cuando el mouse está **sobre** el enlace.
- **`:active`**: En el momento en que se **hace clic** (el botón está presionado).

---

⚠️ **Restricciones de `:visited`**

Por razones de privacidad, los navegadores limitan lo que puedes cambiar
con `:visited`. Solo puedes modificar:
- `color`
- `background-color`
- `border-color`
- `outline-color`
- `column-rule-color`

No puedes cambiar `display`, `position`, `width`, `height`, etc.
Esto es para evitar que los sitios web puedan espiar el historial del usuario.

---

🔵 **Ejemplo completo de enlaces**

------------------------------------------------------------
HTML
------------------------------------------------------------
<nav>
    <a href="https://www.google.com">Google</a>
    <a href="https://www.wikipedia.org">Wikipedia</a>
    <a href="https://www.github.com">GitHub</a>
</nav>
------------------------------------------------------------

CSS:
------------------------------------------------------------
nav a:link {
    color: #007BFF;
    padding: 10px 15px;
    display: inline-block;
    transition: background-color 0.3s;
}

nav a:visited {
    color: #6C757D;
}

nav a:hover {
    background-color: #E9ECEF;
    border-radius: 5px;
}

nav a:active {
    background-color: #DC3545;
    color: white;
}
------------------------------------------------------------

===============================================================================
2. PSEUDO-CLASES DE INTERACCIÓN (:hover, :focus, :active)
===============================================================================

Estas pseudo-clases se aplican a cualquier elemento, no solo enlaces.

---

📌 **`:hover`**

Se activa cuando el usuario coloca el mouse sobre un elemento.

------------------------------------------------------------
CSS
------------------------------------------------------------
.boton:hover {
    background-color: #0056b3;
    transform: scale(1.05);
}

.card:hover {
    box-shadow: 0 8px 25px rgba(0,0,0,0.2);
}
------------------------------------------------------------

---

📌 **`:focus`**

Se activa cuando el elemento tiene el foco (por ejemplo, al navegar con el teclado).

Es fundamental para la **accesibilidad**.

------------------------------------------------------------
CSS
------------------------------------------------------------
input:focus {
    outline: 3px solid #007BFF;
    border-color: #007BFF;
}

button:focus {
    outline: 2px solid #FFC107;
    outline-offset: 2px;
}
------------------------------------------------------------

---

🔴 **NUNCA elimines `:focus` sin alternativa**
/* ❌ Muy mala práctica */
input:focus {
outline: none;
}

/* ✅ Siempre proporciona un indicador visible */
input:focus {
outline: 3px solid #007BFF;
outline-offset: 2px;
}

text

Las personas que navegan por teclado necesitan el `:focus` para saber dónde están.

---

📌 **`:active`**

Se activa mientras se hace clic (el mouse está presionado).

------------------------------------------------------------
CSS
------------------------------------------------------------
.boton:active {
    transform: scale(0.95);
    background-color: #004085;
}
------------------------------------------------------------

---

🔵 **Ejemplo combinado**

------------------------------------------------------------
CSS
------------------------------------------------------------
.boton {
    background-color: #007BFF;
    color: white;
    padding: 10px 20px;
    border: none;
    border-radius: 5px;
    transition: all 0.2s;
}

.boton:hover {
    background-color: #0056b3;
}

.boton:focus {
    outline: 3px solid #FFC107;
    outline-offset: 2px;
}

.boton:active {
    transform: scale(0.95);
}
------------------------------------------------------------

===============================================================================
3. PSEUDO-CLASES DE FORMULARIO
===============================================================================

Estas pseudo-clases permiten estilizar campos de formulario según su estado.

---

📌 **`:checked`**

Se aplica a elementos que están marcados (checkbox, radio).

------------------------------------------------------------
CSS
------------------------------------------------------------
input[type="checkbox"]:checked {
    accent-color: #007BFF;
}

/* Cambiar el estilo de un label cuando el checkbox está marcado */
input[type="checkbox"]:checked + label {
    color: #007BFF;
    font-weight: bold;
}
------------------------------------------------------------

---

📌 **`:disabled` y `:enabled`**

Se aplican a campos que están deshabilitados o habilitados.

------------------------------------------------------------
CSS
------------------------------------------------------------
input:disabled {
    background-color: #E9ECEF;
    color: #6C757D;
    cursor: not-allowed;
}

input:enabled {
    background-color: white;
}
------------------------------------------------------------

---

📌 **`:required` y `:optional`**

Se aplican a campos que son obligatorios u opcionales.

------------------------------------------------------------
CSS
------------------------------------------------------------
input:required {
    border-color: #DC3545;
}

input:optional {
    border-color: #6C757D;
}
------------------------------------------------------------

---

📌 **`:valid` y `:invalid`**

Se aplican cuando el valor del campo es válido o inválido según el tipo o patrón.

------------------------------------------------------------
CSS
------------------------------------------------------------
input:valid {
    border-color: #28A745;
    background-color: #f0fff0;
}

input:invalid {
    border-color: #DC3545;
    background-color: #fff0f0;
}
------------------------------------------------------------

---

🔵 **Ejemplo completo de formulario**

------------------------------------------------------------
HTML
------------------------------------------------------------
<form>
    <label for="nombre">Nombre (requerido):</label>
    <input type="text" id="nombre" required>

    <label for="email">Email (opcional):</label>
    <input type="email" id="email">

    <label for="acepto">Acepto términos:</label>
    <input type="checkbox" id="acepto">

    <button type="submit">Enviar</button>
</form>
------------------------------------------------------------

CSS:
------------------------------------------------------------
input:required {
    border: 2px solid #007BFF;
}

input:optional {
    border: 2px solid #CED4DA;
}

input:valid {
    border-color: #28A745;
    background-color: #F0FFF0;
}

input:invalid {
    border-color: #DC3545;
    background-color: #FFF0F0;
}

input:disabled {
    background-color: #E9ECEF;
    cursor: not-allowed;
}

input:checked + label {
    color: #007BFF;
}

button:focus {
    outline: 3px solid #FFC107;
}
------------------------------------------------------------

===============================================================================
4. PSEUDO-CLASES ESTRUCTURALES
===============================================================================

Estas pseudo-clases seleccionan elementos basándose en su posición dentro del DOM.

---

📌 **`:first-child` y `:last-child`**

Seleccionan el primer y último hijo de un contenedor.

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Primer párrafo de cualquier contenedor */
p:first-child {
    font-weight: bold;
}

/* Último elemento de una lista */
li:last-child {
    border-bottom: none;
}
------------------------------------------------------------

---

📌 **`:nth-child(n)`**

Selecciona el enésimo hijo.

| Sintaxis | Descripción |
|----------|-------------|
| `:nth-child(2)` | El segundo hijo |
| `:nth-child(even)` | Todos los hijos pares |
| `:nth-child(odd)` | Todos los hijos impares |
| `:nth-child(3n)` | Cada tercer hijo (3, 6, 9, ...) |
| `:nth-child(3n+1)` | 1, 4, 7, 10, ... |
| `:nth-child(n+3)` | Todos los hijos desde el tercero en adelante |

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Segundo elemento de una lista */
li:nth-child(2) {
    background-color: #F0F0F0;
}

/* Filas pares de una tabla (efecto zebra) */
tr:nth-child(even) {
    background-color: #F8F9FA;
}

/* Cada 3 elementos */
.item:nth-child(3n) {
    border-right: none;
}
------------------------------------------------------------

---

📌 **`:nth-of-type(n)`**

Similar a `:nth-child`, pero solo cuenta elementos del mismo tipo.

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Segundo párrafo de su contenedor */
p:nth-of-type(2) {
    font-style: italic;
}
------------------------------------------------------------

---

📌 **`:first-of-type` y `:last-of-type`**

Primer y último elemento de un tipo específico.

------------------------------------------------------------
CSS
------------------------------------------------------------
/* El primer párrafo de cualquier contenedor */
p:first-of-type {
    font-size: 1.2em;
}

/* El último párrafo de cualquier contenedor */
p:last-of-type {
    margin-bottom: 0;
}
------------------------------------------------------------

---

📌 **`:nth-last-child(n)`**

Igual que `:nth-child`, pero contando desde el final.

------------------------------------------------------------
CSS
------------------------------------------------------------
/* El penúltimo elemento de una lista */
li:nth-last-child(2) {
    background-color: #E9ECEF;
}
------------------------------------------------------------

---

📌 **`:only-child`**

Selecciona un elemento que es el único hijo de su contenedor.

------------------------------------------------------------
CSS
------------------------------------------------------------
div:only-child {
    background-color: gold;
}
------------------------------------------------------------

---

📌 **`:empty`**

Selecciona elementos que no tienen contenido (ni texto, ni hijos).

------------------------------------------------------------
CSS
------------------------------------------------------------
div:empty {
    display: none;
}
------------------------------------------------------------

---

🔵 **Ejemplo completo: tablas con efectos**

HTML:
------------------------------------------------------------
<table>
    <tr><td>Fila 1</td></tr>
    <tr><td>Fila 2</td></tr>
    <tr><td>Fila 3</td></tr>
    <tr><td>Fila 4</td></tr>
    <tr><td>Fila 5</td></tr>
</table>
------------------------------------------------------------

CSS:
------------------------------------------------------------
tr:nth-child(odd) {
    background-color: #F8F9FA;
}

tr:nth-child(even) {
    background-color: #FFFFFF;
}

tr:first-child {
    font-weight: bold;
}

tr:last-child {
    border-bottom: 2px solid #007BFF;
}
------------------------------------------------------------

===============================================================================
5. PSEUDO-CLASE DE NEGACIÓN (:not())
===============================================================================

`:not()` selecciona todos los elementos que **no** coinciden con el selector proporcionado.

---

📌 **Sintaxis**
:not(selector)

text

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Todos los párrafos que no tienen la clase .destacado */
p:not(.destacado) {
    color: #6C757D;
}

/* Todos los inputs que no son de tipo submit */
input:not([type="submit"]) {
    border: 1px solid #CED4DA;
}

/* Todos los elementos de una lista excepto el primero */
li:not(:first-child) {
    margin-top: 10px;
}
------------------------------------------------------------

---

📌 **Ventajas**

- Selectores más expresivos y precisos.
- Evita tener que escribir selectores complejos.
- Mejora la mantenibilidad.

---

⚠️ **Limitaciones**

No puedes usar `:not()` dentro de `:not()` en versiones antiguas
(hoy en día los navegadores modernos sí lo soportan).
/* ❌ No funciona en navegadores antiguos */
:not(:not(p)) { ... }

text

===============================================================================
6. PSEUDO-CLASES DE CONTENIDO Y URL
===============================================================================

---

📌 **`:target`**

Selecciona el elemento que es el objetivo de un enlace de ancla (fragmento de URL).

------------------------------------------------------------
HTML
------------------------------------------------------------
<a href="#seccion1">Ir a Sección 1</a>
<h2 id="seccion1">Sección 1</h2>

CSS:
------------------------------------------------------------
:target {
    background-color: #FFF3CD;
    border-left: 5px solid #FFC107;
    padding: 10px;
}
------------------------------------------------------------

---

📌 **`:lang()`**

Selecciona elementos con un atributo `lang` específico.

------------------------------------------------------------
CSS
------------------------------------------------------------
p:lang(es) {
    quotes: "«" "»";
}

p:lang(en) {
    quotes: "“" "”";
}
------------------------------------------------------------

===============================================================================
7. PSEUDO-CLASES Y ESPECIFICIDAD
===============================================================================

Las pseudo-clases tienen la misma especificidad que una **clase** (10).

| Selector | Especificidad |
|----------|---------------|
| `p` | 1 |
| `.clase` | 10 |
| `p:hover` | 1 + 10 = 11 |
| `#id:hover` | 100 + 10 = 110 |

Esto significa que las pseudo-clases tienen prioridad sobre selectores de tipo
pero menos que un ID.

===============================================================================
ERRORES COMUNES CON PSEUDO-CLASES
===============================================================================

❌ **Error 1: Olvidar el orden LVHA**

   `:link` → `:visited` → `:hover` → `:active`
   Si el orden es incorrecto, algunos estados no se aplicarán.

---

❌ **Error 2: Eliminar `:focus` sin alternativa**

   Nunca hagas `:focus { outline: none; }` sin proporcionar otro indicador visual.

---

❌ **Error 3: Usar `:hover` solo para escritorio**

   En móviles, `:hover` funciona de manera diferente (se activa al tocar).
   Considera si realmente necesitas `:hover` en dispositivos táctiles.

---

❌ **Error 4: No usar `:not()` correctamente**

   `:not(.clase)` es diferente de `:not(.clase, .otra)` (la coma no funciona en versiones antiguas).

---

❌ **Error 5: Usar `:nth-child` sin entender la base**

   `:nth-child(2)` selecciona el segundo hijo **de cualquier tipo**.
   Si el segundo hijo es un `span`, pero tú esperabas un `p`, no se seleccionará.

GLOSARIO DEL CAPÍTULO
===============================================================================

- **Pseudo-clase**: Selector que se aplica en función del estado de un elemento.
- **:link**: Enlace no visitado.
- **:visited**: Enlace visitado.
- **:hover**: Elemento con el mouse encima.
- **:active**: Elemento en el momento del clic.
- **:focus**: Elemento con el foco (teclado).
- **:checked**: Checkbox o radio marcado.
- **:disabled**: Campo deshabilitado.
- **:enabled**: Campo habilitado.
- **:required**: Campo obligatorio.
- **:valid**: Campo con valor válido.
- **:invalid**: Campo con valor inválido.
- **:first-child**: Primer hijo de un contenedor.
- **:last-child**: Último hijo de un contenedor.
- **:nth-child(n)**: Enésimo hijo.
- **:nth-of-type(n)**: Enésimo hijo de un tipo específico.
- **:first-of-type**: Primer hijo de un tipo específico.
- **:last-of-type**: Último hijo de un tipo específico.
- **:not()**: Niega un selector.
- **:target**: Elemento objetivo de un ancla.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre `:nth-child` y `:nth-of-type`?**

   `:nth-child` cuenta todos los hijos, sin importar el tipo.
   `:nth-of-type` solo cuenta hijos del mismo tipo de elemento.

---

❓ **¿Por qué los enlaces no muestran mi estilo `:hover`?**

   Puede ser porque el orden es incorrecto (LVHA).
   También puede ser que haya otro selector con mayor especificidad.

---

❓ **¿Cómo puedo hacer que un botón cambie al pasar el mouse?**
.boton:hover { background-color: #0056b3; }

text
text

---

❓ **¿Puedo usar `:not()` con varios selectores?**

Sí, en navegadores modernos: `:not(.clase1, .clase2)`.
En navegadores antiguos, no.

---

❓ **¿Qué es `:focus-visible`?**

Es una pseudo-clase más moderna que se aplica solo cuando el foco se activa por teclado,
no por clic. Útil para mostrar el foco solo a usuarios de teclado.
