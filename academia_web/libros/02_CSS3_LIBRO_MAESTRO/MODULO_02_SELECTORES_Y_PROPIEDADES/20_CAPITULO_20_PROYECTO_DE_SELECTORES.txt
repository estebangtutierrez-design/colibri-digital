===============================================================================
EL PROYECTO: GALERÍA DE PERFILES PROFESIONALES
===============================================================================

Vamos a construir una **galería de perfiles profesionales**.

Imagina que estás creando una página para mostrar el equipo de una empresa,
o una comunidad de desarrolladores, o simplemente una galería de tarjetas
de presentación para amigos.

Cada perfil incluirá:
- Foto o avatar.
- Nombre y cargo.
- Habilidades (etiquetas).
- Botones de acción.
- Estado (en línea, ocupado, ausente).

---

📌 **Requisitos del diseño**

- Diseño moderno con tarjetas en grid.
- Colores consistentes (paleta definida).
- Efectos interactivos (hover, focus, active).
- Decoración con pseudo-elementos.
- Accesible (contraste, focus visible).
- Responsivo.

---

📌 **Estructura del proyecto**
galeria-perfiles/
├── index.html
├── style.css
└── assets/
└── avatar-placeholder.png (opcional)

===============================================================================
PASO 3: EXPLICACIÓN DE LOS SELECTORES UTILIZADOS
===============================================================================

Vamos a repasar cómo hemos utilizado **cada tipo de selector** en este proyecto.

---

📌 **Selector universal (*)**
{
margin: 0;
padding: 0;
box-sizing: border-box;
}

text

Se usa para el reset básico. Tiene especificidad (0,0,0,0).

---

📌 **Selectores de tipo (elemento)**
body { ... }
h1 { ... }
p { ... }
img { ... }

text

Se usan para estilos globales de elementos.

---

📌 **Selectores de clase (.clase)**
.galeria { ... }
.tarjeta { ... }
.tarjeta__nombre { ... }
.boton--primario { ... }

text

Son el pilar del proyecto. La mayoría de los estilos usan clases.

---

📌 **Selectores de ID (no usados)**

En este proyecto hemos evitado usar IDs para mantener la especificidad baja.
Esta es una buena práctica que hemos aplicado.

---

📌 **Selectores descendientes (espacio)**
.tarjeta .habilidad { ... }
.tarjeta__avatar img { ... }

text

Seleccionan elementos dentro de otros.

---

📌 **Selectores de hijo directo (>)**
.tarjeta > h2 { ... }

text

Selecciona solo los títulos que son hijos directos de la tarjeta.

---

📌 **Agrupación de selectores (,)**
.titulo-principal, .subtitulo { ... }
.boton--primario, .boton--secundario { ... }

text

Comparten estilos comunes.

---

📌 **Pseudo-clases**
.tarjeta:hover { ... }
.habilidad:hover { ... }
.boton:active { ... }
.boton:focus-visible { ... }
.tarjeta__avatar:focus-within { ... }
.habilidad:first-child { ... }
.habilidad:last-child { ... }

text

Se usan para interactividad y estados.

---

📌 **Pseudo-elementos (::)**
.titulo-principal::after { ... }
.tarjeta__estado::after { ... }

text

Se usan para contenido generado y decoración.

---

📌 **Selectores de atributo (implícitos)**

Aunque no se usaron directamente `[attr]`, el HTML usa atributos como `class`
que son seleccionados por los selectores de clase.

---

📌 **Especificidad en acción**

Ejemplo de especificidad en el proyecto:

| Selector | Especificidad |
|----------|---------------|
| `.tarjeta` | (0,0,1,0) = 10 |
| `.tarjeta:hover` | (0,0,2,0) = 20 |
| `.tarjeta .habilidad` | (0,0,2,0) = 20 |
| `.tarjeta > h2` | (0,0,1,1) = 11 |

La especificidad se mantiene baja y plana, lo que facilita la mantenibilidad.

PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Por qué no usamos IDs en este proyecto?

Porque los IDs tienen alta especificidad y dificultan la mantenibilidad.
Usar clases mantiene la especificidad baja y plana.

❓ ¿Cómo sé qué selector usar en cada caso?

Para estilos globales: selectores de tipo.

Para estilos reutilizables: clases.

Para elementos únicos y JavaScript: IDs.

Para interactividad: pseudo-clases.

Para decoración: pseudo-elementos.

❓ ¿Cómo puedo ver la especificidad de un selector?

Usa el inspector de Chrome o Firefox. Cada regla muestra su especificidad.

❓ ¿Qué significa "especificidad baja y plana"?

Significa usar selectores con baja especificidad (clases) y evitar anidar
demasiado (más de 3 niveles), lo que facilita la mantenibilidad.

❓ ¿Puedo usar este proyecto en mi portafolio real?

¡Sí! Es un excelente proyecto para mostrar tu dominio de los selectores CSS.
Personalízalo y hazlo tuyo.
