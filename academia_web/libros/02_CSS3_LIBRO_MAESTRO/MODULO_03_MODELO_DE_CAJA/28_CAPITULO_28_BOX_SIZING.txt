===============================================================================
INTRODUCCIÓN: ¿QUÉ ES BOX-SIZING?
===============================================================================

`box-sizing` es una propiedad CSS que define cómo se calcula el ancho
y el alto de un elemento.

Hay dos modelos:

1. **`content-box`** (por defecto): `width` y `height` se aplican **solo al contenido**.
   El padding y el border se suman al tamaño total.

2. **`border-box`** (recomendado): `width` y `height` incluyen **contenido + padding + border**.
   El padding y el border se restan del contenido.

---

📌 **Representación visual**

**content-box (por defecto)**
+---------------------------------------------------+
| +---------------------------------------------+ |
| | +---------------------------------------+ | |
| | | CONTENIDO (content) | | |
| | | width: 200px | | |
| | +---------------------------------------+ | |
| | padding: 20px | |
| +---------------------------------------------+ |
| border: 5px |
+---------------------------------------------------+
Ancho total = 200 + 20 + 20 + 5 + 5 = 250px

text

**border-box (recomendado)**
+---------------------------------------------------+
| +---------------------------------------------+ |
| | +---------------------------------------+ | |
| | | CONTENIDO (content) | | |
| | | width: 200px | | |
| | +---------------------------------------+ | |
| | padding: 20px | |
| +---------------------------------------------+ |
| border: 5px |
+---------------------------------------------------+
Ancho total = 200px (padding y border incluidos)
Contenido = 200 - 40 - 10 = 150px

text

---

📌 **Sintaxis**
box-sizing: content-box; /* Modelo por defecto /
box-sizing: border-box; / Modelo recomendado */

text

===============================================================================
1. BOX-SIZING: CONTENT-BOX (MODELO POR DEFECTO)
===============================================================================

`content-box` es el modelo de caja tradicional de CSS.
Es el valor por defecto en todos los navegadores.

---

📌 **Cómo funciona**

- `width` define el ancho del **contenido**.
- `height` define el alto del **contenido**.
- El padding y el border se **suman** al tamaño total.

---

🔵 **Fórmula**
Ancho total = width + padding-left + padding-right + border-left + border-right
Alto total = height + padding-top + padding-bottom + border-top + border-bottom

text

---

🔵 **Ejemplo**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    width: 200px;
    height: 100px;
    padding: 20px;
    border: 5px solid #333;
    box-sizing: content-box;   /* Por defecto */
    background-color: #3498DB;
}
------------------------------------------------------------

**Cálculo del ancho total:**
- width: 200px
- padding-left: 20px
- padding-right: 20px
- border-left: 5px
- border-right: 5px

**Total:** 200 + 20 + 20 + 5 + 5 = **250px**

**Cálculo del alto total:**
- height: 100px
- padding-top: 20px
- padding-bottom: 20px
- border-top: 5px
- border-bottom: 5px

**Total:** 100 + 20 + 20 + 5 + 5 = **150px**

---

📌 **Problemas con content-box**

1. **Cálculos complicados**: para saber el tamaño total, debes sumar padding y border.
2. **Sorpresas inesperadas**: si cambias el padding, el elemento cambia de tamaño.
3. **Layouts difíciles**: en layouts con porcentajes, el padding puede romper el diseño.

---

🔵 **Ejemplo del problema**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Quieres dos columnas de 50% cada una */
.columna {
    width: 50%;
    padding: 20px;
    border: 2px solid #333;
    float: left;
    box-sizing: content-box;   /* Por defecto */
}
------------------------------------------------------------

**Problema:** cada columna ocupará 50% + 20px + 20px + 2px + 2px = 50% + 44px.
Dos columnas ocuparán más del 100% del contenedor.
Una columna se romperá y caerá a la siguiente línea.

===============================================================================
2. BOX-SIZING: BORDER-BOX (MODELO RECOMENDADO)
===============================================================================

`border-box` es el modelo de caja moderno.
Es el valor recomendado por la mayoría de los desarrolladores.

---

📌 **Cómo funciona**

- `width` define el ancho **total** (incluye padding y border).
- `height` define el alto **total** (incluye padding y border).
- El contenido se ajusta automáticamente restando padding y border.

---

🔵 **Fórmula**
Ancho total = width (ya incluye padding y border)
Alto total = height (ya incluye padding y border)
Espacio para contenido = width - padding-left - padding-right - border-left - border-right

text

---

🔵 **Ejemplo**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    width: 200px;
    height: 100px;
    padding: 20px;
    border: 5px solid #333;
    box-sizing: border-box;   /* Modelo recomendado */
    background-color: #3498DB;
}
------------------------------------------------------------

**Cálculo del ancho total:**
- width: 200px (incluye padding y border)
- Espacio para contenido: 200 - 20 - 20 - 5 - 5 = **150px**

**Cálculo del alto total:**
- height: 100px (incluye padding y border)
- Espacio para contenido: 100 - 20 - 20 - 5 - 5 = **50px**

---

📌 **Ventajas de border-box**

1. **Cálculos simples**: el tamaño que defines es el tamaño total.
2. **Predecible**: cambiar el padding no cambia el tamaño total.
3. **Layouts más fáciles**: dos columnas de 50% siempre ocupan el 50% del contenedor.
4. **Más intuitivo**: "quiero que esta caja mida 200px" y mide 200px.

---

🔵 **Ejemplo del problema resuelto**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Dos columnas de 50% cada una, con padding y borde */
.columna {
    width: 50%;
    padding: 20px;
    border: 2px solid #333;
    float: left;
    box-sizing: border-box;   /* ¡Solución! */
}
------------------------------------------------------------

**Resultado:** cada columna ocupa exactamente el 50% del contenedor.
El padding y el borde se incluyen dentro de ese 50%.

===============================================================================
3. COMPARATIVA: CONTENT-BOX VS BORDER-BOX
===============================================================================

---

📌 **Tabla comparativa**

| Característica | content-box | border-box |
|----------------|-------------|------------|
| **Valor por defecto** | Sí | No |
| **Width incluye** | Solo contenido | Contenido + padding + border |
| **Height incluye** | Solo contenido | Contenido + padding + border |
| **Padding y border** | Suma al tamaño | No suma al tamaño |
| **Cálculo** | Complejo | Simple |
| **Predecible** | No | Sí |
| **Recomendado** | No | Sí |

---

🔵 **Ejemplo comparativo con el mismo código**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="caja content-box">content-box</div>
<div class="caja border-box">border-box</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.caja {
    width: 200px;
    padding: 20px;
    border: 5px solid #333;
    margin-bottom: 20px;
    background-color: #3498DB;
    color: white;
}

.content-box {
    box-sizing: content-box;
    /* Ancho total: 250px */
}

.border-box {
    box-sizing: border-box;
    /* Ancho total: 200px */
}
------------------------------------------------------------

**Resultado:** La primera caja es más grande (250px) que la segunda (200px),
aunque ambas tienen el mismo width, padding y border.

===============================================================================
4. CÓMO APLICAR BORDER-BOX GLOBALMENTE
===============================================================================

Para evitar tener que aplicar `box-sizing: border-box` a cada elemento,
puedes aplicar esta regla globalmente.

---

📌 **La regla universal (recomendada)**

------------------------------------------------------------
CSS
------------------------------------------------------------
* {
    box-sizing: border-box;
}
------------------------------------------------------------

---

📌 **La regla universal con herencia (más robusta)**

------------------------------------------------------------
CSS
------------------------------------------------------------
html {
    box-sizing: border-box;
}

*, *::before, *::after {
    box-sizing: inherit;
}
------------------------------------------------------------

---

📌 **Explicación**

- `html { box-sizing: border-box; }` → Establece border-box en el elemento raíz.
- `*, *::before, *::after { box-sizing: inherit; }` → Todos los elementos heredan border-box.

Este enfoque permite que si algún elemento necesita `content-box`,
puedas establecerlo explícitamente sin afectar al resto.

---

🔵 **Ejemplo completo del reset**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Reset con box-sizing: border-box */
html {
    box-sizing: border-box;
}

*, *::before, *::after {
    box-sizing: inherit;
}

/* Ahora todos los elementos usan border-box por defecto */
/* Si quieres content-box en algún elemento específico: */
.elemento-especial {
    box-sizing: content-box;
}
------------------------------------------------------------

---

📌 **¿Dónde colocar esta regla?**

Coloca esta regla al principio de tu archivo CSS, después de los resets
(de márgenes, paddings, etc.), o antes de cualquier otro estilo.

===============================================================================
5. BOX-SIZING EN DIFERENTES CONTEXTOS
===============================================================================

---

📌 **En layouts con porcentajes**

`border-box` es esencial para layouts con porcentajes.

------------------------------------------------------------
CSS
------------------------------------------------------------
* {
    box-sizing: border-box;
}

.contenedor {
    width: 100%;
    max-width: 1200px;
    margin: 0 auto;
}

.columna {
    width: 33.33%;
    padding: 20px;
    float: left;
}

/* Con border-box, las columnas ocupan exactamente 33.33% */
/* Sin border-box, 33.33% + 40px (padding) = se rompe */
------------------------------------------------------------

---

📌 **En formularios**

Los inputs con `width: 100%` y padding son un problema clásico.

------------------------------------------------------------
CSS
------------------------------------------------------------
* {
    box-sizing: border-box;
}

input[type="text"] {
    width: 100%;
    padding: 12px 16px;
    border: 2px solid #ced4da;
    border-radius: 8px;
}

/* Con border-box, el input ocupa el 100% del contenedor */
/* Sin border-box, el input ocupa el 100% + padding + border */
------------------------------------------------------------

---

📌 **En botones**

Los botones con padding y borde también se benefician.

------------------------------------------------------------
CSS
------------------------------------------------------------
* {
    box-sizing: border-box;
}

.boton {
    padding: 12px 24px;
    border: 2px solid #007BFF;
    background-color: #007BFF;
    color: white;
    border-radius: 8px;
}

/* El botón tiene el tamaño que le das sin sorpresas */
------------------------------------------------------------

===============================================================================
6. ¿POR QUÉ BORDER-BOX ES LA MEJOR PRÁCTICA?
===============================================================================

---

📌 **Razón 1: Intuitivo**

- "Quiero una caja de 200px" → mide 200px.
- Simple y directo.

---

📌 **Razón 2: Predecible**

- Cambiar el padding no cambia el tamaño total.
- El diseño no se rompe al añadir padding.

---

📌 **Razón 3: Más fácil para layouts**

- Los porcentajes funcionan como se espera.
- Dos columnas de 50% ocupan exactamente el 50%.

---

📌 **Razón 4: Menos cálculos**

- No necesitas sumar padding y border al ancho.
- Ahorra tiempo y evita errores.

---

📌 **Razón 5: Estándar moderno**

- Todos los frameworks modernos (Bootstrap, Tailwind, Foundation) usan border-box.
- La mayoría de los desarrolladores profesionales lo usan.

---

📌 **Razón 6: Mejor para responsividad**

- En diseños responsivos, border-box simplifica los cálculos.
- Los media queries son más fáciles de manejar.

===============================================================================
7. CUÁNDO USAR CONTENT-BOX
===============================================================================

Aunque `border-box` es el modelo recomendado, hay casos donde
`content-box` puede ser útil.

---

📌 **Casos específicos**

1. **Cuando trabajas con elementos que deben tener un tamaño de contenido exacto**.
   Por ejemplo, una imagen que debe tener exactamente 200px de ancho.

2. **Cuando estás trabajando con un diseño que ya usa content-box**.
   En proyectos heredados, es mejor mantener la consistencia.

3. **Cuando necesitas que el padding no afecte al tamaño**.
   Por ejemplo, para crear efectos visuales específicos.

---

⚠️ **Sin embargo**

En la gran mayoría de los casos, **border-box** es la mejor opción.
Si no estás seguro, usa `border-box`.

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: BOX-SIZING COMO UNA CAJA DE HERRAMIENTAS
===============================================================================

Imagina que tienes dos cajas de herramientas.

- **content-box** es una caja que mide 20cm de ancho.
  Pero cuando pones herramientas dentro, la caja se hace más grande.
  El tamaño que ves no es el tamaño que tienes.

- **border-box** es una caja que mide 20cm de ancho.
  Las herramientas caben dentro sin que la caja se haga más grande.
  El tamaño que ves es el tamaño que tienes.

Con `border-box`, la caja tiene el tamaño que tú defines.
Con `content-box`, la caja siempre es más grande de lo que esperas.

===============================================================================
ERRORES COMUNES CON BOX-SIZING
===============================================================================

❌ **Error 1: No usar box-sizing: border-box globalmente**

   Cada elemento debe tener `box-sizing: border-box` para que el diseño sea predecible.
   **Solución:** aplica `* { box-sizing: border-box; }`.

---

❌ **Error 2: Olvidar que content-box es el valor por defecto**

   Si no defines `box-sizing`, el navegador usa `content-box`.
   **Solución:** siempre define `box-sizing: border-box`.

---

❌ **Error 3: Usar border-box solo en algunos elementos**

   Si unos elementos usan border-box y otros content-box, el diseño será inconsistente.
   **Solución:** aplica border-box globalmente.

---

❌ **Error 4: No considerar que los pseudo-elementos también necesitan box-sizing**

   `::before` y `::after` también necesitan `box-sizing: border-box`.
   **Solución:** usa `*, *::before, *::after { box-sizing: border-box; }`.

---

❌ **Error 5: Cambiar box-sizing en mitad del proyecto**

   Cambiar de content-box a border-box en un proyecto existente puede romper el layout.
   **Solución:** si estás empezando, usa border-box desde el principio.

GLOSARIO DEL CAPÍTULO
===============================================================================

- **Box-sizing**: Propiedad CSS que define cómo se calcula el tamaño de un elemento.
- **Content-box**: Modelo por defecto donde width y height solo incluyen el contenido.
- **Border-box**: Modelo recomendado donde width y height incluyen contenido, padding y border.
- **Padding**: Espacio interno entre el contenido y el borde.
- **Border**: Línea alrededor del padding.
- **Width**: Ancho del elemento (según el modelo de box-sizing).
- **Height**: Alto del elemento (según el modelo de box-sizing).

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre content-box y border-box?**

   `content-box`: width y height solo afectan al contenido.
   `border-box`: width y height incluyen contenido, padding y border.

---

❓ **¿Por qué border-box es mejor?**

   Porque es más intuitivo, predecible y facilita los cálculos.
   El elemento tiene exactamente el tamaño que defines.

---

❓ **¿Cómo aplico border-box a todos los elementos?**

   Con `* { box-sizing: border-box; }`.

---

❓ **¿Qué pasa si uso content-box en un layout con porcentajes?**

   El layout se romperá porque el padding y el border suman al ancho,
   haciendo que los elementos ocupen más del porcentaje definido.

---

❓ **¿Puedo usar content-box en algunos elementos y border-box en otros?**

   Sí, pero es mejor mantener consistencia en todo el proyecto.
