===============================================================================
INTRODUCCIÓN: ¿QUÉ ES EL MARGIN?
===============================================================================

El margin es el espacio exterior entre el **borde** de un elemento
y los elementos que lo rodean.

Imagina que estás colocando cuadros en una pared.

- Cada cuadro es un **elemento**.
- El espacio entre un cuadro y el siguiente es el **margin**.
- El espacio entre el cuadro y la pared (si hay marco) es el **padding** (opcional).

Sin margin, los cuadros estarían pegados unos a otros.
Con margin, cada cuadro tiene su propio espacio.

---

📌 **Representación visual del margin**
+---------------------------------------------------+
| MARGIN (margen) |
| +---------------------------------------------+ |
| | BORDE (border) | |
| | +---------------------------------------+ | |
| | | PADDING (relleno) | | |
| | | +---------------------------------+ | | |
| | | | CONTENIDO (content) | | | |
| | | +---------------------------------+ | | |
| | +---------------------------------------+ | |
| +---------------------------------------------+ |
| |
+---------------------------------------------------+

text

---

📌 **Propiedades de margin**

| Propiedad | Descripción |
|-----------|-------------|
| `margin` | Shorthand para los cuatro lados |
| `margin-top` | Margen superior |
| `margin-right` | Margen derecho |
| `margin-bottom` | Margen inferior |
| `margin-left` | Margen izquierdo |

---

📌 **Características del margin**

1. **Es externo**: el margin está fuera del borde, no dentro.
2. **No afecta al tamaño total**: el margin no se suma al ancho y alto (solo al espacio ocupado).
3. **No tiene color de fondo**: el margin es transparente.
4. **Puede ser negativo**: los márgenes negativos mueven elementos.
5. **Colapsa verticalmente**: los márgenes verticales se combinan.
6. **Acepta `auto`**: `margin: auto` centra el elemento horizontalmente.

===============================================================================
1. MARGIN: SINTAXIS BÁSICA
===============================================================================

---

📌 **Margin con un valor (todos los lados)**
margin: valor;

text

Aplica el mismo margin a los cuatro lados.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    margin: 20px;   /* 20px en todos los lados */
    background-color: #3498DB;
    color: white;
    padding: 10px;
}
------------------------------------------------------------

---

📌 **Margin con dos valores (arriba/abajo, izquierda/derecha)**
margin: vertical horizontal;

text

- El primer valor se aplica a **arriba y abajo**.
- El segundo valor se aplica a **izquierda y derecha**.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    margin: 10px 20px;   /* 10px arriba/abajo, 20px izquierda/derecha */
}
------------------------------------------------------------

---

📌 **Margin con tres valores (arriba, izquierda/derecha, abajo)**
margin: arriba horizontal abajo;

text

- El primer valor se aplica a **arriba**.
- El segundo valor se aplica a **izquierda y derecha**.
- El tercer valor se aplica a **abajo**.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    margin: 10px 20px 30px;   /* 10px arriba, 20px izquierda/derecha, 30px abajo */
}
------------------------------------------------------------

---

📌 **Margin con cuatro valores (arriba, derecha, abajo, izquierda)**
margin: arriba derecha abajo izquierda;

text

- El primer valor se aplica a **arriba**.
- El segundo valor se aplica a **derecha**.
- El tercer valor se aplica a **abajo**.
- El cuarto valor se aplica a **izquierda**.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    margin: 10px 20px 30px 40px;   /* Orden: arriba, derecha, abajo, izquierda */
}
------------------------------------------------------------

---

📌 **Regla mnemotécnica (sentido horario)**

Los valores de margin se aplican en **sentido horario**:
arriba → derecha → abajo → izquierda

text

Es como las agujas del reloj: empiezas arriba y giras a la derecha.

---

🔵 **Ejemplo completo**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    margin-top: 10px;
    margin-right: 20px;
    margin-bottom: 30px;
    margin-left: 40px;
    background: #3498DB;
    color: white;
    padding: 10px;
}

/* Es equivalente a: */
.caja {
    margin: 10px 20px 30px 40px;
}
------------------------------------------------------------

===============================================================================
2. MARGIN CON UNIDADES
===============================================================================

El margin acepta diferentes unidades de medida.

---

📌 **Unidades comunes**

| Unidad | Descripción | Ejemplo |
|--------|-------------|---------|
| `px` | Píxeles | `margin: 20px;` |
| `em` | Relativo al tamaño de fuente del elemento | `margin: 1.5em;` |
| `rem` | Relativo al tamaño de fuente de la raíz | `margin: 1.5rem;` |
| `%` | Porcentaje del ancho del contenedor | `margin: 5%;` |
| `auto` | El navegador calcula automáticamente | `margin: 0 auto;` |

---

🔵 **Ejemplo con diferentes unidades**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja1 {
    margin: 20px;         /* 20 píxeles */
}

.caja2 {
    margin: 1.5em;        /* 1.5 veces el tamaño de fuente del elemento */
}

.caja3 {
    margin: 1.5rem;       /* 1.5 veces el tamaño de fuente de la raíz */
}

.caja4 {
    margin: 5%;           /* 5% del ancho del contenedor */
}

.caja5 {
    margin: 0 auto;       /* Centrado horizontal */
}
------------------------------------------------------------

---

📌 **Recomendación**

- Usa `rem` para margin en la mayoría de los casos.
- Usa `auto` para centrar elementos.
- Usa `%` para márgenes fluidos en layouts responsivos.
- Usa `px` solo cuando necesites un tamaño exacto.

===============================================================================
3. MARGIN: AUTO (CENTRADO)
===============================================================================

`margin: auto` es una de las herramientas más útiles en CSS.

---

📌 **Cómo funciona `margin: auto`**

El navegador calcula automáticamente el espacio sobrante
y lo distribuye uniformemente.

---

📌 **Centrar horizontalmente**

Para centrar un elemento horizontalmente, necesitas:

1. Un `width` definido (no `auto`).
2. `margin: 0 auto` o `margin-left: auto; margin-right: auto;`.

---

🔵 **Ejemplo de centrado**

------------------------------------------------------------
CSS
------------------------------------------------------------
.contenedor {
    width: 600px;
    margin: 0 auto;   /* Centrado horizontal */
    background: #f0f0f0;
    padding: 20px;
}
------------------------------------------------------------

**Resultado:** El contenedor se centra en su elemento padre.

---

📌 **Centrar con margin: auto en flexbox**

En Flexbox, `margin: auto` tiene un comportamiento especial.

------------------------------------------------------------
CSS
------------------------------------------------------------
.contenedor {
    display: flex;
    height: 200px;
    background: #f0f0f0;
}

.hijo {
    margin: auto;   /* Centra vertical y horizontalmente */
    background: #3498DB;
    padding: 20px;
    color: white;
}
------------------------------------------------------------

---

📌 **Margin: auto en dirección horizontal y vertical**

- `margin: auto` → centra en ambos ejes (si el contenedor lo permite).
- `margin: 0 auto` → centra solo horizontalmente.
- `margin: auto 0` → centra solo verticalmente (en flexbox/grid).

---

⚠️ **Importante**

`margin: auto` **no** funciona para centrar verticalmente en el modelo de caja normal.
Para centrado vertical, usa Flexbox o Grid.

===============================================================================
4. COLAPSO DE MÁRGENES (MARGIN COLLAPSE)
===============================================================================

El colapso de márgenes es un fenómeno donde los márgenes verticales
de dos elementos adyacentes se combinan, y el mayor de los dos prevalece.

---

📌 **Reglas del colapso de márgenes**

1. **Solo ocurre en dirección vertical** (top y bottom).
2. **No ocurre en dirección horizontal** (left y right).
3. **El mayor de los dos márgenes prevalece**.
4. **Ocurre entre elementos hermanos adyacentes**.
5. **También ocurre entre un elemento padre y su primer/último hijo**.

---

🔵 **Ejemplo de colapso entre hermanos**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="caja1">Caja 1</div>
<div class="caja2">Caja 2</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.caja1 {
    margin-bottom: 30px;
    background: #3498DB;
    padding: 10px;
    color: white;
}

.caja2 {
    margin-top: 20px;
    background: #28A745;
    padding: 10px;
    color: white;
}
------------------------------------------------------------

**Resultado:** El espacio entre las cajas será de **30px**, no de 50px.
El mayor de los dos márgenes (30px) prevalece.

---

🔵 **Ejemplo de colapso entre padre e hijo**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="padre">
    <div class="hijo">Hijo</div>
</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.padre {
    background: #f0f0f0;
}

.hijo {
    margin-top: 30px;
    background: #3498DB;
    padding: 10px;
    color: white;
}
------------------------------------------------------------

**Resultado:** El margin-top del hijo se "escapa" y se aplica al padre.
El padre se mueve hacia abajo 30px.

---

📌 **Cómo evitar el colapso de márgenes**

1. **Añadir `padding` o `border` al padre**.
2. **Añadir `overflow: hidden` al padre**.
3. **Usar `display: flex` o `grid` en el padre**.
4. **Usar `display: inline-block` en los hijos**.
5. **Usar `float` en los hijos** (no recomendado).

---

🔵 **Ejemplo: evitar colapso con padding**

------------------------------------------------------------
CSS
------------------------------------------------------------
.padre {
    padding-top: 1px;   /* Evita el colapso */
    background: #f0f0f0;
}

.hijo {
    margin-top: 30px;
    background: #3498DB;
    padding: 10px;
    color: white;
}
------------------------------------------------------------

---

🔵 **Ejemplo: evitar colapso con overflow**

------------------------------------------------------------
CSS
------------------------------------------------------------
.padre {
    overflow: hidden;   /* Evita el colapso */
    background: #f0f0f0;
}

.hijo {
    margin-top: 30px;
    background: #3498DB;
    padding: 10px;
    color: white;
}
------------------------------------------------------------

===============================================================================
5. MÁRGENES NEGATIVOS
===============================================================================

Los márgenes pueden tener valores negativos.

---

📌 **Sintaxis**
margin: -20px;
margin-top: -10px;

text

---

📌 **¿Qué hacen los márgenes negativos?**

- **Mueven el elemento** en dirección opuesta.
- Pueden **solapar** elementos.
- Pueden **estirar** o **encoger** espacios.

---

🔵 **Ejemplo de margin negativo**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    margin-top: -20px;   /* Se mueve 20px hacia arriba */
    background: #3498DB;
    padding: 10px;
    color: white;
}
------------------------------------------------------------

---

📌 **Usos de márgenes negativos**

1. **Solapar elementos** (efectos visuales).
2. **Crear layouts complejos** (avanzado).
3. **Contrarrestar márgenes de otros elementos**.

---

⚠️ **Advertencia**

Los márgenes negativos son una técnica avanzada.
Úsalos con cuidado y asegúrate de entender su efecto.

===============================================================================
6. MARGIN VS PADDING
===============================================================================

Una de las confusiones más comunes en CSS es la diferencia entre
padding y margin.

---

📌 **Comparativa**

| Característica | Padding | Margin |
|----------------|---------|--------|
| **Ubicación** | Dentro del borde | Fuera del borde |
| **Color de fondo** | Hereda el color del elemento | Transparente (no hereda) |
| **Afecta al tamaño** | Sí (suma al tamaño) | No (no suma al tamaño) |
| **Colapso** | No colapsa | Sí (vertical) |
| **Valores negativos** | No permitidos | Permitidos |
| **Uso** | Espacio interno | Espacio externo |

---

🔵 **Ejemplo visual**
+---------------------------------------------------+
| MARGIN (margen) |
| +---------------------------------------------+ |
| | BORDE (border) | |
| | +---------------------------------------+ | |
| | | PADDING (relleno) | | |
| | | +---------------------------------+ | | |
| | | | CONTENIDO (content) | | | |
| | | +---------------------------------+ | | |
| | +---------------------------------------+ | |
| +---------------------------------------------+ |
| |
+---------------------------------------------------+

text

---

📌 **Cuándo usar padding**

- Para separar el contenido del borde.
- Para mejorar la legibilidad.
- Para aumentar el área de clic.
- Para crear espacios internos.

---

📌 **Cuándo usar margin**

- Para separar elementos entre sí.
- Para centrar elementos (`margin: 0 auto`).
- Para crear espacios entre secciones.
- Para alinear elementos en un layout.

---

🔵 **Ejemplo comparativo**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    padding: 20px;   /* Espacio interno entre contenido y borde */
    margin: 20px;    /* Espacio externo entre esta caja y la siguiente */
    border: 2px solid #333;
}
------------------------------------------------------------

===============================================================================
7. MARGIN EN DIFERENTES TIPOS DE ELEMENTOS
===============================================================================

El comportamiento del margin varía según el tipo de elemento.

---

📌 **Elementos de bloque (block)**

- El margin se aplica normalmente en los cuatro lados.
- El margin vertical colapsa con otros elementos.
- El margin horizontal no colapsa.

**Ejemplos:** `<div>`, `<p>`, `<h1>`, `<section>`.

---

📌 **Elementos en línea (inline)**

- El margin vertical (top/bottom) **no se aplica** (es ignorado).
- El margin horizontal (left/right) sí se aplica.

**Ejemplos:** `<span>`, `<a>`, `<strong>`.

---

🔵 **Ejemplo con inline**

------------------------------------------------------------
CSS
------------------------------------------------------------
span {
    margin: 20px;   /* El vertical (top/bottom) no funciona */
    background: #3498DB;
    color: white;
}
------------------------------------------------------------

---

📌 **Elementos en línea-bloque (inline-block)**

- Se comportan como inline, pero aceptan margin como block.
- El margin vertical sí se aplica.

**Ejemplos:** `<img>`, `<button>`, elementos con `display: inline-block`.

---

🔵 **Ejemplo con inline-block**

------------------------------------------------------------
CSS
------------------------------------------------------------
span {
    display: inline-block;
    margin: 20px;   /* Funciona en los cuatro lados */
    background: #3498DB;
    color: white;
}
------------------------------------------------------------

===============================================================================
8. BUENAS PRÁCTICAS CON MARGIN
===============================================================================

---

📌 **1. Usa `margin: 0 auto` para centrar elementos**
.elemento {
width: 300px;
margin: 0 auto;
}

text

---

📌 **2. Usa márgenes consistentes en todo el proyecto**

Define variables CSS para márgenes.
:root {
--margin-small: 0.5rem;
--margin-medium: 1rem;
--margin-large: 2rem;
}

.tarjeta {
margin-bottom: var(--margin-medium);
}

text

---

📌 **3. Ten cuidado con el colapso de márgenes**

Asegúrate de entender cómo funciona el colapso para evitar sorpresas.

---

📌 **4. No uses márgenes negativos sin necesidad**

Los márgenes negativos pueden romper el layout fácilmente.

---

📌 **5. Prefiere margin a padding para separar elementos**

El margin separa elementos. El padding separa contenido del borde.

---

📌 **6. En Flexbox y Grid, usa gap en lugar de margin**

`gap` es más limpio y no tiene colapso.
.contenedor {
display: flex;
gap: 20px; /* Separa elementos sin margin */
}

text

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: EL MARGIN COMO EL ESPACIO ENTRE CASAS
===============================================================================

Imagina que los elementos CSS son casas en una calle.

- La casa es el **elemento**.
- El jardín dentro de la casa es el **padding**.
- La valla es el **borde**.
- El espacio entre una casa y la siguiente es el **margin**.

- Si el margin es 0, las casas están pegadas.
- Si el margin es grande, hay mucho espacio entre casas.
- Si el margin es negativo, las casas se solapan.

El margin es el espacio que respeta la privacidad de cada casa.

---

📌 **Otro ejemplo: los asientos en un teatro**

- Cada asiento es un **elemento**.
- El espacio dentro del asiento (donde te sientas) es el **padding**.
- El espacio entre asientos es el **margin**.

Sin margin, los asientos están pegados y es incómodo.
Con margin, cada asiento tiene su espacio.

===============================================================================
ERRORES COMUNES CON MARGIN
===============================================================================

❌ **Error 1: Confundir padding con margin**

   Padding es interno. Margin es externo.
   **Solución:** recuerda la analogía: padding es el aire dentro de la casa,
   margin es el espacio entre casas.

---

❌ **Error 2: No entender el colapso de márgenes**

   Los márgenes verticales colapsan.
   **Solución:** tenlo en cuenta al diseñar.

---

❌ **Error 3: Usar `margin: auto` sin width**

   `margin: auto` solo funciona si el elemento tiene un `width` definido (no auto).

---

❌ **Error 4: Usar márgenes negativos sin entenderlos**

   Los márgenes negativos son avanzados.
   **Solución:** usa márgenes positivos siempre que puedas.

---

❌ **Error 5: Usar margin en elementos inline verticalmente**

   El margin vertical no funciona en inline.
   **Solución:** usa `display: inline-block` o `block`.

---

❌ **Error 6: No usar variables CSS para márgenes**

   Tener márgenes consistentes mejora el diseño.
   **Solución:** define variables para márgenes.

GLOSARIO DEL CAPÍTULO

- **Margin**: Espacio exterior entre el borde y otros elementos.
- **Margin-top**: Margen superior.
- **Margin-right**: Margen derecho.
- **Margin-bottom**: Margen inferior.
- **Margin-left**: Margen izquierdo.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre padding y margin?**

   Padding es espacio **dentro** del borde (entre contenido y borde).
   Margin es espacio **fuera** del borde (entre elementos).

---

❓ **¿Qué es el colapso de márgenes?**

   Es cuando dos márgenes verticales se encuentran y el mayor prevalece.
   Solo ocurre en dirección vertical.

---

❓ **¿Cómo centro un elemento horizontalmente?**

   Con `margin: 0 auto` y un `width` definido.

---

❓ **¿Puedo usar margin con valores negativos?**

   Sí, pero es una técnica avanzada. Úsala con cuidado.

---

❓ **¿Qué es `margin: auto` en Flexbox?**

   En Flexbox, `margin: auto` centra el elemento en el eje principal
   (o en ambos ejes si se usa en ambos).
