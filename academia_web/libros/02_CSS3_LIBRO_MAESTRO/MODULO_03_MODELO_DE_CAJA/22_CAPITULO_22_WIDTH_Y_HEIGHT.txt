===============================================================================
INTRODUCCIÓN: EL TAMAÑO DE LOS ELEMENTOS
===============================================================================

En el capítulo anterior vimos que cada elemento es una caja con contenido,
padding, borde y margen.

Pero, ¿cómo definimos el tamaño de esa caja?

Las propiedades principales son:

- **`width`**: define el ancho del **contenido** (o de la caja, según el `box-sizing`).
- **`height`**: define el alto del **contenido** (o de la caja, según el `box-sizing`).

Pero hay más:
- **`min-width`**: ancho mínimo.
- **`max-width`**: ancho máximo.
- **`min-height`**: alto mínimo.
- **`max-height`**: alto máximo.

Estas propiedades te permiten controlar exactamente el tamaño
de los elementos en tu página.

===============================================================================
1. WIDTH: EL ANCHO DE LOS ELEMENTOS
===============================================================================

`width` define el ancho del elemento.

---

📌 **Sintaxis**
width: valor;

text

---

📌 **Valores comunes**

| Valor | Descripción |
|-------|-------------|
| `auto` | El navegador calcula el ancho (por defecto) |
| `unidad` | px, em, rem, %, vw, etc. |
| `min-content` | El ancho mínimo necesario para el contenido |
| `max-content` | El ancho máximo que el contenido necesita |
| `fit-content` | Se ajusta al contenido, pero con límites |

---

🔵 **Ejemplo básico**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    width: 200px;   /* Ancho fijo de 200 píxeles */
    background-color: #3498DB;
    padding: 10px;
    color: white;
}
------------------------------------------------------------

---

📌 **width: auto (por defecto)**

El navegador calcula automáticamente el ancho del elemento.

- **Elementos de bloque (block)**: ocupan todo el ancho disponible.
- **Elementos en línea (inline)**: ocupan solo el ancho de su contenido.

---

🔵 **Ejemplo con auto**

------------------------------------------------------------
CSS
------------------------------------------------------------
div {
    width: auto;   /* Ocupa todo el ancho del contenedor */
    background: #f0f0f0;
}

span {
    width: auto;   /* Ocupa solo el ancho de su texto */
    background: #3498DB;
    color: white;
}
------------------------------------------------------------

---

📌 **width con porcentaje (%)**

El ancho es relativo al tamaño del **contenedor padre**.

---
🔵 **Ejemplo con %**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="contenedor">
    <div class="caja">50% de ancho</div>
</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.contenedor {
    width: 600px;
    background: #e0e0e0;
    padding: 10px;
}

.caja {
    width: 50%;   /* 50% de 600px = 300px */
    background: #3498DB;
    color: white;
    padding: 10px;
}
------------------------------------------------------------

---

📌 **width con unidades relativas (em, rem)**

El ancho es relativo al tamaño de fuente.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    width: 20rem;   /* Si font-size base es 16px, 20rem = 320px */
}
------------------------------------------------------------

---

📌 **width con unidades de viewport (vw)**

El ancho es relativo al ancho de la ventana del navegador.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    width: 50vw;   /* 50% del ancho de la ventana */
}
------------------------------------------------------------

---

📌 **width: min-content, max-content, fit-content**

Estos valores son más avanzados y ajustan el ancho según el contenido.

------------------------------------------------------------
CSS
------------------------------------------------------------
.min {
    width: min-content;   /* El ancho más pequeño que puede tener el contenido */
}

.max {
    width: max-content;   /* El ancho más grande que necesita el contenido */
}

.fit {
    width: fit-content;   /* Se ajusta al contenido, pero con límites */
}
------------------------------------------------------------

---

🔵 **Ejemplo comparativo**

HTML:
------------------------------------------------------------
<div class="contenedor">
    <div class="caja auto">auto: ocupa todo el espacio</div>
    <div class="caja fija">200px: tamaño fijo</div>
    <div class="caja porcentaje">50%: mitad del contenedor</div>
    <div class="caja viewport">50vw: mitad de la pantalla</div>
</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.contenedor {
    width: 600px;
    background: #f0f0f0;
    padding: 10px;
}

.caja {
    margin-bottom: 10px;
    background: #3498DB;
    color: white;
    padding: 10px;
}

.auto {
    width: auto;
}

.fija {
    width: 200px;
}

.porcentaje {
    width: 50%;
}

.viewport {
    width: 50vw;
}
------------------------------------------------------------

===============================================================================
2. HEIGHT: EL ALTO DE LOS ELEMENTOS
===============================================================================

`height` define el alto del elemento.

---

📌 **Sintaxis**
height: valor;

text

---

📌 **Valores comunes**

| Valor | Descripción |
|-------|-------------|
| `auto` | El navegador calcula el alto (por defecto) |
| `unidad` | px, em, rem, %, vh, etc. |
| `min-content` | El alto mínimo necesario para el contenido |
| `max-content` | El alto máximo que el contenido necesita |

---

🔵 **Ejemplo básico**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    height: 150px;   /* Alto fijo de 150 píxeles */
    background-color: #28A745;
    color: white;
    padding: 10px;
}
------------------------------------------------------------

---

📌 **height: auto (por defecto)**

El alto se ajusta automáticamente al contenido.

---

🔵 **Ejemplo con auto**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    height: auto;   /* Se ajusta al contenido */
    background: #28A745;
    color: white;
    padding: 10px;
}
------------------------------------------------------------

---

📌 **height con porcentaje (%)**

El alto es relativo al tamaño del **contenedor padre**.

**⚠️ IMPORTANTE: Para usar `height: %`, el contenedor padre debe tener una altura definida.**

---

🔵 **Ejemplo con % que funciona**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="contenedor">
    <div class="caja">50% de alto</div>
</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.contenedor {
    height: 300px;   /* El padre tiene una altura definida */
    background: #e0e0e0;
}

.caja {
    height: 50%;   /* 50% de 300px = 150px */
    background: #28A745;
    color: white;
}
------------------------------------------------------------

---

🔵 **Ejemplo con % que NO funciona**

------------------------------------------------------------
CSS
------------------------------------------------------------
.contenedor {
    /* No tiene height definido */
    background: #e0e0e0;
}

.caja {
    height: 50%;   /* ¡NO funciona! porque el padre no tiene altura definida */
    background: #28A745;
}
------------------------------------------------------------

---

📌 **height con unidades de viewport (vh)**

El alto es relativo al alto de la ventana del navegador.

------------------------------------------------------------
CSS
------------------------------------------------------------
.hero {
    height: 100vh;   /* Ocupa toda la altura de la pantalla */
}
------------------------------------------------------------

---

📌 **height: 100% vs height: 100vh**

| Valor | Descripción |
|-------|-------------|
| `100%` | Relativo al **contenedor padre** (necesita que el padre tenga altura) |
| `100vh` | Relativo al **viewport** (siempre funciona) |

---

🔵 **Ejemplo de página completa**

------------------------------------------------------------
CSS
------------------------------------------------------------
html, body {
    height: 100%;   /* Necesario para que los hijos puedan usar % */
    margin: 0;
    padding: 0;
}

.hero {
    height: 100%;   /* Ocupa toda la altura del padre (html/body) */
    background: linear-gradient(135deg, #667eea, #764ba2);
}
------------------------------------------------------------

---

📌 **height con min-content, max-content**

Similar a width.

------------------------------------------------------------
CSS
------------------------------------------------------------
.min {
    height: min-content;   /* El alto más pequeño posible */
}

.max {
    height: max-content;   /* El alto que necesita todo el contenido */
}
------------------------------------------------------------

===============================================================================
3. MIN-WIDTH, MAX-WIDTH: LIMITAR EL ANCHO
===============================================================================

`min-width` y `max-width` permiten establecer límites al ancho de un elemento.

---

📌 **min-width**

Define el **ancho mínimo** que puede tener el elemento.
Nunca será más pequeño que este valor.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    min-width: 200px;   /* Nunca será menor de 200px */
    width: 50%;         /* Si 50% es menos de 200px, se usará 200px */
}
------------------------------------------------------------

---

📌 **max-width**

Define el **ancho máximo** que puede tener el elemento.
Nunca será más grande que este valor.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    max-width: 600px;   /* Nunca será mayor de 600px */
    width: 100%;        /* Si 100% es más de 600px, se usará 600px */
}
------------------------------------------------------------

---

🔵 **Ejemplo de diseño responsivo con max-width**

------------------------------------------------------------
CSS
------------------------------------------------------------
.contenedor {
    max-width: 1200px;   /* Límite máximo */
    width: 90%;          /* En pantallas pequeñas, ocupa el 90% */
    margin: 0 auto;      /* Centrado */
    background: #f0f0f0;
    padding: 20px;
}
------------------------------------------------------------

**Resultado:** En pantallas grandes, el contenedor tendrá un máximo de 1200px.
En pantallas pequeñas, ocupará el 90% del ancho disponible.

---

📌 **Combinación de min-width y max-width**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    min-width: 300px;
    max-width: 500px;
    width: 50%;
}
------------------------------------------------------------

**Resultado:** La caja tendrá entre 300px y 500px de ancho.
Si el 50% del contenedor es menor de 300px, usará 300px.
Si el 50% del contenedor es mayor de 500px, usará 500px.

===============================================================================
4. MIN-HEIGHT, MAX-HEIGHT: LIMITAR EL ALTO
===============================================================================

`min-height` y `max-height` permiten establecer límites al alto de un elemento.

---

📌 **min-height**

Define el **alto mínimo** que puede tener el elemento.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    min-height: 150px;   /* Nunca será menor de 150px */
    height: auto;        /* Se ajusta al contenido, pero al menos 150px */
}
------------------------------------------------------------

---

📌 **max-height**

Define el **alto máximo** que puede tener el elemento.

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    max-height: 300px;   /* Nunca será mayor de 300px */
    overflow: auto;      /* Muestra scroll si el contenido excede */
}
------------------------------------------------------------

---

🔵 **Ejemplo: contenedor de comentarios**

------------------------------------------------------------
CSS
------------------------------------------------------------
.comentarios {
    max-height: 200px;   /* Altura máxima */
    overflow-y: auto;    /* Scroll vertical si hay más comentarios */
    border: 1px solid #ccc;
    padding: 10px;
}
------------------------------------------------------------

**Resultado:** El contenedor tendrá scroll si hay más comentarios de los que caben.

===============================================================================
5. WIDTH Y HEIGHT EN DIFERENTES TIPOS DE ELEMENTOS
===============================================================================

El comportamiento de `width` y `height` varía según el tipo de elemento.

---

📌 **Elementos de bloque (block)**

- `width: auto` (por defecto): ocupan todo el ancho disponible.
- `height: auto` (por defecto): se ajustan al contenido.
- Se les puede asignar cualquier width y height.

**Ejemplos:** `<div>`, `<p>`, `<h1>`, `<section>`, `<article>`, `<header>`, `<footer>`.

---

📌 **Elementos en línea (inline)**

- `width` y `height` **no se aplican** (son ignorados).
- Ocupan solo el espacio de su contenido.

**Ejemplos:** `<span>`, `<a>`, `<strong>`, `<em>`, `<img>` (excepto imágenes, que son inline-block).

---

📌 **Elementos en línea-bloque (inline-block)**

- Se comportan como inline (fluyen en línea).
- Pero aceptan `width` y `height`.

**Ejemplos:** `<img>`, `<button>`, elementos con `display: inline-block`.

---

📌 **Elementos de bloque con display: inline-block**

Para que un elemento en línea acepte `width` y `height`, usa `display: inline-block`.

------------------------------------------------------------
CSS
------------------------------------------------------------
span {
    display: inline-block;
    width: 100px;
    height: 50px;
    background: #3498DB;
    color: white;
    text-align: center;
}
------------------------------------------------------------

===============================================================================
6. WIDTH Y HEIGHT CON BOX-SIZING
===============================================================================

El comportamiento de `width` y `height` depende del `box-sizing`.

---

📌 **Con box-sizing: content-box (por defecto)**

- `width` y `height` definen el tamaño del **contenido**.
- Padding y border se suman.

---

📌 **Con box-sizing: border-box (recomendado)**

- `width` y `height` definen el tamaño **total** (incluye padding y border).
- El contenido se ajusta automáticamente.

---

🔵 **Ejemplo comparativo**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* content-box */
.caja1 {
    width: 200px;
    padding: 20px;
    border: 5px solid #333;
    box-sizing: content-box;
    /* Ancho total = 200 + 40 + 10 = 250px */
}

/* border-box */
.caja2 {
    width: 200px;
    padding: 20px;
    border: 5px solid #333;
    box-sizing: border-box;
    /* Ancho total = 200px (padding y border incluidos) */
}
------------------------------------------------------------

===============================================================================
7. WIDTH Y HEIGHT EN DISEÑO RESPONSIVO
===============================================================================

En diseño responsivo, el uso correcto de `width` y `height` es fundamental.

---

📌 **Reglas de oro para diseño responsivo**

1. **Usa `%` y `vw` para anchos flexibles.**
2. **Usa `max-width` para limitar el tamaño en pantallas grandes.**
3. **Usa `vh` para elementos que deben ocupar toda la pantalla.**
4. **No uses `px` para tamaños principales (usa `rem` o `%`).**
5. **Aplica `box-sizing: border-box` globalmente.**

---

🔵 **Ejemplo: layout responsivo**

------------------------------------------------------------
CSS
------------------------------------------------------------
* {
    box-sizing: border-box;
}

.contenedor {
    max-width: 1200px;
    width: 90%;
    margin: 0 auto;
    padding: 20px;
}

.columna {
    width: 48%;
    float: left;
    margin: 1%;
    padding: 20px;
    background: #f0f0f0;
}

@media (max-width: 768px) {
    .columna {
        width: 100%;
        float: none;
        margin: 10px 0;
    }
}
------------------------------------------------------------

===============================================================================
ERRORES COMUNES CON WIDTH Y HEIGHT
===============================================================================

❌ **Error 1: Usar height: 100% sin que el padre tenga altura**

   `height: 100%` solo funciona si el padre tiene una altura definida.
   Solución: define `height` en el padre o usa `vh`.

---

❌ **Error 2: Usar width: 100% en un elemento con padding y border sin box-sizing**

   El elemento ocupará más del 100% (porque padding y border suman).
   Solución: usa `box-sizing: border-box`.

---

❌ **Error 3: Usar px para todo**

   Los píxeles no escalan con la pantalla.
   Solución: usa `%`, `rem`, `vw` o `vh`.

---

❌ **Error 4: Asumir que height: auto es siempre la mejor opción**

   A veces necesitas una altura fija para lograr un diseño.
   Usa `min-height` si necesitas un mínimo pero quieres que crezca.

---

❌ **Error 5: Olvidar que los elementos inline no aceptan width ni height**

   Usa `display: inline-block` o `display: block` si necesitas dimensiones.

GLOSARIO DEL CAPÍTULO
===============================================================================

- **width**: Ancho del contenido (o de la caja con border-box).
- **height**: Alto del contenido (o de la caja con border-box).
- **min-width**: Ancho mínimo que puede tener un elemento.
- **max-width**: Ancho máximo que puede tener un elemento.
- **min-height**: Alto mínimo que puede tener un elemento.
- **max-height**: Alto máximo que puede tener un elemento.
- **auto**: Valor por defecto; el navegador calcula el tamaño.
- **%**: Relativo al contenedor padre.
- **vw**: 1% del ancho del viewport.
- **vh**: 1% del alto del viewport.
- **min-content**: El tamaño mínimo necesario para el contenido.
- **max-content**: El tamaño máximo que necesita el contenido.
- **fit-content**: Ajusta al contenido con límites.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Cuál es la diferencia entre width: 100% y width: auto?**

   `width: 100%` ocupa exactamente el 100% del ancho del contenedor.
   `width: auto` (por defecto) ocupa todo el espacio disponible
   (que generalmente también es el 100%, pero con diferencias sutiles).

---

❓ **¿Por qué height: 100% no funciona en mi elemento?**

   Porque el elemento padre no tiene una altura definida.
   Necesitas que todos los padres tengan `height: 100%` o usar `height: 100vh`.

---

❓ **¿Qué es mejor, usar px o % para el ancho?**

   Depende del contexto. Usa `%` para layouts fluidos y responsivos.
   Usa `px` solo para elementos que necesiten un tamaño exacto (bordes, sombras).

---

❓ **¿Cómo puedo centrar un elemento horizontalmente?**

   Con `margin: 0 auto` y un `width` definido.

---

❓ **¿Qué hace max-width: 100%?**

   El elemento nunca superará el ancho de su contenedor.
   Es ideal para imágenes responsivas.
