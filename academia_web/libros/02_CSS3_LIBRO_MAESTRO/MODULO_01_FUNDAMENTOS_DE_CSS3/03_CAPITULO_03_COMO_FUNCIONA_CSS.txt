===============================================================================
EL VIAJE DE UNA PÁGINA WEB
===============================================================================

Para entender cómo funciona CSS, primero debemos entender
el viaje completo de una página web desde que el usuario escribe
una URL hasta que ve la página renderizada en su pantalla.

Este viaje tiene varias etapas.

===============================================================================
ETAPA 1: EL USUARIO SOLICITA UNA PÁGINA
===============================================================================

El usuario escribe una dirección en el navegador, por ejemplo:
https://www.ejemplo.com

El navegador hace una solicitud al servidor web.
El servidor responde enviando un archivo HTML.

===============================================================================
ETAPA 2: EL NAVEGADOR LEE EL HTML
===============================================================================

El navegador recibe el HTML y comienza a leerlo línea por línea.
Construye una estructura en memoria llamada **DOM (Document Object Model)**.

El DOM es un árbol que representa toda la estructura de la página.
Cada etiqueta HTML se convierte en un nodo del DOM.

Por ejemplo, este HTML:

------------------------------------------------------------
EJEMPLO DE HTML
------------------------------------------------------------
<!DOCTYPE html>
<html>
<head>
    <title>Mi Página</title>
    <link rel="stylesheet" href="estilos.css">
</head>
<body>
    <header>
        <h1>Bienvenido</h1>
    </header>
    <main>
        <p>Este es un párrafo.</p>
    </main>
</body>
</html>
------------------------------------------------------------

Se convierte en un árbol DOM:

------------------------------------------------------------
ÁRBOL DOM (REPRESENTACIÓN)
------------------------------------------------------------
Documento
  └── html
        ├── head
        │     ├── title
        │     └── link
        └── body
              ├── header
              │     └── h1
              └── main
                    └── p
------------------------------------------------------------

El DOM es la base sobre la que se aplicarán los estilos.

===============================================================================
ETAPA 3: EL NAVEGADOR DESCARGA Y LEE EL CSS
===============================================================================

Mientras el navegador construye el DOM, también descarga los archivos CSS.

Hay tres formas de aplicar CSS:

1. **CSS externo**: En un archivo aparte, vinculado con <link>.
2. **CSS interno**: Dentro de una etiqueta <style> en el <head>.
3. **CSS en línea**: Directamente en el atributo style de un elemento HTML.

El navegador descarga todos los archivos CSS y los combina en
una sola hoja de estilos.

Luego, el navegador lee cada regla CSS y la "parsea" (analiza)
para convertirla en una estructura que pueda entender.

Cada regla CSS tiene dos partes:

------------------------------------------------------------
SINTÁXIS DE UNA REGLA CSS
------------------------------------------------------------
selector {
    propiedad: valor;
}
------------------------------------------------------------

Por ejemplo:

------------------------------------------------------------
EJEMPLO DE REGLA CSS
------------------------------------------------------------
h1 {
    color: blue;
    font-size: 32px;
}
------------------------------------------------------------

- **Selector**: Define a qué elementos se aplica el estilo.
- **Declaración**: La propiedad y su valor.

El navegador extrae todas estas reglas y las guarda en una estructura
llamada **CSSOM (CSS Object Model)**.

===============================================================================
ETAPA 4: EL NAVEGADOR COMBINA DOM Y CSSOM
===============================================================================

Aquí ocurre la magia.

El navegador toma el DOM (la estructura de la página) y el CSSOM
(los estilos disponibles) y los combina para saber qué estilo
corresponde a cada elemento.

Este proceso tiene cuatro pasos fundamentales:

1. **Coincidencia de selectores (Match)**: El navegador mira cada
   selector CSS y encuentra todos los elementos del DOM que coinciden.

2. **Cálculo de cascada**: Cuando un elemento coincide con varios
   selectores, el navegador decide qué regla tiene prioridad.

3. **Herencia**: Algunas propiedades se transmiten de padres a hijos
   si no están definidas explícitamente.

4. **Cálculo de valores finales**: Cada propiedad obtiene un valor
   definitivo que se aplicará al elemento.

===============================================================================
LA CASCADA: EL CORAZÓN DE CSS
===============================================================================

El término "Cascading" (en cascada) es fundamental en CSS.
La cascada es el algoritmo que el navegador utiliza para decidir
qué estilo aplicar cuando hay conflictos.

Imagina una cascada de agua.
El agua cae desde lo más alto hasta lo más bajo.
En CSS, las reglas también caen desde las más importantes hasta las menos importantes.

El orden de prioridad en la cascada es:

===============================================================================
NIVEL 1: IMPORTANCIA Y ORIGEN
===============================================================================

| Origen            | Prioridad |
|-------------------|-----------|
| Estilo en línea   | 1 (más alto) |
| CSS del autor     | 2 |
| CSS del usuario   | 3 |
| CSS del navegador | 4 (más bajo) |

---

🔴 **¿Qué significa esto?**

1. **CSS del navegador (User Agent)**: Cada navegador tiene su propia
   hoja de estilos predeterminada. Por ejemplo, Chrome hace que los
   títulos <h1> sean grandes y en negrita. Estos son los estilos
   más básicos.

2. **CSS del usuario**: Si el usuario configura su navegador para
   usar una fuente más grande (por ejemplo, por problemas de visión),
   esos estilos tienen prioridad sobre los del navegador.

3. **CSS del autor (el desarrollador web)**: Eres tú. Tus estilos
   tienen prioridad sobre los del usuario y del navegador.

4. **CSS en línea**: Los estilos escritos directamente en el atributo
   style de un elemento HTML tienen la máxima prioridad.

---

🟡 **Ejemplo práctico**

------------------------------------------------------------
ESTILO DEL NAVEGADOR (predeterminado)
------------------------------------------------------------
h1 {
    font-size: 2em;
    font-weight: bold;
}
------------------------------------------------------------

------------------------------------------------------------
ESTILO DEL AUTOR (tú)
------------------------------------------------------------
h1 {
    font-size: 32px;
    color: blue;
}
------------------------------------------------------------

------------------------------------------------------------
ESTILO EN LÍNEA
------------------------------------------------------------
<h1 style="color: red; font-size: 48px;">Título</h1>
------------------------------------------------------------

**Resultado final:**
- Color: rojo (porque el estilo en línea tiene prioridad).
- Tamaño: 48px (porque el estilo en línea tiene prioridad).

===============================================================================
NIVEL 2: ESPECIFICIDAD
===============================================================================

Cuando hay conflictos entre reglas del mismo origen (por ejemplo,
dos reglas del CSS del autor), el navegador usa la especificidad.

La especificidad es como un sistema de puntuación.
Cuanto más específico es un selector, mayor prioridad tiene.

La jerarquía de especificidad es:

| Selector              | Especificidad | Ejemplo          |
|-----------------------|---------------|------------------|
| Estilo en línea       | 1000          | style="..."      |
| ID                    | 100           | #mi-id           |
| Clase, pseudo-clase   | 10            | .mi-clase, :hover|
| Elemento, pseudo-elemento | 1         | h1, p, ::before  |
| Universal (*)         | 0             | *                |
| Combinadores (>, +, ~)| 0             | >, +, ~          |

---

📌 **¿Cómo se calcula?**

La especificidad se calcula como un número de cuatro dígitos:
(a, b, c, d)

- a: Estilos en línea (0 o 1).
- b: Número de IDs.
- c: Número de clases, pseudo-clases y atributos.
- d: Número de elementos y pseudo-elementos.

---

🔵 **Ejemplo 1: h1**

------------------------------------------------------------
h1 { color: red; }
------------------------------------------------------------

- a=0, b=0, c=0, d=1
- Especificidad: (0,0,0,1) = 1

---

🔵 **Ejemplo 2: .titulo**

------------------------------------------------------------
.titulo { color: blue; }
------------------------------------------------------------

- a=0, b=0, c=1, d=0
- Especificidad: (0,0,1,0) = 10

---

🔵 **Ejemplo 3: #mi-id**

------------------------------------------------------------
#mi-id { color: green; }
------------------------------------------------------------

- a=0, b=1, c=0, d=0
- Especificidad: (0,1,0,0) = 100

---

🔵 **Ejemplo 4: header h1.titulo**

------------------------------------------------------------
header h1.titulo { color: purple; }
------------------------------------------------------------

- a=0, b=0, c=1 (por la clase .titulo), d=2 (header y h1)
- Especificidad: (0,0,1,2) = 12

---

🔴 **Regla de oro**

**Gana el selector con mayor especificidad.**
Si dos selectores tienen la misma especificidad,
**gana el que aparece más tarde en el archivo CSS.**

===============================================================================
NIVEL 3: ORDEN DE APARICIÓN
===============================================================================

Cuando dos reglas tienen la misma especificidad,
el navegador aplica la que aparece **más abajo** en el archivo CSS.

Esto se llama "orden de cascada".

---

🔵 **Ejemplo**

------------------------------------------------------------
CSS
------------------------------------------------------------
p {
    color: red;
}

p {
    color: blue;
}
------------------------------------------------------------

**Resultado:** El texto será azul, porque la segunda regla
aparece después y tiene la misma especificidad.

---

📌 **Esto también aplica a diferentes archivos CSS**

Si tienes dos archivos CSS vinculados en el HTML:

------------------------------------------------------------
HTML
------------------------------------------------------------
<link rel="stylesheet" href="estilos1.css">
<link rel="stylesheet" href="estilos2.css">
------------------------------------------------------------

El archivo **estilos2.css** tendrá prioridad sobre estilos1.css
cuando tengan la misma especificidad.

===============================================================================
HERENCIA EN CSS
===============================================================================

La herencia es otro mecanismo fundamental de CSS.

**Algunas propiedades se heredan de los elementos padres a los hijos.**

Esto significa que si defines un color en el body,
todos los elementos dentro del body (a menos que se especifique lo contrario)
tendrán ese color.

---

🔵 **Ejemplo de herencia**

------------------------------------------------------------
CSS
------------------------------------------------------------
body {
    color: blue;
    font-family: Arial;
}
------------------------------------------------------------

Todos los elementos dentro del body heredarán:
- Color azul.
- Fuente Arial.

---

📌 **Propiedades que se heredan comúnmente:**

| Propiedad        | ¿Se hereda? |
|------------------|-------------|
| color            | Sí          |
| font-family      | Sí          |
| font-size        | Sí          |
| font-weight      | Sí          |
| text-align       | Sí          |
| line-height      | Sí          |
| letter-spacing   | Sí          |
| visibility       | Sí          |
| background       | No          |
| border           | No          |
| margin           | No          |
| padding          | No          |
| width            | No          |
| height           | No          |
| display          | No          |
| position         | No          |

---

🔴 **¡Atención!**

No todas las propiedades se heredan.
La herencia está diseñada para propiedades relacionadas con el texto.
Las propiedades de diseño (margin, padding, border, etc.)
no se heredan para evitar resultados indeseados.

===============================================================================
VALORES ESPECIALES: INHERIT, INITIAL Y UNSET
===============================================================================

CSS ofrece tres valores especiales para controlar la herencia.

---

📌 **inherit**

Fuerza a un elemento a heredar el valor de su padre.

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
p {
    color: inherit;  /* Hereda el color del padre */
}
------------------------------------------------------------

---

📌 **initial**

Restablece una propiedad a su valor inicial (el valor predeterminado
según el estándar CSS).

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
p {
    color: initial;  /* Vuelve al color predeterminado del navegador */
}
------------------------------------------------------------

---

📌 **unset**

Actúa como inherit si la propiedad se hereda normalmente,
y como initial si no se hereda.

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
p {
    color: unset;    /* Si color se hereda (sí), actúa como inherit */
    border: unset;   /* Si border no se hereda, actúa como initial */
}
------------------------------------------------------------

===============================================================================
EL PROCESO COMPLETO: DE CSS A PANTALLA
===============================================================================

Ahora vamos a unir todo.

Cuando un navegador recibe una página, sigue estos pasos exactos:

---

**PASO 1: Construir el DOM**
   El navegador lee el HTML y crea el árbol DOM.

**PASO 2: Construir el CSSOM**
   El navegador descarga y procesa todos los archivos CSS,
   construyendo el CSSOM.

**PASO 3: Combinar DOM + CSSOM → Render Tree**
   El navegador combina el DOM y el CSSOM para crear el **Render Tree**.
   El Render Tree contiene solo los elementos que serán renderizados
   (los elementos con display: none no aparecen en el Render Tree).

**PASO 4: Calcular la cascada**
   Para cada elemento del Render Tree, el navegador aplica el algoritmo
   de cascada (origen, especificidad, orden) para determinar qué estilos
   se aplican.

**PASO 5: Calcular la herencia**
   Las propiedades que se heredan se propagan de padres a hijos
   cuando no están definidas explícitamente.

**PASO 6: Layout (maquetación)**
   El navegador calcula la posición y el tamaño de cada elemento
   en la página. Esto se llama "layout" o "reflow".

**PASO 7: Pintado (Paint)**
   El navegador "pinta" cada elemento en la pantalla.

**PASO 8: Compositing (composición)**
   El navegador combina todas las capas pintadas en la imagen final
   que ves en tu pantalla.

---

🦉 **El Profesor Búho dice:**

"Este proceso ocurre en milisegundos, cada vez que cargas una página.
El navegador hace todo este trabajo para que tú puedas ver
una página bonita y funcional."

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: CSS ES COMO UN RESTAURANTE
===============================================================================

Imagina que CSS es un restaurante.

- **El DOM** es la lista de ingredientes que tienes.
- **El CSSOM** es el libro de recetas.
- **El Render Tree** es el plato que vas a preparar.
- **La cascada** es el chef decidiendo qué receta usar cuando hay varias.
- **La herencia** es el sabor base que se transmite de un plato a otro.

- **Especificidad** es como un chef con más experiencia tiene prioridad
  sobre un aprendiz.
- **El orden de aparición** es como si el último chef en llegar
  tiene la última palabra.

===============================================================================
ERRORES COMUNES RELACIONADOS CON EL FUNCIONAMIENTO DE CSS
===============================================================================

❌ **Error 1: Usar !important demasiado**

   !important es una "bala de plata" que rompe la cascada.
   Úsalo solo cuando sea estrictamente necesario (casi nunca).

   ¿Por qué es malo? Porque anula la especificidad y el orden,
   haciendo que el CSS sea difícil de mantener.

   El orden de prioridad con !important es:
   - Estilo en línea con !important (máxima prioridad).
   - CSS del autor con !important.
   - CSS del usuario con !important.
   - CSS del navegador con !important.
   - Estilo en línea normal.
   - CSS del autor normal.

---

❌ **Error 2: No entender la especificidad**

   Muchos desarrolladores novatos no entienden por qué
   un estilo no se aplica. La respuesta casi siempre es:
   "Otro selector tiene mayor especificidad."

   Aprende a calcular la especificidad y siempre la entenderás.

---

❌ **Error 3: Confundir herencia con cascada**

   La herencia y la cascada son mecanismos diferentes.
   - La cascada decide qué regla se aplica cuando hay conflicto.
   - La herencia propaga valores de padres a hijos.

   No son lo mismo.

---

❌ **Error 4: No usar herramientas del navegador**

   Los navegadores modernos (Chrome, Firefox, Edge) tienen
   herramientas de desarrollo que te muestran exactamente
   qué estilos se están aplicando y por qué.

   Aprende a usar el Inspector de estilos. Es tu mejor amigo.

---

❌ **Error 5: Escribir CSS sin estructura**

   Si escribes CSS sin entender la cascada, terminarás con un
   archivo lleno de reglas que se sobreescriben unas a otras.

   Organiza tu CSS: primero los estilos generales,
   luego los específicos, y finalmente las excepciones.

===============================================================================
GLOSARIO DEL CAPÍTULO
===============================================================================

- **DOM (Document Object Model)**: Árbol que representa la estructura
  de un documento HTML.

- **CSSOM (CSS Object Model)**: Árbol que representa todas las reglas
  CSS de un documento.

- **Render Tree**: Árbol que combina el DOM y el CSSOM, conteniendo
  solo los elementos que serán renderizados.

- **Cascada**: Algoritmo que determina qué estilo se aplica cuando
  hay conflictos entre reglas CSS.

- **Especificidad**: Sistema de puntuación que determina la prioridad
  de un selector CSS.

- **Herencia**: Mecanismo por el cual las propiedades se transmiten
  de elementos padres a elementos hijos.

- **Layout**: Proceso de calcular el tamaño y la posición de cada elemento.

- **Paint**: Proceso de "pintar" los elementos en la pantalla.

- **Compositing**: Proceso de combinar capas para crear la imagen final.

- **!important**: Palabra clave que anula la cascada y la especificidad
  (usar con mucha precaución).

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Por qué mi estilo no se aplica?**

   Tres razones posibles:
   1. Otro selector tiene más especificidad.
   2. Otro selector aparece después en el archivo CSS.
   3. Hay un error de sintaxis (falta un punto y coma, un corchete, etc.).

---

❓ **¿Es mejor usar clases o IDs?**

   Clases, siempre que sea posible. Las clases son reutilizables
   y tienen menor especificidad, lo que hace que el CSS sea más fácil
   de mantener. Los IDs deberían usarse solo para elementos únicos
   (como un header o un footer).

---

❓ **¿Qué es el "modo estándar" y el "modo quirks"?**

   El modo estándar es el modo de renderizado moderno que sigue
   los estándares del W3C. El modo quirks es un modo de compatibilidad
   que imita el comportamiento de navegadores antiguos.

   Para asegurar el modo estándar, usa <!DOCTYPE html> al inicio
   de tu archivo HTML.

---

❓ **¿Puedo usar !important siempre para asegurar que mis estilos se apliquen?**

   No. !important es una solución temporal que genera problemas
   a largo plazo. Úsalo solo cuando sea absolutamente necesario
   (por ejemplo, para anular estilos de un framework que no puedes modificar).

---

❓ **¿Qué hago si dos archivos CSS tienen estilos conflictivos?**

   El orden de los archivos en el HTML (la etiqueta <link>)
   determina la prioridad cuando la especificidad es la misma.
   El último archivo vinculado tiene prioridad.
