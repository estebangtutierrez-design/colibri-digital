===============================================================================
INTRODUCCIÓN: ¿QUÉ ES UN SELECTOR?
===============================================================================

Un selector es un patrón que el navegador utiliza para seleccionar
elementos HTML a los que aplicar estilos.

Cada regla CSS comienza con un selector.

------------------------------------------------------------
SINTAXIS BÁSICA
------------------------------------------------------------
selector {
    propiedad: valor;
}
------------------------------------------------------------

El navegador examina el documento HTML, busca todos los elementos
que coinciden con el selector y les aplica las declaraciones.

---

📌 **Tipos de selectores básicos**

| Tipo | Ejemplo | Descripción |
|------|---------|-------------|
| Universal | `*` | Selecciona todos los elementos |
| Tipo | `h1` | Selecciona todos los elementos de ese tipo |
| Clase | `.destacado` | Selecciona elementos con esa clase |
| ID | `#logo` | Selecciona el elemento con ese ID |
| Agrupación | `h1, h2, h3` | Selecciona varios tipos a la vez |
| Descendiente | `div p` | Selecciona p dentro de div |
| Pseudoclase | `a:hover` | Selecciona en estado especial |

Vamos a explorar cada uno en detalle.

===============================================================================
SELECTOR UNIVERSAL (*)
===============================================================================

El selector universal (`*`) selecciona **todos** los elementos HTML
de la página.

---

🔵 **Ejemplo**

------------------------------------------------------------
CSS
------------------------------------------------------------
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
------------------------------------------------------------

---

📌 **¿Para qué sirve?**

Se usa comúnmente para **resetear estilos** predeterminados del navegador.
Cada navegador aplica sus propios márgenes y paddings por defecto.
Con `*` puedes eliminarlos todos de una vez.

---

⚠️ **Advertencia**

El selector universal tiene **especificidad 0**.
Es el selector menos específico de todos.

Además, usarlo para todo puede afectar el rendimiento si la página es muy grande,
pero para proyectos normales es perfectamente aceptable.

---

🟡 **Ejemplo práctico**

------------------------------------------------------------
HTML
------------------------------------------------------------
<!DOCTYPE html>
<html>
<head>
    <style>
        * {
            color: red;
        }
    </style>
</head>
<body>
    <h1>Título</h1>
    <p>Párrafo</p>
    <div>Div</div>
</body>
</html>
------------------------------------------------------------

Todos los elementos se verán en color rojo.

===============================================================================
SELECTOR DE TIPO (ELEMENTO)
===============================================================================

El selector de tipo selecciona todos los elementos HTML de un tipo específico.

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: blue;
}

p {
    font-size: 16px;
}

div {
    background-color: #f0f0f0;
}
------------------------------------------------------------

---

📌 **Uso común**

- `h1, h2, h3, h4, h5, h6` → Encabezados.
- `p` → Párrafos.
- `a` → Enlaces.
- `img` → Imágenes.
- `ul, ol` → Listas.
- `table` → Tablas.

---

🟡 **Ejemplo completo**

------------------------------------------------------------
HTML
------------------------------------------------------------
<h1>Título principal</h1>
<p>Este es un párrafo.</p>
<p>Otro párrafo.</p>
<div>Contenedor genérico.</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
h1 {
    color: darkblue;
    font-size: 2em;
}

p {
    color: #333;
    line-height: 1.5;
}

div {
    border: 1px solid #ccc;
    padding: 10px;
}
------------------------------------------------------------

---

🔴 **Características del selector de tipo**

- **Especificidad: 1** (baja).
- Aplica a **todos** los elementos de ese tipo.
- Es útil para estilos generales.
- Si necesitas más precisión, usa clases o IDs.

===============================================================================
SELECTOR DE CLASE (.clase)
===============================================================================

El selector de clase selecciona todos los elementos que tienen
un atributo `class` con el valor especificado.

En HTML:
<elemento class="nombre-clase"> ```
En CSS:

text
.nombre-clase {
    propiedad: valor;
}
🔵 Ejemplos

HTML

<p class="destacado">Este texto está destacado.</p> <p>Este texto es normal.</p> <div class="destacado">Este div también está destacado.</div> ------------------------------------------------------------
CSS:

.destacado {
color: red;
font-weight: bold;
background-color: yellow;
}

Resultado: Todos los elementos con clase "destacado" se verán
en rojo, negrita y con fondo amarillo.

📌 Ventajas de las clases

Reutilizables: Puedes usar la misma clase en varios elementos.

Específicas: Tienen mayor prioridad que los selectores de tipo.

Flexibles: Un elemento puede tener varias clases.

🔵 Múltiples clases en un elemento

HTML

<p class="destacado grande">Texto importante y grande</p> ------------------------------------------------------------
CSS:

.destacado {
color: red;
}

.grande {
font-size: 24px;
}

El párrafo será rojo y con tamaño 24px.

🔴 Características del selector de clase

Se escribe con un punto (.) antes del nombre.

La clase se define en HTML con el atributo class.

Especificidad: 10 (media).

Puede usarse en cualquier elemento.

📌 Buenas prácticas para nombres de clase

✅ Buenos nombres:

.boton-primario

.encabezado-principal

.texto-destacado

.contenedor-flex

❌ Malos nombres:

.rojo (describe el estilo, no el propósito)

.c1 (no es descriptivo)

.texto-que-es-rojo-y-grande (demasiado largo)

Consejo: Nombra las clases por su función, no por su apariencia.
Así, si cambias el color de .boton-primario de azul a verde,
el nombre sigue teniendo sentido.

===============================================================================
SELECTOR DE ID (#id)
===============================================================================

El selector de ID selecciona el elemento único que tiene
un atributo id con el valor especificado.

En HTML:

text
<elemento id="nombre-id">
En CSS:

text
#nombre-id {
    propiedad: valor;
}
🔵 Ejemplo

HTML

<h1 id="titulo-principal">Bienvenido</h1> <p>Este es un párrafo normal.</p> ------------------------------------------------------------
CSS:

#titulo-principal {
color: darkblue;
font-size: 3em;
text-align: center;
}

📌 Características del ID

Se escribe con un numeral (#) antes del nombre.

El ID se define en HTML con el atributo id.

Debe ser único en la página (solo un elemento con ese ID).

Especificidad: 100 (alta).

Tiene mayor prioridad que clases y selectores de tipo.

🔴 ID vs Clase: ¿Cuándo usar cada uno?

Característica	Clase (.)	ID (#)
Uso	Múltiples elementos	Elemento único
Especificidad	10	100
Reutilizable	Sí	No
Semántica	Grupo	Identificador único
Uso recomendado	Estilos reutilizables	Elementos únicos (header, footer, etc.)
🔵 Regla de oro

Usa clases para casi todo.

Usa ID solo para elementos que aparecen una sola vez
(como el encabezado principal, el pie de página, o enlaces de navegación).

⚠️ Atención

Los IDs tienen alta especificidad.
Si usas muchos IDs, puede ser difícil sobrescribir estilos más adelante.
Por eso la recomendación general es usar clases siempre que sea posible.

===============================================================================
AGRUPACIÓN DE SELECTORES
===============================================================================

Puedes aplicar los mismos estilos a varios selectores agrupándolos
con comas.

🔵 Ejemplo

CSS (sin agrupar)

h1 {
color: blue;
text-align: center;
}

h2 {
color: blue;
text-align: center;
}

h3 {
color: blue;
text-align: center;
}

🔵 CSS agrupado

h1, h2, h3 {
color: blue;
text-align: center;
}

📌 Ventajas de la agrupación

Código más limpio.

Fácil de mantener (un solo cambio afecta a todos).

Menos líneas de código.

🔵 Agrupación con diferentes tipos

h1, .titulo, #encabezado {
font-family: Arial, sans-serif;
font-weight: bold;
}

🔴 Advertencia

Cuando agrupas, todos los selectores reciben los mismos estilos.
Si necesitas estilos diferentes, no los agrupes.

===============================================================================
SELECTORES DESCENDIENTES
===============================================================================

El selector descendiente selecciona elementos que están dentro
de otros elementos, sin importar la profundidad.

🔵 Sintaxis

selector1 selector2 {
propiedad: valor;
}

Esto selecciona todos los selector2 que están dentro de selector1.

🔵 Ejemplo

HTML

<div> <p>Párrafo dentro de div</p> <section> <p>Otro párrafo dentro de div > section</p> </section> </div> <p>Párrafo fuera del div</p> ------------------------------------------------------------
CSS:

div p {
color: red;
}

Resultado: Los dos primeros párrafos serán rojos.
El último no (está fuera del div).

📌 Profundidad ilimitada

Los selectores descendientes pueden anidarse todo lo que quieras.

div section p span {
color: blue;
}

Pero cuidado: cuanto más específico, menos reutilizable.

⚠️ Buenas prácticas

Usa selectores descendientes con moderación.

Demasiada profundidad hace que el CSS sea difícil de mantener.

Prefiere clases en lugar de selectores descendientes largos.

🔴 Ejemplo de selector descendiente largo

body header nav ul li a {
color: red;
}

Mejor opción:

.nav-link {
color: red;
}

===============================================================================
PSEUDO-CLASES BÁSICAS
===============================================================================

Las pseudo-clases permiten seleccionar elementos en estados especiales.

📌 :hover

Se activa cuando el usuario pasa el mouse sobre el elemento.

a:hover {
color: red;
text-decoration: underline;
}

📌 :first-child

Selecciona el primer hijo de un elemento.

p:first-child {
font-weight: bold;
}

📌 :last-child

Selecciona el último hijo de un elemento.

li:last-child {
border-bottom: none;
}

📌 :nth-child(n)

Selecciona el enésimo hijo.

li:nth-child(2) {
background-color: #f0f0f0;
}

/* Patrón: cada 2 elementos */
li:nth-child(even) {
background-color: #e0e0e0;
}

li:nth-child(odd) {
background-color: #f0f0f0;
}

📌 :not(selector)

Selecciona todos los elementos que no coinciden con el selector.

p:not(.destacado) {
color: #666;
}

ERRORES COMUNES CON SELECTORES
===============================================================================

❌ Error 1: Confundir clase e ID

.mi-clase vs #mi-id
Si usas el símbolo incorrecto, el estilo no se aplica.

❌ Error 2: Olvidar el punto o el numeral

En lugar de .destacado, escribes destacado.
Sin el punto, CSS lo interpreta como selector de tipo.

❌ Error 3: Espacios en nombres de clase o ID

Los nombres de clase e ID no pueden tener espacios.

text
class="mi clase"  ❌
class="mi-clase"  ✅
class="mi_clase"  ✅
❌ Error 4: Selector descendiente demasiado largo

body header nav ul li a es excesivo.
Prefiere una clase: .nav-link.

❌ Error 5: No usar :hover en el orden correcto

Para enlaces, el orden recomendado (LVHA):

text
a:link { ... }
a:visited { ... }
a:hover { ... }
a:active { ... }

GLOSARIO DEL CAPÍTULO
===============================================================================

Selector: Patrón que identifica elementos HTML para aplicar estilos.

Selector universal (*): Selecciona todos los elementos.

Selector de tipo: Selecciona elementos por su nombre de etiqueta (h1, p, etc.).

Selector de clase (.clase): Selecciona elementos con una clase específica.

Selector de ID (#id): Selecciona el elemento con un ID único.

Agrupación: Combinar selectores con comas para compartir estilos.

Selector descendiente: Selecciona elementos dentro de otros.

Pseudo-clase: Palabra clave que define un estado especial del elemento.

:hover: Estado cuando el mouse está sobre el elemento.

:first-child: Primer hijo de un elemento.

:last-child: Último hijo de un elemento.

:nth-child(n): Enésimo hijo de un elemento.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Qué selector debo usar siempre?

Prefiere clases para casi todo. Son reutilizables,
tienen especificidad media y son fáciles de mantener.

❓ ¿Puedo usar un ID y una clase en el mismo elemento?

Sí. Por ejemplo: <div id="principal" class="contenedor">.

❓ ¿Qué ocurre si dos selectores tienen la misma especificidad?

Gana el que aparece más abajo en el archivo CSS.

❓ ¿Es malo usar el selector universal (*)?

No. Es útil para resets globales. Pero no abuses de él
para estilos específicos porque tiene especificidad 0.

❓ ¿Cuántas clases puede tener un elemento?

Las que quieras. Solo sepáralas con espacios:
<p class="destacado grande text-center">.
