===============================================================================
INTRODUCCIÓN: ¿PARA QUÉ SIRVEN LOS COMENTARIOS?
===============================================================================

Los comentarios son texto en el código que el navegador ignora por completo.
No afectan al diseño ni al comportamiento de la página.

Pero para el desarrollador, los comentarios son oro puro.

---

📌 **Usos principales de los comentarios**

1. **Explicar el propósito** de una sección o regla.
2. **Aclarar decisiones** complejas o no obvias.
3. **Advertir** sobre posibles problemas o limitaciones.
4. **Organizar** el archivo en secciones lógicas.
5. **Desactivar temporalmente** código (depuración).
6. **Documentar** la autoría, fecha y versión.
7. **Guiar a otros desarrolladores** que lean tu código.

---

🔴 **La regla de oro de los comentarios**

**Comenta el "por qué", no el "qué".**

- ❌ Mal comentario: `/* color azul para el título */` (el código ya lo muestra).
- ✅ Buen comentario: `/* Azul corporativo, según guía de marca */` (explica el contexto).

El código te dice *qué* hace.
Los comentarios te dicen *por qué* lo hace.

===============================================================================
SINTAXIS DE LOS COMENTARIOS EN CSS
===============================================================================

En CSS solo hay un tipo de comentario: el comentario de bloque.
Pero se puede usar de dos maneras: una línea o varias líneas.

---

📌 **Comentario de una línea**

------------------------------------------------------------
SINTAXIS
------------------------------------------------------------
/* Esto es un comentario de una línea */
------------------------------------------------------------

🔵 **Ejemplo**

------------------------------------------------------------
/* Estilos para el encabezado principal */
h1 {
    color: blue;
    font-size: 32px;
}
------------------------------------------------------------

---

📌 **Comentario de varias líneas (bloque)**

------------------------------------------------------------
SINTAXIS
------------------------------------------------------------
/*
   Esto es un comentario
   que ocupa varias líneas.
   El navegador ignora todo.
*/
------------------------------------------------------------

🔵 **Ejemplo**

------------------------------------------------------------
/*
   ============================================================
   SECCIÓN: ESTILOS DEL ENCABEZADO
   ============================================================
   Estos estilos controlan el aspecto del header principal.
   Incluyen el logo, el menú de navegación y el título.
*/
header {
    background-color: #333;
    padding: 20px;
}

header h1 {
    color: white;
    font-size: 2em;
}
------------------------------------------------------------

---

⚠️ **Importante**

Los comentarios en CSS no pueden anidarse.

✅ Correcto:
/* Comentario simple */

text

❌ Incorrecto:
/* Comentario /* anidado */ */

text

El navegador interpretará el primer `*/` como el final del comentario y el resto del código será interpretado como CSS, lo que puede romper la página.

===============================================================================
CÓMO USAR COMENTARIOS PARA ORGANIZAR TU CSS
===============================================================================

Una de las mejores prácticas es usar comentarios para dividir
el archivo CSS en secciones lógicas.

---

📌 **Estructura recomendada para un archivo CSS**

------------------------------------------------------------
/*
   ============================================================
   NOMBRE DEL PROYECTO
   Autor: Tu nombre
   Fecha: 2026-06-29
   Descripción: Estilos principales del sitio web.
   ============================================================
*/

/* ============================================================
   1. ESTILOS GLOBALES (reset, variables, etc.)
   ============================================================ */

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

/* ============================================================
   2. TIPOGRAFÍA
   ============================================================ */

body {
    font-family: Arial, sans-serif;
    color: #333;
}

h1, h2, h3 {
    font-family: Georgia, serif;
}

/* ============================================================
   3. ENCABEZADO Y NAVEGACIÓN
   ============================================================ */

header {
    background-color: #1a1a2e;
    padding: 20px;
}

/* ============================================================
   4. CONTENIDO PRINCIPAL
   ============================================================ */

main {
    max-width: 1200px;
    margin: 0 auto;
    padding: 20px;
}

/* ============================================================
   5. PIE DE PÁGINA
   ============================================================ */

footer {
    background-color: #1a1a2e;
    color: white;
    padding: 15px;
    text-align: center;
}

/* ============================================================
   6. UTILIDADES Y CLASES AUXILIARES
   ============================================================ */

.text-center {
    text-align: center;
}

.text-uppercase {
    text-transform: uppercase;
}
------------------------------------------------------------

---

🟢 **Ventajas de esta organización**

1. **Navegación rápida**: Puedes desplazarte fácilmente a la sección que necesitas.
2. **Mantenimiento sencillo**: Sabes exactamente dónde está cada cosa.
3. **Trabajo en equipo**: Todos siguen la misma estructura.
4. **Escalabilidad**: Cuando el proyecto crece, es fácil añadir nuevas secciones.

===============================================================================
CUÁNDO COMENTAR (Y CUÁNDO NO)
===============================================================================

Saber cuándo comentar es tan importante como saber cómo comentar.

---

📌 **✅ Situaciones donde debes comentar**

1. **Explicar decisiones complejas**:
/* Usamos display: flex en lugar de float porque es más compatible
con diseños modernos y más fácil de mantener */
.container {
display: flex;
justify-content: space-between;
}

text

2. **Advertir sobre limitaciones**:
/* Este selector no funciona en IE11 debido a la falta de soporte
para CSS Grid. Se recomienda usar flexbox como fallback. */
.grid {
display: grid;
}

text

3. **Organizar secciones**:
/* ============================================================
SECCIÓN: FORMULARIO DE CONTACTO
============================================================ */

text

4. **Desactivar código temporalmente**:
/* .old-style {
background-color: red;
} */

text

5. **Documentar la autoría y propósito**:
/*
Autor: María Pérez
Fecha: 2026-06-29
Propósito: Estilos para la página de productos
*/

text

6. **Explicar valores mágicos**:
/* Altura calculada: 100vh - 100px (header) - 50px (footer) */
.main-content {
min-height: calc(100vh - 150px);
}

text

---

📌 **❌ Situaciones donde NO debes comentar**

1. **Código que habla por sí mismo**:
❌ `/* Título en rojo */ h1 { color: red; }`
✅ `h1 { color: red; }` (el código es claro)

2. **Código obvio**:
❌ `/* Márgen de 10px */ margin: 10px;`
✅ `margin: 10px;` (no necesita explicación)

3. **Comentarios que se vuelven obsoletos**:
Si el código cambia, el comentario debe actualizarse.
Un comentario obsoleto es peor que no tener comentario.

4. **Comentarios que repiten el código**:
❌ `/* Propiedades del contenedor */ .container { display: flex; }`
✅ `/* Contenedor flexible para alinear elementos horizontalmente */ .container { display: flex; }`

---

🔴 **Regla práctica**

Si tu código es tan complejo que necesitas un comentario para explicarlo,
**considera si puedes simplificar el código primero**.

A veces, escribir código más claro es mejor que escribir comentarios.

===============================================================================
COMENTARIOS PARA DEPURACIÓN
===============================================================================

Los comentarios son una herramienta excelente para depurar (encontrar y corregir errores).

---

📌 **Técnicas de depuración con comentarios**

1. **Desactivar reglas problemáticas**:

Si sospechas que una regla CSS está causando un problema,
coméntala para ver si el problema desaparece.
/* .problema {
position: absolute;
top: 0;
} */

text

2. **Aislar secciones**:

Comenta todo el CSS y ve descomentando sección por sección
para identificar dónde está el error.

3. **Añadir comentarios de prueba**:
/* PRUEBA: ¿El fondo se aplica? */
body {
background-color: red;
}

text

4. **Marcar código temporal**:
/* TODO: Revisar esta regla para mobile */
.sidebar {
width: 300px;
}

text

---

🟢 **Etiquetas útiles para comentarios de depuración**

| Etiqueta | Significado |
|----------|-------------|
| `TODO:` | Algo que hay que hacer |
| `FIXME:` | Algo que está roto y hay que arreglar |
| `BUG:` | Un error conocido |
| `OPTIMIZE:` | Algo que se puede optimizar |
| `HACK:` | Una solución temporal o no ideal |

🔵 **Ejemplo**
/* FIXME: El menú no se alinea correctamente en pantallas pequeñas */
nav ul {
display: flex;
}

text

===============================================================================
BUENAS PRÁCTICAS PARA COMENTARIOS
===============================================================================

Aquí tienes una lista de buenas prácticas para escribir comentarios efectivos.

---

📌 **1. Sé conciso**

   Un buen comentario es breve pero completo.
   No escribas párrafos innecesarios.

---

📌 **2. Usa un lenguaje claro y consistente**

   Escribe en español (o en el idioma del equipo).
   Mantén un tono profesional.

---

📌 **3. Mantén los comentarios actualizados**

   Si cambias el código, actualiza el comentario.
   Un comentario desactualizado es confuso y puede llevar a errores.

---

📌 **4. Usa un formato consistente**

   Define un estilo de comentario y síguelo en todo el proyecto.

   Ejemplo:
/* ============================================================
ESTILOS DEL ENCABEZADO
============================================================ */

text

---

📌 **5. No comentes todo**

Los comentarios deben agregar valor, no ruido.
Si el código es claro, no necesita comentario.

---

📌 **6. Usa comentarios para explicar el "por qué"**

El código muestra "qué" hace.
Los comentarios explican "por qué" lo hace.

---

📌 **7. Usa comentarios para generar documentación**

Algunas herramientas pueden extraer comentarios para generar
documentación automática. Usa un formato estándar si es necesario.

EJEMPLO COMPLETO: PROYECTO CON COMENTARIOS
===============================================================================

Vamos a ver un archivo CSS completo con comentarios bien escritos.

------------------------------------------------------------
style.css
------------------------------------------------------------
/*
============================================================
MI SITIO PERSONAL
============================================================
Autor: Estudiante Lucas EGA Academy
Fecha: 2026-06-29
Descripción: Estilos principales del sitio personal.

Estructura:
1. Reset y variables
2. Tipografía
3. Layout general
4. Encabezado
5. Contenido principal
6. Artículos
7. Barra lateral
8. Pie de página
9. Utilidades
============================================================
*/

/* ============================================================
1. RESET Y VARIABLES
============================================================ */

/* Reset básico para eliminar márgenes y paddings por defecto */
* {
 margin: 0;
 padding: 0;
 box-sizing: border-box;
}

/* Variables CSS (colores principales del sitio) */
:root {
 --color-primario: #2d3436;
 --color-secundario: #0984e3;
 --color-fondo: #f5f6fa;
 --color-texto: #2d3436;
 --color-blanco: #ffffff;
 --color-gris: #dfe6e9;
}

/* ============================================================
2. TIPOGRAFÍA
============================================================ */

/* Fuente base: usa una fuente moderna y legible */
body {
 font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
 color: var(--color-texto);
 background-color: var(--color-fondo);
 line-height: 1.6;
}

/* Títulos: más elegantes y con mayor peso */
h1, h2, h3, h4 {
 font-family: 'Georgia', serif;
 color: var(--color-primario);
 margin-bottom: 0.5em;
}

h1 {
 font-size: 2.5em;
}

/* ============================================================
3. LAYOUT GENERAL
============================================================ */

/* Contenedor principal: centrado y con ancho máximo */
.container {
 max-width: 1200px;
 margin: 0 auto;
 padding: 0 20px;
}

/* ============================================================
4. ENCABEZADO
============================================================ */

header {
 background-color: var(--color-primario);
 color: var(--color-blanco);
 padding: 20px 0;
 border-bottom: 4px solid var(--color-secundario);
}

header h1 {
 color: var(--color-blanco);
 margin: 0;
}

/* ============================================================
5. CONTENIDO PRINCIPAL
============================================================ */

main {
 padding: 40px 0;
 min-height: 400px;
}

/* ============================================================
6. ARTÍCULOS
============================================================ */

article {
 background-color: var(--color-blanco);
 border-radius: 8px;
 padding: 20px;
 margin-bottom: 30px;
 box-shadow: 0 2px 5px rgba(0,0,0,0.1);
}

article h2 {
 color: var(--color-secundario);
}

/* ============================================================
7. BARRA LATERAL
============================================================ */

aside {
 background-color: var(--color-gris);
 padding: 20px;
 border-radius: 8px;
}

/* ============================================================
8. PIE DE PÁGINA
============================================================ */

footer {
 background-color: var(--color-primario);
 color: var(--color-blanco);
 padding: 20px 0;
 text-align: center;
 margin-top: 40px;
}

/* ============================================================
9. UTILIDADES
============================================================ */

/* Clase para ocultar elementos visualmente pero mantenerlos
accesibles para lectores de pantalla */
.sr-only {
 position: absolute;
 width: 1px;
 height: 1px;
 padding: 0;
 margin: -1px;
 overflow: hidden;
 clip: rect(0, 0, 0, 0);
 border: 0;
}

/* Clase para alinear texto al centro */
.text-center {
 text-align: center;
}

===============================================================================
ERRORES COMUNES CON COMENTARIOS
===============================================================================

❌ **Error 1: Comentar código obvio**

❌ `/* Texto rojo */ p { color: red; }`
✅ `/* Color de alerta para mensajes de error */ p.error { color: red; }`

---

❌ **Error 2: No actualizar comentarios**

Si cambias el código y no el comentario, el comentario se vuelve obsoleto.

---

❌ **Error 3: Comentarios demasiado largos**

Un comentario de 10 líneas puede ser innecesario.
Divide el código en secciones y comenta cada parte.

---

❌ **Error 4: No usar comentarios para organización**

Un archivo CSS sin comentarios de sección es difícil de navegar.

---

❌ **Error 5: Comentarios que explican mal**

Un comentario incorrecto es peor que no tener comentario.
Si no estás seguro, no comentes o consulta con alguien.

===============================================================================
GLOSARIO DEL CAPÍTULO
===============================================================================

- **Comentario**: Texto en el código que el navegador ignora; se usa para documentar.
- **Comentario de una línea**: Comentario que ocupa solo una línea (`/* texto */`).
- **Comentario de bloque**: Comentario que puede ocupar varias líneas (`/* texto */`).
- **Documentación**: Información escrita que explica cómo funciona el código.
- **Depuración**: Proceso de encontrar y corregir errores.
- **TODO**: Comentario que indica una tarea pendiente.
- **FIXME**: Comentario que indica un error que debe corregirse.
- **Sección**: Parte del archivo CSS agrupada por función.
- **Código autodocumentado**: Código que es tan claro que no necesita comentarios.
- **Comentario obsoleto**: Comentario que ya no refleja el código actual.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Puedo usar comentarios para desactivar varias líneas de CSS?**

Sí, usando un comentario de bloque que abarque varias líneas.
/*
.clase {
propiedad: valor;
otra-propiedad: otro-valor;
}
*/

text

---

❓ **¿Los comentarios afectan al rendimiento de la página?**

No, el navegador ignora los comentarios completamente.
No afectan al rendimiento.

---

❓ **¿Qué hago si tengo que escribir un comentario muy largo?**

Divide el comentario en varias líneas dentro de `/* */`.
/*
Este es un comentario muy largo
que explica una decisión compleja
que tomamos durante el desarrollo.
*/

text

---

❓ **¿Debo comentar todo mi código?**

No. Comenta solo lo que necesita explicación.
El código claro no necesita comentarios.

---

❓ **¿Cómo manejo los comentarios en proyectos grandes?**

Usa una estructura de secciones estandarizada.
Define un formato de comentarios para todo el equipo.
Usa herramientas que generen documentación automática
(como JSDoc para CSS, aunque no es común).

- El código sin comentarios como un mapa sin leyenda.
- Los comentarios como la memoria del proyecto.

- Archivo CSS completo con comentarios.
- Comparación de comentarios útiles y no útiles.
