===============================================================================
INTRODUCCIÓN: EL COLOR EN LA WEB
===============================================================================

En CSS, el color se utiliza principalmente para dos propiedades:

1. **color**: Define el color del texto.
2. **background-color**: Define el color de fondo.

Pero también se usa en bordes (border-color), sombras, degradados y más.

---

📌 **Los modelos de color**

Un modelo de color es un sistema matemático para representar colores.
Los más comunes en CSS son:

- **Nombres de color** (palabras clave).
- **Hexadecimal** (#RRGGBB).
- **RGB** (Red, Green, Blue) → `rgb(255, 0, 0)`.
- **RGBA** (RGB con transparencia) → `rgba(255, 0, 0, 0.5)`.
- **HSL** (Hue, Saturation, Lightness) → `hsl(0, 100%, 50%)`.
- **HSLA** (HSL con transparencia) → `hsla(0, 100%, 50%, 0.5)`.

Cada uno tiene sus ventajas y casos de uso.
Vamos a explorarlos todos.

===============================================================================
1. NOMBRES DE COLOR
===============================================================================

CSS ofrece 140 nombres de color predefinidos, como `red`, `blue`, `green`,
`yellow`, `orange`, `purple`, `pink`, `brown`, `black`, `white`, etc.

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: red;
}

p {
    color: navy;
}

div {
    background-color: lightgray;
}
------------------------------------------------------------

---

📌 **Ventajas**

- Fáciles de recordar y escribir.
- Ideales para prototipos rápidos.
- No requieren herramientas externas.

---

📌 **Desventajas**

- Limitados a 140 colores.
- No permiten control preciso (no hay "rojo oscuro" estándar).
- No soportan transparencia.

---

📌 **Lista de colores comunes**

| Nombre      | Valor aproximado |
|-------------|------------------|
| red         | #FF0000          |
| green       | #008000          |
| blue        | #0000FF          |
| black       | #000000          |
| white       | #FFFFFF          |
| gray        | #808080          |
| yellow      | #FFFF00          |
| orange      | #FFA500          |
| purple      | #800080          |
| pink        | #FFC0CB          |
| brown       | #A52A2A          |
| navy        | #000080          |
| teal        | #008080          |
| silver      | #C0C0C0          |
| gold        | #FFD700          |

---

📌 **Referencia completa**

Puedes ver todos los 140 nombres de color en:
https://developer.mozilla.org/es/docs/Web/CSS/named-color

---

⚠️ **Consejo del Profesor Búho**

Usa nombres de color solo para pruebas rápidas.
Para proyectos serios, usa hexadecimal, RGB o HSL.

===============================================================================
2. CÓDIGOS HEXADECIMALES (#RRGGBB)
===============================================================================

El formato hexadecimal es el más común en CSS.
Representa colores con seis dígitos hexadecimales (0-9, A-F).

---

📌 **Estructura**
#RRGGBB

text

- **RR**: Cantidad de rojo (00 a FF).
- **GG**: Cantidad de verde (00 a FF).
- **BB**: Cantidad de azul (00 a FF).

- `00` es el valor más bajo (sin color).
- `FF` es el valor más alto (máximo color).

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: #FF0000;   /* Rojo puro */
}

p {
    color: #00FF00;   /* Verde puro */
}

div {
    background-color: #0000FF;   /* Azul puro */
}

body {
    background-color: #F5F5F5;   /* Gris muy claro */
}

footer {
    color: #333333;   /* Gris oscuro */
}
------------------------------------------------------------

---

📌 **Formato abreviado (#RGB)**

Si los tres pares de dígitos son iguales, puedes abreviar a tres dígitos.

- `#FF0000` → `#F00` (rojo).
- `#00FF00` → `#0F0` (verde).
- `#0000FF` → `#00F` (azul).
- `#333333` → `#333` (gris).
- `#FFFFFF` → `#FFF` (blanco).
- `#000000` → `#000` (negro).

---

🔵 **Ejemplos abreviados**

------------------------------------------------------------
h1 {
    color: #F00;   /* Rojo */
}

p {
    color: #333;   /* Gris oscuro */
}

body {
    background: #FFF;   /* Blanco */
}
------------------------------------------------------------

---

📌 **Ventajas**

- Preciso y amplio (más de 16 millones de colores).
- Muy utilizado en diseño web.
- Fácil de copiar de herramientas de diseño (Photoshop, Figma).

---

📌 **Desventajas**

- No es intuitivo para humanos (¿#2C3E50 es azul oscuro?).
- No incluye transparencia de forma nativa (hay que usar rgba o hsla para eso).

---

🟡 **Cómo elegir un color hexadecimal**

Si quieres un tono específico, usa herramientas como:
- **Color picker** de Chrome DevTools.
- **Figma**, **Photoshop** o **GIMP**.
- Sitios web como **Coolors.co** o **ColorHexa**.

===============================================================================
3. RGB (RED, GREEN, BLUE) - rgb()
===============================================================================

El modelo RGB usa tres valores numéricos entre 0 y 255
para representar la cantidad de rojo, verde y azul.

---

📌 **Estructura**
rgb(rojo, verde, azul)

text

- `0` = sin color.
- `255` = color máximo.

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: rgb(255, 0, 0);      /* Rojo */
}

p {
    color: rgb(0, 255, 0);      /* Verde */
}

div {
    background-color: rgb(0, 0, 255);   /* Azul */
}

body {
    background-color: rgb(245, 245, 245);   /* Gris claro */
}

footer {
    color: rgb(51, 51, 51);   /* Gris oscuro */
}
------------------------------------------------------------

---

📌 **Ventajas**

- Más intuitivo que hexadecimal (puedes imaginar la mezcla).
- Soportado en todos los navegadores.
- Permite valores con decimales en CSS moderno (opcional).

---

📌 **Desventajas**

- No tan compacto como hexadecimal.
- No incluye transparencia (para eso está rgba).

===============================================================================
4. RGBA (RGB CON TRANSPARENCIA) - rgba()
===============================================================================

rgba() es igual que rgb(), pero añade un cuarto valor:
el canal **alfa**, que controla la transparencia.

---

📌 **Estructura**
rgba(rojo, verde, azul, alfa)

text

- **alfa**: Número entre 0 (totalmente transparente) y 1 (totalmente opaco).

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: rgba(255, 0, 0, 0.5);   /* Rojo semitransparente */
}

p {
    background-color: rgba(0, 0, 255, 0.3);   /* Fondo azul transparente */
}

div {
    background-color: rgba(0, 0, 0, 0.7);   /* Fondo negro con 70% de opacidad */
}
------------------------------------------------------------

---

📌 **Casos de uso**

1. **Superposiciones**: Fondos semitransparentes sobre imágenes.
2. **Efectos de capa**: Texto sobre imágenes con mejor contraste.
3. **Sutileza**: Colores suaves que no distraen.
4. **Animaciones**: Transiciones entre opacidades.

---

🔵 **Ejemplo práctico: texto sobre imagen**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="hero">
    <h1>Bienvenido a mi sitio</h1>
    <p>Texto sobre una imagen de fondo.</p>
</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.hero {
    background-image: url('fondo.jpg');
    background-size: cover;
    padding: 100px 20px;
    text-align: center;
}

.hero h1 {
    color: white;
    background-color: rgba(0, 0, 0, 0.5);   /* Fondo oscuro semitransparente */
    padding: 20px;
    display: inline-block;
    border-radius: 8px;
}
------------------------------------------------------------

===============================================================================
5. HSL (HUE, SATURATION, LIGHTNESS) - hsl()
===============================================================================

HSL es un modelo de color más cercano a cómo los humanos perciben y describen colores.

---

📌 **Estructura**
hsl(matiz, saturación, luminosidad)

text

- **Matiz (Hue)**: Ángulo en la rueda de color (0 a 360).
  - 0 = rojo
  - 120 = verde
  - 240 = azul
  - 360 = rojo (de nuevo)

- **Saturación**: Intensidad del color (0% a 100%).
  - 0% = gris (sin color).
  - 100% = color completamente saturado.

- **Luminosidad**: Claridad del color (0% a 100%).
  - 0% = negro.
  - 100% = blanco.
  - 50% = color "puro".

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: hsl(0, 100%, 50%);   /* Rojo puro */
}

p {
    color: hsl(240, 100%, 50%);   /* Azul puro */
}

div {
    background-color: hsl(120, 100%, 50%);   /* Verde puro */
}

body {
    background-color: hsl(0, 0%, 90%);   /* Gris claro (90% de luz) */
}

footer {
    color: hsl(0, 0%, 20%);   /* Gris oscuro (20% de luz) */
}
------------------------------------------------------------

---

📌 **Ventajas de HSL**

1. **Intuitivo**: Es fácil ajustar el tono girando el ángulo.
2. **Armonías sencillas**: Para crear paletas, solo cambia el matiz.
3. **Control preciso de claridad**: Ajusta la luminosidad para versiones más claras u oscuras.
4. **Muy usado en diseño**: Es el modelo preferido por muchos diseñadores.

---

📌 **Ejemplo de armonía de colores**

Usa un solo matiz y varía la saturación y luminosidad:

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Paleta de azules */
.azul-oscuro   { color: hsl(210, 100%, 20%); }
.azul-medio    { color: hsl(210, 100%, 40%); }
.azul-claro    { color: hsl(210, 100%, 60%); }
.azul-muy-claro { color: hsl(210, 100%, 80%); }
------------------------------------------------------------

===============================================================================
6. HSLA (HSL CON TRANSPARENCIA) - hsla()
===============================================================================

hsla() es igual que hsl(), pero añade el canal alfa para transparencia.

---

📌 **Estructura**
hsla(matiz, saturación, luminosidad, alfa)

text

- **alfa**: 0 (transparente) a 1 (opaco).

---

🔵 **Ejemplos**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: hsla(0, 100%, 50%, 0.5);   /* Rojo semitransparente */
}

p {
    background-color: hsla(240, 100%, 50%, 0.3);   /* Fondo azul transparente */
}

div {
    background-color: hsla(0, 0%, 0%, 0.7);   /* Fondo negro con 70% de opacidad */
}
------------------------------------------------------------

---

📌 **Cuándo usar HSL vs RGB vs Hexadecimal**

| Modelo | Ideal para... |
|--------|---------------|
| **Hexadecimal** | Copiar colores de herramientas de diseño, simplicidad. |
| **RGB** | Control fino, cuando necesitas números exactos. |
| **HSL** | Crear paletas, ajustar tono, saturación y luminosidad de forma intuitiva. |
| **RGBA/HSLA** | Cuando necesitas transparencia. |

---

🟢 **Recomendación del Profesor Búho**

Aprende HSL. Es el modelo más poderoso para diseñar.
Una vez que entiendes el matiz, la saturación y la luminosidad,
puedes crear paletas completas con solo cambiar unos números.

===============================================================================
7. CURRENTCOLOR
===============================================================================

**currentColor** es una palabra clave especial en CSS que toma
el valor de la propiedad `color` del elemento actual.

---

🔵 **Ejemplo**

------------------------------------------------------------
CSS
------------------------------------------------------------
h1 {
    color: blue;
    border: 2px solid currentColor;  /* El borde será azul */
    box-shadow: 0 0 10px currentColor;  /* La sombra será azul */
}
------------------------------------------------------------

---

📌 **Ventajas**

- Mantiene la coherencia (todos los elementos usan el mismo color).
- Facilita cambios: cambia el `color` y todo lo que usa `currentColor` se actualiza.

---

🔵 **Ejemplo práctico**

------------------------------------------------------------
CSS
------------------------------------------------------------
.boton {
    color: #fff;
    background-color: #007BFF;
    border: 2px solid currentColor;   /* Borde blanco (porque color es blanco) */
    box-shadow: 0 2px 5px rgba(0,0,0,0.3);
}

.boton:hover {
    color: #FFD700;   /* Al hacer hover, el borde se vuelve dorado */
}
------------------------------------------------------------

---

📌 **Casos de uso**

- Bordes que coinciden con el color del texto.
- Iconos (usando SVG o fuentes de iconos) que heredan el color del texto.
- Sombras que se adaptan al color del texto.

===============================================================================
8. CONTRASTE Y ACCESIBILIDAD
===============================================================================

Elegir colores no es solo cuestión de estética.
**El contraste** es fundamental para que todos los usuarios puedan leer tu contenido.

---

📌 **¿Qué es el contraste?**

Es la diferencia de luminosidad entre el texto y su fondo.
Un buen contraste asegura que el texto sea legible.

---

📌 **Estándares de accesibilidad (WCAG)**

Las pautas WCAG (Web Content Accessibility Guidelines) definen:

- **AA (nivel mínimo)**: Contraste de al menos 4.5:1 para texto normal.
- **AAA (nivel mejorado)**: Contraste de al menos 7:1 para texto normal.

---

📌 **¿Cómo verificar el contraste?**

Usa herramientas como:

- **WebAIM Contrast Checker**: https://webaim.org/resources/contrastchecker/
- **Chrome DevTools** (inspecciona el elemento y mira la herramienta de contraste).
- **Figma** o **Adobe Color**.

---

🔴 **Ejemplos de mal contraste**

❌ Texto gris claro sobre fondo blanco (apenas legible).
❌ Texto rojo sobre fondo verde (difícil de leer para daltónicos).
❌ Texto pequeño sobre fondo con imagen de fondo (poco contraste).

---

🟢 **Buenas prácticas**

1. **Texto oscuro sobre fondo claro** (alta legibilidad).
2. **Texto claro sobre fondo oscuro** (también válido).
3. **Evita pares de colores que los daltónicos confundan** (rojo/verde, azul/amarillo).
4. **Usa herramientas de simulación de daltonismo** para probar tus diseños.
5. **No dependas solo del color** para comunicar información (usa también iconos o texto).

---

🔵 **Ejemplo de diseño accesible**

------------------------------------------------------------
CSS
------------------------------------------------------------
body {
    background-color: #FFFFFF;   /* Fondo blanco */
    color: #222222;             /* Texto gris oscuro (alto contraste) */
}

.boton {
    background-color: #0056b3;   /* Azul oscuro */
    color: white;               /* Texto blanco */
    border: none;
    padding: 10px 20px;
    border-radius: 5px;
}
------------------------------------------------------------

Contraste: blanco sobre azul oscuro → muy legible.

===============================================================================
9. PALETAS DE COLORES Y ARMONÍA
===============================================================================

Crear una paleta de colores es un arte y una ciencia.
Aquí tienes algunas técnicas para crear armonías.

---

📌 **1. Monocromática**

Usa un solo matiz y varía saturación y luminosidad.

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
hsl(210, 100%, 20%)   /* Azul muy oscuro */
hsl(210, 100%, 40%)   /* Azul medio */
hsl(210, 100%, 60%)   /* Azul claro */
hsl(210, 100%, 80%)   /* Azul muy claro */
------------------------------------------------------------

---

📌 **2. Análoga**

Usa matices adyacentes en la rueda de colores.

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
hsl(10, 80%, 50%)    /* Rojo-naranja */
hsl(30, 80%, 50%)    /* Naranja */
hsl(50, 80%, 50%)    /* Amarillo */
------------------------------------------------------------

---

📌 **3. Complementaria**

Usa matices opuestos en la rueda (diferencia de 180°).

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
hsl(200, 80%, 50%)   /* Azul */
hsl(20, 80%, 50%)    /* Naranja (opuesto al azul) */
------------------------------------------------------------

---

📌 **4. Triádica**

Usa tres matices equidistantes (120° de diferencia).

------------------------------------------------------------
EJEMPLO
------------------------------------------------------------
hsl(0, 70%, 50%)     /* Rojo */
hsl(120, 70%, 50%)   /* Verde */
hsl(240, 70%, 50%)   /* Azul */
------------------------------------------------------------

---

📌 **Herramientas para crear paletas**

- **Coolors.co**: Genera paletas automáticamente.
- **Adobe Color**: Paletas basadas en teoría del color.
- **Color Hunt**: Inspiración con paletas prediseñadas.

===============================================================================
ERRORES COMUNES CON COLORES
===============================================================================

❌ **Error 1: Usar colores sin suficiente contraste**

   Asegúrate de que el texto sea legible.
   Usa herramientas de verificación de contraste.

---

❌ **Error 2: Depender solo del color para información**

   No uses color como único indicador (por ejemplo, "los campos en rojo son obligatorios").
   Añade también texto o iconos.

---

❌ **Error 3: Usar demasiados colores**

   Limítate a 3-5 colores principales en un diseño.
   Demasiados colores crean confusión visual.

---

❌ **Error 4: No probar en diferentes pantallas**

   Los colores pueden verse diferentes en distintos monitores.
   Siempre prueba en varias pantallas y dispositivos.

---

❌ **Error 5: Usar hexadecimal sin entenderlo**

   Aprende RGB y HSL para tener más control.

GLOSARIO DEL CAPÍTULO
===============================================================================

- **Color**: Propiedad CSS que define el color del texto.
- **background-color**: Propiedad CSS que define el color de fondo.
- **Hex (#RRGGBB)**: Representación hexadecimal de colores.
- **RGB**: Modelo de color basado en rojo, verde y azul.
- **RGBA**: RGB con transparencia (canal alfa).
- **HSL**: Modelo de color basado en matiz, saturación y luminosidad.
- **HSLA**: HSL con transparencia (canal alfa).
- **Alfa**: Canal de transparencia (0 = invisible, 1 = opaco).
- **currentColor**: Palabra clave que toma el valor de la propiedad color.
- **Contraste**: Diferencia de luminosidad entre texto y fondo.
- **Accesibilidad**: Diseño que permite que todos los usuarios, incluidos personas con discapacidad, puedan usar el sitio.
- **Paleta de colores**: Conjunto de colores que se usan en un diseño.
- **Matiz (Hue)**: El ángulo en la rueda de color.
- **Saturación**: Intensidad del color.
- **Luminosidad**: Claridad del color.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Qué modelo de color debería usar?**

   Para proyectos profesionales, HSL es el más recomendado por su flexibilidad.
   Hex es bueno cuando copias colores de herramientas de diseño.
   RGB es útil si trabajas con valores numéricos exactos.

---

❓ **¿Cómo hago para que un color tenga transparencia?**

   Usa `rgba()` o `hsla()`. También puedes usar `opacity`, pero afecta a todo el elemento
   (incluido el contenido), mientras que `rgba` solo afecta al fondo.

---

❓ **¿Cuál es la diferencia entre `opacity` y `rgba`?**

   - `opacity`: Afecta a todo el elemento y su contenido (texto, bordes, etc.).
   - `rgba`: Solo afecta al color específico (fondo, texto, borde) y no al contenido.

---

❓ **¿Qué es currentColor y cuándo usarlo?**

   `currentColor` toma el valor de la propiedad `color` del elemento.
   Úsalo para bordes, sombras o iconos que quieras que coincidan con el color del texto.

---

❓ **¿Cómo puedo verificar el contraste de mis colores?**

   Usa herramientas como WebAIM Contrast Checker o el inspector de Chrome.
