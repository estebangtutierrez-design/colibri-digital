===============================================================================
ANTES DE EMPEZAR: LA ESTRUCTURA DEL PROYECTO
===============================================================================

Para trabajar con CSS, necesitamos una estructura de archivos organizada.

Un proyecto web típico tiene esta estructura:

------------------------------------------------------------
ESTRUCTURA DE UN PROYECTO WEB BÁSICO
------------------------------------------------------------
mi_proyecto/
│
├── index.html
├── style.css
└── assets/
    └── (imágenes, fuentes, etc.)
------------------------------------------------------------

- **index.html**: El archivo principal HTML.
- **style.css**: El archivo CSS donde escribiremos los estilos.
- **assets/**: Carpeta para recursos (imágenes, fuentes, etc.).

---

📌 **Nomenclatura recomendada**

- Usa siempre minúsculas para los nombres de archivos.
- Usa guiones bajos (_) o guiones (-) para separar palabras.
  Ejemplo: `mi_estilo.css` o `mi-estilo.css`.
- El archivo principal CSS suele llamarse `style.css` o `styles.css`.

---

🔴 **¡Importante!**

El archivo CSS **debe tener la extensión .css**.
Nunca .txt, .doc, .pdf o cualquier otra.

El navegador solo reconoce archivos CSS con la extensión .css.

===============================================================================
PASO 1: CREAR EL ARCHIVO CSS
===============================================================================

Abre tu editor de texto (Bluefish, Visual Studio Code, Geany, etc.).

1. Crea un nuevo archivo.
2. Guárdalo con el nombre:
style.css

text
3. Asegúrate de que la extensión sea .css.

---

🟡 **Ubicación del archivo**

Coloca `style.css` en la misma carpeta que tu `index.html`.

Si tu proyecto crece, puedes organizar los CSS en subcarpetas:
mi_proyecto/
│
├── index.html
├── css/
│ └── style.css
└── assets/

text

Para este capítulo, mantendremos `style.css` en la raíz del proyecto.

===============================================================================
PASO 2: ESCRIBIR NUESTRO PRIMER CÓDIGO CSS
===============================================================================

Abre `style.css` y escribe lo siguiente:

------------------------------------------------------------
CÓDIGO CSS (style.css)
------------------------------------------------------------
/*
   ============================================================
   MI PRIMER ARCHIVO CSS
   Estilos para mi primera página web
   ============================================================
*/

/* Estilo para el título principal */
h1 {
    color: blue;
    font-size: 36px;
    text-align: center;
}

/* Estilo para los párrafos */
p {
    color: #333333;
    font-size: 18px;
    line-height: 1.6;
}

/* Estilo para el fondo de la página */
body {
    background-color: #f0f0f0;
    font-family: Arial, sans-serif;
    margin: 20px;
}
------------------------------------------------------------
FIN DEL CÓDIGO
------------------------------------------------------------

===============================================================================
PASO 3: CREAR EL ARCHIVO HTML
===============================================================================

Ahora necesitamos un archivo HTML que use este CSS.

Crea un archivo llamado `index.html` en la misma carpeta y escribe:

------------------------------------------------------------
CÓDIGO HTML (index.html)
------------------------------------------------------------
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi primera página con CSS</title>
    <!-- Aquí vinculamos nuestro archivo CSS -->
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>¡Bienvenido a mi primera página con CSS!</h1>
    <p>Este es un párrafo de ejemplo. El texto se ve más limpio y profesional gracias a CSS.</p>
    <p>CSS nos permite controlar colores, fuentes, márgenes y mucho más.</p>
</body>
</html>
------------------------------------------------------------
FIN DEL CÓDIGO
------------------------------------------------------------

===============================================================================
PASO 4: VINCULAR CSS CON HTML
===============================================================================

La magia ocurre en esta línea dentro del <head>:

------------------------------------------------------------
<link rel="stylesheet" href="style.css">
------------------------------------------------------------

- **rel="stylesheet"**: Indica que es una hoja de estilos.
- **href="style.css"**: Ruta al archivo CSS.

---

📌 **Otras formas de vincular CSS**

1. **CSS interno** (dentro del HTML):

------------------------------------------------------------
<style>
    h1 { color: blue; }
</style>
------------------------------------------------------------

2. **CSS en línea** (directamente en el elemento):

------------------------------------------------------------
<h1 style="color: blue;">Título</h1>
------------------------------------------------------------

---

🔴 **¿Cuál es la mejor?**

**Siempre que sea posible, usa CSS externo (archivo .css).**
- Es más limpio.
- Es reutilizable en varias páginas.
- Es más fácil de mantener.
- Mejora el rendimiento (se cachea).

===============================================================================
PASO 5: ABRIR LA PÁGINA EN EL NAVEGADOR
===============================================================================

1. Guarda ambos archivos (`index.html` y `style.css`).
2. Abre `index.html` en tu navegador (doble clic o arrastrar al navegador).

Deberías ver:
- Fondo gris claro.
- Título azul, centrado y grande.
- Texto de los párrafos en gris oscuro, con un tamaño legible.
- Fuente Arial (o sans-serif).

---

🟢 **Si no ves los estilos, verifica:**

1. Que el archivo CSS esté en la misma carpeta que el HTML.
2. Que el nombre del archivo CSS sea exactamente `style.css`.
3. Que la etiqueta `<link>` esté bien escrita.
4. Que hayas guardado ambos archivos.
5. Que no haya errores de sintaxis en el CSS (faltan punto y coma, corchetes, etc.).

EXPLICACIÓN LÍNEA POR LÍNEA DEL CÓDIGO CSS
===============================================================================

Vamos a desglosar cada parte de nuestro primer CSS para que entiendas
exactamente qué hace cada línea.

---

**Línea 1-5: Comentarios de bloque**

------------------------------------------------------------
/*
   ============================================================
   MI PRIMER ARCHIVO CSS
   Estilos para mi primera página web
   ============================================================
*/
------------------------------------------------------------

Los comentarios en CSS se escriben entre `/*` y `*/`.
Todo lo que está dentro es ignorado por el navegador.
Sirven para documentar el código.

---

**Línea 7: Comentario de una línea**

------------------------------------------------------------
/* Estilo para el título principal */
------------------------------------------------------------

También puedes usar comentarios de una línea.
Son útiles para explicar reglas específicas.

---

**Líneas 8-12: Regla para h1**

------------------------------------------------------------
h1 {
    color: blue;
    font-size: 36px;
    text-align: center;
}
------------------------------------------------------------

- **Selector**: `h1` → se aplica a todos los elementos `<h1>`.
- **Declaración 1**: `color: blue;` → texto azul.
- **Declaración 2**: `font-size: 36px;` → tamaño de fuente de 36 píxeles.
- **Declaración 3**: `text-align: center;` → texto centrado.

---

**Líneas 14-18: Regla para p**

------------------------------------------------------------
p {
    color: #333333;
    font-size: 18px;
    line-height: 1.6;
}
------------------------------------------------------------

- **Selector**: `p` → se aplica a todos los elementos `<p>`.
- **color: #333333;**: Color gris oscuro (notación hexadecimal).
- **font-size: 18px;**: Tamaño de fuente de 18 píxeles.
- **line-height: 1.6;**: Espaciado entre líneas (1.6 veces el tamaño de fuente).

---

**Líneas 20-24: Regla para body**

------------------------------------------------------------
body {
    background-color: #f0f0f0;
    font-family: Arial, sans-serif;
    margin: 20px;
}
------------------------------------------------------------

- **Selector**: `body` → se aplica al cuerpo de la página.
- **background-color: #f0f0f0;**: Fondo gris muy claro.
- **font-family: Arial, sans-serif;**: Fuente Arial, y si no está disponible, cualquier fuente sans-serif.
- **margin: 20px;**: Margen exterior de 20 píxeles alrededor del contenido.

===============================================================================
CÓMO FUNCIONA LA SINTAXIS DE CSS
===============================================================================

Toda regla CSS sigue esta estructura:

------------------------------------------------------------
SINTAXIS GENERAL
------------------------------------------------------------
selector {
    propiedad: valor;
    propiedad: valor;
}
------------------------------------------------------------

- El **selector** indica qué elemento se va a estilizar.
- Las **declaraciones** van entre `{` y `}`.
- Cada declaración tiene una **propiedad** y un **valor**, separados por `:`.
- Cada declaración termina con `;` (punto y coma).

---

📌 **Reglas de sintaxis importantes**

1. **El punto y coma es obligatorio** después de cada declaración.
2. **Los espacios no son importantes** (pero ayudan a la legibilidad).
3. **Los nombres de propiedades son sensibles a mayúsculas** (siempre en minúsculas).
4. **Los valores pueden ser palabras clave, números, colores, etc.**

---

🔴 **Ejemplo de error común**

------------------------------------------------------------
h1 {
    color blue;          /* ❌ Falta los dos puntos (:) */
    font-size: 36px      /* ❌ Falta el punto y coma (;) */
}
------------------------------------------------------------

**Código correcto:**
------------------------------------------------------------
h1 {
    color: blue;         /* ✅ Correcto */
    font-size: 36px;     /* ✅ Correcto */
}
------------------------------------------------------------

===============================================================================
USANDO EL INSPECTOR DEL NAVEGADOR (HERRAMIENTA DE DESARROLLO)
===============================================================================

Una de las herramientas más importantes para trabajar con CSS
es el **inspector del navegador**.

---

📌 **Cómo abrirlo**

- **Chrome**: F12 o Clic derecho → Inspeccionar.
- **Firefox**: F12 o Clic derecho → Inspeccionar.
- **Edge**: F12 o Clic derecho → Inspeccionar.

---

🔍 **¿Qué puedes hacer con él?**

1. **Ver los estilos aplicados** a cualquier elemento.
2. **Modificar CSS en vivo** y ver los cambios al instante.
3. **Desactivar reglas** para comprobar su efecto.
4. **Ver la especificidad** de cada regla.
5. **Agregar nuevas reglas** temporalmente.
6. **Depurar errores** de diseño.

---

🟡 **Práctica guiada**

1. Abre tu página en Chrome.
2. Haz clic derecho en el título h1 → Inspeccionar.
3. En el panel de estilos (a la derecha), verás todas las reglas CSS que afectan a ese elemento.
4. Modifica el valor de `color: blue;` a `color: red;` y observa cómo cambia al instante.
5. Desmarca (check) la regla para ver cómo se ve sin ella.
6. Agrega una nueva regla: `text-decoration: underline;` y observa.

---

💡 **El inspector es tu mejor amigo.**
Úsalo siempre que tengas dudas sobre por qué un estilo no se aplica
o cómo se vería un cambio sin modificar el archivo original.

===============================================================================
ERRORES COMUNES AL CREAR UN PRIMER ARCHIVO CSS
===============================================================================

❌ **Error 1: No vincular el archivo CSS correctamente**

La etiqueta `<link>` debe estar dentro del `<head>`.
El atributo `href` debe apuntar a la ubicación correcta del archivo.

Incorrecto:
<link rel="stylesheet" href="style.css"> <!-- fuera del <head> --> ```
Correcto:

text
<head>
    <link rel="stylesheet" href="style.css">
</head>
❌ Error 2: Olvidar la extensión .css

El archivo debe llamarse style.css, no style.txt o style.

❌ Error 3: Errores de sintaxis

Olvidar los dos puntos (:) entre propiedad y valor.

Olvidar el punto y coma (;) al final de la declaración.

Olvidar cerrar las llaves (}).

Usa un editor con resaltado de sintaxis para detectar estos errores.

❌ Error 4: No guardar los archivos

Después de modificar el CSS, guarda el archivo (Ctrl+S) y actualiza
el navegador (F5) para ver los cambios.

❌ Error 5: Confundir class y id en CSS

Clase: .mi-clase (se escribe con un punto).

ID: #mi-id (se escribe con un numeral).

Si usas #mi-clase cuando debería ser .mi-clase, el estilo no se aplicará.

===============================================================================
GLOSARIO DEL CAPÍTULO
===============================================================================

CSS externo: Archivo .css separado del HTML, vinculado con <link>.

CSS interno: Estilos definidos dentro de <style> en el <head>.

CSS en línea: Estilos aplicados directamente en el atributo style de un elemento.

Comentario: Texto en el código que el navegador ignora; se usa para documentar.

Regla CSS: Conjunto de selector y declaraciones entre llaves.

Declaración: Par propiedad:valor que define un estilo específico.

Selector: Parte de la regla CSS que indica a qué elementos se aplica.

Propiedad: Característica a modificar (color, font-size, etc.).

Valor: El valor asignado a la propiedad (blue, 36px, etc.).

Inspector: Herramienta del navegador para depurar y modificar CSS en vivo.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ ¿Puedo tener varios archivos CSS en una misma página?

Sí. Puedes vincular varios archivos con múltiples etiquetas <link>.
El orden en que los enlaces aparecen determina la prioridad.

Ejemplo:

text
<link rel="stylesheet" href="normalize.css">
<link rel="stylesheet" href="style.css">
style.css tendrá prioridad sobre normalize.css
cuando haya conflictos de la misma especificidad.

❓ ¿Puedo usar CSS en una página sin crear un archivo?

Sí, con CSS interno (<style>) o CSS en línea (style="...").
Pero no es recomendable para proyectos grandes.

❓ ¿Qué es mejor, clase o ID para estilos?

Clase (.) es mejor para estilos reutilizables.
ID (#) es para elementos únicos (y tiene mayor especificidad).

Como regla general: usa clases para casi todo y guarda los IDs
para elementos realmente únicos (como un header o un footer).

❓ ¿Qué hago si mi CSS no se actualiza en el navegador?

Puede ser por caché. Prueba:

Actualizar la página con Ctrl+F5 (actualización forzada).

Abrir la página en modo incógnito.

Limpiar la caché del navegador.

❓ ¿Cómo sé qué propiedades existen y qué valores aceptan?

Consulta la documentación de MDN (Mozilla Developer Network):
https://developer.mozilla.org/es/docs/Web/CSS

Es la mejor referencia para CSS.
