===============================================================================
INTRODUCCIГ“N: ВҝQUГү ES DISPLAY?
===============================================================================

`display` es una propiedad CSS que define el **tipo de caja** de un elemento.
Determina cГіmo se comporta el elemento en el flujo del documento.

Cada elemento HTML tiene un valor de `display` por defecto:

- **Elementos de bloque (block)**: `<div>`, `<p>`, `<h1>`, `<section>`, `<article>`, `<header>`, `<footer>`, `<nav>`, `<main>`, `<ul>`, `<ol>`, `<li>`, `<table>`, `<form>`, `<hr>`, `<address>`, `<blockquote>`, `<pre>`, `<figure>`, `<figcaption>`.

- **Elementos en lГӯnea (inline)**: `<span>`, `<a>`, `<strong>`, `<em>`, `<b>`, `<i>`, `<u>`, `<s>`, `<small>`, `<sub>`, `<sup>`, `<img>`, `<input>`, `<button>`, `<label>`, `<select>`, `<textarea>`, `<br>`, `<code>`, `<var>`, `<samp>`, `<kbd>`, `<abbr>`, `<cite>`, `<q>`, `<time>`, `<mark>`.

- **Elementos en lГӯnea-bloque (inline-block)**: `<img>`, `<input>`, `<button>`, `<textarea>`, `<select>`, `<video>`, `<canvas>`.

---

рҹ“Ң **Valores principales de display**

| Valor | Comportamiento |
|-------|----------------|
| `block` | Ocupa todo el ancho disponible. Se apila verticalmente. Acepta width, height, padding, border, margin. |
| `inline` | Solo ocupa el espacio de su contenido. No acepta width ni height. Se coloca horizontalmente. |
| `inline-block` | Se comporta como inline (fluye en lГӯnea) pero acepta width y height como block. |
| `none` | Oculta el elemento. No ocupa espacio en la pГЎgina. |

===============================================================================
1. DISPLAY: BLOCK (ELEMENTOS DE BLOQUE)
===============================================================================

Los elementos con `display: block` son los mГЎs comunes en HTML.

---

рҹ“Ң **CaracterГӯsticas**

1. **Ocupan todo el ancho disponible** (por defecto, 100% del contenedor).
2. **Se apilan verticalmente** (cada uno en una nueva lГӯnea).
3. **Aceptan `width`, `height`, `padding`, `border`, `margin`** en los cuatro lados.
4. **El valor por defecto** para la mayorГӯa de elementos estructurales.

---

рҹ”ө **Ejemplo: elementos block por defecto**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div>Div 1</div>
<div>Div 2</div>
<div>Div 3</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
div {
    background-color: #3498DB;
    color: white;
    padding: 10px;
    margin: 5px 0;
}
------------------------------------------------------------

**Resultado:** Los tres divs se apilan verticalmente, ocupando todo el ancho.

---

рҹ“Ң **Usar `display: block` en otros elementos**

Puedes convertir un elemento en lГӯnea (como `<span>` o `<a>`) en bloque.

------------------------------------------------------------
CSS
------------------------------------------------------------
a {
    display: block;
    padding: 10px;
    background-color: #3498DB;
    color: white;
    margin: 5px 0;
    text-decoration: none;
}
------------------------------------------------------------

**Resultado:** Los enlaces ahora ocupan todo el ancho y se apilan verticalmente.

---

рҹ“Ң **Usos comunes de `display: block`**

1. **MenГәs de navegaciГіn verticales**.
2. **Botones que ocupan todo el ancho**.
3. **Secciones y contenedores**.
4. **Tarjetas y tarjetas de producto**.

===============================================================================
2. DISPLAY: INLINE (ELEMENTOS EN LГҚNEA)
===============================================================================

Los elementos con `display: inline` son aquellos que fluyen en el texto.

---

рҹ“Ң **CaracterГӯsticas**

1. **Ocupan solo el espacio de su contenido** (no todo el ancho).
2. **Se colocan horizontalmente** (uno al lado del otro).
3. **NO aceptan `width` ni `height`** (se ignoran).
4. **Aceptan `padding` horizontal (left/right)** pero no vertical.
5. **Aceptan `margin` horizontal (left/right)** pero no vertical.
6. **El valor por defecto** para elementos de texto como `<span>`, `<a>`, `<strong>`.

---

рҹ”ө **Ejemplo: elementos inline por defecto**

------------------------------------------------------------
HTML
------------------------------------------------------------
<a href="#">Enlace 1</a>
<a href="#">Enlace 2</a>
<a href="#">Enlace 3</a>
------------------------------------------------------------

CSS:
------------------------------------------------------------
a {
    background-color: #28A745;
    color: white;
    padding: 10px;   /* El padding vertical no se aplica */
    margin: 10px;    /* El margin vertical no se aplica */
    text-decoration: none;
}
------------------------------------------------------------

**Resultado:** Los enlaces se colocan horizontalmente.
El padding y margin vertical no tienen efecto.

---

рҹ“Ң **Intentar usar width en inline no funciona**

------------------------------------------------------------
CSS
------------------------------------------------------------
span {
    width: 200px;     /* вқҢ Ignorado */
    height: 100px;    /* вқҢ Ignorado */
    background-color: #3498DB;
    color: white;
    padding: 10px;
}
------------------------------------------------------------

**Resultado:** el span ocupa solo el espacio de su texto.

---

рҹ“Ң **Usos comunes de `display: inline`**

1. **Enlaces en navegaciГіn horizontal**.
2. **Texto dentro de pГЎrrafos**.
3. **Elementos de formato** (strong, em, i).
4. **Iconos y pequeГұos elementos decorativos**.

---

рҹ“Ң **CГіmo hacer que un elemento inline funcione con width y height**

Usa `display: inline-block` (ver siguiente secciГіn).

===============================================================================
3. DISPLAY: INLINE-BLOCK (ELEMENTOS HГҚBRIDOS)
===============================================================================

`inline-block` combina lo mejor de `inline` y `block`.

---

рҹ“Ң **CaracterГӯsticas**

1. **Se colocan horizontalmente** (como inline).
2. **Aceptan `width`, `height`, `padding`, `border`, `margin`** (como block).
3. **Mantienen el flujo en lГӯnea** (no rompen la lГӯnea).
4. **Гҡtiles para layouts con elementos lado a lado**.

---

рҹ”ө **Ejemplo: inline-block para crear columnas**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="columna">Columna 1</div>
<div class="columna">Columna 2</div>
<div class="columna">Columna 3</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.columna {
    display: inline-block;
    width: 30%;
    padding: 20px;
    background-color: #3498DB;
    color: white;
    margin: 1%;
    box-sizing: border-box;
    border-radius: 8px;
}
------------------------------------------------------------

**Resultado:** Tres columnas de 30% de ancho, lado a lado.

---

рҹ“Ң **Ventajas de inline-block**

1. **Se puede controlar el ancho y alto**.
2. **Se pueden poner elementos lado a lado** sin floats o flexbox.
3. **Respetan el espaciado horizontal**.
4. **Son compatibles con todos los navegadores**.

---

рҹ“Ң **Desventajas de inline-block**

1. **El espacio en blanco entre elementos** (en HTML) crea un espacio visible.
   (SoluciГіn: eliminar espacios o usar `font-size: 0` en el padre).

2. **No permiten centrado vertical fГЎcil** (como flexbox).

---

рҹ”ө **Ejemplo del problema del espacio en blanco**

HTML
------------------------------------------------------------

<div class="columna">2</div>
<div class="columna">3</div>
------------------------------------------------------------

**Problema:** Hay un espacio entre las columnas debido a los saltos de lГӯnea en el HTML.

**SoluciГіn 1:** Eliminar espacios en el HTML.
<div class="columna">1</div><div class="columna">2</div><div class="columna">3</div> ```
SoluciГіn 2: Usar font-size: 0 en el padre.

рҹ“Ң Usos comunes de display: inline-block

MenГәs de navegaciГіn horizontales (con enlaces).

GalerГӯas de imГЎgenes.

рҹ“Ң CaracterГӯsticas

рҹ“Ң Usos comunes de display: none

MenГәs desplegables (mostrar/ocultar).

рҹ“Ң Usos comunes de visibility: hidden

===============================================================================
ERRORES COMUNES CON DISPLAY
===============================================================================

вқҢ Error 1: Usar width y height en elementos inline

No funciona. Los elementos inline ignoran width y height.
SoluciГіn: usa display: inline-block o display: block.

вқҢ Error 2: Usar padding y margin vertical en inline

No funciona. El padding y margin vertical son ignorados en inline.
SoluciГіn: usa display: inline-block o display: block.

вқҢ Error 3: Confundir display: none con visibility: hidden

display: none elimina el elemento del flujo.
visibility: hidden oculta el elemento pero mantiene el espacio.

вқҢ Error 4: No considerar el espacio en blanco con inline-block

Los espacios en el HTML crean espacios visibles entre elementos inline-block.
SoluciГіn: elimina espacios o usa font-size: 0 en el padre.

вқҢ Error 5: Usar inline-block para layouts complejos

Para layouts complejos, es mejor usar Flexbox o Grid.
SoluciГіn: aprende Flexbox y Grid (capГӯtulos posteriores).
