Hoy vas a aprender a dominar la profundidad en CSS.

===============================================================================
INTRODUCCIÓN: ¿QUÉ ES Z-INDEX?
===============================================================================

`z-index` es una propiedad CSS que controla el orden de superposición
de los elementos en el eje Z (profundidad).

---

📌 **El eje Z**

Imagina que tu página tiene tres dimensiones:

- **Eje X**: horizontal (izquierda → derecha).
- **Eje Y**: vertical (arriba → abajo).
- **Eje Z**: profundidad (detrás → delante).

`z-index` controla la posición en el eje Z.

---

📌 **Sintaxis**
z-index: valor;

text

---

📌 **Valores de z-index**

| Valor | Comportamiento |
|-------|----------------|
| `auto` | El elemento hereda el orden de apilamiento de su padre (por defecto). |
| `número` | Un número entero (positivo o negativo). Mayor número = más arriba. |
| `initial` | Restablece al valor por defecto (`auto`). |
| `inherit` | Hereda el valor del elemento padre. |

---

📌 **Características de z-index**

1. **Solo funciona en elementos con `position: relative`, `absolute`, `fixed` o `sticky`**.
2. **No funciona en elementos con `position: static`** (el valor por defecto).
3. **Valores más altos = más cerca del usuario** (se ven encima).
4. **Valores más bajos = más lejos del usuario** (se ven debajo).
5. **Valores negativos** colocan el elemento por debajo del flujo normal.

---

🟢 **Regla de oro**

**`z-index` solo funciona en elementos POSICIONADOS**
**(`position` diferente de `static`).**

===============================================================================
1. Z-INDEX: VALORES NUMÉRICOS
===============================================================================

Los valores numéricos (enteros) controlan el orden de superposición.

---

🔵 **Ejemplo: tres elementos superpuestos**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="caja1">Caja 1</div>
<div class="caja2">Caja 2</div>
<div class="caja3">Caja 3</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.caja1, .caja2, .caja3 {
    position: absolute;
    width: 200px;
    height: 150px;
    padding: 20px;
    color: white;
    font-weight: bold;
    border-radius: 8px;
}

.caja1 {
    top: 20px;
    left: 20px;
    background-color: #3498DB;
    z-index: 1;
}

.caja2 {
    top: 60px;
    left: 60px;
    background-color: #28A745;
    z-index: 2;
}

.caja3 {
    top: 100px;
    left: 100px;
    background-color: #DC3545;
    z-index: 3;
}
------------------------------------------------------------

**Resultado:** La caja 3 (z-index: 3) está por encima de todas.
La caja 2 (z-index: 2) está en el medio.
La caja 1 (z-index: 1) está por debajo.

---

📌 **Valores positivos y negativos**

| z-index | Posición |
|---------|----------|
| `100` | Muy por encima |
| `10` | Por encima |
| `1` | Ligeramente por encima |
| `0` | En el flujo normal |
| `-1` | Por debajo del flujo normal |
| `-100` | Muy por debajo |

---

🔵 **Ejemplo: z-index negativo**

------------------------------------------------------------
CSS
------------------------------------------------------------
.caja {
    position: absolute;
    width: 200px;
    height: 150px;
    background-color: #3498DB;
    z-index: -1;   /* Por debajo del flujo normal */
}
------------------------------------------------------------

**Resultado:** el elemento se coloca por debajo del contenido normal.

===============================================================================
2. Z-INDEX: AUTO (VALOR POR DEFECTO)
===============================================================================

`z-index: auto` es el valor por defecto.
El orden de apilamiento se determina por el orden en el HTML
(el último elemento está por encima).

---

🔵 **Ejemplo con z-index: auto**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="caja1">Caja 1 (primera)</div>
<div class="caja2">Caja 2 (segunda)</div>
<div class="caja3">Caja 3 (tercera)</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.caja1, .caja2, .caja3 {
    position: absolute;
    width: 200px;
    height: 150px;
    padding: 20px;
    color: white;
}

.caja1 {
    top: 20px;
    left: 20px;
    background-color: #3498DB;
    z-index: auto;   /* Por defecto */
}

.caja2 {
    top: 60px;
    left: 60px;
    background-color: #28A745;
    z-index: auto;
}

.caja3 {
    top: 100px;
    left: 100px;
    background-color: #DC3545;
    z-index: auto;
}
------------------------------------------------------------

**Resultado:** La caja 3 (la última en el HTML) está por encima.
La caja 2 está en el medio.
La caja 1 está por debajo.

---

📌 **Cuando usar z-index: auto**

- Cuando no necesitas controlar la superposición.
- Cuando quieres que el elemento herede el contexto de apilamiento de su padre.

===============================================================================
3. CONTEXTO DE APILAMIENTO (STACKING CONTEXT)
===============================================================================

El **contexto de apilamiento** es un concepto fundamental para entender `z-index`.

---

📌 **¿Qué es el contexto de apilamiento?**

Es un "grupo" de elementos que se apilan juntos, independientemente
de otros grupos.

---

📌 **¿Qué crea un nuevo contexto de apilamiento?**

1. **`position: relative/absolute/fixed/sticky` con `z-index`** (diferente de `auto`).
2. **`opacity` menor de 1**.
3. **`transform`** aplicado.
4. **`filter`** aplicado.
5. **`will-change`** con valor.
6. **`container-type`** con valor.
7. **`isolation: isolate`**.
8. **Elementos `<html>` y `<body>`** (por defecto).

---

🔵 **Ejemplo: contexto de apilamiento**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="padre">
    <div class="hijo1">Hijo 1</div>
    <div class="hijo2">Hijo 2</div>
</div>
<div class="otro">Otro elemento</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.padre {
    position: relative;
    z-index: 10;   /* Crea un contexto de apilamiento */
}

.hijo1 {
    position: absolute;
    z-index: 100;   /* Solo dentro del contexto del padre */
}

.otro {
    position: absolute;
    z-index: 20;   /* Está en otro contexto */
}
------------------------------------------------------------

**Resultado:** El hijo1 tiene z-index 100 dentro del contexto del padre.
El elemento "otro" tiene z-index 20 en otro contexto.
Si "otro" debe estar por encima del padre, su z-index debe ser mayor que el del padre
(no solo mayor que el del hijo).

---

📌 **Regla de oro del contexto de apilamiento**

**Los elementos en diferentes contextos de apilamiento se comparan**
**por el z-index del contexto, no por el z-index de los elementos internos.**

===============================================================================
4. POR QUÉ Z-INDEX A VECES NO FUNCIONA
===============================================================================

Uno de los problemas más comunes con `z-index` es que "no funciona".

---

📌 **Razón 1: El elemento no tiene position definido**

`z-index` solo funciona en elementos con `position: relative`, `absolute`, `fixed` o `sticky`.

❌ **No funciona:**
.elemento {
z-index: 100; /* ❌ No funciona (position: static) */
}

text

✅ **Funciona:**
.elemento {
position: relative;
z-index: 100; /* ✅ Funciona */
}

text

---

📌 **Razón 2: El elemento está en un contexto de apilamiento diferente**

Si los elementos están en diferentes contextos, el z-index se compara a nivel del contexto.

❌ **No funciona:**
.padre1 { z-index: 5; }
.padre2 { z-index: 10; }
.hijo-de-padre1 { z-index: 999; } /* No importa, está dentro del padre1 */

text

✅ **Solución:** Ajusta el z-index del padre.

---

📌 **Razón 3: El elemento está oculto por un fondo o un elemento que no debería**

Un elemento con fondo puede tapar a otro si está por encima.

✅ **Solución:** Verifica el orden y los z-index de todos los elementos.

---

📌 **Razón 4: Overflow: hidden en el padre**

`overflow: hidden` en el padre puede recortar elementos hijos con z-index.

✅ **Solución:** Evita `overflow: hidden` si necesitas que los hijos se superpongan.

---

🔵 **Ejemplo: z-index que no funciona**

------------------------------------------------------------
HTML
------------------------------------------------------------
<div class="contenedor">
    <div class="elemento-bajo">Elemento bajo</div>
    <div class="elemento-alto">Elemento alto</div>
</div>
------------------------------------------------------------

CSS:
------------------------------------------------------------
.contenedor {
    position: relative;
    z-index: 1;   /* Contexto de apilamiento */
}

.elemento-bajo {
    position: absolute;
    z-index: 10;   /* Dentro del contexto del contenedor */
    background: #3498DB;
}

.elemento-alto {
    position: absolute;
    z-index: 20;   /* Dentro del contexto del contenedor */
    background: #DC3545;
}
------------------------------------------------------------

**Problema:** `elemento-alto` tiene z-index 20, que es mayor que 10, pero si el contenedor tiene un z-index bajo, todo el contexto puede estar por debajo de otro contexto.

---

📌 **Solución general para z-index que no funciona**

1. **Asegura que el elemento tenga `position` definido**.
2. **Verifica que ambos elementos estén en el mismo contexto de apilamiento**.
3. **Asegura que el z-index del contexto sea suficiente**.
4. **Usa z-index altos (1000+) para elementos que deben estar siempre encima** (como modales).

===============================================================================
5. Z-INDEX EN EL DISEÑO MODERNO
===============================================================================

`z-index` se usa en muchos escenarios modernos.

---

📌 **1. Menús desplegables y submenús**

------------------------------------------------------------
CSS
------------------------------------------------------------
.menu {
    position: relative;
    z-index: 10;
}

.submenu {
    position: absolute;
    z-index: 20;   /* Encima del menú */
}
------------------------------------------------------------

---

📌 **2. Modales y popups**

------------------------------------------------------------
CSS
------------------------------------------------------------
.modal {
    position: fixed;
    z-index: 1000;   /* Encima de todo */
}
------------------------------------------------------------

---

📌 **3. Tooltips (información emergente)**

------------------------------------------------------------
CSS
------------------------------------------------------------
.tooltip {
    position: absolute;
    z-index: 100;
}
------------------------------------------------------------

---

📌 **4. Barras de navegación fijas**

------------------------------------------------------------
CSS
------------------------------------------------------------
.nav {
    position: fixed;
    z-index: 100;
}
------------------------------------------------------------

---

📌 **5. Superposiciones (overlays)**

------------------------------------------------------------
CSS
------------------------------------------------------------
.overlay {
    position: fixed;
    z-index: 999;
    background: rgba(0,0,0,0.5);
}
------------------------------------------------------------

---

🔵 **Ejemplo: sistema de capas común**

------------------------------------------------------------
CSS
------------------------------------------------------------
/* Capa base: contenido normal */
.contenido {
    position: relative;
    z-index: 1;
}

/* Capa media: menú desplegable */
.menu-desplegable {
    position: absolute;
    z-index: 100;
}

/* Capa alta: notificaciones, tooltips */
.notificacion {
    position: fixed;
    z-index: 1000;
}

/* Capa superior: modales, overlays */
.modal {
    position: fixed;
    z-index: 9999;
}
------------------------------------------------------------

===============================================================================
6. BUENAS PRÁCTICAS CON Z-INDEX
===============================================================================

---

📌 **1. Usa valores consistentes**

Define un sistema de z-index para tu proyecto.
:root {
--z-index-base: 1;
--z-index-menu: 100;
--z-index-modal: 1000;
--z-index-overlay: 999;
--z-index-tooltip: 500;
--z-index-notification: 5000;
}

text

---

📌 **2. No uses valores extremos innecesariamente**

No uses `z-index: 99999` si no es necesario.
Mantén los valores en rangos razonables (1-1000).

---

📌 **3. Agrupa z-index por capas lógicas**

- 0-9: contenido normal.
- 10-99: elementos contextuales (menús, dropdowns).
- 100-999: superposiciones (overlays).
- 1000+: modales, popups.

---

📌 **4. Documenta el sistema de z-index**

Añade comentarios en tu CSS explicando el sistema de capas.

---

📌 **5. No dependas de z-index para estructuras complejas**

Para layouts complejos, usa Flexbox o Grid y minimiza el uso de `z-index`.

===============================================================================
LA ANALOGÍA DEL PROFESOR BÚHO: Z-INDEX COMO UN EDIFICIO DE PISOS
===============================================================================

Imagina que tu página es un edificio.

- Cada elemento es un piso del edificio.
- **z-index** es el número del piso.

- Los pisos bajos (z-index: 1, 2, 3) están más cerca del suelo.
- Los pisos altos (z-index: 10, 20, 30) están más cerca del cielo.
- Un piso con número más alto siempre está por encima de uno más bajo.

Pero hay un detalle: cada **contexto de apilamiento** es como un edificio diferente.
Un piso del edificio A puede tener z-index 100, pero si el edificio A es más bajo que el edificio B, el z-index del piso no importa.

===============================================================================
ERRORES COMUNES CON Z-INDEX
===============================================================================

❌ **Error 1: Usar z-index sin position**

   `z-index` solo funciona en elementos con `position` diferente de `static`.
   **Solución:** añade `position: relative`, `absolute`, `fixed` o `sticky`.

---

❌ **Error 2: No considerar el contexto de apilamiento**

   Los elementos en diferentes contextos no se comparan directamente.
   **Solución:** asegura que estén en el mismo contexto o ajusta el contexto padre.

---

❌ **Error 3: Usar z-index: 99999 para todo**

   Los valores extremos dificultan la gestión del z-index.
   **Solución:** usa un sistema de capas coherente.

---

❌ **Error 4: No documentar el sistema de z-index**

   Si varios desarrolladores trabajan en el proyecto, el z-index puede volverse caótico.
   **Solución:** documenta el sistema de capas.

---

❌ **Error 5: Asumir que z-index funciona en elementos con overflow: hidden**

   `overflow: hidden` puede recortar elementos hijos que quieran superponerse.
   **Solución:** tenlo en cuenta al diseñar.

GLOSARIO DEL CAPÍTULO
===============================================================================

- **Z-index**: Propiedad CSS que controla la profundidad de un elemento.
- **Eje Z**: La dimensión de profundidad (detrás/delante).
- **Contexto de apilamiento**: Un grupo de elementos que se apilan juntos.
- **Posicionado**: Elemento con `position` diferente de `static`.

===============================================================================
PREGUNTAS FRECUENTES (FAQ)
===============================================================================

❓ **¿Por qué z-index no funciona en mi elemento?**

   Porque el elemento no tiene `position` definido.
   Añade `position: relative;` (u otro valor) y funcionará.

---

❓ **¿Qué es el contexto de apilamiento?**

   Es un "grupo" de elementos que se apilan juntos.
   Los elementos en diferentes contextos no se comparan directamente.

---

❓ **¿Qué valores puede tener z-index?**

   Números enteros (positivos, negativos o cero) y `auto`.

---

❓ **¿Cómo puedo saber si un elemento tiene un contexto de apilamiento?**

   Si tiene `position` con `z-index` (diferente de auto), `opacity` < 1, `transform`, etc.

---

❓ **¿Es malo usar z-index: 999999?**

   No es "malo", pero es una mala práctica.
   Es mejor usar un sistema de capas coherente.
