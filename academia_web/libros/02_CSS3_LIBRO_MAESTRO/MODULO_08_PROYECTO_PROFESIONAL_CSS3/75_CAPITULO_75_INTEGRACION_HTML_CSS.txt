===============================================================================
BUENAS PRÁCTICAS DE INTEGRACIÓN
===============================================================================

  • HTML semántico → CSS más limpio.
  • Clases con significado, no con estilo.
  • Componentes reutilizables.
  • Separación de responsabilidades.
  • Mantenibilidad a futuro.

===============================================================================
HTML SEMÁNTICO
===============================================================================

  <header>   → Cabecera de la página o sección.
  <nav>      → Navegación principal.
  <main>     → Contenido principal (único por página).
  <article>  → Contenido independiente (blog, noticia).
  <section>  → Agrupación temática.
  <aside>    → Contenido complementario (sidebar).
  <footer>   → Pie de página o sección.

  No uses <div> para todo. La semántica mejora:
    • SEO
    • Accesibilidad (lectores de pantalla)
    • Mantenibilidad del CSS

===============================================================================
NOMBRES DE CLASES: METODOLOGÍA BEM
===============================================================================

  BEM = Bloque__Elemento--Modificador

  .card               → Bloque
  .card__title        → Elemento del bloque
  .card__button       → Elemento del bloque
  .card__button--large → Modificador

  Ventajas:
    • Sin conflictos de nombres.
    • Clara jerarquía visual.
    • Fácil de escalar.

===============================================================================
CSS ORGANIZADO
===============================================================================

  /* ============================================
     1. RESET / NORMALIZE
     ============================================ */
  * { margin: 0; padding: 0; box-sizing: border-box; }

  /* ============================================
     2. VARIABLES (custom properties)
     ============================================ */
  :root {
    --primary: #2563eb;
    --font-base: 1rem;
  }

  /* ============================================
     3. TIPOGRAFÍA BASE
     ============================================ */
  body { font-family: ...; font-size: var(--font-base); }

  /* ============================================
     4. LAYOUT (grid, flexbox, contenedores)
     ============================================ */
  .container { ... }
  .grid { ... }

  /* ============================================
     5. COMPONENTES (navbar, cards, botones)
     ============================================ */
  .btn { ... }
  .card { ... }

  /* ============================================
     6. ESTADOS Y UTILIDADES
     ============================================ */
  .is-active { ... }
  .hidden { display: none; }

  /* ============================================
     7. MEDIA QUERIES (mobile-first)
     ============================================ */
  @media (min-width: 768px) { ... }

ARCHIVOS RECOMENDADOS
===============================================================================

  Para proyectos pequeños a medianos:

  /proyecto/
  ├── index.html
  ├── css/
  │   ├── style.css        → archivo principal
  │   └── normalize.css    → reset opcional
  └── assets/
      └── img/

  Para proyectos grandes:

  /proyecto/
  ├── index.html
  ├── css/
  │   ├── base/
  │   │   ├── reset.css
  │   │   └── variables.css
  │   ├── components/
  │   │   ├── navbar.css
  │   │   ├── card.css
  │   │   └── button.css
  │   └── pages/
  │       └── home.css
  └── assets/
      ├── img/
      └── fonts/

CÓDIGO PRÁCTICO: COMPONENTE REUTILIZABLE
===============================================================================

  <style>
    /* Componente: .btn */
    .btn {
      display: inline-block;
      padding: 12px 28px;
      border-radius: 8px;
      font-size: 1rem;
      font-weight: 600;
      text-decoration: none;
      border: none;
      cursor: pointer;

      background: #2563eb;
      color: white;
    }

    .btn--primary:hover { background: #1d4ed8; }

    .btn--secondary {
      background: #e2e8f0;
      color: #334155;
    }

    .btn--secondary:hover { background: #cbd5e1; }

    .btn--danger {
      background: #ef4444;
      color: white;
    }

    .btn--danger:hover { background: #dc2626; }

    .btn--small {
      padding: 8px 16px;
      font-size: 0.85rem;
    }

    .btn--large {
      padding: 16px 40px;
      font-size: 1.125rem;
    }
  </style>

  <h2>Componente Botón reutilizable</h2>

===============================================================================
RESUMEN DE BUENAS PRÁCTICAS
===============================================================================

  ✓ HTML semántico (header, nav, main, section, article).
  ✓ Nombres de clases con BEM o similar.
  ✓ CSS organizado por secciones con comentarios.
  ✓ Variables CSS para valores repetidos.
  ✓ Componentes reutilizables (btn, card, form).
  ✓ Archivos separados para proyectos grandes.
  ✓ Mobile-first en media queries.
  ✓ Sin !important. Sin IDs en CSS.
