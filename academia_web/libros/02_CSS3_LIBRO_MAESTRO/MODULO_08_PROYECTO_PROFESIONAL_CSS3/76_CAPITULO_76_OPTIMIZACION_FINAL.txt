===============================================================================
¿POR QUÉ OPTIMIZAR?
===============================================================================

  • 53% de usuarios abandonan si tarda más de 3s en cargar.
  • 1 segundo de retraso = 7% menos conversiones.
  • Google usa velocidad como factor de ranking.
  • Sitios lentos consumen más batería y datos móviles.

===============================================================================
MÉTRICAS CLAVE (WEB VITALS)
===============================================================================

  LCP (Largest Contentful Paint):
    → El elemento más grande pintado.
    → Bueno: < 2.5s

  FID (First Input Delay):
    → Tiempo hasta que la página responde al primer clic.
    → Bueno: < 100ms

  CLS (Cumulative Layout Shift):
    → Estabilidad visual (elementos que se mueven).
    → Bueno: < 0.1

===============================================================================
OPTIMIZACIÓN DE CSS
===============================================================================

  1. Minificar

     Antes:
       .card { padding: 20px; border-radius: 8px; }
     Después:
       .card{padding:20px;border-radius:8px}

     Herramientas: cssnano, clean-css.

  2. Purgar CSS no usado

     Antes: 120KB de Bootstrap (95% no usado).
     Después: 8KB con PurgeCSS.

     Herramientas: PurgeCSS, UnCSS.

  3. CSS crítico inline

     <head>
       <style>/* CSS para above the fold */</style>
       <link rel="stylesheet" href="style.css" media="print"
             onload="this.media='all'">
     </head>

     El CSS completo se carga sin bloquear el renderizado.

  4. Compresión en servidor

     Gzip reduce ~70% del tamaño.
     Brotli reduce ~20% más que Gzip.

===============================================================================
OPTIMIZACIÓN DE IMÁGENES
===============================================================================

  • Formato moderno: AVIF > WebP > JPEG.
  • Compresión: 70–85% calidad suele ser suficiente.
  • Dimensiones correctas: no sirvas 2000px si el contenedor es 400px.
  • Lazy loading: loading="lazy" en imágenes debajo del fold.
  • srcset: el navegador elige el tamaño exacto.

===============================================================================
HERRAMIENTAS
===============================================================================

  Lighthouse (Chrome DevTools):
    → Auditoría completa de rendimiento, accesibilidad, SEO.

  PageSpeed Insights (pagespeed.web.dev):
    → Versión online con datos de campo (usuarios reales).

  WebPageTest (webpagetest.org):
    → Pruebas detalladas desde múltiples ubicaciones.

  GTmetrix (gtmetrix.com):
    → Análisis con recomendaciones accionables.

CÓDIGO PRÁCTICO
===============================================================================

  <!DOCTYPE html>
  <html lang="es">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Página optimizada</title>

    <!-- CSS crítico inline -->
    <style>
      * { margin: 0; padding: 0; box-sizing: border-box; }
      body { font-family: Arial, sans-serif; background: #f8f9fa; color: #333; }
      header { background: #1e293b; color: white; padding: 40px 20px; text-align: center; }
      header h1 { font-size: clamp(1.5rem, 4vw, 2.5rem); margin-bottom: 8px; }
      header p { opacity: 0.9; }
      .container { max-width: 1000px; margin: 0 auto; padding: 20px; }
      .card { background: white; border-radius: 8px; padding: 24px; margin-bottom: 16px; box-shadow: 0 2px 8px rgba(0,0,0,0.06); }
      .card h2 { font-size: 1.25rem; margin-bottom: 8px; }
      .card p { color: #666; line-height: 1.6; }
    </style>

    <!-- CSS completo (carga no bloqueante) -->
    <link rel="stylesheet" href="style.css" media="print" onload="this.media='all'">

    <!-- Imagen hero precargada -->
    <link rel="preload" as="image" href="hero.webp" imagesrcset="hero-400.webp 400w, hero-800.webp 800w">

    <!-- Fuentes -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  </head>
  <body>
    <header>
      <h1>Página optimizada</h1>
      <p>Carga rápida, buen rendimiento, usuarios felices.</p>
    </header>

    <div class="container">
      <div class="card">
        <h2>CSS crítico inline</h2>
        <p>Lo que ves al cargar ya está aquí. El resto del CSS
        se carga sin bloquear.</p>
      </div>
      <div class="card">
        <h2>Imágenes con lazy loading</h2>
        <p>Las imágenes de abajo se cargan solo cuando estás por
        verlas. Ahorra datos y velocidad inicial.</p>
      </div>
      <div class="card">
        <h2>Fuentes optimizadas</h2>
        <p>Usamos preconnect para reducir el tiempo de conexión
        con Google Fonts.</p>
      </div>
    </div>

    <!-- Imágenes con lazy loading -->
    <section style="max-width:1000px;margin:0 auto;padding:0 20px 40px;display:grid;grid-template-columns:1fr 1fr;gap:16px;">
      <img src="https://placehold.co/600x400/e2e8f0/64748b?text=Lazy+1" loading="lazy" alt="Imagen 1" style="width:100%;border-radius:8px;">
      <img src="https://placehold.co/600x400/e2e8f0/64748b?text=Lazy+2" loading="lazy" alt="Imagen 2" style="width:100%;border-radius:8px;">
    </section>
  </body>
  </html>

GLOSARIO
===============================================================================

CLS:          Cumulative Layout Shift (estabilidad).
FID:          First Input Delay (interactividad).
PurgeCSS:     Elimina CSS no usado.
CSS crítico:  CSS mínimo para el contenido visible.
Gzip/Brotli:  Compresión del servidor.
Lazy loading: Carga diferida de recursos.
