"Las filas son la dimensión vertical del grid. Controlar su altura
es clave para layouts proporcionados y contenido bien presentado."
                         — Profesor Búho AI

===============================================================================
BIENVENIDA
===============================================================================

Si las columnas son el esqueleto horizontal, las filas son el
vertical. Juntas forman la cuadrícula completa.

En este capítulo verás cómo definir alturas de fila, cómo usar
grid-template-rows, cómo funcionan las filas implícitas y cómo
controlar la alineación vertical del contenido.

===============================================================================
OBJETIVOS
===============================================================================

✓ Definir filas con grid-template-rows.
✓ Usar diferentes unidades para altura de filas.
✓ Controlar filas implícitas con grid-auto-rows.
✓ Usar minmax() para alturas flexibles.
✓ Alinear contenido verticalmente en las filas.
✓ Crear layouts de altura completa (100vh).

===============================================================================
TEORÍA
===============================================================================

grid-template-rows funciona igual que grid-template-columns pero
para el eje vertical.

grid-template-rows: 100px 200px 100px;
→ Tres filas de 100px, 200px y 100px.

Las mismas unidades funcionan:
- px, em, rem → tamaño fijo.
- % → porcentaje del contenedor.
- fr → fracción del espacio vertical disponible.
- min-content → altura mínima para el contenido.
- max-content → altura completa del contenido.
- auto → el navegador decide.
- minmax(mín, máx) → rango flexible.

FILAS IMPLÍCITAS:

Si hay más items que filas definidas, el navegador crea filas
implícitas. Su tamaño se controla con grid-auto-rows.

grid-auto-rows: 120px;
→ Toda fila nueva mide 120px de alto.

LAYOUT DE PANTALLA COMPLETA:

html { height: 100%; }
body { height: 100%; }
.grid { height: 100vh; grid-template-rows: auto 1fr auto; }
→ Header automático, contenido flexible, footer automático.

===============================================================================
ANALOGÍA
===============================================================================

Las filas son los pisos de un edificio.

- grid-template-rows → defines la altura de cada piso.
- 1fr → piso que se adapta a la altura disponible.
- auto → piso que mide solo lo que necesita.
- minmax(100px, 1fr) → piso de mínimo 100px que se estira.
- grid-auto-rows → los pisos extra (implícitos) tienen esta altura.

El layout clásico "header-content-footer" es un edificio de 3
pisos: recepción (auto), oficinas (1fr), estacionamiento (auto).

===============================================================================
SINTAXIS
===============================================================================

/* Filas con diferentes alturas */
.grid {
    grid-template-rows: 80px 1fr 60px;
}

/* Filas con minmax */
.grid {
    grid-template-rows: minmax(100px, auto) 1fr;
}

/* Filas explícitas + implícitas */
.grid {
    grid-template-rows: 100px 100px;
    grid-auto-rows: 80px;
}

/* Layout de pantalla completa */
.grid {
    height: 100vh;
    grid-template-rows: auto 1fr auto;
}

/* Filas con span */
.item {
    grid-row: 1 / 3;
}

/* Nombrar líneas de fila */
.grid {
    grid-template-rows: [header] auto [content] 1fr [footer] auto [fin];
}

=======================================================================
HEADER-CONTENT-FOOTER CON GRID
=======================================================================

El layout más usado en la web: header arriba, contenido al medio,
footer abajo.

body {
    display: grid;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
    margin: 0;
}

- header: auto (solo lo que necesita).
- main: 1fr (ocupa todo el espacio vertical disponible).
- footer: auto (solo lo que necesita).

Esto hace que el footer siempre esté al fondo incluso si el
contenido es poco. Sin Grid (o Flexbox), esto requería trucos.

=======================================================================
ERRORES COMUNES
=======================================================================

1. grid-template-rows sin efecto porque el contenedor no tiene altura.
   → Dale height: 100vh o min-height al contenedor.

2. auto en filas se confunde con 1fr.
   → auto = altura del contenido. 1fr = ocupa el espacio restante.

3. Filas implícitas muy pequeñas (por defecto auto).
   → Define grid-auto-rows con el tamaño deseado.

4. Usar % en filas sin que el padre tenga altura definida.
   → El % necesita una referencia. Sin ella, no funciona.

=======================================================================
GLOSARIO
=======================================================================

grid-template-rows: Define la altura de las filas.
grid-auto-rows: Altura de filas implícitas.
fr: Fracción del espacio vertical disponible.
min-content: Altura mínima para mostrar el contenido.
max-content: Altura completa sin forzar saltos.
auto: Altura automática según el contenido.
100vh: Altura completa del viewport.
