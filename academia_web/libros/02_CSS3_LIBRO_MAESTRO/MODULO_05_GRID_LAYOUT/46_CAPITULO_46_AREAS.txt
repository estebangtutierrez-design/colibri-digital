"grid-template-areas es la forma más visual de maquetar con Grid.
Dibujas el layout con letras en el CSS y los elementos se colocan

                         — Profesor Búho AI

===============================================================================
BIENVENIDA
===============================================================================

Hasta ahora has posicionado elementos con grid-column y grid-row.
Eso funciona, pero puede volverse tedioso en layouts complejos.

grid-template-areas te permite definir zonas nombradas en tu grid
usando una representación visual en ASCII. Es como dibujar el
layout en tu código.

.header  .header  .header
.sidebar .main    .main
.sidebar .main    .main
.footer  .footer  .footer

Cada zona (header, sidebar, main, footer) es un área que puedes
asignar a un elemento con grid-area.

===============================================================================
OBJETIVOS
===============================================================================

✓ Definir áreas con grid-template-areas.
✓ Asignar elementos a áreas con grid-area.
✓ Usar puntos (.) para celdas vacías.
✓ Reordenar elementos visualmente.
✓ Crear layouts complejos de forma legible.

===============================================================================
TEORÍA
===============================================================================

grid-template-areas recibe una cadena de texto por cada fila.
Cada palabra dentro de las comillas representa una celda.

Los nombres de área se repiten para ocupar varias celdas.
Los puntos (.) representan celdas vacías.

REGLAS:
1. Cada área debe ser rectangular (no formas L).
2. Todas las filas deben tener el mismo número de columnas.
3. Los espacios separan las áreas en la misma fila.
4. Las filas se separan con nuevas líneas o espacios.

SINTAXIS:

.grid {
    grid-template-columns: 200px 1fr;
    grid-template-rows: auto 1fr auto;
    grid-template-areas:
        "header  header"
        "sidebar main"
        "footer  footer";
}

.header-nombre  { grid-area: header; }
.sidebar-nombre { grid-area: sidebar; }
.main-nombre    { grid-area: main; }
.footer-nombre  { grid-area: footer; }

El orden en HTML no importa. Cada elemento se coloca según
su grid-area.

===============================================================================
ANALOGÍA
===============================================================================

grid-template-areas es como un plano de construcción:

    ┌─────────┬─────────┐
    │ HEADER  │ HEADER  │
    ├─────────┼─────────┤
    │ SIDEBAR │  MAIN   │
    │         │         │
    ├─────────┴─────────┤
    │      FOOTER       │
    └───────────────────┘

Dibujas el plano en CSS y cada pieza (header, sidebar, main,
footer) sabe exactamente dónde colocarse. No importa en qué
orden estén en el HTML.

===============================================================================
SINTAXIS COMPLETA
===============================================================================

/* Layout clásico */
.layout {
    display: grid;
    grid-template-columns: 250px 1fr;
    grid-template-rows: auto 1fr auto;
    grid-template-areas:
        "header header"
        "sidebar main"
        "footer footer";
    min-height: 100vh;
    gap: 10px;
}

/* Layout con celda vacía */
.layout-2 {
    grid-template-areas:
        "header header header"
        "main   .      sidebar"
        "footer footer footer";
}
/* El punto (.) es una celda vacía */

/* Layout con 3 columnas */
.layout-3 {
    grid-template-areas:
        "header header  header"
        "left   center  right"
        "footer footer  footer";
}

=======================================================================
VENTAJAS DE GRID AREAS
=======================================================================

1. LEGIBILIDAD: ves el layout en el CSS como un dibujo.
2. REORDENAMIENTO: cambias la posición sin tocar el HTML.
3. RESPONSIVO: en media queries cambias solo las áreas.
4. MANTENIBLE: un vistazo y entiendes la estructura.

Ejemplo responsivo:

.layout {
    grid-template-areas:
        "header"
        "sidebar"
        "main"
        "footer";
}

@media (min-width: 768px) {
    .layout {
        grid-template-areas:
            "header header"
            "sidebar main"
            "footer footer";
    }
}

===============================================================================
ERRORES COMUNES
===============================================================================

1. Áreas no rectangulares (forma de L).
   → grid-template-areas solo acepta rectángulos.

2. Número desigual de columnas entre filas.
   → Todas las filas deben tener la misma cantidad de áreas.

3. Olvidar grid-area en los elementos.
   → Si el elemento no tiene grid-area, se coloca automáticamente.

4. Nombres con espacios.
   → Los nombres deben ser una sola palabra. Usa guiones.

EJERCICIOS
===============================================================================

   main (centro), aside (derecha), footer (abajo).

2. Crea una galería de 5 imágenes donde la primera ocupe
   2 columnas y 2 filas, y las otras 4 ocupen 1 celda cada una.

3. Crea un layout responsivo que en móvil sea una columna
   y en desktop pase a: header, sidebar + main, footer.

4. Diseña una página de dashboard con áreas: header, stats,
   chart, table, sidebar, footer.

===============================================================================
GLOSARIO
===============================================================================

grid-template-areas: Define áreas nombradas en el grid.
grid-area: Asigna un elemento a un área nombrada.
Punto (.): Celda vacía en grid-template-areas.
Área rectangular: Una zona que ocupa un rectángulo de celdas.
Reordenamiento visual: Cambiar la posición sin alterar el HTML.
