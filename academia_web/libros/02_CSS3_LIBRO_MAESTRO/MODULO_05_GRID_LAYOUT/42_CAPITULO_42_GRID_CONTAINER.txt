"El contenedor grid es el lienzo. Sin él, no hay cuadrícula.
Sus propiedades definen la estructura: columnas, filas, alineación
y comportamiento del espacio sobrante."
                         — Profesor Búho AI

===============================================================================
BIENVENIDA
===============================================================================

En el capítulo anterior viste los fundamentos de Grid. Ahora
profundizamos en las propiedades del **contenedor grid** — el
elemento padre que tiene `display: grid`.

Hay propiedades que ya conoces (grid-template-columns,
grid-template-rows, gap) y otras nuevas que controlan cómo se
comporta el grid cuando el contenido no llena todo el espacio:
justify-content, align-content y place-content.

===============================================================================
OBJETIVOS
===============================================================================

✓ Usar justify-content para alinear el grid horizontalmente.
✓ Usar align-content para alinear el grid verticalmente.
✓ Controlar el tamaño de las filas implícitas.
✓ Diferenciar auto, 1fr, min-content, max-content, minmax().
✓ Usar grid-auto-rows y grid-auto-columns.

===============================================================================
TEORÍA
===============================================================================

PROPIEDADES DEL CONTENEDOR GRID:

1. grid-template-columns → define las columnas.
2. grid-template-rows → define las filas.
3. gap → espacio entre celdas.
4. justify-content → alinea el grid en el eje horizontal (inline).
5. align-content → alinea el grid en el eje vertical (block).
6. grid-auto-rows → tamaño por defecto de filas implícitas.
7. grid-auto-columns → tamaño por defecto de columnas implícitas.

JUSTIFY-CONTENT y ALIGN-CONTENT solo tienen efecto cuando el
contenedor es más grande que el grid (sobra espacio).

VALORES para justify-content y align-content:
- start (por defecto) → al inicio.
- end → al final.
- center → centrado.
- stretch → estirado para ocupar todo el espacio.
- space-between → espacio igual entre columnas/filas.
- space-around → espacio igual alrededor.
- space-evenly → espacio totalmente igual.

Grid EXPLÍCITO vs IMPLÍCITO:
- Explícito: lo que defines con grid-template-*.
- Implícito: lo que el navegador crea cuando hay más elementos
  de los que caben en el grid explícito.

===============================================================================
ANALOGÍA
===============================================================================

Imagina un tablero de ajedrez sobre una mesa grande.

- grid-template-columns → defines el tamaño de cada casilla.
- gap → el espacio entre casillas.
- justify-content → mueves el tablero a la izquierda/derecha.
- align-content → mueves el tablero arriba/abajo.
- grid-auto-rows → si pones más piezas de las que caben, el
  tablero se estira con filas de ese tamaño.

===============================================================================
SINTAXIS
===============================================================================

.contenedor {
    display: grid;
    grid-template-columns: 100px 100px 100px;
    grid-template-rows: 100px 100px;
    gap: 10px;

    /* Alineación del grid dentro del contenedor */
    justify-content: center;  /* horizontal */
    align-content: center;    /* vertical */

    /* Filas implícitas */
    grid-auto-rows: 80px;
}

===============================================================================
MINMAX()
===============================================================================

minmax(mínimo, máximo) define un rango de tamaño:

grid-template-columns: minmax(100px, 1fr);
→ Cada columna mide al menos 100px y como máximo 1fr.

grid-template-columns: repeat(3, minmax(150px, 1fr));
→ 3 columnas que miden al menos 150px y se estiran hasta
  ocupar el espacio disponible.

Esto es muy útil para grids responsivos sin media queries.

===============================================================================
AUTO-FILL Y AUTO-FIT
===============================================================================

repeat(auto-fill, minmax(150px, 1fr))
→ Crea tantas columnas de 150px como quepan, y si sobra
  espacio, las columnas se estiran.

repeat(auto-fit, minmax(150px, 1fr))
→ Similar, pero las columnas vacías colapsan a 0.

Diferencia práctica:
- auto-fill: mantiene las columnas aunque estén vacías.
- auto-fit: colapsa las columnas vacías.

===============================================================================
ERRORES COMUNES
===============================================================================

1. justify-content no hace nada si el grid ocupa todo el ancho.
   → Usa columnas con tamaño fijo para ver el efecto.

2. Olvidar que grid-auto-rows solo afecta a filas implícitas.
   → Las filas explícitas se controlan con grid-template-rows.

3. Usar align-content en un grid que no tiene altura definida.
   → El contenedor debe tener height para que funcione.

===============================================================================
GLOSARIO
===============================================================================

grid-template-columns: Define las columnas del grid.
grid-template-rows: Define las filas del grid.
justify-content: Alinea el grid en el eje horizontal.
align-content: Alinea el grid en el eje vertical.
grid-auto-rows: Tamaño de filas implícitas.
minmax(): Define un rango mínimo y máximo.
auto-fill: Crea tantas columnas como quepan.
auto-fit: Similar pero colapsa las vacías.
gap: Espacio entre celdas.
