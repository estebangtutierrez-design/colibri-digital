--- 1. INTRODUCCIÓN A LAS EXCEPCIONES ---

Una excepción es una interrupción del flujo normal de un programa debido a
una condición anómala. En C#, las excepciones son objetos que derivan de
System.Exception. Cuando ocurre un error, se "lanza" (throw) una excepción
y el runtime busca un bloque catch que pueda manejarla.

Ejemplo simple de división por cero:

```csharp
using System;

class Program
{
    static void Main()
    {
        int a = 10;
        int b = 0;
        // Esto lanzará DivideByZeroException
        int resultado = a / b;  // Excepción no controlada
    }
}
```

Si no se maneja la excepción, el programa termina abruptamente.

--- 2. ESTRUCTURA TRY/CATCH/FINALLY ---

La estructura básica para manejar excepciones consta de tres bloques:

- try: Contiene el código que puede lanzar una excepción.
- catch: Captura y maneja la excepción.
- finally: Se ejecuta siempre, haya o no excepción (ideal para limpieza).

Sintaxis:

```csharp
try
{
    // Código propenso a errores
}
catch (TipoDeExcepcion ex)
{
    // Manejo de la excepción
}
finally
{
    // Código de limpieza (siempre se ejecuta)
}
```

Ejemplo completo:

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("=== MANEJO DE EXCEPCIONES ===");
        
        try
        {
            Console.Write("Ingrese un número: ");
            int numerador = int.Parse(Console.ReadLine());
            
            Console.Write("Ingrese otro número: ");
            int denominador = int.Parse(Console.ReadLine());
            
            int resultado = numerador / denominador;
            Console.WriteLine($"Resultado: {resultado}");
        }
        catch (DivideByZeroException)
        {
            Console.WriteLine("Error: No se puede dividir por cero.");
        }
        catch (FormatException)
        {
            Console.WriteLine("Error: Formato de número inválido.");
        }
        finally
        {
            Console.WriteLine("Bloque finally ejecutado. Limpieza completada.");
        }
        
        Console.WriteLine("Programa continúa después del try/catch.");
    }
}
```

--- 3. MÚLTIPLES BLOQUES CATCH ---

Podemos tener varios bloques catch para manejar diferentes tipos de
excepción de manera específica. El orden es importante: primero deben
ir las excepciones más específicas y luego las más generales.

```csharp
using System;

class Program
{
    static void Main()
    {
        string[] datos = {"123", "abc", "9999999999999", null, "45"};
        
        foreach (string dato in datos)
        {
            try
            {
                Console.WriteLine($"Procesando: '{dato}'");
                if (dato == null)
                    throw new ArgumentNullException(nameof(dato), "El dato no puede ser nulo");
                
                int numero = int.Parse(dato);
                int resultado = 100 / numero;
                Console.WriteLine($"  OK -> {numero}, 100/{numero} = {resultado}");
            }
            catch (ArgumentNullException ex) when (ex.ParamName == "dato")
            {
                Console.WriteLine($"  Error: Argumento nulo - {ex.Message}");
            }
            catch (FormatException)
            {
                Console.WriteLine("  Error: Formato inválido.");
            }
            catch (OverflowException)
            {
                Console.WriteLine("  Error: Número demasiado grande.");
            }
            catch (DivideByZeroException)
            {
                Console.WriteLine("  Error: División por cero.");
            }
            catch (Exception ex)
            {
                // Catch general: debe ir al final
                Console.WriteLine($"  Error inesperado: {ex.GetType().Name} - {ex.Message}");
            }
        }
    }
}
```

--- 4. FILTROS DE EXCEPCIÓN (WHEN) ---

C# 6 introdujo los filtros de excepción con la palabra clave when:

```csharp
using System;
using System.Net.Http;

class Program
{
    static async System.Threading.Tasks.Task Main()
    {
        try
        {
            using var client = new HttpClient();
            var response = await client.GetStringAsync("https://ejemplo.com/no-existe");
        }
        catch (HttpRequestException ex) when (ex.Message.Contains("404"))
        {
            Console.WriteLine("Recurso no encontrado (404).");
        }
        catch (HttpRequestException ex) when (ex.Message.Contains("500"))
        {
            Console.WriteLine("Error interno del servidor (500).");
        }
        catch (HttpRequestException ex)
        {
            Console.WriteLine($"Error de red: {ex.Message}");
        }
    }
}
```

--- 5. SYSTEM.EXCEPTION Y JERARQUÍA ---

System.Exception es la clase base de todas las excepciones en .NET.
Principales clases derivadas:

- SystemException: Para excepciones del runtime.
  - AccessViolationException
  - ArrayTypeMismatchException
  - IndexOutOfRangeException
  - InvalidCastException
  - NullReferenceException
  - OutOfMemoryException
  - StackOverflowException
  - DivideByZeroException
- ArgumentException: Para argumentos inválidos.
  - ArgumentNullException
  - ArgumentOutOfRangeException
- InvalidOperationException
- NotSupportedException
- IOException: Para errores de E/S.
  - FileNotFoundException
  - DirectoryNotFoundException
  - EndOfStreamException
- FormatException
- OverflowException
- ApplicationException: Históricamente para aplicaciones (hoy se recomienda
  heredar directamente de Exception).

```csharp
using System;
using System.Collections;
using System.IO;

class Program
{
    static void Main()
    {
        Console.WriteLine("=== JERARQUÍA DE EXCEPCIONES ===\n");
        
        // NullReferenceException
        try
        {
            string s = null;
            Console.WriteLine(s.Length);
        }
        catch (NullReferenceException ex)
        {
            Console.WriteLine($"NullReferenceException: {ex.Message}");
        }
        
        // IndexOutOfRangeException
        try
        {
            int[] arr = {1, 2, 3};
            Console.WriteLine(arr[10]);
        }
        catch (IndexOutOfRangeException ex)
        {
            Console.WriteLine($"IndexOutOfRangeException: {ex.Message}");
        }
        
        // InvalidCastException
        try
        {
            object obj = 123;
            string str = (string)obj;
        }
        catch (InvalidCastException ex)
        {
            Console.WriteLine($"InvalidCastException: {ex.Message}");
        }
        
        // ArgumentOutOfRangeException
        try
        {
            var list = new ArrayList();
            list.Add("Item");
            Console.WriteLine(list[-1]);
        }
        catch (ArgumentOutOfRangeException ex)
        {
            Console.WriteLine($"ArgumentOutOfRangeException: {ex.Message}");
        }
        
        // FileNotFoundException
        try
        {
            File.ReadAllText("archivo_inexistente.txt");
        }
        catch (FileNotFoundException ex)
        {
            Console.WriteLine($"FileNotFoundException: {ex.Message}");
        }
    }
}
```

--- 6. INNEREXCEPTION ---

InnerException permite anidar excepciones. Es útil cuando una excepción
es consecuencia de otra. Se usa frecuentemente para preservar la excepción
original mientras se lanza una nueva con más contexto.

```csharp
using System;
using System.IO;

class Program
{
    static void Main()
    {
        try
        {
            ProcesarArchivo("datos.txt");
        }
        catch (Exception ex)
        {
            Console.WriteLine("=== EXCEPCIÓN PRINCIPAL ===");
            Console.WriteLine($"Tipo: {ex.GetType().Name}");
            Console.WriteLine($"Mensaje: {ex.Message}");
            Console.WriteLine($"Método: {ex.TargetSite}");
            
            if (ex.InnerException != null)
            {
                Console.WriteLine("\n=== INNER EXCEPTION ===");
                Console.WriteLine($"Tipo: {ex.InnerException.GetType().Name}");
                Console.WriteLine($"Mensaje: {ex.InnerException.Message}");
                Console.WriteLine($"StackTrace interno:\n{ex.InnerException.StackTrace}");
            }
            
            Console.WriteLine($"\nStackTrace completo:\n{ex.StackTrace}");
        }
    }
    
    static void ProcesarArchivo(string ruta)
    {
        try
        {
            string contenido = File.ReadAllText(ruta);
            int numero = int.Parse(contenido);
            Console.WriteLine($"Número leído: {numero}");
        }
        catch (IOException ex)
        {
            // Envolvemos la excepción original con una nueva
            throw new InvalidOperationException(
                "Error al procesar el archivo. Verifique que exista y tenga permisos.",
                ex);
        }
        catch (FormatException ex)
        {
            throw new InvalidOperationException(
                "El archivo no contiene un número válido.",
                ex);
        }
    }
}
```

--- 7. STACKTRACE ---

StackTrace contiene la pila de llamadas que llevaron a la excepción.
Es vital para depuración.

```csharp
using System;
using System.Diagnostics;

class Program
{
    static void Main()
    {
        try
        {
            Nivel1();
        }
        catch (Exception ex)
        {
            Console.WriteLine("=== DEMO STACKTRACE ===");
            Console.WriteLine($"Mensaje: {ex.Message}");
            Console.WriteLine($"\nStackTrace:\n{ex.StackTrace}");
            
            // Podemos analizar el stack frame por frame
            var stackTrace = new StackTrace(ex, true);
            Console.WriteLine($"\nNúmero de frames: {stackTrace.FrameCount}");
            
            for (int i = 0; i < stackTrace.FrameCount; i++)
            {
                var frame = stackTrace.GetFrame(i);
                var metodo = frame.GetMethod();
                Console.WriteLine($"\nFrame {i}:");
                Console.WriteLine($"  Método: {metodo?.DeclaringType?.Name}.{metodo?.Name}");
                Console.WriteLine($"  Archivo: {frame.GetFileName()}");
                Console.WriteLine($"  Línea: {frame.GetFileLineNumber()}");
            }
        }
    }
    
    static void Nivel1()
    {
        Nivel2();
    }
    
    static void Nivel2()
    {
        Nivel3();
    }
    
    static void Nivel3()
    {
        // Esto causará la excepción
        throw new InvalidOperationException("Error simulado en Nivel3");
    }
}
```

--- 8. LANZAR EXCEPCIONES (THROW) ---

Usamos throw para lanzar una excepción explícitamente:

```csharp
using System;

class Program
{
    static void Main()
    {
        try
        {
            ValidarEdad(-5);
        }
        catch (ArgumentException ex)
        {
            Console.WriteLine($"Error de validación: {ex.Message}");
            Console.WriteLine($"Parámetro: {ex.ParamName}");
        }
        
        try
        {
            ProcesarPedido(null);
        }
        catch (ArgumentNullException ex)
        {
            Console.WriteLine($"Error: {ex.Message}");
        }
        
        // Relanzar excepción (throw sin argumentos preserva stack)
        try
        {
            try
            {
                throw new InvalidOperationException("Error interno");
            }
            catch
            {
                Console.WriteLine("Atrapada, pero relanzamos...");
                throw;  // Relanza la misma excepción
            }
        }
        catch (InvalidOperationException ex)
        {
            Console.WriteLine($"Relanzada: {ex.Message}");
        }
    }
    
    static void ValidarEdad(int edad)
    {
        if (edad < 0)
            throw new ArgumentOutOfRangeException(
                nameof(edad), 
                "La edad no puede ser negativa.");
        
        if (edad > 150)
            throw new ArgumentOutOfRangeException(
                nameof(edad),
                "La edad no puede superar los 150 años.");
        
        Console.WriteLine($"Edad válida: {edad}");
    }
    
    static void ProcesarPedido(string pedido)
    {
        if (pedido == null)
            throw new ArgumentNullException(nameof(pedido));
        
        Console.WriteLine($"Procesando pedido: {pedido}");
    }
}
```

--- 9. BUENAS PRÁCTICAS ---

```csharp
using System;
using System.IO;

class BuenasPracticas
{
    // ✅ CORRECTO: Atrapar solo excepciones que podemos manejar
    public string LeerConfiguracion(string ruta)
    {
        try
        {
            return File.ReadAllText(ruta);
        }
        catch (FileNotFoundException)
        {
            // Creamos el archivo con valores por defecto
            string defecto = "{}";
            File.WriteAllText(ruta, defecto);
            return defecto;
        }
    }
    
    // ✅ CORRECTO: Usar finally para limpiar recursos
    public void ProcesarArchivoSeguro(string ruta)
    {
        FileStream fs = null;
        try
        {
            fs = File.OpenRead(ruta);
            // Procesar...
        }
        finally
        {
            if (fs != null)
                fs.Dispose();
        }
    }
    
    // ✅ CORRECTO: Usar using (equivale a try/finally)
    public void ProcesarConUsing(string ruta)
    {
        using (var fs = File.OpenRead(ruta))
        {
            // Procesar...
        }
    }
    
    // ✅ CORRECTO: No ocultar excepciones
    public int Dividir(int a, int b)
    {
        try
        {
            return a / b;
        }
        catch (DivideByZeroException)
        {
            // ❌ MAL: catch vacío
            // throw; // ✅ CORRECTO: relanzar si no podemos manejarla
            return 0;  // ❌ MAL: devolver valor por defecto sin registrar
        }
    }
    
    // ✅ CORRECTO: Preservar InnerException
    public void MetodoEnvoltorio()
    {
        try
        {
            // Operación riesgosa
        }
        catch (IOException ex)
        {
            throw new ApplicationException("Error en operación.", ex);
        }
    }
}
```

--- 10. EJEMPLO COMPLETO: CALCULADORA ROBUSTA ---

```csharp
using System;
using System.Collections.Generic;

class CalculadoraRobusta
{
    static void Main()
    {
        Console.WriteLine("=== CALCULADORA ROBUSTA ===\n");
        
        var historial = new List<string>();
        
        while (true)
        {
            try
            {
                Console.Write("Ingrese operación (num1 op num2) o 'salir': ");
                string entrada = Console.ReadLine();
                
                if (entrada?.ToLower() == "salir")
                    break;
                
                string[] partes = entrada?.Split(' ');
                
                if (partes == null || partes.Length != 3)
                {
                    Console.WriteLine("Formato: num1 +|-|*|/ num2");
                    continue;
                }
                
                if (!double.TryParse(partes[0], out double num1))
                {
                    Console.WriteLine($"'{partes[0]}' no es un número válido.");
                    continue;
                }
                
                string operador = partes[1];
                
                if (!double.TryParse(partes[2], out double num2))
                {
                    Console.WriteLine($"'{partes[2]}' no es un número válido.");
                    continue;
                }
                
                double resultado = Calcular(num1, num2, operador);
                string linea = $"{num1} {operador} {num2} = {resultado}";
                Console.WriteLine($"Resultado: {linea}");
                historial.Add(linea);
            }
            catch (DivideByZeroException)
            {
                Console.WriteLine("ERROR: No se puede dividir por cero.");
            }
            catch (InvalidOperationException ex)
            {
                Console.WriteLine($"ERROR: {ex.Message}");
            }
            catch (FormatException)
            {
                Console.WriteLine("ERROR: Formato de número incorrecto.");
            }
            catch (OverflowException)
            {
                Console.WriteLine("ERROR: Número fuera de rango.");
            }
            catch (Exception ex)
            {
                Console.WriteLine($"ERROR INESPERADO: {ex.GetType().Name}: {ex.Message}");
            }
        }
        
        Console.WriteLine("\n=== HISTORIAL DE OPERACIONES ===");
        foreach (string h in historial)
            Console.WriteLine($"  {h}");
    }
    
    static double Calcular(double a, double b, string op)
    {
        switch (op)
        {
            case "+":
                if (b > double.MaxValue - a)
                    throw new OverflowException("Suma causa desbordamiento.");
                return a + b;
            case "-":
                return a - b;
            case "*":
                return a * b;
            case "/":
                if (b == 0)
                    throw new DivideByZeroException();
                return a / b;
            default:
                throw new InvalidOperationException($"Operador '{op}' no soportado.");
        }
    }
}
```

--- 11. EXCEPCIONES Y ASINCRONÍA ---

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        Console.WriteLine("=== EXCEPCIONES EN ASINCRONÍA ===\n");
        
        try
        {
            await TareaConErrorAsync();
        }
        catch (InvalidOperationException ex)
        {
            Console.WriteLine($"Excepción capturada: {ex.Message}");
            if (ex.InnerException != null)
                Console.WriteLine($"Inner: {ex.InnerException.Message}");
        }
        
        // Múltiples tareas con WhenAll
        try
        {
            Task t1 = TareaOkAsync("T1", 1000);
            Task t2 = TareaErrorAsync("T2", 500);
            Task t3 = TareaOkAsync("T3", 800);
            
            await Task.WhenAll(t1, t2, t3);
        }
        catch (Exception ex)
        {
            // WhenAll solo lanza la primera excepción
            Console.WriteLine($"\nWhenAll capturó: {ex.Message}");
        }
    }
    
    static async Task TareaConErrorAsync()
    {
        await Task.Delay(100);
        throw new InvalidOperationException("Error en tarea asíncrona",
            new ArgumentException("Causa interna"));
    }
    
    static async Task TareaOkAsync(string nombre, int ms)
    {
        await Task.Delay(ms);
        Console.WriteLine($"  Tarea '{nombre}' completada.");
    }
    
    static async Task TareaErrorAsync(string nombre, int ms)
    {
        await Task.Delay(ms);
        throw new InvalidOperationException($"Error en tarea '{nombre}'");
    }
}
```

--- 12. EXCEPCIONES AGREGADAS (AggregateException) ---

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Console.WriteLine("=== AGGREGATE EXCEPTION ===\n");
        
        try
        {
            Task t = Task.Run(() =>
            {
                throw new InvalidOperationException("Error en Task.Run");
            });
            t.Wait();  // Wait() envuelve en AggregateException
        }
        catch (AggregateException ag)
        {
            ag.Handle(ex =>
            {
                Console.WriteLine($"Manejando: {ex.GetType().Name}: {ex.Message}");
                return true;  // Indica que la excepción fue manejada
            });
        }
        
        // TPL con múltiples excepciones
        try
        {
            var tasks = new Task[3];
            tasks[0] = Task.Run(() => throw new AccessViolationException("Error A"));
            tasks[1] = Task.Run(() => throw new InvalidOperationException("Error B"));
            tasks[2] = Task.Run(() => throw new ArgumentException("Error C"));
            
            Task.WaitAll(tasks);
        }
        catch (AggregateException ag)
        {
            Console.WriteLine($"\nAggregateException con {ag.InnerExceptions.Count} excepciones:");
            foreach (var ex in ag.InnerExceptions)
                Console.WriteLine($"  - {ex.GetType().Name}: {ex.Message}");
        }
    }
}
```

--- 13. LOGGING DE EXCEPCIONES ---

```csharp
using System;
using System.IO;

class ExceptionLogger
{
    private readonly string _logPath;
    
    public ExceptionLogger(string logPath)
    {
        _logPath = logPath;
    }
    
    public void Log(Exception ex)
    {
        try
        {
            using (var writer = File.AppendText(_logPath))
            {
                writer.WriteLine($"[{DateTime.Now:yyyy-MM-dd HH:mm:ss}]");
                writer.WriteLine($"  Tipo: {ex.GetType().FullName}");
                writer.WriteLine($"  Mensaje: {ex.Message}");
                writer.WriteLine($"  StackTrace: {ex.StackTrace}");
                
                if (ex.InnerException != null)
                {
                    writer.WriteLine($"  InnerException:");
                    writer.WriteLine($"    Tipo: {ex.InnerException.GetType().FullName}");
                    writer.WriteLine($"    Mensaje: {ex.InnerException.Message}");
                    writer.WriteLine($"    StackTrace: {ex.InnerException.StackTrace}");
                }
                
                if (ex.Data.Count > 0)
                {
                    writer.WriteLine($"  Data:");
                    foreach (var key in ex.Data.Keys)
                        writer.WriteLine($"    {key}: {ex.Data[key]}");
                }
                
                writer.WriteLine(new string('-', 50));
            }
        }
        catch
        {
            // Silencioso: no podemos lanzar otra excepción mientras
            // registramos una excepción previa.
            Console.Error.WriteLine("Fallo al registrar excepción en el log.");
        }
    }
    
    public static void Demo()
    {
        var logger = new ExceptionLogger("errores.log");
        
        try
        {
            // Simulamos una operación compleja
            var data = LeerDatos();
            var parsed = ParsearDatos(data);
            Console.WriteLine($"Datos procesados: {parsed}");
        }
        catch (Exception ex)
        {
            logger.Log(ex);
            Console.WriteLine("Excepción registrada. Revise errores.log");
        }
    }
    
    static string LeerDatos()
    {
        throw new FileNotFoundException("Archivo de datos no encontrado.",
            "datos_2024.csv");
    }
    
    static int ParsearDatos(string data)
    {
        if (string.IsNullOrEmpty(data))
            throw new ArgumentException("Datos vacíos o nulos.");
        return int.Parse(data);
    }
}
```

--- 14. CUANDO NO USAR EXCEPCIONES ---

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("=== CUÁNDO NO USAR EXCEPCIONES ===\n");
        
        // ❌ MAL: Usar excepciones para control de flujo
        for (int i = -5; i <= 5; i++)
        {
            try
            {
                Console.WriteLine($"Raíz de {i} = {CalcularRaizConExcepcion(i)}");
            }
            catch (InvalidOperationException)
            {
                Console.WriteLine($"Raíz de {i} = No es número positivo");
            }
        }
        
        Console.WriteLine();
        
        // ✅ CORRECTO: Validación previa sin excepciones
        for (int i = -5; i <= 5; i++)
        {
            if (i >= 0)
                Console.WriteLine($"Raíz de {i} = {Math.Sqrt(i):F4}");
            else
                Console.WriteLine($"Raíz de {i} = No es número positivo");
        }
    }
    
    static double CalcularRaizConExcepcion(int n)
    {
        if (n < 0)
            throw new InvalidOperationException("Número negativo.");
        return Math.Sqrt(n);
    }
}
```

1. Las excepciones son objetos que derivan de System.Exception y representan
   condiciones anómalas en la ejecución.

2. La estructura try/catch/finally permite manejar errores de forma
   controlada.

3. Podemos tener múltiples bloques catch para manejar diferentes tipos de
   excepción, ordenados de más específico a más general.

4. InnerException permite preservar la cadena de excepciones originales.

5. StackTrace proporciona la pila de llamadas para depuración.

6. Las buenas prácticas incluyen: no usar excepciones para control de flujo,
   no atrapar Exception en catch general si no es necesario, siempre limpiar
   recursos en finally (o usar using), y registrar las excepciones.

Ejercicio 1: Escriba un programa que lea un archivo de texto y muestre su

Ejercicio 2: Cree un método que valide una contraseña (mínimo 8 caracteres,

Ejercicio 3: Implemente un programa que procese una lista de números en un

Ejercicio 5: Escriba un logger de excepciones que guarde en un archivo la
fecha, tipo, mensaje, stack trace y datos adicionales de cada excepción.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Cuál es la diferencia entre throw y throw ex?
R: throw (sin argumentos) relanza la excepción preservando el stack trace
original. throw ex la relanza pero reinicia el stack trace, perdiendo la
información de dónde ocurrió originalmente. Siempre use throw a secas.

P: ¿Es malo capturar Exception general?
R: Generalmente sí. Capture solo las excepciones que puede manejar. Capturar
Exception general puede ocultar bugs y excepciones críticas como
OutOfMemoryException o StackOverflowException.

P: ¿Cuándo debo usar finally?
R: Siempre que necesite liberar recursos (archivos, conexiones, etc.).
El bloque finally se ejecuta incluso si hay return en try/catch.

P: ¿Puedo tener try sin catch?
R: Sí, try/finally es válido. Se usa cuando solo necesita garantizar la
limpieza de recursos pero no manejar la excepción en ese nivel.

P: ¿Qué es una excepción no controlada?
R: Es una excepción que ningún bloque catch maneja. El runtime muestra
un mensaje de error y finaliza el programa (o el hilo).

================================================================================
  DESAFÍO DEL ARQUITECTO
================================================================================

Diseñe e implemente un sistema de procesamiento de transacciones bancarias
que cumpla con los siguientes requisitos:

1. Lea transacciones desde un archivo CSV con formato:
   ID,Fecha,Tipo(Deposito/Retiro),Monto,Cuenta

2. Valide cada transacción antes de procesarla (monto positivo, tipo
   válido, cuenta existente, saldo suficiente para retiros).

3. Use excepciones personalizadas (por ahora, use las built-in) para cada
   tipo de error de validación.

4. Registre todas las excepciones en un archivo de log con timestamp.

5. Al final, muestre un resumen de transacciones exitosas y fallidas.

6. Use InnerException cuando una excepción sea consecuencia de otra.

Requisitos técnicos:
- Use try/catch/finally en todos los niveles.
- Use filtros when donde sea apropiado.
- No use excepciones para control de flujo.
- Use using para recursos IDisposable.
- Implemente un contador de errores por tipo.

Extensión sugerida: Implemente reintentos automáticos para errores
recuperables (como IOException) con un máximo de 3 intentos.
