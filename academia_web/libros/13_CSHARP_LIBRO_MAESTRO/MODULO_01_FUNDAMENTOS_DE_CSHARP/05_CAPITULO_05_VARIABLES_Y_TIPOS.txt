1. DECLARACIÓN Y ASIGNACIÓN DE VARIABLES
===============================================================================
Una variable es un espacio en memoria que almacena un valor que puede cambiar
durante la ejecución del programa.

**1.1 Sintaxis básica**

```csharp
using System;

// Declaración: tipo nombre;
int edad;
double precio;
string nombre;

// Asignación: nombre = valor;
edad = 30;
precio = 19.99;
nombre = "Ana";

// Declaración + asignación en una línea
int cantidad = 100;
double altura = 1.75;
string ciudad = "Madrid";

// Múltiples variables del mismo tipo en una línea
int x = 10, y = 20, z = 30;

// Variable sin inicializar (debe asignarse antes de usar)
int resultado;
resultado = x + y;
Console.WriteLine($"Resultado: {resultado}");
```

**1.2 Reglas para nombres de variables**

- Deben comenzar con letra o guión bajo (_).
- Pueden contener letras, dígitos y guiones bajos.
- Sensibles a mayúsculas/minúsculas: `nombre`, `Nombre`, `NOMBRE`.
- No pueden ser palabras reservadas del lenguaje: `int`, `class`, `if`, etc.
- Convención: usar camelCase (primera letra minúscula, siguientes palabras mayúscula).

```csharp
using System;

// ✅ Nombres válidos
int contador;
string _nombre;
string nombreCompleto;
int contadorDeElementos123;

// ❌ Nombres inválidos
// int 123contador;     // Comienza con dígito
// string nombre-completo; // Guión no permitido
// int class;           // Palabra reservada

// Palabras reservadas pueden usarse con prefijo @
int @class = 10;  // Evitar si es posible
Console.WriteLine(@class);
```

2. SISTEMA DE TIPOS EN C#
===============================================================================
C# es un lenguaje fuertemente tipado y con seguridad de tipos (type-safe).
Cada variable tiene un tipo definido que determina qué valores puede almacenar
y qué operaciones se pueden realizar.

**2.1 Tipos por Categoría**

```
┌──────────────────────────────────────────────────┐
│                  SISTEMA DE TIPOS C#              │
├──────────────────────┬───────────────────────────┤
│   TIPOS VALOR        │   TIPOS REFERENCIA        │
│   (Value Types)      │   (Reference Types)       │
├──────────────────────┼───────────────────────────┤
│   int, double, bool  │   string, class, array    │
│   char, decimal      │   interface, delegate     │
│   struct, enum       │   record, object          │
│   Almacenan valor    │   Almacenan referencia    │
│   directamente.      │   (puntero al objeto).    │
│   En la pila (stack).│   En el heap (montón).    │
└──────────────────────┴───────────────────────────┘
```

**2.2 Tabla de Tipos Integrados**

| Tipo | .NET Type | Tamaño | Rango | Precisión |
|------|-----------|--------|-------|-----------|
| `byte` | Byte | 8 bits | 0 a 255 | Entero |
| `sbyte` | SByte | 8 bits | -128 a 127 | Entero |
| `short` | Int16 | 16 bits | -32,768 a 32,767 | Entero |
| `ushort` | UInt16 | 16 bits | 0 a 65,535 | Entero |
| `int` | Int32 | 32 bits | -2^31 a 2^31-1 | Entero |
| `uint` | UInt32 | 32 bits | 0 a 2^32-1 | Entero |
| `long` | Int64 | 64 bits | -2^63 a 2^63-1 | Entero |
| `ulong` | UInt64 | 64 bits | 0 a 2^64-1 | Entero |
| `float` | Single | 32 bits | ±1.5e-45 a ±3.4e38 | ~7 dígitos |
| `double` | Double | 64 bits | ±5.0e-324 a ±1.7e308 | ~15-16 dígitos |
| `decimal` | Decimal | 128 bits | ±1.0e-28 a ±7.9e28 | 28-29 dígitos |
| `bool` | Boolean | 8 bits | true / false | - |
| `char` | Char | 16 bits | U+0000 a U+FFFF | - |
| `string` | String | Variable | - | - |
| `object` | Object | Variable | - | - |

```csharp
using System;

Console.WriteLine("=== TIPOS DE DATOS EN C# ===");
Console.WriteLine();

// Enteros
byte nivel = 255;

bool activo = true;
char inicial = 'A';

Console.WriteLine($"byte: {nivel}");
Console.WriteLine($"short: {temperatura}");

Console.WriteLine($"long: {distanciaEstelar}");
Console.WriteLine($"float: {peso}");
Console.WriteLine($"double: {precio}");
Console.WriteLine($"decimal: {salario}");
Console.WriteLine($"bool: {activo}");
Console.WriteLine($"char: {inicial}");
Console.WriteLine($"string: {nombre}");
Console.WriteLine($"object: {cualquierCosa}");
```

3. TIPOS ENTEROS
===============================================================================
**3.1 int (32 bits)**

El tipo entero más común. Usado para la mayoría de operaciones numéricas.

```csharp
using System;

int edad = 30;
int anio = DateTime.Now.Year;
int poblacion = 45000000;

// Límites
Console.WriteLine($"int.MinValue = {int.MinValue}");
Console.WriteLine($"int.MaxValue = {int.MaxValue}");

// Literales
int decimal_int = 42;       // Decimal
int hex_int = 0x2A;         // Hexadecimal (prefijo 0x)
int bin_int = 0b101010;     // Binario (prefijo 0b, C# 7+)

Console.WriteLine($"Decimal: {decimal_int}, Hex: {hex_int}, Binario: {bin_int}");

// Separadores de dígitos (C# 7+)
int mil = 1_000;
int millon = 1_000_000;
int numeroGrande = 123_456_789;

Console.WriteLine($"Con separadores: {mil}, {millon}, {numeroGrande}");
```

**3.2 long (64 bits)**

Para números enteros muy grandes.

```csharp
using System;

long distancia = 9460730472580800;  // Año luz en km
long poblacionMundial = 8_000_000_000;  // 8 mil millones

// Sufijo L (opcional pero recomendado para claridad)
long especifico = 1234567890123L;

Console.WriteLine($"Distancia (año luz): {distancia} km");
Console.WriteLine($"Población mundial: {poblacionMundial}");
Console.WriteLine($"long.MinValue = {long.MinValue}");
Console.WriteLine($"long.MaxValue = {long.MaxValue}");
```

**3.3 byte, short**

Para valores pequeños donde quieres ahorrar memoria.

```csharp
using System;

byte edadByte = 25;      // 0-255, suficiente para edad humana
short alturaCm = 175;    // -32,768 a 32,767
ushort anchoPantalla = 1920;  // Solo positivos

// Cuidado con el desbordamiento (overflow)
checked  // Lanza OverflowException si hay desbordamiento
{
    byte maximo = 255;
    // maximo = maximo + 1;  // OverflowException en checked mode
}

Console.WriteLine($"byte: {edadByte}, short: {alturaCm}, ushort: {anchoPantalla}");
```

4. TIPOS DE PUNTO FLOTANTE
===============================================================================
**4.1 float (32 bits) y double (64 bits)**

Usados para números con parte decimal. `double` es el predeterminado para
literales con punto decimal.

```csharp
using System;

// float requiere sufijo f
float peso = 72.5f;
float altura = 1.75f;

// double (por defecto)
double precio = 19.99;
double pi = 3.141592653589793;
double notacionCientifica = 1.5e10;  // 15,000,000,000

Console.WriteLine($"float peso: {peso}");
Console.WriteLine($"double pi: {pi}");
Console.WriteLine($"Notación científica: {notacionCientifica}");

// Problemas de precisión
double a = 0.1;
double b = 0.2;
double suma = a + b;
Console.WriteLine($"0.1 + 0.2 = {suma}");  // 0.30000000000000004
Console.WriteLine($"¿Es exactamente 0.3? {suma == 0.3}");  // False

// Valores especiales
double infinitoPositivo = double.PositiveInfinity;
double infinitoNegativo = double.NegativeInfinity;
double noEsNumero = double.NaN;  // Not a Number

Console.WriteLine($"1.0 / 0.0 = {1.0 / 0.0}");  // Infinity
Console.WriteLine($"-1.0 / 0.0 = {-1.0 / 0.0}"); // -Infinity
Console.WriteLine($"0.0 / 0.0 = {0.0 / 0.0}");   // NaN

// Precisión
Console.WriteLine();
Console.WriteLine($"float: {sizeof(float)} bytes, precisión ~7 dígitos");
Console.WriteLine($"double: {sizeof(double)} bytes, precisión ~15-16 dígitos");
```

**4.2 decimal (128 bits)**

Para cálculos financieros y monetarios donde la precisión es crítica.

```csharp
using System;

// decimal requiere sufijo m
decimal salario = 3500.50m;
decimal precioProducto = 19.99m;
decimal interes = 0.035m;  // 3.5%

// Operaciones precisas
decimal cantidad = 100.00m;
decimal impuesto = 0.21m;     // IVA 21%
decimal total = cantidad * (1 + impuesto);
Console.WriteLine($"Total con IVA: {total:C}");

// Decimal es preciso para dinero
decimal d1 = 0.1m;
decimal d2 = 0.2m;
decimal sumaDecimal = d1 + d2;
Console.WriteLine($"decimal: 0.1 + 0.2 = {sumaDecimal}");  // 0.3 exacto

Console.WriteLine();
Console.WriteLine($"decimal: {sizeof(decimal)} bytes, precisión 28-29 dígitos");
Console.WriteLine($"decimal.MinValue = {decimal.MinValue}");
Console.WriteLine($"decimal.MaxValue = {decimal.MaxValue}");
```

5. TIPOS BOOL, CHAR Y STRING
===============================================================================
**5.1 bool — Verdadero/Falso**

```csharp
using System;

bool esMayorEdad = true;
bool estaActivo = false;
bool esValido = (10 > 5);  // true

// Operaciones lógicas
bool a = true;
bool b = false;
Console.WriteLine($"a AND b = {a && b}");   // False
Console.WriteLine($"a OR b = {a || b}");    // True
Console.WriteLine($"NOT a = {!a}");          // False
Console.WriteLine($"a XOR b = {a ^ b}");    // True

// Parseo
bool deString = bool.Parse("true");
bool.TryParse("false", out bool deTryParse);

Console.WriteLine($"Parseado: {deString}, TryParse: {deTryParse}");
```

**5.2 char — Carácter Unicode**

```csharp
using System;

char letra = 'A';
char digito = '7';
char simbolo = '$';
char espacio = ' ';

// Unicode
char omega = '\u03A9';  // Ω (Omega griega)

Console.WriteLine($"Letra: {letra} (código: {(int)letra})");

// Métodos útiles
Console.WriteLine($"¿Es letra? {char.IsLetter(letra)}");

Console.WriteLine($"¿Es mayúscula? {char.IsUpper(letra)}");

```

**5.3 string — Cadena de texto**

```csharp
using System;

// Declaración
string nombre = "Ana María";
string vacio = "";
string nulo = null;

// Inmutabilidad: los strings no se modifican, se crean nuevos

// Concatenación

// StringBuilder para muchas concatenaciones
using System.Text;
var sb = new StringBuilder();
for (int i = 0; i < 100; i++)
{
    sb.Append(i).Append(", ");
}
Console.WriteLine(sb.ToString()[..50] + "...");

// String properties and methods
string texto = "  Programación en C#  ";
Console.WriteLine($"Original: '{texto}'");
Console.WriteLine($"Length: {texto.Length}");
Console.WriteLine($"Trim: '{texto.Trim()}'");

Console.WriteLine($"ToLower: '{texto.ToLower().Trim()}'");
Console.WriteLine($"Substring(0, 5): '{texto.Substring(0, 5)}'");
Console.WriteLine($"Contains 'C#': {texto.Contains("C#")}");
Console.WriteLine($"StartsWith '  ': {texto.StartsWith("  ")}");
Console.WriteLine($"EndsWith '#  ': {texto.EndsWith("#  ")}");
Console.WriteLine($"Replace 'C#' con 'CSharp': {texto.Replace("C#", "CSharp")}");
Console.WriteLine($"IndexOf 'C#': {texto.IndexOf("C#")}");

// Verbatim string (@) para rutas y texto multilínea
string ruta = @"C:\Users\Ana\Documentos";
string verbatim = @"Línea 1
Línea 2
Línea 3";
Console.WriteLine($"Ruta: {ruta}");

// Raw string literal (C# 11+)
string json = """
{
    "nombre": "Ana",
    "edad": 30
}
""";
Console.WriteLine(json);

// Escape sequences
string conComillas = "Dijo: \"Hola\"";

string nuevaLinea = "Primera línea\nSegunda línea";

Console.WriteLine(nuevaLinea);
Console.WriteLine(tabulador);
```

6. LA PALABRA CLAVE var — INFERENCIA DE TIPOS
===============================================================================
`var` le dice al compilador: "descubre el tipo por ti mismo". El tipo se
determina en tiempo de compilación (no es tipado dinámico).

```csharp
using System;
using System.Collections.Generic;

// El compilador infiere el tipo
var numero = 42;           // int
var texto = "Hola";        // string
var precio = 19.99;        // double
var activo = true;         // bool
var letra = 'A';           // char
var saldo = 3500.50m;      // decimal

// Con tipos complejos
var lista = new List<int>();           // List<int>
var diccionario = new Dictionary<string, int>();  // Dictionary<string, int>
var anonimo = new { Nombre = "Ana", Edad = 30 };  // Anonymous type

// LINQ
var numeros = new[] { 1, 2, 3, 4, 5 };
var pares = numeros.Where(n => n % 2 == 0);  // IEnumerable<int>

// Cuándo usar var:
// ✅ Tipo obvio por el lado derecho: var nombre = "Ana";
// ✅ Tipos complejos o largos: var dic = new Dictionary<string, List<int>>();
// ✅ Tipos anónimos: var persona = new { Nombre = "Ana" };
// ✅ LINQ: var resultado = lista.Where(x => x > 5);
// ❌ Cuando el tipo NO es obvio: var resultado = Metodo(); // ¿Qué retorna?
// ❌ Para mejorar legibilidad de tipos simples

Console.WriteLine($"numero es {numero.GetType().Name}");
Console.WriteLine($"texto es {texto.GetType().Name}");
Console.WriteLine($"precio es {precio.GetType().Name}");
Console.WriteLine($"activo es {activo.GetType().Name}");
Console.WriteLine($"letra es {letra.GetType().Name}");
Console.WriteLine($"saldo es {saldo.GetType().Name}");
```

7. TIPOS ANULABLES (NULLABLE TYPES)
===============================================================================
Por defecto, los tipos valor (int, double, bool, etc.) NO pueden ser null.
Los tipos referencia (string, class, etc.) SÍ pueden ser null.

**7.1 Nullable<T> y ?**

```csharp
using System;

// Forma explícita
Nullable<int> edad1 = null;

// Forma abreviada (sufijo ?)
int? edad2 = null;
double? precio = null;
bool? activo = null;

// Asignación de valor
edad2 = 30;
precio = 19.99m;
activo = true;

// Verificar si tiene valor
if (edad2.HasValue)
{
    Console.WriteLine($"Edad: {edad2.Value}");
}
else
{
    Console.WriteLine("Edad no especificada");
}

// Operador de fusión null (??)
int edadReal = edad2 ?? 18;  // Si edad2 es null, usa 18
Console.WriteLine($"Edad real: {edadReal}");

// Null-coalescing assignment (??=) (C# 8+)
int? valor = null;
valor ??= 10;  // Asigna 10 solo si valor es null
Console.WriteLine($"Valor después de ??=: {valor}");

// Operador condicional null (?.)
int? longitud = ObtenerNombre()?.Length;
Console.WriteLine($"Longitud: {longitud}");  // null si el método retorna null

string? ObtenerNombre()
{
    // Podría retornar null
    return null;
}
```

**7.2 Nullable Reference Types (C# 8+)**

```csharp
#nullable enable

using System;

// string? puede ser null
string? nombre = null;

// string NO puede ser null (el compilador advierte)
// string nombre = null; // Warning!

// El compilador obliga a verificar null
if (nombre is not null)
{
    Console.WriteLine(nombre.Length);  // Seguro
}

// Operador de supresión null (!)
string noNulo = nombre!;  // Le decimos al compilador que confiamos
Console.WriteLine(noNulo.Length);  // Peligroso si realmente es null

// Null-forgiving operator
void Procesar(string? valor)
{
    _ = valor!.Length;  // Suprime la advertencia
}
```

8. EL TIPO object Y EL SISTEMA UNIFICADO
===============================================================================
En C#, TODOS los tipos heredan de `System.Object`. Esto incluye tanto tipos
valor como tipos referencia.

```csharp
using System;

// object es la clase base universal
object obj1 = 42;           // Boxing de int
object obj2 = "Hola";      // string (ya es referencia)
object obj3 = 3.14;        // Boxing de double
object obj4 = true;         // Boxing de bool
object obj5 = new { X = 10 };  // Cualquier objeto

// Boxing: Tipo valor → object (copiado al heap)
int numero = 42;
object boxed = numero;  // Boxing implícito
Console.WriteLine($"Boxed: {boxed}");

// Unboxing: object → Tipo valor (con verificación de tipo)
object valor = 42;
int unboxed = (int)valor;  // Unboxing explícito (requiere cast)
Console.WriteLine($"Unboxed: {unboxed}");

// El unboxing debe ser al tipo exacto
object dato = 42;
// long erroneo = (long)dato;  // InvalidCastException!
long correcto = (int)dato;    // Primero a int, luego a long
Console.WriteLine($"Correcto: {correcto}");

// GetType() revela el tipo real en runtime
Console.WriteLine($"42 es de tipo: {42.GetType().Name}");
Console.WriteLine($"'Hola' es de tipo: {"Hola".GetType().Name}");
Console.WriteLine($"3.14 es de tipo: {3.14.GetType().Name}");
Console.WriteLine($"true es de tipo: {true.GetType().Name}");

// ToString() — Todos los tipos tienen representación string
Console.WriteLine($"42.ToString() = {42.ToString()}");
Console.WriteLine($"3.14.ToString(\"F2\") = {3.14.ToString("F2")}");
Console.WriteLine($"true.ToString() = {true.ToString()}");
```

9. CONVERSIÓN DE TIPOS
===============================================================================
**9.1 Conversión implícita (segura, sin pérdida de datos)**

```csharp
using System;

// De menor a mayor capacidad (sin pérdida)
int entero = 100;
long largo = entero;       // int → long (implícito)
float flotante = entero;   // int → float (implícito)
double doble = entero;     // int → double (implícito)
double doble2 = flotante;  // float → double (implícito)

// Jerarquía de conversiones implícitas:
// byte → short → int → long → float → double
//                int → decimal
//                long → decimal
// char → int

Console.WriteLine($"int {entero} → long {largo}");
Console.WriteLine($"int {entero} → float {flotante}");
Console.WriteLine($"int {entero} → double {doble}");
```

**9.2 Conversión explícita (cast) — puede perder datos**

```csharp
using System;

// De mayor a menor capacidad (posible pérdida)
double doble = 123.456;
int entero = (int)doble;  // Trunca: 123 (pierde .456)
Console.WriteLine($"double {doble} → int {entero} (pérdida)");

long largo = 1000L;
int entero2 = (int)largo;  // Seguro si no supera int.MaxValue
Console.WriteLine($"long {largo} → int {entero2}");

// Conversión checked: lanza excepción si hay desbordamiento
checked
{
    int max = int.MaxValue;
    // int resultado = max + 1;  // OverflowException
}

// Conversión unchecked: permite desbordamiento silencioso
unchecked
{
    int max = int.MaxValue;
    int resultado = max + 1;  // Vuelve a int.MinValue
    Console.WriteLine($"Desbordamiento: {max} + 1 = {resultado}");
}
```

**9.3 Conversión con métodos Parse y TryParse**

```csharp
using System;

// Parse: lanza excepción si falla
int entero = int.Parse("123");
double doble = double.Parse("123.45");
bool booleano = bool.Parse("true");

Console.WriteLine($"Parse '123' → int: {entero}");
Console.WriteLine($"Parse '123.45' → double: {doble}");

// TryParse: retorna bool, no lanza excepción
if (int.TryParse("abc", out int resultado))
{
    Console.WriteLine($"Éxito: {resultado}");
}
else
{
    Console.WriteLine("Fallo en TryParse (esperado)");
}

// Ejemplo con validación
Console.Write("Ingresa un número: ");
string entrada = Console.ReadLine();
if (int.TryParse(entrada, out int numeroValido))
{
    Console.WriteLine($"Número válido: {numeroValido}");
}
else
{
    Console.WriteLine("¡Eso no es un número válido!");
}
```

**9.4 Conversión con la clase Convert**

```csharp
using System;

// Convert proporciona métodos para conversiones entre tipos base
int entero = Convert.ToInt32("123");
double doble = Convert.ToDouble("123.45");
bool booleano = Convert.ToBoolean("true");
char caracter = Convert.ToChar(65);  // 'A' (código ASCII)

Console.WriteLine($"Convert.ToInt32: {entero}");
Console.WriteLine($"Convert.ToDouble: {doble}");
Console.WriteLine($"Convert.ToBoolean: {booleano}");
Console.WriteLine($"Convert.ToChar(65): {caracter}");

// Convert maneja null (devuelve valor default)
int? nulo = null;
int conDefault = Convert.ToInt32(nulo);  // 0
Console.WriteLine($"Convert con null: {conDefault}");
```

**9.5 Conversión con as/is (solo tipos referencia)**

```csharp
using System;

object valor = "Hola Mundo";

// is: verifica si una variable es de cierto tipo
if (valor is string)
{
    Console.WriteLine("valor es un string");
}

// is + pattern matching (C# 7+)
if (valor is string texto)
{
    Console.WriteLine($"Longitud del texto: {texto.Length}");
}

// as: intenta convertir, retorna null si falla (no excepción)
object objeto = "Texto";
string? cadena = objeto as string;
if (cadena is not null)
{
    Console.WriteLine($"Conversión exitosa: {cadena}");
}

// as con tipos que NO coinciden retorna null
object numero = 42;
string? noString = numero as string;  // null
Console.WriteLine($"as string con int: {(noString is null ? "null" : noString)}");
```

12. DIFERENCIA ENTRE TIPOS VALOR Y REFERENCIA
===============================================================================
**12.1 Comportamiento en asignación**

```csharp
using System;

// TIPOS VALOR: se copia el valor
int a = 10;
int b = a;  // b obtiene una COPIA del valor de a
b = 20;     // a sigue siendo 10
Console.WriteLine($"Valor: a={a}, b={b}");  // a=10, b=20

// TIPOS REFERENCIA: se copia la referencia
int[] arr1 = [1, 2, 3];
int[] arr2 = arr1;  // arr2 apunta al MISMO array
arr2[0] = 999;      // arr1 también se ve afectado
Console.WriteLine($"Referencia: arr1[0]={arr1[0]}, arr2[0]={arr2[0]}");  // 999, 999

// Ejemplo con clases
class PersonaClass
{
    public string Nombre { get; set; }
}

PersonaClass p1 = new PersonaClass { Nombre = "Ana" };
PersonaClass p2 = p1;  // misma referencia
p2.Nombre = "Luis";    // p1.Nombre también cambia
Console.WriteLine($"p1.Nombre={p1.Nombre}, p2.Nombre={p2.Nombre}");  // Luis, Luis
```

**12.2 struct vs class**

```csharp
using System;

// struct: tipo valor
struct PuntoValor
{
    public int X;
    public int Y;
}

// class: tipo referencia
class PuntoReferencia
{
    public int X;
    public int Y;
}

var pv1 = new PuntoValor { X = 10, Y = 20 };
var pv2 = pv1;  // Copia independiente
pv2.X = 100;
Console.WriteLine($"Struct: pv1.X={pv1.X}, pv2.X={pv2.X}");  // 10, 100

var pr1 = new PuntoReferencia { X = 10, Y = 20 };
var pr2 = pr1;  // Misma referencia
pr2.X = 100;
Console.WriteLine($"Class: pr1.X={pr1.X}, pr2.X={pr2.X}");  // 100, 100
```

14. CHECKED Y UNCHECKED — CONTROL DE DESBORDAMIENTO
===============================================================================
```csharp
using System;

// checked: lanza OverflowException en desbordamiento
checked
{
    try
    {
        int max = int.MaxValue;
        int resultado = max + 1;
        Console.WriteLine($"Esto no se ejecuta");
    }
    catch (OverflowException ex)
    {
        Console.WriteLine($"Overflow detectado (checked): {ex.Message}");
    }
}

// unchecked: permite desbordamiento silencioso (por defecto)
unchecked
{
    int max = int.MaxValue;
    int resultado = max + 1;
    Console.WriteLine($"Overflow silencioso (unchecked): {resultado}");
}

// Configuración global en .csproj:
// <CheckForOverflowUnderflow>true</CheckForOverflowUnderflow>
```

RESUMEN DEL CAPÍTULO
===============================================================================
- Las variables se declaran con `tipo nombre;` y se asignan con `=`.
- C# tiene tipos valor (int, double, bool, char, struct, enum) que almacenan
  el valor directamente, y tipos referencia (string, class, array, object) que
  almacenan una referencia al objeto en el heap.
- Los principales tipos numéricos son int (32 bits), long (64 bits), float,
  double y decimal (128 bits, alta precisión para dinero).
- bool almacena true/false, char almacena un carácter Unicode, string almacena
  texto inmutable.
- `var` permite inferencia de tipos en tiempo de compilación; útil para tipos
  complejos y anónimos.
- Los tipos anulables (int?, double?, bool?) permiten que los tipos valor
  puedan ser null, usando `?` después del tipo.
- `object` es la clase base de todos los tipos. Boxing convierte tipo valor a
  object; unboxing hace la conversión inversa.
- Las conversiones pueden ser implícitas (sin pérdida), explícitas/ cast
  (con posible pérdida), o mediante Parse/TryParse/Convert.
- `default(T)` devuelve el valor por defecto de un tipo (0 para numéricos,
  false para bool, null para referencias).
- `const` define valores fijos en compilación; `readonly` en tiempo de
  ejecución.
- `checked` y `unchecked` controlan el comportamiento en desbordamientos
  aritméticos.
- Los strings son inmutables; cada "modificación" crea un nuevo string.
- El operador `??` proporciona un valor por defecto si una variable es null.
- String interpolation ($"...{variable}...") es la forma moderna de formatear.

PREGUNTAS FRECUENTES
===============================================================================
❓ **¿Cuál es la diferencia entre int y long?**
int es de 32 bits (rango ±2.1 mil millones), long es de 64 bits (rango
±9.2 trillones). Usa int para la mayoría de casos y long cuando necesites
números más grandes.

❓ **¿Cuándo usar decimal vs double?**
Usa decimal para dinero y cálculos financieros donde la precisión es crítica.
Usa double para cálculos científicos, gráficos y cuando el rendimiento importa.

❓ **¿Qué significa 'var' exactamente?**
var le dice al compilador "infiere el tipo". Es 100% tipado estáticamente: el
tipo se determina en compilación, no en ejecución. var es equivalente a
escribir el tipo explícitamente.

❓ **¿Por qué no debería usar var siempre?**
var puede hacer el código menos legible cuando el tipo no es obvio desde el
contexto (ej: `var x = Metodo();`). Úsalo cuando el tipo sea claro: literales,
constructores (`new`), LINQ.

❓ **¿Qué es boxing y unboxing?**
Boxing: convertir un tipo valor a object (se copia al heap). Unboxing: la
operación inversa. Tiene costo de rendimiento, evítalo en bucles intensivos.

❓ **¿Cómo elijo entre struct y class?**
Los structs son tipos valor (copia, pila, más rápidos). Úsalos para datos
pequeños e inmutables (< 16 bytes). Las clases son tipos referencia (heap,
más flexibles). Por defecto, usa class.

❓ **¿Qué es 'string?' (desde C# 8)?**
Es una string que puede ser null. En proyectos con Nullable habilitado, el
compilador te obliga a verificar null antes de usarla, previniendo
NullReferenceException.

❓ **¿Por qué se usa más 'int' que 'long' o 'short'?**
int es el tamaño natural del procesador (32 bits). Es eficiente y suficiente
para la mayoría de casos. short ahorra memoria en arrays grandes. long se usa
cuando int no alcanza.

❓ **¿Qué pasa si divido dos enteros?**
El resultado es entero (truncado). `7 / 2 = 3`, no 3.5. Para obtener decimal,
convierte al menos un operando a double: `7 / 2.0` o `(double)7 / 2`.

❓ **¿Cómo convierto un número a string?**
Usa `.ToString()`, `$"{numero}"`, `Convert.ToString(numero)`, o
`string.Format("{0}", numero)`.
