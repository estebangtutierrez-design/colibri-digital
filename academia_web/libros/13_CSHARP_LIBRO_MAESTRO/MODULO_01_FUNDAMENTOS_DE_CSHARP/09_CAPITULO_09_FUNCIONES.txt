1. DEFINICIÓN DE MÉTODOS — SINTAXIS BÁSICA
===============================================================================
Un método es un bloque de código con nombre que realiza una tarea específica.

```csharp
using System;

// Sintaxis básica:
// [modificadores] tipo_retorno NombreMetodo([parámetros])
// {
//     // Cuerpo del método
//     return valor;
// }

// Método simple que no retorna nada (void) ni recibe parámetros
void Saludar()
{
    Console.WriteLine("¡Hola desde un método!");
}

// Método que recibe un parámetro
void SaludarPersona(string nombre)
{
    Console.WriteLine($"¡Hola, {nombre}!");
}

// Método que retorna un valor
int Sumar(int a, int b)
{
    return a + b;
}

// Método con múltiples parámetros y retorno
double CalcularIMC(double peso, double altura)
{
    return peso / (altura * altura);
}

// Llamar a los métodos
Saludar();
SaludarPersona("Ana");
int resultado = Sumar(10, 20);
Console.WriteLine($"10 + 20 = {resultado}");

double imc = CalcularIMC(70, 1.75);
Console.WriteLine($"IMC: {imc:F2}");
```

2. PARÁMETROS Y VALORES DE RETORNO
===============================================================================
```csharp
using System;

// Método sin parámetros y sin retorno
void MostrarBienvenida()
{
    Console.WriteLine("╔═══════════════════╗");
    Console.WriteLine("║   BIENVENIDO     ║");
    Console.WriteLine("╚═══════════════════╝");
}

// Método con un parámetro y sin retorno
void MostrarMensaje(string mensaje, ConsoleColor color = ConsoleColor.White)
{
    Console.ForegroundColor = color;
    Console.WriteLine(mensaje);
    Console.ResetColor();
}

// Método con múltiples parámetros y retorno
int CalcularPotencia(int baseNum, int exponente)
{
    int resultado = 1;
    for (int i = 0; i < exponente; i++)
    {
        resultado *= baseNum;
    }
    return resultado;
}

// Método que retorna un bool
bool EsPar(int numero)
{
    return numero % 2 == 0;
}

// Método que retorna múltiples valores (tupla)
(int suma, int producto) CalcularOperaciones(int a, int b)
{
    return (a + b, a * b);
}

// Método con expresión body (=>)
int Cuadrado(int x) => x * x;

// Uso
MostrarBienvenida();
MostrarMensaje("Mensaje importante", ConsoleColor.Yellow);

int potencia = CalcularPotencia(2, 10);
Console.WriteLine($"2^10 = {potencia}");

Console.WriteLine($"7 es par? {EsPar(7)}");

var operaciones = CalcularOperaciones(5, 3);
Console.WriteLine($"Suma: {operaciones.suma}, Producto: {operaciones.producto}");

Console.WriteLine($"Cuadrado de 9: {Cuadrado(9)}");

// Pasando por valor (copia)
int x = 10;
void Incrementar(int valor)
{
    valor++;
    Console.WriteLine($"Dentro del método: {valor}");  // 11
}
Incrementar(x);
Console.WriteLine($"Fuera del método: {x}");  // 10 (sin cambios)

// Pasando por referencia (ref)
void IncrementarRef(ref int valor)
{
    valor++;
    Console.WriteLine($"Dentro del método: {valor}");  // 11
}
int y = 10;
IncrementarRef(ref y);
Console.WriteLine($"Fuera del método: {y}");  // 11 (cambiado)
```

3. PARÁMETROS ref, out, in
===============================================================================
**3.1 ref — Referencia (lectura/escritura)**

```csharp
using System;

// ref: la variable debe estar inicializada ANTES de pasarla
void Intercambiar(ref int a, ref int b)
{
    int temp = a;
    a = b;
    b = temp;
}

int x = 10, y = 20;
Console.WriteLine($"Antes: x={x}, y={y}");
Intercambiar(ref x, ref y);
Console.WriteLine($"Después: x={x}, y={y}");

// ref con tipos referencia
void AgregarElemento(ref List<string> lista, string elemento)
{
    if (lista is null)
        lista = new List<string>();
    lista.Add(elemento);
}

List<string>? nombres = null;
AgregarElemento(ref nombres, "Ana");
AgregarElemento(ref nombres, "Luis");
Console.WriteLine($"Lista: {string.Join(", ", nombres!)}");
```

**3.2 out — Salida (solo escritura)**

```csharp
using System;

// out: la variable no necesita inicializarse; debe asignarse dentro del método
bool Dividir(int dividendo, int divisor, out int cociente, out int resto)
{
    if (divisor == 0)
    {
        cociente = 0;
        resto = 0;
        return false;
    }
    cociente = dividendo / divisor;
    resto = dividendo % divisor;
    return true;
}

if (Dividir(10, 3, out int cociente, out int resto))
{
    Console.WriteLine($"10 / 3 = {cociente}, resto = {resto}");
}
else
{
    Console.WriteLine("División por cero");
}

// TryParse es un ejemplo clásico de out
if (int.TryParse("123", out int numero))
{
    Console.WriteLine($"Número: {numero}");
}

// Múltiples valores de salida con out
void ObtenerEstadisticas(int[] numeros, out int minimo, out int maximo, out double promedio)
{
    minimo = numeros.Min();
    maximo = numeros.Max();
    promedio = numeros.Average();
}

int[] datos = [10, 25, 3, 47, 18, 32];
ObtenerEstadisticas(datos, out int min, out int max, out double prom);
Console.WriteLine($"Min: {min}, Max: {max}, Prom: {prom:F2}");
```

**3.3 in — Referencia de solo lectura**

```csharp
using System;

// in: la referencia es de solo lectura (más eficiente para structs grandes)
void MostrarCoordenada(in Punto punto)
{
    // punto.X = 10;  // Error: no se puede modificar
    Console.WriteLine($"Coordenada: ({punto.X}, {punto.Y})");
}

var punto = new Punto(5, 10);
MostrarCoordenada(in punto);  // in es opcional aquí

// in es útil con structs grandes para evitar copias
void ProcesarRectangulo(in Rectangulo rect)
{
    int area = rect.Ancho * rect.Alto;
    Console.WriteLine($"Área: {area}");
}

var rect = new Rectangulo(100, 50);
ProcesarRectangulo(in rect);

struct Punto
{
    public int X;
    public int Y;
    public Punto(int x, int y) { X = x; Y = y; }
}

struct Rectangulo
{
    public int Ancho;
    public int Alto;
    public Rectangulo(int ancho, int alto) { Ancho = ancho; Alto = alto; }
}
```

4. SOBRECARGA DE MÉTODOS (METHOD OVERLOADING)
===============================================================================
La sobrecarga permite tener múltiples métodos con el mismo nombre pero
diferentes parámetros.

```csharp
using System;

// Sobrecarga: mismo nombre, diferentes parámetros
void Mostrar(int valor)
{
    Console.WriteLine($"Entero: {valor}");
}

void Mostrar(double valor)
{
    Console.WriteLine($"Double: {valor:F2}");
}

void Mostrar(string valor)
{
    Console.WriteLine($"String: '{valor}'");
}

void Mostrar(int valor, string formato)
{
    Console.WriteLine($"Formateado: {valor.ToString(formato)}");
}

void Mostrar(params int[] valores)
{
    Console.WriteLine($"Array: {string.Join(", ", valores)}");
}

// Uso de la sobrecarga
Mostrar(42);
Mostrar(3.1416);
Mostrar("Hola Mundo");
Mostrar(255, "X");
Mostrar(1, 2, 3, 4, 5);

// Sobrecarga de constructores (aunque no usamos clases formales aquí)
// La sobrecarga también aplica a métodos de instancia y constructores

// Sobrecarga con diferente número de parámetros
int Sumar(int a, int b) => a + b;
int Sumar(int a, int b, int c) => a + b + c;
int Sumar(int a, int b, int c, int d) => a + b + c + d;

Console.WriteLine($"\nSuma (2): {Sumar(1, 2)}");
Console.WriteLine($"Suma (3): {Sumar(1, 2, 3)}");
Console.WriteLine($"Suma (4): {Sumar(1, 2, 3, 4)}");

// Sobrecarga con tipos diferentes
string Formatear(int valor) => $"Número: {valor}";
string Formatear(double valor) => $"Decimal: {valor:F2}";
string Formatear(DateTime fecha) => $"Fecha: {fecha:dd/MM/yyyy}";

Console.WriteLine(Formatear(42));
Console.WriteLine(Formatear(3.14));
Console.WriteLine(Formatear(DateTime.Now));
```

6. EXPRESSION-BODIED MEMBERS
===============================================================================
Sintaxis compacta para métodos y propiedades de una sola línea (C# 6+).

```csharp
using System;

// Expression-bodied method (=>)
int Duplicar(int x) => x * 2;
double AreaCirculo(double radio) => Math.PI * radio * radio;
bool EsPositivo(int n) => n > 0;
string ObtenerSaludo(string nombre) => $"¡Hola, {nombre}!";

// Expression-bodied property (solo get)
string NombreAplicacion => "C# Master Book";
DateTime MomentoActual => DateTime.Now;
double Version => 1.0;

// Expression-bodied property con set (C# 7+)
int _contador;
int Contador
{
    get => _contador;
    set => _contador = value >= 0 ? value : 0;
}

// Expression-bodied constructor (C# 7+)
class Punto
{
    public int X { get; }
    public int Y { get; }
    public Punto(int x, int y) => (X, Y) = (x, y);
    public void Deconstruct(out int x, out int y) => (x, y) = (X, Y);
}

// Expression-bodied destructor (C# 7+)
class Recurso : IDisposable
{
    private bool _disposed = false;
    ~Recurso() => Dispose(false);
    public void Dispose() => Dispose(true);
    private void Dispose(bool disposing) { if (!_disposed) { _disposed = true; } }
}

// Uso
Console.WriteLine(Duplicar(21));
Console.WriteLine($"Área círculo radio 5: {AreaCirculo(5):F2}");
Console.WriteLine(ObtenerSaludo("Ana"));
Console.WriteLine($"App: {NombreAplicacion}, Versión: {Version}");

Contador = 5;
Console.WriteLine($"Contador: {Contador}");

var p = new Punto(3, 4);
var (x, y) = p;
Console.WriteLine($"Punto: ({x}, {y})");
```

7. FUNCIONES LOCALES (LOCAL FUNCTIONS)
===============================================================================
Funciones definidas dentro de otras funciones (C# 7+).

```csharp
using System;

void ProcesarPedido(int idPedido)
{
    Console.WriteLine($"Procesando pedido #{idPedido}...");
    
    // Función local — solo visible dentro de ProcesarPedido
    decimal CalcularImpuesto(decimal subtotal, decimal tasa)
    {
        return subtotal * tasa;
    }
    
    decimal CalcularDescuento(decimal subtotal, int antiguedad)
    {
        // Puede acceder a variables del método padre
        if (antiguedad > 12)
        {
            int descuentoPorcentaje = 10;
            return subtotal * descuentoPorcentaje / 100;
        }
        return 0;
    }
    
    // También pueden ser expression-bodied
    decimal CalcularEnvio(decimal subtotal) => subtotal > 100 ? 0 : 9.99m;
    
    // Uso de las funciones locales
    decimal subtotal = 250.00m;
    decimal impuesto = CalcularImpuesto(subtotal, 0.21m);
    decimal descuento = CalcularDescuento(subtotal, 18);
    decimal envio = CalcularEnvio(subtotal);
    decimal total = subtotal + impuesto - descuento + envio;
    
    Console.WriteLine($"  Subtotal: {subtotal:C2}");
    Console.WriteLine($"  Impuesto: {impuesto:C2}");
    Console.WriteLine($"  Descuento: -{descuento:C2}");
    Console.WriteLine($"  Envío: {envio:C2}");
    Console.WriteLine($"  Total: {total:C2}");
}

ProcesarPedido(1001);

// Las funciones locales pueden ser estáticas (C# 8+)
void Calcular()
{
    // Las funciones locales static no pueden acceder a variables del padre
    static int Sumar(int a, int b) => a + b;
    
    Console.WriteLine($"Suma: {Sumar(5, 3)}");
}

Calcular();

// Las funciones locales pueden usar iteradores (yield)
IEnumerable<int> ObtenerPares(int limite)
{
    IEnumerable<int> GenerarPares()
    {
        for (int i = 0; i <= limite; i += 2)
        {
            yield return i;
        }
    }
    
    return GenerarPares();
}

Console.WriteLine("\nPares hasta 10:");
foreach (var par in ObtenerPares(10))
{
    Console.Write($"{par} ");
}
Console.WriteLine();

// Las funciones locales pueden ser async
async Task<string> DescargarDatosAsync(string url)
{
    async Task<string> HttpGetAsync(string url)
    {
        using var client = new HttpClient();
        return await client.GetStringAsync(url);
    }
    
    return await HttpGetAsync(url);
}
```

8. DELEGADOS — REFERENCIAS A FUNCIONES
===============================================================================
Los delegados permiten tratar funciones como datos.

```csharp
using System;

// Declaración de un delegado
delegate int Operacion(int a, int b);

// Métodos que coinciden con la firma del delegado
int Sumar(int x, int y) => x + y;
int Restar(int x, int y) => x - y;
int Multiplicar(int x, int y) => x * y;

// Crear instancia del delegado
Operacion op = Sumar;
Console.WriteLine($"Sumar: {op(10, 5)}");

// Cambiar la función referenciada
op = Restar;
Console.WriteLine($"Restar: {op(10, 5)}");

op = Multiplicar;
Console.WriteLine($"Multiplicar: {op(10, 5)}");

// Delegados predefinidos: Func, Action, Predicate
// Func<entrada, salida>
Func<int, int> cuadrado = x => x * x;
Console.WriteLine($"Cuadrado (Func): {cuadrado(7)}");

Func<int, int, int> sumarFunc = (a, b) => a + b;
Console.WriteLine($"Suma (Func): {sumarFunc(10, 20)}");

// Action — no retorna valor
Action<string> mostrar = msg => Console.WriteLine(msg);
mostrar("Mensaje desde Action");

// Predicate — retorna bool
Predicate<int> esPar = n => n % 2 == 0;
Console.WriteLine($"¿8 es par? {esPar(8)}");

// Multicast delegate (varios métodos)
void Metodo1() => Console.WriteLine("Método 1 ejecutado");
void Metodo2() => Console.WriteLine("Método 2 ejecutado");
void Metodo3() => Console.WriteLine("Método 3 ejecutado");

Action accion = Metodo1;
accion += Metodo2;  // Agregar método
accion += Metodo3;
Console.WriteLine("\nMulticast delegate:");
accion();  // Ejecuta los tres en orden

accion -= Metodo2;  // Quitar método
Console.WriteLine("\nDespués de quitar Método 2:");
accion();  // Solo ejecuta Metodo1 y Metodo3
```

9. MÉTODOS DE EXTENSIÓN
===============================================================================
Los métodos de extensión permiten agregar funcionalidad a tipos existentes.

```csharp
using System;
using System.Text;

// Métodos de extensión: deben estar en una clase estática
public static class StringExtensions
{
    // Primer parámetro con 'this' indica el tipo a extender
    public static string Invertir(this string texto)
    {
        if (string.IsNullOrEmpty(texto)) return texto;
        char[] chars = texto.ToCharArray();
        Array.Reverse(chars);
        return new string(chars);
    }
    
    public static int ContarPalabras(this string texto)
    {
        if (string.IsNullOrWhiteSpace(texto)) return 0;
        return texto.Split(' ', StringSplitOptions.RemoveEmptyEntries).Length;
    }
    
    public static string Truncar(this string texto, int maxLongitud)
    {
        if (string.IsNullOrEmpty(texto) || texto.Length <= maxLongitud)
            return texto;
        return texto[..maxLongitud] + "...";
    }
}

public static class IntExtensions
{
    public static bool EsPar(this int numero) => numero % 2 == 0;
    
    public static bool EsPrimo(this int numero)
    {
        if (numero < 2) return false;
        for (int i = 2; i * i <= numero; i++)
            if (numero % i == 0) return false;
        return true;
    }
    
    public static string ALetras(this int numero)
    {
        string[] unidades = ["cero", "uno", "dos", "tres", "cuatro",
                            "cinco", "seis", "siete", "ocho", "nueve"];
        return numero >= 0 && numero <= 9 ? unidades[numero] : numero.ToString();
    }
}

public static class ListExtensions
{
    public static void Mostrar<T>(this List<T> lista, string titulo = "")
    {
        if (!string.IsNullOrEmpty(titulo))
            Console.WriteLine(titulo);
        Console.WriteLine($"[{string.Join(", ", lista)}]");
    }
}

// Uso de métodos de extensión
string texto = "Hola Mundo desde C#";

Console.WriteLine($"Original: {texto}");
Console.WriteLine($"Invertido: {texto.Invertir()}");
Console.WriteLine($"Palabras: {texto.ContarPalabras()}");
Console.WriteLine($"Truncado (10): {texto.Truncar(10)}");

int numero = 7;
Console.WriteLine($"\n{numero}:");
Console.WriteLine($"  ¿Es par? {numero.EsPar()}");
Console.WriteLine($"  ¿Es primo? {numero.EsPrimo()}");
Console.WriteLine($"  En letras: {numero.ALetras()}");

var lista = new List<int> { 1, 2, 3, 4, 5 };
lista.Mostrar("Números:");

// Los métodos de extensión también funcionan con LINQ
var filtrados = lista.Where(n => n.EsPar()).ToList();
filtrados.Mostrar("Pares:");
```

10. RECURSIÓN
===============================================================================
La recursión ocurre cuando un método se llama a sí mismo.

```csharp
using System;

// Factorial recursivo
long Factorial(int n)
{
    if (n <= 1) return 1;  // Caso base
    return n * Factorial(n - 1);  // Llamada recursiva
}

// Fibonacci recursivo
int Fibonacci(int n)
{
    if (n <= 1) return n;
    return Fibonacci(n - 1) + Fibonacci(n - 2);
}

// Potencia recursiva
int Potencia(int baseNum, int exponente)
{
    if (exponente == 0) return 1;
    return baseNum * Potencia(baseNum, exponente - 1);
}

// Recorrer directorios (simulado)
void ExplorarDirectorio(string directorio, int nivel = 0)
{
    string indent = new string(' ', nivel * 2);
    Console.WriteLine($"{indent}📁 {directorio}");
    
    if (nivel < 3)  // Límite para evitar recursión infinita
    {
        // Simular subdirectorios
        for (int i = 1; i <= 3; i++)
        {
            ExplorarDirectorio($"{directorio}/Sub{i}", nivel + 1);
        }
    }
}

// Uso
Console.WriteLine($"Factorial de 7: {Factorial(7)}");
Console.WriteLine($"Fibonacci de 10: {Fibonacci(10)}");
Console.WriteLine($"2^10 = {Potencia(2, 10)}");

Console.WriteLine("\nEstructura de directorios:");
ExplorarDirectorio("Proyecto");

// Fibonacci con memoización (optimización)
int FibonacciFast(int n, int[]? memo = null)
{
    memo ??= new int[n + 1];
    Array.Fill(memo, -1);
    return Fib(n, memo);
    
    int Fib(int n, int[] memo)
    {
        if (memo[n] != -1) return memo[n];
        if (n <= 1) return n;
        memo[n] = Fib(n - 1, memo) + Fib(n - 2, memo);
        return memo[n];
    }
}

Console.WriteLine($"\nFibonacci rápido de 40: {FibonacciFast(40)}");
```

11. MÉTODOS ASÍNCRONOS (async/await)
===============================================================================
```csharp
using System;
using System.Net.Http;
using System.Threading.Tasks;

async Task<string> DescargarContenidoAsync(string url)
{
    using var client = new HttpClient();
    client.Timeout = TimeSpan.FromSeconds(10);
    return await client.GetStringAsync(url);
}

async Task ProcesarMultipleAsync()
{
    string[] urls = [
        "https://example.com",
        "https://httpbin.org/get",
        "https://jsonplaceholder.typicode.com/posts/1"
    ];
    
    var tareas = urls.Select(url => DescargarContenidoAsync(url));
    string[] resultados = await Task.WhenAll(tareas);
    
    for (int i = 0; i < urls.Length; i++)
    {
        Console.WriteLine($"URL {urls[i]}: {resultados[i][..50]}...");
    }
}

// Método async que retorna un valor
async Task<int> CalcularSumaAsync(int a, int b)
{
    await Task.Delay(100);  // Simula trabajo asíncrono
    return a + b;
}

// Uso (en Main asíncrono)
Console.WriteLine("Iniciando descargas...");
try
{
    int resultado = await CalcularSumaAsync(10, 20);
    Console.WriteLine($"Suma asíncrona: {resultado}");
    
    // await ProcesarMultipleAsync();  // Descomentar para probar
}
catch (HttpRequestException ex)
{
    Console.WriteLine($"Error de red: {ex.Message}");
}
catch (TaskCanceledException)
{
    Console.WriteLine("Operación cancelada (timeout)");
}

Console.WriteLine("Programa finalizado");
```

```csharp
// 1. Responsabilidad única: un método, una tarea
// ❌ Malo
void ProcesarUsuario(string nombre, int edad)
{

void MostrarUsuario(string nombre, int edad) => Console.WriteLine($"{nombre}: {edad}");
void GuardarEnBaseDeDatos(string nombre, int edad) { /* ... */ }
void EnviarEmail(string nombre) { /* ... */ }

// 2. Nombres descriptivos
// ❌ Malo
void P(int a, int b) { /* ... */ }

// ✅ Bueno
void ProcesarPago(decimal monto, int idCliente) { /* ... */ }

// 3. Número reducido de parámetros (ideal <= 3)
// ❌ Malo
void CrearUsuario(string n, string e, int ed, string t, string d, bool a) { }

// ✅ Bueno: Agrupar en un objeto
void CrearUsuario(Usuario usuario) { }

record Usuario(string Nombre, string Email, int Edad, string Telefono, string Direccion, bool Activo);

// 4. Evitar efectos secundarios (side effects)
int _estadoGlobal = 0;

// ❌ Malo: modifica estado global
void IncrementarEstado() => _estadoGlobal++;

// ✅ Bueno: recibe y retorna
int Incrementar(int valor) => valor + 1;

// 5. Validar parámetros al inicio
string ObtenerSubstring(string texto, int inicio, int longitud)
{
    if (texto is null) throw new ArgumentNullException(nameof(texto));
    if (inicio < 0) throw new ArgumentOutOfRangeException(nameof(inicio));
    if (longitud < 0) throw new ArgumentOutOfRangeException(nameof(longitud));
    if (inicio + longitud > texto.Length)
        throw new ArgumentException("Rango fuera del texto");
    
    return texto.Substring(inicio, longitud);
}

// 6. Preferir métodos puros (sin estado)
// ✅ Determinista: misma entrada → misma salida
int SumaPura(int a, int b) => a + b;

// ❌ No determinista (depende de DateTime.Now)
int ObtenerDiaMes() => DateTime.Now.Day;

// 7. Usar expression body para métodos simples
int DuplicarValor(int x) => x * 2;

// 8. Documentar con XML comments
/// <summary>
/// Calcula el área de un círculo dado su radio.
/// </summary>
/// <param name="radio">Radio del círculo (debe ser >= 0).</param>
/// <returns>Área del círculo.</returns>
double AreaCirculoDocumentado(double radio)
{
    if (radio < 0) throw new ArgumentException("El radio no puede ser negativo");
    return Math.PI * radio * radio;
}
```

RESUMEN DEL CAPÍTULO
===============================================================================
- Los métodos se definen con: `[modificadores] tipoRetorno Nombre([parámetros])`.
- Los métodos pueden retornar void (nada), un valor, o una tupla de valores.
- `ref` pasa el parámetro por referencia (lectura/escritura); la variable debe
  estar inicializada.
- `out` pasa el parámetro por referencia (solo escritura); no necesita
  inicialización previa.
- `in` pasa el parámetro por referencia de solo lectura; útil para structs
  grandes.
- La sobrecarga (overloading) permite múltiples métodos con el mismo nombre
  pero diferentes parámetros.
- `params` permite un número variable de argumentos del mismo tipo.
- Los parámetros opcionales tienen valores por defecto; deben ir al final.
- Los argumentos con nombre permiten pasar parámetros en cualquier orden.
- `expression-bodied members` (=>) ofrecen sintaxis compacta para métodos y
  propiedades de una línea.
- Las funciones locales (C# 7+) son métodos dentro de métodos; pueden acceder
  a variables del método padre y ser estáticas.
- Los métodos de extensión agregan funcionalidad a tipos existentes sin
  modificar el tipo original.
- Los delegados (Func, Action, Predicate) permiten tratar funciones como datos.
- La recursión ocurre cuando un método se llama a sí mismo; requiere un caso
  base para evitar recursión infinita.
- async/await permite métodos asíncronos que no bloquean el hilo principal.
- Buenas prácticas: responsabilidad única, nombres descriptivos, pocos
  parámetros, validar entradas, evitar efectos secundarios, preferir métodos
  puros.

PREGUNTAS FRECUENTES
===============================================================================
❓ **¿Cuál es la diferencia entre ref y out?**
ref requiere que la variable esté inicializada antes de pasarla. out no necesita
inicialización, pero el método debe asignarle un valor antes de retornar.

❓ **¿Puedo tener métodos con el mismo nombre y mismos parámetros pero
diferente retorno?**
No. La sobrecarga se basa en los parámetros (número, tipo, orden), no en el
tipo de retorno. El compilador no puede distinguir solo por el retorno.

❓ **¿Qué son los métodos de extensión y cómo se definen?**
Son métodos estáticos en una clase estática que extienden un tipo existente.
Se definen con `this` antes del primer parámetro. Ej: `static int Duplicar(
this int n) => n * 2;`

❓ **¿Cuándo usar una función local vs un método privado?**
Usa funciones locales cuando solo sean relevantes dentro de un método
específico y quieras encapsular lógica auxiliar. Usa métodos privados cuando
la funcionalidad sea reusable por otros métodos de la clase.

❓ **¿Qué son los delegados y por qué son útiles?**
Los delegados son tipos que representan referencias a métodos. Permiten pasar
funciones como parámetros, crear callbacks, eventos y programación funcional.
Func<T>, Action<T> y Predicate<T> son delegados predefinidos.

❓ **¿Cómo puedo retornar múltiples valores de un método?**
Usando tuplas: `(int, string) Metodo() => (42, "Hola");`. También con
parámetros out, o creando una clase/record contenedor.

❓ **¿Qué es un método puro?**
Un método cuyo resultado depende solo de sus parámetros (no de estado global)
y no produce efectos secundarios (no modifica estado externo). Ej: `Sumar(a,b)`.

❓ **¿Qué es la recursión de cola (tail recursion)?**
Es una recursión donde la llamada recursiva es la última operación del método.
C# no optimiza tail recursion automáticamente, a diferencia de F#.

❓ **¿Qué significa void en un método?**
void indica que el método no retorna ningún valor. El método ejecuta su código
pero no produce un resultado utilizable mediante return.

❓ **¿Puedo usar async/await en métodos void?**
Técnicamente sí, pero no es recomendable. Los métodos async void no pueden ser
esperados y los errores son difíciles de capturar. Solo úsalos para event
handlers. Prefiere async Task.
