1. INTRODUCCIÓN A SYSTEM.TEXT.JSON
----------------------------------------------------------------------

System.Text.Json es el serializador JSON incluido en .NET Core 3.0+ y
disponible para .NET Framework vía paquete NuGet.

Namespace: System.Text.Json

Clases principales:
  - JsonSerializer: métodos estáticos Serialize y Deserialize.
  - JsonSerializerOptions: configuración del serializador.
  - JsonDocument: parsea JSON sin mapear a objetos.
  - JsonElement: representa un valor JSON dentro de JsonDocument.
  - JsonNode: modelo JSON mutable (Nodo, Objeto, Arreglo, Valor).
  - JsonConverter<T>: conversores personalizados.

  ```csharp
  using System.Text.Json;
  using System.Text.Json.Serialization;
  ```

Ventajas sobre Newtonsoft.Json:
  - Mayor rendimiento (hasta 2x más rápido).
  - Menor asignación de memoria.
  - API nativa de .NET (sin dependencias externas).
  - Soporte para Span<T> y Utf8JsonReader/Writer.
  - Seguridad por defecto (no llama a setters no públicos).

2. JSONSERIALIZER.SERIALIZE
----------------------------------------------------------------------

Serializa un objeto C# a string JSON.

  ```csharp
  using System.Text.Json;

  var persona = new Persona
  {
      Nombre = "Ana García",
      Edad = 28,
      Activo = true,
      Salario = 45000.50m
  };

  string json = JsonSerializer.Serialize(persona);
  Console.WriteLine(json);
  // {"Nombre":"Ana García","Edad":28,"Activo":true,"Salario":45000.50}
  ```

Sobrecargas:
  ```csharp
  // A string
  string json = JsonSerializer.Serialize(obj);

  // A string con opciones
  string json = JsonSerializer.Serialize(obj, options);

  // A tipo específico
  string json = JsonSerializer.Serialize(obj, typeof(Persona));

  // A Stream (eficiente para archivos)
  using FileStream fs = File.Create("persona.json");
  await JsonSerializer.SerializeAsync(fs, persona);

  // A Stream con opciones
  await JsonSerializer.SerializeAsync(fs, persona, options);
  ```

Serialización a archivo:
  ```csharp
  var opciones = new JsonSerializerOptions { WriteIndented = true };
  string json = JsonSerializer.Serialize(persona, opciones);
  File.WriteAllText("persona.json", json);
  ```

Serialización asincrónica directa a archivo:
  ```csharp
  using FileStream fs = File.Create("personas.json");
  await JsonSerializer.SerializeAsync(fs, listaPersonas,
      new JsonSerializerOptions { WriteIndented = true });
  ```

3. JSONSERIALIZER.DESERIALIZE
----------------------------------------------------------------------

Deserializa string JSON a objeto C#.

  ```csharp
  string json = @"{
      ""Nombre"": ""Ana García"",
      ""Edad"": 28,
      ""Activo"": true,
      ""Salario"": 45000.50
  }";

  Persona? p = JsonSerializer.Deserialize<Persona>(json);
  Console.WriteLine($"Nombre: {p?.Nombre}, Edad: {p?.Edad}");
  ```

Sobrecargas:
  ```csharp
  // Desde string
  var obj = JsonSerializer.Deserialize<T>(json);
  var obj2 = JsonSerializer.Deserialize(json, typeof(T));

  // Desde Stream (eficiente para archivos grandes)
  using FileStream fs = File.OpenRead("personas.json");
  var personas = await JsonSerializer.DeserializeAsync<List<Persona>>(fs);

  // Desde ReadOnlySpan<byte>
  ReadOnlySpan<byte> utf8Json = File.ReadAllBytes("datos.json");
  var obj3 = JsonSerializer.Deserialize<Persona>(utf8Json);
  ```

Deserialización desde archivo:
  ```csharp
  string json = File.ReadAllText("config.json");
  Config? config = JsonSerializer.Deserialize<Config>(json);
  ```

Deserialización asincrónica:
  ```csharp
  using FileStream fs = File.OpenRead("productos.json");
  List<Producto>? productos = await JsonSerializer.DeserializeAsync<List<Producto>>(fs);
  ```

Manejo de nullables:
  ```csharp
  // Si el JSON puede ser null
  Persona? p = JsonSerializer.Deserialize<Persona>("null");

  // Si una propiedad podría faltar en el JSON
  public class Config
  {
      public string? Nombre { get; set; } // null si no está en JSON
      public int Puerto { get; set; } = 8080; // valor por defecto
  }
  ```

4. JSONSERIALIZEROPTIONS
----------------------------------------------------------------------

JsonSerializerOptions permite personalizar el comportamiento.

  ```csharp
  var options = new JsonSerializerOptions
  {
      WriteIndented = true,           // JSON formateado con saltos de línea
      PropertyNamingPolicy = JsonNamingPolicy.CamelCase, // camelCase
      PropertyNameCaseInsensitive = true, // Ignorar mayúsculas al deserializar
      IncludeFields = true,           // Incluir campos (no solo propiedades)
      DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull, // Ignorar null
      IgnoreReadOnlyProperties = true,// Ignorar propiedades de solo lectura
      Encoder = System.Text.Encodings.Web.JavaScriptEncoder.UnsafeRelaxedJsonEscaping
  };

  string json = JsonSerializer.Serialize(persona, options);
  ```

PropiedadNamingPolicy:
  ```csharp
  // Sin política: mantiene nombres originales
  // JsonNamingPolicy.CamelCase: "Nombre" → "nombre"
  // JsonNamingPolicy.KebabCaseLower: "Nombre" → "nombre" (no en .NET 6-)
  // JsonNamingPolicy.SnakeCaseLower: "Nombre" → "nombre" (a partir de .NET 8)
  ```

Ejemplo completo:
  ```csharp
  var options = new JsonSerializerOptions
  {
      PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
      WriteIndented = true,
      PropertyNameCaseInsensitive = true
  };

  var producto = new Producto
  {
      NombreProducto = "Laptop",
      PrecioUnitario = 1200.99m,
      EnStock = true
  };

  string json = JsonSerializer.Serialize(producto, options);
  Console.WriteLine(json);
  // {
  //   "nombreProducto": "Laptop",
  //   "precioUnitario": 1200.99,
  //   "enStock": true
  // }

  var deserializado = JsonSerializer.Deserialize<Producto>(json, options);
  ```

5. ATRIBUTOS DE SERIALIZACIÓN
----------------------------------------------------------------------

System.Text.Json proporciona atributos para controlar la serialización.

[JsonPropertyName]: Especifica el nombre en JSON.
  ```csharp
  public class Usuario
  {
      [JsonPropertyName("id")]
      public int Id { get; set; }

      [JsonPropertyName("nombre_completo")]
      public string NombreCompleto { get; set; }

      [JsonPropertyName("es_admin")]
      public bool EsAdmin { get; set; }
  }

  // Serializa: {"id":1,"nombre_completo":"Ana","es_admin":true}
  ```

[JsonIgnore]: Excluye una propiedad de la serialización.
  ```csharp
  public class Cuenta
  {
      public string Usuario { get; set; }

      [JsonIgnore]
      public string Contrasena { get; set; }

      [JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)]
      public string? Token { get; set; }
  }
  ```

[JsonInclude]: Incluye campos o propiedades privadas.
  ```csharp
  public class Persona
  {
      [JsonInclude]
      public string nombre; // campo público

      [JsonInclude]
      internal string _secreto; // campo internal
  }
  ```

[JsonConverter]: Especifica un conversor personalizado.
  ```csharp
  [JsonConverter(typeof(DateTimeConverter))]
  public class Evento
  {
      public string Nombre { get; set; }
  }
  ```

[JsonExtensionData]: Captura propiedades adicionales del JSON.
  ```csharp
  public class Producto
  {
      public string Nombre { get; set; }

      [JsonExtensionData]
      public Dictionary<string, JsonElement>? DatosExtra { get; set; }
  }

  // JSON: {"nombre":"Laptop","color":"rojo","peso":2.5}
  // DatosExtra contendrá "color" y "peso"
  ```

[JsonRequired]: La propiedad debe estar presente en JSON.
  ```csharp
  public class Pedido
  {
      [JsonRequired]
      public int Id { get; set; }

      public string? Observaciones { get; set; }
  }
  // Lanza JsonException si "id" falta en el JSON
  ```

7. SERIALIZACIÓN DE OBJETOS ANIDADOS
----------------------------------------------------------------------

Objetos complejos se serializan naturalmente:
  ```csharp
  public class Empresa
  {
      public string Nombre { get; set; }
      public Direccion Direccion { get; set; }
      public List<Empleado> Empleados { get; set; }
  }

  public class Direccion
  {
      public string Calle { get; set; }
      public string Ciudad { get; set; }
  }

  public class Empleado
  {
      public string Nombre { get; set; }
      public string Puesto { get; set; }
  }

  var empresa = new Empresa
  {
      Nombre = "TechCorp",
      Direccion = new Direccion { Calle = "Av. Principal", Ciudad = "Madrid" },
      Empleados = new List<Empleado>
      {
          new() { Nombre = "Ana", Puesto = "Dev" },
          new() { Nombre = "Luis", Puesto = "QA" }
      }
  };

  string json = JsonSerializer.Serialize(empresa, new JsonSerializerOptions
  {
      WriteIndented = true
  });
  // {
  //   "Nombre": "TechCorp",
  //   "Direccion": { "Calle": "Av. Principal", "Ciudad": "Madrid" },
  //   "Empleados": [
  //     { "Nombre": "Ana", "Puesto": "Dev" },
  //     { "Nombre": "Luis", "Puesto": "QA" }
  //   ]
  // }
  ```

Referencias circulares: Lanza JsonException. Usa ReferenceHandler para
manejarlas:
  ```csharp
  var options = new JsonSerializerOptions
  {
      ReferenceHandler = ReferenceHandler.IgnoreCycles // Ignora ciclos
  };
  ```

8. JSONDOCUMENT Y JSONELEMENT (JSON DINÁMICO)
----------------------------------------------------------------------

Cuando no tienes una clase destino, usa JsonDocument para parsear JSON
dinámicamente.

  ```csharp
  string json = @"{
      ""nombre"": ""Ana"",
      ""edad"": 28,
      ""direccion"": { ""ciudad"": ""Madrid"" },
      ""hobbies"": [""leer"", ""correr""]
  }";

  using JsonDocument doc = JsonDocument.Parse(json);
  JsonElement root = doc.RootElement;

  // Acceder a propiedades
  string nombre = root.GetProperty("nombre").GetString();
  int edad = root.GetProperty("edad").GetInt32();
  string ciudad = root.GetProperty("direccion").GetProperty("ciudad").GetString();

  // Array
  JsonElement hobbies = root.GetProperty("hobbies");
  foreach (JsonElement hobby in hobbies.EnumerateArray())
      Console.WriteLine(hobby.GetString());

  // Verificar tipo
  if (root.TryGetProperty("telefono", out JsonElement telefono))
      Console.WriteLine($"Teléfono: {telefono}");
  ```

Métodos de JsonElement:
  - GetString(), GetInt32(), GetInt64(), GetDouble(), GetDecimal()
  - GetBoolean(), GetGuid(), GetDateTime()
  - GetRawText(): devuelve el JSON original para este elemento
  - ValueKind: enum (Object, Array, String, Number, True, False, Null)
  - EnumerateObject(), EnumerateArray()
  - TryGetProperty(), TryGetValue()

  ```csharp
  JsonElement elem = root.GetProperty("edad");

  switch (elem.ValueKind)
  {
      case JsonValueKind.String:
          Console.WriteLine($"String: {elem.GetString()}");
          break;
      case JsonValueKind.Number:
          Console.WriteLine($"Number: {elem.GetInt32()}");
          break;
      case JsonValueKind.True:
      case JsonValueKind.False:
          Console.WriteLine($"Bool: {elem.GetBoolean()}");
          break;
      case JsonValueKind.Null:
          Console.WriteLine("Null");
          break;
  }
  ```

JsonNode (JSON mutable, .NET 6+):
  ```csharp
  using System.Text.Json.Nodes;

  // Crear desde cero
  var node = new JsonObject
  {
      ["nombre"] = "Ana",
      ["edad"] = 28,
      ["direccion"] = new JsonObject
      {
          ["ciudad"] = "Madrid"
      }
  };
  node["edad"] = 29; // Modificar
  node["activo"] = true; // Añadir
  string json = node.ToJsonString(new JsonSerializerOptions { WriteIndented = true });

  // Parsear desde string
  JsonNode? parsed = JsonNode.Parse(json);
  string nombre = parsed?["nombre"]?.GetValue<string>();
  ```

9. CONVERSORES PERSONALIZADOS
----------------------------------------------------------------------

Implementa JsonConverter<T> para controlar la serialización de tipos
complejos o no soportados.

  ```csharp
  public class DateOnlyJsonConverter : JsonConverter<DateOnly>
  {
      private const string Formato = "yyyy-MM-dd";

      public override DateOnly Read(ref Utf8JsonReader reader,
          Type typeToConvert, JsonSerializerOptions options)
      {
          return DateOnly.Parse(reader.GetString()!);
      }

      public override void Write(Utf8JsonWriter writer,
          DateOnly value, JsonSerializerOptions options)
      {
          writer.WriteStringValue(value.ToString(Formato));
      }
  }

  // Uso
  var options = new JsonSerializerOptions();
  options.Converters.Add(new DateOnlyJsonConverter());

  string json = JsonSerializer.Serialize(new { Fecha = DateOnly.FromDateTime(DateTime.Now) }, options);
  ```

Conversor para enum como string:
  ```csharp
  public class EnumStringConverter<T> : JsonConverter<T> where T : struct, Enum
  {
      public override T Read(ref Utf8JsonReader reader,
          Type typeToConvert, JsonSerializerOptions options)
      {
          return Enum.Parse<T>(reader.GetString()!, ignoreCase: true);
      }

      public override void Write(Utf8JsonWriter writer,
          T value, JsonSerializerOptions options)
      {
          writer.WriteStringValue(value.ToString());
      }
  }
  ```

Conversor con JsonConverterAttribute:
  ```csharp
  [JsonConverter(typeof(EnumStringConverter<Color>))]
  public enum Color { Rojo, Verde, Azul }
  ```

10. POLIMORFISMO Y HERENCIA
----------------------------------------------------------------------

Serialización polimórfica con atributos:
  ```csharp
  [JsonDerivedType(typeof(Gato), "gato")]
  [JsonDerivedType(typeof(Perro), "perro")]
  public class Animal
  {
      public string Nombre { get; set; }
  }

  public class Gato : Animal
  {
      public bool LeGustaLaHierba { get; set; }
  }

  public class Perro : Animal
  {
      public bool SabeLadrar { get; set; }
  }

  // Serialización
  Animal miMascota = new Gato { Nombre = "Garfield", LeGustaLaHierba = true };
  string json = JsonSerializer.Serialize(miMascota);
  // {"$type":"gato","Nombre":"Garfield","LeGustaLaHierba":true}

  // Deserialización
  var animal = JsonSerializer.Deserialize<Animal>(json);
  Console.WriteLine(animal is Gato); // true
  ```

Polimorfismo con JsonPolymorphismOptions:
  ```csharp
  var options = new JsonSerializerOptions
  {
      PolymorphismOptions = new JsonPolymorphismOptions
      {
          TypeDiscriminatorPropertyName = "$tipo",
          IgnoreUnrecognizedTypeDiscriminators = true,
          UnknownDerivedTypeHandling = JsonUnknownDerivedTypeHandling.FailSerialization
      }
  };
  ```

11. RENDIMIENTO Y BUENAS PRÁCTICAS
----------------------------------------------------------------------

a) Cachea JsonSerializerOptions: crear opciones es costoso. Crea una
   instancia estática o singleton.

  ```csharp
  private static readonly JsonSerializerOptions _jsonOptions = new()
  {
      PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
      WriteIndented = false
  };
  ```

b) Usa serialización asincrónica con streams para archivos grandes.

c) Para API REST, usa JsonSerializerOptions global en ASP.NET Core.

d) Para rendimiento máximo, usa Utf8JsonReader y Utf8JsonWriter.

e) No serialices propiedades que puedan exponer datos sensibles; usa
   [JsonIgnore].

f) Usa [JsonPropertyName] para nombres de propiedades no convencionales.

g) Prefiere clases selladas para DTOs de serialización.

h) Usa JsonSerializerDefaults.Web para opciones típicas de API web:
  ```csharp
  var options = new JsonSerializerOptions(JsonSerializerDefaults.Web);
  // camelCase, case-insensitive, etc.
  ```

          public DateTime FechaRegistro { get; set; }
      }

      public class Producto
      {
          [JsonPropertyName("id")]
          public int Id { get; set; }

          [JsonPropertyName("nombre_producto")]
          public string NombreProducto { get; set; }

          [JsonPropertyName("precio")]
          public decimal Precio { get; set; }

          [JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)]
          public string? Categoria { get; set; }

          [JsonIgnore]
          public string CodigoInterno { get; set; }
      }

      public class Cuenta
      {
          public string Usuario { get; set; }
          public string Email { get; set; }

          [JsonIgnore]
          public string Contrasena { get; set; }
      }

      public class ProductoConExtra
      {
          public string Nombre { get; set; }

          [JsonExtensionData]
          public Dictionary<string, JsonElement>? Extra { get; set; }
      }

======================================================================
  PREGUNTAS FRECUENTES (FAQ)
======================================================================

P: ¿System.Text.Json reemplaza a Newtonsoft.Json?
R: Sí, es el serializador oficial de .NET. Newtonsoft.Json sigue siendo
   válido, pero System.Text.Json es más rápido y nativo.

P: ¿Cómo manejo fechas en un formato específico?
R: Usa un JsonConverter personalizado. Por defecto usa ISO 8601.

P: ¿Por qué mi propiedad no se serializa?
R: Debe tener getter público. Si es un campo, usa [JsonInclude]
   o options.IncludeFields = true.

P: ¿Cómo ignoro propiedades null al serializar?
R: options.DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull.

P: ¿System.Text.Json soporta JSON5 o comentarios?
R: Sí, con JsonDocumentOptions o JsonSerializerOptions:
   options.ReadCommentHandling = JsonCommentHandling.Skip.

  DESAFÍO DEL ARQUITECTO

Implementa un "Motor de Serialización JSON Universal" que:

  - Serialice cualquier objeto a JSON con control granular:
    - Incluir/excluir propiedades por nombre o tipo.
    - Transformar nombres (camelCase, snake_case, kebab-case).
    - Controlar profundidad máxima.
    - Enmascarar valores sensibles (contraseñas, tokens).
  - Deserialice JSON a objetos con validación:
    - Tipos requeridos, rangos numéricos, longitudes de string.
    - Valores por defecto para propiedades faltantes.
    - Reporte de errores detallado (qué propiedad falló y por qué).
  - Use expresiones lambda para configurar reglas:

  ```csharp
  var config = new SerializationConfig<Persona>()
      .Include(p => p.Nombre)
      .Include(p => p.Edad, max: 120)
      .Exclude(p => p.Salario)
      .TransformNames(NamingPolicy.SnakeCase)
      .MaskSensitive(p => p.Contrasena, "***");
  ```

Pistas:
  - Usa Expression<Func<T, object?>> para las reglas.
  - JsonSerializerOptions con JsonConverter personalizado.
  - Dictionary<string, JsonConverter> para mapeo de propiedades.
  - Reflection para construir el plan de serialización.
  - IValidatableObject o validación personalizada.

Criterios de éxito:
  - Configuración declarativa y fluida.
  - Validación en deserialización con mensajes de error claros.
  - Enmascaramiento de datos sensibles.
  - Soporte para herencia y polimorfismo.
  - Buen rendimiento (cachear el plan de serialización).
