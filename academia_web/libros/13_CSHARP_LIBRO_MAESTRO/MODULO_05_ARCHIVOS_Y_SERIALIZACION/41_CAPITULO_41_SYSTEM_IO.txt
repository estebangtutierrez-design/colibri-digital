1. INTRODUCCIÓN A SYSTEM.IO
----------------------------------------------------------------------

System.IO es el espacio de nombres base en .NET para todas las operaciones de
entrada y salida. Proporciona clases para trabajar con archivos, streams,
lectores, escritores y directorios.

Jerarquía principal:
  - Stream (abstracta): FileStream, MemoryStream, BufferedStream
  - TextReader/TextWriter: StreamReader, StreamWriter, StringReader
  - BinaryReader/BinaryWriter
  - File, FileInfo, Directory, DirectoryInfo (utilidades estáticas/instancia)

La clase base Stream define operaciones como Read, Write, Seek, Flush, Close.
FileStream es la implementación concreta para archivos del sistema operativo.

----------------------------------------------------------------------
2. LA CLASE FILESTREAM
----------------------------------------------------------------------

FileStream permite leer y escribir bytes desde y hacia archivos físicos.
Opera a nivel de bytes, lo que significa que es la capa más cercana al
sistema operativo. Cualquier operación de más alto nivel (StreamReader,
BinaryReader, etc.) termina utilizando un FileStream internamente.

Constructores principales:
  - FileStream(string path, FileMode mode)
  - FileStream(string path, FileMode mode, FileAccess access)
  - FileStream(string path, FileMode mode, FileAccess access, FileShare share)
  - FileStream(string path, FileMode mode, FileAccess access, FileShare share, int bufferSize)

Parámetros clave:
  - path: Ruta absoluta o relativa al archivo.
  - mode: Cómo abrir o crear el archivo (FileMode).
  - access: Tipo de operaciones permitidas (FileAccess).
  - share: Nivel de acceso concurrente (FileShare).
  - bufferSize: Tamaño del búfer interno (por defecto 4096 bytes).

Ejemplo básico:
  ```csharp
  using FileStream fs = new FileStream("datos.bin", FileMode.OpenOrCreate);
  // Trabajar con el stream...
  ```

----------------------------------------------------------------------
3. FILEMODE: CÓMO ABRIR O CREAR ARCHIVOS
----------------------------------------------------------------------

FileMode es una enumeración que especifica cómo el sistema operativo debe
abrir un archivo.

Valores de FileMode:
  - CreateNew: Crea un nuevo archivo. Lanza IOException si ya existe.
  - Create: Crea un nuevo archivo. Si existe, lo sobrescribe.
  - Open: Abre un archivo existente. Lanza FileNotFoundException si no existe.
  - OpenOrCreate: Abre si existe, o crea uno nuevo si no.
  - Truncate: Abre un archivo existente y lo trunca a cero bytes.
  - Append: Abre el archivo para añadir al final. Crea si no existe.

  ```csharp
  using FileStream fs1 = new FileStream("nuevo.txt", FileMode.CreateNew);
  using FileStream fs2 = new FileStream("existente.txt", FileMode.Open);
  using FileStream fs3 = new FileStream("log.txt", FileMode.Append);
  ```

----------------------------------------------------------------------
4. FILEACCESS: PERMISOS DE LECTURA Y ESCRITURA
----------------------------------------------------------------------

FileAccess controla qué tipo de operaciones se permiten en el FileStream.

Valores:
  - Read: Solo lectura.
  - Write: Solo escritura.
  - ReadWrite: Lectura y escritura.

Especificar el acceso correcto mejora la seguridad y el rendimiento. Si solo
necesitas leer, no solicites permisos de escritura.

  ```csharp
  using FileStream fsRead = new FileStream("lectura.txt", FileMode.Open, FileAccess.Read);
  using FileStream fsWrite = new FileStream("salida.txt", FileMode.Create, FileAccess.Write);
  using FileStream fsReadWrite = new FileStream("datos.dat", FileMode.OpenOrCreate, FileAccess.ReadWrite);
  ```

Combinaciones comunes:
  - FileMode.Open + FileAccess.Read: Lectura de archivo existente.
  - FileMode.Create + FileAccess.Write: Escritura de archivo nuevo.
  - FileMode.OpenOrCreate + FileAccess.ReadWrite: Lectura/escritura flexible.

----------------------------------------------------------------------
5. FILESHARE: CONTROL DE ACCESO CONCURRENTE
----------------------------------------------------------------------

FileShare define el nivel de acceso que otros procesos o hilos tienen sobre
el archivo mientras tú lo estás usando.

Valores:
  - None: Acceso exclusivo. Otros no pueden abrir el archivo.
  - Read: Otros pueden leer pero no escribir.
  - Write: Otros pueden escribir pero no leer.
  - ReadWrite: Otros pueden leer y escribir.
  - Delete: Otros pueden eliminar el archivo.
  - Inheritable: El identificador del archivo puede ser heredado por procesos hijo.

  ```csharp
  using FileStream fs = new FileStream("compartido.txt", FileMode.OpenOrCreate,
      FileAccess.Read, FileShare.ReadWrite);
  ```

Para archivos de log donde múltiples procesos escriben, usa FileShare.ReadWrite.
Para archivos de configuración, FileShare.Read es suficiente.

----------------------------------------------------------------------
6. IDISPOSABLE Y EL BLOQUE USING
----------------------------------------------------------------------

FileStream implementa IDisposable. Es crítico liberar recursos no administrados
(identificadores de archivo del SO) tan pronto como sea posible.

Mala práctica (fuga de recursos):
  ```csharp
  FileStream fs = new FileStream("datos.txt", FileMode.Open);
  // Si ocurre una excepción aquí, el archivo queda bloqueado.
  fs.Close();
  ```

Buena práctica con try/finally:
  ```csharp
  FileStream fs = null;
  try
  {
      fs = new FileStream("datos.txt", FileMode.Open);
      // Operaciones
  }
  finally
  {
      fs?.Close();
  }
  ```

Mejor práctica con using:
  ```csharp
  using (FileStream fs = new FileStream("datos.txt", FileMode.Open))
  {
      // Operaciones - se libera automáticamente al salir del bloque
  }
  ```

C# 8+: Declaración using simplificada:
  ```csharp
  using FileStream fs = new FileStream("datos.txt", FileMode.Open);
  // Operaciones - se libera al salir del ámbito
  ```

El bloque using garantiza que Dispose() se llame incluso si ocurre una excepción.
Internamente, el compilador genera un bloque try/finally.

----------------------------------------------------------------------
7. BUFFERING Y RENDIMIENTO
----------------------------------------------------------------------

FileStream tiene un búfer interno para reducir las llamadas al sistema
operativo. El tamaño predeterminado es 4096 bytes, pero puedes cambiarlo.

  ```csharp
  // Búfer más grande para archivos grandes
  using FileStream fs = new FileStream("grande.dat", FileMode.Open,
      FileAccess.Read, FileShare.Read, 65536); // 64 KB
  ```

Consejos de rendimiento:
  - Use búfer grande (64-256 KB) para archivos grandes (>100 MB).
  - Para archivos pequeños (<1 MB), 4096 bytes es suficiente.
  - Para escritura secuencial, FileOptions.SequentialScan optimiza el caché.
  - Para acceso aleatorio, FileOptions.RandomAccess.

  ```csharp
  using FileStream fs = new FileStream("video.mp4", FileMode.Open,
      FileAccess.Read, FileShare.Read, 65536, FileOptions.SequentialScan);
  ```

----------------------------------------------------------------------
8. MANEJO DE EXCEPCIONES EN OPERACIONES DE ARCHIVO
----------------------------------------------------------------------

Las operaciones de archivo son propensas a errores externos. Siempre debes
manejar las excepciones apropiadas.

Excepciones comunes:
  - FileNotFoundException: El archivo no existe.
  - DirectoryNotFoundException: El directorio no existe.
  - UnauthorizedAccessException: Sin permisos.
  - IOException: Error general de I/O.
  - PathTooLongException: Ruta demasiado larga.
  - SecurityException: Restricciones de seguridad.

  ```csharp
  try
  {
      using FileStream fs = new FileStream("datos.txt", FileMode.Open);
      // Operaciones
  }
  catch (FileNotFoundException ex)
  {
      Console.WriteLine($"Archivo no encontrado: {ex.FileName}");
  }
  catch (UnauthorizedAccessException)
  {
      Console.WriteLine("Sin permisos para acceder al archivo.");
  }
  catch (IOException ex)
  {
      Console.WriteLine($"Error de E/S: {ex.Message}");
  }
  ```

----------------------------------------------------------------------
9. COMPARATIVA: STREAM VS READER/WRITER
----------------------------------------------------------------------

Elegir la herramienta correcta:
  - FileStream: Bytes crudos. Lectura/escritura binaria, posicionamiento.
  - StreamReader/StreamWriter: Texto codificado (UTF-8, ASCII, etc.).
  - BinaryReader/BinaryWriter: Tipos primitivos (int, double, etc.).
  - File.ReadAllText/WriteAllText: Conveniencia para archivos pequeños.

Regla general:
  - Bytes → FileStream
  - Texto → StreamReader/StreamWriter
  - Primitivos → BinaryReader/BinaryWriter
  - Archivos pequeños → Métodos estáticos de File

10. BUENAS PRÁCTICAS
----------------------------------------------------------------------

  a) Siempre usa using o using declaration para liberar recursos.
  b) Especifica FileAccess y FileShare explícitamente.
  c) Maneja excepciones específicas, no solo Exception genérica.
  d) Usa FileMode.Append para logs en lugar de Open + Seek.
  e) No asumas que una ruta existe o es accesible.
  f) Para archivos grandes, usa streaming, no cargues todo en memoria.
  g) Usa bufferSize adecuado para el tipo de operación.
  h) Considera FileOptions.SequentialScan para lectura lineal.

  CÓDIGO DE EJEMPLO COMPLETO
================================================================================

  ```csharp
  using System;
  using System.IO;
  using System.Text;

  namespace Capitulo41.FileStreamDemo
  {
      class Program
      {
          static void Main(string[] args)
          {
              string filePath = "demo_filestream.bin";

              // Demostración 1: FileMode y FileAccess
              Console.WriteLine("=== DEMO 1: CREACIÓN Y ESCRITURA BÁSICA ===");
              EscribirBytes(filePath);
              LeerBytes(filePath);

              // Demostración 2: Append
              Console.WriteLine("\n=== DEMO 2: APPEND ===");
              AnadirBytes(filePath);
              LeerBytes(filePath);

              // Demostración 3: FileShare concurrente
              Console.WriteLine("\n=== DEMO 3: ACCESO COMPARTIDO ===");

              // Demostración 4: Posicionamiento (Seek)
              Console.WriteLine("\n=== DEMO 4: SEEK ===");
              LeerEnPosicion(filePath);

              // Demostración 5: Buffer personalizado
              Console.WriteLine("\n=== DEMO 5: BUFFER GRANDE ===");
              CopiarConBufferGrande(filePath, "copia_buffer.bin");

              // Limpieza
              if (File.Exists(filePath)) File.Delete(filePath);
              if (File.Exists("copia_buffer.bin")) File.Delete("copia_buffer.bin");

              Console.WriteLine("\n¡Todos los ejemplos completados!");
          }

          static void EscribirBytes(string path)
          {
              byte[] datos = Encoding.UTF8.GetBytes("Hola, System.IO!");
              using FileStream fs = new FileStream(path, FileMode.Create,
                  FileAccess.Write, FileShare.None);
              fs.Write(datos, 0, datos.Length);
              Console.WriteLine($"Escritos {datos.Length} bytes.");
          }

          static void LeerBytes(string path)
          {
              using FileStream fs = new FileStream(path, FileMode.Open,
                  FileAccess.Read, FileShare.Read);
              byte[] buffer = new byte[fs.Length];
              int bytesLeidos = fs.Read(buffer, 0, buffer.Length);
              string texto = Encoding.UTF8.GetString(buffer, 0, bytesLeidos);
              Console.WriteLine($"Leídos {bytesLeidos} bytes: {texto}");
          }

          static void AnadirBytes(string path)
          {
              byte[] datos = Encoding.UTF8.GetBytes(" [Más datos añadidos]");
              using FileStream fs = new FileStream(path, FileMode.Append,
                  FileAccess.Write, FileShare.Read);
              fs.Write(datos, 0, datos.Length);
              Console.WriteLine($"Añadidos {datos.Length} bytes al final.");
          }

          static void LeerEnPosicion(string path)
          {
              using FileStream fs = new FileStream(path, FileMode.Open,
                  FileAccess.Read, FileShare.Read);
              // Leemos desde la posición 5
              fs.Seek(5, SeekOrigin.Begin);
              byte[] buffer = new byte[10];
              int bytesLeidos = fs.Read(buffer, 0, buffer.Length);
              string texto = Encoding.UTF8.GetString(buffer, 0, bytesLeidos);
              Console.WriteLine($"Desde posición 5: {texto} (posición actual: {fs.Position})");
          }

          static void CopiarConBufferGrande(string origen, string destino)
          {
              using FileStream fsOrigen = new FileStream(origen, FileMode.Open,
                  FileAccess.Read, FileShare.Read, 65536, FileOptions.SequentialScan);
              using FileStream fsDestino = new FileStream(destino, FileMode.Create,
                  FileAccess.Write, FileShare.None, 65536);
              byte[] buffer = new byte[8192];
              int bytesLeidos;
              while ((bytesLeidos = fsOrigen.Read(buffer, 0, buffer.Length)) > 0)
              {
                  fsDestino.Write(buffer, 0, bytesLeidos);
              }
              Console.WriteLine("Archivo copiado con buffer de 64 KB.");
          }
      }
  }
  ```

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Cuál es la diferencia entre FileStream y StreamReader?
R: FileStream opera a nivel de bytes (binario), mientras que StreamReader
   opera a nivel de caracteres (texto) aplicando una codificación.

P: ¿Qué pasa si no uso using en un FileStream?
R: El archivo quedará bloqueado hasta que el GC finalice el objeto, lo que
   puede causar agotamiento de recursos y archivos inaccesibles.

P: ¿FileMode.Append permite leer?
R: No. Append solo permite escritura. Para leer y añadir, usa
   FileMode.OpenOrCreate + FileAccess.ReadWrite.

P: ¿Puedo usar FileStream con archivos de red?
R: Sí, siempre que la ruta UNC sea accesible y tengas permisos.

P: ¿Cuándo debo usar FileOptions.Asynchronous?
R: Para operaciones de I/O asíncronas con ReadAsync/WriteAsync, típicamente
   en aplicaciones GUI o servidores que necesitan escalar.

11. EJEMPLO AVANZADO: GESTOR DE ARCHIVOS CON FILESTREAM
----------------------------------------------------------------------

A continuación, un ejemplo más avanzado que implementa un gestor simple
de archivos con copia, split y verificación.

  ```csharp
  using System;
  using System.IO;
  using System.Security.Cryptography;
  using System.Text;

  public class GestorArchivos
  {
      public static void CopiarConProgreso(string origen, string destino)
      {
          using FileStream fsOrigen = new FileStream(origen, FileMode.Open,
              FileAccess.Read, FileShare.Read, 65536, FileOptions.SequentialScan);
          using FileStream fsDestino = new FileStream(destino, FileMode.Create,
              FileAccess.Write, FileShare.None, 65536);

          byte[] buffer = new byte[81920]; // 80 KB
          int bytesLeidos;
          long totalBytes = fsOrigen.Length;
          long progreso = 0;

          while ((bytesLeidos = fsOrigen.Read(buffer, 0, buffer.Length)) > 0)
          {
              fsDestino.Write(buffer, 0, bytesLeidos);
              progreso += bytesLeidos;
              double porcentaje = (double)progreso / totalBytes * 100;
              Console.Write($"\rProgreso: {porcentaje:F1}% ({progreso}/{totalBytes} bytes)");
          }
          Console.WriteLine("\nCopia completada.");
      }

      public static void SplitArchivo(string origen, int parteTamanoMB)
      {
          int parteTamano = parteTamanoMB * 1024 * 1024;
          using FileStream fsOrigen = new FileStream(origen, FileMode.Open,
              FileAccess.Read, FileShare.Read);

          byte[] buffer = new byte[81920];
          int parteNum = 0;

          while (fsOrigen.Position < fsOrigen.Length)
          {
              string partePath = $"{origen}.part{parteNum:D3}";
              int bytesRestantes = (int)Math.Min(parteTamano, fsOrigen.Length - fsOrigen.Position);

              using FileStream fsParte = new FileStream(partePath, FileMode.Create,
                  FileAccess.Write, FileShare.None, 65536);
              int bytesAEscribir = bytesRestantes;
              int bytesLeidos;

              while (bytesAEscribir > 0 &&
                     (bytesLeidos = fsOrigen.Read(buffer, 0,
                         Math.Min(buffer.Length, bytesAEscribir))) > 0)
              {
                  fsParte.Write(buffer, 0, bytesLeidos);
                  bytesAEscribir -= bytesLeidos;
              }

              Console.WriteLine($"Parte {parteNum}: {partePath} ({bytesRestantes} bytes)");
              parteNum++;
          }
      }

      public static string CalcularSHA256(string path)
      {
          using FileStream fs = new FileStream(path, FileMode.Open,
              FileAccess.Read, FileShare.Read, 65536, FileOptions.SequentialScan);
          using SHA256 sha256 = SHA256.Create();
          byte[] hash = sha256.ComputeHash(fs);
          return BitConverter.ToString(hash).Replace("-", "").ToLower();
      }

      public static void Main(string[] args)
      {
          Console.WriteLine("=== GESTOR DE ARCHIVOS CON FILESTREAM ===");
          Console.WriteLine($"SHA256 de este ejecutable: {CalcularSHA256(typeof(GestorArchivos).Assembly.Location)}");
          Console.WriteLine("Listo para usar los métodos de la clase.");
      }
  }
  ```

12. FILESTREAM ASÍNCRONO PARA APLICACIONES ESCALABLES
----------------------------------------------------------------------

Las operaciones asincrónicas con FileStream mejoran la escalabilidad.

  ```csharp
  public static async Task CopiarAsync(string origen, string destino)
  {
      await using FileStream fsOrigen = new FileStream(origen, FileMode.Open,
          FileAccess.Read, FileShare.Read, 65536,
          FileOptions.Asynchronous | FileOptions.SequentialScan);
      await using FileStream fsDestino = new FileStream(destino, FileMode.Create,
          FileAccess.Write, FileShare.None, 65536,
          FileOptions.Asynchronous);

      byte[] buffer = new byte[81920];
      int bytesLeidos;

      while ((bytesLeidos = await fsOrigen.ReadAsync(buffer, 0, buffer.Length)) > 0)
      {
          await fsDestino.WriteAsync(buffer, 0, bytesLeidos);
      }

      Console.WriteLine("Copia asincrónica completada.");
  }

  public static async Task<byte[]> LeerArchivoAsync(string path)
  {
      await using FileStream fs = new FileStream(path, FileMode.Open,
          FileAccess.Read, FileShare.Read, 4096, FileOptions.Asynchronous);
      byte[] buffer = new byte[fs.Length];
      await fs.ReadAsync(buffer, 0, buffer.Length);
      return buffer;
  }

  public static async Task EscribirArchivoAsync(string path, byte[] datos)
  {
      await using FileStream fs = new FileStream(path, FileMode.Create,
          FileAccess.Write, FileShare.None, 4096, FileOptions.Asynchronous);
      await fs.WriteAsync(datos, 0, datos.Length);
  }
  ```

El uso de FileOptions.Asynchronous permite que el hilo principal no se
bloquee mientras el sistema operativo realiza la operación de I/O.

13. COMPARATIVA COMPLETA DE MODOS Y ACCESOS
----------------------------------------------------------------------

Matriz de compatibilidad de combinaciones FileMode + FileAccess:

  FileMode        | Read  | Write | ReadWrite
  ----------------|-------|-------|----------
  CreateNew       | No    | Sí    | No
  Create          | No    | Sí    | No
  Open            | Sí    | Sí    | Sí
  OpenOrCreate    | Sí    | Sí    | Sí
  Truncate        | No    | Sí    | No
  Append          | No    | Sí    | No

Ejemplos prácticos de cada combinación:

  ```csharp
  // Solo lectura de archivo existente
  using var fs1 = new FileStream("datos.txt", FileMode.Open, FileAccess.Read);

  // Crear nuevo o sobrescribir para escritura
  using var fs2 = new FileStream("salida.txt", FileMode.Create, FileAccess.Write);

  // Añadir al final (logs)
  using var fs3 = new FileStream("log.txt", FileMode.Append, FileAccess.Write);

  // Lectura y escritura en archivo existente
  using var fs4 = new FileStream("datos.dat", FileMode.OpenOrCreate, FileAccess.ReadWrite);
  ```

Combinaciones FileShare recomendadas:

  Escenario                        | FileShare
  ---------------------------------|-------------------
  Logs de aplicación               | ReadWrite
  Archivos de configuración        | Read
  Archivos temporales              | None
  Lectura concurrente              | Read
  Procesamiento batch exclusivo    | None
  Archivos compartidos en red      | ReadWrite
  Backup durante operación         | Read
  Datos críticos (transacciones)   | None
