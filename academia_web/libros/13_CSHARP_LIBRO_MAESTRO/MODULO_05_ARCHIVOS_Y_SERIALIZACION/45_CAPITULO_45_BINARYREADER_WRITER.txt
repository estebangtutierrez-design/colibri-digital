1. INTRODUCCIÓN A BINARYREADER Y BINARYWRITER
----------------------------------------------------------------------

BinaryReader y BinaryWriter son clases que envuelven un Stream y proporcionan
métodos para leer/escribir tipos de datos de .NET en formato binario.

Características:
  - Escriben/leen tipos primitivos en formato compacto.
  - Strings se escriben con prefijo de longitud (7-bit encoded).
  - Usan Little-Endian por defecto (como x86/x64).
  - No realizan ninguna codificación de caracteres (es binario puro).

Jerarquía:
  - BinaryWriter : IDisposable
  - BinaryReader : IDisposable

  ```csharp
  using FileStream fs = new FileStream("datos.bin", FileMode.Create);
  using BinaryWriter bw = new BinaryWriter(fs);
  // Escribir datos...
  ```

----------------------------------------------------------------------
2. BINARYWRITER: ESCRITURA DE TIPOS PRIMITIVOS
----------------------------------------------------------------------

BinaryWriter tiene métodos Write sobrecargados para cada tipo primitivo:

  ```csharp
  using BinaryWriter bw = new BinaryWriter(File.Open("tipos.bin", FileMode.Create));

  // Tipos enteros
  bw.Write((byte)255);        // 1 byte
  bw.Write((sbyte)-128);      // 1 byte
  bw.Write((short)-32768);    // 2 bytes (Int16)
  bw.Write((ushort)65535);    // 2 bytes (UInt16)
  bw.Write(42);               // 4 bytes (Int32)
  bw.Write(42U);              // 4 bytes (UInt32)
  bw.Write(123456789L);       // 8 bytes (Int64)
  bw.Write(123456789UL);      // 8 bytes (UInt64)

  // Tipos de punto flotante
  bw.Write(3.14159f);         // 4 bytes (Single)
  bw.Write(3.14159265358979); // 8 bytes (Double)
  bw.Write(123.456m);         // 16 bytes (Decimal)

  // Otros tipos
  bw.Write(true);             // 1 byte (Boolean)
  bw.Write('A');              // 2 bytes (Char, UTF-16)
  bw.Write("Hola binario");   // Longitud prefijo + UTF-8

  bw.Close();
  ```

Cada tipo tiene un tamaño fijo en bytes:
  - bool: 1 byte
  - byte/sbyte: 1 byte
  - char: 2 bytes (UTF-16)
  - short/ushort: 2 bytes
  - int/uint: 4 bytes
  - float: 4 bytes
  - long/ulong: 8 bytes
  - double: 8 bytes
  - decimal: 16 bytes
  - string: variable (prefijo de longitud + UTF-8)

----------------------------------------------------------------------
3. BINARYWRITER: CONSTRUCTORES Y OPCIONES
----------------------------------------------------------------------

Constructores:
  1. BinaryWriter(Stream output)
  2. BinaryWriter(Stream output, Encoding encoding) // encoding para strings
  3. BinaryWriter(Stream output, Encoding encoding, bool leaveOpen)

  ```csharp
  // Constructor básico
  using FileStream fs = new FileStream("datos.bin", FileMode.Create);
  using BinaryWriter bw = new BinaryWriter(fs);

  // Con encoding específico para strings
  using BinaryWriter bw2 = new BinaryWriter(fs, Encoding.UTF8);

  // Con leaveOpen: no cerrar el stream al disponer el writer
  using BinaryWriter bw3 = new BinaryWriter(fs, Encoding.UTF8, leaveOpen: true);
  // fs sigue abierto aquí
  ```

Propiedades:
  - BaseStream: el stream subyacente.
  - Encoding: encoding usado para strings (solo lectura).

  ```csharp
  using BinaryWriter bw = new BinaryWriter(fs);
  Console.WriteLine($"Stream subyacente: {bw.BaseStream.GetType().Name}");
  Console.WriteLine($"Encoding para strings: {bw.Encoding.EncodingName}");
  ```

----------------------------------------------------------------------
4. BINARYREADER: LECTURA DE TIPOS PRIMITIVOS
----------------------------------------------------------------------

BinaryReader tiene métodos de lectura específicos para cada tipo:

  ```csharp
  using BinaryReader br = new BinaryReader(File.Open("tipos.bin", FileMode.Open));

  byte b = br.ReadByte();          // 1 byte
  sbyte sb = br.ReadSByte();       // 1 byte
  short s = br.ReadInt16();        // 2 bytes
  ushort us = br.ReadUInt16();     // 2 bytes
  int i = br.ReadInt32();          // 4 bytes
  uint ui = br.ReadUInt32();       // 4 bytes
  long l = br.ReadInt64();         // 8 bytes
  ulong ul = br.ReadUInt64();      // 8 bytes
  float f = br.ReadSingle();       // 4 bytes
  double d = br.ReadDouble();      // 8 bytes
  decimal dec = br.ReadDecimal();  // 16 bytes
  bool bo = br.ReadBoolean();      // 1 byte
  char c = br.ReadChar();          // 2 bytes
  string str = br.ReadString();    // variable

  br.Close();
  ```

Es crítico leer los datos en el MISMO ORDEN en que fueron escritos.
Si escribes int, double, string, debes leer int, double, string.

----------------------------------------------------------------------
5. BINARYREADER: CONSTRUCTORES Y MÉTODOS
----------------------------------------------------------------------

Constructores:
  1. BinaryReader(Stream input)
  2. BinaryReader(Stream input, Encoding encoding)
  3. BinaryReader(Stream input, Encoding encoding, bool leaveOpen)

  ```csharp
  using FileStream fs = new FileStream("datos.bin", FileMode.Open,
      FileAccess.Read, FileShare.Read);
  using BinaryReader br = new BinaryReader(fs, Encoding.UTF8, leaveOpen: true);
  ```

Métodos adicionales:
  - Read(): Lee el siguiente byte y avanza.
  - Read(byte[], int, int): Lee un bloque de bytes.
  - Read(char[], int, int): Lee un bloque de caracteres.
  - ReadBytes(int count): Lee exactamente count bytes.
  - ReadChars(int count): Lee exactamente count caracteres.
  - PeekChar(): Lee el siguiente carácter sin avanzar.
  - ReadBoolean, ReadByte, ReadBytes, ... (métodos tipados)

  ```csharp
  using BinaryReader br = new BinaryReader(File.Open("datos.bin", FileMode.Open));

  // Leer un bloque de bytes
  byte[] header = br.ReadBytes(8);
  Console.WriteLine($"Header: {BitConverter.ToString(header)}");

  // Leer datos estructurados
  int id = br.ReadInt32();
  string nombre = br.ReadString();
  double saldo = br.ReadDouble();

  // Leer arrays
  int count = br.ReadInt32();
  double[] valores = new double[count];
  for (int i = 0; i < count; i++)
      valores[i] = br.ReadDouble();
  ```

6. STRINGS EN FORMATO BINARIO
----------------------------------------------------------------------

BinaryWriter no escribe strings como texto plano. Usa un formato especial:

  1. Prefijo: la longitud del string codificada en "7-bit encoded integer".
  2. Datos: los bytes del string en UTF-8 (por defecto).

El 7-bit encoded integer es una forma compacta de almacenar enteros:
  - Valores 0-127: 1 byte.
  - Valores 128-16383: 2 bytes.
  - Así sucesivamente.

  ```csharp
  using BinaryWriter bw = new BinaryWriter(File.Open("str.bin", FileMode.Create));
  bw.Write("Hola");          // 1 byte (5) + 4 bytes UTF-8 = 5 bytes
  bw.Write("");              // 1 byte (0)
  bw.Write("¡Hola mundo!");  // 1 byte (13) + 13 bytes UTF-8 = 14 bytes
  bw.Write("ñññ");           // 1 byte (6) + 6 bytes UTF-8 = 7 bytes

  // Leerlos
  using BinaryReader br = new BinaryReader(File.Open("str.bin", FileMode.Open));
  Console.WriteLine(br.ReadString()); // "Hola"
  Console.WriteLine(br.ReadString()); // ""
  Console.WriteLine(br.ReadString()); // "¡Hola mundo!"
  Console.WriteLine(br.ReadString()); // "ñññ"
  ```

Si necesitas un formato de string diferente, usa Write(byte[]) para control
manual:

  ```csharp
  // Escribir string como UTF-8 sin prefijo de longitud
  string texto = "Mi formato personalizado";
  byte[] bytesTexto = Encoding.UTF8.GetBytes(texto);
  bw.Write(bytesTexto.Length); // prefijo manual (Int32)
  bw.Write(bytesTexto);        // datos
  ```

7. ENDIANNESS Y COMPATIBILIDAD
----------------------------------------------------------------------

Endianness se refiere al orden en que se almacenan los bytes de un número.

  - Little-Endian: el byte menos significativo primero. Usado por x86/x64/ARM.
  - Big-Endian: el byte más significativo primero. Usado por redes (TCP/IP),
    algunas arquitecturas (Motorola 68k), y ciertos formatos de archivo.

BinaryReader/BinaryWriter usan Little-Endian por defecto.

Para leer/escribir en Big-Endian:
  ```csharp
  // No hay soporte directo, pero puedes invertir los bytes
  static int ReadBigEndianInt32(BinaryReader br)
  {
      byte[] bytes = br.ReadBytes(4);
      if (BitConverter.IsLittleEndian)
          Array.Reverse(bytes);
      return BitConverter.ToInt32(bytes);
  }

  static void WriteBigEndianInt32(BinaryWriter bw, int value)
  {
      byte[] bytes = BitConverter.GetBytes(value);
      if (BitConverter.IsLittleEndian)
          Array.Reverse(bytes);
      bw.Write(bytes);
  }
  ```

IPAddress.HostToNetworkOrder / NetworkToHostOrder:
  ```csharp
  // Convertir de host (LE) a network (BE)
  int networkOrder = IPAddress.HostToNetworkOrder(hostValue);
  byte[] beBytes = BitConverter.GetBytes(networkOrder);

  // Convertir de network (BE) a host (LE)
  int hostValue = IPAddress.NetworkToHostOrder(networkBytes);
  ```

Para formatos que requieren endianness específico:
  ```csharp
  public class BigEndianBinaryWriter
  {
      private readonly BinaryWriter _bw;
      public BigEndianBinaryWriter(Stream s) => _bw = new BinaryWriter(s);

      public void Write(int value)
      {
          byte[] bytes = BitConverter.GetBytes(value);
          if (BitConverter.IsLittleEndian) Array.Reverse(bytes);
          _bw.Write(bytes);
      }

      public void Write(short value) { /* similar */ }
      public void Write(long value) { /* similar */ }
      // etc.
  }
  ```

8. DISEÑO DE FORMATOS BINARIOS
----------------------------------------------------------------------

Diseñar un formato binario requiere planificación. Elementos típicos:

a) Magic Number: bytes fijos al inicio para identificar el formato.
  ```csharp
  // Escribir header
  bw.Write(new byte[] { 0x4D, 0x59, 0x46, 0x4D }); // "MYFM"
  bw.Write((int)1); // versión
  ```

b) Versionado: incluir número de versión para compatibilidad futura.
  ```csharp
  const int FORMAT_VERSION = 2;
  bw.Write(FORMAT_VERSION);
  ```

c) Tabla de contenidos: offsets a secciones del archivo.
  ```csharp
  long posTabla = bw.BaseStream.Position;
  bw.Write((long)0); // placeholder offset de datos
  bw.Write((int)0);  // placeholder count

  // ... escribir datos ...

  // Volver y actualizar tabla
  long datosOffset = bw.BaseStream.Position;
  // Actualizar en la tabla de contenidos
  ```

d) Checksum: verificación de integridad.
  ```csharp
  int checksum = CalcularChecksum(datos);
  bw.Write(checksum);
  ```

Ejemplo completo:
  ```csharp
  public class FormatoPersona
  {
      private const byte MAGIC = 0xAB;
      private const byte VERSION = 1;

      public static void Escribir(string path, Persona p)
      {
          using BinaryWriter bw = new BinaryWriter(File.Open(path, FileMode.Create));
          bw.Write(MAGIC);
          bw.Write(VERSION);
          bw.Write(p.Id);
          bw.Write(p.Nombre);
          bw.Write(p.Edad);
          bw.Write(p.Salario);
          bw.Write(p.Activo);
      }

      public static Persona Leer(string path)
      {
          using BinaryReader br = new BinaryReader(File.Open(path, FileMode.Open));
          byte magic = br.ReadByte();
          if (magic != MAGIC) throw new InvalidDataException("Formato inválido");
          byte version = br.ReadByte();
          if (version > VERSION) throw new InvalidDataException("Versión no soportada");

          return new Persona
          {
              Id = br.ReadInt32(),
              Nombre = br.ReadString(),
              Edad = br.ReadInt32(),
              Salario = br.ReadDouble(),
              Activo = br.ReadBoolean()
          };
      }
  }
  ```

9. COMPARATIVA: BINARIO VS TEXTO
----------------------------------------------------------------------

  Aspecto             | Binario (BinaryR/W)  | Texto (StreamR/W)
  --------------------|----------------------|---------------------
  Tamaño              | Compacto             | Mayor (números en ASCII)
  Legibilidad humana  | No                   | Sí
  Velocidad           | Más rápido           | Más lento (parsing)
  Precisión numérica  | Exacta               | Sujeto a redondeo
  Edición manual      | No (hex editor)      | Sí (notepad)
  Portabilidad        | Endianness matters   | UTF-8 universal
  Strings             | Prefijo+UTF-8        | Texto plano
  Complejidad         | Mayor                | Menor
  Uso típico          | Formatos propietarios| Config, logs, CSV

Regla práctica:
  - Usa binario si: tamaño importa, rendimiento crítico, no necesitas
    editar manualmente, trabajas con datos estructurados.
  - Usa texto si: necesitas depuración fácil, edición manual, intercambio
    entre sistemas, estándares abiertos (JSON, XML, CSV).

10. BUENAS PRÁCTICAS
----------------------------------------------------------------------

a) Siempre escribe y lee en el mismo orden. Crear una especificación del
   formato para referencia.

b) Incluye un magic number y versión en tus formatos binarios para
   detectar errores y permitir evolución.

c) Para archivos grandes (>100 MB), considera buffer de 64 KB+ en el
   FileStream subyacente.

d) Usa using para garantizar que los datos se escriban completamente.

e) String en binario usa prefijo de longitud; no uses ReadString() en un
   stream que no fue escrito con Write(string).

f) Verifica la longitud de arrays antes de leerlos para evitar asignaciones
   masivas.

g) Para datos críticos, añade checksum o CRC.

h) Documenta el formato binario (offsets, tipos, versión) para futuros
   desarrolladores.

  ```csharp
          static void EscribirTipos(string path)
          {
              using BinaryWriter bw = new BinaryWriter(File.Open(path, FileMode.Create));
              bw.Write(42);
              bw.Write(3.14159);
              bw.Write(true);
              bw.Write('X');
              bw.Write("Hola binario");
              bw.Write((byte)0xFF);
              bw.Write(123456789012345L);
          }

          static void LeerTipos(string path)
          {
              using BinaryReader br = new BinaryReader(File.Open(path, FileMode.Open));
              Console.WriteLine($"  int: {br.ReadInt32()}");
              Console.WriteLine($"  double: {br.ReadDouble()}");
              Console.WriteLine($"  bool: {br.ReadBoolean()}");
              Console.WriteLine($"  char: {br.ReadChar()}");
              Console.WriteLine($"  string: {br.ReadString()}");
              Console.WriteLine($"  byte: {br.ReadByte():X2}");
              Console.WriteLine($"  long: {br.ReadInt64()}");
          }

          static void EscribirPersonas(string path, List<Persona> personas)
          {
              using BinaryWriter bw = new BinaryWriter(File.Open(path, FileMode.Create));
              bw.Write(personas.Count);
              foreach (var p in personas)
              {
                  bw.Write(p.Id);
                  bw.Write(p.Nombre);
                  bw.Write(p.Edad);
                  bw.Write(p.Salario);
                  bw.Write(p.Activo);
              }
          }

          static List<Persona> LeerPersonas(string path)
          {
              var result = new List<Persona>();
              using BinaryReader br = new BinaryReader(File.Open(path, FileMode.Open));
              int count = br.ReadInt32();
              for (int i = 0; i < count; i++)
              {
                  result.Add(new Persona
                  {
                      Id = br.ReadInt32(),
                      Nombre = br.ReadString(),
                      Edad = br.ReadInt32(),
                      Salario = br.ReadDouble(),
                      Activo = br.ReadBoolean()
                  });
              }
              return result;
          }

          static void EscribirMatriz(string path)
          {
              using BinaryWriter bw = new BinaryWriter(File.Open(path, FileMode.Create));
              int filas = 3, cols = 4;
              bw.Write(filas);
              bw.Write(cols);
              for (int i = 0; i < filas; i++)
                  for (int j = 0; j < cols; j++)
                      bw.Write(i * 10.0 + j);
          }

          static void LeerMatriz(string path)
          {
              using BinaryReader br = new BinaryReader(File.Open(path, FileMode.Open));
              int filas = br.ReadInt32();
              int cols = br.ReadInt32();
              Console.WriteLine($"  Matriz {filas}x{cols}:");
              for (int i = 0; i < filas; i++)
              {
                  Console.Write("  ");
                  for (int j = 0; j < cols; j++)
                      Console.Write($"{br.ReadDouble(),6:F1} ");
                  Console.WriteLine();
              }
          }

          static void EscribirConMagic(string path)
          {
              using BinaryWriter bw = new BinaryWriter(File.Open(path, FileMode.Create));
              bw.Write(0x4D414769); // MAGIC: "MAGic"
              bw.Write((byte)1);    // versión
              bw.Write("Contenido con verificación");
          }

          static void LeerConMagic(string path)
          {
              using BinaryReader br = new BinaryReader(File.Open(path, FileMode.Open));
              int magic = br.ReadInt32();
              if (magic != 0x4D414769)
              {
                  Console.WriteLine("  ERROR: Magic number inválido");
                  return;
              }
              byte version = br.ReadByte();
              string contenido = br.ReadString();
              Console.WriteLine($"  Versión {version}: {contenido}");
          }

          static void EscribirBigEndian(string path, int value)
          {
              using BinaryWriter bw = new BinaryWriter(File.Open(path, FileMode.Create));
              byte[] bytes = BitConverter.GetBytes(value);
              if (BitConverter.IsLittleEndian) Array.Reverse(bytes);
              bw.Write(bytes);
              Console.WriteLine($"  Escrito {value:X8} en Big-Endian");
          }

          static void LeerBigEndian(string path)
          {
              using BinaryReader br = new BinaryReader(File.Open(path, FileMode.Open));
              byte[] bytes = br.ReadBytes(4);
              if (BitConverter.IsLittleEndian) Array.Reverse(bytes);
              int value = BitConverter.ToInt32(bytes);
              Console.WriteLine($"  Leído {value:X8} (Big-Endian)");
          }
      }

      public class Persona
      {
          public int Id { get; set; }
          public string Nombre { get; set; }
          public int Edad { get; set; }
          public double Salario { get; set; }
          public bool Activo { get; set; }
      }
  }
  ```

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Puedo mezclar BinaryReader con StreamReader en el mismo archivo?
R: Técnicamente sí, pero es mala práctica. Cada uno interpreta el stream
   de forma diferente (binario vs texto). Usa uno solo.

P: ¿Qué pasa si leo con el tipo incorrecto?
R: Obtendrás datos corruptos. No hay verificación de tipos en tiempo de
   ejecución. El orden y tipo deben coincidir exactamente.

P: ¿BinaryWriter.Write(string) es compatible entre plataformas?
R: Sí. Usa UTF-8 para los datos y 7-bit encoded integer para la longitud.
   El formato es consistente en todas las plataformas .NET.

P: ¿Cuánto espacio ahorra el binario frente al texto?
R: Depende. Un int en texto ocupa 1-10 bytes; en binario siempre 4.
   Para datos numéricos, el ahorro suele ser del 50-70%.

P: ¿BinaryReader/Writer son seguros para multihilo?
R: No. No tienen sincronización interna. Usa lock si accedes desde
   múltiples hilos.
