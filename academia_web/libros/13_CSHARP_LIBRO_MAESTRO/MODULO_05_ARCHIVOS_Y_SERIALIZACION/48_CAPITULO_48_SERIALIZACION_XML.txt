1. INTRODUCCIÓN A LA SERIALIZACIÓN XML EN .NET
----------------------------------------------------------------------

.NET proporciona dos serializadores XML principales:

a) XmlSerializer (System.Xml.Serialization):
   - Control total sobre la estructura XML.
   - Soporta atributos [XmlElement], [XmlAttribute], [XmlIgnore], etc.
   - Requiere que la clase sea pública y tenga constructor sin parámetros.
   - No requiere atributos específicos de serialización.

b) DataContractSerializer (System.Runtime.Serialization):
   - Orientado a contratos de datos (WCF, SOAP).
   - Usa atributos [DataContract] y [DataMember].
   - Más flexible con tipos complejos.
   - Soporta herencia y referencias circulares.

Namespace principales:
  ```csharp
  using System.Xml.Serialization;
  using System.Runtime.Serialization;
  ```

----------------------------------------------------------------------
2. XMLSERIALIZER: FUNDAMENTOS
----------------------------------------------------------------------

XmlSerializer convierte objetos a XML y viceversa.

  ```csharp
  using System.Xml.Serialization;

  // Clase a serializar
  public class Persona
  {
      public string Nombre { get; set; }
      public int Edad { get; set; }
      public bool Activo { get; set; }
  }

  // Serializar a string
  var persona = new Persona { Nombre = "Ana García", Edad = 28, Activo = true };
  var serializer = new XmlSerializer(typeof(Persona));

  using var writer = new StringWriter();
  serializer.Serialize(writer, persona);
  string xml = writer.ToString();
  Console.WriteLine(xml);

  // Deserializar desde string
  using var reader = new StringReader(xml);
  Persona? deserializada = serializer.Deserialize(reader) as Persona;
  Console.WriteLine($"Nombre: {deserializada?.Nombre}");
  ```

Serializar a archivo:
  ```csharp
  var serializer = new XmlSerializer(typeof(Persona));
  using FileStream fs = File.Create("persona.xml");
  serializer.Serialize(fs, persona);
  ```

Deserializar desde archivo:
  ```csharp
  var serializer = new XmlSerializer(typeof(Persona));
  using FileStream fs = File.OpenRead("persona.xml");
  Persona? p = serializer.Deserialize(fs) as Persona;
  ```

XML generado por defecto:
  ```xml
  <?xml version="1.0" encoding="utf-8"?>
  <Persona xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
           xmlns:xsd="http://www.w3.org/2001/XMLSchema">
    <Nombre>Ana García</Nombre>
    <Edad>28</Edad>
    <Activo>true</Activo>
  </Persona>
  ```

3. XMLSERIALIZER: ATRIBUTOS DE CONTROL
----------------------------------------------------------------------

Atributos para personalizar la estructura XML:

[XmlRoot]: Controla el nombre del elemento raíz.
  ```csharp
  [XmlRoot("empleado")]
  public class Persona { ... }
  // <empleado>...</empleado>
  ```

[XmlElement]: Controla el nombre del elemento.
  ```csharp
  public class Persona
  {
      [XmlElement("nombre_completo")]
      public string Nombre { get; set; }

      [XmlElement("edad")]
      public int Edad { get; set; }
  }
  // <nombre_completo>Ana</nombre_completo>
  ```

[XmlAttribute]: Propiedad como atributo XML.
  ```csharp
  public class Producto
  {
      [XmlAttribute("id")]
      public int Id { get; set; }

      [XmlElement("nombre")]
      public string Nombre { get; set; }

      [XmlElement("precio")]
      public decimal Precio { get; set; }
  }
  // <Producto id="1">
  //   <nombre>Laptop</nombre>
  //   <precio>1200.99</precio>
  // </Producto>
  ```

[XmlIgnore]: Excluye propiedad de la serialización.
  ```csharp
  public class Cuenta
  {
      public string Usuario { get; set; }

      [XmlIgnore]
      public string Contrasena { get; set; }
  }
  ```

[XmlText]: Propiedad como contenido textual del elemento.
  ```csharp
  public class Mensaje
  {
      [XmlAttribute("tipo")]
      public string Tipo { get; set; }

      [XmlText]
      public string Contenido { get; set; }
  }
  // <Mensaje tipo="error">Archivo no encontrado</Mensaje>
  ```

[XmlArray] y [XmlArrayItem]: Control de arrays.
  ```csharp
  public class Empresa
  {
      [XmlArray("empleados")]
      [XmlArrayItem("empleado")]
      public List<Persona> Empleados { get; set; }
  }
  // <empleados>
  //   <empleado>...</empleado>
  //   <empleado>...</empleado>
  // </empleados>
  ```

[XmlInclude]: Incluye tipos derivados.
  ```csharp
  [XmlInclude(typeof(Gato))]
  [XmlInclude(typeof(Perro))]
  public class Animal { ... }

  public class Gato : Animal { ... }
  public class Perro : Animal { ... }
  ```

4. XMLSERIALIZER: NAMESPACES
----------------------------------------------------------------------

Los namespaces XML evitan conflictos de nombres.

  ```csharp
  var ns = new XmlSerializerNamespaces();
  ns.Add("", "http://ejemplo.com/personas"); // namespace por defecto
  ns.Add("dir", "http://ejemplo.com/direcciones");

  var serializer = new XmlSerializer(typeof(PersonaConDireccion));
  using var writer = new StringWriter();
  serializer.Serialize(writer, persona, ns);
  ```

Definir namespace en atributos:
  ```csharp
  [XmlRoot(Namespace = "http://ejemplo.com/facturas")]
  public class Factura
  {
      [XmlElement(Namespace = "http://ejemplo.com/comun")]
      public int Numero { get; set; }
  }
  ```

Eliminar namespaces por defecto:
  ```csharp
  var emptyNs = new XmlSerializerNamespaces();
  emptyNs.Add("", "");
  serializer.Serialize(writer, obj, emptyNs);
  // Elimina xmlns:xsi y xmlns:xsd
  ```

5. XMLSERIALIZER: COLECCIONES Y ARRAYS
----------------------------------------------------------------------

Colecciones simples:
  ```csharp
  public class Equipo
  {
      [XmlArray("miembros")]
      [XmlArrayItem("miembro")]
      public List<string> Integrantes { get; set; }
  }
  // <miembros>
  //   <miembro>Ana</miembro>
  //   <miembro>Luis</miembro>
  // </miembros>
  ```

Arrays sin contenedor (elementos directamente bajo el padre):
  ```csharp
  public class Equipo
  {
      [XmlElement("miembro")]
      public string[] Integrantes { get; set; }
  }
  // <Equipo>
  //   <miembro>Ana</miembro>
  //   <miembro>Luis</miembro>
  // </Equipo>
  ```

Diccionarios: XmlSerializer no soporta diccionarios directamente.
Solución: crear una clase serializable.

  ```csharp
  public class Configuracion
  {
      [XmlArray("items")]
      [XmlArrayItem("item")]
      public List<ConfigItem> Items { get; set; }
  }

  public class ConfigItem
  {
      [XmlAttribute("clave")]
      public string Clave { get; set; }

      [XmlText]
      public string Valor { get; set; }
  }
  // <items>
  //   <item clave="timeout">30</item>
  //   <item clave="host">localhost</item>
  // </items>
  ```

----------------------------------------------------------------------
6. DATACONTRACTSERIALIZER
----------------------------------------------------------------------

DataContractSerializer es el serializador de WCF/SOAP.

  ```csharp
  using System.Runtime.Serialization;
  using System.Xml;

  [DataContract]
  public class Persona
  {
      [DataMember]
      public string Nombre { get; set; }

      [DataMember]
      public int Edad { get; set; }

      [DataMember(Name = "activo")]
      public bool Activo { get; set; }
  }

  // Serializar
  var persona = new Persona { Nombre = "Ana", Edad = 28, Activo = true };
  var serializer = new DataContractSerializer(typeof(Persona));

  using var writer = new StringWriter();
  using var xmlWriter = XmlWriter.Create(writer, new XmlWriterSettings { Indent = true });
  serializer.WriteObject(xmlWriter, persona);
  string xml = writer.ToString();

  // Deserializar
  using var reader = new StringReader(xml);
  using var xmlReader = XmlReader.Create(reader);
  Persona? deserializada = serializer.ReadObject(xmlReader) as Persona;
  ```

XML generado por DataContractSerializer:
  ```xml
  <?xml version="1.0" encoding="utf-8"?>
  <Persona xmlns:i="http://www.w3.org/2001/XMLSchema-instance"
           xmlns="http://schemas.datacontract.org/2004/07/">
    <Nombre>Ana</Nombre>
    <Edad>28</Edad>
    <activo>true</activo>
  </Persona>
  ```

----------------------------------------------------------------------
7. DATACONTRACTSERIALIZER: ATRIBUTOS
----------------------------------------------------------------------

[DataContract]: Marca la clase como contrato de datos.
  ```csharp
  [DataContract(Namespace = "http://ejemplo.com/personas")]
  public class Persona { ... }
  ```

[DataMember]: Marca propiedades/fields para serializar.
  ```csharp
  [DataContract]
  public class Producto
  {
      [DataMember(Name = "id", Order = 1)]
      public int Id { get; set; }

      [DataMember(Name = "nombre", Order = 2, IsRequired = true)]
      public string Nombre { get; set; }

      [DataMember(Name = "precio", Order = 3)]
      public decimal Precio { get; set; }

      [DataMember(Name = "fecha_creacion", Order = 4, EmitDefaultValue = false)]
      public DateTime FechaCreacion { get; set; }
  }
  ```

[IgnoreDataMember]: Excluye propiedad.
  ```csharp
  [DataContract]
  public class Cuenta
  {
      [DataMember]
      public string Usuario { get; set; }

      [IgnoreDataMember]
      public string Contrasena { get; set; }
  }
  ```

Propiedades adicionales de DataMember:
  - Name: nombre en XML (por defecto, el nombre de la propiedad).
  - Order: orden de los elementos en XML.
  - IsRequired: lanza excepción si falta en deserialización.
  - EmitDefaultValue: incluir valores por defecto (false para omitir).

----------------------------------------------------------------------
8. DIFERENCIAS: XMLSERIALIZER VS DATACONTRACTSERIALIZER
----------------------------------------------------------------------

  Aspecto                     | XmlSerializer              | DataContractSerializer
  ----------------------------|----------------------------|---------------------------
  Namespace base              | System.Xml.Serialization   | System.Runtime.Serialization
  Atributos requeridos        | Ninguno (opcionales)       | [DataContract] + [DataMember]
  Constructor sin parámetros  | Requerido                  | Requerido
  Tipos soportados            | Tipos primitivos, objetos  | Todos, incluyendo genéricos
  Diccionarios                | No nativo                  | Sí
  Herencia                    | [XmlInclude]               | [KnownType]
  Referencias circulares      | No soportado               | Sí (con preserveObjectReferences)
  Control de XML              | Total (atributos, orden)   | Limitado (Order, Name)
  Rendimiento                 | Más lento (reflection)     | Más rápido
  Uso típico                  | Documentos XML, config     | WCF, SOAP, contratos de datos

Cuándo usar cada uno:
  - XmlSerializer: cuando necesitas control total sobre la estructura XML,
    archivos de configuración, documentos XML estándar.
  - DataContractSerializer: cuando trabajas con servicios WCF/SOAP,
    necesitas rendimiento, o requieres soporte para herencia avanzada.

----------------------------------------------------------------------
9. XML VS JSON: COMPARATIVA
----------------------------------------------------------------------

  Aspecto                     | XML                          | JSON
  ----------------------------|------------------------------|------------------------------
  Legibilidad humana          | Media (verboso)              | Alta (conciso)
  Tamaño                      | Mayor (tags de cierre)       | Menor (ligero)
  Velocidad de parseo         | Más lento                    | Más rápido
  Tipos de datos              | Solo texto (parsear)         | Nativos (number, bool, null)
  Atributos vs propiedades    | Atributos + elementos        | Solo propiedades
  Namespaces                  | Sí (xmlns)                   | No
  Esquema/validación          | XSD                          | JSON Schema
  Namespaces                  | Sí                           | No
  Documentación               | XSLT, XPath, XQuery          | No equivalente
  Soporte en APIs modernas    | Disminuyendo                 | Estándar de facto
  Comentarios                 | Sí                           | No (JSON5 sí)
  Herramientas .NET           | XmlSerializer, LINQ to XML   | System.Text.Json, Newtonsoft

Cuándo usar XML:
  - Documentos con metadatos (atributos mezclados con contenido).
  - Sistemas empresariales/legados.
  - Servicios SOAP.
  - Configuraciones complejas (web.config, app.config).
  - Formatos de oficina (Office Open XML, OpenDocument).

Cuándo usar JSON:
  - APIs REST.
  - Configuraciones simples.
  - Comunicación navegador-servidor.
  - Microservicios.
  - Almacenamiento NoSQL (MongoDB, etc.).

10. BUENAS PRÁCTICAS
----------------------------------------------------------------------

a) Cachea XmlSerializer: el constructor es costoso porque genera código
   de serialización dinámicamente.

  ```csharp
  private static readonly XmlSerializer _serializer =
      new XmlSerializer(typeof(Persona));
  ```

b) Usa XmlReader.Create con XmlReaderSettings para validación.

c) Para archivos grandes, usa XmlWriter con configuración de buffer.

d) Define namespaces explícitamente para evitar xmlns:xsi y xmlns:xsd
   redundantes.

e) XmlSerializer no es seguro para multihilo: no compartas una instancia
   entre hilos.

f) DataContractSerializer es seguro para multihilo (puedes cachearlo).

g) Para evitar fugas de memoria con XmlSerializer, usa el constructor
   que acepta tipos adicionales o el cacheo de ensamblados.

h) Siempre maneja InvalidOperationException que lanza XmlSerializer
   cuando el XML no coincide con la clase esperada.

          static string SerializarAXml<T>(T obj)
          {
              var serializer = new XmlSerializer(typeof(T));
              var ns = new XmlSerializerNamespaces();
              ns.Add("", "");
              using var sw = new StringWriter();
              serializer.Serialize(sw, obj, ns);
              return sw.ToString();
          }

          static T? DeserializarDeXml<T>(string xml)
          {
              var serializer = new XmlSerializer(typeof(T));
              using var sr = new StringReader(xml);
              return serializer.Deserialize(sr) as T;
          }

          static string SerializarConNamespaces(Factura factura)
          {
              var serializer = new XmlSerializer(typeof(Factura));
              var ns = new XmlSerializerNamespaces();
              ns.Add("fac", "http://ejemplo.com/facturas");
              ns.Add("com", "http://ejemplo.com/comun");
              using var sw = new StringWriter();
              serializer.Serialize(sw, factura, ns);
              return sw.ToString();
          }

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Por qué XmlSerializer requiere constructor sin parámetros?
R: Porque crea objetos usando Activator.CreateInstance. Si tu clase no
   tiene constructor público sin parámetros, usa DataContractSerializer.

P: ¿Cómo serializo propiedades privadas con XmlSerializer?
R: No se puede. XmlSerializer solo serializa miembros públicos.
   DataContractSerializer puede serializar privados con [DataMember].

P: ¿XmlSerializer soporta Dictionary?
R: No directamente. Debes convertir el diccionario a una lista de pares
   clave-valor serializable.

P: ¿Por qué mi XML tiene xmlns:xsi y xmlns:xsd?
R: Son namespaces por defecto para tipos y esquemas. Puedes eliminarlos
   usando XmlSerializerNamespaces vacío.

P: ¿Cuál es más rápido, XmlSerializer o DataContractSerializer?
R: DataContractSerializer es generalmente más rápido porque utiliza
   contratos optimizados. XmlSerializer genera código dinámico.

================================================================================

Implementa un "Generador de Documentos XML desde Esquemas" que:

  - Lea un XSD (XML Schema Definition) y genere clases C# automáticamente
    con los atributos XmlSerializer apropiados.
  - Permita crear instancias de esas clases mediante una API fluida.
  - Serialice las instancias a XML validado contra el XSD original.
  - Soporte tipos simples, complejos, arrays, herencia y namespaces.

  ```csharp
  // Ejemplo de uso
  var generador = new XmlGenerator("factura.xsd");
  var factory = generador.CreateFactory();

  var factura = factory.Create("Factura")
      .Set("Numero", 1001)
      .Set("Cliente", "Empresa XYZ")
      .Set("Fecha", DateTime.Now)
      .AddToArray("Lineas", linea => linea
          .Set("Producto", "Laptop")
          .Set("Cantidad", 2)
          .Set("Precio", 1500.00m));

  string xml = factura.ToXml(validar: true);
  ```

Pistas:
  - Usa System.Xml.Schema para leer el XSD.
  - Reflection.Emit o System.CodeDom para generar clases.
  - XmlSerializer para serialización.
  - XmlSchemaValidator para validación.
  - Namespaces: extrae targetNamespace del XSD.

Criterios de éxito:
  - Genera clases que se compilan y serializan correctamente.
  - Validación contra XSD (errores reportados con detalle).
  - API fluida intuitiva.
  - Soporta tipos complejos anidados.
  - Manejo de namespaces correcto.
