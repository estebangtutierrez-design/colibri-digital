¡Bienvenido al Capítulo 29 del C# Master Book! Los tipos nullable
permiten que los tipos de valor (structs, int, bool, DateTime, etc.)
puedan representar la ausencia de valor, algo que antes solo era posible
con tipos de referencia. Con Nullable<T>, el operador ?, el operador
?? (null-coalescing), ??= (null-coalescing assignment), y el operador
?. (null-conditional), C# ofrece un conjunto completo de herramientas
para trabajar con valores nulos de forma segura y expresiva.

En este capítulo también exploraremos el contexto nullable de C# 8+,
que permite al compilador analizar y advertir sobre posibles null
reference exceptions, llevando la seguridad de tipos a otro nivel.

29.1 — NULLABLE<T> — INTRODUCCIÓN
-----------------------------------

Nullable<T> es un struct que envuelve un tipo de valor para que pueda
representar null. La sintaxis T? es un azúcar sintáctico para Nullable<T>.

```csharp
// Sin nullable: int no puede ser null
int normal = 42;
// int normal = null; // Error de compilación

// Con nullable: int? sí puede ser null
int? nullable = null;
int? conValor = 42;

// Nullable<T> es equivalente a:
Nullable<int> explícito = new Nullable<int>(42);
Nullable<int> nulo = new Nullable<int>(); // null

// Propiedades de Nullable<T>
bool tieneValor = conValor.HasValue; // True
int valorReal = conValor.Value;      // 42

// Acceso seguro con GetValueOrDefault
int seguro = nullable.GetValueOrDefault(); // 0 (default de int)
int conDefault = nullable.GetValueOrDefault(100); // 100

// Comparación
Console.WriteLine(nullable == null);  // True
Console.WriteLine(nullable.HasValue); // False
Console.WriteLine(conValor.HasValue); // True
```

29.2 — OPERADOR ?? (NULL-COALESCING)
--------------------------------------

El operador ?? devuelve el valor de la izquierda si no es null, o el
valor de la derecha si es null.

```csharp
int? a = null;
int? b = 42;

int resultado1 = a ?? 0;   // 0 (a es null)
int resultado2 = b ?? 0;   // 42 (b no es null)

// Encadenamiento
int? x = null;
int? y = null;
int? z = 99;
int final = x ?? y ?? z ?? -1; // 99

// Con string (reference type nullable)
string? nombre = null;
string saludo = $"Hola, {nombre ?? "Invitado"}!";
Console.WriteLine(saludo); // Hola, Invitado!

// Con objetos
List<int>? lista = null;
int count = lista?.Count ?? 0;
Console.WriteLine(count); // 0
```

29.3 — OPERADOR ??= (NULL-COALESCING ASSIGNMENT)
---------------------------------------------------

El operador ??= asigna el valor de la derecha a la izquierda solo si
la izquierda es null.

```csharp
int? contador = null;
contador ??= 0;
Console.WriteLine(contador); // 0

contador ??= 100;
Console.WriteLine(contador); // 0 (no cambia porque ya tiene valor)

// Caso de uso común: inicialización lazy
List<string>? items = null;
items ??= new List<string>();
items.Add("Elemento");

// Con diccionarios
public class Config
{
    private Dictionary<string, string>? _cache;

    public string Obtener(string clave)
    {
        _cache ??= new Dictionary<string, string>();
        return _cache.TryGetValue(clave, out var val) ? val : "";
    }

    public void Establecer(string clave, string valor)
    {
        _cache ??= new Dictionary<string, string>();
        _cache[clave] = valor;
    }
}

// Con propiedades
private string? _nombre;
public string Nombre
{
    get => _nombre ?? "Sin nombre";
    set => _nombre = value;
}
```

29.4 — OPERADOR ?. (NULL-CONDITIONAL)
---------------------------------------

El operador ?. accede a miembros (propiedades, métodos, indexadores)
de forma segura, devolviendo null si el objeto es null.

```csharp
Persona? persona = null;

// Sin null-conditional (peligroso)
// string n = persona.Nombre; // NullReferenceException

// Con null-conditional
string? nombre = persona?.Nombre; // null (no lanza excepción)
int? edad = persona?.Edad;       // null

// Encadenamiento
string? ciudad = persona?.Direccion?.Ciudad;

// Con métodos
persona?.Saludar(); // No hace nada si es null

// Con indexadores
List<int>? numeros = null;
int? primero = numeros?[0]; // null

// Con delegados
Action? accion = null;
accion?.Invoke(); // Seguro, no lanza excepción

// Eventos
EventHandler? miEvento = null;
miEvento?.Invoke(this, EventArgs.Empty); // Seguro

// Combinación con ??
string resultado = persona?.Nombre ?? "Invitado";
Console.WriteLine(resultado); // Invitado

int edadReal = persona?.Edad ?? -1;
Console.WriteLine(edadReal); // -1
```

29.5 — PATTERN MATCHING CON NULL
-----------------------------------

```csharp
int? valor = null;

// Patrón null
if (valor is null)
    Console.WriteLine("Es null");

// Patrón not null
if (valor is not null)
    Console.WriteLine($"Valor: {valor}");

// Switch con null
string clasificar = valor switch
{
    null => "Sin valor",
    > 0 => "Positivo",
    < 0 => "Negativo",
    0 => "Cero"
};

// Property pattern con null
if (persona is { Nombre: not null, Edad: > 18 })
    Console.WriteLine("Mayor de edad con nombre");

// Tuple pattern con null
if ((persona?.Nombre, persona?.Edad) is (not null, > 0))
    Console.WriteLine("Persona válida");

// Operador is not null (forma clásica)
if (valor.HasValue)
    Console.WriteLine($"Tiene valor: {valor.Value}");
```

29.6 — NULLABLE REFERENCE TYPES (C# 8+)
------------------------------------------

A partir de C# 8, puedes habilitar nullable reference types para que
el compilador analice y advierta sobre posibles null.

```csharp
// Habilitar nullable context (en .csproj o con #nullable directive)
#nullable enable

public class Estudiante
{
    // Propiedades referencia no-nullable (deben inicializarse)
    public string Nombre { get; set; } = string.Empty;
    public string Apellido { get; set; } = "";

    // Propiedades referencia nullable
    public string? Email { get; set; }
    public string? Telefono { get; set; }

    // Propiedad no-nullable con inicialización tardía
    public string NombreCompleto => $"{Nombre} {Apellido}";

    public Estudiante(string nombre, string apellido)
    {
        Nombre = nombre;
        Apellido = apellido;
    }

    public void MostrarInfo()
    {
        // Email? requiere ?. o comprobación
        Console.WriteLine($"Email: {Email ?? "No especificado"}");
        Console.WriteLine($"Teléfono: {Telefono ?? "No especificado"}");

        // Sin advertencia: NombreCompleto es no-nullable
        Console.WriteLine(NombreCompleto);
    }
}

// Uso
var est = new Estudiante("Ana", "García");
est.Email = "ana@email.com";
est.MostrarInfo();
```

29.7 — ANOTACIONES DE NULLABILIDAD
-------------------------------------

```csharp
#nullable enable

public class Servicio
{
    // Parámetro no-nullable
    public void Procesar(string entrada)
    {
        Console.WriteLine(entrada.ToUpper()); // Seguro
    }

    // Parámetro nullable
    public void ProcesarOpcional(string? entrada)
    {
        // El compilador advierte si no compruebas null
        Console.WriteLine(entrada?.ToUpper() ?? "NULO");
    }

    // Retorno no-nullable
    public string ObtenerNombre() => "Ejemplo";

    // Retorno nullable
    public string? ObtenerOpcional(bool existe)
    {
        return existe ? "Valor" : null;
    }

    // Parámetro de salida nullable
    public bool TryObtenerValor(out string? resultado)
    {
        resultado = null;
        return false;
    }

    // El operador ! (null-forgiving) suprime advertencias
    public void UsarNullForgiving()
    {
        string? posibleNull = ObtenerOpcional(false);
        string seguro = posibleNull!; // Le decimos al compilador que no es null
        Console.WriteLine(seguro.Length); // Sin advertencia
        // Pero si es null en runtime, lanza excepción
    }
}
```

29.8 — ATRIBUTOS DE NULLABILIDAD
-----------------------------------

```csharp
using System.Diagnostics.CodeAnalysis;

public class AtributosNullability
{
    // [NotNull]: el valor de retorno nunca es null
    [return: NotNull]
    public string NuncaNull() => "Siempre valor";

    // [MaybeNull]: el valor de retorno puede ser null
    [return: MaybeNull]
    public string? PuedeNull(bool condicion) => condicion ? "Valor" : null;

    // [NotNullWhen]: indica que el parámetro out no es null cuando el método devuelve true
    public bool TryGetValue(string key, [NotNullWhen(true)] out string? valor)
    {
        if (key == "existe")
        {
            valor = "Encontrado";
            return true;
        }
        valor = null;
        return false;
    }

    // [MemberNotNull]: indica que la propiedad no será null después de llamar al método
    [MemberNotNull(nameof(Nombre))]
    public void Inicializar()
    {
        Nombre = "Inicializado";
    }

    public string Nombre { get; set; } = "";
}

// Uso
var attr = new AtributosNullability();
if (attr.TryGetValue("existe", out string? valor))
    Console.WriteLine(valor.Length); // Sin advertencia, gracias a [NotNullWhen]
```

29.9 — NULLABLE VALUE TYPES AVANZADO
--------------------------------------

```csharp
// Nullable booleano (tres estados: true, false, null)
bool? opcion = null;

if (opcion == true) Console.WriteLine("Sí");
else if (opcion == false) Console.WriteLine("No");
else Console.WriteLine("Indefinido");

// Operaciones con nullable bool
bool? a = true;
bool? b = null;
Console.WriteLine(a & b); // null (AND con null da null)
Console.WriteLine(a | b); // true (OR con true da true)
Console.WriteLine(a ^ b); // null

// Nullable DateTime
DateTime? fechaNacimiento = null;
DateTime? hoy = DateTime.Today;

// Comparación de nullable
if (fechaNacimiento > hoy)
    Console.WriteLine("¡Fecha futura!");

// LINQ con nullable
var personas = new List<Persona?>
{
    new Persona { Nombre = "Ana", Edad = 30 },
    null,
    new Persona { Nombre = "Luis", Edad = null }
};

var nombres = personas
    .Where(p => p?.Edad > 18)
    .Select(p => p?.Nombre ?? "Anónimo")
    .ToList();

// Nullable en structs personalizados
public struct Punto
{
    public double X, Y;
}

Punto? puntoNull = null;
Punto? puntoValido = new Punto { X = 1, Y = 2 };
Console.WriteLine(puntoValido?.X ?? 0); // 1
```

29.10 — CONVERSIÓN ENTRE NULLABLE Y NO-NULLABLE
--------------------------------------------------

```csharp
int? nullable = 42;

// De nullable a no-nullable (requiere comprobación o !)
if (nullable.HasValue)
{
    int valor = nullable.Value; // Seguro
}

int directo = nullable ?? 0;    // Con valor por defecto
int forzado = nullable!.Value;  // Null-forgiving (peligroso si es null)

// De no-nullable a nullable (siempre seguro)
int noNullable = 100;
int? convertido = noNullable;   // Implícito
int? convertido2 = (int?)noNullable; // Explícito

// Parseo de string a nullable
string entrada = "42";
int? resultado = int.TryParse(entrada, out int val) ? val : null;
Console.WriteLine(resultado ?? -1); // 42

// En LINQ, FirstOrDefault devuelve default (0 para int) pero podemos usar null
var numeros = new List<int>();
int? primero = numeros.Select(n => (int?)n).FirstOrDefault();
Console.WriteLine(primero ?? -1); // -1 (porque la lista está vacía)
```

29.11 — NULLABLE Y GENÉRICOS
------------------------------

```csharp
public static class NullableHelper
{
    public static T? ToNullable<T>(this T valor) where T : struct =>
        valor;

    public static T? ParseNullable<T>(string? valor) where T : struct
    {
        if (string.IsNullOrWhiteSpace(valor))
            return null;

        try
        {
            return (T)Convert.ChangeType(valor, typeof(T));
        }
        catch
        {
            return null;
        }
    }

    public static T? MaxNullable<T>(T? a, T? b) where T : struct, IComparable<T>
    {
        if (!a.HasValue) return b;
        if (!b.HasValue) return a;
        return a.Value.CompareTo(b.Value) > 0 ? a : b;
    }

    public static T? SumNullable<T>(T? a, T? b) where T : struct
    {
        if (!a.HasValue || !b.HasValue) return null;
        dynamic da = a.Value;
        dynamic db = b.Value;
        return (T?)(da + db);
    }
}

// Uso
int? v1 = NullableHelper.ParseNullable<int>("42"); // 42
int? v2 = NullableHelper.ParseNullable<int>("abc"); // null

int? max = NullableHelper.MaxNullable<int>(100, 200); // 200
int? maxNull = NullableHelper.MaxNullable<int>(null, 200); // 200
```

```csharp
public class BusquedaRequest
{
    public string? Query { get; set; }
    public int? Pagina { get; set; }
    public int? TamanoPagina { get; set; }
    public DateTime? Desde { get; set; }
    public DateTime? Hasta { get; set; }
    public bool? Activo { get; set; }
}

public class BusquedaService
{
    public List<string> Buscar(BusquedaRequest request)
    {
        // Uso de ?? para valores predeterminados
        int pagina = request.Pagina ?? 1;
        int tamano = request.TamanoPagina ?? 20;

        Console.WriteLine($"Buscando: {request.Query ?? "*"}");
        Console.WriteLine($"Página {pagina} de {tamano} resultados");

        // Uso de ?. para acceso seguro
        if (request.Desde.HasValue)
            Console.WriteLine($"Desde: {request.Desde:yyyy-MM-dd}");

        if (request.Hasta.HasValue)
            Console.WriteLine($"Hasta: {request.Hasta:yyyy-MM-dd}");

        if (request.Activo ?? true)
            Console.WriteLine("Filtrando solo activos");

        return new List<string> { "Resultado1", "Resultado2" };
    }
}

// Uso
var busqueda = new BusquedaRequest
{
    Query = "C#",
    Pagina = 2,
    TamanoPagina = 10
};

new BusquedaService().Buscar(busqueda);
```

29.14 — EJEMPLO COMPLETO: SISTEMA DE CONFIGURACIÓN CON NULLABLES
------------------------------------------------------------------

```csharp
#nullable enable

using System;
using System.Collections.Generic;
using System.IO;
using System.Text.Json;

public class ConfiguracionApp
{
    private Dictionary<string, object?> _valores = new();
    private string _rutaArchivo;

    public ConfiguracionApp(string rutaArchivo)
    {
        _rutaArchivo = rutaArchivo;
        Cargar();
    }

    // Indexador nullable
    public object? this[string clave]
    {
        get => _valores.TryGetValue(clave, out var val) ? val : null;
        set => _valores[clave] = value;
    }

    public T? Obtener<T>(string clave) where T : struct
    {
        var val = _valores.TryGetValue(clave, out var obj) ? obj : null;
        return val switch
        {
            T t => t,
            JsonElement je when je.ValueKind == JsonValueKind.Number =>
                (T?)Convert.ChangeType(je.GetRawText(), typeof(T)),
            _ => null
        };
    }

    public T? ObtenerObjeto<T>(string clave) where T : class
    {
        var val = _valores.TryGetValue(clave, out var obj) ? obj : null;

        // Pattern matching con null
        return val switch
        {
            T t => t,
            JsonElement je => JsonSerializer.Deserialize<T>(je.GetRawText()),
            _ => null
        };
    }

    public string ObtenerString(string clave, string? defaultValor = null)
    {
        return _valores.TryGetValue(clave, out var val)
            ? val?.ToString() ?? defaultValor ?? ""
            : defaultValor ?? "";
    }

    public int ObtenerInt(string clave, int defaultValor = 0)
    {
        var str = ObtenerString(clave);
        return int.TryParse(str, out int result) ? result : defaultValor;
    }

    public bool ObtenerBool(string clave, bool defaultValor = false)
    {
        var str = ObtenerString(clave)?.ToLower();
        return str switch
        {
            "true" or "1" or "yes" => true,
            "false" or "0" or "no" => false,
            _ => defaultValor
        };
    }

    public void Establecer(string clave, object? valor)
    {
        _valores[clave] = valor;
    }

    public void Guardar()
    {
        string json = JsonSerializer.Serialize(_valores, new JsonSerializerOptions
        {
            WriteIndented = true
        });
        File.WriteAllText(_rutaArchivo, json);
        Console.WriteLine($"Configuración guardada en {_rutaArchivo}");
    }

    private void Cargar()
    {
        if (!File.Exists(_rutaArchivo))
        {
            Console.WriteLine("Archivo de configuración no encontrado. Usando valores predeterminados.");
            return;
        }

        string json = File.ReadAllText(_rutaArchivo);
        var datos = JsonSerializer.Deserialize<Dictionary<string, JsonElement>>(json);

        if (datos is null)
        {
            Console.WriteLine("Error al deserializar configuración.");
            return;
        }

        foreach (var kvp in datos)
            _valores[kvp.Key] = kvp.Value;

        Console.WriteLine($"Configuración cargada ({_valores.Count} valores)");
    }

    public void MostrarTodo()
    {
        foreach (var kvp in _valores)
        {
            string valorStr = kvp.Value?.ToString() ?? "(null)";
            Console.WriteLine($"  {kvp.Key}: {valorStr}");
        }
    }
}

public class Programa
{
    public static void Main()
    {
        string ruta = "config.json";
        var config = new ConfiguracionApp(ruta);

        // Establecer valores (algunos null)
        config.Establecer("app_name", "MiApp");
        config.Establecer("version", "1.0.0");
        config.Establecer("max_usuarios", 100);
        config.Establecer("debug_mode", true);
        config.Establecer("db_connection", null!); // explícitamente null
        config.Establecer("feature_x_enabled", null); // también null

        config.Guardar();

        // Leer valores con coalescing
        string appName = config.ObtenerString("app_name") ?? "AppDefault";
        int maxUsers = config.ObtenerInt("max_usuarios", 50);
        bool debug = config.ObtenerBool("debug_mode");
        string? dbConn = config.ObtenerString("db_connection");
        string featureX = config.ObtenerString("feature_x_enabled") ?? "deshabilitado";

        Console.WriteLine($"\n=== CONFIGURACIÓN CARGADA ===");
        Console.WriteLine($"App: {appName}");
        Console.WriteLine($"Max Usuarios: {maxUsers}");
        Console.WriteLine($"Debug: {debug}");
        Console.WriteLine($"DB Connection: {dbConn ?? "No configurado"}");
        Console.WriteLine($"Feature X: {featureX}");

        // Demostrar null-conditional con evento
        EventHandler? onCambio = null;
        onCambio?.Invoke(null, EventArgs.Empty); // Seguro

        // Demostrar ??=
        List<string>? items = null;
        items ??= new List<string>();
        items.Add("Nuevo item");
        Console.WriteLine($"Items: {items?.Count ?? 0}");

        // Demostrar pattern matching con null
        string? test = null;
        string mensaje = test switch
        {
            not null when test.Length > 0 => $"Texto: {test}",
            not null => "String vacío",
            null => "Valor nulo"
        };
        Console.WriteLine(mensaje);

        // Nullable value types
        int? contador = null;
        contador ??= 0;
        contador++;
        Console.WriteLine($"Contador: {contador}");

        DateTime? ultimoAcceso = null;
        string ultimo = ultimoAcceso?.ToString("g") ?? "Nunca";
        Console.WriteLine($"Último acceso: {ultimo}");

        Console.WriteLine("\nConfiguración final:");
        config.MostrarTodo();
    }
}
```

================================================================================
  FAQ — PREGUNTAS FRECUENTES
================================================================================

P: ¿Nullable<T> es un tipo de valor o referencia?
R: Es un tipo de valor (struct). Nullable<int> es un struct.

P: ¿Cuándo debo usar int? vs Nullable<int>?
R: Usa int? por brevedad. Son equivalentes.

P: ¿El operador ?. funciona con indexadores?
R: Sí: dict?[key] devuelve null si dict es null.

P: ¿Puedo tener un nullable de un nullable?
R: No. int?? no tiene sentido. El compilador lo simplifica a int?.

P: ¿Qué diferencia hay entre ?? y ?: con comprobación null?
R: ?? es más legible: a ?? b vs a != null ? a : b.

P: ¿Los nullable reference types afectan al runtime?
R: No. Solo son análisis en tiempo de compilación. El runtime
   no cambia.

P: ¿Debo usar #nullable enable en todos los proyectos?
R: Sí, es recomendable. Ayuda a prevenir millones de NRE.

P: ¿Qué es el null-forgiving operator !?
R: Le dice al compilador "confía en mí, esto no es null".
   Úsalo solo cuando estés 100% seguro.

  DESAFÍO DEL ARQUITECTO
================================================================================

Diseña un framework de repositorio de datos que maneje valores nullables
de forma integral. El framework debe:

- Repositorio<T, TId> donde TId? puede ser null para entidades nuevas.
- Métodos: GetById(TId id) que devuelve T? (puede no existir).
- GetAll() que filtra elementos null (el repositorio interno puede
  tener entradas null).
- Insert(T entity) que valide que la entidad no tenga propiedades
  requeridas null usando atributos personalizados.
- Update(T entity) que use ??= para preservar valores existentes
  cuando el nuevo valor es null (merge parcial).
- Delete(TId id) que devuelva bool? indicando si se eliminó (true),
  no existía (false), o error (null).
- Query fluido con métodos que devuelvan resultados nullable:
  - FirstOrDefault() -> T?
  - SingleOrDefault() -> T?
  - Where(Expression) que maneje correctamente null en las expresiones.
- Soporte para lazy loading donde propiedades navegacionales sean T?
  y se carguen con ?. y ??.
- Cache integrado que use ConcurrentDictionary<TId, T?> donde los
  valores null representen "no en caché" vs "no existe en BD".
- Serialización a JSON que omita propiedades null por defecto.
- Un sistema de logging que registre cada operación con valores
  nullables formateados.

Requisitos adicionales:
- Todo el código debe estar en #nullable enable.
- Usar [NotNull], [MaybeNull], [NotNullWhen] en todas las APIs.
- Tests que demuestren que nunca se produce NRE.
- Benchmark comparativo con repositorio sin nullable analysis.

Este desafío integra todos los conceptos del capítulo en un framework
profesional de acceso a datos.
