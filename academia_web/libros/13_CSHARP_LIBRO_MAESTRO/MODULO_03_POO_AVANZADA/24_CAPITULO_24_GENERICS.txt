¡Bienvenido al Capítulo 24 del C# Master Book! Los genéricos son una de
las características más poderosas de C#. Permiten definir clases,
interfaces, métodos y delegados con tipos parametrizados, posponiendo
la especificación del tipo concreto hasta el momento de uso. Esto
elimina el boxing/unboxing, proporciona seguridad de tipos en tiempo
de compilación y permite escribir código reusable para cualquier tipo.

Desde List<T> hasta Dictionary<K,V>, pasando por interfaces como
IEnumerable<T> y delegados como Func<T>, los genéricos están en todas
partes. En este capítulo aprenderás a crear tus propios tipos genéricos,
aplicar restricciones (constraints) y dominar las técnicas avanzadas
de genericidad.

24.1 — ¿POR QUÉ GENÉRICOS?
----------------------------

Antes de los genéricos (C# 1.0-1.1), las colecciones almacenaban object,
lo que causaba:

1. Falta de seguridad de tipos (podías poner un string en una lista de ints).
2. Boxing/unboxing costoso para tipos de valor.
3. Necesidad de castear constantemente.

```csharp
// Sin genéricos (antes de C# 2.0)
ArrayList lista = new ArrayList();
lista.Add(123);       // boxing
lista.Add("texto");   // error permitido en compilación
int x = (int)lista[0]; // casting necesario

// Con genéricos
List<int> numeros = new List<int>();
numeros.Add(123);      // sin boxing
// numeros.Add("texto"); // error en compilación
int y = numeros[0];     // sin casting
```

24.2 — CLASES GENÉRICAS

```csharp
public class Repositorio<T>
{
    private readonly List<T> _items = new();

    public void Agregar(T item) => _items.Add(item);
    public T Obtener(int index) => _items[index];
    public int Count => _items.Count;
    public bool Eliminar(T item) => _items.Remove(item);
```

```csharp
public class Par<TKey, TValue>
{
    public TKey Llave { get; }
    public TValue Valor { get; }

    public Par(TKey llave, TValue valor)
    {
        Llave = llave;
        Valor = valor;
    }

    public void Mostrar()
    {
        Console.WriteLine($"{Llave}: {Valor}");
    }
}

// Uso
var entrada1 = new Par<int, string>(1, "Uno");
var entrada2 = new Par<string, double>("Pi", 3.1416);
entrada1.Mostrar(); // 1: Uno
entrada2.Mostrar(); // Pi: 3.1416
```

24.4 — MÉTODOS GENÉRICOS
--------------------------

Los métodos pueden ser genéricos aunque la clase no lo sea.

```csharp
public class Utilidades
{
    // Método genérico
    public static T Maximo<T>(T a, T b) where T : IComparable<T>
    {
        return a.CompareTo(b) >= 0 ? a : b;
    }

    // Intercambiar valores
    public static void Intercambiar<T>(ref T a, ref T b)
    {
        T temp = a;
        a = b;
        b = temp;
    }

    // Crear lista desde parámetros
    public static List<T> CrearLista<T>(params T[] items) => new(items);
}

// Uso
int max = Utilidades.Maximo(10, 20);           // 20
string maxStr = Utilidades.Maximo("ABC", "XYZ"); // XYZ

int x = 5, y = 10;
Utilidades.Intercambiar(ref x, ref y);
Console.WriteLine(x); // 10

var lista = Utilidades.CrearLista(1, 2, 3, 4, 5);
```

24.5 — RESTRICCIONES (CONSTRAINTS)
------------------------------------

Las restricciones limitan qué tipos pueden usarse como argumentos de tipo.

```csharp
public class EjemploRestricciones
{
    // T debe ser tipo de referencia (class, string, array, delegate)
    public class ClaseRestriccion<T> where T : class { }

    // T debe ser tipo de valor (struct, enum, int, etc.)
    public class StructRestriccion<T> where T : struct { }

    // T debe tener un constructor sin parámetros
    public class NewRestriccion<T> where T : new() { }

    // T debe implementar una interfaz
    public class InterfazRestriccion<T> where T : IComparable<T> { }

    // T debe heredar de una clase base
    public class BaseRestriccion<T> where T : EventArgs { }

    // Combinación de restricciones
    public class MultiRestriccion<T> where T : class, IComparable<T>, new() { }

    // Restricción para múltiples parámetros
    public class MultiParamRestriccion<TKey, TValue>
        where TKey : class, IComparable<TKey>
        where TValue : struct { }
}

// Restricción not null (C# 8+ con nullable context)
public class NotNullRestriction<T> where T : notnull { }
```

Ejemplos prácticos:

```csharp
// Repositorio con ID comparable
public class RepositorioConId<T, TId>
    where T : IIdentificable<TId>
    where TId : IEquatable<TId>
{
    private readonly Dictionary<TId, T> _items = new();

    public void Agregar(T item) => _items[item.Id] = item;
    public T? Obtener(TId id) => _items.TryGetValue(id, out var item) ? item : default;
    public bool Eliminar(TId id) => _items.Remove(id);
}

public interface IIdentificable<TId>
{
    TId Id { get; }
}

public class Producto : IIdentificable<int>
{
    public int Id { get; set; }
    public string Nombre { get; set; } = "";
}
```

```csharp
public class RepositorioEnMemoria<T> : IRepositorio<T>
    where T : class, IIdentificable<int>
{
    private readonly Dictionary<int, T> _datos = new();
    private int _siguienteId = 1;

    public T Obtener(int id) =>
        _datos.TryGetValue(id, out var item) ? item : null!;

    public IEnumerable<T> Todos() => _datos.Values;

    public void Agregar(T entidad)
    {
        var id = _siguienteId++;
        // Asignar id mediante reflexión o interfaz

    }

    public void Actualizar(T entidad) =>
        _datos[entidad.Id] = entidad;

    public void Eliminar(int id) =>
        _datos.Remove(id);
}
```

24.8 — DELEGADOS GENÉRICOS
----------------------------

```csharp
// Delegados genéricos predefinidos
// Func<T, TResult> — recibe T, devuelve TResult
// Action<T> — recibe T, no devuelve nada
// Predicate<T> — recibe T, devuelve bool

public class EjemploDelegados
{
    public static List<T> Filtrar<T>(List<T> items, Predicate<T> predicate)
    {
        return items.Where(item => predicate(item)).ToList();
    }

    public static T[] Transformar<TIn, TOut>(TIn[] items, Func<TIn, TOut> transformador)
    {
        return items.Select(transformador).ToArray();
    }

    public static void Procesar<T>(List<T> items, Action<T> accion)
    {
        foreach (var item in items) accion(item);
    }
}

// Uso
var numeros = new List<int> { 1, 2, 3, 4, 5, 6 };

var pares = EjemploDelegados.Filtrar(numeros, n => n % 2 == 0);
Console.WriteLine(string.Join(", ", pares)); // 2, 4, 6

var cuadrados = EjemploDelegados.Transformar(numeros.ToArray(), n => n * n);
Console.WriteLine(string.Join(", ", cuadrados)); // 1, 4, 9, 16, 25, 36

EjemploDelegados.Procesar(numeros, n => Console.Write($"{n} ")); // 1 2 3 4 5 6
```

24.9 — COVARIANZA Y CONTRAVARIANZA
------------------------------------

```csharp
// Covarianza: out T — puedo usar un tipo más derivado (IEnumerable<object> = IEnumerable<string>)
// Contravarianza: in T — puedo usar un tipo más base (Action<string> = Action<object>)

// Covarianza (out)
IEnumerable<string> cadenas = new List<string> { "a", "b", "c" };
IEnumerable<object> objetos = cadenas; // OK — IEnumerable<out T>

// Contravarianza (in)
Action<object> accionObj = obj => Console.WriteLine(obj);
Action<string> accionStr = accionObj; // OK — Action<in T>

// Interfaz personalizada
public interface IProductor<out T> // covarianza
{
    T Producir();
}

public interface IConsumidor<in T> // contravarianza
{
    void Consumir(T item);
}

public class Productor<T> : IProductor<T>
{
    private readonly T _item;
    public Productor(T item) => _item = item;
    public T Producir() => _item;
}

public class Consumidor<T> : IConsumidor<T>
{
    public void Consumir(T item) => Console.WriteLine($"Consumido: {item}");
}

// Uso
IProductor<string> prodStr = new Productor<string>("Hola");
IProductor<object> prodObj = prodStr; // covarianza

IConsumidor<object> consObj = new Consumidor<object>();
IConsumidor<string> consStr = consObj; // contravarianza
```

24.12 — PATRÓN REPOSITORY CON GENÉRICOS
-----------------------------------------

```csharp
// Entidad base
public abstract class Entidad<TId>
{
    public TId Id { get; protected set; }

    protected Entidad(TId id) => Id = id;

    public override bool Equals(object? obj) =>
        obj is Entidad<TId> other && Id!.Equals(other.Id);

    public override int GetHashCode() => Id!.GetHashCode();

    public static bool operator ==(Entidad<TId> a, Entidad<TId> b) =>
        a?.Equals(b) ?? b is null;
    public static bool operator !=(Entidad<TId> a, Entidad<TId> b) =>
        !(a == b);
}

// Repositorio genérico
public interface IRepository<T, TId> where T : Entidad<TId>
{
    T? GetById(TId id);
    IEnumerable<T> GetAll();
    void Add(T entity);
    void Update(T entity);
    void Delete(TId id);
    bool Exists(TId id);
}

// Implementación en memoria
public class InMemoryRepository<T, TId> : IRepository<T, TId>
    where T : Entidad<TId>
{
    protected readonly Dictionary<TId, T> _items = new();

    public T? GetById(TId id) =>
        _items.TryGetValue(id, out var item) ? item : null;

    public IEnumerable<T> GetAll() => _items.Values;

    public void Add(T entity) => _items[entity.Id] = entity;

    public void Update(T entity) => _items[entity.Id] = entity;

    public void Delete(TId id) => _items.Remove(id);

    public bool Exists(TId id) => _items.ContainsKey(id);

    public int Count => _items.Count;
}

// Uso concreto
public class UsuarioEntidad : Entidad<int>
{
    public string Nombre { get; set; }
    public string Email { get; set; }

    public UsuarioEntidad(int id, string nombre, string email) : base(id)
    {
        Nombre = nombre;
        Email = email;
    }
}

public class UsuarioRepository : InMemoryRepository<UsuarioEntidad, int>
{
    public UsuarioEntidad? GetByEmail(string email) =>
        _items.Values.FirstOrDefault(u => u.Email == email);

    public IEnumerable<UsuarioEntidad> GetActivos() =>
        _items.Values.Where(u => u.Nombre != "Desactivado");
}
```

```csharp
public interface IFabrica<T>
{
    T Crear();
}

public class FabricaSimple<T> : IFabrica<T> where T : new()
{
    public T Crear() => new();
}

public class FabricaConParametros<T> : IFabrica<T>
{
    private readonly object[] _parametros;
    public FabricaConParametros(params object[] parametros) => _parametros = parametros;

    public T Crear() => (T)Activator.CreateInstance(typeof(T), _parametros)!;
}

public class FabricaRegistrada<T>
{
    private readonly Dictionary<string, Func<T>> _registros = new();

    public void Registrar(string clave, Func<T> factory) =>
        _registros[clave] = factory;

    public T Crear(string clave) =>
        _registros.TryGetValue(clave, out var factory) ? factory() :
            throw new KeyNotFoundException($"No hay fábrica registrada para '{clave}'");
}
```

24.17 — RENDIMIENTO Y GENÉRICOS
---------------------------------

```csharp
public class ComparadorRendimiento
{
    public static long ProbarArrayList(int count)
    {
        var sw = System.Diagnostics.Stopwatch.StartNew();
        var lista = new System.Collections.ArrayList();
        for (int i = 0; i < count; i++)
            lista.Add(i); // boxing
        int suma = 0;
        for (int i = 0; i < count; i++)
            suma += (int)lista[i]!; // unboxing
        sw.Stop();
        return sw.ElapsedMilliseconds;
    }

    public static long ProbarListGeneric(int count)
    {
        var sw = System.Diagnostics.Stopwatch.StartNew();
        var lista = new List<int>();
        for (int i = 0; i < count; i++)
            lista.Add(i); // sin boxing
        int suma = 0;
        for (int i = 0; i < count; i++)
            suma += lista[i]; // sin unboxing
        sw.Stop();
        return sw.ElapsedMilliseconds;
    }
}

// Los genéricos son significativamente más rápidos para tipos de valor
// debido a la eliminación de boxing/unboxing
```

24.18 — EJEMPLO COMPLETO: SISTEMA DE CACHÉ GENÉRICO
------------------------------------------------------

```csharp
using System;
using System.Collections.Concurrent;
using System.Collections.Generic;
using System.Linq;

public class Cache<TKey, TValue> where TKey : notnull
{
    private readonly ConcurrentDictionary<TKey, CacheItem<TValue>> _cache = new();
    private readonly TimeSpan _tiempoVida;
    private readonly int _maxItems;

    public Cache(TimeSpan tiempoVida, int maxItems = 10000)
    {
        _tiempoVida = tiempoVida;
        _maxItems = maxItems;
    }

    public TValue? Get(TKey key)
    {
        if (_cache.TryGetValue(key, out var item) && !item.Expirado)
        {
            item.UltimoAcceso = DateTime.UtcNow;
            return item.Valor;
        }
        if (item?.Expirado == true)
            _cache.TryRemove(key, out _);
        return default;
    }

    public void Set(TKey key, TValue value)
    {
        if (_cache.Count >= _maxItems)
            LimpiarExpirados();

        _cache[key] = new CacheItem<TValue>(value, DateTime.UtcNow.Add(_tiempoVida));
    }

    public bool Contains(TKey key) =>
        _cache.TryGetValue(key, out var item) && !item.Expirado;

    public void Remove(TKey key) => _cache.TryRemove(key, out _);

    public void Clear() => _cache.Clear();

    public int Count => _cache.Count;

    private void LimpiarExpirados()
    {
        var expirados = _cache.Where(kvp => kvp.Value.Expirado).ToList();
        foreach (var kvp in expirados)
            _cache.TryRemove(kvp.Key, out _);
    }

    public IEnumerable<TKey> KeysActivas =>
        _cache.Where(kvp => !kvp.Value.Expirado).Select(kvp => kvp.Key);

    // Estadísticas
    public int ItemsExpirados =>
        _cache.Count(kvp => kvp.Value.Expirado);

    public int ItemsActivos =>
        _cache.Count(kvp => !kvp.Value.Expirado);

    private class CacheItem<T>
    {
        public T Valor { get; }
        public DateTime Expiracion { get; private set; }
        public DateTime UltimoAcceso { get; set; }
        public bool Expirado => DateTime.UtcNow >= Expiracion;

        public CacheItem(T valor, DateTime expiracion)
        {
            Valor = valor;
            Expiracion = expiracion;
            UltimoAcceso = DateTime.UtcNow;
        }
    }
}

// Restricción con tipo enum
public class EnumCache<T> where T : Enum
{
    private static readonly Dictionary<string, T> _porNombre =
        Enum.GetValues<T>().ToDictionary(e => e.ToString(), e => e);

    private static readonly Dictionary<T, string> _porValor =
        Enum.GetValues<T>().ToDictionary(e => e, e => e.ToString());

    public static T? FromName(string name, bool ignoreCase = false)
    {
        if (ignoreCase)
            return _porNombre.FirstOrDefault(kvp =>
                string.Equals(kvp.Key, name, StringComparison.OrdinalIgnoreCase)).Value;
        return _porNombre.GetValueOrDefault(name);
    }

    public static string? FromValue(T value) =>
        _porValor.GetValueOrDefault(value);
}

public class Programa
{
    public static void Main()
    {
        // Cache de productos
        var cache = new Cache<int, string>(TimeSpan.FromMinutes(5));

        cache.Set(1, "Laptop Gamer");
        cache.Set(2, "Monitor 4K");
        cache.Set(3, "Teclado Mecánico");

        Console.WriteLine(cache.Get(1)); // Laptop Gamer
        Console.WriteLine(cache.Contains(4)); // False

        cache.Remove(2);
        Console.WriteLine(cache.Contains(2)); // False

        // Demostración de constraints
        var repoInt = new Repositorio<int>();
        repoInt.Agregar(10);

        // EnumHelper con restricción
        var dias = EnumHelper<DayOfWeek>.GetValues();
        Console.WriteLine(string.Join(", ", dias));

        // Factory genérico
        var factory = new FabricaSimple<List<int>>();
        var lista = factory.Crear();
        Console.WriteLine(lista.Count); // 0

        // Medir rendimiento
        Console.WriteLine($"\nArrayList: {ComparadorRendimiento.ProbarArrayList(1000000)} ms");
        Console.WriteLine($"List<int>: {ComparadorRendimiento.ProbarListGeneric(1000000)} ms");
    }
}
```

│  restricciones (where) para limitar los tipos permitidos.           │
│  Interfaces como IEnumerable<T> usan covarianza (out) y             │
│  contravarianza (in) para flexibilidad. Los genéricos son           │
│  fundamentales en colecciones, repositorios, fábricas y cualquier   │
│  código que deba funcionar con múltiples tipos.                     │
└──────────────────────────────────────────────────────────────────────┘

================================================================================
  FAQ — PREGUNTAS FRECUENTES
================================================================================

P: ¿Puedo usar operadores como + o == con tipos genéricos?
R: No directamente. No hay restricción que garantice que T tenga un
   operador +. Usa interfaces o delegados para resolverlo.

P: ¿Qué es la reificación en genéricos?
R: En .NET, los genéricos son reificados: el tipo real se conserva en
   tiempo de ejecución (typeof(T) funciona). En Java hay type erasure.

P: ¿Puedo hacer new T() sin restricción?
R: No. Necesitas la restricción where T : new() para usar new T().

P: ¿Los structs genéricos tienen costo de rendimiento?
R: Muy poco. El JIT compila código especializado para cada tipo de valor,
   mientras que los tipos de referencia comparten una implementación.

P: ¿Puedo usar var con tipos genéricos?
R: Sí: var lista = new List<int>();

P: ¿Qué diferencia hay entre List<T> y ArrayList?
R: List<T> es genérico (seguro, sin boxing). ArrayList no es genérico
   (usa object, requiere casting y boxing).

P: ¿Puedo tener un método genérico estático en una clase no genérica?
R: Sí, los métodos genéricos pueden existir en cualquier clase.

P: ¿Qué son los genéricos abiertos y cerrados?
R: Abierto: List<>. Cerrado: List<int>. Solo los cerrados pueden
   instanciarse.

================================================================================
  DESAFÍO DEL ARQUITECTO
================================================================================

Diseña un framework de mapeo objeto-objeto (Object Mapper) genérico
similar a AutoMapper utilizando genéricos, expresiones y reflexión.
El framework debe:

- Clase principal: Mapper<TSource, TDestination>.
- Método Map(TSource source): TDestination.
- Soporte para mapeo automático por nombre de propiedades.
- Configuración manual con ForMember(dest => dest.Propiedad, opt => opt.MapFrom(src => src.Otra)).
- Ignorar propiedades con Ignore().
- Mapeo de colecciones (List<TDest> MapList(List<TSrc>)).
- Mapeo anidado (propiedades que son objetos).
- Soporte para tipos anónimos.
- Performance: usar compilación de expresiones (Expression Trees)
  para que el mapeo sea rápido después de la primera configuración.
- Validación: advertir si hay propiedades en TDestination que no
  se mapean en la configuración.
- Perfiles (profiles): agrupar configuraciones de mapeo.
- Mapeo condicional: MapWhen(src => src.Valor > 0, dest => dest.Destino).
- Benchmarks comparativos con mapeo manual y con AutoMapper real.

Este desafío integra genéricos, reflexión, expresiones lambda,
delegados y patrones de configuración en un solo proyecto.
