--------------------------------------------------------------------
  1. ¿QUÉ ES LA PROGRAMACIÓN ORIENTADA A OBJETOS?
--------------------------------------------------------------------

La Programación Orientada a Objetos (POO) es un paradigma de programación que
organiza el código en torno a "objetos" en lugar de funciones y procedimientos
lineales. Un objeto es una entidad que combina:

  • Estado: datos almacenados en campos (variables internas).
  • Comportamiento: acciones que puede realizar, definidas en métodos.

Los cuatro pilares de la POO son:

  1. Abstracción:  Modelar entidades del mundo real ignorando detalles
                   irrelevantes y enfocándose en lo esencial.
  2. Encapsulación: Ocultar los detalles internos de un objeto y exponer
                   solo lo necesario mediante una interfaz pública.
  3. Herencia:     Crear nuevas clases basadas en clases existentes,
                   reutilizando y extendiendo su comportamiento.
  4. Polimorfismo:  Hacer que objetos de diferentes tipos respondan al
                   mismo mensaje de formas distintas.

En C#, TODO es un objeto. Incluso los tipos primitivos como `int` y `bool`
tienen métodos y propiedades porque heredan de `System.Object`.

--------------------------------------------------------------------
  2. ¿QUÉ ES UNA CLASE?
--------------------------------------------------------------------

Una clase es la definición abstracta de un concepto. Es como un molde o un
plano arquitectónico. Define:

  • Los atributos (campos/propiedades) que tendrá cada objeto.
  • Los métodos (comportamientos) que cada objeto podrá ejecutar.

La clase NO ocupa memoria hasta que se crea una instancia de ella (un objeto).

Sintaxis básica de definición de clase:

  [modificadores] class NombreDeLaClase
  {
      // Campos (variables internas)
      // Propiedades
      // Métodos
      // Constructores
      // Eventos, etc.
  }

Ejemplo:

```csharp
public class Persona
{
    // Campos (atributos)
    public string nombre;
    public int edad;

    // Método (comportamiento)
    public void Saludar()
    {
        Console.WriteLine($"Hola, soy {nombre} y tengo {edad} años.");
    }
}
```

--------------------------------------------------------------------
  3. ¿QUÉ ES UN OBJETO?
--------------------------------------------------------------------

Un objeto es una INSTANCIA concreta de una clase. Es la materialización del
plano. Ocupa memoria en el heap y tiene su propia copia de los campos.

Creación de un objeto (instanciación):

  NombreClase variable = new NombreClase();

  • `new`: operador que reserva memoria en el heap e inicializa el objeto.
  • La variable guarda una REFERENCIA (dirección de memoria), no el objeto
    en sí.

Ejemplo:

```csharp
Persona persona1 = new Persona();
persona1.nombre = "Ana";
persona1.edad = 30;
persona1.Saludar();  // Salida: Hola, soy Ana y tengo 30 años.
```

Cada objeto es independiente. Si creas dos objetos, cada uno tiene su propio
conjunto de campos:

```csharp
Persona p1 = new Persona();
Persona p2 = new Persona();

p1.nombre = "Ana";
p2.nombre = "Luis";

Console.WriteLine(p1.nombre);  // Ana
Console.WriteLine(p2.nombre);  // Luis
```

--------------------------------------------------------------------
  4. CAMPOS (FIELDS)
--------------------------------------------------------------------

Los campos son variables declaradas directamente dentro de la clase.
Almacenan el estado del objeto.

Sintaxis:

  [modificadores] tipo nombreCampo;

Tipos de campos:

  • Campos de instancia:  cada objeto tiene su propia copia.
  • Campos estáticos:     compartidos por todos los objetos de la clase
                          (se declaran con `static`).
  • Campos de solo lectura: se asignan solo en la declaración o en el
                          constructor (con `readonly`).
  • Campos constantes:    valor fijo en tiempo de compilación (con `const`).

Ejemplo completo:

```csharp
public class Empleado
{
    // Campos de instancia
    public string nombre;
    public double salario;

    // Campo estático (compartido)
    public static int contadorEmpleados;

    // Campo readonly (solo se asigna una vez)
    public readonly int idUnico;

    // Campo const (constante de clase)
    public const double SALARIO_MINIMO = 15000;
}
```

Los campos deben inicializarse antes de usarse. Si no se inicializan
explícitamente, el compilador les asigna valores predeterminados:
  • 0 para tipos numéricos
  • false para bool
  • null para tipos referencia

```csharp
public class PruebaCampos
{
    public int entero;         // 0 por defecto
    public double decimal;     // 0.0 por defecto
    public bool bandera;       // false por defecto
    public string texto;       // null por defecto
    public char caracter;      // '\0' por defecto
}
```

--------------------------------------------------------------------
  5. MÉTODOS (METHODS)
--------------------------------------------------------------------

Los métodos definen el comportamiento de los objetos. Son bloques de código
que realizan una tarea específica.

Sintaxis:

  [modificadores] tipoRetorno NombreMetodo([parámetros])
  {
      // Cuerpo del método
      return valor;  // si el tipo de retorno no es void
  }

Tipos de métodos:

  • Métodos de instancia: operan sobre una instancia específica.
  • Métodos estáticos:    pertenecen a la clase, no a la instancia.
  • Métodos virtuales:    pueden ser sobreescritos en clases derivadas.
  • Métodos abstractos:   declarados sin implementación, deben ser
                          implementados en clases derivadas.

Ejemplo de métodos de instancia:

```csharp
public class Calculadora
{
    // Método con retorno
    public int Sumar(int a, int b)
    {
        return a + b;
    }

    // Método void (sin retorno)
    public void MostrarResultado(int resultado)
    {
        Console.WriteLine($"El resultado es: {resultado}");
    }

    // Método con parámetros opcionales
    public int Multiplicar(int a, int b = 1)
    {
        return a * b;
    }

    // Método con parámetros con nombre (out)
    public bool Dividir(int a, int b, out double resultado)
    {
        if (b == 0)
        {
            resultado = 0;
            return false;
        }
        resultado = (double)a / b;
        return true;
    }
}
```

Uso:

```csharp
Calculadora calc = new Calculadora();
int suma = calc.Sumar(5, 3);
calc.MostrarResultado(suma);             // El resultado es: 8

int mult = calc.Multiplicar(4, 3);       // 12
int mult2 = calc.Multiplicar(4);         // 4 (usa b=1 por defecto)

double division;
bool exito = calc.Dividir(10, 3, out division);
if (exito) Console.WriteLine(division);  // 3.333...
```

--------------------------------------------------------------------
  6. EL OPERADOR `new`
--------------------------------------------------------------------

El operador `new` es la puerta de entrada a la creación de objetos. Realiza
varias tareas críticas:

  1. Reserva memoria en el heap para el objeto.
  2. Inicializa los campos a sus valores predeterminados.
  3. Llama al constructor (si existe) para personalizar la inicialización.
  4. Devuelve una referencia al objeto creado.

Sintaxis:

  Tipo variable = new Tipo([argumentos]);

El operador `new` también puede usarse para:

  • Inicializar arrays: `int[] numeros = new int[5];`
  • Crear tipos anónimos: `var obj = new { Nombre = "Ana", Edad = 30 };`

Importante: Sin `new`, la variable es `null` y cualquier intento de acceder
a sus miembros lanzará `NullReferenceException`.

```csharp
public class Cliente
{
    public string nombre;
    public void Mostrar() => Console.WriteLine(nombre);
}

// Correcto:
Cliente c1 = new Cliente();
c1.nombre = "María";
c1.Mostrar();  // María

// Incorrecto (NullReferenceException):
Cliente c2 = null;
c2.nombre = "Juan";  // ERROR: NullReferenceException
```

--------------------------------------------------------------------
  7. MEMORIA: HEAP vs STACK
--------------------------------------------------------------------

Entender la memoria en .NET es crucial para escribir código eficiente y evitar
fugas de memoria o errores de referencia.

┌─────────────────────────────────────────────────────────────────┐
│                         MEMORIA EN .NET                         │
├──────────────┬──────────────────────────────────────────────────┤
│    STACK     │                      HEAP                        │
├──────────────┼──────────────────────────────────────────────────┤
│ Tipos valor  │ Tipos referencia (objetos)                       │
│ (struct,     │ (class, array, string, delegate)                 │
│  int, bool,  │                                                  │
│  double, etc)│                                                  │
├──────────────┼──────────────────────────────────────────────────┤
│ LIFO         │ Asignación dinámica                              │
│ Rápido       │ Más lento (requiere GC)                         │
│ Tamaño       │ Capacidad grande                                  │
│ limitado     │                                                   │
├──────────────┼──────────────────────────────────────────────────┤
│ Se libera al │ Se libera por el Garbage Collector (GC)          │
│ salir del    │ cuando no hay referencias                        │
│ ámbito       │                                                   │
└──────────────┴──────────────────────────────────────────────────┘

Stack:
  • Almacena variables locales y parámetros.
  • Cada llamada a método crea un nuevo "stack frame".
  • Las variables de tipos valor (int, double, bool, struct) se
    almacenan directamente aquí.
  • La memoria se gestiona automáticamente: al salir del ámbito,
    se libera.

Heap:
  • Almacena objetos (instancias de clases).
  • La variable en stack contiene una REFERENCIA (dirección) al objeto
    en heap.
  • El Garbage Collector (GC) gestiona la liberación de memoria cuando
    ya no hay referencias al objeto.

Ejemplo visual:

```csharp
public void Ejemplo()
{
    int a = 10;               // a está en el STACK
    string s = "hola";        // s (referencia) en STACK, "hola" en HEAP
    Persona p = new Persona(); // p (referencia) en STACK, objeto en HEAP
}
```

Representación:

  STACK:
  ┌──────────────┐
  │ a = 10       │
  ├──────────────┤
  │ s = ref ─────┼─────> HEAP: "hola"
  ├──────────────┤
  │ p = ref ─────┼─────> HEAP: Persona { nombre: null, edad: 0 }
  └──────────────┘

Cuando un método termina, su stack frame se elimina. Si no hay más
referencias a un objeto en heap, el GC lo recolecta.

```csharp
public void DemostracionMemoria()
{
    Persona p1 = new Persona();  // Se crea objeto en heap
    Persona p2 = p1;            // p2 apunta al MISMO objeto

    p1.nombre = "Ana";
    Console.WriteLine(p2.nombre);  // Ana (mismo objeto)

    p1 = null;  // p1 ya no referencia al objeto
    // p2 aún referencia al objeto → NO se recolecta

    p2 = null;  // Ahora no hay referencias → el GC puede recolectarlo
}
```

--------------------------------------------------------------------
  8. CICLO DE VIDA DE UN OBJETO
--------------------------------------------------------------------

Todo objeto en C# sigue este ciclo:

  1. Declaración:   Se declara una variable del tipo de la clase.
  2. Instanciación: Se usa `new` para reservar memoria e inicializar.
  3. Uso:           Se accede a sus campos, métodos y propiedades.
  4. Liberación:    Cuando no hay referencias, el GC lo recolecta.

```csharp
public void CicloDeVida()
{
    // 1 y 2: Declaración e instanciación
    Pedido pedido = new Pedido();

    // 3: Uso
    pedido.Producto = "Laptop";
    pedido.Cantidad = 2;
    decimal total = pedido.CalcularTotal();
    Console.WriteLine($"Total: {total}");

    // 4: Al salir del método, pedido sale de ámbito
    // Si no hay otras referencias, el GC recolecta el objeto
}
```

--------------------------------------------------------------------
  9. EJEMPLO COMPLETO: SISTEMA DE BIBLIOTECA
--------------------------------------------------------------------

```csharp
using System;

namespace SistemaBiblioteca
{
    // Definición de la clase Libro
    public class Libro
    {
        // Campos (atributos)
        public string titulo;
        public string autor;
        public int anioPublicacion;
        public bool prestado;
        public static int totalLibros = 0;

        // Métodos
        public void Prestar()
        {
            if (prestado)
            {
                Console.WriteLine($"'{titulo}' ya está prestado.");
            }
            else
            {
                prestado = true;
                Console.WriteLine($"'{titulo}' ha sido prestado.");
            }
        }

        public void Devolver()
        {
            if (!prestado)
            {
                Console.WriteLine($"'{titulo}' no estaba prestado.");
            }
            else
            {
                prestado = false;
                Console.WriteLine($"'{titulo}' ha sido devuelto.");
            }
        }

        public void MostrarInformacion()
        {
            Console.WriteLine($"Título: {titulo}");
            Console.WriteLine($"Autor: {autor}");
            Console.WriteLine($"Año: {anioPublicacion}");
            Console.WriteLine($"Estado: {(prestado ? "Prestado" : "Disponible")}");
            Console.WriteLine();
        }

        public static void MostrarTotalLibros()
        {
            Console.WriteLine($"Total de libros en la biblioteca: {totalLibros}");
        }
    }

    public class Programa
    {
        public static void Main()
        {
            // Crear objetos (instanciar)
            Libro libro1 = new Libro();
            libro1.titulo = "Cien Años de Soledad";
            libro1.autor = "Gabriel García Márquez";
            libro1.anioPublicacion = 1967;
            Libro.totalLibros++;

            Libro libro2 = new Libro();
            libro2.titulo = "1984";
            libro2.autor = "George Orwell";
            libro2.anioPublicacion = 1949;
            Libro.totalLibros++;

            // Usar los objetos
            libro1.MostrarInformacion();
            libro2.MostrarInformacion();

            libro1.Prestar();
            libro1.Prestar();  // Intento de préstamo doble
            libro1.Devolver();
            libro1.Devolver();  // Intento de devolución doble

            Libro.MostrarTotalLibros();

            // Demostración de referencias
            Libro mismaRef = libro1;
            mismaRef.titulo = "Cien Años de Soledad (Edición Especial)";
            Console.WriteLine($"El título original también cambió: {libro1.titulo}");
        }
    }
}
```

Salida esperada:

  Título: Cien Años de Soledad
  Autor: Gabriel García Márquez
  Año: 1967
  Estado: Disponible

  Título: 1984
  Autor: George Orwell
  Año: 1949
  Estado: Disponible

  'Cien Años de Soledad' ha sido prestado.
  'Cien Años de Soledad' ya está prestado.
  'Cien Años de Soledad' ha sido devuelto.
  'Cien Años de Soledad' no estaba prestado.
  Total de libros en la biblioteca: 2
  El título original también cambió: Cien Años de Soledad (Edición Especial)

  10. BUENAS PRÁCTICAS
--------------------------------------------------------------------

  ✓ Usa nombres significativos para clases (sustantivos, PascalCase).
  ✓ Mantén las clases con una sola responsabilidad (Principio SRP).
  ✓ Inicializa los campos en el constructor, no en la declaración.
  ✓ Prefiere propiedades públicas sobre campos públicos.
  ✓ Usa `readonly` para campos que no deben cambiar tras la construcción.
  ✓ Documenta las clases con comentarios XML.
  ✓ No expongas campos internos directamente; usa encapsulación.

```csharp
// Bien
public class Producto
{
    public string Nombre { get; set; }  // Propiedad, no campo público
    public decimal Precio { get; set; }
}

// Mal
public class Producto
{
    public string nombre;  // Campo público (rompe encapsulación)
    public decimal precio;
}
```

  ✓ Declara variables con el tipo más específico posible.
  ✓ Usa `var` solo cuando el tipo sea obvio por el lado derecho.

```csharp
// Bien
var lista = new List<int>();
var producto = new Producto();

// Evitar
var x = 42;  // Preferir explícito en tipos simples
```

--------------------------------------------------------------------
  12. CLASES PARCIALES (partial)
--------------------------------------------------------------------

C# permite dividir la definición de una clase en múltiples archivos usando
la palabra clave `partial`. Es útil en proyectos grandes o código generado.

```csharp
// Archivo 1: Persona.Parte1.cs
public partial class Persona
{
    public string Nombre { get; set; }
}

// Archivo 2: Persona.Parte2.cs
public partial class Persona
{
    public void Saludar()
    {
        Console.WriteLine($"Hola, soy {Nombre}");
    }
}
```

El compilador combina ambas partes en una sola clase.

--------------------------------------------------------------------
  13. CLASES ANIDADAS
--------------------------------------------------------------------

Una clase puede definirse dentro de otra clase. Se usa para agrupar
lógicamente clases que solo tienen sentido en el contexto de la clase
externa.

```csharp
public class Universidad
{
    public string Nombre { get; set; }

    // Clase anidada
    public class Departamento
    {
        public string Nombre { get; set; }
        public int CantidadProfesores { get; set; }
    }
}

// Uso
Universidad uni = new Universidad();
uni.Nombre = "UNI";
Universidad.Departamento depto = new Universidad.Departamento();
depto.Nombre = "Ciencias de la Computación";
```

--------------------------------------------------------------------
  14. CAMPOS ESTÁTICOS Y MÉTODOS ESTÁTICOS
--------------------------------------------------------------------

Los miembros estáticos pertenecen a la clase, no a la instancia. Se accede
a ellos mediante el nombre de la clase, no mediante un objeto.

```csharp
public class Configuracion
{
    // Campo estático
    public static string Version = "1.0.0";

    // Campo estático readonly
    public static readonly DateTime FechaCompilacion = DateTime.Now;

    // Método estático
    public static void MostrarVersion()
    {
        Console.WriteLine($"Versión: {Version}");
    }
}

// Uso (sin crear ninguna instancia)
Console.WriteLine(Configuracion.Version);
Configuracion.MostrarVersion();
```

Características de miembros estáticos:

  • Existen desde que se carga la clase hasta que finaliza el programa.
  • Son compartidos por todas las instancias.
  • Los métodos estáticos solo pueden acceder a otros miembros estáticos
    directamente (no pueden acceder a campos de instancia sin un objeto).

```csharp
public class EjemploEstatico
{
    public int instancia = 10;
    public static int estatico = 20;

    public void MetodoInstancia()
    {
        Console.WriteLine(instancia);  // OK
        Console.WriteLine(estatico);   // OK
    }

    public static void MetodoEstatico()
    {
        // Console.WriteLine(instancia);  // ERROR: no hay 'this'
        Console.WriteLine(estatico);   // OK
    }
}
```

--------------------------------------------------------------------
  15. EL TIPO `System.Object`
--------------------------------------------------------------------

Todas las clases en C# heredan implícitamente de `System.Object`. Esto
significa que toda clase tiene los siguientes miembros:

  • `Equals(object obj)`:        Compara si dos objetos son iguales.
  • `GetHashCode()`:            Obtiene un código hash del objeto.
  • `ToString()`:               Representación en string del objeto.
  • `GetType()`:                Obtiene el tipo exacto del objeto.
  • `MemberwiseClone()`:        Crea una copia superficial (protegido).

```csharp
public class Persona
{
    public string Nombre { get; set; }
}

// Toda clase tiene estos métodos:
Persona p1 = new Persona { Nombre = "Ana" };
Persona p2 = new Persona { Nombre = "Ana" };

Console.WriteLine(p1.ToString());       // Persona (nombre de la clase)
Console.WriteLine(p1.GetType());        // Persona
Console.WriteLine(p1.Equals(p2));       // False (referencias distintas)
Console.WriteLine(p1.GetHashCode());    // Código hash único
```

Podemos SOBREESCRIBIR estos métodos para darles comportamiento específico:

```csharp
public class Persona
{
    public string Nombre { get; set; }

    public override string ToString()
    {
        return $"Persona: {Nombre}";
    }

    public override bool Equals(object obj)
    {
        if (obj is Persona otra)
            return this.Nombre == otra.Nombre;
        return false;
    }

    public override int GetHashCode()
    {
        return Nombre?.GetHashCode() ?? 0;
    }
}

// Ahora:
Persona p1 = new Persona { Nombre = "Ana" };
Persona p2 = new Persona { Nombre = "Ana" };

Console.WriteLine(p1.ToString());       // Persona: Ana
Console.WriteLine(p1.Equals(p2));       // True (comparan por nombre)
```

  RESUMEN DEL CAPÍTULO
================================================================================

  • Una CLASE es un plano o molde que define la estructura y el
    comportamiento de los objetos.

  • Un OBJETO es una instancia concreta de una clase; ocupa memoria en
    el heap y tiene su propia copia de los campos de instancia.

  • La palabra clave `new` reserva memoria en el heap, inicializa el
    objeto y devuelve una referencia.

  • Los CAMPOS almacenan el estado del objeto; pueden ser de instancia,
    estáticos, readonly o constantes.

  • Los MÉTODOS definen el comportamiento; pueden ser de instancia o
    estáticos, con retorno o void, con parámetros opcionales o out.

  • La MEMORIA se divide en STACK (tipos valor, rápido, tamaño limitado)
    y HEAP (tipos referencia, gestionado por el GC).

  • Los miembros ESTÁTICOS pertenecen a la clase, no a la instancia.

  • Toda clase hereda de `System.Object` y puede sobreescribir sus
    métodos (ToString, Equals, GetHashCode).

  • Sin `new`, la variable es null y produce NullReferenceException.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ❓ ¿Puedo tener una clase sin ningún campo ni método?
  ───
  Sí, es una clase vacía. Puede ser útil como marcador de posición o
  para usos avanzados como atributos o tipos genéricos.

  ```csharp
  public class Marcador { }
  ```

  ❓ ¿Qué diferencia hay entre `class` y `struct`?
  ───
  `class` es tipo referencia (heap, GC, null permitido, herencia).
  `struct` es tipo valor (stack, no null por defecto, no herencia,
  copia por valor). Se explica a detalle más adelante.

  ❓ ¿Por qué no debo usar campos públicos?
  ───
  Los campos públicos rompen la encapsulación. Cualquier código puede
  modificar el estado interno del objeto sin control. Las propiedades
  permiten validación y control de acceso.

  ❓ ¿Qué pasa si no uso `new`?
  ───
  La variable vale `null` y cualquier acceso a sus miembros lanza
  `NullReferenceException`.

  ❓ ¿Los métodos estáticos son más rápidos?
  ───
  Marginalmente, porque no requieren el despacho virtual ni la
  comprobación de null. Pero la diferencia es ínfima; úsalos cuando
  tengan sentido semántico (comportamiento sin estado de instancia).

  ❓ ¿Cuándo se llama al Garbage Collector?
  ───
  El GC se ejecuta automáticamente cuando .NET detecta presión de
  memoria. No se puede predecir exactamente cuándo. Se puede forzar
  con `GC.Collect()` pero no se recomienda.

  ❓ ¿Qué es una referencia?
  ───
  Es una dirección de memoria que "apunta" al objeto en el heap.
  Múltiples variables pueden referenciar el mismo objeto.
