================================================================================
  BIENVENIDA
================================================================================

La herencia es uno de los cuatro pilares de la Programación Orientada a
Objetos. Permite crear nuevas clases basadas en clases existentes,
heredando sus campos, propiedades, métodos y comportamientos.

En C#, la herencia se expresa con el operador `:` y es la base de todo el
sistema de tipos: cada clase que creas hereda implícitamente de
System.Object.

En este capítulo exploraremos:

  • Sintaxis de herencia con `: base`.
  • Constructores en jerarquías de herencia.
  • La palabra clave `sealed` para evitar herencia.
  • Los operadores `is` y `as` para comprobación de tipos.
  • Conversiones (casting) entre tipos base y derivados.
  • El polimorfismo (preparación para el próximo capítulo).
  • Buenas prácticas con herencia.

La herencia bien aplicada permite código reusable, extensible y bien
organizado. Mal aplicada, puede crear jerarquías frágiles y difíciles de
mantener. Aprendamos a usarla correctamente.

--------------------------------------------------------------------
  1. ¿QUÉ ES LA HERENCIA?
--------------------------------------------------------------------

La herencia es un mecanismo que permite crear una nueva clase (clase
derivada, subclase) a partir de una clase existente (clase base,
superclase).

La clase derivada:
  • HEREDA todos los miembros públicos, protected e internal.
  • NO hereda constructores, destructores ni miembros privados.
  • PUEDE agregar nuevos miembros.
  • PUEDE sobrescribir miembros virtuales.

Relación "es-un": la clase derivada ES UN tipo de la clase base.

```csharp
// Clase base
public class Animal
{
    public string Nombre { get; set; }
    public int Edad { get; set; }

    public void Comer()
    {
        Console.WriteLine($"{Nombre} esta comiendo.");
    }

    public void Dormir()
    {
        Console.WriteLine($"{Nombre} esta durmiendo.");
    }
}

// Clase derivada (hereda de Animal)
public class Perro : Animal
{
    public void Ladrar()
    {
        Console.WriteLine($"{Nombre} dice: ¡Guau!");
    }
}

// Clase derivada (hereda de Animal)
public class Gato : Animal
{
    public void Maullar()
    {
        Console.WriteLine($"{Nombre} dice: ¡Miau!");
    }
}

// Uso:
Perro perro = new Perro();
perro.Nombre = "Fido";
perro.Edad = 3;
perro.Comer();    // Heredado de Animal
perro.Dormir();   // Heredado de Animal
perro.Ladrar();   // Propio de Perro

Gato gato = new Gato();
gato.Nombre = "Misu";
gato.Maullar();   // Propio de Gato
```

Beneficios de la herencia:

  ✓ REUTILIZACIÓN: el código común está en la clase base.
  ✓ EXTENSIBILIDAD: puedes agregar comportamiento específico.
  ✓ POLIMORFISMO: tratar objetos derivados como base.
  ✓ ORGANIZACIÓN: jerarquías lógicas de tipos.

--------------------------------------------------------------------
  2. SINTAXIS DE HERENCIA
--------------------------------------------------------------------

Sintaxis básica:

  class ClaseDerivada : ClaseBase
  {
      // Nuevos miembros
  }

C# soporta HERENCIA SIMPLE: una clase solo puede heredar directamente
de una clase base. Pero puede implementar múltiples interfaces.

```csharp
public class Base { }
public class Derivada : Base { }  // Herencia simple

// public class Derivada : Base, OtraClase { }  // ERROR: herencia multiple
// public class Derivada : Base, IComparable, IDisposable { }  // OK: interfaces
```

--------------------------------------------------------------------
  3. LA CLASE BASE UNIVERSAL: System.Object
--------------------------------------------------------------------

Toda clase en C# hereda implícitamente de System.Object. Incluso si no
escribes ` : object`, el compilador lo agrega.

```csharp
// Estas dos declaraciones son equivalentes:
public class MiClase { }
public class MiClase : object { }

// System.Object proporciona:
public class EjemploObject
{
    public override string ToString() => "Ejemplo";
    public override bool Equals(object obj) => base.Equals(obj);
    public override int GetHashCode() => base.GetHashCode();
    public Type GetType() => base.GetType();
    // protected object MemberwiseClone() => ...;
}
```

--------------------------------------------------------------------
  4. CONSTRUCTORES EN HERENCIA
--------------------------------------------------------------------

Los constructores NO se heredan. Cada clase debe definir sus propios
constructores. La clase derivada DEBE llamar a un constructor de la
clase base usando `: base()`.

Si no se especifica, el compilador intenta llamar al constructor sin
parámetros de la clase base. Si no existe, da error.

```csharp
public class Persona
{
    public string Nombre { get; set; }
    public int Edad { get; set; }

    public Persona(string nombre, int edad)
    {
        Nombre = nombre;
        Edad = edad;
        Console.WriteLine($"Persona creada: {nombre}");
    }
}

public class Empleado : Persona
{
    public string Puesto { get; set; }
    public decimal Salario { get; set; }

    public Empleado(string nombre, int edad, string puesto, decimal salario)
        : base(nombre, edad)  // Llama al constructor de Persona
    {
        Puesto = puesto;
        Salario = salario;
        Console.WriteLine($"Empleado creado: {nombre} - {puesto}");
    }
}

// Uso:
Empleado emp = new Empleado("Ana", 30, "Ingeniera", 50000);
// Salida:
// Persona creada: Ana
// Empleado creado: Ana - Ingeniera
```

Flujo de ejecución de constructores:

  1. Se ejecutan los inicializadores de campo de la clase DERIVADA.
  2. Se ejecuta el constructor de la clase BASE.
  3. Se ejecuta el cuerpo del constructor de la DERIVADA.

```csharp
public class A
{
    public A()
    {
        Console.WriteLine("Constructor A");
    }
}

public class B : A
{
    public B() : base()
    {
        Console.WriteLine("Constructor B");
    }
}

public class C : B
{
    public C() : base()
    {
        Console.WriteLine("Constructor C");
    }
}

// new C() imprime:
// Constructor A
// Constructor B
// Constructor C
```

Si la clase base no tiene constructor sin parámetros, la derivada DEBE
llamar explícitamente a un constructor con parámetros:

```csharp
public class BaseSinDefault
{
    public string Valor { get; }

    public BaseSinDefault(string valor)
    {
        Valor = valor;
    }
}

public class Derivada : BaseSinDefault
{
    public Derivada() : base("Valor por defecto") { }
    public Derivada(string v) : base(v) { }
}
```

--------------------------------------------------------------------
  5. LA PALABRA CLAVE `base`
--------------------------------------------------------------------

La palabra clave `base` tiene dos usos:

  1. Llamar a un constructor de la clase base (`: base()`).
  2. Acceder a miembros de la clase base desde la derivada.

```csharp
public class Animal
{
    public string Nombre { get; set; }

    public virtual void HacerSonido()
    {
        Console.WriteLine("Sonido generico de animal.");
    }
}

public class Perro : Animal
{
    public override void HacerSonido()
    {
        base.HacerSonido();  // Llama al metodo de la base
        Console.WriteLine("¡Guau guau!");
    }
}

// Uso:
Perro p = new Perro();
p.HacerSonido();
// Salida:
// Sonido generico de animal.
// ¡Guau guau!
```

`base` solo puede usarse dentro de un constructor o un método de instancia
(no en métodos estáticos). Y solo accede al nivel inmediato de herencia.

--------------------------------------------------------------------
  6. SEALED: EVITAR HERENCIA
--------------------------------------------------------------------

La palabra clave `sealed` impide que una clase sea heredada o que un
método/propiedad sea sobrescrito.

```csharp
// Clase sellada: no se puede heredar
public sealed class ConfiguracionFinal
{
    public string Servidor { get; set; }
    public int Puerto { get; set; }
}

// public class MiConfig : ConfiguracionFinal { }  // ERROR: sealed

// Metodo sellado: no se puede sobrescribir en mas niveles
public class Base
{
    public virtual void MetodoVirtual() { }
}

public class Derivada : Base
{
    public sealed override void MetodoVirtual()  // Sellado aqui
    {
        Console.WriteLine("No se puede sobrescribir mas.");
    }
}

public class SubDerivada : Derivada
{
    // public override void MetodoVirtual() { }  // ERROR: sealed
}
```

¿Cuándo usar sealed?

  • Cuando la clase está completa y no debe extenderse.
  • Por razones de seguridad (evitar manipulación).
  • Por rendimiento (el JIT puede optimizar llamadas virtuales).
  • En clases de framework para evitar malos usos.

```csharp
// Ejemplo de clase sealed en .NET: System.String
public sealed class String { /* ... */ }
```

--------------------------------------------------------------------
  7. OPERADOR `is` Y PATTERN MATCHING
--------------------------------------------------------------------

El operador `is` verifica si un objeto es de un tipo determinado. Desde
C# 7, también permite "pattern matching" con declaración de variable.

```csharp
public class Animal { }
public class Perro : Animal { public void Ladrar() => Console.WriteLine("Guau"); }
public class Gato : Animal { public void Maullar() => Console.WriteLine("Miau"); }

public void ProcesarAnimal(Animal animal)
{
    // Forma tradicional (antes de C# 7)
    if (animal is Perro)
    {
        Perro p = (Perro)animal;
        p.Ladrar();
    }

    // Pattern matching (C# 7+)
    if (animal is Perro perro)
    {
        perro.Ladrar();  // Variable 'perro' ya tipada
    }
    else if (animal is Gato gato)
    {
        gato.Maullar();
    }

    // Switch con pattern matching (C# 7+)
    switch (animal)
    {
        case Perro p:
            p.Ladrar();
            break;
        case Gato g:
            g.Maullar();
            break;
        case null:
            Console.WriteLine("Animal es null");
            break;
        default:
            Console.WriteLine("Tipo desconocido");
            break;
    }
}

// Switch expression (C# 8+)
public string ObtenerSonido(Animal animal) => animal switch
{
    Perro => "Guau",
    Gato => "Miau",
    _ => "???"
};
```

`is` también puede verificar patrones más complejos:

```csharp
public void Clasificar(object obj)
{
    if (obj is int entero && entero > 0)
        Console.WriteLine($"Entero positivo: {entero}");
    else if (obj is string texto && texto.Length > 5)
        Console.WriteLine($"Texto largo: {texto}");
    else if (obj is DateTime fecha && fecha.Year > 2020)
        Console.WriteLine($"Fecha reciente: {fecha}");
    else
        Console.WriteLine($"Tipo: {obj?.GetType().Name ?? "null"}");
}
```

--------------------------------------------------------------------
  8. OPERADOR `as`
--------------------------------------------------------------------

El operador `as` intenta convertir un objeto a un tipo. Si la conversión
falla, devuelve `null` (NO lanza excepción).

```csharp
public class Animal { }
public class Perro : Animal { public void Ladrar() => Console.WriteLine("Guau"); }

Animal animal = new Perro();

// Con as (seguro, no lanza excepcion):
Perro p = animal as Perro;
if (p != null)
{
    p.Ladrar();
}

// Equivalente con is:
if (animal is Perro perro)
{
    perro.Ladrar();
}
```

Diferencia entre `is` + cast y `as`:

```csharp
Animal animal = new Perro();

// Opcion 1: is + cast (dos comprobaciones)
if (animal is Perro)
{
    Perro p = (Perro)animal;  // Segundo cast
    p.Ladrar();
}

// Opcion 2: as + null check (una comprobacion)
Perro p2 = animal as Perro;
if (p2 != null)
{
    p2.Ladrar();
}

// Opcion 3: pattern matching (una comprobacion, recomendada)
if (animal is Perro p3)
{
    p3.Ladrar();
}
```

`as` solo funciona con tipos referencia. Para tipos valor, usa `is`:

```csharp
object obj = 42;

// int? nullable = obj as int;   // ERROR: int es tipo valor
int? numero = obj as int?;        // OK: nullable es tipo referencia
if (numero.HasValue) { /* ... */ }
```

--------------------------------------------------------------------
  9. CASTING (CONVERSIÓN DE TIPOS)
--------------------------------------------------------------------

En herencia, podemos convertir entre tipos base y derivados:

  • UPCASTING:   convertir de derivado a base (SIEMPRE seguro).
  • DOWNCASTING: convertir de base a derivado (requiere verificación).

```csharp
public class Animal { }
public class Perro : Animal { public void Ladrar() => Console.WriteLine("Guau"); }

// Upcasting (implicito, siempre seguro)
Perro perro = new Perro();
Animal animal = perro;  // Upcasting: Perro -> Animal
object obj = perro;     // Upcasting: Perro -> object

// Downcasting (explicito, requiere verificacion)
Perro p1 = (Perro)animal;  // Cast explicito (puede lanzar InvalidCastException)
Perro p2 = animal as Perro; // as: devuelve null si falla

if (animal is Perro p3)
{
    p3.Ladrar();  // Pattern matching + downcasting seguro
}
```

Reglas de casting:

  ✓ Upcasting es SIEMPRE seguro (implícito).
  ✓ Downcasting requiere verificación.
  ✓ Usa `as` o `is` para downcasting seguro.
  ✓ El cast directo `(Tipo)` lanza `InvalidCastException` si falla.

```csharp
public void DemostrarCasting()
{
    object[] objetos = { "Hola", 42, new Perro(), new Gato() };

    foreach (var obj in objetos)
    {
        if (obj is string texto)
            Console.WriteLine($"String: {texto}");
        else if (obj is int numero)
            Console.WriteLine($"Entero: {numero}");
        else if (obj is Perro perro)
            perro.Ladrar();
        else if (obj is Gato gato)
            gato.Maullar();
        else
            Console.WriteLine($"Tipo desconocido: {obj.GetType().Name}");
    }
}
```

--------------------------------------------------------------------
  10. EJEMPLO COMPLETO: SISTEMA DE VEHÍCULOS
--------------------------------------------------------------------

```csharp
using System;

namespace SistemaVehiculos
{
    // Clase base
    public class Vehiculo
    {
        private static int _contador = 0;

        public int Id { get; }
        public string Marca { get; set; }
        public string Modelo { get; set; }
        public int Anio { get; set; }
        public double VelocidadActual { get; protected set; }
        public bool Encendido { get; protected set; }

        public Vehiculo(string marca, string modelo, int anio)
        {
            Id = ++_contador;
            Marca = marca;
            Modelo = modelo;
            Anio = anio;
            VelocidadActual = 0;
            Encendido = false;
        }

        public void Encender()
        {
            if (Encendido) { Console.WriteLine($"{Id}: Ya esta encendido."); return; }
            Encendido = true;
            Console.WriteLine($"{Id}: Vehiculo encendido.");
        }

        public void Apagar()
        {
            if (!Encendido) { Console.WriteLine($"{Id}: Ya esta apagado."); return; }
            if (VelocidadActual > 0) { Console.WriteLine($"{Id}: Detenga el vehiculo primero."); return; }
            Encendido = false;
            Console.WriteLine($"{Id}: Vehiculo apagado.");
        }

        public virtual void Acelerar(double kmh)
        {
            if (!Encendido) { Console.WriteLine($"{Id}: Encienda el vehiculo primero."); return; }
            VelocidadActual += kmh;
            Console.WriteLine($"{Id}: Acelerando. Velocidad: {VelocidadActual} km/h");
        }

        public void Frenar(double kmh)
        {
            VelocidadActual = Math.Max(0, VelocidadActual - kmh);
            Console.WriteLine($"{Id}: Frenando. Velocidad: {VelocidadActual} km/h");
        }

        public virtual void MostrarInfo()
        {
            Console.WriteLine($"Vehiculo #{Id}: {Marca} {Modelo} ({Anio})");
            Console.WriteLine($"  Velocidad: {VelocidadActual} km/h");
            Console.WriteLine($"  Estado: {(Encendido ? "Encendido" : "Apagado")}");
        }
    }

    // Clase derivada: Coche
    public class Coche : Vehiculo
    {
        public int NumeroPuertas { get; set; }
        public string TipoCombustible { get; set; }

        public Coche(string marca, string modelo, int anio, int puertas, string combustible)
            : base(marca, modelo, anio)
        {
            NumeroPuertas = puertas;
            TipoCombustible = combustible;
        }

        public override void Acelerar(double kmh)
        {
            base.Acelerar(kmh * 1.2);  // Los coches aceleran mas rapido
        }

        public override void MostrarInfo()
        {
            base.MostrarInfo();
            Console.WriteLine($"  Puertas: {NumeroPuertas}, Combustible: {TipoCombustible}");
        }
    }

    // Clase derivada: Moto
    public class Moto : Vehiculo
    {
        public bool TieneSidecar { get; set; }

        public Moto(string marca, string modelo, int anio, bool sidecar)
            : base(marca, modelo, anio)
        {
            TieneSidecar = sidecar;
        }

        public void HacerCaballito()
        {
            if (!Encendido) { Console.WriteLine($"{Id}: Encienda la moto primero."); return; }
            Console.WriteLine($"{Id}: ¡Haciendo caballito!");
        }

        public override void MostrarInfo()
        {
            base.MostrarInfo();
            Console.WriteLine($"  Sidecar: {(TieneSidecar ? "Si" : "No")}");
        }
    }

    // Clase derivada: Camion
    public sealed class Camion : Vehiculo
    {
        public double CapacidadCarga { get; set; }
        public double CargaActual { get; private set; }

        public Camion(string marca, string modelo, int anio, double capacidad)
            : base(marca, modelo, anio)
        {
            CapacidadCarga = capacidad;
            CargaActual = 0;
        }

        public bool Cargar(double kg)
        {
            if (CargaActual + kg > CapacidadCarga)
            {
                Console.WriteLine($"{Id}: Excede la capacidad maxima de {CapacidadCarga} kg.");
                return false;
            }
            CargaActual += kg;
            Console.WriteLine($"{Id}: Cargados {kg} kg. Carga actual: {CargaActual} kg");
            return true;
        }

        public override void Acelerar(double kmh)
        {
            double factor = 1.0 - (CargaActual / CapacidadCarga) * 0.5;
            base.Acelerar(kmh * Math.Max(0.3, factor));
        }

        public override void MostrarInfo()
        {
            base.MostrarInfo();
            Console.WriteLine($"  Capacidad: {CapacidadCarga} kg, Carga: {CargaActual} kg");
        }
    }

    public class Programa
    {
        public static void Main()
        {
            // Upcasting: tratar derivados como base
            Vehiculo[] vehiculos = new Vehiculo[]
            {
                new Coche("Toyota", "Corolla", 2022, 4, "Gasolina"),
                new Moto("Yamaha", "MT-07", 2023, false),
                new Camion("Volvo", "FH", 2021, 20000)
            };

            // Polimorfismo: cada vehiculo se comporta segun su tipo
            foreach (Vehiculo v in vehiculos)
            {
                v.Encender();
                v.Acelerar(50);
                v.MostrarInfo();
                Console.WriteLine();
            }

            // Downcasting con is/as
            foreach (Vehiculo v in vehiculos)
            {
                if (v is Coche coche)
                    Console.WriteLine($"Coche detectado: {coche.NumeroPuertas} puertas.");
                else if (v is Moto moto)
                {
                    moto.HacerCaballito();
                }
                else if (v is Camion camion)
                {
                    camion.Cargar(5000);
                }
            }

            // Demostracion de sealed
            Camion camion = new Camion("Ford", "Serie F", 2023, 15000);
            camion.Cargar(8000);
            camion.Acelerar(60);
            camion.MostrarInfo();
        }
    }
}
```

--------------------------------------------------------------------
  11. HERENCIA VS COMPOSICIÓN
--------------------------------------------------------------------

Un principio importante de diseño es: "Prefiere composición sobre herencia"
(Favor composition over inheritance).

  • HERENCIA: relación "es-un" (Perro es un Animal).
  • COMPOSICIÓN: relación "tiene-un" (Coche tiene un Motor).

La herencia debe usarse cuando hay una verdadera relación jerárquica.
Para reutilización de código, la composición suele ser más flexible.

```csharp
// Herencia (relacion "es-un")
public class Animal { }
public class Perro : Animal { }

// Composicion (relacion "tiene-un")
public class Motor
{
    public void Encender() { }
    public void Apagar() { }
}

public class Coche
{
    private Motor _motor = new Motor();  // Composicion

    public void Encender() => _motor.Encender();
    public void Apagar() => _motor.Apagar();
}
```

¿Cuándo usar herencia?

  ✓ Relación "es-un" clara y natural.
  ✓ La clase base tiene comportamiento que todas las derivadas comparten.
  ✓ Necesitas polimorfismo (tratar objetos derivados como base).

¿Cuándo usar composición?

  ✓ Relación "tiene-un" o "usa-un".
  ✓ Necesitas reutilizar código sin acoplar jerarquías.
  ✓ La funcionalidad puede variar en tiempo de ejecución.
  ✓ Quieres evitar jerarquías profundas y frágiles.

--------------------------------------------------------------------
  12. BUENAS PRÁCTICAS
--------------------------------------------------------------------

  ✓ Usa herencia solo para relaciones "es-un" genuinas.
  ✓ Mantén la jerarquía poco profunda (máximo 3-4 niveles).
  ✓ Declara la clase como `sealed` a menos que estés diseñando para
    herencia.
  ✓ Documenta la intención: "Esta clase está diseñada para ser heredada."
  ✓ Usa `protected` para miembros que las derivadas necesiten, pero no
    los hagas públicos si no es necesario.
  ✓ Prefiere polimorfismo virtual sobre condicionales con `is`.
  ✓ No abuses de la herencia para reutilizar código (usa composición).
  ✓ En constructores, no llames a métodos virtuales (puede llamarse
    antes de que la clase derivada esté lista).

```csharp
// MAL: llamar a metodo virtual en constructor
public class Base
{
    public Base()
    {
        Inicializar();  // PELIGRO: puede llamar a implementacion derivada
    }
    public virtual void Inicializar() { }
}

// BIEN: el constructor solo inicializa lo minimo
public class Base
{
    public Base() { }
    public virtual void Inicializar() { }
}
```

  RESUMEN DEL CAPÍTULO
================================================================================

  • La HERENCIA permite crear clases derivadas que heredan miembros de
    una clase base (relación "es-un").

  • La sintaxis es `class Derivada : Base`.

  • Los CONSTRUCTORES NO se heredan. La derivada llama al base con
    `: base(parámetros)`.

  • `sealed` impide la herencia de clases o la sobrescritura de métodos.

  • `is` verifica el tipo con pattern matching (C# 7+).

  • `as` convierte tipos de forma segura (devuelve null si falla).

  • UPCASTING (derivado -> base) es siempre seguro e implícito.

  • DOWNCASTING (base -> derivado) requiere verificación.

  • Prefiere COMPOSICIÓN sobre herencia para reutilización de código.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ───
  ❓ ¿Puedo heredar de varias clases?
  ───
  No. C# soporta herencia simple de clases. Pero puedes implementar
  múltiples interfaces.

  ───
  ❓ ¿Qué es la "herencia en diamante"?
  ───
  Es el problema de la herencia múltiple donde una clase hereda de dos
  clases que tienen el mismo método. C# evita esto con herencia simple.

  ───
  ❓ ¿Puedo cambiar el modificador de acceso al sobrescribir?
  ───
  No. El modificador de acceso debe ser el mismo o menos restrictivo.
  No puedes hacer public un método protected, ni viceversa.

  ───
  ❓ ¿Cómo sé cuándo usar virtual?
  ───
  Cuando diseñes la clase base y anticipes que las clases derivadas
  pueden necesitar cambiar ese comportamiento.

  ───
  ❓ ¿Qué pasa si no llamo a base() explícitamente?
  ───
  El compilador llama al constructor sin parámetros de la base. Si no
  existe, da error de compilación.

  ───
  ❓ ¿Puedo heredar de struct?
  ───
  No. Los structs son tipos valor y no soportan herencia. Solo las
  clases (tipos referencia) pueden heredar.

  ───
  ❓ ¿Qué es una clase abstracta y cómo se relaciona con herencia?
  ───
  Una clase abstracta no puede instanciarse directamente. Sirve como
  base para otras clases. La veremos en el capítulo 18.

================================================================================
  CIERRE DEL CAPÍTULO
================================================================================

La herencia es un mecanismo poderoso que permite modelar jerarquías de
tipos y reutilizar comportamiento. Bien usada, produce código elegante y
extensible. Mal usada, crea acoplamiento y fragilidad.

Recuerda: la herencia es una relación "es-un". Si no es natural, prefiere
composición. Y siempre diseña tus clases base pensando en cómo serán
extendidas.

En el próximo capítulo exploraremos el POLIMORFISMO, el tercer pilar de la
POO, que permite que objetos de diferentes tipos respondan al mismo mensaje
de formas distintas.
