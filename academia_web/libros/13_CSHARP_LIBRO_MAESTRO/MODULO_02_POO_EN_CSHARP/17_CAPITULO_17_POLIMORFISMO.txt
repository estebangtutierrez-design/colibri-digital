  BIENVENIDA
================================================================================

El polimorfismo es el tercer pilar de la POO y, quizás, el más poderoso.
La palabra viene del griego "poli" (muchos) y "morphé" (forma). En esencia,
el polimorfismo permite que objetos de diferentes tipos respondan al mismo
mensaje de formas distintas.

En C#, el polimorfismo se manifiesta principalmente a través de:
  • Métodos virtuales y su sobrescritura (override).
  • Ocultamiento de métodos con `new`.
  • Interfaces y su implementación.
  • Clases abstractas.

El polimorfismo nos permite escribir código GENÉRICO que trabaja con la
clase base o la interfaz, y se comporta correctamente sin importar qué
clase derivada concreta esté usando.

Este capítulo se centra en el polimorfismo con clases (virtual/override/new).
El polimorfismo con interfaces se tratará en el Capítulo 19.

--------------------------------------------------------------------
  1. ¿QUÉ ES EL POLIMORFISMO?
--------------------------------------------------------------------

El polimorfismo es la capacidad de un objeto de comportarse de diferentes
maneras según su tipo real, incluso cuando se accede a través de una
referencia de tipo base.

```csharp
public class Animal
{
    public virtual void HacerSonido()
    {
        Console.WriteLine("Sonido generico de animal.");
    }
}

public class Perro : Animal
{
    public override void HacerSonido()
    {
        Console.WriteLine("¡Guau guau!");
    }
}

public class Gato : Animal
{
    public override void HacerSonido()
    {
        Console.WriteLine("¡Miau!");
    }
}

// Polimorfismo en accion:
Animal[] animales = { new Perro(), new Gato(), new Perro() };

foreach (Animal a in animales)
{
    a.HacerSonido();  // Cada uno hace su propio sonido
}

// Salida:
// ¡Guau guau!
// ¡Miau!
// ¡Guau guau!
```

Sin polimorfismo, tendríamos que escribir:

```csharp
foreach (Animal a in animales)
{
    if (a is Perro) ((Perro)a).HacerSonido();
    else if (a is Gato) ((Gato)a).HacerSonido();
}
```

El polimorfismo elimina estos condicionales y hace el código más limpio,
extensible y mantenible.

--------------------------------------------------------------------
  2. MECANISMO INTERNO: TABLA VIRTUAL (VTABLE)
----------------------------------------------------

Cuando el compilador encuentra un método virtual, crea una "vtable"
(tabla virtual) para la clase. Cada objeto tiene un puntero a su vtable.

```
  Perro vtable:                    Gato vtable:
  ┌───────────────────┐           ┌───────────────────┐
  │ HacerSonido() ────┼──Perro()  │ HacerSonido() ────┼──Gato()
  │ ToString() ───────┼──Object() │ ToString() ───────┼──Object()
  │ Equals() ─────────┼──Object() │ Equals() ─────────┼──Object()
  │ GetHashCode() ────┼──Object() │ GetHashCode() ────┼──Object()
  └───────────────────┘           └───────────────────┘
```

Cuando se llama a `a.HacerSonido()` donde `a` es de tipo `Animal`, el
runtime:
  1. Sigue el puntero vtable del objeto.
  2. Busca HacerSonido() en esa tabla.
  3. Ejecuta la implementación correcta.

Esto se llama "despacho virtual" (virtual dispatch) y ocurre en tiempo
de ejecución. Tiene un costo mínimo (una indirección) pero aporta enorme
flexibilidad.

```csharp
public class DemoVTable
{
    public virtual void MetodoVirtual() { }
    public void MetodoNormal() { }
}

// MetodoNormal: el compilador sabe exactamente qué llamar
// MetodoVirtual: el runtime decide según el tipo real
```

--------------------------------------------------------------------
  3. VIRTUAL Y OVERRIDE
--------------------------------------------------------------------

  • `virtual`: declara que un método puede ser sobrescrito en clases
    derivadas.
  • `override`: reemplaza la implementación del método virtual en la
    clase derivada.

Reglas de virtual/override:

  ✓ El método base debe ser virtual, abstract o override.
  ✓ La firma debe ser idéntica (nombre, parámetros, retorno).
  ✓ El modificador de acceso no puede ser más restrictivo.
  ✓ No se puede sobrescribir un método static, private o sealed.
  ✓ Los métodos override son virtuales por defecto (pueden seguir
    siendo sobrescritos).

```csharp
public class Empleado
{
    public string Nombre { get; set; }
    public decimal SalarioBase { get; set; }

    public virtual decimal CalcularBono()
    {
        return SalarioBase * 0.05m;
    }

    public virtual void MostrarInfo()
    {
        Console.WriteLine($"Empleado: {Nombre}, Salario: {SalarioBase:C}");
    }
}

public class Vendedor : Empleado
{
    public decimal VentasDelMes { get; set; }

    public override decimal CalcularBono()
    {
        return base.CalcularBono() + VentasDelMes * 0.02m;
    }

    public override void MostrarInfo()
    {
        base.MostrarInfo();
        Console.WriteLine($"Ventas: {VentasDelMes:C}, Bono total: {CalcularBono():C}");
    }
}

public class Gerente : Empleado
{
    public decimal BonoFijo { get; set; }

    public override decimal CalcularBono()
    {
        return base.CalcularBono() + BonoFijo;
    }
}

// Uso polimorfico:
Empleado[] empleados = {
    new Empleado { Nombre = "Ana", SalarioBase = 30000 },
    new Vendedor { Nombre = "Luis", SalarioBase = 25000, VentasDelMes = 50000 },
    new Gerente { Nombre = "Carlos", SalarioBase = 40000, BonoFijo = 10000 }
};

foreach (var emp in empleados)
{
    Console.WriteLine($"{emp.Nombre}: Bono = {emp.CalcularBono():C}");
}
// Salida:
// Ana: Bono = $1,500.00
// Luis: Bono = $2,250.00
// Carlos: Bono = $12,000.00
```

--------------------------------------------------------------------
  4. NEW (OCULTACIÓN DE MÉTODOS)
--------------------------------------------------------------------

La palabra clave `new` en un método derivado OCULTA el método de la clase
base, en lugar de sobrescribirlo.

A diferencia de override, con `new`:
  • No hay polimorfismo: si llamas a través de la base, se ejecuta el
    método de la base.
  • No se requiere virtual en la clase base.
  • El método base no se reemplaza, solo se "esconde".

```csharp
public class Base
{
    public void Mostrar()
    {
        Console.WriteLine("Metodo de Base");
    }
}

public class Derivada : Base
{
    public new void Mostrar()
    {
        Console.WriteLine("Metodo de Derivada (oculto)");
    }
}

// Uso:
Derivada d = new Derivada();
d.Mostrar();              // "Metodo de Derivada (oculto)"

Base b = d;               // Upcasting
b.Mostrar();              // "Metodo de Base" (NO polimorfismo)

// Con override (polimorfismo):
public class BaseVirtual
{
    public virtual void Mostrar() { Console.WriteLine("Base"); }
}

public class DerivadaOverride : BaseVirtual
{
    public override void Mostrar() { Console.WriteLine("Derivada"); }
}

DerivadaOverride dor = new DerivadaOverride();
BaseVirtual bv = dor;
bv.Mostrar();  // "Derivada" (SI hay polimorfismo)
```

Comparación override vs new:

┌─────────────────────────┬──────────────────┬─────────────────────────┐
│      CARACTERÍSTICA      │    override       │          new            │
├─────────────────────────┼──────────────────┼─────────────────────────┤
│ Requiere virtual        │ Sí                │ No                      │
│ Polimorfismo            │ Sí                │ No                      │
│ Llamada por base        │ Va al derivado    │ Va al base              │
│ Intención               │ Extender/cambiar  │ Ocultar (nueva versión) │
│ Compatible con sealed   │ Sí (sealed over.) │ Sí                      │
└─────────────────────────┴──────────────────┴─────────────────────────┘

¿Cuándo usar `new`?
  • Cuando heredas de una clase que no puedes modificar y quieres
    redefinir un método no virtual.
  • Cuando deliberadamente quieres romper el polimorfismo (raro).
  • Cuando migras código y necesitas compatibilidad hacia atrás.

--------------------------------------------------------------------
  5. POLIMORFISMO CON MÚLTIPLES NIVELES
--------------------------------------------------------------------

El polimorfismo funciona a través de toda la cadena de herencia:

```csharp
public class Animal
{
    public virtual void Saludar() => Console.WriteLine("Animal saluda");
}

public class Perro : Animal
{
    public override void Saludar() => Console.WriteLine("Perro saluda");
}

public class PastorAleman : Perro
{
    public override void Saludar() => Console.WriteLine("Pastor Aleman saluda");
}

// Uso:
Animal a1 = new PastorAleman();
a1.Saludar();  // "Pastor Aleman saluda" (ultimo override)

Animal a2 = new Perro();
a2.Saludar();  // "Perro saluda"

// Con new interrumpiendo la cadena:
public class PerroConNew : Animal
{
    public new void Saludar() => Console.WriteLine("Perro con new");
}

public class PastorConNew : PerroConNew
{
    public override void Saludar() => Console.WriteLine("Pastor con new");
    // ERROR: Saludar en PerroConNew no es virtual (es new)
    // Necesitariamos new de nuevo o virtual en PerroConNew
}
```

--------------------------------------------------------------------
  6. MÉTODOS SELLADOS (SEALED OVERRIDE)
--------------------------------------------------------------------

Un método override puede marcarse como `sealed` para evitar que clases
más derivadas lo sobrescriban:

```csharp
public class Animal
{
    public virtual void HacerSonido() { }
}

public class Perro : Animal
{
    public sealed override void HacerSonido()
    {
        Console.WriteLine("Guau");
    }
}

public class PastorAleman : Perro
{
    // public override void HacerSonido() { }  // ERROR: sealed
}
```

Es útil cuando quieres permitir un nivel de personalización pero no más.

--------------------------------------------------------------------
  7. EL PROBLEMA DE LLAMAR A MÉTODOS VIRTUALES EN CONSTRUCTORES
--------------------------------------------------------------------

NUNCA llames a métodos virtuales en un constructor. La clase derivada
aún no está completamente inicializada.

```csharp
public class Base
{
    public Base()
    {
        Inicializar();  // ¡PELIGRO!
    }

    public virtual void Inicializar()
    {
        Console.WriteLine("Base.Inicializar");
    }
}

public class Derivada : Base
{
    private string _dato = "Importante";

    public override void Inicializar()
    {
        Console.WriteLine($"Derivada.Inicializar: {_dato.Length}");
        // _dato es null si el constructor de Base llama a Inicializar
        // antes de que se ejecute el inicializador de campo de Derivada
    }
}

// new Derivada() lanzaria NullReferenceException
```

--------------------------------------------------------------------
  8. POLIMORFISMO CON OPERADORES 'is' y 'as'
--------------------------------------------------------------------

Aunque el polimorfismo evita los condicionales de tipo, a veces necesitas
comportamiento específico de la clase derivada:

```csharp
public class Animal
{
    public virtual void Mover() => Console.WriteLine("Animal se mueve");
}

public class Pez : Animal
{
    public override void Mover() => Console.WriteLine("Pez nada");
    public void Nadar() => Console.WriteLine("Nadando...");
}

public class Ave : Animal
{
    public override void Mover() => Console.WriteLine("Ave vuela");
    public void Volar() => Console.WriteLine("Volando...");
}

public class Zoologico
{
    public void ProcesarAnimal(Animal animal)
    {
        // Comportamiento polimorfico (funciona para todos)
        animal.Mover();

        // Comportamiento especifico (cuando es necesario)
        if (animal is Pez pez)
            pez.Nadar();
        else if (animal is Ave ave)
            ave.Volar();
    }
}
```

--------------------------------------------------------------------
  9. POLIMORFISMO CON PROPIEDADES
--------------------------------------------------------------------

Las propiedades también pueden ser virtuales y sobrescritas:

```csharp
public abstract class Figura
{
    public abstract double Area { get; }
    public abstract double Perimetro { get; }
    public virtual string Nombre => "Figura";
}

public class Circulo : Figura
{
    public double Radio { get; }

    public Circulo(double radio) => Radio = radio;

    public override double Area => Math.PI * Radio * Radio;
    public override double Perimetro => 2 * Math.PI * Radio;
    public override string Nombre => $"Circulo (r={Radio})";
}

public class Rectangulo : Figura
{
    public double Ancho { get; }
    public double Alto { get; }

    public Rectangulo(double ancho, double alto)
    {
        Ancho = ancho;
        Alto = alto;
    }

    public override double Area => Ancho * Alto;
    public override double Perimetro => 2 * (Ancho + Alto);
    public override string Nombre => $"Rectangulo {Ancho}x{Alto}";
}

// Uso polimorfico:
Figura[] figuras = { new Circulo(5), new Rectangulo(4, 6) };
foreach (var f in figuras)
{
    Console.WriteLine($"{f.Nombre}: Area={f.Area:F2}, Perimetro={f.Perimetro:F2}");
}
```

--------------------------------------------------------------------
  10. EJEMPLO COMPLETO: SISTEMA DE NÓMINA POLIMÓRFICA
--------------------------------------------------------------------

```csharp
using System;

namespace NominaPolimorfica
{
    public enum TipoContrato { TiempoCompleto, MedioTiempo, PorHoras, Comision }

    public abstract class Empleado
    {
        public int Id { get; }
        public string Nombre { get; set; }
        public string Apellido { get; set; }
        public string NombreCompleto => $"{Nombre} {Apellido}";

        protected Empleado(string nombre, string apellido)
        {
            Id = new Random().Next(1000, 9999);
            Nombre = nombre;
            Apellido = apellido;
        }

        // Metodo abstracto: cada tipo lo implementa
        public abstract decimal CalcularPagoMensual();

        // Metodo virtual: puede ser sobrescrito
        public virtual decimal CalcularImpuesto()
        {
            return CalcularPagoMensual() * 0.16m;
        }

        public virtual void MostrarInfo()
        {
            Console.WriteLine($"ID: {Id} - {NombreCompleto}");
            Console.WriteLine($"  Pago Mensual: ${CalcularPagoMensual():F2}");
            Console.WriteLine($"  Impuesto: ${CalcularImpuesto():F2}");
            Console.WriteLine($"  Neto: ${CalcularPagoMensual() - CalcularImpuesto():F2}");
        }
    }

    public class EmpleadoTiempoCompleto : Empleado
    {
        public decimal SalarioAnual { get; set; }
        public decimal BonoAnual { get; set; }

        public EmpleadoTiempoCompleto(string nombre, string apellido,
            decimal salarioAnual, decimal bonoAnual)
            : base(nombre, apellido)
        {
            SalarioAnual = salarioAnual;
            BonoAnual = bonoAnual;
        }

        public override decimal CalcularPagoMensual()
        {
            return (SalarioAnual + BonoAnual) / 12;
        }

        public override void MostrarInfo()
        {
            Console.WriteLine("--- Tiempo Completo ---");
            base.MostrarInfo();
            Console.WriteLine($"  Salario Anual: ${SalarioAnual:F2}");
            Console.WriteLine($"  Bono: ${BonoAnual:F2}");
        }
    }

    public class EmpleadoPorHoras : Empleado
    {
        public decimal TarifaPorHora { get; set; }
        public int HorasTrabajadas { get; set; }

        public EmpleadoPorHoras(string nombre, string apellido,
            decimal tarifa, int horas)
            : base(nombre, apellido)
        {
            TarifaPorHora = tarifa;
            HorasTrabajadas = horas;
        }

        public override decimal CalcularPagoMensual()
        {
            return TarifaPorHora * HorasTrabajadas * 4; // 4 semanas
        }

        public override void MostrarInfo()
        {
            Console.WriteLine("--- Por Horas ---");
            base.MostrarInfo();
            Console.WriteLine($"  Tarifa: ${TarifaPorHora:F2}/h");
            Console.WriteLine($"  Horas/sem: {HorasTrabajadas}");
        }
    }

    public class EmpleadoPorComision : Empleado
    {
        public decimal VentasRealizadas { get; set; }
        public decimal PorcentajeComision { get; set; } // 0.05 = 5%

        public EmpleadoPorComision(string nombre, string apellido,
            decimal ventas, decimal porcentaje)
            : base(nombre, apellido)
        {
            VentasRealizadas = ventas;
            PorcentajeComision = porcentaje;
        }

        public override decimal CalcularPagoMensual()
        {
            return VentasRealizadas * PorcentajeComision;
        }

        public override decimal CalcularImpuesto()
        {
            // Los comisionistas tienen impuesto reducido
            return base.CalcularImpuesto() * 0.8m;
        }

        public override void MostrarInfo()
        {
            Console.WriteLine("--- Por Comision ---");
            base.MostrarInfo();
            Console.WriteLine($"  Ventas: ${VentasRealizadas:F2}");
            Console.WriteLine($"  Comision: {PorcentajeComision:P}");
        }
    }

    public class Programa
    {
        public static void Main()
        {
            Empleado[] nomina = {
                new EmpleadoTiempoCompleto("Ana", "Lopez", 60000, 5000),
                new EmpleadoPorHoras("Carlos", "Ruiz", 25, 40),
                new EmpleadoPorComision("Maria", "Garcia", 80000, 0.08m),
                new EmpleadoTiempoCompleto("Pedro", "Infante", 48000, 3000),
                new EmpleadoPorHoras("Laura", "Diaz", 30, 20)
            };

            decimal totalNomina = 0;

            foreach (var emp in nomina)
            {
                emp.MostrarInfo();
                totalNomina += emp.CalcularPagoMensual();
                Console.WriteLine();
            }

            Console.WriteLine("=================================");
            Console.WriteLine($"TOTAL NOMINA MENSUAL: ${totalNomina:F2}");

            // Demostracion de downcasting para comportamiento especifico
            foreach (var emp in nomina)
            {
                if (emp is EmpleadoTiempoCompleto tc)
                    Console.WriteLine($"{tc.NombreCompleto} tiene bono de ${tc.BonoAnual:F2}");
                else if (emp is EmpleadoPorHoras ph)
                    Console.WriteLine($"{ph.NombreCompleto} trabaja {ph.HorasTrabajadas}h/semana");
                else if (emp is EmpleadoPorComision com)
                    Console.WriteLine($"{com.NombreCompleto} vendio ${com.VentasRealizadas:F2}");
            }
        }
    }
}
```

--------------------------------------------------------------------
  11. POLIMORFISMO vs CONDICIONALES
--------------------------------------------------------------------

El polimorfismo elimina la necesidad de condicionales basados en tipos:

```csharp
// SIN polimorfismo (feo, fragil, dificil de extender)
public void ProcesarPago(object empleado)
{
    if (empleado is EmpleadoTiempoCompleto tc)
        Console.WriteLine(tc.SalarioAnual / 12);
    else if (empleado is EmpleadoPorHoras ph)
        Console.WriteLine(ph.TarifaPorHora * ph.HorasTrabajadas);
    else if (empleado is EmpleadoPorComision com)
        Console.WriteLine(com.VentasRealizadas * com.PorcentajeComision);
}

// CON polimorfismo (limpio, extensible, mantenible)
public void ProcesarPago(Empleado empleado)
{
    Console.WriteLine(empleado.CalcularPagoMensual());
}
```

Ventajas del polimorfismo sobre condicionales:

  ✓ Abierto a extensión: agregas un nuevo tipo sin modificar el código
    existente (Principio Open/Closed).
  ✓ Elimina duplicación de lógica de selección.
  ✓ Centraliza el comportamiento en cada clase.
  ✓ Más fácil de probar (cada clase se prueba por separado).

--------------------------------------------------------------------
  12. BUENAS PRÁCTICAS
--------------------------------------------------------------------

  ✓ Usa virtual solo cuando anticipes que las derivadas NECESITEN
    cambiar el comportamiento.

  ✓ Declara los métodos como sealed cuando la implementación no deba
    cambiar más.

  ✓ No llames a métodos virtuales en constructores.

  ✓ Prefiere override sobre new (new rompe el polimorfismo).

  ✓ Diseña pensando en polimorfismo desde la clase base.

  ✓ Usa el Principio de Sustitución de Liskov (LSP): las clases
    derivadas deben poder sustituir a la base sin alterar el
    comportamiento esperado.

  ✓ Si una clase derivada no necesita cambiar un método virtual,
    no lo sobrescribas (hereda la implementación base).

  ✓ Documenta qué métodos virtuales están diseñados para ser
    sobrescritos y cómo.

```csharp
/// <summary>
/// Calcula el bono del empleado. Sobrescribir para personalizar
/// la formula de calculo.
/// </summary>
public virtual decimal CalcularBono()
{
    return SalarioBase * 0.05m;
}
```

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • POLIMORFISMO: objetos de diferentes tipos responden al mismo mensaje
    de formas distintas.

  • `virtual` permite que un método sea sobrescrito.

  • `override` reemplaza la implementación del método virtual.

  • `new` OCULTA el método base (NO hay polimorfismo).

  • DIFERENCIA: override respeta el polimorfismo (llamada por referencia
    base ejecuta el método derivado); new NO (llamada por base ejecuta
    el base).

  • Sealed override detiene la cadena de sobrescritura.

  • No llames a métodos virtuales en constructores.

  • Las propiedades también pueden ser virtuales/override.

  • El polimorfismo reemplaza condicionales de tipo, haciendo el código
    más extensible y mantenible.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ───
  ❓ ¿override es siempre virtual?
  ───
  Sí, por defecto. Pero puedes marcarlo como sealed para detener la
  cadena de sobrescritura.

  ───
  ❓ ¿new y override pueden coexistir en la misma clase?
  ───
  Sí, pero es confuso y no recomendable.

  ───
  ❓ ¿Puedo sobrescribir un método estático?
  ───
  No. Los métodos estáticos no son virtuales. Pertenecen a la clase,
  no a la instancia.

  ───
  ❓ ¿El rendimiento se ve afectado por métodos virtuales?
  ───
  Mínimamente. Hay una indirección extra (vtable). En el 99% de los
  casos no es relevante. El JIT puede optimizar en algunos casos.

  ───
  ❓ ¿Qué es el "Principio de Sustitución de Liskov" (LSP)?
  ───
  Es un principio SOLID que dice: si S es subtipo de T, entonces los
  objetos de T pueden sustituirse por objetos de S sin alterar las
  propiedades del programa. Básicamente: las derivadas deben comportarse
  como la base.

  ───
  ❓ ¿Puedo tener un método virtual en un struct?
  ───
  Los structs no soportan herencia, por lo que no pueden tener métodos
  virtuales. Pero pueden implementar interfaces (polimorfismo por
  interfaz).

  SISTEMA DE PROCESAMIENTO DE PAGOS POLIMÓRFICO

  Diseña un sistema de procesamiento de pagos usando polimorfismo:

  Clase base abstracta `Pago`:
    • Monto (decimal), Fecha (DateTime), Descripcion
    • Metodo abstracto Procesar(): bool
    • Metodo virtual MostrarDetalle()
    • Metodo virtual CalcularComision(): decimal (default 0)

  Clases derivadas:
    • PagoTarjetaCredito:
      - NumeroTarjeta, CVV, FechaVencimiento
      - Comision: 3% del monto
      - Procesar: validar tarjeta (simulado)
    • PagoPayPal:
      - Email, Token
      - Comision: 2% del monto
      - Procesar: validar token
    • PagoTransferencia:
      - Banco, CuentaOrigen, CuentaDestino
      - Comision: 0.5% (min $5, max $50)
      - Procesar: verificar fondos
    • PagoEfectivo:
      - Sucursal, Recibo
      - Sin comision
      - Procesar: generar recibo

  Clase `ProcesadorPagos`:
    • Lista de Pago
    • AgregarPago(Pago)
    • ProcesarTodos(): procesa cada uno y registra resultado
    • TotalComisiones: decimal
    • TotalProcesado: decimal

    • Usa polimorfismo para procesarlos todos
    • Muestra detalle de cada uno
    • Muestra totales y comisiones
    • Demostrar que agregar un nuevo tipo de pago NO requiere

================================================================================
  CIERRE DEL CAPÍTULO
================================================================================

El polimorfismo es el corazón del diseño orientado a objetos flexible.
Permite escribir código que trabaja con abstracciones, no con concretos,
haciendo que el sistema sea extensible sin modificar lo existente.

Recuerda: virtual/override es la herramienta, pero el polimorfismo es el
concepto. Diseña tus clases pensando en cómo serán usadas polimórficamente.

En el próximo capítulo exploraremos las CLASES ABSTRACTAS, que son la base
perfecta para jerarquías polimórficas.
