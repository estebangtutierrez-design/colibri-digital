En el capítulo anterior aprendiste a definir clases y crear objetos con `new`.
Pero hay una pieza fundamental que apenas mencionamos: el CONSTRUCTOR.

El constructor es el método especial que se ejecuta automáticamente cuando se
crea un objeto. Es la primera oportunidad que tienes para establecer el estado
inicial del objeto, validar parámetros, inicializar recursos y garantizar que
el objeto comience su vida en un estado válido.

Sin constructores bien diseñados, tus objetos pueden terminar en estados
inconsistentes. Con ellos, tienes control total sobre cómo y con qué valores
se crean las instancias.

En este capítulo exploraremos:
  • Constructores por defecto e implícitos.
  • Constructores parametrizados.
  • Sobrecarga de constructores.
  • Encadenamiento con `: this()`.
  • Constructores estáticos.
  • Constructores privados y patrones de creación.
  • Inicializadores de objeto y expresiones con `new`.

Empecemos.

--------------------------------------------------------------------
  1. ¿QUÉ ES UN CONSTRUCTOR?
--------------------------------------------------------------------

Un constructor es un método especial que:

  • Se llama AUTOMÁTICAMENTE al usar `new`.
  • Tiene el MISMO NOMBRE que la clase.
  • NO TIENE tipo de retorno (ni siquiera `void`).
  • SIRVE para inicializar los campos del objeto.
  • Puede recibir parámetros para personalizar la inicialización.

Sintaxis básica:

```csharp
public class NombreClase
{
    // Constructor
    public NombreClase()
    {
        // Código de inicialización
    }
}
```

El constructor se ejecuta después de que los campos se inicializan a sus
valores predeterminados (0, false, null).

Orden de ejecución durante `new`:

  1. Se reserva memoria en el heap para el objeto.
  2. Los campos se inicializan a sus valores predeterminados.
  3. Se ejecutan los inicializadores de campo (si los hay).
  4. Se ejecuta el código del constructor.

```csharp
public class Demo
{
    public int a;        // Inicializado a 0
    public string b;     // Inicializado a null
    public bool c;       // Inicializado a false
    public int d = 42;   // Inicializado a 42 (inicializador de campo)

    public Demo()
    {
        // En este punto:
        // a = 0 (predeterminado)
        // b = null (predeterminado)
        // c = false (predeterminado)
        // d = 42 (por el inicializador de campo)

        a = 10;  // Sobrescribimos el valor predeterminado
    }
}
```

--------------------------------------------------------------------
  2. CONSTRUCTOR POR DEFECTO (DEFAULT CONSTRUCTOR)
--------------------------------------------------------------------

El constructor por defecto es aquel que NO recibe parámetros.

Si NO defines ningún constructor en tu clase, el compilador de C# genera
automáticamente un constructor por defecto público que no hace nada
(llamado "constructor implícito").

Si defines AL MENOS UN constructor (parametrizado o no), el compilador
NO genera el constructor implícito.

```csharp
public class SinConstructorDefinido
{
    public string Mensaje { get; set; }
}

// El compilador genera implícitamente:
// public SinConstructorDefinido() { }

// Podemos usarlo:
SinConstructorDefinido obj = new SinConstructorDefinido();
obj.Mensaje = "Hola";
```

```csharp
public class ConConstructorPropio
{
    public int Valor { get; set; }

    public ConConstructorPropio()
    {
        Valor = 1;
        Console.WriteLine("Constructor por defecto ejecutado.");
    }
}

ConConstructorPropio obj = new ConConstructorPropio();  // Imprime mensaje
```

Si defines un constructor parametrizado y aún quieres el constructor por
defecto, debes definirlo explícitamente:

```csharp
public class Producto
{
    public string Nombre { get; set; }
    public decimal Precio { get; set; }

    // Constructor parametrizado
    public Producto(string nombre, decimal precio)
    {
        Nombre = nombre;
        Precio = precio;
    }

    // Debemos definir explícitamente el constructor por defecto si lo queremos
    public Producto() { }
}

// Ahora ambos son válidos:
Producto p1 = new Producto();                          // Campos null/0
Producto p2 = new Producto("Laptop", 15000);           // Campos inicializados
```

--------------------------------------------------------------------
  3. CONSTRUCTORES PARAMETRIZADOS
--------------------------------------------------------------------

Los constructores parametrizados reciben argumentos que se usan para
inicializar los campos del objeto.

Permiten crear objetos en un estado específico desde el inicio.

```csharp
public class Persona
{
    public string Nombre { get; set; }
    public int Edad { get; set; }
    public string Email { get; set; }

    // Constructor con 2 parámetros
    public Persona(string nombre, int edad)
    {
        Nombre = nombre;
        Edad = edad;
        Email = "sin-correo@dominio.com";  // Valor por defecto
    }

    // Constructor con 3 parámetros (sobrecarga)
    public Persona(string nombre, int edad, string email)
    {
        Nombre = nombre;
        Edad = edad;
        Email = email;
    }
}

// Uso:
Persona p1 = new Persona("Ana", 25);
Persona p2 = new Persona("Luis", 30, "luis@email.com");

Console.WriteLine($"{p1.Nombre}, {p1.Edad}, {p1.Email}");
Console.WriteLine($"{p2.Nombre}, {p2.Edad}, {p2.Email}");
```

--------------------------------------------------------------------
  4. SOBRECARGA DE CONSTRUCTORES
--------------------------------------------------------------------

Al igual que los métodos, los constructores pueden SOBRECARGARSE: definir
múltiples constructores con diferentes listas de parámetros.

El compilador selecciona el constructor correcto según los argumentos
proporcionados en `new`.

```csharp
public class Rectangulo
{
    public double Ancho { get; set; }
    public double Alto { get; set; }

    // Constructor sin parámetros (cuadrado de 1x1)
    public Rectangulo()
    {
        Ancho = 1;
        Alto = 1;
    }

    // Constructor con 1 parámetro (cuadrado)
    public Rectangulo(double lado)
    {
        Ancho = lado;
        Alto = lado;
    }

    // Constructor con 2 parámetros
    public Rectangulo(double ancho, double alto)
    {
        Ancho = ancho;
        Alto = alto;
    }
}

// Uso de cada sobrecarga:
Rectangulo r1 = new Rectangulo();        // 1x1
Rectangulo r2 = new Rectangulo(5);       // 5x5 (cuadrado)
Rectangulo r3 = new Rectangulo(4, 6);    // 4x6

double area1 = r1.Ancho * r1.Alto;  // 1
double area2 = r2.Ancho * r2.Alto;  // 25
double area3 = r3.Ancho * r3.Alto;  // 24
```

Reglas de la sobrecarga:

  • Los parámetros deben diferir en número o tipo.
  • El tipo de retorno NO se considera (los constructores no tienen retorno).
  • El compilador usa el "mejor ajuste" según los argumentos.

```csharp
public class EjemploSobrecarga
{
    public EjemploSobrecarga() { }
    public EjemploSobrecarga(int a) { }
    public EjemploSobrecarga(double a) { }
    public EjemploSobrecarga(int a, string b) { }
    // public EjemploSobrecarga(int b) { }  // ERROR: duplicado
}
```

--------------------------------------------------------------------
  5. ENCADENAMIENTO DE CONSTRUCTORES WITH `: this()`
--------------------------------------------------------------------

El encadenamiento permite que un constructor llame a otro constructor de
la MISMA clase usando la sintaxis `: this(parámetros)`.

Esto evita duplicar código de inicialización.

```csharp
public class Empleado
{
    public string Nombre { get; set; }
    public string Puesto { get; set; }
    public decimal Salario { get; set; }
    public DateTime FechaIngreso { get; set; }

    // Constructor principal (con toda la lógica)
    public Empleado(string nombre, string puesto, decimal salario, DateTime fechaIngreso)
    {
        Nombre = nombre;
        Puesto = puesto;
        Salario = salario;
        FechaIngreso = fechaIngreso;
    }

    // Constructor con 3 parámetros (sin fecha, usa hoy)
    public Empleado(string nombre, string puesto, decimal salario)
        : this(nombre, puesto, salario, DateTime.Now)
    {
    }

    // Constructor con 2 parámetros (puesto por defecto: "Sin asignar")
    public Empleado(string nombre, decimal salario)
        : this(nombre, "Sin asignar", salario, DateTime.Now)
    {
    }

    // Constructor sin parámetros (valores por defecto)
    public Empleado()
        : this("Sin nombre", "Sin asignar", 0, DateTime.Now)
    {
    }
}

// Uso:
Empleado e1 = new Empleado("Ana", "Gerente", 50000, new DateTime(2020, 1, 15));
Empleado e2 = new Empleado("Luis", "Analista", 35000);      // Fecha = hoy
Empleado e3 = new Empleado("Carlos", 25000);                // Puesto = "Sin asignar"
Empleado e4 = new Empleado();                                // Todo por defecto
```

Ventajas del encadenamiento:

  ✓ Reduce duplicación de código.
  ✓ La lógica de inicialización está en un solo lugar.
  ✓ Facilita el mantenimiento (cambias en un solo sitio).
  ✓ Obliga a pasar por las validaciones del constructor principal.

Ejemplo con validación en el constructor principal:

```csharp
public class CuentaBancaria
{
    public string Titular { get; set; }
    public double Saldo { get; set; }
    public string TipoCuenta { get; set; }

    // Constructor principal con validación
    public CuentaBancaria(string titular, double saldoInicial, string tipoCuenta)
    {
        if (string.IsNullOrWhiteSpace(titular))
            throw new ArgumentException("El titular es obligatorio.");

        if (saldoInicial < 0)
            throw new ArgumentException("El saldo inicial no puede ser negativo.");

        Titular = titular;
        Saldo = saldoInicial;
        TipoCuenta = tipoCuenta ?? "Ahorros";
    }

    // Constructor con 2 parámetros (tipo por defecto)
    public CuentaBancaria(string titular, double saldoInicial)
        : this(titular, saldoInicial, "Ahorros")
    {
    }

    // Constructor con 1 parámetro (saldo 0, tipo por defecto)
    public CuentaBancaria(string titular)
        : this(titular, 0, "Ahorros")
    {
    }
}
```

El flujo de ejecución con `: this()`:

  1. Se ejecuta el constructor al que se encadena PRIMERO.
  2. Luego se ejecuta el cuerpo del constructor actual.

```csharp
public class DemoFlujo
{
    public DemoFlujo()
        : this(42)
    {
        Console.WriteLine("Constructor sin parámetros");
    }

    public DemoFlujo(int valor)
        : this(valor, "por defecto")
    {
        Console.WriteLine("Constructor con 1 parámetro");
    }

    public DemoFlujo(int valor, string texto)
    {
        Console.WriteLine("Constructor con 2 parámetros (el primero)");
    }
}

// new DemoFlujo() imprime:
// Constructor con 2 parámetros (el primero)
// Constructor con 1 parámetro
// Constructor sin parámetros
```

--------------------------------------------------------------------
  6. INICIALIZADORES DE CAMPO
--------------------------------------------------------------------

Los campos pueden inicializarse directamente en su declaración, antes de
que se ejecute el constructor. Esto se llama "inicializador de campo".

```csharp
public class Configuracion
{
    // Inicializadores de campo
    public string Servidor = "localhost";
    public int Puerto = 8080;
    public int Timeout = 30;
    public readonly Guid Identificador = Guid.NewGuid();
    public static DateTime InicioAplicacion = DateTime.Now;
}

// Se ejecutan ANTES del constructor
```

Orden de inicialización:

  1. Campos estáticos (en orden de declaración).
  2. Inicializadores de campos de instancia (en orden de declaración).
  3. Constructor de la clase base.
  4. Constructor de la clase actual.

```csharp
public class Pedido
{
    public int Id { get; set; }
    public DateTime FechaCreacion { get; set; } = DateTime.Now;
    public List<string> Productos { get; set; } = new List<string>();
    public string Estado { get; set; }

    // El constructor puede sobrescribir los valores de los inicializadores
    public Pedido()
    {
        Estado = "Pendiente";
    }
}

// Al crear: FechaCreacion = DateTime.Now (del inicializador)
// Estado = "Pendiente" (del constructor)
```

--------------------------------------------------------------------
  7. CONSTRUCTORES ESTÁTICOS
--------------------------------------------------------------------

Un constructor estático se usa para inicializar los miembros ESTÁTICOS de
una clase. Características:

  • Se declara con `static`.
  • NO tiene parámetros.
  • NO tiene modificadores de acceso.
  • Se ejecuta AUTOMÁTICAMENTE ANTES de:
      - Crear cualquier instancia de la clase.
      - Acceder a cualquier miembro estático de la clase.
  • Se ejecuta UNA SOLA VEZ en toda la vida de la aplicación.
  • No se puede llamar explícitamente.

```csharp
public class ConfiguracionGlobal
{
    public static string Version { get; private set; }
    public static DateTime FechaCompilacion { get; private set; }
    public static string Entorno { get; private set; }

    // Constructor estático
    static ConfiguracionGlobal()
    {
        Console.WriteLine("Constructor estático ejecutado.");
        Version = "2.1.0";
        FechaCompilacion = DateTime.Now;
        Entorno = ObtenerEntorno();
    }

    private static string ObtenerEntorno()
    {
        return Environment.GetEnvironmentVariable("ASPNETCORE_ENVIRONMENT") ?? "Production";
    }
}

// El constructor estático se ejecuta aquí (primera vez que se accede):
Console.WriteLine(ConfiguracionGlobal.Version);
```

Usos comunes del constructor estático:

  • Cargar configuración desde archivos.
  • Inicializar conexiones a bases de datos (singleton).
  • Registrar tipos en contenedores DI.
  • Inicializar datos compartidos (por ejemplo, cargar plantillas).

```csharp
public class GestorBaseDeDatos
{
    public static string CadenaConexion { get; private set; }

    static GestorBaseDeDatos()
    {
        try
        {
            CadenaConexion = File.ReadAllText("config/db_connection.txt");
            Console.WriteLine("Cadena de conexión cargada exitosamente.");
        }
        catch (Exception ex)
        {
            CadenaConexion = "DefaultConnection";
            Console.WriteLine($"Error cargando configuración: {ex.Message}");
        }
    }

    public void Conectar()
    {
        Console.WriteLine($"Conectando con: {CadenaConexion}");
    }
}
```

Consideraciones:

  • Si el constructor estático lanza una excepción, la clase queda inutilizable
    para toda la duración del AppDomain.
  • El constructor estático se ejecuta en un hilo seguro (el runtime garantiza
    que se ejecute una sola vez, incluso en multihilo).
  • No se puede combinar `static` con `: base()` o `: this()`.

--------------------------------------------------------------------
  8. CONSTRUCTORES PRIVADOS
--------------------------------------------------------------------

Un constructor privado solo puede ser llamado desde dentro de la propia clase.
Se usa para:

  • Patrón Singleton:    Una sola instancia de la clase.
  • Clases estáticas:    Antes de C# 2.0 (hoy usamos `static class`).
  • Factory methods:     Controlar cómo se crean los objetos.

```csharp
public class Singleton
{
    private static Singleton _instancia;
    private static readonly object _lock = new object();

    // Constructor privado: nadie externo puede hacer new Singleton()
    private Singleton()
    {
        Console.WriteLine("Instancia única creada.");
    }

    // Método público estático para obtener la instancia
    public static Singleton ObtenerInstancia()
    {
        if (_instancia == null)
        {
            lock (_lock)
            {
                if (_instancia == null)
                {
                    _instancia = new Singleton();
                }
            }
        }
        return _instancia;
    }

    public void HacerAlgo()
    {
        Console.WriteLine("Singleton haciendo algo.");
    }
}

// Uso:
// Singleton s = new Singleton();  // ERROR: el constructor es privado

Singleton s1 = Singleton.ObtenerInstancia();
Singleton s2 = Singleton.ObtenerInstancia();

Console.WriteLine(s1 == s2);  // True (misma instancia)
```

Otro ejemplo: Factory Method

```csharp
public class Punto
{
    public double X { get; }
    public double Y { get; }

    // Constructor privado
    private Punto(double x, double y)
    {
        X = x;
        Y = y;
    }

    // Factory methods
    public static Punto CrearOrigen()
    {
        return new Punto(0, 0);
    }

    public static Punto CrearDesdeCoordenadas(double x, double y)
    {
        return new Punto(x, y);
    }

    public static Punto CrearDesdePolar(double angulo, double distancia)
    {
        double x = distancia * Math.Cos(angulo);
        double y = distancia * Math.Sin(angulo);
        return new Punto(x, y);
    }
}

// Uso:
Punto origen = Punto.CrearOrigen();
Punto punto1 = Punto.CrearDesdeCoordenadas(3, 4);
Punto punto2 = Punto.CrearDesdePolar(Math.PI / 4, 10);
```

--------------------------------------------------------------------
  9. INICIALIZADORES DE OBJETO (OBJECT INITIALIZERS)
--------------------------------------------------------------------

Los inicializadores de objeto permiten asignar valores a propiedades al
momento de crear el objeto, sin necesidad de un constructor parametrizado.

Sintaxis:

```csharp
NombreClase obj = new NombreClase
{
    Propiedad1 = valor1,
    Propiedad2 = valor2
};
```

Ejemplo:

```csharp
public class Producto
{
    public string Nombre { get; set; }
    public decimal Precio { get; set; }
    public int Stock { get; set; }
    public string Categoria { get; set; }
}

// Sin inicializador (requiere constructor por defecto + asignaciones)
Producto p1 = new Producto();
p1.Nombre = "Laptop";
p1.Precio = 20000;
p1.Stock = 10;
p1.Categoria = "Electrónica";

// Con inicializador (más compacto)
Producto p2 = new Producto
{
    Nombre = "Laptop",
    Precio = 20000,
    Stock = 10,
    Categoria = "Electrónica"
};
```

Los inicializadores también pueden combinarse con constructores:

```csharp
public class Empleado
{
    public string Nombre { get; set; }
    public int Edad { get; set; }
    public string Puesto { get; set; }
    public decimal Salario { get; set; }

    public Empleado(string nombre)
    {
        Nombre = nombre;
    }
}

// Constructor + inicializador
Empleado emp = new Empleado("Ana")
{
    Edad = 30,
    Puesto = "Ingeniera",
    Salario = 60000
};
```

--------------------------------------------------------------------
  10. EXPRESIONES CON `new` Y TIPOS ANÓNIMOS
--------------------------------------------------------------------

C# también permite usar `new` para crear tipos anónimos (sin definir una
clase explícita).

```csharp
var persona = new
{
    Nombre = "Ana",
    Edad = 30,
    Ciudad = "Lima"
};

Console.WriteLine(persona.Nombre);  // Ana
Console.WriteLine(persona.GetType().Name);  // <>f__AnonymousType0`3
```

Los tipos anónimos son inmutables (sus propiedades son solo lectura) y se
usan comúnmente en consultas LINQ.

```csharp
var productos = new[]
{
    new { Id = 1, Nombre = "Laptop", Precio = 20000 },
    new { Id = 2, Nombre = "Mouse", Precio = 150 },
    new { Id = 3, Nombre = "Teclado", Precio = 300 }
};

var caros = productos.Where(p => p.Precio > 500);
foreach (var p in caros)
{
    Console.WriteLine($"{p.Nombre}: ${p.Precio}");
}
```

--------------------------------------------------------------------
  11. EJEMPLO COMPLETO: SISTEMA DE RESERVAS
--------------------------------------------------------------------

```csharp
using System;

namespace SistemaReservas
{
    public class Reserva
    {
        public int Id { get; set; }
        public string Cliente { get; set; }
        public DateTime FechaReserva { get; set; }
        public DateTime FechaEvento { get; set; }
        public int NumeroPersonas { get; set; }
        public string Estado { get; private set; }
        public decimal MontoTotal { get; set; }

        private static int _contador = 0;
        public static int TotalReservas => _contador;
        public static decimal IngresosTotales { get; private set; }

        // Constructor principal (con toda la lógica)
        public Reserva(string cliente, DateTime fechaEvento,
                       int numeroPersonas, decimal montoPorPersona)
        {
            if (string.IsNullOrWhiteSpace(cliente))
                throw new ArgumentException("El nombre del cliente es obligatorio.");

            if (fechaEvento <= DateTime.Now)
                throw new ArgumentException("La fecha del evento debe ser futura.");

            if (numeroPersonas <= 0)
                throw new ArgumentException("Debe haber al menos 1 persona.");

            if (montoPorPersona <= 0)
                throw new ArgumentException("El monto por persona debe ser positivo.");

            Id = ++_contador;
            Cliente = cliente;
            FechaReserva = DateTime.Now;
            FechaEvento = fechaEvento;
            NumeroPersonas = numeroPersonas;
            Estado = "Confirmada";
            MontoTotal = numeroPersonas * montoPorPersona;

            IngresosTotales += MontoTotal;
        }

        // Constructor con solo cliente y fecha (valores por defecto)
        public Reserva(string cliente, DateTime fechaEvento)
            : this(cliente, fechaEvento, 2, 100)
        {
            Console.WriteLine("Reserva creada con valores por defecto (2 personas, $100 c/u).");
        }

        // Constructor sin parámetros (valores por defecto totales)
        public Reserva()
            : this("Cliente Genérico", DateTime.Now.AddDays(7), 1, 50)
        {
        }

        // Métodos
        public void Cancelar()
        {
            if (Estado == "Cancelada")
            {
                Console.WriteLine($"La reserva {Id} ya está cancelada.");
                return;
            }

            Estado = "Cancelada";
            IngresosTotales -= MontoTotal;
            Console.WriteLine($"Reserva {Id} cancelada.");
        }

        public void ModificarPersonas(int nuevoNumero, decimal montoPorPersona)
        {
            if (Estado == "Cancelada")
            {
                Console.WriteLine("No se puede modificar una reserva cancelada.");
                return;
            }

            decimal montoAnterior = MontoTotal;
            NumeroPersonas = nuevoNumero;
            MontoTotal = nuevoNumero * montoPorPersona;

            IngresosTotales += MontoTotal - montoAnterior;
            Console.WriteLine($"Reserva {Id} actualizada: {NumeroPersonas} personas, total ${MontoTotal}");
        }

        public void MostrarResumen()
        {
            Console.WriteLine("=================================");
            Console.WriteLine($"RESERVA #{Id}");
            Console.WriteLine($"Cliente:     {Cliente}");
            Console.WriteLine($"Fecha Evento: {FechaEvento:dd/MM/yyyy}");
            Console.WriteLine($"Personas:    {NumeroPersonas}");
            Console.WriteLine($"Total:       ${MontoTotal:F2}");
            Console.WriteLine($"Estado:      {Estado}");
            Console.WriteLine("=================================");
        }

        public static void MostrarEstadisticas()
        {
            Console.WriteLine($"Total reservas: {TotalReservas}");
            Console.WriteLine($"Ingresos totales: ${IngresosTotales:F2}");
        }
    }

    public class Programa
    {
        public static void Main()
        {
            // Usar constructor con validación
            Reserva r1 = new Reserva("Ana López", new DateTime(2025, 6, 15), 10, 250);
            r1.MostrarResumen();

            // Usar constructor con valores por defecto
            Reserva r2 = new Reserva("Carlos Ruiz", new DateTime(2025, 7, 20));
            r2.MostrarResumen();

            // Usar constructor por defecto
            Reserva r3 = new Reserva();
            r3.MostrarResumen();

            // Modificar una reserva
            r2.ModificarPersonas(5, 120);

            // Cancelar una reserva
            r1.Cancelar();
            r1.Cancelar();  // Intento doble

            // Estadísticas
            Reserva.MostrarEstadisticas();

            // Inicializador de objeto
            Reserva r4 = new Reserva
            {
                Cliente = "Pedro Infante",
                FechaEvento = new DateTime(2025, 12, 25)
            };
            // Nota: esto solo funciona si hay un constructor sin parámetros
        }
    }
}
```

--------------------------------------------------------------------
  12. BUENAS PRÁCTICAS CON CONSTRUCTORES
--------------------------------------------------------------------

  ✓ Valida los parámetros en el constructor y lanza excepciones si son
    inválidos (ArgumentException, ArgumentNullException, etc.).

  ✓ Usa encadenamiento `: this()` para evitar duplicar lógica.

  ✓ Mantén los constructores simples: solo deben inicializar el estado,
    no hacer trabajo pesado (conectar BD, leer archivos grandes).

  ✓ Si un constructor tiene muchos parámetros (más de 4-5), considera usar
    un patrón Builder o un objeto de parámetros.

  ✓ No llames a métodos virtuales en el constructor (pueden ejecutarse
    antes de que la clase derivada esté completamente inicializada).

  ✓ Documenta los constructores con comentarios XML.

  ✓ Usa `readonly` para campos que se asignan en el constructor y no
    cambian después.

```csharp
/// <summary>
/// Representa un producto en el inventario.
/// </summary>
public class Producto
{
    /// <summary>
    /// Crea un producto con nombre y precio.
    /// </summary>
    /// <param name="nombre">Nombre del producto (obligatorio).</param>
    /// <param name="precio">Precio unitario (debe ser > 0).</param>
    /// <exception cref="ArgumentNullException">Si nombre es null.</exception>
    /// <exception cref="ArgumentOutOfRangeException">Si precio <= 0.</exception>
    public Producto(string nombre, decimal precio)
    {
        Nombre = nombre ?? throw new ArgumentNullException(nameof(nombre));
        Precio = precio > 0 ? precio : throw new ArgumentOutOfRangeException(nameof(precio));
    }

    public string Nombre { get; }
    public decimal Precio { get; }
}
```

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • Los CONSTRUCTORES son métodos especiales que se ejecutan al crear un
    objeto con `new`. No tienen tipo de retorno y se llaman como la clase.

  • El CONSTRUCTOR POR DEFECTO no tiene parámetros. Si no defines ningún
    constructor, el compilador genera uno implícito.

  • Los CONSTRUCTORES PARAMETRIZADOS reciben argumentos para inicializar
    el objeto en un estado específico.

  • La SOBRECARGA permite definir múltiples constructores con diferentes
    parámetros.

  • El ENCADENAMIENTO con `: this()` permite que un constructor llame a
    otro, evitando duplicación.

  • Los CONSTRUCTORES ESTÁTICOS inicializan miembros estáticos. Se ejecutan
    una sola vez, automáticamente.

  • Los CONSTRUCTORES PRIVADOS controlan cómo y cuándo se crean objetos
    (Singleton, Factory).

  • Los INICIALIZADORES DE OBJETO y de campo ofrecen formas alternativas
    de inicializar datos.

  • Siempre valida parámetros en el constructor para garantizar objetos
    en estado válido.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ❓ ¿Puedo tener un constructor `void`?
  ───
  No. Los constructores no tienen tipo de retorno, ni siquiera `void`.
  Si pones `void`, el compilador lo trata como un método normal, no
  como constructor.

  ❓ ¿Puedo llamar a un constructor manualmente después de creado el objeto?
  ───
  No directamente. Los constructores solo se invocan con `new`. No
  puedes llamar a `obj.Constructor()`. Si necesitas reinicializar,
  crea un método `Reset()`.

  ❓ ¿Puedo tener un constructor `async`?
  ───
  No. Los constructores no pueden ser async. Si necesitas operaciones
  asíncronas durante la inicialización, usa un patrón Factory:
  `public static async Task<Clase> CrearAsync() { ... }`.

  ❓ ¿Qué es mejor: constructor con muchos parámetros o inicializadores?
  ───
  Depende. Los constructores garantizan que el objeto se cree en un
  estado válido (validación). Los inicializadores son más flexibles
  pero no validan. Usa constructores cuando el estado sea obligatorio;
  inicializadores para propiedades opcionales.

  ❓ ¿Los constructores pueden ser `private protected`?
  ───
  Sí, pueden tener cualquier modificador de acceso: public, private,
  protected, internal, protected internal, private protected.

  ❓ ¿Puedo tener un constructor en un `struct`?
  ───
  Sí. Los structs pueden tener constructores parametrizados (C# 10+)
  y también constructores sin parámetros (C# 10+). Antes de C# 10,
  los structs siempre tenían un constructor implícito sin parámetros.

  ❓ ¿Se heredan los constructores?
  ───
  No. Las clases derivadas no heredan los constructores de la clase
  base. Deben definir los suyos propios y pueden llamar al constructor
  base con `: base()`.

================================================================================
  CIERRE DEL CAPÍTULO
================================================================================

Los constructores son mucho más que simples inicializadores. Son la puerta
de entrada controlada al mundo de los objetos, permitiéndonos garantizar que
cada instancia comience su vida en un estado válido y predecible.

Hemos visto desde el constructor por defecto hasta el constructor estático,
pasando por el encadenamiento y los constructores privados. Dominar los
constructores es esencial para escribir código robusto y mantenible.

DESTRUCTORES y el patrón Dispose, para gestionar la limpieza de recursos.
