Si las clases abstractas son contratos parciales, las INTERFACES son
contratos PUROS: definen QUÉ hacer, sin decir CÓMO.

Una interfaz en C# es un tipo de referencia que define un conjunto de
miembros (métodos, propiedades, eventos, indexadores) sin implementación.
Cualquier clase o struct que implemente la interfaz se compromete a
proporcionar la implementación de todos esos miembros.

Las interfaces son fundamentales en .NET. El framework entero está
diseñado en torno a interfaces: IComparable, IEnumerable, IDisposable,
INotifyPropertyChanged, IQueryable... son la columna vertebral de la
reutilización y el desacoplamiento.

En este capítulo exploraremos:

  • Definición e implementación de interfaces.
  • Interfaces del framework: IComparable, IEnumerable, IComparer.
  • Implementación explícita de interfaces.
  • Múltiples interfaces y resolución de conflictos.
  • Default interface methods (C# 8+).
  • Principio de Segregación de Interfaces (ISP).
  • Interfaces vs clases abstractas.

--------------------------------------------------------------------
  1. ¿QUÉ ES UNA INTERFAZ?
--------------------------------------------------------------------

Una interfaz define un CONTRATO. Es una declaración de capacidades que
una clase o struct se compromete a implementar.

Sintaxis:

```csharp
public interface INombreInterfaz
{
    // Metodo (sin implementacion)
    void HacerAlgo();

    // Propiedad
    int Valor { get; set; }

    // Evento
    event EventHandler AlgoOcurrio;

    // Indexador
    int this[int index] { get; set; }
}
```

Características:

  • Los miembros son implícitamente `public` (no se usan modificadores).
  • No pueden tener campos, constantes ni constructores.
  • No pueden tener implementación (excepto default methods C# 8+).
  • Una clase/struct puede implementar VARIAS interfaces.
  • Las interfaces pueden heredar de otras interfaces.

```csharp
// Definicion de interfaz
public interface IVolador
{
    void Volar();
    int AltitudMaxima { get; }
}

// Implementacion
public class Aguila : IVolador
{
    public void Volar()
    {
        Console.WriteLine("El aguila vuela a gran altura.");
    }

    public int AltitudMaxima => 3000;
}

public class Avion : IVolador
{
    public void Volar()
    {
        Console.WriteLine("El avion vuela usando motores.");
    }

    public int AltitudMaxima => 10000;
}
```

--------------------------------------------------------------------
  2. IMPLEMENTACIÓN DE INTERFACES
--------------------------------------------------------------------

Para implementar una interfaz, la clase/struct debe proporcionar
implementación para TODOS sus miembros.

```csharp
public interface IReproducible
{
    void Reproducir();
    void Pausar();
    void Detener();
    string ObtenerInfo();
}

public class Cancion : IReproducible
{
    public string Titulo { get; set; }
    public string Artista { get; set; }

    public void Reproducir()
    {
        Console.WriteLine($"Reproduciendo: {Titulo} - {Artista}");
    }

    public void Pausar()
    {
        Console.WriteLine("Cancion pausada.");
    }

    public void Detener()
    {
        Console.WriteLine("Cancion detenida.");
    }

    public string ObtenerInfo()
    {
        return $"{Titulo} - {Artista}";
    }
}

public class Video : IReproducible
{
    public string Nombre { get; set; }

    public void Reproducir()
    {
        Console.WriteLine($"Reproduciendo video: {Nombre}");
    }

    public void Pausar()
    {
        Console.WriteLine("Video pausado.");
    }

    public void Detener()
    {
        Console.WriteLine("Video detenido.");
    }

    public string ObtenerInfo()
    {
        return $"Video: {Nombre}";
    }
}

// Uso polimorfico:
IReproducible[] items = {
    new Cancion { Titulo = "Bohemian Rhapsody", Artista = "Queen" },
    new Video { Nombre = "Tutorial C#" }
};

foreach (var item in items)
{
    item.Reproducir();
    Console.WriteLine(item.ObtenerInfo());
}
```

--------------------------------------------------------------------
  3. INTERFACES DEL FRAMEWORK .NET
--------------------------------------------------------------------

.NET proporciona muchas interfaces fundamentales. Conocerlas es esencial.

IComparable: para ordenar objetos

```csharp
public class Persona : IComparable<Persona>
{
    public string Nombre { get; set; }
    public int Edad { get; set; }

    public int CompareTo(Persona other)
    {
        if (other == null) return 1;
        return this.Edad.CompareTo(other.Edad);
    }

    public override string ToString() => $"{Nombre} ({Edad})";
}

// Uso:
List<Persona> personas = new List<Persona>
{
    new Persona { Nombre = "Ana", Edad = 30 },
    new Persona { Nombre = "Luis", Edad = 25 },
    new Persona { Nombre = "Carlos", Edad = 35 }
};

personas.Sort();  // Ordena por edad (gracias a IComparable)
Console.WriteLine(string.Join(", ", personas));  // Luis (25), Ana (30), Carlos (35)
```

IComparer: para ordenar con criterios alternativos

```csharp
public class PersonaNombreComparer : IComparer<Persona>
{
    public int Compare(Persona x, Persona y)
    {
        return string.Compare(x?.Nombre, y?.Nombre, StringComparison.Ordinal);
    }
}

// Uso:
personas.Sort(new PersonaNombreComparer());  // Ordena por nombre
```

IEnumerable: para recorrer colecciones con foreach

```csharp
public class DiasSemana : IEnumerable<string>
{
    private string[] _dias = { "Lun", "Mar", "Mie", "Jue", "Vie", "Sab", "Dom" };

    public IEnumerator<string> GetEnumerator()
    {
        foreach (var dia in _dias)
            yield return dia;
    }

    System.Collections.IEnumerator System.Collections.IEnumerable.GetEnumerator()
    {
        return GetEnumerator();
    }
}

// Uso:
DiasSemana dias = new DiasSemana();
foreach (var dia in dias)
    Console.WriteLine(dia);
```

ICloneable: para clonar objetos

```csharp
public class Producto : ICloneable
{
    public string Nombre { get; set; }
    public decimal Precio { get; set; }

    public object Clone()
    {
        return new Producto { Nombre = this.Nombre, Precio = this.Precio };
    }
}

// Uso:
Producto p1 = new Producto { Nombre = "Laptop", Precio = 15000 };
Producto p2 = (Producto)p1.Clone();
```

--------------------------------------------------------------------
  4. IMPLEMENTACIÓN EXPLÍCITA DE INTERFACES
--------------------------------------------------------------------

Cuando una clase implementa dos interfaces con el mismo método, o cuando
queremos ocultar un miembro de la interfaz de la API pública de la clase,
usamos implementación explícita.

Sintaxis: `void IInterfaz.Metodo() { }` (sin modificador de acceso).

```csharp
public interface IGrafico
{
    void Dibujar();
}

public interface IImprimible
{
    void Dibujar();  // Mismo nombre!
}

public class Documento : IGrafico, IImprimible
{
    // Implementacion explicita: solo accesible a traves de la interfaz
    void IGrafico.Dibujar()
    {
        Console.WriteLine("Dibujando grafico del documento.");
    }

    void IImprimible.Dibujar()
    {
        Console.WriteLine("Imprimiendo documento.");
    }

    // Metodo propio (opcional)
    public void Mostrar()
    {
        Console.WriteLine("Mostrando documento.");
    }
}

// Uso:
Documento doc = new Documento();
doc.Mostrar();
// doc.Dibujar();  // ERROR: ambiguo

IGrafico g = doc;
g.Dibujar();  // "Dibujando grafico del documento."

IImprimible i = doc;
i.Dibujar();  // "Imprimiendo documento."
```

Otro uso común de implementación explícita: ocultar miembros "peligrosos":

```csharp
public class ColeccionSegura : IEnumerable<int>
{
    private int[] _datos = { 1, 2, 3, 4, 5 };

    // GetEnumerator normal (publico, seguro)
    public IEnumerator<int> GetEnumerator()
    {
        foreach (var d in _datos)
            yield return d;
    }

    // Implementacion explicita del no-generico (oculto)
    System.Collections.IEnumerator System.Collections.IEnumerable.GetEnumerator()
    {
        return GetEnumerator();
    }
}
```

--------------------------------------------------------------------
  5. INTERFACES CON PROPIEDADES, EVENTOS E INDEXADORES
--------------------------------------------------------------------

Las interfaces pueden declarar propiedades, eventos e indexadores:

```csharp
public interface IAlmacenable
{
    int Id { get; set; }
    string Nombre { get; set; }
    DateTime FechaCreacion { get; }

    event EventHandler<EventArgs> DatosCambiados;
}

public interface IListaPersonalizada<T>
{
    T this[int index] { get; set; }
    int Count { get; }
    void Add(T item);
    T RemoveAt(int index);
}

public class MiLista<T> : IListaPersonalizada<T>
{
    private List<T> _items = new List<T>();

    public T this[int index]
    {
        get => _items[index];
        set => _items[index] = value;
    }

    public int Count => _items.Count;

    public void Add(T item) => _items.Add(item);

    public T RemoveAt(int index)
    {
        var item = _items[index];
        _items.RemoveAt(index);
        return item;
    }
}
```

--------------------------------------------------------------------
  6. HERENCIA ENTRE INTERFACES
--------------------------------------------------------------------

Las interfaces pueden heredar de otras interfaces:

```csharp
public interface IEntidad
{
    int Id { get; }
}

public interface IAuditable : IEntidad
{
    DateTime FechaCreacion { get; }
    string CreadoPor { get; }
}

public interface IEliminable : IEntidad
{
    bool Eliminado { get; }
    DateTime? FechaEliminacion { get; }
}

// Una clase que implementa IAuditable debe implementar
// todos los miembros de IAuditable E IEntidad:
public class Producto : IAuditable
{
    public int Id { get; set; }
    public DateTime FechaCreacion { get; set; }
    public string CreadoPor { get; set; }
    public string Nombre { get; set; }
}

// Herencia multiple de interfaces:
public class Pedido : IAuditable, IEliminable
{
    public int Id { get; set; }
    public DateTime FechaCreacion { get; set; }
    public string CreadoPor { get; set; }
    public bool Eliminado { get; set; }
    public DateTime? FechaEliminacion { get; set; }
    public decimal Total { get; set; }
}
```

--------------------------------------------------------------------
  7. DEFAULT INTERFACE METHODS (C# 8+)
--------------------------------------------------------------------

Desde C# 8, las interfaces pueden tener implementaciones por defecto.
Esto permite agregar nuevos métodos a interfaces existentes sin romper
las clases que las implementan.

```csharp
public interface ILogger
{
    void Log(string mensaje);

    // Default implementation (C# 8+)
    void LogInfo(string mensaje)
    {
        Log($"[INFO] {mensaje}");
    }

    void LogError(string mensaje)
    {
        Log($"[ERROR] {mensaje}");
    }

    void LogWarning(string mensaje)
    {
        Log($"[WARNING] {mensaje}");
    }
}

public class LoggerConsola : ILogger
{
    public void Log(string mensaje)
    {
        Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] {mensaje}");
    }

    // No necesitamos implementar LogInfo, LogError, LogWarning
    // porque ya tienen implementacion por defecto
}

// Uso:
ILogger logger = new LoggerConsola();
logger.Log("Mensaje directo");
logger.LogInfo("Esto es informativo");
logger.LogError("Algo salio mal");
```

Si una clase quiere personalizar un default method, simplemente lo
implementa:

```csharp
public class LoggerDecorado : ILogger
{
    public void Log(string mensaje)
    {
        Console.WriteLine($">>> {mensaje} <<<");
    }

    public void LogError(string mensaje)
    {
        Log($"*** ERROR: {mensaje} ***");  // Personalizado
    }
    // LogInfo y LogWarning usan la implementacion por defecto
}
```

--------------------------------------------------------------------
  8. PRINCIPIO DE SEGREGACIÓN DE INTERFACES (ISP)
--------------------------------------------------------------------

El Principio de Segregación de Interfaces (ISP, parte de SOLID) dice:
"Los clientes no deben ser forzados a depender de interfaces que no usan."

Es mejor tener interfaces PEQUEÑAS y ESPECÍFICAS que una grande y
generalista.

```csharp
// MAL: interfaz GRANDE (gorda)
public interface ITrabajador
{
    void Trabajar();
    void Comer();
    void Dormir();
    void Programar();
    void Diseniar();
    void Reunirse();
}

// Solo puede implementar todo aunque no lo necesite
public class Robot : ITrabajador
{
    public void Trabajar() { }
    public void Comer() { throw new NotSupportedException(); }  // No come
    public void Dormir() { throw new NotSupportedException(); }  // No duerme
    public void Programar() { }
    public void Diseniar() { }
    public void Reunirse() { }
}
```

```csharp
// BIEN: interfaces SEGREGADAS (especificas)
public interface ITrabajador
{
    void Trabajar();
}

public interface IAlimentable
{
    void Comer();
}

public interface IDescansable
{
    void Dormir();
}

public interface IProgramador
{
    void Programar();
}

public interface IDiseniador
{
    void Diseniar();
}

public interface IColaborador
{
    void Reunirse();
}

// Robot solo implementa lo que necesita:
public class Robot : ITrabajador, IProgramador
{
    public void Trabajar() => Console.WriteLine("Robot trabajando");
    public void Programar() => Console.WriteLine("Robot programando");
}

// Humano implementa todo:
public class Humano : ITrabajador, IAlimentable, IDescansable, IProgramador, IColaborador
{
    public void Trabajar() => Console.WriteLine("Trabajando");
    public void Comer() => Console.WriteLine("Comiendo");
    public void Dormir() => Console.WriteLine("Durmiendo");
    public void Programar() => Console.WriteLine("Programando");
    public void Reunirse() => Console.WriteLine("En reunion");
}
```

--------------------------------------------------------------------
  9. EJEMPLO COMPLETO: SISTEMA DE PAGOS CON INTERFACES
--------------------------------------------------------------------

```csharp
using System;

namespace SistemaPagos
{
    // Interfaces segun ISP
    public interface IPagable
    {
        decimal Monto { get; }
        bool ProcesarPago();
    }

    public interface IValidable
    {
        bool Validar();
    }

    public interface IReembolsable
    {
        bool Reembolsar();
    }

    public interface INotificable
    {
        void Notificar(string mensaje);
    }

    // Interfaces base para metodos de pago
    public interface IMetodoPago : IPagable, IValidable
    {
        string ObtenerResumen();
    }

    public interface IPagoTarjeta : IMetodoPago
    {
        string UltimosDigitos { get; }
    }

    public interface IPagoPayPal : IMetodoPago
    {
        string Email { get; }
    }

    // Implementaciones
    public class PagoTarjetaCredito : IPagoTarjeta, IReembolsable, INotificable
    {
        public string NumeroTarjeta { get; }
        public string CVV { get; }
        public decimal Monto { get; }
        public string UltimosDigitos => NumeroTarjeta?.Substring(NumeroTarjeta.Length - 4) ?? "0000";
        public bool Procesado { get; private set; }

        public PagoTarjetaCredito(string numero, string cvv, decimal monto)
        {
            NumeroTarjeta = numero;
            CVV = cvv;
            Monto = monto;
        }

        public bool Validar()
        {
            return NumeroTarjeta?.Length == 16
                && CVV?.Length == 3
                && Monto > 0;
        }

        public bool ProcesarPago()
        {
            if (!Validar()) return false;
            Console.WriteLine($"Procesando pago de ${Monto:F2} con tarjeta ****{UltimosDigitos}");
            Procesado = true;
            return true;
        }

        public bool Reembolsar()
        {
            if (!Procesado) return false;
            Console.WriteLine($"Reembolsando ${Monto:F2} a tarjeta ****{UltimosDigitos}");
            Procesado = false;
            return true;
        }

        public void Notificar(string mensaje)
        {
            Console.WriteLine($"Notificacion tarjeta: {mensaje}");
        }

        public string ObtenerResumen()
        {
            return $"Tarjeta ****{UltimosDigitos}: ${Monto:F2}";
        }
    }

    public class PagoPayPal : IPagoPayPal, IReembolsable
    {
        public string Email { get; }
        public string Token { get; }
        public decimal Monto { get; }

        public PagoPayPal(string email, string token, decimal monto)
        {
            Email = email;
            Token = token;
            Monto = monto;
        }

        public bool Validar()
        {
            return Email?.Contains("@") == true
                && Token?.Length > 5
                && Monto > 0;
        }

        public bool ProcesarPago()
        {
            if (!Validar()) return false;
            Console.WriteLine($"Procesando pago de ${Monto:F2} via PayPal ({Email})");
            return true;
        }

        public bool Reembolsar()
        {
            Console.WriteLine($"Reembolsando ${Monto:F2} a PayPal ({Email})");
            return true;
        }

        public string ObtenerResumen()
        {
            return $"PayPal ({Email}): ${Monto:F2}";
        }
    }

    public class PagoEfectivo : IMetodoPago
    {
        public decimal Monto { get; }
        public string CodigoRecibo { get; }

        public PagoEfectivo(decimal monto)
        {
            Monto = monto;
            CodigoRecibo = Guid.NewGuid().ToString("N").Substring(0, 8).ToUpper();
        }

        public bool Validar() => Monto > 0;

        public bool ProcesarPago()
        {
            if (!Validar()) return false;
            Console.WriteLine($"Pago en efectivo de ${Monto:F2}. Recibo: {CodigoRecibo}");
            return true;
        }

        public string ObtenerResumen()
        {
            return $"Efectivo: ${Monto:F2} (Recibo: {CodigoRecibo})";
        }
    }

    public class Programa
    {
        public static void Main()
        {
            IMetodoPago[] pagos = {
                new PagoTarjetaCredito("1234567890123456", "123", 150.50m),
                new PagoPayPal("cliente@email.com", "TOKEN123", 89.99m),
                new PagoEfectivo(200m)
            };

            foreach (var pago in pagos)
            {
                Console.WriteLine($"\n--- {pago.GetType().Name} ---");
                Console.WriteLine(pago.ObtenerResumen());
                Console.WriteLine($"Valido: {pago.Validar()}");
                Console.WriteLine($"Procesado: {pago.ProcesarPago()}");

                // Uso de interfaces especificas con is
                if (pago is IReembolsable reembolso)
                {
                    Console.WriteLine("(Reembolsable)");
                }

                if (pago is INotificable notif)
                {
                    notif.Notificar("Pago recibido");
                }
            }
        }
    }
}
```

--------------------------------------------------------------------
  10. INTERFACES vs CLASES ABSTRACTAS (REVISIÓN)
--------------------------------------------------------------------

┌─────────────────────────┬──────────────────────┬─────────────────────────┐
│      CARACTERÍSTICA      │   CLASE ABSTRACTA     │       INTERFAZ         │
├─────────────────────────┼──────────────────────┼─────────────────────────┤
│ Palabra clave            │ abstract class        │ interface              │
│ Instanciable             │ No                   │ No                     │
│ Herencia múltiple        │ No                   │ Sí                     │
│ Campos                   │ Sí                   │ No                     │
│ Constructores            │ Sí                   │ No                     │
│ Métodos con implement.   │ Sí                   │ Sí (C# 8+ default)     │
│ Modificadores de acceso  │ Todos                │ public (solo)          │
│ Constantes               │ Sí                   │ No                     │
│ Relación                 │ "Es-un" (jerarquía)  │ "Puede-hacer" (capac.) │
│ Uso principal            │ Estado + comportam.  │ Contrato puro          │
└─────────────────────────┴──────────────────────┴─────────────────────────┘

Regla de oro:
  • Si hay estado compartido (campos) -> clase abstracta.
  • Si es solo un contrato de comportamiento -> interfaz.
  • Si no sabes cuál usar -> prefiere interfaz (más flexible).

--------------------------------------------------------------------
  11. BUENAS PRÁCTICAS
--------------------------------------------------------------------

  ✓ Nombra interfaces con prefijo "I" (IComparable, IDisposable).
  ✓ Mantén las interfaces pequeñas y cohesivas (ISP).
  ✓ No crees interfaces con un solo método a menos que sea necesario.
  ✓ Prefiere interfaces sobre clases abstractas para contratos.
  ✓ Usa default interface methods con cuidado (pueden crear ambigüedad).
  ✓ Implementa explícitamente cuando haya conflictos de nombres.
  ✓ Programa contra interfaces, no contra implementaciones.

```csharp
// Programar contra interfaces (DI, desacoplamiento)
public interface IRepositorioProductos
{
    Producto ObtenerPorId(int id);
    void Guardar(Producto p);
}

public class RepositorioProductoSql : IRepositorioProductos { /* SQL */ }
public class RepositorioProductoJson : IRepositorioProductos { /* JSON */ }
public class RepositorioProductoMock : IRepositorioProductos { /* Test */ }
```

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • Una INTERFAZ define un contrato que las clases/structs deben cumplir.

  • Las interfaces pueden contener métodos, propiedades, eventos e
    indexadores (sin implementación, hasta C# 8).

  • Una clase puede implementar VARIAS interfaces.

  • .NET proporciona interfaces fundamentales: IComparable, IEnumerable,
    IDisposable, IComparer, ICloneable, etc.

  • La IMPLEMENTACIÓN EXPLÍCITA resuelve conflictos entre interfaces
    y oculta miembros.

  • Las interfaces pueden HEREDAR de otras interfaces.

  • DEFAULT INTERFACE METHODS (C# 8+) permiten agregar métodos con
    implementación por defecto.

  • ISP: prefiere interfaces pequeñas y específicas.

  • Programa contra interfaces, no contra implementaciones concretas.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ───
  ❓ ¿Puedo instanciar una interfaz?
  ───
  No. Pero puedes usar una variable de tipo interfaz para referenciar
  un objeto que la implemente.

  ───
  ❓ ¿Una interfaz puede tener campos?
  ───
  No (hasta C# 10). Las interfaces solo definen miembros sin estado.
  Si necesitas estado, usa clase abstracta.

  ───
  ❓ ¿Puedo implementar dos interfaces con el mismo método?
  ───
  Sí. Usa implementación explícita para diferenciarlos.

  ───
  ❓ ¿Qué interfaz debo usar para ordenar objetos?
  ───
  IComparable<T> para orden natural de la clase. IComparer<T> para
  criterios de ordenación alternativos.

  ───
  ❓ ¿Qué interfaz hace que una clase funcione con foreach?
  ───
  IEnumerable<T> (o IEnumerable). Proporciona GetEnumerator().

  ───
  ❓ ¿Las interfaces mejoran el rendimiento?
  ───
  No necesariamente. Las llamadas a través de interfaces tienen un
  costo de indirección virtual. Pero los beneficios de diseño superan
  el mínimo impacto.

  ───
  ❓ ¿Puedo tener un método static en una interfaz?
  ───
  Sí, desde C# 11. Las interfaces pueden tener miembros static y
  abstract static (para operadores genéricos).

  SISTEMA DE PROCESAMIENTO DE ARCHIVOS CON INTERFACES

  Diseña un sistema de procesamiento de archivos usando interfaces
  siguiendo ISP y programación contra interfaces.

  Interfaces base:
    • IArchivo: Nombre, Extension, Tamanio, Ruta
    • ILegible: LeerContenido() -> string
    • IEscribible: EscribirContenido(string) -> void
    • ICompresible: Comprimir() -> IArchivo, Descomprimir() -> IArchivo
    • IValidable: ValidarIntegridad() -> bool
    • IConvertible<T>: Convertir() -> T

  Clases:
    • ArchivoTexto: ILegible, IEscribible, IValidable
    • ArchivoImagen: IArchivo, ILegible (metadatos), IConvertible<ArchivoImagen>
    • ArchivoZip: IArchivo, ICompresible, ILegible (listar contenido)
    • ArchivoEjecutable: IArchivo, IValidable (checksum)

  Clase `ProcesadorArchivos`:
    • Metodo Generico Procesar<T>(T archivo) where T : IArchivo
    • Metodo LeerYMostrar(ILegible legible)
    • Metodo ValidarYReportar(IValidable validable)

  Programa:
    • Crea archivos de cada tipo
    • Usa los metodos genericos del procesador
    • Demuestra polimorfismo de interfaces
    • Demuestra que interfaces pequeñas permiten
      combinar capacidades libremente

  ENTREGA: Codigo completo con interfaces segregadas,
  herencia de interfaces, programacion contra interfaces,
  y metodos genericos con restricciones de interfaz.

Las interfaces son el mecanismo fundamental para el desacoplamiento en C#.
.Definen contratos limpios que cualquier clase puede cumplir, permitiendo
polimorfismo sin las restricciones de la herencia simple.

Programar contra interfaces (no contra implementaciones concretas) es uno
de los principios más importantes del diseño de software moderno.
Permite testing, intercambiabilidad y evolución del código.
