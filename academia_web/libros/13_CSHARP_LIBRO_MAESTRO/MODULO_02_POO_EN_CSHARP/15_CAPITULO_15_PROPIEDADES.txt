  BIENVENIDA

En el capítulo anterior aprendiste a encapsular campos con propiedades. Pero
las propiedades en C# son mucho más poderosas de lo que parece a simple vista.

Desde las auto-propiedades hasta las propiedades con expresión corporal
(expression-bodied), pasando por los init-only setters y las propiedades
computadas, C# ofrece un conjunto rico y elegante de herramientas para
controlar el acceso a los datos.

En este capítulo exploraremos en profundidad:

  • Auto-propiedades ({ get; set; }) y su campo de respaldo implícito.
  • Propiedades computadas (con lógica en el getter).
  • Expression-bodied properties (=>).
  • Init-only setters (C# 9+).
  • Propiedades readonly.
  • Propiedades estáticas.
  • Propiedades virtuales, abstractas y de interfaz.
  • Mejores prácticas y casos de uso.

Las propiedades son uno de los mecanismos más usados en C#. Dominarlas te
permitirá escribir código más limpio, expresivo y seguro.

--------------------------------------------------------------------
  1. AUTO-PROPIEDADES (AUTO-IMPLEMENTED PROPERTIES)
--------------------------------------------------------------------

Las auto-propiedades son la forma más compacta de declarar propiedades.
El compilador genera automáticamente un campo de respaldo privado.

Sintaxis:

```csharp
public class Persona
{
    public string Nombre { get; set; }
    public int Edad { get; set; }
}
```

Esto equivale a:

```csharp
public class Persona
{
    private string _nombre;
    private int _edad;

    public string Nombre
    {
        get { return _nombre; }
        set { _nombre = value; }
    }

    public int Edad
    {
        get { return _edad; }
        set { _edad = value; }
    }
}
```

Ventajas de las auto-propiedades:

  ✓ Código más limpio y conciso.
  ✓ El compilador genera el campo de respaldo.
  ✓ Se pueden modificar después para agregar lógica sin cambiar la API.
  • Se pueden inicializar con valores por defecto.

```csharp
public class Producto
{
    public string Nombre { get; set; } = "Sin nombre";
    public decimal Precio { get; set; } = 0.0m;
    public int Stock { get; set; } = 0;
    public bool Activo { get; set; } = true;
}
```

También se pueden usar con distintos modificadores de acceso:

```csharp
public class Cuenta
{
    public string Titular { get; set; }
    public decimal Saldo { get; private set; }  // Solo lectura externa
    public int NumeroCuenta { get; }            // Get-only (init en constructor)

    public Cuenta(string titular, decimal saldoInicial, int numeroCuenta)
    {
        Titular = titular;
        Saldo = saldoInicial;
        NumeroCuenta = numeroCuenta;
    }
}
```

--------------------------------------------------------------------
  2. PROPIEDADES COMPUTADAS (COMPUTED PROPERTIES)
--------------------------------------------------------------------

Una propiedad computada no tiene campo de respaldo. Su valor se calcula
dinámicamente en el getter a partir de otros datos.

```csharp
public class Circulo
{
    public double Radio { get; set; }

    // Propiedades computadas (solo get)
    public double Diametro => Radio * 2;
    public double Circunferencia => 2 * Math.PI * Radio;
    public double Area => Math.PI * Radio * Radio;
}

// Uso:
Circulo c = new Circulo { Radio = 5 };
Console.WriteLine(c.Area);             // 78.5398...
Console.WriteLine(c.Circunferencia);   // 31.4159...
```

Otro ejemplo más elaborado:

```csharp
public class Empleado
{
    public string Nombre { get; set; }
    public string Apellido { get; set; }
    public decimal SalarioBase { get; set; }
    public int AniosExperiencia { get; set; }
    public bool TieneCertificacion { get; set; }

    // Propiedades computadas
    public string NombreCompleto => $"{Nombre} {Apellido}";
    public string Iniciales => $"{Nombre?[0]}{Apellido?[0]}".ToUpper();

    public decimal Bono
    {
        get
        {
            decimal bono = SalarioBase * 0.05m * AniosExperiencia;
            if (TieneCertificacion) bono += 1000;
            return Math.Min(bono, SalarioBase * 0.3m);
        }
    }

    public decimal SalarioTotal => SalarioBase + Bono;

    public string Nivel
    {
        get
        {
            if (AniosExperiencia >= 10) return "Senior";
            if (AniosExperiencia >= 5) return "Mid-Level";
            if (AniosExperiencia >= 2) return "Junior";
            return "Trainee";
        }
    }
}

// Uso:
Empleado emp = new Empleado
{
    Nombre = "Ana",
    Apellido = "Lopez",
    SalarioBase = 50000,
    AniosExperiencia = 8,
    TieneCertificacion = true
};

Console.WriteLine(emp.NombreCompleto);  // Ana Lopez
Console.WriteLine(emp.Nivel);           // Mid-Level
Console.WriteLine(emp.SalarioTotal);    // 50000 + 50000*0.05*8 + 1000 = 71000
```

Características:

  • Las propiedades computadas suelen ser get-only.
  • Se recalculan cada vez que se accede a ellas.
  • No almacenan estado adicional.
  • Pueden tener lógica compleja en el getter.

--------------------------------------------------------------------
  3. EXPRESSION-BODIED PROPERTIES (=>)
--------------------------------------------------------------------

Desde C# 6, las propiedades de solo get pueden usar la sintaxis de
expresión corporal (=>) para una notación más concisa.

```csharp
public class Rectangulo
{
    public double Ancho { get; set; }
    public double Alto { get; set; }

    // Expression-bodied properties
    public double Area => Ancho * Alto;
    public double Perimetro => 2 * (Ancho + Alto);
    public bool EsCuadrado => Ancho == Alto;
}
```

Desde C# 7, también se puede usar expression-bodied para setters y
constructores, pero para propiedades completas la sintaxis es:

```csharp
private string _nombre;
public string Nombre
{
    get => _nombre;
    set => _nombre = value ?? throw new ArgumentNullException(nameof(Nombre));
}
```

Comparación de estilos:

```csharp
// Estilo tradicional
public string NombreCompleto
{
    get { return $"{Nombre} {Apellido}"; }
}

// Expression-bodied (C# 6+)
public string NombreCompleto => $"{Nombre} {Apellido}";

// Setter con expression-bodied (C# 7+)
public string Nombre
{
    get => _nombre;
    set => _nombre = value;
}
```

--------------------------------------------------------------------
  4. INIT-ONLY SETTERS (C# 9+)
--------------------------------------------------------------------

`init` es un modificador para setters que permite la asignación solo durante
la inicialización del objeto (en el constructor o con inicializadores de
objeto). Después de la creación, la propiedad es efectivamente readonly.

```csharp
public class Persona
{
    public string Nombre { get; init; }
    public int Edad { get; init; }
    public string Email { get; init; }
}

// Uso con inicializador de objeto (OK):
Persona p = new Persona
{
    Nombre = "Ana",
    Edad = 30,
    Email = "ana@email.com"
};

// Modificar después (ERROR):
// p.Nombre = "Luis";  // CS8852: La propiedad init-only solo se puede asignar en init

// También se puede asignar en el constructor:
public class Producto
{
    public string Codigo { get; init; }
    public string Nombre { get; init; }
    public decimal Precio { get; init; }

    public Producto(string codigo, string nombre, decimal precio)
    {
        Codigo = codigo;
        Nombre = nombre;
        Precio = precio;
    }
}
```

`init` es ideal para crear objetos inmutables que pueden inicializarse
con la sintaxis de inicializadores de objeto.

Ventajas:

  • Inmutabilidad después de la creación.
  • Sintaxis limpia con inicializadores.
  • Ideal para DTOs, modelos de vista, configuraciones.
  • Compatible con records (donde es el comportamiento predeterminado).

Diferencia entre get-only y init:

```csharp
// Get-only: solo se asigna en el constructor
public class A
{
    public string Nombre { get; }
    public A(string nombre) => Nombre = nombre;
}

// Init: se asigna en constructor O con inicializador
public class B
{
    public string Nombre { get; init; }
}

// Con init podemos hacer:
B b = new B { Nombre = "Test" };

// Con get-only no podemos:
// A a = new A { Nombre = "Test" };  // ERROR
```

--------------------------------------------------------------------
  5. PROPIEDADES ESTÁTICAS
--------------------------------------------------------------------

Las propiedades estáticas pertenecen a la clase, no a las instancias.
Se accede a ellas mediante el nombre de la clase.

```csharp
public class ConfiguracionApp
{
    public static string Version { get; set; } = "1.0.0";
    public static string Entorno { get; private set; }
    public static DateTime Inicio { get; } = DateTime.Now;

    static ConfiguracionApp()
    {
        Entorno = Environment.GetEnvironmentVariable("ASPNETCORE_ENVIRONMENT") ?? "Production";
    }
}

// Uso:
Console.WriteLine(ConfiguracionApp.Version);
Console.WriteLine(ConfiguracionApp.Entorno);
```

Características:

  • Se accede con `Clase.Propiedad`, no con `instancia.Propiedad`.
  • Pueden tener get y set con diferentes accesos.
  • Pueden ser get-only (readonly estáticas).
  • Son compartidas por todas las instancias.
  • Útiles para configuraciones globales, contadores, cachés.

```csharp
public class ContadorInstancias
{
    private static int _contador;

    public static int TotalInstancias => _contador;
    public int Id { get; }

    public ContadorInstancias()
    {
        Id = ++_contador;
    }
}
```

--------------------------------------------------------------------
  6. PROPIEDADES VIRTUALES Y ABSTRACTAS
--------------------------------------------------------------------

Las propiedades pueden ser `virtual` (sobrescribibles en clases derivadas)
y `abstract` (declaradas sin implementación, deben implementarse en
clases derivadas).

```csharp
public abstract class Figura
{
    // Propiedad abstracta (sin implementacion)
    public abstract double Area { get; }
    public abstract double Perimetro { get; }

    // Propiedad virtual (con implementacion por defecto)
    public virtual string Nombre => "Figura generica";

    public void MostrarInfo()
    {
        Console.WriteLine($"{Nombre}: Area = {Area:F2}, Perimetro = {Perimetro:F2}");
    }
}

public class Circulo : Figura
{
    public double Radio { get; set; }

    public Circulo(double radio) => Radio = radio;

    public override double Area => Math.PI * Radio * Radio;
    public override double Perimetro => 2 * Math.PI * Radio;
    public override string Nombre => $"Circulo (r={Radio})";
}

public class Rectangulo : Figura
{
    public double Ancho { get; set; }
    public double Alto { get; set; }

    public Rectangulo(double ancho, double alto)
    {
        Ancho = ancho;
        Alto = alto;
    }

    public override double Area => Ancho * Alto;
    public override double Perimetro => 2 * (Ancho + Alto);
    public override string Nombre => $"Rectangulo ({Ancho}x{Alto})";
}

// Uso:
Figura[] figuras = { new Circulo(5), new Rectangulo(4, 6) };
foreach (var f in figuras)
    f.MostrarInfo();
```

--------------------------------------------------------------------
  7. PROPIEDADES EN INTERFACES
--------------------------------------------------------------------

Las interfaces pueden declarar propiedades (sin implementación en la
interfaz, a menos que sean default interface methods C# 8+).

```csharp
public interface IEntidad
{
    int Id { get; }
    DateTime FechaCreacion { get; }
    string ToString();  // Metodo
}

public interface IProducto
{
    string Nombre { get; set; }
    decimal Precio { get; set; }
    string CodigoBarras { get; init; }
}

public class Producto : IProducto, IEntidad
{
    public int Id { get; init; }
    public DateTime FechaCreacion { get; init; }
    public string Nombre { get; set; }
    public decimal Precio { get; set; }
    public string CodigoBarras { get; init; }

    public Producto()
    {
        Id = new Random().Next(1000, 9999);
        FechaCreacion = DateTime.Now;
    }
}
```

Desde C# 8, las interfaces pueden tener propiedades con implementación
por defecto (default interface methods):

```csharp
public interface ILogger
{
    string Prefijo { get; }
    void Log(string mensaje);

    // Default implementation (C# 8+)
    void LogInfo(string mensaje) => Log($"[INFO] {Prefijo}: {mensaje}");
    void LogError(string mensaje) => Log($"[ERROR] {Prefijo}: {mensaje}");
}

public class LoggerConsola : ILogger
{
    public string Prefijo => "App";

    public void Log(string mensaje)
    {
        Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] {mensaje}");
    }
}

// Uso:
ILogger log = new LoggerConsola();
log.LogInfo("Iniciando aplicacion");
log.LogError("Error simulado");
```

--------------------------------------------------------------------
  8. PROPIEDADES CON CACHÉ (LAZY INITIALIZATION)
--------------------------------------------------------------------

Las propiedades pueden usar inicialización diferida (lazy) para calcular
el valor solo cuando se necesita y almacenarlo en caché.

```csharp
public class Reporte
{
    private string _contenido;
    private bool _contenidoCalculado;

    public string Contenido
    {
        get
        {
            if (!_contenidoCalculado)
            {
                Console.WriteLine("Calculando contenido del reporte...");
                _contenido = GenerarContenido();
                _contenidoCalculado = true;
            }
            return _contenido;
        }
    }

    private string GenerarContenido()
    {
        // Simular operación costosa
        System.Threading.Thread.Sleep(1000);
        return $"Reporte generado el {DateTime.Now}";
    }
}

// O usando Lazy<T>:
public class ReporteLazy
{
    private readonly Lazy<string> _contenidoLazy;

    public ReporteLazy()
    {
        _contenidoLazy = new Lazy<string>(() =>
        {
            Console.WriteLine("Calculando (Lazy)...");
            System.Threading.Thread.Sleep(1000);
            return $"Reporte lazy: {DateTime.Now}";
        });
    }

    public string Contenido => _contenidoLazy.Value;
}
```

--------------------------------------------------------------------
  9. PROPIEDADES INDEXADAS (INDEXERS)
--------------------------------------------------------------------

C# permite definir propiedades con parámetros, llamadas "indexers".
Permiten usar la sintaxis de corchetes [] en objetos.

```csharp
public class Inventario
{
    private Dictionary<string, int> _productos = new();

    // Indexer
    public int this[string codigo]
    {
        get => _productos.TryGetValue(codigo, out var stock) ? stock : 0;
        set => _productos[codigo] = value;
    }

    // Indexer con dos parámetros
    public int this[string codigo, string almacen]
    {
        get
        {
            Console.WriteLine($"Buscando {codigo} en almacen {almacen}");
            return _productos.TryGetValue($"{codigo}_{almacen}", out var stock) ? stock : 0;
        }
    }
}

// Uso:
Inventario inv = new Inventario();
inv["LAP-001"] = 10;
inv["MON-002"] = 25;
Console.WriteLine(inv["LAP-001"]);  // 10
Console.WriteLine(inv["TEC-003"]);  // 0
Console.WriteLine(inv["LAP-001", "Principal"]);  // Busca con almacen
```

--------------------------------------------------------------------
  10. EJEMPLO COMPLETO: SISTEMA DE GESTIÓN DE EMPLEADOS
--------------------------------------------------------------------

```csharp
using System;

namespace GestionEmpleados
{
    public enum TipoEmpleado
    {
        TiempoCompleto,
        MedioTiempo,
        Contratista
    }

    public class Empleado
    {
        private static int _contador = 1000;
        private decimal _salarioBase;

        // Auto-propiedades
        public int Id { get; } = ++_contador;
        public string Nombre { get; set; }
        public string Apellido { get; set; }
        public TipoEmpleado Tipo { get; init; }
        public DateTime FechaIngreso { get; init; }
        public bool Activo { get; set; } = true;

        // Propiedad con validación
        public decimal SalarioBase
        {
            get => _salarioBase;
            set
            {
                if (value < 0)
                    throw new ArgumentException("El salario no puede ser negativo.");
                _salarioBase = value;
            }
        }

        // Propiedades computadas
        public string NombreCompleto => $"{Nombre} {Apellido}";
        public int Antiguedad => DateTime.Now.Year - FechaIngreso.Year;

        public decimal Bono
        {
            get
            {
                decimal bono = SalarioBase * 0.05m * Antiguedad;
                return Math.Min(bono, SalarioBase * 0.5m);
            }
        }

        public decimal SalarioTotal => SalarioBase + Bono;

        public string Nivel
        {
            get
            {
                return Antiguedad switch
                {
                    >= 10 => "Senior",
                    >= 5 => "Intermedio",
                    >= 2 => "Junior",
                    _ => "Trainee"
                };
            }
        }

        // Expresión corporal
        public override string ToString() =>
            $"[{Id}] {NombreCompleto} - {Tipo} - {Nivel} - ${SalarioTotal:F2}";

        public void MostrarInfo()
        {
            Console.WriteLine("=================================");
            Console.WriteLine($"ID:       {Id}");
            Console.WriteLine($"Nombre:   {NombreCompleto}");
            Console.WriteLine($"Tipo:     {Tipo}");
            Console.WriteLine($"Nivel:    {Nivel}");
            Console.WriteLine($"Salario:  ${SalarioBase:F2}");
            Console.WriteLine($"Bono:     ${Bono:F2}");
            Console.WriteLine($"Total:    ${SalarioTotal:F2}");
            Console.WriteLine($"Ingreso:  {FechaIngreso:dd/MM/yyyy}");
            Console.WriteLine($"Antiguedad: {Antiguedad} años");
            Console.WriteLine($"Estado:   {(Activo ? "Activo" : "Inactivo")}");
            Console.WriteLine("=================================");
        }
    }

    public class Programa
    {
        public static void Main()
        {
            // Uso de init con inicializadores
            Empleado e1 = new Empleado
            {
                Nombre = "Ana",
                Apellido = "Lopez",
                SalarioBase = 50000,
                Tipo = TipoEmpleado.TiempoCompleto,
                FechaIngreso = new DateTime(2015, 3, 1)
            };

            Empleado e2 = new Empleado
            {
                Nombre = "Carlos",
                Apellido = "Ruiz",
                SalarioBase = 35000,
                Tipo = TipoEmpleado.MedioTiempo,
                FechaIngreso = new DateTime(2020, 6, 15)
            };

            e1.MostrarInfo();
            e2.MostrarInfo();

            Console.WriteLine($"\nTotal empleados creados: {e2.Id - 1000}");

            // Demostración de init (no se puede modificar despues)
            // e1.Tipo = TipoEmpleado.Contratista;  // ERROR: init only
            // e1.FechaIngreso = DateTime.Now;      // ERROR: init only
        }
    }
}
```

--------------------------------------------------------------------
  12. BUENAS PRÁCTICAS
--------------------------------------------------------------------

  ✓ Usa auto-propiedades por defecto; agrega lógica solo cuando sea
    necesario.

  ✓ Usa `init` para propiedades que no deben cambiar después de la
    inicialización.

  ✓ Usa propiedades computadas para valores derivados (no almacenes
    datos que puedes calcular).

  ✓ Prefiere expression-bodied para propiedades simples y de una línea.

  ✓ No hagas que los getters tengan efectos secundarios (cambiar estado).

  ✓ Los setters deben validar y lanzar excepciones apropiadas.

  ✓ Usa propiedades estáticas para datos compartidos por todas las
    instancias.

  ✓ Documenta propiedades que realizan validación.

  ✓ Para colecciones, expón IReadOnlyList<T> o IEnumerable<T> en lugar
    de List<T> directamente.

```csharp
public class Pedido
{
    private readonly List<string> _items = new();

    public string Cliente { get; init; }
    public DateTime Fecha { get; init; } = DateTime.Now;
    public IReadOnlyList<string> Items => _items.AsReadOnly();
    public int TotalItems => _items.Count;

    public void AgregarItem(string item)
    {
        if (string.IsNullOrWhiteSpace(item))
            throw new ArgumentException("Item invalido.");
        _items.Add(item);
    }
}
```

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • Las AUTO-PROPIEDADES ({ get; set; }) son la forma más concisa de
    declarar propiedades con un campo de respaldo generado por el compilador.

  • Las PROPIEDADES COMPUTADAS calculan su valor dinámicamente a partir
    de otros datos.

  • La sintaxis EXPRESSION-BODIED (=>) permite escribir propiedades de
    una línea de forma elegante.

  • Los INIT-ONLY SETTERS permiten asignar valores en la inicialización
    (constructor o inicializador) y luego son readonly.

  • Las propiedades pueden ser STATIC, VIRTUAL, ABSTRACT, y pueden
    declararse en INTERFACES.

  • Los INDEXERS permiten usar la sintaxis de corchetes [] en objetos.

  • Las propiedades con LAZY INITIALIZATION difieren el cálculo costoso
    hasta que se necesita el valor.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ───
  ❓ ¿Cuándo usar auto-propiedad vs campo público?
  ───
  Siempre usa auto-propiedad. Te da flexibilidad futura (puedes agregar
  validación sin cambiar la API) y es compatible con binding, serialización.

  ───
  ❓ ¿Puedo tener un setter privado en auto-propiedad?
  ───
  Sí: public string Nombre { get; private set; }. Muy común.

  ───
  ❓ ¿Qué diferencia hay entre { get; } y { get; init; }?
  ───
  get-only: solo se asigna en el constructor. init: se asigna en
  constructor O con inicializador de objeto.

  ───
  ❓ ¿Las propiedades computadas son más lentas?
  ───
  Se ejecutan cada vez que se accede. Si el cálculo es costoso, usa
  Lazy<T> o cache manual.

  ───
  ❓ ¿Puedo usar ref returns con propiedades?
  ───
  No directamente. Las propiedades son métodos (get/set). Pero desde C# 7
  puedes tener ref returns en métodos.

  ───
  ❓ ¿Las propiedades pueden ser async?
  ───
  No. Las propiedades no pueden ser async. Si necesitas una operación
  asíncrona, usa un método.

  ───
  ❓ ¿Cómo documentar propiedades?
  ───
  Con comentarios XML: /// <summary>...</summary>

  SISTEMA DE GESTIÓN DE HOTEL

  Diseña un sistema de reservas de hotel con propiedades avanzadas:

  Clase `Habitacion`:
    • Numero (init), Tipo (init), PrecioPorNoche (init)
    • Capacidad (init: 1-6 personas)
    • Disponible (get; set)
    • Propiedades computadas:
      - Descripcion: "Habitacion {Numero} - {Tipo}"
      - PrecioConIVA: PrecioPorNoche * 1.16
    • Indexer para acceder a servicios adicionales por nombre

  Clase `Reserva`:
    • Cliente (init), Habitacion (init)
    • FechaIngreso (init), FechaSalida (init)
    • Propiedades computadas:
      - Noches: (FechaSalida - FechaIngreso).Days
      - Subtotal: Noches * Habitacion.PrecioPorNoche
      - IVA: Subtotal * 0.16
      - Total: Subtotal + IVA
      - Estado: "Confirmada" si FechaIngreso > hoy, etc.

  Clase `Hotel`:
    • Nombre (init), Direccion (init)
    • Lista de habitaciones
    • Indexer por numero de habitacion
    • Propiedad: HabitacionesDisponibles (filtro)
    • Propiedad: PorcentajeOcupacion (computada)

  Programa:
    • Crea un hotel con 10 habitaciones de distintos tipos
    • Crea 3-4 reservas
    • Muestra info completa usando todas las propiedades
