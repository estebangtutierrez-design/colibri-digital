  1. ¿POR QUÉ PRUEBAS UNITARIAS?
  ───────────────────────────────
  Las pruebas unitarias automatizadas son fragmentos de código que verifican
  que otro código funcione correctamente. Beneficios:

    • Detección temprana de errores.
    • Refactorización segura (sabes al instante si algo se rompió).
    • Documentación viva del comportamiento esperado.
    • Diseño más limpio (el código testeable suele ser mejor código).
    • Reducción del costo de mantenimiento.

  2. PIRÁMIDE DE PRUEBAS
  ───────────────────────
    ┌──────────┐
    │   E2E    │  (pocas, lentas, caras)
    ├──────────┤
    │Integrac. │  (algunas)
    ├──────────┤
    │Unitarias │  (muchas, rápidas, baratas) ← ESTAMOS AQUÍ
    └──────────┘

  Regla: 70% unitarias, 20% integración, 10% E2E.

  3. XUNIT VS NUNIT
  ──────────────────
  • xUnit: moderno, creado por el mismo autor de NUnit v2. Más simple,
    mejor integración con .NET Core. Usa [Fact] y [Theory].
  • NUnit: maduro, muy usado en empresas. Usa [Test], [TestCase].

  En este capítulo usaremos xUnit (el estándar de facto en .NET Core),
  pero los conceptos aplican a ambos.

  4. ASSERT
  ─────────
  La clase Assert contiene métodos para verificar condiciones:
    • Assert.Equal(expected, actual)
    • Assert.NotEqual(expected, actual)
    • Assert.True(condition)
    • Assert.False(condition)
    • Assert.Throws<T>(() => ...)
    • Assert.Null(object)
    • Assert.NotNull(object)
    • Assert.Contains(item, collection)

  5. MOCKING CON MOQ
  ───────────────────
  Moq es una librería que crea objetos simulados (mocks) de interfaces o
  clases abstractas. Permite aislar la unidad bajo prueba de sus dependencias.

  6. TDD (TEST DRIVEN DEVELOPMENT)
  ─────────────────────────────────
  TDD es una metodología donde escribes la prueba ANTES del código:

    Rojo:  Escribe una prueba que falla (aún no hay código).
    Verde: Escribe el código mínimo para que la prueba pase.
    Refactor: Mejora el código sin cambiar su comportamiento.

  Este ciclo se conoce como "Red-Green-Refactor".

  // ============================
  // MiApp.Tests/CalculadoraTests.cs
  // ============================

  using Xunit;
  using MiApp.Application;

  namespace MiApp.Tests
  {
      public class CalculadoraTests
      {
          private readonly Calculadora _calc;

          public CalculadoraTests()
          {
              _calc = new Calculadora();
          }

          [Fact]
          public void Sumar_DosNumeros_RetornaSuma()
          {
              // Arrange
              int a = 5;
              int b = 3;

              // Act
              int resultado = _calc.Sumar(a, b);

              // Assert
              Assert.Equal(8, resultado);
          }

          [Fact]
          public void Dividir_DivisionPorCero_LanzaExcepcion()
          {
              // Arrange
              int a = 10;
              int b = 0;

              // Act & Assert
              var ex = Assert.Throws<DivideByZeroException>(
                  () => _calc.Dividir(a, b));

              Assert.Contains("cero", ex.Message);
          }

          [Theory]
          [InlineData(1, 2, 3)]
          [InlineData(-1, -2, -3)]
          [InlineData(0, 0, 0)]
          [InlineData(100, -50, 50)]
          public void Sumar_VariosCasos_RetornaSumaEsperada(
              int a, int b, int esperado)
          {
              // Act
              int resultado = _calc.Sumar(a, b);

              // Assert
              Assert.Equal(esperado, resultado);
          }
      }
  }

  // ============================
  // MiApp.Application/Calculadora.cs
  // ============================

  using System;

  namespace MiApp.Application
  {
      public class Calculadora
      {
          public int Sumar(int a, int b) => a + b;

          public int Restar(int a, int b) => a - b;

          public int Multiplicar(int a, int b) => a * b;

          public int Dividir(int a, int b)
          {
              if (b == 0)
                  throw new DivideByZeroException(
                      "No se puede dividir por cero.");
              return a / b;
          }
      }
  }

  // ============================
  // MiApp.Application/Servicios/ClienteService.cs

  using System;
  using System.Threading.Tasks;
  using MiApp.Domain.Entities;
  using MiApp.Domain.Interfaces;

  namespace MiApp.Application.Servicios
  {
      public class ClienteService
      {
          private readonly IClienteRepositorio _repositorio;
          private readonly IEmailService _emailService;

          public ClienteService(
              IClienteRepositorio repositorio,
              IEmailService emailService)
          {
              _repositorio = repositorio;
              _emailService = emailService;
          }

          public async Task<Cliente> RegistrarClienteAsync(
              string nombre, string email)
          {
              if (string.IsNullOrWhiteSpace(nombre))
                  throw new ArgumentException("El nombre es obligatorio.");

              if (string.IsNullOrWhiteSpace(email))
                  throw new ArgumentException("El email es obligatorio.");

              var existe = await _repositorio.ObtenerPorEmailAsync(email);
              if (existe != null)
                  throw new InvalidOperationException(
                      "Ya existe un cliente con ese email.");

              var cliente = new Cliente
              {
                  Id = Guid.NewGuid(),
                  Nombre = nombre,
                  Email = email,
                  FechaRegistro = DateTime.UtcNow
              };

              await _repositorio.AgregarAsync(cliente);
              await _emailService.EnviarBienvenidaAsync(email, nombre);

              return cliente;
          }
      }
  }

  // ============================
  // MiApp.Tests/ClienteServiceTests.cs
  // ============================

  using System;
  using System.Threading.Tasks;
  using Moq;
  using Xunit;
  using MiApp.Application.Servicios;
  using MiApp.Domain.Entities;
  using MiApp.Domain.Interfaces;

  namespace MiApp.Tests
  {
      public class ClienteServiceTests
      {
          private readonly Mock<IClienteRepositorio> _repoMock;
          private readonly Mock<IEmailService> _emailMock;
          private readonly ClienteService _service;

          public ClienteServiceTests()
          {
              _repoMock = new Mock<IClienteRepositorio>();
              _emailMock = new Mock<IEmailService>();
              _service = new ClienteService(
                  _repoMock.Object, _emailMock.Object);
          }

          [Fact]
          public async Task RegistrarClienteAsync_ClienteValido_RetornaCliente()
          {
              // Arrange
              string nombre = "Juan Pérez";
              string email = "juan@email.com";

              _repoMock.Setup(r => r.ObtenerPorEmailAsync(email))
                       .ReturnsAsync((Cliente)null);

              _repoMock.Setup(r => r.AgregarAsync(It.IsAny<Cliente>()))
                       .ReturnsAsync((Cliente c) => c);

              _emailMock.Setup(e => e.EnviarBienvenidaAsync(email, nombre))
                        .Returns(Task.CompletedTask);

              // Act
              var cliente = await _service.RegistrarClienteAsync(
                  nombre, email);

              // Assert
              Assert.NotNull(cliente);
              Assert.Equal(nombre, cliente.Nombre);
              Assert.Equal(email, cliente.Email);

              _repoMock.Verify(r => r.AgregarAsync(It.IsAny<Cliente>()),
                  Times.Once);

              _emailMock.Verify(e => e.EnviarBienvenidaAsync(email, nombre),
                  Times.Once);
          }

          [Fact]
          public async Task RegistrarClienteAsync_EmailDuplicado_LanzaExcepcion()
          {
              // Arrange
              string email = "existente@email.com";
              _repoMock.Setup(r => r.ObtenerPorEmailAsync(email))
                       .ReturnsAsync(new Cliente { Email = email });

              // Act & Assert
              var ex = await Assert.ThrowsAsync<InvalidOperationException>(
                  () => _service.RegistrarClienteAsync("Test", email));

              Assert.Contains("ya existe", ex.Message.ToLower());
          }

          [Theory]
          [InlineData("", "email@test.com")]
          [InlineData(null, "email@test.com")]
          [InlineData("Nombre", "")]
          [InlineData("Nombre", null)]
          public async Task RegistrarClienteAsync_DatosInvalidos_LanzaExcepcion(
              string nombre, string email)
          {
              // Act & Assert
              await Assert.ThrowsAsync<ArgumentException>(
                  () => _service.RegistrarClienteAsync(nombre, email));
          }
      }
  }

  using System.Collections.Generic;
  using Xunit;

  namespace MiApp.Tests
  {
      public class ValidadorTests
      {
          public static IEnumerable<object[]> DatosEmailValidos =>
              new List<object[]>
              {
                  new object[] { "user@domain.com", true },
                  new object[] { "a@b.co", true },
                  new object[] { "user+tag@domain.com", true },
                  new object[] { "", false },
                  new object[] { "invalido", false },
                  new object[] { "@domain.com", false },
                  new object[] { "user@", false },
                  new object[] { null, false },
              };

          [Theory]
          [MemberData(nameof(DatosEmailValidos))]
          public void ValidarEmail_RetornaResultadoEsperado(
              string email, bool esperado)
          {
              // Arrange
              var validador = new ValidadorEmail();

              // Act
              bool resultado = validador.EsValido(email);

              // Assert
              Assert.Equal(esperado, resultado);
          }

          [Theory]
          [ClassData(typeof(DatosPrueba))]
          public void ValidarConClaseExterna(int input, bool esperado)
          {
              var validador = new ValidadorNumero();
              bool resultado = validador.EsPar(input);
              Assert.Equal(esperado, resultado);
          }
      }

      public class ValidadorEmail
      {
          public bool EsValido(string email)
          {
              if (string.IsNullOrWhiteSpace(email))
                  return false;

              try
              {
                  var addr = new System.Net.Mail.MailAddress(email);
                  return addr.Address == email;
              }
              catch
              {
                  return false;
              }
          }
      }

      public class ValidadorNumero
      {
          public bool EsPar(int numero) => numero % 2 == 0;
      }

      public class DatosPrueba : TheoryData<int, bool>
      {
          public DatosPrueba()
          {
              Add(2, true);
              Add(3, false);
              Add(0, true);
              Add(-4, true);
              Add(-5, false);
          }
      }
  }

  Ejemplo 5: TDD — Ciclo Red-Green-Refactor
  ──────────────────────────────────────────────────────────────────────────────

  // PASO 1 (ROJO): Escribir la prueba que falla

  [Fact]
  public void CalcularDescuento_ClientePremium_Retorna15Porciento()
  {
      // Arrange
      var service = new DescuentoService();
      decimal montoOriginal = 1000;

      // Act
      decimal descuento = service.CalcularDescuento(
          montoOriginal, EsPremium: true);

      // Assert
      Assert.Equal(150, descuento); // 15% de 1000 = 150
  }

  // PASO 2 (VERDE): Escribir el código mínimo

  public class DescuentoService
  {
      public decimal CalcularDescuento(
          decimal monto, bool esPremium)
      {
          if (esPremium)
              return monto * 0.15m;
          return 0;
      }
  }

  // PASO 3 (REFACTOR): Mejorar el código

  public class DescuentoServiceRefactorizado
  {
      private const decimal DescuentoPremium = 0.15m;
      private const decimal DescuentoNormal = 0.05m;
      private const decimal TopeDescuento = 500;

      public decimal CalcularDescuento(
          decimal monto, bool esPremium)
      {
          decimal porcentaje = esPremium
              ? DescuentoPremium
              : DescuentoNormal;

          decimal descuento = monto * porcentaje;
          return Math.Min(descuento, TopeDescuento);
      }
  }

  // Pruebas actualizadas tras refactor:

  public class DescuentoServiceTests
  {
      [Theory]
      [InlineData(1000, true, 150)]
      [InlineData(1000, false, 50)]
      [InlineData(5000, true, 500)] // Tope
      [InlineData(10000, true, 500)] // Tope
      [InlineData(0, true, 0)]
      public void CalcularDescuento_RetornaValorEsperado(
          decimal monto, bool premium, decimal esperado)
      {
          var service = new DescuentoServiceRefactorizado();
          var resultado = service.CalcularDescuento(monto, premium);
          Assert.Equal(esperado, resultado);
      }
  }

  using System;
  using System.Linq;
  using System.Threading.Tasks;
  using Microsoft.EntityFrameworkCore;
  using Xunit;
  using MiApp.Domain.Entities;
  using MiApp.Infrastructure.Data;
  using MiApp.Infrastructure.Repositories;

  namespace MiApp.Tests.Integration
  {
      public class ProductoRepositorioTests
      {
          private readonly AppDbContext _context;
          private readonly ProductoRepositorio _repositorio;

          public ProductoRepositorioTests()
          {
              var options = new DbContextOptionsBuilder<AppDbContext>()
                  .UseInMemoryDatabase(databaseName: Guid.NewGuid().ToString())
                  .Options;

              _context = new AppDbContext(options);
              _repositorio = new ProductoRepositorio(_context);
          }

          [Fact]
          public async Task AgregarAsync_ProductoValido_AgregaYRetorna()
          {
              // Arrange
              var producto = new Producto("Test", 100, 10);

              // Act
              var resultado = await _repositorio.AgregarAsync(producto);

              // Assert
              Assert.NotNull(resultado);
              Assert.NotEqual(Guid.Empty, resultado.Id);

              var enDb = await _context.Productos.FindAsync(resultado.Id);
              Assert.NotNull(enDb);
              Assert.Equal("Test", enDb.Nombre);
          }

          [Fact]
          public async Task ObtenerTodosAsync_ConDatos_RetornaTodos()
          {
              // Arrange
              _context.Productos.AddRange(
                  new Producto("Prod1", 10, 5),
                  new Producto("Prod2", 20, 3));
              await _context.SaveChangesAsync();

              // Act
              var productos = await _repositorio.ObtenerTodosAsync();

              // Assert
              Assert.Equal(2, productos.Count());
          }

          [Fact]
          public async Task EliminarAsync_ProductoExistente_RetornaTrue()
          {
              // Arrange
              var producto = new Producto("Eliminar", 50, 2);
              _context.Productos.Add(producto);
              await _context.SaveChangesAsync();

              // Act
              var resultado = await _repositorio.EliminarAsync(producto.Id);

              // Assert
              Assert.True(resultado);
              Assert.Equal(0, await _context.Productos.CountAsync());
          }
      }
  }

  // FluentAssertions ofrece una sintaxis más legible:

  // using FluentAssertions;

  [Fact]
  public void ValidarConFluentAssertions()
  {
      // Arrange
      var calc = new Calculadora();

      // Act
      Action act = () => calc.Dividir(10, 0);

      // Assert (fluent)
      act.Should()
         .Throw<DivideByZeroException>()
         .WithMessage("*cero*");

      // También para valores:
      int resultado = calc.Sumar(2, 3);
      resultado.Should().Be(5);
      resultado.Should().BePositive();
      resultado.Should().BeGreaterThan(4);
  }

================================================================================
  [PREGUNTAS FRECUENTES — FAQ]
================================================================================

  P: ¿Debo probar métodos privados?
  R: No directamente. Prueba el comportamiento público que los usa. Si un
     método privado es muy complejo, considera extraerlo a otra clase.

  P: ¿Cuánto tiempo debo dedicar a pruebas?
  R: Como mínimo, el mismo tiempo que al código de producción. Un buen ratio
     es 1:1 o 2:1 (más código de prueba que de producción).

  P: ¿Qué cubrir primero?
  R: La lógica de negocio crítica: cálculos, reglas, validaciones. Luego
     los casos límite (edge cases) y manejo de errores.

  P: ¿Pruebas unitarias o de integración?
  R: Principalmente unitarias. Usa integración para la comunicación con
     bases de datos, APIs externas y el sistema de archivos.

  P: ¿Qué hago con código legacy sin pruebas?
  R: Cada vez que toques ese código, agrega pruebas para el cambio que
     haces (como un "cinturón de seguridad"). Con el tiempo, la cobertura
     aumentará.
