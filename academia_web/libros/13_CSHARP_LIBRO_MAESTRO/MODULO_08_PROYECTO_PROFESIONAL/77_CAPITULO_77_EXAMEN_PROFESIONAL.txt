  Ejercicio 1: Clase genérica Repository<T>

  Escribe una interfaz IRepository<T> con métodos:
    Task<T> GetByIdAsync(int id)
    Task<IEnumerable<T>> GetAllAsync()
    Task<T> AddAsync(T entity)
    Task UpdateAsync(T entity)
    Task DeleteAsync(T entity)

  Luego implementa EFRepository<T> usando EF Core.

  SOLUCIÓN REFERENCIA:

  public interface IRepository<T> where T : class
  {
      Task<T> GetByIdAsync(int id);
      Task<IEnumerable<T>> GetAllAsync();
      Task<T> AddAsync(T entity);
      Task UpdateAsync(T entity);
      Task DeleteAsync(T entity);
  }

  public class EFRepository<T> : IRepository<T> where T : class
  {
      private readonly DbContext _context;
      private readonly DbSet<T> _dbSet;

      public EFRepository(DbContext context)
      {
          _context = context;
          _dbSet = context.Set<T>();
      }

      public async Task<T> GetByIdAsync(int id)
          => await _dbSet.FindAsync(id);

      public async Task<IEnumerable<T>> GetAllAsync()
          => await _dbSet.ToListAsync();

      public async Task<T> AddAsync(T entity)
      {
          _dbSet.Add(entity);
          await _context.SaveChangesAsync();
          return entity;
      }

      public async Task UpdateAsync(T entity)
      {
          _context.Entry(entity).State = EntityState.Modified;
          await _context.SaveChangesAsync();
      }

      public async Task DeleteAsync(T entity)
      {
          _dbSet.Remove(entity);
          await _context.SaveChangesAsync();
      }
  }

  Ejercicio 2: Middleware personalizado

  Escribe un middleware que registre el tiempo de cada solicitud HTTP.

  SOLUCIÓN REFERENCIA:

  public class TimingMiddleware
  {
      private readonly RequestDelegate _next;
      private readonly ILogger<TimingMiddleware> _logger;

      public TimingMiddleware(RequestDelegate next, ILogger<TimingMiddleware> logger)
      {
          _next = next;
          _logger = logger;
      }

      public async Task InvokeAsync(HttpContext context)
      {
          var sw = Stopwatch.StartNew();
          await _next(context);
          sw.Stop();
          _logger.LogInformation("{Method} {Path} => {StatusCode} en {ElapsedMs}ms",
              context.Request.Method,
              context.Request.Path,
              context.Response.StatusCode,
              sw.ElapsedMilliseconds);
      }
  }

  Escribe una consulta LINQ que devuelva los 10 clientes que más han gastado
  en total, incluyendo su nombre, email y total gastado.

  SOLUCIÓN REFERENCIA:

  var topClientes = await _context.Pedidos
      .GroupBy(p => p.ClienteId)
      .Select(g => new

          Total = g.Sum(p => p.Total)

  Bug 1: ¿Qué error tiene este código?

  public async Task<List<Cliente>> ObtenerClientes()
  {
      using var ctx = new AppDbContext();
      return await ctx.Clientes.ToListAsync();
  }

  RESPUESTA: El DbContext se está disposed antes de que la consulta termine
  porque el using está fuera del await. Solución: usar using de forma correcta
  o inyectar DbContext con DI (Scoped).

  ---

  Bug 2: ¿Qué error tiene este código?

  public decimal CalcularDescuento(decimal monto)
  {
      if (monto > 1000)
          return monto * 0.1m;
      else if (monto > 500)
          return monto * 0.05m;
      else
          return monto;
  }

  RESPUESTA: El orden de las condiciones es incorrecto. Si monto = 1500,
  la primera condición (monto > 1000) se cumple y devuelve 10%, pero debería
  ser 10% solo para montos entre 500 y 1000. Solución: invertir el orden:
  primero monto > 1000, o ajustar la lógica.

  ---

  Bug 3: ¿Qué error tiene este código?

  public string Concatenar(List<string> palabras)
  {
      string resultado = "";
      foreach (var p in palabras)
          resultado += p + ",";
      return resultado;
  }

  RESPUESTA: Ineficiente para listas grandes. Cada concatenación crea un
  nuevo string. Solución: usar StringBuilder o string.Join.

  ---

  Bug 4: ¿Qué error tiene este código?

  public class ProductoController : ControllerBase
  {
      [HttpDelete("{id}")]
      public IActionResult Eliminar(int id)
      {
          var producto = _context.Productos.Find(id);
          _context.Productos.Remove(producto);
          _context.SaveChanges();
          return NoContent();
      }
  }

  RESPUESTA: No verifica si producto existe. Si Find devuelve null,
  Remove lanza NullReferenceException. Además, no usa async/await.

  ---

  Bug 5: ¿Qué error tiene este código?

  public int ObtenerEdad(DateTime fechaNacimiento)
  {
      return DateTime.Now.Year - fechaNacimiento.Year;
  }

  RESPUESTA: No considera si ya pasó el cumpleaños este año. Una persona
  nacida el 31/12/2020 tendría 5 años el 01/01/2026 con este cálculo,
  cuando en realidad tiene 5 recién desde el 31/12. Solución: comparar
  mes y día.
