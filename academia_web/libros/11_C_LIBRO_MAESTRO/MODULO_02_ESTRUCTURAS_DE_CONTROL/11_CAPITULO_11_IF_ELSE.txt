  -------------------------------------------------------------------------
  11.1  ¿QUÉ ES UNA ESTRUCTURA DE CONTROL?
  -------------------------------------------------------------------------

  Una estructura de control es una instrucción que altera el flujo
  secuencial de un programa. En C las estructuras de control se
  dividen en tres categorías:

    1. Secuenciales  → el programa avanza línea por línea.
    2. Condicionales → if else switch (toman decisiones).
    3. Repetitivas   → while for do...while (iteran).

  La más básica de las condicionales es if.

  -------------------------------------------------------------------------
  11.2  SENTENCIA if
  -------------------------------------------------------------------------

  La sintaxis de if es:

    if (condición) {
        // bloque de código si la condición es verdadera
    }

  La condición debe ir entre paréntesis. Si la condición se evalúa
  como verdadera (cualquier valor distinto de cero) el bloque se
  ejecuta. Si es falsa (cero) se salta.

  EJEMPLO 1: IF SIMPLE

    #include <stdio.h>

    int main() {
        int edad = 18;

        if (edad >= 18) {
            printf("Eres mayor de edad.\n");
        }

        return 0;
    }

  Salida:
    Eres mayor de edad.

  -------------------------------------------------------------------------
  11.3  SENTENCIA if...else
  -------------------------------------------------------------------------

  Cuando queremos ejecutar un bloque si la condición es verdadera
  y otro si es falsa usamos else:

    if (condición) {
        // bloque verdadero
    } else {
        // bloque falso
    }

  EJEMPLO 2: IF...ELSE

    #include <stdio.h>

    int main() {
        int nota = 45;

        if (nota >= 60) {
            printf("Aprobado.\n");
        } else {
            printf("Reprobado.\n");
        }

        return 0;
    }

  Salida:
    Reprobado.

  -------------------------------------------------------------------------
  11.4  ESCALERA else if
  -------------------------------------------------------------------------

  Cuando hay múltiples condiciones que evaluar en cadena usamos
  else if:

    if (condición1) {
        // bloque 1
    } else if (condición2) {
        // bloque 2
    } else if (condición3) {
        // bloque 3
    } else {
        // bloque por defecto
    }

  El programa evalúa de arriba abajo. En cuanto encuentra una
  condición verdadera ejecuta su bloque y sale de la estructura.

  EJEMPLO 3: ESCALERA ELSE IF

    #include <stdio.h>

    int main() {
        int nota;

        printf("Ingrese la nota (0-100): ");
        scanf("%d", &nota);

        if (nota >= 90) {
            printf("Calificación: A\n");
        } else if (nota >= 80) {
            printf("Calificación: B\n");
        } else if (nota >= 70) {
            printf("Calificación: C\n");
        } else if (nota >= 60) {
            printf("Calificación: D\n");
        } else {
            printf("Calificación: F\n");
        }

        return 0;
    }

  Salida (para nota=85):
    Calificación: B

  -------------------------------------------------------------------------
  11.5  OPERADORES RELACIONALES
  -------------------------------------------------------------------------

  Los operadores relacionales se usan dentro de la condición del
  if para comparar valores:

    OPERADOR  SIGNIFICADO        EJEMPLO (verdadero si...)
    ========  ================   =========================
    ==        igual a            a == b
    !=        diferente de       a != b
    <         menor que          a < b
    >         mayor que          a > b
    <=        menor o igual      a <= b
    >=        mayor o igual      a >= b

  IMPORTANTE: No confundas = (asignación) con == (comparación).

  -------------------------------------------------------------------------
  11.6  ERROR COMÚN: ASIGNACIÓN EN LUGAR DE COMPARACIÓN
  -------------------------------------------------------------------------

  Este es uno de los errores más frecuentes en C:

  INCORRECTO:
    if (x = 5) {   // ¡asigna 5 a x no compara!
        ...
    }

  CORRECTO:
    if (x == 5) {  // compara x con 5
        ...
    }

  ¿Por qué el incorrecto "funciona"? Porque en C cualquier valor
  distinto de cero es verdadero. La asignación x = 5 devuelve 5
  que es verdadero por lo que el bloque siempre se ejecuta.

  Para evitar esto algunos programadores escriben:

    if (5 == x) {   // notación Yoda: si olvidas un = da error
        ...
    }

  Porque 5 = x no es válido (no se puede asignar a una constante)
  y el compilador lo detecta.

  -------------------------------------------------------------------------
  11.7  if ANIDADOS
  -------------------------------------------------------------------------

  Podemos colocar un if dentro de otro if. Esto se llama anidación.

    if (condición1) {
        if (condición2) {
            // ambas condiciones son verdaderas
        }
    }

  EJEMPLO 4: IF ANIDADO

    #include <stdio.h>

    int main() {
        int edad;
        char tieneLicencia;

        printf("Edad: ");
        scanf("%d", &edad);

        if (edad >= 18) {
            printf("¿Tienes licencia? (s/n): ");
            scanf(" %c", &tieneLicencia);

            if (tieneLicencia == 's' || tieneLicencia == 'S') {
                printf("Puedes conducir.\n");
            } else {
                printf("Necesitas una licencia.\n");
            }
        } else {
            printf("Eres menor de edad no puedes conducir.\n");
        }

        return 0;
    }

  -------------------------------------------------------------------------
  11.8  MÚLTIPLES CONDICIONES CON OPERADORES LÓGICOS
  -------------------------------------------------------------------------

  Podemos combinar condiciones usando operadores lógicos:

    &&  → AND lógico (todas deben ser verdaderas)
    ||  → OR lógico  (al menos una debe ser verdadera)
    !   → NOT lógico (invierte el valor)

  EJEMPLO 5: CONDICIONES COMPUESTAS

    #include <stdio.h>

    int main() {
        int edad;
        float ingresos;

        printf("Edad: ");
        scanf("%d", &edad);
        printf("Ingresos mensuales: ");
        scanf("%f", &ingresos);

        if (edad >= 18 && edad <= 65 && ingresos >= 1000) {
            printf("Califica para el crédito.\n");
        } else {
            printf("No califica para el crédito.\n");
        }

        return 0;
    }

  -------------------------------------------------------------------------
  11.9  BUENAS PRÁCTICAS CON if
  -------------------------------------------------------------------------

  1. Siempre usa llaves {} incluso para una sola línea.
     Esto mejora la legibilidad y evita errores al añadir código.

  2. Mantén la indentación consistente (2 o 4 espacios).

  3. No anides más de 3 niveles. Si es necesario extrae
     la lógica a una función.

  4. Evalúa primero las condiciones más probables o más
     específicas en una escalera else if.

  5. Usa paréntesis para agrupar condiciones compuestas
     aunque no sean estrictamente necesarios.

  6. Prefiere la notación Yoda (5 == x) si tiendes a
     confundir = con ==.

  -------------------------------------------------------------------------
  11.10  EJEMPLO COMPLETO: CALCULADORA DE IMC
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        float peso altura imc;

        printf("=== CALCULADORA DE IMC ===\n");
        printf("Peso (kg): ");
        scanf("%f", &peso);
        printf("Altura (m): ");
        scanf("%f", &altura);

        imc = peso / (altura * altura);

        printf("Tu IMC es: %.2f\n", imc);

        if (imc < 18.5) {
            printf("Clasificación: Bajo peso\n");
        } else if (imc < 25) {
            printf("Clasificación: Normal\n");
        } else if (imc < 30) {
            printf("Clasificación: Sobrepeso\n");
        } else if (imc < 35) {
            printf("Clasificación: Obesidad grado I\n");
        } else if (imc < 40) {
            printf("Clasificación: Obesidad grado II\n");
        } else {
            printf("Clasificación: Obesidad grado III\n");
        }

        return 0;
    }

  -------------------------------------------------------------------------
  11.11  EL PROBLEMA DEL else PÉNDULO
  -------------------------------------------------------------------------

  En C el else se asocia al if más cercano. Esto puede causar
  ambigüedad:

    if (x > 0)
        if (y > 0)
            printf("x e y positivos\n");
    else
        printf("x negativo o cero\n");

  El else está alineado con el primer if visualmente pero el
  compilador lo asocia al segundo if. Usar llaves evita esto:

    if (x > 0) {
        if (y > 0) {
            printf("x e y positivos\n");
        }
    } else {
        printf("x negativo o cero\n");
    }

  -------------------------------------------------------------------------
  11.12  TABLA DE VERDAD DE CONDICIONES
  -------------------------------------------------------------------------

  Toda condición en C se evalúa como:

    Valor 0          → falso
    Valor != 0       → verdadero

  Esto incluye:

    if (0)           → nunca ejecuta
    if (1)           → siempre ejecuta
    if (-1)          → siempre ejecuta (es distinto de cero)
    if (variable)    → verdadero si variable != 0

  -------------------------------------------------------------------------
  11.13  COMPARACIÓN DE CADENAS
  -------------------------------------------------------------------------

  Para comparar cadenas NO uses == (compara direcciones no
  contenido). Usa strcmp() de <string.h>:

    #include <stdio.h>
    #include <string.h>

    int main() {
        char clave[20];

        printf("Ingrese la clave: ");
        scanf("%19s", clave);

        if (strcmp(clave, "secreta") == 0) {
            printf("Acceso concedido.\n");
        } else {
            printf("Acceso denegado.\n");
        }

        return 0;
    }

  strcmp devuelve:
    • 0 si las cadenas son iguales
    • < 0 si la primera es menor
    • > 0 si la primera es mayor

  -------------------------------------------------------------------------
  11.14  OPERADOR TERNARIO COMO ALTERNATIVA
  -------------------------------------------------------------------------

  Para asignaciones simples condicionales podemos usar el
  operador ternario:

    int max = (a > b) ? a : b;

  Es equivalente a:

    int max;
    if (a > b) {
        max = a;
    } else {
        max = b;
    }

  Veremos esto a detalle en el capítulo 18.

===============================================================================
  PREGUNTAS FRECUENTES (FAQ)
===============================================================================

  P: ¿Es obligatorio usar llaves {} en if?
  R: No pero es una buena práctica. Sin llaves solo la primera
     línea después del if pertenece al bloque.
     if (x > 0) printf("Positivo"); printf("Siempre"); 
     La segunda línea se ejecuta siempre.

  P: ¿Cuántos else if puedo encadenar?
  R: No hay límite teórico pero por legibilidad no se
     recomienda más de 5-7. Para muchos casos usa switch.

  P: ¿Puedo poner un if dentro de un else?
  R: Sí. El anidamiento es libre pero debe mantenerse legible.

  P: ¿Qué diferencia hay entre = y ==?
  R: = asigna un valor a una variable. == compara dos valores.
     if (x = 5) asigna 5 a x y siempre es verdadero.
     if (x == 5) compara x con 5.

  P: ¿Cómo se comparan cadenas en C?
  R: Con strcmp() de <string.h>. strcmp(cad1 cad2) == 0
     indica igualdad.

  P: ¿Qué significa "else péndulo"?
  R: Es la ambigüedad que ocurre cuando un else podría
     asociarse a más de un if. En C siempre se asocia al
     if más cercano. Las llaves resuelven la ambigüedad.

  P: ¿Se puede usar if sin else?
  R: Sí perfectamente. else es opcional.

  P: ¿Qué pasa si la condición contiene efectos secundarios?
  R: Las condiciones se evalúan de izquierda a derecha con
     short-circuit. Por ejemplo if (x != 0 && 10/x > 2)
     no divide por cero porque si x == 0 la segunda parte
     no se evalúa.
