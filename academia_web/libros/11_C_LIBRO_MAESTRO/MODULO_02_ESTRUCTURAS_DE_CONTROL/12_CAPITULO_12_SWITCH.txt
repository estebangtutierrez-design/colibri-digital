  BIENVENIDA

  Bienvenido al Capítulo 12.

  En el capítulo anterior aprendiste a tomar decisiones con
  if y else. Pero cuando tienes muchas opciones posibles
  una escalera de else if puede volverse larga y difícil
  de leer.

  Aquí entra switch: la estructura condicional múltiple
  de C. Con switch puedes evaluar una expresión y elegir
  entre múltiples caminos de forma clara y eficiente.

  Es perfecto para menús estados códigos de error y
  cualquier situación donde una variable pueda tomar
  varios valores discretos.

  -------------------------------------------------------------------------
  12.1  ¿QUÉ ES switch?
  -------------------------------------------------------------------------

  switch es una estructura de control que evalúa una
  expresión y compara su valor contra una lista de
  constantes (case). Cuando encuentra una coincidencia
  ejecuta el código asociado hasta encontrar un break.

  Sintaxis general:

    switch (expresión) {
        case constante1:
            // código
            break;
        case constante2:
            // código
            break;
        ...
        default:
            // código cuando ningún case coincide
    }

  La expresión debe ser de tipo entero (int char short
  long enum). No puede ser float double ni string.

  -------------------------------------------------------------------------
  12.2  COMPONENTES DE switch
  -------------------------------------------------------------------------

  switch (expresión):
    Evalúa la expresión una sola vez al inicio.

  case valor:
    Etiqueta que contiene un valor constante. Si la
    expresión coincide se ejecuta desde ahí.

  break:
    Sale de la estructura switch. Sin break el flujo
    continúa al siguiente case (fall-through).

  default:
    Opcional. Se ejecuta si ningún case coincide.
    Puede ir al final o en cualquier posición.

  -------------------------------------------------------------------------
  12.3  EJEMPLO BÁSICO: DÍA DE LA SEMANA
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        int dia;

        printf("Ingrese un número (1-7): ");
        scanf("%d", &dia);

        switch (dia) {
            case 1:
                printf("Lunes
");
                break;
            case 2:
                printf("Martes
");
                break;
            case 3:
                printf("Miércoles
");
                break;
            case 4:
                printf("Jueves
");
                break;
            case 5:
                printf("Viernes
");
                break;
            case 6:
                printf("Sábado
");
                break;
            case 7:
                printf("Domingo
");
                break;
            default:
                printf("Número inválido (1-7)
");
        }

        return 0;
    }

  -------------------------------------------------------------------------
  12.4  EL FALL-THROUGH
  -------------------------------------------------------------------------

  El fall-through ocurre cuando un case no termina con
  break y la ejecución continúa al siguiente case.

  A veces es intencional y útil:

    #include <stdio.h>

    int main() {
        char c;

        printf("Ingrese una letra: ");
        scanf(" %c", &c);

        switch (c) {
            case 'a':
            case 'e':
            case 'i':
            case 'o':
            case 'u':
                printf("Es una vocal.
");
                break;
            default:
                printf("Es consonante o símbolo.
");
        }

        return 0;
    }

  También funciona con rangos numéricos:

    switch (nota) {
        case 0:
        case 1:
        case 2:
        case 3:
        case 4:
        case 5:
            printf("Reprobado
");
            break;
        case 6:
        case 7:
            printf("Regular
");
            break;
        case 8:
        case 9:
            printf("Bueno
");
            break;
        case 10:
            printf("Excelente
");
            break;
    }

  -------------------------------------------------------------------------
  12.5  switch CON char
  -------------------------------------------------------------------------

  Podemos usar caracteres en los case porque char es
  un tipo entero en C:

    #include <stdio.h>

    int main() {
        char operador;
        int a = 10 b = 5;

        printf("Operador (+, -, *, /): ");
        scanf(" %c", &operador);

        switch (operador) {
            case '+':
                printf("%d + %d = %d
", a, b a + b);
                break;
            case '-':
                printf("%d - %d = %d
", a, b a - b);
                break;
            case '*':
                printf("%d * %d = %d
", a, b a * b);
                break;
            case '/':
                printf("%d / %d = %d
", a, b a / b);
                break;
            default:
                printf("Operador inválido.
");
        }

        return 0;
    }

  -------------------------------------------------------------------------
  12.6  switch ANIDADOS
  -------------------------------------------------------------------------

  Podemos anidar switch dentro de otro switch:

    #include <stdio.h>

    int main() {
        int nivel opcion;

        printf("Nivel de acceso (1-3): ");
        scanf("%d", &nivel);

        switch (nivel) {
            case 1:
                printf("Menú básico:
");
                printf("1. Ver perfil
");
                printf("2. Salir
");
                break;
            case 2:
                printf("Menú intermedio:
");
                printf("1. Ver perfil
");
                printf("2. Editar perfil
");
                printf("3. Salir
");
                break;
            case 3:
                printf("Menú administrador:
");
                printf("1. Ver perfil
");
                printf("2. Editar perfil
");
                printf("3. Usuarios
");
                printf("4. Configuración
");
                printf("5. Salir
");

                printf("Seleccione opción: ");
                scanf("%d", &opcion);

                switch (opcion) {
                    case 1:
                        printf("Perfil del admin
");
                        break;
                    case 3:
                        printf("Gestión de usuarios
");
                        break;
                    case 4:
                        printf("Configuración del sistema
");
                        break;
                    default:
                        printf("Opción no disponible
");
                }
                break;
            default:
                printf("Nivel inválido.
");
        }

        return 0;
    }

  -------------------------------------------------------------------------
  12.7  switch CON ENUMERACIONES
  -------------------------------------------------------------------------

  Las enumeraciones hacen que switch sea más legible:

    #include <stdio.h>

    enum Estado { INACTIVO ACTIVO SUSPENDIDO BANEADO };

    int main() {
        enum Estado usuario = SUSPENDIDO;

        switch (usuario) {
            case INACTIVO:
                printf("Cuenta inactiva.
");
                break;
            case ACTIVO:
                printf("Cuenta activa.
");
                break;
            case SUSPENDIDO:
                printf("Cuenta suspendida.
");
                break;
            case BANEADO:
                printf("Cuenta baneada.
");
                break;
        }

        return 0;
    }

  -------------------------------------------------------------------------
  12.8  ERRORES COMUNES CON switch
  -------------------------------------------------------------------------

  1. Olvidar el break (fall-through no intencional):

       case 1:
           printf("Opción 1\n");
           // falta break → ejecuta también case 2
       case 2:
           printf("Opción 2\n");
           break;

  2. Usar float o double en la expresión:

       float x = 3.5;
       switch (x) {  // ERROR: float no permitido
           case 3.5: ...
       }

  3. Usar variables en case:

       int valor = 10;
       switch (x) {
           case valor:  // ERROR: debe ser constante
       }

  4. Olvidar default para manejar valores inesperados.

  5. Dos case con el mismo valor:

       case 5: ...
       case 5: ...  // ERROR: duplicado

  -------------------------------------------------------------------------
  12.9  switch vs if-else
  -------------------------------------------------------------------------

  ¿Cuándo usar cada uno?

  USA switch CUANDO:
    • La expresión es de tipo entero.
    • Hay 3 o más opciones discretas.
    • Las opciones son valores constantes conocidos.
    • Necesitas claridad y legibilidad.

  USA if-else CUANDO:
    • Las condiciones son rangos (x > 10 && x < 20).
    • Las condiciones son expresiones complejas.
    • Comparas floats o strings.
    • Solo tienes 1 o 2 opciones.

  EJEMPLO DE COMPARACIÓN:

  Con if-else:

    if (opcion == 1) {
        printf("Rojo\n");
    } else if (opcion == 2) {
        printf("Verde\n");
    } else if (opcion == 3) {
        printf("Azul\n");
    } else {
        printf("Desconocido\n");
    }

  Con switch:

    switch (opcion) {
        case 1: printf("Rojo\n");    break;
        case 2: printf("Verde\n");   break;
        case 3: printf("Azul\n");    break;
        default: printf("Desconocido\n");
    }

  El switch es más limpio y rápido.

  -------------------------------------------------------------------------
  12.10  EL ORDEN DE default
  -------------------------------------------------------------------------

  default puede colocarse en cualquier posición pero
  por convención va al final:

    switch (x) {
        default:
            printf("Por defecto\n");
            break;
        case 1:
            printf("Uno\n");
            break;
    }

  Si default está al inicio y no tiene break también
  hay fall-through hacia los case siguientes.

  -------------------------------------------------------------------------
  12.11  switch Y SALTOS EN LÍNEA
  -------------------------------------------------------------------------

  En C23 se añadió la sintaxis simplificada con
  case -> pero en C estándar (C11 C17) se usa la
  sintaxis tradicional con case: y break.

  No uses goto dentro de switch a menos que sea
  absolutamente necesario. Rompe la estructura.

  -------------------------------------------------------------------------
  12.12  EJEMPLO COMPLETO: MENÚ DE RESTAURANTE
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        int opcion cantidad;
        float total = 0;

        printf("=== MENÚ DEL DÍA ===\n");
        printf("1. Desayuno     - $5.50\n");
        printf("2. Almuerzo      - $8.00\n");
        printf("3. Cena          - $10.00\n");
        printf("4. Postre        - $3.50\n");
        printf("5. Bebida        - $2.00\n");
        printf("6. Salir\n");

        do {
            printf("\nSeleccione opción: ");
            scanf("%d", &opcion);

            switch (opcion) {
                case 1:
                    printf("¿Cuántos desayunos? ");
                    scanf("%d", &cantidad);
                    total += cantidad * 5.50;
                    printf("Agregado.\n");
                    break;
                case 2:
                    printf("¿Cuántos almuerzos? ");
                    scanf("%d", &cantidad);
                    total += cantidad * 8.00;
                    break;
                case 3:
                    printf("¿Cuántas cenas? ");
                    scanf("%d", &cantidad);
                    total += cantidad * 10.00;
                    break;
                case 4:
                    printf("¿Cuántos postres? ");
                    scanf("%d", &cantidad);
                    total += cantidad * 3.50;
                    break;
                case 5:
                    printf("¿Cuántas bebidas? ");
                    scanf("%d", &cantidad);
                    total += cantidad * 2.00;
                    break;
                case 6:
                    printf("Total a pagar: $%.2f\n", total);
                    printf("¡Gracias por su visita!\n");
                    break;
                default:
                    printf("Opción no válida.\n");
            }
        } while (opcion != 6);

        return 0;
    }

===============================================================================
  RESUMEN DEL CAPÍTULO
===============================================================================

  • switch evalúa una expresión entera contra constantes.
  • case etiqueta cada valor posible.
  • break sale del switch sin break hay fall-through.
  • default maneja valores no contemplados.
  • La expresión debe ser entera (int char short long).
  • No se permiten floats ni strings en case.
  • El fall-through puede ser útil para agrupar casos.
  • switch es más legible que if-else para 3+ opciones.
  • Siempre incluye default para robustez.
  • Las enumeraciones mejoran la legibilidad del switch.

===============================================================================
  PREGUNTAS FRECUENTES (FAQ)
===============================================================================

  P: ¿Por qué switch solo acepta tipos enteros?
  R: Porque switch funciona con saltos calculados en
     una tabla de saltos (jump table). Esto requiere
     valores enteros conocidos en tiempo de compilación.

  P: ¿Qué pasa si olvido el break?
  R: La ejecución continúa al siguiente case (fall-through).
     A veces es intencional pero a menudo es un error.

  P: ¿Se puede usar switch con cadenas?
  R: No directamente. Debes usar if con strcmp() o
     convertir la cadena a un entero (hash o enum).

  P: ¿Default es obligatorio?
  R: No pero es una buena práctica. Ayuda a manejar
     valores inesperados.

  P: ¿Puedo poner default en medio de los cases?
  R: Sí pero no es recomendable. Mejor al final.

  P: ¿Cuál es la diferencia entre switch y if-else?
  R: switch solo compara igualdad contra constantes.
     if-else puede evaluar rangos y expresiones complejas.
     switch suele ser más rápido para muchas opciones.

  P: ¿Se pueden anidar switch?
  R: Sí completamente.

  P: ¿Por qué no se permiten floats en switch?
  R: La igualdad exacta con floats es peligrosa por
     errores de precisión. Además la tabla de saltos
     requiere valores enteros discretos.

  │      SISTEMA DE GESTIÓN DE BIBLIOTECA (CONSOLA)            │
  └─────────────────────────────────────────────────────────────┘

  Desarrolla un programa que simule un sistema de gestión
  de biblioteca con menú interactivo usando switch.

  REQUISITOS:

  1. Menú principal con opciones:
     1. Prestar libro
     2. Devolver libro
     3. Consultar disponibilidad
     4. Multas
     5. Reportes
     6. Salir

  2. Submenús con switch anidados:
     • Prestar libro: solicitar código del libro y ID
       del socio. Validar que el libro esté disponible.
       Si está disponible marcarlo como prestado.
     • Devolver libro: solicitar código y registrar
       devolución. Si hay retraso calcular multa.
     • Consultar disponibilidad: mostrar libros disponibles.
     • Multas: calcular multa según días de retraso
       ($2 por día con tope de $100).
     • Reportes: mostrar estadísticas (libros prestados
       disponibles con multa).

  3. Datos iniciales:
     • 5 libros con código título y estado.
     • 3 socios con ID nombre y libros prestados.

  4. Reglas de negocio:
     • Un socio puede tener máximo 3 libros prestados.
     • El plazo de préstamo es 7 días.
     • Multa = $2 por día de retraso sin exceder $100.
     • Los libros dañados tienen multa adicional de $50.

  5. Validaciones:
     • Códigos de libro existen.
     • IDs de socio existen.
     • No prestar libro ya prestado.
     • No devolver libro no prestado.

  REQUISITOS TÉCNICOS:
  • Usa switch para todos los menús.
  • Implementa fall-through intencional en al menos
    un lugar.
  • Usa if solo para validaciones.
  • Código indentado con 4 espacios.
  • Cada submenú debe tener opción "Volver".
