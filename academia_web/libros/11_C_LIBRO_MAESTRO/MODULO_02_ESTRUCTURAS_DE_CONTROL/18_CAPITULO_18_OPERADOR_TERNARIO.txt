  -------------------------------------------------------------------------
  18.1  SINTASIS DEL OPERADOR TERNARIO
  -------------------------------------------------------------------------


  El operador ternario tiene tres partes:

    condicion ? expresion_si_verdadero : expresion_si_falso

  La condicion se evalua primero. Si es verdadera
  (distinta de cero), se evalua y devuelve la primera
  expresion. Si es falsa, la segunda.

  EJEMPLO 1: BASICO

```c
#include <stdio.h>

int main() {
    int x = 10, y = 20;
    int maximo;

    maximo = (x > y) ? x : y;

    printf("El mayor es: %d\n", maximo);
    printf("El menor es: %d\n", (x < y) ? x : y);

    return 0;
}```

  -------------------------------------------------------------------------
  18.2  TERNARIO vs IF-ELSE
  -------------------------------------------------------------------------


  TERNARIO:
    int max = (a > b) ? a : b;

  EQUIVALENTE IF-ELSE:
    int max;
    if (a > b) {
        max = a;
    } else {
        max = b;
    }

  El ternario es mas compacto, pero if-else es
  mas claro para logicas complejas.

  -------------------------------------------------------------------------
  18.3  ASIGNACIONES CONDICIONALES
  -------------------------------------------------------------------------


```c
#include <stdio.h>

int main() {
    int nota = 75;
    char *resultado;
    float bono = 1000.0f;

    // Asignacion condicional de string
    resultado = (nota >= 60) ? "Aprobado" : "Reprobado";

    // Asignacion condicional de numero
    bono = (nota >= 90) ? bono * 1.5f : bono;

    printf("Estado: %s\n", resultado);
    printf("Bono: $%.2f\n", bono);

    return 0;
}```

  -------------------------------------------------------------------------
  18.4  TERNARIO EN PRINTF
  -------------------------------------------------------------------------


  Una aplicacion comun es dentro de printf:

```c
#include <stdio.h>

int main() {
    int edad = 17;

    printf("Eres %s de edad.\n",
           (edad >= 18) ? "mayor" : "menor");

    int numero = 7;
    printf("%d es %s\n", numero,
           (numero % 2 == 0) ? "par" : "impar");

    return 0;
}```

  -------------------------------------------------------------------------
  18.5  TERNARIO ANIDADO
  -------------------------------------------------------------------------


  Se pueden anidar ternarios, pero CUIDADO con
  la legibilidad:

```c
#include <stdio.h>

int main() {
    int nota = 85;
    char *calificacion;

    calificacion = (nota >= 90) ? "A" :
                   (nota >= 80) ? "B" :
                   (nota >= 70) ? "C" :
                   (nota >= 60) ? "D" : "F";

    printf("Nota: %d -> %s\n", nota, calificacion);

    // Equivalente con if-else:
    if (nota >= 90)      calificacion = "A";
    else if (nota >= 80) calificacion = "B";
    else if (nota >= 70) calificacion = "C";
    else if (nota >= 60) calificacion = "D";
    else                 calificacion = "F";

    return 0;
}```

  -------------------------------------------------------------------------
  18.6  TERNARIO CON MULTIPLES CONDICIONES
  -------------------------------------------------------------------------


```c
#include <stdio.h>

int main() {
    int edad = 25;
    char tieneLicencia = 's';

    // Combinando condiciones
    char *puedeConducir = (edad >= 18 && tieneLicencia == 's')
                          ? "Puede conducir"
                          : "No puede conducir";

    printf("%s\n", puedeConducir);

    return 0;
}```

  -------------------------------------------------------------------------
  18.7  TERNARIO COMO ARGUMENTO DE FUNCION
  -------------------------------------------------------------------------


```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int a = 5, b = 10;

    // Usar ternario directamente como argumento
    printf("Max: %d\n", (a > b) ? a : b);

    // Ternario para elegir comportamiento
    int *ptr = (a < b) ? &a : &b;
    printf("Menor: %d\n", *ptr);

    // Con funciones
    int diferencia = abs((a > b) ? a - b : b - a);
    printf("Diferencia: %d\n", diferencia);

    return 0;
}```

  -------------------------------------------------------------------------
  18.8  TERNARIO CON OPERACIONES
  -------------------------------------------------------------------------


```c
#include <stdio.h>

int main() {
    int x = 5;

    // El ternario puede contener expresiones
    int resultado = (x > 0) ? x * 2 : x * 3;

    // Tambien llamadas a funciones
    printf("%s\n",
           (x % 2 == 0) ? "Es par" : "Es impar");

    // Asignacion multiple
    int a = 1, b = 2, c;
    c = (a < b) ? (a = 10, b = 20, a + b) : 0;
    // Esto es peligroso, no recomendado

    printf("Resultado: %d, a=%d, b=%d\n", c, a, b);

    return 0;
}```

  -------------------------------------------------------------------------
  18.9  TERNARIO vs IF-ELSE: RENDIMIENTO
  -------------------------------------------------------------------------


  En la mayoria de los compiladores modernos,
  ternario e if-else generan el mismo codigo
  de maquina. No hay diferencia de rendimiento.

  La decision es puramente de ESTILO y LEGIBILIDAD.

  -------------------------------------------------------------------------
  18.10  ERRORES COMUNES CON TERNARIO
  -------------------------------------------------------------------------


  1. Olvidar los parentesis en la condicion:
       int x = a > b ? a : b;  // funciona pero confuso

  2. Mezclar tipos incompatibles:
       char *s = (cond) ? "texto" : 42;  // ERROR

  3. Ternarios demasiado largos (ilegibles):
       x = (a) ? (b) ? (c) ? 1 : 2 : 3 : 4;

  4. Usar ternario donde if-else es mas claro:
       (cond) ? funcionA() : funcionB(); // mejor if-else

  -------------------------------------------------------------------------
  18.11  CUANDO USAR Y CUANDO EVITAR
  -------------------------------------------------------------------------


  USA TERNARIO CUANDO:
  * Asignas un valor basado en una condicion simple.
  * Usas el resultado directamente (printf, return).
  * La expresion cabe en una linea (max 80 cols).

  EVITA TERNARIO CUANDO:
  * La condicion es compleja (multiples &&, ||).
  * Las expresiones tienen efectos secundarios.
  * Necesitas mas de un nivel de anidacion.
  * El bloque verdadero/falso tiene varias lineas.

  -------------------------------------------------------------------------
  18.12  EJEMPLO COMPLETO: CALCULADORA DE PRECIOS
  -------------------------------------------------------------------------


```c
#include <stdio.h>

int main() {
    float precio, descuento, total;
    int cantidad;
    char *categoria;

    printf("Precio unitario: ");
    scanf("%f", &precio);
    printf("Cantidad: ");
    scanf("%d", &cantidad);

    // Ternario para descuento por cantidad
    descuento = (cantidad >= 100) ? 0.20f :
                (cantidad >= 50)  ? 0.15f :
                (cantidad >= 20)  ? 0.10f :
                (cantidad >= 10)  ? 0.05f : 0.0f;

    total = precio * cantidad * (1 - descuento);

    // Ternario para categoria de compra
    categoria = (cantidad >= 100) ? "Mayorista" :
                (cantidad >= 10)  ? "Mediano"   : "Minorista";

    printf("\n--- FACTURA ---\n");
    printf("Categoria: %s\n", categoria);
    printf("Subtotal: $%.2f\n", precio * cantidad);
    printf("Descuento: %.0f%%\n", descuento * 100);
    printf("Total: $%.2f\n", total);

    return 0;
}```

  RESUMEN DEL CAPITULO
===============================================================================

  * El operador ternario (?:) es un if-else compacto
  * Sintaxis: condicion ? valor_true : valor_false
  * Perfecto para asignaciones condicionales simples
  * Los ternarios anidados son peligrosos para la legibilidad
  * Comunmente usado en printf y return
  * No hay diferencia de rendimiento con if-else
  * Elige ternario por claridad, no por velocidad
  * Si no cabe en una linea, usa if-else

===============================================================================
  PREGUNTAS FRECUENTES (FAQ)
===============================================================================

  P: El operador ternario es mas rapido que if-else?
  R: No. Los compiladores modernos generan el mismo codigo de maquina para ambos.

  P: Puedo usar ternario sin asignar el resultado?
  R: Si, pero es confuso. Si solo ejecutas efectos secundarios, usa if-else.

  P: Los tipos en las dos ramas deben ser iguales?
  R: Deben ser compatibles. Si no, el compilador dara una advertencia o error.

  P: Cuantos ternarios puedo anidar?
  R: Tecnicamente ilimitados, pero mas de 2 niveles es ilegible.

  P: El ternario funciona con cualquier tipo de dato?
  R: Funciona con tipos escalares: int, float, char, punteros. Con structs depende del compilador.

  P: Puedo usar ternario en una macro?
  R: Si, es comun en macros para evitar if-else que no funcionan bien en macros.

  P: Que significa 'lvalue required' con ternario?
  R: El resultado del ternario es un rvalue (no se le puede asignar un valor).

  -------------------------------------------------------------------------
  18.13  TERNARIO CON PUNTEROS
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        int a = 100, b = 200;
        int *ptr;

        // Asignar puntero segun condicion
        ptr = (a > b) ? &a : &b;

        printf("El mayor es: %d\n", *ptr);

        // Ternario para elegir entre operaciones
        int x = 5;
        int *destino = (x > 0) ? &a : &b;
        *destino = 999;

        printf("a = %d, b = %d\n", a, b);

        return 0;
    }

  -------------------------------------------------------------------------
  18.14  TERNARIO EN MACROS
  -------------------------------------------------------------------------

    #include <stdio.h>

    #define MAX(a, b) ((a) > (b) ? (a) : (b))
    #define MIN(a, b) ((a) < (b) ? (a) : (b))
    #define ABS(x)    ((x) < 0 ? -(x) : (x))
    #define PAR(x)    ((x) % 2 == 0 ? 1 : 0)
    #define SIGN(x)   ((x) > 0 ? 1 : ((x) < 0 ? -1 : 0))

    int main() {
        int p = 15, q = 25;
        int r = -10;

        printf("MAX(%d, %d) = %d\n", p, q, MAX(p, q));
        printf("MIN(%d, %d) = %d\n", p, q, MIN(p, q));
        printf("ABS(%d) = %d\n", r, ABS(r));
        printf("PAR(%d) = %s\n", p, PAR(p) ? "si" : "no");
        printf("SIGN(%d) = %d\n", r, SIGN(r));

        return 0;
    }

  -------------------------------------------------------------------------
  18.15  TERNARIO EN ESTRUCTURAS DE CONTROL
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        // Ternario en while
        int limite = 10;
        int paso = 1;
        int i = 0;

        printf("Ascendente: ");
        while ((paso > 0) ? (i < limite) : (i >= 0)) {
            printf("%d ", i);
            i += paso;
        }
        printf("\n");

        // Ternario para decidir direccion
        int inicio = 10, fin = 0, dir = -1;
        i = inicio;

        printf("Descendente: ");
        while ((dir > 0) ? (i <= fin) : (i >= fin)) {
            printf("%d ", i);
            i += dir;
        }
        printf("\n");

        return 0;
    }

  -------------------------------------------------------------------------
  18.17  TERNARIO CON OPERACIONES LOGICAS
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        int x = 5, y = 10, z = 15;

        // Combinacion de ternario y logicos
        int resultado = (x > 0 && y > 0)
                       ? (z > 0 ? x + y + z : x + y)
                       : 0;

        printf("Resultado: %d\n", resultado);

        // Ternario dentro de ternario con logicos
        char *clasificacion =
            (x > 0 && y > 0 && z > 0) ? "Todos positivos" :
            (x < 0 && y < 0 && z < 0) ? "Todos negativos" :
            "Mixtos";

        printf("Clasificacion: %s\n", clasificacion);

        return 0;
    }

  -------------------------------------------------------------------------
  18.18  TERNARIO VS IF-ELSE: CASOS PRACTICOS
  -------------------------------------------------------------------------

    #include <stdio.h>

    int main() {
        int valor = 42;

        // CASO 1: Asignacion simple -> TERNARIO
        char *tipo = (valor % 2 == 0) ? "par" : "impar";

        // CASO 2: Accion multiple -> IF-ELSE
        if (valor > 100) {
            printf("Valor grande\n");
            valor = 100;
        } else {
            printf("Valor normal\n");
        }

        // CASO 3: Return condicional -> TERNARIO
        // return (encontrado) ? 0 : -1;

        // CASO 4: Efectos secundarios -> IF-ELSE
        // (condicion) ? funcionA() : funcionB(); // MAL: mejor if-else

        printf("Tipo: %s, Valor: %d\n", tipo, valor);

        return 0;
    }

===============================================================================
  RESPUESTAS A PREGUNTAS FRECUENTES ADICIONALES
===============================================================================

  P: El ternario genera codigo mas rapido?
  R: No. Los compiladores modernos optimizan ambos de la
     misma forma. Elige por legibilidad.

  P: Puedo usar ternario para llamar funciones?
  R: Si, pero es mejor usar if-else si las funciones
     tienen efectos secundarios importantes.

  P: Que tipo de dato devuelve el ternario?
  R: El tipo se determina por las dos expresiones.
     Deben ser compatibles o el compilador hara una
     conversion implicita.

  P: El ternario es un lvalue o rvalue?
  R: En C estandar es un rvalue (no se le puede asignar).
     En C++ puede ser lvalue bajo ciertas condiciones.

  P: Como documentar un ternario complejo?
  R: Si no cabe en una linea, usa if-else. Si lo usas,
     comentalo: /* clasifica nota en A-F */.
