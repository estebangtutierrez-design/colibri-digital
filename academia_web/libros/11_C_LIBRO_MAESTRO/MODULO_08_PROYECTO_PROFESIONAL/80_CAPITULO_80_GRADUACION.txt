=== 12. GLOSARIO DE TERMINOS ===

A continuacion, un glosario de los terminos mas importantes
que has aprendido:

  ABSTRACCION: Ocultar detalles de implementacion, exponiendo
  solo la interfaz.

  ALINEACION: Como se organizan los datos en memoria para
  acceso eficiente por la CPU.

  ARITMETICA DE PUNTEROS: Operaciones matematicas sobre
  direcciones de memoria.

  ASLR: Address Space Layout Randomization. Tecnica de
  seguridad que aleatoriza las direcciones de memoria.

  BACKTRACE: Lista de llamadas a funcion activas en la pila.

  BREAKPOINT: Punto de detencion en GDB.

  BUFFER OVERFLOW: Escribir mas datos de los que un buffer
  puede contener.

  CACHE: Memoria rapida cerca de la CPU que almacena datos
  frecuentemente usados.

  CALLBACK: Funcion pasada como argumento a otra funcion.

  COMPLEJIDAD CICLOMATICA: Medida de la cantidad de caminos
  independientes en un programa.

  CORE DUMP: Archivo que contiene el estado de la memoria
  de un programa cuando falla.

  DANGLING POINTER: Puntero que referencia memoria ya
  liberada.

  DECLARACION: Anunciar la existencia de un simbolo.

  DEFINICION: Proporcionar la implementacion de un simbolo.

  DESBORDAMIENTO: Exceder la capacidad de un tipo de dato.

  DOXIGEN: Herramienta de documentacion para C/C++.

  FLOATING POINT: Representacion de numeros reales en
  computacion.

  FORMAT STRING: Cadena de formato usada por printf y
  funciones similares.

  FUGAS DE MEMORIA: Memoria asignada pero nunca liberada.

  HEAP: Area de memoria para asignacion dinamica.

  INLINE: Reemplazar una llamada a funcion con el cuerpo
  de la funcion.

  INTERFAZ: Archivo .h que define el contrato de un modulo.

  LTO: Link-Time Optimization. Optimizacion entre archivos
  durante el enlazado.

  MEMORY BARRIER: Instruccion que asegura orden en accesos
  a memoria en CPUs multinucleo.

  MUTEX: Mecanismo de sincronizacion para exclusion mutua.

  NUL: Caracter de terminacion de string.

  NULL: Macro que representa un puntero nulo.

  PADDING: Espacio adicional entre miembros de un struct
  para alineacion.

  PGO: Profile-Guided Optimization. Optimizacion basada
  en perfiles de ejecucion.

  PIPE: Mecanismo de comunicacion entre procesos.

  PUNTERO A FUNCION: Variable que almacena la direccion
  de una funcion.

  RACE CONDITION: Comportamiento incorrecto dependiente
  del orden de ejecucion de hilos.

  RECURSION: Tecnica donde una funcion se llama a si misma.

  RESTRICT: Keyword que indica que un puntero no se solapa
  con otros.

  SEGMENTATION FAULT: Acceso a memoria no permitida.

  STACK: Region de memoria para variables locales y
  llamadas a funcion.

  STATIC: Palabra clave que modifica la duracion o
  visibilidad de una variable/funcion.

  STRING: Arreglo de caracteres terminado en nulo.

  STRUCT: Tipo de dato compuesto por miembros de
  diferentes tipos.

  TIPO OPACO: Struct declarado pero no definido en el .h,
  ocultando su implementacion.

  TOCTOU: Time Of Check Time Of Use. Race condition entre
  verificacion y uso.

  UNDEFINED BEHAVIOR: Comportamiento no especificado por
  el estandar C.

  UNION: Tipo que puede contener diferentes tipos en la
  misma ubicacion de memoria.

  VALGRIND: Herramienta de instrumentacion dinamica para
  detectar errores de memoria.

  VARIADIC: Funcion que acepta numero variable de argumentos.

  VLA: Variable Length Array. Array con tamano determinado
  en tiempo de ejecucion.

  VOLATILE: Keyword que indica que una variable puede
  cambiar fuera del flujo normal.

  WATCHPOINT: Breakpoint que se activa cuando una variable
  cambia.

DEPURACION:
  gdb ./prog
  gdb ./prog core
  gdb -p PID
  valgrind --leak-check=full ./prog
  valgrind --tool=helgrind ./prog

ANALISIS:
  cppcheck --enable=all .
  scan-build gcc -c prog.c
  perf stat ./prog
  perf record ./prog && perf report

SISTEMA:
  strace -f ./prog
  ltrace ./prog
  ldd ./prog
  nm ./prog
  objdump -d ./prog

=== 6. PROYECTO DE GRADUACIÓN ===

Para celebrar tu graduación, aquí tienes un proyecto especial:
un mini shell (intérprete de comandos) en C, que integra todo
lo aprendido.

```c
// minishell.c - Shell de comandos minimalista
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/wait.h>
#include <fcntl.h>

#define MAX_COMANDO 1024
#define MAX_ARGUMENTOS 64

static void mostrar_prompt(void) {
    char cwd[1024];
    if (getcwd(cwd, sizeof(cwd))) {
        printf("\033[32m%s\033[0m $ ", cwd);
    } else {
        printf("$ ");
    }
    fflush(stdout);
}

static int ejecutar_comando(char** args) {
    pid_t pid = fork();

    if (pid == 0) {
        // Proceso hijo
        execvp(args[0], args);
        // Si llegamos aquí, hubo error
        fprintf(stderr, "Error: comando no encontrado: %s\n", args[0]);
        exit(1);
    } else if (pid > 0) {
        // Proceso padre
        int status;
        waitpid(pid, &status, 0);
        return WEXITSTATUS(status);
    } else {
        perror("fork");
        return -1;
    }
}

static int tokenizar(char* linea, char** args) {
    int count = 0;
    char* token = strtok(linea, " \t\n");

    while (token && count < MAX_ARGUMENTOS - 1) {
        args[count++] = token;
        token = strtok(NULL, " \t\n");
    }

    args[count] = NULL;
    return count;
}

int main(void) {
    char linea[MAX_COMANDO];
    char* args[MAX_ARGUMENTOS];

    printf("MiniShell v1.0 - Escribe 'salir' para terminar\n");

    while (1) {
        mostrar_prompt();

        if (!fgets(linea, sizeof(linea), stdin)) {
            printf("\n");
            break;
        }

        // Eliminar newline
        linea[strcspn(linea, "\n")] = '\0';

        // Ignorar líneas vacías
        if (strlen(linea) == 0) continue;

        // Comando interno: salir
        if (strcmp(linea, "salir") == 0 || strcmp(linea, "exit") == 0) {
            printf("Hasta luego!\n");
            break;
        }

        // Comando interno: cd
        if (strncmp(linea, "cd ", 3) == 0) {
            char* dir = linea + 3;
            if (chdir(dir) != 0) {
                perror("cd");
            }
            continue;
        }

        // Tokenizar y ejecutar
        int argc = tokenizar(linea, args);
        if (argc > 0) {
            ejecutar_comando(args);
        }
    }

    return 0;
}
```

Compílalo y pruébalo:

```bash
gcc -Wall -Wextra -std=c11 -o minishell minishell.c
./minishell
```

Este mini shell demuestra que puedes integrar: fork/exec, manejo
de procesos, strings, punteros, E/S estándar, y diseño limpio.

Ocho módulos. Ochenta capítulos. Cientos de ejemplos de código.
Miles de líneas escritas, depuradas, optimizadas y aseguradas.

Pero esto no es un adiós: es un hasta luego.

El mundo de C es vasto y fascinante. Cada día hay nuevos desafíos:
nuevos kernels que construir, nuevos dispositivos que programar,
nuevos algoritmos que implementar, nuevas vulnerabilidades que
prevenir.

Lleva contigo las lecciones aprendidas:
  - La paciencia de GDB cuando buscas un bug.
  - La disciplina de Valgrind cuando verificas memoria.
  - La visión arquitectónica cuando diseñas sistemas.
  - La conciencia de seguridad cuando escribes cada línea.

Y recuerda siempre las palabras de Dennis Ritchie, creador de C:

"C es un lenguaje simple, no es un lenguaje simple de dominar.
Pero vale la pena el esfuerzo."

Tú has hecho ese esfuerzo. Ahora disfruta los frutos.

¡Felicidades, C Master!
