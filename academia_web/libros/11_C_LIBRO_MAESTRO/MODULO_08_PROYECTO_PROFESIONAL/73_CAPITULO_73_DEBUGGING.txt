=== 1. INTRODUCCIÓN A GDB ===

GDB (GNU Debugger) es un depurador interactivo que te permite:

  - Ejecutar un programa paso a paso.
  - Detener la ejecución en puntos específicos.
  - Inspeccionar el valor de variables y memoria.
  - Modificar variables en tiempo de ejecución.
  - Analizar la pila de llamadas cuando ocurre un crash.

Compilación para depuración:

```bash
# El flag -g incluye información de depuración en el ejecutable
gcc -g -o programa programa.c

# Para optimización mínima con información de depuración
gcc -g -Og -o programa programa.c

# NUNCA uses -g con -O2/-O3 para depurar (las variables pueden
# desaparecer y el código se reordena)
```

Inicio de GDB:

```bash
# Iniciar con un ejecutable
gdb ./programa

# Iniciar con un core dump (post-mortem)
gdb ./programa core

# Adjuntarse a un proceso en ejecución
gdb -p 1234
```

=== 2. COMANDOS ESENCIALES DE GDB ===

```c
// programa_ejemplo.c - Para nuestras demostraciones
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    char  nombre[64];
    int   edad;
    float altura;
} Persona;

void saludar(Persona* p) {
    printf("Hola, soy %s y tengo %d años.\n",
           p->nombre, p->edad);
}

int calcular_imc(Persona* p) {
    // IMC = peso / (altura * altura)
    // Bug: no hay verificación de altura > 0
    float imc = 75.0f / (p->altura * p->altura);
    return (int)imc;
}

int main(void) {
    Persona* p = malloc(sizeof(Persona));
    if (!p) return 1;

    strcpy(p->nombre, "Lucas");
    p->edad = 30;
    p->altura = 1.75f;

    saludar(p);

    int imc = calcular_imc(p);
    printf("IMC: %d\n", imc);

    free(p);
    return 0;
}
```

Compilar y depurar:

```bash
gcc -g -o prog programa_ejemplo.c
gdb ./prog
```

Comandos básicos dentro de GDB:

```
(gdb) break main          # Poner breakpoint en main
(gdb) run                 # Ejecutar el programa
(gdb) next                # Ejecutar siguiente línea (sin entrar en funciones)
(gdb) step                # Ejecutar siguiente línea (entrando en funciones)
(gdb) continue            # Continuar hasta el siguiente breakpoint
(gdb) print variable      # Mostrar valor de variable
(gdb) print *p            # Mostrar contenido apuntado por p
(gdb) list                # Mostrar código fuente alrededor de la línea actual
(gdb) info locals         # Mostrar variables locales
(gdb) backtrace           # Mostrar pila de llamadas
(gdb) quit                # Salir de GDB
```

=== 3. BREAKPOINTS AVANZADOS ===

Los breakpoints (puntos de ruptura) son la herramienta más usada en GDB.

TIPOS DE BREAKPOINTS:

```bash
# Breakpoint en una línea específica
break programa.c:42

# Breakpoint en una función
break calcular_imc

# Breakpoint condicional
break programa.c:42 if x > 10

# Breakpoint temporal (se elimina después de la primera vez)
tbreak main

# Breakpoint con ignorar (no se detiene las primeras N veces)
ignore 1 5

# Breakpoint en una dirección de memoria
break *0x555555555151
```

WATCHPOINTS: Se detienen cuando una variable cambia.

```bash
# Detenerse cuando 'x' cambie
watch x

# Detenerse cuando 'x' se lea
rwatch x

# Detenerse cuando 'x' cambie o se lea
awatch x

# Watchpoint condicional
watch x if x > 100
```

CATCHPOINTS: Se detienen en eventos específicos.

```bash
# Detenerse en llamadas a malloc
catch syscall malloc

# Detenerse cuando se lanza una señal
catch signal SIGSEGV

# Detenerse cuando se lanza una excepción de C++
catch throw

# Detenerse en fork/clone
catch fork
catch vfork
catch clone
```

=== 4. NAVEGACIÓN POR LA PILA (BACKTRACE) ===

Cuando un programa falla, la pila de llamadas muestra el camino
que llevó al error.

```c
// crash.c - Programa con fallo
#include <stdio.h>

void funcion_c() {
    int* p = NULL;
    *p = 42;  // Segmentation fault!
}

void funcion_b() {
    funcion_c();
}

void funcion_a() {
    funcion_b();
}

int main() {
    funcion_a();
    return 0;
}
```

Depuración post-mortem:

```bash
$ gcc -g -o crash crash.c
$ ./crash
Segmentation fault (core dumped)

$ gdb ./crash core
```

=== 5. INSPECCIÓN DE MEMORIA ===

Una de las características más poderosas de GDB es la capacidad
de inspeccionar memoria directamente.

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    char str[] = "Hola, GDB!";
    char* heap = malloc(32);

    strcpy(heap, "Memoria dinámica");

    printf("%s\n", str);

    free(heap);
    return 0;
}
```

Comandos de inspección de memoria:

```
(gdb) break main
(gdb) run
(gdb) next 4

(gdb) print arr
$1 = {10, 20, 30, 40, 50}

(gdb) print &arr
$2 = (int (*)[5]) 0x7fffffffe210

(gdb) x/5wd 0x7fffffffe210
0x7fffffffe210:	10	20	30	40	50

(gdb) x/10cb str
0x7fffffffe200:	72 'H'	111 'o'	108 'l'	97 'a'	44 ','	32 ' '	71 'G'	68 'D'

(gdb) x/s str
0x7fffffffe200:	"Hola, GDB!"

(gdb) x/8gx heap
0x5555555592a0:	0x6d614d206174616d	0x696e616d696e2061
0x5555555592b0:	0x0000000000000061	0x0000000000000000

(gdb) x/s heap
0x5555555592a0:	"Memoria dinámica"
```

Formato de x (examine):
- x: hexadecimal
- d: decimal
- u: unsigned decimal
- o: octal
- t: binario
- c: carácter
- s: string
- i: instrucción
- f: float

Tamaños:
- b: 1 byte
- h: 2 bytes
- w: 4 bytes (word)
- g: 8 bytes (giant)

=== 6. DEPURACIÓN DE FUGAS DE MEMORIA ===

GDB combinado con técnicas de interceptación puede ayudar a
encontrar fugas de memoria.

```c
// leaks.c - Programa con fuga de memoria
#include <stdio.h>
#include <stdlib.h>

void crear_fuga() {
    int* datos = malloc(100 * sizeof(int));
    // No hay free(datos) -> FUGA
}

int main() {
    for (int i = 0; i < 100; i++) {
        crear_fuga();
    }
    printf("Creadas 100 fugas\n");
    return 0;
}
```

Técnica: breakpoint en malloc con seguimiento:

```
(gdb) break main
(gdb) run

# Configurar seguimiento de malloc
(gdb) break malloc
(gdb) commands
> silent
> printf "malloc(%zu) en %p\n", $rdi, $rax
> continue
> end

(gdb) break free
(gdb) commands
> silent
> printf "free(%p)\n", $rdi
> continue
> end

(gdb) continue
malloc(400) en 0x5555555592a0
malloc(400) en 0x5555555596b0
malloc(400) en 0x555555559ab0
...
```

Para un análisis más profundo, podemos registrar todas las
asignaciones y liberaciones:

```bash
# Usar gdb con logging
gdb -batch -ex "run" -ex "bt" ./leaks 2>&1 | tee gdb.log
```

=== 7. DEPURACIÓN DE CONDICIONES DE CARRERA ===

Para programas multihilo, GDB permite controlar la ejecución de
cada hilo individualmente.

```c
// race.c - Condición de carrera
#include <stdio.h>
#include <pthread.h>

int contador = 0;

void* incrementar(void* arg) {
    for (int i = 0; i < 100000; i++) {
        contador++;  // Race condition!
    }
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, incrementar, NULL);
    pthread_create(&t2, NULL, incrementar, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Contador: %d (esperado: 200000)\n", contador);
    return 0;
}
```

Depuración multihilo:

```
$ gcc -g -pthread -o race race.c
$ gdb ./race

(gdb) break incrementar
(gdb) run

(gdb) info threads
  Id   Target Id         Frame

* 3    Thread 0x7ffff7fbf700 (LWP 12347) "race" incrementar (arg=0x0) at race.c:8
```

=== 8. DEPURACIÓN AVANZADA: SCRIPTS PERSONALIZADOS ===

GDB soporta scripts para automatizar tareas repetitivas.

```gdb
# .gdbinit o script.gdb
set print pretty on
set pagination off
set confirm off

define rastrear_malloc
    break malloc
    commands
        silent
        printf "malloc(%zu) = %p\n", $rdi, $rax
        continue
    end
end

define rastrear_free
    break free
    commands
        silent
        printf "free(%p)\n", $rdi
        continue
    end
end

define reporte_fugas
    printf "=== REPORTE DE MEMORIA ===\n"
    info registers rsp
    maintenance info sections
end

document rastrear_malloc
    Configura un breakpoint en malloc que imprime cada asignación.
end
```

Uso:

```bash
$ gdb -x script.gdb ./programa
(gdb) rastrear_malloc
(gdb) rastrear_free
(gdb) run
```

=== 9. DEPURACIÓN DE PROGRAMAS OPTIMIZADOS ===

Cuando un programa está compilado con optimización (-O2, -O3),
GDB puede mostrar información confusa porque las variables pueden
haber sido optimizadas a registros o eliminadas.

```c
// optimizado.c
#include <stdio.h>

int main() {
    int a = 10;
    int b = 20;
    int c = a + b;  // El compilador puede calcular esto en tiempo de compilación
    printf("Resultado: %d\n", c);
    return 0;
}
```

```
$ gcc -g -O2 -o opt optimizado.c
$ gdb ./opt

(gdb) break main
(gdb) run

(gdb) print a
$1 = <optimized out>

(gdb) print b
$2 = <optimized out>

(gdb) info locals
No symbol table info available for a
```

Para depurar programas optimizados:

```bash
# Compilar con -Og (optimización amigable para depuración)
gcc -g -Og -o opt optimizado.c

# Forzar que las variables no se optimicen con volatile en el código
```

=== 10. DEPURACIÓN DE CORRUPCIÓN DE MEMORIA ===

La corrupción de memoria produce los bugs más difíciles de encontrar.
GDB puede ayudar a identificar dónde ocurre la corrupción.

```c
// corruption.c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    char buffer[16];
    int  flag = 0;

    // Bug: escribir más allá del buffer
    strcpy(buffer, "ESTE STRING ES DEMASIADO LARGO PARA EL BUFFER");
    // En este punto, flag está corrupto

    if (flag == 0) {
        printf("flag sigue siendo 0\n");
    } else {
        printf("flag está corrupto: %d\n", flag);
    }

    return 0;
}
```

Depuración:

```
$ gcc -g -fstack-protector-all -o corruption corruption.c
$ gdb ./corruption

(gdb) run

(gdb) print &buffer

(gdb) print &flag
```

=== 11. REVERSE DEBUGGING ===

GDB 7.0+ soporta depuración inversa, permitiendo retroceder en
la ejecución.

```
(gdb) target record-full  # Activar registro de ejecución

(gdb) break corruption.c:11
(gdb) run

(gdb) reverse-next  # Retroceder una línea

(gdb) reverse-step   # Retroceder entrando en funciones

(gdb) reverse-continue  # Retroceder hasta el breakpoint anterior

(gdb) print flag
$1 = 0  # Antes de la corrupción
```

=== 14. CASO PRÁCTICO: DEBUGGEANDO UN SERVIDOR WEB ===

```c
// mini_servidor.c - Con bugs intencionales
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <arpa/inet.h>

#define BUFFER_SIZE 256

void manejar_cliente(int client_fd) {
    char buffer[BUFFER_SIZE];
    char respuesta[512];

    // Bug 1: No se verifica el valor de retorno de read()
    read(client_fd, buffer, BUFFER_SIZE);  // Podría leer -1

    // Bug 2: No se valida que buffer tenga terminación nula
    if (strstr(buffer, "GET /") != NULL) {
        strcpy(respuesta, "HTTP/1.1 200 OK\r\nContent-Type: text/plain\r\n\r\n");
        strcat(respuesta, "Hola Mundo!");
        write(client_fd, respuesta, strlen(respuesta));
    }

    // Bug 3: No se cierra el socket
}

int main() {
    int server_fd = socket(AF_INET, SOCK_STREAM, 0);
    struct sockaddr_in addr = {
        .sin_family = AF_INET,
        .sin_port = htons(8080),
        .sin_addr.s_addr = INADDR_ANY
    };

    bind(server_fd, (struct sockaddr*)&addr, sizeof(addr));
    listen(server_fd, 5);

    while (1) {
        int client_fd = accept(server_fd, NULL, NULL);
        manejar_cliente(client_fd);
        close(client_fd);  // Bug 4: close debería estar dentro de manejar_cliente
    }
}
```

________________________________________________________________________________
  PREGUNTAS FRECUENTES
________________________________________________________________________________

P: ¿Por qué GDB dice "variable optimized out"?
R: El compilador optimizó la variable. Compila con -Og en lugar
de -O2/-O3 para preservar las variables durante la depuración.

P: ¿Cómo depuro un programa que ya se colgó?
R: Si tienes un core dump, usa: gdb ./programa core. Si no,
configura el sistema para generar core dumps con: ulimit -c unlimited.

P: ¿Cómo salgo de un bucle infinito en GDB?
R: Presiona Ctrl+C para interrumpir la ejecución y volver al
prompt de GDB.

P: ¿Puedo cambiar el valor de una variable mientras depuro?
R: Sí, usa: set variable = nuevo_valor. Esto es muy útil para
probar correcciones sin recompilar.

P: ¿GDB funciona con programas en C++?
R: Sí, GDB soporta C++, incluyendo namespaces, templates y
excepciones. Usa comandos como break MyClass::method.
