1. EL PROGRAMA "HELLO, WORLD"
===============================================================================

**1.1 El origen de una tradición**

El programa "Hello, World" apareció por primera vez en 1972 en un
informe interno de Bell Labs titulado *"A Tutorial Introduction to
the Language B"* de Brian Kernighan. Cuando Kernighan y Ritchie
escribieron "The C Programming Language" en 1978, incluyeron el
mismo ejemplo, y desde entonces se ha convertido en el programa
iniciático de cualquier lenguaje de programación.

**1.2 Tu primer programa**

Aquí está. Léelo con atención, aunque no entiendas cada detalle
todavía. Lo analizaremos línea por línea a continuación.

```c
/*
 * primer_programa.c - Mi primer programa en C
 * Muestra un mensaje en la consola.
 */

#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}
```

**1.3 Compilar y ejecutar**

Para ver este programa en acción:

```bash

# Luego compila:

# Ejecuta:
./hola
```

Salida esperada:
```
Hello, World!
```

2. ANATOMÍA DE UN PROGRAMA EN C
===============================================================================

Vamos a diseccionar el programa línea por línea.

**2.1 Los comentarios**

```c
/*
 * primer_programa.c - Mi primer programa en C
 * Muestra un mensaje en la consola.
 */
```

Estas líneas son **comentarios**. El compilador las ignora por
completo. Sirven para documentar el código. En C hay dos tipos:

- `/* ... */` — Comentarios multilínea.
- `// ...` — Comentarios de una línea (desde C99).

```c
#include <stdio.h>  // Este es un comentario de una línea

/* Este es un
   comentario
   multilínea */
```

**2.2 La directiva #include**

```c
#include <stdio.h>
```

Esta línea es una **directiva de preprocesador**. Le dice al
compilador: "antes de compilar, incluye el contenido del archivo
`stdio.h` aquí".

`stdio.h` es un archivo de cabecera (header) que pertenece a la
**Biblioteca Estándar de C**. Su nombre significa "Standard Input
Output". Contiene las declaraciones de funciones como:

- `printf()` — Imprimir texto formateado en la salida estándar.
- `scanf()` — Leer datos formateados desde la entrada estándar.
- `getchar()` — Leer un carácter desde la entrada estándar.
- `putchar()` — Escribir un carácter en la salida estándar.

Sin `#include <stdio.h>`, el compilador no sabría qué es `printf`
y produciría un error.

```c
/* Sin #include <stdio.h>, esto no compilaría */
#include <stdio.h>  /* ¡Esta línea es obligatoria para usar printf! */

int main(void) {
    printf("Siempre incluye <stdio.h> cuando uses printf o scanf.\n");
    return 0;
}
```

**2.3 La función main()**

```c
int main(void) {
```

Esta línea declara la **función principal** del programa. Analicemos
sus partes:

- `int` — El tipo de dato que la función devuelve (un entero).
- `main` — El nombre de la función. `main` es especial: es el punto
  de entrada del programa. Cuando ejecutas un programa, el sistema
  operativo llama a `main`.
- `(void)` — Los parámetros que recibe la función. `void` significa
  "ningún parámetro".

Todo programa en C **debe** tener una función `main`. Es donde
comienza la ejecución. Sin `main`, el enlazador producirá un error.

**Formas válidas de main():**

```c
int main(void)                /* Sin parámetros - recomendada */
int main()                    /* Parámetros no especificados */
int main(int argc, char *argv[])  /* Con argumentos de línea de comandos */
void main()                   /* Incorrecto - el estándar requiere int */
```

El estándar C (C11) dice que `main` debe devolver `int`. Usar
`void main()` es incorrecto aunque algunos compiladores lo acepten.

```c
#include <stdio.h>

int main(void) {
    printf("La función main es el corazón de todo programa en C.\n");
    printf("Sin ella, no hay programa.\n");
    return 0;
}
```

**2.4 La función printf()**

```c
    printf("Hello, World!\n");
```

`printf` es una función de la biblioteca estándar que **imprime
texto formateado** en la consola.

Analicemos sus partes:

- `printf` — El nombre de la función.
- `("Hello, World!\n")` — El argumento que le pasamos. Es una
  **cadena de caracteres** (string).
- `\n` — Un **carácter de escape** que representa "nueva línea"
  (newline). Hace que el cursor salte a la línea siguiente.
- `;` — El **punto y coma** termina la instrucción. En C, todas
  las instrucciones terminan con `;`.

**Caracteres de escape comunes:**

| Secuencia | Significado              |
|-----------|--------------------------|
| `\n`      | Nueva línea (newline)    |
| `\t`      | Tabulación horizontal    |
| `\\`      | Barra invertida literal  |
| `\"`      | Comilla doble literal    |
| `\'`      | Comilla simple literal   |
| `\0`      | Carácter nulo            |
| `\r`      | Retorno de carro         |
| `\b`      | Retroceso (backspace)    |

```c
#include <stdio.h>

int main(void) {
    printf("Caracteres de escape en accion:\n");
    printf("Columna1\tColumna2\tColumna3\n");
    printf("Valor1\t\tValor2\t\tValor3\n");
    printf("Comilla doble: \" así se escapa\n");
    printf("Barra invertida: \\ así se escapa\n");
    printf("Una línea\nY otra línea\nY una tercera\n");
    return 0;
}
```

**2.5 La instrucción return**

```c
    return 0;
```

`return` finaliza la ejecución de una función y devuelve un valor
a quien la llamó. En el caso de `main`, ese "quien llamó" es el
sistema operativo.

- `return 0;` — Indica que el programa terminó **correctamente**.
- `return 1;` o cualquier otro valor — Indica un **error**.

El sistema operativo puede usar este valor para saber si el
programa funcionó bien. En scripts de shell, puedes verificar
este valor con `$?`:

```bash
./mi_programa
echo $?  # Imprime 0 si todo fue bien
```

```c
#include <stdio.h>

int main(void) {
    printf("El valor de retorno de main es fundamental.\n");
    printf("return 0 = éxito, return 1 = error.\n");
    printf("Los sistemas operativos verifican este valor.\n");
    return 0;  /* Todo salió bien */
}
```

4. ERRORES COMUNES DEL PRINCIPIANTE
===============================================================================

Parte del aprendizaje es cometer errores. Aquí están los más
frecuentes y cómo solucionarlos.

**4.1 Error: punto y coma olvidado**

```c
#include <stdio.h>

int main(void) {
    printf("Hola mundo")  /* ⚠️ ERROR: falta ; */
    return 0;
}
```

**Error del compilador:**
```
hola.c:5:5: error: expected ';' before 'return'
```

```c
#include <stdio.h>

int main(void) {
    printf("Hola mundo");  /* ✅ Punto y coma presente */
    return 0;
}
```

**4.2 Error: comillas mal cerradas**

```c
#include <stdio.h>

int main(void) {
    printf("Hola mundo);  /* ⚠️ ERROR: comillas desbalanceadas */
    return 0;
}
```

**4.3 Error: #include mal escrito**

```c
#include <studio.h>  /* ⚠️ ERROR: stdio, no studio */

int main(void) {
    printf("Hola\n");
    return 0;
}
```

**4.4 Error: main mal declarada**

```c
#include <stdio.h>

void main() {  /* ⚠️ ERROR: debe devolver int */
    printf("Hola\n");
}
```

**4.5 Error: olvidar la llave de cierre**

```c
#include <stdio.h>

int main(void) {
    printf("Hola\n");
    /* ⚠️ ERROR: falta la llave de cierre } */
```

```c
#include <stdio.h>

int main(void) {
    printf("Los errores son normales al aprender.\n");
    printf("Lo importante es leer los mensajes y corregirlos.\n");
    printf("Cada error que resuelves te hace mejor programador.\n");
    return 0;
}
```

5. CÓDIGOS DE SALIDA Y SU SIGNIFICADO
===============================================================================

El valor que devuelve `main` es más importante de lo que parece.

**5.1 Convención de códigos de retorno**

| Código | Significado                |
|--------|----------------------------|
| 0      | Ejecución exitosa          |
| 1      | Error genérico             |
| 2      | Uso incorrecto             |
| 127    | Comando no encontrado      |
| 130    | Terminado por Ctrl+C       |

**5.2 Verificando el código de retorno**

En Linux/macOS:

```bash
./mi_programa
echo "Código de retorno: $?"
```

En Windows (PowerShell):

```powershell
.\mi_programa.exe
echo "Código de retorno: $LASTEXITCODE"
```

**5.3 Usando constantes simbólicas para códigos de retorno**

```c
#include <stdio.h>
#include <stdlib.h>  /* Para EXIT_SUCCESS y EXIT_FAILURE */

int main(void) {
    int resultado;

    printf("¿Cuánto es 2 + 2? ");
    resultado = 4;

    if (resultado == 4) {
        printf("¡Correcto!\n");
        return EXIT_SUCCESS;  /* Equivalente a return 0 */
    } else {
        printf("Incorrecto.\n");
        return EXIT_FAILURE;  /* Equivalente a return 1 */
    }
}
```

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    printf("Usa EXIT_SUCCESS y EXIT_FAILURE de <stdlib.h>.\n");
    printf("Son más legibles que 0 y 1.\n");
    return EXIT_SUCCESS;
}
```

6. BUENAS PRÁCTICAS DESDE EL PRIMER DÍA
===============================================================================

Aunque estás empezando, es crucial adoptar buenos hábitos desde
ahora.

**6.1 Nombres de archivos**

- Usa nombres descriptivos: `calculadora.c`, no `a.c`.
- Solo letras minúsculas, números y guiones bajos.
- Extensión `.c` para código fuente, `.h` para cabeceras.

**6.2 Indentación consistente**

```c
/* INCORRECTO: difícil de leer */
int main(void) {
printf("Hola\n");
    printf("Adiós\n");
  return 0;
}

/* CORRECTO: indentación clara y consistente */
int main(void) {
    printf("Hola\n");
    printf("Adiós\n");
    return 0;
}
```

**6.3 Líneas en blanco para separar secciones**

```c
#include <stdio.h>

int main(void) {
    /* Sección de inicialización */
    int edad = 25;

    /* Sección de procesamiento */
    edad = edad + 1;

    /* Sección de salida */
    printf("Edad: %d\n", edad);

    return 0;
}
```

**6.4 Comentarios útiles, no obvios**

```c
/* MAL: comenta lo obvio */
int x = 5;  /* Asigna 5 a x */

/* BIEN: explica el porqué */
int max_usuarios = 100;  /* Límite del plan gratuito */
```

```c
#include <stdio.h>

int main(void) {
    printf("═══ GUÍA DE ESTILO PARA PRINCIPIANTES ═══\n\n");
    printf("✓ Nombres descriptivos para archivos y variables\n");
    printf("✓ Indentación consistente (4 espacios recomendados)\n");
    printf("✓ Líneas en blanco para separar secciones\n");
    printf("✓ Comentarios que explican el porqué, no el qué\n");
    printf("✓ Punto y coma al final de cada instrucción\n");
    printf("✓ Llaves siempre presentes, incluso en una línea\n");
    printf("✓ return 0 al final de main para indicar éxito\n");
    return 0;
}
```

7. PROGRAMA INTERACTIVO: SALUDO PERSONALIZADO


8. DEPURACIÓN BÁSICA
===============================================================================

**8.1 Leyendo errores del compilador**

Los mensajes de error de GCC siguen este formato:

```
archivo.c:número_línea:número_columna: error: mensaje
    código problemático
    ^~~~~~ señalización del problema
```

Ejemplo:
```c
#include <stdio.h>

int main(void) {
    printf("Hola\n")  /* Falta ; */
    return 0;
}
```

GCC mostrará:
```
hola.c:5:5: error: expected ';' before 'return'
    5 |     return 0;
      |     ^~~~~~
```

**8.2 Usando advertencias (-Wall -Wextra)**

Siempre compila con `-Wall -Wextra`. Estas opciones activan
advertencias que te ayudan a detectar problemas potenciales:

```bash
gcc -Wall -Wextra programa.c -o programa
```

```c
#include <stdio.h>

int main(void) {
    int x;  /* Variable no inicializada */
    printf("x = %d\n", x);  /* ⚠️ Advertencia: x no está inicializada */
    return 0;
}
```

**8.3 Depuración con printf**

Una técnica simple pero efectiva: usa `printf` para mostrar valores
intermedios y entender qué está haciendo tu programa.

```c
#include <stdio.h>

int main(void) {
    int a = 5;
    int b = 3;

    printf("DEBUG: a = %d, b = %d\n", a, b);

    int suma = a + b;
    printf("DEBUG: suma = %d\n", suma);

    printf("%d + %d = %d\n", a, b, suma);
    return 0;
}
```

```c
#include <stdio.h>

int main(void) {
    printf("La depuración con printf es tu mejor amiga.\n");
    printf("Cuando algo no funciona, imprime variables.\n");
    printf("Así descubrirás dónde está el problema.\n");
    return 0;
}
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué se usa `return 0;` y no otro número?**
Por convención, el valor 0 indica que el programa terminó
correctamente. Cualquier valor distinto de cero indica un error
o condición anormal. El sistema operativo puede verificar este
valor para tomar decisiones (por ejemplo, en scripts).

❓ **¿Qué pasa si no pongo `return 0;` al final de main?**
En C99 y posteriores, si `main` no tiene `return`, el compilador
automáticamente retorna 0 al alcanzar el final de la función. Sin
embargo, es buena práctica escribirlo explícitamente para mayor
claridad.

❓ **¿Puedo tener más de una función main?**
No. Un programa solo puede tener una función `main`. Es el punto de
entrada único. Si tienes múltiples archivos, solo uno debe contener
`main`.

❓ **¿Qué diferencia hay entre `printf` y `puts`?**
`puts` imprime una cadena y añade automáticamente un salto de línea
al final. `printf` es más versátil porque permite formatear la salida
con especificadores como `%d`, `%s`, etc.

Diseña un programa en C llamado `cartel.c` que genere un cartel

   - **Nombre del estudiante** (usando una variable que puedas

   - **Fecha actual** (usando el preprocesador `__DATE__`).
   - **Versión del compilador** (usando `__VERSION__`).

4. **Pie de página**: Al final del cartel, muestra el número total
   de líneas que ocupa el cartel.

5. **Múltiples idiomas**: El cartel debe tener una versión en español
   y otra en inglés. Usa una variable `int idioma` donde 0 = español,
   1 = inglés, y muestra el cartel en el idioma seleccionado.

El objetivo es crear una salida profesional y atractiva, demostrando
que desde el primer programa puedes producir resultados visualmente
impresionantes.

