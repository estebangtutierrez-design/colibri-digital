1. EL COMPILADOR GCC
===============================================================================

**1.1 ¿Qué es GCC?**

GCC (GNU Compiler Collection) es un conjunto de compiladores de
código abierto creado por Richard Stallman y la Free Software
Foundation como parte del proyecto GNU. Originalmente llamado
"GNU C Compiler", hoy soporta múltiples lenguajes: C, C++, Objective-C,
Fortran, Ada, Go y más.

GCC es el compilador de C más utilizado en el mundo, especialmente
en sistemas Unix y Linux. Es la herramienta que usaremos durante
todo este curso.

**1.2 ¿Por qué GCC?**

- **Gratuito y de código abierto**: Sin licencias, sin restricciones.
- **Estándar**: Soporta C89, C99, C11, C17 y parcialmente C23.
- **Portable**: Disponible en prácticamente cualquier plataforma.
- **Potente**: Optimizaciones avanzadas que producen código muy rápido.
- **Estandar de facto**: Es el compilador de referencia para Linux y
  la mayoría de sistemas embebidos.

```c
/* Este programa verificará si tu compilador funciona */
#include <stdio.h>

int main(void) {
    printf("Si ves este mensaje, ¡tu compilador funciona!\n");
    printf("Compilador: %s\n", __VERSION__);
    printf("Estándar: %ld\n", __STDC_VERSION__);
    return 0;
}
```

2. INSTALACIÓN DE GCC
===============================================================================

**2.1 En Linux (Ubuntu/Debian)**

En distribuciones basadas en Debian (Ubuntu, Linux Mint, etc.),
abre una terminal y ejecuta:

```bash
sudo apt update
sudo apt install build-essential
```

El paquete `build-essential` incluye GCC, G++ y otras herramientas
esenciales como `make`. Para instalar solo GCC:

```bash
sudo apt install gcc
```

Verifica la instalación:

```bash
gcc --version
```

Deberías ver algo como:
```
gcc (Ubuntu 14.2.1-1) 14.2.1
Copyright (C) 2024 Free Software Foundation, Inc.
...
```

**2.2 En Linux (Fedora/RHEL/CentOS)**

En distribuciones basadas en Red Hat:

```bash
sudo dnf install gcc
# o en versiones antiguas:
sudo yum install gcc
```

**2.3 En Linux (Arch/Manjaro)**

```bash
sudo pacman -S gcc
```

**2.4 En macOS**

Si usas macOS, tienes dos opciones:

**Opción A: Xcode Command Line Tools** (recomendada)

```bash
xcode-select --install
```

Esto instala Clang, que es el compilador de C de Apple (es compatible
con GCC, y usaremos `gcc` como alias). Xcode CLT incluye también
`make`, `git` y otras herramientas.

**Opción B: Homebrew + GCC**

```bash
brew install gcc
```

Luego usa `gcc-14` (o la versión instalada) en lugar de `gcc`.

Verifica:

```bash
gcc --version
# o
clang --version
```

**2.5 En Windows**

En Windows hay varias opciones:

**Opción A: MinGW-w64 (recomendada)**

1. Descarga MSYS2 desde https://www.msys2.org/
2. Sigue las instrucciones de instalación.
3. Abre "MSYS2 UCRT64" y ejecuta:
```bash
pacman -S mingw-w64-ucrt-x86_64-gcc
```

4. Agrega `C:\msys64\ucrt64\bin` a tu PATH del sistema.
5. Abre una nueva terminal y verifica:
```bash
gcc --version
```

**Opción B: WSL (Windows Subsystem for Linux)**

1. Instala WSL desde Microsoft Store o con:
```powershell
wsl --install
```

2. Abre la terminal de Ubuntu en WSL y sigue los pasos de Linux.

**Opción C: Cygwin**

Descarga e instala Cygwin desde https://www.cygwin.com/, seleccionando
el paquete `gcc-core` durante la instalación.

```c

    printf("═══ VERIFICACIÓN DEL ENTORNO ═══\n\n");
    printf("Sistema operativo: ");

#ifdef __linux__
    printf("Linux\n");
#elif defined(_WIN32) || defined(_WIN64)
    printf("Windows\n");
#elif defined(__APPLE__) && defined(__MACH__)

    printf("MSVC\n");
#else
    printf("Desconocido\n");
#endif

    return 0;
}
```

3. COMPILACIÓN DESDE LA TERMINAL
===============================================================================

**3.1 El proceso de compilación**

Compilar un programa en C consta de cuatro etapas:

1. **Preprocesamiento**: El preprocesador maneja las directivas
   `#include`, `#define`, `#if`, etc. Genera un archivo `.i`.

2. **Compilación**: El compilador traduce el código C preprocesado
   a lenguaje ensamblador. Genera un archivo `.s`.

3. **Ensamblado**: El ensamblador convierte el código ensamblador
   a código máquina (objeto). Genera un archivo `.o`.

4. **Enlazado**: El enlazador combina los archivos objeto con las
   bibliotecas para producir el ejecutable final.

```
  fuente.c ──→[preprocesador]──→ fuente.i ──→[compilador]──→ fuente.s
                                                                  │
                                                             [ensamblador]
                                                                  │
                                                               fuente.o
                                                                  │
                                                             [enlazador]
                                                                  │
                                                              ejecutable
```

**3.2 Compilación básica: un solo archivo**

Crea un archivo llamado `hola.c`:

```c
#include <stdio.h>

int main(void) {
    printf("¡Hola, mundo desde C!\n");
    return 0;
}
```

Compílalo:

```bash
gcc hola.c
```

Esto produce un ejecutable llamado `a.out` (Linux/macOS) o `a.exe`
(Windows). Ejecútalo:

```bash
./a.out
```

**3.3 Compilación con nombre de salida personalizado**

```bash
gcc hola.c -o hola
```

Esto genera un ejecutable llamado `hola` (o `hola.exe` en Windows).

```bash
./hola
```

**3.4 Opciones comunes de compilación**

| Opción         | Descripción                                          |
|---------------|------------------------------------------------------|
| `-o <archivo>`| Especifica el nombre del archivo de salida           |
| `-Wall`       | Activa todas las advertencias comunes                |
| `-Wextra`     | Activa advertencias adicionales                      |
| `-Werror`     | Trata las advertencias como errores                  |
| `-std=c11`    | Usa el estándar C11                                  |
| `-O2`         | Optimización nivel 2 (rendimiento)                   |
| `-g`          | Incluye información de depuración (GDB)              |
| `-v`          | Modo verbose (muestra el proceso de compilación)     |

Ejemplo completo:

```bash
gcc -Wall -Wextra -std=c11 -O2 -g programa.c -o programa
```

```c
/* Programa para probar diferentes opciones de compilación */
#include <stdio.h>

int main(void) {
    printf("Compilado con:\n");
    printf("  - Wall: %s\n", "activado");
    printf("  - Wextra: %s\n", "activado");
    printf("  - std: c11\n");
    printf("  - O2: optimización activada\n");
    printf("  - g: símbolos de depuración incluidos\n");
    return 0;
}
```

4. COMPILACIÓN MULTIARCHIVO
===============================================================================

En proyectos reales, el código se divide en múltiples archivos.
Veamos cómo compilar proyectos de varios archivos.

**4.1 Estructura típica**

```
proyecto/
├── main.c      (contiene main)
├── funciones.c (implementaciones)
└── funciones.h (declaraciones)
```

**4.2 Archivo de cabecera (`funciones.h`)**

```c
#ifndef FUNCIONES_H
#define FUNCIONES_H

int suma(int a, int b);
int resta(int a, int b);
void imprimir_resultado(int resultado);

#endif
```

**4.3 Archivo de implementación (`funciones.c`)**

```c
#include <stdio.h>
#include "funciones.h"

int suma(int a, int b) {
    return a + b;
}

int resta(int a, int b) {
    return a - b;
}

void imprimir_resultado(int resultado) {
    printf("Resultado: %d\n", resultado);
}
```

**4.4 Archivo principal (`main.c`)**

```c
#include <stdio.h>
#include "funciones.h"

int main(void) {
    int a = 10, b = 5;

    imprimir_resultado(suma(a, b));
    imprimir_resultado(resta(a, b));

    return 0;
}
```

**4.5 Compilación multiarchivo**

Método 1: Compilar todo junto:
```bash
gcc main.c funciones.c -o programa
```

Método 2: Compilar por separado (recomendado para proyectos grandes):
```bash
gcc -c main.c -o main.o
gcc -c funciones.c -o funciones.o
gcc main.o funciones.o -o programa
```

La compilación separada permite que, al modificar un archivo, solo
se recompile ese archivo y se vuelvan a enlazar los objetos.

```c
#include <stdio.h>

int main(void) {
    printf("La compilación separada ahorra tiempo en proyectos grandes.\n");
    printf("Solo se recompila el archivo modificado.\n");
    printf("Luego se enlazan todos los objetos.\n");
    return 0;
}
```

5. VISUAL STUDIO CODE
===============================================================================

**5.1 ¿Por qué VS Code?**

Visual Studio Code es un editor de código fuente ligero pero
extremadamente potente, desarrollado por Microsoft. Es la elección
perfecta para aprender C por varias razones:

- **Gratuito y open source**.
- **Multiplataforma**: Windows, macOS, Linux.
- **Extensible**: Miles de extensiones para cualquier necesidad.
- **Integración con terminal**: Compila y ejecuta sin salir del editor.
- **IntelliSense**: Autocompletado inteligente para C.
- **Depuración integrada**: Puedes ejecutar paso a paso tu código.

**5.2 Instalación de VS Code**

**Linux (Ubuntu/Debian)**:

```bash
# Descargar e instalar el paquete .deb
wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -D -o root -g root -m 644 packages.microsoft.gpg /etc/apt/keyrings/packages.microsoft.gpg
sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
rm -f packages.microsoft.gpg
sudo apt update
sudo apt install code
```

O instala desde la interfaz gráfica descargando el `.deb` desde
https://code.visualstudio.com/.

**Linux (Fedora)**:

```bash
sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'
sudo dnf install code
```

**macOS**:

Descarga el archivo `.zip` desde https://code.visualstudio.com/ y
arrastra VS Code a la carpeta de Aplicaciones.

O con Homebrew:
```bash
brew install --cask visual-studio-code
```

**Windows**:

Descarga el instalador desde https://code.visualstudio.com/ y ejecútalo.
Marca la opción "Agregar a PATH" durante la instalación.

```c
#include <stdio.h>

    printf("VS Code + C = La combinación perfecta para aprender.\n");
    printf("Ligero, rápido y con todas las herramientas necesarias.\n");
    return 0;
}
```

6. EXTENSIONES ESENCIALES PARA C
===============================================================================

Una vez instalado VS Code, necesitas instalar extensiones para
convertirlo en un entorno de desarrollo de C completo.

**6.1 Extensiones obligatorias**

1. **C/C++ (ms-vscode.cpptools)**: Proporciona IntelliSense,
   depuración y navegación de código. Es la extensión oficial de
   Microsoft para C y C++.

2. **C/C++ Extension Pack (ms-vscode.cpptools-extension-pack)**:
   Incluye C/C++, CMake Tools y otras herramientas útiles.

3. **Code Runner (formulahendry.code-runner)**: Permite ejecutar
   código con un clic, mostrando la salida en la consola integrada.

**6.2 Extensiones recomendadas**

4. **Prettier (esbenp.prettier-vscode)**: Formateador de código.
5. **GitLens (eamodio.gitlens)**: Herramientas avanzadas de Git.
6. **Material Icon Theme (pkief.material-icon-theme)**: Iconos
   bonitos para los archivos.
7. **Bracket Pair Colorizer 2 (coenraads.bracket-pair-colorizer-2)**:
   Colorea pares de paréntesis/llaves para facilitar la lectura.
8. **Indent Rainbow (oderwat.indent-rainbow)**: Colorea la
   indentación para mejor legibilidad.

**6.3 Cómo instalar extensiones**

1. Abre VS Code.
2. Haz clic en el icono de extensiones (cuadrado en la barra lateral)
   o presiona `Ctrl+Shift+X`.
3. Busca el nombre de la extensión.
4. Haz clic en "Instalar".

```c
#include <stdio.h>

int main(void) {
    printf("Las extensiones convierten VS Code en un IDE completo.\n");
    printf("IntelliSense, debugging, formato automático...\n");

    return 0;
}
```

7. CONFIGURACIÓN DE VS CODE PARA C
===============================================================================

**7.1 Crear un archivo de configuración de compilación**

Crea un archivo llamado `.vscode/tasks.json` en la raíz de tu
proyecto para configurar la compilación con un atajo de teclado:

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Compilar C",
            "type": "shell",
            "command": "gcc",
            "args": [
                "-Wall",
                "-Wextra",
                "-std=c11",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "problemMatcher": ["$gcc"]
        }
    ]
}
```

**7.2 Configuración de depuración**

Crea `.vscode/launch.json` para depurar con GDB:

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Depurar C",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "setupCommands": [
                {
                    "description": "Activar formato bonito para GDB",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ],
            "preLaunchTask": "Compilar C"
        }
    ]
}
```

**7.3 Configuración del editor**

Crea `.vscode/settings.json` para personalizar el editor:

```json
{
    "editor.fontSize": 14,
    "editor.tabSize": 4,
    "editor.insertSpaces": true,
    "editor.renderWhitespace": "all",
    "editor.minimap.enabled": true,
    "files.autoSave": "onFocusChange",
    "C_Cpp.default.compilerPath": "/usr/bin/gcc",
    "C_Cpp.default.cStandard": "c11",
    "code-runner.runInTerminal": true,
    "code-runner.executorMap": {
        "c": "cd $dir && gcc -Wall -Wextra -std=c11 $fileName -o $fileNameWithoutExt && ./$fileNameWithoutExt"
    }
}
```

9. HERRAMIENTAS ADICIONALES
===============================================================================

Además de GCC y VS Code, hay herramientas que todo programador de C
debe conocer:

**9.1 GDB (GNU Debugger)**

El depurador estándar para C. Permite ejecutar paso a paso, inspeccionar
variables, establecer puntos de interrupción y analizar el estado del
programa en cualquier momento.

```bash
gcc -g programa.c -o programa  # compilar con símbolos de depuración
gdb ./programa                  # iniciar depurador
```

Comandos básicos de GDB:
- `break main` — Punto de interrupción en main.
- `run` — Ejecutar el programa.
- `next` — Siguiente línea (sin entrar en funciones).
- `step` — Siguiente línea (entrando en funciones).
- `print variable` — Mostrar valor de una variable.
- `quit` — Salir de GDB.

**9.2 Valgrind**

Detector de fugas de memoria y errores de memoria:

```bash
valgrind --leak-check=full ./programa
```

Valgrind te dirá si:
- Olvidaste liberar memoria.
- Accedes a memoria ya liberada.
- Escribes fuera de los límites de un array.

**9.3 Make**

Herramienta de automatización de compilación para proyectos grandes:

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -std=c11 -g
TARGET = programa
OBJS = main.o funciones.o

$(TARGET): $(OBJS)
	$(CC) $(OBJS) -o $(TARGET)

main.o: main.c funciones.h
	$(CC) $(CFLAGS) -c main.c -o main.o

funciones.o: funciones.c funciones.h
	$(CC) $(CFLAGS) -c funciones.c -o funciones.o

clean:
	rm -f $(OBJS) $(TARGET)

run: $(TARGET)
	./$(TARGET)
```

**9.4 Formateadores y analizadores**

- **indent**: Formatea automáticamente el código C.
- **cppcheck**: Analizador estático que detecta bugs potenciales.
- **clang-tidy**: Analizador estático basado en Clang.

```c
#include <stdio.h>

int main(void) {
    printf("Un buen programador de C conoce sus herramientas:\n");
    printf("  GCC   → Compila\n");
    printf("  GDB   → Depura\n");
    printf("  VG    → Valida memoria\n");
    printf("  Make  → Automatiza\n");
    printf("  Cppch → Analiza\n");
    return 0;
}
```

===============================================================================
10. SOLUCIÓN DE PROBLEMAS COMUNES
===============================================================================

**10.1 "gcc: command not found"**

El compilador no está instalado o no está en el PATH.

- Linux: instala `build-essential` o `gcc`.
- macOS: ejecuta `xcode-select --install`.
- Windows: verifica que MinGW esté en el PATH.

**10.2 Error de compilación: undefined reference to 'main'**

El enlazador no encuentra la función `main`. Asegúrate de que tu
programa tenga `int main(void)` o `int main(int argc, char *argv[])`.

**10.3 Error: 'some_function' was not declared in this scope**

Olvidaste incluir la cabecera correspondiente o declarar el prototipo
de la función.

**10.4 VS Code no encuentra el compilador**

Configura la ruta del compilador en `settings.json`:

```json
"C_Cpp.default.compilerPath": "/usr/bin/gcc"
```

**10.5 Code Runner no ejecuta el programa**

Asegúrate de que `code-runner.runInTerminal` esté en `true` en la
configuración.

```c
#include <stdio.h>

int main(void) {
    printf("Los errores son parte del aprendizaje.\n");
    printf("Cada error que resuelves te hace más fuerte.\n");
    printf("Lee los mensajes, entiéndelos, resuélvelos.\n");
    return 0;
}
```

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo usar otro compilador?**
Sí, hay varias opciones: Clang (más moderno, errores más claros),
MSVC (solo Windows, en Visual Studio) y TCC (muy rápido pero
limitado). En este curso usaremos GCC por ser el estándar universal.

❓ **¿Puedo usar otro editor?**
¡Por supuesto! Los programadores de C usan Vim, Emacs, CLion,
Sublime Text, etc. VS Code es el que recomiendo por su equilibrio
entre simplicidad y potencia, pero la herramienta es secundaria;
lo importante es el conocimiento.

❓ **No tengo permisos de administrador para instalar GCC. ¿Qué hago?**
Usa un compilador online como Replit, OnlineGDB o Compiler Explorer
(https://godbolt.org/). También puedes usar WSL en Windows sin
necesidad de permisos administrativos completos.

❓ **¿Qué hago si la compilación tarda mucho?**
En proyectos pequeños (como los de este curso), la compilación es
instantánea. Si notas lentitud, verifica que tu antivirus no esté
analizando los archivos objeto, o usa la compilación separada con
archivos `.o`.

DESAFÍO DEL ARQUITECTO
===============================================================================

🏗️ **Arquitecto del Entorno: El Script de Automatización**

Diseña un script (en Bash para Linux/macOS o en PowerShell para
Windows) que automatice la creación de un proyecto de C completo.

El script debe:

1. Crear la estructura de directorios:
```
nombre_proyecto/
├── src/           (código fuente .c)
├── include/       (cabeceras .h)
├── build/         (archivos .o y ejecutable)
└── Makefile       (archivo de compilación)
```

2. Generar un `main.c` básico en `src/` que imprima "Proyecto C
   inicializado correctamente".

3. Generar un archivo `Makefile` que compile todo el proyecto con
   las opciones: `-Wall -Wextra -std=c11 -g`.

4. Compilar el proyecto automáticamente al finalizar.

5. El script debe aceptar un argumento: el nombre del proyecto.
   Ejemplo: `./crear_proyecto.sh mi_juego`

6. Verificar que GCC esté instalado antes de comenzar.

El reto no es solo escribir el script, sino hacerlo robusto:
- Verificar que el directorio no exista antes de crearlo.
- Mostrar mensajes informativos de cada paso.
- Manejar errores (si no hay GCC, si no se puede compilar, etc.).
- Ser multiplataforma (al menos Linux/macOS con Bash).

Este desafío te preparará para trabajar en proyectos reales de C,
donde la organización y automatización son clave.
