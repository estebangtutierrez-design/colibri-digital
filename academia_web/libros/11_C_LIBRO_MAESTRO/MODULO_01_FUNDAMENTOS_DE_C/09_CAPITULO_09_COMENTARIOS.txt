1. TIPOS DE COMENTARIOS EN C
===============================================================================

C tiene dos formas de incluir comentarios:

**1.1 Comentarios de una linea (//)**

Introducidos en C99. Comienzan con // y terminan al final de la linea.

```c
// Esto es un comentario de una linea
int x = 10;  // Comentario al final de una linea
```

**1.2 Comentarios multilinea (/* */)**

Disponibles desde K&R C (original). Todo lo que esta entre /* y */
es ignorado por el compilador.

```c
/* Esto es un comentario
   multilinea que abarca
   varias lineas de texto */
int y = 20;
```

```c
#include <stdio.h>

int main(void) {
    // Comentario de una linea: recomendado para notas breves
    printf("Hola, ");

    /* Comentario multilinea:
       util para explicaciones
       mas detalladas */
    printf("Mundo!
");

    return 0;
}
```

**1.3 Comentarios anidados**

Los comentarios /* */ NO se pueden anidar en C estandar.

```c
/* Esto es un comentario /* con intento de anidar */ ERROR */
```

El compilador ve el primer */ como el cierre del comentario,
dejando "ERROR */" como codigo, lo que causa error de compilacion.

```c
#include <stdio.h>

int main(void) {
    /* Usa // dentro de /* */ si necesitas "anidar" */
    // printf("Comentado temporalmente
");

    printf("Los comentarios anidados no son validos en C clasico.
");
    printf("Usa // si necesitas comentar un bloque con /* */ dentro.
");

    return 0;
}
```

2. BUENOS COMENTARIOS VS MALOS COMENTARIOS
===============================================================================

**2.1 Malos comentarios**

```c
/* MAL: comentario obvio que no anade informacion */
int a = 0;  // Asigna 0 a la variable a

/* MAL: comentario desactualizado */
int calcular_total(int precio, int impuesto) {
    // Antes: precio * impuesto
    return precio + impuesto;  // Ahora suma
}

/* MAL: comentario ruidoso */
// ********************************************
// *         INICIO DE LA FUNCION MAIN        *
// ********************************************
int main(void) {  // Funcion principal del programa
```

**2.2 Buenos comentarios**

```c
/* BIEN: explica el PORQUE, no el QUE */
int buffer[TAM_MAX];  // TAM_MAX debe ser multiplo de 4 para alineacion

/* BIEN: documenta una decision de diseno */
// Usamos busqueda lineal en lugar de binaria porque el array
// casi siempre tiene menos de 10 elementos (datos de produccion)
int indice = buscar_lineal(datos, objetivo);

/* BIEN: advierte sobre una trampa */
// ATENCION: esta funcion NO es reentrante. Usa una variable global.
void procesar_evento(void) {
    static int contador = 0;
    contador++;
}
```

```c
#include <stdio.h>
#include <math.h>

int main(void) {
    printf("=== BUENOS VS MALOS COMENTARIOS ===

");

    printf("MAL comentario:
");
    printf("  // Suma 1 a i  (obvio, no anade valor)

");

    printf("BUEN comentario:
");
    printf("  // Incrementa i en 1 para avanzar al siguiente
");
    printf("  // caracter. El bucle termina al encontrar '\0'.

");

    printf("Regla de oro:
");
    printf("  Comenta el POR QUE, no el QUE.
");
    printf("  El codigo ya dice lo que hace;
");
    printf("  los comentarios deben decir por que lo hace.
");

    return 0;
}
```

3. COMENTARIOS DE DOCUMENTACION (DOXYGEN)
===============================================================================

Doxygen es una herramienta que genera documentacion automatica a
partir de comentarios estructurados en el codigo.

**3.1 Formato basico de Doxygen**

```c
/**
 * @brief Breve descripcion de la funcion.
 *
 * Descripcion detallada de lo que hace la funcion,
 * sus algoritmos, casos especiales, etc.
 *
 * @param parametro1 Descripcion del primer parametro.
 * @param parametro2 Descripcion del segundo parametro.
 * @return Descripcion del valor de retorno.
 */
```

**3.2 Comandos Doxygen comunes**

| Comando       | Descripcion                          |
|---------------|--------------------------------------|
| @brief        | Descripcion breve                    |
| @param        | Documenta un parametro               |
| @return       | Documenta el valor de retorno        |
| @see          | Referencia a otra funcion            |
| @note         | Nota importante                      |
| @warning      | Advertencia                          |
| @deprecated   | Marca como obsoleto                  |
| @author       | Autor del codigo                     |
| @version      | Version                              |
| @date         | Fecha de creacion/modificacion       |

```c
#include <stdio.h>

/**
 * @brief Calcula el factorial de un numero entero no negativo.
 *
 * El factorial de n (n!) se define como n * (n-1) * (n-2) * ... * 1.
 * Para n = 0, el factorial es 1 por definicion.
 *
 * @param n Numero entero no negativo (n >= 0).
 * @return El factorial de n como unsigned long long.
 *         Devuelve 1 si n = 0.
 *
 * @warning El resultado puede desbordarse para n > 20.
 * @see combinaciones()
 */
unsigned long long factorial(int n) {
    unsigned long long resultado = 1;

    for (int i = 2; i <= n; i++) {
        resultado *= i;
    }

    return resultado;
}

/**
 * @brief Verifica si un numero es primo.
 *
 * Un numero primo es aquel que solo es divisible
 * por 1 y por si mismo.
 *
 * @param n Numero entero positivo a verificar.
 * @return 1 si es primo, 0 si no lo es.
 *
 * @note Esta implementacion es basica; para numeros
 *       grandes considera la Criba de Eratostenes.
 */
int es_primo(int n) {
    if (n < 2) return 0;
    if (n == 2) return 1;
    if (n % 2 == 0) return 0;

    for (int i = 3; i * i <= n; i += 2) {
        if (n % i == 0) return 0;
    }

    return 1;
}

int main(void) {
    printf("=== DOCUMENTACION CON DOXYGEN ===

");

    printf("Factorial de 10: %llu
", factorial(10));
    printf("?7 es primo? %s
", es_primo(7) ? "Si" : "No");
    printf("?10 es primo? %s
", es_primo(10) ? "Si" : "No");

    printf("
Ejecuta: doxygen -g
");
    printf("Para generar la documentacion automaticamente.
");

    return 0;
}
```

5. COMENTARIOS DE DEPURACION TEMPORAL
===============================================================================

Los comentarios son utiles durante el desarrollo para desactivar
codigo temporalmente.

**5.1 Comentar codigo para depuracion**

```c
#include <stdio.h>

int main(void) {
    int a = 10, b = 0;

    // Desactivado temporalmente para pruebas
    // printf("DEBUG: a = %d, b = %d
", a, b);

    // if (b == 0) {
    //     printf("Error: division por cero
");
    //     return 1;
    // }

    int resultado = a / b;  // Esto fallara si b = 0
    printf("Resultado: %d
", resultado);

    return 0;
}
```

**5.2 Directivas de depuracion con #ifdef**

Mejor que comentar manualmente es usar compilacion condicional:

```c
#include <stdio.h>

#define DEBUG  // Activar modo depuracion

int main(void) {
    int a = 10, b = 5;

#ifdef DEBUG
    printf("[DEBUG] a = %d, b = %d
", a, b);
#endif

    int suma = a + b;

#ifdef DEBUG
    printf("[DEBUG] suma = %d
", suma);
#endif

    printf("%d + %d = %d
", a, b, suma);

    return 0;
}
```

**5.3 Comentarios TODO y FIXME**

Marca el codigo que necesita atencion futura:

```c
// TODO: Implementar validacion de entrada de usuario
// FIXME: Esta funcion falla con valores negativos
// HACK: Solucion temporal mientras se implementa el modulo de login
// XXX: Optimizar este bucle para grandes volumenes de datos
```

```c
#include <stdio.h>

int main(void) {
    printf("=== MARCAS DE CODIGO ===

");

    printf("TODO  - Tarea pendiente de implementar
");
    printf("FIXME - Error conocido que debe corregirse
");
    printf("HACK  - Solucion temporal (volver a revisar)
");
    printf("XXX   - Alerta sobre algo problematico
");
    printf("NOTE  - Nota importante
");

    printf("
Usa estas marcas consistentemente:
");
    printf("  // TODO: Validar entrada del usuario
");
    printf("  // FIXME: Corregir desbordamiento con n>100
");

    return 0;
}
```

6. CODIGO AUTO-DOCUMENTADO
===============================================================================

El mejor comentario es el que no necesitas escribir porque el
codigo se explica por si mismo.

**6.1 Nombres descriptivos**

```c
/* MAL: nombres que obligan a comentar */
int c(int a, int b) { return a * b; }

/* BIEN: el nombre lo dice todo */
int calcular_area_rectangulo(int ancho, int alto) {
    return ancho * alto;
}
```

**6.2 Funciones pequenas con proposito claro**

```c
/* MAL: funcion grande que hace de todo */
void procesar(void);

/* BIEN: funciones con responsabilidad unica */
int leer_edad_usuario(void);
int verificar_edad_valida(int edad);
void mostrar_mensaje_edad(int edad);
```

**6.3 Constantes con nombre vs numeros magicos**

```c
/* MAL: numero magico */
if (edad > 65) { ... }

/* BIEN: constante con nombre */
#define EDAD_JUBILACION 65
if (edad > EDAD_JUBILACION) { ... }
```

```c
#include <stdio.h>

#define TEMPERATURA_CONCION_AGUA 100
#define TEMPERATURA_CONCIONAGUA 100

int main(void) {
    printf("=== CODIGO AUTO-DOCUMENTADO ===

");

    printf("Principios:

");
    printf("1. Nombres descriptivos de variables y funciones
");
    printf("   int calcular_edad(int anio_nacimiento);

");
    printf("2. Funciones pequenas con una sola responsabilidad
");
    printf("   - leer_datos()
");
    printf("   - validar_datos()
");
    printf("   - procesar_datos()
");
    printf("   - mostrar_resultados()

");
    printf("3. Constantes con nombre en lugar de numeros magicos

");
    printf("4. Estructuras de control claras
");
    printf("   - Evita condiciones demasiado complejas
");
    printf("   - Usa funciones auxiliares para claridad
");

    return 0;
}
```

7. CUANDO NO COMENTAR
===============================================================================

Tan importante como saber comentar es saber **cuando no hacerlo**.

**7.1 No comentes el codigo obvio**

```c
// MAL: estos comentarios no anaden informacion
int x = 5;           // Asigna 5 a x
x++;                 // Incrementa x en 1
printf("%d", x);     // Imprime x
```

**7.2 No comentes codigo mal escrito; reescribelo**

En lugar de comentar codigo confuso, hazlo mas claro:

```c
/* MAL: codigo confuso + comentario */
int f(int* a, int n) {  // Ordena el array
    int t, i, j;
    for (i = 0; i < n-1; i++)
        for (j = 0; j < n-i-1; j++)
            if (a[j] > a[j+1]) { t = a[j]; a[j] = a[j+1]; a[j+1] = t; }
}

/* BIEN: codigo claro sin necesidad de comentarios */
void ordenar_burbuja(int array[], int longitud) {
    for (int i = 0; i < longitud - 1; i++) {
        for (int j = 0; j < longitud - i - 1; j++) {
            if (array[j] > array[j + 1]) {
                int temporal = array[j];
                array[j] = array[j + 1];
                array[j + 1] = temporal;
            }
        }
    }
}
```

**7.3 No mantengas comentarios desactualizados**

Un comentario falso es peor que ningun comentario. Si cambias
el codigo, actualiza los comentarios.

```c
/* MAL: comentario que ya no es cierto (antes era precio * impuesto) */
return precio + impuesto;  // Calcula precio * impuesto
```

```c
#include <stdio.h>

int main(void) {
    printf("=== CUANDO NO COMENTAR ===

");

    printf("NO comentar:
");
    printf("  - Lo que el codigo ya dice claramente
");
    printf("  - Codigo mal escrito (reescribelo)
");
    printf("  - Informacion desactualizada

");

    printf("SI comentar:
");
    printf("  - El POR QUE de una decision de diseno
");
    printf("  - Algoritmos complejos o no obvios
");
    printf("  - Limitaciones y advertencias
");
    printf("  - API y parametros de funciones
");
    printf("  - TODO y FIXME para trabajo futuro
");

    return 0;
}
```

9. EJEMPLO COMPLETO: ARCHIVO BIEN COMENTADO
===============================================================================

```c
/**
 * @file    gestor_estudiantes.c
 * @brief   Sistema de gestion de estudiantes.
 * @author  Lucas EGA Academy
 * @date    2026-07-02
 * @version 1.0.0
 *
 * Proporciona funciones para administrar estudiantes,
 * incluyendo registro, calificaciones y reportes.
 *
 * @see gestor_estudiantes.h
 */

#include <stdio.h>
#include <string.h>

#define MAX_ESTUDIANTES 1000  // Limite del plan actual
#define MAX_NOMBRE 100        // Longitud maxima del nombre

/**
 * @brief Estructura que representa un estudiante.
 */
typedef struct {
    int id;                    // Identificador unico
    char nombre[MAX_NOMBRE];  // Nombre completo
    int edad;                  // Edad en anios
    float promedio;            // Promedio general (0.0 - 10.0)
    int activo;                // 1 = activo, 0 = inactivo
} Estudiante;

/**
 * @brief Busca un estudiante por su ID.
 *
 * Recorre el array de estudiantes y devuelve el indice
 * del que coincide con el ID buscado.
 *
 * @param estudiantes Array de estudiantes.
 * @param num_estudiantes Numero total de estudiantes.
 * @param id_buscado ID del estudiante a buscar.
 * @return Indice del estudiante, o -1 si no se encuentra.
 *
 * @warning Los IDs se asumen unicos.
 */
int buscar_por_id(Estudiante estudiantes[], int num_estudiantes, int id_buscado) {
    for (int i = 0; i < num_estudiantes; i++) {
        if (estudiantes[i].id == id_buscado) {
            return i;  // Encontrado
        }
    }
    return -1;  // No encontrado
}

/**
 * @brief Calcula el promedio general de todos los estudiantes activos.
 *
 * @param estudiantes Array de estudiantes.
 * @param num_estudiantes Numero total de estudiantes.
 * @return Promedio general de los estudiantes activos.
 *         Devuelve 0.0 si no hay estudiantes activos.
 */
float calcular_promedio_general(Estudiante estudiantes[], int num_estudiantes) {
    float suma = 0.0;
    int count = 0;

    for (int i = 0; i < num_estudiantes; i++) {
        if (estudiantes[i].activo) {
            suma += estudiantes[i].promedio;
            count++;
        }
    }

    // Evitar division por cero
    return (count > 0) ? (suma / count) : 0.0;
}

int main(void) {
    printf("=== EJEMPLO DE ARCHIVO BIEN COMENTADO ===

");

    printf("Este archivo muestra las mejores practicas de
");
    printf("documentacion profesional en C:

");
    printf("1. Cabecera de archivo con informacion completa
");
    printf("2. Documentacion de funciones con Doxygen
");
    printf("3. Comentarios de linea para decisiones de diseno
");
    printf("4. Documentacion de parametros y valores de retorno
");
    printf("5. Advertencias sobre casos especiales
");

    return 0;
}
```

RESUMEN DEL CAPITULO
===============================================================================

- C tiene dos tipos de comentarios: // (una linea, C99) y /* */ (multilinea).
- Los comentarios /* */ NO se pueden anidar en C estandar.
- Comenta el POR QUE, no el QUE. El codigo ya dice lo que hace.
- Doxygen permite generar documentacion automatica desde comentarios
  estructurados (@brief, @param, @return, @see, etc.).
- Usa TODO, FIXME, HACK, XXX para marcar trabajo pendiente.
- El codigo auto-documentado (nombres claros, funciones pequenas)
  reduce la necesidad de comentarios.
- No comentes codigo obvio ni mantengas comentarios desactualizados.
- Prefiere reescribir codigo confuso en lugar de explicarlo con
  comentarios.
- La consistencia en el estilo de comentarios es importante en
  proyectos colaborativos.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

?** ?Debo comentar cada linea de codigo?**
No. Comentar cada linea es contraproducente: anade ruido visual y
hace mas dificil mantener los comentarios sincronizados con el
codigo. Comenta solo cuando el codigo no sea obvio por si mismo.

?** ?Que hago si cambio el codigo pero no los comentarios?**
Actualiza los comentarios inmediatamente. Un comentario desactualizado
es peor que ningun comentario porque activamente engana al lector.
Si no puedes mantener los comentarios, no los escribas.

?** ?Cual es el mejor estilo de comentarios?**
El que sea consistente en tu proyecto. Para codigo nuevo, recomiendo:
- // para comentarios breves y notas al final de linea.
- /* */ para bloques de documentacion Doxygen.
- Sigue el estilo existente si trabajas en un proyecto consolidado.

?** ?Debo usar Doxygen en todos mis proyectos?**
Doxygen es excelente para bibliotecas y APIs publicas. Para proyectos
pequenos o personales, los comentarios simples son suficientes. La
regla es: documenta para tu audiencia. Si otros usaran tu codigo,
usa Doxygen.

**Arquitecto de Documentacion: El Generador de Reportes**

Disena un programa en C llamado `gestor_biblioteca.c` que implemente
un sistema simple de gestion de biblioteca, pero el verdadero reto
es la **documentacion del codigo**.

Requisitos del programa:

2. Funciones para: agregar libro, buscar por titulo, prestar libro,
   devolver libro, listar todos los libros.
3. Un menu interactivo para que el usuario use el sistema.

Requisitos de DOCUMENTACION:
