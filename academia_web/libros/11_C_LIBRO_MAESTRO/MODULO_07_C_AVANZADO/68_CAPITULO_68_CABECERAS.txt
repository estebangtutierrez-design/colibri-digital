===============================================================================
1. DECLARACIÓN VS. DEFINICIÓN
===============================================================================

Para entender los archivos de cabecera, primero debemos comprender la
diferencia fundamental entre declaración y definición.

1.1 Declaración

Una declaración le dice al compilador que algo existe, sin reservar memoria.
Puede declararse múltiples veces.

```c
// Declaraciones de variables (no reservan memoria)
extern int contador_global;
extern const double PI;

// Declaraciones de funciones (prototipos)
int suma(int a, int b);
void procesar_datos(const char *nombre, int longitud);

// Declaraciones de tipos (pueden repetirse si son identicas)
struct Punto;
typedef struct Punto Punto_t;

// Declaraciones de struct (incompleto / forward declaration)
struct ArbolBinario;
```

1.2 Definición

Una definición crea la entidad y reserva memoria (o genera código). Solo
puede definirse una vez (regla de una definición - ODR).

```c
// Definiciones de variables
int contador_global = 0;
const double PI = 3.1415926535;

// Definiciones de funciones
int suma(int a, int b) {
    return a + b;
}

// Definiciones de struct (completo)
struct Punto {
    int x;
    int y;
};

// Definición de typedef
typedef struct Punto Punto_t;
```

1.3 ¿Qué va en el .h y qué en el .c?

```c
// archivo: modulo.h
#ifndef MODULO_H
#define MODULO_H

// DECLARACIONES: van en el .h
extern int variable_compartida;
void funcion_publica(int x);
int obtener_valor(void);

// Tipos: van en el .h si son necesarios fuera
typedef struct {
    int id;
    char nombre[64];
} Registro;

#endif
```

```c
// archivo: modulo.c
#include "modulo.h"

// DEFINICIONES: van en el .c
int variable_compartida = 42;

void funcion_publica(int x) {
    variable_compartida = x;
}

int obtener_valor(void) {
    return variable_compartida;
}

// Funciones privadas: solo en .c, con static
static void funcion_privada(void) {
    // Solo visible en este archivo
}
```

===============================================================================
2. INCLUDE GUARDS
===============================================================================

Los include guards evitan que un archivo de cabecera se incluya múltiples
veces, lo que causaría errores de redefinición.

2.1 Estilo tradicional (#ifndef)

```c
#ifndef NOMBRE_ARCHIVO_H
#define NOMBRE_ARCHIVO_H

// ... contenido del header ...

#endif /* NOMBRE_ARCHIVO_H */
```

2.2 Estilo #pragma once

```c
#pragma once

// ... contenido del header ...
// MAS SIMPLE, pero no es C estándar (aunque todos los compiladores
// modernos lo soportan: GCC, Clang, MSVC, ICC, etc.)
```

2.3 Comparación

```c
// VENTAJAS de #ifndef:
// - Es C estándar (ISO C)
// - Funciona en todos los compiladores
// - Permite control adicional (ej: incluir condicional)

// VENTAJAS de #pragma once:
// - Menos código, menos errores de copia/pega
// - No hay riesgo de nombres duplicados
// - Compilación potencialmente más rápida

// RECOMENDACION: Use #pragma once para proyectos nuevos,
// #ifndef si necesita portabilidad máxima a compiladores exoticos

// Ejemplo de error con #ifndef:
// Si dos headers usan el mismo nombre de guarda accidentalmente:
// #ifndef CONFIG_H
// #define CONFIG_H  // Si OTRO header usa CONFIG_H, uno sera excluido!

// Por convencion, el nombre del guarda incluye la ruta:
// LIB_CORE_UTILS_H
// LIB_NET_SOCKET_H
// APP_MAIN_H
```

2.4 Include guards anidados y condicionales

```c
#ifndef PLATFORM_H
#define PLATFORM_H

// Include condicional basado en plataforma
#if defined(__linux__)
    #define PLATFORM_LINUX 1
    #include <linux/limits.h>
#elif defined(__APPLE__)
    #define PLATFORM_MACOS 1
    #include <sys/syslimits.h>
#elif defined(_WIN32)
    #define PLATFORM_WINDOWS 1
    #include <windows.h>
#else
    #error "Plataforma no soportada"
#endif

// Guarda de version
#if defined(DEBUG) && !defined(DEBUG_H)
    #include "debug.h"
    #define DEBUG_H
#endif

#endif /* PLATFORM_H */
```

3. TIPOS Y ESTRUCTURAS EN CABECERAS
===============================================================================

3.1 Typedefs y structs

```c
#ifndef TIPOS_H
#define TIPOS_H

#include <stdint.h>
#include <stdbool.h>
#include <stddef.h>

// Tipos basicos del proyecto
typedef uint8_t  u8;
typedef uint16_t u16;
typedef uint32_t u32;
typedef uint64_t u64;
typedef int8_t   i8;
typedef int16_t  i16;
typedef int32_t  i32;
typedef int64_t  i64;
typedef float    f32;
typedef double   f64;

// Struct completo (todos los miembros visibles)
typedef struct {
    double x;
    double y;
    double z;
} Vector3D;

// Forward declaration (struct incompleto)
typedef struct ListaEnlazada ListaEnlazada;

// Enumeraciones
typedef enum {
    ESTADO_INACTIVO = 0,
    ESTADO_ACTIVO,
    ESTADO_PAUSADO,
    ESTADO_ERROR
} EstadoSistema;

// Struct con funciones callback
typedef void (*CallbackError)(int codigo, const char *mensaje);
typedef int (*Comparador)(const void *a, const void *b);

#endif /* TIPOS_H */
```

3.2 Forward declarations para reducir dependencias

```c
// archivo: nodo.h
#ifndef NODO_H
#define NODO_H

// Forward declaration: solo necesitamos saber que existe
typedef struct Nodo Nodo;

// Funciones que usan Nodo*
Nodo* nodo_crear(int valor);
void  nodo_destruir(Nodo *n);
int   nodo_obtener_valor(const Nodo *n);

#endif
```

```c
// archivo: nodo.c
#include "nodo.h"
#include <stdlib.h>

// Definicion completa solo en .c
struct Nodo {
    int valor;
    Nodo *siguiente;
    Nodo *anterior;
};

Nodo* nodo_crear(int valor) {
    Nodo *n = malloc(sizeof(Nodo));
    if (n) {
        n->valor = valor;
        n->siguiente = NULL;
        n->anterior = NULL;
    }
    return n;
}

void nodo_destruir(Nodo *n) {
    free(n);
}

int nodo_obtener_valor(const Nodo *n) {
    return n->valor;
}
```

3.3 Estructuras opacas (OOP en C)

```c
// archivo: interfaz_grafica.h
#ifndef INTERFAZ_GRAFICA_H
#define INTERFAZ_GRAFICA_H

// Tipo opaco: el usuario nunca ve los miembros
typedef struct Ventana Ventana;

// API publica
Ventana* ventana_crear(int ancho, int alto, const char *titulo);
void ventana_destruir(Ventana *v);
void ventana_mostrar(Ventana *v);
void ventana_redimensionar(Ventana *v, int ancho, int alto);
int  ventana_esta_abierta(const Ventana *v);
void ventana_set_titulo(Ventana *v, const char *titulo);

#endif
```

```c
// archivo: interfaz_grafica.c
#include "interfaz_grafica.h"
#include <stdlib.h>
#include <string.h>

// La definicion completa esta solo en .c
struct Ventana {
    int ancho;
    int alto;
    char *titulo;
    int handle;
    bool abierta;
};

Ventana* ventana_crear(int ancho, int alto, const char *titulo) {
    Ventana *v = malloc(sizeof(Ventana));
    if (v) {
        v->ancho = ancho;
        v->alto = alto;
        v->titulo = strdup(titulo);
        v->handle = 0;
        v->abierta = true;
    }
    return v;
}
```

===============================================================================
4. VARIABLES EXTERNAS
===============================================================================

Para compartir variables entre archivos .c, se declaran con extern en el
.h y se definen en un solo .c.

4.1 Variable global compartida

```c
// archivo: config.h
#ifndef CONFIG_H
#define CONFIG_H

// Declaracion extern (NO definicion)
extern int nivel_log;
extern const char *archivo_config;
extern int modo_seguro;

// Constantes
#define VERSION "2.0.0"
#define MAX_BUFFER 4096
#define TIMEOUT_DEFAULT 30

// Constante global (definida aqui)
static const int MAX_INTENTOS = 3;

#endif
```

```c
// archivo: config.c
#include "config.h"

// Definiciones (unica vez)
int nivel_log = 2;
const char *archivo_config = "/etc/app/config.ini";
int modo_seguro = 1;
```

4.2 Peligros de las variables globales

```c
// archivo: peligros.h
#ifndef PELIGROS_H
#define PELIGROS_H

// MAL: definir variable en header (causa duplicacion si se incluye
// en multiples .c)
// int contador = 0;  // NO HACER ESTO!

// BIEN: declarar extern
extern int contador;

// MAL: variable estatica en header (cada .c tiene su copia!)
// static int copias_separadas = 0;  // Cada .c tendra su propia variable!

// BIEN: inline functions (C99+)
inline int maximo(int a, int b) {
    return (a > b) ? a : b;
}

#endif
```

4.3 Acceso controlado con funciones getter/setter

```c
// archivo: estado.h
#ifndef ESTADO_H
#define ESTADO_H

#include <stdbool.h>

// No exponemos la variable directamente
// En su lugar, proveemos funciones de acceso

void   estado_inicializar(void);
bool   estado_obtener_conectado(void);
void   estado_set_conectado(bool conectado);
int    estado_obtener_ultimo_error(void);
void   estado_set_ultimo_error(int error);
const char* estado_obtener_mensaje_error(void);

#endif
```

```c
// archivo: estado.c
#include "estado.h"

// Variables estaticas (privadas al modulo)
static bool conectado = false;
static int ultimo_error = 0;
static const char *mensajes[] = {
    "Sin error",
    "Error de conexion",
    "Timeout",
    "Acceso denegado"
};

void estado_inicializar(void) {
    conectado = false;
    ultimo_error = 0;
}

bool estado_obtener_conectado(void) {
    return conectado;
}

// ... etc
```

5. FUNCIONES INLINE Y MACROS
===============================================================================

5.1 Funciones inline en headers

```c
#ifndef INLINE_UTILS_H
#define INLINE_UTILS_H

#include <string.h>
#include <ctype.h>

// Funciones inline - ideales para operaciones pequenias y frecuentes
// El compilador puede expandirlas en el lugar de llamada

inline int entero_absoluto(int x) {
    return x < 0 ? -x : x;
}

inline int min_int(int a, int b) {
    return a < b ? a : b;
}

inline int max_int(int a, int b) {
    return a > b ? a : b;
}

inline int clamp(int valor, int min, int max) {
    if (valor < min) return min;
    if (valor > max) return max;
    return valor;
}

inline int esta_en_rango(int valor, int min, int max) {
    return valor >= min && valor <= max;
}

// Funciones inline con punteros
inline void swap_int(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

inline char char_a_mayuscula(char c) {
    return (c >= 'a' && c <= 'z') ? c - 32 : c;
}

#endif
```

5.2 Macros vs funciones inline

```c
#ifndef MACROS_VS_INLINE_H
#define MACROS_VS_INLINE_H

// MACROS: expansion textual, sin tipo, sin depuracion
#define CUADRADO(x) ((x) * (x))
#define MAX(a, b)   ((a) > (b) ? (a) : (b))
#define MIN(a, b)   ((a) < (b) ? (a) : (b))

// Problemas con macros:
// CUADRADO(++x) -> ((++x) * (++x))  // Incrementa x DOS veces!
// MAX("hola", "mundo")  // Compila pero no hace lo esperado

// Funciones inline: con tipo, seguras, depurables
inline int cuadrado_int(int x) { return x * x; }

// Macro con do-while para statements
#define LOG_ERROR(mensaje) do { \
    fprintf(stderr, "ERROR: %s (%s:%d)\n", \
            mensaje, __FILE__, __LINE__); \
} while (0)

// Macro para obtener el numero de elementos de un array
#define ARRAY_SIZE(arr) (sizeof(arr) / sizeof((arr)[0]))

#endif
```

5.3 Macros con parámetros variádicos

```c
#ifndef DEBUG_MACROS_H
#define DEBUG_MACROS_H

#include <stdio.h>

// Macros con argumentos variadicos
#ifdef DEBUG
    #define DEBUG_PRINT(fmt, ...) \
        fprintf(stderr, "[DEBUG] %s:%d: " fmt "\n", \
                __FILE__, __LINE__, ##__VA_ARGS__)
#else
    #define DEBUG_PRINT(fmt, ...)  // No-op en release
#endif

// Macro de asercion propia
#define ASSERT(cond, msg) do { \
    if (!(cond)) { \
        fprintf(stderr, "ASSERTION FAILED: %s (%s:%d)\n", \
                msg, __FILE__, __LINE__); \
        abort(); \
    } \
} while (0)

// Macro para marcar parametros no usados
#define UNUSED(x) ((void)(x))

#endif
```

6. DEPENDENCIAS ENTRE CABECERAS
===============================================================================

6.1 Inclusión mínima y orden

```c
// BUENA PRACTICA: incluir solo lo necesario
// Si el .h solo usa int*, no necesita incluir <stdlib.h>

// archivo: lista.h
#ifndef LISTA_H
#define LISTA_H

#include <stddef.h>  // size_t
#include <stdbool.h> // bool

// Forward declarations en lugar de includes innecesarios
typedef struct Nodo Nodo;

typedef struct {
    Nodo *cabeza;
    size_t longitud;
} Lista;

// Funciones
Lista* lista_crear(void);
void   lista_destruir(Lista *l);
bool   lista_insertar(Lista *l, int valor, size_t pos);
bool   lista_eliminar(Lista *l, size_t pos);
int    lista_obtener(const Lista *l, size_t pos);
size_t lista_longitud(const Lista *l);
bool   lista_vacia(const Lista *l);

#endif
```

6.2 Inclusiones circulares

```c
// PROBLEMA: inclusion circular
// a.h incluye b.h, y b.h incluye a.h
// Solucion: forward declarations

// archivo: a.h (solucion)
#ifndef A_H
#define A_H

// No incluir b.h!
// Solo forward declaration
struct B;

typedef struct {
    int id;
    struct B *b_relacionado;  // Puntero: no necesita definicion completa
} A;

void a_procesar(A *a);

#endif
```

```c
// archivo: b.h (solucion)
#ifndef B_H
#define B_H

struct A;  // forward declaration

typedef struct {
    int valor;
    struct A *a_relacionado;
} B;

void b_procesar(B *b);

#endif
```

6.3 Header dependencies graph

```c
// Ilustracion de dependencias

// main.c
#include "app.h"        // app.h -> ui.h, core.h
                        // ui.h -> core.h, event.h
                        // core.h <- ya incluido, guarda lo protege
                        // event.h -> core.h (ya incluido)

// app.h

// ui.h
#include "core.h"      // Protegido por #ifndef CORE_H
#include "event.h"

// event.h
#include "core.h"      // Protegido por #ifndef CORE_H

// core.h
#ifndef CORE_H
#define CORE_H
// ... definiciones base ...
#endif
```

8.1 Platform-specific headers

```c
#ifndef PLATAFORMA_H
#define PLATAFORMA_H

    #define COMPILADOR_GCC 1

    #define FORCE_INLINE __attribute__((always_inline))
    #define PACKED __attribute__((packed))
    #define UNUSED __attribute__((unused))
#elif defined(_MSC_VER)
    #define COMPILADOR_MSVC 1
    #define INLINE __inline
    #define FORCE_INLINE __forceinline
    #define PACKED __pragma(pack(push, 1))
    #define UNUSED
#else
    #define COMPILADOR_DESCONOCIDO 1
    #define INLINE inline
    #define FORCE_INLINE
    #define PACKED
    #define UNUSED
#endif

// Deteccion de sistema operativo
#if defined(_WIN32) || defined(_WIN64)
    #define OS_WINDOWS 1
    #include <windows.h>
    #define SLEEP(ms) Sleep(ms)
#elif defined(__linux__)
    #define OS_LINUX 1
    #include <unistd.h>
    #define SLEEP(ms) usleep((ms) * 1000)
#elif defined(__APPLE__) || defined(__MACH__)
    #define OS_MACOS 1
    #include <unistd.h>
    #define SLEEP(ms) usleep((ms) * 1000)
#endif

// Deteccion de arquitectura

#elif defined(__aarch64__)
    #define ARCH_ARM64 1
#elif defined(__arm__) || defined(_M_ARM)
    #define ARCH_ARM 1
#endif

#endif
```

8.2 Cabeceras de depuración

```c
#ifndef DEBUG_UTILS_H
#define DEBUG_UTILS_H

#include <stdio.h>
#include <stdlib.h>

// Niveles de log
typedef enum {
    LOG_TRACE = 0,
    LOG_DEBUG,
    LOG_INFO,
    LOG_WARN,
    LOG_ERROR,
    LOG_FATAL
} LogLevel;

#ifndef LOG_LEVEL
    #define LOG_LEVEL LOG_DEBUG
#endif

#define LOG_COLOR_RESET   "\033[0m"
#define LOG_COLOR_RED     "\033[31m"
#define LOG_COLOR_GREEN   "\033[32m"
#define LOG_COLOR_YELLOW  "\033[33m"
#define LOG_COLOR_BLUE    "\033[34m"
#define LOG_COLOR_CYAN    "\033[36m"

#if LOG_LEVEL <= LOG_TRACE
    #define LOG_TRACE(fmt, ...) \
        fprintf(stderr, LOG_COLOR_CYAN "[TRACE] " LOG_COLOR_RESET \
                "%s:%d: " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)
#else
    #define LOG_TRACE(fmt, ...)
#endif

#if LOG_LEVEL <= LOG_DEBUG
    #define LOG_DEBUG(fmt, ...) \
        fprintf(stderr, LOG_COLOR_BLUE "[DEBUG] " LOG_COLOR_RESET \
                "%s:%d: " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)
#else
    #define LOG_DEBUG(fmt, ...)
#endif

#if LOG_LEVEL <= LOG_INFO
    #define LOG_INFO(fmt, ...) \
        fprintf(stderr, LOG_COLOR_GREEN "[INFO] " LOG_COLOR_RESET \
                fmt "\n", ##__VA_ARGS__)
#else
    #define LOG_INFO(fmt, ...)
#endif

    #define LOG_WARN(fmt, ...) \
        fprintf(stderr, LOG_COLOR_YELLOW "[WARN] " LOG_COLOR_RESET \
                fmt "\n", ##__VA_ARGS__)
#else
    #define LOG_WARN(fmt, ...)
#endif

#if LOG_LEVEL <= LOG_ERROR
    #define LOG_ERROR(fmt, ...) \
        fprintf(stderr, LOG_COLOR_RED "[ERROR] " LOG_COLOR_RESET \
                "%s:%d: " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)
#else
    #define LOG_ERROR(fmt, ...)

    fprintf(stderr, LOG_COLOR_RED "[FATAL] " LOG_COLOR_RESET \

#endif
```

9.1 Antipatrones comunes

```c
// ANTIPATRON 1: Definir variables en el header
// header.h
int contador_global = 0;  // MAL! Cada .c que incluya tendra su copia
// CORRECCION: usar extern en .h, definir en un solo .c

// ANTIPATRON 2: Incluir headers innecesarios
// header.h
#include <stdio.h>    // Solo necesitamos size_t? -> <stddef.h>
#include <stdlib.h>   // Solo necesitamos bool? -> <stdbool.h>
// CORRECCION: incluir solo lo minimo necesario

// ANTIPATRON 3: Funciones definidas en header sin inline
// header.h
int maximo(int a, int b) {  // MAL! Si se incluye en varios .c, error
    return a > b ? a : b;
}
// CORRECCION: declarar como inline o poner en .c

// ANTIPATRON 4: Olvidar include guards
// header.h - SIN GUARDAS! -> error si se incluye dos veces
// CORRECCION: Siempre usar #ifndef / #define / #endif

// ANTIPATRON 5: Dependencias circulares
// a.h incluye b.h, b.h incluye a.h
// CORRECCION: forward declarations

// ANTIPATRON 6: Mezclar interfaz e implementacion
// header.h define funciones completas (no inline) ademas de prototipos
// CORRECCION: solo prototipos en .h, implementacion en .c
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Los headers (.h) contienen declaraciones; los .c contienen definiciones
- Include guards (#ifndef/#define/#endif o #pragma once) evitan doble inclusion
- Las variables globales se declaran con extern en .h y se definen en un .c
- Forward declarations reducen dependencias y evitan inclusiones circulares
- Las funciones inline pueden definirse en headers segun el estandar C99+
- Los tipos opacos (struct incompleto) ocultan la implementacion
- extern "C" es necesario para compatibilidad con compiladores C++
- Incluir solo lo minimo necesario: usar <stddef.h> en vez de <stdio.h>
- Convencion: prefijo del modulo en funciones, MAYUSCULAS en macros
- Un buen header documenta la interfaz sin exponer detalles internos

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿pragma once es estándar?** No es estándar ISO C, pero es soportado
por todos los compiladores modernos (GCC, Clang, MSVC, ICC). Para máxima
portabilidad, use #ifndef tradicional.

❓ **¿Puedo definir una función en un header?** Sí, si es inline (C99+).
Las funciones no-inline causan errores de "multiple definition" si el
header se incluye en varios .c.

❓ **¿Qué es forward declaration?** Declarar un tipo sin definirlo:
struct Nombre; o typedef struct Nombre Nombre; Permite usar punteros al
tipo sin incluir su definicion completa.

❓ **¿Cómo evito dependencias circulares?** Use forward declarations,
separe interfaces en archivos mas pequenios, y disene jerarquias de
dependencias aciclicas.

❓ **¿Debo incluir <stdio.h> en mi header solo porque uso size_t?** No.
Incluya <stddef.h> que define size_t sin arrastrar toda la E/S.

❓ **¿Qué hago con headers del sistema vs locales?** Use <> para headers
del sistema (#include <stdio.h>) y "" para locales (#include "miheader.h").

❓ **¿Cuántos headers debe tener un proyecto?** Depende del tamano. Como
regla: un header por modulo logico. No tenga headers de 1 funcion ni
headers de 1000 declaraciones.
