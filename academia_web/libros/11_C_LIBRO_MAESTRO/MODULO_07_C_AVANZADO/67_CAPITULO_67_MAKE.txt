===============================================================================
1. FUNDAMENTOS DE MAKE
===============================================================================

Make es una herramienta de automatización que determina qué partes de un
programa necesitan ser recompiladas basándose en las dependencias entre
archivos. Fue creada por Stuart Feldman en 1977 en Bell Labs, y sigue siendo
el estándar de facto para proyectos C/C++.

1.1 El problema que resuelve Make

```c
// Sin Make, compilar un proyecto con 3 archivos seria:
// gcc -c main.c -o main.o
// gcc -c funciones.c -o funciones.o
// gcc -c utils.c -o utils.o
// gcc main.o funciones.o utils.o -o programa
// Y recompilar TODO si un solo archivo cambia

// Con Make, solo recompila lo necesario automaticamente
```

1.2 Estructura básica de un Makefile

Un Makefile contiene reglas con la siguiente estructura:

```
target: dependencias
<tab> comando
```

Donde:
- target: el archivo a generar (o un nombre de tarea)
- dependencias: archivos necesarios para crear el target
- comando: la acción a ejecutar (DEBE comenzar con TAB, no espacios)

```makefile
# Makefile básico
programa: main.o funciones.o
    gcc main.o funciones.o -o programa

main.o: main.c
    gcc -c main.c -o main.o

funciones.o: funciones.c
    gcc -c funciones.c -o funciones.o

clean:
    rm -f *.o programa
```

1.3 Primer Makefile funcional

Creemos un proyecto pequeño para demostrar Make:

```c
// main.c
#include <stdio.h>
#include "funciones.h"

int main()
{
    printf("Resultado: %d\n", suma(10, 20));
    return 0;
}
```

```c
// funciones.h
#ifndef FUNCIONES_H
#define FUNCIONES_H
int suma(int a, int b);
#endif
```

```c
// funciones.c
#include "funciones.h"

int suma(int a, int b)
{
    return a + b;
}
```

```makefile
# Makefile
programa: main.o funciones.o
    gcc main.o funciones.o -o programa

main.o: main.c funciones.h
    gcc -c main.c -o main.o

funciones.o: funciones.c funciones.h
    gcc -c funciones.c -o funciones.o

clean:
    rm -f *.o programa
```

2. VARIABLES EN MAKEFILE
===============================================================================

Las variables hacen que los Makefiles sean más flexibles y mantenibles.

2.1 Declaración y uso de variables

```makefile
# Declaracion
CC = gcc
CFLAGS = -Wall -Wextra -O2
LDFLAGS = -lm
LIBS = -lpthread
TARGET = mi_programa
SRCDIR = src
OBJDIR = obj

# Uso: $(NOMBRE_VARIABLE)
$(TARGET): main.o utils.o
    $(CC) main.o utils.o $(LDFLAGS) $(LIBS) -o $(TARGET)

main.o: main.c
    $(CC) $(CFLAGS) -c main.c -o main.o

utils.o: utils.c
    $(CC) $(CFLAGS) -c utils.c -o utils.o
```

2.2 Tipos de asignación

```makefile
# Asignacion simple (evaluada en el momento de uso)
CC = gcc
CFLAGS = -Wall

# Asignacion condicional (solo si no esta definida)
CXX ?= g++
OPTIMIZE ?= -O2

# Asignacion append (+)
CFLAGS += -Wextra   # Ahora CFLAGS = -Wall -Wextra

# Asignacion immediata (evaluada en el momento de la definicion)
DIR = /tmp
PATH := $(DIR)/bin:$(PATH)

# Asignacion con shell
FECHA := $(shell date +%Y%m%d)
ARCHIVOS := $(shell ls *.c)
```

2.3 Variables automáticas

Make proporciona variables que se evalúan automáticamente en cada regla:

```makefile
# $@ : El nombre del target
# $< : El nombre de la primera dependencia
# $^ : Lista de todas las dependencias
# $? : Lista de dependencias mas recientes que el target
# $* : La parte del patron que coincide con % (en reglas de patron)

programa: main.o funciones.o utils.o
    @echo "Creando $@ desde $^"
    $(CC) $^ -o $@

main.o: main.c funciones.h
    $(CC) $(CFLAGS) -c $< -o $@

# Demostracion
variables:
    @echo "Target: $@"
    @echo "Primera dep: $<"
    @echo "Todas las dep: $^"
    @echo "Modificadas: $?"
```

2.4 Makefile con variables automáticas

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -g
LDFLAGS =
LIBS =
TARGET = programa

$(TARGET): main.o funciones.o
    $(CC) $(LDFLAGS) $^ -o $@ $(LIBS)

main.o: main.c funciones.h
    $(CC) $(CFLAGS) -c $< -o $@

funciones.o: funciones.c funciones.h
    $(CC) $(CFLAGS) -c $< -o $@

.PHONY: clean all
clean:
    rm -f *.o $(TARGET)

all: $(TARGET)
```

===============================================================================
3. REGLAS DE PATRÓN
===============================================================================

Las reglas de patrón evitan repetir la misma regla para cada archivo.

3.1 Reglas implícitas y de patrón

```makefile
# Regla de patron: convierte %.c en %.o
%.o: %.c
    $(CC) $(CFLAGS) -c $< -o $@

# Con esta regla, no necesitamos reglas individuales para cada .c
# make generara automaticamente main.o, funciones.o, utils.o, etc.

TARGET = programa
OBJS = main.o funciones.o utils.o

$(TARGET): $(OBJS)
    $(CC) $(LDFLAGS) $^ -o $@ $(LIBS)

# Dependencias automaticas (opcional)
main.o: funciones.h
funciones.o: funciones.h
utils.o: utils.h

.PHONY: clean
clean:
    rm -f $(OBJS) $(TARGET)
```

3.2 Reglas implícitas de Make

Make tiene reglas implícitas incorporadas. Por ejemplo, sabe cómo compilar
.c a .o sin que se lo digamos explícitamente:

```makefile
# Reglas implicitas comunes:
# %.o: %.c  -> usa $(CC) $(CFLAGS) -c
# %: %.o    -> usa $(CC) $(LDFLAGS) -o $@ $^ $(LDLIBS)

# Podemos confiar en ellas:
TARGET = programa
OBJS = main.o funciones.o utils.o

$(TARGET): $(OBJS)
    $(CC) $(LDFLAGS) $^ -o $@ $(LDLIBS)

# No necesitamos regla para .c.o - Make la provee
# Pero podemos sobreescribir las implicitas
%.o: %.c
    $(CC) $(CFLAGS) -c -o $@ $<

.PHONY: clean
clean:
    rm -f $(OBJS) $(TARGET)
```

3.3 Makefile completo con reglas de patrón

```makefile
# ============================================================
# Makefile para proyecto C con reglas de patron
# ============================================================

CC = gcc
CFLAGS = -Wall -Wextra -Werror -pedantic -std=c17
CFLAGS_DEBUG = -g -O0 -DDEBUG
CFLAGS_RELEASE = -O3 -DNDEBUG
LDFLAGS =
LDLIBS = -lm

TARGET = app
SRCDIR = src
OBJDIR = obj
INCDIR = include

SRCS = $(wildcard $(SRCDIR)/*.c)
OBJS = $(patsubst $(SRCDIR)/%.c, $(OBJDIR)/%.o, $(SRCS))
DEPS = $(OBJS:.o=.d)

# Modo por defecto: debug
MODE ?= debug
ifeq ($(MODE), release)
    CFLAGS += $(CFLAGS_RELEASE)
else
    CFLAGS += $(CFLAGS_DEBUG)
endif

# Incluir directorios
CFLAGS += -I$(INCDIR)

# Target principal
$(TARGET): $(OBJS) | $(OBJDIR)
    $(CC) $(LDFLAGS) $^ -o $@ $(LDLIBS)

# Regla de patron para compilar
$(OBJDIR)/%.o: $(SRCDIR)/%.c
    @mkdir -p $(@D)
    $(CC) $(CFLAGS) -MMD -MP -c $< -o $@

# Incluir dependencias automaticas
-include $(DEPS)

# Targets phony
.PHONY: all clean debug release help

all: $(TARGET)

debug:
    $(MAKE) MODE=debug

release:
    $(MAKE) MODE=release

clean:
    rm -rf $(OBJDIR) $(TARGET)

$(OBJDIR):
    mkdir -p $(OBJDIR)

help:
    @echo "Objetivos disponibles:"
    @echo "  all       - Compila el programa (debug por defecto)"
    @echo "  debug     - Compila con simbolos de depuracion"
    @echo "  release   - Compila con optimizacion"
    @echo "  clean     - Limpia archivos generados"
    @echo "  help      - Muestra esta ayuda"
```

===============================================================================
4. TARGETS PHONY Y ESPECIALES
===============================================================================

4.1 .PHONY: targets que no son archivos

Cuando un target no representa un archivo (como clean, all, test), debemos
declararlo como .PHONY para evitar conflictos con archivos del mismo nombre:

```makefile
# Sin .PHONY, si existe un archivo llamado "clean" en el directorio,
# Make pensara que clean esta actualizado y no ejecutara el comando

.PHONY: clean all test install uninstall

clean:
    rm -rf build/ dist/

all: programa

test: programa
    ./programa --test

install: programa
    cp programa /usr/local/bin/

uninstall:
    rm -f /usr/local/bin/programa
```

4.2 Targets especiales

```makefile
# .DEFAULT: regla por defecto si no se encuentra ninguna otra
.DEFAULT:
    @echo "Target desconocido: $@"
    @echo "Use 'make help' para ver los objetivos disponibles"

# .SILENT: ejecuta sin mostrar los comandos
.SILENT: clean

# .IGNORE: ignora errores en los comandos
.IGNORE: test_flaky

# .PRECIOUS: no borra archivos intermedios
.PRECIOUS: %.o

# .INTERMEDIATE: archivos intermedios que pueden borrarse
.INTERMEDIATE: %.s

# .SECONDARY: como intermediate pero no se borran
.SECONDARY: %.d

# .DELETE_ON_ERROR: borra target si el comando falla
.DELETE_ON_ERROR:
```

4.3 Targets con ordenación (`|`)

El operador `|` separa dependencias normales de dependencias de orden.
Las dependencias de orden solo necesitan existir, no importa su timestamp:

```makefile
OUTDIR = build
$(OUTDIR)/programa: $(OBJS) | $(OUTDIR)
    $(CC) $^ -o $@

$(OUTDIR):
    mkdir -p $(OUTDIR)

# El directorio $(OUTDIR) debe existir, pero no hace que se recompile
# si cambia su timestamp
```

5. DIRECTIVAS CONDICIONALES Y FUNCIONES
===============================================================================

5.1 Condicionales

```makefile
CC = gcc

# Condicional simple
ifeq ($(OS), Windows_NT)
    RM = del /Q
    EXT = .exe
else
    RM = rm -f
    EXT =
endif

# ifdef / ifndef
ifdef DEBUG
    CFLAGS = -g -O0 -DDEBUG
else
    CFLAGS = -O2 -DNDEBUG
endif

# ifeq / ifneq con comparaciones
VERSION = 2
ifeq ($(VERSION), 1)
    SRCS = main_v1.c
else
    SRCS = main_v2.c
endif

TARGET = programa$(EXT)

$(TARGET): $(SRCS)
    $(CC) $(CFLAGS) $< -o $@

clean:
    $(RM) $(TARGET)
```

5.2 Funciones de texto

```makefile
# Funcion shell
FECHA := $(shell date +%Y-%m-%d)
ARCHIVOS_C := $(shell find src -name "*.c")
USUARIO := $(shell whoami)

# Funciones de strings
NOMBRE = programa-$(FECHA)
MAYUSCULAS := $(shell echo $(NOMBRE) | tr a-z A-Z)

# Funcion subst
TEXTO = archivo.c archivo.h README.md
SOLO_C := $(filter %.c, $(TEXTO))
SOLO_H := $(filter %.h, $(TEXTO))
SIN_MD := $(filter-out %.md, $(TEXTO))

# Funcion patsubst (pattern substitution)
SRCDIR = src
OBJDIR = obj
SRCS = src/main.c src/utils.c src/io.c
OBJS = $(patsubst $(SRCDIR)/%.c, $(OBJDIR)/%.o, $(SRCS))

# Funcion dir / notdir / basename / suffix
RUTA = src/subdir/archivo.c
DIR := $(dir $(RUTA))        # src/subdir/
ARCHIVO := $(notdir $(RUTA)) # archivo.c
BASE := $(basename $(RUTA))  # src/subdir/archivo
EXT := $(suffix $(RUTA))     # .c

# Funcion wildcard
SRCS := $(wildcard src/*.c)
HDRS := $(wildcard include/*.h)

# Funcion foreach
DIRS = src src/utils src/io
TARGET_DIRS := $(foreach dir, $(DIRS), $(OBJDIR)/$(dir))
$(TARGET_DIRS):
    mkdir -p $@
```

5.3 Makefile con versionado automático

```makefile
CC = gcc
CFLAGS = -Wall -Wextra
VERSION_MAJOR = 1
VERSION_MINOR = 0
VERSION_PATCH := $(shell git rev-list --count HEAD 2>/dev/null || echo 0)
VERSION = $(VERSION_MAJOR).$(VERSION_MINOR).$(VERSION_PATCH)

# Generar archivo de version
version.h: Makefile
    @echo "Generando version.h..."
    @echo "#ifndef VERSION_H" > version.h
    @echo "#define VERSION_H" >> version.h
    @echo "#define VERSION_MAJOR $(VERSION_MAJOR)" >> version.h
    @echo "#define VERSION_MINOR $(VERSION_MINOR)" >> version.h
    @echo "#define VERSION_PATCH $(VERSION_PATCH)" >> version.h
    @echo '#define VERSION_STR "$(VERSION)"' >> version.h
    @echo '#define COMPILE_DATE "$(shell date)"' >> version.h
    @echo '#define COMPILE_USER "$(shell whoami)"' >> version.h
    @echo "#endif" >> version.h

OBJS = main.o utils.o
programa: $(OBJS) version.h
    $(CC) $(CFLAGS) $^ -o $@

main.o: main.c version.h
    $(CC) $(CFLAGS) -c $< -o $@

utils.o: utils.c
    $(CC) $(CFLAGS) -c $< -o $@

.PHONY: clean
clean:
    rm -f *.o programa version.h
```

```c
// main.c (usa version.h generado)
#include <stdio.h>
#include "version.h"

int main()
{
    printf("Programa v%s\n", VERSION_STR);
    printf("Compilado: %s por %s\n", COMPILE_DATE, COMPILE_USER);
    return 0;
}
```

===============================================================================
6. DEPENDENCIAS AUTOMÁTICAS (GCC -MMD)
===============================================================================

GCC puede generar automáticamente las dependencias de cada archivo .c,
eliminando la necesidad de mantenerlas manualmente.

6.1 Generación de dependencias con -MMD

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -MMD -MP
# -MMD: genera archivos .d con dependencias
# -MP: anade targets phony para headers, evitando errores

SRCDIR = src
OBJDIR = obj
SRCS = $(wildcard $(SRCDIR)/*.c)
OBJS = $(patsubst $(SRCDIR)/%.c, $(OBJDIR)/%.o, $(SRCS))
DEPS = $(OBJS:.o=.d)

TARGET = programa

$(TARGET): $(OBJS)
    $(CC) $(LDFLAGS) $^ -o $@

$(OBJDIR)/%.o: $(SRCDIR)/%.c | $(OBJDIR)
    $(CC) $(CFLAGS) -c $< -o $@

$(OBJDIR):
    mkdir -p $(OBJDIR)

# Incluir las dependencias generadas
-include $(DEPS)

.PHONY: clean
clean:
    rm -rf $(OBJDIR) $(TARGET)
```

6.2 Contenido de un archivo .d típico

```
# build/main.d (generado automaticamente por GCC)
build/main.o: src/main.c include/funciones.h include/utils.h \
  /usr/include/stdio.h /usr/include/stdlib.h

funciones.h:

utils.h:

# Esto permite que Make detecte automaticamente cuando
# cualquier header incluido cambia
```

7. DIRECTORIOS MÚLTIPLES Y PROYECTOS GRANDES
===============================================================================

7.1 Estructura con src/ y include/

```makefile
# Makefile para proyecto con subdirectorios
PROJECT = mi_app
CC = gcc
CFLAGS = -Wall -Wextra -Iinclude
LDFLAGS =
LDLIBS = -lm

SRCDIR = src
INCDIR = include
BUILDDIR = build

SRCS = $(shell find $(SRCDIR) -name '*.c')
OBJS = $(patsubst $(SRCDIR)/%.c, $(BUILDDIR)/%.o, $(SRCS))
DEPS = $(OBJS:.o=.d)
TARGET = $(BUILDDIR)/$(PROJECT)

.PHONY: all clean run

all: $(TARGET)

$(TARGET): $(OBJS)
    @echo "Linkeando $@..."
    $(CC) $(LDFLAGS) $^ -o $@ $(LDLIBS)

$(BUILDDIR)/%.o: $(SRCDIR)/%.c
    @mkdir -p $(dir $@)
    $(CC) $(CFLAGS) -MMD -MP -c $< -o $@

run: $(TARGET)
    ./$(TARGET)

clean:
    rm -rf $(BUILDDIR)

-include $(DEPS)
```

7.2 Makefile recursivo (para proyectos con submódulos)

```makefile
# Makefile principal que delega a sub-Makefiles
SUBDIRS = lib/core lib/net lib/ui app

.PHONY: all clean $(SUBDIRS)

all: $(SUBDIRS)

$(SUBDIRS):
    $(MAKE) -C $@

clean:
    for dir in $(SUBDIRS); do \
        $(MAKE) -C $$dir clean; \
    done

# Alternativa con paralelismo
# NOTA: $(SUBDIRS) declarado como .PHONY evita conflictos
```

7.3 Makefile para biblioteca y programa

```makefile
# Makefile para biblioteca estatica + programa de prueba
CC = gcc
AR = ar
CFLAGS = -Wall -Wextra -Iinclude
ARFLAGS = rcs

LIB_DIR = lib
SRC_DIR = src
INC_DIR = include
TEST_DIR = test
BUILD_DIR = build

LIB_SRCS = $(wildcard $(SRC_DIR)/lib/*.c)
LIB_OBJS = $(patsubst $(SRC_DIR)/lib/%.c, $(BUILD_DIR)/lib/%.o, $(LIB_SRCS))
LIB_TARGET = $(BUILD_DIR)/libmi_app.a

TEST_SRCS = $(wildcard $(TEST_DIR)/*.c)
TEST_OBJS = $(patsubst $(TEST_DIR)/%.c, $(BUILD_DIR)/test/%.o, $(TEST_SRCS))
TEST_TARGETS = $(patsubst $(TEST_DIR)/%.c, $(BUILD_DIR)/test/%, $(TEST_SRCS))

.PHONY: all lib test clean

all: lib test

# Construir biblioteca
lib: $(LIB_TARGET)

$(LIB_TARGET): $(LIB_OBJS)
    @mkdir -p $(dir $@)
    $(AR) $(ARFLAGS) $@ $^

$(BUILD_DIR)/lib/%.o: $(SRC_DIR)/lib/%.c
    @mkdir -p $(dir $@)
    $(CC) $(CFLAGS) -MMD -MP -c $< -o $@

# Construir tests
test: $(TEST_TARGETS)

$(BUILD_DIR)/test/%: $(BUILD_DIR)/test/%.o $(LIB_TARGET)
    @mkdir -p $(dir $@)
    $(CC) $(CFLAGS) $^ -o $@ -L$(BUILD_DIR) -lmi_app -lm

$(BUILD_DIR)/test/%.o: $(TEST_DIR)/%.c
    @mkdir -p $(dir $@)
    $(CC) $(CFLAGS) -MMD -MP -c $< -o $@

clean:
    rm -rf $(BUILD_DIR)
```

===============================================================================
9. MAKEFILE PARA DIFERENTES CONFIGURACIONES
===============================================================================

9.1 Debug vs Release vs Profile

```makefile
# Configuraciones de compilacion
CC = gcc
TARGET = programa
SRCS = main.c utils.c io.c
OBJS = $(SRCS:.c=.o)

# Configuraciones por defecto
BUILD_TYPE ?= debug

# Flags base
CFLAGS = -Wall -Wextra -Wpedantic
LDFLAGS =

ifeq ($(BUILD_TYPE), debug)
    CFLAGS += -g -O0 -DDEBUG -fsanitize=address
    LDFLAGS += -fsanitize=address
    BUILD_DIR = build/debug
else ifeq ($(BUILD_TYPE), release)
    CFLAGS += -O3 -DNDEBUG -march=native -flto
    LDFLAGS += -flto -s
    BUILD_DIR = build/release
else ifeq ($(BUILD_TYPE), profile)
    CFLAGS += -O2 -g -DNDEBUG -pg
    LDFLAGS += -pg
    BUILD_DIR = build/profile
else
    $(error Build type no valido: $(BUILD_TYPE))
endif

OBJS := $(addprefix $(BUILD_DIR)/, $(OBJS))
TARGET := $(BUILD_DIR)/$(TARGET)

$(TARGET): $(OBJS)
    @mkdir -p $(dir $@)
    $(CC) $(LDFLAGS) $^ -o $@

$(BUILD_DIR)/%.o: %.c
    @mkdir -p $(dir $@)
    $(CC) $(CFLAGS) -c $< -o $@

.PHONY: all debug release profile clean

all: debug
debug: BUILD_TYPE = debug
debug: $(TARGET)

release: BUILD_TYPE = release
release: $(TARGET)

profile: BUILD_TYPE = profile
profile: $(TARGET)

clean:
    rm -rf build/

# USO:
# make          -> compila debug
# make release  -> compila release
# make profile  -> compila con profiling
# make BUILD_TYPE=release -> compila release
```

9.2 Makefile con opción de verbose

```makefile
# Control de verbosidad
V ?= 0  # 0 = quiet, 1 = verbose

ifeq ($(V), 0)
    Q = @
    VECHO = @echo
else
    Q =
    VECHO = @true
endif

CC = gcc
CFLAGS = -Wall -Wextra
TARGET = programa
OBJS = main.o utils.o

$(TARGET): $(OBJS)
    $(VECHO) "  LINK    $@"
    $(Q)$(CC) $^ -o $@

main.o: main.c
    $(VECHO) "  CC      $<"
    $(Q)$(CC) $(CFLAGS) -c $< -o $@

utils.o: utils.c
    $(VECHO) "  CC      $<"
    $(Q)$(CC) $(CFLAGS) -c $< -o $@

clean:
    $(VECHO) "  CLEAN"
    $(Q)rm -f $(OBJS) $(TARGET)

# $ make V=1  -> muestra todos los comandos
# $ make     -> muestra solo resumenes
```

10. MAKEFILE PARA PROYECTO COMPLETO

10.1 Makefile profesional para proyecto C

```makefile
#==============================================================================
# Makefile para Proyecto C Profesional
#==============================================================================

# Herramientas
CC      ?= gcc
AR      ?= ar
RM      ?= rm -f
MKDIR   ?= mkdir -p
INSTALL ?= install

# Flags de compilacion
CFLAGS   ?= -Wall -Wextra -Wpedantic -Wshadow -Wconversion
CFLAGS   += -Wstrict-prototypes -Wmissing-prototypes
CFLAGS   += -std=c17 -D_POSIX_C_SOURCE=200809L
DEPFLAGS ?= -MMD -MP

# Flags de linkeo
LDFLAGS ?=
LDLIBS  ?= -lm

# Directorios
SRCDIR   ?= src
INCDIR   ?= include
BUILDDIR ?= build
LIBDIR   ?= $(BUILDDIR)/lib
BINDIR   ?= $(BUILDDIR)/bin
TESTDIR  ?= test
DOCDIR   ?= doc

# Archivos
TARGET   := $(BINDIR)/app
LIBRARY  := $(LIBDIR)/libcore.a
SRCS     := $(shell find $(SRCDIR) -name '*.c' ! -path '*/test/*')
OBJS     := $(patsubst $(SRCDIR)/%.c, $(BUILDDIR)/obj/%.o, $(SRCS))
DEPS     := $(OBJS:.o=.d)
TEST_SRCS := $(wildcard $(TESTDIR)/*.c)
TEST_OBJS := $(patsubst $(TESTDIR)/%.c, $(BUILDDIR)/test/%.o, $(TEST_SRCS))
TEST_BINS := $(patsubst $(TESTDIR)/%.c, $(BUILDDIR)/test/%, $(TEST_SRCS))

# Informacion del proyecto
NAME    ?= app
VERSION ?= 1.0.0
AUTHOR  ?= Lucas EGA Academy

#==============================================================================
# Targets principales
#==============================================================================

.PHONY: all lib bin test clean install uninstall help

all: lib bin

# Biblioteca estatica
lib: $(LIBRARY)

$(LIBRARY): $(OBJS)
	@$(MKDIR) $(LIBDIR)
	@echo "  AR      $@"
	$(Q)$(AR) rcs $@ $^

# Ejecutable principal
bin: $(TARGET)

$(TARGET): $(LIBRARY)
	@$(MKDIR) $(BINDIR)
	@echo "  LINK    $@"
	$(Q)$(CC) $(LDFLAGS) -L$(LIBDIR) main.c -o $@ -lcore $(LDLIBS)

# Regla de compilacion generica
$(BUILDDIR)/obj/%.o: $(SRCDIR)/%.c
	@$(MKDIR) $(dir $@)
	@echo "  CC      $<"
	$(Q)$(CC) $(CFLAGS) $(DEPFLAGS) -I$(INCDIR) -c $< -o $@

#==============================================================================
# Tests
#==============================================================================

test: $(TEST_BINS)
	@for test in $^; do \
		echo "  TEST    $$test"; \
		$$test || exit 1; \
	done

$(BUILDDIR)/test/%: $(TESTDIR)/%.c $(LIBRARY)
	@$(MKDIR) $(dir $@)
	@echo "  CC      $<"
	$(Q)$(CC) $(CFLAGS) -I$(INCDIR) -L$(LIBDIR) $< -o $@ -lcore $(LDLIBS)

#==============================================================================
# Limpieza
#==============================================================================

clean:
	@echo "  CLEAN   $(BUILDDIR)"
	$(Q)$(RM) -r $(BUILDDIR)

distclean: clean
	@echo "  CLEAN   *~ *.bak"
	$(Q)$(RM) *~ *.bak

#==============================================================================
# Instalacion
#==============================================================================

PREFIX ?= /usr/local
INSTALL_BIN := $(DESTDIR)$(PREFIX)/bin
INSTALL_LIB := $(DESTDIR)$(PREFIX)/lib
INSTALL_INC := $(DESTDIR)$(PREFIX)/include/$(NAME)

install: $(TARGET) $(LIBRARY)
	@$(MKDIR) $(INSTALL_BIN) $(INSTALL_LIB) $(INSTALL_INC)
	$(INSTALL) -m 755 $(TARGET) $(INSTALL_BIN)/
	$(INSTALL) -m 644 $(LIBRARY) $(INSTALL_LIB)/
	$(INSTALL) -m 644 $(wildcard $(INCDIR)/*.h) $(INSTALL_INC)/

uninstall:
	$(RM) $(INSTALL_BIN)/$(notdir $(TARGET))
	$(RM) $(INSTALL_LIB)/$(notdir $(LIBRARY))
	$(RM) -r $(INSTALL_INC)

#==============================================================================
# Utilidades
#==============================================================================

help:
	@echo "================================================================================"
	@echo "  $(NAME) v$(VERSION) by $(AUTHOR)"
	@echo "================================================================================"
	@echo ""
	@echo "  Targets:"
	@echo "    all       - Compila biblioteca y ejecutable"
	@echo "    lib       - Compila solo la biblioteca"
	@echo "    bin       - Compila solo el ejecutable"
	@echo "    test      - Compila y ejecuta tests"
	@echo "    clean     - Limpia archivos generados"
	@echo "    install   - Instala en $(PREFIX)"
	@echo "    uninstall - Desinstala"
	@echo ""
	@echo "  Variables:"
	@echo "    CC        - Compilador (default: gcc)"
	@echo "    CFLAGS    - Flags de compilacion"
	@echo "    PREFIX    - Prefijo de instalacion (default: /usr/local)"
	@echo ""
	@echo "  Ejemplos:"
	@echo "    make                   - Compilacion por defecto"
	@echo "    make CC=clang CFLAGS=-O3  - Compila con clang optimizado"
	@echo "    make clean all         - Recompila desde cero"
	@echo "    make install PREFIX=~  - Instala en el home"
	@echo "================================================================================"

#==============================================================================
# Dependencias
#==============================================================================

-include $(DEPS)

# Silenciar comandos (descomentar para produccion)
Q ?= @
```

RESUMEN DEL CAPÍTULO
===============================================================================

- Make automatiza la compilación usando reglas target:dependencias -> comando
- Las variables ($(VAR)) hacen los Makefiles mantenibles y flexibles
- Variables automáticas: $@ (target), $< (primera dep), $^ (todas las dep)
- Reglas de patrón (%.o: %.c) evitan repetir reglas para cada archivo
- .PHONY declara targets que no son archivos (clean, all, test)
- -MMD genera dependencias automáticas; include las importa
- Las funciones (wildcard, patsubst, shell, foreach) manipulan texto
- Los condicionales (ifeq, ifdef) permiten configuraciones multiplataforma
- BUILD_TYPE (debug/release/profile) maneja diferentes configuraciones
- Q=@ silencia comandos; V=1 controla verbosidad

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué Make dice "missing separator. Stop"?** Porque usaste espacios
en lugar de TAB para indentar los comandos. Make exige TAB, no espacios.

❓ **¿Cómo veo los comandos que Make ejecuta?** Use `make V=1` o no use @
al inicio de los comandos. O ejecute `make -n` para simular sin ejecutar.

❓ **¿Qué significa .PHONY?** Declara que un target no representa un archivo
real. Sin .PHONY, si existe un archivo "clean", Make pensará que está
actualizado y no ejecutará el comando.

❓ **¿Cómo compilo en paralelo?** `make -j$(nproc)` o `make -j4`. Make
ejecuta reglas independientes en paralelo, acelerando significativamente.

❓ **¿Cómo depuro un Makefile?** Use `make -p` (imprime la base de datos),
`make -d` (debug), `make -n` (dry-run), y `$(info VARIABLE = $(VARIABLE))`
para depurar valores.

❓ **¿Make o CMake?** Make es más simple y directo para proyectos pequeños/
medianos. CMake es más potente y multiplataforma, pero más complejo. Make
sigue siendo excelente para proyectos C puros.

❓ **¿Cómo ignoro errores en un comando?** Anteponga `-` al comando:
`-rm -f *.o`. O use .IGNORE para todo el target.
