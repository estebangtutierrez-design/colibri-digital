1. INTRODUCCIÓN AL PREPROCESADOR
===============================================================================

El preprocesador de C es un procesador de texto que opera antes de la
compilación. No conoce C, no entiende tipos de datos, ni variables, ni
funciones. Solo sabe de directivas que comienzan con el símbolo #.

Las directivas del preprocesador son instrucciones que se ejecutan en una fase
temprana. Todo lo que el preprocesador produce es luego pasado al compilador.

Fases de compilación en C:
1. Manejo de caracteres trigraph y reemplazo de secuencias
2. División en líneas lógicas (unión de líneas con \)
3. Tokenización y eliminación de comentarios
4. EJECUCIÓN DEL PREPROCESADOR
5. Compilación a código ensamblador
6. Ensamblado a código objeto
7. Enlazado (linking)

El preprocesador trabaja en la fase 4 y sus resultados son texto puro que el
compilador procesa después.

Ejemplo simple del flujo:

```c
// entrada.c
#include <stdio.h>
#define MENSAJE "Hola Mundo"

int main(void) {
    printf("%s\n", MENSAJE);
    return 0;
}
```

Después del preprocesamiento, el código se vería así (simplificado):

```c
// contenido de stdio.h insertado aquí...
// ... cientos de líneas ...

int main(void) {
    printf("%s\n", "Hola Mundo");
    return 0;
}
```

La macro MENSAJE fue reemplazada por su valor literal "Hola Mundo" y el
contenido de stdio.h fue insertado.

Podemos ver el resultado del preprocesador con el flag -E de gcc:

```c
// gcc -E entrada.c -o salida.i
// Esto genera el archivo preprocesado sin compilar
```

===============================================================================
2. #define — CONSTANTES SIMBÓLICAS
===============================================================================

La directiva #define es la más utilizada del preprocesador. Permite definir
constantes simbólicas que serán reemplazadas textualmente en el código.

Sintaxis básica:

```c
#define NOMBRE valor
```

Donde NOMBRE sigue las reglas de identificadores de C (letras, dígitos,
guión bajo, no comienza con dígito) y por convención se escribe en
MAYÚSCULAS.

Ejemplos fundamentales:

```c
#include <stdio.h>

#define PI 3.1415926535
#define RADIO 10.0
#define NOMBRE_PROGRAMA "Calculadora de Área"
#define MAX_ESTUDIANTES 100
#define EDAD_MINIMA 18
#define ACTIVO 1
#define INACTIVO 0

int main(void) {
    double area = PI * RADIO * RADIO;
    
    printf("=== %s ===\n", NOMBRE_PROGRAMA);
    printf("Radio: %.2f\n", RADIO);
    printf("Área del círculo: %.4f\n", area);
    printf("Capacidad máxima: %d estudiantes\n", MAX_ESTUDIANTES);
    
    int estado = ACTIVO;
    if (estado == ACTIVO) {
        printf("Sistema activo\n");
    }
    
    return 0;
}
```

Las constantes #define no tienen tipo. Son reemplazos textuales. El compilador
ve el valor directamente y deduce el tipo según el contexto.

Múltiples líneas con \:

```c
#include <stdio.h>

#define ENCABEZADO \
    "==================================\n" \
    "   SISTEMA DE GESTIÓN ACADÉMICA   \n" \
    "==================================\n"

int main(void) {
    printf(ENCABEZADO);
    printf("Bienvenido al sistema\n");
    return 0;
}
```

#define con expresiones (¡cuidado!):

```c
#include <stdio.h>

#define CUADRADO(x) x * x      // ¡PELIGRO! Sin paréntesis
#define CUADRADO_SEGURO(x) ((x) * (x))

int main(void) {
    int a = 5;
    printf("CUADRADO(5) = %d\n", CUADRADO(5));         // 25
    printf("CUADRADO(3+2) = %d\n", CUADRADO(3+2));     // 11 = 3+2*3+2 = 3+6+2 = 11 ¡ERROR!
    printf("CUADRADO_SEGURO(3+2) = %d\n", CUADRADO_SEGURO(3+2)); // 25 = ((3+2)*(3+2))
    
    int b = 3;
    printf("CUADRADO(b+1) = %d\n", CUADRADO(b+1));     // 7 = 3+1*3+1 = 3+3+1 ¡ERROR!
    printf("CUADRADO_SEGURO(b+1) = %d\n", CUADRADO_SEGURO(b+1)); // 16
    
    return 0;
}
```

Problema clásico con macros:

```c
#include <stdio.h>

#define DOBLE(x) x + x
#define DOBLE_SEGURO(x) ((x) + (x))

int main(void) {
    int a = 5;
    int resultado = 10 * DOBLE(a);
    printf("10 * DOBLE(5) = %d\n", resultado); // 10 * 5 + 5 = 55, ¡no 100!
    
    resultado = 10 * DOBLE_SEGURO(a);
    printf("10 * DOBLE_SEGURO(5) = %d\n", resultado); // 100
    
    return 0;
}
```

Definición de constantes con valores compuestos:

```c
#include <stdio.h>

#define MSG_ERROR "Error: Operación no permitida"
#define MSG_EXITO "Operación completada con éxito"
#define SEPARADOR "----------------------------------------\n"
#define FORMATO_TABLA "| %-20s | %5s | %8s |\n"

int main(void) {
    printf(SEPARADOR);
    printf(FORMATO_TABLA, "Nombre", "Edad", "Código");
    printf(SEPARADOR);
    printf(FORMATO_TABLA, "Ana López", "22", "A001");
    printf(FORMATO_TABLA, "Carlos Ruiz", "25", "A002");
    printf(FORMATO_TABLA, "María García", "21", "A003");
    printf(SEPARADOR);
    
    printf("%s\n", MSG_EXITO);
    return 0;
}
```

===============================================================================
3. #include — INSERCIÓN DE ARCHIVOS
===============================================================================

La directiva #include inserta el contenido de un archivo en la posición actual.
Es la base del sistema de módulos de C.

Formas de #include:

```c
#include <archivo.h>  // Busca en directorios del sistema (estándar)
#include "archivo.h"  // Busca primero en directorio actual, luego en sistema
```

Diferencia entre <> y "":

<> busca en las rutas de inclusión del sistema (configuradas con -I en gcc).
"" busca primero en el directorio del archivo fuente, y si no encuentra,
busca en las rutas del sistema igual que <>.

Ejemplo de inclusión de bibliotecas estándar:

```c
#include <stdio.h>    // Entrada/Salida estándar
#include <stdlib.h>   // Funciones de utilidad general
#include <string.h>   // Funciones de manipulación de cadenas
#include <math.h>     // Funciones matemáticas
#include <time.h>     // Funciones de tiempo
#include <ctype.h>    // Clasificación de caracteres
#include <errno.h>    // Manejo de errores
#include <assert.h>   // Aserciones de depuración
#include <limits.h>   // Límites de tipos enteros
#include <float.h>    // Límites de tipos punto flotante
#include <stddef.h>   // Definiciones estándar (NULL, size_t, etc.)
#include <stdint.h>   // Tipos enteros de ancho fijo
#include <stdbool.h>  // Tipo bool y constantes true/false

int main(void) {
    printf("Todas las bibliotecas estándar incluidas\n");
    printf("INT_MAX = %d\n", INT_MAX);
    printf("FLT_MAX = %e\n", FLT_MAX);
    printf("RAND_MAX = %d\n", RAND_MAX);
    return 0;
}
```

Inclusión de archivos propios:

```c
// Supongamos que tenemos estos archivos:
// "mis_constantes.h"   - Definiciones de constantes
// "mis_funciones.h"    - Prototipos de funciones
// "mis_estructuras.h"  - Definiciones de estructuras

#include <stdio.h>
#include "mis_constantes.h"
#include "mis_estructuras.h"
#include "mis_funciones.h"

int main(void) {
    Estudiante e;
    iniciar_estudiante(&e, "Ana", 22, 9.5);
    mostrar_estudiante(&e);
    return 0;
}
```

Anidación de #include (un archivo incluye a otro):

```c
// config.h
#ifndef CONFIG_H
#define CONFIG_H

#define VERSION "2.0"
#define MAX_USUARIOS 500

#include "constantes_red.h"
#include "constantes_seguridad.h"

#endif
```

Protección contra inclusión múltiple (include guards):

```c
// mi_biblioteca.h
#ifndef MI_BIBLIOTECA_H
#define MI_BIBLIOTECA_H

int suma(int a, int b);
int resta(int a, int b);

#endif
```

Ejemplo de dependencias con includes:

```c
// Utilizamos varios archivos de encabezado anidados
#include <stdio.h>
#include "config.h"
#include "utilidades/logger.h"
#include "utilidades/validacion.h"
#include "modelos/usuario.h"
#include "modelos/producto.h"
#include "servicios/autenticacion.h"
#include "servicios/pagos.h"

int main(void) {
    printf("Sistema v%s iniciado\n", VERSION);
    log_info("Sistema iniciado correctamente");
    
    Usuario admin = crear_usuario("admin", "admin123");
    if (autenticar(&admin)) {
        Producto p = crear_producto("Laptop", 1200.00);
        if (procesar_pago(&admin, &p)) {
            log_info("Pago procesado exitosamente");
        }
    }
    
    return 0;
}
```

===============================================================================
4. #undef — CANCELACIÓN DE MACROS
===============================================================================

La directiva #undef elimina la definición de una macro previamente definida
con #define. Es útil para redefinir macros o para limpiar el espacio de
nombres.

Sintaxis:

```c
#undef NOMBRE
```

Ejemplo básico:

```c
#include <stdio.h>

#define VALOR 100

int main(void) {
    printf("VALOR = %d\n", VALOR); // 100
    
#undef VALOR
    // printf("VALOR = %d\n", VALOR); // ERROR: VALOR no está definido
    
#define VALOR 200
    printf("VALOR = %d\n", VALOR); // 200
    
    return 0;
}
```

Uso de #undef para configuraciones:

```c
#include <stdio.h>

#define DEBUG          // activamos debug
#define NIVEL_LOG 3

#ifdef DEBUG
    #define LOG(msg) printf("[DEBUG] %s\n", msg)
#else
    #define LOG(msg)
#endif

void funcion_sensible(void) {
    LOG("Entrando a funcion_sensible");
    // ... lógica crítica ...
    LOG("Saliendo de funcion_sensible");
}

int main(void) {
    LOG("Inicio del programa");
    funcion_sensible();
    
#undef DEBUG
#undef NIVEL_LOG
    
    // A partir de aquí, DEBUG no está definido
    // LOG("Esto no se imprimiría si estuviera en otra función");
    
    printf("Programa finalizado\n");
    return 0;
}
```

#undef con macros de función:

```c
#include <stdio.h>

#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define MIN(a, b) ((a) < (b) ? (a) : (b))

int main(void) {
    int x = 10, y = 20;
    printf("MAX(%d, %d) = %d\n", x, y, MAX(x, y));
    printf("MIN(%d, %d) = %d\n", x, y, MIN(x, y));
    
#undef MAX
#undef MIN
    
    // Ahora MAX y MIN ya no están definidos
    // printf("%d\n", MAX(5, 3)); // ERROR: MAX no está definido
    
    // Podemos redefinir con otro significado
#define MAX(a, b) ((a) >= (b) ? "mayor o igual" : "menor")
    
    printf("Resultado: %s\n", MAX(5, 3));
    
    return 0;
}
```

Uso avanzado con cambio de comportamiento:

```c
#include <stdio.h>

#define TIPO_DATO int
#define FORMATO "%d"

int main(void) {
    TIPO_DATO a = 42;
    printf("Valor = " FORMATO "\n", a);
    
#undef TIPO_DATO
#undef FORMATO
    
#define TIPO_DATO double
#define FORMATO "%.2f"
    
    TIPO_DATO b = 3.14159;
    printf("Valor = " FORMATO "\n", b);
    
#undef TIPO_DATO
#undef FORMATO
    
#define TIPO_DATO char*
#define FORMATO "%s"
    
    TIPO_DATO c = "Hola Mundo";
    printf("Valor = " FORMATO "\n", c);
    
    return 0;
}
```

5. MACROS PREDEFINIDAS DEL ESTÁNDAR
===============================================================================

El estándar C define varias macros que están siempre disponibles para el
programador. Estas macros proporcionan información sobre el código fuente y
el entorno de compilación.

Macros predefinidas obligatorias (C99 y posteriores):

```c
#include <stdio.h>

int main(void) {
    printf("=== MACROS PREDEFINIDAS ===\n\n");
    
    printf("__FILE__:     %s\n", __FILE__);       // Nombre del archivo actual
    printf("__LINE__:     %d\n", __LINE__);       // Línea actual
    printf("__DATE__:     %s\n", __DATE__);       // Fecha de compilación
    printf("__TIME__:     %s\n", __TIME__);       // Hora de compilación
    printf("__STDC__:     %d\n", __STDC__);       // 1 si compilador es estándar
    printf("__STDC_VERSION__: %ld\n", __STDC_VERSION__); // Versión del estándar
    
    return 0;
}
```

Uso de __FILE__ y __LINE__ para depuración:

```c
#include <stdio.h>

#define LOG_INFO(msg) \
    printf("[INFO] %s:%d: %s\n", __FILE__, __LINE__, msg)

#define LOG_ERROR(msg) \
    fprintf(stderr, "[ERROR] %s:%d: %s\n", __FILE__, __LINE__, msg)

#define ASSERT_TRUE(cond) \
    do { \
        if (!(cond)) { \
            fprintf(stderr, "ASSERT FALLIDO: %s:%d: '%s' es falso\n", \
                    __FILE__, __LINE__, #cond); \
        } \
    } while(0)

int dividir(int a, int b, int* resultado) {
    if (b == 0) {
        LOG_ERROR("División por cero detectada");
        return -1;
    }
    *resultado = a / b;
    LOG_INFO("División exitosa");
    return 0;
}

int main(void) {
    LOG_INFO("Inicio del programa");
    
    int res;
    ASSERT_TRUE(dividir(10, 2, &res) == 0);
    printf("10 / 2 = %d\n", res);
    
    ASSERT_TRUE(dividir(10, 0, &res) != 0);
    
    LOG_INFO("Fin del programa");
    return 0;
}
```

Función que imprime información de depuración:

```c
#include <stdio.h>

void mostrar_info_compilacion(void) {
    printf("╔══════════════════════════════════════╗\n");
    printf("║     INFORMACIÓN DE COMPILACIÓN       ║\n");
    printf("╠══════════════════════════════════════╣\n");
    printf("║ Archivo:     %-26s║\n", __FILE__);
    printf("║ Línea:       %-26d║\n", __LINE__);
    printf("║ Fecha:       %-26s║\n", __DATE__);
    printf("║ Hora:        %-26s║\n", __TIME__);
    printf("║ Estándar:    %-26s║\n",
        #ifdef __STDC_VERSION__
            #if __STDC_VERSION__ >= 201112L
                "C11"
            #elif __STDC_VERSION__ >= 199901L
                "C99"
            #elif __STDC_VERSION__ >= 199409L
                "C95"
            #else
                "C89/C90"
            #endif
        #else
            "ANSI C"
        #endif
    );
    printf("╚══════════════════════════════════════╝\n");
}

int main(void) {
    mostrar_info_compilacion();
    return 0;
}
```

Uso de __LINE__ para generar etiquetas únicas:

```c
#include <stdio.h>

#define CONCATENAR(x, y) x ## y
#define ETIQUETA_UNICA(linea) CONCATENAR(etiqueta_, linea)
#define CREAR_ETIQUETA ETIQUETA_UNICA(__LINE__)

int main(void) {
    int CREAR_ETIQUETA = 100;    // int etiqueta_50 = 100
    int CREAR_ETIQUETA = 200;    // int etiqueta_51 = 200
    
    printf("etiqueta_50 = %d\n", etiqueta_50);
    printf("etiqueta_51 = %d\n", etiqueta_51);
    
    return 0;
}
```

Macro para generar nombres de variables únicos:

```c
#include <stdio.h>

#define CONCAT(x, y) x ## y
#define VAR_UNICA(base) CONCAT(base, __LINE__)

int main(void) {
    int VAR_UNICA(contador) = 10;  // int contador85 = 10
    int VAR_UNICA(contador) = 20;  // int contador86 = 20
    
    printf("contador85 = %d\n", contador85);
    printf("contador86 = %d\n", contador86);
    
    return 0;
}
```

6. #pragma — DIRECTIVAS DEL COMPILADOR
===============================================================================

La directiva #pragma proporciona instrucciones específicas al compilador.
Es una directiva de implementación: cada compilador puede definir sus propios
pragmas. El estándar dice que los pragmas no reconocidos deben ser ignorados.

#pragma once (evitar inclusión múltiple):

```c
// encabezado.h
#pragma once  // Equivalente a #ifndef + #define + #endif

int funcion_ejemplo(void);
```

#pragma GCC dependency (verificar dependencias):

```c
#include <stdio.h>

// Verificar que cierto archivo no es más reciente que el actual
// #pragma GCC dependency "config.h"
// #pragma GCC dependency "config.h" "¡config.h ha sido modificado!"

int main(void) {
    printf("Dependencias verificadas\n");
    return 0;
}
```

#pragma GCC poison (envenenar identificadores):

```c
#include <stdio.h>

// Prohibir el uso de ciertos identificadores
#pragma GCC poison malloc
#pragma GCC poison free
#pragma GCC poison goto

int main(void) {
    printf("Funciones peligrosas envenenadas\n");
    
    // int* p = malloc(100);  // ERROR: malloc está envenenado
    // free(p);               // ERROR: free está envenenado
    // goto fin;              // ERROR: goto está envenenado
    
    return 0;
}
```

#pragma GCC warning/error (mensajes controlados):

```c
#include <stdio.h>

// Generar warnings/errors específicos
// #pragma GCC warning "Este código usa funciones obsoletas"
// #pragma GCC error "Esta versión no es compatible con el sistema"

#define USAR_FUNCION_ANTIGUA
#ifdef USAR_FUNCION_ANTIGUA
    #pragma message "ATENCIÓN: Se está usando una función obsoleta"
#endif

int main(void) {
    printf("Compilación con mensajes controlados\n");
    return 0;
}
```

#pragma pack (control de alineación):

```c
#include <stdio.h>

// Sin pack (alineación natural)
struct SinPack {
    char c;
    int i;
    char d;
};

// Con pack (sin relleno)
#pragma pack(1)
struct ConPack {
    char c;
    int i;
    char d;
};
#pragma pack() // Restaurar alineación por defecto

int main(void) {
    printf("sizeof(struct SinPack) = %zu bytes\n", sizeof(struct SinPack));
    printf("sizeof(struct ConPack) = %zu bytes\n", sizeof(struct ConPack));
    printf("Esperado con pack(1): %zu bytes\n", sizeof(char) + sizeof(int) + sizeof(char));
    
    return 0;
}
```

Pragmas para optimización:

```c
#include <stdio.h>

// Sugerir optimización para una función
#pragma GCC optimize("O3")

int funcion_optimizada(int n) {
    int suma = 0;
    for (int i = 0; i < n; i++) {
        suma += i * i;
    }
    return suma;
}

#pragma GCC reset_options

int main(void) {
    printf("Resultado: %d\n", funcion_optimizada(1000));
    return 0;
}
```

===============================================================================
7. #error — GENERACIÓN DE ERRORES EN PREPROCESAMIENTO
===============================================================================

La directiva #error genera un error en tiempo de compilación con un mensaje
personalizado. Es útil para verificar condiciones que deben cumplirse antes
de compilar.

Sintaxis:

```c
#error mensaje
```

Validación de versión del estándar:

```c
#include <stdio.h>

#if !defined(__STDC_VERSION__) || __STDC_VERSION__ < 199901L
    #error "Este programa requiere al menos C99"
#endif

int main(void) {
    printf("Compilando con C99 o superior\n");
    
    // Característica de C99: declaración en for
    for (int i = 0; i < 5; i++) {
        printf("i = %d\n", i);
    }
    
    return 0;
}
```

Validación de plataforma:

```c
#include <stdio.h>

#if defined(__linux__)
    #define PLATAFORMA "Linux"
#elif defined(_WIN32)
    #define PLATAFORMA "Windows"
#elif defined(__APPLE__)
    #define PLATAFORMA "macOS"
#else
    #error "Plataforma no soportada. Este programa requiere Linux, Windows o macOS"
#endif

int main(void) {
    printf("Compilando para: %s\n", PLATAFORMA);
    return 0;
}
```

Validación de límites:

```c
#include <limits.h>
#include <stdio.h>

#if INT_MAX < 2147483647
    #error "Se requiere un sistema con int de al menos 32 bits"
#endif

#if CHAR_BIT != 8
    #error "Se requieren bytes de 8 bits"
#endif

int main(void) {
    printf("Sistema compatible: int=%zu bits, char=%d bits\n",
           sizeof(int) * 8, CHAR_BIT);
    return 0;
}
```

Validación de configuración:

```c
#include <stdio.h>

#define MAX_BUFFER 256

#ifndef MAX_BUFFER
    #error "MAX_BUFFER debe estar definido"
#endif

#if MAX_BUFFER < 64
    #error "MAX_BUFFER debe ser al menos 64"
#elif MAX_BUFFER > 65536
    #error "MAX_BUFFER no debe exceder 65536"
#endif

int main(void) {
    char buffer[MAX_BUFFER];
    printf("Buffer de %d bytes creado\n", MAX_BUFFER);
    return 0;
}
```

Verificación de componentes requeridos:

```c
#include <stdio.h>

// Componentes opcionales
#define TIENE_MATEMATICAS
// #define TIENE_RED
#define TIENE_GRAFICOS

#ifndef TIENE_MATEMATICAS
    #error "El módulo matemático es obligatorio para compilar"
#endif

#ifndef TIENE_RED
```

10. BUENAS PRÁCTICAS CON EL PREPROCESADOR
===============================================================================

Reglas de oro para usar el preprocesador:

1. Usar #define para constantes (C prefiere const o enum, pero #define es
   válido para macros preprocesador)
2. Siempre poner paréntesis alrededor de los parámetros de macros
3. Siempre poner paréntesis alrededor de la macro completa
4. Evitar efectos secundarios en argumentos de macros
5. Usar do { ... } while(0) para macros multi-sentencia
6. Preferir enum sobre #define para constantes enteras relacionadas
7. Preferir const sobre #define para constantes con tipo
8. Nunca usar #define para redefinir palabras clave del lenguaje

Ejemplo de buenas prácticas:

```c
#include <stdio.h>

// Mal: sin paréntesis
#define CUBO_MAL(x) x * x * x

// Bien: con paréntesis
#define CUBO(x) ((x) * (x) * (x))

// Mal: macro multi-sentencia sin do-while
#define SWAP_MAL(a, b) { int tmp = a; a = b; b = tmp; }

// Bien: macro multi-sentencia con do-while
#define SWAP(a, b, tipo) \
    do { \
        tipo tmp = a; \
        a = b; \
        b = tmp; \
    } while(0)

int main(void) {
    int x = 3, y = 5;
    
    printf("CUBO(3) = %d\n", CUBO(3));
    printf("CUBO(1+2) = %d\n", CUBO(1+2)); // ((1+2)*(1+2)*(1+2)) = 27
    
    printf("Antes: x=%d, y=%d\n", x, y);
    SWAP(x, y, int);
    printf("Después: x=%d, y=%d\n", x, y);
    
    // Efecto secundario: EVITAR
    int arr[] = {1, 2, 3, 4, 5};
    int i = 0;
    // printf("%d\n", CUBO(arr[i++])); // i++ se ejecuta 3 veces
    i = 0;
    int val = arr[i++];
    printf("Cubo seguro: %d\n", CUBO(val)); // i++ solo una vez
    
    return 0;
}
```

RESUMEN DEL CAPÍTULO

- El preprocesador procesa el código antes de la compilación usando directivas
  que comienzan con #
- #define crea constantes simbólicas y macros que son reemplazadas textualmente
- #include inserta el contenido de archivos en el código fuente
- #undef elimina definiciones de macros previamente creadas
- Las macros predefinidas como __FILE__, __LINE__, __DATE__, __TIME__ y
  __STDC__ proporcionan información contextual valiosa
- #pragma ofrece control específico del compilador (pack, optimize, poison)
- #error genera errores en tiempo de preprocesamiento para validaciones
- Las macros con parámetros requieren paréntesis cuidadosos para evitar errores
- El sistema de includes permite organizar código en módulos reutilizables
- Las macros predefinidas del compilador permiten detección multiplataforma

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre #define y const?** #define es un reemplazo
textual sin tipo que ocurre en tiempo de preprocesamiento. const es una
variable con tipo en tiempo de compilación. #define no ocupa memoria, const sí.
En C moderno se prefiere const para la mayoría de casos.

❓ **¿Puedo usar #include dentro de una función?** Sí, pero no es recomendable.
La directiva #include puede aparecer en cualquier lugar del archivo, pero
por claridad y mantenibilidad debe ir al inicio.

❓ **¿Qué pasa si incluyo el mismo archivo dos veces?** Sin include guards,
el contenido se duplica, causando errores de redefinición. Por eso se usan
#ifndef/#define/#endif o #pragma once.

❓ **¿Cómo veo el resultado del preprocesador?** Con gcc -E archivo.c -o
archivo.i. Esto genera el archivo preprocesado listo para compilar.

❓ **¿#pragma once es estándar?** No es parte del estándar C, pero es
soportado por la mayoría de compiladores (GCC, Clang, MSVC). Es más limpio
que los include guards tradicionales pero menos portátil.
