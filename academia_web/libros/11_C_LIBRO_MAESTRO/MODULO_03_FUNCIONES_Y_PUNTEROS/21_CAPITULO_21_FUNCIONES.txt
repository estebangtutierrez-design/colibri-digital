1. ¿QUÉ ES UNA FUNCIÓN?
================================================================================

Una función es un bloque de código reutilizable que realiza una tarea
específica. Tiene un nombre único, puede recibir datos (parámetros) y
puede devolver un resultado.

Sintaxis general:

  tipo_retorno nombre_funcion(tipo param1, tipo param2, ...) {
      // cuerpo de la función
      return valor;
  }

Ejemplo mínimo:

--------------------------------------------------------------------------------
  #include <stdio.h>

  void saludar(void) {
      printf("Hola, mundo!\n");
  }

  int main(void) {
      saludar();
      return 0;
  }
--------------------------------------------------------------------------------

Componentes:
  • tipo_retorno:  El tipo del valor que devuelve (int, float, void...).
  • nombre:        Identificador único (siguiendo reglas de C).
  • parámetros:    Lista de variables que recibe la función.
  • cuerpo:        Bloque de sentencias entre llaves { }.
  • return:        Envía un valor al punto de llamada.

================================================================================
2. DEFINICIÓN VS. DECLARACIÓN (PROTOTIPO)
================================================================================

DEFINICIÓN:
  Es el cuerpo completo de la función. Incluye el encabezado y el
  bloque de código. El compilador reserva memoria y genera código.

DECLARACIÓN (PROTOTIPO):
  Es solo el encabezado seguido de punto y coma. Le dice al compilador
  "esta función existe, tiene tal firma". Permite llamar a la función
  antes de definirla.

Ejemplo de prototipo:

--------------------------------------------------------------------------------
  #include <stdio.h>

  // PROTOTIPO: declaración sin cuerpo
  int sumar(int a, int b);

  int main(void) {
      int resultado = sumar(5, 3);
      printf("Suma: %d\n", resultado);
      return 0;
  }

  // DEFINICIÓN: cuerpo completo
  int sumar(int a, int b) {
      return a + b;
  }
--------------------------------------------------------------------------------

¿Por qué usar prototipos?
  • Permiten definir funciones después de main().
  • El compilador verifica tipos en tiempo de compilación.
  • Facilitan la organización en archivos de encabezado (.h).

Si omitimos el prototipo, el compilador asume que la función devuelve
int y no verifica los parámetros (esto es peligroso y obsoleto).

--------------------------------------------------------------------------------
  // MAL: sin prototipo, el compilador asume int implícitamente
  #include <stdio.h>

  int main(void) {
      double r = dividir(10.0, 3.0); // comportamiento indefinido
      printf("%f\n", r);
      return 0;
  }

  double dividir(double a, double b) {
      return a / b;
  }
--------------------------------------------------------------------------------

Este código puede compilar con advertencias pero el resultado será
incorrecto porque el compilador trata dividir() como int -> int.

================================================================================
3. TIPO DE RETORNO
================================================================================

El tipo de retorno puede ser cualquier tipo de datos de C:
  • Tipos básicos: int, char, float, double
  • Modificados: long, unsigned, short
  • void (sin retorno)
  • Punteros: int*, char*, etc.
  • Estructuras y uniones
  • Tipos definidos con typedef

Función con tipo de retorno int:

--------------------------------------------------------------------------------
  int obtener_edad(void) {
      return 25;
  }
--------------------------------------------------------------------------------

Función con retorno float:

--------------------------------------------------------------------------------
  float promedio(int a, int b) {
      return (a + b) / 2.0f;
  }
--------------------------------------------------------------------------------

Función con retorno char:

--------------------------------------------------------------------------------
  char primera_letra(const char* str) {
      return str[0];
  }
--------------------------------------------------------------------------------

Función void (sin retorno):

--------------------------------------------------------------------------------
  void mostrar_menu(void) {
      printf("=== MENU PRINCIPAL ===\n");
      printf("1. Nueva partida\n");
      printf("2. Cargar\n");
      printf("3. Salir\n");
  }
--------------------------------------------------------------------------------

Si una función declarada con retorno no void no ejecuta return,
el comportamiento es indefinido:

--------------------------------------------------------------------------------
  // ERROR GRAVE: falta return
  int cuadrado(int x) {
      int r = x * x;
      // falta return r;
  }

  int main(void) {
      int v = cuadrado(5);
      printf("%d\n", v); // basura
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
4. FUNCIONES DE BIBLIOTECA VS. FUNCIONES DE USUARIO
================================================================================

FUNCIONES DE BIBLIOTECA:
  Vienen en archivos de encabezado estándar:
  • <stdio.h>:  printf, scanf, fopen, fclose...
  • <stdlib.h>: malloc, free, atoi, rand, exit...
  • <string.h>: strlen, strcpy, strcmp, strcat...
  • <math.h>:   sqrt, pow, sin, cos, floor, ceil...
  • <ctype.h>:  isdigit, isalpha, toupper, tolower...

FUNCIONES DE USUARIO:
  Las que nosotros escribimos para organizar nuestro código.

Ejemplo combinando ambas:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <ctype.h>
  #include <string.h>

  // función de usuario
  int contar_vocales(const char* str) {
      int cont = 0;
      for (int i = 0; str[i] != '\0'; i++) {
          char c = tolower(str[i]); // función de biblioteca
          if (c == 'a' || c == 'e' || c == 'i' ||
              c == 'o' || c == 'u') {
              cont++;
          }
      }
      return cont;
  }

  int main(void) {
      char texto[100];
      printf("Ingrese texto: ");
      fgets(texto, sizeof(texto), stdin);

      int vocales = contar_vocales(texto);
      printf("Vocales encontradas: %d\n", vocales);
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
5. FLUJO DE LLAMADA: EL STACK DE EJECUCIÓN
================================================================================

Cuando se llama a una función, ocurre lo siguiente:

  1. Se evalúan los argumentos.
  2. Se reserva un marco (stack frame) en la pila.
  3. Se guarda la dirección de retorno (siguiente instrucción).
  4. Los parámetros se copian en el marco.
  5. Se transfiere el control a la función.
  6. Se ejecuta el cuerpo.
  7. Se almacena el valor de retorno.
  8. Se libera el marco de la pila.
  9. Se retoma la ejecución en la dirección guardada.

Visualización:

--------------------------------------------------------------------------------
  Pila (stack) durante la ejecución:

  Antes de llamar a f():
  +-------------------+
  |   main()          |
  +-------------------+

  Al llamar a f():
  +-------------------+
  |   f(): marco      |
  |   - parámetros    |
  |   - variables     |
  |   - dir. retorno  |
  +-------------------+
  |   main()          |
  +-------------------+

  Al retornar de f():
  +-------------------+
  |   main()          |
  +-------------------+
--------------------------------------------------------------------------------

Código de ejemplo:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int duplicar(int x) {
      int temp = x * 2;  // variable local en el marco de duplicar
      return temp;
  }

  int main(void) {
      int a = 10;
      int b = duplicar(a); // a se copia al parámetro x
      printf("%d\n", b);   // 20
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
6. FUNCIONES CON VARIOS PARÁMETROS
================================================================================

Una función puede tener cero o más parámetros.

CERO PARÁMETROS (se usa void):

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <time.h>

  void imprimir_timestamp(void) {
      time_t t = time(NULL);
      printf("Timestamp: %ld\n", t);
  }
--------------------------------------------------------------------------------

VARIOS PARÁMETROS:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdbool.h>

  bool es_valido(int edad, float peso, float altura) {
      if (edad < 18) return false;
      if (peso <= 0) return false;
      if (altura <= 0) return false;
      return true;
  }

  int main(void) {
      if (es_valido(25, 70.5, 1.75)) {
          printf("Datos válidos\n");
      }
      return 0;
  }
--------------------------------------------------------------------------------

ORDEN DE EVALUACIÓN:
  El estándar de C no especifica el orden en que se evalúan los
  argumentos. Esto puede generar comportamientos inesperados:

--------------------------------------------------------------------------------
  int f(int a, int b) { return a + b; }

  int x = 0;
  printf("%d\n", f(++x, ++x));   // COMPORTAMIENTO INDEFINIDO
  // No se sabe si x se incrementa 1 o 2 veces antes de cada uso
--------------------------------------------------------------------------------

NUNCA modifiques una variable dos veces entre dos puntos de secuencia.

================================================================================
7. FUNCIONES DENTRO DE FUNCIONES (ANIDADAS NO)
================================================================================

C NO PERMITE definir funciones dentro de otras funciones.
Todo debe estar en el ámbito global. Lo que sí se puede es
LLAMAR a una función desde otra.

--------------------------------------------------------------------------------
  #include <stdio.h>

  int suma(int a, int b) {
      return a + b;
  }

  void operacion(void) {
      int r = suma(10, 20); // llamada válida
      printf("Resultado: %d\n", r);
  }
--------------------------------------------------------------------------------

No existe la función anidada como en Python o Pascal:

--------------------------------------------------------------------------------
  // ESTO NO COMPILA
  void externa(void) {
      void interna(void) { // error de sintaxis
          printf("interna\n");
      }
      interna();
  }
--------------------------------------------------------------------------------

================================================================================
8. FUNCIONES Y ARCHIVOS MÚLTIPLES
================================================================================

En proyectos reales, las funciones se distribuyen en archivos:

  main.c          → función main()
  operaciones.c   → implementación de funciones
  operaciones.h   → prototipos (header)

operaciones.h:
--------------------------------------------------------------------------------
  #ifndef OPERACIONES_H
  #define OPERACIONES_H

  int sumar(int a, int b);
  int restar(int a, int b);
  int multiplicar(int a, int b);
  double dividir(int a, int b);

  #endif
--------------------------------------------------------------------------------

operaciones.c:
--------------------------------------------------------------------------------
  #include "operaciones.h"

  int sumar(int a, int b) { return a + b; }
  int restar(int a, int b) { return a - b; }
  int multiplicar(int a, int b) { return a * b; }
  double dividir(int a, int b) { return (double)a / b; }
--------------------------------------------------------------------------------

main.c:
--------------------------------------------------------------------------------
  #include <stdio.h>
  #include "operaciones.h"

  int main(void) {
      printf("%d\n", sumar(10, 5));
      printf("%d\n", restar(10, 5));
      printf("%d\n", multiplicar(10, 5));
      printf("%.2f\n", dividir(10, 5));
      return 0;
  }
--------------------------------------------------------------------------------

Compilación:
  gcc -c operaciones.c   → operaciones.o
  gcc -c main.c         → main.o
  gcc main.o operaciones.o -o programa

O en un solo paso:
  gcc main.c operaciones.c -o programa

================================================================================
9. FUNCIONES INLINE (C99 en adelante)
================================================================================

La sugerencia inline le pide al compilador que expanda el código
de la función en el punto de llamada para evitar overhead.

--------------------------------------------------------------------------------
  #include <stdio.h>

  static inline int max(int a, int b) {
      return (a > b) ? a : b;
  }

  int main(void) {
      int m = max(100, 200); // el compilador puede expandir como:
                              // int m = (100 > 200) ? 100 : 200;
      printf("Max: %d\n", m);
      return 0;
  }
--------------------------------------------------------------------------------

inline es una SUGERENCIA, no una orden. El compilador decide.
Útil para funciones muy pequeñas y llamadas frecuentes.

================================================================================
10. FUNCIONES Y EL CALIFICADOR CONST
================================================================================

Podemos usar const para indicar que un parámetro no será modificado:

--------------------------------------------------------------------------------
  #include <stdio.h>

  void imprimir_mensaje(const char* mensaje) {
      // mensaje[0] = 'X'; // ERROR: const no permite modificar
      printf("%s\n", mensaje);
  }

  int main(void) {
      const char* saludo = "Hola, C Master!";
      imprimir_mensaje(saludo);
      return 0;
  }
--------------------------------------------------------------------------------

const en parámetros mejora la seguridad y documenta la intención.

================================================================================
11. FUNCIONES SIN PARÁMETROS: VOID EXPLÍCITO
================================================================================

En C, una función declarada con () vacío NO especifica que no
tiene parámetros. Solo dice "no se especifican los parámetros".
Para decir explícitamente "sin parámetros", usamos void:

--------------------------------------------------------------------------------
  // AMBIGUO: no especifica parámetros
  int funcion();

  // EXPLÍCITO: sin parámetros
  int funcion(void);
--------------------------------------------------------------------------------

Ejemplo del peligro:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int f();      // sin especificar

  int main(void) {
      printf("%d\n", f(10, 20, 30)); // compila aunque no espera args
      return 0;
  }

  int f(void) {
      return 42;
  }
--------------------------------------------------------------------------------

El código compila (quizás con advertencia) porque f() sin void
no especifica la lista de parámetros. Siempre usar void.

================================================================================
12. BUENAS PRÁCTICAS CON FUNCIONES
================================================================================

  1. Una función debe hacer una sola cosa (principio SRP).
  2. El nombre debe reflejar claramente su propósito.
  3. Limitar el número de parámetros (ideal ≤ 4).
  4. Siempre prototipar antes de usar.
  5. Documentar el propósito, parámetros y retorno.
  6. Preferir funciones cortas (≤ 30 líneas).
  7. Usar const para parámetros de solo lectura.
  8. Inicializar siempre las variables locales.
  9. Evitar efectos secundarios no documentados.
  10. Mantener la cohesión: todo en la función debe estar
      relacionado con su propósito.

Ejemplo de función bien diseñada:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdbool.h>
  #include <ctype.h>

  /**
   * Verifica si una cadena contiene solo dígitos.
   * @param str Cadena a verificar.
   * @return true si todos los caracteres son dígitos.
   */
  bool es_numerico(const char* str) {
      if (str == NULL || str[0] == '\0') return false;
      for (int i = 0; str[i] != '\0'; i++) {
          if (!isdigit(str[i])) return false;
      }
      return true;
  }

  int main(void) {
      printf("%d\n", es_numerico("12345"));   // 1
      printf("%d\n", es_numerico("12a45"));   // 0
      printf("%d\n", es_numerico(""));        // 0
      return 0;
  }
--------------------------------------------------------------------------------

RESUMEN DEL CAPÍTULO
================================================================================

  • Una función es un bloque de código reutilizable con nombre.
  • Tiene tipo de retorno, nombre y parámetros.
  • El prototipo declara la función sin definirla.
  • void como retorno indica que no devuelve nada.
  • void como parámetro indica que no recibe argumentos.
  • Las funciones se organizan en la pila (stack).
  • C no permite funciones anidadas.
  • inline sugiere expansión en el punto de llamada.
  • Usar const en parámetros de solo lectura.
  • Los prototipos van en archivos .h separados.

================================================================================
PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Puedo tener una función dentro de otra función?
  R: No. C no permite definiciones anidadas. Todas las funciones
     deben estar en el ámbito global. Lo que sí puedes es LLAMAR
     a una función desde otra.

  P: ¿Qué pasa si llamo a una función antes de declararla?
  R: En C moderno (C99+), el compilador emite un error. En modos
     antiguos, asume que devuelve int y genera código incorrecto.

  P: ¿Cuántos parámetros puede tener una función?
  R: El estándar no fija un límite teórico, pero en la práctica
     más de 6-7 parámetros indica mal diseño. Prefiere usar
     una estructura.

  P: ¿Qué diferencia hay entre definir y declarar una función?
  R: Definir es escribir el cuerpo completo. Declarar (prototipo)
     es solo el encabezado con punto y coma. La declaración le
     dice al compilador "esto existe, confía en mí".

  P: ¿Una función void puede tener return?
  R: Sí, puede tener return; (sin valor) para salir antes.
     return con valor es error de compilación en funciones void.

  P: ¿Qué es el stack frame?
  R: Es el bloque de memoria que se reserva en la pila para una
     llamada a función. Contiene parámetros, variables locales
     y la dirección de retorno. Se libera al retornar.

  P: ¿inline siempre acelera el programa?
  R: No. inline elimina overhead de llamada pero puede aumentar
     el tamaño del código (bloat). Útil solo para funciones
     muy pequeñas y llamadas frecuentes.
