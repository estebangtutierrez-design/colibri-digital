================================================================================
1. RECURSIÓN DE COLA (TAIL RECURSION)
================================================================================

Una función es de cola cuando la llamada recursiva es la ÚLTIMA
operación que realiza la función (no hay más instrucciones después).

RECURSIÓN DE COLA:

--------------------------------------------------------------------------------
  void contar(int n) {
      if (n == 0) return;
      printf("%d ", n);
      contar(n - 1); // última operación → tail recursion
  }

  int main(void) {
      contar(5); // 5 4 3 2 1
      return 0;
  }
--------------------------------------------------------------------------------

RECURSIÓN NO-COLAR (la llamada no es la última operación):

--------------------------------------------------------------------------------
  int factorial(int n) {
      if (n <= 1) return 1;
      return n * factorial(n - 1); // multiplicación DESPUES de la
                                     // llamada → NO es tail
  }
  // Equivalente a:
  // int temp = factorial(n - 1);
  // return n * temp; // operacion pendiente despues de llamada
--------------------------------------------------------------------------------

En factorial, después de factorial(n-1) aún falta multiplicar
por n, por lo que no es recursión de cola.

La recursión de cola es importante porque el compilador puede
optimizarla (TCO: Tail Call Optimization) reutilizando el marco
de pila actual en lugar de crear uno nuevo.

================================================================================
2. CONVERSIÓN A RECURSIÓN DE COLA
--------------------------------------------------------------------------------

Podemos reescribir funciones para que sean de cola usando un
ACUMULADOR (parámetro extra que acumula el resultado).

FACTORIAL CON RECURSIÓN DE COLA:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int factorial_tail(int n, int acumulador) {
      if (n <= 1) return acumulador;
      return factorial_tail(n - 1, n * acumulador);
      // La llamada es la última operación
  }

  // Función envoltorio
  int factorial(int n) {
      return factorial_tail(n, 1);
  }

  int main(void) {
      for (int i = 0; i <= 10; i++) {
          printf("%d! = %d\n", i, factorial(i));
      }
      return 0;
  }
--------------------------------------------------------------------------------

FIBONACCI CON RECURSIÓN DE COLA:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int fib_tail(int n, int a, int b) {
      if (n == 0) return a;
      if (n == 1) return b;
      return fib_tail(n - 1, b, a + b);
  }

  int fib(int n) {
      return fib_tail(n, 0, 1);
  }

  int main(void) {
      for (int i = 0; i <= 15; i++) {
          printf("F(%d) = %d\n", i, fib(i));
      }
      return 0;
  }
--------------------------------------------------------------------------------

Comparación de eficiencia:

  Fibonacci tradicional:   O(2^n),  fib(40) → ~330M llamadas
  Fibonacci cola:          O(n),    fib(40) → 40 llamadas

La versión de cola es TAN eficiente como la iterativa.

SUMA DE ARREGLO CON RECURSIÓN DE COLA:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int suma_tail(const int arr[], int n, int acum) {
      if (n <= 0) return acum;
      return suma_tail(arr + 1, n - 1, acum + arr[0]);
  }

  int suma_arr(const int arr[], int n) {
      return suma_tail(arr, n, 0);
  }

  int main(void) {
      int datos[] = {10, 20, 30, 40, 50};
      printf("Suma: %d\n", suma_arr(datos, 5)); // 150
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
3. TAIL CALL OPTIMIZATION (TCO)
--------------------------------------------------------------------------------

Cuando el compilador detecta recursión de cola, puede aplicar
TCO: en lugar de crear un nuevo stack frame, reutiliza el actual.

SIN TCO (normal):
  factorial(5)
    → factorial(4)  [nuevo frame]
      → factorial(3)  [nuevo frame]
        → ...

CON TCO (optimizado):
  factorial(5)
    → (reutiliza frame) → factorial(4)
    → (reutiliza frame) → factorial(3)
    → ...

El resultado es que la recursión de cola es TAN eficiente como
la iteración en términos de uso de pila.

COMPROBAR SI EL COMPILADOR APLICA TCO:

Compila con optimización:
  gcc -O2 -S programa.c   # genera ensamblador
  gcc -O2 -foptimize-sibling-calls programa.c

En GCC, TCO se activa con -O2, -O3, o -Os. Con -O0 no se activa.

DEMOSTRACIÓN PRÁCTICA (probando stack overflow):

--------------------------------------------------------------------------------
  #include <stdio.h>

  // Sin TCO (potencial stack overflow)
  int suma_rec(int n) {
      if (n == 0) return 0;
      return n + suma_rec(n - 1); // NO tail
  }

  // Con TCO (sin stack overflow con optimizacion)
  int suma_tco(int n, int acum) {
      if (n == 0) return acum;
      return suma_tco(n - 1, acum + n); // tail
  }

  int main(void) {
      // Probar con un valor grande
      // Con -O0: ambas pueden fallar
      // Con -O2: suma_tco no debe fallar
      // printf("%d\n", suma_rec(100000));   // probable overflow
      // printf("%d\n", suma_tco(100000, 0)); // OK con TCO
      return 0;
  }
--------------------------------------------------------------------------------

Nota: Los compiladores de C NO ESTÁN OBLIGADOS a aplicar TCO.
Depende de la implementación y el nivel de optimización.

================================================================================
4. RECURSIÓN DE ÁRBOL (TREE RECURSION)
--------------------------------------------------------------------------------

Una función con recursión de árbol se llama a sí misma MÚLTIPLES
veces en cada paso recursivo, formando un árbol de llamadas.

FIBONACCI TRADICIONAL (recursión de árbol):

--------------------------------------------------------------------------------
  int fib_arbol(int n) {
      if (n == 0) return 0;
      if (n == 1) return 1;
      return fib_arbol(n - 1) + fib_arbol(n - 2); // dos llamadas
  }
  // Arbol de llamadas: O(2^n)
--------------------------------------------------------------------------------

RECORRIDO DE ÁRBOL BINARIO (recursión de árbol natural):

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdlib.h>

  typedef struct Nodo {
      int dato;
      struct Nodo* izq;
      struct Nodo* der;
  } Nodo;

  Nodo* crear_nodo(int dato) {
      Nodo* n = malloc(sizeof(Nodo));
      n->dato = dato;
      n->izq = n->der = NULL;
      return n;
  }

  // Recorrido inorder (árbol recursivo natural)
  void inorder(Nodo* raiz) {
      if (raiz == NULL) return;
      inorder(raiz->izq);        // llamada recursiva 1
      printf("%d ", raiz->dato);
      inorder(raiz->der);        // llamada recursiva 2
  }

  // Preorder
  void preorder(Nodo* raiz) {
      if (raiz == NULL) return;
      printf("%d ", raiz->dato);
      preorder(raiz->izq);
      preorder(raiz->der);
  }

  // Postorder
  void postorder(Nodo* raiz) {
      if (raiz == NULL) return;
      postorder(raiz->izq);
      postorder(raiz->der);
      printf("%d ", raiz->dato);
  }

  int altura(Nodo* raiz) {
      if (raiz == NULL) return -1;
      int alt_izq = altura(raiz->izq);
      int alt_der = altura(raiz->der);
      return 1 + (alt_izq > alt_der ? alt_izq : alt_der);
  }

  void liberar_arbol(Nodo* raiz) {
      if (raiz == NULL) return;
      liberar_arbol(raiz->izq);
      liberar_arbol(raiz->der);
      free(raiz);
  }

  int main(void) {
      Nodo* raiz = crear_nodo(10);
      raiz->izq = crear_nodo(5);
      raiz->der = crear_nodo(15);
      raiz->izq->izq = crear_nodo(3);
      raiz->izq->der = crear_nodo(7);
      raiz->der->izq = crear_nodo(12);
      raiz->der->der = crear_nodo(18);

      printf("Inorder: ");   inorder(raiz);   printf("\n");
      printf("Preorder: ");  preorder(raiz);  printf("\n");
      printf("Postorder: "); postorder(raiz); printf("\n");
      printf("Altura: %d\n", altura(raiz));

      liberar_arbol(raiz);
      return 0;
  }
--------------------------------------------------------------------------------

5. DIVIDE Y VENCERÁS RECURSIVO
--------------------------------------------------------------------------------

Los algoritmos divide y vencerás son naturalmente recursivos:
  1. Dividir el problema en subproblemas más pequeños.
  2. Conquistar recursivamente cada subproblema.
  3. Combinar las soluciones.

MERGE SORT:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdlib.h>
  #include <string.h>

  void merge(int arr[], int izq, int medio, int der) {
      int n1 = medio - izq + 1;
      int n2 = der - medio;

      int* L = malloc(n1 * sizeof(int));
      int* R = malloc(n2 * sizeof(int));

      memcpy(L, arr + izq, n1 * sizeof(int));
      memcpy(R, arr + medio + 1, n2 * sizeof(int));

      int i = 0, j = 0, k = izq;
      while (i < n1 && j < n2) {
          if (L[i] <= R[j]) arr[k++] = L[i++];
          else              arr[k++] = R[j++];
      }
      while (i < n1) arr[k++] = L[i++];
      while (j < n2) arr[k++] = R[j++];

      free(L);
      free(R);
  }

  void merge_sort(int arr[], int izq, int der) {
      if (izq < der) {
          int medio = izq + (der - izq) / 2;
          merge_sort(arr, izq, medio);      // dividir
          merge_sort(arr, medio + 1, der);  // dividir
          merge(arr, izq, medio, der);      // combinar
      }
  }

  int main(void) {
      int arr[] = {38, 27, 43, 3, 9, 82, 10};
      int n = sizeof(arr) / sizeof(arr[0]);

      merge_sort(arr, 0, n - 1);

      for (int i = 0; i < n; i++)
          printf("%d ", arr[i]);
      printf("\n");
      return 0;
  }
--------------------------------------------------------------------------------

QUICK SORT:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int particionar(int arr[], int bajo, int alto) {
      int pivote = arr[alto];
      int i = bajo - 1;

      for (int j = bajo; j < alto; j++) {
          if (arr[j] < pivote) {
              i++;
              int temp = arr[i];
              arr[i] = arr[j];
              arr[j] = temp;
          }
      }
      int temp = arr[i + 1];
      arr[i + 1] = arr[alto];
      arr[alto] = temp;
      return i + 1;
  }

  void quick_sort(int arr[], int bajo, int alto) {
      if (bajo < alto) {
          int pi = particionar(arr, bajo, alto);
          quick_sort(arr, bajo, pi - 1);
          quick_sort(arr, pi + 1, alto);
      }
  }

  int main(void) {
      int arr[] = {10, 7, 8, 9, 1, 5};
      int n = sizeof(arr) / sizeof(arr[0]);

      quick_sort(arr, 0, n - 1);

      for (int i = 0; i < n; i++)
          printf("%d ", arr[i]);
      printf("\n");
      return 0;
  }
--------------------------------------------------------------------------------

6. RECURSIÓN vs ITERACIÓN: ANÁLISIS PROFUNDO
--------------------------------------------------------------------------------

FACTORIAL: TRES VERSIONES

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <time.h>

  int fact_no_tail(int n) {
      if (n <= 1) return 1;
      return n * fact_no_tail(n - 1);
  }

  int fact_tail(int n, int acc) {
      if (n <= 1) return acc;
      return fact_tail(n - 1, n * acc);
  }

  int fact_iter(int n) {
      int r = 1;
      for (int i = 2; i <= n; i++) r *= i;
      return r;
  }

  #define MEDIR(f, n, veces) do { \
      clock_t start = clock(); \
      for (int i = 0; i < veces; i++) (void)f(n); \
      printf("%s: %ld ms\n", #f, (clock()-start)*1000/CLOCKS_PER_SEC); \
  } while(0)

  int main(void) {
      int n = 12;
      int veces = 10000000;

      MEDIR(fact_iter, n, veces);
      MEDIR(fact_tail, n, veces);
      MEDIR(fact_no_tail, n, veces);

      return 0;
  }
--------------------------------------------------------------------------------

Resultados típicos (con -O2):
  fact_iter:    ~50 ms
  fact_tail:    ~50 ms (TCO aplicado)
  fact_no_tail: ~120 ms (no puede aplicar TCO)

Sin optimización (-O0):
  fact_iter:    ~80 ms
  fact_tail:    ~200 ms (sin TCO, paga overhead de llamadas)
  fact_no_tail: ~250 ms

CUÁNDO USAR CADA UNA:

  ITERACIÓN:
    • Cuando la eficiencia es crítica.
    • Bucles simples (for, while).
    • Cuando la profundidad podría ser grande.

  RECURSIÓN (normal):
    • Problemas naturalmente recursivos.
    • Árboles, grafos, backtracking.
    • Código más legible y cercano a la definición.

  RECURSIÓN DE COLA:
    • Como替代 a iteración en lenguaje funcional.
    • Cuando esperas TCO del compilador.
    • Transformación directa de problema recursivo.

7. RECURSIÓN Y MEMORIA: EL HEAP COMO ALTERNATIVA
--------------------------------------------------------------------------------

Para recursión muy profunda, podemos simular la pila en el heap
(usando un arreglo o una pila explícita). Esto evita stack overflow.

RECURSIÓN vs. PILA EXPLÍCITA (DFS en árbol):

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdlib.h>

  typedef struct Nodo { int dato; struct Nodo* izq; struct Nodo* der; } Nodo;

  // Recursivo (natural pero limitado por profundidad)
  void dfs_rec(Nodo* raiz) {
      if (raiz == NULL) return;
      printf("%d ", raiz->dato);
      dfs_rec(raiz->izq);
      dfs_rec(raiz->der);
  }

  // Iterativo con pila explicita (control total)
  void dfs_iter(Nodo* raiz) {
      if (raiz == NULL) return;
      Nodo** pila = malloc(1000 * sizeof(Nodo*));
      int tope = 0;
      pila[tope++] = raiz;

      while (tope > 0) {
          Nodo* actual = pila[--tope];
          printf("%d ", actual->dato);
          if (actual->der) pila[tope++] = actual->der;
          if (actual->izq) pila[tope++] = actual->izq;
      }
      free(pila);
  }
  // La version iterativa puede manejar arboles mucho mas profundos
  // porque usa heap en lugar de stack.
--------------------------------------------------------------------------------

8. RECURSIÓN MÚTUA AVANZADA
--------------------------------------------------------------------------------

MÁQUINA DE ESTADOS CON RECURSIÓN MÚTUA:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdbool.h>
  #include <ctype.h>

  void estado_espacios(const char**, int*);
  void estado_palabra(const char**, int*);

  void estado_espacios(const char** str, int* cont) {
      while (**str != '\0' && isspace(**str)) (*str)++;
      if (**str == '\0') return;
      (*cont)++;
      estado_palabra(str, cont);
  }

  void estado_palabra(const char** str, int* cont) {
      while (**str != '\0' && !isspace(**str)) (*str)++;
      estado_espacios(str, cont);
  }

  int contar_palabras(const char* str) {
      int cont = 0;
      estado_espacios(&str, &cont);
      return cont;
  }

  int main(void) {
      printf("%d\n", contar_palabras("  Hola  mundo  C  ")); // 3
      printf("%d\n", contar_palabras(""));                   // 0
      printf("%d\n", contar_palabras("  uno  "));            // 1
      return 0;
  }

9. BACKTRACKING AVANZADO: LAS 8 REINAS
--------------------------------------------------------------------------------

Problema clásico de backtracking recursivo:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdbool.h>
  #include <stdlib.h>

  #define N 8

  void imprimir_tablero(int tablero[N][N]) {
      for (int i = 0; i < N; i++) {
          for (int j = 0; j < N; j++)
              printf("%c ", tablero[i][j] ? 'Q' : '.');
          printf("\n");
      }
      printf("\n");
  }

  bool es_seguro(int tablero[N][N], int fila, int col) {
      for (int i = 0; i < col; i++)
          if (tablero[fila][i]) return false;

      for (int i = fila, j = col; i >= 0 && j >= 0; i--, j--)
          if (tablero[i][j]) return false;

      for (int i = fila, j = col; i < N && j >= 0; i++, j--)
          if (tablero[i][j]) return false;

      return true;
  }

  bool resolver(int tablero[N][N], int col) {
      if (col >= N) return true;

      for (int i = 0; i < N; i++) {
          if (es_seguro(tablero, i, col)) {
              tablero[i][col] = 1;
              if (resolver(tablero, col + 1))
                  return true;
              tablero[i][col] = 0; // backtrack
          }
      }
      return false;
  }

  int main(void) {
      int tablero[N][N] = {0};

      if (resolver(tablero, 0)) {
          printf("Solucion para %d reinas:\n", N);
          imprimir_tablero(tablero);
      } else {
          printf("No hay solucion\n");
      }
      return 0;
  }

10. RECURSIÓN CON GENERACIÓN DE PERMUTACIONES
--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
  #include <stdio.h>

  void intercambiar(char* a, char* b) {
      char t = *a; *a = *b; *b = t;
  }

  void permutar(char* arr, int inicio, int fin) {
      if (inicio == fin) {
          printf("%s\n", arr);
          return;
      }
      for (int i = inicio; i <= fin; i++) {
          intercambiar(&arr[inicio], &arr[i]);
          permutar(arr, inicio + 1, fin);
          intercambiar(&arr[inicio], &arr[i]); // backtrack
      }
  }

  int main(void) {
      char str[] = "ABC";
      int n = 0;
      while (str[n] != '\0') n++;

      printf("Permutaciones de %s:\n", str);
      permutar(str, 0, n - 1);
      return 0;
  }
--------------------------------------------------------------------------------

11. RECURSIÓN EN GRÁFICOS: FLOOD FILL
--------------------------------------------------------------------------------

El algoritmo de relleno por inundación es recursivo y se usa en
editores de imágenes.

--------------------------------------------------------------------------------
  #include <stdio.h>

  #define FILAS 5
  #define COLS 5

  void flood_fill(int img[FILAS][COLS], int x, int y,
                  int color_viejo, int color_nuevo) {
      if (x < 0 || x >= FILAS || y < 0 || y >= COLS)
          return;
      if (img[x][y] != color_viejo)
          return;

      img[x][y] = color_nuevo;

      flood_fill(img, x + 1, y, color_viejo, color_nuevo);
      flood_fill(img, x - 1, y, color_viejo, color_nuevo);
      flood_fill(img, x, y + 1, color_viejo, color_nuevo);
      flood_fill(img, x, y - 1, color_viejo, color_nuevo);
  }

  void imprimir_img(int img[FILAS][COLS]) {
      for (int i = 0; i < FILAS; i++) {
          for (int j = 0; j < COLS; j++)
              printf("%d ", img[i][j]);
          printf("\n");
      }
  }

  int main(void) {
      int imagen[FILAS][COLS] = {
          {1, 1, 1, 2, 2},
          {1, 1, 1, 2, 2},
          {1, 0, 1, 2, 2},
          {2, 2, 2, 2, 2},
          {2, 2, 2, 0, 0}
      };

      printf("Original:\n");
      imprimir_img(imagen);

      flood_fill(imagen, 0, 0, 1, 9);

      printf("\nRellenado (1 -> 9):\n");
      imprimir_img(imagen);
      return 0;
  }
--------------------------------------------------------------------------------

12. CRITERIOS PARA DECIDIR: RECURSIÓN vs ITERACIÓN
--------------------------------------------------------------------------------

USA RECURSIÓN CUANDO:
  • El problema se define naturalmente en términos recursivos.
  • La profundidad es pequeña y predecible.
  • El código iterativo sería demasiado complejo (árboles, Hanoi).
  • La claridad del código es más importante que la eficiencia.
  • Backtracking o divide y vencerás.

USA ITERACIÓN CUANDO:
  • El problema es un simple bucle.
  • La profundidad de recursión puede ser grande.
  • La eficiencia es crítica (embebido, tiempo real).
  • El compilador no aplica TCO.
  • El mantenimiento de la pila es costoso.

COMPLEJIDADES COMUNES:

  Algoritmo          Temporal     Espacial (pila)
  factorial rec      O(n)          O(n)
  factorial tail     O(n)          O(1) con TCO
  factorial iter     O(n)          O(1)
  fibonacci rec      O(2^n)        O(n)
  fibonacci tail     O(n)          O(1) con TCO
  fibonacci iter     O(n)          O(1)
  merge sort         O(n log n)    O(n) + O(log n) pila
  quick sort         O(n log n)    O(log n) pila (promedio)
  Hanoi              O(2^n)        O(n)
  árbol inorder      O(n)          O(h) h = altura

================================================================================
RESUMEN DEL CAPÍTULO
================================================================================

  • Recursión de cola: la llamada recursiva es la última operación.
  • TCO (Tail Call Optimization) permite reutilizar stack frames.
  • La recursión de cola es tan eficiente como la iteración.
  • Recursión de árbol: múltiples llamadas por paso (Fibonacci).
  • Divide y vencerás: merge sort, quick sort.
  • Backtracking: 8 reinas, permutaciones.
  • La pila explícita en heap evita stack overflow.
  • Convertir recursión a iteración: usar acumuladores.
  • La elección recursión vs iteración depende del contexto.

PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Qué compiladores soportan TCO?
  R: GCC y Clang con -O2, -O3, -Os. MSVC no siempre lo hace.
     ICC (Intel) también lo soporta.

  P: ¿Cómo sé si mi compilador aplicó TCO?
  R: Compila con -S y revisa el ensamblador. Busca si hay
     instrucciones de CALL o JMP. Con TCO, verás JMP en lugar
     de CALL. También puedes probar con una recursión muy
     profunda y ver si hay stack overflow.

  P: ¿Cuándo NO se aplica TCO?
  R: Cuando hay operaciones después de la llamada, cuando la
     función es variádica, o cuando hay destructores (en C++).
     En C puro, si es tail, se puede aplicar.

  P: ¿La recursión de cola es siempre mejor?
  R: Con TCO, sí. Sin TCO, es igual que cualquier recursión
     (sigue creando stack frames). La ventaja es conceptual:
     puedes expresar el problema recursivamente con eficiencia
     iterativa.

  P: ¿Cómo convierto cualquier recursión a cola?
  R: Identifica qué información necesita persistir entre
     llamadas y pásala como parámetro acumulador. El caso base
     retorna el acumulador.

  P: ¿Cuál es la profundidad máxima de recursión segura?
  R: Generalmente unos 10,000 a 100,000 frames (depende del
     tamaño del frame y el stack del SO). Con TCO, no hay
     límite práctico.

  P: ¿Qué es mejor, recursión de cola o iteración directa?
  R: Iteración directa es más explícita. Recursión de cola es
     más declarativa. Elige según el estilo del equipo y el
     contexto del problema.

DESAFÍO DEL ARQUITECTO
================================================================================

  Implementa un solucionador de Sudoku usando backtracking
  recursivo. Requisitos:

    • Tablero 9x9 representado como int[9][9] (0 = vacío).
    • Función bool es_valido(int tablero[9][9], int fila,
      int col, int num) que verifique si num puede colocarse
      en (fila, col) según reglas del Sudoku.
    • Función bool resolver(int tablero[9][9]) que use
      backtracking recursivo para encontrar la solución.
    • Función void imprimir_tablero(int tablero[9][9])
    • main() debe:
        - Definir un tablero inicial con algunos valores.
        - Mostrar el tablero inicial.
        - Resolverlo y mostrar la solución.
        - Indicar si tiene solución o no.

  Puntos extra:
    - Heurística: elegir la celda con menos opciones primero
      (MRV - Minimum Remaining Values).
    - Leer el tablero desde un archivo.
    - Contar cuántas recursiones se realizaron.
    - Generar tableros válidos aleatorios.

  El objetivo es dominar el backtracking recursivo con poda
  (pruning) para reducir el espacio de búsqueda.

================================================================================
CIERRE DEL CAPÍTULO
================================================================================

  La recursión avanzada es una herramienta de pensamiento tanto
  como de programación. La recursión de cola, el divide y
  vencerás, y el backtracking son patrones que aparecen en
  innumerables problemas. Elegir sabiamente entre recursión
  e iteración, y optimizar cuando sea necesario, es la marca
  del programador maduro.
