Bienvenido al Capítulo 26. Los punteros son la característica
más poderosa (y peligrosa) de C. Un puntero es una variable que
almacena una dirección de memoria. Dominar los punteros es lo que
separa al programador de C del verdadero maestro.

"Los punteros son como cuchillos: útiles si sabes usarlos,
peligrosos si no." — Anónimo

================================================================================
OBJETIVOS DEL CAPÍTULO
================================================================================

  • Comprender qué es un puntero y cómo se representa en memoria.
  • Declarar e inicializar punteros correctamente.
  • Usar los operadores & (dirección) y * (indirección).
  • Realizar aritmética de punteros.
  • Usar NULL para punteros seguros.
  • Relacionar punteros con arreglos.
  • Evitar errores clásicos (punteros colgantes, fugas).

================================================================================
1. ¿QUÉ ES UN PUNTERO?
================================================================================

Un puntero es una variable que contiene una dirección de memoria.
En sistemas de 64 bits, ocupa 8 bytes; en 32 bits, 4 bytes.

MEMORIA (visual):

  Dirección    Contenido
  0x7fff0000   42         ← int x = 42;
  0x7fff0008   0x7fff0000 ← int* p = &x;
                           El puntero p guarda la dirección de x

DECLARACIÓN:

  tipo* nombre;       // estilo 1 (preferido)
  tipo *nombre;       // estilo 2
  tipo * nombre;      // estilo 3

Los tres significan lo mismo: "nombre es un puntero a tipo".

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int x = 42;
      int* p = &x;  // p apunta a x

      printf("Valor de x: %d\n", x);       // 42
      printf("Direccion de x: %p\n", (void*)&x);
      printf("Valor de p: %p\n", (void*)p);
      printf("Valor apuntado por p: %d\n", *p); // 42

      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
2. OPERADOR & (DIRECCIÓN)
================================================================================

El operador & obtiene la dirección de memoria de una variable.

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int a = 10;
      char b = 'Z';
      double c = 3.14;

      printf("Direccion de a: %p\n", (void*)&a);
      printf("Direccion de b: %p\n", (void*)&b);
      printf("Direccion de c: %p\n", (void*)&c);

      // Las direcciones suelen diferir en el tamaño del tipo
      return 0;
  }
--------------------------------------------------------------------------------

& NO se puede usar con:
  • Constantes (5, 'a', 3.14) → &5 es un error.
  • Expresiones aritméticas → &(a + b) es un error.
  • Variables register → register int x; &x es error.

Sí se puede usar con:
  • Variables (cualquier tipo).
  • Elementos de arreglos (&arr[0]).
  • Campos de estructuras (&s.campo).
  • Funciones (el nombre es la dirección).

================================================================================
3. OPERADOR * (INDIRECCIÓN / DEREFERENCIA)
================================================================================

El operador * accede al valor almacenado en la dirección que
contiene el puntero. Se llama "dereferenciar" o "indirectar".

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int x = 100;
      int* p = &x;

      // Leer a través del puntero
      printf("x via p: %d\n", *p); // 100

      // Modificar a través del puntero
      *p = 200;  // ahora x = 200
      printf("x modificado: %d\n", x); // 200

      // Equivalencia: *p es lo mismo que x
      *p = *p + 50;
      printf("x: %d\n", x); // 250

      return 0;
  }
--------------------------------------------------------------------------------

DEREFERENCIA MÚLTIPLE (puntero a puntero):

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int x = 42;
      int* p = &x;    // puntero a int
      int** pp = &p;  // puntero a puntero a int

      printf("x = %d\n", x);
      printf("*p = %d\n", *p);
      printf("**pp = %d\n", **pp); // 42

      **pp = 100; // modifica x a través de pp
      printf("x = %d\n", x); // 100

      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
4. EL PUNTERO NULL
================================================================================

NULL es un valor especial que indica "puntero que no apunta a
nada". Está definido en <stddef.h>, <stdio.h>, <stdlib.h>, etc.

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int* p = NULL; // inicialización segura

      // SIEMPRE verificar antes de dereferenciar
      if (p != NULL) {
          printf("%d\n", *p);
      } else {
          printf("Puntero NULL, no se puede dereferenciar\n");
      }

      // Dereferenciar NULL → SEGMENTATION FAULT
      // int x = *p; // ¡CRASH!
      return 0;
  }
--------------------------------------------------------------------------------

BUENAS PRÁCTICAS:
  • Inicializar punteros a NULL si no tienen dirección válida.
  • Verificar que no sean NULL antes de usarlos.
  • Después de free(), asignar NULL al puntero.
  • En parámetros de funciones, verificar NULL al inicio.

#define NULL vs nullptr (C23):
  En C23 se introduce nullptr (similar a C++), pero en C11/C17
  se usa NULL (definido como ((void*)0) o simplemente 0).

================================================================================
5. PUNTEROS Y ARREGLOS
================================================================================

En C, el nombre de un arreglo es un puntero constante al primer
elemento. arr es equivalente a &arr[0].

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int arr[5] = {10, 20, 30, 40, 50};

      printf("arr:     %p\n", (void*)arr);
      printf("&arr[0]: %p\n", (void*)&arr[0]);
      // Son la misma dirección

      // Acceso mediante índice
      printf("arr[2] = %d\n", arr[2]);    // 30

      // Acceso mediante puntero (equivalente)
      printf("*(arr+2) = %d\n", *(arr + 2)); // 30

      return 0;
  }
--------------------------------------------------------------------------------

ARR[I] ES AZÚCAR SINTÁCTICO PARA *(ARR + I):
  • arr[i] ≡ *(arr + i)
  • &arr[i] ≡ arr + i

DEMOSTRACIÓN:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int nums[] = {5, 10, 15, 20, 25};
      int* p = nums; // p apunta al primer elemento

      for (int i = 0; i < 5; i++) {
          printf("nums[%d] = %d  *(p+%d) = %d  p[%d] = %d\n",
                  i, nums[i],
                  i, *(p + i),
                  i, p[i]); // p[i] también funciona!
      }
      return 0;
  }
--------------------------------------------------------------------------------

DIFERENCIA CLAVE: sizeof:
  • sizeof(arr) → tamaño total del arreglo (20 bytes para int[5]).
  • sizeof(p)   → tamaño del puntero (8 bytes en 64 bits).

ARR NO ES MODIFICABLE:
  El nombre del arreglo es un puntero constante. arr++ no está
  permitido. Pero p++, sí.

--------------------------------------------------------------------------------
  // arr++; // ERROR: arr es constante
  p++; // OK: p es una variable puntero
  printf("Ahora p apunta a: %d\n", *p); // 20 (segundo elemento)
--------------------------------------------------------------------------------

================================================================================
6. ARITMÉTICA DE PUNTEROS
--------------------------------------------------------------------------------

La aritmética de punteros escala según el tamaño del tipo al
que apunta. p + 1 avanza sizeof(tipo) bytes, no 1 byte.

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int arr[] = {10, 20, 30, 40, 50};
      int* p = arr;

      printf("p     = %p\n", (void*)p);
      printf("p + 1 = %p\n", (void*)(p + 1));
      printf("Diferencia: %ld bytes\n",
             (char*)(p + 1) - (char*)p); // 4 (sizeof(int))

      // Recorrer el arreglo con aritmética
      for (int i = 0; i < 5; i++) {
          printf("%d ", *(p + i));
      }
      printf("\n");
      return 0;
  }
--------------------------------------------------------------------------------

ARITMÉTICA CON DIFERENTES TIPOS:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      char c_arr[] = "Hola";
      int  i_arr[] = {1, 2, 3, 4};
      double d_arr[] = {1.0, 2.0, 3.0};

      char* cp = c_arr;
      int* ip = i_arr;
      double* dp = d_arr;

      printf("char:   %p -> %p (salto %ld)\n",
             (void*)cp, (void*)(cp + 1), (long)(cp + 1) - (long)cp);
      printf("int:    %p -> %p (salto %ld)\n",
             (void*)ip, (void*)(ip + 1), (long)(ip + 1) - (long)ip);
      printf("double: %p -> %p (salto %ld)\n",
             (void*)dp, (void*)(dp + 1), (long)(dp + 1) - (long)dp);
      return 0;
  }
--------------------------------------------------------------------------------

OPERACIONES PERMITIDAS:
  • p + n, p - n: avanzar/retroceder n elementos.
  • p++: avanzar al siguiente elemento.
  • p--: retroceder al elemento anterior.
  • p1 - p2: distancia (en elementos) entre dos punteros.
  • p1 == p2, p1 != p2: comparación de igualdad.
  • p1 < p2, p1 > p2: comparación relacional (mismo arreglo).

OPERACIONES NO PERMITIDAS:
  • p + p (suma de dos punteros).
  • p / n, p % n.
  • Multiplicación de punteros.

RESTA DE PUNTEROS:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int arr[] = {10, 20, 30, 40, 50};
      int* inicio = &arr[0];
      int* fin = &arr[4];

      printf("Elementos entre inicio y fin: %ld\n", fin - inicio); // 4

      // Recorrido con punteros
      for (int* p = inicio; p <= fin; p++) {
          printf("%d ", *p);
      }
      printf("\n");
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
7. PUNTEROS A DIFERENTES TIPOS
--------------------------------------------------------------------------------

PUNTERO A CHAR (STRING):

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      const char* mensaje = "Hola, mundo!";
      // mensaje es un puntero al primer carácter

      while (*mensaje != '\0') {
          putchar(*mensaje);
          mensaje++;
      }
      printf("\n");

      // Equivalente con índice:
      // for (int i = 0; mensaje[i] != '\0'; i++)
      //     putchar(mensaje[i]);
      return 0;
  }
--------------------------------------------------------------------------------

PUNTERO A DOUBLE:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      double valores[] = {3.14, 2.71, 1.41, 1.73};
      double* p = valores;
      double suma = 0.0;

      for (int i = 0; i < 4; i++) {
          suma += *(p + i);
      }
      printf("Suma: %.2f\n", suma);
      return 0;
  }
--------------------------------------------------------------------------------

PUNTERO A VOID (void*):

  void* es un puntero genérico. Puede apuntar a cualquier tipo.
  NO se puede dereferenciar directamente (necesita casteo).

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int x = 42;
      double y = 3.14;
      void* ptr;

      ptr = &x; // void* apunta a int
      printf("int: %d\n", *(int*)ptr); // casteo necesario

      ptr = &y; // void* apunta a double
      printf("double: %.2f\n", *(double*)ptr);

      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
8. PUNTEROS COLGANTES (DANGLING POINTERS)
--------------------------------------------------------------------------------

Un puntero colgante apunta a memoria que ya no es válida.
Dereferenciarlo es COMPORTAMIENTO INDEFINIDO.

CAUSAS COMUNES:

  1. Devolver puntero a variable local.

--------------------------------------------------------------------------------
  int* colgante(void) {
      int local = 42;
      return &local; // local se destruye al retornar
  }

  int main(void) {
      int* p = colgante();
      // printf("%d\n", *p); // COMPORTAMIENTO INDEFINIDO
      return 0;
  }
--------------------------------------------------------------------------------

  2. Usar puntero después de free().

--------------------------------------------------------------------------------
  #include <stdlib.h>

  int main(void) {
      int* p = malloc(sizeof(int));
      *p = 100;
      free(p);
      // p ahora es colgante
      // *p = 200; // COMPORTAMIENTO INDEFINIDO
      p = NULL; // buena práctica
      return 0;
  }
--------------------------------------------------------------------------------

  3. Puntero a memoria fuera del ámbito.

--------------------------------------------------------------------------------
  int* salir_del_bloque(void) {
      int arr[10];
      return arr; // arr se destruye al salir
  }
--------------------------------------------------------------------------------

CÓMO EVITAR PUNTEROS COLGANTES:
  • Inicializar punteros a NULL.
  • Asignar NULL después de free().
  • No retornar direcciones de variables automáticas.
  • Usar herramientas como Valgrind, AddressSanitizer.

================================================================================
9. PUNTEROS A ESTRUCTURAS
--------------------------------------------------------------------------------

Los punteros a estructuras son muy comunes. Usan el operador ->
para acceder a miembros.

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <string.h>

  typedef struct {
      char nombre[50];
      int edad;
  } Persona;

  int main(void) {
      Persona p1;
      Persona* ptr = &p1;

      // Acceso mediante ->
      strcpy(ptr->nombre, "Lucas");
      ptr->edad = 30;

      // Equivalente a (*ptr).nombre, pero más limpio
      printf("Nombre: %s, Edad: %d\n", ptr->nombre, ptr->edad);

      // ptr->nombre  ≡  (*ptr).nombre
      // ptr->edad    ≡  (*ptr).edad

      return 0;
  }
--------------------------------------------------------------------------------

PUNTEROS DINÁMICOS A ESTRUCTURAS:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdlib.h>
  #include <string.h>

  typedef struct {
      char* nombre;
      int edad;
  } Persona;

  Persona* crear_persona(const char* nombre, int edad) {
      Persona* p = malloc(sizeof(Persona));
      if (p == NULL) return NULL;
      p->nombre = malloc(strlen(nombre) + 1);
      if (p->nombre == NULL) {
          free(p);
          return NULL;
      }
      strcpy(p->nombre, nombre);
      p->edad = edad;
      return p;
  }

  void destruir_persona(Persona* p) {
      if (p == NULL) return;
      free(p->nombre);
      free(p);
  }

  int main(void) {
      Persona* p = crear_persona("Ana", 25);
      if (p) {
          printf("%s: %d\n", p->nombre, p->edad);
          destruir_persona(p);
      }
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
10. ARREGLOS DE PUNTEROS
--------------------------------------------------------------------------------

Podemos tener arreglos donde cada elemento es un puntero.

ARREGLO DE STRINGS:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      // Arreglo de punteros a char (cada uno apunta a un string)
      const char* nombres[] = {
          "Lucas",
          "Ana",
          "Carlos",
          "Maria"
      };
      int n = sizeof(nombres) / sizeof(nombres[0]);

      for (int i = 0; i < n; i++) {
          printf("nombres[%d] = %s (addr: %p)\n",
                  i, nombres[i], (void*)nombres[i]);
      }
      return 0;
  }
--------------------------------------------------------------------------------

ARREGLO DE PUNTEROS A INT:

--------------------------------------------------------------------------------
  #include <stdio.h>

  int main(void) {
      int a = 10, b = 20, c = 30;
      int* ptrs[] = {&a, &b, &c};

      for (int i = 0; i < 3; i++) {
          printf("*ptrs[%d] = %d\n", i, *ptrs[i]);
      }

      // Modificar via puntero
      *ptrs[1] = 200;
      printf("b ahora es: %d\n", b); // 200
      return 0;
  }
--------------------------------------------------------------------------------

================================================================================
11. PUNTEROS A PUNTEROS
--------------------------------------------------------------------------------

Se usan para modificar un puntero desde una función (necesario
para listas enlazadas, árboles, etc.).

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdlib.h>

  void asignar_memoria(int** pp, int size) {
      *pp = malloc(size * sizeof(int));
      if (*pp != NULL) {
          (*pp)[0] = 42; // inicializar
      }
  }

  int main(void) {
      int* p = NULL;
      asignar_memoria(&p, 10);

      if (p != NULL) {
          printf("p[0] = %d\n", p[0]); // 42
          free(p);
      }
      return 0;
  }
--------------------------------------------------------------------------------

PUNTERO A PUNTERO PARA MODIFICAR PUNTERO EN FUNCIÓN:

--------------------------------------------------------------------------------
  #include <stdio.h>
  #include <stdlib.h>

  void redimensionar(int** arr, int nuevo_size) {
      int* temp = realloc(*arr, nuevo_size * sizeof(int));
      if (temp != NULL) {
          *arr = temp;
      }
  }

  int main(void) {
      int* datos = malloc(5 * sizeof(int));
      for (int i = 0; i < 5; i++) datos[i] = i * 10;

      redimensionar(&datos, 10);
      datos[9] = 999;

      for (int i = 0; i < 10; i++)
          printf("%d ", datos[i]);
      printf("\n");

      free(datos);
      return 0;
  }
--------------------------------------------------------------------------------

12. DEBUGGING CON PUNTEROS: ERRORES COMUNES
--------------------------------------------------------------------------------

ERROR 1: No inicializar el puntero.

--------------------------------------------------------------------------------
  int main(void) {
      int* p; // sin inicializar (basura)
      *p = 10; // COMPORTAMIENTO INDEFINIDO (probable crash)
      return 0;
  }
--------------------------------------------------------------------------------

ERROR 2: Confundir * en declaración vs * en uso.

--------------------------------------------------------------------------------
  int* p, q;  // p es int*, q es int (NO int*)
  // Correcto: int *p, *q;
--------------------------------------------------------------------------------

ERROR 3: Olvidar los paréntesis con estructuras.

--------------------------------------------------------------------------------
  typedef struct { int x, y; } Punto;
  Punto p = {10, 20};
  Punto* ptr = &p;
  // *ptr.x  → *(ptr.x) → ERROR: ptr no tiene campo x
  // (*ptr).x → correcto
  // ptr->x  → correcto (preferido)
--------------------------------------------------------------------------------

ERROR 4: Buffer overflow con aritmética.

--------------------------------------------------------------------------------
  int arr[5];
  int* p = arr;
  *(p + 100) = 0; // escribe fuera del arreglo: ¡peligro!
--------------------------------------------------------------------------------

ERROR 5: Comparar punteros de distintos arreglos.

--------------------------------------------------------------------------------
  int a[5], b[5];
  int* pa = a;
  int* pb = b;
  // if (pa < pb) // COMPORTAMIENTO INDEFINIDO (no es el mismo
  //             // objeto agregado)
--------------------------------------------------------------------------------

================================================================================
RESUMEN DEL CAPÍTULO
================================================================================

  • Un puntero almacena una dirección de memoria.
  • & obtiene la dirección de una variable.
  • * dereferencia un puntero (accede al valor apuntado).
  • arr[i] ≡ *(arr + i).
  • La aritmética de punteros escala según sizeof(tipo).
  • NULL = puntero que no apunta a nada.
  • void* es un puntero genérico (necesita casteo para usar).
  • -> accede a miembros de struct vía puntero.
  • Nunca dereferenciar punteros sin inicializar o NULL.
  • Asignar NULL después de free().

================================================================================
PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Cuánto ocupa un puntero?
  R: En sistemas de 64 bits, 8 bytes. En 32 bits, 4 bytes.
     Independientemente del tipo al que apunte.

  P: ¿Qué es más rápido, acceso por índice o por puntero?
  R: Los compiladores modernos generan el mismo código para
     arr[i] y *(arr + i). Usa el que sea más legible.

  P: ¿Por qué se usa void*?
  R: Para punteros genéricos. malloc() retorna void*.
     Las funciones qsort() y bsearch() lo usan. Permite
     escribir código que trabaja con cualquier tipo.

  P: ¿Qué pasa si dereferencio NULL?
  R: Comportamiento indefinido que típicamente resulta en
     una violación de segmento (segmentation fault) y el
     programa termina.

  P: ¿Se puede sumar dos punteros?
  R: No. No tiene sentido sumar direcciones. Sí se pueden
     restar para obtener la distancia en elementos.

  P: ¿arr y &arr son lo mismo?
  R: Casi. arr es puntero a arr[0]. &arr es puntero al
     arreglo completo (tipo int(*)[5]). Apuntan a la misma
     dirección pero tienen tipos diferentes.

  P: ¿Qué es un puntero a puntero?
  R: Una variable que almacena la dirección de otro puntero.
     Se usa para modificar punteros desde funciones y para
     arreglos bidimensionales dinámicos.

        - Ingresar dos matrices 3x3.

      dobles (int**).
