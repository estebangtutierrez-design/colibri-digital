  "La búsqueda es la operación gemela del ordenamiento. Para buscar
   rápido, primero ordena. Para ordenar rápido, primero entiende
   cómo buscar."
                                                    — Donald E. Knuth

--------------------------------------------------------------------------------

  ▸ BIENVENIDA

  Una vez que tenemos datos almacenados en arrays, la operación más
  común después del ordenamiento es la búsqueda. ¿Cómo encontrar un
  elemento específico? ¿Existe? ¿En qué posición está?

  Existen dos enfoques fundamentales:
  • Búsqueda Lineal (secuencial): recorre todo el array hasta encontrar
    el elemento. Funciona con arrays ordenados o desordenados.
  • Búsqueda Binaria: divide el array ordenado por la mitad en cada
    paso. Es mucho más rápida, pero requiere que el array esté ordenado.

  En este capítulo implementaremos ambos algoritmos, analizaremos su
  eficiencia y exploraremos variantes como la búsqueda de primeros/
  últimos/contadores de ocurrencias.

  Aprenderás a elegir el algoritmo adecuado según el contexto y a
  implementar búsquedas en arrays de enteros, strings y estructuras.

  --- 1. BÚSQUEDA LINEAL (SECUENCIAL) ---

  Es el algoritmo más simple: recorre el array desde el inicio hasta
  el final, comparando cada elemento con el valor buscado.

  Algoritmo:
  1. Para i desde 0 hasta n-1:
     a. Si arr[i] == valor, devuelve i.
  2. Devuelve -1 (no encontrado).

  Implementación:

    int busqueda_lineal(int arr[], int n, int valor) {
        for (int i = 0; i < n; i++) {
            if (arr[i] == valor)
                return i;
        }
        return -1;
    }

  Con punteros:

    int *busqueda_lineal_p(int *inicio, int *fin, int valor) {
        while (inicio < fin) {
            if (*inicio == valor)
                return inicio;
            inicio++;
        }
        return NULL;
    }

  Complejidad:
  • Peor caso: O(n) — el elemento está al final o no existe.
  • Mejor caso: O(1) — el elemento está en la primera posición.
  • Promedio: O(n) — recorre n/2 elementos en promedio.
  • Espacio: O(1).

  Ventajas: no requiere ordenamiento, funciona con cualquier tipo.
  Desventajas: lenta para arrays grandes.

  --- 2. BÚSQUEDA LINEAL EN STRINGS ---

    int busqueda_str(char *arr[], int n, const char *clave) {
        for (int i = 0; i < n; i++) {
            if (strcmp(arr[i], clave) == 0)
                return i;
        }
        return -1;
    }

  Búsqueda parcial (por prefijo):

    int busqueda_prefijo(char *arr[], int n, const char *prefijo) {
        size_t len = strlen(prefijo);
        for (int i = 0; i < n; i++) {
            if (strncmp(arr[i], prefijo, len) == 0)
                return i;
        }
        return -1;
    }

  --- 3. BÚSQUEDA BINARIA ---

  La búsqueda binaria (binary search) es un algoritmo extremadamente
  eficiente que funciona dividiendo repetidamente el array ordenado
  por la mitad y determinando en qué mitad debe estar el elemento.

  Requisito FUNDAMENTAL: el array debe estar ORDENADO.

  Algoritmo:
  1. Establece izq = 0, der = n - 1.
  2. Mientras izq <= der:
     a. medio = (izq + der) / 2.
     b. Si arr[medio] == valor, devuelve medio.
     c. Si arr[medio] < valor, izq = medio + 1 (busca en la mitad derecha).
     d. Si arr[medio] > valor, der = medio - 1 (busca en la mitad izquierda).
  3. Devuelve -1 (no encontrado).

  Visualización: buscar 7 en [1, 3, 5, 7, 9, 11, 13]

    izq=0, der=6 → medio=3 → arr[3]=7 → ¡encontrado!

  Buscar 6 en [1, 3, 5, 7, 9, 11, 13]:

    izq=0, der=6 → medio=3 → arr[3]=7 > 6 → der=2
    izq=0, der=2 → medio=1 → arr[1]=3 < 6 → izq=2
    izq=2, der=2 → medio=2 → arr[2]=5 < 6 → izq=3
    izq=3 > der=2 → no encontrado, devuelve -1

  Implementación iterativa:

    int busqueda_binaria(int arr[], int n, int valor) {
        int izq = 0, der = n - 1;
        while (izq <= der) {
            int medio = izq + (der - izq) / 2;  // evitar desbordamiento
            if (arr[medio] == valor)
                return medio;
            if (arr[medio] < valor)
                izq = medio + 1;
            else
                der = medio - 1;
        }
        return -1;
    }

  Implementación recursiva:

    int busqueda_binaria_rec(int arr[], int izq, int der, int valor) {
        if (izq > der) return -1;

        int medio = izq + (der - izq) / 2;

        if (arr[medio] == valor) return medio;
        if (arr[medio] < valor)
            return busqueda_binaria_rec(arr, medio + 1, der, valor);
        else
            return busqueda_binaria_rec(arr, izq, medio - 1, valor);
    }

  Complejidad:
  • Peor caso: O(log n) — se necesitan log₂(n) comparaciones.
  • Mejor caso: O(1) — el elemento está justo en el medio.
  • Espacio: O(1) iterativa, O(log n) recursiva (por la pila).

  Para n = 1,000,000: la búsqueda binaria necesita solo ~20 comparaciones.
  La búsqueda lineal podría necesitar 1,000,000.

  --- 4. PRIMERA Y ÚLTIMA OCURRENCIA ---

  Cuando el array tiene elementos duplicados ordenados, podemos encontrar
  la primera o última ocurrencia con variantes de la búsqueda binaria.

  Primera ocurrencia:

    int primera_ocurrencia(int arr[], int n, int valor) {
        int izq = 0, der = n - 1;
        int resultado = -1;
        while (izq <= der) {
            int medio = izq + (der - izq) / 2;
            if (arr[medio] == valor) {
                resultado = medio;
                der = medio - 1;  // sigue buscando a la izquierda
            } else if (arr[medio] < valor) {
                izq = medio + 1;
            } else {
                der = medio - 1;
            }
        }
        return resultado;
    }

  Última ocurrencia:

    int ultima_ocurrencia(int arr[], int n, int valor) {
        int izq = 0, der = n - 1;
        int resultado = -1;
        while (izq <= der) {
            int medio = izq + (der - izq) / 2;
            if (arr[medio] == valor) {
                resultado = medio;
                izq = medio + 1;  // sigue buscando a la derecha
            } else if (arr[medio] < valor) {
                izq = medio + 1;
            } else {
                der = medio - 1;
            }
        }
        return resultado;
    }

  Contar ocurrencias:

    int contar_ocurrencias(int arr[], int n, int valor) {
        int primero = primera_ocurrencia(arr, n, valor);
        if (primero == -1) return 0;
        int ultimo = ultima_ocurrencia(arr, n, valor);
        return ultimo - primero + 1;
    }

  --- 5. BÚSQUEDA BINARIA EN STRINGS ---

  Con array de strings ordenado:

    int busqueda_binaria_str(char *arr[], int n, const char *clave) {
        int izq = 0, der = n - 1;
        while (izq <= der) {
            int medio = izq + (der - izq) / 2;
            int cmp = strcmp(arr[medio], clave);
            if (cmp == 0) return medio;
            if (cmp < 0) izq = medio + 1;
            else der = medio - 1;
        }
        return -1;
    }

  --- 6. BÚSQUEDA BINARIA CON ESTRUCTURAS ---

    typedef struct {
        int id;
        char nombre[50];
    } Registro;

    int buscar_por_id(Registro arr[], int n, int id_buscar) {
        int izq = 0, der = n - 1;
        while (izq <= der) {
            int medio = izq + (der - izq) / 2;
            if (arr[medio].id == id_buscar) return medio;
            if (arr[medio].id < id_buscar) izq = medio + 1;
            else der = medio - 1;
        }
        return -1;
    }

  --- 7. BÚSQUEDA POR INTERPOLACIÓN ---

  Similar a la búsqueda binaria, pero en lugar de ir al medio, calcula
  una posición basada en el valor buscado (como buscar en un diccionario).

    int busqueda_interpolacion(int arr[], int n, int valor) {
        int izq = 0, der = n - 1;
        while (izq <= der && valor >= arr[izq] && valor <= arr[der]) {
            if (izq == der) {
                if (arr[izq] == valor) return izq;
                return -1;
            }
            int pos = izq + ((double)(der - izq) /
                      (arr[der] - arr[izq])) * (valor - arr[izq]);

            if (arr[pos] == valor) return pos;
            if (arr[pos] < valor) izq = pos + 1;
            else der = pos - 1;
        }
        return -1;
    }

  Complejidad: O(log log n) en promedio para datos uniformemente
  distribuidos. O(n) en el peor caso.

  --- 8. BÚSQUEDA CON BARRERA (SENTINEL) ---

  Optimización de la búsqueda lineal: añadir el valor buscado al final
  del array (como barrera) evita comprobar el límite en cada iteración.

    int busqueda_con_barrera(int arr[], int n, int valor) {
        int ultimo = arr[n - 1];
        arr[n - 1] = valor;  // coloca la barrera

        int i = 0;
        while (arr[i] != valor) i++;

        arr[n - 1] = ultimo;  // restaura

        if (i < n - 1 || arr[n - 1] == valor)
            return i;
        return -1;
    }

  Reduce las comparaciones de 2n a n, pero modifica temporalmente
  el array. No se usa en la práctica moderna (los compiladores
  optimizan igual).

  --- 9. bsearch — BÚSQUEDA BINARIA EN BIBLIOTECA ESTÁNDAR ---

  C proporciona bsearch en <stdlib.h> para búsqueda binaria genérica.

  Prototipo:

    void *bsearch(const void *key, const void *base,
                  size_t nmemb, size_t size,
                  int (*compar)(const void *, const void *));

  • key: puntero al valor a buscar.
  • base: puntero al array (ordenado).
  • nmemb: número de elementos.
  • size: tamaño de cada elemento.
  • compar: función de comparación (misma que qsort).

  Devuelve puntero al elemento encontrado, o NULL.

  Ejemplo con enteros:

    #include <stdio.h>
    #include <stdlib.h>

    int cmp(const void *a, const void *b) {
        return *(const int*)a - *(const int*)b;
    }

    int main() {
        int arr[] = {1, 3, 5, 7, 9, 11, 13};
        int n = sizeof(arr) / sizeof(arr[0]);
        int clave = 7;

        int *resultado = bsearch(&clave, arr, n, sizeof(int), cmp);

        if (resultado)
            printf("Encontrado: %d en posicion %td\n",
                   *resultado, resultado - arr);
        else
            printf("No encontrado\n");

        return 0;
    }

  Con strings:

    int cmp_str(const void *a, const void *b) {
        return strcmp(*(const char**)a, *(const char**)b);
    }

    int main() {
        char *palabras[] = {"abeja", "delfin", "gato", "perro"};
        int n = sizeof(palabras) / sizeof(palabras[0]);
        char *clave = "gato";

        char **resultado = bsearch(&clave, palabras, n,
                                    sizeof(char*), cmp_str);

        if (resultado)
            printf("Encontrado: %s\n", *resultado);

        return 0;
    }

  EJEMPLO 3: Contar ocurrencias en array ordenado

    #include <stdio.h>

    int primera(int arr[], int n, int v) {
        int izq = 0, der = n - 1, res = -1;
        while (izq <= der) {
            int med = izq + (der - izq) / 2;
            if (arr[med] == v) { res = med; der = med - 1; }
            else if (arr[med] < v) izq = med + 1;
            else der = med - 1;
        }
        return res;
    }

    int ultima(int arr[], int n, int v) {
        int izq = 0, der = n - 1, res = -1;
        while (izq <= der) {
            int med = izq + (der - izq) / 2;
            if (arr[med] == v) { res = med; izq = med + 1; }
            else if (arr[med] < v) izq = med + 1;
            else der = med - 1;
        }
        return res;
    }

    int main() {
        int arr[] = {1, 2, 2, 2, 3, 3, 4, 5, 5, 5, 5, 6};
        int n = sizeof(arr) / sizeof(arr[0]);

        int p = primera(arr, n, 5);
        int u = ultima(arr, n, 5);
        printf("Primer 5 en %d, ultimo en %d, total: %d\n",
               p, u, u - p + 1);

        srand(time(NULL));

        for (int i = 0; i < N; i++)
            arr[i] = rand() % 100000;

        qsort(arr, N, sizeof(int), cmp);

        int buscar = arr[N / 2];

        clock_t t1 = clock();
        for (int i = 0; i < N; i++)
            if (arr[i] == buscar) break;
        clock_t t2 = clock();

        int *r = bsearch(&buscar, arr, N, sizeof(int), cmp);
        clock_t t3 = clock();

        printf("Lineal:  %.6f s\n", (double)(t2 - t1) / CLOCKS_PER_SEC);
        printf("Binaria: %.6f s\n", (double)(t3 - t2) / CLOCKS_PER_SEC);

        free(arr);
        return 0;
    }

  --- 11. BÚSQUEDA LINEAL vs BINARIA ---

  Característica     | Lineal             | Binaria
  -------------------|--------------------|---------------------
  Requisito          | Ninguno            | Array ordenado
  Complejidad        | O(n)               | O(log n)
  n=100              | 100 comparaciones  | ~7 comparaciones
  n=10,000           | 10,000 comp.       | ~14 comparaciones
  n=1,000,000        | 1,000,000 comp.    | ~20 comparaciones
  Implementación     | Simple             | Moderada
  Tipo de datos      | Cualquiera         | Cualquiera (comparable)
  Búsqueda única     | Adecuada           | Mejor (si ya está ordenado)
  Búsquedas múltiples | Ineficiente        | Muy eficiente

  Regla práctica: si haces más de una búsqueda, ordena y usa binaria.
  Si solo buscas una vez en datos desordenados, usa lineal.

  --- 12. ERRORES COMUNES ---

  1. Usar búsqueda binaria en un array no ordenado:

     int arr[] = {5, 2, 8, 1, 9};
     bsearch(&clave, arr, 5, sizeof(int), cmp);  // ¡incorrecto!

  2. Cálculo incorrecto del medio (desbordamiento):

     int medio = (izq + der) / 2;  // puede desbordar si izq+der > INT_MAX
     int medio = izq + (der - izq) / 2;  // seguro

  3. Condición de bucle incorrecta:

     while (izq < der)  // puede salir temprano
     while (izq <= der) // correcto

  4. No actualizar correctamente izq/der:

     if (arr[med] < valor) izq = med;     // puede causar bucle infinito
     if (arr[med] < valor) izq = med + 1; // correcto

  5. Usar bsearch con tipo incorrecto en la clave:

     int clave = 42;
     bsearch(&clave, arr, n, sizeof(int), cmp);  // &clave, no clave

  6. Comparación incorrecta en strings para bsearch:

     // Incorrecto: pasar char** no char*
     char *clave = "hola";
     bsearch(clave, ...)  // mal
     bsearch(&clave, ...) // bien

  7. No verificar que bsearch devuelva NULL.

  8. En búsqueda lineal con strings, confundir == con strcmp.

  9. Usar sizeof en parámetro de array en funciones de búsqueda.

 10. Olvidar que bsearch requiere el array ordenado con la misma
     función de comparación.

--------------------------------------------------------------------------------

  ▸ RESUMEN DEL CAPÍTULO

  • Búsqueda lineal: recorre todo el array, O(n), sin requisitos.
  • Búsqueda binaria: divide el array ordenado, O(log n), muy rápida.
  • La búsqueda binaria requiere que el array esté ordenado.
  • bsearch es la función genérica de la biblioteca estándar.
  • Se pueden encontrar primera y última ocurrencia con variantes.
  • lower_bound/upper_bound encuentran puntos de inserción.
  • Para búsquedas únicas en datos pequeños, la lineal es suficiente.
  • Para múltiples búsquedas, ordena y usa binaria.
  • Siempre usa izq + (der - izq) / 2 para evitar desbordamiento.
  • La función de comparación de bsearch debe coincidir con la de qsort.

  ▸ PREGUNTAS FRECUENTES (FAQ)

  P: ¿Por qué la búsqueda binaria es tan rápida?
  R: Porque en cada paso descarta la mitad del array. Para n=1,000,000,
     solo se necesitan log₂(1,000,000) ≈ 20 comparaciones.

  P: ¿Puedo usar búsqueda binaria en un array desordenado?
  R: No. La búsqueda binaria asume que el array está ordenado. Si el
     array no está ordenado, el algoritmo puede fallar.

  P: ¿Qué es mejor, búsqueda lineal o binaria?
  R: Depende. Si el array es pequeño (<100) o solo buscas una vez, la
     lineal es más simple. Si el array es grande o buscas múltiples
     veces, la binaria es muy superior.

  P: ¿Cómo funciona bsearch?
  R: bsearch implementa búsqueda binaria genérica. Recibe un puntero
     al valor clave, el array, el número de elementos, el tamaño de
     cada elemento y una función de comparación.

  P: ¿Qué es lower_bound?
  R: lower_bound devuelve el primer índice donde se podría insertar un
     valor para mantener el orden. La primera posición donde arr[i] >=
     valor. Es útil para búsquedas por rango.

  P: ¿La búsqueda lineal puede ser más rápida que la binaria?
  R: Para arrays muy pequeños (<20 elementos), la lineal puede ser más
     rápida debido a la localidad de caché y menos sobrecarga. Pero
     para arrays grandes, la binaria gana siempre.

  P: ¿Qué pasa si hay elementos duplicados? ¿Qué índice devuelve la
     búsqueda binaria?
  R: La búsqueda binaria estándar puede devolver cualquier índice donde
     aparezca el valor. Para controlar esto, se usan las variantes de
     primera/última ocurrencia.

  P: ¿Cómo busco en un array de strings con bsearch?
  R: Necesitas pasar un puntero al puntero del string clave, y la
     función de comparación debe recibir char** y hacer el cast
     correspondiente.
