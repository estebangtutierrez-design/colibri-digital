  ▸ CONTENIDO

  --- 1. ¿QUÉ ES UN ARRAY? ---

  Un array (arreglo o vector) es una colección de elementos del mismo
  tipo de dato, almacenados en posiciones contiguas de memoria. Todos
  los elementos comparten el mismo nombre y se distinguen por un índice
  numérico.

  Características fundamentales:

  • Homogeneidad: todos los elementos son del mismo tipo.
  • Contigüidad: los elementos ocupan posiciones consecutivas en RAM.
  • Tamaño fijo: en C estático, el tamaño se define en tiempo de
    compilación y no puede cambiar.
  • Indexación: cada elemento se accede mediante un entero (índice).
  • Base cero: el primer elemento está en la posición 0.

  Representación gráfica de un array de 5 enteros:

    Índice:    0    1    2    3    4
             +----+----+----+----+----+
             | 10 | 20 | 30 | 40 | 50 |
             +----+----+----+----+----+
    Dirección: 0x100 0x104 0x108 0x10C 0x110

  Cada entero ocupa 4 bytes (en la mayoría de sistemas). Si el array
  comienza en 0x100, el elemento [0] está en 0x100, [1] en 0x104, etc.

  --- 2. DECLARACIÓN DE ARRAYS ---

  La sintaxis para declarar un array es:

    tipo nombre[tamaño];

  Donde:
  • tipo: cualquier tipo de dato de C (int, float, char, double, etc.).
  • nombre: identificador del array.
  • tamaño: constante entera positiva (número de elementos).

  Ejemplos:

    int notas[30];        // 30 enteros para notas de estudiantes
    float temperaturas[7]; // 7 floats para temperaturas de una semana
    char letras[26];       // 26 caracteres para el alfabeto
    double promedios[10];  // 10 doubles para promedios

  El tamaño debe ser una constante conocida en tiempo de compilación:

    int n = 10;
    int arr[n];  // VLA (Variable Length Array) — C99, no recomendado

  Los VLA (Variable Length Arrays) se introdujeron en C99 pero son
  opcionales desde C11. Es mejor usar constantes simbólicas:

    #define TAM 100
    int arr[TAM];

  O constantes con enum:

    enum { TAM = 100 };
    int arr[TAM];

  --- 3. INICIALIZACIÓN DE ARRAYS ---

  C ofrece varias formas de inicializar un array:

  a) Inicialización completa con lista:

    int arr[5] = {10, 20, 30, 40, 50};

  b) Inicialización parcial (el resto se rellena con 0):

    int arr[5] = {10, 20};  // arr = {10, 20, 0, 0, 0}

  c) Inicialización sin tamaño (el compilador cuenta):

    int arr[] = {10, 20, 30, 40, 50};  // tamaño = 5

  d) Inicialización con Designated Initializers (C99):

    int arr[5] = {[2] = 30, [4] = 50};  // arr = {0, 0, 30, 0, 50}

  e) Inicialización a cero:

    int arr[5] = {0};     // todos los elementos son 0
    int arr[5] = {};      // no válido en C estándar

  f) Asignación posterior (elemento por elemento):

    int arr[5];
    arr[0] = 10;
    arr[1] = 20;
    arr[2] = 30;
    arr[3] = 40;
    arr[4] = 50;

  IMPORTANTE: No se puede asignar un array completo después de declarado:

    int a[5] = {1,2,3,4,5};
    int b[5];
    b = a;  // ERROR: no se puede asignar arrays directamente

  Hay que copiar elemento por elemento o usar memcpy().

  --- 4. ACCESO A ELEMENTOS: EL ÍNDICE ---

  Se accede a un elemento del array usando el operador corchete []:

    arr[indice]

  El índice es un número entero que comienza en 0 y llega hasta
  tamaño-1.

    int arr[5] = {10, 20, 30, 40, 50};

    printf("%d\n", arr[0]);  // 10
    printf("%d\n", arr[2]);  // 30
    printf("%d\n", arr[4]);  // 50

  Se puede usar cualquier expresión entera como índice:

    int i = 3;
    printf("%d\n", arr[i]);    // 40
    printf("%d\n", arr[i-1]);  // 30
    printf("%d\n", arr[2*2]);  // 50

  El índice también se puede usar para modificar elementos:

    arr[1] = 99;  // arr = {10, 99, 30, 40, 50}

  --- 5. RECORRIDO DE ARRAYS CON BUCLES ---

  La forma más común de trabajar con arrays es recorrerlos con bucles:

  Con for:

    int arr[5] = {10, 20, 30, 40, 50};

    for (int i = 0; i < 5; i++) {
        printf("arr[%d] = %d\n", i, arr[i]);
    }

  Con while:

    int i = 0;
    while (i < 5) {
        printf("arr[%d] = %d\n", i, arr[i]);
        i++;
    }

  Para llenar un array con entrada del usuario:

    int numeros[10];
    for (int i = 0; i < 10; i++) {
        printf("Ingrese numero %d: ", i+1);
        scanf("%d", &numeros[i]);
    }

  Para sumar todos los elementos:

    int suma = 0;
    for (int i = 0; i < 10; i++) {
        suma += numeros[i];
    }
    printf("Suma total: %d\n", suma);

  --- 6. CÁLCULO DEL TAMAÑO DE UN ARRAY ---

  El operador sizeof devuelve el tamaño en bytes:

    int arr[10];
    printf("Tamaño del array: %zu bytes\n", sizeof(arr));
    printf("Tamaño de un elemento: %zu bytes\n", sizeof(arr[0]));
    printf("Número de elementos: %zu\n", sizeof(arr) / sizeof(arr[0]));

  Es muy común crear una macro para esto:

    #define LONGITUD(arr) (sizeof(arr) / sizeof(arr[0]))

    int arr[] = {1,2,3,4,5};
    for (size_t i = 0; i < LONGITUD(arr); i++) {
        printf("%d ", arr[i]);
    }

  PRECAUCIÓN: sizeof en un puntero NO devuelve el tamaño del array:

    void funcion(int arr[]) {  // arr es un puntero aquí
        printf("%zu\n", sizeof(arr));  // 8 (tamaño de un puntero)
    }

  Esto se debe a que cuando un array se pasa a una función, decae a
  puntero. Veremos esto en detalle en el Capítulo 33.

  --- 7. PELIGRO: ÍNDICES FUERA DE LÍMITES ---

  C NO realiza comprobación de límites en los arrays. Si accedes a
  una posición fuera del rango, el programa se comporta de forma
  indefinida (undefined behavior).

    int arr[5] = {1,2,3,4,5};
    arr[5] = 99;   // ¡FUERA DE LÍMITES! (índice 5, cuando el máximo es 4)
    arr[-1] = 10;  // ¡FUERA DE LÍMITES! (índice negativo)

  Consecuencias posibles:
  • Sobrescritura de otras variables del programa.
  • Corrupción de memoria.
  • Fallo de segmentación (segfault).
  • Vulnerabilidad de seguridad (buffer overflow).

  Es responsabilidad del programador asegurarse de que los índices
  estén siempre dentro del rango válido.

  --- 8. EJEMPLOS COMPLETOS ---

  EJEMPLO 1: Estadísticas básicas con arrays

    #include <stdio.h>
    #define NUM_ESTUDIANTES 5

    int main() {
        float notas[NUM_ESTUDIANTES];
        float suma = 0.0, promedio;
        float max, min;

        for (int i = 0; i < NUM_ESTUDIANTES; i++) {
            printf("Nota del estudiante %d: ", i + 1);
            scanf("%f", &notas[i]);
            suma += notas[i];
        }

        promedio = suma / NUM_ESTUDIANTES;
        max = min = notas[0];

        for (int i = 1; i < NUM_ESTUDIANTES; i++) {
            if (notas[i] > max) max = notas[i];
            if (notas[i] < min) min = notas[i];
        }

        printf("\n--- ESTADÍSTICAS ---\n");
        printf("Suma:     %.2f\n", suma);
        printf("Promedio: %.2f\n", promedio);
        printf("Máximo:   %.2f\n", max);
        printf("Mínimo:   %.2f\n", min);

        return 0;
    }

  EJEMPLO 2: Invertir un array

    #include <stdio.h>
    #define TAM 7

    int main() {
        int arr[TAM] = {1, 2, 3, 4, 5, 6, 7};
        int temp;

        printf("Array original: ");
        for (int i = 0; i < TAM; i++)
            printf("%d ", arr[i]);
        printf("\n");

        for (int i = 0; i < TAM / 2; i++) {
            temp = arr[i];
            arr[i] = arr[TAM - 1 - i];
            arr[TAM - 1 - i] = temp;
        }

        printf("Array invertido: ");
        for (int i = 0; i < TAM; i++)
            printf("%d ", arr[i]);
        printf("\n");

        return 0;
    }

  EJEMPLO 3: Contar frecuencias de notas

    #include <stdio.h>
    #define NUM_NOTAS 10

    int main() {
        int notas[NUM_NOTAS] = {7, 5, 8, 5, 9, 7, 6, 8, 7, 10};
        int frecuencias[11] = {0};  // índices 0-10

        for (int i = 0; i < NUM_NOTAS; i++) {
            frecuencias[notas[i]]++;
        }

        printf("Nota | Frecuencia\n");
        printf("-----|-----------\n");
        for (int i = 0; i <= 10; i++) {
            if (frecuencias[i] > 0) {
                printf("  %2d |     %d\n", i, frecuencias[i]);
            }
        }

        return 0;
    }

  EJEMPLO 4: Desplazamiento circular (rotación)

    #include <stdio.h>
    #define TAM 5

    int main() {
        int arr[TAM] = {1, 2, 3, 4, 5};
        int temp = arr[TAM - 1];

        for (int i = TAM - 1; i > 0; i--) {
            arr[i] = arr[i - 1];
        }
        arr[0] = temp;

        printf("Rotado a la derecha: ");
        for (int i = 0; i < TAM; i++)
            printf("%d ", arr[i]);
        printf("\n");

        return 0;
    }

  EJEMPLO 5: Eliminar duplicados (array ordenado)

    #include <stdio.h>
    #define TAM 8

    int main() {
        int arr[TAM] = {1, 2, 2, 3, 4, 4, 4, 5};
        int temp[TAM];
        int j = 0;

        for (int i = 0; i < TAM; i++) {
            if (i == 0 || arr[i] != arr[i - 1]) {
                temp[j++] = arr[i];
            }
        }

        printf("Sin duplicados: ");
        for (int i = 0; i < j; i++)
            printf("%d ", temp[i]);
        printf("\n");

        return 0;
    }

  EJEMPLO 6: Array de caracteres

    #include <stdio.h>

    int main() {
        char vocales[5] = {'a', 'e', 'i', 'o', 'u'};

        for (int i = 0; i < 5; i++) {
            printf("vocales[%d] = '%c'\n", i, vocales[i]);
        }

        return 0;
    }

  EJEMPLO 7: Buscar el valor máximo y su posición

    #include <stdio.h>
    #define TAM 10

    int main() {
        int arr[TAM] = {34, 12, 89, 5, 67, 23, 90, 1, 45, 78};
        int max = arr[0];
        int pos = 0;

        for (int i = 1; i < TAM; i++) {
            if (arr[i] > max) {
                max = arr[i];
                pos = i;
            }
        }

        printf("Máximo: %d en posición %d\n", max, pos);

        return 0;
    }

  EJEMPLO 8: Producto punto de dos vectores

    #include <stdio.h>
    #define TAM 3

    int main() {
        int a[TAM] = {1, 2, 3};
        int b[TAM] = {4, 5, 6};
        int producto = 0;

        for (int i = 0; i < TAM; i++) {
            producto += a[i] * b[i];
        }

        printf("Producto punto: %d\n", producto);  // 1*4 + 2*5 + 3*6 = 32

        return 0;
    }

  EJEMPLO 9: Fibonacci con array

    #include <stdio.h>
    #define TERMINOS 20

    int main() {
        int fib[TERMINOS];
        fib[0] = 0;
        fib[1] = 1;

        for (int i = 2; i < TERMINOS; i++) {
            fib[i] = fib[i - 1] + fib[i - 2];
        }

        printf("Serie de Fibonacci (%d términos):\n", TERMINOS);
        for (int i = 0; i < TERMINOS; i++) {
            printf("%d ", fib[i]);
        }
        printf("\n");

        return 0;
    }

  EJEMPLO 10: Array como contador de dígitos

    #include <stdio.h>

    int main() {
        int digitos[10] = {0};
        long numero = 8463291847L;

        while (numero > 0) {
            digitos[numero % 10]++;
            numero /= 10;
        }

        printf("Frecuencia de dígitos:\n");
        for (int i = 0; i < 10; i++) {
            printf("Dígito %d: %d veces\n", i, digitos[i]);
        }

        return 0;
    }

  --- 9. CONSTANTES Y TAMAÑOS FIJOS ---

  Siempre usa constantes simbólicas para los tamaños:

    #define TAM 100
    int arr[TAM];

  O enum:

    enum { MAX_ALUMNOS = 50 };
    int edades[MAX_ALUMNOS];

  Ventajas:
  • El código es más legible.
  • Si cambia el tamaño, solo se modifica un lugar.
  • Se evitan números mágicos.

  --- 10. ERRORES COMUNES ---

  1. Olvidar que los índices empiezan en 0:

     for (int i = 1; i <= TAM; i++)  // ERROR: se sale por 1
         arr[i] = i;

  2. Confundir tamaño con índice máximo:

     int arr[10];
     arr[10] = 5;  // ERROR: índice máximo es 9

  3. No inicializar el array:

     int arr[5];
     printf("%d", arr[0]);  // valor basura

  4. Usar sizeof en un parámetro de función:

     void func(int arr[]) {
         size_t n = sizeof(arr);  // ERROR: es tamaño de puntero
     }

  5. Asignar arrays con =:

     int a[5], b[5];
     a = b;  // ERROR

  6. Usar un VLA con tamaño negativo o cero:

     int n = -5;
     int arr[n];  // comportamiento indefinido

  ▸ RESUMEN DEL CAPÍTULO

  • Un array es una colección homogénea de elementos almacenados en
    posiciones contiguas de memoria.
  • Se declara como: tipo nombre[tamaño];
  • Los índices comienzan en 0 y terminan en tamaño-1.
  • Se puede inicializar con { ... } en la declaración.
  • No hay comprobación de límites en C: es responsabilidad del
    programador.
  • sizeof(arr) / sizeof(arr[0]) da el número de elementos.
  • Los arrays se recorren típicamente con bucles for.
  • Los arrays estáticos tienen tamaño fijo en tiempo de compilación.
  • Los VLA permiten tamaño variable pero no son portables.
  • Usa constantes simbólicas para definir tamaños.

  ▸ PREGUNTAS FRECUENTES (FAQ)

  P: ¿Por qué los arrays en C empiezan en 0 y no en 1?
  R: Por razones históricas y de eficiencia. El índice representa el
     desplazamiento desde la dirección base del array. arr[i] es
     equivalente a *(arr + i). Si empezara en 1, el compilador tendría
     que restar 1 en cada acceso, lo que sería ineficiente.

  P: ¿Qué pasa si uso un índice negativo?
  R: Es comportamiento indefinido. Puede acceder a memoria anterior al
     array, corromper datos o causar un segfault. Nunca uses índices
     negativos.

  P: ¿Puedo cambiar el tamaño de un array después de declararlo?
  R: No en C estático. Los arrays estáticos tienen tamaño fijo. Para
     tamaños dinámicos necesitas memoria dinámica con malloc/realloc
     (lo veremos en módulos posteriores).

  P: ¿Cuál es la diferencia entre int arr[] e int *arr?
  R: En muchos contextos son intercambiables debido al decay. Pero
     sizeof(arr) en el primer caso (dentro del ámbito de declaración)
     da el tamaño total del array, mientras que sizeof(arr) en el
     segundo da el tamaño del puntero.

  P: ¿Los VLA son seguros?
  R: No siempre. Si el tamaño es muy grande, puede desbordar la pila
     (stack overflow). Además, desde C11 son opcionales, por lo que
     no todo compilador los soporta. Es mejor evitarlos.

  P: ¿Cómo paso un array a una función?
  R: Se pasa automaticamente como puntero al primer elemento. La
     función recibe void func(int arr[]) o void func(int *arr), y
     normalmente también se pasa el tamaño como parámetro separado.

  P: ¿Qué es el buffer overflow?
  R: Es cuando se escribe más allá de los límites de un array,
     sobrescribiendo memoria adyacente. Es una de las vulnerabilidades
     de seguridad más comunes en C.

  P: ¿Puedo tener un array de tamaño 0?
  R: En C estándar, los arrays de tamaño 0 no están permitidos (aunque
     algunos compiladores los aceptan como extensión). Los VLA sí pueden
     tener tamaño 0.

  ▸ CIERRE DEL CAPÍTULO

  Has dado el primer gran paso en el mundo de las estructuras de datos
  en C. Los arrays son la herramienta fundamental para trabajar con
  colecciones de datos. Dominarlos es esencial antes de avanzar a
  conceptos más complejos.

  En el próximo capítulo exploraremos los arrays multidimensionales,
  que te permitirán trabajar con matrices, tablas y datos en 2D y 3D.
  Es como pasar de una línea recta a un plano, y luego a un cubo.

  PRACTICA, PRACTICA, PRACTICA. Los arrays son como los ladrillos de
  una construcción: cuanto mejor los manejes, más sólidos serán tus
  programas.
