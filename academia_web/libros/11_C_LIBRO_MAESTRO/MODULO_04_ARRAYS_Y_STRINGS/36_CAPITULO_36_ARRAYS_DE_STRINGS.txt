  una lista de nombres? ¿O las líneas de un archivo? ¿O los argumentos

  sus diferencias en memoria y eficiencia, y aprenderemos a trabajar
  con argumentos de línea de comandos (argc/argv). También veremos
  cómo ordenar, buscar y manipular listas de strings.

  ▸ CONTENIDO

  --- 1. ¿QUÉ ES UN ARRAY DE STRINGS? ---

  Un array de strings es una colección de strings. En C hay dos formas
  principales de implementarlo:

  1. Matriz 2D de caracteres:
       char nombres[5][20];
     • 5 strings, cada uno de hasta 19 caracteres + '\0'.
     • Memoria contigua: 5 * 20 = 100 bytes.
     • Acceso: nombres[i][j] → carácter j del string i.
     • printf("%s", nombres[i]) → imprime el string i.

  2. Array de punteros a char:
       char *nombres[5];
     • 5 punteros, cada uno apunta a un string.
     • Los strings pueden tener longitudes diferentes.
     • Memoria no necesariamente contigua.
     • Más flexible, pero requiere inicialización cuidadosa.

  --- 2. MATRIZ 2D DE CARACTERES ---

  Declaración:

    char frutas[4][10] = {
        "Manzana",
        "Banana",
        "Cereza",
        "Durazno"
    };

  En memoria: 4 filas de 10 bytes cada una.

    Fila 0: M a n z a n a \0 . .
    Fila 1: B a n a n a \0 . . .
    Fila 2: C e r e z a \0 . . .
    Fila 3: D u r a z n o \0 . .

  Ventajas:
  • Memoria contigua (buena localidad).
  • Las strings se pueden modificar individualmente.
  • No requiere gestión de memoria adicional.

  Desventajas:
  • Desperdicio de espacio si los strings son de longitudes dispares.
  • Tamaño máximo fijo por string.
  • Número de strings fijo.

  Recorrer:

    for (int i = 0; i < 4; i++)
        printf("%s\n", frutas[i]);

  Acceder a un carácter:

    printf("%c\n", frutas[0][2]);  // 'n' (tercera letra de "Manzana")

  --- 3. ARRAY DE PUNTEROS A CHAR ---

  Declaración con inicialización:

    char *colores[] = {
        "Rojo",
        "Verde",
        "Azul",
        "Amarillo"
    };

  En memoria: el array contiene punteros (8 bytes cada uno), y los
  strings literales están en una región de solo lectura.

  Ventajas:
  • No desperdicia espacio (cada string ocupa exactamente lo que necesita).
  • Fácil de reordenar (solo se mueven punteros, no datos).
  • Flexible para añadir/eliminar strings.

  Desventajas:
  • Los strings literales son inmutables (no se pueden modificar).
  • Para strings mutables, hay que asignar memoria dinámicamente.
  • Los datos no son contiguos (peor localidad de caché).

  Recorrer:

    for (int i = 0; i < 4; i++)
        printf("%s\n", colores[i]);

  --- 4. MATRIZ 2D vs ARRAY DE PUNTEROS ---

  Característica     | char arr[N][M]    | char *arr[N]
  -------------------|-------------------|-----------------------
  Memoria            | Contigua          | Dispersa
  Tamaño por string  | Fijo (M)          | Variable
  Modificable        | Sí                | Depende (literales no)
  Desperdicio        | Posible           | Mínimo
  Reordenar          | Lento (copia)     | Rápido (punteros)
  sizeof(arr)        | N*M*sizeof(char)  | N*sizeof(char*)
  Paso a función     | arr[][M]          | char **arr

  --- 5. ARRAY DE STRINGS MUTABLES (CON MEMORIA DINÁMICA) ---

  Para tener strings mutables con longitudes variables, se combina
  un array de punteros con memoria dinámica:

    #include <stdio.h>
    #include <stdlib.h>
    #include <string.h>

    int main() {
        char *nombres[3];

        nombres[0] = malloc(20);
        strcpy(nombres[0], "Ana");

        nombres[1] = malloc(30);
        strcpy(nombres[1], "Luis Fernando");

        nombres[2] = strdup("Pedro");  // strdup = malloc + strcpy

        for (int i = 0; i < 3; i++)
            printf("%s\n", nombres[i]);

        for (int i = 0; i < 3; i++)
            free(nombres[i]);

        return 0;
    }

  strdup no es parte del estándar C (es POSIX), pero muchos compiladores
  la soportan. Puedes implementarla fácilmente:

    char *mi_strdup(const char *s) {
        char *p = malloc(strlen(s) + 1);
        if (p) strcpy(p, s);
        return p;
    }

  --- 6. argc y argv — ARGUMENTOS DE LÍNEA DE COMANDOS ---

  Todo programa en C puede recibir argumentos de la línea de comandos
  mediante los parámetros de main:

    int main(int argc, char *argv[])

  • argc (argument count): número de argumentos (incluyendo el nombre
    del programa).
  • argv (argument vector): array de strings con los argumentos.

  Ejemplo: ./programa hola mundo 123

    argv[0] = "./programa"
    argv[1] = "hola"
    argv[2] = "mundo"
    argv[3] = "123"
    argc = 4

  argv[argc] es siempre NULL (centinela).

  Programa mínimo:

    #include <stdio.h>

    int main(int argc, char *argv[]) {
        printf("Nombre del programa: %s\n", argv[0]);
        printf("Argumentos: %d\n", argc - 1);

        for (int i = 1; i < argc; i++)
            printf("  argv[%d] = %s\n", i, argv[i]);

        return 0;
    }

  argv también se puede declarar como char **argv (es equivalente).

  --- 7. PROCESAMIENTO DE ARGUMENTOS ---

  Ejemplo: calculadora de línea de comandos

    #include <stdio.h>
    #include <stdlib.h>

    int main(int argc, char *argv[]) {
        if (argc != 4) {
            fprintf(stderr, "Uso: %s num1 operador num2\n", argv[0]);
            return 1;
        }

        double a = atof(argv[1]);
        double b = atof(argv[3]);
        double r;

        switch (argv[2][0]) {
            case '+': r = a + b; break;
            case '-': r = a - b; break;
            case 'x': r = a * b; break;
            case '/': r = a / b; break;
            default:
                fprintf(stderr, "Operador invalido: %s\n", argv[2]);
                return 1;
        }

        printf("%.2f %c %.2f = %.2f\n", a, argv[2][0], b, r);
        return 0;
    }

  --- 8. ORDENAMIENTO DE ARRAYS DE STRINGS ---

  Con array de punteros, ordenar es eficiente (solo se mueven punteros):

    #include <stdio.h>
    #include <string.h>

    void ordenar(char *arr[], int n) {
        for (int i = 0; i < n - 1; i++) {
            for (int j = i + 1; j < n; j++) {
                if (strcmp(arr[i], arr[j]) > 0) {
                    char *temp = arr[i];
                    arr[i] = arr[j];
                    arr[j] = temp;
                }
            }
        }
    }

    int main() {
        char *palabras[] = {"perro", "gato", "abeja", "delfin", "cabra"};
        int n = sizeof(palabras) / sizeof(palabras[0]);

        ordenar(palabras, n);

        for (int i = 0; i < n; i++)
            printf("%s\n", palabras[i]);

        return 0;
    }

  Con matriz 2D, el ordenamiento requiere copiar filas completas
  (más costoso). Se usa strcpy para intercambiar.

  --- 9. BÚSQUEDA EN ARRAYS DE STRINGS ---

  Búsqueda lineal:

    int buscar(char *arr[], int n, const char *clave) {
        for (int i = 0; i < n; i++)
            if (strcmp(arr[i], clave) == 0)
                return i;
        return -1;
    }

  Búsqueda binaria (array ordenado):

    int busqueda_binaria(char *arr[], int n, const char *clave) {
        int izq = 0, der = n - 1;
        while (izq <= der) {
            int medio = (izq + der) / 2;
            int cmp = strcmp(arr[medio], clave);
            if (cmp == 0) return medio;
            if (cmp < 0) izq = medio + 1;
            else der = medio - 1;
        }
        return -1;
    }

  --- 10. LEER ARCHIVO EN ARRAY DE STRINGS ---

    #include <stdio.h>
    #include <stdlib.h>
    #include <string.h>

    #define MAX_LINEAS 1000
    #define MAX_LONG 256

    int main() {
        char *lineas[MAX_LINEAS];
        int num_lineas = 0;
        char buffer[MAX_LONG];
        FILE *f = fopen("texto.txt", "r");

        if (!f) {
            perror("Error al abrir archivo");
            return 1;
        }

        while (fgets(buffer, sizeof(buffer), f) && num_lineas < MAX_LINEAS) {
            buffer[strcspn(buffer, "\n")] = '\0';
            lineas[num_lineas] = malloc(strlen(buffer) + 1);
            strcpy(lineas[num_lineas], buffer);
            num_lineas++;
        }
        fclose(f);

        for (int i = 0; i < num_lineas; i++) {
            printf("Linea %d: %s\n", i, lineas[i]);
            free(lineas[i]);
        }

        return 0;
    }

  EJEMPLO 2: Ordenar nombres (matriz 2D)

    #include <stdio.h>
    #include <string.h>

    int main() {
        char nombres[][20] = {"Carlos", "Ana", "Pedro", "Maria", "Luis"};
        int n = sizeof(nombres) / sizeof(nombres[0]);

        for (int i = 0; i < n - 1; i++) {
            for (int j = i + 1; j < n; j++) {
                if (strcmp(nombres[i], nombres[j]) > 0) {
                    char temp[20];
                    strcpy(temp, nombres[i]);
                    strcpy(nombres[i], nombres[j]);
                    strcpy(nombres[j], temp);
                }
            }
        }

        for (int i = 0; i < n; i++)
            printf("%s\n", nombres[i]);

        return 0;
    }

  EJEMPLO 5: Buscar palabra en lista

    #include <stdio.h>
    #include <string.h>

    int main() {
        char *frutas[] = {"manzana", "banana", "cereza", "durazno", "uva"};
        int n = sizeof(frutas) / sizeof(frutas[0]);
        char busqueda[30];

        printf("Que fruta buscas? ");
        fgets(busqueda, 30, stdin);
        busqueda[strcspn(busqueda, "\n")] = '\0';

        int encontrada = 0;
        for (int i = 0; i < n; i++) {
            if (strcmp(frutas[i], busqueda) == 0) {
                printf("'%s' encontrada en posicion %d\n", busqueda, i);
                encontrada = 1;
                break;
            }
        }

        if (!encontrada)
            printf("'%s' no esta en la lista\n", busqueda);

        return 0;
    }

  --- 12. argv COMO MATRIZ ---

  Aunque argv se declara como char *argv[] o char **argv, se puede
  acceder a caracteres individuales:

    int main(int argc, char *argv[]) {
        // argv[i] es un string (char*)
        // argv[i][j] es un carácter

        printf("%c\n", argv[0][0]);  // primer carácter del programa
        printf("%c\n", argv[1][2]);  // tercer carácter del primer argumento

        return 0;
  }

  argv[argc] es NULL, lo que permite recorrerlo sin argc:

    for (char **p = argv; *p; p++)
        printf("%s\n", *p);

  --- 13. ERRORES COMUNES ---

  1. Desbordar una fila en una matriz 2D:

     char nombres[5][10];
     strcpy(nombres[0], "Nombre muy largo");  // overflow

  2. Modificar strings literales en array de punteros:

     char *colores[] = {"Rojo", "Verde"};
     colores[0][0] = 'r';  // undefined behavior

  3. Olvidar liberar memoria dinámica:

     char *arr[10];
     arr[0] = malloc(100);
     // ... usar ...
     // free(arr[0]);  // olvidado → leak

  4. No verificar argc antes de usar argv:

     // Sin verificar:
     printf("%s", argv[1]);  // crash si no hay argumentos

  5. Confundir sizeof en arrays de punteros pasados a función:

     void func(char *arr[]) {
         int n = sizeof(arr) / sizeof(arr[0]);  // ERROR
     }

  6. Olvidar que strtok modifica el string al tokenizar.

  7. Usar strcpy en lugar de strncpy con filas de matriz 2D.

  8. No terminar correctamente con '\0' al construir strings.

  ▸ RESUMEN DEL CAPÍTULO

  • Hay dos formas principales de crear arrays de strings: matrices 2D
    de char (contiguas, tamaño fijo) y arrays de punteros a char
    (flexibles, tamaño variable).
  • Las matrices 2D desperdician espacio si los strings son de longitudes
    dispares, pero tienen buena localidad de memoria.
  • Los arrays de punteros son más flexibles y eficientes para ordenar.
  • argv (argument vector) y argc (argument count) son los parámetros
    de main que reciben los argumentos de línea de comandos.
  • argv[0] es el nombre del programa; argv[argc] es NULL.
  • Para strings mutables con memoria dinámica, usa malloc + strcpy.
  • strdup combina malloc y strcpy en una sola llamada.
  • Siempre libera la memoria asignada dinámicamente.
  • Para ordenar arrays de strings, strcmp + intercambio de punteros.
  • Para leer archivos en arrays de strings, fgets + malloc por línea.

  ▸ PREGUNTAS FRECUENTES (FAQ)

  P: ¿Cuándo debo usar matriz 2D y cuándo array de punteros?
  R: Usa matriz 2D cuando todos los strings tengan longitud similar y
     el número sea fijo. Usa array de punteros cuando las longitudes
     varíen mucho o necesites flexibilidad para añadir/eliminar elementos.

  P: ¿Cómo se pasa un array de strings a una función?
  R: void func(char *arr[], int n) o void func(char **arr, int n).
     En ambos casos, arr es un puntero a puntero.

  P: ¿Qué representa argc?
  R: El número de argumentos en la línea de comandos. argc siempre es
     al menos 1 (el nombre del programa).

  P: ¿argv[argc] es siempre NULL?
  R: Sí, el estándar C garantiza que argv[argc] es un puntero nulo.

  P: ¿Puedo modificar argv?
  R: Sí, puedes modificar los strings individuales (si no son literales).
     Algunos sistemas permiten modificarlos para cambiar la salida de
     ps/otros comandos, pero no es portable.

  P: ¿Cómo leo un archivo completo en un array de strings?
  R: Cuenta las líneas primero (o usa un tamaño máximo), luego asigna
     memoria para cada línea con malloc y copia el contenido.

  P: ¿Qué es strdup?
  R: Es una función POSIX que duplica un string: asigna memoria con
     malloc y copia el contenido con strcpy. No es C estándar pero
     está ampliamente disponible.

  P: ¿Cómo ordeno un array de strings sin modificar los originales?
  R: Crea un array de punteros a los strings originales y ordena los
     punteros. Los strings originales no se mueven.
