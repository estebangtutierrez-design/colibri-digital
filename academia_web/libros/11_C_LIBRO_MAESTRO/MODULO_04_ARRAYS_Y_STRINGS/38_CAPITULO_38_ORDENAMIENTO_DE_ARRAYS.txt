  ▸ BIENVENIDA

  Ordenar datos es una de las operaciones más comunes en programación.
  Ya sea ordenar nombres alfabéticamente, números de menor a mayor, o
  registros por fecha, el ordenamiento está en el corazón de innumerables
  aplicaciones.

  En este capítulo estudiaremos tres algoritmos de ordenamiento clásicos:
  • Bubble Sort (burbuja): el más simple de entender, aunque ineficiente.
  • Selection Sort (selección): intuitivo y fácil de implementar.
  • Insertion Sort (inserción): eficiente para datos casi ordenados.

  Además, exploraremos qsort, la función de ordenamiento de la biblioteca
  estándar de C, que implementa QuickSort de forma genérica.

  No solo aprenderás a ordenar, sino que entenderás cómo piensa un
  algoritmo: las comparaciones, los intercambios y la complejidad.

  --- 1. EL PROBLEMA DEL ORDENAMIENTO ---

  Dado un array de elementos, el ordenamiento consiste en reorganizarlos
  según un criterio de comparación (orden ascendente, descendente,
  alfabético, etc.).

  Entrada:  [5, 2, 8, 1, 9, 3]
  Salida:   [1, 2, 3, 5, 8, 9]  (ascendente)

  Características deseables de un algoritmo de ordenamiento:
  • Correcto: produce el orden esperado.
  • Eficiente: minimiza comparaciones e intercambios.
  • In-place: ordena sin usar memoria auxiliar significativa.
  • Estable: mantiene el orden relativo de elementos iguales.

  --- 2. BUBBLE SORT (ORDENAMIENTO DE BURBUJA) ---

  Bubble Sort funciona comparando pares adyacentes e intercambiándolos
  si están en el orden incorrecto. Cada pasada "hace burbujear" el
  elemento más grande hacia el final.

  Algoritmo:
  1. Recorre el array desde i=0 hasta n-2.
  2. Si arr[i] > arr[i+1], intercambia.
  3. Repite hasta que no haya intercambios.

  Visualización (primera pasada):
    [5, 2, 8, 1, 9] → [2, 5, 8, 1, 9] → [2, 5, 8, 1, 9] → [2, 5, 1, 8, 9] → [2, 5, 1, 8, 9]
    El 9 "burbujea" hasta el final.

  Implementación:

    void bubble_sort(int arr[], int n) {
        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
        }
    }

  Optimización: si en una pasada no hay intercambios, el array ya está
  ordenado y podemos terminar antes.

    void bubble_sort_opt(int arr[], int n) {
        for (int i = 0; i < n - 1; i++) {
            int intercambio = 0;
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                    intercambio = 1;
                }
            }
            if (!intercambio) break;
        }
    }

  Complejidad:
  • Peor caso: O(n²) (array invertido).
  • Mejor caso: O(n) (array ya ordenado, con optimización).
  • Espacio: O(1) (in-place).
  • Estable: sí (intercambia solo si es mayor, no si es igual).

  --- 3. SELECTION SORT (ORDENAMIENTO POR SELECCIÓN) ---

  Selection Sort divide el array en una parte ordenada (a la izquierda)
  y otra desordenada (a la derecha). En cada paso, encuentra el mínimo
  de la parte desordenada y lo coloca al final de la parte ordenada.

  Algoritmo:
  1. Para i desde 0 hasta n-2:
     a. Encuentra el mínimo en arr[i...n-1].
     b. Intercambia arr[i] con el mínimo.

  Visualización:
    [5, 2, 8, 1, 9] → min=1 → [1, 2, 8, 5, 9]
    [1, 2, 8, 5, 9] → min=2 → [1, 2, 8, 5, 9]
    [1, 2, 8, 5, 9] → min=5 → [1, 2, 5, 8, 9]
    [1, 2, 5, 8, 9] → min=8 → [1, 2, 5, 8, 9]

  Implementación:

    void selection_sort(int arr[], int n) {
        for (int i = 0; i < n - 1; i++) {
            int min_idx = i;
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[min_idx])
                    min_idx = j;
            }
            if (min_idx != i) {
                int temp = arr[i];
                arr[i] = arr[min_idx];
                arr[min_idx] = temp;
            }
        }
    }

  Complejidad:
  • Peor caso: O(n²).
  • Mejor caso: O(n²) (siempre hace el mismo número de comparaciones).
  • Espacio: O(1).
  • Estable: no (el intercambio puede saltar sobre elementos iguales).

  --- 4. INSERTION SORT (ORDENAMIENTO POR INSERCIÓN) ---

  Insertion Sort construye el array ordenado uno por uno, tomando
  elementos del array desordenado e insertándolos en la posición
  correcta dentro de la parte ordenada.

  Algoritmo:
  1. Para i desde 1 hasta n-1:
     a. Toma arr[i] como clave.
     b. Desplaza a la derecha todos los elementos mayores que clave.
     c. Inserta clave en el hueco creado.

  Visualización:
    [5, 2, 8, 1, 9] → clave=2 → [2, 5, 8, 1, 9]
    [2, 5, 8, 1, 9] → clave=8 → [2, 5, 8, 1, 9]
    [2, 5, 8, 1, 9] → clave=1 → [1, 2, 5, 8, 9]
    [1, 2, 5, 8, 9] → clave=9 → [1, 2, 5, 8, 9]

  Implementación:

    void insertion_sort(int arr[], int n) {
        for (int i = 1; i < n; i++) {
            int clave = arr[i];
            int j = i - 1;
            while (j >= 0 && arr[j] > clave) {
                arr[j + 1] = arr[j];
                j--;
            }
            arr[j + 1] = clave;
        }
    }

  Complejidad:
  • Peor caso: O(n²) (array invertido).
  • Mejor caso: O(n) (array ya ordenado).
  • Espacio: O(1).
  • Estable: sí (inserta después de elementos iguales).

  Insertion Sort es el más eficiente de los tres para arrays pequeños
  o casi ordenados.

  --- 5. COMPARACIÓN DE ALGORITMOS ---

  Algoritmo       | Mejor caso | Peor caso | Promedio | Estable | In-place
  ----------------|------------|-----------|----------|---------|---------
  Bubble Sort     | O(n)       | O(n²)     | O(n²)    | Sí      | Sí
  Selection Sort  | O(n²)      | O(n²)     | O(n²)    | No      | Sí
  Insertion Sort  | O(n)       | O(n²)     | O(n²)    | Sí      | Sí

  Número de operaciones para n=1000:
  • Bubble: ~500,000 comparaciones.
  • Selection: ~500,000 comparaciones, 1000 intercambios.
  • Insertion: ~250,000 en promedio.

  Para n grande (>10000), estos algoritmos son demasiado lentos.
  Ahí entran QuickSort, MergeSort y HeapSort (O(n log n)).

  --- 6. qsort — ORDENAMIENTO RÁPIDO GENÉRICO ---

  La biblioteca estándar de C proporciona qsort, que implementa
  QuickSort de forma genérica (funciona con cualquier tipo).

  Prototipo:

    void qsort(void *base, size_t nmemb, size_t size,
               int (*compar)(const void *, const void *));

  Parámetros:
  • base: puntero al inicio del array.
  • nmemb: número de elementos.
  • size: tamaño de cada elemento (sizeof).
  • compar: función de comparación.

  La función de comparación debe devolver:
  • <0 si el primer elemento va antes.
  • 0 si son iguales.
  • >0 si el primer elemento va después.

  Ordenar enteros:

    #include <stdio.h>
    #include <stdlib.h>

    int cmp_int(const void *a, const void *b) {
        int va = *(const int*)a;
        int vb = *(const int*)b;
        if (va < vb) return -1;
        if (va > vb) return 1;
        return 0;
        // o simplemente: return va - vb;
    }

    int main() {
        int arr[] = {5, 2, 8, 1, 9, 3};
        int n = sizeof(arr) / sizeof(arr[0]);

        qsort(arr, n, sizeof(int), cmp_int);

        for (int i = 0; i < n; i++)
            printf("%d ", arr[i]);
        printf("\n");

        return 0;
    }

  Orden descendente: cambiar el signo de la comparación.

  --- 7. ORDENAR STRINGS CON qsort ---

    #include <stdio.h>
    #include <stdlib.h>
    #include <string.h>

    int cmp_str(const void *a, const void *b) {
        return strcmp(*(const char**)a, *(const char**)b);
    }

    int main() {
        char *palabras[] = {"perro", "gato", "abeja", "delfin"};
        int n = sizeof(palabras) / sizeof(palabras[0]);

        qsort(palabras, n, sizeof(char*), cmp_str);

        for (int i = 0; i < n; i++)
            printf("%s\n", palabras[i]);

        return 0;
    }

  Nota: los parámetros de cmp_str son punteros a los elementos del
  array. Como cada elemento es un char*, los parámetros son char**.
  Por eso necesitamos el cast y la doble desreferencia.

    }

  EJEMPLO 1: Bubble Sort con optimización y conteo

    #include <stdio.h>

    void bubble_sort(int arr[], int n) {
        int pasadas = 0, intercambios = 0;
        for (int i = 0; i < n - 1; i++) {
            int intercambio = 0;
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int t = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = t;
                    intercambio = 1;
                    intercambios++;
                }
            }
            pasadas++;

        }
        printf("Pasadas: %d, Intercambios: %d\n", pasadas, intercambios);
    }

    int main() {
        int arr[] = {64, 34, 25, 12, 22, 11, 90};
        int n = sizeof(arr) / sizeof(arr[0]);

        bubble_sort(arr, n);
        for (int i = 0; i < n; i++) printf("%d ", arr[i]);
        printf("\n");

        return 0;
    }

  EJEMPLO 6: Medición de tiempo

    #include <stdio.h>
    #include <stdlib.h>
    #include <time.h>

    #define N 10000

    int cmp(const void *a, const void *b) {
        return *(const int*)a - *(const int*)b;
    }

    int main() {
        int arr[N];
        srand(time(NULL));

        for (int i = 0; i < N; i++)
            arr[i] = rand() % 10000;

        clock_t inicio = clock();
        qsort(arr, N, sizeof(int), cmp);
        clock_t fin = clock();

        printf("Tiempo qsort (%d elementos): %.3f segundos\n",
               N, (double)(fin - inicio) / CLOCKS_PER_SEC);

        return 0;
    }

    #include <stdio.h>
    #include <stdlib.h>
    #include <string.h>

    typedef struct {
        char nombre[30];
        int edad;
        float promedio;
    } Estudiante;

    int cmp_est(const void *a, const void *b) {
        Estudiante *ea = (Estudiante*)a;

        int cmp = strcmp(ea->nombre, eb->nombre);

        if (ea->edad != eb->edad)

        if (ea->promedio > eb->promedio) return 1;
        return 0;
    }

            {"Luis", 22, 90.0},
            {"Ana", 19, 92.0}

        int n = sizeof(e) / sizeof(e[0]);

        qsort(e, n, sizeof(Estudiante), cmp_est);

        for (int i = 0; i < n; i++)
            printf("%s, %d, %.1f\n", e[i].nombre, e[i].edad, e[i].promedio);

    }

  --- 10. CUÁNDO USAR CADA ALGORITMO ---

  • Bubble Sort: solo con fines educativos. Nunca en producción.
  • Selection Sort: cuando el número de intercambios es costoso
    (hace máximo n intercambios). No es estable.
  • Insertion Sort: arrays pequeños (<50) o casi ordenados. Es el
    mejor de los tres para datos en vivo (online sorting).
  • qsort: para cualquier otro caso. Es O(n log n) y está altamente
    optimizado.

  --- 11. ERRORES COMUNES ---

  1. Índices incorrectos en los bucles (off-by-one):

     for (int i = 0; i < n; i++)  // debe ser i < n-1 en burbuja
         for (int j = 0; j < n-1; j++)

  2. Olvidar la optimización de Bubble Sort (seguir iterando cuando
     ya está ordenado).

  3. Usar comparación incorrecta en qsort:

     arr[j] > arr[j+1] → ascendente
     arr[j] < arr[j+1] → descendente

  4. Cast incorrecto en la función de comparación de qsort:

     int cmp(const void *a, const void *b) {
         return *(int*)a - *(int*)b;  // correcto
     }

  5. No verificar que los strings estén terminados en '\0'.

  6. Usar = en lugar de == en comparaciones.

  7. Olvidar pasar sizeof(tipo) a qsort.

  8. No considerar la estabilidad cuando es importante.

  ▸ RESUMEN DEL CAPÍTULO

  • Bubble Sort: compara e intercambia adyacentes, O(n²).
  • Selection Sort: encuentra el mínimo y lo coloca al inicio, O(n²).
  • Insertion Sort: inserta cada elemento en su lugar, O(n²), pero
    O(n) en el mejor caso.
  • Los tres son in-place, pero solo Bubble e Insertion son estables.
  • qsort de la biblioteca estándar es la opción práctica para la
    mayoría de los casos.
  • qsort usa una función de comparación que devuelve <0, 0, >0.
  • Para strings, la comparación en qsort requiere doble desreferencia.
  • Insertion Sort es el mejor para arrays pequeños o casi ordenados.
  • Para grandes volúmenes de datos, se necesitan algoritmos O(n log n).

  ▸ PREGUNTAS FRECUENTES (FAQ)

  P: ¿Por qué se llama Bubble Sort?
  R: Porque los elementos más grandes "burbujean" hacia el final del
     array como burbujas en un líquido.

  P: ¿Cuál es el algoritmo de ordenamiento más rápido?
  R: Para arrays grandes, QuickSort (O(n log n) promedio) es uno de
     los más rápidos. MergeSort y HeapSort también son O(n log n).

  P: ¿Qué significa que un algoritmo sea "estable"?
  R: Que mantiene el orden relativo de elementos con claves iguales.
     Si dos elementos tienen el mismo valor, el que aparece primero en
     el original, aparece primero en el ordenado.

  P: ¿Por qué Insertion Sort es mejor que Bubble Sort?
  R: Insertion Sort hace aproximadamente la mitad de comparaciones
     que Bubble Sort para datos aleatorios, y es O(n) para datos ya
     ordenados (Bubble también puede ser O(n) con optimización).

  P: ¿Cómo funciona la función de comparación de qsort?
  R: Recibe punteros a dos elementos. Debe devolver negativo si el
     primero va antes, 0 si son iguales, positivo si el primero va
     después. Los parámetros son void* por lo que necesitas cast.

  P: ¿Puedo ordenar en orden descendente con qsort?
  R: Sí, simplemente invierte la comparación: return *(int*)b - *(int*)a.

  P: ¿Qué es la complejidad O(n²)?
  R: Significa que el tiempo de ejecución crece proporcionalmente al
     cuadrado del número de elementos. Si n se duplica, el tiempo se
     cuadruplica.

  P: ¿Debo implementar mi propio algoritmo de ordenamiento o usar qsort?
  R: Usa qsort siempre que puedas. Los algoritmos manuales son para
     aprendizaje o casos muy específicos.
