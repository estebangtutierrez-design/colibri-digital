===============================================================================
1. INTRODUCCIÓN A free()
===============================================================================

La función free() es la contraparte de malloc(). Su propósito es devolver
al sistema operativo la memoria que previamente se reservó en el heap.

Prototipo:

```c
#include <stdlib.h>

void free(void *ptr);
```

La función recibe un puntero a la memoria que se desea liberar. Este puntero
debe haber sido devuelto por malloc(), calloc() o realloc(). Si ptr es NULL,
free() no hace nada (es seguro llamar free(NULL)).

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = (int *)malloc(sizeof(int));
    if (p == NULL) {
        fprintf(stderr, "Error: no se pudo asignar memoria\n");
        return 1;
    }

    *p = 42;
    printf("Valor: %d\n", *p);

    free(p);  // Devolvemos la memoria al sistema

    return 0;
}
```

Cada llamada a malloc() reserva un bloque en el heap. free() marca ese
bloque como disponible para futuras asignaciones. El bloque no desaparece
de la memoria virtual del proceso, pero su contenido queda inválido.

Es fundamental entender que free() NO cambia el valor del puntero. El
puntero sigue apuntando a la misma dirección, pero esa dirección ya no
es válida para acceder. Por eso decimos que el puntero queda "colgante".

===============================================================================
2. EL CICLO DE VIDA DE UN BLOQUE EN EL HEAP
===============================================================================

El heap es una región de memoria administrada por el heap manager
(generalmente implementado por malloc/free en la biblioteca estándar de C).

Cuando llamas a malloc(size_t size):
  1. El heap manager busca un bloque libre lo suficientemente grande.
  2. Si no encuentra, solicita más memoria al kernel (sbrk o mmap).
  3. Marca el bloque como "en uso".
  4. Devuelve un puntero al inicio del bloque (después del header).

Cuando llamas a free(void *ptr):
  1. El heap manager recibe el puntero.
  2. Accede al header del bloque (justo antes de ptr).
  3. Marca el bloque como "libre".
  4. Opcionalmente coalesce bloques libres adyacentes.
  5. Si es posible, devuelve memoria al kernel.

Cada bloque en el heap tiene típicamente esta estructura:

  +-------------------+
  | Header (metadatos) |  ← Información de tamaño, estado, etc.
  +-------------------+
  | Datos del usuario  |  ← El puntero que devuelve malloc()
  +-------------------+

El header contiene:
  - Tamaño del bloque (incluyendo el header)
  - Flags (bloque libre/en uso)
  - Punteros a bloques anterior/siguiente (para gestión de lista libre)

#include <stdio.h>
#include <stdlib.h>

int main() {
    printf("Demostración del ciclo de vida en el heap\n\n");

    printf("1. Estado inicial del heap\n");
    printf("2. Reservamos 3 bloques:\n");

    int *a = (int *)malloc(sizeof(int));
    int *b = (int *)malloc(sizeof(int));
    int *c = (int *)malloc(sizeof(int));

    printf("   a = %p, b = %p, c = %p\n\n", (void*)a, (void*)b, (void*)c);

    printf("3. Liberamos el bloque b:\n");
    free(b);
    printf("   Bloque b liberado. a y c siguen ocupados.\n\n");

    printf("4. Solicitamos un bloque de tamaño similar:\n");
    int *d = (int *)malloc(sizeof(int));
    printf("   d = %p (debería coincidir con la dirección de b)\n", (void*)d);
    printf("   El heap manager reutilizó el bloque liberado.\n\n");

    printf("5. Liberamos todos los bloques y finalizamos.\n");
    free(a);
    free(c);
    free(d);

    return 0;
}
```

El heap manager puede reutilizar bloques liberados, lo que hace que la
memoria sea eficiente. Sin embargo, también puede fragmentarse si se
liberan y asignan bloques de diferentes tamaños de manera intercalada.

===============================================================================
3. PUNTEROS COLGANTES (DANGLING POINTERS)
===============================================================================

Un puntero colgante es un puntero que apunta a una dirección de memoria
que ya ha sido liberada. Acceder a través de un puntero colgante es
Comportamiento Indefinido (Undefined Behavior).

Ejemplo clásico:

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = (int *)malloc(sizeof(int));
    if (p == NULL) return 1;

    *p = 100;
    printf("Antes de free: %d\n", *p);

    free(p);  // Liberamos la memoria

    // ¡PELIGRO! p es ahora un puntero colgante
    printf("Después de free: %d\n", *p);  // UB: comportamiento indefinido

    *p = 200;  // UB: estamos escribiendo en memoria liberada

    return 0;
}
```

Este programa puede:
  - Funcionar "aparentemente bien" (lo peor: el error pasará desapercibido)
  - Fallar con segmentation fault
  - Sobrescribir metadatos del heap manager
  - Corromper datos de otros bloques

El comportamiento indefinido significa que ANY puede pasar. El programa
puede funcionar mil veces y fallar en la milésima primera ejecución.

Casos comunes de punteros colgantes:

1. Liberar memoria y seguir usándola (el ejemplo anterior).
2. Devolver un puntero a una variable local (stack) desde una función.
3. Dos punteros apuntan al mismo bloque y uno lo libera.

```c
#include <stdio.h>
#include <stdlib.h>

// Caso 2: Devolver puntero a variable local
int *funcion_peligrosa(void) {
    int variable_local = 42;
    return &variable_local;  // ¡ERROR! variable_local se destruye al salir
}

// Caso 3: Dos punteros al mismo bloque
int main() {
    int *ptr1 = (int *)malloc(sizeof(int));
    int *ptr2 = ptr1;  // Ambos apuntan al mismo lugar

    if (ptr1 == NULL) return 1;

    *ptr1 = 123;
    printf("ptr1: %d, ptr2: %d\n", *ptr1, *ptr2);

    free(ptr1);
    ptr1 = NULL;

    // ptr2 sigue apuntando a memoria liberada
    // printf("%d\n", *ptr2);  // UB: ptr2 es colgante

    return 0;
}
```

La solución a los punteros colgantes es simple pero requiere disciplina:
inmediatamente después de free(), asigna NULL al puntero.

===============================================================================
4. ASIGNAR NULL TRAS free()
===============================================================================

La regla de oro de la memoria dinámica:

free(ptr);
ptr = NULL;

¿Por qué es tan importante? Porque elimina de raíz el problema del puntero
colgante. Si ptr es NULL, cualquier intento de acceder a *ptr causará un
segmentation fault inmediato y detectable, en lugar de un comportamiento
indefinido impredecible.

```c
#include <stdio.h>
#include <stdlib.h>

void liberar_seguro(int **pp) {
    free(*pp);
    *pp = NULL;  // El puntero original ahora es NULL
}

int main() {
    int *datos = (int *)malloc(10 * sizeof(int));
    if (datos == NULL) return 1;

    for (int i = 0; i < 10; i++) {
        datos[i] = i * 10;
    }

    liberar_seguro(&datos);

    // Ahora datos es NULL
    if (datos == NULL) {
        printf("El puntero fue liberado correctamente.\n");
    }

    // Intentar acceder causaría un crash inmediato:
    // datos[0] = 99;  // Segmentation fault (detectable)

    return 0;
}
```

Beneficios de esta práctica:

1. **Detección temprana**: Acceder a NULL causa un crash inmediato.
2. **Seguridad en múltiples free**: free(NULL) es seguro (no hace nada).
3. **Claridad del código**: Un puntero NULL indica "no apunta a nada válido".
4. **Depuración más fácil**: Los crashes por NULL son más fáciles de rastrear.

Patrón recomendado para liberar estructuras:

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    char *nombre;
    int edad;
} Persona;

void persona_liberar(Persona *p) {
    if (p == NULL) return;
    free(p->nombre);  // Liberar campos dinámicos internos
    p->nombre = NULL;
    free(p);          // Liberar la estructura misma
    // No asignamos NULL a p porque es una copia
}

int main() {
    Persona *p = (Persona *)malloc(sizeof(Persona));
    if (p == NULL) return 1;

    p->nombre = (char *)malloc(50 * sizeof(char));
    if (p->nombre == NULL) {
        free(p);
        return 1;
    }

    strcpy(p->nombre, "Ana García");
    p->edad = 28;

    printf("Persona: %s, %d años\n", p->nombre, p->edad);

    persona_liberar(p);
    p = NULL;  // Quien llama también asigna NULL

    return 0;
}
```

Este patrón de "liberación en cascada" es esencial cuando trabajas con
estructuras anidadas que contienen memoria dinámica.

===============================================================================
5. DOBLE LIBERACIÓN (DOUBLE FREE)
===============================================================================

El double free ocurre cuando llamas a free() dos veces sobre el mismo
puntero (sin haberlo reasignado entre medias). Esto corrompe las
estructuras internas del heap manager.

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = (int *)malloc(sizeof(int));
    if (p == NULL) return 1;

    *p = 42;

    free(p);   // Primera liberación: correcta
    free(p);   // ¡DOUBLE FREE! Comportamiento indefinido

    return 0;
}
```

Consecuencias del double free:

1. El heap manager intenta marcar como libre un bloque que ya está libre.
2. Corrompe las listas internas de bloques libres.
3. futuras llamadas a malloc() pueden devolver direcciones inválidas.
4. Posible escalada a vulnerabilidad de seguridad (heap overflow).

¿Por qué es peligroso?

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    // Demostración del caos que causa un double free
    int *a = (int *)malloc(sizeof(int));
    int *b = (int *)malloc(sizeof(int));
    int *c = (int *)malloc(sizeof(int));

    *a = 10; *b = 20; *c = 30;

    printf("Antes del double free:\n");
    printf("a = %p (%d), b = %p (%d), c = %p (%d)\n",
           (void*)a, *a, (void*)b, *b, (void*)c, *c);

    free(a);
    free(a);  // DOUBLE FREE: corrompe el heap

    // Ahora el heap está corrupto
    printf("\nDespués del double free:\n");

    int *d = (int *)malloc(sizeof(int));
    if (d) {
        *d = 99;
        printf("d = %p (%d)\n", (void*)d, *d);
    }

    int *e = (int *)malloc(sizeof(int));
    if (e) {
        *e = 88;
        printf("e = %p (%d)\n", (void*)e, *e);
    }

    printf("\nEl programa puede fallar aquí o mostrar datos corruptos.\n");

    free(b);
    free(c);
    // No liberamos a otra vez (ya causamos suficiente daño)

    return 0;
}
```

La solución: asignar NULL inmediatamente después de free().

```c
int *p = (int *)malloc(sizeof(int));
// ... uso ...
free(p);
p = NULL;  // Ahora free(p) no haría nada

// Si por error llamamos a free(p) de nuevo:
free(p);  // Seguro: free(NULL) no hace nada
```

Esta simple práctica elimina el riesgo de double free y dangling pointers
en un solo movimiento.

===============================================================================
6. CORRUPCIÓN DEL HEAP
===============================================================================

La corrupción del heap es una de las pesadillas más temidas en C. Ocurre
cuando escribes o lees fuera de los límites de un bloque asignado, o cuando
manipulas incorrectamente los metadatos del heap.

Causas comunes:

1. **Buffer overflow en el heap**: escribir más allá del bloque asignado.
2. **Use-after-free**: acceder a memoria ya liberada.
3. **Double free**: liberar el mismo bloque dos veces.
4. **Invalid free**: liberar un puntero que no vino de malloc().

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    // Buffer overflow en el heap
    char *buffer = (char *)malloc(5 * sizeof(char));
    if (buffer == NULL) return 1;

    strcpy(buffer, "Hola mundo!");  // ¡Desbordamiento! Solo hay espacio para 4+1

    printf("Buffer: %s\n", buffer);

    // El desbordamiento corrompió los metadatos del bloque siguiente
    free(buffer);  // Probable crash aquí

    return 0;
}
```

¿Cómo detectar corrupción del heap?

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    // Invalid free: liberar un puntero que no es de malloc
    int stack_var = 42;
    int *ptr = &stack_var;

    // free(ptr);  // ¡ERROR! ptr no está en el heap

    // Liberar un puntero interno (no el inicio del bloque)
    int *datos = (int *)malloc(10 * sizeof(int));
    if (datos == NULL) return 1;

    // free(datos + 5);  // ¡ERROR! No es el inicio del bloque

    free(datos);  // Correcto

    return 0;
}
```

Síntomas de heap corrupto:
  - malloc() devuelve NULL sin motivo aparente.
  - free() causa segmentation fault.
  - Los valores de los datos cambian misteriosamente.
  - El programa falla en puntos aparentemente no relacionados.
  - Los fallos son intermitentes y difíciles de reproducir.

Herramientas de detección:
  - Valgrind (memcheck)
  - AddressSanitizer (ASan)
  - Electric Fence
  - glibc MALLOC_CHECK_

```c
// Compilar con AddressSanitizer para detectar corrupción:
// gcc -fsanitize=address -g programa.c -o programa

#include <stdio.h>
#include <stdlib.h>

int main() {
    char *datos = (char *)malloc(10);
    datos[10] = 'X';  // ASan detectará este out-of-bounds
    free(datos);
    return 0;
}
```

La corrupción del heap es traicionera porque los síntomas aparecen mucho
después del error original. Por eso es crucial usar herramientas de
detección desde el principio del desarrollo.

===============================================================================
7. EL HEAP MANAGER: CÓMO RASTREA LAS ASIGNACIONES
===============================================================================

Para entender free() a fondo, debemos conocer cómo el heap manager
(implementado en malloc/free) rastrea los bloques de memoria.

El heap manager mantiene una estructura de datos interna (típicamente
una lista enlazada o un árbol) que contiene todos los bloques libres
y ocupados. Cada bloque tiene un "header" (cabecera) con metadatos.

Estructura típica de un bloque en glibc (ptmalloc):

  +-------------------+
  | prev_size         |  ← Tamaño del bloque anterior (si está libre)
  +-------------------+
  | size              |  ← Tamaño de este bloque (con flags)
  +-------------------+
  | Datos del usuario |  ← El puntero que devuelve malloc()
  |                   |
  |                   |
  +-------------------+
  | (padding)         |  ← Alineación a 8 o 16 bytes
  +-------------------+

Los flags en el campo size indican:
  - PREV_INUSE (bit 0): el bloque anterior está en uso
  - IS_MMAPPED (bit 1): el bloque fue asignado con mmap()
  - NON_MAIN_ARENA (bit 2): el bloque pertenece a un arena no principal

Cuando llamamos a free():

```c
#include <stdio.h>
#include <stdlib.h>

/*
 * Vista interna simplificada de cómo funciona free():
 *
 * 1. free(ptr) recibe el puntero del usuario.
 * 2. Calcula la dirección del header: header = (struct chunk_header *)ptr - 1
 * 3. Lee el tamaño del bloque desde el header.
 * 4. Marca el bloque como libre (limpia el flag PREV_INUSE del SIGUIENTE bloque).
 * 5. Intenta fusionar (coalesce) con bloques libres adyacentes.
 * 6. Si el bloque fusionado es lo suficientemente grande, lo añade a la lista
 *    de bloques libres (o al árbol de bins).
 * 7. Si es posible, devuelve memoria al kernel (con sbrk o munmap).
 */

int main() {
    printf("Demostración conceptual del heap manager\n\n");

    printf("El heap manager organiza la memoria en 'chunks':\n");
    printf("  [Header|Datos][Header|Datos][Header|Datos]...\n\n");

    printf("Cada header contiene:\n");
    printf("  - Tamaño del chunk (incluyendo header)\n");
    printf("  - Flags (PREV_INUSE, IS_MMAPPED, NON_MAIN_ARENA)\n");
    printf("  - Punteros fd/bk (solo en chunks libres)\n\n");

    printf("Cuando liberamos un chunk:\n");
    printf("  1. Se marca como libre\n");
    printf("  2. Se fusiona con chunks libres adyacentes\n");
    printf("  3. Se añade a la lista de chunks libres\n");
    printf("  4. Puede devolverse al kernel si está al final del heap\n\n");

    printf("Asignamos varios bloques para ver el comportamiento:\n");

    void *p1 = malloc(64);
    void *p2 = malloc(64);
    void *p3 = malloc(64);

    printf("  p1 = %p\n", p1);
    printf("  p2 = %p\n", p2);
    printf("  p3 = %p\n", p3);
    printf("  Distancia p2-p1 = %ld bytes\n", (char*)p2 - (char*)p1);
    printf("  El header ocupa %ld bytes (típicamente 16-32)\n",
           (char*)p2 - (char*)p1 - 64);

    free(p1);  // Se marca como libre, se fusiona si es posible
    free(p3);  // Se marca como libre
    free(p2);  // Se fusiona con p1 y p3 (coalescing)

    printf("\nTodos los bloques liberados y fusionados.\n");

    return 0;
}
```

Cuando los bloques se liberan y son adyacentes, el heap manager los
fusiona (coalescing). Esto reduce la fragmentación y permite que
futuras asignaciones grandes tengan éxito.

Si el bloque liberado está al final del heap y es lo suficientemente
grande, el heap manager puede devolver la memoria al kernel usando
sbrk() con un valor negativo o munmap().

===============================================================================
8. PATRONES AVANZADOS DE LIBERACIÓN
===============================================================================

Existen patrones establecidos para manejar la liberación de memoria
en diferentes contextos.

8.1. Liberación perezosa (Lazy Free)

En algunos casos, es preferible diferir la liberación para mejorar el
rendimiento. Esto es común en sistemas de tiempo real o en collectores
de memoria personalizados.

```c
#include <stdio.h>
#include <stdlib.h>

#define MAX_DEFERRED 100

typedef struct {
    void *bloques[MAX_DEFERRED];
    int count;
} DeferredFree;

void deferred_init(DeferredFree *df) {
    df->count = 0;
}

void deferred_add(DeferredFree *df, void *ptr) {
    if (df->count < MAX_DEFERRED) {
        df->bloques[df->count++] = ptr;
    } else {
        free(ptr);  // Liberar inmediatamente si está lleno
    }
}

void deferred_flush(DeferredFree *df) {
    for (int i = 0; i < df->count; i++) {
        free(df->bloques[i]);
    }
    df->count = 0;
}

int main() {
    DeferredFree df;
    deferred_init(&df);

    for (int i = 0; i < 10; i++) {
        int *p = (int *)malloc(sizeof(int));
        *p = i;
        deferred_add(&df, p);
    }

    // Liberar todos de una vez
    deferred_flush(&df);

    return 0;
}
```

8.2. Pool de memoria con liberación por lotes

En lugar de liberar cada bloque individualmente, se liberan todos juntos.

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Bloque {
    void *ptr;
    struct Bloque *siguiente;
} Bloque;

typedef struct {
    Bloque *cabecera;
    int count;
} MemoryPool;

void pool_init(MemoryPool *pool) {
    pool->cabecera = NULL;
    pool->count = 0;
}

void pool_add(MemoryPool *pool, void *ptr) {
    Bloque *b = (Bloque *)malloc(sizeof(Bloque));
    if (b == NULL) return;
    b->ptr = ptr;
    b->siguiente = pool->cabecera;
    pool->cabecera = b;
    pool->count++;
}

void pool_free_all(MemoryPool *pool) {
    Bloque *actual = pool->cabecera;
    while (actual != NULL) {
        Bloque *temp = actual;
        free(actual->ptr);  // Liberar el bloque del usuario
        actual = actual->siguiente;
        free(temp);         // Liberar el nodo del pool
    }
    pool->cabecera = NULL;
    pool->count = 0;
}

int main() {
    MemoryPool pool;
    pool_init(&pool);

    for (int i = 0; i < 100; i++) {
        int *p = (int *)malloc(sizeof(int));
        if (p == NULL) break;
        *p = i * 2;
        pool_add(&pool, p);
    }

    printf("Pool creado con %d bloques.\n", pool.count);

    // Liberar todo de una sola vez
    pool_free_all(&pool);
    printf("Todos los bloques liberados.\n");

    return 0;
}
```

8.3. Macro de liberación segura

Podemos definir una macro que simplifique el patrón free + NULL:

```c
#include <stdio.h>
#include <stdlib.h>

#define FREE_AND_NULL(ptr) do { \
    free(ptr);                 \
    (ptr) = NULL;              \
} while(0)

int main() {
    int *datos = (int *)malloc(100 * sizeof(int));
    if (datos == NULL) return 1;

    for (int i = 0; i < 100; i++) {
        datos[i] = i;
    }

    FREE_AND_NULL(datos);

    // datos es NULL ahora
    // FREE_AND_NULL(datos);  // Seguro: free(NULL) no hace nada

    return 0;
}
```

O una función genérica con puntero a puntero:

```c
#include <stdio.h>
#include <stdlib.h>

void safe_free(void **pp) {
    if (pp != NULL && *pp != NULL) {
        free(*pp);
        *pp = NULL;
    }
}

#define FREE_SAFE(ptr) safe_free((void**)&(ptr))

int main() {
    char *nombre = (char *)malloc(100);
    int *edades = (int *)malloc(50 * sizeof(int));

    if (nombre == NULL || edades == NULL) {
        FREE_SAFE(nombre);
        FREE_SAFE(edades);
        return 1;
    }

    // ... uso ...

    FREE_SAFE(nombre);
    FREE_SAFE(edades);

    return 0;
}
```

Estos patrones reducen drásticamente los errores de memoria y hacen el
código más mantenible.

===============================================================================
9. free() CON ESTRUCTURAS COMPLEJAS
===============================================================================

Cuando trabajamos con estructuras que contienen múltiples campos
dinámicos, la liberación debe hacerse en orden inverso a la asignación.

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    char *titulo;
    char *autor;
    int *calificaciones;
    int num_calificaciones;
} Libro;

Libro *libro_crear(const char *titulo, const char *autor) {
    Libro *libro = (Libro *)malloc(sizeof(Libro));
    if (libro == NULL) return NULL;

    libro->titulo = (char *)malloc(strlen(titulo) + 1);
    libro->autor = (char *)malloc(strlen(autor) + 1);

    if (libro->titulo == NULL || libro->autor == NULL) {
        free(libro->titulo);
        free(libro->autor);
        free(libro);
        return NULL;
    }

    strcpy(libro->titulo, titulo);
    strcpy(libro->autor, autor);
    libro->calificaciones = NULL;
    libro->num_calificaciones = 0;

    return libro;
}

int libro_agregar_calificacion(Libro *libro, int calificacion) {
    int *nuevas = (int *)realloc(libro->calificaciones,
                                 (libro->num_calificaciones + 1) * sizeof(int));
    if (nuevas == NULL) return -1;

    libro->calificaciones = nuevas;
    libro->calificaciones[libro->num_calificaciones] = calificacion;
    libro->num_calificaciones++;
    return 0;
}

void libro_liberar(Libro *libro) {
    if (libro == NULL) return;

    // Liberar en orden inverso a la asignación
    free(libro->calificaciones);
    libro->calificaciones = NULL;

    free(libro->autor);
    libro->autor = NULL;

    free(libro->titulo);
    libro->titulo = NULL;

    free(libro);
    // Nota: no podemos asignar NULL al puntero del llamante
}

int main() {
    Libro *libro = libro_crear("C Master Book", "Profesor Búho AI");
    if (libro == NULL) return 1;

    libro_agregar_calificacion(libro, 95);
    libro_agregar_calificacion(libro, 88);
    libro_agregar_calificacion(libro, 92);

    printf("Libro: %s por %s\n", libro->titulo, libro->autor);
    printf("Calificaciones: ");
    for (int i = 0; i < libro->num_calificaciones; i++) {
        printf("%d ", libro->calificaciones[i]);
    }
    printf("\n");

    libro_liberar(libro);
    libro = NULL;  // El llamante debe asignar NULL

    return 0;
}
```

La regla general: libera en el orden inverso al que asignaste.
Si asignaste A, luego B, luego C, libera C, luego B, luego A.
Esto evita referencias colgantes durante la liberación.

10. CASOS ESPECIALES DE free()
===============================================================================

10.1. free(NULL) es seguro

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *p = NULL;

    // Esto es completamente seguro:
    free(p);  // No hace nada

    // Incluso múltiples veces:
    free(p);
    free(p);

    printf("free(NULL) es seguro.\n");
    return 0;
}
```

Esto es útil en funciones de limpieza que no saben si un puntero
fue asignado o no.

10.2. free() después de realloc()

realloc() puede devolver una dirección diferente. Si lo hace, la
dirección original queda inválida. Nunca llames free() a la
dirección original después de realloc().

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *datos = (int *)malloc(5 * sizeof(int));
    if (datos == NULL) return 1;

    for (int i = 0; i < 5; i++) datos[i] = i;

    int *mas_datos = (int *)realloc(datos, 10 * sizeof(int));
    if (mas_datos == NULL) {
        // realloc falló: datos original sigue siendo válido
        free(datos);
        return 1;
    }

    // realloc tuvo éxito: datos original puede estar liberado
    // NO hacer: free(datos);
    datos = mas_datos;  // Actualizar el puntero

    for (int i = 5; i < 10; i++) datos[i] = i * 10;

    free(datos);  // Liberar el nuevo bloque
    datos = NULL;

    return 0;
}
```

10.3. Liberar memoria alineada (aligned_alloc)

Si usaste aligned_alloc(), usa free() normalmente (en C11):

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    // Asignar memoria alineada a 64 bytes (para SIMD, etc.)
    void *ptr = aligned_alloc(64, 256);
    if (ptr == NULL) return 1;

    // free() funciona con aligned_alloc en C11
    free(ptr);

    return 0;
}
```

===============================================================================
RESUMEN DEL CAPÍTULO
===============================================================================

- free() devuelve la memoria del heap al sistema; cada malloc() debe
  tener su correspondiente free().
- Los punteros colgantes (dangling pointers) apuntan a memoria liberada;
  acceder a ellos es Comportamiento Indefinido.
- Asignar NULL inmediatamente después de free() elimina el riesgo de
  dangling pointers y double free.
- El double free corrompe las estructuras internas del heap y puede
  causar fallos impredecibles o vulnerabilidades de seguridad.
- La corrupción del heap ocurre por buffer overflows, use-after-free,
  double free o invalid free.
- El heap manager mantiene metadatos en cada bloque (tamaño, flags)
  y fusiona bloques libres adyacentes (coalescing).
- free(NULL) es seguro y no hace nada.
- Al liberar estructuras complejas, hazlo en orden inverso a la
  asignación.
- Usa herramientas como Valgrind y AddressSanitizer para detectar
  errores de memoria.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué pasa si llamo a free() con un puntero que no fue devuelto por malloc?**
Es Comportamiento Indefinido. Puede corromper el heap, causar un crash
inmediato o, peor, corromper datos silenciosamente. Nunca liberes punteros
que apunten al stack o a memoria estática.

❓ **¿Debo comprobar si ptr es NULL antes de llamar a free()?**
No es necesario. free(NULL) está definido como una operación segura que
no hace nada. Sin embargo, es buena práctica mantener tus punteros en
estado NULL después de liberarlos.

❓ **¿free() realmente devuelve la memoria al sistema operativo?**
Depende de la implementación. En muchos sistemas, free() marca el bloque
como disponible para reutilización, pero no lo devuelve al kernel
inmediatamente. La memoria se devuelve al kernel cuando el bloque está
al final del heap y es lo suficientemente grande.

❓ **¿Qué es mejor: free() y NULL como macro o como función?**
Ambos enfoques son válidos. Una macro como FREE_SAFE(ptr) es conveniente
y no tiene overhead. Una función con void** es más segura tipicamente
pero requiere más tipeo. La macro es más común en la práctica.
