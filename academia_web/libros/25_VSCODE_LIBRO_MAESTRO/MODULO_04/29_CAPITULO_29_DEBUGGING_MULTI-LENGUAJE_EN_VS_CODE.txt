CONTENIDO DEL CAPÍTULO
===============================================================================

1. DEBUGGER INTEGRADO

Soporta Node.js, Python, Java, C++, C#, Go, Ruby, PHP mediante extensiones.

Todos comparten la misma interfaz y conceptos: breakpoints, watch, call stack.



2. LAUNCH.JSON

Configuraciones en .vscode/launch.json. launch: inicia el programa. attach:

se conecta a proceso existente. Múltiples configuraciones por proyecto.



3. BREAKPOINTS CONDICIONALES

Condiciones (expresiones que se evalúan), hit counts (número de ejecuciones),

logpoints (mensajes sin detener).



4. DEBUG CONSOLE

Terminal DEBUG CONSOLE durante pausa. Ejecuta expresiones en el contexto

actual. Prueba cambios sin modificar el código.

EJEMPLO DE CÓDIGO
----------------------------------------

{
  "configurations": [
    {
      "name": "Python: Flask",
      "type": "python",
      "request": "launch",
      "module": "flask",
      "env": { "FLASK_APP": "app.py", "FLASK_ENV": "development" },
      "args": ["run", "--no-reload"],
      "justMyCode": true
    },
    {
      "name": "Node: Attach",
      "type": "node",
      "request": "attach",
      "processId": "${command:PickProcess}",
      "protocol": "inspector"
    }
  ]
}
