CONTENIDO DEL CAPÍTULO
===============================================================================

1. LOGPOINTS

Mensajes en consola sin detener ejecución. Como console.log sin modificar

código. Clic derecho > Add Logpoint. Variables entre {}.



2. BREAKPOINTS CONDICIONALES

Se activan solo cuando expresión es verdadera. Bugs en condiciones específicas.

Ej: userId === 12345 && total > 1000. Hit count: >= 5.



3. DATA BREAKPOINTS (WATCHPOINTS)

Se detienen cuando una variable cambia su valor. Útil para encontrar dónde

se modifica una variable inesperadamente. Desde panel WATCH.



4. FUNCTION BREAKPOINTS

Breakpoints en todas las llamadas a una función sin saber su ubicación.

Usa 'Debug: Add Function Breakpoint' e ingresa el nombre.

EJEMPLO DE CÓDIGO
----------------------------------------

# Logpoint: "Procesando usuario ${userId} con ${products.length} productos"
# Conditional: userId === 12345 && total > 1000
# Hit count: >= 5 (5ta ejecucion)
# Function BP: processOrder
