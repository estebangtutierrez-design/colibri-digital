CONTENIDO DEL CAPÍTULO
===============================================================================

1. FILE NESTING

Agrupa archivos relacionados bajo uno principal. Component.tsx puede anidar

Component.css, Component.test.tsx, Component.stories.tsx.



2. CONFIGURACIÓN

explorer.fileNesting.enabled: true. explorer.fileNesting.patterns define

las reglas de anidamiento usando el nombre base del archivo.



3. ORGANIZACIÓN DE PROYECTOS

Estructura de carpetas consistente. Nombres descriptivos. Agrupar por

funcionalidad. Archivos de máximo 300 líneas.



4. OUTLINE (Ctrl+Shift+O)

Muestra estructura del archivo actual: funciones, clases, imports.

Navegación rápida entre símbolos del archivo.

EJEMPLO DE CÓDIGO
----------------------------------------

{
  "explorer.fileNesting.enabled": true,
  "explorer.fileNesting.patterns": {
    "*.tsx": "${capture}.css, ${capture}.test.tsx, ${capture}.stories.tsx",
    "*.ts": "${capture}.test.ts, ${capture}.spec.ts",
    "package.json": "package-lock.json, yarn.lock, pnpm-lock.yaml"
  }
}
