CONTENIDO DEL CAPÍTULO
===============================================================================

1. CAMBIOS PARCIALES

Stage de líneas específicas dentro de un archivo. Abre el diff, selecciona

líneas, Stage Selected Ranges. Commits atómicos y limpios.



2. COMMITS ATÓMICOS

Cada commit = un cambio lógico único. Divide cambios grandes con stage

selectivo. Mensajes claros siguiendo Conventional Commits.



3. STASH

Guardar cambios temporales. Source Control > ... > Stash. Aplicar, pop,

listar stashes. Incluir untracked files opcionalmente.



4. AMEND

Modificar el último commit. Solo en commits no publicados. Para cambios

complejos, usa interactive rebase desde terminal o GitLens.

EJEMPLO DE CÓDIGO
----------------------------------------

# Conventional Commits:
# feat: new feature
# fix: bug fix
# docs: documentation
# refactor: code refactoring
# test: adding tests
# chore: maintenance

# Ejemplo: feat(auth): add JWT refresh endpoint
# Closes #123
