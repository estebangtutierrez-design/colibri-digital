CONTENIDO DEL CAPÍTULO
===============================================================================

1. MERGE EDITOR

Tres paneles: Current (nuestro), Incoming (entrante), Result (resultado).

Colores: verde = aceptado, azul = por revisar, rojo = conflicto.



2. ACEPTAR CAMBIOS

Botones: Accept Current, Accept Incoming, Accept Both. Navegar conflictos

con F7/Shift+F7. Editar manualmente en el panel combinado.



3. ESTRATEGIAS

Current: mantener nuestros cambios. Incoming: aceptar del otro (feature).

Both: combinar (ambas partes añadieron código diferente). Manual: editar.



4. CONFIGURACIÓN

git.mergeEditor: true para el nuevo editor. mergeEditor.showBase para

ver el ancestro común. mergeEditor.showNonConflictingChanges.

EJEMPLO DE CÓDIGO
----------------------------------------

{
  "git.mergeEditor": true,
  "mergeEditor.showNonConflictingChanges": true
}
# Atajos:
# Ctrl+Alt+Enter - aceptar conflicto actual
# F7 - siguiente conflicto
# Shift+F7 - conflicto anterior
