CONTENIDO DEL CAPÍTULO
===============================================================================

1. ES MODULES

import/export de ES6. Default exports y named exports. Type-only

exports/imports: export type, import type. Isomórficos con bundlers.



2. NAMESPACES

namespace Validation {}. Anidamiento. Export para visibilidad.

Ambient namespaces: declare namespace. Útil para librerías globales.

Preferir ES modules sobre namespaces en proyectos modernos.



3. RESOLUCIÓN DE MÓDULOS

moduleResolution: node, node16, bundler. paths y baseUrl en tsconfig.

RootDir y outDir para estructura de salida. Declaration files .d.ts.

Triple-slash directives: /// <reference path="..." />.

EJEMPLO DE CÓDIGO
----------------------------------------

// user.ts
export interface User {
  id: number
  name: string
}

export function createUser(name: string): User {
  return { id: Date.now(), name }
}

// app.ts
import type { User } from './user.js'
import { createUser } from './user.js'

const user = createUser('Alice')

// Re-export
export { User, createUser }
export * from './validators.js'
