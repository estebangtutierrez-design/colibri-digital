1. ENUMS NUMÉRICOS

enum Direction { Up = 1, Down, Left, Right }. Auto-incremento.

Reverse mapping: Direction[1] -> 'Up'. Enums const: sin código runtime.

Enums heterogéneos (mixtos) posibles pero no recomendados.



2. ENUMS STRING

enum Color { Red = 'RED', Green = 'GREEN' }. Más legibles en runtime.

Requieren inicialización. No tienen reverse mapping automático.

Útiles para APIs, estados, tipos de eventos.



3. UNION TYPES Y LITERALS

type Status = 'active' | 'inactive'. type Dice = 1 | 2 | 3 | 4 | 5 | 6.

Template literal types: type Event = `on${Change}`. as const para

objetos literales. Discriminated unions con type narrowing.

EJEMPLO DE CÓDIGO
----------------------------------------

enum Direction {
  Up = 'UP',
  Down = 'DOWN',
  Left = 'LEFT',
  Right = 'RIGHT'
}

const dir: Direction = Direction.Up

// Union type
type HttpMethod = 'GET' | 'POST' | 'PUT' | 'DELETE'

// Discriminated union
interface Circle { kind: 'circle'; radius: number }
interface Square { kind: 'square'; side: number }
type Shape = Circle | Square

function area(s: Shape): number {
  if (s.kind === 'circle') return Math.PI * s.radius ** 2
  return s.side ** 2
}
