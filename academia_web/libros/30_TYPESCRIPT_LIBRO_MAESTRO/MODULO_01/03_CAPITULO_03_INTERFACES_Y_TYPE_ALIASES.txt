CONTENIDO DEL CAPÍTULO
===============================================================================

1. INTERFACES

interface User { name: string; age?: number; readonly id: number; }

Extensión: extends. Implementación en clases. Declaration merging.

Interfaces describen la forma de un objeto.



2. TYPE ALIASES

type Point = { x: number; y: number; }. type ID = string | number.

type puede: uniones, intersecciones, tipos condicionales, mapped.

Interfaces vs Types: interface extensible, type no. Usar interface

para objetos públicos, type para uniones/complejos.



3. UTILITY TYPES

Partial<T>, Required<T>, Readonly<T>, Pick<T,K>, Omit<T,K>.

Record<K,T>, Exclude<T,U>, Extract<T,U>, NonNullable<T>.

ReturnType<T>, Parameters<T>. Awaited<T> para async.

EJEMPLO DE CÓDIGO
----------------------------------------

interface User {
  name: string
  email?: string
  readonly id: number
}

type Status = 'active' | 'inactive' | 'pending'
type UserWithStatus = User & { status: Status }

// Utility Types
const partial: Partial<User> = {}
const picked: Pick<User, 'name' | 'email'> = { name: 'test' }

// Type from function
function createUser(name: string): User {
  return { id: 1, name }
}
type CreateUserReturn = ReturnType<typeof createUser>
