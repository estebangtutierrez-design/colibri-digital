CONTENIDO DEL CAPÍTULO
===============================================================================

1. BUILDER PATTERN

Clase Builder con métodos que retornan this. Tipado progresivo.

Patrón común para queries, configuraciones, y objetos complejos.



2. STRATEGY PATTERN

Interface Strategy con método execute. Implementaciones concretas.

Registry de estrategias con Map<string, Strategy>. Genéricos para

tipos de entrada/salida variables.



3. FACTORY PATTERN

Factory function que retorna tipo según input. Discriminated unions

para diferentes configuraciones. Generic factory pattern.

Dependency injection con tipos. Container tipado.

EJEMPLO DE CÓDIGO
----------------------------------------

// Builder pattern
class QueryBuilder<T extends Record<string, any>> {
  private conditions: string[] = []
  
  where<K extends keyof T>(key: K, value: T[K]): this {
    this.conditions.push(`${String(key)} = ${value}`)
    return this
  }
  
  build(): string {
    return `SELECT * WHERE ${this.conditions.join(' AND ')}`
  }
}

// Factory pattern
type ServiceFactory<T> = {
  create(config: T): Service
}

function createFactory<T extends ServiceConfig>(
  ServiceClass: new (config: T) => Service
): ServiceFactory<T> {
  return {
    create: (config) => new ServiceClass(config)
  }
}
