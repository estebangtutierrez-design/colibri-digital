1. BRANDED TYPES

type UserId = string & { __brand: 'UserId' }. Brand único.

Previene mezclar IDs de diferentes entidades. Compile-time safety.

Solo casting explícito crea el tipo. Runtime: sigue siendo string.



2. OPAQUE TYPES

Declarar tipos en módulo separado. Solo el módulo puede crear.

export type UserId = string & { readonly __brand: unique symbol }.

Tagged types con unique symbol. Alternativa a classes wrapper.



3. NOMINAL VS STRUCTURAL

TypeScript tiene structural typing. Brands emulan nominal typing.

Útil para: IDs, emails, sanitized strings, money amounts.

Fléche: librería para brand types. Newtype pattern.

EJEMPLO DE CÓDIGO
----------------------------------------

// Branded type
type Brand<T, B> = T & { __brand: B }
type UserId = Brand<string, 'UserId'>
type OrderId = Brand<string, 'OrderId'>

function createUserId(id: string): UserId {
  return id as UserId
}

function getUser(id: UserId): Promise<User>
function getOrder(id: OrderId): Promise<Order>

// Error de compilación - tipos incompatibles
const uid = createUserId('abc')
// getOrder(uid)  // Error!

// Opaque type con unique symbol
declare const EMAIL_BRAND: unique symbol
type Email = string & { readonly [EMAIL_BRAND]: typeof EMAIL_BRAND }

function createEmail(s: string): Email | null {
  return s.includes('@') ? (s as Email) : null
}
