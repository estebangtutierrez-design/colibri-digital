CONTENIDO DEL CAPÍTULO
===============================================================================

1. VARIANCE EN TYPESCRIPT

Covariant: T en posición de output (return type). Contravariant: T en

posición de input (parameter). Invariant: ambas posiciones.

TypeScript: parámetros de función son contravariantes (strict mode).



2. STRICT FUNCTION TYPES

strictFunctionTypes: true. Parámetros bivariantes en métodos (this).

Function vs method declarations: métodos tienen bivariance.

Implicaciones para EventEmitter, callback handlers.



3. VARIANCE EN GENERICS

interface Producer<T> { produce(): T } (covariant).

interface Consumer<T> { consume(t: T): void } (contravariant).

interface Box<T> { get(): T; set(t: T): void } (invariant).

readonly para covarianza en arrays.

EJEMPLO DE CÓDIGO
----------------------------------------

// Covarianza (output)
type Producer<out T> = () => T

// Contravarianza (input)
type Consumer<in T> = (value: T) => void

// Invarianza
type Box<T> = {
  get(): T
  set(value: T): void
}

// Implicación práctica
class Animal {}
class Dog extends Animal {}

const dogs: Producer<Dog> = () => new Dog()
const animals: Producer<Animal> = dogs  // OK: covariant

const animalConsumer: Consumer<Animal> = (a: Animal) => {}
const dogConsumer: Consumer<Dog> = animalConsumer  // OK: contravariant

// Invariant: no se puede asignar
// const dogBox: Box<Dog> = animalBox  // Error!
