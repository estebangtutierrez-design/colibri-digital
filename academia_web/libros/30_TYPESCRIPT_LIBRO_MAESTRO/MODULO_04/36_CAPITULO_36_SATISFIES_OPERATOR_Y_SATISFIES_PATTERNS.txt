CONTENIDO DEL CAPÍTULO
===============================================================================

1. SATISFIES OPERATOR

satisfies: valida tipo sin cambiar el tipo inferido. Más estricto

que type annotation. Preserva el tipo literal exacto.

const palette = { red: [255, 0, 0] } satisfies Record<string, number[]>



2. PATRONES COMUNES

Config objects con tipos estrictos. Color palette, theme tokens.

API responses con validación. Enum-like objects. Tuples validated.

Record keys tipados con valores literales.



3. SATISFIES VS TYPE ANNOTATION

Annotation: asigna tipo, pierde valor literal. satisfies: valida,

preserva tipo exacto. satisfies + as const para readonly.

type Colors = 'red' | 'green' | 'blue'
const colors = ['red', 'green'] satisfies Colors[]

EJEMPLO DE CÓDIGO
----------------------------------------

type Color = [number, number, number]
type Palette = Record<string, Color>

const palette = {
  red: [255, 0, 0],
  green: [0, 255, 0],
  blue: [0, 0, 255],
  // yellow: [255, 255, 0, 0],  // Error: too many elements
} satisfies Palette

// Cada color inferido como [number, number, number] literal
// No pierde la estructura exacta del objeto

// Api response validation
const response = {
  status: 200,
  data: { id: 1, name: 'test' }
} satisfies { status: number; data: { id: number; name: string } }
