1. ¿QUÉ ES UNA INTERFACE?

Una interface es un tipo de referencia que contiene:
- Métodos abstractos (sin implementación).
- Métodos default (Java 8+).
- Métodos static.
- Constantes (static final).

```java
interface NombreInterface {
    int VALOR_MAXIMO = 100; // public static final

    void metodoAbstracto(); // public abstract

    default void metodoDefault() {
        System.out.println("Default method");
    }

    static void metodoStatic() {
        System.out.println("Static method");
    }
}
```

2. DECLARACIÓN Y USO

```java
interface Imprimible {
    void imprimir();
    String getNombreDocumento();
    int getNumeroPaginas();
}

interface Escaneable {
    void escanear();
    int getResolucion();
}

class DocumentoPDF implements Imprimible {
    private String nombre;
    private int paginas;

    public DocumentoPDF(String nombre, int paginas) {
        this.nombre = nombre;
        this.paginas = paginas;
    }

    @Override
    public void imprimir() {
        System.out.println("Imprimiendo PDF: " + nombre);
    }

    @Override
    public String getNombreDocumento() { return nombre; }

    @Override
    public int getNumeroPaginas() { return paginas; }
}
```

3. HERENCIA MÚLTIPLE DE TIPO

Una clase implementa múltiples interfaces separadas por comas:

```java
class ImpresoraMultifuncional implements Imprimible, Escaneable {
    @Override
    public void imprimir() {
        System.out.println("Imprimiendo...");
    }

    @Override
    public String getNombreDocumento() { return "Doc"; }

    @Override
    public int getNumeroPaginas() { return 10; }

    @Override
    public void escanear() {
        System.out.println("Escaneando...");
    }

    @Override
    public int getResolucion() { return 300; }
}
```

Interfaces pueden extender otras interfaces:

```java
interface Vehiculo {
    void acelerar();
    void frenar();
}

interface VehiculoElectrico extends Vehiculo {
    void cargarBateria();
    int getNivelBateria();
}

class Tesla implements VehiculoElectrico {
    private int bateria = 100;

    @Override
    public void acelerar() {
        System.out.println("Acelerando silenciosamente...");
    }

    @Override
    public void frenar() {
        System.out.println("Frenando con regeneración...");
    }

    @Override
    public void cargarBateria() {
        bateria = 100;
        System.out.println("Batería cargada.");
    }

    @Override
    public int getNivelBateria() { return bateria; }
}
```

4. DEFAULT METHODS (Java 8+)

Permiten agregar métodos con implementación sin romper clases existentes:

```java
interface OperacionMatematica {
    double ejecutar(double a, double b);

    default void mostrarOperacion(String nombre, double a, double b) {
        System.out.println(nombre + ": " + a + " op " + b + " = " + ejecutar(a, b));
    }

    default boolean esValido(double a, double b) {
        return Double.isFinite(a) && Double.isFinite(b);
    }
}

class Suma implements OperacionMatematica {
    @Override
    public double ejecutar(double a, double b) { return a + b; }
    // mostrarOperacion y esValido se heredan
}

class Division implements OperacionMatematica {
    @Override
    public double ejecutar(double a, double b) {
        if (b == 0) throw new ArithmeticException("División por cero");
        return a / b;
    }

    @Override
    public boolean esValido(double a, double b) {
        return OperacionMatematica.super.esValido(a, b) && b != 0;
    }
}
```

Resolución de conflictos: si dos interfaces tienen el mismo default method,
la clase debe sobrescribirlo:

```java
interface A { default void saludar() { System.out.println("Hola A"); } }
interface B { default void saludar() { System.out.println("Hola B"); } }

class C implements A, B {
    @Override
    public void saludar() {
        A.super.saludar(); // Decide cuál usar
    }
}
```

5. INTERFACES FUNCIONALES (@FunctionalInterface)

Tienen exactamente un método abstracto. Son la base de las expresiones lambda.

```java
@FunctionalInterface
interface Calculador {
    int calcular(int x, int y);
}

@FunctionalInterface
interface Saludador {
    void saludar(String nombre);

    default void despedir(String nombre) {
        System.out.println("Adiós, " + nombre);
    }
}
```

Uso con clases anónimas y lambdas:

```java
public class TestLambdas {
    public static void main(String[] args) {
        // Clase anónima
        Calculador suma = new Calculador() {
            @Override
            public int calcular(int x, int y) { return x + y; }
        };

        // Lambda
        Calculador resta = (x, y) -> x - y;
        Calculador multi = (x, y) -> x * y;
        Calculador division2 = (x, y) -> x / y;

        System.out.println("Suma: " + suma.calcular(10, 5));
        System.out.println("Resta: " + resta.calcular(10, 5));
        System.out.println("Multi: " + multi.calcular(10, 5));

        // Con saludador
        Saludador s = nombre -> System.out.println("¡Hola, " + nombre + "!");
        s.saludar("Ana");
        s.despedir("Ana");
    }
}
```

6. INTERFACES COMUNES EN JAVA

java.lang.Comparable<T>: para ordenar objetos.
```java
interface Comparable<T> {
    int compareTo(T o);
}

class Persona implements Comparable<Persona> {
    private String nombre;
    private int edad;

    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    @Override
    public int compareTo(Persona otra) {
        return Integer.compare(this.edad, otra.edad);
    }
}
```

java.util.Comparator<T>: para comparar de diferentes formas.
```java
import java.util.Comparator;

class ComparadorPorNombre implements Comparator<Persona> {
    @Override
    public int compare(Persona p1, Persona p2) {
        return p1.getNombre().compareTo(p2.getNombre());
    }
}
```

java.lang.Runnable: para hilos.
```java
class Tarea implements Runnable {
    @Override
    public void run() {
        System.out.println("Ejecutando en hilo: " +
                         Thread.currentThread().getName());
    }
}
```

7. EJEMPLO COMPLETO: SISTEMA DE AUTENTICACIÓN

```java
interface Autenticable {
    boolean autenticar(String credencial);
    String getTipoAutenticacion();
    void cerrarSesion();

    default boolean validarCredencial(String credencial) {
        return credencial != null && !credencial.isEmpty();
    }
}

interface Auditable {
    void registrarAccion(String accion);
    String[] getHistorialAcciones();
}

class UsuarioLocal implements Autenticable, Auditable {
    private String usuario;
    private String passwordHash;
    private String[] historial;
    private int contador;
    private boolean sesionActiva;

    public UsuarioLocal(String usuario, String password) {
        this.usuario = usuario;
        this.passwordHash = Integer.toHexString(password.hashCode());
        this.historial = new String[100];
        this.contador = 0;
    }

    @Override
    public boolean autenticar(String password) {
        if (!validarCredencial(password)) return false;

        String hash = Integer.toHexString(password.hashCode());
        if (hash.equals(passwordHash)) {
            sesionActiva = true;
            registrarAccion("Inicio de sesión: " + usuario);
            return true;
        }
        registrarAccion("Intento fallido: " + usuario);
        return false;
    }

    @Override
    public String getTipoAutenticacion() { return "LOCAL"; }

    @Override
    public void cerrarSesion() {
        sesionActiva = false;
        registrarAccion("Cierre de sesión: " + usuario);
    }

    @Override
    public void registrarAccion(String accion) {
        if (contador < historial.length) {
            historial[contador++] = java.time.LocalDateTime.now() + " - " + accion;
        }
    }

    @Override
    public String[] getHistorialAcciones() {
        String[] activas = new String[contador];
        System.arraycopy(historial, 0, activas, 0, contador);
        return activas;
    }

    public boolean isSesionActiva() { return sesionActiva; }
}

class UsuarioLDAP implements Autenticable {
    private String usuario;
    private boolean sesionActiva;

    public UsuarioLDAP(String usuario) {
        this.usuario = usuario;
    }

    @Override
    public boolean autenticar(String token) {
        // Simular autenticación LDAP
        if (token != null && token.length() > 10) {
            sesionActiva = true;
            System.out.println("Autenticación LDAP exitosa.");
            return true;
        }
        return false;
    }

    @Override
    public String getTipoAutenticacion() { return "LDAP"; }

    @Override
    public void cerrarSesion() {
        sesionActiva = false;
        System.out.println("Sesión LDAP cerrada.");
    }
}
```

```java
public class SistemaAutenticacion {
    public static void main(String[] args) {
        UsuarioLocal user1 = new UsuarioLocal("admin", "1234");
        UsuarioLDAP user2 = new UsuarioLDAP("corporate");

        Autenticable[] usuarios = {user1, user2};

        for (Autenticable u : usuarios) {
            System.out.println("\nTipo: " + u.getTipoAutenticacion());
            boolean ok = u.autenticar("1234");
            System.out.println("Autenticación: " + (ok ? "Exitosa" : "Fallida"));
            u.cerrarSesion();
        }

        // Acceso a métodos específicos de interfaz Auditable
        if (user1 instanceof Auditable) {
            Auditable aud = (Auditable) user1;
            String[] hist = aud.getHistorialAcciones();
            System.out.println("\nHistorial:");
            for (String h : hist) {
                if (h != null) System.out.println("  " + h);
            }
        }
    }
}
```

8. INTERFAZ vs CLASE ABSTRACTA

| Característica       | Interface              | Clase Abstracta         |
|----------------------|------------------------|-------------------------|
| Instanciable         | No                     | No                      |
| Métodos abstractos   | Sí                     | Sí                      |
| Métodos concretos    | Default/static (8+)    | Sí                      |
| Atributos            | static final           | Cualquier tipo          |
| Constructores        | No                     | Sí                      |
| Herencia múltiple    | Sí (implementar varias)| No (extends solo una)   |
| Estado               | No                     | Sí                      |
| Uso                  | Contrato puro          | Base con implementación |

Regla: Prefiere interface cuando solo necesites un contrato. Usa clase abstracta
cuando las subclases compartan estado o comportamiento.

9. INTERFACES EN JAVA 9+

Java 9 introdujo métodos privados en interfaces:

```java
interface CalculadoraAvanzada {
    default double operacionCompleja(double a, double b, double c) {
        return validarYCalcular(a, b, c);
    }

    private double validarYCalcular(double a, double b, double c) {
        if (a < 0 || b < 0 || c < 0) {
            throw new IllegalArgumentException("Valores negativos no permitidos");
        }
        return a * b + Math.sqrt(c);
    }
}
```

10. BUENAS PRÁCTICAS

1. Nombres de interfaces: adjetivos (Comparable, Runnable) o sustantivos con
   -able (Serializable, Cloneable).
2. Mantener interfaces pequeñas: Principio de Segregación de Interfaces (ISP).
3. Usar @FunctionalInterface para interfaces funcionales.
4. Preferir interfaces sobre clases abstractas para contratos.
5. Documentar el contrato con JavaDoc.
6. No usar interfaces para constantes (es un antipatrón).

13. PREGUNTAS FRECUENTES (FAQ)

P: ¿Puede una interface tener constructor?
R: No. Las interfaces no pueden tener constructores.

P: ¿Puede una interface tener atributos?
R: Sí, pero son implícitamente public static final (constantes).

P: ¿Cuál es la diferencia entre extends e implements?
R: extends es para heredar de una clase. implements es para implementar una
interface.

P: ¿Una interface puede extender otra interface?
R: Sí, usando extends. Una interface puede extender múltiples interfaces.

P: ¿Puedo crear un objeto de una interface?
R: No directamente. Pero puedes crear una clase anónima o lambda.

P: ¿Qué es una interface funcional?
R: Una interface con exactamente un método abstracto. Base de lambdas.

P: ¿Las interfaces reemplazan a las clases abstractas?
R: No. Cada una tiene su propósito. Abstractas para estado compartido,
interfaces para contratos.

14. DESAFÍO DEL ARQUITECTO

Construye un sistema de plugin procesable usando interfaces.

Interfaces base:
- Procesable: void procesar(), String getNombre(), default void mostrarInfo().
- Configurable: void configurar(String clave, String valor), String getConfig().
- Reportable: String generarReporte(), default void imprimirReporte().

Clases que implementan múltiples interfaces:
- ProcesadorTexto: implementa Procesable, Configurable.
- ProcesadorImagen: implementa Procesable, Configurable.
- GeneradorPDF: implementa Procesable, Reportable.
- AnalizadorDatos: implementa Procesable, Configurable, Reportable.

SistemaGestorPlugins:
- Almacena Procesable[] plugins.
- Métodos: registrarPlugin, ejecutarTodos, generarReportesPlugin.
- Usa polimorfismo para procesar cada plugin según sus interfaces.

Incluir una clase de prueba que demuestre el uso de todas las interfaces.

Este desafío integra: interfaces, herencia múltiple de tipo, polimorfismo con
interfaces, default methods y instanceof.

15. CONCLUSIÓN

Las interfaces son esenciales en Java. Permiten:
- Definir contratos puros (abstracción total).
- Herencia múltiple de tipo.
- Código débilmente acoplado (programar para interfaces).
- Expresiones lambda (interfaces funcionales).
- Extensiones sin romper código (default methods).
