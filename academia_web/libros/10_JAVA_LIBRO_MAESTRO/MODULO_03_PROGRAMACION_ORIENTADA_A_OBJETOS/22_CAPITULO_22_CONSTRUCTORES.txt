1. ¿QUÉ ES UN CONSTRUCTOR?

Un constructor es un bloque de código especial que se ejecuta cuando se crea una
instancia de una clase mediante el operador new. Su propósito principal es
inicializar el estado del objeto, asignando valores iniciales a sus atributos.

Características fundamentales de los constructores:
- Tienen el MISMO nombre que la clase.
- NO tienen tipo de retorno (ni siquiera void).
- No pueden ser static, final, abstract ni synchronized.
- Se ejecutan automáticamente al usar new.
- Pueden tener parámetros.
- Pueden sobrecargarse (múltiples constructores con distintos parámetros).

═══════════════════════════════════════════════════════════════════════════════════

2. EL CONSTRUCTOR POR DEFECTO

Si no definimos ningún constructor en una clase, Java proporciona automáticamente
un constructor por defecto (sin parámetros) que:

1. Tiene visibilidad pública (misma que la clase).
2. No tiene parámetros.
3. Inicializa las variables de instancia a sus valores por defecto.
4. No ejecuta ningún código adicional.

```java
class Persona {
    String nombre;
    int edad;
}

// El compilador genera:
// Persona() { super(); }
```

El constructor por defecto DESAPARECE tan pronto como definimos al menos un
constructor en la clase. Si necesitamos un constructor sin parámetros además de
otros, debemos definirlo explícitamente.

```java
class Persona {
    String nombre;
    int edad;

    // Constructor con parámetros
    Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    // ERROR: Persona() ya no existe porque definimos otro constructor
}

// Persona p = new Persona(); // ¡Error de compilación!
```

3. CONSTRUCTORES CON PARÁMETROS

Los constructores con parámetros nos permiten recibir valores iniciales para los
atributos del objeto, garantizando que el objeto se cree en un estado válido.

```java
class Persona {
    String nombre;
    int edad;
    String dni;

    // Constructor con parámetros
    Persona(String nombre, int edad, String dni) {
        this.nombre = nombre;
        this.edad = edad;
        this.dni = dni;
    }
}
```

Uso:

```java
Persona p = new Persona("Ana García", 28, "12345678A");
```

Ventajas:
- Garantiza que el objeto se inicialice completamente.
- Evita estados inconsistentes (objetos con atributos null o cero).
- Hace el código más legible y conciso.

4. SOBRECARGA DE CONSTRUCTORES (CONSTRUCTOR OVERLOADING)

La sobrecarga de constructores permite definir múltiples constructores con
diferentes listas de parámetros. Java selecciona el constructor adecuado según
los argumentos proporcionados en new.

Reglas para la sobrecarga:
- Los constructores deben diferenciarse en número, tipo u orden de parámetros.
- No pueden diferenciarse solo por el nombre del parámetro.
- Cada constructor proporciona una forma diferente de inicializar el objeto.

```java
class Rectangulo {
    double base;
    double altura;
    String color;

    // Constructor sin parámetros
    Rectangulo() {
        this.base = 1.0;
        this.altura = 1.0;
        this.color = "blanco";
    }

    // Constructor con dos parámetros
    Rectangulo(double base, double altura) {
        this.base = base;
        this.altura = altura;
        this.color = "blanco";
    }

    // Constructor con tres parámetros
    Rectangulo(double base, double altura, String color) {
        this.base = base;
        this.altura = altura;
        this.color = color;
    }

    // Constructor que recibe otro Rectángulo (copia)
    Rectangulo(Rectangulo otro) {
        this.base = otro.base;
        this.altura = otro.altura;
        this.color = otro.color;
    }
}
```

Uso de la sobrecarga:

```java
Rectangulo r1 = new Rectangulo();                     // 1.0 x 1.0, blanco
Rectangulo r2 = new Rectangulo(5.0, 3.0);             // 5.0 x 3.0, blanco
Rectangulo r3 = new Rectangulo(4.0, 6.0, "rojo");    // 4.0 x 6.0, rojo
Rectangulo r4 = new Rectangulo(r3);                   // Copia de r3
```

5. ENCADENAMIENTO DE CONSTRUCTORES CON this()

La palabra clave this() permite llamar a un constructor desde otro constructor
de la misma clase. Esto evita la duplicación de código y centraliza la lógica de
inicialización.

Reglas de this():
- Debe ser la PRIMERA sentencia dentro del constructor.
- Solo puede usarse dentro de un constructor.
- Cada constructor puede llamar a un solo this().
- Ayuda a mantener el código DRY (Don't Repeat Yourself).

```java
class Empleado {
    String nombre;
    String departamento;
    double salario;
    String email;

    // Constructor principal (el más completo)
    Empleado(String nombre, String departamento, double salario, String email) {
        this.nombre = nombre;
        this.departamento = departamento;
        this.salario = salario;
        this.email = email;
    }

    // Constructor que delega: solo nombre y salario
    Empleado(String nombre, double salario) {
        this(nombre, "General", salario, nombre.toLowerCase() + "@empresa.com");
    }

    // Constructor que delega: solo nombre
    Empleado(String nombre) {
        this(nombre, 30000.0);
    }

    // Constructor sin parámetros (valores por defecto)
    Empleado() {
        this("Sin Asignar");
    }

    void mostrar() {
        System.out.println(nombre + " | " + departamento +
                         " | $" + salario + " | " + email);
    }
}
```

Flujo de llamadas:

```
new Empleado()
  → this("Sin Asignar")
    → this("Sin Asignar", 30000.0)
      → this("Sin Asignar", "General", 30000.0, "sin asignar@empresa.com")
        → Asigna todos los atributos
```

Uso:

```java
Empleado e1 = new Empleado("Carlos Ruiz", "IT", 55000, "carlos@empresa.com");
Empleado e2 = new Empleado("Ana López", 40000);   // departamento: General
Empleado e3 = new Empleado("Luis");                // salario: 30000, depto: General
Empleado e4 = new Empleado();                      // nombre: Sin Asignar, salario: 30000

e1.mostrar();
e2.mostrar();
e3.mostrar();
e4.mostrar();
```

Salida:

```
Carlos Ruiz | IT | $55000.0 | carlos@empresa.com
Ana López | General | $40000.0 | ana lopez@empresa.com
Luis | General | $30000.0 | luis@empresa.com
Sin Asignar | General | $30000.0 | sin asignar@empresa.com
```

6. DIFERENCIA ENTRE CONSTRUCTOR Y MÉTODO

| Característica       | Constructor                        | Método                          |
|----------------------|------------------------------------|---------------------------------|
| Nombre              | Mismo que la clase                 | Cualquier nombre                |
| Tipo de retorno     | Ninguno (ni void)                  | Obligatorio (void si no retorna)|
| Llamada             | Solo con new                       | objeto.metodo()                 |
| this()              | Sí, primera línea                  | No                              |
| Herencia            | No se hereda                       | Se hereda                       |
| Inicialización      | Estado del objeto                  | Comportamiento                  |

7. VALIDACIÓN EN CONSTRUCTORES

Los constructores son el lugar ideal para validar los datos antes de crear el
objeto. Si los datos no son válidos, podemos lanzar una excepción.

```java
class CuentaBancaria {
    String numeroCuenta;
    String titular;
    double saldo;

    CuentaBancaria(String numeroCuenta, String titular, double saldoInicial) {
        // Validaciones
        if (numeroCuenta == null || numeroCuenta.length() < 10) {
            throw new IllegalArgumentException(
                "El número de cuenta debe tener al menos 10 dígitos.");
        }
        if (titular == null || titular.isEmpty()) {
            throw new IllegalArgumentException(
                "El titular no puede estar vacío.");
        }
        if (saldoInicial < 0) {
            throw new IllegalArgumentException(
                "El saldo inicial no puede ser negativo.");
        }

        this.numeroCuenta = numeroCuenta;
        this.titular = titular;
        this.saldo = saldoInicial;
    }

    void mostrar() {
        System.out.println("Cuenta: " + numeroCuenta);
        System.out.println("Titular: " + titular);
        System.out.println("Saldo: $" + saldo);
    }
}
```

```java
public class PruebaValidacion {
    public static void main(String[] args) {
        try {
            CuentaBancaria c1 = new CuentaBancaria(
                "1234567890", "María Pérez", 1000.0);
            c1.mostrar();

            // Esto lanzará excepción
            CuentaBancaria c2 = new CuentaBancaria(
                "123", "Pedro", -500.0);
            c2.mostrar(); // No se ejecuta

        } catch (IllegalArgumentException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }
}
```

8. CONSTRUCTOR DE COPIA

Un constructor de copia crea un nuevo objeto a partir de otro existente,
copiando sus valores. Es útil para crear copias independientes.

```java
class Punto {
    int x;
    int y;

    Punto(int x, int y) {
        this.x = x;
        this.y = y;
    }

    // Constructor de copia
    Punto(Punto otro) {
        this(otro.x, otro.y); // Usa this() para llamar al constructor principal
    }

    void mover(int dx, int dy) {
        this.x += dx;
        this.y += dy;
    }

    void mostrar() {
        System.out.println("(" + x + ", " + y + ")");
    }
}
```

```java
public class TestCopia {
    public static void main(String[] args) {
        Punto p1 = new Punto(3, 4);
        Punto p2 = new Punto(p1); // Copia independiente

        System.out.print("p1: "); p1.mostrar();
        System.out.print("p2: "); p2.mostrar();

        p1.mover(1, 1); // Solo afecta a p1

        System.out.println("Después de mover p1:");
        System.out.print("p1: "); p1.mostrar();
        System.out.print("p2: "); p2.mostrar(); // p2 no se modifica
    }
}
```

Salida:

```
p1: (3, 4)
p2: (3, 4)
Después de mover p1:
p1: (4, 5)
p2: (3, 4)
```

⚠️ Importante: El constructor de copia realiza una copia superficial (shallow
copy). Si la clase contiene referencias a objetos mutables, estas se compartirán.

9. EJEMPLO COMPLETO: SISTEMA DE GESTIÓN DE ESTUDIANTES

```java
// Archivo: Estudiante.java
public class Estudiante {
    String id;
    String nombreCompleto;
    int edad;
    String carrera;
    double[] calificaciones;

    // Constructor principal
    Estudiante(String id, String nombreCompleto, int edad,
               String carrera, double[] calificaciones) {
        this.id = id;
        this.nombreCompleto = nombreCompleto;
        this.edad = edad;
        this.carrera = carrera;
        this.calificaciones = calificaciones;
    }

    // Constructor con datos básicos (genera ID automático)
    Estudiante(String nombreCompleto, int edad, String carrera) {
        this("EST-" + System.currentTimeMillis(),
             nombreCompleto, edad, carrera, new double[]{0, 0, 0});
    }

    // Constructor solo con nombre y carrera
    Estudiante(String nombreCompleto, String carrera) {
        this(nombreCompleto, 18, carrera);
    }

    // Constructor de copia
    Estudiante(Estudiante otro) {
        this(otro.id, otro.nombreCompleto, otro.edad,
             otro.carrera, otro.calificaciones.clone());
    }

    double calcularPromedio() {
        double suma = 0;
        for (double calif : calificaciones) {
            suma += calif;
        }
        return calificaciones.length > 0 ? suma / calificaciones.length : 0;
    }

    void mostrarInfo() {
        System.out.println("ID: " + id);
        System.out.println("Nombre: " + nombreCompleto);
        System.out.println("Edad: " + edad);
        System.out.println("Carrera: " + carrera);
        System.out.print("Calificaciones: ");
        for (double c : calificaciones) {
            System.out.print(c + " ");
        }
        System.out.println();
        System.out.println("Promedio: " + String.format("%.2f", calcularPromedio()));
        System.out.println("Estado: " + (calcularPromedio() >= 3.0 ?
                           "Aprobado" : "Reprobado"));
    }
}
```

```java
public class SistemaEstudiantes {
    public static void main(String[] args) {
        System.out.println("=== SISTEMA DE GESTIÓN DE ESTUDIANTES ===\n");

        // Usando diferentes constructores
        double[] notasAna = {4.5, 3.8, 4.2};
        Estudiante ana = new Estudiante("EST-001", "Ana García",
                                        22, "Ingeniería de Sistemas", notasAna);

        Estudiante carlos = new Estudiante("Carlos López", 25,
                                          "Matemáticas");

        Estudiante luis = new Estudiante("Luis Torres", "Física");

        // Copia de Ana
        Estudiante copiaAna = new Estudiante(ana);

        ana.mostrarInfo();
        System.out.println();
        carlos.mostrarInfo();
        System.out.println();
        luis.mostrarInfo();
        System.out.println();

        // Modificar la copia y verificar que es independiente
        copiaAna.calificaciones[0] = 5.0;
        System.out.println("=== DESPUÉS DE MODIFICAR COPIA ===");
        System.out.println("Original - Calificación 1: " + ana.calificaciones[0]);
        System.out.println("Copia    - Calificación 1: " + copiaAna.calificaciones[0]);
    }
}
```

10. BUENAS PRÁCTICAS CON CONSTRUCTORES

1. Validar parámetros: Siempre validar los datos recibidos en el constructor.
2. Usar this() para evitar duplicación: Encadena constructores cuando sea
   posible.
3. Mantener constructores simples: La lógica compleja no pertenece al
   constructor.
4. Inicializar todos los atributos: Evita NullPointerException.
5. Preferir constructores con parámetros: En lugar de crear objeto vacío y
   luego llenar atributos.
6. Considerar el patrón Builder: Para clases con muchos atributos opcionales.
7. Documentar excepciones: Indicar qué excepciones puede lanzar el constructor.

11. ERRORES COMUNES CON CONSTRUCTORES

Error 1: Llamar al constructor como un método
```java
Persona p = new Persona(); // Bien
p.Persona(); // Error: no se puede llamar explícitamente
```

Error 2: Olvidar que el constructor por defecto desaparece
```java
class A {
    A(int x) {}
}
A a = new A(); // Error: no existe constructor sin parámetros
```

Error 3: Llamar a this() después de otras sentencias
```java
A() {
    System.out.println("Hola"); // Error: this() debe ser la primera línea
    this(10);
}
```

Error 4: Ciclo infinito con this()
```java
A() { this(10); }
A(int x) { this(); } // Error de compilación: llamada recursiva
```

13. SOLUCIONES A LOS EJERCICIOS

Solución 1: Clase Libro

```java
public class Libro {
    String titulo;
    String autor;
    String isbn;
    int paginas;
    double precio;

    Libro(String titulo, String autor, String isbn,
          int paginas, double precio) {
        this.titulo = titulo;
        this.autor = autor;
        this.isbn = isbn;
        this.paginas = paginas;
        this.precio = precio;
    }

    Libro(String titulo, String autor) {
        this(titulo, autor, "ISBN-" + System.currentTimeMillis(),
             0, 0.0);

        this("Sin título", "Sin autor");
    }

    Libro(Libro otro) {
        this(otro.titulo, otro.autor, otro.isbn,
             otro.paginas, otro.precio);
    }

    void mostrar() {
        System.out.println(titulo + " - " + autor + " (" + isbn + ")");
    }
}
```

14. PREGUNTAS FRECUENTES (FAQ)

P: ¿Los constructores se heredan?
R: No, los constructores no se heredan. Sin embargo, una subclase puede llamar al
constructor de la superclase con super().

P: ¿Puedo tener un constructor privado?
R: Sí, es útil para implementar patrones como Singleton o Factory.

P: ¿Qué es el constructor por defecto?
R: Es el constructor sin parámetros que Java proporciona automáticamente si no
definimos ningún constructor. Desaparece si definimos al menos un constructor.

P: ¿Puedo llamar a un constructor explícitamente después de crear el objeto?
R: No, los constructores solo se ejecutan una vez con new.

P: ¿Puede un constructor ser final?
R: No, los constructores no pueden ser final, static, abstract ni synchronized.

P: ¿Qué diferencia hay entre this y this()?
R: this es una referencia al objeto actual. this() es una llamada a otro
constructor de la misma clase.

P: ¿Puedo usar return en un constructor?
R: No, los constructores no tienen tipo de retorno. Si usas return, debe ser sin
valor: return; Esto termina el constructor antes de tiempo.

encadenamiento con this(), validación, arrays y constructor de copia profunda.

16. CONCLUSIÓN DEL CAPÍTULO

Los constructores son una herramienta fundamental en Java. Nos permiten:
- Inicializar objetos de manera controlada y segura.
- Proporcionar múltiples formas de crear objetos mediante sobrecarga.
- Reutilizar lógica de inicialización con this().
- Validar datos antes de crear el objeto.
- Crear copias independientes con constructores de copia.

Dominar los constructores es esencial para escribir código robusto y mantenible.
