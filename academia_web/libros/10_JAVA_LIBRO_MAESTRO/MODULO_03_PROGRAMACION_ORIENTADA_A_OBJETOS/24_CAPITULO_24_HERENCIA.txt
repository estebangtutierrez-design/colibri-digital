1. ¿QUÉ ES LA HERENCIA?

La herencia es un mecanismo que permite a una clase (subclase o clase hija)
adquirir los atributos y métodos de otra clase (superclase o clase padre).

Relación "es-un" (is-a):
- Un Perro es un Animal.
- Un Coche es un Vehículo.
- Un Profesor es un Empleado.

En Java, la herencia se implementa con la palabra clave extends:

```java
class Superclase {
    // Atributos y métodos
}

class Subclase extends Superclase {
    // Atributos y métodos adicionales o modificados
}
```

Beneficios de la herencia:
- Reutilización de código: La subclase hereda todo lo no privado.
- Organización jerárquica: Modela relaciones naturales.
- Polimorfismo: Una subclase puede tratarse como su superclase.
- Extensibilidad: Podemos añadir funcionalidad sin modificar código existente.

2. SINTAXIS BÁSICA: extends

```java
// Superclase
class Animal {
    String nombre;
    int edad;

    void comer() {
        System.out.println(nombre + " está comiendo.");
    }

    void dormir() {
        System.out.println(nombre + " está durmiendo.");
    }
}

// Subclase
class Perro extends Animal {
    String raza;

    void ladrar() {
        System.out.println(nombre + " dice: ¡Guau guau!");
    }
}

// Otra subclase
class Gato extends Animal {
    String color;

    void maullar() {
        System.out.println(nombre + " dice: ¡Miau!");
    }
}
```

```java
public class TestHerencia {
    public static void main(String[] args) {
        Perro perro = new Perro();
        perro.nombre = "Max";
        perro.edad = 3;
        perro.raza = "Labrador";

        Gato gato = new Gato();
        gato.nombre = "Luna";
        gato.edad = 2;
        gato.color = "Negro";

        // Métodos heredados de Animal
        perro.comer();
        perro.dormir();
        perro.ladrar(); // Método propio

        System.out.println();

        gato.comer();
        gato.dormir();
        gato.maullar(); // Método propio
    }
}
```

3. LA PALABRA CLAVE super

super es una referencia a la superclase. Se utiliza para:

1. Llamar al constructor de la superclase: super(parametros).
2. Acceder a miembros de la superclase ocultos por la subclase.

═══════════════════════════════════════════════════════════════════════════════════

3.1 super() PARA CONSTRUCTORES

La primera línea de un constructor de subclase debe llamar a un constructor de
la superclase, ya sea explícitamente con super() o implícitamente (Java añade
super() automáticamente si existe el constructor por defecto).

```java
class Vehiculo {
    String marca;
    String modelo;
    int anio;

    Vehiculo(String marca, String modelo, int anio) {
        this.marca = marca;
        this.modelo = modelo;
        this.anio = anio;
        System.out.println("Constructor Vehiculo");
    }
}

class Coche extends Vehiculo {
    int numeroPuertas;
    boolean tieneAire;

    Coche(String marca, String modelo, int anio,
          int numeroPuertas, boolean tieneAire) {
        super(marca, modelo, anio); // DEBE ser la primera línea
        this.numeroPuertas = numeroPuertas;
        this.tieneAire = tieneAire;
        System.out.println("Constructor Coche");
    }
}
```

Si la superclase NO tiene constructor sin parámetros, la subclase DEBE llamar
explícitamente a super(parametros).

```java
class A {
    A(int x) { }
}

class B extends A {
    B() {
        super(10); // Obligatorio
    }
}
```

═══════════════════════════════════════════════════════════════════════════════════

3.2 super PARA ACCEDER A MIEMBROS

Podemos usar super.miembro para acceder a métodos o atributos de la superclase
que han sido ocultados o sobrescritos.

```java
class Padre {
    String nombre = "Padre";

    void saludar() {
        System.out.println("Hola desde la clase Padre");
    }
}

class Hijo extends Padre {
    String nombre = "Hijo";

    void mostrarNombres() {
        System.out.println("Nombre desde esta clase: " + this.nombre);
        System.out.println("Nombre desde superclase: " + super.nombre);
    }

    void saludar() {
        super.saludar(); // Llama al método de Padre
        System.out.println("Hola desde la clase Hijo");
    }
}
```

4. SOBRESCRITURA DE MÉTODOS (METHOD OVERRIDING)

La sobrescritura permite a una subclase proporcionar una implementación específica
de un método que ya está definido en su superclase.

Reglas de sobrescritura:
1. El método debe tener la misma firma (nombre + parámetros).
2. El tipo de retorno debe ser el mismo o un subtipo (covariante).
3. El modificador de acceso no puede ser más restrictivo.
4. El método en la superclase no debe ser final.
5. El método en la superclase no debe ser static (eso sería ocultación).
6. Se usa la anotación @Override para verificar la sobrescritura.

```java
class Animal {
    void hacerSonido() {
        System.out.println("El animal hace un sonido genérico.");
    }
}

class Perro extends Animal {
    @Override
    void hacerSonido() {
        System.out.println("El perro ladra: ¡Guau guau!");
    }
}

class Gato extends Animal {
    @Override
    void hacerSonido() {
        System.out.println("El gato maúlla: ¡Miau!");
    }
}
```

5. LA ANOTACIÓN @Override

@Override es una anotación que le indica al compilador que estamos sobrescribiendo
un método de la superclase. El compilador verificará que exista ese método en la
superclase, evitando errores por escribir mal el nombre o los parámetros.

```java
class Base {
    void metodo(int x) { }
}

class Derivada extends Base {
    @Override
    void metodo(int x) { } // Correcto

    // @Override
    // void metodo(String s) { } // Error: no sobrescribe, es sobrecarga
}
```

Beneficios de @Override:
- Detecta errores en tiempo de compilación.
- Hace explícita la intención de sobrescribir.
- Mejora la legibilidad del código.

6. LA CLASE Object

Todas las clases en Java heredan directa o indirectamente de la clase Object.
Esto significa que todos los objetos tienen acceso a los métodos de Object.

Métodos principales de Object:
- toString(): Representación en String del objeto.
- equals(Object obj): Compara objetos.
- hashCode(): Código hash del objeto.
- getClass(): Obtiene la clase del objeto en tiempo de ejecución.
- clone(): Crea una copia del objeto (requiere implementar Cloneable).

```java
class Persona {
    String nombre;
    int edad;

    Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    @Override
    public String toString() {
        return "Persona{nombre='" + nombre + "', edad=" + edad + "}";
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        Persona persona = (Persona) obj;
        return edad == persona.edad && nombre.equals(persona.nombre);
    }

    @Override
    public int hashCode() {
        return java.util.Objects.hash(nombre, edad);
    }
}
```

7. JERARQUÍA DE HERENCIA Y CONSTRUCTORES

Cuando se crea un objeto de una subclase, se ejecutan los constructores de toda
la cadena de herencia, desde la superclase más alta hasta la subclase.

```java
class A {
    A() {
        System.out.println("Constructor A");
    }
}

class B extends A {
    B() {
        System.out.println("Constructor B");
    }
}

class C extends B {
    C() {
        System.out.println("Constructor C");
    }
}

public class TestJerarquia {
    public static void main(String[] args) {
        C c = new C();
        // Salida:
        // Constructor A
        // Constructor B
        // Constructor C
    }
}
```

8. EJEMPLO COMPLETO: SISTEMA DE EMPLEADOS CON HERENCIA

```java
// Superclase
class Empleado {
    protected String nombre;
    protected String id;
    protected double salarioBase;

    public Empleado(String nombre, String id, double salarioBase) {
        this.nombre = nombre;
        this.id = id;
        this.salarioBase = salarioBase;
    }

    public double calcularSalario() {
        return salarioBase;
    }

    public void trabajar() {
        System.out.println(nombre + " está trabajando.");
    }

    public void mostrarInfo() {
        System.out.println("ID: " + id);
        System.out.println("Nombre: " + nombre);
        System.out.println("Salario: $" + String.format("%.2f", calcularSalario()));
    }
}

// Subclase: Gerente
class Gerente extends Empleado {
    private double bonoGestion;
    private String departamento;

    public Gerente(String nombre, String id, double salarioBase,
                   double bonoGestion, String departamento) {
        super(nombre, id, salarioBase);
        this.bonoGestion = bonoGestion;
        this.departamento = departamento;
    }

    @Override
    public double calcularSalario() {
        return super.calcularSalario() + bonoGestion;
    }

    @Override
    public void trabajar() {
        System.out.println(nombre + " está gestionando el departamento " +
                         departamento);
    }

    public void realizarEvaluacion() {
        System.out.println(nombre + " está realizando evaluaciones.");
    }

    @Override
    public void mostrarInfo() {
        super.mostrarInfo();
        System.out.println("Departamento: " + departamento);
        System.out.println("Bono gestión: $" + bonoGestion);
    }
}

// Subclase: Desarrollador
class Desarrollador extends Empleado {
    private String lenguajePrincipal;
    private int proyectosCompletados;

    public Desarrollador(String nombre, String id, double salarioBase,
                         String lenguajePrincipal, int proyectosCompletados) {
        super(nombre, id, salarioBase);
        this.lenguajePrincipal = lenguajePrincipal;
        this.proyectosCompletados = proyectosCompletados;
    }

    @Override
    public double calcularSalario() {
        return super.calcularSalario() + (proyectosCompletados * 200);
    }

    @Override
    public void trabajar() {
        System.out.println(nombre + " está escribiendo código en " +
                         lenguajePrincipal);
    }

    public void compilar() {
        System.out.println(nombre + " está compilando el proyecto.");
    }

    @Override
    public void mostrarInfo() {
        super.mostrarInfo();
        System.out.println("Lenguaje: " + lenguajePrincipal);
        System.out.println("Proyectos: " + proyectosCompletados);
    }
}

// Subclase: Becario
class Becario extends Empleado {
    private String universidad;
    private int duracionMeses;

    public Becario(String nombre, String id, double salarioBase,
                   String universidad, int duracionMeses) {
        super(nombre, id, salarioBase);
        this.universidad = universidad;
        this.duracionMeses = duracionMeses;
    }

    @Override
    public void trabajar() {
        System.out.println(nombre + " está aprendiendo y apoyando al equipo.");
    }

    public void estudiar() {
        System.out.println(nombre + " está estudiando en " + universidad);
    }

    @Override
    public void mostrarInfo() {
        super.mostrarInfo();
        System.out.println("Universidad: " + universidad);
        System.out.println("Duración: " + duracionMeses + " meses");
    }
}
```

```java
public class SistemaEmpleados {
    public static void main(String[] args) {
        System.out.println("=== SISTEMA DE EMPLEADOS ===\n");

        Gerente gerente = new Gerente("Ana López", "G-001",
                                      5000, 1500, "Tecnología");
        Desarrollador dev = new Desarrollador("Carlos Ruiz", "D-001",
                                              3500, "Java", 5);
        Becario becario = new Becario("Luis Torres", "B-001",

        // Polimorfismo: cada uno ejecuta su versión del método

            emp.mostrarInfo();
            System.out.println();
        }

        System.out.println("=== JORNADA LABORAL ===");
        for (Empleado emp : empleados) {
            emp.trabajar();
        }

        System.out.println();
        System.out.println("=== SALARIOS ===");
        for (Empleado emp : empleados) {
            System.out.println(emp.nombre + ": $" +
                             String.format("%.2f", emp.calcularSalario()));
        }
    }
}
```

9. TIPOS DE HERENCIA EN JAVA

Java soporta:
- Herencia simple: Una clase puede extender solo una superclase.
- Herencia multinivel: A extends B, B extends C.
- Herencia jerárquica: Varias clases extienden la misma superclase.

Java NO soporta:
- Herencia múltiple: Una clase no puede extender múltiples clases. Esto se
  soluciona con interfaces (capítulo 27).

```
Herencia simple:                Herencia multinivel:
    Animal                          Vehiculo
      |                               |
      +--- Perro                      Coche
      |                                 |
      +--- Gato                         CocheElectrico

Herencia jerárquica:
        Empleado
      /    |    \
  Gerente  Dev  Becario
```

10. MODIFICADOR protected

El modificador protected permite que los miembros de una clase sean accesibles
desde las subclases (incluso en diferentes paquetes) y desde clases del mismo
paquete.

```java
public class CuentaBancaria {
    protected String numeroCuenta;
    protected double saldo;

    protected void actualizarSaldo(double monto) {
        this.saldo += monto;
    }
}

class CuentaAhorros extends CuentaBancaria {
    public void aplicarInteres(double tasa) {
        double interes = saldo * tasa / 100;
        actualizarSaldo(interes); // Acceso protected
    }
}
```

11. HERENCIA vs. COMPOSICIÓN

La composición (tiene-un) es una alternativa a la herencia. En lugar de "es-un",
modelamos relaciones "tiene-un".

```java
// Herencia: Coche ES-UN Vehiculo
class Coche extends Vehiculo { }

// Composición: Coche TIENE-UN Motor
class Coche {
    private Motor motor;
}
```

Regla general: Preferir composición sobre herencia.
- Usa herencia cuando la relación sea claramente "es-un".
- Usa composición cuando la relación sea "tiene-un" o "usa-un".

```java
// MAL: Herencia forzada
class Pato {
    void volar() { }
}
class PatoJuguete extends Pato { } // Un pato de juguete no vuela

// BIEN: Composición
class PatoJuguete {
    private Pato pato; // Delega comportamiento
}
```

12. BUENAS PRÁCTICAS CON HERENCIA

1. Diseñar para herencia o prohibirla: Documentar si una clase está diseñada
   para ser extendida o marcarla como final.
2. Usar @Override siempre: Para detectar errores en tiempo de compilación.
3. No heredar por reutilización: Hereda porque existe una relación "es-un".
4. Preferir composición sobre herencia: En caso de duda, usa composición.
5. Mantener jerarquías poco profundas: Máximo 3-4 niveles.
6. Llamar a super en constructores: Asegurar inicialización correcta.
7. protected para miembros que las subclases necesiten.

14. SOLUCIONES A LOS EJERCICIOS

Solución 1: Jerarquía de Figuras

```java
class Figura {
    double calcularArea() { return 0; }
}

class Circulo extends Figura {
    private double radio;
    Circulo(double radio) { this.radio = radio; }
    @Override
    double calcularArea() { return Math.PI * radio * radio; }
}

class Rectangulo extends Figura {
    private double base, altura;
    Rectangulo(double base, double altura) {
        this.base = base;
        this.altura = altura;
    }
    @Override
    double calcularArea() { return base * altura; }
}

class Triangulo extends Figura {
    private double base, altura;
    Triangulo(double base, double altura) {
        this.base = base;
        this.altura = altura;
    }
    @Override
    double calcularArea() { return (base * altura) / 2; }
}
```

15. PREGUNTAS FRECUENTES (FAQ)

P: ¿Puedo heredar de múltiples clases en Java?
R: No, Java solo soporta herencia simple. Para herencia múltiple se usan
interfaces.

P: ¿Qué es la sobrescritura de métodos?
R: Es redefinir un método de la superclase en la subclase con la misma firma pero
diferente implementación.

P: ¿Puedo sobrescribir un método static?
R: No, los métodos static se ocultan (hiding), no se sobrescriben.

P: ¿Qué pasa si no llamo a super() en el constructor?
R: Java llama automáticamente a super() (sin parámetros). Si la superclase no
tiene constructor sin parámetros, habrá error de compilación.

P: ¿Puedo evitar que una clase sea heredada?
R: Sí, usando la palabra clave final: final class MiClase { }.

P: ¿Puedo evitar que un método sea sobrescrito?
R: Sí, declarándolo como final: public final void metodo() { }.

P: ¿Qué es el tipo de retorno covariante?
R: Es permitir que el método sobrescrito devuelva un subtipo del tipo de retorno
original.
