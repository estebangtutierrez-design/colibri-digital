1. INTRODUCCIÓN A LA PROGRAMACIÓN ORIENTADA A OBJETOS

La Programación Orientada a Objetos (POO) es un paradigma de programación que
organiza el código en torno a "objetos" en lugar de funciones y procedimientos.
Un objeto es una entidad que contiene datos (atributos) y comportamientos
(métodos).

Java es un lenguaje orientado a objetos desde su concepción. Todo en Java, con
excepción de los tipos primitivos, gira en torno a objetos y clases.

Los cuatro pilares fundamentales de la POO son:

1. Encapsulación: Ocultar los detalles internos de un objeto y proteger sus datos.
2. Herencia: Crear nuevas clases basadas en clases existentes.
3. Polimorfismo: Un mismo método puede comportarse de diferentes formas.
4. Abstracción: Ocultar la complejidad mostrando solo lo esencial.

En este capítulo nos enfocaremos en los dos conceptos más básicos: clases y
objetos.

═══════════════════════════════════════════════════════════════════════════════════

2. ¿QUÉ ES UNA CLASE?

Una clase es un modelo o plano (blueprint) que define la estructura y el
comportamiento de un tipo de objeto. Describe qué atributos (datos) y qué métodos
(funciones) tendrán todos los objetos de ese tipo.

Piensa en una clase como el molde para hacer galletas: el molde define la forma,
pero no es la galleta. Cada galleta que hornees con ese molde será un objeto
distinto.

La sintaxis básica para definir una clase en Java es:

```java
class NombreClase {
    // Atributos (variables de instancia)
    // Métodos (comportamientos)
}
```

Las clases en Java siguen las siguientes reglas:
- El nombre debe comenzar con mayúscula (convención CamelCase).
- El archivo debe llamarse igual que la clase pública.
- Una clase puede contener atributos, métodos, constructores y bloques.

═══════════════════════════════════════════════════════════════════════════════════

3. ¿QUÉ ES UN OBJETO?

Un objeto es una instancia concreta de una clase. Es una entidad que existe en
memoria y tiene:
- Estado: representado por los valores de sus atributos.
- Comportamiento: definido por sus métodos.
- Identidad: cada objeto es único, aunque tenga los mismos valores que otro.

Para crear un objeto usamos la palabra clave new seguida del constructor de la
clase:

```java
NombreClase objeto = new NombreClase();
```

Cuando ejecutamos new ocurre lo siguiente:
1. Se reserva memoria en el heap para el nuevo objeto.
2. Se inicializan los atributos a sus valores por defecto.
3. Se ejecuta el constructor de la clase.
4. Se devuelve una referencia al objeto creado.

═══════════════════════════════════════════════════════════════════════════════════

4. VARIABLES DE INSTANCIA (ATRIBUTOS)

Las variables de instancia son los datos que posee cada objeto. Se declaran dentro
de la clase pero fuera de cualquier método. Cada objeto tiene su propia copia de
estas variables.

Tipos de variables en una clase:
- Variables de instancia: pertenecen a cada objeto.
- Variables de clase (static): pertenecen a la clase, compartidas por todos.
- Variables locales: declaradas dentro de métodos, solo existen durante la
  ejecución del método.

Valores por defecto de las variables de instancia:
- byte, short, int, long: 0
- float, double: 0.0
- char: '\u0000'
- boolean: false
- Referencias (objetos): null

═══════════════════════════════════════════════════════════════════════════════════

5. MÉTODOS DE INSTANCIA

Los métodos definen el comportamiento de los objetos. Un método es un bloque de
código que realiza una tarea específica y puede devolver un valor.

Sintaxis de un método:

```java
tipoRetorno nombreMetodo(parametros) {
    // Cuerpo del método
    return valor;
}
```

Si el método no devuelve nada, usamos void.

═══════════════════════════════════════════════════════════════════════════════════

6. LA REFERENCIA this

La palabra clave this es una referencia al objeto actual. Se utiliza para:
- Diferenciar entre variables de instancia y parámetros con el mismo nombre.
- Llamar a otros constructores de la misma clase.
- Pasar el objeto actual como parámetro.

```java
class Persona {
    private String nombre;

    public void setNombre(String nombre) {
        this.nombre = nombre; // this.nombre es el atributo, nombre es el parámetro
    }
}
```

7. EJEMPLO COMPLETO: SISTEMA DE BIBLIOTECA

A continuación, presentamos un ejemplo completo que modela un sistema simple de
biblioteca usando clases y objetos.

```java
// Archivo: Libro.java
public class Libro {
    // Variables de instancia
    String titulo;
    String autor;
    int anioPublicacion;
    boolean disponible;
    int numeroPaginas;
    String genero;

    // Método para mostrar información del libro
    void mostrarInformacion() {
        System.out.println("=== INFORMACIÓN DEL LIBRO ===");
        System.out.println("Título: " + titulo);
        System.out.println("Autor: " + autor);
        System.out.println("Año de publicación: " + anioPublicacion);
        System.out.println("Género: " + genero);
        System.out.println("Páginas: " + numeroPaginas);
        System.out.println("Disponible: " + (disponible ? "Sí" : "No"));
        System.out.println("==============================");
    }

    // Método para prestar el libro
    void prestar() {
        if (disponible) {
            disponible = false;
            System.out.println("El libro '" + titulo + "' ha sido prestado.");
        } else {
            System.out.println("El libro '" + titulo + "' no está disponible.");
        }
    }

    // Método para devolver el libro
    void devolver() {
        if (!disponible) {
            disponible = true;
            System.out.println("El libro '" + titulo + "' ha sido devuelto.");
        } else {
            System.out.println("El libro '" + titulo + "' ya estaba disponible.");
        }
    }

    // Método para verificar disponibilidad
    boolean estaDisponible() {
        return disponible;
    }

    // Método que usa this para comparar con otro libro
    boolean esMismoGenero(Libro otroLibro) {
        return this.genero.equals(otroLibro.genero);
    }
}
```

```java
public class Biblioteca {
    String nombre;
    String direccion;
    Libro libro1;

        libro1.autor = "Gabriel García Márquez";
        libro1.anioPublicacion = 1967;
        libro1.disponible = true;
        libro1.numeroPaginas = 417;
        libro1.genero = "Realismo Mágico";

        libro2 = new Libro();
        libro2.titulo = "Don Quijote de la Mancha";
        libro2.autor = "Miguel de Cervantes";
        libro2.anioPublicacion = 1605;
        libro2.disponible = true;
        libro2.numeroPaginas = 863;
        libro2.genero = "Novela";

        libro3 = new Libro();
        libro3.titulo = "1984";
        libro3.autor = "George Orwell";
        libro3.anioPublicacion = 1949;
        libro3.disponible = true;
        libro3.numeroPaginas = 328;
        libro3.genero = "Distopía";
    }

    void mostrarCatalogo() {
        System.out.println("=== CATÁLOGO DE " + nombre + " ===");
        libro1.mostrarInformacion();
        System.out.println();
        libro2.mostrarInformacion();
        System.out.println();
        libro3.mostrarInformacion();
    }

    void realizarPrestamo(int opcion) {
        switch (opcion) {
            case 1: libro1.prestar(); break;
            case 2: libro2.prestar(); break;
            case 3: libro3.prestar(); break;
            default: System.out.println("Opción inválida.");
        }
    }

    void realizarDevolucion(int opcion) {
        switch (opcion) {
            case 1: libro1.devolver(); break;
            case 2: libro2.devolver(); break;
            case 3: libro3.devolver(); break;
            default: System.out.println("Opción inválida.");
        }
    }

    void librosDelMismoGenero() {
        System.out.println("Comparando géneros entre libros:");
        if (libro1.esMismoGenero(libro2)) {
            System.out.println(libro1.titulo + " y " + libro2.titulo +
                             " son del mismo género.");
        } else {
            System.out.println(libro1.titulo + " y " + libro2.titulo +
                             " son de géneros diferentes.");
        }
        if (libro1.esMismoGenero(libro3)) {
            System.out.println(libro1.titulo + " y " + libro3.titulo +
                             " son del mismo género.");
        } else {
            System.out.println(libro1.titulo + " y " + libro3.titulo +
                             " son de géneros diferentes.");
        }
        if (libro2.esMismoGenero(libro3)) {
            System.out.println(libro2.titulo + " y " + libro3.titulo +
                             " son del mismo género.");
        } else {
            System.out.println(libro2.titulo + " y " + libro3.titulo +
                             " son de géneros diferentes.");
        }
    }
}
```

8. EJEMPLO ADICIONAL: CLASE RECTÁNGULO

Veamos un ejemplo más matemático para reforzar conceptos:

```java
public class Rectangulo {
    // Variables de instancia
    double base;
    double altura;
    String color;

    // Métodos de cálculo
    double calcularArea() {
        return base * altura;
    }

    double calcularPerimetro() {
        return 2 * (base + altura);
    }

    double calcularDiagonal() {
        return Math.sqrt(base * base + altura * altura);
    }

    // Método que usa this para comparar
    boolean esMayor(Rectangulo otro) {
        return this.calcularArea() > otro.calcularArea();
    }

    // Método que redimensiona el rectángulo
    void escalar(double factor) {
        this.base *= factor;
        this.altura *= factor;
    }

    void mostrar() {
        System.out.println("Rectángulo " + color + ": " +
                         base + " x " + altura +
                         " | Área: " + calcularArea() +
                         " | Perímetro: " + calcularPerimetro() +
                         " | Diagonal: " + String.format("%.2f", calcularDiagonal()));
    }
}
```

```java
public class PruebaRectangulo {
    public static void main(String[] args) {
        // Crear rectángulos
        Rectangulo r1 = new Rectangulo();
        r1.base = 5.0;
        r1.altura = 3.0;
        r1.color = "rojo";

        Rectangulo r2 = new Rectangulo();
        r2.base = 4.0;
        r2.altura = 6.0;
        r2.color = "azul";

        // Mostrar información
        r1.mostrar();
        r2.mostrar();

        // Comparar
        if (r1.esMayor(r2)) {
            System.out.println("El rectángulo rojo es mayor.");
        } else {
            System.out.println("El rectángulo azul es mayor o igual.");
        }

        // Escalar
        System.out.println("\nEscalando rectángulo rojo por factor 2...");
        r1.escalar(2.0);
        r1.mostrar();
    }
}
```

9. RELACIÓN ENTRE CLASES Y OBJETOS EN MEMORIA

Entender cómo se organizan las clases y objetos en memoria es fundamental:

1. La clase (el .class) se carga en el Metaspace (antes PermGen).
2. Las variables de instancia se almacenan en el heap dentro del objeto.
3. Los métodos se almacenan una sola vez en el Metaspace, todos los objetos
   comparten el mismo código.
4. La referencia a un objeto se guarda en la pila (stack) y apunta al objeto
   en el heap.

```
Memoria:
+-----------+
| Pila      |          +----------+
| (Stack)   |          |  Heap    |
|           |          |          |
| r1 ───────┼─────────>│ Libro    |
|           |          | titulo   |
| r2 ───────┼─────────>│ autor    |
|           |          | anio     |
+-----------+          | ...      |
                       +----------+
```

10. BUENAS PRÁCTICAS AL DEFINIR CLASES

1. Una clase debe representar una sola entidad lógica (Principio de
   Responsabilidad Única).
2. Los nombres de clases deben ser sustantivos en singular con PascalCase.
3. Los atributos deben ser privados (encapsulación, lo veremos en capítulo 23).
4. Los métodos deben ser verbos que describan su acción.
5. Inicializar las variables de instancia apropiadamente.
6. Mantener la cohesión: todos los métodos deben estar relacionados con el
   propósito de la clase.

Solución 2: Clase CuentaBancaria

```java
public class CuentaBancaria {
    String numeroCuenta;
    String titular;
    double saldo;

    void depositar(double monto) {
        if (monto > 0) {
            this.saldo += monto;
            System.out.println("Depósito exitoso. Nuevo saldo: " + saldo);
        } else {
            System.out.println("El monto debe ser positivo.");
        }
    }

    void retirar(double monto) {
        if (monto > 0 && monto <= this.saldo) {
            this.saldo -= monto;
            System.out.println("Retiro exitoso. Nuevo saldo: " + saldo);
        } else {
            System.out.println("Fondos insuficientes o monto inválido.");
        }
    }

    void mostrarSaldo() {
        System.out.println("Titular: " + titular);
        System.out.println("Cuenta: " + numeroCuenta);
        System.out.println("Saldo: $" + saldo);
    }
}
```

13. PREGUNTAS FRECUENTES (FAQ)

P: ¿Puedo tener múltiples clases en un mismo archivo?
R: Sí, pero solo una puede ser pública. El archivo debe llamarse como la clase
pública. Las demás clases tendrán visibilidad package-private.

P: ¿Qué ocurre si no asigno un valor a una variable de instancia?
R: Java le asigna un valor por defecto: 0 para números, false para boolean,
null para referencias.

P: ¿Cuál es la diferencia entre una clase y un objeto?
R: La clase es el modelo (el plano), el objeto es la instancia concreta (la casa
construida a partir del plano).

P: ¿Qué significa que una variable de tipo objeto contenga null?
R: Significa que la referencia no apunta a ningún objeto en memoria. Intentar
acceder a métodos o atributos de una referencia null lanza una
NullPointerException.

P: ¿Es obligatorio usar this?
R: No, es opcional cuando no hay ambigüedad. Sin embargo, es buena práctica
usarlo para hacer explícito que nos referimos a la variable de instancia.

P: ¿Puedo crear un objeto sin usar new?
R: Hay otras formas (reflexión, deserialización, clone), pero new la más común.

P: ¿Qué es el garbage collector?
R: Es el mecanismo de Java que destruye automáticamente los objetos que ya no
tienen referencias, liberando memoria del heap.

15. CONCLUSIÓN DEL CAPÍTULO

Hemos dado el primer paso en el mundo de la programación orientada a objetos.
Ahora entiendes que:

- Una clase es un plano que define atributos y comportamientos.
- Un objeto es una instancia concreta de una clase, creada con new.
- Cada objeto tiene su propio estado (valores de atributos).
- La referencia this apunta al objeto actual.
- Los métodos definen el comportamiento de los objetos.

Estos conceptos son la base de todo el desarrollo moderno en Java. En el próximo
capítulo exploraremos los constructores, que nos permitirán inicializar objetos
de manera más elegante y controlada.
