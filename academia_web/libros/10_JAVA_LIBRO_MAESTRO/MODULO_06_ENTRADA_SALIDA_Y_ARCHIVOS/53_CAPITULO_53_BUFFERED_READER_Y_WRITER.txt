1. EL PATRÓN DECORADOR EN java.io
----------------------------------

Java.io utiliza el patrón de diseño Decorador. Las clases "núcleo"
(FileReader, FileWriter) leen/escriben directamente del sistema. Las
clases "decoradoras" (BufferedReader, BufferedWriter) envuelven a las
núcleo añadiendo funcionalidad (buffering, métodos convenientes).

  FileReader fr = new FileReader("/tmp/archivo.txt");
  BufferedReader br = new BufferedReader(fr);

  // También en una línea:
  BufferedReader br = new BufferedReader(new FileReader("/tmp/archivo.txt"));

Este patrón permite combinar funcionalidades: BufferedReader puede
envolver cualquier Reader (no solo FileReader).

2. BUFFEREDREADER — CONSTRUCTORES
----------------------------------

  • BufferedReader(Reader in)
    Crea un buffer con tamaño predeterminado (8192 caracteres).

  • BufferedReader(Reader in, int sz)
    Crea un buffer con tamaño personalizado (sz caracteres).

  BufferedReader br1 = new BufferedReader(new FileReader("/tmp/a.txt"));
  BufferedReader br2 = new BufferedReader(new FileReader("/tmp/b.txt"), 16384);

Un buffer más grande puede mejorar el rendimiento en archivos grandes,
pero consume más memoria. El tamaño óptimo suele ser múltiplo de 4096.

3. READLINE() — LA JOYA DE BUFFEREDREADER
------------------------------------------

readLine() lee una línea completa (hasta '\n', '\r', o '\r\n') y devuelve
un String sin el carácter(es) de fin de línea. Devuelve null al llegar al
final del archivo.

  try (BufferedReader br = new BufferedReader(new FileReader("/tmp/archivo.txt"))) {
      String linea;
      while ((linea = br.readLine()) != null) {
          System.out.println(linea);
      }
  }

Este es probablemente el patrón más utilizado en Java para leer archivos
de texto. Es simple, claro y eficiente.

4. LECTURA DE ARCHIVOS GRANDES
-------------------------------

Para archivos muy grandes (cientos de MB o GB), readLine() con buffer
es la forma más eficiente de procesar línea por línea sin cargar todo
el archivo en memoria:

  try (BufferedReader br = new BufferedReader(new FileReader("/tmp/gigante.txt"))) {
      String linea;
      long lineas = 0;
      while ((linea = br.readLine()) != null) {
          // Procesar línea...
          lineas++;
      }
      System.out.println("Total líneas: " + lineas);
  }

5. OTROS MÉTODOS DE BUFFEREDREADER
------------------------------------

  • read(): lee un carácter (igual que Reader).
  • read(char[] cbuf, int off, int len): lee en bloque.
  • skip(long n): salta n caracteres.
  • ready(): indica si el stream está listo para ser leído.
  • mark(int readAheadLimit): marca la posición actual.
  • reset(): vuelve a la última posición marcada.
  • lines(): (Java 8+) devuelve un Stream<String> de líneas.

Ejemplo con lines() (Java 8+):

  try (BufferedReader br = new BufferedReader(new FileReader("/tmp/datos.txt"))) {
      long totalChars = br.lines()
              .filter(linea -> !linea.startsWith("#"))
              .mapToLong(String::length)
              .sum();
      System.out.println("Total chars (sin comentarios): " + totalChars);
  }

Nota: lines() devuelve un Stream que debe cerrarse. El try-with-resources
cierra el BufferedReader que a su vez cierra el Stream.

6. EL MÉTODO MARK() Y RESET()
------------------------------

BufferedReader soporta marcas (mark), lo que permite "retroceder" en la
lectura:

  try (BufferedReader br = new BufferedReader(new FileReader("/tmp/datos.txt"))) {
      // Marcar posición actual con límite de 100 caracteres hacia adelante
      br.mark(100);
      String linea1 = br.readLine();
      String linea2 = br.readLine();
      System.out.println("Leído: " + linea1 + " | " + linea2);
      br.reset(); // Volver a la marca
      String linea1Again = br.readLine();
      System.out.println("Releído: " + linea1Again);
  }

mark(int readAheadLimit) especina cuántos caracteres se pueden leer antes
de que la marca se invalide.

7. BUFFEREDWRITER — CONSTRUCTORES
-----------------------------------

  • BufferedWriter(Writer out)
    Buffer de tamaño predeterminado (8192 caracteres).

  • BufferedWriter(Writer out, int sz)
    Buffer de tamaño personalizado.

  BufferedWriter bw1 = new BufferedWriter(new FileWriter("/tmp/salida.txt"));
  BufferedWriter bw2 = new BufferedWriter(new FileWriter("/tmp/salida.txt"), 16384);

8. ESCRITURA CON BUFFER
-------------------------

BufferedWriter acumula caracteres en un buffer interno y los escribe al
destino cuando el buffer está lleno, o cuando se llama a flush().

  try (BufferedWriter bw = new BufferedWriter(new FileWriter("/tmp/salida.txt"))) {
      bw.write("Primera línea.");
      bw.newLine(); // Salto de línea portable
      bw.write("Segunda línea.");
      bw.newLine();
      bw.write("Tercera línea.");
      bw.newLine();
  }

9. NEWLINE() — SALTO DE LÍNEA PORTABLE
----------------------------------------

newLine() escribe el separador de línea apropiado para la plataforma:
  • Windows: \r\n
  • Linux/Mac: \n
  • Mac clásico (pre-OS X): \r

Esto hace que el código sea portátil. No uses \n directamente si el
archivo se va a leer en distintas plataformas.

10. FLUSH() — VACIAR EL BUFFER
-------------------------------

flush() fuerza la escritura del contenido del buffer al destino.

  try (BufferedWriter bw = new BufferedWriter(new FileWriter("/tmp/log.txt", true))) {
      bw.write("Evento: " + new java.util.Date());
      bw.newLine();
      bw.flush(); // Asegurar que se escriba inmediatamente
      // ... más escrituras ...
  }

Útil en logs o aplicaciones en tiempo real donde los datos deben
persistir inmediatamente.

11. COMPARACIÓN DE RENDIMIENTO
-------------------------------

Demostración conceptual de la diferencia de rendimiento:

  // SIN BUFFER: FileReader directo
  long inicio = System.currentTimeMillis();
  try (FileReader fr = new FileReader("/tmp/grande.txt")) {
      int c;
      while ((c = fr.read()) != -1) { /* descartar */ }
  }
  long fin = System.currentTimeMillis();
  System.out.println("FileReader: " + (fin - inicio) + " ms");

  // CON BUFFER: BufferedReader
  inicio = System.currentTimeMillis();
  try (BufferedReader br = new BufferedReader(new FileReader("/tmp/grande.txt"))) {
      int c;
      while ((c = br.read()) != -1) { /* descartar */ }
  }
  fin = System.currentTimeMillis();
  System.out.println("BufferedReader: " + (fin - inicio) + " ms");

En la práctica, BufferedReader puede ser 10-100 veces más rápido que
FileReader para lectura carácter por carácter, dependiendo del tamaño
del archivo y del buffer.

12. TAMAÑO DEL BUFFER
-----------------------

  • Tamaño predeterminado: 8192 caracteres (8 KB).
  • Para la mayoría de casos, el tamaño predeterminado es adecuado.
  • Para archivos muy grandes (>100 MB), probar con 32 KB o 64 KB.
  • Para operaciones interactivas (consola), el tamaño predeterminado basta.
  • Regla general: múltiplos de 4096 (tamaño de página en discos).

13. BUFFEREDREADER CON SYSTEM.IN
---------------------------------

BufferedReader no solo sirve para archivos. También se usa para leer
desde la consola de forma eficiente:

  try (BufferedReader br = new BufferedReader(new InputStreamReader(System.in))) {
      System.out.print("Ingrese su nombre: ");
      String nombre = br.readLine();
      System.out.println("Hola, " + nombre + "!");
  }

System.in es un InputStream (bytes). InputStreamReader actúa como puente
de bytes a caracteres, y BufferedReader añade el buffer.

14. LÍNEAS CON STREAM API (JAVA 8+)
------------------------------------

BufferedReader.lines() devuelve un Stream<String> que permite procesar
el archivo de forma funcional:

  try (BufferedReader br = new BufferedReader(new FileReader("/tmp/datos.txt"))) {
      br.lines()
              .filter(linea -> !linea.trim().isEmpty())
              .map(String::toUpperCase)
              .forEach(System.out::println);
  }

El Stream se cierra automáticamente al cerrar el BufferedReader.

15. BUFFEREDWRITER CON PRINTWRITER
-----------------------------------

Combinación común para salida con formato:

  try (PrintWriter pw = new PrintWriter(
          new BufferedWriter(new FileWriter("/tmp/reporte.txt")))) {
      pw.println("=== REPORTE ===");
      pw.printf("Fecha: %s%n", new java.util.Date());
      pw.printf("Total: %,d unidades%n", 1024);
  }

import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.InputStreamReader;
import java.io.IOException;
import java.io.PrintWriter;
import java.util.stream.Stream;

public class Capitulo53BufferedReaderWriter {

    public static void main(String[] args) {
        System.out.println("=== CAPÍTULO 53: BUFFERED READER & WRITER ===\n");

        String archivo = "/tmp/cap53_ejemplo.txt";
        String archivoGrande = "/tmp/cap53_grande.txt";

        // 1. Escritura con BufferedWriter
        System.out.println("--- Escritura con BufferedWriter ---");
        try (BufferedWriter bw = new BufferedWriter(new FileWriter(archivo))) {
            bw.write("Línea 1: Hola desde BufferedWriter");
            bw.newLine();
            bw.write("Línea 2: Esto es una prueba");
            bw.newLine();
            bw.write("Línea 3: Con saltos de línea portables");
            bw.newLine();
            bw.write("Línea 4: ñoño áéíóú");
            bw.newLine();
            bw.flush();
            System.out.println("Archivo escrito correctamente.");
        } catch (IOException e) {
            System.err.println("Error escritura: " + e.getMessage());
        }

        // 2. Lectura con readLine()
        System.out.println("\n--- Lectura con readLine() ---");
        try (BufferedReader br = new BufferedReader(new FileReader(archivo))) {
            String linea;
            int numLinea = 0;
            while ((linea = br.readLine()) != null) {
                System.out.println("L" + (++numLinea) + ": " + linea);
            }
        } catch (IOException e) {
            System.err.println("Error lectura: " + e.getMessage());
        }

        // 3. Lectura con tamaño de buffer personalizado
        System.out.println("\n--- Buffer personalizado (16 KB) ---");
        try (BufferedReader br = new BufferedReader(
                new FileReader(archivo), 16384)) {
            char[] buffer = new char[4096];
            int leidos;
            StringBuilder sb = new StringBuilder();
            while ((leidos = br.read(buffer)) != -1) {
                sb.append(buffer, 0, leidos);
            }
            System.out.println("Total caracteres: " + sb.length());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 4. Modo append con BufferedWriter
        System.out.println("\n--- Append con BufferedWriter ---");
        try (BufferedWriter bw = new BufferedWriter(
                new FileWriter(archivo, true))) {
            bw.write("Línea añadida al final.");
            bw.newLine();
            bw.write("Otra línea más.");
            bw.newLine();
            System.out.println("Datos añadidos.");
        } catch (IOException e) {
            System.err.println("Error append: " + e.getMessage());
        }

        // 5. lines() con Stream API (Java 8+)
        System.out.println("\n--- lines() con Stream ---");
        try (BufferedReader br = new BufferedReader(new FileReader(archivo))) {
            Stream<String> lineas = br.lines();
            lineas.filter(l -> l.contains("Línea"))
                  .map(String::toUpperCase)
                  .forEach(l -> System.out.println(">> " + l));
        } catch (IOException e) {
            System.err.println("Error lines: " + e.getMessage());
        }

        // 6. mark() y reset()
        System.out.println("\n--- mark() y reset() ---");
        try (BufferedReader br = new BufferedReader(new FileReader(archivo))) {
            br.mark(200); // Permite leer hasta 200 chars sin perder la marca
            String primera = br.readLine();
            String segunda = br.readLine();
            System.out.println("Primera lectura: " + primera);
            System.out.println("Segunda lectura: " + segunda);
            br.reset(); // Volver al mark
            String releida = br.readLine();
            System.out.println("Releída desde mark: " + releida);
        } catch (IOException e) {
            System.err.println("Error mark: " + e.getMessage());
        }

        // 7. Archivo grande simulado y medición de rendimiento
        System.out.println("\n--- Simulación de archivo grande ---");
        // Crear archivo de prueba con 10000 líneas
        try (BufferedWriter bw = new BufferedWriter(
                new FileWriter(archivoGrande))) {
            for (int i = 0; i < 10000; i++) {
                bw.write("Esta es la línea número " + i
                         + " del archivo de prueba.");
                bw.newLine();
            }
            System.out.println("Archivo de " + 10000 + " líneas creado.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // Medir FileReader sin buffer
        System.out.println("\n--- Comparación de rendimiento ---");
        long inicioFR = System.currentTimeMillis();
        try (FileReader fr = new FileReader(archivoGrande)) {
            while (fr.read() != -1) { /* vacío */ }
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        long finFR = System.currentTimeMillis();

        long inicioBR = System.currentTimeMillis();
        try (BufferedReader br = new BufferedReader(
                new FileReader(archivoGrande))) {
            while (br.read() != -1) { /* vacío */ }
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        long finBR = System.currentTimeMillis();

        long inicioBRL = System.currentTimeMillis();
        try (BufferedReader br = new BufferedReader(
                new FileReader(archivoGrande))) {
            while (br.readLine() != null) { /* vacío */ }
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        long finBRL = System.currentTimeMillis();

        System.out.println("FileReader (char a char):     " + (finFR - inicioFR) + " ms");
        System.out.println("BufferedReader (char a char): " + (finBR - inicioBR) + " ms");
        System.out.println("BufferedReader (readLine):    " + (finBRL - inicioBRL) + " ms");

        // 8. Escribir con newLine() portable
        System.out.println("\n--- newLine() portable ---");
        try (BufferedWriter bw = new BufferedWriter(
                new FileWriter("/tmp/cap53_newline.txt"))) {
            bw.write("Windows usa \\r\\n");
            bw.newLine();
            bw.write("Linux/Mac usan \\n");
            bw.newLine();
            bw.write("¡Salto portable!");
            bw.newLine();
            System.out.println("Archivo con saltos portables creado.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 9. Leer archivo línea por línea con contador
        System.out.println("\n--- Contador de líneas ---");
        try (BufferedReader br = new BufferedReader(new FileReader(archivoGrande))) {
            long totalLineas = br.lines().count();
            System.out.println("Total líneas: " + totalLineas);
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 10. Combinación con PrintWriter
        System.out.println("\n--- PrintWriter + BufferedWriter ---");
        try (PrintWriter pw = new PrintWriter(
                new BufferedWriter(new FileWriter("/tmp/cap53_print.txt")))) {
            pw.println("Reporte generado con PrintWriter");
            pw.println("================================");
            pw.printf("Fecha: %s%n", new java.util.Date());
            pw.printf("Valor: %,.2f%n", 1234567.89);
            System.out.println("PrintWriter completado.");
            System.out.println("Error: " + pw.checkError());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 11. Leer desde consola (simulado con System.in y BufferedReader)
        System.out.println("\n--- Lectura desde consola (simulado) ---");
        System.out.println("(En un entorno real leería de System.in)");
        // Esto funcionaría en una consola interactiva:
        // try (BufferedReader br = new BufferedReader(
        //         new InputStreamReader(System.in))) {
        //     System.out.print("Escribe algo: ");
        //     String entrada = br.readLine();
        //     System.out.println("Escribiste: " + entrada);
        // }

        // 12. Tamaño predeterminado del buffer
        System.out.println("\n--- Tamaño del buffer ---");
        // El tamaño predeterminado es 8192 en la mayoría de JDK.
        // Podemos verificarlo (no hay getter público, pero podemos asumirlo).
        System.out.println("Tamaño buffer default: 8192 chars (8 KB)");

        // 13. Leer archivo con skip()
        System.out.println("\n--- skip() con BufferedReader ---");
        try (BufferedReader br = new BufferedReader(new FileReader(archivo))) {
            System.out.println("Primeras líneas saltando la primera:");
            br.readLine(); // Saltar primera línea
            String linea;
            while ((linea = br.readLine()) != null) {
                System.out.println("  " + linea);
            }
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 14. TransferTo (BufferedReader hereda de Reader)
        System.out.println("\n--- transferTo() con BufferedReader ---");
        try (BufferedReader br = new BufferedReader(new FileReader(archivo));
             BufferedWriter bw = new BufferedWriter(
                     new FileWriter("/tmp/cap53_transfer.txt"))) {
            br.transferTo(bw);
            System.out.println("transferTo completado.");
        } catch (IOException e) {
            System.err.println("Error transferTo: " + e.getMessage());
        }

        // Limpieza
        System.out.println("\n--- Limpieza ---");
        new java.io.File(archivo).delete();
        new java.io.File(archivoGrande).delete();
        new java.io.File("/tmp/cap53_newline.txt").delete();
        new java.io.File("/tmp/cap53_print.txt").delete();
        new java.io.File("/tmp/cap53_transfer.txt").delete();
        System.out.println("Archivos temporales eliminados.");

        System.out.println("\n=== FIN CAPÍTULO 53 ===");
    }
}

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

• BufferedReader y BufferedWriter añaden buffering a la E/S de caracteres
  usando el patrón Decorador. El buffer interno reduce las llamadas al SO.

• BufferedReader.readLine() lee líneas completas devolviendo String o null
  al llegar al final del archivo. Es el método más usado para leer texto.

• BufferedReader.lines() (Java 8+) devuelve un Stream<String> para
  procesamiento funcional.

• mark() y reset() permiten marcar posiciones y retroceder en la lectura.

• BufferedWriter.newLine() escribe el separador de línea portable según
  la plataforma (\r\n en Windows, \n en Unix/Mac).

• flush() fuerza la escritura inmediata del buffer al destino.

• El tamaño predeterminado del buffer es 8192 caracteres. Se puede
  especificar un tamaño personalizado en el constructor.

• BufferedReader + FileReader puede ser 10-100 veces más rápido que
  FileReader solo para lectura carácter por carácter.

• BufferedReader también se usa para lectura eficiente desde consola
  (System.in) usando InputStreamReader como puente.

   tabla alineada. Soporta comillas en los campos.

   real, mostrando las nuevas líneas que se añadan (como tail -f).
   Usa BufferedReader con mark/reset o RandomAccessFile.

   texto de gran tamaño (100 MB, 500 MB) con datos ficticios para probar
   rendimiento. Debe aceptar tamaño y nombre de archivo como argumentos.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿BufferedReader es seguro para usar con múltiples hilos?
R: No, BufferedReader no está sincronizado. Si varios hilos leen del
   mismo BufferedReader, los resultados son impredecibles. Usa
   sincronización externa o crea un BufferedReader por hilo.

P: ¿Puedo usar readLine() y luego read() en el mismo BufferedReader?
R: Sí, pero con cuidado. readLine() consume hasta el salto de línea
   inclusive, y read() empezará en el primer carácter de la siguiente
   línea. El buffer interno mantiene la posición correcta.

P: ¿Cuál es el tamaño de buffer óptimo?
R: Para la mayoría de aplicaciones, el valor predeterminado (8192) es
   adecuado. Para archivos muy grandes (>500 MB) o accesos lentos (disco
   de red), probar con 32768 o 65536. El beneficio de buffers más grandes
   sigue la ley de rendimientos decrecientes.

P: ¿Por qué mi BufferedWriter no escribe nada hasta que cierro?
R: Porque los datos se acumulan en el buffer interno. Si necesitas
   escritura inmediata, llama a flush() después de cada escritura
   importante, especialmente en aplicaciones de logging.

P: ¿newLine() es realmente necesario o puedo usar \n?
R: newLine() garantiza portabilidad. Si usas \n directamente, los
   archivos generados en Windows se verán como una sola línea en
   el Bloc de Notas (aunque otros editores lo manejan bien).
   newLine() usa System.lineSeparator() internamente.

P: ¿BufferedReader con InputStreamReade y System.in funciona siempre?
R: Sí, pero ten en cuenta que System.in es un solo InputStream compartido.
   Si envuelves System.in en múltiples BufferedReader, solo el primero
   leerá datos correctamente porque el buffer consumirá bytes.

  "PROCESADOR DE ARCHIVOS DE LOG CON ANÁLISIS EN TIEMPO REAL"

  Diseña un sistema que procese archivos de log en formato:

  [2025-01-15 14:30:22] [INFO]  [main]    Aplicación iniciada
  [2025-01-15 14:30:23] [ERROR] [worker1] Conexión fallida: timeout
  [2025-01-15 14:30:25] [WARN]  [worker1] Reintentando conexión...
  [2025-01-15 14:31:00] [ERROR] [worker2] Error crítico en módulo X

  Requisitos:

  1. Parser de log: Cada línea se parsea en: timestamp, nivel, hilo,
     mensaje. Usa BufferedReader para leer.

  2. Filtros en tiempo real: Permite filtrar por nivel (ERROR, WARN,
     INFO, DEBUG), por hilo, o por rango de fechas.

  3. Estadísticas: Calcula y muestra cada 5 segundos:
     • Total de líneas procesadas.
     • Conteo por nivel de severidad.
     • Los 3 hilos con más errores.
     • Tasa de procesamiento (líneas/segundo).

  4. Exportación de reportes: Genera un archivo de reporte con las
     estadísticas al detener el procesamiento (Ctrl+C).

  5. Modo tail: Monitorea un archivo que está siendo escrito (como
     tail -f), procesando nuevas líneas en tiempo real.

  6. Alertas: Si se detectan más de 10 ERRORES en un minuto, escribe
     una alerta en un archivo de alertas separado.

  7. Mínimo 4 clases: LogParser, LogStats, LogReporter, LogMonitor.

  VALORACIÓN:
  • 3 estrellas: Parseo + filtros + estadísticas básicas.
  • 4 estrellas: Modo tail + exportación de reportes.
  • 5 estrellas: Alertas configurables + interfaz interactiva por
    consola (comandos como /stats, /filter ERROR, /tail).
