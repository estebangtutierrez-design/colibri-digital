BIENVENIDA
----------
Bienvenido al Capitulo 59! Toda aplicacion necesita configuracion: rutas de
base de datos, URLs de servicios, valores por defecto, etc. La clase
java.util.Properties es la forma mas simple y clasica de manejar
configuracion en Java. Permite almacenar pares clave=valor en archivos
de texto plano (.properties) o en XML. En este capitulo aprenderas a
cargar, modificar y guardar archivos de propiedades, asi como a usar
valores por defecto y tecnicas avanzadas como propiedades encadenadas.

FRASE DEL DIA
-------------
"La configuracion es el ADN de una aplicacion."
— Anonimo

OBJETIVOS DEL CAPITULO
----------------------
• Comprender el formato de archivos .properties.
• Crear y cargar archivos de propiedades con load().
• Guardar propiedades con store().
• Leer y escribir valores con getProperty() y setProperty().
• Usar valores por defecto en getProperty().
• Trabajar con XML: loadFromXML() y storeToXML().
• Iterar sobre todas las propiedades.
• Encadenar propiedades (System.getProperties() + archivo).
• Internacionalizacion con ResourceBundle.
• Buenas practicas en configuracion de aplicaciones.

1. LA CLASE PROPERTIES
-----------------------

Properties extiende Hashtable<Object, Object> y esta disenada
especificamente para manejar archivos de configuracion en formato
clave=valor.

  public class Properties extends Hashtable<Object, Object> {
      public Properties();
      public Properties(Properties defaults);
      // ...
  }

Caracteristicas principales:
  • Almacena pares String -> String.
  • Soporta valores por defecto encadenados.
  • Metodos load() y store() para I/O.
  • Compatible con XML.
  • No es thread-safe (sincronizar externamente si es necesario).

2. FORMATO DEL ARCHIVO .properties
-----------------------------------

  # Comentario (linea que empieza con # o !)
  clave=valor
  otra.clave = otro valor
  ruta = C:\\Users\\usuario\\docs  # Barras invertidas se escapan
  mensaje = Linea 1\\
  Linea 2                           # Continuacion con \\
  bandera = true
  numero = 42

Reglas del formato:
  • # o ! al inicio de linea = comentario.
  • clave y valor separados por =, :, o espacio en blanco.
  • Los espacios alrededor de la clave/valor se ignoran.
  • \ al final de linea = continuacion en la siguiente linea.
  • Secuencias de escape: \n, \r, \t, \\, \uXXXX (Unicode).
  • No usar acentos directamente (usar \uXXXX o UTF-8 con load(Reader)).

3. CONSTRUCTORES
------------------

  Properties props = new Properties(); // Sin valores por defecto

  Properties defaults = new Properties();
  defaults.setProperty("host", "localhost");
  defaults.setProperty("puerto", "8080");
  Properties props = new Properties(defaults); // Con defaults

4. LOAD() — CARGAR DESDE ARCHIVO
----------------------------------

  // Desde InputStream (ISO-8859-1 por defecto)
  try (FileInputStream fis = new FileInputStream("/tmp/config.properties")) {
      Properties props = new Properties();
      props.load(fis);
      System.out.println(props.getProperty("host"));
  }

  // Desde Reader (respeta codificacion)
  try (FileReader fr = new FileReader("/tmp/config.properties",
          StandardCharsets.UTF_8)) {
      Properties props = new Properties();
      props.load(fr);
  }

  // Desde XML
  try (FileInputStream fis = new FileInputStream("/tmp/config.xml")) {
      Properties props = new Properties();
      props.loadFromXML(fis);
  }

5. STORE() — GUARDAR A ARCHIVO
--------------------------------

  try (FileOutputStream fos = new FileOutputStream("/tmp/config.properties")) {
      Properties props = new Properties();
      props.setProperty("host", "localhost");
      props.setProperty("puerto", "3306");
      props.store(fos, "Configuracion de base de datos");
  }

  // Como XML
  try (FileOutputStream fos = new FileOutputStream("/tmp/config.xml")) {
      Properties props = new Properties();
      props.setProperty("host", "localhost");
      props.setProperty("puerto", "3306");
      props.storeToXML(fos, "Config XML", StandardCharsets.UTF_8.name());
  }

6. GETPROPERTY() Y SETPROPERTY()
----------------------------------

  Properties props = new Properties();

  // Setear
  props.setProperty("db.url", "jdbc:mysql://localhost:3306/mydb");
  props.setProperty("db.user", "admin");
  props.setProperty("db.password", "secreta");
  props.setProperty("db.pool.size", "10");

  // Obtener
  String url = props.getProperty("db.url");
  String user = props.getProperty("db.user");
  String pass = props.getProperty("db.password");

  // Con valor por defecto
  String timeout = props.getProperty("db.timeout", "30000");

7. VALORES POR DEFECTO ENCADENADOS
------------------------------------

  Properties globalDefaults = new Properties();
  globalDefaults.setProperty("debug", "false");
  globalDefaults.setProperty("locale", "en_US");

  Properties appDefaults = new Properties(globalDefaults);
  appDefaults.setProperty("debug", "true"); // Sobreescribe

  Properties appProps = new Properties(appDefaults);
  appProps.load(new FileReader("app.properties"));

  // Orden de busqueda: appProps -> appDefaults -> globalDefaults

8. ITERAR SOBRE PROPIEDADES
-----------------------------

  // keySet()
  for (String key : props.stringPropertyNames()) {
      System.out.println(key + " = " + props.getProperty(key));
  }

  // list(PrintStream) — util para depuracion
  props.list(System.out);

  // entrySet()
  for (Map.Entry<Object, Object> entry : props.entrySet()) {
      System.out.println(entry.getKey() + " = " + entry.getValue());
  }

9. SYSTEM.PROPERTIES
---------------------

Java mantiene propiedades del sistema que pueden combinarse con las
propias de la aplicacion:

  // Listar propiedades del sistema
  System.getProperties().list(System.out);

  // Obtener una propiedad del sistema
  String javaVersion = System.getProperty("java.version");
  String userHome = System.getProperty("user.home");
  String osName = System.getProperty("os.name");

  // Establecer propiedad del sistema
  System.setProperty("mi.app.version", "1.0.0");

  // Cargar desde archivo y combinar con sistema
  Properties props = new Properties();
  props.load(new FileReader("config.properties"));
  props.putAll(System.getProperties()); // Sistema sobreescribe

10. RESOURCEBUNDLE — INTERNACIONALIZACION
------------------------------------------

ResourceBundle usa archivos .properties para internacionalizacion:

  // Archivos: messages_en_US.properties, messages_es_ES.properties
  // messages_en_US.properties:
  //   greeting=Hello
  //   farewell=Goodbye

  // messages_es_ES.properties:
  //   greeting=Hola
  //   farewell=Adios

  Locale locale = new Locale("es", "ES");
  ResourceBundle bundle = ResourceBundle.getBundle("messages", locale);
  String saludo = bundle.getString("greeting"); // "Hola"

11. Buenas Practicas
---------------------

  • No hardcodear configuraciones en el codigo.
  • Usar archivos .properties para configuracion externa.
  • No incluir contrasenas en texto plano (usar variables de entorno).
  • Usar valores por defecto para parametros opcionales.
  • Documentar cada propiedad con comentarios en el archivo.
  • Separar configuracion por entorno (dev, test, prod).
  • No modificar archivos .properties dentro del JAR (usar externos).
  • Considerar formatos modernos como YAML o JSON para configuraciones
    complejas.

import java.io.*;
import java.nio.charset.StandardCharsets;
import java.util.*;

public class Capitulo59Propiedades {

    public static void main(String[] args) {
        System.out.println("=== CAPITULO 59: PROPIEDADES ===\n");

        // 1. Crear y guardar propiedades
        System.out.println("--- Crear y guardar ---");
        Properties props = new Properties();
        props.setProperty("db.host", "localhost");
        props.setProperty("db.port", "3306");
        props.setProperty("db.name", "mi_app");
        props.setProperty("db.user", "admin");
        props.setProperty("db.password", "s3cr3t");
        props.setProperty("app.debug", "true");
        props.setProperty("app.version", "2.0.0");
        props.setProperty("app.locale", "es_ES");

        try (FileOutputStream fos = new FileOutputStream("/tmp/cap59_config.properties")) {
            props.store(fos, "Configuracion de Mi App");
            System.out.println("Propiedades guardadas.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 2. Cargar propiedades
        System.out.println("\n--- Cargar propiedades ---");
        Properties cargadas = new Properties();
        try (FileInputStream fis = new FileInputStream("/tmp/cap59_config.properties")) {
            cargadas.load(fis);
            System.out.println("Propiedades cargadas (" + cargadas.size() + ")");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 3. Leer valores
        System.out.println("\n--- Leer valores ---");
        String host = cargadas.getProperty("db.host");
        String port = cargadas.getProperty("db.port");
        String name = cargadas.getProperty("db.name");
        String user = cargadas.getProperty("db.user");
        String debug = cargadas.getProperty("app.debug");

        System.out.println("Host: " + host);
        System.out.println("Port: " + port);
        System.out.println("DB: " + name);
        System.out.println("User: " + user);
        System.out.println("Debug: " + debug);

        // 4. Valor por defecto
        System.out.println("\n--- Valor por defecto ---");
        String timeout = cargadas.getProperty("db.timeout", "30000");
        String maxConns = cargadas.getProperty("db.max.connections", "10");
        System.out.println("timeout: " + timeout + " (default si no existe)");
        System.out.println("maxConnections: " + maxConns);

        // 5. Iterar sobre propiedades
        System.out.println("\n--- Iterar sobre propiedades ---");
        for (String key : cargadas.stringPropertyNames()) {
            System.out.println("  " + key + " = " + cargadas.getProperty(key));
        }

        // 6. List (depuracion)
        System.out.println("\n--- list() ---");
        cargadas.list(System.out);

        // 7. Valores por defecto encadenados
        System.out.println("\n--- Defaults encadenados ---");
        Properties globalDefaults = new Properties();
        globalDefaults.setProperty("app.debug", "false");
        globalDefaults.setProperty("app.locale", "en_US");
        globalDefaults.setProperty("app.theme", "light");

        Properties appProps = new Properties(globalDefaults);
        appProps.setProperty("app.debug", "true"); // Sobreescribe
        appProps.setProperty("app.locale", "es_ES");

        System.out.println("debug: " + appProps.getProperty("app.debug"));
        System.out.println("locale: " + appProps.getProperty("app.locale"));
        System.out.println("theme: " + appProps.getProperty("app.theme"));
        System.out.println("noexiste: " + appProps.getProperty("noexiste", "valor_default"));

        // 8. Guardar y cargar en XML
        System.out.println("\n--- XML ---");
        Properties xmlProps = new Properties();
        xmlProps.setProperty("servicio.url", "https://api.ejemplo.com/v1");
        xmlProps.setProperty("servicio.timeout", "5000");
        xmlProps.setProperty("servicio.retries", "3");

        try (FileOutputStream fos = new FileOutputStream("/tmp/cap59_config.xml")) {
            xmlProps.storeToXML(fos, "Configuracion en XML",
                    StandardCharsets.UTF_8.name());
            System.out.println("XML guardado.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        Properties xmlCargadas = new Properties();
        try (FileInputStream fis = new FileInputStream("/tmp/cap59_config.xml")) {
            xmlCargadas.loadFromXML(fis);
            System.out.println("XML cargado:");
            xmlCargadas.list(System.out);
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 9. Propiedades del sistema
        System.out.println("\n--- Propiedades del sistema ---");
        System.out.println("java.version: " + System.getProperty("java.version"));
        System.out.println("user.home: " + System.getProperty("user.home"));
        System.out.println("user.dir: " + System.getProperty("user.dir"));
        System.out.println("os.name: " + System.getProperty("os.name"));
        System.out.println("file.separator: " + System.getProperty("file.separator"));

        // 10. System.setProperty
        System.out.println("\n--- System.setProperty ---");
        System.setProperty("mi.app.inicio",
                new java.util.Date().toString());
        System.out.println("App inicio: " + System.getProperty("mi.app.inicio"));

        // 11. Cargar desde Reader (codificacion explicita)
        System.out.println("\n--- Carga con Reader (UTF-8) ---");
        try (FileReader fr = new FileReader("/tmp/cap59_config.properties",
                StandardCharsets.UTF_8)) {
            Properties readerProps = new Properties();
            readerProps.load(fr);
            System.out.println("Cargado con Reader: " + readerProps.size() + " props");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 12. Modificar y guardar
        System.out.println("\n--- Modificar y guardar ---");
        Properties editProps = new Properties();
        try (FileReader fr = new FileReader("/tmp/cap59_config.properties",
                StandardCharsets.UTF_8)) {
            editProps.load(fr);
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        editProps.setProperty("app.version", "2.1.0");
        editProps.setProperty("app.updated",
                new java.util.Date().toString());
        editProps.remove("db.password"); // No guardar contrasena

        try (FileOutputStream fos = new FileOutputStream(
                "/tmp/cap59_config_updated.properties")) {
            editProps.store(fos, "Configuracion actualizada");
            System.out.println("Propiedades actualizadas guardadas.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 13. Convertir a mapa
        System.out.println("\n--- Properties a Map ---");
        Map<String, String> mapa = new HashMap<>();
        for (String key : editProps.stringPropertyNames()) {
            mapa.put(key, editProps.getProperty(key));
        }
        System.out.println("Mapa: " + mapa.size() + " entradas");

        // 14. Properties desde entrada del usuario
        System.out.println("\n--- Creacion programatica ---");
        Properties commandLineProps = new Properties();
        commandLineProps.setProperty("input.file", "/tmp/datos.txt");
        commandLineProps.setProperty("output.file", "/tmp/resultados.txt");
        commandLineProps.setProperty("encoding", "UTF-8");
        commandLineProps.setProperty("overwrite", "false");

        // Simular procesamiento
        String input = commandLineProps.getProperty("input.file");
        String output = commandLineProps.getProperty("output.file");
        String encoding = commandLineProps.getProperty("encoding", "UTF-8");
        boolean overwrite = Boolean.parseBoolean(
                commandLineProps.getProperty("overwrite", "false"));

        System.out.println("Input: " + input);
        System.out.println("Output: " + output);
        System.out.println("Encoding: " + encoding);
        System.out.println("Overwrite: " + overwrite);

        // Limpieza
        System.out.println("\n--- Limpieza ---");
        String[] archivos = {
            "/tmp/cap59_config.properties",
            "/tmp/cap59_config.xml",
            "/tmp/cap59_config_updated.properties"
        };
        for (String a : archivos) {
            new java.io.File(a).delete();
        }
        System.out.println("Archivos temporales eliminados.");

        System.out.println("\n=== FIN CAPITULO 59 ===");
    }
}

================================================================================
  RESUMEN DEL CAPITULO
================================================================================

• Properties (java.util.Properties) almacena pares clave=valor como String.
  Extiende Hashtable y es la forma clasica de configuracion en Java.

• load(InputStream) carga desde archivo .properties en ISO-8859-1.
  load(Reader) respeta la codificacion especificada.
  loadFromXML(InputStream) carga desde XML.

• store(OutputStream, comments) guarda en formato .properties.
  storeToXML(OutputStream, comment, encoding) guarda en XML.

• getProperty(key) y getProperty(key, defaultValue) obtienen valores.
  setProperty(key, value) establece valores.

• Valores por defecto encadenados: se pasan al constructor
  new Properties(defaults).

• stringPropertyNames() devuelve el conjunto de claves.
  list(PrintStream) es util para depuracion.

• System.getProperties() devuelve las propiedades de la JVM.
  System.getProperty(key) obtiene propiedades del sistema.

• ResourceBundle usa archivos .properties para internacionalizacion,
  seleccionando el archivo segun el Locale.

• Buenas practicas: no hardcodear config, usar defaults, separar por
  entorno, no incluir secretos en propiedades.

================================================================================
  EJERCICIOS PROPUESTOS
================================================================================

1. Cargador de configuracion: Crea una clase ConfigManager que cargue
   propiedades desde multiples fuentes en orden de prioridad:
   1) Sistema (system properties), 2) Archivo externo,
   3) Archivo interno en JAR, 4) Valores por defecto.

2. Editor de propiedades: Programa interactivo que liste, agregue,
   modifique y elimine propiedades de un archivo .properties.
   Debe guardar los cambios al salir.

3. Validador de configuracion: Dado un archivo .properties y un archivo
   de esquema (que define que claves son obligatorias, opcionales, tipos
   esperados), valida que la configuracion sea correcta.

4. Exportador a JSON: Lee un archivo .properties y genera un archivo
   JSON equivalente con la misma estructura.

5. Variables de entorno: Implementa un Properties que soporte
   referencias a variables de entorno: db.password=${DB_PASSWORD}.
   Resuelve las referencias al cargar.

6. Propiedades por ambiente: Carga configuracion comun (config.properties)
   y luego sobreescribe con config-{env}.properties segun el ambiente
   (dev, test, prod). Usa System.getProperty("env") para detectar.

7. Monitor de propiedades: Programa que monitoree un archivo .properties
   y cuando cambie, recargue automaticamente la configuracion sin
   reiniciar la aplicacion (hot reload).

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: Por que mis acentos no se ven bien al cargar un .properties?
R: load(InputStream) usa ISO-8859-1. Usa load(Reader) con la codificacion
   correcta (UTF-8) para soportar acentos y otros caracteres.

P: Properties es thread-safe?
R: No. Properties extiende Hashtable que es sincronizado, pero las
   operaciones compuestas (get + set) no son atomicas. Sincroniza
   externamente si usas multiples hilos.

P: Como guardar Properties en orden alfabetico?
R: Properties no garantiza orden. Usa un TreeMap para ordenar:
   Properties ordenadas = new Properties();
   TreeMap<String, String> sorted = new TreeMap<>(props);
   sorted.forEach((k,v) -> ordenadas.setProperty(k, v));

P: Cual es la diferencia entre Properties y ResourceBundle?
R: Properties es para configuracion general. ResourceBundle extiende
   Properties con soporte de internacionalizacion (locale) y
   jerarquia de archivos.

P: Puedo usar Properties para leer archivos YAML?
R: No directamente. Properties solo lee .properties y XML. Para YAML
   usa bibliotecas como SnakeYAML.

P: Como proteger contrasenas en archivos .properties?
R: No guardes contrasenas en texto plano. Usa variables de entorno,
   un vault de secretos (HashiCorp Vault), o encripta el valor y
   desencriptalo en memoria al cargar.

  DESAFIO DEL ARQUITECTO

  "SISTEMA DE CONFIGURACION JERARQUICO CON HOT RELOAD"

  Disena un sistema de configuracion que soporte propiedades jerarquicas,
  herencia, tipos, validacion y recarga en caliente.

  Requisitos:

  1. Formato jerarquico: Soporta claves anidadas usando puntos:
     database.host, database.port, database.credentials.user.
     Internamente, almacena como propiedades planas pero permite
     acceso estructurado: config.getSection("database").get("host").

  2. Tipos inferidos y convertidos: getInt(), getBoolean(), getDouble(),
     getDuration(), getFile(). Con versiones con default:
     getInt("key", 42).

  3. Validacion al cargar: Define un esquema con tipo, rango, valores
     permitidos, obligatoriedad. Si la validacion falla, reporta todos
     los errores y no aplica la configuracion.

  4. Hot reload: Monitorea el archivo de configuracion con WatchService
     (NIO). Al detectar cambios, recarga y valida automaticamente.
     Notifica a los componentes registrados via listener/callback.

  5. Perfiles: Soporta perfiles (dev, test, prod). Las propiedades
     especificas del perfil sobreescriben las bases. Se activa con
     System.setProperty("profile", "dev") o argumento de JVM.

  6. Variables interpoladas: Soporta referencias a otras propiedades:
     database.url = jdbc:mysql://${database.host}:${database.port}/db

  7. Exportacion: Exporta la configuracion activa (resuelta) a .properties,
     XML, JSON y YAML.

  8. Minimo 5 clases: Config, ConfigSection, ConfigSchema,
     ConfigValidator, ConfigWatcher.

  VALORACION:
  3*: Carga basica + tipos + perfiles.
  4*: Validacion + hot reload + variables interpoladas.
  5*: Exportacion multiple + listeners + testing completo.
