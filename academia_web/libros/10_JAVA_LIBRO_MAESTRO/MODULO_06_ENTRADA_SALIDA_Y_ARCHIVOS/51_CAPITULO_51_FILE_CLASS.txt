1. INTRODUCCIÓN A LA CLASE FILE
--------------------------------

La clase java.io.File es una representación abstracta de rutas de archivos y
directorios. Desde su aparición en JDK 1.0, ha sido la clase fundamental para
interactuar con el sistema de archivos. Es importante entender que File no
representa el contenido del archivo, sino su ubicación (path) y atributos.

Un objeto File puede referirse a:
  • Un archivo que existe o que aún no existe.
  • Un directorio.
  • Una ruta que no corresponde a ningún recurso real.

Ejemplo conceptual:
  File f = new File("/home/usuario/documento.txt");
  Este objeto representa la ruta /home/usuario/documento.txt, pero no crea
  ningún archivo físico hasta que invoquemos createNewFile().

2. CONSTRUCTORES DE FILE
------------------------

La clase File ofrece varios constructores:

  a) File(String pathname)
     Crea una instancia con la ruta dada.
     File f = new File("/tmp/miarchivo.txt");

  b) File(String parent, String child)
     Construye una ruta combinando el directorio padre y el nombre hijo.
     File f = new File("/tmp", "miarchivo.txt");

  c) File(File parent, String child)
     Similar al anterior, pero el padre es otro objeto File.
     File dir = new File("/tmp");
     File f = new File(dir, "miarchivo.txt");

  d) File(URI uri)
     Crea una instancia a partir de un URI.
     File f = new File(new URI("file:///tmp/miarchivo.txt"));

3. SEPARADORES DE RUTAS
-----------------------

Java proporciona constantes estáticas para manejar separadores de forma
independiente del sistema operativo:

  • File.separator       → "/" en Linux/Mac, "\\" en Windows
  • File.separatorChar   → versión char del separador
  • File.pathSeparator   → ":" en Linux/Mac, ";" en Windows
  • File.pathSeparatorChar → versión char

Buenas prácticas: usar File.separator en lugar de codificar "/" o "\\".

4. VERIFICAR EXISTENCIA: exists()
---------------------------------

El método exists() devuelve true si el archivo o directorio representado
por el objeto File existe físicamente en el sistema de archivos.

  File f = new File("/tmp/archivo.txt");
  if (f.exists()) {
      System.out.println("El archivo existe.");
  } else {
      System.out.println("El archivo no existe.");
  }

Es importante siempre verificar la existencia antes de realizar operaciones
que asuman que el recurso está presente.

5. CREAR ARCHIVOS: createNewFile()
-----------------------------------

El método createNewFile() crea un archivo vacío si y solo si no existe ya.
Devuelve boolean indicando si se creó (true) o ya existía (false).

  File f = new File("/tmp/nuevo_archivo.txt");
  try {
      boolean creado = f.createNewFile();
      if (creado) {
          System.out.println("Archivo creado exitosamente.");
      } else {
          System.out.println("El archivo ya existe.");
      }
  } catch (IOException e) {
      System.err.println("Error al crear el archivo: " + e.getMessage());
  }

Detalles importantes:
  • Lanza IOException si ocurre un error de E/S.
  • Lanza SecurityException si no hay permisos de escritura.
  • La operación es atómica: ningún otro proceso puede crear el archivo
    entre la verificación y la creación.

6. CREAR DIRECTORIOS: mkdir() Y mkdirs()
----------------------------------------

mkdir() crea un directorio. El directorio padre debe existir.
mkdirs() crea el directorio y todos los directorios padre necesarios.

  File dir = new File("/tmp/mi_directorio");
  if (dir.mkdir()) {
      System.out.println("Directorio creado.");
  } else {
      System.out.println("No se pudo crear el directorio.");
  }

  // mkdirs() crea toda la jerarquía
  File jerarquia = new File("/tmp/a/b/c/d");
  if (jerarquia.mkdirs()) {
      System.out.println("Jerarquía de directorios creada.");
  }

  /* Diferencia clave:
     mkdir()  → falla si el padre no existe.
     mkdirs() → crea los padres automáticamente.
  */

7. LISTAR CONTENIDO: listFiles() Y list()
-----------------------------------------

list() devuelve un String[] con los nombres de archivos y directorios.
listFiles() devuelve un File[] con objetos File completos.

  File directorio = new File("/tmp");
  if (directorio.isDirectory()) {
      String[] nombres = directorio.list();
      for (String nombre : nombres) {
          System.out.println(nombre);
      }

      File[] archivos = directorio.listFiles();
      for (File archivo : archivos) {
          System.out.println(archivo.getAbsolutePath());
      }
  }

Filtrado con FilenameFilter y FileFilter:

  // FilenNameFilter: filtra por nombre
  FilenameFilter filtroTxt = (dir, nombre) -> nombre.endsWith(".txt");
  String[] txts = directorio.list(filtroTxt);

  // FileFilter: filtra por objeto File
  FileFilter soloDirectorios = File::isDirectory;
  File[] dirs = directorio.listFiles(soloDirectorios);

8. ELIMINAR: delete()
---------------------

delete() elimina un archivo o directorio. El directorio debe estar vacío.

  File f = new File("/tmp/archivo_a_eliminar.txt");
  if (f.exists()) {
      boolean eliminado = f.delete();
      if (eliminado) {
          System.out.println("Archivo eliminado.");
      } else {
          System.out.println("No se pudo eliminar el archivo.");
      }
  }

Para eliminar directorios con contenido debemos recorrer recursivamente:

  public static void eliminarRecursivo(File f) {
      if (f.isDirectory()) {
          File[] hijos = f.listFiles();
          if (hijos != null) {
              for (File h : hijos) {
                  eliminarRecursivo(h);
              }
          }
      }
      f.delete();
  }

deleteOnExit() programa la eliminación cuando la JVM termina:

  File temp = new File("/tmp/temporal.txt");
  temp.createNewFile();
  temp.deleteOnExit(); // Se borrará al finalizar la JVM

9. METADATOS DEL ARCHIVO
-------------------------

File proporciona varios métodos para consultar metadatos:

  • length()         → Tamaño en bytes.
  • lastModified()   → Timestamp de última modificación (long).
  • canRead()        → ¿Se puede leer?
  • canWrite()       → ¿Se puede escribir?
  • canExecute()     → ¿Se puede ejecutar?
  • isFile()         → ¿Es un archivo regular?
  • isDirectory()    → ¿Es un directorio?
  • isHidden()       → ¿Está oculto?
  • getAbsolutePath()→ Ruta absoluta.
  • getCanonicalPath()→ Ruta canónica (sin . ni ..).
  • getName()        → Nombre del archivo/carpeta.
  • getParent()      → Directorio padre.
  • toPath()         → Convierte a java.nio.file.Path.

  File f = new File("/tmp/ejemplo.txt");
  System.out.println("Tamaño: " + f.length() + " bytes");
  System.out.println("Última modificación: " + new Date(f.lastModified()));
  System.out.println("Lectura: " + f.canRead());
  System.out.println("Escritura: " + f.canWrite());
  System.out.println("Ejecución: " + f.canExecute());
  System.out.println("Absoluta: " + f.getAbsolutePath());
  System.out.println("Canónica: " + f.getCanonicalPath());

10. RUTAS ABSOLUTAS VS RELATIVAS
---------------------------------

  • Ruta absoluta: Comienza desde la raíz del sistema de archivos.
    Ejemplo Linux: "/home/usuario/docs/archivo.txt"
    Ejemplo Windows: "C:\\Users\\usuario\\docs\\archivo.txt"

  • Ruta relativa: Se interpreta desde el directorio de trabajo actual.
    Ejemplo: "docs/archivo.txt"

  File absoluto = new File("/tmp/archivo.txt");
  File relativo = new File("archivo.txt"); // directorio de trabajo actual

  System.out.println(absoluto.isAbsolute()); // true
  System.out.println(relativo.isAbsolute()); // false

11. RENOMBRAR: renameTo()
--------------------------

renameTo(File dest) renombra o mueve un archivo. Devuelve boolean.

  File origen = new File("/tmp/viejo.txt");
  File destino = new File("/tmp/nuevo.txt");
  if (origen.renameTo(destino)) {
      System.out.println("Renombrado exitosamente.");
  } else {
      System.out.println("El renombrado falló.");
  }

Nota: renameTo() puede fallar entre sistemas de archivos diferentes. Para
operaciones más robustas, usar Files.move() de NIO.

12. ARCHIVOS TEMPORALES
------------------------

createTempFile() crea archivos temporales en el directorio temporal del SO.

  File temp = File.createTempFile("prefijo_", ".tmp");
  System.out.println("Temp: " + temp.getAbsolutePath());
  temp.deleteOnExit();

  // Especificar directorio
  File temp2 = File.createTempFile("log_", ".log", new File("/tmp"));

13. LIMITACIONES DE LA CLASE FILE
---------------------------------

La clase File tiene varias limitaciones que llevaron a la creación del
paquete java.nio.file (NIO.2) en Java 7:

  • No puede leer ni escribir contenido de archivos (solo metadatos).
  • Los métodos delete() y renameTo() devuelven boolean en lugar de lanzar
    excepciones claras.
  • No soporta enlaces simbólicos de forma adecuada.
  • No proporciona notificaciones de cambios en el sistema de archivos.
  • Escalabilidad limitada para operaciones masivas.
  • La gestión de atributos es rudimentaria.

A pesar de estas limitaciones, File sigue siendo ampliamente utilizado y es
fundamental entenderlo antes de migrar a NIO. En el capítulo 58 abordaremos
NIO en profundidad.

================================================================================
  CÓDIGO DE EJEMPLO COMPLETO
================================================================================

import java.io.File;
import java.io.FilenameFilter;
import java.io.FileFilter;
import java.io.IOException;
import java.util.Date;

public class Capitulo51FileClass {

    public static void main(String[] args) {
        System.out.println("=== CAPÍTULO 51: FILE CLASS ===\n");

        // 1. Crear objetos File
        File directorioTmp = new File("/tmp");
        File archivoEjemplo = new File("/tmp/ejemplo_file.txt");
        File directorioNuevo = new File("/tmp/mi_directorio_java");
        File jerarquia = new File("/tmp/nivel1/nivel2/nivel3");

        // 2. Verificar existencia
        System.out.println("--- exists() ---");
        System.out.println("¿/tmp existe? " + directorioTmp.exists());
        System.out.println("¿ejemplo_file.txt existe? " + archivoEjemplo.exists());

        // 3. Crear archivo
        System.out.println("\n--- createNewFile() ---");
        try {
            boolean creado = archivoEjemplo.createNewFile();
            System.out.println("Archivo creado: " + creado + " | Ruta: "
                    + archivoEjemplo.getAbsolutePath());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 4. Crear directorio
        System.out.println("\n--- mkdir() y mkdirs() ---");
        boolean dirCreado = directorioNuevo.mkdir();
        System.out.println("Directorio creado: " + dirCreado);

        boolean jerarquiaCreada = jerarquia.mkdirs();
        System.out.println("Jerarquía creada: " + jerarquiaCreada);

        // 5. Listar contenido
        System.out.println("\n--- listFiles() ---");
        File[] archivosTmp = directorioTmp.listFiles();
        if (archivosTmp != null) {
            System.out.println("Total elementos en /tmp: " + archivosTmp.length);
            int contador = 0;
            for (File f : archivosTmp) {
                if (contador < 10) {
                    String tipo = f.isDirectory() ? "[DIR]" : "[FILE]";
                    System.out.println("  " + tipo + " " + f.getName());
                    contador++;
                }
            }
            if (archivosTmp.length > 10) {
                System.out.println("  ... y " + (archivosTmp.length - 10) + " más");
            }
        }

        // 6. Filtrar con FilenameFilter
        System.out.println("\n--- FilenameFilter ---");
        FilenameFilter filtroTxt = (dir, nombre) -> nombre.endsWith(".txt");
        String[] txts = directorioTmp.list(filtroTxt);
        if (txts != null && txts.length > 0) {
            System.out.println("Archivos .txt en /tmp:");
            for (String t : txts) {
                System.out.println("  " + t);
            }
        } else {
            System.out.println("No hay archivos .txt en /tmp");
        }

        // 7. FileFilter
        System.out.println("\n--- FileFilter (solo directorios) ---");
        FileFilter soloDirs = File::isDirectory;
        File[] directorios = directorioTmp.listFiles(soloDirs);
        if (directorios != null) {
            System.out.println("Directorios en /tmp:");
            int count = 0;
            for (File d : directorios) {
                if (count < 5) {
                    System.out.println("  " + d.getName());
                    count++;
                }
            }
        }

        // 8. Metadatos
        System.out.println("\n--- Metadatos ---");
        if (archivoEjemplo.exists()) {
            System.out.println("Nombre: " + archivoEjemplo.getName());
            System.out.println("Ruta absoluta: " + archivoEjemplo.getAbsolutePath());
            try {
                System.out.println("Ruta canónica: " + archivoEjemplo.getCanonicalPath());
            } catch (IOException e) {
                System.err.println("Error canónico: " + e.getMessage());
            }
            System.out.println("Directorio padre: " + archivoEjemplo.getParent());
            System.out.println("Tamaño: " + archivoEjemplo.length() + " bytes");
            System.out.println("Última mod: " + new Date(archivoEjemplo.lastModified()));
            System.out.println("¿Se puede leer? " + archivoEjemplo.canRead());
            System.out.println("¿Se puede escribir? " + archivoEjemplo.canWrite());
            System.out.println("¿Se puede ejecutar? " + archivoEjemplo.canExecute());
            System.out.println("¿Es archivo? " + archivoEjemplo.isFile());
            System.out.println("¿Es directorio? " + archivoEjemplo.isDirectory());
            System.out.println("¿Está oculto? " + archivoEjemplo.isHidden());
        }

        // 9. Crear archivo temporal
        System.out.println("\n--- createTempFile() ---");
        try {
            File temp = File.createTempFile("cap51_", ".tmp");
            System.out.println("Temporal creado: " + temp.getAbsolutePath());
            temp.deleteOnExit();

            File tempPersonalizado = File.createTempFile("log_", ".log",
                    new File("/tmp"));
            System.out.println("Temp personalizado: " +
                    tempPersonalizado.getAbsolutePath());
            tempPersonalizado.deleteOnExit();
        } catch (IOException e) {
            System.err.println("Error al crear temp: " + e.getMessage());
        }

        // 10. Separadores
        System.out.println("\n--- Separadores ---");
        System.out.println("separator: " + File.separator);
        System.out.println("separatorChar: " + File.separatorChar);
        System.out.println("pathSeparator: " + File.pathSeparator);
        System.out.println("pathSeparatorChar: " + File.pathSeparatorChar);

        // 11. ListRoots (raíces del sistema)
        System.out.println("\n--- listRoots() ---");
        File[] raices = File.listRoots();
        System.out.println("Raíces del sistema:");
        for (File raiz : raices) {
            long total = raiz.getTotalSpace();
            long libre = raiz.getFreeSpace();
            long usable = raiz.getUsableSpace();
            System.out.println("  " + raiz.getPath()
                    + " | Total: " + (total / (1024 * 1024 * 1024)) + " GB"
                    + " | Libre: " + (libre / (1024 * 1024 * 1024)) + " GB"
                    + " | Usable: " + (usable / (1024 * 1024 * 1024)) + " GB");
        }

        // 12. Comparar rutas
        System.out.println("\n--- Comparación de rutas ---");
        File f1 = new File("/tmp/archivo.txt");
        File f2 = new File("/tmp/./archivo.txt");
        System.out.println("equals: " + f1.equals(f2));
        try {
            System.out.println("Canónicas iguales: "
                    + f1.getCanonicalFile().equals(f2.getCanonicalFile()));
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 13. toPath() (puente a NIO)
        System.out.println("\n--- toPath() ---");
        java.nio.file.Path path = archivoEjemplo.toPath();
        System.out.println("Path NIO: " + path);

        // 14. Limpieza final
        System.out.println("\n--- Limpieza ---");
        System.out.println("Eliminando archivo de ejemplo...");
        if (archivoEjemplo.exists()) {
            boolean eliminado = archivoEjemplo.delete();
            System.out.println("Archivo eliminado: " + eliminado);
        }

        System.out.println("Eliminando jerarquía...");
        eliminarRecursivo(jerarquia);
        System.out.println("Jerarquía eliminada.");

        System.out.println("Eliminando directorio...");
        if (directorioNuevo.exists()) {
            directorioNuevo.delete();
        }

        System.out.println("\n=== FIN CAPÍTULO 51 ===");
    }

    public static void eliminarRecursivo(File f) {
        if (f.isDirectory()) {
            File[] hijos = f.listFiles();
            if (hijos != null) {
                for (File h : hijos) {
                    eliminarRecursivo(h);
                }
            }
        }
        boolean eliminado = f.delete();
        if (!eliminado) {
            System.out.println("No se pudo eliminar: " + f.getAbsolutePath());
        }
    }
}

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

• La clase java.io.File representa rutas de archivos y directorios, no su
  contenido. Permite crear, eliminar, renombrar y consultar metadatos.

• exists() verifica si el recurso existe físicamente. Siempre debe usarse
  antes de operaciones que asuman la existencia del archivo.

• createNewFile() crea un archivo vacío de forma atómica. Devuelve true si
  se creó, false si ya existía. Lanza IOException.

• mkdir() crea un directorio (requiere que el padre exista), mientras que
  mkdirs() crea toda la jerarquía de padres necesaria.

• list() devuelve nombres (String[]) y listFiles() devuelve objetos File[].
  Ambos aceptan filtros: FilenameFilter y FileFilter.

• delete() elimina un archivo o directorio vacío. Para directorios con
  contenido se necesita eliminación recursiva. deleteOnExit() programa la
  eliminación al terminar la JVM.

• Los metadatos disponibles incluyen tamaño (length()), fecha de última
  modificación (lastModified()), permisos (canRead(), canWrite(),
  canExecute()), y tipo (isFile(), isDirectory(), isHidden()).

• File tiene limitaciones que motivaron la creación de NIO: no maneja
  contenido, retorna boolean en lugar de excepciones, y tiene soporte
  limitado para enlaces simbólicos y atributos avanzados.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿File.createNewFile() y File.exists() son operaciones atómicas?
R: createNewFile() es atómica a nivel de sistema operativo. exists() no
   lo es: entre la verificación y una operación posterior el estado puede
   cambiar. Para operaciones atómicas de creación/verificación use
   createNewFile() o Files.createFile() de NIO.

P: ¿Por qué renameTo() a veces falla sin motivo aparente?
R: renameTo() puede fallar si el origen y destino están en sistemas de
   archivos diferentes (monturas distintas). En esos casos, la implementación
   puede requerir copiar y borrar. Files.move() de NIO es más fiable.

P: ¿Cómo elimino un directorio que no está vacío?
R: Debes recorrer recursivamente todos los hijos y eliminar primero los
   archivos y subdirectorios vacíos, como muestra el método
   eliminarRecursivo() del ejemplo. O usar Files.walk() de NIO.

P: ¿Cuál es la diferencia entre getAbsolutePath() y getCanonicalPath()?
R: getAbsolutePath() resuelve la ruta contra el directorio de trabajo
   actual pero no elimina ".", ".." ni enlaces simbólicos.
   getCanonicalPath() resuelve completamente la ruta eliminando estos
   elementos y siguiendo enlaces.

P: ¿File es seguro en entornos multihilo?
R: File en sí es inmutable (sus métodos no modifican el estado interno),
   pero las operaciones sobre el sistema de archivos no son atómicas a
   nivel de aplicación. Para operaciones concurrentes seguras, use
   bloqueos externos o las operaciones atómicas de NIO.

P: ¿Puedo usar File para leer el contenido de un archivo?
R: No. File solo maneja metadatos y operaciones de creación/eliminación.
   Para leer contenido necesitas FileInputStream, FileReader, Scanner o
   las clases de NIO como Files.readAllLines().

P: ¿Qué hace exactamente deleteOnExit()?
R: Registra el archivo para ser eliminado cuando la JVM termine
   normalmente. No funciona si la JVM termina abruptamente (kill -9).
   Es útil para limpiar archivos temporales.

  "SISTEMA DE RESPALDO INCREMENTAL"

  Diseña e implementa un sistema de respaldo (backup) que cumpla los
  siguientes requisitos:

  1. Dado un directorio origen y un directorio destino, el sistema debe
     copiar todos los archivos que hayan sido modificados desde el último
     respaldo.

  2. Mantén un archivo de metadatos (backup_metadata.txt) que registre la
     fecha y hora de cada respaldo, junto con la lista de archivos
     copiados.

  3. Los directorios deben replicarse en el destino manteniendo la misma
     estructura jerárquica.

  4. Soporta exclusiones: un archivo .backupignore en el directorio origen
     puede contener patrones (glob) de archivos a excluir.

  5. Implementa tres modos:
     • FULL: respaldo completo (copia todo aunque no haya cambios).
     • INCREMENTAL: solo archivos modificados desde el último respaldo.
     • DIFFERENTIAL: solo archivos modificados desde el último FULL.

  6. Genera un reporte al finalizar con: cantidad de archivos copiados,
     espacio total transferido, duración del proceso, y errores (si los
     hubo).

  7. Mínimo 3 clases: BackupEngine, MetadataManager, FileFilter.

  VALORACIÓN:
  • 3 estrellas: Funcionalidad básica (full e incremental) con File.
  • 4 estrellas: Modo diferencial + .backupignore con patrones glob.
  • 5 estrellas: Interfaz de línea de comandos robusta + reporte detallado.
