1. LA JERARQUÍA READER Y WRITER
--------------------------------

Java organiza la E/S de caracteres en dos jerarquías de clases abstractas:

  Reader (lectura de caracteres):
  • java.io.Reader (abstracta)
    • BufferedReader
    • CharArrayReader
    • FilterReader
    • InputStreamReader
      • FileReader
    • PipedReader
    • StringReader

  Writer (escritura de caracteres):
  • java.io.Writer (abstracta)
    • BufferedWriter
    • CharArrayWriter
    • FilterWriter
    • OutputStreamWriter
      • FileWriter
    • PipedWriter
    • StringWriter
    • PrintWriter

FileReader y FileWriter son subclases respectivamente de InputStreamReader
y OutputStreamWriter. Estas, a su vez, son "puentes" entre bytes y
caracteres.

2. FILEREADER — LECTURA DE ARCHIVOS DE TEXTO
---------------------------------------------

FileReader permite leer archivos de texto carácter por carácter o en
bloques. Hereda de InputStreamReader, que hereda de Reader.

Constructores principales:
  • FileReader(String fileName)
  • FileReader(File file)
  • FileReader(FileDescriptor fd)

Ejemplo básico de lectura:

  try (FileReader fr = new FileReader("/tmp/lectura.txt")) {
      int caracter;
      while ((caracter = fr.read()) != -1) {
          System.out.print((char) caracter);
      }
  } catch (IOException e) {
      e.printStackTrace();
  }

El método read() devuelve un int que representa el carácter leído (0-65535)
o -1 si se alcanzó el final del archivo. Se debe hacer casting a char.

3. LECTURA EN BLOQUE CON ARREGLO DE CARACTERES
-----------------------------------------------

Para mejorar el rendimiento, es mejor leer en bloques:

  try (FileReader fr = new FileReader("/tmp/lectura.txt")) {
      char[] buffer = new char[1024];
      int caracteresLeidos;
      while ((caracteresLeidos = fr.read(buffer)) != -1) {
          String texto = new String(buffer, 0, caracteresLeidos);
          System.out.print(texto);
      }
  } catch (IOException e) {
      e.printStackTrace();
  }

read(char[] cbuf) devuelve el número de caracteres leídos, o -1 si es fin
del archivo. read(char[] cbuf, int off, int len) permite leer hasta len
caracteres a partir de la posición off del arreglo.

4. SKIP() — SALTAR CARACTERES
------------------------------

Reader proporciona el método skip(long n) para saltar n caracteres:

  try (FileReader fr = new FileReader("/tmp/datos.txt")) {
      fr.skip(10); // Saltar los primeros 10 caracteres
      int c;
      while ((c = fr.read()) != -1) {
          System.out.print((char) c);
      }
  }

5. FILEWRITER — ESCRITURA DE ARCHIVOS DE TEXTO
-----------------------------------------------

FileWriter permite escribir caracteres en archivos de texto.

Constructores principales:
  • FileWriter(String fileName)
  • FileWriter(String fileName, boolean append)
  • FileWriter(File file)
  • FileWriter(File file, boolean append)
  • FileWriter(FileDescriptor fd)

El parámetro append determina si se añade al final (true) o se sobreescribe
el archivo (false, valor por defecto).

Métodos de escritura:
  • write(int c): escribe un solo carácter.
  • write(char[] cbuf): escribe un arreglo completo.
  • write(char[] cbuf, int off, int len): escribe una porción.
  • write(String str): escribe una cadena.
  • write(String str, int off, int len): escribe una porción de cadena.
  • flush(): vacía el buffer al archivo.
  • append(CharSequence csq): implementa Appendable.

Ejemplo básico de escritura:

  try (FileWriter fw = new FileWriter("/tmp/salida.txt")) {
      fw.write("Hola, mundo!\n");
      fw.write("Segunda línea.\n");
      fw.write("Tercera línea.\n");
      fw.flush(); // Forzar escritura
  } catch (IOException e) {
      e.printStackTrace();
  }

6. MODO APPEND
---------------

Para añadir contenido sin borrar lo existente:

  try (FileWriter fw = new FileWriter("/tmp/log.txt", true)) {
      fw.write("Nueva entrada de log: " + new java.util.Date() + "\n");
  }

Cada ejecución añadirá una línea sin borrar las anteriores.

7. FLUSH() Y CLOSE() — LA IMPORTANCIA DE CERRAR
-------------------------------------------------

  • flush(): fuerza la escritura de cualquier dato en buffer al destino.
    Garantiza que los datos lleguen al archivo físico.

  • close(): cierra el recurso liberando los recursos del sistema operativo.
    Antes de cerrar, hace flush() automáticamente.

  • Siempre usar try-with-resources (Capítulo 57) para garantizar el
    cierre automático, o un bloque finally si usas try-catch tradicional.

  try (FileWriter fw = new FileWriter("/tmp/archivo.txt")) {
      fw.write("Contenido importante...");
      // No es necesario llamar flush() ni close() explicitamente
  } // try-with-resources llama close() automáticamente

8. LIMITACIÓN DE FILEWRITER: CODIFICACIÓN
------------------------------------------

FileReader y FileWriter usan la codificación de caracteres predeterminada
de la plataforma (Charset.defaultCharset()). En sistemas Linux suele ser
UTF-8, en Windows puede ser CP1252. Esto puede causar problemas de
portabilidad.

  // ¿Qué codificación se está usando?
  System.out.println(Charset.defaultCharset());

Para especificar una codificación explícita, debemos usar
InputStreamReader con FileInputStream, y OutputStreamWriter con
FileOutputStream:

  // UTF-8 explícito para lectura
  try (InputStreamReader isr =
          new InputStreamReader(new FileInputStream("/tmp/datos.txt"), "UTF-8")) {
      int c;
      while ((c = isr.read()) != -1) {
          System.out.print((char) c);
      }
  }

  // UTF-8 explícito para escritura
  try (OutputStreamWriter osw =
          new OutputStreamWriter(new FileOutputStream("/tmp/salida.txt"), "UTF-8")) {
      osw.write("Texto con codificación UTF-8 explícita");
  }

A partir de Java 10, usar Charset:

  try (InputStreamReader isr =
          new InputStreamReader(new FileInputStream("/tmp/datos.txt"),
                  java.nio.charset.StandardCharsets.UTF_8)) { ... }

9. COPY - COPIA DE ARCHIVOS DE TEXTO
-------------------------------------

Un caso de uso clásico: copiar un archivo de texto.

  public static void copiarArchivo(String origen, String destino)
          throws IOException {
      try (FileReader fr = new FileReader(origen);
           FileWriter fw = new FileWriter(destino)) {
          char[] buffer = new char[8192];
          int caracteres;
          while ((caracteres = fr.read(buffer)) != -1) {
              fw.write(buffer, 0, caracteres);
          }
      } // Ambos recursos se cierran automáticamente
  }

10. MANEJO DE EXCEPCIONES
--------------------------

Las operaciones de E/S pueden lanzar:
  • FileNotFoundException: el archivo no existe (solo lectura).
  • IOException: error general de E/S.
  • SecurityException: permisos insuficientes.

  try {
      FileReader fr = new FileReader("/tmp/no_existe.txt");
      // ...
  } catch (FileNotFoundException e) {
      System.err.println("Archivo no encontrado: " + e.getMessage());
  } catch (IOException e) {
      System.err.println("Error de E/S: " + e.getMessage());
  }

11. USO CONTRYE — CÓDIGO ANTIGUO (ANTES DE JAVA 7)
----------------------------------------------------

Antes de try-with-resources (Java 7), se usaba finally para cerrar:

  FileReader fr = null;
  FileWriter fw = null;
  try {
      fr = new FileReader("/tmp/origen.txt");
      fw = new FileWriter("/tmp/destino.txt");
      int c;
      while ((c = fr.read()) != -1) {
          fw.write(c);
      }
  } catch (IOException e) {
      e.printStackTrace();
  } finally {
      if (fr != null) {
          try { fr.close(); } catch (IOException e) { e.printStackTrace(); }
      }
      if (fw != null) {
          try { fw.close(); } catch (IOException e) { e.printStackTrace(); }
      }
  }

Este patrón es verboso y propenso a errores. Hoy en día siempre se
recomienda try-with-resources.

12. EL MÉTODO TRANSFERTO() (JAVA 9+)
--------------------------------------

A partir de Java 9, Reader tiene el método transferTo(Writer out) que
transfiere todo el contenido del Reader al Writer:

  try (FileReader fr = new FileReader("/tmp/origen.txt");
       FileWriter fw = new FileWriter("/tmp/destino.txt")) {
      fr.transferTo(fw);
  }

Es la forma más concisa de copiar contenido textual entre streams.

13. PRINTWRITER VS FILEWRITER
------------------------------

PrintWriter extiende Writer y ofrece métodos convenientes como
print(), println(), printf(). Puede envolver un FileWriter:

  try (PrintWriter pw = new PrintWriter(new FileWriter("/tmp/salida.txt"))) {
      pw.println("Línea 1");
      pw.println("Línea 2");
      pw.printf("Número: %d\n", 42);
  }

PrintWriter no lanza IOException (los chequea internamente con
checkError()). Es útil para generar texto formateado.

================================================================================
  CÓDIGO DE EJEMPLO COMPLETO
================================================================================

import java.io.FileReader;
import java.io.FileWriter;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.io.IOException;
import java.io.FileNotFoundException;
import java.io.PrintWriter;
import java.nio.charset.StandardCharsets;

public class Capitulo52FileReaderFileWriter {

    public static void main(String[] args) {
        System.out.println("=== CAPÍTULO 52: FILE READER & FILE WRITER ===\n");
        String archivo = "/tmp/cap52_ejemplo.txt";

        // 1. Escritura básica
        System.out.println("--- Escritura básica con FileWriter ---");
        try (FileWriter fw = new FileWriter(archivo)) {
            fw.write("Hola desde FileWriter!\n");
            fw.write("Esto es una prueba.\n");
            fw.write("Caracteres especiales: ñoño áéíóú.\n");
            fw.write("Número: " + 42 + "\n");
            fw.flush();
            System.out.println("Archivo escrito exitosamente.");
        } catch (IOException e) {
            System.err.println("Error escritura: " + e.getMessage());
        }

        // 2. Lectura básica carácter por carácter
        System.out.println("\n--- Lectura básica con FileReader ---");
        try (FileReader fr = new FileReader(archivo)) {
            int c;
            System.out.print("Contenido: ");
            while ((c = fr.read()) != -1) {
                System.out.print((char) c);
            }
            System.out.println(); // Salto de línea extra
        } catch (FileNotFoundException e) {
            System.err.println("Archivo no encontrado: " + e.getMessage());
        } catch (IOException e) {
            System.err.println("Error lectura: " + e.getMessage());
        }

        // 3. Lectura con buffer de caracteres
        System.out.println("--- Lectura con arreglo de caracteres ---");
        try (FileReader fr = new FileReader(archivo)) {
            char[] buffer = new char[512];
            int leidos;
            StringBuilder sb = new StringBuilder();
            while ((leidos = fr.read(buffer)) != -1) {
                sb.append(buffer, 0, leidos);
            }
            System.out.println("Contenido completo:\n" + sb.toString());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 4. Modo append
        System.out.println("--- Modo append ---");
        try (FileWriter fw = new FileWriter(archivo, true)) {
            fw.write("Línea añadida al final.\n");
            fw.write("Otra línea añadida.\n");
            System.out.println("Contenido añadido.");
        } catch (IOException e) {
            System.err.println("Error append: " + e.getMessage());
        }

        // 5. Verificar append
        System.out.println("\n--- Contenido después de append ---");
        try (FileReader fr = new FileReader(archivo)) {
            fr.transferTo(System.out); // Java 9+
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        System.out.println();

        // 6. Lectura con skip()
        System.out.println("--- skip() ---");
        try (FileReader fr = new FileReader(archivo)) {
            fr.skip(5);
            System.out.println("Después de saltar 5 caracteres:");
            int c;
            while ((c = fr.read()) != -1) {
                System.out.print((char) c);
            }
            System.out.println();
        } catch (IOException e) {
            System.err.println("Error skip: " + e.getMessage());
        }

        // 7. Escritura con arreglo de caracteres
        System.out.println("--- write(char[]) ---");
        try (FileWriter fw = new FileWriter("/tmp/array_demo.txt")) {
            char[] chars = "Demo con arreglo de caracteres.\n".toCharArray();
            fw.write(chars);
            fw.write("Otra línea.\n".toCharArray(), 0, 6); // "Otra l"
            System.out.println("Escrito con arreglos.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 8. Copia de archivos
        System.out.println("\n--- Copia de archivos ---");
        String destino = "/tmp/cap52_copia.txt";
        try {
            copiarArchivo(archivo, destino);
            System.out.println("Archivo copiado a: " + destino);
        } catch (IOException e) {
            System.err.println("Error copia: " + e.getMessage());
        }

        // 9. Especificar codificación con InputStreamReader/OutputStreamWriter
        System.out.println("\n--- Codificación explícita UTF-8 ---");
        try (OutputStreamWriter osw = new OutputStreamWriter(
                new FileOutputStream("/tmp/utf8_demo.txt"),
                StandardCharsets.UTF_8)) {
            osw.write("Texto con codificación UTF-8 explícita.\n");
            osw.write("Caracteres: ñoño áéíóú ü ñ Ñ © ® ™\n");
            System.out.println("Archivo UTF-8 escrito.");
        } catch (IOException e) {
            System.err.println("Error UTF-8: " + e.getMessage());
        }

        // Leer con codificación explícita
        try (InputStreamReader isr = new InputStreamReader(
                new FileInputStream("/tmp/utf8_demo.txt"),
                StandardCharsets.UTF_8)) {
            System.out.println("Lectura con UTF-8:");
            int c;
            while ((c = isr.read()) != -1) {
                System.out.print((char) c);
            }
            System.out.println();
        } catch (IOException e) {
            System.err.println("Error lectura UTF-8: " + e.getMessage());
        }

        // 10. TransferTo (Java 9+)
        System.out.println("--- transferTo() ---");
        try (FileReader fr = new FileReader(archivo);
             FileWriter fw = new FileWriter("/tmp/transfer_demo.txt")) {
            fr.transferTo(fw);
            System.out.println("transferTo completado.");
        } catch (IOException e) {
            System.err.println("Error transferTo: " + e.getMessage());
        }

        // 11. PrintWriter para formato
        System.out.println("\n--- PrintWriter ---");
        try (PrintWriter pw = new PrintWriter(
                new FileWriter("/tmp/print_demo.txt"))) {
            pw.println("PrintWriter demo:");
            pw.printf("Entero: %d | Flotante: %.2f | Cadena: %s%n",
                      42, 3.14159, "Hola");
            System.out.println("PrintWriter completado.");
            System.out.println("Estado de error: " + pw.checkError());
        } catch (IOException e) {
            System.err.println("Error PrintWriter: " + e.getMessage());
        }

        // 12. Verificar archivos creados
        System.out.println("\n--- Archivos creados en /tmp ---");
        String[] archivosCreados = {
            "cap52_ejemplo.txt",
            "cap52_copia.txt",
            "array_demo.txt",
            "utf8_demo.txt",
            "transfer_demo.txt",
            "print_demo.txt"
        };
        for (String a : archivosCreados) {
            java.io.File f = new java.io.File("/tmp/" + a);
            System.out.println("  " + a + ": " + (f.exists() ? "EXISTE" : "NO EXISTE")
                    + " (" + f.length() + " bytes)");
        }

        // 13. Charset default
        System.out.println("\n--- Charset predeterminado ---");
        System.out.println("Default: " + java.nio.charset.Charset.defaultCharset());

        System.out.println("\n=== FIN CAPÍTULO 52 ===");
    }

    public static void copiarArchivo(String origen, String destino)
            throws IOException {
        try (FileReader fr = new FileReader(origen);
             FileWriter fw = new FileWriter(destino)) {
            char[] buffer = new char[8192];
            int caracteres;
            while ((caracteres = fr.read(buffer)) != -1) {
                fw.write(buffer, 0, caracteres);
            }
        }
    }
}

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

• FileReader y FileWriter son las clases más simples para leer y escribir
  archivos de texto en Java. Heredan de InputStreamReader/OutputStreamWriter.

• FileReader lee caracteres con read() (devuelve int, -1 para EOF) y
  read(char[], off, len) para lectura en bloque.

• FileWriter escribe caracteres con write(int), write(char[]),
  write(String), y write(String, off, len).

• El modo append (segundo parámetro = true) permite añadir al final sin
  sobrescribir.

• Siempre cerrar los recursos. try-with-resources es la forma recomendada.

• flush() fuerza la escritura inmediata de los datos en buffer.

• Limitación importante: FileReader y FileWriter usan la codificación
  predeterminada de la plataforma. Para controlar la codificación, usar
  InputStreamReader con FileInputStream y un Charset explícito.

• Reader.transferTo(Writer) desde Java 9 permite copiar contenido de
  forma concisa entre streams de caracteres.

• PrintWriter ofrece métodos convenientes (println, printf) y puede
  envolver FileWriter para salida formateada.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿FileReader y FileWriter son eficientes para archivos grandes?
R: No. Para archivos grandes, el acceso carácter por carácter es muy lento.
   Se recomienda usar BufferedReader/BufferedWriter para añadir buffering,
   o leer en bloques grandes (8 KB o más).

P: ¿Qué codificación usa FileReader por defecto?
R: Usa la codificación predeterminada de la JVM, obtenida con
   Charset.defaultCharset(). Depende del sistema operativo y la
   configuración regional. Para evitar problemas de portabilidad,
   especifica la codificación explícitamente.

P: Si uso try-with-resources, ¿necesito llamar a flush()?
R: No es necesario. close() (llamado automáticamente por try-with-resources)
   invoca flush() antes de cerrar. Sin embargo, si necesitas garantizar
   que los datos se escriban inmediatamente (ej: logs en tiempo real),
   puedes llamar a flush() explícitamente.

P: ¿Cuál es la diferencia entre FileWriter y OutputStreamWriter?
R: FileWriter es una subclase de OutputStreamWriter que crea automáticamente
   un FileOutputStream. OutputStreamWriter requiere que pases el
   OutputStream explícitamente, lo que permite especificar la codificación.

P: ¿Puedo leer un archivo binario con FileReader?
R: No, FileReader es para archivos de texto (caracteres). Para archivos
   binarios usa FileInputStream. Usar FileReader con binarios corromperá
   los datos debido a la conversión de codificación.

P: ¿Qué ocurre si escribo en un archivo que no existe?
R: FileWriter crea el archivo si no existe (si el directorio padre sí
   existe). Si el directorio padre no existe, lanza FileNotFoundException.

  "PROCESADOR DE ARCHIVOS DE CONFIGURACIÓN"

  Diseña un sistema que procese archivos de configuración con el formato:

  # Comentarios
  clave = valor
  usuario = "juan perez"
  puerto = 8080
  depuracion = true
  lista = manzana, pera, uva
  [seccion1]
  clave_hija = valor_hija

  Requisitos:

  1. Parseador básico: Ignora líneas en blanco y comentarios (#).
     Separa clave=valor y trimea ambos lados.

  2. Soporte para strings con comillas: "valor con espacios" debe
     preservar el contenido interno.

  3. Inferencia de tipos: Detecta automáticamente enteros, booleanos
     (true/false), doubles, y trata el resto como String.

  4. Secciones: Soporta [seccion] para agrupar claves. La clave
     compuesta sería "seccion.clave".

  5. Exportación: Guarda la configuración en un nuevo archivo con
     formato limpio y orden alfabético de claves.

  6. Validación: Detecta errores de sintaxis y reporta línea y tipo
     de error (clave duplicada, formato inválido, sección sin cerrar).

  7. Mínimo 3 clases: ConfigParser, ConfigSection, ConfigValidator.

  VALORACIÓN:
  • 3 estrellas: Parseo básico + tipos inferidos + exportación.
  • 4 estrellas: Secciones + comillas + detección de errores.
  • 5 estrellas: Archivos incluidos (import "otro.conf") + herencia
    de secciones.

  CAPÍTULO 53: BUFFERED READER Y BUFFERED WRITER
================================================================================

En el Capítulo 53 exploraremos BufferedReader y BufferedWriter, las
clases que añaden una capa de buffering a la E/S de caracteres.
Aprenderemos cómo el buffer interno mejora drásticamente el rendimiento
