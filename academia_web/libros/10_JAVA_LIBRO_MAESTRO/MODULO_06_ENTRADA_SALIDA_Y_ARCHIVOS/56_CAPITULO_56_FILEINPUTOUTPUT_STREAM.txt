1. BYTE STREAMS VS CHARACTER STREAMS
--------------------------------------

  Byte streams (InputStream/OutputStream):
  • Trabajan con bytes (8 bits).
  • No realizan conversión de codificación.
  • Adecuados para datos binarios: imágenes, audio, video, ZIP, etc.
  • Clases principales: FileInputStream, FileOutputStream.

  Character streams (Reader/Writer):
  • Trabajan con caracteres Unicode (16 bits o más).
  • Realizan conversión usando Charset.
  • Adecuados para archivos de texto.
  • Clases principales: FileReader, FileWriter.

  Elección:
  • Si son datos legibles por humanos → character streams.
  • Si son datos binarios o se desconoce el formato → byte streams.

2. FILEINPUTSTREAM — LECTURA DE BYTES
---------------------------------------

FileInputStream permite leer bytes de un archivo.

  Constructores:
  • FileInputStream(String name)
  • FileInputStream(File file)
  • FileInputStream(FileDescriptor fd)

  Métodos principales:
  • read(): Lee un byte (0-255) o -1 si es EOF.
  • read(byte[] b): Lee bytes en un arreglo. Devuelve número leído.
  • read(byte[] b, int off, int len): Lee hasta len bytes.
  • available(): Estima cuántos bytes pueden leerse sin bloqueo.
  • skip(long n): Salta y descarta n bytes.
  • close(): Cierra el stream.

  Ejemplo básico:

  try (FileInputStream fis = new FileInputStream("/tmp/archivo.bin")) {
      int byteLeido;
      while ((byteLeido = fis.read()) != -1) {
          System.out.printf("%02X ", byteLeido);
      }
  }

3. FILEOUTPUTSTREAM — ESCRITURA DE BYTES
-----------------------------------------

FileOutputStream permite escribir bytes en un archivo.

  Constructores:
  • FileOutputStream(String name)
  • FileOutputStream(String name, boolean append)
  • FileOutputStream(File file)
  • FileOutputStream(File file, boolean append)
  • FileOutputStream(FileDescriptor fd)

  Métodos principales:
  • write(int b): Escribe un byte (los 8 bits menos significativos).
  • write(byte[] b): Escribe un arreglo completo.
  • write(byte[] b, int off, int len): Escribe una porción.
  • flush(): Vacía el buffer (si hay capas superiores).
  • close(): Cierra el stream.

  Ejemplo básico:

  try (FileOutputStream fos = new FileOutputStream("/tmp/salida.bin")) {
      fos.write(0x48); // 'H'
      fos.write(0x65); // 'e'
      fos.write(0x6C); // 'l'
      byte[] datos = { 0x6C, 0x6F, 0x21 }; // "lo!"
      fos.write(datos);
  }

4. LECTURA/ESCRITURA EN BLOQUE
--------------------------------

Leer byte a byte es extremadamente lento. Siempre usar arreglos:

  try (FileInputStream fis = new FileInputStream("/tmp/entrada.bin");
       FileOutputStream fos = new FileOutputStream("/tmp/salida.bin")) {
      byte[] buffer = new byte[8192];
      int bytesLeidos;
      while ((bytesLeidos = fis.read(buffer)) != -1) {
          fos.write(buffer, 0, bytesLeidos);
      }
  }

Tamaño de buffer recomendado:
  • 4096 (4 KB): mínimo recomendado.
  • 8192 (8 KB): buen balance.
  • 65536 (64 KB): para archivos muy grandes.
  • 1048576 (1 MB): para máxima velocidad en discos modernos.

5. COPY — COPIA DE ARCHIVOS BINARIOS
--------------------------------------

  public static void copiarArchivo(String origen, String destino)
          throws IOException {
      try (FileInputStream fis = new FileInputStream(origen);
           FileOutputStream fos = new FileOutputStream(destino)) {
          byte[] buffer = new byte[8192];
          int bytesLeidos;
          while ((bytesLeidos = fis.read(buffer)) != -1) {
              fos.write(buffer, 0, bytesLeidos);
          }
      }
  }

  // Java 9+: transferTo() copia directamente
  public static void copiarConTransferTo(String origen, String destino)
          throws IOException {
      try (FileInputStream fis = new FileInputStream(origen);
           FileOutputStream fos = new FileOutputStream(destino)) {
          fis.transferTo(fos); // Simple y eficiente
      }
  }

6. AVAILABLE() Y SKIP()
-------------------------

  • available(): Devuelve una estimación de bytes que pueden leerse
    sin bloqueo. Útil para buffers dinámicos:

  try (FileInputStream fis = new FileInputStream("/tmp/datos.bin")) {
      int disponibles = fis.available();
      byte[] buffer = new byte[disponibles];
      fis.read(buffer);
      System.out.println("Leídos " + buffer.length + " bytes de una vez.");
  }

  • skip(long n): Salta n bytes:

  try (FileInputStream fis = new FileInputStream("/tmp/header.bin")) {
      fis.skip(100); // Saltar header de 100 bytes
      int contenido = fis.read();
  }

7. BUFFEREDINPUTSTREAM Y BUFFEREDOUTPUTSTREAM
-----------------------------------------------

Añaden buffering a los byte streams, similar a BufferedReader/Writer:

  try (BufferedInputStream bis = new BufferedInputStream(
           new FileInputStream("/tmp/grande.bin"));
       BufferedOutputStream bos = new BufferedOutputStream(
           new FileOutputStream("/tmp/copia.bin"))) {
      byte[] buffer = new byte[8192];
      int leidos;
      while ((leidos = bis.read(buffer)) != -1) {
          bos.write(buffer, 0, leidos);
      }
      bos.flush(); // Forzar escritura de buffer residual
  }

  // Con buffer de tamaño personalizado
  BufferedInputStream bis = new BufferedInputStream(
      new FileInputStream("/tmp/grande.bin"), 65536);

8. DATAINPUTSTREAM Y DATAOUTPUTSTREAM
---------------------------------------

Permiten leer/escribir tipos primitivos de forma portable:

  try (DataOutputStream dos = new DataOutputStream(
           new FileOutputStream("/tmp/primitivos.dat"))) {
      dos.writeInt(42);
      dos.writeDouble(3.14159);
      dos.writeUTF("Hola");
      dos.writeBoolean(true);
      dos.writeLong(123456789L);
  }

  try (DataInputStream dis = new DataInputStream(
           new FileInputStream("/tmp/primitivos.dat"))) {
      int i = dis.readInt();
      double d = dis.readDouble();
      String s = dis.readUTF();
      boolean b = dis.readBoolean();
      long l = dis.readLong();
      System.out.println(i + ", " + d + ", " + s + ", " + b + ", " + l);
  }

DataOutputStream escribe en big-endian (network byte order), lo que
garantiza portabilidad entre plataformas.

9. SEQUENCEINPUTSTREAM — COMBINAR MÚLTIPLES STREAMS
-----------------------------------------------------

SequenceInputStream concatena varios InputStreams como si fueran uno:

  try (FileInputStream f1 = new FileInputStream("/tmp/parte1.bin");
       FileInputStream f2 = new FileInputStream("/tmp/parte2.bin");
       SequenceInputStream seq = new SequenceInputStream(f1, f2)) {
      byte[] buffer = new byte[8192];
      int leidos;
      while ((leidos = seq.read(buffer)) != -1) {
          // Procesar...
      }
  }

  // Con enumeración de streams
  Vector<InputStream> streams = new Vector<>();
  streams.add(new FileInputStream("/tmp/a.bin"));
  streams.add(new FileInputStream("/tmp/b.bin"));
  SequenceInputStream seq = new SequenceInputStream(streams.elements());

10. BYTEARRAYINPUT/OUTPUT STREAM
----------------------------------

Para trabajar con bytes en memoria:

  // Escribir en un arreglo
  ByteArrayOutputStream baos = new ByteArrayOutputStream();
  baos.write(0x41);
  baos.write("Hola".getBytes());
  byte[] resultado = baos.toByteArray();

  // Leer desde un arreglo
  ByteArrayInputStream bais = new ByteArrayInputStream(resultado);
  int b;
  while ((b = bais.read()) != -1) {
      System.out.print((char) b);
  }

11. FILEINPUTSTREAM CON FILES — OBTENER TAMAÑO
-----------------------------------------------

File.length() da el tamaño exacto del archivo, útil para buffers:

  File archivo = new File("/tmp/datos.bin");
  long tamano = archivo.length();
  if (tamano < Integer.MAX_VALUE) {
      byte[] todo = new byte[(int) tamano];
      try (FileInputStream fis = new FileInputStream(archivo)) {
          fis.read(todo); // Leer todo el archivo de una vez
      }
  }

Cuidado: leer archivos enteros en memoria puede causar OutOfMemoryError.

12. MANEJO DE EXCEPCIONES
--------------------------

  try (FileInputStream fis = new FileInputStream("/tmp/no_existe.bin")) {
      // ...
  } catch (FileNotFoundException e) {
      System.err.println("Archivo no encontrado: " + e.getMessage());
  } catch (SecurityException e) {
      System.err.println("Permiso denegado: " + e.getMessage());
  } catch (IOException e) {
      System.err.println("Error de E/S: " + e.getMessage());
  }

13. APLICACIONES COMUNES
--------------------------

  • Copia de archivos: Backup, sincronización.
  • Procesamiento de imágenes: Leer cabeceras, metadatos.
  • Carga de recursos: Binarios empaquetados en JAR.
  • Criptografía: Leer archivos para encriptar/desencriptar.

import java.io.*;
import java.util.*;

public class Capitulo56FileInputOutputStream {

    public static void main(String[] args) {
        System.out.println("=== CAPÍTULO 56: FILEINPUT/OUTPUT STREAM ===\n");

        String archivoBin = "/tmp/cap56_ejemplo.bin";
        String copiaBin = "/tmp/cap56_copia.bin";

        // 1. Escribir bytes básicos
        System.out.println("--- Escritura básica de bytes ---");
        try (FileOutputStream fos = new FileOutputStream(archivoBin)) {
            fos.write(0x48); // H
            fos.write(0x65); // e
            fos.write(0x6C); // l
            fos.write(0x6C); // l
            fos.write(0x6F); // o
            fos.write(0x20); // espacio
            fos.write(0x42); // B
            fos.write(0x69); // i
            fos.write(0x6E); // n
            fos.write(0x21); // !
            fos.write('\n');
            System.out.println("Bytes escritos: Hello Bin!");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 2. Leer byte a byte
        System.out.println("\n--- Lectura byte a byte ---");
        try (FileInputStream fis = new FileInputStream(archivoBin)) {
            int b;
            System.out.print("Contenido: ");
            while ((b = fis.read()) != -1) {
                System.out.print((char) b);
            }
            System.out.println();
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 3. Escribir arreglo de bytes
        System.out.println("\n--- Escritura con arreglo ---");
        byte[] datosComplejos = {
            0x48, 0x6F, 0x6C, 0x61, 0x20,
            0x4D, 0x75, 0x6E, 0x64, 0x6F, 0x21, 0x0A,
            0x45, 0x73, 0x74, 0x6F, 0x20, 0x65, 0x73,
            0x20, 0x62, 0x69, 0x6E, 0x61, 0x72, 0x69, 0x6F, 0x0A
        };
        try (FileOutputStream fos = new FileOutputStream(
                "/tmp/cap56_array.bin")) {
            fos.write(datosComplejos);
            System.out.println("Arreglo de bytes escrito.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 4. Leer con buffer de bytes
        System.out.println("\n--- Lectura con buffer ---");
        try (FileInputStream fis = new FileInputStream("/tmp/cap56_array.bin")) {
            byte[] buffer = new byte[4096];
            int leidos;
            System.out.print("Contenido: ");
            while ((leidos = fis.read(buffer)) != -1) {
                System.out.print(new String(buffer, 0, leidos));
            }
            System.out.println();
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 5. Copia de archivos
        System.out.println("\n--- Copia de archivos ---");
        try {
            copiarArchivo(archivoBin, copiaBin);
            System.out.println("Archivo copiado de " + archivoBin
                    + " a " + copiaBin);
            // Verificar
            File original = new File(archivoBin);
            File copiado = new File(copiaBin);
            System.out.println("Tamaño original: " + original.length()
                    + " bytes");
            System.out.println("Tamaño copia: " + copiado.length() + " bytes");
        } catch (IOException e) {
            System.err.println("Error copia: " + e.getMessage());
        }

        // 6. transferTo (Java 9+)
        System.out.println("\n--- transferTo() ---");
        try (FileInputStream fis = new FileInputStream(archivoBin);
             FileOutputStream fos = new FileOutputStream(
                     "/tmp/cap56_transfer.bin")) {
            fis.transferTo(fos);
            System.out.println("Copia con transferTo completada.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 7. available() y skip()
        System.out.println("\n--- available() y skip() ---");
        try (FileInputStream fis = new FileInputStream(archivoBin)) {
            System.out.println("Bytes disponibles: " + fis.available());
            fis.skip(3); // Saltar "Hel"
            System.out.println("Después de skip(3): " + fis.available());
            int r = fis.read();
            System.out.println("Siguiente byte: " + (char) r + " (0x"
                    + Integer.toHexString(r) + ")");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 8. DataInputStream/DataOutputStream
        System.out.println("\n--- DataInputStream/DataOutputStream ---");
        try (DataOutputStream dos = new DataOutputStream(
                new FileOutputStream("/tmp/cap56_datos.dat"))) {
            dos.writeInt(1024);
            dos.writeDouble(3.14159265358979);
            dos.writeUTF("Java Master Book");
            dos.writeBoolean(true);
            dos.writeLong(System.currentTimeMillis());
            System.out.println("Datos primitivos escritos.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        try (DataInputStream dis = new DataInputStream(
                new FileInputStream("/tmp/cap56_datos.dat"))) {
            System.out.println("Int: " + dis.readInt());
            System.out.println("Double: " + dis.readDouble());
            System.out.println("UTF: " + dis.readUTF());
            System.out.println("Boolean: " + dis.readBoolean());
            System.out.println("Long: " + dis.readLong());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 9. BufferedInputStream/OutputStream
        System.out.println("\n--- Buffered Streams ---");
        try (BufferedInputStream bis = new BufferedInputStream(
                new FileInputStream(archivoBin), 8192);
             BufferedOutputStream bos = new BufferedOutputStream(
                new FileOutputStream("/tmp/cap56_buffered.bin"), 8192)) {
            byte[] buffer = new byte[4096];
            int leidos;
            while ((leidos = bis.read(buffer)) != -1) {
                bos.write(buffer, 0, leidos);
            }
            bos.flush();
            System.out.println("Copia con buffered streams completada.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 10. ByteArrayInputStream/OutputStream
        System.out.println("\n--- ByteArray Streams ---");
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        try (DataOutputStream dos = new DataOutputStream(baos)) {
            dos.writeShort(42);
            dos.writeFloat(2.718f);
            dos.writeUTF("Demo en memoria");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        byte[] datosMemoria = baos.toByteArray();
        System.out.println("Bytes en memoria: " + datosMemoria.length);

        // Leer desde memoria
        try (DataInputStream dis = new DataInputStream(
                new ByteArrayInputStream(datosMemoria))) {
            System.out.println("Short: " + dis.readShort());
            System.out.println("Float: " + dis.readFloat());
            System.out.println("UTF: " + dis.readUTF());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 11. Leer archivo entero en memoria
        System.out.println("\n--- Leer archivo entero ---");
        try {
            byte[] contenido = leerArchivoEntero(archivoBin);
            System.out.println("Archivo leído completo: "
                    + contenido.length + " bytes");
            System.out.println("Como texto: "
                    + new String(contenido, java.nio.charset.StandardCharsets.UTF_8).trim());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 12. Crear archivo binario con patrones
        System.out.println("\n--- Patrones binarios ---");
        try (FileOutputStream fos = new FileOutputStream("/tmp/cap56_patron.bin")) {
            byte[] patron = new byte[256];
            for (int i = 0; i < 256; i++) {
                patron[i] = (byte) i;
            }
            fos.write(patron);
            System.out.println("Patrón de 0-255 escrito.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        try (FileInputStream fis = new FileInputStream("/tmp/cap56_patron.bin")) {
            System.out.print("Primeros 16 bytes: ");
            for (int i = 0; i < 16; i++) {
                int b = fis.read();
                System.out.printf("%02X ", b);
            }
            System.out.println();
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 13. Modo append
        System.out.println("\n--- Modo append ---");
        try (FileOutputStream fos = new FileOutputStream("/tmp/cap56_append.bin")) {
            fos.write("Primera línea\n".getBytes());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        try (FileOutputStream fos = new FileOutputStream(
                "/tmp/cap56_append.bin", true)) {
            fos.write("Segunda línea añadida\n".getBytes());
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
        try (FileInputStream fis = new FileInputStream("/tmp/cap56_append.bin")) {
            System.out.print("Contenido con append: ");
            byte[] buf = new byte[1024];
            int l = fis.read(buf);
            System.out.print(new String(buf, 0, l));
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // 14. Comparar rendimiento
        System.out.println("\n--- Comparación de rendimiento ---");
        String tempBig = "/tmp/cap56_grande.bin";
        try (FileOutputStream fos = new FileOutputStream(tempBig)) {
            byte[] bloque = new byte[8192];
            Arrays.fill(bloque, (byte) 0xAB);
            for (int i = 0; i < 128; i++) { // 1 MB
                fos.write(bloque);
            }
            System.out.println("Archivo de 1 MB creado.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }

        // Medir sin buffer byte a byte
        long t1 = System.nanoTime();
        try (FileInputStream fis = new FileInputStream(tempBig)) {
            while (fis.read() != -1) {}
        } catch (IOException e) {}
        long t1f = System.nanoTime();
        double ms1 = (t1f - t1) / 1_000_000.0;

        // Medir con buffer
        long t2 = System.nanoTime();
        try (BufferedInputStream bis = new BufferedInputStream(
                new FileInputStream(tempBig))) {
            while (bis.read() != -1) {}
        } catch (IOException e) {}
        long t2f = System.nanoTime();
        double ms2 = (t2f - t2) / 1_000_000.0;

        // Medir con buffer de bloque
        long t3 = System.nanoTime();
        try (BufferedInputStream bis = new BufferedInputStream(
                new FileInputStream(tempBig))) {
            byte[] buf = new byte[8192];
            while (bis.read(buf) != -1) {}
        } catch (IOException e) {}
        long t3f = System.nanoTime();
        double ms3 = (t3f - t3) / 1_000_000.0;

        System.out.printf("FileInputStream byte a byte:   %.2f ms%n", ms1);
        System.out.printf("BufferedInputStream byte a byte: %.2f ms%n", ms2);
        System.out.printf("BufferedInputStream bloque 8KB:  %.2f ms%n", ms3);

        // Limpieza
        System.out.println("\n--- Limpieza ---");
        String[] limpiar = {
            archivoBin, copiaBin, "/tmp/cap56_array.bin",
            "/tmp/cap56_transfer.bin", "/tmp/cap56_datos.dat",
            "/tmp/cap56_buffered.bin", "/tmp/cap56_patron.bin",
            "/tmp/cap56_append.bin", tempBig
        };
        for (String f : limpiar) {
            new File(f).delete();
        }
        System.out.println("Archivos temporales eliminados.");

        System.out.println("\n=== FIN CAPÍTULO 56 ===");
    }

    public static void copiarArchivo(String origen, String destino)
            throws IOException {
        try (FileInputStream fis = new FileInputStream(origen);
             FileOutputStream fos = new FileOutputStream(destino)) {
            byte[] buffer = new byte[8192];
            int bytesLeidos;
            while ((bytesLeidos = fis.read(buffer)) != -1) {
                fos.write(buffer, 0, bytesLeidos);
            }
        }
    }

    public static byte[] leerArchivoEntero(String ruta) throws IOException {
        File archivo = new File(ruta);
        byte[] contenido = new byte[(int) archivo.length()];
        try (FileInputStream fis = new FileInputStream(archivo)) {
            int offset = 0;
            int leidos;
            while (offset < contenido.length
                    && (leidos = fis.read(contenido, offset,
                        contenido.length - offset)) != -1) {
                offset += leidos;
            }
        }
        return contenido;
    }
}

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

• FileInputStream y FileOutputStream son streams de bytes para leer y
  escribir datos binarios. No realizan conversión de codificación.

• read() lee un byte (0-255) o -1 en EOF. read(byte[]) lee en bloque.

• write(int) escribe un byte. write(byte[]) escribe un arreglo.
  write(byte[], off, len) escribe una porción.

• Siempre usar buffers (arreglos de bytes) para mejorar rendimiento. El
  tamaño óptimo suele ser entre 8 KB y 64 KB.

• InputStream.transferTo(OutputStream) (Java 9+) copia datos de forma
  eficiente y concisa.

• BufferedInputStream y BufferedOutputStream añaden buffering transparente.

• DataInputStream y DataOutputStream permiten leer/escribir tipos
  primitivos de forma portable (big-endian).

• available() estima bytes disponibles sin bloqueo. skip() salta bytes.

• ByteArrayInputStream y ByteArrayOutputStream trabajan con bytes en
  memoria, útiles para buffers y transformaciones.

• La elección entre byte streams y character streams depende del tipo
  de datos: binarios o texto.

  EJERCICIOS PROPUESTOS
================================================================================

1. Copia con barra de progreso: Modifica el copiador de archivos para
   mostrar una barra de progreso en porcentaje. Usa File.length() para
   conocer el tamaño total.

2. Visor hexadecimal: Programa que lea cualquier archivo y muestre su
   contenido en formato hexadecimal lado a lado con la representación
   ASCII, 16 bytes por línea (como hexdump o xxd).

3. Calculadora de hash: Lee un archivo y calcula su hash MD5, SHA-256
   y SHA-512 usando java.security.MessageDigest. Muestra los resultados
   en hexadecimal.

4. Buscador de secuencias: Dado un archivo binario y una secuencia de
   bytes (ej: 0x89, 0x50, 0x4E, 0x47 para PNG), encuentra todas las
   ocurrencias y reporta sus posiciones (offsets).

5. Editor hexadecimal simple: Lee un archivo binario, permite al usuario
   modificar bytes en posiciones específicas, y guarda los cambios en
   un nuevo archivo. Muestra una vista hexadecimal interactiva.

6. Divisor de archivos: Divide un archivo grande en partes de N bytes
   cada una. Luego implementa un "unidor" que reconstruya el archivo
   original a partir de las partes. Verifica la integridad.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

P: ¿Cuál es la diferencia entre FileInputStream y FileReader?
R: FileInputStream lee bytes, FileReader lee caracteres. FileReader usa
   un Charset para convertir bytes a caracteres. Para archivos binarios
   usa FileInputStream; para texto usa FileReader.

P: ¿Puedo usar FileOutputStream para escribir texto?
R: Sí, convirtiendo String a bytes con getBytes(). Pero es mejor usar
   FileWriter o OutputStreamWriter con un Charset para texto.

P: ¿Qué tan grande debe ser el buffer para leer archivos?
R: Para la mayoría de casos, 8 KB (8192 bytes) ofrece buen rendimiento.
   Para archivos muy grandes, 64 KB o 1 MB pueden ser más rápidos.
   El tamaño óptimo depende del sistema operativo y hardware.

P: ¿Por qué mi copia de archivo con read()/write() byte a byte es tan
   lenta?
R: Cada llamada a read()/write() involucra una llamada al sistema
   operativo. Con buffer, lees/escribes muchos bytes por llamada,
   reduciendo drásticamente el número de llamadas al sistema.

P: ¿FileOutputStream sobrescribe o añade?
R: Por defecto (new FileOutputStream("archivo")) sobrescribe. Con
   new FileOutputStream("archivo", true) añade al final (append).

P: ¿Qué pasa si leo un archivo de texto con FileInputStream?
R: Funciona, pero obtienes bytes, no caracteres. Debes convertir los
   bytes a String usando el Charset correcto. Es más seguro usar
   InputStreamReader o Files.newBufferedReader().

P: ¿DataOutputStream es compatible entre distintas plataformas?
R: Sí, DataOutputStream siempre escribe en big-endian (network byte
   order), garantizando que DataInputStream en cualquier plataforma
   pueda leer los datos correctamente.

  DESAFÍO DEL ARQUITECTO
================================================================================

  "EXTRACTOR DE METADATOS DE ARCHIVOS BINARIOS"

  Diseña un sistema que analice archivos binarios y extraiga metadatos
  según su tipo (magic number). Debe soportar al menos 5 formatos.

  Requisitos:

  1. Detector de tipo por magic numbers: Los primeros bytes del archivo
     identifican el formato:
     • PNG: 89 50 4E 47 0D 0A 1A 0A
     • JPEG: FF D8 FF
     • PDF: 25 50 44 46
     • ZIP: 50 4B 03 04
     • GIF: 47 49 46 38 (39 61 o 37 61 para versiones)
     • Class Java: CA FE BA BE

  2. Extractores de metadatos específicos:
     • PNG: dimensiones (del chunk IHDR), tipo de color, profundidad.
     • JPEG: dimensiones, tabla de cuantización, comentarios.
     • PDF: número de páginas (xref table), versión.
     • ZIP: lista de archivos internos, tamaño comprimido/no comprimido.
     • GIF: dimensiones, número de colores, animación.
     • Java Class: versión, constant pool, interfaces.

  3. Reporte: Genera un reporte en texto con todos los metadatos
     extraídos. También exporta a JSON.

  4. Soporte para archivos corruptos: Detecta inconsistencias y reporta
     errores de integridad sin lanzar excepciones fatales.

  5. Modo batch: Procesa todos los archivos de un directorio y genera
     un reporte consolidado.

  6. Mínimo 4 clases: MagicNumberDetector, MetadataExtractor,
     MetadataReport, BinaryAnalyzer.

  VALORACIÓN:
  • 3 estrellas: Detector de tipo + extractor para 2 formatos.
  • 4 estrellas: Extractores para 4 formatos + reporte JSON.
  • 5 estrellas: Detección de corrupción + modo batch + 6 formatos.
