1. EL PROGRAMA HOLA MUNDO
===============================================================================

Sin más preámbulos, aquí está el programa más famoso de la historia de la
programación, ahora en su versión Java:

```java
public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("¡Hola, Mundo!");
    }
}
```

¡Felicidades! Acabas de escribir tu primer programa Java. Ahora vamos a
analizar cada línea en detalle para que entiendas exactamente qué hace
cada parte.

```java
// Versión comentada del Hola Mundo

// public: Esta clase es accesible desde cualquier parte
// class: Estamos declarando una clase (unidad básica en OOP)
// HolaMundo: Nombre de la clase (debe coincidir con el nombre del archivo)
public class HolaMundo {

    // public: El método es accesible desde fuera de la clase
    // static: El método pertenece a la clase, no a una instancia
    // void: El método no devuelve ningún valor
    // main: Nombre del método (punto de entrada del programa)
    // String[] args: Argumentos de línea de comandos (array de strings)
    public static void main(String[] args) {

        // System: Clase del sistema
        // out: Flujo de salida estándar (consola)
        // println: Método que imprime una línea y añade un salto de línea
        System.out.println("¡Hola, Mundo!");

    } // Fin del método main
} // Fin de la clase HolaMundo
```

**Reglas importantes:**

1. El nombre de la clase DEBE coincidir con el nombre del archivo.
   - Clase: HolaMundo → Archivo: HolaMundo.java
   - Java es sensible a mayúsculas/minúsculas (case-sensitive).

2. El método main es el punto de entrada del programa. Sin él, el programa
   no puede ejecutarse.

3. Cada instrucción termina con punto y coma (;).

4. Las llaves { } delimitan bloques de código.

2. EL MÉTODO MAIN EN DETALLE
===============================================================================

El método main es el punto de entrada de toda aplicación Java. Cuando
ejecutas java HolaMundo, la JVM busca el método main con esta firma exacta:

```java
public static void main(String[] args)
```

Vamos a desglosar cada palabra:

**public:** Modificador de acceso. Significa que el método es accesible
desde cualquier otra clase. La JVM necesita acceder a main desde fuera
de la clase, por lo que debe ser public.

**static:** Indica que el método pertenece a la clase, no a una instancia
(objeto) de la clase. La JVM llama a main sin crear un objeto de la clase,
por lo que debe ser static.

**void:** Indica que el método no devuelve ningún valor. El método main
finaliza cuando el programa termina, pero no "retorna" un valor al sistema
operativo (a diferencia de C/C++ donde main devuelve int).

**main:** Es el nombre del método. Debe ser exactamente "main" (en
minúsculas). La JVM busca específicamente este nombre.

**String[] args:** Es el parámetro del método. String[] significa "array
de Strings". args es el nombre del parámetro (puede llamarse de otra forma,
como argumentos o params, pero args es la convención). Estos son los
argumentos de línea de comandos.

```java
// Demostración de diferentes formas de declarar main

class VariacionesMain {
    // Forma estándar (la más usada)
    public static void main(String[] args) {
        System.out.println("Forma 1: public static void main(String[] args)");
    }
    
    // Otra sintaxis válida (el array puede ir después del nombre)
    // public static void main(String args[]) { }
    
    // Con argumentos de línea de comandos
    // public static void main(String... args) { } // varargs
}
```

Solo la forma exacta `public static void main(String[] args)` es reconocida
por la JVM como punto de entrada. Cualquier variación hará que el programa
no pueda ejecutarse (aunque compile sin errores).

```java
// Programa que muestra los argumentos de línea de comandos

class MostrarArgumentos {
    public static void main(String[] args) {
        System.out.println("Número de argumentos: " + args.length);
        
        for (int i = 0; i < args.length; i++) {
            System.out.println("Argumento " + i + ": " + args[i]);
        }
        
        if (args.length == 0) {
            System.out.println("No se pasaron argumentos.");
            System.out.println("Ejecuta: java MostrarArgumentos hola mundo 123");
        }
    }
}
```

**Ejecutar con argumentos:**
```bash
java MostrarArgumentos Java es genial
# Salida:
# Número de argumentos: 3
# Argumento 0: Java
# Argumento 1: es
# Argumento 2: genial
```

3. SYSTEM.OUT.PRINTLN Y OTRAS FORMAS DE SALIDA
===============================================================================

System.out.println es la forma más común de mostrar información en la
consola. Pero Java ofrece varias variantes:

```java
// Diferentes formas de imprimir en consola

class SalidaConsola {
    public static void main(String[] args) {
        // println: Imprime y añade salto de línea
        System.out.println("Hola con salto de línea");
        System.out.println("Esta línea aparece debajo");
        
        // print: Imprime SIN salto de línea
        System.out.print("Esto se imprime ");
        System.out.print("en la misma línea");
        
        // Salto de línea manual
        System.out.println(); // Línea en blanco
        
        // printf: Imprime con formato (similar a C)
        String nombre = "Ana";
        int edad = 25;
        System.out.printf("Nombre: %s, Edad: %d años%n", nombre, edad);
        
        // Caracteres especiales
        System.out.println("Línea 1\nLínea 2");       // \n = salto de línea
        System.out.println("Columna1\tColumna2");      // \t = tabulador
        System.out.println("Comilla: \"");              // \" = comilla doble
        System.out.println("Barra invertida: \\");      // \\ = barra invertida
    }
}
```

**Tabla de métodos de salida:**

| Método | Descripción | Ejemplo |
|--------|-------------|---------|
| System.out.println(x) | Imprime x + salto de línea | println("Hola") |
| System.out.print(x) | Imprime x sin salto de línea | print("Hola") |
| System.out.printf(format, args) | Imprime con formato | printf("Edad: %d", 25) |
| System.out.format(format, args) | Igual que printf | format("Valor: %.2f", 3.14) |

System.out es un objeto de tipo PrintStream que representa la "salida
estándar". También existen System.err para errores:

```java
// System.out vs System.err

class SalidaVsError {
    public static void main(String[] args) {
        // Salida estándar (normalmente en la consola)
        System.out.println("Mensaje informativo");
        
        // Salida de error (puede resaltarse en rojo en la terminal)
        System.err.println("Mensaje de error");
        
        // Útil para separar la salida normal de los errores
        System.out.println("Procesando...");
        if (args.length == 0) {
            System.err.println("ERROR: No se proporcionaron argumentos");
        }
    }
}
```

4. COMPILACIÓN Y EJECUCIÓN PASO A PASO
===============================================================================

Ahora que entiendes el código, vamos a compilarlo y ejecutarlo paso a paso.

**Paso 1: Crear el archivo**

Crea un archivo llamado HolaMundo.java con el siguiente contenido:

```java
public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("¡Hola, Mundo!");
    }
}
```

Puedes usar cualquier editor de texto: notepad, nano, vim, VSCode, etc.

**Paso 2: Compilar con javac**

Abre una terminal en el directorio donde guardaste el archivo y ejecuta:

```bash
javac HolaMundo.java
```

Si no hay errores, el comando no muestra nada (el silencio es éxito). Se
habrá generado un archivo HolaMundo.class en el mismo directorio.

**Paso 3: Ejecutar con java**

```bash
java HolaMundo
```

Deberías ver:
```
¡Hola, Mundo!
```

```java
// Programa que muestra información de compilación

class InformacionCompilacion {
```

5. ERRORES COMUNES Y CÓMO SOLUCIONARLOS
===============================================================================

Es normal cometer errores al empezar. Veamos los más comunes y cómo
solucionarlos:

```java
// Este programa contiene errores comunes (NO EJECUTAR)
// Los errores están comentados para que los estudies

class ErroresProgramacion {
    public static void main(String[] args) {
        System.out.println("=== ERRORES COMUNES EN JAVA ===");
        
        // ❌ Error 1: Olvidar el punto y coma
        // System.out.println("Hola")  // Error: ';' expected
        
        // ❌ Error 2: Usar comillas simples para strings
        // System.out.println('Hola'); // Error: unclosed character literal
        
        // ❌ Error 3: Nombre de clase diferente al del archivo
        // Archivo se llama Prueba.java pero la clase es Otra
        // Error: class Otra is public, should be declared in Otra.java
        
        // ❌ Error 4: Escribir mal el método main
        // public static void main(String args) // Error: falta []
        // public void main(String[] args)      // Error: falta static
        
        // ❌ Error 5: Olvidar cerrar llaves
        // public static void main(String[] args) {
        //     System.out.println("Hola");
        // // Error: reached end of file while parsing
        
        System.out.println("¡Revisa tu código si ves errores!");
    }
}
```

**Tabla de errores de compilación comunes:**

| Mensaje de Error | Causa | Solución |
|------------------|-------|----------|
| ';' expected | Falta punto y coma al final de una línea | Agregar ; |
| class X is public, should be declared in X.java | El nombre de la clase pública no coincide con el archivo | Renombrar clase o archivo |
| cannot find symbol | Usas una variable o método que no existe | Verificar nombre/import |
| Main method not found in class X | Falta main o la firma es incorrecta | Agregar public static void main(String[] args) |
| reached end of file while parsing | Falta una llave de cierre } | Revisar y cerrar todos los bloques |
| illegal character: '\u00a0' | Carácter invisible (espacio no separable) | Reescribir la línea |
| unclosed string literal | Falta comilla doble de cierre | Agregar " al final |

```java
// Programa de diagnóstico de errores

class DiagnosticoErrores {
    public static void main(String[] args) {
        System.out.println("=== DIAGNÓSTICO DE ERRORES ===");
        System.out.println();
        System.out.println("Si tu programa no compila:");
        System.out.println("1. Revisa la línea que indica el error");
        System.out.println("2. Verifica mayúsculas/minúsculas");
        System.out.println("3. Confirma que todas las llaves cierran");
        System.out.println("4. Asegura que el nombre del archivo coincide con la clase");
        System.out.println("5. Verifica que main está escrito correctamente");
        System.out.println();
        System.out.println("El compilador javac te dice exactamente");
        System.out.println("dónde está el error y qué esperaba.");
        System.out.println("¡Lee los mensajes de error con atención!");
        System.out.println("======================================");
    }
}
```

6. BUENAS PRÁCTICAS DESDE EL PRINCIPIO
===============================================================================

Aunque estás empezando, es importante adoptar buenas prácticas desde el
primer día. Formar hábitos correctos ahora te ahorrará dolores de cabeza
en el futuro.

**Nombres de clases:** Usan PascalCase (CamelCase con primera letra mayúscula).
Cada palabra comienza con mayúscula: MiPrimeraClase, Calculadora, Usuario.

**Nombres de métodos:** Usan camelCase (primera letra minúscula, siguientes
palabras mayúsculas): calcularTotal(), obtenerNombre(), main().

**Nombres de variables:** camelCase: edad, nombreCompleto, totalPrecio.

**Constantes:** MAYÚSCULAS_CON_GUIONES_BAJOS: VALOR_PI, MAX_USUARIOS.

**Indentación:** Usa 4 espacios por nivel de sangría (no tabs, o configura
tu editor para que los tabs sean 4 espacios).

**Comentarios:** Usa // para una línea y /* */ para múltiples líneas. Los
comentarios deben explicar el POR QUÉ, no el QUÉ.

```java
/**
 * Este es un comentario de documentación (Javadoc).
 * Se usa para generar documentación automática.
 * 
 * Clase que demuestra buenas prácticas de estilo.
 */
class BuenasPracticas {
    
    // Constante: MAYÚSCULAS con guiones bajos
    static final String MENSAJE_BIENVENIDA = "Bienvenido a Java";
    
    // Método main: punto de entrada
    public static void main(String[] args) {
        // Variable local: camelCase
        String nombreUsuario = "Estudiante";
        int edad = 25;
        
        // Método con nombre descriptivo
        imprimirSaludo(nombreUsuario, edad);
    }
    
    // Método: verbo en infinitivo + complemento
    static void imprimirSaludo(String nombre, int edad) {
        System.out.println(MENSAJE_BIENVENIDA);
        System.out.println("Nombre: " + nombre);
        System.out.println("Edad: " + edad);
    }
}
```

**Regla de oro:** El código debe ser legible para humanos, no solo para
máquinas. Escribe pensando en quien leerá tu código en el futuro (incluido
tú mismo dentro de 6 meses).

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener múltiples clases en un mismo archivo?**
Sí, pero solo una puede ser pública. Las demás serán package-private
(accesibles solo dentro del mismo paquete).

❓ **¿Qué pasa si no pongo static en el método main?**
El programa compilará pero no se ejecutará. La JVM mostrará: "Main method
is not static in class X". El método main debe ser static.

❓ **¿El archivo .class contiene código legible?**
No. El bytecode es binario y no legible por humanos directamente. Puedes
usar javap (desensamblador) para verlo, pero no es necesario.

❓ **¿Puedo cambiar el nombre del parámetro args?**
Sí. String[] argumentos o String[] params funcionan igual. Lo importante
es el tipo String[] y la posición.

❓ **¿Por qué mi IDE no muestra la salida del programa?**
En algunos IDEs la consola de salida puede estar oculta. Busca una
pestaña llamada "Run", "Output" o "Console" en tu IDE.
