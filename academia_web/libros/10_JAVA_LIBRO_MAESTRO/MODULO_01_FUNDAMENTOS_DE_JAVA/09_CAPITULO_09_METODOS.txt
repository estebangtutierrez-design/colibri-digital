¡Bienvenido al capítulo de los métodos! Hasta ahora, todo nuestro código
ha estado dentro del método main. Pero a medida que los programas crecen,
necesitamos organizar mejor el código. Los métodos (también llamados
funciones) son la herramienta fundamental para lograrlo.

Un método es un bloque de código que realiza una tarea específica y puede
ser reutilizado múltiples veces. Piensa en ellos como pequeñas máquinas:
reciben datos de entrada (parámetros), procesan algo y devuelven un
resultado (o realizan una acción).

En este capítulo aprenderás a declarar métodos, pasar parámetros, devolver
valores y sobrecargar métodos (overloading). Estos conceptos son la base
de la programación estructurada y el primer paso hacia la programación
orientada a objetos.

===============================================================================
1. ¿QUÉ ES UN MÉTODO?
===============================================================================

Un método es un conjunto de instrucciones agrupadas bajo un nombre, que
realiza una tarea específica. Los métodos permiten:

1. **Reutilización:** Escribes el código una vez y lo usas muchas veces.
2. **Organización:** Divide problemas grandes en partes pequeñas.
3. **Abstracción:** Oculta los detalles de implementación.
4. **Mantenibilidad:** Cambias una sola vez y afecta a todas las llamadas.

**Sintaxis de un método:**
```
modificador tipoRetorno nombre(parámetros) {
    // cuerpo del método
    return valor;
}
```

```java
// Primeros métodos

class IntroduccionMetodos {
    public static void main(String[] args) {
        System.out.println("=== INTRODUCCIÓN A MÉTODOS ===");
        
        // Llamar a un método sin parámetros
        mostrarMensaje();
        
        // Llamar a un método con parámetros
        saludar("Ana");
        saludar("Carlos");
        
        // Llamar a un método que retorna un valor
        int resultado = sumar(5, 3);
        System.out.println("5 + 3 = " + resultado);
        
        // Usar el método directamente
        System.out.println("10 + 20 = " + sumar(10, 20));
        
        // Método void: no retorna valor
        imprimirLinea("Fin del programa");
    }
    
    // Método sin parámetros y sin retorno (void)
    static void mostrarMensaje() {
        System.out.println("¡Bienvenido al estudio de métodos en Java!");
    }
    
    // Método con parámetros y sin retorno (void)
    static void saludar(String nombre) {
        System.out.println("¡Hola, " + nombre + "!");
    }
    
    // Método con parámetros y con retorno (int)
    static int sumar(int a, int b) {
        return a + b;
    }
    
    // Método void que imprime una línea
    static void imprimirLinea(String mensaje) {
        System.out.println(">>> " + mensaje + " <<<");
    }
}
```

2. DECLARACIÓN DE MÉTODOS: PARTES FUNDAMENTALES
===============================================================================

Cada método tiene las siguientes partes:

```java
// Anatomía completa de un método

class AnatomiaMetodo {
    public static void main(String[] args) {
        System.out.println("=== ANATOMÍA DE UN MÉTODO ===");
        
        double resultado = calcularIMC(70, 1.75);
        System.out.printf("IMC: %.2f - %s%n", resultado, clasificarIMC(resultado));
    }
    
    /**
     * MODIFICADORES:
     * - public: accesible desde cualquier clase
     * - static: pertenece a la clase (no a una instancia)
     * 
     * TIPO DE RETORNO: double
     * 
     * NOMBRE: calcularIMC (verbo en infinitivo + complemento)
     * 
     * PARÁMETROS: double peso, double altura
     * 
     * CUERPO: { ... }
     */
    public static double calcularIMC(double peso, double altura) {
        return peso / (altura * altura);
    }
    
    /**
     * MODIFICADOR: private (solo accesible dentro de esta clase)
     * 
     * TIPO DE RETORNO: String
     * 
     * NOMBRE: clasificarIMC
     * 
     * PARÁMETRO: double imc
     */
    private static String clasificarIMC(double imc) {
        if (imc < 18.5) return "Bajo peso";
        if (imc < 25) return "Normal";
        if (imc < 30) return "Sobrepeso";
        return "Obesidad";
    }
}
```

**Elementos de la declaración de un método:**

| Elemento | Descripción | Ejemplos |
|----------|-------------|----------|
| Modificador de acceso | Controla visibilidad | public, private, protected, (default) |
| Modificador de clase | static, final, abstract | static (pertenece a la clase) |
| Tipo de retorno | Lo que devuelve | int, double, String, void (nada) |
| Nombre | Identificador del método | calcularTotal, mostrarMenu |
| Parámetros | Datos de entrada (opcional) | (int a, String b) |
| Cuerpo | Código entre llaves | { ... } |
| Return | Devuelve el resultado (si no es void) | return valor; |

```java
// Diferentes firmas de métodos

class FirmasMetodos {
    public static void main(String[] args) {
        System.out.println("=== DIFERENTES FIRMAS DE MÉTODOS ===");
        
        // Sin parámetros y void
        mostrarSeparador();
        
        // Con un parámetro y void
        System.out.print("Cuadrado de 7: ");
        mostrarCuadrado(7);
        
        // Con múltiples parámetros y retorno
        int resultado = maximo(10, 7);
        System.out.println("Máximo entre 10 y 7: " + resultado);
        
        // Sin parámetros y con retorno
        double pi = obtenerPi();
        System.out.println("Pi: " + pi);
        
        // Con parámetros de diferentes tipos
        String info = formatearInfo("Ana", 25, 1.75);
        System.out.println("Info: " + info);
    }
    
    // void sin parámetros
    static void mostrarSeparador() {
        System.out.println("======================");
    }
    
    // void con un parámetro
    static void mostrarCuadrado(int n) {
        System.out.println(n * n);
    }
    
    // con retorno y parámetros
    static int maximo(int a, int b) {
        return a > b ? a : b;
    }
    
    // con retorno sin parámetros
    static double obtenerPi() {
        return 3.14159;
    }
    
    // múltiples parámetros de diferentes tipos
    static String formatearInfo(String nombre, int edad, double altura) {
        return String.format("%s, %d años, %.2f m", nombre, edad, altura);
    }
}
```

3. PARÁMETROS Y RETURN
===============================================================================

```java
// Parámetros y return en detalle

class ParametrosReturn {
    public static void main(String[] args) {
        System.out.println("=== PARÁMETROS Y RETURN ===\n");
        
        // Paso por valor (copia del valor)
        int x = 10;
        System.out.println("Antes de modificar: x = " + x);
        modificarPrimitivo(x);
        System.out.println("Después de modificar: x = " + x + " (¡no cambió!)");
        
        // Return múltiple (a través de un objeto)
        System.out.println("\n=== OPERACIONES MATEMÁTICAS ===");
        int a = 15;
        int b = 4;
        
        System.out.printf("%d + %d = %d%n", a, b, sumar(a, b));
        System.out.printf("%d - %d = %d%n", a, b, restar(a, b));
        System.out.printf("%d * %d = %d%n", a, b, multiplicar(a, b));
        System.out.printf("%d / %d = %.2f%n", a, b, dividir(a, b));
        System.out.printf("%d %% %d = %d%n", a, b, modulo(a, b));
        
        // Métodos que retornan boolean
        System.out.println("\n=== MÉTODOS BOOLEANOS ===");
        System.out.println("¿Es par 7? " + esPar(7));
        System.out.println("¿Es par 10? " + esPar(10));
        System.out.println("¿Está entre 1 y 10? " + estaEnRango(5, 1, 10));
        System.out.println("¿Está entre 1 y 10? " + estaEnRango(15, 1, 10));
        
        // Método que retorna String
        System.out.println("\n=== GENERACIÓN DE MENSAJES ===");
        String mensaje = generarSaludo("Profesor Búho", "mañana");
        System.out.println(mensaje);
        
        // El return puede estar en cualquier lugar
        System.out.println("\n=== RETURN TEMPRANO ===");
        System.out.println("Factorial de 5: " + factorial(5));
        System.out.println("Factorial de -3: " + factorial(-3) + " (inválido)");
    }
    
    // Demostración de paso por valor
    static void modificarPrimitivo(int valor) {
        valor = 100;  // Solo modifica la copia local
        System.out.println("   Dentro del método: valor = " + valor);
    }
    
    static int sumar(int a, int b) { return a + b; }
    static int restar(int a, int b) { return a - b; }
    static int multiplicar(int a, int b) { return a * b; }
    static double dividir(int a, int b) { return (double) a / b; }
    static int modulo(int a, int b) { return a % b; }
    
    static boolean esPar(int numero) {
        return numero % 2 == 0;
    }
    
    static boolean estaEnRango(int valor, int min, int max) {
        return valor >= min && valor <= max;
    }
    
    static String generarSaludo(String nombre, String momento) {
        return String.format("¡Buen%s %s, %s! ¿Cómo estás hoy?", 
            momento.equals("mañana") ? "os" : "as",
            momento, nombre);
    }
    
    // Return temprano (early return)
    static int factorial(int n) {
        if (n < 0) {
            return -1;  // Código de error
        }
        if (n == 0 || n == 1) {
            return 1;
        }
        int resultado = 1;
        for (int i = 2; i <= n; i++) {
            resultado *= i;
        }
        return resultado;
    }
}
```

**Paso por valor:** En Java, los parámetros se pasan por valor. Esto
significa que se crea una copia del valor. Si el método modifica el
parámetro, la variable original no se ve afectada (para tipos primitivos).

Para objetos, se pasa la copia de la referencia, por lo que puedes
modificar el objeto pero no reasignar la referencia.

```java
// Paso por valor con objetos

class PasoPorValorObjetos {
    public static void main(String[] args) {
        System.out.println("=== PASO POR VALOR CON OBJETOS ===");
        
        StringBuilder sb = new StringBuilder("Hola");
        System.out.println("Antes: " + sb);
        
        modificarStringBuilder(sb);
        System.out.println("Después: " + sb + " (¡el objeto se modificó!)");
        
        // Pero no se puede reasignar la referencia
        StringBuilder otro = new StringBuilder("Original");
        System.out.println("\nAntes de reasignar: " + otro);
        intentarReasignar(otro);
        System.out.println("Después de reasignar: " + otro + " (no cambió!)");
    }
    
    static void modificarStringBuilder(StringBuilder sb) {
        sb.append(" Mundo");  // Modifica el objeto
        System.out.println("   Dentro (append): " + sb);
    }
    
    static void intentarReasignar(StringBuilder sb) {
        sb = new StringBuilder("Nuevo");  // Solo cambia la copia local
        System.out.println("   Dentro (reasignar): " + sb);
    }
}
```

4. MÉTODOS STATIC VS DE INSTANCIA
===============================================================================

Hasta ahora hemos usado métodos static (pertenecen a la clase). Pero
también existen métodos de instancia (pertenecen a un objeto).

```java
// Static vs Instancia

class MetodosStaticVsInstancia {
    // Variable de clase (static)
    static int contadorGlobal = 0;
    
    // Variable de instancia (no static)
    String nombre;
    
    // Constructor
    MetodosStaticVsInstancia(String nombre) {
        this.nombre = nombre;
        contadorGlobal++;
    }
    
    public static void main(String[] args) {
        System.out.println("=== STATIC VS INSTANCIA ===\n");
        
        // Método static: se llama desde la clase
        System.out.println("Método static:");
        mostrarInfoGlobal();
        System.out.println("   PI = " + getPI());
        System.out.println("   Contador = " + contadorGlobal);
        
        System.out.println();
        
        // Método de instancia: necesita un objeto
        MetodosStaticVsInstancia obj1 = new MetodosStaticVsInstancia("Ana");
        MetodosStaticVsInstancia obj2 = new MetodosStaticVsInstancia("Luis");
        
        System.out.println("Métodos de instancia:");
        System.out.println("   Objeto 1: " + obj1.saludar());
        System.out.println("   Objeto 2: " + obj2.saludar());
        
        String info1 = obj1.obtenerInfo();
        String info2 = obj2.obtenerInfo();
        System.out.println("   " + info1);
        System.out.println("   " + info2);
        
        // Static puede acceder a static
        System.out.println("\nContador global: " + contadorGlobal);
        
        // Static NO puede acceder directamente a variables de instancia
        // System.out.println(nombre); // Error!
    }
    
    // Método static: pertenece a la clase
    static void mostrarInfoGlobal() {
        System.out.println("   Clase: MetodosStaticVsInstancia");
        System.out.println("   Versión: 1.0");
    }
    
    static double getPI() {
        return Math.PI;
    }
    
    // Método de instancia: pertenece a cada objeto
    String saludar() {
        return "¡Hola! Soy " + nombre;
    }
    
    String obtenerInfo() {
        return "Usuario: " + nombre + " (ID: " + contadorGlobal + ")";
    }
}
```

**Diferencias clave:**

| Característica | static | Instancia |
|----------------|--------|-----------|
| Pertenece a | La clase | El objeto |
| Se llama con | Clase.metodo() | objeto.metodo() |
| Accede a variables static | Sí | Sí |
| Accede a variables instancia | No (no hay this) | Sí |
| Necesita objeto? | No | Sí |
| Memoria | Una copia compartida | Una copia por objeto |

```java
// Cuándo usar static y cuándo no

class CuandoUsarStatic {
    public static void main(String[] args) {
        System.out.println("=== ¿CUÁNDO USAR STATIC? ===");
        System.out.println();
        
        // ✅ Usa static cuando:
        System.out.println("✅ USAR STATIC:");
        System.out.println("   - Métodos de utilidad (Math.pow(), Utils.formatear())");
        System.out.println("   - Constantes (public static final)");
        System.out.println("   - Métodos que no dependen del estado del objeto");
        System.out.println("   - El método main (es static)");
        System.out.println("   - Contadores globales");
        System.out.println();
        
        // ❌ No uses static cuando:
        System.out.println("❌ EVITAR STATIC:");
        System.out.println("   - El método necesita acceder a estado de instancia");
        System.out.println("   - Necesitas polimorfismo (override)");
        System.out.println("   - El método debe ser sobrescrito en subclases");
        System.out.println("   - Abusar de static dificulta las pruebas unitarias");
        
        // Ejemplo de método de utilidad static
        System.out.println("\nEjemplo: Método de utilidad static");
        System.out.println("   " + Utilidades.formatearMoneda(1234.5));
        System.out.println("   " + Utilidades.esEmailValido("test@test.com"));
        System.out.println("   " + Utilidades.generarUUID());
    }
}

// Clase de utilidades con métodos static
class Utilidades {
    static String formatearMoneda(double cantidad) {
        return String.format("$%,.2f", cantidad);
    }
    
    static boolean esEmailValido(String email) {
        return email != null && email.contains("@") && email.contains(".");
    }
    
    static String generarUUID() {
        return java.util.UUID.randomUUID().toString();
    }
}
```

5. SOBRECARGA DE MÉTODOS (OVERLOADING)
===============================================================================

La sobrecarga de métodos (overloading) permite tener varios métodos con
el MISMO nombre pero DIFERENTES parámetros. Java distingue los métodos
por su firma (nombre + tipo y cantidad de parámetros).

```java
// Sobrecarga de métodos

class Sobrecarga {
    public static void main(String[] args) {
        System.out.println("=== SOBRECARGA DE MÉTODOS ===\n");
        
        System.out.println("El mismo nombre 'sumar' con diferentes parámetros:");
        System.out.println();
        
        // sumar(int, int)
        System.out.println("sumar(int, int): " + sumar(5, 3));           // 8
        
        // sumar(int, int, int)
        System.out.println("sumar(int, int, int): " + sumar(5, 3, 2));   // 10
        
        // sumar(double, double)
        System.out.println("sumar(double, double): " + sumar(5.5, 3.2)); // 8.7
        
        // sumar(double, double, double)
        System.out.println("sumar(double, double, double): " + sumar(5.5, 3.2, 1.3)); // 10.0
        
        // sumar(String, String)
        System.out.println("sumar(String, String): " + sumar("Hola ", "Mundo")); // Hola Mundo
        
        // sumar(int[])
        int[] numeros = {1, 2, 3, 4, 5};
        System.out.println("sumar(int[]): " + sumar(numeros));           // 15
        
        System.out.println("\n=== OTROS EJEMPLOS DE SOBRECARGA ===");
        
        // println es un ejemplo de sobrecarga
        System.out.println("System.out.println está sobrecargado:");
        System.out.println(42);           // println(int)
        System.out.println(3.14);         // println(double)
        System.out.println(true);         // println(boolean)
        System.out.println('A');          // println(char)
        System.out.println("Texto");      // println(String)
        
        // Nuestro método imprimir sobrecargado
        imprimir("Hola");
        imprimir(42);
        imprimir(3.14);
        imprimir(true);
        imprimir("Nombre", "Edad");
    }
    
    // Versiones sobrecargadas de sumar
    static int sumar(int a, int b) {
        System.out.println("  → sumar(int, int)");
        return a + b;
    }
    
    static int sumar(int a, int b, int c) {
        System.out.println("  → sumar(int, int, int)");
        return a + b + c;
    }
    
    static double sumar(double a, double b) {
        System.out.println("  → sumar(double, double)");
        return a + b;
    }
    
    static double sumar(double a, double b, double c) {
        System.out.println("  → sumar(double, double, double)");
        return a + b + c;
    }
    
    static String sumar(String a, String b) {
        System.out.println("  → sumar(String, String)");
        return a + b;
    }
    
    static int sumar(int[] valores) {
        System.out.println("  → sumar(int[])");
        int total = 0;
        for (int v : valores) {
            total += v;
        }
        return total;
    }
    
    // Sobrecarga de imprimir
    static void imprimir(String texto) {
        System.out.println("📝 String: " + texto);
    }
    
    static void imprimir(int numero) {
        System.out.println("🔢 Int: " + numero);
    }
    
    static void imprimir(double decimal) {
        System.out.println("📊 Double: " + decimal);
    }
    
    static void imprimir(boolean bool) {
        System.out.println("✅ Boolean: " + bool);
    }
    
    static void imprimir(String clave, String valor) {
        System.out.println("📋 Par: " + clave + " = " + valor);
    }
}
```

**Reglas de la sobrecarga:**
1. El nombre del método debe ser el mismo.
2. Los parámetros deben diferir en tipo, cantidad o ambos.
3. El tipo de retorno NO forma parte de la firma (no puedes sobrecargar
   solo cambiando el retorno).
4. El modificador de acceso NO afecta la sobrecarga.

```java
// Reglas de sobrecarga - qué es válido y qué no

class ReglasSobrecarga {
    public static void main(String[] args) {
        System.out.println("=== REGLAS DE SOBRECARGA ===");
    }
    
    // ✅ Válido: diferente cantidad de parámetros
    void metodo(int a) { }
    void metodo(int a, int b) { }
    
    // ✅ Válido: diferente tipo de parámetros
    void metodo(int a) { }
    void metodo(double a) { }
    
    // ✅ Válido: diferente orden
    void metodo(int a, double b) { }
    void metodo(double a, int b) { }
    
    // ❌ Inválido: solo cambiar el tipo de retorno
    // int calcular() { return 0; }
    // double calcular() { return 0.0; } // Error
    
    // ❌ Inválido: solo cambiar el modificador
    // public void ejecutar() { }
    // private void ejecutar() { } // Error
    
    // ❌ Inválido: solo cambiar el nombre del parámetro
    // void hacer(int x) { }
    // void hacer(int y) { } // Error (misma firma)
}
```

6. COMPOSICIÓN DE MÉTODOS
===============================================================================

Un método puede llamar a otros métodos. Esto se llama composición y es
fundamental para crear programas modulares.

```java
// Composición de métodos

class ComposicionMetodos {
    public static void main(String[] args) {
        System.out.println("=== COMPOSICIÓN DE MÉTODOS ===\n");
        
        String nombre = "Ana López";
        double[] calificaciones = {85.5, 90.0, 78.5, 92.0};
        
        // Un método que llama a otros métodos
        procesarEstudiante(nombre, calificaciones);
        
        System.out.println();
        
        // Otro estudiante
        procesarEstudiante("Luis Pérez", new double[]{70.0, 65.5, 80.0, 72.5});
    }
    
    // Método principal que compone otros métodos
    static void procesarEstudiante(String nombre, double[] notas) {
        System.out.println("═══════════════════════════════");
        System.out.println("  Estudiante: " + nombre);
        System.out.println("───────────────────────────────");
        
        mostrarNotas(notas);
        
        double promedio = calcularPromedio(notas);
        String estado = determinarEstado(promedio);
        String calificacion = convertirACalificacion(promedio);
        
        mostrarResultado(promedio, estado, calificacion);
        System.out.println("═══════════════════════════════\n");
    }
    
    static void mostrarNotas(double[] notas) {
        System.out.print("  Notas: ");
        for (int i = 0; i < notas.length; i++) {
            System.out.printf("%.1f", notas[i]);
            if (i < notas.length - 1) System.out.print(", ");
        }
        System.out.println();
    }
    
    static double calcularPromedio(double[] notas) {
        double suma = 0;
        for (double nota : notas) {
            suma += nota;
        }
        return suma / notas.length;
    }
    
    static String determinarEstado(double promedio) {
        return promedio >= 60 ? "APROBADO" : "REPROBADO";
    }
    
    static String convertirACalificacion(double promedio) {
        if (promedio >= 90) return "A (Excelente)";
        if (promedio >= 80) return "B (Bueno)";
        if (promedio >= 70) return "C (Regular)";
        if (promedio >= 60) return "D (Suficiente)";
        return "F (Insuficiente)";
    }
    
    static void mostrarResultado(double promedio, String estado, String calificacion) {
        System.out.printf("  Promedio: %.2f%n", promedio);
        System.out.println("  Estado: " + estado);
        System.out.println("  Calificación: " + calificacion);
    }
}
```

```java
// Calculadora modular con composición de métodos

class CalculadoraModular {
    public static void main(String[] args) {
        System.out.println("=== CALCULADORA MODULAR ===\n");
        
        ejecutarCalculadora(10, 5);

        mostrarOperacion("Multiplicación", a, b, "*", multiplicar(a, b));
        mostrarOperacion("División", a, b, "/", dividir(a, b));
        mostrarOperacion("Potencia", a, b, "^", potencia(a, b));
        
        System.out.println();
    }
    
    static double sumar(double a, double b) { return a + b; }
    static double restar(double a, double b) { return a - b; }

    static double dividir(double a, double b) { return b != 0 ? a / b : Double.NaN; }
    static double potencia(double base, double exp) { return Math.pow(base, exp); }
    
    static void mostrarOperacion(String nombre, double a, double b, String operador, double resultado) {

    }
```

7. MÉTODOS CON VARARGS (ARGUMENTOS VARIABLES)
===============================================================================

Java permite métodos que acepten un número variable de argumentos usando
los tres puntos (...) en el parámetro.

```java
// Varargs: número variable de argumentos

class Varargs {
    public static void main(String[] args) {
        System.out.println("=== VARARGS (ARGUMENTOS VARIABLES) ===\n");
        
        // Un método varargs puede recibir 0, 1, o N argumentos
        System.out.println("sumar(): " + sumar());                 // 0
        System.out.println("sumar(5): " + sumar(5));              // 1
        System.out.println("sumar(1, 2, 3): " + sumar(1, 2, 3));  // 3
        System.out.println("sumar(1, 2, 3, 4, 5): " + sumar(1, 2, 3, 4, 5)); // 5
        
        System.out.println("\n=== OTROS EJEMPLOS ===");
        
        System.out.printf("Máximo de (3, 7, 2, 9, 1): %d%n", maximo(3, 7, 2, 9, 1));
        System.out.printf("Máximo de (100, 50): %d%n", maximo(100, 50));
        System.out.printf("Máximo de (-5): %d%n", maximo(-5));
        
        String mensaje = concatenar("Java", "es", "genial", "!");
        System.out.println("Concatenación: " + mensaje);
        
        String lista = listarElementos("Ana", "Luis", "Carlos");
        System.out.println("Lista: " + lista);
        
        // También funciona con arrays
        System.out.println("\nCon array:");
        int[] valores = {10, 20, 30};
        System.out.println("sumar(array): " + sumar(valores));
    }
    
    // Varargs: los ... indican que acepta 0 o más ints
    static int sumar(int... numeros) {
        int total = 0;
        for (int n : numeros) {
            total += n;
        }
        return total;
    }
    
    // Varargs con otros parámetros (el varargs debe ser el ÚLTIMO)
    static int maximo(int primero, int... resto) {
        System.out.print("  maximo(" + primero);
        for (int r : resto) System.out.print(", " + r);
        System.out.print(") → ");
        
        int max = primero;
        for (int r : resto) {
            if (r > max) max = r;
        }
        return max;
    }
    
    // Varargs con String
    static String concatenar(String... palabras) {
        StringBuilder sb = new StringBuilder();
        for (String p : palabras) {
            sb.append(p).append(" ");
        }
        return sb.toString().trim();
    }
    
    // Varargs con formato
    @SafeVarargs
    static <T> String listarElementos(T... elementos) {
        if (elementos.length == 0) return "Vacío";
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < elementos.length; i++) {
            if (i > 0) sb.append(", ");
            sb.append(elementos[i]);
        }
        return sb.toString();
    }
}
```

**Reglas de Varargs:**
1. Solo puede haber UN parámetro varargs por método.
2. El varargs debe ser el ÚLTIMO parámetro.
3. Internamente, el compilador lo trata como un array.
4. Puedes pasar un array directamente o los elementos separados por coma.

RESUMEN DEL CAPÍTULO
===============================================================================

- Un método agrupa código para realizar una tarea específica y reutilizable.
- La firma de un método incluye nombre + tipo y cantidad de parámetros.
- El tipo de retorno void indica que el método no devuelve nada.
- Los parámetros se pasan por valor (copia del valor).
- static significa que el método pertenece a la clase, no a una instancia.
- La sobrecarga permite múltiples métodos con el mismo nombre pero
  diferentes parámetros.
- La composición de métodos permite construir programas modulares.
- Los varargs (...) permiten pasar un número variable de argumentos.
- Un método debe HACER UNA SOLA COSA y hacerla bien.
- Los nombres de métodos deben ser verbos descriptivos en camelCase.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Puedo tener un método dentro de otro método?**
No. En Java no se permiten métodos anidados (a diferencia de JavaScript
o Python). Todos los métodos deben declararse dentro de la clase, pero
no dentro de otros métodos.

❓ **¿Un método puede devolver varios valores?**
No directamente. Un método solo puede devolver UN valor. Si necesitas
devolver varios, puedes usar un array, una lista, o crear una clase
específica para agrupar los valores.

❓ **¿Qué es el método main? ¿Por qué es static?**
El método main es el punto de entrada del programa. Es static porque la
JVM lo llama sin crear una instancia de la clase. Si no fuera static,
la JVM tendría que instanciar la clase, lo cual es un paso extra.

❓ **¿Puedo llamar a un método static desde un método de instancia?**
Sí. Los métodos static son accesibles desde cualquier contexto (static
o de instancia). Pero un método static NO puede acceder directamente a
variables o métodos de instancia.

❓ **¿Cuál es la diferencia entre parámetro y argumento?**
El parámetro es la variable definida en la declaración del método. El
argumento es el valor real que se pasa al invocar el método. Por ejemplo:
en sumar(int a, int b), a y b son parámetros; en sumar(5, 3), 5 y 3
son argumentos.
