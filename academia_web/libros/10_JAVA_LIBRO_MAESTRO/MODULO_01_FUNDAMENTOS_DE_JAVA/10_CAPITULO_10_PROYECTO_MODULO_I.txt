1. DESCRIPCIÓN DEL PROYECTO
===============================================================================

**Sistema de Gestión Académica (SGA) v1.0**

Vamos a construir una aplicación de consola que permita gestionar
estudiantes, cursos y calificaciones. El sistema debe:

**Requisitos funcionales:**

1. **Gestión de Estudiantes:**
   - Registrar un nuevo estudiante (nombre, email, teléfono).
   - Listar todos los estudiantes registrados.
   - Buscar un estudiante por nombre o email.
   - Actualizar información de un estudiante.

2. **Gestión de Cursos:**
   - Registrar un nuevo curso (nombre, código, créditos).
   - Listar todos los cursos disponibles.
   - Buscar un curso por nombre o código.

3. **Gestión de Calificaciones:**
   - Asignar una calificación a un estudiante en un curso.
   - Calcular el promedio de un estudiante.
   - Mostrar el historial académico de un estudiante.
   - Mostrar estadísticas de un curso (promedio, max, min).

4. **Características del sistema:**
   - Menú principal interactivo.
   - Validación de entrada del usuario.
   - Mensajes de error claros y descriptivos.
   - Formato de salida profesional (tablas).
   - Capacidad para almacenar datos en arrays (sin base de datos).

**Restricciones técnicas:**
- Usar solo lo aprendido en el Módulo I (tipos, operadores, Strings,
  Scanner, métodos, arrays).
- No usar colecciones (List, ArrayList) ni POO avanzada (herencia).
- Todo el código debe estar organizado en métodos bien nombrados.
- Los datos se almacenan en arrays de tamaño fijo (máximo 100 estudiantes,
  50 cursos).

    static void asigCal() {

        int c = leerInt("  ID curso: ", 0, cCur - 1);

        System.out.printf("  Asignada: %.1f (%s)%n", n, aLetra(n));

    static void histEst() {
        if (cEst == 0) { System.out.println("  No hay estudiantes."); return; }
        listEst();
        int e = leerInt("  ID estudiante: ", 0, cEst - 1);
        System.out.printf("%n=== HISTORIAL: %s ===%n", noms[e].toUpperCase());
        double suma = 0; int cnt = 0;
        for (int i = 0; i < cCal; i++) {
            if (eId[i] == e) {
                System.out.printf("  %-25s %.1f (%s)%n", cortar(nomC[cId[i]], 23), notas[i], aLetra(notas[i]));
                suma += notas[i]; cnt++;
            }
        }
        if (cnt == 0) { System.out.println("  Sin calificaciones."); return; }
        double p = suma / cnt;
        System.out.printf("  Promedio: %.2f (%s)%n", p, aLetra(p));
    }

    static void estCur() {
        if (cCur == 0) { System.out.println("  No hay cursos."); return; }
        listCur();
        int c = leerInt("  ID curso: ", 0, cCur - 1);
        double suma = 0; int cnt = 0, apro = 0;
        double max = -1, min = 101;
        for (int i = 0; i < cCal; i++) {
            if (cId[i] == c) {
                double n = notas[i]; suma += n; cnt++;
                if (n > max) max = n; if (n < min) min = n;
                if (n >= 60) apro++;
            }
        }
        if (cnt == 0) { System.out.println("  Sin calificaciones."); return; }
        double p = suma / cnt;
        System.out.printf("  Prom: %.2f | Max: %.2f | Min: %.2f%n", p, max, min);
        System.out.printf("  Aprob: %d/%d (%.1f%%) | %s%n", apro, cnt, (double)apro/cnt*100, aLetra(p));
    }

    static String aLetra(double n) {
        return n >= 90 ? "A" : n >= 80 ? "B" : n >= 70 ? "C" : n >= 60 ? "D" : "F";
    }

            if (notas[i] >= 60) apr[cId[i]]++;
        }
        System.out.printf("%-4s %-25s %-8s %-10s%n", "ID", "CURSO", "PROM", "% APROB");
        for (int i = 0; i < cCur; i++) {
            if (cnt[i] > 0) {
                double p = sum[i] / cnt[i];
                System.out.printf("%-4d %-25s %-8.2f %-9.1f%%%n", i, cortar(nomC[i], 23), p, (double)apr[i]/cnt[i]*100);

    // ========== VALIDACION Y UTILIDADES ==========

    static int leerInt(String msg, int min, int max) {
        while (true) {
            System.out.print(msg);
            if (sc.hasNextInt()) {
                int v = sc.nextInt(); sc.nextLine();
                if (v >= min && v <= max) return v;
                System.out.printf("  Entre %d y %d.%n", min, max);
            } else { System.out.println("  Numero invalido."); sc.nextLine(); }
        }
    }

    static double leerDbl(String msg, double min, double max) {
        while (true) {
            System.out.print(msg);
            if (sc.hasNextDouble()) {
                double v = sc.nextDouble(); sc.nextLine();
                if (v >= min && v <= max) return v;
                System.out.printf("  Entre %.1f y %.1f.%n", min, max);
            } else { System.out.println("  Invalido."); sc.nextLine(); }
        }
    }

    static String leerStr(String msg, int min, int max) {
        while (true) {
            System.out.print(msg);
            String s = sc.nextLine().trim();
            if (s.length() >= min && s.length() <= max) return s;
            System.out.printf("  Entre %d y %d caracteres.%n", min, max);
        }
    }

    static String leerEmail() {
        while (true) {
            System.out.print("  Email: ");
            String e = sc.nextLine().trim().toLowerCase();
            if (e.contains("@") && e.contains(".") && e.length() >= 5) {
                boolean dup = false;
                for (int i = 0; i < cEst; i++) if (emails[i] != null && emails[i].equals(e)) { dup = true; break; }
                if (!dup) return e;
                System.out.println("  Email duplicado.");
            } else System.out.println("  Email invalido.");
        }
    }

    }

    static void pausa() {
        System.out.print("\n  Enter para continuar...");
        sc.nextLine();
    }

    static String cortar(String s, int max) {
        if (s == null || s.length() <= max) return s;
        return s.substring(0, max - 3) + "...";
    }
}

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Por qué usamos arrays paralelos en lugar de objetos?**
Porque en el Módulo I aún no hemos cubierto programación orientada a
objetos. Los arrays paralelos son una forma simple de almacenar datos
relacionados. En el Módulo II usaremos clases y objetos.

❓ **¿Cómo puedo hacer que los arrays sean dinámicos?**
Los arrays en Java tienen tamaño fijo. Si necesitas crecimiento dinámico,
puedes usar ArrayList (lo veremos en módulos posteriores) o crear un nuevo
array más grande y copiar los datos.

❓ **¿Qué pasa si el usuario ingresa datos incorrectos repetidamente?**
Los métodos de validación usan bucles while que se repiten hasta que la
entrada sea correcta. El programa no avanzará hasta recibir datos válidos.

❓ **¿Puedo agregar más funcionalidades al proyecto?**
¡Por supuesto! Este proyecto es tu base. Puedes agregar persistencia
(guardar/cargar datos), interfaz gráfica, más reportes, etc.

❓ **¿El proyecto tiene que ser exactamente igual?**
No. El diseño que te presento es una guía. Siéntete libre de mejorarlo,
cambiar la interfaz, agregar funcionalidades o modificar la estructura
según tu criterio.
