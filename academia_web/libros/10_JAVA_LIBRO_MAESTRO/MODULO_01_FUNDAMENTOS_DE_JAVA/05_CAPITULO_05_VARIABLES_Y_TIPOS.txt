1. ¿QUÉ ES UNA VARIABLE?
===============================================================================

Una variable es un espacio en la memoria del ordenador que almacena un
valor. Puedes pensar en ella como una caja etiquetada: la etiqueta es el
nombre de la variable y el contenido es el valor que almacena.

En Java, toda variable debe ser declarada antes de usarse. La declaración
especifica el tipo de dato y el nombre:

```
tipo nombreVariable;
```

```java
// Declaración de variables en Java

class DeclaracionVariables {
    public static void main(String[] args) {
        // Declaración: tipo + nombre
        int edad;
        double precio;
        char letra;
        boolean activo;
        
        // Asignación: dar un valor a la variable
        edad = 25;
        precio = 19.99;
        letra = 'A';
        activo = true;
        
        // Declaración + asignación en una línea
        String nombre = "Ana";
        int cantidad = 100;
        
        // Las variables se pueden reasignar
        edad = 26;
        System.out.println("Edad actualizada: " + edad);
        
        // Imprimir valores
        System.out.println("Nombre: " + nombre);
        System.out.println("Edad: " + edad);
        System.out.println("Precio: " + precio);
        System.out.println("Letra: " + letra);
        System.out.println("Activo: " + activo);
        System.out.println("Cantidad: " + cantidad);
    }
}
```

**Reglas para nombrar variables:**

1. Deben comenzar con una letra (a-z, A-Z), guión bajo (_) o símbolo de dólar ($).
2. Los siguientes caracteres pueden ser letras, dígitos, guiones bajos o dólares.
3. No pueden usar palabras reservadas de Java (int, class, public, etc.).
4. Son sensibles a mayúsculas/minúsculas: edad, Edad, EDAD son diferentes.
5. Por convención, se usa camelCase: nombreCompleto, fechaNacimiento.

```java
// Ejemplos de nombres válidos e inválidos

class NombresVariables {
    public static void main(String[] args) {
        // Válidos
        int edad = 25;
        int _edad = 25;        // válido, pero no recomendado
        int $edad = 25;         // válido, pero no recomendado
        int edadUsuario = 25;   // camelCase (recomendado)
        int edad2 = 25;         // válido
        int año = 2024;         // válido (con ñ)
        
        // Inválidos (descomentar para ver errores)
        // int 2edad = 25;      // Error: comienza con dígito
        // int edad usuario = 25; // Error: espacio
        // int class = 25;      // Error: palabra reservada
        // int edad-usuario = 25; // Error: guión
        
        System.out.println("Nombres válidos demostrados");
    }
}
```

2. TIPOS PRIMITIVOS ENTEROS
===============================================================================

Java tiene 8 tipos primitivos. Los primeros 4 son para números enteros:

**byte:** 8 bits, rango de -128 a 127. Útil para ahorrar memoria en arrays grandes.

**short:** 16 bits, rango de -32,768 a 32,767.

**int:** 32 bits, rango de -2^31 a 2^31-1 (aproximadamente ±2.1 mil millones).
Es el tipo entero más usado por defecto.

**long:** 64 bits, rango de -2^63 a 2^63-1. Para números muy grandes. Se
sufija con L o l (recomendado L mayúscula).

```java
// Los tipos enteros en Java

class TiposEnteros {
    public static void main(String[] args) {
        // byte: 8 bits (-128 a 127)
        byte peque = 100;
        System.out.println("byte: " + peque);
        // byte error = 200; // Error: fuera de rango
        
        // short: 16 bits (-32,768 a 32,767)
        short mediano = 30000;
        System.out.println("short: " + mediano);
        
        // int: 32 bits (valor por defecto para enteros)
        int normal = 2000000;
        System.out.println("int: " + normal);
        
        // long: 64 bits (sufijo L)
        long grande = 10000000000L;  // sin la L, error de compilación
        System.out.println("long: " + grande);
        
        // Valores máximos y mínimos
        System.out.println("\n=== VALORES LÍMITE ===");
        System.out.println("byte:  " + Byte.MIN_VALUE + " a " + Byte.MAX_VALUE);
        System.out.println("short: " + Short.MIN_VALUE + " a " + Short.MAX_VALUE);
        System.out.println("int:   " + Integer.MIN_VALUE + " a " + Integer.MAX_VALUE);
        System.out.println("long:  " + Long.MIN_VALUE + " a " + Long.MAX_VALUE);
        
        // Literales especiales
        int hex = 0xFF;        // 255 en hexadecimal
        int bin = 0b1101;      // 13 en binario
        int oct = 077;         // 63 en octal
        int conGuiones = 1_000_000; // 1000000 (mejor legibilidad)
        
        System.out.println("\nHexadecimal 0xFF: " + hex);
        System.out.println("Binario 0b1101: " + bin);
        System.out.println("Con guiones 1_000_000: " + conGuiones);
    }
}
```

**Regla práctica:** Usa int para la mayoría de los casos. Usa long solo
cuando sepas que los valores superarán los 2.1 mil millones. Usa byte y
short solo cuando ahorrar memoria sea crítico (arrays muy grandes).

```java
// Comparación de tamaños en memoria

class MemoriaEnteros {
    public static void main(String[] args) {
        System.out.println("=== TAMAÑO EN MEMORIA ===");
        System.out.println("byte:  1 byte (8 bits)");
        System.out.println("short: 2 bytes (16 bits)");
        System.out.println("int:   4 bytes (32 bits)");
        System.out.println("long:  8 bytes (64 bits)");
        System.out.println();
        System.out.println("Ejemplo: Un array de 1 millón de ints");
        System.out.println("usa ~4 MB de memoria.");
        System.out.println("El mismo array de bytes usa solo ~1 MB.");
    }
}
```

3. TIPOS PRIMITIVOS DECIMALES
===============================================================================

Para números con parte fraccionaria, Java ofrece dos tipos:

**float:** 32 bits, precisión simple (~6-7 dígitos decimales). Se sufija
con f o F.

**double:** 64 bits, precisión doble (~15-16 dígitos decimales). Es el
tipo por defecto para números decimales. Se puede sufijar con d o D
(opcional).

```java
// Tipos decimales (punto flotante)

class TiposDecimales {
    public static void main(String[] args) {
        // float: 32 bits, sufijo f
        float precio = 19.99f;
        System.out.println("float: " + precio);
        
        // double: 64 bits (por defecto para decimales)
        double pi = 3.141592653589793;
        System.out.println("double: " + pi);
        
        // Notación científica
        double notacion = 1.5e10;  // 1.5 x 10^10 = 15000000000
        System.out.println("Notación científica: " + notacion);
        
        double pequeno = 1.5e-3;   // 0.0015
        System.out.println("Muy pequeño: " + pequeno);
        
        // Valores especiales
        double infinito = Double.POSITIVE_INFINITY;
        double negativoInfinito = Double.NEGATIVE_INFINITY;
        double noNumero = Double.NaN;
        
        System.out.println("\n=== VALORES ESPECIALES ===");
        System.out.println("Infinito positivo: " + infinito);
        System.out.println("Infinito negativo: " + negativoInfinito);
        System.out.println("NaN (Not a Number): " + noNumero);
        
        // División por cero no lanza excepción (en double/float)
        System.out.println("5.0 / 0.0 = " + (5.0 / 0.0));
        System.out.println("0.0 / 0.0 = " + (0.0 / 0.0));
        
        // Precisión limitada
        System.out.println("\n=== PRECISIÓN LIMITADA ===");
        System.out.println("0.1 + 0.2 = " + (0.1 + 0.2)); // ¡No es exactamente 0.3!
    }
}
```

**ADVERTENCIA:** Los tipos float y double tienen precisión limitada. Para
cálculos financieros o científicos que requieran exactitud, debes usar
BigDecimal (lo veremos en módulos avanzados).

```java
// Problemas de precisión con punto flotante

class PrecisionLimitada {
    public static void main(String[] args) {
        // ¡Esto NO funciona como esperas!
        double resultado = 0.0;
        for (int i = 0; i < 10; i++) {
            resultado += 0.1;
        }
        System.out.println("0.1 + 0.1 + ... + 0.1 (10 veces) = " + resultado);
        System.out.println("¿Es exactamente 1.0? " + (resultado == 1.0));
        
        // Comparación segura
        double margen = 0.0001;
        System.out.println("¿Casi 1.0? " + (Math.abs(resultado - 1.0) < margen));
        
        // Para dinero, usa BigDecimal
        System.out.println("\nPara cálculos exactos usa BigDecimal:");
        java.math.BigDecimal bd = java.math.BigDecimal.valueOf(0.1);
        bd = bd.add(java.math.BigDecimal.valueOf(0.2));
        System.out.println("0.1 + 0.2 con BigDecimal = " + bd);
    }
}
```

4. CHAR Y BOOLEAN
===============================================================================

Completamos los 8 tipos primitivos con char y boolean:

**char:** 16 bits, almacena un único carácter Unicode. Se usa con comillas
simples ('A', 'ñ', '3', '\n'). Puede contener cualquier carácter Unicode.

**boolean:** Solo dos valores: true o false. Se usa para condiciones lógicas.
No es un número (a diferencia de C/C++ donde 0 es false y !=0 es true).

```java
// Los tipos char y boolean

class CharYBoolean {
    public static void main(String[] args) {
        // === CHAR ===
        char letraA = 'A';
        char letraN = 'ñ';
        char digito = '9';
        char espacio = ' ';
        char saltoLinea = '\n';
        char tabulador = '\t';
        
        // Valores Unicode
        char omega = '\u03A9';  // Letra griega Omega mayúscula
        char corazon = '\u2665'; // Símbolo de corazón
        char yen = '\u00A5';     // Símbolo de yen
        
        System.out.println("=== CHAR ===");
        System.out.println("Letra A: " + letraA);
        System.out.println("Letra ñ: " + letraN);
        System.out.println("Omega: " + omega);
        System.out.println("Corazón: " + corazon);
        System.out.println("Yen: " + yen);
        
        // char puede tratarse como número (código Unicode)
        System.out.println("Código Unicode de 'A': " + (int) letraA);
        System.out.println("Código Unicode de 'ñ': " + (int) letraN);
        
        // === BOOLEAN ===
        boolean esMayorEdad = true;
        boolean tienePermiso = false;
        boolean resultado = (10 > 5); // true
        
        System.out.println("\n=== BOOLEAN ===");
        System.out.println("Es mayor de edad: " + esMayorEdad);
        System.out.println("Tiene permiso: " + tienePermiso);
        System.out.println("10 > 5: " + resultado);
        
        // Operaciones lógicas
        boolean y = esMayorEdad && tienePermiso; // false
        boolean o = esMayorEdad || tienePermiso; // true
        boolean no = !esMayorEdad;               // false
        
        System.out.println("Y lógico: " + y);
        System.out.println("O lógico: " + o);
        System.out.println("Negación: " + no);
    }
}
```

5. INFERENCIA DE TIPOS CON VAR
===============================================================================

Desde Java 10, puedes usar la palabra clave 'var' para declarar variables
locales. El compilador infiere el tipo basándose en el valor asignado:

```java
// Inferencia de tipos con var

class InferenciaVar {
    public static void main(String[] args) {
        // var: el compilador deduce el tipo
        var nombre = "Ana";         // String
        var edad = 25;              // int
        var precio = 19.99;         // double
        var activo = true;          // boolean
        var letra = 'A';            // char
        var saldo = 1000L;          // long (por la L)
        var peso = 65.5f;           // float (por la f)
        
        System.out.println("=== INFERENCIA CON VAR ===");
        System.out.println("nombre es de tipo: " + ((Object) nombre).getClass().getName());
        System.out.println("edad es de tipo: " + ((Object) edad).getClass().getName());
        
        // var con expresiones
        var suma = 10 + 20;         // int
        var concatenacion = "Hola " + "Mundo"; // String
        var mayor = (10 > 5);       // boolean
        
        System.out.println("suma: " + suma);
        System.out.println("concatenacion: " + concatenacion);
        System.out.println("mayor: " + mayor);
        
        // var no se puede usar sin inicializar
        // var x; // Error: cannot infer type for local variable
        
        // var no se puede usar para campos de clase o parámetros
        // (solo para variables locales)
    }
}
```

**Limitaciones de var:**

1. Solo para variables locales (no campos de clase, ni parámetros, ni
   tipos de retorno).
2. Debe inicializarse en la misma línea (no puedes declarar var x;).
3. No se puede asignar null (a menos que haya un casteo).
4. No se puede usar en declaraciones múltiples: var a = 1, b = 2; // Error.

```java
// Cuándo usar var y cuándo no

class UsoVar {
    public static void main(String[] args) {
        // ✅ Buen uso: el tipo es obvio por el lado derecho
        var nombre = "Ana";
        var lista = new java.util.ArrayList<String>();
        
        // ✅ Buen uso: tipos largos y complejos
        var mapa = new java.util.HashMap<String, java.util.List<Integer>>();
        
        // ❌ Mal uso: pérdida de legibilidad
        // var resultado = metodoComplejo();  // ¿Qué tipo devuelve?
        // Mejor: TipoResultado resultado = metodoComplejo();
        
        System.out.println("Usa var cuando mejore la legibilidad,");
        System.out.println("no cuando la empeore.");
    }
}
```

**Recomendación:** Usa var cuando el tipo sea obvio por el contexto
(new ArrayList<>(), "texto", 42) y cuando mejore la legibilidad. Evítalo
cuando el tipo no sea evidente.

6. CASTING: CONVERSIÓN DE TIPOS
===============================================================================

El casting (o moldeado) es la conversión de un tipo de dato a otro. Java
distingue dos tipos de casting:

**Casting implícito (widening):** Convierte automáticamente de un tipo
menor a uno mayor. No hay pérdida de datos.

**Casting explícito (narrowing):** Convierte manualmente de un tipo mayor
a uno menor. PUEDE HABER pérdida de datos. Debes hacerlo explícitamente
con (tipo).

```java
// Casting implícito y explícito

class CastingTipos {
    public static void main(String[] args) {
        System.out.println("=== CASTING IMPLÍCITO (WIDENING) ===");
        // De menor a mayor: no requiere sintaxis especial
        byte b = 100;
        short s = b;       // byte -> short (implícito)
        int i = s;         // short -> int (implícito)
        long l = i;        // int -> long (implícito)
        float f = l;       // long -> float (implícito, posible pérdida)
        double d = f;      // float -> double (implícito)
        
        System.out.println("byte " + b + " -> short " + s);
        System.out.println("short " + s + " -> int " + i);
        System.out.println("int " + i + " -> long " + l);
        
        // char a int (implícito)
        char letra = 'A';
        int codigo = letra;  // char -> int
        System.out.println("char 'A' -> int " + codigo);
        
        System.out.println("\n=== CASTING EXPLÍCITO (NARROWING) ===");
        // De mayor a menor: requiere (tipo)
        double pi = 3.14159;
        int entero = (int) pi;  // double -> int (pierde decimales)
        System.out.println("double " + pi + " -> int " + entero);
        
        // int a byte (puede desbordarse)
        int grande = 200;
        byte pequeno = (byte) grande;
        System.out.println("int " + grande + " -> byte " + pequeno + " (¡desbordado!)");
        
        // long a int
        long largo = 123456789L;
        int normal = (int) largo;
        System.out.println("long " + largo + " -> int " + normal);
        
        // char desde código numérico
        char letraDesdeCodigo = (char) 65;
        System.out.println("(char)65 -> " + letraDesdeCodigo);
    }
}
```

**Jerarquía de tipos para casting implícito:**
```
byte -> short -> int -> long -> float -> double
                 ^
                 |
                char
```

El único caso especial: char puede convertirse implícitamente a int (y
tipos mayores), pero no al revés sin casting explícito.

```java
// Casting con pérdida de datos

class PerdidaDatosCasting {
    public static void main(String[] args) {
        System.out.println("=== PÉRDIDA DE DATOS EN CASTING ===");
        
        // Caso 1: Desbordamiento (overflow)
        int grande = 300;
        byte pequeno = (byte) grande;
        // 300 - 256 = 44 (byte solo llega a 127, da la vuelta)
        System.out.println(grande + " -> byte: " + pequeno);
        
        // Caso 2: Pérdida de decimales (truncamiento)
        double decimal = 9.99;
        int entero = (int) decimal;
        System.out.println(decimal + " -> int: " + entero);
        
        // Caso 3: Pérdida de precisión (long -> float)
        long largo = 123456789123456789L;
        float flotante = largo;
        System.out.println(largo + " -> float: " + flotante + " (precisión perdida)");
        
        System.out.println("\n⚠️ Siempre que hagas narrowing casting,");
        System.out.println("   verifica que el valor quepa en el destino.");
    }
}
```

7. WRAPPERS: LA CONTRAPARTE OBJETO DE LOS PRIMITIVOS
===============================================================================

Además de los 8 tipos primitivos, Java ofrece clases "wrapper" que
envuelven a cada primitivo en un objeto. Se usan cuando necesitas tratar
un primitivo como objeto (por ejemplo, en colecciones).

| Primitivo | Wrapper |
|-----------|---------|
| byte | Byte |
| short | Short |
| int | Integer |
| long | Long |
| float | Float |
| double | Double |
| char | Character |
| boolean | Boolean |

```java
// Clases wrapper de los tipos primitivos

class Wrappers {
    public static void main(String[] args) {
        System.out.println("=== CLASES WRAPPER ===");
        
        // Autoboxing: primitivo -> wrapper (automático desde Java 5)
        int primitivo = 42;
        Integer wrapper = primitivo;  // autoboxing
        System.out.println("Autoboxing: " + wrapper);
        
        // Unboxing: wrapper -> primitivo (automático)
        Integer obj = Integer.valueOf(100);
        int valor = obj;  // unboxing
        System.out.println("Unboxing: " + valor);
        
        // Métodos útiles de los wrappers
        System.out.println("\n=== MÉTODOS ÚTILES ===");
        System.out.println("Integer.parseInt('123'): " + Integer.parseInt("123"));
        System.out.println("Double.parseDouble('3.14'): " + Double.parseDouble("3.14"));
        System.out.println("Boolean.parseBoolean('true'): " + Boolean.parseBoolean("true"));
        
        System.out.println("Integer.MAX_VALUE: " + Integer.MAX_VALUE);
        System.out.println("Double.MIN_VALUE: " + Double.MIN_VALUE);
        
        // Conversión de entero a binario/hexadecimal
        System.out.println("42 en binario: " + Integer.toBinaryString(42));
        System.out.println("255 en hexadecimal: " + Integer.toHexString(255));
        
        // Comparación de wrappers
        Integer a = 127;
        Integer b = 127;
        Integer c = 200;
        Integer d = 200;
        
        System.out.println("\n=== COMPARACIÓN DE WRAPPERS ===");
        System.out.println("127 == 127: " + (a == b));    // true (cacheado)
        System.out.println("200 == 200: " + (c == d));    // false (NO cacheado)
        System.out.println("Usa .equals() para comparar wrappers:");
        System.out.println("200.equals(200): " + c.equals(d)); // true
    }
}
```

**Nota importante:** Los valores Integer entre -128 y 127 se cachean,
por lo que == funciona. Para valores fuera de ese rango, usa .equals().

RESUMEN DEL CAPÍTULO
===============================================================================

- Una variable es un espacio de memoria con un nombre y un tipo.
- Java tiene 8 tipos primitivos: byte, short, int, long, float, double, char, boolean.
- int es el tipo entero por defecto; double es el tipo decimal por defecto.
- Usa long con sufijo L, float con sufijo f.
- char almacena un carácter Unicode; boolean solo true o false.
- var (Java 10+) permite inferencia de tipos para variables locales.
- Casting implícito (widening) convierte de menor a mayor sin pérdida.
- Casting explícito (narrowing) requiere (tipo) y puede perder datos.
- Los wrappers son la versión objeto de los primitivos.
- Para cálculos exactos con decimales usa BigDecimal, no double/float.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre 5 y 5.0 en Java?**
5 es un int; 5.0 es un double (por defecto). Si haces 5/2 obtienes 2
(división entera), pero 5.0/2 obtienes 2.5.

❓ **¿Por qué 0.1 + 0.2 no es exactamente 0.3?**
Por la representación binaria de números de punto flotante. Algunos
números decimales no pueden representarse exactamente en binario. Usa
BigDecimal para precisión exacta.

❓ **¿Puedo usar var para declarar una constante?**
Sí: final var MAX = 100; funciona. El compilador infiere el tipo y final
impide la reasignación.

❓ **¿Qué pasa si divido un entero por 0?**
Lanza ArithmeticException: / by zero. Con double/float retorna Infinity
o NaN.

❓ **¿Cuándo debo usar float en lugar de double?**
Casi nunca. double tiene más precisión y es el estándar. Usa float solo
cuando ahorrar memoria sea crítico (arrays muy grandes de decimales).

🏗️ **Calculadora de Tipos de Datos**

Diseña un programa Java que:

1. Declare variables de los 8 tipos primitivos con valores iniciales.
2. Realice todas las conversiones de widening posibles (byte->short->
   int->long->float->double) y muestre cada paso.
3. Realice al menos 3 conversiones de narrowing y muestre si hubo pérdida.
4. Use var para declarar al menos 5 variables de tipos diferentes.
5. Muestre el tamaño en bytes de cada tipo primitivo usando los wrappers
   (Byte.BYTES, Integer.BYTES, etc.).
6. Incluya un ejemplo de autoboxing y unboxing.
7. Use parseInt() para convertir un String a int y muestre el resultado.

Ejemplo de salida:
```
=== CALCULADORA DE TIPOS ===

1. Declaración de primitivos:
   byte: 100
   short: 30000
   int: 2000000
   ...

2. Widening casting (implícito):
   byte 100 -> short 100 -> int 100 -> long 100 -> float 100.0 -> double 100.0

3. Narrowing casting (explícito):
   double 3.14159 -> int 3 (pérdida: 0.14159)

4. Variables con var:
   nombre = "Ana" (String)
   edad = 25 (int)
   ...

5. Tamaño de tipos:
   byte: 1 byte, short: 2 bytes, int: 4 bytes, ...

6. Autoboxing: int 42 -> Integer 42
   Unboxing: Integer 100 -> int 100

7. String "123" -> int 123
```

Este desafío integra todos los conceptos de tipos de datos, variables,
casting y wrappers en un solo programa.
