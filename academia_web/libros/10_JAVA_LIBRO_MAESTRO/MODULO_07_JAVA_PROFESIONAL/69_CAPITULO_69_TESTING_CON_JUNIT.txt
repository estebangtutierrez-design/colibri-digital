1. ¿POR QUÉ SON IMPORTANTES LOS TESTS?
===============================================================================

Las pruebas unitarias son el primer nivel de defensa contra bugs.

```java
// Sin tests: código sin verificación
public class Calculadora {
    public int sumar(int a, int b) {
        return a + b;
    }

    public int dividir(int a, int b) {
        return a / b; // ¿Qué pasa si b es 0?
    }
}

// ¿Cómo sabemos que funciona? Solo por "confianza"
// El testing cambia eso: cada método tiene verificación automática
```

Beneficios de las pruebas unitarias:
1. Detección temprana de errores: encuentras bugs antes de llegar a QA.
2. Documentación viva: los tests muestran cómo debe usarse el código.
3. Refactorización segura: puedes cambiar código sabiendo que los tests
   detectarán si algo se rompe.
4. Diseño mejorado: escribir tests te obliga a crear código más modular.
5. Confianza: despliegas con la certeza de que las funcionalidades clave
   funcionan.

```xml

<dependencies>

        <artifactId>junit-jupiter</artifactId>
```

```gradle
// Gradle: build.gradle
dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.2'

    useJUnitPlatform()
}
```

Estructura de proyecto recomendada:
```
src/
  main/java/     -> Código de producción
  test/java/     -> Tests (mismo package que el código)
```

```java
// Código a probar
class Calculadora {
    public int sumar(int a, int b) {
        return a + b;
    }

    public int restar(int a, int b) {
        return a - b;
    }

    public int multiplicar(int a, int b) {
        return a * b;
    }

    public int dividir(int a, int b) {
        if (b == 0) {
            throw new IllegalArgumentException("No se puede dividir por cero");
        }
        return a / b;
    }
}

// Tests (usando JUnit 5)
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculadoraTest {

    private final Calculadora calc = new Calculadora();

    @Test
    void testSumar() {
        assertEquals(5, calc.sumar(2, 3));
        assertEquals(0, calc.sumar(-1, 1));
        assertEquals(-5, calc.sumar(-2, -3));
    }

    @Test
    void testRestar() {
        assertEquals(3, calc.restar(5, 2));
        assertEquals(-3, calc.restar(2, 5));
    }

    @Test
    void testMultiplicar() {
        assertEquals(6, calc.multiplicar(2, 3));
        assertEquals(0, calc.multiplicar(0, 5));
        assertEquals(-6, calc.multiplicar(-2, 3));
    }

    @Test
    void testDividir() {
        assertEquals(2, calc.dividir(6, 3));
        assertEquals(0, calc.dividir(1, 3)); // División entera
    }

    @Test
    void testDividirPorCero() {
        assertThrows(IllegalArgumentException.class,
            () -> calc.dividir(5, 0));
    }
}
```

4. ASERCIONES EN JUNIT 5
===============================================================================

Las aserciones verifican que el resultado es el esperado.

```java
import org.junit.jupiter.api.Test;
import java.util.*;
import static org.junit.jupiter.api.Assertions.*;

class AsercionesDemo {

    @Test
    void asercionesBasicas() {
        // assertEquals: verifica igualdad
        assertEquals(4, 2 + 2);
        assertEquals("Hola", "Hola");
        // Con mensaje personalizado
        assertEquals(4, 2 + 2, "La suma debería ser 4");
        // Con supplier de mensaje (evaluación perezosa)
        assertEquals(4, 2 + 2, () -> "Error en la suma");

        // assertNotEquals: verifica que NO sean iguales
        assertNotEquals(5, 2 + 2);

        // assertTrue / assertFalse
        assertTrue(5 > 3);
        assertFalse(5 < 3);
        assertTrue("Java".startsWith("J"), "Debería empezar con J");
    }

    @Test
    void asercionesObjetos() {
        // assertNull / assertNotNull
        assertNull(null);
        assertNotNull("Objeto");
        assertNotNull(new ArrayList<>(), "La lista no debería ser nula");

        // assertSame / assertNotSame (por referencia)
        String s1 = "Hola";
        String s2 = "Hola";
        String s3 = new String("Hola");
        assertSame(s1, s2);     // misma referencia (interned)
        assertNotSame(s1, s3);  // diferente referencia

        // assertInstanceOf
        assertInstanceOf(List.class, new ArrayList<>());
        assertInstanceOf(Collection.class, new ArrayList<>());
    }

    @Test
    void asercionesArrays() {
        // assertArrayEquals
        int[] esperado = {1, 2, 3};
        int[] actual = {1, 2, 3};
        assertArrayEquals(esperado, actual);

        String[] expected = {"a", "b"};
        String[] actual2 = {"a", "b"};
        assertArrayEquals(expected, actual2);
    }

    @Test
    void asercionesIterables() {
        List<String> lista = Arrays.asList("a", "b", "c");
        assertIterableEquals(Arrays.asList("a", "b", "c"), lista);
    }

    @Test
    void asercionesExcepciones() {
        // assertThrows: verifica que se lanza una excepción
        Exception exception = assertThrows(
            ArithmeticException.class,
            () -> { int x = 1 / 0; }
        );
        assertEquals("/ by zero", exception.getMessage());

        // assertDoesNotThrow: verifica que NO se lanza excepción
        assertDoesNotThrow(() -> { int x = 1 / 1; });
    }

    @Test
    void asercionesTiempo() {
        // assertTimeout: verifica que complete dentro del tiempo
        assertTimeout(java.time.Duration.ofMillis(100), () -> {
            Thread.sleep(50);
        });

        // assertTimeoutPreemptively: aborta si excede el tiempo
        assertTimeoutPreemptively(java.time.Duration.ofMillis(200), () -> {
            Thread.sleep(100);
        });
    }

    // fail(): falla el test intencionalmente
    @Test
    void testFallidoIntencional() {
        // fail("Este test siempre falla - implementación pendiente");
    }
}
```

5. CICLO DE VIDA: @BEFOREEACH, @AFTEREACH, @BEFOREALL, @AFTERALL
===============================================================================

```java
import org.junit.jupiter.api.*;

class CicloVidaTest {

    @BeforeAll
    static void initAll() {
        System.out.println("@BeforeAll: Se ejecuta UNA VEZ antes de todos los tests");
        // Útil para: inicializar recursos costosos (conexiones BD, pools)
    }

    @BeforeEach
    void init() {
        System.out.println("  @BeforeEach: Se ejecuta ANTES de cada test");
        // Útil para: crear objetos, resetear estado
    }

    @Test
    void testUno() {
        System.out.println("    Test 1 ejecutándose");
        assertTrue(true);
    }

    @Test
    void testDos() {
        System.out.println("    Test 2 ejecutándose");
        assertTrue(true);
    }

    @AfterEach
    void tearDown() {
        System.out.println("  @AfterEach: Se ejecuta DESPUÉS de cada test");
        // Útil para: limpiar recursos, cerrar archivos
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("@AfterAll: Se ejecuta UNA VEZ después de todos los tests");
        // Útil para: cerrar conexiones, liberar recursos globales
    }
}
```

Orden de ejecución:
```
@BeforeAll
  @BeforeEach
    @Test metodo1
  @AfterEach
  @BeforeEach
    @Test metodo2
  @AfterEach
@AfterAll
```

6. TESTS PARAMETRIZADOS
===============================================================================

Los tests parametrizados ejecutan el mismo test con diferentes datos.

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.*;
import static org.junit.jupiter.api.Assertions.*;

class ParametrizadosDemo {

    // @ValueSource: valores simples
    @ParameterizedTest
    @ValueSource(strings = {"", "  ", "\t", "\n"})
    void testStringsVacios(String input) {
        assertTrue(input.trim().isEmpty());
    }

    @ParameterizedTest
    @ValueSource(ints = {2, 4, 6, 8, 10})
    void testNumerosPares(int numero) {
        assertEquals(0, numero % 2);
    }

    // @CsvSource: múltiples parámetros
    @ParameterizedTest
    @CsvSource({
        "1,    1,    2",
        "-1,   1,    0",
        "0,    0,    0",
        "100,  200,  300"
    })
    void testSumarParametrizado(int a, int b, int esperado) {
        Calculadora calc = new Calculadora();
        assertEquals(esperado, calc.sumar(a, b));
    }

    // @CsvFileSource: desde archivo CSV externo
    @ParameterizedTest
    @CsvFileSource(resources = "/datos-suma.csv", numLinesToSkip = 1)
    void testSumarDesdeArchivo(int a, int b, int esperado) {
        Calculadora calc = new Calculadora();
        assertEquals(esperado, calc.sumar(a, b));
    }

    // @MethodSource: desde un método que genera datos
    @ParameterizedTest
    @MethodSource("proveerNumeros")
    void testConMethodSource(int numero, boolean esperado) {
        assertEquals(esperado, numero > 0);
    }

    static java.util.stream.Stream<Arguments> proveerNumeros() {
        return java.util.stream.Stream.of(
            Arguments.of(5, true),
            Arguments.of(-3, false),
            Arguments.of(0, false)
        );
    }

    // @EnumSource: valores de un enum
    enum Dia { LUNES, MARTES, MIERCOLES, JUEVES, VIERNES, SABADO, DOMINGO }

    @ParameterizedTest
    @EnumSource(value = Dia.class, names = {"SABADO", "DOMINGO"})
    void testFinde(Dia dia) {
        assertTrue(dia == Dia.SABADO || dia == Dia.DOMINGO);
    }
}
```

7. @NESTED: TESTS ANIDADOS
===============================================================================

@Nested agrupa tests relacionados dentro de una clase externa.

```java
import org.junit.jupiter.api.*;

class StackTest {

    private java.util.Stack<String> stack;

    @BeforeEach
    void init() {
        stack = new java.util.Stack<>();
    }

    @Test
    void testStackVacia() {
        assertTrue(stack.isEmpty());
    }

    @Nested
    class CuandoHayElementos {

        @BeforeEach
        void agregarElementos() {
            stack.push("A");
            stack.push("B");
        }

        @Test
        void testNoVacia() {
            assertFalse(stack.isEmpty());
            assertEquals(2, stack.size());
        }

        @Test
        void testPop() {
            assertEquals("B", stack.pop());
            assertEquals(1, stack.size());
        }

        @Test
        void testPeek() {
            assertEquals("B", stack.peek());
            assertEquals(2, stack.size()); // peek no elimina
        }

        @Nested
        class CuandoStackEstaLlena {

            @BeforeEach
            void llenarStack() {
                for (int i = 0; i < 8; i++) {
                    stack.push("X");
                }
            }

            @Test
            void testTamanoMaximo() {
                // Cualquier test que requiera stack lleno
                assertTrue(stack.size() > 5);
            }
        }
    }

    @Nested
    class CuandoEstaVacia {

        @Test
        void testPopLanzaExcepcion() {
            assertThrows(java.util.EmptyStackException.class,
                () -> stack.pop());
        }
    }
}
```

9. TDD (TEST DRIVEN DEVELOPMENT)
===============================================================================

TDD es una metodología donde primero escribes el test y luego el código.

Ciclo TDD: Rojo -> Verde -> Refactorizar

1. ROJO: Escribe un test que falla (la funcionalidad no existe).
2. VERDE: Escribe el código mínimo para que el test pase.
3. REFACTORIZAR: Mejora el código manteniendo los tests verdes.

```java
// Ejemplo TDD: Calculadora de descuentos

// PASO 1: Escribir el test PRIMERO (Rojo)
import org.junit.jupiter.api.*;

class CalculadoraDescuentosTest {

    private CalculadoraDescuentos calc;

    @BeforeEach
    void init() {
        calc = new CalculadoraDescuentos();
    }

    // Primer test: descuento básico
    @Test
    void testDescuentoMenor100() {
        assertEquals(0.0, calc.calcularDescuento(50.0));
    }

    @Test
    void testDescuentoEntre100y500() {
        // 10% de descuento
        assertEquals(10.0, calc.calcularDescuento(100.0));
        assertEquals(50.0, calc.calcularDescuento(500.0));
    }

    @Test
    void testDescuentoMayor500() {
        // 20% de descuento
        assertEquals(100.0, calc.calcularDescuento(500.01));
        assertEquals(200.0, calc.calcularDescuento(1000.0));
    }

    @Test
    void testDescuentoMaximo() {
        // Máximo descuento: $500
        assertEquals(500.0, calc.calcularDescuento(5000.0));
        assertEquals(500.0, calc.calcularDescuento(10000.0));
    }

    @Test
    void testMontoInvalido() {
        assertThrows(IllegalArgumentException.class,
            () -> calc.calcularDescuento(-10));
    }
}

// PASO 2: Implementar el código para que los tests pasen (Verde)
class CalculadoraDescuentos {

    public double calcularDescuento(double monto) {
        if (monto < 0) {
            throw new IllegalArgumentException(
                "El monto no puede ser negativo");
        }

        double descuento;
        if (monto < 100) {
            descuento = 0;
        } else if (monto <= 500) {
            descuento = monto * 0.10;
        } else {
            descuento = monto * 0.20;
        }

        return Math.min(descuento, 500.0); // Descuento máximo $500
    }
}

// PASO 3: Refactorizar (mejorar código, tests deben seguir pasando)
// El código ya está limpio, pero podríamos extraer constantes
```

10. MOCKITO: SIMULAR DEPENDENCIAS
===============================================================================

En aplicaciones reales, las clases dependen de otras (BD, APIs, archivos).
Mockito permite crear objetos simulados (mocks) para aislar el código.

```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```

```java
import org.junit.jupiter.api.*;
import org.mockito.*;
import static org.mockito.Mockito.*;
import static org.junit.jupiter.api.Assertions.*;

// Dependencias
interface RepositorioUsuario {
    Usuario buscarPorId(int id);
    void guardar(Usuario u);
}

class Usuario {
    private int id;
    private String nombre;
    private String email;

    public Usuario(int id, String nombre, String email) {
        this.id = id;
        this.nombre = nombre;
        this.email = email;
    }
    public int getId() { return id; }
    public String getNombre() { return nombre; }
    public String getEmail() { return email; }
}

class ServicioNotificacion {
    public boolean enviarEmail(String to, String subject, String body) {
        // Enviar email real (simulado)
        return true;
    }
}

// Clase a probar
class ServicioUsuario {
    private final RepositorioUsuario repositorio;
    private final ServicioNotificacion notificacion;

    public ServicioUsuario(RepositorioUsuario repo, ServicioNotificacion notif) {
        this.repositorio = repo;
        this.notificacion = notif;
    }

    public Usuario registrarUsuario(String nombre, String email) {
        Usuario u = new Usuario(0, nombre, email);
        repositorio.guardar(u);
        notificacion.enviarEmail(email, "Bienvenido",
            "Hola " + nombre + ", bienvenido!");
        return u;
    }

    public String obtenerNombreUsuario(int id) {
        Usuario u = repositorio.buscarPorId(id);
        return u != null ? u.getNombre() : "Usuario no encontrado";
    }
}

// Tests con Mockito
class ServicioUsuarioTest {

    @Mock
    private RepositorioUsuario repositorioMock;

    @Mock
    private ServicioNotificacion notificacionMock;

    @InjectMocks
    private ServicioUsuario servicio;

    @BeforeEach
    void init() {
        MockitoAnnotations.openMocks(this);
    }

    @Test
    void testRegistrarUsuario() {
        // Configurar comportamiento del mock
        Usuario esperado = new Usuario(1, "Ana", "ana@email.com");

        // Simular que guardar() funciona
        doNothing().when(repositorioMock).guardar(any(Usuario.class));
        when(notificacionMock.enviarEmail(
            anyString(), anyString(), anyString()))
            .thenReturn(true);

        // Ejecutar
        Usuario resultado = servicio.registrarUsuario("Ana", "ana@email.com");

        // Verificar
        assertNotNull(resultado);
        assertEquals("Ana", resultado.getNombre());

        // Verificar que se llamaron los métodos esperados
        verify(repositorioMock).guardar(any(Usuario.class));
        verify(notificacionMock).enviarEmail(
            eq("ana@email.com"), anyString(), anyString());
    }

    @Test
    void testObtenerNombreUsuarioExistente() {
        // Simular repositorio
        when(repositorioMock.buscarPorId(1))
            .thenReturn(new Usuario(1, "Carlos", "carlos@email.com"));

        String nombre = servicio.obtenerNombreUsuario(1);
        assertEquals("Carlos", nombre);

        verify(repositorioMock).buscarPorId(1);
    }

    @Test
    void testObtenerNombreUsuarioNoExistente() {
        when(repositorioMock.buscarPorId(99)).thenReturn(null);

        String nombre = servicio.obtenerNombreUsuario(99);
        assertEquals("Usuario no encontrado", nombre);
    }
}
```

Anotaciones clave de Mockito:
- @Mock: Crea un mock de la clase/interfaz.
- @InjectMocks: Crea la instancia real e inyecta los mocks.
- @Spy: Crea un spy (mock parcial, usa el objeto real).
- @Captor: Captura argumentos para verificación avanzada.

11. EJEMPLO PRÁCTICO: TEST DE UN REPOSITORIO
===============================================================================

```java
// Código de producción
class Producto {
    private int id;
    private String nombre;
    private double precio;

    public Producto(int id, String nombre, double precio) {
        this.id = id;
        this.nombre = nombre;
        this.precio = precio;
    }

    // Getters
    public int getId() { return id; }
    public String getNombre() { return nombre; }
    public double getPrecio() { return precio; }
}

interface ProductoRepository {
    Producto findById(int id);
    List<Producto> findAll();
    Producto save(Producto p);
    void deleteById(int id);
}

class ProductoService {
    private final ProductoRepository repo;

    public ProductoService(ProductoRepository repo) {
        this.repo = repo;
    }

    public double calcularPrecioPromedio() {
        List<Producto> productos = repo.findAll();
        if (productos.isEmpty()) return 0.0;
        return productos.stream()
            .mapToDouble(Producto::getPrecio)
            .average()
            .orElse(0.0);
    }

    public Producto aplicarDescuento(int productoId, double porcentaje) {
        Producto p = repo.findById(productoId);
        if (p == null) {
            throw new IllegalArgumentException("Producto no encontrado");
        }
        double nuevoPrecio = p.getPrecio() * (1 - porcentaje / 100);
        return repo.save(new Producto(p.getId(), p.getNombre(), nuevoPrecio));
    }
}

// Tests
import org.junit.jupiter.api.*;
import org.mockito.*;
import static org.mockito.Mockito.*;
import static org.junit.jupiter.api.Assertions.*;
import java.util.*;

class ProductoServiceTest {

    @Mock
    private ProductoRepository repoMock;

    @InjectMocks
    private ProductoService service;

    @BeforeEach
    void init() {
        MockitoAnnotations.openMocks(this);
    }

    @Test
    void testPrecioPromedioConProductos() {
        List<Producto> productos = Arrays.asList(
            new Producto(1, "A", 100),
            new Producto(2, "B", 200),
            new Producto(3, "C", 300)
        );
        when(repoMock.findAll()).thenReturn(productos);

        double promedio = service.calcularPrecioPromedio();
        assertEquals(200.0, promedio, 0.001);
    }

    @Test
    void testPrecioPromedioSinProductos() {
        when(repoMock.findAll()).thenReturn(Collections.emptyList());
        assertEquals(0.0, service.calcularPrecioPromedio());
    }

    @Test
    void testAplicarDescuento() {
        Producto original = new Producto(1, "Laptop", 1000);
        when(repoMock.findById(1)).thenReturn(original);
        when(repoMock.save(any())).thenAnswer(inv -> inv.getArgument(0));

        Producto resultado = service.aplicarDescuento(1, 10);

        assertEquals(900.0, resultado.getPrecio(), 0.001);
        assertEquals("Laptop", resultado.getNombre());

        verify(repoMock).findById(1);
        verify(repoMock).save(any(Producto.class));
    }

    @Test
    void testAplicarDescuentoProductoInexistente() {
        when(repoMock.findById(99)).thenReturn(null);

        assertThrows(IllegalArgumentException.class,
            () -> service.aplicarDescuento(99, 10));
    }
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

En este capítulo hemos aprendido:

• Las pruebas unitarias son esenciales para el desarrollo profesional.
  Garantizan que el código funciona y facilitan la refactorización.

• JUnit 5 es el framework de testing estándar. Usa anotaciones como @Test,
  @BeforeEach, @AfterEach, @BeforeAll, @AfterAll.

• Las aserciones (assertEquals, assertTrue, assertThrows, etc.) verifican
  que los resultados sean los esperados.

• @ParameterizedTest ejecuta el mismo test con diferentes datos usando
  @ValueSource, @CsvSource, @MethodSource, etc.

• @Nested agrupa tests relacionados jerárquicamente.

• @Tag permite filtrar qué tests ejecutar (rápidos, lentos, etc.).

• TDD (Test Driven Development) invierte el orden: primero el test, luego
  el código. Ciclo: Rojo -> Verde -> Refactorizar.

• Mockito permite simular dependencias para aislar el código bajo prueba.

===============================================================================
PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Qué porcentaje de cobertura debo tener?**
No hay un número mágico. El objetivo no es "100% de cobertura" sino
"pruebas significativas". Enfócate en probar la lógica de negocio
crítica y los casos borde.

❓ **¿Los tests deben ser rápidos?**
Sí. Los tests unitarios deben ejecutarse en milisegundos. Si un test
tarda segundos, probablemente es un test de integración, no unitario.

❓ **¿Qué diferencia hay entre unitario y de integración?**
Unitario: prueba una unidad aislada (con mocks). Integración: prueba
varios componentes juntos (con BD real, API real).

❓ **¿Debo probar métodos privados?**
No directamente. Prueba la funcionalidad a través de métodos públicos.
Si un método privado es muy complejo, considera extraerlo a una clase
separada.

❓ **¿Cada test debe ser independiente?**
Sí. Los tests no deben depender unos de otros ni de un orden específico.
@BeforeEach garantiza un estado limpio para cada test.

❓ **¿Cómo nombro mis tests?**
En JUnit 5, los nombres de métodos pueden ser en español con @DisplayName.
Ej: @DisplayName("Debe lanzar excepción cuando el saldo es insuficiente").

🏗️ **Sistema de Testing Automatizado para un E-commerce**

Diseña e implementa un conjunto completo de tests para un sistema de
carrito de compras de una tienda online.

Clases a probar:
1. Carrito: agregar, eliminar, actualizar cantidad, calcular total.
2. CalculadorImpuestos: calcula impuestos según país y tipo de producto.
3. ProcesadorPagos: valida tarjeta, procesa pago, maneja rechazos.
4. GestorEnvios: calcula costo de envío, tiempo estimado, seguimiento.
5. ServicioPedidos: orquesta el flujo completo de compra.

Requisitos:
1. Tests unitarios con JUnit 5 para cada clase.
2. Tests parametrizados para variaciones de precio, cantidad, país.
3. Mockito para simular dependencias (ProcesadorPagos real no debe usarse).
4. Tests anidados con @Nested para diferentes escenarios.
5. Tags para clasificar tests (@Tag("rapidos"), @Tag("criticos")).
6. Cobertura mínima del 90% en lógica de negocio.

Implementa:
- Todas las clases de producción (simplificadas pero funcionales).
- Tests completos con casos normales, bordes y error.
- Documentación de los tests con @DisplayName descriptivo.
- Un reporte de cobertura (simulado o real si usas herramientas).

Este desafío integra todo lo aprendido sobre testing y simula un escenario
real de aseguramiento de calidad en un proyecto profesional.
