1. ¿QUÉ ES JDBC?
===============================================================================

JDBC (Java Database Connectivity) es una API de Java que permite a las
aplicaciones Java interactuar con bases de datos relacionales. Fue
introducida en JDK 1.1 y sigue siendo la base de la mayoría de frameworks
de persistencia como Hibernate, JPA y MyBatis.

```java
// La esencia de JDBC: Connection, Statement, ResultSet
// Este es un vistazo general, lo veremos en detalle más adelante

import java.sql.*;

public class ConceptoJDBC {
    public static void main(String[] args) {
        // 1. Driver (gestionado automáticamente desde JDBC 4.0)
        // 2. Connection
        // 3. Statement
        // 4. ResultSet
        System.out.println("JDBC: Java Database Connectivity");
        System.out.println("API estándar para acceso a BD desde Java");
        System.out.println("Independiente del proveedor de BD");
    }
}
```

Arquitectura JDBC:
- DriverManager: Gestiona los drivers registrados.
- Driver: Implementación específica para cada BD (MySQL, PostgreSQL, etc.).
- Connection: Representa una conexión activa con la BD.
- Statement: Objeto para ejecutar sentencias SQL.
- ResultSet: Conjunto de resultados de una consulta.

===============================================================================
2. TIPOS DE DRIVERS JDBC
===============================================================================

Existen 4 tipos de drivers JDBC:

Tipo 1: JDBC-ODBC Bridge (obsoleto/eliminado desde Java 8).
Tipo 2: Driver nativo que usa bibliotecas del cliente de la BD.
Tipo 3: Driver de red que traduce las llamadas JDBC a un protocolo propio.
Tipo 4: Driver puro Java que traduce JDBC al protocolo nativo de red de la BD.

Hoy en día, el Tipo 4 es el estándar. Ejemplos:
- MySQL: com.mysql.cj.jdbc.Driver
- PostgreSQL: org.postgresql.Driver
- Oracle: oracle.jdbc.OracleDriver
- H2 (embebida): org.h2.Driver

Para usar un driver, debes agregar el JAR correspondiente al classpath.
Con Maven/Gradle, agregas la dependencia:

```xml
<!-- Ejemplo: dependencia Maven para MySQL -->
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <version>8.0.33</version>
</dependency>
```

```java
// Verificar qué drivers están disponibles
import java.sql.*;

public class DriversDisponibles {

        System.out.println("Drivers JDBC registrados:");

            System.out.println("  - " + d.getClass().getName() +
                " v" + d.getMajorVersion() + "." + d.getMinorVersion());
        }
    }
}
```

3. CONEXIÓN A LA BASE DE DATOS
===============================================================================

Para conectarte a una BD necesitas:
1. La URL de conexión (formato específico según la BD).
2. El usuario y contraseña.
3. El driver en el classpath.

```java
import java.sql.*;

public class ConexionBasica {
    public static void main(String[] args) {
        // URL de conexión para diferentes bases de datos:
        // MySQL:   jdbc:mysql://localhost:3306/mi_bd
        // PostgreSQL: jdbc:postgresql://localhost:5432/mi_bd
        // H2:      jdbc:h2:~/mi_bd
        // Oracle:  jdbc:oracle:thin:@localhost:1521:XE

        String url = "jdbc:mysql://localhost:3306/java_master";
        String user = "root";
        String password = "password";

        Connection conexion = null;

        try {
            // Establecer conexión
            conexion = DriverManager.getConnection(url, user, password);
            System.out.println("¡Conexión exitosa!");
            System.out.println("BD: " + conexion.getCatalog());
            System.out.println("AutoCommit: " + conexion.getAutoCommit());

        } catch (SQLException e) {
            System.err.println("Error de conexión:");
            System.err.println("  Código: " + e.getErrorCode());
            System.err.println("  SQL State: " + e.getSQLState());
            System.err.println("  Mensaje: " + e.getMessage());
        } finally {
            // Cerrar conexión en finally
            if (conexion != null) {
                try {
                    conexion.close();
                    System.out.println("Conexión cerrada.");
                } catch (SQLException e) {
                    System.err.println("Error al cerrar: " + e.getMessage());
                }
            }
        }
    }
}
```

Desde Java 7 (try-with-resources), el cierre es automático:

```java
            System.out.println("Versión: " + conexion.getMetaData().getDatabaseProductVersion());
        } catch (SQLException e) {
            System.err.println("Error: " + e.getMessage());
        }
    }
}
```

4. STATEMENT: EJECUTAR CONSULTAS SQL
===============================================================================

Statement se usa para ejecutar sentencias SQL estáticas.

```java
import java.sql.*;

public class StatementDemo {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/java_master";
        String user = "root";
        String password = "password";

        String sql = "SELECT id, nombre, email, salario FROM empleados";

        try (Connection con = DriverManager.getConnection(url, user, password);
             Statement stmt = con.createStatement();
             ResultSet rs = stmt.executeQuery(sql)) {

            System.out.println("Empleados:");
            System.out.println("ID | NOMBRE | EMAIL | SALARIO");
            System.out.println("---|--------|-------|--------");

            while (rs.next()) {
                int id = rs.getInt("id");
                String nombre = rs.getString("nombre");
                String email = rs.getString("email");
                double salario = rs.getDouble("salario");

                System.out.printf("%2d | %-6s | %-10s | %.2f%n",
                    id, nombre, email, salario);
            }

        } catch (SQLException e) {
            System.err.println("Error: " + e.getMessage());
        }
    }
}
```

Métodos de Statement:
- executeQuery(sql): Para SELECT, devuelve ResultSet.
- executeUpdate(sql): Para INSERT, UPDATE, DELETE, devuelve filas afectadas.
- execute(sql): Para cualquier SQL, devuelve boolean (true = ResultSet).
- executeBatch(): Ejecuta un lote de sentencias.

```java
public class ExecuteUpdateDemo {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/java_master";
        String user = "root";
        String password = "password";

        try (Connection con = DriverManager.getConnection(url, user, password);
             Statement stmt = con.createStatement()) {

            // INSERT
            String insert = "INSERT INTO empleados (nombre, email, salario) " +
                "VALUES ('Ana López', 'ana@email.com', 45000.00)";
            int filas = stmt.executeUpdate(insert);
            System.out.println("Insertadas: " + filas + " filas");

            // UPDATE
            String update = "UPDATE empleados SET salario = salario * 1.1 " +
                "WHERE salario < 40000";
            filas = stmt.executeUpdate(update);
            System.out.println("Actualizadas: " + filas + " filas");

            // DELETE
            String delete = "DELETE FROM empleados WHERE id = 1";
            filas = stmt.executeUpdate(delete);
            System.out.println("Eliminadas: " + filas + " filas");

        } catch (SQLException e) {
            System.err.println("Error: " + e.getMessage());
        }
    }
}
```

5. RESULTSET: PROCESAR RESULTADOS
===============================================================================

ResultSet representa el conjunto de resultados de una consulta. Es un cursor
que apunta a una fila y avanza con next().

```java
import java.sql.*;
import java.time.LocalDate;

public class ResultSetDemo {
    public static void main(String[] args) throws SQLException {
        String url = "jdbc:mysql://localhost:3306/java_master";
        String user = "root";
        String password = "password";

        String sql = "SELECT * FROM productos WHERE precio > ?";

        try (Connection con = DriverManager.getConnection(url, user, password);
             PreparedStatement ps = con.prepareStatement(sql)) {

            ps.setDouble(1, 100.0);

            try (ResultSet rs = ps.executeQuery()) {
                // Metadatos del resultado
                ResultSetMetaData meta = rs.getMetaData();
                int columnas = meta.getColumnCount();

                System.out.println("Columnas: " + columnas);
                for (int i = 1; i <= columnas; i++) {
                    System.out.println("  " + i + ": " +
                        meta.getColumnName(i) + " (" +
                        meta.getColumnTypeName(i) + ")");
                }

                // Recorrer resultados
                System.out.println("\nProductos:");
                while (rs.next()) {
                    // Acceso por nombre de columna
                    int id = rs.getInt("id");
                    String nombre = rs.getString("nombre");
                    double precio = rs.getDouble("precio");

                    // Acceso por índice de columna (1-based)
                    int stock = rs.getInt(4);
                    // Si hay fecha: rs.getDate("fecha_creacion").toLocalDate()

                    System.out.printf("  %2d | %-15s | $%6.2f | %3d stock%n",
                        id, nombre, precio, stock);
                }
            }
        }
    }
}
```

Métodos get de ResultSet:
- Tipos numéricos: getInt(), getLong(), getDouble(), getBigDecimal().
- Texto: getString().
- Fechas: getDate(), getTime(), getTimestamp().
- Binarios: getBytes(), getBlob(), getClob().
- Booleanos: getBoolean().

Todos tienen variantes por nombre de columna (getInt("id")) y por índice
(getInt(1)).

7. GESTIÓN DE RECURSOS Y BUENAS PRÁCTICAS
===============================================================================

La gestión incorrecta de recursos JDBC es una de las causas más comunes de
problemas en aplicaciones reales.

```java
public class GestionRecursos {
    // MALA PRÁCTICA: No cerrar recursos
    public void malaGestion() throws SQLException {
        Connection con = DriverManager.getConnection("...");
        Statement stmt = con.createStatement();
        ResultSet rs = stmt.executeQuery("SELECT * FROM tabla");
        // No cierra nada -> fuga de conexiones
    }

    // BUENA PRÁCTICA: try-with-resources
    public void buenaGestion() {
        String sql = "SELECT * FROM empleados WHERE departamento = ?";

        try (Connection con = DriverManager.getConnection("url", "user", "pass");
             PreparedStatement ps = con.prepareStatement(sql)) {

            ps.setString(1, "Ventas");

            try (ResultSet rs = ps.executeQuery()) {
                while (rs.next()) {
                    System.out.println(rs.getString("nombre"));
                }
            }
            // ResultSet se cierra automáticamente

        } catch (SQLException e) {
            System.err.println("Error: " + e.getMessage());
        }
        // PreparedStatement, Connection se cierran automáticamente
    }

    // BUENA PRÁCTICA: Closeable de forma manual (pre-Java 7)
    public void gestionManual() {
        Connection con = null;
        Statement stmt = null;
        ResultSet rs = null;
        try {
            con = DriverManager.getConnection("...");
            stmt = con.createStatement();
            rs = stmt.executeQuery("SELECT * FROM tabla");
            while (rs.next()) { /* procesar */ }
        } catch (SQLException e) {
            // Manejar error
        } finally {
            // Cerrar en orden inverso
            try { if (rs != null) rs.close(); } catch (SQLException e) {}
            try { if (stmt != null) stmt.close(); } catch (SQLException e) {}
            try { if (con != null) con.close(); } catch (SQLException e) {}
        }
    }
}
```

Reglas de oro:
1. Siempre cierra en el orden inverso: ResultSet -> Statement -> Connection.
2. Usa try-with-resources siempre que sea posible.
3. No mantengas conexiones abiertas más tiempo del necesario.
4. Un pool de conexiones (ver capítulo 65) es mejor que abrir/cerrar cada vez.

8. PATRÓN DAO (DATA ACCESS OBJECT)
===============================================================================

El patrón DAO encapsula el acceso a datos y separa la lógica de negocio de
la lógica de persistencia.

```java
// Modelo
class Empleado {
    private int id;
    private String nombre;
    private String email;
    private double salario;

    public Empleado() {}
    public Empleado(String nombre, String email, double salario) {
        this.nombre = nombre;
        this.email = email;
        this.salario = salario;
    }
    // Getters y setters
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
    public double getSalario() { return salario; }
    public void setSalario(double salario) { this.salario = salario; }

    @Override
    public String toString() {
        return "Empleado{id=" + id + ", nombre='" + nombre + "'}";
    }
}

// Interfaz DAO
interface EmpleadoDAO {
    Empleado buscarPorId(int id);
    List<Empleado> listarTodos();
    void insertar(Empleado emp);
    void actualizar(Empleado emp);
    void eliminar(int id);
}

// Implementación DAO con JDBC
class EmpleadoDAOImpl implements EmpleadoDAO {
    private static final String URL = "jdbc:mysql://localhost:3306/java_master";
    private static final String USER = "root";
    private static final String PASS = "password";

    private Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASS);
    }

    @Override
    public Empleado buscarPorId(int id) {
        String sql = "SELECT * FROM empleados WHERE id = ?";
        try (Connection con = getConnection();
             PreparedStatement ps = con.prepareStatement(sql)) {
            ps.setInt(1, id);
            try (ResultSet rs = ps.executeQuery()) {
                if (rs.next()) {
                    return mapearEmpleado(rs);
                }
            }
        } catch (SQLException e) {
            throw new RuntimeException("Error buscando empleado", e);
        }
        return null;
    }

    @Override
    public List<Empleado> listarTodos() {
        List<Empleado> lista = new java.util.ArrayList<>();
        String sql = "SELECT * FROM empleados";
        try (Connection con = getConnection();
             Statement stmt = con.createStatement();
             ResultSet rs = stmt.executeQuery(sql)) {
            while (rs.next()) {
                lista.add(mapearEmpleado(rs));
            }
        } catch (SQLException e) {
            throw new RuntimeException("Error listando empleados", e);
        }
        return lista;
    }

    @Override
    public void insertar(Empleado emp) {
        String sql = "INSERT INTO empleados (nombre, email, salario) VALUES (?, ?, ?)";
        try (Connection con = getConnection();
             PreparedStatement ps = con.prepareStatement(sql,
                 Statement.RETURN_GENERATED_KEYS)) {
            ps.setString(1, emp.getNombre());
            ps.setString(2, emp.getEmail());
            ps.setDouble(3, emp.getSalario());
            ps.executeUpdate();
            try (ResultSet keys = ps.getGeneratedKeys()) {
                if (keys.next()) {
                    emp.setId(keys.getInt(1));
                }
            }
        } catch (SQLException e) {
            throw new RuntimeException("Error insertando empleado", e);
        }
    }

    @Override
    public void actualizar(Empleado emp) {
        String sql = "UPDATE empleados SET nombre=?, email=?, salario=? WHERE id=?";
        try (Connection con = getConnection();
             PreparedStatement ps = con.prepareStatement(sql)) {
            ps.setString(1, emp.getNombre());
            ps.setString(2, emp.getEmail());
            ps.setDouble(3, emp.getSalario());
            ps.setInt(4, emp.getId());
            ps.executeUpdate();
        } catch (SQLException e) {
            throw new RuntimeException("Error actualizando empleado", e);
        }
    }

    @Override
    public void eliminar(int id) {
        String sql = "DELETE FROM empleados WHERE id = ?";
        try (Connection con = getConnection();
             PreparedStatement ps = con.prepareStatement(sql)) {
            ps.setInt(1, id);
            ps.executeUpdate();
        } catch (SQLException e) {
            throw new RuntimeException("Error eliminando empleado", e);
        }
    }

    private Empleado mapearEmpleado(ResultSet rs) throws SQLException {
        Empleado emp = new Empleado();
        emp.setId(rs.getInt("id"));
        emp.setNombre(rs.getString("nombre"));
        emp.setEmail(rs.getString("email"));
        emp.setSalario(rs.getDouble("salario"));
        return emp;
    }
}

// Uso del DAO
public class DAODemo {
    public static void main(String[] args) {
        EmpleadoDAO dao = new EmpleadoDAOImpl();

        // Insertar
        Empleado nuevo = new Empleado("Carlos Ruiz", "carlos@email.com", 55000);
        dao.insertar(nuevo);
        System.out.println("Insertado: " + nuevo.getId());

        // Listar
        System.out.println("\nTodos los empleados:");
        for (Empleado e : dao.listarTodos()) {
            System.out.println("  " + e);
        }

        // Buscar
        Empleado encontrado = dao.buscarPorId(1);
        System.out.println("\nBuscado: " + encontrado);

        // Actualizar
        if (encontrado != null) {
            encontrado.setSalario(encontrado.getSalario() * 1.1);
            dao.actualizar(encontrado);
            System.out.println("Actualizado: " +
                dao.buscarPorId(encontrado.getId()));
        }

        // Eliminar
        dao.eliminar(nuevo.getId());
        System.out.println("Eliminado empleado " + nuevo.getId());
    }
}
```

9. SQL INJECTION Y SEGURIDAD
===============================================================================

Nunca concatenes valores directamente en SQL. Usa siempre PreparedStatement
para prevenir inyección SQL.

```java
public class SQLInjectionDemo {
    public static void main(String[] args) {
        String inputMalicioso = "' OR '1'='1";

        // PELIGROSO: concatenación directa
        String sqlInseguro = "SELECT * FROM usuarios WHERE user = '" +
            inputMalicioso + "' AND pass = 'x'";
        // Resultado: SELECT * FROM usuarios WHERE user = '' OR '1'='1' AND pass = 'x'
        // Esto devuelve TODOS los usuarios

        // SEGURO: PreparedStatement
        String sqlSeguro = "SELECT * FROM usuarios WHERE user = ? AND pass = ?";

        try (Connection con = DriverManager.getConnection("...", "...", "...");
             PreparedStatement ps = con.prepareStatement(sqlSeguro)) {
            ps.setString(1, inputMalicioso); // Se trata como literal, no como SQL
            ps.setString(2, "x");
            try (ResultSet rs = ps.executeQuery()) {
                // No devolverá nada si el usuario no existe
            }
        } catch (SQLException e) {
            System.err.println("Error: " + e.getMessage());
        }
    }
}
```

PreparedStatement también ofrece beneficios de rendimiento: las consultas se
precompilan y pueden reutilizarse.

10. EJEMPLO PRÁCTICO: GESTOR DE INVENTARIO
===============================================================================

Un ejemplo completo que integra todos los conceptos.

```java
public class GestorInventario {
    private static final String URL = "jdbc:mysql://localhost:3306/tienda";
    private static final String USER = "root";
    private static final String PASS = "password";

    // Crear tabla (si no existe)
    public void inicializarBD() {
        String sql = "CREATE TABLE IF NOT EXISTS productos (" +
            "id INT AUTO_INCREMENT PRIMARY KEY," +
            "nombre VARCHAR(100) NOT NULL," +
            "precio DECIMAL(10,2) NOT NULL," +
            "stock INT NOT NULL DEFAULT 0," +
            "categoria VARCHAR(50)" +
            ")";

        try (Connection con = DriverManager.getConnection(URL, USER, PASS);
             Statement stmt = con.createStatement()) {
            stmt.execute(sql);
            System.out.println("Tabla productos lista.");
        } catch (SQLException e) {
            System.err.println("Error inicializando BD: " + e.getMessage());
        }
    }

    // Reporte de productos con bajo stock
    public void reporteBajoStock(int limite) {
        String sql = "SELECT id, nombre, precio, stock FROM productos " +
            "WHERE stock < ? ORDER BY stock ASC";

        try (Connection con = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement ps = con.prepareStatement(sql)) {

            ps.setInt(1, limite);

            try (ResultSet rs = ps.executeQuery()) {
                System.out.println("\n=== PRODUCTOS CON BAJO STOCK ===");
                System.out.printf("%-4s %-20s %-10s %-6s%n",
                    "ID", "NOMBRE", "PRECIO", "STOCK");
                System.out.println("---- -------------------- ---------- ------");

                boolean hay = false;
                while (rs.next()) {
                    hay = true;
                    System.out.printf("%-4d %-20s $%-8.2f %-6d%n",
                        rs.getInt("id"),
                        rs.getString("nombre"),
                        rs.getDouble("precio"),
                        rs.getInt("stock"));
                }

                if (!hay) {
                    System.out.println("No hay productos con bajo stock.");
                }
            }

        } catch (SQLException e) {
            System.err.println("Error en reporte: " + e.getMessage());
        }
    }

    public static void main(String[] args) {
        GestorInventario gestor = new GestorInventario();
        gestor.inicializarBD();
        gestor.reporteBajoStock(10);
    }
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

En este capítulo hemos aprendido:

• JDBC es la API estándar de Java para acceso a bases de datos relacionales,
  independiente del proveedor de BD.

• Los drivers Tipo 4 son los más usados: implementación pura Java que
  traduce JDBC al protocolo nativo de red.

• Connection se obtiene con DriverManager.getConnection(url, user, pass).

• Statement ejecuta consultas SQL estáticas; PreparedStatement para
  consultas parametrizadas (más seguro y eficiente).

• ResultSet es un cursor para recorrer filas resultado. Se accede con
  next() y se extraen valores con getInt(), getString(), etc.

• Los recursos JDBC deben cerrarse siempre. try-with-resources es la
  forma más limpia de hacerlo.

• El patrón DAO separa la lógica de acceso a datos de la lógica de negocio.

• La inyección SQL se previene usando PreparedStatement en lugar de
  concatenar strings.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Necesito instalar algo más para usar JDBC?**
Sí, necesitas el driver JDBC de tu base de datos (archivo JAR). Si usas
Maven/Gradle, añades la dependencia. Desde JDBC 4.0, el driver se carga
automáticamente si está en el classpath.

❓ **¿Qué base de datos me recomiendas para aprender?**
H2 es excelente para aprender: es una BD embebida en Java, no necesita
instalación, soporta modo memoria y es compatible con SQL estándar.

❓ **¿Qué es una conexión pool y por qué la necesito?**
Abrir y cerrar conexiones es costoso. Un pool mantiene un conjunto de
conexiones reutilizables. Lo veremos en detalle en el capítulo 65.

❓ **¿Statement o PreparedStatement?**
Siempre PreparedStatement. Es más seguro (previene inyección SQL), más
eficiente (precompila la consulta) y más legible.

❓ **¿Qué es ResultSet.TYPE_SCROLL_INSENSITIVE?**
Permite moverte hacia adelante y atrás en el ResultSet, y acceder a filas
por posición absoluta. Por defecto, ResultSet solo avanza (TYPE_FORWARD_ONLY).

❓ **¿Cómo manejo fechas en JDBC?**
Usa java.sql.Date, java.sql.Time y java.sql.Timestamp. Con Java 8+, los
métodos getDate() y getTimestamp() se convierten con toLocalDate() y
toLocalDateTime().
