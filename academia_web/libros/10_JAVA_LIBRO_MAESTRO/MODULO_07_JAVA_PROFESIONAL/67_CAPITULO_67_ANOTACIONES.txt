===============================================================================
1. ¿QUÉ SON LAS ANOTACIONES?
===============================================================================

Las anotaciones son una forma de añadir metadatos al código fuente. Se
identifican con el símbolo @ y pueden aplicarse a clases, métodos, campos,
parámetros, constructores, variables locales y otras anotaciones.

```java
// Las anotaciones comienzan con @
// Ejemplos de anotaciones integradas:

public class EjemploAnotaciones {

    @Override
    public String toString() {
        return "Esta anotación verifica que estamos sobrescribiendo";
    }

    @Deprecated
    public void metodoViejo() {
        System.out.println("Este método no debería usarse");
    }

    @SuppressWarnings("unchecked")
    public void metodoConAdvertencias() {
        java.util.List lista = new java.util.ArrayList();
        lista.add("sin tipo genérico");
    }
}
```

Las anotaciones pueden clasificarse en tres categorías:
1. Anotaciones para el compilador: @Override, @Deprecated, @SuppressWarnings.
2. Anotaciones para procesamiento en tiempo de compilación: generación de
   código, validación (ej: Lombok, Nullable).
3. Anotaciones en tiempo de ejecución: accesibles via Reflection (ej: Spring,
   JPA, JUnit).

===============================================================================
2. @OVERRIDE
===============================================================================

@Override es una anotación que indica que un método está sobrescribiendo un
método de la superclase o implementando un método de una interfaz.

```java
interface Animal {
    void hacerSonido();
    void comer(String alimento);
}

class Perro implements Animal {

    @Override
    public void hacerSonido() {
        System.out.println("Guau guau!");
    }

    @Override
    public void comer(String alimento) {
        System.out.println("El perro come " + alimento);
    }
}

// Ejemplo de error que @Override previene
class Gato implements Animal {

    @Override
    public void hacerSonido() {
        System.out.println("Miau");
    }

    // Esto NO compilaría porque el nombre del método es incorrecto
    // @Override
    // public void comerAlimento(String a) { } // Error: no sobrescribe nada
}

// Sobrescritura de métodos de Object
class Persona {
    private String nombre;
    private int edad;

    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    @Override
    public String toString() {
        return "Persona{nombre='" + nombre + "', edad=" + edad + "}";
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        Persona persona = (Persona) obj;
        return edad == persona.edad && nombre.equals(persona.nombre);
    }

    @Override
    public int hashCode() {
        return java.util.Objects.hash(nombre, edad);
    }
}

public class OverrideDemo {
    public static void main(String[] args) {
        Perro perro = new Perro();
        perro.hacerSonido();
        perro.comer("carne");

        Persona p1 = new Persona("Ana", 30);
        Persona p2 = new Persona("Ana", 30);

        System.out.println(p1); // Usa toString()
        System.out.println("¿Iguales? " + p1.equals(p2));
    }
}
```

@Override es crucial porque:
- El compilador verifica que realmente estás sobrescribiendo un método.
- Previene errores por tipear mal el nombre del método.
- Protege contra cambios en la superclase que eliminen el método.
- Mejora la legibilidad del código.

3. @DEPRECATED
===============================================================================

@Deprecated marca elementos que ya no deberían usarse. El compilador genera
una advertencia cuando alguien los usa.

```java
class SistemaAntiguo {

    @Deprecated
    public void metodoObsoleto() {
        System.out.println("Este método está obsoleto");
    }

    // Con Javadoc indicando la alternativa
    @Deprecated
    public void login(String user, String pass) {
        // Versión antigua, insegura
    }

    public void loginSeguro(String user, String pass, boolean usarSSL) {
        // Versión nueva, segura
        System.out.println("Login seguro");
    }
}

// También puedes deprecar clases enteras
@Deprecated
class ClaseObsoleta {
    public void hacerAlgo() {
        System.out.println("Clase obsoleta");
    }
}

// @Deprecated con since y forRemoval (Java 9+)
class SistemaActualizado {

    @Deprecated(since = "2.0", forRemoval = true)
    public void metodoAEliminar() {
        System.out.println("Será eliminado en la próxima versión");
    }

    @Deprecated(since = "1.5")
    public void metodoAntiguo() {
        System.out.println("Obsoleto desde v1.5");
    }

    public void metodoNuevo() {
        System.out.println("Usa este método en su lugar");
    }
}

public class DeprecatedDemo {
    @SuppressWarnings("deprecation")
    public static void main(String[] args) {
        SistemaAntiguo sistema = new SistemaAntiguo();

        // El compilador genera advertencia aquí
        sistema.metodoObsoleto();

        // Usando @SuppressWarnings("deprecation") en el método
        // eliminamos la advertencia

        SistemaActualizado actualizado = new SistemaActualizado();
        actualizado.metodoAntiguo();
        // metodoAEliminar produce advertencia más fuerte
        // porque tiene forRemoval = true
    }
}
```

Desde Java 9, @Deprecated tiene dos elementos:
- since: Versión desde la que está deprecado.
- forRemoval: true si se planea eliminar en el futuro.

4. @SUPPRESSWARNINGS
===============================================================================

@SuppressWarnings le dice al compilador que no genere ciertas advertencias.

```java
import java.util.*;

public class SuppressWarningsDemo {

    @SuppressWarnings("unchecked")
    public void sinAdvertenciaGenericos() {
        List lista = new ArrayList(); // Raw type
        lista.add("String en raw list");
        // Normalmente: warning "unchecked"
    }

    @SuppressWarnings("deprecation")
    public void sinAdvertenciaDeprecado() {
        SistemaAntiguo s = new SistemaAntiguo();
        s.metodoObsoleto(); // Normalmente: warning "deprecation"
    }

    @SuppressWarnings("rawtypes")
    public void sinAdvertenciaRawTypes() {
        List lista = new ArrayList(); // Sin warning por raw type
    }

    @SuppressWarnings("serial")
    public class SinWarningSerial implements java.io.Serializable {
        // Sin warning por falta de serialVersionUID
    }

    @SuppressWarnings("all") // Desactiva TODAS las advertencias
    public void sinNingunaAdvertencia() {
        // Úsalo con cuidado, solo en casos extremos
    }

    @SuppressWarnings({"unchecked", "rawtypes", "deprecation"})
    public void multiplesAdvertencias() {
        List lista = new ArrayList();
        SistemaAntiguo s = new SistemaAntiguo();
        s.metodoObsoleto();
    }

    public static void main(String[] args) {
        SuppressWarningsDemo demo = new SuppressWarningsDemo();
        demo.sinAdvertenciaGenericos();
        demo.sinAdvertenciaDeprecado();
        demo.sinAdvertenciaRawTypes();
        demo.sinNingunaAdvertencia();
        demo.multiplesAdvertencias();
    }
}
```

Advertencias comunes:
- unchecked: Operaciones con tipos raw o casts sin verificación.
- deprecation: Uso de código marcado como @Deprecated.
- rawtypes: Uso de clases genéricas sin parámetros de tipo.
- serial: Clases Serializable sin serialVersionUID.
- all: Todas las advertencias.

5. OTRAS ANOTACIONES INTEGRADAS
===============================================================================

```java
import java.lang.annotation.Native;

// @FunctionalInterface: indica que una interfaz tiene un solo método abstracto
@FunctionalInterface
interface Operacion {
    int ejecutar(int a, int b);
    // Solo UN método abstracto permite uso como lambda
    // boolean equals(Object obj); // Este es de Object, no cuenta
}

// @SafeVarargs: suprime advertencias sobre operaciones inseguras con varargs
class EjemploSafeVarargs {

    @SafeVarargs
    public final <T> List<T> combinarListas(List<T>... listas) {
        List<T> resultado = new ArrayList<>();
        for (List<T> lista : listas) {
            resultado.addAll(lista);
        }
        return resultado;
    }

    @SafeVarargs
    public static <T> void imprimir(T... elementos) {
        for (T e : elementos) {
            System.out.print(e + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        List<String> a = List.of("A", "B");
        List<String> b = List.of("C", "D");
        EjemploSafeVarargs ej = new EjemploSafeVarargs();
        List<String> combinada = ej.combinarListas(a, b);
        System.out.println(combinada);

        imprimir("Hola", "Mundo", "Java");
    }
}

// @Native: indica que un campo es accesible desde código nativo (JNI)
class EjemploNative {
    @Native
    public static final int CONSTANTE_NATIVA = 42;
}
```

6. CREACIÓN DE ANOTACIONES PERSONALIZADAS
===============================================================================

Para crear una anotación se usa @interface. Se pueden definir elementos
(parámetros) que actúan como métodos.

```java
import java.lang.annotation.*;

// Anotación simple (sin elementos)
@interface Info {
}

// Anotación con elementos
@interface Autor {
    String nombre();
    String fecha() default "desconocida";
    String[] revisores() default {};
}

// Anotación con un único elemento (convención: value)
@interface Version {
    double value();
}

// Uso de las anotaciones personalizadas
@Info
@Autor(nombre = "Profesor Búho", fecha = "2024-01-15")
@Version(1.0)
class ClaseAnotada {

    @Autor(nombre = "Asistente",
           fecha = "2024-02-01",
           revisores = {"Alumno1", "Alumno2"})
    public void metodoDocumentado() {
        System.out.println("Método con anotaciones");
    }

    @Version(1.5)
    public void metodoVersionado() {
        System.out.println("Este método es v1.5");
    }

    @Autor(nombre = "Invitado")
    public void metodoSinFecha() {
        // Usa la fecha por defecto: "desconocida"
        System.out.println("Usa valor por defecto para fecha");
    }
}
```

Reglas para elementos de anotaciones:
- Tipos permitidos: primitivos, String, Class, enum, anotaciones, arrays de estos.
- No pueden ser null (los valores por defecto tampoco).
- Pueden tener valores por defecto (default).
- Si un elemento tiene valor por defecto, es opcional al usar la anotación.
- Si la anotación tiene un solo elemento llamado "value", se puede omitir
  el nombre: @Version(1.0) en lugar de @Version(value = 1.0).

7. META-ANOTACIONES
===============================================================================

Las meta-anotaciones se aplican a otras anotaciones para configurar su
comportamiento.

```java
import java.lang.annotation.*;

// @Retention: indica hasta dónde se conserva la anotación
@Retention(RetentionPolicy.SOURCE)  // Solo en código fuente, se descarta al compilar
@interface AnotacionSource { }

@Retention(RetentionPolicy.CLASS)   // En el class file, pero no accesible en runtime
@interface AnotacionClass { }

@Retention(RetentionPolicy.RUNTIME) // Accesible en tiempo de ejecución via Reflection
@interface AnotacionRuntime { }

// @Target: indica a qué elementos se puede aplicar la anotación
@Target(ElementType.TYPE)           // Solo a clases, interfaces, enums
@interface AnotacionTipo { }

@Target(ElementType.METHOD)         // Solo a métodos
@interface AnotacionMetodo { }

@Target(ElementType.FIELD)          // Solo a campos
@interface AnotacionCampo { }

@Target(ElementType.PARAMETER)      // Solo a parámetros
@interface AnotacionParametro { }

@Target(ElementType.CONSTRUCTOR)    // Solo a constructores
@interface AnotacionConstructor { }

// Múltiples targets
@Target({ElementType.TYPE, ElementType.METHOD})
@interface AnotacionMultiple { }

// @Documented: la anotación aparecerá en el Javadoc generado
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface MetodoDocumentado {
    String descripcion();
}

// @Inherited: la anotación se hereda a las subclases
@Inherited
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
@interface Entidad {
    String nombre();
}

@Entidad(nombre = "Base")
class ClaseBase { }

class ClaseDerivada extends ClaseBase { }
// ClaseDerivada HEREDA la anotación @Entidad

// Uso completo
@AnotacionTipo
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface Medible {
    String unidad() default "ms";
    String descripcion();
}
```

8. PROCESAMIENTO DE ANOTACIONES EN TIEMPO DE EJECUCIÓN
===============================================================================

Con Reflection podemos leer anotaciones en tiempo de ejecución.

```java
import java.lang.annotation.*;
import java.lang.reflect.*;

// Anotación que procesaremos
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface Auditable {
    String accion();
    String nivel() default "INFO";
}

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.FIELD)
@interface Validar {
    int minLongitud() default 0;
    int maxLongitud() default Integer.MAX_VALUE;
    boolean noNulo() default false;
}

// Clase que usa las anotaciones
class ServicioUsuario {

    @Auditable(accion = "CREAR_USUARIO", nivel = "ALTA")
    public void crearUsuario(String nombre) {
        System.out.println("Usuario " + nombre + " creado");
    }

    @Auditable(accion = "ELIMINAR_USUARIO")
    public void eliminarUsuario(int id) {
        System.out.println("Usuario " + id + " eliminado");
    }

    public void metodoNormal() {
        System.out.println("Método sin auditoría");
    }
}

class UsuarioEntidad {

    @Validar(noNulo = true, minLongitud = 3, maxLongitud = 50)
    private String nombre;

    @Validar(noNulo = true)
    private String email;

    private int edad;

    public UsuarioEntidad(String nombre, String email) {
        this.nombre = nombre;
        this.email = email;
    }
}

// Procesador de anotaciones
class ProcesadorAnotaciones {

    // Procesar @Auditable
    public static void auditarMetodos(Class<?> clazz) {
        System.out.println("\n=== Auditoría de " + clazz.getSimpleName() + " ===");

        for (Method metodo : clazz.getDeclaredMethods()) {
            Auditable auditable = metodo.getAnnotation(Auditable.class);
            if (auditable != null) {
                System.out.println("Método: " + metodo.getName());
                System.out.println("  Acción: " + auditable.accion());
                System.out.println("  Nivel: " + auditable.nivel());
            }
        }
    }

    // Procesar @Validar
    public static void validarCampos(Object obj) throws Exception {
        Class<?> clazz = obj.getClass();
        System.out.println("\n=== Validando " + clazz.getSimpleName() + " ===");

        for (Field campo : clazz.getDeclaredFields()) {
            Validar validar = campo.getAnnotation(Validar.class);
            if (validar != null) {
                campo.setAccessible(true);
                Object valor = campo.get(obj);

                System.out.println("Campo: " + campo.getName());

                if (validar.noNulo() && valor == null) {
                    throw new Exception(campo.getName() + " no puede ser nulo");
                }

                if (valor instanceof String texto) {
                    if (validar.noNulo() && (texto == null || texto.trim().isEmpty())) {
                        throw new Exception(campo.getName() + " no puede estar vacío");
                    }
                    if (texto.length() < validar.minLongitud()) {
                        throw new Exception(campo.getName() +
                            " debe tener al menos " + validar.minLongitud() +
                            " caracteres");
                    }
                    if (texto.length() > validar.maxLongitud()) {
                        throw new Exception(campo.getName() +
                            " debe tener máximo " + validar.maxLongitud() +
                            " caracteres");
                    }
                }
                System.out.println("  -> Válido");
            }
        }
    }

    public static void main(String[] args) throws Exception {
        // Procesar auditoría
        auditarMetodos(ServicioUsuario.class);

        // Procesar validación
        UsuarioEntidad usuario = new UsuarioEntidad("Ana", "ana@email.com");
        validarCampos(usuario);

        // Esto lanzaría excepción:
        // UsuarioEntidad invalido = new UsuarioEntidad("An", null);
        // validarCampos(invalido);
    }
}
```

10. EJEMPLO PRÁCTICO: FRAMEWORK DE VALIDACIÓN CON ANOTACIONES
===============================================================================

Un mini-framework de validación usando anotaciones personalizadas.

```java
import java.lang.annotation.*;
import java.lang.reflect.*;

// Definición de anotaciones de validación
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.FIELD)
@interface NotNull {
    String mensaje() default "El campo no puede ser nulo";
}

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.FIELD)
@interface Tamaño {
    int min() default 0;
    int max() default Integer.MAX_VALUE;
    String mensaje() default "El campo debe tener entre {min} y {max} caracteres";
}

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.FIELD)
@interface Rango {
    double min() default Double.MIN_VALUE;
    double max() default Double.MAX_VALUE;
    String mensaje() default "El valor debe estar entre {min} y {max}";
}

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.FIELD)
@interface Email {
    String mensaje() default "Formato de email inválido";
}

// Clase modelo con validaciones
class Producto {

    @NotNull(mensaje = "El nombre del producto es obligatorio")
    @Tamaño(min = 3, max = 100,
        mensaje = "El nombre debe tener entre {min} y {max} caracteres")
    private String nombre;

    @Rango(min = 0.01, max = 99999.99,
        mensaje = "El precio debe estar entre {min} y {max}")
    private double precio;

    @Rango(min = 0, mensaje = "El stock no puede ser negativo")
    private int stock;

    @Email(mensaje = "El email del proveedor no es válido")
    private String emailProveedor;

    public Producto(String nombre, double precio, int stock, String emailProveedor) {
        this.nombre = nombre;
        this.precio = precio;
        this.stock = stock;
        this.emailProveedor = emailProveedor;
    }
}

// Motor de validación genérico
class Validador {
    public static class ErrorValidacion {
        private final String campo;
        private final String mensaje;

        public ErrorValidacion(String campo, String mensaje) {
            this.campo = campo;
            this.mensaje = mensaje;
        }

        @Override
        public String toString() {
            return campo + ": " + mensaje;
        }
    }

    public static List<ErrorValidacion> validar(Object obj) {
        List<ErrorValidacion> errores = new java.util.ArrayList<>();
        Class<?> clazz = obj.getClass();

        for (Field campo : clazz.getDeclaredFields()) {
            campo.setAccessible(true);
            Object valor;
            try {
                valor = campo.get(obj);
            } catch (Exception e) {
                errores.add(new ErrorValidacion(campo.getName(),
                    "Error accediendo al campo: " + e.getMessage()));
                continue;
            }

            // Validar @NotNull
            NotNull notNull = campo.getAnnotation(NotNull.class);
            if (notNull != null && valor == null) {
                errores.add(new ErrorValidacion(campo.getName(),
                    notNull.mensaje()));
            }

            // Validar @Tamaño
            Tamaño tamaño = campo.getAnnotation(Tamaño.class);
            if (tamaño != null && valor instanceof String texto) {
                if (texto.length() < tamaño.min() || texto.length() > tamaño.max()) {
                    String msg = tamaño.mensaje()
                        .replace("{min}", String.valueOf(tamaño.min()))
                        .replace("{max}", String.valueOf(tamaño.max()));
                    errores.add(new ErrorValidacion(campo.getName(), msg));
                }
            }

            // Validar @Rango
            Rango rango = campo.getAnnotation(Rango.class);
            if (rango != null && valor instanceof Number num) {
                double d = num.doubleValue();
                if (d < rango.min() || d > rango.max()) {
                    String msg = rango.mensaje()
                        .replace("{min}", String.valueOf(rango.min()))
                        .replace("{max}", String.valueOf(rango.max()));
                    errores.add(new ErrorValidacion(campo.getName(), msg));
                }
            }

            // Validar @Email (simple)
            Email email = campo.getAnnotation(Email.class);
            if (email != null && valor instanceof String texto
                    && !texto.isEmpty()) {
                if (!texto.matches("^[\\w.-]+@[\\w.-]+\\.\\w{2,}$")) {
                    errores.add(new ErrorValidacion(campo.getName(),
                        email.mensaje()));
                }
            }
        }

        return errores;
    }
}

public class FrameworkValidacionDemo {
    public static void main(String[] args) {
        // Producto válido
        Producto valido = new Producto("Laptop Gamer", 1299.99, 10, "proveedor@email.com");
        List<Validador.ErrorValidacion> errores = Validador.validar(valido);
        System.out.println("Producto válido: " +
            (errores.isEmpty() ? "OK" : errores));

        // Producto inválido
        Producto invalido = new Producto("AB", 0.0, -5, "email-invalido");
        errores = Validador.validar(invalido);
        System.out.println("\nProducto inválido:");
        errores.forEach(System.out::println);
    }
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

En este capítulo hemos aprendido:

• Las anotaciones son metadatos que añaden información al código sin
  alterar directamente su comportamiento.

• @Override verifica que un método realmente sobrescribe otro del padre.

• @Deprecated marca elementos obsoletos (con since y forRemoval desde
  Java 9).

• @SuppressWarnings suprime advertencias específicas del compilador.

• Otras anotaciones integradas: @FunctionalInterface, @SafeVarargs,
  @Native.

• Las anotaciones personalizadas se crean con @interface y pueden tener
  elementos con valores por defecto.

• Las meta-anotaciones (@Retention, @Target, @Documented, @Inherited)
  configuran el comportamiento de las anotaciones.

• Las anotaciones se procesan en tiempo de ejecución con Reflection
  (getAnnotation, getDeclaredAnnotations).

• Los frameworks modernos usan intensivamente anotaciones para
  configuración declarativa (Spring, JPA, JUnit).

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Cuál es la diferencia entre comentarios y anotaciones?**
Los comentarios son ignorados por el compilador. Las anotaciones son
metadatos procesables por el compilador, herramientas y frameworks.

❓ **¿Puedo extender una anotación?**
No, las anotaciones no pueden heredar de otras anotaciones. Pero puedes
combinar anotaciones y usar @Inherited para que se hereden a subclases.

❓ **¿Las anotaciones afectan el rendimiento?**
Las anotaciones con RetentionPolicy.RUNTIME se cargan en memoria, pero
el impacto es mínimo. El procesamiento con Reflection sí tiene overhead.

❓ **¿Por qué @Override es importante?**
Porque el compilador verifica que estás sobrescribiendo correctamente.
Sin @Override, un error tipográfico crearía un método nuevo en lugar de
sobrescribir, causando bugs difíciles de encontrar.

❓ **¿Qué es RetentionPolicy?**
Define cuánto tiempo se conserva la anotación: SOURCE (solo código fuente),
CLASS (en bytecode pero no accesible), RUNTIME (accesible via Reflection).

❓ **¿Puedo usar anotaciones en anotaciones?**
Sí, son las meta-anotaciones. @Retention y @Target son ejemplos de
anotaciones aplicadas a anotaciones.

🏗️ **Framework de Pruebas Unitarias Simple (Mini-JUnit)**

Diseña e implementa un mini-framework de pruebas unitarias similar a JUnit
usando anotaciones personalizadas y Reflection.

Requisitos:
1. Anotaciones: @Test, @Before, @After, @BeforeClass, @AfterClass.
2. El framework debe ejecutar todos los métodos anotados con @Test.
3. @Before se ejecuta antes de cada test, @After después de cada test.
4. Si un test lanza una excepción, se marca como fallido.
5. El framework debe reportar: total tests, exitosos, fallidos, tiempo.
6. Soportar @Test(expected = Exception.class) para tests que deben fallar.

Implementa:
- Definición de todas las anotaciones con @Retention(RUNTIME).
- Clase TestRunner que procesa las anotaciones con Reflection.
- Ejecuta los tests en orden: Before -> Test -> After.
- Captura excepciones y las reporta adecuadamente.
- Muestra estadísticas de ejecución.

Este desafío te mostrará cómo se construyen los frameworks de testing
desde cero, revelando la magia detrás de herramientas como JUnit.
