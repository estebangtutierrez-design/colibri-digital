1. ¿QUÉ ES REFLECTION?
===============================================================================

Reflection es la capacidad de un programa para examinarse a sí mismo y
modificar su propio comportamiento en tiempo de ejecución.

```java
// La esencia de Reflection: el objeto Class
public class QueEsReflection {
    public static void main(String[] args) throws Exception {
        // Toda clase en Java tiene un objeto Class asociado
        // Reflection te permite acceder a ese objeto y más

        System.out.println("Reflection: introspección en tiempo de ejecución");
        System.out.println("Permite:");
        System.out.println("  - Obtener metadatos de clases");
        System.out.println("  - Invocar métodos dinámicamente");
        System.out.println("  - Acceder a campos privados");
        System.out.println("  - Crear objetos sin usar new");
        System.out.println("  - Procesar anotaciones");
    }
}
```

Casos de uso de Reflection:
1. Frameworks: Spring, Hibernate, JUnit, Jackson.
2. IDEs: Autocompletado, navegación, refactorización.
3. Servidores de aplicaciones: Carga dinámica de clases.
4. Herramientas de desarrollo: Debuggers, profilers.
5. Serialización/Deserialización: JSON, XML.

2. OBTENER EL OBJETO CLASS
===============================================================================

Class es la puerta de entrada a Reflection. Hay tres formas de obtenerlo:

```java
public class ObtenerClass {
    public static void main(String[] args) throws Exception {
        // 1. .class (en tiempo de compilación)
        Class<String> claseString = String.class;
        Class<Integer> claseInt = Integer.class;
        Class<Persona> clasePersona = Persona.class;

        System.out.println("1. .class:");
        System.out.println("  String.class: " + claseString.getName());

        // 2. getClass() (en tiempo de ejecución, desde una instancia)
        Persona persona = new Persona("Ana", 30);
        Class<?> clasePersona2 = persona.getClass();
        System.out.println("2. getClass(): " + clasePersona2.getName());

        // 3. Class.forName() (carga dinámica desde string)
        Class<?> claseForName = Class.forName("java.util.ArrayList");
        System.out.println("3. forName: " + claseForName.getName());

        // Verificar igualdad
        System.out.println("\n¿Son iguales? " +
            (clasePersona == clasePersona2)); // true

        // Tipos primitivos
        System.out.println("\nClases de primitivos:");
        System.out.println("  int.class: " + int.class);
        System.out.println("  double.class: " + double.class);
        System.out.println("  boolean.class: " + boolean.class);
        System.out.println("  void.class: " + void.class);
    }
}

class Persona {
    private String nombre;
    private int edad;

    public Persona() {}

    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    public void saludar() {
        System.out.println("Hola, soy " + nombre);
    }

    private void metodoPrivado() {
        System.out.println("Método privado ejecutado");
    }

    @Override
    public String toString() {
        return "Persona{" + nombre + ", " + edad + "}";
    }
}
```

Class.forName() es especialmente importante porque permite cargar clases
dinámicamente usando su nombre completo como string. Esto es fundamental
para plugins, drivers y frameworks.

3. INSPECCIONAR CLASES: MÉTODOS
===============================================================================

getMethods() devuelve los métodos públicos (incluyendo heredados).
getDeclaredMethods() devuelve todos los métodos declarados en la clase.

```java
import java.lang.reflect.Method;
import java.lang.reflect.Modifier;

public class InspeccionarMetodos {
    public static void main(String[] args) {
        Class<?> clazz = String.class;

        System.out.println("=== Métodos públicos de String ===");
        Method[] metodosPublicos = clazz.getMethods();
        System.out.println("Total públicos (incluye heredados): " +
            metodosPublicos.length);

        // Mostrar primeros 10
        for (int i = 0; i < Math.min(10, metodosPublicos.length); i++) {
            Method m = metodosPublicos[i];
            System.out.println("  " + m.getReturnType().getSimpleName() +
                " " + m.getName() + "(" +
                parametrosToString(m.getParameterTypes()) + ")");
        }

        System.out.println("\n=== Métodos declarados de Persona ===");
        Class<?> personaClass = Persona.class;
        Method[] metodosDeclarados = personaClass.getDeclaredMethods();

        for (Method m : metodosDeclarados) {
            System.out.println("  " + Modifier.toString(m.getModifiers()) +
                " " + m.getReturnType().getSimpleName() +
                " " + m.getName() + "(" +
                parametrosToString(m.getParameterTypes()) + ")");

            // Anotaciones del método
            System.out.println("    Anotaciones: " +
                java.util.Arrays.toString(m.getDeclaredAnnotations()));
        }
    }

    static String parametrosToString(Class<?>[] params) {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < params.length; i++) {
            if (i > 0) sb.append(", ");
            sb.append(params[i].getSimpleName());
        }
        return sb.toString();
    }
}
```

Métodos clave de Class para inspección:
- getMethods(): métodos públicos (incluye heredados).
- getDeclaredMethods(): todos los métodos declarados.
- getMethod(name, paramTypes): método público específico.
- getDeclaredMethod(name, paramTypes): cualquier método específico.
- getConstructors(), getDeclaredConstructors(): constructores.
- getFields(), getDeclaredFields(): campos.
- getAnnotations(), getDeclaredAnnotations(): anotaciones.
- getSuperclass(): superclase.
- getInterfaces(): interfaces implementadas.
- getModifiers(): modificadores (public, static, final, etc.).

4. INSPECCIONAR CLASES: CAMPOS
===============================================================================

```java
import java.lang.reflect.Field;
import java.lang.reflect.Modifier;

class Empleado {
    public String nombrePublico;
    private double salario;
    protected String departamento;
    static int contador = 0;
    final String EMPRESA = "Tech Corp";
}

public class InspeccionarCampos {
    public static void main(String[] args) {
        Class<?> clazz = Empleado.class;

        System.out.println("=== Campos públicos (incluye heredados) ===");
        for (Field f : clazz.getFields()) {
            System.out.println("  " + f.getType().getSimpleName() +
                " " + f.getName());
        }

        System.out.println("\n=== Todos los campos declarados ===");
        for (Field f : clazz.getDeclaredFields()) {
            System.out.println("  " + Modifier.toString(f.getModifiers()) +
                " " + f.getType().getSimpleName() +
                " " + f.getName());
        }

        // Obtener campos específicos
        System.out.println("\n=== Campo específico ===");
        try {
            Field salarioField = clazz.getDeclaredField("salario");
            System.out.println("  Campo: " + salarioField);
            System.out.println("  Tipo: " + salarioField.getType());
            System.out.println("  Modificadores: " +
                Modifier.toString(salarioField.getModifiers()));
            System.out.println("  Accesible: " + salarioField.canAccess(null));
        } catch (NoSuchFieldException e) {
            System.out.println("  Campo no encontrado");
        }
    }
}
```

5. CREAR INSTANCIAS DINÁMICAMENTE
===============================================================================

Podemos crear objetos sin usar new, usando Reflection.

```java
import java.lang.reflect.Constructor;

public class CrearInstancias {
    public static void main(String[] args) throws Exception {
        // 1. newInstance() de Class (usa constructor sin args)
        Class<?> clazz = Class.forName("Persona");

        // En Java 9+ newInstance() está deprecado. Usar Constructor
        // Constructor<?> constructor = clazz.getConstructor();
        // Persona p1 = (Persona) constructor.newInstance();

        // 2. Con constructor específico
        Constructor<?> constructorConArgs =
            clazz.getConstructor(String.class, int.class);
        Persona p2 = (Persona) constructorConArgs.newInstance("Carlos", 25);
        System.out.println("Creada con Constructor: " + p2);

        // 3. Constructor privado (setAccessible)
        // Constructor<?> privado = clazz.getDeclaredConstructor();
        // privado.setAccessible(true);
        // Persona p3 = (Persona) privado.newInstance();

        // 4. Array de objetos
        int[] dims = {5};
        Object array = java.lang.reflect.Array.newInstance(
            Persona.class, dims);
        System.out.println("Array de Personas creado: " +
            java.lang.reflect.Array.getLength(array) + " elementos");

        // 5. Clases anidadas
        Class<?> innerClass = Class.forName(
            "CrearInstancias$ClaseInterna");
        Constructor<?> innerConstructor = innerClass.getDeclaredConstructor();
        innerConstructor.setAccessible(true);
        Object inner = innerConstructor.newInstance();
        System.out.println("Clase interna instanciada: " + inner.getClass());
    }

    static class ClaseInterna {
        public ClaseInterna() {
            System.out.println("Constructor interno ejecutado");
        }
    }
}
```

6. INVOCAR MÉTODOS DINÁMICAMENTE
===============================================================================

El método invoke() de Method permite llamar a cualquier método en tiempo
de ejecución.

```java
import java.lang.reflect.Method;

public class InvocarMetodos {
    public static void main(String[] args) throws Exception {
        Persona persona = new Persona("María", 28);

        // 1. Invocar método público
        Method toStringMethod = Persona.class.getMethod("toString");
        String resultado = (String) toStringMethod.invoke(persona);
        System.out.println("toString invocado: " + resultado);

        // 2. Invocar método con parámetros
        // Supongamos que hay un método: public void setNombre(String n)
        Method setNombreMethod =
            Persona.class.getMethod("setNombre", String.class);
        setNombreMethod.invoke(persona, "Laura Modificada");
        System.out.println("Después de setNombre: " + persona);

        // 3. Invocar método privado
        Method metodoPrivado =
            Persona.class.getDeclaredMethod("metodoPrivado");
        metodoPrivado.setAccessible(true); // Necesario para acceder a privados
        metodoPrivado.invoke(persona);

        // 4. Invocar método estático
        Method estaticoMethod =
            Utilidades.class.getMethod("saludar", String.class);
        String saludo = (String) estaticoMethod.invoke(null, "Mundo");
        System.out.println("Saludo estático: " + saludo);

        // 5. Manejo de excepciones
        try {
            Method metodoRiesgoso =
                Persona.class.getMethod("metodoQueLanza");
            metodoRiesgoso.invoke(persona);
        } catch (java.lang.reflect.InvocationTargetException e) {
            // La excepción real está encapsulada en InvocationTargetException
            System.out.println("Excepción en método: " +
                e.getCause().getClass().getSimpleName() +
                ": " + e.getCause().getMessage());
        }
    }
}

class Utilidades {
    public static String saludar(String nombre) {
        return "Hola, " + nombre + "!";
    }
}
```

7. ACCEDER Y MODIFICAR CAMPOS
===============================================================================

```java
import java.lang.reflect.Field;

public class AccederCampos {
    public static void main(String[] args) throws Exception {
        Persona persona = new Persona("Ana", 30);

        // 1. Leer campo público
        // Supongamos que Persona tuviera un campo público
        // Field nombrePublico = Persona.class.getField("nombrePublico");
        // String valor = (String) nombrePublico.get(persona);

        // 2. Leer campo privado (con setAccessible)
        Field nombreField = Persona.class.getDeclaredField("nombre");
        nombreField.setAccessible(true);
        String nombre = (String) nombreField.get(persona);
        System.out.println("Nombre (privado) leído: " + nombre);

        Field edadField = Persona.class.getDeclaredField("edad");
        edadField.setAccessible(true);
        int edad = edadField.getInt(persona);
        System.out.println("Edad (privado) leída: " + edad);

        // 3. Modificar campo privado
        nombreField.set(persona, "Ana Modificada");
        edadField.setInt(persona, 31);
        System.out.println("Después de modificar: " + persona);

        // 4. Campo estático
        Field contadorField = Empleado.class.getDeclaredField("contador");
        contadorField.setAccessible(true);
        int valorContador = contadorField.getInt(null); // null para estáticos
        System.out.println("Contador estático: " + valorContador);
        contadorField.setInt(null, 100);
        System.out.println("Contador modificado: " + Empleado.contador);

        // 5. Campo final (cuidado: puede no funcionar en todos los casos)
        Field empresaField = Empleado.class.getDeclaredField("EMPRESA");
        empresaField.setAccessible(true);
        // empresaField.set(null, "Otra Empresa"); // No recomendado
    }
}
```

8. CONSTRUCTORES CON REFLECTION
===============================================================================

```java
import java.lang.reflect.Constructor;

public class ConstructoresReflection {
    public static void main(String[] args) throws Exception {
        Class<?> clazz = Persona.class;

        System.out.println("=== Constructores ===");
        for (Constructor<?> c : clazz.getDeclaredConstructors()) {
            System.out.println("  " + c);
            System.out.println("    Parámetros: " + c.getParameterCount());
            Class<?>[] paramTypes = c.getParameterTypes();
            for (Class<?> p : paramTypes) {
                System.out.println("      - " + p.getSimpleName());
            }
        }

        // Crear instancias con diferentes constructores
        Constructor<?> sinArgs = clazz.getDeclaredConstructor();
        sinArgs.setAccessible(true);
        Persona p1 = (Persona) sinArgs.newInstance();
        System.out.println("\nSin args: " + p1);

        Constructor<?> conArgs = clazz.getConstructor(String.class, int.class);
        Persona p2 = (Persona) conArgs.newInstance("Roberto", 35);
        System.out.println("Con args: " + p2);

        // Constructor con varargs
        Constructor<?> varargsConstructor =
            clazz.getDeclaredConstructor(String[].class);

        // Verificar si un constructor es accesible
        System.out.println("\n¿Constructor público? " +
            Modifier.isPublic(conArgs.getModifiers()));
    }
}
```

9. PROXY DINÁMICO CON REFLECTION
===============================================================================

Java permite crear proxies dinámicos que implementan interfaces en tiempo
de ejecución. Es la base de AOP (Programación Orientada a Aspectos).

```java
import java.lang.reflect.*;

interface Servicio {
    void ejecutar(String tarea);
    int calcular(int x, int y);
}

// Implementación real
class ServicioReal implements Servicio {
    @Override
    public void ejecutar(String tarea) {
        System.out.println("Ejecutando: " + tarea);
    }

    @Override
    public int calcular(int x, int y) {
        return x + y;
    }
}

// Manejador de invocación (Interceptor)
class LoggingHandler implements InvocationHandler {
    private final Object target;

    public LoggingHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args)
            throws Throwable {
        System.out.println("[LOG] Antes de: " + method.getName());
        Object resultado = method.invoke(target, args != null ? args : new Object[0]);
        System.out.println("[LOG] Después de: " + method.getName() +
            " -> " + resultado);
        return resultado;
    }
}

public class ProxyDinamico {
    public static void main(String[] args) {
        ServicioReal real = new ServicioReal();

        Servicio proxy = (Servicio) Proxy.newProxyInstance(
            Servicio.class.getClassLoader(),
            new Class<?>[]{Servicio.class},
            new LoggingHandler(real)
        );

        // Llamadas al proxy (son interceptadas)
        proxy.ejecutar("Tarea importante");
        int resultado = proxy.calcular(10, 20);
        System.out.println("Resultado: " + resultado);

        // Verificar tipo
        System.out.println("\n¿Es proxy? " + Proxy.isProxyClass(proxy.getClass()));
        System.out.println("Handler: " + Proxy.getInvocationHandler(proxy));
    }
}
```

10. REFLECTION CON GENÉRICOS Y ANOTACIONES
===============================================================================

```java
import java.lang.reflect.*;
import java.util.*;

public class ReflectionGenericosAnotaciones {

    public static void main(String[] args) throws Exception {
        // Obtener información genérica
        Method metodo = MiClase.class.getMethod("getLista");
        Type returnType = metodo.getGenericReturnType();

        if (returnType instanceof ParameterizedType parametrizado) {
            System.out.println("Tipo raw: " + parametrizado.getRawType());
            System.out.println("Argumentos tipo:");
            for (Type arg : parametrizado.getActualTypeArguments()) {
                System.out.println("  - " + arg);
            }
        }

        // Parámetros genéricos del método
        Method metodoConParam =
            MiClase.class.getMethod("procesar", List.class);
        for (Type param : metodoConParam.getGenericParameterTypes()) {
            if (param instanceof ParameterizedType pt) {
                System.out.println("Parámetro genérico: " +
                    pt.getActualTypeArguments()[0]);
            }
        }

        // Anotaciones
        Method metodoAnotado =
            MiClase.class.getMethod("metodoAnotado");
        Deprecated dep = metodoAnotado.getAnnotation(Deprecated.class);
        if (dep != null) {
            System.out.println("since: " + dep.since());
            System.out.println("forRemoval: " + dep.forRemoval());
        }

        // Arrays: obtener tipo de componente
        int[] arr = new int[10];
        System.out.println("Tipo array: " + arr.getClass().getComponentType());
    }
}

class MiClase {
    public List<String> getLista() {
        return List.of("A", "B");
    }

    public <T> void procesar(List<T> items) {}

    @Deprecated(since = "2.0", forRemoval = true)
    public void metodoAnotado() {}
}
```

11. RENDIMIENTO Y SEGURIDAD
===============================================================================

Reflection tiene implicaciones importantes que debes conocer:

```java
public class RendimientoReflection {
    public static void main(String[] args) throws Exception {
        int ITERACIONES = 10_000_000;

        // Llamada normal
        Persona p = new Persona("Test", 1);
        long inicio = System.nanoTime();

        for (int i = 0; i < ITERACIONES; i++) {
            p.toString();
        }

        long finNormal = System.nanoTime() - inicio;

        // Llamada con Reflection
        Method metodo = Persona.class.getMethod("toString");

        inicio = System.nanoTime();
        for (int i = 0; i < ITERACIONES; i++) {
            metodo.invoke(p);
        }
        long finReflection = System.nanoTime() - inicio;

        System.out.println("Rendimiento (" + ITERACIONES + " iteraciones):");
        System.out.println("  Llamada normal:    " + finNormal / 1_000_000 + " ms");
        System.out.println("  Reflection:        " + finReflection / 1_000_000 + " ms");
        System.out.println("  Diferencia:        " +
            (finReflection / (double) finNormal) + "x más lento");

        // setAccessible(true) acelera
        metodo.setAccessible(true);
        inicio = System.nanoTime();
        for (int i = 0; i < ITERACIONES; i++) {
            metodo.invoke(p);
        }
        long finReflectionAcc = System.nanoTime() - inicio;
        System.out.println("\nCon setAccessible(true): " +
            finReflectionAcc / 1_000_000 + " ms");
    }
}
```

Reglas de seguridad y buenas prácticas:
1. Reflection es más lento: 2-50x dependiendo de la operación.
2. setAccessible(true) puede violar encapsulación.
3. Reflection puede romper singletons, inmutabilidad y otras garantías.
4. SecurityManager puede restringir Reflection.
5. Prefiere MethodHandle (moderno) para mejor rendimiento.
6. Cachea los objetos Method/Field si los usas repetidamente.
7. No uses Reflection donde las interfaces y la herencia son suficientes.

12. METHODHANDLE (ALTERNATIVA MODERNA)
===============================================================================

Desde Java 7, MethodHandle ofrece una alternativa más rápida a Reflection.

```java
import java.lang.invoke.*;

public class MethodHandleDemo {
    public static void main(String[] args) throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // Obtener MethodHandle para toString
        MethodHandle toStringMH = lookup.findVirtual(
            Persona.class, "toString",
            MethodType.methodType(String.class));

        Persona p = new Persona("Ana", 30);

        // Invocar
        String resultado = (String) toStringMH.invoke(p);
        System.out.println("MethodHandle toString: " + resultado);

        // MethodHandle para constructor
        MethodHandle constructorMH = lookup.findConstructor(
            Persona.class,
            MethodType.methodType(void.class, String.class, int.class));

        Persona p2 = (Persona) constructorMH.invoke("Luis", 40);
        System.out.println("MethodHandle constructor: " + p2);

        // MethodHandle para método estático
        MethodHandle estaticoMH = lookup.findStatic(
            Utilidades.class, "saludar",
            MethodType.methodType(String.class, String.class));

        String saludo = (String) estaticoMH.invoke("Mundo");
        System.out.println("MethodHandle estático: " + saludo);

        // MethodHandle para campo
        MethodHandle getterMH = lookup.findGetter(
            Persona.class, "nombre", String.class);
        String nombre = (String) getterMH.invoke(p);
        System.out.println("MethodHandle getter: " + nombre);

        // Rendimiento: MethodHandle es casi tan rápido como llamada directa
        int ITERACIONES = 10_000_000;
        long inicio = System.nanoTime();
        for (int i = 0; i < ITERACIONES; i++) {
            toStringMH.invoke(p);
        }
        long fin = System.nanoTime() - inicio;
        System.out.println("MethodHandle (" + ITERACIONES + "): " +
            fin / 1_000_000 + " ms");
    }
}
```

RESUMEN DEL CAPÍTULO
===============================================================================

En este capítulo hemos aprendido:

• Reflection permite inspeccionar y manipular clases, métodos, campos y
  constructores en tiempo de ejecución.

• Class es la puerta de entrada. Se obtiene con .class, getClass() o
  Class.forName().

• getMethods(), getFields(), getConstructors() inspeccionan miembros
  públicos. getDeclaredMethods() etc. inspeccionan todos los miembros.

• Crear instancias: Constructor.newInstance() (preferido sobre
  Class.newInstance() deprecado).

• Invocar métodos: Method.invoke(objeto, args). Los métodos privados
  requieren setAccessible(true).

• Acceder a campos: Field.get(objeto), Field.set(objeto, valor).

• Proxy dinámico: Proxy.newProxyInstance() con InvocationHandler para
  interceptar llamadas a interfaces.

• MethodHandle (Java 7+) es una alternativa más rápida y moderna que
  Reflection pura.

• Reflection tiene costo de rendimiento y puede romper encapsulación.
  Úsala con cuidado y solo cuando sea necesaria.

PREGUNTAS FRECUENTES
===============================================================================

❓ **¿Reflection es seguro?**
Reflection puede romper la encapsulación (acceder a campos privados,
invocar métodos privados). En entornos controlados es seguro, pero en
código que maneja datos sensibles debe usarse con cuidado.

❓ **¿Por qué Class.newInstance() está deprecado?**
Porque no propaga las excepciones del constructor correctamente (envuelve
toda excepción en InstantiationException). Usa Constructor.newInstance().

❓ **¿setAccessible(true) siempre funciona?**
No. Si hay un SecurityManager configurado, puede denegar el permiso.
En entornos normales (sin SecurityManager), funciona siempre.

❓ **¿Cuándo NO debo usar Reflection?**
Cuando puedes lograr lo mismo con polimorfismo, interfaces o genéricos.
Reflection añade complejidad y reduce el rendimiento.

❓ **¿Qué es InvocationTargetException?**
Es una excepción que envuelve la excepción real lanzada por el método
invocado. Usa getCause() para obtener la excepción original.

❓ **¿MethodHandle reemplaza a Reflection?**
No completamente. MethodHandle es más rápido y moderno, pero Reflection
ofrece más funcionalidad de inspección. Son complementarios.

DESAFÍO DEL ARQUITECTO
===============================================================================

🏗️ **Framework ORM Simple (Mini-Hibernate)**

Diseña e implementa un mini-framework ORM que use Reflection para mapear
objetos Java a tablas de base de datos, similar a Hibernate pero simple.

Requisitos:
1. Anotaciones: @Tabla(nombre), @Columna(nombre), @Id, @AutoIncrement.
2. El framework debe generar INSERT, SELECT, UPDATE, DELETE automáticamente.
3. Soporte para tipos básicos: String, int, long, double, boolean.
4. Mapeo de nombres: de "nombreCampo" a "nombre_campo" (camelCase a snake_case).
5. Métodos genéricos: guardar(obj), buscar(Class, id), eliminar(obj), listarTodos(Class).

Implementa:
- Definición de anotaciones (@Retention(RUNTIME), @Target).
- Clase EntityMapper que genere SQL usando Reflection.
- Clase SimpleORM que integre con JDBC.
- Una entidad de ejemplo (Usuario, Producto) y pruebas funcionales.

Este desafío te mostrará cómo funcionan internamente frameworks como
Hibernate y te dará una comprensión profunda de Reflection aplicada.
