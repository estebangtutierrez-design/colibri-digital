----------------------------------------------------------------------
  15.1  SINTAXIS DEL BUCLE do...while
----------------------------------------------------------------------

El bucle do...while tiene una sintaxis diferente a while: el cuerpo se
ejecuta primero y luego se evalúa la condición.

    do {
        // Código a ejecutar al menos una vez
    } while (condición);

El punto y coma al final es obligatorio. Es fácil olvidarlo.

Flujo de ejecución:

  1. Ejecutar el cuerpo del bucle.
  2. Evaluar la condición.
  3. Si es true, volver al paso 1.
  4. Si es false, continuar después del bucle.

Esto garantiza que el cuerpo se ejecute SIEMPRE al menos una vez,
independientemente de la condición.

Ejemplo básico:

    int i = 1;
    do {
        System.out.println("Iteración: " + i);
        i++;
    } while (i <= 5);

    System.out.println("Bucle terminado");

Salida:

    Iteración: 1
    Iteración: 2
    Iteración: 3
    Iteración: 4
    Iteración: 5
    Bucle terminado

----------------------------------------------------------------------
  15.2  DO...WHILE vs. WHILE
----------------------------------------------------------------------

La diferencia fundamental es cuándo se evalúa la condición:

  - while: condición al INICIO → puede ejecutarse 0 veces.
  - do...while: condición al FINAL → se ejecuta siempre al menos 1 vez.

Comparación visual:

    while (condición) {      do {
        // cuerpo               // cuerpo
    }                        } while (condición);

Ejemplo que muestra la diferencia:

    int x = 10;

    // while: condición false desde el inicio → NO se ejecuta
    while (x < 5) {
        System.out.println("While: " + x);
    }

    // do...while: se ejecuta UNA vez aunque la condición sea false
    do {
        System.out.println("Do-while: " + x);
    } while (x < 5);

Salida:

    Do-while: 10

El while no imprime nada porque la condición x < 5 es false desde el
inicio. El do...while imprime una vez antes de evaluar la condición.

¿Cuándo usar cada uno?

  Usa while cuando:
    - Puede que no necesites ejecutar el cuerpo (validación sin intento).
    - La condición debe verificarse rigurosamente antes de actuar.
    - Estás leyendo datos con centinela (while es más natural).

  Usa do...while cuando:
    - El cuerpo debe ejecutarse siempre al menos una vez.
    - Estás implementando un menú interactivo.
    - Necesitas mostrar algo antes de preguntar si continuar.
    - El usuario debe ver las opciones antes de decidir.

----------------------------------------------------------------------
  15.3  MENÚS INTERACTIVOS CON do...while
----------------------------------------------------------------------

El caso de uso más común de do...while son los menús interactivos. El
menú debe mostrarse al menos una vez, y luego repetirse hasta que el
usuario elija salir.

Estructura típica:

    int opcion;
    do {
        // Mostrar menú
        System.out.println("=== MENÚ ===");
        System.out.println("1. Opción 1");
        System.out.println("2. Opción 2");
        System.out.println("3. Salir");
        System.out.print("Seleccione: ");
        opcion = scanner.nextInt();

        // Procesar opción
        switch (opcion) {
            case 1 -> // acción 1
            case 2 -> // acción 2
            case 3 -> System.out.println("Saliendo...");
            default -> System.out.println("Opción inválida");
        }
    } while (opcion != 3);

Ventajas de do...while para menús:

  1. El menú se muestra siempre al menos una vez.
  2. El código de mostrar el menú está dentro del bucle.
  3. La condición de salida es clara y explícita.
  4. No hay duplicación de código (como en while que requiere
     mostrar el menú antes y dentro del bucle).

Ejemplo completo sin do...while (con while):

    // Con while: hay que duplicar la lectura
    System.out.println("=== MENÚ ===");
    System.out.println("1. Opción 1");
    System.out.println("2. Salir");
    System.out.print("Seleccione: ");
    opcion = scanner.nextInt();

    while (opcion != 2) {
        // procesar
        System.out.println("=== MENÚ ===");
        System.out.println("1. Opción 1");
        System.out.println("2. Salir");
        System.out.print("Seleccione: ");
        opcion = scanner.nextInt();
    }

Con do...while se evita esta duplicación.

----------------------------------------------------------------------
  15.4  VALIDACIÓN DE ENTRADA CON do...while
----------------------------------------------------------------------

Do...while es natural para validación de entrada porque el usuario debe
ingresar al menos un valor antes de verificar su validez.

    int edad;
    do {
        System.out.print("Ingrese su edad (0-150): ");
        edad = scanner.nextInt();
        if (edad < 0 || edad > 150) {
            System.out.println("Edad inválida. Intente de nuevo.");
        }
    } while (edad < 0 || edad > 150);

    System.out.println("Edad registrada: " + edad);

Comparación con while:

    // Con while: necesitas leer antes del bucle
    System.out.print("Ingrese su edad (0-150): ");
    edad = scanner.nextInt();
    while (edad < 0 || edad > 150) {
        System.out.print("Edad inválida. Ingrese de nuevo: ");
        edad = scanner.nextInt();
    }

    // Con do...while: más natural (lee al menos una vez)
    do {
        System.out.print("Ingrese su edad (0-150): ");
        edad = scanner.nextInt();
    } while (edad < 0 || edad > 150);

El do...while es más limpio para validación: no necesitas una lectura
separada antes del bucle.

Validación con mensaje de error personalizado:

    double nota;
    do {
        System.out.print("Ingrese la nota (0-100): ");
        nota = scanner.nextDouble();
        if (nota < 0 || nota > 100) {
            System.out.println("Error: la nota debe estar entre 0 y 100.");
        }
    } while (nota < 0 || nota > 100);

----------------------------------------------------------------------
  15.5  JUEGOS CON do...while
----------------------------------------------------------------------

Do...while es excelente para juegos donde el jugador debe hacer al menos
un intento, o donde se pregunta "¿Jugar de nuevo?" al final.

Ejemplo: juego de adivinanza con repetición.

    import java.util.Scanner;
    import java.util.Random;

    public class JuegoAdivinanzaRepetido {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            Random random = new Random();
            String jugarDeNuevo;

            do {
                int numeroSecreto = random.nextInt(100) + 1;
                int intento;
                int intentos = 0;

                System.out.println("\n=== ADIVINA EL NÚMERO (1-100) ===");

                do {
                    System.out.print("Intento " + (intentos + 1) + ": ");
                    intento = scanner.nextInt();
                    intentos++;

                    if (intento < numeroSecreto) {
                        System.out.println("El número es MAYOR.");
                    } else if (intento > numeroSecreto) {
                        System.out.println("El número es MENOR.");
                    }
                } while (intento != numeroSecreto);

                System.out.println("¡Correcto! Lo lograste en " + intentos + " intentos.");

                System.out.print("¿Jugar de nuevo? (s/n): ");
                jugarDeNuevo = scanner.next();

            } while (jugarDeNuevo.equalsIgnoreCase("s"));

            System.out.println("Gracias por jugar.");
            scanner.close();
        }
    }

Observa los dos do...while anidados:
  - El interior controla los intentos de adivinanza.
  - El exterior controla la repetición del juego completo.

----------------------------------------------------------------------
  15.6  DO...WHILE ANIDADOS
----------------------------------------------------------------------

Al igual que while y for, los do...while pueden anidarse.

Ejemplo: tabla de multiplicar con do...while anidados.

    int i = 1;
    do {
        int j = 1;
        do {
            System.out.printf("%4d", i * j);
            j++;
        } while (j <= 10);
        System.out.println();
        i++;
    } while (i <= 10);

Ejemplo: menú de menús (submenús con do...while).

    int opcionPrincipal;
    do {
        System.out.println("=== SISTEMA PRINCIPAL ===");
        System.out.println("1. Gestión de usuarios");
        System.out.println("2. Reportes");
        System.out.println("3. Salir");
        System.out.print("Seleccione: ");
        opcionPrincipal = scanner.nextInt();

        switch (opcionPrincipal) {
            case 1 -> {
                int opcionUsuarios;
                do {
                    System.out.println("--- GESTIÓN DE USUARIOS ---");
                    System.out.println("1. Crear usuario");
                    System.out.println("2. Editar usuario");
                    System.out.println("3. Eliminar usuario");
                    System.out.println("4. Volver al menú principal");
                    System.out.print("Seleccione: ");
                    opcionUsuarios = scanner.nextInt();

                    switch (opcionUsuarios) {
                        case 1 -> System.out.println("Usuario creado.");
                        case 2 -> System.out.println("Usuario editado.");
                        case 3 -> System.out.println("Usuario eliminado.");
                        case 4 -> System.out.println("Volviendo...");
                        default -> System.out.println("Opción inválida.");
                    }
                } while (opcionUsuarios != 4);
            }
            case 2 -> {
                System.out.println("Generando reportes...");
            }
            case 3 -> System.out.println("Saliendo del sistema...");
            default -> System.out.println("Opción inválida.");
        }
    } while (opcionPrincipal != 3);

----------------------------------------------------------------------
  15.7  PATRÓN DE REPETICIÓN CON CONFIRMACIÓN
----------------------------------------------------------------------

Un patrón común es "hacer algo, preguntar si desea continuar, repetir
si la respuesta es sí". Este patrón es natural con do...while.

    String respuesta;
    do {
        // Realizar la acción
        System.out.println("Procesando...");
        System.out.println("Operación completada.");

        // Preguntar si continuar
        System.out.print("¿Desea realizar otra operación? (s/n): ");
        respuesta = scanner.next();
    } while (respuesta.equalsIgnoreCase("s"));

    System.out.println("Programa terminado.");

Este patrón se usa en:
  - Procesamiento por lotes.
  - Entrada de múltiples registros.
  - Juegos por rondas.
  - Simulaciones iterativas.

Ejemplo 1: Cajero automático completo.

    import java.util.Scanner;

    public class CajeroCompleto {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            double saldo = 1000.0;
            int opcion;

            do {
                System.out.println("\n=== CAJERO AUTOMÁTICO ===");
                System.out.printf("Saldo disponible: $%.2f%n", saldo);
                System.out.println("1. Depositar");
                System.out.println("2. Retirar");
                System.out.println("3. Consultar saldo");
                System.out.println("4. Salir");
                System.out.print("Seleccione una opción: ");

                // Validar que sea entero
                while (!scanner.hasNextInt()) {
                    System.out.print("Por favor, ingrese un número: ");
                    scanner.next();
                }
                opcion = scanner.nextInt();

                switch (opcion) {
                    case 1 -> {
                        System.out.print("Monto a depositar: $");
                        double deposito = scanner.nextDouble();
                        while (deposito <= 0) {
                            System.out.print("Monto inválido. Ingrese un valor positivo: $");
                            deposito = scanner.nextDouble();
                        }
                        saldo += deposito;
                        System.out.printf("Depósito exitoso. Nuevo saldo: $%.2f%n", saldo);
                    }
                    case 2 -> {
                        System.out.print("Monto a retirar: $");
                        double retiro = scanner.nextDouble();
                        while (retiro <= 0 || retiro > saldo) {
                            if (retiro <= 0) {
                                System.out.print("Monto inválido. Ingrese un valor positivo: $");
                            } else {
                                System.out.printf("Saldo insuficiente. Máximo disponible: $%.2f%n", saldo);
                                System.out.print("Ingrese un monto menor: $");
                            }
                            retiro = scanner.nextDouble();
                        }
                        saldo -= retiro;
                        System.out.printf("Retiro exitoso. Nuevo saldo: $%.2f%n", saldo);
                    }
                    case 3 -> {
                        System.out.printf("Su saldo actual es: $%.2f%n", saldo);
                    }
                    case 4 -> {
                        System.out.println("Gracias por usar nuestro cajero.");
                    }
                    default -> {
                        System.out.println("Opción no válida. Intente de nuevo.");
                    }
                }
            } while (opcion != 4);

            scanner.close();
        }
    }

Ejemplo 2: Calculadora con repetición.

    import java.util.Scanner;

    public class CalculadoraRepetitiva {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            String continuar;

            System.out.println("=== CALCULADORA ===");

            do {
                System.out.print("Ingrese el primer número: ");
                double a = scanner.nextDouble();

                System.out.print("Ingrese el operador (+, -, *, /): ");
                char op = scanner.next().charAt(0);

                System.out.print("Ingrese el segundo número: ");
                double b = scanner.nextDouble();

                double resultado;
                boolean operacionValida = true;

                switch (op) {
                    case '+' -> resultado = a + b;
                    case '-' -> resultado = a - b;
                    case '*' -> resultado = a * b;
                    case '/' -> {
                        if (b == 0) {
                            System.out.println("Error: División entre cero.");
                            operacionValida = false;
                            resultado = 0;
                        } else {
                            resultado = a / b;
                        }
                    }
                    default -> {
                        System.out.println("Operador no válido.");
                        operacionValida = false;
                        resultado = 0;
                    }
                }

                if (operacionValida) {
                    System.out.printf("%.2f %c %.2f = %.2f%n", a, op, b, resultado);
                }

                System.out.print("¿Desea realizar otro cálculo? (s/n): ");
                continuar = scanner.next();

            } while (continuar.equalsIgnoreCase("s"));

            System.out.println("Calculadora cerrada.");
            scanner.close();
        }
    }

Ejemplo 3: Sistema de inicio de sesión.

    import java.util.Scanner;

    public class SistemaLogin {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);

            String usuarioCorrecto = "admin";
            String passwordCorrecto = "1234";
            boolean autenticado = false;
            int intentos = 0;
            final int MAX_INTENTOS = 3;

            System.out.println("=== SISTEMA DE INICIO DE SESIÓN ===");

            do {
                System.out.print("Usuario: ");
                String usuario = scanner.nextLine();

                System.out.print("Contraseña: ");
                String password = scanner.nextLine();

                if (usuario.equals(usuarioCorrecto) && password.equals(passwordCorrecto)) {
                    autenticado = true;
                    System.out.println("¡Bienvenido al sistema!");
                } else {
                    intentos++;
                    int restantes = MAX_INTENTOS - intentos;
                    if (restantes > 0) {
                        System.out.printf("Credenciales incorrectas. Le quedan %d intento(s).%n", restantes);
                    } else {
                        System.out.println("Ha excedido el número de intentos. Sistema bloqueado.");
                    }
                }
            } while (!autenticado && intentos < MAX_INTENTOS);

            if (autenticado) {
                // Menú principal (otro do...while)
                int opcion;
                do {
                    System.out.println("\n=== MENÚ PRINCIPAL ===");
                    System.out.println("1. Ver perfil");
                    System.out.println("2. Configuración");
                    System.out.println("3. Cerrar sesión");
                    System.out.print("Seleccione: ");
                    opcion = scanner.nextInt();
                    scanner.nextLine();

                    switch (opcion) {
                        case 1 -> System.out.println("Mostrando perfil del usuario...");
                        case 2 -> System.out.println("Abriendo configuración...");
                        case 3 -> System.out.println("Cerrando sesión...");
                        default -> System.out.println("Opción inválida.");
                    }
                } while (opcion != 3);
            }

            scanner.close();
        }
    }

Ejemplo 4: Generador de contraseñas (con repetición).

    import java.util.Scanner;
    import java.util.Random;

    public class GeneradorPasswords {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            Random random = new Random();
            String caracteres = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789!@#$%";
            String continuar;

            System.out.println("=== GENERADOR DE CONTRASEÑAS ===");

            do {
                int longitud;
                do {
                    System.out.print("Longitud de la contraseña (8-32): ");
                    longitud = scanner.nextInt();
                    if (longitud < 8 || longitud > 32) {
                        System.out.println("La longitud debe estar entre 8 y 32.");
                    }
                } while (longitud < 8 || longitud > 32);

                StringBuilder password = new StringBuilder();
                for (int i = 0; i < longitud; i++) {
                    int indice = random.nextInt(caracteres.length());
                    password.append(caracteres.charAt(indice));
                }

                System.out.println("Contraseña generada: " + password);

                System.out.print("¿Generar otra contraseña? (s/n): ");
                continuar = scanner.next();

            } while (continuar.equalsIgnoreCase("s"));

            System.out.println("Programa terminado.");
            scanner.close();
        }
    }

Ejemplo 5: Promedio de calificaciones con repetición.

    import java.util.Scanner;

    public class SistemaCalificaciones {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            String continuar;

            System.out.println("=== SISTEMA DE CALIFICACIONES ===");

            do {
                int numMaterias;
                do {
                    System.out.print("¿Cuántas materias tiene? (1-20): ");
                    numMaterias = scanner.nextInt();
                    if (numMaterias < 1 || numMaterias > 20) {
                        System.out.println("Número inválido (1-20).");
                    }
                } while (numMaterias < 1 || numMaterias > 20);

                double suma = 0;
                for (int i = 1; i <= numMaterias; i++) {
                    double nota;
                    do {
                        System.out.print("Nota de materia " + i + " (0-100): ");
                        nota = scanner.nextDouble();
                        if (nota < 0 || nota > 100) {
                            System.out.println("Nota inválida (0-100).");
                        }
                    } while (nota < 0 || nota > 100);
                    suma += nota;
                }

                double promedio = suma / numMaterias;
                System.out.printf("%nSu promedio es: %.2f%n", promedio);

                String calificacion = promedio >= 90 ? "Excelente" :
                                     promedio >= 80 ? "Notable" :
                                     promedio >= 70 ? "Bueno" :
                                     promedio >= 60 ? "Suficiente" :
                                     "Reprobado";
                System.out.println("Calificación: " + calificacion);

                System.out.print("%n¿Desea registrar otro alumno? (s/n): ");
                continuar = scanner.next();

            } while (continuar.equalsIgnoreCase("s"));

            System.out.println("Sistema finalizado.");
            scanner.close();
        }
    }

    }

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • do...while evalúa la condición DESPUÉS de ejecutar el cuerpo,
    garantizando al menos una ejecución.
  • La sintaxis es: do { ... } while (condición); (punto y coma
    obligatorio al final).
  • La diferencia fundamental con while es que while evalúa antes
    (puede ejecutarse 0 veces) y do...while evalúa después (mínimo
    1 ejecución).
  • El uso más común de do...while son los menús interactivos, donde
    el menú debe mostrarse al menos una vez.
  • También es excelente para validación de entrada (el usuario debe
    ingresar al menos un valor).
  • Los juegos con repetición ("¿Jugar de nuevo?") se implementan
    naturalmente con do...while.
  • do...while puede anidarse para crear menús con submenús.
  • El patrón de confirmación (hacer, preguntar, repetir) es
    idiomático con do...while.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Cuándo debo usar do...while en lugar de while?
  R: Cuando el cuerpo del bucle debe ejecutarse al menos una vez sin
  importar la condición. Los casos típicos son menús interactivos y
  validación de entrada.

  P: ¿Es obligatorio el punto y coma al final del do...while?
  R: Sí. Es un error de sintaxis muy común olvidarlo. El compilador
  marcará un error si falta el punto y coma.

  P: ¿Puedo usar break y continue en do...while?
  R: Sí, break y continue funcionan igual que en while y for. break
  sale del bucle, continue salta a la evaluación de la condición.

  P: ¿Puedo tener un do...while infinito?
  R: Sí, con do { ... } while (true); que se ejecuta para siempre a
  menos que haya un break interno.

  P: ¿El do...while es más eficiente que while?
  R: No hay diferencia de rendimiento significativa. La diferencia es
  puramente semántica y de estilo.

  P: ¿Puedo declarar variables en la condición del do...while?
  R: No, la condición solo acepta una expresión booleana. La variable
  debe declararse antes.

  P: ¿Se puede usar do...while con Scanner para validar tipos?
  R: Sí, combinado con hasNextInt()/hasNextDouble() es muy efectivo.

  P: ¿Hay alguna situación donde while no pueda reemplazar a do...while?
  R: Técnicamente, todo do...while se puede reescribir como while
  duplicando el cuerpo antes del bucle. Pero do...while evita esa
  duplicación, resultando en código más limpio.

Has completado el Capítulo 15 del JAVA MASTER BOOK. El bucle do...while
completa tu arsenal de estructuras de repetición. Ahora tienes tres
herramientas poderosas: while (tal vez no se ejecute), for (contador
preciso), y do...while (al menos una vez).

El do...while es el rey indiscutible de los menús interactivos. Su
garantía de ejecución mínima lo hace perfecto para interfaces de usuario
basadas en texto, juegos con repetición, y validación de entrada.
