Bienvenido al Capítulo 16 del JAVA MASTER BOOK. Has dominado los tres
bucles de Java: while, for y do...while. Ahora es momento de aprender
cómo controlarlos con precisión quirúrgica.

Break y continue son sentencias que modifican el flujo normal de un bucle.
Break termina el bucle inmediatamente. Continue salta a la siguiente
iteración. Y cuando se combinan con etiquetas (labels), pueden controlar
bucles externos desde bucles internos, una técnica avanzada pero
poderosa.

En este capítulo exploraremos desde el break simple en bucles, pasando
por continue, hasta los labeled break/continue que pueden interrumpir
bucles anidados desde cualquier nivel.

¡Prepárate para afinar el control de tus bucles!

----------------------------------------------------------------------
  16.1  LA SENTENCIA break EN BUCLES
----------------------------------------------------------------------

break termina la ejecución del bucle más interno en el que se encuentra.
El control se transfiere a la primera instrucción después del bucle.

Sintaxis:

    while (condición) {
        if (condiciónDeSalida) {
            break;
        }
        // código
    }

Ejemplo: buscar un número en una secuencia.

    int[] numeros = {10, 20, 30, 40, 50, 60};
    int busqueda = 40;
    boolean encontrado = false;

    for (int i = 0; i < numeros.length; i++) {
        if (numeros[i] == busqueda) {
            encontrado = true;
            System.out.println("Encontrado en índice: " + i);
            break; // Sale del bucle al encontrar
        }
    }

    if (!encontrado) {
        System.out.println("No encontrado.");
    }

Sin break, el bucle seguiría iterando aunque ya hubiera encontrado el
valor. Con break, optimizamos la búsqueda.

Ejemplo: leer contraseña con 3 intentos (break temprano).

    import java.util.Scanner;

    public class LoginConBreak {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            String passwordCorrecto = "secreto";
            int maxIntentos = 3;

            for (int i = 1; i <= maxIntentos; i++) {
                System.out.print("Intento " + i + " - Contraseña: ");
                String password = scanner.nextLine();

                if (password.equals(passwordCorrecto)) {
                    System.out.println("¡Acceso concedido!");
                    break; // Sale del bucle si acierta
                }

                if (i < maxIntentos) {
                    System.out.println("Contraseña incorrecta. Te quedan "
                            + (maxIntentos - i) + " intentos.");
                } else {
                    System.out.println("Has agotado los intentos. Acceso denegado.");
                }
            }

            scanner.close();
        }
    }

break también funciona en while y do...while:

    int i = 0;
    while (true) { // Bucle infinito intencional
        System.out.println("Iteración: " + i);
        i++;
        if (i >= 5) {
            break; // Salir después de 5 iteraciones
        }
    }

----------------------------------------------------------------------
  16.2  break EN switch (REPASO)
----------------------------------------------------------------------

Ya vimos break en switch en el capítulo 12. Es importante notar que en
switch, break termina el case, no el bucle que lo contiene.

    for (int i = 1; i <= 5; i++) {
        switch (i) {
            case 1:
                System.out.println("Uno");
                break; // Sale del switch, no del for
            case 2:
                System.out.println("Dos");
                break;
            default:
                System.out.println("Otro: " + i);
        }
    }

En este caso, break sale del switch y continúa con la siguiente
iteración del for. Si quisieras salir del for desde dentro del switch,
necesitarías una etiqueta (labeled break).

----------------------------------------------------------------------
  16.3  LA SENTENCIA continue
----------------------------------------------------------------------

continue salta el resto del cuerpo del bucle y pasa a la siguiente
iteración. En while y do...while, continue transfiere a la evaluación
de la condición. En for, continue transfiere a la actualización y luego
a la condición.

Sintaxis:

    for (int i = 0; i < 10; i++) {
        if (condiciónDeSalto) {
            continue;
        }
        // Código que se salta si continue se ejecuta
    }

Ejemplo: imprimir solo números pares usando continue.

    for (int i = 1; i <= 10; i++) {
        if (i % 2 != 0) {
            continue; // Salta impares
        }
        System.out.println("Par: " + i);
    }

El código anterior imprime: Par: 2, Par: 4, Par: 6, Par: 8, Par: 10

Ejemplo: procesar solo calificaciones aprobatorias.

    double[] calificaciones = {85, 45, 92, 38, 76, 55, 90};

    for (double nota : calificaciones) {
        if (nota < 60) {
            System.out.println("Reprobado: " + nota + " (no procesado)");
            continue;
        }
        // Solo las aprobatorias pasan aquí
        System.out.println("Aprobado: " + nota + " - ¡Felicidades!");
    }

continue en un while con actualización al final:

    int i = 0;
    while (i < 10) {
        i++;
        if (i % 2 == 0) {
            continue; // Vuelve a evaluar la condición
        }
        System.out.println("i = " + i);
    }

Nota: en while, si continue se ejecuta antes de la actualización, la
variable de control no se actualiza y el bucle puede volverse infinito.

    int i = 0;
    while (i < 10) {
        if (i == 5) {
            continue; // ¡PELIGRO!: i nunca se incrementa, bucle infinito
        }
        System.out.println(i);
        i++;
    }

Para evitar esto, coloca la actualización antes del continue, o usa for
(que actualiza automáticamente).

----------------------------------------------------------------------
  16.4  break vs. continue
----------------------------------------------------------------------

Diferencia fundamental:

  - break: TERMINA el bucle por completo.
  - continue: SALTA a la SIGUIENTE ITERACIÓN del bucle.

Visualmente:

    for (int i = 1; i <= 5; i++) {
        if (i == 3) {
            break;   // i=3 → termina todo
        }
        System.out.print(i + " ");
    }
    // Imprime: 1 2

    for (int i = 1; i <= 5; i++) {
        if (i == 3) {
            continue; // i=3 → salta esta iteración
        }
        System.out.print(i + " ");
    }
    // Imprime: 1 2 4 5

Comparación:

  Característica     | break          | continue
  -------------------|----------------|-------------------
  Efecto             | Termina bucle  | Salta iteración
  Siguiente ejecución| Fuera del bucle| Próxima iteración
  En bucle anidado   | Sale del interno| Continúa externo
  Con etiquetas      | Sale al label  | Continua en label
  En switch          | Termina case   | (no aplica)

----------------------------------------------------------------------
  16.5  LABELED BREAK (break CON ETIQUETA)
----------------------------------------------------------------------

Una etiqueta (label) es un identificador seguido de dos puntos que se
coloca antes de un bucle. labeled break permite salir de un bucle
externo desde dentro de un bucle interno.

Sintaxis:

    externo: for (int i = 0; i < 5; i++) {
        interno: for (int j = 0; j < 5; j++) {
            if (condición) {
                break externo; // Sale del bucle externo
            }
        }
    }

Ejemplo: buscar un valor en una matriz 2D.

    int[][] matriz = {
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9}
    };

    int busqueda = 5;
    boolean encontrado = false;

    buscar: for (int i = 0; i < matriz.length; i++) {
        for (int j = 0; j < matriz[i].length; j++) {
            if (matriz[i][j] == busqueda) {
                System.out.println("Encontrado " + busqueda
                        + " en [" + i + "][" + j + "]");
                encontrado = true;
                break buscar; // Sale de AMBOS bucles
            }
        }
    }

    if (!encontrado) {
        System.out.println("No encontrado.");
    }

Sin labeled break, tendríamos que usar una variable flag:

    boolean encontrado = false;
    for (int i = 0; i < matriz.length && !encontrado; i++) {
        for (int j = 0; j < matriz[i].length && !encontrado; j++) {
            if (matriz[i][j] == busqueda) {
                System.out.println("Encontrado en [" + i + "][" + j + "]");
                encontrado = true;
            }
        }
    }

El labeled break es más legible en estos casos.

Ejemplo: procesar pedidos hasta encontrar una bandera de parada.

    pedidos: for (int dia = 1; dia <= 7; dia++) {
        System.out.println("Día " + dia + ":");
        for (int pedido = 1; pedido <= 10; pedido++) {
            if (dia == 5 && pedido == 3) {
                System.out.println("  ¡Alto! Parada general.");
                break pedidos;
            }
            System.out.println("  Procesando pedido " + pedido);
        }
    }

----------------------------------------------------------------------
  16.6  LABELED CONTINUE (continue CON ETIQUETA)
----------------------------------------------------------------------

labeled continue salta a la siguiente iteración del bucle etiquetado,
no del bucle más interno.

Sintaxis:

    externo: for (int i = 0; i < 5; i++) {
        for (int j = 0; j < 5; j++) {
            if (condición) {
                continue externo; // Continúa con la siguiente i
            }
        }
    }

Ejemplo: procesar solo filas que cumplan una condición.

    int[][] datos = {
        {1, -2, 3},
        {-4, 5, -6},
        {7, 8, 9},
        {-1, -2, -3}
    };

    filas: for (int i = 0; i < datos.length; i++) {
        for (int j = 0; j < datos[i].length; j++) {
            if (datos[i][j] < 0) {
                System.out.println("Fila " + i + " tiene negativos. Saltando...");
                continue filas; // Salta toda la fila
            }
        }
        // Solo llegamos aquí si la fila no tiene negativos
        System.out.print("Fila " + i + " (todos positivos): ");
        for (int j = 0; j < datos[i].length; j++) {
            System.out.print(datos[i][j] + " ");
        }
        System.out.println();
    }

Salida:

    Fila 0 tiene negativos. Saltando...
    Fila 1 tiene negativos. Saltando...
    Fila 2 (todos positivos): 7 8 9
    Fila 3 tiene negativos. Saltando...

Otro ejemplo: calendario, saltar fines de semana (simplificado).

    dias: for (int mes = 1; mes <= 12; mes++) {
        for (int dia = 1; dia <= 30; dia++) {
            if (dia == 29 || dia == 30) {
                // Simulamos que 29 y 30 son fines de semana
                continue dias; // Salta al siguiente mes
            }
            System.out.println("Mes " + mes + ", Día " + dia + ": laboral");
        }
    }

----------------------------------------------------------------------
  16.7  PATRONES Y BUENAS PRÁCTICAS
----------------------------------------------------------------------

Patrón 1: Búsqueda con break.

    int[] datos = {10, 20, 30, 40, 50};
    int objetivo = 30;
    int indice = -1;

    for (int i = 0; i < datos.length; i++) {
        if (datos[i] == objetivo) {
            indice = i;
            break;
        }
    }

    System.out.println(indice != -1 ? "Encontrado en: " + indice : "No encontrado");

Patrón 2: Filtrado con continue.

    for (int i = 0; i < 100; i++) {
        if (i % 5 != 0) continue; // Solo múltiplos de 5
        if (i % 3 != 0) continue; // Solo múltiplos de 3
        System.out.println(i + " es múltiplo de 3 y 5");
    }

Patrón 3: continue en validación de datos.

    for (String linea : lineas) {
        if (linea.isEmpty() || linea.startsWith("#")) {
            continue; // Saltar líneas vacías o comentarios
        }
        procesarLinea(linea);
    }

Patrón 4: Menú con break para salir.

    while (true) {
        mostrarMenu();
        int opcion = scanner.nextInt();
        if (opcion == 0) {
            System.out.println("Saliendo...");
            break;
        }
        procesarOpcion(opcion);
    }

Buenas prácticas:

  1. No abuses de break/continue. A veces un if/else es más claro.
  2. Evita labeled break/continue si una variable flag funciona bien.
  3. No uses continue en while si la actualización está después.
  4. labeled break suele ser más claro que múltiples condiciones.
  5. break en switch es normal; break en bucles debe tener propósito claro.

Malas prácticas:

    // MAL: lógica confusa con múltiples breaks
    for (int i = 0; i < 10; i++) {
        if (i == 3) break;
        for (int j = 0; j < 10; j++) {
            if (j == 5) break;
            if (i + j == 7) break;
            System.out.println(i + "," + j);
        }
    }

    // MEJOR: refactorizar a métodos con nombres descriptivos
    procesarDatos();
    // ... donde procesarDatos() contiene la lógica

Ejemplo 1: Encontrar el primer número primo en un rango.

    public class PrimerPrimo {
        public static void main(String[] args) {
            int limite = 100;

            encontrar: for (int n = 2; n <= limite; n++) {
                for (int i = 2; i <= Math.sqrt(n); i++) {
                    if (n % i == 0) {
                        continue encontrar; // No es primo, siguiente número
                    }
                }
                System.out.println("El primer primo es: " + n);
                break; // Encontrado, salimos
            }
        }
    }

Ejemplo 2: Validar contraseña con reglas (continue).

    import java.util.Scanner;

    public class ValidadorPasswordReglas {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            String password;

            while (true) {
                System.out.print("Ingrese contraseña (8+ chars, mayúsc, minúsc, dígito): ");
                password = scanner.nextLine();

                if (password.length() < 8) {
                    System.out.println("Muy corta.");
                    continue;
                }

                boolean mayuscula = false;
                boolean minuscula = false;
                boolean digito = false;

                for (char c : password.toCharArray()) {
                    if (Character.isUpperCase(c)) mayuscula = true;
                    if (Character.isLowerCase(c)) minuscula = true;
                    if (Character.isDigit(c)) digito = true;
                }

                if (!mayuscula) {
                    System.out.println("Falta mayúscula.");
                    continue;
                }
                if (!minuscula) {
                    System.out.println("Falta minúscula.");
                    continue;
                }
                if (!digito) {
                    System.out.println("Falta dígito.");
                    continue;
                }

                System.out.println("¡Contraseña válida!");
                break;
            }

            scanner.close();
        }
    }

Ejemplo 3: Procesamiento de matriz con labeled break.

    public class ProcesamientoMatriz {
        public static void main(String[] args) {
            int[][] ventas = {
                {120, 80, 200, 150},
                {300, 250, 0, 180},
                {90, 110, 130, 0},
                {400, 350, 300, 250}
            };

            // Buscar el primer mes con 0 ventas (dato faltante)
            buscarCero: for (int i = 0; i < ventas.length; i++) {
                for (int j = 0; j < ventas[i].length; j++) {
                    if (ventas[i][j] == 0) {
                        System.out.println("Dato faltante encontrado en "
                                + "vendedor " + (i + 1)
                                + ", mes " + (j + 1));
                        break buscarCero;
                    }
                }
            }

            // Procesar solo vendedores con todas las ventas positivas
            System.out.println("\nVendedores completos:");
            vendedores: for (int i = 0; i < ventas.length; i++) {
                for (int j = 0; j < ventas[i].length; j++) {
                    if (ventas[i][j] == 0) {
                        continue vendedores; // Vendedor con datos faltantes
                    }
                }
                int total = 0;
                for (int venta : ventas[i]) {
                    total += venta;
                }
                System.out.println("Vendedor " + (i + 1) + ": " + total);
            }
        }
    }

Ejemplo 4: Juego de memoria numérica.

    import java.util.Scanner;
    import java.util.Random;

    public class JuegoMemoria {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);
            Random random = new Random();

            juego: while (true) {
                int numeroSecreto = random.nextInt(100) + 1;
                System.out.println("\n--- NUEVA RONDA ---");
                System.out.println("Adivina el número (1-100). Tienes 5 intentos.");

                for (int intento = 1; intento <= 5; intento++) {
                    System.out.print("Intento " + intento + ": ");
                    int guess = scanner.nextInt();

                    if (guess == numeroSecreto) {
                        System.out.println("¡Correcto! Lo lograste en " + intento + " intentos.");
                        break; // Sale del for
                    }

                    if (intento == 5) {
                        System.out.println("Agotaste los intentos. Era: " + numeroSecreto);
                    }
                }

                System.out.print("¿Jugar de nuevo? (s/n): ");
                String respuesta = scanner.next();
                if (!respuesta.equalsIgnoreCase("s")) {
                    System.out.println("Gracias por jugar.");
                    break juego; // Sale del while externo
                }
            }

            scanner.close();
        }
    }

  16.9  CÓDIGO COMPLETO DE EJEMPLO

```java
import java.util.Scanner;

/**
 * Sistema de Análisis de Datos de Encuestas
 * Demostración completa de break, continue, labeled break y
 * labeled continue.
 */
public class AnalisisEncuestas {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("===========================================");
        System.out.println("  SISTEMA DE ANÁLISIS DE ENCUESTAS");
        System.out.println("===========================================");

        // Configuración
        System.out.print("Número de encuestados: ");
        int numEncuestados = scanner.nextInt();

        System.out.print("Número de preguntas por encuesta: ");
        int numPreguntas = scanner.nextInt();

        // Matriz de respuestas: [encuestado][pregunta]
        int[][] respuestas = new int[numEncuestados][numPreguntas];
        int contadorEncuestasValidas = 0;

        // --- CARGA DE DATOS CON VALIDACIÓN (continue y labeled break) ---
        System.out.println("\n--- CARGA DE RESPUESTAS ---");
        System.out.println("Ingrese respuestas (1-5 para cada pregunta).");

        encuestados: for (int i = 0; i < numEncuestados; i++) {
            System.out.println("\nEncuestado " + (i + 1) + ":");

            for (int j = 0; j < numPreguntas; j++) {
                System.out.print("  Pregunta " + (j + 1) + " (1-5): ");
                int respuesta = scanner.nextInt();

                // Validar rango
                if (respuesta < 1 || respuesta > 5) {
                    System.out.println("  Respuesta inválida. Encuesta descartada.");
                    continue encuestados; // Salta al siguiente encuestado
                }

                respuestas[i][j] = respuesta;
            }

            // Si llegamos aquí, todas las respuestas son válidas
            contadorEncuestasValidas++;

            // Verificar si ya tenemos suficientes encuestas válidas
            if (contadorEncuestasValidas >= 5) {
                System.out.println("\n¡Meta de 5 encuestas válidas alcanzada!");
                break encuestados; // Salimos del bucle externo
            }
        }

        // --- ANÁLISIS CON labeled continue ---
        System.out.println("\n===========================================");
        System.out.println("  ANÁLISIS DE RESULTADOS");
        System.out.println("===========================================");
        System.out.println("Encuestas válidas procesadas: " + contadorEncuestasValidas);

        if (contadorEncuestasValidas == 0) {
            System.out.println("No hay datos para analizar.");
            scanner.close();
            return;
        }

        // Calcular promedio por pregunta (saltando encuestas inválidas)
        System.out.println("\n--- PROMEDIO POR PREGUNTA ---");

        preguntas: for (int j = 0; j < numPreguntas; j++) {
            int suma = 0;
            int contador = 0;

            for (int i = 0; i < numEncuestados; i++) {
                int resp = respuestas[i][j];

                // Saltar encuestas no completadas (quedaron en 0)
                if (resp == 0) {
                    continue; // Salta esta iteración interna
                }

                suma += resp;
                contador++;
            }

            if (contador == 0) {
                System.out.println("  Pregunta " + (j + 1) + ": Sin datos");
                continue preguntas; // Salta a la siguiente pregunta
            }

            double promedio = (double) suma / contador;
            String nivel = promedio >= 4.0 ? "Alto" :
                          promedio >= 3.0 ? "Medio" : "Bajo";
            System.out.printf("  Pregunta %d: %.2f (%s)%n",
                    j + 1, promedio, nivel);
        }

        // --- DETECCIÓN DE PATRONES CON labeled break ---
        System.out.println("\n--- DETECCIÓN DE PATRONES ---");

        // Buscar un encuestado que haya dado 5 en todas las preguntas
        encontrarPerfecto: for (int i = 0; i < numEncuestados; i++) {
            for (int j = 0; j < numPreguntas; j++) {
                if (respuestas[i][j] != 5) {
                    continue encontrarPerfecto; // No es perfecto
                }
                if (respuestas[i][j] == 0) {
                    continue encontrarPerfecto; // Encuesta incompleta
                }
            }
            System.out.println("Encuestado " + (i + 1)
                    + ": ¡Puntuación perfecta en todas las preguntas!");
            break encontrarPerfecto; // Solo nos interesa el primero
        }

        // Buscar encuestados con alguna respuesta de 1 (muy insatisfecho)
        System.out.println("\nEncuestados con alguna respuesta de 1 (muy insatisfecho):");
        boolean hayInsatisfechos = false;

        for (int i = 0; i < numEncuestados; i++) {
            for (int j = 0; j < numPreguntas; j++) {
                if (respuestas[i][j] == 1) {
                    System.out.println("  Encuestado " + (i + 1)
                            + ", Pregunta " + (j + 1));
                    hayInsatisfechos = true;
                    break; // Sale del for interno, continúa con el externo
                }
            }
        }

        if (!hayInsatisfechos) {
            System.out.println("  No se encontraron respuestas de 1.");
            System.out.println("  ¡Todos los encuestados están satisfechos!");
        }

        // --- RESUMEN CON continue para filtrar ---
        System.out.println("\n--- RESUMEN DE SATISFACCIÓN ---");

        int totalRespuestas = 0;
        int sumaTotal = 0;

        for (int i = 0; i < numEncuestados; i++) {
            for (int j = 0; j < numPreguntas; j++) {
                if (respuestas[i][j] == 0) {
                    continue; // Ignorar respuestas no contestadas
                }
                sumaTotal += respuestas[i][j];
                totalRespuestas++;
            }
        }

        if (totalRespuestas > 0) {
            double promedioGlobal = (double) sumaTotal / totalRespuestas;
            System.out.printf("Promedio global: %.2f / 5.00%n", promedioGlobal);
            System.out.printf("Satisfacción general: %.1f%%%n",
                    (promedioGlobal / 5.0) * 100);

            String categoria = promedioGlobal >= 4.5 ? "Excelente" :
                              promedioGlobal >= 4.0 ? "Muy Buena" :
                              promedioGlobal >= 3.0 ? "Buena" :
                              promedioGlobal >= 2.0 ? "Regular" : "Mala";
            System.out.println("Categoría: " + categoria);
        }

        // --- EJEMPLO DE break EN while INFINITO ---
        System.out.println("\n--- OPCIÓN DE EXPORTACIÓN ---");
        while (true) {
            System.out.print("¿Exportar resultados a CSV? (s/n): ");
            String exportar = scanner.next();
            if (exportar.equalsIgnoreCase("s")) {
                System.out.println("Exportando... (simulación)");
                break; // Sale del while infinito
            } else if (exportar.equalsIgnoreCase("n")) {
                System.out.println("Exportación cancelada.");
                break;
            } else {
                System.out.println("Por favor, responda 's' o 'n'.");
                continue; // Vuelve a preguntar
            }
        }

        System.out.println("\n===========================================");
        System.out.println("  ANÁLISIS COMPLETADO");
        System.out.println("===========================================");
        scanner.close();
    }
}
```

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • break termina el bucle actual inmediatamente y transfiere el control
    a la siguiente instrucción después del bucle.
  • continue salta el resto del cuerpo del bucle y continúa con la
    siguiente iteración.
  • En for, continue ejecuta la actualización y luego la condición.
  • En while, continue vuelve directamente a la condición (cuidado con
    la actualización).
  • labeled break (break etiqueta) sale del bucle con la etiqueta
    especificada, útil para salir de bucles anidados.
  • labeled continue (continue etiqueta) salta a la siguiente iteración
    del bucle etiquetado, útil para continuar bucles externos.
  • break en switch es diferente de break en bucles (termina el case).
  • No abuses de break/continue; úsalos cuando mejoren la legibilidad.
  • labeled break/continue son herramientas avanzadas para situaciones
    específicas de bucles anidados.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Puedo usar break en un if?
  R: No directamente. break solo funciona dentro de bucles (for, while,
  do...while) o switch. Si necesitas salir de un if dentro de un bucle,
  break funciona porque está dentro del bucle.

  P: ¿Qué pasa si uso continue en un switch?
  R: continue no tiene significado en switch (a menos que el switch esté
  dentro de un bucle, en cuyo caso continue afecta al bucle, no al switch).

  P: ¿Es mala práctica usar break/continue?
  R: No, son herramientas legítimas cuando se usan apropiadamente. La
  mala práctica es abusar de ellos haciendo el código confuso. Úsalos
  cuando mejoren la claridad.

  P: ¿break y continue pueden usarse en bucles anidados?
  R: Sí. break sale del bucle más interno. continue salta a la siguiente
  iteración del bucle más interno. Para afectar bucles externos, usa
  etiquetas.

  P: ¿Son lo mismo break en switch y break en bucles?
  R: La sintaxis es la misma, pero el efecto es diferente: en switch,
  break termina el case; en bucles, termina el bucle. Si un switch está
  dentro de un bucle, break solo termina el switch.

  P: ¿Puedo usar múltiples etiquetas en un bucle?
  R: Sí, pero solo la etiqueta más cercana al bucle es útil. Las etiquetas
  adicionales se ignoran.

  P: ¿El labeled break es como goto?
  R: Es similar en el sentido de que transfiere el control, pero mucho
  más limitado. labeled break solo puede salir de bucles, no saltar a
  cualquier parte del código como goto.

  P: ¿continue funciona en un do...while igual que en while?
  R: Sí, pero en do...while la condición se evalúa después de ejecutar
  el cuerpo. Si continue se ejecuta, el control va a la condición.
