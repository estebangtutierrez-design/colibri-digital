----------------------------------------------------------------------
  12.1  LA SENTENCIA switch TRADICIONAL
----------------------------------------------------------------------

La sentencia switch evalúa una expresión y ejecuta el bloque de código
asociado al valor que coincide. Es una alternativa más limpia y legible
a largas cadenas de if/else if cuando se comparan valores constantes.

Sintaxis del switch tradicional:

    switch (expresión) {
        case valor1:
            // Código para valor1
            break;
        case valor2:
            // Código para valor2
            break;
        case valor3:
            // Código para valor3
            break;
        default:
            // Código para cualquier otro valor
            break;
    }

La expresión debe ser de un tipo compatible: byte, short, char, int,
String (desde Java 7), enum, o wrapper classes (Byte, Short, Character,
Integer).

Ejemplo básico con días de la semana:

    int dia = 3;
    String nombreDia;

    switch (dia) {
        case 1:
            nombreDia = "Lunes";
            break;
        case 2:
            nombreDia = "Martes";
            break;
        case 3:
            nombreDia = "Miércoles";
            break;
        case 4:
            nombreDia = "Jueves";
            break;
        case 5:
            nombreDia = "Viernes";
            break;
        case 6:
            nombreDia = "Sábado";
            break;
        case 7:
            nombreDia = "Domingo";
            break;
        default:
            nombreDia = "Día inválido";
            break;
    }

    System.out.println("El día " + dia + " es " + nombreDia);

Cada case representa un posible valor de la expresión. Cuando se encuentra
una coincidencia, se ejecuta el código desde ese case hasta que se
encuentra un break (o hasta el final del switch).


----------------------------------------------------------------------
  12.2  LA SENTENCIA break
----------------------------------------------------------------------

La sentencia break dentro de un switch termina la ejecución del bloque
switch y transfiere el control a la siguiente instrucción después del
switch. Sin break, ocurre el "fall-through": la ejecución continúa con
el siguiente case, incluso si no hay coincidencia.

Fall-through (pase de línea):

    int mes = 2;
    switch (mes) {
        case 1:
            System.out.println("Enero");
        case 2:
            System.out.println("Febrero");  // Se ejecuta
        case 3:
            System.out.println("Marzo");    // También se ejecuta (fall-through)
        case 4:
            System.out.println("Abril");    // También se ejecuta
            break;
        default:
            System.out.println("Mes inválido");
    }

Sin break en case 2 y case 3, la ejecución continúa hasta encontrar un
break o el final del switch. En este ejemplo, se imprimirían Febrero,
Marzo y Abril.

El fall-through puede ser intencional y útil en ciertos casos, como
cuando múltiples valores deben ejecutar el mismo código.

Fall-through intencional:

    int dia = 6;
    switch (dia) {
        case 1:
        case 2:
        case 3:
        case 4:
        case 5:
            System.out.println("Día laboral");
            break;
        case 6:
        case 7:
            System.out.println("Fin de semana");
            break;
        default:
            System.out.println("Día inválido");
            break;
    }

Aquí, los días 1-5 comparten el mismo código "Día laboral", y los días
6-7 comparten "Fin de semana". Este patrón es idiomático en Java y
evita repetición de código.

Siempre incluye break al final de cada case a menos que tengas una razón
explícita para permitir el fall-through. El fall-through accidental es
uno de los errores más comunes con switch.


----------------------------------------------------------------------
  12.3  LA CLAÚSULA default
----------------------------------------------------------------------

El caso default se ejecuta cuando ningún case coincide con el valor de
la expresión. Es opcional, pero se recomienda incluirlo siempre como
medida de seguridad.

    char calificacion = 'X';

    switch (calificacion) {
        case 'A':
            System.out.println("Excelente");
            break;
        case 'B':
            System.out.println("Notable");
            break;
        case 'C':
            System.out.println("Aprobado");
            break;
        case 'D':
            System.out.println("Deficiente");
            break;
        case 'F':
            System.out.println("Reprobado");
            break;
        default:
            System.out.println("Calificación no válida: " + calificacion);
            break;
    }

El default puede colocarse en cualquier posición dentro del switch,
pero por convención se coloca al final. Si se coloca en medio y no
tiene break, también causará fall-through.

Posición no convencional de default:

    switch (codigo) {
        case 1:
            System.out.println("Opción 1");
            break;
        default:
            System.out.println("Opción por defecto");
            // Sin break: ¡fall-through intencional!
        case 2:
            System.out.println("Opción 2");
            break;
    }

En este ejemplo, si codigo es 1, imprime "Opción 1". Si es cualquier
otro valor, ejecuta default y luego también case 2 (fall-through).
Esto es válido pero confuso y considerado mala práctica.


----------------------------------------------------------------------
  12.4  TIPOS DE DATOS SOPORTADOS
----------------------------------------------------------------------

El switch tradicional soporta los siguientes tipos:

  - Tipos primitivos enteros: byte, short, char, int
  - Wrappers de tipos enteros: Byte, Short, Character, Integer
  - String (desde Java 7)
  - Enumeraciones (enum)

NO soporta: long, float, double, boolean, o tipos de objetos arbitrarios.

Ejemplo con String:

    String comando = "salir";

    switch (comando) {
        case "ayuda":
            System.out.println("Comandos disponibles: iniciar, detener, salir");
            break;
        case "iniciar":
            System.out.println("Sistema iniciado.");
            break;
        case "detener":
            System.out.println("Sistema detenido.");
            break;
        case "salir":
            System.out.println("Saliendo del programa...");
            break;
        default:
            System.out.println("Comando no reconocido: " + comando);
            break;
    }

La comparación de String en switch usa internamente el método equals(),
por lo que distingue mayúsculas y minúsculas.

Ejemplo con enum:

    enum EstadoPedido { PENDIENTE, ENVIADO, ENTREGADO, CANCELADO }

    EstadoPedido estado = EstadoPedido.ENVIADO;

    switch (estado) {
        case PENDIENTE:
            System.out.println("El pedido está pendiente de procesar.");
            break;
        case ENVIADO:
            System.out.println("El pedido ha sido enviado.");
            break;
        case ENTREGADO:
            System.out.println("El pedido fue entregado.");
            break;
        case CANCELADO:
            System.out.println("El pedido fue cancelado.");
            break;
    }

Nota: en los enum, no se usa la sintaxis EstadoPedido.PENDIENTE dentro
del case; solo el nombre de la constante. Esto es porque Java ya sabe
el tipo del enum por la expresión del switch.


----------------------------------------------------------------------
  12.5  ENHANCED SWITCH (JAVA 14+)
----------------------------------------------------------------------

A partir de Java 14, se introdujo el "enhanced switch" o switch mejorado,
que ofrece dos grandes novedades:

  1. Sintaxis de flecha (->): más concisa y sin riesgo de fall-through.
  2. Switch como expresión: puede devolver un valor.

Sintaxis de flecha:

    switch (expresión) {
        case valor1 -> // código o expresión
        case valor2 -> {
            // múltiples líneas
        }
        case valor3 -> // código
        default -> // código
    }

Con la sintaxis de flecha, no se necesita break. Cada case es
independiente y no hay fall-through. La flecha indica que solo se
ejecutará el bloque de ese case.

Ejemplo con flecha:

    String dia = "LUNES";
    String tipo;

    switch (dia.toUpperCase()) {
        case "LUNES", "MARTES", "MIÉRCOLES", "JUEVES", "VIERNES" ->
            tipo = "Laboral";
        case "SÁBADO", "DOMINGO" ->
            tipo = "Fin de semana";
        default ->
            tipo = "Desconocido";
    }

    System.out.println(dia + " es día " + tipo);

Observa que múltiples valores pueden combinarse en una sola línea
con comas, simplificando el fall-through intencional.

Switch como expresión:

    String dia = "SÁBADO";
    String tipo = switch (dia.toUpperCase()) {
        case "LUNES", "MARTES", "MIÉRCOLES", "JUEVES", "VIERNES" -> "Laboral";
        case "SÁBADO", "DOMINGO" -> "Fin de semana";
        default -> "Desconocido";
    };

    System.out.println(tipo);

Aquí, switch devuelve un valor que se asigna directamente a la variable
tipo. El switch expresión debe ser exhaustivo: debe cubrir todos los
casos posibles, o incluir default.

Cuando el bloque de un case tiene varias líneas y necesitas devolver
un valor, se usa la palabra clave yield:

    int mes = 5;
    String estacion = switch (mes) {
        case 12, 1, 2 -> {
            String e = "Verano";
            System.out.println("Es " + e);
            yield e;
        }
        case 3, 4, 5 -> {
            String e = "Otoño";
            System.out.println("Es " + e);
            yield e;
        }
        case 6, 7, 8 -> {
            String e = "Invierno";
            System.out.println("Es " + e);
            yield e;
        }
        case 9, 10, 11 -> {
            String e = "Primavera";
            System.out.println("Es " + e);
            yield e;
        }
        default -> "Mes inválido";
    };

    System.out.println("Estación: " + estacion);

yield es similar a return, pero dentro de un switch expresión. Devuelve
el valor y termina la evaluación del switch.

Comparativa entre switch tradicional y enhanced:

    // Tradicional
    switch (nota) {
        case 10:
        case 9:
            System.out.println("Sobresaliente");
            break;
        case 8:
        case 7:
            System.out.println("Notable");
            break;
        case 6:
            System.out.println("Bien");
            break;
        case 5:
            System.out.println("Suficiente");
            break;
        default:
            System.out.println("Insuficiente");
            break;
    }

    // Enhanced con flecha y casos múltiples
    switch (nota) {
        case 10, 9  -> System.out.println("Sobresaliente");
        case 8, 7   -> System.out.println("Notable");
        case 6      -> System.out.println("Bien");
        case 5      -> System.out.println("Suficiente");
        default     -> System.out.println("Insuficiente");
    }

    // Enhanced como expresión
    String resultado = switch (nota) {
        case 10, 9  -> "Sobresaliente";
        case 8, 7   -> "Notable";
        case 6      -> "Bien";
        case 5      -> "Suficiente";
        default     -> "Insuficiente";
    };

El enhanced switch es más seguro (sin fall-through accidental), más
conciso (múltiples casos en una línea) y más expresivo (puede devolver
valores). Se recomienda usar enhanced switch siempre que trabajes con
Java 14 o superior.


----------------------------------------------------------------------
  12.6  switch ANIDADO
----------------------------------------------------------------------

Al igual que if, los switch pueden anidarse dentro de otros switch,
aunque la legibilidad puede verse afectada.

Ejemplo: menú de opciones con submenús.

    int opcionPrincipal = 2;

    switch (opcionPrincipal) {
        case 1:
            System.out.println("Gestión de usuarios");
            int subOpcion = 3;
            switch (subOpcion) {
                case 1 -> System.out.println("Crear usuario");
                case 2 -> System.out.println("Editar usuario");
                case 3 -> System.out.println("Eliminar usuario");
                default -> System.out.println("Subopción inválida");
            }
            break;
        case 2:
            System.out.println("Gestión de productos");
            break;
        case 3:
            System.out.println("Reportes");
            break;
        default:
            System.out.println("Opción inválida");
            break;
    }

En general, si necesitas switch anidados, considera refactorizar la
lógica en métodos separados para mantener el código limpio.


----------------------------------------------------------------------
  12.7  switch vs. if/else
----------------------------------------------------------------------

¿Cuándo usar switch y cuándo usar if/else? Esta es una pregunta común.

Usa switch cuando:

  - Comparas una sola variable contra múltiples valores constantes.
  - Los valores son conocidos en tiempo de compilación.
  - El número de casos es moderado (3 o más).
  - La lógica para cada caso es independiente.

Usa if/else cuando:

  - Las condiciones son rangos (ej., nota > 60).
  - Las condiciones son expresiones booleanas complejas.
  - Necesitas comparar objetos con lógica personalizada.
  - Las condiciones son de diferentes tipos o variables.
  - Necesitas else if con condiciones no relacionadas.

Ejemplo de cuándo NO usar switch:

    // Mal uso de switch (no es adecuado para rangos)
    int nota = 75;
    switch (nota / 10) {  // Truco para usar switch con rangos
        case 10:
        case 9:  // 90-100
            System.out.println("A");
            break;
        case 8:  // 80-89
            System.out.println("B");
            break;
        case 7:  // 70-79
            System.out.println("C");
            break;
        case 6:  // 60-69
            System.out.println("D");
            break;
        default:
            System.out.println("F");
            break;
    }

    // Mejor usar if/else para rangos
    if (nota >= 90) {
        System.out.println("A");
    } else if (nota >= 80) {
        System.out.println("B");
    } else if (nota >= 70) {
        System.out.println("C");
    } else if (nota >= 60) {
        System.out.println("D");
    } else {
        System.out.println("F");
    }


----------------------------------------------------------------------
  12.8  ERRORES COMUNES
----------------------------------------------------------------------

Error #1: Olvidar el break (fall-through accidental).

    switch (codigo) {
        case 1:
            System.out.println("Opción 1");
            // Olvidé el break
        case 2:
            System.out.println("Opción 2");
            break;
    }

Si codigo es 1, se imprimirán ambas líneas. Siempre verifica que cada
case tenga su break a menos que quieras fall-through intencional.

Error #2: Usar tipos no soportados.

    long numero = 100L;
    switch (numero) {  // ERROR: long no es soportado
        // ...
    }

Usa int o convierte el long a int si es necesario.

Error #3: Olvidar el default.

Aunque opcional, omitir default significa que valores inesperados
pasarán desapercibidos. Inclúyelo siempre por seguridad.

Error #4: Variables duplicadas en diferentes cases.

    switch (x) {
        case 1:
            int valor = 10;
            System.out.println(valor);
            break;
        case 2:
            int valor = 20;  // ERROR: variable ya definida
            System.out.println(valor);
            break;
    }

En el switch tradicional, todos los cases comparten el mismo ámbito.
Para evitarlo, usa bloques { } dentro de cada case:

    switch (x) {
        case 1: {
            int valor = 10;
            System.out.println(valor);
            break;
        }
        case 2: {
            int valor = 20;  // OK: ámbito separado por llaves
            System.out.println(valor);
            break;
        }
    }

Con enhanced switch y flecha, cada case tiene su propio ámbito, por lo
que este problema no ocurre.

Error #5: Comparación incorrecta con String.

    String color = "Rojo";
    switch (color) {
        case "rojo":  // No coincide por mayúsculas
            System.out.println("Color rojo");
            break;
    }

La comparación es sensible a mayúsculas. Usa toUpperCase() o
toLowerCase() en la expresión para evitar este problema.

Error #6: Usar switch con valores null.

    String texto = null;
    switch (texto) {  // Lanza NullPointerException
        case "hola":
            System.out.println("Hola");
            break;
    }

Si la expresión del switch puede ser null, verifícalo antes:

    if (texto == null) {
        System.out.println("Valor nulo");
    } else {
        switch (texto) {
            // cases...
        }
    }

Buenas prácticas:

  1. Siempre incluye un case default.
  2. Siempre incluye break en cada case (tradicional).
  3. Usa enhanced switch (->) cuando sea posible.
  4. Agrupa casos que comparten lógica.
  5. No uses switch para rangos o condiciones complejas.
  6. Verifica null antes del switch.
  7. Mantén cada case simple; si es complejo, extrae a un método.

Ejemplo 1: Calculadora de operaciones básicas.

    import java.util.Scanner;

    public class CalculadoraSwitch {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);

            System.out.print("Ingrese el primer número: ");
            double num1 = scanner.nextDouble();

            System.out.print("Ingrese el operador (+, -, *, /): ");
            char operador = scanner.next().charAt(0);

            System.out.print("Ingrese el segundo número: ");
            double num2 = scanner.nextDouble();

            double resultado;

            switch (operador) {
                case '+' -> resultado = num1 + num2;
                case '-' -> resultado = num1 - num2;
                case '*' -> resultado = num1 * num2;
                case '/' -> {
                    if (num2 == 0) {
                        System.out.println("Error: División entre cero.");
                        return;
                    }
                    resultado = num1 / num2;
                }
                default -> {
                    System.out.println("Operador no válido.");
                    return;
                }
            }

            System.out.printf("%.2f %c %.2f = %.2f%n", num1, operador, num2, resultado);
            scanner.close();
        }
    }

Ejemplo 2: Convertidor de unidades de temperatura.

    import java.util.Scanner;

    public class ConvertidorTemperatura {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);

            System.out.println("CONVERTIDOR DE TEMPERATURA");
            System.out.println("1. Celsius a Fahrenheit");
            System.out.println("2. Fahrenheit a Celsius");
            System.out.println("3. Celsius a Kelvin");
            System.out.println("4. Kelvin a Celsius");
            System.out.print("Seleccione una opción: ");
            int opcion = scanner.nextInt();

            System.out.print("Ingrese la temperatura: ");
            double temperatura = scanner.nextDouble();

            double resultado = switch (opcion) {
                case 1 -> (temperatura * 9 / 5) + 32;
                case 2 -> (temperatura - 32) * 5 / 9;
                case 3 -> temperatura + 273.15;
                case 4 -> temperatura - 273.15;
                default -> Double.NaN;
            };

            if (Double.isNaN(resultado)) {
                System.out.println("Opción no válida.");
            } else {
                String[] nombres = {"", "°F", "°C", "K", "°C"};
                System.out.printf("Resultado: %.2f %s%n", resultado, nombres[opcion]);
            }

            scanner.close();
        }
    }

Ejemplo 3: Sistema de gestión de pedidos con enum.

    public class GestionPedidos {

        enum EstadoPedido {
            PENDIENTE, PAGADO, PREPARANDO, ENVIADO, ENTREGADO, CANCELADO
        }

        public static void main(String[] args) {

            EstadoPedido estado = EstadoPedido.PREPARANDO;

            String mensaje = switch (estado) {
                case PENDIENTE -> "El pedido está pendiente de pago.";
                case PAGADO -> "El pago ha sido recibido. Preparando pedido...";
                case PREPARANDO -> "El pedido está siendo preparado.";
                case ENVIADO -> "El pedido ha sido enviado al cliente.";
                case ENTREGADO -> "El pedido fue entregado con éxito.";
                case CANCELADO -> "El pedido fue cancelado.";
            };

            System.out.println("Estado: " + estado);
            System.out.println("Mensaje: " + mensaje);

            // Acción según el estado
            switch (estado) {
                case PENDIENTE, PAGADO, PREPARANDO ->
                    System.out.println("El pedido está en proceso.");
                case ENVIADO, ENTREGADO ->
                    System.out.println("El pedido está completado.");
                case CANCELADO ->
                    System.out.println("El pedido fue cancelado.");
            }
        }
    }

Ejemplo 4: Menú de restaurante.

    import java.util.Scanner;

    public class MenuRestaurante {
        public static void main(String[] args) {
            Scanner scanner = new Scanner(System.in);

            System.out.println("=== MENÚ DEL RESTAURANTE ===");
            System.out.println("1. Entradas");
            System.out.println("2. Platos principales");
            System.out.println("3. Bebidas");
            System.out.println("4. Postres");
            System.out.print("Seleccione una categoría: ");
            int categoria = scanner.nextInt();

            String seleccion;
            double precio = 0;

            switch (categoria) {
                case 1 -> {
                    System.out.println("--- ENTRADAS ---");
                    System.out.println("1. Ensalada César");
                    System.out.println("2. Sopa del día");
                    System.out.println("3. Bruschettas");
                    System.out.print("Elija una opción: ");
                    int opcion = scanner.nextInt();
                    seleccion = switch (opcion) {
                        case 1 -> { precio = 8.50; yield "Ensalada César"; }
                        case 2 -> { precio = 6.00; yield "Sopa del día"; }
                        case 3 -> { precio = 7.50; yield "Bruschettas"; }
                        default -> "Opción inválida";
                    };
                }
                case 2 -> {
                    System.out.println("--- PLATOS PRINCIPALES ---");
                    System.out.println("1. Lomo saltado");
                    System.out.println("2. Paella");
                    System.out.println("3. Lasagna");
                    System.out.print("Elija una opción: ");
                    int opcion = scanner.nextInt();
                    seleccion = switch (opcion) {
                        case 1 -> { precio = 18.00; yield "Lomo saltado"; }
                        case 2 -> { precio = 22.00; yield "Paella"; }
                        case 3 -> { precio = 16.50; yield "Lasagna"; }
                        default -> "Opción inválida";
                    };
                }
                case 3 -> {
                    System.out.println("--- BEBIDAS ---");
                    System.out.println("1. Agua mineral");
                    System.out.println("2. Refresco");
                    System.out.println("3. Café");
                    System.out.print("Elija una opción: ");
                    int opcion = scanner.nextInt();
                    seleccion = switch (opcion) {
                        case 1 -> { precio = 2.50; yield "Agua mineral"; }
                        case 2 -> { precio = 3.00; yield "Refresco"; }
                        case 3 -> { precio = 3.50; yield "Café"; }
                        default -> "Opción inválida";
                    };
                }
                case 4 -> {
                    System.out.println("--- POSTRES ---");
                    System.out.println("1. Tiramisú");
                    System.out.println("2. Helado");
                    System.out.println("3. Flan");
                    System.out.print("Elija una opción: ");
                    int opcion = scanner.nextInt();
                    seleccion = switch (opcion) {
                        case 1 -> { precio = 9.00; yield "Tiramisú"; }
                        case 2 -> { precio = 5.50; yield "Helado"; }
                        case 3 -> { precio = 6.00; yield "Flan"; }
                        default -> "Opción inválida";
                    };
                }
                default -> seleccion = "Categoría inválida";
            }

            if (precio > 0) {
                System.out.println("Ha seleccionado: " + seleccion);
                System.out.printf("Precio: $%.2f%n", precio);
            } else {
                System.out.println(seleccion);
            }

            scanner.close();
        }

import java.util.Scanner;

/**
 * Sistema de Gestión de Biblioteca Digital
 * Demostración completa de switch tradicional y enhanced switch,
 * incluyendo switch como expresión, fall-through intencional,
 * y manejo de default.

public class BibliotecaDigital {

    enum EstadoLibro { DISPONIBLE, PRESTADO, RESERVADO, EN_REPARACION }

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("========================================");
        System.out.println("  BIBLIOTECA DIGITAL - JAVA MASTER BOOK");
        System.out.println("========================================");

        // --- Menú principal con switch expresión ---
        System.out.println("\n--- MENÚ PRINCIPAL ---");
        System.out.println("1. Buscar libro");
        System.out.println("2. Prestar libro");
        System.out.println("3. Devolver libro");
        System.out.println("4. Estado de libros");
        System.out.println("5. Reportes");
        System.out.println("0. Salir");
        System.out.print("Seleccione una opción: ");

        int opcion = scanner.nextInt();
        scanner.nextLine();

        System.out.println();

        String mensaje = switch (opcion) {
            case 1 -> "Ha seleccionado: BUSCAR LIBRO";
            case 2 -> "Ha seleccionado: PRESTAR LIBRO";
            case 3 -> "Ha seleccionado: DEVOLVER LIBRO";
            case 4 -> "Ha seleccionado: ESTADO DE LIBROS";
            case 5 -> "Ha seleccionado: REPORTES";
            case 0 -> "Saliendo del sistema...";
            default -> "Opción no válida. Intente nuevamente.";
        };

        System.out.println(mensaje);

        // --- Lógica según la opción usando switch tradicional con fall-through ---
        System.out.println("\n--- PROCESANDO OPCIÓN ---");

        switch (opcion) {
            case 1:
                System.out.println("Ingrese el título o autor a buscar:");
                String busqueda = scanner.nextLine();
                System.out.println("Buscando: " + busqueda + "...");
                System.out.println("(Simulación de búsqueda completada)");
                break;

            case 2:
                System.out.print("Ingrese el ID del libro a prestar: ");
                int idPrestamo = scanner.nextInt();
                System.out.println("Procesando préstamo del libro ID: " + idPrestamo);
                System.out.println("(Simulación de préstamo completada)");
                break;

            case 3:
                System.out.print("Ingrese el ID del libro a devolver: ");
                int idDevolucion = scanner.nextInt();
                System.out.println("Procesando devolución del libro ID: " + idDevolucion);
                System.out.println("(Simulación de devolución completada)");
                break;

            case 4:
                // Demostración de enum con switch
                System.out.println("\n--- ESTADOS DE LIBROS ---");
                EstadoLibro estado = EstadoLibro.DISPONIBLE;

                String textoEstado = switch (estado) {
                    case DISPONIBLE -> "El libro está disponible para préstamo.";
                    case PRESTADO -> "El libro está actualmente prestado.";
                    case RESERVADO -> "El libro está reservado.";
                    case EN_REPARACION -> "El libro está en reparación.";
                };
                System.out.println(textoEstado);

                // Demostración de fall-through intencional con switch tradicional
                System.out.println("\nAcciones permitidas según estado:");
                switch (estado) {
                    case DISPONIBLE:
                    case RESERVADO:
                        System.out.println("- Se puede prestar");
                        System.out.println("- Se puede reservar");
                        break;
                    case PRESTADO:
                        System.out.println("- Se puede devolver");
                        System.out.println("- Se puede renovar");
                        break;
                    case EN_REPARACION:
                        System.out.println("- Solo consulta en sala");
                        break;
                }
                break;

            case 5:
                System.out.println("\n--- REPORTES ---");
                System.out.println("1. Libros más prestados");
                System.out.println("2. Usuarios con más préstamos");
                System.out.println("3. Libros atrasados");
                System.out.print("Seleccione tipo de reporte: ");
                int tipoReporte = scanner.nextInt();

                String reporte = switch (tipoReporte) {
                    case 1 -> "Generando reporte de libros más prestados...";
                    case 2 -> "Generando reporte de usuarios activos...";
                    case 3 -> "Generando reporte de devoluciones atrasadas...";
                    default -> "Tipo de reporte no válido.";
                };
                System.out.println(reporte);
                break;

            case 0:
                System.out.println("Gracias por usar la Biblioteca Digital.");
                break;

            default:
                System.out.println("Opción no reconocida. Por favor, intente de nuevo.");
                break;
        }

        // --- Demostración de switch anidado con submenú ---
        if (opcion == 1 || opcion == 2) {
            System.out.println("\n--- SUBMENÚ DE ACCIÓN ---");
            System.out.println("¿Desea realizar otra operación?");
            System.out.print("1. Sí / 2. No: ");
            int continuar = scanner.nextInt();

            String accion = switch (continuar) {
                case 1 -> "Volviendo al menú principal...";
                case 2 -> "Saliendo del sistema...";
                default -> "Opción inválida. Saliendo...";
            };
            System.out.println(accion);
        }

        // --- Demostración de switch con yield en bloque ---
        System.out.println("\n--- RESUMEN DE SESIÓN ---");

        String resumen = switch (opcion) {
            case 1 -> {
                String detalle = "Búsqueda realizada";
                System.out.println("Detalle interno: " + detalle);
                yield "Resumen: Operación de búsqueda completada.";
            }
            case 2 -> {
                String detalle = "Préstamo procesado";
                System.out.println("Detalle interno: " + detalle);
                yield "Resumen: Operación de préstamo completada.";
            }
            case 3 -> {
                String detalle = "Devolución procesada";
                System.out.println("Detalle interno: " + detalle);
                yield "Resumen: Operación de devolución completada.";
            }
            case 4 -> {
                yield "Resumen: Consulta de estado realizada.";
            }
            case 5 -> {
                yield "Resumen: Reporte generado.";
            }
            default -> "Resumen: No se realizó ninguna operación.";
        };

        System.out.println(resumen);
        System.out.println("========================================");
        scanner.close();
    }
}

================================================================================
  RESUMEN DEL CAPÍTULO
================================================================================

  • switch evalúa una expresión y ejecuta el bloque del case coincidente.
  • Los tipos soportados son: byte, short, char, int, String, enum y sus
    wrappers.
  • break termina la ejecución del switch; sin él ocurre fall-through.
  • El fall-through puede ser intencional para agrupar casos.
  • default maneja cualquier valor no contemplado por los cases.
  • El enhanced switch (Java 14+) usa sintaxis de flecha (->) y no tiene
    fall-through accidental.
  • El switch expresión devuelve un valor y requiere exhaustividad.
  • yield devuelve un valor desde un bloque de switch expresión.
  • switch es preferible a if/else para comparar una variable contra
    múltiples valores constantes.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  P: ¿Puedo usar long en un switch?
  R: No. long no está soportado en switch. Debes convertir a int o usar
  if/else. float, double y boolean tampoco son soportados.

  P: ¿Qué versión de Java necesito para el enhanced switch?
  R: El enhanced switch con sintaxis de flecha está disponible desde Java
  14 como feature estándar. Java 12 y 13 lo tenían como preview.

  P: ¿El switch expresión siempre necesita default?
  R: No siempre. Si cubres todos los valores posibles del tipo (por
  ejemplo, todos los valores de un enum), el compilador no exige default.
  Para tipos con infinitos valores (int, String), es obligatorio default
  o cubrir exhaustivamente.

  P: ¿Cuál es la diferencia entre yield y return en un switch?
  R: yield se usa dentro de un switch expresión para devolver un valor y
  terminar la evaluación del switch. return termina el método completo.
  En un switch expresión, no puedes usar return; usas yield.

  P: ¿Puedo mezclar switch tradicional con enhanced switch?
  R: No en la misma sentencia switch. Cada switch debe ser completamente
  tradicional (con dos puntos) o completamente enhanced (con flechas).

  P: ¿Qué ocurre si ningún case coincide y no hay default?
  R: Simplemente no se ejecuta ningún bloque. El programa continúa con
  la siguiente instrucción después del switch.

  P: ¿El switch con String es eficiente?
  R: Sí. Java compila los switch de String usando códigos hash y
  comparaciones con equals(), con complejidad O(1) promedio.

  P: ¿Puedo tener un switch dentro de otro switch?
  R: Sí, se permite la anidación. Sin embargo, anidar más de 2 niveles
  afecta la legibilidad. Considera refactorizar a métodos.

limpio y expresivo cuando necesitas manejar múltiples casos basados en
