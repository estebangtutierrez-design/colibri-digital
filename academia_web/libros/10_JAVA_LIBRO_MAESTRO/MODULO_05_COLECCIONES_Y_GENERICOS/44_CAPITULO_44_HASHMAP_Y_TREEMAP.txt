BIENVENIDO

Bienvenido al capítulo dedicado a los mapas, posiblemente la
estructura de datos más versátil y poderosa del Collection
Framework. Mientras que las listas y conjuntos almacenan
elementos individuales, los mapas almacenan pares clave-valor,
permitiéndote asociar información de manera significativa.

Imagina un diccionario: buscas una palabra (clave) y obtienes
su definición (valor). Eso es exactamente un mapa. Desde contar
frecuencias de palabras hasta implementar cachés, bases de datos
en memoria, o configuraciones de aplicación, los mapas están
en todas partes en el desarrollo Java.

LA INTERFAZ MAP

Map no extiende Collection (es una jerarquía separada).
Representa una estructura que asocia claves únicas con valores.

Características:
  • Claves únicas: no puede haber dos claves iguales
  • Un valor por clave: cada clave se asocia a exactamente un valor
  • Los valores pueden repetirse
  • No tiene índice posicional

```java
public interface Map<K, V> {
    V put(K key, V value);
    V get(Object key);
    V remove(Object key);
    boolean containsKey(Object key);
    boolean containsValue(Object value);
    int size();
    boolean isEmpty();
    void clear();
    Set<K> keySet();
    Collection<V> values();
    Set<Map.Entry<K, V>> entrySet();
}
```

Implementaciones principales:
  • HashMap: basado en tabla hash, O(1), sin orden
  • TreeMap: basado en árbol rojo-negro, O(log n), ordenado
  • LinkedHashMap: tabla hash + lista enlazada, orden de
    inserción/acceso
  • Hashtable: legacy, sincronizado (no usar)

HASHMAP EN PROFUNDIDAD

HashMap es la implementación más usada de Map. Almacena las
claves en una tabla hash, permitiendo acceso O(1) promedio.

Estructura interna (Java 8+):
  • Arreglo de "nodos" (buckets)
  • Cada nodo es un: hash | clave | valor | next
  • Cuando hay colisiones, se forma una lista enlazada (o árbol
    si la lista supera 8 elementos)

```java
// Estructura de un nodo (simplificada)
static class Node<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;
}
```

Parámetros importantes:
  • Capacidad inicial (default 16): número de buckets
  • Factor de carga (default 0.75): cuando size > capacidad * 0.75,
    se redimensiona (duplica la capacidad)
  • Rehashing: cuando se redimensiona, todos los elementos se
    recolocan según su nuevo hashCode % nuevaCapacidad

Colisiones en HashMap:
  • Ocurren cuando dos claves diferentes tienen el mismo hashCode
  • En Java 8+, si una lista supera 8 nodos, se convierte en
    árbol rojo-negro para mantener O(log n) en el peor caso
  • Java 8+ también usa árboles para mejorar rendimiento contra
    ataques de colisión hash (HashDoS)

Calculando índice del bucket:
  int index = (n - 1) & hash   // donde n es la capacidad

Esta operación es equivalente a hash % n pero más rápida, y
funciona porque n siempre es potencia de 2.

CREACIÓN Y OPERACIONES BÁSICAS CON MAP

```java
import java.util.HashMap;
import java.util.Map;

public class MapBasico {
    public static void main(String[] args) {
        // Crear mapa
        Map<String, Integer> edades = new HashMap<>();

        // put(K, V): añadir o actualizar
        edades.put("Ana", 25);
        edades.put("Luis", 30);
        edades.put("Carlos", 22);
        edades.put("Ana", 26);  // Actualiza el valor

        // get(Object): obtener valor por clave
        Integer edadAna = edades.get("Ana");      // 26
        Integer edadPedro = edades.get("Pedro");  // null

        // getOrDefault (Java 8+)
        int edad = edades.getOrDefault("Pedro", 0);  // 0, no null

        // containsKey / containsValue
        boolean existe = edades.containsKey("Ana");   // true
        boolean hayEdad = edades.containsValue(30);   // true

        // remove
        edades.remove("Carlos");

        // size / isEmpty
        System.out.println("Tamaño: " + edades.size());
        System.out.println("¿Vacío?: " + edades.isEmpty());

        // putIfAbsent (Java 8+): solo añade si la clave no existe
        edades.putIfAbsent("Luis", 99);  // No actualiza, ya existe

        // replace (Java 8+)
        edades.replace("Ana", 26, 27);   // Solo reemplaza si el valor actual es 26

        // clear
        edades.clear();
    }
}
```

productos.entrySet().stream()

MÉTODOS AVANZADOS (JAVA 8+)

```java
Map<String, Integer> ventas = new HashMap<>();
ventas.put("Ana", 100);
ventas.put("Luis", 200);
ventas.put(null, 50);  // HashMap permite clave null

// compute: calcular nuevo valor basado en clave y valor actual
ventas.compute("Ana", (k, v) -> v == null ? 0 : v + 50);
// Ana → 150

// computeIfAbsent: calcular solo si la clave no existe
ventas.computeIfAbsent("Pedro", k -> 0);
// Pedro → 0 (se añade)

// computeIfPresent: calcular solo si la clave existe
ventas.computeIfPresent("Luis", (k, v) -> v + 100);
// Luis → 300

// merge: combinar valor existente con nuevo valor
ventas.merge("Ana", 30, Integer::sum);  // Ana: 150 + 30 = 180
ventas.merge("Nuevo", 50, Integer::sum); // Nuevo: 50 (no había valor)

// replaceAll
ventas.replaceAll((k, v) -> v * 2);
```

TREEMAP: MAPA ORDENADO

TreeMap implementa SortedMap y NavigableMap. Las claves se
almacenan ordenadas según su orden natural o un Comparator.

Características:
  • O(log n) en operaciones básicas
  • Claves siempre ordenadas
  • No permite null como clave (desde Java 7)
  • Implementa SortedMap y NavigableMap

```java
import java.util.*;

public class EjemploTreeMap {
    public static void main(String[] args) {
        // Orden natural (alfabético para String)
        Map<String, Integer> treeMap = new TreeMap<>();
        treeMap.put("Carlos", 30);
        treeMap.put("Ana", 25);
        treeMap.put("Luis", 28);
        System.out.println(treeMap);  // {Ana=25, Carlos=30, Luis=28}

        // TreeMap con orden inverso
        Map<String, Integer> inverso = new TreeMap<>(Comparator.reverseOrder());
        inverso.put("Carlos", 30);
        inverso.put("Ana", 25);
        inverso.put("Luis", 28);
        System.out.println(inverso);  // {Luis=28, Carlos=30, Ana=25}

        // TreeMap con Comparator personalizado
        Map<String, Integer> porLongitud = new TreeMap<>(
            (a, b) -> Integer.compare(a.length(), b.length())
        );
        porLongitud.put("tres", 3);
        porLongitud.put("siete", 5);
        porLongitud.put("dos", 2);
        System.out.println(porLongitud);  // {dos=2, tres=3, siete=5}
    }
}
```

Métodos de navegación:

```java
TreeMap<Integer, String> mapa = new TreeMap<>();
mapa.put(10, "Diez");
mapa.put(20, "Veinte");
mapa.put(30, "Treinta");
mapa.put(40, "Cuarenta");
mapa.put(50, "Cincuenta");

// SortedMap methods
mapa.firstKey();        // 10
mapa.lastKey();         // 50
mapa.headMap(30);       // {10=Diez, 20=Veinte}
mapa.tailMap(30);       // {30=Treinta, 40=Cuarenta, 50=Cincuenta}
mapa.subMap(20, 40);    // {20=Veinte, 30=Treinta}

// NavigableMap methods
mapa.lowerKey(30);      // 20 (estrictamente menor)
mapa.floorKey(25);      // 20 (menor o igual)
mapa.ceilingKey(25);    // 30 (mayor o igual)
mapa.higherKey(30);     // 40 (estrictamente mayor)

mapa.lowerEntry(30);    // 20=Veinte
mapa.floorEntry(25);    // 20=Veinte
mapa.ceilingEntry(25);  // 30=Treinta
mapa.higherEntry(30);   // 40=Cuarenta

mapa.descendingMap();   // {50=Cincuenta, ..., 10=Diez}
mapa.pollFirstEntry();  // Elimina y devuelve 10=Diez
mapa.pollLastEntry();   // Elimina y devuelve 50=Cincuenta
```

LINKEDHASHMAP: ORDEN DE INSERCIÓN Y ACCESO

LinkedHashMap mantiene una lista doblemente enlazada de las
entradas, permitiendo:
  • Orden de inserción (por defecto)
  • Orden de acceso (útil para cachés LRU)

```java
import java.util.*;

public class LinkedHashMapExample {
    public static void main(String[] args) {
        // Orden de inserción
        Map<String, Integer> insercion = new LinkedHashMap<>();
        insercion.put("Z", 1);
        insercion.put("A", 2);
        insercion.put("M", 3);
        System.out.println("Orden inserción: " + insercion);
        // {Z=1, A=2, M=3}

        // Orden de acceso (accessOrder = true)
        Map<String, Integer> acceso = new LinkedHashMap<>(16, 0.75f, true);
        acceso.put("A", 1);
        acceso.put("B", 2);
        acceso.put("C", 3);
        acceso.get("A");  // Acceso mueve "A" al final
        acceso.get("B");  // Acceso mueve "B" al final
        System.out.println("Orden acceso: " + acceso);
        // {C=3, A=1, B=2}
    }
}
```

Caché LRU con LinkedHashMap:

```java
class LRUCache<K, V> extends LinkedHashMap<K, V> {
    private final int capacidad;

    public LRUCache(int capacidad) {
        super(capacidad, 0.75f, true);  // accessOrder=true
        this.capacidad = capacidad;
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        return size() > capacidad;
    }
}

// Uso
LRUCache<Integer, String> cache = new LRUCache<>(3);
cache.put(1, "Uno");
cache.put(2, "Dos");
cache.put(3, "Tres");
System.out.println(cache);  // {1=Uno, 2=Dos, 3=Tres}
cache.get(1);               // Accede a 1
cache.put(4, "Cuatro");     // El LRU es 2 (no visitado recientemente)
System.out.println(cache);  // {3=Tres, 1=Uno, 4=Cuatro}
```

EJEMPLO: CONTADOR DE FRECUENCIA DE PALABRAS

```java
import java.util.*;

public class ContadorFrecuencia {
    public static Map<String, Integer> contarPalabras(String texto) {
        Map<String, Integer> frecuencias = new HashMap<>();
        String[] palabras = texto.toLowerCase().split("[\\s\\p{Punct}]+");

        for (String palabra : palabras) {
            if (palabra.isEmpty()) continue;
            // Forma clásica
            // frecuencias.put(palabra, frecuencias.getOrDefault(palabra, 0) + 1);

            // Forma con merge
            frecuencias.merge(palabra, 1, Integer::sum);
        }
        return frecuencias;
    }

    public static Map<String, Integer> topN(Map<String, Integer> mapa, int n) {
        return mapa.entrySet().stream()
            .sorted(Map.Entry.<String, Integer>comparingByValue().reversed())
            .limit(n)
            .collect(LinkedHashMap::new,
                (m, e) -> m.put(e.getKey(), e.getValue()),
                Map::putAll);
    }

    public static void main(String[] args) {
        String texto = "El sol brilla sobre el mar y el sol se oculta tras el mar";

        Map<String, Integer> frecuencias = contarPalabras(texto);
        System.out.println("Frecuencias: " + frecuencias);

        System.out.println("\nTop 3 palabras más frecuentes:");
        Map<String, Integer> top = topN(frecuencias, 3);
        top.forEach((palabra, count) ->
            System.out.println(palabra + ": " + count + " veces"));
    }
}
```

EJEMPLO: AGENDA TELEFÓNICA

```java
import java.util.*;

public class AgendaTelefonica {
    private Map<String, String> contactos = new HashMap<>();

    public void agregarContacto(String nombre, String telefono) {
        if (contactos.containsKey(nombre)) {
            System.out.println("Contacto actualizado: " + nombre);
        }
        contactos.put(nombre, telefono);
    }

    public String buscarTelefono(String nombre) {
        return contactos.getOrDefault(nombre, "Contacto no encontrado");
    }

    public boolean eliminarContacto(String nombre) {
        return contactos.remove(nombre) != null;
    }

    public void listarContactos() {
        if (contactos.isEmpty()) {
            System.out.println("Agenda vacía.");
            return;
        }
        System.out.println("=== CONTACTOS ===");
        contactos.forEach((nombre, telefono) ->
            System.out.println(nombre + ": " + telefono));
    }

    public List<String> buscarPorPrefijo(String prefijo) {
        List<String> resultado = new ArrayList<>();
        for (String nombre : contactos.keySet()) {
            if (nombre.toLowerCase().startsWith(prefijo.toLowerCase())) {
                resultado.add(nombre + ": " + contactos.get(nombre));
            }
        }
        return resultado;
    }

    public static void main(String[] args) {
        AgendaTelefonica agenda = new AgendaTelefonica();

        agenda.agregarContacto("Ana García", "555-0101");
        agenda.agregarContacto("Luis Pérez", "555-0102");
        agenda.agregarContacto("Carlos Ruiz", "555-0103");

        agenda.listarContactos();

        System.out.println("\nTeléfono de Ana: " + agenda.buscarTelefono("Ana García"));

        System.out.println("\nContactos que empiezan con 'A':");
        agenda.buscarPorPrefijo("A").forEach(System.out::println);

        agenda.eliminarContacto("Carlos Ruiz");
        System.out.println("\nDespués de eliminar a Carlos:");
        agenda.listarContactos();
    }
}
```

EJEMPLO: AGRUPACIÓN DE ELEMENTOS

```java
import java.util.*;
import java.util.stream.Collectors;

public class AgrupacionEjemplos {
    public static void main(String[] args) {
        List<Empleado> empleados = Arrays.asList(
            new Empleado("Ana", "Ventas", 50000),
            new Empleado("Luis", "IT", 60000),
            new Empleado("Carlos", "Ventas", 55000),
            new Empleado("María", "IT", 65000),
            new Empleado("Pedro", "RH", 45000)
        );

        // Agrupar por departamento (manual)
        Map<String, List<Empleado>> porDepto = new HashMap<>();
        for (Empleado e : empleados) {
            porDepto.computeIfAbsent(e.getDepartamento(),
                k -> new ArrayList<>()).add(e);
        }

        System.out.println("=== Empleados por departamento ===");
        porDepto.forEach((dept, list) -> {
            System.out.println(dept + ": " + list.size() + " empleados");
            list.forEach(e -> System.out.println("  - " + e.getNombre()));
        });

        // Salario promedio por departamento
        Map<String, Double> promedioSalario = new HashMap<>();
        for (Map.Entry<String, List<Empleado>> entry : porDepto.entrySet()) {
            double promedio = entry.getValue().stream()
                .mapToDouble(Empleado::getSalario)
                .average()
                .orElse(0);
            promedioSalario.put(entry.getKey(), promedio);
        }

        System.out.println("\n=== Salario promedio por departamento ===");
        promedioSalario.forEach((dept, prom) ->
            System.out.printf("%s: $%.2f%n", dept, prom));
    }
}

class Empleado {
    private String nombre;
    private String departamento;
    private double salario;

    public Empleado(String nombre, String departamento, double salario) {
        this.nombre = nombre;
        this.departamento = departamento;
        this.salario = salario;
    }

    public String getNombre() { return nombre; }
    public String getDepartamento() { return departamento; }
    public double getSalario() { return salario; }
}
```

NULL EN MAPAS

```java
Map<String, String> hashMap = new HashMap<>();
hashMap.put(null, "valorNull");      // OK
hashMap.put("clave", null);          // OK
System.out.println(hashMap.get(null));     // "valorNull"
System.out.println(hashMap.get("clave"));  // null
System.out.println(hashMap.containsKey("clave"));  // true (existe aunque valor sea null)

Map<String, String> treeMap = new TreeMap<>();
treeMap.put(null, "valor");  // NullPointerException
```

Diferencia entre clave null y clave ausente:
  • get() devuelve null en ambos casos
  • containsKey() distingue entre ambos

```java
Map<String, String> mapa = new HashMap<>();
mapa.put("clave", null);

System.out.println(mapa.get("clave"));        // null
System.out.println(mapa.get("inexistente"));  // null
System.out.println(mapa.containsKey("clave"));        // true
System.out.println(mapa.containsKey("inexistente"));  // false
```

ERRORES COMUNES Y BUENAS PRÁCTICAS

Error #1: Iterar sobre keySet y hacer get dentro del bucle

```java
Map<String, Integer> mapa = new HashMap<>();

// MAL: dos búsquedas por clave (ineficiente)
for (String clave : mapa.keySet()) {
    Integer valor = mapa.get(clave);  // Segunda búsqueda
}

// BIEN: iterar sobre entrySet
for (Map.Entry<String, Integer> entry : mapa.entrySet()) {
    String clave = entry.getKey();
    Integer valor = entry.getValue();
}
```

Error #2: Usar objetos mutables como clave

```java
// MAL: si la clave cambia, el mapa se corrompe
Map<List<Integer>, String> mapa = new HashMap<>();
List<Integer> key = new ArrayList<>(List.of(1, 2, 3));
mapa.put(key, "valor");
key.add(4);  // ¡MODIFICACIÓN! El hash cambia
System.out.println(mapa.get(key));  // probablemente null
```

Error #3: Asumir que put siempre añade

```java
Map<String, Integer> mapa = new HashMap<>();
mapa.put("clave", 1);
Integer anterior = mapa.put("clave", 2);  // anterior = 1
System.out.println(mapa.get("clave"));  // 2 (reemplazó)
```

Buena práctica: Inicializar con capacidad estimada

```java
// Si esperas 1000 entradas
Map<String, Integer> mapa = new HashMap<>(2000);  // /0.75 = ~1334
```

Buena práctica: Usar Map.of() para mapas pequeños e inmutables

```java
Map<String, Integer> inmutable = Map.of(
    "A", 1, "B", 2, "C", 3
);
// Hasta 10 entradas. Para más, usar Map.ofEntries()
```

RESUMEN DEL CAPÍTULO

• Map almacena pares clave-valor con claves únicas.

• HashMap: O(1) promedio, sin orden, permite null (clave y valor).

• TreeMap: O(log n), ordenado por clave, implementa SortedMap
  y NavigableMap para navegación avanzada.

• LinkedHashMap: O(1), mantiene orden de inserción o acceso
  (útil para cachés LRU).

• Los métodos put(), get(), remove(), containsKey() son
  fundamentales.

• Java 8+ añadió métodos poderosos: compute(), merge(),
  putIfAbsent(), forEach(), getOrDefault().

• No uses objetos mutables como claves en HashMap.

• Programa contra la interfaz Map para flexibilidad.

PREGUNTAS FRECUENTES

P: ¿Por qué HashMap permite null pero TreeMap no?
R: HashMap usa hashCode() que maneja null (hash 0). TreeMap
   necesita comparar claves con compareTo(), y null no es
   comparable.

P: ¿Cuál es el factor de carga óptimo?
R: 0.75 es el valor por defecto y es óptimo para la mayoría de
   casos. Menor (0.5) gasta más memoria pero menos colisiones.
   Mayor (0.9) ahorra memoria pero más colisiones.

P: ¿HashMap garantiza orden de inserción?
R: No. Ni siquiera en Java 8+. Si necesitas orden de inserción,
   usa LinkedHashMap. Si necesitas orden natural, usa TreeMap.

P: ¿Cómo itero sobre un Map de manera eficiente?
R: Usa entrySet() con for-each o forEach lambda. Evita iterar
   keySet y llamar get() dentro del bucle.

P: ¿Qué pasa si dos claves tienen el mismo hashCode?
R: Ocurre una colisión. Los elementos se almacenan en la misma
   cubeta formando una lista enlazada (o árbol si hay más de 8).
   El rendimiento se degrada de O(1) a O(n) u O(log n).

DESAFÍO DEL ARQUITECTO

SISTEMA DE RECOMENDACIÓN DE PRODUCTOS

Implementa un motor de recomendación simple que use mapas como
estructura principal. Debe funcionar así:

  • Usuarios: Map<String, Set<String>> — usuario → conjunto de
    productos que ha comprado/visto

  • Productos: Map<String, Set<String>> — producto → conjunto de
    usuarios que lo compraron

  • Matriz de co-ocurrencia: Map<String, Map<String, Integer>>
    — producto A → (producto B → veces que A y B se compraron
    juntos)

Funcionalidades:
  1. Registrar compra: un usuario compra un producto
  2. Recomendar para usuario: basado en "a los usuarios que
     compraron X también les gustó Y"
  3. Recomendar producto similar: basado en co-ocurrencia
  4. Top productos: productos más comprados
  5. Usuarios similares: usuarios que compraron productos
     similares

Requisitos adicionales:
  • Implementar filtrado colaborativo básico (item-based)
  • Normalizar puntuaciones de recomendación (0.0 a 1.0)
  • Excluir productos ya comprados de las recomendaciones
  • Soportar al menos 10,000 usuarios y 1,000 productos

Extra:
  • Implementar una caché LRU para recomendaciones frecuentes
  • Añadir un mapa de popularidad para productos nuevos sin
    historial de compras (cold start)

Este desafío integra todos los tipos de mapas en un escenario
realista de machine learning y sistemas de recomendación.

Has conquistado los mapas en Java. HashMap te ha mostrado cómo

clave-valor.
