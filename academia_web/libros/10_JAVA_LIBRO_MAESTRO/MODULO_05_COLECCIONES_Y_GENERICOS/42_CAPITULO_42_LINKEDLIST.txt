BIENVENIDO

Bienvenido al capítulo dedicado a LinkedList, la implementación

ESTRUCTURA INTERNA DE LINKEDLIST

LinkedList está implementada como una lista doblemente enlazada.
Cada elemento se almacena en un nodo que contiene:

  • El dato (elemento)
  • Referencia al nodo anterior (prev)
  • Referencia al nodo siguiente (next)

Estructura conceptual:

  null ← [prev|A|next] ⇄ [prev|B|next] ⇄ [prev|C|next] → null
          ↑ first                    ↑ last

La clase mantiene referencias al primer nodo (first) y al
último nodo (last), además del tamaño (size).

```java
// Estructura interna simplificada
private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;

    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}

transient Node<E> first;
transient Node<E> last;
transient int size = 0;
```

Cuando añades un elemento al final, simplemente creas un nuevo
nodo y ajustas las referencias del antiguo último nodo y del
nuevo nodo.

CREACIÓN DE LINKEDLIST

```java
import java.util.LinkedList;
import java.util.List;
import java.util.Deque;
import java.util.Queue;

// Forma 1: constructor vacío
LinkedList<String> lista1 = new LinkedList<>();

// Forma 2: a partir de otra colección
LinkedList<String> lista2 = new LinkedList<>(lista1);

// Forma 3: como List
List<String> comoLista = new LinkedList<>();

// Forma 4: como Queue
Queue<String> comoCola = new LinkedList<>();

// Forma 5: como Deque
Deque<String> comoDeque = new LinkedList<>();
```

LinkedList implementa tres interfaces principales:
  • List — comportamiento de lista
  • Deque — cola doble (extiende Queue)
  • Queue — comportamiento de cola

Esto la convierte en una de las clases más versátiles del
Collection Framework.

MÉTODOS HEREDADOS DE LIST

Los mismos que ArrayList, pero con diferente rendimiento:

```java
LinkedList<String> lista = new LinkedList<>();
lista.add("A");        // O(1) al final
lista.add("B");
lista.add("C");

String primero = lista.getFirst();  // O(1)
String ultimo = lista.getLast();    // O(1)
String cualquiera = lista.get(1);   // O(n) - recorre desde el inicio

lista.set(1, "X");      // O(n) - necesita encontrar el nodo

lista.remove(0);        // O(1) - elimina el primero
lista.remove(1);        // O(n) - necesita encontrar el nodo
lista.remove("C");      // O(n) - busca lineal
```

IMPORTANTE: get(index) y set(index) son O(n) en LinkedList,
a diferencia de ArrayList que son O(1). LinkedList recorre los
nodos desde el extremo más cercano (inicio o final).

================================================================================
MÉTODOS ESPECÍFICOS DE LINKEDLIST

--- addFirst(E e) / addLast(E e) ---

Añade al principio o al final de la lista.

```java
LinkedList<String> lista = new LinkedList<>();
lista.addFirst("Primero");
lista.addLast("Último");
lista.addFirst("Nuevo Primero");
// [Nuevo Primero, Primero, Último]
```

--- getFirst() / getLast() ---

Obtiene el primer o último elemento sin eliminarlo.

```java
String primero = lista.getFirst();
String ultimo = lista.getLast();
```

--- removeFirst() / removeLast() ---

Elimina y devuelve el primer o último elemento.

```java
String eliminado = lista.removeFirst();
String ultimoEliminado = lista.removeLast();
```

--- offerFirst(E e) / offerLast(E e) ---

Versiones de addFirst/addLast que devuelven false en lugar de
lanzar excepción si la colección está llena (aplica para
colas con capacidad limitada; LinkedList tiene capacidad
ilimitada, así que siempre devuelven true).

--- pollFirst() / pollLast() ---

Versiones de removeFirst/removeLast que devuelven null en
lugar de lanzar excepción cuando la lista está vacía.

--- peekFirst() / peekLast() ---

Versiones de getFirst/getLast que devuelven null en lugar de
lanzar excepción cuando la lista está vacía.

--- pop() / push(E e) ---

Operaciones de pila. push() añade al principio, pop() elimina
y devuelve el primero.

```java
LinkedList<String> pila = new LinkedList<>();
pila.push("A");  // addFirst
pila.push("B");
pila.push("C");
System.out.println(pila.pop());  // "C" - removeFirst
```

--- element() / offer(E e) / peek() / poll() / remove() ---

Operaciones heredadas de Queue. Ver capítulo 45 para detalles.

ARRAYLIST VS LINKEDLIST: COMPARATIVA

┌─────────────────────────┬─────────────┬─────────────┐
│ OPERACIÓN               │ ARRAYLIST   │ LINKEDLIST  │
├─────────────────────────┼─────────────┼─────────────┤
│ get(index)              │ O(1)        │ O(n)        │
│ add(E) al final         │ O(1)*       │ O(1)        │
│ add(index, E)           │ O(n)        │ O(n)        │
│ addFirst(E)             │ O(n)        │ O(1)        │
│ remove(index)           │ O(n)        │ O(n)        │
│ removeFirst()           │ O(n)        │ O(1)        │
│ remove(Object)          │ O(n)        │ O(n)        │
│ contains(Object)        │ O(n)        │ O(n)        │
│ Iteración               │ O(n)        │ O(n)        │
│ Memoria por elemento    │ Bajo        │ Alto        │
└─────────────────────────┴─────────────┴─────────────┘

* add() en ArrayList es O(1) amortizado; O(n) cuando se
  redimensiona.

Cuándo usar ArrayList:
  • Acceso frecuente por índice
  • Iteración predominante
  • Inserciones/eliminaciones al final
  • Bajo consumo de memoria

Cuándo usar LinkedList:
  • Inserciones/eliminaciones frecuentes al principio
  • Operaciones de cola (FIFO) o pila (LIFO)
  • Necesitas implementar Deque
  • No necesitas acceso aleatorio frecuente

EJEMPLO: LINKEDLIST COMO COLA (FIFO)

```java
import java.util.LinkedList;
import java.util.Queue;

public class ColaSupermercado {
    public static void main(String[] args) {
        Queue<String> cola = new LinkedList<>();

        // Llegan clientes
        cola.offer("Cliente 1");
        cola.offer("Cliente 2");
        cola.offer("Cliente 3");

        System.out.println("Atendiendo clientes:");
        while (!cola.isEmpty()) {
            String cliente = cola.poll();
            System.out.println("Atendiendo a: " + cliente);
            System.out.println("Cola restante: " + cola);
        }
        System.out.println("Todos los clientes atendidos.");
    }
}
```

EJEMPLO: LINKEDLIST COMO PILA (LIFO)

```java
import java.util.Deque;
import java.util.LinkedList;

public class HistorialNavegacion {
    private Deque<String> historial = new LinkedList<>();

    public void visitarPagina(String url) {
        historial.push(url);
        System.out.println("Visitando: " + url);
    }

    public void retroceder() {
        if (!historial.isEmpty()) {
            String pagina = historial.pop();
            System.out.println("Retrocediendo desde: " + pagina);
        } else {
            System.out.println("No hay páginas anteriores.");
        }
    }

    public String paginaActual() {
        return historial.peek();
    }

    public static void main(String[] args) {
        HistorialNavegacion navegador = new HistorialNavegacion();

        navegador.visitarPagina("google.com");
        navegador.visitarPagina("github.com");
        navegador.visitarPagina("stackoverflow.com");

        System.out.println("\nPágina actual: " + navegador.paginaActual());

        navegador.retroceder();
        navegador.retroceder();
        System.out.println("Página actual: " + navegador.paginaActual());
    }
}
```

EJEMPLO: COMPARATIVA DE RENDIMIENTO

```java
import java.util.ArrayList;
import java.util.LinkedList;
import java.util.List;

public class BenchmarkListas {
    public static void main(String[] args) {
        int elementos = 100_000;
        int operaciones = 10_000;

        // --- Inserción al final ---
        System.out.println("=== INSERCIÓN AL FINAL ===");

        long inicio = System.nanoTime();
        List<Integer> arrayList = new ArrayList<>();
        for (int i = 0; i < elementos; i++) arrayList.add(i);
        long finArrayList = System.nanoTime();
        System.out.println("ArrayList: " + (finArrayList - inicio) / 1_000_000 + " ms");

        inicio = System.nanoTime();
        List<Integer> linkedList = new LinkedList<>();
        for (int i = 0; i < elementos; i++) linkedList.add(i);
        long finLinkedList = System.nanoTime();
        System.out.println("LinkedList: " + (finLinkedList - inicio) / 1_000_000 + " ms");

        // --- Inserción al principio ---
        System.out.println("\n=== INSERCIÓN AL PRINCIPIO ===");

        arrayList = new ArrayList<>();
        inicio = System.nanoTime();
        for (int i = 0; i < operaciones; i++) arrayList.add(0, i);
        finArrayList = System.nanoTime();
        System.out.println("ArrayList addFirst: " + (finArrayList - inicio) / 1_000_000 + " ms");

        linkedList = new LinkedList<>();
        inicio = System.nanoTime();
        for (int i = 0; i < operaciones; i++) ((LinkedList<Integer>) linkedList).addFirst(i);
        finLinkedList = System.nanoTime();
        System.out.println("LinkedList addFirst: " + (finLinkedList - inicio) / 1_000_000 + " ms");

        // --- Acceso por índice ---
        System.out.println("\n=== ACCESO POR ÍNDICE ===");

        inicio = System.nanoTime();
        for (int i = 0; i < operaciones; i++) arrayList.get(i);
        finArrayList = System.nanoTime();
        System.out.println("ArrayList get: " + (finArrayList - inicio) / 1_000_000 + " ms");

        inicio = System.nanoTime();
        for (int i = 0; i < operaciones; i++) linkedList.get(i);
        finLinkedList = System.nanoTime();
        System.out.println("LinkedList get: " + (finLinkedList - inicio) / 1_000_000 + " ms");
    }
}
```

EJEMPLO: GESTIÓN DE TURNOS (LISTA CIRCULAR CON LINKEDLIST)

```java
import java.util.Iterator;
import java.util.LinkedList;

public class GestionTurnos {
    private LinkedList<String> turnos = new LinkedList<>();
    private Iterator<String> iterador;

    public void registrarCliente(String nombre) {
        turnos.addLast(nombre);
        System.out.println(nombre + " registrado. Turnos: " + turnos.size());
    }

    public String siguienteTurno() {
        if (turnos.isEmpty()) return "No hay turnos pendientes";
        return "Turno de: " + turnos.removeFirst();
    }

    public void atenderCliente() {
        System.out.println(siguienteTurno());
    }

    public void saltarCliente(String nombre) {
        if (turnos.remove(nombre)) {
            turnos.addLast(nombre);
            System.out.println(nombre + " movido al final de la cola.");
        }
    }

    public void mostrarCola() {
        System.out.println("Cola actual: " + turnos);
    }

    public static void main(String[] args) {
        GestionTurnos sistema = new GestionTurnos();

        sistema.registrarCliente("Ana");
        sistema.registrarCliente("Luis");
        sistema.registrarCliente("Carlos");
        sistema.registrarCliente("María");

        sistema.mostrarCola();
        sistema.atenderCliente();
        sistema.saltarCliente("Carlos");
        sistema.mostrarCola();
        sistema.atenderCliente();
        sistema.atenderCliente();
        sistema.atenderCliente();
        sistema.atenderCliente();
    }
}
```

EJEMPLO: EDITOR DE TEXTO CON DESHACER

```java
import java.util.LinkedList;
import java.util.Deque;

public class EditorTexto {
    private StringBuilder texto = new StringBuilder();
    private Deque<String> pilaDeshacer = new LinkedList<>();
    private Deque<String> pilaRehacer = new LinkedList<>();

    public void escribir(String nuevoTexto) {
        pilaDeshacer.push(texto.toString());
        texto.append(nuevoTexto);
        pilaRehacer.clear();
        System.out.println("Texto actual: '" + texto + "'");
    }

    public void deshacer() {
        if (!pilaDeshacer.isEmpty()) {
            pilaRehacer.push(texto.toString());
            texto = new StringBuilder(pilaDeshacer.pop());
            System.out.println("Deshacer. Texto actual: '" + texto + "'");
        } else {
            System.out.println("No hay acciones para deshacer.");
        }
    }

    public void rehacer() {
        if (!pilaRehacer.isEmpty()) {
            pilaDeshacer.push(texto.toString());
            texto = new StringBuilder(pilaRehacer.pop());
            System.out.println("Rehacer. Texto actual: '" + texto + "'");
        } else {
            System.out.println("No hay acciones para rehacer.");
        }
    }

    public static void main(String[] args) {
        EditorTexto editor = new EditorTexto();
        editor.escribir("Hola ");
        editor.escribir("mundo ");
        editor.escribir("cruel");
        editor.deshacer();
        editor.deshacer();
        editor.rehacer();
        editor.escribir("hermoso");
    }
}
```

ERRORES COMUNES Y BUENAS PRÁCTICAS

Error #1: Usar get(index) en bucles grandes con LinkedList

```java
LinkedList<String> lista = new LinkedList<>();
// ... 100,000 elementos

// MAL: O(n²) — get(i) recorre desde el principio cada vez
for (int i = 0; i < lista.size(); i++) {
    System.out.println(lista.get(i));
}

// BIEN: O(n) — usa iterador o for-each
for (String s : lista) {
    System.out.println(s);
}
```

Error #2: Asumir que LinkedList siempre es mejor para insertar

LinkedList solo es mejor que ArrayList para inserciones al
principio. Para inserciones en medio, ambas son O(n). En la
práctica, ArrayList suele ser más rápido incluso ahí porque
tiene menor overhead de memoria y mejor localidad de caché.

Error #3: Usar LinkedList cuando solo necesitas Deque

Si solo necesitas operaciones de cola/pila, considera ArrayDeque
que tiene mejor rendimiento y menor consumo de memoria.

Buena práctica: Usar la interfaz apropiada

```java
// Como lista
List<String> lista = new LinkedList<>();

// Como cola
Queue<String> cola = new LinkedList<>();

// Como cola doble
Deque<String> deque = new LinkedList<>();
```

RESUMEN DEL CAPÍTULO

• LinkedList es una lista doblemente enlazada: cada nodo tiene
  referencias al elemento anterior y siguiente.

• Implementa List, Deque y Queue, ofreciendo gran versatilidad.

• Inserción y eliminación al principio: O(1) — muy eficiente.

• Acceso por índice: O(n) — lento comparado con ArrayList.

• Ofrece métodos específicos: addFirst, addLast, getFirst,
  getLast, removeFirst, removeLast, push, pop, peek.

• Puede usarse como lista, cola (FIFO) o pila (LIFO).

• Elige ArrayList para acceso frecuente por índice;
  LinkedList para operaciones frecuentes en los extremos.

• ArrayDeque suele ser mejor que LinkedList para operaciones
  de cola/pila puras.

PREGUNTAS FRECUENTES

P: ¿LinkedList consume más memoria que ArrayList?
R: Sí, significativamente. Cada elemento en LinkedList requiere
   almacenar dos referencias adicionales (prev y next), además
   del objeto Node. En ArrayList solo se almacena la referencia
   en el arreglo. Para listas grandes, la diferencia puede ser
   de 3x-4x más memoria.

P: ¿Cuándo debo preferir LinkedList sobre ArrayList?
R: Cuando la operación principal sea insertar o eliminar al
   principio de la lista, o cuando necesites una estructura que
   funcione como cola y lista simultáneamente.

P: ¿Es LinkedList mejor que ArrayList para inserciones en medio?
R: No necesariamente. Aunque LinkedList no requiere desplazar
   elementos, necesita O(n) para llegar a la posición. Además,
   el overhead de crear nodos hace que ArrayList a menudo sea
   más rápido en la práctica para listas medianas.

P: ¿Por qué LinkedList implementa Deque pero ArrayList no?
R: Porque LinkedList puede agregar/eliminar en ambos extremos
   en O(1), mientras que ArrayList requiere O(n) para operaciones
   al principio (desplazar todos los elementos).

P: ¿LinkedList es thread-safe?
R: No. Para uso concurrente, debes sincronizarla externamente
   con Collections.synchronizedList() o usar ConcurrentLinkedDeque.

    - terminarProceso(): mueve a terminados
    - mostrarEstado(): imprime todas las colas

    100ms y luego vuelve al final de la cola de listos.

  • Implementar promoción de prioridad: cada 10 segundos de

Requisitos adicionales:
  • Usar LinkedList como Deque para operaciones eficientes
  • Implementar tanto FIFO (cola de listos) como LIFO (para
    procesos que acaban de llegar con alta prioridad)
  • Mostrar estadísticas: tiempo promedio de espera, throughput

Extras:
  • Simular la ejecución en tiempo real con hilos
  • Implementar herencia de prioridad (evitar inversión)

Este desafío integra todos los conceptos de LinkedList y
demuestra su poder en un escenario de sistemas operativos.

LinkedList representa una alternativa poderosa a ArrayList cuando
el patrón de uso involucra operaciones frecuentes en los extremos
de la lista. Su implementación como lista doblemente enlazada le
confiere propiedades únicas: inserción y eliminación O(1) al
principio y al final, y la capacidad de funcionar como cola, pila
y lista simultáneamente.
