COLECCIÓN: CONCEPTO FUNDAMENTAL

Una colección es un objeto que agrupa múltiples elementos en una
sola unidad. Java proporciona el Collection Framework, una
arquitectura unificada para almacenar y manipular grupos de
objetos.

El framework incluye:

  • Interfaces: contratos que definen comportamiento
  • Implementaciones: clases concretas que realizan esos contratos
  • Algoritmos: métodos utilitarios para manipular colecciones

La interfaz raíz es Collection, que extiende Iterable. De ella
derivan List, Set y Queue.

================================================================================
LA INTERFAZ LIST

List es una interfaz que representa una secuencia ordenada de
elementos. Características principales:

  • Ordenada: los elementos mantienen el orden de inserción
  • Indexada: cada elemento tiene una posición numérica
  • Permite duplicados: puedes almacenar el mismo valor varias veces
  • Acceso posicional: puedes obtener, insertar o eliminar por índice

Implementaciones principales de List:
  • ArrayList: basada en un arreglo redimensionable
  • LinkedList: basada en nodos doblemente enlazados
  • Vector: versión sincronizada (legacy, poco usada hoy)

================================================================================
ARRAYLIST EN PROFUNDIDAD

ArrayList es una implementación de List respaldada por un arreglo
dinámico. Cuando el arreglo interno se llena, ArrayList crea un
nuevo arreglo más grande (generalmente un 50% mayor) y copia los
elementos existentes.

Características clave:
  • Acceso O(1): obtener elementos por índice es instantáneo
  • Inserción O(n): insertar al principio o en medio requiere
    desplazar elementos
  • Eliminación O(n): similar a inserción, requiere desplazamiento
  • Redimensionamiento automático: crece según necesidad
  • No sincronizado: no es thread-safe (pero es más rápido)

Estructura interna simplificada:

  Object[] elementData;  // arreglo interno
  int size;              // cantidad de elementos actuales

Cuando haces add(), el tamaño interno crece así:

  int newCapacity = oldCapacity + (oldCapacity >> 1);
  // Incremento del 50%

================================================================================
CREACIÓN DE ARRAYLIST

Formas de crear un ArrayList:

```java
import java.util.ArrayList;
import java.util.List;

// Forma 1: capacidad por defecto (10)
ArrayList<String> lista1 = new ArrayList<>();

// Forma 2: capacidad inicial especificada
ArrayList<String> lista2 = new ArrayList<>(100);

// Forma 3: a partir de otra colección
ArrayList<String> lista3 = new ArrayList<>(lista1);

// Forma 4: usando el tipo interfaz (programación orientada a interfaz)
List<String> lista4 = new ArrayList<>();
```

La programación orientada a interfaz (forma 4) es la práctica
recomendada. Al declarar la variable como List, puedes cambiar
la implementación a LinkedList, Vector, etc. sin modificar el
código cliente.

Capacidad inicial: si sabes que almacenarás muchos elementos,
especificar una capacidad inicial evita redimensionamientos
costosos. Por ejemplo, si esperas 10,000 elementos:

```java
List<String> usuarios = new ArrayList<>(10_000);
```

Esto crea un arreglo interno de 10,000 posiciones de una vez,
evitando múltiples redimensionamientos.

MÉTODOS PRINCIPALES DE ARRAYLIST

--- add(E e) ---

Añade un elemento al final de la lista. Devuelve siempre true.

```java
List<String> frutas = new ArrayList<>();
frutas.add("Manzana");
frutas.add("Banana");
frutas.add("Cereza");
// Lista: [Manzana, Banana, Cereza]
```

--- add(int index, E element) ---

Inserta un elemento en una posición específica, desplazando los
elementos existentes hacia la derecha.

```java
frutas.add(1, "Arándano");
// Lista: [Manzana, Arándano, Banana, Cereza]
```

--- get(int index) ---

Obtiene el elemento en la posición indicada.

```java
String fruta = frutas.get(2);  // "Banana"
```

--- set(int index, E element) ---

Reemplaza el elemento en la posición indicada.

```java
frutas.set(1, "Mango");
// Lista: [Manzana, Mango, Banana, Cereza]
```

--- remove(int index) ---

Elimina el elemento en la posición indicada y lo devuelve. Los
elementos posteriores se desplazan a la izquierda.

```java
String eliminado = frutas.remove(2);  // "Banana"
// Lista: [Manzana, Mango, Cereza]
```

--- remove(Object o) ---

Elimina la primera ocurrencia del objeto especificado.

```java
frutas.remove("Manzana");
// Lista: [Mango, Cereza]
```

--- size() ---

Devuelve la cantidad de elementos en la lista.

```java
int total = frutas.size();  // 2
```

--- isEmpty() ---

Devuelve true si la lista no contiene elementos.

```java
boolean vacia = frutas.isEmpty();  // false
```

--- contains(Object o) ---

Devuelve true si la lista contiene el elemento especificado.

```java
boolean tieneMango = frutas.contains("Mango");  // true
```

--- indexOf(Object o) ---

Devuelve el índice de la primera ocurrencia, o -1 si no existe.

```java
int pos = frutas.indexOf("Mango");  // 0
```

--- clear() ---

Elimina todos los elementos de la lista.

```java
frutas.clear();
// Lista: []
```

--- toArray() ---

Convierte la lista a un arreglo.

```java
Object[] arregloObjetos = frutas.toArray();
String[] arregloString = frutas.toArray(new String[0]);
```

--- addAll(Collection<? extends E> c) ---

Añade todos los elementos de una colección al final.

```java
List<String> masFrutas = List.of("Kiwi", "Papaya");
frutas.addAll(masFrutas);
```

--- removeAll(Collection<?> c) ---

Elimina todos los elementos que están en la colección dada.

```java
List<String> eliminar = List.of("Mango", "Kiwi");
frutas.removeAll(eliminar);
```

--- retainAll(Collection<?> c) ---

Conserva solo los elementos que están en la colección dada.

```java
List<String> conservar = List.of("Cereza", "Papaya");
frutas.retainAll(conservar);
```

--- subList(int fromIndex, int toIndex) ---

Devuelve una vista de una porción de la lista (no copia).

```java
List<String> sub = frutas.subList(0, 2);
// Elementos en índices 0 y 1
```

AUTO-BOXING Y UNBOXING EN COLECCIONES

Las colecciones no pueden almacenar tipos primitivos, solo
objetos. Java realiza automáticamente la conversión:

```java
List<Integer> numeros = new ArrayList<>();

// Auto-boxing: int → Integer
numeros.add(42);       // Java convierte int 42 a Integer.valueOf(42)
numeros.add(100);

// Unboxing: Integer → int
int valor = numeros.get(0);  // Java convierte Integer a int

// Esto fallaría con tipos primitivos:
// List<int> errores = new ArrayList<>();  // ERROR: no compila
```

Los tipos primitivos soportados por sus wrappers:
  • boolean → Boolean
  • byte → Byte
  • char → Character
  • short → Short
  • int → Integer
  • long → Long
  • float → Float
  • double → Double

ITERACIÓN SOBRE ARRAYLIST

Forma 1: for clásico con índice

```java
List<String> nombres = List.of("Ana", "Luis", "Carlos");

for (int i = 0; i < nombres.size(); i++) {
    System.out.println(nombres.get(i));
}
```

Forma 2: for-each (enhanced for loop)

```java
for (String nombre : nombres) {
    System.out.println(nombre);
}
```

Forma 3: Iterator

```java
Iterator<String> it = nombres.iterator();
while (it.hasNext()) {
    String nombre = it.next();
    System.out.println(nombre);
}
```

Forma 4: ListIterator (bidireccional)

```java
ListIterator<String> lit = nombres.listIterator();
while (lit.hasNext()) {
    System.out.println(lit.next());
}
// Recorrido inverso
while (lit.hasPrevious()) {
    System.out.println(lit.previous());
}
```

Forma 5: forEach con lambda (Java 8+)

```java
nombres.forEach(n -> System.out.println(n));
nombres.forEach(System.out::println);  // method reference
```

Forma 6: Stream API (Java 8+)

```java
nombres.stream()
       .filter(n -> n.startsWith("A"))
       .forEach(System.out::println);
```

Rendimiento: el for-each y el forEach lambda son los más legibles
y generalmente los más usados. El for con índice es útil cuando
necesitas modificar la lista mientras iteras.

EJEMPLO COMPLETO: GESTIÓN DE ESTUDIANTES

```java
import java.util.ArrayList;
import java.util.List;

public class GestionEstudiantes {
    public static void main(String[] args) {
        List<Estudiante> estudiantes = new ArrayList<>();

        estudiantes.add(new Estudiante("A001", "Ana García", 8.5));
        estudiantes.add(new Estudiante("A002", "Luis Pérez", 7.2));
        estudiantes.add(new Estudiante("A003", "Carlos Ruiz", 9.1));
        estudiantes.add(new Estudiante("A004", "María López", 6.8));

        System.out.println("=== Lista de Estudiantes ===");
        for (Estudiante e : estudiantes) {
            System.out.println(e);
        }

        System.out.println("\nPromedio del curso: " + calcularPromedio(estudiantes));

        Estudiante mejor = estudiantes.get(0);
        for (Estudiante e : estudiantes) {
            if (e.getNota() > mejor.getNota()) {
                mejor = e;
            }
        }
        System.out.println("Mejor estudiante: " + mejor.getNombre());

        estudiantes.removeIf(e -> e.getNota() < 7.0);
        System.out.println("\nEstudiantes aprobados:");
        estudiantes.forEach(System.out::println);
    }

    public static double calcularPromedio(List<Estudiante> lista) {
        double suma = 0;
        for (Estudiante e : lista) {
            suma += e.getNota();
        }
        return lista.isEmpty() ? 0 : suma / lista.size();
    }
}

class Estudiante {
    private String id;
    private String nombre;
    private double nota;

    public Estudiante(String id, String nombre, double nota) {
        this.id = id;
        this.nombre = nombre;
        this.nota = nota;
    }

    public String getNombre() { return nombre; }
    public double getNota() { return nota; }

    @Override
    public String toString() {
        return id + " | " + nombre + " | Nota: " + nota;
    }
}
```

EJEMPLO: COMPARANDO ARRAYS VS ARRAYLIST

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class ArrayVsArrayList {
    public static void main(String[] args) {
        // ARRAY tradicional
        System.out.println("=== ARRAY TRADICIONAL ===");
        String[] array = new String[3];
        array[0] = "A";
        array[1] = "B";
        array[2] = "C";
        // array[3] = "D";  // ArrayIndexOutOfBoundsException
        System.out.println(Arrays.toString(array));

        // ARRAYLIST
        System.out.println("\n=== ARRAYLIST ===");
        List<String> lista = new ArrayList<>();
        lista.add("A");
        lista.add("B");
        lista.add("C");
        lista.add("D");  // Sin problema, crece automáticamente
        System.out.println(lista);

        // Eliminación
        // array[1] = null;  // No se redimensiona, deja hueco
        lista.remove(1);    // Se redimensiona automáticamente
        System.out.println("Después de eliminar índice 1: " + lista);

        // Búsqueda
        int indiceArray = -1;
        for (int i = 0; i < array.length; i++) {
            if ("C".equals(array[i])) {
                indiceArray = i;
                break;
            }
        }
        int indiceLista = lista.indexOf("C");
        System.out.println("Índice de 'C' en array: " + indiceArray);
        System.out.println("Índice de 'C' en lista: " + indiceLista);
    }
}
```

EJEMPLO: CAPACIDAD Y RENDIMIENTO

```java
import java.util.ArrayList;
import java.util.List;

public class CapacidadArrayList {
    public static void main(String[] args) {
        // ArrayList sin capacidad inicial
        long inicio = System.nanoTime();
        List<Integer> sinCapacidad = new ArrayList<>();
        for (int i = 0; i < 1_000_000; i++) {
            sinCapacidad.add(i);
        }
        long fin = System.nanoTime();
        System.out.println("Sin capacidad inicial: " +
            (fin - inicio) / 1_000_000 + " ms");

        // ArrayList CON capacidad inicial
        inicio = System.nanoTime();
        List<Integer> conCapacidad = new ArrayList<>(1_000_000);
        for (int i = 0; i < 1_000_000; i++) {
            conCapacidad.add(i);
        }
        fin = System.nanoTime();
        System.out.println("Con capacidad inicial: " +
            (fin - inicio) / 1_000_000 + " ms");

        // Diferencia: la segunda suele ser 2-3 veces más rápida
    }
}
```

ERRORES COMUNES Y BUENAS PRÁCTICAS

Error #1: Usar tipos raw (sin genérico)

```java
// MAL: tipo raw, sin seguridad de tipos
ArrayList lista = new ArrayList();
lista.add("Hola");
lista.add(42);  // Compila, pero peligroso

// BIEN: parametrizado con genericos
ArrayList<String> lista = new ArrayList<>();
```

Error #2: Modificar la lista durante la iteración con for-each

```java
List<String> items = new ArrayList<>(List.of("A", "B", "C"));

// MAL: ConcurrentModificationException
for (String item : items) {
    if (item.equals("B")) {
        items.remove(item);  // ¡EXCEPCIÓN!
    }
}

// BIEN: usar Iterator.remove()
Iterator<String> it = items.iterator();
while (it.hasNext()) {
    if (it.next().equals("B")) {
        it.remove();
    }
}

// BIEN (Java 8+): removeIf
items.removeIf(item -> item.equals("B"));
```

Error #3: Confundir capacity con size

```java
List<String> lista = new ArrayList<>(100);
System.out.println(lista.size());  // 0, no 100
// Capacidad es el espacio interno reservado, no los elementos
```

Buena práctica: programar contra interfaces

```java
// MAL: atado a la implementación
ArrayList<String> lista = new ArrayList<>();

// BIEN: flexible, puedes cambiar a LinkedList mañana
List<String> lista = new ArrayList<>();
```

RESUMEN DEL CAPÍTULO

• ArrayList es una implementación de List basada en un arreglo
  dinámico que se redimensiona automáticamente.

• Proporciona acceso O(1) por índice, pero inserciones y
  eliminaciones en medio son O(n).

• Los métodos principales son: add(), get(), set(), remove(),
  size(), isEmpty(), contains(), indexOf(), clear().

• Las colecciones no aceptan tipos primitivos; Java realiza
  auto-boxing/unboxing automáticamente.

• Existen múltiples formas de iterar: for clásico, for-each,
  Iterator, ListIterator, forEach lambda y Stream API.

• Es buena práctica programar contra la interfaz List.

• Especificar la capacidad inicial mejora el rendimiento cuando
  conoces el tamaño aproximado.

• No debes modificar una lista mientras la iteras con for-each;
  usa Iterator.remove() o removeIf().

PREGUNTAS FRECUENTES

P: ¿Por qué ArrayList es más lento que un array?
R: ArrayList tiene una capa de abstracción sobre el array interno.
   Cada llamada a métodos como get(), add() tiene un overhead
   mínimo. Para la mayoría de aplicaciones la diferencia es
   insignificante. Si necesitas máximo rendimiento y tamaño fijo,
   usa arrays.

P: ¿Cuándo debo especificar la capacidad inicial?
R: Cuando conoces aproximadamente el tamaño final de la lista y
   este es grande (> 10,000 elementos). Esto evita múltiples
   redimensionamientos costosos.

P: ¿ArrayList es thread-safe?
R: No. Para acceso concurrente debes usar:
   • Collections.synchronizedList(new ArrayList<>())
   • CopyOnWriteArrayList (del paquete concurrent)
   • Sincronización manual con bloques synchronized

P: ¿Cómo convierto un ArrayList a un array?
R: Usa toArray():
   String[] array = lista.toArray(new String[0]);

P: ¿Cuál es la diferencia entre capacity y size?
R: Capacity es el tamaño del arreglo interno (memoria reservada).
   Size es la cantidad real de elementos almacenados.
