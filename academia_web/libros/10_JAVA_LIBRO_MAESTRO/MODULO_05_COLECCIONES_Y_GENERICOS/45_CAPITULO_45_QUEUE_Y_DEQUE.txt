BIENVENIDO

Bienvenido al capítulo de colas en Java. Las colas representan
una de las estructuras de datos más fundamentales en informática.
Desde la cola de impresión hasta la gestión de procesos en un
sistema operativo, las colas están en todas partes.

En este capítulo exploraremos Queue (FIFO), Deque (doble extremo)
y PriorityQueue (cola con prioridad). Cada una tiene reglas
distintas sobre cómo se insertan y extraen los elementos, y
cada una es óptima para escenarios diferentes.

LA INTERFAZ QUEUE

Queue extiende Collection y representa una colección diseñada
para mantener elementos en un orden de procesamiento específico
(generalmente FIFO: First-In, First-Out).

```java
public interface Queue<E> extends Collection<E> {
    boolean add(E e);      // Inserta, lanza excepción si falla
    boolean offer(E e);    // Inserta, devuelve false si falla

    E remove();            // Elimina y devuelve, lanza excepción si vacía
    E poll();              // Elimina y devuelve, null si vacía

    E element();           // Obtiene sin eliminar, lanza excepción si vacía
    E peek();              // Obtiene sin eliminar, null si vacía
}
```

Dos tipos de métodos:
  • Los que lanzan excepción cuando la cola está vacía o llena:
    add(), remove(), element()
  • Los que retornan valor especial (null/false):
    offer(), poll(), peek()

Para colas ilimitadas (como LinkedList o PriorityQueue), add()
y offer() son equivalentes. Para colas con capacidad limitada
(como ArrayBlockingQueue), offer() es preferible.

================================================================================
PRIORITYQUEUE: COLA CON PRIORIDAD

PriorityQueue es una cola donde los elementos se ordenan según
su prioridad natural (Comparable) o un Comparator. El elemento
con mayor prioridad (menor según el orden) sale primero.

Características:
  • No es FIFO: el orden de salida depende de la prioridad
  • Basada en un heap binario (no en una lista ordenada)
  • O(log n) en inserción y extracción
  • O(1) para peek()
  • No permite null
  • No sincronizada

```java
import java.util.PriorityQueue;
import java.util.Queue;

public class PriorityQueueExample {
    public static void main(String[] args) {
        // Cola de prioridad natural (menor número = mayor prioridad)
        Queue<Integer> cola = new PriorityQueue<>();

        cola.offer(30);
        cola.offer(10);
        cola.offer(20);
        cola.offer(5);

        System.out.println("Peek: " + cola.peek());  // 5 (el menor)

        while (!cola.isEmpty()) {
            System.out.print(cola.poll() + " ");  // 5 10 20 30
        }
        // Salida: 5 10 20 30 (ordenados)

        // PriorityQueue con orden inverso (mayor número = mayor prioridad)
        Queue<Integer> prioridadInversa = new PriorityQueue<>(
            (a, b) -> b - a
        );
        prioridadInversa.offer(30);
        prioridadInversa.offer(10);
        prioridadInversa.offer(20);
        prioridadInversa.offer(5);

        while (!prioridadInversa.isEmpty()) {
            System.out.print(prioridadInversa.poll() + " ");  // 30 20 10 5
        }
    }
}
```

PriorityQueue con objetos personalizados:

```java
import java.util.PriorityQueue;
import java.util.Queue;

class Tarea implements Comparable<Tarea> {
    private String nombre;
    private int prioridad;  // 1 = más alta, 5 = más baja

    public Tarea(String nombre, int prioridad) {
        this.nombre = nombre;
        this.prioridad = prioridad;
    }

    @Override
    public int compareTo(Tarea otra) {
        return Integer.compare(this.prioridad, otra.prioridad);
    }

    @Override
    public String toString() {
        return "[Prioridad " + prioridad + "] " + nombre;
    }
}

public class GestorTareasPrioritarias {
    public static void main(String[] args) {
        Queue<Tarea> tareas = new PriorityQueue<>();

        tareas.offer(new Tarea("Revisar email", 3));
        tareas.offer(new Tarea("Bug crítico en producción", 1));
        tareas.offer(new Tarea("Documentar API", 5));
        tareas.offer(new Tarea("Reunión con cliente", 2));
        tareas.offer(new Tarea("Actualizar dependencias", 4));

        System.out.println("Procesando tareas por prioridad:");
        while (!tareas.isEmpty()) {
            Tarea t = tareas.poll();
            System.out.println("→ " + t);
            // Simular procesamiento...
        }
    }
}
```

DEQUE: COLA DOBLE

Deque (Double-Ended Queue) extiende Queue y permite operaciones
en ambos extremos: puedes insertar, obtener y eliminar tanto al
principio como al final.

```java
public interface Deque<E> extends Queue<E> {
    // Métodos de ambos extremos (excepción / valor especial)
    void addFirst(E e);      boolean offerFirst(E e);
    void addLast(E e);       boolean offerLast(E e);
    E removeFirst();         E pollFirst();
    E removeLast();          E pollLast();
    E getFirst();            E peekFirst();
    E getLast();             E peekLast();

    // Métodos de pila
    void push(E e);     // addFirst
    E pop();            // removeFirst

    // Métodos de cola (heredados)
    boolean offer(E e);     // offerLast
    E poll();               // pollFirst
    E peek();               // peekFirst
}
```

Implementaciones principales:
  • ArrayDeque: basada en arreglo circular, eficiente
  • LinkedList: basada en nodos (vista en capítulo 42)
  • LinkedBlockingDeque: versión concurrente

================================================================================
ARRAYDEQUE: LA IMPLEMENTACIÓN ESTRELLA

ArrayDeque es una implementación de Deque basada en un arreglo
redimensionable. Es más rápida que LinkedList para operaciones
de cola/pila y consume menos memoria.

Características:
  • O(1) en inserciones/eliminaciones en ambos extremos
  • Basada en arreglo circular (ring buffer)
  • No permite null
  • No sincronizada
  • Más rápida que LinkedList para Deque
  • Más rápida que Stack para pila
  • Más rápida que LinkedList para cola

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class ArrayDequeExample {
    public static void main(String[] args) {
        // Como cola (FIFO)
        Deque<String> cola = new ArrayDeque<>();
        cola.offer("A");
        cola.offer("B");
        cola.offer("C");
        System.out.println("Cola FIFO:");
        while (!cola.isEmpty()) {
            System.out.println(cola.poll());  // A, B, C
        }

        // Como pila (LIFO) - ¡mejor que Stack!
        Deque<String> pila = new ArrayDeque<>();
        pila.push("A");
        pila.push("B");
        pila.push("C");
        System.out.println("\nPila LIFO:");
        while (!pila.isEmpty()) {
            System.out.println(pila.pop());  // C, B, A
        }

        // Como cola doble
        Deque<Integer> deque = new ArrayDeque<>();
        deque.addFirst(10);
        deque.addLast(20);
        deque.addFirst(5);
        deque.addLast(25);
        System.out.println("\nDeque: " + deque);  // [5, 10, 20, 25]
        System.out.println("Primero: " + deque.getFirst());  // 5
        System.out.println("Último: " + deque.getLast());    // 25
    }
}
```

Estructura interna de ArrayDeque (ring buffer):

```
ArrayDeque: capacidad 8 (siempre potencia de 2)

head = 3, tail = 7

Índice:  0  1  2  3  4  5  6  7
Valor:   _  _  _  E  F  G  H  _
                ↑           ↑
                head        tail

Añadir al final (tail): se pone en índice 7, tail avanza a 0
Eliminar del inicio (head): se elimina índice 3, head avanza a 4
```

Esta estructura permite que todas las operaciones en ambos
extremos sean O(1) sin necesidad de copiar elementos (hasta
que se necesita redimensionar).

EJEMPLO: COLA DE IMPRESIÓN CON PRIORIDAD

```java
import java.util.PriorityQueue;
import java.util.Queue;

public class ColaImpresion {
    static class TrabajoImpresion implements Comparable<TrabajoImpresion> {
        private String documento;
        private int prioridad;     // 1 = urgente, 5 = normal
        private int paginas;
        private long timestamp;

        public TrabajoImpresion(String doc, int prioridad, int pag) {
            this.documento = doc;
            this.prioridad = prioridad;
            this.paginas = pag;
            this.timestamp = System.nanoTime();
        }

        @Override
        public int compareTo(TrabajoImpresion otro) {
            // Primero por prioridad, luego por orden de llegada
            if (this.prioridad != otro.prioridad) {
                return Integer.compare(this.prioridad, otro.prioridad);
            }
            return Long.compare(this.timestamp, otro.timestamp);
        }

        @Override
        public String toString() {
            return "[P" + prioridad + "] " + documento + " (" + paginas + " pág)";
        }
    }

    public static void main(String[] args) {
        Queue<TrabajoImpresion> cola = new PriorityQueue<>();

        cola.offer(new TrabajoImpresion("Informe mensual", 3, 15));
        cola.offer(new TrabajoImpresion("Contrato urgente", 1, 3));
        cola.offer(new TrabajoImpresion("Manual usuario", 5, 50));
        cola.offer(new TrabajoImpresion("Factura", 2, 1));

        System.out.println("Procesando cola de impresión:");
        while (!cola.isEmpty()) {
            TrabajoImpresion trabajo = cola.poll();
            System.out.println("Imprimiendo: " + trabajo);
            try {
                Thread.sleep(trabajo.paginas * 10);  // Simular impresión
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        System.out.println("Todos los documentos impresos.");
    }
}
```

EJEMPLO: SISTEMA DE UNDO/REDO CON DEQUE

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class SistemaUndoRedo {
    private Deque<String> historialUndo = new ArrayDeque<>();
    private Deque<String> historialRedo = new ArrayDeque<>();
    private String estadoActual = "";

    public void ejecutar(String accion) {
        historialUndo.push(estadoActual);
        estadoActual = accion;
        historialRedo.clear();
        System.out.println("Ejecutado: " + accion);
    }

    public String deshacer() {
        if (historialUndo.isEmpty()) {
            System.out.println("No hay acciones para deshacer.");
            return estadoActual;
        }
        historialRedo.push(estadoActual);
        estadoActual = historialUndo.pop();
        System.out.println("Deshecho. Estado actual: " + estadoActual);
        return estadoActual;
    }

    public String rehacer() {
        if (historialRedo.isEmpty()) {
            System.out.println("No hay acciones para rehacer.");
            return estadoActual;
        }
        historialUndo.push(estadoActual);
        estadoActual = historialRedo.pop();
        System.out.println("Rehecho. Estado actual: " + estadoActual);
        return estadoActual;
    }

    public void verHistorial() {
        System.out.println("Undo stack: " + historialUndo);
        System.out.println("Redo stack: " + historialRedo);
        System.out.println("Actual: " + estadoActual);
    }

    public static void main(String[] args) {
        SistemaUndoRedo editor = new SistemaUndoRedo();

        editor.ejecutar("Escribir 'Hola'");
        editor.ejecutar("Escribir ' Mundo'");
        editor.ejecutar("Poner en negrita");

        editor.verHistorial();

        editor.deshacer();
        editor.deshacer();
        editor.rehacer();

        System.out.println("\nDespués de undo/redo:");
        editor.verHistorial();
    }
}
```

EJEMPLO: VENTANA DESLIZANTE CON DEQUE

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class VentanaDeslizante {
    public static int[] maximosVentana(int[] arr, int k) {
        if (arr == null || k <= 0 || k > arr.length) {
            return new int[0];
        }

        int[] resultado = new int[arr.length - k + 1];
        Deque<Integer> deque = new ArrayDeque<>();  // Guarda índices

        for (int i = 0; i < arr.length; i++) {
            // Eliminar elementos fuera de la ventana actual
            while (!deque.isEmpty() && deque.peekFirst() <= i - k) {
                deque.pollFirst();
            }

            // Eliminar elementos menores que el actual (no serán máximos)
            while (!deque.isEmpty() && arr[deque.peekLast()] < arr[i]) {
                deque.pollLast();
            }

            deque.offerLast(i);

            // El primer elemento de la deque es el máximo de la ventana
            if (i >= k - 1) {
                resultado[i - k + 1] = arr[deque.peekFirst()];
            }
        }
        return resultado;
    }

    public static void main(String[] args) {
        int[] arr = {1, 3, -1, -3, 5, 3, 6, 7};
        int k = 3;

        int[] maximos = maximosVentana(arr, k);
        System.out.print("Array: ");
        for (int n : arr) System.out.print(n + " ");
        System.out.println("\nVentana de tamaño " + k);
        System.out.print("Máximos: ");
        for (int m : maximos) System.out.print(m + " ");
        // Salida: 3 3 5 5 6 7
    }
}
```

Este es un problema clásico de algoritmos donde Deque ofrece
una solución O(n) en lugar de O(n*k).

EJEMPLO: SISTEMA DE ATENCIÓN AL CLIENTE

```java
import java.util.*;

public class SistemaAtencionCliente {
    private Queue<String> colaNormal = new LinkedList<>();
    private Queue<String> colaVip = new LinkedList<>();
    private Deque<String> colaPrioritaria = new ArrayDeque<>();

    public void llegarCliente(String nombre, String tipo) {
        switch (tipo) {
            case "VIP":
                colaVip.offer(nombre);
                break;
            case "NORMAL":
                colaNormal.offer(nombre);
                break;
            case "PRIORITARIO":
                colaPrioritaria.addFirst(nombre);
                break;
        }
        System.out.println(nombre + " (" + tipo + ") ha llegado.");
    }

    public String atenderSiguiente() {
        if (!colaPrioritaria.isEmpty()) {
            return "Prioritario: " + colaPrioritaria.pollFirst();
        }
        if (!colaVip.isEmpty()) {
            return "VIP: " + colaVip.poll();
        }
        if (!colaNormal.isEmpty()) {
            return "Normal: " + colaNormal.poll();
        }
        return "No hay clientes en espera.";
    }

    public void mostrarEstado() {
        System.out.println("\n=== ESTADO COLAS ===");
        System.out.println("Cola normal: " + colaNormal);
        System.out.println("Cola VIP: " + colaVip);
        System.out.println("Cola prioritaria: " + colaPrioritaria);
        System.out.println("Total esperando: " +
            (colaNormal.size() + colaVip.size() + colaPrioritaria.size()));
    }

    public static void main(String[] args) {
        SistemaAtencionCliente sistema = new SistemaAtencionCliente();

        sistema.llegarCliente("Ana", "NORMAL");
        sistema.llegarCliente("Luis", "VIP");
        sistema.llegarCliente("Carlos", "NORMAL");
        sistema.llegarCliente("María", "PRIORITARIO");
        sistema.llegarCliente("Pedro", "VIP");

        sistema.mostrarEstado();

        System.out.println("\nAtendiendo:");
        for (int i = 0; i < 6; i++) {
            System.out.println(sistema.atenderSiguiente());
        }
    }
}
```

ERRORES COMUNES Y BUENAS PRÁCTICAS

Error #1: Iterar sobre PriorityQueue esperando orden

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offerAll(List.of(5, 1, 3, 2, 4));

// MAL: la iteración no garantiza orden
for (int n : pq) {
    System.out.print(n + " ");  // Puede salir desordenado
}

// BIEN: poll() en bucle para obtener orden
while (!pq.isEmpty()) {
    System.out.print(pq.poll() + " ");  // 1 2 3 4 5
}
```

Error #2: Usar Stack (clase legacy) en lugar de Deque

```java
// MAL: Stack es legacy, lento, sincronizado innecesariamente
Stack<String> pilaVieja = new Stack<>();

// BIEN: ArrayDeque como pila (más rápido, más moderno)
Deque<String> pilaNueva = new ArrayDeque<>();
```

Error #3: Confundir Queue con otros tipos de colección

Queue no es para almacenamiento arbitrario. Es para procesamiento
ordenado. No debes acceder a elementos por índice ni iterar
sobre una cola para leer datos — debes poll() para consumir.

Buena práctica: Elegir la implementación correcta

```java
// Cola FIFO simple
Queue<String> cola = new LinkedList<>();

// Cola FIFO eficiente (no necesita null)
Deque<String> colaEficiente = new ArrayDeque<>();

// Cola con prioridad
Queue<Tarea> prioridad = new PriorityQueue<>();

// Pila
Deque<String> pila = new ArrayDeque<>();
```

RESUMEN DEL CAPÍTULO

• Queue representa una colección para procesamiento FIFO, con
  métodos que lanzan excepción (add, remove, element) y métodos
  que retornan valor especial (offer, poll, peek).

• PriorityQueue ordena elementos por prioridad (Comparable o
  Comparator). No es FIFO. La extracción con poll() devuelve
  el elemento de mayor prioridad.

• Deque extiende Queue y permite operaciones en ambos extremos:
  addFirst/addLast, removeFirst/removeLast, getFirst/getLast.

• ArrayDeque es la implementación recomendada para Deque:
  más rápida que LinkedList, más rápida que Stack.

• PriorityQueue está basada en heap binario: O(log n) en
  inserción y extracción, O(1) en consulta.

• Elige LinkedList para colas que necesiten null, ArrayDeque
  para máximo rendimiento, PriorityQueue para prioridades.

PREGUNTAS FRECUENTES

P: ¿Cuál es la diferencia entre poll() y remove()?
R: Ambos eliminan y devuelven el elemento al frente de la cola.
   remove() lanza NoSuchElementException si la cola está vacía;
   poll() devuelve null.

P: ¿Por qué debería usar ArrayDeque en lugar de LinkedList?
R: ArrayDeque es más rápido, consume menos memoria (no tiene
   overhead de nodos), y tiene mejor localidad de caché. La
   única ventaja de LinkedList es que permite null.

P: ¿PriorityQueue garantiza orden en la iteración?
R: No. PriorityQueue solo garantiza que poll() devuelva el
   elemento de mayor prioridad. La iteración interna puede
   mostrar los elementos en cualquier orden (el orden del heap).

P: ¿Cuándo usar add() vs offer()?
R: Para colas ilimitadas (LinkedList, PriorityQueue, ArrayDeque),
   ambos son equivalentes. Para colas con capacidad limitada
   (ArrayBlockingQueue), offer() es preferible porque no lanza
   excepción.

P: ¿Deque reemplaza a Stack?
R: Sí. La documentación oficial de Java recomienda usar Deque
   (especialmente ArrayDeque) en lugar de Stack. Stack extiende
   Vector y tiene overhead de sincronización innecesario.

DESAFÍO DEL ARQUITECTO

SIMULADOR DE SISTEMA OPERATIVO (SCHEDULER MULTINIVEL)

Implementa un planificador de procesos con múltiples colas
usando Queue, PriorityQueue y Deque. El sistema debe simular
la planificación de procesos de un sistema operativo real:

  • Cola de procesos nuevos: Deque (prioridad: los de mayor
    prioridad se ponen al frente)

  • Cola de listos: PriorityQueue (ordenada por prioridad,
    luego por tiempo de llegada)

  • Cola de E/S: Queue (FIFO) — procesos esperando operaciones
    de entrada/salida

  • Cola de terminados: Deque (se añaden al final cuando
    completan)

Algoritmo de planificación: MLFQ (Multi-Level Feedback Queue)
  • Nivel 0: Round Robin (quantum 100ms) — alta prioridad
  • Nivel 1: Round Robin (quantum 200ms) — prioridad media
  • Nivel 2: FCFS (First-Come, First-Served) — baja prioridad

  • Los procesos nuevos comienzan en el nivel 0
  • Si un proceso no completa en su quantum, baja un nivel
  • Si un proceso se bloquea por E/S, al desbloquearse sube
    un nivel (para evitar starvation)

Cada proceso tiene:
  • PID, nombre, prioridad base, tiempo total de CPU,
    tiempo usado, estado, nivel actual

Requisitos adicionales:
  • Simular llegada de procesos en tiempo real
  • Mostrar estadísticas: throughput, tiempo promedio de espera,
    tiempo de retorno, tiempos de respuesta
  • Detectar starvation (procesos esperando más de X tiempo)
  • Implementar aging: cada cierto tiempo, subir de nivel los
    procesos que llevan mucho esperando

Extra:
  • Implementar herencia de prioridad (priority inheritance)
  • Soportar afinidad de CPU (procesos bound a CPU vs I/O bound)
  • Mostrar visualización de las colas en tiempo real

Este desafío integra todos los tipos de colas en un escenario
complejo de sistemas operativos, demostrando el poder de Queue,
PriorityQueue y Deque en aplicaciones del mundo real.

Has completado la exploración de colas en Java. Queue te ha
mostrado cómo procesar elementos en orden de llegada (FIFO).
PriorityQueue ha revelado cómo la prioridad puede alterar el
orden de procesamiento. Y Deque ha demostrado la flexibilidad

ArrayDeque se perfila como la implementación estrella para
