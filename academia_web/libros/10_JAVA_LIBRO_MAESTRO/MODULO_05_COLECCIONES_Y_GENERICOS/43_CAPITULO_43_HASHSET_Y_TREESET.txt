BIENVENIDO

Bienvenido al fascinante mundo de los conjuntos en Java. Si las
listas representan secuencias ordenadas que permiten duplicados,
los conjuntos son colecciones que garantizan la unicidad de sus
elementos. No importa cuántas veces intentes añadir un elemento
duplicado, el conjunto simplemente lo ignorará.

En este capítulo exploraremos las tres implementaciones principales
de Set: HashSet (rápido, sin orden), TreeSet (ordenado, más lento)
y LinkedHashSet (orden de inserción). Cada una tiene un propósito
específico, y dominarlas te permitirá escribir código más limpio
y eficiente cuando necesites eliminar duplicados o trabajar con
operaciones de teoría de conjuntos.

================================================================================
LA INTERFAZ SET

Set es una interfaz que extiende Collection y representa una
colección que no contiene elementos duplicados. Características:

  • Sin duplicados: no pueden existir dos elementos iguales
  • Sin índice: no hay acceso posicional (no hay get(index))
  • Iteración: se itera sobre los elementos, pero el orden
    depende de la implementación

```java
public interface Set<E> extends Collection<E> {
    // Métodos heredados de Collection:
    // add, remove, contains, size, isEmpty, clear, iterator, etc.
    // NO hay get(int index), set(int index), etc.
}
```

Tres implementaciones principales:
  • HashSet: basado en tabla hash, O(1), sin orden garantizado
  • TreeSet: basado en árbol rojo-negro, O(log n), ordenado
  • LinkedHashSet: tabla hash + lista enlazada, O(1), orden de
    inserción

================================================================================
HASHSET EN PROFUNDIDAD

HashSet es la implementación más utilizada de Set. Está respaldada
internamente por un HashMap (sí, un mapa). Cuando añades un
elemento al HashSet, en realidad lo almacenas como clave en un
HashMap con un valor constante dummy.

```java
// Estructura interna simplificada
private transient HashMap<E, Object> map;
private static final Object PRESENT = new Object();

public boolean add(E e) {
    return map.put(e, PRESENT) == null;
}
```

Características:
  • Operaciones O(1) promedio: add, remove, contains
  • No garantiza orden de los elementos
  • Permite un elemento null
  • No sincronizado

La eficiencia de HashSet depende de dos métodos cruciales:
  • hashCode(): distribuye los objetos en "cubetas" (buckets)
  • equals(): determina si dos objetos son iguales

Contrato entre hashCode() y equals():
  1. Si dos objetos son equals(), deben tener el mismo hashCode()
  2. Si dos objetos tienen el mismo hashCode(), pueden o no ser
     equals() (colisión)
  3. hashCode() debe ser consistente: mismo objeto, mismo valor
     durante la ejecución

================================================================================
IMPLEMENTANDO hashCode() Y equals() CORRECTAMENTE

```java
import java.util.Objects;

public class Persona {
    private String nombre;
    private int edad;
    private String email;

    public Persona(String nombre, int edad, String email) {
        this.nombre = nombre;
        this.edad = edad;
        this.email = email;
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Persona persona = (Persona) o;
        return edad == persona.edad &&
               Objects.equals(nombre, persona.nombre) &&
               Objects.equals(email, persona.email);
    }

    @Override
    public int hashCode() {
        return Objects.hash(nombre, edad, email);
    }

    @Override
    public String toString() {
        return nombre + " (" + edad + ") - " + email;
    }
}
```

Reglas de oro:
  • Incluir los mismos campos en ambos métodos
  • Usar campos inmutables (o al menos que no cambien mientras
    el objeto está en un HashSet)
  • NO incluir campos derivados o volátiles
  • Objects.hash() y Objects.equals() simplifican la escritura
  • Si usas IDE (IntelliJ, Eclipse), genera estos métodos
    automáticamente

Consecuencias de implementar mal hashCode():
  • HashSet permite duplicados (pierde su propósito)
  • contains() devuelve false para elementos existentes
  • Rendimiento pésimo (todos los objetos en la misma cubeta)

CREACIÓN Y USO DE HASHSET

```java
import java.util.HashSet;
import java.util.Set;

// Creación
Set<String> nombres = new HashSet<>();
Set<Integer> numeros = new HashSet<>(100);     // capacidad inicial
Set<String> copia = new HashSet<>(nombres);     // copia

// Añadir elementos
nombres.add("Ana");
nombres.add("Luis");
nombres.add("Carlos");
nombres.add("Ana");  // Ignorado, ya existe

System.out.println(nombres);  // Orden impredecible

// Verificar existencia
boolean existe = nombres.contains("Ana");  // true, O(1)

// Eliminar
nombres.remove("Luis");

// Tamaño
System.out.println("Total: " + nombres.size());  // 2

// Iterar (sin orden garantizado)
for (String nombre : nombres) {
    System.out.println(nombre);
}

// Java 8+
nombres.forEach(System.out::println);
```

TREESET EN PROFUNDIDAD

TreeSet es una implementación de Set basada en un TreeMap
(árbol rojo-negro). Los elementos se almacenan ordenados según
su orden natural (Comparable) o un Comparator proporcionado.

Características:
  • Operaciones O(log n): add, remove, contains
  • Elementos siempre ordenados
  • NO permite null (desde Java 7)
  • No sincronizado
  • Implementa NavigableSet y SortedSet

Métodos adicionales de TreeSet:

```java
TreeSet<Integer> numeros = new TreeSet<>();
numeros.addAll(Set.of(5, 2, 8, 1, 9, 3));

// Navegación
numeros.first();      // 1 - primer elemento
numeros.last();       // 9 - último elemento
numeros.lower(5);     // 3 - menor que 5
numeros.higher(5);    // 8 - mayor que 5
numeros.floor(5);     // 5 - menor o igual que 5
numeros.ceiling(5);   // 5 - mayor o igual que 5

// Subconjuntos
numeros.headSet(5);   // [1, 2, 3] - menores que 5
numeros.tailSet(5);   // [5, 8, 9] - mayores o iguales que 5
numeros.subSet(2, 8); // [2, 3, 5] - entre 2 y 8 exclusivo

// Eliminación
numeros.pollFirst();  // Elimina y devuelve el primero (1)
numeros.pollLast();   // Elimina y devuelve el último (9)

// Iteración descendente
for (Integer n : numeros.descendingSet()) {
    System.out.println(n);  // 8, 5, 3, 2
}
```

Para que TreeSet funcione, los elementos deben implementar
Comparable o debes proporcionar un Comparator:

```java
// Opción 1: elementos implementan Comparable
Set<Persona> personas = new TreeSet<>();
// Persona debe implementar Comparable<Persona>

// Opción 2: proporcionar Comparator
Set<Persona> porEdad = new TreeSet<>(
    (p1, p2) -> Integer.compare(p1.getEdad(), p2.getEdad())
);
```

LINKEDHASHSET

LinkedHashSet es una implementación que combina tabla hash con
lista enlazada. Mantiene el orden de inserción de los elementos.

Características:
  • Operaciones O(1): add, remove, contains (como HashSet)
  • Mantiene orden de inserción (como LinkedList)
  • Un poco más lento que HashSet por el overhead de la lista
  • Permite un elemento null
  • Ideal cuando necesitas unicidad + orden de inserción

```java
import java.util.LinkedHashSet;
import java.util.Set;

public class LinkedHashSetExample {
    public static void main(String[] args) {
        Set<String> hashSet = new HashSet<>();
        hashSet.add("Z");
        hashSet.add("A");
        hashSet.add("M");
        hashSet.add("B");
        System.out.println("HashSet: " + hashSet);  // Orden impredecible

        Set<String> linkedHashSet = new LinkedHashSet<>();
        linkedHashSet.add("Z");
        linkedHashSet.add("A");
        linkedHashSet.add("M");
        linkedHashSet.add("B");
        System.out.println("LinkedHashSet: " + linkedHashSet);  // [Z, A, M, B]

        // Útil para eliminar duplicados manteniendo orden
        String[] palabras = {"casa", "árbol", "casa", "sol", "árbol", "luna"};
        Set<String> unicas = new LinkedHashSet<>(List.of(palabras));
        System.out.println("Sin duplicados: " + unicas);  // [casa, árbol, sol, luna]
    }
}
```

OPERACIONES DE TEORÍA DE CONJUNTOS

Set ofrece métodos para realizar operaciones matemáticas de
conjuntos:

```java
import java.util.HashSet;
import java.util.Set;

public class TeoriaConjuntos {
    public static void main(String[] args) {
        Set<Integer> A = new HashSet<>(Set.of(1, 2, 3, 4, 5));
        Set<Integer> B = new HashSet<>(Set.of(4, 5, 6, 7, 8));

        // UNIÓN: elementos en A o en B
        Set<Integer> union = new HashSet<>(A);
        union.addAll(B);
        System.out.println("A ∪ B = " + union);
        // [1, 2, 3, 4, 5, 6, 7, 8]

        // INTERSECCIÓN: elementos en A y en B
        Set<Integer> interseccion = new HashSet<>(A);
        interseccion.retainAll(B);
        System.out.println("A ∩ B = " + interseccion);
        // [4, 5]

        // DIFERENCIA: elementos en A pero no en B
        Set<Integer> diferencia = new HashSet<>(A);
        diferencia.removeAll(B);
        System.out.println("A - B = " + diferencia);
        // [1, 2, 3]

        // DIFERENCIA SIMÉTRICA: elementos en A o B, pero no en ambos
        Set<Integer> diffSimetrica = new HashSet<>(A);
        diffSimetrica.addAll(B);
        Set<Integer> interseccion2 = new HashSet<>(A);
        interseccion2.retainAll(B);
        diffSimetrica.removeAll(interseccion2);
        System.out.println("A Δ B = " + diffSimetrica);
        // [1, 2, 3, 6, 7, 8]

        // SUBCONJUNTO
        Set<Integer> C = new HashSet<>(Set.of(1, 2, 3));
        System.out.println("C ⊆ A? " + A.containsAll(C));  // true
        System.out.println("C ⊆ B? " + B.containsAll(C));  // false
    }
}
```

EJEMPLO: DETECTOR DE DUPLICADOS

```java
import java.util.HashSet;
import java.util.List;
import java.util.Set;

public class DetectorDuplicados {
    public static <T> boolean tieneDuplicados(List<T> lista) {
        Set<T> conjunto = new HashSet<>(lista);
        return conjunto.size() != lista.size();
    }

    public static <T> Set<T> encontrarDuplicados(List<T> lista) {
        Set<T> vistos = new HashSet<>();
        Set<T> duplicados = new HashSet<>();
        for (T elemento : lista) {
            if (!vistos.add(elemento)) {
                duplicados.add(elemento);
            }
        }
        return duplicados;
    }

    public static <T> List<T> sinDuplicados(List<T> lista) {
        return new java.util.ArrayList<>(new LinkedHashSet<>(lista));
    }

    public static void main(String[] args) {
        List<String> palabras = List.of(
            "casa", "perro", "casa", "gato", "perro", "sol"
        );

        System.out.println("¿Tiene duplicados? " + tieneDuplicados(palabras));
        System.out.println("Duplicados: " + encontrarDuplicados(palabras));
        System.out.println("Sin duplicados: " + sinDuplicados(palabras));
    }
}
```

EJEMPLO: GESTIÓN DE ESTUDIANTES ÚNICOS

```java
import java.util.*;

public class GestionEstudiantesUnicos {
    public static void main(String[] args) {
        Set<Estudiante> estudiantes = new HashSet<>();

        estudiantes.add(new Estudiante("A001", "Ana", 4.5));
        estudiantes.add(new Estudiante("A002", "Luis", 3.8));
        estudiantes.add(new Estudiante("A001", "Ana", 4.5));  // Duplicado

        System.out.println("Total estudiantes únicos: " + estudiantes.size());
        // 2 (el tercero es duplicado)

        // TreeSet ordenado por nota
        Set<Estudiante> porNota = new TreeSet<>(
            (e1, e2) -> Double.compare(e2.getNota(), e1.getNota())
        );
        porNota.addAll(estudiantes);
        System.out.println("\nEstudiantes ordenados por nota (descendente):");
        porNota.forEach(System.out::println);
    }
}

class Estudiante {
    private String id;
    private String nombre;
    private double nota;

    public Estudiante(String id, String nombre, double nota) {
        this.id = id;
        this.nombre = nombre;
        this.nota = nota;
    }

    public double getNota() { return nota; }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Estudiante)) return false;
        Estudiante that = (Estudiante) o;
        return Objects.equals(id, that.id);
    }

    @Override
    public int hashCode() {
        return Objects.hash(id);
    }

    @Override
    public String toString() {
        return id + " | " + nombre + " | " + nota;
    }
}
```

Nótese que la unicidad se basa en el id, no en el nombre o nota.
Dos estudiantes con el mismo id son considerados iguales.

EJEMPLO: TEXTO ÚNICO Y ORDENADO CON TREESET

```java
import java.util.*;

public class AnalizadorTexto {
    public static void main(String[] args) {
        String texto = "el sol brilla sobre el mar y el sol se oculta tras el mar";

        String[] palabras = texto.toLowerCase().split(" ");

        // HashSet: palabras únicas sin orden
        Set<String> unicas = new HashSet<>(Arrays.asList(palabras));
        System.out.println("Únicas (sin orden): " + unicas);

        // TreeSet: palabras únicas ordenadas alfabéticamente
        Set<String> ordenadas = new TreeSet<>(Arrays.asList(palabras));
        System.out.println("Únicas (ordenadas): " + ordenadas);

        // LinkedHashSet: palabras únicas en orden de aparición
        Set<String> porAparicion = new LinkedHashSet<>(Arrays.asList(palabras));
        System.out.println("Únicas (orden aparición): " + porAparicion);

        // TreeSet con orden inverso
        Set<String> inverso = new TreeSet<>(Comparator.reverseOrder());
        inverso.addAll(Arrays.asList(palabras));
        System.out.println("Únicas (orden inverso): " + inverso);

        System.out.println("\nTotal palabras: " + palabras.length);
        System.out.println("Palabras únicas: " + unicas.size());
    }
}
```

NAVEGABLE SET Y SORTED SET

TreeSet implementa SortedSet y NavigableSet, interfaces que
proporcionan métodos de navegación:

```java
NavigableSet<Integer> conjunto = new TreeSet<>();
conjunto.addAll(Set.of(10, 20, 30, 40, 50, 60, 70, 80));

// SortedSet methods
conjunto.first();           // 10
conjunto.last();            // 80
conjunto.headSet(40);       // [10, 20, 30]
conjunto.tailSet(40);       // [40, 50, 60, 70, 80]
conjunto.subSet(30, 60);    // [30, 40, 50]

// NavigableSet methods
conjunto.lower(40);         // 30 (estrictamente menor)
conjunto.floor(40);         // 40 (menor o igual)
conjunto.ceiling(45);       // 50 (mayor o igual)
conjunto.higher(40);        // 50 (estrictamente mayor)
conjunto.descendingSet();   // [80, 70, 60, 50, 40, 30, 20, 10]
conjunto.pollFirst();       // 10 (elimina y devuelve)
conjunto.pollLast();        // 80 (elimina y devuelve)

// headSet, tailSet, subSet con inclusividad
conjunto.headSet(40, true);     // incluye 40
conjunto.subSet(30, true, 60, true);  // incluye ambos extremos
```

ERRORES COMUNES Y BUENAS PRÁCTICAS

Error #1: No implementar hashCode()

```java
// MAL: sin hashCode(), cada objeto tiene código hash diferente
class Punto {
    int x, y;
    // equals implementado, pero hashCode NO
}

Set<Punto> puntos = new HashSet<>();
puntos.add(new Punto(1, 2));
puntos.add(new Punto(1, 2));
System.out.println(puntos.size());  // 2 (DEBERÍA SER 1)
```

Error #2: Objeto mutable en HashSet

```java
// MAL: modificar objeto después de añadirlo al set
Set<StringBuilder> set = new HashSet<>();
StringBuilder sb = new StringBuilder("Hola");
set.add(sb);
sb.append(" Mundo");  // Modificación muta el hash
System.out.println(set.contains(sb));  // Puede ser false
```

Error #3: Añadir null a TreeSet

```java
Set<String> tree = new TreeSet<>();
tree.add("Hola");
tree.add(null);  // NullPointerException
```

Buena práctica: Inicializar con capacidad esperada

```java
// Si esperas 10,000 elementos
// HashSet crece automáticamente, pero puedes optimizar
Set<String> set = new HashSet<>(20_000);  // factor de carga 0.75
```

Buena práctica: Usar Set.of() para conjuntos pequeños e inmutables

```java
Set<String> inmutables = Set.of("A", "B", "C");
// No permite null, no permite duplicados
```

RESUMEN DEL CAPÍTULO

• Set es una colección que no permite elementos duplicados.

• HashSet: O(1) en operaciones básicas, sin orden, requiere
  hashCode() y equals() correctamente implementados.

• TreeSet: O(log n), elementos ordenados, implementa SortedSet
  y NavigableSet para navegación.

• LinkedHashSet: O(1) como HashSet pero mantiene orden de
  inserción.

• El contrato hashCode()/equals() es fundamental para el
  correcto funcionamiento de HashSet y LinkedHashSet.

• TreeSet requiere que los elementos sean Comparable o se debe
  proporcionar un Comparator.

• Set soporta operaciones de teoría de conjuntos: addAll
  (unión), retainAll (intersección), removeAll (diferencia).

• Los objetos almacenados en un Set no deben ser mutables.

PREGUNTAS FRECUENTES

P: ¿Por qué HashSet no mantiene orden?
R: HashSet distribuye los elementos en cubetas según su hashCode.
   El orden de iteración depende del número de cubetas y de la
   distribución, que no está diseñada para ser predecible.

P: ¿Puedo tener null en un Set?
R: HashSet y LinkedHashSet permiten un único null. TreeSet NO
   permite null desde Java 7 (lanza NullPointerException).

P: ¿Cuándo usar TreeSet en lugar de HashSet?
R: Cuando necesites los elementos ordenados o necesites
   operaciones de navegación (lower, higher, subSet, etc.).

P: ¿Cómo decido qué campos incluir en equals() y hashCode()?
R: Incluye los campos que definen la identidad del objeto. Para
   una entidad de base de datos, suele ser el id. Para un valor,
   todos los campos relevantes.

P: ¿Qué pasa si cambio un objeto después de añadirlo a un HashSet?
R: Es peligroso. El objeto está en una cubeta basada en su hash
   original. Si cambias el objeto, su hashCode cambia pero no se
   recoloca. Puede volverse "invisible" para contains() y remove().

DESAFÍO DEL ARQUITECTO

MOTOR DE BÚSQUEDA DE TEXTOS (INVERTED INDEX)

Implementa un motor de búsqueda simple que indexe documentos
de texto usando conjuntos. El sistema debe:

  • Indexar múltiples documentos (cada documento tiene título
    y contenido)
  • Construir un índice invertido: Map<String, Set<String>>
    donde la clave es una palabra y el valor es el conjunto de
    títulos de documentos que contienen esa palabra
  • Almacenar las palabras en TreeSet para orden alfabético

Operaciones de búsqueda:
  • AND: documentos que contienen TODAS las palabras buscadas
  • OR: documentos que contienen ALGUNA de las palabras
  • NOT: documentos que contienen la primera pero NO la segunda
  • Frase exacta: documentos que contienen la frase completa

Requisitos adicionales:
  • Normalizar texto: minúsculas, sin puntuación
  • Ignorar stop words (artículos, preposiciones, etc.)
  • Implementar búsqueda con comodín (wildcard): "comput*" debe
    encontrar "computadora", "computación", etc.
  • Mostrar estadísticas: palabra más frecuente, documentos más
    relevantes, cobertura del índice

Extra:
  • Implementar ranking TF-IDF básico
  • Añadir búsqueda por proximidad (palabras cerca entre sí)

Este desafío integra HashSet, TreeSet y operaciones de teoría
de conjuntos en una aplicación real de recuperación de
información.
