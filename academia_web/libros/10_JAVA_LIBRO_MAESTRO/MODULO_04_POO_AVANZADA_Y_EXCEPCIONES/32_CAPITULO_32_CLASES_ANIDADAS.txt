  BIENVENIDO

  En el capítulo anterior exploraste la herencia múltiple a través de
  interfaces. Ahora nos adentraremos en un tema que demuestra la
  sofisticación del diseño de Java: las clases anidadas.

  Una clase anidada es una clase definida dentro de otra clase. Java
  ofrece cuatro tipos: clases anidadas estáticas (static nested), clases
  internas (inner classes), clases locales (local classes) y clases
  anónimas (anonymous classes). Cada una tiene propósitos, reglas de
  acceso y casos de uso específicos.

  Este capítulo cubre los primeros tres tipos. Las clases anónimas, por su
  importancia y extensión, tienen su propio capítulo (el 33).

  Dominar las clases anidadas te permitirá escribir código más cohesivo,
  encapsulado y legible, agrupando clases que lógicamente pertenecen juntas.

  CONTENIDO

  --- 1. CLASIFICACIÓN DE CLASES ANIDADAS ---

  Las clases anidadas en Java se dividen en dos categorías principales:

  1. Clases anidadas estáticas (static nested classes):
     - Se declaran con la palabra clave static.
     - No tienen referencia a la instancia de la clase contenedora.
     - Se comportan como una clase independiente pero agrupada
       lógicamente.

  2. Clases no estáticas (inner classes):
     a) Clases internas (inner classes o member inner classes):
        - Declaradas como miembros de la clase contenedora.
        - Tienen acceso implícito a los miembros de la instancia
          contenedora.
     b) Clases locales (local classes):
        - Definidas dentro de un bloque de código (método, constructor).
        - Visibles solo dentro de ese bloque.
     c) Clases anónimas (anonymous classes):
        - Clases sin nombre, se definen e instancian en una sola
          expresión.
        - Se cubren en detalle en el Capítulo 33.

  ```java
  // Estructura de clasificación

  public class ClaseExterna {
      static class AnidadaEstatica {
          // Clase anidada estática
      }

      class Interna {
          // Clase interna (miembro no estático)
      }

      public void metodo() {
          class Local {
              // Clase local (definida dentro de un método o bloque)
          }
      }
  }
  ```

  --- 2. CLASES ANIDADAS ESTÁTICAS (STATIC NESTED) ---

  Una clase anidada estática es una clase declarada como static dentro
  de otra clase. Se comporta como una clase normal pero su nombre está
  anclado al de la clase contenedora.

  ```java
  public class Continente {
      private String nombre;

      public Continente(String nombre) {
          this.nombre = nombre;
      }

      // Clase anidada estática
      public static class Pais {
          private String nombre;
          private String capital;

          public Pais(String nombre, String capital) {
              this.nombre = nombre;
              this.capital = capital;
          }

          public void mostrarInfo() {
              System.out.println("País: " + nombre);
              System.out.println("Capital: " + capital);
              // No puede acceder a 'nombre' de Continente directamente
          }
      }
  }

  public class PruebaEstatica {
      public static void main(String[] args) {
          // Creación de la clase anidada estática
          Continente.Pais pais = new Continente.Pais("Perú", "Lima");
          pais.mostrarInfo();
      }
  }
  ```

  Características:
  - No requiere una instancia de la clase contenedora.
  - Se instancia como: Contenedora.Anidada obj = new Contenedora.Anidada();
  - Puede acceder a miembros estáticos de la clase contenedora.
  - NO puede acceder a miembros no estáticos de la contenedora (no tiene
    referencia implícita).
  - Puede tener miembros estáticos y no estáticos propios.

  ```java
  // Ejemplo con acceso a miembros estáticos

  public class Configuracion {
      private static String appName = "JavaMaster";
      private String version = "1.0";  // No estático

      public static class Validador {
          public void validar() {
              // Acceso permitido a miembro estático
              System.out.println("Validando: " + appName);

              // Error de compilación: no puede acceder a 'version'
              // System.out.println("Versión: " + version);
          }

          // Puede tener sus propios miembros estáticos
          public static void mostrarInfo() {
              System.out.println("Validador de " + appName);
          }
      }
  }
  ```

  --- 3. PATRÓN BUILDER CON CLASE ANIDADA ESTÁTICA ---

  Un uso clásico de las clases anidadas estáticas es el patrón Builder,
  que permite construir objetos complejos con muchos parámetros opcionales:

  ```java
  public class Computadora {
      private String procesador;
      private int ramGB;
      private int almacenamientoGB;
      private String tarjetaGrafica;
      private boolean tieneSSD;
      private String sistemaOperativo;

      // Constructor privado: solo el Builder puede crear instancias
      private Computadora(Builder builder) {
          this.procesador = builder.procesador;
          this.ramGB = builder.ramGB;
          this.almacenamientoGB = builder.almacenamientoGB;
          this.tarjetaGrafica = builder.tarjetaGrafica;
          this.tieneSSD = builder.tieneSSD;
          this.sistemaOperativo = builder.sistemaOperativo;
      }

      // Clase anidada estática Builder
      public static class Builder {
          private String procesador;
          private int ramGB;
          private int almacenamientoGB;
          private String tarjetaGrafica = "Integrada";
          private boolean tieneSSD = true;
          private String sistemaOperativo = "Windows 10";

          public Builder(String procesador, int ramGB) {
              this.procesador = procesador;
              this.ramGB = ramGB;
          }

          public Builder almacenamiento(int gb) {
              this.almacenamientoGB = gb;
              return this;
          }

          public Builder tarjetaGrafica(String tg) {
              this.tarjetaGrafica = tg;
              return this;
          }

          public Builder ssd(boolean ssd) {
              this.tieneSSD = ssd;
              return this;
          }

          public Builder sistemaOperativo(String so) {
              this.sistemaOperativo = so;
              return this;
          }

          public Computadora build() {
              return new Computadora(this);
          }
      }

      @Override
      public String toString() {
          return "Computadora [CPU=" + procesador + ", RAM=" + ramGB
                  + "GB, Almacenamiento=" + almacenamientoGB
                  + "GB, GPU=" + tarjetaGrafica
                  + ", SSD=" + (tieneSSD ? "Sí" : "No")
                  + ", SO=" + sistemaOperativo + "]";
      }
  }

  public class PruebaBuilder {
      public static void main(String[] args) {
          Computadora pc = new Computadora.Builder("Intel i7", 16)
                  .almacenamiento(512)
                  .tarjetaGrafica("NVIDIA RTX 3060")
                  .ssd(true)
                  .sistemaOperativo("Linux Ubuntu")
                  .build();

          System.out.println(pc);
      }
  }
  ```

  --- 4. CLASES INTERNAS (INNER CLASSES) ---

  Una clase interna es una clase no estática definida como miembro de otra
  clase. A diferencia de las clases anidadas estáticas, una inner class
  tiene acceso implícito a todos los miembros de la instancia contenedora,
  incluso los privados.

  ```java
  public class Biblioteca {
      private String nombre;
      private Libro[] libros;
      private int contador;

      public Biblioteca(String nombre, int capacidad) {
          this.nombre = nombre;
          this.libros = new Libro[capacidad];
          this.contador = 0;
      }

      public void agregarLibro(String titulo, String autor) {
          if (contador < libros.length) {
              libros[contador++] = new Libro(titulo, autor);
          }
      }

      // Clase interna
      public class Libro {
          private String titulo;
          private String autor;

          public Libro(String titulo, String autor) {
              this.titulo = titulo;
              this.autor = autor;
          }

          public void mostrarInfo() {
              // Acceso al miembro privado de la clase externa
              System.out.println("Biblioteca: " + nombre);
              System.out.println("  Título: " + titulo);
              System.out.println("  Autor: " + autor);
          }
      }

      public void listarLibros() {
          for (int i = 0; i < contador; i++) {
              libros[i].mostrarInfo();
          }
      }
  }

  public class PruebaInner {
      public static void main(String[] args) {
          Biblioteca biblio = new Biblioteca("Biblioteca Nacional", 5);

          biblio.agregarLibro("Cien Años de Soledad", "Gabriel García Márquez");
          biblio.agregarLibro("El Quijote", "Miguel de Cervantes");

          biblio.listarLibros();

          // Crear inner class desde fuera
          Biblioteca.Libro libro = biblio.new Libro("1984", "George Orwell");
          libro.mostrarInfo();
      }
  }
  ```

  Sintaxis de creación desde fuera de la clase contenedora:
  ```java
  ClaseExterna.InnerClass obj = instanciaExterna.new InnerClass(args);
  ```

  --- 5. REFERENCIA A LA INSTANCIA CONTENEDORA ---

  Una inner class guarda una referencia implícita a la instancia de la
  clase externa que la creó. Puedes acceder a ella con ClaseExterna.this:

  ```java
  public class Reloj {
      private String marca;
      private int hora = 0;
      private int minuto = 0;

      public Reloj(String marca) {
          this.marca = marca;
      }

      public void setTiempo(int h, int m) {
          this.hora = h;
          this.minuto = m;
      }

      public class Alarma {
          private int horaAlarma;
          private int minutoAlarma;

          public Alarma(int h, int m) {
              this.horaAlarma = h;
              this.minutoAlarma = m;
          }

          public void activar() {
              // Uso de la referencia explícita a la clase externa
              if (Reloj.this.hora == horaAlarma
                      && Reloj.this.minuto == minutoAlarma) {
                  System.out.println("¡ALARMA SONANDO! (" + marca + ")");
              } else {
                  System.out.println("Esperando alarma...");
                  System.out.println("Reloj: " + Reloj.this.hora + ":"
                          + Reloj.this.minuto);
                  System.out.println("Alarma configurada: "
                          + horaAlarma + ":" + minutoAlarma);
              }
          }

          // Si un método de la inner class oculta un nombre de la externa
          public void mostrarMarca() {
              String marca = "Alarma local";
              System.out.println(marca);              // Variable local
              System.out.println(this.marca);         // Variable de inner (error no existe)
              System.out.println(Reloj.this.marca);   // Variable de clase externa
          }
      }

      public Alarma crearAlarma(int h, int m) {
          return new Alarma(h, m);
      }
  }

  public class PruebaAlarma {
      public static void main(String[] args) {
          Reloj reloj = new Reloj("Casio");
          reloj.setTiempo(7, 30);

          Reloj.Alarma alarma = reloj.crearAlarma(7, 30);
          alarma.activar();

          System.out.println();

          reloj.setTiempo(8, 0);
          alarma.activar();
      }
  }
  ```

  --- 6. CLASES LOCALES (LOCAL CLASSES) ---

  Las clases locales se definen dentro de un bloque de código, típicamente
  un método. Son visibles solo dentro de ese bloque y tienen acceso a las
  variables finales (o effectively final) del método que las contiene.

  ```java
  import java.time.LocalDate;
  import java.time.format.DateTimeFormatter;

  public class GestorEventos {
      public void crearEvento(String nombre, int diasHastaEvento) {
          // Variable local (debe ser effectively final para usarse en la clase local)
          LocalDate hoy = LocalDate.now();
          LocalDate fechaEvento = hoy.plusDays(diasHastaEvento);

          // Clase local dentro del método
          class Evento {
              private String nombre;
              private LocalDate fecha;
              private int diasRestantes;

              public Evento(String nombre, LocalDate fecha) {
                  this.nombre = nombre;
                  this.fecha = fecha;
                  this.diasRestantes = LocalDate.now().until(fecha).getDays();
              }

              public void mostrarDetalles() {
                  // Accede a variable del método: hoy (effectively final)
                  System.out.println("=== Evento: " + nombre + " ===");
                  System.out.println("Fecha: " + fecha.format(
                          DateTimeFormatter.ofPattern("dd/MM/yyyy")));
                  System.out.println("Días restantes: " + diasRestantes);
                  System.out.println("Creado desde: " + hoy);

                  // No puede modificar 'hoy' ni 'diasHastaEvento'
              }
          }

          // Uso de la clase local
          Evento evento = new Evento(nombre, fechaEvento);
          evento.mostrarDetalles();
      }

      public void procesarLista(String[] items) {
          // Otra clase local en otro método
          class Procesador {
              public void imprimirItems() {
                  for (int i = 0; i < items.length; i++) {
                      System.out.println((i + 1) + ". " + items[i]);
                  }
              }
          }

          Procesador p = new Procesador();
          p.imprimirItems();
      }
  }

  public class PruebaLocal {
      public static void main(String[] args) {
          GestorEventos gestor = new GestorEventos();
          gestor.crearEvento("Conferencia Java", 30);
          System.out.println();
          gestor.procesarLista(new String[]{"Leche", "Pan", "Huevos"});
      }
  }
  ```

  Características:
  - Definidas dentro de un bloque (método, constructor, inicializador).
  - Visibles solo dentro del bloque donde se definen.
  - Pueden acceder a miembros de la clase contenedora.
  - Pueden acceder a variables locales del método si son final o
    effectively final (Java 8+).
  - No pueden tener declaraciones static (excepto constantes).
  - No pueden tener modificadores de acceso (public, private, protected).

  ```java
  // Ejemplo: effectively final (Java 8+)

  public class Calculadora {
      public int ejecutarOperacion(int a, int b, String tipo) {
          // 'tipo' debe ser effectively final (no se reasigna después de
          // usarse en la clase local)

          class Suma {
              public int calcular() {
                  // 'tipo' es effectively final: no se modifica después
                  if (tipo.equals("suma")) {
                      return a + b;
                  }
                  return 0;
              }
          }

          // Esto causaría error de compilación si se descomenta:
          // tipo = "resta";  // Ya no sería effectively final

          return new Suma().calcular();
      }
  }
  ```

  --- 7. SOMBREAMIENTO Y ACCESO A MIEMBROS ---

  Es posible que nombres de miembros se superpongan entre la clase externa,
  la clase interna y variables locales. Java proporciona reglas claras de
  resolución:

  ```java
  public class Sombreado {
      private String x = "Variable de instancia de Sombreado";

      public class Interna {
          private String x = "Variable de instancia de Interna";

          public void metodo(final String x) {
              // Parámetro del método
              System.out.println("Parámetro x: " + x);

              // Variable de instancia de Interna
              System.out.println("Interna.x: " + this.x);

              // Variable de instancia de Sombreado
              System.out.println("Sombreado.this.x: " + Sombreado.this.x);
          }
      }

      public void probar() {
          Interna i = new Interna();
          i.metodo("Parámetro pasado");
      }
  }

  public class PruebaSombreado {
      public static void main(String[] args) {
          new Sombreado().probar();
      }
  }
  ```

  Salida:
  ```
  Parámetro x: Parámetro pasado
  Interna.x: Variable de instancia de Interna
  Sombreado.this.x: Variable de instancia de Sombreado
  ```

  --- 8. CLASES ANIDADAS Y HERENCIA ---

  Las clases anidadas pueden extenderse y también pueden heredar de otras
  clases:

  ```java
  public class Empresa {
      private String nombre;

      public Empresa(String nombre) {
          this.nombre = nombre;
      }

      // Clase interna que puede ser extendida
      public class Empleado {
          protected String nombreEmpleado;
          protected double salario;

          public Empleado(String nombre, double salario) {
              this.nombreEmpleado = nombre;
              this.salario = salario;
          }

          public void mostrarInfo() {
              System.out.println("Empresa: " + Empresa.this.nombre);
              System.out.println("Empleado: " + nombreEmpleado);
              System.out.println("Salario: $" + salario);
          }
      }

      // Clase anidada estática
      public static class Departamento {
          private String nombre;

          public Departamento(String nombre) {
              this.nombre = nombre;
          }

          public void mostrarNombre() {
              System.out.println("Departamento: " + nombre);
          }
      }
  }

  // Extensión de inner class fuera de la clase contenedora
  class Gerente extends Empresa.Empleado {
      private double bono;

      public Gerente(Empresa empresa, String nombre,
                     double salario, double bono) {
          empresa.super(nombre, salario);  // Sintaxis especial
          this.bono = bono;
      }

      @Override
      public void mostrarInfo() {
          super.mostrarInfo();
          System.out.println("Bono: $" + bono);
          System.out.println("Total: $" + (salario + bono));
      }
  }

  public class PruebaHerencia {
      public static void main(String[] args) {
          Empresa empresa = new Empresa("TechCorp");

          Empresa.Empleado emp = empresa.new Empleado("Ana López", 3500);
          emp.mostrarInfo();

          System.out.println();

          Gerente gerente = new Gerente(empresa, "Carlos Ruiz", 6000, 2000);
          gerente.mostrarInfo();

          System.out.println();

          Empresa.Departamento dept = new Empresa.Departamento("IT");
          dept.mostrarNombre();
      }
  }
  ```

  La sintaxis para extender una inner class es:
  ```java
  class Hija extends Externa.Interna {
      Hija(Externa ref, args) {
          ref.super(args);
      }
  }
  ```

  --- 9. CLASES INTERNAS ANÓNIMAS (AVANCE) ---

  Aunque las clases anónimas se cubren a fondo en el capítulo 33, vale la
  pena ver un ejemplo básico aquí para entender su relación con las clases
  locales:

  ```java
  public class PruebaRapida {
      public static void main(String[] args) {
          // Clase anónima implementando una interfaz
          Runnable tarea = new Runnable() {
              @Override
              public void run() {
                  System.out.println("Ejecutando tarea desde clase anónima");
              }
          };

          Thread hilo = new Thread(tarea);
          hilo.start();
      }
  }
  ```

  Las clases anónimas son esencialmente clases locales sin nombre que se
  definen e instancian en una sola expresión.

  --- 10. BUENAS PRÁCTICAS ---

  1. Usa clases anidadas estáticas cuando la clase auxiliar esté
     estrechamente asociada con la clase externa pero no necesite acceso
     a su instancia (ej. Builder, comparadores).

  2. Usa inner classes cuando necesites acceso a miembros privados de la
     clase contenedora y la relación sea parte del diseño (ej. iteradores,
     nodos de listas enlazadas).

  3. Usa clases locales cuando la lógica solo sea relevante dentro de un
     método específico y no merezca una clase separada.

  4. Evita inner classes muy grandes. Si la clase interna supera las 20-30
     líneas, considera convertirla en una clase separada.

  5. Prefiere clases anidadas estáticas sobre inner classes a menos que
     necesites explícitamente acceso a la instancia contenedora.

  6. Documenta bien la relación entre la clase contenedora y sus clases
     anidadas.

  7. Ten cuidado con el sombreamiento: usa ClaseExterna.this.miembro para
     desambiguar.

  8. Para construir objetos complejos, el patrón Builder con una clase
     anidada estática es la mejor opción.

  ```java
  // Ejemplo de buenas prácticas: Iterador con inner class

  import java.util.Iterator;
  import java.util.NoSuchElementException;

  public class ListaEnlazada<T> implements Iterable<T> {
      private Nodo<T> cabeza;

      public void agregar(T dato) {
          Nodo<T> nuevo = new Nodo<>(dato);
          if (cabeza == null) {
              cabeza = nuevo;
          } else {
              Nodo<T> actual = cabeza;
              while (actual.siguiente != null) {
                  actual = actual.siguiente;
              }
              actual.siguiente = nuevo;
          }
      }

      // Inner class privada para nodos (acceso a datos internos)
      private static class Nodo<T> {
          T dato;
          Nodo<T> siguiente;

          Nodo(T dato) {
              this.dato = dato;
          }
      }

      @Override
      public Iterator<T> iterator() {
          return new IteradorLista();
      }

      // Inner class que implementa el iterador
      private class IteradorLista implements Iterator<T> {
          private Nodo<T> actual = cabeza;

          @Override
          public boolean hasNext() {
              return actual != null;
          }

          @Override
          public T next() {
              if (!hasNext()) {
                  throw new NoSuchElementException();
              }
              T dato = actual.dato;
              actual = actual.siguiente;
              return dato;
          }
      }

      // Clase anidada estática: fábrica
      public static <T> ListaEnlazada<T> of(T... elementos) {
          ListaEnlazada<T> lista = new ListaEnlazada<>();
          for (T elem : elementos) {
              lista.agregar(elem);
          }
          return lista;
      }
  }

  public class PruebaLista {
      public static void main(String[] args) {
          ListaEnlazada<String> lista = ListaEnlazada.of("A", "B", "C");

          for (String s : lista) {
              System.out.println(s);
          }
      }
  }
  ```

  --- 11. COMPARATIVA DE TIPOS DE CLASES ANIDADAS ---

  +========================+============+============+============+
  | Característica         | Estática   | Interna    | Local      |
  +========================+============+============+============+
  | Requiere instancia     | No         | Sí         | Depende*   |
  | externa                |            |            |            |
  +------------------------+------------+------------+------------+
  | Acceso a miembros      | Solo       | Todos      | Todos      |
  | externos               | estáticos  |            |            |
  +------------------------+------------+------------+------------+
  | Puede tener miembros   | Sí         | No (solo   | No (solo   |
  | static                 |            | constantes)| constantes)|
  +------------------------+------------+------------+------------+
  | Visibilidad fuera      | Sí, con    | Sí, con    | No         |
  | de la contenedora      | nombre     | nombre     |            |
  +------------------------+------------+------------+------------+
  | Puede ser declarada    | Sí         | Sí         | No (solo   |
  | como public/private    |            |            | en bloque) |
  +------------------------+------------+------------+------------+
  | Caso de uso típico     | Builder,   | Iterador,  | Lógica     |
  |                        | operaciones| evento,    | temporal   |
  |                        | agrupadas  | nodo       | en método  |
  +------------------------+------------+------------+------------+

  *Una clase local necesita una instancia de la clase contenedora solo si
  accede a miembros de instancia.

  PREGUNTAS FRECUENTES (FAQ)

  1. ¿Cuál es la diferencia entre clase anidada estática e interna?
     La estática no tiene referencia a la instancia contenedora; la
     interna sí, y por tanto puede acceder a sus miembros no estáticos.

  2. ¿Se puede instanciar una inner class sin una instancia de la externa?
     No. Necesitas una instancia de la clase externa: externa.new Interna().

  3. ¿Las clases locales pueden acceder a variables del método?
     Sí, siempre que sean final o effectively final (no se reasignen).

  4. ¿Puedo extender una inner class desde fuera de la clase contenedora?
     Sí, pero necesitas una instancia de la externa: ref.super(args).

  5. ¿Qué ventajas tienen las clases anidadas?
     Encapsulación mejorada, código más legible al agrupar clases
     relacionadas, y acceso a miembros privados.

  6. ¿Las clases anidadas generan archivos .class separados?
     Sí. Por ejemplo, Externa$Interna.class para inner classes,
     Externa$1.class para clases anónimas.

  7. ¿Puedo tener una clase anidada dentro de otra clase anidada?
     Sí. Java permite anidamiento en múltiples niveles.

  8. ¿Las interfaces pueden tener clases anidadas?
     Sí. Las interfaces pueden contener clases anidadas (implícitamente
     públicas y estáticas).

  ```java
  public interface Operable {
      void operar();

      // Clase anidada dentro de una interfaz
      class Util {
          public static void mostrarEstado() {
              System.out.println("Estado operativo");
          }
      }
  }
  ```

  RESUMEN DEL CAPÍTULO

  En este capítulo has aprendido:

  • Java soporta cuatro tipos de clases anidadas: estáticas, internas
    (miembro), locales y anónimas.

  • Las clases anidadas estáticas no requieren instancia de la contenedora
    y no acceden a miembros no estáticos de esta.

  • Las inner classes tienen acceso completo a todos los miembros de la
    clase contenedora (incluso privados).

  • Las clases locales se definen dentro de un método y solo son visibles
    allí. Pueden acceder a variables effectively final del bloque.

  • La sintaxis ref.super() permite extender inner classes fuera de la
    contenedora.

  • El patrón Builder se implementa limpiamente con clases anidadas
    estáticas.

  • Las inner classes son ideales para implementar iteradores, nodos y
    otros elementos con fuerte acoplamiento lógico.

  • Se debe preferir composición y clases anidadas estáticas sobre inner
    classes a menos que se necesite acceso explícito a la instancia
    contenedora.
