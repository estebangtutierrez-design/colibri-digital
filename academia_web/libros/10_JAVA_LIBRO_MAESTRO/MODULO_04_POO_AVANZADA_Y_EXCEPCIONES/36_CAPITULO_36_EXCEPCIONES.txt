  --- 1. JERARQUÍA DE EXCEPCIONES ---

  Todas las excepciones en Java son subclases de Throwable:

  ```
  Object
    └── Throwable
          ├── Error (irrecuperable, no se debe capturar)
          │     ├── OutOfMemoryError
          │     ├── StackOverflowError
          │     └── ...
          └── Exception (recuperable, se debe manejar)
                ├── IOException (checked)
                │     ├── FileNotFoundException
                │     ├── EOFException
                │     └── ...
                ├── SQLException (checked)
                ├── ClassNotFoundException (checked)
                └── RuntimeException (unchecked)
                      ├── NullPointerException
                      ├── IllegalArgumentException
                      ├── IndexOutOfBoundsException
                      ├── ArithmeticException
                      └── ...
  ```

  ```java
  public class JerarquiaExcepciones {
      public static void main(String[] args) {
          // Error: no se debe capturar (irrecuperable)
          // throw new OutOfMemoryError("Memoria insuficiente");

          // RuntimeException (unchecked): no obligatorio capturar
          try {
              int division = 10 / 0;
          } catch (ArithmeticException e) {
              System.out.println("Error aritmético: " + e.getMessage());
          }

          // Exception (checked): obligatorio manejar o declarar
          try {
              throw new Exception("Excepción checked");
          } catch (Exception e) {
              System.out.println("Excepción capturada: " + e.getMessage());
          }
      }
  }
  ```

  --- 2. EXCEPCIONES CHECKED VS UNCHECKED ---

  Las excepciones checked son aquellas que el compilador obliga a manejar
  (mediante try-catch o throws). Son subclases de Exception pero no de
  RuntimeException. Ejemplos: IOException, SQLException.

  Las excepciones unchecked son subclases de RuntimeException. El
  compilador no obliga a manejarlas, aunque es buena práctica hacerlo.
  Ejemplos: NullPointerException, IllegalArgumentException.

  ```java
  import java.io.*;

  public class CheckedVsUnchecked {
      public static void main(String[] args) {
          // Unchecked: compila sin try-catch (pero podría fallar en runtime)
          String texto = null;
          // texto.length();  // NullPointerException en runtime

          // Checked: el compilador obliga a manejar o declarar
          try {
              FileReader file = new FileReader("archivo.txt");
          } catch (FileNotFoundException e) {
              System.out.println("Archivo no encontrado");
          }
      }

      // Método que declara excepción checked con throws
      public static void leerArchivo() throws IOException {
          FileReader file = new FileReader("archivo.txt");
          file.read();
          file.close();
      }

      // Método que puede lanzar unchecked (no requiere declaración)
      public static int dividir(int a, int b) {
          if (b == 0) {
              throw new ArithmeticException("División por cero");
          }
          return a / b;
      }
  }
  ```

  Regla general: usa excepciones checked para condiciones razonablemente
  recuperables que el llamador debe manejar. Usa unchecked para errores
  de programación (bugs) o condiciones que no se espera manejar.

  --- 3. TRY-CATCH ---

  El bloque try rodea el código que puede lanzar una excepción. El bloque
  catch maneja la excepción.

  ```java
  import java.util.*;

  public class TryCatchBasico {
      public static void main(String[] args) {
          Scanner scanner = new Scanner(System.in);

          System.out.print("Ingresa un número: ");
          String entrada = scanner.nextLine();

          try {
              int numero = Integer.parseInt(entrada);
              System.out.println("Número válido: " + numero);
          } catch (NumberFormatException e) {
              System.out.println("Error: '" + entrada + "' no es un número válido.");
              System.out.println("Detalle: " + e.getMessage());
          }

          // Múltiples catch
          try {
              int[] numeros = {1, 2, 3};
              System.out.println(numeros[5]);  // IndexOutOfBounds
          } catch (ArrayIndexOutOfBoundsException e) {
              System.out.println("Índice fuera de rango: " + e.getMessage());
          } catch (NullPointerException e) {
              System.out.println("Array nulo: " + e.getMessage());
          }

          // El orden de los catch importa (de más específico a más general)
          try {
              Object obj = "Hola";
              Integer num = (Integer) obj;  // ClassCastException
          } catch (ClassCastException e) {
              System.out.println("Error de casteo: " + e.getMessage());
          } catch (RuntimeException e) {
              System.out.println("Error en runtime: " + e.getMessage());
          } catch (Exception e) {
              System.out.println("Error general: " + e.getMessage());
          }

          scanner.close();
      }
  }
  ```

  --- 4. MULTI-CATCH (JAVA 7+) ---

  Desde Java 7, puedes capturar múltiples tipos de excepción en un solo
  bloque catch separándolos con |.

  ```java
  import java.util.*;
  import java.io.*;

  public class MultiCatch {
      public static void main(String[] args) {
          // Sin multi-catch (Java 6 y anteriores)
          try {
              int[] arr = new int[5];
              arr[10] = 10 / 0;
          } catch (ArrayIndexOutOfBoundsException e) {
              System.out.println("Error de índice");
          } catch (ArithmeticException e) {
              System.out.println("Error aritmético");
          }

          // Con multi-catch (Java 7+)
          try {
              int[] arr = new int[5];
              arr[10] = 10 / 0;
          } catch (ArrayIndexOutOfBoundsException | ArithmeticException e) {
              System.out.println("Error de operación: " + e.getClass().getSimpleName());
              // e es efectivamente final (no se puede reasignar)
          }

          // Ejemplo más realista
          try {
              leerConfiguracion("config.txt");
          } catch (IOException | NumberFormatException e) {
              System.err.println("Error al leer configuración: " + e.getMessage());
          }
      }

      public static void leerConfiguracion(String archivo)
              throws IOException, NumberFormatException {
          BufferedReader br = new BufferedReader(new FileReader(archivo));
          String linea = br.readLine();
          int valor = Integer.parseInt(linea);
          System.out.println("Valor de configuración: " + valor);
          br.close();
      }
  }
  ```

  --- 5. FINALLY ---

  El bloque finally se ejecuta siempre, haya o no excepción. Es ideal
  para liberar recursos (cerrar archivos, conexiones, etc.).

  ```java
  import java.io.*;

  public class BloqueFinally {
      public static void main(String[] args) {
          BufferedReader reader = null;
          try {
              reader = new BufferedReader(new FileReader("datos.txt"));
              String linea = reader.readLine();
              System.out.println("Contenido: " + linea);
          } catch (FileNotFoundException e) {
              System.out.println("Archivo no encontrado");
          } catch (IOException e) {
              System.out.println("Error de lectura: " + e.getMessage());
          } finally {
              // Se ejecuta SIEMPRE: con o sin excepción
              System.out.println("Cerrando recursos...");
              try {
                  if (reader != null) {
                      reader.close();
                      System.out.println("Recurso cerrado exitosamente.");
                  }
              } catch (IOException e) {
                  System.out.println("Error al cerrar: " + e.getMessage());
              }
          }

          // finally se ejecuta incluso con return
          System.out.println("Resultado: " + metodoConFinally());
      }

      public static int metodoConFinally() {
          try {
              return 42;
          } finally {
              System.out.println("Finally se ejecuta ANTES del return");
              // Aquí no se recomienda poner return (oculta el return anterior)
          }
      }
  }
  ```

  Casos donde finally NO se ejecuta:
  1. System.exit() en el try.
  2. JVM crash.
  3. Hilo interrumpido (Thread.stop).

  ```java
  public class FinallyNoEjecutado {
      public static void main(String[] args) {
          try {
              System.out.println("Dentro del try");
              System.exit(0);  // JVM termina aquí
          } finally {
              System.out.println("Esto NO se imprime");
          }
      }
  }
  ```

  --- 6. TRY-WITH-RESOURCES (JAVA 7+) ---

  El try-with-resources cierra automáticamente los recursos que
  implementan AutoCloseable o Closeable. No necesitas finally explícito.

  ```java
  import java.io.*;
  import java.util.*;

  public class TryWithResources {
      public static void main(String[] args) {
          // Sintaxis básica
          try (BufferedReader br = new BufferedReader(
                  new FileReader("archivo.txt"))) {
              String linea;
              while ((linea = br.readLine()) != null) {
                  System.out.println(linea);
              }
          } catch (FileNotFoundException e) {
              System.out.println("Archivo no encontrado");
          } catch (IOException e) {
              System.out.println("Error de E/S: " + e.getMessage());
          }
          // br se cierra automáticamente aquí (incluso con excepción)

          // Múltiples recursos
          try (BufferedReader br = new BufferedReader(
                  new FileReader("entrada.txt"));
               BufferedWriter bw = new BufferedWriter(
                  new FileWriter("salida.txt"))) {

              String linea;
              while ((linea = br.readLine()) != null) {
                  bw.write(linea);
                  bw.newLine();
              }
              System.out.println("Archivo copiado exitosamente.");
          } catch (IOException e) {
              System.out.println("Error de E/S: " + e.getMessage());
          }

          // Recursos declarados antes (Java 9+)
          BufferedReader br = new BufferedReader(
                  new FileReader("archivo.txt"));
          try (br) {  // Java 9+: variable effectively final
              System.out.println(br.readLine());
          } catch (IOException e) {
              System.out.println("Error: " + e.getMessage());
          }

          // AutoCloseable: clase personalizada
          try (RecursoPersonalizado r = new RecursoPersonalizado()) {
              r.usar();
          } catch (Exception e) {
              System.out.println("Error: " + e.getMessage());
          }
      }
  }

  class RecursoPersonalizado implements AutoCloseable {
      public RecursoPersonalizado() {
          System.out.println("Recurso abierto");
      }

      public void usar() {
          System.out.println("Usando recurso...");
          if (Math.random() > 0.5) {
              throw new RuntimeException("Error en recurso");
          }
      }

      @Override
      public void close() throws Exception {
          System.out.println("Recurso cerrado automáticamente");
      }
  }
  ```

  --- 7. THROWS: DECLARAR EXCEPCIONES ---

  La palabra clave throws en la firma del método indica qué excepciones
  checked puede lanzar el método. Los llamadores deben manejarlas o
  declararlas a su vez.

  ```java
  import java.io.*;

  public class DeclaracionThrows {
      public static void main(String[] args) {
          try {
              leerArchivo("datos.txt");
              procesarConfiguracion("config.xml");
          } catch (IOException e) {
              System.out.println("Error de E/S: " + e.getMessage());
          } catch (ConfiguracionInvalidaException e) {
              System.out.println("Config inválida: " + e.getMessage());
          }
      }

      // throws con múltiples excepciones checked
      public static void leerArchivo(String path)
              throws FileNotFoundException, IOException {
          if (path == null) {
              throw new IllegalArgumentException("Path no puede ser nulo");  // unchecked
          }
          if (!path.endsWith(".txt")) {
              throw new FileNotFoundException("Formato incorrecto: " + path);
          }

          BufferedReader br = new BufferedReader(new FileReader(path));
          // Puede lanzar IOException
          String linea = br.readLine();
          System.out.println("Leído: " + linea);
          br.close();
      }

      // throws con excepción personalizada
      public static void procesarConfiguracion(String archivo)
              throws ConfiguracionInvalidaException {
          // Simular error de configuración
          throw new ConfiguracionInvalidaException(
                  "El archivo " + archivo + " tiene formato inválido");
      }
  }

  class ConfiguracionInvalidaException extends Exception {
      public ConfiguracionInvalidaException(String mensaje) {
          super(mensaje);
      }
  }
  ```

  --- 8. THROW: LANZAR EXCEPCIONES ---

  La palabra clave throw se usa para lanzar una excepción explícitamente.

  ```java
  import java.util.*;

  public class LanzarExcepciones {
      public static void main(String[] args) {
          try {
              validarEdad(-5);
          } catch (IllegalArgumentException e) {
              System.out.println("Validación: " + e.getMessage());
          }

          try {
              retirarDinero(500, 2000);
          } catch (SaldoInsuficienteException e) {
              System.out.println("Error bancario: " + e.getMessage());
              System.out.println("Saldo actual: " + e.getSaldoActual());
              System.out.println("Intento retiro: " + e.getMontoIntento());
          }

          // Re-lanzar excepción (con o sin cambios)
          try {
              metodoInterno();
          } catch (Exception e) {
              // Relanzar con más información
              throw new RuntimeException("Error en main", e);
          }
      }

      public static void validarEdad(int edad) {
          if (edad < 0) {
              throw new IllegalArgumentException(
                      "La edad no puede ser negativa: " + edad);
          }
          if (edad > 150) {
              throw new IllegalArgumentException(
                      "Edad fuera de rango: " + edad);
          }
          System.out.println("Edad válida: " + edad);
      }

      public static void retirarDinero(double saldo, double monto)
              throws SaldoInsuficienteException {
          if (monto > saldo) {
              throw new SaldoInsuficienteException(saldo, monto);
          }
          System.out.println("Retiro exitoso: $" + monto);
      }

      public static void metodoInterno() throws Exception {
          try {
              int resultado = 10 / 0;
          } catch (ArithmeticException e) {
              // Relanzar como otro tipo
              throw new Exception("Error en cálculo", e);
          }
      }
  }

  class SaldoInsuficienteException extends Exception {
      private double saldoActual;
      private double montoIntento;

      public SaldoInsuficienteException(double saldo, double monto) {
          super("Saldo insuficiente: $" + saldo + " para retirar $" + monto);
          this.saldoActual = saldo;
          this.montoIntento = monto;
      }

      public double getSaldoActual() { return saldoActual; }
      public double getMontoIntento() { return montoIntento; }
  }
  ```

  --- 9. ENCADENAMIENTO DE EXCEPCIONES ---

  Java permite encadenar excepciones, preservando la causa original del
  error. El constructor de Throwable acepta una causa.

  ```java
  import java.io.*;

  public class Encadenamiento {
      public static void main(String[] args) {
          try {
              leerDatosUsuario("usuarios.txt");
          } catch (ProcesamientoException e) {
              System.out.println("Error en procesamiento:");
              System.out.println("  Mensaje: " + e.getMessage());

              // Obtener la causa original
              Throwable causa = e.getCause();
              System.out.println("  Causa: " + causa.getClass().getName());
              System.out.println("  Causa msg: " + causa.getMessage());
          }
      }

      public static void leerDatosUsuario(String archivo)
              throws ProcesamientoException {
          try {
              BufferedReader br = new BufferedReader(
                      new FileReader(archivo));
              String linea = br.readLine();
              System.out.println("Datos: " + linea);
              br.close();
          } catch (FileNotFoundException e) {
              // Encadenar la excepción original
              throw new ProcesamientoException(
                      "No se encontró el archivo de usuarios", e);
          } catch (IOException e) {
              throw new ProcesamientoException(
                      "Error leyendo datos de usuarios", e);
          }
      }
  }

  class ProcesamientoException extends Exception {
      public ProcesamientoException(String mensaje, Throwable causa) {
          super(mensaje, causa);
      }

      public ProcesamientoException(String mensaje) {
          super(mensaje);
      }
  }
  ```

  --- 10. TRY CON RECURSOS MÚLTIPLES Y SUPRESIÓN ---

  Cuando se usa try-with-resources, si ocurre una excepción tanto en el
  try como en el close(), la del close() se suprime (suppressed). Puedes
  recuperar las excepciones suprimidas.

  ```java
  import java.io.*;

  public class SupresionExcepciones {
      public static void main(String[] args) {
          try {
              leerConError();
          } catch (Exception e) {
              System.out.println("Excepción principal: " + e.getMessage());
              System.out.println("Supimidas: "
                      + e.getSuppressed().length);

              for (Throwable s : e.getSuppressed()) {
                  System.out.println("  Suprimida: " + s.getMessage());
              }
          }
      }

      public static void leerConError() throws Exception {
          try (RecursoQueFalla r1 = new RecursoQueFalla("Recurso1");
               RecursoQueFalla r2 = new RecursoQueFalla("Recurso2")) {
              throw new Exception("Error en el bloque try");
          }
      }
  }

  class RecursoQueFalla implements AutoCloseable {
      private String nombre;

      public RecursoQueFalla(String nombre) {
          this.nombre = nombre;
          System.out.println(nombre + " abierto");
      }

      @Override
      public void close() throws Exception {
          System.out.println(nombre + " cerrando...");
          throw new Exception("Error al cerrar " + nombre);
      }
  }
  ```

  Salida:
  ```
  Recurso1 abierto
  Recurso2 abierto
  Recurso2 cerrando...
  Recurso1 cerrando...
  Excepción principal: Error en el bloque try
  Suprimidas: 2
    Suprimida: Error al cerrar Recurso2
    Suprimida: Error al cerrar Recurso1
  ```

  --- 11. BUENAS PRÁCTICAS ---

  1. Captura excepciones específicas, no genéricas (evita catch(Exception)).
  2. No uses excepciones para control de flujo normal.
  3. Libera recursos siempre (try-with-resources es la mejor opción).
  4. Documenta las excepciones que lanza un método con @throws.
  5. Preserva la causa original al relanzar excepciones.
  6. No ignores excepciones (catch vacío es casi siempre incorrecto).
  7. Lanza excepciones con mensajes descriptivos.
  8. Prefiere excepciones unchecked para errores de programación.
  9. Usa multi-catch para manejar múltiples excepciones de la misma forma.
  10. No exageres: no rodees cada línea con try-catch. Agrupa operaciones.

  ```java
  // Malo: captura genérica y no hace nada
  try {
      // código
  } catch (Exception e) {
      // Vacío: ignora el error
  }

  // Malo: excepción para control de flujo
  try {
      Integer.parseInt(valor);
      // es número
  } catch (NumberFormatException e) {
      // no es número
  }
  // Bueno: usar métodos de validación
  if (valor != null && valor.matches("\\d+")) {
      // es número
  }

  // Malo: excepción sin contexto
  throw new Exception("Error");

  // Bueno: excepción con contexto
  throw new IllegalArgumentException(
          "El email '" + email + "' no tiene formato válido");
  ```

  --- 12. EJEMPLO INTEGRADOR ---

  ```java
  import java.io.*;
  import java.util.*;

  public class SistemaGestionArchivos {
      private static final String ARCHIVO_LOG = "aplicacion.log";

      public static void main(String[] args) {
          Scanner scanner = new Scanner(System.in);
          int opcion;

          do {
              System.out.println("\n=== SISTEMA DE GESTIÓN DE ARCHIVOS ===");
              System.out.println("1. Leer archivo");
              System.out.println("2. Escribir en archivo");
              System.out.println("3. Copiar archivo");
              System.out.println("4. Mostrar estadísticas");
              System.out.println("5. Salir");
              System.out.print("Seleccione: ");

              try {
                  opcion = Integer.parseInt(scanner.nextLine());
                  procesarOpcion(opcion, scanner);
              } catch (NumberFormatException e) {
                  System.out.println("Error: Debe ingresar un número.");
                  opcion = 0;
              } catch (OperacionArchivoException e) {
                  System.out.println("Error de operación: " + e.getMessage());
                  registrarError(e);
                  opcion = 0;
              }
          } while (opcion != 5);

          scanner.close();
          System.out.println("Programa finalizado.");
      }

      private static void procesarOpcion(int opcion, Scanner scanner)
              throws OperacionArchivoException {
          switch (opcion) {
              case 1 -> leerArchivo(scanner);
              case 2 -> escribirArchivo(scanner);
              case 3 -> copiarArchivo(scanner);
              case 4 -> mostrarEstadisticas();
              case 5 -> System.out.println("Saliendo...");
              default -> System.out.println("Opción inválida.");
          }
      }

      private static void leerArchivo(Scanner scanner)
              throws OperacionArchivoException {
          System.out.print("Nombre del archivo: ");
          String nombre = scanner.nextLine();

          // Try-with-resources
          try (BufferedReader br = new BufferedReader(
                  new FileReader(nombre))) {
              System.out.println("\n=== CONTENIDO ===");
              String linea;
              int numLinea = 1;
              while ((linea = br.readLine()) != null) {
                  System.out.printf("%3d: %s%n", numLinea++, linea);
              }
          } catch (FileNotFoundException e) {
              throw new OperacionArchivoException(
                      "El archivo '" + nombre + "' no existe.", e);
          } catch (IOException e) {
              throw new OperacionArchivoException(
                      "Error al leer '" + nombre + "': " + e.getMessage(), e);
          }
      }

      private static void escribirArchivo(Scanner scanner)
              throws OperacionArchivoException {
          System.out.print("Nombre del archivo: ");
          String nombre = scanner.nextLine();

          if (nombre == null || nombre.trim().isEmpty()) {
              throw new IllegalArgumentException(
                      "El nombre del archivo no puede estar vacío");
          }

          System.out.print("Contenido (línea vacía para terminar):\n");

          try (BufferedWriter bw = new BufferedWriter(
                  new FileWriter(nombre, true))) {  // true: append
              String linea;
              while (!(linea = scanner.nextLine()).isEmpty()) {
                  bw.write(linea);
                  bw.newLine();
              }
              System.out.println("Contenido escrito en '" + nombre + "'.");
          } catch (IOException e) {
              throw new OperacionArchivoException(
                      "Error al escribir en '" + nombre + "'", e);
          }
      }

      private static void copiarArchivo(Scanner scanner)
              throws OperacionArchivoException {
          System.out.print("Archivo origen: ");
          String origen = scanner.nextLine();
          System.out.print("Archivo destino: ");
          String destino = scanner.nextLine();

          try (BufferedReader br = new BufferedReader(
                  new FileReader(origen));
               BufferedWriter bw = new BufferedWriter(
                  new FileWriter(destino))) {

              char[] buffer = new char[8192];
              int leidos;
              while ((leidos = br.read(buffer)) != -1) {
                  bw.write(buffer, 0, leidos);
              }
              System.out.println("Archivo copiado de '" + origen
                      + "' a '" + destino + "'.");
          } catch (FileNotFoundException e) {
              throw new OperacionArchivoException(
                      "Archivo origen no encontrado: " + origen, e);
          } catch (IOException e) {
              throw new OperacionArchivoException(
                      "Error al copiar archivos", e);
          }
      }

      private static void mostrarEstadisticas()
              throws OperacionArchivoException {
          File archivo = new File(ARCHIVO_LOG);

          if (!archivo.exists()) {
              System.out.println("No hay registro de errores.");
              return;
          }

          try (BufferedReader br = new BufferedReader(
                  new FileReader(archivo))) {
              int errores = 0;
              long tamano = archivo.length();
              String linea;
              while ((linea = br.readLine()) != null) {
                  errores++;
              }
              System.out.println("=== ESTADÍSTICAS ===");
              System.out.println("Errores registrados: " + errores);
              System.out.println("Tamaño del log: " + tamano + " bytes");

          } catch (IOException e) {
              throw new OperacionArchivoException(
                      "Error al leer estadísticas", e);
          }
      }

      private static void registrarError(Exception e) {
          try (BufferedWriter bw = new BufferedWriter(
                  new FileWriter(ARCHIVO_LOG, true))) {
              bw.write(new Date() + " - " + e.getClass().getSimpleName()
                      + ": " + e.getMessage());
              bw.newLine();
          } catch (IOException ex) {
              System.err.println("No se pudo registrar el error en el log.");
          }
      }
  }

  class OperacionArchivoException extends Exception {
      public OperacionArchivoException(String mensaje, Throwable causa) {
          super(mensaje, causa);
      }

      public OperacionArchivoException(String mensaje) {
          super(mensaje);
      }
  }
  ```

  +========================================+====================================+==============================================+
  | Enfoque                                | Ventajas                          | Desventajas                                 |
  +========================================+====================================+==============================================+
  | try-catch-finally                      | Compatible con Java 5/6           | Verboso, olvidos de close()                 |
  +----------------------------------------+------------------------------------+----------------------------------------------+
  | try-with-resources (Java 7+)           | Cierre automático, suprimidas     | Solo con AutoCloseable                      |
  +----------------------------------------+------------------------------------+----------------------------------------------+
  | try-with-resources (Java 9+)           | Variables externas                | Deben ser effectively final                  |
  +----------------------------------------+------------------------------------+----------------------------------------------+

  PREGUNTAS FRECUENTES (FAQ)

  1. ¿Cuál es la diferencia entre throw y throws?
     throw lanza una excepción explícitamente. throws declara qué
     excepciones puede lanzar un método.

  2. ¿Se puede tener try sin catch?
     Sí, si tienes finally. Pero al menos debe haber catch o finally.

  3. ¿Qué es try-with-resources?
     Un try que declara uno o más recursos (AutoCloseable) que se cierran
     automáticamente al final del bloque.

  4. ¿finally se ejecuta siempre?
     Casi siempre. No se ejecuta si ocurre System.exit(), JVM crash o
     el hilo es detenido.

  5. ¿Puedo tener múltiples catch para una misma excepción?
     No. Pero puedes tener múltiples catch para diferentes excepciones.

  6. ¿Las excepciones checked deben manejarse obligatoriamente?
     Sí. El compilador obliga a capturarlas (try-catch) o declararlas
     (throws).

  7. ¿Puedo crear mis propias excepciones?
     Sí, extendiendo Exception (checked) o RuntimeException (unchecked).

  8. ¿Qué son las excepciones suprimidas?
     Excepciones que ocurren durante el cierre de recursos en
     try-with-resources, que se agregan como suprimidas a la excepción
     principal.

  9. ¿Puedo lanzar una excepción desde finally?
     Sí, pero puede ocultar la excepción original del try.

  10. ¿Es mejor lanzar checked o unchecked?
        Depende. Para condiciones recuperables, checked. Para errores de
        programación, unchecked.

  • Encadenamiento de excepciones (causa original).
  • Excepciones suprimidas en try-with-resources.
  • Buenas prácticas: no capturar Exception genérica, no usar
    excepciones para control de flujo, liberar recursos siempre.
