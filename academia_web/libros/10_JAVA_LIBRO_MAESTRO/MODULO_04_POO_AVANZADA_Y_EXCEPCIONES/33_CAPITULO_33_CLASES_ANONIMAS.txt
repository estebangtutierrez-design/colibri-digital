  BIENVENIDO

  En el capítulo anterior exploraste las clases anidadas, incluyendo las
  estáticas, internas y locales. Ahora llega el turno de las clases
  anónimas: la forma más concisa de definir e instanciar una clase en una
  sola expresión.

  Una clase anónima no tiene nombre y se define en el mismo lugar donde
  se instancia. Es perfecta para situaciones donde necesitas una
  implementación única y desechable de una interfaz o clase abstracta,
  típicamente en escenarios de callbacks, event handlers y listeners.

  Este capítulo te enseñará la sintaxis, los casos de uso principales y
  las limitaciones de las clases anónimas, preparando el terreno para el
  siguiente capítulo sobre expresiones lambda, que en muchos casos
  reemplazan a las clases anónimas con una sintaxis aún más compacta.

--------------------------------------------------------------------------------

  OBJETIVOS DEL CAPÍTULO

  1. Comprender la sintaxis de las clases anónimas en Java.
  2. Implementar clases anónimas a partir de interfaces y clases

  4. Aplicar el patrón callback con clases anónimas.
  5. Diferenciar entre clases anónimas, locales y lambdas.
  6. Identificar las limitaciones de las clases anónimas.

  CONTENIDO

  --- 1. SINTÁXIS BÁSICA ---

  Una clase anónima combina la definición de clase y la creación de
  instancia en una sola expresión:

  ```java
  new Interfaz() {
      // Implementación de métodos
  };

  new ClaseAbstracta() {
      // Implementación de métodos abstractos
      // y/o sobrescritura de métodos concretos
  };
  ```

  La sintaxis completa es:

  ```java
  new NombreClaseOInterfaz(argumentosConstructor) {
      // Cuerpo de la clase anónima
  };
  ```

  Cuando se basa en una interfaz, no hay argumentos de constructor (las
  interfaces no tienen constructores). Cuando se basa en una clase, se
  pasan los argumentos del constructor de la clase padre.

  ```java
  public class EjemploBasico {
      public static void main(String[] args) {
          // Clase anónima implementando una interfaz
          Runnable tarea = new Runnable() {
              @Override
              public void run() {
                  System.out.println("¡Hola desde una clase anónima!");
              }
          };

          tarea.run();

          // Clase anónima extendiendo una clase abstracta
          Thread hilo = new Thread() {
              @Override
              public void run() {
                  System.out.println("Ejecutando en un hilo anónimo");
              }
          };

          hilo.start();
      }
  }
  ```

  --- 2. CLASES ANÓNIMAS A PARTIR DE UNA INTERFAZ ---

  Este es el uso más común. Cuando necesitas una implementación única de
  una interfaz, puedes crearla directamente sin crear una clase separada.

  ```java
  public interface Saludador {
      void saludar(String nombre);
      void despedirse(String nombre);
  }

  public class PruebaSaludador {
      public static void main(String[] args) {
          // Clase anónima implementando la interfaz Saludador
          Saludador s = new Saludador() {
              @Override
              public void saludar(String nombre) {
                  System.out.println("¡Hola, " + nombre + "! Bienvenido.");
              }

              @Override
              public void despedirse(String nombre) {
                  System.out.println("¡Adiós, " + nombre + "! Hasta pronto.");
              }
          };

          s.saludar("Ana");
          s.despedirse("Ana");
      }
  }
  ```

  La clase anónima implementa TODOS los métodos abstractos de la interfaz.
  Si la interfaz tiene múltiples métodos, todos deben ser implementados.

  ```java
  // Ejemplo con Comparator

  import java.util.Arrays;
  import java.util.Comparator;

  public class OrdenamientoAnonimo {
      public static void main(String[] args) {
          String[] palabras = {"Java", "Python", "C", "JavaScript", "Go"};

          // Ordenar por longitud usando clase anónima
          Arrays.sort(palabras, new Comparator<String>() {
              @Override
              public int compare(String a, String b) {
                  return Integer.compare(a.length(), b.length());
              }
          });

          System.out.println("Ordenado por longitud: "
                  + Arrays.toString(palabras));

          // Orden inverso con otra clase anónima
          Arrays.sort(palabras, new Comparator<String>() {
              @Override
              public int compare(String a, String b) {
                  return b.compareTo(a);
              }
          });

          System.out.println("Orden alfabético inverso: "
                  + Arrays.toString(palabras));
      }
  }
  ```

  --- 3. CLASES ANÓNIMAS A PARTIR DE UNA CLASE ---

  También puedes extender una clase concreta o abstracta mediante una clase
  anónima. Puedes sobrescribir métodos y agregar nuevos miembros (aunque
  estos últimos solo serán accesibles desde la propia expresión).

  ```java
  public class Animal {
      private String nombre;

      public Animal(String nombre) {
          this.nombre = nombre;
      }

      public void sonido() {
          System.out.println(nombre + " hace un sonido genérico.");
      }
  }

  public class PruebaAnimalAnonimo {
      public static void main(String[] args) {
          // Clase anónima extendiendo Animal
          Animal perro = new Animal("Rex") {
              @Override
              public void sonido() {
                  System.out.println("¡Guau guau! Soy " + super.toString());
              }

              // Método adicional (solo accesible desde la anónima)
              public void moverCola() {
                  System.out.println("Moviendo la cola...");
              }
          };

          perro.sonido();
          // perro.moverCola();  // Error: no visible desde tipo Animal
      }
  }
  ```

  Cuando extiendes una clase, la clase anónima no puede tener constructor
  explícito (usa el de la clase padre). Se pueden inicializar miembros
  con un bloque inicializador de instancia.

  ```java
  public class Ventana {
      private String titulo;

      public Ventana(String titulo) {
          this.titulo = titulo;
      }

      public void abrir() {
          System.out.println("Abriendo ventana: " + titulo);
      }
  }

  public class PruebaVentana {
      public static void main(String[] args) {
          // Clase anónima con bloque inicializador
          Ventana ventana = new Ventana("Mi Ventana") {
              private int ancho;
              private int alto;

              // Bloque inicializador de instancia
              {
                  ancho = 800;
                  alto = 600;
              }

              @Override
              public void abrir() {
                  super.abrir();
                  System.out.println("Dimensiones: " + ancho + "x" + alto);
              }
          };

          ventana.abrir();
      }
  }
  ```

  --- 4. CAPTURA DE VARIABLES LOCALES ---

  Al igual que las clases locales, las clases anónimas pueden acceder a
  variables del método que las contiene, siempre que sean final o
  effectively final (Java 8+).

  ```java
  public interface Operacion {
      int ejecutar(int x);
  }

  public class Calculadora {
      public static void main(String[] args) {
          int factor = 10;  // effectively final

          // Clase anónima que captura 'factor'
          Operacion multiplicar = new Operacion() {
              @Override
              public int ejecutar(int x) {
                  return x * factor;  // Captura la variable local
              }
          };

          System.out.println("10 * 5 = " + multiplicar.ejecutar(5));

          // Esto causaría error si se descomenta:
          // factor = 20;  // Ya no sería effectively final
      }
  }
  ```

  En versiones anteriores a Java 8, las variables debían ser declaradas
  explícitamente como final:

  ```java
  public class AntesDeJava8 {
      public void metodo() {
          final int constante = 42;  // Obligatorio declarar final

          Operacion op = new Operacion() {
              @Override
              public int ejecutar(int x) {
                  return x + constante;
              }
          };
      }
  }
  ```

  --- 5. MANEJO DE EVENTOS Y LISTENERS ---

  Uno de los usos más extendidos de las clases anónimas es en el manejo
  de eventos, particularmente en interfaces gráficas (GUI) y
  programación reactiva.

  ```java
  import javax.swing.*;
  import java.awt.event.*;

  public class VentanaEventos {
      public static void main(String[] args) {
          JFrame frame = new JFrame("Ejemplo de Eventos");
          JButton boton = new JButton("Haz clic");

          // Clase anónima como listener de acción
          boton.addActionListener(new ActionListener() {
              @Override
              public void actionPerformed(ActionEvent e) {
                  System.out.println("¡Botón presionado!");
                  JOptionPane.showMessageDialog(null,
                          "Has hecho clic en el botón");
              }
          });

          // Clase anónima como listener de foco
          boton.addFocusListener(new FocusListener() {
              @Override
              public void focusGained(FocusEvent e) {
                  System.out.println("Botón ganó el foco");
              }

              @Override
              public void focusLost(FocusEvent e) {
                  System.out.println("Botón perdió el foco");
              }
          });

          // Clase anónima como listener de teclado
          boton.addKeyListener(new KeyListener() {
              @Override
              public void keyTyped(KeyEvent e) {
                  System.out.println("Tecla tipeada: " + e.getKeyChar());
              }

              @Override
              public void keyPressed(KeyEvent e) {
                  System.out.println("Tecla presionada: " + e.getKeyCode());
              }

              @Override
              public void keyReleased(KeyEvent e) {
                  System.out.println("Tecla liberada: " + e.getKeyCode());
              }
          });

          frame.add(boton);
          frame.setSize(300, 200);
          frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
          frame.setVisible(true);
      }
  }
  ```

  Aunque Swing ya no es la tecnología moderna de GUI, este patrón de
  diseño con clases anónimas para eventos se replica en muchos frameworks.

  ```java
  // Simulación de eventos sin Swing

  import java.util.ArrayList;
  import java.util.List;

  // Interfaz para listener
  interface EventListener {
      void onEvent(String tipo, String data);
  }

  class FuenteEventos {
      private List<EventListener> listeners = new ArrayList<>();

      public void addListener(EventListener listener) {
          listeners.add(listener);
      }

      public void dispararEvento(String tipo, String data) {
          for (EventListener l : listeners) {
              l.onEvent(tipo, data);
          }
      }

      public void simularEventos() {
          System.out.println("=== Simulando eventos... ===");
          dispararEvento("CLICK", "Usuario hizo clic en botón OK");
          dispararEvento("HOVER", "Mouse sobre el botón Cancelar");
          dispararEvento("FOCUS", "Campo email obtuvo el foco");
      }
  }

  public class SistemaEventos {
      public static void main(String[] args) {
          FuenteEventos fuente = new FuenteEventos();

          // Múltiples listeners con clases anónimas
          fuente.addListener(new EventListener() {
              @Override
              public void onEvent(String tipo, String data) {
                  System.out.println("[LOGGER] Evento: " + tipo
                          + " - " + data);
              }
          });

          fuente.addListener(new EventListener() {
              @Override
              public void onEvent(String tipo, String data) {
                  if (tipo.equals("CLICK")) {
                      System.out.println("[PROCESADOR] Procesando clic: "
                              + data);
                  }
              }
          });

          fuente.simularEventos();
      }
  }
  ```

  Salida:
  ```
  === Simulando eventos... ===
  [LOGGER] Evento: CLICK - Usuario hizo clic en botón OK
  [PROCESADOR] Procesando clic: Usuario hizo clic en botón OK
  [LOGGER] Evento: HOVER - Mouse sobre el botón Cancelar
  [LOGGER] Evento: FOCUS - Campo email obtuvo el foco
  ```

  --- 6. PATRÓN CALLBACK CON CLASES ANÓNIMAS ---

  El patrón callback permite pasar un comportamiento como argumento a un
  método, que lo invocará en el momento adecuado.

  ```java
  interface Callback {
      void onCompletado(String resultado);
      void onError(String mensaje);
  }

  class TareaLarga {
      public void ejecutar(String parametro, Callback callback) {
          System.out.println("Iniciando tarea con: " + parametro);

          try {
              // Simular trabajo
              String resultado = parametro.toUpperCase();
              Thread.sleep(500);  // Simula latencia

              if (parametro.isEmpty()) {
                  callback.onError("El parámetro no puede estar vacío");
              } else {
                  callback.onCompletado(resultado);
              }
          } catch (InterruptedException e) {
              callback.onError("Tarea interrumpida: " + e.getMessage());
          }
      }
  }

  public class PruebaCallback {
      public static void main(String[] args) {
          TareaLarga tarea = new TareaLarga();

          // Callback con clase anónima
          tarea.ejecutar("Hola Mundo", new Callback() {
              @Override
              public void onCompletado(String resultado) {
                  System.out.println("¡Tarea completada!");
                  System.out.println("Resultado: " + resultado);
              }

              @Override
              public void onError(String mensaje) {
                  System.err.println("Error: " + mensaje);
              }
          });

          System.out.println("Programa principal continúa...");

          // Otro callback para caso de error
          tarea.ejecutar("", new Callback() {
              @Override
              public void onCompletado(String resultado) {
                  System.out.println("Completado: " + resultado);
              }

              @Override
              public void onError(String mensaje) {
                  System.err.println("¡ERROR! " + mensaje);
              }
          });
      }
  }
  ```

  --- 7. CLASES ANÓNIMAS COMO ARGUMENTOS ---

  Un patrón muy común es pasar una clase anónima directamente como
  argumento de método, sin asignarla a una variable:

  ```java
  import java.util.*;

  public class ArgumentosAnonimos {
      public static void main(String[] args) {
          List<String> lista = new ArrayList<>(
                  Arrays.asList("Banana", "Manzana", "Cereza", "Dátil"));

          // Clase anónima pasada como argumento
          Collections.sort(lista, new Comparator<String>() {
              @Override
              public int compare(String a, String b) {
                  return Integer.compare(a.length(), b.length());
              }
          });
          System.out.println("Por longitud: " + lista);

          // Otra clase anónima inline
          lista.sort(new Comparator<String>() {
              @Override
              public int compare(String a, String b) {
                  return b.charAt(0) - a.charAt(0);
              }
          });
          System.out.println("Por primera letra (inverso): " + lista);

          // Filtro con clase anónima
          List<String> filtradas = filtrar(lista, new Filtro() {
              @Override
              public boolean aceptar(String s) {
                  return s.startsWith("C") || s.startsWith("D");
              }
          });
          System.out.println("Filtradas: " + filtradas);
      }

      interface Filtro {
          boolean aceptar(String s);
      }

      static List<String> filtrar(List<String> lista, Filtro f) {
          List<String> resultado = new ArrayList<>();
          for (String s : lista) {
              if (f.aceptar(s)) {
                  resultado.add(s);
              }
          }
          return resultado;
      }
  }
  ```

  Este patrón de "estrategia" con clases anónimas es la base de las
  expresiones lambda, que veremos en el capítulo 34.

  --- 8. CLASES ANÓNIMAS Y MÉTODOS DEFAULT ---

  Las clases anónimas pueden sobrescribir métodos default y estáticos
  (aunque los estáticos no se heredan, los default sí).

  ```java
  interface SaludadorMejorado {
      void saludar(String nombre);

      default void despedirse(String nombre) {
          System.out.println("Adiós por defecto, " + nombre);
      }

      default void agradecer(String nombre) {
          System.out.println("Gracias por defecto, " + nombre);
      }
  }

  public class PruebaDefaults {
      public static void main(String[] args) {
          // Sobrescribiendo solo algunos métodos default
          SaludadorMejorado s = new SaludadorMejorado() {
              @Override
              public void saludar(String nombre) {
                  System.out.println("¡Qué tal, " + nombre + "!");
              }

              @Override
              public void despedirse(String nombre) {
                  System.out.println("¡Nos vemos, " + nombre + "!");
              }
              // agradecer() usa la implementación default
          };

          s.saludar("Luis");
          s.despedirse("Luis");
          s.agradecer("Luis");
      }
  }
  ```

  --- 9. LIMITACIONES DE LAS CLASES ANÓNIMAS ---

  1. No pueden tener constructores explícitos: usan el constructor de la
     clase padre. Para inicializar, se usa un bloque de instancia {}.

  2. No pueden ser genéricas: no se pueden declarar parámetros de tipo
     para la clase anónima (aunque pueden usar los tipos de contexto).

  3. No pueden ser abstractas: deben implementar todos los métodos
     abstractos de la interfaz/clase padre.

  4. No pueden contener miembros estáticos (excepto constantes final).

  5. No pueden ser usadas como tipo de retorno o variable: su tipo es el
     de la superclase/interfaz que extienden/implementan.

  6. Solo pueden extender UNA clase o implementar UNA interfaz.

  7. Si se usan extensivamente, el código puede volverse difícil de leer.

  ```java
  public class Limitaciones {
      // Error: no se pueden declarar métodos estáticos
      // Runnable r = new Runnable() {
      //     static void metodo() { }  // Error
      // };

      // Error: no pueden ser genéricas
      // Comparator<T> c = new Comparator<T>() { };  // Error

      // Solución para inicialización: bloque de instancia
      Runnable r = new Runnable() {
          private int id;
          {
              id = 42;  // Bloque inicializador
          }

          @Override
          public void run() {
              System.out.println("ID: " + id);
          }
      };
  }
  ```

  --- 10. CLASES ANÓNIMAS VS LAMBDAS (AVANCE) ---

  Aunque las expresiones lambda (Capítulo 34) a menudo reemplazan a las
  clases anónimas, hay diferencias importantes:

  - Las clases anónimas pueden implementar interfaces con múltiples
    métodos abstractos; las lambdas solo pueden implementar interfaces
    funcionales (un solo método abstracto).

  - Las clases anónimas crean un nuevo ámbito (this se refiere a la
    clase anónima); en lambdas, this se refiere al ámbito circundante.

  - Las clases anónimas pueden tener variables de instancia y bloques
    inicializadores; las lambdas no.

  ```java
  public class Comparacion {
      private String mensaje = "Clase contenedora";

      public void probar() {
          // Clase anónima: 'this' se refiere a la clase anónima
          Runnable anonima = new Runnable() {
              private String mensaje = "Clase anónima";

              @Override
              public void run() {
                  System.out.println(this.mensaje);  // Clase anónima
                  System.out.println(Comparacion.this.mensaje); // Contenedora
              }
          };

          // Lambda: 'this' se refiere al ámbito circundante
          Runnable lambda = () -> {
              // this.mensaje se refiere a Comparacion.this.mensaje
              System.out.println(this.mensaje);  // Clase contenedora
          };

          anonima.run();
          lambda.run();
      }
  }

  public class PruebaComparacion {
      public static void main(String[] args) {
          new Comparacion().probar();
      }
  }
  ```

  Salida:
  ```
  Clase anónima
  Clase contenedora
  Clase contenedora
  ```

  --- 11. EJEMPLO INTEGRADOR: SISTEMA DE NOTIFICACIONES ---

  ```java
  import java.util.ArrayList;
  import java.util.List;

  // Interfaz base para notificaciones
  interface Notificador {
      void enviar(String mensaje);

      default void enviarMultiple(List<String> mensajes) {
          for (String msg : mensajes) {
              enviar(msg);
          }
      }

      default void log(String mensaje) {
          System.out.println("[LOG] " + mensaje);
      }
  }

  class SistemaNotificaciones {
      private List<Notificador> canales = new ArrayList<>();

      public void agregarCanal(Notificador canal) {
          canales.add(canal);
          System.out.println("Canal de notificación agregado.");
      }

      public void notificarATodos(String mensaje) {
          System.out.println("\n=== Notificando a todos los canales ===");
          System.out.println("Mensaje: " + mensaje + "\n");

          for (Notificador canal : canales) {
              canal.enviar(mensaje);
          }
      }

      public void notificarPorLotes(List<String> mensajes) {
          System.out.println("\n=== Notificación por lotes ===");
          for (Notificador canal : canales) {
              canal.enviarMultiple(mensajes);
          }
      }
  }

  public class AppNotificaciones {
      public static void main(String[] args) {
          SistemaNotificaciones sistema = new SistemaNotificaciones();

          // Canal Email con clase anónima
          sistema.agregarCanal(new Notificador() {
              private String email = "admin@example.com";

              @Override
              public void enviar(String mensaje) {
                  System.out.println("[EMAIL a " + email + "] " + mensaje);
                  log("Email enviado a " + email);
              }

              @Override
              public void log(String mensaje) {
                  System.out.println("[LOG-EMAIL] " + mensaje);
              }
          });

          // Canal SMS con clase anónima
          sistema.agregarCanal(new Notificador() {
              private String telefono = "+51-999888777";
              private int contador = 0;

              @Override
              public void enviar(String mensaje) {
                  contador++;
                  System.out.println("[SMS a " + telefono
                          + " (#" + contador + ")] " + mensaje);
              }
          });

          // Canal Slack con clase anónima
          sistema.agregarCanal(new Notificador() {
              private String canal = "#general";

              @Override
              public void enviar(String mensaje) {
                  System.out.println("[SLACK en " + canal + "] " + mensaje);
              }

              @Override
              public void enviarMultiple(List<String> mensajes) {
                  System.out.println("[SLACK] Enviando " + mensajes.size()
                          + " mensajes a " + canal + "...");
                  for (String msg : mensajes) {
                      enviar(msg);
                  }
              }
          });

          // Pruebas
          sistema.notificarATodos("¡Servidor reiniciado con éxito!");

          List<String> alertas = new ArrayList<>();
          alertas.add("ALERTA: CPU al 95%");
          alertas.add("ALERTA: Memoria al 85%");
          alertas.add("INFO: Backup completado");

          sistema.notificarPorLotes(alertas);

          System.out.println("\n=== Sistema de notificaciones activo ===");
      }
  }
  ```

  Salida (simplificada):
  ```
  Canal de notificación agregado.
  Canal de notificación agregado.
  Canal de notificación agregado.

  === Notificando a todos los canales ===
  Mensaje: ¡Servidor reiniciado con éxito!

  [EMAIL a admin@example.com] ¡Servidor reiniciado con éxito!
  [LOG-EMAIL] Email enviado a admin@example.com
  [SMS a +51-999888777 (#1)] ¡Servidor reiniciado con éxito!
  [SLACK en #general] ¡Servidor reiniciado con éxito!
  ...
  ```

  PREGUNTAS FRECUENTES (FAQ)

  1. ¿Las clases anónimas tienen nombre?
     No. El compilador les asigna nombres sintéticos como Clase$1.class.

  2. ¿Cuándo debo usar una clase anónima vs una clase separada?
     Usa clase anónima cuando la implementación sea única, pequeña y
     específica del contexto. Si se repite o es compleja, crea una clase.

  3. ¿Puedo tener múltiples instancias de una clase anónima?
     Sí. Cada vez que escribes "new Interfaz() { ... }" creas una nueva
     instancia de una nueva clase anónima.

  4. ¿Las clases anónimas pueden tener herencia?
     No. Una clase anónima solo puede extender una clase o implementar
     una interfaz, no ambas.

  5. ¿Puedo usar genéricos con clases anónimas?
     La clase anónima no declara parámetros de tipo, pero puede usar los
     del ámbito donde se define.

  6. ¿Cómo inicializo una clase anónima?
     Con un bloque de instancia {} dentro del cuerpo de la clase anónima.

  7. ¿Sobrescriben las clases anónimas los métodos default?
     Sí, pueden sobrescribir cualquier método heredado.

  8. ¿Las expresiones lambda siempre reemplazan a las clases anónimas?
     Solo cuando la interfaz es funcional (un solo método abstracto).
     Las clases anónimas siguen siendo necesarias para interfaces con
     múltiples métodos.

  RESUMEN DEL CAPÍTULO

  En este capítulo has aprendido:

  • Las clases anónimas son clases sin nombre definidas e instanciadas
    en una sola expresión.

  • Se usan para implementar interfaces o extender clases de manera
    concisa y específica al contexto.

  • Son ideales para callbacks y event handlers.

  • Capturan variables effectively final del ámbito circundante.

  • No pueden tener constructores explícitos, pero pueden usar bloques
    inicializadores de instancia.

  • Su this se refiere a la propia clase anónima (diferente de las
    lambdas).

  • No son adecuadas para lógicas que se repiten o son muy extensas.

  • Las expresiones lambda (Capítulo 34) pueden reemplazarlas cuando se
    trata de interfaces funcionales.

  DESAFÍO DEL ARQUITECTO

  Diseña un sistema de procesamiento de encuestas en tiempo real que use
  clases anónimas intensivamente.

  Requisitos:
  - Interface Pregunta con métodos: mostrar(), responder(String respuesta),
    y default obtenerEstadisticas().
  - Interface Encuesta con métodos: iniciar(), agregarPregunta(Pregunta),
    mostrarResultados().
  - Interface Validador con: boolean validar(String respuesta).
  - Crea una clase Encuestadora con un método realizarEncuesta(Encuesta).
  - Usa exclusivamente clases anónimas para:
    * Crear preguntas de tipo texto con validación (mínimo 3 caracteres)
    * Crear preguntas de tipo sí/no con validación
    * Crear preguntas de calificación (1-5) con validación
    * Crear la encuesta que contenga estas preguntas
    * Implementar listeners que notifiquen cuando una pregunta sea
      respondida

  Extensión opcional: Agrega persistencia de respuestas en un arreglo
  y genera un reporte final con estadísticas de cada pregunta.

  CIERRE DEL CAPÍTULO

  Las clases anónimas son una herramienta expresiva que Java puso a tu
  disposición para escribir código más conciso y situado. Aunque las
  expresiones lambda las han eclipsado para interfaces funcionales, las
  clases anónimas siguen siendo esenciales para interfaces con múltiples
  métodos y para situaciones donde necesitas un ámbito propio (this).

  natural de las clases anónimas para interfaces funcionales, que llevará
  tu código a un nivel de expresividad aún mayor.
