  76.1. ¿QUÉ ES SPRING BOOT?
-------------------------------------------------------------------------------

Spring Boot es un proyecto de Spring que simplifica la creación de
aplicaciones independientes y de producción. Fue creado por el equipo
de Spring (liderado por Phil Webb) en 2014.

  PROBLEMAS QUE SPRING BOOT RESUELVE:

  Antes de Spring Boot, crear una aplicación Spring requería:
  1. Configurar Maven con múltiples dependencias (y versiones compatibles)
  2. Escribir XML de configuración de Spring (cientos de líneas)
  3. Configurar el servidor web (Tomcat, Jetty) externamente
  4. Configurar el archivo web.xml
  5. Crear y configurar el DataSource
  6. Configurar JPA, Transaction Manager, etc.
  7. Configurar el view resolver (JSP, Thymeleaf)
  8. Y mucho más...

  Con Spring Boot:
  1. Agregas un starter (spring-boot-starter-web)
  2. Escribes tu código
  3. Ejecutas la aplicación

  CARACTERÍSTICAS PRINCIPALES:

  • Starters: Dependencias preconfiguradas que agrupan librerías
    relacionadas.

  • Autoconfiguración: Spring Boot configura automáticamente los
    componentes basándose en las dependencias del classpath.

  • Servidor embebido: Incluye Tomcat, Jetty o Undertow. No necesitas
    instalar un servidor externo.

  • application.properties/yml: Configuración externalizada y
    jerárquica.

  • Actuator: Endpoints de monitoreo y gestión para producción.

  • DevTools: Herramientas de desarrollo (recarga automática, etc.).

  • CLI: Interfaz de línea de comandos para prototipado rápido.

  • Spring Initializr: Generador de proyectos web.

  76.2.1. SPRING INITIALIZR

  La forma más rápida de crear un proyecto Spring Boot es usando
  Spring Initializr: https://start.spring.io/

  Configuración típica:
  - Project: Maven (o Gradle)
  - Language: Java
  - Spring Boot: 3.2.x (última estable)
  - Group: com.lucasega
  - Artifact: mi-aplicacion
  - Dependencies: Spring Web, Spring Data JPA, H2 Database, Lombok

-------------------------------------------------------------------------------
  76.3. CLASE PRINCIPAL Y ARRANQUE
-------------------------------------------------------------------------------

La clase principal es el punto de entrada de la aplicación Spring Boot.

```java
package com.lucasega;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class MiAplicacionApplication {

    public static void main(String[] args) {
        SpringApplication.run(MiAplicacionApplication.class, args);
    }
}
```

  ¿QUÉ HACE @SpringBootApplication?

  Es una anotación compuesta que incluye:

  1. @Configuration: Marca la clase como fuente de definición de beans
  2. @EnableAutoConfiguration: Activa la autoconfiguración de Spring Boot
  3. @ComponentScan: Escanea el paquete y subpaquetes en busca de
     componentes (@Component, @Service, @Repository, @Controller)

  EQUIVALENTE A:

```java
@Configuration
@EnableAutoConfiguration
@ComponentScan(basePackages = "com.lucasega")
public class MiAplicacionApplication {
    // ...
}
```

  ARRANQUE PERSONALIZADO:

```java
@SpringBootApplication
public class AppPersonalizada {

    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(AppPersonalizada.class);

        // Deshabilitar banner
        app.setBannerMode(Banner.Mode.OFF);

        // Configuración adicional
        app.setDefaultProperties(Collections.singletonMap(
            "server.port", "9090"
        ));

        app.run(args);
    }
}
```

  │ Starter                        │ Funcionalidad                 │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-web        │ Aplicaciones web (MVC +       │

  │ spring-boot-starter-webflux    │ Aplicaciones reactivas        │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-data-jpa   │ JPA + Hibernate               │
  │ spring-boot-starter-data-      │ MongoDB                       │
  │ mongodb                        │                               │
  │ spring-boot-starter-data-      │ Redis                         │
  │ redis                          │                               │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-security   │ Seguridad (autenticación,     │
  │                                │ autorización)                 │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-thymeleaf  │ Motor de plantillas Thymeleaf │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-test       │ JUnit 5, Mockito, TestContain.│
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-actuator   │ Monitoreo y gestión           │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-validation │ Bean Validation (Hibernate    │
  │                                │ Validator)                    │

  │                                │ Aspectos                      │
  ├────────────────────────────────┼────────────────────────────────┤
  │ spring-boot-starter-mail       │ Envío de correos electrónicos │

  │                                │ (Quartz Scheduler)            │

  ¿CÓMO FUNCIONAN INTERNAMENTE?

  Cada starter tiene un archivo spring.factories en META-INF que
  lista las configuraciones automáticas que deben activarse.

  Por ejemplo, spring-boot-starter-web incluye:
  - spring-boot-starter-tomcat (servidor embebido)
  - spring-webmvc (Spring MVC)
  - spring-boot-starter-json (Jackson)
  - Y todas las dependencias transitivas necesarias

-------------------------------------------------------------------------------
  76.5. AUTOCONFIGURACIÓN
-------------------------------------------------------------------------------

La autoconfiguración es el corazón de Spring Boot. Basándose en las
dependencias del classpath, Spring Boot configura automáticamente los
componentes necesarios.

  EJEMPLO: ¿Cómo sabe Spring Boot que debe configurar un DataSource?

  1. Detecta H2, HikariCP y JPA en el classpath
  2. Configura automáticamente:
     - DataSource (H2 en memoria)
     - EntityManagerFactory
     - TransactionManager
     - JPA Repository
     - H2 Console (en desarrollo)

  PERSONALIZACIÓN DE LA AUTOCONFIGURACIÓN:

```java
@SpringBootApplication
@EnableAutoConfiguration(exclude = {
    DataSourceAutoConfiguration.class,  // Excluir DataSource automático
    HibernateJpaAutoConfiguration.class // Excluir JPA automático
})
public class AppPersonalizada {
    // ...
}
```

  CONDICIONALES DE AUTOCONFIGURACIÓN:

```java
@Configuration
@ConditionalOnClass(DataSource.class) // Solo si DataSource está en classpath
@ConditionalOnMissingBean(DataSource.class) // Solo si no hay un DataSource definido
@ConditionalOnProperty(prefix = "app.db", name = "enabled", havingValue = "true")
@ConditionalOnWebApplication
@ConditionalOnNotWebApplication
public class MiAutoConfiguracion {
    // ...
}
```

  CREAR TU PROPIA AUTOCONFIGURACIÓN:

```java
package com.lucasega.autoconfig;

import org.springframework.boot.autoconfigure.condition.ConditionalOnClass;
import org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
@ConditionalOnClass(name = "com.lucasega.mi Libreria")
public class MiAutoConfiguracion {

    @Bean
    @ConditionalOnMissingBean
    public MiServicio miServicio() {
        return new MiServicio();
    }
}
```

```properties
# META-INF/spring/org.springframework.boot.autoconfigure.
# AutoConfiguration.imports
com.lucasega.autoconfig.MiAutoConfiguracion
```

```java
import java.util.Map;

    private List<String> dominiosPermitidos;

    public static class Seguridad {
        private boolean enabled = true;
        private int maxIntentos = 3;
        private long timeoutSesion = 3600;

        // getters y setters
    }

    // getters y setters de AppProperties
}
```

```java
// Uso de @ConfigurationProperties en un servicio
@Service

}
```

  76.8. SERVIDOR EMBEBIDO
-------------------------------------------------------------------------------

Spring Boot incluye un servidor web embebido, eliminando la necesidad de
instalar y configurar un servidor externo.

  SERVIDORES SOPORTADOS:

  ┌──────────┬──────────────────────────────────────────────────────┐
  │ Servidor │ Características                                     │
  ├──────────┼──────────────────────────────────────────────────────┤
  │ Tomcat   │ Por defecto. Maduro, ampliamente usado.             │
  │           │ Soporta Servlet 6.0, WebSocket, HTTP/2             │
  ├──────────┼──────────────────────────────────────────────────────┤
  │ Jetty    │ Más ligero. Ideal para entornos con recursos        │
  │          │ limitados. Soporta Servlet, WebSocket, HTTP/2       │
  ├──────────┼──────────────────────────────────────────────────────┤
  │ Undertow │ Alto rendimiento. Desarrollado por Red Hat.         │
  │          │ NIO-based, muy rápido.                              │
  └──────────┴──────────────────────────────────────────────────────┘

  CAMBIO DE SERVIDOR:

```xml
<!-- Excluir Tomcat y agregar Jetty -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-tomcat</artifactId>
        </exclusion>
    </exclusions>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-jetty</artifactId>
</dependency>
```

  76.9. SPRING BOOT DEVTOOLS
-------------------------------------------------------------------------------

DevTools mejora la experiencia de desarrollo con funcionalidades como
recarga automática, reinicio rápido y configuraciones de desarrollo.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <scope>runtime</scope>
    <optional>true</optional>
</dependency>
```

  FUNCIONALIDADES:

  1. Reinicio automático (Live Reload):
     - Detecta cambios en archivos y reinicia la aplicación
     - Mucho más rápido que un reinicio completo (usa cargadores de
       clases duales)

  2. LiveReload:
     - Recarga automática del navegador cuando hay cambios
     - Necesita extensión LiveReload en el navegador

  3. Configuraciones de desarrollo:
     - Template caching desactivado (Thymeleaf, FreeMarker)
     - H2 Console habilitada por defecto
     - Logging DEBUG para capa web

  4. Despliegue remoto:
     - Soporte para reinicio remoto

```properties
# Deshabilitar DevTools específicamente
spring.devtools.restart.enabled=false

# Excluir directorios del reinicio
spring.devtools.restart.exclude=static/**,public/**,templates/**

# Habilitar LiveReload
spring.devtools.livereload.enabled=true
```

  76.10. SPRING BOOT ACTUATOR
-------------------------------------------------------------------------------

Actuator proporciona endpoints para monitorear y gestionar la aplicación
en producción.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

  ENDPOINTS PRINCIPALES:

  ┌──────────────────┬──────────────────────────────────────────────┐
  │ Endpoint         │ Descripción                                  │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ /actuator/health │ Estado de salud de la aplicación             │
  │ /actuator/info   │ Información personalizada de la app          │
  │ /actuator/metrics│ Métricas (memoria, CPU, etc.)               │
  │ /actuator/env    │ Variables de entorno y configuración         │
  │ /actuator/beans  │ Todos los beans de Spring                   │
  │ /actuator/mappings│ Mapeos de rutas HTTP                       │
  │ /actuator/loggers│ Configuración de loggers en tiempo real      │
  │ /actuator/threaddump│ Dump de hilos                            │
  │ /actuator/heapdump│ Dump de memoria heap                        │
  │ /actuator/shutdown│ Apagar la aplicación (POST)                 │
  └──────────────────┴──────────────────────────────────────────────┘

```properties
# Exponer endpoints
management.endpoints.web.exposure.include=health,info,metrics,env,beans
management.endpoints.web.exposure.include=*  # Todos (cuidado en prod)
```

```java
import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;

@Component
public class ServicioExternoHealthIndicator implements HealthIndicator {

    public Health health() {
        try {
            // Verificar conexión con servicio externo
            boolean servidorExternoOK = verificarServicio();
            if (servidorExternoOK) {
                return Health.up()
                    .withDetail("servicio", "Disponible")
                    .withDetail("tiempoRespuesta", "120ms")
                    .build();
            } else {
                return Health.down()
                    .withDetail("servicio", "No disponible")
                    .withDetail("error", "Timeout de conexión")
                    .build();
            }
        } catch (Exception e) {
            return Health.down(e)
                .withDetail("servicio", "Error crítico")
                .build();
        }
    }

    private boolean verificarServicio() {
        // Lógica de verificación
        return true;
    }
}
```

```java
// Service
package com.lucasega.service;

import com.lucasega.model.Tarea;
import com.lucasega.repository.TareaRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;
import java.util.Optional;

@Service
@Transactional
public class TareaService {

    private final TareaRepository repository;

    public TareaService(TareaRepository repository) {
        this.repository = repository;
    }

    public Tarea crear(Tarea tarea) {
        return repository.save(tarea);
    }

    @Transactional(readOnly = true)
    public List<Tarea> listarTodas() {
        return repository.findAll();
    }

    @Transactional(readOnly = true)
    public List<Tarea> listarPendientes() {
        return repository.findByCompletadaFalse();
    }

    @Transactional(readOnly = true)
    public Optional<Tarea> buscarPorId(Long id) {
        return repository.findById(id);
    }

    public Tarea completar(Long id) {
        Tarea tarea = repository.findById(id)
            .orElseThrow(() -> new RuntimeException("Tarea no encontrada: " + id));
        tarea.setCompletada(true);
        return repository.save(tarea);
    }

    public void eliminar(Long id) {
        repository.deleteById(id);
    }
}
```

```java
// Controller
package com.lucasega.controller;

import com.lucasega.model.Tarea;
import com.lucasega.service.TareaService;
import jakarta.validation.Valid;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/tareas")
public class TareaController {

    private final TareaService service;

    public TareaController(TareaService service) {
        this.service = service;
    }

    @GetMapping
    public ResponseEntity<List<Tarea>> listarTodas() {
        return ResponseEntity.ok(service.listarTodas());
    }

    @GetMapping("/{id}")
    public ResponseEntity<Tarea> obtenerPorId(@PathVariable Long id) {
        return service.buscarPorId(id)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }

    @PostMapping
    public ResponseEntity<Tarea> crear(@Valid @RequestBody Tarea tarea) {
        Tarea nueva = service.crear(tarea);
        return ResponseEntity.status(HttpStatus.CREATED).body(nueva);
    }

    @PutMapping("/{id}/completar")
    public ResponseEntity<Tarea> completar(@PathVariable Long id) {
        Tarea tarea = service.completar(id);
        return ResponseEntity.ok(tarea);
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> eliminar(@PathVariable Long id) {
        service.eliminar(id);
        return ResponseEntity.noContent().build();
    }
}
```

  RESUMEN DEL CAPÍTULO
-------------------------------------------------------------------------------

  • Spring Boot simplifica drásticamente la creación de aplicaciones
    Spring con autoconfiguración, starters y servidor embebido.

  • @SpringBootApplication combina @Configuration,
    @EnableAutoConfiguration y @ComponentScan en una sola anotación.

  • Los starters son dependencias agrupadas que incluyen todo lo
    necesario para una funcionalidad específica.

  • La autoconfiguración configura componentes basándose en las
    dependencias del classpath (convención sobre configuración).

  • application.properties y application.yml permiten externalizar
    toda la configuración de la aplicación.

  • Los perfiles (spring.profiles.active) permiten configuraciones
    específicas por entorno (desarrollo, pruebas, producción).

  • @ConfigurationProperties mapea propiedades a objetos Java
    tipados y validados.

  • El servidor embebido (Tomcat, Jetty, Undertow) elimina la
    necesidad de instalación y configuración externa.

  • DevTools mejora la productividad con reinicio automático,
    LiveReload y configuraciones de desarrollo.

  • Actuator proporciona endpoints de monitoreo y gestión para
    entornos de producción.

  • Spring Boot se integra perfectamente con JUnit 5, Mockito y
    TestContainers para testing.

================================================================================
  PREGUNTAS FRECUENTES (FAQ)
================================================================================

  ❓ ¿Spring Boot reemplaza a Spring Framework?
  R: No. Spring Boot se basa en Spring Framework. Es una capa que
     simplifica su uso, pero todo lo que sabes de Spring sigue
     aplicando.

  ❓ ¿Qué versión de Spring Boot debería usar?
  R: Siempre la última versión estable. Actualmente 3.2.x (con Java 17).
     Spring Boot 2.x aún se mantiene pero está en fin de vida.

  ❓ ¿Qué es el fat JAR de Spring Boot?
  R: Es un JAR ejecutable que incluye todas las dependencias y el
     servidor embebido. Se crea con spring-boot-maven-plugin.

  ❓ ¿Cómo despliego una aplicación Spring Boot?
  R: java -jar app.jar. También puedes generar un WAR para desplegar
     en servidores externos.

  ❓ ¿Spring Boot es adecuado para microservicios?
  R: Sí, es la plataforma más usada para microservicios Java.
     Spring Cloud extiende Spring Boot para patrones de microservicios.

  ❓ ¿Puedo usar Spring Boot con IntelliJ Community (gratuito)?
  R: Sí, pero no tendrás el asistente de Spring Initializr integrado.
     Puedes usar start.spring.io desde el navegador.

  ❓ ¿Cómo manejo errores globalmente en Spring Boot?
  R: Con @ControllerAdvice y @ExceptionHandler para manejo centralizado
     de excepciones.
