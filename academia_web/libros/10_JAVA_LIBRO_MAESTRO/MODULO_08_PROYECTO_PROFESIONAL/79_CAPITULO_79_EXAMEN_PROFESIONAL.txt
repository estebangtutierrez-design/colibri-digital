  PREGUNTA 1.3 (3 ptos) - Herencia y polimorfismo

```java
abstract class Animal {
    public String hacerSonido() { return "Sonido generico"; }
}

class Perro extends Animal {
    @Override
    public String hacerSonido() { return "Guau"; }
    public String moverCola() { return "Moviendo cola"; }
}

class Gato extends Animal {
    @Override
    public String hacerSonido() { return "Miau"; }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Perro();
        Animal a2 = new Gato();
        
        System.out.println(a1.hacerSonido()); // ?
        // System.out.println(a1.moverCola()); // ? ¿Compila?
        System.out.println(a2.hacerSonido()); // ?
        
        Perro p = (Perro) a1;
        System.out.println(p.moverCola()); // ?
        
        // Gato g = (Gato) a1; // ? ¿Que pasa?
    }
}
```

  Explica linea por linea que sucede y por que.

public class CalculadoraService {

}

  c) Null pointer
  d) Montos grandes

  79.12. HOJA DE RESPUESTAS (TEORICAS)
-------------------------------------------------------------------------------

  RESPUESTA 1.1: a) true, false, true, true
  - a == b: true (Integer 10 en cache -128 a 127, autounboxing)
  - b == c: false (b es de cache, c es new Integer, objetos diferentes)
  - b.equals(c): true (equals compara valores)
  - a == c: true (autounboxing de c, compara primitivos)

  RESPUESTA 1.2:
  - s1 y s3 apuntan al mismo objeto en String Pool (literales)
  - s2 es un objeto nuevo en el heap
  - Se crean 2 objetos: s1/s3 (pool) y s2 (heap)
  - s1 == s2: false (diferentes objetos)
  - s1 == s3: true (mismo objeto en pool)

  RESPUESTA 2.1:
  - size: 3 (Ana se actualiza, null solo cuenta una vez)
  - get("Ana"): 26 (el ultimo valor)
  - get(null): 41

  RESPUESTA 3.1:
  - Problema: condicion de carrera en cuenta++
  - Solucion 1: synchronized en ambos metodos
  - Solucion 2: AtomicInteger
  - Solucion 3: ReentrantLock

  RESPUESTA 4.2:
  - PreparedStatement: precompilado, evita SQL injection
  - Statement: concatenacion de strings, inseguro
  - CallableStatement: para stored procedures
  - Mejor practica: PreparedStatement siempre (seguridad, rendimiento)

  RESPUESTA 6.2:
  - Mejor implementacion: Enum Singleton
    - Thread-safe por definicion
    - Serializacion segura
    - Protegido contra reflexion
    - Una sola linea de codigo

  RESPUESTA 7.3:
  a) jackson-databind 2.15.2 (primera declaracion a misma profundidad
     desde spring-boot-starter-web)
  b) hibernate-core 6.3.1.Final (ruta mas corta desde starter-data-jpa.
     Hay conflicto con 5.6.15.Final de otra-libreria pero gana la de
     Spring por ruta mas corta)
  c) Ver POM con exclusiones
  d) Ver POM con dependencyManagement

  DESAFÍO DEL ARQUITECTO
================================================================================

  🏆 CONVIERTE EL PROYECTO INTEGRADOR EN UN PRODUCTO REAL

  El proyecto integrador del examen puede convertirse en un producto
  real. Los pasos adicionales son:

  1. Mejora la UX/UI:
     - Diseno responsive
     - Temas claros/oscuros
     - Animaciones y transiciones
     - Accesibilidad (WCAG 2.1)

  2. Funcionalidades premium:
     - Notificaciones en tiempo real (WebSocket)
     - Reportes exportables (PDF, Excel)
     - Multi-idioma (i18n)
     - Integracion con pasarela de pagos real

  3. Escalabilidad:
     - Cache con Redis
     - Cola de mensajes (RabbitMQ)
     - Microservicios (dividir modulo de facturacion)
     - Base de datos replicada

  4. Produccion real:
     - Dominio personalizado con HTTPS
     - Monitoreo 24/7
     - SLA y soporte
     - Documentacion para usuarios

  💡 Publica el proyecto en GitHub, agrega un README profesional,
  y compartelo en tu portafolio de LinkedIn.

  ¡Este proyecto puede ser tu carta de presentacion al mundo laboral!
