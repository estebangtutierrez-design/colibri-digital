CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT ADD

git add <archivo> prepara cambios para commit. git add . agrega todo.

git add -p agrega interactivamente por fragmentos (hunks).

Útil para commits atómicos y limpios.



2. GIT COMMIT

git commit -m "mensaje" crea un commit. Buenas prácticas: presente imperativo,

máx 50 chars título, línea en blanco luego cuerpo de 72 chars.

Conventional Commits: feat:, fix:, docs:, refactor:, test:, chore:.



3. GIT STATUS Y SHORT STATUS

git status muestra cambios. git status -s formato compacto: ?? sin trackear,

M modificado, A agregado, D borrado. R renombrado, U sin merge.

EJEMPLO DE CÓDIGO
----------------------------------------

echo "# Mi Proyecto" > README.md
git status -s
# ?? README.md (untracked)

git add README.md
git status -s
# A  README.md (staged)

git commit -m "feat: add initial README"
git status -s
# (clean)

git log --oneline -3
