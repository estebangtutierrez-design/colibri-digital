CONTENIDO DEL CAPÍTULO
===============================================================================

1. REMOTOS

git remote add origin <url>. git remote -v lista. origin es convención.

git push -u origin main establece upstream. --set-upstream para tracking.



2. PUSH Y PULL

git push envía commits. git pull = git fetch + git merge (o rebase).

git fetch solo descarga, no fusiona. Útil para revisar cambios antes.



3. RAMAS REMOTAS

origin/main, origin/feature. git branch -r ramas remotas.

git switch -c local-branch origin/remote-branch para trackear.

git fetch --prune limpia referencias a ramas remotas eliminadas.

EJEMPLO DE CÓDIGO
----------------------------------------

git remote add origin https://github.com/usuario/proyecto.git
git push -u origin main
# Empuja y establece tracking

git fetch origin
git log origin/main --oneline -5
# Revisar cambios sin fusionar

git pull --rebase
# Traer cambios con rebase en lugar de merge

git remote -v
# Listar repositorios remotos
