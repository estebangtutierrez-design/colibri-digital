CONTENIDO DEL CAPÍTULO
===============================================================================

1. RAMAS EN GIT

git branch <nombre> crea rama. git branch lista ramas (* activa).

git switch <rama> cambia. git checkout -b <rama> crea y cambia.

Las ramas son punteros ligeros a commits. Crear ramas es instantáneo.



2. FUSIÓN (MERGE)

git merge <rama> incorpora cambios. Fast-forward si no hay divergencia.

Merge commit cuando hay historia divergente. Conflictos requieren

resolución manual. git mergetool para visualizar conflictos.



3. ELIMINAR RAMAS

git branch -d <rama> elimina rama fusionada. -D fuerza eliminación.

git push origin --delete <rama> elimina remota. Mantener ramas limpias.

EJEMPLO DE CÓDIGO
----------------------------------------

git branch feature-auth
git switch feature-auth
# Trabajar en la rama...
git add . && git commit -m "feat: add authentication"

git switch main
git merge feature-auth
# Integrar cambios

git branch -d feature-auth
# Eliminar rama fusionada

git branch -a
# Listar todas las ramas locales y remotas
