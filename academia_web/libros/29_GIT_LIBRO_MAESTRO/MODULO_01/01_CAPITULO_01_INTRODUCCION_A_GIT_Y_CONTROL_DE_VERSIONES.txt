CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES GIT?

Git es un sistema de control de versiones distribuido creado por Linus Torvalds

en 2005. Permite rastrear cambios en el código fuente, coordinar trabajo en

equipo y mantener un historial completo del proyecto.



2. DIFERENCIA CON SVN/CVS

A diferencia de los sistemas centralizados, Git mantiene una copia completa del

repositorio en cada desarrollador. Esto permite trabajar sin conexión y

proporciona redundancia total del proyecto.



3. LOS TRES ESTADOS DE GIT

Modified: el archivo cambió pero no está en staging. Staged: marcado para la

próxima confirmación. Committed: los datos están almacenados en la BD local.

El flujo básico es: working directory -> staging area -> repository.



4. SHA-1 Y OBJETOS

Cada commit se identifica por un hash SHA-1. Git almacena snapshots completos

del proyecto. Todo se verifica mediante checksum. Es imposible modificar un

archivo sin que Git lo detecte.

EJEMPLO DE CÓDIGO
----------------------------------------

git init
# Crea un nuevo repositorio en el directorio actual

git add archivo.txt
# Agrega archivo.txt al área de staging

git commit -m "Primer commit"
# Confirma los cambios con un mensaje descriptivo

git status
# Muestra el estado actual del repositorio

git log --oneline --graph --all
# Visualiza el historial de manera compacta
