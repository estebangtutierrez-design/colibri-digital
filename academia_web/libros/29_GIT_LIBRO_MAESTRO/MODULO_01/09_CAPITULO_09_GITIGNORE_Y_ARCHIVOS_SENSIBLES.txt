CONTENIDO DEL CAPÍTULO
===============================================================================

1. GITIGNORE

Archivo .gitignore en la raíz del proyecto. Patrones: *.log, node_modules/,

build/, .env, .DS_Store. Negación con !. Directorios con /.

GitHub mantiene plantillas por lenguaje/lenguaje.



2. ARCHIVOS SENSIBLES

Nunca commitear .env, claves API, certificados, passwords. Usar variables

de entorno. git-secrets y truffleHog para escanear repositorios.



3. ELIMINAR DEL HISTORIAL

git rm --cached quita de staging sin borrar. git filter-branch elimina

archivos de todo el historial. BFG Repo-Cleaner para limpieza masiva.

git forget: herramienta especializada.

EJEMPLO DE CÓDIGO
----------------------------------------

# .gitignore
node_modules/
dist/
.env
*.log
.DS_Store
coverage/
.vscode/

# Forzar archivo de configuración de ejemplo
!config.example.env

git rm --cached .env
echo ".env" >> .gitignore
git add .gitignore
git commit -m "chore: remove .env from tracking"
