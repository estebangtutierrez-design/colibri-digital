CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT LFS

Git Large File Storage: reemplaza archivos grandes con punteros.

El contenido real se almacena en servidor LFS (GitHub, GitLab).

git lfs track "*.psd" para rastrear patrones. .gitattributes config.



2. INSTALACIÓN Y USO

git lfs install una vez. git lfs track "*.zip". git lfs ls-files.

git lfs migrate para migrar archivos existentes. Pull automático.

GitHub: 2GB gratis, pago por ancho de banda adicional.



3. ALTERNATIVAS

Git LFS es el estándar. Alternativas: git-annex para almacenamiento

distribuido. Git submodules para repos separados. Storage externo

con scripts personalizados (AWS S3, Google Cloud Storage).

EJEMPLO DE CÓDIGO
----------------------------------------

git lfs install
git lfs track "*.psd"
git lfs track "*.zip"
git add .gitattributes

# Migrar archivos existentes
git lfs migrate import --include="*.psd" --everything

# Ver estado
git lfs ls-files --size

git lfs status
git lfs pull
