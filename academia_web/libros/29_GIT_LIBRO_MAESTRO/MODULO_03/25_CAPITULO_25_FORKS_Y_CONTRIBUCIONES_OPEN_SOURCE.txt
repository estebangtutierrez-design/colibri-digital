CONTENIDO DEL CAPÍTULO
===============================================================================

1. FORKS

Copia del repositorio en tu cuenta. git remote add upstream para

sincronizar. git fetch upstream y git merge upstream/main.

Ideal para contribuir sin permisos de escritura.



2. CONTRIBUCIÓN OPEN SOURCE

Fork -> Clone -> Branch -> Commit -> Push -> PR (de fork a original).

Comunicación: issues primero, discussions, contribution guidelines.

Conventional Commits, firmar CLA, seguir code style del proyecto.



3. MANTENEDORES

Evaluar PRs, dar feedback constructivo, mergear o cerrar con razón.

Reconocer contribuciones. Código de conducta. Plantillas de contribución.

Semantic release automatizado. Changelog generado.

EJEMPLO DE CÓDIGO
----------------------------------------

git clone https://github.com/tu-usuario/proyecto.git
git remote add upstream https://github.com/original/proyecto.git
git fetch upstream
git switch -c feat/mi-contribucion
git commit -m "feat: add contribution"
git push origin feat/mi-contribucion
# Crear PR en GitHub desde tu fork al repositorio original
