CONTENIDO DEL CAPÍTULO
===============================================================================

1. ISSUES

Reporte de bugs, solicitudes de features, tareas. Título descriptivo.

Labels: bug, enhancement, help wanted, good first issue. Milestones

para agrupar por versión. Asignees y project boards.



2. ISSUES TEMPLATES

Bug report: pasos para reproducir, comportamiento esperado/actual, logs.

Feature request: problema que resuelve, solución propuesta, alternativas.

Config.yml en .github/ISSUE_TEMPLATE para personalizar.



3. PROJECT BOARDS

Kanban: Todo, In Progress, Done. Automatizaciones: mover cards al

asignar, al abrir/cerrar PR. Sprint planning con milestones.

EJEMPLO DE CÓDIGO
----------------------------------------

# .github/ISSUE_TEMPLATE/bug_report.yml
name: Bug Report
description: Reportar un error
labels: [bug]
body:
  - type: textarea
    attributes:
      label: Descripción
      description: Describe el bug
  - type: textarea
    attributes:
      label: Pasos para reproducir
      placeholder: |
        1. Ir a '...'
        2. Click en '...'
  - type: textarea
    attributes:
      label: Comportamiento esperado
