CONTENIDO DEL CAPÍTULO
===============================================================================

1. BRANCH PROTECTION RULES

Configurables en GitHub/GitLab. Requerir PRs, aprobaciones, status

checks. Evitar pushes directos a main/release. Requerir código

actualizado. Incluir administradores o no.



2. STATUS CHECKS

CI debe pasar. Cobertura de código mínima. Sin conflictos con base.

Revisión requerida. Firma de commits. Linting y formato.



3. POLÍTICAS AVANZADAS

CODEOWNERS: revisores automáticos por archivo/directorio. Firmas GPG

obligatorias. Commit signing verification. Lineal history requirement.

Deploy keys y environment protection.

EJEMPLO DE CÓDIGO
----------------------------------------

# CODEOWNERS
default codeowners @team-lead
/src/auth/ @security-team
/docs/ @docs-team
*.md @docs-team

# GitHub Settings
# Settings > Branches > Add rule
# Branch: main
# ✅ Require PR
# ✅ Require approvals (2)
# ✅ Dismiss stale reviews
# ✅ Require status checks
# ✅ Require linear history
# ✅ Include administrators
