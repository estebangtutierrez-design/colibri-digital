CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT HOOKS

Scripts que se ejecutan en eventos específicos. pre-commit: antes de

commit. pre-push: antes de push. commit-msg: validar mensaje.

post-merge, post-checkout, prepare-commit-msg.



2. HERRAMIENTA PRE-COMMIT

Framework para gestionar hooks. .pre-commit-config.yaml con repos y

hooks. Ejecuta linters, formateadores, chequeos de seguridad.

Auto-fix y validación. Compatible con múltiples lenguajes.



3. HOOKS PERSONALIZADOS

.git/hooks/ en cada repo. git init --template para distribuir.

Husky (Node) gestiona hooks vía package.json. lefthook (Go).

NO poner secretos en hooks. Hooks pueden omitirse con --no-verify.

EJEMPLO DE CÓDIGO
----------------------------------------

# .pre-commit-config.yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.4.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files
  - repo: https://github.com/commitizen-tools/commitizen
    rev: v2.42.1
    hooks:
      - id: commitizen

# Instalar hooks
# pre-commit install
# pre-commit run --all-files
