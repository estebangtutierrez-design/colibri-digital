CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES GITOPS?

Operaciones de infraestructura usando Git como fuente única de verdad.

Declarativo: estado deseado en Git. Automatización: reconcilia estado

real con deseado. Pull vs push. Implementación con ArgoCD, Flux.



2. FLUJO DE GITOPS

Cambio en Git -> CI valida -> Herramienta GitOps detecta -> Sincroniza.

Rollback: git revert. Audit trail completo. Seguridad: sin acceso directo

a clúster. Developer self-service via PR.



3. ARGOCD Y FLUX

ArgoCD: UI web, SSO, multi-cluster, sync policies, health checks.

Flux: más simple, CI-friendly, compatible con Kustomize y Helm.

Ambos: auto-sync, webhooks, notifications, secret management.

EJEMPLO DE CÓDIGO
----------------------------------------

# ArgoCD Application
apiVersion: argoproj.io/v1alpha1
kind: Application
spec:
  project: default
  source:
    repoURL: https://github.com/org/infra.git
    path: k8s/production
    targetRevision: main
  destination:
    server: https://kubernetes.default.svc
    namespace: production
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

# Flux Kustomization
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
spec:
  interval: 5m
  path: ./clusters/production
  prune: true
