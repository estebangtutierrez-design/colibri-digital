CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT PARA DOCUMENTACIÓN

Docs como código. Markdown, AsciiDoc, MDX. Editado en el mismo flujo

que el código. PRs para revisión de docs. Versionado de documentación.

MkDocs, Docusaurus, GitBook. CI para build y deploy.



2. WIKIS EN GIT

GitHub Wikis: repositorio Git separado. git clone wiki-url para editar

local. Sidebar y footer personalizados. GitLab Wikis integradas.

Confluence + Git: sincronización bidireccional.



3. DIAGRAMAS COMO CÓDIGO

Mermaid: diagramas en Markdown (GitHub nativo). PlantUML, Draw.io

integración. Diagrams.net para edición visual. Structurizr para C4.

Versionados y revisables como código.

EJEMPLO DE CÓDIGO
----------------------------------------

```mermaid
graph TD
  A[Feature Branch] -->|PR| B{CI Checks}
  B -->|Pass| C[Code Review]
  C -->|Approve| D[Merge to Main]
  D -->|Auto| E[Deploy to Staging]
  E -->|Tests| F{Approved?}
  F -->|Yes| G[Deploy Production]
  F -->|No| H[Rollback]
  H --> D
```
