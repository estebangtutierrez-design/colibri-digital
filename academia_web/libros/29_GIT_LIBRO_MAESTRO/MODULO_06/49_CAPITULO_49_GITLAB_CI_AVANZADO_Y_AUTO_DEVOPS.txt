CONTENIDO DEL CAPÍTULO
===============================================================================

1. GITLAB CI AVANZADO

Child/parent pipelines. DAG (needs:). Rules y changes. Includes:

reutilizar config. Template y extends. Variables protegidas.

Manual jobs con when: manual. Environments con rollback.



2. AUTO DEVOPS

Build, test, code quality, SAST, license scanning, container scanning,

deploy a staging/production. Auto detecta lenguaje. Usa Docker y

Kubernetes. Personalizable con Auto DevOps template.



3. GITLAB PAGES Y REVIEW APPS

Review apps: deploy dinámico por MR. GitLab Pages para documentación.

GitLab Registry para contenedores. GitLab Terraform integration.

GitLab Packages para artifacts. Dependency Proxy.

EJEMPLO DE CÓDIGO
----------------------------------------

# .gitlab-ci.yml con child pipelines
stages:
  - build
  - test
  - deploy

variables:
  APP_NAME: my-app

include:
  - template: Auto-DevOps.gitlab-ci.yml

build:app:
  stage: build
  script:
    - docker build -t \$CI_REGISTRY_IMAGE .
    - docker push \$CI_REGISTRY_IMAGE

deploy:review:
  stage: deploy
  script:
    - echo "Deploying review app"
  environment:
    name: review/\$CI_COMMIT_REF_SLUG
    url: https://\$CI_COMMIT_REF_SLUG.\$APP_NAME.example.com
  only:
    - merge_requests
