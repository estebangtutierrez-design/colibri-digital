CONTENIDO DEL CAPÍTULO
===============================================================================

1. GLOBAL GIT TEMPLATES

git init --template /path/to/templates. Distribuye hooks a todos los

repos nuevos. git config init.templateDir. Centraliza políticas de

hooks: validación de commits, formatos, convenciones.



2. GITOOKS

Framework para gestionar hooks centralizados en monorepos. Hooks

versionados en el repo. .githooks/ directorio. git config core.hooksPath

.githooks. Más portable que hooks en .git/hooks.



3. EJEMPLOS CORPORATIVOS

Validar: JIRA ticket en mensaje, tamaño de diff, archivos prohibidos.

Firma GPG obligatoria. Scan de secretos. Notificar a Slack.

Integrar con sistema de ticketing. Logging centralizado.

EJEMPLO DE CÓDIGO
----------------------------------------

# .githooks/commit-msg
#!/bin/bash
# Validar formato: JIRA-123: tipo: mensaje
PATTERN="^[A-Z]+-[0-9]+: (feat|fix|docs|refactor|test|chore): .+$"

if ! grep -qE "$PATTERN" "$1"; then
  echo "Error: commit message must match JIRA-XXX: type: message"
  exit 1
fi

# hook para pre-commit
# .githooks/pre-commit
git diff --cached --name-only | while read file; do
  if [[ "$file" == *.env ]]; then
    echo "Error: .env files cannot be committed"
    exit 1
  fi
done

chmod +x .githooks/*
