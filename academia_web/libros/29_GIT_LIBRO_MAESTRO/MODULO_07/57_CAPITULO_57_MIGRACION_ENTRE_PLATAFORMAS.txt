CONTENIDO DEL CAPÍTULO
===============================================================================

1. MIGRACIÓN COMPLETA

git clone --bare old-repo.git. cd repo.git. git push --mirror new-url.

Preserva todas las ramas, tags, historial. Verificar nuevo repo.

Actualizar remotos locales. Comunicar cambios al equipo.



2. MIGRACIÓN PARCIAL

git filter-repo para migrar subdirectorio a nuevo repo. Preserva

historial de ese directorio. git subtree split para extraer.

Útil cuando se divide un monorepo en repos individuales.



3. CONVERSIÓN DE PLATAFORMA

GitHub -> GitLab: mirror push o API. Bitbucket -> GitHub: importer.

Azure -> GitHub: GitHub Importer. SVN -> Git: git svn clone.

Preservar: issues, PRs, wikis, webhooks. Herramientas de terceros.

EJEMPLO DE CÓDIGO
----------------------------------------

git clone --bare https://github.com/org/repo.git
cd repo.git
git push --mirror https://gitlab.com/org/repo.git

# Extraer subdirectorio
git filter-repo --path src/library --path tests/library

# Actualizar remotos locales
git remote set-url origin <new-url>
git fetch --all
git branch -a
