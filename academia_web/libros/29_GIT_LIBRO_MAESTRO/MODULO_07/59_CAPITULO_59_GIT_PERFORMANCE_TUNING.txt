CONTENIDO DEL CAPÍTULO
===============================================================================

1. OPTIMIZACIÓN DE REPOS

git config core.preloadindex true. git config core.fscache true.

git config gc.auto 256. --depth para shallow clones. Separar

repos grandes: dividir en submódulos. Usar alternates para objetos.



2. GIT CONFIG PARA RENDIMIENTO

core.deltaBaseCacheLimit (96m-512m). core.bigFileThreshold (512m).

pack.threads = CPU count. pack.windowMemory (1g). pack.deltaCacheSize.

fetch.writeCommitGraph para generación de gráficos.



3. BENCHMARKING

git version --build-options. hyperfine para medir tiempos. Profiling:

GIT_TRACE=1, GIT_TRACE_PERFORMANCE=1. time git command. Medir antes/después.

EJEMPLO DE CÓDIGO
----------------------------------------

git config core.preloadindex true
git config core.fscache true
git config fetch.writeCommitGraph true

# Medir tiempos
time git status

GIT_TRACE=1 git fetch
GIT_TRACE_PERFORMANCE=1 git clone <url>

# Build-options para ver optimizaciones
git version --build-options

# Generar commit-graph
git commit-graph write --reachable
