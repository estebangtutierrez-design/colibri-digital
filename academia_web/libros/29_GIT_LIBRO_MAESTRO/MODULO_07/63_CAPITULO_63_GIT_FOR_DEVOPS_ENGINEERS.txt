CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT EN CI/CD

Pipeline triggers: push, tag, PR. Checkout strategies: depth, lfs,

submodules. Fetch tags. Matrix builds con versiones. Git status

en jobs. Version tagging automático con semantic-release.



2. GIT PARA INFRASTRUCTURE AS CODE

Terraform + Git: estado, módulos, providers. Ansible playbooks

versionados. Helm charts. Kubernetes manifests. GitOps con Flux/Argo.

Policy as Code: OPA, Kyverno. Security as Code: Trivy, Checkov.



3. GIT EN MICROSERVICIOS

Repos por servicio vs monorepo. Shared libraries versionadas.

API contracts en Git. CI/CD multi-servicio. Dependencia entre

servicios: git submodules, subtrees, package registry.

EJEMPLO DE CÓDIGO
----------------------------------------

# Pipeline de infraestructura
git tag v1.0.0-infra
git push origin v1.0.0-infra

# Checkout con submódulos en pipeline
git submodule update --init --recursive

# Version tag automático
# semantic-release analiza commits y genera versión

# Terraform con Git
terraform init
terraform workspace new production
git add -A
git commit -m "infra: update production config"
