1. ESTRATEGIA DE BACKUP

git clone --mirror como backup. Automatizar con cron/scheduler.

Múltiples ubicaciones: local, cloud, offsite. Frecuencia según

actividad. Política de retención. Probar restauración periódicamente.



2. BUNDLE PARA BACKUP

git bundle create backup-$(date +%Y%m%d).bundle --all. Archivo único

portátil. git bundle verify check integrity. Restore: git clone

backup.bundle. Ideal para backup offline o transferencia manual.



3. DISASTER RECOVERY PLAN

Documentar procedimientos. Tiempo objetivo de recuperación (RTO).

Punto objetivo de recuperación (RPO). Roles y responsabilidades.

Pruebas periódicas. Checklist de DR. Notificar al equipo.

EJEMPLO DE CÓDIGO
----------------------------------------

#!/bin/bash
# Backup script
REPO_DIR="/var/git/repos"
BACKUP_DIR="/backup/git"
DATE=$(date +%Y%m%d_%H%M)

for repo in "$REPO_DIR"/*/; do
  name=$(basename "$repo")
  git bundle create "$BACKUP_DIR/$name-$DATE.bundle" --all
  echo "Backed up: $name"
done
