CONTENIDO DEL CAPÍTULO

1. POLÍTICAS DE CUMPLIMIENTO

SOX, HIPAA, GDPR, PCI-DSS requieren: quién cambió qué, cuándo y por qué.

GPG signing obligatorio. Commit messages con referencias a tickets.

Aprobación mínima de 2 reviewers. Branch protection obligatorio.



2. AUDIT LOG

Git log completo es audit trail. git log --format="%H,%an,%ae,%ad,%s".

Exportar a SIEM. GitHub Audit Log: eventos de administración. GitLab

Audit Events. Retención según política corporativa.



3. REPORTES Y DASHBOARDS

Herramientas: GitStats, GitInspector, Gitential. Métricas: commits/

semana, autores activos, líneas cambiadas, tiempo entre commits.

Dashboard de cumplimiento: signature coverage, review coverage.

EJEMPLO DE CÓDIGO
----------------------------------------

# Export audit log
git log --format="%H|%an|%ae|%ai|%s" --since="2024-01-01" > audit.csv

# Verificar firmas en todos los commits
git log --show-signature --format="%H %GF" | grep -c "Good "

# Verificar cobertura de review
git log --merges --first-parent --format="%H,%an"

# Script de compliance
for commit in $(git rev-list --all); do
  if ! git verify-commit $commit 2>/dev/null; then
    echo "Unsigned commit: $commit"
  fi
done
