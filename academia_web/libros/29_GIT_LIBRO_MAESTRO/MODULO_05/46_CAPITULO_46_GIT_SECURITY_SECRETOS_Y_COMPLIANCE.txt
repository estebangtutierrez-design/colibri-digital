CONTENIDO DEL CAPÍTULO
===============================================================================

1. DETECCIÓN DE SECRETOS

git-secrets: hooks que escanean commits. truffleHog: escanea historial.

Detectan API keys, tokens, passwords. Integrados en CI para bloquear

pushes con secretos. Github secret scanning automático.



2. CUMPLIMIENTO Y AUDITORÍA

Firma obligatoria de commits (GPG/SSH). Logs de acceso al repo.

Branch protection enforce. Audit log en GitHub/GitLab. Políticas de

retención de repositorios. Backup y disaster recovery.



3. REPOSITORIOS PRIVADOS

GitHub Advanced Security para enterprise. Dependabot alerts. CodeQL

scanning. Secret scanning para partners. IP allow lists. SAML SSO.

2FA obligatorio. Políticas de creación de repositorios.

EJEMPLO DE CÓDIGO
----------------------------------------

# .gitattributes para archivos sensibles
*.p12 filter=git-crypt diff=git-crypt
*.key filter=git-crypt diff=git-crypt

# git-secrets hooks
git secrets --register-aws
git secrets --add 'api[_-]?key'
git secrets --add 'secret[_-]?token'
git secrets --scan
git secrets --scan-history

# .env.template (nunca .env real)
DB_HOST=localhost
DB_USER=usuario
