CONTENIDO DEL CAPÍTULO
===============================================================================

1. SEMANTIC VERSIONING

MAJOR.MINOR.PATCH (2.5.1). MAJOR: breaking changes. MINOR: new features

backward-compatible. PATCH: bug fixes. Pre-release: 2.0.0-beta.1.

Build metadata: 2.0.0+build.20240115. semver.org especificación.



2. CONVENTIONAL COMMITS

feat: -> MINOR. fix: -> PATCH. BREAKING CHANGE en body -> MAJOR.

Estándar: feat, fix, docs, style, refactor, perf, test, build, ci,

chore, revert. Automatic version bump con semantic-release.



3. CHANGELOG AUTOMATIZADO

Herramientas: standard-version, semantic-release, release-please.

Generan CHANGELOG.md desde conventional commits. Git tags automáticos.

Notas de release para GitHub Releases. Consistencia en todo el equipo.

EJEMPLO DE CÓDIGO
----------------------------------------

# Conventional Commits
feat: add user profile page
fix: correct login validation
feat: implement search (BREAKING CHANGE)
docs: update API reference
refactor: extract form component

# Semantic-release config
# .releaserc.json
{
  "branches": ["main"],
  "plugins": [
    "@semantic-release/commit-analyzer",
    "@semantic-release/release-notes-generator",
    "@semantic-release/changelog",
    "@semantic-release/github"
  ]
}
