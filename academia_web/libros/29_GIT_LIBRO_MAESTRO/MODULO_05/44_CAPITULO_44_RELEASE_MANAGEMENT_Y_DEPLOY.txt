CONTENIDO DEL CAPÍTULO
===============================================================================

1. ESTRATEGIAS DE RELEASE

Release branches: estabilización antes de main. Canary releases: % de

tráfico. Blue-green deployment: dos entornos completos. Rolling update:

gradual. Feature flags para activar/desactivar en producción.



2. GIT TAGS Y RELEASES

Release tags: git tag -a v1.2.3. CI genera notas de release automáticas.

GitHub Releases con assets binarios. Changelog generado automáticamente.

Firmar releases con GPG. Checksums para verificar integridad.



3. AUTOMATIZACIÓN

GitHub Actions / GitLab CI detectan push a release/*. Build, test,

deploy automático. Rollback: git revert release tag o deploy anterior.

Mantener artefactos de release. Política de retención.

EJEMPLO DE CÓDIGO
----------------------------------------

name: Release
on:
  push:
    branches: [main]

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm ci && npm test && npm run build
      - uses: cycjimmy/semantic-release-action@v3
        env:
          GITHUB_TOKEN: \${{ secrets.GITHUB_TOKEN }}

# Rollback
git revert HEAD
git push origin main
git tag -a v1.2.3-rollback -m "Rollback to v1.2.2"
