CONTENIDO DEL CAPÍTULO
===============================================================================

1. GITHUB FLOW

Rama única principal (main). Feature branches cortas. Pull requests

obligatorios. Deploy continuo desde main. Ideal para CI/CD.

Mínima sobrecarga de procesos. Ampliamente adoptado en SaaS.



2. TRUNK-BASED DEVELOPMENT

Commits directos a main (o rama troncal). Feature flags para ocultar

código incompleto. Integraciones múltiples al día. Equipos maduros.

Requiere: buena suite de tests, feature flags, despliegue rápido.



3. COMPARATIVA

Git Flow: estructura, control, releases calendarizados. GitHub Flow:

simplicidad, velocidad, deploy continuo. Trunk-based: máximo flujo,

equipos pequeños, madurez técnica alta. Elegir según contexto.

EJEMPLO DE CÓDIGO
----------------------------------------

# GitHub Flow
git switch -b feat/payment-integration
# Implementar...
git push origin feat/payment-integration
# PR -> Review -> Merge to main

# Trunk-based
git switch main
git pull
# Cambio pequeño con feature flag
if (featureFlags.newPayment) {
  // Nuevo código
}
git commit -m "Add payment gateway (flag: newPayment)"
