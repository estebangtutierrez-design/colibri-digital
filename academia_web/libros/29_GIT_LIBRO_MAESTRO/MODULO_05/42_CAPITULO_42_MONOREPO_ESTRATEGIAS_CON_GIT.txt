CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES UN MONOREPO?

Repositorio único con múltiples proyectos. Código compartido, builds

unificados, versionado atómico. Conveniente pero complejo de escalar.

Google, Microsoft, Uber usan monorepos. Herramientas: Bazel, Nx, Turborepo.



2. GIT EN MONOREPOS

git subtree para dependencias compartidas. git submodules para equipos

independientes. Git sparse checkout para clonar partes específicas.

git partial clone: clonar solo archivos necesarios.



3. MEJORES PRÁCTICAS

CI/CD selectivo: solo afectados. CODEOWNERS por directorio. Políticas

de permisos granulares. Herramientas: Nx affected, Lerna, changesets.

Documentar estructura. Evitar dependencias circulares.

EJEMPLO DE CÓDIGO
----------------------------------------

git clone --filter=blob:none <url>
# Partial clone: evita descargar blobs

git sparse-checkout init --cone
git sparse-checkout set packages/frontend packages/shared
# Solo checkout directorios específicos

git clone --depth 1 <url>
# Shallow clone para CI

git worktree add ../api-pkg packages/api
# Worktree para un paquete específico
