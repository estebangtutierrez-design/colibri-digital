CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES CHERRY-PICK?

Aplica un commit específico de otra rama a la actual. Útil para

hotfixes o cuando necesitas un cambio sin toda la rama. Crea un nuevo

commit con cambios idénticos pero hash diferente.



2. CUÁNDO USAR

Hotfix a producción desde develop. Seleccionar características

específicas. Portar parches entre versiones. NO para integración normal.



3. PRÁCTICAS SEGURAS

Siempre incluir referencia al commit original en el mensaje. git cherry

para ver commits que faltan. cherry-pick -n para aplicar sin commitear.

Resolver conflictos igual que en merge.

EJEMPLO DE CÓDIGO
----------------------------------------

git switch main
git cherry-pick abc123
# Aplica commit abc123 en main

git cherry-pick -n xyz789
# Aplica cambios sin hacer commit

git cherry-pick --continue
# Continuar después de conflicto

git cherry-pick --abort
# Abortar operación

git log --oneline main..feature/git cherry-pick --signoff abc123
