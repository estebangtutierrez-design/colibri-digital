CONTENIDO DEL CAPÍTULO
===============================================================================

1. FAST-FORWARD MERGE

Ocurre cuando la rama base no tuvo nuevos commits. Git simplemente

mueve el puntero. Historial lineal. git merge --ff-only para forzar.



2. THREE-WAY MERGE

Cuando ambas ramas avanzaron. Git crea merge commit con dos padres.

Registra explícitamente la fusión. git merge --no-ff fuerza merge commit.



3. ESTRATEGIA DE FUSIÓN

--squash: fusiona como un solo commit. --no-commit: revisar antes.

Merge strategy: recursive, ours, theirs. Estrategia subtree para

dependencias. Usar --no-ff para feature branches en GitHub Flow.

EJEMPLO DE CÓDIGO
----------------------------------------

git switch main
git merge feature
# Fast-forward si es posible

git merge --no-ff feature
# Fuerza merge commit aunque sea fast-forward

git log --oneline --graph
# Visualizar el merge o fast-forward

git merge --squash feature
git commit -m "feat: add all feature changes as one"
