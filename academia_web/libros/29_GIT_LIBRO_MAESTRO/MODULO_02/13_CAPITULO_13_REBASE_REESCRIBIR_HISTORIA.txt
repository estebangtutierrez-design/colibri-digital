CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES REBASE?

git rebase main reaplica los commits de la rama actual sobre main.

Historia lineal sin merge commits. Útil antes de integrar cambios.

Reescribe la historia: NO USAR en ramas compartidas.



2. REBASE INTERACTIVO

git rebase -i HEAD~3. Acciones: pick, reword, edit, squash, fixup,

drop. Reordenar, combinar o eliminar commits. Limpiar historia local.



3. REBASE VS MERGE

Merge preserva historia exacta (incluyendo paralelismo). Rebase crea

historia lineal. Git rebase produce historia más limpia. Regla de oro:

merge público, rebase local.

EJEMPLO DE CÓDIGO
----------------------------------------

git switch feature
git rebase main
# Reaplica feature sobre main

# Rebase interactivo
git rebase -i HEAD~3
# pick abc123 feat: add login
# squash def456 fix: typo in login
# pick ghi789 feat: add dashboard

git rebase --continue
# Continuar después de resolver conflictos

git rebase --abort
# Deshacer el rebase en caso de problemas
