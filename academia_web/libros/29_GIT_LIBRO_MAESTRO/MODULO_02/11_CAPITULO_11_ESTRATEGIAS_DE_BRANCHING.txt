CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT FLOW

Ramas: main, develop, feature/, release/, hotfix/. Flujo completo con

merge dedicados. Ideal para proyectos con versiones semánticas.

Cada release se etiqueta y tiene mantenimiento separado.



2. GITHUB FLOW

Simplicidad: main + feature branches. Pull requests para integrar.

Despliegue continuo desde main. Ideal para CI/CD. Equipos ágiles.

Menos sobrecarga que Git Flow.



3. TRUNK-BASED DEVELOPMENT

Rama única (trunk/main). Commits pequeños y frecuentes. Feature flags

para ocultar código incompleto. Integración continua constante.

Ideal para equipos maduros y despliegue continuo.

EJEMPLO DE CÓDIGO
----------------------------------------

# Git Flow
git flow init
git flow feature start nueva-funcionalidad
git flow feature finish nueva-funcionalidad
git flow release start v1.0.0
git flow hotfix start bug-critico

# GitHub Flow
git switch -c feat/nueva-funcionalidad
git commit -m "Implement new feature"
git push origin feat/nueva-funcionalidad
# Crear Pull Request en GitHub

# Trunk-based
git switch main
git pull
git commit -m "Add small increment" -m "Feature flagged behind NEW_FEATURE"
