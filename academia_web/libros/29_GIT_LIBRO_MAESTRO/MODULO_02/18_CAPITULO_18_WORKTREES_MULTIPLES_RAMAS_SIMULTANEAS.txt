CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ SON LOS WORKTREES?

Git worktree permite tener múltiples ramas checkout en diferentes

directorios simultáneamente. git worktree add <path> <branch>.

Comparten el repositorio (objetos, refs) pero tienen working tree

independiente. Ideal para context switching.



2. CUÁNDO USAR WORKTREES

Hotfix urgente mientras desarrollas. Revisar PRs sin cambiar contexto.

Ejecutar tests en diferentes ramas. Comparar comportamiento.

Mantener documentación separada del desarrollo principal.



3. GESTIÓN

git worktree list. git worktree prune. worktree remove. Mover o

eliminar con cuidado: los worktrees refs persisten. Prune limpia.

Límite práctico: 3-4 worktrees simultáneos.

EJEMPLO DE CÓDIGO
----------------------------------------

git worktree add ../hotfix-fix hotfix/critical-bug
# Working tree separado para hotfix

git worktree add ../docs-preview main --no-checkout
# Sin checkout para documentos

git worktree list
# /repo             [main]
# /repo/../hotfix   [hotfix]

git worktree remove hotfix
git worktree prune
