CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT FILTER-BRANCH

Reescribe historial aplicando filtros. Eliminar archivos, modificar

mensajes, cambiar autor. Peligroso: reescribe TODOS los commits.

Lento en repos grandes. Uso cuidadoso y planificado.



2. BFG REPO-CLEANER

Alternativa rápida a filter-branch. bfg --delete-files env.txt.

bfg --replace-text passwords.txt. Mucho más rápido en repos grandes.

Mantiene los tags. Compatible con filter-branch.



3. MEJORES PRÁCTICAS

Siempre backup antes. Comunicar al equipo: todos deben re-clonar.

git push --force-with-lease (no --force). git gc --aggressive --prune=now

después. Limpiar reflog y backups después de verificar.

EJEMPLO DE CÓDIGO
----------------------------------------

git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch .env' \
  --prune-empty --tag-name-filter cat -- --all

# BFG (más rápido)
java -jar bfg.jar --delete-files "*.log" .
java -jar bfg.jar --replace-text passwords.txt .

git push --force-with-lease --all
git push --force-with-lease --tags
