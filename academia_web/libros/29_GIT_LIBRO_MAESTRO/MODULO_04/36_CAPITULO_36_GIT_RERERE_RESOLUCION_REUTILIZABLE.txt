CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES RERERE?

Reuse Recorded Resolution. Git recuerda cómo resolviste conflictos.

Reaplica automáticamente en conflictos similares. Activar con

git config rerere.enabled true. Útil en rebases largos.



2. FUNCIONAMIENTO

Registra estado pre-conflicto y resolución. En conflictos futuros

con el mismo texto, aplica la solución guardada. git rerere diff

para ver resoluciones. git rerere status estado actual.



3. CUÁNDO USAR

Rebases que reintroducen conflictos. Cherry-picks repetitivos.

Merges entre ramas activas. Merge de rama feature a main y viceversa.

No confiar ciegamente: revisar siempre la resolución automática.

EJEMPLO DE CÓDIGO
----------------------------------------

git config --global rerere.enabled true

# Durante un rebase con conflictos
git rerere status
git rerere diff
# Ver qué resoluciones se reutilizarán

git rerere forget <path>
# Olvidar resolución guardada para archivo

git rerere clear
# Limpiar historial de resoluciones
