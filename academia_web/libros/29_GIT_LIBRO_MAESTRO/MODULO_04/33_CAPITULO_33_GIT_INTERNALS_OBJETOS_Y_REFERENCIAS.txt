CONTENIDO DEL CAPÍTULO
===============================================================================

1. OBJETOS GIT

Blob: contenido de archivo. Tree: directorio (nombres, modos, SHAs).

Commit: snapshot + metadata (autor, fecha, padres, mensaje).

Tag anotado: tag + objeto. Almacenados en .git/objects/.



2. REFERENCIAS

.git/refs/heads/ (ramas), .git/refs/tags/ (tags), .git/HEAD.

git rev-parse HEAD para resolver a SHA. git cat-file -p <sha>

para ver contenido del objeto. git ls-tree para trees.



3. PACK FILES

Git comprime objetos en .pack para eficiencia. git gc empaqueta.

git verify-pack verifica integridad. git count-objects estadísticas.

Auto-gc se ejecuta periódicamente. gc.auto configura umbral.

EJEMPLO DE CÓDIGO
----------------------------------------

echo "hello" | git hash-object --stdin -w
# Crea blob y devuelve SHA

git cat-file -p HEAD
git cat-file -t HEAD

git rev-parse main
git ls-tree HEAD

git count-objects -vH
# Estadísticas de almacenamiento

git gc --aggressive --prune=now
