CONTENIDO DEL CAPÍTULO
===============================================================================

1. ¿QUÉ ES EL REFLOG?

Registro local de todos los movimientos de HEAD. git reflog muestra.

Incluye resets, rebases, merges. Git mantiene reflog por 90 días.

El reflog es local: no se comparte con push/pull.



2. RECUPERAR COMMITS PERDIDOS

git reflog muestra referencias. git checkout HEAD@{n} o reset a

un reflog. Útil después de reset --hard, rebase fallido, o commits

huérfanos. El commit sigue existiendo si está en reflog.



3. REBASE RECOVERY

Después de rebase, git reflog feature para ver antes del rebase.

git checkout -b recovered-feature HEAD@{10}. Cherry-pick commits

necesarios. No depender del reflog para recuperación rutinaria.

EJEMPLO DE CÓDIGO
----------------------------------------

git reflog
# abc123 HEAD@{0}: commit: feat: add login
# def456 HEAD@{1}: rebase: finished
# ghi789 HEAD@{2}: rebase: checkout main
# old789 HEAD@{3}: commit: WIP

git reset --hard HEAD@{3}
# Recuperar estado anterior

git reflog feature --date=iso
# Reflog de rama específica con fechas
