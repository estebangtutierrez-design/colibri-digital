CONTENIDO DEL CAPÍTULO
===============================================================================

1. FIRMA DIGITAL CON GPG

git commit -S firma con GPG. Verificar con git log --show-signature.

Verificación automática si commit.gpgsign = true. Requiere clave GPG

asociada a cuenta GitHub/GitLab.



2. CONFIGURACIÓN GPG

gpg --gen-key o usar claves existentes. git config user.signingkey.

Agregar clave pública a GitHub. git log --show-signature verifica.

GitHub muestra badge Verified en commits firmados.



3. SSH SIGNATURES

Alternativa moderna a GPG. git config gpg.format ssh. git log

--show-signature funciona igual. Claves SSH ya configuradas.

Más simple que GPG. Adoptado por GitHub recientemente.

EJEMPLO DE CÓDIGO
----------------------------------------

gpg --full-generate-key
gpg --list-secret-keys --keyid-format LONG
git config user.signingkey ABCDEF1234567890

git commit -S -m "feat: signed commit"
git log --show-signature -1

git config commit.gpgsign true
# Todos los commits se firmarán automáticamente

# SSH signatures
git config gpg.format ssh
git config user.signingkey ~/.ssh/id_ed25519.pub
