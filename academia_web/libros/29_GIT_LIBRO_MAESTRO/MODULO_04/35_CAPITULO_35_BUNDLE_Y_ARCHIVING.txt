CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT BUNDLE

Empaqueta objetos Git en un archivo. git bundle create repo.bundle HEAD

main. Transferir sin conexión de red. backup portátil del repo.

git bundle verify y git clone repo.bundle para restaurar.



2. GIT ARCHIVE

Crea archivo tar/zip del código sin historial. git archive -o proyecto.zip

HEAD. Útil para distribuir releases. --prefix para prefijo de directorio.

Omitir .gitignore o .gitattributes con --no-<pattern>.



3. CASOS DE USO

Backups offline de repositorios. Transferencia por USB/air-gapped.

Distribuir código fuente sin historial. Archivos de release.

Migración entre servidores sin conexión directa.

EJEMPLO DE CÓDIGO
----------------------------------------

git bundle create backup.bundle --all
# Back up completo del repo

git bundle create increment.bundle HEAD~5..HEAD
# Últimos 5 commits

git clone backup.bundle repo-restaurado/

git archive -o proyecto-v1.0.zip HEAD
# Código sin historial

git archive -o proyecto-v1.0.tar.gz --prefix=proyecto/ main
