CONTENIDO DEL CAPÍTULO
===============================================================================

1. GIT NOTES

Añadir metadatos a commits sin modificar el commit original.

git notes add -m "Nota: revisión aprobada". Almacenadas en refs/notes/.

git log --show-notes. Compartir con git push --ref=refs/notes/*.



2. USOS PRÁCTICOS

Añadir resultados de code review. Información de CI. Referencias a

tickets. Anotaciones de auditoría. Estado de aprobación.

Metadatos de compliance. Fechas de deploy.



3. LIMITACIONES

No reemplazan commit messages. No se transfieren en clone normal.

Requiere push explícito. No integrado en GitHub/GitLab UI.

Alternativas modernas: pull request comments, issue references.

EJEMPLO DE CÓDIGO
----------------------------------------

git notes add -m "Reviewed by: equipo-seguridad"
git notes add -m "Approved: 2024-01-15"
git log --show-notes

git notes append -m "Deployed to production: v2.3.1"

git push origin refs/notes/*
# Compartir notas con el equipo

git notes list
# Listar todos los commits con notas
