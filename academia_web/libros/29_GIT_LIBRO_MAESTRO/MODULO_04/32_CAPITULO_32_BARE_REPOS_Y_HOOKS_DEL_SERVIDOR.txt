CONTENIDO DEL CAPÍTULO
===============================================================================

1. BARE REPOSITORIES

Repos sin working directory: git init --bare. Solo contienen objetos

Git y refs. Usados en servidores (GitHub, GitLab). El estándar para

repositorios centralizados. Directorio nombrado repo.git.



2. HOOKS DEL LADO DEL SERVIDOR

pre-receive, update, post-receive en el bare repo. Validar antes de

aceptar push. CI triggers. Notificaciones. Actualizar deploy.

Implementar políticas: validar formato commit, firmas GPG.



3. UPDATE HOOK

Recibe: ref name, old SHA, new SHA. Puede rechazar push. Validar por

rama: main requiere PR, releases permiten tags, features reglas.

Ejemplo: evitar pushes directos a main. Forzar squash merges.

EJEMPLO DE CÓDIGO
----------------------------------------

git init --bare proyecto.git

# hooks/pre-receive
#!/bin/bash
while read oldrev newrev refname
do
  if [ "$refname" = "refs/heads/main" ]; then
    echo "Direct push to main rejected"
    exit 1
  fi
done

chmod +x hooks/pre-receive
